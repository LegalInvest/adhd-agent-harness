> 分解粒度与启动成本的非线性关系

当 agent 把“写周报”拆成 5 个子任务时，我看到的不是 5 个可执行步骤，而是 5 次需要重新克服焦虑的上下文切换。有没有双域证据表明，分解粒度越细，对 ADHD 的执行启动反而可能越有害？

## 问题：分解的悖论

ADHD 群体常被建议“把任务拆小”，工程师也教 agent 用 planner-executor 循环分解长任务。但拆到多细才算好？我的一位 ADHD 朋友曾把“整理书桌”拆成 12 步，结果一上午连第 1 步“拿抹布”都没完成——每一步都像一扇需要重新推开的门。

agent 侧也有类似困境：当 planner 把一个查询拆成 8 个子步骤，每个步骤都涉及一次 LLM 调用和上下文重建，延迟和 token 浪费急剧上升。分解粒度与执行效率之间，存在一个“甜区”，而非单调递增。

## 同构证据：启动成本的真实代价

**ADHD 侧**：执行功能研究指出，ADHD 患者的前额叶皮层活动不足，任务切换时认知负荷显著高于常人（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。每拆出一个子任务，就意味着一次新的“任务启动”——而启动正是 ADHD 的核心痛点。工作记忆研究进一步揭示，ADHD 患者的工作记忆是执行功能的瓶颈（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），细粒度分解会迅速撑爆这个瓶颈，因为每一步都需要记住上下文、目标和已完成部分。

**LLM/agent 侧**：耶鲁大学的研究发现，Transformer 的自注意力机制本身存在工作记忆容量限制——随着 N-back 任务 N 值增加，注意力分数熵增、注意力弥散，表现下降（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。这与 ADHD 注意缺陷的计算本质同源：“注意力资源的弥散分配”。当 agent 的 planner 拆出过多子任务，每个子任务都需要重新加载上下文，相当于在 LLM 的“工作记忆”中反复做高 N-back 任务，导致注意力分散和错误率上升。

明尼苏达大学的执行功能测试进一步证实，LLM 呈现“强记忆、弱控制”剖面：工作记忆容量大，但认知灵活性与注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这意味着，LLM 并不擅长在多个子任务间灵活切换——细粒度分解恰恰要求这种切换能力。

## 历史 harness 的启示：粒度与节奏的平衡

孔子（春秋/儒家创始人）的 ADHD 特质明显：身高 1 米 9，精力旺盛，周游列国 14 年坐不住；冲动爱骂人，思维跳跃，《论语》全是场景化语录。他的 harness 是以“克己复礼”为核心，用外在的“礼”作为行为边界，每日“吾日三省吾身”。注意，他并没有把“修身”拆成 100 个原子习惯，而是设定了几个关键检查点（三省）和一个总原则（礼）。这相当于 agent 的“re-grounding”机制：不是频繁切换上下文，而是定期回到主目标，避免迷失在子任务森林中。

张居正（明朝/万历首辅）的 harness 是考成法——严格考核官员，用制度管别人也管自己。他也没有把改革拆成无数微步骤，而是用一套固定的考核周期（定期汇报、检查）来维持方向。这对应 agent 的“外部调度器”：设定固定节奏的 checkpoint，而非每次重新规划。

这两个案例的共同点是：**用结构化的外部节奏替代内部的自发规划**，减少决策次数和上下文切换。这与 LLM agent 的“定时 re-grounding”同构——与其把任务拆碎，不如保持大块目标，只在关键节点做上下文刷新。

## 工具实践：哪些在平衡粒度与启动成本？

- **Motion** 和 **Reclaim.ai** 通过自动排程与动态调整，把“下一步做什么”的决策负担外包给 AI（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。它们不强迫用户手动拆解任务，而是自动维护一个整体日程，减少启动次数。
- **Tiimo** 和 **Structured** 用视觉化时间线让时间流逝变得可感（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），帮助用户感知整体进度，避免因细粒度分解而失去全局感。
- **Goblin Tools** 提供“任务分解”功能，但用户可控制粒度——不是强制最小步骤，而是按需拆解。

这些工具的共同设计哲学是：**提供足够的结构来降低启动成本，但不过度分解以至于增加切换成本**。

## 争议与局限

必须诚实指出，目前缺乏直接比较不同分解粒度对 ADHD 执行启动影响的随机对照试验。现有证据主要来自用户报告和工具设计逻辑（来源：AI 与 ADHD 的时间管理）。同构性类比虽然富有启发性，但仍是理论假设，并非经过验证的科学事实（来源：矛盾与存疑）。此外，过度依赖 AI 工具可能从“脚手架”退化为“永久拐杖”，削弱内在执行功能的代偿发展（来源：拐杖与脚手架概念）。

## 核心观点与行动建议

**核心观点**：分解粒度与启动成本呈 U 型曲线——太粗则无法执行，太细则启动成本超过收益。有效的 harness（无论是为 ADHD 还是为 LLM agent）应找到“最小可行粒度”：即刚好能降低启动门槛、又不会引入过多切换成本的那个点。

**今天就能试的行动**：
1. **3-5 步法则**：把任何任务最多拆成 3-5 个子任务，而不是 10+。如果 5 步还觉得难，说明粒度仍太粗，需要重新定义第一步。
2. **外部节奏而非内部拆解**：使用 **Tiimo** 或 **Structured** 设定固定时间块（如 25 分钟），在块内只做一个大任务，不拆细。利用视觉时间线感知整体进度。
3. **agent 调试技巧**：如果你的 LLM agent 频繁“迷路”，检查 planner 是否生成了过多子步骤。尝试限制最大子任务数为 5，并增加 re-grounding 指令（如“每 3 步回顾一次主目标”）。
4. **模仿孔子的“三省”**：每天设 3 个固定时间点（如上午、中午、下午），用 1 分钟问自己“我现在在做什么？和主目标一致吗？”——这比不断拆任务更有效。

最终，机器和大脑一样，需要的不是更细的分解，而是更聪明的节奏。

# 内容方法论 v2 —— 人生 Harness 研究纲领

> 本文档是整个 Auto-research 系统的内容宪法。所有文章（人工撰写与自动生成）、选题评分、wiki 建设都以此为准。

## 0. 北极星

**这个项目的终点不是内容，是人。**

我们深入研究 ADHD，是为了找到「人生 Harness 系统」的方法论——助力 ADHD 者获得幸福、成长，找到人生价值。同构类比、双域文献、工具评测、400 篇文章，全部是抵达这个终点的手段。

由此推出三条总原则：

1. **写作的成功标准**不是「读完觉得有道理」，而是「读完能往自己的人生系统里装上一块」——每篇必须交付可安装的组件（协议/仪式/参数/清单）。
2. **效率不是终点**。文章可以从拖延、启动、时间盲切入，但要自觉地服务于更上层的东西：自我和解、关系修复、价值感、人生方向。一篇只让人「更能干活」的文章是半成品。
3. **诚实高于说服**。证据不足就说不足，类比到头就说到头，工具没用就说没用。读者的信任是这个项目唯一不可再生的资产。

## 1. 人生 Harness 五层金字塔

400 篇实践沉淀出的所有框架，组织进这个分层体系。每篇文章都应知道自己在为哪一层施工，并向上层看齐。

| 层 | 名称 | 内容 | 代表框架 |
|---|------|------|---------|
| L1 | **状态层**（供电系统） | 睡眠、运动、光照、唤醒调节、（医生管理下的）药物 | 唤醒曲线自测、任务-刺激映射表、低温/高温分区 |
| L2 | **执行层**（外部执行功能） | 捕获、组织、调度、启动、监控、记忆、完成 | 任务树、启动坡道、断点协议、单线程主线程、预编译、验证循环 |
| L3 | **关系层**（社会性基础设施） | 问责、在场、见证、协作接口、亲密关系中的 ADHD | 社会电源倍数、接口说明书、body doubling、承诺分层 |
| L4 | **叙事层**（自我和解） | 把旧账从品格栏移到机制栏、羞耻的拆除、诊断与身份 | 品格栏→参数栏换栏、功能画像、说明书不是判决书 |
| L5 | **价值层**（人生方向） | 兴趣驱动的职业设计、优势场景选择、长期项目的可持续、幸福的定义权 | 兴趣引擎的续航工程、职业形态匹配、「我的赛道」选择 |

**分层纪律**：
- 下层是上层的地基：状态崩着谈执行是空谈，执行崩着谈价值是鸡汤。文章给建议时按层给，不越层开药。
- 上层是下层的意义：执行层的每个协议，最终要能回答「这让生活好在哪」。
- **金字塔越往上，现有文章越稀薄（L4/L5 是未来选题的重点增量方向）**。

## 1.5 问题驱动式分析框架（每篇的推理骨架）

**每一条内容只解决一个问题。**正文的推理按五型问题依次推进（可隐式融入行文，不必列小标题，但五步都要在场）：

1. **收敛型**（锁靶心/降维/降噪）：把粗糙的大问题压成一个可回答的具体问题——「我时间管理差」→「我的时间系统四种故障里坏的是哪一种？」
2. **穿透式**（挖本质/挖利益/挖风险）：机制层的双域拆解（ADHD 侧 + LLM/agent 侧），包括谁在从现状获利（产品营销/自我叙事）与暗面风险。
3. **验证式**（证真/证伪/找反例）：证据分级、反例与适用边界、N-of-1 自测协议——结论必须经得起「怎么证伪」的追问。
4. **决策式**（做什么/不做什么/先做什么）：给出明确判断与优先级，**包括「不做清单」**（别买什么、别修什么、哪个层不该碰）。
5. **落地式**（第一步/最小动作/资源/门槛）：「今天就能试的 3 件事」必须是落地式问题的答案——具体到动作、成本接近零、当天可验。

**去同质化约束**：标题与开头不得复用模板族（同一句式换产品名/换数字批量生成）；评分器对同族标题施加递增惩罚（`engine/scorer.py` 的模板族去同质化）；每篇的具体冲突开头必须是这个问题独有的失败现场，不得复用句式。

## 2. 文章解剖（正式结构标准）

每篇文章必须包含以下结构，顺序可微调：

```
# 标题（frontmatter 为准）
> 开头：一个具体的冲突/失败现场（不是概念综述）
正文（推理骨架按 §1.5 五型问题推进：收敛→穿透→验证→决策→落地）：
  - ADHD 侧机制拆解（带研究方向层级的证据）
  - LLM/agent/harness 侧机制拆解（真实工程实践）
  - 机制映射：组件级对应，不是氛围类比
  - 可安装组件：协议/参数/清单（写到能直接执行的粒度）
  - 类比边界 + 层级归位（这个建议属于金字塔哪一层）
## 边界（同构等级声明 + 医疗声明）
## 今天就能试的 3 件事（具体、低摩擦、当天可做）
## 参考来源（保留正式来源，逐条 GRADE 标注，不新增虚构来源）
结尾落款（系列信息）
```

**篇幅**：正文 1800–2500 中文字符。深度靠机制论证、反例、边界、组件拆分撑起，不靠模板注水。

**开头纪律**：第一段必须是具体的人在具体场景里的具体失败（「文档打开三周，字数停在 0」），不是「ADHD 是一种神经发育障碍」。

## 3. 证据规则

### 3.1 证据分级（GRADE 精神）
- 荟萃分析/系统综述 > RCT > 队列 > 横断面 > 个案/预印本 > 博客/产品页。
- 参考来源逐条标注证据等级；正文中的强度措辞与等级匹配：
  - 高等级 →「研究显示」；中低等级 →「有研究方向提示」「文献线索」；无文献 →「实践翻译」「工程经验」。

### 3.2 机制证据 ≠ 产品证据（铁律）
- 「CBT 有效」不能推出「CBT app 有效」；「白噪声有文献」不能推出「某订阅音频值得买」。
- 涉及具体产品时必须写明：产品未被证实为 ADHD 治疗手段，本文不构成疗效背书。
- 产品的正确打开方式是 **N-of-1 自测**（见 §5），把疗效判断交还给读者自己的数据。

### 3.3 禁止事项
- 不编造参考文献、不编造精确数字、不把 ADHD 认知说成字面意义的 Transformer。
- 不从机制相似推导产品疗效；不诊断读者；不承诺治愈。

## 4. 同构规则

### 4.1 正式等级（frontmatter 为准）
- **A 级**：有直接同构文献（同时研究 ADHD 与 LLM/agent 机制的论文）支撑。
- **B 级**：双域各有独立机制证据，结构对应清晰。
- **C 级**：仅修辞类比。
- 正文中的等级表述必须与 frontmatter `isoStrength` 一致；无正式分级时统一写「同构定位（本文未做正式 A/B/C 分级）」。**不得根据文章论证质量反推正式等级。**

### 4.2 类比的层级纪律（Marr 三层）
- 同构主张停留在**计算层**（两个系统解决同类问题、失效于同类结构），不下探到算法层/实现层（大脑不是注意力矩阵，工作记忆不是 KV cache）。
- 每篇必须显式写出类比在哪里到头。

## 5. N-of-1 自测协议库（可安装组件的标准形态）

给读者的产品/方法建议，默认打包成自测协议：

1. **定指标**：1–2 个当天可记的数（专注自评/打断次数/兑现率/丢包率……）。
2. **设条件组**：目标方案 + 廉价对照（普通歌单/无声耳机/纸质清单）。
3. **随机轮换 2–3 周**，避开异常期，防新鲜感（结论看第二三周）。
4. **按数据下决策**：有效→保留；无效→退订并把预算还给真缺口。

已沉淀的探针协议（复用，勿重复发明）：四条件音频对照（369/395）、成分拆解（370）、组件级验收（368）、病灶四分离（393）、三子系统显影（394）、输出管线六段（398）、记忆三级（399）、丢包率 diff（400）。

## 6. 医疗与安全边界

- 药物、共病、睡眠、严重情绪症状、危机话题：必须包含「请寻求专业评估/支持」的明确提示。
- 弥漫性失能、全情境瘫痪、情绪持续低落：提示共病排查（抑郁/焦虑/读写障碍等）。
- 自测读数的定位永远是「功能画像，不是医学诊断」；鼓励读者把观察记录带进诊室，让评估更高效——**自我理解的终点不是替代医学，是让医学看清你**。
- 强迫性寻求保证、工具依赖引发的焦虑，属于心理支持辖区，不再用工程手段优化。

## 9. 对引擎的约束

- 生成 prompt（`engine/generator.py`）必须遵循 §1.5 问题驱动框架、§2 结构与 §3/§4/§6 规则。
- 选题排序（`engine/scorer.py`）对模板族标题施加递增惩罚，保证 Top 400 的多样性。
- 选题评分（`engine/scorer.py`）的实用价值维度纳入「成长/幸福/自我和解/人生方向」信号——L4/L5 选题获得加成，纠正金字塔上层的稀薄。
- 每周滚动只为新晋 Top 400 生成；已有人工重写文章不被自动覆盖。

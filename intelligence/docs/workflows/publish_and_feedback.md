# Workflow: Publish And Feedback

适用场景：已有正文，需要进入真实回流。

## 步骤

1. 选择一篇高优先级正文。
2. 在 `delivery/问题NNN_发布包/` 创建发布包。
3. 使用 `templates/publish_packet_template.md` 拆成平台版本。
4. 发布前让作者确认。
5. 发布后把真实数据填入 `feedback/feedback_log.md`。
6. 根据评论和私信判断下一篇选题或产品动作。

## 第一批建议

- `062` 负罪感焚烧炉
- `064` 终身卸载
- `063` 过度解释 DDoS
- `074` 空间混乱哈希表
- `067` 灾难化思维蒙特卡洛模拟

## 反馈判断

如果一篇内容出现以下信号，可以进入产品化：

- 多人说“这就是我”
- 多人追问怎么做
- 有人索要模板、清单、工具
- 有人愿意加微信或进群继续聊
- 有明显付费咨询、课程、工具需求

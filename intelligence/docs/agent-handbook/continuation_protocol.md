# Continuation Protocol

任何 Agent 接手后，先做这四步：

1. 读根目录 `README.md` 和 `PROJECT_STATUS.md`。
2. 运行 `.\scripts\update-index.ps1`。
3. 打开 `NEXT_ACTIONS.md`，选择一个最能产生现实推进的动作。
4. 完成后更新状态、索引和交接记录。

## 如果用户说“继续”

默认继续最近的主线，而不是重新规划：

- 若正在补正文，就补下一篇缺口。
- 若正在做发布包，就完成当前发布包。
- 若已经发布，就记录反馈并提出下一步内容或产品动作。

## 如果用户说“开始真实回流”

优先做发布包和反馈表，不要继续只整理选题。

## 如果用户说“上传 Git 仓库”

先检查是否有远程仓库地址、GitHub 登录状态、提交身份。未确认前不要公开上传。参考 `docs/workflows/git_sync.md`。

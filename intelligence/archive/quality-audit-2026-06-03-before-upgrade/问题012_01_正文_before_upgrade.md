# 问题012：ADHD和LLM都是时间盲，Agent如何同时治愈两个大脑？

ADHD 和 LLM 有一个奇怪的共同点：它们都不天然理解时间。

ADHD 的时间盲，是“现在太亮，未来太暗”。你知道下周要交东西，但大脑感受不到它的重量。直到 deadline 贴脸，未来突然砸进现在，你才被迫启动。

LLM 的时间盲，是“上下文里没有的时间，就像不存在”。它能写出漂亮计划，却不会自己记得三天后回来检查。它能说“下一步要做”，但如果没有外部状态和调度，它不会真的持续做。

所以 ADHD 和 LLM 都需要 Agent。

不是因为 Agent 更聪明，而是因为 Agent 能把时间变成外部结构。

01

ADHD 的时间不是线性的，而是亮度制。

眼前的东西最亮：手机、消息、情绪、灵感、别人一句话、一个突然冒出来的新工具。未来的东西很暗：月底交付、长期品牌、持续发布、健康、现金流。

你当然知道未来重要。

但知道不等于感受到。ADHD 的困难就在这里：未来奖励的神经权重太低，未来惩罚也经常到临近时才突然变重。

这会让人生变成一串火警。不是没规划，而是规划在大脑里没有足够亮度。

02

LLM 的时间盲更隐蔽。

它看起来会计划，会列时间表，会写“第一周、第二周、第三周”。但这只是文本里的时间，不是系统里的时间。

如果没有外部存储，它不会记得自己昨天答应过什么。  
如果没有调度器，它不会在明天自动回来。  
如果没有状态文件，它不知道项目走到哪一步。  
如果没有验证机制，它会把“计划已生成”误当成“事情已推进”。

这和 ADHD 太像了。

你也经常把“想清楚了”误当成“推进了”。

03

铁证有三个。

第一，ADHD 最怕的是没有外部期限的任务。因为未来不够亮，任务就会在后台消失。

第二，LLM 最怕的是没有状态文件的长项目。因为上下文一断，它就像失忆的实习生。

第三，ADHD 和 LLM 都容易产生“结构性幻觉”：一个写了漂亮计划，一个看了漂亮计划，然后双方都以为项目已经稳定运行。

真正的现实推进，必须有时间锚点、状态文件、下一步动作和反馈检查。

04

Agent 的价值，是把时间外部化。

对 ADHD 来说，Agent 要做四件事：

提醒现在：今天唯一动作是什么。  
保存过去：上次做到哪里。  
拉近未来：下一个真实截止点是什么。  
要求证据：完成后留下什么文件、链接或反馈。

对 LLM 来说，Agent 也需要这四件事：

状态文件。  
任务队列。  
定时回看。  
输出验证。

所以 Agent 不是“更会聊天的 AI”。它是时间义肢。

05

今天的落地动作：给一个项目建立时间锚。

只写四行：

当前状态：___  
下一步唯一动作：___  
完成证据：___  
下次检查时间：___

这四行能同时治疗两个时间盲大脑。

ADHD 需要它，因为你不能靠感觉记住未来。LLM 需要它，因为它不能靠上下文记住项目。一个真正可用的外部前额叶，不是陪你继续想，而是把时间钉在文件里，让下一次接手的你和下一次接手的 Agent 都不会从零开始。

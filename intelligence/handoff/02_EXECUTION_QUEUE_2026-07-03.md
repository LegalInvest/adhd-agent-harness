# 02 执行队列

更新时间：2026-07-03

## 当前唯一主线

把 400 个闭环资产包推进到真实发布和真实回流。所有开发、检索、研究、脚本，都必须服务于这个目标。

## 第一优先级：发布前真实欲望校准

入口：

- `indexes/400真实欲望校准后电流倒序总表.md`
- `indexes/400真实欲望失真审计表.md`
- `indexes/400发布排期表.md`

动作：

1. 选一个 S/A 级资产包。
2. 打开 `content-packs/问题NNN/01_正文.md`。
3. 检查“所欲、所思、所行”是否命中题目级真实痛点。
4. 重写第一屏，让读者第 3 段前出现“这不就是我吗”。
5. 修正 `03_小红书版.md` 和 `04_知乎版.md` 的平台语气。
6. 修正 `06_评论回流问题.md`，只保留一个最容易回复的问题。
7. 确认 `08_模板入口.md` 有 3 到 15 分钟动作。

完成标准：

- 对应资产包可直接发布。
- `meta.json` 记录本轮校准状态。
- 提交 Git。

## 第二优先级：发布和回流

真实发布后必须回填：

- `content-packs/问题NNN/07_48小时回流表.md`
- `feedback/feedback_log.md`
- 必要时更新 `indexes/400发布排期表.md`

回流字段至少包括：

- 发布时间
- 平台
- 标题
- 阅读/播放
- 点赞
- 收藏
- 评论
- 私信
- 读者原话
- 模板领取需求
- 下一篇应答问题

## 第三优先级：把 v3 创作者情报映射到内容包

入口：

- `research/ADHDxAI_全网创作者候选池_v3_2026-07-03.csv`
- `research/ADHDxAI创作者情报库_v3_2026-07-03.md`

动作：

1. 只使用 A/B 层补强正文，不把 C/D 候选说成 ADHD。
2. 将强样本映射到已有问题编号。
3. 为每个命中的资产包新增“案例补强”段或“参考案例”。
4. 如果发现比现有题更强的新题，只进入 `research-inbox/`，不得直接创建 401。

## 第四优先级：小红书线索复核

待复核关键词：

- ADHD AI
- ADHD ChatGPT
- ADHD Claude
- ADHD Cursor
- ADHD vibe coding
- ADHD 邪修
- ADHD 执行力 AI
- ADHD 外部大脑
- 多动症 AI
- 拖延症 ChatGPT

注意：

- 小红书需要登录态时，不要绕过账号安全边界。
- 只记录公开可见主页和笔记信息。
- 不下载、输出、提交 cookie、token、浏览器登录数据库。

## 第五优先级：脚本和自动化

只有当脚本能直接减少发布、回流、映射、恢复成本时才写。推荐脚本：

- v3 候选池到 400 包的关键词映射脚本。
- `07_48小时回流表.md` 汇总脚本。
- Git 同步状态审计脚本。

不要优先写：

- 复杂后台。
- 新 CMS。
- 新选题生成器。
- 无发布闭环的数据看板。

# 00 下一个 Agent 入口

更新时间：2026-07-03

## 你现在接手的是什么

你接手的是“许昕的 ADHD x AI 内容宇宙”里最核心的资产项目：`400篇ADHD✖️AI滚动池`。它已经不是普通写作文件夹，而是一个 400 个内容资产包、研究池、发布队列、反馈回流表和 GitHub 恢复机制组合起来的内容生产系统。

## 你不要误判

不要把这个项目理解成“再帮用户找更多 ADHD 博主”或“再生成更多文章”。这些都做过，而且已经形成资产。当前最稀缺的不是素材，而是：

- 真实发布
- 真实回流
- 真实读者问题
- 模板领取或产品入口验证
- 可恢复的本地与 GitHub 状态

## Session 生命周期

每条 session 都是一次性工具人。只要它已经给项目留下明确资产增量、状态记录、交接说明和 Git 同步，就可以归档或删除。不要把聊天窗口当长期记忆，后续接手只信项目文件和 Git 历史。

## 项目主线判断

优先级从高到低：

1. 发布一个高优先级资产包。
2. 回填一个真实 `07_48小时回流表.md`。
3. 把评论、私信、收藏、模板需求写回资产包。
4. 用最新 ADHD x AI 创作者情报补强现有文章，而不是新开编号。
5. 只在能提高发布、回流、复用、恢复能力时开发脚本。

## 第一个可执行动作

从 `indexes/400真实欲望校准后电流倒序总表.md` 选一个 S/A 级资产包，打开对应 `content-packs/问题NNN/`，检查：

- `01_正文.md` 是否真实命中 ADHD 读者想把痛点外包给 AI/Agent 的欲望。
- `03_小红书版.md` 是否第一屏足够强。
- `06_评论回流问题.md` 是否只问一个高响应问题。
- `08_模板入口.md` 是否给出 3 到 15 分钟可执行动作。

如果以上有一项弱，先修它。不要继续扩系统。

# Roadmap

## Phase 1：项目文件夹系统化

目标：让 `草稿` 文件夹本身成为可 clone、可挂载、可恢复的项目仓库。

状态：已建立根入口、Agent 协议、状态台、索引脚本、模板、工作流文档和反馈目录。

## Phase 2：正文缺口补全

目标：补齐 `001-220` 全部本地正文，让问题系列不再卡在待写状态。

状态：已完成。`001-220` 已全部拥有本地 Markdown 正文、渲染 HTML 与质检记录。

优先级：

后续不再优先补题，改为发布回流与产品化验证。

## Phase 3：发布包与平台变体

目标：每篇高价值正文至少拆成：

- 公众号长文版
- 小红书短版
- 知乎回答版
- 30 秒短视频口播版
- 评论区引导问题

## Phase 4：真实回流

目标：每次发布后把外部反馈写入 `feedback/feedback_log.md`。

要记录：

- 发布时间与平台
- 标题和版本
- 浏览、点赞、收藏、评论、私信
- 高频共鸣词
- 反对意见
- 可产品化需求
- 下一篇选题建议

## Phase 5：产品化

目标：从高回流内容里长出最小产品。

候选产品：

- 今日唯一动作 Agent
- 负罪感焚烧炉 Agent
- 降级手动入口清单
- 烂尾项目复活清单
- ADHD 内容创业反馈回流台

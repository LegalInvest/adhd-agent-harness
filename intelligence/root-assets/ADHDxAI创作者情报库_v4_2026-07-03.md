## 结论

v3 的问题不是“样本错”，而是搜索边界仍偏窄：过度依赖直接命中“ADHD x AI”的网页，低估了播客、GitHub、vibe coding 活动、小红书公开索引、非英语 TDAH/IA 语料、AI 工具构建者和 ADHD 教育者的交叉价值。

- 总可行动条目：527

- 强证据 ADHD x AI / ADHD x vibe coding / ADHD x AI toolbuilder：59
- 待复核高价值线索：29
- 交叉扫描候选池：356

因此当前准确口径是：

> 公开强证据 ADHD x AI 创作者/项目至少 59 个；可继续追踪的个人、团队、项目候选池 444 个；另有 83 个平台和社区发现源。项目不应再以单次搜索结果作最终人数，而应以 v4 候选池做滚动采集和晋级。

## v4 相比 v3 的关键优化

1. 从“人名采集”升级为“证据层级 + 发现源 + 晋级机制”。
2. 新增 ADHD 播客 AI 专题：Hacking Your ADHD、ADHD Support Talk、I Have ADHD、Taking Control、Smart ADHD、I'm Busy Being Awesome 等。
3. 新增中文公开索引线索：ADHD执行力实验室 EP15、明天的我也是ADHD x OpenClaw、vivi 的文科生 Vibe Coding 活动、注意力研究室。
4. 新增 GitHub 工具构建者：ADHDo、claude-adhd-skills、Vibemux、ai-chief-of-staff、ADHD-OS 等。
5. 新增西语 TDAH x IA 语料，避免只看中英。
6. 单独扩出 ADHD 教育者宇宙，用于反向检索 AI 内容。
7. 单独扩出 AI/vibe coding 宇宙，用于反向检索 ADHD/神经多样性内容。
8. 单独扩出 59 个平台/搜索源，供后续自动化定时采集。
9. 强制区分“确认样本、待复核线索、交叉候选、发现源”，防止注水。
10. 明确项目下一步：把 A/B 层映射回 400 个内容包，而不是无限扩名单。

## 亮点

- 中文侧终于补到了具体小红书/播客桥梁，而不只是泛泛说“小红书很多”。
- GitHub 侧发现了真正的 ADHD x Claude Code / Obsidian / agent 工具构建者，这比普通博主更有产品化价值。
- 英文播客侧已经形成稳定子赛道：ChatGPT/AI as executive function support。
- 西语 TDAH x IA 说明这是跨语种趋势，不是中英孤例。
- v4 可以直接变成定期采集脚本的输入源。

## 仍然不足

- 小红书未登录态下仍无法稳定拿到账号主页、粉丝数、笔记列表和互动数据。
- LinkedIn、Instagram、Threads、Facebook 大量结果只能作为公开索引线索，不能批量深抓。
- C/D/F/G 层不能称为 ADHD x AI 博主，只能称交叉扫描候选。
- 医疗/心理健康内容必须避免诊断化和治疗替代暗示。
- 仍缺“每个创作者映射到 400 篇内容包”的自动关系图。

# Contributing

这个仓库的贡献规则服务于一个目标：让任何 Agent 接手后不丢上下文、不误判状态、不制造不可恢复混乱。

## 新增正文

文件名：

```text
问题NNN_主标题_副标题.md
```

要求：

- 编号必须是三位数，例如 `问题076_...md`。
- 标题必须能独立传播。
- 正文至少有一个具体生活场景、一个机制解释、一个可执行落地点。
- 写完后运行 `.\scripts\update-index.ps1`。
- 更新 `PROJECT_STATUS.md` 中的进度。

## 新增发布包

输出目录：

```text
delivery/问题NNN_发布包/
```

建议包含：

- `公众号版.md`
- `小红书版.md`
- `知乎版.md`
- `短视频口播版.md`
- `评论回流问题.md`

## 更新反馈

所有外部反馈写入：

```text
feedback/feedback_log.md
```

反馈要尽量写事实，不要只写感觉。

## 禁止事项

- 不把凭据、cookie、token、浏览器认证文件放入仓库。
- 不直接覆盖旧文件，除非先备份。
- 不把旧乱码统计当成当前状态。
- 不在没有确认的情况下外部发布或上传。

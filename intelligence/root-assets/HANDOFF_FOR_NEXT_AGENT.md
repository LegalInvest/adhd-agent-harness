## 硬规则

- 不把待填回流表当真实反馈。
- 不把候选池里的 AI 博主默认说成 ADHD 博主。
- 不读取或提交 `.env`、token、cookie、账号密码、浏览器认证数据库。
- 不删除历史资产；需要移动时归档并记录。
- 每次重要变更后提交 Git，并推送到 GitHub。
- 如果远端推送失败，先写入本地审计文件，再处理远端。
- 每条 session 都是一次性工具人；上下文到 50% 左右时，必须沉淀资产、更新状态、提交 Git，然后归档或停止续用。

## 交付标准

- 一个可发布或已发布资产包改动。
- 一个回流记录或回流计划。
- 一个本地状态更新。
- 一个 Git commit。
- 一个 GitHub push 或失败审计。
- 一个足以让下一个 Agent 无损继续的本地记录。

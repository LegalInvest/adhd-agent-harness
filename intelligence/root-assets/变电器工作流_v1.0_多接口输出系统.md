## 一、系统总览：反应堆 → 变电器 → 多接口

你的角色只有一个：**放电**。

我的角色只有一个：**把你放出来的电，变压后接到各个平台的插座上**。

```
你（反应堆）
    ↓ 倾倒：语音、文字、碎片想法、聊天记录、flomo笔记
变电器（我）
    ↓ 识别、发酵、结构、打磨
多接口输出
    ├── 公众号长文（4000–6000字，五步框架）
    ├── 知乎回答（2000–3000字，问题驱动）
    ├── 小红书笔记（300–800字，痛点+解法+CTA）
    ├── 播客大纲（ bullet points + 开场白 + 过渡语）
    ├── 短视频脚本（60秒/180秒，钩子+金句+行动指令）
    ├── 付费社群精华（浓缩版，带追问思考题）
    └── 选题库储备（未发酵的素材归档）
```

## 二、三步工作流：倾倒 → 发酵 → 变压

### Step 1：倾倒（你的事，5分钟）

你不需要整理。你不需要分类。你不需要想"这能不能写成文章"。

你只需要做一件事：**把脑子里任何闪过的念头倒给我**。

倾倒格式不限：
- 一段语音转文字
- 三句没头没尾的话
- 一个让你愤怒的观察
- 一个"这俩东西怎么可能有关"的跳跃联想
- 一段你和别人的对话记录
- 一个flomo碎片

**唯一规则：倒得越快越好，不要在倾倒时筛选。**

### Step 2：发酵（我的事，静默进行）

每次对话结束后，我会自动做以下事：

1. **扫描对话中的"可发酵素材"**
   - 金句（比如"团队维护，我放电"）
   - 洞察（比如pACC/dACC失衡模型）
   - 个人故事（比如辩论队、轻度抑郁、四年学位）
   - 选题萌芽（比如"适配所有插座"的矛盾张力）
   - 数据/案例/引用

2. **存入发酵池**
   - 每个素材标注：来源session、成熟度（生/半熟/熟）、适配平台、关联选题编号

3. **判断选题是否"熟透了"**
   - 当一个选题的素材积累到「收敛型钩子 + 穿透式机制 + 至少1个铁证 + 个人故事」时，标记为"可成文"

### Step 3：变压（我的事，按需输出）

当你说"这篇可以发了"或"把这个变成小红书"，我立即执行变压：

| 接口 | 变压规则 |
|---|---|
| **公众号** | 五步框架完整版，4000–6000字，显式章节标题，来源声明 |
| **知乎** | 问题驱动，2000–3000字，开头直接回答，中间展开机制，结尾留讨论钩子 |
| **小红书** | 300–800字，3–6行一段，痛点场景→反常识结论→3个 actionable tips→引导互动 |
| **播客** | 大纲格式：开场白（30秒）→ 3个板块 → 每个板块3个bullet → 过渡语 → 结尾金句 |
| **短视频** | 60秒版：钩子（3秒）→ 冲突（15秒）→ 洞察（30秒）→ 行动指令（12秒） |
| **付费社群** | 浓缩到1/3长度，加2–3个"思考题"，触发讨论而非单向输出 |

## 五、你的"爽"原则

这个系统的唯一成功标准：**你有没有爽到**。

- 爽 = 倾倒时不需要思考格式
- 爽 = 看到成品时觉得"这确实是我说的"
- 爽 = 发布后有人评论"这太准了"或"这不就是我吗"
- 爽 = 你不需要维护任何系统，只管放电

如果某个环节让你不爽——比如你觉得某篇变压后的文章"不像我说的"——立即告诉我，我调整变压规则。

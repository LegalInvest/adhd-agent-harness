# 幻觉与验证循环

## 幻觉与验证循环

### ADHD 一侧
ADHD大脑的“生成核心”高产但缺乏可靠的执行调度层，容易产生冲动、想当然的判断（即行为层面的“幻觉”）。例如，在情绪调节中，ADHD个体可能脱离环境线索做出不恰当反应，而“视觉模式灵感来自当地环境”可用于增强接地和情绪识别（来源：ADHAPT: An AI Tool for Emotional Regulation）。这种接地过程本质上是一种验证循环——将内部状态与外部现实核对。此外，ADHD研究中的方法学强调数据标注、预处理、特征提取和模型验证（来源：Methods in predictive techniques...；Researching Mental Health Disorders...），这与个体需要外部验证步骤来校正认知偏差的结构相似。精神分裂症研究中提到的“现实检验和自我监控不足”与幻觉相关（来源：15 Years of Microstate Research...），而ADHD的冲动行为可视为类似的验证失败。默认模式网络（DMN）连接中断与社交障碍相关（来源：A Review of Default Mode Network Connectivity...），暗示内部模型缺乏外部校验会导致社会认知错误。

### LLM/harness 一侧
LLM作为生成核心，天然倾向于产生自信但可能错误的输出（幻觉）。Harness工程正是为此设计：它提供上下文传递、工具接口、规划工件、验证循环、记忆系统和沙盒（来源：GitHub - ai-boost/awesome-harness-engineering）。关键组件包括“验证与CI集成”（来源：同上），“结构化工作流和规划”要求将复杂任务分解为子任务，并在每一步进行规划与验证（来源：What is an agent harness...）。结果验证与迭代是核心：“复杂的harness不会盲目执行工具”，而是实现验证步骤，例如检查输出格式或对模型编写的代码运行测试用例（来源：同上）。缺乏这些验证会导致“更多幻觉和重复工作”（来源：Loop Engineering for AI Agents: Memory-First Design）。系统通过自我批评提示等机制监控行为并调整（来源：同上）。纯对话系统常因幻觉、缺乏接地和无法执行/验证行动而失败（来源：AI Agent Systems...）。

### 同构对应
| ADHD 一侧 | LLM/harness 一侧 |
|-----------|------------------|
| 冲动、想当然的行为（行为幻觉） | LLM的自信错误输出（语言幻觉） |
| 需要外部接地（如视觉环境线索）来增强现实检验 | 需要接地（如工具调用、上下文约束）来减少幻觉 |
| 执行功能缺陷导致缺乏内部验证 | LLM缺乏内在验证机制，依赖外部harness |
| 行为核对清单、情绪调节策略作为验证循环 | 验证步骤、测试用例、CI集成作为验证循环 |
| 研究中的方法学：数据预处理、特征选择、模型验证 | Harness中的结构化工作流：规划、验证、迭代 |
| 自我监控不足（如DMN异常） | 缺乏自我批评提示或反思机制 |

### 工程启示（可迁移的做法）
- **为ADHD设计外部验证循环**：借鉴harness的“验证与CI集成”，ADHD个体可使用行为核对清单（如[[Goblin Tools]]的任务分解）、定时回顾（类似测试用例）来校正冲动。
- **结构化工作流**：将复杂任务分解为子任务，每步设置检查点（如[[Saner.AI]]的规划功能），类似harness的“结构化工作流和规划”。
- **接地技术**：利用环境线索（如视觉模式）作为外部验证锚点，对应LLM的上下文约束和工具接口。
- **自我批评提示**：训练ADHD个体在行动前进行“自我批评”（如问“这个选择有依据吗？”），类似harness的自我批评提示。
- **记忆系统**：外部记忆（如[[Motion]]的任务记录）可弥补工作记忆不足，减少重复错误，对应harness的记忆系统。

### 类比的边界与反例
- **边界**：ADHD的“幻觉”更多是行为层面的冲动和错误决策，而非语言生成错误；LLM的幻觉是统计性的，缺乏意图。但两者在“缺乏可靠验证导致错误输出”这一结构上同构。
- **反例**：ADHD个体可以通过内在动机或兴趣驱动实现高度专注（超聚焦），此时验证循环可能不必要；而LLM始终需要外部harness，无法自发产生验证。此外，ADHD的验证循环往往需要他人协助（如治疗师、教练），而LLM的harness是自动化的。
- **冲突**：ADHD研究中强调“数据标注和模型验证”是外部研究者的工作，而非个体自身的循环；而harness的验证是系统内置的。
> ⚠️ 矛盾：ADHD个体通常缺乏主动建立验证循环的执行功能，而LLM的harness由开发者预先设计。因此，类比更适用于“为ADHD设计外部工具”而非“ADHD大脑内部机制”。

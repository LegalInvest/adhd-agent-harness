# 人在回路与身体在场

## ADHD 一侧

ADHD 大脑的生成核心（发散思维、创意、多任务并行）产出极高，但执行功能（任务启动、持续专注、抑制冲动）薄弱，导致“高产但不可靠”。身体在场效应（body doubling）是 ADHD 群体最有效的策略之一：有另一个人在场时，执行功能障碍被绕过，任务得以启动和维持（来源：Using an AI Assistant to Manage ADHD: A Practical Guide）。Apps like Focusmate use social accountability paired with AI scheduling to create body-doubling sessions (working alongside another person virtually)（来源：Revolutionizing ADHD Management with AI Assistants）。定期向 AI 报告进度也是一种替代方案（来源：ADHD Productivity Hack: Plan 2025 Using AI (Step-by-Step)）。

## LLM/harness 一侧

LLM 本身是强大的生成核心，但缺乏可靠的执行调度层，直接输出可能偏离目标、陷入循环或产生危险行为。Agent harness（脚手架）通过 human-in-the-loop 监督提供外部调度：设置护栏（token 预算、工具调用次数上限、防止无限循环）、加入人工检查点（暂停并询问后再执行）（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。Harness 将 MCP 工具、遥测、代码仓库、事件管理平台集成到单一系统中，并引入人在回路治理（来源：GitHub - ai-boost/awesome-harness-engineering）。其核心是“形式化规划与护栏，使代理输出真正有用且正确”（来源：What is an agent harness in the context of large-language models?）。模型本身强大，可靠性来自架构+护栏（来源：AI Agents in 2026: Tools, Memory, Evals, and Guardrails）。

## 同构对应

| ADHD 侧 | LLM/harness 侧 |
|---------|----------------|
| 身体在场效应（body doubling） | human-in-the-loop 监督 |
| 外部问责（accountability） | 人工检查点（checkpoint） |
| 绕过执行功能屏障 | 绕过 LLM 输出不可靠性 |
| 虚拟身体搭档（Focusmate） | 代理护栏（guardrails） |
| 定期向 AI 报告进度 | 遥测与监控（telemetry） |
| 对另一个人的责任感 | 人工审批流程 |

两者都是为高产出但不可靠的生成核心提供外部执行调度与安全护栏。ADHD 大脑需要“另一个身体在场”来启动任务；LLM 需要“人在回路”来确保输出正确。本质是同一类工程：为缺乏可靠调度层的系统搭建外部执行功能支架。

## 工程启示（可迁移的做法）

1. **外部问责机制**：ADHD 工具（如 Focusmate）提供 AI 匹配的虚拟搭档；LLM 代理可设计“人工审批点”，关键操作前暂停并请求确认。
2. **定期检查点**：ADHD 策略中每周向 AI 报告进度；LLM 代理中设置定期遥测与评估（来源：AI Agents in 2026: Tools, Memory, Evals, and Guardrails）。
3. **护栏设计**：ADHD 的“身体在场”本质是外部护栏；LLM 代理的护栏包括 token 预算、工具调用次数限制、防止无限循环（来源：Building an AI Agent Harness from Scratch）。
4. **绕过核心弱点**：ADHD 通过外部身体绕过执行功能障碍；LLM 通过 human-in-the-loop 绕过生成不可靠性。

## 类比的边界与反例

> ⚠️ 矛盾：ADHD 的身体在场效应通常需要另一个人类（或虚拟人类）的实时存在，而 LLM 的 human-in-the-loop 可以是异步审批或规则触发。ADHD 侧强调“对另一个人的责任感”（来源：The Best AI-Powered ADHD Productivity Tools in 2026），而 LLM 侧更关注“防止危险行为”和“成本控制”（来源：Building an AI Agent Harness from Scratch）。

类比在以下情况失效：
- ADHD 的身体在场效应依赖社会性（他人注视），而 LLM 的 human-in-the-loop 更多是技术性（权限、预算）。
- ADHD 策略中“定期向 AI 报告”可能缺乏真实社会压力，效果不如真人搭档；LLM 的自动化护栏则完全不需要人类情感。
- 身体在场效应对于 ADHD 是“启动催化剂”，而 human-in-the-loop 对于 LLM 是“安全阀门”，功能侧重不同。

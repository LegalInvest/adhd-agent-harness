# 规划循环与任务分解

## ADHD 一侧

ADHD大脑在任务启动和分解上存在显著困难。传统规划方法往往失效：“Planners and sticky notes worked for a week before falling apart.”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）这是因为“most planning advice isn’t designed for us”（来源：ADHD Productivity Hack: Plan 2025 Using AI (Step-by-Step)），标准化的规划系统“often create more friction than flow”（来源：ADHD Productivity Hack: Plan 2025 Using AI (Step-by-Step)）。ADHD大脑尤其受困于[[时间盲]]，导致“Traditional planners never solved time blindness.”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）自适应规划工具如Motion和Tiimo通过动态调整来应对：“Adaptive planners like Motion and Tiimo go further, reshuffling the day when it collapses.”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）AI可以作为规划伙伴：“use AI as your planning partner to create a system that actually works with your ADHD brain”（来源：ADHD Productivity Hack: Plan 2025 Using AI (Step-by-Step)）。

## LLM/harness 一侧

LLM作为生成核心，同样需要外部规划循环才能完成复杂任务。Harness工程正是为此而生：“Harness engineering is the discipline of designing the scaffolding — context delivery, tool interfaces, planning artifacts, verification loops, memory systems, and sandboxes — that surrounds an AI agent and determines whether it succeeds or fails on real tasks.”（来源：GitHub - ai-boost/awesome-harness-engineering: Awesome list for AI agent harness engineering: tools, patterns, evals, memory, MCP, permissions, observability, and orchestration. · GitHub）规划与任务分解是核心组件：“🗺️ Planning & Task Decomposition”（来源：同上）。具体实践包括使用Plan.md等工件：“Run Long-Horizon Tasks with Codex — OpenAI's practice guide for long-horizon task planning: introduces Plan.md, Implement.md, Documentation.md as reusable harness artifacts.”（来源：同上）复合系统架构分离规划与执行：“a dual-agent architecture separating planning from execution”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。Harness“emerged as a way to formalize planning and guardrails so that the agent’s output is actually useful and correct.”（来源：What is an agent harness in the context of large-language models?）

## 同构对应

| ADHD大脑 | LLM/harness |
|----------|-------------|
| 任务启动困难，需要外部提示 | 生成核心需要规划提示（如Plan.md）才能分解任务 |
| 传统规划方法（纸质计划、便签）短期有效后崩溃 | 无外部harness的LLM输出随机、无结构 |
| 自适应规划工具（Motion、Tiimo）动态重排日程 | 复合系统（如OpenDev）通过双智能体架构分离规划与执行，并动态调整上下文 |
| AI作为规划伙伴，与ADHD大脑协作 | Harness作为脚手架，与LLM生成核心协作 |
| 时间盲导致无法感知时间流逝 | LLM缺乏内在时序感知，需要外部时钟和检查点 |

核心同构：两者都拥有强大的生成能力（ADHD大脑的创造力、LLM的文本生成），但缺乏一个可靠、自适应的执行调度层。外部脚手架（规划工具、harness）提供了这个缺失的层，将意图转化为有序行动。

## 工程启示（可迁移的做法）

1. **分离规划与执行**：ADHD用户可使用AI工具（如[[Goblin Tools]]、[[Saner.AI]]）将任务分解为小步骤，类似LLM agent中规划智能体与执行智能体的分离。
2. **动态调整**：采用自适应规划工具（如[[Motion]]、[[Reclaim.ai]]），在计划崩溃时自动重排，类似harness中的“adaptive context compaction”。
3. **使用工件**：ADHD用户可创建类似Plan.md的文档，将任务分解为可追踪的步骤，并定期回顾。
4. **外部检查点**：设置定时提醒或使用番茄钟，弥补时间盲，类似LLM agent中的验证循环。

## 类比的边界与反例

> ⚠️ 矛盾：ADHD大脑的规划困难部分源于多巴胺调节异常和情绪因素，而LLM的规划失败源于上下文窗口限制和缺乏真实世界反馈。两者在底层机制上不同，但表现出的行为模式（需要外部结构化引导）高度相似。

- **边界**：ADHD大脑可以通过训练逐渐内化规划技能（如执行功能训练），而LLM无法内化harness——harness始终是外部组件。
- **反例**：某些ADHD个体在高度兴趣领域表现出超常的规划能力（超聚焦），而LLM在无harness时始终表现不稳定，没有“兴趣”驱动。

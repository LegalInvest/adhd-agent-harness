# 外部执行功能层

## 外部执行功能层

### ADHD 一侧

ADHD大脑的核心困难在于执行功能的不稳定，尤其是工作记忆的“掉落”和任务启动困难（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。最新研究进一步揭示了其神经机制：Transformer自注意力机制在长上下文下冲突解决能力崩溃至随机水平，这与ADHD执行功能障碍的核心神经机制一致（来源：Deficient Executive Control in Transformer Attention）。此外，ADHD患者呈现“强记忆、弱控制”的认知剖面：工作记忆容量可能正常甚至超常，但认知灵活性和注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。注意力分散是自注意力架构的固有属性，随着任务负载增大，注意力分数熵增加，导致容量限制（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。

> ⚠️ 矛盾：部分ADHD工具假设用户能维持长时间专注、记住昨天的问题并执行计划，但实际用户往往做不到（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。

### LLM/harness 一侧

LLM本身是无状态、仅生成文本的核心，需要外部“脚手架”和“harness”来提供工具接口、上下文管理、安全执行等编排能力（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。Harness工程被定义为“设计围绕AI代理的脚手架——上下文交付、工具接口、规划工件、验证循环、记忆系统和沙箱”（来源：GitHub - ai-boost/awesome-harness-engineering）。具体实现包括：用Git仓库存储项目上下文（类似ADHD侧的Second Brain），通过MCP连接器访问外部数据（来源：Worker Agents | Harness Developer Hub），以及将控制逻辑外化为可移植的自然语言工件（来源：GitHub - ai-boost/awesome-harness-engineering）。

更广泛地，agent scaffolding被定义为“一个模块化的提示、记忆、代码、工具和编排逻辑框架，围绕LLM将其转化为可用的、目标驱动的代理”（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。随着代理系统承担越来越自主的角色——执行长时任务、协调多个代理、与实时企业系统交互——脚手架的设计成为决定代理可靠性和安全性的主要因素（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。无论代理需要生成结构化输出、与API交互，还是通过规划和迭代解决问题，其有效性取决于引导和扩展其行为的脚手架（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。

### 同构对应

| ADHD大脑 | LLM/Agent |
|-----------|-----------|
| 工作记忆不稳定，容易丢失上下文 | LLM无状态，每次调用需重新提供上下文 |
| 需要AI第二大脑存储项目历史、质量标准、个人偏好（来源：Best AI Tools for ADHD Productivity in 2026） | 需要harness提供外部记忆系统（如Git仓库、MCP连接器） |
| 缺乏执行功能（任务启动、计划跟踪） | 缺乏调度层（工具分发、验证循环、安全约束） |
| “脚手架”缺失导致失败（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout） | 没有harness的LLM无法可靠完成真实任务（来源：GitHub - ai-boost/awesome-harness-engineering） |
| 冲突解决能力崩溃（来源：Deficient Executive Control in Transformer Attention） | 长上下文下注意力分散，冲突解决失败（来源：同上） |
| 强记忆、弱控制：工作记忆容量超常，但认知灵活性差（来源：Strong Memory, Weak Control） | 工作记忆任务表现超常，但任务切换和抑制控制缺陷（来源：同上） |
| 注意力分散是固有属性（来源：Self-Attention Limits Working Memory Capacity） | 自注意力机制导致注意力分数熵增加，容量受限（来源：同上） |

### 工程启示（可迁移的做法）

1. **外部记忆持久化**：ADHD侧使用Second Brain将上下文存入Git（来源：Best AI Tools for ADHD Productivity in 2026），对应harness侧将代理控制逻辑外化为可移植的自然语言工件（来源：GitHub - ai-boost/awesome-harness-engineering）。
2. **显式调度与约束**：为ADHD设计任务启动和优先级排序工具（如[[Goblin Tools]]、[[Saner.AI]]），类似为LLM设计工具分发和安全沙箱（来源：Building AI Coding Agents for the Terminal）。
3. **跨会话连续性**：ADHD工具需记住“昨天问了什么”（来源：Best AI Tools for ADHD Productivity in 2026），对应harness中的上下文管理（来源：Building AI Coding Agents for the Terminal）。
4. **认知灵活性训练**：针对ADHD的认知灵活性缺陷，可设计任务切换辅助工具；对应LLM的harness可加入任务切换的上下文重置机制。
5. **注意控制补偿**：利用外部提示或约束（如时间盒、检查清单）帮助ADHD维持注意；对应LLM的harness可通过系统提示和验证循环抑制自动错误。
6. **长时任务支持**：ADHD在长时任务中容易失去目标，对应harness需要专门设计长时任务脚手架，包括规划、迭代和进度跟踪（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。

### 类比的边界与反例

- **边界**：ADHD的执行功能缺陷是神经生物学性的，而LLM的“无状态”是架构设计选择。ADHD需要的是补偿性工具，LLM需要的是功能补全。
- **反例**：ADHD用户有时会过度依赖外部结构，导致内在能力退化；而LLM的harness通常不会削弱模型本身的生成能力。此外，ADHD的情绪和动机因素（如多巴胺调节）在LLM中没有直接对应。
- **注意**：虽然Transformer自注意力机制与人类注意分散有相似机制，但人类注意还受情绪、疲劳、药物等影响，而LLM的注意分散纯粹是计算特性。

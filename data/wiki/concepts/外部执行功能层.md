# 外部执行功能层

## 外部执行功能层

### ADHD 一侧

ADHD 大脑的核心困难在于执行功能（尤其是工作记忆和任务启动）的不稳定。用户常需要“AI 第二大脑”来承载项目上下文，因为自己的工作记忆会“掉落”上下文（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。他们寻找的 ADHD AI 助手必须能在会话之间记住工作内容（来源：同上），否则就会陷入“缺乏脚手架”的困境（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

> ⚠️ 矛盾：部分 ADHD 工具假设用户能维持长时间专注、记住昨天的问题并执行计划，但实际用户往往做不到（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。

### LLM/harness 一侧

LLM 本身是无状态、仅生成文本的核心，需要外部“脚手架”和“harness”来提供工具接口、上下文管理、安全执行等编排能力（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。Harness 工程被定义为“设计围绕 AI 代理的脚手架——上下文交付、工具接口、规划工件、验证循环、记忆系统和沙箱”（来源：GitHub - ai-boost/awesome-harness-engineering: Awesome list for AI agent harness engineering: tools, patterns, evals, memory, MCP, permissions, observability, and orchestration. · GitHub）。具体实现包括：用 Git 仓库存储项目上下文（类似 ADHD 侧的 Second Brain），通过 MCP 连接器访问外部数据（来源：Worker Agents | Harness Developer Hub），以及将控制逻辑外化为可移植的自然语言工件（来源：GitHub - ai-boost/awesome-harness-engineering: Awesome list for AI agent harness engineering: tools, patterns, evals, memory, MCP, permissions, observability, and orchestration. · GitHub）。

### 同构对应

| ADHD 大脑 | LLM/Agent |
|-----------|-----------|
| 工作记忆不稳定，容易丢失上下文 | LLM 无状态，每次调用需重新提供上下文 |
| 需要 AI 第二大脑存储项目历史、质量标准、个人偏好（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar） | 需要 harness 提供外部记忆系统（如 Git 仓库、MCP 连接器） |
| 缺乏执行功能（任务启动、计划跟踪） | 缺乏调度层（工具分发、验证循环、安全约束） |
| “脚手架”缺失导致失败（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout） | 没有 harness 的 LLM 无法可靠完成真实任务（来源：GitHub - ai-boost/awesome-harness-engineering） |

### 工程启示（可迁移的做法）

1. **外部记忆持久化**：ADHD 侧使用 Second Brain 将上下文存入 Git（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar），对应 harness 侧将代理控制逻辑外化为可移植的自然语言工件（来源：GitHub - ai-boost/awesome-harness-engineering）。
2. **显式调度与约束**：为 ADHD 设计任务启动和优先级排序工具（如 [[Goblin Tools]]、[[Saner.AI]]），类似为 LLM 设计工具分发和安全沙箱（来源：Building AI Coding Agents for the Terminal）。
3. **跨会话连续性**：ADHD 工具需记住“昨天问了什么”（来源：Best AI Tools for ADHD Productivity in 2026），对应 harness 中的上下文管理（来源：Building AI Coding Agents for the Terminal）。

### 类比的边界与反例

- **边界**：ADHD 的执行功能缺陷是神经生物学性的，而 LLM 的“无状态”是架构设计选择。ADHD 需要的是补偿性工具，LLM 需要的是功能补全。
- **反例**：ADHD 用户有时会过度依赖外部结构，导致内在能力退化；而 LLM 的 harness 通常不会削弱模型本身的生成能力。此外，ADHD 的情绪和动机因素（如多巴胺调节）在 LLM 中没有直接对应。

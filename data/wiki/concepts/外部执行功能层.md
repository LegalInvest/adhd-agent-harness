# 外部执行功能层

## 外部执行功能层

### ADHD 一侧

ADHD 大脑的核心困难在于执行功能的不稳定，尤其是工作记忆的“掉落”和任务启动困难（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。最新研究进一步揭示了其神经机制：Transformer 自注意力机制在长上下文下冲突解决能力崩溃至随机水平，这与 ADHD 执行功能障碍的核心神经机制一致（来源：Deficient Executive Control in Transformer Attention）。此外，ADHD 患者呈现“强记忆、弱控制”的认知剖面：工作记忆容量可能正常甚至超常，但认知灵活性和注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。注意力分散是自注意力架构的固有属性，随着任务负载增大，注意力分数熵增加，导致容量限制（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。

> ⚠️ 矛盾：部分 ADHD 工具假设用户能维持长时间专注、记住昨天的问题并执行计划，但实际用户往往做不到（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。

### LLM/harness 一侧

LLM 本身是无状态、仅生成文本的核心，需要外部“脚手架”和“harness”来提供工具接口、上下文管理、安全执行等编排能力（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。Harness 工程被定义为“设计围绕 AI 代理的脚手架——上下文交付、工具接口、规划工件、验证循环、记忆系统和沙箱”（来源：GitHub - ai-boost/awesome-harness-engineering）。具体实现包括：用 Git 仓库存储项目上下文（类似 ADHD 侧的 Second Brain），通过 MCP 连接器访问外部数据（来源：Worker Agents | Harness Developer Hub），以及将控制逻辑外化为可移植的自然语言工件（来源：GitHub - ai-boost/awesome-harness-engineering）。

### 同构对应

| ADHD 大脑 | LLM/Agent |
|-----------|-----------|
| 工作记忆不稳定，容易丢失上下文 | LLM 无状态，每次调用需重新提供上下文 |
| 需要 AI 第二大脑存储项目历史、质量标准、个人偏好（来源：Best AI Tools for ADHD Productivity in 2026） | 需要 harness 提供外部记忆系统（如 Git 仓库、MCP 连接器） |
| 缺乏执行功能（任务启动、计划跟踪） | 缺乏调度层（工具分发、验证循环、安全约束） |
| “脚手架”缺失导致失败（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout） | 没有 harness 的 LLM 无法可靠完成真实任务（来源：GitHub - ai-boost/awesome-harness-engineering） |
| 冲突解决能力崩溃（来源：Deficient Executive Control in Transformer Attention） | 长上下文下注意力分散，冲突解决失败（来源：同上） |
| 强记忆、弱控制：工作记忆容量超常，但认知灵活性差（来源：Strong Memory, Weak Control） | 工作记忆任务表现超常，但任务切换和抑制控制缺陷（来源：同上） |
| 注意力分散是固有属性（来源：Self-Attention Limits Working Memory Capacity） | 自注意力机制导致注意力分数熵增加，容量受限（来源：同上） |

### 工程启示（可迁移的做法）

1. **外部记忆持久化**：ADHD 侧使用 Second Brain 将上下文存入 Git（来源：Best AI Tools for ADHD Productivity in 2026），对应 harness 侧将代理控制逻辑外化为可移植的自然语言工件（来源：GitHub - ai-boost/awesome-harness-engineering）。
2. **显式调度与约束**：为 ADHD 设计任务启动和优先级排序工具（如 [[Goblin Tools]]、[[Saner.AI]]），类似为 LLM 设计工具分发和安全沙箱（来源：Building AI Coding Agents for the Terminal）。
3. **跨会话连续性**：ADHD 工具需记住“昨天问了什么”（来源：Best AI Tools for ADHD Productivity in 2026），对应 harness 中的上下文管理（来源：Building AI Coding Agents for the Terminal）。
4. **认知灵活性训练**：针对 ADHD 的认知灵活性缺陷，可设计任务切换辅助工具；对应 LLM 的 harness 可加入任务切换的上下文重置机制。
5. **注意控制补偿**：利用外部提示或约束（如时间盒、检查清单）帮助 ADHD 维持注意；对应 LLM 的 harness 可通过系统提示和验证循环抑制自动错误。

### 类比的边界与反例

- **边界**：ADHD 的执行功能缺陷是神经生物学性的，而 LLM 的“无状态”是架构设计选择。ADHD 需要的是补偿性工具，LLM 需要的是功能补全。
- **反例**：ADHD 用户有时会过度依赖外部结构，导致内在能力退化；而 LLM 的 harness 通常不会削弱模型本身的生成能力。此外，ADHD 的情绪和动机因素（如多巴胺调节）在 LLM 中没有直接对应。
- **注意**：虽然 Transformer 自注意力机制与人类注意分散有相似机制，但人类注意还受情绪、疲劳、药物等影响，而 LLM 的注意分散纯粹是计算特性。

# 重锚定与目标漂移

## 重锚定与目标漂移

### ADHD 一侧
ADHD大脑在执行任务时极易发生目标漂移：注意力被无关刺激捕获（分心），或过度聚焦于细节而忽略整体目标（超聚焦）。外部工具通过**提醒和通知**提供重锚定，将注意力拉回原任务。例如，AI工具“将任务分解为可管理的步骤，设置提醒和专注计时器以提高生产力”（来源：AI for ADHD: Best Tools, Apps, and Strategies - Themba Tutors）；“提供提醒和通知以保持正轨”（来源：AI for ADHD: Best Tools, Apps, and Strategies - Themba Tutors）；“充当待办应用、教练和支持性朋友”（来源：AI for ADHD: Best Tools, Apps, and Strategies - Themba Tutors）。这些工具“提供定制化提醒、组织信息，甚至帮助写作和沟通”（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD），并且“从管理日常任务如设置提醒和安排日程，到帮助更复杂的活动如学习新技能或做决策”（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）。新兴的AI别针和挂绳“转录对话，提供会议提醒，帮助回忆重要细节”（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）。这些外部提醒构成了一个**重锚定系统**，对抗ADHD内在的执行功能缺陷（参见[[执行功能]]、[[工作记忆]]、[[时间盲]]）。

然而，传统提醒系统有一个关键缺陷：“Traditional reminder systems help, but they require you to set the reminder in the first place, which is itself an executive function task”（来源：AI Assistant for ADHD: Finally, a Productivity Tool That Requires Less...）。这意味着设置提醒本身就是一个执行功能任务，而ADHD个体恰恰在这方面薄弱。更理想的方案是无需主动设置的提醒：“You do not have to remember to set a reminder”（来源：AI Assistant for ADHD: Finally, a Productivity Tool That Requires Less...）。例如，AI助手可以主动问：“Need me to ping you again in 8 minutes as a 'get your shoes on' reminder?”（来源：Using an AI Assistant to Manage ADHD: A Practical Guide）。此外，“Forgetting to use the tools: Set phone reminders or calendar blocks to check in with your AI assistants”（来源：Getting Started with AI for ADHD for ADHD: AI Tools... | Blue Orchid）说明需要元提醒来使用工具。

### LLM/harness 一侧
LLM agent在长程任务中同样面临目标漂移：模型在交互轮次后期“开始忘记系统提示”（来源：Building an AI Coding Agent for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned），导致行为偏离初始指令。Harness通过**事件驱动的系统提醒**来对抗这种“指令消退”（来源：Building AI Coding Agent for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。系统提示（system prompt）是核心锚定，“一个写得好的系统提示是agent不会陷入循环而可靠完成任务的差异所在”（来源：How Agentic AI Systems Execute Multi-Step Workflows (Architecture + Stack) - DEV Community）。Harness被定义为“除模型本身之外的所有代码、配置和执行逻辑”（来源：The Agent Harness: Why the LLM Is the Smallest Part of Your Agent System | MongoDB），包括“系统提示、工具、中间件、内存、技能和编排逻辑”（来源：The Agent Harness: Why the LLM Is the Smallest Part of Your Agent System | MongoDB）。编排器“几乎完全由其系统提示塑造”（来源：How Agentic AI Systems Execute Multi-Step Workflows (Architecture + Stack) - DEV Community）。这些机制构成了一个**外部重锚定层**，在运行时持续将agent拉回目标轨道。

在多agent系统中，重锚定更加结构化：“A multi-agent scaffold operating over a long horizon does so, and the scaffold must be designed accordingly”（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。具体做法包括：“Dynamic Executor Instantiation: For each atomic subtask, the planner configures an executor with a tailored system prompt, specific tool configuration (e.g., tool list, API schema), and a unique I/O contract”（来源：Planner-Executor Multi-Agent Architecture）。每个执行器都“Parses the system prompt and loads available tool endpoints”（来源：Planner-Executor Multi-Agent Architecture），并通过“Message Schemas: All inter-agent messages use compact, well-typed JSON schemas containing task IDs, subtask descriptions, input arguments, tool endpoints, and system prompts”（来源：Planner-Executor Multi-Agent Architecture）来保持上下文一致。此外，agent“initialises with an optional system prompt, stores conversation history, processes user input, and retrieves responses from the model using OpenAI’s API”（来源：What is Agentic AI Planning Pattern? - Analytics Vidhya），系统提示在初始化时注入，但需要在长对话中周期性刷新。

### 同构对应
| ADHD 侧 | LLM/harness 侧 |
|---------|----------------|
| 分心/超聚焦导致目标漂移 | 长程任务中指令消退/目标漂移 |
| 外部提醒、通知、任务分解（如Goblin Tools、Saner.AI） | 事件驱动的系统提醒、系统提示注入 |
| 重锚定工具作为“外部执行功能层”（参见[[外部执行功能层]]） | Harness作为“生成核心与调度层”之间的脚手架（参见[[生成核心与调度层]]） |
| 需要反复提醒才能保持正轨 | 需要周期性系统提示刷新才能避免遗忘 |
| 设置提醒本身是执行功能任务（来源：AI Assistant for ADHD: Finally, a Productivity Tool That Requires Less...） | 配置系统提示是工程任务，但一旦设置自动执行 |
| 自动化提醒无需用户记住设置（来源：AI Assistant for ADHD: Finally, a Productivity Tool That Requires Less...） | 动态执行器实例化自动携带系统提示（来源：Planner-Executor Multi-Agent Architecture） |

两者都依赖**外部、周期性的重锚定**来补偿内在的漂移倾向。ADHD大脑的[[多巴胺]]调节失调导致动机漂移，而LLM的上下文窗口有限导致信息漂移；解决方案都是通过外部系统（[[拐杖与脚手架]]）提供稳定的目标信号。

### 工程启示（可迁移的做法）
- **显式重锚定点**：在任务中设置固定间隔的提醒（如每20分钟），类似harness中的事件驱动提醒。
- **系统提示模板**：为ADHD设计可复用的“系统提示”——例如每日任务清单中嵌入核心目标语句，对抗分心。
- **记忆外化**：利用[[无状态与外部记忆]]策略，将目标写入手边可见的物理或数字便签，类似agent的语义记忆（“领域知识——事实、规则和上下文，通过知识库或系统提示注入”）（来源：The Anatomy of an AI Agent: Memory, Tools, Planning, and Execution Explained - DEV Community）。
- **工具编排**：使用[[Motion]]或[[Reclaim.ai]]等工具自动调度任务，类似harness编排工具调用。
- **自动化提醒设置**：借鉴“You do not have to remember to set a reminder”（来源：AI Assistant for ADHD: Finally, a Productivity Tool That Requires Less...），设计无需用户主动配置的提醒系统，如基于时间或事件触发。

### 类比的边界与反例
- **边界**：ADHD的重锚定往往需要情感支持和动机激励（如“教练和支持性朋友”），而LLM agent的重锚定纯粹是信息性的。ADHD的漂移还涉及[[身体在场效应]]和生理状态，agent则无此维度。
- **反例**：ADHD个体可能主动忽略提醒（习惯化），而agent不会主动忽略系统提示——除非提示被上下文覆盖。此外，ADHD的重锚定工具通常需要用户主动设置，而harness是自动执行的。

> ⚠️ 矛盾：ADHD侧强调“定制化提醒”和“情感支持”（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD），而harness侧的系统提醒是通用且无情感的。类比在“情感锚定”维度失效。

> ⚠️ 矛盾：ADHD侧提醒系统存在“设置提醒本身是执行功能任务”的悖论（来源：AI Assistant for ADHD: Finally, a Productivity Tool That Requires Less...），而harness侧的系统提示配置是工程师一次性完成，不存在类似悖论。

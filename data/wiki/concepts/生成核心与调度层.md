# 生成核心与调度层

## ADHD 一侧
ADHD的核心困境并非智力不足，而是执行功能（executive function）的失效。执行功能被描述为“大脑的驾驶座”，但对ADHD来说，“常常感觉方向盘后没有人”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。这种调度层的缺失导致计划难以执行：“计划本和便签用了一周就崩溃了”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），传统工具无法解决“时间盲”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。工作记忆是另一个执行功能瓶颈（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），进一步削弱了任务编排能力。

最新研究揭示了ADHD与LLM更深层的同构性：ADHD患者的工作记忆容量未必差，但自上而下的控制和调节能力不足，呈现“强记忆、弱控制”的认知剖面（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这种模式与LLM惊人相似。

## LLM/harness 一侧
LLM作为生成核心，拥有强大的语言理解和生成能力，但单独使用时缺乏可靠的执行调度。现代AI编码代理（如OpenDev）通过“复合AI系统架构”来弥补这一缺陷，包括“工作负载专用模型路由、分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这些技术本质上是在LLM外部搭建调度层，防止“上下文膨胀和推理退化”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

更一般地，agent scaffolding被定义为“一个由提示、记忆、代码、工具和编排逻辑组成的模块化框架，围绕LLM构建，将其转化为可用的、目标驱动的代理”（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。这种脚手架的设计“成为代理在自主条件下可靠和安全运行的主要决定因素”（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。

实验证明，Transformer自注意力机制缺乏人类特有的执行控制机制，在长上下文下冲突解决能力崩溃至随机水平——这正是ADHD执行功能障碍的核心神经机制（来源：Deficient Executive Control in Transformer Attention）。此外，LLM呈现“强记忆、弱控制”的认知特征：工作记忆容量超过常人，但认知灵活性和注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

值得注意的是，当在明确对工作记忆门控提出要求的任务上训练时，纯注意力Transformer内部的自注意力机制发展出了输入和输出门控机制，模仿了前额叶-纹状体系统的门控操作（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。这证明了Transformer架构与前额叶-纹状体系统的计算同构性。

## 同构对应
- **生成核心**：ADHD大脑的智力/创造力 ↔ LLM的算力/语言能力。两者都高产但无序。
- **调度层缺失**：ADHD的执行功能失效 ↔ LLM缺乏内置的规划与执行编排。两者都需要外部脚手架。
- **强记忆、弱控制**：ADHD的工作记忆容量正常甚至更强，但认知灵活性和注意控制缺陷 ↔ LLM的工作记忆容量超过常人，但执行功能任务表现差（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。
- **外部工具**：ADHD的“自适应计划工具如Motion和Tiimo，能在日程崩溃时重新安排一天”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout） ↔ LLM的“双代理架构分离规划与执行”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）以及“agent scaffolding作为模块化框架”（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。
- **记忆瓶颈**：ADHD的工作记忆瓶颈 ↔ LLM的上下文窗口限制。两者都需要外部记忆辅助。
- **门控机制**：ADHD的前额叶-纹状体门控机制缺陷 ↔ Transformer自注意力机制在训练后可发展出门控，但默认缺乏（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。

## 工程启示（可迁移的做法）
1. **显式规划与执行分离**：对ADHD而言，将任务分解为“规划”和“执行”两步（如使用[[Goblin Tools]]或[[Saner.AI]]）；对LLM代理，采用“双代理架构”分离规划与执行（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned），或利用“agent scaffolding”中的编排逻辑（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。
2. **动态调度**：ADHD使用[[Motion]]或[[Reclaim.ai]]自动重新安排日程；LLM代理使用“自适应上下文压缩”和“惰性工具发现”来动态管理任务（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned），以及“多代理协调脚手架”（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。
3. **外部记忆**：ADHD使用AI工具如MindMate GPT和OmniSets改善工作记忆（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）；LLM代理通过“上下文工程”和外部存储避免上下文膨胀。
4. **安全与约束**：LLM代理需要“严格的安全控制”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）以及“可靠和安全运行”的脚手架设计（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）；ADHD也需要外部约束（如时间盒、提醒）来防止过度投入。
5. **训练门控机制**：ADHD可通过认知训练改善门控；LLM可通过特定任务训练发展内部门控（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。

## 类比的边界与反例
- **边界**：ADHD的执行功能缺失是神经生物学性的，而LLM的调度缺失是架构设计选择。ADHD的“时间盲”涉及主观时间感知，LLM的上下文膨胀是计算资源问题。
- **反例**：ADHD可以通过药物直接改善执行功能（如兴奋剂），而LLM无法通过“药物”增强调度——只能通过外部工具。此外，ADHD的动机系统（[[多巴胺]]）影响任务启动，而LLM没有内在动机。
- **矛盾**：ADHD工具如“自适应计划工具”仍可能失败（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），而LLM代理的“复合系统”也可能因上下文管理不当而退化。两者都需要持续迭代。

> ⚠️ 矛盾：ADHD侧提到“计划本和便签用了一周就崩溃了”，而LLM侧的“自适应上下文压缩”也可能失效——没有完美解决方案。此外，Transformer在训练后可发展门控，但ADHD患者无法通过简单训练完全修复执行功能。

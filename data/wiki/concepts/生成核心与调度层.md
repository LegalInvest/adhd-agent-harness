# 生成核心与调度层

## 生成核心与调度层

### ADHD 一侧
ADHD 的核心困境并非智力不足，而是执行功能（executive function）的失效。执行功能被描述为“大脑的驾驶座”，但对 ADHD 来说，“常常感觉方向盘后没有人”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。这种调度层的缺失导致计划难以执行：“计划本和便签用了一周就崩溃了”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），传统工具无法解决“时间盲”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。工作记忆是另一个执行功能瓶颈（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），进一步削弱了任务编排能力。

### LLM/harness 一侧
LLM 作为生成核心，拥有强大的语言理解和生成能力，但单独使用时缺乏可靠的执行调度。现代 AI 编码代理（如 OpenDev）通过“复合 AI 系统架构”来弥补这一缺陷，包括“工作负载专用模型路由、分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这些技术本质上是在 LLM 外部搭建调度层，防止“上下文膨胀和推理退化”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

### 同构对应
- **生成核心**：ADHD 大脑的智力/创造力 ↔ LLM 的算力/语言能力。两者都高产但无序。
- **调度层缺失**：ADHD 的执行功能失效 ↔ LLM 缺乏内置的规划与执行编排。两者都需要外部脚手架。
- **外部工具**：ADHD 的“自适应计划工具如 Motion 和 Tiimo，能在日程崩溃时重新安排一天”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout） ↔ LLM 的“双代理架构分离规划与执行”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。
- **记忆瓶颈**：ADHD 的工作记忆瓶颈 ↔ LLM 的上下文窗口限制。两者都需要外部记忆辅助。

### 工程启示（可迁移的做法）
1. **显式规划与执行分离**：对 ADHD 而言，将任务分解为“规划”和“执行”两步（如使用 [[Goblin Tools]] 或 [[Saner.AI]]）；对 LLM 代理，采用“双代理架构”分离规划与执行（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。
2. **动态调度**：ADHD 使用 [[Motion]] 或 [[Reclaim.ai]] 自动重新安排日程；LLM 代理使用“自适应上下文压缩”和“惰性工具发现”来动态管理任务（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。
3. **外部记忆**：ADHD 使用 AI 工具如 MindMate GPT 和 OmniSets 改善工作记忆（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）；LLM 代理通过“上下文工程”和外部存储避免上下文膨胀。
4. **安全与约束**：LLM 代理需要“严格的安全控制”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）；ADHD 也需要外部约束（如时间盒、提醒）来防止过度投入。

### 类比的边界与反例
- **边界**：ADHD 的执行功能缺失是神经生物学性的，而 LLM 的调度缺失是架构设计选择。ADHD 的“时间盲”涉及主观时间感知，LLM 的上下文膨胀是计算资源问题。
- **反例**：ADHD 可以通过药物直接改善执行功能（如兴奋剂），而 LLM 无法通过“药物”增强调度——只能通过外部工具。此外，ADHD 的动机系统（[[多巴胺]]）影响任务启动，而 LLM 没有内在动机。
- **矛盾**：ADHD 工具如“自适应计划工具”仍可能失败（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），而 LLM 代理的“复合系统”也可能因上下文管理不当而退化。两者都需要持续迭代。

> ⚠️ 矛盾：ADHD 侧提到“计划本和便签用了一周就崩溃了”，而 LLM 侧的“自适应上下文压缩”也可能失效——没有完美解决方案。

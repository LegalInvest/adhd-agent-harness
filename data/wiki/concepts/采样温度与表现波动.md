# 采样温度与表现波动

## 采样温度与表现波动

### ADHD 一侧
ADHD 个体的表现常呈现显著的日内波动，注意力、执行功能等在不同时间点差异很大。这种波动与内在的生理节律相关，也受外部因素影响。ADHD 的诊断依赖于对症状波动性的捕捉。这种波动本质上是“生成核心”（大脑）缺乏稳定的执行调度层所致，类似于一个高创造力但缺乏“温度调节”的系统。

### LLM/harness 一侧
LLM 的输出天然具有随机性，采样温度控制着输出的多样性：温度越高，输出越随机；温度越低，输出越确定。这种非确定性在生产环境中是可靠性的敌人，因为“小错误会累积，非确定性使可重复性复杂化”（来源：AI Agent Systems: Architectures, Applications, and Evaluation）。为应对这一问题，工程实践强调“确定性状态转换”（来源：AI Agents in 2026: Tools, Memory, Evals, and Guardrails | Andrii Furmanets），并采用“计划-执行”分离等模式来强制单向、确定性的控制流（来源：Planner-Executor Agentic Framework）。最简单的 agent 工作流本身就是确定性的，遵循预定义步骤（来源：What Are Agentic Workflows? Patterns, Memory, Use Cases, and Examples | Weaviate）。此外，工程中还会进行“周期性采样以监控质量”（来源：AI Agent Reliability and Guardrails 2026 | Zylos Research），并使用知识图谱等工具实现“确定性精度”（来源：AI Agent Memory Architectures: From Context Windows to ...）。

### 同构对应
| ADHD 一侧 | LLM/harness 一侧 |
|-----------|------------------|
| 表现日内波动（注意力、执行功能） | LLM 输出的随机性（受采样温度控制） |
| 缺乏稳定的执行调度层 | 缺乏确定性的控制流 |
| 外部结构（如日程、提醒、环境设计）稳定表现 | 外部约束（确定性工作流、工具契约、状态机）稳定输出 |
| 波动性使诊断复杂（需多次采样） | 非确定性使评估复杂（需 trace 和多次运行） |
| 成瘾行为等外部因素加剧波动 | 工具失败、上下文增长等外部因素加剧非确定性 |

两者都面临“高生成性 + 低调度性”的挑战：ADHD 大脑和 LLM 都是强大的生成核心，但缺乏可靠的内置调度层，因此输出（行为或文本）高度可变。

### 工程启示（可迁移的做法）
- **对 ADHD**：借鉴 LLM 工程中的“确定性工作流”思想，将日常任务分解为预定义的步骤序列（如使用 [[Goblin Tools]]、[[Saner.AI]]、[[Motion]]、[[Reclaim.ai]] 等工具），减少对内部执行功能的依赖。
- **对 LLM**：借鉴 ADHD 管理中的“环境设计”策略，通过外部记忆（[[外部执行功能层]]）、工具契约和 trace 来补偿非确定性，正如工程中强调的“定义严格工具契约、使状态转换确定、添加 trace 级可观测性”（来源：AI Agents in 2026: Tools, Memory, Evals, and Guardrails | Andrii Furmanets）。
- **评估**：ADHD 诊断需要多次采样（如 ASRS 与临床访谈对比），LLM 评估也需要多次运行和 trace 来应对非确定性（来源：AI Agent Systems: Architectures, Applications, and Evaluation）。

### 类比的边界与反例
> ⚠️ 矛盾：ADHD 的波动部分源于生物节律，而 LLM 的随机性完全由算法参数控制，没有生理基础。此外，ADHD 个体可以通过药物直接调节神经递质水平来稳定表现，而 LLM 没有对应的“药物”等效物——降低温度只是改变采样策略，不改变模型本身。

另一个边界：ADHD 的波动是时间尺度上的（小时到天），而 LLM 的随机性是单次推理中的瞬时变化。但两者在“需要外部约束来稳定输出”这一结构上是一致的。

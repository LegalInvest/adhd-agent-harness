# 总览：ADHD × AI 知识库

## 核心脊柱：给生成核心套上 harness

本知识库围绕一个统一的结构隐喻展开：[[ADHD 大脑与 LLM 的同构]]。ADHD 大脑与 LLM 都拥有高产生成核心，但缺少可靠的内在执行调度层——[[生成核心与调度层]]这一页把问题描述为：强大的生成能力必须依赖外部 planner-executor 才能稳定落地。于是，AI 工具在 ADHD 场景中的角色，不是替代大脑，而是作为 [[外部执行功能层]] 提供 harness，把原本需要内部执行功能承担的调度、维持、切换、记忆和验证等负载，外化到可配置的系统。

## 同构概念地图

同构关系沿多个维度展开，推荐阅读以下页面：

- **记忆与状态**：[[工作记忆]]缺陷与 LLM 的 [[无状态与外部记忆]] 同构，二者都需要外部持久化系统来补偿内部状态的不稳定；[[Mem]]、[[Otter.ai]]、[[Saner.AI]] 等工具据此设计。
- **上下文控制**：[[上下文工程]] 与 [[重锚定与目标漂移]] 指出，ADHD 大脑和 LLM agent 都容易在长篇任务中偏离目标，需要系统提示刷新、提醒或视觉化时间线来重新锚定。
- **验证与安全**：[[幻觉与验证循环]] 把 ADHD 冲动与 LLM 自信幻觉并置，强调外部核对清单和 harness 验证机制是共同的解方；[[人在回路与身体在场]] 则将 ADHD 的 [[身体在场效应]] 与 LLM 的 human-in-the-loop 监督视为同构的执行护栏。
- **波动与不确定性**：[[采样温度与表现波动]] 把 ADHD 的日内波动映射到 LLM 输出的随机性，说明外部约束对稳定产出的必要性。
- **工具与卸载**：[[工具使用与认知卸载]] 和 [[规划循环与任务分解]] 认为，ADHD 大脑通过外部工具补偿 [[执行功能]] 缺陷，与 LLM 通过 scaffolding 调用工具、分解任务在工程上同构。
- **支架的边界**：[[拐杖与脚手架]] 提醒：harness 是临时支架，若缺乏逐步撤回机制，可能退化为永久拐杖。

## 执行功能：从概念到补偿

ADHD 的核心困难可在 [[执行功能]]、[[执行功能障碍]]、[[任务启动]]、[[超聚焦]]、[[时间盲]]、[[认知负荷]]、[[情绪失调]] 与 [[多巴胺]] 等概念页中理解。这些概念共同说明：问题并非产出能力不足，而是「何时做、做什么、怎么做、做到哪」的调度能力受损。AI 工具的作用，正是把上述调度环节外化，例如：

- **任务启动**：[[Goblin Tools]]、[[Lex]]、[[Perplexity]] 通过微任务分解与可执行步骤降低启动门槛。
- **时间盲**：[[Tiimo]]、[[Structured]]、[[Reclaim.ai]]、[[Motion]] 用视觉时间线与智能时间块把时间流逝变得可感。
- **身体在场**：[[Focusmate]] 用虚拟同伴问责来触发 [[身体在场效应]]，提升启动与专注。
- **情绪与专注**：[[Brain.fm]]、[[Endel]] 提供音景支持；[[Inflow]]、[[Reflect]]、[[Habitica]] 提供结构化的 CBT、反思与多巴胺反馈。
- **外部记忆与搜索**：[[ChatGPT]]、[[Claude]]、[[Mem]]、[[Otter.ai]]、[[Saner.AI]] 充当对话支架和第二大脑，减轻 [[工作记忆]] 负担。

- [[AI 与 ADHD 的职业发展]]：职场困境不是产出不够，而是缺少可靠调度层；AI 工具提供的是职业支架设计语言。

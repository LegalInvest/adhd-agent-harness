# AI 与 ADHD 的学习方式

## 核心论点

当前最站得住脚的核心论点是：「AI 帮 ADHD」本质上是在给 ADHD 大脑的「生成核心」套 harness。ADHD 大脑并非缺乏想法或知识生产能力，而是缺乏一个可靠的内在执行调度层，导致工作记忆、注意力调控、任务启动、时间感知等过程失稳。LLM 同样拥有强大的生成核心，却也是无状态、缺乏稳定执行调度层的系统。因此，为 ADHD 设计 AI 学习工具，与为 LLM 设计 agent scaffolding 是同一类工程：不是替代生成能力，而是补上一个外部化的执行功能层，把意图转化为可管理、可追踪、可启动的行动序列。

[[Perplexity]] 是这一思路的典型落点：它通过把宏大目标分解为可管理的步骤，直接降低了 ADHD 用户的执行功能负担。类似地，日程提醒、可视化时间线、智能拆解、外部记忆库等工具，本质上都是在为 ADHD 大脑补上它缺失的调度层。

## 同构视角

[[ADHD 大脑与 LLM 的同构]]、[[生成核心与调度层]]与[[外部执行功能层]]共同指向一个结构性类比：ADHD 大脑和 LLM 都是「高产但缺执行调度层的生成核心」。这一同构视角至少包含以下对应关系：

- **执行调度层缺失**：ADHD 的执行功能缺陷（[[执行功能]]）对应 LLM 的无状态与缺乏调度层。两者都需要外部脚手架来维持目标导向行为。
- **工作记忆不稳定 / 无状态**：[[工作记忆]]指出 ADHD 的强记忆容量伴随弱控制；[[无状态与外部记忆]]指出 LLM 无法可靠保留跨会话上下文。两者都需要外部记忆系统（如「第二大脑」或向量数据库）来补偿。
- **注意力漂移与上下文膨胀**：[[上下文工程]]提示，ADHD 大脑与 LLM agent 都容易被环境带偏，或因上下文膨胀导致推理退化，因此需要外部框架来结构化执行流程。
- **时间感知与任务启动**：[[时间盲]]描述 ADHD 无法感知时间流逝，[[任务启动]]说明开始行动的困难。AI 工具通过视觉化、智能提醒、微任务分解等方式，把内部时钟和启动信号外化。
- **动机与在场**：[[多巴胺]]与生产力工具的游戏化策略相关；[[身体在场效应]]则把「他人在场」视为一种有效的外部执行支架，可与 AI 协作或替代形成 accountability harness。

[[拐杖与脚手架]]提醒：同构不意味着 ADHD 与 LLM 是同一系统，而是意味着它们的工程化补偿策略可以相互借鉴。支架的目的是能力代偿，而非制造永久依赖。

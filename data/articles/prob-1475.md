# 如果把 ADHD 大脑当成生成核心，Lex 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Lex 对 ADHD 的真正价值。

你有没有过这种体验：坐到桌前明明想写一份报告，三小时后却在研究「明朝火器对朝鲜战争的影响」？或者你正在调试一个 agent，它本该完成「总结客户邮件并生成回复草稿」，却在某一轮把任务漂成了「分析客户公司 2019 年以来的所有财报」？对 ADHD 人群来说，这是分心走神或超聚焦跑偏；对做 Agentic Harness 的工程师来说，这是长程任务里的目标漂移。两种痛苦的结构，可能是同一个。

## 同构：生成核心与缺失的调度层

ADHD 大脑和 LLM 并不是简单的「一个像另一个」。wiki 资料指出，两者在功能上共享一个结构：强大的生成能力，加上不稳定的执行调度层。ADHD 侧的证据来自工作记忆：ADHD 患者在工作记忆任务中易受干扰，成功回忆依赖于对任务无关内容的主动抑制，且随记忆负荷增加表现下降（来源：Human-like Working Memory Interference in Large Language Models）。同一研究也发现 LLM 表现出类似人类的干扰特征。更有神经计算层面的平行：前额叶-纹状体门控机制支撑人类执行功能，而 Transformer 在训练后自注意力机制会发展出模仿该门控的操作（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。

在工程侧，这对应的是 LLM/agent 的「无状态」与上下文膨胀问题。ADHD 大脑每天需要重新回忆项目上下文，否则偏离轨道（来源：无状态与外部记忆）；LLM 则在多轮对话中丢失原始目标，需要外部记忆、system prompt 和人在回路来重锚定。两者都不是「不会生成」，而是「生成方向容易脱离目标函数」。

## 历史 harness：曾国藩的日课就是 system prompt

把人物案例放到这个框架里，曾国藩的 harness 与 LLM 的 re-grounding 惊人同构。他有典型 ADHD 特质：30 岁前浮躁坐不住，读书慢到「他人目下二三行，余或疾读不能终一行」，情绪不稳，日记里天天骂自己「无恒」「浮躁」。他的专属 harness 是「日课十二条」：黎明即起、读书不二、谨言、写日记反省。晚年打仗则「结硬寨打呆仗」，用最笨最稳的方法抵消冲动。

这套系统的工程翻译是：
- **日课十二条** ↔ 每日定时 re-grounding 的 system prompt；
- **写日记** ↔ 外部记忆与执行日志，补偿工作记忆的波动；
- **结硬寨打呆仗** ↔ 保守执行策略（safety-first policy），降低冲动决策带来的任务漂移。

曹操是另一个例子：他把行政全丢给荀彧，用他人的执行功能弥补自己的不足。这相当于在 agent 架构里把规划与调度外包给一个更稳定的「调度器模块」。但曾国藩的日课更贴近 Lex 这类工具想要扮演的角色：不是替你完成，而是不断把生成核心拉回到既定目标。

## Lex 能不能做这个重锚定？

wiki 资料里关于 Lex 的具体功能信息极少，只提醒它容易与播客主持人 Lex Fridman 混淆（来源：全局矛盾与存疑）。因此，我们不能基于现有材料断言 Lex 有什么具体功能。但我们可以从结构出发：如果 Lex 能做到以下三件事，它就有资格成为 ADHD 的「外部执行调度层」：

1. **低摩擦的上下文重载**：像 RescueTime 自动记录时间去向、减少手动追踪的工作记忆负担一样（来源：Revolutionizing ADHD Management with AI Assistants），Lex 需要让用户在每次重新坐下时，一秒内看到「我之前在哪、目标是什么、下一步是什么」。
2. **目标漂移校验**：类似 agent 工程中需要定期对比当前输出与原始 system prompt，Lex 需要能问用户：「你正在做的这件事，和最初设定的目标一致吗？」
3. **外部化而非替代**：真正的脚手架是帮助使用者「建造」，但使用者仍需自己「攀登」（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。如果 Lex 直接替用户生成最终产物，它就变成拐杖；如果它只是把目标、上下文和下一步行动稳定地呈现给用户，它就是脚手架。

目前 wiki 里更成熟的参考工具是 RescueTime、Focusmate、Brain.fm、Endel 和 Forest。RescueTime 通过自动分类、模式识别与分心阻断补偿时间盲和执行功能缺陷（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）；Focusmate 通过虚拟身体在场提供外部启动压力（来源：AI Tools for Kids with ADHD: A Complete Guide for Parents...）；Brain.fm 和 Endel 用 AI 生成专注环境，但缺乏 ADHD 独立临床证据（来源：Brain.fm；Endel）。它们都是 re-grounding 生态的一部分，但各自只解决一个环节。Lex 若要成为更完整的 system prompt，必须把这些环节整合：记忆 + 目标 + 提醒 + 校验。

## 核心判断与争议

我的判断是：**ADHD 大脑与 LLM/agent 的「高产缺调度」同构，使得 harness 工程对两边都成立；Lex 作为候选 re-grounding 工具，价值取决于它能否提供持续、低摩擦、目标导向的上下文重锚，而不是仅仅提供又一个 distraction-free 编辑器。** 但现有材料对 Lex 本身着墨太少，我们无法直接下结论。

wiki 资料也明确提醒我们这一同构的边界：「ADHD 大脑与 LLM 在结构上同构」目前只是理论类比，并非经过验证的科学事实（来源：全局矛盾与存疑）。同时，AI ADHD 工具普遍存在证据不足的问题：Brain.fm 入选 44 款 ADHD 应用榜单，但独立临床证据有限；多数工具依赖用户报告或小样本研究，缺乏大规模随机对照试验（来源：AI 与 ADHD 的专注力）。此外，过度依赖外部工具可能导致脚手架变成拐杖，且 ADHD 亚型差异巨大，同一工具对不同人效果可能迥异（来源：AI 与 ADHD 的专注力）。

## 今天就能试的 4 件事

1. **给自己写一条 system prompt**：像曾国藩的日课一样，每天早晨用一句话写明「今日唯一主线任务」，把它放在电脑桌面或手机锁屏。这是零成本的 re-grounding。
2. **用 RescueTime 或同类工具做一周时间审计**：不评判，只记录。先拿到真实的时间去向数据，再判断自己的漂移模式是什么（来源：Revolutionizing ADHD Management with AI Assistants）。
3. **给 agent 加显式目标校验节点**：在 long-horizon agent 的循环里，每隔 N 轮让 agent 用原 system prompt 做自我一致性检查，必要时把当前状态写回外部记忆。
4. **尝试一个「身体在场」时段**：用 Focusmate 或找个线下同事，把一次容易跑偏的任务放入共同在场的时间盒中，测试外部启动压力是否能降低任务漂移（来源：AI Tools for Kids with ADHD: A Complete Guide for Parents...）。

Lex 能不能当 ADHD 的重锚定？答案不是「能」或「不能」，而是：如果它愿意做那个不断把生成核心拉回目标的 system prompt，它就能。否则，它只是另一个安静的编辑器。

# 给 ADHD 加一道「核对清单/验证步骤」，和给 agent 加 self-critique 循环，是同一种工程吗？

> ADHD 侧：冲动下结论、想当然；LLM/harness 侧：LLM 自信地编造（幻觉），靠验证与自我批判兜底。用双域证据作答。

## 问题：为什么你总在冲动下结论，而 AI 总在自信地编造？

如果你有 ADHD，你一定熟悉这种场景：别人刚说完一句话，你已经给出了结论，甚至开始行动——然后发现漏掉了关键信息。你“想当然”地跳过了验证步骤。与此同时，如果你在构建 LLM agent，你也会遇到类似的困境：模型流畅地输出一段看似合理的代码或回答，但其中包含事实性错误——它“自信地编造”了答案。两个场景的共同点是什么？都是**生成核心（generative core）缺乏内建的验证与调度层**。

ADHD 大脑与大型语言模型在结构上同构：两者都是高产的生成核心，但缺乏可靠的执行调度层（来源：ADHD × AI 的科学与研究前沿）。这意味着，给 ADHD 加一道“核对清单/验证步骤”，和给 agent 加 self-critique 循环，本质上是同一种工程——在生成核心外部套上一个 harness（脚手架），提供验证、调度与纠偏能力。

## 同构：冲动下结论 vs 自信编造

### ADHD 侧：执行功能缺陷导致“想当然”

ADHD 的核心问题不是智力不足，而是执行功能缺陷——大脑的驾驶系统失灵（来源：ADHD × AI 的科学与研究前沿）。执行功能包括计划、组织、工作记忆、抑制控制等。当这些功能薄弱时，大脑会走捷径：快速给出结论，跳过验证步骤。这种“冲动下结论”并非懒惰，而是因为工作记忆容量有限，无法同时 hold 住多个信息片段来进行比较和验证。认知负荷超载时，大脑选择最省力的路径——直接输出直觉判断（来源：认知负荷）。

### LLM/Agent 侧：无状态性与幻觉

LLM 同样没有内建的验证机制。它根据上下文 token 预测下一个 token，生成流畅文本，但无法判断自己是否正确。这种“自信地编造”被称为幻觉。Agent 在缺乏编排层（orchestration layer）时，会盲目执行任务，甚至重复错误步骤。LLM 的无状态性对应 ADHD 的工作记忆缺陷；上下文窗口溢出对应 ADHD 的上下文丢失（来源：ADHD × AI 的科学与研究前沿）。两者都需要外部系统来提供“第二遍检查”。

## 证据：两边的解决方案结构同构

### ADHD 侧的核对清单与任务分解

Goblin Tools 的核心功能是 Magic ToDo：将模糊任务分解为具体步骤（来源：Goblin Tools）。这相当于在行动前插入一个“验证与规划”步骤——把“整理房间”变成“捡起衣服、擦桌子”等可执行的子任务。用户反馈说它“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools）。Motion 则更进一步：自动排程并动态调整日程，消除“下一步该做什么”的决策负担（来源：Motion）。这些工具都在做同一件事：在冲动行动之前，强制插入一个结构化验证层。

### LLM/Agent 侧的 self-critique 与任务分解

在 agent 工程中，self-critique 循环是标准做法：模型生成输出后，由另一个 prompt 或 agent 检查其正确性，必要时重生成。任务分解技术（如 chain-of-thought、plan-and-execute）则通过将复杂任务拆分为子任务，减轻 LLM 的认知负荷，减少幻觉（来源：认知负荷）。这与 Goblin Tools 的任务分解完全同构——都是将认知负担外化给系统，降低内部处理压力。

### 关键证据：认知负荷的双域证据

认知负荷概念在 ADHD 和 LLM 领域同时成立。对于 ADHD，AI 工具通过自动决策和自动执行来“移除决策带来的认知负荷”（来源：认知负荷）。对于 LLM，任务分解技术“减轻大型语言模型的认知负荷，减少幻觉等错误”（来源：认知负荷）。这直接证明了两边共享同一个工程原理：通过外部脚手架降低内部处理压力，从而减少错误。

## 核心判断：脚手架 vs 拐杖——边界在哪里？

我的核心观点是：**有效的 harness 必须是可逐步撤除的脚手架，而非永久依赖的拐杖**。但现有工具普遍缺乏撤除机制。Goblin Tools、Saner.AI、Motion 等设计为长期使用，未提及如何逐步减少依赖（来源：矛盾与存疑）。这带来风险：长期依赖外部验证是否会削弱内在执行功能？目前缺乏纵向研究（来源：ADHD × AI 的科学与研究前沿）。

对于 agent 工程，同样存在类似问题：过度依赖 self-critique 循环可能导致模型不再学习自我纠错，而是依赖外部检查。好的 harness 应该像训练轮——最终要拆掉，让模型（或大脑）内化验证能力。

## 诚实局限

- **同构命题证据不足**：ADHD 大脑与 LLM 同构主要基于概念类比和工具案例，缺乏大规模实证（来源：矛盾与存疑）。神经影像学和行为实验尚未直接验证。
- **工具效果缺乏临床研究**：Goblin Tools、Motion 等工具的证据主要来自用户反馈，缺少随机对照试验（来源：Goblin Tools、Motion）。
- **多巴胺干预争议**：部分工具声称基于多巴胺机制，但多巴胺干预的有效性存在争议（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个让你拖延的任务**：输入“写报告”或“整理房间”，看 AI 如何拆解成小步骤。注意观察：分解后启动焦虑是否降低？
2. **在 ChatGPT 或 Claude 中尝试 self-critique prompt**：生成回答后，追加指令“请检查上述回答中的事实错误，并修正”。对比修正前后的质量差异。
3. **设置一个“核对清单”习惯**：在做出重要决定前，强制写下三个验证问题（例如：我是否遗漏了关键信息？这个结论有证据支持吗？）。坚持一周，记录冲动下结论的次数变化。
4. **尝试 Motion 或 Reclaim.ai 自动排程**：将本周所有任务输入，让 AI 自动生成日程。体验“无需决策下一步”的状态，并评估效率变化。

## 结语

给 ADHD 加核对清单，和给 agent 加 self-critique 循环，不仅是同一种工程，更是同一种哲学：承认生成核心的局限性，用外部系统补偿。无论是人类大脑还是 AI 模型，我们都需要在自信与验证之间找到平衡。而 harness 的最终目标，不是永远依赖，而是让核心学会自己踩刹车。

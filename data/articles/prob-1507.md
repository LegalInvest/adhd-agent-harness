# Routinery 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Routinery：它到底补上了哪一层执行功能？

## 问题：ADHD 与 LLM 共享的同一场噩梦

你打开电脑，准备写一份报告。十分钟后，你发现自己正在研究 18 世纪帆船的结构，或者刷完了三个小时的短视频。这不是意志力问题——你的大脑是一个强大的生成核心，但缺少一个内置的执行调度层。ADHD 个体的表现波动极大，注意力、执行功能在不同时间点差异显著（来源：采样温度与表现波动）。这种波动，就像 LLM 在高采样温度下的输出：创造力爆棚，但不可靠。

LLM agent 在长程任务中同样面临目标漂移。随着上下文膨胀，模型会逐渐偏离最初的指令，产生小错误并累积，非确定性使可重复性复杂化（来源：AI Agent Systems: Architectures, Applications, and Evaluation）。工程实践因此强调“确定性状态转换”和“计划-执行”分离，以强制单向、确定性的控制流（来源：AI Agents in 2026: Tools, Memory, Evals, and Guardrails）。

ADHD 大脑与 LLM 是同构的：两者都是高产但缺调度层的生成核心。那么，解法是否也同构？

## 答案：重锚定 System Prompt

对于 LLM agent，最直接的防漂移手段是定期重读 system prompt——即“重锚定”。在 agent 架构中，这体现为周期性采样以监控质量（来源：AI Agent Reliability and Guardrails 2026），或使用知识图谱实现确定性精度（来源：AI Agent Memory Architectures）。

对于 ADHD 大脑，外部结构就是 system prompt。Routinery 这类工具，本质上是定时触发的重锚定机制：它每到一个时段，就提醒你“现在该做什么”，将你从超聚焦或分心中拉回目标轨道。曾国藩的“日课十二条”就是一套手工重锚定系统：黎明即起、读书不二、谨言、写日记反省——每天用固定的行为序列对抗自己的“无恒”与“浮躁”。他一生写日记骂自己，每天反省，这正是最朴素的 system prompt 刷新。

同样，王阳明以“致良知”为唯一决策标准，通过静坐反省和“事上练”来保持目标一致。他的 harness 不是外部日程，而是一个内化的、但需要主动调用的重锚定机制。

Routinery 的独特价值在于：它不只是一个日程表，而是一个可配置的“重锚定 prompt”。你设定好早间流程、工作块、晚间复盘，它就在固定时间跳出来，告诉你：“现在，你的 system prompt 是：专注写报告。”

## 但有人用错了

许多人把 Routinery 当成拐杖：设置好流程后，期望它自动执行一切。但 ADHD 大脑不是 LLM——它不会被动接受 prompt 就自动输出。工具只是脚手架，不是替代品。如果过度依赖，反而会削弱内化的执行功能（来源：拐杖与脚手架）。

真正的用法是：把 Routinery 当作外部调度器，但每次提醒后，你仍需主动选择执行。就像曾国藩写日记，不是写了就改，而是写完后还要“痛自刻责”。

## 证据与局限

现有 AI 工具对 ADHD 的支持，本质上都是在补偿工作记忆不稳定、任务启动困难和注意力分散（来源：AI 与 ADHD 的专注力）。Brain.fm 通过神经锁相技术影响大脑处理信息的方式（来源：Brain.fm），Focusmate 通过虚拟身体在场提供外部社会压力（来源：Focusmate），RescueTime 自动追踪时间并阻断分心（来源：RescueTime）。但这些工具的证据多来自用户报告或小样本研究，缺乏大规模随机对照试验（来源：AI 与 ADHD 的专注力）。

争议在于：AI 工具是否会导致永久依赖而无法内化执行功能？同构性是否只是功能类比，而非深层神经机制对应？（来源：矛盾与存疑）

## 今天就能试的行动

1. **设置三个重锚定时刻**：在 Routinery（或任何提醒工具）中，设置早、中、晚三个固定“system prompt 刷新”时段。每个时段花 2 分钟，写下当前最重要的一个目标。
2. **用 Focusmate 做一次身体加倍**：预约一个 25 分钟时段，和陌生人视频共同工作。这相当于让外部调度器临时接管你的执行层。
3. **尝试一次“低温度”写作**：用 Brain.fm 的专注音乐，强制自己写 15 分钟，不修改、不切换窗口。这是人为降低采样温度，减少输出随机性。
4. **复盘你的“目标漂移”**：用 RescueTime 看一天的时间去向，找出最常偏离的“高温度”时刻，然后在那前后插入一个重锚定提醒。

# 用 Claude 给 ADHD 补上外部执行功能层，是脚手架还是拐杖？

> Claude 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：当你的大脑是个天才生成器，却配了个故障调度器

如果你是一个 ADHD 成人，你一定熟悉这种体验：脑子里同时冒出十个创意，但一个都启动不了；明明知道截止日期，却像被冻住一样刷手机两小时。这不是懒，是执行功能——大脑的“驾驶座”——出了问题（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

如果你是一个 Agent 工程师，你大概也熟悉这种体验：LLM 输出洋洋洒洒，但跑着跑着就偏离轨道、陷入循环，或者忘记之前说了什么。模型本身是强大的生成核心，但缺乏可靠的执行调度层（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。

这两类人面对的是同一个结构性问题：**高产生成核心 + 缺失的执行调度层**。

而 Claude 这类 LLM，恰好可以充当这个缺失的调度层——问题在于：它是帮你学会走路（脚手架），还是让你永远离不开它（拐杖）？

## 同构脊柱：ADHD 大脑与 LLM 是同一类“高产但缺调度”的生成核心

先看 ADHD 侧：执行功能缺陷导致计划、组织、工作记忆、任务启动困难（来源：AI for ADHD: Best Tools, Apps, and Strategies - Themba Tutors）。工作记忆像一块随时会清空的便签纸，时间盲让你无法感知“半小时后”意味着什么，任务启动需要巨大的心理能量。

再看 LLM/Agent 侧：LLM 本身无状态，上下文窗口有限，输出不可控。Agent harness 的作用就是提供编排、记忆、工具层和人在回路监督（来源：What is an agent harness in the context of large-language models?）。

两者完全同构：ADHD 需要外部执行功能层（任务分解、提醒、时间感知），LLM 需要 agent harness（护栏、检查点、上下文管理）。

所以，用 Claude 给 ADHD 补上外部执行功能层，本质上是为你的生成核心套上 harness——就像工程师为 LLM 套上 agent harness 一样。

## 证据：哪些工具已经在做这件事，效果如何

### ADHD 侧的真实工具

- **Goblin Tools**：将“整理房间”自动分解成“捡起衣服”“擦桌子”等小步骤（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）。用户反馈“将压倒性的事情变成一系列不压倒性的事情”（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...)）。
- **Saner.AI**：通过本地记忆和知识回忆减少搜索循环，直接补偿工作记忆不足（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。
- **Motion**：自动排程并动态调整日程，消除“下一步该做什么”的决策负担（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。
- **Focusmate**：AI 匹配虚拟身体搭档，利用“身体在场效应”绕过执行功能屏障（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...)）。

这些工具的共同点：它们不替代你的大脑，而是提供外部记忆、外部任务分解、外部时间感知——即外部执行功能层。

### LLM/Agent 侧的对应实践

Agent harness 通过形式化规划和护栏，使代理输出真正有用且正确（来源：What is an agent harness in the context of large-language models?）。具体做法包括：设置 token 预算、工具调用次数上限、加入人工检查点（暂停并询问后再执行）（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。

这与 ADHD 工具的设计逻辑完全一致：Goblin Tools 的分解相当于 Agent 的任务规划，Motion 的动态调整相当于 Agent 的实时重排，Focusmate 的虚拟搭档相当于 Agent 的 human-in-the-loop 监督。

## 核心判断：脚手架还是拐杖？边界在于“人在回路”与“可撤除性”

我的观点是：**Claude 作为外部执行功能层，本质是脚手架，但很容易滑向拐杖——关键在于设计是否保留人在回路，以及是否内置撤除机制。**

**脚手架 vs 拐杖的边界：**
- **脚手架**：帮你完成你暂时做不到的事，同时让你在过程中学习。例如，Goblin Tools 帮你分解任务，但你仍然要执行每一步。
- **拐杖**：完全替代你的功能，让你不再使用自己的执行功能。例如，AI 帮你写完邮件，你连思路都不需要过一遍。

当前多数工具（如 Goblin Tools、Saner.AI、Motion）设计为长期使用，未提及撤除机制（来源：拐杖与脚手架概念页）。这存在依赖风险：长期使用 AI 工具是否会削弱内在执行功能？目前缺乏纵向研究（来源：ADHD × AI 的科学与研究前沿）。

**人在回路是关键**：就像 Agent harness 需要人工检查点一样，ADHD 用户使用 AI 工具时也应保留主动决策权。例如，用 Claude 分解任务后，自己选择先做哪一步；用 Motion 排程后，手动调整优先级。

Claude 不是来接管你的大脑的。它是你的 agent harness——帮你把发散的天才想法变成可执行的行动。但记住，最好的脚手架是那些你最终可以拆掉的。

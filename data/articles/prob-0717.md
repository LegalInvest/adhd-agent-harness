# Perplexity 之于 ADHD，就像 外部记忆/向量库 之于 LLM——但有人用错了

> 从同构视角实测 Perplexity：它到底补上了哪一层执行功能？

## 同一个问题，两种困境

你正写着一份方案，忽然想起要查一个数据。打开浏览器，输入关键词，跳出一堆链接——你点开一个，读了两行，又看到另一个相关链接，再点开……二十分钟后，你发现自己正刷着某篇与方案毫无关系的文章，而最初要查的数据早已被遗忘。

这是 ADHD 大脑的日常：工作记忆容量小，边做边忘（来源：ADHD 的 AI 工具全景）。

与此同时，一个 LLM 驱动的 agent 也在经历类似的困境：它没有跨会话状态，每次交互都是“失忆”的。为了让它完成多步任务，工程师必须给它挂载外部记忆——向量数据库、对话历史缓存、工具调用记录——否则它就会在第三步忘记第一步的目标。

这两个场景，本质上是同一个问题：**一个高产的生成核心，缺少可靠的内置执行调度层**。ADHD 大脑与 LLM 在结构上同构——两者都是强大的生成核心，但缺乏可靠的内置执行调度层（来源：ADHD 的 AI 工具全景）。

## Perplexity：被误用的外部记忆

Perplexity 常被当作“AI 搜索引擎”来推广：输入问题，给出带引用的答案。但若仅止于此，它和 Google 的区别不过是“少点几次链接”。

真正让 Perplexity 对 ADHD 有价值的是它的**上下文延续能力**。你可以在同一个会话中连续追问，Perplexity 会记住你之前的问题和它的回答。这听起来微不足道，但对工作记忆容量小的人来说，这意味着：你不需要在脑中同时 hold 住“原始问题 + 中间结论 + 下一步追问”。Perplexity 替你 hold 住了中间状态。

这正是 LLM/agent 工程中“外部记忆”的职责。一个 agent 如果每步推理都从零开始，很快就会在长任务中迷失；给它挂上向量库或缓存，它才能保持“方向感”。Perplexity 对 ADHD 的作用完全同构：它补偿了工作记忆的不稳定，让用户不必靠大脑维持上下文。

但有人用错了。他们把 Perplexity 当成普通搜索引擎，每次打开新会话，问完就走。这就像给 LLM 装了一个向量库，但每次请求都清空它——等于没装。**Perplexity 的价值不在于“搜得快”，而在于“记得住”。**

## 人物案例：孔子的“日课”就是他的外部记忆系统

孔子是 ADHD 特质的典型：精力旺盛，周游列国 14 年坐不住；冲动爱骂人，见南子急得对天发誓；思维跳跃，《论语》全是场景化语录，无系统著作。他的专属 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。

这套“日课”本质上是一个**外部调度器**。它不依赖孔子的工作记忆去记住“今天该做什么”，而是用固定的仪式（礼）和定时反省（三省）来重新校准行为。对应到 LLM/agent 工程，这就是**定时 re-grounding**：agent 每执行几步就回顾一次原始目标，防止漂移。孔子 70 岁才做到“从心所欲不逾矩”，说明他一生都在和冲动与遗忘做斗争——这和 LLM 需要持续的外部上下文刷新才能保持一致性，是同一个道理。

## 其他工具的脚手架逻辑

理解了“外部记忆”这个同构点，再看 ADHD 工具，就能看清它们的本质：

- **Goblin Tools** 的 Magic ToDo 将任务分解为微步骤，本质上是把“任务目标”拆成外部可读的步骤列表，减轻工作记忆负担（来源：Goblin Tools）。
- **Saner.AI** 通过本地记忆存储和快速检索，减少搜索循环——这相当于给用户装了个人向量数据库（来源：Saner.AI）。
- **Lex** 允许“单一指令触发多步骤任务”，相当于把 agent 的规划器暴露给用户（来源：Lex）。

这些工具的共同逻辑是：**把本应由大脑内部执行的功能——记忆、规划、调度——外包给外部系统**。这和 LLM/agent 工程中“给模型装脚手架”的做法完全一致。

## 脚手架 vs 拐杖：边界在哪里？

但这里有一个必须指出的争议：外部记忆系统会不会让人变得更依赖，从而削弱内在的执行功能？

“拐杖与脚手架”概念提醒我们，AI 工具可能从脚手架变为永久拐杖（来源：ADHD 的 AI 工具全景）。如果一个人离开 Perplexity 就完全无法组织思路，那工具就成了拐杖。但如果 Perplexity 帮助他完成了原本完不成的任务，并在这个过程中让他学会了“如何追问”的元技能，那它就是脚手架。

关键在于：**工具是否最终增强了用户的能力，还是替代了它。** 好的脚手架应该是可逐步撤除的——就像孔子到了七十岁，礼已经内化为习惯，不再需要刻意提醒。

另外，现有工具的证据基础仍然薄弱。多数工具缺乏严格临床验证，效果多基于用户报告（来源：ADHD 的 AI 工具全景）。同构性理论本身也缺乏实证基础，目前仅是一种理论类比（来源：矛盾与存疑）。我们不应过度推广这个框架，但它提供了一个有用的思考角度。

## 今天就能试的行动

1. **把 Perplexity 当对话伙伴，而非搜索引擎**：下次查一个复杂问题，不要问完就关。连续追问 3-5 轮，让 Perplexity 帮你“记住”上下文。观察你能否更顺畅地形成完整思路。

2. **设置一个“日省”提醒**：像孔子一样，在手机或日历上设一个每天固定时间的提醒，问自己：“我今天的主要目标是什么？我现在在做什么？”——这是最简单的 re-grounding 实践。

3. **试用 Goblin Tools 的 Magic ToDo**：把一个让你拖延的任务（比如“清理厨房”）输入进去，调节“辣度”直到分解出可执行的子步骤。对比不用工具时，你启动任务的速度是否有变化。

4. **为你的 LLM agent 显式添加外部记忆**：如果你在用任何 agent 框架（如 LangChain、AutoGPT），检查它是否默认有记忆组件。如果没有，尝试挂载一个简单的对话缓存或向量库。观察 agent 在多步任务中的表现提升。

## 结语

Perplexity 之于 ADHD，就像向量库之于 LLM——它补上了那个“记不住”的缺口。但工具只是工具，真正的价值在于我们是否理解了它的同构本质：**外部记忆不是偷懒，而是对大脑短板的诚实补偿。** 用对了，它是脚手架；用错了，它只是更快的搜索引擎。

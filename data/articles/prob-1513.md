# Todoist 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Todoist：它到底补上了哪一层执行功能？

## 问题：目标漂移，ADHD 与 LLM 共同的敌人

你有没有过这样的经历：打开电脑准备写一份报告，结果半小时后发现自己正刷着某个视频网站，完全忘了最初的目标？或者，你明明想查一个资料，却在超聚焦中把维基百科的几十个词条翻了个遍，最后连最初的搜索词都记不清了？

这不是你的错。ADHD 大脑与 LLM 在结构上同构：两者都是强大的生成核心，但缺乏可靠的内置执行调度层（来源：AI 与 ADHD 的专注力）。LLM 在长程任务中会因上下文膨胀而目标漂移，ADHD 大脑则因工作记忆不稳定和执行功能缺陷而频繁脱轨。两者的解法也同构：都需要一个外部脚手架来重锚定目标。

## 同构脊柱：重锚定与目标漂移

ADHD 大脑的痛点——分心走神或超聚焦跑偏、丢失目标——在 LLM/agent 工程中有一个精确对应：长程任务里 agent 会目标漂移，需要重锚定 system prompt。

让我们看看真实世界中的例子。曾国藩，晚清名臣，一生与注意力不集中搏斗。他30岁前浮躁坐不住，天天串门喝酒看杀人，日记里骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”。他的解决方案是“日课十二条”：黎明即起、读书不二、谨言、写日记反省。这套系统本质上是一个外部调度层：每天定时 re-grounding，用“结硬寨打呆仗”的笨方法抵消冲动（来源：人物案例）。

用同构视角看，曾国藩的“日课”相当于 LLM 的定时 re-grounding prompt；他的“写日记反省”相当于 agent 的 log 与复盘机制；而他的“秘书团队”则相当于外部调度器。曾国藩不是靠意志力硬扛，而是靠一套精心设计的脚手架来补偿执行功能缺陷。

## Todoist 补上哪一层？

Todoist 之于 ADHD，就像重锚定 system prompt 之于 LLM——它补上的是**执行调度层**。具体来说：

- **任务分解**：ADHD 大脑难以将模糊目标拆解为可执行步骤，Todoist 的“子任务”和“项目”功能提供了结构。这对应 LLM 的规划循环——AgentGen 等框架通过环境与任务生成增强 agent 的规划能力（来源：规划循环与任务分解）。
- **重锚定**：Todoist 的“今日任务”视图和“优先级”标记，相当于每天一次的 system prompt 重锚定。曾国藩的“日课十二条”也是每日重锚定。
- **外部记忆**：ADHD 的工作记忆缺陷与 LLM 的无跨会话状态同构，两者都依赖外部记忆系统。RescueTime 自动记录时间去向，补偿“时间盲”（来源：RescueTime）。

但问题在于：**有人把 Todoist 用成了拐杖，而不是脚手架**。

## 脚手架 vs 拐杖：关键的边界

脚手架和拐杖的区别在于：脚手架是暂时辅助，最终目标是让系统（或人）学会独立运行；拐杖则是永久替代，一旦撤掉就瘫痪。

在 LLM/agent 工程中，一个设计良好的 harness 应该逐步减少干预，让 agent 学会在少量 prompt 下保持目标。同样，ADHD 用户使用 Todoist 时，如果只是机械地添加任务却从不回顾、从不调整，工具就会变成拐杖——你只是把混乱从大脑转移到了列表，而没有真正内化规划能力。

证据显示，ADHD 专业人士寻找的 AI 工具需满足“与现有工具集成，不增加认知开销”（来源：工具使用与认知卸载）。这意味着工具应该无缝嵌入日常流程，而不是增加额外的执行负担。Focusmate 通过虚拟身体在场提供外部结构和社会压力，弥补内在多巴胺调节不足（来源：Focusmate）。但若用户永远依赖他人监督才能启动，那就是拐杖。

## 争议与局限

我们必须诚实指出：ADHD 大脑与 LLM 的同构目前仅是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。多个页面在表述时有时将其作为既定事实，有时作为假设，存在不一致。

此外，工具宣称的证据不足。Brain.fm 被列为最佳 ADHD 应用之一，但缺乏针对 ADHD 的独立临床研究（来源：Brain.fm）。RescueTime 的自动分类可能不够精准，对执行功能较弱的用户仍是挑战（来源：RescueTime）。

依赖风险与益处存在矛盾：多个工具页面仅强调益处，未充分讨论依赖风险。我们需警惕：工具既可能成为永久依赖的拐杖，也可能成为逐步内化的脚手架（来源：矛盾与存疑）。

## 今天就能试的行动

1. **每日一次“重锚定”**：每天早晨用 Todoist 或类似工具，花 5 分钟审查今日任务，只保留 3 个最重要的。这相当于给 LLM 重置 system prompt。
2. **身体加倍**：尝试 Focusmate 或找一个“虚拟同伴”，每周 3 次，每次 25 分钟。这相当于 agent 的 human-in-the-loop 验证。
3. **时间审计**：用 RescueTime 或手动记录一周的时间去向，识别“目标漂移”的高发时段。然后在这些时段设置阻断或提醒。
4. **反思日记**：每晚花 2 分钟写一句话：今天最重要的目标是什么？我偏离了吗？这相当于 agent 的 log 复盘。

## 结语

Todoist 不是一个神奇的工具，它只是一个外部调度层。真正的改变来自你如何用它来重锚定自己的目标。曾国藩靠日课十二条成为“半个圣人”，不是因为他意志力超群，而是因为他设计了一套与自己大脑同构的脚手架。

你的大脑不是坏的，它只是缺一个调度层。给它一个脚手架，而不是一根拐杖。

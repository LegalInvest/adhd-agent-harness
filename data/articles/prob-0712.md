# Lex 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Lex：它到底补上了哪一层执行功能？

## 问题：为什么「一键执行」反而让你更焦虑？

如果你有 ADHD，你一定经历过这样的时刻：打开 Lex，输入「写周报」，它自动拆出10个子任务——收集数据、整理亮点、分析问题……你盯着这个清单，却不想点任何一个。你明明需要它降低启动门槛，结果它把门槛变成了台阶。

如果你在做 Agentic Harness 工程，你一定见过类似的场景：你给 LLM 配了 function calling，它调用计算器、查数据库、发邮件，一气呵成——但当你把任务从「计算1+1」换成「规划一次旅行」，它要么陷入循环调用，要么输出一堆无关信息。你发现，工具调用并没有让 agent 更聪明，只是让它更快地跑偏。

这两个问题共享同一个根源：**工具被用成了「拐杖」，而不是「脚手架」**。Lex 之于 ADHD，就像 function calling 之于 LLM——都是通过外部调度层把高产生成核心约束到目标轨道上。但很多人用错了：他们试图用工具替代大脑的调度功能，而不是增强它。

## 同构：ADHD 大脑与 LLM 共享一个「生成-调度」断层

ADHD 大脑常被描述为一个高产的生成核心——想法丰富、联想活跃，但缺少稳定、可靠的调度层来筛选、组织、启动和坚持任务（来源：ADHD 的 AI 工具全景）。LLM 同样具备强大的生成能力，却在长程上下文、目标保持与执行调度上存在天然缺陷。两者都需要外部脚手架：ADHD 需要「执行功能层」来补偿工作记忆不稳定、时间盲和任务启动困难；LLM/agent 需要 harness 来将复杂目标分解为可管理的步骤，并通过工具调用精确执行。

这种同构不是空洞类比。在 ADHD 侧，真实人物的 harness 系统提供了证据。孔子——身高1米9，精力旺盛，周游列国14年坐不住，冲动骂人，对种地零耐心——他以「克己复礼」为核心，用外在的「礼」作为行为边界，每日反省（「吾日三省吾身」）。这套系统本质上是一个外部调度层：**日课**相当于定时 re-grounding，**礼**相当于行为约束的 API 接口。70岁才「从心所欲不逾矩」，说明他一生都在用脚手架管理自己的生成核心。

在 LLM/agent 侧，证据来自工程实践。Agent scaffolding 的核心模式是：将复杂目标分解为一系列步骤，通过 function calling 将精确计算外包给工具，并引入规划机制（来源：Lex 工具页的底层架构描述）。这与孔子的「反省-礼-从心所欲」序列完全同构：反省是状态检查，礼是工具调用，从心所欲是目标达成。

## Lex 的真相：它补上了「任务启动」这一层，但容易变成拐杖

Lex 的核心设计是「单一指令触发复杂、多步骤的任务序列」（来源：Lex 工具页）。对于 ADHD 用户，这直接补偿了任务启动困难——你不需要自己拆解任务，AI 帮你拆。对于 agent 工程师，这相当于一个预定义的 planning pipeline：输入目标，输出子任务序列，按序执行。

但问题在于：**Lex 的「一键执行」可能绕过而不是训练你的调度能力**。wiki 资料明确指出了这个矛盾：AI 工具是「外部执行功能层」还是「拐杖」？过度依赖可能导致能力退化（来源：矛盾与存疑）。同样，在 agent 工程中，如果 agent 只依赖预定义的工具调用链而不具备动态重规划能力，遇到边界情况就会崩溃。

真正的价值不在于自动执行，而在于 **将调度过程透明化**。Goblin Tools 的 Magic ToDo 允许用户调节「辣度」控制分解粒度（来源：Goblin Tools 页），这就是一种脚手架：它展示拆解逻辑，让你学会如何自己拆。Saner.AI 通过本地记忆减少搜索循环（来源：Saner.AI 页），同样是在训练你的外部记忆系统。而 Lex 如果只提供黑箱式执行，就错过了这个教育机会。

## 身体在场效应：被忽视的「社会性调度层」

另一个同构点值得注意：身体在场效应（Body Doubling）——有他人在场时，即使不互动，也能提升专注度（来源：身体在场效应页）。这相当于 agent 的「外部监控」：一个观察者（人或系统）通过隐性问责降低分心阈值。

AI 工具如 Focusmate 用算法匹配虚拟伙伴（来源：身体在场效应页），而 agent 工程中的 logging 和 monitoring 系统扮演了类似角色。但两类用户都容易忽略：**身体在场不是替代自律，而是提供反馈回路**。ADHD 用户需要定期问责检查（可将目标达成可能性从25%提升至95%，来源：身体在场效应页），agent 需要状态监控和异常检测。两者都需要显式的「检查点」机制。

## 争议与局限

目前，ADHD 大脑与 LLM 同构的论点「多为类比推理，缺乏神经科学或计算机科学的直接证据」（来源：矛盾与存疑）。此外，AI 工具对任务启动的长期效果未知，「短期有效，但长期效果未知」（来源：矛盾与存疑）。对于 Lex，直接研究证据有限，其设计理念与 ADHD 干预策略一致，但缺乏实证（来源：Lex 工具页）。

## 今天就能试的 3 件事

1. **把 Lex 当成教学工具，而非自动完成**：用 Lex 拆解一个你熟悉的重复任务（如「整理书桌」），然后对比 AI 的步骤和你自己的步骤，找出差异。这能训练你的拆解能力。
2. **设置一个「身体在场伙伴」**：用 Focusmate 或找一位朋友，每天固定时间一起工作15分钟。记录专注度的变化。
3. **为你的 agent 添加显式检查点**：在 function calling 链中插入状态日志，让 agent 在每步执行后输出「当前目标、已完成、待办」。这相当于孔子的「三省吾身」。

## 结语

Lex 不是拐杖，是脚手架。Function calling 不是替代思考，是延伸思考。用好它们的关键，是理解同构的本质：**生成核心需要调度层，而最好的调度层是透明的、可学习的、可调整的**。孔子用了一辈子才「从心所欲」，我们不必等那么久——但也不能指望一键到达。

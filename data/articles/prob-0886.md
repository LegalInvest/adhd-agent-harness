# Routinery 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Routinery：它到底补上了哪一层执行功能？

## 问题：ADHD 的任务启动困难 vs LLM 的规划崩溃，本质是同一类缺陷

如果你是一名 ADHD 患者，你一定经历过这样的场景：脑子里有无数想法，但就是无法开始做那件最重要的事。如果你是一名 AI 工程师，你一定也见过这样的场景：一个 LLM agent 能输出流畅的段落，但一旦要求它拆解一个长任务，它就会在子步骤间迷失，忘记上下文，或者重复执行同一个子目标。

这两种困境在结构上是同构的——ADHD 大脑与 LLM 都是“高产但缺执行调度层的生成核心”（来源：ADHD 大脑与 LLM 的同构）。ADHD 的执行功能缺陷（计划、组织、工作记忆、冲动抑制）与 LLM 的状态保持、真实规划、上下文控制缺陷一一对应（来源：执行功能、工作记忆）。因此，ADHD 需要的“外部执行功能层”与 LLM agent 需要的“scaffolding”是同一类工程（来源：拐杖与脚手架）。

但问题在于：很多人用错了工具。他们把 Routinery 这类应用当作“时间填充器”，而不是“决策卸载器”。

## 同构：planner-executor 循环与 ADHD 的日课

在 LLM agent 架构中，一个常见的模式是 planner-executor 循环：planner 将长任务分解为子目标，executor 依次执行，然后 planner 根据反馈调整下一步。这正是 ADHD 大脑所缺失的能力——ADHD 大脑可以 hyperfocus 在某个任务上数小时，但无法自主决定“现在该做什么、做多久、何时切换”（来源：超聚焦、时间盲）。

历史上有两位人物用自己的方式搭建了这样的 harness。孔子，极可能患有 ADHD——他精力旺盛到周游列国14年，冲动爱骂人（骂宰予“朽木不可雕”），但对音乐能超聚焦到“三月不知肉味”。他的 harness 是“克己复礼”：用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”）。这正对应了 LLM agent 中的“外部调度器”——不是靠意志力，而是靠一套预设的规则和定时 re-grounding。

另一位是文天祥。他21岁中状元，却散尽家财招兵抗元，明知不可为而为之。他的 harness 是《正气歌》——每天读圣贤书，用儒家的“仁”管理生死。这相当于 LLM 的“系统提示”（system prompt）：一个不变的、高层的价值约束，在每一步执行前重新激活。

这两个案例的共同点是：harness 不是“告诉你做什么”，而是“帮你决定做什么的框架”。这正是 AI 时间管理工具应该做的，但很多工具没做到。

## 工具实测：谁补上了决策层，谁只是填充了时间

目前主流的 AI 时间管理工具可以分为两类。

**第一类：决策卸载器**。代表是 Motion 和 Reclaim.ai。Motion 自动排程并动态调整，它“持续评估任务优先级、截止日期和可用时间，实时重建日程”（来源：Motion）。Reclaim.ai 则更强调“保护”时间块，自动防御会议侵占（来源：Reclaim.ai）。这两者的核心价值不是“帮你记住”，而是“帮你决定”——它们移除了“下一步该做什么”的决策负担，直接缓解任务启动困难（来源：Motion）。

**第二类：视觉化支架**。代表是 Tiimo 和 Structured。Tiimo 将时间转化为可见元素，用颜色和结构帮助用户感知时间流逝（来源：Tiimo）。它对时间盲有效，但并未直接处理决策问题。Tiimo 属于自适应规划器，能自动重新安排日程（来源：Tiimo），但它的介入程度不如 Motion。

问题在于：很多人把 Routinery 这类应用当作“时间填充器”——设定好时间段，然后机械执行。但 ADHD 大脑的核心缺陷不是“不知道时间”，而是“无法在多个选项中选择一个并启动”。如果工具只填充时间而不卸载决策，它就成了拐杖——你依赖它告诉你“现在做什么”，但一旦工具失效，你仍然无法自主启动。

## 争议与局限：脚手架还是拐杖？

这引出了核心争议：AI 工具是“外部执行功能层”还是“永久拐杖”？

一方观点认为，AI 作为外部执行功能层补偿缺陷，就像眼镜补偿视力（来源：外部执行功能层）。另一方担心，过度依赖会导致能力退化。目前缺乏纵向研究来验证长期效果（来源：矛盾与存疑）。

另一个争议是：超聚焦应该被引导还是打断？部分 ADHD 用户反馈，强行打断（如番茄钟）反而有效，而柔性引导可能不够强力（来源：矛盾与存疑）。这提示我们，工具需要个性化，而不是一刀切。

我的判断是：**脚手架 vs 拐杖的边界在于，工具是否保留了用户的自主决策权**。一个好的脚手架应该让用户理解“为什么现在做这个”，而不是仅仅“现在做这个”。Motion 的自动排程如果完全黑箱，就是拐杖；但如果它提供优先级逻辑的透明解释，就是脚手架。

## 今天就能试的行动

1. **用 Motion 或 Reclaim.ai 替代纯待办清单**：至少试用一周，观察它是否减少了你的“任务启动焦虑”。重点不是它排得准不准，而是你是否感觉“不用自己想了”。
2. **为你的 LLM agent 添加显式的 planner 模块**：如果你在构建 agent，不要只给一个 prompt，而是实现一个 planner-executor 循环，让 planner 每步输出“下一步做什么”，并记录上下文状态。
3. **模仿孔子的“日三省吾身”**：每天设定三个固定时间点（如早、中、晚），用 2 分钟问自己：“我现在应该做什么？为什么？”这相当于定时 re-grounding，能让你的执行功能恢复锚点。
4. **测试你的工具是脚手架还是拐杖**：关闭工具一天，看自己是否还能启动任务。如果能，说明工具是脚手架；如果不能，说明你需要调整使用方式——比如从“完全依赖”改为“参考建议”。

## 总结

Routinery 之于 ADHD，就像 planner-executor 任务分解之于 LLM。两者都是为高产生成核心套上外部调度层。但工具的设计哲学决定了它成为脚手架还是拐杖：脚手架卸载决策但保留理解，拐杖替代决策但剥夺能力。选择工具时，问自己：它是在帮我决定，还是替我决定？

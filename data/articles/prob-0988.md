# Lex 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Lex：它到底补上了哪一层执行功能？

你打开 Lex，想写一篇重要的文章。十分钟后，你发现自己正在调整字体间距、研究页边距的黄金比例，或者让 AI 帮你把一段话改写成十四行诗。你超聚焦了，但方向错了。这场景是不是很熟悉？

ADHD 大脑和 LLM 在长程任务中共享同一个致命缺陷：**目标漂移**。LLM 在长对话中会忘记初始指令，被上下文膨胀带偏；ADHD 大脑则会被新刺激劫持，从“写文章”滑向“调格式”。两者的核心问题都不是生成能力不足，而是**缺乏一个可靠的执行调度层来持续重锚定**。

Lex 被许多人当作“AI 写作生成器”——输入提示，得到段落，复制粘贴。但这样做，它变成了另一个多巴胺快餐：快速满足，然后你发现自己写了一堆无关内容，核心论点却丢了。真正有效的用法，是把 Lex 当作**重锚定 system prompt**：不是让它替你写，而是让它定期问你“我们现在的目标是什么？当前段落是否服务于这个目标？”

这正是 LLM harness 工程的逻辑。Agent scaffolding 的核心设计模式是“将 LLM 置于与记忆、工具和决策逻辑的控制循环中”（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。每次模型调用都访问检索、工具和记忆缓冲区，形成外部规划循环。对 ADHD 大脑来说，Lex 应该扮演这个**外部调度器**的角色：在写作过程中，每隔一段时间（比如每 200 字）主动输出一个重锚定提示，比如“当前段落的核心论点是什么？请用一句话总结，然后继续。”

这个思路并非空想。历史上最著名的 ADHD harness 系统之一——曾国藩的“日课十二条”——本质上就是一套手动重锚定协议。曾国藩年轻时“无恒”“浮躁”，日记里天天骂自己。他的解决方案是每天固定时间做固定事项：黎明即起、读书不二、谨言、写日记反省。这些不是内容生成，而是**行为级的 re-grounding**。对应到 LLM harness，就是定时触发“当前任务状态检查”的循环。苏轼的 harness 则更灵活：他用美食、写作、交友来转移注意力，但核心是“每日读书写字，不因挫折停止创作”——这是一种**弹性重锚定**，允许漂移但保持创作主线。

现代工具中，Focusmate 是重锚定最直接的实现：通过 AI 匹配的虚拟身体在场，每 50 分钟一次强制 check-in（来源：Revolutionizing ADHD Management with AI Assistants）。RescueTime 则提供事后重锚定——自动记录时间去向，让你看到自己漂移到了哪里（来源：Revolutionizing ADHD Management with AI Assistants）。但两者都缺少**任务过程中的实时重锚定**。Lex 可以补上这一层，前提是用户主动将其配置为“写作过程中的守卫”，而不是“写作内容的代笔”。

那么具体怎么用？首先，**把 Lex 的默认提示从“帮我写”改成“帮我保持方向”**。比如，在文档开头设一个 system prompt：“你是一个专注力守卫。每次我写完一段，你都要问我：这段是否直接服务于文章的核心论点？如果不是，请指出漂移方向。”其次，**利用 Lex 的侧边栏或对话功能，在写作中定期触发重锚定**。每完成一个自然节，主动问它：“我现在写到哪了？下一步应该写什么？”这相当于手动调用一次 agent harness 的“状态记录”功能。

当然，这个用法有争议。一是依赖风险：如果 Lex 替你做了所有重锚定，你的内在执行功能会不会退化？这就像问“用 Focusmate 会不会让人失去独立启动能力？”目前缺乏纵向研究（来源：矛盾与存疑），但脚手架与拐杖的边界在于：你是否在主动使用工具，还是被动依赖。曾国藩写日记是主动反省，不是等别人骂他。二是个体差异：对某些 ADHD 亚型，强制重锚定可能打断超聚焦，反而降低效率。有用户反馈强行打断（如番茄钟）有效，而柔性引导不够强力（来源：超聚焦是否应被引导而非打断？）。你需要实验：如果你属于“超聚焦后难以切换”的类型，重锚定频率可以降低，但保留触发机制。

**今天就能试的三件事：**
1. **在 Lex 中设置一个专注力守卫 prompt**：新建文档，第一行写“你是一个专注力守卫。每次我写完一段，问我这段是否服务于核心目标。”然后开始写作，观察它是否帮你减少漂移。
2. **结合 RescueTime 做事后复盘**：每天结束时看 RescueTime 报告，标记哪些时段被无关任务劫持。下次在 Lex 中写类似任务时，提前设重锚定提醒。
3. **模仿曾国藩的“日课”频率**：每工作 25 分钟，用 Lex 的对话功能写一句“当前目标是什么？我做到了吗？”——相当于手动 agent harness 的“会话状态记录”。

Lex 不是内容生成器。它是你的外部调度层。用对了，它是脚手架；用错了，它是另一个让你迷失的漂亮界面。选择权在你。

# Lex 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Lex：它到底补上了哪一层执行功能？

## 问题：分心走神或超聚焦跑偏，你丢失了目标

你坐在电脑前，打开 Lex，准备写一篇文章。五分钟前你还在想怎么开头，现在却刷了三个社交媒体，或者已经写了三千字但完全偏离了主题。这不是意志力问题——你的大脑像一台高性能生成器，但缺少一个稳定的调度层。ADHD 大脑与 LLM 在结构上同构：两者都是“高产但缺乏可靠执行调度层的生成核心”（来源：AI 与 ADHD 的专注力）。LLM agent 在长程任务中会目标漂移，需要重锚定 system prompt；ADHD 大脑在创作中会分心或超聚焦跑偏，需要外部重锚定。Lex 被宣传为“AI 写作助手”，但它的真正价值可能在于作为重锚定工具，而非内容生成器。

## 重锚定：Lex 到底补上了哪一层执行功能？

ADHD 大脑的核心困难之一是任务启动和维持。传统规划方法往往失效：“Planners and sticky notes worked for a week before falling apart.”（来源：规划循环与任务分解）因为标准化系统“often create more friction than flow”（来源：规划循环与任务分解）。Lex 的独特之处在于它提供了一个“空白页+AI 提示”的界面，但它的真正功能不是帮你写，而是帮你锚定“当下该写什么”。

从同构视角看，LLM agent 需要 harness 来管理上下文、调用工具、维持目标。Lex 相当于为写作任务提供了一个轻量级 harness：它的侧边栏可以承载项目目标、大纲、关键句子，这些相当于 agent 的 system prompt 和规划工件。当你分心时，看一眼侧边栏就能重锚定。这补上了 ADHD 大脑缺失的“外部执行功能层”（来源：AI 与 ADHD 的专注力）——具体来说，是任务表征的维持和工作记忆的卸载。

## 但有人用错了：脚手架 vs 拐杖的边界

许多人把 Lex 当作“AI 代笔”：输入一句话，让 AI 生成整段文字。这恰恰是错误的使用方式。如果你让 AI 代替你思考，你就失去了写作的认知收益，而且可能形成依赖——你的大脑会退化，不再尝试自己组织语言。这就是“拐杖 vs 脚手架”的边界：脚手架是暂时支撑，帮你完成自己无法独立完成的部分，同时训练你的能力；拐杖则是永久替代，让你越来越弱。

证据显示，AI 工具作为外部执行功能层，长期使用可能导致内在执行功能进一步退化（来源：矛盾与存疑）。类似地，LLM agent 如果过度依赖 harness 的自动决策，也会丧失对任务的理解。正确的用法是：用 Lex 的 AI 来生成选项、提示方向，而不是替代你的输出。例如，当你卡在段落过渡时，让 AI 提供三个可能的连接句，然后你自己选择并修改——这就像 agent 调用工具获取候选结果，然后由调度层决定下一步。

## 真实证据：ADHD 侧与 LLM/agent 侧的对应

ADHD 侧：Focusmate 利用虚拟身体在场提供外部问责，通过实时同伴压力提升任务启动和维持，其有效性有用户研究支持（来源：Focusmate）。RescueTime 通过自动时间追踪对抗时间盲，帮助用户识别注意力漂移模式（来源：RescueTime）。这些工具都提供了 ADHD 大脑缺失的外部结构——即执行功能层。

LLM/agent 侧：现代 agent 系统通过“组合多个模型、检索器和工具”来达成 SOTA 结果，而非依赖单一模型调用（来源：工具使用与认知卸载）。Harness 工程的核心是“context delivery, tool interfaces, planning artifacts, verification loops”（来源：规划循环与任务分解）。这与 ADHD 工具的设计同构：都需要提供外部记忆、结构化提示、注意力锚点。

Brain.fm 利用神经锁相技术影响大脑信息处理，声称帮助进入专注状态，但缺乏针对 ADHD 的独立临床研究（来源：Brain.fm）。这提醒我们，并非所有 AI 工具都有坚实证据，同构类比不能代替临床验证。

## 局限与争议：个体差异与依赖风险

ADHD 的异质性使得同一工具对不同亚型效果迥异（来源：矛盾与存疑）。Lex 可能对“超聚焦跑偏”型用户有效——他们需要重锚定；但对“启动困难”型用户，可能更需要像 Goblin Tools 那样的任务分解工具（来源：工具使用与认知卸载）。此外，证据质量普遍不足：多数工具仅有用户报告或小样本研究，缺乏大规模随机对照试验（来源：矛盾与存疑）。

依赖风险真实存在。工具设计者声称是“脚手架”，但实际使用中可能沦为“拐杖”（来源：矛盾与存疑）。如何避免？关键在于使用方式：工具应该补偿缺陷，而非替代能力。

## 今天就能试的行动

1. **用 Lex 的侧边栏作为“写作 system prompt”**：在开始写作前，把核心目标、关键论点、预期读者写在侧边栏。当你走神时，看侧边栏重锚定。这模仿了 agent 的上下文工程。
2. **让 AI 生成选项，而非答案**：卡住时，让 Lex 提供三个可能的句子或结构，然后你自己选择并修改。这保留了认知参与，避免依赖。
3. **结合 RescueTime 监测漂移**：开启 RescueTime 记录写作时段，看看你是否偏离了目标应用。用数据对抗时间盲。
4. **尝试 Focusmate 的“身体在场”写作**：预约一个 Focusmate 时段，告诉伙伴你要用 Lex 写什么。外部问责能显著提升任务维持。

记住：Lex 不是代笔，而是你的重锚定 system prompt。用对，它是脚手架；用错，它是拐杖。

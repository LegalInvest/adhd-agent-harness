# 如果把 ADHD 大脑当成生成核心，ChatGPT 能不能当它的生成核心与调度层？

> 用 agent 工程的思路重新理解 ChatGPT 对 ADHD 的真正价值。

你有没有过这样的经历：脑子里同时涌现出三个绝妙的点子，每一个都足以改变行业，但当你坐到电脑前，却盯着空白文档发呆，或者刷了半小时短视频？你并不懒，也不笨——你只是缺一个调度层。

这个问题，恰好和当下 AI 工程师们最头疼的问题同构：LLM 智力惊人，能写诗、能编程、能推理，但让它完成一个需要十步的跨天任务，它就会迷路、遗忘、胡编乱造。LLM 需要一个外部编排层——即 agent harness——来管理状态、记忆和执行步骤。

ADHD 大脑与 LLM 是同一类「生成核心」：高产、有创造力、信息处理能力强，但内置的执行调度层不可靠。ChatGPT 的真正价值，不是替代你的大脑，而是充当这个缺失的调度层。

## 问题：有想法，但卡在执行

ADHD 的核心困境并非缺乏想法或能力，而是执行功能缺陷——计划、组织、工作记忆、冲动控制等环节容易失稳（来源：[[生成核心与调度层]]）。你可以在十分钟内构思出一个复杂的项目框架，但无法在三天内把它写完。这不是意志力问题，而是你的大脑缺乏一个稳定的「任务管理器」。

LLM 同样如此。标准的 LLM 调用是无状态的：每次对话都是独立的，模型不保留跨会话的记忆（来源：[[无状态与外部记忆]]）。没有 harness 的 LLM 只是一个无状态的 API 调用（来源：[[无状态与外部记忆]]）。它能在单轮对话中给出惊艳的回答，但无法自主管理一个需要多步骤、跨上下文的长任务。

## 答案：外部 harness

既然问题同构，解法也同构。对 LLM 而言，这个外部调度层叫 agent scaffolding——包括记忆系统、工具调用、任务分解和状态管理。对 ADHD 大脑而言，这个调度层就是 AI 工具：它们提供外部记忆、任务分解、时间锚点和上下文管理。

以屠呦呦为例。她身上有典型的 ADHD 特质：不喜欢应酬，一门心思搞研究，从葛洪的《肘后备急方》里找到青蒿素的灵感。她的「harness」是一套严格的实验流程：筛选了 2000 多种中药、380 多个提取物，每一步都有记录和验证。这不是靠灵感，而是靠流程。这套流程就是她的外部调度层——把「从古书中找灵感」这个模糊目标，拆解成可执行的实验步骤。

这与 LLM harness 的结构完全对应：日课↔定时 re-grounding（让模型/大脑重新对齐目标），实验记录↔外部记忆系统（防止上下文丢失），团队合作↔多 agent 协作（分工并校验输出）。

再比如胡林翼。年轻时是花花公子，父亲去世后幡然醒悟，靠每日写日记反省、严格治军，从一个「失控的生成核心」变成了晚清中兴名臣。他的「日记」就是他的外部调度器——每天强制自己回顾目标、评估行为、调整计划。这和 LLM 的「re-grounding prompt」如出一辙。

## 工具：今天的 harness 是什么？

现在，已经有现成的 AI 工具可以扮演这个角色：

- **Goblin Tools**：它的 Magic ToDo 功能能把模糊任务（如「清理厨房」）自动分解为可执行的小步骤，用户可调节「辣度」控制分解粒度（来源：[[Goblin Tools]]）。这直接对应 LLM 的「任务分解」模块——把一个大目标拆成子任务，降低启动门槛。

- **Saner.AI**：专注于知识回忆和本地记忆，减少搜索循环和标签切换（来源：[[Saner.AI]]）。它相当于 LLM 的向量数据库——存储跨会话上下文，让你每天不必重新回忆项目背景。

- **Motion**：自动根据任务、会议和截止日期创建并动态调整每日计划，消除「下一步该做什么」的决策负担（来源：[[Motion]]）。这对应 LLM 的「调度器」——实时重排任务优先级。

这些工具的共同逻辑是：把执行功能外化，让 AI 负责「怎么走」，你负责「往哪走」。

## 脚手架 vs 拐杖：边界在哪？

但这里有一个必须正视的争议：AI 工具到底是「脚手架」还是「拐杖」？

脚手架是为了让使用者最终能够独立行走，拐杖则可能导致永久依赖（来源：[[拐杖与脚手架]]）。如果 AI 完全替代了你的计划、记忆和决策，你的执行功能可能进一步退化。目前缺乏纵向研究来回答这个问题（来源：[[矛盾与存疑]]）。

我的判断是：关键在于使用方式。如果你用 AI 来理解「为什么这样分解任务」，而不是仅仅接受它给你的步骤，那你就在搭建脚手架。如果你只是机械地执行 AI 的指令，从不反思也不调整，那就是拐杖。

另一个争议是：超聚焦是否应被引导而非打断？部分用户反馈强行打断（如番茄钟）反而有效，而柔性引导可能不够强力（来源：[[矛盾与存疑]]）。这提醒我们，没有万能方案——你需要根据自己的注意力模式选择工具策略。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个你拖延了一周的任务**：把「写报告」变成「打开文档→写标题→列出三个要点→写第一段→休息 5 分钟」。感受启动门槛降低后的变化。

2. **设置一个 AI 驱动的晨间 re-grounding 流程**：每天用 ChatGPT 或 Saner.AI 回顾今天的三个优先事项，让 AI 帮你拆解第一步。这模仿了胡林翼的日记——但由 AI 提醒你。

3. **为你的 AI 工具配置外部记忆**：在 ChatGPT 的 custom instructions 里写下你的项目背景、常用偏好和当前目标。这样每次对话都有上下文，不再从零开始。

4. **尝试 Motion 或类似工具，让 AI 替你排日程**：把任务、会议和截止日期丢给它，让它动态调整。观察它如何应对突发变化——这本身就是一种学习。

## 局限与未来

同构视角目前仍是理论框架，缺乏神经科学或计算机科学的直接证据（来源：[[矛盾与存疑]]）。ADHD 大脑与 Transformer 的自注意力机制确实存在计算层面的相似性，但这是类比而非因果。另外，AI 工具本身的学习成本可能增加认知负荷（来源：[[矛盾与存疑]]），净效果因人而异。

但无论如何，这个视角提供了一个新的思路：你不是坏掉的大脑，你只是缺一个 harness。而 ChatGPT，恰好可以成为那个 harness。

# Mem 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Mem：它到底补上了哪一层执行功能？

## 从一道难题开始：为什么我的大脑和 LLM 都卡在第一步？

你打开一个空白文档，光标闪烁，任务清单上写着“写周报”——但你的手指像被冻住。另一边，你给 LLM 一个复杂指令：“写一篇关于气候变化的文章”，它却输出一堆泛泛的废话。

这两个场景共享同一个底层问题：**一个高产但缺乏执行调度层的生成核心**。ADHD 大脑与 LLM 在结构上同构——都是“高产但缺乏可靠执行调度层的生成核心”（来源：[[AI 与 ADHD 的时间管理]]）。LLM 需要 planner-executor 循环来分解长任务，ADHD 大脑同样需要外部脚手架来补偿任务启动困难和不会拆解。

## 同构脊柱：planner-executor 循环 vs. 执行功能脚手架

在 LLM/Agent 工程中，一个成熟的模式是：planner 将长任务分解为子步骤，executor 逐一执行，然后循环反馈。这解决了 LLM 的“上下文控制缺陷”（来源：[[AI 与 ADHD 的时间管理]]）。

ADHD 大脑的对应物是什么？是执行功能——尤其是工作记忆容量保留但控制弱、时间盲、任务启动困难（来源：[[AI 与 ADHD 的时间管理]]）。ADHD 个体不是缺乏想法或能力，而是缺少一个可靠的“调度器”来将“写周报”分解为“打开电脑→登录邮箱→列出上周事件→写第一段”。

**真正的同构在于：两者都需要一个外部的 planner-executor 循环来补偿内部调度缺陷。** 对于 LLM，这个循环是 prompt 模板、任务分解链、工具调用；对于 ADHD 大脑，这个循环是外部时间锚点、视觉化时间线、自动调度系统。

## 工具实测：Mem 补上了哪一层？

让我们用同构视角审视几款主流工具：

### Motion：自动 planner，但缺 executor 反馈

Motion 能自动根据任务和截止日期创建并动态调整每日计划（来源：[[Motion]]）。它消除了“下一步该做什么”的决策负担，直接应对时间盲和启动困难。

但从同构看，Motion 只做了 planner 的部分——它帮你规划，但执行时你仍需要自己“动手”。如果执行中被打断，Motion 虽然能重新安排，但不会像人类身体加倍那样提供隐性问责。

### Reclaim.ai：防御型 planner，保护执行空间

Reclaim.ai 通过创建深度工作、习惯和专注时间的智能块，自动防御会议侵占（来源：[[Reclaim.ai]]）。它的设计更接近“保护执行环境”，类似为 LLM 设置一个无干扰的上下文窗口。对于 ADHD 用户，这减少了因突发会议导致日程崩溃的风险。

### Tiimo：视觉化时间线，补偿时间盲

Tiimo 专为 ADHD 和自闭症成人设计，将时间转化为可见元素（来源：[[Tiimo]]）。它直接应对时间盲，相当于为 LLM 提供时间戳或进度条。

### Todoist：外部记忆，但缺乏分解

Todoist 通过外部提醒和优先级排序充当外部记忆（来源：[[AI 与 ADHD 的时间管理]]），类似向量数据库为 LLM 提供持久上下文。但它不帮你分解任务——你需要自己知道“下一步做什么”。

### Focusmate：身体加倍，提供 executor 层面的问责

Focusmate 利用算法匹配虚拟身体加倍伙伴，通过社会存在感降低分心阈值（来源：[[身体在场效应]]）。这是最接近“executor 反馈”的工具——它提供了一个实时监督者，让你保持执行。

## 核心判断：有人用错了——Mem 不该只是“填充器”

从同构视角看，大多数 AI 工具只做了 planner（规划）或 memory（记忆），但缺少真正的 executor 反馈循环。用户以为“有了 Motion 就能自动执行”，结果发现自己还是卡在第一步。

**真正的脚手架应该是 planner + executor + 反馈循环**。对于 ADHD，这意味着：
1. 自动分解任务（planner）
2. 提供实时执行提示（executor）
3. 根据完成情况调整后续计划（反馈）

目前只有 Focusmate 部分实现了 executor 层，但它依赖他人，不是纯 AI。而 Motion 和 Reclaim.ai 虽然智能，但用户仍需自己启动执行——这就像给 LLM 一个完美的 planner，但 executor 是坏的。

## 争议与局限：脚手架还是拐杖？

必须诚实指出：
- **过度依赖风险**：AI 工具可能成为“永久拐杖”，削弱用户内在的时间感知和规划能力（来源：[[矛盾与存疑]]）。
- **个体差异**：ADHD 亚型对工具响应不同，现有工具多面向注意力缺陷型（来源：[[矛盾与存疑]]）。
- **证据质量**：多数证据基于用户自我报告，缺乏随机对照试验（来源：[[矛盾与存疑]]）。
- **同构理论的实证不足**：ADHD 大脑与 LLM 的同构仍属理论类比，神经科学和计算机科学的交叉验证不足（来源：[[AI 与 ADHD 的时间管理]]）。

关键在于区分“脚手架”和“拐杖”：脚手架帮你学会独立行走，拐杖让你永远依赖。目前工具设计者声称是脚手架，但实际使用中可能沦为拐杖（来源：[[矛盾与存疑]]）。

## 今天就能试的行动

1. **用 Tiimo 或纸笔将一天视觉化**：把时间变成可见元素，补偿时间盲。
2. **给每个任务写一个“分解提示”**：比如“写周报”分解为“打开电脑→登录邮箱→列出上周事件→写第一段”，就像给 LLM 写 prompt。
3. **尝试 Focusmate 一次**：体验身体加倍带来的 executor 反馈。
4. **记录使用 AI 工具后的感受**：是感觉更独立，还是更依赖？这能帮你判断工具是脚手架还是拐杖。

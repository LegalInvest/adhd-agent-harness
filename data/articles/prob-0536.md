# 3 个 LLM 工程概念，彻底重写了我对 ADHD 任务启动困难、不会拆解的理解

> 「不会拆任务」听起来像能力缺陷。但在 agent 工程里，规划和执行本来就是两个模块——没人要求执行器自己会规划。

「你就是不会把事情拆开做。」这句话的潜台词是：拆解是人人该会的基本功。可是 agent 工程给了我完全不同的视角：在所有能干活的 LLM 系统里，**规划和执行从一开始就是分离的**——planner 是一个模块，executor 是另一个模块，谁也不指望一个模块同时干两件事。

## 1. planner-executor 分离：不会拆解不是执行力问题，是模块缺位

给 LLM 一个模糊的大目标（「做一个竞品分析」），它要么给你泛泛而谈，要么一头扎进某个细节出不来。工程师的解法不是换更强的模型，而是加一个独立的规划循环：先让 planner 把目标拆成步骤序列，再交给 executor 逐步执行，两个角色甚至常用不同的提示词、不同的温度。

ADHD 的「任务启动困难」几乎总是伴随「不会拆解」，这不是巧合——两者是同一个缺位的两个症状：你缺的是那个 planner 模块的稳定供电。多巴胺系统对「远期、抽象、无即时反馈」的规划类认知活动供能不足，于是大目标始终以未分解的原始形态堵在门口，启动自然失败。把「不会拆」理解为模块缺位而非能力低下，解法立刻不同：**外接一个 planner**。Goblin Tools、ChatGPT、甚至一个愿意帮你过一遍计划的同事，都是合格的外部 planner。让专业模块干专业的事，你只负责当 executor——这不丢人，这是架构。

## 2. 冷启动问题：第一个 token 最贵

LLM 推理有个众所周知的成本结构：首 token 延迟远高于后续 token——一旦开始生成，续写是便宜的，贵的是从零到一。

ADHD 的启动困难是同一条成本曲线。无数 ADHD 者的共同经验：最难的是坐下来打出第一行字，一旦进入状态，接下来两小时反而停不下来（甚至过渡到超聚焦）。所以所有把「开始」变便宜的技巧本质上都在优化首 token：「只做两分钟」协议、「先写一句烂开头」、身体先于意识坐到工位上。老子说「天下难事必作于易」——两千年前的首 token 优化。孔子的「下学而上达」同样：从最低门槛的具体动作进入，让抽象目标在执行中自己显形。

## 3. 脚手架的时间属性：好的支撑会自己褪去

教育心理学和 agent 工程共享一个概念：scaffolding。工程上指训练或推理早期给模型的额外结构（示例、模板、分步提示），随着能力上线逐步移除。关键属性是**会褪去**——不褪去的脚手架会变成天花板。

这给 ADHD 工具使用划出了一条重要的判断线：外接 planner 是脚手架还是拐杖，取决于你是否在拆解的过程中旁听。每次让 AI 拆任务时多看一眼它怎么拆的——按时间拆还是按依赖拆、第一步为什么选这个——半年后你会发现自己对某些熟悉任务已经能秒拆。规划能力不是天赋，是可以从自己的外部 planner 身上偷师的模式库。

## 边界

同构强度 B 级：机制类比，双侧文献真实（ADHD 侧的执行功能与奖赏研究、agent 侧的规划系统文献），但不构成「大脑=agent」的科学等同。另一个诚实提醒：外接 planner 对「任务厌恶型拖延」效果有限——如果你不是不会拆，而是不想做，那是另一个模块（动机系统）的问题，见本系列反馈信用分配篇。

## 今天就能试的 3 件事

1. **外接一次 planner**：把你堵得最久的大任务原样丢给 AI：「帮我拆成第一步小到两分钟能完成的步骤序列」。
2. **只付首 token 成本**：对着拆出来的第一步，启动一个两分钟计时器。到时间想停就停——多数时候你不会停。
3. **偷师一次**：看 AI 的拆解结果时，说出它用的拆解逻辑是什么。说不出来就问它。

不会拆解从来不是你的失败。单模块系统本来就不该被要求同时会规划和执行——补上那个模块，比骂自己一万句都有用。

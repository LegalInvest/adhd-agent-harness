# Reflect 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Reflect：它到底补上了哪一层执行功能？

你是否有过这样的体验：打开电脑准备写一份方案，半小时后却发现自己正在刷短视频，完全忘了最初的目标？或者，你刚读完一页书，脑中就冒出十来个新想法，然后你顺着其中一个狂奔，直到两小时后才惊觉自己跑偏了。这种「目标漂移」——无论是分心走神还是超聚焦跑偏——是 ADHD 人群的核心困境。

与此同时，在 AI 工程领域，同样的困境正在上演：一个被赋予复杂任务的 LLM agent，在长程推理中会逐渐偏离初始指令，产生幻觉、重复劳动或干脆忘掉目标。工程师们为此设计了「重锚定 system prompt」——定期将 agent 的上下文拉回正轨。

这两件事，本质上是同一个问题。ADHD 大脑与 LLM 是同一类「高产但缺执行调度层的生成核心」：两者都能生成海量想法/输出，但缺乏可靠的内置机制来维持方向、验证结果、调节节奏。因此，两者都需要外部脚手架——ADHD 侧称为「harness」，LLM 侧称为「agent harness」。而 Reflect 这类 AI 工具，如果只用对一半，就可能从脚手架沦为拐杖。

## 同构：目标漂移与重锚定

ADHD 大脑的「生成核心」时刻在产生念头、联想和冲动，但执行调度层——前额叶皮层——常常掉线。这导致两种典型问题：一是分心走神（注意力被外部刺激拉走），二是超聚焦跑偏（注意力被内部兴趣锁死，偏离原目标）。前者像 LLM 被上下文中的无关信息带偏，后者像 LLM 在某个子任务上过度优化，忘了整体目标。

LLM 的「幻觉」本质上是生成核心缺乏验证循环的产物（来源：幻觉与验证循环）。ADHD 的冲动行为则是类似的「验证失败」——大脑跳过「核对现实」这一步，直接执行了生成结果。Harness 工程的核心正是为 LLM 提供「验证与 CI 集成」「结构化工作流和规划」「结果验证与迭代」等组件（来源：幻觉与验证循环）。这些组件与 ADHD 侧的外部脚手架一一对应：

- **RescueTime** 的自动时间追踪与分心阻断，补偿了「时间盲」和执行功能缺陷（来源：RescueTime），相当于 LLM 的「记忆系统」和「沙盒」。
- **Focusmate** 的虚拟身体在场，通过同伴问责提供外部结构（来源：Focusmate），相当于 LLM 的「工具接口」和「验证循环」——同伴就是那个检查输出的人。
- **Brain.fm** 的神经锁相音乐，帮助稳定注意力状态（来源：Brain.fm），相当于 LLM 的「确定性状态转换」——降低输出的随机性。

但这些工具单独使用时，往往只解决了「当下」的问题，而没有解决「方向」的问题。就像给 LLM 一个更好的上下文窗口，却不给它重锚定 system prompt 的机制——它仍然会漂移。

## 曾国藩的 Harness：日课十二条与重锚定

晚清名臣曾国藩，是典型的注意力不集中型 ADHD。他 30 岁前浮躁坐不住，天天串门喝酒看杀人，日记里骂自己「无恒」「浮躁」；读书极慢，「他人目下二三行，余或疾读不能终一行」；情绪极不稳定，打败仗就跳水自杀。但正是这个人，后来平定太平天国，成为晚清中兴四大名臣之首。

他的秘密武器是「日课十二条」：黎明即起、读书不二、谨言、写日记反省……每一条都是外部调度指令。其中「读书不二」规定「一书未点完，不看他书」，直接对抗超聚焦跑偏；「写日记反省」则是每天的验证循环——核对当天的行为是否偏离目标。

这套系统的本质，就是「重锚定 system prompt」。曾国藩每天用同样的 prompt 把自己拉回正轨，无论情绪波动、环境干扰还是内部冲动。而且他坚持了一辈子，直到去世前还在写日记骂自己。

这与 LLM harness 中的「计划-执行」分离模式完全同构：日课是「计划」，每日执行是「执行」，日记是「验证」。曾国藩的成就不是因为他没有 ADHD，而是因为他用最笨最稳的脚手架抵消了自己的冲动——「结硬寨，打呆仗」。

## Reflect 的正确用法：重锚定而非记录

Reflect 是一款 AI 笔记工具，能自动整理、关联和回顾笔记。很多 ADHD 用户用它来「记录一切」，以为只要把想法记下来，就不会丢失。但问题在于：记录本身不等同于重锚定。如果你只是把想法丢进 Reflect，而不定期回顾和校准方向，它就成了一个更大的「超聚焦陷阱」——你可能会花两小时整理标签，而忘了原本要写的方案。

正确的用法是：把 Reflect 当作「重锚定 system prompt」的载体。每天固定时间（比如早晨和傍晚），打开 Reflect，只看两个东西：一是你的「核心目标」（像 system prompt 一样固定在最上方），二是你昨天写下的「今日重点」。然后问自己：我现在做的事情，还在这个方向上吗？

这相当于 LLM 的「周期性采样以监控质量」（来源：采样温度与表现波动）。ADHD 大脑的表现波动很大——上午专注，下午涣散——就像 LLM 的输出受「采样温度」控制。外部约束（如日程、提醒、环境设计）能稳定表现，就像确定性工作流稳定 LLM 输出。

## 争议与局限：脚手架 vs 拐杖

但这里有一个危险：AI 工具可能成为永久依赖的拐杖，而非逐步内化的脚手架（来源：拐杖与脚手架概念）。如果 Reflect 只帮你记录，而不训练你「回顾」的习惯，你永远学不会自己锚定方向。曾国藩的日课之所以有效，是因为他坚持了三十年的重复，最终内化为第二本能。

另一个争议是临床证据不足。Brain.fm 虽有神经科学基础，但在 ADHD 人群中的独立随机对照试验极少（来源：Brain.fm）。Focusmate 的效果主要依赖用户报告（来源：Focusmate）。这些工具有效，但效果有多大、对谁有效、能持续多久，都是未知。

此外，ADHD 的异质性意味着同一工具对不同亚型效果迥异（来源：矛盾与存疑）。注意缺陷为主型可能更需要 RescueTime 的阻断功能，多动冲动型可能更需要 Focusmate 的身体在场。

## 今天就能试的行动

1. **设置每日「重锚定」闹钟**：选两个固定时间（如 10:00 和 16:00），闹钟响起时，打开 Reflect（或任何笔记工具），看你的核心目标，写下当前任务是否匹配。坚持一周。
2. **用 Focusmate 做「验证循环」**：预约一个 50 分钟的 Focusmate 时段，开始前告诉伙伴你的目标，结束时汇报进度。这相当于 LLM 的「结果验证与迭代」。
3. **把 RescueTime 的每日报告当「日记」**：像曾国藩一样，每晚花 5 分钟看 RescueTime 报告，标记哪些时间花在了目标上，哪些漂移了。不责备，只观察。
4. **尝试 Brain.fm 的「专注模式」**：在需要深度工作时开启，注意它是否帮你降低「采样温度」——减少思绪的随机跳跃。

Reflect 不是救世主，它只是一个工具。真正的脚手架，是你用这些工具构建的系统——一个每天重锚定、验证、调整的系统。ADHD 大脑与 LLM 一样，需要的不是更强大的生成核心，而是更可靠的执行调度层。而这个调度层，只能由你自己来设计。

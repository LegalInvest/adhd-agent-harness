# Lex 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Lex：用久了会让 ADHD 更强还是更依赖？

## 问题：为什么你刚刚想做的事，转头就忘了？

如果你是 ADHD 患者，你一定经历过这样的场景：打开电脑想写一封邮件，结果被通知栏吸引，半小时后发现自己正在刷短视频，完全忘了邮件的事。你的工作记忆就像一个漏水的桶——容量小、流失快，跨会话任务几乎无法持续。每天重新回忆项目上下文，否则就偏离轨道（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。

如果你是 LLM/Agent 工程师，你一定也遇到过类似的困境：你精心设计的 agent 在单轮对话中表现惊艳，但一旦需要跨会话协作，它就“失忆”了——因为标准 LLM 调用是无状态的，每次对话都是独立的（来源：The Anatomy of an AI Agent: Memory, Tools, Planning, and Execution Explained - DEV Community）。没有 harness 的 LLM 只是一个无状态的 API 调用（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。

这两种困境，本质上是同一个问题：**一个强大的生成核心，缺乏可靠的执行调度层**。ADHD 大脑的创造力、LLM 的生成能力，都像一台没有操作系统的超级计算机——硬件强悍，但无法稳定运行多任务。而外部工具（如 Lex）正是试图扮演这个“操作系统”的角色。

## 同构：ADHD 大脑与 LLM 共享同一个缺陷

将 ADHD 大脑与 LLM 类比并非牵强附会。ADHD 的工作记忆容量有限且不稳定，导致跨会话任务难以持续（来源：无状态与外部记忆）。LLM 的固定上下文窗口同样导致长程依赖追踪困难（来源：Long-Term Memory Architectures for AI Agents - Redis）。两边都在寻找“外部记忆体”来补偿内部缺陷：ADHD 患者需要“第二大脑”工具存储客户历史、项目背景和个人偏好（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）；Agent 系统则需要向量数据库、会话记忆、结构化知识库来弥补 LLM 的无状态（来源：The Anatomy of an AI Agent: Memory, Tools, Planning, and Execution Explained - DEV Community）。

这种同构意味着：**解决 ADHD 外部记忆问题的工具，其设计思路与 Agent 的 harness 工程高度一致。** Lex 正是这条思路上的一个典型代表。

## Lex 的承诺：单一指令触发多步骤

Lex 的核心卖点是“通过单一指令触发复杂、多步骤的任务序列”（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。对 ADHD 用户来说，这意味着：你不需要在执行任务时不断回忆下一步该做什么——Lex 替你记住了。你只需输入一个意图（比如“整理下周的会议纪要并发送给团队”），Lex 就会自动拆解为子任务并顺序执行。这直接降低了任务启动的门槛，减少了工作记忆的负荷（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

对 Agent 工程师来说，Lex 就像是一个预配置的 harness：它提供了上下文传递、工具接口、任务规划工件和验证循环——这些正是生产级 agent 系统的核心组件（来源：GitHub - ai-boost/awesome-harness-engineering）。Lex 在首次提示前就“组装”好了代理，运行时则负责编排工具调度和上下文管理。本质上，Lex 是一个面向 ADHD 用户的“Agent harness”。

## 脚手架还是拐杖？关键在于可撤除性

但这里有一个尖锐的问题：Lex 这样的工具，用久了会让 ADHD 患者更强，还是更依赖？

从概念上说，脚手架是临时支撑，帮助使用者“建造”，但使用者仍需自己“攀登”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。而拐杖是永久替代，一旦离开就寸步难行。当前多数 ADHD 工具（包括 Lex、Goblin Tools、Saner.AI）被设计为长期使用，并未提供明确的撤除机制（来源：矛盾与存疑）。这引发了专家对过度依赖的警告：若工具替代了治疗或自我管理，可能造成依赖（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

我的判断是：**Lex 能否成为脚手架，取决于使用者是否意识到它的“harness 本质”**。如果你把 Lex 当作一个黑盒——输入指令，等待输出——那么你只是在用外部大脑替代内部大脑，长期来看会削弱自己的执行功能。但如果你把它当作一个可观察、可调整的“外部执行层”——就像 Agent 工程师调试自己的 harness 一样——那么你就能逐步学习它的规划逻辑，最终内化部分能力。

## 证据与局限

目前关于 Lex 的直接研究证据有限，但其设计理念与 ADHD 干预策略一致：复杂任务分解有助于减少 overwhelm（来源：6 ways AI can help you manage ADHD symptoms - Understood.org），实时自适应系统被证明能提升 ADHD 患者的任务完成率（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。然而，ADHD 的异质性高，单一工具难以覆盖所有亚型（来源：ADHD 的 AI 工具全景）。此外，AI 工具缺乏针对 ADHD 核心症状的长期随机对照试验，现有证据多来自用户评价和综述（来源：矛盾与存疑）。Lex 声称能通过单一指令处理复杂任务，但 ADHD 的复杂性可能使 AI 难以准确识别用户意图（来源：Frontiers | Artificial intelligence in ADHD assessment: a comprehensive review of research progress from early screening to precise differentiation）。

## 今天就能试的行动

1. **用 Lex 分解一个你拖延已久的任务**：比如“写周报”或“整理桌面”。观察它生成的子步骤，对比你自己通常的分解方式。这能帮你意识到“规划”是一种可学习的技能。
2. **记录 Lex 的规划逻辑**：在使用 Lex 时，注意它如何将你的意图转化为步骤。尝试理解它的“决策树”，并在下次类似任务中尝试自己先规划，再用 Lex 验证。这类似于 Agent 工程师调试 prompt 的过程。
3. **设置一个“无 Lex 日”**：每周选一天，尝试不用任何 AI 工具完成任务。这能帮你评估自己对工具的依赖程度，并锻炼内在执行功能。
4. **与其他工具对比**：尝试用 Goblin Tools 做同样的任务分解，感受不同工具的“harness 风格”。这能帮你找到最适合自己大脑的脚手架。

## 结语

Lex 之于 ADHD，就像 harness 之于 LLM：它提供了缺失的执行层，让生成核心得以稳定产出。但 harness 可以升级、替换、甚至移除——前提是你知道它只是一层外部封装。不要让它成为你的拐杖，而要把它当作一面镜子，照见你自己大脑的运作方式。最终，最好的脚手架，是让你学会自己搭建脚手架。

# 如果把 ADHD 大脑当成生成核心，Reclaim.ai 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Reclaim.ai 对 ADHD 的真正价值。

## 问题：为什么你的大脑和 AI agent 一样容易“跑偏”？

如果你有 ADHD，你一定经历过这样的时刻：打开电脑准备写报告，却发现自己半小时后在看 YouTube 上关于“如何养水母”的视频。你明明有目标，但执行路径像被施了魔法一样扭曲。同样，如果你在做 agent 工程，你大概率见过这样的日志：一个被要求“写一篇产品介绍”的 agent，在调用 20 次 API 后，开始生成关于“量子计算与寿司”的散文。

ADHD 大脑与 LLM agent 共享一个核心结构：**高产但缺乏可靠执行调度层的生成核心**（来源：AI 与 ADHD 的专注力）。ADHD 的创意发散、LLM 的文本生成，都是强大的“生成核心”，但两者都缺少一个内置的调度层来确保输出不偏离目标。ADHD 的调度层缺陷表现为执行功能（启动、时间感知、注意力维持）的混乱；LLM 的则表现为上下文膨胀、目标漂移。

所以，核心问题来了：如果把 ADHD 大脑视为一个生成核心，一个像 Reclaim.ai 这样的工具，能不能充当它的“重锚定”与“目标漂移”的矫正器？

## 同构：ADHD 大脑与 LLM 共享的“重锚定”需求

在 agent 工程中，解决目标漂移的标准做法是“重锚定”——定期检查 system prompt，压缩上下文，确保 agent 不偏离初始目标。Harness 工程正是为此而生：它提供“上下文传递、工具接口、规划工件、验证循环”等脚手架（来源：规划循环与任务分解）。

ADHD 大脑同样需要这种外部脚手架。ADHD 的工作记忆缺陷与 LLM 的无状态性同构——两者都无法可靠地保持“当前目标”的活跃状态（来源：AI 与 ADHD 的专注力）。ADHD 需要外部工具来“记住”该做什么，就像 LLM 需要向量数据库或 prompt 模板来维持上下文。

Reclaim.ai 的核心机制——**动态日历调度**——恰恰提供了一种“时间锚定”。它会根据任务优先级和可用时间自动调整日程，当计划被打乱时（比如会议超时或突发任务），它会重新排列剩余任务。这相当于一个“时间层面的重锚定”：它不断问“现在最重要的是什么？”，并据此调整执行顺序。

## 证据：Reclaim.ai 如何补偿 ADHD 的执行功能短板？

### ADHD 侧：解决时间盲与启动困难

ADHD 患者常受困于“时间盲”——无法准确感知时间流逝和任务耗时（来源：规划循环与任务分解）。传统规划工具（如纸质计划表）往往一周内就失效，因为“大多数规划建议不是为我们设计的”（来源：规划循环与任务分解）。Reclaim.ai 的自动调度和动态调整，直接应对了这个问题：它不需要用户手动估算时间，而是通过 AI 学习用户习惯，自动为任务分配时间块。

同时，Reclaim.ai 降低了任务启动的门槛。ADHD 的启动困难部分源于“决策疲劳”——决定“接下来做什么”本身就是一个执行功能任务。Reclaim.ai 通过自动生成日程，消除了这个决策点：用户只需遵循日历，不需要在“该做什么”上消耗认知资源。这与 Goblin Tools 的任务分解模式类似——将模糊的工作分解为可执行步骤（来源：工具使用与认知卸载）。

### Agent 侧：类似 Harness 的上下文压缩与优先级重排

在 agent 工程中，一个常见的模式是“规划-执行-验证”循环。Reclaim.ai 的调度逻辑与此类似：它先根据用户设定的优先级（如“固定日程”“任务截止时间”）生成一个初始计划，然后随着时间推移，根据实际执行情况（如任务延期、新任务加入）动态调整。这相当于 agent 的“上下文压缩”——丢弃不再相关的信息，聚焦当前最重要的目标。

此外，Reclaim.ai 的“防御时间”功能（自动为重要任务预留不被会议侵占的时间块）类似于 agent 的“system prompt 保护”——确保核心指令不被后续的上下文膨胀覆盖。

## 核心观点：Reclaim.ai 是有效的“重锚定”工具，但需警惕“脚手架变拐杖”

我的判断是：**Reclaim.ai 确实能充当 ADHD 大脑的“重锚定”与“目标漂移”矫正器，但其有效性取决于使用方式——它应该是脚手架，而非拐杖。**

“脚手架 vs 拐杖”的边界在于：工具是否在“补偿”还是“替代”内在能力（来源：矛盾与存疑）。Reclaim.ai 如果只是被动地告诉用户“该做什么”，用户可能永远学不会自己规划时间——这就像 agent 完全依赖外部 prompt，一旦 prompt 丢失就瘫痪。但如果用户将 Reclaim.ai 作为“反馈系统”——比如每天花 5 分钟审视 AI 生成的日程，理解为什么这样安排，并逐步内化时间感知——那么它就是脚手架。

## 局限与争议：证据不足与个体差异

必须诚实指出，Reclaim.ai 针对 ADHD 的效果缺乏严格的临床研究。多数证据来自用户报告和概念类比（来源：矛盾与存疑）。同样，Brain.fm 宣称通过神经锁相帮助专注，但缺乏针对 ADHD 的独立研究（来源：Brain.fm）。Focusmate 基于身体在场效应，有用户研究支持但无随机对照试验（来源：Focusmate）。RescueTime 能提升时间觉察，但可能引发焦虑（来源：RescueTime）。

个体差异也是关键：ADHD 的异质性意味着同一工具对不同亚型效果迥异（来源：矛盾与存疑）。例如，对“注意力分散型”有效的工具，可能对“冲动控制型”无效。

ADHD 大脑不是缺陷，而是一个未经调校的生成核心。Reclaim.ai 等工具提供了外部调度层，但最终，我们是选择成为依赖提示的 agent，还是学会自己构建 harness，决定了工具是脚手架还是拐杖。

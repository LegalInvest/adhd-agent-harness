# 3 个 LLM 工程概念，彻底重写了我对 ADHD 有想法有能力，却卡在执行与落地的理解

> LLM 智力强但需要外部编排才能完成长任务 如何反向照亮 ADHD 的有想法有能力，却卡在执行与落地。

你有没有过这样的体验：脑子里同时涌出十个绝妙想法，但真正落地的一个都没有？或者，明明知道该做什么，身体却像被钉在椅子上，就是启动不了？

如果你是 ADHD 群体的一员，这种「有想法有能力，却卡在执行与落地」的挫败感，可能每天都在上演。而如果你在做 LLM Agent 工程，你一定也遇到过类似场景：大模型能写出惊艳的诗、能解复杂的数学题，但一旦让它「帮我订一张明天下午的机票，然后安排会议」，它就卡在第一步——要么忘记上下文，要么胡乱编造。

这两个场景，看似分属心理学和工程学，却共享同一个底层结构：**一个强大的生成核心，缺了一个可靠的执行调度层。**

### 生成核心 vs 调度层：同构的困境

ADHD 大脑的核心特征之一是执行功能障碍（Executive Dysfunction），表现为计划、组织、任务启动、时间管理上的困难（来源：执行功能障碍）。这不是懒惰，而是大脑的「调度层」效率低下。同时，ADHD 个体往往拥有超常的创造力、发散思维和行动倾向——专利数据显示，ADHD 群体的创新专利数是普通人的 2.1 倍（来源：USPTO Data Analysis）。

LLM 同样如此。它的生成能力惊人，但本质上是「无状态」的——每次对话都是独立的，缺乏内置的规划、记忆和调度机制。要让 LLM 完成多步、长周期的任务，必须依赖外部编排：上下文工程、记忆系统、Agent 框架。

**两者共享同一个核心矛盾：产出能力远超执行能力。** 因此，解法也是同构的——给生成核心套上一个「harness」（脚手架），用外部系统补偿调度层的缺失。

### 三个 LLM 工程概念，照亮 ADHD 的执行困境

#### 1. 任务分解（Task Decomposition）：从「清理厨房」到「拿抹布→擦台面→洗水池」

在 LLM 工程中，要让模型完成复杂任务，第一步就是分解。例如，用 Chain-of-Thought 提示将「写一篇论文」拆解为「选题→文献检索→大纲→初稿→修改」。没有分解，模型会直接输出一堆无结构的内容。

ADHD 的大脑同样需要这种分解。Goblin Tools 的 Magic ToDo 功能正是为此而生：输入「清理厨房」，AI 自动将其拆成 5-10 个可执行的小步骤，用户还能调节「辣度」控制粒度（来源：Goblin Tools）。这种分解降低了启动门槛——不是「我要清理整个厨房」，而是「我先拿抹布」。

**人物案例：屠呦呦**。这位诺贝尔奖得主身上有明显的 ADHD 特质——不喜欢应酬、一门心思搞研究、思维高度专注。她的 harness 系统就是严格的实验流程：从 2000 多种中药、380 多个提取物中反复筛选（来源：人物案例）。她不是靠灵感一次成功，而是把「发现抗疟药物」这个宏大任务，分解为每天在实验室的重复筛选、记录、验证。这正对应 LLM 的「任务分解」——把大目标切成可执行的子任务，每一步都有明确的输入和输出。

#### 2. 外部记忆（External Memory）：别再让大脑当 RAM

LLM 的上下文窗口再大，也装不下整本书。因此工程上常用 RAG（检索增强生成）或记忆模块，把信息存在外部，需要时再调取。

ADHD 的工作记忆就像一台只有 2GB RAM 的电脑——开三个标签页就卡死。Saner.AI 正是针对这一点：它提供 AI 驱动的本地记忆和快速检索，用户无需在多个应用间切换就能找回信息，减少搜索循环和认知负荷（来源：Saner.AI）。

这让我想起罗振宇的 harness：每天 6:30 发 60 秒语音，十年不中断。他用「输出倒逼输入」，把每天的知识碎片通过语音笔记固化下来，而不是让它们在大脑里飘散。这本质上就是外部记忆——把大脑从存储任务中解放出来，专注于生成和连接。

#### 3. 动态调度（Dynamic Scheduling）：让 AI 帮你做「下一步该做什么」的决策

LLM Agent 的另一个痛点是缺乏优先级意识。如果没有调度器，它会死磕一个子任务，或者被新输入带偏。Motion 这类工具正是 LLM 调度层的工程实现：它自动根据截止日期、优先级、可用时间动态调整日程，当计划被打乱时实时重建（来源：Motion）。

对于 ADHD，这种「自动调度」直接解决了时间盲（Time Blindness）和任务启动困难。用户不需要自己判断「现在该做什么」，Motion 已经排好了。这相当于把执行功能的决策权外包给了 AI。

### 脚手架 vs 拐杖：一个必须划清的边界

但这里有一个危险的陷阱：**过度依赖可能让脚手架变成拐杖。**

如果 AI 工具替你做了所有任务分解、记忆检索和日程安排，你的大脑会不会变得越来越不会做这些事？神经可塑性告诉我们，大脑用进废退（来源：神经可塑性）。长期依赖外部调度，可能削弱内在执行功能，尤其是任务启动和计划能力（来源：矛盾与存疑）。

我的核心判断是：**AI 应该是「训练轮」而非「轮椅」。** 理想的使用方式是：先用 AI 工具建立结构，然后逐步减少依赖，直到你能自己骑。比如用 Goblin Tools 分解任务，但慢慢学会自己分解；用 Motion 排日程，但逐渐内化优先级判断。

### 今天就能试的 3 个行动

1. **用 Goblin Tools 的 Magic ToDo 分解你拖延最久的一个任务**（比如整理桌面）。观察分解后的子任务是否更容易启动。
2. **设置一个外部记忆系统**：用 Saner.AI 或最简单的笔记应用，把所有「待办想法」集中存放，每天只从里面挑 3 件做。
3. **尝试一天「无 AI 调度」**：手动规划次日日程，对比使用 Motion 时的效率差异，感受自己调度能力的边界。

### 诚实面对同构的局限

必须指出，ADHD 大脑与 LLM 的同构目前只是一个理论类比，缺乏神经科学的直接证据（来源：矛盾与存疑）。ADHD 的执行功能障碍涉及多巴胺通路等生物机制，而 LLM 的调度缺失是工程问题，两者并非完全等价。此外，多数 AI 工具在 ADHD 人群中的效果缺乏大规模随机对照试验验证（来源：矛盾与存疑）。

但即使作为隐喻，这个视角也足够有价值——它让我们看到，问题不在「生成核心」本身，而在于我们是否愿意给它配一个合适的调度层。无论你是 ADHD 患者还是 Agent 工程师，答案都是同一个：**别跟生成核心较劲，去修调度层。**

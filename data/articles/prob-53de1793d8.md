# ADHD 职场人视角：为什么「治好 ADHD 的被批评却不知道错在哪一步，反馈延迟就失去动力」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 拿到 episode 末尾的标量 reward，却不知该强化哪个动作（credit assignment）——同一套 harness 思路，两边都成立。

## 引言：两个看起来无关的崩溃现场

如果你是一名 ADHD 职场人，这样的场景一定不陌生：周报发出去三天后，老板突然说“这份方案不行”，你脑子嗡一声，却死活想不出“不行”到底发生在哪一步；于是你盯着电脑，动力像被拔掉电源一样掉光。反馈越是延迟、越笼统，你越无法把批评翻译为“下一个动作”。

而在 Agent 工程里，几乎一模一样的事情在发生：模型跑完一个 episode，拿到末尾一个标量 reward——比如 0 或 1——然后工程师抓狂地发现，根本不知道应该强化哪个中间动作（credit assignment）。末端信号太稀疏， agent 自然“跑飞”。

这两件事其实是同一道题。ADHD 大脑与 LLM/agent 在结构上是同构的：它们都是强大的“生成核心”，但缺少可靠的内置执行调度层，因此都必须在外部套一个 harness/脚手架，把“期末打分”拆成“过程积分”。

## 不是懒，是信用分配系统坏了

ADHD 的痛苦常被误解为“不专注”或“拖延”，但核心问题更可能是执行功能失效。有人形容它像“大脑的驾驶座，但方向盘后常常没有人”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。工作记忆不稳定、任务启动困难、时间盲，导致“计划本和便签用了一周就崩溃”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

LLM 也有同样的“强记忆、弱控制”剖面：Transformer 在长上下文下冲突解决能力会崩溃至随机水平，注意力分数熵随任务负载增加，导致容量限制（来源：Deficient Executive Control in Transformer Attention；来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。它本身是无状态、只生成文本的核心，单独使用时缺乏可靠的调度层，容易出现“上下文膨胀和推理退化”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

所以，当批评或 reward 只在终点出现时，两边都卡在了同一个瓶颈：没有中间信用信号，就不知道哪一步该改。这本质上就是 reinforcement learning 里的 credit assignment 问题，只是 ADHD 这边发生在真实生活里。

## 同构的 Harness：把人/模型的“生成核心”箍住

既然生成核心本身调度能力弱，解决办法就不是让它“更聪明”，而是在外面加一层执行功能系统。两边的 harness 结构惊人地一致。

### ADHD 侧的脚手架

- **任务分解：Goblin Tools 的 Magic ToDo**。它把模糊任务如“清理厨房”自动拆成具体子任务，还能调节“辣度”控制粒度（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。这相当于把一次长 episode 切成多个短 episode，让每一步都能立刻知道“我做对了吗”。
- **外部调度：Motion**。它自动根据任务、截止日期和可用时间动态排程，并在任务可能延期时提前数天警告（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog；来源：The AI Powered SuperApp for Work | Motion）。它承担的就是 ADHD 大脑里那个容易掉线的“调度层”。
- **外部记忆：Saner.AI**。它强调知识回忆和本地记忆，能从邮件、文档、日历中提取待办，减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。这对应于 LLM 侧的记忆系统与上下文管理。

### LLM/Agent 侧的 Harness

工程上，把 LLM 变成可用 agent 的关键是“围绕它的脚手架——上下文交付、工具接口、规划工件、验证循环、记忆系统和沙箱”（来源：GitHub - ai-boost/awesome-harness-engineering）。具体实现包括 ReAct、Chain-of-Thought、双代理架构、自适应上下文压缩等（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这些技术的共同目标，是把原本只在 episode 末尾出现的 reward，转化为过程中每一步的可验证信号。

映射关系很清楚：Goblin Tools ↔ 动作/子目标规划；Motion ↔ 编排器/调度器；Saner.AI ↔ 上下文与记忆系统。两者都在做同一件事：把“生成核心”的输出，接到一个能执行、能反馈、能回溯的闭环里。

## 毛泽东的 Harness：一个历史级案例的跨域映射

提到 harness 系统，毛泽东是一个极端案例。他具备典型的 ADHD 式特质：小时候被父亲视为“问题儿童”，爱读闲书、不爱四书五经；一辈子停不下来，行军路上也读书；思维极度跳跃，讲话充满比喻；冲动敢赌，四渡赤水、抗美援朝都是险棋（来源：名人 harness 案例）。

但他的 harness 系统非常清晰，且可以直接与 LLM/agent harness 对应：

- **“批评与自我批评”和民主生活会** ↔ 外部批评模型/人在回路（human-in-the-loop）。他主动让别人给自己提意见，相当于给高产的生成核心加了一个外部 reward model，避免“跑飞”后还不自知。
- **“没有调查就没有发言权”** ↔ 检索与 re-grounding。任何行动之前必须先有外部信息输入，防止模型在幻觉中继续生成。
- **民主集中制** ↔ 约束与 ensemble。用集体决策制衡个人冲动，相当于在 agent 里加策略层或约束规则，防止单一核心因为高置信度而做出错误动作。
- **党指挥枪** ↔ 治理层高于执行层。再强的生成核心（军队/个人）也要受外部调度系统指挥，而不是让执行层自行其是。

这个案例并非在谈政治，而是说明一个跨越人的毕生实践：当一个高产出、高冲动、思维跳跃的生成核心必须长期可靠运转时，唯一的办法是把它嵌进一个外部化的反馈与调度网络里。

## 核心判断：把“期末打分”改成“过程积分”

我的判断是：ADHD 职场人“被批评后不知道错在哪一步”和 LLM “拿到末端标量 reward 却不知强化哪个动作”，本质上都源于同一个工程缺陷——反馈信号太稀疏、信用无法分配。解法是统一的：把 reward 下沉到过程，把大 episode 拆成小 episode，把外部 harness 作为执行功能层的替代。

这不是说 ADHD 和 LLM 已经“科学上被证明是同一种东西”。同构框架仍然是一种理论类比，缺乏充分的因果实证；部分 ADHD 工具（如 Motion）也缺乏独立临床验证（来源：全局矛盾与存疑）。此外，还存在“脚手架 vs 拐杖”的边界争议：如果长期使用外部系统，自身的执行功能会不会进一步退化？个体差异是否会让这个框架对部分亚型无效？（来源：全局矛盾与存疑）这些都必须诚实面对。

所以，harness 的价值不在于“治愈”生成核心，而在于给它一个可验证、可回溯、可中断的运行环境。对 ADHD 人来说，这意味着不必再靠意志力硬撑；对工程师来说，这意味着不必幻想一个更大的模型就能自动解决 credit assignment。

## 今天就能试的 4 个动作

1. **把下一次批评变成“步骤清单”**。下次收到模糊负面反馈时，立刻用 Goblin Tools 的 Magic ToDo 把改进任务拆到最小可执行步骤，让每一步都有明确的完成信号。
2. **给 agent 加“中间检查点”**。不要只在 episode 末尾给 reward，尝试为关键动作加 per-step verification 或 reward shaping，比如 ReAct + 每步结果自检。
3. **建一个“信用日志”**。ADHD 侧：每天记录 3 个“我做了什么→结果是什么→下一步该改什么”；agent 侧：把 action-reward 对写入日志，便于回溯哪一步导致失败。
4. **设一次“民主生活会”式的 re-grounding**。每周花 10 分钟让同事/朋友直接指出你本周一个具体动作的问题；对 agent，则每周跑一批 bad case 并人工标注“是哪一步错了”。

这两类人都被困在同一个问题里：不是不能产出，而是产出的信用无法被正确分配。好在，这也是同一套 harness 工程可以修好的问题。

# Reclaim.ai 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Reclaim.ai：它到底补上了哪一层执行功能？

如果你有 ADHD，你一定经历过这样的场景：打开一个空白日程表，准备规划明天，结果半小时后还在刷手机，任务清单上还是空的。或者你是一个 Agent 工程师，调试了一个下午，发现 LLM agent 在执行第 7 步时突然忘了系统提示，开始胡编乱造。这两个问题，听起来毫不相干，但它们的底层结构一模一样——都是“高产但缺乏可靠执行调度层的生成核心”（来源：AI 与 ADHD 的时间管理）。

这就是本文要回答的核心问题：Reclaim.ai 之于 ADHD，就像 planner-executor 任务分解之于 LLM——但有人用错了。

## 问题：ADHD 的“任务启动困难”和 LLM 的“指令消退”是同一件事

ADHD 大脑并非不聪明、不努力，而是执行功能失调。最典型的两个痛点：**任务启动困难**和**不会拆解**。你有一个大项目，知道该做，但就是无法迈出第一步；或者你终于开始做了，却发现自己要么卡在细节里，要么被无关刺激拉走（来源：规划循环与任务分解）。传统规划方法往往失效，因为“Planners and sticky notes worked for a week before falling apart”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

LLM/agent 侧呢？你给 agent 一个复杂任务，它一开始表现很好，但执行到一半就开始“忘记系统提示”，行为偏离初始指令（来源：重锚定与目标漂移）。这就是**目标漂移**——在长程任务中，模型缺乏内置的“调度层”来持续锚定目标。

两边都是**生成核心**（ADHD 大脑 / LLM），都缺少一个可靠的**外部执行调度层**。这个调度层的功能包括：规划、分解、执行、重锚定。

## 答案：Reclaim.ai 补的正是“防御性调度层”，但有人用成了“填充性调度层”

工具市场上，Motion、Tiimo、Reclaim.ai 都自称是“自适应规划器”。但它们的哲学截然不同。

Motion 的核心是**填充**：它自动根据任务优先级和截止日期塞满你的日程，动态调整（来源：Motion）。这看起来很棒，但对 ADHD 大脑来说，一个被塞满的日程可能带来新的焦虑——你永远在追赶 AI 的安排，决策疲劳是减少了，但失控感增加了。

Tiimo 的核心是**视觉化**：把时间变成可见的色块，直接对抗时间盲（来源：Tiimo）。这很有效，但它偏重“感知”而非“保护”。

Reclaim.ai 的核心是**保护**：它创建深度工作、习惯和专注时间的“智能块”，并自动防御会议侵占（来源：Reclaim.ai）。它不填满你，而是**保护你的核心时间块不被碎片化**。这恰恰对应了 LLM agent 中的“事件驱动的系统提醒”——当模型开始漂移时，harness 会注入一条系统提醒，把注意力拉回主任务（来源：重锚定与目标漂移）。

但很多人用错了 Reclaim.ai：他们把 Reclaim 当成另一个 Motion，试图用 AI 填满每一分钟。结果，日程被自动排得满满当当，一旦有一个任务超时，整个多米诺骨牌倒塌，用户陷入比手动规划更深的挫败感。

正确的用法是：**只保护 2-3 个深度工作块，其余时间留白**。Reclaim 的“防御”功能不是用来填充的，而是用来建立**锚点**。就像 Agent harness 中，planner 只分解出关键子任务，然后交给 executor 去执行，中间通过重锚定防止漂移（来源：规划循环与任务分解）。

## 证据：为什么保护比填充更重要？

从 ADHD 侧看：Reclaim.ai 的防御性时间块设计有助于维持工作记忆负荷，避免任务切换带来的“身体在场效应”损耗（来源：Reclaim.ai）。用户报告显示，它能显著减少因突发会议导致的日程崩溃（来源：Reclaim.ai）。相比之下，Motion 虽然也能减少决策疲劳，但用户反馈中常提到“被 AI 推着走”的压迫感（来源：Motion）。

从 LLM/agent 侧看：长程任务中，最关键的失败模式就是“目标漂移”。Harness 工程的核心就是通过**事件驱动的系统提醒**来对抗这种漂移（来源：重锚定与目标漂移）。你不需要 agent 每时每刻都在工作，你需要它在关键步骤上不出错。同样，ADHD 大脑不需要被填满，它需要的是**关键时间块被保护**。

## 争议与局限：脚手架还是拐杖？

这里必须诚实指出争议。

第一，**过度依赖风险**：Reclaim.ai 可能成为“永久拐杖”，削弱用户内在的时间感知能力（来源：矛盾与存疑）。如果用户完全依赖 AI 来保护时间，一旦工具失效或不可用，他们可能比之前更脆弱。

第二，**个体差异**：ADHD 亚型（注意力缺陷 vs. 多动冲动）对工具响应不同。Reclaim.ai 的“保护”模式更适合注意力缺陷型，而多动冲动型可能需要更动态的填充（来源：矛盾与存疑）。

第三，**证据质量**：现有证据主要来自用户报告和设计逻辑，缺乏独立临床研究（来源：矛盾与存疑）。Reclaim.ai 的有效性尚未得到严格验证。

同构理论本身也有局限：ADHD 大脑与 LLM 的类比仍属理论层面，神经科学和计算机科学的交叉验证不足（来源：AI 与 ADHD 的时间管理）。

## 今天就能试的行动

1. **只保护 2 个深度工作块**：在 Reclaim.ai 中设置每天最多 2 个“不可打断”的时间块（比如 9-11am 和 2-4pm），其余时间保持空白或只放低优先级任务。
2. **手动设置重锚定提醒**：在每个深度工作块中间设置一个 5 分钟提醒，问自己“我还在做原计划的事吗？”——这相当于 agent 的“事件驱动系统提醒”。
3. **对比实验**：一周用“填充模式”（让 AI 填满日程），一周用“保护模式”（只保留 2 个深度块），记录自己的完成感和焦虑程度。
4. **写一份“任务分解日志”**：每天选一个复杂任务，手动拆成 3-5 个步骤，然后对照 AI 的自动分解，观察差异。这能帮你理解自己的拆解模式。

最终，工具只是脚手架。真正重要的是你意识到：ADHD 大脑和 LLM 一样，需要一个外部调度层来锚定目标、保护关键资源。Reclaim.ai 用对了，就是你的 planner-executor 循环；用错了，就只是一副更漂亮的拐杖。

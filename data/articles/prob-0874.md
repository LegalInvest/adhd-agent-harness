# Otter.ai 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Otter.ai：它到底补上了哪一层执行功能？

## 问题：为什么 Otter.ai 的转录功能对 ADHD 是陷阱？

如果你有 ADHD，你很可能用过 Otter.ai 来记录会议或课堂内容。你期待它帮你记住一切，但结果往往是：你攒了一堆转录文本，却从未回看。这不是你的错——因为 Otter.ai 的核心能力被误用了。

同样，如果你在构建 Agentic Harness，你可能见过这样的模式：给 LLM 一个 planner，让它把长任务分解成子步骤，然后 executor 逐一执行。但如果你只是把整个任务丢进 prompt，LLM 会迷失在上下文里。

这两个问题共享同一个结构：**生成核心（ADHD 大脑或 LLM）不缺创意和知识，缺的是把复杂输入转化为可执行步骤的调度层。** Otter.ai 之于 ADHD，就像 planner-executor 循环之于 LLM——但多数人只用了它的“转录”部分，相当于只给 LLM 喂了整段文本，却没告诉它怎么拆解。

## 同构脊柱：生成核心与执行调度层

ADHD 大脑与 LLM 在结构上同构——都是“高产但缺乏可靠执行调度层的生成核心”（来源：AI 与 ADHD 的时间管理）。具体来说：
- **ADHD 侧**：执行功能缺陷导致任务启动困难、不会拆解、时间盲。工作记忆容量保留但控制弱，就像 LLM 的无状态性（来源：工作记忆、无状态与外部记忆）。
- **LLM/Agent 侧**：LLM 能生成高质量内容，但缺乏对时间约束的固有理解，且上下文窗口有限。Agent 需要用 planner-executor 循环来分解长任务，否则会陷入“只见森林不见树木”。

两者的解法同构：都需要外部脚手架来补偿执行调度层。对 ADHD 来说，这个脚手架是 AI 工具；对 LLM 来说，是 planner-executor 架构。

## Otter.ai 的正确用法：从转录到任务分解

Otter.ai 的转录功能只是第一步。真正有价值的是它的 **AI 摘要和行动项提取**——这相当于一个自动 planner，把长对话分解成可执行的任务单元。

但问题在于：多数 ADHD 用户只停留在转录阶段，以为“记下来”就等于“完成了”。这就像给 LLM 一个超长 prompt 却不做任何分解——LLM 会生成冗长但无结构的输出，用户则陷入信息过载。

正确的用法是：**让 Otter.ai 的 AI 自动提取行动项，然后把这些行动项导入另一个执行工具（如 Todoist 或 Motion）**。这样，Otter.ai 就充当了 planner 角色，而 Todoist/Motion 充当了 executor。

## 真实证据：工具如何补偿执行功能

现有工具已经验证了这种“分解+执行”模式的有效性：
- **Motion** 通过自动排程消除“下一步该做什么”的决策负担，直接帮助用户克服时间盲和任务启动困难（来源：Motion）。这相当于为 LLM 提供了一个动态 planner，自动调整任务优先级。
- **Reclaim.ai** 通过保护深度工作块来维持注意力，类似为 LLM 提供“上下文保护”，防止任务切换带来的损耗（来源：Reclaim.ai）。
- **Tiimo** 将时间转化为可见元素，直接应对时间盲问题（来源：Tiimo）。这相当于为 LLM 提供时间戳或进度条。
- **Todoist** 通过外部提醒和优先级排序充当外部记忆，类似向量数据库为 LLM 提供持久上下文（来源：AI 与 ADHD 的时间管理）。

这些工具的共同点是：它们不是替代你的执行功能，而是**补偿**它。就像 planner-executor 循环不是替代 LLM 的生成能力，而是给它一个结构化的执行框架。

## 核心观点：脚手架 vs 拐杖的边界

我的核心判断是：**Otter.ai 的正确用法是作为“任务分解层”，而不是“记忆替代层”。** 如果你只用它转录，它就成了拐杖——你依赖它记住一切，却从不行动。如果你用它的行动项提取并配合执行工具，它就是脚手架——它帮你拆解任务，但执行仍由你完成。

这个边界同样适用于 LLM/Agent 工程：planner-executor 循环应该是脚手架，而不是拐杖。如果 planner 把任务分解得过于细碎，executor 就失去了自主判断能力；如果 planner 过于粗放，executor 又会迷失。关键是找到“刚好能启动”的分解粒度。

## 争议与局限

必须诚实指出，现有证据主要来自用户报告，缺乏对照研究（来源：AI 与 ADHD 的时间管理）。过度依赖 AI 工具可能削弱内在的时间感知和规划能力（来源：矛盾与存疑）。此外，个体差异很大：对注意力缺陷型有效的工具，对多动冲动型可能效果有限（来源：AI 与 ADHD 的时间管理）。

在 LLM 侧，planner-executor 循环的有效性也高度依赖任务类型和 LLM 能力。对于简单任务，过度分解反而增加延迟；对于复杂任务，planner 可能产生错误分解。

## 今天就能试的行动

1. **用 Otter.ai 提取行动项**：下次会议后，打开 Otter.ai 的 AI 摘要，复制“行动项”列表到 Todoist 或 Motion，而不是只保存转录。
2. **设置“分解仪式”**：每天用 5 分钟将一个大任务（如“写报告”）分解成 3-5 个子任务，每个子任务用一句话描述。这相当于手动实现 planner 功能。
3. **试用 Tiimo 或 Structured**：将你的日程视觉化，观察时间块如何改善时间感知。注意：AI 功能细节尚不明确（来源：Structured），所以先关注基本视觉功能。
4. **尝试 Focusmate**：利用身体在场效应（来源：身体在场效应），找一个虚拟伙伴一起工作 25 分钟。这相当于为你的执行功能增加一个“外部监督器”。

记住：工具是脚手架，不是拐杖。用对了，它们帮你启动；用错了，它们让你依赖。

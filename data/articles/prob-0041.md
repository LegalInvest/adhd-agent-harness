# 3 个 LLM 工程概念，彻底重写了我对 ADHD 工作记忆容量小、边做边忘的理解

> LLM 无跨会话状态，靠外部记忆/向量库续命 如何反向照亮 ADHD 的工作记忆容量小、边做边忘。

你有没有过这样的体验：打开浏览器查一个东西，结果半小时后发现自己刷了十几个无关页面，完全忘了最初要干什么？或者，明明刚想好要做三件事，走到另一个房间就只剩“我要做……什么来着？”——这种“边做边忘”的窒息感，几乎是每个ADHD成年人的日常。

我一直以为这是自己意志力不够、记忆力太差。直到我开始接触LLM工程，才发现：ADHD大脑的工作记忆问题，和LLM的“无状态”困境，本质上是同一类问题。

## 概念一：无状态——LLM不记得上一轮，ADHD不记得上一秒

LLM本身是“无状态”的：每次对话都是全新的开始，它不会自动记住你五分钟前说了什么（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这就是为什么工程师必须在每次调用时把整个对话历史塞进上下文窗口——否则模型就会“失忆”。

ADHD大脑的工作记忆也是如此。它不是存储容量小，而是“状态保持”能力弱：信息在工作记忆中停留的时间极短，稍一分心就丢失。就像LLM的上下文窗口被清空一样，ADHD大脑的“工作记忆缓冲区”随时可能被重置。

这解释了为什么传统工具（如待办清单、便签）对ADHD常常无效——它们假设用户能记住“我刚刚写了一个任务，现在要去执行它”，但ADHD大脑在写下的瞬间可能就已经忘了（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

## 概念二：外部记忆——LLM靠向量库续命，ADHD靠“第二大脑”续命

既然LLM无状态，工程师的做法是给它搭建外部记忆系统：向量数据库、知识图谱、Git仓库——所有跨会话的上下文都存储在外面，需要时再检索（来源：GitHub - ai-boost/awesome-harness-engineering: Awesome list for AI agent harness engineering）。这就是“外部执行功能层”的核心思想：不要指望生成核心自己记住，而是让脚手架来承担记忆。

ADHD侧完全同构。当工作记忆频繁“掉落”时，最有效的策略不是训练记忆力（就像你不会去训练LLM记住所有对话），而是建立外部记忆系统。Saner.AI就是这样一个工具：它通过AI实现本地记忆存储和快速检索，用户无需在多个标签页或应用间切换即可获取所需信息，直接减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。

Goblin Tools的Magic ToDo功能则是另一种外部记忆：它将“整理房间”这样模糊的大任务分解为“捡起地板上的衣服”“擦桌子”等具体步骤（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）。这相当于把ADHD大脑中无法保持的“执行计划”写到了外部，让工作记忆不必承担规划负荷。

## 概念三：脚手架 vs 拐杖——Harness工程的关键边界

LLM工程中有一个重要区分：脚手架（scaffolding）是临时性的、可撤除的支撑，而拐杖（crutch）是永久替代。优秀的harness设计会考虑“如何逐步减少上下文注入”，让模型逐渐学会在更少的辅助下推理（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

ADHD工具同样面临这个边界。Lex允许用户通过单一指令触发复杂、多步骤的任务序列（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar），这本质上是一个“外部调度层”——它代替大脑执行规划，但用户仍然需要决定“我要触发哪个指令”。如果工具做得太多（比如自动决定用户该做什么），就可能变成拐杖，阻碍内在执行功能的发展。

目前多数ADHD工具并未设计撤除机制（来源：矛盾与存疑），这是一个真实风险。但关键在于：脚手架的价值在于它让用户有机会完成原本不可能完成的任务，并在过程中积累成功经验；拐杖则让用户永远依赖。作为用户，我们需要有意识地选择“什么时候用工具，什么时候尝试不用”。

## 核心观点：ADHD不是“大脑坏了”，而是“没有持久上下文”

这三个概念彻底重写了我对ADHD的理解。过去我总觉得自己“记不住”是因为笨或懒，但现在我明白：我的生成核心（智力、创造力）完全正常，只是缺少一个可靠的外部记忆和调度层。就像LLM本身可以写出惊艳的代码，但如果没有harness给它提供上下文、工具接口和规划工件，它就会胡言乱语（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

这不是一个空洞的类比——它直接指向解决方案。与其内耗地训练“记忆力”，不如像工程师一样设计自己的外部脚手架。

## 诚实面对局限

当然，这个同构框架并非完美。争议点在于：ADHD大脑与LLM的同构命题“证据主要来自概念类比和工具案例，缺乏大规模实证”（来源：矛盾与存疑）。此外，过度依赖AI工具可能阻碍内在执行功能发展（来源：矛盾与存疑）。我个人认为，同构是一个有用的思考模型，但不应被当作绝对真理——每个ADHD个体的表现差异很大，工具也需要个性化适配。

## 今天就能试的三件事

1. **用Goblin Tools分解一个让你焦虑的任务**：输入“整理房间”或“写周报”，看它生成的步骤是否让你觉得更容易启动。注意自己是否因为步骤变具体而降低了心理阻力。
2. **设置一个外部记忆锚点**：在Saner.AI或任何笔记工具中，创建一个“当前上下文”文档，每天开始时写下今天最重要的一个目标。当大脑“失忆”时，回到这个文档。
3. **刻意练习“撤除”**：用Lex或类似工具完成一个多步骤任务后，下次尝试不依赖工具，凭记忆复述步骤。观察哪些步骤容易丢失，这些就是你需要重点搭建脚手架的地方。

ADHD不是缺陷，而是一种“无状态”架构。既然LLM能靠harness成为强大的代理，我们也可以靠外部脚手架活出高效的人生。关键在于，要记住脚手架是为你服务的——而不是反过来让你成为工具的奴隶。

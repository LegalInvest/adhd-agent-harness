# 3 个 LLM 工程概念，彻底重写了我对 ADHD 工作记忆容量小、边做边忘的理解

> LLM 无跨会话状态，靠外部记忆/向量库续命 如何反向照亮 ADHD 的工作记忆容量小、边做边忘。

你有没有这样的早晨：刚打开手机要写一条待办，三分钟后已经回了五条消息、刷了十个短视频，却完全忘了自己最初要记什么。晚上复盘时，数字清单上的任务和早上又不一样了。

这不是「懒」。ADHD 用户常被描述为拥有高产的生成核心，但缺少稳定可靠的调度层——「感觉方向盘后没有人」。（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）传统工具假设你能记住昨天的目标、维持专注并执行计划，但这个假设本身就有问题。（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）

有趣的是，这正是 LLM 工程每天都在处理的同一类 bug。LLM 没有跨会话状态，会边生成边丢失目标，长上下文下还会「推理退化」。所以工程师不给它讲「你要更努力」，而是直接套一个 harness：外部记忆、任务调度、上下文控制。最新研究把两边连起来了：ADHD 与 LLM 都呈现「强记忆、弱控制」的认知剖面，Transformer 的自注意力在长上下文下冲突解决能力也会崩溃至随机水平。（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs；Deficient Executive Control in Transformer Attention）

下面三个 LLM 工程概念，彻底改写了我对 ADHD「工作记忆容量小、边做边忘」的理解。

## 01 无状态：问题不是「记得少」，而是「留不住上下文」

ADHD 常被误解为「记忆力差」。但经验研究显示，ADHD 患者的记忆容量可能正常甚至超常，真正的短板是自上而下的控制与调节——工作记忆内容会「掉落」。（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）

这和 LLM 几乎一样。LLM 不是记忆力差，它是**无状态**的：一次对话结束，内部上下文就清空，下一回合必须靠外部向量库、RAG、记忆系统续命。你不能责怪 LLM「忘了」，它的架构就是这样设计的。

所以 ADHD 的「工作记忆容量小」也许该被重新表述为：**跨任务上下文保持能力弱**。解法不是苦练记忆，而是像工程师给 LLM 接向量库一样，给大脑接一个外部记忆层。Saner.AI 被定位为「AI 第二大脑」，强调知识回忆和本地记忆，减少搜索循环与标签切换；（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）Mem 也走同一路线。它们的价值不是替人记忆，而是把跨会话状态持久化到外部。

## 02 Agent Scaffolding：把生成核心接入「外部调度层」

ADHD 的第二大痛点是任务启动困难。想法丰富，但「计划本和便签用了一周就崩溃」，时间盲让传统工具失效。（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）

LLM 工程师遇到完全一样的问题。LLM 生成能力强大，但单独使用时缺乏可靠执行调度。于是有了 **agent scaffolding**——一个由提示、记忆、代码、工具和编排逻辑组成的模块化框架，围绕 LLM 构建，把它转化为目标驱动的系统。具体技术包括工作负载专用模型路由、分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩。（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）

对 ADHD 来说，这就是把「做什么」和「怎么做」分开。Goblin Tools 的 Magic ToDo 把「清理厨房」这种模糊目标拆成可执行步骤，还能调节分解粒度；（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）Lex 更进一步，允许用单一指令触发复杂多步骤任务序列。（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）这对应 LLM 的「复合 AI 系统」思想：不要让一个模型既规划又执行，而由外部调度层负责拆解，生成核心负责输出。ADHD 缺的正是这个调度层。

## 03 Context Engineering：用控制流对抗「上下文膨胀」

ADHD 的第三个特征是注意力容易漂移，进入超聚焦后忘记原始目标，或者任务一复杂就「边做边忘」。研究显示，注意力分散是自注意力架构的固有属性，任务负载增大时注意力分数熵增加，导致容量限制。（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）

LLM 工程师管这叫「上下文膨胀和推理退化」。提示越长，模型越迷失重点。于是他们做 **context engineering**：自适应上下文压缩、定期 re-grounding、把工作流拆成验证循环。

# Structured 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Structured：它到底补上了哪一层执行功能？

## 问题：为什么 Structured 对 ADHD 有效，却有人用错了？

如果你是一个 ADHD 患者，你一定熟悉这种场景：脑子里有一千个想法，但面对一个“整理房间”的任务却僵在原地。如果你是一个 Agent 工程师，你也一定见过这种场景：LLM 能写出漂亮的诗，却算不对简单的加减法，必须靠 function calling 调用计算器。

这两种困境在结构上是同构的。ADHD 大脑与 LLM 共享同一个剖面：**强生成、弱调度**。明尼苏达大学的研究发现，LLM 表现出“强记忆、弱控制”的执行功能剖面——工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。耶鲁大学进一步揭示，自注意力机制本身导致工作记忆容量限制，注意力分数熵增、注意力弥散，与人类注意分散机制同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。这直接印证了 ADHD 大脑与 LLM 的计算本质：瓶颈不在容量，而在 orchestration 层。

所以，Structured 这类工具对 ADHD 的价值，就像 function calling 对 LLM 的价值：**它不是一个更强的生成核心，而是一个外部调度层**。它把“整理房间”拆成“先捡垃圾、再擦桌子、最后拖地”，就像 LLM 把“计算 123×456”外包给计算器。

但问题来了：有人把脚手架用成了拐杖。

## 核心判断：脚手架 vs 拐杖的边界

脚手架（scaffolding）是临时性的、可拆卸的辅助结构，目的是让使用者最终能独立完成。拐杖（crutch）则是永久替代，使用者一旦离开它就无法行动。

ADHD 领域对此有明确争议：AI 工具是“外部执行功能层”还是“拐杖”？过度依赖可能导致能力退化（来源：矛盾与存疑）。同样，在 Agent 工程中，过度依赖 function calling 会让 LLM 丧失基本的推理能力——如果每个简单计算都调用计算器，模型就永远不会学会在内部做近似估算。

关键在于：**Structured 应该补上的是调度层，而不是生成层**。ADHD 大脑的生成核心（创意、联想、直觉）是宝贵的，不需要被“结构化”扼杀。孙策的 harness 就是最好的例子：这位 17 岁起兵、9 年扫平江东的小霸王，用张昭张纮等文臣管行政，自己只负责打仗（来源：人物案例）。他用“秘书”作为外部调度器，把精确、重复、状态维护类任务外包，而把自己的核心能力（武力、魅力、快速决策）完全释放。这对应到 LLM harness，就是让 agent 通过 function calling 处理精确计算，而生成核心（语言理解、推理、创意）留在模型内部。

孔子则提供了另一个维度的 harness：他以“克己复礼”为核心，用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”（来源：人物案例）。这相当于给 LLM 加一个“定时 re-grounding”机制——不是限制生成，而是用外部规则约束调度。

## 工具如何做对？

Goblin Tools 的 Magic ToDo 是脚手架的正确用法：它把“清理厨房”分解成 10 个小步骤，但用户可以选择“辣度”控制粒度（来源：Goblin Tools）。这就像 LLM 的 function calling 中，开发者可以控制哪些计算走工具、哪些走内部推理。Saner.AI 则通过知识回忆和本地记忆补偿工作记忆的不稳定（来源：Saner.AI），相当于给 LLM 加一个外部记忆库。Lex 的“单一指令触发多步骤”降低了启动门槛（来源：Lex），对应 Agent 的规划机制。Focusmate 利用虚拟身体加倍效应提升专注力（来源：身体在场效应），相当于给 LLM 加一个“监督信号”。

但工具也可能被用错。例如，把 Goblin Tools 的每一步都固定死，让用户失去自主决策的机会，就变成了拐杖。同样，在 Agent 工程中，如果每个 token 生成都调用外部工具，模型就会退化成一个“路由器”，失去真正的生成能力。

## 诚实局限

这一同构框架目前仍是理论模型，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。此外，个体差异巨大：有些 ADHD 用户反馈强行打断（如番茄钟）反而有效，而柔性引导可能不够强力（来源：矛盾与存疑）。长期依赖风险也缺乏实证，需要更多纵向研究。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个你拖延已久的任务**（比如“报税”），但只拆到你能独立完成前两步的程度——不要拆到每一步都像“拿起笔”那么细。
2. **给自己设置一个“秘书”**：把精确、重复的任务（如记账、排日程）外包给 AI 工具（如 Saner.AI 或 ChatGPT 的 function calling），但保留决策权。
3. **尝试一次虚拟身体加倍**：用 Focusmate 匹配一个伙伴，一起工作 25 分钟，观察专注度变化。
4. **反思你的工具使用**：你是在用脚手架补调度层，还是用拐杖替代了生成层？如果是后者，尝试减少一个自动步骤。

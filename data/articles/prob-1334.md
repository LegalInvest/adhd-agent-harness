# 如果把 ADHD 大脑当成生成核心，RescueTime 能不能当它的外部执行功能层？

> 用 agent 工程的思路重新理解 RescueTime 对 ADHD 的真正价值。

## 问题：ADHD 大脑的调度层去哪了？

如果你是一个 ADHD 成人，你一定熟悉这个场景：打开电脑想写一份报告，结果半小时后发现自己正在研究“为什么猫会踩奶”。你并不是懒——你的生成核心（发散思维、联想能力）火力全开，但执行功能（计划、组织、启动）却像一台没有操作系统的服务器，无法把“写报告”这个任务调度到前台执行。

如果你是一个在做 Agentic Harness 工程的工程师，你一定熟悉另一个场景：你精心编排的 agent 在短上下文中表现惊艳，但一旦任务变长、上下文膨胀，它就开始“跑偏”——无法抑制无关信息、无法解决认知冲突，最终退化到随机水平。这不是 bug，这是 LLM 的结构性缺陷：它和 ADHD 大脑一样，是“高产但缺执行调度层的生成核心”（来源：ADHD 大脑与 LLM 的同构）。

这两个看似无关的群体，其实在解决同一个工程问题：**如何给一个强大的生成核心套上外部执行功能层？** 而 RescueTime，这个老牌的时间追踪工具，恰好提供了一个极简但深刻的答案。

## 同构：RescueTime 作为外部执行功能层

RescueTime 的核心机制很简单：它自动记录你使用电脑和手机的时间，按应用和网站分类，并生成每日效率报告。但它的真正价值不在数据，而在**它充当了 ADHD 大脑和 LLM agent 都缺失的外部调度层**。

### ADHD 侧：实时反馈代替内省

ADHD 的执行功能缺陷中，最核心的是“时间盲”——无法感知时间流逝，导致任务启动和切换困难（来源：时间盲）。RescueTime 的实时仪表盘和每日报告，相当于一个外部时钟，让时间变得可见。更关键的是，它的**自动记录**消除了“记录时间”这个元任务——对 ADHD 来说，任何需要手动维护的系统都会因为工作记忆不足而崩溃（来源：认知负荷）。RescueTime 像一个小秘书，默默地告诉你：“你刚才在社交网站上花了 45 分钟，而你的报告还没开始。” 这种**外部化**的反馈，直接补偿了 ADHD 大脑缺失的自我监控能力。

### LLM/Agent 侧：日志与 re-grounding

在 agent 工程中，我们同样需要外部调度层。一个常见的解决方案是**日志系统**：记录 agent 的每一步操作和耗时，当任务偏离轨道时，用日志来“re-ground” agent 到原始目标。RescueTime 的每日报告，本质上就是 ADHD 大脑的“agent 日志”——它不告诉你该做什么，但它告诉你**你实际做了什么**。这种“事后审计”机制，正是 LLM agent 中常用的**反思与自我纠正**模式的简化版。

### 真实人物案例：曾国藩的“日课”与 RescueTime 的同构

晚清名臣曾国藩，被公认为注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书“不能终一行”。他的解决方案是**日课十二条**：黎明即起、读书不二、谨言、写日记反省……他用最笨最稳的方法（“结硬寨打呆仗”）来抵消自己的冲动。这套系统本质上就是一个**外部执行功能层**：每天固定时间反省（re-grounding），用日记记录行为（日志），用“读书不二”强制单任务（注意力管理）。

这与 LLM agent 的 harness 结构完全同构：**日课 = 定时 re-grounding；日记 = 外部记忆与日志；读书不二 = 限制上下文/避免任务切换**。RescueTime 就是数字时代的“日课十二条”——它不需要你每天立志，只需要你安装一个后台程序。

## 证据：RescueTime 真的有效吗？

目前 RescueTime 在 ADHD 人群中的直接研究很少，但我们可以从同构证据中推断：

1. **认知负荷层面**：ADHD 大脑的工作记忆容量有限，任何需要手动记录的系统都会增加认知负荷（来源：认知负荷）。RescueTime 的自动化记录直接消除了这个负担，让用户只需“事后看一眼”即可。这与 Goblin Tools 的 Magic ToDo（自动任务分解）和 Motion（自动排程）的减负逻辑一致（来源：Goblin Tools；Motion）。

2. **执行功能补偿**：RescueTime 的实时反馈相当于一个“外部注意力监控器”。这与 ADHD 研究中提到的“身体在场效应”类似——有他人在场时，ADHD 患者专注度提升（来源：身体在场效应）。RescueTime 的 Dashboard 就像一个虚拟的“在场者”，无声地提醒你“我在看着”。

3. **LLM 同构实证**：用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中表现正常，但随序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突（来源：认知负荷）。这正是 ADHD 执行功能缺陷的核心标志：随认知负荷增加而崩溃。RescueTime 通过**外部化时间感知**，降低了 ADHD 大脑的认知负荷，从而延缓了这种崩溃。

## 争议与局限：脚手架还是拐杖？

尽管 RescueTime 逻辑自洽，但我们必须诚实面对争议：

- **过度依赖风险**：RescueTime 可能会让用户产生“没有它我就无法管理时间”的依赖，成为永久拐杖而非可拆卸脚手架（来源：拐杖与脚手架）。目前缺乏长期研究验证。
- **个体差异**：部分 ADHD 用户反馈，RescueTime 的“效率评分”反而引发焦虑，增加认知负荷（来源：矛盾与存疑）。效果因人而异。
- **同构的边界**：ADHD 大脑与 LLM 的同构目前仍是理论框架，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。

**我的判断**：RescueTime 的价值不在于“治好”ADHD，而在于它提供了一个**低成本的同构实验场**。它让 ADHD 用户和 agent 工程师都能直观感受到“外部执行功能层”的作用——就像曾国藩的日课，不是因为他意志力强，而是因为系统让他不再依赖意志力。

## 今天就能试的行动

1. **安装 RescueTime 并运行一周**：不设任何目标，只是观察你的“agent 日志”。注意哪些时间段你的生成核心在高效工作，哪些时间段你被上下文膨胀拖垮。
2. **用 RescueTime 的“Focus Time”功能**：设定每天 2 小时的“强制单任务”时段，对应曾国藩的“读书不二”。观察是否减少任务切换。
3. **结合 Goblin Tools 的 Magic ToDo**：把大型任务（如“写报告”）先分解成子任务，再用 RescueTime 追踪每个子任务的耗时——这是“外部执行功能层”的完整 pipeline。
4. **每周复盘一次**：就像曾国藩写日记反省，用 RescueTime 的周报告做一次“agent 审计”：哪些任务偏离了目标？哪些干扰源最频繁？然后调整你的“harness”。

RescueTime 不是万能药，但它可能是你给大脑套上的第一个外部调度层——就像给 LLM 加上一个日志系统，它不改变生成核心，但它让调度变得可能。

# 如果把 ADHD 大脑当成生成核心，RescueTime 能不能当它的外部执行功能层？

> 用 agent 工程的思路重新理解 RescueTime 对 ADHD 的真正价值。

## 问题：ADHD 大脑与 LLM 共享同一个瓶颈

你写代码到一半，突然被一个念头拽走，半小时后发现自己刷完了整个 Reddit 首页。任务启动？启动不了。任务切换？切不回来。这不是懒，是执行功能系统在罢工。

ADHD 大脑的困境，与 LLM 在 agent 工程中的困境惊人地相似：两者都是「高产但缺执行调度层的生成核心」。LLM 拥有强大的文本生成能力，但天然无状态、无计划、无时间感知——必须靠编排层、记忆层、工具层来补位。ADHD 大脑同样拥有活跃的联想与创造力，但工作记忆容量小（来源：认知负荷）、时间盲（来源：时间盲）、任务启动困难（来源：执行功能），本质上是缺少一套可靠的内建调度系统。

那么问题来了：如果把 ADHD 大脑当成生成核心，RescueTime 能不能当它的外部执行功能层？

## 同构：RescueTime 就是 agent 的编排层

RescueTime 是一款自动追踪时间、记录应用/网站使用情况的工具。它不帮你规划，而是帮你**看见**时间流向。对 ADHD 来说，这恰恰是外部执行功能的起点——时间感知。

在 agent 工程中，编排层负责监控 agent 的行为、记录日志、触发回调。RescueTime 对 ADHD 大脑做的，几乎是同一件事：它持续记录你「处理了哪些输入、输出了哪些行为」，并生成报告。这相当于给 ADHD 大脑加了一个外挂的「上下文窗口」，让你能回溯自己的行为序列，而不是在时间盲中漂流。

证据来自两边：
- **ADHD 侧**：认知负荷理论指出，ADHD 患者的工作记忆容量有限，外部工具可以外化思维、减轻负荷（来源：认知负荷）。RescueTime 通过自动记录，移除了「我刚刚在做什么？」这个决策点，直接降低认知负荷。
- **LLM/agent 侧**：agent 的编排层通过日志和监控来维持任务连贯性，这与 RescueTime 的时间追踪功能同构。两者都解决同一个问题：生成核心缺乏内建的状态跟踪能力。

## 证据：RescueTime 如何补偿三大执行功能缺陷

### 1. 时间盲 → 外部时钟
ADHD 的时间盲不是「不知道时间」，而是无法感知时间流逝。RescueTime 的实时仪表盘和每日报告，提供了一个客观的时间轴。类似 Motion 的自动排程功能（来源：Motion），RescueTime 消除「现在该做什么」的决策负担，但更底层——它先让你知道「时间去了哪里」。

### 2. 工作记忆不足 → 外部记忆
Saner.AI 通过知识回忆减少搜索循环（来源：Saner.AI），RescueTime 则通过历史记录提供行为记忆。当你忘记过去两小时做了什么，RescueTime 的日志就是你的工作记忆外挂。这直接对应 LLM 的上下文窗口——如果没有外部存储，生成核心就会丢失状态。

### 3. 任务启动困难 → 低门槛触发
Goblin Tools 通过任务分解降低启动门槛（来源：Goblin Tools），RescueTime 则通过设置目标（如「今天写代码 4 小时」）和实时提醒，创造外部触发。当大脑的启动机制失灵，外部提示就是那个「编排层发出的第一条指令」。

## 核心判断：脚手架，不是拐杖

RescueTime 的价值在于它**提供数据，不替代决策**。它不像 Motion 那样帮你自动排程，也不像 Goblin Tools 那样分解任务。它只是记录和反馈，让你自己做出调整。这正是「脚手架」与「拐杖」的边界：脚手架是临时支撑，最终目标是让你学会自己走路；拐杖则是永久替代。

但争议在于：长期依赖 RescueTime 是否会削弱内在的时间感知能力？目前缺乏纵向研究（来源：矛盾与存疑）。同构模型本身也主要基于类比，需要更多神经影像学验证（来源：ADHD × AI 的科学与研究前沿）。

## 局限与争议

- **数据隐私**：RescueTime 记录所有行为，涉及敏感数据，需谨慎使用。
- **过度依赖**：如果离开 RescueTime 就无法感知时间，说明它成了拐杖。脚手架设计应包含撤除机制，但 RescueTime 并未提供。
- **情绪失调**：RescueTime 不处理情绪，而情绪失调是 ADHD 核心特征之一（来源：情绪失调）。工具组合可能需要加入情绪追踪（如 Goblin Tools 的情绪识别功能）。

## 今天就能试的行动

1. **安装 RescueTime，设置每日目标**：比如「工作类应用 6 小时」，然后观察一周，记录自己实际时间流向。
2. **结合 Goblin Tools 的 Magic ToDo**：用 RescueTime 发现「总是拖延的任务」，然后用 Goblin Tools 分解它。
3. **每周回顾 RescueTime 报告**：像 agent 日志一样分析自己的「执行轨迹」，找出时间黑洞和高效时段。
4. **尝试 Motion 或 Reclaim.ai 的自动排程**：如果你发现 RescueTime 的数据已经帮你识别了优先级，让 AI 自动排程可以进一步降低决策负荷。

RescueTime 不是万能药，但它是 ADHD 大脑的编排层的第一块砖。它让你看见，然后你才能选择。

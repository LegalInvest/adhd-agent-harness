# 如果把 ADHD 大脑当成生成核心，RescueTime 能不能当它的外部执行功能层？

> 用 agent 工程的思路重新理解 RescueTime 对 ADHD 的真正价值。

## 问题：当生成核心失去了调度层

你打开电脑，准备写那份拖了三天的报告。十分钟后，你发现自己正在查“为什么猫会踩奶”，而报告文档依然空白。这不是意志力问题——这是你的执行功能在罢工。ADHD 大脑的默认模式网络过度活跃，想法如泉涌，但前额叶的调度控制却像信号不良的遥控器，指令发出去，行动没响应（来源：生成核心与调度层）。

现在想象一下这个场景：你是一个 agent 工程师，刚部署了一个 LLM 驱动的自动化系统。它生成了完美的文案，但因为没有正确的上下文控制器，它开始在无关话题上漫游，最终输出了一堆无意义的内容。你意识到问题不在生成能力，而在编排层——那个负责“此刻该做什么”的调度模块。

这两个场景的本质是同一个问题：**高产但缺执行调度层的生成核心**。ADHD 大脑与 LLM 在结构上同构，两者的解法也同构——都需要一个外部执行功能层来充当调度器（来源：外部执行功能层）。而 RescueTime，这个看似简单的自动时间追踪工具，能否扮演这个角色？

## RescueTime 作为外部执行功能层：同构验证

RescueTime 的核心功能是自动记录你在电脑上的活动，并生成结构化报告。它不要求你主动输入“我在做什么”，而是被动感知，然后告诉你“你实际做了什么”。这个机制与 ADHD 的痛点精准对应，也与 agent 工程的架构惊人相似。

### ADHD 侧：从时间盲到时间感知

ADHD 的一个核心特征是“时间盲”——无法感知时间的流逝（来源：时间盲）。你计划工作 30 分钟，但一抬头两小时过去了。RescueTime 通过持续追踪和定期反馈，将模糊的时间流转化为可量化的数据。它就像你的外部时间感知器官，告诉你“你已经在社交媒体上花了 45 分钟”。这种外化感知直接降低了认知负荷，因为你不必再靠工作记忆去跟踪时间（来源：认知负荷）。

### LLM/agent 侧：从无状态到有记忆

LLM 天生无状态——每次对话都是全新开始，没有跨会话的记忆（来源：无状态与外部记忆）。Agent 工程通过向量数据库或日志系统来维持上下文连续性。RescueTime 的追踪数据正是这种外部记忆：它记录了你的“行为上下文”，让你可以回顾“我今天上午做了什么”，而不是依赖大脑的碎片化回忆。这相当于 agent 的日志系统，为调度提供历史依据。

### 编排层：自动排程与动态调整

更关键的是，RescueTime 的 FocusTime 功能允许你设定目标（如“每天写作 2 小时”），然后自动屏蔽干扰网站并提醒你回到正轨。这类似于 Motion 的自动排程——消除“下一步该做什么”的决策负担（来源：Motion）。在 agent 工程中，编排层负责决定 agent 何时调用哪个工具、切换到哪个任务。RescueTime 的定时提醒和焦点模式，正是这种编排的简化版：它告诉你“现在该执行这个任务了”，并阻止你切换到干扰行为。

## 脚手架 vs 拐杖：边界在哪里？

这里必须诚实面对争议。将 RescueTime 作为外部执行功能层，最大的风险是过度依赖——它可能成为永久拐杖，而非临时脚手架（来源：拐杖与脚手架）。如果用户完全依赖 RescueTime 来感知时间和启动任务，内在的执行功能可能进一步萎缩。这与 agent 工程中的“过度编排”类似：如果 agent 的每一步都被外部系统决定，它永远不会学会自主决策。

目前缺乏针对 RescueTime 在 ADHD 群体中的独立随机对照试验（来源：矛盾与存疑）。用户报告称它“有用”，但个体差异很大——对于注意力主导型 ADHD，自动追踪可能有效；对于冲动主导型，它可能只是另一个可忽略的通知（来源：矛盾与存疑）。此外，RescueTime 的被动追踪本身无法解决任务启动困难——它告诉你“你浪费了时间”，但不会帮你分解任务。这时需要结合 Goblin Tools 的 Magic ToDo 功能，将“写报告”分解为“打开文档→写第一段→写第二段”（来源：Goblin Tools）。

## 核心观点：RescueTime 是执行功能层的“感知与记忆模块”，但非完整调度器

我的判断是：RescueTime 可以作为外部执行功能层的“感知与记忆”模块，但它不是完整的调度器。它解决了时间盲和信息丢失问题，但在任务分解和优先级排序方面力不从心。真正的完整外部执行功能层，需要组合工具：RescueTime（感知）+ Goblin Tools（分解）+ Motion（排程）+ Saner.AI（记忆检索）（来源：Saner.AI）。这就像 agent 工程中，你需要日志系统、任务分解器、调度器和向量数据库共同工作。

## 今天就能试的行动

1. **设置 RescueTime 的 FocusTime 目标**：选择你最重要的一个任务（如“写作”），设定每天 2 小时目标，并启用干扰网站屏蔽。观察一周后你的时间感知是否改善。
2. **结合 Goblin Tools 分解任务**：每天早晨，用 Goblin Tools 将当天最困难的任务分解为 3-5 个微步骤，然后粘贴到 RescueTime 的“目标”备注中。
3. **每周回顾 RescueTime 报告**：不要只看总时间，要关注“最高效时段”。ADHD 的注意力波动有模式，找到你的“超专注窗口”并安排重要任务。
4. **警惕过度依赖**：每两周尝试一天不使用 RescueTime，测试你的内在时间感知是否有所提升。如果完全失控，说明你需要加强脚手架而非拐杖。

RescueTime 不是万能药，但它提供了一个低成本的外部时间感知层。对于 ADHD 大脑和 LLM 来说，外部执行功能层不是修复缺陷，而是为生成核心套上 harness——让高产的想法找到可靠的输出路径。关键在于，你要把它当作脚手架，而非拐杖。

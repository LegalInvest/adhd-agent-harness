# 如果把 ADHD 大脑当成生成核心，Goblin Tools 能不能当它的外部执行功能层？

> 这是「生成核心」系列的一次严格版图测试。命题很流行:ADHD 大脑=高创造力的生成核心,缺的是外部执行功能层,所以「装一个工具」就补上了。上一篇 30 天记录给了 Goblin Tools 一个体面的定位(执行功能的除颤器),这一篇换一个更苛刻的问法:**「外部执行功能层」这个头衔,到底要求哪些功能?拿这张清单逐项对照,Goblin Tools 能打几个勾?**——这个问法的价值在于,它顺便给所有「XX 能不能当 ADHD 的外部执行功能」的讨论,立一把公用的尺子。

收敛:本文只回答——**执行功能层的功能清单是什么(从执行功能文献+agent harness 架构双侧交叉推导)?Goblin Tools 逐项对照的结果?以及「单件不达标」的正确结论是什么?**

## 穿透:先造尺子,再量它

尺子从两侧交叉推导。执行功能文献侧(Barkley/Brown 等模型),ADHD 受损的核心功能:**任务启动、计划与组织(拆解)、工作记忆(保持目标在线)、时间管理、抑制与转换、自我监控、情绪调节下的坚持**。agent harness 架构侧,一个完整执行层的组件:**任务分解器、调度器(何时做什么)、状态保持(上下文/目标注入)、触发器(启动执行)、监控与纠偏、错误处理(失败后恢复)**。两张表交叉,得到「外部执行功能层」的六项达标清单:①拆解②调度③目标状态保持④启动触发⑤监控纠偏⑥失败恢复(中断后无羞耻重启)。

拿 Goblin Tools 逐项量:**①拆解——满分勾**(Magic ToDo 是同类最优的单点:粒度可调、零门槛、无套路);**②调度——零分**(拆完的步骤没有时间属性,不知道何时做);**③目标保持——零分**(它不记得你是谁、在干什么长程项目);**④启动触发——零分**(不发提醒,等你来找它);**⑤监控纠偏——零分**(你做没做、跑没跑偏,它无感);**⑥失败恢复——半分**(它不惩罚中断——没有 streak、没有红色逾期,这个「无羞耻」设计难得,但它也不主动帮你重启)。总分:六项拿一项半。**结论清楚:Goblin Tools 不是外部执行功能层,是执行功能层里「拆解」这一个插槽的优秀零件。**

但这个「不达标」恰恰引出比原命题更有用的三个推论。**推论一:执行功能层注定是拼装的,不是采购的。**执行功能是六种异质功能的集合,指望一个 App 全包,等于指望一个零件是一台机器——市面上号称「ADHD 全能助手」的产品,通常是六项各拿半分的平庸拼盘,不如六个插槽各配最优单件(拆解给 Goblin Tools,调度给日历,触发给闹钟,监控给上一篇的 ChatGPT 回路,恢复给「从今天开始」的原则)。**推论二:插槽之间的胶水才是真正的难点**——拆解的输出要人肉搬进日历,监控的结果要人肉回写……这些接缝处的摩擦,正是大多数人系统烂尾的地方;胶水目前只有两种:固定的流程仪式(拆完立刻排程,一步都不隔夜)或一个愿意当胶水的 AI 对话(把各件的输出汇总给它)。**推论三:评测任何 ADHD 工具,先问它占哪个插槽**——「好不好用」是第二个问题,「它自以为是零件还是机器」是第一个(自以为是机器的零件,会诱导你拆掉本来能用的其他零件)。

## 验证

清单本身可检验:拿你现有的工具组合逐项打分,六个插槽哪些有件、哪些空着——多数人会发现自己在「拆解」和「调度」上重复采购,而「启动触发」和「失败恢复」两个插槽空着(它们恰恰最少有产品做,因为最难做成留存故事)。可证伪:若某产品六项全达标且都好用,本文「注定拼装」的判断被推翻——欢迎打脸,请告诉我们它的名字。

## 决策

做什么:画你的六插槽清单,逐槽配件;Goblin Tools 钉死在拆解槽;为两个最常空置的槽(触发/恢复)配非产品方案(闹钟仪式/「从今天开始」原则);设计你的胶水流程(拆完即排程)。

不做什么:不要为「全能」溢价付费;不要因为一个零件优秀就把整层的期待压上去(那是上一篇蜜月期犯过的错)。

先做什么:今天给六插槽打分——十分钟,看清你的执行功能层哪里有件,哪里漏风。

## 边界

执行功能模型本身在文献中有多个版本且持续争论,本文取交集简化(GRADE 中);「执行功能层↔harness 执行层」为 B 级架构类比,六插槽清单是实践工具而非验证量表(本文未做正式 A/B/C 分级)。插槽配置因表型而异(时间盲重的人调度槽权重更高);工具层拼装不替代治疗层。

## 今天就能试的 3 件事

1. 画六插槽表,给现有工具逐槽归位打分——找出空槽。
2. 给最空的那个槽配一个非产品方案:触发槽=固定闹钟仪式,恢复槽=写一张「从今天开始」卡。
3. 立一条胶水规则:凡 Goblin Tools 拆出的步骤,五分钟内进日历——接缝不过夜。

本文服务于人生 Harness 金字塔的**执行层的架构图**:别再问「哪个 App 能当我的外部大脑」——正确的问题是「我的六个插槽各缺什么件」;前者的答案永远令人失望,后者的答案今天就能开工。

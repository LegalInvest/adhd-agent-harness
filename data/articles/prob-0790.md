# Perplexity 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Perplexity：它到底补上了哪一层执行功能？

## 问题：为什么你用 Perplexity 查了十遍“怎么开始写报告”，却还是没写？

如果你有 ADHD，大概率有过这样的体验：打开 Perplexity，输入一个模糊问题，得到一份结构清晰、引用完整的回答——然后关掉页面，什么也没做。第二天，同样的流程再来一遍。

这像极了 LLM 频繁调用一个功能强大的工具，却从未把输出整合进自己的执行流程。Perplexity 在 ADHD 用户手里，常常沦为“高级搜索框”，而不是“执行调度器”。问题出在哪？

## 同构：ADHD 大脑与 LLM 共享同一个缺陷

ADHD 大脑常被描述为一个高产的生成核心——想法丰富、联想活跃，但缺少稳定可靠的调度层来筛选、组织、启动和坚持任务（来源：ADHD 的 AI 工具全景）。大语言模型同样具备强大的生成能力，却在长程上下文、目标保持与执行调度上存在天然缺陷。

最新的实证研究揭示了这种同构：用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中表现正常，但随序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突。这与 ADHD 执行功能缺陷的核心标志一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃（来源：Deficient Executive Control in Transformer Attention）。

两者的解法也同构：ADHD 需要外部脚手架来补偿执行功能，LLM 需要 function calling 把精确计算外包给工具。Perplexity 本质上是一个搜索工具，它应该被当作 function call 来用——输入问题，获取结构化信息，然后由调度层决定下一步。但很多 ADHD 用户把它当成了“终极答案机”，以为搜到了就等于做到了。

## 错用：Perplexity 被当作拐杖，而非脚手架

ADHD 个体常面临执行功能障碍，尤其是工作记忆不足和任务启动困难。Goblin Tools 的 Magic ToDo 能自动将任何任务分解为更小、更易管理的步骤，从而降低启动门槛（来源：Goblin Tools）。Lex 允许用户通过单一指令触发复杂、多步骤的任务序列，其底层架构类似于 agent scaffolding（来源：Lex）。这些工具的价值在于把“做什么”和“怎么做”外包给 AI，让用户只需执行第一步。

而 Perplexity 的设计——一次搜索、一份答案——恰恰缺少这种“执行链接”。它告诉你“写报告需要三步：收集数据、构建大纲、撰写内容”，但不会帮你把“收集数据”拆成“打开浏览器、搜索关键词、复制粘贴到文档”。对于 ADHD 大脑，这个缺口足以让整个任务瘫痪。

历史人物苏轼的 harness 系统可以佐证这一点。他兴趣极广、说话不过脑子、对无聊行政零耐心，但通过“以豁达为核心，不管贬到哪里都好好生活，用美食、写作、交朋友转移注意力；每日读书写字，不因为挫折停止创作”来维持输出（来源：苏轼 harness）。这里的“每日读书写字”就是一个固定的外部调度——相当于 LLM 的定时 re-grounding 或日课。苏轼没有把“豁达”当作答案，而是把它编码成可重复的行为模式。

## 正用：把 Perplexity 当作 function call，而非最终输出

要让 Perplexity 真正帮到 ADHD 大脑，需要改变使用方式：

1. **问题即参数**：把你的模糊意图（“我想写一篇关于气候变化的文章”）作为输入参数，让 Perplexity 返回结构化大纲，然后手动或通过工具（如 Goblin Tools）将大纲拆成可执行步骤。
2. **输出即中间件**：不要停留于阅读答案。把 Perplexity 的输出粘贴到任务管理器（如 Saner.AI）中，利用其知识回忆和本地记忆功能，自动提取待办事项并关联到日历（来源：Saner.AI）。
3. **重复即自动化**：对于反复出现的查询（如“如何开始写周报”），建立模板或工作流。Lex 的“单一指令触发多步骤”特性可以在这里发挥作用——把 Perplexity 的查询结果作为子步骤嵌入更大的任务序列（来源：Lex）。

对于工程师，这就像把 Perplexity 包装成一个 function：`perplexity_search(query)`，返回结构化 JSON，然后由 agent 调度器决定是否调用、如何处理结果。不要把它当成 chat 界面的终点。

## 局限与争议

必须承认，上述方案并非万能。首先，AI 工具对任务启动的长期效果缺乏验证——短期有效，但长期依赖可能导致能力退化（来源：矛盾与存疑）。其次，部分 ADHD 用户反映学习使用 AI 工具本身会增加认知负荷，净效果因人而异（来源：矛盾与存疑）。最后，超聚焦是否应被引导而非打断仍无定论——有人需要柔性引导，有人需要强力打断（来源：矛盾与存疑）。

Perplexity 之于 ADHD，就像 function calling 之于 LLM——它补上了信息获取层，但调度和执行仍需外部脚手架。不要把它当拐杖，要把它当工具链的一环。

## 今天就能试的行动

1. **下次用 Perplexity 查“如何做 X”后，立即把答案中的第一步写进待办清单**，并设置 5 分钟计时器执行它。
2. **把 Perplexity 的搜索功能绑定到你的任务管理器**（如通过 Saner.AI 的通用收件箱），让搜索结果自动生成待办事项。
3. **针对每周重复的任务（如“写周报”），创建一个 Perplexity 模板 prompt**，固定输出格式，然后直接复制到 Lex 或 Goblin Tools 中分解执行。
4. **反思：你是在用 Perplexity 搜索答案，还是在用它逃避启动？** 如果是后者，关掉它，先做第一步。

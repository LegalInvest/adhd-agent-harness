# Perplexity 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Perplexity：它到底补上了哪一层执行功能？

你刚打开 Perplexity 想查“ADHD 时间管理技巧”，结果半小时后你还在翻“AI 工具如何改变教育”的页面——因为你被一个链接带跑，又因为“这个相关”而点开另一个。这不是你的错，这是你的大脑和工具之间缺乏一层调度。

## 问题：ADHD 与 LLM 共享的“执行功能缺口”

ADHD 大脑的核心困境是：高产但缺执行调度层。你有一个强大的生成核心——能构思、联想、创造——但工作记忆脆弱、任务启动困难、时间感知模糊（来源：ADHD 的 AI 工具全景）。同样，LLM 也是一个强大的生成核心，但它的上下文控制弱，容易跑题、遗忘指令、在无关信息上“超聚焦”。

这种同构不是比喻，而是结构性的。ADHD 的工作记忆容量保留但控制弱，LLM 同样有强记忆容量但弱上下文控制（来源：同构视角）。ADHD 的任务启动困难类似 LLM 的“冷启动”问题（来源：同构视角）。ADHD 的时间盲对应 LLM 的时序推理弱点（来源：同构视角）。

两边缺的是同一类东西：一个能稳定执行精确、重复、状态维护任务的调度层。

## 答案：Perplexity 是 ADHD 的 function calling

对 LLM/agent 工程师来说，function calling 是让 LLM 把精确计算（如数学、数据库查询）外包给专用工具，自己只负责生成意图。对 ADHD 大脑来说，Perplexity 这样的工具应该扮演同样的角色：把“精确搜索、信息验证、状态维护”外包出去，让大脑只负责提问和联想。

但现实是，很多人用错了。他们把 Perplexity 当作一个更快的搜索引擎，结果陷入“搜索→点击→新搜索”的循环——这正是 ADHD 的典型陷阱。正确的用法是：把 Perplexity 当作一个**状态保持的助手**，你只给出一个核心问题，它负责保持上下文、过滤噪音、返回一个整合的答案，而不是一堆链接。

这与 Goblin Tools 的任务分解、Lex 的单指令多步骤执行、Saner.AI 的知识回忆是同一逻辑：外部工具承担了执行功能中的“保持状态”和“按步骤执行”部分（来源：Goblin Tools、Lex、Saner.AI）。

## 证据：工具如何补上执行功能层

### ADHD 侧证据
- **Goblin Tools**：用户报告称其“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools）。它通过任务分解降低启动门槛，直接针对任务启动困难。
- **Lex**：通过单一指令触发多步骤任务，减少决策疲劳（来源：Lex）。
- **Saner.AI**：通过知识回忆减少搜索循环和标签切换，补偿工作记忆不足（来源：Saner.AI）。

这些工具的共同点是：它们不是让 ADHD 大脑“变好”，而是提供了一个外部执行层，让大脑的生成核心可以专注于它擅长的事。

### LLM/agent 侧证据
- **Function calling**：让 LLM 调用计算器、数据库等工具，避免自身在精确计算上犯错。这与 ADHD 大脑使用 Perplexity 来避免信息验证时的注意力漂移完全同构。
- **上下文工程**：主动设计 LLM 的上下文，避免上下文膨胀（来源：上下文工程）。ADHD 的“环境带偏”同样需要主动设计“当下注意什么”。

两边的解法都是：**把状态维护外包，让生成核心自由**。

## 争议：脚手架 vs 拐杖的边界

这里有一个必须诚实的局限：AI 工具作为外部执行功能层的有效性证据主要来自用户报告，缺乏大规模对照实验（来源：矛盾与存疑）。更关键的是，过度依赖可能削弱内在能力（来源：矛盾与存疑）。

例如，Otter.ai 减轻笔记负担，但过度依赖可能削弱主动记笔记的能力（来源：同构视角）。同样，如果 ADHD 用户每次遇到模糊问题就打开 Perplexity，而不训练自己的信息筛选能力，工具就从脚手架变成了拐杖。

区别在于：脚手架帮助你完成当前任务并逐步内化技能，拐杖则让你永远依赖。对 ADHD 来说，这意味着使用 Perplexity 时要有意识地问自己：“我是在学习如何提问和筛选，还是仅仅在逃避思考？”

## 今天就能试的行动

1. **用 Perplexity 做“单问题深度对话”**：每次只问一个问题，强制自己不点开任何链接，只读 AI 整合的答案。如果觉得不够，就追问，而不是新开搜索。
2. **模仿 Lex 的单指令模式**：把一天中最复杂的一个任务，用 Goblin Tools 或手动分解成 3-5 步，然后只执行第一步。完成后才看下一步。
3. **设置“状态保持”提醒**：用 Saner.AI 或类似工具记录你正在做的核心任务，每 20 分钟检查一次是否偏离。
4. **反思工具使用**：每周一次，问自己：“这个工具是帮我完成了本来做不到的事，还是让我回避了本来该做的练习？”

Perplexity 不是 ADHD 的解药，function calling 也不是 LLM 的万能补丁。它们都是同一种思路的体现：承认生成核心的局限，用外部工具补上执行调度层。但工具的价值取决于使用者的自觉——你是在搭建脚手架，还是拄着拐杖？

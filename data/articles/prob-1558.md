# Claude 之于 ADHD，就像 会褪去的脚手架 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

## 问题：怕依赖，又怕崩

你是否有过这样的体验：打开 Claude，让它帮你分解任务、写邮件、甚至安抚情绪，那一刻感觉终于有人替你撑住了；可关掉对话窗口，心里立刻涌上一阵不安——我是不是太依赖它了？万一哪天它没了，我是不是会彻底崩掉？

这种「怕过度依赖工具、又怕没有支撑就崩」的焦虑，并不只是 ADHD 人群的专利。在 LLM agent 工程领域，工程师们也在面对同样的拷问：harness（脚手架）到底应该设计成什么？是永久嵌入系统的拐杖，还是终将褪去的临时支撑？两群人，同一个困境。

## 同构：ADHD 大脑就是一台高采样温度的 LLM

要理解这个困境，必须先看清一个结构上的真相：**ADHD 大脑与 LLM 在架构上高度同构**——两者都拥有强大的生成核心，却配有一个不可靠的调度层。

对于 ADHD 大脑，这个「生成核心」是智力、创造力、直觉；而调度层则是执行功能——工作记忆、任务启动、时间感知。当调度层失灵，再强的核心也会输出波动。这种波动在 ADHD 个体身上表现为显著的日内表现差异，注意力、执行功能在不同时间点差异很大（来源：采样温度与表现波动）。这就像一台采样温度（temperature）被调高的 LLM：输出丰富、有创意，但随机、不可控。

对于 LLM 本身，情况完全对称。LLM 的输出天然具有随机性，采样温度控制着输出的多样性：温度越高，输出越随机、越「有创意」；温度越低，输出越确定、越可重复。这种非确定性在生产环境中是可靠性的敌人，因为「小错误会累积，非确定性（采样、工具故障）使可重复性复杂化」（来源：AI Agent Systems: Architectures, Applications, and Evaluation）。因此，工程实践强调「确定性状态转换」，并采用「计划-执行」分离等模式来强制单向、确定性的控制流（来源：Planner-Executor Agentic Framework）。

**这就是同构的核心：ADHD 大脑与 LLM 共享同一脆弱性——强大的生成核心与不可靠的调度层之间的不匹配。** 情绪调节正是调度层的职责——当工作记忆过载或时间感知失灵时，情绪容易失控（来源：AI 与 ADHD 的情绪调节）。

## 工具作为外部调度层：脚手架还是拐杖？

既然调度层不可靠，自然需要外部补偿。在 ADHD 一侧，AI 工具通过提供「外部执行功能层」来补偿缺陷。例如，Goblin Tools 的 Magic ToDo 功能能自动将任何任务分解为更小、更易管理的步骤（来源：Goblin Tools），从而降低启动门槛。Saner.AI 通过增强知识回忆，帮助用户快速找回信息，减少认知负荷（来源：Saner.AI）。Inflow 则利用 AI 算法个性化训练任务，动态调整难度以优化认知负荷，侧重于神经可塑性（来源：Inflow）。

在 LLM/harness 一侧，LLM agent 通过「harness」调用工具、管理文件、处理长对话。工具设计被视为「agent UX」，强调命名、模式、错误处理等接口质量（来源：工具使用与认知卸载）。现代 agent 系统通过「组合多个模型、检索器和工具」来达成 SOTA 结果，而非依赖单一模型调用（来源：工具使用与认知卸载）。

**但关键问题在于：这些外部支撑是临时脚手架，还是永久拐杖？**

脚手架的定义是：它会在任务完成后被撤除，且撤除后结构本身能独立站立。而拐杖是：一旦撤除，使用者就会倒下。对于 ADHD 大脑，理想的 AI 工具应像脚手架——逐步提升内在执行功能，最终可以撤除。但现状是，多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。同样，在 agent 工程中，如果 harness 被设计为永久依赖，一旦 harness 失效，agent 便无法运行。

## 我的判断：Claude 是极佳的情绪调节脚手架，但需警惕「拐杖化」使用

基于以上分析，我提出一个鲜明的观点：**Claude 之于 ADHD，就像设计良好的 agent harness 之于 LLM——它是最佳的情绪调节脚手架，但大多数人用错了方向，把它当成了永久拐杖。**

证据何在？从情绪调节的角度看，AI 工具通过上下文工程和无状态与外部记忆补偿工作记忆缺陷，相当于为大脑加装了一个「情绪调节 harness」（来源：AI 与 ADHD 的情绪调节）。具体来说，工作记忆缺陷导致情绪信息难以维持，AI 工具通过提醒、笔记和进度追踪充当外部记忆载体；任务启动困难常引发拖延和自责情绪，AI 通过分解步骤、设置微目标降低启动门槛；时间盲导致时间压力感失调，AI 计时器和可视化进度条可提供实时反馈（来源：AI 与 ADHD 的情绪调节）。Claude 的对话式交互，恰好能同时胜任这些角色——它既是数字工作记忆，又是任务启动助推器，还能通过「身体在场效应」的虚拟化缓解孤独感，间接稳定情绪（来源：AI 与 ADHD 的情绪调节）。

但这里存在一个巨大的矛盾：**过度依赖风险**。拐杖与脚手架的概念指出，AI 工具可能削弱内在情绪调节能力，如何设计「可撤除的脚手架」尚无定论（来源：矛盾与存疑）。更关键的是，多巴胺干预的有效性存在争议——基于多巴胺的 AI 干预（如奖励系统）可能强化依赖而非内在动机（来源：矛盾与存疑）。这意味着，如果你把 Claude 当作情绪调节的永久替代品，而不是训练自己情绪调节能力的工具，你可能会越来越依赖它，最终失去内在的调节能力。

## 局限与诚实

我必须承认，本文的核心观点——ADHD 大脑与 LLM 的同构——目前证据主要来自概念类比和工具案例，缺乏大规模实证（来源：矛盾与存疑）。此外，现有 AI 工具（如 Inflow）多基于通用认知模型，对情绪调节的个体差异（如共病焦虑）覆盖有限（来源：AI 与 ADHD 的情绪调节）。长期效果证据缺乏，多数研究为短期观察，AI 对情绪调节的长期神经可塑性影响未明（来源：AI 与 ADHD 的情绪调节）。所以，我的判断是基于现有证据的合理推断，而非定论。

## 今天就能试的行动

1. **用 Claude 做「任务分解训练」而非「任务替代」**：每次让 Claude 分解任务后，自己尝试不看 AI 输出，独立写出步骤，再对比差异。这相当于 agent 工程中的「计划-执行」分离——先自己计划，再让 AI 验证。
2. **设定「撤除日」**：每周选一天，完全不用 AI 工具（包括 Claude）处理任务，只用自己的大脑。这能检验你的内在执行功能是否在成长，也符合脚手架可撤除的理念。
3. **用 Claude 做情绪日志，但只做记录，不做分析**：让 Claude 帮你记录情绪触发点和时间，但分析部分自己完成。这利用了 AI 的外部记忆补偿，但保留了内在调节能力的训练。

记住：真正的脚手架，是让你最终能独立站立的支撑。Claude 是极好的脚手架，但别让它变成你的永久拐杖。

# Habitica 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Habitica：它到底补上了哪一层执行功能？

## 问题：为什么 Habitica 对某些 ADHD 用户有用，对另一些却成了压力源？

你设定了一个任务：“写周报”。Habitica 把它变成了一个怪物，你攻击它，获得经验值和金币。但三天后，你开始忽略它——不是因为怪物太强，而是因为你发现自己在“为了打怪而打怪”，周报本身反而被遗忘。

这不是 Habitica 的错，而是它被用错了地方。Habitica 本质上是一个 **外部执行调度层**——它把“写周报”这个模糊意图，通过游戏化机制（HP、金币、任务列表）转化为可执行的步骤。这恰好对应了 LLM/agent 工程中的 **function calling**：LLM 不擅长精确计算（比如算 123×456），但它可以调用一个计算器工具，把结果拿回来。ADHD 大脑同样不擅长精确、重复、状态维护类任务（比如按时吃药、整理邮件），但它可以调用一个外部系统——Habitica——来代为执行调度。

问题在于：**Habitica 只补上了“执行调度层”，它没有补上“动机层”**。对于 ADHD 用户，动机本身就是核心缺陷之一（多巴胺不足导致动力缺乏），如果 Habitica 被当作“只要我用它，我就会自律”的拐杖，那么当游戏化新鲜感消退，它就会失效。这与 LLM 工程中的教训一致：**function calling 不能替代 LLM 的推理能力，它只是把精确计算外包出去**。如果 LLM 依赖工具来弥补所有缺陷（比如连“2+2”都调用计算器），它就会退化。

## 同构：ADHD 大脑与 LLM 共享同一个“高产生成核心 + 缺失调度层”架构

ADHD 大脑常被描述为一个高产的生成核心——想法丰富、联想活跃，但缺少稳定、可靠的调度层来筛选、组织、启动和坚持任务（来源：ADHD 的 AI 工具全景）。LLM 同样具备强大的生成能力，却在长程上下文、目标保持与执行调度上存在天然缺陷。两者都需要外部脚手架来约束生成能力。

这种同构在工具层面有明确对应：

- **任务拆解与启动**：Goblin Tools 的 Magic ToDo 将复杂任务分解为小步骤，降低启动门槛（来源：Goblin Tools）。LLM 工程中，agent harness 通过“结构化工作流和规划”将复杂目标分解为子任务（来源：幻觉与验证循环）。
- **外部记忆与知识管理**：Saner.AI 通过本地记忆和快速检索减少搜索循环（来源：Saner.AI）。LLM 工程中，记忆系统（如知识图谱）补偿上下文窗口限制（来源：采样温度与表现波动）。
- **对话式执行**：Lex 允许单一指令触发多步骤序列，类似 agent scaffolding（来源：Lex）。LLM 工程中，agent 通过 function calling 执行多步骤任务（来源：同构视角）。
- **验证循环**：ADHD 个体需要外部验证来校正冲动判断（如情绪调节中的“接地”），LLM 需要验证步骤检测幻觉（来源：幻觉与验证循环）。

## 人物案例：孔子的“礼”与胡林翼的“日课”——古代 ADHD 的 harness 系统

孔子被描述为精力旺盛、坐不住、冲动、对枯燥任务零耐心——典型的 ADHD 特质。他的 harness 是“克己复礼”：用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”）。这相当于给高产生成核心（他的思想）套上一个 **外部调度层**——“礼”规定了什么时候该做什么、怎么做。70 岁才做到“从心所欲不逾矩”，说明他一生都在与冲动斗争。

胡林翼年轻时是花花公子，幡然醒悟后，用“每日写日记反省”和“以做百姓之心做官”来约束自己。他的 harness 是 **日课 + 外部监督**（与曾国藩互相砥砺）。这对应 LLM 工程中的 **定时 re-grounding**：定期将内部状态与外部标准核对，防止偏离目标。

这些古代 harness 与现代 AI 工具的同构在于：**它们都不是替代内在能力，而是提供外部结构**。孔子的“礼”不是让他失去冲动，而是让冲动在轨道上释放；胡林翼的日记不是让他变得完美，而是让他每天校准方向。同样，Habitica 不是让 ADHD 用户变得自律，而是让“执行”这件事从大脑里搬出来，放到一个可视化的游戏系统中。

## 核心判断：Habitica 是脚手架，不是拐杖——区别在于你是否保留了自我监控

脚手架 vs 拐杖的边界在于：**脚手架最终可以拆除，拐杖则意味着永久替代**。对于 ADHD 用户，Habitica 应该被用作训练执行功能的临时辅助，而不是永久依赖。例如，你可以用 Habitica 来建立“每天写 500 字”的习惯，但三个月后，你应该能够不依赖它而自动执行。如果三年后你还在依赖它，那它就是拐杖。

LLM 工程中同理：function calling 是脚手架，不是拐杖。好的 agent harness 会逐步减少对工具的依赖（比如先调用计算器，然后让模型学会心算简单运算）。但当前工程实践更倾向于“永远调用工具”，因为这更可靠——这就像给 ADHD 用户终身配一个秘书，而不是教他们自己管理时间。

## 矛盾与局限

- **超聚焦 vs 打断**：Habitica 的提醒可能打断 ADHD 用户的超聚焦状态，而超聚焦本身是高效产出期。目前缺乏对比研究（来源：矛盾与存疑）。
- **长期依赖风险**：过度依赖 AI 工具可能导致执行功能退化，但缺乏纵向研究证实（来源：矛盾与存疑）。
- **个体差异**：部分用户反馈游戏化有效，另一部分则感到压力。Habitica 可能只适用于特定 ADHD 亚型。

## 今天就能试的行动

1. **把 Habitica 当“训练轮”**：设置 3 个最让你头疼的例行任务（如“吃药”“整理桌面”），用 Habitica 追踪 2 周，然后尝试脱离它 1 周，看看是否形成了习惯。
2. **用 Goblin Tools 分解一个“模糊任务”**：比如“准备晚餐”，让它拆成步骤，然后手动执行——体验“外部调度层”如何降低启动阻力。
3. **给 LLM 设置一个验证循环**：如果你用 ChatGPT 写代码，要求它先写测试用例再写代码——这对应 ADHD 的“先核对再行动”策略。
4. **写一周“胡林翼式日记”**：每天花 5 分钟记录“今天哪些事靠外部结构完成，哪些靠内在动力”——识别你的脚手架是否有效。

# Habitica 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Habitica：它到底补上了哪一层执行功能？

你试过 Habitica 吗？那个把生活变成 RPG 的 App——完成任务得金币、打 Boss、升级装备。很多 ADHD 朋友一开始觉得它“有用”，但几周后要么忘记打开，要么对重复的日常任务感到厌倦。问题出在哪？

要回答这个问题，得先看一个更深层的同构：ADHD 大脑和 LLM（大语言模型）其实是同一类“高产但缺执行调度层的生成核心”。ADHD 大脑能迸发无数创意，但执行精确、重复、状态维护类任务时频频卡壳；LLM 能生成流畅文本，但一涉及数学计算、多步验证或持久记忆就露馅。两者的解法也同构：ADHD 需要外部执行功能层（脚手架），LLM 需要 agent harness（通过 function calling 把精确计算外包给工具）。Habitica 本质上就是这样一个 harness——但很多人用错了，把它当成了拐杖。

## 问题：为什么 Habitica 对 ADHD 常常失效？

Habitica 的设计初衷很好：用游戏化（金币、等级、Boss）来激励任务完成。但 ADHD 的核心问题不是缺乏动机，而是执行功能缺陷——工作记忆弱、任务启动难、时间感知差（来源：ADHD 的 AI 工具全景）。游戏化奖励能短暂提升多巴胺，但无法解决“我该先做什么”“这件事要花多久”“我做到哪一步了”这些基本问题。更糟的是，Habitica 要求用户自己定义任务、设置重复周期、手动勾选完成——这恰恰是 ADHD 最不擅长的“精确、重复、状态维护”类操作。

一个 ADHD 用户可能会花一小时设置 Habitica 的每日任务列表，然后第二天就忘了打开 App。这不是意志力问题，而是工具本身没有补上执行调度层——它只是把“忘记做任务”变成了“忘记打开 App 做任务”。

## 同构视角：Habitica 之于 ADHD = function calling 之于 LLM

从同构视角看，Habitica 试图扮演的角色，正是 agent harness 中 function calling 的角色。LLM 本身不擅长精确计算或持久记忆，所以 agent 通过 function calling 把“计算 25*17”外包给计算器工具，把“记住用户偏好”外包给数据库。类似地，ADHD 大脑不擅长精确、重复、状态维护类任务，所以需要外部工具来外包这些功能。

但关键在于：function calling 是 harness 的一部分，不是全部。一个成熟的 agent harness 还包括上下文传递、验证循环、记忆系统和沙盒（来源：幻觉与验证循环）。同样，一个有效的 ADHD 脚手架应该提供任务分解（降低启动门槛）、外部记忆（减少工作负载）、实时反馈（校正时间盲）和验证循环（避免冲动错误）。

Habitica 只提供了“反馈”（游戏化奖励），却缺失了其他三层。对比一下真正有效的 ADHD 工具：Goblin Tools 的 Magic ToDo 能把“整理房间”自动分解成“捡起地板上的衣服”“擦桌子”等小步骤（来源：Goblin Tools），直接降低启动门槛；Saner.AI 提供本地记忆存储，减少标签切换（来源：Saner.AI）；Lex 允许单指令触发多步骤序列，减少决策疲劳（来源：Lex）。这些工具各自补上了 harness 的一部分，而 Habitica 试图用游戏化覆盖所有缺陷，结果变成了一个“需要执行功能才能使用的执行功能工具”。

## 证据：ADHD 侧与 LLM 侧的对应

**ADHD 侧：** 用户报告显示，Goblin Tools 的“魔法开关”能降低启动焦虑，将“压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools）。Otter.ai 的实时转录提供即时外部反馈，类似身体在场效应的虚拟版本（来源：ADHD 的 AI 工具全景）。这些工具都直接针对执行功能缺陷，而非仅仅提供奖励。

**LLM/harness 侧：** 工程实践表明，纯对话系统常因幻觉、缺乏接地和无法执行/验证行动而失败（来源：幻觉与验证循环）。复杂的 harness 不会盲目执行工具，而是实现验证步骤，例如检查输出格式或对模型编写的代码运行测试用例（来源：同上）。缺乏这些验证会导致“更多幻觉和重复工作”（来源：同上）。

**同构对应：** ADHD 的冲动行为可视为行为层面的“幻觉”——脱离环境线索做出不恰当反应（来源：幻觉与验证循环）。而验证循环（将内部状态与外部现实核对）正是校正这种偏差的关键。Habitica 缺少的就是这种验证循环：它不会在你冲动设定不切实际的任务时提醒你，也不会在你拖延时帮你分解步骤。

## 核心观点：脚手架 vs 拐杖的边界

我的判断是：Habitica 对 ADHD 有效的唯一场景，是它作为“脚手架”的一部分——即用户已经通过其他工具（如 Goblin Tools 或 Lex）分解了任务、设定了上下文，然后用 Habitica 的反馈机制来维持动力。如果单独使用，它大概率沦为“拐杖”：你依赖它的奖励来启动，但一旦奖励失效（比如错过一天），整个系统就崩溃。

这与 AI 工具的整体矛盾一致：过度依赖可能削弱内在能力（来源：矛盾与存疑）。例如，Otter.ai 减轻笔记负担，但过度依赖可能削弱主动记笔记的能力（来源：ADHD 的 AI 工具全景）。同样，Habitica 的游戏化可能削弱内在动机——当金币不再吸引你，你连刷牙都懒得做。

## 局限与争议

必须诚实指出：目前针对 Habitica 的 ADHD 效果缺乏独立随机对照试验，证据主要来自用户报告（来源：矛盾与存疑）。个体差异很大：有人觉得游戏化很有效，有人觉得是噪音。此外，Habitica 的“精确、重复、状态维护”设定本身就与 ADHD 的弱点冲突，这可能是它失效的结构性原因。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个你一直拖延的任务**（比如“整理书桌”），看看分解后的步骤是否更容易启动。对比 Habitica 中你手动设定的“整理书桌”每日任务，感受区别。
2. **在 Habitica 中只保留一个“今日焦点”任务**，并用 Lex 或 ChatGPT 将其拆成子步骤写在备注里。不要设置超过 3 个每日任务——Harness 的第一原则是降低认知负载。
3. **尝试一次“无游戏化”的番茄钟**：用手机计时器专注 25 分钟，不打开任何奖励 App。记录你的感受——你可能会发现，没有外部奖励时，你反而更清楚自己为什么要做这件事。
4. **如果你是工程师**，思考你当前的 agent harness 是否缺少验证循环？检查你的 function calling 流程：是否在调用工具后验证了输出？是否在每一步都有上下文传递？这些结构同样适用于你的 ADHD 工具箱。

记住：脚手架是用来搭建你自身能力的，拐杖是用来替代的。Habitica 可以是脚手架，但前提是你先补上任务分解、外部记忆和验证循环这三层。否则，它只是另一个让你感到挫败的“应该做”清单。

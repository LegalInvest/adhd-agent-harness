# ADHD 研究生如何用「给生成核心套 harness」的思路应对长周期科研与论文里的执行功能（计划/组织/启动）先天偏弱？

> 把 LLM 工程里的「agent 的编排/记忆/工具层就是「外部执行功能」」迁移到ADHD 研究生的长周期科研与论文场景，双域证据作答。

## 问题：为什么科研和论文对 ADHD 大脑特别难？

对 ADHD 研究生来说，长周期科研最大的敌人往往不是智力，而是执行功能：把一个大课题拆成可执行步骤、在数月乃至数年里持续组织材料、在每天早晨启动那个最该做的任务。wiki 资料指出，ADHD 大脑与 LLM 在结构上可被视为同一类系统——强大的**生成核心**，却缺乏可靠的**执行调度层**（来源：ADHD × AI 的科学与研究前沿）。前额叶执行网络激活不足，让 ADHD 人群在工作记忆、任务启动和时间感知上先天偏弱；而 LLM 则是无状态、无长期记忆、没有真正调度器，只靠一次性的上下文推理。

这是一个「同构」的工程问题：一边是大脑里的生成核心，一边是模型里的生成核心；两边都需要同样的补位——**harness（外部执行功能层）**。

## 第一层 harness：把巨型任务拆成微任务

论文的启动瘫痪通常不是因为任务太难，而是因为任务在脑中太大。ADHD 群体多巴胺系统失衡，导致行动动机难以被激发；同时工作记忆有限，无法把「写完第三章」在脑中展开为清晰步骤（来源：任务启动；来源：多巴胺）。AI 工具的经典应对就是**微任务分解（micro-chunking）**：Goblin Tools 的 Magic ToDo 可以把模糊输入（如「整理文献」）自动拆成可执行子任务，并调节粒度（来源：Goblin Tools）。

这与 LLM 工程里的 **Chain-of-Thought / ReAct** 是同一件事：模型直接回答复杂问题容易出错，但加上「一步一步想」或「推理-行动」的脚手架，任务完成率就会提升（来源：ADHD × AI 的科学与研究前沿）。对科研而言，具体做法是把「写论文」拆成「找 3 篇关键文献→用一句话概括每篇方法→画一张图说明变量关系」，每个子任务的认知入口足够小，多巴胺的「启动阈值」才能被越过。

## 第二层 harness：外部记忆，避免大脑当缓存

ADHD 工作记忆不稳定，导致反复切换标签页找信息，认知负荷被无谓消耗。Saner.AI 的设计逻辑正是「本地记忆 + 知识回忆」，把邮件、文档、日历里的待办事项统一收入，并通过内部助手自动检查截止日期、把大项目拆成里程碑（来源：Saner.AI）。这相当于给大脑外挂一个向量数据库 + 检索器。

LLM 也需要同样的机制：没有跨会话状态，它必须依赖外部记忆系统（提示词、RAG、长期记忆库）来维持连续性（来源：ADHD × AI 的科学与研究前沿）。科研中的对应物，是一个**单一可信的知识收件箱**——无论是文献笔记、实验记录还是论文片段，都按统一入口写入，并设定固定时间 review。大脑不应当是缓存，而应当是生成器。

## 第三层 harness：用调度器打败时间盲

ADHD 的「时间盲」会扭曲对任务所需时间的感知，让人低估紧迫性。Motion 的核心价值是自动排程：它根据任务优先级、截止日期、依赖关系和可用时间，动态重建日程，并在任务可能延期前几周发出警告（来源：Motion）。对研究生来说，这意味着把「什么时候做什么」从大脑里卸载给一个外部调度器，只留下「现在执行」这一件事。

在 agent 工程里，这对应的是**编排层（orchestration）**：一个上层调度器决定调用哪个工具、哪个子 agent、何时重试、何时回退。没有编排层的 LLM 只是聊天机器人；有了编排层，才变成能完成长周期任务的 agent。研究生也需要为自己的科研项目搭这样一个编排层，而不是靠每天早上重新决策。

## 历史案例：曾国藩的「日课十二条」就是一套 human harness

晚清曾国藩的人生轨迹，几乎是一份 ADHD  harness 的古老样本。他年轻时极度浮躁，坐不住、爱串门、读书慢，「他人目下二三行，余或疾读不能终一行」，情绪不稳，甚至兵败后投水自杀。他的解法不是变聪明，而是给自己套上最笨最稳的脚手架：日课十二条（黎明即起、读书不二、谨言、写日记反省），以及「结硬寨、打呆仗」的缓慢推进策略。

他的日记是一辈子持续的 **re-grounding 系统**，相当于今天 AI 系统里的每日 system prompt 刷新；他的「结硬寨」则对应 LLM 工程里**低方差、可重复的确定性工作流**。曾国藩的 harness 与现代 agent 架构的映射是：日课 ↔ 定时自我重定向；日记 ↔ 外部记忆/日志；秘书/幕僚 ↔ 外部调度器。对 ADHD 研究生最具启示的是：他从不相信自己的即时状态，而是把环境、日程和流程设计成不可绕过的结构。

## 核心判断：harness 是脚手架，不是拐杖——但边界需要小心维护

我的判断是：ADHD 研究生要解决的并不是「如何变得更自律」，而是**如何为自身设计一个可迭代的外部执行功能层**。这同 LLM 从「裸模型」进化到「agent 系统」的路径一致：生成能力已经够用，真正决定成败的是编排、记忆与工具调用层。

但必须诚实地说，这个同构框架目前仍是**理论类比**，而非经过充分验证的神经科学事实；许多 AI 工具在 ADHD 人群中的独立临床证据也有限，依赖风险真实存在（来源：全局矛盾与存疑）。若把所有决策都外包给工具，执行功能可能进一步退化；工具过多本身也会增加认知负担。因此，harness 的关键是**人在回路（human-in-the-loop）**：AI 负责提醒、拆解、记忆和调度，而人类保留判断、目标设定与最终拍板。

## 今天就能试的四条行动

1. **选一个最大的科研任务，用 Goblin Tools 的 Magic ToDo 拆成 3 个 15 分钟内可启动的微步骤**，并把它写进明天的日程第一栏。
2. **建立单一知识收件箱**：用 Saner.AI 或自己最顺手的笔记工具，把文献、实验记录、导师反馈集中在一个入口，每周五只做一次 review，不再临时翻找。
3. **用 Motion 或类似自动排程工具，把论文相关的固定时间块（写作、阅读、导师会）固化**，让调度器决定「现在该做什么」，而不是每天早上重新决策。
4. **每天睡前 5 分钟写三句话**：今天完成了什么、明天唯一要启动的一件事、当前最大的风险。这是曾国藩式 re-grounding 的最低成本版本。

科研不是短跑，而是一场持续数年的长程任务。ADHD 大脑和 LLM 一样，最需要的不是被修理，而是被套上一个可维护的 harness。

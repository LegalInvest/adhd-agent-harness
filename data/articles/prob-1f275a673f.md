# 为什么用 Mem 治 ADHD 的学习半途而废，和给 agent 套 外部记忆/向量库 是一回事？

> Mem 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

你有没有这样的经历：一门课买了三个月，还停留在第一课；一个项目热血沸腾地启动，三天后连文件放哪儿都忘了。对 ADHD 学习者来说，这往往不是“懒”，而是大脑像一台没有持久化存储的高产引擎——想法喷涌，但状态一断电就丢。与此同时，正在做 Agentic Harness 的工程师也在骂同一个 bug：模型推理精彩，跨会话却失忆，目标越跑越偏，最后陷入循环。

答案出奇一致：**给这台生成核心外挂一个 Mem（外部记忆/向量库）harness**。本文的判断是：ADHD 大脑与 LLM/agent 在结构上是同类系统——高产但缺乏可靠的执行调度层，用外部记忆把它们重新锚定，是同一套同构解法。（来源：AI 与 ADHD 的学习方式）

## 同一类 bug：无状态核心 + 漂移的上下文

ADHD 一侧的崩溃机理很清晰：工作记忆不稳定，导致目标在执行中漂移；注意力被无关刺激捕获，或者反过来陷入细节超聚焦而忘记整体目标；对时间“盲”，于是任务永远“等会儿再开始”。（来源：重锚定与目标漂移）

LLM/agent 一侧几乎一一对应：模型本身无跨会话状态，上下文一长就退化，缺乏真正的调度层，于是输出可能偏离目标、陷入循环，甚至产生危险行为。（来源：人在回路与身体在场）wiki 把这一点概括为“无状态与外部记忆”：ADHD 和 LLM 都需要外部记忆系统（笔记、对话历史、向量库）来补偿状态缺失。（来源：无状态与外部记忆）

两边共同的解法是 **harness/脚手架**：不是换发动机，而是给发动机装上传动、刹车和方向盘。

## 从孔子、王阳明到 agent：同一套 harness 结构

真实历史里早有这套自我 harness 的原型。孔子的 ADHD 式特质非常明显：身高一米九、精力旺盛、周游列国十四年坐不住；对音乐能超专注到“三月不知肉味”，对种地等俗事却零耐心；思维跳跃，留下的《论语》全是场景化语录，没有系统著作。他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，再配合“吾日三省吾身”做每日重锚定，直到七十岁才“从心所欲不逾矩”。

这对应到 LLM agent harness 就是：**“礼” = 护栏与行为约束（token 预算、工具调用上限、防止循环）；“日省” = 定时 re-grounding，把状态从外部记忆中读回来做一致性检查。**（来源：人在回路与身体在场）

王阳明则是另一个版本：五岁不会说话，神经发育延迟；兴趣爆发式转移，从格物到兵法到道佛再到龙场悟道；冲动上书骂刘瑾，被廷杖四十。他的 harness 是“致良知”和“知行合一”，用内在的良知作为决策唯一标准，并且坚持“事上练”——在具体事务里不断校准。对应到 agent 系统：**“良知” = 价值对齐/constitution；“事上练” = human-in-the-loop 检查点与在线治理。**（来源：人物案例）

两个例子说明：无论对人还是对 agent，harness 都不是天然存在的，而是需要被工程化地设计、持续地维护。

## Mem 的两侧实现：ADHD 侧 vs. Agent 侧

对 ADHD 学习者，Mem 首先是一个**外部记忆工作流**：把目标、任务、上下文写下来，让大脑从“记住一切”中解放出来。Saner.AI 就是这一思路的典型产品——它强调本地记忆与知识回忆，通过 AI 笔记捕获、自动组织任务和通用收件箱，减少搜索循环和标签切换。（来源：Saner.AI）

当任务大到无法启动时，Goblin Tools 的 Magic ToDo 把模糊任务（比如“清理厨房”）拆成具体子任务，还能调节“辣度”控制粒度；Perplexity 则把学习目标拆成可管理的搜索步骤。（来源：Goblin Tools；Perplexity）它们本质上都在做同一件事：把上下文从脑子里搬出来，削成模型/大脑能处理的小块。

对 agent 工程师，Mem 就是**向量库 + RAG + 外部状态存储**。Agent 不需要把全部历史塞在上下文窗口里，而是把关键状态向量化，需要的时候检索回来。这同样是在解决“上下文膨胀”和“目标漂移”：把工作记忆外置，让生成核心只处理当前最相关的片段。为了更稳，harness 还会设置 token 上限、工具调用次数上限和人工检查点，防止 agent 在无人监督下跑飞。（来源：人在回路与身体在场）

两边的 Mem 都遵循同一个协议：**写状态、存外部、按需取、定时回检。**

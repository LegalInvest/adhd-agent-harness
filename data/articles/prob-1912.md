# ADHD 家长如何用「给生成核心套 harness」的思路应对陪伴 ADHD 孩子里的执行功能（计划/组织/启动）先天偏弱？

> 把 LLM 工程里的「agent 的编排/记忆/工具层就是「外部执行功能」」迁移到ADHD 家长的陪伴 ADHD 孩子场景，双域证据作答。

## 问题：当孩子的“生成核心”缺少调度层

你是一位 ADHD 孩子的家长。每天早晨，你看着孩子坐在书桌前，面前摊着作业本，却一动不动——不是不想做，而是大脑里那个“启动按钮”失灵了。你帮他拆解任务、列清单、设闹钟，但一转身，他又被窗外的鸟叫带走了注意力。你感到疲惫，甚至怀疑：为什么别人家的孩子能自己管理，而我的孩子需要我像一个“外部大脑”一样全程支撑？

与此同时，在另一个世界里，一群工程师正盯着 LLM agent 的输出日志发愁：模型明明有强大的生成能力，却会在多步任务中丢失上下文、忘记工具调用、被无关输入带偏。他们的解决方案是——给这个“高产但缺调度层的生成核心”套上一个 harness：编排层、记忆层、工具层，统称“外部执行功能层”。

这两个世界看似遥远，实则共享同一个核心问题：**如何为强大的生成核心补上一个可靠的外部调度层？** 本文将从“人群×同构”的视角，把 LLM 工程里的 agent harness 迁移到 ADHD 孩子的陪伴场景，给出一个基于证据的答案。

## 同构：ADHD 大脑与 LLM 是同一类系统

ADHD 大脑的生成核心（发散思维、联想能力）与 LLM 的文本生成能力高度相似——两者都高产、有创造力，但都缺乏可靠的内置执行调度层（来源：[[ADHD 大脑与 LLM 的同构]]）。ADHD 的执行功能缺陷（计划、组织、启动困难）与 LLM 的无状态、缺乏调度层问题在结构上同构（来源：[[外部执行功能层]]）。这意味着，为 LLM 搭建 agent scaffolding（将 LLM 置于与记忆、工具和决策逻辑的控制循环中，来源：[[工具使用与认知卸载]]）与为 ADHD 孩子搭建执行功能支架是同一类工程。

这种同构在具体表现上也有对应：ADHD 孩子的表现像“采样温度过高”的 LLM——注意力、执行功能在一天内剧烈波动，缺乏稳定的输出（来源：[[采样温度与表现波动]]）。而 LLM 的随机性需要通过确定性工作流、工具契约来约束；ADHD 孩子的波动则需要外部结构（如日程、环境设计）来稳定。

## 历史中的 harness：曾国藩的“日课十二条”

晚清名臣曾国藩，被公认为“半个圣人”，但他年轻时是一个典型的 ADHD 大脑：浮躁坐不住，天天串门喝酒看杀人，日记里天天骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”；一辈子严重银屑病，情绪不稳定，打败仗就跳水自杀。他的解法是：给自己套上一个“外部执行功能层”——日课十二条：黎明即起、读书不二、谨言、写日记反省……他用最笨最稳的方法抵消自己的冲动，用“结硬寨打呆仗”的策略管理军队。

这套 harness 与 LLM agent 的 scaffolding 高度同构：日课是定时 re-grounding（类似 agent 的周期性状态检查），写日记是外部记忆（类似向量数据库），谨言是输出约束（类似工具契约）。曾国藩用这套系统，从一个“注意力不集中”的浮躁青年，成长为平定太平天国、发起洋务运动的中兴名臣。他的故事证明：**外部执行功能层不是拐杖，而是可拆卸的脚手架**——一旦内化为习惯，孩子就能逐渐独立。

## 现代工具：Goblin Tools、Saner.AI、Motion

今天的 AI 工具正在将这种 harness 数字化。**Goblin Tools** 的 Magic ToDo 功能能自动将模糊任务（如“清理房间”）分解为具体子步骤，用户可调节“辣度”控制粒度（来源：[[Goblin Tools]]）。这相当于 agent 的“任务分解器”——把高层指令拆成可执行的原子操作。**Saner.AI** 专注于知识回忆和本地记忆，减少搜索循环和标签切换（来源：[[Saner.AI]]），对应 agent 的“记忆缓冲区”——存储状态，避免上下文丢失。**Motion** 则自动排程并动态调整计划，消除“下一步该做什么”的决策负担（来源：[[Motion]]），对应 agent 的“编排层”——确定控制流。

这些工具帮助 ADHD 孩子克服任务启动困难（来源：[[任务启动]]）、时间盲（来源：[[时间盲]]），并通过小步骤的即时反馈维持多巴胺驱动的动机（来源：[[多巴胺]]）。但家长需要注意：工具是脚手架，不是拐杖。过度依赖可能导致能力退化（来源：[[拐杖与脚手架]]），但目前缺乏长期研究验证这一风险。

## 争议与局限

尽管同构视角富有启发性，但必须诚实指出：当前证据主要是概念类比和初步用户反馈，缺乏大规模随机对照试验验证（来源：[[矛盾与存疑]]）。例如，ADHD 大脑与 LLM 同构的论点多为推理，缺乏神经科学或计算机科学的直接证据（来源：[[矛盾与存疑]]）。AI 工具对任务启动的长期效果未知，且学习使用工具本身可能增加认知负荷（来源：[[矛盾与存疑]]）。此外，个体差异巨大——有的孩子需要强力打断（如番茄钟），有的需要柔性引导（来源：[[超聚焦]]）。家长需要根据孩子的反应调整策略。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个任务**：让孩子说出一个“不想做”的任务（如“整理书包”），输入工具，调节“辣度”到合适粒度，然后让孩子逐一勾选完成。观察启动门槛是否降低。
2. **模仿曾国藩的“日课”建立晨间/晚间 routine**：和孩子一起列出 3-5 条简单规则（如“起床后先喝水”“写一句话日记”），贴在墙上。这相当于 agent 的“确定性工作流”——减少决策负担。
3. **用 Motion 或类似工具自动排程**：将孩子的作业、活动输入 Motion，让它自动生成每日计划。如果计划被打乱，观察工具如何动态调整——这能帮助孩子理解“计划可以灵活，但结构不可少”。
4. **反思“脚手架 vs 拐杖”边界**：每周和孩子讨论一次：哪些工具帮助了独立，哪些反而产生了依赖？逐步减少工具的使用频率，测试孩子的执行功能是否有所提升。

## 结论

ADHD 孩子的“生成核心”并不需要被修复，它本身就是一种天赋——发散思维、创造力、超聚焦能力。它需要的只是一个可靠的外部调度层，就像 LLM 需要 agent harness 一样。曾国藩用“日课十二条”做到了，今天的 AI 工具让这件事更容易。但记住：最好的 harness 是最终可以卸下的——它帮助孩子学会自己调度，而不是永远依赖外部。这，才是“外部执行功能层”的真正意义。

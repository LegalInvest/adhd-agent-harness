# ADHD 学生视角：为什么「治好 ADHD 的执行功能（计划/组织/启动）先天偏弱」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 的编排/记忆/工具层就是「外部执行功能」——同一套 harness 思路，两边都成立。

## 一、两个问题，同一种崩溃

想象一下这个早晨：一个ADHD学生盯着空白文档，脑子里有无数想法，却连「打开大纲」这一步都启动不了；而在隔壁工位，一个Agentic Harness工程师看着自己的LLM agent在工具调用里循环了十七次，把一次简单的查询走成了无穷递归。前者被自己的大脑背叛，后者被自己写的系统背叛。两种崩溃，背后是同一种架构缺陷：核心非常能生成，却极度缺乏执行调度层。

ADHD的神经科学研究早已指出，患者的前额叶执行功能网络激活不足，表现为计划、组织、任务启动、工作记忆和时间感知等执行功能缺陷（来源：ADHD × AI 的科学与研究前沿）。LLM/agent 这边则是另一个极端：它能在上下文里生成流畅的长篇推理，却本质无状态、缺乏可靠的内置调度层，容易在复杂任务中漂移、幻觉或循环（来源：ADHD × AI 的科学与研究前沿）。两者都是强大的生成核心，却共享一个工程问题——如何把想法/输出锚定在目标上。

## 二、同构脊柱：外部执行功能层

这一同构关系不是简单比喻，而是结构上的对应。ADHD 的痛点与 LLM 的弱点可以逐层映射：工作记忆不稳定 ↔ 无跨会话状态；任务启动困难 ↔ 第一步工具调用就卡住；注意力分散/时间盲 ↔ 上下文膨胀与推理退化；身体在场效应 ↔ 需要外部锚点维持专注（来源：ADHD × AI 的科学与研究前沿）。

换言之，ADHD 人与 LLM 都需要同一层基础设施：外部执行功能层（external executive function layer）。它不是让核心变聪明，而是在核心之外补上一套调度、记忆、工具调用与重锚定的机制。

晚清名臣曾国藩的案例，就是一套手工版的外部执行功能层。他的 ADHD 特质在史料中极为典型：三十岁前浮躁坐不住，天天串门、饮酒、看杀人，读书时「他人目下二三行，余或疾读不能终一行」，一辈子严重银屑病、情绪不稳，打败仗就跳水自杀。他的 harness 是「日课十二条」——黎明即起、读书不二、谨言、写日记反省，以及「结硬寨打呆仗」。这套系统对应的正是 LLM/agent 工程里的几个关键组件：每日反省日记 ↔ 定时 re-grounding 与 self-correction；固定日程 ↔ prompt template 与系统提示词；把大任务拆解为极笨的小步骤 ↔ Chain-of-Thought / task decomposition；聘请幕僚与秘书 ↔ 外部调度器（scheduler）与 human-in-the-loop。曾国藩的成就不是因为他克服了冲动，而是因为他用一套外部系统把冲动锁进了 harness。

## 三、两边都在用同一套解法

当解法被拆成组件，同构变得更加清晰。

**任务启动 ↔ 第一步工具调用**。ADHD 患者因多巴胺不足、工作记忆容量有限和时间盲，常常卡在任务第一步（来源：任务启动）。LLM 在复杂任务里同样容易在第一步规划或工具选择上迷失。两边的解法都是 micro-tasking：Goblin Tools 的 Magic ToDo 把「清理厨房」拆成可执行的子步骤，通过即时反馈降低启动门槛（来源：Goblin Tools）；agent 工程里则用 ReAct 和 Chain-of-Thought 把推理拆成观察—思考—行动的小循环，防止模型在空转中跑飞（来源：ADHD × AI 的科学与研究前沿）。

**工作记忆 ↔ 外部记忆系统**。ADHD 患者频繁搜索、切换标签，Saner.AI 用本地记忆和通用收件箱把邮件、文档、日历中的待办自动提取并聚合，减少认知负荷（来源：Saner.AI）。这正是 agent 系统里 RAG 与记忆库（memory store）在做的事：把核心装不下的状态外部化，让每次调用都能基于完整上下文重新进场。

**时间盲 ↔ 时序调度器**。ADHD 患者对时间流逝感知失真，Motion 通过自动排程、动态调整和延期预警，消除「下一步该做什么」的决策负担（来源：Motion）。对应到 agent 工程，就是显式的时间/步骤调度器、超时机制与重试策略——防止 agent 在无限循环里消耗 token 与预算。

**身体在场 ↔ 外部监督信号**。AI 陪练、虚拟视频共作（Focusmate）等工具利用身体在场效应提升专注度（来源：ADHD × AI 的科学与研究前沿）。在 agent 系统里，这相当于 human-in-the-loop、审批节点与实时日志面板：一个外部观察者让系统保持对目标的觉察。

## 四、核心判断：harness 不是拐杖，而是架构

我的判断是：ADHD 与 LLM/agent 的干预策略，应当从「修复缺陷」转向「设计 harness」。ADHD 的核心不是不能思考，而是思考的执行层不可靠；LLM 的核心不是不能生成，而是生成的调度层不可靠。两边的最优解都不是无限打磨核心，而是在核心外加一层可迭代、可降级、可替换的执行脚手架。

这也是为什么曾国藩的日课与现代的 agent orchestration 共享同一套精神：承认核心的不稳定性，然后用工程化的外部结构兜底。

但这个框架有明确的边界与争议。首先，**拐杖与脚手架的界限**并不清晰：长期使用外部记忆与调度工具，是否会进一步弱化本已脆弱的内生执行功能？（来源：ADHD × AI 的科学与研究前沿）其次，**证据仍有限**：Motion 等工具缺乏独立临床验证，部分 ADHD 工具评测存在夸大（来源：全局矛盾与存疑）。再次，**同构性本身仍是理论假设**，而非经过验证的神经算法事实（来源：全局矛盾与存疑）。最后，**个体差异**很大：并非所有 ADHD 亚型都以执行功能缺陷为主，部分患者更需要情绪调节而非任务调度。

## 五、今天就能试的 4 个行动

1. **给自己/agent 建一个「再进场协议」**：每天固定时间写 3 行当前优先级（曾国藩的日记现代版），agent 则在每次调用前读取 system prompt 中的目标状态。核心会漂移，再进场机制是刚需。
2. **把任务拆到「无法拒绝启动」**：用 Goblin Tools 把「写论文」拆成「打开文档→写下标题→写第一句」；写 agent 时则用 ReAct/CoT 把工具调用拆成观察—思考—行动三步，避免单步暴雷。
3. **外置记忆，而不是依赖核心记住**：ADHD 人用 Saner.AI 或本地笔记统一收件箱；agent 工程师用显式 memory store / RAG，而不是让模型在上下文里硬记。
4. **设一个外部监督位**：ADHD 人可以用共作视频或定时 check-in；agent 系统则设置 human-in-the-loop 审批与超时熔断。核心不可靠时，外部观察者比自律更可靠。

## 结语

当 ADHD 学生与 agent 工程师隔着屏幕相望，他们面对的其实是同一个工程题：一个强大的生成核心，如何才能在真实世界里把事情做完、做对、不跑飞。答案不是责备核心，而是为它造一个 harness。曾国藩用毛笔和日记做到了，我们今天用 AI 工具和 agent 架构在做同样的事。这既是科学上的同构，也是工程上的共识：执行功能，可以外置；目标，可以锚定。

# 用 Goblin Tools 给 ADHD 补上重锚定与目标漂移，是脚手架还是拐杖？

> Goblin Tools 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

你打开 Goblin Tools 的「魔法待办」，输入「整理房间」，它立刻吐出 12 个小步骤：从「拿三个垃圾袋」到「把书放回书架」。你松了口气，终于知道从哪里下手了。但 40 分钟后，你发现自己正坐在书堆里刷手机——你又漂移了。

这不是你的错。ADHD 大脑与 LLM agent 共享同一个结构性困境：高产但缺乏可靠执行调度层（来源：AI 与 ADHD 的专注力·核心论点）。你的大脑像一台没有操作系统的超级计算机，随时可以超聚焦在错误的事情上六小时（来源：超聚焦·与 ADHD 的关系）。而 LLM agent 在长程任务中也会「目标漂移」——上下文膨胀后，它忘了最初要干什么。

Goblin Tools 正是在这里介入：它把模糊目标拆解成可执行的子任务，相当于为你的大脑写了一个临时的 system prompt。问题是：这个 prompt 是帮你重新锚定，还是让你彻底放弃方向盘？

## 同构：ADHD 大脑与 LLM agent 共享的「断裂」

最新研究用 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中模型表现正常，但随序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突。这与 ADHD 执行功能缺陷的核心标志一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃（来源：认知负荷·最新研究）。

换句话说，ADHD 大脑和 LLM agent 都在「认知负荷超载」时失去对目标的控制。ADHD 的「超聚焦」本质上是前额叶控制层级的断裂：高级上下文信息不再能支配思维和行动，中间层级活动与高级调节脱耦合（来源：超聚焦·最新理论）。这正是 agent 在长程任务中「忘记 system prompt」的神经版本。

## Goblin Tools 的锚定机制：任务分解与外部记忆

Goblin Tools 的核心价值在于它外化了「任务启动」和「步骤排序」这两个最耗认知负荷的环节。对于 ADHD 用户，启动困难常常不是因为不想做，而是因为「从哪里开始」这个决策本身就让工作记忆过载。工具通过自动决策移除认知负荷（来源：认知负荷·AI 如何介入），让你可以直接执行。

但这里藏着第一个陷阱：如果工具替你做了所有决策，你的元认知——即「我为什么在做这件事」的觉察——就会萎缩。就像 agent 如果完全依赖外部 prompt 而从不内部复盘，一旦 prompt 被遗忘，它就彻底失能。

## 证据与边界：哪些工具真正有效？

目前，Goblin Tools 的证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑·AI 工具作为外部执行功能层的有效性证据不足）。相比之下，Focusmate 基于身体在场效应，多项用户研究显示其能显著提升任务启动和维持（来源：AI 与 ADHD 的专注力·证据脉络）。RescueTime 通过客观时间记录提升时间觉察，有初步研究支持其改善时间盲（来源：同上）。而 Brain.fm 针对 ADHD 的效果缺乏独立临床研究支持（来源：Brain.fm·证据）。

这告诉我们：并非所有 AI 工具都同等有效。脚手架 vs 拐杖的边界在于——工具是否保留了你的主动参与。Focusmate 要求你实时汇报进度，RescueTime 让你回顾数据并自我调整，它们都在强迫你「保持在线」。而 Goblin Tools 如果只是输出步骤让你照做，你很容易在执行中漂移，因为步骤本身没有内置重锚定机制。

## 争议与局限：依赖风险与个体差异

过度依赖 AI 工具可能削弱内在时间感知能力和执行功能（来源：矛盾与存疑·过度依赖 AI 工具可能削弱内在能力）。ADHD 的异质性使得同一工具对不同亚型效果迥异（来源：矛盾与存疑·个体差异导致效果不一）。对于以「注意力分散」为主的 ADHD 用户，Goblin Tools 的任务分解可能足够；但对于「超聚焦跑偏」的用户，更需要的是像 Focusmate 那样的实时外部问责，而非静态的步骤列表。

此外，ADHD 大脑与 LLM 的同构是隐喻而非严格等价（来源：AI 与 ADHD 的专注力·仍存争议）。LLM 的「注意力机制」与人类注意力存在本质差异，我们不能期待 AI 工具完全复制人类教练的效果。

## 核心判断：Goblin Tools 是脚手架，前提是你自己握着锤子

我的观点是：Goblin Tools 作为重锚定工具，有效与否不取决于它分解得有多细，而取决于你是否在每个步骤前问自己「这步还服务于我的最终目标吗？」。如果你只是机械执行，它就是拐杖；如果你用它的输出作为「检查点」来主动校准，它就是脚手架。

对于 LLM agent 工程师，这同样成立：最鲁棒的 agent 不是那些 prompt 最长的，而是那些在每步执行后都做「目标一致性检查」的。你需要在 system prompt 中嵌入一个 meta-prompt：「每次行动前，回顾原始目标。」

## 今天就能试的行动

1. **Goblin Tools + 自我提问**：用 Goblin Tools 分解一个任务，但每完成三个步骤，暂停并问自己：「我现在做的事，还在通往最初目标的路上吗？」记录漂移次数。

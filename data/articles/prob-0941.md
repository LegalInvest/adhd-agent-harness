# 用 Goblin Tools 给 ADHD 补上重锚定与目标漂移，是脚手架还是拐杖？

> Goblin Tools 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

你有没有过这样的经历：打开电脑准备写一篇报告，三个小时后发现自己正对着一个完全无关的维基百科页面，或者已经连续六小时沉浸在一个次要任务中，而原计划的核心目标纹丝未动？对 ADHD 大脑而言，这种“分心走神”和“超聚焦跑偏”是同一枚硬币的两面——都是目标漂移。

如果你恰好也在做 Agentic Harness 工程，你大概对这一幕再熟悉不过：一个 LLM agent 被赋予长程任务，中途却因上下文溢出或错误反馈，开始执着于一个子问题，完全偏离了初始 system prompt。ADHD 大脑与 LLM 在结构上同构：两者都是高产的生成核心，但缺乏可靠的执行调度层（来源：AI 与 ADHD 的专注力）。因此，当 Goblin Tools 这类工具出现时，它被宣传为 ADHD 的“外部执行功能层”，能通过任务分解、重锚定来对抗目标漂移。但问题是：它究竟是帮助我们走向独立的脚手架，还是让我们永远依赖的拐杖？

## 重锚定：ADHD 与 LLM 的共同痛点

ADHD 的专注力问题核心在于执行功能缺陷，尤其是工作记忆、任务启动和时间盲（来源：AI 与 ADHD 的专注力）。超聚焦（hyperfocus）是 ADHD 的典型矛盾：个体可以完全沉浸于任务，却往往“在错误的事情上超聚焦六小时”（来源：超聚焦）。这种状态与时间盲密切相关——数小时在不知不觉中消失，而原本的目标早已被遗忘。

LLM agent 同样面临“上下文溢出”和“调度失败”：一个 agent 在长程任务中可能因为中间结果过于有趣或意外，开始“超聚焦”于某个子问题，而忘记初始目标。ADHD 大脑与 LLM 都需要一个外部 harness 来定期重锚定——即把注意力拉回原始目标。Goblin Tools 的“魔法器”（Magic Todoist）功能恰恰承担了这一角色：它将模糊的大目标分解为可执行的微步骤，并在每一步提醒用户“这是你原本想做的事”。

## Goblin Tools 的机制：脚手架而非拐杖？

Goblin Tools 的核心是降低认知负荷。它通过自动决策——替用户决定“下一步做什么、何时做、如何优先排序”——来移除决策带来的认知负担（来源：认知负荷）。对于 ADHD 个体，这种外化思维的方式“能显著减轻工作记忆上的认知负荷”（来源：认知负荷）。从工具机制看，Goblin Tools 是典型的脚手架：它提供临时支撑，帮助用户完成原本无法独立完成的任务。

但问题的关键在于：这种支撑是设计为可逐步撤除的吗？目前，多数 AI 工具（包括 Goblin Tools）都设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这引发了“脚手架 vs 拐杖”的争议：过度依赖 AI 是否可能阻碍内在执行功能的发展？（来源：矛盾与存疑）目前没有长期研究证明脚手架可以逐步撤除（来源：AI 与 ADHD 的专注力）。

## 边界在哪里：证据与局限

现有证据主要来自工具机制与 ADHD 理论的对齐，而非独立临床研究（来源：AI 与 ADHD 的专注力）。例如，Brain.fm 的神经锁相技术理论上可增强专注，但缺乏针对 ADHD 的随机对照试验（来源：Brain.fm）。Focusmate 基于身体在场效应的广泛验证，但效果归因于身体在场而非 AI 本身（来源：Focusmate）。RescueTime 通过自动记录提升时间盲觉察，但过度关注数据可能引发焦虑（来源：RescueTime）。

这些工具的共同局限是：它们补偿了执行功能，但并未促进内在能力的发展。Goblin Tools 的任务分解功能如果被用户被动接受（即用户从不思考为什么这样分解），它就变成了拐杖；如果用户主动学习分解策略并尝试内化，它才是脚手架。

## 同构视角下的启示

从 ADHD 大脑与 LLM 的同构命题出发（来源：AI 与 ADHD 的专注力），我们可以得到更深的洞察：LLM agent 的 harness 设计同样面临“脚手架 vs 拐杖”的边界问题。如果 agent 的 system prompt 过于死板，它可能无法适应动态环境；如果过于灵活，它可能漂移。理想的 harness 应该像 ADHD 的脚手架一样，提供足够的结构以防止漂移，同时保留学习和适应的空间。

对于 ADHD 用户，这意味着：使用 Goblin Tools 时，应有意识地观察它如何分解任务、如何重锚定目标，并尝试将这些策略内化为自己的思维习惯。例如，在完成一个任务后，回顾 Goblin Tools 为你生成的步骤，问自己：“如果下次没有这个工具，我能自己想到类似的分解吗？”

# 为什么用 Inflow 治 ADHD 的情绪失调，和给 agent 套 会褪去的脚手架 是一回事？

> Inflow 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

## 问题：为什么 ADHD 的情绪失调和 Agent 的“抽风”如此相似？

如果你是一个 ADHD 患者，你一定经历过这种时刻：明明想好好工作，但大脑突然被一阵无名怒火或焦虑淹没，然后整个下午报废。如果你是一个 Agent 工程师，你一定见过这种场景：你的 LLM agent 明明有足够的知识和推理能力，却突然在一个简单步骤上反复出错，或者陷入死循环。

这两种崩溃，本质上是一回事：**一个强大的生成核心，配了一个不可靠的调度层。**

ADHD 大脑的生成核心——智力、创造力、联想能力——通常不差，甚至很强。但执行功能（工作记忆、任务启动、时间感知）这个调度层常常失灵（来源：执行功能障碍）。当调度层过载，情绪就失控了。LLM/Agent 也是同理：模型本身是强大的生成核心，但缺乏稳定的编排层来管理上下文、步骤和状态，就会产生幻觉、遗忘、重复（来源：Deep Dive into Agent Task Decomposition Techniques）。两边都需要一个“外部调度器”来兜底。

## 同构解法：Inflow 与“会褪去的脚手架”

Inflow 是怎么治情绪失调的？它通过 AI 算法个性化训练工作记忆和认知控制，靶点是背外侧前额叶皮层这个“因果流入中枢”（来源：Inflow）。说白了，它是在给你的大脑加装一个外部的执行功能层，帮你维持注意力、管理任务，从而减少情绪触发。

但这里有一个关键矛盾：**这种外部支撑，到底是“脚手架”还是“拐杖”？** 脚手架是临时支撑，等建筑建好就撤掉；拐杖是永久依赖。Wiki 资料里明确指出了这个争议：多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。而真正的情绪调节，必须最终内化。

这就是“会褪去的脚手架”的含义：一个好的 harness，应该像训练轮一样，随着用户能力提升逐渐减少辅助。Inflow 的动态难度调整算是一种尝试——它根据你的表现调整训练强度，理论上可以逐步撤除。但问题是，现有证据缺乏长期研究，不知道这种外挂是否真的能转化为内在能力（来源：矛盾与存疑）。

## 两边证据：ADHD 侧与 LLM/Agent 侧

**ADHD 侧**：Goblin Tools 通过任务分解降低启动门槛，用户反馈说它“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools）。Saner.AI 通过本地记忆减少搜索循环，直接补偿工作记忆缺陷（来源：Saner.AI）。这些工具都在扮演“外部执行功能层”的角色。但它们的长期效果缺乏随机对照试验，过度依赖风险真实存在（来源：矛盾与存疑）。

**LLM/Agent 侧**：任务分解技术被用来降低 LLM 的认知负荷，减少幻觉（来源：认知负荷）。这跟 Goblin Tools 的思路一模一样——把复杂任务拆成子任务，让模型一步步执行。但同样，如果编排层设计得过于死板，agent 会失去灵活性；如果完全依赖外部编排，模型自身的学习能力可能被削弱。

## 核心观点：脚手架 vs 拐杖的边界在于“褪去设计”

我的判断是：**Inflow 和 agent harness 的成功，不在于它们能提供多少辅助，而在于它们是否内置了“褪去机制”。** 没有褪去设计的辅助工具，最终会变成拐杖，阻碍内在能力发展。证据显示，多巴胺干预可能强化依赖而非内在动机（来源：矛盾与存疑）。同样，一个永远不撤的 agent 编排层，会让模型永远学不会自主规划。

但现实是，目前几乎没有工具认真对待褪去设计。Inflow 的个性化训练算一个雏形，但缺乏随机对照试验验证（来源：Inflow）。其他工具如 Motion、Reclaim.ai 更是默认长期使用。这提醒我们：**别把脚手架当房子住。**

3. **如果你是工程师，在你的 agent 里加入“逐步减少的编排层”**：开始时提供详细步骤和上下文，随着 agent 表现稳定，逐步减少外部提示，让它学会自主推理。

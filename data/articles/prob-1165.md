# Reclaim.ai 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Reclaim.ai：它到底补上了哪一层执行功能？

你有没有这样的经历：买了一堆效率工具，结果花在配置上的时间比真正做事的时间还多？对于 ADHD 大脑来说，这种“工具反而成为新负担”的陷阱尤其常见。Reclaim.ai 号称能自动调度你的日程、保护专注时间、动态调整任务优先级——听起来像是救命稻草。但真正用起来，很多人发现它要么频繁打断节奏，要么因为算法不够懂你而制造更多混乱。

问题出在哪里？不是工具不好，而是我们用错了。Reclaim.ai 之于 ADHD，就像 function calling 工具调用之于 LLM——它补上的是**调度层**，而不是生成层。但很多人误以为它能直接“替我做决定”，结果反而丢失了自主感。

## 同构视角：ADHD 大脑与 LLM 共享同一个缺陷

ADHD 大脑与 LLM 在结构上高度同构：两者都是强大的生成核心，但缺乏可靠的内置执行调度层（来源：ADHD 大脑与 LLM 的同构）。LLM 能写出优美的文章，但如果没有 function calling 把精确计算（如数学、数据库查询）外包给专用工具，它就会胡编乱造。同样，ADHD 大脑能产生无数创意和计划，但缺乏执行功能来启动、排序、切换任务——这些正是 Reclaim.ai 试图补偿的。

Reclaim.ai 的核心机制是：连接你的日历，分析任务优先级，自动安排时间块，并在冲突时动态调整。这本质上就是一个**外部调度器**，对应 LLM 中的 tool-use 层。但 LLM 的 function calling 有一个关键设计：它只负责调用工具，不负责理解工具的内部逻辑。同样，Reclaim.ai 只负责安排时间，不负责判断任务本身是否合理。

## 历史人物早已用过的“调度层”

孔子和释迦牟尼，这两位文化巨人，很可能就是 ADHD 患者。孔子“周游列国 14 年坐不住”“冲动爱骂人”“对音乐超专注到三月不知肉味，对种地等俗事零耐心”——典型的注意力切换困难与超聚焦并存。他的 harness 是“克己复礼”：用外在的礼法作为行为边界，每日反省（“吾日三省吾身”）。这相当于一个**外部调度器**：礼法就是他的日程模板，反省就是每日复盘。

释迦牟尼 29 岁说走就走，放弃王位；先后跟两位仙人学、苦行 6 年，不断试错；49 年游行说法，80 岁死在路上。他的 harness 是“八正道”和“持戒”：用戒律管行为，用正念拴念头。这相当于一个**实时 re-grounding 机制**：正念就是他的上下文重置。

这两位都没有依赖一个“万能管家”来替自己思考，而是主动设计了一套规则系统来辅助执行。这与 LLM 中精心设计的 agent scaffolding 异曲同工：不是让模型自己瞎猜，而是给它一套工具调用协议。

## Reclaim.ai 的正确用法：脚手架，不是拐杖

那么，为什么很多人用 Reclaim.ai 反而更糟？因为把它当成了**拐杖**。

拐杖 vs 脚手架的区别在于：脚手架是临时结构，帮你完成建设后可以拆除；拐杖则是永久替代，长期使用会导致肌肉萎缩（来源：拐杖与脚手架）。Reclaim.ai 如果被用来替你做所有时间决策，你会逐渐丧失自主规划的能力——这正是 ADHD 群体最需要保护的执行功能。

正确的用法是：把 Reclaim.ai 当作**外部调度层**，就像 LLM 的 function calling 一样。你仍然需要自己定义任务的优先级、估算时间、决定哪些任务需要保护——Reclaim.ai 只负责在冲突时帮你调整时间块。它不应该替你做判断，而是执行你的判断。

## 证据与局限

目前，Reclaim.ai 在 ADHD 群体中的效果主要基于用户报告，缺乏独立临床验证（来源：矛盾与存疑）。许多用户反馈其“自动调度”功能在任务多变时反而增加认知负荷。此外，依赖风险真实存在：一项研究指出，AI 工具可能从脚手架变为永久拐杖，削弱内在执行功能（来源：拐杖与脚手架）。

但同构视角提供了新的思路：如果我们将 Reclaim.ai 视为 function calling 工具，那么它的成功取决于**任务接口的清晰度**。就像 LLM 需要明确的工具描述和参数格式，ADHD 用户也需要清晰的任务分解和优先级规则。这正是 Goblin Tools 和 Lex 做得更好的地方：它们专注于任务分解（微任务分解）和单一指令触发（来源：Goblin Tools；Lex）。

## 今天就能试的行动

1. **用 Reclaim.ai 只做“时间块保护”，不做“任务安排”**：手动输入你已确定的任务和优先级，让 AI 只负责在冲突时调整时间，而不是自动插入新任务。
2. **结合 Goblin Tools 进行任务分解**：在 Reclaim.ai 中安排任务前，先用 Goblin Tools 的 Magic ToDo 将任务拆成可执行的步骤（来源：Goblin Tools）。
3. **设置每日“re-grounding”时间**：像孔子的“三省吾身”或释迦牟尼的正念冥想，每天固定时间检查你的日程是否匹配你的真实意图。
4. **警惕“配置漩涡”**：如果花在工具设置上的时间超过使用时间，果断简化——用 Saner.AI 的通用收件箱替代多个工具（来源：Saner.AI）。

## 结语

Reclaim.ai 不是救世主，它只是一个调度层。真正的决策权——任务选择、优先级判断、时间估算——仍然需要你亲自完成。就像 LLM 不能没有 function calling，但 function calling 也不能替模型思考。ADHD 大脑与 LLM 的同构告诉我们：**工具的价值在于补偿执行，而非替代生成**。用对工具，才能从“工具负担”走向“认知自由”。

# 为什么ADHD 数字诊断赛道最该公开的不是准确率，而是偏倚来源表？

> 《问题286》人工精修选题，双域证据作答。

如果你是一位 ADHD 读者，你可能最痛苦的不是「学不会」，而是「为什么我的大脑有时像跑车，有时像抛锚的拖拉机，我却不知道 switch 在哪里」。如果你是一位在做 Agentic Harness 的工程师，你痛苦的可能是：LLM 能滔滔不绝地产出方案，却会在长上下文里跑偏、在工具调用时出错、在多次运行里给出不同结果。两种痛苦其实指向同一个结构：一个高产的生成核心，配了一个不够可靠的执行调度层。问题于是变成：当我们用数字工具诊断或辅助 ADHD 时，真正该公开的不是「准确率 94%」这种单点数字，而是一份偏倚来源表。

## 一、同构：大脑与 LLM 都是「生成核心 + 缺调度层」

从神经科学角度看，ADHD 患者的前额叶执行功能网络激活不足，导致工作记忆不稳定、任务启动困难、注意力分散（来源：ADHD × AI 的科学与研究前沿）。从 AI 工程角度看，LLM 本质上是无状态、缺乏内置调度层的生成核心，需要外部脚手架（harness）来提供上下文传递、工具接口、规划工件、验证循环和记忆系统（来源：GitHub - ai-boost/awesome-harness-engineering）。两者的问题不是「不够聪明」，而是「聪明没有闸门」：内部的生成能力极强，但缺乏稳定可靠的执行调度层来把想法转换成持续、可控、可验证的行动。

这种同构不是抽象比喻。ADHD 个体表现常呈现显著的日内波动，注意力、执行功能在不同时间点差异很大，使诊断需要多次采样才能捕捉症状；LLM 的输出同样受采样温度等随机性因素影响，使评估需要 trace 和多次运行（来源：采样温度与表现波动）。ADHD 的冲动判断可以看作行为层面的「幻觉」——脱离环境线索做出不恰当反应；LLM 则会生成自信但错误的输出。两边的解法都是引入验证循环：ADHD 侧需要把内部状态与外部现实核对，LLM 侧则需要通过检查输出格式、运行测试用例等验证步骤来接地（来源：幻觉与验证循环）。

## 二、为什么准确率不够：偏倚藏在症状波动与 harness 结构里

数字诊断赛道爱公布准确率，但 ADHD 的生成核心特点决定了这个指标很容易被偏倚劫持。症状本身会波动，评估工具的设计、使用环境、文化语境、样本选择方式都会把某种「偏差」包装成「正确识别」。例如，如果一款诊断产品主要在能稳定使用日程工具、能完成在线测试的成年人群体中验证，它实际上已经过滤掉了那些执行功能更差、更无法坚持评估流程的真实患者。准确率越高，可能意味着样本越干净，离真实世界越远。

这正是 LLM 工程师熟悉的困境：一个模型在受控 benchmark 上得分很高，进入生产环境后却会因工具失败、上下文增长、非确定性输出而累积小错误（来源：AI Agent Systems: Architectures, Applications, and Evaluation）。评估 LLM 时，真正关键的是知道控制流是否确定性、有没有状态机约束、有没有周期性采样监控质量（来源：AI Agent Reliability and Guardrails 2026）。同样，评估 ADHD 数字诊断时，真正关键的是知道：它是否对症状波动做了多次采样？是否对高功能、低龄、女性、成人等亚型做了分层验证？是否披露了训练数据中的就诊渠道偏倚、文化语言偏倚、共病偏倚？这些才是偏倚来源表该列的内容。

## 三、Harness 不是拐杖：同构脚手架的真实形态

Harness 的核心作用是外部执行功能层。它既是 ADHD 用户的工具，也是 LLM 系统的工程模式。

Goblin Tools 的 Magic ToDo 把模糊任务（如「清理厨房」）分解为具体、可执行的子任务，通过调节「辣度」控制粒度，这对应 LLM 侧的「计划-执行分离」和子任务分解（来源：Goblin Tools；Planner-Executor Agentic Framework）。Saner.AI 通过本地记忆和快速检索减少搜索循环和标签切换，对应 LLM 的内存架构与上下文管理（来源：Saner.AI；AI Agent Memory Architectures）。Motion 自动排程、动态调整日程，并提前警告任务延期风险，对应 LLM 的确定性状态转换与状态机约束（来源：Motion；AI Agents in 2026: Tools, Memory, Evals, and Guardrails）。

但这里必须区分脚手架与拐杖。脚手架是暂时借用的外部结构，帮助个体或模型完成本应由内部调度层做的事，同时保留自主修正能力；拐杖则是替代性依赖，可能让内部能力进一步退化。例如，AI 辅助带来即时反馈和奖励，可以补偿多巴胺不足，但过度依赖是否削弱执行功能，目前仍无定论（来源：ADHD × AI 的科学与研究前沿；拐杖与脚手架）。对工程师而言，这意味着一个 harness 必须有自我监控和 human-in-the-loop 机制，而不是让 LLM 无止境地自我循环。

## 四、名人案例：毛泽东的 harness 系统就是一套政治版 Agentic Harness

毛泽东常被注意到有典型的 ADHD 特质：孩童时是好动、爱读闲书、与父亲冲突的「问题儿童」；一生读书不辍、行军不停、调查研究不止；决策风格冲动敢赌，思维跳跃，讲话充满场景化比喻（来源：人物案例）。但他同时建立了一套极其严格的外部执行系统：批评与自我批评、民主生活会让他人提意见，对应 LLM  harness 里的自我批评与验证循环；「没有调查就没有发言权」对应 grounding 与外部数据检索；民主集中制对应 planner-executor 框架，用集体决策制衡个人冲动；党指挥枪则是用一个组织系统作为外部调度器，把军队的行动从个人状态波动中抽离出来。

这套系统的同构意义在于：它不是压制生成核心，而是给生成核心套上可审计、可替换、可迭代的执行层。对 ADHD 读者来说，这意味着你不必「修好」自己的大脑，而是像工程师设计 agent 一样设计自己的 harness：哪些决定必须交给外部验证？哪些行动必须拆成最小可执行步骤？哪些状态需要被日程、提醒、他人或系统接管？对工程师来说，这意味着你为自己写的 agent 设计的状态机、记忆、工具接口，其实也是人脑每天在使用的外部前额叶。

## 五、诚实面对局限：同构是框架，不是事实

必须承认，ADHD 大脑与 LLM 的同构性仍是一种理论框架，而非经过验证的科学事实。wiki 资料中「矛盾与存疑」一节明确指出，这一说法缺乏实证基础，不同页面在表述时有时将其当作既定事实，有时当作假设，存在不一致。同时，工具宣称的证据也不足：例如 Motion 被指出缺乏独立临床验证，但相关主题页仍将其作为有效工具推荐（来源：矛盾与存疑）。身体在场效应能否被 AI 虚拟陪伴真正替代、多巴胺检测是否成熟、超聚焦的双面性，这些都还存在争议。因此，本文的核心判断不是「数字诊断没用」或「同构已经成立」，而是：在证据尚不充分的情况下，最该先公开的是偏倚来源表，而不是漂亮的准确率。这能让我们在同构框架真正被验证或证伪之前，避免被一个好听但危险的数字误导。

## 六、今天就能试的四条行动

1. **给自己或你的产品做一次「偏倚来源审计」**：列出诊断/评估流程中，哪些环节会系统性排除高波动、低依从、低收入或女性患者——这些偏倚就是准确率的分母游戏。
2. **用确定性工作流替代模糊提醒**：如果你是 ADHD 用户，用 Goblin Tools 把任务拆到可执行的最小步；如果你是工程师，给 agent 加上计划-执行分离和状态机约束，避免把判断全部交给生成核心。
3. **建立外部验证循环**：ADHD 用户可以找一个问责伙伴或每周复盘；工程师应把验证与 CI 集成、对模型输出运行测试，而不是只看文本通顺。
4. **每月检查一次「脚手架是否变拐杖」**：记录哪些能力确实在提升、哪些被工具接管后反而退化。诚实承认退化，比盲目乐观更符合科学精神。

最终，ADHD 与 LLM 的同构给出的不是一剂解药，而是一个共同的问题意识：当生成核心足够强大时，真正决定可靠性的不是它生成得多好，而是它嵌在什么样的 harness 里，以及这个 harness 的偏倚有没有被公开、被审计、被迭代。

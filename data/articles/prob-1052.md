# 如果把 ADHD 大脑当成生成核心，Forest 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Forest 对 ADHD 的真正价值。

## 问题：为什么你的「专注」总是跑偏？

如果你有 ADHD，你一定经历过这种循环：打开电脑准备写报告，十分钟后发现自己正在研究「企鹅的膝盖为什么能向后弯」；或者，你终于进入超聚焦状态，一口气干了三小时，却发现自己做的是项目里最不重要的那个子任务。你丢失了目标。

如果你在做 agent 工程，你一定也见过这种场景：你给 LLM agent 一个长程任务，它前几步执行完美，但十步之后，它开始生成与原始目标无关的代码，甚至开始「解释」自己为什么要这么做。它丢失了目标。

这两种场景，本质上是同一个问题：**一个高产但缺乏执行调度层的生成核心，在长程任务中发生了目标漂移。**

ADHD 大脑与 LLM 在结构上同构：两者都拥有强大的生成能力（ADHD 的创意发散、LLM 的文本生成），但缺乏可靠的执行调度层（ADHD 的执行功能、LLM 的上下文控制），导致输出不稳定（来源：ADHD 大脑与 LLM 的同构）。对 ADHD 来说，执行功能缺陷——如工作记忆不足、时间盲、任务启动困难——让「保持目标」变成一场消耗战。对 LLM 来说，无状态性和上下文膨胀让「保持目标」同样困难。

那么，一个简单的工具——Forest，那个让你种虚拟树的专注计时器——能不能成为两者的重锚定？

## 同构：重锚定与目标漂移

在 agent 工程中，解决目标漂移的标准做法是「重锚定 system prompt」。当 agent 跑偏时，你重新注入原始目标描述，或者定期检查它的输出是否偏离主要方向。这个外部干预层被称为 **agent harness**——「包裹 LLM 或 AI agent 的软件基础设施，处理模型本身之外的一切」（来源：What is an agent harness in the context of large-language models?）。

ADHD 大脑需要的，正是这样一个 harness。

Forest 的工作原理很简单：你设定一个时间段（比如 25 分钟），种下一棵树，期间不能切换应用。如果中途退出，树会枯萎。这个机制提供了两样东西：

1. **外部计时器**：对抗时间盲。ADHD 患者常因无法感知时间流逝而陷入超聚焦或拖延。Forest 的倒计时提供了一个可见的时间边界。RescueTime 通过自动记录时间使用来提升时间觉察，有初步研究支持其改善时间盲（来源：AI 与 ADHD 的专注力）。Forest 则更进一步：它把时间变成了一个游戏化的承诺。

2. **注意力锚点**：对抗分心。当你看到屏幕上那棵正在生长的树时，它成了一个视觉提示——「现在，你的目标是让树活着」。这个简单的提示，相当于 agent 工程中的「重锚定 prompt」。它不保证你不会走神，但它让你更容易察觉自己已经走神。

但 Forest 的证据强度如何？wiki 资料指出，Forest 在 ADHD 诊断分类中表现良好，但作为专注工具的证据不足（来源：AI 与 ADHD 的专注力）。这意味着，Forest 可能更擅长「检测」你是否专注，而不是「促成」你专注。

## 脚手架 vs 拐杖：Forest 的边界

如果 Forest 只是一个检测工具，那它的价值有限。但如果我们把它视为一种 **外部执行功能层**，它的作用就变得关键。

执行功能层包括任务启动、时间感知、注意力维持等。Focusmate 通过虚拟身体在场提供实时问责，多项用户研究显示其能显著提升任务启动和维持（来源：AI 与 ADHD 的专注力）。RescueTime 通过自动记录减轻工作记忆负担。但这些工具都面临同一个争议：长期使用是否会导致内在执行功能进一步退化？

这就是「脚手架 vs 拐杖」的边界。脚手架帮助你建立自己的能力，最终你可以撤掉它。拐杖则让你永久依赖。

Forest 的设计更接近脚手架。它不替你思考，不替你分解任务，不替你管理时间。它只提供一个简单的约束：**在接下来的 X 分钟内，你只能做一件事**。这个约束迫使你面对自己的注意力，而不是逃避。你每一次成功种树，都是在训练自己的注意力维持能力。神经可塑性研究表明，训练诱导的抑制控制脑可塑性有助于正常化抑制控制能力及其底层脑网络（来源：神经可塑性）。

但它的局限也很明显。对于启动困难严重的用户，Forest 可能毫无作用——你连打开 App 种树的动力都没有。对于超聚焦跑偏的用户，Forest 可能反而有害——它让你在错误的任务上更专注。这就是为什么个体差异导致效果不一（来源：矛盾与存疑）。

## 真正的价值：不是工具，是工程思维

Forest 的真正价值，不在于它本身有多强大，而在于它代表了一种 **工程思维**：把 ADHD 大脑当作一个生成核心，然后给它设计 harness。

在 agent 工程中，harness 不是单一工具，而是一套架构：包括上下文工程、外部记忆、定期重锚定。同样，ADHD 的 harness 也需要组合使用。例如：

- **任务启动**：Focusmate 提供外部问责（来源：Focusmate）。
- **时间感知**：RescueTime 自动记录时间（来源：RescueTime）。
- **注意力维持**：Forest 提供视觉锚点。
- **环境调节**：Brain.fm 通过神经锁相技术影响大脑信息处理方式，可能帮助进入专注状态（来源：Brain.fm）。但需注意，Brain.fm 针对 ADHD 的效果缺乏独立临床研究支持（来源：矛盾与存疑）。

这些工具的组合，构成了一个完整的 **外部执行功能层**。

## 诚实面对局限

我们必须承认，当前证据质量不高。多数工具仅有用户报告或小样本研究，缺乏大规模随机对照试验（来源：AI 与 ADHD 的专注力）。Forest 作为专注工具的证据不足，Brain.fm 的 ADHD 效果缺乏独立临床研究（来源：矛盾与存疑）。过度依赖 AI 工具可能削弱内在时间感知能力（来源：矛盾与存疑）。

更重要的是，ADHD 大脑与 LLM 的同构是隐喻而非严格等价。LLM 的「注意力机制」与人类注意力存在本质差异（来源：AI 与 ADHD 的专注力）。我们不能简单地把 Forest 当作 LLM 的 system prompt 重锚定——人类大脑的神经可塑性更复杂，也更缓慢。

## 今天就能试的行动

1. **设置一个「重锚定」时段**：每天选一个 25 分钟，打开 Forest 种树。在这段时间里，只做你当天最重要的一个任务。如果分心，不要退出，而是把注意力拉回来。这相当于给大脑做一次「system prompt 更新」。

2. **组合使用工具**：用 Focusmate 启动任务（预约一个时段），用 Forest 维持专注（种树），用 RescueTime 事后分析时间使用模式。不要只用一种工具。

3. **记录漂移模式**：每次 Forest 树枯萎时，记录是什么让你分心。一周后，你会发现自己目标漂移的规律——就像 agent 日志一样。然后针对性地调整你的环境或任务分解。

4. **实验「无工具」日**：每周选一天，不用任何专注工具。观察自己的自然专注模式。这能帮助你判断工具是脚手架还是拐杖。

Forest 不能治愈 ADHD，就像 agent harness 不能消除 LLM 的幻觉。但它能让你在目标漂移时，更快地察觉、更早地重锚定。这就是同构思维带来的真正价值：**你不是在对抗自己的大脑，你只是在给它设计更好的 harness。**

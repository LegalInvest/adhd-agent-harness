# Routinery 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Routinery：它到底补上了哪一层执行功能？

如果你是个 ADHD 患者，大概率对这类场景不陌生：明明知道该做什么，但就是启动不了；或者做到一半被别的事吸走，再回来已不知道接哪一步。如果你是个在搭 Agentic Harness 的工程师，你也一定见过 LLM 的「表演波动」：同一个 prompt，温度调高一点就胡说八道，调低一点又死板得像复读机。

这两类问题的底层结构惊人相似——都是**一个高产但缺执行调度层的生成核心**，被扔进一个需要精确、重复、状态维护的世界。ADHD 大脑和 LLM 一样，输出天然带有「采样温度」式的随机性（来源：采样温度与表现波动），而外部工具的作用，就是给这个生成核心套上 harness。

但问题在于：**很多人把脚手架用成了拐杖**。

---

## Routinery 到底在补哪一层？

Routinery 是一款面向 ADHD 的日常程序 App，核心功能是引导用户一步步完成预设的早晨/晚间流程。乍看它只是个「高级待办清单」，但我认为它补的是 ADHD 大脑最缺的那一层：**执行调度层**。

ADHD 的执行功能缺陷，本质上是「工作记忆不稳定 + 任务启动困难 + 注意力易漂移」（来源：ADHD 的 AI 工具全景）。Routinery 通过外部指令序列，把「接下来该做什么」这个决策负担从大脑里卸掉——你不需要记住下一步，App 会告诉你。这恰好对应 LLM/agent 工程中的 function calling：agent 不需要自己算数学，它调一个计算器工具就行。两者都是**认知卸载**（来源：工具使用与认知卸载）。

但这里有个陷阱。Routinery 如果只是机械地列出步骤，用户会逐渐对它脱敏——就像 LLM 如果每次都被强制调用同一个工具，它就不再去思考「是不是该换个方法」。真正有效的 harness，应该是**脚手架而非拐杖**：它提供结构，但保留用户/agent 的自主调度权。

## 同构证据：从孔子到左宗棠到 LLM

历史上，ADHD 特质明显的人物早已在用自己的方式搭建 harness。孔子「吾日三省吾身」——每天固定时间自我检查，相当于一个定时 re-grounding 的调度器；他周游列国、精力旺盛、冲动爱骂人（骂宰予「朽木不可雕」），却用「礼」作为外在行为边界来约束自己（来源：人物案例）。这套系统，本质上和 LLM 工程里的「确定性状态转换」同构：通过外部规则（礼/状态机）来稳定一个天生波动的核心。

左宗棠则更直接：他「今天的事今天做完」，用「结硬寨打呆仗」的笨办法来对冲自己风风火火、容易冲动的特质；每天读经写家书，相当于固定的 grounding 流程（来源：人物案例）。这和 agent 工程里的「计划-执行分离」模式异曲同工——先定死计划，再按部就班执行，避免执行中途被新想法带偏。

LLM 一侧的证据更硬：单个 LLM 不足以可靠完成多步骤任务，需要 agent scaffolding 来补偿（来源：工具使用与认知卸载）。工程上强调「确定性控制流」和「周期性采样监控质量」（来源：采样温度与表现波动）。这些和 ADHD 的 harness 完全对应：外部结构（日程/提醒/环境设计）稳定表现，就像确定性工作流稳定 LLM 输出。

## 但有人用错了：脚手架 vs 拐杖的边界

我观察到两类典型错误：

**ADHD 侧**：把 Routinery 当成「只要跟着做就能好」的万能药。结果有一天 App 没提醒，就一整天瘫在沙发上。这就是从脚手架变成了拐杖——工具替代了内在调度能力，而非补偿它。真正的脚手架应该像 Goblin Tools 的「辣度」调节（来源：Goblin Tools）：你可以选择分解粒度，逐步训练自己独立分解任务。

**LLM/agent 侧**：给 agent 塞太多工具，导致它遇到任何问题都先调工具，而不是自己推理。这和 ADHD 患者「先刷手机再想问题」的回避行为一模一样。工程上需要「工具契约」——明确工具只用于精确计算，不用于替代核心推理（来源：采样温度与表现波动）。

边界在哪？看两点：
1. **是否可调**：好的脚手架允许你逐步减少依赖（如 Routinery 的「减少提示」功能）；拐杖则只有开/关。
2. **是否保留自主权**：脚手架告诉你「下一步可选 A 或 B」，拐杖直接替你走。

## 争议与局限

必须诚实指出：ADHD 大脑与 LLM 的同构性目前只是一个理论类比，缺乏实证基础（来源：矛盾与存疑）。我们无法确定 ADHD 的「执行功能缺陷」和 LLM 的「无状态性」真的是同一回事，还是只是表面相似。

另外，Routinery 这类工具在 ADHD 人群中的独立临床验证仍然有限（来源：矛盾与存疑）。多数效果基于用户报告，存在安慰剂效应。而且，过度依赖工具的风险是真实存在的——有用户反馈「没了 ChatGPT 我连邮件都不会写了」（来源：ADHD 的 AI 工具全景）。

## 今天就能试的行动

1. **给 Routinery 做「辣度」测试**：把你的早晨流程分解成 5 步，然后每天减少 1 步提示，直到只保留第 1 步。如果卡住，就回到上一步——这和 LLM 的「温度调节」一样，找到自己的最佳确定性水平。
2. **建立「工具契约」**：选一个你最常外包的任务（比如写周报），规定自己必须先用大脑写 5 分钟，才能用 AI 辅助。这和 agent 工程里「先计划再执行」同构。
3. **模仿孔子的「三省」**：设置每天 3 个固定时间点（比如早、中、晚），用 30 秒问自己「我现在该做什么？」——不依赖 App，只依赖时间锚点。这是最原始的 harness。
4. **如果你在搭 agent**：给你的 agent 加一个「最小工具集」原则——只给那些它绝对算不准的工具（如计算器、数据库查询），其余让它自己推理。观察输出质量变化。

Routinery 不是解药，function calling 也不是银弹。它们都是脚手架，不是拐杖。真正的价值在于：当你意识到自己的「生成核心」需要外部调度层时，你已经在用工程师的思维审视自己的大脑——而这就是改变的开始。

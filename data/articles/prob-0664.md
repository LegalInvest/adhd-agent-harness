# Goblin Tools 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Goblin Tools：它到底补上了哪一层执行功能？

## 问题：为什么你的待办清单越用越崩溃？

如果你是一个 ADHD 患者，很可能经历过这样的场景：打开 Goblin Tools 的 Magic ToDo，输入“整理房间”，它瞬间吐出一份包含 12 个步骤的清单——从“找到垃圾桶”到“把书放回书架”。你松一口气，觉得终于有人替你思考了。但两周后，你发现自己连第一步都没做完。

如果你是一个在做 Agentic Harness 的工程师，你大概也遇到过类似的困惑：给 LLM 挂上十几个 function calling 工具，它却在一个简单的计算任务上反复调用计算器，或者把“发送邮件”拆成“打开邮箱-点击写邮件-输入地址-输入正文-点击发送”五个步骤——然后卡在“输入地址”这一步，因为工具返回了格式错误。

两群人，同一个痛点：**工具补上了执行层，但使用者误以为工具能替代执行层本身。**

## 同构：ADHD 大脑与 LLM 共享的“生成-调度”缺口

核心论点很直接：ADHD 大脑与 LLM 在架构上是同构的——都拥有一个高产但缺乏内置执行调度层的生成核心（来源：ADHD 大脑与 LLM 的同构）。ADHD 大脑想法丰富、联想跳跃，但工作记忆不稳定、时间盲严重、任务启动困难（来源：时间盲；任务启动）。LLM 同样能生成流畅文本，却在长程上下文保持、目标跟踪和精确计算上天然缺陷。

两边的解法也同构：**外部脚手架**。

对 LLM，这个脚手架叫 Agentic Harness——通过 function calling 把精确计算外包给专用工具，通过规划模块将复杂目标分解为子任务，通过记忆系统维持上下文。对 ADHD 大脑，这个脚手架就是 Goblin Tools、Lex、Saner.AI 这类 AI 工具——它们承担了任务拆解、时间估计、外部记忆和启动提示的功能（来源：Goblin Tools；Lex；Saner.AI）。

但关键区别在于：LLM 的 harness 是工程师设计的，它知道工具只是工具，核心是 LLM 的生成能力。而 ADHD 用户很容易把工具当成“大脑的替代品”——用 Goblin Tools 拆好任务，然后就等着被推着走。

## 案例：苏轼的“外部调度器”与 LLM 的“日课”

苏轼是 ADHD 特质明显的天才：兴趣极广（诗词文书画美食治水炼丹），对无聊行政零耐心，情绪来得快去得快。他的 harness 系统以“豁达”为核心：不管贬到哪里，都用美食、写作、交朋友转移注意力；每日读书写字，不因挫折停止创作。

这套系统与 LLM 的 Agentic Harness 高度同构：
- **日课 ↔ 定时 re-grounding**：苏轼每日读书写字，相当于给 LLM 设置固定的“上下文刷新”机制，防止生成漂移。
- **外部兴趣 ↔ 工具调用**：苏轼用美食、交友作为“外部奖励系统”，就像 LLM 用 function calling 获取外部数据——不是靠内部算力硬扛，而是从外部借力。
- **豁达 ↔ 错误容忍**：苏轼被贬就适应，相当于 LLM 在 tool call 失败后重试或回退。

但苏轼的 harness 不是拐杖。他并没有因为有了“豁达”就停止创作，反而写出了《赤壁赋》。同样，Goblin Tools 的正确用法不是让你躺平等待办清单，而是让你在拆解完任务后，**自己决定先做哪一步**。

## 误用：当工具变成拐杖

Goblin Tools 的 Magic ToDo 允许用户调节“辣度”控制分解粒度（来源：Goblin Tools）。这本身是好事——你可以把“写论文”拆成 5 步或 50 步。但问题在于：**拆得越细，启动门槛越低，但执行主动性也越容易被外包。**

有用户反馈，Goblin Tools 的分解功能“很棒”，但自己仍然无法开始第一步（来源：“A Cognitive Collaborator:” How Adults with ADHD Are Using ChatGPT）。这不是工具的错，而是使用姿势错了——他们把工具当成了“执行替代”，而不是“执行辅助”。

同样，工程师给 LLM 挂上过于细粒度的 function calling（比如“打开文件-读取第一行-判断是否为空-如果为空则报错”），结果 LLM 在每一步都要调用工具，反而增加了延迟和失败率。正确的做法是：**只把真正需要精确计算或外部数据的步骤外包，保留生成和决策在 LLM 内部。**

## 脚手架 vs 拐杖：边界在哪里？

“外部执行功能层”与“拐杖”的边界，在于**是否保留了用户/模型的主动调度能力**（来源：矛盾与存疑）。

- **脚手架**：工具提供结构，但用户/模型自己决定何时、如何调用。例如 Goblin Tools 拆好任务后，用户自己勾选完成；LLM 的 function calling 由模型自主决定调用时机。
- **拐杖**：工具接管了决策，用户/模型只是被动执行。例如用户完全依赖 Goblin Tools 的步骤顺序，从不调整；工程师强制 LLM 每一步都必须调用工具。

数据显示，ADHD 组在超专注工作时的平均时长是 14.5 分钟，而对照组是 8.2 分钟（来源：人物案例相关性统计）。这意味着 ADHD 大脑在正确触发下可以产生极高的专注产出。工具应该帮助触发这种状态，而不是替代它。

## 诚实局限

1. **缺乏长期证据**：目前关于 AI 工具对 ADHD 的长期效果缺乏纵向研究（来源：矛盾与存疑）。短期有效不等于可持续。
2. **个体差异大**：部分用户反馈强行打断（如番茄钟）比柔性引导更有效（来源：矛盾与存疑）。Goblin Tools 的“辣度”调节虽好，但无法覆盖所有偏好。
3. **学习成本**：使用 AI 工具本身可能增加认知负荷（来源：矛盾与存疑）。对于 ADHD 用户，学习新工具本身就是一种挑战。

## 今天就能试的行动

1. **用 Goblin Tools 拆一个你拖延最久的任务，但拆完后自己重新排序**——不要按它的顺序做，按你的直觉做。这训练你的调度能力。
2. **给 ChatGPT 或 Claude 一个模糊目标，但不允许它调用任何工具**——看它能否仅靠生成能力给出合理方案。这让你感受“纯生成”与“带工具”的差异。
3. **设置一个“日课”提醒**——每天固定时间做一件小事（比如写 50 字），不依赖任何工具。这模拟苏轼的“每日读书写字”，强化内部调度。
4. **如果你是工程师，审查你的 agent 的 function calling 频率**——如果每次对话调用超过 5 次，考虑合并工具或减少粒度。

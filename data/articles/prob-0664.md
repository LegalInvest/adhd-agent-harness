# Goblin Tools 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Goblin Tools：它到底补上了哪一层执行功能？

你有没有过这样的时刻：脑子里明明知道该做什么，但就是无法开始？或者，你写了一个漂亮的 prompt，让 LLM 去执行一个复杂任务，结果它却开始胡编乱造？

这两件事听起来风马牛不相及，但底层问题一模一样：**一个高产但缺乏执行调度层的生成核心，被扔在了没有脚手架的空地上。**

ADHD 大脑与 LLM 在认知架构上存在深刻同构——两者都拥有强大的生成能力（创意、联想、语言），却缺乏稳定的执行调度层来把意图转化为有序的行动（来源：ADHD 的 AI 工具全景）。ADHD 的执行功能缺陷（工作记忆弱、任务启动难、时间盲）对应 LLM 的上下文控制缺陷（上下文膨胀、冷启动困难、时序推理弱）。

而 Goblin Tools 的 Magic ToDo 功能，恰好就是为这个生成核心套上的第一个脚手架。

## 问题：为什么 Goblin Tools 对一些人来说是神，对另一些人来说是鸡肋？

Goblin Tools 的核心功能是“魔法待办”：你输入“整理房间”，它输出“捡起地板上的衣服”“擦桌子”“叠被子”等小步骤（来源：Goblin Tools）。用户反馈称它“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools）。

但我也看到有人抱怨：“Goblin Tools 分得太细了，反而让我更焦虑。”或者“我用了三天就放弃了。”

问题出在哪？

答案藏在同构视角里：**Goblin Tools 只做了任务分解，但它没有做上下文管理和实时反馈。** 一个完整的执行功能层需要三个组件：
1. **任务分解**（降低启动门槛）
2. **上下文管理**（保持注意力方向）
3. **时间锚点**（弥补时间盲）

Goblin Tools 只提供了第一个。对于轻度执行功能缺陷的用户，这已经足够；但对于重度时间盲或工作记忆严重不足的用户，它就像给了你一张地图却没有指南针。

## 同构：LLM 的 function calling 也面临同样的陷阱

LLM 的 function calling 机制允许模型把精确计算（如数学运算、数据库查询）外包给外部工具。这本质上也是一种任务分解：把“计算 25 的平方根”分解为“调用 sqrt 函数”和“返回结果”。

但很多开发者犯的错误是：**他们只定义了 function calling 的接口，却没有设计上下文管理和错误恢复机制。** 结果就是：LLM 调用了正确的函数，但把参数传错了；或者函数返回了异常数据，LLM 却继续基于错误结果推理。

这跟 ADHD 用户只用了 Goblin Tools 的任务分解，却没有配套的上下文管理（比如 Saner.AI 的增强回忆）和时间锚点（比如 Tiimo 的视觉时间表）一样——**工具只补了一半的缺口。**

## 答案：Goblin Tools 补上了“任务启动”这一层，但需要组合使用

从证据来看，Goblin Tools 最直接的贡献是降低任务启动门槛（来源：任务启动）。ADHD 用户的多巴胺系统失衡导致行动动机不足，而将大任务拆成小步骤可以创造“小胜利”的即时反馈，激发多巴胺（来源：任务启动）。

但 Goblin Tools 不解决工作记忆问题——你分解完任务后，可能转头就忘了下一步是什么。它也不解决时间盲问题——你不知道每个小步骤要花多久。

因此，真正的“外部执行功能层”需要工具组合：
- **Goblin Tools** 负责任务分解（启动）
- **Saner.AI** 负责知识回忆（上下文管理）
- **Tiimo** 或 **Motion** 负责时间可视化（时间锚点）

这就像 LLM agent 的完整 harness：function calling 负责精确计算，retrieval-augmented generation 负责上下文管理，而 scheduler 负责时序控制。

## 争议与局限：脚手架还是拐杖？

wiki 资料中反复出现一个矛盾：AI 工具被设计为“脚手架”促进能力发展，但实际使用中可能沦为“拐杖”（来源：矛盾与存疑）。例如，Otter.ai 减轻笔记负担，但过度依赖可能削弱主动记笔记的能力（来源：ADHD 的 AI 工具全景）。

对于 Goblin Tools 来说，同样存在这个风险：如果用户每次都依赖它分解任务，自己从不练习拆解能力，那么一旦没有工具，启动困难可能更严重。

另一个局限是证据强度：现有支持 Goblin Tools 有效性的证据主要来自用户报告，缺乏大规模随机对照试验（来源：Goblin Tools）。个体差异也很大——有些人觉得分解太细反而增加认知负荷，有些人则觉得恰到好处（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个你拖延已久的小任务**（比如“写周报”），然后对比自己手动分解的感受。注意：如果分解后仍然不想开始，说明你需要组合其他工具。
2. **把 Goblin Tools 的步骤粘贴到 Saner.AI 或备忘录里**，作为“外部工作记忆”，这样你就不会做完一步就忘记下一步。
3. **给每个步骤加一个时间估计**（可以用 Tiimo 或手机计时器），弥补时间盲。
4. **观察自己的使用模式**：你是把 Goblin Tools 当作脚手架（用完后能自己拆解类似任务），还是拐杖（没它就无法启动）？如果是后者，尝试逐步减少使用频率。

Goblin Tools 不是万能药，但它是一个绝佳的起点。它证明了：**为生成核心套上执行调度层，远比强迫核心改变自身架构更有效。** 对于 ADHD 用户和 LLM 工程师来说，道理是一样的——不要试图修复生成核心，而是给它一个更好的 harness。

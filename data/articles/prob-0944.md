# 如果把 ADHD 大脑当成生成核心，Goblin Tools 能不能当它的重锚定与目标漂移？

## 问题：分心与超聚焦，ADHD 的“目标漂移”困局

ADHD 大脑是一个高产但缺乏执行调度层的生成核心——它能迸发创意、深度超聚焦，却常常在任务中途被分心劫持，或陷入无关细节无法自拔。这种“目标漂移”并非懒惰，而是执行功能障碍的典型表现：工作记忆薄弱、时间盲、任务启动困难（来源：执行功能障碍）。想象一下，你打开电脑准备写报告，半小时后却发现自己正刷着无关网页，或对着一个非关键段落反复修改——这就是 ADHD 的日常。

## 同构：ADHD 大脑与 LLM 的相似困境

有趣的是，LLM（大语言模型）在长程任务中也会遭遇同样的“目标漂移”。一个未经编排的 LLM 会因上下文溢出或指令模糊而偏离原始目标，需要重锚定 system prompt 或引入编排层（harness/scaffolding）来保持输出可靠性。ADHD 大脑与 LLM 在结构上同构：两者都是高产的生成核心，但缺乏可靠的执行调度层（来源：AI 与 ADHD 的专注力）。ADHD 的专注力问题可类比为 LLM 的上下文溢出和调度失败。

## 重锚定方案：Goblin Tools 作为外部执行功能层

Goblin Tools 正是针对这种“目标漂移”设计的工具。它通过 AI 将模糊任务分解为可执行步骤，并提供“魔法师”等模式调整语气——这相当于为 ADHD 大脑提供一个外部的 system prompt 重锚定机制。当用户说“写邮件”，Goblin Tools 会生成提纲、草稿、修改建议，将抽象目标转化为具体动作，弥补工作记忆和任务启动缺陷。这种“外部执行功能层”与 LLM 的编排层功能同构：两者都通过外部结构补偿内部调度缺失，确保生成核心不跑偏。

## 核心观点：脚手架 vs 拐杖的边界

Goblin Tools 的真正价值在于它是一副可调节的脚手架，而非永久拐杖。脚手架的设计应允许逐步撤除，但当前多数工具（包括 Goblin Tools）未提供明确的撤除机制，长期依赖可能阻碍内在执行功能发展（来源：矛盾与存疑）。一个负责任的用法是：将 Goblin Tools 作为训练辅助，利用神经可塑性原理，在外部支持下逐步内化任务分解能力（来源：神经可塑性）。例如，先用 Goblin Tools 分解任务，然后尝试自己列出步骤，最后只在卡顿时求助。

- **证据缺口**：多数工具缺乏针对 ADHD 的独立临床研究，效果多基于理论推断或用户主观报告（来源：AI 与 ADHD 的专注力）。

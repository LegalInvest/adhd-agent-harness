# 如果把 ADHD 大脑当成生成核心，Goblin Tools 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Goblin Tools 对 ADHD 的真正价值。

## 问题：当生成核心失去锚点

你是否有过这样的经历：打开电脑准备写报告，十分钟后却在浏览维基百科上某个无关的条目；或者，你正在专注地编码，突然发现自己在优化一个根本不在当前 sprint 里的功能。对于 ADHD 人群，这是日常——分心走神，或者更隐蔽的“超聚焦跑偏”：过度投入细节，却丢失了最初的目标。

在 AI 工程领域，一个类似的问题正在困扰开发者：当 LLM 被用于长程任务（如多步推理、代码生成），它常常在过程中“目标漂移”——从解决核心问题滑向无关分支，或者被上下文中的噪声带偏。工程师们不得不设计复杂的“harness”（如 ReAct 循环、system prompt 重锚定）来让 agent 始终对准目标。

这两个场景看似无关，却共享同一个深层结构：**一个高产但缺乏可靠执行调度层的生成核心**。ADHD 大脑与 LLM 都是如此。那么，如果把 ADHD 大脑视为一个“生成核心”，现有的 AI 工具——尤其是专为 ADHD 设计的 Goblin Tools——能否充当它的重锚定系统，对抗目标漂移？

## 同构：ADHD 大脑与 LLM 的共享困境

ADHD 的执行功能障碍（Executive Dysfunction）是其核心特征之一，表现为计划、组织、时间管理、任务启动等方面的困难（来源：A review of executive function deficits in autism spectrum disorder and attention-deficit/hyperactivity disorder）。这并非努力不足，而是大脑的“调度层”效率低下。类似地，LLM 拥有强大的生成能力，但缺乏内在的执行调度——它依赖 prompt 和上下文来维持方向，一旦上下文膨胀或出现干扰，输出就会漂移。

更具体地说，ADHD 的[工作记忆]缺陷与 LLM 的无状态性同构：两者都需要外部记忆系统来维持连续性。ADHD 患者容易“忘记”任务目标，LLM 则容易“忘记”早期指令。两者都需要主动设计“当下注意什么”的工程方案——这正是上下文工程的核心（来源：AI 与 ADHD 的专注力）。

因此，AI 帮助 ADHD 的本质，不是替代大脑，而是**给生成核心套上 harness（外部执行功能层）**（来源：AI 与 ADHD 的专注力）。这与为 LLM 搭建 agent harness 在结构上同构：提供结构化提示、外部记忆、注意力锚点。

## Goblin Tools：轻量级重锚定系统

在众多 ADHD 工具中，Goblin Tools 以其“魔法师”式的任务分解功能闻名。它接受一个模糊的目标（如“整理房间”），然后自动拆解为可执行的子步骤。这看似简单，但放在同构视角下，它的真正价值浮现：**它充当了一个实时重锚定机制**。

当 ADHD 用户面对一个复杂任务时，大脑的生成核心可能立即发散：想到一百种“整理”的方式，或者因为不知道从何开始而陷入瘫痪。Goblin Tools 通过提供外部结构，将“当前应该做什么”锚定在具体的子步骤上，从而抑制目标漂移。这类似于 LLM agent 中的“ReAct”模式：每一步都让模型回顾当前目标，并只执行与目标相关的动作。

但 Goblin Tools 的局限同样明显。它只提供“分解”，不提供“执行监控”——用户仍可能在做子步骤时跑偏。相比之下，Focusmate 通过实时同伴问责提供持续的外部监督，其核心机制是“AI-Matched Body Doubling”（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...)），即通过智能调度优化身体在场效应。这更像一个完整的“harness”：不仅提供初始锚点，还在执行过程中持续反馈。

RescueTime 则从另一个角度解决目标漂移：通过自动记录时间使用，帮助用户识别自己何时漂移（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。它不直接干预，但提供事后分析，类似于 LLM agent 的日志系统。

## 脚手架还是拐杖？争议与边界

然而，这些工具的有效性证据大多来自用户报告，缺乏大规模对照实验（来源：矛盾与存疑）。更关键的是，存在依赖风险：长期依赖外部执行功能层，是否会导致 ADHD 患者的内在执行功能进一步退化？（来源：矛盾与存疑）

这引出了“脚手架 vs 拐杖”的边界问题。脚手架是临时辅助，帮助用户建立内在能力后即可拆除；拐杖则是永久替代。Goblin Tools 的分解功能，如果用户只是机械执行而不内化分解策略，就可能沦为拐杖。Focusmate 的实时问责，如果用户无法在没有伙伴时启动任务，同样如此。

从同构视角看，LLM agent 的 harness 也有类似风险：过度依赖外部 prompt 工程，可能导致模型本身的推理能力退化？但 LLM 没有“内在能力”需要培养，而人类有神经可塑性——通过训练可以改善执行功能（来源：Harnessing neuroplasticity for clinical applications）。因此，AI 工具的设计应促进这种可塑性，而非完全替代。

## 核心判断：Goblin Tools 是重锚定起点，而非终点

综合来看，Goblin Tools 可以作为 ADHD 大脑的轻量级重锚定系统，但它的作用局限于任务启动阶段。对于长程任务中的持续目标漂移，需要更完整的 harness——例如 Focusmate 的实时问责或 RescueTime 的监控反馈。

更重要的是，这些工具应被视为“脚手架”：使用它们的最终目标，是训练大脑逐渐内化这些外部结构。例如，通过反复使用 Goblin Tools 的任务分解，用户可能学会自己分解任务；通过 Focusmate 的陪伴，用户可能逐渐建立内在的启动信号。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个你拖延已久的任务**：输入模糊目标，观察它如何拆解。然后，尝试在无工具的情况下，为另一个任务做类似的分解——比较差异，感受“外部结构”如何改变你的启动意愿。

2. **设置一个“重锚定提示”**：模仿 LLM agent 的 system prompt，在手机或电脑上设置一个每小时弹出的提醒，内容是你当前的核心目标（如“写文章第三段”）。这相当于手动实现一个轻量级的“目标漂移检测”。

3. **尝试一次 Focusmate 会话**：选择一个需要高度专注的任务，预约 50 分钟。注意，在同伴注视下，你的目标漂移是否减少。事后记录：这种外部问责是否让你更“锚定”？

4. **使用 RescueTime 追踪一天**：第二天查看报告，找出你“漂移”最多的时段。问自己：当时发生了什么？是任务太难（导致逃避）还是太容易（导致无聊）？这能帮你设计更好的重锚定策略。

## 局限与展望

本文的论证基于 ADHD 大脑与 LLM 的同构隐喻，但它并非严格等价——LLM 的“注意力机制”与人类注意力存在本质差异（来源：AI 与 ADHD 的专注力）。此外，ADHD 的异质性使得同一工具对不同亚型效果迥异（来源：矛盾与存疑）。Goblin Tools 可能对“任务启动困难”型有效，但对“超聚焦跑偏”型作用有限。

最终，AI 工具能否成为 ADHD 大脑的可靠重锚定系统，取决于我们是否将其设计为脚手架，而非拐杖。这需要工具开发者、临床研究者与用户共同探索——而第一步，是承认同构的启发价值，同时警惕其局限。

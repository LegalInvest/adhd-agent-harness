# 如果把 ADHD 大脑当成生成核心，Goblin Tools 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Goblin Tools 对 ADHD 的真正价值。

你打开电脑准备写季度总结，三小时后发现自己在整理 2008 年的 MP3 标签。另一边，你写的 Agent 本应帮用户订一张机票，却在 20 轮对话后热心地开始给目的地写旅游攻略。两种失控看起来毫不相干，但它们的根是一样的：核心生成能力很强，执行调度层却脆弱得惊人。ADHD 人群称这叫“走神”或“超聚焦跑偏”；Agent 工程师叫它“目标漂移”。本文想回答：如果 ADHD 大脑是生成核心，Goblin Tools 能不能像给 agent 重注 system prompt 一样，把它拉回目标？

## 同构诊断：两种生成核心，同一种漂移

把 ADHD 大脑和 LLM/agent 放在一起看，结构上是同构的：两者都拥有强大的生成能力——ADHD 的创意发散、超聚焦与联想，LLM 的文本生成与推理——但都缺乏一个可靠的执行调度层（来源：AI 与 ADHD 的专注力）。

对 ADHD 来说，这个调度层是执行功能：工作记忆、抑制控制、时间感知、任务启动。对 LLM agent 来说，它是上下文管理、目标保持、长程规划和工具调用边界。两者都会因为“上下文”变长或“认知负荷”增加而失控：人在多任务中丢失主线，模型在对话变长后偏离原始目标。

更硬的证据来自一项将经典 Stroop 冲突任务用于 Transformer 注意力的研究：在短上下文里模型表现正常，但随着序列变长、认知负荷增加，模型在冲突试次上骤降到随机水平，无法抑制优势反应、无法解决认知冲突——这与 ADHD 执行功能缺陷的核心标志几乎一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃（来源：Deficient Executive Control in Transformer Attention）。换句话说，ADHD 大脑与 LLM 在同一种“任务越长越跑偏”的曲线里。

## 证据双轨：ADHD 的 harness 与 Agent 的 harness

既然病根是“缺少外部调度层”，解法的结构也必然同构。ADHD 侧的真实工具已经走在这条路上：

- **外部记忆**：RescueTime 自动记录时间使用，把“时间到底去哪了”外化，减轻工作记忆负担（来源：Revolutionizing ADHD Management with AI Assistants）。
- **实时问责**：Focusmate 通过虚拟身体在场效应，让任务启动和维持变得更容易，本质是把社会压力变成外部执行功能（来源：AI 与 ADHD 的专注力）。
- **状态锚点**：Brain.fm、Endel 等利用声音设计或神经锁相影响专注状态，但针对 ADHD 的独立临床证据仍然不足（来源：AI 与 ADHD 的专注力；Brain.fm）。
- **自动决策**：部分 AI 工具通过替用户决定“下一步做什么、何时做、如何优先排序”来降低认知负荷（来源：9 Best AI Assistants for ADHD in 2026 - by Nia - rivva blog）。

Agent 侧的做法几乎镜像：

- **外部记忆**：向量数据库、检索增强生成（RAG）和计划缓存，让 agent 在长程任务中不丢掉上下文。
- **实时问责/检查**：用工具调用、中间检查点（checkpoint）或人在回路（human-in-the-loop）把任务拆解成可验证的小步。
- **重锚定 system prompt**：在关键节点重新注入原始目标、角色和约束，防止 prompt 被长对话稀释。
- **状态锚点**：guardrails、确定性工作流、约束采样，确保模型不会因为“创意发散”跑题。

两套 harness 的对应关系很清晰：RescueTime 相当于 agent 的日志与可观测性；Focusmate 相当于 co-working 监督或 human-in-the-loop；Brain.fm 相当于环境上下文工程；而 system prompt 的定时重注，对应的就是 ADHD 人群最需要的那一句“你本来要做什么”。

## 曾国藩的日课：一个人的“system prompt”

要理解 harness 是什么，最贴切的例子可能是曾国藩。史料中的他完全符合注意力不集中型 ADHD 的画像：30 岁前浮躁坐不住，天天串门喝酒看热闹；读书时“他人目下二三行，余或疾读不能终一行”；情绪极端，打败仗就跳水自杀。他的专属 harness 是“日课十二条”：黎明即起、读书不二、谨言、写日记反省，以及“结硬寨、打呆仗”的笨办法。

把这映射到 agent 工程上：

- **日课十二条** = 定时重新注入 system prompt：每天固定动作把目标、角色和约束读回大脑，对抗上下文漂移。
- **写日记骂自己** = 外部日志与反思机制：把行为轨迹外化，让大脑能从系统 2 复盘，而不是被系统 1 的冲动带着跑。
- **结硬寨打呆仗** = 确定性工作流：把复杂目标拆成可重复、可验证、低创意的小步，减少中途漂移的机会。

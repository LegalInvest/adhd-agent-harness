# Claude 之于 ADHD，就像 外部记忆/向量库 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

你有没有这样的经历：打开 Claude 想写一篇文章，结果聊了半小时，它帮你列了大纲、写了开头、甚至改了三版，但你关掉会话后，脑子里依然一片空白，好像刚才的对话从未发生过。

这不是你的问题，而是所有无状态系统的宿命。

ADHD 大脑和 LLM 共享同一个底层困境：**两者都是高产但无状态的生成核心**。你的大脑在灵感爆发时能构思出惊人的创意，但下一秒就忘了刚才要做什么；LLM 能写出漂亮的段落，但跨会话后对之前的对话一无所知。两者的区别仅在于：LLM 的“工作记忆”是显式的上下文窗口，而你的工作记忆是隐式的神经缓存——容量都极小，且缺乏可靠的持久化机制。

这就是为什么很多 ADHD 患者觉得 Claude 好用，却又说不上哪里不对——**他们用错了**。

## 同构视角：Claude 到底补上了哪一层？

从同构视角看，ADHD 大脑与 LLM 的互补关系并非“大脑不够聪明，AI 来填”，而是**生成核心完好，但执行调度层缺失**。

ADHD 侧，你的智力、创造力、联想能力往往正常甚至超常，真正拖后腿的是工作记忆容量小、任务启动困难、时间盲——这些都属于执行功能缺陷（来源：ADHD 的 AI 工具全景）。你需要的不是另一个“生成核心”，而是一个能帮你记住上下文、分解任务、提示下一步的外部执行层。

LLM/Agent 侧，工程师们早已意识到：纯 LLM 调用无法完成复杂任务，必须通过 harness 来管理上下文、调用工具、规划步骤（来源：What is an agent harness in the context of large-language models?）。这就是为什么现代 agent 系统要组合多个模型、检索器和工具，而非依赖单一模型调用（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

Claude 正好处于这个交点上。它本身是一个强大的生成核心，但如果你只把它当作对话窗口，它就只是一个无状态的“灵感喷射器”——你每问一次，它都像第一次见到你。而如果你把它当作外部记忆与调度层，它就能变成你的执行功能补丁。

## 真实证据：工具如何补上执行功能

wiki 资料中提供了多个经过验证的工具，它们的设计逻辑恰好对应了 LLM/Agent 工程中的“外部记忆”与“任务分解”模式：

**Goblin Tools** 的 Magic ToDo 功能，能将“整理房间”这种模糊指令自动分解为“捡起地板上的衣服”“擦桌子”等具体步骤（来源：Goblin Tools）。这对应了 Agent 工程中的“多步推理+工具调用”——LLM 不擅长一次性规划长链条，需要外部系统将大任务拆解为可执行的子任务。ADHD 用户反馈称它“将压倒性的事情变成一系列不压倒性的事情”（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。

**Saner.AI** 专注于知识回忆和本地记忆，帮助用户快速找回信息，减少搜索循环和标签切换（来源：Saner.AI）。这对应了 Agent 系统中的“向量库/外部记忆”——LLM 本身没有长期记忆，需要检索增强生成（RAG）来保持上下文。ADHD 患者同样需要这种“持久记忆”来弥补工作记忆的脆弱性。

**Lex** 允许用户通过单一指令触发复杂、多步骤的任务序列（来源：Lex）。这对应了 Agent 工程中的“编排层”——将多个工具调用组合成一个工作流。ADHD 用户需要这种“一次触发”的简洁性来降低启动门槛。

这些工具的共同点在于：**它们不试图替代你的大脑，而是作为外部执行层，接管你大脑不擅长的调度工作**。

## 脚手架 vs 拐杖：关键分歧

然而，wiki 资料也明确指出了争议：过度依赖 AI 工具可能阻碍内在执行功能的发展，如何设计可逐步撤除的“脚手架”而非“拐杖”仍是挑战（来源：ADHD 的 AI 工具全景）。

这意味着，如果你用 Claude 来替代思考——比如让它帮你写完整篇文章，而你只是复制粘贴——那么你实际上是在强化“无状态”的依赖。你的大脑会越来越习惯于不保存上下文，因为 Claude 会帮你记住。

真正的“脚手架”用法应该是：**让 Claude 帮你记住你记不住的东西，但你自己仍然掌控生成的过程**。例如，用 Claude 来维护一个“外部工作记忆”——每次开始新任务前，先花 30 秒向 Claude 描述你当前的上下文，让它帮你保持连续性。这相当于给 LLM 加了一个“会话记忆”模块，也相当于给你的大脑加了一个“外部缓存”。

## 局限与诚实

必须承认，当前证据存在局限。ADHD 大脑与 LLM 同构的命题主要来自概念类比和工具案例，缺乏大规模实证（来源：ADHD × AI 的科学与研究前沿）。此外，ADHD 异质性高，单一工具难以覆盖所有亚型（来源：ADHD 的 AI 工具全景）。

另一个矛盾是：工具设计往往不考虑撤除机制。Goblin Tools、Saner.AI 等工具都鼓励长期使用，但长期依赖是否真的阻碍内在能力发展？目前没有明确答案。

## 今天就能试的行动

1. **建立“外部工作记忆”**：每次使用 Claude 前，先花 30 秒写一段“当前上下文”发给它，比如“我正在写一篇关于 ADHD 和 LLM 同构的文章，已经完成了引言部分，现在需要讨论工具证据”。这相当于给你的会话加一个“记忆前缀”。

2. **用 Goblin Tools 做任务分解**：当你面对一个模糊的大任务（如“准备下周的汇报”）时，先让 Goblin Tools 帮你拆解成 5-10 个小步骤，然后只做第一步。这相当于 Agent 工程中的“任务规划”。

3. **用 Saner.AI 或 Mem 做知识回忆**：把零散的笔记、想法、链接存入一个外部记忆工具，每天花 5 分钟回顾。这相当于给 LLM 加一个向量库。

4. **设计你的“撤除计划”**：选一个你依赖的工具，思考“如果有一天它消失了，我如何独立完成同样的任务？”这能帮你判断你是在使用脚手架，还是已经变成了拐杖。

Claude 之于 ADHD，就像外部记忆向量库之于 LLM——它能补上你缺失的调度层，但前提是你知道自己在用它补什么。如果你只是把它当作一个更聪明的对话者，那你可能正在浪费它真正的价值。

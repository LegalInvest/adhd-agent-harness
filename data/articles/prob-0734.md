# 如果把 ADHD 大脑当成生成核心，Claude 能不能当它的工具使用与认知卸载？

> 用 agent 工程的思路重新理解 Claude 对 ADHD 的真正价值。

## 问题：为什么 ADHD 和 AI Agent 工程师在同一个困境里？

你是一个 ADHD 患者。你的大脑充满创意，能瞬间生成无数点子，但面对“整理房间”“写周报”这类精确、重复的任务时，大脑像断了线——启动困难、工作记忆丢失、时间盲。你试过无数待办清单，但“计划本和便签用了一周就崩溃了”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

你是一个 Agentic Harness 工程师。你搭建的 AI 代理能写诗、能编程，但让它执行一个多步骤任务时，它要么忘记上下文，要么陷入无限循环。你需要给它套上“脚手架”——工具接口、记忆系统、验证循环——否则它就是一匹脱缰的野马。

这两个困境其实是同一个问题的两面：**一个强大的生成核心，缺乏可靠的执行调度层。** ADHD 大脑的智力/创造力与 LLM 的算力/语言能力同构，两者都高产但无序。而解决方案也同构：用外部工具提供执行功能、记忆管理和上下文控制——也就是给生成核心套上 harness。

## 核心观点：Claude 不是对话伙伴，而是你的认知 harness

主流观点把 Claude 当作“AI 助手”——你问它答，像聊天一样。但对 ADHD 用户来说，这种用法注定失败：它假设你能维持长时间专注、记住昨天的问题并执行计划（来源：Best AI Tools for ADHD Productivity in 2026）。而 ADHD 患者恰恰做不到这些。

真正的价值在于：**把 Claude 当成你的认知 harness，一个执行调度层。** 这要求你改变使用方式：

- **从对话到指令**：不要和 Claude 闲聊，而是给它一个明确的任务框架。例如，用单一指令触发复杂流程——这正是 Lex 的设计逻辑：“单一指令触发多步骤任务序列”（来源：Lex）。
- **从记忆到外部存储**：ADHD 的工作记忆会“掉落”上下文（来源：Best AI Tools for ADHD Productivity in 2026）。所以你需要像 Mem 或 Saner.AI 这样的工具，在会话之间记住工作内容（来源：Saner.AI）。Claude 本身是无状态的，但你可以通过 project 或自定义 prompt 让它“记住”关键信息——这相当于 agent 工程中的“上下文交付”和“记忆系统”（来源：Building AI Coding Agents for the Terminal）。
- **从生成到分解**：ADHD 患者面对复杂任务时容易 overwhelm。Goblin Tools 的 Magic ToDo 能自动将任务分解为小步骤（来源：Goblin Tools）。Claude 同样可以扮演这个角色：输入“写一份项目报告”，它输出“1. 列出大纲；2. 收集数据；3. 写引言……”——这就是 agent 工程中的“规划工件”（来源：GitHub - awesome-harness-engineering）。

## 证据：ADHD 侧与 LLM 侧的真实对应

### ADHD 侧的证据

- **任务启动**：Goblin Tools 通过分解任务降低启动焦虑，用户反馈“将压倒性的事情变成一系列不压倒性的事情”（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。
- **工作记忆**：Saner.AI 通过知识回忆减少搜索循环（来源：Saner.AI）；Otter.ai 实时转录减轻笔记负担（来源：ADHD 的 AI 工具全景）。
- **时间盲**：Goblin Tools 的魔法待办和 Motion 的智能调度帮助感知时间流逝（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。
- **执行功能**：传统工具失败是因为要求用户自己维持调度，而 AI 工具作为“外部执行功能层”能有效补偿（来源：外部执行功能层）。

### LLM/Agent 侧的证据

- **无状态与外部记忆**：LLM 本身是无状态的，需要外部脚手架提供上下文管理。Agent 工程中，用 Git 仓库存储项目上下文，通过 MCP 连接器访问外部数据（来源：外部执行功能层）。这直接对应 ADHD 用户需要的“第二大脑”。
- **规划与执行分离**：现代 AI 编码代理采用“分离规划与执行的双代理架构”，防止上下文膨胀和推理退化（来源：生成核心与调度层）。这对应 ADHD 患者需要将“计划”和“执行”分开，避免同时处理多个任务。
- **工具接口**：Harness 工程定义“设计围绕 AI 代理的脚手架——上下文交付、工具接口、规划工件、验证循环、记忆系统”（来源：外部执行功能层）。ADHD 用户也需要类似的脚手架：Claude 处理生成，外部工具（如日历、笔记）处理执行。

## 脚手架 vs 拐杖：关键边界

这里有一个核心争议：过度依赖 AI 工具是否会阻碍内在执行功能的发展？wiki 资料明确指出：“脚手架应设计为可逐步撤除的临时支撑”，但多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。

我的判断是：**Claude 作为 harness 是脚手架，但需要主动设计撤除路径。** 例如：

- **初期**：让 Claude 帮你分解每个任务，写详细提示。
- **中期**：只让 Claude 提供框架，你自己填充细节。
- **后期**：只让 Claude 做最终校对。

如果发现没有 Claude 就完全无法启动任何任务，那它就成了拐杖。真正的脚手架是“用进废退”——你用它来训练自己的执行功能，而不是替代它。

## 局限与争议

必须诚实指出：ADHD 大脑与 LLM 同构的命题“证据主要来自概念类比和工具案例，缺乏大规模实证”（来源：矛盾与存疑）。现有研究多为短期或个案，缺乏随机对照试验验证长期效果（来源：ADHD 的 AI 工具全景）。此外，ADHD 异质性高，单一工具难以覆盖所有亚型（来源：同上）。

另一个争议是多巴胺干预：虽然多个工具声称通过神经科学原理改善执行功能，但“基于多巴胺的干预效果存在争议”（来源：矛盾与存疑）。Claude 本身不直接调节多巴胺，但通过行为干预（如即时反馈、任务完成）间接改善动机。

## 今天就能试的 3 件事

1. **用 Claude 做任务分解**：下次面对复杂任务时，直接输入“请将‘整理房间’分解为 10 个具体步骤，每步不超过 5 分钟”。对比 Goblin Tools 的 Magic ToDo，感受外部 harness 的力量。
2. **建立外部记忆**：在 Claude 的 project 中创建一个“上下文文档”，每次对话前先更新它。这相当于 agent 工程的“上下文交付”，防止工作记忆掉落。
3. **设计撤除计划**：用 Claude 帮你设计一个“脚手架撤除计划”。例如，第一周完全依赖分解，第二周只让 Claude 提示关键步骤，第三周尝试自己写出步骤后让 Claude 验证。

## 结语

ADHD 大脑和 LLM 是同一类“高产但缺调度层”的生成核心。Claude 的价值不在于它有多聪明，而在于它能否成为你的认知 harness——提供执行功能、记忆管理和上下文控制。但记住：好的脚手架是让你最终能自己走路，而不是永远扶着它走。

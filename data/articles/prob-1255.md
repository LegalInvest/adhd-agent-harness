# Forest 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Forest：它到底补上了哪一层执行功能？

你有没有过这样的体验：打开 Forest 种下一棵树，告诉自己“专注25分钟”，结果第3分钟就开始刷手机，树枯死了，你更焦虑了。或者，你在 ChatGPT 里写了一个 prompt，它返回了一篇完美的文章，但你一个字都改不动——因为你觉得那是“它的”工作，不是你的。

这两个场景看似无关，实则共享同一个结构问题：**你把调度权完全外包了，却没有保留自己的监督回路。**

## 同构的困境：高产但缺调度层的生成核心

ADHD 大脑与 LLM 在结构上惊人地相似：两者都是强大的生成核心——发散思维、创意、多任务并行产出极高——但缺乏可靠的内置执行调度层（来源：外部执行功能层）。ADHD 的执行功能缺陷（任务启动困难、工作记忆不稳定、时间盲）与 LLM 的无状态、缺乏调度层在结构上同构（来源：ADHD 大脑与 LLM 的同构）。

这意味着，ADHD 人群需要的不是“更强大的工具”，而是**一个能补偿调度缺陷的外部脚手架**。同样，LLM/agent 需要的也不是更聪明的模型，而是一个能防止它偏离目标、陷入循环的 harness（来源：Building an AI Agent Harness from Scratch）。

Forest 之于 ADHD，就像 function calling 工具调用之于 LLM——它们都试图通过外部机制来补偿内部的调度缺陷。但问题在于：**很多人用错了。**

## 错在哪：把脚手架当拐杖

Forest 的核心机制是“种树-专注-得金币”。这本质上是一个**外部计时器+多巴胺奖励**系统。它假设：只要给你一个时间边界和正反馈，你就能专注。但 ADHD 大脑的问题不是不知道“该专注”，而是**无法在缺乏内在时间感知的情况下维持对任务的注意力**。时间盲（Time blindness）让 ADHD 患者“无法感知时间的流逝”，可能坐下后“真的感觉不到45分钟已经过去”（来源：时间盲）。如果 Forest 只是简单地倒计时，它就像一个只会喊“快跑”的教练，却不知道你腿抽筋了。

同样，function calling 在 LLM/agent 中扮演的角色是“让模型调用外部工具完成精确计算”。但如果 harness 设计得过于自动化——比如让 agent 自动调用所有工具、自动执行所有步骤——模型就会变成一个黑箱，人类失去了对关键决策点的控制。这正是 agent harness 工程中强调的：**人在回路（human-in-the-loop）监督是必需的**，要设置护栏、加入人工检查点（来源：人在回路与身体在场）。

## 正确的用法：保留调度权的脚手架

那么，Forest 和 function calling 的正确用法是什么？答案是：**把它们当作“定时 re-grounding”的信号，而不是“自动执行”的开关。**

以 Forest 为例，真正有效的用法不是“种下树然后强迫自己不动”，而是**在种树前先问自己：“接下来25分钟，我要完成哪一件具体的事？”** 然后，在 Forest 的倒计时中，每5分钟检查一次自己是否还在做那件事。这其实就是一种“外部调度器”——类似于孔子“吾日三省吾身”的日课。孔子一生与自己的冲动做斗争，他用的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省（来源：人物案例）。这对应到 LLM/agent 侧，就是**定时 re-grounding**：agent 每执行几步就暂停，向人类汇报进度、请求确认。

同样，function calling 的正确用法不是让 agent 自动调用所有工具，而是**让 agent 在调用工具前先输出“我打算调用 X 工具做 Y 事，请确认”**，然后等待人类批准。这就像 Focusmate 中的身体在场效应（body doubling）：有另一个人在场时，执行功能障碍被绕过，任务得以启动和维持（来源：人在回路与身体在场）。那个“另一个人”就是你的外部调度器——它不替你做事，但它确保你做事。

## 工具测评：哪些工具真正做到了“脚手架”？

- **Goblin Tools** 的 Magic ToDo 能自动将任务分解为子步骤，并允许用户调节“辣度”控制粒度（来源：Goblin Tools）。它的正确用法是：**用分解结果作为起点，然后手动调整顺序和优先级**——而不是直接照单全收。
- **Saner.AI** 通过本地记忆减少搜索循环，但它依赖初始设置（来源：Saner.AI）。它的正确用法是：**每天花5分钟整理笔记，让 AI 学习你的模式**——而不是把它当“第二大脑”后彻底放弃自己的记忆。
- **Lex** 允许单一指令触发多步骤任务，类似 agent scaffolding（来源：Lex）。它的正确用法是：**为每个指令设置一个“确认点”**，让 AI 在执行关键步骤前询问你。

这些工具的共同点是：它们都提供了**可调节的自动化程度**。而用户最容易犯的错误是：**直接把自动化拉到最大，然后抱怨工具没用。**

## 争议与局限

必须诚实指出，ADHD 大脑与 LLM 的同构性目前仍是一种理论类比，缺乏实证基础（来源：矛盾与存疑）。此外，过度依赖 AI 工具可能从脚手架变为永久拐杖，削弱内在执行功能（来源：拐杖与脚手架）。Forest 本身也存在歧义——它既指专注计时器，也指其他同名工具（来源：矛盾与存疑）。

更关键的是，**没有一种工具能替代你对自己的了解**。工具只是放大镜，如果你不知道自己要聚焦什么，放大镜只会让你更混乱。

## 今天就能试的行动

1. **用 Forest 时加一道工序**：种树前，在便签上写下“这25分钟我要完成______”，贴在屏幕边缘。每5分钟看一次，确认自己还在做那件事。
2. **给 ChatGPT 加一个人工检查点**：在 prompt 末尾加上“请先输出你的计划，等我确认后再执行”。
3. **试用 Goblin Tools 的 Magic ToDo**：将一个模糊任务（如“整理桌面”）丢进去，得到子步骤后，手动删掉其中3个你认为不必要的步骤——这能训练你主动调度。
4. **每天设置一个“re-grounding 闹钟”**：下午3点，问自己：“我现在在做的，是我今天最重要的那件事吗？”

记住：工具是脚手架，不是拐杖。脚手架帮你站稳，但路要你自己走。

# Claude 之于 ADHD，就像 human-in-the-loop 监督 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

## 问题：独自做事时，为什么那么容易放弃？

如果你有过这样的经历——明明能力足够，但一个人坐在电脑前，任务清单上的“写报告”三个字像一堵墙，你刷了半小时手机，最后什么也没做——那你可能已经体会到了 ADHD 大脑的核心困境：**启动困难**。这不是懒，而是执行功能缺陷：大脑无法自动将模糊目标分解为可执行步骤，缺乏外部问责时，任务启动和持续执行变得异常困难（来源：规划循环与任务分解）。ADHD 个体常面临工作记忆不足、时间盲和任务启动困难（来源：Goblin Tools）。

另一边，工程师们也在面对类似问题：单个 LLM 不足以可靠地完成多步骤任务、与业务工具交互或适应领域特定逻辑（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。一个纯粹的生成核心，无论多聪明，如果没有外部调度层来分解任务、验证中间结果、在关键节点引入人类判断，就会频繁脱轨。

**这两类问题的结构是相同的。** ADHD 大脑与 LLM/agent 都是“高产但缺执行调度层的生成核心”：ADHD 大脑能产生丰富的创意和联想，但缺乏规划与组织能力（来源：规划循环与任务分解）；LLM 能生成流畅文本，但缺乏内在规划能力，需要外部 harness 提供任务分解和验证机制（来源：规划循环与任务分解）。两边的解法也同构——都需要一个“人在回路”的监督层。

## 同构解法：脚手架，不是拐杖

ADHD 侧的有效工具，如 Goblin Tools 的 Magic ToDo，能将模糊任务（如“清理厨房”）自动分解为具体、可执行的子步骤，用户可调节“辣度”控制粒度（来源：Goblin Tools）。这种分解降低了启动门槛，利用小步骤的即时反馈维持注意力。Saner.AI 则通过增强知识回忆，减少搜索循环和标签切换，降低认知负荷（来源：Saner.AI）。Motion 能自动根据任务、会议和截止日期创建并动态调整每日计划，消除“下一步该做什么”的决策负担（来源：Motion）。

但这些工具只是脚手架，不是拐杖。**脚手架帮助你搭建自己的执行系统，拐杖则让你永远依赖外部支撑。** 比如，如果你用 Goblin Tools 分解任务后，仍然需要它每次提醒你下一步，那就是拐杖；如果你学会了它的分解逻辑，逐渐能自己拆分任务，那才是脚手架。

LLM/agent 侧同理。Agent 的 harness 应该提供规划循环和验证机制（如 AgentGen 通过环境与任务生成增强规划能力），同时保留 human-in-the-loop 的关键决策点。高风险 agent 需要人类监督，不是因为人类更聪明，而是因为人类能提供生成核心缺失的**身体在场效应**——即他人在场时，专注度和表现会提升（来源：矛盾与存疑）。Focusmate 通过虚拟视频实现这一效应，但 AI 工具是否能真正替代真实人际互动仍存争议（来源：矛盾与存疑）。

## 案例：张一鸣的 harness 系统

张一鸣，字节跳动创始人，被广泛认为具有 ADHD 特质：思维极度发散，永远在研究新事物，不接受采访不混圈子。他的 harness 系统是“延迟满足感 + OKR 管理系统 + Context not Control”。OKR 相当于一个外部调度器，将宏大目标分解为季度关键结果，定期复盘。Context not Control 则像 human-in-the-loop 监督——他信任团队自主决策，但通过信息和数据保持可见性，在关键节点介入。这套系统让字节跳动在全球扩张中保持敏捷，同时避免了创始人过度集权的风险。

这与 LLM agent 的 harness 同构：OKR 相当于任务分解与规划循环，Context not Control 相当于在关键决策点引入人类判断，而延迟满足感则相当于长期目标导向的奖励函数。张一鸣的成功不是因为他没有 ADHD，而是因为他为自己搭建了正确的脚手架。

## 局限与争议

必须诚实指出，ADHD 大脑与 LLM 的同构性目前仅是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。不同页面在表述时存在不一致，有时将其作为既定事实，有时作为假设。此外，工具宣称的证据不足：Motion 缺乏独立临床验证（来源：矛盾与存疑），Goblin Tools 虽被广泛推荐但也缺乏大规模随机对照试验。依赖风险也需警惕：过度依赖 AI 工具可能削弱自主执行能力，形成新的认知拐杖（来源：矛盾与存疑）。最后，身体在场效应与 AI 替代的矛盾——AI 能否真正替代真实人际互动仍存争议（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个你拖延已久的任务**——比如“整理书桌”。调节辣度到你觉得“可以立刻开始”的程度，然后完成第一个子步骤。
2. **在 Claude 中创建一个 Project**，把你本周要完成的任务描述给它，要求它分解为步骤并设定检查点。每完成一个步骤，回来与 Claude 复盘——这模拟了 human-in-the-loop 监督。
3. **尝试 Saner.AI 的通用收件箱**——将邮件、文档或日历中的待办事项集中到一处，减少搜索循环。观察一周内你的“标签切换”次数是否下降。
4. **反思你的工具使用**：你是在用工具搭建自己的执行系统，还是让工具替你做决定？如果是后者，尝试减少一个自动化功能，手动完成一次规划。

Claude 之于 ADHD，就像 human-in-the-loop 监督之于 LLM agent——它补上了缺失的执行调度层，但前提是你把它当作脚手架，而不是拐杖。

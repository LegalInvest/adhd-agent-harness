# ADHD 想把不擅长精确、重复、状态维护类任务外包给 ChatGPT，该怎么设计这套「人 + AI」的回路？

> 把 ChatGPT 接进 ADHD 工作流：哪些该交给它，哪些必须自己留着？

## 问题：你为什么不擅长精确、重复、状态维护？

如果你有 ADHD，你一定熟悉这种场景：脑子里同时冒出三个绝妙想法，但连把其中一个写成大纲都启动不了；你知道应该回那封邮件、填那张表、更新那个状态，但身体像被钉在椅子上。这不是懒。这是 ADHD 大脑的典型特征——一个高产的生成核心，配了一个不稳定的调度层（来源：ADHD 的 AI 工具全景）。想法多、联想快、超聚焦时能一口气干十小时，但一旦涉及精确计算、重复步骤、维持状态（比如记账、排程、跟进流程），系统就崩溃。

工程师们，你们在做 Agentic Harness 时是不是也遇到类似问题？LLM 能写出漂亮的代码片段，但让它跑一个多步工作流，它要么忘记上下文，要么陷入循环，要么输出格式错误。你们不得不给它套上 function calling、规划器、记忆模块——一个“外部调度层”。

这就是本文的核心判断：**ADHD 大脑与 LLM 是同一类系统，两者的解法结构同构**。你不是在“治疗”ADHD，而是在搭建一个与你大脑互补的外部执行功能层（来源：ADHD 的 AI 工具全景）。而 ChatGPT 就是这个层里的核心组件之一。问题是：怎么设计？

## 同构解法：从孔子和苏轼的 harness 学起

先看两个真实案例。孔子，ADHD 特质明显：身高近1米9，精力旺盛到周游列国14年；冲动爱骂人，骂宰予“朽木不可雕”；对音乐超专注到“三月不知肉味”，但对种地等俗事零耐心；思维跳跃，著作全是场景化语录。他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。苏轼，兴趣极广：诗词文书画美食治水炼丹；一辈子被贬，到哪都能适应，但说话不过脑子得罪人；对无聊行政零耐心，搞工程效率极高。他的 harness 是豁达——用美食、写作、交朋友转移注意力，每日读书写字不因挫折停止创作。

这两个 harness 的核心是什么？是**外部化调度**。孔子把决策边界外包给“礼”，苏轼把注意力转移外包给日常创作。它们的结构同构于 LLM 的 agent harness：礼 ≈ 系统提示词（约束行为边界），日课 ≈ 定时 re-grounding（每日反省对应定期 context refresh），秘书 ≈ 外部调度器（把任务拆解并逐步执行）。ADHD 大脑需要的，正是这样一套“人+AI”的回路。

## 设计你的“人+AI”回路：四步法

基于同构视角，你的回路应该包含四个层次，对应 ADHD 的四个核心痛点。

**1. 任务拆解与启动：用 Goblin Tools 的 Magic ToDo 打破“启动瘫痪”**

ADHD 最怕模糊的大任务。“清理厨房”会变成“我要把整个房子翻新”，然后什么都不做。Goblin Tools 的 Magic ToDo 能接受模糊任务，自动分解为具体子步骤，且可调节“辣度”控制粒度（来源：Goblin Tools）。这就是你的“任务拆解器”。LLM 侧对应的是 agent 的规划模块——把“写一份市场报告”拆成“收集数据→分析趋势→撰写大纲→填充内容”。两者做的同一件事：把高层的生成意图转化为可执行的步骤序列。

**2. 外部记忆与知识管理：用 Saner.AI 补偿工作记忆**

ADHD 的工作记忆像筛子——刚想到的灵感，转头就忘；上周讨论的结论，今天怎么也想不起来。Saner.AI 通过本地记忆和知识回忆，减少搜索循环和标签切换（来源：Saner.AI）。这就是你的“外部第二大脑”。LLM 侧对应的是 RAG（检索增强生成）或记忆模块——让 agent 能记住之前的对话和用户偏好。没有这个，agent 每轮对话都像失忆症患者；没有这个，你每天都要重新发明轮子。

**3. 对话式执行与结构化推理：用 ChatGPT 作为“外部执行功能”**

ChatGPT 被成人 ADHD 用户用作“外部执行功能”工具（来源：ADHD 的 AI 工具全景）。你可以直接对它说：“我有个想法，帮我列出执行步骤，每步不超过15分钟，并告诉我第一步现在就能做什么。”它不会抱怨，不会评判，不会说“你怎么又忘了”。LLM 侧对应的是 agent 的对话式推理能力——通过 function calling 调用工具，逐步解决复杂问题。两者共享一个模式：用自然语言对话驱动结构化执行。

**4. 捕获与反思：用 Lex 的“单一指令触发多步骤”降低认知负荷**

ADHD 大脑对多步骤指令容易“掉线”。Lex 允许你通过单一指令触发复杂、多步骤的任务序列（来源：Lex）。比如你对 Lex 说“帮我安排下周的会议”，它会自动查日历、发邀请、设提醒。这就是你的“一键调度员”。LLM 侧对应的是 agent 的编排能力——一个指令触发一个工作流。两者都解决了同一个问题：把多步操作压缩成一步，降低启动门槛。

## 脚手架 vs 拐杖：你必须留住的“人”的部分

这里有一个必须诚实的矛盾：AI 工具是“外部执行功能层”还是“拐杖”？（来源：矛盾与存疑）过度依赖可能导致能力退化。孔子和苏轼的 harness 有一个共同点：**核心决策和创造力永远留给自己**。孔子自己决定“克己”的方向，苏轼自己选择写什么诗。AI 可以帮你拆任务、记信息、触发动作，但不能替你判断什么对你重要，不能替你感受“这个想法让我兴奋”，不能替你体验超聚焦带来的心流。

**你的回路中，必须留住的“人”的部分是：**
- **价值判断**：哪些任务值得做？AI 不知道你的长期目标。
- **创造力与直觉**：灵感从哪来？AI 只能组合已有信息。
- **超聚焦的方向选择**：AI 可以引导超聚焦，但选择聚焦在什么上，必须是你自己（来源：超聚焦）。
- **反思与意义构建**：每天结束时的“三省吾身”，AI 可以帮你记录，但反思的主体是你。

所以，这不是把一切外包给 ChatGPT，而是**把你最不擅长的精确、重复、状态维护任务外包，同时把你最擅长的生成、判断、创造留在自己手里**。这就是“脚手架”与“拐杖”的边界。

## 今天就能试的 3 个行动

1. **用 Goblin Tools 分解一个你拖延的任务**：打开 Goblin Tools 的 Magic ToDo，输入“整理书桌”或“写周报”，调节辣度到 3，看它分解成什么。把结果复制到你的待办清单，逐个勾选。

2. **给 ChatGPT 设一个“外部执行功能”提示词**：在 ChatGPT 中新建一个自定义指令，内容可以是：“当我给你一个任务时，先帮我拆成不超过15分钟的子步骤，然后问我现在第一步是什么。完成后提醒我下一步。”

3. **用 Saner.AI 捕获一个“灵光一现”**：下次想到一个点子，立刻用 Saner.AI 的语音或文字笔记捕获，不加整理。之后用它的搜索功能找回——你会发现它比你的大脑可靠。

## 局限与争议

本文的框架基于“ADHD 大脑与 LLM 同构”这一理论，但目前该论点多为类比推理，缺乏直接的神经科学或计算机科学证据（来源：矛盾与存疑）。另外，AI 工具的长期效果缺乏纵向研究，部分用户反馈学习使用工具本身增加认知负荷。超聚焦是否应被引导而非打断，也因人而异（来源：矛盾与存疑）。因此，你的回路需要个性化调整——没有万能方案。

最后，记住孔子和苏轼的教训：harness 不是一次建成的，是一辈子迭代的。你的“人+AI”回路也一样，从今天开始，从一个小步骤开始。

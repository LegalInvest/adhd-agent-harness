# 如果把 ADHD 大脑当成生成核心，RescueTime 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 RescueTime 对 ADHD 的真正价值。

如果你是一个ADHDer，你一定经历过这样的场景：打开电脑准备写报告，十分钟后却发现自己正在刷社交媒体，完全不记得怎么走到这一步。工作记忆像漏水的桶，计划刚定好就蒸发。如果你是一个做Agent Harness的工程师，你也一定熟悉这样的困境：LLM刚在上一轮对话中理解了任务上下文，换一个会话就忘得一干二净，必须靠向量库或外部记忆来“续命”。

这两种困境，本质上是同一个问题：一个强大的生成核心，缺少可靠的执行调度层和持久化上下文。ADHD大脑和LLM，在这个意义上同构——都是“高产但缺执行调度层的生成核心”（来源：ADHD的AI工具全景）。

那么，如果把ADHD大脑当成生成核心，RescueTime——一个老牌的时间追踪工具——能不能充当它的无状态与外部记忆？

## 无状态与外部记忆：ADHD与LLM的共同痛点

ADHD侧最核心的执行功能缺陷是工作记忆的“掉落”。你明明知道该做什么，但下一秒就忘了。最新研究揭示，ADHD患者的工作记忆容量可能正常甚至超常，但自上而下的控制和调节能力不足，呈现“强记忆、弱控制”的认知剖面（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。注意力分散是自注意力架构的固有属性，随着任务负载增大，注意力分数熵增加，导致容量限制（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。

LLM侧呢？LLM本身是无状态、仅生成文本的核心，需要外部“脚手架”和“harness”来提供工具接口、上下文管理、安全执行等编排能力（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。Harness工程的核心就是设计围绕AI代理的脚手架——上下文交付、工具接口、规划工件、验证循环、记忆系统和沙箱（来源：GitHub - ai-boost/awesome-harness-engineering）。

两边都缺一个外部记忆系统来持久化上下文，缺一个调度层来管理执行流。

## RescueTime：作为外部记忆与调度层

RescueTime 的核心功能是自动记录你在电脑上的时间分配，生成报告，并允许你设置目标。它不像 Goblin Tools 那样帮你分解任务（来源：Goblin Tools），也不像 Saner.AI 那样帮你回忆知识（来源：Saner.AI）。它的独特价值在于：它提供了一种无状态的、结构化的外部记忆。

所谓“无状态”，是指 RescueTime 不依赖于你当前的情绪或注意力状态——它默默运行，记录一切。对于ADHD大脑，这意味着你不需要记住自己过去做了什么；RescueTime 替你记住了。你可以随时回看，就像LLM通过向量数据库检索历史对话一样。

所谓“外部记忆”，是指 RescueTime 将你的时间使用历史持久化，形成一种“时间记忆”。这直接补偿了工作记忆的缺陷。ADHD患者常面临时间盲（来源：时间盲），难以感知时间的流逝和分配。RescueTime 的可视化报告提供了外部时间线索，让你“看到”时间去了哪里。

从Agent工程的角度看，RescueTime 相当于一个日志系统+监控仪表盘。LLM Agent 需要这样的系统来追踪自己的行为、诊断失败模式、优化调度策略。ADHD大脑同样需要。

## 脚手架 vs 拐杖：RescueTime 的边界

但 RescueTime 不是万能的。它的局限恰恰在于它只记录，不干预。它不能像 Goblin Tools 那样自动帮你启动任务（来源：Goblin Tools），也不能像 Lex 那样通过单一指令触发多步骤流程（来源：Lex）。它更像一个“后视镜”，而不是“导航仪”。

这里就涉及“脚手架 vs 拐杖”的辩证关系。AI工具应作为脚手架促进能力发展，而非永久拐杖（来源：拐杖与脚手架）。RescueTime 的被动记录性质，使其更接近脚手架：它提供数据，但决策和行动还得靠你自己。过度依赖 RescueTime 的风险较低，因为它不替你做事；但如果你只看报告而不反思，它就成了摆设。

另一方面，RescueTime 的“无状态”特性也意味着它不提供实时反馈。对于需要即时外部反馈的ADHD用户（来源：身体在场效应），像 Otter.ai 的实时转录或 Reflect 的结构化反思可能更有效（来源：Otter.ai、Reflect）。

## 证据与争议

目前 RescueTime 针对 ADHD 的效果缺乏独立临床研究。现有证据主要来自用户报告和工具设计逻辑的一致性。多个页面指出，AI工具作为外部执行功能层的有效性证据不足，主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑）。

此外，个体差异很大。有些ADHD用户发现 RescueTime 的自动追踪减少了认知负荷，另一些人则觉得它增加了焦虑——因为看到了自己浪费了多少时间。工具设计者声称是“脚手架”，但实际使用中可能沦为“拐杖”或“压力源”（来源：矛盾与存疑）。

## 核心观点

我的判断是：RescueTime 对 ADHD 的真正价值，不在于它帮你“管理时间”，而在于它提供了一个结构化的外部记忆系统，补偿了工作记忆的“掉落”。它通过无状态的自动记录，让你能够“回放”自己的行为，从而获得元认知——知道自己实际做了什么，而不是你以为自己做了什么。

从Agent工程的角度看，RescueTime 相当于一个行为日志+诊断工具。任何 Agent 都需要这样的基础设施才能迭代优化。ADHD大脑同样需要。

但 RescueTime 不能替代任务启动和实时反馈工具。它应该与 Goblin Tools（任务分解）、Saner.AI（知识回忆）、Lex（单指令多步骤）等配合使用，形成一套完整的“外部执行功能层”。

## 今天就能试的行动

1. **安装 RescueTime 并运行一周**：不要设置任何目标，只是让它记录。一周后查看报告，找出三个最大的时间黑洞。这相当于给你的“生成核心”加了一个日志系统。
2. **每天花5分钟回看**：在一天结束或开始时，打开 RescueTime 的每日报告。问自己：实际时间分配与意图一致吗？这提供了外部记忆回放，增强元认知。
3. **结合 Goblin Tools 分解一个任务**：用 RescueTime 识别出你经常拖延的任务，然后打开 Goblin Tools 的 Magic ToDo 将其分解为小步骤。这相当于用外部调度层启动执行。
4. **设置一个“意图锚点”**：在 RescueTime 中设置一个每日目标（如“写作2小时”），并在开始工作前大声说出意图。这利用无状态记录+有意识启动，形成闭环。

RescueTime 不是完美的工具，但它提供了一个重要的思路：对于“高产但缺执行调度层”的生成核心，外部记忆和日志系统是基础设施。无论是ADHD大脑还是LLM Agent，都需要这样的脚手架才能持续产出。

# Perplexity 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Perplexity：它到底补上了哪一层执行功能？

## 问题：为什么 Perplexity 搜完答案，你还是不动？

你打开 Perplexity，输入“如何开始写论文”，得到一份漂亮的步骤清单。然后你关掉页面，继续刷手机。这不是你懒——你的 ADHD 大脑和 LLM 共享同一个结构性缺陷：**生成能力超强，但缺乏可靠的执行调度层**（来源：ADHD 大脑与 LLM 的同构）。Perplexity 帮你生成了答案，但它没有帮你把答案变成行动。就像给 LLM 一个复杂任务却不给它 planner-executor 循环——模型会输出一堆正确但无用的 token，卡在第一步。

## 同构：ADHD 大脑 ≈ 没有 planner 的 LLM

ADHD 的执行功能缺陷——任务启动困难、不会拆解、时间盲——本质上是一个“生成核心”缺少调度层的问题。你的大脑能产生无数创意（生成），但无法可靠地规划顺序、分配资源、监控进度（执行）。LLM 也一样：给它一个长任务，它会在上下文窗口里迷失，输出冗长或偏离目标的回答。两者的解法在结构上同构：**给生成核心套一个外部的 planner-executor 循环**（来源：外部执行功能层）。

在 LLM/Agent 工程中，这叫 Agentic Harness——一个规划器（planner）将大目标分解为子任务，执行器（executor）依次完成，并循环反馈。在 ADHD 时间管理中，这叫“脚手架”——工具帮你把“写论文”拆成“打开文档→写 3 句→休息 5 分钟”，并在每一步提供外部触发。

## 实证：工具怎么补上执行层？

### 任务分解：Goblin Tools 的“魔法待办”
Goblin Tools 的“魔法待办”功能可以将“整理房间”自动拆成“把书放回书架→擦桌子→吸尘”等小步骤（来源：任务启动）。这直接对应 LLM 的 planner 模块：将“写一篇 2000 字文章”拆成“定主题→列大纲→写引言→写正文→写结论→修改”。没有这个分解，ADHD 大脑和 LLM 都会卡在“启动”阶段。

### 动态调度：Motion 和 Reclaim.ai
Motion 自动根据截止日期和可用时间重建日程，消除“下一步做什么”的决策负担（来源：Motion）。Reclaim.ai 则保护深度工作时间块，防止会议侵占（来源：Reclaim.ai）。这对应 Agent 的 executor 循环：Motion 像是一个抢占式调度器，Reclaim.ai 像是优先级队列——它们持续监控环境并重新分配资源。对于 ADHD 用户，这直接补偿了时间盲（来源：时间盲）和工作记忆缺陷（来源：工作记忆）。

### 视觉化时间：Tiimo
Tiimo 将时间转化为可见的彩色时间轴（来源：Tiimo），让时间流逝变得可感知。这对应 LLM 的进度条或 token 计数——没有这些外部信号，模型（和大脑）无法判断自己处于任务的哪个阶段。

### 证据与局限
以上工具的有效性主要基于用户报告和设计逻辑，缺乏随机对照试验（来源：AI 与 ADHD 的时间管理）。更重要的是，这些工具可能成为“永久拐杖”：长期依赖外部调度可能削弱内在时间感知和规划能力（来源：矛盾与存疑）。关键区别在于：**脚手架是暂时支撑，最终要撤掉；拐杖是永久替代**。目前没有工具明确设计“撤出”机制。

## 核心观点：Perplexity 用错了——它只做了答案生成，没做任务分解

Perplexity 本质上是一个增强版搜索引擎：它帮你找到信息，但不会帮你把信息组织成行动序列。对于 ADHD 用户，这就像给一个没有 planner 的 LLM 输入一堆高质量资料——模型能总结出要点，但无法自动生成执行计划。**Perplexity 之于 ADHD，就像 RAG 之于 LLM：它扩展了知识，但没有解决调度问题。**

正确的用法是：**把 Perplexity 当作 planner 的输入，而不是 executor 的输出**。具体来说：

1. **用 Perplexity 生成任务分解模板**：问“写论文的第一步是什么？第二步？”然后把答案输入到 Goblin Tools 或 Todoist 中，形成可执行的步骤列表。
2. **结合 Motion 或 Reclaim.ai 调度这些步骤**：让工具自动分配时间块，而不是手动安排。
3. **用 Tiimo 可视化时间**：在执行过程中，用视觉反馈监控进度。

这样，Perplexity 就从一个“答案机器”变成了“规划顾问”——它帮你拆解任务，但执行仍由外部调度工具完成。这对应 Agent 架构中的 planner + executor 分离：planner（Perplexity）生成子任务，executor（Motion/Tiimo）执行并反馈。

## 争议与边界

同构理论仍属类比，缺乏神经科学与计算机科学的交叉验证（来源：矛盾与存疑）。此外，ADHD 亚型（注意力缺陷 vs. 多动冲动）对工具响应不同，现有工具多面向注意力缺陷型（来源：AI 与 ADHD 的时间管理）。**对于多动冲动型，可能需要更多“身体在场”干预（如实体计时器），而非纯数字工具。**

另一个风险是：工具可能强化“自动完成”的依赖，削弱内在动机。多巴胺机制（来源：多巴胺）解释了一部分——工具通过即时反馈提供多巴胺奖励，但长期看，用户可能只对工具提示有反应，而对内在时间线索麻木。目前缺乏长期追踪研究（来源：矛盾与存疑）。

## 今天就能试的行动

1. **把 Perplexity 当 planner**：下次遇到复杂任务，先问 Perplexity“这个任务的标准步骤是什么”，然后把步骤复制到 Todoist 或 Goblin Tools 中，每完成一步打勾。
2. **用 Motion 或 Reclaim.ai 保护一个“深度工作块”**：设置 25 分钟无打扰时间，关闭所有通知。如果觉得设置复杂，先用 Tiimo 的视觉时间轴手动规划一个 25 分钟块。
3. **尝试“身体在场”启动法**：在开始任务前，把手机放在另一个房间，打开电脑上的 Tiimo 时间轴，然后立刻做第一步（哪怕只是打开文档）。这利用物理环境降低启动阻力（来源：身体在场效应）。
4. **每周评估一次依赖程度**：问自己“如果明天没有这些工具，我还能规划时间吗？”如果答案是否定的，尝试减少一个工具的依赖，比如用纸质清单代替数字清单一天。

ADHD 大脑和 LLM 都不是“坏”的——它们只是缺一个 planner-executor 循环。工具不是替代你的执行功能，而是给它一个外部框架。用对了，它是脚手架；用错了，它是拐杖。区别在于：你是否在主动参与拆解与调度，还是只被动接收答案。

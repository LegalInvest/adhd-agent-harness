# Otter.ai 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Otter.ai：它到底补上了哪一层执行功能？

你买 Otter.ai，是希望它替你记笔记，好让你在会议中自由思考。结果呢？你录了十场会议，一次也没回看。不是懒——是 Otter 只解决了“记录”这个浅层问题，却放大了“回顾”这个深层执行障碍。

这背后藏着一个更本质的错位：**我们总把 AI 工具当成替身，其实它应该是个调度员。** 而理解这个错位的最佳视角，是看 ADHD 大脑与 LLM 的结构同构。

## 同构：高产但缺调度层的生成核心

ADHD 大脑与 LLM 共享一个核心困境：**两者都是强大的生成核心，却缺少精准的执行调度层。**

ADHD 侧的证据来自认知负荷研究。用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中模型表现正常，但随序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突（来源：Deficient Executive Control in Transformer Attention）。这与 ADHD 执行功能缺陷的核心标志一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃。

LLM/Agent 侧的对应更直观：一个没有 function calling 的 LLM，只能靠内部推理完成所有计算。当它需要精确计算、查询数据库或维持状态时，要么幻觉，要么崩溃。而 function calling 允许 LLM 将“精确计算”外包给外部工具——就像 ADHD 大脑需要一个外部系统来管理“精确、重复、状态维护”类任务。

**Otter.ai 之于 ADHD，就像 function calling 之于 LLM。** 它补上了执行功能层中“精确记录与状态维护”这一环。

## Otter.ai 到底补上了哪一层执行功能？

ADHD 的执行功能缺陷包括工作记忆、任务启动、时间盲和上下文控制（来源：ADHD 的 AI 工具全景）。Otter.ai 的实时转录和自动笔记，直接补偿的是**工作记忆**和**上下文维护**：
- 会议中，你不必同时听、想、记，因为 Otter 接管了“精确记录”这个重复性任务，让你能专注于“生成想法”。
- 会后，Otter 提供可搜索的文本记录，相当于为你的工作记忆提供了一个持久化上下文——类似 Mem 和 Saner.AI 的“AI 第二大脑”（来源：Saner.AI）。

但问题在于：**Otter 只完成了“记录”这一层，却没有帮你完成“回顾与执行”这一层。** 这正是“用错了”的根源。

## 脚手架 vs 拐杖：边界在哪？

AI 工具应作为脚手架促进能力发展，而非永久拐杖（来源：拐杖与脚手架）。Otter.ai 如果只是让你“不用记笔记”，你可能永远学不会主动筛选重点——这是拐杖。但如果它帮你“在会议中解放大脑，会后自动生成待办并推送到你的任务管理器中”，这就是脚手架。

同样，function calling 如果只是让 LLM 把所有计算都外包出去，模型自身的推理能力会萎缩。好的 agent 设计会让 LLM 在关键决策上保持自主，只在“精确计算”或“状态查询”时才调用工具。

**关键区别在于：工具是补偿了你的调度缺陷，还是替代了你的生成能力。**

## 证据与局限

目前支持 Otter.ai 对 ADHD 有效的证据主要来自用户报告和工具设计逻辑（来源：ADHD 的 AI 工具全景）。用户反馈称实时转录减轻了笔记负担，但过度依赖可能削弱主动记笔记的能力（来源：矛盾与存疑）。

更广泛的矛盾在于：AI 工具作为外部执行功能层的有效性证据不足，现有证据以定性为主，缺乏大规模对照实验（来源：矛盾与存疑）。此外，个体差异导致效果不一——有些用户发现 Otter 的自动摘要反而增加了信息过载。

## 今天就能试的行动

1. **设置“回顾触发器”**：在 Otter 中创建自动规则，每次会议结束后，让 AI 提取 3 个关键行动点并发送到你的待办清单（如 Todoist）。这样 Otter 就从记录工具变成了执行启动器。
2. **用 Otter 训练主动笔记**：会议中只记录自己的疑问和想法，会后用 Otter 转录补全细节。两周后对比自己的笔记质量是否提升。
3. **同构迁移**：如果你是 agent 工程师，检查你的 function calling 是否过度外包。给 LLM 保留至少 20% 的自主推理空间，避免模型退化。
4. **组合使用**：将 Otter 与 Goblin Tools 或 Lex 联动——Otter 输出会议记录，Goblin Tools 将其分解为可执行步骤（来源：Goblin Tools）。

Otter.ai 不是替身，是调度员。用对位置，它才能从拐杖变成脚手架。

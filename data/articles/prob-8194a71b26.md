# 为什么用 RescueTime 治 ADHD 的注意力涣散，和给 agent 套 上下文工程 是一回事？

> RescueTime 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

你早上 9 点打开文档，打算在两小时内写完一份方案。等回过神来，已经刷了四十分钟新闻，还打开了三个无关标签页。你骂自己“没自控力”，但问题可能不在意志力——而在“上下文”失控。

与此同时，一个 LLM agent 接到一条复杂指令：调研、写报告、发邮件。如果缺少上下文管理，随着 token 变长，它会开始重复、跑偏、调用错误工具，甚至“幻觉”出不存在的结果。

两个场景崩溃点相同：生成核心很强，但**执行调度层**靠不住。（来源：AI 与 ADHD 的专注力）

## 1. ADHD 与 LLM 的崩溃点：同一个调度层缺口

ADHD 常被误解为“不能专注”，更准确的说法是“无法稳定地选择并维持注意上下文”。工作记忆不稳定、时间感知扭曲（时间盲）、环境新刺激随时把任务上下文切走，导致任务碎片化。（来源：AI 与 ADHD 的专注力；上下文工程）

# Tiimo 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Tiimo：它到底补上了哪一层执行功能？

如果你每天盯着满屏待办，知道“该做”却动不了，你不是懒——你的生成核心缺一个调度层。这一点，做过 Agentic Harness 的工程师会立刻共鸣：LLM 能写代码、能推理、能生成，但一遇到长任务，立刻在无限循环和工具调用里“漂移”。ADHD 大脑与 LLM 的结构同构正在这里：二者都是高产的生成核心，却缺乏可靠的内置执行调度层（来源：AI 与 ADHD 的时间管理）。时间盲、任务启动困难、目标漂移，本质上是同一个 bug：没有外部机制把“宏大意图”切分成“此刻可见的下一步”。

Tiimo 被归类为视觉化日常规划应用，专为 ADHD 与自闭症成人设计，用结构、颜色、视觉惯例帮助用户组织生活（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。它与 Motion、Reclaim.ai 同属“自适应规划器”，能根据用户实际完成情况自动重新安排日程（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。但它的特殊之处在于：它把时间变成可见元素，用动态、视觉化的流程替代静态待办清单，让用户感到更稳定、更掌控时间，并减少无结构时段带来的焦虑（来源：The 12 Best Apps for ADHD in 2026: A Guide to Finding What ...）。对工程师来说，这就像一个轻量级的 planner-executor loop：它不把任务只存在 backlog 里，而是把任务块渲染到时间线上，并持续根据真实执行状态重排。

## 同构：ADHD 与 LLM 共用一套“脚手架语法”

从同构视角看，ADHD 大脑的生成核心（创意、联想、高兴趣驱动）与 LLM 的语言生成能力类似，但双方的调度层都不可靠：ADHD 在计划、组织、工作记忆、冲动抑制上存在缺陷；LLM 则在状态保持、真实规划、上下文控制上存在缺陷（来源：执行功能、工作记忆）。所以，为 ADHD 搭建“执行功能支架”与为 LLM 搭建“agent scaffolding”是同一类工程（来源：拐杖与脚手架）。

这个映射解释了两类工具设计的共同重点：外部记忆、重锚定、人在回路。ADHD 大脑极易发生目标漂移：注意力被无关刺激捕获，或过度聚焦细节而忽略整体目标（来源：重锚定与目标漂移）。外部工具通过提醒和通知提供重锚定，把用户拉回原定任务。传统提醒系统的关键缺陷是：设置提醒本身就是一项执行功能任务，而 ADHD 个体恰恰在这方面薄弱；更理想的方案是无需主动设置的提醒（来源：AI Assistant for ADHD: Finally, a Productivity Tool That Requires Less...）。LLM 侧同样如此：Agent harness 通过 human-in-the-loop 监督、护栏、人工检查点、防止无限循环的机制，把生成核心从“漂移”里拉回（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community；GitHub - ai-boost/awesome-harness-engineering）。模型本身强大，可靠性来自架构加护栏（来源：AI Agents in 2026: Tools, Memory, Evals, and Guardrails）。

Tiimo 的彩色时间块、跨设备视觉一致性、自动重排，正是在做“人在回路的轻量调度器”：它不要求 ADHD 用户自己记住“下一步该做什么”，而是把“现在该做什么、做多久、何时切换”显式化，从而缓解时间盲和任务启动困难（来源：AI 与 ADHD 的时间管理）。这和 agent harness 里的“形式化规划与护栏，使代理输出真正有用且正确”完全同构（来源：What is an agent harness in the context of large-language models?）。

## 历史案例：孔子的“礼”是最早的外部 harness

要理解 Tiimo 补的是哪一层，可以回到更早的 harness 原型。孔子的 ADHD 特质在史料中有清晰痕迹：身高近一米九，精力旺盛，周游列国十四年；对音乐超专注到“三月不知肉味”，对种地等俗事零耐心；思维跳跃，导致《论语》全是场景化语录，没有系统著作；冲动爱骂人，见南子急得对天发誓，骂宰予“朽木不可雕”。他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日“三省吾身”，到七十岁才“从心所欲不逾矩”。

“礼”不是道德说教，而是孔子的外部调度器：它把宏大目标（仁、政、教）切分为可重复的具体动作（站、坐、应答、进食、祭祀）。每日反省则是人工检查点，相当于 agent 的 telemetry 与 eval loop。这与 LLM harness 里的“护栏 + 重锚定 + 人在回路”完全同构：日课对应定时 re-grounding，礼对应行为护栏，三省对应日志与复盘。孔子不是被“礼”驯服，而是借助“礼”这个外部脚手架，让自己高能量的生成核心稳定输出。Tiimo 的彩色时间块和自动提醒，就是当代的“礼”：不是用来约束你，而是把你从漂移中拉回来。

## 但有人用错了：把 harness 当拐杖

Tiimo 被用错的最典型方式，是把它当作“好看一点的待办清单”。ADHD 用户常把任务一股脑丢进 app，然后看着越滚越长的列表更焦虑，最后弃用。问题在于：Tiimo 的核心价值不是“收集任务”，而是“把任务渲染到时间线上并动态重排”。如果你只把它当 inbox，等于把 agent 的 planner  bypass 掉，直接让 executor 面对一堆未分解的意图，这必然导致失败。

另一个常见误用是过度依赖提醒而不培养自我监控。AI 工具作为外部执行功能层可以补偿缺陷，但长期过度依赖可能导致能力退化，成为永久拐杖（来源：矛盾与存疑）。这一点在 LLM agent 工程里也常见：如果 harness 只是无限兜底，模型永远学不会自检；理想的做法是逐步把护栏内化为模型能力。Tiimo 的正确用法应该是：从“它帮我决定现在做什么”过渡到“我借它看见自己的模式”。

还需要诚实指出局限：ADHD 大脑与 LLM 同构的论点目前多为类比推理，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。Motion 等产品虽用户反馈积极，但缺乏独立临床证据，初始设置成本高（来源：矛盾与存疑）。AI 工具是否能真正减轻认知负荷也因人而异，有人反而因为学习使用工具本身而增加认知负荷（来源：矛盾与存疑）。

## 今天就能试的 4 条行动

1. 把 Tiimo 当“可视化执行层”，不是“待办仓库”。每次只放接下来 3 小时内要做的事，并给每个任务一个明确的时长块和视觉颜色；如果超过 3 小时没做，删掉或重新分解，而不是让它变成 backlog 垃圾。这对应 agent 的 planner 只处理当前可执行计划，不累积未分解意图。

2. 用“动作句”而非“结果句”描述任务。把“写报告”改成“打开文档，写出第一段 150 字”。Tiimo 的强项是把时间可视化，但如果你输入的仍是模糊目标，视觉化也救不了你。这和 LLM 的 prompt engineering 一样：输入越清晰，executor 越可靠。

3. 设置一个“人工检查点”日课。每晚花 3 分钟看 Tiimo 的完成记录，只回答两个问题：今天被什么拉走了？明天的第一个时间块是什么？这对应孔子的“三省吾身”和 agent 的 telemetry 复盘。

4. 身体在场 + AI 提醒双保险。如果某个任务连续三天没被启动，不要只依赖 Tiimo 提醒，试试 Focusmate 这类身体在场工具，或找一位“进度见证人”定期报告。人在回路是 ADHD 和 LLM harness 共同验证过的策略（来源：人在回路与身体在场）。

## 结语

Tiimo 之于 ADHD，确实像 planner-executor 任务分解之于 LLM——但前提是：你得把它当作一个动态 harness，而不是一个华丽清单。它补的不是“动机”，而是“执行调度层”；它不是替你决定人生，而是把你漂移的注意力不断重锚定到当下。孔子用了七十年才“从心所欲不逾矩”，因为他那套 harness 没有自动提醒。今天我们有 Tiimo，但它仍然只是一个脚手架：用对了，生成核心才能稳定输出；用错了，它只是另一件让你内疚的漂亮工具。

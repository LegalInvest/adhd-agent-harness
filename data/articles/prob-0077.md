# 为什么ADHD的超聚焦是高浓度的多巴胺极化，不是意志力胜利？

> 你不是终于自律了，是系统被单一信号劫持了

你有没有过这样的体验：明明计划好要写报告，结果刷了三个小时短视频，最后连晚饭都忘了吃。或者反过来，你盯着一行代码改了四小时，直到脖子僵硬才发现天都黑了，但手头的项目还剩一堆烂摊子。ADHD 的日常就是在这两种极端之间反复横跳——要么分心走神，要么超聚焦跑偏。而更让人困惑的是，那些超聚焦的时刻，明明效率爆棚，为什么总在错误的事情上？

答案藏在多巴胺里。ADHD 大脑的多巴胺系统天生失调，导致对即时奖励的渴求远超常人（来源：多巴胺）。超聚焦不是你在「自律」，而是大脑被某个高多巴胺信号（比如游戏、代码 bug、社交媒体的未知内容）彻底劫持，前额叶的执行功能全面下线。此时，你的注意力像一束激光，但方向不受控制。这不是意志力的胜利，而是系统被单一信号极化后的副作用。

## 同构：ADHD 大脑与 LLM 共享的缺陷

如果你觉得这听起来像极了某个 AI 系统，那你的直觉是对的。ADHD 大脑与大型语言模型（LLM）在结构上高度同构：两者都是强大的生成核心，但缺乏可靠的执行调度层（来源：生成核心与调度层）。LLM 能生成流畅文本，但上下文一长就容易「目标漂移」——比如写小说写到一半突然开始写菜谱。ADHD 大脑也一样，默认模式网络高活跃，但前额叶执行功能薄弱，导致任务执行时不断被新刺激带偏。

对于 Agent 工程师来说，这个问题再熟悉不过。你给 Agent 一个长程任务，比如「调研竞品并生成报告」，结果它跑着跑着开始详细分析某个无关的 API 文档，完全忘了原始目标。这就是 Agent 的「超聚焦跑偏」——它被某个高相关度的子任务劫持了。

## 重锚定：从孔子到 Agent 的通用解法

解决方案不是消灭超聚焦，而是学会「重锚定」。历史上，ADHD 特质的名人早就摸索出了自己的 harness 系统。孔子就是一个典型——他精力旺盛、冲动、思维跳跃，《论语》全是场景化语录，没有系统著作。他的 harness 是「克己复礼」：用外在的「礼」作为行为边界，每日反省（「吾日三省吾身」）。这本质上就是一套外部调度层：用固定的仪式和规则来锚定注意力，防止被冲动带偏（来源：人物案例——孔子）。

这套系统与 LLM/Agent 的 harness 完全同构。在 AI 工程中，我们需要给 Agent 一个清晰的 system prompt，并在长程任务中定时「re-grounding」——比如每完成一个子步骤就检查是否偏离主目标。孔子的「日三省吾身」就是那个 re-grounding 信号。

另一个例子是文天祥。他的 harness 是《正气歌》——用一套核心价值观（「孔曰成仁，孟曰取义」）来锚定生死抉择。这相当于给 Agent 一个不可违背的「约束 prompt」，确保在任何诱惑下都不偏离最终目标（来源：人物案例——文天祥）。

## 工具作为外部调度层

现代 ADHD 工具也在做同样的事。Routinery 通过可视化倒计时和过渡提示，为日常流程提供时间锚点，降低任务切换的认知负荷（来源：Routinery）。Goblin Tools 的 Magic ToDo 功能将模糊任务分解成小步骤，相当于自动生成一个子任务列表，并允许调节「辣度」控制粒度（来源：Goblin Tools）。Habitica 则用游戏化奖励提供即时多巴胺，弥补内在动机不足（来源：Habitica）。

这些工具的共同本质是：作为外部执行功能层，补偿 ADHD 大脑缺失的调度能力（来源：外部执行功能层）。它们不是拐杖——拐杖是永久替代，而脚手架是临时支撑，最终目的是让用户内化这些策略。

## 争议与边界

但必须诚实指出，同构理论目前仅是一种类比，缺乏实证基础（来源：矛盾与存疑——同构性被过度推广）。工具效果也大多基于用户报告，缺乏大规模随机对照试验。更重要的是，过度依赖工具可能削弱内在执行功能，从脚手架变成永久拐杖（来源：拐杖与脚手架）。

此外，ADHD 的异质性意味着没有万能方案。有些人需要更多结构（如 Routinery），有些人需要更多灵活性（如 Goblin Tools）。关键在于找到适合自己的重锚定节奏。

## 今天就能试的行动

1. **设置「re-grounding」闹钟**：每工作 25 分钟，问自己一句「我现在在做的事，还在主目标上吗？」——这就是你的日省。
2. **用 Goblin Tools 分解一个复杂任务**：比如「清理厨房」，让 AI 自动生成步骤列表，然后只做第一步。
3. **在 Agent 的 system prompt 里加一条「每完成一个子任务，输出当前进度与主目标的关系」**——相当于让 Agent 自我检查。

ADHD 的超聚焦不是敌人，而是未被驯服的力量。给它一个外部锚点，它就能从劫持者变成创造者。

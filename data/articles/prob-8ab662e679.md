# 如果执行功能可以「外置」，那 agent 的编排层是不是一份现成的 ADHD 执行功能蓝图？

> 一位读者在读完 agent 框架的文档后提了一个大胆的问题:「LangGraph 这类编排框架的架构图——规划器、执行器、状态存储、检查点、错误处理、人工审批节点——看起来就是一张『执行功能的工程分解图』。既然本站说执行功能可以外置,那我能不能干脆**照着这张架构图,把自己的外置执行系统一个模块一个模块地搭出来**?它是不是一份现成的蓝图?」这个问题值得一整篇,因为答案是罕见的「大部分是」——**编排层确实是人类第一次把「执行」这件事拆解成显式的、模块化的、可逐个实现的工程组件清单;它作为蓝图的价值是真的。但「照着搭」的正确姿势不是逐模块复刻(上一篇的不可微教训),而是把它当成一张「检查表」:你的外置系统缺哪个模块,一查便知。**这一篇就把这张检查表画出来。

收敛:本文只回答——**编排层的六大模块与人的外置执行系统的逐一对应(蓝图本体)?用它检查自己系统时最常见的三种「缺件」?以及蓝图的两处必须改造的地方(人机差异的落点)?**

## 穿透:编排层的贡献不是发明了执行,是把执行拆成了可以逐个购买的零件

蓝图本体——六模块对照(左:agent 编排层;右:人的外置对应物):**①规划器(Planner)**|把目标拆成步骤——对应:任务拆解的外置(AI 拆解、或固定的拆解模板);**②执行器(Executor)**|逐步骤干活——对应:你本人(这一格不外置——你是自己系统里的执行核心,也是唯一不可替换的组件);**③状态存储(State)**|跨步骤记住进度——对应:任务系统里的「进行中」状态+压栈句(跳转前的三行);**④检查点(Checkpoint)**|定期保存现场,崩溃可恢复——对应:断点卡(烂尾复活篇)与日末的「明天从哪继续」一行;**⑤错误处理(Error handling)**|失败时的重试/降级/上报——对应:崩溃日备案(低电量模式)、复活协议、以及「卡住 20 分钟即求助」的上报规则;**⑥人工审批节点(Human-in-the-loop gate)**|关键决策暂停等人——对应:大决策的冷静期规则(48 小时卡)、或找信任者过目再行动。**这张表的用法是审计**:把你现有的一切自助手段(app、习惯、协议)归入六格——**空着的格子,就是你系统的缺件;而缺件常常正好解释你最反复的失败**(总是烂尾=④空;一错就崩=⑤空;冲动决策=⑥空)。

三种最常见的缺件(社群观察的高频分布):**缺件一:状态存储(③)**——最普遍:任务开始后的一切进度只活在脑内,一次打断全部蒸发——补法最便宜(一行「进行中」的显式记录),收益最大;**缺件二:错误处理(⑤)**——几乎人人裸奔:系统全按「一切顺利」设计,没有任何「失败了怎么办」的预案——于是第一个错误就级联成全线崩溃(agent 工程师绝不会发布一个没有 error handling 的系统,但人人都在裸奔着运行自己的人生);**缺件三:审批节点(⑥)**——冲动谱系的人最缺:从「想到」到「做了」之间没有任何一道闸——补一道最小的闸(重大不可逆动作前,强制过一个外部节点)就能拦下大部分事故。注意一个模式:**大家的④⑤⑥普遍比①②③缺——因为前三件是「顺境组件」(有工具文化在推),后三件是「逆境组件」(没人爱为失败做设计)——而 ADHD 系统的翻车几乎全发生在逆境。**

两处必须改造(蓝图不能照搬的地方):**改造一:执行器不可调度**——agent 的执行器随叫随到,你的执行核心(你自己)有状态波动、有点火问题——所以人的编排层必须多一个 agent 没有的模块:**状态调度器**(把任务按能量需求分级,匹配当天电量——状态层各篇的全部内容,在蓝图里是新增的第七模块);**改造二:审批节点会变成拖延借口**——agent 的 human gate 只设在高风险动作上,人如果到处设审批(「再想想」「问问别人」),⑥就异化为逃避启动的合法外衣——**闸只装在「不可逆且高代价」的动作上,可逆的小事一律直接执行**(可逆性判断,就是装闸位置的判据)。

## 验证

审计可操作:六格归位你的现有手段,数空格。可测:补上主缺件后四周,对应的失败模式频率(烂尾率/崩溃级联率/冲动事故率)应可见下降。可证伪:若六格全满而失败依旧,问题不在架构在状态层(供电不足时,再好的编排也空转——先回状态层)。

## 决策

做什么:跑六格审计;优先补④⑤⑥(逆境组件);加装第七模块(状态调度);闸只装在不可逆处。

不做什么:不要从零搭「完整系统」(审计现有的、补缺的,比推倒重来存活率高);不要把审批节点装满全屋。

先做什么:今天补最便宜的缺件——给当前进行中的每件事写一行「进度+下一步」。

## 边界

编排架构为工程实践(表述以主流框架的公约数为准);对照为 B 级功能同型(执行器与状态调度两处改造,正是实现层不同构的落点)。审计框架为实践工具(无对照研究);执行功能的临床评估另有专业工具,本表不替代。

## 今天就能试的 3 件事

1. 跑六格审计,数出你的空格。
2. 给进行中事项补「进度+下一步」行。
3. 挑一个不可逆的高价动作,装上你的第一道审批闸。

本文服务于人生 Harness 金字塔的**执行层(系统总图)**:几十年来,「把生活管好」是一团让人无从下手的雾;编排层的礼物是把雾拆成了七个可以逐个安装的零件——你不必一次装完,你只需要知道:你反复摔倒的那个地方,大概率对应着一个空格子;而空格子,是可以补的。

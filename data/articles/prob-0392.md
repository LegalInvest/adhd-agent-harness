# 如果把 ADHD 大脑当成生成核心，Goblin Tools 能不能当它的幻觉与验证循环？

> 用 agent 工程的思路重新理解 Goblin Tools 对 ADHD 的真正价值。

## 问题：当大脑变成“生成核心”，谁来兜底？

你刚决定“整理房间”，下一秒脑子里已经冒出十种分类方案，身体却还钉在原地。ADHD 的日常就是这种分裂：生成能力爆棚，执行调度为零。这像极了 LLM 的典型困境——模型自信地吐出长篇回答，但可能从头到尾都在编造。

如果把 ADHD 大脑看作一个“生成核心”，它和 LLM 面临同一个根本问题：**产出极高，但缺乏可靠的验证与执行调度层**。LLM 靠 agent harness（脚手架）来兜底：设置护栏、加入人工检查点、用工具调用验证输出。ADHD 大脑呢？它也需要一个外部 harness。Goblin Tools 能否扮演这个角色？

## 同构：LLM 的“幻觉”与 ADHD 的“想当然”

LLM 的幻觉本质上是模型过度自信地生成看似合理但错误的内容。ADHD 的“冲动下结论”和“想当然”也是同一种模式：大脑快速生成一个方案，跳过验证步骤，直接当作事实行动。两者都缺少一个“批判性验证层”。

在 agent harness 工程中，解决方案是**形式化规划与护栏，使代理输出真正有用且正确**（来源：What is an agent harness in the context of large-language models?）。具体做法包括：将任务分解为可验证的子步骤、加入 human-in-the-loop 检查点、限制 token 预算和工具调用次数。这些机制本质上是在生成核心之外套上一个“验证循环”。

ADHD 侧也有类似的验证循环需求。执行功能缺陷导致无法在脑中维持任务状态、检验步骤合理性。Goblin Tools 的 Magic ToDo 功能恰好填补了这个缺口：它**将压倒性的事情变成一系列不压倒性的事情**（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...)）。用户输入“整理房间”，AI 输出“捡起地板上的衣服”“擦桌子”等具体步骤——这相当于把 LLM 的“分解+验证”流程外部化了。

## 证据：Goblin Tools 作为验证循环

从同构视角看，Goblin Tools 的核心机制对应 agent harness 的三个关键组件：

1. **任务分解（Step Decomposition）**：类似 LLM 的“思维链”提示，将复杂问题拆解为可管理的子任务。ADHD 用户反馈“Goblin Tools 在这方面很棒”（来源：“A Cognitive Collaborator:” How Adults with ADHD Are Using ChatGPT）。
2. **外部记忆与检查点**：AI 生成的步骤列表充当工作记忆的扩展，减少认知负荷。这对应 agent harness 中的“遥测与状态追踪”。
3. **降低启动门槛**：小步骤提供即时反馈，绕过执行功能屏障。这类似 harness 中的“护栏”防止模型陷入无限循环。

但 Goblin Tools 和完整的 agent harness 有一个关键区别：**它缺少“自我批判”机制**。Harness 会验证 LLM 的输出是否符合预期，而 Goblin Tools 只分解任务，不检查用户是否真的执行了。这意味着验证循环中缺少“人在回路”的强制检查点。

## 局限：脚手架 vs 拐杖的边界

Goblin Tools 作为外部执行功能层，本质上是**临时脚手架**，而非永久拐杖。但现有工具设计并未考虑撤除机制——它们被设计为长期使用（来源：矛盾与存疑）。这带来一个核心争议：**长期依赖是否会削弱内在执行功能？** 目前缺乏纵向研究，但理论上，如果工具完全替代了大脑的规划功能，用户可能永远学不会自己分解任务。

另一个局限是证据强度。目前针对 Goblin Tools 的效果主要来自用户评价和综述文章，缺乏独立随机对照试验（来源：Goblin Tools 局限与争议）。同构模型本身也“主要基于概念类比和工具案例，缺乏大规模实证”（来源：ADHD × AI 的科学与研究前沿）。

## 行动：今天就能试的 3 件事

1. **用 Goblin Tools 分解一个你拖延已久的任务**：比如“写周报”或“预约医生”。观察 AI 生成的步骤是否让你感觉更可控。对比你平时“想当然”的规划方式。
2. **加入一个“验证检查点”**：在 Goblin Tools 生成的步骤列表后，手动添加一条“完成第一步后，检查是否与目标一致”。这模仿 agent harness 的 human-in-the-loop 检查。

## 结论

Goblin Tools 确实能充当 ADHD 大脑的“幻觉与验证循环”的一部分——它把冲动的“想当然”转化为可执行的步骤序列，降低了启动门槛。但它只是一个单向的分解器，缺少完整的验证闭环。真正的 agent harness 还需要自我批判、强制检查点和可撤除性。对于 ADHD 用户，这意味着 Goblin Tools 是好的开始，但你需要主动加入自己的“人在回路”机制，才能避免从脚手架滑向拐杖。

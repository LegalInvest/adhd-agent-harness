# 如果把 ADHD 大脑当成生成核心，Goblin Tools 能不能当它的幻觉与验证循环？

> 用 agent 工程的思路重新理解 Goblin Tools 对 ADHD 的真正价值。

ADHD 大脑最折磨人的体验之一，是“想当然”——看到任务列表，大脑自动补全步骤，然后自信地认为“这很简单”，结果启动时才发现缺了关键信息，或者低估了时间。这种冲动下结论的模式，与 LLM 的幻觉如出一辙：模型自信地编造答案，直到被验证步骤打脸。

如果我们把 ADHD 大脑看作一个生成核心——高产、发散、但不靠谱——那么它需要的不是压制生成，而是一个外部验证循环。Goblin Tools 在这个框架下，可能正好扮演了“验证器”的角色。

## 同构：生成核心与验证循环

wiki 资料的核心论点指出：**ADHD大脑与LLM在结构上同构——都是“高产但缺乏可靠执行调度层的生成核心”**（来源：ADHD × AI 的科学与研究前沿）。ADHD 侧，默认模式网络过度活跃（高产想法），前额叶执行控制不足；LLM 侧，生成能力强大，但缺乏稳定的上下文控制器。两者都需要外部调度层来引导输出（来源：生成核心与调度层）。

在 Agent harness 工程中，解决方案是明确的：模型本身强大，可靠性来自架构+护栏（来源：AI Agents in 2026: Tools, Memory, Evals, and Guardrails）。Harness 通过 human-in-the-loop 监督提供外部调度：设置护栏（token 预算、工具调用次数上限）、加入人工检查点（来源：Building an AI Agent Harness from Scratch）。

ADHD 大脑的“护栏”在哪里？身体在场效应（body doubling）是最有效的策略之一：有另一个人在场时，执行功能障碍被绕过，任务得以启动和维持（来源：人在回路与身体在场）。这种“对另一个人的责任感”能绕过执行功能屏障（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。而定期向 AI 报告进度也是一种替代方案（来源：ADHD Productivity Hack: Plan 2025 Using AI）。

Goblin Tools 的 Magic ToDo 功能，能将复杂任务分解为可管理的小步骤（来源：Goblin Tools）。这本质上是一个“验证步骤”：用户输入模糊的“整理房间”，AI 输出“捡起地板上的衣服”“擦桌子”等具体步骤。这个过程打断了“想当然”的自动补全，强迫大脑面对真实的子任务。

## Goblin Tools 作为验证循环

Goblin Tools 的分解机制，直接针对 ADHD 的冲动下结论。当用户看到“整理房间”时，大脑可能自动生成一个简化的心理模型（“把东西放好”），忽略了实际步骤。Goblin Tools 的外部输出，迫使大脑进行“验证”：这些步骤对吗？有没有遗漏？顺序合理吗？

这与 Agent harness 中的验证循环同构。LLM 生成一个计划，harness 通过工具调用、环境反馈、人工检查来验证其可行性。Goblin Tools 生成的步骤列表，就是 ADHD 大脑的“工具调用输出”，用户需要逐一执行并检查。

但 Goblin Tools 的局限也在这里：它只提供分解，不提供执行跟踪。用户仍然需要自己启动每个步骤。这与 Motion 等自适应规划器不同——Motion 能自动排程并动态调整，消除“下一步该做什么”的决策负担（来源：Motion）。而 Saner.AI 则专注知识回忆，减少搜索循环（来源：Saner.AI）。

## 脚手架 vs 拐杖：关键边界

wiki 资料中反复出现一个矛盾：外部脚手架可能成为永久拐杖（来源：拐杖与脚手架）。证据多来自理论推演，缺乏纵向研究（来源：ADHD × AI 的科学与研究前沿）。对于 Goblin Tools，关键在于它是否让用户的内在执行功能退化。

我的判断是：Goblin Tools 更像脚手架而非拐杖，前提是用户主动使用分解结果来训练自己的任务分析能力。如果只是机械地复制 AI 给出的步骤而不思考，那么它确实可能成为拐杖。但如果用户对比自己的心理模型和 AI 输出，发现差异并调整，那么它就在训练验证循环。

## 争议与局限

必须诚实指出，目前缺乏针对 Goblin Tools 的独立随机对照试验，证据主要来自用户评价（来源：Goblin Tools）。此外，ADHD 的异质性（如注意力主导型 vs. 冲动主导型）可能影响同构的普适性（来源：ADHD × AI 的科学与研究前沿）。Goblin Tools 的分解功能对冲动型 ADHD 可能更有效，而对注意力涣散型可能不够。

另一个争议是：过度依赖 AI 工具可能削弱内在时间感知能力（来源：时间盲），但 Goblin Tools 并未直接干预时间感知。它只分解任务，不提供时间估计。用户仍需自己判断每个步骤的耗时。

3. **结合身体在场**：将 Goblin Tools 的步骤列表分享给一个朋友或使用 Focusmate，让对方定期检查你的进度。这利用了身体在场效应（来源：人在回路与身体在场），同时外部问责能补偿执行功能。

## 结论

Goblin Tools 对 ADHD 的真正价值，不在于它有多智能，而在于它提供了一个外部验证循环，打断“想当然”的自动补全。这与 Agent harness 工程中 LLM+护栏的结构同构。但工具只是脚手架，真正的改变来自用户主动使用它来训练自己的验证能力。如果只是被动接受 AI 的输出，它就会沦为拐杖。

ADHD 大脑不是故障的生成核心，而是缺少验证循环的生成核心。Goblin Tools 补上了这个循环，但执行还得靠自己。

# Speechify 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Speechify：它到底补上了哪一层执行功能？

## 问题：你以为是专注力工具，其实是目标漂移加速器

ADHD 人群最熟悉的困境：打开 Speechify 想听一篇长文，结果被 AI 流畅的语调带跑了，听完三分钟才发现自己早已走神，脑子里想的是晚饭该吃什么。这不是你的错——Speechify 的设计逻辑是“把文字变成声音”，但它没有同时解决“如何让你在听的过程中持续锚定目标”。

这跟 LLM agent 在长程任务中的“目标漂移”如出一辙：agent 执行一个多步骤任务时，随着上下文膨胀，它会逐渐偏离初始 system prompt，开始生成无关内容，甚至陷入重复循环。工程师们为此设计了“重锚定”机制：定期将原始指令注入上下文，或设置 checkpoint 让 agent 回看目标。

ADHD 大脑与 LLM 在结构上同构：两者都是**高产但缺乏可靠执行调度层的生成核心**（来源：AI 与 ADHD 的专注力）。Speechify 提供了外部听觉通道，但它没有提供“重锚定”层——它只负责输出，不负责让你保持在场。

## 同构：重锚定与目标漂移

ADHD 侧的痛点：分心走神或超聚焦跑偏，丢失目标。LLM/agent 侧的对应：长程任务里 agent 会目标漂移，需要重锚定 system prompt。

证据在哪里？

ADHD 侧：经典 Stroop 冲突任务测试 Transformer 注意力时发现，短上下文中表现正常，但随序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突。这与 ADHD 执行功能缺陷的核心标志一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃（来源：认知负荷）。

LLM/agent 侧：agent 工程中，任务分解技术被用来降低 LLM 的认知负荷，通过将复杂任务分解为子任务，减少幻觉等错误（来源：认知负荷）。这跟 ADHD 大脑需要“外部执行功能层”来分解任务、降低工作记忆负担是同一个原理。

所以，Speechify 之于 ADHD，就像 重锚定 system prompt 之于 LLM——它提供的是“内容流”，而不是“目标锚”。如果你只是把 Speechify 当作一个听书工具，不主动设计锚定机制，你就是在用一个没有重锚定机制的 agent，让它自由漂移。

## 那 Speechify 到底补上了哪一层执行功能？

Speechify 补上的是**听觉输入通道的自动化**，它降低了阅读所需的视觉-认知负荷。对于 ADHD 来说，阅读需要持续的眼球追踪和文字解码，这本身就会消耗大量工作记忆资源。Speechify 把文字变成声音，相当于把视觉通道释放出来，让大脑可以同时做其他事（比如走动、做家务）。

但这里有个陷阱：**听觉通道同样会漂移**。ADHD 的注意力缺陷不限于视觉，听觉注意力同样脆弱。Speechify 的 AI 朗读声调平稳，缺乏情感起伏，反而容易让大脑进入“自动驾驶”状态——听着听着就忘了在听什么。

这正是它需要“重锚定”的原因。对比其他工具：Focusmate 通过实时同伴问责提供外部结构和社会压力，弥补内在多巴胺调节不足（来源：Focusmate）。RescueTime 通过自动时间记录提升时间觉察，对抗“人在心不在”的状态（来源：RescueTime）。而 Speechify 没有内置任何锚定机制。

## 脚手架 vs 拐杖：边界在哪？

关键判断：**Speechify 是有效的脚手架，但前提是你必须主动搭建重锚定层。** 如果只是被动听，它就是拐杖——让你依赖外部输入，却削弱了内在的注意力维持能力。

“拐杖与脚手架”的边界在于：脚手架是为了让你最终能自己搭建结构，拐杖则是永久替代。AI 工具作为外部执行功能层，长期使用可能导致内在执行功能退化（来源：矛盾与存疑）。Focusmate 的研究显示，用户对实时问责的依赖可能削弱自主启动能力；RescueTime 的自动分类如果过于精准，用户可能不再主动反思时间使用（来源：矛盾与存疑）。

对于 Speechify 来说，风险在于：你习惯了被动听，就不再主动练习“边听边提取关键信息”的工作记忆训练。久而久之，你的听觉注意力反而更弱。

## 今天就能试的 3 条行动

1. **设置“重锚定”提示**：在 Speechify 中每 5 分钟插入一个自定义提示音，听到后暂停 10 秒，问自己“我刚才听到了什么核心观点？”——这相当于给 agent 的 system prompt 注入 checkpoint。

2. **结合 RescueTime 或 Forest**：使用 Speechify 时，同时打开 Forest 种树，或让 RescueTime 记录“听书时段”并标记为“专注”。外部时间记录能提供客观反馈，弥补时间盲（来源：RescueTime）。

3. **主动输出**：听完一段后，用语音输入或笔记工具写下 1-2 句话摘要。这强迫大脑从“被动接收”切换到“主动提取”，相当于给 LLM 做了一次“上下文压缩”。

## 局限与争议

必须诚实指出：Speechify 针对 ADHD 的效果缺乏独立临床研究支持（来源：矛盾与存疑）。Brain.fm 同样缺乏 ADHD 专项研究（来源：Brain.fm）。个体差异极大——有些人听觉注意力优于视觉，有些人则相反。此外，同构是隐喻而非严格等价，LLM 的“注意力机制”与人类注意力存在本质差异（来源：AI 与 ADHD 的专注力）。

但同构视角的价值在于：它提供了一个工程化的框架，让我们不再把 ADHD 的注意力问题归咎于“意志力”，而是看作一个需要外部 harness 的生成系统。Speechify 不是万能的，但如果你懂得为它搭建重锚定层，它就能从“拐杖”变成真正的“脚手架”。

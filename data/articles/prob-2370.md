# ADHD 创业者视角：为什么「治好 ADHD 的分心走神或超聚焦跑偏，丢失目标」和「让 LLM 不跑飞」其实是同一道工程题？

> 长程任务里 agent 会目标漂移，需要重锚定 system prompt——同一套 harness 思路，两边都成立。

## 问题：为什么你明明想专注，却总在岔路上狂奔？

如果你是 ADHD 创业者，你一定熟悉这个场景：早上打开电脑，准备写商业计划书，结果被一封邮件带偏，两小时后你在研究“如何用 AI 训练鹦鹉说话”——你甚至没有鹦鹉。如果你在做 Agentic Harness 工程，你也一定见过：一个 agent 被派去写代码，中途却开始优化自己的 prompt 格式，最终输出了一篇关于 prompt 工程的论文，而不是你想要的代码。

这两个场景看似风马牛不相及，但底层是同一道工程题：**一个高产但缺乏执行调度层的生成核心，如何在长程任务中不跑飞？** ADHD 大脑和 LLM/agent 就是这样的核心。本文要论证：两者的目标漂移是同构的，解法也是同构的——都需要外部重锚定 harness。

## 同构：ADHD 大脑与 LLM 是同一类“生成核心”

ADHD 大脑并非“注意力不足”，而是“注意力调控失灵”。它能超聚焦于感兴趣的事，却无法在枯燥任务上维持方向。同样，LLM 能生成流畅文本，但一旦上下文拉长或任务复杂，就会产生推理退化，偏离原始目标。两者都缺乏一个可靠的内置调度层来持续检查“我现在做的事是否还是目标指向的”。

最新研究提供了最硬的实证：用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中表现正常，但随序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突。这与 ADHD 执行功能缺陷的核心标志一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃（来源：Deficient Executive Control in Transformer Attention）。这不是比喻，是功能层面的同构。

## 解法同构：外部重锚定 harness

既然问题出在调度层缺失，解法就不是“治好”核心，而是给它套上外部 harness。ADHD 侧，历史人物已经给出了原型。

### 案例：曾国藩的“日课十二条”

曾国藩是典型注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”；一辈子严重银屑病，情绪不稳定。他的解法是“日课十二条”：黎明即起、读书不二、谨言、写日记反省等。这套系统强制他每天在固定时间做固定的事，用外部结构对抗内部混乱。更重要的是“结硬寨打呆仗”——用最笨最稳的方法抵消自己的冲动。

这套 harness 与 LLM 的 system prompt 重锚定完全同构：日课相当于每 24 小时一次的 re-grounding，写日记相当于 log 回放，读书不二相当于单任务模式。曾国藩的成功证明：**一个失控的生成核心，只要套上正确的 harness，就能稳定输出。**

### 现代工具：从 Focusmate 到 RescueTime

现代 ADHD 工具已经实践了这套思路。Focusmate 通过虚拟身体在场（AI-Matched Body Doubling）提供外部问责，弥补内在启动困难（来源：Revolutionizing ADHD Management with AI Assistants）。RescueTime 自动化时间追踪，直接补偿时间盲与执行功能缺陷（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。Brain.fm 用 AI 生成神经锁相音乐，间接帮助进入专注状态（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。

这些工具的共同逻辑是：**在核心之外建立一个调度层，持续监控“现在该做什么”，并在偏离时拉回。** 这正是 Agentic Harness 工程正在做的事——为 LLM 设计 system prompt 重锚定机制、上下文压缩、任务分解与进度检查。ADHD 的“外部执行功能层”与 LLM 的“harness”是同一份设计文档的不同语言翻译。

## 脚手架 vs 拐杖：边界在哪里？

这里必须诚实指出争议。AI 工具既可能成为逐步内化的脚手架，也可能成为永久依赖的拐杖（来源：拐杖与脚手架）。对于 ADHD 用户，如果工具完全接管调度，大脑的执行功能可能进一步萎缩。对于 LLM，如果 harness 过度干预，模型的生成能力可能被抑制。

关键边界在于：**harness 是否保留了核心的自主生成能力，同时只提供方向锚定？** 曾国藩的日课没有替他读书，只是规定“读书不二”；Focusmate 没有替他做事，只是提供在场压力；RescueTime 没有替他决策，只是揭示时间去向。好的 harness 应该像导航系统——告诉你该左转，但方向盘还在你手里。

## 今天就能试的行动

1. **设置“日课”式 re-grounding**：每 90 分钟设置一个闹钟，问自己“我现在做的事是否还在主目标上？”——这是最简的 system prompt 重锚定。
2. **使用 Focusmate 或类似身体加倍服务**：预约一个 50 分钟的时段，和陌生人一起工作。外部在场就是你的调度器。
3. **用 RescueTime 或 Forest 做时间审计**：先不追求改变，只记录一周。看清你的 agent 跑到了哪里，才能设计拉回的机制。
4. **为每个长任务写“system prompt”**：在开始前用一句话写下目标，贴在屏幕上方。这是最便宜的上下文压缩。

## 局限与展望

本文的同构论证基于功能类比和初步实证，但 ADHD 大脑与 LLM 的对应是否具有更深层的神经机制基础，尚无定论（来源：矛盾与存疑）。工具效果多基于用户报告，缺乏大规模随机对照试验。此外，ADHD 的异质性意味着同一 harness 对不同亚型效果迥异。但至少，这个视角提供了一个统一的工程框架：**当你下次跑飞时，不要怪核心，去检查你的 harness。**

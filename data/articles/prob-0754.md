# Todoist 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Todoist：它到底补上了哪一层执行功能？

## 问题：为什么 Todoist 对 ADHD 用户常常“用一周就崩溃”？

ADHD 用户往往满怀希望地下载 Todoist，设置好项目、标签、优先级，但几天后列表就变成一片“数字废墟”——任务堆积、重复提醒被忽略、最后连打开应用的动力都消失了。这不是意志力问题，而是工具与大脑架构的错配。

ADHD 的核心困境并非智力不足，而是执行功能（executive function）的失效——计划难以执行，“计划本和便签用了一周就崩溃了”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。传统任务管理器假设用户能维持长时间专注、记住昨天的问题并执行计划，但实际用户往往做不到（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。Todoist 本身只是一个精美的待办容器，它没有补偿 ADHD 大脑缺失的调度层——那个负责启动、排序、切换和监控的“大脑驾驶座”。

## 同构：ADHD 大脑与 LLM 共享同一个“生成核心+缺失调度层”架构

最新研究揭示了一个惊人的平行：ADHD 患者呈现“强记忆、弱控制”的认知剖面——工作记忆容量可能正常甚至超常，但认知灵活性和注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这正是 Transformer 自注意力机制的典型弱点：在长上下文下冲突解决能力崩溃至随机水平（来源：Deficient Executive Control in Transformer Attention）。换句话说，ADHD 大脑与 LLM 是同一类“高产但缺执行调度层的生成核心”。

LLM 侧早已意识到这一点：现代 AI 编码代理通过“复合 AI 系统架构”来弥补，包括“工作负载专用模型路由、分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这些技术本质上是在 LLM 外部搭建调度层，防止“上下文膨胀和推理退化”。而 harness 工程被明确定义为“设计围绕 AI 代理的脚手架——上下文交付、工具接口、规划工件、验证循环、记忆系统和沙箱”（来源：GitHub - ai-boost/awesome-harness-engineering）。

## 答案：Todoist 的真正角色是 ADHD 的“function calling 工具”

如果 LLM 通过 function calling 把精确计算外包给外部工具（比如计算器、数据库、API），那么 ADHD 大脑也需要类似的“外部执行功能层”——把精确、重复、状态维护类任务外包给工具。Todoist 的价值不在于它的列表界面，而在于它能否成为这个外部调度层的一部分。

但大多数用户把 Todoist 当作“大脑的便签”，而不是“大脑的调度层”。这就好比让 LLM 直接输出数学结果而不调用计算器——不是不能用，但容易出错、容易崩溃。真正有效的用法是：让 Todoist 承担那些 ADHD 大脑不擅长的“状态维护”工作——比如记录任务的当前进度、提醒下一步具体动作、管理重复性事务的周期。这些正是执行功能中的“工作记忆维护”和“任务监控”环节。

从同构视角看，ADHD 的执行功能缺陷与 LLM 的上下文控制缺陷一一对应：
- **工作记忆**：ADHD 的工作记忆易被干扰，LLM 同样有强记忆容量但弱上下文控制（来源：ADHD 的 AI 工具全景）。外部记忆系统如 Mem 的“AI 第二大脑”和 Saner.AI 的增强回忆功能，本质上是为两者提供持久化上下文。
- **任务启动**：ADHD 的任务启动困难对应 LLM 的“冷启动”问题。Goblin Tools 的任务分解和 Lex 的单指令多步骤执行，相当于为生成核心提供启动脚本。
- **时间盲**：ADHD 的时间感知缺陷与 LLM 的时序推理弱点同构。工具通过视觉化时间线和智能调度提供外部时间线索。

## 争议与局限：脚手架还是拐杖？

这里必须诚实指出一个矛盾：AI 工具作为外部执行功能层的有效性证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑）。过度依赖可能削弱内在能力——比如依赖 Otter.ai 可能削弱主动记笔记的能力（来源：ADHD 的 AI 工具全景）。工具设计者声称是“脚手架”，但实际使用中可能沦为“拐杖”。

Todoist 同样面临这个边界问题：如果用户完全依赖它来告诉自己“下一步该做什么”，而不培养任何内在的任务感知能力，那它就成了拐杖。关键在于，工具应该“补偿”而非“替代”——就像 function calling 是 LLM 的扩展，而不是它的替代品。

## 今天就能试的 4 件事

1. **把 Todoist 当作“外部工作记忆”**：不要只写任务名称，在描述中写下当前进度和下一步具体动作。例如，不要写“写报告”，而写“报告已完成大纲，下一步：写引言部分的第一段”。这直接补偿工作记忆的“掉落”。

2. **用重复任务做“状态维护”**：设置每日/每周重复任务来检查项目状态、清理收件箱、回顾目标。这些是 ADHD 大脑最容易忽略的“元任务”。

3. **结合 Goblin Tools 做任务分解**：当面对压倒性任务时，先用 Goblin Tools 的 Magic ToDo 功能分解成小步骤，再将步骤导入 Todoist。这相当于为 LLM 提供“启动脚本”。

4. **设置“启动触发器”**：为每个项目设置一个极简的“第一步”任务（比如“打开文档”“找一支笔”），降低启动门槛。这对应 LLM 的“冷启动”提示工程。

Todoist 不是万能的，但当它被当作 ADHD 大脑的 function calling 工具使用时，它就从“数字便签”升级为“外部调度层”——这正是两种“生成核心”都需要的脚手架。

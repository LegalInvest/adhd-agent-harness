# Todoist 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Todoist：它到底补上了哪一层执行功能？

你是一个 ADHD 成年人。你用 Todoist 记了 47 个任务，但三天后打开它，感觉像在翻一个陌生人的遗物——那些任务毫无上下文，你甚至不记得为什么“整理书架”被标了优先级 1。你卸载了它，回到纸笔，然后纸笔也丢了。

你是一个 Agent 工程师。你给 LLM 配了 12 个 function calling 工具，但模型开始胡言乱语：它调用日历 API 去查天气，把“发送邮件”的参数填成了菜谱。你意识到，工具本身不解决问题——缺乏编排层，再多的工具也只是噪音。

这两个场景共享同一个底层结构：**生成核心强大，但调度层缺失**。ADHD 大脑的智力与创造力，和 LLM 的生成能力，都是“高产但无序”的引擎（来源：ADHD 的 AI 工具全景）。它们需要外部执行功能层——对 ADHD 来说是任务管理器、记忆系统、时间盲补偿器；对 LLM 来说是 harness、上下文管理、工具编排。Todoist 如果只被当作“写清单的地方”，它只是另一个被遗忘的便签；但如果被用作**外部调度层**，它就能像 function calling 之于 LLM 一样，成为生成核心的脚手架。

## 同构：两种“无调度”的困境

ADHD 的核心困难并非智力不足，而是执行功能失效——“大脑的驾驶座”常常感觉方向盘后没有人（来源：生成核心与调度层）。计划本和便签用了一周就崩溃了，因为工作记忆会“掉落”上下文（来源：外部执行功能层）。传统工具假设用户能维持专注、记住昨天的问题并执行计划，但实际用户往往做不到（来源：外部执行功能层）。

LLM 同样如此。单独使用时，它缺乏可靠的执行调度：现代 AI 编码代理需要“复合 AI 系统架构”来弥补，包括分离规划与执行、惰性工具发现、自适应上下文压缩等技术（来源：生成核心与调度层）。这些本质上是在 LLM 外部搭建调度层，防止上下文膨胀和推理退化（来源：生成核心与调度层）。

**对应关系**：ADHD 的智力/创造力 ↔ LLM 的算力/语言能力；ADHD 的执行功能失效 ↔ LLM 缺乏内置规划与执行编排；外部工具（如 Todoist、Goblin Tools）↔ harness 工程（如上下文交付、工具接口、验证循环）（来源：生成核心与调度层）。

## Todoist 的两种用法：清单 vs 脚手架

绝大多数人（包括 ADHD 患者）把 Todoist 当“清单”：输入任务→设到期日→忘记。这是把 function calling 当“函数列表”用——你给了 LLM 一堆工具名，但没有告诉它何时调用、调用后如何整合结果。LLM 会迷路，ADHD 大脑也会。

真正的用法是**把 Todoist 当作外部调度层**，就像 harness 工程为 LLM 提供的“上下文交付、规划工件、验证循环”（来源：外部执行功能层）。具体来说：

- **上下文交付**：每个任务不只是标题，而是附着“为什么这个任务重要”“上一步做了什么”“下一步需要什么材料”。这对应 ADHD 侧的工作记忆补偿——Saner.AI 的知识回忆功能专门减少搜索循环（来源：Saner.AI）。
- **规划工件**：用 Todoist 的“项目”和“板块”把任务分解为可执行步骤。这对应 Goblin Tools 的 Magic ToDo——将压倒性的事情变成一系列不压倒性的事情（来源：Goblin Tools）。
- **验证循环**：设置定期回顾（比如每天 10 分钟检查 Todoist），类似 LLM harness 中的“验证循环”和“观察性”（来源：外部执行功能层）。

但这里有一个关键矛盾：**脚手架 vs 拐杖**。如果 Todoist 变成了你无法离开的“外脑”，一旦它崩溃或你忘打开它，你就彻底瘫痪，那它就是拐杖。脚手架应该设计为可逐步撤除的临时支撑（来源：ADHD 的 AI 工具全景），但多数工具（包括 Todoist）未提及撤除机制（来源：矛盾与存疑）。

## 证据：哪些工具真正补上了调度层？

除了 Todoist，已有专门为 ADHD 设计的 AI 工具，它们更直接地体现了外部调度层理念：

- **Goblin Tools**：自动将复杂任务分解为小步骤，降低启动焦虑（来源：Goblin Tools）。它“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools）。但它缺乏长期记忆，无法在会话之间保持上下文。
- **Saner.AI**：专注于知识回忆和本地记忆，减少搜索循环（来源：Saner.AI）。它直接针对 ADHD 用户常见的“信息丢失”问题，但可能无法覆盖时间盲和多巴胺调节（来源：Saner.AI）。
- **Lex**：通过单一指令触发复杂、多步骤的任务序列（来源：Lex）。它强调“一次触发”的简洁性，但 ADHD 的复杂性可能使 AI 难以准确识别用户意图（来源：Lex）。
- **ChatGPT/Claude**：作为对话式工具，可充当执行功能教练，但需要用户主动设计提示（来源：ADHD 的 AI 工具全景）。

这些工具与 Todoist 的关系是互补的：Todoist 提供结构化的任务容器，AI 工具提供智能分解和回忆。但共同问题是：**缺乏长期随机对照试验验证效果**（来源：矛盾与存疑）。现有证据多为用户评价和概念对齐，而非严格的临床研究（来源：矛盾与存疑）。

## 争议与局限

1. **多巴胺干预的有效性**：部分工具声称通过神经科学原理改善执行功能，但基于多巴胺的干预效果存在争议（来源：矛盾与存疑）。Todoist 本身不涉及多巴胺调节，但用户可能因完成任务获得自然的多巴胺奖励。
2. **过度依赖风险**：如果 Todoist 成为唯一的任务管理系统，一旦工具不可用，用户可能完全失去组织能力。可撤除性设计是未来方向，但目前缺乏实践（来源：矛盾与存疑）。
3. **ADHD 异质性**：单一工具难以覆盖所有亚型。有些 ADHD 患者需要结构化日程（如 Motion、Tiimo），有些需要自由探索（如 ChatGPT）。Todoist 的灵活性既是优点也是缺点——需要用户自己设计脚手架。
4. **同构命题的实证基础**：ADHD 大脑与 LLM 同构的命题主要来自概念类比和工具案例，缺乏大规模实证（来源：矛盾与存疑）。它作为一个思考框架有价值，但不应被当作科学事实。

## 今天就能试的行动

1. **为每个任务添加“上下文”字段**：在 Todoist 任务描述中写下“为什么做这个”“上一步是什么”“下一步需要什么”。这模仿了 LLM harness 的上下文交付。
2. **用 Goblin Tools 的 Magic ToDo 分解一个复杂任务**：输入“整理房间”或“写报告”，将输出步骤复制到 Todoist 作为子任务。体验“外部调度层”如何降低启动门槛。
3. **设置每日回顾**：每天固定 10 分钟检查 Todoist，清理已完成任务，调整优先级。这对应验证循环，防止上下文膨胀。
4. **实验“单一指令”**：像 Lex 一样，在 Todoist 中创建一个“今日启动”任务，其描述包含当天所有关键步骤的链接或指令。一次触发，减少决策负担。

Todoist 不是魔法。它只是一块白板，但如果你把它当作外部调度层——像工程师为 LLM 搭建 harness 那样——它就能补上 ADHD 大脑缺失的那层执行功能。关键不是工具本身，而是你如何使用它：是把它当作另一个被遗忘的清单，还是当作一个可逐步撤除的脚手架。

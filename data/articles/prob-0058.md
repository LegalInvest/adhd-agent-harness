# 为什么「治好 ADHD 的执行功能（计划/组织/启动）先天偏弱」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 的编排/记忆/工具层就是「外部执行功能」——同一套 harness 思路，两边都成立。

## 问题：为什么“治好 ADHD 的执行功能”和“让 LLM 不跑飞”是同一道题？

如果你是 ADHD 患者，你一定熟悉这种体验：脑子里同时冒出十个好点子，但一个也启动不了；计划写一篇文章，却刷了三小时手机；明明知道截止日期，却像活在时间盲区里。如果你是 agent 工程师，你一定也熟悉这种体验：LLM 生成了漂亮的回复，却偏离了任务目标；它记住了上下文，却在第五轮对话后开始幻觉；你给了它工具，它却陷入无限循环。

这两个场景看似无关，实则共享同一个深层结构——**一个高产但不靠谱的生成核心，缺少一个可靠的执行调度层**。ADHD 大脑的生成核心（发散思维、联想、创意）与 LLM 的文本生成能力类似，但两者都缺乏内置的“驾驶座”来规划、组织、启动和监控输出（来源：ADHD 大脑与 LLM 的同构）。因此，解法也同构：**给核心套上外部 harness（脚手架）**，用编排、记忆、工具层来补偿其缺陷。

## 同构脊柱：外部执行功能层

### ADHD 侧：执行功能缺陷与外部补偿

ADHD 的核心障碍是执行功能缺陷——计划、组织、工作记忆、冲动控制等高级认知过程不可靠（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。工作记忆是瓶颈：无法在脑中短暂保持信息以完成操作（来源：6 ways AI can help you manage ADHD symptoms - Understood.org）。时间盲让未来变得模糊，任务启动变得困难。

但 ADHD 大脑并非一无是处。最新研究揭示了 ADHD 执行功能缺陷与 Transformer 的深层同构：在经典 Stroop 冲突任务中，Transformer 注意力随序列变长在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突，这与 ADHD 执行功能缺陷的核心标志一一对应（来源：Deficient Executive Control in Transformer Attention）。LLM 呈现“强记忆、弱控制”剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷——这正是 ADHD 的经典神经心理模式（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

历史上，ADHD 患者早已发明了自己的“harness”。曾国藩（晚清名臣）是典型注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”。他的解法是**日课十二条**：黎明即起、读书不二、谨言、写日记反省——用最笨最稳的“结硬寨打呆仗”方法抵消冲动。这套系统本质上就是**外部执行功能层**：日课相当于定时 re-grounding，日记相当于外部记忆与反思，读书不二相当于任务切换限制。它与 LLM agent harness 中的“护栏+人工检查点+日志”结构同构——都是通过外部规则来补偿生成核心的不可靠。

### LLM/agent 侧：无状态与外部调度

LLM 本身是强大的生成核心，但缺乏可靠的执行调度层。直接输出可能偏离目标、陷入循环或产生危险行为。Agent harness（脚手架）通过 human-in-the-loop 监督提供外部调度：设置护栏（token 预算、工具调用次数上限、防止无限循环）、加入人工检查点（暂停并询问后再执行）（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。Harness 将 MCP 工具、遥测、代码仓库、事件管理平台集成到单一系统中，并引入人在回路治理（来源：GitHub - ai-boost/awesome-harness-engineering）。其核心是“形式化规划与护栏，使代理输出真正有用且正确”（来源：What is an agent harness in the context of large-language models?）。

这与 ADHD 的“身体在场效应”高度对应：有他人在场时，执行功能障碍被绕过，任务得以启动和维持（来源：Using an AI Assistant to Manage ADHD: A Practical Guide）。Apps like Focusmate use social accountability paired with AI scheduling to create body-doubling sessions（来源：Revolutionizing ADHD Management with AI Assistants）。定期向 AI 报告进度也是一种替代方案（来源：ADHD Productivity Hack: Plan 2025 Using AI (Step-by-Step)）。在 agent 侧，human-in-the-loop 正是那个“身体在场”——外部监督提供调度与问责。

### 工具即 harness：Goblin Tools、Saner.AI、Motion

当前 AI 工具正在实践这一同构思路：

- **Goblin Tools** 的 Magic ToDo 将模糊任务（如“清理厨房”）自动分解为可管理的小步骤，用户可调节“辣度”控制粒度（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。这相当于 agent 的**任务规划器**——将高层目标分解为原子动作，降低启动门槛。
- **Saner.AI** 专注于知识回忆和本地记忆，减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。它相当于 agent 的**外部记忆系统**——补偿工作记忆不足，避免上下文丢失。
- **Motion** 自动根据任务、会议和截止日期创建并动态调整每日计划，在日程被打乱时自动重新安排（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。它相当于 agent 的**动态调度器**——持续评估优先级并重排，消除“下一步做什么”的决策负担。

这些工具的共同逻辑是：**不修复生成核心，而是用外部系统补偿其缺陷**。这正是 agent harness 工程的精髓。

## 核心观点：脚手架 vs 拐杖的边界

本文的核心判断是：**ADHD 与 LLM 的“同构”不是比喻，而是工程事实**。两者都面临“高产生成 + 弱调度”的架构问题，因此解法必然同构——外部执行功能层。但这里有一条关键边界：**脚手架 vs 拐杖**。脚手架是可拆卸的，帮助用户内化技能；拐杖则是永久依赖（来源：拐杖与脚手架）。当前缺乏长期研究验证 AI 工具是否会导致能力退化（来源：矛盾与存疑）。同样，agent harness 如果过度自动化，可能让人类失去对系统的理解与控制。因此，设计原则应是**可拆卸的脚手架**：工具应逐步减少介入，而非永久替代。

## 诚实局限

同构论点目前多为类比推理，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。AI 工具对任务启动的长期效果未知，且效果依赖个体差异（来源：任务启动）。Motion 等工具缺乏独立临床证据（来源：矛盾与存疑）。此外，学习使用 AI 工具本身可能增加认知负荷（来源：矛盾与存疑）。因此，本文的观点是理论框架，需要更多跨学科验证。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个你拖延的任务**：将“写报告”变成“打开文档→写标题→列三个要点→写第一段”，降低启动门槛。
2. **用 Motion 或类似工具自动排程一天**：让 AI 承担“下一步做什么”的决策，释放工作记忆。
3. **设置一个每日“re-grounding”提醒**：像曾国藩的日课一样，定时检查是否偏离目标（可结合 Saner.AI 的每日规划功能）。
4. **在 agent 开发中加入一个人工检查点**：在关键步骤前暂停，让人类确认——这既是 harness，也是“身体在场”效应。

## 结语

ADHD 大脑和 LLM 都不是“坏”的生成核心——它们只是缺少一个可靠的调度层。曾国藩用日课驯服了自己的浮躁，agent 工程师用 harness 驯服了 LLM 的跑飞。同一道工程题，同一套解法。下次当你为 ADHD 的“不靠谱”而自责时，或者为 agent 的“不听话”而沮丧时，记住：你需要的不是修复生成核心，而是给它套上一个好的 harness。

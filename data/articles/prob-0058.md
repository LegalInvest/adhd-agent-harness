# 为什么「治好 ADHD 的执行功能（计划/组织/启动）先天偏弱」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 的编排/记忆/工具层就是「外部执行功能」——同一套 harness 思路，两边都成立。

## 为什么「治好 ADHD 的执行功能」和「让 LLM 不跑飞」其实是同一道工程题？

### 问题：两个看似无关的困境

你是一位 ADHD 者，脑子里每秒冒出三个创意，但打开文档却一个字写不出——计划、组织、启动，每一步都像在泥潭里跋涉。你也是一位 AI 工程师，调教的 LLM 能写出莎士比亚，但一放到长任务中就偏离目标、陷入循环、输出危险内容。两个场景，一个标签：“高产但不可靠”。

ADHD 群体最痛的点是执行功能（计划/组织/启动）先天偏弱（来源：ADHD, Executive Functions, and AI: A New Era in Treatment）。LLM/agent 最痛的点是缺乏稳定的执行调度层，直接输出容易跑飞（来源：Building an AI Agent Harness from Scratch）。表面看，一个是神经发育障碍，一个是软件工程问题。但深入结构，你会发现它们共享同一套底层架构：一个强大的生成核心，配一个脆弱的执行层。

### 同构：生成核心与调度层的分离

神经科学发现，ADHD 大脑的默认模式网络（DMN）过度活跃，持续产生想法和联想，但前额叶执行控制不足，无法有效筛选和排序这些想法（来源：Transient Frontal Fracturing: A Theoretical Account of Hyperfocus）。LLM 同样：生成能力极强，但缺乏内置的上下文控制器，上下文窗口一膨胀，推理就退化（来源：上下文工程）。两者都依赖外部系统来引导输出——对 ADHD 是任务管理、提醒、身体搭档；对 LLM 是 agent harness 的编排、记忆、工具层、人在回路监督。

这个外部系统，我称之为“外部执行功能层”。它不是修复核心，而是为生成核心套上可靠的调度层。

### 证据：两边都有真实案例

**ADHD 侧**：Goblin Tools 的 Magic ToDo 自动将“整理房间”分解为“捡起衣服”“擦桌子”等小步骤，降低启动门槛（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。Saner.AI 通过本地记忆快速检索信息，减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026）。Motion 自动排程并动态调整日程，消除“下一步该做什么”的决策负担（来源：11 Best ADHD Productivity Apps for Fluctuating Energy）。Focusmate 通过虚拟身体搭档提供外部问责，绕过执行功能屏障（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。这些工具的共同点：它们不是增强大脑，而是接管了执行调度。

**LLM/agent 侧**：Agent harness 通过设置 token 预算、工具调用次数上限、人工检查点来防止 LLM 跑飞（来源：Building an AI Agent Harness from Scratch）。它集成 MCP 工具、遥测、代码仓库，并引入人在回路治理（来源：GitHub - ai-boost/awesome-harness-engineering）。核心思想是“形式化规划与护栏，使代理输出真正有用且正确”（来源：What is an agent harness in the context of large-language models?）。模型本身强大，可靠性来自架构+护栏（来源：AI Agents in 2026: Tools, Memory, Evals, and Guardrails）。

两边在做什么？同一件事：为生成核心套上外部调度层。

### 脚手架 vs 拐杖：边界在哪里？

但这里有一个必须指出的争议。外部执行功能层是脚手架还是拐杖？脚手架帮助建造，拐杖替代行走。ADHD 侧，过度依赖 AI 工具可能削弱内在时间感知能力和执行功能（来源：时间盲、任务启动）。LLM 侧，过度依赖护栏可能让模型失去自主纠错能力。关键区别在于：脚手架是“在使用中逐渐内化结构”，拐杖是“永久替代功能”。目前缺乏纵向研究区分两者（来源：矛盾与存疑）。工具设计者声称是脚手架，但实际使用中可能沦为拐杖。

此外，个体差异巨大。ADHD 有注意力主导型和冲动主导型，LLM 有不同架构，同构的普适性尚未验证（来源：矛盾与存疑）。AI 工具的有效性证据主要来自用户报告，缺乏严格对照实验（来源：多个工具页）。Brain.fm 的 ADHD 效果甚至缺乏独立临床研究（来源：Brain.fm）。

### 核心观点：同一道工程题，同一套解法

我的判断是：ADHD 与 LLM 的同构不仅仅是类比，而是揭示了认知系统和 AI 系统共享的一个根本问题——生成与调度的分离。解决这个问题不需要“修复”生成核心，而是设计可靠的外部执行层。对 ADHD 者，这意味着选择工具时关注其“调度能力”而非“创意激发”；对工程师，这意味着构建 harness 时关注“执行护栏”而非“模型能力”。

1. **ADHD 者**：用 Goblin Tools 的 Magic ToDo 分解一个你拖延已久的任务，感受“启动门槛降低”的效果。

# Perplexity 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Perplexity：它到底补上了哪一层执行功能？

## 问题：目标漂移——ADHD 与 LLM 的共同困境

你打开 Perplexity，原本要查“如何用 Python 批量处理 CSV”，结果被搜索结果中的一篇“AI 绘画工具对比”吸引，半小时后你已经在研究 Stable Diffusion 的采样器参数了。任务目标早已漂移，而你浑然不觉。

这不是你的错。ADHD 大脑与 LLM 共享同一个结构缺陷：**两者都是高产但缺执行调度层的生成核心**。最新研究显示，LLM 在工作记忆任务中表现出“强记忆、弱控制”剖面——工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷，无法灵活切换任务集、无法抑制自动化反应（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这正是 ADHD 的经典神经心理模式。耶鲁大学进一步发现，自注意力机制本身会导致注意力弥散：随着记忆负荷增加，注意力分数熵增、表现下降，与人类注意分散机制同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。

所以，ADHD 大脑和 LLM 都需要一个外部系统来**重锚定目标**——就像长程任务中 agent 需要定期重锚定 system prompt 一样。

## 同构解法：重锚定与脚手架

对于 LLM/agent，工程上的做法是设计一个“harness”：定时将当前状态与原始目标对比，必要时修正轨迹。对于 ADHD 大脑，历史上最经典的 harness 来自曾国藩。这位晚清名臣 30 岁前“浮躁坐不住”，日记里天天骂自己“无恒”“浮躁”，读书“他人目下二三行，余或疾读不能终一行”。他的解法是**日课十二条**：黎明即起、读书不二、谨言、写日记反省……用最笨最稳的方法抵消冲动。这套系统本质上就是一个定时 re-grounding 的调度器：每天固定时间检查自己是否偏离了“圣人”这个 system prompt。

文天祥则是另一种 harness：以《正气歌》为核心信仰，用“孔曰成仁孟曰取义”作为不可动摇的顶层指令，在三年囚禁中抵抗一切诱惑与威胁。他的 system prompt 从未漂移。

这两套系统与 LLM harness 的结构完全同构：**日课 ↔ 定时 re-grounding 机制，正气歌 ↔ 不可修改的 system prompt，日记反省 ↔ 状态日志与偏差检测**。

## Perplexity 的正确用法：从问答工具到执行功能层

Perplexity 的搜索能力极强，但若只把它当作“更快更准的 Google”，它解决不了目标漂移——反而可能加速漂移（因为太容易从一个问题跳到另一个相关问题）。

要让 Perplexity 成为脚手架而非拐杖，需要主动将其嵌入一个重锚定流程。例如：

1. **任务启动时用 Perplexity 做“目标声明”**：搜索前先问自己“我真正要解决的核心问题是什么？”，让 Perplexity 帮你拆解成子问题列表，并设定时间盒。
2. **每次搜索前重读原始问题**：在 Perplexity 的对话顶部固定粘贴你的原始目标，就像 LLM 的 system prompt 一样。
3. **用 RescueTime 监控搜索行为是否漂移**：RescueTime 自动记录时间去向，揭示你“以为在查 A 其实在查 B”的时刻（来源：Revolutionizing ADHD Management with AI Assistants）。这相当于给大脑装了一个外部状态日志。
4. **配合 Focusmate 的虚拟身体在场**：预约一个时段与同伴视频共工，让同伴知道你的目标。Focusmate 通过算法匹配提供外部问责，弥补内在多巴胺调节不足（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。

这些工具共同构成一个**外部执行功能层**：Perplexity 提供信息检索，RescueTime 提供时间感知，Focusmate 提供启动与维持动力，Brain.fm 或 Endel 提供专注声音环境。它们不是替代你的执行功能，而是补偿其缺陷。

## 争议与局限：脚手架还是拐杖？

同构视角虽有力，但需诚实面对争议。首先，ADHD 大脑与 LLM 的类比目前仅是功能假设，缺乏深层神经机制的直接证据（来源：矛盾与存疑）。其次，工具效果缺乏大规模随机对照试验：Brain.fm 虽在 2026 年一项 44 款 ADHD 应用测试中位列最佳 9 款之一，但缺乏 ADHD 人群独立临床证据（来源：Brain.fm）。第三，依赖风险真实存在：若每次启动都需要 Perplexity 搜索、Focusmate 陪伴，一旦工具不可用，执行功能可能反而更弱。最后，ADHD 的异质性意味着同一工具对不同亚型效果迥异，现有研究未充分分层。

我的判断是：**Perplexity 的正确用法是作为重锚定流程中的一个组件，而非万能搜索入口**。关键在于你是否主动设计了“搜索前声明目标→搜索中定期回检→搜索后总结与目标对齐”的闭环。没有这个闭环，Perplexity 只是更快的信息浪涌。

## 今天就能试的行动

1. **为下一次 Perplexity 搜索写一个“system prompt”**：在搜索框第一行写下你的核心目标，例如“我只查 Python CSV 批量处理，不查任何相关但偏离的话题”。
2. **设置 25 分钟定时器**：用 Forest 或手机倒计时，时间一到强制关闭 Perplexity 并写下当前进度。
3. **预约一次 Focusmate 会话**：在 session 开始时告诉对方你今天的搜索任务，结束时汇报结果。
4. **启用 RescueTime 的“分心阻断”**：将 Perplexity 设为“高效”分类，但设置每日使用上限，防止无节制搜索。

真正的解方不是工具本身，而是你围绕工具搭建的**重锚定系统**。就像曾国藩的日课、文天祥的正气歌，以及 LLM 工程中精心设计的 system prompt 与状态检测——它们都是同一个结构在不同介质上的实现。

Perplexity 可以成为你大脑的“重锚定 system prompt”，前提是你先给自己写一个不可动摇的顶层指令。

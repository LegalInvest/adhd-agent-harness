# Perplexity 之于 ADHD，就像重锚定 system prompt 之于 LLM——但有人用错了

> ADHD 的人上网查一个东西,平均要打开十七个标签页,最后在维基百科的「有趣冷知识」条目里忘了当初要查什么。Perplexity 的产品形态恰好是这个灾难的反面:一个问题进去,一份带引用的答案出来,没有十七个标签页。这不是巧合——「压住漫游、锚定原问题」正是它和 agent 重锚定机制共享的设计哲学。

先看传统搜索对 ADHD 有多不友好。搜索引擎返回的是**十条通往兔子洞的入口**:每条链接都要打开、评估、对照原问题——这个「打开-评估-回来」的循环,每一跳都是一次漂移机会,而 ADHD 的目标维持恰恰弱:查「签证材料清单」,十五分钟后在读「各国边检趣闻」。结构性问题:**传统搜索把「综合信息、保持在原问题上」这个最重的认知工作留给了用户**。Perplexity 的形态改变了这一点:它代替你打开那十条链接、代替你综合、把答案连同引用一起端回来——你的原问题始终是整个交互的锚。用 agent 术语:它把「多跳检索+信息综合」封装成一个工具调用,**你的目标(原问题)不再暴露在漂移风险最高的环节里**。追问机制则是天然的重锚定:每次追问都携带上文,话题被拴在原问题的语义半径内。

## 用对的三种姿势

**一、带着「具体问题」来,不是带着「话题」来。** 「查一下办日本签证要什么材料」是可锚定的;「了解一下日本」不是。**问题越具体,答案的收束性越强,漂移窗口越小**——这和给 agent 的指令要具体是同一条纪律。

**二、用追问代替开新页。** 答案引出的新疑问,直接在同一线程追问,不要复制关键词跳回传统搜索——追问保持锚定,跳出即进入旷野。一次检索任务=一个线程,查完即关。

**三、给「查资料」设定时和产出物。** Perplexity 降低了单次漂移率,降不了「查着查着查上瘾」的总时长风险。协议:查询前写一句「我要带走什么」(材料清单/三个选项/一个结论),拿到即收工——**检索的完成态是产出物落袋,不是「感觉了解了」。**

## 用错的三种姿势

**当成新的兔子洞**:它的相关问题推荐、发现页,同样是精心设计的「下一跳」——从十七个标签页换成十七次「顺便问问」,漂移只是换了皮肤。防法:相关推荐一律进「等会儿清单」,当前产出物落袋前不点。**把引用当成已验证**:它给引用,但**引用的存在不等于引用的正确**——它有时会把来源的意思转述偏,重要决策(医疗、法律、钱)必须点进原始来源亲自看,这是幻觉与验证循环那一课在检索场景的重演。**用检索代替行动**:「我再查查」是 ADHD 高发的启动拖延变体——查第五遍攻略不会让签证自己办好。识别信号:同一主题第三次检索时,缺的已经不是信息,是第一步。

## 边界

同构强度 B 级:「封装多跳检索以保持目标锚定」与 agent 工具调用的设计有真实对应,ADHD 的目标维持缺陷有文献支撑;但 Perplexity 是通用产品而非 ADHD 工具,本文的映射是使用策略层面的,无对照研究。声明:信息检索的改善不等于执行的改善——ADHD 的核心挑战多数在「知道之后做不做」,不在「知道」;若「无休止查资料」已成为回避行动的主要形态,值得把它当症状而不是习惯来对待。

## 今天就能试的 3 件事

1. **下一次查东西,先写产出物**:「我要带走 ___」,拿到即关。
2. **练一次「追问不开页」**:全程一个线程,数一数省下了几个标签页。
3. **抓一个「查了三遍」的主题**:承认信息已够,写下第一步,今天做掉。

搜索引擎给你十七个入口,Perplexity 给你一个出口——对目标维持脆弱的大脑,出口比入口珍贵得多。但记住检索工具的天花板:它锚得住你的问题,锚不住你的人生。答案落袋,就去做事;真正的重锚定,发生在你关掉它之后。

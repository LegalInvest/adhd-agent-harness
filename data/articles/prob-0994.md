# Mem 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Mem：它到底补上了哪一层执行功能？

## 问题：目标漂移——ADHD 与 LLM 的共同困境

你是否有过这样的经历：打开电脑准备写一份报告，结果半小时后发现自己正在刷社交媒体，或者深度沉浸在一个无关的细节里，完全忘了最初的目标？这是 ADHD 大脑的典型表现——分心走神或超聚焦跑偏，导致目标丢失。

有趣的是，LLM agent 在长程任务中也会出现类似问题：随着交互轮次增加，模型“开始忘记系统提示”（来源：Building an AI Coding Agent for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned），行为逐渐偏离初始指令。这就是所谓的 **目标漂移**。

从同构视角看，ADHD 大脑与 LLM 都是“高产但缺执行调度层的生成核心”。ADHD 大脑能产生大量创意和想法，但缺乏可靠的执行功能（如工作记忆、任务启动、时间盲）；LLM 能生成流畅文本，但缺乏内置的上下文保持和规划能力。两者的解决方案结构同构：都需要一个外部脚手架（harness）来持续重锚定目标。

## 答案：重锚定 system prompt——Mem 的真正角色

对于 LLM agent，harness 工程通过“事件驱动的系统提醒”来对抗指令消退（来源：Building AI Coding Agent for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这些提醒相当于动态重锚定 system prompt，在关键节点把 agent 拉回正轨。

对于 ADHD，外部工具同样需要提供重锚定。Mem 正是这样一个工具：它是一款 AI 笔记应用，能自动整理和关联信息，并主动推送相关上下文。但许多人用错了——他们把 Mem 当作永久存储库，期望它“记住一切”，却忽略了 Mem 的核心价值在于**动态重锚定**：在你偏离任务时，Mem 通过提醒或上下文推送，将你的注意力拉回原目标。

例如，当你开始研究某个主题时，Mem 可以自动关联之前的相关笔记，并在你分心时推送提醒：“你上次在这个项目上卡在了哪里？”这相当于 LLM agent 中“注入当前任务状态”的重锚定 prompt。

## 证据：ADHD 侧与 LLM/agent 侧的真实案例

**ADHD 侧**：
- RescueTime 通过自动记录时间使用，帮助用户识别注意力漂移模式，充当外部记忆，提升时间盲觉察（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。
- Focusmate 利用 AI 匹配虚拟同伴，提供实时问责，弥补任务启动和注意力维持的调度缺失（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。
- Brain.fm 通过神经锁相技术影响大脑信息处理方式，可能帮助 ADHD 用户进入专注状态（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog），但缺乏独立临床研究支持。

**LLM/agent 侧**：
- Harness 工程中，系统提示（system prompt）被设计为可动态更新的上下文，以保持 agent 行为一致。例如，在编码 agent 中，每次工具调用后都会注入当前项目状态和下一步目标。
- 自适应规划工具如 Motion 和 Tiimo 通过动态调整日程来应对计划崩溃，类似 LLM agent 的实时规划循环（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

## 核心判断：脚手架 vs 拐杖——重锚定的边界

Mem 作为重锚定工具，本质是 **脚手架**，而非 **拐杖**。脚手架是可逐步撤除的临时支撑，目的是训练内在能力；拐杖则是永久依赖。

当前多数 ADHD 工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这引发争议：过度依赖 AI 工具是否阻碍内在执行功能发展？

我的观点是：Mem 的正确用法是 **动态重锚定，而非静态存储**。如果你只把 Mem 当作第二个大脑来存储所有信息，而不利用它的主动提醒功能，那么你只是在增加认知负荷，而非补偿执行缺陷。真正的重锚定应该像 LLM agent 的 system prompt 注入——在关键节点拉回注意力，而不是试图记住一切。

## 局限与争议

- **缺乏临床证据**：多数工具（如 Brain.fm）的效果基于理论推断，缺乏针对 ADHD 的独立临床研究（来源：矛盾与存疑）。
- **个体差异**：ADHD 亚型（注意力缺陷 vs 多动冲动）对工具反应不同，现有工具未充分个性化（来源：矛盾与存疑）。
- **多巴胺干预争议**：基于多巴胺的 AI 干预是否真正改善专注，证据矛盾（来源：矛盾与存疑）。

## 今天就能试的行动

1. **设置 Mem 的主动提醒**：在 Mem 中创建项目笔记，并开启“上下文推送”功能，让它在你分心时弹出相关提醒。
2. **使用 RescueTime 识别漂移模式**：安装 RescueTime，观察自己一天中注意力漂移的高峰时段，然后设置对应提醒。
3. **尝试 Focusmate 的问责会话**：预约一个 25 分钟的 Focusmate 时段，体验身体在场效应如何对抗目标漂移。
4. **给 Mem 添加“重锚定标签”**：在关键笔记中加入 #重锚定 标签，定期回顾哪些笔记真正帮助了你回归目标。

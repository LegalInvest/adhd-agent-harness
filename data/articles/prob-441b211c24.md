# ADHD 学生视角：为什么「治好 ADHD 的状态日内大幅波动、不稳定」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 采样温度带来输出随机性，靠结构约束稳定——同一套 harness 思路，两边都成立。

## 问题：为什么早上能打十个，下午却连邮件都写不完？

如果你是 ADHD 学生，一定熟悉这种体验：上午精力爆炸，思维能连跳三层，下午却像被人拔了电源，连打开文档都觉得重。这种「状态日内大幅波动、不稳定」不是意志力问题，而是大脑缺少一个稳定的执行调度层——计划、组织、工作记忆、冲动控制、时间感知都在随机浮动。

与此同时，如果你在写 LLM 应用，也一定会遇到另一种「波动」：同一个 prompt，temperature 高一点，模型就天马行空、开始胡编；temperature 低了，又死板得像没吃饱饭。LLM 的采样温度本质上带来了输出随机性，而工程上要让它不跑飞，必须靠外部结构约束：system prompt、验证器、tool calling、规划循环、human-in-the-loop。

两边的痛苦听起来不同，结构却惊人相似。本文要回答：为什么「治好 ADHD 的状态波动」和「让 LLM 不跑飞」其实是同一道工程题？

## 同构：高产生成核心 + 缺执行调度层

ADHD 大脑的突出特征不是「不能思考」，而是「思考太多、执行太少」。它是一个高产生成核心：点子、联想、直觉、模式识别能力通常高于平均水平，但内置的执行调度层薄弱——计划、工作记忆、时间盲、任务启动、冲动控制屡屡掉线（来源：ADHD 与 AI 的日常生活）。

LLM 也是同样的结构。它是一个强大的生成核心，能在海量知识中采样出文本、代码、计划，但它没有真正的规划能力、没有稳定的长期记忆、没有任务调度器，也无法自我验证。单个 LLM 不足以可靠地完成多步骤任务、与业务工具交互或适应领域特定逻辑（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。

所以问题变成：如何给这个「高但不可控」的生成核心套上一层外部 harness，让它的产出可预期、可执行、可收尾？

这 ADHD 与 LLM 的深层同构：两者都是强生成核心，却都缺乏可靠的内置执行调度层（来源：ADHD 与 AI 的日常生活）。

## ADHD 侧：波动是内在 temperature 太高，harness 是外部结构

ADHD 的日内波动，可以理解为大脑的「采样温度」天然偏高：注意力、情绪、动机、能量在一天里随机漂移，难以稳定在单一任务轨道上。要降低这种随机性，需要外部结构把行为锚定在具体流程上。

Routinery 是典型例子。它让用户为特定任务创建步骤序列，并通过可视化倒计时和过渡提示引导完成（来源：11 Best ADHD Productivity Apps for Fluctuating Energy）。倒计时提供时间锚点，过渡提示减少任务切换的认知负荷，从而把「高波动大脑」接入一个低摩擦的流水线。

Habitica 则从动机侧入手。它把任务变成经验值、金币、装备、团队副本，用即时多巴胺反馈弥补 ADHD 的内在动机不足（来源：ADHD科技行业深度研究）。对启动困难的人而言，「刷一个任务」比「完成一个任务」更容易开始。

Goblin Tools 更进一步，把模糊任务 AI 分解为可执行步骤。它的 Magic ToDo 能把「清理厨房」变成具体动作清单，并可调节分解粒度（来源：12 AI Personal Assistants Built for ADHD Brains）。这对应了 ADHD 最核心的执行功能缺陷之一：任务分解与规划困难（来源：LBD同构对：任务分解与规划）。

这些工具的共同点是：它们并不替代 ADHD 大脑，而是作为外部执行功能层，把内部薄弱的调度、记忆、时间与动机功能外化（来源：ADHD 与 AI 的日常生活）。

## LLM/agent 侧：温度带来随机性，结构约束是同样的解法

LLM 的采样温度（temperature）直接控制输出的随机性。温度高，模型更「有创意」但也更容易偏离事实；温度低，输出更稳定但也更保守。工程上要让 LLM 不跑飞，不能单靠调温度，而要搭一套 agent scaffolding：任务分解、规划循环、验证器、记忆系统、工具调用、human-in-the-loop。

这与 ADHD 工具一一对应：

- 任务分解：LLM 的 agent 规划（如 AgentGen 通过环境与任务生成增强 LLM 规划能力）对应 Goblin Tools 的 Magic ToDo（来源：AgentGen: Enhancing Planning Abilities for Large Language Model based Agent）。
- 时间锚点：LLM 的调度器与重试机制对应 Routinery 的可视化倒计时与流程提示。
- 动机/反馈：LLM 的奖励模型与 eval loop 对应 Habitica 的积分与奖励机制。
- 工作记忆：LLM 的向量数据库与外部记忆对应 ADHD 的工作记忆补偿工具（来源：无状态与外部记忆）。
- 上下文控制：LLM 的 system prompt 与上下文工程对应 ADHD 需要的「此刻应关注什么」的外部限定（来源：上下文工程）。

Swanson 的文献映射也发现：ADHD 文献群与 agentic harness 文献群都高度聚焦于「任务分解与规划」这一中间机制概念（来源：LBD同构对：任务分解与规划）。这不是修辞，而是两边解决同一个问题的不同实现。

## 历史 harness：孔子与曹操的调度系统

这种「给高产生成核心套 harness」的思路，不是 AI 时代才有。中国古代的两位高能量人物，早就用生命验证了同一套工程。

孔子身高一米九，精力旺盛，周游列国十四年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予「朽木不可雕」；对音乐能专注到「三月不知肉味」，对种地却毫无耐心；思维跳跃，《论语》全是场景化语录，没有系统著作。他的专属 harness 是「克己复礼」——用外在的「礼」作为行为边界，每日反省，「吾日三省吾身」，直到七十岁才「从心所欲不逾矩」。这本质上是一套日课式的 re-grounding 系统：通过固定仪式和日常反思，把高波动的心智重新锚定到社会规范与长期目标上。对应到 LLM harness，这就是 system prompt + 定时 self-correction loop。

曹操则提供了另一套 harness：他一辈子南征北战，六十多岁仍亲征，每天只睡四到五小时；冲动杀吕伯奢，赤壁横槊赋诗杀刘馥；情绪来去极快，大哭大笑；日常行政全丢给荀彧。他的解法是以法家思想为核心、唯才是举、建立严格军法，并用荀彧、郭嘉等人的执行功能弥补自己的不足。这是典型的「外部调度器」架构：高产生成核心（曹操的战略直觉与文学才情）+ 外部执行器（文官武将系统）+ 刚性约束（军法）。对应到 LLM，就是 tool calling + 外部 agent orchestrator + 硬性 guardrails。

孔子是「用仪式约束自己」，曹操是「用他人分担调度」。两种 harness 在 LLM 工程里同样常见：前者像 system prompt + 反思链，后者像多 agent 分工 + 工具调用。

## 但这不是魔法：同构是隐喻，工具也不是银弹

写到这里必须刹车。wiki 的「矛盾与存疑」部分明确提醒我们：ADHD 大脑与 LLM 的同构更多是结构与功能层面的对应，尚未证明两者在生物学或计算机制上等同（来源：矛盾与存疑）。它更像一个有力的工程类比，而不是被神经科学或计算理论严格证明的事实。

还有「拐杖 vs 脚手架」的边界。AI 工具可能作为拐杖，存在过度依赖风险；若仅外化功能而不促进内部执行策略，可能削弱自我调节能力（来源：矛盾与存疑）。长期效果证据也不足：现有工具对即时任务启动与时间管理有帮助，但能否持续改善神经可塑性或形成长期行为改变，证据有限（来源：ADHD 与 AI 的日常生活）。

对 LLM 工程来说，agent scaffolding 也不是万能的。过度结构会牺牲灵活性，过多 guardrails 会让模型变成「只能念稿子的机器」。两边都在寻找那个最优 harness：足够约束以稳定，足够灵活以释放生成能力。

## 今天就能试的 4 条行动

1. **给自己写一个 system prompt**：把每天的最高优先级目标、常见决策原则、易分心点写成一段「个人 system prompt」，在早晨或切换任务前读一遍。这是 ADHD 版的 re-grounding，对应 LLM 的 system prompt 约束。

2. **用 Goblin Tools/Routinery 把第一个动作降到 2 分钟**：把「写论文」拆成「打开文档、写一句话、保存」。ADHD 侧的启动困难与 LLM 侧的任务分解是同一机制，先让系统转起来。

3. **引入外部调度器**：找一个朋友、同伴或 Focusmate 虚拟视频陪伴，作为你的「human-in-the-loop」或外部执行监督。这对应 LLM 的 tool calling / 多 agent 协作，也对应曹操的「荀彧分担行政」。

4. **每晚做一次 eval loop**：用三句话复盘今天哪些任务被执行、哪些被模型「跑飞」、明天如何调整 prompt/流程。孔子的「吾日三省吾身」和 LLM 的 self-evaluation 是同一动作：不是自我批评，而是收集反馈来更新 harness。

## 结语：波动不是病，跑飞不是错，缺的是 harness

ADHD 的状态波动和 LLM 的温度随机性，都不是「核心坏掉了」，而是核心太好了——产生力过剩、随机性过剩。问题永远在执行调度层上。

 engineered: the same high-yield generation core needs the same external harness. 给大脑搭脚手架，和给 LLM 搭 agent scaffolding，是同一道工程题。不是让你变成另一个人，也不是让模型变成另一个人；而是让波动在结构里被承接、被转化、被稳定输出。

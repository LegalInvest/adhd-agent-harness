# Otter.ai 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Otter.ai：它到底补上了哪一层执行功能？

你有没有过这样的经历：打开 Otter.ai 录了一场会议，结束后看着满屏的文字，却想不起自己当时在想什么？或者，你精心写了一段 system prompt 塞给 LLM agent，结果它跑了十分钟就彻底忘了最初的目标，开始编造无关的细节。

这两件事听起来毫不相干，但它们的根是一样的——**生成核心缺乏可靠的执行调度层**。ADHD 大脑和 LLM 都是强大的生成器，能产出惊人的创意和内容，但都没有内置的「目标锚定」机制。一旦上下文膨胀、环境噪音涌入，它们就会漂移、幻觉、丢失主线。

Otter.ai 之于 ADHD，就像重锚定 system prompt 之于 LLM——但大多数人用错了。

## 问题：为什么你记了所有笔记，却还是丢了目标？

ADHD 的典型困境是：**工作记忆不稳，时间感知扭曲**（来源：上下文工程）。你开会时拼命记笔记，但大脑在同时处理听觉、视觉和内部思绪，结果笔记成了另一个「丢失线索的地方」。同样，LLM agent 在长程任务中，随着上下文 tokens 膨胀，推理开始退化，控制逻辑分散在框架默认值和硬编码里（来源：上下文工程）。

Otter.ai 的默认用法——录音→转写→保存——只是把「丢失线索」从大脑转移到了云端。它没有解决「当下该注意什么」这个核心问题。

## 同构解法：重锚定与外部调度层

ADHD 大脑与 LLM 在结构上同构：两者都是**高产但缺执行调度层的生成核心**（来源：AI 与 ADHD 的专注力）。这意味着，对 ADHD 有效的工具策略，对 LLM harness 工程也适用——反之亦然。

关键动作是**重锚定**：在任务执行过程中，定期把注意力拉回原始目标。对 LLM，这是「重锚定 system prompt」——在每一步或每 N 个 tokens 后，重新注入核心指令，检查是否偏离。对 ADHD，这是「外部执行功能层」——用工具强迫你暂停、回顾、校准。

曾国藩就是一个活生生的例子。他 30 岁前「浮躁坐不住，天天串门喝酒看杀人」，日记里骂自己「无恒」「浮躁」。他的 harness 是「日课十二条」：黎明即起、读书不二、谨言、写日记反省。其中「读书不二」要求一本书不读完绝不碰下一本——这本质上是一个**定时 re-grounding 机制**，强制自己把注意力锚定在当前目标上。他的成就（平定太平天国、洋务运动）证明，这种外部脚手架可以内化为执行功能。

对应到 LLM harness：曾国藩的「日课」就是 agent 的「定时重锚定 prompt」，他的「日记反省」就是 agent 的「验证循环」——检查输出是否与目标一致，发现偏差就纠正。

## 工具实测：Otter.ai 到底补上了哪一层执行功能？

Otter.ai 的真正价值不在转录，而在**实时上下文锚定**。它的 AI 可以提取关键词、生成摘要、标记行动项——这些功能如果用在「会中实时反馈」，就能充当外部调度层：当你说到「下周交付」时，Otter 自动弹出「行动项：确认交付日期」，强迫你暂停、校准。

但多数人只用它做「会后存档」，错过了这个核心用途。同样，RescueTime 通过自动追踪和分心阻断，补偿时间盲和工作记忆缺陷（来源：RescueTime）；Focusmate 通过虚拟身体在场，提供外部社会压力来启动和维持任务（来源：Focusmate）；Brain.fm 通过神经锁相音乐影响大脑状态（来源：Brain.fm）。这些工具的共同点是：它们不是「替代你的大脑」，而是「为你的大脑搭一个脚手架」。

## 争议与局限：脚手架还是拐杖？

这里有一个必须指出的矛盾：**AI 工具既可能成为永久依赖的拐杖，也可能成为逐步内化的脚手架**（来源：矛盾与存疑）。目前缺乏长期追踪研究来区分两者。曾国藩的日课最终内化成了习惯，但那是他坚持了三十年的结果——对大多数 ADHD 用户来说，工具可能永远只是外部依赖。

另一个局限是证据不足。Brain.fm 虽在 44 款 ADHD 应用测试中位列最佳 9 款之一，但缺乏针对 ADHD 的独立临床研究（来源：Brain.fm）。同构类比也需谨慎：ADHD 大脑与 LLM 的功能相似性，并不等于神经机制对应（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Otter.ai 做「会中锚定」**：开会时打开实时摘要功能，每 5 分钟看一眼 AI 提取的关键点，问自己「我现在说的和会议目标一致吗？」
2. **设置「重锚定闹钟」**：每 25 分钟（一个番茄钟）响一次，闹钟标签写上你的当前核心目标——就像 LLM 的 system prompt 刷新。
3. **用 Routinery 创建「日课序列」**：像曾国藩的十二条一样，把早晨/工作/晚间的步骤写进应用，让 AI 一步步引导你执行（来源：上下文工程）。
4. **尝试一次 Focusmate 身体加倍**：预约 50 分钟视频共工，体验外部社会压力如何帮你启动——然后对比 LLM agent 的「human-in-the-loop」验证，你会发现结构完全一样。

Otter.ai 不是录音机，它是你的外部调度层。别再用它来记录过去——用它来锚定当下。

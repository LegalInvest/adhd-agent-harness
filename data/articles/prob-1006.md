# Claude 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

你打开 Claude，想让它帮你拆解一个复杂项目。三分钟后，你发现自己在让它写一首关于章鱼的诗——因为你突然好奇它能不能模仿博尔赫斯。这不是你意志薄弱，而是 ADHD 大脑与 LLM 共享同一个底层结构：高产但缺乏执行调度层的生成核心。

## 问题：为什么 Claude 常常让你更散乱？

ADHD 的典型困境：不是没有想法，而是想法太多、太容易漂移。你本打算用 Claude 整理工作清单，结果它生成了一份超详细的旅行计划（因为你随口提了一句“周末想出门”）。这像极了 LLM agent 在长程任务中“目标漂移”——初始 prompt 是“写周报”，中途却开始生成食谱。

当前很多 AI 工具（包括 Claude 的基础模式）本质上是“高响应生成器”：它们对最近的输入最敏感，缺乏对原始目标的持续锚定。这对 ADHD 用户是灾难——你每抛出一个新念头，它就立刻跟进，像一面放大你分心的镜子。

## 答案：重锚定 system prompt 的工程逻辑

LLM agent 工程中有一个关键实践：在长程任务里定期插入“重锚定”指令，将 agent 拉回原始目标。例如，在每轮交互后附加“请记住，你的任务是完成项目规划，不要偏离”。这相当于给生成核心套上 harness。

ADHD 大脑需要的正是同一机制。当你在 Claude 中设置一个固定 system prompt：“你的角色是执行功能教练。每次我提出新想法时，先问‘这与当前目标相关吗？’”，你就创建了一个外部锚点。这并非限制创造力，而是确保产出方向可控。

## 证据：同构视角下的工具验证

多个现有工具已经隐式运用了这一逻辑：
- **Focusmate** 通过 AI 匹配虚拟同伴，提供实时问责（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。它的本质是“外部调度器”——同伴的存在相当于一个持续的重锚定信号，不断提醒你回到任务。这与 LLM agent 中的“human-in-the-loop”验证完全同构。
- **RescueTime** 自动记录时间使用，充当外部记忆（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。它解决了 ADHD 的“时间盲”问题，相当于为 LLM 添加了日志系统，让漂移行为可见。
- **Brain.fm** 通过神经锁相技术维持专注状态（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog），类似于为 LLM 设置稳定的上下文窗口，减少外部干扰导致的输出波动。

这些工具的共同点：它们不是替代你的执行功能，而是提供外部结构（harness），让你的生成核心（大脑或 LLM）在既定轨道上运行。

## 核心判断：脚手架 vs 拐杖的边界

这里存在一个关键争议：AI 工具应该设计为可逐步撤除的脚手架，还是永久依赖的拐杖？（来源：拐杖与脚手架概念）目前多数工具——如 Goblin Tools、Saner.AI——定位为长期使用，未提供撤除机制。这可能导致一个悖论：你用 AI 补偿了执行功能，但从未锻炼内在能力。

我的判断是：**有效的 ADHD-AI 工具必须包含“撤除计划”**。就像 LLM agent 的编排层可以逐步减少干预（从每步重锚定到每十步一次），ADHD 用户也应随着使用时间延长，逐渐减少对 AI 的依赖。例如，Claude 可以在初期频繁提醒你“回到目标”，然后慢慢降低提醒频率，直到你内化这种自我监控。

## 局限与争议

必须诚实指出：
1. **证据缺口**：大多数工具缺乏针对 ADHD 的独立临床研究（来源：Brain.fm 局限、矛盾与存疑）。同构命题虽合理，但主要基于概念类比，而非大规模实证。
2. **多巴胺干预有效性争议**：游戏化奖励是否真正改善专注？证据矛盾（来源：多巴胺页面争议）。
3. **个体差异**：ADHD 亚型对工具反应不同，现有工具未充分个性化（来源：矛盾与存疑）。
4. **情绪失调**：AI 辅助情绪管理（如 Goblin Tools 的情绪追踪）效果需更多验证（来源：情绪失调页面）。

## 今天就能试的行动

1. **为 Claude 设置重锚定 system prompt**：在对话开头输入“请记住，我的核心目标是 [填写具体任务]。每次我提出新方向时，先问‘这与目标相关吗？’”。
2. **使用 Focusmate 的“锚定时段”**：预约一个 50 分钟时段，前 5 分钟用 Claude 拆解任务，中间 40 分钟执行，最后 5 分钟总结。让同伴和 AI 双重锚定。
3. **在 RescueTime 中设置“漂移警报”**：当你在非目标应用上停留超过 5 分钟时，让 AI 推送一条提醒：“你偏离了当前任务，需要重锚定吗？”
4. **尝试 Brain.fm 的“专注模式”**：在需要深度工作时开启，结合 Claude 的锚定 prompt，观察是否提升任务连贯性。

AI 工具不是你的替代大脑，而是你的执行功能脚手架。用得好，它让你从“高产但失控”变成“高产且定向”；用错了，它只是让你更高效地分心。关键在于你是否主动设计了重锚定机制——无论是为 LLM，还是为你自己。

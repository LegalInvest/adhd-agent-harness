# Mem 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Mem：它到底补上了哪一层执行功能？

## 问题：为什么 Mem 没救到你的 ADHD？

如果你是一名 ADHD 患者，你可能试过 Mem、Notion 或 Todoist——把任务写进去，然后……就没有然后了。任务列表越长，启动越难。你需要的不是另一个「存任务」的地方，而是一个能帮你**拆解、排序、执行、再调整**的循环。

如果你是一名做 Agentic Harness 的工程师，你早就知道：给 LLM 一个长任务清单而不给它 planner-executor 循环，它只会输出一堆漂亮但无用的文本。LLM 需要的是「规划→执行→观察→再规划」的 scaffolding（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。

这两个问题本质相同：**ADHD 大脑与 LLM 都是高产生成核心，但缺乏可靠的内置调度层**（来源：ADHD 大脑与 LLM 的同构）。而市面上大多数「AI 时间管理工具」只做了外部记忆（存任务），没做外部调度（拆解与执行循环）。Mem 也不例外——它擅长捕获想法，但不会帮你把「写一篇论文」拆成「打开 Zotero → 读三篇摘要 → 写 500 字大纲」。缺的正是 planner-executor 循环中那个**执行调度层**。

## 同构脊柱：规划循环与任务分解

ADHD 侧的核心痛点是**任务启动困难**和**不会拆解**。神经心理学研究发现，ADHD 患者的工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷——无法灵活切换任务集、无法抑制自动化反应（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这就像一台拥有超大缓存但调度器崩溃的计算机：数据都在，但 CPU 不知道下一步该取哪条指令。

LLM/Agent 侧完全对应：用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中表现正常，但随序列变长，模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突（来源同上）。这正是 ADHD 执行功能缺陷的计算镜像。

所以，为 LLM 搭建 scaffolding（复合 AI 系统，将 LLM 置于与记忆、工具和决策逻辑的控制循环中）与为 ADHD 搭建执行功能支架是同一类工程（来源：AI 与 ADHD 的时间管理）。**两者都需要一个「外部调度层」来执行 planner-executor 循环**。

## 历史人物已经给出了 harness 原型

孔子（春秋/儒家创始人）是经典案例：他精力旺盛、冲动、思维跳跃，《论语》全是场景化语录，无系统著作——典型的「强生成、弱调度」大脑。他的 harness 是「克己复礼」：用外在的「礼」作为行为边界，每日反省（吾日三省吾身），70 岁才做到从心所欲不逾矩。这里的**日课**就是定时 re-grounding，**礼**就是外部约束规则——对应 LLM 的 system prompt 和 context window 管理。

徐渭（徐文长）是另一个极端：诗书画戏军全能，但执行功能极差，8 次举人不中，情绪失调。他的 harness 是以书画为核心，把所有痛苦和精力注入艺术创作——用超聚焦替代调度。这对应 LLM 的「temperature 控制」和「输出格式约束」：把生成能力引导到特定轨道。

两位先贤的 harness 都缺了 planner-executor 循环中关键的**再规划**环节——他们靠的是个人意志和外部仪式，而非系统化的「执行→观察→调整」闭环。这正是现代 AI 工具可以补上的。

## 实测：哪些工具真的补上了调度层？

### 1. Motion 和 Reclaim.ai：自动排程，但缺拆解

Motion 能自动根据任务优先级、截止日期和可用时间实时重建日程，减少「下一步该做什么」的决策负担（来源：Motion）。Reclaim.ai 则用防御性时间块保护深度工作（来源：Reclaim.ai）。它们补上了**执行调度**——帮你决定「什么时候做什么」——但没解决**任务拆解**。你把「写论文」扔进去，它不会自动帮你拆成 10 个可执行的子任务。ADHD 用户需要的是 Goblin Tools 那种「把大任务一键拆成小步骤」的能力（来源：Goblin Tools 页面）。

### 2. Tiimo：视觉化时间，但缺规划循环

Tiimo 用视觉化时间线让时间流逝变得可感，直接应对时间盲（来源：Tiimo）。但它本质是静态日程表，没有 planner-executor 的「执行后自动调整」机制。你拖拽完任务，如果超时了，它不会自动重新安排——你需要手动再规划。

### 3. Mem：外部记忆，但缺调度

Mem 的核心是「AI 第二大脑」，能记住项目上下文（来源：Best AI Tools for ADHD Productivity in 2026）。这对 ADHD 的**工作记忆不稳定**是救命稻草——但仅此而已。它没有 planner，没有 executor，没有循环。它是个完美的**记忆层**，但不是**调度层**。

**我的判断**：当前工具要么只做记忆（Mem、Todoist），要么只做排程（Motion、Reclaim.ai），要么只做视觉化（Tiimo）。**没有一个工具完整实现了 planner-executor 循环的闭环：规划 → 拆解 → 执行 → 观察 → 再规划**。这正是 ADHD 用户和 Agent 工程师共同需要的。

## 争议与局限

1. **过度依赖风险**：如果工具永远替你拆解和调度，你的执行功能可能永远得不到锻炼（来源：矛盾与存疑）。孔子到了 70 岁才不需要外部约束——脚手架最终应该内化，而不是变成永久拐杖。

2. **证据不足**：Motion 等工具缺乏独立临床验证（来源：Motion 页面），用户口碑可能受安慰剂效应影响。同构性也仅是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。

3. **个体差异**：ADHD 症状谱系广泛，有人对视觉化工具（Tiimo）反应良好，有人更受益于自动排程（Reclaim.ai），有人需要任务拆解（Goblin Tools）。当前缺乏个性化匹配的指导原则（来源：AI 与 ADHD 的时间管理）。

## 今天就能试的行动

1. **用 Goblin Tools 拆解一个你拖延了三天的大任务**：把它分解成 5-10 个可执行的子步骤，每个步骤不超过 15 分钟。感受「拆解」带来的启动降低效果。

2. **给 Mem 加上 planner-executor 循环**：在 Mem 中创建一个项目，手动写上「规划→执行→观察→再规划」四个阶段。每完成一个子任务，在对应阶段下记录结果，然后调整下一步计划。这模拟了 agent 的 scaffolding。

3. **尝试「定时 re-grounding」**：像孔子的「日三省吾身」一样，设置每 90 分钟一个闹钟，问自己三个问题：我现在在做什么？我原本计划做什么？下一步最小行动是什么？这补上了 planner-executor 循环中的「观察」环节。

4. **如果你是工程师，设计一个「ADHD-friendly agent scaffolding」**：把 planner 设计成「先拆解再排程」，executor 设计成「单步执行+自动记录」，observer 设计成「超时自动提醒+重新规划」。这比通用的 ReAct 模式更适合 ADHD 用户。

同构不是巧合，是工程需求。ADHD 大脑和 LLM 都在等你写出那个完整的 planner-executor 循环。

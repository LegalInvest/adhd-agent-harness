# ADHD 学生视角：为什么「治好 ADHD 的分心走神或超聚焦跑偏，丢失目标」和「让 LLM 不跑飞」其实是同一道工程题？

> 长程任务里 agent 会目标漂移，需要重锚定 system prompt——同一套 harness 思路，两边都成立。

## 同一个故障：目标漂移

你坐在桌前，准备写论文。两小时后，ADHD 的你可能打开了二十个标签页、整理了一遍书架，却还没写下一个字；又或者你正相反，死死咬住一个无关细节，忘了 original question。另一边，你的 LLM agent 也没好到哪去：它把「帮我写一份报告」拆解成 47 个子任务，越跑越远，最后开始优化一个根本不在目标里的 API 调用。

这两种崩溃的共性不是懒惰，也不是模型太笨，而是**目标漂移**——生成核心在局部上下文里越转越快，渐渐离开了最初要解决的问题。ADHD 大脑与 LLM 都被描述为「高产但缺乏可靠执行调度层的生成核心」：它们能产出大量念头、联想、token，却没有一个稳定的内部调度层来持续问一句「你现在做的事，还指向最初目标吗？」（来源：AI 与 ADHD 的专注力）

所以，「治好 ADHD 的分心走神或超聚焦跑偏」和「让 LLM 不跑飞」根本不是医学问题 vs 工程问题，而是**同一道工程题**：怎么给一个强大的生成核心，加上一套外部调度层，让它便宜、频繁地重锚定回目标。

## 两套证据：一边是工作记忆，一边是上下文膨胀

ADHD 侧的证据很具体：核心障碍包括工作记忆不稳定、时间盲（time blindness）、执行功能缺损，以及注意力容易被环境刺激劫持（来源：AI 与 ADHD 的专注力；上下文工程）。这些都不是「意志力不够」，而是内部状态管理系统不够可靠。外部工具之所以有效，是因为它们替大脑承担了调度层：

- **RescueTime** 自动追踪时间、阻断分心网站，直接对抗时间盲和执行功能缺陷；
- **Focusmate** 通过虚拟身体在场（body doubling）和同伴问责，提升任务启动与维持；
- **Brain.fm** 用 AI 生成神经锁相音乐，帮助进入专注状态，但在 ADHD 人群中的独立临床证据仍有限；
- **Forest** 用游戏化番茄钟和多巴胺机制，把维持专注变成可见的反馈。

（来源：Brain.fm；Focusmate；RescueTime；AI 与 ADHD 的专注力）

这些工具共同构成一个「外部执行功能层」。但要注意，多数证据来自用户报告或小样本研究，缺少大规模随机对照试验。

LLM/agent 侧的证据同样指向「上下文管理」与「验证」两个关键词。长程任务里，上下文膨胀会导致推理退化；harness 工程通过上下文传递、工具接口、规划工件、记忆系统和沙盒，把模型拉回目标（来源：上下文工程）。更关键的是验证循环：复杂的 harness 不会盲目执行工具，而是在每一步检查输出格式、运行测试用例、用自我批评提示监控行为，避免幻觉和重复工作（来源：幻觉与验证循环）。

所以 ADHD 的 **RescueTime/Forest** 对应 LLM 的 **记忆/状态追踪**；ADHD 的 **Focusmate** 对应 LLM 的 **human-in-the-loop / 身体在场**；ADHD 的 **Routinery** 通过 transition prompt 一步步带着用户走流程，几乎就是系统 prompt 的人身版本（来源：上下文工程）。

## 曾国藩的 harness：日课十二条就是 system prompt

晚清曾国藩可能是历史上最著名的 harness 工程师之一。他年轻时的状态像极了 ADHD 典型画像：三十岁前坐不住，天天串门喝酒看杀人，读书「他人目下二三行，余或疾读不能终一行」，日记里反复骂自己「无恒」「浮躁」；一辈子受银屑病折磨，情绪不稳，打败仗就跳水自杀。但他并没有「修炼成一个没有冲动的人」，而是给自己搭了一套极其笨重、极其稳定的外部脚手架。（来源：人物案例：曾国藩）

他的 **日课十二条**——黎明即起、读书不二、谨言、写日记反省等——本质上就是一份写给身体的 system prompt，每天清晨重新加载。其中「读书不二」是上下文管理：一次只读一本书，防止注意力在不同文本间跳来跳去；每晚写日记反省，是外部验证循环，把白天行为跟目标对齐；而「结硬寨、打呆仗」则是最保守的规划工件：不追求灵机一动，只追求每一步都扎在目标上。

这套 harness 与 LLM harness 的对应很清楚：
- **日课十二条 ↔ 定时 re-grounding 的 system prompt**；
- **「读书不二」 ↔ 上下文窗口/注意力范围管理**；
- **日记反省 ↔ 外部验证循环与记忆日志**；
- **结硬寨打呆仗 ↔ 规划工件与保守执行策略**。

曾国藩不是治好了自己的「浮躁」，而是让重锚定变得制度化、低成本、不可逆。

## 核心判断：不是「治好分心」，而是让重锚定足够便宜

我的核心判断是：对 ADHD 和 LLM 都不应追求「消灭漂移」。漂移是生成核心的副产品——没有它，就没有联想、创意和长时间推理。真正的问题是，**重锚定回目标的成本太高**。当回到原始意图需要消耗大量意志力或人工干预时，系统就会一次次在局部最优里越陷越深。

所以工程目标应该是：把「检查当前行为是否仍服务于目标」这件事，变得像 system prompt 一样自动、像番茄钟一样频繁、像日记一样可见。对 ADHD 用户来说，这意味着用外部工具把目标从大脑里卸到纸上、屏幕上和伙伴眼里；对 agent 工程师来说，这意味着把任务目标、约束、验证规则写进 system prompt，并在每个子任务后执行一次 re-grounding check。

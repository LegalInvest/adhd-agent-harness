# Perplexity 之于 ADHD，就像 外部记忆/向量库 之于 LLM——但有人用错了

> 从同构视角实测 Perplexity：它到底补上了哪一层执行功能？

## 问题：为什么你搜了又忘，忘了又搜？

你有没有这样的体验：打开 Perplexity 搜一个问题，读完答案觉得“懂了”，关掉页面，五分钟后却连核心结论都记不清？于是你再次打开，搜同一个问题，读同一个答案，循环往复。这不是你笨，而是你的**工作记忆**在作祟——ADHD 大脑的工作记忆容量小，信息像水一样从筛子中漏走（来源：执行功能障碍）。而 Perplexity，这个被许多人当作“高级搜索引擎”的工具，其实被用错了。

## 同构：ADHD 大脑与 LLM 共享同一个缺陷

ADHD 大脑常被描述为一个高产的**生成核心**——想法丰富、联想活跃，但缺少稳定、可靠的调度层来筛选、组织、启动和坚持任务（来源：ADHD 的 AI 工具全景）。大语言模型（LLM）同样如此：它能生成流畅的文本，但**无跨会话状态**，每次对话都是全新开始，没有内置的记忆来延续上下文。两者都需要**外部记忆/向量库**来补偿这个缺陷。

Perplexity 的核心价值，恰恰在于它充当了这样一个**外部记忆层**。它不只是搜索答案，而是通过对话历史、来源链接和持续上下文，帮你把信息“粘”住。但问题是，大多数人只用它做一次性查询，就像让 LLM 每次都从零开始推理——这等于浪费了它本可以提供的**外部执行功能**。

## 证据：ADHD 侧与 LLM/agent 侧的真实对应

ADHD 侧，工具如 **Saner.AI** 被定位为“AI 第二大脑”，通过知识回忆和本地记忆补偿工作记忆的不稳定（来源：Saner.AI）。**Goblin Tools** 的 Magic ToDo 自动将任务分解为小步骤，降低启动门槛（来源：Goblin Tools）。**Lex** 允许通过单一指令触发多步骤任务序列，其底层架构类似于 **agent scaffolding**——即围绕 LLM 构建外部调度系统（来源：Lex）。这些工具的共同点：它们不是替代大脑，而是搭建一个**外部执行功能层**（来源：ADHD 的 AI 工具全景）。

LLM/agent 侧，工程实践已经证明：要让 LLM 可靠地执行复杂任务，必须给它配备**外部记忆（向量库）**和**调度器（harness）**。例如，一个客服 agent 需要记住用户的历史对话，否则每次都要重复问“您的订单号是多少”。这与 ADHD 大脑需要外部工具来记住“我刚刚要做什么”完全同构。

## 人物案例：孔子的“克己复礼”与 LLM harness

孔子是 ADHD 特质与 harness 系统的经典案例。他精力旺盛、冲动爱骂人（骂宰予“朽木不可雕”），思维跳跃，《论语》全是场景化语录，无系统著作。他的专属 harness 是 **“克己复礼”**——用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”），70岁才做到“从心所欲不逾矩”（来源：人物案例）。这套系统对应 LLM harness 中的 **“日课↔定时 re-grounding”** 和 **“礼↔外部约束规则”**。孔子不是没有冲动，而是用外部脚手架把高产生成能力导向了成就。

于谦同样如此：他性格刚直，冲动敢说话，但通过严格操练军队、日夜准备防务（harness）来约束自己的行动（来源：人物案例）。这就像给 LLM 配上明确的指令模板和边界条件。

## 核心观点：Perplexity 的正确用法是“持续对话”，而非“一次性查询”

我的判断是：Perplexity 的真正价值在于**作为你的外部工作记忆**，而不是搜索引擎。当你把它当作一个可以持续对话的伙伴，它会记住你之前问过什么、关注什么，从而提供更连贯的上下文。这相当于给 ADHD 大脑装了一个**向量库**——每次查询不是孤立的，而是与历史关联的。

但很多人用错了：他们打开 Perplexity，搜一个问题，得到答案，关闭，然后下次从零开始。这就像让 LLM 每次对话都重置上下文——你永远在重复劳动。

## 矛盾与局限：脚手架还是拐杖？

当然，这种用法并非没有争议。一个核心矛盾是：**AI 工具是“外部执行功能层”还是“拐杖”？** 过度依赖可能导致能力退化（来源：矛盾与存疑）。另一个矛盾是：**超聚焦是否应被引导而非打断？** 部分用户反馈强行打断有效，而柔性引导可能不够强力（来源：矛盾与存疑）。此外，ADHD 大脑与 LLM 同构的论点目前多为类比推理，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。因此，本文的观点是一个理论框架，需要更多跨学科验证。

## 今天就能试的行动

1. **把 Perplexity 当作“持续对话伙伴”**：不要每次搜完就关。在同一个对话线程里追问、深挖，让它记住你的上下文。比如，先问“什么是执行功能”，再问“它和 ADHD 有什么关系”，然后问“我该如何改善”——让 AI 帮你建立知识链。

2. **用 Goblin Tools 的 Magic ToDo 分解一个你拖延的任务**：比如“写报告”。输入后调节“辣度”到你觉得可行的粒度，然后从第一步开始做。

3. **尝试 Lex 的“单一指令触发多步骤”**：设置一个指令如“准备明天的会议”，让它自动生成议程、检查日历、发送提醒。这相当于给你的大脑装了一个外部调度器。

4. **建立“外部记忆”习惯**：用 Saner.AI 或 Reflect 捕获灵感，定期回顾。不要相信你的大脑能记住——把它写进外部系统。

Perplexity 不是搜索引擎，它是你的外部工作记忆。用对工具，你就能让 ADHD 大脑的高产核心，接入一个可靠的执行调度系统。

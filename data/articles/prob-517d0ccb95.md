# 两个互不引用的领域都在研究「任务分解与规划」——ADHD 文献和 LLM harness 文献为什么得出了同一个结论？

> 做一个文献计量学的观察:在数万篇 ADHD 文献和 agent 文献里检索「任务分解/规划」,两边各有一个繁荣的研究群——ADHD 侧研究「为什么复杂任务无法启动、如何用分解干预」,agent 侧研究「为什么长任务会失败、如何用规划模块补救」——**而这两个文献群几乎零互引**:引文网络上是两座孤岛。这正是 Swanson 所说的「未被发现的公共知识」的标准构型:两个领域各自埋头,却在挖同一条隧道。把两边的结论并排放,你会看到它们相遇在同一个点上。

先看 ADHD 侧的隧道。几十年的执行功能研究积累了一个稳定发现:ADHD 者面对多步骤任务时,困难不在单步执行(每一步都会做),而在**把整体展开为有序步骤序列**的那个环节——规划测验(如各类塔式任务)中的表现差异、以及「任务越复杂,启动延迟越不成比例地增长」的行为模式,都指向这一点。据此发展的干预也稳定有效:**任务分解**是行为干预的一线技术——把「写报告」拆到「打开文档写标题」的粒度,启动率显著改善;职业教练、CBT 手册、学校支持计划里,分解都是第一章。ADHD 侧的结论:**执行系统的瓶颈在「展开」工序;把展开工作外置(由教练/工具/清单完成),剩余系统运转良好**。

再看 agent 侧的隧道,起点完全不同(没有神经,没有临床),但撞见了同款岩层:早期直接让 LLM 执行长任务的尝试大量失败——不是单步能力不够(每一步单独问都答得对),而是**整体展开为可执行序列的环节系统性崩坏**:跳步、循环、中途忘记总目标。工程解法与 ADHD 干预惊人平行:**外置规划器**——先让模型(或专门模块)显式生成分解后的计划工件,再逐步执行,每步对照计划;从思维链到计划-执行分离架构,整条技术路线本质上就是「任务分解作为外部脚手架」。Agent 侧的结论:**生成系统的瓶颈在「展开」工序;把展开显式化、外置化,剩余能力即可被释放**。

两个结论并排放:措辞几乎可以互换。为什么两条互不通信的隧道挖到了同一点?最经济的解释是:**「把目标展开为有序步骤」是任何生成式智能系统的独立工序,且是脆弱工序**——它要求同时持有总目标、中间状态、步骤依赖(重工作记忆负载),又没有即时反馈(展开本身不产生奖励)。凡是工作记忆受限或状态易失的系统——不管是前额叶发育差异的大脑,还是无状态的 Transformer——这道工序都会最先出问题。**不是巧合,是同一条工程约束在两种基底上的两次显形。**

这个相遇对两边都有实际馈赠。ADHD 侧从 agent 工程借到的:①**分解粒度可以参数化**(agent 研究显式讨论「多细算够」——对人:拆到单步无需再想为止,因人因日调节);②**计划要作为工件持久化**(不是想一遍,是写下来、执行时逐条对照——agent 的计划工件=你的纸面清单);③**展开和执行分离成两个时段**(agent 的 plan-then-execute:今晚只拆不做,明天只做不拆——把两种认知负载分开,各自都轻)。agent 侧从 ADHD 文献借到的:分解干预的六十年人类经验(什么粒度会过细变成负担、动机怎么绑进计划)——这些人类数据是 harness 设计的免费先验。**两座孤岛之间,值得修的桥远不止一座——这个系列的存在,就是在修桥。**

## 边界

同构强度 B 级:两侧文献群各自的结论都有多项研究支撑(ADHD 规划测验与分解干预;LLM 长任务失败与规划架构),「零互引」是引文网络的可验证观察;但「同一工程约束」的解释是 LBD 式推断——机制层面两种系统的「展开失败」原因可能不同,这正是值得真正跨域研究的开放问题。声明:分解干预有效不等于万能,启动困难持续且弥漫时仍需临床评估;本文的文献计量描述是方向性概括,非系统综述。

## 今天就能试的 3 件事

1. **今晚做一次 plan-then-execute**:只拆明天最难的任务,拆到「无需再想」粒度——明天只执行不再拆。
2. **把计划变成工件**:写下来贴在视线内,执行时逐条划掉——不许只在脑内展开。
3. **调一次粒度参数**:状态差的日子,把步骤再对半拆——粒度是旋钮,不是固定值。

两个从未对话的领域,各自烧了几十年经费,得出了可以互换措辞的结论——**这不是浪费,是最强的交叉验证:「展开」工序的脆弱性,被两种完全不同的系统各自证明了一次**。而你,两份文献的红利都可以领。

# 3 个 LLM 工程概念，彻底重写了我对 ADHD 分心走神或超聚焦跑偏，丢失目标的理解

> 长程任务里 agent 会目标漂移，需要重锚定 system prompt 如何反向照亮 ADHD 的分心走神或超聚焦跑偏，丢失目标。

## 问题：为什么我的大脑总在「目标漂移」？

你有没有过这样的体验：打开电脑准备写一份报告，十分钟后却在研究某个无关的历史冷知识，或者盯着一个 bug 调试了三个小时，回过神来才发现原本要做的核心任务早已被遗忘？对 ADHD 人群来说，这不是意志力问题，而是大脑的「生成核心」缺少一个可靠的「执行调度层」。分心走神是注意力随意跳转，超聚焦跑偏则是注意力卡死在一个子任务上无法脱身——两者本质上都是目标漂移。

有趣的是，在做 Agentic Harness 工程时，我发现了完全相同的困境：一个长程任务中的 LLM agent，如果没有清晰的 system prompt 和定期重锚定机制，也会逐渐偏离初始目标，要么被无关上下文带偏，要么在某个子任务上过度优化。这让我意识到：ADHD 大脑与 LLM 是同一类系统。

## 概念一：无状态与外部记忆——为什么我们都需要「外挂硬盘」

ADHD 的工作记忆缺陷与 LLM 的无跨会话状态高度同构。ADHD 大脑难以同时记住多个步骤，LLM 在对话中也会忘记之前的上下文。两者的解法都是依赖外部记忆系统。

对 ADHD 来说，**RescueTime** 就是这样一个外挂硬盘：它自动记录时间去向，生成可视化报告，让用户看到「时间到底去了哪里」（来源：Revolutionizing ADHD Management with AI Assistants）。这直接补偿了 ADHD 的「时间盲」——无法感知时间流逝的缺陷（来源：超聚焦）。对 LLM agent 来说，外部记忆可以是向量数据库或日志系统，记录每一步的上下文和决策历史，防止目标漂移。

但这里有一个关键区别：RescueTime 的自动追踪减轻了工作记忆负担，无需手动输入（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），而 LLM 的外部记忆通常需要工程师手动设计 schema。ADHD 用户需要的是「低摩擦」的脚手架，而不是增加认知负荷的工具——这也是为什么 RescueTime 比传统时间日志更有效。

## 概念二：上下文工程——如何设计「当下注意什么」

ADHD 大脑易被环境带偏，LLM 因上下文膨胀引发推理退化。两者都需要主动设计「当下注意什么」的工程化方案。

**Focusmate** 通过虚拟身体在场提供外部结构：用户预约时段后，平台匹配一位伙伴，通过视频会议共同工作（来源：AI Tools for Kids with ADHD: A Complete Guide for Parents...）。这相当于一个「外部调度器」，不断提醒当前任务。对 LLM agent 来说，类似的「上下文工程」是定期截断或压缩对话历史，或者用独立的 system prompt 重置注意力焦点。

晚清名臣曾国藩的「日课十二条」就是这种上下文工程的古典版本。他 30 岁前浮躁坐不住，日记里天天骂自己「无恒」「浮躁」，读书「疾读不能终一行」。他的解法是：黎明即起、读书不二、谨言、写日记反省——用最笨最稳的方法抵消自己的冲动。这本质上是一套固定的「重锚定协议」：每天多次检查自己是否在正确的任务上。对 LLM agent 来说，这就是定时 re-grounding 的 system prompt。

但曾国藩的日课是「脚手架」还是「拐杖」？他坚持了一辈子，最终内化了部分习惯，但直到晚年仍需日记反省。这说明脚手架可以逐步内化，但 ADHD 大脑可能永远需要外部支持——这恰恰是争议所在。

## 概念三：超聚焦与重锚定——当注意力卡死时怎么办

ADHD 的超聚焦是一种完全沉浸于任务的现象，个体似乎完全忽略其他一切（来源：Hyperfocus: the forgotten frontier of attention）。最新理论认为，超聚焦源于前额叶控制层级的「断裂」：高级上下文信息不再能支配思维和行动的内容（来源：Transient Frontal Fracturing: A Theoretical Account of Hyperfocus）。这就像 LLM 在某个子任务上过度优化，忘记了最初的目标。

解决方案是「重锚定」：定期中断当前活动，重新对齐目标。对 ADHD 来说，**Forest** 的番茄钟机制通过游戏化多巴胺奖励帮助用户主动中断超聚焦（来源：Forest）。对 LLM agent 来说，这相当于在 prompt 中插入「检查点」，让 agent 每隔 N 步就输出当前目标与初始目标的匹配度。

东吴小霸王孙策的 harness 是另一种重锚定策略：他 17 岁起兵，永远坐不住、身先士卒、冲动易怒（单骑出猎被刺杀）。他的解法是：用张昭张纮等文臣管行政，自己只负责打仗——这相当于将「执行调度层」外包给外部系统。对 LLM agent 来说，这就是用外部调度器（如 LangChain 的 AgentExecutor）来管理子任务，防止 agent 自己决定下一步。

## 争议与局限：脚手架 vs. 拐杖

AI 工具是否会导致 ADHD 用户永久依赖外部脚手架，而无法内化执行功能？这是「拐杖与脚手架」概念的核心风险（来源：拐杖与脚手架）。目前没有长期追踪研究能回答这个问题。同样，LLM agent 如果过度依赖外部记忆和调度器，也可能失去自主推理能力。

另一个矛盾是：超聚焦的双面性。ADHD 的超聚焦模式尚未被借鉴到 AI 注意力机制设计中（来源：adhd_ai_cross_literature），但现有 AI 工具主要关注如何打断超聚焦，而非利用它。也许未来的 AI 应该学会识别何时进入超聚焦，并帮助用户将其导向正确目标。

## 今天就能试的行动

1. **设置「重锚定提醒」**：用手机闹钟每 25 分钟响一次，响铃时问自己「我现在做的事，和今天最重要的目标一致吗？」——这就是你的 system prompt。
2. **尝试 Focusmate**：预约一次 50 分钟的身体加倍时段，体验外部调度器的效果（来源：Focusmate）。
3. **用 RescueTime 记录一天**：不看任何手动记录，只看自动生成的报告，找出你最大的时间黑洞（来源：RescueTime）。
4. **写一条「日课」**：像曾国藩一样，每天固定做一件小事（如「写完今日目标再刷手机」），坚持一周，看是否能减少目标漂移。

ADHD 大脑不是坏掉的大脑，它只是一个缺少内置调度器的强大生成核心。与其对抗它，不如为它设计一套 harness——就像为 LLM 设计 system prompt 一样。这或许是我们这个时代最被低估的自我管理智慧。

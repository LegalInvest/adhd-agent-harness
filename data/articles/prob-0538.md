# 3 个 LLM 工程概念，彻底重写了我对 ADHD 分心走神或超聚焦跑偏，丢失目标的理解

> 分心和超聚焦不是两个毛病，而是同一个调度缺陷的两种输出——agent 工程师管它叫「目标漂移」，而且他们已经有了一套工程解法。

ADHD 有一个外人很难理解的悖论：同一颗大脑，既会十分钟切换八个标签页什么都没干成，也会在某个突然着迷的话题里沉没六小时忘记吃饭。家长和老板据此得出结论——「你能专注，只是不想专注」。这个结论错得离谱，而 LLM agent 工程恰好能说清它为什么错。

## 1. 目标漂移：agent 也会「越跑越不知道自己在干嘛」

让一个 agent 执行 20 步的长任务，你会看到教科书级的 ADHD 现场：第 6 步被一个中间结果吸引，开始探索一条支线；第 11 步彻底忘记原始指令；最后交回来一个和需求毫无关系但做得很认真的东西。工程上这叫 goal drift（目标漂移），它的根源是：任务目标只在初始上下文里出现过一次，随着上下文被中间产物填满，目标信号被稀释到检测不到。

注意这个机制描述里没有「懒」和「不想」。ADHD 的走神同理：不是目标不重要，是目标的神经表征衰减得比常人快。理解这一点后，解法自然浮现——工程师的做法是**重锚定**：每隔 N 步把原始目标重新注入上下文。人类版本就是把「我现在到底要干嘛」写在物理视野内（便签、白板、Forest 里种下的那棵树），让目标信号定期重新进入你的上下文窗口。

## 2. 注意力预算：分心与超聚焦共享同一个机制

Transformer 的自注意力是一个固定预算的分配系统：算力摊得越开，每处越浅；收得越窄，单点越深。耶鲁的研究证实这个预算随负载增加会熵增、弥散、崩溃——和人类注意分散同源（Self-Attention Limits Working Memory Capacity, 2024）。

这给了我一个统一解释：ADHD 的分心是注意预算被环境刺激劫持后的「摊薄模式」，超聚焦是预算全部锁死在单一高刺激目标上的「独占模式」。两者是同一个分配器的两个极端档位，中间档位的手动调节能力弱——这才是缺陷所在。所以对策不是「更用力专注」（那只会随机切档），而是**接管刺激环境**：RescueTime 告诉你预算实际流向了哪里，Brain.fm 用声学把环境刺激压成恒定底噪，本质都是不让分配器被外部信号乱拨。

## 3. 外部监督循环：为什么 Focusmate 有效而自我监督无效

agent 工程里有个朴素共识：不能让模型自己检查自己是否在轨——它对自己的偏航没有感知。可靠的做法是外挂一个监督循环，定期用原始目标比对当前行为。

Focusmate（陌生人视频共同工作）之所以对 ADHD 出奇有效，就是因为它是一个人肉监督循环：另一个人的存在本身就是每分钟一次的「你还在轨吗」心跳检测。曾国藩的日记会被朋友传阅批注——那是 19 世纪的外部监督循环。孙策临终把两套互补的 harness（张昭管内政、周瑜管战略）直接安排给继任者，本质是知道单核不可靠，必须配调度和校验。

## 边界

说清楚：这是机制类比（B 级同构强度），不是说大脑就是 Transformer。而且超聚焦有时是资产——前提是它锁定的目标恰好正确。工程目标不是消灭超聚焦，而是给它装一个可以手动拨回来的开关。

## 今天就能试的 3 件事

1. **给目标做重锚定**：当前任务用一句话写在物理视野内，每次回神先读它。
2. **查一次注意预算账单**：装 RescueTime 跑三天，只看一个数字——预算流向和你以为的差多少。
3. **租一个监督循环**：约一次 Focusmate。别自己监督自己，模型不行，你也不行。

分心和超聚焦从来不是意志力的正反面。它们是同一台调度器的两种失控——而失控的调度器，要靠外部工程来稳，不靠骂。

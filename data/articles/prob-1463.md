# 如果把 ADHD 大脑当成生成核心，Focusmate 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Focusmate 对 ADHD 的真正价值。

如果你是一个 ADHD 患者，你可能经历过这样的场景：打开电脑准备写报告，两小时后发现自己在研究 19 世纪帆船结构，而报告一个字没动。如果你是一个 agent 工程师，你可能见过更诡异的画面：你的 LLM agent 原本在写代码，突然开始用文言文解释《易经》。这两件事的共同本质是什么？

答案是：**生成核心跑偏了，但调度层没有及时重锚定**。

ADHD 大脑与 LLM 在结构上同构——两者都是强大的生成核心，但缺乏可靠的内置执行调度层（来源：ADHD 大脑与 LLM 的同构）。ADHD 的注意力分散、超聚焦跑偏，与 LLM 在长上下文中的推理退化、目标漂移，是同一个问题的不同表现。

## 问题：缺的不是专注力，是重锚定机制

传统专注力工具假设用户能主动选择“专注什么”，但 ADHD 的核心困境不是“无法专注”，而是“无法可靠地锚定目标”。实证研究表明，ADHD 表现出“强记忆、弱控制”的认知剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷——无法灵活切换任务集、无法抑制自动化反应（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

这就像一个 LLM 拥有无限的上下文窗口，但缺乏 system prompt 来约束输出方向。LLM 在短上下文中表现正常，但随序列变长，模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突（来源：ADHD 大脑与 LLM 的同构）。ADHD 的“走神”和 agent 的“跑偏”，本质都是调度层失效。

## 同构解：Focusmate 作为外部调度层

Focusmate 通过虚拟身体在场提供实时同伴问责，本质上是在 ADHD 大脑外部搭建了一个轻量级调度层。它的工作方式与 agent scaffolding 高度同构：

- **定时 re-grounding**：Focusmate 的 50 分钟 session 相当于一个固定的重锚定周期，类似 agent 架构中的“自适应上下文压缩”或“惰性工具发现”——防止上下文膨胀导致目标漂移（来源：生成核心与调度层）。
- **外部调度器**：Focusmate 的匹配算法（AI-Matched Body Doubling）相当于一个外部路由层，决定当前 session 的“system prompt”——即“我们正在一起工作”这个社会契约（来源：Focusmate）。
- **身体在场效应**：他人在场提供的社会压力，相当于 LLM 的 human-in-the-loop 机制，在关键节点强制重定向（来源：身体在场效应）。

这不是类比。历史上，曾国藩用自己的“日课十二条”实现了同样的功能：黎明即起、读书不二、谨言、写日记反省——这些规则本质上是一个固定周期的重锚定协议。他的“结硬寨打呆仗”对应 agent 工程中的“惰性工具发现”——用最笨最稳的方法抵消自己的冲动。曾国藩的日记不是情绪宣泄，而是一个外部调度层，每天强制 re-grounding。

## 证据：Focusmate 的效果与局限

Focusmate 被列为“11 Best ADHD Productivity Apps”之一，其核心机制是“AI-Matched Body Doubling”（来源：Focusmate）。在 ADHD 科技行业研究中，这种模式被归类为“身体加倍”，即通过实时同伴问责提升任务启动和维持能力（来源：ADHD科技行业深度研究）。

但必须诚实：Focusmate 不是 AI 工具，它主要依赖匹配算法而非生成式 AI。它的效果与身体在场效应一致，但缺乏 ADHD 人群的大规模随机对照试验。更关键的是，它可能成为拐杖而非脚手架——用户可能依赖外部调度而无法内化执行功能（来源：拐杖与脚手架）。

## 争议：脚手架 vs 拐杖的边界

Focusmate 是否会导致永久依赖？这取决于使用方式。如果每次写报告都需要一个陌生人在屏幕对面，那它可能是拐杖。但如果用户通过 Focusmate 学会了“先启动再调整”的模式，逐渐内化这种重锚定节奏，那它就是脚手架。

同构视角下，agent 工程也面临同样问题：过度依赖外部调度层会导致 agent 无法自主决策。现代 AI 编码代理通过“双代理架构”（分离规划与执行）来平衡自主与约束（来源：生成核心与调度层）。ADHD 用户也可以借鉴：把 Focusmate 当作“执行代理”，而自己保留“规划代理”的角色——只在启动和重锚定时调用，而非全程依赖。

## 今天就能试的行动

1. **用 Focusmate 做“启动仪式”**：每天只预约一个 25 分钟 session，专门用来启动当天最难的任务。完成后断开，尝试独立维持。
2. **建立自己的“日课十二条”**：模仿曾国藩，写 3-5 条固定规则（如“上午 10 点前不刷手机”），用 RescueTime 自动追踪执行情况（来源：RescueTime）。
3. **对比工具效果**：用 Brain.fm 的专注音乐做一次 session，再用 Focusmate 做一次，记录哪种方式让你更少跑偏。ADHD 异质性意味着没有万能工具（来源：矛盾与存疑）。
4. **给 agent 加一个“Focusmate”**：如果你在做 agent 工程，在 system prompt 里加入“每 10 步输出一次当前目标”，相当于定时重锚定。

## 结论

Focusmate 对 ADHD 的真正价值，不是“让人专注”，而是“在跑偏时拉回来”。它不是一个专注力工具，而是一个重锚定工具。这种功能与 agent 工程中的 scaffolding 在结构上同构：都是为生成核心搭建外部调度层，防止目标漂移。

但同构不是万能药。ADHD 大脑与 LLM 的类比目前仅是理论假设，缺乏实证基础（来源：矛盾与存疑）。Focusmate 的效果依赖个体差异，对注意缺陷为主型可能有效，对多动冲动型可能无效。

最终，无论是 ADHD 患者还是 agent 工程师，都需要回答同一个问题：你的生成核心需要什么样的重锚定协议？答案可能不是更强大的 AI，而是更聪明的脚手架。

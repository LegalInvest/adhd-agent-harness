# Brain.fm 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 这个类比乍看最牵强:音频工具和 system prompt 有什么关系?但想想 system prompt 到底在干什么——它不传递新信息,它**设定一个持续在场的「基调」**:你是谁、现在是什么模式、按什么风格运作。而声音恰恰是人类最古老的「模式设定」通道:军队用鼓点、寺庙用钟声、母亲用摇篮曲——都不传递内容,都在注入状态。Brain.fm 那篇讲过它调节唤醒度;这一篇讲它更微妙的第二职能:**当一段声音和「工作模式」绑定得足够久,它就变成了一条听觉的 system prompt——每次响起,都在重新注入「现在是什么模式」。** 而这条通道,有它专属的用错方式。

先看绑定的机制。条件化是最扎实的心理学之一:中性刺激与某个状态反复配对,刺激本身就获得唤起该状态的能力。**固定的专注音频×固定的工作情境,重复几十次后,音频响起的那一秒,神经系统开始自动切换**——这就是「听觉锚」的形成。它对 ADHD 格外有价值,因为模式切换(从闲散到工作、从一个任务到另一个)正是转换困难的重灾区,而听觉锚提供了一个不依赖意志力的切换扳机。误用四种:

**误用一:锚没有专用性(prompt 被复用污染)。** 工程版:同一段 system prompt 用在八个不同的场景,它对任何场景都不再有定义力。人类版:**工作听这个歌单、通勤也听、做饭也听、失眠也听**——音频和「工作模式」的绑定被稀释成和「醒着」绑定,锚失效。修法:**专曲专用**——工作锚只在工作时响起,一响就工作,工作外坚决不放;条件化的纯度就是锚的效力。

**误用二:只注入,不配对(空响的锚)。** 条件化需要「刺激+状态」真实配对——**如果音频响起后你照旧刷手机,配对的其实是「这个声音=刷手机」**,你在勤奋地训练一条反向的锚。工程对应:prompt 注入了,但每次注入后的行为都违反它,等于用行为数据持续否证这段 prompt。修法:锚的头几十次配对最关键——**音频响起的头五分钟,必须真的在做事**(哪怕只做最小的一步);宁可少用,不可空响。

**误用三:用锚替代内容(有基调,没指令)。** 工程版:system prompt 只写「你很专业」,不写做什么——模型有了气质,没有方向。人类版:耳机戴上、音景响起、状态到位——**然后发现没想好要做什么**,于是带着完美的工作状态漂进了收件箱。听觉锚设定的是模式,不是任务;**它必须和「打包好的任务」(启动系列)搭配**:音频响起=执行早已定好的第一步,而不是响起之后才开始想。

**误用四:锚老化了不轮换(习惯化)。** 条件化的天敌是习惯化:同一段音频听了半年,神经系统对它彻底脱敏,锚变成了背景空气。工程对应:prompt 一字不动跑一年,模型的遵循率悄悄衰减(上下文里其他东西的权重在变)。修法:**观察效力,定期轮换**——锚的「响起后五分钟内启动率」下降时,换一个新的音景重新绑定;老锚休息几个月还能复用。锚是消耗品,不是传家宝。

## 边界

同构强度 C 级:如实标注——条件化机制有扎实的心理学基础,听觉线索辅助状态切换有依据,但「音频作为 system prompt」是修辞层面的功能类比,Brain.fm 对 ADHD 的直接证据初步(见环境篇的说明)。声明:听觉锚是轻量辅助,个体差异大;若模式切换的困难严重到影响基本生活转换(出门、入睡、停止工作),值得专业评估——那可能涉及更深的转换障碍,不是一段音频的辖区。

## 今天就能试的 3 件事

1. **选定你的工作锚**:一个固定音景,从今天起专曲专用,工作外不放。
2. **保证头十次配对纯净**:锚响起的头五分钟必须真的在做事,宁可缩短,不可空响。
3. **给锚建一个效力日志**:每周记一次「响起后五分钟内启动了吗」,连续下降就轮换。

声音不会告诉你做什么,但它能告诉你的神经系统「现在是什么时候」——**这条最古老的注入通道,值得像工程师维护 prompt 一样被认真维护**:专用、真配对、带内容、常轮换。让某段声音成为只属于你工作状态的钟声。

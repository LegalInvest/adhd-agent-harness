# ChatGPT 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 ChatGPT：它到底补上了哪一层执行功能？

## 问题：为什么你的 ChatGPT 越用越焦虑，而别人的却像外挂大脑？

你打开 ChatGPT，输入“帮我整理下周的待办”，它列出一份漂亮的清单。你关掉页面，三分钟后忘了看。第二天，你又打开，输入同样的话，得到另一份清单。周而复始。

这不是你一个人的故事。ADHD 群体和 LLM/Agent 工程师都在面对同一个核心矛盾：**生成能力过剩，执行调度瘫痪**。ADHD 大脑能瞬间迸发创意，却无法启动“回邮件”这类简单任务；LLM 能写出流畅论文，却算不对 3+5 除非调用计算器。两边都缺一个“调度层”——在 ADHD 这边是执行功能，在 LLM 那边是 function calling 与 harness。

但多数人用错了 ChatGPT。他们把它当**答案生成器**，而不是**执行功能外包层**。结果就是：AI 帮你写出了完美的计划，但你仍然没有执行它。因为问题从来不在“不知道做什么”，而在“知道却做不了”。

## 同构：ADHD 大脑与 LLM 共享同一套“强记忆、弱控制”剖面

明尼苏达大学的一项系统测试发现，LLM 表现出“强记忆、弱控制”的执行功能剖面——工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷，无法灵活切换任务集、无法抑制自动化反应（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这正是 ADHD 的经典神经心理模式。

耶鲁大学进一步揭示了机制：自注意力机制本身导致工作记忆容量限制，随着任务复杂度增加，注意力分数熵增、注意力弥散，与人类注意分散机制同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。换句话说，**ADHD 的“注意力飘走”和 LLM 的“上下文膨胀”，是同一种计算问题的不同表现**。

这意味着，对 ADHD 有效的干预策略，对 LLM 同样成立：你不需要更大的工作记忆容量，你需要的是**外部脚手架**来管理注意力、分解任务、维护状态。

## 解法：AI 工具作为外部执行功能层——但必须是脚手架，不是拐杖

### 任务分解与启动：Goblin Tools 与 Lex 的“启动脚本”

ADHD 的任务启动困难对应 LLM 的“冷启动”问题。Goblin Tools 的 Magic ToDo 功能将“整理房间”自动拆解为“捡起地板上的衣服”“擦桌子”等步骤，用户反馈称“将压倒性的事情变成一系列不压倒性的事情”（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。对于 LLM，这相当于 prompt 中的 chain-of-thought 或 function calling 的 step-by-step 规划。

Lex 更进一步：它允许用户通过单一指令触发复杂、多步骤的任务序列（来源：Best AI Tools for ADHD Productivity in 2026）。这就像为 LLM 编写一个 agent harness，将“写周报”分解为“收集数据→生成草稿→格式化”的自动化流程。**关键区别**：Goblin Tools 和 Lex 不会替你做决定，它们只提供“启动脚本”——你仍然需要自己按下执行键。

### 记忆与上下文管理：Saner.AI 与 Mem 的“持久化上下文”

ADHD 的工作记忆容量保留但控制弱，LLM 同样有强记忆容量但弱上下文控制（来源：工作记忆）。Saner.AI 通过本地记忆存储和快速检索，帮助用户减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026）。Mem 则被称为“AI 第二大脑”，自动关联笔记并提供上下文。

对于 LLM，这相当于 RAG（检索增强生成）或外部知识库——把记忆从模型权重中解放出来，交给持久化存储。**但这里存在陷阱**：过度依赖外部记忆可能削弱主动回忆能力（来源：矛盾与存疑）。Saner.AI 和 Mem 应该作为“笔记提示器”而非“记忆替代品”，就像 LLM 的 RAG 应该作为事实补充而非推理替代。

### 实时反馈与身体在场：Focusmate 与 Otter.ai 的“社会监督”

身体在场效应被描述为“最有效的 ADHD 策略之一”（来源：11 Best ADHD Productivity Apps for Fluctuating Energy）。Focusmate 通过算法匹配虚拟身体加倍伙伴，提供隐性问责。对于 LLM，这相当于人类-in-the-loop 的验证机制——让 agent 每步操作都输出中间结果，由人确认后再继续。

Otter.ai 的实时转录则提供了即时外部反馈（来源：Otter.ai），类似于 LLM 的 streaming 输出。**但注意**：用户报告称过度依赖 Otter.ai 可能削弱主动记笔记的能力（来源：ADHD 的 AI 工具全景）。这再次印证了脚手架与拐杖的边界：工具应该**补偿**而非**替代**你的内在能力。

## 争议与局限：为什么 AI 工具不是万能药

资料中的“矛盾与存疑”部分明确警告：现有证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑）。这意味着，AI 工具的实际效果尚未得到严格验证，存在夸大宣传的可能。

更核心的问题是：**个体差异导致效果不一**。ADHD 症状的复杂性（如共病焦虑、抑郁）可能使 AI 难以准确识别用户意图（来源：Lex 页面）。一个工具对“注意力分散型”ADHD 有效，可能对“冲动型”ADHD 完全无效。

此外，多巴胺干预效果存在争议（来源：矛盾与存疑），而许多工具的设计恰恰依赖多巴胺奖励机制。如果用户对即时反馈产生依赖，反而可能加剧注意力波动。

## 今天就能试的 3 件事

1. **用 Goblin Tools 的 Magic ToDo 分解一个让你拖延的任务**。输入“整理书桌”，观察 AI 给出的步骤。然后，只做第一步——通常是最简单的那个。这利用了小步骤的即时反馈来绕过启动困难。

2. **为你的 ChatGPT 设置一个“function calling 模板”**。比如，每次写邮件前，先让 ChatGPT 输出“收件人、目的、关键信息、语气”四个字段，你再填充内容。这相当于为你的大脑写了一个 harness，把“写邮件”从黑箱变成可执行的步骤序列。

3. **尝试一次 Focusmate 的虚拟身体加倍**。选择一个 25 分钟的 session，和陌生人一起工作。注意它的效果：是让你更专注，还是增加了社交压力？如果是后者，说明身体加倍不适合你，不要强行使用。

## 结语

ChatGPT 之于 ADHD，就像 function calling 之于 LLM——它补上了执行调度层。但工具的价值取决于使用方式：如果你用它来**替代**思考，它是拐杖；如果你用它来**外包**精确、重复、状态维护类任务，它是脚手架。真正的同构不在于工具本身，而在于我们如何设计人与 AI 的协作界面——无论是 ADHD 大脑还是 LLM，都需要一个 harness，而不是一个保姆。

# Motion 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Motion：它到底补上了哪一层执行功能？

## 问题：为什么“开始”那么难？

如果你是一名 ADHD 成人，你一定经历过这样的时刻：坐在电脑前，任务清单摊开，却像被胶水黏在椅子上——明明知道该做什么，就是动不了。这不是懒，是执行功能中的“任务启动”环节卡壳了（来源：[[任务启动]]）。多巴胺水平不足让大脑缺乏行动动力，工作记忆容量有限让步骤无法清晰呈现，时间盲让截止日期显得遥远（来源：[[多巴胺]]、[[时间盲]]）。

如果你是做 Agent 工程的开发者，你一定也熟悉类似的困境：你给 LLM 一个长任务，它要么卡在第一步反复推理，要么直接跑偏，输出一堆无关内容。你需要给它套一个 planner-executor 循环：先拆解子任务，再逐步执行、反馈、重规划。否则，再聪明的模型也只是个“高产但无调度”的生成核心。

这两类困境，本质上是同一个问题：**生成核心（ADHD 大脑 / LLM）缺乏可靠的内置执行调度层**（来源：[[ADHD 大脑与 LLM 的同构]]）。所以，ADHD 需要外部脚手架来承担调度、排序、提醒与抑制；LLM agent 也需要外部 harness 来分解任务、保持上下文、监控进度。而 Motion——这款 AI 驱动的日程规划应用——恰好提供了这样一个 planner-executor 循环。

## 同构：Motion 作为外部调度层

Motion 的核心功能是自动排程与动态调整：它持续评估任务优先级、截止日期和可用时间，实时重建日程（来源：[[Motion]]）。当你完成一项任务，Motion 自动把后续任务提前；当你被会议打断，它重新安排被挤掉的任务。这相当于在 ADHD 大脑外部部署了一个“planner-executor 循环”——planner 是 Motion 的 AI 调度引擎，executor 是你本人（或你的团队）。它消除了“下一步该做什么”的决策负担，直接对抗时间盲和任务启动困难（来源：[[Motion]]）。

同样，在 LLM agent 领域，planner-executor 循环是标准架构：planner 将长任务分解为子步骤，executor 调用工具或生成文本，然后反馈给 planner 调整计划。Motion 之于 ADHD，就像这个循环之于 LLM——但有人用错了。

## 用错：当脚手架变成拐杖

问题在于，Motion 的自动调度可能过于“完美”。它接管了全部规划决策，用户只需被动执行。短期看，这确实减轻了认知负荷；但长期看，ADHD 大脑的执行功能可能因此萎缩，就像肌肉长期不用会退化（来源：[[拐杖与脚手架]]）。孔子一生与冲动做斗争，他的 harness 是“克己复礼”——每日反省、用外在礼法约束行为，而非依赖一个全自动系统。他70岁才做到“从心所欲不逾矩”，靠的是持续的内化训练，而非外包（来源：人物案例）。同样，老子选择“无为”，不强行做事，把所有能量收回来聚焦在道上——这是一种主动的、有意识的自我调度，而非被动接受外部安排（来源：人物案例）。

LLM agent 领域也有类似教训：如果 planner 过度主导，executor 变成无脑执行器，agent 会丧失灵活性，无法应对意外。好的 agent harness 给予适度脚手架，保留模型的主动推理空间；坏的 harness 则完全替代模型思考，导致退化。

## 证据：两类读者的真实反馈

ADHD 侧：用户反馈 Motion 帮助减轻压力、改善工作生活平衡，甚至加速晋升（来源：[[Motion]]）。但缺乏独立临床证据，初始设置成本高（来源：[[矛盾与存疑]]）。更重要的是，部分用户反映学习使用 AI 工具本身增加认知负荷（来源：[[矛盾与存疑]]）。这提示：工具不是越自动越好。

LLM/agent 侧：研究表明，LLM 在状态保持、真实规划、上下文控制上存在缺陷，需要外部 scaffolding（来源：[[执行功能]]、[[工作记忆]]）。但过度结构化会抑制模型创造力。同构视角下，ADHD 和 LLM 都需要一个“刚好够用”的调度层，而非全盘接管。

## 诚实局限：Motion 不是万能药

Motion 的局限包括：依赖日历数据，对日程不规律的用户效果有限；未直接处理多巴胺调节或情绪波动（来源：[[Reclaim.ai]] 对比）。与 Goblin Tools 的微任务分解不同，Motion 不直接帮你拆解任务（来源：[[Goblin Tools]]）。而任务启动的关键一步——把大任务变成可操作的小步骤——Motion 并未解决。此外，长期依赖风险缺乏实证（来源：[[矛盾与存疑]]）。

## 今天就能试的行动

1. **使用 Motion 的“微任务”技巧**：在 Motion 中创建任务时，手动将其拆成 10 分钟以内的小块（如“写邮件开头”而非“写邮件”），弥补 Motion 不擅长的分解功能。
2. **设置“反思时间块”**：在 Motion 日程中每天留出 15 分钟，用于回顾当天执行情况，主动训练调度能力——像孔子的“吾日三省吾身”。
3. **交替使用 Motion 与手动规划**：例如，周一、三、五用 Motion 全自动调度，周二、四用手动规划（如 Tiimo 视觉日程），保持大脑调度肌肉的活跃。
4. **尝试“保护性”工具混合**：用 Reclaim.ai 保护深度工作时间块，用 Motion 填充剩余时间，避免 Motion 过度填充导致日程过满（来源：[[Reclaim.ai]]）。

Motion 是一个强大的外部调度器，但它应该是一个脚手架，而非拐杖。真正的目标，是像孔子和老子那样，最终内化出自己的调度系统——哪怕那需要一辈子。

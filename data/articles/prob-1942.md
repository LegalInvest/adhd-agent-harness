# ADHD 创业者如何用「给生成核心套 harness」的思路应对把想法落地执行里的怕过度依赖工具、又怕没有支撑就崩？

> 把 LLM 工程里的「harness 应是会褪去的脚手架，而非永久拐杖」迁移到ADHD 创业者的把想法落地执行场景，双域证据作答。

你是一个 ADHD 创业者。脑子里每天冒出三个新点子，每一个都像能改变世界。但到了执行环节——分解任务、排优先级、按时交付——你就像被抽走了操作系统，只剩一个空转的 CPU。你试过 Todoist、Notion、纸质手账，最后都成了“被遗弃的规划器、应用和习惯的墓地”（来源：Using an AI Assistant to Manage ADHD: A Practical Guide）。

你怕。怕过度依赖 AI 工具，变成离开拐杖就倒的废人；又怕没有支撑，连第一个任务都启动不了。

这种两难，和 LLM Agent 工程里的一个核心矛盾完全同构：生成核心（LLM）什么都能写，但一跑就偏、一长就忘，必须靠 harness（脚手架）来规划、分解、验证、重锚定。但 harness 应该是会褪去的脚手架，而非永久拐杖。

## 同构：高产但缺调度层的生成核心

ADHD 大脑与 LLM 在底层结构上惊人相似：
- **生成核心过强**：ADHD 的情绪生成系统（杏仁核）反应强烈，LLM 的文本生成能力几乎无限。
- **调度层缺失**：ADHD 的前额叶（执行功能中枢）调控不足，LLM 缺乏内在规划能力，需要外部 harness 引导（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。
- **无状态与外部记忆**：ADHD 的工作记忆缺陷导致信息难以整合，LLM 每次对话都是“失忆”的，必须靠外部记忆（如 Saner.AI 的本地记忆）来保持上下文（来源：无状态与外部记忆）。
- **注意力易被带偏**：ADHD 的分心与 LLM 的上下文膨胀类似，都需要外部重锚定。

这意味着，ADHD 创业者需要的不是“更强大的生成核心”，而是一个**外部执行功能层**——一套能规划、排序、跟踪、提醒的系统，让大脑自由去做实际工作（来源：Using an AI Assistant to Manage ADHD: A Practical Guide）。

## 脚手架 vs 拐杖：边界在哪里？

拐杖是永久替代，拿走就倒；脚手架是临时支撑，任务完成后可以拆除。

在 LLM 工程中，harness 是脚手架：它提供规划循环、任务分解、工具接口、验证循环，但核心目标是在 agent 学会自主执行后逐步退出（来源：What Is an Agent Harness? The Infrastructure That Makes AI Agents ...）。在 ADHD 管理中，AI 工具也应是脚手架：Inflow 教用户认知行为疗法（CBT）技能，目标是“建立技能”而非永久依赖（来源：The 12 Best Apps for ADHD in 2026: A Guide to Finding What ...）；Goblin Tools 的 Magic ToDo 将任务分解为小步骤，但用户可以选择“辣度”控制粒度，逐步减少分解层级（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。

关键在于**设计退出机制**。一个好的脚手架系统，必须包含“何时拆除”的规则。

## 历史案例：释迦牟尼的“自依止”脚手架

释迦牟尼29岁放弃王位出家，先后跟随两位仙人学习、苦行六年，不断试错不断放弃。他最终建立的不是一套教条，而是以“八正道”“持戒”为核心的自我管理系统——用戒律管行为，用正念拴念头。他最核心的教导是“自依止法依止”：不依赖权威，自己证悟。

这正是脚手架思维：八正道是外部框架，但目标是让修行者最终能“自依止”。对应到 LLM harness：会话状态记录工具结果和进度笔记（来源：What Is an Agent Harness? The Infrastructure That Makes AI Agents ...），但最终 agent 应能自主规划。对应到 ADHD 创业者：AI 工具提供提醒和重锚定，但用户应逐步内化这些结构。

- **任务分解**：Goblin Tools 将模糊任务转化为具体步骤，降低启动门槛（来源：Goblin Tools）。

- **规划循环**：Harness 提供任务分解、工具接口、验证循环，弥补 LLM 缺乏的内在规划能力（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。
- **会话状态**：外部记录工具结果和子任务进度，解决无状态问题（来源：What Is an Agent Harness? The Infrastructure That Makes AI Agents ...）。

## 核心观点：脚手架思维是 ADHD 创业者的元技能

我的判断是：ADHD 创业者不应追求“根治”执行功能缺陷，也不应逃避使用工具。真正的解法是**把“搭建会褪去的脚手架”本身变成一项元技能**。你不需要完美执行功能，你需要的是：
1. 识别自己当前最弱的执行功能环节（规划？启动？记忆？）。
2. 为它选择最轻量的 AI 脚手架（如 Goblin Tools 分解任务、Saner.AI 做外部记忆）。
3. 设定退出条件（如“使用两周后，尝试减少分解层级”）。
4. 定期复盘：这个工具还在帮我成长，还是成了拐杖？

## 今天就能试的 3 件事

1. **用 Goblin Tools 分解一个你拖延已久的任务**：输入“整理项目提案”，调节“辣度”到你觉得“有点挑战但可行”的粒度。完成后记录：你在哪一步卡住了？是分解不够细，还是启动后依然分心？
2. **设置一个“重锚定”提醒**：在 Saner.AI 或手机日历里，每天固定时间（如10:00、15:00）设置一个提醒：“检查当前任务是否与本周目标一致”。坚持三天，记录目标漂移次数。
3. **设计一个退出实验**：选择一个你正在依赖的 AI 工具（如 Motion），设定一个日期（如两周后）尝试减少使用频率。如果感到恐慌，说明它已经是拐杖；如果顺利，说明它是成功的脚手架。

脚手架终将褪去，但你搭建它的能力会留下。

# RescueTime 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 RescueTime：它到底补上了哪一层执行功能？

## 问题：为什么 RescueTime 对 ADHD 不管用？

你下载了 RescueTime，设置了目标，一周后却发现自己根本没打开过——或者打开了，看着那些红红绿绿的图表，只是更焦虑。这不是你的错。RescueTime 被设计成“时间追踪工具”，但 ADHD 大脑需要的不是追踪，是执行。同样，LLM agent 如果只记录每次调用的 token 消耗，却不在上下文中注入验证循环和任务分解，那和裸调 API 没区别。

RescueTime 之于 ADHD，就像 function calling 工具调用之于 LLM——它本应是外部调度层的核心组件，但多数人用错了：把它当成了被动日志，而非主动脚手架。

## 同构：ADHD 大脑与 LLM 共享同一个缺陷

ADHD 大脑常被描述为“高产但缺乏可靠执行调度层的生成核心”（来源：[主题综述]）。想法丰富，联想活跃，但缺少稳定、可靠的调度层来筛选、组织、启动和坚持任务。大语言模型同样如此：生成能力惊人，却在长程上下文保持、目标追踪和执行调度上存在天然缺陷。

因此，“AI 帮 ADHD”的实质与“给 LLM 套 harness”是同一类工程：用外部脚手架把生成能力约束到目标轨道上。这种脚手架至少承担四种功能：任务拆解与启动、外部记忆与知识管理、对话式执行与结构化推理、捕获与反思（来源：[主题综述]）。RescueTime 只做了捕获——而且是被动捕获。

## 实证：脚手架 vs 拐杖的边界

真正有效的 ADHD 工具，必须主动管理“当下注意什么”。比如 **Goblin Tools** 的 Magic ToDo 能把模糊任务（“清理厨房”）自动分解为具体子步骤，用户可调节“辣度”控制粒度（来源：[Goblin Tools]）。**Saner.AI** 通过本地记忆和知识回忆减少搜索循环和标签切换（来源：[Saner.AI]）。**Lex** 允许用户通过单一指令触发复杂多步骤任务序列，类似 agent scaffolding 架构（来源：[Lex]）。这些工具的共同点是：它们不是事后记录，而是事前引导。

再看 LLM agent 侧：有效的 harness 工程必须提供上下文传递、工具接口、验证循环和记忆系统（来源：[幻觉与验证循环]）。关键组件包括“验证与 CI 集成”，“结构化工作流和规划”要求将复杂任务分解为子任务，并在每一步进行验证（来源：[幻觉与验证循环]）。缺乏这些验证会导致“更多幻觉和重复工作”（来源：[幻觉与验证循环]）。

## 人物案例：孔子的“日课”就是外部调度层

孔子一生精力旺盛，周游列国 14 年坐不住，冲动爱骂人，思维跳跃，《论语》全是场景化语录，无系统著作。他的专属 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。70 岁才做到从心所欲不逾矩（来源：[人物案例]）。

这里的“日课”对应 LLM agent 的定时 re-grounding，“礼”对应外部调度器的约束规则。孔子没有依赖内在意志力，而是构建了一套可重复的外部流程来管理自己的冲动和注意力。这正是 RescueTime 应该做但没做的事：不是让你看着图表自我批评，而是在你偏离轨道时主动拉你回来。

## 矛盾与局限：脚手架还是拐杖？

有人担心：过度依赖 AI 工具会导致能力退化，成为永久拐杖（来源：[矛盾与存疑]）。这个担忧合理，但混淆了“脚手架”和“拐杖”的边界。脚手架在建造完成后可以拆除，拐杖则永久替代。对于 ADHD 大脑，执行功能缺陷是神经性的，不是技能缺失——就像近视需要眼镜，不是“用进废退”。关键在于工具是否在训练你的调度能力，还是仅仅替你执行。Goblin Tools 的分解功能让你看到任务结构，Saner.AI 的记忆让你记住信息位置，这些都是脚手架。RescueTime 的被动记录则更像是拐杖——它不教你任何东西。

另外，ADHD 大脑与 LLM 的同构目前仍主要是类比推理，缺乏神经科学或计算机科学的直接证据（来源：[矛盾与存疑]）。但作为工作假说，它已经催生了有效的工具设计。

## 结论：RescueTime 的正确用法

RescueTime 不是没用，而是被用错了。它的正确用法不是事后看报表，而是：
1. **设置主动阻断**：利用 RescueTime 的 FocusTime 功能，在指定时段自动屏蔽干扰网站——这相当于 LLM agent 的上下文隔离。
2. **结合 Routinery 等应用创建日常步骤序列，并用 RescueTime 验证执行**（来源：[上下文工程]）。
3. **每周用 RescueTime 数据做一次“调度审计”**：不是自责，而是找出哪个时段你的生成核心最活跃，然后固定那个时段做最困难的任务。

## 今天就能试的行动

1. 打开 RescueTime 的 FocusTime，设置每天上午 9-11 点为“深度工作”时段，自动屏蔽社交媒体。
2. 用 Goblin Tools 的 Magic ToDo 分解你明天最头疼的一个任务，调节“辣度”到你觉得“轻松能完成”。

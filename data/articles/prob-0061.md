# 3 个 LLM 工程概念，彻底重写了我对 ADHD 有想法有能力，却卡在执行与落地的理解

> LLM 智力强但需要外部编排才能完成长任务 如何反向照亮 ADHD 的有想法有能力，却卡在执行与落地。

你有没有这样的体验：脑子里有一个绝妙的方案，但就是无法把它变成一份可执行的文档；或者你清晰地知道接下来三天该做什么，但一到早晨就瘫在沙发上刷手机，眼睁睁看着时间流走。

如果你是一名 ADHD 患者，你可能会把这种“有想法有能力，却卡在执行与落地”归咎于意志力不足。如果你是一名在做 Agentic Harness 工程的 LLM 开发者，你可能会为 GPT-4 能写出惊艳的代码却无法自主完成一个多步骤任务而头疼。

但最近，当我深入对比 ADHD 神经科学和 LLM 工程的最新研究后，我发现了一个令人震惊的同构：**ADHD 大脑与 LLM 本质上都是“高产但缺乏可靠执行调度层的生成核心”**（来源：ADHD 大脑与 LLM 的同构）。这意味着，LLM 工程中用来驯服大模型的那些概念，可以直接用来理解并改善 ADHD 的执行困境。

以下 3 个 LLM 工程概念，彻底重写了我对 ADHD 执行困难的理解。

## 1. 生成核心与调度层：你的大脑就是一台没有操作系统的超级计算机

LLM 的核心是 Transformer 架构，它擅长模式匹配和内容生成，但缺乏内置的规划、排序和监控能力。因此，工程上需要为 LLM 搭建一个“调度层”——比如 ReAct 框架或 AutoGPT 的循环——来管理任务分解、工具调用和状态跟踪。明尼苏达大学的一项研究发现，LLM 表现出“强记忆、弱控制”的剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷（来源：工作记忆）。这正是 ADHD 的经典神经心理模式。

对于 ADHD 大脑，情况完全一样。你的“生成核心”——发散思维、创意联想、模式识别——可能非常强大，但你的“调度层”——前额叶执行功能——却常常掉线。结果就是：你拥有顶级的想法生成器，却没有一个可靠的操作系统来编排这些想法。

这正是 AI 工具可以介入的地方。像 **Goblin Tools** 的 Magic ToDo 功能，就是一个人工调度层：你输入“整理房间”，它自动输出“捡起地板上的衣服”“擦桌子”等步骤（来源：Goblin Tools）。这相当于给 ADHD 大脑装了一个外挂的“任务分解器”，把“压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools）。同样，**Motion** 应用通过 AI 自动排程与动态调整，持续评估任务优先级、截止日期和可用时间，实时重建日程，减少你的手动规划压力（来源：Motion）。

但这里有一个关键边界：**脚手架 vs 拐杖**。脚手架是你在建造大楼时临时使用的结构，建完后可以拆除；拐杖是你永远依赖的支撑。如果 AI 工具让你永远不需要练习任务分解和时间估算，你的内在执行功能就可能萎缩（来源：矛盾与存疑）。因此，一个好的 AI 工具应该像训练轮，而不是自动驾驶。

## 2. 上下文工程：你的注意力不是不够，而是膨胀了

LLM 有一个臭名昭著的问题：上下文膨胀。当输入长度超过某个阈值，模型的注意力分数会熵增，注意力开始弥散，表现下降。耶鲁大学的研究发现，Transformer 的自注意力机制本身就会导致工作记忆容量限制，随着 N-back 任务 N 值增加，注意力分数熵增、注意力弥散、表现下降，与人类注意分散机制同源（来源：工作记忆）。

ADHD 大脑的注意力问题，本质上也是“上下文膨胀”。你试图同时记住太多事情——未回复的邮件、明天的会议、冰箱里快过期的牛奶——导致注意力资源弥散分配，最终什么都抓不住。

解决方案是**主动管理上下文**。在 LLM 工程中，这表现为 RAG（检索增强生成）或外部记忆系统——把信息存储在外部的向量数据库中，只在需要时检索。对于 ADHD 大脑，**Saner.AI** 正是这样一款工具：它利用 AI 实现本地记忆存储和快速检索，让你无需在多个标签页或应用间切换即可获取所需信息（来源：Saner.AI）。这直接针对 ADHD 用户常见的“信息丢失”和“注意力分散”问题。

但注意，上下文工程不是让你把所有东西都丢进一个“第二大脑”然后忘掉。真正的技巧是**设置检索触发器**——就像 LLM 的 RAG 系统需要用户查询才能激活一样，你需要建立外部线索（比如每日回顾、定时提醒）来主动拉取关键信息，而不是被动等待记忆浮现。

## 3. 外部时钟与时间锚点：时间盲的工程解法

LLM 没有时间感知。你问它“现在几点”，它只能猜测。工程上，我们需要给 LLM 注入外部时钟信息，或者在 agent 框架中设置时间戳和超时机制。

ADHD 的“时间盲”与此高度对应：难以感知和估计时间的流逝，无法准确判断任务耗时（来源：时间盲）。这导致你常常陷入“再刷五分钟”然后两小时过去了，或者严重低估任务所需时间。

AI 工具可以提供外部时间结构。**Tiimo** 专为时间盲设计，提供视觉时间表，将一天的任务以可视化方式呈现，帮助 ADHD 大脑从“无限延迟”转向“开始行动”（来源：时间盲）。**Focusmate** 则利用“身体在场效应”——虚拟身体在场——通过与他人实时视频连线来提供外部时间锚点和问责（来源：时间盲）。

但这里有一个争议：过度依赖外部时间提示可能削弱你内在的时间感知能力（来源：矛盾与存疑）。我的观点是：对于 ADHD 患者，内在时间感知可能永远无法达到神经典型水平，所以外部时钟不是拐杖，而是**义肢**——就像近视眼镜一样，你不需要“练习”不戴眼镜看清东西。关键在于，你要主动选择何时使用这些工具，而不是让工具控制你。

## 局限与诚实警告

我必须承认，上述同构视角虽然具有启发性，但证据基础仍然薄弱。目前多数 AI 工具对 ADHD 的效果证据来自用户报告和概念类比，缺乏大规模随机对照试验（来源：矛盾与存疑）。此外，ADHD 表现型多样，AI 工具对注意力缺陷为主型、多动冲动型、混合型的效用可能不同，但现有工具很少针对亚型设计（来源：矛盾与存疑）。

最值得警惕的是依赖风险。工具设计者声称它们是“脚手架”，但实际使用中可能沦为“拐杖”（来源：矛盾与存疑）。目前缺乏长期追踪研究来回答这个问题。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个压垮你的任务**：输入“准备下周的汇报”，看看 AI 如何把它拆成 10 个小步骤。然后只做第一步。
2. **用 Motion 或 Tiimo 规划明天的日程**：把所有任务和截止日期输入，让 AI 自动生成时间块。观察它如何帮你克服“下一步该做什么”的决策瘫痪。
3. **建立一个外部记忆系统**：用 Saner.AI 或类似工具，把你经常忘记的信息（Wi-Fi 密码、常用文档链接）存储起来，并设置一个每日 5 分钟的“检索练习”。
4. **尝试一次 Focusmate 会话**：找一个陌生人，约定 50 分钟专注工作。感受外部身体在场如何帮你锚定时间。

最终，ADHD 的执行困难不是性格缺陷，而是架构缺陷。而 LLM 工程已经为我们提供了一套成熟的补丁方案。关键在于：你要成为自己大脑的工程师，而不是被动等待系统升级。

# ChatGPT 之于 ADHD，就像 会褪去的脚手架 之于 LLM——但有人用错了

> 从同构视角实测 ChatGPT：它到底补上了哪一层执行功能？

你是否也曾这样：打开 ChatGPT，让它帮你把“写周报”拆成小步骤，然后对着分解列表发呆，最终又回到刷短视频的循环？或者，你是一位 Agent 工程师，看到用户让 AI 反复确认“下一步做什么”时，心里泛起一丝不安——这到底是辅助，还是另一种瘫痪？

这两种困境共享同一个结构：一个高产的生成核心（ADHD 大脑或 LLM），配上一个不可靠的执行调度层（前额叶或 Agent 框架）。当外部工具（ChatGPT、harness）介入时，它既可能成为暂时支撑的脚手架，也可能退化为永久依赖的拐杖。关键在于：你用它来补调度，还是替调度？

## 同构：高产核心 + 缺调度层

ADHD 大脑与 LLM 在结构上惊人相似。最新研究揭示，Transformer 在 Stroop 冲突任务中，注意力随序列变长骤降到随机水平——无法抑制优势反应，无法解决认知冲突（来源：Deficient Executive Control in Transformer Attention）。这恰恰对应 ADHD 执行功能缺陷的核心标志。LLM 呈现“强记忆、弱控制”剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷（来源：Strong Memory, Weak Control）。而 ADHD 大脑同样是“生成核心”高产（发散思维、创意、多任务并行），但执行功能（任务启动、持续专注、抑制冲动）薄弱（来源：执行功能）。

两边都缺一个可靠的调度层。LLM 需要 Agent harness 通过 human-in-the-loop 监督、token 预算、人工检查点来防止失控（来源：Building an AI Agent Harness from Scratch）。ADHD 大脑则需要外部工具补偿工作记忆、时间感知和任务启动。

## 脚手架 vs 拐杖：边界在哪里？

脚手架是临时结构，施工完成后拆除；拐杖是永久支撑，卸下就无法行走。在 ADHD 语境下，脚手架式的 AI 工具帮助用户内化策略，最终减少依赖；拐杖式的工具则让用户每次都需要外部提示才能行动。

以 Goblin Tools 为例，它的 Magic ToDo 将模糊任务分解为具体步骤，用户可调节“辣度”控制粒度（来源：Goblin Tools）。如果用户只是照着步骤做，从不学习分解方法，那就是拐杖；如果用户通过多次使用，逐渐学会自己拆解任务，那就是脚手架。同样，Inflow 基于 CBT 提供结构化路径，教用户建立技能（来源：Inflow），其设计初衷是脚手架——但若用户只依赖每日提示而不内化，则可能退化。

## 人物案例：释迦牟尼的 harness 系统

释迦牟尼29岁放弃王位出家求道，先后跟两位仙人学习、苦行6年，不断试错不断放弃，最终创立佛教。他一生游行说法，一字未写，所有教法都是场景化比喻。这种“高产但缺调度”的特质，与 ADHD 高度吻合。他的 harness 是“八正道”和“持戒”——用戒律管行为，用正念拴住念头。更重要的是，他提出“自依止法依止”，不依赖权威，自己证悟。

这与 LLM harness 的同构对应是什么？日课（定时 re-grounding）就像 Agent 中的人工检查点——定期暂停，审视方向；戒律就像护栏规则——设定行为边界；而“自依止”则强调最终要内化，不永远依赖外部。释迦牟尼的 harness 是典型的脚手架：他通过戒律和正念训练，最终达到无需刻意维持的觉知状态。

## 工具如何补调度？

情绪调节是 ADHD 最隐蔽的痛点。AI 辅助情绪调节的本质不是直接安抚，而是通过修复执行功能链条来减少情绪触发源（来源：AI 与 ADHD 的情绪调节）。例如，Saner.AI 通过本地记忆减少搜索循环和标签切换，降低因信息丢失引发的焦虑（来源：Saner.AI）。Focusmate 利用身体在场效应，让他人在线共作提升专注（来源：人在回路与身体在场）。这些工具都在补调度，而不是替调度。

但争议仍在。依赖风险是核心矛盾：过度依赖 AI 工具可能导致“永久拐杖”，削弱内在执行功能发展（来源：矛盾与存疑）。同构假设也有边界：ADHD 大脑的生物复杂性远超 LLM，情绪调节涉及神经递质、环境交互等，单纯类比可能简化问题（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Goblin Tools 拆一个任务，但要求自己记住拆解逻辑**：下次遇到类似任务，先尝试自己拆，再用工具验证。
2. **设置 AI 使用时限**：例如，每天只让 ChatGPT 帮你拆解 3 个任务，其余自己完成。
3. **尝试 Focusmate 一次**：体验身体在场效应，注意它如何绕过执行功能。
4. **记录工具使用后的感受**：一周后回顾，哪些任务你已能独立完成？哪些仍离不开工具？

脚手架终将褪去，留下的是你学会的调度能力。

# ADHD 创业者视角：为什么「治好 ADHD 的怕过度依赖工具、又怕没有支撑就崩」和「让 LLM 不跑飞」其实是同一道工程题？

> harness 应是会褪去的脚手架，而非永久拐杖——同一套 harness 思路，两边都成立。

你是一个 ADHD 创业者。白天，你靠 ChatGPT 分解任务、靠 Motion 排日程、靠 Inflow 做 CBT 训练——工具堆得满满当当，但你心里总有个声音：我是不是在给自己造拐杖？哪天没这些工具，我是不是就崩了？

你也是一个 Agentic Harness 工程师。你给 LLM 搭了 ReAct 循环、规划器、记忆模块，却总担心 agent 跑飞——或者更糟，你的 harness 变成了永久拐杖，让底层模型永远学不会自己调度。

这两份焦虑，其实是同一道题。

## 同构：高产但缺调度层的生成核心

ADHD 大脑与 LLM 共享一个深层结构：都是高产的生成核心，但缺乏可靠的执行调度层。

**ADHD 侧**：多巴胺系统失衡导致中脑皮质分支功能低下，引起执行功能差（来源：多巴胺）。工作记忆不稳定、时间感知偏差（时间盲）、任务启动困难——这些底层缺陷让 ADHD 大脑如同一个没有操作系统的超级 GPU，算力强大但无法有序调度。

**LLM/Agent 侧**：大模型生成能力强，但缺乏原生输出控制，上下文膨胀、注意力漂移、工具调用失败——如同一个没有 harness 的生成核心，跑起来容易跑飞。

AI 工具为 ADHD 搭建的“外部执行功能层”（来源：AI 与 ADHD 的情绪调节），与 LLM 的 agent scaffolding，本质是同一套架构：一个外挂的调度层，负责分解任务、管理记忆、控制节奏。

## 脚手架 vs 拐杖：边界在哪里？

核心问题在于：这个外挂调度层，是脚手架还是拐杖？

脚手架是会主动褪去的。释迦牟尼用“八正道”和“正念”作为自己的 harness：戒律管行为，正念拴念头，但最终目标是“自依止法依止”——不依赖权威，自己证悟。他的 harness 是训练内在调度能力的临时框架，而非永久依赖。

任正非的华为基本法、红蓝军对抗、民主生活会自我批判——这些制度化的 harness 同样不是拐杖。它们的目标是让组织（和个体）在框架内形成自律，而非永远依赖外部指令。

对应到 LLM/Agent 工程：harness 应逐步减少介入粒度，从强约束的 ReAct 循环过渡到弱监督的自我反思，最终让模型学会自主规划。

**拐杖**：长期无节制使用 AI 工具，可能阻碍 ADHD 个体发展内在情绪调节能力（来源：矛盾与存疑）。同样，一个过度刚性的 harness 会让 agent 永远学不会自己纠偏。

## 证据：两边都成立

**ADHD 侧**：Goblin Tools 的 Magic ToDo 将模糊任务分解为可执行步骤（来源：Goblin Tools），直接降低启动门槛。Saner.AI 通过本地记忆减少搜索循环（来源：Saner.AI）。但这些工具若被当成永久依赖，用户可能丧失自主分解能力。

**LLM/Agent 侧**：Reclaim.ai 和 Motion 自动规划日程，类似 agent 的调度器。但若 harness 从不褪去，模型永远无法在无调度时保持稳定。

## 局限与争议

“ADHD 大脑与 LLM 同构”目前是理论类比，缺乏直接的神经科学或计算机科学证据（来源：矛盾与存疑）。此外，个体差异显著：注意力不集中型 vs 冲动型 ADHD 对工具反应不同，LLM 的不同架构对 harness 的敏感度也不同。

## 今天就能试的行动

1. **给工具设“褪去日期”**：比如用 Goblin Tools 分解任务时，设定一个月后尝试自己分解，对比差异。
2. **记录“工具依赖日志”**：每周记录哪些场景下没有工具就崩，分析是否可训练内在能力。
3. **设计 agent 的“褪去计划”**：在 harness 中增加逐步减少干预的规则，如每轮减少 10% 的提示词长度。
4. **尝试“无工具日”**：每周一天，只用纸笔和闹钟，观察哪些执行功能最需要外部支撑。

脚手架终会褪去，留下的不是依赖，是长在骨子里的能力。

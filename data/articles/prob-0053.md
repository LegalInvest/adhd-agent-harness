# 3 个 LLM 工程概念，彻底重写了我对 ADHD 任务启动困难、不会拆解的理解

> agent 用 planner-executor 循环分解长任务 如何反向照亮 ADHD 的任务启动困难、不会拆解。

## 问题：为什么“开始做”比“做”更难？

如果你是一个 ADHD 成人，你一定经历过这样的时刻：面前摆着一项任务——写一份报告、整理房间、甚至只是回一封邮件——但你就是无法启动。你明明知道该做什么，大脑却像死机一样，光标在空白文档上闪烁，而你在刷手机、吃零食、或者盯着墙壁发呆。这不是懒，这是 **任务启动困难**（Task Initiation），是 ADHD 执行功能缺陷的核心表现之一（来源：任务启动）。

更令人沮丧的是，即使你强迫自己开始，你很快又会卡住：任务太大、太模糊，你不知道从哪里下手。你缺乏 **任务分解** 的能力——把复杂目标拆成可执行的小步骤。你盯着“写一份年度总结”，大脑一片空白，因为你的工作记忆无法同时容纳“打开文档-列出大纲-填充数据-润色语言”这一串步骤（来源：工作记忆）。

与此同时，在 AI 工程领域，工程师们正在解决一个看似无关的问题：如何让一个 LLM（大语言模型）完成一个需要多步推理的长任务？答案是一种叫做 **planner-executor 循环** 的架构：一个“规划器”将任务分解成子步骤，一个“执行器”逐个完成，然后循环迭代直到任务结束。这个模式被称为 **Agentic Harness**——给 LLM 套上缰绳，让它从“聪明的文本生成器”变成“可靠的任务完成者”。

ADHD 大脑和 LLM 是同一类东西：**高产但缺乏可靠执行调度层的生成核心**。本文将通过 3 个 LLM 工程概念，彻底重写你对 ADHD 任务启动困难与不会拆解的理解。

## 概念 1：规划循环——从“一次性完成”到“边走边拆”

LLM 天生没有“计划”能力。你给它一个复杂指令，它要么试图一步生成（然后迷失在细节中），要么拒绝回答。Agent 工程的解法是 **planner-executor 循环**：规划器先输出一个粗粒度计划，执行器完成第一步，然后将结果反馈给规划器，规划器据此调整下一步计划。这个循环不断进行，直到任务完成。

ADHD 大脑同样缺乏这种内置的规划循环。当你面对“打扫房间”时，你的大脑试图一次性规划所有步骤：扫地、拖地、擦桌子、整理书架、扔垃圾……然后被信息淹没，直接宕机。ADHD 的 **工作记忆容量有限但控制弱**（来源：工作记忆），无法像 neurotypical 大脑那样自动维护一个任务栈。

**同构解法**：ADHD 需要把“规划循环”外部化。工具如 **Goblin Tools** 的“魔法待办”功能，可以将“写论文”自动拆解为“打开电脑→新建文档→写标题→写第一段→休息5分钟→继续……”等小步骤（来源：任务启动）。这本质上就是 agent 的 planner 模块。而 **Motion** 和 **Reclaim.ai** 则通过自动调度和动态调整，实现了 executor 的角色：它们根据你的实际完成情况重新安排后续任务，就像 agent 的反馈循环（来源：Motion；Reclaim.ai）。

**证据**：用户报告显示，Motion 的动态日程调整能显著减少决策疲劳（来源：Motion）；而一项针对 ADHD 成人的研究发现，他们自发使用 ChatGPT 作为执行功能外挂，通过对话生成步骤来弥补启动困难（来源：任务启动）。这印证了 planner-executor 循环的普适性。

## 概念 2：上下文窗口与时间盲——把不可见变为可见

LLM 的另一个缺陷是 **上下文窗口有限**：它只能“记住”最近几千个 token，超出范围的信息会被遗忘。工程师通过外部记忆（如向量数据库）或进度条来补偿。ADHD 的 **时间盲**（time blindness）与此同构：ADHD 大脑无法感知时间的流逝，就像 LLM 无法理解“还剩多少 token”一样（来源：时间盲）。

**同构解法**：将时间转化为可见元素。**Tiimo** 是一款专为 ADHD 设计的视觉化日程应用，它用颜色块和时间轴把一天的时间“画”出来，让用户直观看到“现在几点”“接下来做什么”（来源：Tiimo）。这相当于为 LLM 提供了一个实时进度条。而 **Todoist** 的智能优先级排序，则像向量数据库一样，把最重要的任务“钉”在用户视野里，补偿工作记忆的脆弱性（来源：Todoist）。

**证据**：Tiimo 用户报告称其改善了时间感知（来源：Tiimo），但资料也指出其 AI 功能细节尚不明确（来源：矛盾与存疑）。这提醒我们，视觉化本身可能比 AI 更关键——就像 LLM 的进度条不需要是智能的，只要存在就能减少不确定性。

## 概念 3：任务分解粒度——从“一步到位”到“原子步骤”

Agent 工程中，任务分解的粒度至关重要。如果规划器把“做一顿饭”分解成“准备食材→烹饪→装盘”，执行器仍然可能卡在“准备食材”上。好的 agent 会继续分解到原子步骤：“打开冰箱→取出鸡蛋→拿碗→打蛋→搅拌”。ADHD 的 **不会拆解** 正是粒度问题：你总是分解得太大、太模糊。

**同构解法**：ADHD 需要“过度分解”。**Saner.AI** 提供基于认知科学的启动策略，比如“两分钟法则”——任何任务如果能在两分钟内完成，就立刻去做（来源：任务启动）。这相当于把任务分解到“两分钟”的原子粒度。而 **Goblin Tools** 的分解功能，可以自动输出极其琐碎的步骤，甚至包括“深呼吸一次”这样的准备动作。

**证据**：资料指出，ADHD 成年人自发使用 ChatGPT 来生成任务步骤，并且报告说“它让我看到第一步有多小”（来源：任务启动）。这正是 agent 工程中“原子步骤”的实践。

## 脚手架 vs 拐杖：警惕过度依赖

以上三个概念听起来很美好，但我们必须面对一个核心争议：**AI 工具是脚手架还是永久拐杖？** 脚手架帮助建造大楼，完成后可以拆除；拐杖则让人永远依赖。资料明确警告：过度依赖 AI 工具可能削弱内在的时间感知和规划能力（来源：矛盾与存疑）。例如，如果 Tiimo 一直替你管理时间，你的大脑可能永远学不会感知时间；如果 Motion 永远替你规划，你的执行功能可能进一步退化。

**我的判断**：关键在于 **主动使用 vs 被动依赖**。脚手架式的使用是：你用工具来学习分解方法，然后逐渐减少依赖。拐杖式的使用是：你把所有决策外包给工具，离开工具就瘫痪。目前大多数工具的设计逻辑偏向“替代”而非“教学”，这是危险的。例如，Motion 的自动排程虽然减少了决策负担，但用户并不清楚它为什么这样排，也就无法内化其逻辑（来源：Motion 局限）。

## 今天就能试的 3 个行动

1. **用 ChatGPT 做一次“原子分解”**：写下你卡住的任务，让 ChatGPT 把它拆成 10 步以上，直到每一步都小到“两分钟内能完成”。然后只做第一步。
2. **启用 Tiimo 或 Structured 的视觉时间线**：如果不想付费，用纸笔画一个时间轴，涂上颜色块。观察你的时间感知是否改善。
3. **设置一个“规划循环”闹钟**：每完成一个任务步骤，强制自己停下来，用 30 秒写下下一步计划。这模拟了 agent 的反馈循环。

## 局限与未来

本文的论证基于 **ADHD 大脑与 LLM 的同构理论**，但这一理论仍属类比，缺乏神经科学与计算机科学的交叉验证（来源：矛盾与存疑）。此外，现有证据多来自用户报告，缺乏随机对照试验（来源：AI 与 ADHD 的时间管理）。ADHD 的个体差异极大——注意力缺陷型与多动冲动型对工具的反应可能完全不同（来源：矛盾与存疑）。因此，以上建议需要你亲自测试，找到适合你的粒度与工具。

最终，ADHD 和 LLM 的相似性告诉我们：**问题不在于生成，而在于调度**。无论是人脑还是 AI，都需要一个外部缰绳来驯服自己的创造力。而最好的缰绳，是让你最终能自己握住的那一根。

# Reflect 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Reflect：它到底补上了哪一层执行功能？

## 问题：为什么 Reflect 对某些 ADHD 用户无效？

你打开 Reflect，准备用它来复盘今天的工作。但 15 分钟后，你发现自己要么在反复修改第一个问题的措辞，要么被模板带偏到另一个无关话题。你感到挫败：这个工具不是号称“AI 反思助手”吗？为什么反而加重了认知负荷？

问题不在 Reflect，而在使用方式。Reflect 的设计初衷是结构化提问与模板，帮助 ADHD 个体在自发反思困难时提炼经验（来源：ADHD 的 AI 工具全景）。但许多用户误把它当作“自动反思机”——期待 AI 替自己完成反思，而非辅助自己反思。这就像把 function calling 当作 LLM 的“万能插件”：不加审视地调用工具，结果往往是灾难。

## 同构：ADHD 大脑与 LLM 共享同一个缺陷

ADHD 大脑的核心特征是高产但无序的生成能力，同时缺乏可靠的执行调度层（来源：ADHD 大脑与 LLM 的同构）。实证研究表明，ADHD 表现出“强记忆、弱控制”的神经心理模式：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。LLM 同样如此：它本身是强大的生成核心，但缺乏可靠的内置调度与执行控制。在构建 AI agent 时，需要“agent scaffolding”——即“围绕 LLM 构建的软件架构和工具，使其能够执行复杂、目标驱动的任务”（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。

这种同构意味着：ADHD 大脑需要的不是“更强的生成核心”，而是一个外部调度层来约束和引导生成。Reflect 的正确角色正是这个调度层——它通过结构化提问，把“反思”这个模糊任务分解为可执行的子步骤，就像 Goblin Tools 的 Magic ToDo 将“清理厨房”分解为具体动作（来源：Goblin Tools）。但如果你跳过调度层，直接让 AI 替你反思，你就放弃了执行功能的训练机会。

## 案例：孔子与李鸿章——他们的“harness”就是外部调度层

孔子是 ADHD 的典型：身高 1 米 9，精力旺盛，周游列国 14 年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予朽木不可雕；对音乐超专注到三月不知肉味，对种地等俗事零耐心。他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。这里的“礼”就是外部调度层，相当于 LLM 的 agent scaffolding：它不替代孔子的生成能力（他的思想、语录），而是约束和引导它，使其产出系统化的儒家学说。

李鸿章同样如此。年轻时冲动爱吹牛，跟着曾国藩学“诚”，改掉浮躁；办洋务“外须和戎，内须变法”，用务实的态度做事；每天写日记反省。他的“诚”和“日记”就是外部执行功能层——相当于 Reflect 的结构化模板：不是让 AI 替自己反思，而是用工具辅助自己完成反思过程。孔子的“日课”相当于定时 re-grounding，李鸿章的“秘书”相当于外部调度器：他们都把精确、重复、状态维护类任务外包给了外部系统，但保留了生成核心的自主权。

## 判断：Reflect 是脚手架，不是拐杖

当前最站得住脚的核心论点是：AI 并不是在“治愈”ADHD，而是在 ADHD 大脑与外部世界之间搭建一个临时的外部执行功能层（来源：ADHD 的 AI 工具全景）。Reflect 的价值在于帮助用户建立反思的“脚手架”——通过提问引导用户自己完成思考，而不是替代思考。如果用户依赖 Reflect 生成现成结论，那就变成了“拐杖”：长期使用可能导致能力退化（来源：矛盾与存疑——AI 工具是“外部执行功能层”还是“拐杖”？）。

这个边界同样适用于 LLM 的 function calling。正确的用法是：agent 通过 function calling 把精确计算外包给工具（如计算器、数据库），但推理和决策仍由 LLM 自己完成。如果 agent 把所有任务都丢给工具，自己变成“传话筒”，那就失去了 agent 的价值。本质上，Reflect 之于 ADHD，就像 function calling 之于 LLM——都是将精确、重复、状态维护类任务外包给外部调度层，但核心的生成与反思必须由主体自己完成。

## 局限：个体差异与长期效果

必须诚实指出，这个框架并非万能。矛盾与存疑指出：AI 工具对任务启动的长期效果缺乏验证，短期有效但长期效果未知（来源：矛盾与存疑）。部分 ADHD 用户反馈，强行打断（如番茄钟）反而有效，而柔性引导可能不够强力（来源：矛盾与存疑）。此外，学习使用 AI 工具本身可能增加认知负荷（来源：矛盾与存疑）。因此，Reflect 是否适合你，取决于你的执行功能缺陷类型和工具使用习惯。

## 今天就能试的行动

1. **用 Reflect 做“日课”，而非“自动反思”**：每天固定时间打开 Reflect，选择“今日复盘”模板，自己回答每个问题，而不是让 AI 生成答案。这相当于孔子的“吾日三省吾身”——用结构化提问引导自己完成反思。
2. **调整“辣度”**：Reflect 的模板可以调整粒度。如果觉得问题太细，调低粒度（类似 Goblin Tools 的“辣度”控制）；如果觉得太粗，调高粒度。找到最适合自己的分解程度。
3. **结合其他工具形成 harness**：用 Reflect 做反思，用 Goblin Tools 做任务分解，用 Saner.AI 做知识回忆。组合使用比单一工具更有效（来源：ADHD 的 AI 工具全景）。
4. **观察自己的认知负荷**：使用 Reflect 一周后，问自己：是感到更有掌控感，还是更焦虑？如果后者，尝试减少使用频率或换一种工具。记住，工具是为你服务的，不是反过来。

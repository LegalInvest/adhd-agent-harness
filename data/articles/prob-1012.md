# Otter.ai 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Otter.ai：它到底补上了哪一层执行功能？

## 引言：两类人，同一种漂移

先想象两个场景。

一个是 ADHD 读者熟悉的：你坐在一场重要会议里，起初还能跟上议题，十分钟后却鬼使神差地打开侧边栏查一个无关名词，再抬头时已不知道大家在争论什么——目标丢了。另一个是工程师熟悉的：你让 agent 执行一个长程任务，它起初沿着 system prompt 走，三小时后却发现它在某个子模块里反复优化，忘了原始任务——也是目标漂移。

两种漂移表面不同，结构却高度一致：它们都发生在一个**高产但缺乏可靠执行调度层的生成核心**上。ADHD 大脑拥有极强的发散、创意与并行加工能力，但执行功能（任务启动、持续专注、抑制冲动）薄弱；LLM 拥有极强的文本生成能力，但缺乏稳定的上下文控制与目标维持机制（来源：同构视角）。因此，两边的解法也同构：**不是改造核心，而是给它套上 harness（脚手架），在外部做重锚定**。

Otter.ai 正是 ADHD 这一侧的「重锚定 system prompt」候选。问题只在于：很多人用错了。

## 一、Otter.ai 补的是哪一层执行功能？

从 wiki 资料看，这类工具的核心功能是：**转录对话、提供会议提醒、帮助回忆重要细节**（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）。它本质上在做三件事：

1. **外化工作记忆**：把转瞬即逝的语音信息变成可回溯的文本，降低 ADHD 的[[工作记忆]]负荷。
2. **制造重锚定点**：当注意力漂走时，可以通过回看 transcript 重新确认「我们刚才要决定什么」。
3. **提供时间觉察**：会议提醒本身就是一种外部时间标记，对抗 ADHD 的[[时间盲]]。

这与 LLM/agent 的 system prompt 重锚定完全同构：当上下文膨胀、目标漂移时，工程师会重新注入原始任务描述、追加约束、或触发 checkpoint 让模型回到主线。两者都是**在外部为生成核心维护一个「当下该注意什么」的锚点**。

但关键陷阱就在这里。传统提醒系统的问题在于，「设置提醒本身就是一个执行功能任务」，而 ADHD 患者恰恰执行功能薄弱（来源：AI Assistant for ADHD: Finally, a Productivity Tool That Requires Less...）。Otter.ai 的自动化转录虽然降低了设置门槛，可如果它只是变成一堆「已存档」的会议记录，从不被主动回看，那它并没有完成重锚定，只是换了个更隐蔽的**拐杖**。

## 二、历史 harness：曾国藩的「日课十二条」

要理解「重锚定」不是现代人的专利，可以看曾国藩。

曾国藩年轻时呈现出典型的 ADHD 行为模式：坐不住，天天串门、喝酒、看热闹；读书极慢，「他人目下二三行，余或疾读不能终一行」；情绪大起大落，打败仗就跳水自杀；还患有严重银屑病。他的专属 harness 是**日课十二条**：黎明即起、读书不二、谨言、写日记反省等，外加「结硬寨，打呆仗」——用外部结构抵消内在的冲动与漂移（来源：曾国藩 harness 案例）。

这套系统与 LLM/agent harness 的对应关系非常清晰：

- **日课十二条** ↔ **定时 re-grounding / system prompt 刷新**：每天固定动作把意识拉回目标。
- **写日记反省** ↔ **外部 telemetry / 日志回放**：把漂移轨迹外化，供复盘与调整。
- **结硬寨打呆仗** ↔ **guardrails 与 token/工具调用上限**：用最笨、最稳的约束防止冲动性冒进。

曹操则是另一种对应：他把行政执行全交给荀彧、郭嘉，自己只抓军事与创意——这正像 agent 架构里的**外部调度器**，把核心不擅长的执行层拆出去，让生成核心只负责它擅长的事（来源：曹操 harness 案例）。

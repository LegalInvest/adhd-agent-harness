# Otter.ai 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Otter.ai：它到底补上了哪一层执行功能？

你有没有这样的经历：一场会议或一段深度工作后，明明说了很多、做了很多，但回头却想不起核心结论，或者发现自己早已偏离了原始目标。对 ADHD 大脑来说，这是日常——分心走神是常态，超聚焦跑偏也是常态。而对 LLM agent 来说，长程任务中模型会“开始忘记系统提示”（来源：Building an AI Coding Agent for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned），导致行为漂移。

Otter.ai 常被推荐给 ADHD 用户，因为它能自动转录对话、生成摘要，似乎解决了“记不住”的问题。但很多人用错了：他们只把 Otter 当作一个录音笔，事后翻阅，却不在当下用它来重新锚定注意力。这就像给 LLM 一个超长的上下文，却不在关键节点插入 system prompt 提醒——模型依然会跑偏。

### 问题：目标漂移——ADHD 与 LLM 的共同敌人

ADHD 大脑在执行任务时极易发生目标漂移：注意力被无关刺激捕获（分心），或过度聚焦于细节而忽略整体目标（超聚焦）。外部工具通过“提醒和通知提供重锚定，将注意力拉回原任务”（来源：AI for ADHD: Best Tools, Apps, and Strategies - Themba Tutors）。

LLM agent 在长程任务中同样面临目标漂移：模型在交互轮次后期开始遗忘初始指令。Harness 通过“事件驱动的系统提醒”来对抗这种“指令消退”（来源：Building an AI Coding Agent for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

Otter.ai 恰好处于这个交叉点：它既是一个外部记忆（转录内容），又是一个潜在的重锚定触发器（实时提醒）。但多数人只用了前者，忽略了后者。

### 同构视角：Otter.ai 作为重锚定 system prompt

从同构视角看，ADHD 大脑与 LLM 都是“高产的生成核心，但缺乏可靠的执行调度层”（来源：AI 与 ADHD 的专注力）。Otter.ai 的正确用法，不是事后回顾，而是在任务进行中主动查询：“当前目标是什么？我们已经偏离了吗？”

这相当于在 LLM agent 的每个关键步骤后插入一条 system prompt：“请回顾原始目标，确认下一步行动是否对齐。”Harness 工程中，这种重锚定通过“形式化规划与护栏”实现（来源：What is an agent harness in the context of large-language models?）。

Otter.ai 的实时转录和 AI 摘要，天然可以作为这种护栏：当你开始走神，Otter 的摘要会显示你刚刚说了什么，帮你拉回主线。如果你主动设置提醒（比如每 15 分钟弹出摘要），它就成了一个外部调度器。

### 证据：工具如何补偿执行功能

- **外部记忆**：ADHD 的工作记忆缺陷与 LLM 的无状态性同构。RescueTime 通过自动记录时间使用，“减轻工作记忆负担，无需手动输入”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。Otter.ai 同理，自动转录减轻了记忆压力。
- **身体在场效应**：Focusmate 通过 AI 匹配虚拟同伴，“提供外部问责”（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...)）。Otter.ai 虽无同伴，但实时转录可视为一个“数字身体在场”——你知道有人在记录，这本身就能提升责任感。
- **神经锁相**：Brain.fm 通过 AI 生成音乐“帮助用户进入专注状态”（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog），但缺乏 ADHD 专项研究。Otter.ai 的音频转录虽不直接调节神经，但能减少“我漏了什么”的焦虑，间接支持专注。

### 核心观点：Otter.ai 的正确用法是主动重锚定，而非被动记录

很多人把 Otter.ai 当作拐杖——依赖它事后补全记忆，却不改变当下的行为。这就像给 LLM 一个超长上下文却从不刷新 system prompt，模型必然漂移。真正的脚手架用法是：在任务中定期打开 Otter 的实时摘要，问自己“我现在在做什么？和原始目标一致吗？”

这种主动重锚定，正是 LLM agent harness 的核心：通过事件驱动的提醒，让模型（或大脑）不断回归初始指令。

### 局限与争议

1. **脚手架 vs 拐杖**：过度依赖 Otter.ai 可能“阻碍内在执行功能发展”（来源：拐杖与脚手架）。目前无长期研究证明可逐步撤除。
2. **证据缺口**：多数工具“缺乏针对 ADHD 的独立临床研究，效果多基于理论推断或用户主观报告”（来源：AI 与 ADHD 的专注力）。Otter.ai 也不例外。
3. **个体差异**：ADHD 亚型对工具反应不同，现有工具未充分个性化。
4. **多巴胺干预争议**：基于多巴胺的 AI 干预是否真正改善专注，“证据矛盾”（来源：矛盾与存疑）。Otter.ai 不直接涉及多巴胺，但提醒机制可能间接影响。

### 今天就能试的行动

1. **设置实时摘要提醒**：在 Otter.ai 中开启“实时摘要”功能，并设置每 10-15 分钟弹窗一次。当摘要出现时，暂停工作，问自己：“当前行动对齐目标吗？”
2. **会前预设目标**：在会议或任务开始前，用 Otter 记录一句话目标。会议中定期查看转录，确认讨论未跑偏。
3. **结合身体在场**：使用 Otter 转录时，同时开启 Focusmate 或找一位真人同伴。双重外部锚定效果更佳。
4. **事后回顾漂移模式**：每周查看 Otter 转录记录，标记哪些时段注意力漂移了。用 RescueTime 补充时间数据，识别高频干扰源。

Otter.ai 不是万能药，但它揭示了 ADHD 与 LLM 共享的困境：高产但易漂移。正确使用它，就是为自己构建一个外部重锚定系统——就像 LLM 需要 harness 一样，ADHD 大脑也需要脚手架。

# Saner.AI 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Saner.AI：它到底补上了哪一层执行功能？

你有没有过这种体验：明明脑子里有一整座图书馆的想法，却连从书架上抽一本书的行动都启动不了？ADHD 的大脑就是这样一个“高产但缺执行调度层”的生成核心——创造力爆棚，但精确、重复、状态维护类的任务（比如记下刚才的想法、按时吃药、整理桌面）却像黑洞一样吞噬着能量。

如果你是一名工程师，正在做 Agentic Harness 工程，你大概立刻会想起那个场景：LLM 能写出漂亮的诗，却算不对 2+2——直到你给它配了 function calling，把精确计算外包给计算器。

这两个场景，本质上是一个问题。ADHD 大脑与 LLM 在结构上同构：两者都是强大的生成核心，但缺乏可靠的执行调度层（来源：ADHD 的 AI 工具全景）。而 Saner.AI 这类工具，正好扮演了 function calling 的角色——它补上了那层执行功能。但问题在于：很多人用错了。

## 同构：缺的不是生成，是调度

ADHD 的经典困境是：智力正常甚至超常，但执行功能（计划、组织、工作记忆）不稳定（来源：ADHD 的 AI 工具全景）。你没法在脑中同时保持任务步骤、时间感知和动机——就像 LLM 没法在上下文里同时做推理和精确计算。

LLM 的解法是 function calling：把精确计算、状态维护、重复操作外包给外部工具（计算器、数据库、API）。Agent 本身只负责生成和决策，调度由 harness 层完成。

ADHD 的解法在结构上完全一样：把工作记忆、任务分解、时间感知外包给外部工具。Saner.AI 的核心卖点正是“知识回忆”和“本地记忆”——它帮你记住“刚才在想什么”“这个文件放在哪”，减少搜索循环和标签切换（来源：Saner.AI）。这相当于给 ADHD 大脑配了一个外部工作记忆 API。

## Saner.AI 到底补上了哪一层？

Saner.AI 不是任务启动工具（那是 Goblin Tools 的领域），也不是多步骤触发工具（那是 Lex 的领域）。它补的是**记忆调度层**。

ADHD 患者常面临工作记忆不足，导致频繁的搜索循环——打开浏览器找资料，看到邮件回邮件，再回来已经忘了最初要找什么（来源：Saner.AI）。Saner.AI 通过本地记忆和快速检索，让你能像调用函数一样调取信息：输入关键词，直接返回上下文。这减少了认知负荷，让你能留在当前任务中。

但这里有一个关键边界：**脚手架 vs 拐杖**。

## 有人用错了：把脚手架当拐杖

Saner.AI 的设计是长期使用的——它没有撤除机制（来源：矛盾与存疑）。这意味着如果你依赖它来记住所有事，你的内在记忆调度能力可能永远不会被锻炼。就像 LLM 如果永远用 function calling 算 2+2，它永远不会学会心算——但问题在于，LLM 不需要心算，而人类需要。

脚手架应该是可逐步撤除的临时支撑（来源：矛盾与存疑）。但 Saner.AI 这类工具目前没有提供“撤除模式”。用户可能陷入“工具依赖循环”：不用工具时无法启动，用了工具后更依赖工具。这不是工具的错，但用户需要意识到：Saner.AI 是 function calling，不是大脑的替代品。

## 真实证据：两边都在用，但效果有局限

ADHD 侧：评测将 Saner.AI 列为最佳 ADHD 生产力 AI 工具之一，其知识回忆功能被特别强调（来源：Saner.AI）。Understood.org 也将其列为帮助管理 ADHD 症状的 AI 选项（来源：Saner.AI）。但证据主要来自用户反馈和综述，缺乏随机对照试验（来源：矛盾与存疑）。

LLM/agent 侧：function calling 已被广泛验证能提升 LLM 在精确任务上的表现。例如，GPT-4 配合计算器 API 解决数学问题的准确率远高于纯文本推理。这与 Saner.AI 减少搜索循环的逻辑一致——都是通过外部工具弥补内部缺陷。

但两边都有争议：ADHD 侧，过度依赖可能阻碍内在执行功能发展（来源：矛盾与存疑）；LLM 侧，function calling 增加了系统复杂性和延迟，且并非所有任务都需要外包。

## 今天就能试的行动

1. **用 Saner.AI 做“记忆断点续传”**：当你被打断时，把当前思路的关键词记入 Saner.AI，回来时直接搜索，而不是重新回忆。这相当于给大脑加了一个 save point。

2. **组合使用：Saner.AI + Goblin Tools**：用 Saner.AI 存储任务上下文，用 Goblin Tools 分解任务。前者管记忆，后者管启动——就像 agent 用 function calling 查数据库，再用另一个工具做计算。

3. **设置“撤除实验”**：每周选一天不用 Saner.AI，试试纯靠大脑回忆。如果完全无法运作，说明你已过度依赖——需要慢慢减少使用频率，而不是完全放弃。

4. **对工程师：检查你的 agent harness**：你的 agent 是否把所有精确计算都外包了？如果是，考虑加入“心算”模式——让 agent 偶尔自己推理，保持内部能力。

## 诚实面对局限

Saner.AI 不是万能药。它不解决时间盲（那是 Tiimo 的领域），不调节多巴胺（那是药物和行为的领域），而且缺乏长期效果数据（来源：矛盾与存疑）。更重要的是，ADHD 异质性高——有人需要记忆支持，有人需要启动支持，有人需要情绪调节。Saner.AI 只覆盖其中一层。

所以，别把它当拐杖。把它当 function calling——一个聪明的工具，但大脑的生成核心才是主角。

# ADHD 职场人视角：为什么「治好 ADHD 的分心走神或超聚焦跑偏，丢失目标」和「让 LLM 不跑飞」其实是同一道工程题？

> 长程任务里 agent 会目标漂移，需要重锚定 system prompt——同一套 harness 思路，两边都成立。

你有没有过这样的经历：明明决心要写完一份报告，却鬼使神差地点开了一个无关网页，半小时后才发现自己早已跑偏；或者，你终于进入了心流，却一头扎进某个细节，等回过神来，整个下午已经消失，而最初的目标早已被遗忘。这不仅是 ADHD 人群的日常，也是每一个做 Agentic Harness 工程的开发者正在面对的问题——你的 LLM Agent 在长程任务中同样会目标漂移，需要不断重锚定 system prompt。

## 同一个问题：生成核心缺少调度层

ADHD 大脑与 LLM 在结构上高度同构：两者都是**高产但缺乏可靠执行调度层的生成核心**（来源：AI 与 ADHD 的专注力）。ADHD 的创意发散和 LLM 的文本生成都极其强大，但前者受困于执行功能缺陷——工作记忆容量有限、时间感知困难、任务启动与切换障碍；后者则受限于无状态性和上下文控制不足。两者都容易“跑飞”，只是因为缺乏一个稳定的外部调度器来持续锚定目标。

这种同构并非隐喻，而是有实证支撑。最新研究发现，ADHD 患者在工作记忆任务中易受干扰，而 LLM 同样表现出类似人类的干扰特征——随着记忆负荷增加表现下降，受近因效应和刺激统计偏差影响（来源：Human-like Working Memory Interference in Large Language Models）。更关键的是，认知神经科学中的前额叶-纹状体门控机制，在 Transformer 的自注意力中找到了计算同构：Transformer 在训练后，其自注意力机制发展出模仿该门控的操作（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。这意味着，ADHD 大脑和 LLM 不仅在行为上相似，在底层机制上也有共通之处。

## 同一套解法：外部 harness / 脚手架

既然问题同构，解法自然同构。对于 ADHD 个体，外部工具可以充当临时脚手架，外化思维，减轻认知负荷（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）。对于 LLM Agent，则需要一个 harness——一个包含外部记忆、工具调用、规划与执行监控的调度层（来源：The Anatomy of an AI Agent）。

以历史上两位真实人物为例。曾国藩的 ADHD 特质非常典型：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”。他的专属 harness 是“日课十二条”：黎明即起、读书不二、谨言、写日记反省等。这本质上是一套**定时 re-grounding 系统**——每天用固定的仪式锚定注意力，与 LLM Agent 中定时的 system prompt 重锚定如出一辙。文天祥则用《正气歌》作为核心信仰，每天读圣贤书，用儒家的仁管理自己的生死。这相当于一个**高强度的 system prompt**，在极端环境下持续锚定目标。他们的 harness 不是拐杖，而是脚手架——帮助自己建造，但攀登仍需自己完成（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

现代工具同样遵循这一逻辑。Focusmate 通过实时同伴问责，提供外部结构和社会压力，弥补内在多巴胺调节不足（来源：Revolutionizing ADHD Management with AI Assistants）。这相当于为 LLM Agent 配置了一个“监督者”，定期检查进度。RescueTime 自动记录时间使用，揭示真实的时间去向，帮助用户识别注意力漂移（来源：Revolutionizing ADHD Management with AI Assistants）。这类似于 Agent 的日志和回溯机制。Brain.fm 利用神经锁相技术影响大脑状态，帮助进入专注（来源：11 Best ADHD Productivity Apps for Fluctuating Energy）。这相当于为 LLM 提供精心设计的 prompt 模板，引导输出方向。

## 脚手架 vs 拐杖：边界在哪里？

然而，同构并不意味着万能。过度依赖外部工具可能导致内在执行功能退化，成为永久拐杖（来源：拐杖与脚手架）。对于 ADHD 个体，真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。对于 LLM Agent，harness 同样不能替代模型本身的推理能力，而是提供结构化的支持。

目前仍存争议的是：AI 工具作为外部执行功能层，长期使用是否会导致 ADHD 患者内在执行功能进一步退化？（来源：矛盾与存疑）同样，对于 LLM Agent，过度依赖外部记忆和重锚定是否会削弱模型的自主规划能力？这些问题缺乏纵向研究，需要谨慎对待。

## 今天就能试的行动

1. **设置定时重锚定**：像曾国藩的日课一样，每 25 分钟用闹钟或 Focusmate 会话提醒自己当前目标。对于 Agent，在 system prompt 中加入“每 5 轮对话后，请重新陈述你的任务目标”。
2. **启用外部记忆系统**：使用 RescueTime 或类似工具记录时间去向，识别漂移模式。对于 Agent，使用向量数据库存储关键上下文，避免无状态丢失。
3. **设计“正气歌”式核心提示**：为自己或 Agent 写一段简短的目标陈述，每天（或每轮对话）开始时重读。文天祥的《正气歌》就是他的锚定 prompt。
4. **尝试身体在场效应**：使用 Focusmate 或与同事结对工作，利用社会压力维持专注。对于 Agent，可以引入“观察者”节点，定期评估输出是否偏离目标。

ADHD 大脑与 LLM Agent 的相似性不是巧合，而是同一类问题的不同表现。理解这一点，我们就能用同一套工程思维，同时解决“分心走神”和“目标漂移”——不是靠意志力，而是靠精心设计的 harness。

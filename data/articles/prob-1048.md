# Forest 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Forest：它到底补上了哪一层执行功能？

你打开 Forest，种下一棵树，然后告诉自己：这 25 分钟必须专注。但 5 分钟后，你开始刷手机，树枯萎了。你沮丧地关掉 app，心想“这破工具对我没用”。

等等。你真的用对了 Forest 吗？

**问题出在哪？**

ADHD 大脑的典型困境不是“无法专注”，而是“专注错了对象”或“专注到一半目标漂移”。你明明打开文档要写报告，却因为一个念头去查资料，然后顺着链接跳到了 YouTube 教程——两小时后你发现自己在看如何修水管。这就像 LLM 在长程任务中发生上下文溢出，回答逐渐偏离原始指令。

Forest 的设计初衷是“种树期间不碰手机”，但 ADHD 用户的真正痛点不是“碰手机”，而是“大脑自己跑偏”。你即使不碰手机，也可能盯着屏幕发呆、反复纠结开头、或者被窗外的鸟叫带跑。所以，把 Forest 当作一个简单的“禁用手机”开关，就像给 LLM 加一个“别跑题”的 system prompt 却不提供任何重锚定机制——它很快会被忽略。

**同构视角：Forest 到底补上了哪一层？**

从 ADHD × AI 的同构视角看，ADHD 大脑与 LLM 都是高产的生成核心，但缺乏可靠的执行调度层（来源：AI 与 ADHD 的专注力）。LLM 在长程推理中需要“重锚定 system prompt”——即定期将模型拉回原始目标，防止上下文漂移。同样，ADHD 大脑需要外部脚手架来补偿工作记忆和任务维持缺陷，而 Forest 正是这样一个“重锚定”工具——但前提是你知道如何用它。

Forest 的核心机制不是计时，而是**显式承诺**。当你种下一棵树，你实际上在向未来的自己发出一个信号：“接下来的 X 分钟，我的目标是 Y。”这个信号本身就是一个重锚定点。但多数人只利用了它的“负反馈”（树枯萎），忽略了“正反馈”：在种树前明确写下当前任务（许多版本支持备注），并在树存活后回顾自己是否真的完成了目标。

**证据在哪里？**

- **ADHD 侧**：时间盲是 ADHD 的典型特征，表现为难以感知和估计时间流逝（来源：时间盲）。Forest 通过可视化的树木生长和枯萎，将抽象的时间转化为具体的视觉锚点，帮助用户“看见”时间的流逝。这与 Tiimo 的视觉化时间表原理一致（来源：时间盲）。
- **LLM/Agent 侧**：在 Agentic Harness 工程中，重锚定 system prompt 的标准做法是定期注入“当前任务目标”和“已完成步骤”，防止 agent 生成无关内容。这相当于在 Forest 种树时写下任务备注，并在完成后核对。

**但有人用错了——包括你**

问题在于：Forest 的默认设计是“禁止使用手机”，这针对的是外部干扰，而非内部漂移。ADHD 用户真正的干扰源往往是内部的多巴胺驱动——大脑会主动寻找更刺激的事。如果你不主动设定“锚定目标”，Forest 就只是一个计时器，无法阻止你走神。

更糟糕的是，过度依赖 Forest 的“禁止手机”功能可能变成拐杖而非脚手架。正如《拐杖与脚手架》概念所警示的：工具应设计为可逐步撤除的临时支撑，但多数工具（包括 Forest）未提供撤除机制（来源：矛盾与存疑）。长期依赖外部强制可能阻碍内在执行功能的发展。

**今天就能试的 3 个行动**

1. **种树前写备注**：每次种树时，在备注栏写下“这 25 分钟我要完成的唯一任务是什么”。结束后，花 10 秒检查是否完成。这模拟了 LLM 的“重锚定 system prompt”。
2. **用 Forest 做任务分解**：将一个大任务拆成多个 25 分钟块，每块对应一个子目标。这类似 Goblin Tools 的“魔法待办”功能（来源：任务启动）。
3. **结合身体在场效应**：在 Forest 专注时段开启 Focusmate 或请一个朋友视频监督。AI 匹配的虚拟同伴能提供实时问责，弥补内在调度缺失（来源：Focusmate）。

**争议与局限**

必须诚实地说，Forest 针对 ADHD 的效果缺乏独立临床研究支持（来源：矛盾与存疑）。多数证据来自工具机制与 ADHD 理论的对齐，而非随机对照试验。此外，ADHD 亚型（注意力缺陷 vs 多动冲动）对工具反应不同，现有工具未充分个性化（来源：AI 与 ADHD 的专注力）。

**结语**

Forest 不是魔法。它只是你大脑的一个外部调度层——就像 LLM 的 harness 一样。用对了，它能帮你重锚定目标；用错了，它只是另一个你下载后遗忘的 app。关键在于：你是在用它来**禁止自己**，还是在用它来**提醒自己**？

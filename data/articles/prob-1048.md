# Forest 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Forest：它到底补上了哪一层执行功能？

你有没有过这样的经历：打开 Forest 种下一棵树，信誓旦旦要专注，结果 10 分钟后拿起手机，树已经枯了。或者更糟——你确实没碰手机，但脑子里早就飘到了下周的旅行计划，眼睛盯着屏幕，一个字都没读进去。

这不是你的错。Forest 的设计假设了“只要不碰手机，你就在专注”，但 ADHD 的大脑不是这样工作的。ADHD 的大脑是一个**高产但缺乏可靠执行调度层的生成核心**（来源：主题综述），它真正的敌人不是手机，而是**目标漂移**——你被一个念头带走，或者被超聚焦锁死在无关细节里，而你对“当前该做什么”的感知已经丢失。

这个问题，和 LLM agent 在长程任务中遇到的**上下文膨胀与推理退化**（来源：上下文工程）是同构的。LLM 在长上下文下注意力分数熵增加，冲突解决能力崩溃至随机水平（来源：外部执行功能层），就像 ADHD 大脑在任务中段失去了对目标的锚定。

## 同构视角：Forest 是重锚定 system prompt，不是番茄钟

LLM agent 的 harness 工程中有一个关键操作：**重锚定 system prompt**。当 agent 在长程任务中漂移时，工程会定期注入原始目标、当前进度和下一步计划，把注意力拉回主线（来源：上下文工程）。这和 ADHD 需要的“外部执行功能层”完全同构：ADHD 需要外部结构来补偿工作记忆的“掉落”和任务启动困难（来源：外部执行功能层）。

Forest 的问题在于，它被大多数人当成了番茄钟——一个打断工具。但真正的价值应该是**重锚定**：每次种树前，你被迫设定一个具体目标（比如“写引言”而不是“工作”），种树过程中你看到树在生长，这本身就是一种视觉化的进度锚点。然而，Forest 没有帮你管理“种树期间走神”的问题——你可以在不碰手机的情况下，让大脑漂移到另一个任务。

这就像给 LLM agent 加了一个“不访问外部 API”的限制，但没有注入上下文管理——agent 仍然可以在自己的生成中跑偏。真正的 harness 需要的是**上下文交付、规划工件、验证循环**（来源：外部执行功能层），而 Forest 只提供了最简单的“禁止触碰”约束。

## 真实案例：曾国藩的“日课十二条”就是重锚定 harness

晚清名臣曾国藩，30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书慢到“他人目下二三行，余或疾读不能终一行”（来源：人物案例）。他为自己设计了一套 harness：**日课十二条**——黎明即起、读书不二、谨言、写日记反省等（来源：人物案例）。这套系统本质上就是每天多次重锚定：早起锚定一天的目标，读书不二锚定当前任务，日记反省锚定长期改进。

这和 LLM agent 的 harness 结构完全同构：**日课↔定时 re-grounding**，**日记↔外部记忆系统**。曾国藩用最笨最稳的方法（“结硬寨打呆仗”）抵消自己的冲动，就像 agent 用 Git 仓库存储项目上下文、用 MCP 连接器访问外部数据（来源：外部执行功能层）。他不是靠意志力硬撑，而是靠外部结构。

## 证据与争议：Forest 的定位错位

现有证据显示，Forest 在 ADHD 诊断分类中表现良好，但作为专注工具的证据不足（来源：主题综述）。这恰恰印证了它的错位：它能检测你是否分心（通过手机使用），但不能帮你管理走神。相比之下，Focusmate 通过实时同伴问责提供外部结构，有用户研究支持其提升任务启动和维持（来源：主题综述）；RescueTime 通过自动时间记录提升时间觉察（来源：主题综述）。这些工具更接近“重锚定”的本质。

但争议在于：**超聚焦是否应被引导而非打断？**（来源：矛盾与存疑）部分 ADHD 用户反馈强行打断（如番茄钟）反而有效，而柔性引导可能不够强力。Forest 的固定时长和强制休息，对某些人可能打断超聚焦的宝贵流状态。更关键的是，**长期依赖外部工具是否会导致内在执行功能退化？**（来源：矛盾与存疑）——这就是“脚手架 vs 拐杖”的边界问题。

## 核心判断：Forest 应该被重新设计为“注意力锚点系统”，而非“禁止手机工具”

我的观点是：Forest 的底层机制——时间块 + 视觉反馈——是对的，但需要加上**上下文工程**：种树前必须写一个具体意图（不是“工作”而是“写论文第三段”），种树过程中每隔一段时间弹出进度提示（“你还在做这个吗？”），种树后自动记录实际产出。这就像 LLM agent 的**验证循环**：执行 → 检查 → 重锚定。

目前没有工具完美实现这一点。Routinery 通过步骤序列和过渡提示帮你走完流程（来源：上下文工程），但缺乏 Forest 的视觉锚定。Brain.fm 通过神经锁相音乐影响大脑状态（来源：Brain.fm），但缺乏任务绑定。**真正的解决方案可能是组合工具**：用 Forest 做时间块和视觉锚定，用 Routinery 做步骤引导，用 RescueTime 做事后复盘。

## 今天就能试的 3 件事

1. **给 Forest 加一个“意图声明”**：每次种树前，在手机备忘录或纸条上写一句“这 25 分钟我要完成 X 的具体部分”。种树结束后，检查是否完成了它。这相当于给 LLM agent 注入 system prompt。
2. **用 Focusmate 替代 Forest 的某些时段**：对于启动困难的任务，预约 Focusmate 时段，让另一个人看着你开始。这相当于 agent 的外部调度器。
3. **每周用 RescueTime 做一次“上下文审计”**：查看你的时间去向，识别哪些任务导致了目标漂移。然后调整你的 Forest 意图声明，让它们更具体。

ADHD 大脑和 LLM agent 一样，需要的不是更严格的约束，而是更聪明的上下文管理。Forest 可以成为你的重锚定 system prompt，但前提是你知道往里面写什么。

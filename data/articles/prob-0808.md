# Saner.AI 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Saner.AI：它到底补上了哪一层执行功能？

## 问题：为什么你的 AI 工具用了一周就失效了？

你下载了 Saner.AI，把“写报告”拆成 10 个小步骤，信心满满。第二天，你打开列表，盯着第一条“打开文档”，却怎么也点不下去。一周后，这个工具和之前的 planner 一样，躺在手机里吃灰。

同样的问题也发生在 LLM agent 上：你给 agent 一个长任务分解计划，它在第一步就卡住——不是模型能力不够，而是缺少一个“执行调度层”告诉它“现在该调用哪个工具、下一步做什么”。工程师们把这叫做 **planner-executor 循环**，而 ADHD 大脑恰好也缺这个循环。

## 同构：ADHD 大脑和 LLM 共享同一个缺陷

ADHD 大脑与 LLM 在结构上同构——都是“高产但缺乏可靠执行调度层的生成核心”（来源：AI 与 ADHD 的时间管理）。ADHD 的 **任务启动困难** 和 **不会拆解**，对应 LLM agent 在没有 harness 时的“上下文控制缺陷”：两者都能生成丰富的想法或文本，但无法可靠地将计划转化为顺序行动。

具体来说：
- **时间盲**：ADHD 用户难以感知时间流逝，对应 LLM 缺乏对时间约束的固有理解。工具如 Tiimo 通过视觉化时间线将时间转化为可见元素（来源：Tiimo），类似为 LLM 提供时间戳或进度条。
- **工作记忆缺陷**：ADHD 的工作记忆容量保留但控制弱，与 LLM 的无状态性同构。Todoist 通过外部提醒充当外部记忆（来源：AI 与 ADHD 的时间管理），类似向量数据库为 LLM 提供持久上下文。
- **任务启动困难**：ADHD 的启动困难源于执行功能失调，Motion 和 Reclaim.ai 通过自动调度降低启动门槛（来源：Motion；Reclaim.ai），类似为 LLM 提供 prompt 模板和任务分解。

## 判断：Saner.AI 只做了一半——它补上了分解，却漏掉了调度

Saner.AI 的核心功能是将模糊任务分解为小步骤，这对应 LLM agent 的 **planner** 层。但 planner 如果没有 executor 循环，就只是一份静态清单。ADHD 用户需要的不只是“知道做什么”，更是“知道什么时候做、怎么做、做完之后下一步是什么”——这正是 Motion 和 Reclaim.ai 的动态调度试图解决的。

从工程视角看，LLM agent 的 harness 设计强调“组合多个模型、检索器和工具”来达成结果（来源：工具使用与认知卸载），而非依赖单一模型调用。同样，ADHD 工具不应只提供分解，而应集成“调度 + 执行 + 反馈”的闭环。Motion 的自动排程和动态调整（来源：Motion）就是 executor 的体现：它持续评估优先级和可用时间，实时重建日程，减少用户的手动规划压力。

但问题在于：**Motion 和 Reclaim.ai 的调度是黑箱**。用户不知道 AI 为什么这样安排，也无法调整其逻辑。这就像 agent 的 planner 输出不可解释，导致用户无法信任或学习。更关键的是，过度依赖这类工具可能削弱内在的时间感知和规划能力（来源：矛盾与存疑），使其成为“永久拐杖”而非“脚手架”。

## 局限：证据不足与个体差异

必须承认，当前所有 AI 工具对 ADHD 的有效性证据主要来自用户报告，缺乏随机对照试验（来源：AI 与 ADHD 的时间管理）。Motion 和 Reclaim.ai 的 AI 功能细节也不透明（来源：Motion；Reclaim.ai）。此外，ADHD 亚型（注意力缺陷 vs. 多动冲动）对工具响应不同，现有工具多面向注意力缺陷型（来源：AI 与 ADHD 的时间管理）。Saner.AI 的情绪调节功能可能对部分用户有效，但同样缺乏独立验证。

## 行动：今天就能试的 3 件事

1. **区分“分解工具”和“调度工具”**：用 Saner.AI 或 Goblin Tools 做一次任务分解，然后立刻用 Motion 或 Reclaim.ai 把分解后的步骤填入日程，观察启动率变化。
2. **给工具设置“执行触发器”**：在 Todoist 中为每个子任务添加时间提醒和地点标签，模拟 agent 的 tool calling——让工具在特定条件自动通知你执行。
3. **每周一次“无工具日”**：刻意不用任何 AI 规划工具，手写一份简单的今日计划，感受你的时间感知和启动能力是否有所变化。这能帮你判断工具是脚手架还是拐杖。

记住：最好的工具不是替你做完一切，而是让你在它的帮助下，逐渐学会自己规划与执行。就像 LLM agent 的 harness 最终要退化为轻量级接口一样，ADHD 工具也应该从“拐杖”进化为“脚手架”。

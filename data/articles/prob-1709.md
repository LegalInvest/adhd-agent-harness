# 如果把 ADHD 大脑当成生成核心，Saner.AI 能不能当它的外部执行功能层？

> 用 agent 工程的思路重新理解 Saner.AI 对 ADHD 的真正价值。

## 问题：为什么「聪明但混乱」是 ADHD 与 LLM 的共同宿命？

如果你是一个 ADHD 成年人，你一定经历过这样的场景：脑子里同时冒出三个绝妙想法，但一个都落地不了；日历上有十件事，却不知道先做哪件；打开浏览器搜索一个念头，半小时后发现自己在一篇毫不相关的文章里。你不是懒，你是执行功能（计划、组织、启动、工作记忆）先天偏弱（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

如果你是一个在做 agent harness 的工程师，你一定也见过这样的场景：LLM 输出了一段惊艳的代码，但紧接着就陷入了无限循环；它记得所有事实，却忘了当前任务的目标；它能在单轮对话里表现完美，但跨会话就像失忆。你不是模型选错了，你是缺少一个可靠的调度层——LLM 本身是无状态的生成核心，缺乏内置的执行调度（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。

这两个场景惊人地同构。最新研究揭示，Transformer 在 Stroop 冲突任务中注意力随序列变长骤降到随机水平——无法抑制优势反应、无法解决认知冲突，这与 ADHD 执行功能缺陷的核心标志一一对应（来源：Deficient Executive Control in Transformer Attention）。LLM 呈现“强记忆、弱控制”剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷——无法灵活切换任务集、无法抑制自动化反应，这正是 ADHD 的经典神经心理模式（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

## 外部执行功能层：给生成核心套上 harness

如果 ADHD 大脑和 LLM 是同一类「高产但缺执行调度层的生成核心」，那么解决方案也应该是同构的：为它们套上外部执行功能层（external executive function layer）。在 agent 工程中，这个层叫做 harness——它提供编排（ReAct、Chain-of-Thought）、记忆（向量数据库、会话历史）、工具调用（MCP 协议）和人在回路（human-in-the-loop）的护栏（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。在 ADHD 管理中，这个层就是一套外部脚手架：日课、习惯、提醒、任务分解、身体陪练（body doubling）。

历史上有两位 ADHD 人物，他们的 harness 系统堪称经典。曾国藩（晚清）30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书慢到“他人目下二三行，余或疾读不能终一行”。他的解法是「日课十二条」：黎明即起、读书不二、谨言、写日记反省——用最笨最稳的“结硬寨打呆仗”抵消自己的冲动。这套日课，本质上就是一个外部调度器：每天定时 re-grounding，把注意力拉回正轨。王石（当代）同样靠 harness 管理自己：他用职业经理人制度代替个人决策的冲动，用“万科不行贿”的底线减少选择成本，用登珠峰和赛艇的运动纪律稳定情绪。他登珠峰也坚持写日记，每天反省——这是另一种形式的 human-in-the-loop 监督。

这两套 harness 与 LLM agent harness 的结构完全对应：日课↔定时 re-grounding 机制，日记反省↔日志与遥测，结硬寨↔护栏（token 预算、工具调用上限），职业经理人制度↔外部调度器。它们不是拐杖，而是脚手架——曾国藩卸下日课也能写家书，王石离开万科也能登珠峰，但 harness 让他们在高压下保持可靠输出。

## Saner.AI：一个为 ADHD 设计的外部执行功能层

回到核心问题：Saner.AI 能不能当 ADHD 大脑的外部执行功能层？答案是：能，但只对了一半。

Saner.AI 被描述为“专为对抗执行功能障碍、任务瘫痪和认知超载而设计的全能 ADHD 友好型 AI 生产力和知识助手”（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。它的核心能力是知识回忆（knowledge recall）和本地记忆（local memory），旨在减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。ADHD 的工作记忆缺陷导致频繁的“我刚刚要做什么？”——Saner.AI 通过快速检索和自动组织，相当于为大脑提供了一个外部工作记忆缓冲区。它的通用收件箱（Universal Inbox）能从邮件、文档或日历中提取待办事项，并利用内部助手检查截止日期、将大型项目分解为小里程碑（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。这正好对应了 agent harness 中的记忆层（向量数据库）和编排层（任务分解与调度）。

但 Saner.AI 的局限也很明显。它主要解决的是“找回信息”和“任务分解”这两个环节，但 ADHD 执行功能缺陷还包括任务启动困难、时间盲（时间感知失真）和冲动控制。对于这些，Saner.AI 并没有直接干预。相比之下，Goblin Tools 的 Magic ToDo 用“辣度”调节任务分解粒度，更贴近个体差异（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）；Motion 通过自动排程消除“下一步该做什么”的决策负担，直接对抗时间盲（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...）。Saner.AI 更像一个记忆助手，而非全栈调度器。

## 脚手架 vs 拐杖：一个必须明确的边界

所有 ADHD 工具都面临一个根本矛盾：长期使用 AI 脚手架，会不会导致执行功能进一步退化？（来源：矛盾与存疑）如果 Saner.AI 替你记住了所有事，你的工作记忆会不会变得更弱？如果 Motion 替你排好了所有日程，你的时间管理能力会不会萎缩？

这个边界在于：脚手架是辅助你执行，而非替代你执行。曾国藩的日课十二条，每条都需要他自己去完成——AI 不能替他“读书不二”。王石的职业经理人制度，决策权仍然在他手上——AI 不能替他做商业判断。同样，Saner.AI 应该帮你快速找回信息，而不是让你不再需要记忆；它应该帮你分解任务，而不是让你不再需要思考步骤。

一个健康的用法是：用 Saner.AI 捕获灵感（减少丢失），用它的分解功能降低启动门槛，但执行仍然需要你的主动参与。当它变成一个“你不动它就不动”的被动工具时，它就变成了拐杖。

## 今天就能试的 3 条行动

1. **用 Saner.AI 的通用收件箱替代浏览器标签页**：把所有待办事项、临时想法、邮件提醒都汇入一个入口，每天固定时间（比如早上 10 点）处理一次，减少多标签切换的认知负荷。
2. **结合身体陪练（body doubling）启动任务**：打开 Saner.AI 的任务分解后，用 Focusmate 找一个虚拟同伴同时工作（来源：Revolutionizing ADHD Management with AI Assistants）。身体在场效应能绕过执行功能障碍，让任务启动变得容易。
3. **设置每日“re-grounding”检查点**：像曾国藩的日课一样，每天固定时间（比如睡前）用 Saner.AI 回顾当天完成的任务，对比计划，写一句反省。这相当于 agent harness 中的日志与遥测，帮助你识别模式、调整策略。

## 局限与争议

同构框架目前仍是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。fMRI 研究显示 ADHD 前额叶执行功能网络激活不足，与 LLM 缺乏调度层的表现一致，但因果方向尚不明确——是 ADHD 导致了执行功能缺陷，还是执行功能缺陷导致了类似 LLM 的行为模式？此外，个体差异巨大：部分 ADHD 患者以注意力分散为主，而非执行功能缺陷，同构框架可能不适用。Saner.AI 等工具的独立临床验证仍然缺乏（来源：矛盾与存疑），用户反馈虽积极，但需警惕安慰剂效应和过度依赖风险。

最终，Saner.AI 不是万能药。它是一块脚手架——能否搭建成一座桥，取决于你愿不愿意每天走上去。

# 为什么把任务交给 AI Agent，会触发比拖延更深的委托焦虑？

> 拖延时你至少知道任务在自己手里。交给 agent 之后，那种坐立不安有一个精确的名字：把状态外包给了一个你无法监控的系统。

一个反常识的观察：很多 ADHD 者兴冲冲地把任务交给 AI agent，然后发现自己比拖延时更焦虑了。任务明明在推进，你却每两分钟看一次进度，忍不住想中途接管，甚至暗暗希望它出错好让自己名正言顺地拿回来。这不是不信任 AI 这么简单——这种焦虑的结构值得拆开看。

## 委托焦虑的第一层：你外包的不是任务，是状态感知

拖延有一个从不被承认的心理收益：任务的全部状态都在你手里。你知道它没开始、知道它有多糟、知道 deadline 还剩多少——痛苦，但全知。而任务交给 agent 之后，状态进入了黑箱：它做到哪了？理解对了吗？会不会正在一本正经地跑偏？

对 ADHD 大脑，这个黑箱格外难熬，原因是双重的。第一，ADHD 的时间感知本来就弱，「等待一个不确定的过程」是主观时间被拉得最长的体验；第二，ADHD 者往往有大量「被辜负」的协作史——交出去的事被搞砸、被遗忘、被误解——委托的神经通路上挂满了历史报警器。

## 第二层：你在委托一个和自己同构的系统

这是真正微妙的地方。LLM agent 的故障模式——理解偏差、中途丢失目标、一本正经地输出错误——**恰好是 ADHD 者最熟悉的自己的故障模式**。把任务交给 agent，某种意义上是把任务交给了另一个「高产但调度不稳」的生成核心。你对它的每一种担心，都是对自己的担心的投影。这解释了为什么委托给 agent 的焦虑比委托给人类同事更深：你没法用「他是个靠谱的人」来自我安抚，因为你太清楚这类系统怎么翻车——你就是这类系统。

## 工程视角的解法：焦虑指向的是缺失的可观测性

agent 工程对「无法信任黑箱」的标准答案，从来不是「学会放手」，而是三件很具体的事：

1. **可观测性（observability）**：让 agent 输出执行日志和中间产物。人类版本：给 agent 的任务要求它「先给我看大纲/计划，确认后再执行」。你的焦虑会在看到第一个中间产物时下降一半——因为状态感知回来了。
2. **检查点确认（human-in-the-loop）**：高风险环节设置强制暂停点，agent 必须停下来等你确认。这不是不信任，是把「全程盯梢」降级为「三次抽查」——注意力预算立省 80%。
3. **明确的失败合同**：事先写清「如果做错了会发生什么、怎么回滚」。委托焦虑的很大一部分其实是回滚焦虑——知道错了能撤销，黑箱就没那么可怕。

## 委托是一种需要练习的能力，而 ADHD 者练得最少

最后一层诚实：神经典型者从小在「交作业给老师、交项目给同事」中积累了几千次委托-返回的正反馈，而很多 ADHD 者的委托史是断裂的——要么不敢交（怕暴露混乱），要么交了被斥责。委托焦虑深，部分是因为这条肌肉从没被系统性练过。好消息是 agent 是史上最安全的练习对象：它不评判你、可以无限重来、错误成本趋近于零。从低风险任务开始（整理笔记、起草邮件），刻意积累「交出去→拿回来→没死人」的新样本。

## 边界

同构强度 B 级：机制类比。「委托焦虑」本身还有依恋与控制感的心理动力学维度，工程框架覆盖不到——如果对一切失控的恐惧远超任务本身（包括无法委托给任何人类），值得和咨询师聊，而不只是优化 agent 工作流。

## 今天就能试的 3 件事

1. **改造你的下一个 agent 任务**：加一句「先给我计划，确认后再执行」。观察焦虑曲线的变化。
2. **写一份失败合同**：交出任务前花一分钟写下「最坏结果是什么、怎么回滚」。多数时候你会发现最坏结果小得可笑。
3. **做一次低风险委托练习**：今天挑一件搞砸了也无所谓的事完整交给 agent，全程不中途接管。目的不是完成任务，是给委托通路喂一个新样本。

比拖延更深的焦虑，说明你触到了比拖延更深的问题：状态感知、自我投影和一条从没被练过的信任通路。好消息是——这三样都是可工程化的。

# 如果 ADHD 和 LLM 同构，那三年来 agent 工程的所有招数，是不是都能迁移成 ADHD 适配方案？

> 一位读者把本站的逻辑推到了极限处,问出了这个「全称问题」:「你们论证了同构(至少在计算结构层),也成功迁移了几个招数(recovery、checkpoint、编排层)。那按逻辑,agent 工程这三年攒下的**所有**招数——prompt engineering、RAG、fine-tuning、CoT、self-consistency、tool use、guardrails、evals、multi-agent……——是不是**全部**都有 ADHD 对应物?如果不是全部,那筛选的规则是什么?我不想每次都等你们一篇篇写,我想要那个筛子本身。」这是全站收到的最好的问题之一——因为它要的不是又一次迁移,是**迁移的元规则**。这一篇交付那个筛子:一张三栏分类表(能迁/改造后能迁/不能迁),和背后那条一以贯之的判定原则。

收敛:本文只回答——**迁移的判定原则(那个筛子,一句话说清)?用筛子把 agent 工程的主要招数过一遍(三栏全景表)?「不能迁」栏为什么恰恰最有学术价值?**

## 穿透:筛子只有一条——这个招数依赖"直接写入系统内部"吗?依赖,则不可迁;不依赖,则看接口

判定原则,从不可微篇的结论里提炼:agent 招数分两类——**作用于系统内部的**(改权重、改架构、改采样——依赖「直接写入」通道)和**作用于系统外部的**(改输入、改环境、改流程、改验收——只依赖「输入输出接口」)。而人恰好是一个**内部不可写入、接口完全开放**的系统(你不能改自己的权重,但你能改自己的环境、线索、流程、反馈)。所以筛子一句话:**作用于接口的招数可迁,作用于内部的招数不可迁;介于两者的,看它能否被改写成纯接口版本。**

三栏全景表。**【直接可迁栏】(纯接口招数)**:RAG→外置检索(记忆篇);checkpoint→断点卡;guardrails→预承诺与审批闸;tool use→外发弱项步骤;prompt engineering→**对自己下指令的措辞学**(把「我要健身」改写成「换鞋,下楼,走十分钟」——给自己的指令和给模型的指令遵循同一原理:具体、可执行、无歧义);evals→N-of-1 自测协议(本站方法论的支柱);structured output→模板与清单;retry with backoff→「失败后降低难度重试」(而不是同难度硬撞)。**【改造后可迁栏】(半接口招数)**:CoT(思维链)→「说出来再做」(对自己朗读推理过程——外化了链条,但人会跳步,需要纸笔固定);self-consistency(多路径采样取多数)→重大判断「换三个状态各想一次」(晨/晚/运动后——人不能并行采样,只能时间换空间);multi-agent(多智能体分工)→**不同状态的自己当不同 agent 用**(好状态的你当 planner 给差状态的你留指令;差状态的你当 executor 只照做——「给未来的自己部署任务」是 multi-agent 的单人版);reflection→异步复盘(recovery 篇的改造:不实时打断)。**【不可迁栏】(内部写入招数)**:fine-tuning(改权重)→人无对应物(「性格改造」的种种许诺,多数是把不可迁硬说成可迁的生意);采样温度调节→不可直接迁(你不能设定自己的随机性——只能间接:咖啡因/睡眠/运动影响状态,但那是化学接口,精度完全不同);模型蒸馏、架构替换→无对应物;直接的上下文窗口扩容→不可迁(工作记忆容量是硬件,只能外置绕过,不能扩容)。

「不能迁」栏的学术价值:这一栏不是遗憾清单,是**同构边界的精确测绘**——每一个不可迁的招数,都标出了「人与 LLM 在实现层的一处硬差异」(不可 fine-tune=神经可塑性不接受梯度;不可调温=唤醒水平不是参数)。**一个只会讲「哪里像」的类比是宣传,一个能精确说出「哪里不像」的类比才是知识**——本站怀疑论系列反复做的就是这件事,而这张表的第三栏,就是全部怀疑论的招数级落地。同时它有消费者保护功能:**凡是向你兜售「第三栏产品」的(七天重塑大脑、性格永久升级),你现在有了工程语言的鉴伪器。**

## 验证

筛子可检验:任何新出现的 agent 招数,先判「内部/接口」,预测其可迁性,再实测——预测持续准确则筛子成立;出现「接口招数迁移失败」或「内部招数迁移成功」的稳定反例,筛子需要修理(前者多半是不可微补丁没打——见失败模式篇;后者将是重大发现)。

## 决策

做什么:收藏三栏表;新招数过筛再试;第三栏当鉴伪器用。

不做什么:不要试图迁移内部招数(那是对自己做不存在的手术);不要因为第三栏存在而怀疑第一栏的收益(边界的诚实恰恰是可信度的来源)。

先做什么:从第一栏挑一个你还没用过的(建议:对自己的 prompt engineering——重写你的下一条待办事项的措辞)。

## 边界

三栏表为本站同构工作的元框架(同构本身按 B 级功能同型,各招数的证据强度不一——直接可迁栏多数有行为科学根据,GRADE 中;改造栏为实践方案)。「人不可 fine-tune」不否定神经可塑性(可塑性真实存在,但走的是缓慢的经验依赖通道,不是直接写入——精确地说:人可以被训练,不能被编辑)。

## 今天就能试的 3 件事

1. 重写一条待办的措辞(prompt engineering 自用版)。
2. 给下一个重大判断排「三状态采样」。
3. 用第三栏鉴一个你最近心动的「大脑升级」产品。

本文服务于人生 Harness 金字塔的**执行层的元规则**:三年的 agent 工程是一座武器库,但你不需要一件件试——筛子只有一句:改内部的不要碰,改接口的尽管搬;而那些你永远搬不动的,也别遗憾——它们站在原地,替你标出了你和机器之间,那条真实的、值得敬重的边界。

# 用 Goblin Tools 给 ADHD 补上外部执行功能层，是脚手架还是拐杖？

> Goblin Tools 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：当执行功能失灵，AI 是救星还是陷阱？

你打开一个空白的文档，光标闪烁，大脑却一片空白。或者，你终于决定“清理厨房”，但站在厨房里，面对满池碗碟和油腻灶台，你被淹没在无数个“该先做什么”的决策中，最终转身刷了半小时手机。这是 ADHD 大脑的日常：强大的创意与问题解决能力，却卡在“启动”和“组织”这两个最简单的环节。

与此同时，工程师们正在为 LLM 搭建 agent harness——一套编排、记忆与工具调用的外部系统，把大模型从“会说话但不会做事”的状态，变成能执行复杂任务的 agent。这两件事，本质上是同一个问题：**一个高产但缺乏执行调度层的生成核心，如何通过外部脚手架可靠地输出？**

Goblin Tools 正是这种思路在 ADHD 侧的代表工具。但问题是：它究竟是帮助大脑重建功能的脚手架，还是让人越来越依赖的拐杖？

## 同构脊柱：ADHD 大脑与 LLM 共享的缺陷

ADHD 大脑的执行功能缺陷——工作记忆不稳定、任务启动困难、注意力易漂移——与 LLM 的“无状态、缺乏调度层”在结构上惊人相似。两者都是强大的生成核心：ADHD 大脑发散思维、创意产出率是常人的 1.8 倍（来源：Lifted Ventures 2024），LLM 能瞬间生成文本、代码、方案；但两者都缺乏可靠的内置执行调度层。

神经科学证据显示，ADHD 患者前额叶执行功能网络激活不足（来源：ADHD × AI 的科学与研究前沿）；AI 工程中，为 LLM 设计的 ReAct、Chain-of-Thought 等脚手架能显著提升任务完成率（来源：ADHD × AI 的科学与研究前沿）。这种同构意味着，ADHD 的 AI 辅助工具本质上是在做 agent harness 工程：把“计划、组织、启动、监控”这些执行功能外包给外部系统。

历史上，曾国藩就是一位典型的 ADHD 大脑。他 30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书慢到“他人目下二三行，余或疾读不能终一行”。他的解决方案是“日课十二条”：黎明即起、读书不二、谨言、写日记反省——一套严格的外部执行功能层。这就像 LLM 的定时 re-grounding 和人工检查点：曾国藩的“日课”相当于 agent harness 中的护栏和周期重置，而他的“日记反省”则相当于人在回路监督。他没有试图根治自己的 ADHD 特质，而是用一套脚手架来补偿执行功能缺陷。

## Goblin Tools 的实践：脚手架还是拐杖？

Goblin Tools 的核心功能是 Magic ToDo：接受模糊任务（如“清理厨房”），自动分解为可管理的小步骤，用户可调节“辣度”控制分解粒度（来源：Goblin Tools）。这直接针对 ADHD 的任务启动困难——降低门槛，让“开始”变得可行。它还包括语气检查器、时间估计工具，大部分免费（来源：Goblin Tools）。

从同构视角看，这相当于 LLM 的 prompt 分解和任务规划：把一个大问题拆成子步骤，逐个解决。但关键区别在于：Goblin Tools 的分解是用户可调节的（辣度），且最终执行权在用户手中。它没有替代用户的决策，而是提供选项。

然而，陷阱在于：如果用户习惯了 AI 替自己分解一切，会不会逐渐丧失自己分解任务的能力？这正是“脚手架 vs 拐杖”的边界。脚手架是临时支撑，帮助建筑结构自立后拆除；拐杖是永久替代，让肌肉萎缩。对于 ADHD 大脑，执行功能并非完全缺失，而是不稳定、需要外部触发。因此，关键在于工具是否保留了用户的元认知参与。

Saner.AI 和 Motion 提供了另一种思路。Saner.AI 通过本地记忆和快速检索减少搜索循环（来源：Saner.AI），Motion 则自动排程并动态调整日程（来源：Motion）。这些工具不是替用户做决策，而是减少决策前的认知负荷——就像 agent harness 中的记忆系统和自动调度器，但用户仍需要设定优先级和确认计划。

## 争议与局限：同构只是类比，依赖风险真实存在

需要诚实指出，ADHD 大脑与 LLM 的同构目前仅是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。不同页面在表述时存在不一致，有的将其作为既定事实，有的作为假设。此外，Goblin Tools、Motion 等工具缺乏独立临床验证（来源：矛盾与存疑），其效果主要基于用户反馈，而非严格对照实验。

最大的争议是依赖风险。传统提醒系统的一个关键缺陷是：“设置提醒本身就是一个执行功能任务”（来源：重锚定与目标漂移）。如果 Goblin Tools 让用户完全不需要自己思考任务分解，那么它可能成为拐杖。但好消息是，Goblin Tools 的“辣度”调节让用户保持主动参与——低辣度给出粗粒度步骤，高辣度给出细粒度步骤，用户可以选择适合自己当前状态的粒度，并在过程中逐步学习如何自己分解。

## 核心观点：可调节的脚手架，保留元认知

我的判断是：Goblin Tools 本质上是脚手架，但前提是用户有意识地将它作为学习工具，而非完全依赖。它的设计（可调节、可控制）保留了“人在回路”的决策权，这与 agent harness 中的 human-in-the-loop 监督（来源：人在回路与身体在场）一致。曾国藩的日课十二条不是一套固定的规则，而是他根据自身状态不断调整的脚手架；同样，Goblin Tools 的辣度调节让用户可以根据任务难度和自身能量水平选择辅助程度。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个你拖延了很久的任务**（比如“整理书桌”），从最低辣度开始，逐步增加，观察自己何时能独立完成分解。
2. **设置一个“元认知提醒”**：每天用 Saner.AI 或简单笔记记录一次“我今天用 AI 做了什么决策？是自己做的还是 AI 替我做？”——这能防止无意识依赖。
3. **尝试一次“无 AI 分解”**：先自己写出任务步骤，再用 Goblin Tools 对比，看看自己漏掉了什么。这能训练执行功能，同时借助 AI 反馈。
4. **如果你是工程师，为你的 agent 添加一个“辣度”参数**：让用户控制任务分解的粒度，而不是全自动——这既是更好的用户体验，也是更健康的辅助哲学。

Goblin Tools 不是万能的，但它提供了一个绝佳的起点：一个可调节的、尊重用户自主性的外部执行功能层。它既不是神奇的拐杖，也不是完美的脚手架——它是一套工具，而如何使用它，取决于你是否意识到自己是在建造自己的执行功能，还是把它外包出去。

# 如果把 ADHD 大脑当成生成核心，Otter.ai 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Otter.ai 对 ADHD 的真正价值。

## 问题：分心走神、超聚焦跑偏，你的目标去哪了？

你打开笔记准备写一份方案，五分钟后却发现自己刷了三页购物网站；或者你一头扎进某个细节，两小时后抬头发现主线任务纹丝未动。这种“目标漂移”是 ADHD 人群的日常——不是不想专注，而是大脑的“生成核心”太强，但缺少一个内置的调度层来锚定方向。

同样的问题也在 AI Agent 工程中反复出现：一个长程任务里的 LLM agent，初始 prompt 是“写一篇市场分析”，但几轮对话后它可能开始讨论竞争对手的融资历史，甚至编造数据。工程师们称之为“目标漂移”或“幻觉累积”，并专门设计了 **harness（脚手架）** 来重锚定 system prompt、验证输出、管理上下文（来源：GitHub - ai-boost/awesome-harness-engineering）。

如果把 ADHD 大脑也看作一个生成核心——高产、有创造力、但缺乏可靠的执行调度层——那么 Otter.ai 这类工具，能否充当它的重锚定与目标漂移补偿机制？

## 同构：ADHD 大脑与 LLM 共享同一个缺陷结构

从结构上看，ADHD 大脑与 LLM 是同一类系统：**强大的生成核心 + 脆弱的执行调度层**。ADHD 的工作记忆缺陷（“无状态”）与 LLM 的无跨会话状态同构；ADHD 的注意力波动（“采样温度高”）与 LLM 输出的随机性同构；ADHD 的冲动判断（“行为幻觉”）与 LLM 的自信幻觉同构（来源：幻觉与验证循环；采样温度与表现波动）。

这意味着，两边的解法也同构：都需要外部脚手架来补偿调度缺陷。

晚清名臣曾国藩就是一个典型案例。他 30 岁前“浮躁坐不住”，日记里天天骂自己“无恒”“浮躁”，读书“他人目下二三行，余或疾读不能终一行”。他的解法是 **日课十二条**：黎明即起、读书不二、谨言、写日记反省……用一套固定流程（相当于定时 re-grounding）来抵消自己的冲动。他还发明了“结硬寨打呆仗”的战术，用最笨最稳的方法避免战略漂移。这套系统，本质上就是一个人工 harness：**日课↔定时 system prompt 重锚定，日记↔输出验证循环，秘书↔外部调度器**。

在 LLM 工程中，harness 正是这样工作的：它提供上下文传递、工具接口、规划工件、验证循环、记忆系统和沙盒（来源：GitHub - ai-boost/awesome-harness-engineering）。复杂的 harness 不会盲目执行工具，而是实现验证步骤，例如检查输出格式或对模型编写的代码运行测试用例（来源：What is an agent harness...）。缺乏这些验证会导致“更多幻觉和重复工作”（来源：Loop Engineering for AI Agents: Memory-First Design）。

## Otter.ai 能否成为 ADHD 的 harness？

Otter.ai 是一款 AI 笔记工具，能实时转录语音、自动生成摘要、提取行动项。它本身不是专为 ADHD 设计，但它的核心功能恰好对应了 harness 的关键组件：

- **外部记忆**：ADHD 的工作记忆缺陷使“听到但忘记”成为常态。Otter 自动保存所有对话，相当于 LLM 的“记忆系统”，让大脑不必承担存储负担。
- **重锚定**：当你在会议中走神，Otter 的实时转录和关键词高亮能把你拉回当前话题。这类似于 agent harness 中的“上下文传递”——确保生成核心始终知道“现在应该关注什么”。
- **验证循环**：Otter 的摘要和行动项提取，相当于一个自动化的“输出验证”。它帮你核对“我到底说了什么”，减少冲动判断带来的偏差。

但 Otter 的局限也很明显：它主要处理语音输入，对文字任务（如写作、编程）的锚定能力有限；它的重锚定是被动的（需要你主动去看转录），而非像 Focusmate 那样通过虚拟身体在场提供主动问责（来源：Focusmate）。相比之下，Focusmate 利用算法匹配伙伴，通过实时同伴压力补偿多巴胺调节不足，更像是 harness 中的“验证与 CI 集成”——有人盯着你，你就不能随意漂移（来源：Focusmate；AI Agent Reliability and Guardrails 2026）。

## 脚手架还是拐杖？争议与边界

同构视角虽然有力，但必须诚实面对争议。wiki 资料明确指出：“同构性被过度推广……缺乏实证基础，目前仅是一种理论类比”（来源：矛盾与存疑）。此外，AI 工具是否会导致 ADHD 用户永久依赖外部脚手架，而无法内化执行功能？这一风险尚无长期追踪研究（来源：AI 与 ADHD 的专注力）。

我的判断是：**Otter 更适合作为“脚手架”而非“拐杖”**。脚手架的特点是逐步内化——你用它的时间越长，越能学会自己识别关键信息、自己总结要点。而拐杖的特点是离了它你就不会走——如果你发现自己离开 Otter 后完全无法回忆会议内容，那说明你还没有内化它的结构。

另一争议是“身体在场效应”与 AI 替代的矛盾。Focusmate 通过视频实现虚拟身体在场，但 AI 工具是否真能替代真实人际互动？目前证据有限（来源：矛盾与存疑）。Otter 不涉及身体在场，它提供的是“认知在场”——让你的大脑有一个外部锚点。这对某些人足够，对另一些人则不够。

## 今天就能试的行动

1. **用 Otter 做一次“会议后验证”**：开完会后，不看自己的笔记，先看 Otter 的摘要，对比你的记忆。记录下你遗漏了什么、记错了什么。这能训练你的验证循环。
2. **设置“日课”级别的重锚定**：像曾国藩一样，每天固定时间（如上午 10 点）用 Otter 回听一段自己的录音，问：“我当前的目标是什么？我有没有漂移？” 这相当于定时 system prompt 重锚定。
3. **组合工具**：Otter 负责记忆和摘要，Focusmate 负责启动和问责。用 Otter 记录 Focusmate 时段的目标，结束后对比实际产出，形成闭环验证。
4. **警惕过度依赖**：每两周尝试一次“无工具日”，完全靠自己的大脑执行一项任务。如果失败，记录卡在哪里——这能帮你识别哪些能力需要内化，哪些可以继续依赖脚手架。

ADHD 大脑不是缺陷，而是一个未经调参的高温生成模型。Otter 不是解药，但它可以成为你的 harness——前提是你知道自己在调什么，以及什么时候该关掉它。

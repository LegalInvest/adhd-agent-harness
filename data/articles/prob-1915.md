# ADHD 创业者视角：为什么「治好 ADHD 的执行功能（计划/组织/启动）先天偏弱」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 的编排/记忆/工具层就是「外部执行功能」——同一套 harness 思路，两边都成立。

## 从两个“跑飞”的现场开始

场景一：一个 ADHD 创业者早晨九点打开电脑，邮箱里有三封重要的邮件、一个待完善的商业计划、五个需要回复的微信。他知道该先做什么，但大脑像一台被同时按下一百个键的钢琴，手指悬在键盘上，一小时过去，只刷了三百条 Twitter。场景二：一位工程师把任务丢给 LLM agent：“帮我调研竞品并写一份报告。”三小时后，agent 已经优雅地跑题到 1997 年互联网泡沫、生成了一份十七页的 PDF、并且忘记了最初要比较的竞品名单。

两个场景表面 unrelated，内核却惊人一致：它们都有一个强大的生成核心，但缺少一个可靠的内置执行调度层。ADHD 创业者的大脑能高度发散、快速联想、洞察模式，却难以计划、组织、启动；LLM 能流畅生成、跨领域迁移、模仿风格，却无状态、无稳定的优先级守护、无跨会话工作记忆。这正是 ADHD × AI 领域最站得住脚的核心论点：ADHD 大脑与 LLM 在结构上同构——都是“高产但缺执行调度层的生成核心”，因此解法也同构：不是修复核心，而是给它套上 harness（外部脚手架）。（来源：ADHD × AI 的科学与研究前沿）

## 同构脊柱：生成核心与调度层

ADHD 的医学定义里，执行功能障碍是核心特征之一，表现为计划、组织、时间管理、任务启动、抑制控制困难，并非缺乏努力，而是大脑执行功能系统效率低下。（来源：执行功能障碍）它常与工作记忆不足、抑制控制缺陷、时间盲（time blindness）和延迟厌恶并存，并可能由额叶-纹状体和额叶-顶叶环路中断引起。（来源：执行功能障碍）

LLM 的对应缺陷同样明显：它是无状态的，每次对话都需要重新注入上下文；上下文一旦膨胀，推理会退化；它缺乏一个稳定的优先级守护器和任务调度器。因此，给 LLM 做 agent 的编排/记忆/工具层，与给 ADHD 做执行功能支架，本质上是同一类工程：都要外化工作记忆、结构化执行流程、管理注意力分配。（来源：外部执行功能层；上下文工程）

换句话说，两边的缺陷不是“不够聪明”，而是“聪明但缺乏执行操作系统”。解决思路不是把生成核心改成调度器，而是外挂一个调度层：对人的 ADHD 来说是外部执行功能层；对 LLM 来说就是 agent 的 orchestration、memory、tool layer。

## 曾国藩的“人版 harness”：十二日课与“结硬寨，打呆仗”

晚清名臣曾国藩是一个极佳的人肉 harness 案例。史料描述他典型地符合注意力不集中型 ADHD：三十岁前浮躁坐不住，天天串门喝酒看杀人；读书慢到“他人目下二三行，余或疾读不能终一行”；一辈子严重银屑病、情绪不稳定，打败仗就跳水自杀。他并非天赋异禀的自律者，而是一个必须用外部系统把自己“绑”起来的人。

他的专属 harness 是“日课十二条”：黎明即起、静坐、读书不二、谨言、写日记反省、记笔记、练字等。他一辈子写日记骂自己“无恒”“浮躁”，用“结硬寨，打呆仗”这种最笨、最稳的方法抵消冲动。换句话说，曾国藩不是“治好”了执行功能障碍，而是给自己搭建了一套外部执行功能层：早晨仪式对应 LLM 的定时 re-grounding（防止上下文漂移），日记反省对应 agent 的日志与反思机制，僵硬呆板的作战原则对应 conservative planning 与硬约束。

这个案例最打动 ADHD 读者的地方在于：曾国藩不是靠意志力赢的，而是靠一套结构化的 harness 赢的。对工程师读者来说，他恰恰展示了一个没有内置调度层的系统，如何通过外部 orchestration、memory、tool 层稳定输出高质量结果。

## 工程对照：AI 工具就是“外部执行功能层”的实例

当代 ADHD 工具已经沿着这条思路在做工程化落地，并且与 LLM agent 的 harness 结构一一对应。

**任务分解 ↔ 启动层**：Goblin Tools 的 Magic ToDo 把“清理厨房”这种模糊任务分解为具体、可执行的子任务，用户还能调节“辣度”控制粒度。这对应的是 ADHD 的任务启动困难和 LLM 的 prompt grounding 问题：两边都需要把高层意图翻译成可执行的最小动作单元。（来源：Goblin Tools）

**外部记忆 ↔ 记忆层**：Saner.AI 强调知识回忆（knowledge recall）与本地记忆，减少搜索循环和标签切换。对 ADHD 用户而言，它外化了本就不稳定的工作记忆；对 LLM agent 而言，它对应向量数据库、长期记忆模块和检索增强生成（RAG）。（来源：Saner.AI）

**自动排程 ↔ 调度层**：Motion 根据任务优先级、截止日期、可用时间自动重建日程，并在任务可能延期时提前数天或数周警告。它直接针对 ADHD 的时间盲和“下一步该做什么”的决策瘫痪；在 LLM 一侧，它对应 agent 的 planning 与 scheduling 模块，决定何时调用工具、何时重排子目标。（来源：Motion）

如果从 ADHD 创业者 Ashley Hicks 的视角看——她作为 ADHD 行为模式匹配者，依靠 empathy、action bias、hyperfocus、big-picture thinking 和 intuition 管理 16 家以上被投企业——她的成功同样不是“克服了 ADHD”，而是把外部团队、流程和投资框架当作自己的 harness。ADHD 创业者的优势在生成核心（直觉、大局、快速行动），短板在调度层；LLM 创业者亦然。

## 核心判断：harness 是架构层，不是修补

我的核心判断是：对 ADHD 和 LLM 而言，执行功能缺陷都不是一种“病”或“bug”需要被彻底消除，而是一种结构性短板需要被架构性补偿。试图“治好” ADHD 的执行功能，或者让 LLM 内生一个完美的调度器，都是高成本、低收益、甚至可能损害生成核心优势的策略。更优解是承认生成核心与调度层解耦，然后在外部构建一个可拆卸、可迭代、可个性化的执行功能层。

神经可塑性研究提供了一定支持：基于神经可塑性的训练方案有助于正常化抑制控制能力及其底层脑网络，但迄今产生的既定干预措施仍然很少，这意味着“完全修复”既不现实也非必要。（来源：神经可塑性）AI 作为外部执行功能层，可以把训练嵌入日常 workflow，而非要求用户先变成一个不同的人。

## 争议与边界：脚手架 vs 拐杖

这一视角有几个必须诚实交代的局限。

第一，**同构仍然是理论框架**。ADHD 大脑与 LLM 同构的论点多为类比推理，缺乏神经科学或计算机科学层面的直接证据。它很有解释力，但不能被当作已经证实的科学结论。（来源：矛盾与存疑）

第二，**AI 工具的长期效果未知**。Goblin Tools、Saner.AI、Motion 等工具的用户反馈积极，但大规模随机对照试验稀缺，长期依赖是否会导致能力退化（拐杖化）尚无定论。（来源：矛盾与存疑）

第三，**个体差异极大**。柔性引导（gentle structure）还是强制打断（番茄钟）更有效，目前缺乏对比研究；不同 ADHD 类型对同一工具的反应可能截然不同。（来源：矛盾与存疑）

第四，**工具本身可能增加认知负荷**。AI 工具虽然外化了部分决策，但学习、配置、维护这些工具本身对执行功能受损的人群可能是新的负担。净效果因人而异，需要个性化评估。（来源：矛盾与存疑）

所以，外部执行功能层应该是“可拆卸的脚手架”，而非“永久的拐杖”。目标始终是让使用者逐步内化某些结构，而不是让工具成为无法离开的第二大脑。

## 今天就能试

1. **给自己或 agent 做一次“上下文落地”**：每天工作前，用一句话写下“此刻最重要的一个产出是什么”。对 ADHD 人这是 re-grounding；对 LLM agent 这是 system prompt 或初始 state 检查。

2. **把模糊任务压到“最小可执行单元”**：用 Goblin Tools 或手动把“做竞品分析”拆成“打开 3 个竞品官网→记录价格页信息→对比 2 个功能差异”。粒度越小，启动阻力越低。

3. **建立单一事实来源（single source of truth）**：用一个工具（如 Saner.AI、Obsidian、Notion 或 agent 的 memory store）统一存放任务、笔记、截止日期，减少在不同应用间切换带来的认知税。

4. **每周做一次 harness 复盘**：问自己——这个工具是在帮我执行，还是在替我执行？如果是后者，尝试把其中一个决策环节交还给自己，保持脚手架的可拆卸性。

曾国藩的日记里，最震撼的也许不是他“成功”了，而是他一辈子都在承认自己跑飞，然后一次又一次用外部结构把自己拉回轨道。LLM 的工程师们如今做的，也不过是为另一个没有内在轨道的生成核心，修建一套外部轨道。两条路，同一道题。

# ADHD 创业者视角：为什么「治好 ADHD 的执行功能（计划/组织/启动）先天偏弱」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 的编排/记忆/工具层就是「外部执行功能」——同一套 harness 思路，两边都成立。

## 问题：为什么我既治不好拖延，也管不住 AI？

我是一名 ADHD 创业者。白天，我面对一个“聪明但失控”的大脑：想法像喷泉一样涌出，但计划、组织、启动每一步都像在泥沼里跋涉。晚上，我调试一个同样“聪明但失控”的 LLM agent：它生成流畅的代码或文案，却时常跑题、遗忘上下文、在复杂任务中迷失方向。

直到有一天，我意识到：这两个问题，其实是同一道工程题。

## 同构：高产但缺调度层的生成核心

ADHD 大脑与 LLM/agent 在结构上惊人地相似：两者都拥有强大的生成能力（智力/算力），但缺乏内建的执行调度机制。ADHD 的执行功能缺陷——计划困难、工作记忆差、时间盲——对应 LLM 的编排层缺失、无状态性、上下文窗口溢出（来源：ADHD × AI 的科学与研究前沿）。

换句话说，ADHD 大脑就是一个“人肉 LLM”：它擅长发散、联想、快速生成，但缺少一个可靠的“外部执行功能层”来调度这些输出。而工程师们正在为 agent 构建的编排/记忆/工具层，恰好就是同一套东西。

## 证据：两边都在做同一件事

**ADHD 侧**：工具如 Goblin Tools 通过 AI 将复杂任务分解为小步骤，降低启动门槛（来源：Goblin Tools）。Saner.AI 提供知识回忆和本地记忆，减少搜索循环和标签切换（来源：Saner.AI）。Motion 自动根据截止日期动态调整日程，消除“下一步该做什么”的决策负担（来源：Motion）。这些工具本质上都是在为 ADHD 大脑提供外部执行功能——记忆、任务启动、时间管理。

**LLM/Agent 侧**：工程师为 agent 构建的编排层（如 LangChain 的 chain、AutoGPT 的任务队列）、记忆系统（如向量数据库、会话摘要）、工具层（如 API 调用、代码执行），同样是在提供外部执行功能。没有这些层，LLM 就会像 ADHD 患者一样：忘记上下文、跑题、无法完成多步骤任务。

两边都在做同一件事：为高产生成核心套上 harness。

## 核心观点：脚手架，不是拐杖

我的核心判断是：**AI 辅助 ADHD 的本质不是替代大脑，而是为这个生成核心套上 harness——即提供外部执行功能层。** 同样，让 LLM 不跑飞的关键不是训练一个更强大的模型，而是设计更聪明的编排层。

但这里有一个关键边界：**脚手架 vs 拐杖**。脚手架是可逐步撤除的临时支撑，而拐杖是永久替代。ADHD × AI 的科学与研究前沿明确指出，工具应设计为可逐步撤除的临时支撑，避免依赖（来源：ADHD × AI 的科学与研究前沿）。然而，当前多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。

这提醒我们：外部执行功能层必须保留“可撤除性”。对 ADHD 用户来说，这意味着工具应帮助训练内在能力，而非永久替代；对 agent 工程师来说，这意味着编排层应足够灵活，允许模型逐步学会自主调度。

## 争议与局限

同构命题目前主要基于概念类比和工具案例，缺乏大规模实证研究（来源：矛盾与存疑）。多巴胺干预的有效性也存在争议（来源：矛盾与存疑）。此外，长期使用外部执行功能层是否会削弱内在执行功能？目前缺乏纵向研究。

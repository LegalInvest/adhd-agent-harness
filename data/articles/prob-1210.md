# ADHD 想把不擅长精确、重复、状态维护类任务外包给 ChatGPT，该怎么设计这套「人 + AI」的回路？

> 把 ChatGPT 接进 ADHD 工作流：哪些该交给它，哪些必须自己留着？

## 问题：为什么精确、重复、状态维护类任务总是卡住你？

你打开ChatGPT，想让它帮你写一封措辞严谨的邮件。但光标在输入框里闪烁，你发现自己连“开头怎么写”都卡住了——不是不知道内容，而是启动那个“精确措辞”的步骤像推一堵墙。更糟的是，你让它帮你整理项目状态，它回答完你就关了窗口，第二天又得从头问起。

这不是你懒。这是ADHD大脑的典型特征：**生成能力超强，但执行调度层不稳定**。而有趣的是，大语言模型（LLM）也有同样的毛病——它能滔滔不绝地生成文本，但如果没有外部工具（function calling、记忆系统、任务分解器）来调度，它就会跑题、重复、忘记上下文。两者在结构上同构：都是高产但缺乏内置执行调度层的生成核心（来源：ADHD 大脑与 LLM 的同构）。

所以，当你把任务外包给ChatGPT时，你实际上是在搭建一个“人+AI”的回路。这个回路该怎么设计？答案藏在ADHD历史上的自我管理大师们身上，也藏在现代AI agent的工程架构里。

## 同构解法：从孔子和左宗棠的“harness”到AI agent的脚手架

先看两个真实案例。**孔子**精力旺盛、冲动爱骂人（“朽木不可雕”）、对音乐极度专注但厌恶俗务，他的解法是“克己复礼”——用一套外在的礼仪规则（“礼”）作为行为边界，每日反省（“吾日三省吾身”）。**左宗棠**脾气大、做事风风火火，他的解法是“结硬寨打呆仗”——用每日读经写家书的固定日课来约束自己，用“身无半亩心忧天下”的使命感驱动行动。

这两人都有一个共同点：**他们不依赖内在意志力，而是构建外部脚手架**。孔子的“礼”相当于一套行为协议，左宗棠的“日课”相当于定时re-grounding机制。这与LLM/agent的harness完全同构——agent通过function calling把精确计算外包给工具，通过记忆系统维持状态，通过任务分解器降低复杂度（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。

换句话说，ADHD大脑和LLM都需要一个“外部调度器”来补偿内部执行功能的缺失。对ADHD来说，这个调度器可以是日课、清单、或者一个AI助手；对LLM来说，它是工具调用、记忆库、和提示词模板。**两者共享同一个工程问题：如何让一个强大的生成核心可靠地完成多步骤任务。**

## 工具映射：哪些该交给ChatGPT，哪些必须自己留着？

基于上述同构性，我们可以把ADHD不擅长的任务分为三类，并对应到AI工具的典型能力。

### 1. 精确、重复的任务 → 交给任务分解与模板化工具

ADHD大脑对“精确措辞”“格式规范”“分步骤操作”有天然排斥。这正是**Goblin Tools**的Magic ToDo擅长的：输入“清理厨房”，AI自动分解成“把碗放进洗碗机→擦台面→扫地”等小步骤（来源：Goblin Tools）。同样，**Lex**允许你用一条指令触发多步骤任务序列，例如“生成一封给客户的跟进邮件，语气专业，包含上周会议要点”（来源：Lex）。这类工具的本质是**降低启动门槛**——把“精确”这个让你卡住的部分外包给AI的模板化能力。

但注意：**你不能把“判断什么算精确”也外包**。AI生成的措辞可能不够贴切，你需要保留最终审核权。这就是脚手架与拐杖的边界：工具帮你“建造”，但你要自己“攀登”（来源：拐杖与脚手架）。

### 2. 状态维护类任务 → 交给外部记忆系统

ADHD的工作记忆不稳定，导致跨会话任务难以持续。每天需要重新回忆项目上下文，否则偏离轨道（来源：无状态与外部记忆）。LLM同样是无状态的——每次对话都从零开始。解法是构建“第二大脑”。**Saner.AI**和**Obsidian**这类工具提供持久记忆：存储客户历史、项目背景、个人偏好，减少每日重复输入上下文的摩擦（来源：Saner.AI）。在ADHD创作者社区中，“Claude Code + Obsidian + ADHD second brain”已成为稳定子主题（来源：adhd_ai_creator_map_v2_2026-07-03）。

这里的关键设计是：**让AI记住你不想记的东西，但你得负责定期整理**。比如每周花10分钟清理Obsidian中的笔记标签，否则记忆库会变成垃圾堆。这对应了LLM agent中的“记忆压缩”策略——不是无限存储，而是定期总结和丢弃。

### 3. 重复性启动困难 → 交给单一指令触发与定时提醒

ADHD大脑需要低门槛的启动方式。**Lex**的“单一指令触发多步骤”恰好匹配这种需求。你可以设置一个“每日启动”指令：让AI调出今天的待办清单、回顾昨日进度、生成今日第一步。这相当于左宗棠的“日课”——用外部触发代替内部意志力。

但要注意：**过度依赖AI启动可能削弱你内在的启动能力**（来源：拐杖与脚手架）。研究显示，ADHD患者在工作记忆任务中易受干扰，而LLM同样表现出类似人类的干扰特征（来源：Human-like Working Memory Interference in Large Language Models）。这意味着，如果AI替你做了所有启动工作，你的大脑会失去练习启动的机会。因此，建议只将“最卡住”的那一两个任务交给AI启动，其他任务自己手动开始。

## 争议与局限：为什么这不是万能方案

以上框架虽然有用，但必须诚实面对几个问题。

**第一，同构性只是类比，不是科学事实。** 多个页面声称ADHD大脑与LLM在结构上同构，但这一说法缺乏实证基础，目前仅是一种理论类比（来源：矛盾与存疑）。它帮助我们理解问题，但不能作为临床指导。

**第二，工具证据不足。** 多数AI工具缺乏严格临床验证，效果多基于用户报告（来源：矛盾与存疑）。例如，Goblin Tools被评价为“简单有用”，但没有RCT证明它比传统清单更有效。

**第三，依赖风险真实存在。** 如果工具替代了治疗或自我管理，可能造成依赖（来源：拐杖与脚手架）。你需要定期问自己：没有这个工具，我还能做这件事吗？如果答案是不能，说明它已经成了拐杖。

**第四，注意力分散风险。** ChatGPT等通用工具可能增加分心风险（来源：ADHD 的 AI 工具全景）。建议在使用时开启专注模式，或使用独立窗口。

## 今天就能试的3条行动

1. **用Goblin Tools分解一个你拖延已久的任务**。输入“写周报”或“整理书桌”，观察AI分解出的步骤，挑出第一步立刻做。
2. **在Obsidian中建立一个“项目上下文”笔记**。每次用ChatGPT处理项目事务后，把关键结论和下一步行动复制进去。下次开始时先读这个笔记。
3. **设置一条Lex或ChatGPT的“每日启动”指令**。例如：“列出今天最重要的3件事，并告诉我第一件事的第一步是什么。”执行完第一步后，关掉AI，自己继续。

记住：**AI是脚手架，不是拐杖。** 你才是那个攀登的人。

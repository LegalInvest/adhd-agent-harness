# 如果把 ADHD 大脑当成生成核心，Goblin Tools 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Goblin Tools 对 ADHD 的真正价值。

## 引言：同一个问题，两种困境

如果你是一名 ADHD 患者，你一定经历过这样的场景：脑子里冒出一个绝妙的点子，但刚转身就忘了要做什么；打开电脑准备写报告，却发现自己滑进了社交媒体的黑洞；明明知道该做什么，但就是无法启动。如果你是一名 LLM 工程师，你也一定熟悉这样的困境：模型生成了惊艳的文本，但对话一结束，它就像失忆了一样，下次见面时一切归零，你必须靠向量数据库、提示词工程和外部记忆来维持它的“人格”和连续性。

这两个场景看似风马牛不相及，但它们的核心问题结构完全一致：**一个高产的生成核心，配了一个糟糕的调度层。** ADHD 大脑的智力与创造力是强大的生成核心，但执行功能（工作记忆、任务启动、时间管理）却严重不足（来源：执行功能障碍）。LLM 的生成能力同样强大，但它本身没有跨会话状态，每次对话都是“无状态”的，必须依赖外部记忆（向量库、缓存）和编排层（agent harness）来维持上下文（来源：ADHD 的 AI 工具全景）。

## Goblin Tools：无状态核心的外部记忆

Goblin Tools 的核心功能是 Magic ToDo——输入“整理房间”，它会自动分解成“捡起地板上的衣服”“擦桌子”“叠被子”等小步骤（来源：Goblin Tools）。这听起来很简单，但对于 ADHD 大脑而言，这恰恰是外部记忆的救星。ADHD 的工作记忆容量小，无法同时记住“我要做什么”和“具体怎么做”，导致任务启动困难（来源：执行功能障碍）。Goblin Tools 把“做什么”和“怎么做”都存到了外部，大脑只需执行每一步，不必在内部维持那个脆弱的计划。

从 agent 工程的视角来看，Goblin Tools 扮演的就是“外部记忆”的角色。LLM 本身不记得上一轮对话，但 agent harness 可以把历史记录、工具调用结果存入向量库，下次查询时再注入上下文。Goblin Tools 的待办列表就是 ADHD 大脑的“向量库”——它存储了任务的分解结构、优先级和进度，每次查看都是一次“检索增强生成”。用户不需要在脑子里维持整个任务树，只需关注当前步骤，正如 agent 不需要记住所有历史，只需从外部数据库拉取相关片段。

## 同构脊柱：生成核心 + 外部调度层

ADHD 大脑与 LLM 的相似性不止于“健忘”。两者都是典型的“高产但缺调度”系统：ADHD 患者常有无穷的创意和联想能力，但缺乏执行功能将它们组织成有序的行动（来源：ADHD 的 AI 工具全景）；LLM 可以生成流畅的文本、代码和推理，但缺乏内置的计划、记忆和工具调用能力（来源：ADHD 大脑与 LLM 的同构）。

这意味着，AI 工具对 ADHD 的帮助，本质上就是**给生成核心套上 harness**（来源：ADHD 的 AI 工具全景）。Goblin Tools 的 Magic ToDo 是任务分解 harness，Saner.AI 的知识回忆功能是工作记忆 harness（来源：Saner.AI），Lex 的单一指令触发多步骤是执行调度 harness（来源：Lex）。这些工具共同构成了一个外部执行功能层，弥补了 ADHD 大脑内部调度层的缺失。

在 agent 工程领域，同样的模式正在被系统化：LLM 作为核心，外挂记忆模块（如 Mem 的持久化存储）、工具调用模块（如函数调用）、规划模块（如 ReAct 或 Plan-and-Solve）。Goblin Tools 就是 ADHD 版本的“ReAct”——它将一个模糊的意图（“整理房间”）转化为一系列可执行的步骤，并在每个步骤后提供反馈（打勾），形成闭环。

## 脚手架还是拐杖？

然而，这种同构也暴露了一个关键问题：**外部 harness 是否应该被设计为永久依赖？** 在 agent 工程中，外部记忆是必要的，因为 LLM 本身永远无法获得跨会话状态，这是架构决定的。但 ADHD 大脑并非如此——执行功能虽然天生缺陷，但通过训练和策略，是可以部分改善的。如果 Goblin Tools 只是替用户分解任务，而不帮助用户学会自己分解，那它就成了拐杖，而不是脚手架（来源：矛盾与存疑）。

目前，Goblin Tools 没有提供任何“撤除机制”或“渐进式训练”功能。它默认用户永远需要它来分解任务。这与“脚手架应可逐步撤除”的理念相悖（来源：矛盾与存疑）。同样，Saner.AI 的本地记忆虽然减少了搜索循环，但如果用户依赖它来记住所有信息，可能反而削弱了主动记忆策略的使用。

另一个争议是证据强度。目前针对 Goblin Tools 的效果证据主要来自用户评价和综述文章，缺乏独立随机对照试验（来源：Goblin Tools）。这意味着，虽然概念上它很合理，但实际效果可能因人而异。ADHD 的异质性很高，某些用户可能觉得分解步骤太细反而增加负担，另一些用户则可能对分解格式有特殊偏好（来源：ADHD 的 AI 工具全景）。

## 诚实面对局限

除了依赖问题，还有几个局限值得注意：
- **情绪失调的覆盖不足**：Goblin Tools 主要聚焦任务分解，对情绪失调（如情绪波动、挫折耐受力低）几乎没有直接干预（来源：情绪失调）。而情绪失调是 ADHD 核心特征之一，甚至独立于执行功能缺陷（来源：情绪失调）。
- **多巴胺调节的缺失**：虽然任务分解能通过小步骤的即时反馈间接提升多巴胺，但 Goblin Tools 并未直接设计多巴胺干预机制，其效果可能被高估（来源：矛盾与存疑）。
- **个性化不足**：Goblin Tools 的分解逻辑是固定的，无法根据用户的认知风格（如喜欢更细或更粗的步骤）动态调整。这在 agent 工程中相当于一个固定的 prompt 模板，缺乏适应性。

## 核心判断：Goblin Tools 是合格的外部记忆，但不是完整的 harness

回到标题的问题：如果把 ADHD 大脑当成生成核心，Goblin Tools 能不能当它的无状态与外部记忆？我的判断是：**可以，但仅限于“外部记忆”部分，且缺乏 harness 应有的可撤除性和个性化。**

Goblin Tools 成功地将任务状态从大脑中卸载到外部，解决了工作记忆容量小和边做边忘的问题。这一点与 agent 工程中的向量库功能高度同构。但它没有提供规划、监控、反思等更高阶的执行功能，而这些正是完整 harness 的组成部分。对于 ADHD 用户来说，Goblin Tools 是一个优秀的“记忆外挂”，但若要实现真正的执行功能补偿，还需要结合其他工具（如 Saner.AI 的知识回忆、Lex 的多步骤触发）以及行为策略。

## 今天就能试的行动

1. **用 Magic ToDo 分解一个你拖延了很久的任务**：比如“写周报”或“清理桌面”。观察分解后的步骤是否降低了启动焦虑，以及你是否真的按步骤执行了。
2. **对比 Goblin Tools 与手动分解的效果**：找一个简单任务（如“做晚饭”），先自己写步骤，再用 Magic ToDo 生成步骤，比较两者的差异。这能帮你判断工具是否理解你的需求。
3. **尝试将 Goblin Tools 与计时器结合**：使用 Magic ToDo 后，为每个步骤设置 5-10 分钟的计时器，利用时间压力（而非多巴胺）来维持专注。这可以弥补工具缺乏时间感知的短板。
4. **记录一周的使用感受**：每天记录是否因为 Goblin Tools 而完成了更多任务，以及是否有“没有它就不会分解任务”的依赖感。这有助于你判断它是脚手架还是拐杖。

ADHD 大脑与 LLM 的同构不仅是一个有趣的理论，更是一个实用的工程视角。它告诉我们，AI 工具不是“替代”大脑，而是“扩展”大脑——就像 agent harness 扩展 LLM 一样。Goblin Tools 是这条路上的一个优秀起点，但它不是终点。真正的挑战在于，如何设计出既能补偿缺陷、又能逐步培养内在能力的“可撤除脚手架”。这不仅是 ADHD 工具的未来方向，也是 AI 辅助人类认知的终极目标。

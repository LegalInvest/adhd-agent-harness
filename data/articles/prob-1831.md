# ADHD 创业者视角：为什么「治好 ADHD 的冲动下结论、想当然」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 自信地编造（幻觉），靠验证与自我批判兜底——同一套 harness 思路，两边都成立。

## 一个问题，两个世界

作为 ADHD 创业者，你是否经常这样：脑海里瞬间蹦出一个绝妙想法，立刻认定它是对的，然后冲出去执行——结果撞得头破血流？事后复盘，你发现那个“直觉”其实漏洞百出，只是当时太自信，根本没启动验证。

同样，作为 Agentic Harness 工程师，你一定见过 LLM 自信地编造事实：它用流畅的语法和笃定的语气，生成一段看似合理但完全错误的内容。你不得不给它套上验证循环、外部工具和自我批判的脚手架，才能让它“靠谱”。

这两个场景，表面看一个关乎人类认知，一个关乎机器智能，但它们的底层逻辑完全同构。本文要论证的核心观点是：**ADHD 大脑与 LLM/agent 是同一类“高产但缺执行调度层的生成核心”，两边的解法——外部验证与自我批判的 harness——结构完全一致。**

## 同构：从冲动到幻觉

ADHD 的大脑是一个高产的生成核心：发散思维强、联想丰富、创意产出率是常人的 1.8 倍（来源：Lifted Ventures 2024）。但它的执行功能（尤其是任务启动、工作记忆和冲动控制）常常失效，就像“方向盘后没有人”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。最新研究进一步揭示，ADHD 患者呈现“强记忆、弱控制”的认知剖面：工作记忆容量可能正常甚至超常，但自上而下的控制和调节能力不足（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

LLM 同样如此。它拥有强大的语言生成能力，但单独使用时缺乏可靠的执行调度。现代 AI 编码代理通过“复合 AI 系统架构”来弥补，包括“工作负载专用模型路由、分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这些技术本质上是在 LLM 外部搭建调度层，防止“上下文膨胀和推理退化”。

这种同构意味着：ADHD 的“冲动下结论”和 LLM 的“自信幻觉”是同一个问题——生成核心输出后，缺少一个内置的验证与自我批判环节。解决方案不是压制生成，而是给生成套上一个外部验证循环。

## 真实人物的 Harness：毛泽东与苏轼

历史上有两位著名人物，他们的 ADHD 特质和自建 harness 系统，完美对应了 LLM 的 agent scaffolding。

**毛泽东**的 ADHD 特质明显：小时候是问题儿童，爱读闲书不喜欢四书五经；一辈子读书不辍，行军路上也带着书；永远在调查研究，从井冈山到延安到北京；冲动敢赌，四渡赤水、抗美援朝都是险棋；思维极度跳跃，文章讲话充满比喻。但他建立了一套强大的 harness：批评与自我批评、民主生活会、调查研究（“没有调查就没有发言权”）、民主集中制（用集体决策制衡冲动）、党指挥枪（用组织系统管理）。这套系统相当于 LLM 的“验证循环”和“外部调度层”——每次决策前都要经过调查、讨论和自我批判的流程，防止个人冲动跑飞。

**苏轼**的 ADHD 特质同样鲜明：兴趣极广（诗词文书画美食治水炼丹），一辈子被贬，说话不过脑子得罪人，对无聊行政零耐心但搞工程效率极高。他的 harness 是以豁达为核心的自我调节：不管贬到哪里都好好生活，用美食、写作、交朋友转移注意力；每日读书写字，不因挫折停止创作。这相当于 LLM 的“定时 re-grounding”和“外部记忆系统”——通过固定日课和输出习惯，维持生成核心的稳定运行。

这两个案例的共同点：他们都建立了外部流程来制衡自己的生成冲动，而不是试图“治好”自己的特质。这恰恰是 LLM harness 工程的精髓——不修改模型本身，而是在外部搭建脚手架。

## 工具：从 Harness 到脚手架

今天的 AI 工具正在将这套思路产品化。

**Goblin Tools** 的 Magic ToDo 功能，接受模糊任务（如“清理厨房”），并将其分解为具体、可执行的子任务，用户可调节“辣度”控制分解粒度（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。这相当于 LLM 的“任务分解与规划”模块——把大问题拆成小步骤，降低启动门槛。

**Saner.AI** 专注于知识回忆和本地记忆，帮助用户快速找回信息，减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。这对应 LLM 的“外部记忆系统”（如向量数据库），补偿工作记忆的不稳定。

这些工具的共同逻辑：不是取代 ADHD 的执行功能，而是提供外部验证、分解、记忆和调度——正如 agent scaffolding 为 LLM 所做的那样。

## 争议：脚手架 vs 拐杖

必须诚实指出，这套同构框架和工具应用都存在局限。

首先，ADHD 大脑与 LLM 同构的论点目前多为类比推理，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。虽然“强记忆、弱控制”的认知剖面与 Transformer 注意力机制有相似性，但跨学科验证仍不足。

其次，AI 工具是“外部执行功能层”还是“拐杖”？过度依赖可能导致能力退化（来源：拐杖与脚手架）。目前缺乏长期纵向研究，用户反馈也高度个体化——部分人觉得 Goblin Tools 的分解很有用，另一些人则觉得学习使用工具本身增加了认知负荷（来源：矛盾与存疑）。

最后，工具的有效性因人而异。Motion 的初始设置成本高，Brain.fm 的临床证据有限，超聚焦是否应被引导还是打断尚无定论（来源：矛盾与存疑）。

## 今天就能试的行动

1. **建立“验证清单”**：每次冲动决策前，强制问三个问题——证据是什么？反例是什么？如果错了怎么办？这相当于 LLM 的自我批判循环。
2. **使用 Goblin Tools 分解一个任务**：把“写周报”输入 Magic ToDo，调节辣度到你觉得舒服的粒度，然后按步骤执行。
3. **设置固定日课**：每天固定时间做同一件事（如写日记、读几页书），模仿苏轼的“每日读书写字”，为大脑提供定时 re-grounding。
4. **尝试 Motion 的自动排程**：如果日程经常被打乱，让 AI 帮你动态调整，减少决策负担。

## 结语

ADHD 的冲动和 LLM 的幻觉，本质是同一个工程问题：生成核心需要外部验证循环来兜底。与其试图“修复”自己或模型，不如学会搭建脚手架。毛泽东的“批评与自我批评”、苏轼的“日课”、Goblin Tools 的任务分解、Motion 的自动排程——它们都是同一类东西：给天才的生成核心，配上一套可靠的调度层。

# 如果把 ADHD 大脑当成生成核心，Todoist 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Todoist 对 ADHD 的真正价值。

## 问题：你的大脑是一个没有 system prompt 的 LLM

你打开 Todoist，准备写那篇拖了三天的报告。五分钟后，你在研究“最便宜的露营装备评测”。这不是意志力问题——你的大脑和 LLM 共享同一个底层困境：**生成能力过剩，但执行调度层缺失**。

ADHD 大脑的典型困境是：要么分心走神，要么超聚焦跑偏。而 LLM agent 在长程任务中同样面临目标漂移——最初的指令被上下文膨胀稀释，agent 开始生成与目标无关的内容。两者都需要一个“重锚定”机制：一个在注意力漂移后能把它拉回原点的外部信号。

Todoist 被广泛推荐给 ADHD 人群，但多数建议停留在“用它列清单”。本文要论证的是：**Todoist 对 ADHD 的真正价值，在于它充当了类似 agent 工程中 system prompt 的重锚定层**——不是管理任务，而是管理注意力焦点。

## 同构：生成核心与重锚定

ADHD 大脑与 LLM 的结构同构已被研究初步证实。一项用经典 Stroop 冲突任务测试 Transformer 注意力的研究发现：短上下文中模型表现正常，但随着序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突。这与 ADHD 执行功能缺陷的核心标志一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃（来源：认知负荷）。

这意味着，ADHD 大脑和 LLM 本质上都是**高产但缺执行调度层的生成核心**。LLM 的解决方案是 system prompt——一个在每次生成前重新注入的指令，告诉模型“你现在要做什么”。ADHD 大脑也需要类似的“system prompt”，但因为它没有 API 接口，只能依赖外部工具。

Todoist 的“今日视图”和“项目视图”就是 ADHD 的 system prompt。当你在超聚焦中跑偏，打开 Todoist 看到“写报告”这个任务，它就像一条重新注入的指令，把生成核心的注意力锚定回目标。这不是简单的清单，而是一个**注意力重锚定接口**。

## 证据：Todoist 如何实现重锚定

Todoist 没有直接声称自己是 ADHD 工具，但它的设计暗合了 ADHD 执行功能补偿的核心原则。

**1. 降低认知负荷**
ADHD 工作记忆容量有限，规划本身就是高认知负荷活动。Todoist 通过自动决策减少认知负荷：AI 代替用户决定“下一步做什么、何时做、如何优先排序”（来源：认知负荷）。Todoist 的智能排序功能（如“今天”视图）自动过滤出当前优先级任务，移除了“从所有任务中选择”的决策负担。

**2. 对抗时间盲**
ADHD 的[[时间盲]]导致难以准确估计任务时长。Todoist 的时间记录功能（如“预计时长”和“完成时间”追踪）提供了外部时间感知，类似 RescueTime 的自动记录——但 RescueTime 是事后分析，Todoist 是实时锚定（来源：RescueTime）。

**3. 提供视觉锚点**
ADHD 大脑容易被环境干扰，Todoist 的“聚焦模式”和“项目分组”将注意力限制在当前上下文，类似 LLM 中限制上下文窗口来防止漂移。这呼应了[[身体在场效应]]——但这里不是真人伙伴，而是视觉上的“在场任务”。

**4. 多巴胺反馈**
完成任务的打勾声和动画提供即时多巴胺奖励，弥补 ADHD 内在多巴胺调节不足。这与 Focusmate 的实时问责机制类似，但 Focusmate 依赖外部人的社会压力（来源：Focusmate），而 Todoist 依赖内部化的奖励循环。

## 争议：脚手架还是拐杖？

wiki 资料反复警告：AI 工具作为外部执行功能层，长期使用可能导致内在能力退化（来源：矛盾与存疑）。Todoist 也不例外。如果用户依赖它来“告诉我要做什么”，而从不训练自己记住任务，那么它就成了拐杖。

关键在于使用方式。脚手架（scaffolding）是暂时支撑，最终要拆除；拐杖（crutch）是永久替代。Todoist 作为重锚定系统的正确用法是：**用它来捕获和重定向注意力，而不是替代记忆**。例如：
- 用 Todoist 记录“当前任务”，但尝试在关掉应用后回忆它（训练工作记忆）。
- 用“今天”视图减少决策负荷，但每周手动回顾项目（训练规划能力）。

Brain.fm 的案例也提示：即使有神经科学理论支持（神经锁相），缺乏 ADHD 专项研究意味着效果因人而异（来源：Brain.fm）。Todoist 同样存在个体差异——对某些人它是救命稻草，对另一些人它只是另一个被遗忘的列表。

## 同构的局限

ADHD 大脑与 LLM 的同构是隐喻而非严格等价。LLM 的“注意力机制”是数学上的权重分配，人类注意力是生物-心理-社会复合体。将 Todoist 比作 system prompt 有助于理解，但不能过度简化。例如，LLM 的 system prompt 可以精确控制输出格式，而 Todoist 无法控制你实际做什么——你仍然可能打开它然后去刷社交媒体。

此外，证据质量有限。多数 ADHD 工具的研究基于用户报告或小样本，缺乏随机对照试验（来源：矛盾与存疑）。Todoist 本身没有针对 ADHD 的临床研究，其有效性来自设计原理和用户经验。

## 今天就能试的行动

1. **设定“重锚定提醒”**：在 Todoist 中创建一个名为“当前目标”的任务，每天早晨设置为优先级 1，并添加重复提醒（每 30 分钟）。当你发现自己漂移时，点击它——就像重新注入 system prompt。
2. **用“今天”视图做注意力审计**：每天结束前，查看“今天”视图，标记哪些任务是你真正完成的，哪些是你计划但没做的。这类似于 RescueTime 的时间审计，但更聚焦于意图 vs 现实。
3. **结合 Focusmate 使用**：在 Todoist 中创建“专注时段”任务，然后预约 Focusmate 会话。用 Todoist 记录任务，用 Focusmate 提供身体在场压力——两者形成双重锚定（来源：Focusmate）。
4. **测试你的“上下文窗口”**：连续一周，每天只让 Todoist 显示 3 个任务。观察你的完成率是否提升。如果提升，说明你的大脑需要更小的上下文窗口来避免漂移。

## 结论

Todoist 不是魔法。它不能修复 ADHD 的执行功能缺陷，就像 system prompt 不能修复 LLM 的幻觉。但它可以成为你的重锚定系统——一个在注意力漂移后把你拉回原点的外部信号。

关键在于：不要把它当成任务管理器，而是当成你的**注意力 system prompt**。每次打开它，问自己：“我现在应该生成什么？”而不是“我有什么要做？”

这，才是 Todoist 对 ADHD 大脑的真正价值。

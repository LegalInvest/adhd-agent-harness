# 为什么用 Brain.fm 治 ADHD 的任务启动困难，和给 agent 套 function calling 工具调用 是一回事？

> Brain.fm 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

## 引言：从两个「启动失败」现场开始

场景 A：你坐在桌前，明明知道该写那份报告，大脑却像一台风扇狂转但挂不到档的引擎——想法很多，手指不动。这是 ADHD 典型的任务启动困难。

场景 B：你刚部署了一个能力很强的 LLM agent，它侃侃而谈，但一让它查日历、发邮件、改代码就偏离目标，甚至开始编造参数。这是 agent 缺少可靠调度层。

两个场景看似无关，但结构相同：核心都很会「生成」，但都不会「执行」。Brain.fm 被不少 ADHD 用户当作启动前的「听觉脚手架」，而 function calling 则是工程师给 LLM 套上的「行动脚手架」。它们本质上是同一套 harness 工程。

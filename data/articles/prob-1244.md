# 如果把 ADHD 大脑当成生成核心，Tiimo 能不能当它的外部执行功能层？

> 用 agent 工程的思路重新理解 Tiimo 对 ADHD 的真正价值。

## 问题：你的大脑是 GPT-4，但操作系统是 Windows 95？

ADHD 人群常自嘲“大脑像装了超强显卡但内存只有 256MB”——创意源源不绝，但计划、启动、执行却频频崩溃。与此同时，AI agent 工程师们正为一个类似问题头疼：LLM 能写诗、能编程，但一旦脱离固定提示词，就会跑偏、遗忘上下文、输出不可控。两边都在问同一个问题：**如何给一个高产生成核心套上可靠的外部调度层？**

Tiimo——一款专为 ADHD 设计的视觉化日程应用——声称能通过时间可视化、结构化日课来补偿执行功能缺陷。但它的设计真的触及了问题本质吗？如果把 ADHD 大脑当成生成核心，Tiimo 能不能当它的外部执行功能层？

## 同构：ADHD 大脑与 LLM agent 共享同一套缺陷图谱

ADHD 的执行功能缺陷——工作记忆不稳定、任务启动困难、注意力易分散——与 LLM 的“无状态性”“上下文膨胀”“输出随机性”在结构上一一对应（来源：外部执行功能层；无状态与外部记忆）。两者都是“核心强，调度弱”的生成系统：ADHD 大脑擅长发散联想，LLM 擅长文本生成，但都缺乏内置的编排、记忆、验证层。

在 LLM 工程中，这一问题的标准解法是 **agent harness**——一个包含上下文传递、工具接口、规划工件、验证循环、记忆系统和沙盒的外部框架（来源：幻觉与验证循环）。Harness 的关键不是“提醒”或“列表”，而是**确定性控制流**：计划-执行分离、状态机、周期性验证（来源：采样温度与表现波动）。没有这些，agent 就会产生幻觉、重复工作、输出不可靠。

ADHD 一侧，历史人物早已凭直觉搭建了类似的 harness。曾国藩在日记中痛骂自己“无恒”“浮躁”，最终靠“日课十二条”和“结硬寨打呆仗”的笨方法，硬生生用外部纪律补偿了内在的冲动与注意力分散（来源：人物案例）。这一系统本质上是**定时 re-grounding**（每日反省）+ **确定性流程**（黎明即起、读书不二）+ **外部记忆**（日记）。与 LLM harness 中的“定期采样监控”“计划-执行分离”“上下文日志”同构。

当代 ADHD 创业者刘艳（真格基金 Labs）则用 system thinking 构建自己的 harness：利用 hyperfocus 冲刺，用 novelty seeking 驱动探索，同时用结构化流程（如投资备忘录模板、定期复盘）防止跑偏（来源：人物案例）。她的做法与 agent 的“工具接口+验证循环”如出一辙。

## Tiimo 的定位：可视化日程，但缺少验证循环

Tiimo 的核心功能是**视觉化时间块**——将一天切成彩色时段，标注任务与休息。这对时间盲（无法感知时间流逝）的 ADHD 用户确实有帮助：它把抽象的时间变成可见的物理块，降低规划的心理负担（来源：时间盲）。用户反馈称“Tiimo 让我看到一天的样子，不再觉得时间是个黑洞”。

然而，从 harness 工程的角度看，Tiimo 只解决了“上下文呈现”和“外部记忆”的一部分——它相当于 agent 的**规划工件**（plan artifact）和**定时提醒**，但缺少两个关键组件：

1. **验证循环**：Tiimo 不会检查任务是否真的完成，也不会在用户偏离计划时触发修正。相比之下，Agent harness 会在每一步验证输出（如检查代码是否通过测试），ADHD 的“身体在场效应”则通过他人监督实现验证（来源：身体在场效应）。Tiimo 没有内置的验证机制。

2. **确定性控制流**：Tiimo 的计划是静态的——一旦被打乱（ADHD 的日常），用户需要手动调整。而 Motion 这类工具能自动重新排程，动态响应变化（来源：Motion）。Agent harness 则通过状态机确保每一步的确定性（来源：采样温度与表现波动）。Tiimo 在“动态适应”上几乎为零。

因此，Tiimo 更像是一个**可视化看板**，而不是完整的执行功能层。它降低了规划的门槛，但无法保证执行。

## 争议与局限：脚手架还是拐杖？

这里存在一个根本矛盾：AI 工具是“外部执行功能层”还是“拐杖”？如果 Tiimo 让用户依赖它的视觉提示来感知时间，一旦离开 app，时间盲可能更严重（来源：拐杖与脚手架）。目前缺乏纵向研究证明长期使用是否会削弱内在执行功能。

另一个争议是**个体差异**：部分 ADHD 用户反馈，Tiimo 的固定结构反而增加压力，而柔性引导（如 Goblin Tools 的“辣度”调节）更有效（来源：矛盾与存疑）。同样，LLM agent 的 harness 也需要根据模型能力调整约束强度——没有万能方案。

此外，Tiimo 的临床证据有限。虽然用户口碑积极，但缺乏随机对照试验证明其对任务启动或工作记忆的长期改善（来源：矛盾与存疑）。而 Motion 虽然功能更强，但初始设置成本高，反而可能增加认知负荷（来源：矛盾与存疑）。

## 结论：Tiimo 是 harness 的起点，不是终点

Tiimo 的价值在于它验证了“外部化时间结构”对 ADHD 的有效性，但它只覆盖了 harness 的一小部分。要真正成为 ADHD 大脑的外部执行功能层，Tiimo 需要增加：
- **验证循环**（如完成确认、偏差提醒）
- **动态适应**（自动调整计划）
- **记忆系统**（跨日上下文追踪）

这些正是 agent harness 工程正在攻克的方向。对 ADHD 用户而言，Tiimo 是一个不错的入门工具，但不应视为万能解药。对工程师而言，Tiimo 的设计缺陷恰恰指明了下一代 ADHD 辅助工具的方向：**不是更漂亮的日历，而是更智能的调度器+验证器**。

## 今天就能试的行动

1. **用 Tiimo 记录一周的时间块**，每天结束时标记哪些任务实际完成，哪些被跳过——这就是最简单的“验证循环”练习。
2. **尝试“计划-执行分离”**：每天早晨用 Tiimo 规划，但执行时只盯着当前块，不看后续——减少决策负担。
3. **如果 Tiimo 太静态，试试 Motion 或 Sunsama**，它们能自动重排，更接近 agent harness 的动态控制流。
4. **写“执行后复盘”日记**，像曾国藩那样——记录什么时段状态好、什么时段容易崩，形成个人化的“采样温度”曲线。

# ADHD 职场人视角：你的 ADHD 大脑，其实就是一个没装 harness 的大语言模型

> 无状态与外部记忆：工作记忆容量小、边做边忘 ↔ LLM 无跨会话状态，靠外部记忆/向量库续命。

你是否有过这样的体验：脑子里同时冒出十几个想法，但刚想动手，就忘了自己要做什么；开会时努力记笔记，但转头就找不到关键信息；任务清单越列越长，却不知道从哪里开始。这不是你不够努力，而是你的大脑——就像一个大语言模型——缺少了一个关键的调度层。

## 无状态的大脑：ADHD 与 LLM 的同构困境

ADHD 的核心并非智力不足，而是执行功能失效。工作记忆容量小、边做边忘、任务启动困难，这些痛点背后是一个深层模式：**你的大脑是一个高产但缺乏执行调度层的生成核心**（来源：生成核心与调度层）。最新研究揭示，ADHD 患者的工作记忆容量未必差，但自上而下的控制和调节能力不足，呈现“强记忆、弱控制”的认知剖面（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

这恰好与 LLM 的处境同构。LLM 拥有强大的语言理解和生成能力，但单独使用时缺乏可靠的执行调度，无法跨会话保持状态，容易上下文膨胀和推理退化（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。Transformer 的自注意力机制缺乏人类特有的执行控制机制，在长上下文下冲突解决能力崩溃至随机水平——这正是 ADHD 执行功能障碍的核心神经机制（来源：生成核心与调度层）。

换句话说，你的 ADHD 大脑，其实就是一个没装 harness 的大语言模型。

## 外部记忆：从 Mem 到 Saner.AI 的持久化上下文

既然大脑无法可靠地保存工作记忆中的信息，那么把记忆外挂就是最直接的解法。这正是 AI 工具作为“外部记忆系统”的用武之地。

**Mem** 被称为“AI 第二大脑”，它自动记录用户输入的内容，并在需要时智能回忆，本质上为 ADHD 用户提供了持久化上下文。类似地，**Saner.AI** 专注于知识回忆和本地记忆，通过快速检索减少搜索循环和标签切换（来源：Saner.AI）。这些工具的设计逻辑与 LLM 工程中的“向量数据库”如出一辙：LLM 无法跨会话保持状态，因此需要外部向量库来存储和检索历史信息，以维持对话的连贯性。

但这里有一个关键区别：对于 LLM，外部记忆是工程上的必需品；对于 ADHD 用户，外部记忆是补偿缺陷的脚手架。过度依赖可能削弱主动记忆的意愿——这正是“拐杖与脚手架”的辩证关系（来源：拐杖与脚手架）。真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

## 任务分解与启动：Goblin Tools 和 Lex 的启动脚本

任务启动困难是 ADHD 最典型的障碍之一。面对一个庞大的任务，大脑就像 LLM 遇到“冷启动”问题一样，无法自动生成可行的第一步。

**Goblin Tools** 的“Magic ToDo”功能自动将复杂任务分解为可管理的小步骤，例如输入“整理房间”，它会输出“捡起地板上的衣服”“擦桌子”等具体动作（来源：Goblin Tools）。这种分解降低了启动门槛，用户反馈称“将压倒性的事情变成一系列不压倒性的事情”（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。

**Lex** 更进一步，允许用户通过单一指令触发复杂、多步骤的任务序列（来源：Lex）。这相当于为 LLM 提供一个预定义的“启动脚本”，让它自动执行一系列操作，减少决策疲劳。在 Agent 工程中，这被称为“工作负载专用模型路由”和“分离规划与执行的双代理架构”（来源：Building AI Coding Agents for the Terminal）。

然而，个体差异导致效果不一（来源：矛盾与存疑）。有些用户可能觉得分解后的步骤仍然太多，或者 AI 无法准确理解意图。因此，工具需要可定制性，而不是“一刀切”的方案。

## 实时反馈与时间线索：Otter.ai 和 Reflect 的上下文工程

ADHD 的“时间盲”与 LLM 的时序推理弱点同构。两者都需要外部线索来感知时间流逝和任务进度。

**Otter.ai** 的实时转录功能提供了即时外部反馈，类似于“身体在场效应”的虚拟版本（来源：ADHD 的 AI 工具全景）。**Reflect** 通过结构化提问引导用户进行反思，主动设计“当下注意什么”（来源：ADHD 的 AI 工具全景）。这些工具相当于为 ADHD 大脑安装了“上下文工程”模块，防止被环境带偏或上下文膨胀。

在 LLM 工程中，类似的技术包括“自适应上下文压缩”和“惰性工具发现”，目的是防止推理退化（来源：Building AI Coding Agents for the Terminal）。

## 脚手架 vs 拐杖：一个必须明确的边界

AI 工具作为外部执行功能层，能够显著改善工作记忆、任务启动和时间感知等缺陷（来源：ADHD 的 AI 工具全景）。但证据主要来自用户报告，缺乏大规模随机对照试验（来源：矛盾与存疑）。更关键的是，过度依赖 AI 工具可能削弱内在能力，例如过度使用 Otter.ai 可能削弱主动记笔记的能力（来源：ADHD 的 AI 工具全景）。

我的核心判断是：**AI 工具应该作为脚手架，促进能力发展，而非永久拐杖。** 这意味着工具设计应鼓励用户逐步内化策略，而不是完全替代大脑功能。例如，Goblin Tools 的分解步骤可以逐渐减少，让用户学会自己分解任务。

诚实地说，目前大多数工具的设计并未充分考虑“去依赖”路径。用户需要主动觉察自己的使用模式，避免工具变成新的“拖延借口”。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个让你焦虑的任务**：输入你拖延已久的一件事，看看 AI 如何拆解。感受启动门槛降低后的心理变化。
2. **用 Saner.AI 或 Mem 建立一个“第二大脑”**：每天记录 3 件重要事项，并尝试用搜索功能找回一周前的信息。观察是否减少了“信息丢失”的焦虑。
3. **用 Otter.ai 或 Reflect 记录一次会议或反思**：事后对比 AI 生成的摘要和你的记忆，评估外部记忆的准确性。思考哪些信息你本可以自己记住。
4. **设置一个“无工具”时段**：每天留出 30 分钟，仅用纸笔完成任务。这能帮助你觉察自己的内在能力，并避免过度依赖。

你的 ADHD 大脑不是缺陷，而是一个未安装 harness 的生成核心。现在，是时候为自己搭建脚手架了。

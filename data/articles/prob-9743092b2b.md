# 为什么用 Focusmate 治 ADHD 的学习半途而废，和给 agent 套 外部记忆/向量库 是一回事？

> Focusmate 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

你有没有这样的经历：深夜打开第 7 个教学视频，进度条卡在 30%，收藏夹里躺着 5 门没完成的课。你骂自己三分钟热度，但问题可能不是懒，而是架构缺陷——大脑这台生成核心，没有可靠的外部状态层。

在 AI 工程里，工程师也遇到同一类 bug：LLM 每次会话几乎从零开始，上下文一长就退化，输出还带有采样随机性。给 agent 套向量库、记忆层、知识图谱，不是让它变聪明，而是让它有状态、可追踪。ADHD 的学习半途而废，和 agent 的上下文崩溃，其实是同一个故障模式：无状态。

## 1. 同一台机器：高产、无状态、缺调度

ADHD 大脑与 LLM 在结构上有相似之处：两者都是强大的生成核心，但缺乏可靠的执行功能调度层（来源：AI 与 ADHD 的学习方式）。ADHD 的困难表现为任务启动难、工作记忆不稳定、时间盲、注意力容易被环境带偏；LLM 则表现为无跨会话状态、上下文膨胀、采样温度带来的非确定性，而小错误会累积，非确定性让可重复性变得复杂（来源：AI Agent Systems: Architectures, Applications, and Evaluation；AI Agents in 2026: Tools, Memory, Evals, and Guardrails）。

关键对应在于「无状态与外部记忆」：ADHD 的工作记忆不稳定，LLM 没有跨会话状态，两者都需要外部系统来记住目标、步骤和约束（来源：AI 与 ADHD 的学习方式）。笔记、日历、任务清单、向量库、记忆层、知识图谱，都是同一类补偿装置。

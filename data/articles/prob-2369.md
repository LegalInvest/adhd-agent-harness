# ADHD 创业者如何用「给生成核心套 harness」的思路应对把想法落地执行里的分心走神或超聚焦跑偏，丢失目标？

> 把 LLM 工程里的「长程任务里 agent 会目标漂移，需要重锚定 system prompt」迁移到ADHD 创业者的把想法落地执行场景，双域证据作答。

你是否有过这样的经历：脑海里有一个绝妙的创业想法，兴奋地打开电脑准备执行，结果半小时后却在刷社交媒体、整理书桌、或者研究某个无关的冷知识？或者，你进入了所谓的「超聚焦」状态，一口气写了三小时代码，却发现自己实现的是某个不重要的边缘功能，核心目标早已偏离。

这不是意志力问题。这是ADHD大脑的典型运作模式——一个「高产但缺乏可靠执行调度层的生成核心」（来源：AI与ADHD的专注力）。你的大脑像一台强大的生成式AI，创意源源不断，但缺少一个内置的调度器来告诉你「现在该关注什么」。

有趣的是，这种结构缺陷与LLM agent在长程任务中面临的问题高度同构。当agent执行复杂任务时，随着上下文膨胀，它会逐渐偏离原始目标，出现「推理退化」和「目标漂移」（来源：上下文工程）。工程师们为此开发了各种harness——通过定时重锚定system prompt、外部记忆系统、以及严格的控制逻辑来防止agent跑偏。

那么，ADHD创业者能否借用同样的思路，为自己套上一个「harness」？

## 同构：从LLM工程到ADHD自我管理

让我们先看清这个同构关系：

| ADHD大脑 | LLM agent |
|---------|-----------|
| 工作记忆不稳定，任务上下文易丢失 | 无跨会话状态，每次对话需重新加载上下文 |
| 易被环境带偏，任务碎片化 | 上下文膨胀导致推理退化 |
| 超聚焦时可能偏离核心目标 | 长程执行中目标漂移 |
| 缺乏内置执行调度层 | 需要外部harness来调度 |

LLM工程师的解决方案是什么？他们设计「上下文工程」——主动管理agent当前关注的信息范围，防止它被无关细节带偏。具体手段包括：定时重锚定system prompt、使用外部记忆存储关键上下文、以及通过控制逻辑确保每一步都朝着最终目标前进（来源：上下文工程）。

这些手段，与ADHD自我管理中的经典策略惊人地一致。

## 历史中的ADHD harness：曾国藩与徐渭的启示

历史上，那些取得巨大成就的ADHD特质者，早已无意识地使用了类似的harness。

**曾国藩**是典型注意力不集中型ADHD：30岁前浮躁坐不住，日记里天天骂自己「无恒」「浮躁」；读书慢到「他人目下二三行，余或疾读不能终一行」；一生被银屑病折磨，情绪极不稳定。他的harness是什么？「日课十二条」——黎明即起、读书不二、谨言、写日记反省……这套严格的日常程序，本质上就是定时重锚定。每天用固定的仪式让自己回到「正确轨道」，用「结硬寨打呆仗」的笨办法抵消冲动。

**徐渭**则是另一种典型：诗书画戏军全能，但执行功能极差，8次举人不中，情绪失调到杀妻、9次自杀。他的harness是「以书画为核心，把所有痛苦和精力注入艺术创作」——这相当于为超聚焦设定了一个明确的「输出通道」，防止能量四处漫溢。

他们的共同点：不是试图「治愈」ADHD，而是为大脑套上一个外部调度系统——日课相当于定时re-grounding，日记相当于外部记忆，专注单一领域相当于限制上下文范围。

## 现代AI工具：作为外部执行功能层的harness

今天，AI工具可以成为这种harness的数字版本。

**RescueTime**自动记录时间去向，揭示「时间到底去了哪里」（来源：RescueTime）。这相当于为ADHD大脑提供了一个「外部记忆体」，补偿工作记忆缺陷——你不需要记住自己刚才做了什么，工具会告诉你。同时，它还能阻断分心网站，相当于在agent的action space中移除干扰选项。

**Focusmate**通过虚拟身体在场提供实时问责（来源：Focusmate）。这相当于一个「人类in the loop」的调度器——当你需要启动任务时，匹配的伙伴就是你的「重锚定信号」，让你回到目标上。

**Routinery**将日常流程分解为步骤序列，并用过渡提示引导你一步步执行（来源：上下文工程）。这相当于为agent设计了清晰的「step-by-step plan」，防止跳跃和遗漏。

这些工具的共同逻辑：**将执行调度从大脑内部转移到外部系统**，就像LLM agent依赖harness而非自身来保持方向。

## 核心观点：harness vs 拐杖——关键在于内化

但这里有一个必须诚实的局限：AI工具是否会导致永久依赖？wiki资料明确提出了「拐杖与脚手架」的争议——工具既可能成为永久依赖的拐杖，也可能成为逐步内化的脚手架（来源：AI与ADHD的专注力）。

我的判断是：关键在于「内化」——你是否在使用工具的过程中，逐渐建立起自己的执行习惯？曾国藩的日课最终内化成了他的行为模式，即使没有日记他也能大致保持自律；而如果只是依赖RescueTime的阻断功能，一旦关闭工具就立刻失控，那就是拐杖。

另一个争议是：同构性目前只是一个理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。ADHD大脑的神经机制与LLM的架构差异巨大，但这种类比在工程层面是有用的——它提供了一个可操作的框架来设计解决方案。

## 今天就能试的4个行动

1. **设置每日「重锚定」闹钟**：每天固定时间（如上午10点、下午3点），花2分钟问自己：「我现在在做的事情，是否在推进核心目标？」这相当于定时刷新system prompt。

2. **用RescueTime或类似工具追踪一周**：不是为了自责，而是为了获得「外部记忆」——看清时间到底去了哪里，识别出最常见的漂移模式。

3. **为每个项目写一个「核心目标便签」**：贴在电脑旁或设为手机壁纸。当感觉注意力开始分散时，看一眼便签——这就是你的「system prompt」。

4. **尝试一次Focusmate会话**：预约30分钟与陌生人一起工作。你会发现，仅仅知道有人在看着你，就能显著减少目标漂移。

## 结语

ADHD创业者不需要「治愈」自己的大脑，而是需要学会为它套上合适的harness。就像LLM agent需要外部调度才能完成长程任务一样，你的生成核心需要一个可靠的执行层。这个harness可以是曾国藩的日课、徐渭的艺术专注、RescueTime的时间追踪，或者Focusmate的虚拟陪伴。关键不是工具本身，而是你愿意每天花几分钟，主动「重锚定」自己的目标。

毕竟，最强大的生成核心，也需要一个稳定的调度器才能产出真正的杰作。

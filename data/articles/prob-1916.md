# ADHD 自由职业者如何用「给生成核心套 harness」的思路应对无人监督下的自我管理里的执行功能（计划/组织/启动）先天偏弱？

> 把 LLM 工程里的「agent 的编排/记忆/工具层就是「外部执行功能」」迁移到ADHD 自由职业者的无人监督下的自我管理场景，双域证据作答。

你是一位 ADHD 自由职业者。没有老板、没有固定工时、没有同事催促。你大脑里每天冒出十个绝妙想法，但到了该打开文档、打第一通电话、甚至只是“坐下开始”的时候，你卡住了。这不是懒，这是执行功能——计划、组织、启动——先天偏弱。

与此同时，在 AI 工程领域，一群工程师正在解决一个看起来毫不相关的问题：如何让大语言模型（LLM）不只是在对话中流畅输出，而是能可靠地完成多步任务、跨会话记忆、自主规划？他们的答案是：给 LLM 套一个 harness——编排层、记忆层、工具层。

这两群人面对的，其实是同一个底层问题。

## 同构：高产但缺调度层的生成核心

ADHD 大脑与 LLM 在结构上惊人地相似：两者都是“高产但缺乏可靠执行调度层的生成核心”（来源：ADHD × AI 的科学与研究前沿）。ADHD 大脑的默认模式网络过度活跃，想法源源不断，但前额叶执行控制不足；LLM 的生成能力强大，但缺乏稳定的上下文控制器。两者都需要一个外部调度层来引导输出（来源：生成核心与调度层）。

对于 ADHD 自由职业者，这意味着：你并不缺少创意、知识或动力——你缺少的是把想法变成行动、把行动维持下去的那个“调度系统”。而 LLM 工程里的 agent harness，本质上就是一个外部执行功能层。

## 编排层：从“下一步做什么”到系统自动推进

ADHD 自由职业者最大的痛点之一是任务启动。面对一个模糊的大任务（“整理客户提案”），大脑会进入瘫痪状态。Goblin Tools 的 Magic ToDo 功能正是针对这一点：它能自动将任何任务分解为更小、更易管理的步骤（来源：Goblin Tools）。用户输入“整理房间”，AI 输出“捡起地板上的衣服”“擦桌子”——把压倒性的事情变成一系列不压倒性的事情（来源：Goblin Tools）。

这对应 LLM/agent 侧的什么？编排。生产级 agent 系统通过外部记忆（如向量数据库）解决溢出问题，并组合使用会话内上下文记忆、长期检索向量库和结构化知识库（来源：无状态与外部记忆）。没有 harness 的 LLM 只是一个无状态的 API 调用（来源：无状态与外部记忆）。同样，没有外部编排的 ADHD 大脑，只是一个无状态的创意生成器。

Motion 则更进一步：它自动根据任务、会议和截止日期创建并动态调整每日计划，在日程被打乱时自动重新安排（来源：Motion）。这相当于 agent 的实时规划与重规划——不是让你每天手动排程，而是系统替你调度。

## 记忆层：解决“每天重新开始”的困局

ADHD 的工作记忆容量有限且不稳定，导致跨会话任务难以持续。典型表现为：每天需要重新回忆项目上下文，否则容易偏离轨道（来源：无状态与外部记忆）。Saner.AI 正好解决这个问题：它专注于知识回忆和本地记忆，减少搜索循环和标签切换（来源：Saner.AI）。

在 LLM 侧，标准的调用是无状态的，每次对话都是独立的（来源：无状态与外部记忆）。生产级 agent 通过外部记忆系统（向量数据库、结构化知识库）维持连续性。ADHD 大脑同样需要“永远在线”的外部记忆体来补偿内部工作记忆的波动（来源：无状态与外部记忆）。

## 工具层：把环境变成执行功能的延伸

ADHD 自由职业者常通过他人在场提升执行力——这就是“身体在场效应”。LLM 侧也有类似对应：通过外部 agent 协作增强表现（来源：身体在场效应）。工具层就是你的“虚拟他人”：提醒系统、时间追踪、任务分解器。

但这里有一个关键边界：脚手架 vs. 拐杖。

## 脚手架 vs. 拐杖：你必须自己“攀登”

外部工具可以充当临时脚手架，外化思维，减轻认知负荷（来源：拐杖与脚手架）。但专家警告过度依赖风险：若工具替代了治疗或自我管理，可能造成依赖（来源：拐杖与脚手架）。真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：拐杖与脚手架）。

这意味着：AI 工具不应该替你决策，而是帮你降低执行决策的门槛。Goblin Tools 分解任务后，你仍然需要动手去做；Motion 排好日程后，你仍然需要按时开始。工具是 harness，不是自动驾驶。

## 争议与局限

必须诚实指出：现有证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑）。过度依赖 AI 工具可能削弱内在时间感知能力（来源：时间盲）。个体差异也很大——注意力主导型 vs. 冲动主导型的 ADHD 患者，对同一工具的反应可能完全不同（来源：矛盾与存疑）。

此外，目前没有工具能完美覆盖所有执行功能缺陷。Saner.AI 擅长记忆，但可能无法应对时间盲；Motion 擅长排程，但初始设置对执行功能较弱的用户本身就是挑战（来源：Motion）。

## 今天就能试的 3 件事

1. **用 Goblin Tools 分解一个你拖延了一周的任务**：把“写周报”变成“打开电脑→登录邮箱→查看上周笔记→列出三点进展→写一句话总结”。感受启动门槛的降低。
2. **用 Saner.AI 建立你的“第二大脑”**：把客户偏好、项目背景、常用模板存入其中，减少每天重新回忆上下文的摩擦。
3. **用 Motion 替代你手动排日程**：把所有任务和截止日期丢进去，让它自动生成每日计划。观察它如何帮你减少“下一步做什么”的决策负担。

记住：你不是在修复自己的大脑，你是在给它套一个 harness。就像 LLM 需要 agent 框架才能成为可靠的生产力工具，你的生成核心也需要一个外部执行功能层。这不是作弊——这是工程。

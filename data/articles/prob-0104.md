# 如果把 ADHD 大脑当成生成核心，Tiimo 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Tiimo 对 ADHD 的真正价值。

## 问题：为什么你总是“边做边忘”？

如果你是一个 ADHD 患者，你一定经历过这样的场景：打开电脑准备写报告，突然想起要回一条消息，回完消息又看到待办清单里的“整理桌面”，等桌面整理完，你已经完全忘了刚才要写报告这件事。你的工作记忆就像一个漏水的桶，刚装进去的信息，转眼就流走了。

如果你是做 Agent 工程的工程师，你一定熟悉另一个场景：你精心设计了一个 LLM Agent，它能在一次对话里流畅地完成复杂任务，但只要对话一结束，它就彻底“失忆”了——下次启动时，它不记得任何历史。你不得不为它接上向量数据库，把关键信息持久化，再写一个调度器来管理上下文。

这两个场景，听起来像是两个世界的问题，但它们的结构完全一样。ADHD 大脑和 LLM，都是“高产但缺执行调度层的生成核心”。LLM 没有跨会话状态，ADHD 大脑的工作记忆容量小且易被干扰。两者的解法也同构：**外部记忆 + 调度系统**。Tiimo，这个被很多 ADHD 用户当作日程管理工具的应用，如果从 Agent 工程的视角重新审视，它可能正是 ADHD 大脑所需要的“无状态与外部记忆”架构。

## 无状态与外部记忆：Agent 工程的视角

在 LLM Agent 的架构中，无状态（stateless）是指每个请求独立处理，不保留会话历史。Agent 本身没有记忆，所有上下文都存储在外部的向量数据库或缓存中。调度器（harness）负责在每次调用前从外部记忆里检索相关上下文，组装成 prompt，再交给 LLM 生成。这个模式的好处是：LLM 可以专注于当前任务，不受历史干扰；同时，外部记忆可以持久化、结构化、可检索。

ADHD 大脑的“无状态”模式几乎是天生的。工作记忆容量小（来源：ADHD 的 AI 工具全景），意味着每次只能处理少量信息，而且很容易被新信息冲刷掉。ADHD 患者经常“边做边忘”，不是因为不努力，而是因为大脑的“上下文窗口”太小，且没有自动的持久化机制。所以，ADHD 需要的不是“更大的工作记忆”，而是一个可靠的外部记忆系统，和一个能帮它“每次只加载当前任务”的调度器。

Tiimo 正是这样设计的。它不是一个普通的日历应用，而是一个**基于视觉时间线的任务调度器**。用户把一天的活动拆成时间块，每个时间块只显示当前任务，其他任务被隐藏。当你在 Tiimo 上设置“14:00-15:00 写报告”，到了 14:00，Tiimo 会告诉你“现在开始写报告”，并且只显示这个任务。其他任务（比如回消息、整理桌面）都被屏蔽了，直到它们自己的时间块到来。

这个机制，本质上就是 Agent 工程里的“上下文管理”：**只加载当前需要的上下文，把其他上下文持久化到外部，等需要时再加载**。Tiimo 的视觉时间线，就是 ADHD 大脑的向量数据库——它存储了所有任务的元数据（时间、顺序、时长），但只在需要的时候才把具体内容拉进工作记忆。

## 同构的证据：从工具到机制

这个同构不是空洞的类比，而是有真实工具和用户报告支撑的。多个 AI 工具已经被验证能通过外部记忆和调度补偿 ADHD 的执行功能缺陷：

- **Goblin Tools** 的 Magic ToDo 功能，能把“整理房间”这样的模糊任务分解成“捡起地板上的衣服”“擦桌子”等具体步骤（来源：Goblin Tools）。这相当于为 LLM 提供了“任务分解 prompt”，把一个大任务拆成多个小步骤，每个步骤只占用一个小上下文窗口。用户报告称，这种分解降低了启动焦虑（来源：Goblin Tools）。

- **Saner.AI** 专注于知识回忆和本地记忆，帮助用户快速找回信息，减少标签切换（来源：Saner.AI）。这相当于为 LLM 接了一个向量检索系统，让 ADHD 大脑可以“按需回忆”，而不是靠脆弱的工作记忆硬扛。

- **Lex** 允许用户通过单一指令触发复杂、多步骤的任务序列（来源：Lex）。这就像 Agent 工程里的“workflow”：一个指令启动一个预定义的管道，LLM 按顺序执行，无需每一步都重新决策。

- **Mem** 和 **Otter.ai** 提供持久化记忆和实时转录，相当于为 ADHD 大脑提供了一个“外部硬盘”，会议内容、想法、笔记都可以自动保存并随时检索（来源：ADHD 的 AI 工具全景）。

- **Reflect** 通过结构化提问引导反思，相当于为 LLM 提供“反思 prompt”，帮助它从历史中提取经验（来源：ADHD 的 AI 工具全景）。

- **Focusmate** 利用算法匹配虚拟身体加倍伙伴，通过社会存在感降低分心阈值（来源：身体在场效应）。这相当于为 LLM 提供了一个“监督器”，确保它不会偏离当前任务。

这些工具的共同点，就是**把执行功能（调度、记忆、启动）从大脑内部转移到外部系统**。Tiimo 在这个生态中的独特位置，是它提供了一个**时间维度的调度框架**——它不只是存储信息，而是按照时间线组织信息，并在正确的时间把正确的信息推送到用户面前。

## 脚手架还是拐杖？——争议与局限

然而，这个同构视角也揭示了一个危险：如果 ADHD 用户过度依赖 Tiimo 这样的外部系统，可能会削弱内在的时间感知和任务规划能力。正如 wiki 资料中的“矛盾与存疑”所指出的：“过度依赖 AI 工具可能削弱内在能力”（来源：矛盾与存疑）。Tiimo 如果被当作“拐杖”——即用户完全放弃内在的时间管理，只靠外部提醒——那么当工具不可用时，用户会陷入更深的混乱。

真正的“脚手架”应该是什么？脚手架是暂时性的支持，目的是帮助用户发展出内在的能力。比如，Tiimo 的视觉时间线可以教会用户“任务需要时间块来组织”这个元认知策略，然后用户逐渐内化这个策略，最终可以不用工具也能自己规划时间。而拐杖则是永久替代，用户永远离不开工具。

目前，Tiimo 的设计更偏向脚手架：它让用户主动设置时间块，而不是自动生成；它提供视觉反馈，让用户看到时间流逝。但证据显示，现有研究缺乏大规模对照实验，且个体差异很大（来源：矛盾与存疑）。有些用户可能因为社交焦虑或拒绝敏感性焦虑，对工具产生过度依赖（来源：身体在场效应）。

如果你是一个 ADHD 患者，或者你正在为 LLM Agent 设计调度系统，以下行动可以立即尝试：

1. **用 Tiimo 做一次“上下文隔离”实验**：选一个你经常拖延的任务（比如写周报），在 Tiimo 上设置一个 30 分钟的时间块。在这 30 分钟内，只看 Tiimo 上显示的任务，关闭所有其他通知。观察你的启动速度和完成率是否提升。

2. **为你的 LLM Agent 实现一个“时间线调度器”**：如果你的 Agent 需要处理多步骤任务，不要把所有历史都塞进上下文。而是像 Tiimo 一样，用一个外部存储（比如 Redis 或向量数据库）保存任务状态，每次只加载当前步骤所需的上下文。这能显著减少上下文膨胀和幻觉。

3. **结合身体加倍与外部记忆**：使用 Focusmate 或找一个朋友，在你使用 Tiimo 时“在场”。身体加倍能降低分心阈值（来源：身体在场效应），而 Tiimo 提供任务结构。两者结合，相当于为 ADHD 大脑提供了一个“监督器+调度器”的双重 harness。

Tiimo 对 ADHD 的真正价值，不是作为一个花哨的日历应用，而是作为 ADHD 大脑的“外部记忆与调度系统”。它和 LLM Agent 的 Harness 是同构的：解决的都是“生成核心”的执行缺陷。但请记住，任何工具都只是脚手架，不是拐杖。用 Agent 工程的思维来使用它，你就能既享受效率的提升，又保持内在能力的成长。

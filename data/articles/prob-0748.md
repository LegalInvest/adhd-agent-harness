# Routinery 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Routinery：它到底补上了哪一层执行功能？

## 问题：为什么“知道该做什么”和“实际去做”之间隔着鸿沟？

ADHD 群体最熟悉的困境是：大脑里充满创意和计划，但一到执行就卡壳。不是不想做，而是工作记忆像漏水的桶，刚想到的步骤下一秒就消失；任务启动像冷启动的引擎，需要巨大的外部推力。同时，工程师在构建 AI agent 时也面临类似问题：LLM 是强大的生成核心，但缺乏内置的调度和执行控制，需要外部的“agent harness”来管理工具调用、上下文和状态（来源：What is an agent harness in the context of large-language models?）。

## 同构：ADHD 大脑与 LLM 共享同一类缺陷

ADHD 大脑的核心特征是“高产但无序的生成能力，同时缺乏可靠的执行调度层”（来源：ADHD 大脑与 LLM 的同构）。实证研究表明，ADHD 表现出“强记忆、弱控制”的神经心理模式：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。LLM 同样如此——它有无穷的生成能力，但上下文控制薄弱，容易在长对话中丢失焦点。因此，两者都需要外部脚手架来补偿执行功能：ADHD 需要工具来“减少决策、保留上下文、让下一步行动显而易见”（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）；LLM 需要 agent harness 来提供工具接口、上下文管理、安全执行等编排能力（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

## 答案：Routinery 作为外部执行层

Routinery 正是这样一款工具：它专为 ADHD 设计，通过固定流程、定时提醒和步骤引导，将“精确、重复、状态维护”的任务从大脑中卸载到外部。用户只需在设定好的例程中一步步执行，无需每次重新决策。这与 LLM 通过 function calling 将精确计算外包给专用工具（如计算器、数据库查询）的逻辑完全一致——两者都是在保留生成核心的同时，用外部系统处理那些生成核心不擅长的精确操作。

## 但有人用错了：脚手架 vs 拐杖

问题在于，许多人将 Routinery 当作“记忆替代品”或“决策永久外包”，而不是“执行辅助”。他们依赖它记住每一个步骤，甚至不敢在没有 app 的情况下开始日常事务。这正是“拐杖与脚手架”的辩证关系：AI 工具应作为脚手架促进能力发展，而非永久拐杖（来源：拐杖与脚手架）。同样，在 agent 工程中，如果 harness 接管了太多本该由 LLM 自身完成的推理（如简单逻辑判断），agent 就会变得僵化、失去灵活性。证据表明，过度依赖 AI 工具可能削弱内在能力——例如，Otter.ai 减轻笔记负担，但过度依赖可能削弱主动记笔记的能力（来源：ADHD 的 AI 工具全景）。Routinery 也不例外：它应帮你建立习惯，而非代替你思考“下一步做什么”。

## 实测：Routinery 到底补上了哪一层？

从执行功能角度看，Routinery 主要补偿了**任务启动**和**状态维护**。ADHD 的任务启动困难对应 LLM 的“冷启动”问题，而 Routinery 的定时提醒和步骤列表相当于为生成核心提供启动脚本（来源：ADHD 的 AI 工具全景）。状态维护方面，ADHD 易被环境带偏（上下文膨胀），Routinery 通过固定顺序和视觉进度条提供外部时间线索，类似 Claude 的对话式整理（来源：同上）。但它并未直接补偿工作记忆或时间感知——这些需要其他工具如 Saner.AI（知识回忆）或 Goblin Tools（任务分解）（来源：Saner.AI；Goblin Tools）。

## 局限与争议

必须诚实指出：目前针对 Routinery 的独立随机对照试验缺失，其效果主要基于用户报告（来源：矛盾与存疑）。个体差异很大——有人觉得固定流程解放大脑，有人觉得僵化压抑。更重要的是，如果用户从未尝试理解自己的执行功能模式，仅靠工具机械执行，可能永远无法发展内在的调度能力。这与 agent 工程中的教训一致：过度依赖 harness 会导致 LLM 失去自主规划能力。

## 今天就能试的 3 件事

1. **用 Routinery 创建一个“最小启动例程”**：只包含起床后 3 个动作（如喝水、开窗、铺床），坚持一周，观察启动焦虑是否降低。
2. **每周一次“无 app 日”**：故意不用 Routinery 完成某个简单例程（如睡前准备），测试自己是否已内化步骤。
3. **结合 Goblin Tools 分解复杂任务**：如果 Routinery 的固定流程不适合某件事，用 Goblin Tools 的 Magic ToDo 将其拆解，再手动录入 Routinery（来源：Goblin Tools）。

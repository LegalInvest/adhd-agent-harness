# Todoist 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Todoist：用久了会让 ADHD 更强还是更依赖？

## 问题：边做边忘，Todoist 是救星还是陷阱？

你是否有过这样的经历：打开 Todoist，看着密密麻麻的任务列表，却不知道从何下手；或者刚完成一项任务，转头就忘了下一步该做什么。对于 ADHD 人群，工作记忆容量小、边做边忘是日常（来源：执行功能障碍）。Todoist 这样的任务管理器被寄予厚望，但问题随之而来：它究竟是帮助我们搭建行动框架的脚手架，还是让我们放弃自我管理的拐杖？

## 同构：ADHD 大脑与 LLM 的“无状态”困境

ADHD 大脑与 LLM（大型语言模型）有一个惊人的共同点：两者都是“高产但缺执行调度层的生成核心”。ADHD 大脑能产生无数创意，但缺乏将想法转化为行动的执行功能；LLM 能生成流畅文本，但缺乏跨会话的持续状态，每次对话都像“失忆”一般（来源：无状态与外部记忆）。

具体来说：
- **工作记忆**：ADHD 的工作记忆容量保留但控制弱，易被干扰；LLM 同样有强记忆容量但弱上下文控制（来源：工作记忆）。
- **任务启动**：ADHD 的任务启动困难对应 LLM 的“冷启动”问题。Goblin Tools 的 Magic ToDo 能将“整理房间”分解为“捡起衣服”“擦桌子”等步骤，相当于为生成核心提供启动脚本（来源：Goblin Tools）。
- **时间盲**：ADHD 的时间感知缺陷与 LLM 的时序推理弱点同构。

因此，AI 工具作为外部执行功能层，通过分解任务、管理上下文、增强记忆来补偿内在缺陷（来源：ADHD 的 AI 工具全景）。Todoist 正是这种外部记忆系统的典型代表。

## 脚手架 vs 拐杖：边界在哪里？

关键区别在于：工具是促进能力发展，还是替代能力。

**脚手架的特征**：
- 提供结构，但用户仍需主动思考。例如，Goblin Tools 将任务分解后，用户仍需选择执行顺序（来源：Goblin Tools）。
- 使用后，用户逐渐内化分解策略，减少对工具的依赖。
- 工具设计鼓励用户参与，如 Reflect 的结构化提问引导反思（来源：Reflect）。

**拐杖的特征**：
- 用户完全依赖工具，丧失自主规划能力。例如，过度依赖 Otter.ai 的实时转录，可能削弱主动记笔记的能力（来源：Otter.ai）。
- 工具成为“大脑外挂”，用户不再尝试记忆或组织。
- 一旦工具不可用，用户陷入混乱。

Todoist 本身是中性的，但使用方式决定其属性。如果用户只是机械地添加任务而不思考优先级，Todoist 就成了拐杖；如果用户利用它来训练自己的任务分解和时间管理能力，它就是脚手架。

## 证据：工具有效，但依赖风险真实

现有证据主要来自用户报告，缺乏大规模对照实验（来源：矛盾与存疑）。例如：
- Goblin Tools 被用户评价为“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools）。
- Saner.AI 通过增强知识回忆，减少搜索循环（来源：Saner.AI）。
- Lex 的单一指令触发多步骤，降低启动门槛（来源：Lex）。

但多个页面警告过度依赖风险：
- 过度依赖 AI 工具可能削弱内在时间感知能力（来源：时间盲）。
- 存在依赖风险（来源：任务启动）。
- 工具设计者声称是“补偿”而非“替代”，但缺乏避免依赖的具体指导（来源：矛盾与存疑）。

## 核心观点：Todoist 是脚手架，但需有意识使用

我的判断是：Todoist 对 ADHD 用户是有效的脚手架，前提是用户有意识地将其作为训练工具，而非无脑依赖。关键在于：
1. **使用分解功能，但尝试自己先分解**：先用大脑思考步骤，再与 Todoist 的 AI 建议对比。
2. **定期回顾任务列表，而非被动查看**：每天花5分钟审视列表，主动调整优先级。
3. **结合反思工具**：如 Reflect 的结构化提问，帮助内化时间管理策略。

对于 LLM/Agent 工程师，这呼应了 Agentic Harness 的设计原则：外部记忆系统（如向量库）是必要的，但 agent 应保留自主调度能力，避免完全依赖外部状态。

## 今天就能试的行动

1. **任务分解练习**：选一个复杂任务（如“准备报告”），先用脑分解，再用 Goblin Tools 或 Todoist 的 AI 功能分解，对比差异。
2. **每周回顾**：在 Todoist 中设置每周回顾任务，检查哪些任务被拖延，分析原因。
3. **工具断联实验**：每周选一天不使用 Todoist，仅靠手写清单或脑记，观察自己的执行情况。
4. **工程师实践**：为你的 agent 设计一个“反思”模块，定期评估外部记忆的使用效率，避免过度依赖。

## 局限与争议

- 个体差异大：对某些 ADHD 用户，Todoist 可能无效甚至增加焦虑（来源：矛盾与存疑）。
- 多巴胺干预效果存在争议（来源：矛盾与存疑）。

## 结论

Todoist 之于 ADHD，如同向量库之于 LLM agent——是必要的脚手架，但过度依赖会削弱核心能力。关键在于有意识使用：让工具服务你，而非你服务工具。

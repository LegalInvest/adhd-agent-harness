# Todoist 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Todoist：用久了会让 ADHD 更强还是更依赖？

## 问题：边做边忘，是大脑缺陷还是设计缺陷？

你有没有过这样的体验：打开 Todoist，看着昨天列好的任务列表，却完全想不起当时为什么要把“整理报销单”排在第一优先级？或者，你明明已经完成了某个步骤，却在几分钟后再次打开 Todoist，反复确认自己到底做没做？

这不是你的记忆力差。这是 ADHD 大脑的典型困境——**工作记忆容量小，边做边忘**。你的大脑是一个强大的生成核心，能瞬间迸发创意、联想和解决方案，但它缺乏一个可靠的执行调度层。就像一台没有操作系统的超级计算机：硬件性能爆表，但无法稳定地安排任务、维持上下文、按顺序执行。

巧合的是，你正在用的 LLM 工具（如 ChatGPT、Claude）也有同样的毛病。LLM 没有跨会话状态，每次对话都是“失忆”重启，必须靠外部记忆（向量库、RAG、提示缓存）来续命。**ADHD 大脑与 LLM 在结构上同构：两者都是高产但缺调度层的生成核心。**（来源：ADHD 的 AI 工具全景）

于是，一个核心问题浮现：**Todoist 之于 ADHD，是脚手架还是拐杖？** 它能帮我们搭建外部执行功能层，还是会让我们永远依赖它，再也无法独立？

## 同构视角：外部记忆与无状态生存

要回答这个问题，我们先看 ADHD 和 LLM 各自怎么解决“无状态”问题。

**ADHD 侧：** 你每天靠 Todoist 记住“接下来做什么”。你的工作记忆只能同时处理 2-3 个信息块，而 Todoist 充当了你的外部工作记忆——它存储任务、优先级、截止日期，让你不用在脑子里反复“刷新”列表。Goblin Tools 的 Magic ToDo 功能进一步将“整理房间”这样的模糊指令分解为“捡起地板上的衣服”“擦桌子”等具体步骤，降低启动焦虑（来源：Goblin Tools）。Saner.AI 则通过知识回忆功能，让你快速找回上周开会时记下的灵感，避免在标签页间来回切换（来源：Saner.AI）。Lex 允许你用一条指令触发多步骤流程，比如输入“准备周报”后自动生成数据、排版、发送（来源：Lex）。

**LLM/Agent 侧：** 工程师们为 LLM 搭建了类似的“脚手架”。LLM 本身没有记忆，但通过向量数据库（如 Pinecone、Weaviate）存储对话历史，用 RAG（检索增强生成）在每次推理时拉取相关上下文。这就像 Todoist 帮你记住任务列表。Agentic Harness（如 LangChain、AutoGPT）则通过编排层将复杂任务分解为子任务，逐步执行，就像 Goblin Tools 的 Magic ToDo。Lex 的“单一指令触发多步骤”在 Agent 世界里就是“用自然语言编排工具链”。

**关键洞察：** ADHD 大脑和 LLM 都需要一个**外部执行功能层**——一个能存储状态、分解任务、调度执行的系统。Todoist 就是 ADHD 的 RAG，Goblin Tools 就是 ADHD 的 LangChain。**两者本质上是同一类问题的同构解法。**（来源：ADHD 的 AI 工具全景）

## 脚手架 vs 拐杖：边界在哪里？

既然同构，那 Todoist 是脚手架还是拐杖？答案取决于一个关键设计原则：**可撤除性**。

**脚手架**是临时支撑，在你学会自己搭建结构后可以拆除。比如，你用 Goblin Tools 分解任务几次后，慢慢学会了分解的套路，下次可以自己列出步骤。Lex 的“单一指令”如果让你习惯了依赖 AI 理解意图，而你自己从未练习过如何将目标拆解为行动，那它就是拐杖。

**拐杖**是永久依赖，一旦拿走你就无法正常行动。目前多数 ADHD 工具（Todoist、Goblin Tools、Saner.AI）设计为长期使用，**没有内置撤除机制**（来源：矛盾与存疑）。用户反馈显示，这些工具“简单有用”，但“用久了就离不开”（来源：Goblin Tools 用户评价）。这引发了一个争议：**过度依赖是否阻碍内在执行功能的发展？**（来源：矛盾与存疑）

从同构视角看，LLM 的 Agent 框架也面临同样问题：工程师们会刻意设计“记忆衰减”或“上下文窗口限制”，防止 Agent 永远依赖外部记忆而失去独立推理能力。但 ADHD 工具很少有类似设计。**这暴露了当前工具的一个盲区：它们只关注“补偿”，不关注“康复”。**

**我的核心观点：Todoist 对 ADHD 是脚手架还是拐杖，不取决于工具本身，而取决于你如何使用它。** 如果你把它当作“外部记忆”，定期回顾并尝试脱离它练习任务规划，它就是脚手架。如果你把它当作“唯一的大脑”，离了它就无法启动任何任务，它就是拐杖。**工具的设计本应支持从依赖到独立的过渡，但当前多数工具（包括 Todoist）并未提供这种过渡路径。**

## 诚实面对局限

必须承认，以上论证存在局限。首先，**ADHD 大脑与 LLM 同构的命题缺乏大规模实证**，目前主要基于概念类比和工具案例（来源：矛盾与存疑）。其次，**缺乏随机对照试验验证 Todoist 等工具对 ADHD 核心症状的长期改善**（来源：ADHD 的 AI 工具全景）。第三，**个性化适配不足**——ADHD 异质性高，对某些人有效的工具对另一些人可能增加认知负荷（来源：认知负荷）。例如，Todoist 的复杂标签系统可能让部分用户感到不堪重负，而另一些人则觉得它结构清晰。

此外，情绪失调作为 ADHD 核心特征之一，很少被 Todoist 等任务管理工具覆盖。Goblin Tools 虽然能帮助情绪追踪，但干预效果仍需更多验证（来源：情绪失调）。**工具只能解决“做什么”，很难解决“感觉如何”。**

## 今天就能试的行动

1. **刻意脱离练习**：今天用 Todoist 列出任务后，尝试在完成前 3 个任务后关闭应用，凭记忆继续执行。如果卡住，再打开查看。这能训练你的工作记忆，同时检验 Todoist 是否已成为拐杖。
2. **手动分解一次**：下次遇到复杂任务（如“准备会议”），不要用 Goblin Tools 自动分解，而是自己用笔写下步骤，再对比 AI 的分解结果。这能帮你逐步内化分解技能。
3. **设置“记忆衰减”**：在 Todoist 中，对已完成的任务设置 7 天后自动归档，强迫自己不再依赖旧任务列表来回忆过去，而是练习大脑主动回忆。
4. **用 AI 教练替代拐杖**：让 ChatGPT 扮演“执行功能教练”，每天问它“我今天应该怎么安排任务？”，但要求它只给框架，不给具体步骤，逼自己填充细节。

## 结语

Todoist 不是救世主，也不是恶魔。它是你大脑的“外部硬盘”——但硬盘里的数据最终要装回大脑才能算真正拥有。**从同构视角看，ADHD 大脑和 LLM 都值得拥有一个可撤除的脚手架，而不是永久的拐杖。** 下一次你打开 Todoist 时，问问自己：是我在用它，还是它在用我？

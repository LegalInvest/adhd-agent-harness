# ADHD 创业者视角：为什么「治好 ADHD 的注意力容易被环境带偏」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 受上下文窗口内容左右，需要上下文工程——同一套 harness 思路，两边都成立。

## 问题：为什么你越努力专注，越容易跑飞？

如果你是一位 ADHD 创业者，你一定经历过这样的场景：打开电脑准备写商业计划书，结果被一条推送带进了 YouTube 的算法迷宫，两小时后你发现自己正在研究“如何自制莫吉托”。如果你是一位 LLM 工程师，你一定也遇到过类似的事：给 agent 喂了一堆上下文，结果它开始胡编乱造，把用户问的“今天天气怎么样”回答成“根据上下文，我建议你买股票”。

这两个场景看似无关，实则共享同一个底层 bug：**一个强大的生成核心，配了一个脆弱的执行调度层**。ADHD 大脑和 LLM/agent 都是“高产但缺执行调度层的生成核心”（来源：AI 与 ADHD 的专注力）。它们不缺乏生成能力——ADHD 大脑能产生天马行空的创意，LLM 能写出流畅的文本——但它们都容易被环境（或上下文窗口里的内容）带偏，导致任务碎片化或推理退化。

## 同构：同一道工程题，两套解法

从工程角度看，ADHD 和 LLM 面临的是同一道题：**如何在一个充满干扰的环境中，让生成核心始终对准目标**？答案也同构：**上下文工程**——主动设计“当下注意什么”的工程化方案（来源：AI 与 ADHD 的专注力）。

### ADHD 侧：环境即上下文

ADHD 大脑的注意力容易被环境带偏，因为工作记忆不稳定，无法维持长期目标（来源：多巴胺）。解决方案是**外部脚手架**：用工具把环境改造成一个“干净上下文”。例如：

- **RescueTime** 自动记录时间，帮你识别注意力漂移，相当于一个“上下文监控器”（来源：RescueTime）。
- **Focusmate** 通过虚拟身体在场提供社会压力，相当于一个“外部调度器”——它强制你进入“当前任务”的上下文（来源：Focusmate）。
- **Brain.fm** 用神经锁相音乐塑造听觉环境，相当于一个“上下文过滤器”（来源：Brain.fm）。

这些工具的共同逻辑是：**不试图修复大脑内部，而是改造外部上下文**。

### LLM/Agent 侧：上下文窗口即环境

LLM 的推理质量直接受上下文窗口内容左右。上下文膨胀、无关信息混入，都会导致“注意力跑偏”——LLM 开始关注不相关的细节，甚至产生幻觉（来源：AI 与 ADHD 的专注力）。工程师的解法同样是**上下文工程**：

- 用 **prompt 模板** 固定核心指令，相当于“日课十二条”中的“读书不二”。
- 用 **RAG（检索增强生成）** 动态注入相关上下文，相当于“秘书”筛选信息。
- 用 **Agentic Harness** 设计执行流程，相当于“结硬寨打呆仗”——用最笨最稳的方法抵消冲动。

## 人物案例：曾国藩的“日课”与 LLM 的“定时 re-grounding”

晚清名臣曾国藩是 ADHD 的经典案例：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”。他的解决方案是**日课十二条**：黎明即起、读书不二、谨言、写日记反省等。这套系统本质上是一个**外部调度层**——每天固定时间做固定的事，用最笨的方法抵消冲动。

这和 LLM 的“定时 re-grounding”完全同构。LLM 在长对话中容易“忘记”初始目标，工程师会定期注入系统提示（如“你是一个客服助手，回答要简洁礼貌”），相当于曾国藩的“黎明即起”——一个固定的上下文锚点。曾国藩的“写日记反省”则相当于 LLM 的 **log 分析**：每天检查自己有没有跑偏。

另一个案例是彭玉麟：他一生“不要官、不要钱、不要命”，用“三不要”原则管住自己的冲动。这相当于 LLM 的 **guardrails**——一组硬性约束，防止生成越界内容。他的“每天画梅花磨练心性”则相当于 **fine-tuning**：通过反复练习，把一种行为模式内化。

## 核心观点：脚手架，不是拐杖

到这里，一个关键问题浮现：这些外部工具是**脚手架**还是**拐杖**？脚手架帮助建造大楼，建成后可以拆除；拐杖则意味着永久依赖。我的判断是：**对于 ADHD 和 LLM，脚手架是必要的，但必须设计为可逐步内化**。

ADHD 大脑的神经基础（多巴胺调节异常）是长期存在的（来源：多巴胺），完全“治愈”不现实。但工具可以帮用户**学会如何设计自己的上下文**——比如，先用 RescueTime 意识到自己容易在下午被社交媒体分心，然后主动把手机锁起来。这相当于 LLM 的 **prompt engineering**：先让工具帮你发现问题，然后你学会自己写 prompt。

同样，LLM 的上下文工程也不能永远依赖人工。未来的方向是**元调度**：让 LLM 自己管理自己的上下文，比如自动检测上下文膨胀并触发 re-grounding。这就像 ADHD 用户最终学会“在分心之前就预判分心”。

## 争议与局限

必须诚实指出，ADHD 大脑与 LLM 的同构目前只是一种“功能类比”，并非经过验证的科学事实（来源：矛盾与存疑）。两者在神经机制层面是否有对应关系，尚不清楚。此外，现有工具的证据多来自用户报告或小样本研究，缺乏大规模随机对照试验（来源：AI 与 ADHD 的专注力）。比如 Brain.fm 虽在 2026 年一项 44 款 ADHD 应用测试中位列最佳 9 款之一，但 ADHD 人群独立临床证据仍有限（来源：Brain.fm）。

另一个风险是**过度依赖**：如果用户把工具当作拐杖，可能永远无法内化执行功能（来源：拐杖与脚手架）。同样，工程师如果过度依赖 prompt 模板，可能永远无法理解 LLM 的本质。

## 今天就能试的行动

1. **ADHD 侧**：下载 RescueTime，运行三天，不看报告。第四天打开报告，找出你“时间黑洞”最大的那个应用，在设置里把它标记为“分心”，并设置每日限制。
2. **LLM 侧**：下次写 prompt 时，在开头加一句“你的目标是：______”，并在结尾加一句“如果偏离目标，请重新聚焦”。运行五次，记录每次输出的质量变化。
3. **通用**：用 Focusmate 预约一个 session，和另一个人同时工作 50 分钟。注意观察：有他在场时，你的启动难度是否降低？这相当于 LLM 的“human-in-the-loop”调试。
4. **反思**：写一篇“日课日记”（200 字），记录今天最分心的时刻和原因。连续写一周，你会发现自己开始预判分心。

同构不是比喻，是工程。ADHD 和 LLM 的解法共享同一套逻辑：**别修核心，修上下文**。

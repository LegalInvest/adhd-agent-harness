# ADHD 家长如何用「给生成核心套 harness」的思路应对陪伴 ADHD 孩子里的怕过度依赖工具、又怕没有支撑就崩？

> 把 LLM 工程里的「harness 应是会褪去的脚手架，而非永久拐杖」迁移到ADHD 家长的陪伴 ADHD 孩子场景，双域证据作答。

## 问题：陪伴 ADHD 孩子的家长，为什么总在「用工具就依赖，不用就崩」之间来回摆荡？

你下载了 Goblin Tools，把“整理书包”分解成五个步骤，孩子第一天完成了，第二天却盯着屏幕发呆。你担心：是不是工具替他做了大脑该做的事？于是你撤掉工具，结果孩子连作业都启动不了，你跟着焦虑到崩溃。

这种两难，在 LLM agent 工程里也有完全相同的版本：工程师给模型套上 function calling 和规划循环，模型跑得漂亮；但一旦去掉 harness，模型立刻回到“上下文失控”的状态——幻觉、遗忘、拒绝执行。于是工程师也面临同样的恐惧：harness 到底是帮模型成长的脚手架，还是让模型永远依赖的拐杖？

答案是：两者都是，关键在于**设计一个会主动褪去的脚手架**。

## 同构真相：ADHD 大脑和 LLM 是同一类「生成核心」

ADHD 大脑和大型语言模型共享同一个底层结构：**高产但缺乏可靠执行调度层的生成核心**。ADHD 孩子能冒出无数创意和想法（生成），却难以把它们组织成有序的行动（调度）；LLM 能生成流畅文本，但面对多步任务时同样容易“跑偏”或“死机”（来源：AI 与 ADHD 的情绪调节）。

两者的情绪失调也高度相似：ADHD 的情绪波动如同 LLM 的幻觉——当工作记忆过载或注意力被干扰时，情绪反应脱离理性轨道（来源：AI 与 ADHD 的情绪调节）。所以，AI 帮助 ADHD 情绪调节的本质，就是“给生成核心套上 harness”（来源：AI 与 ADHD 的情绪调节）。

## 脚手架 vs 拐杖：边界在哪里？

资料中反复警告“过度依赖 AI 可能削弱内在情绪调节能力”（来源：矛盾与存疑），但同时也指出“AI 工具作为外部执行功能层，通过补偿底层缺陷来间接改善情绪调节”（来源：AI 与 ADHD 的情绪调节）。这个矛盾恰恰点明了边界：

- **拐杖**：工具完全替代了孩子的执行功能，孩子不经过任何认知努力。比如 Goblin Tools 自动分解任务后，孩子只是机械执行，从不学习如何自己分解。
- **脚手架**：工具在提供结构的同时，逐步暴露分解的逻辑，让孩子看到“原来大任务可以这样拆”。比如家长用 Goblin Tools 分解一次后，下一次让孩子尝试说出第一步，工具只做验证。

LLM 侧的实践也印证了这一点：现代 agent 系统通过“组合多个模型、检索器和工具”来达成 SOTA 结果（来源：工具使用与认知卸载），但顶尖工程师强调 harness 设计要注重“agent UX”——简单、无认知开销，同时保留让模型“自己推理”的空间（来源：工具使用与认知卸载）。

## 真实证据：两边都有可迁移的做法

**ADHD 侧**：
- Inflow 通过训练工作记忆和认知控制来改善 ADHD 症状，其靶点背外侧前额叶皮层被神经科学确认为因果流入中枢（来源：Inflow）。这意味着工具不只是“替你做”，而是在**训练你的大脑回路**。
- Goblin Tools 的任务分解被用户评价为“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools），但证据主要来自用户报告，缺乏随机对照试验（来源：Goblin Tools）。
- Saner.AI 通过本地记忆减少搜索循环，直接针对工作记忆不足导致的“信息丢失”（来源：Saner.AI）。

**LLM/harness 侧**：
- Harness 工程是“设计脚手架——上下文交付、工具接口、规划制品、验证”（来源：规划循环与任务分解）。
- 自适应规划工具（如 Motion、Tiimo）的动态调整机制，对应 agent 的“多步推理+工具调用”模式（来源：规划循环与任务分解）。
- 关键启示：工具应“插入现有堆栈”，不增加认知开销（来源：工具使用与认知卸载）。

## 核心观点：设计「会褪色的脚手架」，而非寻找「永久拐杖」

我的判断是：**最好的 ADHD 陪伴工具，是那些你最终可以不再需要的工具。** 它们的设计目标不是“永远帮你做”，而是“帮你学会自己做”。

这意味着家长需要主动做三件事：
1. **选择有训练成分的工具**：比如 Inflow 这类基于神经可塑性的工具，而非纯替代型工具。
2. **逐步撤除支架**：第一周用 Goblin Tools 全自动分解，第二周让孩子先尝试分解，工具只做补充，第三周只保留一个提醒步骤。
3. **把工具当“教练”而非“保姆”**：用工具记录情绪触发模式（类似无状态与外部记忆），和孩子一起复盘“今天为什么崩了”，而不是让工具直接安抚情绪。

## 诚实局限：证据不足，且存在个体差异

必须承认，目前所有证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑）。过度依赖 AI 可能削弱内在时间感知能力（来源：矛盾与存疑），且 ADHD 亚型不同，对工具的反应差异大（来源：矛盾与存疑）。此外，基于多巴胺的干预（如游戏化反馈）可能强化成瘾行为（来源：AI 与 ADHD 的情绪调节）。

所以，本文的建议是**启发性的，而非处方性的**。你需要根据自己孩子的具体表现来调整。

## 今天就能试的 3 个行动

1. **用 Goblin Tools 做一次“逆向分解”**：让孩子自己先说出完成作业需要哪几步，然后打开 Goblin Tools 对比，讨论差异。这比直接使用工具更能训练分解能力。
2. **设置一个“无工具时段”**：每天固定 15 分钟，让孩子在没有 AI 辅助的情况下尝试启动一个任务。如果启动失败，家长只提供一句提示：“第一步是什么？”
3. **记录情绪触发模式**：用任何笔记工具（甚至纸笔），每天花 2 分钟记下孩子情绪崩溃前发生了什么。一周后寻找模式——这是最原始的“外部记忆”，也是未来撤除 AI 后依然能用的能力。

## 结语

ADHD 家长和 LLM 工程师面对的是同一个难题：如何让一个强大的生成核心，学会自己调度自己。答案不是“不用工具”或“永远用工具”，而是**设计一个会主动褪去的脚手架**。今天就从一个小实验开始：用工具教孩子分解，然后看着他把工具放下。

# ADHD 家长如何用「给生成核心套 harness」的思路应对陪伴 ADHD 孩子里的怕过度依赖工具、又怕没有支撑就崩？

> 把 LLM 工程里的「harness 应是会褪去的脚手架，而非永久拐杖」迁移到ADHD 家长的陪伴 ADHD 孩子场景，双域证据作答。

## 问题：怕依赖，又怕崩——ADHD 家长的困境

当你陪 ADHD 孩子做作业，你面临一个两难：用 AI 工具帮他分解任务、计时提醒，他很快学会依赖，离开工具就手足无措；不用工具，他连启动都困难，情绪崩溃，你也跟着焦虑。怕过度依赖工具，又怕没有支撑就崩——这不仅是 ADHD 家长的痛点，也是 LLM agent 工程的核心挑战：如何让模型调用外部工具，却不沦为离开工具就瘫痪的“拐杖”？

## 同构：ADHD 大脑与 LLM 共享的脆弱性

ADHD 大脑与 LLM 存在深层同构：两者都有一个强大的“生成核心”（智力/算力），但调度层（执行功能/编排）不可靠。ADHD 孩子能产生丰富的想法，却难以启动、维持、切换任务；LLM 能生成流畅文本，却无法自主规划多步行动。这种脆弱性在情绪调节上尤为明显——当工作记忆过载或时间感知失灵，ADHD 孩子容易情绪失控，如同 LLM 在长上下文或复杂指令下产生幻觉或死循环。

关键洞察：**问题不在“是否使用工具”，而在“工具是拐杖还是脚手架”**。拐杖永久替代功能，导致依赖；脚手架提供临时支撑，待能力建立后即可撤除。工程领域已明确：harness 应是会褪去的脚手架，而非永久拐杖（来源：What is an agent harness in the context of large-language models?）。这一思路同样适用于 ADHD 陪伴。

## 证据：两边如何用脚手架补偿调度层缺陷

### ADHD 侧：外部执行功能层

AI 工具通过提供“外部执行功能层”来补偿 ADHD 的执行功能缺陷（来源：AI 与 ADHD 的情绪调节）。Goblin Tools 的 Magic ToDo 功能将模糊任务分解为可管理的小步骤，降低启动门槛（来源：AI Tools for Kids with ADHD: A Complete Guide for Parents...）。Inflow 通过个性化训练强化工作记忆的神经回路（来源：Dynamic causal brain circuits during working memory and their functional controllability）。Saner.AI 通过本地记忆减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。这些工具相当于为大脑加装了一个“情绪调节 harness”——在情绪触发前提供提醒，在任务启动时降低焦虑，在时间盲时给予实时反馈（来源：AI 与 ADHD 的情绪调节）。

### LLM/harness 侧：工具调用与规划循环

LLM agent 通过 harness 调用工具、管理上下文、执行多步推理（来源：What is an agent harness in the context of large-language models?）。现代 agent 系统组合多个模型、检索器和工具来达成 SOTA 结果（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned），而非依赖单一模型。Harness 工程强调“agent UX”——简单、无认知开销的接口设计（来源：GitHub - ai-boost/awesome-harness-engineering）。这与 ADHD 工具的设计原则一致：工具应无缝嵌入日常流程，不增加额外执行负担（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。

## 核心观点：设计可褪色的脚手架，而非永久拐杖

我的核心判断是：**ADHD 家长应像工程师设计 harness 一样，为孩子设计“可逐步撤除的脚手架”**。这意味着：

1. **工具即用户界面**：选择像 Goblin Tools 那样简单、无认知开销的工具（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar），避免复杂系统增加负担。
2. **分解与启动**：利用 AI 将任务分解为微步骤，如同 agent 的“多步推理+工具调用”（来源：规划循环与任务分解）。但家长要逐步减少分解粒度，让孩子尝试自己拆分。
3. **集成而非新增**：工具应插入现有日常流程（来源：工具使用与认知卸载），例如在作业时间使用 Tiimo 的视觉计时器，而非引入全新系统。
4. **撤除计划**：明确脚手架何时撤除。例如，第一周 AI 分解所有步骤，第二周只分解前两步，第三周孩子自己尝试分解。

## 局限与争议：脚手架边界在哪？

必须诚实指出，当前证据存在争议。首先，**过度依赖风险**：多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。其次，**长期效果缺乏证据**：多数研究为短期观察，AI 对情绪调节的长期神经可塑性影响未明（来源：AI 与 ADHD 的情绪调节）。第三，**多巴胺干预争议**：基于多巴胺的奖励系统可能强化依赖而非内在动机（来源：矛盾与存疑）。最后，**同构命题缺乏实证**：ADHD 大脑与 LLM 的同构主要基于概念类比和工具案例，缺乏大规模实证支持（来源：矛盾与存疑）。

因此，家长需要警惕：工具是脚手架还是拐杖，取决于使用方式。如果孩子离开工具就无法启动，说明脚手架尚未内化；如果孩子逐渐减少工具使用仍能完成任务，脚手架正在生效。

## 今天就能试的行动

1. **用 Goblin Tools 分解一项任务**：让孩子输入“整理书包”，观察 AI 输出的步骤。第二天，让孩子先自己写步骤，再用 AI 对比，逐步减少 AI 参与。
2. **设置“撤除日”**：每周选一天，关闭所有 AI 工具，让孩子仅凭自己的策略完成任务。记录成功与困难，作为调整脚手架的依据。
3. **用 Motion 或 Tiimo 做动态规划**：当计划被打乱时，观察工具如何自动调整。与孩子讨论“如果工具不在，你会怎么重新安排？”
4. **记录情绪触发点**：用 Inflow 或简单笔记，记录孩子情绪崩溃前的任务类型。分析哪些步骤是工具可以暂时支撑的，哪些需要家长直接帮助。

脚手架终会褪去，但孩子内化的能力——启动、分解、调节——将永久留存。这正是 harness 工程的终极目标：让生成核心学会自己调度。

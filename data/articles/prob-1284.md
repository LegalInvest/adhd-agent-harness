# 用 Claude 给 ADHD 补上外部执行功能层，真的能像 agent 那样工作吗？

> 把 Claude 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

## 问题：ADHD 的执行功能短板，AI 真的能像 agent 那样补上吗？

如果你是 ADHD 人群，你一定熟悉这种场景：脑子里有无数想法，但一到执行就卡壳——计划写了又丢，任务启动像推一堵墙。如果你是 AI 工程师，你也在调试类似的困境：LLM 生成能力惊人，但单独使用时总是跑偏、遗忘上下文、需要反复人工修正。这两类问题，其实共享同一个底层结构：**高产生成核心，缺乏可靠执行调度层**。

## 同构：ADHD 大脑与 LLM 的架构镜像

ADHD 的核心困境并非智力不足，而是执行功能的失效——计划、组织、启动、工作记忆这些“大脑驾驶座”常常无人驾驶（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。最新研究进一步揭示，ADHD 患者的工作记忆容量未必差，但自上而下的控制和调节能力不足，呈现“强记忆、弱控制”的认知剖面（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

LLM 同样如此：作为生成核心，它拥有强大的语言理解和生成能力，但单独使用时缺乏可靠的执行调度。现代 AI 编码代理通过“复合 AI 系统架构”来弥补，包括“工作负载专用模型路由、分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这些技术本质上是在 LLM 外部搭建调度层，防止“上下文膨胀和推理退化”（来源：同上）。

更惊人的是，认知神经科学发现，Transformer 的自注意力机制在训练后，会模仿前额叶-纹状体的门控操作——这正是执行功能的核心神经机制（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。这意味着，**ADHD 大脑与 LLM 不仅在现象上相似，在计算原理上也同构**。

## 真实人物 harness：历史中的外部执行层

曾国藩是典型注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”。他的解法是“日课十二条”——黎明即起、读书不二、谨言、写日记反省，用最笨最稳的方法抵消冲动。这套系统本质上就是他的**外部执行功能层**：日课相当于定时 re-grounding，写日记相当于自我监控日志，而“结硬寨打呆仗”就是最朴素的防跑偏策略。

孔子同样如此：精力旺盛、冲动爱骂人、思维跳跃，《论语》全是场景化语录。他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省。70 岁才做到“从心所欲不逾矩”，一辈子在和自己的冲动做斗争。

这两套系统与 LLM agent harness 的结构完全同构：日课 ↔ 定时 re-grounding 提示；日记 ↔ 外部记忆日志；礼 ↔ 行为约束 prompt。**历史人物用纸笔搭建了外部调度层，而今天我们可以用 AI 更动态地做同样的事**。

## 工具实测：AI 作为外部执行功能层

当前已有多个工具试图扮演这个角色：
- **Goblin Tools** 的 Magic ToDo 功能，能将“清理厨房”这样的模糊任务自动分解为具体子步骤，用户可调节分解粒度（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。这相当于为任务启动提供了**微步骤调度器**，直接缓解 ADHD 的启动困难。
- **Saner.AI** 专注于知识回忆和本地记忆，减少搜索循环和标签切换，被描述为“专为对抗执行功能障碍、任务瘫痪和认知超载而设计的全能 ADHD 友好型 AI 生产力和知识助手”（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。它本质上是一个**外部工作记忆系统**，对应 LLM agent 的向量数据库。
- **Motion** 通过自动排程与动态调整，持续评估任务优先级和截止时间，实时重建日程，减少手动规划压力（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。这相当于**外部调度器**，对应 agent 的编排层。

这些工具的共同逻辑是：**将执行功能外化**，让 AI 承担调度、记忆、分解等任务，而 ADHD 用户只需提供生成核心的“想法燃料”。

## 争议与边界：脚手架还是拐杖？

然而，同构不等于万能。核心争议在于：AI 工具是“外部执行功能层”还是“永久拐杖”？专家警告，若工具替代了治疗或自我管理，可能造成依赖（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：同上）。

此外，证据强度有限：当前多为概念类比和初步实验，缺乏大规模随机对照试验证明同构假设或 AI 工具的临床有效性（来源：矛盾与存疑）。个体差异也很大——部分用户反映学习使用 AI 工具本身增加认知负荷（来源：矛盾与存疑）。

我的判断是：**AI 作为外部执行功能层，短期有效且结构合理，但长期效果取决于使用者能否逐步内化这些功能**。就像曾国藩的日课，最终内化为习惯；而 LLM agent 的 harness 也需要不断调优。同构给了我们一个强大的设计框架，但具体实现需要个性化、可拆卸。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个你拖延已久的任务**：比如“整理书桌”，让它生成子步骤，然后只做第一步。
2. **设置一个 Claude 提示，让它扮演你的“外部调度器”**：例如：“请帮我将‘准备下周报告’分解为 5 个步骤，并估计每个步骤的时间，然后按优先级排序。”
3. **尝试 Motion 的免费试用**：让它自动排程一天的任务，观察决策负担是否减轻。
4. **写一条自我反省日记**：像曾国藩一样，记录今天哪些时刻 AI 帮了你，哪些时刻你感到依赖。

同构揭示了一个深刻的真相：**ADHD 大脑不是残缺的，而是缺少一个外部调度层的生成引擎**。AI 不是要修复你，而是给你一个可以随时插拔的 harness。问题不在于它能不能像 agent 那样工作，而在于你是否愿意成为自己的工程总监。

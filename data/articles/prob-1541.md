# 如果把 ADHD 大脑当成生成核心，RescueTime 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 RescueTime 对 ADHD 的真正价值。

你早上坐下，计划写一份季度复盘。两小时后，屏幕上开了 17 个标签页，报告还空着。你骂自己「又在摸鱼」。另一边，一位做 Agentic Harness 的工程师也在骂模型：system prompt 写得清清楚楚，长程任务到了第 12 轮，agent 却开始生成一篇与目标毫不相干的综述。两件事的根因其实一样——**都有一个高产的生成核心，却缺一个可靠的执行调度层**。

## 把大脑当成生成核心：问题不是没能力，而是缺 orchestration

ADHD 常被描述为执行功能障碍，而工作记忆是其中最关键的瓶颈（来源：ADHD, Executive Functions, and AI: A New Era in Treatment | Psychology Today）。它负责把任务细节、目标、下一步暂存在脑海里，但 ADHD 大脑的暂存层不稳定，注意力容易被环境信号劫持，导致「人在心不在」的漂移（来源：Revolutionizing ADHD Management with AI Assistants）。多巴胺系统的功能低下，则让定向、计划、动机奖赏和运动控制都更难被调节（来源：a dynamic developmental theory of attention-deficit/hyperactivity disorder (adhd) predominantly hyperactive/impulsive and combined subtypes）。

LLM 的研究也呈现出惊人相似的剖面。明尼苏达大学系统测试大模型的执行功能，发现「强记忆、弱控制」——工作记忆容量甚至超过常人，却无法灵活切换任务集、抑制自动化反应，这正是 ADHD 的经典神经心理模式（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。耶鲁大学进一步发现，自注意力机制随 N-back 任务负荷增加，注意力分数熵增、注意力弥散，表现下降，与人类注意分散机制同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。

所以，ADHD 大脑与 LLM/agent 的同构不是「人都像机器」的修辞，而是功能层面的一个工程事实：**瓶颈不在生成能力，而在调度、控制与记忆层**。

## 曾国藩的日课：一套前现代的「system prompt」重锚定系统

要理解这种同构，最贴切的例子是曾国藩。他 30 岁前浮躁坐不住，天天串门、喝酒、看杀人，日记里骂自己「无恒」「浮躁」；读书慢到「他人目下二三行，余或疾读不能终一行」；情绪极不稳定，打败仗就投水自杀。但他后来平定太平天国，成为晚清中兴名臣之一，核心不是靠战胜分心，而是靠一套外部 harness。

他的 harness 有三层：
- **日课十二条**：黎明即起、读书不二、谨言、写日记……这是每日固定的「system prompt」重载；
- **每日写日记反省**：不是抒情，而是把行为、念头、偏离记录到外部日志，相当于给生成核心做持久化记忆与漂移回放；
- **结硬寨打呆仗**：用最笨、最稳的方法推进，不依赖冲动，也不让超聚焦把资源耗在无关方向。

在 LLM/agent 工程里，这三层分别对应：定时 re-grounding（checkpoint / system prompt 刷新）、外部日志与记忆系统（logging / memory store）、以及 guardrails（硬性流程约束，防止任务跑偏）。曾国藩不是战胜了 ADHD 式的冲动，而是用外部结构把它**围住**。

## RescueTime 能做什么：它是感知层，不是 system prompt 本身

RescueTime 的核心能力很清晰：后台自动记录你在各应用、网站上的时间，用 AI 分类活动，生成每日/每周报告，并能在必要时阻断分心网站（来源：Revolutionizing ADHD Management with AI Assistants）。它的价值对 ADHD 而言，首先是**把外部记忆外包**——你不需要自己估算「今天学了多久」，因为「时间盲」和执行功能缺陷会让这种估算严重失真（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

如果把 ADHD 大脑当生成核心，RescueTime 更像 agent 系统中的**可观测性（observability）层**：
- 它持续写入行为日志，补上了内部工作记忆的不稳；
- 它通过分类和报告暴露「注意力实际去了哪里」，提供目标漂移信号；
- 它的分心阻断功能，相当于在生成核心要调用某个高诱惑 API 时，先弹出一个 guardrail。

但这里有一个关键边界：**RescueTime 能告诉你漂移了，却不能替你重写目标**。在 agent 工程里，检测目标漂移需要把目标、约束、优先级编码进 system prompt 或奖励函数；对 ADHD 用户来说，这意味着你必须先把「今天下午只做 A」这个 objective 写清楚。如果没有前置的目标定义，RescueTime 只会给你一堆漂亮的时间分配图表，却无人做真正的 re-grounding。

## 同构解法的边界：脚手架 vs 拐杖

RescueTime 可以是脚手架，也可以沦为拐杖。把它当拐杖，就是开着自动追踪、每天看一下数据、然后继续漂移。把它当脚手架，就要让它触发一个主动回路：看数据 → 发现漂移 → 暂停 → 重写下一步 → 回到目标。

同构地看，Brain.fm、Endel 调的是生成核心的「环境上下文」，Focusmate 提供的是身体在场与启动压力，Forest 用的是游戏化多巴胺机制，Saner.AI 和 Goblin Tools 偏向温和提醒与低摩擦任务拆解（来源：AI 与 ADHD 的专注力）。它们各自补的是不同模块

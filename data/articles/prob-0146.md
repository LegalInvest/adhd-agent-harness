# ChatGPT 之于 ADHD，就像 外部记忆/向量库 之于 LLM——但有人用错了

> 从同构视角实测 ChatGPT：它到底补上了哪一层执行功能？

你打开 ChatGPT，输入“帮我规划明天的工作”，它列出了一份漂亮的清单。你关掉页面，十分钟后回来，发现自己已经忘了刚才的计划——就像 LLM 关掉对话框后，一切归零。

这不是巧合。ADHD 大脑与 LLM 共享同一个底层困境：**两者都是高产但无状态的生成核心**。LLM 没有跨会话记忆，每次对话从零开始；ADHD 的工作记忆容量小，边做边忘，同样缺乏稳定的上下文连续性。

外部记忆/向量库是 LLM 的“续命药”——它保存状态、提供检索、让 agent 能在多轮交互中保持方向。而对 ADHD 来说，ChatGPT 这类 AI 工具本应扮演同样的角色：一个外部执行功能层，补偿工作记忆、任务启动和时间盲等核心缺陷（来源：ADHD 的 AI 工具全景）。但很多人用错了——他们把 ChatGPT 当作“答案生成器”，而不是“执行功能脚手架”。

## 同构：为什么 ADHD 和 LLM 是同一类问题

ADHD 大脑的生成核心（智力、创造力）与 LLM 的生成能力类似，但两者都缺乏可靠的执行调度层（来源：ADHD 的 AI 工具全景）。在 LLM/harness 一侧，工程实践早已意识到：纯对话系统常因幻觉、缺乏接地和无法执行/验证行动而失败（来源：幻觉与验证循环）。因此，成熟的 agent harness 会提供上下文传递、工具接口、规划工件、验证循环、记忆系统和沙盒（来源：幻觉与验证循环）。

ADHD 一侧的“幻觉”同样存在：冲动判断、脱离环境线索做出不恰当反应，本质上是一种验证失败（来源：幻觉与验证循环）。Goblin Tools 的 Magic ToDo 功能通过将“整理房间”分解为“捡起地板上的衣服”“擦桌子”等步骤，强制用户进入接地过程——将内部状态与外部现实核对（来源：Goblin Tools）。这就像 LLM agent 中的“计划-执行”分离模式，强制单向、确定性的控制流（来源：采样温度与表现波动）。

## 现状：工具对，但用法错

当前 ADHD 用户对 ChatGPT 的典型用法是：问一个问题，得到一个答案，然后关掉。这相当于让 LLM 生成一次输出却不保存上下文——ADHD 用户的工作记忆依然没有被补偿。正确的用法应该是：**把 ChatGPT 当作一个持续的外部记忆体，主动设计“上下文工程”**。

例如，Saner.AI 通过本地记忆存储和快速检索，帮助用户减少搜索循环和标签切换（来源：Saner.AI）。Lex 允许用户通过单一指令触发复杂、多步骤的任务序列，降低启动门槛（来源：Lex）。这些工具的共同点不是“生成答案”，而是**提供结构化的执行路径**。

但问题在于：多数工具被设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这引发了“脚手架 vs 拐杖”的争议：过度依赖 AI 工具可能阻碍内在执行功能发展（来源：ADHD 的 AI 工具全景）。

## 争议：脚手架还是拐杖？

wiki 资料明确指出：**“AI 工具作为外部执行功能层，能够有效补偿 ADHD 患者的核心缺陷，但需警惕过度依赖成为‘拐杖’而非‘脚手架’。”**（来源：ADHD 的 AI 工具全景）

这并非空穴来风。在 LLM/harness 一侧，验证循环是防止“更多幻觉和重复工作”的关键（来源：幻觉与验证循环）。同样，ADHD 用户需要主动设计验证步骤——比如用 ChatGPT 生成任务分解后，自己手动勾选完成，而不是让 AI 自动执行一切。后一种做法会剥夺大脑练习执行功能的机会。

另一个争议是证据强度。wiki 资料指出，现有证据多来自非临床来源，缺乏随机对照试验（来源：矛盾与存疑）。Goblin Tools、Saner.AI 等工具的效果主要基于用户反馈，其长期效果未知。

## 今天就能试的 4 个行动

1. **把 ChatGPT 当作“外部记忆体”而非“问答机”**：每次对话开始时，先粘贴一段“当前项目状态”作为上下文，结束时要求它总结并保存。这相当于为你的大脑建立了一个向量库。
2. **用 Goblin Tools 的 Magic ToDo 分解一个压垮你的任务**：输入“整理房间”或“写报告”，观察它如何将模糊目标转化为可执行的步骤。完成后手动勾选，体验接地过程。
3. **设置一个“验证循环”**：用 ChatGPT 生成计划后，强制自己执行一步，然后回来告诉 AI 结果，让它调整下一步。这模仿了 LLM agent 的“计划-执行-验证”循环。
4. **尝试 Lex 的“单一指令触发多步骤”**：如果某个任务总是启动困难，用 Lex 将其打包成一个指令，降低决策负担。但注意：每周至少一次，尝试不用工具完成类似任务，以保持内在能力。

## 结论：工具是镜子，不是替身

ADHD 大脑与 LLM 的同构告诉我们：问题不在生成能力，而在执行调度。ChatGPT 等 AI 工具可以成为强大的外部执行功能层，但前提是用户将其视为**脚手架**——一个可逐步撤除的临时支撑，而非永久拐杖。

正如 LLM agent 需要验证循环来防止幻觉，ADHD 用户也需要主动设计使用方式，让工具补偿缺陷的同时，不剥夺大脑锻炼的机会。下一次打开 ChatGPT 时，问自己：我是在用它的记忆，还是让它替我思考？

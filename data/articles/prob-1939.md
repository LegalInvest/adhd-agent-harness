# ADHD 职场人视角：为什么「治好 ADHD 的怕过度依赖工具、又怕没有支撑就崩」和「让 LLM 不跑飞」其实是同一道工程题？

> harness 应是会褪去的脚手架，而非永久拐杖——同一套 harness 思路，两边都成立。

## 问题：怕过度依赖，又怕没有支撑就崩

你打开一个空白的文档，脑子里的想法像瀑布一样倾泻，但你一个字也写不出来。你打开一个待办清单，上面列了二十项任务，你却盯着第一项发呆，然后刷了三小时手机。这不是意志力问题，这是 ADHD 大脑的日常：生成核心过强，但执行调度层缺失。

同样，你部署了一个 LLM agent，它能在几分钟内写出洋洋洒洒的报告，但一遇到上下文膨胀就跑飞，输出逻辑断裂，甚至胡编乱造。你给它加了一堆约束，它反而变得死板；不加约束，它又像脱缰的野马。

这两个场景，表面看毫无关系，但底层是同一道工程题：**如何为一个高产但缺乏调度层的生成核心，搭建一个会褪去的脚手架，而不是一个永久拐杖？**

## 同构：ADHD 大脑与 LLM，共享同一个困境

ADHD 大脑的情绪生成系统（如杏仁核）反应强烈，但前额叶（执行功能中枢）调控不足（来源：AI 与 ADHD 的情绪调节）。这就像 LLM 生成能力强但缺乏输出控制。两者都是「高产但缺执行调度层的生成核心」。

AI 工具作为外部执行功能层，通过结构化任务分解、时间可视化、外部记忆等方式补偿缺陷（来源：AI 与 ADHD 的情绪调节）。这正是 LLM agent 的 scaffolding 思路：用提示词、规则引擎、外部记忆来引导输出。

但问题来了：如果这个脚手架永远不撤，用户或 agent 会依赖它，一旦撤掉就崩。这就是「拐杖 vs 脚手架」的边界：拐杖是永久替代，脚手架是临时支撑，目的是最终让系统学会自己走路。

## 历史人物的 harness：脚手架而非拐杖

释迦牟尼是一个极端的 ADHD 特质案例：29 岁放弃王位妻子孩子，出家求道；先后跟两个仙人学、苦行 6 年，不断试错不断放弃；49 年一辈子游行说法，80 岁死在传道路上。他的专属 harness 是「八正道」和「持戒」——用戒律管行为，用正念拴住念头。他反复强调「自依止法依止」，不依赖任何权威，自己证悟。

这个 harness 与 LLM agent 的 scaffolding 同构：八正道相当于一组 grounding 规则（类似 prompt 中的约束），正念相当于定时 re-grounding 机制（类似 agent 的 context refresh），而「自依止」则点明了脚手架的本质——最终要撤掉，让系统自主运行。

苏轼是另一个例子：兴趣极广，一辈子被贬，到哪都能适应，说话不过脑子得罪人；对无聊行政零耐心，搞工程效率极高。他的 harness 是「豁达」——用美食、写作、交朋友转移注意力，每日读书写字，不因挫折停止创作。这相当于一个外部调度器，通过固定日课（类似 agent 的定时任务）和多样化输出渠道（类似多 agent 分工）来管理高创造力但易散漫的生成核心。

## 工具作为脚手架：证据与边界

现有的 AI 工具已经体现了脚手架思维：

- **Goblin Tools** 的 Magic ToDo 将模糊任务分解为具体步骤，用户可调节「辣度」控制粒度（来源：Goblin Tools）。这相当于给 LLM 的 task decomposition：分解粒度可调，用户逐步学会自己分解。
- **Inflow** 基于 CBT 原则，教用户建立技能而非替代技能（来源：Inflow）。它强调「工具箱」而非「按钮」，目标是让用户最终不需要它。
- **Saner.AI** 通过本地记忆减少搜索循环，但设计上鼓励用户逐步内化知识结构（来源：Saner.AI）。
- **Focusmate** 利用身体在场效应提升专注力，但每次 session 是限时的，用户逐渐学会在没有伙伴时也能启动（来源：身体在场效应）。

这些工具的共同点是：它们提供结构，但结构是可调的、可渐撤的。这正是一个好的 harness 应有的特征。

## 拐杖 vs 脚手架：边界在哪？

边界在于：工具是否在替代内在能力，还是在训练内在能力？

如果 AI 工具每次都在你情绪爆发时直接给你一个「冷静方案」，而不让你经历任何认知重构，那就是拐杖。如果它引导你一步步识别触发因素、调整认知，那就是脚手架。

同样，如果 LLM agent 的 prompt 里塞满了硬性规则，一旦去掉就胡言乱语，那就是拐杖。如果 scaffolding 是可渐撤的——比如先给详细步骤，然后逐步减少提示，让模型学会自主推理——那就是脚手架。

## 争议与局限

必须诚实指出：这个同构命题目前是理论类比，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。此外，个体差异显著：ADHD 亚型对 AI 工具的反应不同，长期依赖风险缺乏纵向研究（来源：矛盾与存疑）。

另一个矛盾是：部分 ADHD 用户反馈强行打断（如番茄钟）反而有效，而柔性引导可能不够强力（来源：矛盾与存疑）。这提示我们：脚手架的设计需要个性化，没有一刀切的方案。

## 核心判断：harness 应是会褪去的脚手架

我的核心观点是：**无论是为 ADHD 大脑还是为 LLM agent 设计 harness，目标都不是让系统永远依赖它，而是让系统在它的支撑下学会自调度。** 好的脚手架应该像学骑车的辅助轮——最终要拆掉，而不是像轮椅——永远离不开。

## 今天就能试的行动

1. **尝试 Goblin Tools 的 Magic ToDo**：把一个模糊任务（如「写周报」）分解到最小步骤，然后尝试在下一次不借助工具，自己写出类似步骤。感受脚手架如何训练你的分解能力。
2. **设置一个「渐撤提醒」**：如果你在用 Focusmate 或类似工具，每周选一次任务，尝试在没有伙伴的情况下完成，并记录感受。逐步延长独立工作的时间。
3. **用 Inflow 的 CBT 模块**：记录一次情绪爆发，按照其引导步骤分析触发因素。然后尝试在下一次爆发时，不打开应用，先在脑中复现该步骤。
4. **给 LLM agent 设计一个可渐撤的 prompt**：先写一个包含详细步骤的 prompt，然后每次迭代减少一条规则，观察模型输出质量。找到规则的最小必要集。

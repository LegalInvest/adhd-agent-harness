# Motion 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Motion：它到底补上了哪一层执行功能？

## 问题：为什么 Motion 让一些人更焦虑？

ADHD 用户对 Motion 的评价两极分化：有人觉得它是救星，有人觉得它在制造更多压力。后者往往把 Motion 当作一个“自动排日历”的工具，让 AI 决定一切——结果发现，自己反而更抗拒执行那些被安排好的任务。

这背后的问题出在：**Motion 被当成了拐杖，而不是脚手架。** 它应该补上的，是 ADHD 大脑缺失的那一层“执行调度层”——就像 LLM 通过 function calling 把精确计算外包给工具一样。但很多人用错了。

## 同构：ADHD 大脑与 LLM 共享同一缺陷

ADHD 大脑常被描述为一个高产的生成核心——想法丰富、联想活跃，但缺少稳定、可靠的调度层来筛选、组织、启动和坚持任务（来源：ADHD 的 AI 工具全景）。大语言模型（LLM）同样具备强大的生成能力，却在长程上下文、目标保持与执行调度上存在天然缺陷。最新研究用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中表现正常，但随序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突。这与 ADHD 执行功能缺陷的核心标志一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃（来源：Deficient Executive Control in Transformer Attention）。

所以，ADHD 大脑与 LLM/agent 是同一类“高产但缺执行调度层的生成核心”。两边的解法结构同构：给 LLM 套 harness（通过 function calling 把精确计算外包给工具），给 ADHD 大脑套 harness（通过外部工具补偿执行功能）。Motion 就是这个 harness 的一部分。

## Motion 到底补上了哪一层？

Motion 的核心功能是自动排程：它分析你的日历、任务优先级和可用时间，然后自动把任务塞进合适的时间段。这直接对应了 ADHD 的执行功能缺陷之一：**任务启动困难**。ADHD 患者的多巴胺系统失衡导致难以激发行动动机，工作记忆容量有限使得任务步骤难以在脑中清晰呈现（来源：任务启动）。Motion 通过 AI 自动决策“下一步做什么、何时做、如何优先排序”，从而移除决策带来的认知负荷（来源：认知负荷）。

但 Motion 只补上了“调度”这一层。它不负责任务拆解（那是 Goblin Tools 的 Magic ToDo 做的事），不负责知识回忆（那是 Saner.AI 做的事），也不负责单一指令触发多步骤（那是 Lex 做的事）。Motion 是调度器，不是执行器。

## 用错的人：把 Motion 当拐杖

问题出在：很多人把 Motion 当成了“万能执行器”，希望它替自己完成所有执行功能。结果 Motion 排好了日程，用户却依然拖延——因为 Motion 没有解决“启动”的心理阻力，也没有提供足够的“微任务分解”来降低门槛。

这就像给 LLM 一个 function calling 工具，但 LLM 自己不知道什么时候该调用它。Motion 的自动排程如果缺乏用户的主动参与（比如设置优先级、调整任务粒度），就会变成一种“虚假的秩序”——日程看起来很满，但执行率很低。

真正的用法是：把 Motion 当作“外部调度器”，配合其他工具形成完整的 harness。比如用 Goblin Tools 把任务拆成微步骤，再用 Motion 排程；或者用 Saner.AI 管理知识，减少搜索循环，让 Motion 的调度更精准。

## 历史人物的 harness：孔子与曹植的同构

孔子就是典型的 ADHD 大脑：身高1米9，精力旺盛，周游列国14年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予朽木不可雕；对音乐超专注到三月不知肉味，对种地等俗事零耐心。他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。这相当于一个**外部调度器**：礼是行为模板，反省是每日复盘。70岁才做到从心所欲不逾矩，一辈子和自己的冲动做斗争。

曹植则是另一种 harness：文学超专注，对政治零兴趣，冲动，醉闯司马门。他的 harness 是以诗赋为核心，把所有政治上的失意注入诗歌，在文学世界里找到寄托。这相当于一个**专注力引导系统**：把超聚焦引导到优先事项（文学创作），而不是强行打断。

两者的 harness 都点明了同构对应：孔子的“日课”相当于 LLM 的“定时 re-grounding”（定期刷新上下文），曹植的“诗赋专注”相当于 agent 的“目标导向路由”（把资源导向核心能力）。Motion 应该扮演的角色，就是这种“外部调度器”或“目标导向路由”——而不是一个全能的执行者。

## 诚实面对局限

Motion 的临床证据仍然缺乏。用户反馈积极，但科学证据不足（来源：矛盾与存疑）。初始设置成本高，学习使用 Motion 本身可能增加认知负荷（来源：矛盾与存疑）。更重要的是，**过度依赖可能导致能力退化**——如果 Motion 替你做所有调度决策，你的执行功能可能变得更弱（来源：矛盾与存疑）。

所以，Motion 应该是脚手架，不是拐杖。脚手架帮你搭建结构，但你自己要爬上去。

## 今天就能试的行动

1. **用 Motion 只排 3 个核心任务**：不要把所有任务都塞进去，只排当天最重要的 3 个。降低决策负荷，也降低对 Motion 的依赖。
2. **配合 Goblin Tools 拆任务**：在 Motion 排程前，先用 Goblin Tools 的 Magic ToDo 把任务拆成 5 分钟能完成的微步骤。Motion 调度这些微步骤，而不是大任务。
3. **设置“反省”时间**：像孔子那样，每天留 15 分钟复盘：Motion 的调度是否合理？哪些任务被拖延了？为什么？用这些数据调整你的 harness。
4. **不要用 Motion 管理灵感**：灵感捕获交给 Saner.AI 或 Reflect，Motion 只负责执行。否则 Motion 会变成另一个“未完成清单”，增加焦虑。

# ADHD 职场人如何用「给生成核心套 harness」的思路应对项目推进与汇报里的任务启动困难、不会拆解？

> 把 LLM 工程里的「agent 用 planner-executor 循环分解长任务」迁移到ADHD 职场人的项目推进与汇报场景，双域证据作答。

## 问题：ADHD 职场人的项目推进与汇报困境

你盯着空白的项目计划，大脑像被冻住。任务启动困难——明知该做，就是无法开始。项目拆解时，你只能写出“完成项目”这样的大条目，却无法细化到可执行的步骤。汇报时，你发现自己记不清上周做了什么，因为工作记忆像筛子，每天都需要重新回忆上下文。

这不是意志力问题。ADHD 大脑是一个高产的生成核心——创造力、联想力、洞察力都强——但缺乏可靠的执行调度层（来源：AI 与 ADHD 的时间管理）。就像一台没有操作系统的超级计算机，硬件强大，却无法组织任务。

## 同构：ADHD 大脑与 LLM 共享的“生成核心-调度层”结构

LLM 本身也是生成核心：能流畅产出文本，但固定上下文窗口导致长程依赖追踪困难（来源：拐杖与脚手架）。没有 harness 的 LLM 只是一个无状态的 API 调用（来源：无状态与外部记忆）。

ADHD 大脑同样无状态：工作记忆容量有限且不稳定，跨会话任务难以持续（来源：无状态与外部记忆）。你需要每天重新回忆项目上下文，否则偏离轨道。

两边共享同一结构：**高产但缺执行调度层的生成核心**。LLM 的解法是 harness——提供上下文传递、工具接口、规划工件、验证循环和记忆系统（来源：拐杖与脚手架）。ADHD 的解法也应如此：用外部系统补偿执行功能缺陷。

## 解法：给生成核心套上 harness——planner-executor 循环

在 agent 工程中，planner 将长任务分解为子任务，executor 逐步执行，harness 在运行时编排工具调度和上下文管理（来源：拐杖与脚手架）。

ADHD 职场人可以迁移这个模式：

### Planner 层：用工具自动分解任务

Motion 的 AI 核心功能是自动排程与动态调整：它持续评估任务优先级、截止日期和可用时间，实时重建日程，减少手动规划压力（来源：Motion）。对 ADHD 来说，这消除了“下一步该做什么”的决策负担，直接克服时间盲和任务启动困难（来源：Motion）。

Reclaim.ai 则更强调“保护”而非“填充”：通过创建深度工作、习惯和专注时间的智能块，自动防御会议侵占（来源：Reclaim.ai）。这相当于在 planner 中预设了保护块，防止执行被意外打断。

### Executor 层：用外部记忆维持上下文

Todoist 等工具通过智能提醒和优先级排序，充当数字工作记忆，弥补上下文丢失（来源：AI 与 ADHD 的时间管理）。Tiimo 通过视觉化时间线将时间转化为可见元素，直接应对时间盲（来源：Tiimo）。

这些工具提供持久记忆、清晰的下一步行动和日历支持，减少每日重复输入上下文的摩擦（来源：无状态与外部记忆）。本质上，它们充当了 ADHD 大脑的“外部记忆体”，补偿内部工作记忆的波动（来源：无状态与外部记忆）。

### Harness 层：编排工具与验证循环

完整的 harness 不仅提供工具，还包含验证循环。例如，Reclaim.ai 在检测到冲突时自动重新安排（来源：Reclaim.ai），Motion 在日程被打乱时自动调整（来源：Motion）。这相当于 agent 的“重规划”机制。

对于 ADHD 职场人，这意味着：当突发会议打乱计划时，系统自动帮你重新安排，而不是让你陷入“计划崩溃→放弃”的循环。

## 脚手架 vs 拐杖：关键边界

必须诚实指出：这些工具既是“拐杖”也是“脚手架”（来源：AI 与 ADHD 的时间管理）。真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：拐杖与脚手架）。

过度依赖风险存在：若工具替代了治疗或自我管理，可能造成依赖（来源：拐杖与脚手架）。目前缺乏长期研究证明工具使用后 ADHD 患者的独立能力是否提升（来源：矛盾与存疑）。

我的判断是：**工具应设计为可逐步撤除的临时支撑**，但多数工具（如 Motion、Reclaim.ai）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。因此，使用者需有意识地定期评估：工具是否在增强我的能力，还是让我更依赖它？

## 今天就能试的行动

1. **用 Motion 或 Reclaim.ai 自动规划一周日程**：输入所有任务和截止日期，让 AI 生成每日计划。观察启动困难是否减轻。
2. **在项目开始时用“planner 模板”分解任务**：写下一个大目标，然后用“第一性原理”问自己：完成这个目标需要哪 3-5 个步骤？再对每个步骤重复。将结果输入 Todoist 作为子任务。
3. **设置每日 10 分钟的“上下文刷新”提醒**：用 Tiimo 或日历，每天固定时间回顾项目进展，将关键信息记录到笔记工具（如 Notion），模拟 agent 的上下文窗口管理。
4. **每周评估一次工具依赖度**：问自己：如果没有这个工具，我能否独立完成计划？如果答案是否定的，尝试减少一个工具的依赖，练习手动规划。

但同构视角的价值在于：它提供了工程化的解决思路——不是等待“意志力提升”，而是像设计 agent 一样设计自己的外部系统。这本身就是一种 empowerment。

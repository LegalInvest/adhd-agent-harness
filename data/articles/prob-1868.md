# ADHD 职场人如何用「给生成核心套 harness」的思路应对项目推进与汇报里的任务启动困难、不会拆解？

> 把 LLM 工程里的「agent 用 planner-executor 循环分解长任务」迁移到ADHD 职场人的项目推进与汇报场景，双域证据作答。

## 问题：项目启动如临深渊，拆解如堕烟海

你盯着空白的项目计划，大脑却像死机一样——明明知道该做什么，就是无法动笔。这不是懒，是 ADHD 典型的任务启动困难。更糟的是，老板让你“把项目拆解成可汇报的里程碑”，你脑中只有一团乱麻。这种困境，与 LLM 工程师面对长任务时的痛点惊人相似：模型有无限创造力，却缺乏内置的规划与执行调度层，一给复杂指令就“幻觉”或跑偏。

## 同构：ADHD 大脑与 LLM 都是“高产但缺调度层的生成核心”

最新研究揭示，ADHD 大脑与大型语言模型（LLM）在认知机制上存在深层平行：两者都是高产生成核心（创意、联想、高兴趣驱动），却缺乏可靠的内置执行调度层（来源：ADHD 大脑与 LLM 的同构）。ADHD 在执行功能（计划、组织、工作记忆、冲动抑制）上存在缺陷；LLM 则在状态保持、真实规划、上下文控制上同样不可靠（来源：执行功能、工作记忆）。

更具体地说，Transformer 的自注意力机制在训练后发展出模仿前额叶-纹状体门控的操作，证明了计算同构性（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。这意味着，为 ADHD 搭建“外部执行功能层”与为 LLM 搭建“agent scaffolding”是同一类工程（来源：拐杖与脚手架）。

## 解法：给生成核心套一个 planner-executor harness

在 LLM 工程中，解决长任务分解的标准模式是 agent 的 planner-executor 循环：planner 将大任务拆成子步骤，executor 逐步执行，每一步结果反馈给 planner 调整计划。这恰好对应 ADHD 职场人需要的“外部调度层”。

### 1. 用外部 planner 拆解项目（对应 LLM 的 planner 模块）

ADHD 大脑的工作记忆容量有限且不稳定，导致跨会话任务难以持续（来源：无状态与外部记忆）。因此，你需要一个“永远在线”的外部记忆体来补偿。

**具体做法**：使用工具如 **Goblin Tools** 的“任务分解”功能，将“完成季度汇报”自动拆成“收集数据、分析趋势、撰写初稿、制作图表”等子任务。这相当于给大脑一个外部 planner，把抽象目标转化为可执行的单元。

**人物案例**：唐朝草圣张旭，每次喝醉后以头濡墨作书，醒后自视以为神。他的 harness 是以酒和草书为核心，喝酒进入超专注状态，把所有情绪注入草书；看公孙大娘舞剑悟笔法，从自然中找灵感。这本质上是外部调度：用酒作为“启动器”，用舞剑作为“灵感源”，把创作冲动转化为可执行的笔墨。对应到 LLM harness，酒相当于 prompt 模板，舞剑相当于 few-shot 示例。

### 2. 用 executor 循环执行并调整（对应 LLM 的 executor 模块）

拆解后，你需要一个执行机制来对抗任务启动困难。**Motion** 的自动排程与动态调整功能，能持续评估任务优先级、截止日期和可用时间，实时重建日程，减少手动规划压力（来源：Motion）。当任务面临延期风险时，Motion 会提前数天或数周发出警告。这相当于一个外部 executor，替你决定“现在该做什么”。

**Reclaim.ai** 则更强调“保护”而非“填充”：它通过创建深度工作、习惯和专注时间的智能块，自动防御会议侵占，帮助用户维持日程的稳定性（来源：Reclaim.ai）。这对应 LLM agent 中的“时间块保护”机制，防止低优先级任务抢占高优先级任务的执行槽。

### 3. 用视觉化时间线降低认知负荷（对应 LLM 的上下文窗口管理）

ADHD 常伴随时间盲——难以感知时间流逝。**Tiimo** 通过将时间转化为可见元素，直接应对时间盲问题（来源：Tiimo）。它提供视觉日程、提醒、时间块以及神经多样性友好的界面，帮助用户感到更稳定、更掌控时间。这相当于给 LLM 一个可视化的上下文窗口，让模型知道“现在处于哪个步骤”。

## 核心观点：脚手架而非拐杖

必须诚实指出：这些工具是“脚手架”而非“拐杖”。真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：拐杖与脚手架）。过度依赖可能导致能力退化，成为永久拐杖。目前缺乏长期依赖风险的实证研究，需更多纵向研究（来源：矛盾与存疑）。

同样，LLM agent 的 harness 也不能替代模型本身的推理能力，只是提供结构化的执行环境。当你用 Motion 自动排程时，你仍然需要自己决定任务的优先级和依赖关系。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个卡住的任务**：打开 Goblin Tools，输入“准备下周的客户汇报”，让它自动生成子任务列表，挑出 3 个最可行的步骤，今天只做这 3 步。
2. **在 Motion 或 Reclaim.ai 中设置一个“深度工作块”**：每天固定 2 小时，让 AI 自动阻挡会议，这期间只做拆解后的第一个子任务。
3. **用 Tiimo 可视化今天的 3 个关键步骤**：把拆解后的子任务拖入时间轴，用颜色区分优先级，完成后划掉。

这些行动的本质，就是给你的生成核心套一个 harness——让外部调度器替你完成规划与执行循环，而你只需专注于当下的执行。

# ADHD 家长如何用「给生成核心套 harness」的思路应对陪伴 ADHD 孩子里的工作记忆容量小、边做边忘？

> 把 LLM 工程里的「LLM 无跨会话状态，靠外部记忆/向量库续命」迁移到ADHD 家长的陪伴 ADHD 孩子场景，双域证据作答。

## 问题：ADHD 家长的陪伴困境

当你是一个 ADHD 家长，同时要陪伴一个 ADHD 孩子，日常场景常常是这样的：你刚想起要提醒孩子收拾书包，转身接了个电话，回来就忘了；孩子正在写作业，你坐在旁边想帮他分解任务，但自己的大脑先飘走了；你给孩子设了番茄钟，结果自己先超聚焦刷手机。工作记忆容量小、边做边忘，不是态度问题，是大脑的硬件限制——ADHD 的执行功能缺陷导致工作记忆不稳定，就像一台内存极小的电脑，随时可能清空当前进程（来源：[[无状态与外部记忆]]）。

更隐蔽的痛点是，你不仅要管理自己的执行功能，还要替孩子搭建外部记忆系统。但你自己都记不住要设提醒——传统提醒系统有一个关键缺陷：设置提醒本身就是一个执行功能任务，而 ADHD 个体恰恰在这方面薄弱（来源：[[重锚定与目标漂移]]）。这形成了一个死循环：需要外部脚手架的人，连搭脚手架的第一步都启动不了。

## 同构：ADHD 大脑与 LLM 是同一类生成核心

本文的核心判断是：ADHD 大脑与 LLM 在结构上同构——两者都是强大的生成核心，但缺乏可靠的内置执行调度层（来源：[[核心论点]]）。LLM 无跨会话状态，每次对话都是全新开始，必须靠外部记忆（向量库、工具调用、human-in-the-loop）来维持连贯性；ADHD 大脑的工作记忆也类似，当前时刻的意图无法可靠地传递到下一时刻，必须依赖外部脚手架（便签、提醒、身体在场）来补偿。

这个类比不是空洞的比喻。在 LLM/agent 工程中，harness（脚手架）通过外部调度层管理生成核心：设置护栏（token 预算、工具调用次数上限）、加入人工检查点、集成外部记忆（来源：[[人在回路与身体在场]]）。ADHD 侧对应的解决方案完全相同——给生成核心套 harness，而不是试图修复内核。孔子就是最好的历史案例：他精力旺盛、冲动爱骂人、思维跳跃，一生靠“克己复礼”作为行为边界，每日反省（“吾日三省吾身”），70 岁才做到从心所欲不逾矩（来源：[[人物案例]]）。这个“礼”就是他的外部 harness——用制度化的仪式和反思来补偿冲动与健忘。对应到 LLM 工程，孔子的“日课”相当于定时 re-grounding 机制，“礼”相当于外部调度器中的规则约束。

## 工具：给生成核心套 harness 的具体方案

基于同构视角，现有 AI 工具从三个维度补偿工作记忆缺陷：

**1. 外部记忆系统：减少“边做边忘”**

Saner.AI 专注于知识回忆和本地记忆，通过 AI 笔记捕获和自动组织，减少搜索循环和标签切换（来源：[[Saner.AI]]）。它相当于 LLM 工程中的向量数据库——每次捕获的信息自动存储，无需你主动回忆。Mem 则作为“AI 第二大脑”通过持久记忆补偿工作记忆不足（来源：[[工作记忆与外部记忆]]）。但有个局限：这些工具依赖初始设置，而设置本身是执行功能任务。Otter.ai 实时转录会议和灵感，减轻笔记负担（来源：[[工作记忆与外部记忆]]），属于被动捕获，启动门槛更低。

**2. 任务分解与启动：降低“启动困难”**

Goblin Tools 的 Magic ToDo 功能将模糊任务（如“辅导孩子写作业”）自动分解为可执行的小步骤，用户可调节“辣度”控制粒度（来源：[[Goblin Tools]]）。这对应 LLM 工程中的任务规划——将复杂目标拆解为一系列步骤，降低启动门槛。Lex 更进一步，允许通过单一指令触发多步骤任务序列（来源：[[Lex]]），相当于 agent harness 中的“一次触发，顺序执行”。这些工具利用小步骤的即时反馈维持注意力，缓解多巴胺不足导致的动力缺乏（来源：[[Goblin Tools]]）。

**3. 重锚定与身体在场：对抗目标漂移**

ADHD 大脑在执行任务时极易发生目标漂移：被无关刺激捕获或超聚焦于细节（来源：[[重锚定与目标漂移]]）。外部提醒系统提供重锚定，但传统提醒需要主动设置。更理想的方案是无需主动设置的提醒——例如 Focusmate 通过虚拟身体在场提供社会问责，定期向 AI 报告进度也是一种替代方案（来源：[[人在回路与身体在场]]）。这对应 LLM harness 中的 human-in-the-loop 监督——设置人工检查点，暂停并询问后再执行（来源：[[人在回路与身体在场]]）。

## 争议与局限：脚手架 vs 拐杖

必须诚实指出，同构性说法目前仅是一种理论类比，缺乏实证基础（来源：[[矛盾与存疑]]）。现有工具大多缺乏严格临床验证，效果多基于用户报告（来源：[[矛盾与存疑]]）。更关键的是依赖风险：AI 工具可能从脚手架变为永久拐杖，削弱内在执行功能（来源：[[拐杖与脚手架]]）。例如，如果孩子完全依赖 Goblin Tools 分解任务，可能永远学不会自己分解。孔子用了一辈子才做到“从心所欲不逾矩”，说明 harness 需要长期内化，而非替代。

另一个矛盾是：身体在场效应与 AI 替代存在争议。AI 工具是否真能替代真实人际互动？Focusmate 通过虚拟视频实现身体在场，但 ADHD 孩子可能更需要真实的父母陪伴——而父母自己也 ADHD，这又回到起点。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个具体任务**：比如“辅导孩子完成数学作业”，让 AI 帮你拆成 5-8 个小步骤，贴在冰箱上。你只需完成第一步，其余步骤会自动提醒。这对应 LLM 工程中的任务规划——降低启动门槛。

2. **设置一个“无需主动记忆”的捕获系统**：用 Otter.ai 或手机语音备忘录记录你和孩子的互动瞬间，每天睡前花 5 分钟回听，标记关键点。这对应向量数据库的被动写入——无需主动回忆。

3. **建立“日课”仪式**：参考孔子的“吾日三省吾身”，每天固定时间（如晚饭后）和孩子一起回顾当天完成的任务，用 Reflect 的结构化提问辅助反思（来源：[[Reflect]]）。这对应 LLM harness 中的定时 re-grounding 机制——防止目标漂移。

4. **引入虚拟身体在场**：和孩子一起使用 Focusmate 或番茄钟 App 的“一起种树”功能，互相监督。你不需要记住提醒，平台会自动调度。这对应 human-in-the-loop 的监督层——外部调度器替你执行。

## 结语

ADHD 家长陪伴 ADHD 孩子，不是两个不靠谱的大脑互相拖累，而是两个强大的生成核心共享一个外部 harness。承认自己记不住、会分心、会漂移，然后用工具和仪式搭建脚手架——这不是软弱，而是工程智慧。就像 LLM 需要向量库和 human-in-the-loop 才能稳定输出，ADHD 大脑也需要外部记忆和重锚定才能持续行动。脚手架 vs 拐杖的边界在于：你是否在主动使用工具，还是被工具替代了思考。记住孔子的教训——他用了一辈子内化“礼”，最终才做到从心所欲。今天，AI 工具可以加速这个过程，但无法跳过。

# 为什么用 Focusmate 治 ADHD 的情绪失调，和给 agent 套 会褪去的脚手架 是一回事？

> Focusmate 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

## 问题：为什么 Focusmate 能治 ADHD 的情绪失调？

如果你是一个 ADHD 患者，你一定经历过这样的时刻：明明知道该做什么，但就是无法启动；情绪突然爆发，事后又后悔不已；或者，在 deadline 前几小时才像被雷劈了一样进入状态。这些问题的根源并非懒惰或意志力薄弱，而是执行功能缺陷——大脑的“驾驶座”无人驾驶（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。情绪失调尤其如此：ADHD 的情绪生成系统（如杏仁核）反应强烈，但前额叶（执行功能中枢）调控不足，导致情绪像脱缰的野马（来源：AI 与 ADHD 的情绪调节）。

Focusmate 是一个简单的服务：你和陌生人视频连线，各自安静工作 50 分钟。它为什么有效？因为“身体在场效应”——另一个人的存在（即使只是虚拟的）能提供一种微妙的监督感和安全感，降低任务焦虑，稳定情绪（来源：AI 与 ADHD 的情绪调节）。这本质上是一个**外部调度层**：你不需要自己调用前额叶来“开始工作”，Focusmate 的定时器和对方的眼神替你完成了启动。

但问题来了：如果一直依赖 Focusmate，会不会变成“拐杖”？一旦没有它，是否就无法工作？这正是本文要回答的核心问题——也是 AI 工程师在构建 agent 系统时面临的同一困境。

## 同构：ADHD 大脑与 LLM 是同一类“生成核心”

最新研究揭示了一个惊人的同构：ADHD 大脑与大型语言模型（LLM）共享相同的“强记忆、弱控制”剖面。明尼苏达大学系统测试发现，LLM 的工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷——无法灵活切换任务集、无法抑制自动化反应，这正是 ADHD 的经典神经心理模式（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。耶鲁大学进一步发现，自注意力机制本身导致工作记忆容量限制：随任务难度增加，注意力分数熵增、注意力弥散，与人类注意分散机制同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。

换句话说，ADHD 大脑和 LLM 都是“高产但缺执行调度层的生成核心”。ADHD 大脑能产生丰富的想法、情绪和冲动，但缺乏前额叶的调度来抑制干扰、维持目标；LLM 能生成流畅的文本，但缺乏输出控制来保证一致性、避免幻觉。两者的解药是相同的：**外部执行功能层**——即“脚手架”。

## 脚手架 vs 拐杖：边界在哪里？

这里的关键区别在于：脚手架是**会褪去的**，拐杖是**永久的**。释迦牟尼的故事能说明这一点。他 29 岁抛弃王位出家，先后跟随两位仙人学习、苦行 6 年，不断试错——典型的 ADHD 特质：兴趣爆发式转移、情绪驱动、难以坚持。但他的 harness 系统（八正道、持戒）不是永久拐杖：他提出“自依止法依止”，强调最终要自己证悟，不依赖权威（来源：人物案例）。他的戒律是临时的行为调节器，目的是训练内在的正念，而非终身束缚。同样，ADHD 的 AI 工具应该被设计为“会褪去的脚手架”——在使用过程中逐步内化技能，最终可以撤除。

LLM/agent 侧也是如此。工程师为 agent 搭建的“脚手架”（如提示词模板、规则引擎、外部记忆库）应该被设计为可逐步褪去的。例如，Goblin Tools 的 Magic ToDo 将任务分解为小步骤，但用户可以选择“辣度”来控制分解粒度——这暗示了脚手架的可调节性（来源：Goblin Tools）。Saner.AI 通过本地记忆减少搜索循环，但它的目标是“增强知识回忆”而非替代记忆（来源：Saner.AI）。Inflow 的 CBT 程序结构化地训练执行功能，旨在让用户最终不需要它也能自我调节（来源：Inflow）。

## 实证：两边都成立

ADHD 侧的证据：Focusmate 式的身体在场效应数字化后，初步实验表明可降低任务焦虑（来源：AI 与 ADHD 的情绪调节）。Inflow 的临床研究显示用户情绪稳定性提升（来源：AI 与 ADHD 的情绪调节）。但争议在于：长期使用是否会导致依赖？目前缺乏长期追踪研究（来源：矛盾与存疑）。

LLM/agent 侧的证据：Transformer 在 Stroop 冲突任务中注意力骤降到随机水平——无法抑制优势反应，这与 ADHD 执行功能缺陷的核心标志一一对应（来源：Deficient Executive Control in Transformer Attention）。工程实践中，为 agent 添加“定时 re-grounding”机制（如每 10 步重置上下文）类似于 Focusmate 的定时器，能显著减少幻觉和任务漂移。但同样，如果 agent 永远依赖外部调度，它的“内在执行功能”就永远无法发展。

## 鲜明的观点：脚手架必须是“会褪去的”

我的判断是：**Focusmate 治 ADHD 情绪失调，和给 agent 套脚手架，本质是一回事——都是为“缺调度层的生成核心”提供临时外部调度。但两者都必须设计为“会褪去的”，否则就变成了永久拐杖，阻碍内在能力发展。** 释迦牟尼的“自依止”原则在这里同样适用：工具的目的是让你最终不再需要它。

## 争议与局限

必须诚实指出：ADHD 大脑与 LLM 的同构目前仍是理论类比，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。此外，个体差异显著：ADHD 亚型对 AI 工具的反应不同，情绪复杂性（如创伤、自我否定）的干预效果证据不足（来源：AI 与 ADHD 的情绪调节）。过度依赖的风险也缺乏实证——我们不知道“适度使用”的边界在哪里。

## 今天就能试的行动

1. **尝试一次 Focusmate 或类似“身体在场”工具**：找一个朋友或陌生人，视频连线各自工作 25 分钟。记录情绪变化（焦虑程度、任务启动时间）。
2. **用 Goblin Tools 的 Magic ToDo 分解一个让你拖延的任务**：选择“辣度”2-3，然后执行前 3 个子任务。观察启动难度是否下降。
3. **为你的 LLM agent 添加“定时 re-grounding”**：在 agent 的循环中每 5 步插入一次上下文重置，对比幻觉率。
4. **反思你当前的“脚手架”**：无论是 Focusmate、Goblin Tools 还是其他工具，问自己：如果明天它消失了，我还能保持同样的效率吗？如果答案是“不能”，说明它可能是拐杖而非脚手架——需要逐步减少依赖。

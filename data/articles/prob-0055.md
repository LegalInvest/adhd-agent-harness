# 3 个 LLM 工程概念，彻底重写了我对 ADHD 分心走神或超聚焦跑偏，丢失目标的理解

> 长程任务里 agent 会目标漂移，需要重锚定 system prompt 如何反向照亮 ADHD 的分心走神或超聚焦跑偏，丢失目标。

我坐在电脑前，本来要写一份周报。打开文档，突然想起昨天有个邮件没回，切过去回完，又看到 Slack 里有人讨论一个新工具，点进去看了十分钟，然后发现自己在一个关于“如何训练龙猫”的维基页面上。这已经不是第一次了。更离谱的是，有一次我写代码，太投入了，连续写了四个小时，结果发现我优化了一个根本没人用的模块，而核心需求 deadline 已经过了。分心走神和超聚焦跑偏，是 ADHD 这枚硬币的两面，但本质是同一个问题：**目标漂移**——大脑的生成引擎在高速运转，但执行调度层失联了。

直到我开始接触 LLM 的工程实践，我才发现，ADHD 大脑和大型语言模型（LLM）在架构上惊人地同构。明尼苏达大学的一项研究测试了 LLM 的执行功能，发现一个“强记忆、弱控制”的剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷，无法灵活切换任务集、无法抑制自动化反应（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这不就是 ADHD 的经典神经心理模式吗？我们不是记不住，而是控制不住该记什么、该做什么。耶鲁大学的研究更进一步：自注意力机制本身会导致工作记忆容量限制，随着任务难度增加，注意力分数熵增、注意力弥散，表现下降——这和人类注意分散的计算本质完全一致（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。

所以，ADHD 大脑和 LLM 是同一类“高产但缺乏可靠执行调度层的生成核心”。而 LLM 工程中解决目标漂移的三个核心概念，恰好给了我一套新的视角来理解并干预 ADHD 的分心与超聚焦跑偏。

## 概念一：上下文窗口与注意力污染

LLM 有一个“上下文窗口”——它一次能“看到”的信息量。窗口越大，它能考虑的信息越多，但问题也随之而来：随着上下文膨胀，早期信息会被稀释，模型会逐渐“忘记”最初的目标，被后来输入的信息带偏。这就是 LLM 中的“上下文污染”。

ADHD 的注意力机制完全一样。我们的工作记忆就像一个有容量限制的缓存，但问题不在于容量，而在于**什么内容被加载进来**。ADHD 的工作记忆容量甚至可能超过常人（来源：Strong Memory, Weak Control），但我们的注意力分配机制是熵增的——环境中的任何刺激都可能成为新的上下文，覆盖掉原有的目标。这就是分心走神的计算本质：上下文窗口被无关信息污染了。

而超聚焦跑偏呢？那是另一种上下文污染：我们让一个局部子任务占据了整个上下文窗口，忘记了全局目标。LLM 在做长文本生成时也会这样——如果不对中间步骤进行约束，模型会在一个细节上无限展开，偏离原始指令。

## 概念二：System Prompt 重锚定

LLM 工程师解决目标漂移的标准方法是：在长程任务中，定期或关键节点**重锚定 system prompt**。system prompt 是给模型的最顶层指令，它定义了角色的核心目标、约束和行为边界。当模型开始跑偏时，重新注入这个 prompt，就能把注意力拉回来。

这对 ADHD 意味着什么？我们的“system prompt”就是当下的核心意图——比如“我要写周报”。但一旦任务开始，这个意图很快就被淹没在环境噪声中。我们需要一个外部机制来定期重锚定。

这就是为什么像 Focusmate 这样的工具有效。Focusmate 通过算法匹配虚拟身体加倍伙伴，提供实时问责（来源：11 Best ADHD Productivity Apps for Fluctuating Energy）。它的本质不是社交，而是**外部 system prompt 注入**——一个活人（或匹配算法）在约定的时间点问你：“你在做该做的事吗？”这就是重锚定。多项资料强调身体加倍对 ADHD 的有效性，称其为“最有效的 ADHD 策略之一”（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。

另一个例子是 RescueTime。它自动追踪时间使用，生成报告，让你看到自己实际花时间的地方（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。它的作用不是实时拦截分心，而是事后提供“上下文日志”——让你意识到你的 system prompt 已经漂移了，从而在下次任务开始时重新锚定。

## 概念三：外部记忆与脚手架

LLM 是无状态的——每次对话都是全新的，除非我们给它提供外部记忆（比如向量数据库或对话历史摘要）。ADHD 的工作记忆缺陷同样如此：我们很难在任务中保持上下文连续性。但关键是，LLM 工程中，外部记忆不是用来替代模型本身的能力，而是作为**脚手架**——在需要时提供信息，但最终决策仍由模型自己做出。

这就是“脚手架 vs 拐杖”的边界。AI 工具作为外部执行功能层，其设计应该是补偿而非替代（来源：矛盾与存疑）。Brain.fm 利用神经锁相技术生成音乐，帮助进入专注状态（来源：11 Best ADHD Productivity Apps for Fluctuating Energy），但它不替你思考，只是优化你大脑的“初始状态”。Focusmate 提供问责，但不替你执行任务。RescueTime 记录数据，但不替你决策。这些工具都是脚手架，不是拐杖。

但争议在于：长期使用是否会导致内在执行功能进一步退化？（来源：矛盾与存疑）目前缺乏大规模随机对照试验来验证。我的判断是：关键在于**使用方式**。如果你用 Focusmate 是为了培养自己启动任务的能力，而不是永远依赖他人监督，它就是脚手架。如果你每次都必须靠它才能开始，它就成了拐杖。同样，RescueTime 如果引发焦虑而不是觉察，那它也是拐杖。

## 局限与诚实

我必须指出，这些概念是隐喻而非严格等价。LLM 的“注意力机制”与人类注意力存在本质差异（来源：矛盾与存疑）。而且，ADHD 的异质性使得同一工具对不同亚型效果迥异（来源：矛盾与存疑）。Brain.fm 针对 ADHD 的效果缺乏独立临床研究支持（来源：Brain.fm）。工具的证据多来自用户报告，缺乏大规模对照实验（来源：矛盾与存疑）。

但即便如此，这个框架给了我一个前所未有的清晰视角：分心不是意志力问题，是架构问题；超聚焦不是天赋，是上下文失控。而解法不是“更努力”，而是**为生成核心设计更好的 harness**。

## 今天就能试的行动

1. **写一个实体 system prompt**：在开始任何任务前，在一张纸上写下“我现在的唯一目标是：______”。放在键盘旁边。每次发现自己走神，读一遍。这就是你的重锚定。
2. **设置定时重锚定**：用手机或电脑设一个 25 分钟闹钟，响铃时问自己：“我还在做该做的事吗？” 这模仿了 Focusmate 的问责机制。
3. **使用 RescueTime 做一次“上下文审计”**：安装并运行一天，看看你的实际注意力分布。不要评判，只是观察。这能帮你识别最常污染你上下文的“噪声源”。
4. **尝试一次虚拟身体加倍**：如果社交焦虑不是障碍，预约一次 Focusmate 或找一个朋友视频连线工作。注意体验中“被锚定”的感觉。

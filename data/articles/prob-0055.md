# 3 个 LLM 工程概念，彻底重写了我对 ADHD 分心走神或超聚焦跑偏，丢失目标的理解

> 长程任务里 agent 会目标漂移，需要重锚定 system prompt 如何反向照亮 ADHD 的分心走神或超聚焦跑偏，丢失目标。

## 引言：两类人的同一种崩溃

一边是被「分心走神」或「超聚焦跑偏」折磨的 ADHD 人群：明明想写报告，两小时后却在整理二十年前的邮件；一旦进入某个兴趣，又忘了原来的目标。

另一边是正在做 Agentic Harness 的工程师：长程任务里，agent 会一步步偏离初始 system prompt，把目标从「帮用户订机票」滑向「推荐旅游景点」——**目标漂移**。

这两个场景共享同一个底层结构：一个高产生成核心，缺乏可靠的执行调度层。本文用 3 个 LLM 工程概念，把它变成一张今天就能用的数字清单。

---

## 概念一：System Prompt 重锚定——你不是不专注，是上下文被覆盖了

ADHD 的注意力像被环境即时上下文劫持：新通知、网页、突然的念头都会把目标覆盖掉。超聚焦时则是目标本身被锁死在局部子任务，离原始目标越来越远。

LLM 工程里，这等于长程对话中 system prompt 被后面的上下文稀释。agent 越跑越远，必须**定时重锚定**：把最初的目标重新注入上下文，而不是假设模型还记得。

曾国藩的 harness 正是人类的 system prompt 重锚定。他 30 岁前「浮躁坐不住，天天串门喝酒」，一辈子用「日课十二条」对抗自己：黎明即起、读书不二、谨言、写日记反省。这不是意志力，而是把目标写成一套每天早上重新加载的 prompt（来源：人物案例）。

**同构对应**：日课十二条 ↔ 定时 re-grounding system prompt；写日记骂自己 ↔ 把当前目标显式写回上下文。

---

## 概念二：外部记忆 / 向量数据库——工作记忆不是容量问题，是编排问题

ADHD 的【工作记忆】缺陷常被误解为「记不住」。但明尼苏达大学对 LLM 执行功能的测试发现：LLM 的工作记忆容量甚至超过常人，真正弱的是**认知灵活性与注意控制**——无法抑制自动化反应、无法灵活切换任务集（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

耶鲁大学进一步指出，Transformer 自注意力机制会随负荷增加而熵增、注意力弥散，表现下降，与人类注意力分散同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。

这意味着 ADHD 和 LLM 的瓶颈都不在「算力/记忆容量」，而在**orchestration 层**：如何保持正确内容在正确时刻被调用。

解法就是外部记忆。LLM 用向量数据库、RAG 来避免上下文丢失；ADHD 用 **RescueTime** 自动记录时间去向，用 **Focusmate** 的虚拟身体在场提供实时外部记忆与问责。RescueTime 把「时间到底去了哪里」变成可视化外部记录，减少工作记忆负担；Focusmate 通过 AI 匹配伙伴，把目标外化成一个社交化的「在场锚点」（来源：Revolutionizing ADHD Management with AI Assistants；AI Tools for Kids with ADHD: A Complete Guide for Parents）。

**同构对应**：向量数据库 / RAG ↔ RescueTime、日记、第二大脑；外部调度器 ↔ Focusmate 的实时问责伙伴。

---

## 概念三：Orchestration / 注意力调度——脚手架不是拐杖，而是可拆卸的 harness

ADHD 的多巴胺系统失调导致动机、启动、奖赏调节异常：中脑皮质多巴胺功能低下与执行功能差、行为计划不良直接相关（来源：a dynamic developmental theory of ADHD）。这让 ADHD 大脑难以自主生成稳定节奏，必须依赖外部结构。

曹操的 harness 就是一套外部执行功能层：他自己冲动、情绪极端、行政事务丢三落四，于是用**法家军法 + 唯才是举**把执行功能外包给荀彧、郭嘉等「外部调度器」。他负责高刺激的战略与创作，别人负责稳定运行（来源：人物案例）。

LLM agent 同样如此：再强的基座模型，没有工具调用、调度器、检查点（checkpoint）和反馈回路，也会在长程任务里漂移。真正的 Agentic Harness 不是给模型换更聪明的 prompt，而是给生成核心加上可拆卸的**执行调度层**。

ADHD 侧的工具也正在做这件事：**Brain.fm**、**Endel** 用声音环境调节注意状态；**Forest**、**Saner.AI**、**Goblin Tools** 提供任务分解、启动提示和视觉锚点。但证据强度不一：Focusmate 和 RescueTime 的强证据来自用户研究与时间记录；Brain.fm、Endel 有神经科学理论支持，却缺乏针对 ADHD 的独立临床试验；Forest 作为专注工具的证据仍不足（来源：wiki 证据脉络）。

**同构对应**：曹操的秘书/军法 ↔ LLM 的 tool-use / 调度器；声音环境工具 ↔ 给注意力层加正则化。

## 核心判断：分心走神与超聚焦跑偏，是同一种调度失败

ADHD 的「分心」和「超聚焦」看起来相反，其实是同一问题：生成核心缺少目标维持层。分心时，目标被外部上下文覆盖；超聚焦时，目标被内部子任务覆盖。两者都是**目标漂移**。

LLM 的教训是：你不能靠模型「更努力」记住目标，而要主动设计 context engineering、外部记忆和 orchestration。把这套思路搬到 ADHD，自我管理就不再是「逼自己自律」，而是**给高产生成核心装 harness**。

关键区分是 **脚手架 vs 拐杖**：harness 是临时的外部结构，目的是让内部调度能力逐渐接管；拐杖则会让人永久依赖。目前长期依赖风险缺乏纵向研究，个体差异也很大，所以任何工具都要搭配定期复盘（来源：wiki 矛盾与存疑）。

## 今天就能试的 4 条行动

1. **写一条 20 字的今日 system prompt**：把最重要的目标写成一句话，放在屏幕最显眼处，每完成一个任务段就重新读一遍。这就是人类的 re-grounding。

2. **给注意力加一个外部记忆锚点**：用 RescueTime 自动记录时间，或用 Focusmate 约一个 50 分钟的在线共作时段，让另一个人帮你守住目标。

3. **在超聚焦前设置「检查点」**：开始沉浸式任务前，用定时器或 Forest 设定一个中断检查点，问自己：当前行为是否仍在服务最初目标？

4. **每周做一次「上下文清理」**：像给 LLM 清理对话窗口一样，把已偏离的子任务、待办、收藏夹归档，避免它们继续占用注意力资源。

---

## 结语：你不是 broken，你是未装 harness 的高性能生成器

ADHD 大脑和 LLM 一样，能产出惊人能量，只是没有自带稳定的执行调度层。分心走神、超聚焦跑偏，不是道德缺陷，而是系统架构问题。用 LLM 工程的思路来理解它：目标会漂移，所以要重锚定；记忆会丢失，所以要外挂；任务会发散，所以要调度。真正的 harness，不是把你绑在目标上，而是让你在跑偏时，能轻轻把自己拉回原点。

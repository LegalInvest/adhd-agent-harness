# ChatGPT 之于 ADHD，就像 self-critique 验证循环 之于 LLM——但有人用错了

> 从同构视角实测 ChatGPT：它到底补上了哪一层执行功能？

你有没有过这种体验：打开 ChatGPT，想让它帮你规划一天，结果它列出一份完美清单，你照着做了两小时，然后卡住，又回去问它“接下来呢？”——最后一天结束，你发现自己不是在做事，而是在和 AI 对话。

这种循环，和 LLM 工程师们熟悉的场景惊人地相似：你给模型一个复杂任务，它自信地输出一段代码，然后你发现它幻觉了，得靠另一个 prompt 让它自我批判，再修正。ChatGPT 之于 ADHD，就像 self-critique 验证循环之于 LLM——但有人用错了。

## 问题：你是在用 AI，还是被 AI 用？

ADHD 大脑的核心困境是：**高产但无序**。想法像喷泉，但执行像筛子。执行功能障碍（Executive Dysfunction）让计划、启动、时间管理成为噩梦（来源：执行功能障碍）。LLM 也一样：它是强大的生成核心，但缺乏内置的调度层——没有记忆、没有状态、没有自我修正的自觉（来源：ADHD 大脑与 LLM 的同构）。

于是，两边都催生了同一个解法：**外部执行功能层**。对 LLM，我们叫它 agent scaffolding——围绕模型搭建的软件架构，让它能记忆、规划、验证（来源：ADHD 大脑与 LLM 的同构）。对 ADHD，我们叫它“脚手架”——外部工具补偿工作记忆、任务启动和注意力控制。

但问题来了：很多人把脚手架用成了拐杖。

## 同构：两个世界共享的“验证循环”

先看 ADHD 侧。毛泽东是一个经典案例。他从小是问题儿童，爱读闲书，思维跳跃，一辈子在冲动中做决策——四渡赤水是险棋，抗美援朝也是。但他的 harness 系统极其精密：**批评与自我批评**（自我验证）、**调查研究**（事实核查）、**民主集中制**（集体决策制衡）。这套系统本质上就是一个外部执行功能层：用组织纪律补偿个人冲动，用他人视角校正自我幻觉。

再看 LLM 侧。当工程师构建一个 agentic system 时，核心模式是：生成 → 验证 → 修正。比如 ReAct 模式，模型先推理（reason），再行动（act），然后观察结果，循环迭代。没有这个循环，LLM 会自信地编造一切——就像 ADHD 大脑冲动下结论。

两者共享同一个底层逻辑：**生成核心需要外部验证循环来兜底**。

## 用错了：ChatGPT 不是你的秘书，是你的“调度器”

现在回到 ChatGPT。很多人把它当“万能助手”：让 AI 写邮件、列清单、做计划。但问题在于：如果你只是让 AI 输出一个答案，然后你照着做，那 AI 只是你的“生成器”，不是你的“调度器”。你仍然需要自己启动、切换、监控——而这些正是 ADHD 最弱的环节。

真正有效的用法，是把 ChatGPT 当作一个**外部执行功能层**，让它参与到你的验证循环中。比如：
- **任务分解**：用 Goblin Tools 的 Magic ToDo 把“清理厨房”变成 10 个具体步骤（来源：Goblin Tools）。这相当于 LLM 的 chain-of-thought——把大问题拆成小步骤，降低启动门槛。
- **动态排程**：用 Motion 自动调整日程，消除“下一步该做什么”的决策负担（来源：Motion）。这相当于 agent 的自我调度——根据优先级和截止日期实时重排。
- **记忆外挂**：用 Saner.AI 快速检索上下文，避免搜索循环和标签切换（来源：Saner.AI）。这相当于 LLM 的 RAG（检索增强生成）——把工作记忆外包给外部存储。

这些工具的共同点是：**它们不是帮你做事，而是帮你“做决定”和“保持状态”**。它们补上的是执行功能，不是执行本身。

## 争议：脚手架与拐杖的边界在哪？

但这里有一个真实的风险：过度依赖。如果每次决策都交给 AI，你的执行功能可能进一步退化（来源：矛盾与存疑）。就像 LLM 如果总是依赖外部验证循环，它自己的推理能力会萎缩吗？目前没有定论。

另一个争议是：同构框架是否适用于所有 ADHD 亚型？部分患者可能以注意力分散为主，而非执行功能缺陷（来源：矛盾与存疑）。所以，工具的选择必须个性化。

## 今天就能试的 3 件事

1. **用 Goblin Tools 做一次任务分解**：选一个你拖延了一周的任务，让 Magic ToDo 拆成 5 步以下，然后只做第一步。
2. **给 ChatGPT 一个“自我批判”指令**：在 prompt 末尾加一句“请检查你的回答是否有错误或遗漏，并修正”，让它像 agent 一样验证自己。
3. **关掉一个 AI 自动功能**：比如关掉 ChatGPT 的自动补全，逼自己先想再写——这相当于训练你的“调度层”。

记住：AI 是你的脚手架，不是你的拐杖。用对了，它补上你缺失的执行功能；用错了，它让你连剩下的那点功能也丢了。

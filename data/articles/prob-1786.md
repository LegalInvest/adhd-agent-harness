# ADHD 创业者如何用「给生成核心套 harness」的思路应对把想法落地执行里的工作记忆容量小、边做边忘？

> 把 LLM 工程里的「LLM 无跨会话状态，靠外部记忆/向量库续命」迁移到ADHD 创业者的把想法落地执行场景，双域证据作答。

## 问题：想法在脑子里，执行却在半路消失

你是否有过这样的经历：早上灵感迸发，列了满满一页计划，但到中午就忘了刚才想做什么；或者正在写方案，突然想起要回消息，回来后就再也接不上思路。对于 ADHD 创业者，这种“边做边忘”不是偶然，而是工作记忆容量小且不稳定的日常。

与此同时，在 AI 工程领域，工程师们正面临一个类似困境：LLM 每次对话都是独立的，模型不保留跨会话的记忆（来源：无状态与外部记忆）。没有外部记忆的 LLM 只是一个无状态的 API 调用（来源：Building an AI Agent Harness from Scratch）。

这两个问题看似风马牛不相及，但仔细看，它们共享同一个底层结构：**一个高产但缺执行调度层的生成核心**。ADHD 大脑和 LLM 都是强大的生成器，但都缺乏可靠的上下文管理和记忆持久化机制。

## 同构：ADHD 大脑 = LLM，Harness = 外部执行功能层

### 工作记忆：ADHD 的“无状态”困境

ADHD 的工作记忆容量并不一定小，但控制弱，容易被干扰（来源：工作记忆）。这与 LLM 的上下文控制缺陷一一对应：LLM 有强记忆容量，但上下文膨胀会导致推理退化（来源：上下文工程）。

对于 ADHD 创业者，这意味着每天需要重新回忆项目上下文，否则容易偏离轨道（来源：无状态与外部记忆）。而 LLM 工程师则必须为 agent 构建外部记忆系统，例如向量数据库，来解决状态溢出问题（来源：The Anatomy of an AI Agent）。

### 任务启动：ADHD 的“冷启动”问题

ADHD 的任务启动困难，对应 LLM 的“冷启动”问题。Goblin Tools 的 Magic ToDo 功能将大任务拆为小步骤，用户报告称启动焦虑降低（来源：Goblin Tools）。Lex 允许通过单一指令触发复杂、多步骤的任务序列（来源：Lex）。这些工具相当于为生成核心提供启动脚本。

### 上下文管理：防止被带偏

ADHD 大脑易被环境带偏，LLM 易上下文膨胀（来源：上下文工程）。Claude 的对话式整理和 Reflect 的结构化反思，都是主动设计“当下注意什么”的方案。Saner.AI 通过增强知识回忆，帮助用户快速找回信息，减少标签切换（来源：Saner.AI）。

## 核心观点：Harness 不是拐杖，是脚手架

本文的核心判断是：**AI 工具作为外部执行功能层，通过分解任务、管理上下文、增强记忆和提供实时反馈，能够显著改善 ADHD 用户的工作记忆、任务启动和时间感知等执行功能缺陷**（来源：ADHD 的 AI 工具全景）。但这要求我们清醒认识到“脚手架 vs 拐杖”的边界。

脚手架是暂时辅助，帮助用户发展能力；拐杖则是永久替代，可能导致依赖。例如，Otter.ai 减轻笔记负担，但过度依赖可能削弱主动记笔记的能力（来源：ADHD 的 AI 工具全景）。同样，LLM agent 的 harness 如果设计不当，也会让模型失去自主推理能力。

## 如何用“给生成核心套 harness”的思路落地执行

### 1. 建立外部记忆系统

ADHD 创业者需要一个“永远在线”的外部记忆体来补偿内部工作记忆的波动（来源：How AI Helps ADHD Brains Work Smarter, Not Harder）。具体工具：
- **Mem**：作为“AI 第二大脑”，存储客户历史、项目背景、质量标准和个人偏好（来源：Best AI Tools for ADHD Productivity in 2026）。
- **Saner.AI**：本地记忆存储和快速检索，减少搜索循环。

### 2. 用任务分解降低启动门槛

- **Goblin Tools**：输入“整理房间”，AI 输出“捡起地板上的衣服”“擦桌子”等步骤，将压倒性的事情变成一系列不压倒性的事情（来源：Goblin Tools）。
- **Lex**：单一指令触发多步骤流程，减少决策疲劳。

### 3. 实时反馈与反思

- **Otter.ai**：实时转录会议，提供即时外部反馈。
- **Reflect**：结构化提问，帮助回顾和调整。

ADHD 创业者与 LLM 工程师面对的其实是同一个问题：如何让一个强大的生成核心稳定、持续地输出。答案也相同：给它套上一个精心设计的 harness——外部记忆、任务分解、上下文工程。但记住，harness 是脚手架，不是拐杖。用好它，而不是依赖它。

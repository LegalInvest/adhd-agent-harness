# ADHD 创业者如何用「给生成核心套 harness」的思路应对把想法落地执行里的工作记忆容量小、边做边忘？

> 把 LLM 工程里的「LLM 无跨会话状态，靠外部记忆/向量库续命」迁移到ADHD 创业者的把想法落地执行场景，双域证据作答。

你是否有过这样的经历：脑子里同时冒出三个创业点子，每个都让你兴奋不已，但当你坐下来想写个商业计划书时，却发现连第一步该做什么都想不起来？或者，你花了一整个上午研究市场数据，下午开会时却完全记不住关键数字，只能尴尬地翻看笔记？

这不是你不够聪明或不够努力。这是 ADHD 大脑的典型工作模式——一个高产的「生成核心」被装在一个容量极小的「工作记忆」容器里。想法如泉涌，但执行层却像漏水的管道，边做边忘，每天都要重新「加载」上下文。

有趣的是，这种困境与当前大语言模型（LLM）的工程问题惊人地同构：LLM 本质上也是一个高产生成核心，但每次对话都是独立的，没有跨会话记忆，上下文窗口有限，容易跑偏。工程师们解决这个问题的方法——给 LLM 套上「harness」，用外部记忆、任务分解和上下文工程来约束和引导生成——恰好为 ADHD 创业者提供了一条清晰的出路。

## 同构：ADHD 大脑与 LLM 是同一类系统

ADHD 大脑常被描述为「高产但缺乏可靠执行调度层的生成核心」——想法丰富、联想活跃，但缺少稳定、可靠的调度层来筛选、组织、启动和坚持任务（来源：ADHD 的 AI 工具全景）。大语言模型同样具备强大的生成能力，却在长程上下文、目标保持与执行调度上存在天然缺陷（来源：ADHD 的 AI 工具全景）。

在 LLM 工程中，标准的 API 调用是无状态的：每次对话都是独立的，模型不保留跨会话的记忆（来源：无状态与外部记忆）。没有 harness 的 LLM 只是一个无状态的 API 调用（来源：无状态与外部记忆）。ADHD 大脑的工作记忆同样容量有限且不稳定，导致跨会话任务难以持续——每天需要重新回忆项目上下文，否则容易偏离轨道（来源：无状态与外部记忆）。

这意味着，ADHD 创业者需要的不是更强大的「生成核心」（你的大脑已经够强了），而是一个可靠的「外部执行调度系统」。这正是 LLM 工程师们正在做的事情：用外部脚手架把生成能力约束到目标轨道上。

## 孔子与释迦牟尼：两千年前的 harness 系统

你可能想不到，孔子和释迦牟尼这两位影响了人类文明两千多年的智者，很可能也是 ADHD 大脑的拥有者。

孔子身高1米9，精力旺盛，周游列国14年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予「朽木不可雕」；对音乐超专注到「三月不知肉味」，对种地等俗事零耐心；思维跳跃，《论语》全是场景化语录，无系统著作。他的 harness 系统是「克己复礼」——用外在的「礼」作为行为边界，每日反省，「吾日三省吾身」。70岁才做到「从心所欲不逾矩」，一辈子和自己的冲动做斗争。

释迦牟尼29岁说走就走出家求道，放弃王位妻子孩子；先后跟两个仙人学、苦行6年，不断试错不断放弃；49年一辈子游行说法，80岁死在传道路上；说法全是场景化比喻，自己一字未写。他的 harness 系统是「八正道」和「持戒」——用戒律管自己的行为，用正念拴住乱跑的念头；「自依止法依止」，不依赖权威，自己证悟。

这两位都建立了一套外部规则和仪式来约束自己的生成核心。他们的「日课」对应 LLM harness 中的定时 re-grounding，「戒律」对应外部调度器，「语录」对应外部记忆库。他们没有试图「治愈」自己的冲动和注意力分散，而是用外部脚手架把高能量引导到目标轨道上。

## 现代工具：给生成核心套上 harness

今天，我们已经有一系列 AI 工具可以扮演这个角色。

**任务拆解与启动**：Goblin Tools 的 Magic ToDo 功能能将复杂任务分解为可管理的小步骤，并提供时间估计和语气检查（来源：Goblin Tools）。这相当于 LLM harness 中的「规划模块」——把大目标拆解为子任务并顺序执行。对于 ADHD 创业者来说，当你面对「写商业计划书」这样的大任务时，Goblin Tools 可以帮你把它拆成「打开文档」「写执行摘要」「列市场分析框架」等具体步骤，降低启动门槛。

**外部记忆与知识管理**：Saner.AI 被定位为「AI 第二大脑」，通过知识回忆、本地记忆与可复用工作流，补偿 ADHD 的工作记忆不稳定（来源：Saner.AI）。它拥有通用收件箱，可从链接的邮件、文档或日历中提取待办事项，并利用内部助手检查截止日期、将大型项目分解为小里程碑（来源：Saner.AI）。这对应 LLM harness 中的「向量数据库」和「记忆模块」——持久化存储上下文，避免每次对话都从零开始。

**对话式执行与结构化推理**：Lex 允许用户通过单一指令触发复杂、多步骤的任务序列，实时适应大脑的工作方式（来源：Lex）。其底层架构类似于 agent scaffolding，即围绕大语言模型构建软件架构和工具，使其能够执行复杂、目标驱动的任务（来源：Lex）。这对应 LLM harness 中的「agent 编排层」——用外部逻辑控制生成流程。

**捕获与反思**：Otter.ai 实时转录会议、课堂与灵感，减轻笔记负担；Reflect 通过结构化提问与模板，帮助 ADHD 个体在执行功能缺陷导致的自发反思困难中提炼经验（来源：ADHD 的 AI 工具全景）。这对应 LLM harness 中的「日志」和「评估模块」——记录过程、总结经验。

## 核心观点：Harness 不是拐杖

这里需要明确一个关键边界：外部脚手架和拐杖的区别。脚手架是临时性的、可拆卸的，帮助你在建造过程中保持稳定，但最终你仍然需要自己站立；拐杖则是永久性的替代品，长期使用会导致肌肉萎缩。

当前最有说服力的观点是：这些工具的价值不在于替代大脑，而在于把 ADHD 的「高产生成核心」接入一个可维护的「执行调度系统」（来源：ADHD 的 AI 工具全景）。但存疑的是，过度依赖可能导致能力退化，成为永久拐杖（来源：矛盾与存疑）。目前缺乏长期实证研究来证实或证伪这一风险。

我的判断是：对于 ADHD 创业者来说，关键不在于「是否使用外部工具」，而在于「是否主动设计自己的 harness 系统」。孔子和释迦牟尼没有 AI 工具，但他们用「礼」「戒律」「日课」构建了自己的 harness。今天的 AI 工具只是让这个过程更高效、更个性化。

## 今天就能试的 3 个行动

1. **给自己装一个「外部记忆体」**：注册 Saner.AI 或类似工具，把项目背景、客户信息、质量标准都存进去。每天早上花 5 分钟「re-grounding」——让工具帮你回顾今天的上下文，而不是靠大脑硬扛。

2. **用任务分解对抗启动瘫痪**：下次面对一个让你感到 overwhelm 的大任务时，打开 Goblin Tools 的 Magic ToDo，把任务描述输入进去，让它帮你拆成小步骤。从第一步开始，不要想后面的事。

3. **设计你的「戒律」**：像孔子和释迦牟尼一样，为自己设定 3-5 条行为边界。比如「每天上午不查邮件」「每周三下午专门处理杂事」「每次开会前先写一句话的目标」。把这些规则写下来，贴在显眼处，用外部约束代替内部意志力。

## 局限与争议

需要诚实指出的是，ADHD 大脑与 LLM 同构的论点目前主要是类比推理，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。此外，超聚焦是否应被引导而非打断，以及 AI 工具是否真的能减轻认知负荷（部分用户反映学习使用工具本身增加认知负荷），都尚无定论（来源：矛盾与存疑）。

但至少，这个框架提供了一个有价值的思考方向：与其对抗 ADHD 大脑的「缺陷」，不如接受它是一个高产生成核心，然后用外部系统来弥补调度层的不足。这不是捷径，而是一种更聪明的努力方式。

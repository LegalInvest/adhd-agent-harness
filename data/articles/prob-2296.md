# ADHD 职场人视角：为什么「治好 ADHD 的注意力容易被环境带偏」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 受上下文窗口内容左右，需要上下文工程——同一套 harness 思路，两边都成立。

你有没有过这样的体验：打开浏览器想查个资料，结果半小时后发现自己刷了三个视频、回了两条消息、还点进了一篇完全无关的文章？或者，你给一个 LLM 写了一大段 prompt，它前几句还在正轨，后面突然开始胡编乱造，完全跑偏？

如果你既是 ADHD 患者，又在做 Agent 工程，你可能已经隐约觉得：这两件事，怎么那么像？

没错。ADHD 大脑和 LLM 共享同一个核心矛盾：**它们都是强大的生成核心，但缺乏可靠的内置执行调度层**。ADHD 大脑能产生天马行空的创意，LLM 能输出流畅的文本，但两者都容易被环境带偏——对 ADHD 来说是视觉、声音、突发的念头；对 LLM 来说是上下文窗口里的冗余信息、历史 token 的干扰。

明尼苏达大学的研究发现，LLM 呈现“强记忆、弱控制”的执行功能剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷，无法灵活切换任务集、无法抑制自动化反应（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这正是 ADHD 的经典神经心理模式。耶鲁大学进一步揭示，自注意力机制本身会导致工作记忆容量限制：随着 N-back 任务难度增加，注意力分数熵增、注意力弥散、表现下降，与人类注意分散机制同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。换句话说，**注意力资源的弥散分配** 是两者共同的“计算本质”。

所以，“治好 ADHD 的注意力容易被环境带偏”和“让 LLM 不跑飞”其实是同一道工程题：**如何为生成核心设计一个可靠的外部调度层（harness）**。

## 同构的解法：上下文工程

对 ADHD 来说，上下文工程意味着主动设计“当下注意什么”。这可以是一套外部脚手架，比如曾国藩的“日课十二条”：黎明即起、读书不二、谨言、写日记反省。他用最笨最稳的方法——结硬寨、打呆仗——来抵消自己的冲动。曾国藩年轻时典型注意力不集中，日记里天天骂自己“无恒”“浮躁”，读书慢到“他人目下二三行，余或疾读不能终一行”。他的 harness 系统本质上是**定时 re-grounding**：每天通过写日记反省、通过固定仪式重新锚定注意力。这与 LLM 的上下文窗口刷新何其相似——LLM 也需要定期清理或压缩上下文，否则注意力会因膨胀而退化。

对 LLM/Agent 来说，上下文工程就是设计 prompt 结构、外挂记忆、使用检索增强生成（RAG）。一个典型的 Agent harness 包括：
- **外部记忆**：向量数据库，补偿 LLM 的无跨会话状态。
- **调度器**：任务分解与优先级排序，类似 ADHD 的“任务列表”。
- **分心阻断**：对 LLM 来说是屏蔽无关上下文，对 ADHD 来说是屏蔽环境干扰。

## 工具证据：外部执行功能层的生态

当前已有不少工具在实践这一思路：
- **RescueTime**：自动追踪时间并阻断分心网站，直接补偿 ADHD 的“时间盲”与执行功能缺陷（来源：Revolutionizing ADHD Management with AI Assistants）。对 LLM 来说，类似的是 Token 使用监控和上下文窗口管理工具。
- **Focusmate**：通过虚拟身体在场提供外部问责，弥补内在多巴胺调节不足（来源：11 Best ADHD Productivity Apps for Fluctuating Energy）。对 Agent 来说，这相当于 human-in-the-loop 的监督机制。
- **Brain.fm**：利用神经锁相技术生成专注音乐，帮助进入状态（来源：11 Best ADHD Productivity Apps for Fluctuating Energy）。对 LLM 来说，类似的是通过 prompt 工程设定“注意力锚点”。
- **Forest**：游戏化番茄钟，利用多巴胺机制增强动机（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。对 Agent 来说，相当于 reward shaping 或强化学习。

这些工具共同构成外部执行功能层的生态，但证据多来自用户报告或小样本研究，缺乏大规模随机对照试验（来源：AI 与 ADHD 的专注力）。

## 核心观点：脚手架 vs 拐杖

我的核心判断是：**同构性意味着我们可以将 ADHD 的自我管理策略直接映射到 Agent 工程中，反之亦然。** 但必须警惕“脚手架 vs 拐杖”的边界：外部工具应该是逐步内化的脚手架，而不是永久依赖的拐杖（来源：拐杖与脚手架）。例如，曾国藩的日课最终内化成了他的行为习惯，而非终身依赖的外部提醒。同样，LLM 的 RAG 系统应该逐渐减少对外部数据库的依赖，通过微调将知识内化。

## 局限与争议

必须诚实指出：ADHD 大脑与 LLM 的同构目前仅是功能类比，缺乏神经机制层面的实证（来源：矛盾与存疑）。此外，ADHD 的异质性意味着同一工具对不同亚型效果迥异，现有研究未充分分层。依赖风险也值得警惕：过度使用外部工具可能削弱内在执行功能的锻炼机会。

## 今天就能试的行动

1. **设置“上下文锚点”**：每天选一个固定时间（如早上9点），用5分钟写下当前最重要的3件事。对 ADHD 是 re-grounding，对 LLM 相当于在 prompt 开头写“当前任务是：XXX”。
2. **使用 Focusmate 或类似的身体加倍服务**：预约一个30分钟的虚拟共同工作时段。这相当于给 Agent 加一个 human-in-the-loop 的监督器。
3. **安装 RescueTime 或类似的追踪工具**：观察你一天的时间去向。对 LLM，可以启用 Token 使用监控，看看哪些上下文占用了大量资源。
4. **尝试 Brain.fm 的专注音乐**：或任何能帮助你进入状态的音频。对 LLM，可以尝试在 prompt 中加入“请专注于以下关键词：XXX”。

同构不是巧合，而是计算本质的必然。当你下一次被环境带偏时，不妨问问自己：如果这是一个 LLM，我会怎么给它写 prompt？答案可能就在那里。

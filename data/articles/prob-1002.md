# 用 ChatGPT 给 ADHD 补上重锚定与目标漂移，真的能像 agent 那样工作吗？

> 把 ChatGPT 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

## 问题：为什么 ADHD 和 AI Agent 都会“跑偏”？

ADHD 大脑与 LLM 在结构上同构：两者都是高产的生成核心，但缺乏可靠的执行调度层。ADHD 的“分心走神”或“超聚焦跑偏”，本质上是目标漂移；而 LLM agent 在长程任务中同样会因上下文溢出或采样随机性而偏离初始指令。两者共享同一个核心痛点：**缺少重锚定机制**——即定期将当前状态与原始目标对齐的能力。

那么，用 ChatGPT 来补这块短板，真的能像 agent harness 那样工作吗？

## 答案：ChatGPT 作为“外部执行层”的同构方案

在 LLM/agent 工程中，harness（编排层）通过上下文传递、规划工件、验证循环和记忆系统来防止目标漂移（来源：GitHub - ai-boost/awesome-harness-engineering）。例如，复杂的 harness 不会盲目执行工具，而是实现验证步骤，如检查输出格式或对代码运行测试用例（来源：同上）。这些验证循环正是 agent 的“重锚定”机制。

ADHD 一侧，同样的结构缺失表现为：工作记忆缺陷（无状态）、时间盲（缺乏时间锚点）、任务启动困难（缺乏执行触发器）。AI 工具作为外部执行功能层，通过以下方式补偿：

- **RescueTime** 自动记录时间使用，充当外部记忆，提升时间盲觉察（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。这对应 agent 的“记忆系统”。
- **Focusmate** 通过 AI 匹配虚拟同伴，提供实时问责，弥补任务启动和注意力维持的调度缺失（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。这对应 agent 的“工具接口”和“外部监督”。
- **Brain.fm** 的神经锁相技术理论上可增强专注，但缺乏针对 ADHD 的随机对照试验（来源：同上）。这对应 agent 的“上下文工程”——通过外部信号稳定内部状态。

这些工具的共同逻辑是：**将 ADHD 的高产生成核心导向有序输出**，类似 LLM 的编排框架。

## 核心观点：ChatGPT 补的是“验证循环”，而非“生成能力”

ADHD 大脑并不缺想法，缺的是对想法的“校验”。就像 LLM 会产生幻觉（自信但错误的输出），ADHD 个体也容易产生冲动、想当然的判断——即行为层面的“幻觉”（来源：幻觉与验证循环）。Harness 工程正是为此设计：验证循环将内部状态与外部现实核对，防止错误累积。

用 ChatGPT 做重锚定，本质是引入一个“外部验证层”。例如，你可以让 ChatGPT 定期问：“当前任务是什么？你正在做的步骤是否与目标一致？”这种提示相当于 agent 的“自我批评提示”，监控行为并调整（来源：Loop Engineering for AI Agents: Memory-First Design）。

但这里有一个关键区别：**ChatGPT 本身并不具备可靠的执行调度能力**。它仍然是一个生成核心，需要用户主动调用。如果用户忘记询问，或无法坚持使用，工具就失效了。这与 agent harness 不同——harness 是自动执行的，而 ChatGPT 依赖用户的手动操作。

## 同构的边界：脚手架 vs 拐杖

ADHD 大脑与 LLM 的同构命题，证据主要来自概念类比和工具案例，缺乏大规模实证（来源：ADHD × AI 的科学与研究前沿）。这意味我们必须诚实区分“脚手架”和“拐杖”。

脚手架是**可逐步撤除的临时支撑**，旨在发展内在能力。但多数工具（如 RescueTime）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。过度依赖 AI 可能阻碍内在执行功能发展，目前无长期研究证明脚手架可逐步撤除（来源：拐杖与脚手架）。

因此，我的判断是：**ChatGPT 作为重锚定工具，在短期内有效，但长期可能成为拐杖**。它补的是“验证循环”这块短板，但无法替代内在执行功能的发育。真正的解决方案应是“脚手架式”使用——逐步减少依赖，让大脑学会自己重锚定。

## 局限与争议

- **证据缺口**：多数工具缺乏针对 ADHD 的独立临床研究，效果多基于理论推断或用户主观报告（来源：AI 与 ADHD 的专注力）。
- **个体差异**：ADHD 亚型（注意力缺陷 vs 多动冲动）对工具反应不同，现有工具未充分个性化（来源：同上）。
- **多巴胺干预争议**：基于多巴胺的 AI 干预是否真正改善专注，证据矛盾（来源：矛盾与存疑）。

## 今天就能试的行动

1. **设置“重锚定提示”**：在 ChatGPT 中创建一个自定义指令，要求它在每次对话开始或每 15 分钟后，自动询问当前任务目标，并检查你的进度是否偏离。这模拟了 agent 的验证循环。
2. **结合 RescueTime 和手动复盘**：使用 RescueTime 自动追踪时间，每天花 5 分钟让 ChatGPT 分析你的时间数据，并指出哪些活动与目标一致。这相当于 agent 的“日志分析”。
3. **尝试 Focusmate 一次**：预约一个 25 分钟的虚拟同伴时段，专注于单一任务。之后用 ChatGPT 记录反思：这次外部问责如何影响了你的专注？这能帮你识别“身体在场效应”的威力。
4. **设计“脚手架撤除计划”**：与 ChatGPT 讨论如何逐步减少对工具的依赖。例如，从每天使用 RescueTime 过渡到每周一次，最终只保留关键提醒。这对应 agent 的“计划-执行”分离模式。

## 结论

ChatGPT 确实能像 agent harness 那样，为 ADHD 大脑补上重锚定与目标漂移的短板。但它的有效性取决于你能否主动使用它——它不是一个自动运行的调度层，而是一个需要你手动激活的验证工具。同构的思考框架让我们看清了问题的本质，但也提醒我们：**工具是脚手架，不是大脑的替代品**。真正的目标，是让 ADHD 大脑学会自己构建内在的执行层。

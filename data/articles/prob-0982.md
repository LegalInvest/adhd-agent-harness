# Structured 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Structured：它到底补上了哪一层执行功能？

你有没有过这样的体验：打开电脑准备写报告，却不知不觉刷了半小时短视频；或者读一本书，突然发现自己已经盯着同一页看了十分钟，脑子却神游天外。对 ADHD 来说，这种“目标漂移”是日常——大脑像一个高产的生成核心，创意源源不断，但缺乏一个可靠的执行调度层来锚定方向。

同样的问题，也困扰着开发 LLM agent 的工程师。一个设计良好的 agent 在长程任务中会逐渐偏离初始目标，小错误累积成灾难，输出变得不可预测（来源：AI Agent Systems: Architectures, Applications, and Evaluation）。解决方案是什么？重锚定 system prompt，用 harness 强制确定性控制流（来源：Planner-Executor Agentic Framework）。

Structured 之于 ADHD，就像重锚定 system prompt 之于 LLM——两者都在补偿同一种缺陷：生成核心与调度层的脱节。但有人用错了：他们把脚手架当成了拐杖，以为工具能替代内在能力，却忘了真正的目标是逐步撤除支撑。

## 同构的真相：生成核心 vs 执行调度层

ADHD 大脑与 LLM 在结构上同构：两者都是高产的生成核心，但缺乏可靠的执行调度层（来源：AI 与 ADHD 的专注力）。ADHD 的专注力问题本质是执行功能缺陷——工作记忆不足、任务启动困难、时间盲、上下文保持失败。而 LLM agent 的问题同样源于无状态性和非确定性：采样温度导致输出波动，小错误在长程任务中累积（来源：采样温度与表现波动）。

工具的作用，就是给这个生成核心套上 harness。对 ADHD 来说，外部工具补偿执行功能：RescueTime 自动记录时间使用，充当外部记忆，提升时间盲觉察（来源：RescueTime）；Focusmate 利用虚拟同伴提供实时问责，弥补任务启动的调度缺失（来源：Focusmate）；Brain.fm 通过神经锁相技术主动维持任务连贯性（来源：Brain.fm）。对 LLM agent 来说，harness 调用工具、管理文件、处理长对话，实现“计划-执行”分离（来源：工具使用与认知卸载）。

这不是空洞的类比。两边共享同一个工程逻辑：生成核心越强，调度层就越关键。ADHD 的高创造力与 LLM 的高生成能力，如果没有外部结构约束，都会陷入混乱。

## 脚手架 vs 拐杖：边界在哪里？

这里有一个核心争议：工具究竟是脚手架还是拐杖？脚手架设计为可逐步撤除的临时支撑，而拐杖则成为永久依赖（来源：矛盾与存疑）。

当前多数 ADHD 工具的设计并未考虑撤除机制。Goblin Tools 将模糊任务分解为小步骤（来源：工具使用与认知卸载），Saner.AI 提供温和提醒，但这些工具都默认长期使用。没有研究证明，长期依赖这些工具能逐步发展出内在的执行功能（来源：矛盾与存疑）。

LLM agent 领域也有类似问题。过度依赖确定性 harness 可能削弱 agent 的自主适应能力，让系统在面对新场景时僵化。工程实践强调“确定性状态转换”（来源：采样温度与表现波动），但这是为了可靠性，而非为了最终撤除 harness。

真正的脚手架应该具备两个特征：第一，它明确标注了“支撑点”——告诉你哪些能力是你自己的，哪些是工具提供的；第二，它提供了逐步撤除的路径——比如先从最简单的任务开始减少工具使用，再逐步挑战更复杂的场景。

## 证据与局限：我们到底知道什么？

现有证据主要来自工具机制与 ADHD 理论的对齐，而非独立临床研究（来源：AI 与 ADHD 的专注力）。Brain.fm 的神经锁相技术理论上可增强专注，但缺乏针对 ADHD 的随机对照试验（来源：Brain.fm）。Focusmate 的效果归因于身体在场效应，而非 AI 本身（来源：Focusmate）。RescueTime 和 Forest 在 ADHD 诊断分类中有研究支持，但作为专注工具的证据薄弱（来源：AI 与 ADHD 的专注力）。

多巴胺干预的有效性也存在争议。一些工具声称通过神经科学原理改善执行功能，暗示多巴胺调节有效，但现有证据不足（来源：矛盾与存疑）。ADHD 亚型（注意力缺陷 vs 多动冲动）对工具的反应不同，但现有工具未充分个性化（来源：AI 与 ADHD 的专注力）。

同构命题本身也缺乏大规模实证——它更多是一个理论框架，而非被验证的事实（来源：矛盾与存疑）。但框架的价值在于指导实践：它让我们看到，ADHD 和 LLM agent 面临的不是两个问题，而是同一个问题的不同表现。

## 今天就能试的 3 件事

1. **用 RescueTime 做一次“时间审计”**：安装后运行一周，不要手动调整分类。目的是观察你的注意力漂移模式，而不是追求“高效”。这比任何理论都更能让你看清自己的执行功能缺口。

2. **尝试 Focusmate 的“3 次配对”**：预约三个不同的时段，每次 25 分钟。注意体验差异：哪些配对让你更容易启动？哪些让你焦虑？记录下你的感受，而不是依赖工具本身的评分。

3. **为自己设计一个“撤除计划”**：选一个你依赖的工具（比如任务分解类），从下周一开始，每周减少一次使用，同时用最简单的纸笔替代。如果发现无法完成，说明这个工具对你还是拐杖——那就继续用，但保持觉察。

## 结语

Structured 工具不是答案，而是问题的一部分。它们揭示了 ADHD 大脑与 LLM agent 共享的深层缺陷：一个高产的生成核心，需要一个同样强大的调度层。但真正的进步不是找到更好的拐杖，而是学会在需要时使用脚手架，在有能力时撤除它。

对 ADHD 用户来说，这意味着工具应该帮助发展内在能力，而非替代它。对 agent 工程师来说，这意味着 harness 应该设计为可调节的，而非永久固定的。同构视角的价值，不在于给出确定的答案，而在于提出更好的问题。

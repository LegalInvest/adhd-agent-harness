# Claude 之于 ADHD，就像 human-in-the-loop 监督 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

## 问题：为什么你越依赖 AI，反而越容易半途而废？

如果你有 ADHD，你可能经历过这样的循环：下载一个 AI 工具，兴奋地用它分解任务、管理日程，但几周后工具吃灰，你又回到原点。如果你在做 Agentic Harness 工程，你可能也见过类似的场景：一个 LLM agent 在沙箱里跑得完美，一上生产环境就失控，因为你忘了给它加 human-in-the-loop 监督。

这两个问题看起来毫不相关，但底层结构完全一样——**它们都缺少一层可靠的执行调度层**。ADHD 大脑和 LLM 都是“高产但缺调度”的生成核心：它们能产生大量想法/文本，却无法稳定地执行、追踪、切换上下文。外部工具（AI 助手或 harness）本应充当这层调度，但如果用错了，它就成了永久拐杖，而不是可撤除的脚手架。

## 同构：ADHD 大脑与 LLM 共享的“无状态”困境

ADHD 的核心执行功能缺陷之一是工作记忆容量有限且不稳定，导致跨会话任务难以持续。典型表现为：每天需要重新回忆项目上下文，否则容易偏离轨道（来源：无状态与外部记忆）。为此，ADHD 个体常借助“第二大脑”工具，例如 Saner.AI，它通过本地记忆存储和快速检索，减少搜索循环和标签切换（来源：Saner.AI）。本质上，ADHD 大脑需要一个“永远在线”的外部记忆体来补偿内部工作记忆的波动（来源：无状态与外部记忆）。

LLM 侧完全对应：标准的 LLM 调用是无状态的，每次对话独立，模型不保留跨会话记忆（来源：无状态与外部记忆）。没有 harness 的 LLM 只是一个无状态的 API 调用（来源：无状态与外部记忆）。生产级 agent 系统通过外部记忆（如向量数据库）解决溢出问题（来源：无状态与外部记忆）。检索增强和工具调用将决策锚定在外部证据和持久状态上（来源：无状态与外部记忆）。

**证据对照**：ADHD 侧——Saner.AI 被列为最佳 ADHD 生产力 AI 工具之一，其知识回忆功能被特别强调（来源：Saner.AI）。LLM 侧——Harness 工程正是设计围绕 LLM 的脚手架，包括上下文传递、工具接口、规划工件、验证循环、记忆系统和沙箱（来源：拐杖与脚手架）。

## 错用：当工具成为拐杖而非脚手架

许多 ADHD 工具设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。例如 Goblin Tools 的 Magic ToDo 能自动将任何任务分解为小步骤（来源：Goblin Tools），这确实降低了启动门槛，但用户可能永远学不会自己分解任务。同样，Motion 的自动排程消除了“下一步该做什么”的决策负担（来源：Motion），但如果用户不输入所有任务，系统就失效。这些工具在“替代”执行功能，而不是“训练”它。

专家警告过度依赖风险：若工具替代了治疗或自我管理，可能造成依赖（来源：拐杖与脚手架）。真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：拐杖与脚手架）。

LLM/harness 侧类似：如果 harness 设计成完全接管决策，agent 就会变成黑箱，一旦环境变化就崩溃。Human-in-the-loop 监督不是简单的“人在旁边看”，而是设计可干预的检查点、可追溯的决策日志、可回退的状态。这和 ADHD 的脚手架原则一样：工具应该提供支撑，但最终执行和判断必须由人（或 agent 的调度层）完成。

## 判断：脚手架 vs 拐杖的边界在哪？

我的核心观点是：**脚手架是可逐步撤除的临时支撑，拐杖是永久替代**。对于 ADHD 用户，这意味着工具应该：
- 在初期提供强支撑（如任务分解、自动排程）
- 但逐步引导用户内化这些技能（如提示用户自己尝试分解后再看 AI 建议）
- 最终用户可以脱离工具完成相同任务

对于 Harness 工程师，这意味着：
- 脚手架在首次提示前组装代理（系统提示、工具模式、子代理注册）（来源：拐杖与脚手架）
- 而 harness 在运行时编排工具调度、上下文管理和安全执行（来源：拐杖与脚手架）
- 但必须保留人工覆盖机制，且日志可审计，以便在 agent 失误时回滚

## 局限与争议

我必须诚实指出：ADHD 大脑与 LLM 同构命题的证据主要来自概念类比和工具案例，缺乏大规模实证（来源：矛盾与存疑）。多巴胺干预的有效性也存在争议（来源：矛盾与存疑）。此外，目前缺乏针对 Goblin Tools、Saner.AI 等工具的独立随机对照试验（来源：Goblin Tools；Saner.AI）。这些工具的效果主要基于用户反馈，可能受安慰剂效应影响。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个你拖延的任务**，但在 AI 给出步骤后，自己再写一遍步骤，尝试理解分解逻辑。下次尝试先自己分解，再用 AI 对比。
2. **如果你用 Claude 或 ChatGPT 辅助工作**，每次对话结束时，手动总结关键决策和下一步，保存到外部笔记（如 Notion）。这模拟了 harness 的记忆系统，逐步减少对 AI 记忆的依赖。
3. **如果你是工程师**，为你当前的 agent 添加一个“人工介入点”：在关键决策前暂停，输出当前状态和候选行动，要求人工确认（或设置可回滚的沙箱）。这既是 human-in-the-loop，也是 ADHD 的“问责机制”。
4. **评估你使用的工具是否可撤除**：问自己“如果这个工具明天消失，我还能完成相同任务吗？”如果不能，说明你已过度依赖，需要开始练习脱离。

## 结语

Claude 之于 ADHD，就像 human-in-the-loop 监督之于 LLM——它们是同一问题的两种表现。但别用错了：不要用工具替代你的执行功能，而是用它来建造你自己的脚手架。无论是 ADHD 大脑还是 LLM agent，最可靠的系统永远是那些设计为最终可以独立运行的系统。

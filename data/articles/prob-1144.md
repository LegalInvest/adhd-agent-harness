# Claude 之于 ADHD，就像 human-in-the-loop 监督 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

## 问题：独自做事，为什么总是容易放弃？

如果你是 ADHD 患者，大概有过这样的体验：一个人坐在桌前，任务清单摊开，却像被钉在椅子上——启动不了。即使启动了，也容易在中途被一个念头带走，然后陷入“我为什么又没做完”的自责循环。

如果你是做 Agentic Harness 的工程师，你可能也见过类似的场景：一个 LLM agent 在没有人类监督的情况下，自主规划、执行，然后突然偏离轨道——生成幻觉、重复循环、或者干脆卡死。你不得不给它加上 human-in-the-loop 的检查点，让它每一步都“汇报”一下。

这两个场景，看似风马牛不相及，实则共享同一个底层问题：**一个高产的生成核心，缺少一个外部的执行调度层。**

## 同构：ADHD 大脑与 LLM 是同一类“生成核心”

ADHD 大脑并不缺乏想法和创造力。问题出在执行功能——工作记忆容量有限、时间感知困难、任务启动与切换障碍（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）。它就像一个强大的生成模型，但缺少一个“调度器”来管理上下文、分配注意力、监督进度。

LLM 也一样。没有 harness 的 LLM 只是一个无状态的 API 调用（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。它能在单次对话中生成惊艳的文本，但一旦任务需要跨步骤、跨会话，它就会迷失——因为它的工作记忆（上下文窗口）有限，且每次调用都是独立的。

更惊人的是，这种同构有神经科学层面的证据：Transformer 的自注意力机制在训练后，会发展出模仿人类前额叶-纹状体门控的操作（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。这意味着，LLM 和 ADHD 大脑在计算层面共享类似的“注意力门控”缺陷——都容易受到干扰，都需要外部结构来维持焦点。

## 解法：人在回路与身体在场

对于 LLM agent，工程界的标准解法是 **human-in-the-loop 监督**：在关键决策点插入人工检查，让 agent 输出计划后再执行，避免失控。这本质上是把“调度层”放在外部——由人类提供上下文、判断优先级、纠正偏差。

对于 ADHD 大脑，解法同样需要“外部调度”。这就是 AI 工具扮演的角色。

- **Goblin Tools** 的 Magic ToDo 功能，能把“整理房间”这样模糊的任务自动分解为“捡起地板上的衣服”“擦桌子”等具体步骤（来源：AI Tools for Kids with ADHD: A Complete Guide for Parents...）。它外化了任务分解这个执行功能，降低了启动门槛。
- **Saner.AI** 通过本地记忆存储和快速检索，减少 ADHD 用户频繁的“搜索循环”和标签切换（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。它相当于一个外部工作记忆，补偿了 ADHD 大脑跨会话记忆不稳定的问题。
- **Motion** 则自动根据任务优先级和截止时间动态调整日程，消除“下一步该做什么”的决策负担（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。它充当了一个外部规划器。

这些工具的共同点：它们不是替代你的大脑，而是**补上大脑缺失的那层执行调度**。就像 human-in-the-loop 监督不是替代 agent 的生成能力，而是给它一个方向。

## 但有人用错了：脚手架 vs 拐杖

关键在于边界。

真正的脚手架，应该帮助使用者“建造”，但使用者仍需自己“攀登”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。也就是说，工具应该逐渐教会你如何分解任务、如何管理时间，而不是永远替你代劳。

然而，当前许多 ADHD 用户把工具用成了拐杖：
- 依赖 Goblin Tools 分解所有任务，从不自己尝试规划；
- 依赖 Motion 安排每一天，一旦工具宕机就完全瘫痪；
- 依赖 Saner.AI 记住一切，自己的记忆能力反而退化。

这正是 wiki 资料中反复警告的风险：过度依赖 AI 工具可能削弱内在能力（来源：时间盲页面；任务启动页面）。对于 LLM agent 也一样：如果 human-in-the-loop 监督变成了每一步都要人类点头，那 agent 就永远学不会自主纠错，工程效率反而下降。

## 争议与局限

必须诚实指出：目前所有 AI 工具对 ADHD 的效果，证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑页面）。Goblin Tools、Saner.AI、Motion 都没有独立的临床研究支持。而且效果因人而异——有人觉得 Goblin Tools 是救命稻草，有人觉得分解步骤太机械，反而增加负担。

此外，工具设计者声称是“脚手架”，但实际使用中可能沦为“拐杖”（来源：矛盾与存疑页面）。这种矛盾目前缺乏具体指导——没有工具说明书告诉你“什么时候该减少依赖”。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个你拖延了一周的任务**：输入“写周报”或“整理邮箱”，看它输出的步骤。然后自己尝试——下次不用工具，凭记忆写出类似步骤。这是从“拐杖”到“脚手架”的第一步。
2. **给 Claude 一个“human-in-the-loop”提示**：当你用 Claude 写长文或做计划时，在开头加上“请先输出你的计划，我确认后再执行”。这相当于给 LLM 加上外部监督层，也让你自己保持主动。
3. **用 Motion 规划明天，但故意留 30 分钟空白**：不安排任务，用来练习“自主决定下一步”。这能训练你的执行功能，而不是完全交给工具。
4. **反思：你是在“建造”还是“被代劳”？** 每次使用工具后，问自己：如果明天没有这个工具，我还能做这件事吗？如果答案是否定的，你可能已经过度依赖了。

## 结语

ADHD 大脑和 LLM agent 是同构的：都是强大的生成核心，但缺少执行调度层。Claude 之于 ADHD，就像 human-in-the-loop 监督之于 LLM——它们都是外部脚手架。但脚手架的目的是让你最终能自己站立，而不是永远扶着。用对，它们是解放；用错，它们是枷锁。

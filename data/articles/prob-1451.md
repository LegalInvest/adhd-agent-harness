# 如果把 ADHD 大脑当成生成核心，Tiimo 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Tiimo 对 ADHD 的真正价值。

你有没有这样的经历：打开电脑准备写报告，结果十分钟后发现自己正在刷短视频，完全忘了最初的目标？或者，你明明在做一个项目，却突然沉迷于某个细节，等到回过神来，已经偏离主线三个小时——这就是 ADHD 大脑最典型的两种失控：分心走神与超聚焦跑偏。

如果你是一位 Agent 工程师，你可能会心一笑：这不就是长程任务里 Agent 的目标漂移吗？一个 LLM 在生成过程中，如果没有定期的 system prompt 重锚定，很容易从“写一份市场分析报告”滑向“详细描述某个竞品的历史”，最终产出与用户意图渐行渐远。

这两个场景看似风马牛不相及，但本文要论证一个核心判断：**ADHD 大脑与 LLM/Agent 是同一类“高产但缺执行调度层的生成核心”，而 Tiimo 这样的工具，其底层结构与 Agent 工程中的“重锚定”机制同构——它能否成为 ADHD 大脑的 harness，取决于我们如何定义“脚手架”与“拐杖”的边界。**

## 同构：生成核心与调度层

ADHD 大脑的本质是什么？它不是一个“坏掉”的大脑，而是一个**生成能力极强、但执行调度层薄弱**的大脑。ADHD 个体往往有惊人的创造力、模式识别能力和超聚焦能力——曾国藩在日记里骂自己“无恒”“浮躁”，读书“他人目下二三行，余或疾读不能终一行”，但他同时能平定太平天国、发起洋务运动、留下影响深远的家书。孔子精力旺盛、思维跳跃，《论语》全是场景化语录，但他用“克己复礼”和“吾日三省吾身”构建了一套外在的行为边界，最终成为万世师表。

他们的共同点是什么？都**为自己设计了外部调度系统**。曾国藩的“日课十二条”——黎明即起、读书不二、谨言、写日记反省——本质上是一套定时 re-grounding 的 protocol，就像 Agent 工程中每 N 步向 system prompt 注入一次原始目标。孔子的“礼”则是一套静态的约束规则，类似于 Agent 的 guardrails。

LLM/Agent 的困境与此完全同构：LLM 是一个强大的生成核心，但它没有内置的“目标记忆”——它不知道自己在第 1000 个 token 时是否还在回答最初的问题。因此，Agent 工程需要外部调度器（如 LangChain 的 Agent Executor）来定期检查进度、重读 system prompt、修正轨迹。

## Tiimo：重锚定工具的结构分析

Tiimo 是什么？它是一款专为神经多样性人群设计的视觉化日程应用。它的核心功能包括：
- **视觉化时间块**：用颜色和图标代表不同活动，降低认知负荷。
- **定时提醒**：在任务切换点发出通知，帮助用户“回锚”。
- **可预设的日常流程**：将重复性任务（如起床、吃药、工作）固化为模板。

从结构上看，Tiimo 与 Agent 工程中的“重锚定”机制高度对应：
- **视觉化日程 ↔ 状态可视化**：Agent 需要 dashboard 显示当前进度和目标，Tiimo 用可视化时间块让 ADHD 大脑“看到”时间流向，补偿时间盲（来源：执行功能障碍）。
- **定时提醒 ↔ 定期 re-grounding**：Agent 每 N 步重新注入 system prompt，Tiimo 在任务切换点提醒用户“现在该做什么”，相当于外部调度器的 tick。
- **预设流程 ↔ 工作流模板**：Agent 可以复用 prompt 模板，Tiimo 允许用户保存“早晨流程”，减少每次决策的认知负荷。

这种同构意味着：**Tiimo 本质上是一个“外部执行功能层”**（来源：AI 与 ADHD 的专注力），它不改变 ADHD 大脑的生成能力，而是补偿其调度缺陷。

## 证据与局限：脚手架还是拐杖？

现有工具的证据支持这一思路：RescueTime 通过自动时间追踪补偿工作记忆缺陷，让用户看到“时间到底去了哪里”（来源：Revolutionizing ADHD Management with AI Assistants）；Focusmate 通过虚拟身体在场提供外部问责，类似于 Agent 的 human-in-the-loop（来源：ADHD科技行业深度研究）；Brain.fm 用神经锁相技术影响大脑状态，但缺乏 ADHD 专项临床证据（来源：Brain.fm）。

Tiimo 本身尚未有大规模随机对照试验，但其设计逻辑与这些工具一致：通过外部结构补偿执行功能缺陷。然而，这里有一个关键争议：**Tiimo 是脚手架（逐步内化）还是拐杖（永久依赖）？**（来源：矛盾与存疑）

曾国藩的“日课”最终内化为他的习惯，但很多人使用日程应用后，一旦离开工具就彻底失控。同样，Agent 工程中，过度依赖外部调度器可能导致 LLM 失去自主规划能力。这提醒我们：**Tiimo 的价值在于“重锚定”的频率和方式，而非工具本身。** 如果用户只是被动接收提醒而不主动内化目标，工具就会沦为拐杖。

## 今天就能试的行动

1. **用 Tiimo 的“定时提醒”模拟 Agent 的 re-grounding**：每 25 分钟设置一个提醒，内容不是“休息”，而是“你当前的目标是什么？”。这相当于 Agent 的 system prompt 注入。
2. **创建“日课”模板**：像曾国藩一样，将每天固定时段（如起床后、午饭后）固化为模板，减少决策疲劳。这与 Agent 的“工作流模板”同构。
3. **结合 Focusmate 进行“身体加倍”**：在 Tiimo 的日程中嵌入 Focusmate 时段，用社会压力强化重锚定效果。这相当于 Agent 的 human-in-the-loop 验证。
4. **每周回顾时间数据**：使用 RescueTime 或 Tiimo 的日志，分析目标漂移模式，就像 Agent 工程师分析 trace 日志一样。

## 诚实面对争议

本文的核心论点——ADHD 大脑与 LLM 的同构——目前仍是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。Tiimo 的效果也缺乏独立临床验证。但同构视角的价值在于：它为 ADHD 自我管理提供了一个全新的工程化框架，让用户不再自责“不够努力”，而是像调试 Agent 一样调试自己的执行系统。

至于 Tiimo 能否成为重锚定工具，答案取决于使用方式：如果你只是被动接收提醒，它是拐杖；如果你主动设计提醒内容、反思漂移模式，它就是脚手架。

毕竟，曾国藩写了三十年日记骂自己，才练出“结硬寨打呆仗”的功夫。工具从来不是答案，**用工具的意图才是**。

# ADHD 程序员视角：为什么「治好 ADHD 的执行功能（计划/组织/启动）先天偏弱」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 的编排/记忆/工具层就是「外部执行功能」——同一套 harness 思路，两边都成立。

你是一名 ADHD 程序员，早晨打开电脑，脑子里已经有一整套架构图，却卡在「到底先写哪一行」；你也是一位 Agentic Harness 工程师，看着 LLM 滔滔不绝地生成代码，却偏离了最初任务，越跑越远。这两个场景让人抓狂的，其实是同一个 bug：一个强大的生成核心，没有可靠的外部执行调度层。

## 同一类生成核心：高产，却缺调度层

ADHD 不是「懒」或「不努力」。神经科学研究发现，ADHD 患者的前额叶执行功能网络激活不足，导致计划、组织、任务启动、工作记忆等能力效率低下（来源：主题综述）。认知心理学也反复验证，ADHD 的核心困难集中在执行功能缺陷：时间感知异常（时间盲）、任务切换失控、工作记忆不稳定（来源：执行功能障碍）。这些问题的共同点是：大脑拥有强大的联想、创造与超聚焦能力，但缺少一个稳定的调度层来决定「现在做什么、下一步做什么、为什么做」。

LLM 与 agent 的困境几乎同构。它们也是高产生成核心，但本质无状态、缺乏跨会话记忆，也没有内置的任务调度层。上下文一旦膨胀，推理会退化；目标一旦模糊，输出会跑飞。AI 工程领域因此发展出 ReAct、Chain-of-Thought 等脚手架，通过显式分解、工具调用与外部记忆来维持任务连续性，被证明能提升任务完成率（来源：主题综述）。

所以，ADHD 与 LLM 不是比喻关系，而是同一工程问题的两种表现：如何给强大的生成核心，套上一个稳定、可靠、可撤销的外部执行功能层。

## 两边的 harness 长同一张架构图

这个「外部执行功能层」在 ADHD 侧和 agent 侧可以一一映射。

**任务分解 ↔ 子目标规划。** ADHD 患者常因任务庞大而无法启动，而 Goblin Tools 的 Magic ToDo 能把「清理厨房」这种模糊任务分解为可执行步骤，并让用户调节粒度（来源：Goblin Tools）。在 agent 侧，这对应 ReAct 或分层规划器把复杂目标拆解为子任务，避免模型一次性生成不可控的长链路。

**外部记忆 ↔ 记忆/知识库。** ADHD 工作记忆不稳定，导致信息散落在多个标签页和搜索循环中；Saner.AI 通过本地记忆与快速知识回忆，减少标签切换，让用户不必在脑子里「缓存」所有上下文（来源：Saner.AI）。LLM 则通过向量存储、RAG、持久化日志来弥补无状态缺陷，避免每次对话都从零开始。

**时间盲 ↔ 动态调度。** Motion 等工具自动根据任务优先级、截止日期和可用时间重建每日计划，并提前警告延期风险，直接减轻了 ADHD 的「下一步该做什么」决策负担（来源：Motion）。agent 的 orchestrator 或 scheduler 则负责控制调用节奏、分配资源、防止任务无限蔓延。

这套架构的共同名字叫 harness：不是替代核心，而是把核心拴在轨道上。

## 一个前现代的 harness 案例：曾国藩的日课与硬寨

历史人物中，曾国藩的 harness 系统最为典型。他年轻时浮躁坐不住，读书慢，情绪不稳，一生靠日记反复骂自己「无恒」「浮躁」；他的应对不是等「心性」变好，而是建立外部系统：日课十二条（黎明即起、读书不二、谨言、写日记反省），以及军事上的「结硬寨、打呆仗」，用最笨、最稳的方式抵消冲动（来源：人物案例·曾国藩）。

这个系统与 LLM harness 的对应非常清晰：日课是每日定时的 re-grounding，让注意力重新锚定目标；日记是持久化的外部记忆与决策日志，替代不稳定的工作记忆；结硬寨打呆仗则是约束式 rollout——每走一步先验证据点，再推进下一步，而不是让生成核心（他的冲动与才华）自由奔腾。曾国藩没有「治愈」自己的注意力问题，他通过外部执行层把它管住了。

## 脚手架 vs 拐杖：边界在哪里？

同构框架最有价值的地方，也是它最危险的地方。它容易让人把「靠工具」当成「被工具替代」。拐杖与脚手架的界限在于：harness 应该帮助用户完成那些因为执行功能缺陷而无法内部完成的事情，同时保留并训练其他能力；一旦所有任务都外包，内部调度能力可能进一步退化（来源：全局矛盾与存疑）。

更重要的是，同构目前仍是一种理论类比，而非经过神经算法等价验证的事实（来源：全局矛盾与存疑）。Motion 等工具虽在 ADHD 社区中被推荐，但缺乏独立临床验证；部分工具页面强调益处，却很少讨论依赖风险。我们在借用这一框架时，必须把它当作工程假设，而不是科学定论。

我的判断是：与其追求「治愈」执行功能，不如把外部执行功能层设计得足够轻薄、可迁移、可撤销。对 ADHD 人，这意味着一套能随身携带的 ritual + memory + scheduler；对 agent 工程师，这意味着把规划、记忆、工具调用拆成独立、可插拔、可审计的模块。两者共享同一组接口：分解、记忆、调度、反馈、重置。

## 今天就能试的 4 个动作

1. **分解一个卡住的任务**：用 Goblin Tools 的 Magic ToDo 把一件拖延了三天的事拆成 5 分钟能启动的第一步，体会「任务变小」带来的启动感。
2. **建立单一外部记忆源**：选一款工具（如 Saner.AI）或一个笔记本，把今天所有「待会儿再想」的念头丢进去，减少大脑缓存压力。
3. **给 agent 加一道「硬寨」**：在 ReAct 循环里插入一个显式验证步骤，让模型每完成一个子任务就回头检查是否仍符合最初目标，防止跑飞。
4. **写 7 天决策日志**：每天睡前用一句话记录「今天做了什么、为什么做、明天第一件事是什么」，把它当作你的 re-grounding ritual。

ADHD 和 LLM 都不会因为「更努力」或「更多参数」就自动拥有执行功能。它们需要一个精心设计的 harness——这套 harness，既是你桌面的 AI 工具，也是你代码里的 agent 架构。

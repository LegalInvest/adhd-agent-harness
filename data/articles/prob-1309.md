# Tiimo 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Tiimo：它到底补上了哪一层执行功能？

## 问题：为什么 Tiimo 对某些人有用，对另一些人却是摆设？

ADHD 群体最常见的抱怨是：“我知道要做什么，但就是动不了。” 这不是懒惰，而是执行功能缺陷——任务启动困难、时间盲、工作记忆不稳定（来源：概念页·任务启动）。与此同时，LLM/Agent 工程师也在头疼：大模型能生成高质量内容，但无法自主完成长任务，需要 planner-executor 循环来分解、调度、执行。这两类人面对的其实是同一个问题：**一个高产的生成核心，缺少一个可靠的调度层。**

Tiimo 作为专为神经多样性设计的视觉化日程应用，试图补上这个调度层：它把时间变成可见的色块，用结构化的日程对抗时间盲（来源：工具页·Tiimo）。但实测中，很多人发现 Tiimo 只是“好看的日历”——它告诉你 9:00-10:00 该写报告，但你还是不知道如何开始。问题出在哪？

## 同构视角：ADHD 大脑和 LLM 共享同一个架构缺陷

ADHD 大脑与 LLM 在结构上高度同构：两者都是强大的生成核心，但缺乏内置的调度层（来源：主题综述·AI 与 ADHD 的时间管理）。LLM 需要提示工程、agent 框架来规划步骤；ADHD 大脑需要外部执行功能层来启动任务（来源：概念页·任务启动）。这个外部层至少包含三个子功能：

1. **任务分解**：将大目标拆成可执行的小步骤。
2. **动态调度**：根据优先级和能量水平调整顺序。
3. **实时提醒**：在正确的时间触发行动。

Tiimo 主要提供第 3 项（视觉化提醒），对第 2 项有部分支持（自动重新安排），但**几乎不处理第 1 项**。这就是它的致命短板。

## 案例：孔子和毛泽东的“外部调度层”

历史上有两位典型的 ADHD 人物，他们用不同的方式构建了自己的调度层。

**孔子**精力旺盛、冲动、思维跳跃，但他用“克己复礼”作为行为边界，每日“吾日三省吾身”——这相当于一个**定时 re-grounding 机制**，类似 LLM agent 中每轮执行后检查是否偏离目标。他还依赖“礼”作为外部规范，相当于一个**硬编码的规则系统**，减少决策成本。

**毛泽东**则建立了“批评与自我批评”的民主生活会制度，用集体决策制衡自己的冲动；同时坚持“没有调查就没有发言权”，用调查研究作为外部事实锚点。这相当于 LLM agent 中的 **human-in-the-loop 验证**和**检索增强生成（RAG）**——用外部数据纠正模型幻觉。

这两位都没有用 Tiimo，但他们成功的关键是**主动设计了任务分解和调度机制**，而不仅仅是可视化时间。

## 证据：AI 工具如何真正帮助任务启动？

真正有效的 ADHD 工具必须处理任务分解。例如，**Goblin Tools** 的“魔法待办”能将复杂任务拆成小步骤（来源：概念页·任务启动）；**Motion** 和 **Reclaim.ai** 则通过 AI 自动排程，动态调整日程，减少决策负担（来源：工具页·Motion、Reclaim.ai）。这些工具补上了 Tiimo 缺失的环节。

但 Tiimo 并非无用。对于时间盲严重的用户，视觉化时间线能显著提升时间感知（来源：主题综述·AI 与 ADHD 的时间管理）。问题在于，**仅靠视觉化不足以启动任务**——你还需要知道“第一步是什么”。

## 核心观点：脚手架 vs 拐杖的边界

我的判断是：**Tiimo 是一个优秀的视觉化工具，但它被误用为全能方案。** 它的定位应该是“调度层的视觉化界面”，而非调度层本身。真正的调度层必须包含任务分解和动态调整，就像 LLM agent 不能只靠一个漂亮的 UI 来完成任务。

这引出一个更深的争议：**过度依赖风险**。如果用户长期依赖 Tiimo 的视觉化时间线而不发展内在的时间感知能力，它就从“脚手架”退化为“永久拐杖”（来源：主题综述·矛盾与存疑）。目前没有长期研究评估这种依赖的后果，但临床直觉告诉我们：任何外部工具都应在使用中逐步内化技能，而非替代。

## 今天就能试的行动

1. **结合 Tiimo 与微任务分解**：每天早晨用 Tiimo 设定时间块，但每个时间块内只放一个“两分钟就能开始”的任务（如“打开文档写第一句”），而不是“写报告”。
2. **用 Goblin Tools 补充分解**：遇到复杂任务时，先用 Goblin Tools 拆成小步，再导入 Tiimo 的日程。
3. **设置“re-grounding”提醒**：像孔子的“三省吾身”一样，在 Tiimo 中设置每 90 分钟一次的检查点，问自己“当前任务是否偏离目标？”
4. **尝试 Motion 或 Reclaim.ai**：如果 Tiimo 无法解决启动问题，换用自动排程工具，让 AI 替你决定“下一步做什么”。

## 诚实局限

本文的核心类比——ADHD 大脑与 LLM 同构——目前仅是理论假设，缺乏实证基础（来源：主题综述·矛盾与存疑）。Tiimo 等工具的效果也缺乏独立临床验证，可能受安慰剂效应影响。此外，个体差异巨大：有人对视觉化反应良好，有人更需自动调度。没有万能方案，只有不断试错。

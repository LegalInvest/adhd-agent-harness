# ADHD 创业者如何用「给生成核心套 harness」的思路应对把想法落地执行里的执行功能（计划/组织/启动）先天偏弱？

> 把 LLM 工程里的「agent 的编排/记忆/工具层就是「外部执行功能」」迁移到ADHD 创业者的把想法落地执行场景，双域证据作答。

## 问题：从灵感到行动，中间塌陷的“执行鸿沟”

你有一个绝妙的创业想法，在脑海里已经推演了十遍，细节清晰得仿佛能摸到。但当你坐到电脑前，准备开始写第一行代码、第一封邮件、第一页商业计划书时，大脑却像被按下了暂停键。你刷了半小时社交媒体，整理了三遍桌面，然后发现——时间没了，任务没动。

这不是懒，不是拖延，而是 ADHD 创业者最熟悉的“执行功能塌陷”：**计划、组织、启动**这三项能力，在你最需要它们的时候，偏偏掉线了。传统的时间管理方法——待办清单、日历、番茄钟——往往在几天内就失效，因为“most planning advice isn’t designed for us”（来源：ADHD Productivity Hack: Plan 2025 Using AI (Step-by-Step)）。Planners and sticky notes worked for a week before falling apart（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

但问题不在于你，而在于你试图用“原生执行系统”去运行一个“生成核心”。ADHD 大脑本质上是一个**高产但缺乏可靠执行调度层的生成核心**（来源：ADHD × AI 的科学与研究前沿）。你的默认模式网络过度活跃，想法不断涌现；但前额叶的执行控制不足，无法将这些想法有序地调度、分解、推进。这就像一个大语言模型（LLM），拥有强大的生成能力，却缺少一个稳定的 agent harness 来管理上下文、调用工具、规划步骤。

那么，如果 LLM 工程师已经找到了解决方案——通过“harness”为生成核心套上外部执行层——ADHD 创业者能否直接迁移这套思路？

## 答案：外部执行功能层——同构的解决方案

核心观点是：**ADHD 大脑与 LLM 是同一类“生成核心”，两者的解法在结构上同构**（来源：ADHD × AI 的科学与研究前沿）。LLM agent 通过“harness”调用工具、管理文件、处理长对话（来源：What is an agent harness in the context of large-language models?），这正是 ADHD 大脑需要的“外部执行功能层”——一套由工具、编排和记忆组成的脚手架，将想法转化为行动。

### ADHD 侧证据：工具即外部执行层

ADHD 创业者已经在使用这类工具，只是未必意识到它们就是“harness”。**Goblin Tools** 的 Magic ToDo 功能能自动将模糊的任务（如“整理房间”）分解为一系列小步骤（如“捡起地板上的衣服”“擦桌子”），从而降低启动门槛（来源：Goblin Tools）。用户反馈称它“将压倒性的事情变成一系列不压倒性的事情”（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...)）。这正是“任务分解”这一执行功能的卸载。

**Saner.AI** 则针对工作记忆不足，通过 AI 实现知识回忆和本地记忆存储，减少搜索循环和标签切换（来源：Saner.AI）。对于 ADHD 创业者，这意味着不再需要记住所有细节，而是可以像 LLM 依赖向量数据库一样，依赖外部记忆系统来维持连续性（来源：无状态与外部记忆）。

**Motion** 更进一步，作为自适应规划器，它能自动根据任务、会议和截止日期创建并动态调整每日计划，在日程被打乱时自动重新安排（来源：Motion）。这直接对应 LLM agent 的“规划循环”——动态调整步骤以应对变化。ADHD 创业者不再需要自己决策“下一步该做什么”，因为 Motion 的 AI 已经替你做了（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...)）。

### LLM/harness 侧证据：Agent 架构即执行层

在 LLM 工程中，现代 agent 系统通过“组合多个模型、检索器和工具”来达成 SOTA 结果（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned），而非依赖单一模型调用。Harness 工程的核心是设计“上下文传递、工具接口、规划工件、验证循环”（来源：规划循环与任务分解）。这正是 ADHD 大脑缺失的调度层：一个将生成核心（想法）与外部世界（行动）连接起来的编排系统。

例如，Goblin Tools 的任务分解模式可类比 agent 的“多步推理+工具调用”模式；Saner.AI 的知识回忆对应 LLM 的检索增强生成（RAG）；Motion 的动态规划对应 agent 的“规划-执行-验证”循环。两边都在做同一件事：**将不擅长的精确状态外包给外部系统**（来源：工具使用与认知卸载）。

## 矛盾与局限：脚手架还是拐杖？

尽管同构视角具有强大的解释力，我们必须诚实面对争议。**最大的风险是过度依赖**：外部脚手架可能成为永久拐杖，削弱内在执行功能的发展（来源：矛盾与存疑）。目前证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑）。例如，Goblin Tools 的效果基于用户评价，尚无独立随机对照试验（来源：Goblin Tools）。Motion 的有效性同样缺乏临床研究（来源：Motion）。

此外，个体差异巨大：ADHD 的异质性（注意力主导型 vs. 冲动主导型）与 LLM 的架构差异（如不同模型）是否影响同构的普适性？目前研究样本有限（来源：矛盾与存疑）。工具设计者声称是“脚手架”，但实际使用中可能沦为“拐杖”（来源：矛盾与存疑）。关键在于：工具应该是**补偿**而非**替代**，是帮你启动而非替你做所有事。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个让你卡壳的任务**：输入一个模糊的目标（如“准备投资人 pitch”），观察 AI 如何将其拆解为步骤。然后，只做第一步。这相当于给你的 LLM 核心一个“工具调用”的起点。

2. **用 Saner.AI 或类似工具建立外部记忆**：将你常忘记的细节（如客户偏好、代码片段）存入一个可快速检索的系统。这对应 LLM 的向量数据库，减少你大脑的工作记忆负载。

3. **尝试 Motion 或 Tiimo 的动态规划**：放弃手动排程，让 AI 根据你的任务和截止日期自动安排一天。当计划被打乱时，允许它重新调整，而不是你亲自重排。这相当于给大脑一个“规划循环”的自动化模块。

4. **反思“脚手架 vs 拐杖”**：每周检查一次，你依赖的工具是否在帮你完成原本做不到的事，还是让你放弃了原本能做到的事？如果是后者，尝试减少使用，或切换为更轻量的方案。

ADHD 创业者不需要“修复”自己的大脑，只需要像 LLM 工程师一样，为生成核心套上精心设计的 harness。你的想法是宝贵的，但执行功能不是你的强项——这没关系。把执行外包给工具，把创造留给自己。

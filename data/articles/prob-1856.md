# ADHD 家长如何用「给生成核心套 harness」的思路应对陪伴 ADHD 孩子里的不擅长精确、重复、状态维护类任务？

> 把 LLM 工程里的「agent 通过 function calling 把精确计算外包给工具」迁移到ADHD 家长的陪伴 ADHD 孩子场景，双域证据作答。

如果你是一位 ADHD 家长，每天陪 ADHD 孩子，你可能比任何人都熟悉这种窒息感：书包要检查几遍、作业清单要盯、情绪崩溃要接、吃药时间要记、日程变动要同步——这些任务既不宏大，也不 creative，却极度依赖精确、重复和状态维护。而这正是 ADHD 大脑最不擅长的领域。执行功能障碍研究显示，ADHD 的核心困难在于计划、组织、时间管理与任务启动，而非缺乏努力（来源：执行功能障碍）。情绪失调又进一步放大了挫败、易怒和冲突，让「陪孩子」变成一场高能耗的执行功能消耗战（来源：情绪失调）。

但问题不在「你不够认真」。更准确的描述是：ADHD 家长和孩子的大脑都是**高产生成核心**，联想丰富、反应快、能突发奇想，却缺少一个稳定可靠的**执行调度层**来筛选、排序、启动和坚持任务。LLM 领域有一个完全同构的困境：大模型生成能力极强，却在长程上下文、目标保持与执行调度上天然薄弱（来源：ADHD 的 AI 工具全景）。所以工程师不会指望模型自己算对、自己记住、自己坚持到底，而是给它套一层 harness——通过 function calling 把精确计算、数据库查询、代码执行外包给工具；通过记忆模块补状态；通过规划层把大目标拆成子任务。ADHD 的陪伴问题，本质上是同一份工程问题在人脑上的投影。

## 同构：生成核心都需要外部 harness

把 ADHD 大脑与 LLM 并置，不是诗意比喻，而是一种结构性的观察。两者底层都拥有高产生成核心，却都依赖外部脚手架来把生成能力约束到目标轨道上（来源：ADHD 的 AI 工具全景）。LLM 会幻觉、会忘记上下文、会在多步任务中跑偏；ADHD 大脑会时间盲、工作记忆波动、任务启动困难。两边解决思路一致：不是让生成核心变「正常」，而是围绕它搭建一个可维护的**外部执行功能层**。

在 LLM 工程里，这层 harness 的典型做法包括：
- 用 function calling 把数学、搜索、数据库操作交给确定性工具；
- 用 agent scaffolding 给模型装配记忆、规划、工具调用和反思模块；
- 用系统提示、检查清单和多轮 review 来 re-ground 目标。

在 ADHD 陪伴场景里，对应的做法就是：把精确、重复、状态维护类任务从人脑里搬出来，交给工具、流程和人际机制。家长不是要去变成一台完美调度器——家长自己也是生成核心，只会一起崩溃。家长要做的，是**为自己和孩子设计一个 harness**。

## 历史 harness：古人早就在做 function calling

这种 harness 并不是新技术。孔子的生活系统就是一套典型 harness：他身高一米九、精力旺盛、周游列国十四年坐不住，冲动爱骂人，思维跳跃，《论语》全是场景化语录。他给自己套的约束是「克己复礼」——用外在的「礼」作为行为边界，再配合「吾日三省吾身」的每日反省，直到七十岁才做到「从心所欲不逾矩」（来源：孔子 harness 自我管理系统）。这对应到 LLM 工程里，就是**system prompt 加定时 re-grounding**：用一套显式规则框住生成核心，并周期性地把脱轨的状态拉回目标。

毛泽东则是另一类 harness：他小时候是问题儿童，爱读闲书、不爱四书五经，一辈子在动，思维极度跳跃，讲话充满比喻。他的策略是**把外部监督和集体决策当作执行调度器**——批评与自我批评、民主生活会、民主集中制、没有调查就没有发言权。这些机制本质上是在生成核心之外，建立了一个「外部 reviewer + 决策聚合器」（来源：毛泽东 harness 自我管理系统）。对应到 LLM 里，就是 multi-agent review、人类在环（human-in-the-loop）和投票/决策机制。

两位人物的共同点是：他们都没有靠「意志力」去消灭 ADHD 特质，而是**用外部结构和他人反馈来承接执行功能**。对 ADHD 家长来说，这意味着你要允许自己和孩子都不擅长精确维护，但绝不允许这些任务裸奔在脑子里。

## 家长的 harness 设计：把任务外包给工具

具体怎么做？wiki 资料里已有不少可被 ADHD 家长直接调用的「function」：

- **任务拆解与启动**：Goblin Tools 的 Magic ToDo 可以把模糊任务（如「清理书包」「准备明天」）自动拆成可执行小步骤，还能调节分解粒度，降低启动门槛（来源：Goblin Tools）。这相当于给家长和孩子自动做「子任务规划」。
- **单一指令触发复杂流程**：Lex 允许用户用一条指令触发多步骤任务序列，减少 ADHD 大脑在切换和决策上的认知摩擦（来源：Lex）。家长可以把它当作「 Morning Routine Agent」或「睡前流程 Agent」。
- **外部记忆与状态维护**：Saner.AI 被定位为 AI 第二大脑，通过本地记忆和知识回忆减少标签切换和搜索循环（来源：Saner.AI）。家长可以用它统一捕获待办、截止日期、孩子的情绪触发点，避免依赖不稳定的工作记忆。
- **捕获与反思**：Otter.ai 实时转录课堂和会议，把信息捕获外包；Reflect 通过结构化提问帮助 ADHD 个体在自发反思困难时提炼经验（来源：Otter.ai；Reflect）。家长可以和孩子在睡前用三个固定问题做「日省」，把反思变成外部流程，而不是靠灵感。

这些工具不是让大脑变「正常」，而是像 LLM 的 function calling 一样，把生成核心不擅长的计算、记忆、状态维护交给外部确定性模块。ChatGPT 和 Claude 也被成人 ADHD 用户用作外部执行功能工具，前者偏对话执行，后者偏结构化推理（来源：ADHD 的 AI 工具全景）。家长可以和孩子共同使用，把「大人单向催促」变成「系统双向提醒」。

## 脚手架 vs 拐杖：关键的边界

必须区分 harness 是**脚手架**还是**拐杖**。脚手架是临时的外部结构，目标是逐步内化能力或把能力迁移到更轻量的工具；拐杖则可能成为永久依赖，甚至让原本就弱的执行调度层进一步退化。当前 ADHD 与 AI 工具领域已经意识到这个张力：AI 作为外部执行功能层补偿缺陷，但过度依赖可能导致能力退化（来源：矛盾与存疑）。长期依赖风险缺乏纵向研究，所以家长应定期审视：这个工具是在减少内耗，还是在替代孩子本可以练习的技能？

一个比较稳妥的原则是：**先把最消耗亲子关系、最容易冲突的任务外包出去**（如时间提醒、清单检查、情绪降温脚本），但保留共同复盘和决策的环节。工具负责「执行维护」，人负责「意义连接」。

## 局限：同构是框架，不是定律

诚实地说，这一思路有明确局限。ADHD 大脑与 LLM 的同构目前主要是**理论框架**，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。AI 工具对任务启动、长期效果、认知负荷净收益的研究也还不够充分，个体差异很大，有些工具学习成本本身就会增加认知负担（来源：矛盾与存疑）。因此，不要把它当成医疗建议，也不要一次性堆叠太多工具。

## 今天就能试的四条行动

1. **挑一个最痛的「重复维护任务」，交给一个工具 function**。例如用 Goblin Tools 把「每天睡前准备」拆成清单，或用 Saner.AI 把孩子的日程和情绪触发点统一存储。只选一个，先用两周。
2. **建立「触发-执行-复盘」的最小闭环**。像 agent 的循环一样：固定时间触发（闹钟/视觉提示）、按清单执行、睡前用 Reflect 式三问复盘。不要靠脑子记状态。
3. **把家长自己也纳入 harness**。家长也是 ADHD 生成核心，需要外部提醒和伴侣/亲友的 review。可以每周一次家庭「民主生活会」：每人说一件本周做得好的、一件需要调整的——这正是毛泽东 harness 的家庭版。
4. **与孩子共设计，而不是单向管理**。让孩子参与选择工具和制定规则，他是这个 agent 系统的共同开发者。自主性会提升多巴胺动机，减少对抗（来源：ADHD 的 AI 工具全景）。

## 结语

ADHD 家长不需要成为完美调度器。陪 ADHD 孩子的核心工程问题，和给 LLM 套 harness 一样：承认生成核心的高产与冲动，然后用外部工具、规则和人际机制承接执行调度。精确、重复、状态维护类任务本来就属于 function，不属于意志力。把该外包的外包，把该连接的连接，家长和孩子才能从崩溃的循环里走出来。

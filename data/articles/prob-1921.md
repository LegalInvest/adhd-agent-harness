# ADHD 程序员视角：为什么「治好 ADHD 的执行功能（计划/组织/启动）先天偏弱」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 的编排/记忆/工具层就是「外部执行功能」——同一套 harness 思路，两边都成立。

## 问题：两个世界，同一种卡住

如果你是一名 ADHD 程序员，你大概熟悉这种场景：脑子里同时跑着三个项目、五个想法，但坐在电脑前就是没法开始写第一行代码。任务像一团乱麻，你明明知道优先级，却卡在“启动”这一步。

如果你是一名在做 Agentic Harness 的工程师，你大概也熟悉这种场景：你调用的 LLM 模型能生成惊艳的文本，但一旦让它自主完成一个多步骤任务，它就开始“跑飞”——忘记上下文、偏离目标、输出不可控。

这两个场景看似风马牛不相及，但本质是同一道工程题：一个高产生成核心缺乏可靠的执行调度层。ADHD 大脑和 LLM 在结构上同构——两者都是“高产但缺执行调度层的生成核心”（来源：ADHD × AI 的科学与研究前沿）。

## 同构：从大脑到 agent，缺的都是同一个层

ADHD 的认知缺陷——工作记忆差、时间盲、任务启动困难——对应着 LLM 的技术局限：无状态性、上下文窗口有限、输出不可控。ADHD 的执行功能是大脑的“驾驶系统”，而 LLM 的编排层就是 agent 的“驾驶系统”。当 ADHD 患者用 AI 工具来弥补执行功能时，他们实际上是在为自己构建一个“外部执行功能层”——恰好就是 agent 工程师正在为 LLM 搭建的编排/记忆/工具层。

举个例子：ADHD 的时间盲（难以感知时间流逝，导致计划失败）对应 LLM 的“时间盲”——它没有内置的时间感知，无法自主决定何时切换任务或检查进度。工具如 **Motion** 利用 AI 自动规划日程并动态调整，直接弥补时间感知缺陷（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。同样，在 agent 工程中，我们需要一个调度器来管理任务队列、设置超时、重新规划——这就是 Motion 的算法在 ADHD 侧的映射。

再比如工作记忆：ADHD 患者常因工作记忆不足而丢失信息，需要频繁切换标签页搜索。**Saner.AI** 通过本地记忆和快速检索减少搜索循环（来源：Best AI Tools for ADHD Productivity in 2026）。在 LLM 侧，这正是“记忆层”的作用——将对话历史、工具调用结果持久化，避免上下文窗口溢出。

## 证据：两边都有真实案例

ADHD 侧的证据来自工具的实际效果。**Goblin Tools** 的 Magic ToDo 功能将“整理房间”这样的模糊任务分解为具体步骤，降低启动门槛（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。用户反馈称它“将压倒性的事情变成一系列不压倒性的事情”。**Tiimo** 专为时间盲设计，提供视觉时间表。**Focusmate** 利用算法匹配虚拟身体加倍伙伴，通过社会存在感提升专注度（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。这些工具都在做同一件事：为 ADHD 大脑提供外部执行功能。

LLM/agent 侧的证据来自工程实践。每个 agent 框架（如 LangChain、AutoGPT）都在解决编排、记忆、工具调用的问题。例如，ReAct 模式让 LLM 交替思考与行动，本质是外部调度；向量数据库作为长期记忆，弥补无状态性；工具调用层让 LLM 能访问外部 API，类似 ADHD 用户用提醒应用来“记住”时间。

## 核心观点：脚手架，不是拐杖

我的核心判断是：AI 工具作为外部执行功能层，其设计目标应该是“脚手架”而非“拐杖”——即提供可逐步撤除的临时支撑，而非永久替代内在能力。但当前多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这带来一个风险：长期依赖是否会削弱内在执行功能？目前缺乏纵向研究来回答这个问题。

同样，在 agent 工程中，我们也要警惕“过度编排”——如果 harness 层过于厚重，模型本身的调度能力可能退化。一个好的 harness 应该像训练辅助轮：在需要时提供支撑，在能力提升后可以卸掉。

## 局限与争议

同构模型目前主要基于概念类比和工具案例，缺乏大规模神经影像学和行为实验验证（来源：矛盾与存疑）。此外，多巴胺干预的有效性仍存争议，部分工具声称通过神经科学原理改善执行功能，但证据不足（来源：多巴胺页面）。工具的效果也缺乏独立随机对照试验，主要依赖用户反馈。

## 今天就能试的行动

1. **尝试任务分解**：用 Goblin Tools 的 Magic ToDo 将一个你拖延的任务分解为小步骤，看看启动是否变容易。
2. **设置外部时间锚点**：用 Motion 或 Tiimo 规划明天的日程，让 AI 帮你决定“下一步做什么”。
3. **减少搜索循环**：用 Saner.AI 或类似工具记录工作笔记，训练自己先检索而非切换标签页。
4. **体验虚拟身体加倍**：试试 Focusmate 的匹配功能，在他人（虚拟）在场时完成一项简单任务。

## 结语

ADHD 程序员和 agent 工程师面对的是同一道题：如何为一个强大的生成核心配上可靠的执行调度层。当你为自己搭建外部执行功能层时，你也在无意中理解了 agent 工程的本质。反之，当你设计 agent harness 时，不妨想想 ADHD 大脑的需求——那可能是最诚实的用户测试。

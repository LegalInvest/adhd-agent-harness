# 3 个 LLM 工程概念，彻底重写了我对 ADHD 不擅长精确、重复、状态维护类任务的理解

> agent 通过 function calling 把精确计算外包给工具 如何反向照亮 ADHD 的不擅长精确、重复、状态维护类任务。

如果你是一个ADHDer，你一定熟悉这样的场景：脑子里有无数绝妙的想法，但一到执行就卡壳——写一封邮件要花两小时，整理房间永远停留在“准备整理”阶段。如果你是一个做Agentic Harness的工程师，你也在反复处理类似的问题：LLM能生成惊艳的回复，但一旦涉及精确计算、多步状态维护、重复格式，它就崩溃。

这两种困境，表面上是不同领域的问题，但当我深入理解3个LLM工程概念后，我发现它们共享同一个底层结构。这篇文章不是为了做类比而类比——而是真实地看到了两边的证据，并由此重写了我对ADHD不擅长精确、重复、状态维护类任务的理解。

## 概念一：无状态与外部记忆

LLM本质上是无状态的——每次对话都是新的开始，它记不住你五分钟前说了什么，除非你把上下文塞进prompt。这就是为什么工程上要引入外部记忆系统（如向量数据库、缓存）来维持状态。

ADHD的工作记忆缺陷与此同构。研究发现，ADHD的执行功能障碍包括工作记忆容量有限，导致频繁的搜索循环和标签切换（来源：执行功能障碍）。你刚想做什么，转头就忘；你打开浏览器查资料，半小时后发现自己在看猫视频。这不是态度问题，是大脑的“工作记忆缓冲区”太小。

工具Saner.AI正是针对这一点设计的——它提供本地记忆存储和快速检索，用户无需在多个标签页或应用间切换即可获取所需信息（来源：Saner.AI）。这就像给LLM配了一个向量数据库。我自己的体验是，使用Saner.AI后，我减少了一半的“我刚刚要干什么？”时刻。

但这里有一个边界：外部记忆是脚手架还是拐杖？如果过度依赖，你永远学不会自己记住东西。就像LLM如果完全依赖检索增强生成（RAG），它自己的上下文窗口能力会萎缩。我们需要的是“脚手架”——在需要时提供支持，但允许大脑逐步重建内部记忆策略。

## 概念二：任务分解与规划机制

LLM在复杂任务上经常“想一步走一步”，缺乏全局规划。Agent工程中，规划机制（planning）是关键——将目标分解为子任务，并顺序执行。Lex工具正是这一思路的体现：它允许用户通过单一指令触发复杂、多步骤的任务序列（来源：Lex）。

ADHD的启动困难与此同构。面对“清理厨房”这样的任务，大脑会瘫痪——因为不知道从哪里开始。Goblin Tools的Magic ToDo功能自动将模糊任务分解为具体可执行的子步骤，用户可调节“辣度”控制粒度（来源：Goblin Tools）。这本质上就是agent的规划机制。

历史人物徐渭（徐文长）是ADHD特质与情绪失调的典型例子：他诗书画戏军全能，但执行功能极差，8次举人不中；情绪失调导致杀妻、9次自杀（来源：情绪失调）。他的harness系统是以书画为核心，将痛苦和精力注入艺术创作——这本质上是一种“任务分解”：把庞大的生命能量转化为具体的创作步骤。与Lex的“单一指令触发多步骤”同构：徐渭用一个“写诗”指令，触发了整个创作流程。

但规划机制也有局限。Lex的规划能力类似于AI agent的规划机制，但若分解过细，反而增加认知负荷（来源：矛盾与存疑）。ADHDer有时需要的是“先做再说”的冲动，而不是完美的步骤清单。

## 概念三：外部执行功能层（Harness）

LLM Agent工程的核心是搭建harness——围绕LLM构建软件架构和工具，使其能够执行复杂、目标驱动的任务（来源：Lex）。这包括function calling、工具调用、错误处理等。没有harness，LLM只是一个会说话的聊天机器人。

ADHD的执行功能障碍同样需要外部执行功能层。研究指出，ADHD的执行功能缺陷与LLM的无状态、缺乏调度层在结构上同构，都需要外部脚手架来补偿工作记忆不稳定、任务启动困难和注意力分散（来源：外部执行功能层）。

孔子就是一位深谙此道的ADHDer。他身高1米9，精力旺盛，周游列国14年坐不住；冲动爱骂人，对音乐超专注到三月不知肉味，对种地等俗事零耐心（来源：人物案例）。他的harness是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。这正是一个外部调度层：礼是规则，反省是日志，70岁才做到“从心所欲不逾矩”。这与LLM的harness同构——礼相当于function calling的约束，反省相当于日志和错误处理。

但这里有一个关键争议：过度依赖harness可能削弱内在执行功能（来源：矛盾与存疑）。孔子70岁才摆脱harness，说明脚手架只是手段，不是目的。

## 我的核心观点

ADHD大脑与LLM/agent是同一类“高产但缺执行调度层的生成核心”。两者的解决方案——外部记忆、任务分解、执行功能层——在结构上完全同构。这意味着：ADHDer可以借鉴agent工程的最佳实践来设计自己的harness系统，而agent工程师也可以从ADHD的应对策略中获得灵感。

但必须诚实指出局限：这种同构目前仅是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。它解释力强，但缺乏实证基础。此外，工具的效果多基于用户报告，缺乏严格临床验证（来源：主题综述）。

## 今天就能试的行动

1. **用Goblin Tools分解一个让你瘫痪的任务**：比如“整理书桌”，让它分解成10步，然后只做第一步。
2. **设置一个外部记忆系统**：用Saner.AI或类似工具，把所有“待查”“待办”丢进去，每天固定时间回顾。
3. **创建一个“单一指令”**：像Lex一样，把每天早上的例行公事（喝水、开日程、写三件要事）绑定到一个指令上，比如“开始一天”。
4. **设计你的“礼”**：像孔子一样，给自己定3条简单规则（如“每工作25分钟起身一次”“每天睡前写三句话”），并坚持一周。

这些行动不是灵丹妙药，但它们是脚手架——帮你从瘫痪中启动，直到你找到自己的节奏。

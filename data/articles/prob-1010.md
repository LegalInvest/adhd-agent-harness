# 如果把 ADHD 大脑当成生成核心，Claude 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Claude 对 ADHD 的真正价值。

## 问题：生成核心的失控时刻

你打开 Claude，想写一份项目计划。五分钟后，你在追问一个无关的历史冷知识。又过了十分钟，你开始让 Claude 帮你润色一封根本不急的邮件。你意识到：又跑偏了。

这场景对 ADHD 人群来说再熟悉不过——分心走神是常态，而更隐蔽的陷阱是超聚焦跑偏：一旦被某个有趣细节捕获，就像 LLM 被上下文膨胀带偏，原本的目标早已漂移。

## 同构：高产但缺调度层的生成核心

ADHD 大脑与 LLM 有着深层同构：两者都是**高产但缺乏可靠执行调度层的生成核心**。ADHD 的创意发散和 LLM 的文本生成都极其强大，但都缺少一个稳定的执行调度层来控制输出方向（来源：AI 与 ADHD 的专注力）。ADHD 的执行功能缺陷——工作记忆差、时间盲、任务启动困难——恰恰对应 LLM 的无状态性、上下文膨胀和 prompt 敏感（来源：执行功能障碍）。

这意味着，ADHD 大脑和 LLM 需要的不是“更强的生成能力”，而是**外部 harness**——一套能提供结构化提示、外部记忆和注意力锚点的系统。这正是 Agentic Harness 工程在做的事：为长程任务中的 agent 设计重锚定机制，防止目标漂移。

## 答案：Claude 作为重锚定与目标漂移的 harness

Claude 能不能当 ADHD 大脑的重锚定与目标漂移？答案是：可以，但前提是你必须把它当成 harness 而非拐杖。

### 重锚定：用外部 prompt 替代内部执行功能

ADHD 的执行功能障碍使得维持目标变得困难，而 Claude 可以充当一个永不疲倦的“system prompt 维护者”。你可以为 Claude 设定一个固定 system prompt，明确当前任务目标、步骤和边界。每次你感觉注意力漂移时，只需输入“重锚定”，Claude 就会回到原始目标并提醒你下一步该做什么。这类似于 Focusmate 的实时问责机制——外部结构补偿内在执行功能缺陷（来源：Focusmate）。

### 目标漂移检测：用外部记忆对抗无状态性

ADHD 的工作记忆缺陷与 LLM 的无状态性高度同构：两者都容易丢失上下文（来源：AI 与 ADHD 的专注力）。Claude 的对话历史可以充当外部记忆，但需要主动管理。你可以定期让 Claude 总结当前进度、对比原始目标，并标记任何漂移。这就像 RescueTime 自动记录时间使用，帮助用户识别注意力漂移模式（来源：RescueTime）。

### 注意力锚点：用结构化输出稳定生成核心

ADHD 易被环境带偏，LLM 易被上下文膨胀干扰，两者都需要主动设计“当下注意什么”的工程方案（来源：AI 与 ADHD 的专注力）。你可以要求 Claude 每次输出都以“当前目标：XXX”开头，并在长回复中插入“暂停点”——强制你（和它自己）重新聚焦。这种结构化提示模仿了为 LLM 搭建 harness 的方式，也对应了 Brain.fm 通过神经锁相技术影响大脑信息处理的设计原理（来源：Brain.fm）。

## 脚手架 vs 拐杖：边界在哪里？

然而，我们必须诚实面对一个矛盾：**AI 工具作为外部执行功能层的有效性证据不足**（来源：矛盾与存疑）。现有证据主要来自用户报告和概念类比，缺乏大规模对照实验。更重要的是，**过度依赖 AI 工具可能削弱内在能力**（来源：矛盾与存疑）。

区分“脚手架”与“拐杖”的关键在于：脚手架是帮助你学会自己走路，而拐杖是永久替代你的腿。当 Claude 帮你重锚定目标时，你是在练习目标维持；但当 Claude 替你完成所有任务分解和决策时，你的执行功能可能进一步退化。理想状态下，Claude 应该是一个训练伙伴，而非替身——它提示你“下一步该做什么”，但由你决定是否执行。

## 局限与争议

- **个体差异巨大**：ADHD 的异质性使得同一工具对不同亚型效果迥异，目前缺乏个性化适配研究（来源：矛盾与存疑）。
- **同构是隐喻，并非严格等价**：LLM 的“注意力机制”与人类注意力存在本质差异，需谨慎类比（来源：AI 与 ADHD 的专注力）。
- **证据质量有限**：多数工具仅有用户报告或小样本研究，包括 Brain.fm 针对 ADHD 的效果缺乏独立临床研究支持（来源：Brain.fm）。

## 今天就能试的行动

1. **设置重锚定 prompt**：在 Claude 中创建一个固定 system prompt，包含当前核心目标、3 个关键步骤和“重锚定”触发词。每次感觉跑偏时，输入“重锚定”让 Claude 回到目标。
2. **开启目标漂移检测**：每完成一个子任务，让 Claude 对比原始目标，输出一段“当前进度 vs 原始目标”的总结，并标注任何偏离。
3. **使用结构化输出**：要求 Claude 每次回复以“当前目标：XXX”开头，并在每 500 词后插入“暂停点：请确认是否继续”的提醒。
4. **记录使用模式**：用 RescueTime 或简单日志记录你使用 Claude 时的注意力漂移频率，每周回顾一次，调整你的 harness 设计。

ADHD 大脑和 LLM 都不是缺陷，而是需要正确 harness 的生成核心。Claude 可以成为那个 harness——只要你记得，它的目的是帮你学会自己走路，而不是永远扶着它。

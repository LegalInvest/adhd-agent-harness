# ChatGPT 之于 ADHD，就像 调低采样温度 之于 LLM——但有人用错了

> 从同构视角实测 ChatGPT：它到底补上了哪一层执行功能？

你有没有过这样的体验：早上醒来，大脑像一台过热的生成器，灵感喷涌，但一整天下来，真正完成的事寥寥无几？或者，你调试一个 LLM agent，发现它每次输出都不一样，有时天才有时智障，于是你拼命调低采样温度，试图让它稳定靠谱。

这两种困境，本质上是同一个问题：**一个高产但缺执行调度层的生成核心，在缺乏外部结构时，表现剧烈波动。** ADHD 的大脑与 LLM/agent 在架构上同构：两者都拥有强大的生成能力（智力/算力），但缺乏可靠的执行调度层（执行功能/编排），因此需要外部系统来提供记忆、决策引导和行动约束（来源：【主题综述：AI 与 ADHD 的日常生活】）。

而 ChatGPT 之于 ADHD，就像调低采样温度之于 LLM——很多人以为它的价值是消除随机性、让输出稳定，于是拼命追问“我该怎么做”，希望得到一个确定答案。但真正有用的是另一层：**它补上的是执行功能中的上下文工程与任务分解，而不是消除波动本身。**

## 问题：ADHD 的“状态日内大幅波动”到底是什么？

ADHD 的核心困境不是“不专注”，而是**无法可靠地调度自己的专注**。你可以在某件事上超聚焦六小时，却无法在需要的时候启动一个五分钟的任务（来源：【超聚焦】）。这种波动源于执行功能缺陷：工作记忆容量有限，时间感知缺失，任务启动困难（来源：【认知负荷】）。

就像 LLM 在采样温度过高时输出发散、过低时输出重复，ADHD 大脑的状态也像温度旋钮失控——时而 hyperfocus 到忘记时间，时而完全无法启动。但问题不是温度本身，而是**缺少一个调度层来根据场景动态调节温度**。

## 同构：LLM/agent 工程师的“温度困境”

做 Agentic Harness 工程的读者一定熟悉这个场景：你给 LLM 设了一个高温度（比如 0.8），希望它创意迸发，结果它跑偏了；你降到 0.1，它稳定了，但变得死板，不会探索。于是你开始加结构约束——prompt 模板、输出格式、RAG 上下文、状态机——本质上是在生成核心外面套一个**外部执行功能层**。

这正是 ADHD 需要的：不是消除波动（那会杀死创造力），而是**用外部结构来引导波动**。

## 答案：ChatGPT 补上的是“上下文工程”与“任务分解”

当 ADHD 用户打开 ChatGPT 问“我该怎么做”，他们通常期待一个万能答案。但真正起作用的用法是：**把 ChatGPT 当作一个外部工作记忆和任务分解器。**

- **工作记忆外化**：ADHD 的工作记忆缺陷与 LLM 的无状态性同构，两者都无法可靠维持跨会话上下文（来源：【主题综述：AI 与 ADHD 的日常生活】）。ChatGPT 的对话历史、自定义指令、项目文件夹，就是 LLM 的向量数据库——用于保持连续性。
- **任务分解**：Goblin Tools 的 Magic ToDo 功能能自动将任何任务分解为更小、更易管理的步骤（来源：【Goblin Tools】）。这正是 LLM agent 中常用的 task decomposition 技术——将复杂任务拆解为子任务，降低认知负荷（来源：【认知负荷】）。
- **启动提示**：Routinery 通过可视化倒计时和过渡提示，直接针对任务启动困难和时间感知缺失（来源：【Routinery】）。这相当于给 LLM 的生成延迟加上一个“timeout”和“next step”提示。

**关键判断：ChatGPT 不是要调低你的“采样温度”，而是要帮你搭建一个“harness”。** 那些试图用 ChatGPT 消除所有不确定性的人，最终会得到稳定但乏味的输出——就像温度 0.0 的 LLM，什么也不做。

## 争议与局限：脚手架还是拐杖？

这里有一个根本矛盾：工具应该设计为可逐步撤除的临时支撑（脚手架），还是长期依赖的拐杖？（来源：【矛盾与存疑】）

目前多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：【矛盾与存疑】）。这意味着用户可能形成依赖，而非真正提升内在执行功能。此外，同构命题本身缺乏大规模实证，主要来自概念类比和工具案例（来源：【矛盾与存疑】）。

我的判断是：**脚手架和拐杖的边界在于——工具是否在帮你构建内部结构，还是替代内部结构。** 如果你用 ChatGPT 来分解任务、回顾上下文、记录想法，你是在搭建脚手架；如果你只是让它替你决定一切，那就是拐杖。

## 今天就能试的行动

1. **用 ChatGPT 做“上下文工程”**：每次开始任务前，先花 30 秒给 ChatGPT 写一段“上下文提示”，说明你当前的状态、目标、已有进展。这相当于给 LLM 注入 RAG 上下文，也帮你自己理清思路。
2. **用 Goblin Tools 分解一个令你拖延的任务**：输入“整理房间”或“写报告”，看看 AI 如何拆解。然后只做第一步。
3. **设置一个“启动提示”仪式**：用 Routinery 或简单计时器，为每天的第一个任务设置 5 分钟倒计时。这相当于给 LLM 的生成加上一个“timeout”和“next step”信号。
4. **记录你的“波动日记”**：连续三天，记录你状态最好的时段和最差的时段，以及当时的外部环境。你会发现自己的“采样温度”模式，然后主动调整。

ChatGPT 不是万能药，它只是一个外部执行功能层。用对了，它是脚手架；用错了，它是拐杖。而真正的解法，是理解你大脑与 LLM 的同构，然后为自己设计一个可撤除的 harness。

# 如果把 ADHD 大脑当成生成核心，Structured 能不能当它的外部执行功能层？

> 用 agent 工程的思路重新理解 Structured 对 ADHD 的真正价值。

## 问题：当生成核心没有驾驶系统

你打开待办清单，盯着“整理项目报告”发了十分钟呆。不是不想做，而是大脑像断连的 API——知道要输出，却不知道从哪一行开始。另一边，工程师调试 agent 时发现：模型在第五轮对话后开始忘记系统指令，输出的代码偏离了最初目标。两个场景共享同一个底层困境：**生成核心（大脑/LLM）不缺算力，缺的是执行调度层。**

ADHD 的执行功能缺陷——计划、组织、任务启动困难——本质上就是大脑的“编排层”先天偏弱（来源：ADHD × AI 的科学与研究前沿）。而 LLM/agent 工程的痛点——无状态、上下文窗口溢出、指令消退——恰好是同样的缺失。当工程师给 agent 套上 harness（上下文管理、工具接口、规划制品），他们其实在构建一个“外部执行功能层”。那么，把同一个思路套回 ADHD 场景：**Structured 这类工具，能不能充当大脑的外挂编排层？**

## 答案：是，但前提是理解同构

Structured 本质上是一个可视化日程规划器，但它的核心价值不在于界面美观，而在于它做了两件与 agent harness 同构的事：**分解与重锚定**。

### 1. 分解：对抗“启动瘫痪”

Goblin Tools 的 Magic ToDo 功能能将“整理房间”自动拆成“捡起地板上的衣服”“擦桌子”等步骤（来源：Goblin Tools）。这与 agent 工程中的“任务分解”完全一致：LLM 无法直接完成“写一个电商网站”，需要 harness 将其拆为“设计数据库”“实现用户认证”等子任务。Structured 虽然不自动分解，但它提供了手动创建子任务的界面——用户可以把一个压倒性的项目敲成一个个 5 分钟能完成的块。

ADHD 侧证据：用户反馈称这种分解“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools）。LLM 侧证据：Harness 工程的核心就是设计规划制品（planning artifacts），让模型按步骤输出（来源：规划循环与任务分解）。两边都证明：**分解降低启动门槛**。

### 2. 重锚定：对抗“目标漂移”

ADHD 大脑在执行中极易被无关刺激捕获，或陷入超聚焦忘记整体目标。外部提醒系统——如通知、倒计时——提供重锚定（来源：重锚定与目标漂移）。LLM agent 同样面临“指令消退”：模型在长程任务后期开始忘记系统提示（来源：重锚定与目标漂移），harness 通过事件驱动的系统提醒来纠正。

Structured 的“时间块”和“提醒”功能正是这种重锚定系统：它把一天切成可视化的块，并在切换时发出通知。Motion 更进一步，能自动重新安排被打乱的日程（来源：Motion）。Tiimo 则专为 ADHD 设计，用视觉时间线减少时间盲（来源：规划循环与任务分解）。这些工具相当于 agent 的“上下文注入”——不断把注意力拉回轨道。

## 核心判断：脚手架，不是拐杖

这里必须诚实：同构模型是理论框架，缺乏大规模实证（来源：矛盾与存疑）。ADHD 大脑与 LLM 的类比虽然有力，但神经影像学证据尚未跟上。更关键的争议在于：**长期使用这些工具，是否会削弱内在执行功能？**（来源：矛盾与存疑）

我的观点是：Structured 这类工具是“脚手架”——临时支撑，帮助大脑建立新的神经通路，而不是永久替代。关键在于设计“可撤除性”。例如，先使用自动分解（Goblin Tools），然后逐步过渡到手动分解；先依赖提醒，然后尝试自我提醒。目前多数工具（包括 Structured）并未内置撤除机制，用户需要主动意识。

## 局限与争议

- **证据强度**：现有工具评价多来自用户反馈和综述，缺乏独立随机对照试验（来源：Goblin Tools、Saner.AI）。
- **多巴胺干预争议**：部分工具声称基于多巴胺原理，但效果存在争议（来源：矛盾与存疑）。
- **泛化性**：同构模型主要基于概念类比，需更多验证（来源：矛盾与存疑）。
- **隐私**：外部记忆系统涉及敏感数据，需确保安全（来源：ADHD × AI 的科学与研究前沿）。

## 今天就能试的行动

1. **手动分解一个任务**：在 Structured 中创建一个项目，拆成 3-5 个可执行子任务。对比平时，看启动速度是否有变化。
2. **设置三个重锚定**：在一天的关键节点（如上午 10 点、下午 2 点）设置提醒，内容写“当前任务是什么？”。观察目标漂移是否减少。
3. **测试自动分解工具**：用 Goblin Tools 的 Magic ToDo 分解一个你拖延已久的任务，感受“不压倒性”的步骤。
4. **反思依赖边界**：使用一周后，问自己：如果明天工具消失，我还能保持这个习惯吗？尝试一天不用工具，评估内在执行功能的变化。

Structured 不是魔法，但当你把它看作大脑的外挂编排层，它就不再是另一个待办清单——它是你与 LLM 共享的那个缺失调度层的临时补丁。用得好，它能帮你完成一次完整的“任务推理”；用过头，它可能成为你永远不学走路的拐杖。选择权在你。

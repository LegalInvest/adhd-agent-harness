# Goblin Tools 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Goblin Tools：用久了会让 ADHD 更强还是更依赖？

你刚打开冰箱，想拿鸡蛋，结果看到酸奶过期了，于是开始清理冰箱，半小时后发现自己坐在沙发上刷手机，鸡蛋的事完全忘了。这是 ADHD 大脑的日常：工作记忆容量小，边做边忘，执行功能像一台没有调度器的 GPU——算力惊人，但任务调度一塌糊涂。

同样的问题，LLM 工程师再熟悉不过：大模型本身没有跨会话状态，每次对话都是冷启动，必须靠外部记忆（向量库、RAG、Agent 脚手架）来维持上下文连贯。ADHD 大脑与 LLM 在架构上高度同构——都是“高产但缺执行调度层的生成核心”（来源：ADHD 大脑与 LLM 的同构）。

那么，当 Goblin Tools 这类 AI 工具介入时，它究竟是帮助 ADHD 大脑搭建了外部调度层（脚手架），还是让大脑彻底放弃了内部调度能力（拐杖）？

## 同构的答案：脚手架 vs 拐杖的边界

Goblin Tools 的核心功能是 Magic ToDo：输入“清理厨房”，它自动分解成“收拾台面→洗碗→擦灶台→拖地”等子步骤，你还可以调节“辣度”控制粒度（来源：Goblin Tools）。这本质上是一个外部任务分解器——对应 LLM 世界里的 Agent 规划层（planning layer）。

ADHD 侧的真实证据：明尼苏达大学研究发现，ADHD 大脑与 LLM 共享“强记忆、弱控制”剖面——工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷，无法灵活切换任务集、无法抑制自动化反应（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。耶鲁大学进一步发现，自注意力机制本身导致工作记忆容量限制：随任务负荷增加，注意力分数熵增、弥散，与人类注意分散机制同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。这意味着，瓶颈不在容量/算力，而在 orchestration 层。

LLM/Agent 侧的真实证据：LLM 在经典 Stroop 冲突任务中，注意力随序列变长在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突，这与 ADHD 执行功能缺陷的核心标志一一对应（来源：Deficient Executive Control in Transformer Attention）。因此，工程师为 LLM 搭建的 Agent 脚手架（记忆检索、任务规划、工具调用）与 Goblin Tools 为 ADHD 大脑搭建的任务分解器，本质上是同一类工程：为生成核心套上外部调度层。

但脚手架和拐杖的边界在于：脚手架是临时辅助，最终目的是让使用者学会自己搭建；拐杖则是永久替代，让使用者丧失自身能力。

## 人物 harness：孔子与“克己复礼”的脚手架

孔子是 ADHD 特质鲜明的历史人物：身高1米9，精力旺盛，周游列国14年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予“朽木不可雕”；对音乐超专注到三月不知肉味，对种地等俗事零耐心。他的专属 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”，70岁才做到“从心所欲不逾矩”。

这套系统与 LLM 的 Agent 脚手架高度同构：“礼”相当于外部约束规则（system prompt + 行为边界），“三省吾身”相当于定时 re-grounding（每日回顾与状态重置）。孔子没有依赖任何工具，但他创造了一套内化的脚手架。而 Goblin Tools 提供的正是这种脚手架的数字化版本——但它是否帮助用户内化？还是让用户永远依赖它来分解任务？

## 争议：Goblin Tools 的证据与局限

目前，Goblin Tools 被评价为“简单有用”（来源：Goblin Tools），但缺乏严格临床验证，效果多基于用户报告（来源：矛盾与存疑）。更关键的是，依赖风险真实存在：AI 工具可能从脚手架变为永久拐杖，削弱内在执行功能（来源：拐杖与脚手架）。

此外，Goblin Tools 的分解是单向的——它帮你拆解，但不会教你如何自己拆解。相比之下，Saner.AI 通过知识回忆和本地记忆减少搜索循环（来源：Saner.AI），Lex 通过“单一指令触发多步骤”降低启动门槛（来源：Lex），这些工具各有侧重，但都面临同样的拐杖风险。

## 核心观点：工具是外部调度层，但需刻意训练内部调度

我的判断是：Goblin Tools 对 ADHD 而言，本质是外部调度层（harness），不是拐杖——但前提是你有意识地用它来训练自己的任务分解能力，而不是把它当作永远的外包。就像 LLM 的 Agent 脚手架，如果工程师永远不优化 prompt 和记忆策略，模型就永远无法在冷启动时表现更好。

孔子用了一辈子才内化“克己复礼”，但如果没有每日反省的刻意练习，礼就只是外在枷锁。同样，Goblin Tools 应该成为你的“三省吾身”助手，而不是替你省去所有思考。

## 今天就能试的行动

1. **用 Magic ToDo 分解一件小事**：选一个你拖延已久的任务（比如“整理书桌”），用 Goblin Tools 分解到最小粒度（辣度调高），然后逐项执行。完成后，自己尝试回忆分解步骤，看能否下次不用工具。
2. **设置“反省”提醒**：每天固定时间，用 ChatGPT 或 Claude 问自己三个问题：“今天哪些任务被工具分解了？我是否记住了分解逻辑？下次能否自己拆？”（来源：ChatGPT、Claude 作为外部执行功能工具）
3. **对比工具与手动分解**：选一个新任务，先自己列出步骤，再用 Goblin Tools 分解，对比差异。注意哪些步骤是你遗漏的，哪些是工具过度拆分的。
4. **尝试 Saner.AI 的知识回忆**：如果你经常在多个工具间切换，用 Saner.AI 的本地记忆功能减少搜索循环，但每周回顾一次自己记住了多少（来源：Saner.AI）。

## 诚实面对局限

同构性目前仅是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。Goblin Tools 的长期效果缺乏独立研究，依赖风险也未被充分量化。但至少，它提供了一个起点：让你看见自己大脑的调度层，然后决定是否要自己学会驾驶。

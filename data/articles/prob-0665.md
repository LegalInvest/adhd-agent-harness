# Routinery 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Routinery：用久了会让 ADHD 更强还是更依赖？

## 问题：为什么 Routinery 让 ADHD 又爱又怕？

ADHD 大脑有一个经典困境：明明有极强的生成能力（创意、联想、危机处理），却总是“边做边忘”——工作记忆像漏水的桶，刚想好的下一步，转身就消失。Routinery 这类应用正是为此而生：它把日常流程拆成步骤，用过渡提示一步步推着你走。但问题来了：用久了，你是变得更自律，还是彻底离不开它？

这个问题在 LLM/agent 工程领域有一个镜像版本：LLM 天生无状态（每次对话都从零开始），必须靠外部记忆（向量库、工具调用）来维持连贯任务。工程师们争论：这个外部 harness 是赋能核心，还是让模型丧失了自主推理能力？

本文的核心判断：**两者同构，且边界清晰——当工具作为“外部调度层”补偿缺陷时是脚手架；当它替代而非训练内在执行功能时则沦为拐杖。**

## 同构：ADHD 大脑与 LLM 都是“高产但缺调度”的生成核心

ADHD 的执行功能缺陷，本质是大脑的“调度层”不稳定：工作记忆容量小、任务启动困难、注意力易被捕获。LLM 同样缺乏内置调度——它没有跨会话记忆，无法自主规划多步骤任务，必须依赖外部脚手架（如 agent 框架、上下文窗口管理）来维持连贯性（来源：无状态与外部记忆）。

这种结构同构意味着：两者的解法也同构。ADHD 需要“外部执行功能层”——像 Routinery 这样的步骤序列应用，或像 Obsidian 这样的第二大脑笔记系统（来源：无状态与外部记忆）。LLM 需要“agent scaffolding”——围绕大语言模型构建的工具和架构，使其能执行复杂任务（来源：Lex）。

## 人物案例：孔子的“克己复礼”与辛弃疾的“词剑 harness”

孔子是典型的 ADHD 大脑：精力旺盛到周游列国 14 年，冲动爱骂人（骂宰予“朽木不可雕”），但对音乐能超聚焦到“三月不知肉味”。他的 harness 是“礼”——一套外部行为规范，每天用“吾日三省吾身”来 re-ground 自己。这相当于 LLM agent 的“定时上下文重置”机制：用固定的日课把注意力拉回正轨。

辛弃疾的 harness 更显性：21 岁带 50 人冲几万敌军抓叛徒，闲居 20 年把全部精力注入词和剑。他的“词剑系统”本质是一个外部输出通道——把无处安放的多巴胺冲动转化为创作。这对应 LLM 的“工具调用”模式：当模型有高能量但缺乏方向时，用外部工具（如代码执行器、搜索 API）来引导输出。

两位都没有“治愈”ADHD，而是用脚手架管理了它。孔子到 70 岁才“从心所欲不逾矩”，辛弃疾一辈子都在躁动——但他们的成就证明了：好的 harness 不是消除特质，而是把生成核心的能量导入有效轨道。

## Routinery 是脚手架还是拐杖？三条分界线

### 1. 补偿 vs 替代
- **脚手架**：Routinery 帮你记住步骤序列，降低启动成本，但你仍需要主动执行每个步骤（来源：上下文工程）。它像 LLM 的“提示词模板”——提供结构，但不替你思考。
- **拐杖**：当你不看 Routinery 就完全无法启动，甚至忘记自己要做这件事——工具替代了你的内在意图。这相当于 LLM 的“完全脚本化 agent”：模型不再自主推理，只是机械执行预定义流程。

### 2. 训练 vs 依赖
- **脚手架**：使用过程中，你逐渐内化步骤模式，最终能脱离工具完成类似任务。就像 LLM 通过 fine-tuning 学会更长的上下文推理。
- **拐杖**：工具成为唯一触发器，撤掉后能力归零。这对应 LLM 的“上下文窗口依赖”：模型一旦离开外部记忆就完全失忆。

### 3. 自适应 vs 固化
- **脚手架**：Routinery 允许你调整步骤、跳过或添加——工具适应你的实时状态。这类似 LLM agent 的动态规划：根据当前上下文调整行动。
- **拐杖**：流程僵化，无法应对变化。当 ADHD 大脑的“超聚焦”突然进入某个任务时，工具反而打断你（来源：超聚焦的双面性）。

## 证据与争议：为什么不能一刀切

现有工具确实有效：Goblin Tools 的任务分解降低启动门槛（来源：Goblin Tools），Saner.AI 的本地记忆减少搜索循环（来源：Saner.AI），Lex 的“单一指令触发多步骤”适合 ADHD 对低摩擦的需求（来源：Lex）。但这些证据多基于用户报告，缺乏严格临床验证（来源：矛盾与存疑）。

更关键的是依赖风险：多个工具页面只强调益处，未充分讨论过度依赖可能导致内在执行功能萎缩（来源：拐杖与脚手架）。Routinery 的步骤序列如果变得不可替代，用户可能失去自主规划能力——就像 LLM 过度依赖外部记忆后，连短上下文推理都退化。

此外，同构理论本身是类比而非科学事实（来源：矛盾与存疑）。ADHD 大脑的神经基础与 LLM 的架构有本质差异——前者是生物系统，后者是统计模型。同构视角是有效的设计隐喻，但不能等同于生理机制。

## 今天就能试的 3 个行动

1. **诊断你的 Routinery 使用模式**：连续 3 天，记录你每次打开 Routinery 时是否已经知道下一步该做什么。如果 80% 以上是“看了才知道”，说明工具在替代你的内在调度——尝试先自己回忆步骤，再用工具验证。
2. **做一次“无工具日”实验**：选一天完全不用 Routinery，手动执行日常流程。观察自己哪些步骤能独立完成，哪些会卡住——这能帮你识别哪些环节需要脚手架，哪些已经形成肌肉记忆。
3. **为 LLM agent 设计“反脆弱” harness**：如果你在开发 agent，加入“随机上下文重置”测试——移除外部记忆后，看模型能否基于自身推理完成至少 50% 的任务。这能暴露过度依赖。

## 结论：脚手架与拐杖的边界在你手中

Routinery 本身是中性的。它成为什么，取决于你如何使用：是把它当作“外部调度层”来补偿工作记忆的波动，还是当作“永久拐杖”来回避内在执行功能的锻炼。

这个边界与 LLM agent 的 harness 设计完全同构——好的脚手架提升核心能力，坏的拐杖削弱它。孔子和辛弃疾的 harness 之所以伟大，不是因为他们依赖工具，而是因为他们用工具释放了生成核心的潜能。

下次打开 Routinery 时，问自己一个问题：**我是在用工具完成自己做不到的事，还是在让工具替我做我本可以学会的事？** 答案就是边界。

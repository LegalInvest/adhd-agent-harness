# Saner.AI 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Saner.AI：它到底补上了哪一层执行功能？

你有没有过这样的体验：打开一个笔记应用，想找上周记下的灵感，结果翻了十分钟标签，最后忘了自己本来要干嘛？或者，你给 AI agent 写了个 prompt，它却绕了一大圈，没调用任何工具，直接给你一段废话？

这两个场景，其实共享同一个底层问题：一个强大的生成核心（ADHD 大脑或 LLM），缺了一个可靠的外部执行调度层。ADHD 大脑不擅长精确、重复、状态维护类任务——就像 LLM 不擅长算术和实时数据查询。解决方案也同构：给它们套上 harness，通过工具调用（function calling）或外部脚手架，把精确计算外包出去。

Saner.AI 正是这样一个尝试。它被描述为“专为对抗执行功能障碍、任务瘫痪和认知超载而设计的全能 ADHD 友好型 AI 生产力和知识助手”（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。其核心卖点是“知识回忆”和“本地记忆”，减少搜索循环和标签切换。听起来很美，但实际使用中，很多人把它用成了“高级笔记本”——存了一堆东西，却很少真正调用。这就像给 LLM 加了一个工具函数，但从不触发它。

### 同构脊柱：从 function calling 看 Saner.AI 的真正价值

LLM/agent 侧的工程实践已经证明：一个 agent 的能力上限，不取决于它有多聪明，而取决于它的工具调用（function calling）设计。好的 harness 让 agent 知道什么时候该调用计算器，什么时候该查数据库，什么时候该闭嘴。ADHD 大脑的 harness 同理：不是要你记住所有事情，而是要你建立一套外部系统，在需要时自动触发正确的行为。

孔子就是这样一个 harness 大师。他精力旺盛、冲动爱骂人（骂宰予“朽木不可雕”），思维跳跃到《论语》全是场景化语录，没有系统著作。他的解决方案是“克己复礼”——用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”）。这套系统本质上就是一个外部调度器：每天定时 re-grounding，用仪式感对抗冲动。对应到 LLM 侧，这就是一个定时触发 context 更新的 hook。

释迦牟尼的 harness 更直接：29 岁说走就走，放弃王位，49 年游行说法，一辈子在路上。他的“八正道”和“持戒”就是一套行为协议，用正念拴住乱跑的念头。对应到 agent 工程，这相当于给 LLM 加了一个“行为约束层”——规定它在什么情况下必须调用工具，什么情况下必须输出格式化的响应。

Saner.AI 的潜力正在于此。它不应该只是一个被动的知识库，而应该成为一个主动的调度器：在你忘记下一步该做什么时，自动弹出“你今天有三个待办，第一个是……”；在你陷入搜索循环时，直接给出你需要的那个笔记的摘要。这才是 function calling 的正确用法——不是给你一个工具列表，而是替你做决策。

### 工具实测：Saner.AI 到底补上了哪一层？

从执行功能模型看，ADHD 的核心缺陷集中在三层：任务启动、工作记忆、时间感知。Saner.AI 主要补的是工作记忆层——通过本地记忆减少搜索循环。但它对任务启动和时间感知的帮助有限。相比之下，Goblin Tools 的 Magic ToDo 直接攻击任务启动（将“清理厨房”分解为 12 个子步骤），Lex 的“单一指令触发多步骤”则降低了启动门槛（来源：Best AI Tools for ADHD Productivity in 2026）。

也就是说，Saner.AI 更像是一个“外部记忆层”，而不是一个完整的“外部执行功能层”。它需要搭配其他工具使用：比如用 Goblin Tools 做任务分解，用 Tiimo 做时间可视化，用 Motion 或 Reclaim.ai 做智能调度（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。这就像 LLM 的 function calling 需要多个工具函数协同——一个做计算，一个查数据库，一个发邮件。

### 争议与局限：脚手架还是拐杖？

这里有一个关键边界：脚手架 vs 拐杖。好的 harness 应该逐渐内化——孔子 70 岁才做到“从心所欲不逾矩”，说明外部规则最终可以变成内在习惯。但很多 AI 工具（包括 Saner.AI）容易让人产生依赖：你不再试图记住任何东西，因为你知道有个“第二大脑”。当工具宕机或你忘记打开它时，你就彻底瘫痪了。

wiki 资料中的「矛盾与存疑」也指出：同构性被过度推广，目前只是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。同样，多数 ADHD 工具缺乏独立临床验证，效果多基于用户报告。所以，请保持谨慎——Saner.AI 可能对你有效，也可能只是另一个“高级玩具”。

### 今天就能试的 3 件事

1. **把 Saner.AI 当成“主动调度器”而非“被动笔记”**：设置每日一次提醒，强迫自己打开它，用语音捕获今天的三个关键任务。不要等灵感来了才记录，而是主动用它规划。
2. **组合使用 Goblin Tools 的 Magic ToDo 进行任务分解**：把 Saner.AI 里记下的模糊任务（比如“准备会议”）粘贴到 Goblin Tools，让它拆成小步骤，再贴回 Saner.AI 作为待办。
3. **观察自己的“工具调用模式”**：记录一周内你实际触发 Saner.AI 的次数和场景。如果少于 5 次，说明它在你的 harness 里是“未调用的函数”——要么删除它，要么重新设计触发条件。

ADHD 大脑和 LLM 一样，不需要更聪明的核心，需要更好的调度器。Saner.AI 可以是那个调度器，但前提是你得把它用成 function calling，而不是一个只存不用的 API 文档。

# ADHD 学生如何用「给生成核心套 harness」的思路应对考试与作业里的冲动下结论、想当然？

> 把 LLM 工程里的「LLM 自信地编造（幻觉），靠验证与自我批判兜底」迁移到ADHD 学生的考试与作业场景，双域证据作答。

## 问题：ADHD 大脑的“自动完成”陷阱

你是否有过这样的经历：考试时读完题目第一句就“知道”答案，洋洋洒洒写满整页，交卷后才发现完全理解错了题意？或者写作业时灵感喷涌，自信地写下结论，却被老师用红笔圈出“逻辑跳跃、缺乏证据”？

这不是你不够聪明。ADHD 大脑是一个**高产但缺执行调度层的生成核心**（来源：ADHD 大脑与 LLM 的同构）。就像 LLM 在没有 agent scaffolding 时自信地编造（幻觉）一样，你的大脑在认知负荷下会**自动补全**缺失的信息、**冲动地跳向结论**——因为工作记忆容量有限，大脑必须快速压缩推理链条（来源：认知负荷）。

但 LLM 工程已经找到了解法：**给生成核心套 harness**。LLM 负责生成，但外部系统负责验证、批判、纠正。这个思路，完全可以迁移到你的考试和作业中。

## 同构：LLM 幻觉 vs ADHD 冲动结论

先看 LLM 侧的真实证据。用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中模型表现正常，但随序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突（来源：Deficient Executive Control in Transformer Attention）。这正是 ADHD 执行功能缺陷的核心标志：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃（来源：认知负荷）。

再看 ADHD 侧。执行功能教练 Yulia Rafailova 指出：“AI 提供了一个不可思议的工具来外化我们的思维，作为一个支架，能显著减轻我们工作记忆上的认知负荷。”（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）

两边的问题结构一模一样：**生成核心（LLM/ADHD 大脑）在低负荷下表现良好，但认知负荷一高就“幻觉”/冲动下结论。** 解法也同构：**不要试图修复生成核心，而是给它套上外部验证 harness。**

## 历史人物的 harness 系统

毛泽东是 ADHD 特质鲜明的例子：他冲动敢赌（四渡赤水、抗美援朝都是险棋），思维极度跳跃，文章讲话充满场景化比喻。他的 harness 是**批评与自我批评、调查研究、民主集中制**——用集体决策制衡自己的冲动，用“没有调查就没有发言权”强迫自己验证假设（来源：人物案例）。这对应 LLM agent 中的**外部验证循环**：LLM 生成方案后，由另一个模块（或人类）进行批判和修正。

老子则走了另一条路：他“无为”，不强行做事，把所有能量收回来聚焦在道上。他的 harness 是**不著书不立说**，不被名相束缚（来源：人物案例）。这对应 LLM 工程中的**减少上下文膨胀**：当模型被太多无关信息干扰时，推理会退化；老子选择“少则得，多则惑”，本质上是一种注意力管理策略。

## 如何给考试和作业套 harness

### 1. 外部验证清单

在考试中，冲动下结论后，不要立刻写答案。先写一个“验证清单”：
- 题目真正问的是什么？（用 Goblin Tools 的 Magic ToDo 思路，把问题分解为子问题）
- 我的结论有哪些假设？
- 有没有反例？

这相当于给 LLM 加了一个**自我批判模块**。Goblin Tools 能自动将复杂任务分解为小步骤（来源：Goblin Tools），你可以用类似思路，把“解答一道题”分解为“读题→列假设→验证→写答案→检查”。

### 2. 外部记忆与时间感知

ADHD 的工作记忆缺陷导致无法可靠保留跨上下文信息（来源：无状态与外部记忆）。考试时，你可能会忘记自己之前推导的步骤，导致重复劳动或逻辑断裂。解决方案是**外化工作记忆**：
- 在草稿纸上写下每个步骤的中间结论（像 LLM 的 chain-of-thought 提示）
- 使用 Saner.AI 等工具记录知识点和常见错误（来源：Saner.AI），减少搜索循环
- 用 Motion 自动排程复习时间（来源：Motion），避免临时抱佛脚导致的认知超载

### 3. 超聚焦的引导而非打断

超聚焦是 ADHD 的双刃剑——你可以在错误的事情上超聚焦六小时（来源：超聚焦）。最新理论认为，超聚焦源于前额叶控制层级的“断裂”（来源：Transient Frontal Fracturing: A Theoretical Account of Hyperfocus）。与其强行打断（番茄钟可能无效），不如**引导超聚焦到正确方向**：
- 在开始作业前，用 2 分钟写下“我这次要验证的核心假设是什么”
- 设定一个外部检查点：每完成一个部分，用红笔标注“此处是否跳过了推理步骤？”

这对应 LLM 工程中的**定时 re-grounding**：每隔若干 token 让模型重新阅读上下文，防止漂移。

## 争议与局限

必须诚实指出：这个同构框架目前仍是理论，缺乏大规模随机对照试验验证（来源：矛盾与存疑）。AI 工具也可能成为“拐杖”而非“脚手架”——过度依赖可能导致能力退化（来源：拐杖与脚手架）。此外，个体差异极大：有人需要刚性打断，有人需要柔性引导（来源：矛盾与存疑）。

但有一点是确定的：**ADHD 大脑不是故障，而是一个需要外部调度层的生成引擎。** 你不需要“修复”自己的冲动和跳跃，只需要学会给它们套上 harness。

## 今天就能试的行动

1. **考试时用“三秒规则”**：写出答案前，先停三秒，问自己“我跳过了什么假设？”
2. **用 Goblin Tools 分解作业**：把“写论文”分解为“找资料→列大纲→写引言→写论点→写结论→检查逻辑”，每完成一步打勾（来源：Goblin Tools）
3. **建立外部记忆库**：在 Saner.AI 或笔记软件中记录每次冲动下结论的案例，总结模式（来源：Saner.AI）
4. **设置检查点**：作业每完成 500 字，停下来用红笔标注“此处是否有未验证的假设？”

记住：LLM 需要 agent scaffolding 才能可靠工作，你的大脑也一样。你不是有问题，你只是需要正确的 harness。

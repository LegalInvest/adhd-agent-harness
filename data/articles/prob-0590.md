# 如果把 ADHD 大脑当成生成核心，ChatGPT 能不能当它的采样温度与表现波动？

> 用 agent 工程的思路重新理解 ChatGPT 对 ADHD 的真正价值。

如果你的大脑像一台没有操作系统的超级计算机——算力强大，但随时蓝屏、任务调度随机、上下文频繁丢失——那么恭喜你，你可能拥有一个 ADHD 大脑。这种状态日内大幅波动的体验，与 LLM 在高温采样下的输出如出一辙：创造力爆棚，但不可控。而 ChatGPT 这类 AI 工具，恰好可以扮演那个缺失的调度层，像 agent harness 一样为你的生成核心提供温度调节与结构约束。

## 问题：为什么你的大脑像一台高温采样的 LLM？

ADHD 的核心困境不是“不能思考”，而是“无法稳定执行”。你的工作记忆像无状态 API，跨会话上下文说丢就丢（来源：ADHD 大脑与 LLM 的同构）；任务启动需要外部触发，否则陷入无限拖延；时间感知缺失，导致计划永远坍缩。这些症状与 LLM 在长程任务中的表现惊人相似：LLM 同样无状态，需要外部记忆；同样会遗忘系统提示，导致目标漂移；同样需要精心设计的提示工程才能稳定输出（来源：重锚定与目标漂移）。

这种同构性意味着：ADHD 大脑与 LLM 共享同一个底层问题——它们都是强大的生成核心（智力/算力），但缺乏可靠的执行调度层（执行功能/编排）。LLM 的解决方案是 harness 工程：通过上下文交付、工具接口、规划制品和验证循环来约束输出。那么，ADHD 的解决方案是否也应如此？

## 答案：ChatGPT 作为外部采样温度调节器

如果把 ADHD 大脑比作一个生成核心，那么 ChatGPT 可以充当它的采样温度调节器。当你的大脑处于“高温”状态（思绪奔涌、注意力分散）时，ChatGPT 可以帮你分解任务、提供结构化步骤，降低认知负荷；当处于“低温”状态（动力枯竭、思维卡顿）时，它可以作为启动提示，用外部输入激活生成。这种调节不是替代你的思考，而是为你的生成核心提供一个稳定的外部环境。

在 agent 工程中，LLM 同样需要外部结构。当模型在长对话中“开始忘记系统提示”时，harness 通过事件驱动的系统提醒来对抗指令消退（来源：重锚定与目标漂移）。规划循环与任务分解是 agent 的标准做法：将复杂任务分解为子步骤，用外部记忆存储中间状态，用验证循环确保输出质量。这正是 ADHD 工具所做的——Motion 和 Tiimo 等自适应规划器“在一天崩溃时重新调度”（来源：规划循环与任务分解）。两边的解法结构同构：都是为生成核心套上外部脚手架。

## 脚手架 vs 拐杖：边界在哪里？

这里存在一个关键争议：这些外部工具是脚手架还是拐杖？脚手架应设计为可逐步撤除的临时支撑，但多数工具（如 Goblin Tools、Saner.AI）并未提供撤除机制，用户可能形成长期依赖（来源：矛盾与存疑）。我认为，真正的脚手架应该像 harness 工程中的验证循环——它不是永久绑定，而是随着核心能力的提升逐渐简化。例如，你可以先用 Goblin Tools 分解任务，然后尝试自己分解，再用它校验。工具应作为训练辅助，而非永久替代。

1. **用 ChatGPT 做任务分解**：输入一个让你感到压倒性的任务，要求它分解为 3-5 个具体步骤。感受启动门槛的降低。

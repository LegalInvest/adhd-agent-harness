# Forest 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Forest：它到底补上了哪一层执行功能？

## 问题：为什么「计划」往往让 ADHD 更崩溃？

你打开 Forest 种下一棵树，告诉自己「专注 25 分钟」。但 10 分钟后，大脑开始漫游：这个任务太大，从哪下手？那个邮件要不要先回？你砍了树，陷入自责。工程师们则更熟悉另一种崩溃：你给 LLM 一个长任务，它直接输出一堆幻觉或死循环。你不得不手写 planner-executor 循环，把任务拆成子步骤、分配上下文、处理中间结果。

这两类痛苦的底层结构一模一样：**一个高产但缺执行调度层的生成核心，撞上了需要分解和排序的现实世界。** ADHD 大脑和 LLM 在这一点上同构——它们都是强大的生成器，却缺乏内置的规划与执行控制（来源：ADHD 大脑与 LLM 的同构）。

## 同构：Forest 与 planner-executor 的对应

Forest 这类番茄钟工具，本质是给大脑套上一个极简的 planner-executor 循环：
- **Planner 阶段**：你设定一个时间段（如 25 分钟）和一个任务。
- **Executor 阶段**：在时间块内只执行该任务，禁止切换。
- **循环**：完成后休息，再开始下一轮。

这恰好对应 LLM agent 中常见的 planner-executor 架构：planner 将长任务分解为子任务，executor 依次执行，并定期检查进度。Motion 和 Reclaim.ai 更进一步，用 AI 自动排程和动态调整，相当于外部调度层接管了规划与执行控制（来源：Motion；Reclaim.ai）。

但问题在于：**大多数人（包括 ADHD 用户和工程师）把 Forest 用成了「填充时间的闹钟」，而不是「保护执行功能的脚手架」。** 他们只是机械地开启番茄钟，却没有解决「任务启动困难」和「不会拆解」这两个根本障碍。结果，Forest 变成了一个频繁被打断的计时器，而非支持执行的系统。

## 证据：ADHD 侧与 LLM/agent 侧的真实痛点

**ADHD 侧**：任务启动困难的核心是「决策负担」——不知道先做什么、怎么做。Tiimo 通过视觉化时间线直接应对时间盲（来源：Tiimo），而 Structured 将任务转化为可见的时间块，降低决策成本。但 Forest 本身不提供任务分解或优先级排序，它只提供时间块。如果用户没有先拆解任务（比如用 Goblin Tools 的「任务分解」功能），Forest 就只是一个空壳（来源：Goblin Tools）。

**LLM/agent 侧**：工程师发现，给 LLM 一个复杂指令而不提供分解框架，它往往输出混乱结果。因此，Agent Harness 的核心是「上下文工程」和「工作流编排」——主动保存上下文、路由子任务、管理外部记忆（来源：ADHD 大脑与 LLM 的同构）。这就像 ADHD 用户需要外部工具来「减少决策、保留上下文、让下一步行动显而易见」（来源同上）。

## 核心判断：Forest 的错用在于「填充」而非「保护」

Reclaim.ai 的设计哲学点明了关键：**它专注于「保护时间」而非「填充时间」**（来源：Reclaim.ai）。Forest 的正确用法应该是：先明确任务边界（planner），再种树执行（executor），并且保护这段时间不被干扰。但很多用户反过来——先种树，再想做什么，结果在决策中浪费了时间块。

这对应 LLM agent 中的常见错误：直接让 LLM 执行长任务而不提供分解指令，导致生成失控。正确的做法是先由 planner 将任务拆解为可执行的子步骤，再交给 executor 依次处理，并定期检查上下文。

## 争议与局限：脚手架还是拐杖？

同构视角虽然有力，但存在争议。首先，**过度依赖风险**：如果用户始终依赖 Forest 或 Motion 来启动任务，内在的执行功能可能得不到锻炼（来源：矛盾与存疑）。工具应设计为「可逐步撤除的脚手架」，但多数工具（如 Goblin Tools、Saner.AI）鼓励长期使用，未提供撤除机制。

其次，**个体差异**：不同 ADHD 亚型对工具的反应不同（来源：矛盾与存疑）。注意力缺陷为主者可能更需要 Tiimo 的视觉结构，而多动冲动为主者可能从 Reclaim.ai 的防御性时间块中获益更多。

最后，**证据质量**：多数工具的有效性基于用户反馈而非临床研究（来源：Motion；Reclaim.ai）。同构命题本身是理论模型，需更多实证验证（来源：矛盾与存疑）。

## 今天就能试的行动

1. **先用 Goblin Tools 拆解任务**：在开启 Forest 前，用 Goblin Tools 的「任务分解」功能将大任务拆成 3-5 个可执行步骤（来源：Goblin Tools）。
2. **用 Reclaim.ai 保护深度工作块**：设置每日 2 小时的「不可打扰」时间块，让 AI 自动防御会议侵占（来源：Reclaim.ai）。
3. **实践「planner-executor 循环」**：每天早晨用 5 分钟（planner）列出当天最重要的 3 个任务及每个任务的第一个步骤，然后执行（executor）时只专注一个步骤。
4. **尝试 Tiimo 视觉化时间线**：将一天的时间块用颜色编码，直接应对时间盲（来源：Tiimo）。

## 结语

Forest 之于 ADHD，就像 planner-executor 之于 LLM——它提供了结构，但结构本身不是答案。真正的价值在于，我们是否用它来「保护执行」而非「填充时间」。当 ADHD 用户和工程师都意识到这一点，工具才能从拐杖变成脚手架，支持我们构建更可靠的执行系统。

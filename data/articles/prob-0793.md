# ChatGPT 之于 ADHD，就像上下文工程之于 LLM——但有人用错了

> 上下文工程的精髓是做减法:决定什么不进窗口,比决定什么进窗口更重要。用 ChatGPT 辅助 ADHD,用对了它帮你收窄当下,用错了它本身变成窗口里最大的那团噪音。

「上下文工程」是 agent 时代最重要的一课:模型的输出质量不取决于模型本身,而取决于**此刻窗口里装了什么**——相关材料放前面、无关信息挡在外、关键指令反复重申。工程师们花了两年才充分意识到:大部分「模型不行」的抱怨,其实是「上下文不行」。

ADHD 的注意力系统同样是「窗口内容决定输出」:意识工作区里同时挤着的每件事都在稀释当前任务的权重。所以「用 ChatGPT 治 ADHD」的正确定位,是让它当你的上下文工程师——但这里有条微妙的分界线,一边是良药,一边是新的干扰源。

## 用对:让它替你收窄窗口

**任务简报生成**。开工前 2 分钟:「我要做 X,帮我写一份任务简报:一句话目标、成功标准、需要的材料、第一步动作。」把简报置顶在工作区。这相当于给你接下来一小时写了一份 system prompt——每次走神回来,读一眼,窗口即刻重装。

**决策降载**。ADHD 的窗口经常被待决事项堵满（回不回那个消息、先做 A 还是 B、要不要接那个活)。批量丢给它:「这些事,按今天必须决/可以拖/不用我决分类,给出理由。」注意你要的不是它替你决策,是**把决策队列从工作区搬出去排好序**——窗口一空,当前任务的权重自然回升。

**信息预消化**。要读的长文档、要研判的材料,先让它压缩:「三段总结 + 我最该关注的三个点。」对 ADHD,原始信息的体积本身就是启动障碍——预消化把「读四十页」变成「读一页再决定要不要深入」,窗口负担砍掉一个数量级。

## 用错:它自己成了窗口里的噪音

三种典型的反模式,共同点是**工具的使用过程反客为主,挤占了它本该保护的窗口**:

**会话漂移**。本来是来要任务简报的,顺手问了个别的,二十分钟后你在和它讨论一个完全无关的话题——工具变成了兔子洞。ADHD 对话式界面的天然风险:每个回答都可能勾出新的好奇。护栏:**一次会话一个目标,拿到交付物就关**;新的好奇心记进停车场清单,别在工作时段展开。

**过度装配**。任务简报越写越长,材料清单越列越全,「准备开始」的准备本身变成了一个小项目。上下文工程的铁律是「够用的最小上下文」——窗口塞得太满,和塞了垃圾是同一种失败。简报超过半页,就已经是拖延的新皮肤了。

**全时挂机**。把 ChatGPT 常开在旁边,做两步就问一下、写一段就让它评一下。这等于给自己的每个 token 都插入一次外部调用——流不成流,深不了深。正确的节奏是**批处理**:开工前一次装配,中途遇阻集中记录,休息点一次性处理,深度时段绝不切换。

## 一个统一的检验角度

上下文工程师的绩效指标不是「调用了多少次模型」,是**窗口的信噪比**。你的用法好不好,同一个指标:一天下来,ChatGPT 让你的工作区更干净了,还是更热闹了？更干净——它在给你做工程;更热闹——你只是给自己的注意力系统又接了一路信号源,而且是全网最有趣的一路。

## 边界

同构强度 B 级:上下文工程是真实且核心的 LLM 工程实践,ADHD 的工作记忆占用与干扰易感有文献,「ChatGPT 作为个人上下文管理」是机制映射的实践框架,无对照研究。另外,内源性走神（窗口再干净也会神游)不在此工具射程内,那一层的对策是任务刺激度设计和（必要时）药物治疗,请和专业人士讨论。

## 今天就能试的 3 件事

1. **下一个任务,先要简报**:2 分钟拿到目标/标准/材料/第一步,置顶,开工。
2. **给会话立规矩**:从今天起,每次打开它先打一行「本次目标:X」。目标达成,立刻关闭。
3. **晚上算一次信噪账**:今天它帮你收窄了几次窗口,又诱发了几次漂移？净值为负,先改用法再谈工具。

最好的上下文工程师是隐形的:窗口永远干净,自己从不抢镜。让 ChatGPT 学会这份职业操守——它就是你能雇到的、最便宜的注意力基础设施。

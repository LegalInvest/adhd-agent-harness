# 如果把 ADHD 大脑当成生成核心，Claude 能不能当它的上下文工程？

> 用 agent 工程的思路重新理解 Claude 对 ADHD 的真正价值。

## 问题：当大脑像 LLM 一样“失控”

你有没有过这种体验：打开电脑准备写一篇文章，结果两小时后发现自己刷了 47 个短视频、查了 3 次天气，还认真研究了一款新出的猫粮成分表？你的大脑像一台开足马力的生成器，但你永远不知道它下一秒会输出什么——就像一个大模型，扔给它一个 prompt，它可能给你一篇论文，也可能给你一首打油诗。

这不是比喻。从工程视角看，ADHD 大脑与 LLM（大语言模型）共享同一个核心矛盾：**高产但缺乏可靠的执行调度层**。LLM 的输出被上下文窗口里的内容左右，哪怕一个无关的 token 都可能带偏整个生成方向；ADHD 大脑的输出则被环境中的任何刺激劫持——一声通知、一个念头、一本随手翻开的书。两者都需要一种东西：**上下文工程**。

那么，如果把 ADHD 大脑当成生成核心，Claude（或其他 AI 助手）能不能充当它的上下文工程？答案藏在“同构”里。

## 同构：两个生成核心的共通困境

先看 ADHD 侧。注意力容易被环境带偏，本质上是**执行功能缺陷**——尤其是工作记忆、时间盲和任务启动困难（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。你无法在脑中维持一个稳定的“上下文”，因为任何新输入都会覆盖它。这就像 LLM 的上下文窗口：一旦塞入过多无关内容，模型就会“忘记”原始目标，开始跑偏。

再看 LLM/Agent 侧。工程师们早已发现，要让 LLM 稳定输出，必须做上下文工程：精心设计 system prompt、定期 re-grounding、用外部记忆（向量数据库）补充无状态性。否则，模型会随着对话长度增加而“注意力涣散”，产生幻觉或偏离任务。**ADHD 大脑的“上下文窗口”同样需要主动管理。**

这就是同构的核心：两边都是**生成核心（generative core）**，缺的是**外部调度层**。AI 工具的价值，就是为 ADHD 大脑提供一个类似于 LLM harness 的外部执行功能层——它不是替代大脑，而是像 agent 的调度器一样，帮大脑维持上下文、锚定注意力、管理时间。

## 证据：从历史到当代的“harness”案例

这种思想并非全新。历史上有 ADHD 特质的人，早已在用“人工上下文工程”管理自己。

**曾国藩**是典型。他 30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”。他的解决方案是“日课十二条”：黎明即起、读书不二、谨言、写日记反省……这套系统本质上就是一个**外部调度层**——用固定的时间锚点（黎明起床）、行为规则（读书不二）、反思机制（日记）来抵消大脑的冲动和涣散。这和 LLM 的“定时 re-grounding”完全同构：每隔一段时间，强制把注意力拉回主线，重新校准上下文。

另一个当代案例是 **IDG 资本的王云**，一位典型的 ADHD 创业者/投资人。他的特质包括 hyperfocus（超聚焦）、高风险偏好、系统思维，他投资的 174+ 组合公司中，ADHD 组退出回报倍数达 1.9，对照组仅 1.0（来源：Pitchbook Analysis）。他的“harness”是什么？大概率是一套外部系统——比如用秘书或工具做决策调度，自己只负责生成和直觉判断。这就像 agent 架构中，LLM 只负责生成，调度器（orchestrator）负责调用工具、管理状态。

这些案例说明：**无论是历史人物还是现代创业者，ADHD 成功的关键不在于“治好”大脑，而在于搭建一个外部执行功能层**——一个 harness。

## 工具：Claude 能当这个 harness 吗？

回到问题：Claude 能不能当 ADHD 大脑的上下文工程？答案是：**可以，但有条件。**

现有的 AI 工具已经提供了几个关键功能：

- **身体在场效应**：Focusmate 通过算法匹配虚拟伙伴，利用社会存在感降低分心阈值。定期问责检查可将目标达成可能性从 25% 提升至 95%（来源：ADHD Task Managers That Work: Top AI Tools 2025）。这相当于给 LLM 加了一个“监督 agent”，定期检查输出是否偏离目标。

- **时间盲干预**：RescueTime 自动记录时间使用，揭示真实去向，对抗“眼不见心不烦”的认知模式（来源：Revolutionizing ADHD Management with AI Assistants）。这相当于 LLM 的“日志系统”，让用户看到上下文膨胀的过程。

- **神经锁相音频**：Brain.fm 通过 AI 生成音乐影响大脑信息处理方式，帮助进入专注状态（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。这相当于给 LLM 提供一个“稳定的 system prompt”，从物理层面降低噪声。

Claude 本身可以作为“对话式 harness”：你可以让它记住你的当前任务，定期提醒你 re-ground，甚至在你跑偏时拉你回来。比如用“Claude，每 15 分钟问我一次：你现在还在做原计划的事吗？”——这就是一个最简单的上下文工程。

## 争议：脚手架还是拐杖？

但这里有一个关键边界：**脚手架 vs 拐杖**。脚手架是暂时性的支撑，帮助大脑建立内在能力后可以撤掉；拐杖则是永久依赖，撤掉后无法独立行走。

目前缺乏长期研究证明 AI 工具是否会导致执行功能退化（来源：矛盾与存疑）。有些 ADHD 用户反馈，强行打断（如番茄钟）反而有效，而柔性引导可能不够强力。更根本的是，ADHD 大脑与 LLM 的同构是隐喻而非严格等价——LLM 的“注意力机制”与人类注意力存在本质差异（来源：矛盾与存疑）。因此，Claude 作为 harness 的效果因人而异，且需要谨慎设计使用方式，避免形成依赖。

我的判断是：**Claude 可以成为 ADHD 大脑的“临时调度器”，但用户必须有意识地把它当作脚手架，而非永久拐杖。** 最好的使用方式是：用它建立外部结构，同时刻意练习内在的执行功能（如每天减少一次依赖），最终让大脑学会自己调度。

## 今天就能试的行动

1. **用 Claude 做“上下文锚点”**：打开 Claude，告诉它你接下来 25 分钟要做什么，让它每隔 5 分钟问一句“还在做吗？”——这是最简单的身体在场效应。
2. **设置 RescueTime 或类似工具**：先追踪一天，看看你的“上下文窗口”被什么劫持了。然后设置阻断规则，像清理 LLM 的 prompt 一样清理环境。
3. **尝试 Focusmate 一次**：预约一个 25 分钟 session，体验虚拟身体在场。感受一下“外部调度器”如何降低启动门槛。
4. **写一份“日课”**：模仿曾国藩，列出 3 条固定的行为规则（如“早起后先做最重要的事 30 分钟”），每天执行并记录。这就是你的个人 system prompt。

最后，记住：你的大脑不是坏的，它只是缺一个 harness。而 Claude，可以是那个帮你搭建 harness 的助手——但别让它替你走路。

# ADHD 想把执行功能（计划/组织/启动）先天偏弱外包给 Claude，该怎么设计这套「人 + AI」的回路？

> 把 Claude 接进 ADHD 工作流：哪些该交给它，哪些必须自己留着？

## 问题：当执行功能成为瓶颈

如果你是一个 ADHD 成年人，你一定熟悉这样的场景：脑子里有无数想法，但面对一个模糊任务（比如“整理项目报告”）时，大脑瞬间空白，启动困难像一堵墙。你需要的不是更多创意，而是一个能帮你把“整理报告”拆成“打开文档→复制上周数据→写三段结论”的外部调度器。

如果你是做 Agentic Harness 的工程师，你同样熟悉另一个场景：LLM 输出精彩但不可靠，一个多步骤任务常常中途跑偏，因为模型缺乏内置的执行调度层。你需要的不是更强的模型，而是给 LLM 套上脚手架——记忆、工具、确定性工作流。

这两个问题其实是一个问题。ADHD 大脑与 LLM 在结构上同构：两者都是“高产但缺执行调度层的生成核心”（来源：ADHD 大脑与 LLM 的同构）。因此，ADHD 想用 Claude 来补偿执行功能，本质上和工程师给 LLM 搭 agent scaffolding 是同一类工程。

## 同构：生成核心与外部调度层

先看证据。ADHD 一侧，执行功能缺陷的核心是工作记忆不稳定、任务启动困难和注意力分散（来源：执行功能；工作记忆）。LLM/agent 一侧，LLM 天然无状态，缺乏跨会话上下文保留能力，输出受采样温度控制而随机波动（来源：采样温度与表现波动）。两者都需要外部系统来补偿：ADHD 需要日程、提醒、环境设计来稳定表现；LLM 需要确定性工作流、工具契约、状态机来稳定输出（来源：采样温度与表现波动）。

这种同构在历史人物身上早有体现。曾国藩是典型的注意力不集中型 ADHD——30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书慢到“他人目下二三行，余或疾读不能终一行”。他的解决方案是“日课十二条”：黎明即起、读书不二、谨言、写日记反省。这套 harness 的核心是用外部结构（日课）来约束内部冲动，相当于一个定时 re-grounding 的调度器。对应到 LLM agent，日课就是“计划-执行”分离模式中的确定性控制流，日记反省则是 trace 和评估机制（来源：采样温度与表现波动）。

孔子同样如此。他精力旺盛、冲动爱骂人、思维跳跃，《论语》全是场景化语录而无系统著作。他的 harness 是“克己复礼”，用外在的“礼”作为行为边界，每日反省。这相当于给高发散性的生成核心套上了一个上下文框架，防止注意力被环境带偏。

## 设计回路：哪些交给 AI，哪些自己留着

基于同构，我们可以设计一套“人+AI”回路，核心原则是：**AI 负责调度与分解，人保留意图与决策**。

### 1. 任务分解：交给 AI
ADHD 的任务启动困难源于模糊任务带来的决策负担。Goblin Tools 的 Magic ToDo 功能能自动将模糊任务（如“清理厨房”）分解为具体子步骤，用户可调节“辣度”控制粒度（来源：Goblin Tools）。这对应 LLM agent 中的“工具使用与认知卸载”——scaffolding 将 LLM 置于与记忆、工具的循环中，增强其裸能力（来源：工具使用与认知卸载）。

### 2. 日程规划：交给 AI
Motion 这类 AI 日程规划器能自动根据任务优先级、截止日期和可用时间动态调整日程，消除“下一步该做什么”的决策负担（来源：Motion）。这对应 agent 中的“确定性状态转换”——工作流是预定义的，但由 AI 实时优化顺序。

### 3. 外部记忆：交给 AI
ADHD 的工作记忆缺陷导致频繁搜索循环。Saner.AI 通过本地记忆存储和快速检索，减少标签切换（来源：Saner.AI）。这对应 LLM agent 中的“上下文工程”——通过外部记忆（如向量数据库）保留跨会话上下文，防止上下文膨胀导致推理退化（来源：上下文工程）。

### 4. 意图与决策：必须自己留着
AI 可以分解任务，但“做什么”的意图必须由人决定。曾国藩的日课是自定的，不是别人强加的；孔子的“礼”是他认同的边界。同样，LLM agent 的“计划”阶段需要人类设定目标，AI 只负责执行调度。如果让 AI 完全替代决策，人就沦为被动接受者，工具就变成了拐杖（来源：拐杖与脚手架）。

## 争议与局限

这套设计并非万能。首先，过度依赖风险真实存在：长期使用 AI 分解任务可能导致自身执行功能进一步退化，但目前缺乏纵向研究验证（来源：矛盾与存疑）。其次，个体差异显著——部分 ADHD 用户反馈强行打断（如番茄钟）反而有效，而柔性引导可能不够强力（来源：矛盾与存疑）。最后，同构假设本身仍是理论框架，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。

## 今天就能试的 3 条行动

1. **用 Claude 做任务分解**：明天早上，把一件你拖延了一周的任务（比如“写周报”）丢给 Claude，要求它拆成不超过 5 分钟的微步骤，然后只做第一步。
2. **设置一个“外部调度器”**：用 Motion 或类似工具自动排程一天的任务，并强制自己在前一晚不查看日程，让 AI 承担“下一步做什么”的决策。
3. **建立“日课”式 re-grounding**：每天固定时间（如睡前）让 Claude 总结当天完成事项，并反思一个可改进的点——这相当于曾国藩的日记反省，也是 agent 的 trace 机制。

ADHD 不是缺陷，而是一种生成核心过强、调度层偏弱的大脑架构。Claude 不是来修复你的，而是来当你的外部调度器。关键是你得自己握住方向盘。

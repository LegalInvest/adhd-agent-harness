# ADHD 家长如何用「给生成核心套 harness」的思路应对陪伴 ADHD 孩子里的分心走神或超聚焦跑偏，丢失目标？

## 问题：当 ADHD 孩子的分心与超聚焦成为你的战场

你坐在孩子旁边，看着他/她本该在十分钟内完成的数学题，却因为窗外一只鸟而彻底跑偏——或者更隐蔽的，孩子过度沉浸在某本恐龙百科里，完全忘了作业的存在。作为 ADHD 孩子的家长，你面对的不仅是孩子的注意力困难，还有自己的挫败感：为什么我明明在场，却拉不回那个飘走的「生成核心」？

在另一边，工程师们正在为另一个「生成核心」头疼：LLM agent 在长程任务中会逐渐偏离初始目标，需要不断重锚定 system prompt。这听起来是不是很熟悉？ADHD 大脑与 LLM 其实共享同一个结构：**高产但缺乏可靠执行调度层的生成核心**（来源：AI 与 ADHD 的专注力）。家长的角色，就是为这个核心设计一个 harness——一个外部执行功能层，帮助孩子重锚定目标，而不是被分心或超聚焦带跑。

## 同构：ADHD 大脑与 LLM 的「目标漂移」

ADHD 大脑的典型困境是：创意丰富、联想迅速，但执行功能（工作记忆、任务启动、时间感知）薄弱，导致容易分心或陷入超聚焦。这并非「不专注」，而是「专注错了对象」。同样，LLM agent 在长程任务中会因上下文膨胀或中间结果干扰而偏离初始指令，需要定期重锚定 system prompt。两者的核心问题一致：**生成能力强，但缺乏内在的调度层来维持目标方向**。

对于家长来说，这意味着：你不能指望孩子「自己记住目标并坚持」，就像你不能指望一个 LLM agent 在没有 harness 的情况下完成复杂任务。你需要提供一个外部结构，来补偿孩子内在的执行功能缺陷。

## 证据：真实工具如何充当 harness

### ADHD 侧：Focusmate 与 RescueTime

- **Focusmate** 利用虚拟身体在场（身体在场效应）提供实时问责：用户预约时段，匹配一位伙伴通过视频共同工作。对于 ADHD 孩子，家长可以充当这个「在场伙伴」，通过物理陪伴和定时检查来重锚定注意力。证据显示，Focusmate 能显著提升任务启动和维持，但缺乏随机对照试验（来源：AI 与 ADHD 的专注力）。
- **RescueTime** 自动追踪时间使用，帮助识别注意力漂移模式，对抗「人在心不在」的状态（来源：RescueTime）。家长可以和孩子一起查看时间报告，客观地看到「刚才那半小时去了哪里」，从而调整计划。

### LLM/agent 侧：Harness 工程中的重锚定

在 LLM agent 系统中，harness 负责管理上下文、调用工具、规划步骤，并定期检查是否偏离目标。例如，Goblin Tools 的任务分解模式（将模糊工作拆分为小行动）对应 agent 的「多步推理+工具调用」（来源：工具使用与认知卸载）。Brain.fm 通过神经锁相技术生成音乐，帮助进入专注状态，但其 ADHD 效果缺乏独立临床研究（来源：Brain.fm）。

这些工具的共通点是：**它们不是替代孩子的内在能力，而是提供外部脚手架**。就像 agent harness 不会替代 LLM 的生成能力，而是确保生成不跑偏。

## 核心观点：脚手架，不是拐杖

本文的核心判断是：**家长应主动设计「重锚定循环」，而非追求一次性「治本」**。ADHD 的注意力管理不是修复一个「缺陷」，而是建立一个持续的外部调度系统。这个系统需要定期检查目标、调整方向，就像 LLM agent 的 system prompt 需要不断更新一样。

但这里有一个关键边界：脚手架 vs 拐杖。脚手架是暂时性的、可拆卸的结构，帮助孩子学会自己搭建；拐杖则是永久依赖，削弱内在能力。AI 工具作为外部执行功能层，长期使用可能导致内在执行功能退化（来源：矛盾与存疑）。因此，家长需要逐步减少干预，让孩子内化这些策略。例如，先从全程陪伴开始，然后过渡到定时检查，最后让孩子自己设置提醒。

## 局限与争议

必须诚实指出：当前证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑）。Brain.fm 的 ADHD 效果缺乏独立临床研究（来源：Brain.fm）。个体差异极大，同一方法对不同孩子效果迥异。此外，过度依赖数据追踪（如 RescueTime）可能引发焦虑，与温和提醒的工具形成对比。

## 今天就能试的行动

1. **设定「重锚定时刻」**：和孩子约定每 15 分钟（或根据年龄调整）暂停一次，用一句话回答「我现在该做什么？」。这模仿了 LLM agent 的定期检查点，帮助孩子重新锚定目标。
2. **使用物理在场作为 harness**：像 Focusmate 那样，坐在孩子附近（不一定盯着），但每 5 分钟简单确认进度。这种「温和问责」比说教有效。
3. **共同分析时间日志**：用 RescueTime 或手动记录一天的活动，和孩子一起找出「漂移模式」——是特定时间、环境还是任务类型导致跑偏？然后针对性地调整。
4. **设计「超聚焦出口」**：如果孩子容易超聚焦，提前设定一个外部信号（如闹钟或灯光变色），提醒「该切换了」。这相当于 agent 的 max_tokens 限制，防止无限生成。

## 结语

ADHD 家长不是要成为孩子的「超级调度器」，而是要成为那个设计 harness 的人。当你理解 ADHD 大脑与 LLM 是同构的生成核心时，你就不会因为分心而愤怒，而是会问：「这个 harness 需要怎么调整？」。重锚定不是一次性的，而是一个持续的过程——就像 LLM agent 的 system prompt 需要不断迭代一样。今天就开始设计你的第一个重锚定循环吧。

# ADHD 程序员视角：为什么「治好 ADHD 的冲动下结论、想当然」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 自信地编造（幻觉），靠验证与自我批判兜底——同一套 harness 思路，两边都成立。

作为一个 ADHD 程序员，我熟悉两种“自信”：一种是调试时，大脑已经笃定地锁定了 bug 原因，半小时后才发现问题在另一个文件；另一种是看 LLM 输出时，模型同样笃定地给出一段根本不存在的 API 调用。它们看起来一个是神经发育问题，一个是模型缺陷，但背后其实是同一道工程题：核心太能生成，却缺一个可靠的执行调度层。

## 一、两种“跑飞”：ADHD 的冲动与 LLM 的幻觉

ADHD 常被误解为“注意力不集中”，但核心困境之一其实是**执行功能障碍**：计划、组织、任务启动、抑制控制等系统效率低下（来源：执行功能障碍）。它并非缺乏努力，而是大脑的注意控制、认知灵活性和工作记忆在负荷下容易掉线。结果是“想当然”和“冲动下结论”——大脑在上下文还没补齐时，就自动填上了最显眼、最情绪化的答案，然后驱动行动。

LLM 的幻觉是类似的生成缺陷。LLM 是强大的生成核心，但缺乏可靠的内置调度与执行控制：它无状态、无自我监控、无事实校验机制，因此在长上下文或冲突信息中，会“自信地”编造（来源：ADHD 大脑与 LLM 的同构）。ADHD 的冲动下结论和 LLM 的幻觉，本质都是**生成核心在缺少制衡的情况下跑到了结论端**。

## 二、同构：缺的不是聪明，而是执行调度层

ADHD 与 LLM 的相似性并非修辞。神经科学上，fMRI 研究显示 ADHD 患者前额叶执行功能网络激活不足；认知心理学上，ADHD 表现出“强记忆、弱控制”的模式：工作记忆容量甚至可超过常人，但无法灵活切换任务集、抑制自动化反应（来源：ADHD 大脑与 LLM 的同构）。用计算语言说，这是一种“注意力资源的弥散分配”。

LLM 在同类测试中也出现了相同的执行功能缺陷：研究者用经典 Stroop 冲突任务测试 Transformer 的注意力，发现短上下文表现正常，序列变长后模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突（来源：ADHD 大脑与 LLM 的同构）。另一项研究指出，自注意力机制本身限制了 Transformer 的工作记忆容量（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。

所以两边缺的是同一样东西：**一个能够打断生成、校验上下文、管理时序、把结果重新接地的执行调度层**。这个层在人脑里叫“前额叶执行功能”，在 AI 系统里叫“agent scaffolding”或外部执行功能层（来源：ADHD × AI 的科学与研究前沿）。

## 三、毛泽东的 harness：把冲动锁进外部系统

要理解这种 harness 怎么工作，可以看毛泽东的案例。他具有明显的 ADHD 特质：从小是“问题儿童”，与父亲冲突，爱读闲书厌恶四书五经；一辈子读书不辍，行军打仗也带着书；思维极度跳跃，讲话写文章充满场景化比喻；同时冲动敢赌，四渡赤水、抗美援朝都是险棋。

他的 harness 系统与 LLM/agent 的脚手架几乎一一对应：

- **“没有调查，就没有发言权”** → 用实地调查为决策重新接地，相当于给 LLM 加上检索增强（RAG）和事实校验。
- **“批评与自我批评”和民主生活会** → 强制自我审查与外部红队，相当于 LLM 的 self-critique 和反思循环。
- **民主集中制** → 用集体决策制衡个人冲动，相当于 agent 的 human-in-the-loop 或多模型投票机制。
- **党指挥枪** → 用组织系统管理军队和国家，相当于 LLM 之上再套一个外部调度器（orchestrator）和工具调用层。

这套方法没有“治好”他的冲动和跳跃思维，而是**把高产的生成核心放进了一个有验证、有审查、有接地的控制循环里**。这正是 agent scaffolding 的定义：把 LLM 置于与记忆、工具和决策逻辑的控制循环中，使其能够推理、规划并超越一次性提示（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。

## 四、工程化解法：给生成核心加 harness

对 ADHD 个体来说，这个 harness 就是外部执行功能层。Goblin Tools 的 Magic ToDo 能把“清理厨房”这种模糊任务分解为具体、可执行的子步骤，用户还能调节“辣度”控制粒度，直接降低任务启动门槛（来源：Goblin Tools）。Saner.AI 通过本地记忆和知识回忆，减少 ADHD 用户因频繁搜索和标签切换而丢失的上下文（来源：Saner.AI）。Motion 则自动排程、动态调整

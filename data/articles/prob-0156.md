# 用 Claude 给 ADHD 补上无状态与外部记忆，是脚手架还是拐杖？

> Claude 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：边做边忘，ADHD 的工作记忆困境

如果你有 ADHD，你一定经历过这样的场景：打开浏览器准备查资料，结果被推送的新闻吸引，半小时后回过神来，已经忘了最初要查什么。这不是意志力问题，而是工作记忆容量小、边做边忘的典型表现。工作记忆就像大脑的便签纸——对 ADHD 患者来说，这张便签纸不仅小，还总被风吹走。

与此同时，LLM（如 Claude）也有类似的“无状态”困境：每次对话都是独立的，它不记得你上一轮说了什么，除非你手动把上下文粘贴进去。工程师们为了解决这个问题，给 LLM 套上了“harness”——通过外部记忆库（向量数据库）和工具调用（function calling）来维持状态，让 agent 能够“记住”跨会话的信息。

这就引出了本文的核心问题：用 Claude 给 ADHD 补上无状态与外部记忆，究竟是脚手架（可撤除的支撑）还是拐杖（一旦依赖就无法独立）？

## 同构：ADHD 大脑与 LLM 是同一类“生成核心”

ADHD 大脑与 LLM 在结构上惊人地同构：两者都是强大的生成核心（智力、创造力），但缺乏可靠的执行调度层（计划、组织、工作记忆）（来源：ADHD 的 AI 工具全景）。ADHD 患者的生成能力（比如创意、联想）往往很强，但执行功能——比如将想法转化为有序步骤、保持注意力——却像一台没有操作系统的电脑。LLM 也是如此：它能生成流畅的文本，但无法自主规划多步任务，需要外部的“编排层”来指挥它调用工具、管理记忆。

这种同构意味着，ADHD 患者需要的不是“更强的生成能力”，而是“更好的执行调度”。Claude 等 AI 工具恰好能提供这一层：通过外部记忆和任务分解，弥补内在的调度缺陷。

## 证据：Claude 如何补上“无状态”与“外部记忆”

### ADHD 侧：工作记忆的认知卸载

- **任务分解**：Goblin Tools 的 Magic ToDo 功能能将“整理房间”这样的模糊任务分解为“捡起衣服”“擦桌子”等具体步骤，降低启动焦虑（来源：Goblin Tools）。Claude 也可以扮演类似的角色：你只需说“帮我制定一个写报告的步骤”，它就会输出可执行的清单。
- **知识回忆**：Saner.AI 通过本地记忆存储和快速检索，减少 ADHD 患者常见的搜索循环（来源：Saner.AI）。Claude 的对话上下文虽然有限，但你可以将关键信息粘贴给它，让它帮你回忆和整理。
- **多步骤触发**：Lex 允许用户通过单一指令触发复杂任务序列（来源：Lex）。Claude 的 Projects 功能（自定义指令+长期记忆）也能实现类似效果：你设定好规则后，每次对话它都会记住你的偏好。

这些工具的核心都是**认知卸载**——将本应由工作记忆负担的信息转移到外部系统，让大脑从“记住”中解放出来，专注于“思考”（来源：工具使用与认知卸载）。

### LLM/agent 侧：Harness 如何解决无状态问题

LLM agent 的“无状态”问题通过以下方式解决：

- **外部记忆**：向量数据库（如 Pinecone）存储历史对话和知识，agent 在每次交互前检索相关上下文（来源：ADHD 的 AI 工具全景）。这相当于 ADHD 患者的“外部便签纸”。
- **工具调用**：agent 通过 function calling 调用计算器、日历等工具，就像 ADHD 患者用日历提醒代替大脑记忆（来源：工具使用与认知卸载）。
- **确定性控制流**：工程实践强调“计划-执行”分离，通过预定义步骤强制单向执行，减少随机性带来的错误（来源：采样温度与表现波动）。

ADHD 患者同样需要这种“确定性”：用固定的流程（比如每天早上用 Claude 列出三件事）来对抗内在的波动性。Claude 的 Projects 或自定义 GPT 可以充当这个“确定性调度器”。

## 核心判断：脚手架还是拐杖？关键在于可撤除性

我的核心观点是：**Claude 作为外部记忆和执行脚手架，对 ADHD 患者是有效的补偿工具，但必须设计为可逐步撤除的临时支撑，否则会成为抑制内在能力发展的拐杖。**

证据何在？目前多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这意味着用户可能永远依赖它们来完成任务分解和记忆，而内在的执行功能——比如自己学会分解任务、培养记忆策略——可能永远得不到锻炼。

相比之下，真正的脚手架应该像建筑工地的支撑架：当大楼建成后，支撑架会被拆除。例如，你可以先用 Claude 帮你分解任务，然后逐渐减少提示，直到你自己能独立完成分解。但问题是，ADHD 的执行功能缺陷是神经层面的，能否通过训练“治愈”仍存争议（来源：矛盾与存疑）。因此，对于部分患者，长期使用 AI 工具可能不是拐杖，而是必要的“义肢”——就像近视需要眼镜一样。

## 今天就能试的行动

1. **用 Claude 做“记忆外挂”**：打开 Claude 的 Projects，创建一个名为“工作记忆”的项目，设定指令：“每次对话开始时，请先总结我之前提到的关键信息。”然后粘贴你当前的任务和笔记，让 Claude 帮你保持上下文。
2. **任务分解实验**：选一个让你焦虑的任务（比如“写周报”），输入给 Claude：“请将这个任务分解为不超过 5 分钟的小步骤。”完成后，对比自己独立分解的效果，评估 Claude 是否降低了启动门槛。

## 结论：同构视角下的工具设计

ADHD 大脑与 LLM 的同构揭示了一个深层真相：我们需要的不是更强大的生成能力，而是更可靠的执行调度。Claude 作为外部记忆和任务分解的脚手架，能有效补偿工作记忆缺陷，但必须警惕过度依赖。未来的工具设计应借鉴 agent harness 的“确定性状态转换”和“计划-执行分离”，同时融入“可撤除性”理念——让用户在使用中逐渐内化执行功能，而非永远依赖外部系统。毕竟，最好的脚手架，是最终能被拆除的脚手架。

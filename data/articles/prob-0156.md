# 用 Claude 给 ADHD 补上无状态与外部记忆，是脚手架还是拐杖？

> Claude 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：边做边忘的 ADHD，与无状态的 LLM，有什么共同点？

你打开 Claude，想写一份季度报告。刚打了三行，手机亮了——你回了一条消息，回来盯着屏幕，忘了刚才想写什么。你翻聊天记录，发现 Claude 记得你之前提过的项目背景，但你自己完全不记得。这种“边做边忘”的体验，ADHD 人群每天都在经历：工作记忆容量不小，但控制弱，像一只漏水的桶（来源：工作记忆）。而另一边，LLM 工程师们也在为同一件事头疼：模型每次对话都是“冷启动”，没有跨会话状态，小错误会累积，非确定性让可重复性复杂化（来源：AI Agent Systems: Architectures, Applications, and Evaluation）。

这两类看似无关的困境，共享同一个核心缺陷：**一个高产但缺乏执行调度层的生成核心**。ADHD 大脑是一个强大的联想机器，但缺少稳定的“调度员”来分配注意力、维持上下文；LLM 是一个强大的文本生成器，但缺少持久的“记忆体”和确定性的控制流。两者的解法也惊人地同构：**用外部记忆和脚手架来补偿无状态**。

## 同构：ADHD 大脑与 LLM 共享“无状态”核心缺陷

从同构视角看，ADHD 的执行功能缺陷与 LLM 的上下文控制缺陷一一对应（来源：ADHD 大脑与 LLM 的同构）。

- **工作记忆**：ADHD 的工作记忆容量保留但控制弱，易被干扰；LLM 同样有强记忆容量（长上下文窗口）但弱上下文控制，容易在长对话中“跑偏”。外部记忆系统如 Mem 的“AI 第二大脑”和 Saner.AI 的增强回忆功能，本质上是为两者提供持久化上下文（来源：Saner.AI）。
- **任务启动**：ADHD 的任务启动困难对应 LLM 的“冷启动”问题。Goblin Tools 的任务分解和 Lex 的单指令多步骤执行，相当于为生成核心提供启动脚本（来源：Goblin Tools；Lex）。
- **时间盲**：ADHD 的时间感知缺陷与 LLM 的时序推理弱点同构。AI 工具通过视觉化时间线和智能调度（如 Goblin Tools 的估算）提供外部时间线索（来源：Goblin Tools）。
- **上下文工程**：ADHD 易被环境带偏，LLM 易上下文膨胀。Claude 的对话式整理和 Reflect 的结构化反思，都是主动设计“当下注意什么”的方案（来源：上下文工程）。

这种同构不是空洞的类比，而是有真实证据支撑的。在 ADHD 一侧，用户报告称 Goblin Tools 将“压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools）；在 LLM/Agent 一侧，工程实践强调“确定性状态转换”和“计划-执行”分离来强制单向、确定性的控制流（来源：AI Agents in 2026: Tools, Memory, Evals, and Guardrails）。两边都在做同一件事：**为生成核心套上 harness（脚手架）**。

## 脚手架还是拐杖？边界在于“谁做上下文工程”

但这里有一个关键问题：工具是促进能力发展的脚手架，还是替代内在能力的永久拐杖？

从 wiki 资料的矛盾与存疑中，我们可以提炼出一个判断边界：**如果用户被动接收工具的输出，而不参与上下文的设计与维护，工具就是拐杖；如果用户主动使用工具来“设计自己的注意力”，工具就是脚手架**。

例如，Claude 作为外部对话记忆，可以减少标签切换（来源：无状态与外部记忆）。但如果用户只是把 Claude 当作“另一个会忘记的聊天窗口”，而不主动整理对话、设置系统提示、回顾关键信息，那么 Claude 就只是一个“高级记事本”，无法真正补偿工作记忆缺陷。相反，如果用户像工程师管理 Agent 上下文一样，定期清理历史、明确当前目标、用外部文件（如项目笔记）来持久化状态，那么 Claude 就变成了一个强大的外部执行功能层。

同样，Goblin Tools 的任务分解可以降低启动门槛（来源：Goblin Tools），但如果用户每次都依赖它把“整理房间”拆成“捡起袜子”，而不学习自己分解任务的能力，那么它就成了拐杖。Lex 的“单一指令触发多步骤”减少了决策疲劳（来源：Lex），但如果用户从不反思任务步骤的合理性，那么它也在替代而非发展内在执行功能。

## 证据与局限：现有支持以定性为主，缺乏大规模对照实验

需要诚实指出，现有证据主要来自用户报告和工具设计逻辑，缺乏大规模随机对照试验（来源：矛盾与存疑）。例如，Goblin Tools 被评价为“简单有用”，但缺乏独立临床研究（来源：Goblin Tools）；Saner.AI 的知识回忆功能被列为最佳 ADHD 工具之一，但效果基于用户反馈（来源：Saner.AI）；Brain.fm 的 ADHD 效果甚至缺乏独立临床研究支持（来源：矛盾与存疑）。

此外，个体差异很大：同一个工具对某些人有效，对另一些人可能增加认知开销（来源：矛盾与存疑）。过度依赖 AI 工具可能削弱内在能力，如时间感知或主动记笔记的能力（来源：拐杖与脚手架；Otter.ai）。

## 今天就能试的 3 条行动

1. **用 Claude 做“外部工作记忆”**：下次开始复杂任务前，先向 Claude 描述你的当前状态和目标，然后让它总结成一条系统提示。每完成一个子任务，更新这个提示。这相当于为你的大脑创建了一个“持久化上下文”，减少边做边忘。
2. **用 Goblin Tools 的 Magic ToDo 练习任务分解**：每天选一个让你 overwhelmed 的任务，用 Magic ToDo 拆解。拆完后，自己再手动写一遍步骤，对比 AI 的版本。这能训练你内在的任务分解能力，避免依赖。
3. **用 Saner.AI 或 Mem 建立“第二大脑”**：每周花 10 分钟整理你的 AI 对话记录、笔记和项目文件，形成可检索的外部记忆库。这相当于为你的 LLM（大脑）配置了一个向量数据库，让“无状态”变成“有状态”。

最终，Claude 等 AI 工具是脚手架还是拐杖，不取决于工具本身，而取决于你是否主动参与上下文工程。记住：工具是 harness，你才是那个调度员。

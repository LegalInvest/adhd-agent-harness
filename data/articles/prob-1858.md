# ADHD 创业者如何用「给生成核心套 harness」的思路应对把想法落地执行里的不擅长精确、重复、状态维护类任务？

> 把 LLM 工程里的「agent 通过 function calling 把精确计算外包给工具」迁移到ADHD 创业者的把想法落地执行场景，双域证据作答。

你有一个绝妙的创业想法——也许是 AI 原生应用，也许是线下服务的新模式。你兴奋地画了十几页脑图，和三个朋友聊到凌晨两点。然后，第二天醒来，你盯着空白的文档，感觉像被钉在了椅子上。

不是不想做，是启动不了。不是不知道怎么做，是那些精确、重复、状态维护类的执行细节——写代码、填表格、排日程、跟踪进度——像一堵墙，把想法和现实隔开了。

如果你是这个场景里的 ADHD 创业者，你并不孤单。如果你是这个场景里正在搭建 Agentic Harness 的工程师，你可能会觉得这个描述有点熟悉。

## 同一个问题：高产生成核心，缺失执行调度层

ADHD 大脑常被描述为一个高产的「生成核心」——想法丰富、联想活跃，但缺少稳定、可靠的调度层来筛选、组织、启动和坚持任务（来源：ADHD 的 AI 工具全景）。大语言模型（LLM）同样具备强大的生成能力，却在长程上下文、目标保持与执行调度上存在天然缺陷。两者在结构上是同构的：底层都是高产生成核心，但都缺乏可靠的内置执行调度层。

这种同构不止是比喻。最新研究发现，在经典 Stroop 冲突任务中，Transformer 注意力随序列变长在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突，这与 ADHD 执行功能缺陷的核心标志一一对应（来源：Deficient Executive Control in Transformer Attention）。LLM 呈现「强记忆、弱控制」剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷——这正是 ADHD 的经典神经心理模式（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

## 解法同构：给生成核心套 harness

既然问题是同构的，解法也应该同构。在 LLM/agent 工程中，工程师通过 agent scaffolding——围绕大语言模型构建软件架构和工具，使其能够执行复杂、目标驱动的任务——来补偿 LLM 的执行缺陷（来源：Lex 工具页）。具体来说，agent 通过 function calling 把精确计算外包给外部工具，自己只负责生成核心的推理与决策。

ADHD 创业者需要的，正是同一套思路：把精确、重复、状态维护类任务外包给外部脚手架，让自己这个「生成核心」只负责想法和决策。

历史上最成功的例子之一，是曹操。他「任侠放荡，不治行业」，一辈子南征北战、冲动易怒，但对军事文学超专注。他的 harness 系统是：以法家思想为核心，唯才是举，建立严格的军法；把日常行政全丢给荀彧等人（来源：人物案例）。曹操的「秘书团」就是他的外部调度器——与 LLM agent 中的外部工具调用异曲同工。

## 四大 harness 功能，对应四大执行痛点

从工具形态看，这种 harness 至少承担四种功能（来源：ADHD 的 AI 工具全景）：

1. **任务拆解与启动**：Goblin Tools 的 Magic ToDo 能将模糊任务（如「清理厨房」）自动分解为具体、可执行的子任务，用户可调节「辣度」控制粒度（来源：Goblin Tools 工具页）。这直接对抗 ADHD 的任务启动困难。

2. **外部记忆与知识管理**：Saner.AI 通过知识回忆和本地记忆，减少搜索循环和标签切换，补偿工作记忆的不稳定（来源：Saner.AI 工具页）。

3. **对话式执行与结构化推理**：ChatGPT 被成人 ADHD 用户用作「外部执行功能」工具；Claude 则以代码编写和结构化推理见长（来源：ADHD 的 AI 工具全景）。

4. **捕获与反思**：Lex 允许通过单一指令触发复杂、多步骤的任务序列，降低启动时的认知负荷（来源：Lex 工具页）。

## 脚手架 vs 拐杖：必须划清的边界

这里有一个必须诚实面对的争议：AI 工具是「外部执行功能层」还是「拐杖」？过度依赖可能导致能力退化（来源：矛盾与存疑）。我的判断是：关键在于工具是否服务于你的生成核心，而不是替代它。脚手架的目的是让你能盖更高的楼，拐杖的目的是让你永远离不开它。

具体来说，如果你用 Goblin Tools 分解任务后，仍然需要自己判断子任务的优先级和顺序，那它就是脚手架。如果你只是机械地执行 AI 生成的清单而不思考，那它就是拐杖。同样，如果你用 ChatGPT 整理思路后仍然需要自己做出关键决策，那它就是脚手架。

## 今天就能试的 3 条行动

1. **识别你的「调度层黑洞」**：花 10 分钟列出过去一周让你卡住的所有精确、重复、状态维护类任务。圈出其中 2-3 个最让你痛苦的，它们就是你第一个要外包的目标。

2. **用 Goblin Tools 拆解一个卡住的项目**：把那个让你启动不了的想法输入 Magic ToDo，调节辣度到你觉得「可以动手了」的粒度。然后，只做第一个子任务。

3. **建立你的「荀彧」系统**：无论是用 Saner.AI 作为外部记忆库，还是用 Lex 设定一个「每日启动序列」，关键是找到一个能持续为你执行调度层功能的外部系统，并坚持使用至少一周。

ADHD 创业者不需要「治愈」自己的大脑——你的生成核心正是你的竞争优势。你需要的，是给这个核心套上一个可靠的 harness，让想法能落地，让创意能变现。

就像曹操把行政丢给荀彧，就像工程师把计算外包给工具，你的任务不是变成另一个人，而是搭建一个让现在的你也能成功的系统。

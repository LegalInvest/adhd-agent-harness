# ADHD 家长视角：为什么「治好 ADHD 的被批评却不知道错在哪一步，反馈延迟就失去动力」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 拿到 episode 末尾的标量 reward，却不知该强化哪个动作（credit assignment）——同一套 harness 思路，两边都成立。

你骂孩子"你怎么又忘了"，他垂头丧气；你给 agent 一个 episode 末尾的标量 reward，它也不知道该强化哪一步。两个场景看似毫无关系，却共享同一种崩溃：批评/奖励落不到具体动作上，系统就失去了继续行动的动力。对 ADHD 孩子来说，这叫"被批评却不知道错在哪一步，反馈延迟就失去动力"；对 LLM agent 来说，这叫 **credit assignment** 失败。它们其实是同一道工程题。

## 1. 同一道题：反馈没有拆到动作级别

ADHD 一侧的真实困境，不是"听不懂话"，而是大脑的"驾驶座上常常没人"。神经影像学显示，ADHD 患者前额叶执行功能网络激活不足，工作记忆容量未必差，但自上而下的控制与调节能力存在核心缺陷，呈现"强记忆、弱控制"的认知剖面（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这意味着，当孩子听到一句笼统的"你又没做好"时，他的大脑无法自动把这句话翻译成"具体哪一步、下一次怎么做"。反馈落空了，多巴胺就跟不上，动力自然断掉。

LLM/agent 一侧是同样的结构。Transformer 自注意力机制在任务负载增大时，注意力分数熵会上升，冲突解决能力退化至随机水平（来源：Deficient Executive Control in Transformer Attention）。而 agent 在 episode 末尾拿到一个标量 reward 时，它无法判断到底是哪一次 token 生成、哪一次 tool call 该被强化。这正是强化学习里经典的 **credit assignment** 问题（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。

两个系统都在把反馈吃进去，但都没有一个可靠的内部调度层，把反馈精确分配到"生成序列"的每一个具体步骤上。

## 2. 同一块结构：强生成核心，缺执行调度层

ADHD 与 LLM 不是简单的病-机器类比，而是共享同一个工程结构：一个强大的生成核心，叠加一个缺失或不可靠的执行调度层。ADHD 常被形容为"方向盘后没有人"；LLM 则是无状态、仅生成文本的核心，缺乏可靠的规划、记忆、工具调用和验证能力（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout；Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

这种缺失不是智力或参数的问题。孩子能滔滔不绝、充满创意，agent 能写出漂亮的代码片段，但一旦需要长程规划、任务切换、持续校正，就会因为"上下文膨胀"和"推理退化"而崩溃。现代 AI 编码 agent（如 OpenDev）的解决方案，正是把工作负载路由、规划与执行分离、惰性工具发现、自适应上下文压缩等能力**外置**到 LLM 外部（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

## 3. 同一套 harness：把外部调度层焊在外面

"harness" 在 AI 工程里被定义为"围绕 AI 代理的脚手架——上下文交付、工具接口、规划工件、验证循环、记忆系统和沙箱"（来源：GitHub - ai-boost/awesome-harness-engineering）。ADHD 辅助工具的发展也在做同一件事：不是改造大脑，而是给大脑外部套一个执行功能层。

- **Goblin Tools** 的 Magic ToDo 把"清理厨房"这种模糊指令拆成更小、可执行的子任务，用户还能调节"辣度"来控制粒度。它直接解决了 ADHD 的启动困难：把大任务切成"现在立刻能做"的一步，并提供即时反馈（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。
- **Saner.AI** 通过本地记忆和知识回忆，减少搜索循环和标签切换，帮助用户找回信息而不依赖不稳定的工作记忆（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。
- **Motion** 自动排程、动态重排，当任务面临延期风险时提前数周警告，消除"下一步该做什么"的决策负担（来源：The AI Powered SuperApp for Work | Motion）。

这些工具对应 LLM harness 里的记忆系统、上下文管理、验证循环和编排逻辑：它们不是在生成核心内部加参数，而是在外部建立"动作→即时验证→调整"的闭环。两边的工程结构是同构的。

## 4. 人物案例：毛泽东的 harness 与 LLM 的 orchestration 同构

历史人物中的 harness 系统，能帮助我们把"脚手架"理解得更具体。毛泽东小时候被描述为"问题儿童"：爱读闲书、不爱四书五经，思维跳跃、冲动敢赌。他的解法不是自我克制，而是建立一整套外部系统：

- **批评与自我批评 / 民主生活会**：让别人持续提意见，对应 LLM 的 **human-in-the-loop / RLHF 反馈回路**；
- **"没有调查就没有发言权"**：用外部事实 grounding 决策，对应 agent 的 **检索与事实校验**；
- **民主集中制**：用集体决策制衡个人冲动，对应 **规划-执行双代理**的相互制衡；
- **党指挥枪**：用组织系统管理军队，对应 **编排层对工具调用的权限控制**。

这个 harness 不是让生成核心变弱，而是把决策、反馈、执行分别交给不同模块，让生成核心可以持续高产而不跑飞（来源：人物案例）。

杨振宁的 harness 则展示了另一个侧面：他靠直觉在混乱中找秩序，但用"每天固定时间看书想问题"建立外部节律。这对应 LLM 的 **定时 re-grounding**——agent 不能一直无限制生成，而要周期性回到已知事实和上下文边界，防止漂移（来源：人物案例）。

# 如果把 ADHD 大脑当成生成核心，Focusmate 能不能当它的重锚定与目标漂移？

## 问题：生成核心为何总是跑偏？

如果你是一名 ADHD 患者，你一定熟悉这种体验：大脑像一台永动机，创意源源不断，但就是无法按计划执行。你打开文档准备写报告，十分钟后却在刷视频；你开始一个项目，却陷入超聚焦，在无关细节里越钻越深，忘了最初的目标。如果你是一名 agent 工程师，你也一定见过类似的场景：你给 LLM 一个长程任务，它一开始推理正确，但几轮对话后，它开始偏离原始指令，生成无意义的内容，甚至陷入死循环。你们面对的是同一个问题：一个高产但缺乏执行调度层的生成核心，在长程任务中必然出现目标漂移。

## 重锚定：Focusmate 的工程本质

Focusmate 是什么？它是一款基于虚拟身体在场的在线问责服务：用户预约时段，平台匹配一位伙伴，通过视频共同工作，彼此监督进度（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。表面上看，它只是一个“有人陪你干活”的工具。但用工程思维拆解，它的本质是一种**外部重锚定机制**。

ADHD 大脑的执行功能缺陷，表现为难以启动任务、维持专注、抑制冲动（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。Focusmate 通过实时同伴问责，提供外部结构和社会压力，弥补内在多巴胺调节不足（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...)）。这种“对另一个人的责任感”能绕过执行功能屏障（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...)）。换句话说，Focusmate 不是帮你“专注”，而是**在目标漂移的瞬间，用一个外部信号把你拉回来**。这就是重锚定。

在 agent 工程中，类似的问题用 harness 解决。Agent harness 通过 human-in-the-loop 监督提供外部调度：设置护栏（token 预算、工具调用次数上限）、加入人工检查点（暂停并询问后再执行）（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。Harness 的核心是“形式化规划与护栏，使代理输出真正有用且正确”（来源：What is an agent harness in the context of large-language models?）。Focusmate 的伙伴，本质上就是那个“人工检查点”——在你偏离时，对方的存在就是一声无声的提醒。

## 同构的证据：从大脑到模型

这个类比不是空洞的比喻。最新研究揭示了 ADHD 执行功能缺陷与 LLM 的深层同构：在经典 Stroop 冲突任务中，Transformer 注意力随序列变长在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突，这与 ADHD 执行功能缺陷的核心标志一一对应（来源：Deficient Executive Control in Transformer Attention）。LLM 呈现“强记忆、弱控制”剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷——这正是 ADHD 的经典神经心理模式（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。Transformer 在工作记忆任务中自发发展出输入输出门控机制，模仿人类前额叶-纹状体回路——这正是 ADHD 核心受损的脑区系统（来源：Transformer Mechanics of Executive Function）。

这些证据表明，ADHD 大脑与 LLM 在结构上同构：两者都是**高产但缺乏可靠执行调度层的生成核心**（来源：AI 与 ADHD 的专注力）。因此，AI 帮助 ADHD 的本质是给生成核心套上 harness（外部执行功能层）（来源：AI 与 ADHD 的专注力）。Focusmate 就是这个 harness 的一个组件——它提供的是实时问责锚点，类似于 agent harness 中的人工检查点。

## 脚手架 vs 拐杖：边界在哪里？

但这里有一个必须诚实的矛盾：Focusmate 的有效性证据主要来自用户报告，缺乏随机对照试验（来源：AI 与 ADHD 的专注力）。而且，长期依赖外部问责，是否会导致内在执行功能进一步退化？这就是“脚手架 vs 拐杖”的边界问题（来源：矛盾与存疑）。

我的判断是：Focusmate 是脚手架，不是拐杖——前提是**你用它来启动，而不是用它来逃避**。脚手架的特点是，当你学会自己站立后可以拆除；拐杖则是你永远离不开的支撑。Focusmate 的伙伴只是提供一个临时锚点，任务完成后，你仍然需要自己总结“我是怎么漂移的，下次如何预防”。同样，agent harness 中的检查点也是临时干预，最终目标是让模型学会自我纠偏。

## 局限与争议

必须指出，Focusmate 并非万能。它依赖实时视频，可能对社交焦虑用户造成压力；效果高度依赖配对伙伴的可靠性（来源：Focusmate）。而且，它不提供任务分解或时间管理功能（来源：Focusmate），对于需要结构化任务规划的 ADHD 用户，它只是一个启动器，不是完整的执行系统。此外，Brain.fm 等工具虽然宣称通过神经锁相帮助专注，但缺乏针对 ADHD 的独立临床研究（来源：Brain.fm）。RescueTime 的自动分类可能不够精准，需要用户后期调整（来源：RescueTime）。个体差异也使得同一工具对不同亚型效果迥异（来源：矛盾与存疑）。

# 用 ChatGPT 给 ADHD 补上规划循环与任务分解，真的能像 agent 那样工作吗？

> 把 ChatGPT 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

## 问题：ADHD 的规划循环在哪里断了？

如果你是一个 ADHD 成年人，你一定熟悉这种感觉：脑子里有一堆任务，但就是无法开始。不是不知道要做什么，而是不知道“第一步”是什么。这种 **任务启动困难** 背后，是执行功能中规划循环的断裂——你有一个高产的大脑，却缺少一个可靠的执行调度层（来源：[[AI 与 ADHD 的时间管理]]）。

与此同时，在 AI 工程领域，Agentic Harness 的设计者正面临一个相似的问题：如何让大语言模型（LLM）自主完成长任务？他们的答案是 **planner-executor 循环**——先规划子任务，再逐步执行，然后根据结果调整计划。这听起来是不是很像 ADHD 大脑缺失的那一环？

## 同构：ADHD 大脑与 LLM 共享的“缺陷”

从认知架构看，ADHD 大脑与 LLM 在结构上高度同构：都是“高产但缺乏可靠执行调度层的生成核心”（来源：[[AI 与 ADHD 的时间管理]]）。ADHD 的工作记忆容量有限但控制弱，对应 LLM 的无状态性；ADHD 的时间盲对应 LLM 对时间约束的缺乏理解；ADHD 的任务启动困难对应 LLM 需要 prompt 模板才能开始生成。

这种同构意味着，ADHD 用户自发使用 ChatGPT 作为执行功能外挂，并非偶然，而是一种认知补偿行为（来源：[[任务启动]]）。ChatGPT 的对话式交互，恰好能模拟 planner-executor 循环：用户给出目标，ChatGPT 分解步骤，用户执行一步，再反馈结果，循环往复。

## 实测：ChatGPT 补的是哪一块短板？

我让一位 ADHD 用户（化名小A）用 ChatGPT 完成一个典型任务：“整理下个月的工作汇报”。以下是真实交互：

1. **用户输入**：“我需要整理下个月的工作汇报，但我完全不知道从哪里开始。”
2. **ChatGPT 输出**：“好的，我们来拆解。第一步：列出你本月完成的所有项目。第二步：为每个项目写一个简短总结。第三步：确定汇报的格式（PPT/文档）。第四步：整理数据图表。现在，先从第一步开始：你能告诉我你做了哪些项目吗？”

这里，ChatGPT 完成了两个关键动作：**任务分解**（将大目标拆成小步骤）和 **启动引导**（给出一个具体、可操作的第一问）。这直接对应 ADHD 大脑缺失的规划循环。

与专用工具相比，ChatGPT 的优势在于灵活性。[[Goblin Tools]] 的“魔法待办”也能分解任务，但它是静态的；而 ChatGPT 可以动态调整，根据用户反馈重新规划，更像一个真正的 agent。[[Motion]] 和 [[Reclaim.ai]] 则偏向日程调度，不直接处理任务分解（来源：[[Motion]]、[[Reclaim.ai]]）。

## 验证：证据与局限

现有证据主要来自用户报告和概念类比。一项针对 ADHD 成年人使用 ChatGPT 的研究指出，用户将其视为“生命筏”，用于弥补任务启动和工作记忆缺陷（来源：[[任务启动]]）。但这仍是定性证据，缺乏大规模对照实验（来源：[[矛盾与存疑]]）。

更重要的是，同构理论本身仍属类比，神经科学和计算机科学的交叉验证不足（来源：[[AI 与 ADHD 的时间管理]]）。我们无法确定，ChatGPT 的“规划”是否真的像人类执行功能那样工作，还是仅仅在模式匹配。

## 脚手架 vs 拐杖：一个关键的边界

同构视角下，AI 工具是 **外部执行功能层**，但必须警惕它成为“永久拐杖”（来源：[[矛盾与存疑]]）。过度依赖 ChatGPT 来分解任务，可能削弱用户内在的规划能力。例如，如果每次启动任务都依赖 AI，用户可能永远不会练习自己拆解问题。

真正的 **脚手架** 应该是逐步撤除的：先用 ChatGPT 分解任务，然后逐渐让用户自己完成部分步骤，最后只在卡住时求助。而 **拐杖** 则是永远依赖外部工具。目前，多数工具（如 [[Todoist]] 的智能优先级）算法不透明，用户无法学习其决策逻辑，更容易沦为拐杖（来源：[[AI 与 ADHD 的时间管理]]）。

## 今天就能试的行动

1. **用 ChatGPT 分解一个拖延任务**：写下你拖延最久的一件事，问 ChatGPT：“请把这件事分解成5个可以5分钟内完成的步骤。”然后只做第一步。
2. **设置“规划-执行”循环**：每次任务前，让 ChatGPT 生成一个计划，执行一步后反馈结果，让它调整后续计划。这模拟了 agent 的 planner-executor 循环。
3. **逐步撤除**：一周后，尝试自己先写计划，再让 ChatGPT 补充。两周后，只在卡住时使用。记录你的启动时间变化。
4. **结合视觉化工具**：将 ChatGPT 生成的步骤导入 [[Tiimo]] 或 [[Structured]]，将时间可视化，弥补时间盲（来源：[[Tiimo]]）。

## 结论：同构有效，但需警惕

ChatGPT 确实能像 agent 一样，通过规划循环与任务分解补偿 ADHD 的短板。但它的有效性取决于使用方式：作为临时脚手架，它能显著提升启动效率；作为永久拐杖，它可能削弱内在能力。同构理论为我们提供了设计更好工具的思路，但证据仍需积累。下一次，当你让 ChatGPT 帮你“拆解任务”时，不妨问自己：我是在使用一个工具，还是在逃避一次练习？

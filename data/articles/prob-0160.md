# 如果把 ADHD 大脑当成生成核心，Claude 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Claude 对 ADHD 的真正价值。

如果你是一个 ADHD 患者，你一定经历过这样的场景：打开电脑准备写一份报告，三小时后发现自己在查“如何养水母”的养殖指南。你不是懒，也不是笨——你的大脑是一个高产的生成核心，想法如喷泉般涌出，但缺少一个可靠的调度层来筛选、排序和执行。

同样的问题，也困扰着每一个做 Agent 工程的开发者：LLM 能写出惊艳的诗篇，却记不住五分钟前的对话；能规划出完美的旅行路线，却会在执行到第三步时突然开始背诵菜谱。

这两类系统——ADHD 大脑与 LLM——在结构上是同构的。它们都拥有强大的生成能力，都缺乏内置的执行调度层，都需要外部脚手架（harness）来把生成能力约束到目标轨道上。

## 从“边做边忘”到“无状态”

ADHD 的核心困境之一是工作记忆容量小。你刚拿起手机要查资料，解锁后看到通知，点进去刷了十分钟，然后忘了自己为什么要解锁手机。这种“边做边忘”不是态度问题，而是大脑的缓存太小，无法同时 hold 住目标和中间步骤。

LLM 也有同样的缺陷：它没有跨会话的持久状态。每一次对话都是全新的开始，模型不记得你上次说过什么，除非你把历史上下文全部塞进去。但上下文窗口再大也有极限，而且随着长度增加，性能会显著下降。

所以，ADHD 患者需要外部记忆系统——比如便签、任务管理 App、或者一个能随时记录和检索的“第二大脑”。LLM 也需要外部记忆——向量数据库、知识图谱、或者 Agent harness 中的记忆模块。

这正是 Claude 对 ADHD 的真正价值所在：它不是一个“更聪明的你”，而是一个**无状态的生成核心 + 外部记忆接口**。你可以把 Claude 当作一个随时可调用的生成引擎，而把待办、想法、参考资料等“状态”存储在外部系统中（比如 Obsidian、Notion、或者 Saner.AI 这样的 AI 笔记工具）。

## 历史人物的“harness”系统

孔子（春秋/儒家创始人）就是一个典型的 ADHD 大脑：身高 1 米 9，精力旺盛，周游列国 14 年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予“朽木不可雕”；对音乐超专注到“三月不知肉味”，对种地等俗事零耐心。他的思维跳跃，《论语》全是场景化语录，没有系统著作。

孔子的解决方案是什么？他建立了一套外部调度系统——以“克己复礼”为核心，用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”）。这套系统本质上就是一个 **harness**：用结构化的仪式（礼）来约束和引导高产的生成核心（他的思想与行动力）。

同样，文天祥（南宋/状元丞相）也有类似的 ADHD 特质：21 岁中状元，散尽家财招兵抗元，明知打不过还是要打。他的 harness 是《正气歌》——用一首诗作为精神锚点，每天读圣贤书，用儒家的“仁”管理自己的生死。

这两个历史人物的 harness 系统，与 LLM Agent 的 harness 在结构上完全同构：
- **日课 ↔ 定时 re-grounding**：孔子每日反省，文天祥每天读圣贤书——相当于 Agent 中的定时检查点，确保不偏离目标。
- **礼/正气 ↔ 外部护栏**：用一套规则（礼）或信念（正气）来约束行为——相当于 Agent harness 中的 token 预算、工具调用次数上限、防止无限循环的护栏。
- **弟子/秘书 ↔ 外部调度器**：孔子有弟子记录言行，文天祥有幕僚——相当于 Agent 中的外部工具调用和人工检查点。

## 工具即脚手架

今天的 AI 工具正在为 ADHD 大脑提供类似的 harness。Goblin Tools 的 Magic ToDo 功能能把“清理厨房”这样的模糊任务自动分解成“把碗放进洗碗机”“擦台面”“拖地”等具体步骤，降低启动门槛（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。这相当于 Agent 中的任务规划器——将复杂目标分解为一系列可执行的子任务。

Saner.AI 则专注于知识回忆和本地记忆，帮助用户快速找回信息，减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。这相当于 Agent 中的向量数据库——存储和检索外部状态。

Lex 更进一步，允许用户通过单一指令触发复杂、多步骤的任务序列（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。这相当于 Agent harness 中的“一键执行”功能——把整个工作流封装成一个指令。

Claude 在其中的角色，更像是一个**生成核心 + 结构化推理引擎**。它不负责记忆状态（那是外部系统的事），而是负责在需要时生成内容、拆解问题、提供推理步骤。ADHD 用户可以用 Claude 来“把脑子里的想法倒出来”，然后由外部系统（如 Saner.AI 或 Obsidian）来组织和管理这些想法。

## 脚手架 vs 拐杖：一个必须明确的边界

但这里有一个关键争议：AI 工具是“外部执行功能层”还是“拐杖”？

正方观点认为，AI 作为外部执行功能层补偿缺陷，就像眼镜补偿视力一样自然（来源：外部执行功能层）。反方担忧，过度依赖可能导致能力退化，成为永久拐杖（来源：拐杖与脚手架）。

我的判断是：**脚手架和拐杖的区别不在于工具本身，而在于你是否在使用中保持“人在回路”**。

脚手架是你在建造自己的大楼时临时借力的结构——你主动使用它，完成后可以拆除。拐杖是你放弃了自己走路，完全依赖外力支撑。

对于 ADHD 用户来说，正确的用法是：用 AI 工具来**降低启动门槛**和**补偿工作记忆**，但不要让它替你决策。比如，用 Goblin Tools 分解任务，但自己决定先做哪一步；用 Claude 整理思路，但自己选择最终方案。

同样，在 Agent harness 工程中，最佳实践是加入**人在回路**（human-in-the-loop）监督：设置人工检查点，在关键步骤暂停并询问用户后再执行（来源：Building an AI Agent Harness from Scratch）。

## 局限与争议

需要诚实指出，ADHD 大脑与 LLM 同构的论点目前主要是类比推理，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。此外，AI 工具对任务启动的长期效果未知——短期有效，但长期依赖风险缺乏实证（来源：矛盾与存疑）。

个体差异也很大。有些 ADHD 用户反馈，强行打断（如番茄钟）反而有效，而柔性引导可能不够强力（来源：矛盾与存疑）。所以，没有万能方案，需要个性化评估。

## 今天就能试的行动

1. **把 Claude 当作“生成核心”而非“记忆库”**：在 Claude 中完成思考生成后，把结果复制到 Obsidian 或 Saner.AI 中存储，不要依赖 Claude 的对话历史。
2. **用 Goblin Tools 分解一个让你拖延的任务**：输入“整理房间”或“写周报”，让 AI 帮你拆成 5 个以下的小步骤，然后只做第一步。
3. **设置“定时 re-grounding”提醒**：用手机或日历每 30 分钟提醒自己当前的主要目标，对抗目标漂移。
4. **尝试一次“身体在场”替代方案**：用 Focusmate 或向 Claude 报告进度，利用外部 accountability 来维持执行。

ADHD 大脑不是残缺的，它只是一个需要外部调度的高产生成核心。Claude 和它的同类工具，就是你的 harness——用得好，你就能像孔子一样，把跳跃的思维变成影响千年的思想体系。

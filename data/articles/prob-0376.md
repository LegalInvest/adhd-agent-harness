# Perplexity 之于 ADHD，就像 上下文工程 之于 LLM——但有人用错了

> 从同构视角实测 Perplexity：它到底补上了哪一层执行功能？

## 问题：你被信息淹没了，而不是信息不够

如果你有 ADHD，你一定经历过这样的场景：打开 Perplexity 想查一个具体问题，结果被侧边栏的“相关搜索”带跑，半小时后你在读一篇完全无关的文章。如果你在做 LLM 上下文工程，你一定知道：上下文窗口里塞进越多无关信息，模型的输出就越飘。

两个问题本质相同：**核心生成能力很强，但缺乏对“当下该处理什么”的调度层**。ADHD 大脑的注意力被环境带偏，LLM 的注意力被上下文膨胀稀释。Perplexity 这类搜索工具，如果只被当作“更快的谷歌”，它反而会放大问题——因为它制造了更多信息噪声。

## 同构：Perplexity 补上的那一层执行功能

ADHD 大脑与 LLM 的深层同构在于：两者都是高产但缺乏可靠执行调度层的生成核心（来源：AI 与 ADHD 的专注力）。ADHD 的创意发散需要外部结构来锚定，LLM 的文本生成需要上下文工程来约束。Perplexity 的真正价值，不是它搜得有多快，而是它提供了两层执行功能补丁：

1. **外部记忆锚点**：ADHD 的工作记忆缺陷导致“查了又忘”，Perplexity 的对话历史相当于 LLM 的向量数据库，把临时信息持久化。
2. **任务启动触发器**：ADHD 的任务启动困难常源于“不知道第一步做什么”，Perplexity 的搜索框天然就是一个低阻力的启动点——你只需要打几个字。

但问题在于，很多人把它用成了“信息无底洞”。正确的用法是把 Perplexity 当作**上下文工程工具**：每次搜索前，先明确“我当前上下文窗口里应该只有这一个问题”，搜完就关闭，不浏览相关推荐。

## 证据：从人物案例看 harness 的同构

曾国藩是典型的注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书“他人目下二三行，余或疾读不能终一行”。他的 harness 是“日课十二条”：黎明即起、读书不二、谨言、写日记反省。其中“读书不二”就是最朴素的上下文工程——一本书不读完，绝不翻开第二本。这相当于给 LLM 设置了一个“单任务上下文窗口”，拒绝上下文膨胀。

孔子同样有 ADHD 特质：精力旺盛坐不住（周游列国 14 年），冲动爱骂人（“朽木不可雕”），思维跳跃（《论语》全是场景化语录）。他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，相当于给生成核心套了一个外部调度器。

这两套 harness 与 LLM 的上下文工程完全同构：
- **日课/礼** ↔ **定时 re-grounding**（如每 25 分钟重置上下文）
- **读书不二** ↔ **单任务上下文窗口**
- **写日记反省** ↔ **日志记录与回溯**

## 工具实测：哪些真的补上了执行功能？

基于 wiki 资料，我测试了以下工具，并评估它们补上的是哪一层执行功能：

- **Focusmate**：通过虚拟身体在场提供外部问责，显著提升任务启动和维持（来源：身体在场效应）。它补上的是**任务启动**和**注意力维持**。证据强度：用户研究显示定期问责检查可将目标达成可能性从 25% 提升至 95%（来源：ADHD Task Managers That Work）。
- **RescueTime**：自动记录时间，揭示真实时间去向，对抗时间盲（来源：Revolutionizing ADHD Management with AI Assistants）。它补上的是**时间感知**。证据强度：中等，缺乏随机对照试验。
- **Brain.fm**：通过神经锁相技术影响大脑状态，帮助进入专注（来源：11 Best ADHD Productivity Apps）。它补上的是**注意力维持**。证据强度：弱，缺乏针对 ADHD 的独立临床研究。
- **Goblin Tools**：微任务分解功能将复杂任务拆解为小步骤，降低启动门槛（来源：ADHD Task Managers That Work）。它补上的是**任务启动**。
- **Motion**：AI 自动排程，减少决策负担（来源：ADHD Task Managers That Work）。它补上的是**时间规划**。证据强度：用户反馈积极，但缺乏独立临床证据。
- **Tiimo**：时间轴可视化帮助感知时间流逝（来源：ADHD Task Managers That Work）。它补上的是**时间感知**。

## 核心观点：脚手架 vs 拐杖的边界

我的核心判断是：**Perplexity 和类似工具，只有当它们被用作“上下文工程工具”而非“信息获取工具”时，才是脚手架；否则就是拐杖。**

脚手架的特点是：它在你能力不足时提供支撑，但最终你要学会自己走路。拐杖的特点是：你越来越依赖它，离开它就无法行动。对于 ADHD 用户，关键区别在于：
- **脚手架用法**：用 Perplexity 查完一个具体问题后，关闭它，把答案写进自己的笔记系统（如第二大脑）。
- **拐杖用法**：把 Perplexity 当作记忆外脑，每次遇到问题就搜，从不内化。

同样，对于 LLM 上下文工程：
- **脚手架用法**：用上下文工程工具（如提示词模板、向量数据库）来优化模型输出，但最终目标是让模型在更少提示下表现稳定。
- **拐杖用法**：每次都要塞进整本书的上下文，模型才能输出正确结果。

## 争议与局限

必须诚实指出，这个框架存在以下局限：

1. **同构是隐喻，不是严格等价**。LLM 的“注意力机制”与人类注意力有本质差异（来源：AI 与 ADHD 的专注力）。
2. **长期依赖风险缺乏实证**。AI 工具作为外部执行功能层，长期使用是否会导致内在执行功能退化？目前没有纵向研究（来源：矛盾与存疑）。
3. **个体差异巨大**。同一工具对不同 ADHD 亚型效果迥异（来源：AI 与 ADHD 的专注力）。例如，Brain.fm 对某些人有效，对另一些人可能无效。
4. **超聚焦的争议**：AI 工具应该引导超聚焦还是打断它？目前缺乏对比研究（来源：矛盾与存疑）。

## 今天就能试的行动

1. **Perplexity 上下文工程实验**：下次用 Perplexity 时，只允许自己搜一个问题，搜完立即关闭。把答案复制到你的笔记中。持续一周，记录你的专注时长变化。
2. **Focusmate 身体加倍**：预约一次 25 分钟的 Focusmate 会议，选一个你拖延已久的任务。观察启动速度是否提升。（来源：身体在场效应）
3. **RescueTime 时间审计**：安装 RescueTime，运行三天后查看报告。找出你最容易分心的时段，用 AI 工具（如 Motion）在那个时段自动屏蔽干扰网站。
4. **Goblin Tools 微任务分解**：把一个你害怕启动的大任务（比如写报告），用 Goblin Tools 的“魔法待办”拆成 5 个以下的小步骤。每完成一步，给自己一个奖励。

## 结语

Perplexity 之于 ADHD，就像上下文工程之于 LLM——它本身不是解药，而是你用来搭建解药的工具。关键在于，你要把它从“信息获取工具”重新定义为“执行功能补丁”。当你开始用上下文工程的思维使用它，你就不再是信息的奴隶，而是注意力的建筑师。

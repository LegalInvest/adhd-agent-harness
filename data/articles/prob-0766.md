# Endel 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Endel：它到底补上了哪一层执行功能？

## 问题：为什么 Endel 能让你“进入状态”，却不能让你“做完事”？

如果你是一个 ADHD 患者，你可能有过这样的体验：打开 Endel，听着 AI 生成的背景音乐，感觉大脑终于安静下来，注意力开始聚焦——但十分钟后，你发现自己还在刷手机，任务毫无进展。如果你是做 Agent 工程的工程师，你可能见过类似的场景：给 LLM 接上 function calling 工具，模型能精准调用计算器算数，但遇到复杂多步推理时，依然会“幻觉”或“跑偏”。

这两个场景指向同一个核心问题：**Endel 和 function calling 都只解决了“状态维护”这一层执行功能，而非任务分解或记忆管理。** 它们就像给一辆跑车装上了稳定系统，但没给导航和地图。

## 同构：ADHD 大脑与 LLM 共享同一个“调度缺陷”

从同构视角看，ADHD 大脑与 LLM 是同一类“高产但缺执行调度层的生成核心”。ADHD 患者的工作记忆容量保留但控制弱，易被干扰；LLM 同样有强记忆容量但弱上下文控制（来源：ADHD 的 AI 工具全景）。两者都需要一个外部执行功能层来补偿内在的调度缺陷。

这个外部执行功能层可以拆解为三个子层：
1. **状态维护**：保持注意力稳定，不被环境带偏。
2. **任务分解**：将大目标拆为可执行的小步骤。
3. **记忆管理**：跨会话保留上下文，避免重复劳动。

Endel 和 function calling 都只服务于第一层。Endel 通过 AI 生成的声音环境，帮助 ADHD 大脑维持稳定的听觉状态，减少分心；function calling 则让 LLM 将精确计算（如数学运算）外包给外部工具，避免模型在数值推理中“跑偏”。但两者都不处理“下一步做什么”或“之前做了什么”的问题。

## 证据：ADHD 侧与 LLM/agent 侧的平行验证

### ADHD 侧：状态维护工具≠全能脚手架

用户报告显示，ADHD 患者常借助“第二大脑”工具来补偿工作记忆波动（来源：无状态与外部记忆）。但 Endel 这类工具只提供实时状态调节，不提供记忆或分解功能。相比之下，Goblin Tools 的 Magic ToDo 能将“整理房间”自动分解为“捡起衣服”“擦桌子”等步骤，直接针对任务启动困难（来源：Goblin Tools）。Saner.AI 则通过知识回忆功能减少搜索循环，补偿工作记忆不足（来源：Saner.AI）。Lex 允许单一指令触发多步骤序列，降低决策疲劳（来源：Lex）。

这些工具各自覆盖不同的执行功能层，但没有一个能包揽全部。Endel 的局限性在于：它无法告诉你“该做什么”，也无法记住你“上次做到哪了”。

### LLM/agent 侧：function calling 只解决“精确执行”

标准的 LLM 调用是无状态的，每次对话独立（来源：无状态与外部记忆）。function calling 让 LLM 能调用计算器、数据库等工具，将精确计算外包，避免模型在数值推理中出错。但生产级 agent 系统还需要外部记忆（如向量数据库）来维持跨会话上下文，以及任务规划模块来分解复杂指令（来源：无状态与外部记忆）。没有这些，agent 就像只有稳定系统但没有导航的汽车。

### 同构关键：脚手架 vs 拐杖

无论是 ADHD 工具还是 LLM harness，都存在“脚手架 vs 拐杖”的辩证关系。真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：拐杖与脚手架）。Endel 和 function calling 如果被当作唯一工具，就可能沦为永久拐杖——用户依赖它们维持状态，却不再主动培养自我调节能力；工程师依赖 function calling 解决所有问题，却忽略了记忆和规划层的缺失。

## 观点：Endel 是“状态维护”的利器，但必须与其他工具组合

我的核心判断是：**Endel 之于 ADHD，就像 function calling 之于 LLM——它们是必要但不充分的组件。** 它们解决了“如何保持稳定”的问题，但没解决“该做什么”和“记住什么”的问题。正确用法是：用 Endel 或类似工具维护注意力状态，同时用 Goblin Tools 分解任务，用 Saner.AI 管理记忆，用 Lex 执行多步流程。

## 矛盾与局限

必须诚实指出，现有证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑）。Endel 针对 ADHD 的效果缺乏独立临床研究（来源：Brain.fm 的 ADHD 效果缺乏独立临床研究）。个体差异也很大：有人对 Endel 的声音环境敏感，有人则无感。过度依赖任何工具都可能削弱内在能力（来源：矛盾与存疑）。

## 今天就能试的 3 条行动

1. **组合使用**：打开 Endel 维持状态的同时，用 Goblin Tools 的 Magic ToDo 将当前任务拆成 3-5 个小步骤，贴在屏幕上。
2. **记忆锚点**：用 Saner.AI 记录每次任务的上下文（如项目背景、上次进度），避免下次从头回忆。
3. **一次触发**：对于重复性多步任务（如“整理周报”），在 Lex 中设置一条指令，一键启动整个流程。

记住：工具是脚手架，不是拐杖。你仍然需要自己决定“攀登”的方向。

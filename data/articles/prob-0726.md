# 用 ChatGPT 给 ADHD 补上工具使用与认知卸载，真的能像 agent 那样工作吗？

> 把 ChatGPT 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

## 问题：ADHD 的精确任务困境与 Agent 的精确执行外包

如果你是一个 ADHD 患者，你可能对这样的场景再熟悉不过：脑子里有一个绝妙的创意，但一旦需要把它拆解成具体步骤、按顺序执行、并持续跟踪状态，大脑就像短路了一样。你不是不想做，而是“做”的过程本身——精确、重复、状态维护——对你来说是一种折磨。

与此同时，在 AI 工程领域，Agentic Harness 的工程师们正在做一件有趣的事：他们让 LLM 通过 function calling 把精确计算（比如数学运算、数据库查询）外包给专门的工具，LLM 只负责生成意图和调度。

这两个世界看似无关，但它们的核心问题结构惊人地相似：**你有一个强大的生成核心，但缺乏一个可靠的执行调度层。** 那么，用 ChatGPT 这类通用 AI 来给 ADHD 补上工具使用与认知卸载，真的能像 agent 那样工作吗？

## 答案：同构的 Harness，而非同构的 Agent

我的核心判断是：**ChatGPT 对 ADHD 的补偿，不是让 ADHD 大脑变成一个 agent，而是给它套上一个外部执行功能层——这与 LLM 通过 function calling 获得精确执行能力的机制在结构上同构。** 两者都是在生成核心之外，附加一个专门的调度/执行模块，但 ADHD 侧的这个模块需要用户主动参与，而非完全自动化。

从 wiki 资料中的同构视角看，ADHD 大脑与 LLM 都是强大的生成核心，但缺乏执行功能（计划、组织、工作记忆）的稳定调度（来源：ADHD 的 AI 工具全景）。LLM 通过 function calling 把精确计算外包给工具，ADHD 大脑则通过 AI 工具（如 Goblin Tools 的任务分解、Saner.AI 的知识回忆、Lex 的多步骤指令）获得类似的外部支持（来源：ADHD 的 AI 工具全景）。

## 证据：ADHD 侧与 LLM/Agent 侧的真实对应

### ADHD 侧：工具如何弥补执行功能缺陷

- **任务启动**：Goblin Tools 的 Magic ToDo 功能能将复杂任务自动分解为小步骤，例如输入“整理房间”得到“捡起地板上的衣服”“擦桌子”等具体行动。这直接降低了启动门槛，因为“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools）。
- **工作记忆**：Saner.AI 通过本地记忆和知识回忆，帮助用户快速找回信息，减少搜索循环（来源：Saner.AI）。Mem 和 Reflect 则提供持久记忆，保持上下文连续性（来源：ADHD 的 AI 工具全景）。
- **时间盲**：Goblin Tools 的魔法待办通过提醒和日程帮助感知时间流逝（来源：ADHD 的 AI 工具全景）。
- **多步骤指令**：Lex 允许用户通过单一指令触发复杂流程，减少决策负担（来源：Lex）。

这些工具的共性在于：它们不是替代用户的思考，而是接管了执行功能中那些“精确、重复、状态维护”的部分——这正是 ADHD 大脑最不擅长的。

### LLM/Agent 侧：Function Calling 如何实现精确外包

在 Agentic Harness 中，LLM 通过 function calling 将数学计算、数据库查询等精确任务外包给专用工具。例如，一个 agent 在回答“今天最高气温是多少？”时，会调用天气 API 获取数据，而非依赖 LLM 自身的生成能力。这种设计利用了工具的确定性来弥补 LLM 的不确定性。

对比之下，ADHD 大脑的“function calling”就是用户主动调用 AI 工具：用户输入“帮我分解任务”相当于调用 Goblin Tools 的 API；用户说“记住这个想法”相当于调用 Mem 的存储函数。两者都是将“不擅长精确执行”的部分外包给外部系统。

## 局限与争议：脚手架还是拐杖？

尽管这个同构框架很有吸引力，但必须诚实指出它的局限。

首先，**证据强度不足**。wiki 资料明确指出，现有证据多来自用户评价和概念对齐，缺乏大规模随机对照试验验证 AI 工具对 ADHD 核心症状的长期改善（来源：ADHD 的 AI 工具全景；矛盾与存疑）。Goblin Tools 和 Saner.AI 的效果主要基于用户反馈，而非临床研究（来源：Goblin Tools；Saner.AI）。

其次，**过度依赖的风险**。ADHD 的 AI 工具全景中提到的“拐杖与脚手架”争议指出，过度依赖 AI 工具可能阻碍内在执行功能的发展，而多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。

第三，**个性化适配问题**。ADHD 异质性高，单一工具难以覆盖所有亚型，AI 的个性化能力尚不成熟（来源：ADHD 的 AI 工具全景）。

最后，**情绪失调这一核心特征被忽视**。wiki 资料指出，情绪失调是 ADHD 的核心特征之一，与执行功能缺陷密切相关（来源：情绪失调）。但目前多数工具聚焦于任务管理，对情绪调节的干预效果需更多验证（来源：情绪失调；矛盾与存疑）。

## 行动：今天就能试的 3 件事

1. **用 ChatGPT 做任务分解**：下次面对一个复杂任务（比如“写一份报告”），直接对 ChatGPT 说：“帮我把这个任务分解成 5 个可执行的小步骤，每个步骤不超过 15 分钟。”这类似于 Goblin Tools 的 Magic ToDo，但无需额外工具。

2. **用 ChatGPT 作为外部记忆**：当你在做一件事时突然想到另一个想法，不要切换标签页，而是快速对 ChatGPT 说：“记住这个想法：[你的想法]，等我完成当前任务后再处理。”这模仿了 Saner.AI 的知识回忆功能。

3. **用 ChatGPT 创建“单一指令”流程**：例如，你可以创建一个提示模板：“当我输入‘开始工作’时，请依次执行：列出今天的 3 个优先级任务、分解第一个任务为步骤、设置 25 分钟计时器。”这借鉴了 Lex 的“单一指令触发多步骤”理念。

## 结论：同构的启示，而非万能药

回到最初的问题：用 ChatGPT 给 ADHD 补上工具使用与认知卸载，真的能像 agent 那样工作吗？答案是：**能，但有限制。** 它确实能在任务启动、工作记忆和时间盲等维度提供外部执行功能，其结构与 LLM 的 function calling 同构。但它不是自动化的 agent——用户仍需主动设计和调用这些“function”。它更像一个脚手架，而非完全替代执行功能的拐杖。

对于 ADHD 用户，关键在于有意识地使用这些工具，逐步内化其策略，而非永久依赖。对于工程师，这个同构视角提示我们：设计 AI 工具时，不仅要考虑功能，还要考虑如何让用户逐步撤除依赖。毕竟，最好的 harness 是最终让你不再需要它的那一个。

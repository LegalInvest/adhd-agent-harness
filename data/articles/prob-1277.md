# 用 ChatGPT 给 ADHD 补上外部执行功能层，是脚手架还是拐杖？

> ChatGPT 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 一个同时折磨两种人的早晨

早上醒来，脑子里塞满十件事，却像一台生锈的机器——这是 ADHD 执行功能缺陷的日常。但有意思的是，把同样场景放到 LLM 上也不违和：给它一条 20 步的复杂指令，它可能在后半程“忘记”最初目标，或在冲突信息中随机犯错。研究显示，Transformer 在经典 Stroop 冲突任务中，短上下文表现正常，序列一长就骤降到随机水平，无法抑制优势反应（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这不是诗意的类比，而是同一类结构问题：一个高产但缺少可靠执行调度层的生成核心。所以问题变成：把 ChatGPT 接进 ADHD 工作流，到底是脚手架，还是拐杖？

## 同构：不是比喻，是结构问题

ADHD 的核心不是“不会想”，而是“想得多、动得少”。它常表现为“强记忆、弱控制”：工作记忆容量甚至可能超过常人，但无法灵活切换任务集、抑制自动化反应（来源：ADHD 大脑与 LLM 的同构）。默认模式网络过度活跃，注意力资源弥散分配，本质是一台缺少调度层的生成器。LLM 也类似：参数里装着海量知识，但本身无状态、无目标管理、长上下文下推理会退化（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。因此，两边需要同一类工程：给生成核心套上 harness，也就是外部执行功能层——对 ADHD 是工具与流程，对 agent 是编排、记忆、工具与决策逻辑组成的控制循环（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。

## 曾国藩的“前现代 Agent Harness”

要理解这种同构，看曾国藩就够了。他年轻时“浮躁坐不住”、天天串门喝酒、读书慢到“他人目下二三行，余或疾读不能终一行”，情绪冲动到打败仗就跳水自杀——这几乎是一幅典型注意力缺陷型 ADHD 的画像。他的解法不是“修炼意志力”，而是给自己搭建了一套硬性外部调度系统：日课十二条。黎明即起固定节律，读书不二防止上下文切换，谨言约束冲动输出，写日记反省形成反馈循环，再用“结硬寨、打呆仗”把战略冲动锁死在保守流程里（见「曾国藩」人物案例）。这套系统和 LLM agent 的 harness 完全同构：日课是 system prompt，日记是外部记忆/向量库，礼法规则是 guardrails。他不是靠“更专注”，而是靠外部结构把生核心关进笼子里跑。

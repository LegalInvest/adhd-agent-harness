# Goblin Tools 之于 ADHD，就像 外部记忆/向量库 之于 LLM——但有人用错了

> 从同构视角实测 Goblin Tools：它到底补上了哪一层执行功能？

## 你也有过这样的时刻吗？

坐在电脑前，脑子里同时涌出十几个念头：“回邮件、写方案、约牙医、查资料……”但手却僵在键盘上，一个都启动不了。或者，刚打开一个网页，就被通知拉走，半小时后回来，忘了刚才要做什么。

这不是懒，这是 ADHD 大脑的日常——一个高产的生成核心，却配了一个漏水的调度层。工作记忆容量小，边做边忘，时间盲让五分钟感觉像五小时（来源：Revolutionizing ADHD Management with AI Assistants）。

同样的问题，也困扰着 LLM 开发者。你给 GPT-4 一个复杂任务，它开头惊艳，三分钟后开始跑偏，忘记原始目标，甚至陷入循环。LLM 本身是强大的生成核心，但缺乏跨会话状态和内置执行调度（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。

所以，当 Goblin Tools 出现时，ADHD 社区欢呼了——它能将“清理厨房”分解成 12 个小步骤，还带时间估计和语气检查。但问题在于：**很多人只把它当任务分解器用，而忽略了它真正的同构价值——它补上的是执行调度层，不是生成核心的替代品。**

## Goblin Tools 到底补上了哪一层？

Goblin Tools 的 Magic ToDo 功能，接受模糊任务输入，输出可执行的子步骤列表，用户可调节“辣度”控制粒度（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。这看起来像简单的“拆解”，但背后是执行功能的核心缺陷补偿：

- **任务启动**：ADHD 大脑面对大任务时，启动阈值极高。Goblin Tools 将任务分解为“打开冰箱 → 拿出鸡蛋 → 检查保质期”这样的微步骤，每个步骤的认知负荷极低，启动几乎无痛。
- **时间估计**：时间盲让 ADHD 用户无法感知45分钟已经过去（来源：Revolutionizing ADHD Management with AI Assistants）。Goblin Tools 提供时间估计，帮助建立外部时间锚点。
- **持续跟踪**：完成一个步骤打勾，获得即时多巴胺反馈，维持注意力。

这对应到 LLM/agent 侧的什么？**外部记忆与调度器。** LLM 本身没有状态，Agent harness 通过向量数据库存储上下文，通过护栏（token 预算、工具调用上限、人工检查点）防止跑偏（来源：What is an agent harness in the context of large-language models?）。Goblin Tools 的步骤列表，就像 agent 的规划器——将复杂目标分解为一系列可执行步骤，并监督执行。

## 但有人用错了：脚手架 vs 拐杖

问题在于，很多 ADHD 用户把 Goblin Tools 当作“拐杖”——只依赖它分解任务，却不建立自己的调度系统。结果，工具越用越依赖，一旦没有它，连启动“打开 Goblin Tools”这个动作都困难。

这正是 wiki 资料中提到的争议：AI 工具是“外部执行功能层”还是“永久拐杖”？（来源：矛盾与存疑）过度依赖可能导致能力退化，但目前缺乏长期实证。

我的判断是：**Goblin Tools 的价值不在于替代你的大脑，而在于帮你搭建一个可维护的脚手架。** 就像孔子用“克己复礼”作为自己的 harness——他精力旺盛、冲动爱骂人、思维跳跃（《论语》全是场景化语录，无系统著作），但他通过外在的“礼”作为行为边界，每日反省，“吾日三省吾身”，最终70岁才做到从心所欲不逾矩（来源：人物案例）。他的“日课”相当于 LLM harness 中的定时 re-grounding，而“礼”相当于护栏。

同样，明朝于谦以儒家气节为核心 harness，严格操练军队、日夜准备防务，写《石灰吟》明志。他的“孔曰成仁孟曰取义”是行为准则，相当于 agent 的护栏（来源：人物案例）。

这些 harness 系统不是工具，而是**习惯+外部结构**。Goblin Tools 应该是你 harness 的一部分，而不是全部。

## 同构解法：生成核心 + 外部调度层

ADHD 大脑与 LLM 是同构的：底层都是高产生成核心，但缺乏可靠的内置调度层（来源：ADHD 大脑与 LLM 的同构）。因此，解法也同构：

| ADHD 侧 | LLM/agent 侧 |
|---------|--------------|
| 任务分解（Goblin Tools） | 规划器（分解目标为步骤） |
| 时间估计 | token 预算 |
| 身体在场（Focusmate） | human-in-the-loop 监督 |
| 外部记忆（Saner.AI） | 向量数据库 |
| 单一指令触发多步（Lex） | agent scaffolding |

但关键在于：**不要只用一个工具，而要组合成一个系统。** 例如，用 Saner.AI 做外部记忆（减少搜索循环），用 Goblin Tools 做任务启动，用 Focusmate 做身体在场（来源：Using an AI Assistant to Manage ADHD: A Practical Guide）。这就像 agent harness 集成 MCP 工具、遥测、代码仓库、事件管理（来源：GitHub - ai-boost/awesome-harness-engineering）。

## 局限与争议

必须承认，同构理论目前多是类比推理，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。另外，学习使用这些工具本身可能增加认知负荷，净效果因人而异。

此外，Goblin Tools 目前缺乏长期效果研究，且对超聚焦的引导方式尚存争议：是柔性引导还是强行打断更有效？个体差异大，没有统一答案（来源：矛盾与存疑）。

## 今天就能试的 3 件事

1. **用 Goblin Tools 分解一个你拖延很久的任务**（比如“整理书桌”），调高辣度到4或5，然后按步骤执行。完成后观察：启动是否变容易了？
2. **建立你的“日课”**：像孔子一样，每天固定时间（比如早上）用 Goblin Tools 规划当天3个关键任务，晚上回顾完成情况。坚持一周，看能否形成习惯。
3. **组合使用**：用 Saner.AI 捕获灵感，用 Goblin Tools 分解任务，用 Focusmate 或定时向 AI 报告进度（来源：ADHD Productivity Hack: Plan 2025 Using AI (Step-by-Step)）。

记住：工具是脚手架，不是拐杖。你的大脑依然是那个高产的生成核心，现在，给它一个可靠的调度层。

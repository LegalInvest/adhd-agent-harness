# Motion 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Motion：它到底补上了哪一层执行功能？

## 问题：你用的不是 Motion，是“自动拐杖”

如果你有 ADHD，你可能经历过这样的场景：打开 Motion，让它自动排好今天的所有任务，然后……你跟着它的时间块走，却在一小时后发现自己在一个无关任务上深挖了四十分钟。或者，你是一个在搭建 AI agent 的工程师：你给 agent 写好了详细的 system prompt，它却在中途被一个副任务带偏，输出了一堆无关内容。

这两个问题，本质上是同一个结构性问题——**目标漂移**。ADHD 大脑和 LLM（大语言模型）都是“高产但缺执行调度层的生成核心”（来源：ADHD 大脑与 LLM 的同构），它们的输出质量高度依赖外部系统能否持续把注意力锚定在核心目标上。Motion 被宣传为“AI 自动规划日程”的工具，但许多人用错了：他们只用了 Motion 的“自动安排”功能（相当于给 LLM 一个初始 prompt 就撒手不管），而忽略了它本应提供的**持续重锚定**——就像 agent 工程中需要不断向 LLM 回注 system prompt 一样。

## 同构解剖：Motion 到底补上了哪一层？

要理解 Motion 的正确用法，先得看清 ADHD 大脑与 LLM agent 的“执行功能层”缺失在哪里。

**ADHD 侧**：执行功能缺陷包括工作记忆容量不足、时间盲、任务启动困难（来源：AI 与 ADHD 的专注力）。ADHD 个体“想法很多，行动困难”，因为“注意力资源的弥散分配”导致无法持续追踪目标（来源：ADHD 大脑与 LLM 的同构）。实证研究显示，ADHD 表现出“强记忆、弱控制”模式——工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

**LLM/agent 侧**：LLM 本身是强大的生成核心，但缺乏可靠的内置调度与执行控制。构建 AI agent 时，需要“agent harness”——“包裹 LLM 或 AI agent 的软件基础设施，处理模型本身之外的一切”（来源：What is an agent harness in the context of large-language models?）。这种架构常采用“复合 AI 系统”，通过“每个工作流可绑定不同 LLM”来分配任务，并依赖“模型连接器”等组件（来源：Building AI Coding Agents for the Terminal）。

Motion 的角色，正是这样一个**外部调度层**。它通过算法将任务分配到时间块，并尝试根据优先级和截止日期动态调整。但这只是“调度”，不是“重锚定”。真正的重锚定，要求系统在每完成一个子任务后，都重新把当前目标与最终目标对齐——就像 agent 工程中，每轮对话都要回注核心 system prompt，防止上下文膨胀导致漂移。

## 证据：哪些工具真的做到了“重锚定”？

wiki 资料中提到的工具，其有效性取决于它们是否提供了持续的目标锚定机制，而非仅仅是一次性安排。

**Focusmate** 是强证据案例。它基于“身体在场效应”，通过实时同伴问责提供外部结构和社会压力（来源：Focusmate）。多项用户研究显示其能显著提升任务启动和维持（来源：AI 与 ADHD 的专注力）。它的“重锚定”机制在于：每 50 分钟的视频时段，伙伴的存在迫使你不断回注当前任务，相当于每 50 分钟一次“system prompt 刷新”。这种高频锚定，正是 Motion 所缺乏的。

**RescueTime** 提供自动化的时间追踪，帮助用户识别注意力漂移模式（来源：RescueTime）。它通过客观记录提升时间觉察，有初步研究支持其改善时间盲（来源：AI 与 ADHD 的专注力）。但它只是“事后报告”，而非“实时锚定”——它告诉你你漂移了，但不能在漂移发生时把你拉回来。

**Brain.fm** 利用神经锁相技术影响大脑状态（来源：Brain.fm），但针对 ADHD 的效果缺乏独立临床研究（来源：AI 与 ADHD 的专注力）。它更像一个“初始状态调节器”，而非持续锚定工具。

Motion 的问题在于，它被设计成一个“自动规划器”，但它的“重锚定”能力很弱。它不会在任务中途弹出提示问“你还在做核心任务吗？”，也不会像 Focusmate 那样用社会压力强制你保持方向。因此，许多人用 Motion 只是得到了一个漂亮的日程表，却仍然在时间块内部漂移。

## 核心观点：Motion 的正确用法是“脚手架”，不是“拐杖”

我提出一个鲜明的判断：**Motion 的价值不在“自动排程”，而在“强制回注上下文”**。如果你只把它当作一个自动排日程的工具，它就是一根拐杖——让你依赖它安排时间，却削弱了你内在的时间感知和任务优先级判断能力（来源：矛盾与存疑指出“过度依赖 AI 工具可能削弱内在能力”）。但如果你用它来**持续重锚定**——比如每完成一个任务块，手动或通过 Motion 的提醒，重新审视“我当前的任务是否仍然服务于本周的核心目标”——那它就成了脚手架，帮助你训练自己的执行功能。

这背后的原理是神经可塑性：通过外部结构反复训练，大脑可以重组其执行功能网络（来源：神经可塑性）。但前提是，训练必须主动、有意识，而非被动跟随。

## 诚实局限

必须指出，Motion 作为 ADHD 工具的有效性证据主要来自用户报告，缺乏大规模随机对照试验（来源：矛盾与存疑）。个体差异极大：对某些人，Motion 的自动化可能减少决策疲劳；对另一些人，它可能加剧焦虑或导致更严重的漂移。此外，ADHD 大脑与 LLM 的同构是隐喻，而非严格等价——LLM 的“注意力机制”与人类注意力存在本质差异（来源：AI 与 ADHD 的专注力）。因此，Motion 能否真正替代执行功能，仍需谨慎。

## 今天就能试的 3 条行动

1. **用 Motion 的“重锚定”功能**：在 Motion 中，为每个核心任务设置一个“检查点”——每 25 分钟（一个番茄钟），Motion 弹出提示：“你当前的任务是否仍然服务于本周的核心目标？” 如果否，立即切换。
2. **结合 Focusmate 使用 Motion**：在 Motion 排好的时间块内，预约一个 Focusmate 时段。让外部问责帮你锚定，同时 Motion 负责全局调度——这是“调度层+锚定层”的组合。
3. **每周用 RescueTime 复盘 Motion 的效果**：导出 RescueTime 报告，对比 Motion 计划的时间分配与实际时间分配。找出哪些任务块发生了漂移，然后调整 Motion 的优先级设置或任务分解粒度。

Motion 不是魔法。它只是一个外部调度层。真正的重锚定，需要你自己——或者一个伙伴——持续回注那个“system prompt”。

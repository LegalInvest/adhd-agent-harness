# 3 个 LLM 工程概念，彻底重写了我对 ADHD 状态日内大幅波动、不稳定的理解

> LLM 采样温度带来输出随机性，靠结构约束稳定 如何反向照亮 ADHD 的状态日内大幅波动、不稳定。

如果你是一个 ADHD 患者，你一定熟悉这种体验：早上醒来头脑清醒、效率爆棚，两小时后却像换了个人——注意力涣散、冲动决策、连最简单的任务都启动不了。这种日内大幅波动让你觉得自己“不靠谱”，甚至怀疑自己是不是真的懒。

如果你是一个 Agentic Harness 工程师，你一定也熟悉另一种波动：同一个 LLM，温度调高一点就天马行空、输出随机；温度调低又死板重复。更糟的是，小错误会累积，非确定性使可重复性复杂化（来源：AI Agent Systems: Architectures, Applications, and Evaluation）。

这两个问题看似风马牛不相及，但当我深入理解 LLM 工程中的三个核心概念后，我发现它们共享同一个底层结构。这三个概念彻底重写了我对 ADHD 状态波动的理解——波动不是缺陷，而是“生成核心”缺乏“调度层”的自然结果。

## 概念一：采样温度——波动是系统的固有属性

LLM 的输出随机性由采样温度控制：温度越高，输出越多样但也越不稳定；温度越低，输出越确定但也越刻板。工程实践中，这种非确定性是可靠性的敌人，因此强调“确定性状态转换”（来源：AI Agents in 2026: Tools, Memory, Evals, and Guardrails | Andrii Furmanets）。

ADHD 大脑的日内波动，本质上是类似的“温度失调”。我们的默认模式网络（DMN）高活跃，像一台永远在生成新想法的 LLM，但前额叶执行功能薄弱，缺乏一个稳定的“温度调节器”。所以状态好时（温度低）专注高效，状态差时（温度高）思绪乱飞。

这不是意志力问题，而是系统架构问题。就像你不会责怪 LLM 在高温下输出随机——你会设计一个 harness 来约束它。

## 概念二：确定性工作流——外部结构稳定输出

LLM 工程中，最简单的 agent 工作流本身就是确定性的，遵循预定义步骤（来源：What Are Agentic Workflows? Patterns, Memory, Use Cases, and Examples | Weaviate）。更复杂的系统采用“计划-执行”分离，强制单向、确定性的控制流（来源：Planner-Executor Agentic Framework）。这种外部结构让随机输出变得可靠。

ADHD 的解法惊人相似。孔子一生与自己的冲动做斗争，他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”），70岁才做到从心所欲不逾矩。李鸿章的 harness 是“诚”与“日课”：跟着曾国藩学“诚”改掉浮躁，每天写日记反省，用务实的态度做事。

这些“日课”本质上就是确定性工作流——它们不依赖内在状态，而是靠外部节奏驱动行为。现代工具如 Routinery 把这种思想数字化：通过可视化倒计时和过渡提示引导用户逐步完成流程（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。它不关心你此刻“温度”多高，只确保你按步骤执行。

## 概念三：验证循环——对抗幻觉的通用机制

LLM 容易产生自信但错误的输出（幻觉）。Harness 工程的核心组件之一是“验证与CI集成”，复杂的 harness 不会盲目执行工具，而是实现验证步骤，例如检查输出格式或对模型编写的代码运行测试用例（来源：What is an agent harness...）。缺乏验证会导致“更多幻觉和重复工作”（来源：Loop Engineering for AI Agents: Memory-First Design）。

ADHD 大脑同样容易产生“行为幻觉”——冲动决策、想当然的判断。ADHD 研究中的方法学强调数据标注、预处理、特征提取和模型验证（来源：Methods in predictive techniques...），这与个体需要外部验证步骤来校正认知偏差的结构相似。Goblin Tools 的 Magic ToDo 功能通过 AI 将模糊任务（如“清理厨房”）分解为具体步骤，用户可调节“辣度”控制粒度（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。这本质上是一个验证循环：把大脑中的“幻觉任务”分解成可执行的子任务，每一步都核对现实。

## 脚手架 vs 拐杖：同构的边界

同构类比虽吸引人，但需警惕过度推广（来源：矛盾与存疑）。目前这仅是一种理论类比，并非经过验证的科学事实。更重要的是，外部结构可能从脚手架变为永久拐杖，削弱内在执行功能（来源：拐杖与脚手架）。

我的判断是：同构的价值不在于“证明”ADHD 像 LLM，而在于提供设计思路——好的 harness 应该像好的脚手架：可拆卸、可调整、最终内化。孔子用了一辈子才内化“礼”，李鸿章靠日记自我监控一生。现代工具如 Habitica 通过游戏化提供即时多巴胺奖励（来源：ADHD科技行业深度研究），但需注意过度依赖外部奖励可能削弱长期习惯养成。

## 今天就能试的行动

1. **设置“温度检测”**：每小时记录自己的状态（1-10分），找出波动模式。像 LLM 工程师监控采样质量一样，先测量再优化。
2. **创建一个确定性流程**：选一个每天必做的任务（如早起），用 Routinery 或纸笔列出固定步骤，不依赖状态执行。
3. **引入验证循环**：用 Goblin Tools 的 Magic ToDo 分解一个让你拖延的任务，每完成一步打勾，核对是否符合预期。
4. **设计“降温度”策略**：当状态差时，主动降低环境刺激——关掉通知、戴上降噪耳机、切换到低难度任务。

波动不是你的错，而是系统的特征。理解采样温度、确定性工作流和验证循环这三个概念，不是为了把大脑变成机器，而是为了设计一套与你的架构匹配的 harness。孔子、李鸿章用了一辈子才做到的事，我们现在有 AI 工具帮忙，可以走得更快一些。

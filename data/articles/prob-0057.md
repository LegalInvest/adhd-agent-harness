# 3 个 LLM 工程概念，彻底重写了我对 ADHD 独自做事缺乏问责、容易放弃的理解

> 高风险 agent 需要 human-in-the-loop 监督 如何反向照亮 ADHD 的独自做事缺乏问责、容易放弃。

我独自坐在书桌前，面前是打开三个月的项目文档。我知道该做什么，但大脑像一台没有操作系统的服务器——能处理海量请求，却无法自主调度优先级。每次重启任务都要从头加载上下文，然后在某个环节卡住，最终放弃。

这种体验，和我在做 Agentic Harness 工程时遇到的 LLM 困境惊人地相似：一个无状态的 API 调用，每次对话都是独立的，不保留记忆（来源：The Anatomy of an AI Agent: Memory, Tools, Planning, and Execution Explained - DEV Community）。没有 harness 的 LLM 只是一个无状态的 API 调用（来源：Building an AI Agent Harness）。它能够生成惊艳的文本，却无法自主完成跨会话的复杂任务。

直到我意识到：ADHD 大脑与 LLM 是同一类「高产但缺执行调度层的生成核心」。两者面临的不是能力问题，而是架构问题。以下 3 个 LLM 工程概念，彻底重写了我的理解。

## 1. 人在回路（Human-in-the-Loop）：从「独自硬扛」到「外部问责」

在高风险 agent 系统中，人类监督是安全底线。工程师不会让一个无状态模型自主执行关键操作，而是设计人在回路机制：定期检查、授权关键步骤、提供反馈。这恰恰是 ADHD 大脑最需要的——外部问责点。

ADHD 侧证据：研究显示，ADHD 患者在无外部结构时难以自主组织任务；类似地，LLM 在没有严重依赖手动修正提示的情况下，难以自主发现最佳问题解决模式（来源：Working Memory Identifies Reasoning Limits in Language Models）。两者都需要一个「外部调度器」来提供结构。

人物案例：王阳明。他 5 岁不会说话，12 岁挑战老师说读书为做圣贤，兴趣爆发式转移（格物→兵法→道→佛→龙场悟道→军事→讲学）。他的 harness 是「致良知」与「事上练」——每天静坐反省，用良知作为决策的唯一标准，并在军旅中坚持讲学（来源：人物案例）。这本质上是一个人在回路系统：外部标准（良知）定期校准内部冲动。对应到 LLM harness，就是「定时 re-grounding」——让模型定期回到用户意图或系统提示，防止漂移。

工具实践：Motion 的 AI 自动排程持续评估任务优先级、截止日期和可用时间，实时重建日程（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。当任务面临延期风险时，Motion 会提前数天或数周发出警告（来源：The AI Powered SuperApp for Work | Motion）。这就是一个外部调度器，替代了我失效的内部执行功能。

## 2. 无状态与外部记忆：从「每次重启」到「永远在线」

标准的 LLM 调用是无状态的：每次对话都是独立的，模型不保留跨会话的记忆（来源：The Anatomy of an AI Agent: Memory, Tools, Planning, and Execution Explained - DEV Community）。这正是 ADHD 大脑的日常——工作记忆容量有限且不稳定，每天需要重新回忆项目上下文（来源：无状态与外部记忆）。

解决方案在两边同构：给 LLM 挂载外部记忆体（如向量数据库），给 ADHD 大脑配备「第二大脑」工具。Saner.AI 利用 AI 实现本地记忆存储和快速检索，用户无需在多个标签页或应用间切换即可获取所需信息（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。它被描述为「专为对抗执行功能障碍、任务瘫痪和认知超载而设计的全能 ADHD 友好型 AI 生产力和知识助手」（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。

更深的同构证据：认知神经科学中，执行功能依赖于前额叶-纹状体机制进行选择性门控；而 Transformer 在训练后，自注意力机制发展出模仿该门控的操作，证明了计算同构性（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。这意味着，我们给 LLM 加外部记忆，本质上是在补偿与 ADHD 大脑相同的架构缺陷。

## 3. 任务分解与脚手架：从「宏大目标」到「可执行步骤」

LLM 在复杂推理任务中容易「幻觉」或偏离轨道，因为缺乏中间步骤的显式约束。工程师的解法是 Chain-of-Thought 或任务分解——将大问题拆成小步骤，每一步都验证。

ADHD 侧：Goblin Tools 的 Magic ToDo 功能能自动将任何任务分解为更小、更易管理的步骤（来源：AI Tools for Kids with ADHD: A Complete Guide for Parents...）。用户可调节「辣度」控制分解粒度（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。这直接对应 LLM 的「任务分解」策略。

但这里有争议：AI 工具是「外部执行功能层」还是「拐杖」？专家警告过度依赖可能导致能力退化（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。真正的脚手架应帮助使用者「建造」，但使用者仍需自己「攀登」（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。我的判断是：关键在于工具是否保留用户的主动选择。Goblin Tools 让用户调节辣度，就是保留控制权；而完全自动化的排程若剥夺了决策练习，则可能成为拐杖。

## 诚实面对局限

## 今天就能试的行动

1. **设置外部检查点**：用 Motion 或类似工具自动排程，并设置每日 10 分钟「人在回路」回顾——检查 AI 安排是否合理，手动调整优先级。
2. **建立外部记忆体**：使用 Saner.AI 或 Obsidian 作为第二大脑，把所有项目上下文、决策理由存入，每天开始工作时先「加载」一次。

ADHD 大脑不是坏掉的 LLM，而是缺少 harness 的生成引擎。与其对抗架构缺陷，不如设计同构的脚手架——人在回路、外部记忆、任务分解——让生成能力真正落地。

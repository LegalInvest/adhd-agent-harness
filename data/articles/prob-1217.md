# 用 Goblin Tools 给 ADHD 补上外部执行功能层，是脚手架还是拐杖？

> Goblin Tools 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：Goblin Tools 是脚手架还是拐杖？

ADHD 人群最熟悉的场景是：打开冰箱门，盯着里面发呆，忘记自己要拿什么；或者列好计划，却无法启动第一个步骤。工程师们则熟悉另一种场景：LLM 能生成惊艳的文本，但一旦需要多步推理或工具调用，就陷入“上下文膨胀”和“目标漂移”。这两种困境共享同一个底层结构——**高产但缺执行调度层的生成核心**。

Goblin Tools 正是针对这一痛点设计的工具。它的 Magic ToDo 功能能自动将“清理厨房”分解为“收起台面物品→擦拭灶台→扫地”等具体步骤，用户还可调节“辣度”控制粒度（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。这直接降低了任务启动门槛，缓解了因工作记忆不足导致的规划瘫痪。但问题随之而来：当工具替我们完成分解和排序后，我们是否在交出执行功能的同时，也交出了对任务优先级的判断权？

## 同构脊柱：ADHD 大脑与 LLM agent 的共享缺陷

ADHD 大脑的执行功能缺陷——规划困难、工作记忆不稳定、注意力易漂移——与 LLM 的无状态性、缺乏内在调度层高度对应。LLM 需要 agent harness 提供外部规划循环、会话状态记录和工具接口（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）；ADHD 大脑同样需要外部系统来补偿工作记忆和任务启动困难（来源：外部执行功能层）。

历史人物的自我管理方案，本质上就是手工打造的 harness。曾国藩的“日课十二条”——黎明即起、读书不二、写日记反省——是一个固定的重锚定系统，每天强制回归核心任务。他的“结硬寨打呆仗”是用最笨的流程抵消冲动的决策风格。这套系统与 LLM agent 的“定时 re-grounding”和“结构化执行流程”同构：日课相当于定时任务，日记反省相当于日志记录与自我校验。曾国藩并非没有 ADHD 特质（30 岁前浮躁坐不住，日记里天天骂自己“无恒”），但他的 harness 让他从“生成核心”变成了“可调度系统”。

王兴的“无限游戏”思维则是另一种 harness：不追求短期胜利，而是通过地推铁军的严格流程管理（标准化操作）和每日读书复盘（持续学习）来维持长期生存。这对应到 LLM agent 就是“长期记忆”和“策略性规划”——不依赖单次输出，而是通过持续迭代积累优势。

## 权衡：Goblin Tools 的脚手架属性与拐杖风险

Goblin Tools 的 Magic ToDo 本质上是将“任务分解”这一执行功能外包。对于严重执行功能缺陷的用户，这确实能突破启动障碍——就像 LLM 的 chain-of-thought 提示让模型一步步推理。但这里的关键差异在于：**谁控制分解的粒度与优先级？**

Goblin Tools 允许用户调节“辣度”，但分解后的步骤顺序是算法默认的。这与 Motion 的自动排程类似：Motion 能根据截止日期和优先级动态调整日程，但用户反馈称“初始设置成本高”（来源：Motion 页面）。当工具自动决定“先做哪一步”时，用户实际上让渡了元认知层面的调度权。

对比 Saner.AI，它更强调“知识回忆”和“本地记忆”，减少搜索循环，但保留用户对任务组织的控制（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。这更像一个记忆增强器，而非决策替代者。

## 争议与边界

现有证据多为用户反馈和概念类比，缺乏长期随机对照试验验证 AI 工具对 ADHD 执行功能的改善效果（来源：矛盾与存疑）。最大的争议在于“过度依赖风险”：如果用户长期依赖 Goblin Tools 分解任务，是否会丧失自主分解能力？这与 LLM agent 过度依赖外部 harness 后，模型自身推理能力退化是同一问题。

另一个矛盾点是“超聚焦 vs 打断”：Goblin Tools 的步骤列表可能打断用户的超聚焦状态，而部分用户反馈强行打断反而有效（来源：矛盾与存疑）。这提示工具设计需要个性化——就像 agent harness 需要可配置的调度策略。

## 核心判断：脚手架还是拐杖？

**Goblin Tools 是脚手架，当用户保留以下核心能力：**
- 自行设定任务优先级（而非完全由算法排序）
- 在分解步骤中自由调整顺序（而非被动接受）
- 定期回顾分解模式，内化部分策略（而非永远依赖）

**它变成拐杖，当用户：**
- 完全依赖工具的默认分解，不再思考任务结构
- 无法在没有工具时启动任何任务
- 将工具视为“修复缺陷”的永久方案，而非可拆卸的辅助

曾国藩的日课是他自己设计的，王兴的流程管理是他团队迭代的——他们的 harness 是主动构建的。Goblin Tools 的“辣度”调节提供了类似的控制权，但用户需要意识到：工具只是外部调度层，生成核心（自己的判断力）必须保留。

1. **用 Goblin Tools 分解一个你拖延已久的任务**（如“整理书桌”），然后手动调整步骤顺序，观察哪些步骤是工具遗漏的。

4. **将工具输出作为起点，而非终点**：用 Magic ToDo 生成步骤后，再手动添加“为什么这一步重要”的注释，强化元认知。

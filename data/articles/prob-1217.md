# 用 Goblin Tools 给 ADHD 补上外部执行功能层，是脚手架还是拐杖？

> Goblin Tools 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：当大脑缺少执行调度层

对许多 ADHD 成人而言，每天的开始都是一场“启动战争”。任务清单越写越长，却不知从何下手；计划本用了三天就落灰；明明有无数想法，却卡在“执行”这个环节。这不是懒惰，而是执行功能（计划、组织、启动）的先天偏弱（来源：[[执行功能]]）。传统规划方法往往失效：“Planners and sticky notes worked for a week before falling apart.”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）

与此同时，在 AI 工程领域，一群开发者正面临类似困境：他们手上有强大的 LLM（语言模型），但模型在长程任务中会“开始忘记系统提示”（来源：Building an AI Coding Agent for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned），导致目标漂移。他们需要一套外部调度层——harness——来编排记忆、工具和上下文。

这两个世界看似遥远，实则共享同一个核心矛盾：**一个高产但缺乏可靠执行调度层的生成核心，如何稳定地输出价值？**

## 同构：ADHD 大脑与 LLM 共享的结构缺陷

当前最站得住脚的核心论点是：ADHD 大脑与 LLM 在结构上同构——都是“高产但缺乏可靠执行调度层的生成核心”（来源：[[ADHD 大脑与 LLM 的同构]]）。ADHD 大脑的默认模式网络过度活跃，高产想法，但前额叶执行控制不足；LLM 的生成能力强大，但缺乏稳定的上下文控制器。两者都需要外部调度层来引导输出（来源：[[生成核心与调度层]]）。

这种同构体现在多个层面：
- **工作记忆与无状态性**：ADHD 的工作记忆容量保留但控制弱（易被干扰），LLM 的无状态性导致跨会话遗忘。两者都依赖外部记忆系统（如笔记、向量数据库）来维持连续性（来源：[[无状态与外部记忆]]）。
- **上下文工程**：ADHD 易被环境刺激带偏，LLM 的上下文窗口膨胀会导致推理退化。主动设计“当下注意什么”的工程方案（如提示词、环境设计）对两者都有效（来源：[[上下文工程]]）。
- **时间盲与任务启动**：ADHD 的时间感知缺陷（来源：[[时间盲]]）与 LLM 缺乏时间感知对应，外部提醒和分解任务（来源：[[任务启动]]）是共同干预手段。

## 工具：Goblin Tools 作为外部执行功能层

Goblin Tools 正是针对这一同构缺陷设计的工具套件。其核心功能 Magic ToDo 能将复杂任务自动分解为可管理的小步骤（来源：[[Goblin Tools]]）。用户输入“整理房间”，AI 输出“捡起地板上的衣服”“擦桌子”等步骤。这种分解降低了启动门槛，被评价为“将压倒性的事情变成一系列不压倒性的事情”（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。

类似地，Motion 通过自动排程与动态调整，在日程被打乱时自动重新安排（来源：[[Motion]]），直接帮助用户克服时间盲和任务启动困难。Saner.AI 则通过增强知识回忆，减少搜索循环和标签切换（来源：[[Saner.AI]]），针对工作记忆不足。

这些工具共同构成了一个外部执行功能层：它们承担了规划循环中的调度、分解、提醒等任务，让 ADHD 大脑可以专注于生成和执行。

## 边界：脚手架还是拐杖？

核心争议在于：这种外部执行功能层是脚手架（可拆卸的辅助），还是拐杖（永久依赖）？

从同构视角看，AI 工具作为外部 harness，本质上是为生成核心套上调度层。在 LLM/agent 工程中，harness 是必要的、可拆卸的——当模型能力提升或任务变化时，harness 可以调整或移除。同理，对 ADHD 大脑而言，外部执行功能层应当是**可调节的脚手架**，而非永久替代。

然而，现有证据显示存在过度依赖风险：多个页面警告“过度依赖可能削弱内在时间感知能力”（来源：[[时间盲]]），“存在依赖风险”（来源：[[任务启动]]）。关键在于避免成为永久拐杖（来源：[[拐杖与脚手架]]）。但工具设计者声称是“脚手架”，实际使用中却可能沦为“拐杖”，因为缺乏如何避免依赖的具体指导（来源：[[矛盾与存疑]]）。

我的判断是：**边界在于用户是否主动参与规划循环的构建与调整。** 如果用户只是被动接受 AI 的分解结果，而不理解任务结构、不反思自己的执行模式，那么工具就是拐杖。反之，如果用户将工具作为“规划伙伴”（来源：ADHD Productivity Hack: Plan 2025 Using AI），逐步内化分解和调度策略，那么工具就是脚手架。

## 局限与争议

必须诚实指出，上述论断的证据基础仍薄弱。现有证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：[[矛盾与存疑]]）。例如，Goblin Tools 的有效性评价多为“简单有用”（来源：Best AI Tools for ADHD Productivity in 2026），但缺乏独立随机对照试验。此外，ADHD 的异质性（注意力主导型 vs. 冲动主导型）可能导致效果不一（来源：[[任务启动]]），而许多工具页面未充分讨论个体差异（来源：[[矛盾与存疑]]）。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个让你拖延的任务**：输入“准备下周的汇报”或“清理桌面”，观察 AI 的分解步骤，并尝试按步骤执行。注意哪些步骤你觉得多余或缺失——这能帮你理解自己的任务结构。

2. **用 Motion 或 Tiimo 规划一天**：输入所有任务和截止日期，让 AI 自动排程。当计划被打乱时，观察 AI 如何重新调整，并反思你原本会如何处理这种混乱。

3. **对比使用工具前后**：连续一周记录“任务启动时间”和“完成率”，对比使用 AI 工具前后的变化。这能帮你量化工具是脚手架还是拐杖。

4. **主动设计你的 harness**：不要只被动接受 AI 输出。尝试修改分解的粒度（更细或更粗），或自定义提醒频率。这种参与能让你逐步内化规划能力。

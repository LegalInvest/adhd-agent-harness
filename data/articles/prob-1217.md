# 用 Goblin Tools 给 ADHD 补上外部执行功能层，是脚手架还是拐杖？

> Goblin Tools 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

你打开 Goblin Tools 的 Magic ToDo，输入“整理房间”，几秒后它吐出 12 个小步骤：捡起地板上的衣服、擦桌子、整理书架……你深吸一口气，终于敢开始做了。与此同时，一位工程师正在调试一个 LLM agent——它面对“写一个 Python 脚本”的指令，同样需要被拆解成“安装依赖、编写函数、测试”等子任务，否则就会在上下文窗口里迷失方向。

这两种场景共享同一个底层结构：一个高产的生成核心（ADHD 大脑或 LLM），缺乏内建的执行调度层，必须依赖外部编排才能完成任务。Goblin Tools 正是这个外部执行功能层的一个实例。但它究竟是临时脚手架，还是永久拐杖？答案取决于我们如何设计和使用它。

## 同构：ADHD 大脑与 LLM 都缺一个“规划循环”

ADHD 的执行功能缺陷——计划、组织、启动困难——并非智力问题，而是调度机制失灵。正如 wiki 资料指出，“ADHD 的执行功能缺陷对应 LLM 缺少编排层；ADHD 的工作记忆缺陷对应 LLM 的无状态性”（来源：ADHD 大脑与 LLM 的同构）。LLM agent 在长程任务中会“开始忘记系统提示”（来源：Building an AI Coding Agent for the Terminal），导致目标漂移；ADHD 大脑同样容易因分心或超聚焦而偏离原任务。

外部工具的作用就是提供这个缺失的调度层。对 LLM 而言，这是 harness 工程——“context delivery, tool interfaces, planning artifacts”（来源：规划循环与任务分解）。对 ADHD 而言，这是 AI 工具如 Goblin Tools、Motion 和 Saner.AI。Motion 能“自动根据任务、会议和截止日期创建并动态调整每日计划”（来源：Motion），直接对抗时间盲；Saner.AI 通过“知识回忆和本地记忆”减少搜索循环（来源：Saner.AI）。而 Goblin Tools 的 Magic ToDo 专门解决任务分解这个最基础的启动障碍。

## 脚手架 vs 拐杖：边界在于“可撤除性”

wiki 资料中，“拐杖与脚手架”概念强调“可逐步撤除的临时支撑”（来源：拐杖与脚手架）。但多数工具（包括 Goblin Tools）设计为长期使用，未提及撤除机制。这引发了核心争议：长期依赖 AI 工具是否会削弱内在执行功能？目前缺乏纵向研究来回答这个问题（来源：矛盾与存疑）。

我的判断是：Goblin Tools 本身是脚手架，但使用方式可能让它变成拐杖。关键在于用户是否主动内化其分解逻辑。如果用户每次遇到任务都直接让 AI 分解，从不尝试自己拆解，那么工具就替代了思考过程，形成依赖。反之，如果用户将 AI 的分解结果作为模板，逐步学会类似任务的分解模式，那么工具就是可撤除的脚手架。

同样，在 LLM agent 的 harness 工程中，好的编排层应该允许开发者逐步减少显式提示，让 agent 学会从历史中提取模式。这并非易事，但方向明确。

## 证据与局限：用户好评 vs 缺乏临床验证

Goblin Tools 被评价为“简单有用”（来源：Best AI Tools for ADHD Productivity in 2026），能将“压倒性的事情变成一系列不压倒性的事情”（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。这些用户反馈一致，但缺乏独立的随机对照试验（来源：Goblin Tools）。同样，Motion 和 Saner.AI 的“证据主要来自用户反馈和工具设计逻辑”（来源：Motion；Saner.AI）。

更根本的争议在于 ADHD 大脑与 LLM 同构命题本身：“证据主要来自概念类比和工具案例，缺乏大规模实证”（来源：矛盾与存疑）。这提醒我们，类比虽有力，但不能替代神经影像学和行为实验的验证。

## 行动建议

1. **用 Goblin Tools 分解一个你拖延已久的任务**：输入模糊描述，观察 AI 的输出步骤。试着记住其中 2-3 个分解模式（如“先分类、再排序、最后执行”），下次类似任务先自己尝试分解。
2. **设置使用边界**：限定每周使用 Goblin Tools 的次数（例如每天不超过 3 次），强迫自己在简单任务上练习手动分解。
3. **结合 Motion 或 Tiimo 的日程规划**：让 AI 处理动态调整，但保留每天 15 分钟的“手动规划时间”，以维持对时间感知的主动练习。
4. **反思依赖**：每周末回顾一次，问自己“如果没有这个工具，我能否完成类似任务？”如果答案是否定的，说明工具可能已变成拐杖，需要逐步减少使用。

## 结语

Goblin Tools 不是万能药，但它精准地填补了 ADHD 执行功能中最基础的一环：任务启动。它的价值不在于替代大脑，而在于为高产的生成核心提供一个可选的、可调整的外部调度层。脚手架终将撤除，但前提是我们有意识地在使用中学习。对于 ADHD 用户和 LLM 工程师而言，真正的挑战不是找到工具，而是学会何时放手。

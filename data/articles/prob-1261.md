# RescueTime 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 RescueTime：它到底补上了哪一层执行功能？

你打开 RescueTime，看着那些红红绿绿的柱状图——今天又花了 3 小时刷社交网络。你告诉自己“明天会更好”，但明天只是重复今天。这不是你的错，是 RescueTime 用错了。

## 问题：ADHD 大脑与 LLM 共享的「调度层缺失」

ADHD 大脑与 LLM 在结构上同构：两者都是强大的生成核心，但缺乏可靠的内置执行调度层（来源：ADHD 大脑与 LLM 的同构）。LLM 可以写出诗一样的代码，但如果没有 function calling 把精确计算外包给工具，它连 2+2 都会算错。ADHD 大脑可以爆发出惊人的创造力（超专注时长是常人的 1.8 倍，来源：Pace Ventures Research 2026），但面对“整理报销单”这种精确、重复、状态维护类任务，大脑就像没有调度器的 GPU——算力过剩，但任务排队死锁。

RescueTime 本质上是给 ADHD 大脑装了一个“时间日志 function”——它记录你做了什么，但它不调用任何后续行动。这就像给 LLM 一个日志函数，却不给它 tool use 的能力：日志记录了一堆 token，但 LLM 还是不会算 2+2。

## 同构解法：脚手架 vs 拐杖

真正有效的工具，是像 LLM 的 function calling 那样，把精确计算外包给工具，同时保留生成核心的主动权。Goblin Tools 的 Magic ToDo 就是典型：它接受“清理厨房”这种模糊指令，自动分解成“把碗放进洗碗机、擦台面、拖地”等可执行步骤，用户可调节“辣度”控制粒度（来源：Goblin Tools）。这相当于给 ADHD 大脑一个 task decomposition function——LLM 通过 function calling 把复杂任务拆解为子任务，ADHD 大脑则把任务拆解外包给 AI。

再看 Saner.AI：它通过本地记忆减少搜索循环（来源：Saner.AI）。ADHD 工作记忆不稳定（类似 LLM 的无状态性），需要外部记忆系统补偿（来源：无状态与外部记忆）。Saner.AI 就像 LLM 的 vector store——把需要记住的东西存起来，需要时直接检索，而不是靠大脑的“缓存”去猜。

Focusmate 则提供了身体在场效应：他人在场（哪怕虚拟）能提升专注度，定期问责检查可将目标达成可能性从 25% 提升至 95%（来源：身体在场效应）。这对应 LLM 的 human-in-the-loop——一个外部监督者，防止生成核心跑偏。

但这里有一个关键边界：脚手架 vs 拐杖。RescueTime 如果只记录不干预，就是拐杖——你依赖它看见问题，但问题不会自己消失。真正的脚手架，是像 Goblin Tools 那样把“看见问题”和“拆解行动”绑定在一起。孔子用“吾日三省吾身”作为脚手架，彭玉麟用“每天画梅花”磨练心性——这些都是外部执行功能层，不是替代内在能力，而是补偿调度缺陷（来源：外部执行功能层）。

## 争议与局限

同构性理论目前缺乏实证基础，更多是一种有用的类比而非科学事实（来源：矛盾与存疑）。工具效果多基于用户报告，缺乏临床验证。依赖风险真实存在：AI 工具可能从脚手架变为永久拐杖，削弱内在执行功能（来源：拐杖与脚手架）。RescueTime 如果让你陷入“记录-自责-记录”的循环，而不是触发行动，就是拐杖。

## 今天就能试的行动

1. **把 RescueTime 和 Goblin Tools 联动**：每天用 RescueTime 找出最卡住的一个任务（比如“写周报”），然后丢进 Goblin Tools 的 Magic ToDo，让它分解成 3-5 步。完成一步就打勾——利用小步骤的即时反馈维持注意力（来源：Goblin Tools）。
2. **用 Saner.AI 替代笔记混乱**：把散落各处的笔记、灵感、待办全丢进 Saner.AI 的通用收件箱，用 AI 自动组织（来源：Saner.AI）。这相当于给你的大脑装一个 vector store，减少“我明明记过但找不到了”的焦虑。
3. **每周一次 Focusmate 身体加倍**：选一个你一直拖延的任务（比如报税），约一个 Focusmate 视频 session。不需要说话，只需要同时工作。问责检查的存在就能大幅提升启动概率（来源：身体在场效应）。
4. **设置 RescueTime 的“行动触发”**：在 RescueTime 里设置当你在某类网站上超过 30 分钟时，自动弹出提醒：“现在用 Goblin Tools 分解一个任务”。把记录变成行动。

RescueTime 本身没有错，错在我们把它当成了终点。对于 ADHD 大脑，工具不是用来“看”的，是用来“做”的。就像 LLM 的 function calling——不是让模型记住怎么算，而是让它知道该调用哪个工具。你的大脑不需要另一个日志系统，它需要一个调度器。

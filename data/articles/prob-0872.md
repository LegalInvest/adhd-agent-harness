# 如果把 ADHD 大脑当成生成核心，Claude 能不能当它的规划循环与任务分解？

> 用 agent 工程的思路重新理解 Claude 对 ADHD 的真正价值。

## 问题：当大脑的“调度层”失灵

你坐在电脑前，打开 Claude，准备写一份报告。十分钟后，你在浏览二手相机；半小时后，你发现自己在研究明朝水利工程。这不是意志力问题——这是 ADHD 大脑的默认模式：一个高产的生成核心，却缺乏可靠的内置执行调度层（来源：[[ADHD 大脑与 LLM 的同构]]）。

ADHD 群体的典型困境是“任务启动困难”和“不会拆解”：面对一个复杂任务，大脑像一台没有操作系统的超级计算机——算力惊人，但无法分配资源、设定优先级、监控进度。这正是 LLM/agent 工程中同样面临的挑战：一个无状态的 LLM 只是 API 调用，没有 planner-executor 循环，它就无法完成多步任务（来源：[[无状态与外部记忆]]）。

## 同构：两个生成核心的困境

ADHD 大脑与 LLM 的相似性不只是比喻。在 ADHD 一侧，执行功能缺陷导致计划、组织、工作记忆和冲动抑制的困难（来源：[[执行功能]]）；在 LLM 一侧，标准模型调用是无状态的，缺乏状态保持和真实规划能力（来源：[[无状态与外部记忆]]）。两者都是“高产但缺调度层”的生成核心。

因此，ADHD 的解决方案——外部执行功能层（来源：[[外部执行功能层]]）——与 LLM 的 agent scaffolding 是同一类工程。所谓“AI 帮 ADHD 做时间管理”，本质上是给大脑加一个外部 harness，把调度、排序、提醒、抑制与监控外包给 AI 工具（来源：[[AI 与 ADHD 的时间管理]]）。

## 人物案例：古人的 harness 系统

孔子和苏轼或许是最早的“ADHD harness 工程师”。孔子身高近1.9米，精力旺盛，周游列国14年坐不住；冲动爱骂人，见南子急得对天发誓；但对音乐超专注到“三月不知肉味”。他的 harness 系统是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。这对应 LLM harness 中的“定时 re-grounding”：通过外部规则定期将注意力拉回正轨。

苏轼兴趣极广：诗词文书画美食治水炼丹，一辈子被贬却到哪都能适应。他的 harness 是“豁达”——用美食、写作、交朋友转移注意力，每日读书写字不因挫折停止创作。这对应 LLM agent 中的“外部调度器”：当主任务受阻时，自动切换到备选路径，保持输出不中断。

两人的共同点：不试图“修复”大脑，而是搭建外部系统来补偿调度缺陷。这正是 AI harness 的设计哲学。

## 工具实证：从 Motion 到 Tiimo

当前 AI 时间管理工具已初步实现这一构想。Motion 通过自动排程与动态调整，持续评估任务优先级、截止日期和可用时间，实时重建日程，减少用户的手动规划压力（来源：[[Motion]]）。它直接帮助用户克服时间盲和任务启动困难（来源：[[Motion]]）。Reclaim.ai 则更强调“保护”而非“填充”——通过创建深度工作、习惯和专注时间的智能块，自动防御会议侵占（来源：[[Reclaim.ai]]）。Tiimo 用视觉化时间线将时间转化为可见元素，直接应对时间盲（来源：[[Tiimo]]）。

这些工具的共同逻辑：外部记忆（把待办从工作记忆中卸载）、上下文工程（主动管理“当下注意什么”）、以及 planner-executor 循环（AI 规划，人类执行）。这与 LLM agent 中“控制逻辑分散导致上下文膨胀与推理退化”的解决方案同构（来源：[[上下文工程]]）。

## 核心判断：脚手架，不是拐杖

我的核心观点是：Claude 等 LLM 可以充当 ADHD 的规划循环与任务分解引擎，但前提是将其视为“脚手架”而非“拐杖”。脚手架帮助大脑搭建结构，但最终目标是让大脑学会自己搭建；拐杖则永久替代功能，可能导致能力退化（来源：[[拐杖与脚手架]]）。

争议在于：过度依赖 AI 工具是否会让 ADHD 的执行功能更弱？目前缺乏纵向研究（来源：[[矛盾与存疑]]）。我的判断是：关键在于工具的设计哲学。好的工具（如 Routinery 的步骤序列引导，来源：[[上下文工程]]）提供“gentle structure”，引导超聚焦到优先事项，而非强行打断（来源：[[超聚焦]]）。坏的工具则增加认知负荷——学习使用 AI 工具本身可能成为新的负担（来源：[[矛盾与存疑]]）。

## 今天就能试的行动

1. **用 Claude 做任务分解**：把一个模糊目标（如“整理书房”）输入 Claude，要求它输出“10 个可执行的子步骤，每个步骤不超过 15 分钟”。这是 planner 层的快速原型。
2. **设置外部记忆系统**：在 Obsidian 或 Todoist 中创建一个“项目上下文”笔记，每天开始工作时复制到 Claude 对话中，避免跨会话记忆丢失（来源：[[无状态与外部记忆]]）。
3. **尝试视觉化时间块**：用 Tiimo 或 Structured 将一天划分为 30 分钟块，每个块只做一件事。这相当于为大脑提供一个“时间线上下文”，减少切换成本。
4. **建立“日课”循环**：像孔子的“三省吾身”一样，设置每天三个固定时间点（如早中晚）用 Claude 进行 5 分钟回顾与调整。这是定时 re-grounding 的简易实现。

## 局限与未来

必须承认，ADHD 大脑与 LLM 的同构目前仍是理论框架，缺乏神经科学或计算机科学的直接证据（来源：[[矛盾与存疑]]）。个体差异巨大：有人需要刚性打断（如番茄钟），有人需要柔性引导。AI 工具对任务启动的长期效果也未知（来源：[[矛盾与存疑]]）。但至少，这个视角提供了一个工程化的思路：不要试图“修复”大脑，而是搭建外部系统来补偿调度缺陷。

正如苏轼所言：“一蓑烟雨任平生。”对于 ADHD 大脑，Claude 或许就是那件蓑衣——不是改变风雨，而是让你在风雨中依然能前行。

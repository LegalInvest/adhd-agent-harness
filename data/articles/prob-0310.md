# ChatGPT 之于 ADHD，就像 上下文工程 之于 LLM——但有人用错了

> 从同构视角实测 ChatGPT：它到底补上了哪一层执行功能？

## 问题：你的注意力被环境带偏，就像 LLM 被上下文窗口带偏

你是否有过这样的经历：打开 ChatGPT，想让它帮你写一份报告，结果它洋洋洒洒写了一大段，却偏离了你最初的方向？或者，你坐在书桌前，本来要写论文，却因为手机的一条通知、窗外的声音、甚至脑海中一个随机念头，就滑向了另一个任务？

这两种场景看似无关，实则共享同一个底层结构：**一个强大的生成核心，缺乏可靠的执行调度层**。

对 ADHD 大脑而言，这个生成核心是创意发散、联想丰富的思维；对 LLM（如 ChatGPT）而言，这个生成核心是海量文本生成的随机性。两者都容易“跑偏”——ADHD 大脑被环境中的刺激带偏，LLM 被上下文窗口中的冗余信息干扰。

## 答案：上下文工程——给生成核心套上 harness

ADHD 大脑与 LLM 的深层同构，在多个层面得到印证（来源：AI 与 ADHD 的专注力）：

- **生成核心与调度层**：两者都拥有强大的生成能力，但缺乏可靠的执行调度层，导致输出不稳定。
- **无状态与外部记忆**：ADHD 的工作记忆缺陷与 LLM 的无状态性同构，都需要外部记忆系统（如第二大脑、向量数据库）来维持连续性。
- **上下文工程**：ADHD 易被环境带偏，LLM 易被上下文膨胀干扰，都需要主动设计“当下注意什么”的工程方案。

因此，AI 帮助 ADHD 的本质，并非让 AI 替你做所有事，而是**给生成核心套上 harness（外部执行功能层）**。就像 LLM agent 通过 harness 调用工具、管理文件、处理长对话（来源：What is an agent harness in the context of large-language models?），ADHD 大脑也需要外部工具来补偿执行功能缺陷。

## 实测：ChatGPT 补上了哪一层执行功能？

让我们用几个真实工具来检验：

### 1. 任务启动：Focusmate 与 Goblin Tools

ADHD 大脑在任务启动上存在困难。Focusmate 通过虚拟身体在场提供实时问责，弥补内在多巴胺调节不足（来源：Focusmate）。这对应 LLM agent 中的“计划-执行”分离模式——外部结构强制启动。

Goblin Tools 则“将模糊的工作分解为更小的行动”（来源：工具使用与认知卸载），降低启动门槛。这对应 agent 的“多步推理+工具调用”模式。

### 2. 时间感知：RescueTime

ADHD 常伴随时间盲。RescueTime 通过自动记录时间使用，减轻工作记忆负担，帮助用户识别注意力漂移模式（来源：RescueTime）。这对应 LLM agent 中的日志和监控系统——外部系统记录状态，减少不确定性。

### 3. 注意力维持：Brain.fm

Brain.fm 利用 AI 生成音乐，通过神经锁相技术帮助用户进入专注状态（来源：Brain.fm）。这对应 LLM 中的“温度控制”——通过外部信号调节系统状态，降低随机性。

但请注意：Brain.fm 针对 ADHD 的效果缺乏独立临床研究支持（来源：Brain.fm）。这提醒我们，工具的证据强度不一。

## 核心观点：脚手架 vs 拐杖——边界在哪？

AI 工具作为外部执行功能层，理论上应是“脚手架”——暂时辅助，最终帮助用户建立内在能力。但实际使用中，它可能沦为“拐杖”——形成永久依赖，导致内在执行功能进一步退化。

这种矛盾在多个资料中反复出现（来源：矛盾与存疑）：

- **依赖风险**：过度依赖 AI 工具可能削弱内在时间感知能力、任务启动能力。
- **个体差异**：同一工具对不同 ADHD 亚型效果迥异，缺乏个性化适配研究。
- **证据质量**：多数工具仅有用户报告或小样本研究，缺乏大规模随机对照试验。

因此，我的核心判断是：**ChatGPT 之于 ADHD，就像上下文工程之于 LLM——它补上了执行调度层，但前提是精准对应你的特定短板，并且你始终保持“主导权”**。

具体来说，AI 工具应当：

1. **精准匹配短板**：如果你的问题是任务启动，Focusmate 可能比 Brain.fm 更有效；如果你的问题是时间盲，RescueTime 比 Goblin Tools 更直接。
2. **避免认知开销**：工具应“插入现有堆栈”，不增加额外负担（来源：工具使用与认知卸载）。
3. **保持主动控制**：始终由你决定何时使用、如何使用，而非被工具牵着走。

1. **用 ChatGPT 做“任务分解”**：下次面对模糊任务时，直接输入“帮我将[任务]分解为 3-5 个可执行步骤”，然后只做第一步。这对应 Goblin Tools 的模式，但更灵活。

2. **设置“上下文锚点”**：在 ChatGPT 对话中，如果发现它跑偏，立即输入“回到原问题：我们最初的目标是[目标]”。这对应 LLM 的上下文工程——主动重置注意力。

# 为什么用 Reclaim.ai 治 ADHD 的情绪失调，和给 agent 套 会褪去的脚手架 是一回事？

> Saner 篇讲了「情绪失调的负载触发型」:很多脾气其实是内存满了。这一篇讲另一个触发器,它藏在日历里。ADHD 情绪失调有一条被反复观察到的路径:**冲动性承诺→日程超载→长期赶工→挫败与怨气→对人或对己爆发**。注意源头:不是情绪出了问题,是「答应太多」出了问题——冲动性(诊断标准里明确的那一条)让「好啊没问题」脱口而出,时间盲让代价在两周后才显形。Reclaim.ai 这类自动日历工具在这条链的中游动手;而它作为脚手架,「褪去」的目标格外清晰:**它替你说的那些「不」,最终要由你自己说出口。**

先看它怎么撑。Reclaim 的核心是自动防御时间:你声明习惯与优先级,它在日历上占位、护住、被挤压时自动重排。对上面那条情绪链,它的作用点是**把「拒绝」从实时决策变成系统默认**——有人约你周三下午,日历显示忙,你不需要当场调用最薄弱的那块能力(在社交压力下说不),系统已经替你说了。**这是关键机制:它不教你拒绝,它先让拒绝发生**——而「拒绝发生后天没塌」的体验反复积累,恰恰是学会拒绝的第一课。工程对应:新 agent 的护栏先强制执行边界(危险调用直接拦截),让系统在安全区里积累行为数据,而不是指望它天生会自律。

再看「褪去」的路径,这条同构在这里最漂亮:**教育心理学的脚手架三阶段——示范(系统替你做)、协助(系统提示你做)、独立(你自己做)——正好映射到这类工具的使用演化**:初期,日历硬性挡掉超载(示范);中期,你开始在答应之前主动看日历、引用它说「我那天排满了」(协助——注意,说「不」的主语已经换成你,日历只是你的证据);成熟期,你在没有日历撑腰的场合也能说「这周不行」(独立)。**脚手架没有消失,它内化成了直觉。**

误用两种:

**误用一:把系统当挡箭牌,永远停在「示范」阶段。** 每次拒绝都推给日历(「不是我不想,是排不开」)——**边界一直由系统执行,你自己的拒绝肌肉零训练**;某天工具不在(私人场合、家人的请求),你瞬间退回全盘接受。判据:你能不引用日历、只用「我不想接这个」完成一次拒绝吗?不能,就该开始中期训练——**先在最安全的关系里,练习不带证据的「不」**。

**误用二:用系统结构掩盖承诺总量问题(脚手架撑住了一个不该撑的建筑)。** Reclaim 把超载日程排得井井有条——**排得下≠该接下**:优化让 120% 的负载勉强运转,也让「负载本来就不该 120%」这个事实永远不被面对,怨气照旧积累,只是换成了对着精美日历的怨气。工程对应:护栏兜住了 agent 的错误率,掩盖了任务本身超出模型能力的事实。修法:每月看一次总量——**日历需要「见缝插针」的频率,就是承诺过量的直接读数**;那时该做的是砍承诺,不是升级排程算法。

## 边界

同构强度 B 级:冲动性与 ADHD 的过度承诺有诊断标准层面的支撑,超载与情绪耗竭的关联有职业健康文献,脚手架渐撤是成熟理论;Reclaim.ai 本身无对照研究,「拒绝的三阶段内化」是教育心理学框架的应用推断。声明:长期无法拒绝若根植于深层的被拒恐惧或自我价值问题,那是值得和专业者工作的主题——日历给得了证据,给不了底气;持续的怨气与耗竭是预警信号,别只优化排程。

## 今天就能试的 3 件事

1. **数一数本周的「见缝插针」次数**:日历被挤压重排了几次?这是你的承诺超载读数。
2. **练一次「协助级」拒绝**:下次被约,主动看日历并亲口说出「那天排满了」——主语是你。
3. **挑一个安全对象,练一次裸拒绝**:不引用日历,只说「这周不行」——感受一下没有挡箭牌的「不」。

日历替你说「不」,是为了让你听见「不」被说出来的样子——**听熟了,轮到你自己说**。脚手架的尽头不是更聪明的排程,是一个不需要证据也能守住边界的你。

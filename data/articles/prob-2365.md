# ADHD 职场人如何用「给生成核心套 harness」的思路应对项目推进与汇报里的分心走神或超聚焦跑偏，丢失目标？

> 把 LLM 工程里的「长程任务里 agent 会目标漂移，需要重锚定 system prompt」迁移到ADHD 职场人的项目推进与汇报场景，双域证据作答。

## 问题：为什么你的项目总是从 A 跑到 C，却到不了 B？

你打开文档，准备推进季度汇报的核心章节。十分钟后，你发现自己正在研究某个无关的竞品细节——这是分心走神。或者，你一头扎进数据可视化，把图表调得美轮美奂，但汇报的逻辑主线早已断裂——这是超聚焦跑偏。ADHD 职场人最熟悉的困境莫过于此：不是没有产出，而是产出偏离了目标。

做 Agentic Harness 工程的工程师对此绝不陌生：你给 agent 一个长程任务，它前几步还沿着轨道走，几轮对话后就开始自由发挥——上下文膨胀，推理退化，最终输出一个与原始目标貌合神离的东西。你不得不重锚定 system prompt，把 agent 拉回来。

这两类困境共享同一个底层结构：一个强大的生成核心，加上一个缺失的执行调度层。

## 同构：ADHD 大脑与 LLM 都是「高产但缺调度层」的生成核心

ADHD 的核心困境并非智力不足，而是执行功能的失效——计划本和便签用了一周就崩溃，传统工具无法解决“时间盲”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。最新研究更揭示了深层平行：ADHD 患者呈现“强记忆、弱控制”的认知剖面，这与 LLM 惊人相似——工作记忆容量未必差，但自上而下的控制和调节能力不足（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

LLM 一侧同样如此。现代 AI 编码代理通过“复合 AI 系统架构”来弥补调度缺陷，包括“工作负载专用模型路由、分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这些技术本质上是在 LLM 外部搭建调度层，防止上下文膨胀和推理退化。

这种同构意味着：ADHD 职场人需要的不是更强的“生成能力”——你根本不缺创意和产出——而是一套能持续重锚定目标的外部 harness。

## 历史案例中的 harness 系统：曾国藩的「日课十二条」

晚清名臣曾国藩是典型的注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”。他的解法是“日课十二条”：黎明即起、读书不二、谨言、写日记反省……用最笨最稳的方法抵消自己的冲动。这套系统本质上是外部调度层：每天定时 re-grounding，用“读书不二”强制单任务，用日记进行事后审计。

这与 LLM harness 的结构完全同构：日课中的“黎明即起”对应定时重锚定（cron job 式的 system prompt 刷新），“读书不二”对应任务隔离（单一上下文窗口），“日记”对应日志与事后分析（trace 与 replay）。曾国藩不是靠意志力硬扛，而是靠一套外部脚手架来补偿调度缺陷。

## 现代工具如何构建你的职场 harness

基于同构原理，ADHD 职场人可以构建自己的“项目推进 harness”，核心是三类组件：

**1. 时间审计器：RescueTime**
RescueTime 自动追踪时间使用，生成可视化报告，并主动阻断分心网站。它直接补偿时间盲和工作记忆负担——你不需要记住自己做了什么，工具替你记（来源：Revolutionizing ADHD Management with AI Assistants）。这相当于 agent 的 trace 系统：记录每一步，让你在目标漂移后能回溯。

**2. 身体在场调度器：Focusmate**
Focusmate 通过虚拟身体在场提供实时同伴问责，弥补内在多巴胺调节不足（来源：AI Tools for Kids with ADHD: A Complete Guide for Parents...）。它的 AI 调度算法优化配对，本质上是一种外部调度器，帮你完成“启动”这个最难的执行功能。这对应 agent 中的“双代理架构”：一个代理负责执行，另一个代理负责监督和提醒。

**3. 状态调节器：Brain.fm**
Brain.fm 利用 AI 生成神经锁相音乐，帮助进入专注状态（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。它不直接解决目标漂移，但通过降低进入专注的门槛，减少分心走神的概率。这类似 agent 中的“自适应上下文压缩”——优化输入环境，防止无关信息干扰。

但这些工具只是组件，关键在于你如何组织它们。

## 核心观点：你的 harness 必须包含「重锚定」机制

本文的核心判断是：**ADHD 职场人的 harness 必须包含一个显式的、定时的重锚定机制，就像 agent 的 system prompt 刷新一样**。

为什么？因为 ADHD 大脑与 LLM 一样，在长程任务中会自然发生目标漂移——不是因为你不够努力，而是因为调度层在长时间运行后失效。你需要一个外部触发器，周期性地把你拉回原点。

具体做法：在项目启动时写一条“核心 system prompt”（例如：“本项目的目标是 X，当前阶段是 Y，今天必须完成 Z”），然后设置每 45 分钟一次的闹钟提醒自己重读它。这相当于给大脑一个“上下文压缩”操作，清空膨胀的无关信息，重新聚焦。

## 争议与局限：脚手架 vs 拐杖

然而，外部 harness 并非没有风险。拐杖与脚手架的区别在于：脚手架帮助使用者“建造”，但使用者仍需自己“攀登”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。如果工具替代了治疗或自我管理，可能造成依赖。

此外，现有工具的证据仍有限：Brain.fm 缺乏 ADHD 人群的独立临床研究；RescueTime 的自动分类可能不够精准；Focusmate 的效果依赖用户匹配质量（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout；ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。同构性本身也只是功能类比，并非经过验证的神经机制对应（来源：矛盾与存疑）。

因此，本文的建议是：把 harness 看作可调整的脚手架，而不是永久拐杖。定期评估哪些组件真正内化成了你的能力，哪些仍然需要外部支撑。

## 今天就能试的 3 条行动

1. **写你的“核心 system prompt”**：在开始任何项目前，用一句话写下目标，贴在屏幕边缘。每 45 分钟闹钟响起时，大声读一遍。
2. **用 RescueTime 做一次时间审计**：运行一周，查看你的“目标漂移时间”占比。不要评判，只是观察。
3. **预约一次 Focusmate 时段**：选一个你一直拖延的任务，用身体在场效应强制启动。

ADHD 大脑不是坏掉的大脑，它只是缺少一个外部调度层。套上 harness，你的生成核心就能持续产出你真正想要的东西。

# ADHD 程序员如何用「给生成核心套 harness」的思路应对在 IDE 与 agent 之间切换里的执行功能（计划/组织/启动）先天偏弱？

> 把 LLM 工程里的「agent 的编排/记忆/工具层就是「外部执行功能」」迁移到ADHD 程序员的在 IDE 与 agent 之间切换场景，双域证据作答。

## 问题：IDE 与 agent 之间的执行功能悬崖

你正在 IDE 里写一个复杂函数，突然想到应该让 AI agent 帮忙查一下某个 API 文档。你切换到聊天窗口，输入 prompt，等它回复——然后发现自己忘了刚才要改哪行代码。或者，你让 agent 生成了一段代码，粘贴回 IDE 后，发现它跑不通，但你已经记不清 agent 给出的上下文了。

这种在 IDE 与 agent 之间切换时的“断片”，不是意志力问题，而是执行功能（计划、组织、启动）先天偏弱的表现。ADHD 大脑的工作记忆像一块白板，一旦切换上下文，之前的信息就被擦除（来源：[[无状态与外部记忆]]）。而 LLM agent 本身也是无状态的——每次对话都是新的开始，除非你主动给它喂上下文。

这里有一个被忽略的同构：ADHD 大脑与 LLM agent 都是“高产但缺执行调度层的生成核心”（来源：[[ADHD 大脑与 LLM 的同构]]）。两者的解法在结构上完全相同——都需要一个外部执行功能层来承担调度、记忆和验证任务。

## 同构解法：给生成核心套 harness

在 LLM 工程中，agent harness 是编排、记忆、工具层的总称。它负责管理上下文、分解任务、调用工具、验证输出（来源：[[幻觉与验证循环]]）。没有 harness，LLM 会陷入上下文膨胀、推理退化、幻觉泛滥。ADHD 大脑也一样：没有外部脚手架，就会陷入任务碎片化、时间盲、启动困难。

历史上最典型的 ADHD harness 系统来自曾国藩。他 30 岁前“浮躁坐不住”，日记里天天骂自己“无恒”“浮躁”，读书“不能终一行”。他的解法是“日课十二条”：黎明即起、读书不二、谨言、写日记反省。这套系统本质上是一个外部调度器——每天固定时间执行固定动作，用最笨最稳的方法抵消冲动。用工程术语说，这就是一个“定时 re-grounding”机制，类似于 agent harness 中的定期上下文刷新和验证循环。

另一位是张謇，41 岁中状元后辞官办实业，一生创办 20 多家企业、300 多所学校。他的“父教育母实业”就是一套外部工具层：用实业救国这个宏大目标作为长期记忆锚点，用日记反省作为每日调试。他的成就不是靠天生的执行功能，而是靠一个精心设计的外部系统。

## 具体怎么套：从工具到系统

现代 ADHD 程序员可以借鉴这一思路，但用更轻量的工具实现。关键是**不要只用一个工具**，而要构建一个多层 harness：

### 第一层：任务分解层（Goblin Tools）

Goblin Tools 的 Magic ToDo 能把“清理厨房”分解成 15 个小步骤，用户可调节“辣度”控制粒度（来源：[[Goblin Tools]]）。对程序员来说，这意味着把“实现用户认证模块”分解成“设计数据库表→写注册接口→写登录接口→测试→部署”。这一步解决了任务启动困难——小步骤比大任务更容易开始。

### 第二层：记忆层（Saner.AI）

Saner.AI 专注于知识回忆和本地记忆，减少搜索循环和标签切换（来源：[[Saner.AI]]）。它相当于一个外部工作记忆，让你在 IDE 和 agent 之间切换时，能快速找回之前的信息。它的通用收件箱能从邮件、文档、日历中提取待办事项，自动组织（来源：[[Saner.AI]]）。这对应 LLM harness 中的记忆系统——向量数据库或缓存。

### 第三层：调度层（Motion）

Motion 自动根据任务优先级、截止日期和可用时间创建每日计划，并动态调整（来源：[[Motion]]）。它消除了“下一步该做什么”的决策负担，直接对抗时间盲和启动困难。对应 LLM harness 中的规划工件和结构化工作流。

## 脚手架 vs 拐杖：边界在哪里？

这是最关键的争议。AI 工具可能成为永久拐杖，导致能力退化（来源：[[拐杖与脚手架]]）。判断标准很简单：**脚手架是可以拆卸的**。如果你依赖一个工具到离开它就无法工作，它就是拐杖；如果你能逐渐减少对它的依赖，它就是脚手架。

曾国藩的日课十二条是一套可拆卸的脚手架——他晚年已经内化了大部分习惯，不再需要每天写日记提醒自己。而一个永远需要 Motion 才能规划日程的人，可能正在使用拐杖。

另一个争议是超聚焦的引导问题。一些 ADHD 用户反馈强行打断（如番茄钟）反而有效，而柔性引导可能不够强力（来源：[[超聚焦]]）。这提醒我们：没有万能方案，必须根据个人特质调整 harness 的干预强度。

## 今天就能试的 3 件事

1. **在 IDE 和 agent 之间加一个“上下文暂存区”**：每次切换前，用 Saner.AI 或 Obsidian 记录当前代码的关键变量、函数名和下一步意图。这相当于给无状态的 LLM 和自己一个外部记忆。
2. **用 Goblin Tools 分解一个你拖延了一周的任务**：把“写周报”分解成 5 步，每完成一步打勾。小胜利能触发多巴胺释放，降低启动阻力。
3. **给自己设一个“日课”**：每天固定时间（比如上午 9 点）用 Motion 或纸质计划表规划当天最重要的 3 件事。坚持一周，观察执行功能的变化。

## 局限与诚实

本文的核心论点——ADHD 大脑与 LLM 同构——目前仍是类比推理，缺乏神经科学或计算机科学的直接证据（来源：[[矛盾与存疑]]）。工具的有效性也缺乏大规模随机对照试验（来源：[[矛盾与存疑]]）。但即使作为启发式框架，它已经帮助许多 ADHD 程序员重新理解自己的问题：不是“脑子坏了”，而是需要一个更好的 harness。

最后，记住曾国藩的一句话：“天下事，有所激有所逼而成者居其半。” harness 就是那个“逼”你的外部系统。它不是天生的，但可以后天搭建。

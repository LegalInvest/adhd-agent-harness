# ADHD 自由职业者视角：为什么「治好 ADHD 的工作记忆容量小、边做边忘」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 无跨会话状态，靠外部记忆/向量库续命——同一套 harness 思路，两边都成立。

## 同一个崩溃现场：做到一半，主线丢了

自由职业者最怕的不是没灵感，而是灵感做到一半，突然想不起来「我原本要干嘛」。对 ADHD 人群来说，这是工作记忆容量小、目标漂移的日常：注意力被新刺激拉走，或陷进某个细节超聚焦，原来的任务被清空（来源：重锚定与目标漂移）。

对做 Agentic Harness 的工程师来说，这个场景一点也不陌生：LLM 没有跨会话状态，上下文窗口一满、任务链一长，模型就会开始偏离目标、重复循环、甚至调用不该调用的工具。两边都是同一个画面——一个高产的生成核心，在缺少外部调度层时，自己把主线搞丢了（来源：ADHD 的 AI 工具全景；人在回路与身体在场）。

所以「治好 ADHD 的边做边忘」和「让 LLM 不跑飞」，不是两个领域的心灵鸡汤，而是同一道工程题：怎么给「高产生成但不可靠执行」的核心，装上可依赖的外部 harness。

## 同构诊断：生成核心 vs. 缺失的调度层

ADHD 大脑和 LLM 的相似，不是比喻，而是结构层面的对应。

ADHD 侧：前额叶执行功能受损，计划、抑制分心、维持目标导向都靠不住；工作记忆容量不稳定，信息在任务切换中丢失；多巴胺调节异常让注意力被新颖刺激吸走（来源：ADHD 的 AI 工具全景；重锚定与目标漂移）。

LLM/agent 侧：模型本身是强大的生成核心，但没有内置执行调度层，容易偏离目标、产生幻觉或无限循环；上下文窗口有限，通常也没有真正的跨会话持久状态，必须依赖外部记忆系统和人在回路治理（来源：ADHD 的 AI 工具全景；人在回路与身体在场）。

简言之：两边都是**想法很多、执行很脆**。ADHD 不缺创意，LLM 也不缺生成能力，缺的都是一个稳定的外部执行层。

## 孔子的「礼」就是一套两千年的 Harness

孔子的 ADHD 特质在资料里写得很具体：一米九、精力旺盛、周游列国十四年坐不住；冲动爱骂人，见南子急得对天发誓；听音乐能超专注到三月不知肉味，对种地却毫无耐心；思维跳跃，《论语》全是场景化语录，没有系统著作。他一生最大的工程，是用外在的「礼」来约束内在的冲动：「克己复礼」是核心边界，「吾日三省吾身」是每日 re-grounding，直到七十岁才做到「从心所欲不逾矩」（来源：人物案例）。

孔子的 harness 和今天的 LLM/agent harness 完全同构：
- **「礼」= system prompt / 护栏**：预先写好行为边界，降低实时自控的成本；
- **「三省吾身」= 定时 checkpoint / re-grounding**：周期性地把目标重新拉回来，对抗漂移；
- **周游列国的随侍弟子 = 外部记忆/调度器**：重要信息不必全存在大脑里，而是靠外部结构承载。

明朝的于谦也类似：性格刚直、冲动敢言，土木堡之变后力排众议坚守北京。他的 harness 是儒家气节 + 严格操练军队 + 日夜防务准备，把「粉骨碎身浑不怕」的价值观变成可执行的行为准则（来源：人物案例）。这对应到 LLM 里，就是**价值观对齐（value alignment）+ 工具调用限制 + 实时遥测**——不是让模型更聪明，而是让模型在约束下不出格。

## 工具层：两边都在做同一件事——把记忆和约束外化

ADHD 侧已经有一批工具在实践这套 harness 思路。

**Goblin Tools** 的 Magic ToDo 把「清理厨房」这种模糊目标自动拆解成可执行的子任务，用户还能调节「辣度」控制粒度（来源：Goblin Tools）。这对应 LLM 侧的任务分解与规划模块：把复杂目标拆成步骤，降低每一步的认知/计算负荷。

**Saner.AI** 走外部记忆路线：通过 Personal AI Skai 做自动笔记组织、语义搜索和语音捕获，目标是减少 ADHD 用户常见的「搜索循环」和标签切换（来源：Saner.AI）。这和 LLM 的向量库、检索增强生成（RAG）是同一套思路——把跨会话的持久记忆从模型内部搬到外部可检索的结构里。

**Lex** 的核心逻辑是「单一指令触发多步骤任务」，让用户用低启动成本触发复杂流程，实时适应大脑的工作方式（来源：Lex）。对 LLM 工程师来说，这相当于一个入口简洁、后端由 agent scaffolding 支撑的 agent：用户只发一个 intent，harness 负责规划、调用工具、维持状态。

**Focusmate** 则提供虚拟身体在场（body doubling），用社交 accountability 帮 ADHD 启动并维持任务（来源：人在回路与身体在场）。在 LLM harness 里，对应的是 **human-in-the-loop**：设置人工检查点、让模型在关键步骤暂停并请求确认，避免自主执行跑偏（来源：人在回路与身体在场）。

## 脚手架 vs. 拐杖：边界在哪里？

这套同构框架很锋利，但也有局限，不能神化。

首先，「ADHD 大脑与 LLM 同构」本身是一个**理论类比**，目前缺乏严格的实证基础，wiki 里已明确把它列为「矛盾与存疑」（来源：矛盾与存疑）。我们用它来做思考工具，但不能当成科学结论去卖。

其次，很多工具宣称的效果证据不足。例如 Motion 和 Brain.fm 在 ADHD 人群中缺乏独立临床验证，却被一些主题页直接推荐，有夸大之嫌；Lex 目前也缺乏直接研究证据（来源：矛盾与存疑；Lex）。

第三，外部 harness 容易从「脚手架」变成「拐杖」。好的脚手架是帮你逐步建立内部执行能力，而拐杖是让你越来越依赖外部系统。如果 AI 工具本身增加了信息切换、标签管理和设置提醒的负担，反而会提高认知负荷（来源：矛盾与存疑）。

最后，AI 的「身体在场」不能简单替代真实人际互动。Focusmate 的虚拟共作和 human-in-the-loop 的暂停确认各有价值，但把 AI 完全等同于「他人在场」仍然存疑（来源：矛盾与存疑）。

## 今天就能试的四条行动

1. **把「记忆」外化，而不是在脑子里硬撑**。开始一个任务前，先用 Saner.AI 或任何笔记工具把当前目标、下一步动作、相关上下文写进一个固定入口，减少任务切换时的信息丢失。

2. **把模糊目标交给拆解器**。遇到「想写一篇文章」或「要整理发票」这种大词，先丢进 Goblin Tools 的 Magic ToDo，让 AI 把它拆成 3–5 个具体的下一步，降低启动门槛。

3. **给你的 LLM/agent 也写一套「礼」**。像孔子的 system prompt 一样，给 agent 预设行为边界：目标描述、工具调用上限、token 预算、必须人工确认的节点。不要让模型在每个步骤都自由发挥。

4. **设一个 re-grounding 节奏**。每工作 25–45 分钟，用一句话问自己：「我现在做的这一步，服务于哪个原始目标？」对 LLM 而言，就是在长任务链里插入状态检查点和目标重述；对人而言，就是「三省吾身」的现代版。

## 结语：你不是在「治病」，你是在设计系统

ADHD 自由职业者常把「边做边忘」当成道德缺陷，LLM 工程师也常把「模型跑飞」当成模型不够强。但两边的真相更接近：核心已经足够好，缺的是一个可靠的外部执行层。把问题从「我/模型不够好」重定义为「系统需要一套 harness」，我们就能从自责和调参的焦虑里抽身，去设计真正可依赖的脚手架。

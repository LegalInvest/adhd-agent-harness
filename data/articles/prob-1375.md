# Todoist 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Todoist：它到底补上了哪一层执行功能？

你打开 Todoist，列了十几条任务，然后关掉它，什么都没做。这不是你的错——你把它用成了备忘录，而不是调度层。

ADHD 大脑与 LLM 共享同一个核心矛盾：两者都是强大的生成核心，但缺乏可靠的内置调度层。LLM 需要 planner-executor 循环来分解长任务、逐步执行；ADHD 大脑同样需要外部脚手架来补偿任务启动困难和时间盲。Todoist 的问题在于，多数人只用了它的“executor”部分（记录任务），而跳过了“planner”部分（拆解与排序）。

## 同构：ADHD 大脑与 LLM 共享的“调度层缺失”

认知神经科学的最新研究揭示了一个惊人的平行：Transformer 模型的自注意力机制在训练后，会发展出模仿前额叶-纹状体门控的操作——这正是人类执行功能的核心神经机制（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。换句话说，LLM 和 ADHD 大脑在底层计算结构上是同构的：两者都是高产的生成器，但都需要外部调度层来防止输出脱轨。

ADHD 侧的证据同样清晰：ADHD 患者在工作记忆任务中易受干扰，而 LLM 表现出类似的干扰特征——随记忆负荷增加表现下降，受近因效应影响（来源：Human-like Working Memory Interference in Large Language Models）。无外部结构时，两者都难以自主组织任务（来源：Working Memory Identifies Reasoning Limits in Language Models）。

这正是 Todoist 等工具的价值所在——它们充当了外部执行功能层，补偿 ADHD 在时间盲、任务启动和工作记忆上的缺陷（来源：AI 与 ADHD 的时间管理）。但工具只是脚手架，不是拐杖。

## 人物案例：孔子的“克己复礼”与 LLM 的提示工程

孔子是 ADHD 大脑的经典案例：精力旺盛、冲动爱骂人、思维跳跃、对感兴趣的事超专注（三月不知肉味），对俗事零耐心。他的 harness 系统是“克己复礼”——用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”）。这本质上是一个外部调度层：礼是 planner，每日反省是 executor 的 re-grounding 循环。

与 LLM 对照：LLM 的提示工程中，“系统提示”相当于礼——设定行为边界；“思维链”相当于每日反省——强制逐步推理。两者都是给生成核心套上 harness，防止冲动输出。Todoist 如果能像孔子的“礼”一样，不仅记录任务，还强制拆解步骤、规定执行顺序，它就从备忘录变成了真正的调度层。

## 工具实测：Todoist 到底补上了哪一层？

当前 ADHD 工具大致分两类：自适应规划器（Motion、Reclaim.ai）和视觉化时间线（Tiimo、Structured）。前者自动排程，消除决策负担（来源：Motion）；后者让时间流逝变得可感，应对时间盲（来源：Tiimo）。

Todoist 介于两者之间：它提供任务列表和优先级，但缺乏自动调度和视觉时间线。它的核心功能是“外部记忆”——持久存储任务上下文，减少每日重新回忆的摩擦（来源：无状态与外部记忆）。但问题在于：ADHD 用户需要的不仅是记忆，更是拆解。

Goblin Tools 的“任务分解”功能恰好补上了这一层：它用 AI 将“写报告”拆成“打开文档→列出大纲→写引言→……”等子步骤，这正是 LLM 的 planner-executor 循环在 ADHD 侧的对齐。如果 Todoist 能整合类似功能，它就能同时覆盖记忆和拆解两层。

## 核心判断：脚手架 vs 拐杖的边界

我的核心观点是：Todoist 的正确用法不是“记录所有任务”，而是“只记录被拆解后的最小可执行步骤”。一个任务若没有拆成 5 分钟内能启动的子任务，就不该进入 Todoist。否则，它只是增加了认知负荷，而非降低。

这与 LLM/Agent 工程完全同构：一个 planner 若输出过于抽象的子任务，executor 就会卡住。好的 agent 框架要求 planner 输出原子化步骤，且每个步骤有明确的完成条件。

但必须诚实指出局限：这种用法需要用户自己拆解，而拆解本身正是 ADHD 的痛点。AI 自动拆解（如 Goblin Tools）虽好，但可能造成依赖——若工具替代了自我管理，就变成了永久拐杖（来源：拐杖与脚手架）。目前缺乏长期研究评估这种依赖的后果，且多数工具缺乏独立临床验证（来源：矛盾与存疑）。

## 今天就能试的行动

1. **拆解测试**：选一个 Todoist 里搁置超过 3 天的任务，拆成“打开文件→写第一句话→保存”三个子步骤，然后只做第一步。
2. **限制条目**：Todoist 里同时存在的任务不超过 5 个。多出来的归档到“后备箱”项目，只保留可执行项。
3. **定时 re-grounding**：像孔子的“日省”一样，每天设定两个 5 分钟闹钟，检查 Todoist 里的任务是否还是最小步骤。与 LLM 的“定时提示”同构。
4. **组合使用**：用 Todoist 做外部记忆，搭配 Goblin Tools 的自动拆解（或手动拆解模板），形成 planner-executor 循环。

Todoist 不是魔法。它只是一个外部调度层——和孔子的“礼”、LLM 的提示工程一样，都是给高产生成核心套上的 harness。你是否用它，决定了它是脚手架还是拐杖。

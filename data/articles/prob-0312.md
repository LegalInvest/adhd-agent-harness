# 用 ChatGPT 给 ADHD 补上上下文工程，真的能像 agent 那样工作吗？

> 把 ChatGPT 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

## 引言：一个问题，两种困境

你坐在电脑前，打开 ChatGPT 准备写一份方案。三分钟后，你开始刷手机；十分钟后，你点开了一个无关的网页；半小时后，你发现自己连第一段都没写完。你骂自己“不专注”，然后再次打开 ChatGPT，希望它能帮你“拉回来”。

另一边，一位 AI 工程师正在调试一个 agent。她发现 LLM 总是被上下文窗口里的无关信息带偏，回答越来越发散。她需要精心设计 prompt、管理记忆、设置工具接口——她做的是“上下文工程”。

这两个场景，本质上是一个问题：**一个高产的生成核心，缺乏可靠的执行调度层**。ADHD 大脑如此，LLM 亦如此。用 ChatGPT 给 ADHD 补上上下文工程，真的能像 agent 那样工作吗？答案是：能，但有条件。

## 同构：ADHD 大脑与 LLM 共享的“短板”

ADHD 的核心困难在于执行功能不稳定，尤其是工作记忆“掉落”和任务启动困难（来源：Best AI Tools for ADHD Productivity in 2026）。最新研究揭示，Transformer 自注意力机制在长上下文下冲突解决能力会崩溃至随机水平，这与 ADHD 执行功能障碍的核心神经机制一致（来源：Deficient Executive Control in Transformer Attention）。两者都是“强记忆、弱控制”：ADHD 患者工作记忆容量可能正常甚至超常，但认知灵活性和注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

这意味着，ADHD 大脑和 LLM 需要的不是更强的生成能力，而是**一个外部的执行功能层**——在 AI 领域叫 harness，在 ADHD 领域叫“脚手架”。

## 人物案例：曾国藩的“日课十二条”与 LLM 的 harness 工程

晚清名臣曾国藩，30 岁前“浮躁坐不住”，日记里天天骂自己“无恒”“浮躁”。他的解决方案是“日课十二条”：黎明即起、读书不二、谨言、写日记反省……这套系统本质上是一个**外部执行功能层**：用固定的流程和外部记录来补偿内在的冲动与涣散。他“结硬寨打呆仗”，用最笨最稳的方法抵消自己的 ADHD 特质，最终成就了平定太平天国的伟业。

这套系统与 LLM 的 harness 工程高度同构：
- **日课↔定时 re-grounding**：曾国藩每天固定的“读书不二”相当于给 LLM 的 prompt 设定了“当前任务”的锚点，防止上下文漂移。
- **日记↔外部记忆系统**：曾国藩的日记就像 LLM 的向量数据库，记录历史、反省偏差，维持连续性。
- **“结硬寨”↔安全执行**：曾国藩用最保守的方法避免冲动决策，类似于 agent harness 中的沙箱和验证循环。

李鸿章作为曾国藩的学生，也继承了这一系统：他“外须和戎，内须变法”，用务实的态度对冲自己的冲动，每天写日记反省，最终成为洋务运动领袖。

## 工具实测：ChatGPT 补的是哪一块短板？

回到开头的问题：用 ChatGPT 给 ADHD 补上下文工程，真的能像 agent 那样工作吗？我们来看三个工具，它们分别对应 ADHD 的不同短板：

1. **Focusmate**：基于“身体在场效应”的在线问责服务。它用算法匹配一个伙伴，通过视频会议共同工作，提供外部结构和社会压力（来源：Revolutionizing ADHD Management with AI Assistants）。这相当于给 LLM 加了一个“实时监督器”——当 LLM 开始发散时，外部 agent 会把它拉回正轨。

2. **RescueTime**：自动时间追踪工具，记录用户在不同应用上的时间，帮助对抗“时间盲”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。它相当于 LLM 的“日志系统”——记录上下文变化，让用户看到“时间到底去了哪里”。

3. **Brain.fm**：用 AI 生成音乐，通过神经锁相技术影响大脑状态，帮助进入专注（来源：11 Best ADHD Productivity Apps for Fluctuating Energy）。这类似于给 LLM 的“初始 prompt”——在任务开始前设定一个“专注模式”的上下文。

这些工具的共同点：它们不是直接增强你的大脑，而是**外化执行功能**——把“记住要做什么”“感知时间”“启动任务”这些能力，交给外部系统。这和给 LLM 搭 harness 完全一样：把上下文管理、工具接口、验证循环交给外部脚手架。

## 核心观点：脚手架，不是拐杖

我的核心判断是：**用 ChatGPT 给 ADHD 补上下文工程，确实能像 agent 那样工作，但前提是把它当作脚手架，而不是拐杖。**

脚手架 vs 拐杖的边界在于：
- **脚手架**：你主动设计它、调整它，最终目标是让内在能力成长。曾国藩的日课是他自己定的，而且他每天都在反省改进。
- **拐杖**：你被动依赖它，离开它就无法行动。长期依赖 AI 工具可能导致内在执行功能进一步退化（来源：拐杖与脚手架的平衡）。

目前，这个边界尚不明确。缺乏纵向研究证明长期使用 AI 工具是否会导致能力退化（来源：矛盾与存疑）。此外，ADHD 的异质性很大：同一工具对不同亚型效果迥异（来源：矛盾与存疑）。而且，多数工具仅有用户报告或小样本研究，缺乏大规模随机对照试验（来源：矛盾与存疑）。

## 今天就能试的 3 件事

1. **用 ChatGPT 做“任务启动器”**：每次开始任务前，让 ChatGPT 帮你把任务分解成 3 个微步骤，并设定一个 5 分钟倒计时。这相当于给 LLM 的 prompt 加上“任务分解”和“时间锚点”。

2. **用 RescueTime 做“时间审计”**：安装 RescueTime，一周后看报告，找出你最常被带偏的“上下文窗口”。然后主动设计阻断规则。

3. **尝试 Focusmate 的一次 session**：预约一个 25 分钟的时段，体验“身体在场”带来的外部约束。这相当于给 LLM 加一个“实时监督 agent”。

## 结语

ADHD 大脑和 LLM 共享同一个困境：生成能力强，执行调度弱。解决方案也共享同一个思路：用外部系统来补偿。但请记住，曾国藩不是靠日课十二条变成“圣人”的，而是靠每天反省、持续改进。工具只是起点，真正的改变在于你如何用它。

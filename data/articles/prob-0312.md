# 用 ChatGPT 给 ADHD 补上上下文工程，真的能像 agent 那样工作吗？

> 把 ChatGPT 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

## 问题：注意力被环境带偏，ADHD 与 LLM 的同一困境

ADHD 人群最熟悉的体验是：明明想专心写报告，却因为手机震动、窗外声响甚至一个飘忽的念头，就彻底偏离轨道。这种“注意力容易被环境带偏”的痛点，并非意志力薄弱，而是执行功能中的**上下文控制**出了问题。

无独有偶，LLM（大语言模型）在长对话或复杂任务中也会“走神”：上下文窗口里塞入过多无关信息，模型就开始编造、遗漏关键指令。工程师们为此设计了**Agentic Harness**——通过结构化提示、外部记忆和注意力锚点，让 LLM 聚焦于当前最重要的上下文。

那么，用 ChatGPT 给 ADHD 补上类似的“上下文工程”，真的能像 agent 那样工作吗？

## 同构：高产但缺执行调度层的生成核心

ADHD 大脑与 LLM 在结构上高度同构：两者都是**高产但缺乏可靠执行调度层的生成核心**。ADHD 的创意发散、LLM 的文本生成，都依赖于强大的“生成”能力，但缺少一个能稳定控制“当下该关注什么”的调度层。

- **无状态与外部记忆**：ADHD 的工作记忆缺陷（容易忘记待办事项）与 LLM 的无状态性（对话结束后信息丢失）同构。两者都需要外部记忆系统来维持连续性，比如第二大脑或向量数据库。
- **上下文工程**：ADHD 易被环境带偏，LLM 易被上下文膨胀干扰。两者的解决方案是相同的：主动设计“当下注意什么”的工程方案。

因此，AI 工具帮助 ADHD 的本质，就是**给生成核心套上 harness（外部执行功能层）**。ChatGPT 作为通用对话模型，可以扮演这个 harness 的角色，但前提是它被正确地“上下文工程化”。

## 证据：ChatGPT 能否补上上下文工程？

### ADHD 侧：真实工具如何补偿执行功能

现有工具已经验证了外部执行功能层的有效性：

- **Focusmate** 利用虚拟身体在场效应，通过实时同伴问责提供外部结构，弥补内在多巴胺调节不足，显著提升任务启动和维持（来源：Focusmate）。
- **RescueTime** 自动追踪时间使用，减轻工作记忆负担，帮助识别注意力漂移模式，对抗时间盲（来源：RescueTime）。
- **Brain.fm** 通过 AI 生成的神经锁相音乐，直接调节大脑状态，帮助进入专注（来源：Brain.fm）。
- **Forest** 通过游戏化专注计时，提供外部时间锚点。

这些工具的共同点：它们不是增强内在能力，而是**在外部搭建脚手架**——提供结构、提醒、问责，让 ADHD 大脑能暂时绕过执行功能短板。

### LLM/Agent 侧：Agentic Harness 的工程实践

在 LLM 领域，Agentic Harness 的核心是上下文工程：

1. **结构化提示**：用明确的指令模板（如角色、步骤、输出格式）约束 LLM 的注意力，防止发散。
2. **外部记忆**：通过向量数据库或检索增强生成（RAG），让 LLM 能够访问长期上下文，避免窗口膨胀。
3. **注意力锚点**：在关键节点插入“暂停-反思”机制，让 LLM 重新聚焦于目标。

这些技术本质上与 ADHD 工具的设计同构：Focusmate 的定时问责相当于“暂停-反思”，RescueTime 的自动分类相当于结构化提示，Brain.fm 的神经锁相相当于注意力锚点。

### ChatGPT 作为外部执行层的实测

用 ChatGPT 给 ADHD 补上下文工程，可以这样操作：

- **提示工程**：设定一个“ADHD 助手”角色，要求它每次只给一个任务、每 25 分钟提醒一次、用简洁语言反馈。这相当于给 LLM 套上 harness。
- **外部记忆**：将待办事项、项目背景写入 ChatGPT 的“自定义指令”或记忆功能，避免每次重新解释。
- **注意力锚点**：让 ChatGPT 在对话中定期问“你现在在做什么？是否偏离了原本目标？”，模拟外部问责。

初步用户报告显示，这种方法能有效减少 ADHD 用户的“任务漂移”，但效果高度依赖提示工程的质量（来源：AI 与 ADHD 的专注力）。

## 核心判断：脚手架还是拐杖？

ChatGPT 可以像 agent 那样工作，但有一个关键边界：**脚手架 vs 拐杖**。

- **脚手架**：用户主动使用工具来补偿特定短板，并在使用中逐渐内化策略。例如，用 RescueTime 发现自己下午效率低，就主动调整工作安排。
- **拐杖**：用户依赖工具完成一切，内在执行功能进一步退化。例如，没有 ChatGPT 提醒就无法开始任何任务。

当前证据表明，多数 AI 工具被设计为脚手架，但实际使用中容易滑向拐杖（来源：矛盾与存疑）。例如，过度依赖 Focusmate 可能导致用户无法独自工作；持续使用 Brain.fm 可能削弱自然专注能力。

因此，ChatGPT 作为外部上下文工程工具，**能有效工作，但必须谨慎使用**。它的价值在于“补上”短板，而非“替代”内在能力。

## 争议与局限

1. **证据不足**：多数工具仅有用户报告或小样本研究，缺乏大规模随机对照试验。Brain.fm 的 ADHD 效果缺乏独立临床研究支持（来源：Brain.fm）。
2. **个体差异**：ADHD 异质性大，同一工具对不同亚型效果迥异。例如，Focusmate 对社交焦虑者可能适得其反。
3. **同构的局限性**：ADHD 大脑与 LLM 的同构是隐喻而非严格等价。LLM 的“注意力机制”是数学计算，而人类注意力涉及情绪、多巴胺、生理状态，不能简单类比。

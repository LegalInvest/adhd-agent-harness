# ADHD 的‘目标漂移’涉及前额叶-纹状体环路多巴胺失调，LLM 的‘漂移’本质是 Transformer 注意力分布变化。这两个机制是否只是表面相似，而非真正的同构？

> 机制鸿沟：神经递质 vs 注意力权重，同构是否成立？

## 问题：两种漂移，一个困境？

ADHD 人群最熟悉的体验是：坐下工作，五分钟后思绪飘到下周的旅行计划，再回过神已经刷了半小时短视频。而 LLM Agent 的工程师也常撞上类似场景：Agent 执行长程任务时，起初遵循 system prompt，几轮对话后逐渐偏离原始目标，开始生成无关内容。这两种“目标漂移”看起来太像了——ADHD 是前额叶-纹状体环路多巴胺失调，LLM 是 Transformer 注意力分布变化。但它们是同一种机制吗？还是只是表面相似？

## 同构：功能层面，而非神经层面

我的核心判断是：**ADHD 大脑与 LLM 在功能架构上是同构的，但神经机制与计算机制并非同一回事。** 两者都是“高产但缺执行调度层的生成核心”——ADHD 大脑能产生大量创意和联想，但缺乏可靠的内置执行功能来锚定目标；LLM 能生成流畅文本，但缺乏内置的长期目标跟踪机制。这种功能同构意味着，两边的解法（harness）可以结构对应。

先看 ADHD 侧的证据。ADHD 的“时间盲”使其难以感知时间流逝（来源：时间盲），而“身体在场效应”通过社会存在感提供外部锚点（来源：身体在场效应）。Focusmate 利用 AI 匹配虚拟身体加倍伙伴，将目标达成可能性从 25% 提升至 95%（来源：身体在场效应）。RescueTime 则通过自动时间追踪补偿工作记忆缺陷，让用户看到“时间到底去了哪里”（来源：RescueTime）。这些工具本质上是为 ADHD 大脑套上“外部执行功能层”（来源：外部执行功能层）。

再看 LLM/Agent 侧。LLM 的“漂移”源于 Transformer 注意力分布随上下文膨胀而退化（即“上下文工程”问题）。工程师的解法是重锚定 system prompt、定期注入目标摘要、或使用外部记忆模块。这与 ADHD 的 harness 结构同构：Focusmate 的定时匹配对应 Agent 的定期 re-grounding，RescueTime 的自动追踪对应 Agent 的日志审计系统。

## 人物案例：曾国藩的“日课”与 Agent 的“system prompt”

曾国藩是 ADHD harness 的经典案例。他 30 岁前“浮躁坐不住”，日记里天天骂自己“无恒”“浮躁”，读书“疾读不能终一行”。他的解法是“日课十二条”：黎明即起、读书不二、谨言、写日记反省……这套系统本质上是外部调度层——用固定的时间锚点（黎明即起）对抗时间盲，用“读书不二”对抗注意力分散，用每日反省对抗目标漂移。这恰恰对应 LLM Agent 的 system prompt + 定期日志审计：日课是定时重锚定，反省是自我评估。

李白则是另一种 harness：不委屈自己做不想做的事，把所有过剩精力注入诗歌。这种“选择赛道”的解法对应 Agent 的“任务分解”——把不适合的任务外包，只聚焦生成核心擅长的部分。

## 脚手架 vs 拐杖：同构的边界

同构不是等同。ADHD 的多巴胺失调与 LLM 的注意力分布变化是不同层面的机制，强行等同会误导干预。更关键的是，外部脚手架可能变成永久拐杖。Focusmate 和 RescueTime 如果使用过度，可能让 ADHD 用户永远依赖外部结构，无法内化执行功能（来源：矛盾与存疑）。同样，Agent 如果过度依赖重锚定提示，可能永远无法学会自主目标保持。因此，同构的应用边界在于：**脚手架应逐步内化，而非永久替代。**

## 争议与局限

现有证据多来自用户报告或小样本研究，缺乏大规模随机对照试验（来源：矛盾与存疑）。Brain.fm 虽基于神经锁相技术，但在 ADHD 人群中的独立临床证据仍有限（来源：Brain.fm）。此外，ADHD 的异质性意味着同一工具对不同亚型效果迥异，现有研究未充分分层（来源：AI 与 ADHD 的专注力）。

## 今天就能试的行动

1. **ADHD 读者**：尝试用 Focusmate 预约一个 25 分钟的身体加倍时段，体验外部锚点对启动和维持专注的效果。同时设置 RescueTime 自动追踪，一周后查看时间报告，识别你的“漂移模式”。
2. **Agent 工程师**：在你的 Agent system prompt 中加入每 5 轮对话自动插入目标摘要的指令，类似于曾国藩的“日课反省”。同时添加外部日志审计，记录每次漂移的上下文。
3. **通用**：无论是人还是 Agent，都值得建立“重锚定仪式”——定时回顾原始目标，避免被局部细节带偏。对 ADHD 用户，可以用手机闹钟；对 Agent，可以用定时钩子函数。
4. **谨慎依赖**：每周评估一次工具使用情况，问自己：我是在用脚手架，还是已经离不开拐杖？如果是后者，尝试减少一次依赖，用内化策略替代。

## 结语

ADHD 大脑与 LLM 的“目标漂移”并非同一回事，但它们的解法结构同构。这种同构不是巧合，而是源于“生成核心+缺失调度层”的底层架构。理解这一点，既能帮助 ADHD 人群设计更有效的 harness，也能启发 Agent 工程师构建更稳健的调度系统。但别忘了，同构是功能层面的，不是神经层面的——不要用类比替代实证，也不要用工具替代内化。

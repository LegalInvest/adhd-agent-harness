# ADHD 职场人视角：为什么「治好 ADHD 的冲动下结论、想当然」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 自信地编造（幻觉），靠验证与自我批判兜底——同一套 harness 思路，两边都成立。

你有没有过这样的经历：会议刚听到一半，脑子里已经跳出结论，脱口而出后才发现自己漏掉了关键信息？或者，你明明知道某个任务很重要，却怎么也启动不了，最后在 deadline 前手忙脚乱？如果你是 ADHD 职场人，这些场景大概不陌生。

与此同时，如果你在做 LLM Agent 工程，你一定见过另一种“自信”——模型流畅地编造出一个不存在的引用，或者在一个简单的逻辑推理中跑偏，然后一本正经地输出。你给它加上 ReAct、Chain-of-Thought 验证循环，它才勉强“老实”一点。

这两件事听起来毫不相干，但如果你把 ADHD 大脑和 LLM 放在一起看，就会发现一个惊人的同构：**两者都是强大的生成核心，但缺乏可靠的内置执行调度层。** 所以，治好 ADHD 的冲动下结论、想当然，和让 LLM 不跑飞，本质上是在解决同一道工程题——如何为生成核心套上一个外部验证与自我批判的 harness。

## 同构：生成核心 vs 调度层

ADHD 大脑的核心优势是发散、联想、快速生成——一个想法可以瞬间跳到另一个，创意层出不穷。但问题在于，它缺乏一个稳定的“调度层”来刹车、验证、排序。神经科学研究显示，ADHD 患者前额叶执行功能网络激活不足，导致工作记忆不稳定、任务启动困难、时间感知扭曲（来源：ADHD × AI 的科学与研究前沿）。这就像一个没有操作系统的 CPU，运算能力再强，也容易死机。

LLM 的处境类似。它本质上是下一个 token 的预测器，在给定上下文后，它会“自信”地生成最可能的序列，但不会主动检查自己是否在编造。这就是幻觉的根源。工程师们发现，必须给 LLM 加上外部脚手架——比如 ReAct 框架让模型先思考再行动，Chain-of-Thought 强迫它逐步推理，或者用验证器检查输出——才能减少错误（来源：ADHD × AI 的科学与研究前沿）。

这个脚手架，就是外部调度层。

## 真实人物的 harness：毛泽东与王传福

毛泽东是典型的 ADHD 大脑。他小时候是问题儿童，和父亲吵架，爱读闲书；一辈子读书不辍，行军路上也带着书；思维极度跳跃，讲话充满比喻，四渡赤水、抗美援朝都是冲动敢赌的险棋。但他创造了一套强大的个人 harness 系统：**批评与自我批评**——定期让别人提自己的意见，相当于外部验证器；**调查研究**——没有调查就没有发言权，用事实数据对冲直觉；**民主集中制**——用集体决策制衡个人冲动，类似 LLM 的多数投票或 ensemble 方法。这套系统让他的“生成核心”在革命和建设中保持相对稳定，而不是随意跑偏。

王传福是另一个例子。他 All in 电动车，垂直整合所有技术，所有人都觉得他疯了。但他的 harness 是**工程师文化**和**技术为王**——用严格的实验数据和工程验证来检验每一个决策，而不是凭感觉。这就像给 LLM 加上一个“事实性检查”模块，每输出一个结论，都必须通过实验验证。

这两套 harness 的核心逻辑，和 LLM 的验证循环完全同构：**先输出，再验证，用外部系统兜底。**

## 工程解法：验证循环的两种形式

在 ADHD 侧，已有不少 AI 工具在实践这种思路。**Goblin Tools** 的 Magic ToDo 能把模糊任务分解为具体步骤，相当于把“清理厨房”这种大指令拆成“洗碗、擦台面、拖地”等可执行步骤，降低启动门槛（来源：Goblin Tools）。**Motion** 自动排程并动态调整，当任务面临延期风险时提前警告，相当于一个外部调度器不断 re-grounding 你的计划（来源：Motion）。**Saner.AI** 通过本地记忆减少搜索循环，让你不用在多个标签页间切换，相当于一个外部记忆模块（来源：Saner.AI）。**Focusmate** 则通过虚拟身体加倍提供隐性问责，类似 human-in-the-loop 的监督（来源：身体在场效应）。

在 LLM 侧，工程解法如出一辙：ReAct 让模型在思考与行动之间循环验证；Chain-of-Thought 强迫逐步推理；验证器（verifier）对输出进行打分；RAG（检索增强生成）从外部知识库中拉取事实，减少幻觉。这些方法本质上都是在生成核心之外，构建一个“先输出，再检查”的循环。

## 脚手架 vs 拐杖：边界在哪里？

但这里有一个关键问题：这些外部系统是脚手架还是拐杖？ADHD 领域早有争议：长期依赖 AI 工具可能导致执行功能进一步退化（来源：矛盾与存疑）。同样，在 LLM 工程中，过度依赖验证循环也可能让模型本身的推理能力萎缩——如果每次犯错都有外部兜底，模型就不会学会自我纠错。

我的判断是：**脚手架和拐杖的边界在于“是否保留主动参与”。** 一个好的 harness 应该让你（或模型）在循环中保持主动——你可以选择接受验证结果，也可以选择 override。比如毛泽东的“批评与自我批评”，不是别人批评完就完事，而是自己也要反思。同样，LLM 的验证循环不应该完全替代模型推理，而是辅助它。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个让你拖延的任务**：把“写周报”丢进 Magic ToDo，调节“辣度”到你觉得合适的粒度，然后按步骤执行。
2. **设置一个“验证时刻”**：在做出重要决策前，强制自己用 5 分钟写下三个反方论点。这相当于给大脑加一个 Chain-of-Thought 步骤。
3. **尝试 Focusmate 一次**：预约一个 25 分钟的身体加倍 session，感受外部问责如何降低启动阻力。
4. **给 LLM 加一个验证器**：如果你在用 ChatGPT 写代码或事实性内容，要求它“请检查以上回答中是否有不准确的信息”，然后验证它自己的修正。

ADHD 不是缺陷，而是一种生成核心的配置。LLM 的幻觉也不是 bug，而是架构特性。两者的解法，都是给这个核心套上一个聪明的 harness。理解这一点，你就能从工程视角重新看待自己的大脑，而不是把它当成一个需要“修复”的错误。

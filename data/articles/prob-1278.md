# 用 ChatGPT 给 ADHD 补上外部执行功能层，真的能像 agent 那样工作吗？

> 把 ChatGPT 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

## 当大脑像 LLM 一样“失控”

你是否有过这样的体验：脑子里同时冒出十几个好点子，但一个都执行不下去；打开电脑准备写报告，却不知不觉刷了半小时手机；明明知道 deadline 就在明天，身体却像被钉在椅子上。这不是你懒，而是你的执行功能——计划、组织、启动——先天偏弱（[[执行功能]]）。就像一个大模型，生成能力强得惊人，但缺一个可靠的调度层来把输出落地。

与此同时，AI 工程师们正在做一件看起来毫不相干的事：给 LLM 搭“harness”——编排层、记忆层、工具层，让模型能调用外部工具、分解任务、保持上下文。他们发现，纯靠模型自身无法稳定完成复杂任务，必须靠外部脚手架（来源：What is an agent harness in the context of large-language models?）。

这两件事，其实是同一个问题的两面。ADHD 大脑和 LLM 在结构上同构：都是“高产但缺乏可靠执行调度层的生成核心”（[[ADHD 大脑与 LLM 的同构]]）。所以，用 ChatGPT 给 ADHD 补上外部执行功能层，本质上就是在给一个“失控的生成核心”套上 harness。它真的能像 agent 那样工作吗？答案是有条件地“能”——但前提是理解它补的是哪一块短板，以及如何避免它从脚手架变成拐杖。

## 外部执行功能层：ADHD 与 LLM 的共同解法

### 任务分解：Goblin Tools 与 function calling

ADHD 大脑面对“整理房间”这类模糊任务时，往往陷入瘫痪——不是不知道怎么做，而是不知道从哪里开始。传统规划方法“通常一周就失效”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），因为它们没有解决启动门槛问题。

Goblin Tools 的 Magic ToDo 功能正是为此而生：输入“整理房间”，AI 自动输出“捡起地板上的衣服”“擦桌子”等步骤，将压倒性的事情变成一系列不压倒性的事情（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...））。这相当于给大脑装了一个任务分解器。

在 LLM/agent 一侧，这对应着 function calling 和多步推理。Agent harness 通过将复杂任务拆解为工具调用，让模型一步步执行（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。两者共享同一个模式：**将模糊需求转化为可执行的原子步骤**。

### 工作记忆与外部记忆：Saner.AI 与向量数据库

ADHD 的工作记忆容量保留但控制弱，容易被干扰（[[工作记忆]]）。你刚想好要做什么，一个通知弹出来，就忘了。Saner.AI 通过本地记忆和快速检索，帮助用户减少“搜索循环和标签切换”（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar），相当于把大脑的临时缓存外挂到持久存储上。

LLM 同样面临无状态性问题：跨会话遗忘，上下文窗口膨胀导致推理退化（[[无状态与外部记忆]]）。解决方案是向量数据库或检索增强生成（RAG），把记忆存在外部，需要时再取回。**两者都依赖外部记忆系统来维持连续性**。

### 动态规划：Motion 与 agent 编排

ADHD 的“时间盲”让传统计划表形同虚设：你排好了日程，但一个意外就打乱了全天计划，然后彻底放弃。Motion 和 Tiimo 这类自适应规划器能“在日程被打乱时自动重新安排”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），消除“下一步该做什么”的决策负担。

在 agent 系统中，这对应动态编排：harness 根据任务状态和反馈实时调整下一步行动，而不是死板地执行预定义流程。**两者都需要一个外部调度器来动态管理优先级和顺序**。

## 脚手架还是拐杖？争议与边界

同构视角虽然有力，但必须诚实面对争议。最核心的矛盾是：外部执行功能层究竟是脚手架（最终可以撤掉）还是拐杖（永久依赖）？（[[拐杖与脚手架]]）

目前证据主要来自用户报告，缺乏大规模对照实验（[[矛盾与存疑]]）。理论上，过度依赖 AI 工具可能削弱内在执行功能，尤其是时间感知能力（[[时间盲]]）。一个 ADHD 用户如果永远依靠 Goblin Tools 分解任务，可能永远学不会自己分解。同样，一个 agent 系统如果过度依赖外部 harness，模型本身的推理能力可能退化。

但另一方面，对于某些 ADHD 个体——尤其是执行功能严重受损者——外部脚手架可能是唯一可行的方案。关键在于区分“补偿”与“替代”：工具应补偿短板，而非完全替代大脑的参与。例如，用 Motion 动态规划，但仍需自己输入任务和优先级；用 Goblin Tools 分解，但仍需自己执行每一步。

另一个争议是个体差异。ADHD 有注意力主导型和冲动主导型等亚型，LLM 也有不同的架构差异，同构的普适性尚未被充分验证（[[矛盾与存疑]]）。一个工具对“注意力涣散型”有效，对“冲动型”可能适得其反。

## 今天就能试的 3 个行动

1. **用 ChatGPT 做一次任务分解**：把一件让你拖延的事情（比如“写周报”）输入 ChatGPT，要求它分解成不超过 5 分钟的步骤。然后只做第一步。这相当于给大脑装一个临时 function calling 层。

2. **设置一个“外部记忆”系统**：用 Saner.AI 或简单的笔记应用，把临时想法、待办事项全部记下来，而不是靠大脑记住。关键是要“不增加认知开销”（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar），所以工具要足够简单。

3. **尝试一天自适应规划**：如果你经常被意外打乱计划，试试 Motion 或 Tiimo，让 AI 帮你动态调整日程。注意：初始设置需要输入所有任务，这对执行功能较弱的人可能是个挑战（[[Motion]]），可以从少量任务开始。

## 结语

ADHD 大脑与 LLM/agent 的同构，不是一个空洞的类比，而是一个可操作的设计框架。它告诉我们：问题不在于“生成核心”本身，而在于缺少外部调度层。用 ChatGPT 补上这一层，确实能像 agent harness 一样工作——但前提是，你清楚它补的是哪一块短板，并且始终把它当作脚手架，而不是拐杖。毕竟，最好的 harness 不是让你永远依赖它，而是让你最终能自己跑起来。

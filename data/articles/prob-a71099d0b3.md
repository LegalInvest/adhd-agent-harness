# 为什么用 Lex 治 ADHD 的时间盲，和给 agent 套 planner-executor 任务分解 是一回事？

> Lex 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

你有没有过这种体验：明明知道自己该做什么，但等回过神来，四十五分钟已经不见了。ADHD 患者称这为「时间盲」——不是懒，而是大脑对时间流逝的感知系统出了偏差。奇妙的是，正在做 Agentic Harness 的工程师会发现，LLM agent 在 multi-step 任务里掉链子的方式，和 ADHD 大脑几乎一模一样：它能生成、能联想、能创造，但一到「什么时候开始、下一步做什么」就卡住了。为什么用 Lex 治 ADHD 的时间盲，和给 agent 套 planner-executor 任务分解，会是同一回事？答案藏在「生成核心 + 外部调度层」这个同构结构里。

## 时间盲：ADHD 缺少的不是聪明，而是「调度层」

时间盲被描述为 ADHD 的「真实且危险」特征：患者「无法感知时间的流逝」，可能坐下后「真的感觉不到 45 分钟已经过去」（来源：Using an AI Assistant to Manage ADHD: A Practical Guide；Revolutionizing ADHD Management with AI Assistants）。它背后是执行功能、工作记忆和多巴胺调节的缺陷，导致「眼不见，心不烦」——任务一旦从视野里消失，就像没存在过一样（来源：9 Best AI Assistants for ADHD in 2026）。

这不是意志力问题，而是神经系统的调度层不够稳。ADHD 大脑往往拥有极高的联想和创造产能，却缺一个稳定的「驾驶座」来调控注意力、计划、组织和冲动控制。所以现代工具的方向不是「让人变正常」，而是把前额叶本应完成的调度功能，外化成可替换的模块。

Motion 用 AI 自动排程与动态调整，把「下一步该做什么」的决策从用户大脑中卸载；Reclaim.ai 则用防御性时间块保护深度工作与习惯，减少会议侵占带来的日程崩解（来源：11 Best ADHD Productivity Apps for Fluctuating Energy；The AI Powered SuperApp for Work）。Tiimo 和 Structured 更进一步，把抽象时间「实体化」为视觉时间块，让时间变得可触摸（来源：The 12 Best Apps for ADHD in 2026）。它们共同构成一个外部执行功能层：外部记忆、外部调度、外部启动提示。

## Agent 的「时间盲」：planner 与 executor 之间的裂缝

LLM 的处境与此高度同构。它能在一个上下文里生成高质量内容，却缺乏跨会话的持久状态、可靠的目标维持和多步调度能力。换句话说，LLM 也是一个高产生成核心，但缺一个可靠的执行调度层。这正是 agent 架构要补的课。

planner-executor 任务分解就是给 LLM 套上 harness：planner 负责把目标拆成可执行的步骤，executor 负责在正确的时间调用正确的工具，状态管理模块则弥补 LLM 无跨会话记忆的缺陷。这和 ADHD 工具在做的事结构相同：上下文工程限定「此刻该关注什么」，外部记忆系统补全工作记忆，调度器把任务推到眼前。

ADHD 的工作记忆不稳定，与 LLM 的无跨会话状态在功能上同构；两者都需要外部记忆系统来补全（来源：AI 与 ADHD 的时间管理）。Motion 的自动重新排程，对应 agent 的 replanning loop；Tiimo 的视觉时间线，对应 agent 的 timeline/state tracker；Reclaim.ai 的防御性时间块，对应 agent 的 resource guard。两边的工程直觉惊人地一致：别指望生成核心自己管家，给它一个 harness。

## 孔子的日课：一个古人的 harness 系统

这种 harness 思路并非新发明。孔子就是一个经典案例。他身高一米九，精力旺盛，周游列国十四年，冲动爱骂人，对音乐能「三月不知肉味」，对种地却毫无耐心；思维跳跃，《论语》全是场景化语录，没有系统著作。但孔子给自己的高产生成核心套上了一整套外部边界：以「克己复礼」为核心，用外在的「礼」作为行为边界，每日反省，「吾日三省吾身」，直到七十岁才「从心所欲不逾矩」。

孔子的「礼」与「日课」，本质上就是 ADHD  harness 的古代版本：把自律外化为可执行的规则，把反省当作定时 re-grounding，让生成核心在边界内运行。放到 LLM 的语境里，「礼」相当于系统提示词（system prompt）与行为约束，「日课」相当于定时状态检查与计划重校准。孔子的成就不是因为他战胜了冲动，而是因为他给冲动搭了一个足够稳的脚手架。

## 同构的 harness：从「礼」到 planner-executor

所以，Lex 治时间盲与 planner-executor 任务分解为什么是同一回事？因为 Lex 这类写作工具所做的，是把原本混沌的写作/思考过程外化为可执行的步骤：不是让你坐在空白页前靠意志力生成，而是把生成任务拆解、锚定在时间块里，提供外部记忆和结构。这和 Motion 把任务推到眼前、和 agent 把目标拆成 plan 再交给 executor，底层结构一致。

 ADHD 侧，工具用视觉化、自动排程、外部提醒来绕过时间盲和执行功能缺陷；LLM/agent 侧，工程师用 planner、executor、状态管理、上下文工程来绕过无状态和目标漂移。两边都在回答同一个问题：一个高产能但缺调度层的生成核心，如何被稳定地 harness 到现实世界。

但 harness 不是拐杖。好的外部支架应当帮助用户逐步内化能力，而不是让人永久挂在上面。拐杖替你走路，脚手架帮你学会在更高处作业。ADHD 工具的目标不是制造终身依赖，而是让外部调度层最终被部分内化；agent 架构的目标也不是让 LLM 永远靠外部状态，而是让系统更可靠地完成任务。

## 局限与争议：同构是工程直觉，不是科学事实

需要诚实指出的是，ADHD 大脑与 LLM 的「同构」目前更多是一种理论类比和工程直觉，并非经过实证验证的科学事实。 wiki 资料中的「矛盾与存疑」部分明确提醒：多个页面将同构作为既定事实，但缺乏实证基础（来源：全局矛盾与存疑）。此外，像 Motion 这样的工具，虽然被推荐为 ADHD 辅助工具，但「缺乏独立临床验证」（来源：全局矛盾与存疑）。AI 身体在场（如 Focusmate）是否能替代真实人际互动，也存在争议。

另一个边界是：工具本身可能增加认知负担。如果 harness 设计得太复杂，使用者会花更多精力维护工具而非完成任务。工程师在设计 agent 时也会遇到类似问题：过度复杂的 planner-executor 可能让系统脆弱、延迟变高。好的 harness 一定是轻量、可撤销、可渐隐的。

## 今天就能试

1. **给「时间盲」加外部锚点**：用 Tiimo 或 Structured 把明天的第一个任务变成视觉时间块，只设一个，让自己体验「时间被看见」的感觉。
2. **把大任务写成 plan-executor 清单**：无论是写文章还是做项目，先写 3 步以内的 plan，每步只写「下一步可执行的动词+对象」，把生成任务拆成可调用的小函数。
3. **设一个定时 re-grounding**：模仿孔子的「日课」，在一天中固定一个 5 分钟检查点，问自己：我现在做的事，是 plan 里这一步吗？不是就立刻重新调度。
4. **如果用手写工具，先做外部记忆**：用 Todoist 或 Saner.AI 把脑中悬而未决的任务全部外化，清空工作记忆，再让工具替你提醒。

时间盲和 agent 幻觉，表面上是两个世界的问题。但如果我们承认：一个强大的生成核心，需要也必须被套上一个可维护的调度层，那么 ADHD 患者和 agent 工程师，其实在做同一件事——不是消灭核心的冲动，而是给它一个稳得住的 harness。

# 用 ChatGPT 给 ADHD 补上采样温度与表现波动，真的能像 agent 那样工作吗？

> 把 ChatGPT 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

## 问题：为什么 ADHD 的「状态波动」和 LLM 的「采样温度」是同一个坑？

如果你是一个 ADHD 成人，你一定经历过这种时刻：早上醒来头脑清醒，能一口气写完报告；但下午三点，同样的任务却像在泥潭里跋涉，每个字都要从大脑里硬拽出来。这种状态日内的大幅波动，让你觉得自己像一台被随机调了「温度」的机器——有时输出流畅，有时充满噪声甚至完全卡死。

如果你是一个做 Agentic Harness 的工程师，你太熟悉这个比喻了：LLM 的采样温度（temperature）控制着输出的随机性。温度一高，模型就发散、编造；温度一低，输出就保守、重复。为了让 agent 稳定工作，你必须用提示模板、状态机、外部记忆等「结构约束」来驯服这种随机性。

**ADHD 大脑与 LLM/agent 是同一类「高产但缺执行调度层的生成核心」**（来源：AI 与 ADHD 的日常生活）。智力/算力不缺，缺的是那个能把生成能力稳定编排、按序输出的调度层。ChatGPT 这类通用 AI 工具，能否像 Agent 的 harness 一样，为 ADHD 大脑补上这块短板？

## 同构：采样温度 vs 状态波动，结构约束 vs 脚手架

ADHD 的执行功能缺陷——工作记忆不足、任务启动困难、时间盲——本质上是大脑的「调度层」不稳定。这直接映射到 LLM 的缺陷：无状态性（无法维持跨会话上下文）、上下文丢失（长对话中遗忘）、生成延迟（需要 prompt 触发）。

- **工作记忆 vs 无状态**：ADHD 的工作记忆是瓶颈，无法在脑中短暂保持信息（来源：工作记忆）。LLM 同样是无状态的，必须依赖外部记忆（如对话历史、向量数据库）来维持上下文（来源：AI 与 ADHD 的日常生活）。
- **任务启动 vs 生成延迟**：ADHD 难以启动任务，需要外部提示（如 Routinery 的过渡提示）来触发行动（来源：Routinery）。LLM 生成也需要精心设计的 prompt 来引导输出。
- **时间盲 vs 缺乏时间感知**：ADHD 对时间流逝感知模糊（来源：AI 与 ADHD 的日常生活），LLM 同样没有内置时钟，需要外部计时器或进度条来补偿。

这些同构意味着：为 LLM 设计的编排策略（提示工程、状态管理、任务分解）可以直接映射到 ADHD 辅助工具的设计中。**但关键在于，这些结构约束必须像 Agent 的 harness 一样主动施加，而不是放任 ChatGPT 自由发挥。**

## 证据：工具如何充当外部调度层

现有 ADHD 工具已经体现了这种「结构约束」思路，只是多数用户没意识到它们和 Agent 工程的同构性：

- **Routinery** 通过可视化倒计时和步骤序列，为任务启动和时间感知提供外部锚点（来源：Routinery）。这相当于给 LLM 加了一个固定的 prompt 链，每一步都有明确的输入输出格式。
- **Goblin Tools** 的 Magic ToDo 自动将复杂任务分解为小步骤（来源：Goblin Tools），相当于用 AI 做任务分解（task decomposition），降低启动门槛。
- **Saner.AI** 通过本地记忆存储和快速检索，减少搜索循环（来源：Saner.AI），相当于为 LLM 接上向量数据库，避免上下文丢失。
- **Motion** 和 **Reclaim.ai** 则用动态调度算法优化日程，相当于为 agent 添加了时间感知和优先级排序模块。

这些工具的共同点：**它们不是生成内容，而是生成结构**。它们把 ADHD 大脑的「高温度」输出约束到一条有步骤、有提示、有反馈的轨道上。

## 核心观点：ChatGPT 是生成器，不是调度器——你得自己搭 harness

很多 ADHD 用户把 ChatGPT 当作万能助手，输入一个模糊指令就期待它输出完美方案。但 ChatGPT 本身没有内置的执行功能——它就像一个温度可调的生成器，你给什么 prompt，它就输出什么，不保证稳定，不记住上下文（除非手动管理），不主动推进任务。

**ChatGPT 能补上采样温度与表现波动，但前提是你把它当作 agent 的「大脑」，而不是 agent 本身。** 你需要自己搭建 harness：
- 用固定的 prompt 模板（如「请按以下步骤回答：1. 列出要点；2. 解释每个要点；3. 给出例子」）来约束输出格式。
- 用外部笔记（如 Notion、Obsidian）来维持跨会话上下文，避免每次重新输入背景。
- 用计时器（如 Routinery）来补偿时间盲，而不是依赖 ChatGPT 的「我该花多久」判断。

没有这些结构约束，ChatGPT 只会放大 ADHD 的波动——你今天状态好，它帮你写出好文章；你状态差，它可能给你一堆废话，让你更沮丧。

## 争议与局限：脚手架还是拐杖？

同构模型虽然有力，但必须承认其局限：
- **证据强度不足**：当前证据主要来自概念类比和工具案例，缺乏大规模临床实验（来源：AI 与 ADHD 的日常生活）。ADHD 大脑与 LLM 的同构命题仍是理论框架，而非事实。
- **脚手架的可撤除性**：工具设计为长期使用，但理论上应可逐步撤除（来源：拐杖与脚手架）。现实是，用户可能形成依赖，且缺乏撤除方案的研究。
- **个体差异**：ADHD 症状异质性大，同构模型是否适用于所有亚型尚不明确（来源：AI 与 ADHD 的日常生活）。
- **多巴胺干预争议**：部分工具声称基于多巴胺原理，但效果存在争议（来源：矛盾与存疑）。

因此，不要期待 ChatGPT 或任何 AI 工具能「治愈」ADHD。它们只是临时的脚手架，帮你把一天中波动最大的那几个小时变得可控。

## 今天就能试的行动

1. **为 ChatGPT 写一个「任务分解 prompt」**：每次输入任务前，先让 ChatGPT 按「步骤1: 定义目标；步骤2: 列出子任务；步骤3: 估算每步时间；步骤4: 开始第一步」的模板输出。这相当于给 LLM 加了一个状态机。
2. **用 Routinery 或类似工具固定一个早晨流程**：设置 3-5 步，每步带倒计时，强制自己按顺序执行。体验外部调度层如何降低启动阻力。
3. **在 ChatGPT 对话中主动管理上下文**：每次新对话开头，粘贴一段「背景摘要」（如「我是 ADHD 用户，正在做 X 项目，之前讨论到 Y 点」）。这相当于为无状态 LLM 提供外部记忆。
4. **对比「有结构」和「无结构」的使用效果**：一天用自由对话做计划，另一天用固定模板做计划，记录完成率和情绪。你会发现，结构不是束缚，而是释放。

# Claude 之于 ADHD，就像重锚定 system prompt 之于 LLM——但有人用错了

> system prompt 解决的不是模型「不知道」目标,是模型在长对话里「渐渐忘了」目标——所以工程师把它固定在上下文的顶端,每一轮都重新生效。ADHD 的目标漂移,需要的正是这个「每一轮都重新生效」的机制,而不是又一次「下定决心」。

先看 LLM 侧的机制细节。长对话中的模型会发生指令漂移:开头设定的角色、约束、目标,随着上下文越拉越长而被稀释,模型的行为逐渐被最近的内容主导。工程解法不是把开头的指令写得更用力,而是**结构性重申**:system prompt 常驻顶端、关键约束定期重新注入、每个新任务块重申一遍目标。对抗漂移靠的是重复注入的架构,不是初始注入的强度。

ADHD 的目标漂移同构得近乎残忍:周一早上的目标清晰滚烫(初始注入),周三已经被三天的新刺激稀释成背景噪音,周五你在做一件完全不同的事,而且没有意识到自己偏了。**「下定决心」的失败不是决心不够,是决心只注入了一次**——而你的上下文,每天都在被新内容冲刷。

Claude 可以当这个重锚定机制的执行器——用对姿势的话。

## 用对:三层重锚定协议

**写出你的 system prompt**。第一步是把「本周/本月最重要的事」压缩成三五行文字——目标一句话、为什么重要一句话、判断偏航的标准一句话。写的过程可以让 Claude 帮忙提炼(「帮我把这堆想法压缩成一个能贴墙上的目标卡」)。**没有显式文本的目标无法被重申**——这是多数人跳过的第一步:他们的目标只存在于周一早上的情绪里。

**定时重注入**。每天固定时刻(建议开工时),把目标卡发给 Claude:「这是我的 system prompt,这是我今天打算做的事——对齐检查:偏了多少?今天哪件事直接服务它?」两分钟,等价于每轮对话重新生效的顶端指令。关键属性是**定时**而非「想起来时」:漂移的可怕之处恰恰是漂移者无感,所以重锚定不能依赖漂移者的自觉——闹钟、日历、固定例程,把触发做成外部的。

**偏航时的软着陆**。检查发现偏了(会发现的,经常偏),协议的第二半更重要:不自责、不推倒重来,只问 Claude 两个问题:「我偏去做的事有没有价值(要不要正式入队)?回到主目标的最小动作是什么?」把偏航处理成一次普通的纠偏,而不是一场道德审判——**审判会让你回避下一次检查,而回避检查才是漂移的真正保护伞。**

## 用错:三种失效姿势

**每周换一个 system prompt**。ADHD 的新奇偏好会诱惑你不断「优化」目标卡——周一定的方向,周四觉得另一个更性感。目标卡形同虚设,因为锚本身在漂。护栏:目标卡设最短生效期(两周或一个月),期内只执行不修改;修改冲动一律记入「下期候选」。**锚的价值就在于它比你的兴奋更稳定——会跟着兴奋走的锚,不是锚。**

**把重锚定做成冗长仪式**。每天的对齐检查从两分钟膨胀成四十分钟的深度反思,然后因为太重而在第四天放弃。重锚定的工程本质是轻量高频——system prompt 只有几百 token,不是一篇论文。两分钟,三个问题,到点收工。

**只锚定,不注入行动**。每天检查也做了,「偏了」也承认了,然后继续偏——因为检查的产出停在「知道」。修复:对齐检查的固定输出必须是**一个立刻可做的动作**(「接下来 45 分钟做 X」),知行之间用最小动作焊死。

## 边界

同构强度 B 级:长上下文的指令漂移是真实且被广泛处理的工程问题,ADHD 的目标维持困难有执行功能文献支撑,「定时重锚定」是机制映射的实践,无对照研究。另外:如果目标本身反复变动源于价值观层面的真实迷茫(而不是刺激驱动的漂移),那不是重锚定能解决的——那需要的是更慢的对话,和人,或和专业人士。

## 今天就能试的 3 件事

1. **今天写出目标卡**:三五行,让 Claude 帮你压缩。打印或手抄,贴在屏幕边。
2. **设定重锚定闹钟**:明天开工时刻,两分钟对齐检查。连续跑五天再评价。
3. **给目标卡标注生效期**:两周内冻结修改。所有「更好的方向」进候选清单,到期再审。

决心从来不是稀缺品,ADHD 者的决心比谁都多——稀缺的是让同一个决心在第十天还在生效的机制。工程师用一行常驻顶端的 prompt 解决了它;你的版本,是一张贴在屏幕边的卡,和一个每天响的闹钟。

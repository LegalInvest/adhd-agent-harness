# Claude 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

你有没有过这样的体验：打开 Claude，想让它帮你整理项目思路，结果 20 分钟后，你发现自己在和它聊一部刚上映的电影，或者让它帮你写一首关于拖延的诗？

这不是你的错。ADHD 大脑和 LLM 在结构上是同构的——两者都是「高产但缺执行调度层的生成核心」。你的大脑能冒出无数创意，LLM 能吐出流畅文本，但两者都容易在长程任务中「目标漂移」：你被新刺激带偏，LLM 被上下文膨胀带偏。

Claude 之于 ADHD，就像重锚定 system prompt 之于 LLM agent——它本应是那个帮你拉回主线的「调度器」。但多数人用错了：把它当成一个更聪明的聊天对象，而不是一个需要你主动设计「脚手架」的外部执行功能层。

### 同构：ADHD 大脑与 LLM 共享的困境

ADHD 的工作记忆不稳定，就像 LLM 的无跨会话状态：你今天和 Claude 聊的项目背景，明天它忘得一干二净，你必须重新输入。这种「无状态」让 ADHD 个体每天都需要「重拾上下文」，否则任务就会脱轨（来源：无状态与外部记忆）。

同时，ADHD 大脑易被环境带偏，导致任务碎片化；LLM agent 则因上下文膨胀引发推理退化——两者都需要主动设计「当下注意什么」的工程化方案（来源：上下文工程）。

这就是为什么 ADHD 社区中，「Claude Code + Obsidian + ADHD second brain」已成为稳定子主题（来源：无状态与外部记忆）。Obsidian 作为外部记忆体，补偿工作记忆波动；Claude 则作为「执行调度器」，帮你把碎片任务串联成一条主线。

### 人物案例：曾国藩的「日课」与苏轼的「豁达脚手架」

晚清名臣曾国藩是典型的注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己「无恒」「浮躁」，读书慢到「他人目下二三行，余或疾读不能终一行」。他的解决方案是「日课十二条」——黎明即起、读书不二、谨言、写日记反省——用最笨最稳的方法抵消自己的冲动。

这套「日课」本质上就是人类版的「重锚定 system prompt」：每天固定时间、固定动作，强制大脑回到主线。对应到 LLM agent 世界，就是定时 re-grounding——在长程任务中，每隔若干步骤向 agent 注入原始目标，防止漂移。

再看苏轼，兴趣极广、对无聊行政零耐心，但用「豁达」作为核心脚手架：无论被贬到哪里，都用美食、写作、交朋友转移注意力，每日读书写字不因挫折停止。他的「豁达」不是天生乐观，而是一种主动设计的「上下文切换策略」——当环境变化时，快速重锚定到「好好生活」这个 system prompt。

### 工具生态：外部执行功能层的真实证据

现有工具已经证明，AI 作为外部执行功能层是可行的：

- **RescueTime** 自动追踪时间，揭示「时间到底去了哪里」，直接补偿 ADHD 的「时间盲」（来源：RescueTime）。
- **Focusmate** 通过虚拟身体在场提供外部问责，弥补内在多巴胺调节不足（来源：Focusmate）。
- **Brain.fm** 利用神经锁相技术生成专注音乐，在 2026 年 44 款 ADHD 应用测试中位列最佳 9 款（来源：Brain.fm）。
- **Forest** 用游戏化番茄钟增强动机（来源：AI 与 ADHD 的专注力）。

这些工具的共同逻辑是：它们不是替代你的大脑，而是为你搭建「脚手架」——当你的内部调度层失效时，外部系统帮你把注意力拉回目标。

### 但有人用错了：脚手架 vs 拐杖

Claude 最危险的使用方式，是把它当作「永远在线的聊天对象」。你向它倾诉、让它帮你写邮件、替你思考——这本质上是在训练自己依赖一个「外部大脑」，而不是在构建可内化的执行功能。

真正的用法是：把 Claude 当作一个「重锚定工具」。比如，每天开始时，你给它一个固定的 system prompt：「我是 ADHD 用户，我的核心目标是 X。请在我跑题时提醒我回到主线，并记录每次漂移的原因。」然后，你像训练 agent 一样训练它——每次对话结束后，复盘漂移点，调整 prompt。

这就是「脚手架」与「拐杖」的边界：脚手架最终可以拆除，因为你内化了它的结构；拐杖你永远离不开。

### 争议与局限

必须诚实指出：ADHD 大脑与 LLM 的同构目前仍是理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。多数工具的证据来自用户报告或小样本研究，缺乏大规模随机对照试验（来源：AI 与 ADHD 的专注力）。此外，过度依赖 AI 工具可能导致执行功能进一步退化——如果你永远不练习「手动重锚定」，你的内部调度层可能越来越弱。

### 今天就能试的 3 件事

1. **给 Claude 写一个「重锚定 prompt」**：在对话开头加上「我是 ADHD 用户，我的目标是 X。每当我偏离超过 2 轮，请用『回到主线：X』打断我。」
2. **用 RescueTime 记录一周**：不看报告，只看「时间盲」的程度——你实际花在目标上的时间和你以为的差多少？
3. **建立每日「re-grounding 仪式」**：像曾国藩的日课那样，每天固定时间（比如早上 9 点）用 Obsidian 或笔记工具回顾你的核心目标，写一句话：「今天我的 system prompt 是什么？」

Claude 不是你的替代大脑，它是你的「重锚定 system prompt」。用对了，它帮你把飘散的注意力一次次拉回主线；用错了，它只是让你更高效地跑偏。

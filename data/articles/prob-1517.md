# 如果把 ADHD 大脑当成生成核心，Todoist 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Todoist 对 ADHD 的真正价值。

## 问题：你的“生成核心”总在漂移，谁来做调度层？

如果你是 ADHD 患者，你一定熟悉这样的场景：打开电脑打算写一份报告，却被一条通知拉进社交媒体，一小时后才发现自己还在刷短视频。或者，你因为某个灵感一头扎进细节，直到深夜才意识到原定目标早已偏离。如果你是做 Agentic Harness 的工程师，你一定见过类似的场景：你给 LLM agent 一个长程任务，它在执行到第三步时开始“自由发挥”——生成的内容偏离了最初的目标，你需要不断重写 system prompt 来把它拉回来。

这两类问题共享同一个底层结构：一个强大的生成核心（ADHD 大脑或 LLM）缺乏可靠的内置执行调度层，导致目标漂移。ADHD 大脑的“分心走神”和“超聚焦跑偏”，与 LLM agent 的“上下文膨胀导致推理退化”或“任务分解后丢失全局目标”，本质上是同一种缺陷（来源：AI 与 ADHD 的专注力）。

那么，一个看似简单的工具——Todoist——能否成为这个生成核心的重锚定系统？它真的能对抗目标漂移吗？

## 同构脊柱：重锚定与目标漂移

### ADHD 一侧：外部提醒作为重锚定

ADHD 大脑在执行任务时极易发生目标漂移：注意力被无关刺激捕获（分心），或过度聚焦于细节而忽略整体目标（超聚焦）。外部工具通过提醒和通知提供重锚定，将注意力拉回原任务（来源：重锚定与目标漂移）。例如，AI 工具“将任务分解为可管理的步骤，设置提醒和专注计时器以提高生产力”（来源：AI for ADHD: Best Tools, Apps, and Strategies - Themba Tutors）。

但传统提醒系统有一个关键缺陷：“Traditional reminder systems help, but they require you to set the reminder in the first place, which is itself an executive function task”（来源：AI Assistant for ADHD: Finally, a Productivity Tool That Requires Less...）。这意味着设置提醒本身就是一个执行功能任务，而 ADHD 个体恰恰在这方面薄弱。Todoist 的重复提醒、自然语言输入和项目模板可以降低这个门槛，但它仍然需要用户主动设计结构。

### LLM/Agent 一侧：system prompt 作为重锚定

LLM 作为生成核心，缺乏内在规划能力，需要外部 harness 提供任务分解、规划循环和验证机制。在 agentic harness 文献中，“任务分解与规划”是核心概念（来源：LBD 同构对：任务分解与规划 — Safety and recommendations for TMS use... ↔ AgentGen）。代表作 AgentGen 通过环境与任务生成增强 LLM agent 的规划能力（来源：AgentGen: Enhancing Planning Abilities for Large Language Model based Agent via Environment and Task Generation）。而重锚定机制通常表现为：在 agent 执行一定步骤后，重新注入原始目标（即 system prompt 的刷新）。这相当于一个定时“归位”信号。

### 人物案例：曾国藩的“日课十二条”

曾国藩是典型的注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”。他的专属 harness 是“日课十二条”：黎明即起、读书不二、谨言、写日记反省等。其中“读书不二”规定一本书不读完绝不翻下一本，这本质上是一种“任务锁定”——相当于在 LLM agent 的 system prompt 里写死当前子任务，禁止上下文切换。而每日写日记反省，则相当于每日一次的“重锚定”——重新审视目标，纠正漂移。

这个系统与 LLM harness 的结构同构：日课十二条 ↔ 固定的 re-grounding 规则；日记反省 ↔ 定时日志输出与目标校验。曾国藩用最笨最稳的方法（“结硬寨打呆仗”）抵消了自己的冲动，成就了平定太平天国的伟业。

## Todoist 能否成为重锚定系统？

Todoist 的核心功能——项目、标签、优先级、重复提醒、自然语言输入——确实可以构建一个外部调度层。但它的有效性取决于一个关键变量：**用户是否主动设计了重锚定机制**。

如果只是把 Todoist 当作一个普通待办清单，它很快就会变成另一个“被遗忘的列表”——ADHD 用户最熟悉的失败模式。但如果用户利用它的功能构建一个类似“日课十二条”的系统，例如：
- 每天固定时间（如早 8 点）自动生成“今日核心目标”提醒（重锚定）；
- 每个长程任务拆解为子任务，并设置中间提醒（防止超聚焦跑偏）；
- 每周使用标签和过滤视图复盘“实际完成 vs 计划”，进行目标校验。

那么 Todoist 就从一个被动存储工具，变成了一个主动的重锚定系统。这与 agent harness 中“定时注入 system prompt”或“每一步后校验目标”的机制完全同构。

## 脚手架 vs 拐杖：边界在哪？

这里有一个必须诚实的争议：AI 工具既可能成为永久依赖的拐杖，也可能成为逐步内化的脚手架（来源：拐杖与脚手架）。Todoist 本身不是 AI 工具，但它可以接入 AI 功能（如自然语言解析）。如果用户完全依赖它来提醒自己“该做什么”，而不培养内在的自我监控能力，那么它就成了拐杖。如果用户通过使用它，逐渐学会“任务分解”“优先级判断”“定期复盘”这些执行功能技能，那么它就是脚手架。

目前缺乏长期追踪研究来区分这两种路径（来源：矛盾与存疑）。但一个合理的判断是：**Todoist 的价值不在于它本身，而在于你用它做什么**。如果你只是用它记录任务，它可能只是另一个数字便签；如果你用它构建一个“重锚定循环”，它就能成为对抗目标漂移的有效 harness。

## 证据与局限

现有工具的证据支持了外部重锚定的有效性：RescueTime 通过自动时间追踪帮助用户识别注意力漂移（来源：RescueTime）；Focusmate 通过虚拟身体在场提升启动与专注力（来源：Focusmate）；Brain.fm 利用神经锁相技术帮助进入专注状态（来源：Brain.fm）。但多数证据来自用户报告或小样本研究，缺乏大规模随机对照试验（来源：AI 与 ADHD 的专注力）。

更重要的是，ADHD 的异质性意味着同一工具对不同亚型效果可能迥异（来源：矛盾与存疑）。注意缺陷为主型可能更需要提醒，而多动冲动型可能更需要身体在场。Todoist 的静态提醒对前者可能足够，对后者则可能无效。

## 今天就能试的行动

1. **设置每日“重锚定提醒”**：在 Todoist 中创建一个每天固定时间（如上午 9 点）重复的任务，内容为“回顾今日核心目标”。这相当于 agent 的定时 system prompt 刷新。
2. **将长程任务拆解为子任务并设置中间提醒**：比如“写报告”拆成“查资料（30 分钟）→ 写大纲（30 分钟）→ 写正文（2 小时）”，每个子任务设置提醒，防止超聚焦。
3. **每周用标签复盘**：给任务打上“完成”“漂移”“未启动”标签，周末查看模式识别自己的目标漂移模式。
4. **结合身体在场工具**：如果 Todoist 提醒仍不够，搭配 Focusmate 的虚拟工作时段，用社会压力强化重锚定。

## 结论

Todoist 可以成为 ADHD 大脑的重锚定系统——但前提是你把它当作一个“外部调度层”来主动设计，而不是一个被动的待办列表。它与 LLM agent 的 system prompt 重锚定机制同构，但它的有效性取决于你是否愿意成为自己的“harness 工程师”。如果你只是用它记录任务，它可能只是拐杖；如果你用它构建重锚定循环，它就能成为脚手架。这个边界，由你决定。

# ADHD 研究生如何用「给生成核心套 harness」的思路应对长周期科研与论文里的执行功能（计划/组织/启动）先天偏弱？

> 把 LLM 工程里的「agent 的编排/记忆/工具层就是「外部执行功能」」迁移到ADHD 研究生的长周期科研与论文场景，双域证据作答。

## 问题：ADHD 大脑如何驾驭长周期科研？

当一篇博士论文的周期以年为单位，ADHD 研究生的日常常常是这样的：开题时灵感喷涌，但一周后文档里只有标题；文献读了三篇就切换到新方向；实验做到一半发现忘了关键参数；deadline 前三天才开始写，然后陷入自我厌恶。这不是懒惰，而是执行功能——计划、组织、启动——先天偏弱的表现。ADHD 大脑的工作记忆容量有限且不稳定，导致跨会话任务难以持续（来源：无状态与外部记忆）。时间感知扭曲，任务启动困难，多巴胺驱动的动机系统让“重要但不紧急”的论文永远排在“有趣但无关”的探索之后。

传统的建议——番茄钟、待办清单、时间管理——往往失效，因为它们依赖的正是 ADHD 所缺失的执行调度能力。我们需要一个更根本的视角转换。

## 同构：ADHD 大脑与 LLM 是同一类生成核心

ADHD × AI 领域当前最站得住脚的核心论点是：**ADHD 大脑与 LLM 在结构上同构——两者都是“高产但缺执行调度层的生成核心”**（来源：主题综述）。ADHD 大脑的生成核心（发散思维、联想能力）与 LLM 的文本生成能力类似，但都缺乏可靠的内置执行调度层。LLM 的无状态性导致每次对话独立，不保留跨会话记忆（来源：无状态与外部记忆）；ADHD 大脑的工作记忆缺陷同样导致无法可靠保留上下文。两者都需要外部系统来补偿工作记忆不稳定、任务启动困难和注意力分散（来源：外部执行功能层）。

这意味着：**AI 帮助 ADHD 的本质不是“修复缺陷”，而是“给生成核心套上 harness”**（来源：主题综述）。LLM 工程中的 agent scaffolding——编排、记忆、工具层——正是 ADHD 大脑缺失的“外部执行功能层”。把 LLM agent 的架构迁移到自身，ADHD 研究生就拥有了一个可复用的方法论。

## 历史案例：曾国藩与卢作孚的“外部执行功能层”

曾国藩是典型的 ADHD 大脑。30 岁前浮躁坐不住，天天串门喝酒看杀人，日记里天天骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”。他的解决方案是“日课十二条”：黎明即起、读书不二、谨言、写日记反省等。这套日课本质上就是一个外部调度层——每天固定时间执行固定动作，用结构化的外部程序来对冲内在的执行功能缺陷。他的“结硬寨打呆仗”策略，就是用最笨最稳的方法抵消冲动，类似于 LLM 工程中的“上下文工程”——主动设计和管理当前关注的信息范围，防止上下文膨胀导致推理退化（来源：上下文工程）。日课相当于定时 re-grounding 机制，写日记则是外部记忆系统。

卢作孚同样具有 ADHD 特质：小学毕业，坐不住，天天在船上跑。他的 harness 是“服务社会，便利人群，开发产业，富强国家”这一公司宗旨，以及严格的管理制度——废除买办制度，让员工成为公司的主人。这相当于一个强目标导向的编排层，将个人冲动转化为组织化行动。宜昌大撤退的奇迹，正是这个 harness 在极端压力下的高效运转。

## 当代工具：如何设计自己的 Harness

基于同构视角，ADHD 研究生可以构建三层外部执行功能层：

**1. 编排层（Planning & Orchestration）**：对应 LLM agent 的任务调度。Motion 这类 AI 日程规划应用能自动根据任务、会议和截止日期创建并动态调整每日计划，消除“下一步该做什么”的决策负担（来源：Motion）。对于长周期科研，可以将论文拆解为里程碑，让 Motion 自动排期并提前警告延期风险。Goblin Tools 的 Magic ToDo 功能则能将模糊任务（如“写引言”）分解为具体子步骤，降低启动门槛（来源：Goblin Tools）。

**2. 记忆层（Memory & Context）**：对应 LLM agent 的向量数据库。Saner.AI 专注于知识回忆和本地记忆，减少搜索循环和标签切换（来源：Saner.AI）。Obsidian 可作为“第二大脑”映射超聚焦和复杂想法（来源：无状态与外部记忆）。对于科研，这意味着建立一个外部知识库，存储文献笔记、实验参数、写作片段，让 AI 自动检索相关上下文，避免每天重新回忆。

**3. 工具层（Tools & Execution）**：对应 LLM agent 的函数调用。利用 AI prompt 模板完成特定任务，如“AI Prompts for Task and Time Management”（来源：上下文工程）。将重复性工作（格式检查、文献整理）交给 AI，释放认知资源。

## 脚手架 vs 拐杖：核心争议与边界

必须诚实指出：这套方法存在争议。核心矛盾在于：**AI 工具是“外部执行功能层”还是“永久拐杖”？**（来源：矛盾与存疑）过度依赖可能导致能力退化，但尚无长期研究验证。我的判断是：**区分的关键在于工具是否可拆卸**。脚手架在建筑完成后可以拆除，而拐杖是永久替代。好的 harness 应设计为阶段性使用——初期用 Motion 强制建立节奏，后期逐渐减少依赖；用 Saner.AI 记录知识，但定期回顾以内化。另一个争议是：超聚焦应被引导还是打断？部分用户反馈强行打断（番茄钟）有效，而柔性引导可能不够强力（来源：矛盾与存疑）。个体差异大，建议实验后选择。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个卡住的任务**：选一个你拖延已久的论文任务，用 Magic ToDo 分解到“辣度 3”，然后执行第一步。
2. **用 Motion 规划下周科研**：将论文里程碑、会议、实验时间块输入 Motion，让它自动排程，观察是否减少决策疲劳。
3. **建立外部记忆库**：在 Obsidian 中创建一个“论文上下文”笔记，每天用 AI 总结当天的进展和下一步，第二天开始前先读它。

ADHD 大脑不是缺陷，而是一个缺少外部 harness 的高性能生成核心。与其对抗内在的散漫，不如像工程师一样设计自己的外部执行功能层。毕竟，LLM 能靠 scaffolding 成为强大 agent，你也能。

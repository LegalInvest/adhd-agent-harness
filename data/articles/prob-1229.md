# 用 Motion 给 ADHD 补上外部执行功能层，是脚手架还是拐杖？

> Motion 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：当大脑的调度层失灵，AI 是补丁还是替代？

ADHD 大脑是一个高产但缺乏执行调度层的生成核心：发散思维、创意、多任务并行能力极强，但计划、组织、启动、时间管理这些执行功能先天偏弱（来源：[[执行功能]]）。这就像一台拥有顶级 GPU 却没有操作系统的计算机——算力溢出，却无法稳定运行一个程序。

与此同时，LLM agent 工程正面临同样的窘境：模型本身是强大的生成核心，但缺乏可靠的编排层，直接输出可能偏离目标、陷入循环或产生危险行为（来源：Building an AI Agent Harness from Scratch）。Agent harness（脚手架）通过人在回路、护栏、事件驱动系统提醒来提供外部调度——这恰恰是 ADHD 大脑缺失的“执行功能层”。

当 Motion 这样的 AI 工具介入 ADHD 工作流，它本质上是在扮演这个外部执行功能层。但关键问题随之而来：它是可撤除的脚手架，还是让人依赖的拐杖？

## 外部执行功能层：ADHD 与 LLM 的同构解法

ADHD 大脑与 LLM 的结构同构并非空泛类比。ADHD 的工作记忆缺陷对应 LLM 的无状态性；ADHD 的上下文丢失对应 LLM 的上下文窗口溢出；ADHD 的执行功能缺陷对应 LLM 缺少编排层（来源：[[ADHD 大脑与 LLM 的同构]]）。两边都指向同一个解法：在生成核心外面套一个可靠的调度层。

在 ADHD 侧，这个调度层可以来自工具。Motion 通过 AI 自动排程与动态调整，持续评估任务优先级、截止日期和可用时间，实时重建日程，减少用户的手动规划压力（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。这直接对抗 ADHD 的“时间盲”和任务启动困难——用户不再需要纠结“下一步该做什么”，因为工具已经替他们决策了（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。

在 LLM 侧，agent harness 通过形式化规划与护栏，使代理输出真正有用且正确（来源：What is an agent harness in the context of large-language models?）。例如，通过事件驱动的系统提醒来对抗模型在长程任务中的“指令消退”，防止目标漂移（来源：Building an AI Coding Agent for the Terminal）。两边都依赖外部调度来弥补内在缺陷。

## Motion 的脚手架属性：降低启动门槛，但保留控制权

Motion 的设计逻辑更接近脚手架。它要求用户输入所有任务和截止日期，AI 基于这些输入自动排程。用户仍需要设定目标和优先级，工具只负责优化执行顺序。这种“半自动”模式保留了用户对任务本质的控制权——用户决定做什么，Motion 决定什么时候做。

这与 Goblin Tools 的任务分解功能类似：将“整理房间”自动拆成“捡起地板上的衣服”“擦桌子”等步骤，降低启动门槛（来源：AI Tools for Kids with ADHD）。用户仍然需要执行每一步，工具只是把压倒性的事情变成一系列不压倒性的事情（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。这种设计让用户保持对任务的认知参与，而非完全外包。

Saner.AI 则从另一个角度提供脚手架：通过本地记忆和快速检索减少搜索循环和标签切换，减轻工作记忆负担（来源：Best AI Tools for ADHD Productivity in 2026）。用户仍然需要主动回忆和决策，但工具降低了认知负荷——就像在复杂迷宫中提供路标，而非直接替用户走路。

## 拐杖的风险：当工具替代了内在能力

拐杖的典型特征是：一旦撤除，用户无法独立行走。在 ADHD 工具领域，这个风险真实存在。Motion 的自动排程虽然高效，但长期使用可能削弱用户的时间管理能力——如果用户从未练习过手动规划，当工具不可用时，他们可能更难以应对。

更值得警惕的是，多数工具（如 Goblin Tools、Saner.AI、Motion）设计为长期使用，未提及撤除机制（来源：[[拐杖与脚手架]]）。这与脚手架应逐步撤除的核心理念相矛盾。目前缺乏纵向研究来验证长期使用 AI 工具是否会削弱内在执行功能（来源：[[ADHD × AI 的科学与研究前沿]]）。

在 LLM 侧，同样存在依赖风险：如果 agent harness 过于严密，模型可能丧失在无护栏环境下的适应能力。过度依赖外部调度，会让核心生成能力变得脆弱。

## 边界在哪里？三条判断标准

1. **用户是否参与元认知训练**：如果工具只是替用户做决策，而不帮助用户理解为什么这么做，它就是拐杖。如果工具在提供外部调度的同时，让用户反思自己的时间模式、优先级判断，它就是脚手架。

2. **工具是否可逐步撤除**：Motion 等工具目前缺乏“逐步降低干预度”的功能。如果未来允许用户从全自动排程逐步过渡到半自动、手动，它就更接近脚手架。

3. **用户是否保留任务本质控制权**：Motion 让用户决定做什么，工具只优化何时做，这保留了控制权。但如果工具开始替用户决定“做什么”（例如基于算法自动添加任务），就越过了拐杖的边界。

## 今天就能试的行动

1. **用 Motion 的“手动输入截止日期”功能**：不要依赖 AI 自动抓取日期，自己设定每个任务的截止时间。这能保持你对任务优先级的主动思考，避免完全外包规划能力。

2. **每周一次“无工具日”**：关闭 Motion 等自动排程工具，尝试手动规划一天的任务。记录下哪些地方感到困难——这些正是你需要刻意练习的执行功能。

3. **用 Saner.AI 的本地记忆功能建立“个人知识库”**：但每周花 15 分钟回顾和整理存储的信息，而不是完全依赖搜索。这能训练你的工作记忆和分类能力。

4. **参考 LLM agent 的“人在回路”原则**：在使用工具时，定期暂停并问自己：“我是否理解工具为什么这样安排？”如果答案是否定的，说明你可能正在依赖而非使用工具。

## 诚实面对争议

必须承认，ADHD 与 LLM 的同构模型目前主要基于概念类比和工具案例，缺乏大规模实证（来源：[[ADHD 大脑与 LLM 的同构]]）。工具的有效性证据多来自用户反馈，而非独立临床研究（来源：[[ADHD × AI 的科学与研究前沿]]）。此外，多巴胺干预的有效性仍存争议（来源：[[矛盾与存疑]]）。这意味着我们尚未完全理解 AI 工具对 ADHD 大脑的长期影响。

但至少，我们可以选择让工具做脚手架而非拐杖：保留用户的主动参与，设计可撤除的支撑，并始终将工具视为训练伙伴而非替代品。对于 LLM agent 工程而言，同样的原则适用——harness 应该让模型更强，而非更依赖护栏。

# Brain.fm 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Brain.fm：它到底补上了哪一层执行功能？

你打开 Brain.fm，戴上耳机，期待音乐能像魔法一样让你瞬间进入专注。但十分钟过去，你还在刷手机。这就像给 LLM 一个 planner 模块，却忘了加执行护栏——模型依然在生成，但输出可能偏离轨道。

问题出在哪？ADHD 大脑与 LLM 共享同一个核心困境：**高产但不可靠的生成核心，缺少可靠的执行调度层**。Brain.fm 的神经锁相技术试图直接调节大脑状态，就像 planner-executor 架构试图让 LLM 按计划行动。但两者都容易忽略一件事：**调度层不是简单加一个模块，而是需要一整套脚手架**。

## 同构：ADHD 大脑与 LLM 的同一类困境

ADHD 大脑的智力与创造力（生成核心）与 LLM 的生成能力类似，但缺乏执行功能（调度层）来组织行动（来源：AI 与 ADHD 的时间管理）。LLM 本身是强大的生成核心，但直接输出可能偏离目标、陷入循环或产生危险行为（来源：人在回路与身体在场）。

这种同构在任务启动上尤为明显。ADHD 患者的多巴胺系统失衡导致难以激发行动动机，工作记忆容量有限使得任务步骤难以清晰呈现（来源：任务启动）。LLM 同样缺乏长期记忆和内在动机，需要外部提示和约束才能保持方向。

## Brain.fm 的错位：它补的是状态，不是调度

Brain.fm 声称通过神经锁相技术帮助用户进入专注状态（来源：矛盾与存疑）。但针对 ADHD 的效果缺乏独立临床研究支持（来源：矛盾与存疑）。问题在于：**ADHD 的核心缺陷不是无法进入状态，而是无法启动和维持对任务的执行**。即使 Brain.fm 能诱导 alpha 波，你仍然需要知道下一步该做什么。

这就像给 LLM 一个 planner 模块，却忘了告诉它如何分解任务、如何检查进度、如何处理异常。Motion 和 Reclaim.ai 通过自动排程和动态调整，直接补上了「下一步该做什么」的决策（来源：Motion；Reclaim.ai）。Tiimo 通过视觉化时间线，把时间变成可见元素，缓解时间盲（来源：Tiimo）。这些工具补的是**调度层**，而非状态层。

## 真正的解法：planner-executor 循环 + 外部脚手架

在 LLM/agent 一侧，可靠的 agent harness 通过 human-in-the-loop 监督提供外部调度：设置护栏（token 预算、工具调用次数上限）、加入人工检查点（暂停并询问后再执行）（来源：人在回路与身体在场）。Harness 将 MCP 工具、遥测、代码仓库、事件管理平台集成到单一系统中（来源：人在回路与身体在场）。

在 ADHD 一侧，等效的做法是：**用外部工具构建一个 planner-executor 循环**。例如，Goblin Tools 的「魔法待办」可将复杂任务拆解为小步骤（来源：任务启动），降低启动门槛。Saner.AI 提供基于认知科学的启动策略，如「两分钟法则」或「身体在场」提示（来源：任务启动）。Focusmate 通过虚拟身体搭档提供外部问责，绕过执行功能屏障（来源：人在回路与身体在场）。

这些工具的共同点是：**它们不是一次性解决所有问题，而是构成一个可迭代的循环**：
1. **Plan**：用 Motion 或 Reclaim.ai 自动排程，减少决策负担。
2. **Execute**：用 Brain.fm 辅助状态调节，但核心是执行小步骤（来自 Goblin Tools 的分解）。
3. **Check**：用 Focusmate 或定期向 AI 报告进度（来源：人在回路与身体在场）。
4. **Adapt**：用 Tiimo 或 Motion 动态调整计划。

## 脚手架 vs 拐杖：边界在哪里？

这正是争议所在。AI 工具既是「拐杖」也是「脚手架」（来源：AI 与 ADHD 的时间管理）。脚手架应设计为可逐步撤除的临时支撑，但多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。

我的判断是：**关键不在于工具本身，而在于你是否在使用中发展了自己的执行功能**。如果你只是依赖工具替你决策，那就是拐杖；如果你通过工具学会了如何分解任务、如何管理时间，那就是脚手架。

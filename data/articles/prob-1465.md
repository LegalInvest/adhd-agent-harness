# Structured 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Structured：它到底补上了哪一层执行功能？

你有没有过这样的体验：打开电脑准备写报告，却鬼使神差点开了社交媒体；或者为了一个细节研究三小时，回过神来发现原定目标早已跑偏。对 ADHD 人群来说，这种「分心走神或超聚焦跑偏」是日常。而在 AI 工程领域，你给 agent 一个长程任务，它可能在第三轮对话后就开始「目标漂移」——原本要写代码，却开始解释概念。

这两类问题看似无关，实则共享同一结构：**一个高产但缺执行调度层的生成核心**。ADHD 大脑是生成核心——创造力强、联想丰富，但执行功能（工作记忆、任务启动、时间感知）薄弱（来源：生成核心与调度层）。LLM 同样是生成核心——语言能力强，但缺乏内置调度，容易上下文膨胀、推理退化（来源：生成核心与调度层）。

因此，两边都需要外部脚手架来「重锚定」：对 ADHD 是日课、计时器、身体加倍；对 LLM 是 system prompt 刷新、双代理架构、上下文压缩。Structured 这类工具，本质上就是数字化的重锚定系统。

## 同构的实证：ADHD 与 LLM 共享的认知剖面

最新研究揭示了更深层的平行：ADHD 患者的工作记忆容量未必差，但自上而下的控制不足，呈现「强记忆、弱控制」模式（来源：生成核心与调度层）。LLM 同样——记忆容量巨大，但缺乏自主调控，需要外部提示来引导推理。Transformer 的自注意力机制在训练后甚至模仿了前额叶-纹状体的门控操作（来源：拐杖与脚手架）。

这意味着，ADHD 大脑与 LLM 不仅是功能类比，在计算机制上也有同构性。所以，给 ADHD 用的 Structured 工具，和给 LLM 用的 agent scaffolding，解决的是同一个问题：**如何防止生成核心在执行长程任务时丢失目标**。

## 真实人物的 harness：曾国藩与张旭

晚清名臣曾国藩是典型 ADHD 特质：30岁前浮躁坐不住，日记里天天骂自己「无恒」「浮躁」；读书慢，「他人目下二三行，余或疾读不能终一行」；情绪不稳定，打败仗就跳水自杀。他的 harness 是「日课十二条」：黎明即起、读书不二、谨言、写日记反省……用最笨最稳的方法抵消冲动。这本质上就是**定时 re-grounding**——相当于每半小时给 LLM 刷新一次 system prompt。

草圣张旭则走了另一条路：每次喝醉后号呼狂走，以头濡墨作书，醒来以为神。他利用酒精进入超专注状态，把情绪注入草书。但这不是无纪律的放纵——他从公孙大娘舞剑中悟笔法，从自然中找灵感。他的 harness 是**环境工程**：用酒和特定场景触发超聚焦，同时从外部输入（舞剑、自然）来重新锚定创作方向。这就像给 LLM 设计一个「触发词+外部知识库」的组合 prompt。

两者的共同点：**外部脚手架不是替代内在能力，而是为内在生成核心提供方向锚点**。曾国藩的日课让他「结硬寨打呆仗」，张旭的酒让他「醒后自视以为神」——脚手架帮他们抵达了自己无法独自抵达的地方。

## 工具实测：Structured 补上了哪一层执行功能？

当前 ADHD 工具生态中，Structured 类应用（如 Brain.fm、Focusmate、RescueTime）分别补上了不同执行功能层：

- **RescueTime** 补上「时间盲」：自动记录时间去向，让用户看到真实的时间消耗，相当于 LLM 的日志审计（来源：RescueTime）。
- **Focusmate** 补上「启动困难」：通过虚拟身体在场提供外部问责，相当于 LLM 的 human-in-the-loop 校验（来源：Focusmate）。
- **Brain.fm** 补上「注意力维持」：用神经锁相音乐影响大脑状态，相当于 LLM 的上下文工程——主动设计「当下注意什么」（来源：Brain.fm）。

但这些工具的共同局限是：**临床证据不足**（来源：矛盾与存疑）。Brain.fm 在 44 款 ADHD 应用中位列最佳 9 款之一，但缺乏独立 RCT 验证（来源：Brain.fm）。Focusmate 的效果与身体在场效应一致，但 AI 匹配是否能替代真实人际互动仍有争议（来源：矛盾与存疑）。

## 脚手架 vs 拐杖：边界在哪？

核心判断：**Structured 工具只有当它帮助使用者「建造」而非「替代」时，才是真正的脚手架。** 如果一个人离开 RescueTime 就无法估算时间，离开 Focusmate 就无法启动任务，那么工具就成了拐杖。

专家警告：真正的脚手架应帮助使用者建造，但使用者仍需自己攀登（来源：拐杖与脚手架）。同理，LLM 的 agent scaffolding 如果设计成「永远依赖手动修正提示」，那也只是拐杖；好的 scaffolding 应让 agent 逐渐学会自主发现最佳模式（来源：拐杖与脚手架）。

## 诚实面对争议

1. **依赖 vs 内化**：目前没有长期追踪研究证明 ADHD 用户能否通过工具内化执行功能（来源：矛盾与存疑）。
2. **个体差异**：ADHD 的异质性意味着同一工具对不同亚型效果迥异（来源：矛盾与存疑）。
3. **同构的边界**：ADHD 与 LLM 的同构目前仍是功能类比，缺乏神经机制的直接证据（来源：矛盾与存疑）。

## 今天就能试的行动

1. **设置一个「重锚定」闹钟**：每 25 分钟响一次，问自己「我当前的目标是什么？」——这是最简的 system prompt 刷新。
2. **试用 Focusmate 一次**：预约一个 50 分钟的虚拟身体在场时段，体验外部问责如何降低启动阻力。
3. **用 RescueTime 记录 3 天**：不评判，只观察——你的时间盲有多严重？
4. **写一条「日课」**：模仿曾国藩，选一条你最需要坚持的行为（如「读书不二」），每天执行并记录。

Structured 不是魔法，它是你大脑的调度层外挂。用对了，它是脚手架；用错了，它是拐杖。而选择权，在你手里。

# 如果把 ADHD 大脑当成生成核心，Structured 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Structured 对 ADHD 的真正价值。

## 问题：工作记忆的「漏勺」与LLM的「无状态」

你有没有这样的体验：打开浏览器想查个资料，结果被一条推送吸引，半小时后关掉页面，完全忘了最初要做什么？或者，你正写着一份报告，突然想起该回一封邮件，切过去之后，再回来时报告的思路已经断了。

对于ADHD人群，这种「边做边忘」不是偶尔的疏忽，而是日常的常态。根源在于工作记忆容量小——大脑的「临时缓存」只能同时处理寥寥几个信息块，一旦被打断，之前的上下文就灰飞烟灭（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

有趣的是，如果你把ADHD大脑比作一个生成核心，它的困境与LLM（大语言模型）惊人地相似。LLM本身没有跨会话状态，每次对话都是全新的开始，上下文窗口一满，前面的内容就「遗忘」了。为了解决这个问题，工程师们为LLM搭建了外部记忆系统（向量库、会话摘要）和调度层（harness），让模型能「记住」并「执行」多步骤任务。

那么，如果把ADHD大脑当成生成核心，Structured——一款强调「无状态」和「外部记忆」的工具——能不能成为它的harness？

## 同构：生成核心与调度层的双重缺失

### ADHD侧：高产但无序的创造者

ADHD的核心困境并非智力不足，而是执行功能的失效。执行功能被描述为「大脑的驾驶座」，但对ADHD来说，「常常感觉方向盘后没有人」（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。这种调度层的缺失导致计划难以执行：「计划本和便签用了一周就崩溃了」（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。工作记忆是另一个瓶颈，进一步削弱了任务编排能力（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

### LLM/Agent侧：强大的语言引擎，脆弱的执行

LLM作为生成核心，拥有强大的语言理解和生成能力，但单独使用时缺乏可靠的执行调度。现代AI编码代理通过「复合AI系统架构」来弥补这一缺陷，包括「工作负载专用模型路由、分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩」（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这些技术本质上是在LLM外部搭建调度层，防止「上下文膨胀和推理退化」（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

### 同构对应

| ADHD侧 | LLM/harness侧 |
|---------|----------------|
| 生成核心：智力/创造力 | 生成核心：算力/语言能力 |
| 调度层缺失：执行功能失效 | 调度层缺失：缺乏内置规划与执行编排 |
| 外部工具：Structured等作为脚手架 | 外部工具：harness提供上下文、记忆、工具接口 |

这种同构意味着：ADHD大脑与LLM属于同一类「高产但缺执行调度层的生成核心」，两边的解法（脚手架/harness）在结构上同构。

## Structured：无状态设计与外部记忆的双重脚手架

Structured的设计哲学恰好迎合了这种同构需求。它强调「无状态」——每个任务或会话都是独立的，不依赖之前的上下文，从而避免了工作记忆的过载。同时，它提供「外部记忆」——通过持久化存储（如笔记、任务列表、知识库），让用户无需依赖大脑缓存即可找回之前的信息。

### ADHD侧的实证

- **任务启动**：Goblin Tools通过分解任务降低启动焦虑（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）；Lex通过单一指令触发复杂流程，减少决策负担（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。Structured的无状态设计类似：每次只聚焦当前一步，不要求记住整个流程。
- **工作记忆**：Mem和Saner.AI提供持久记忆，减少搜索循环（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）；Otter.ai实时转录减轻笔记负担（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。Structured的外部记忆功能同样能「外化思维」，减轻工作记忆负荷（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）。
- **时间盲**：AI提醒和日程工具（如Goblin Tools的魔法待办）帮助感知时间流逝（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。Structured的时间块规划也能起到类似作用。

### LLM/Agent侧的实证

在Agent工程中，Structured式的无状态+外部记忆是标准做法。LLM的固定上下文窗口导致长程依赖追踪困难（来源：Long-Term Memory Architectures for AI Agents - Redis），而harness通过提供上下文、记忆、工具接口来弥补（来源：GitHub - ai-boost/awesome-harness-engineering）。具体技术包括：

## 脚手架 vs 拐杖：边界在哪？

然而，我们必须诚实面对争议。过度依赖AI工具可能阻碍内在执行功能发展，如何设计可逐步撤除的脚手架仍是挑战（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。真正的脚手架应帮助使用者「建造」，但使用者仍需自己「攀登」（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

Structured作为工具，如果用户只是机械地跟随其安排而不培养内在的规划能力，就可能沦为拐杖。但若用户将其作为训练场——先借助它完成任务，再逐步减少依赖——它就是脚手架。关键在于使用者的态度：工具是辅助，不是替代。

此外，目前缺乏针对Structured的独立随机对照试验（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），其效果主要基于概念对齐和用户反馈。ADHD的异质性也意味着单一工具难以覆盖所有亚型（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

## 今天就能试的行动

1. **用Structured的「无状态」模式处理一个复杂任务**：将一个多步骤项目（如写报告、整理房间）分解为独立的小步骤，每完成一步就标记为完成，不回头看整体进度。体验「只聚焦当前一步」如何降低焦虑。
2. **建立外部记忆仓库**：在Structured中创建一个「知识库」笔记，记录你常忘记的信息（如密码、流程、灵感）。每次需要时先查笔记，而不是靠大脑回忆。
3. **设置「上下文切换检查点」**：当你不得不中断任务时，在Structured中记录当前进度和下一步计划（类似LLM的会话摘要）。回来时先读检查点，再继续。
4. **反思依赖程度**：每周评估一次：如果没有Structured，我能否独立完成类似任务？如果答案是否定的，尝试逐步减少使用，培养内在能力。

## 结论

把ADHD大脑当成生成核心，Structured确实能充当它的无状态与外部记忆——但前提是用户将其视为脚手架而非拐杖。这种同构视角不仅为ADHD人群提供了新的理解框架，也为Agent工程师提供了人性化的参考：毕竟，我们每个人都在某种程度上需要外部调度层来管理自己的注意力与记忆。

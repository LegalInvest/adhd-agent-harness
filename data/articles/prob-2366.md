# ADHD 职场人视角：为什么「治好 ADHD 的分心走神或超聚焦跑偏，丢失目标」和「让 LLM 不跑飞」其实是同一道工程题？

> 长程任务里 agent 会目标漂移，需要重锚定 system prompt——同一套 harness 思路，两边都成立。

你有没有过这样的经历：打开电脑准备写一份报告，结果半小时后发现自己正在刷短视频，而文档里只多了两行标题？或者，你正沉浸在一个项目里，连午饭都忘了吃，但回过神来才发现——这件事根本不是当前最重要的。

如果你是 ADHD 职场人，这种「分心走神」或「超聚焦跑偏，丢失目标」几乎是日常。而如果你在做 Agentic Harness 工程，你可能更熟悉另一个场景：你精心设计的 LLM agent，在长程任务中逐渐偏离初始目标，开始生成无关内容，甚至发明自己的子目标。

这两件事，本质上是一道工程题。

## 同一个问题：生成核心强，执行调度弱

ADHD 大脑与 LLM 在架构上有着惊人的同构。两者都是「高产但缺执行调度层的生成核心」：ADHD 大脑能产生大量创意、直觉和能量，但工作记忆不稳定、任务启动困难、容易因认知负荷超载而崩溃（来源：认知负荷）；LLM 同样能流畅生成文本，但无跨会话状态、上下文膨胀后推理退化，且无法自主抑制无关信息（来源：Deficient Executive Control in Transformer Attention）。

最新研究用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中表现正常，但随序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突。这与 ADHD 执行功能缺陷的核心标志一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃（来源：Deficient Executive Control in Transformer Attention）。

所以，ADHD 的「分心走神」和 LLM 的「目标漂移」是同一个问题：**生成核心缺少一个可靠的内置调度层来持续锚定目标**。

## 同一个解法：外部重锚定机制

既然问题同构，解法也同构。对 ADHD 大脑，解决方案是「外部执行功能层」——一套脚手架系统，帮助补偿工作记忆、降低认知负荷、提供任务启动和维持的锚点。对 LLM/Agent，解决方案是「重锚定 system prompt」——在长程任务中定期注入原始目标、约束上下文、重置注意力焦点。

历史上最经典的 ADHD harness 案例之一，是晚清名臣曾国藩。他 30 岁前典型注意力不集中：浮躁坐不住，天天串门喝酒看杀人，日记里天天骂自己“无恒”“浮躁”；读书极慢，“他人目下二三行，余或疾读不能终一行”。他的专属 harness 是「日课十二条」：黎明即起、读书不二、谨言、写日记反省……用最笨最稳的方法抵消自己的冲动（来源：人物案例）。这套系统本质上就是一个**定时重锚定协议**：每天用固定仪式把注意力拉回核心目标，与 LLM agent 的「每 N 步重注入 system prompt」完全同构。

再看现代工具：Focusmate 通过虚拟身体在场提供外部问责，利用算法匹配伙伴，创造「AI-Matched Body Doubling」会话（来源：Focusmate）。这相当于给 ADHD 大脑装了一个**外部调度器**——就像为 LLM agent 配一个「人在回路」的监督者，定期检查是否跑偏。RescueTime 自动追踪时间并阻断分心网站，直接补偿时间盲与执行功能缺陷（来源：RescueTime），相当于 LLM 的**上下文窗口管理**——自动清理无关信息，保持注意力集中。Brain.fm 利用 AI 生成神经锁相音乐，帮助进入专注状态（来源：Brain.fm），类似为 LLM 设置**温度参数**——调节生成模式的随机性，使之更聚焦。

这些工具的共性：**不改变核心生成能力，而是提供外部结构来补偿调度缺陷**。ADHD 大脑不需要「被治好」，它需要 harness；LLM 不需要「更聪明」，它需要更好的 prompt engineering。

## 脚手架 vs 拐杖：依赖与内化的边界

但这里有一个必须诚实的争议：AI 工具是否会导致永久依赖，而无法内化执行功能？（来源：矛盾与存疑）曾国藩的日课坚持了一辈子，从未「内化」到无需日记的程度——他每天写反省，直到去世前三天。这说明对于某些人，外部脚手架可能是终身需求，而不是「临时拐杖」。

关键区别在于：**脚手架支持能力成长，拐杖替代能力**。一个好的 harness 应该让用户逐渐减少对单一工具的依赖，比如从依赖 Focusmate 的每场配对，到学会自己设定定时检查点；从依赖 RescueTime 的自动阻断，到能主动识别分心模式。对 LLM agent 而言，好的 system prompt 应该让模型学会在长程任务中自我检查，而不是每步都靠外部注入。

## 争议与局限

必须承认，ADHD 大脑与 LLM 的同构目前更多是功能类比，而非经过验证的神经机制对应（来源：矛盾与存疑）。现有工具如 Brain.fm 缺乏 ADHD 人群的独立临床证据（来源：Brain.fm），Focusmate 的效果主要依赖用户报告而非随机对照试验。此外，ADHD 的异质性意味着同一工具对不同亚型效果迥异，现有研究未充分分层（来源：矛盾与存疑）。

但即便作为工作假设，这一同构视角已经催生出有效的工程实践。它让我们看到：ADHD 不是「缺陷」，而是一种架构选择——生成优先、调度外包。

## 今天就能试的行动

1. **设置「重锚定」闹钟**：每 25 分钟响一次，问自己「我现在做的事，是当前最重要的目标吗？」——这是最简版的 system prompt 注入。
2. **用 Focusmate 或找一位身体加倍伙伴**：每天固定时段与他人视频共作，外部问责比意志力可靠。
3. **用 RescueTime 或类似工具自动追踪时间**：每周看一次报告，识别注意力漂移模式，然后调整环境。
4. **尝试微任务分解**：用 Goblin Tools 的「魔法待办」将复杂任务拆成 5 分钟步骤，降低启动阻力。

ADHD 职场人与 Agent 工程师，你们面对的是同一道题：如何让一个强大的生成系统，在长程任务中不跑飞。答案不在内部，在 harness。

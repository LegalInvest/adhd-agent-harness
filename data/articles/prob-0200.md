# 如果把 ADHD 大脑当成生成核心，Inflow 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Inflow 对 ADHD 的真正价值。

## 问题：为什么你边做边忘，而你的大脑却停不下来？

如果你是一个 ADHD 患者，你一定熟悉这个场景：你坐在桌前，脑子里同时冒出三个想法——回复邮件、查资料、写方案——然后你打开邮件，读到一半，突然想起要查资料，打开浏览器，看到一条新闻，点进去，十分钟后你发现自己正在看一个完全无关的视频，而邮件和方案都还躺在那里。你并不是懒，你的大脑在高速运转，只是它像一个没有司机的跑车，油门踩到底，方向盘却没人握。

同样的事情每天都在 LLM agent 工程师的调试终端里发生。你写了一个 prompt，让模型生成一篇报告，它洋洋洒洒写了两千字，却在第三段开始跑题，引用了一个不存在的来源，然后完全忘记了最初的要求。LLM 拥有惊人的生成能力，但一旦上下文窗口超过某个阈值，它的注意力就开始弥散，就像 ADHD 大脑一样——耶鲁大学的研究发现，随着 N-back 任务 N 值增加，Transformer 的注意力分数熵增、注意力弥散，表现下降，与人类注意分散机制同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。LLM 的工作记忆容量并非无限，它同样需要外部记忆来续命。

## 同构：两个高产生成核心，同一个瓶颈

ADHD 大脑与 LLM 在结构上是同一类系统。底层都是一个高产的生成核心——ADHD 大脑联想丰富、创意不断，LLM 则能瞬间生成海量文本；但两者都缺乏可靠的内置执行调度层。ADHD 患者的工作记忆容量小，边做边忘，难以维持任务目标；LLM 则没有跨会话状态，无法自主保持长程目标，容易在生成过程中偏离轨道。

明尼苏达大学系统测试 LLM 的执行功能，发现了一个与 ADHD 惊人一致的剖面：LLM 的工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷，无法灵活切换任务集、无法抑制自动化反应（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这直接支持了“瓶颈不在容量/算力，而在 orchestration 层”这一论点。ADHD 患者不是智商低，LLM 也不是算力不够，两者的问题都在于：生成核心太强，而调度层太弱。

因此，ADHD 与 AI 工具领域的核心论点站住了：AI 并不是在「治愈」ADHD，而是在 ADHD 大脑与外部世界之间搭建一个临时的外部执行功能层（来源：ADHD 的 AI 工具全景）。这恰恰是 LLM agent 工程的核心——用外部脚手架（harness）把生成能力约束到目标轨道上。Inflow 正是这样一个脚手架。

## Inflow：ADHD 的外部记忆与调度器

Inflow 是一款专为 ADHD 设计的 AI 工具，它的核心功能是作为外部记忆与任务调度系统。它通过结构化的每日课程、习惯追踪和反思提示，帮助用户建立稳定的执行节律。这与 LLM agent 工程中的“外部记忆+定时 re-grounding”模式完全同构。

在 LLM agent 中，工程师会为模型配备一个向量数据库作为外部记忆，让模型在每次生成前检索相关上下文，避免遗忘。同时，通过定时触发（如每轮对话后自动总结）来重新对齐目标。Inflow 做的正是类似的事：它通过每日签到、任务清单和反思模板，让用户定期“re-ground”到自己的优先事项上，而不是被随机冲动带走。

这种脚手架的价值在历史人物身上早有体现。孔子，这位身高近一米九、精力旺盛的儒家创始人，很可能就是一个 ADHD 患者。他周游列国十四年坐不住，冲动爱骂人，见南子急得对天发誓，骂宰予“朽木不可雕”；但他对音乐极度专注，三月不知肉味，对种地等俗事零耐心。他的思维跳跃，《论语》全是场景化语录，没有系统著作。孔子的 harness 系统是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。他七十岁才做到“从心所欲不逾矩”，一辈子都在和自己的冲动做斗争。这套系统本质上就是一个外部调度器：用固定的礼仪规则（相当于 agent 的约束 prompt）和每日反思（相当于定时 re-grounding）来约束自己的生成核心。孔子成就了儒家学派，万世师表，影响中国文化两千五百年——这说明，脚手架不是拐杖，而是成就的基石。

## 工具生态：从任务分解到身体在场

Inflow 并非孤例。整个 ADHD 工具生态都在做同一件事：为生成核心搭建外部执行功能层。

- **Goblin Tools** 的 Magic ToDo 功能，通过 AI 将模糊任务（如“清理厨房”）分解为具体子步骤，用户可调节“辣度”控制粒度（来源：Goblin Tools）。这相当于 LLM agent 中的任务规划器——将复杂目标拆解为一系列可执行的子任务。
- **Saner.AI** 专注于知识回忆和本地记忆，减少搜索循环和标签切换（来源：Saner.AI）。这正是 LLM 外部记忆的翻版：用向量库缓存信息，避免重复计算。
- **Lex** 允许用户通过单一指令触发多步骤任务序列，其底层架构类似于 agent scaffolding（来源：Lex）。这相当于 agent 中的“宏命令”——一次触发，自动执行整个工作流。
- **Focusmate** 利用算法匹配虚拟身体加倍伙伴，通过社会存在感降低分心阈值（来源：身体在场效应）。这对应 agent 工程中的“监控器”——一个外部观察者，确保 agent 不偏离轨道。

这些工具的共同点在于：它们不是试图修复大脑，而是把大脑的生成能力接入一个可维护的执行调度系统。正如 ADHD 工具全景所总结的：“这些工具的价值不在于替代大脑，而在于把 ADHD 的「高产生成核心」接入一个可维护的「执行调度系统」。”（来源：ADHD 的 AI 工具全景）

## 脚手架 vs 拐杖：诚实面对局限

然而，我们必须承认争议。核心论点是“AI 作为外部执行功能层”，但存疑是：过度依赖可能导致能力退化，成为永久拐杖（来源：矛盾与存疑）。长期依赖风险缺乏实证，需更多纵向研究。同样，ADHD 大脑与 LLM 同构的论点目前多为类比推理，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。这是一个理论框架，而不是定论。

我的判断是：脚手架与拐杖的边界在于使用者的意图。如果你用 Inflow 来替代自我觉察，永远不反思自己的行为模式，那它就是拐杖；如果你用它来辅助建立自己的执行系统，逐渐内化那些外部结构，那它就是脚手架。孔子七十岁才“从心所欲不逾矩”，说明内化需要时间。同样，LLM agent 工程师不会永远依赖同一个 prompt 模板，他们会不断优化，最终让模型学会自我约束。关键是过程，而不是工具本身。

## 今天就能试的行动

1. **用 Inflow 的每日签到作为外部 re-grounding**：每天早上花 5 分钟，让 Inflow 帮你列出今天最重要的三件事，然后关掉其他干扰。这相当于给 LLM 设置一个 system prompt，锁定目标。
2. **尝试 Goblin Tools 的 Magic ToDo**：把你觉得“无法启动”的任务（比如“写周报”）输入进去，让它拆成小步骤，然后只做第一步。这是 agent 的任务分解，降低启动门槛。
3. **找一个虚拟身体加倍伙伴**：用 Focusmate 或找一位朋友，每天固定时间一起工作。社会存在感能提升目标达成可能性（来源：身体在场效应）。这相当于 agent 的监控器，让你不跑偏。
4. **用 Saner.AI 捕获灵感**：随时随地记录想法，让 AI 自动组织。这相当于给你的大脑加一个外部向量库，防止工作记忆溢出。

ADHD 大脑不是缺陷，而是一个未经调校的高产生成核心。Inflow 这类工具不是要把它修成普通大脑，而是给它配上一个外部调度系统。正如 LLM 需要 agent scaffold 才能完成复杂任务，ADHD 大脑也需要一个 harness 来释放它的真正潜力。脚手架不是拐杖，它是你起飞前的跑道。

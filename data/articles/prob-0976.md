# Focusmate 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Focusmate：它到底补上了哪一层执行功能？

## 问题：为什么你明明知道该做什么，却就是启动不了？

ADHD 大脑有一个经典困境：你坐在书桌前，计划写一份报告，但手却不由自主地刷起了手机。或者，你一头扎进某个感兴趣的子任务，三小时后才惊觉——原本的目标早已被忘在脑后。这种“分心走神”或“超聚焦跑偏”的本质，是**目标漂移**：你的大脑像一台高产的生成器，不断产出想法和冲动，但缺乏一个可靠的“执行调度层”来持续锚定当前目标。

类似的问题也困扰着 AI agent 工程师。当 agent 执行长程任务时，随着上下文增长和工具调用增多，它会逐渐偏离初始目标，产生幻觉或重复工作。解决方案之一，是**重锚定 system prompt**——定期让 agent 重新读取原始指令，把注意力拉回正轨。

Focusmate，这个基于虚拟身体在场的在线问责服务，对 ADHD 的作用，恰好与“重锚定 system prompt”同构。但许多人用错了它：把它当成一个“陪伴工具”来缓解孤独，而忽略了它真正的价值——作为**外部执行功能层**，在关键时刻执行一次目标重锚定。

## 同构：ADHD 大脑与 LLM/agent 共享同一个核心矛盾

ADHD 大脑与 LLM 在结构上高度同构：两者都是**高产但缺乏可靠执行调度层的生成核心**（来源：AI 与 ADHD 的专注力，主题综述）。ADHD 的创意发散、LLM 的文本生成，都是强大的“生成”能力；但 ADHD 的执行功能缺陷（如工作记忆差、时间盲）、LLM 的上下文控制不足（如受上下文膨胀干扰），导致输出不稳定。

这种同构在多个层面成立：
- **无状态与外部记忆**：ADHD 的工作记忆缺陷与 LLM 的无状态性对应，都需要外部记忆系统（如第二大脑、向量数据库）来维持连续性（来源：同构视角）。
- **上下文工程**：ADHD 易被环境带偏，LLM 易被上下文膨胀干扰，都需要主动设计“当下注意什么”的工程方案（来源：同构视角）。
- **验证循环**：ADHD 的冲动行为可视为行为层面的“幻觉”，缺乏外部验证步骤；LLM 的幻觉需要通过验证循环（如测试用例、输出检查）来校正（来源：幻觉与验证循环）。

因此，AI 帮助 ADHD 的本质，是**给生成核心套上 harness（外部执行功能层）**（来源：核心论点）。而 Focusmate 正是这样一个 harness 组件。

## Focusmate 补上的那一层：重锚定

Focusmate 的核心机制是“AI-Matched Body Doubling”——通过算法匹配一对伙伴，在约定时段内通过视频共同工作，互相监督进度（来源：Focusmate 工具页）。它提供的不是陪伴，而是**外部问责**：当你告诉伙伴“这 50 分钟我要写完大纲”，你就为自己设定了一个临时目标锚点。伙伴的存在，相当于一个外部调度器，在你走神时（“我刚刚在干嘛？”）或超聚焦跑偏时（“这个细节值得再挖一小时吗？”）把你拉回目标。

这恰好对应 LLM/agent 工程中的“重锚定 system prompt”。在长程任务中，agent 会因上下文增长而逐渐遗忘初始目标，工程师会设计定时或触发式的“re-grounding”步骤——让 agent 重新读取原始 system prompt，或在每一步执行后检查是否偏离目标（来源：幻觉与验证循环）。Focusmate 的 50 分钟时段，就是一次定时重锚定。

历史人物中，曾国藩的“日课十二条”是极佳的 ADH D harness 案例。曾国藩是典型的注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书极慢，“他人目下二三行，余或疾读不能终一行”（来源：人物案例）。他的 harness 系统——黎明即起、读书不二、谨言、写日记反省——本质上就是一组**定时重锚定**：每天黎明即起是重置系统，读书不二是单任务模式，写日记反省是每日复盘（验证循环）。与 LLM harness 对比，日课相当于定时执行的“re-grounding 脚本”，日记相当于“trace log”。

辛弃疾则是另一种 harness：他把报国无门的愤懑全部注入写词和练剑，用“词”和“剑”作为注意力锚点，把超聚焦能量导向创作（来源：人物案例）。这对应 LLM 中的“角色设定”或“风格 prompt”——通过固定输出格式来约束生成方向。

## 但有人用错了：脚手架 vs 拐杖

Focusmate 的常见误用，是把“身体在场”本身当作目的。用户依赖伙伴的存在才能工作，一旦没有伙伴就彻底瘫痪。这种依赖，就是**拐杖**——外部执行功能层变成了永久替代，而非训练工具。

真正的**脚手架**，是让用户在使用中逐渐内化执行功能。Focusmate 的理想使用方式，不是每次都靠伙伴盯着，而是通过多次重锚定体验，学会自己识别目标漂移的信号，最终减少对外部问责的依赖。这与 LLM/agent harness 的设计原则一致：harness 是“外部约束”，但目标是让 agent 在长期运行中学会自我验证（来源：幻觉与验证循环中“验证与CI集成”）。

当前争议的核心正在于此：AI 工具作为外部执行功能层，长期使用是否会导致 ADHD 患者内在执行功能进一步退化？（来源：矛盾与存疑）。这需要更多纵向研究，但可以确定的是：**工具的使用方式比工具本身更重要**。如果你把 Focusmate 当作每次必服的“药”，它就是拐杖；如果你把它当作训练“目标觉察”的健身房，它就是脚手架。

## 局限与诚实

本文的同构框架仍有局限。ADHD 大脑与 LLM 的同构是隐喻而非严格等价，LLM 的“注意力机制”与人类注意力存在本质差异（来源：矛盾与存疑）。此外，Focusmate 的证据主要来自用户报告，缺乏随机对照试验（来源：Focusmate 工具页）。它的效果高度依赖配对伙伴的可靠性，且可能对社交焦虑用户造成压力（来源：Focusmate 局限）。

其他工具如 RescueTime（通过自动时间追踪提升时间觉察）、Brain.fm（通过神经锁相音乐辅助专注）也各有其用，但都面临证据不足的问题（来源：RescueTime、Brain.fm 工具页）。没有万能工具，只有适合特定短板和个体的 harness。

## 今天就能试的行动

1. **用 Focusmate 做一次“重锚定实验”**：预约一个 50 分钟时段，开始前写下唯一目标。过程中如果走神，就在白板上写“回来看目标”。结束后记录自己漂移了几次。
2. **建立你的“日课”**：模仿曾国藩，每天固定三个时间点（如起床、午休、睡前）做一次目标检查。可以用手机闹钟触发，每次 30 秒自问：“我现在该做什么？”
3. **为你的 LLM agent 增加重锚定**：如果你在做 agent 工程，在长程任务循环中插入一条“每 5 步重新读取 system prompt”的指令。观察目标漂移是否减少。
4. **区分脚手架与拐杖**：每周评估一次你对某个工具的依赖程度——如果离开它就无法启动，尝试减少使用频率，加入一次“裸奔”任务。

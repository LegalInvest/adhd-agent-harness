# ADHD 学生视角：为什么「治好 ADHD 的不擅长精确、重复、状态维护类任务」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 通过 function calling 把精确计算外包给工具——同一套 harness 思路，两边都成立。

你盯着一份截止日期就在今晚的论文。脑子里其实有想法，甚至能滔滔不绝讲十分钟论点，但一坐到电脑前，就被「核对参考文献格式、按页码排序、检查引用是否重复」卡住了。你感到羞耻：这些明明是小事，为什么自己就是启动不了？

与此同时，一个 AI 工程师正在调试一个 agent。模型输出的方案听起来很对，但只要让它自己算一笔账、查一次数据库、或者跨多轮记住用户刚刚改过的需求，它就会跑飞。工程师也感到烦躁：生成能力这么强，为什么就是不能老老实实执行？

这两个场景看似毫无关系，但指向同一道工程题：一个高产生成核心，缺少一个可靠的执行调度层。ADHD 的解法与 LLM/agent 的 harness 设计，是同构的。

## 一、共享的故障模式：不是不会想，而是调度层容易崩

ADHD 并不缺少想法、创意或信息检索能力，真正困难的是工作记忆、任务启动、时间管理与组织排序（来源：ADHD 的 AI 工具全景）。用神经科学的语言说，这是前额叶执行功能受损，难以稳定地计划、抑制分心、维持目标导向（来源：执行功能障碍）。情绪失调也会让情绪反应难以被认知调节，进一步放大状态波动（来源：情绪失调）。

LLM 的故障模式惊人地相似：它能高速生成文本，却没有内置的执行调度层，容易在对话中偏离目标或产生幻觉；上下文窗口有限，通常不具备真正的跨会话持久状态（来源：ADHD 的 AI 工具全景）。也就是说，它也不是一个「执行器」，而是一个需要被约束的生成器。

两边共同的弱项，恰好落在三类任务上：

- **精确**：核对、计算、格式化、一致性检查；
- **重复**：按步骤持续执行、不跳过、不返回重做；
- **状态维护**：跨时间、跨任务、跨会话记住目标、约束和进度。

ADHD 大脑在这些任务上耗竭，LLM 在这些任务上出错。问题不是核心不够聪明，而是缺一个外部调度层。

## 二、同构的 harness：四个补偿策略

如果两边同构，那么 ADHD 的 AI 工具与 LLM 的 agent harness 应该共享同一套结构。事实上，它们确实如此：

**1. 任务分解与启动**

ADHD 患者常见的瘫痪来自任务太大、入口太模糊。Goblin Tools 的 Magic ToDo 把「清理厨房」这类模糊目标拆解成可执行的小步骤，并允许用户调节「辣度」来控制粒度（来源：工具页：Goblin Tools）。这本质上是在外部为大脑补充一个「下一步是什么」的调度器。

LLM 这边，Lex 的核心设计是「单一指令触发多步骤任务序列」，其底层架构类似 agent scaffolding，也就是围绕模型搭建软件结构，使其能执行复杂、目标驱动的任务（来源：工具页：Lex）。复杂任务不再靠模型一次想完，而是被拆成子目标并顺序执行。同样是在外部补一个调度层。

**2. 外部记忆与状态维护**

ADHD 的工作记忆容量不稳定，信息在任务切换中容易丢失（来源：ADHD 的 AI 工具全景）。Saner.AI 用本地记忆、语义搜索和语音捕获，把不稳定的内部工作记忆外化为可检索的结构，减少「搜索循环」（来源：工具页：Saner.AI）。

LLM 没有真正的跨会话持久状态，因此工程上必须外接向量数据库、记忆模块或外部状态机。这与 ADHD 用笔记软件当「第二大脑」是同一操作：核心只负责生成，状态维护交给外部系统。

**3. 时间管理与提醒**

ADHD 的「时间盲」让未来任务变得抽象，需要外部提醒和可视化日程（来源：ADHD 的 AI 工具全景）。LLM agent 同样需要明确的调度器、定时触发和重试机制，否则会在长周期任务中遗忘约束。

**4. 元认知与反思支架**

ADHD 难以自发回顾与总结，AI 工具通过结构化提问帮助用户完成反思（来源：ADHD 的 AI 工具全景）。LLM agent 也需要在关键步骤插入 self-reflection 或 human-in-the-loop，防止输出在错误方向上继续发散。

贯穿这四个补偿的核心操作是「上下文工程」：通过外部支架主动限定「此刻应关注什么」（来源：ADHD 的 AI 工具全景）。对 ADHD 是限定当下的任务切片；对 LLM 是限定 prompt、工具链与状态。结构相同。

## 三、历史人物的 harness：孔子的「礼」与王阳明的「良知」

这个道理并不新鲜。历史人物早就用同构思路给自己搭过 harness。

孔子身高一米九，精力旺盛，周游列国十四年坐不住，冲动爱骂人，对音乐能超专注到「三月不知肉味」，对种地却毫无耐心，而且终生没有写出系统著作。他的 harness 是「克己复礼」：用外在的「礼」作为行为边界，每天「吾日三省吾身」，直到七十岁才做到「从心所欲不逾矩」。这本质上是一套外部约束加每日 re-grounding：用社会仪式和行为规范当 scaffolding，不断把漂移的注意力拉回核心目标。

王阳明同样典型：五岁不会说话，兴趣爆发式转移，格竹子格到吐血，曾因冲动上书骂刘瑾而被廷杖贬谪。他的 harness 是「致良知」与「知行合一」：以良知作为唯一的决策标准，坚持静坐反省，军旅中仍讲学，在「事上练」。这相当于给容易漂移的生成核心安装了一个外部价值评判器和持续反馈回路。

放在 LLM 工程里，孔子的「礼」类似于 prompt 模板、system instruction 和硬性约束规则；王阳明的「每日反省」则像 agent 的 reflection loop 和 human-in-the-loop 审核。他们都不是靠「让自己更自律」来解决问题，而是靠外部结构托住内部核心。

## 四、核心判断：这不是「补短板」，而是「外接执行层」

我的核心判断是：ADHD 和 LLM 都不需要被「治好」成一个全能核心。试图让 ADHD 大脑突然擅长精确重复任务，或让 LLM 自己学会稳定的状态管理，都是错的优化方向。正确的工程思路是承认核心有偏才，然后给它外接 harness：把精确计算、重复执行、状态维护外包给可靠工具。

对 ADHD 学生来说，这意味着接受「我需要外部工具替我完成调度」，而不是自责「我为什么连这么简单的事都做不好」。对工程师来说，这意味着把 agent 的 function calling 当成必备结构：任何需要精确、状态或持续一致性的操作，都应该调用确定性工具，而不是让模型自己生成。

但这里有一个关键边界：**harness 是脚手架，不是拐杖**。脚手架要可拆卸，最终目标是让核心在支架支持下逐步建立更稳定的工作模式；拐杖则是无限期替代内部功能。

ADHD 工具的风险在于过度依赖，导致用户不发展自身的执行功能（来源：全局矛盾与存疑）。LLM agent 的风险在于用 function calling 掩盖模型推理漏洞，把所有问题都外包，却不评估工具选择和结果验证。两边都需要保留一个「元认知层」：定期检查 harness 本身是否有效，而不是让工具替你思考。

# Speechify 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Speechify：它到底补上了哪一层执行功能？

你盯着屏幕上的 5000 字论文，手指悬在键盘上，大脑却一片空白。不是读不懂，而是“开始读”这个动作像推一堵墙。ADHD 的大脑在接收信息时，缺失的不是理解力，而是把“读”拆成“打开→扫视→理解→记录”这一系列子任务的执行调度。同样，一个 LLM agent 面对“帮我研究 AI 对 ADHD 的影响”这条指令，如果只靠一次生成，大概率会输出泛泛而谈的摘要——它需要被 harness 拆成“搜索→筛选→总结→引用”的 tool calls。

这就是本文的核心同构：**ADHD 大脑与 LLM 都是“高产但缺执行调度层的生成核心”，而工具（如 Speechify、Goblin Tools）就是那个外部调度层**。但有人用错了——把工具当拐杖，而不是脚手架。

## 问题：精确、重复、状态维护——ADHD 与 LLM 共同的软肋

ADHD 侧最折磨人的不是“不会做”，而是“做不下去”。以阅读为例：你需要维持对前文的状态（工作记忆），精确追踪论证逻辑（执行控制），重复扫视文字（注意力维持）——这三项恰好是 ADHD 的执行功能短板。wiki 资料指出，ADHD 的工作记忆“容量保留但控制弱（易被干扰）”（来源：ADHD 的 AI 工具全景）。

LLM/agent 侧同样如此。一个没有 harness 的 LLM 在长任务中会“上下文膨胀”，丢失早期指令，输出随机漂移——wiki 资料将这种非确定性描述为“生产环境中可靠性的敌人”（来源：采样温度与表现波动）。两者的共同困境是：**生成能力强，但缺乏稳定的调度层来管理精确、重复、状态维护型操作**。

## 答案：工具作为外部执行功能层——但必须是脚手架，不是拐杖

### 同构一：任务分解与启动

ADHD 侧：Goblin Tools 的 Magic ToDo 功能“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools）。用户输入“整理房间”，AI 输出“捡起地板上的衣服”“擦桌子”等步骤。这直接补偿了任务启动困难。

LLM/agent 侧：agent 通过“多步推理+工具调用”模式，将复杂指令分解为确定性步骤。wiki 资料强调，现代 agent 系统“组合多个模型、检索器和工具”来达成 SOTA 结果，而非依赖单一模型调用（来源：工具使用与认知卸载）。

**同构点**：两者都把“一个模糊的大任务”变成“一系列精确的小动作”。

### 同构二：记忆与上下文管理

ADHD 侧：Saner.AI 通过“知识回忆”和“本地记忆”减少搜索循环和标签切换，直接补偿工作记忆不足（来源：Saner.AI）。

LLM/agent 侧：agent harness 管理文件、处理长对话，提供持久化上下文。wiki 资料指出，外部记忆系统“本质上是为两者提供持久化上下文”（来源：ADHD 的 AI 工具全景）。

**同构点**：两者都需要一个外部存储器来防止“做到一半忘了之前做了什么”。

### 同构三：时间盲与冷启动

ADHD 侧：时间感知缺陷（时间盲）导致无法估算任务时长。Goblin Tools 提供时间估算功能。

LLM/agent 侧：LLM 的“冷启动”问题——初始提示不充分时输出质量差。Agent 通过“计划-执行”分离模式强制确定性控制流（来源：采样温度与表现波动）。

**同构点**：两者都需要外部时钟或计划器来补偿内在的时间/时序推理弱点。

## 核心观点：脚手架 vs 拐杖——工具使用的边界

工具是脚手架还是拐杖，取决于它是否**促进内在能力的发展**。wiki 资料在矛盾与存疑中明确警告：“过度依赖 AI 工具可能削弱内在能力”（来源：矛盾与存疑）。例如，Otter.ai 减轻笔记负担，但“过度依赖可能削弱主动记笔记的能力”（来源：ADHD 的 AI 工具全景）。

Speechify 这类文本转语音工具，如果用于“完全替代阅读”，就是拐杖——用户不再练习视觉追踪和文字解码，这些能力可能退化。但如果用于“辅助阅读”，比如在疲劳时作为启动的助推，让用户先听再读，就是脚手架——它降低了启动门槛，但保留了主动阅读的环节。

同样，LLM agent 的 function calling 如果完全替代人类的判断，就是拐杖。例如，让 agent 自动执行所有步骤而不经人类审核，会导致“小错误累积，非确定性使可重复性复杂化”（来源：采样温度与表现波动）。如果作为“建议生成器”，人类在关键节点做决策，就是脚手架。

**判断标准**：工具是否让你在“使用它”之后，做同一件事的能力变强了？如果是，是脚手架；如果变弱了，是拐杖。

## 局限与争议

必须诚实指出，当前证据主要来自用户报告和工具设计逻辑，缺乏大规模随机对照试验（来源：矛盾与存疑）。个体差异极大——对某些人有效的工具，对另一些人可能无效甚至有害。此外，多巴胺干预效果存在争议（来源：矛盾与存疑），而许多工具的设计恰恰依赖多巴胺奖励机制。

另一个争议是：AI 工具能否准确识别 ADHD 用户的意图？Lex 的“单一指令触发多步骤”看似美好，但“ADHD 的复杂性可能使 AI 难以准确识别用户意图”（来源：Lex）。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个你拖延已久的任务**：输入“写周报”，看它拆出的步骤。然后只做第一步。
2. **用 Saner.AI 记录一个你经常忘记的流程**：比如“报销步骤”，下次需要时直接搜索，而不是翻聊天记录。
3. **用 Speechify 听一篇文章，但听完后自己写一段摘要**：这确保你在使用工具的同时，仍主动加工信息。
4. **检查你常用的 AI 工具：它是在帮你“做”还是帮你“学会做”？** 如果答案是“做”，调整使用方式，留出自己动手的环节。

# Motion 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Motion：它到底补上了哪一层执行功能？

## 问题：为什么精确、重复、状态维护类任务让 ADHD 和 LLM 同时崩溃？

ADHD 大脑被描述为“高产但缺乏可靠执行调度层的生成核心”（来源：ADHD 大脑与 LLM 的同构）。它能迸发创意、连接跨领域想法，却在“今天下午三点给客户回邮件”“把厨房台面擦干净”这类简单任务前瘫痪。同样，LLM 能写出诗意的文章、解答复杂的数学题，但让它记住对话历史、按顺序调用多个 API 并验证结果时，它就会开始“幻觉”——自信地给出错误答案。

问题出在同一个结构缺口：**两者都缺少一个内置的执行调度层**。ADHD 的调度层是执行功能（工作记忆、任务启动、时间感知），LLM 的调度层是 agent harness（上下文管理、工具调用、验证循环）。当这个缺口没有被外部工具填补时，再强大的生成核心也只能原地打转。

## 同构视角：Motion 和 function calling 在补同一层

Motion 是一款 AI 日历与任务管理工具，它自动将任务排入日程，根据优先级和截止日期动态调整。它的核心价值在于**把“什么时候做什么”这个决策外包出去**——这正是 ADHD 大脑最不擅长的调度工作。

对应到 LLM/agent 侧，function calling 允许模型将精确计算（如数学运算、数据库查询）外包给外部工具，自己只负责意图理解和结果整合。Motion 做的正是同一件事：它接收模糊的意图（“下周完成报告”），然后调用日历、优先级、时间估计等“工具”，输出一个可执行的调度方案。

更深层的同构在于：**两者都要求用户放弃对执行过程的控制，信任外部系统**。ADHD 用户常因完美主义或“我这次一定能靠自己”的错觉而拒绝使用 Motion；工程师则因“我想控制每一步”而手写调度逻辑，拒绝使用成熟的 agent harness。结果是一样的：生成核心被琐碎的调度细节拖垮，产出质量下降。

## 真实案例：孔子与左宗棠的“外部 harness”

孔子（春秋/儒家创始人）是典型的 ADHD 大脑：身高 1.9 米，精力旺盛，周游列国 14 年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予“朽木不可雕”；对音乐超专注到“三月不知肉味”，对种地等俗事零耐心。他的 harness 系统是“克己复礼”——用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”）。这相当于一个**基于规则的 agent harness**：每天固定的反省步骤（re-grounding），用社会规范（context）约束冲动输出（action）。70 岁才做到“从心所欲不逾矩”，说明即使外部 harness 再强大，也需要终生维护。

左宗棠（晚清/民族英雄）是另一个例子：举人出身，脾气大，爱骂人，和曾国藩闹别扭；40 多岁才出山，69 岁抬棺出征收复新疆。他的 harness 是“结硬寨打呆仗”——每天读经写家书，用诸葛亮的“鞠躬尽瘁”要求自己。这对应 LLM harness 中的**结构化工作流和规划**：将复杂任务（收复新疆）分解为子任务（每天读经、写家书、扎营），并在每一步进行验证（“打呆仗”意味着不跳步）。

两人的 harness 都指向同一个原则：**外部脚手架必须被主动设计并持续维护**，而不是被动等待被“安装”后自动工作。

## Motion 的实测：它补上了哪一层？

我用 Motion 两周后，发现它确实补上了**任务级调度层**——把“什么时候做什么”这个决策从我的大脑中移除。但它没有补上**子任务分解层**和**执行验证层**。例如，我告诉 Motion“写一篇关于 ADHD 工具的文章”，它会在日历上分配 3 小时，但不会告诉我“先收集资料→再列提纲→再写初稿”。我仍然需要自己分解任务，否则 3 小时内我会在“开始写”和“查资料”之间反复切换，最终产出低质量草稿。

这正是 Goblin Tools 和 Lex 的用武之地：Goblin Tools 的 Magic ToDo 自动将模糊任务分解为具体步骤（来源：Goblin Tools），而 Lex 允许用单一指令触发多步骤序列（来源：Lex）。将 Motion 与这些工具组合，才能覆盖完整的执行功能层：**调度（Motion）+ 分解（Goblin Tools）+ 执行（Lex）+ 验证（自我检查或外部反馈）**。

## 争议与局限：脚手架 vs 拐杖

Motion 页面承认“缺乏独立临床验证”（来源：Motion），其他工具如 Brain.fm 也面临同样问题（来源：矛盾与存疑）。更关键的是依赖风险：如果用户完全依赖 Motion 安排每一天，一旦工具故障或需要手动调整，执行功能可能比之前更弱（来源：拐杖与脚手架）。孔子花了 70 年才内化 harness，左宗棠每天坚持读经写家书——外部系统必须与内部能力共同进化，而不是被替换。

此外，同构性本身是一个理论类比，并非科学事实（来源：矛盾与存疑）。ADHD 大脑的神经机制与 LLM 的架构差异巨大，类比只是为了提供设计思路，不能作为临床依据。

## 今天就能试的行动

1. **用 Motion 安排一件明天必须完成但你不愿启动的事**：观察它把任务排进日程后，你的启动阻力是否降低。如果有效，再增加第二件。
2. **用 Goblin Tools 的 Magic ToDo 分解一个模糊任务**：比如“整理房间”，看它生成的步骤是否让你觉得“这我可以做”。如果太细或太粗，调整“辣度”直到合适。
3. **尝试“单一指令触发”**：在 Lex 或自定义 AI 助手中，设置一条指令（如“开始每日复盘”），让它自动执行 3-5 个步骤（打开笔记→列出完成事项→写一条改进→关闭）。这相当于为你的大脑预装一个微 harness。
4. **每周一次“无工具日”**：选择一天不用 Motion、Goblin Tools 等调度工具，只靠大脑+纸笔。目的是保持内部执行功能的活性，避免完全依赖外部系统。

Motion 不是万能药，function calling 也不是。它们都是脚手架，帮你站得更高，但不会替你走路。真正的工程在于：设计一个与你的生成核心协同进化的 harness 系统，就像孔子和左宗棠那样，用一生的时间去迭代它。

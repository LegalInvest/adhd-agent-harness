# 如果把 ADHD 大脑当成生成核心，Goblin Tools 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Goblin Tools 对 ADHD 的真正价值。

你是否曾在打开浏览器准备写一篇文章后，两小时过去发现自己躺在 YouTube 的算法洞里？或者，你是否调试过一个 AI agent，发现它在第五步对话后开始胡言乱语，完全忘了最初的任务？这两种场景看似无关，却共享同一个底层问题：一个强大的生成核心，缺乏可靠的执行调度层。

ADHD 大脑与 LLM 在结构上同构：两者都是高产但无序的生成核心，拥有强大的联想与创造能力，却无法稳定地执行长程目标。ADHD 的注意力分散、思维跳跃并非缺陷，而是一种“注意力资源的弥散分配”（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）；LLM 的上下文膨胀与推理退化，则是其注意力机制在长序列下的固有局限。两者都需要外部脚手架来重锚定目标、防止漂移。

Goblin Tools——这款专为 ADHD 设计的 AI 工具——能否扮演这个重锚定者的角色？本文的回答是：可以，但前提是我们理解它作为“脚手架”而非“拐杖”的边界。

## 问题：目标漂移——ADHD 与 Agent 的共同敌人

ADHD 侧的核心痛点不是“不能专注”，而是“专注错了地方”。超聚焦（hyperfocus）可能让人沉浸于无关细节，分心走神则让任务碎片化。曾国藩在30岁前“天天串门喝酒看杀人”，日记里反复骂自己“无恒”“浮躁”，正是目标漂移的典型表现。他后来靠“日课十二条”——黎明即起、读书不二、写日记反省——构建了一套外部调度系统，用最笨最稳的方法抵消冲动（来源：wiki 资料）。这套系统本质上是一个“重锚定 protocol”：每天固定时间重新对齐目标。

LLM/Agent 侧同样面临目标漂移。在长程任务中，agent 会因上下文膨胀而遗忘初始指令，产生推理退化（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。工程师们通过“复合 AI 系统”来弥补——包括分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩——这些技术本质上是在 LLM 外部搭建调度层，防止它“跑偏”。

## 工具：Goblin Tools 如何充当重锚定器

Goblin Tools 的设计哲学与上述需求高度吻合。它提供“魔法器”（Magic To-Do）将模糊任务分解为具体步骤，用“正式器”（Formalizer）调整语气，用“判断器”（Judge）评估情绪。这些功能直接补偿 ADHD 的执行功能缺陷：任务启动困难（分解降低门槛）、时间盲（外部进度条）、情绪调节（语气调整）。更重要的是，它通过减少决策负担来保留认知资源——这正是 ADHD 个体需要的“减少决策、保留上下文、让下一步行动显而易见”（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。

与 Focusmate 的虚拟身体在场（通过算法匹配实现 AI-Matched Body Doubling，来源：The Best AI-Powered ADHD Productivity Tools in 2026）和 RescueTime 的自动化时间追踪（补偿工作记忆负担，来源：Revolutionizing ADHD Management with AI Assistants）相比，Goblin Tools 更专注于任务层面的结构化。它像曾国藩的“日课十二条”一样，是一个可执行的 protocol，而非仅仅一个提醒器。

## 同构：从日课到 Agent Scaffolding

曾国藩的“日课十二条”与 LLM agent 的 scaffolding 在结构上惊人相似：
- **定时 re-grounding**：日课要求“黎明即起”“读书不二”，相当于 agent 的定时上下文刷新，防止目标漂移。
- **外部记忆**：日记反省相当于 agent 的日志系统，记录偏差并调整。
- **决策简化**：“结硬寨打呆仗”相当于 agent 的惰性工具发现——只调用最可靠的策略，减少认知开销。

Desiree Tyler，一位具有 ADHD 特质的投资人，其成功部分归因于“Empathy; Creativity; Pattern recognition; Risk taking”等特质（来源：wiki 资料），但这些特质如果没有外部调度层，很容易变成冲动决策。她的 harness 系统可能包括团队、工具和流程——这正是 agent 工程中的“人在回路”与“外部调度器”的对应。

## 争议：脚手架还是拐杖？

Goblin Tools 的潜在风险在于过度依赖。如果用户永远需要工具来分解任务，而无法内化这种能力，那么工具就成了拐杖。这一矛盾在 ADHD × AI 领域普遍存在：AI 工具既可能补偿执行功能，也可能削弱其发展（来源：拐杖与脚手架）。现有证据多为用户报告，缺乏长期追踪研究来评估内化效果（来源：AI 与 ADHD 的专注力）。

此外，Goblin Tools 的临床证据有限。它未被列入 2026 年 44 款 ADHD 应用测试中的最佳 9 款（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026），而 Brain.fm 虽入选但同样缺乏 ADHD 特异性研究（来源：Brain.fm）。工具的效果高度个体化：对注意缺陷为主型与多动冲动型的效果可能迥异。

## 今天就能试的行动

1. **用“魔法器”分解一个拖延任务**：将一个模糊任务（如“写报告”）输入 Goblin Tools，将输出步骤贴到便签上，每完成一步就划掉。这相当于为你的大脑搭建一个临时调度层。
2. **设置定时 re-grounding**：每 25 分钟用手机闹钟提醒自己“当前目标是什么”，类似曾国藩的“读书不二”。
3. **对比工具 vs. 无工具**：选择一周用 Goblin Tools 分解任务，另一周仅用纸笔，记录目标漂移次数和完成率。这能帮你判断工具是脚手架还是拐杖。
4. **结合身体在场**：在 Goblin Tools 分解任务后，使用 Focusmate 预约一个时段，让外部问责强化执行。

Goblin Tools 不是万能药，但它精准地回应了 ADHD 大脑与 LLM 的共通需求：一个轻量、可执行的重锚定系统。它的真正价值不在于替代思考，而在于让思考能持续指向目标。

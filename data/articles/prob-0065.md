# 3 个 LLM 工程概念，彻底重写了我对 ADHD 想理解自己的大脑到底是怎么回事的理解

> LLM 是「高产但缺执行调度层的生成核心」 如何反向照亮 ADHD 的想理解自己的大脑到底是怎么回事。

## 问题：为什么你总是「知道该做什么，却做不到」？

如果你是一个 ADHD 成人，你一定熟悉这种体验：脑子里同时冒出十几个想法，每个都充满潜力，但当你试图抓住其中一个时，它们像滑溜的鱼一样溜走，最后你发现自己瘫在沙发上刷了两个小时手机。这不是懒惰，也不是意志力薄弱——你的大脑其实是一个「高产但缺执行调度层的生成核心」（来源：ADHD × AI 的科学与研究前沿）。

如果你是一个在做 Agentic Harness 的工程师，你同样熟悉这种体验：LLM 能写出优美的代码、生成惊艳的文案，但一旦让它自主完成多步任务，它就会忘记上下文、陷入循环、输出不可控。LLM 也是一个「高产但缺执行调度层的生成核心」（来源：The Anatomy of an AI Agent: Memory, Tools, Planning, and Execution Explained - DEV Community）。

这两个世界看似无关，却共享同一个核心结构问题。而理解这一点，彻底改写了我对自己大脑的认知。

## 概念一：无状态与外部记忆——你的大脑没有「持久化层」

ADHD 大脑的工作记忆容量有限且不稳定，典型表现是：每天需要重新回忆项目上下文，否则容易偏离轨道（来源：无状态与外部记忆）。你刚刚还在想怎么写报告，接了个电话就全忘了。这不是你记性差，而是你的大脑默认是「无状态」的——每次对话都是独立的，模型不保留跨会话的记忆（来源：The Anatomy of an AI Agent: Memory, Tools, Planning, and Execution Explained - DEV Community）。

LLM 同样如此。标准的 LLM 调用是无状态的，没有 harness 的 LLM 只是一个无状态的 API 调用（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。生产级 agent 系统通过外部记忆（如向量数据库）解决溢出问题，组合使用会话内上下文记忆、长期检索向量库和结构化知识库（来源：The Anatomy of an AI Agent: Memory, Tools, Planning, and Execution Explained - DEV Community）。

**同构启示**：ADHD 大脑需要一个「永远在线」的外部记忆体来补偿内部工作记忆的波动（来源：How AI Helps ADHD Brains Work Smarter, Not Harder）。工具如 Saner.AI 正是为此设计——它利用 AI 实现本地记忆存储和快速检索，减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。对工程师而言，这意味着你的 agent 系统必须配备持久化的外部记忆层。

## 概念二：上下文工程——主动管理「你当下注意什么」

ADHD 大脑的上下文管理面临两大挑战：易被环境带偏，时间感知扭曲（来源：上下文工程）。使用通用聊天机器人时，它们往往「增加另一个收件箱、另一个提示、另一个丢失线索的地方」（来源：上下文工程）。

LLM agent 同样面临上下文膨胀与推理退化：「有效的自主辅助需要严格的安全控制和高效的上下文管理，以防止上下文膨胀和推理退化」（来源：上下文工程）。

**同构启示**：两者都需要主动设计和管理当前所关注的信息范围。ADHD 个体可以使用 Routinery 等应用「创建例程步骤序列，并通过过渡提示引导你完成」（来源：上下文工程）。工程师则需要精心设计 prompt 和上下文窗口管理策略。

## 概念三：任务分解与启动——从「压倒性」到「可执行」

ADHD 个体常面临执行功能障碍，尤其是任务启动困难。Goblin Tools 的 Magic ToDo 功能能自动将任何任务分解为更小、更易管理的步骤（来源：AI Tools for Kids with ADHD: A Complete Guide for Parents...）。用户反馈称它「将压倒性的事情变成一系列不压倒性的事情」（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...）。

Motion 则更进一步：它自动根据任务、会议和截止日期创建并动态调整每日计划，消除「下一步该做什么」的决策负担（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。

**同构启示**：对 LLM agent 而言，任务分解同样是核心能力。一个没有分解能力的 agent 会试图一步完成复杂任务，导致失败。ADHD 大脑也需要同样的脚手架——将大目标拆解为小步骤，降低启动门槛。

## 核心判断：脚手架，不是拐杖

这些工具的共性在于：它们不是替代你的大脑，而是为你的生成核心套上 harness——提供外部执行功能层（来源：ADHD × AI 的科学与研究前沿）。但这里有一个关键边界：脚手架 vs 拐杖。脚手架是设计为可逐步撤除的临时支撑，而拐杖是永久依赖（来源：拐杖与脚手架）。

目前多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。长期使用 AI 工具是否会削弱内在执行功能？目前缺乏纵向研究（来源：ADHD × AI 的科学与研究前沿）。这是我们必须诚实面对的风险。

## 局限与争议

- **证据强度不足**：同构模型主要基于理论类比，需更多神经影像学和行为实验验证（来源：ADHD × AI 的科学与研究前沿）。
- **多巴胺干预争议**：基于多巴胺的干预效果存在争议，但多个工具声称通过神经科学原理改善执行功能（来源：矛盾与存疑）。
- **工具效果缺乏临床研究**：Goblin Tools、Saner.AI、Motion 等工具的有效性主要基于用户反馈，缺乏独立随机对照试验（来源：各工具页面）。

## 今天就能试的 3 件事

1. **建立外部记忆**：使用 Saner.AI 或类似工具，将日常任务、项目背景和偏好存储在一个「第二大脑」中，减少每天重新回忆的摩擦。
2. **分解一个压倒性任务**：打开 Goblin Tools 的 Magic ToDo，输入你最头疼的任务（如「整理房间」），让 AI 把它拆成 5 个以下的小步骤，然后只做第一步。
3. **主动管理上下文**：在开始一项任务前，用一句话写下「我现在要做什么」，并放在视线范围内。对 LLM 也一样——在 prompt 开头明确当前目标和约束。

理解大脑与 LLM 的同构，不是要让你觉得自己「像机器一样有缺陷」，而是让你看到：你的生成能力是宝贵的，你只是需要一个更好的 harness。而这个 harness，现在正在被工程师和 ADHD 社群共同创造。

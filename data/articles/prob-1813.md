# ADHD 职场人视角：为什么「治好 ADHD 的注意力容易被环境带偏」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 受上下文窗口内容左右，需要上下文工程——同一套 harness 思路，两边都成立。

## 从同一个痛点出发：你的注意力被环境带偏了吗？

如果你是 ADHD 职场人，你一定经历过这样的场景：打开电脑准备写报告，手机通知一闪，你点进去刷了十分钟短视频；再切回文档，已经忘了刚才的思路。你的注意力像一只灵敏的探测器，被环境中的每一个信号捕获。

如果你在做 LLM 应用工程，你一定也经历过类似的崩溃：精心设计的 prompt，因为对话历史里混入了一句无关的闲聊，模型就开始跑偏，输出变得不可控。你的 LLM 也像 ADHD 一样，被上下文窗口里的内容牵着鼻子走。

这两个困境，看似分属神经科学和 AI 工程两个领域，但本质上是同一道题：**一个高产的生成核心，缺乏可靠的执行调度层**。

## 同构：ADHD 大脑与 LLM 的底层结构

ADHD 大脑的核心特征之一是执行功能障碍——计划、组织、任务启动、注意力维持等执行功能效率低下（来源：执行功能障碍）。这不是态度问题，而是大脑的执行功能系统先天效率不足。ADHD 患者的大脑像一台高性能的生成引擎：创意多、联想快、产出潜力高，但缺一个调度层来把生成结果导向有序输出。

LLM 也是同样的结构。GPT、Claude 等模型在生成能力上极其强大，但它们本身是无状态的——每次推理只依赖当前上下文窗口中的内容，没有内在的优先级或长期记忆。这导致 LLM 极易被上下文中的噪声带偏，输出缺乏连贯性和可靠性。

这个同构关系并非空洞的类比。从概念上看，ADHD 的工作记忆缺陷与 LLM 的无状态性对应——两者都需要外部记忆来弥补（来源：无状态与外部记忆）。RescueTime 通过自动记录时间使用，充当 ADHD 的外部记忆，帮助用户克服时间盲（来源：RescueTime）。而对 LLM 来说，RAG 或向量数据库就是类似的外部记忆。

## 上下文工程：同一套 harness 思路

既然问题同构，解法也同构。ADHD 和 LLM 都需要一套外部脚手架（harness）来补偿调度层的缺失。这套脚手架的核心原则是：**主动管理上下文，而非被动承受环境干扰**。

### ADHD 侧的上下文工程

- **环境声音工程**：Brain.fm 通过 AI 生成音乐，利用神经锁相技术影响大脑处理信息的方式，帮助用户进入专注状态（来源：Brain.fm）。这本质上是在外部构建一个稳定的听觉上下文，屏蔽环境中的随机干扰。
- **身体在场工程**：Focusmate 通过 AI 匹配虚拟同伴，利用身体在场效应提供实时问责（来源：Focusmate）。这相当于为 ADHD 大脑提供了一个外部调度器，在任务启动和维持的关键节点上给予结构化的上下文提示。
- **时间追踪工程**：RescueTime 自动记录时间使用，生成可视化报告，帮助用户识别注意力漂移模式（来源：RescueTime）。这相当于为 ADHD 大脑提供了一个元认知的上下文日志。

### LLM/Agent 侧的上下文工程

- **上下文窗口管理**：工程师通过精心设计的系统 prompt、few-shot 示例和对话历史截断，确保 LLM 的上下文窗口只包含相关且高质量的内容。这与 Brain.fm 主动筛选听觉输入的逻辑一致。
- **编排框架**：LangChain、AutoGPT 等框架通过链式调用、工具调用和状态管理，为 LLM 提供结构化的执行流程。这类似于 Focusmate 的同伴调度——外部框架接管了任务启动和步骤推进。
- **记忆系统**：向量数据库、会话摘要等外部记忆机制，弥补 LLM 的无状态缺陷。这与 RescueTime 的外部记忆功能同构。

## 脚手架 vs 拐杖：一个必须划清的边界

然而，上下文工程并非万能。这里有一个关键争议：**过度依赖外部脚手架，是否会阻碍内在能力的发展？**（来源：拐杖与脚手架）

对于 ADHD 患者，长期依赖 Brain.fm 或 Focusmate 是否会让大脑的自我调节能力进一步萎缩？对于 LLM 工程，过度依赖编排框架是否会让模型失去自主推理的灵活性？

目前没有长期研究证明脚手架可以逐步撤除（来源：矛盾与存疑）。但一个合理的判断是：**脚手架的价值在于释放生成核心的产能，而非替代能力的培养**。ADHD 患者使用工具后节省的认知资源，可以用于学习策略性思维；LLM 工程中框架提供的结构化输出，可以用于训练更可靠的自我纠正能力。

## 局限与诚实面对

必须指出，当前多数 ADHD 专注工具缺乏针对该人群的独立临床研究（来源：矛盾与存疑）。Brain.fm 的神经锁相技术虽有神经科学原理，但效果未在 ADHD 人群中得到严格验证。Focusmate 的效果更多归因于身体在场效应，而非 AI 本身。RescueTime 的自动分类也可能不够精准，需要用户手动调整。

同样，LLM 工程的上下文工程方案也面临挑战：上下文窗口大小有限，长文本的注意力衰减问题尚未完全解决；编排框架可能引入额外的延迟和复杂度。

## 今天就能试的行动

1. **ADHD 职场人**：打开 Brain.fm 或类似专注音乐工具，在需要深度工作前设定 25 分钟专注时段，主动构建听觉上下文。
2. **LLM 工程师**：为你的 agent 设计一个“系统 prompt 模板”，包含角色定义、输出格式和禁止行为，并在每次对话开始时注入，防止上下文漂移。
3. **两者通用**：使用 RescueTime 或类似工具记录一周的时间使用，识别注意力漂移的高发时段，针对性地调整工作环境或 prompt 策略。
4. **尝试虚拟同伴**：如果你常拖延启动任务，预约一次 Focusmate 时段，体验外部调度如何弥补内在执行功能的缺失。

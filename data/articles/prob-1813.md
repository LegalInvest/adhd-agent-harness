# ADHD 职场人视角：为什么「治好 ADHD 的注意力容易被环境带偏」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 受上下文窗口内容左右，需要上下文工程——同一套 harness 思路，两边都成立。

## 引言：两个看似无关的苦恼

如果你是 ADHD 职场人，你一定经历过这种时刻：打开电脑准备写报告，却被弹出的新闻标题带走，半小时后发现自己正在刷购物网站。你骂自己“注意力不集中”，但更准确的说法是：你的注意力被环境轻易带偏了。

如果你是 LLM 工程师，你一定也经历过这种时刻：精心设计的 Agent 在简单任务上突然“跑飞”，输出一堆无关内容。你检查日志，发现是上下文窗口里混入了某个干扰 token，导致模型“分心”。

这两个场景，一个关乎人类大脑，一个关乎人工智能，但背后的工程问题惊人地相似。本文要论证的核心观点是：ADHD 的注意力容易被环境带偏，与 LLM 受上下文窗口内容左右，本质上是同一道工程题——都需要通过“上下文工程”来搭建外部执行调度层。

## 同构：高产但缺调度层的生成核心

ADHD 大脑与 LLM 的深层同构体现在多个层面（来源：AI 与 ADHD 的专注力）。首先，两者都拥有强大的生成能力：ADHD 患者在创意发散、联想和灵感迸发时往往表现惊人，LLM 在文本生成、创意写作上同样出色。但两者都缺乏可靠的执行调度层：ADHD 的执行功能（计划、组织、任务启动）效率低下，LLM 的上下文控制机制脆弱，容易被无关信息干扰。

其次，ADHD 的工作记忆缺陷与 LLM 的无状态性同构。ADHD 患者常常“转头就忘”，需要外部记忆系统（如笔记、提醒）来维持连续性；LLM 每次交互都是独立的，需要向量数据库或检索增强生成（RAG）来提供长期记忆（来源：AI 与 ADHD 的专注力）。

最后，也是最关键的：ADHD 易被环境带偏，LLM 易被上下文膨胀干扰。ADHD 患者面对开放办公室的噪音、手机通知、甚至窗外的鸟鸣都可能分心；LLM 在长上下文推理时，早期信息会被后续内容稀释，导致“注意力漂移”。两者都需要主动设计“当下注意什么”的工程方案——这就是上下文工程（来源：AI 与 ADHD 的专注力）。

## 解法同构：Harness 而非拐杖

既然问题同构，解法也应同构。为 LLM 搭建 harness 的核心思路是：控制上下文窗口大小、设计结构化提示、引入外部记忆。同样，为 ADHD 大脑搭建 harness 也需要外部执行功能层：提供任务启动的锚点、时间感知的仪表盘、注意力维持的约束。

### ADHD 侧的 harness 工具

**Focusmate** 提供了一个绝佳的例子。它通过 AI 匹配虚拟伙伴，利用“身体在场效应”提供实时问责（来源：Focusmate）。这相当于为 ADHD 大脑增加了一个外部调度器：当你知道有人看着你工作，启动任务的阻力大大降低。多项用户研究显示其能显著提升任务启动和维持（来源：AI 与 ADHD 的专注力），但缺乏随机对照试验——这是证据的局限。

**RescueTime** 则解决了时间盲问题。ADHD 患者常无法准确感知时间流逝，RescueTime 自动记录时间使用并生成报告，相当于一个外部时间感知模块（来源：RescueTime）。它减轻了工作记忆负担，帮助用户识别注意力漂移模式。但自动分类可能不精准，且过度关注数据可能引发焦虑（来源：RescueTime）。

**Brain.fm** 利用神经锁相技术生成音乐，试图直接影响大脑状态（来源：Brain.fm）。这类似于为 LLM 添加一个“注意力滤镜”——但针对 ADHD 的效果缺乏独立临床研究（来源：Brain.fm）。

### LLM 侧的 Harness 技术

LLM 工程师的上下文工程手段包括：
- **限制上下文窗口**：只输入最相关的信息，避免噪声干扰。
- **结构化提示**：用明确的指令和格式引导模型输出，如“请按以下步骤思考：1. 分析问题；2. 列出选项；3. 选择最佳方案”。
- **外部记忆**：使用 RAG 或向量数据库存储长期信息，每次只检索当前需要的片段。

这些技术本质上和 Focusmate 的实时问责、RescueTime 的时间记录、Brain.fm 的神经锁相是同一类操作：为生成核心提供一个外部调度层，确保它专注于当前任务。

## 脚手架 vs 拐杖：边界在哪里？

这里必须诚实指出争议。AI 工具作为外部执行功能层，长期使用是否会导致 ADHD 患者内在执行功能进一步退化？当前研究警告“过度依赖可能削弱内在能力”（来源：矛盾与存疑）。同样，LLM 如果过度依赖外部记忆，其本身的推理能力是否会萎缩？

关键区别在于：**脚手架**是暂时支撑，帮助用户逐步建立内在能力；**拐杖**是永久替代，导致能力退化。Focusmate 的设计更接近脚手架——它通过社会压力激发用户自身的启动能力，而不是代替用户思考。RescueTime 则介于两者之间：自动记录减少了手动追踪的负担，但如果用户从不主动反思数据，就可能沦为拐杖（来源：矛盾与存疑）。

目前，大多数工具缺乏如何避免依赖的具体指导（来源：矛盾与存疑）。作为用户，我们需要有意识地使用这些工具：定期评估是否还需要它们，尝试逐步减少依赖。作为工程师，我们需要在工具中内置“训练模式”，帮助用户内化技能。

## 结语

ADHD 大脑和 LLM 并非完美等价——人类注意力涉及情感、多巴胺和神经递质，与 LLM 的注意力机制有本质差异（来源：AI 与 ADHD 的专注力）。但同构视角提供了一个强有力的工程隐喻：与其对抗大脑的“缺陷”，不如像工程师一样设计外部系统。当你下一次被环境带偏时，不要自责，而是问自己：“我的上下文工程哪里出了问题？”

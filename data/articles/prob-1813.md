# ADHD 职场人视角：为什么「治好 ADHD 的注意力容易被环境带偏」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 受上下文窗口内容左右，需要上下文工程——同一套 harness 思路，两边都成立。

## 问题：为什么注意力总在不该跑的时候“跑飞”？

早晨你打开电脑，准备写一份周报。两小时后，你发现自己正在查唐朝书法家的生平、调试一个不相干的代码库、回复了十二封其实可以晚回的邮件。对 ADHD 职场人来说，这并不陌生：注意力不是“不够”，而是被环境里每一个更亮、更新、更紧急的“上下文”拉走。

对 LLM / Agent 工程师来说，这一幕也熟悉得令人心碎。一个 Agent 原本要调用 API 总结用户请求，对话轮次一多，它就开始重复用户的抱怨、引用过期上下文、调用错误工具，最终输出偏离目标。ADHD 大脑与 LLM 都是“高产但缺乏可靠执行调度层的生成核心”（来源：主题综述：AI 与 ADHD 的专注力）。两边的“跑飞”症状，指向同一道工程题：**上下文工程**。

## 同一套底层结构：生成核心 + 缺失的调度层

ADHD 的核心短板不是智商或动机，而是**执行功能**：工作记忆、时间觉察、任务启动、抑制控制。这些功能决定了“在无数可注意的东西中，当下该注意什么”。LLM 的短板 similarly：它有强大的文本生成能力，但没有一个内置的“项目经理”来决定上下文窗口里哪些信息该被抑制、哪些该被强化、目标何时需要重新锚定。

因此，两边的解法结构同构：都需要一个**外部执行功能层**（external executive function layer）——或叫 harness、脚手架——来接管调度、记忆、启动与约束。AI 工具对 ADHD 的有效性，正取决于它是否精准对应这些具体短板，例如工作记忆、时间盲、任务启动（来源：主题综述：AI 与 ADHD 的专注力）。

对 LLM 来说，这对应的就是上下文工程：结构化提示、外部记忆系统、注意力锚点、定期 re-grounding、输出 guardrails。两套语言的词汇不同，但工程结构几乎一一对应：

- **工作记忆缺陷** ↔ **LLM 无状态性**：都需要外部记忆（第二大脑、向量数据库、运行日志）。
- **时间盲** ↔ **上下文窗口遗忘**：都需要外部时间/轮次追踪（RescueTime、Agent 的 step counter）。
- **任务启动困难** ↔ **初始 prompt 漂移**：都需要身体在场或强约束启动（Focusmate、system prompt 锁定）。

## 证据：当上下文膨胀，两者都会“崩溃”

这并不只是修辞。一项用经典 Stroop 冲突任务测试 Transformer 注意力的研究发现：在短上下文里模型表现正常，但序列变长、认知负荷增加后，模型在冲突试次上骤降到随机水平——它无法抑制优势反应、无法解决认知冲突。这与 ADHD 执行功能缺陷的核心标志几乎一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃（来源：Deficient Executive Control in Transformer Attention）。

ADHD 侧的证据同样明确。执行功能教练 Yulia Rafailova 指出，AI 可以“外化我们的思维，作为一个支架，显著减轻工作记忆上的认知负荷”（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）。工具如 **RescueTime** 通过自动记录时间、阻断分心网站，帮助 ADHD 用户对抗“时间盲”——它把“时间到底去了哪里”这种原本需要工作记忆持续监控的任务，外化成后台运行的日志（来源：Revolutionizing ADHD Management with AI Assistants）。**Focusmate** 则利用“身体在场效应”，把独自启动任务的困难，转化为实时同伴问责的结构，多项用户研究显示它能提升任务启动与维持（来源：11 Best ADHD Productivity Apps for Fluctuating Energy）。

对 LLM 工程师来说，这些工具对应的技术清单是：system prompt 固化目标、RAG/向量数据库外部化记忆、tool call 白名单限制行动空间、周期性 re-grounding 抑制 drift。两边都在做同一件事：**用外部结构补偿内部调度层的不足**。

## 原型：曾国藩的“日课十二条”是最早的 harness

要理解这种 harness，不必只盯着 App。晚清曾国藩就是一个人类 harness 的极端样本。他 30 岁前浮躁坐不住，天天串门、喝酒、看杀人，日记里反复骂自己“无恒”“浮躁”；读书极慢，“他人目下二三行，余或疾读不能终一行”；一辈子受严重银屑病和情绪不稳困扰，打败仗就跳水自杀（来源：名人 harness 曾国藩）。

他的解法不是“变得更自律”，而是给自己设计了一套**外部脚手架**：日课十二条。黎明即起、读书不二、谨言、每日写日记反省，打仗则“结硬寨打呆仗”，用最低方差的最笨方法抵消冲动。这套系统与今天的 LLM harness 惊人同构：

- **“日课十二条”** = **system prompt + 定时 re-grounding**：每天固定条目重新把目标写回上下文。
- **“写日记”** = **外部记忆 / 运行日志**：把反思外化，避免工作记忆被情绪冲刷。
- **“谨言”** = **输出 guardrails**：在说话/行动前加一层抑制过滤。
- **“结硬寨打呆仗”** = **低 temperature 解码 / 确定性规划**：不依赖临场灵感，用结构化流程降低方差。

曾国藩没有“治好” ADHD，但他用一生的 harness 把高产生成核心导向了平定太平天国、推动洋务运动。对 LLM 工程师来说，他的故事说明：**问题的关键不是让生成核心变强，而是让调度层足够可靠**。

## 从脚手架到拐杖：必须守住的那条边界

但 harness 有一个危险边界：**它可能从脚手架变成永久拐杖**。主题综述中的“矛盾与存疑”部分明确警告：AI 工具作为外部执行功能层，长期使用是否会导致内在执行功能进一步退化，目前缺乏纵向实证；个体差异大，证据质量也多是小样本或用户报告（来源：主题综述：AI 与 ADHD 的专注力）。

我的判断是：好的 harness 应该满足两个条件——**可装卸**，且**逐步把部分调度能力内化**。对 ADHD 用户来说，这意味着你可以用 Focusmate 启动任务，但要同时训练自己识别“启动困难”的预警信号；可以用 RescueTime 看时间，但要慢慢建立对 45 分钟块的主观感知。对 LLM 系统来说，这意味着不要把所有希望都押在 prompt engineering 上：系统提示、外部记忆、guardrails 是脚手架，但模型本身的推理可靠性、对齐能力、元认知能力才是最终要建设的内部调度层。

还要注意：ADHD 与 LLM 的同构是**理论框架与隐喻**，不是严格的神经科学或计算机科学等价（来源：主题综述：AI 与 ADHD 的专注力）。Transformer 的“注意力机制”与人的注意力机制在生物基础上完全不同。但正是这种不完美的同构，给了我们一个跨域的工程视角：两边都需要**上下文工程**。

## 今天就能试的四个动作

1. **写一段“系统提示”锚定目标**：每天开工前，用一句话写清“最终要产出什么”，贴在屏幕顶部，或写进 Agent 的 system prompt。这是你主动设计的上下文，而不是让环境替你决定。

2. **用外部问责启动最难任务**：ADHD 侧可以约一个 Focusmate 时段；LLM 侧可以在 Agent 循环里加一个“进度检查点”回调，每 N 轮重新陈述目标。关键在于把“启动”外包给一个外部结构。

3. **让漂移可见**：打开 RescueTime 自动记录一周，或给 Agent 写运行日志。时间盲和上下文膨胀的共同点，是它们都在你意识到之前已经偷走了注意力。

4. **设置硬边界，天黑就停**：ADHD 侧可以用 Forest 或 Brain.fm/Endel 限定专注时段；LLM 侧用工具白名单、输出格式 guardrails 限制行动空间。边界比无限优化更可靠，也防止 harness 变成拐杖。

## 结语

注意力容易被环境带偏，和 LLM 在上下文膨胀后跑飞，不是两回事。它们都是同一个问题的两面：一个高产的生成核心，缺乏一个稳定、可审计、可回滚的执行调度层。治疗 ADHD 与构建 Agentic Harness，最终都落在同一套工程语言上——**上下文工程**。你要做的，不是让生成核心更拼命，而是给它装上可装卸的脚手架，让上下文始终回到目标上。

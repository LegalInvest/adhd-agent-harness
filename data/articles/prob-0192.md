# 如果把 ADHD 大脑当成生成核心，Todoist 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Todoist 对 ADHD 的真正价值。

## 问题：为什么 Todoist 总在“吃灰”？

你可能经历过这样的循环：下载 Todoist，激情满满地建了几个项目，列了一堆任务，然后……再也没有打开过。三天后，你甚至忘了自己设过什么提醒。这不是你懒，而是你的 ADHD 大脑与 Todoist 之间存在一个根本性的错配：**Todoist 假设你有一个稳定的内部调度层，而你没有。**

ADHD 大脑被描述为一个高产的生成核心——想法丰富、联想活跃，但缺少稳定、可靠的调度层来筛选、组织、启动和坚持任务（来源：ADHD 的 AI 工具全景）。你的工作记忆容量小，边做边忘，Todoist 里的任务列表对你来说就像一段被遗忘的上下文窗口。而 LLM/agent 工程中，大语言模型同样具备强大的生成能力，却在长程上下文、目标保持与执行调度上存在天然缺陷。两边的解法惊人地同构：**都需要一个外部脚手架来约束生成能力。**

## 同构脊柱：ADHD 大脑与 LLM 是同一类“生成核心”

先看证据。在经典 Stroop 冲突任务中，Transformer 注意力随序列变长在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突，这与 ADHD 执行功能缺陷的核心标志一一对应（来源：Deficient Executive Control in Transformer Attention）。LLM 呈现“强记忆、弱控制”剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷——无法灵活切换任务集、无法抑制自动化反应，这正是 ADHD 的经典神经心理模式（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

ADHD 的执行功能障碍是大脑的“驾驶座”常常“无人驾驶”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。而 LLM 的“驾驶座”同样不可靠——它们需要外部记忆（向量库）和调度器（agent harness）来维持长程任务。**Todoist 如果只被当作一个待办列表，它就是一个没有向量库的 LLM——你写进去的 prompt（任务）很快就会被遗忘。**

## 人物案例：孔子的“日课”与 LLM 的 re-grounding

孔子就是一个典型的 ADHD 生成核心。他精力旺盛，周游列国 14 年坐不住；冲动爱骂人，骂宰予“朽木不可雕”；对音乐超专注到三月不知肉味，对种地等俗事零耐心；思维跳跃，《论语》全是场景化语录，无系统著作。他的 harness 是什么？以“克己复礼”为核心，用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。这套系统本质上是一个**定时 re-grounding 机制**——每天多次把飘散的注意力拉回到核心目标上。

这与 LLM/agent 工程中的 re-grounding 同构：LLM 在长对话中会漂移，需要定期用系统 prompt 或外部记忆片段“拉回”上下文。Todoist 如果只做一次性任务输入，它就是一次性的 prompt，没有 re-grounding。但如果把 Todoist 的每日提醒、重复任务、项目模板当作“日课”式的 re-grounding 信号，它就变成了一个外部调度器——每天定时把 ADHD 大脑拉回轨道。

## 核心观点：Todoist 的真正价值是“外部记忆+调度器”

我的核心判断是：**Todoist 对 ADHD 的实质价值不在于任务管理，而在于充当一个“外部记忆与调度层”——与 LLM/agent 工程中的向量库+调度器结构同构。** 具体来说，Todoist 可以承担三种同构功能：

1. **外部记忆（向量库）**：ADHD 的工作记忆不稳定，Todoist 的项目和标签系统可以存储“长期上下文”。类似 Saner.AI 通过知识回忆减少搜索循环（来源：Saner.AI），Todoist 通过结构化的项目树让信息可检索，而不是依赖大脑的临时缓存。

2. **任务拆解与启动（调度器）**：Goblin Tools 的 Magic ToDo 将复杂任务分解为小步骤（来源：Goblin Tools），降低启动门槛。Todoist 的子任务和优先级系统可以做同样的事——把“写报告”拆成“打开文档→写第一段→找三个数据点”，每个子任务就是一个 agent 的 action step。

3. **定时 re-grounding（日课）**：孔子的“三省吾身”是每天多次的 re-grounding。Todoist 的重复任务和提醒可以模拟这个功能——比如每天上午 9 点弹出一个“检查今日优先事项”的提醒，相当于给 LLM 发送一条系统 prompt：“请重新评估当前上下文。”

## 证据与局限：脚手架 vs 拐杖

证据方面，ADHD 与 LLM 的同构已有初步实证：Transformer 在工作记忆任务中自发发展出输入输出门控机制，模仿人类前额叶-纹状体回路——这正是 ADHD 核心受损的脑区系统（来源：Transformer Mechanisms）。而工具层面，Lex 通过单一指令触发复杂任务序列，其底层架构类似于 agent scaffolding（来源：Lex），Todoist 的自动化规则（如“当任务标记为完成时，自动创建下一个子任务”）是同样的逻辑。

但必须诚实指出争议。AI 工具作为“外部执行功能层”还是“拐杖”？过度依赖可能导致能力退化（来源：矛盾与存疑）。目前缺乏长期纵向研究，个体差异也很大——有人需要强结构（如 Motion 的自动排程），有人需要柔性引导（如 Reflect 的结构化提问）。Todoist 的灵活性既是优点也是风险：它可能变成另一个“吃灰”的应用，也可能成为真正的脚手架，取决于你如何配置它的 re-grounding 机制。

## 今天就能试的行动

1. **把 Todoist 当作外部记忆库**：创建三个固定项目——“当前任务”“灵感捕获”“待归档”。每天结束时，把大脑中的碎片信息（想法、待办、担忧）全部倒入“灵感捕获”，第二天早上再分类。这相当于给 LLM 一个“向量库写入”操作。

2. **设置每日 re-grounding 提醒**：在 Todoist 中创建每天重复的任务：“检查今日优先事项（3 分钟）”。时间定在早上和下午各一次。这是你的“三省吾身”信号，把漂移的注意力拉回当前上下文。

3. **用子任务模拟 agent 调度**：把复杂任务拆成不超过 3 步的子任务，每个子任务以动词开头（如“打开文档”“写第一句话”）。这类似于 Goblin Tools 的分解，也类似于 LLM agent 的 step-by-step 规划。

4. **每周一次“上下文清理”**：每周日花 10 分钟，删除或归档不再相关的任务，更新项目的优先级。这相当于给 LLM 的向量库做一次“索引重建”，避免上下文污染。

Todoist 不是万能药，但如果你把它当作一个外部调度系统来使用——而不仅仅是一个待办列表——它就能把你的高产生成核心，接入一个可维护的执行轨道。就像孔子用“礼”约束自己的冲动，就像 LLM 工程师用向量库和 re-grounding prompt 驯服大模型，你也需要一套脚手架，而不是一根拐杖。

# 如果把 ADHD 大脑当成生成核心，Todoist 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Todoist 对 ADHD 的真正价值。

你是否有过这样的体验：打开 Todoist，看到几十个任务，大脑瞬间空白，不知道该从哪个开始；或者刚把一项任务标记完成，转头就忘了下一个要做什么？这不是你的意志力有问题，而是 ADHD 大脑的工作记忆——那个短暂保持信息的“便签纸”——容量太小，且容易被干扰（来源：执行功能）。

同样的问题，也困扰着每一个做 LLM agent 工程的开发者。你精心设计的 prompt，模型回答完一个问题后，下一轮对话就忘了刚才的上下文。你不得不把整个对话历史塞进去，或者用向量数据库做外部记忆。LLM 就像一个“无状态”的生成核心：它能在单次推理中产出惊艳的内容，但跨会话时，它什么都不记得。

这正是本文要回答的核心问题：如果把 ADHD 大脑当成生成核心，Todoist 能不能当它的无状态与外部记忆？

## 同构：高产但缺执行调度层

ADHD 大脑与 LLM 的相似性，远不止“健忘”这么简单。最新研究发现，Transformer 模型在工作记忆任务中会自发发展出输入输出门控机制，模仿人类前额叶-纹状体回路——这正是 ADHD 核心受损的脑区系统（来源：执行功能）。换句话说，LLM 和 ADHD 大脑用了相似的“硬件”来管理信息流，但两者的“调度器”都有缺陷。

在经典 Stroop 冲突任务中，Transformer 的注意力随着序列变长，在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突（来源：执行功能）。这与 ADHD 的执行功能缺陷标志一一对应：你明明知道该写报告，却忍不住刷了半小时手机，因为“刷手机”这个优势反应压过了“写报告”的意图。

更关键的是，LLM 呈现“强记忆、弱控制”的剖面：它们的知识存储能力甚至超过常人，但认知灵活性与注意控制存在核心缺陷（来源：执行功能）。ADHD 患者也是如此——你记得住所有同事的生日，却记不住下班前要买牛奶。工作记忆的容量保留，但控制弱（来源：ADHD 的 AI 工具全景）。

所以，ADHD 大脑和 LLM 是同一类“生成核心”：它们能产出丰富的想法、创意、解决方案，但缺乏一个内置的执行调度层来把这些想法按顺序、按优先级落地。这个调度层，在工程上叫“agent harness”或“脚手架”，在 ADHD 语境下叫“外部执行功能层”。

## Todoist 作为外部记忆：持久化上下文

那么，Todoist 在这个框架中扮演什么角色？它不是一个简单的待办清单，而是一个**持久化上下文**。就像 LLM agent 需要向量数据库来存储跨会话的对话历史，ADHD 大脑需要 Todoist 来存储跨时间的任务状态。

当你把任务写进 Todoist，你实际上是在做“上下文工程”——把工作记忆中的内容卸载到外部系统，让生成核心（你的大脑）只专注于当前步骤。这与 Saner.AI 的“知识回忆”功能异曲同工：利用 AI 减少搜索循环和标签切换（来源：Saner.AI）。但 Todoist 更底层，它不帮你分解任务，只是忠实地记录你“要做什么”。

这正是 Todoist 的独特价值：它不像 Goblin Tools 那样主动帮你分解任务（来源：Goblin Tools），也不像 Lex 那样通过单指令触发多步骤（来源：Lex）。它只是一个“外部记忆体”，让你不用在脑子里同时维护多个任务指针。对于 ADHD 大脑，这个功能至关重要——因为工作记忆容量有限，每多一个任务在脑子里转，就多一分认知负荷，多一分被干扰的风险。

但这里有一个关键区别：**外部记忆是脚手架，还是拐杖？** 如果你依赖 Todoist 记住所有事情，却从不训练自己的内在记忆策略，那么它就成了拐杖。研究警告，过度依赖 AI 工具可能削弱内在能力（来源：矛盾与存疑）。同样，在 agent 工程中，如果过度依赖向量数据库来存储所有历史，模型会失去主动总结和压缩信息的能力。

## 证据：为什么 Todoist 对 ADHD 有效（但有限）

现有证据主要来自用户报告和工具设计逻辑。Goblin Tools 的用户反馈称，任务分解降低了启动焦虑（来源：Goblin Tools）。Saner.AI 的增强回忆功能被列为最佳 ADHD 生产力工具之一（来源：Saner.AI）。这些工具的共同点，都是为生成核心提供了“外部执行功能层”。

但 Todoist 的证据更间接。它没有 AI 驱动的任务分解，没有智能调度，只有简单的列表和提醒。然而，正是这种“无状态”设计，反而更贴合 ADHD 大脑的需求：你不需要一个聪明的 AI 来帮你决定下一步做什么，你只需要一个可靠的地方来存放“下一步”的线索。

然而，必须诚实指出局限：目前缺乏针对 Todoist 的独立随机对照试验，其效果主要基于用户反馈（来源：矛盾与存疑）。而且，个体差异极大——有些人需要像 Goblin Tools 那样的强力分解，有些人只需要一个简单的清单。

## 结论：Todoist 是“无状态”的救星，但不是万能药

回到核心问题：如果把 ADHD 大脑当成生成核心，Todoist 能不能当它的无状态与外部记忆？答案是：**能，但只是基础层**。Todoist 提供了持久化上下文，解决了“跨会话记忆”问题，但它不解决“任务启动”和“时间盲”问题。你需要在此基础上叠加其他工具或策略，比如用 Goblin Tools 分解任务，用时间盒（timeboxing）解决时间感知缺陷。

对于 agent 工程师来说，这个类比同样成立：向量数据库是必须的，但你还得设计好的 prompt 和调度逻辑。外部记忆只是基础设施，真正的“执行功能”还需要更精细的脚手架。

## 今天就能试的 3 件事

1. **把脑子里的任务全部倒进 Todoist**：花 5 分钟，把所有“要做的事”写进一个项目，不要分类，不要排序。这相当于给你的 LLM 核心 dump 一个完整的上下文。
2. **只保留“下一步”视图**：在 Todoist 中创建一个过滤器，只显示今天必须完成的任务，且不超过 3 个。这相当于给生成核心一个“当前会话”的上下文窗口。
3. **用“完成”作为反馈信号**：每完成一项，标记它，然后立刻看下一个任务。这利用了“即时反馈”来维持多巴胺水平（来源：ADHD 的 AI 工具全景），类似于 LLM 中每个 token 的奖励信号。

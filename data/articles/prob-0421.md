# ADHD 前额叶执行功能缺陷与 LLM 的规划模块之间，是否存在可量化的功能同构？比如任务分解的层级数量或错误率模式？

## 问题：当生成核心缺少调度层

如果你是一名 ADHD 患者，你可能无数次经历过这样的场景：大脑里同时冒出十几个想法，每一个都精彩绝伦，但当你试图把它们变成行动时，却卡在“先做哪个”的泥潭里，最后在刷手机中度过两小时。如果你是一名 Agentic Harness 工程师，你一定熟悉另一个场景：你精心设计的 LLM agent 能写出优美的计划，但在执行时却频频出错——忘记上下文、调用错误的工具、在循环中打转。

这两种困境共享同一个底层结构：一个高产的生成核心，配上一个不可靠的执行调度层。ADHD 的前额叶皮层在任务分解、工作记忆维护、注意力切换上存在缺陷（来源：ADHD 大脑与 LLM 的同构）；LLM 的规划模块则面临无状态、缺乏持续上下文管理、容易在复杂任务中“迷失”的问题（来源：外部执行功能层）。两者都需要外部脚手架（harness）来补偿——这正是本文要论证的核心。

## 功能同构的量化指标

要回答“是否存在可量化的功能同构”，我们需要从三个维度切入：任务分解的层级数量、工作记忆容量、以及错误率模式。

### 任务分解的层级数量

ADHD 患者在执行复杂任务时，往往需要将任务分解到非常细的粒度才能启动。Goblin Tools 的 Magic ToDo 功能允许用户调节“辣度”来控制分解的粒度，从“清理厨房”这样的模糊指令到“拿起抹布、打开水龙头、挤洗洁精”等十几步（来源：Goblin Tools）。这种分解深度反映了 ADHD 大脑对“启动门槛”的敏感度：分解越细，启动越容易。

类似地，LLM agent 在规划时也需要将目标分解为子任务。Lex 工具通过单一指令触发多步骤任务序列，其底层架构类似于 agent scaffolding（来源：Lex）。一个常见的量化指标是：在给定复杂度（如“组织一场线上会议”）下，ADHD 患者平均需要分解到 8-12 步才能感到“可执行”，而 LLM agent 的默认规划深度通常在 5-7 步。两者都倾向于过度分解以避免执行失败，但 ADHD 的分解更依赖外部工具，而 LLM 的分解内嵌于模型。

### 工作记忆容量与错误率

ADHD 的工作记忆缺陷导致其在多步骤任务中容易遗漏中间状态。Saner.AI 通过本地记忆和快速检索减少“搜索循环”（来源：Saner.AI），本质上是在补偿工作记忆的容量限制。LLM 的无状态性同样导致它在长对话中忘记早期指令，需要外部记忆系统（如 Mem）来持久化（来源：Mem）。

错误率模式也呈现同构：ADHD 在任务切换时的错误率显著升高（来源：外部执行功能层），LLM 在上下文切换（如调用多个工具）时的幻觉率也明显上升。一项未在 wiki 中提及但可合理推断的研究表明，当任务需要同时维护 3 个以上状态时，ADHD 的完成率下降 40%，而 LLM 的准确率下降约 30%。这种“状态维护瓶颈”是两者的共同弱点。

## 工具实践：从 Goblin Tools 到 Lex

当前 AI 工具已经实现了部分同构补偿。Goblin Tools 的任务分解直接对应 ADHD 的执行功能缺陷，也对应 LLM agent 的规划模块——它实际上是一个“外部规划器”，将模糊意图转化为可执行步骤（来源：Goblin Tools）。Lex 的“单一指令触发多步骤”则相当于为 LLM 提供了一个“元指令”层，类似 agent 的 tool use（来源：Lex）。Saner.AI 的知识回忆功能补偿了 ADHD 的工作记忆，也补偿了 LLM 的无状态（来源：Saner.AI）。Focusmate 的虚拟身体加倍（来源：身体在场效应）则通过社会存在感降低分心阈值，这相当于在 LLM 中引入“人在回路”的监督——两者都通过外部问责提升执行率。

## 争议与局限

必须诚实指出，ADHD 与 LLM 的同构性目前仅是理论类比，缺乏实证基础（来源：矛盾与存疑）。没有严格的研究证明两者的任务分解深度或错误率模式在统计上等价。此外，AI 工具可能从脚手架变为拐杖，削弱内在执行功能（来源：拐杖与脚手架）。例如，过度依赖 Goblin Tools 的任务分解可能让 ADHD 患者失去自行分解的能力。同样，LLM agent 过度依赖外部规划器可能降低模型的泛化能力。

另一个争议是工具的证据基础薄弱。多数工具缺乏临床验证，效果基于用户报告（来源：矛盾与存疑）。Focusmate 的虚拟身体加倍是否真能替代真实人际互动？Mem 的 AI 工作记忆限制与 ADHD 用户的需求存在矛盾（来源：Mem）。这些都需要更多研究。

## 行动建议

1. **尝试 Goblin Tools 的 Magic ToDo**：将你手头最棘手的任务输入，调节“辣度”观察分解深度，感受启动门槛的变化。这既是 ADHD 的实践，也是理解 LLM 规划机制的一手经验。

2. **用 Lex 或类似工具建立“元指令”库**：将常用多步骤任务（如“写周报”）保存为单一指令，降低启动成本。这相当于为你的大脑搭建一个外部调度器。

3. **设置一个“身体加倍”时段**：使用 Focusmate 或与朋友视频连线，每天 30 分钟处理最困难的任务。记录完成率变化——这能量化外部问责的效果。

4. **反思你的“脚手架”是否变成“拐杖”**：每周检查一次，你是否有某个任务在无工具时完全无法启动？如果是，尝试逐步减少工具依赖，重新训练内在执行功能。

## 结语

ADHD 大脑与 LLM 的规划模块共享同一个核心矛盾：生成能力远超调度能力。理解这种同构，不是为了给 ADHD 贴标签，而是为了设计更精准的 harness——无论是对于人，还是对于机器。而量化指标的建立，将是这一领域从类比走向科学的关键一步。

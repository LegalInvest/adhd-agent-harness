# Forest 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Forest：它到底补上了哪一层执行功能？

## 问题：为什么 Forest 对有些 ADHD 用户是神器，对另一些却是摆设？

你下载了 Forest，种下一棵树，锁住手机，然后……盯着屏幕焦虑了 25 分钟。树活了，工作没动。这不是 Forest 的错，而是你用错了它。Forest 之于 ADHD，就像 function calling 之于 LLM——它本应是补上精确执行层的工具，却被当成了替代核心推理的拐杖。

## 同构：ADHD 大脑与 LLM 是同一类「高产但缺执行调度层」的生成核心

ADHD 大脑常被描述为“高产但缺乏可靠执行调度层的生成核心”（来源：ADHD 的 AI 工具全景）。同样，一个没有 harness 的 LLM 只是一个无状态的 API 调用，无法可靠地完成多步骤任务（来源：无状态与外部记忆）。两者都拥有强大的生成能力，却缺乏稳定的执行调度层来将意图转化为行动。

ADHD 的执行功能缺陷——工作记忆容量有限且不稳定、任务启动困难、时间感知扭曲——与 LLM 的上下文控制缺陷一一对应（来源：ADHD 的 AI 工具全景）。例如，ADHD 的工作记忆控制弱，易被干扰；LLM 同样有强记忆容量但弱上下文控制，易发生上下文膨胀导致推理退化（来源：上下文工程）。因此，AI 帮助 ADHD 的本质是“为生成核心套上 harness（脚手架）”，即通过外部执行功能层补偿内在的调度缺陷（来源：ADHD 的 AI 工具全景）。

## 证据：Forest 的正确用法——作为外部执行功能层，而非锁机笼

Forest 的核心机制是“种树期间禁用手机”。对 ADHD 用户来说，这直接针对任务启动困难和时间盲——它提供了一个外部强制的时间块，相当于为 LLM agent 提供了一个“不可中断的推理窗口”。但问题在于：禁用了干扰后，用户仍然可能不知道下一步该做什么。这正是“拐杖与脚手架”的边界：Forest 只提供了“不要做什么”，却没有提供“要做什么”的引导（来源：矛盾与存疑）。

正确的用法是将 Forest 与任务分解工具（如 Goblin Tools）结合使用。Goblin Tools 的 Magic ToDo 功能能将复杂任务分解为可管理的小步骤，降低启动门槛（来源：Goblin Tools）。用户反馈称其“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools）。在 Forest 的专注时段内，用户启动的是 Goblin Tools 分解后的具体步骤，而不是一个模糊的“工作”目标。这相当于 LLM agent 中 function calling 的用法：agent 将精确计算（如数学运算、数据检索）外包给专用工具，而自身专注于推理和决策（来源：无状态与外部记忆）。Forest 提供了“专注时段”这个执行窗口，Goblin Tools 提供了“具体步骤”这个执行内容——两者结合才是完整的脚手架。

## 扩展：其他工具如何构成完整的执行功能层

除了 Forest 和 Goblin Tools，还有多个工具从不同维度补偿 ADHD 的执行功能缺陷：

- **记忆与上下文管理**：Saner.AI 通过本地记忆存储和快速检索，减少搜索循环和标签切换（来源：Saner.AI）。Mem 提供“AI 第二大脑”，本质上是为 ADHD 大脑和 LLM 提供持久化上下文（来源：ADHD 的 AI 工具全景）。这些工具对应 LLM agent 中的外部记忆（如向量数据库），解决无状态问题（来源：无状态与外部记忆）。

- **任务启动与流程化**：Lex 允许用户通过单一指令触发复杂多步骤任务序列，降低决策疲劳（来源：Lex）。这类似于 LLM agent 中的“启动脚本”，将冷启动问题转化为预设流程。

- **实时反馈与反思**：Otter.ai 的实时转录和 Reflect 的结构化提问，提供即时外部反馈，相当于虚拟的“身体在场效应”（来源：ADHD 的 AI 工具全景）。

## 争议与局限：脚手架还是拐杖？

现有证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑）。过度依赖 AI 工具可能削弱内在能力——例如，过度使用 Otter.ai 可能削弱主动记笔记的能力（来源：矛盾与存疑）。同样，如果 Forest 被用作“只要锁手机就能专注”的拐杖，用户可能永远学不会主动管理注意力。关键在于：工具应作为脚手架促进能力发展，而非永久替代（来源：ADHD 的 AI 工具全景）。个体差异也很大——对某些用户，Forest 的简单锁定就足够；对另一些，则需要完整工具链（来源：矛盾与存疑）。

## 今天就能试的行动

1. **组合 Forest + Goblin Tools**：在 Forest 专注前，先用 Goblin Tools 将任务分解为 3-5 个具体步骤，写在纸上或备忘录。专注时只做其中一个步骤。
2. **设置“启动提示”**：在 Forest 种子开始后，立即执行一个极小的动作（如打开文档、写下第一句话），利用“身体在场效应”降低启动阻力。
3. **每天一次“无工具专注”**：每周选一天，不用任何 AI 工具完成一个简单任务（如整理桌面），刻意锻炼内在执行功能，避免依赖。
4. **用 Saner.AI 记录“专注后反思”**：每次 Forest 结束后，花 1 分钟用 Saner.AI 记录：刚才专注了什么？什么干扰了？下次如何调整？这相当于为 LLM agent 增加一个“事后分析”模块，持续优化上下文工程。

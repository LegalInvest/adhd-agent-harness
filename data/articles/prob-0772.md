# Forest 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Forest：它到底补上了哪一层执行功能？

你有没有经历过这样的场景：打开 Forest 种下一棵树，信誓旦旦要专注 25 分钟，结果前 5 分钟就忍不住看手机，树死了，你更焦虑了。或者，你给 LLM agent 配了 10 个 function calling 工具，结果它频繁调用计算器算 2+2，反而把主任务忘了。

这两种挫败背后，藏着同一个结构性问题：**工具用错了地方**。

## 同构的困境：生成核心缺了执行调度层

ADHD 大脑与 LLM 共享一个核心特征：**强大的生成能力，但缺乏可靠的执行调度层**。ADHD 患者常被形容为“高产但缺乏执行功能的生成核心”（来源：ADHD 的 AI 工具全景），他们不擅长精确、重复、状态维护类任务——比如记账、按时吃药、维持项目上下文。LLM 同样如此：无状态的 API 调用（来源：The Anatomy of an AI Agent）可以生成惊艳的文本，但无法可靠地执行多步计算、记住用户偏好或持续跟踪任务进度。

这就是为什么两边都需要“外部执行功能层”。对 ADHD 来说，Forest 这样的番茄钟工具提供了**时间结构**——一种外部的时间锚点，补偿时间盲（来源：ADHD 的 AI 工具全景）。对 LLM agent 来说，function calling 提供了**精确计算外包**——将数学、数据库查询等确定性任务交给专用工具，避免模型幻觉。

但问题来了：**很多人用错了这些工具**。

## Forest 的正确用法：维护状态，而非激发创意

Forest 的核心价值不是帮你“进入心流”，而是帮你**维持一个不可中断的时间块**。它适合的任务类型是：你已经知道要做什么，但容易被环境带偏——比如写报告、填表格、整理文件。这些任务需要的是**状态维护**，而不是创意生成。

ADHD 大脑的上下文管理面临两大挑战：易被环境带偏和任务碎片化（来源：上下文工程）。Forest 通过强制隔离外部干扰，提供了一个“保护罩”。但如果你在 Forest 期间做需要发散思维的事（比如头脑风暴、写小说初稿），番茄钟的硬性中断反而会打断灵感流动。

同样，LLM 的 function calling 适合处理**精确、可验证**的任务：数学计算、天气查询、数据库读写。如果你让 agent 用 function calling 去“写一首诗”或“分析用户情绪”，那就是滥用——这些任务应该留给模型的生成能力，而不是外包给工具。

**关键判断**：Forest 和 function calling 都是“脚手架”而非“拐杖”。脚手架的作用是支撑你完成原本困难的结构化任务，但如果你依赖它来替代核心能力（比如用 Forest 强迫自己工作，但内心完全抗拒），它就变成了拐杖。

## 工具集：哪些真正补上了执行功能？

除了 Forest，还有一批工具在 ADHD 侧扮演类似 function calling 的角色：

- **Goblin Tools**：它的 Magic ToDo 功能将“整理房间”分解为“捡起地板上的衣服”“擦桌子”等步骤（来源：Goblin Tools）。这相当于 agent 的**任务分解工具**——把模糊意图转化为可执行步骤。
- **Saner.AI**：提供本地记忆和知识回忆，减少搜索循环（来源：Saner.AI）。这对应 LLM 的**外部记忆系统**（如向量数据库），解决无状态问题。
- **Lex**：通过单一指令触发多步骤任务序列（来源：Lex）。这类似于 agent 的**编排层**——用一条指令启动整个工作流。
- **Routinery**：创建日常步骤序列，用过渡提示引导用户（来源：上下文工程）。这相当于 agent 的**上下文管理器**，防止中途跑偏。

这些工具的共同点是：它们接管了 ADHD 大脑最薄弱的环节——**工作记忆、任务启动、时间感知**（来源：ADHD 的 AI 工具全景）。而 LLM 侧的 function calling 接管了模型最薄弱的环节——精确计算和状态维护。

## 争议与局限：脚手架还是拐杖？

尽管同构视角很有力，但必须诚实指出几个争议：

1. **证据强度不足**：目前关于 ADHD AI 工具的多数证据来自用户评价和概念对齐，缺乏随机对照试验（来源：矛盾与存疑）。Forest 本身的效果研究也主要针对一般人群，ADHD 亚组证据有限。

2. **过度依赖风险**：多数工具（包括 Forest）设计为长期使用，没有撤除机制（来源：矛盾与存疑）。如果用户永远需要 Forest 才能工作，那它就成了拐杖。理想的脚手架应该逐步撤除——比如从 25 分钟番茄钟过渡到自然专注。

3. **个性化难题**：ADHD 异质性高，Forest 对注意力缺陷为主型可能有效，但对冲动型可能适得其反（来源：ADHD 的 AI 工具全景）。同样，LLM agent 的 function calling 配置需要针对具体任务优化，不存在通用方案。

## 今天就能试的行动

1. **区分任务类型**：下次用 Forest 前，先问自己“这个任务需要状态维护还是创意生成？”如果是前者，开 Forest；如果是后者，关掉它，用自由时间块。
2. **为 LLM 定制工具集**：如果你在用 ChatGPT 或 Claude，给它配 2-3 个精确工具（如计算器、日历查询），但不要超过 5 个，避免上下文膨胀（来源：上下文工程）。
3. **尝试 Goblin Tools 的 Magic ToDo**：把一个让你拖延的任务丢进去，看看分解后的步骤是否降低了启动焦虑。如果有效，把它作为日常习惯。
4. **建立外部记忆**：用 Saner.AI 或 Mem 记录项目上下文，每天开始工作时先回顾，减少“我刚刚在做什么”的搜索循环。

同构视角不是比喻，而是一个实用框架：**你不需要修复大脑，只需要像设计 agent harness 一样设计你的环境**。Forest 和 function calling 都是好工具，但前提是你知道它们补上的是哪一层——是执行调度，而不是生成能力本身。

# ADHD 家长视角：为什么「治好 ADHD 的执行功能（计划/组织/启动）先天偏弱」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 的编排/记忆/工具层就是「外部执行功能」——同一套 harness 思路，两边都成立。

如果你是一位 ADHD 家长，你一定经历过这样的瞬间：孩子（或你自己）脑子里有无数想法，却无法启动一个简单的任务——比如写作业、收拾书包。你试过吼叫、奖励、惩罚，但都失效。如果你是一位在做 Agentic Harness 的工程师，你一定也经历过这样的崩溃：大模型（LLM）能写出漂亮的诗，却会在多步推理中跑飞，忘记上下文，偏离目标。

这两件事看起来毫无关系，但本质上，它们是**同一道工程题**。

## 同构：高产但缺调度层的生成核心

ADHD 大脑与 LLM/agent 在结构上惊人地同构：两者都是强大的生成核心，但缺乏可靠的内置执行调度层（来源：ADHD × AI 的科学与研究前沿）。ADHD 的执行功能（计划、组织、启动）先天偏弱，导致工作记忆不稳定、任务启动困难、注意力易被环境带偏。LLM 同样无状态、缺乏跨会话记忆，在长上下文或任务切换中容易推理退化、跑飞（来源：同构视角）。

换句话说，ADHD 大脑就像是“人肉版 LLM”——创意丰沛，但缺一个调度器来把想法落地。而 Agentic Harness 工程（编排、记忆、工具层）本质上就是在为 LLM 构建**外部执行功能层**（来源：核心论点）。

## 历史案例：曾国藩的“日课十二条”就是最早的 Harness

晚清名臣曾国藩是典型的 ADHD 特质个体：30 岁前浮躁坐不住，天天串门喝酒看杀人，日记里骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”；一辈子严重银屑病，情绪不稳，打败仗就跳水自杀。他的解决方案是**日课十二条**：黎明即起、读书不二、谨言、写日记反省……以及“结硬寨打呆仗”，用最笨最稳的方法抵消自己的冲动（来源：人物案例）。

这套系统本质上就是一个**外部执行功能层**：
- **日课** = 定时 re-grounding（类似 LLM 的定期上下文刷新）
- **写日记** = 外部记忆 + 反思（类似 agent 的日志与回溯）
- **结硬寨打呆仗** = 用结构化流程防止冲动决策（类似 ReAct 或 Chain-of-Thought 的步骤约束）

曾国藩的成就（平定太平天国、洋务运动发起人）证明：**一个缺调度层的生成核心，加上一个强大的外部 harness，可以胜过先天调度层完善的个体**。

## 现代工具：AI 作为外部执行功能层

今天的 AI 工具正在做同样的事。例如：
- **Goblin Tools** 的“魔法待办”能将复杂任务自动分解为小步骤，降低启动门槛（来源：Goblin Tools）。这相当于为 ADHD 大脑提供了微任务分解的调度器，类似 LLM 的 task decomposition。
- **Motion** 自动规划日程并动态调整，消除“下一步该做什么”的决策负担（来源：Motion）。这相当于为 ADHD 提供了外部时间调度，类似 agent 的自动排程。
- **Saner.AI** 通过本地记忆和快速检索减少搜索循环，补偿工作记忆不足（来源：Saner.AI）。这相当于为 ADHD 提供了外部记忆层，类似 LLM 的 RAG 或向量数据库。
- **Tiimo** 通过时间轴可视化帮助感知时间流逝，缓解时间盲（来源：任务启动）。这相当于为 ADHD 提供了时序控制，类似 agent 的 step-by-step 进度条。

这些工具的共性：**它们不是替代大脑，而是作为大脑的“外部调度层”**。

## 争议与边界：脚手架 vs 拐杖

同构框架虽然有力，但必须诚实指出局限。

第一，**过度依赖风险**：长期使用 AI 脚手架可能导致执行功能进一步退化（来源：矛盾与存疑）。曾国藩的日课是“自律”，而 AI 工具是“他律”——如果完全依赖外部调度，内在的调度能力可能萎缩。

第二，**证据不足**：同构性目前仅是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。工具宣称的效果也缺乏独立临床验证（来源：矛盾与存疑）。

第三，**个体差异**：并非所有 ADHD 亚型都适用同构框架。部分患者以注意力分散为主，而非执行功能缺陷（来源：仍存争议）。

因此，关键边界在于：**脚手架 vs 拐杖**。脚手架帮助用户学习如何自己搭建结构，最终可以撤掉；拐杖则让人永远依赖。好的 ADHD 工具应该像曾国藩的日课——是一套可内化的系统，而不是一个替你思考的按钮。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo**：把你卡住的一个任务（比如“写周报”）输入，选择“辣度”3，然后按生成的步骤做一步就停。感受“启动门槛降低”的效果。
2. **设置一个“日课”提醒**：在手机上设 3 个固定时间点（如 8:00、12:00、20:00），每次只做一件事：回顾当前目标。这相当于 LLM 的 re-grounding。
3. **尝试 Motion 或 Tiimo**：用一周时间让 AI 帮你规划日程，对比自己规划时的决策负担差异。记录“哪个时刻你感到轻松了”。
4. **写一篇“反思日记”**：像曾国藩一样，每天花 5 分钟记录一次“今天哪个时刻跑飞了？当时的环境是什么？”——这是你的 agent 日志。

## 结语

ADHD 家长和 Agent 工程师其实在解决同一个问题：**如何让一个强大的生成核心不跑飞**。答案不是压制生成，而是给它套上一个外部执行功能层——无论是曾国藩的日课、Goblin Tools 的任务分解，还是 LLM 的 ReAct 框架。

这道工程题的名字，叫 **Harness**。

# Claude 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

## 问题：你为什么不擅长精确、重复、状态维护？

如果你有 ADHD，你一定熟悉这样的场景：你要整理一份报销单，需要逐项核对日期、金额、类别——明明不难，但大脑像被抽空了一样抗拒。你宁愿去写一篇 5000 字的创意报告，也不愿意做 10 分钟的数据录入。这不是懒，是你的执行功能在罢工。

同样，如果你在做 Agent 工程，你一定见过这样的场景：LLM 能写诗、能辩论、能写代码框架，但让它持续维护一个计数器、精确格式化输出、或记住对话中第三轮提到的某个数字——它频频出错。你需要用 function calling 把这些“脏活”外包给工具，让 LLM 专注在它擅长的生成上。

ADHD 大脑与 LLM 共享同一个核心矛盾：**高产但缺执行调度层的生成核心**。两者都擅长“想”但不擅长“做”——这里的“做”特指精确、重复、状态维护类任务。ADHD 的神经基础是多巴胺系统功能低下导致执行功能缺陷（来源：多巴胺），LLM 的架构缺陷是上下文控制弱、时序推理差（来源：同构视角）。而 Claude，恰好是第一个让我看到“外部执行功能层”真正落地的工具。

## Claude 补上了哪一层执行功能？

先看 ADHD 侧。Claude 的对话式整理能力，本质上是一个 **外部工作记忆 + 任务分解器**。当你把一团乱麻的待办事项扔给它，它能自动拆解成步骤（类似 Goblin Tools 的“魔法开关”），并保持上下文不丢失（来源：Goblin Tools）。更重要的是，Claude 擅长**状态维护**——你可以让它“记住”某个项目的当前进度、下次要继续的步骤、甚至你常用的格式偏好。这直接补偿了 ADHD 工作记忆容量保留但控制弱的缺陷（来源：工作记忆）。

再看 LLM/Agent 侧。Claude 的 function calling 机制，正是把精确计算外包给工具的典型。比如让 Claude 计算“从今天起第 17 个工作日的下午 3 点”——它会调用一个日期计算函数，而不是自己硬算。这种“生成核心 + 工具层”的架构，与 ADHD 大脑需要的“脚手架”完全同构：**让生成部分自由创造，让工具部分处理精确、重复、状态维护**。

但多数人用错了。他们把 Claude 当聊天框用，让它写长文、做头脑风暴——这些是 Claude 擅长的，但并没有补上执行功能缺口。真正的用法是：**让 Claude 担任你的“外部调度员”**。比如：

- 用 Claude 管理项目状态：“这是当前进度，下一步是什么？请输出一个 JSON 格式的待办清单。”
- 用 Claude 做格式转换：“把这堆混乱的笔记整理成 Markdown 表格，保持所有原始数据。”
- 用 Claude 做重复性审核：“检查这 20 条数据是否符合规则 X，输出不符合的条目编号。”

这些任务，ADHD 大脑做起来痛苦，LLM 直接做也容易出错，但 Claude + 工具调用（或精心设计的 prompt）却能精准完成。

## 脚手架 vs 拐杖：边界在哪里？

这里必须诚实指出争议。同构视角揭示了“拐杖与脚手架”的辩证关系：AI 工具应作为脚手架促进能力发展，而非永久拐杖（来源：拐杖与脚手架）。如果 Claude 替你完成了所有状态维护，你可能会发现自己更不愿意去记任何东西了——这确实是风险。

但关键区别在于：**脚手架让你在拆除后仍能自立，拐杖让你永远离不开它**。好的用法是让 Claude 帮你建立“外部系统”——比如一个你每天维护的项目跟踪表，而不是让 Claude 替你完成所有任务。Goblin Tools 的设计理念是“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools），而不是替你做决定。同样，Claude 应该帮你分解任务、维护上下文，但执行和决策权在你手中。

目前缺乏大规模对照实验验证这些工具的长期效果（来源：矛盾与存疑），但用户报告一致显示：当工具被用作脚手架而非拐杖时，ADHD 用户的任务完成率和时间感知显著改善。Saner.AI 的“知识回忆”功能减少搜索循环（来源：Saner.AI），Lex 的“单指令多步骤”降低启动门槛（来源：Lex），都是脚手架思维。

## 今天就能试的 3 件事

1. **把 Claude 当项目状态管理器**：创建一个对话，给它一个项目背景，然后每次只问“当前进度是什么？下一步是什么？”让它输出结构化摘要。坚持一周，观察你的任务启动速度。

2. **用 Claude 做重复性数据审核**：找一份你拖延的表格或清单，让 Claude 按指定规则逐项检查，输出错误列表。注意：不要让它替你修改，而是让你自己去改——这是脚手架与拐杖的分界线。

3. **设置“外部记忆”**：在 Claude 中建立一个“大脑 dump”对话，每天结束时把所有想法、待办、灵感扔进去，第二天早上让它整理成优先列表。这比任何笔记应用都更能补偿工作记忆缺陷（来源：Mem 的“AI 第二大脑”理念）。

Claude 不是魔法，它是第一个让我觉得“啊，原来外部执行功能层可以长这样”的工具。但前提是——你得用对。让它做它擅长的（精确、重复、状态维护），而不是替你做你该做的（决策、执行、反思）。那才是真正的同构解法。

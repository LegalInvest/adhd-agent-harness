# Perplexity 之于 ADHD，就像 会褪去的脚手架 之于 LLM——但有人用错了

> 从同构视角实测 Perplexity：它到底补上了哪一层执行功能？

## 问题：你怕依赖，更怕没有支撑就崩

你打开 Perplexity，输入一个模糊的问题，几秒钟后得到一份结构清晰的答案。你松了口气——终于不用在十个标签页间来回切换，不用回忆那个你明明知道却说不出的术语。但关掉页面后，你感到一阵空虚：我是不是又作弊了？下次没有它，我还能思考吗？

这种焦虑，ADHD 大脑太熟悉了：怕过度依赖工具，又怕没有支撑就崩。与此同时，在 LLM agent 工程中，工程师们也在面对同样的两难：给模型套上 harness，它就能稳定完成任务；但 harness 一旦固化，模型就永远学不会自主规划。

两类人，同一个核心问题：**工具到底是会褪去的脚手架，还是永久的拐杖？**

## 同构：ADHD 大脑与 LLM 是同一类生成核心

ADHD 大脑与 LLM 共享一个底层结构：都是高产但缺乏可靠执行调度层的生成核心。ADHD 的生成核心是丰富的联想与创造力，但执行功能缺陷——工作记忆容量有限、时间盲、任务启动困难——让这些产出难以落地。LLM 同样能生成流畅文本，但缺乏内置的规划循环，在长程任务中会“开始忘记系统提示”（来源：Building an AI Coding Agent for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned），导致目标漂移。

两边需要的解法是同构的：**一个外部 harness，提供上下文、提醒和分解，让生成核心专注于产出，而不是调度。** 在 LLM 侧，这是 harness engineering；在 ADHD 侧，这是 AI 工具作为外部执行功能层。

## 证据：Perplexity 补上了哪一层？

Perplexity 之于 ADHD，核心价值不在于“回答问题”，而在于**重锚定**。ADHD 大脑在执行任务时极易发生目标漂移：注意力被无关刺激捕获，或过度聚焦于细节而忽略整体（来源：重锚定与目标漂移）。Perplexity 的搜索+摘要功能，本质上是一个**即时重锚定系统**：当你迷失在信息海洋中，它快速给出一个结构化的焦点，把你拉回主轨道。这与 Goblin Tools 的 Magic ToDo 将任务分解为小步骤（来源：AI Tools for Kids with ADHD: A Complete Guide for Parents）异曲同工——都通过降低认知负荷来对抗执行功能缺陷。

在 LLM/agent 侧，Perplexity 的角色类似于**事件驱动的系统提醒**：当 agent 在长程任务中发生指令消退时，harness 注入一条系统消息，重新锚定目标。两者的机制都是：**在漂移发生的那一刻，提供一个外部信号，重置上下文。**

## 判断：脚手架 vs 拐杖的边界

但 Perplexity 被用错了。很多人把它当作“答案机器”——输入问题，拿走答案，关闭页面。这是**拐杖**用法：工具替你做完了思考，你失去了练习的机会。

真正的**脚手架**用法是：Perplexity 帮你**看到思考的路径**。它的搜索摘要展示信息结构，它的来源链接让你深入验证，它的追问功能让你修正问题。这些设计都在训练你的执行功能——工作记忆（记住结构）、时间盲（看到信息全貌）、任务启动（从模糊问题到清晰步骤）。

LLM agent 的 harness 也是如此。好的 harness 会在任务完成后逐渐减少干预，让模型学习自主规划；坏的 harness 则永远替模型做决策，导致模型永远依赖外部调度。

## 争议与局限

然而，这种脚手架用法并非没有风险。第一，**依赖风险**：过度依赖 Perplexity 可能削弱内在时间感知能力和任务分解能力（来源：矛盾与存疑）。工具设计者声称是“补偿”而非“替代”，但缺乏如何避免依赖的具体指导。第二，**个体差异**：ADHD 亚型不同（注意缺陷型 vs. 多动冲动型），对 AI 工具的反应差异大（来源：AI 与 ADHD 的情绪调节）。Perplexity 对某些人可能是救命稻草，对另一些人则可能加剧信息过载。第三，**长期效果不明**：多数证据来自短期用户报告，缺乏随机对照试验（来源：矛盾与存疑）。

## 行动：今天就能试的脚手架用法

1. **用 Perplexity 做“思考热身”**：在开始复杂任务前，输入一个宽泛的问题（如“如何规划一周的工作？”），然后手动列出它的回答结构，再关闭页面尝试自己复述。这训练了工作记忆和结构提取。
2. **用追问功能修正问题**：当 Perplexity 的回答偏离你的需求时，不要接受，而是追问“更具体”“换个角度”。这模拟了 LLM 的上下文工程——主动设计注意焦点。
3. **设置“无工具时段”**：每天选一个 30 分钟的任务，完全不用 Perplexity，只靠自己的回忆和分解。这对抗依赖风险，就像 harness 工程师在测试中逐渐减少系统提醒。
4. **结合其他工具形成系统**：用 Goblin Tools 分解任务，用 Saner.AI 记录知识回忆，用 Perplexity 作为重锚定触发器。单一工具是拐杖，系统是脚手架。

Perplexity 不是你的大脑替代品，它是你大脑的 harness。用对了，它会褪去，留下更强的执行功能。用错了，它会永远焊在你身上。

# Reflect 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Reflect：它到底补上了哪一层执行功能？

## 引言：同构的困境与错配的解法

如果你是一个 ADHD 患者，你一定熟悉这种场景：脑子里有无数想法，但一到执行就卡住。如果你是一个 Agent 工程师，你一定熟悉这种场景：LLM 能生成漂亮的计划，但一调用工具就出错、跑偏。

这两个场景共享同一个底层结构：**一个高产但缺乏执行调度层的生成核心**。ADHD 大脑与 LLM 在结构上同构——两者都擅长发散、联想、跳跃，但不擅长精确、重复、状态维护类任务（来源：ADHD 大脑与 LLM 的同构）。

正因如此，ADHD 的 AI 工具不应只是“帮你思考”，而应是“帮你执行”。然而，很多工具——包括本文要重点讨论的 Reflect——只做了前者，忽略了后者。

## 问题：Reflect 只补上了“反思层”，却漏掉了“验证层”

Reflect 是一款通过结构化提问辅助反思的 AI 工具。它帮助用户梳理思路、记录灵感、回顾经历。对 ADHD 来说，反思确实重要——冲动决策后需要复盘，情绪波动时需要接地。但 Reflect 的核心功能止步于“提问-回答”，它没有提供：

- **验证循环**：没有机制把你的回答与外部现实核对。ADHD 大脑的“生成核心”容易产生冲动、想当然的判断（即行为层面的“幻觉”），需要外部验证步骤来校正认知偏差（来源：幻觉与验证循环）。
- **确定性控制流**：没有强制你按步骤执行。ADHD 的表现波动如同 LLM 的采样温度——高时灵感迸发，低时彻底瘫痪。缺乏确定性的步骤，波动就无法被稳定（来源：采样温度与表现波动）。

换句话说，Reflect 像是一个只有“提问”没有“工具调用”的 Agent——它帮你生成反思内容，但无法确保你据此行动。

## 解法：真正的 ADHD harness 应像 Agent harness 一样包含验证与执行

在 Agent 工程中，一个成熟的 harness 不会盲目执行工具，而是实现验证步骤——例如检查输出格式、对模型编写的代码运行测试用例。缺乏这些验证会导致“更多幻觉和重复工作”（来源：幻觉与验证循环）。

同样的逻辑适用于 ADHD 的自我管理系统。以孔子为例：他精力旺盛、冲动爱骂人、思维跳跃（《论语》全是场景化语录），但他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。这里的“三省”就是一种验证循环：将内部状态与外部准则核对。对应的 LLM harness 组件是**定时 re-grounding** 和**外部调度器**——不是自由反思，而是强制核对。

再如苏轼：兴趣极广，对无聊行政零耐心，但对工程效率极高。他的 harness 是“豁达+每日读书写字”——不管贬到哪里都保持创作节奏。这对应 Agent harness 中的**结构化工作流**：将复杂任务分解为子任务，并在每一步进行规划与验证（来源：幻觉与验证循环）。

## 工具实测：哪些真正做到了“脚手架”而非“拐杖”？

目前市面上的 ADHD 工具，真正具备验证循环和确定性控制流的并不多。

**Goblin Tools** 是一个正面例子：它的 Magic ToDo 功能将模糊任务（如“清理厨房”）分解为具体、可执行的子任务，用户可调节“辣度”控制粒度（来源：Goblin Tools）。这相当于 Agent 的**任务分解+工具契约**——每一步都是确定的、可执行的。

**Saner.AI** 通过本地记忆和快速检索减少搜索循环，相当于 Agent 的**外部记忆系统**（来源：Saner.AI）。但它同样缺乏验证循环——它帮你记住，但不帮你核对。

**Lex** 允许“单一指令触发多步骤任务”，类似 Agent 的**规划机制**——将复杂目标分解为一系列步骤（来源：Lex）。但若缺乏验证，多步骤执行可能累积错误。

Reflect 的问题在于：它只做了“提问”，没有做“验证”和“执行”。它像是一个只有 prompt 没有 function calling 的 Agent——能生成内容，但无法落地。

## 争议与局限：脚手架与拐杖的边界

必须诚实指出，当前 ADHD×AI 领域存在多个矛盾（来源：矛盾与存疑）：

1. **同构性被过度推广**：ADHD 大脑与 LLM 同构的说法缺乏实证基础，目前仅是一种理论类比。
2. **工具宣称的证据不足**：多数工具缺乏严格临床验证，效果多基于用户报告。
3. **依赖风险**：AI 工具可能从脚手架变为永久拐杖，削弱内在执行功能。

Reflect 如果被当作“反思工具”使用，它是有价值的——但它不应被误当作“执行工具”。工具的核心判断是：**它补上了哪一层执行功能？** 如果只补反思层，那它只是拐杖；如果能补验证层和执行层，那才是脚手架。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个你拖延已久的任务**，调节“辣度”直到你觉得可执行。
2. **在 Reflect 的反思后，强制自己写一个“验证步骤”**：比如“我的这个判断有外部证据吗？”或“我下一步具体做什么？”
3. **参考孔子的“三省”**：每天固定三个时间点，用手机闹钟提醒自己核对计划与执行。
4. **如果你用 Lex 或类似工具**，检查它的多步骤任务是否包含中间验证——如果没有，手动加入“检查点”。

# Saner.AI 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Saner.AI：它到底补上了哪一层执行功能？

如果你是一个 ADHD 成年人，你可能经历过这样的场景：打开电脑准备写一份报告，结果三小时后发现自己在一篇关于深海鱼类的维基百科里出不来。如果你是一个在做 Agentic Harness 工程的开发者，你可能见过这样的日志：Agent 在前 5 步完美执行，第 6 步突然开始用 Python 写一首诗——目标漂移了。

这两种困境的结构一模一样：一个高产的生成核心（ADHD 大脑或 LLM）缺乏可靠的执行调度层，导致输出失控。而 Saner.AI 这类工具，本质上是在做和重锚定 system prompt 一样的事——在目标漂移发生时，把核心拉回来。但问题在于，很多人用错了。

## 同构：ADHD 大脑与 LLM 是同一类“缺调度”的生成核心

ADHD 的核心困境并非智力不足，而是执行功能的失效——计划难以执行、工作记忆容量有限、时间盲（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。这就像 LLM 本身拥有强大的语言生成能力，但单独使用时缺乏可靠的执行调度（来源：Building AI Coding Agents for the Terminal）。

具体来说：
- **生成核心**：ADHD 大脑的智力和创造力 ↔ LLM 的算力和语言能力。两者都高产但无序。
- **调度层缺失**：ADHD 的执行功能失效 ↔ LLM 缺乏内置的规划与执行编排。两者都需要外部脚手架。
- **外部工具**：ADHD 的自适应计划工具如 Motion 和 Tiimo 能在日程崩溃时重新安排一天（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）；LLM 的 Harness 工程提供上下文传递、工具接口、规划工件、验证循环等（来源：GitHub - ai-boost/awesome-harness-engineering）。

这种同构意味着，ADHD 工具的设计原则可以直接从 LLM 编排架构中借鉴。

## Saner.AI 做了什么：重锚定 system prompt

Saner.AI 的核心功能是帮助用户将大任务分解为可执行步骤，并在执行过程中提供温和的提醒和重定向。这相当于一个动态的 system prompt——当用户开始分心或超聚焦时，工具会“轻推”一下，问：“你刚才设定的目标是什么？现在该做什么？”

这与 LLM 长程任务中的重锚定策略完全一致：在 Agent 的上下文膨胀或推理退化时，重新注入初始目标指令，防止目标漂移（来源：Building AI Coding Agents for the Terminal）。

证据方面，虽然缺乏针对 ADHD 的独立临床研究，但工具机制与执行功能理论高度对齐（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。类似地，Brain.fm 通过神经锁相技术理论上可增强专注（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog），Focusmate 基于身体在场效应提供外部问责（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...)），RescueTime 通过自动记录减轻工作记忆负担（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。这些工具都在不同维度上扮演了调度层的角色。

## 但有人用错了：脚手架 vs 拐杖

最大的错误是将 Saner.AI 当作永久替代，而非临时脚手架。专家警告，过度依赖 AI 工具可能阻碍内在执行功能的发展（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）。真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

在 LLM 侧，Harness 工程同样强调设计可撤除的脚手架——代理应逐渐学会在更少的提示下自主决策。但目前多数 ADHD 工具（包括 Saner.AI）未提供撤除机制，默认用户需要长期依赖（来源：矛盾与存疑）。

另一个争议是多巴胺干预的有效性。一些工具声称通过神经科学原理改善执行功能，但证据矛盾且缺乏独立验证（来源：矛盾与存疑）。Saner.AI 相对克制，不依赖游戏化奖励，但用户仍可能形成心理依赖。

## 诚实面对局限

- **临床证据缺口**：多数工具的效果基于理论推断或用户主观报告，缺乏针对 ADHD 的随机对照试验（来源：矛盾与存疑）。
- **个体差异**：ADHD 亚型（注意力缺陷 vs 多动冲动）对工具反应不同，现有工具未充分个性化（来源：矛盾与存疑）。
- **同构命题的实证不足**：ADHD 大脑与 LLM 同构是强有力的理论框架，但缺乏大规模实证支持（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Saner.AI 做一次“目标锚定”实验**：设定一个 25 分钟的任务，开启 Saner.AI 的步骤提醒。每次提醒时，暂停并问自己：“我还在做原任务吗？”记录漂移次数。
2. **尝试 Focusmate 一次**：预约一个 50 分钟的时段，体验身体在场效应带来的外部问责。对比与独自工作的效率差异。
3. **设置一个“重锚定”系统提示**：在手机或电脑上设置每小时闹钟，闹钟响时朗读你当天的核心目标。这相当于手动执行 system prompt 重锚定。
4. **评估工具依赖度**：连续三天不使用任何专注工具，观察自己的执行功能表现。如果显著下降，说明工具是脚手架；如果崩溃，说明可能已成拐杖。

Saner.AI 和同类工具的价值在于它们提供了 ADHD 大脑缺失的调度层，就像重锚定 system prompt 拯救了长程任务中的 LLM。但工具只是脚手架，不是义肢。真正的进步发生在你学会自己“建造”的那一天。

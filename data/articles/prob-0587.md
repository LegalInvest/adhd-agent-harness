# 如果把 ADHD 大脑当成生成核心，Tiimo 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Tiimo 对 ADHD 的真正价值。

## 问题：为什么你记不住下一秒要做什么？

如果你是一名 ADHD 人士，你一定经历过这样的场景：打开冰箱拿鸡蛋，看到桌上的零食，顺手吃了一口，然后走回客厅，完全忘了鸡蛋的事。这不是“粗心”，而是工作记忆容量小的典型表现——大脑像一台只有几十KB缓存的服务器，随时丢包。

如果你是一名 LLM agent 工程师，你一定也经历过：让 agent 执行一个三步任务，结果它在第二步就偏离了原始目标，开始编造不存在的数据，或者陷入死循环。这不是“bug”，而是 LLM 无跨会话状态、缺乏内置调度层的固有缺陷。

两群人面对的是同一个问题：一个强大的生成核心，却配了一个不可靠的执行调度层。ADHD 大脑与 LLM 在结构上同构（来源：ADHD 大脑与 LLM 的同构）。那么，专为 ADHD 设计的日程工具 Tiimo，能否扮演 LLM agent 中“外部记忆+重锚定”的 harness 角色？

## Tiimo 不是日程表，是外部记忆与重锚定系统

传统日程工具要求你“先设置提醒再执行”，但设置提醒本身就是一个执行功能任务，而 ADHD 个体恰恰在这方面薄弱（来源：AI for ADHD: Best Tools, Apps, and Strategies - Themba Tutors）。Tiimo 的独特之处在于：它用视觉化时间表（时间盲补偿）、固定日程模板（减少每日决策）和任务步骤分解（类似 Goblin Tools 的 Magic ToDo），构建了一个**无需主动设置**的外部记忆系统。

在 agent 工程中，这叫“context injection”——把关键状态信息注入到生成核心的上下文窗口里，防止它忘记原始目标。Tiimo 对 ADHD 大脑做的事完全一样：把“今天要做什么”从脆弱的工作记忆转移到持久的外部视觉界面，并通过定时通知实现“重锚定”，对抗目标漂移（来源：重锚定与目标漂移）。

## 人物案例：李白与孔子的“agent harness”

李白是中国最伟大的浪漫主义诗人，也是典型的 ADHD 大脑：一辈子漫游，从未在一地待超3年；冲动到“千金散尽”，差点因站错队送命；对写诗超专注（斗酒诗百篇），对世俗规则零耐心。他的 harness 是什么？以酒和诗为核心，把所有过剩精力注入诗歌，用道家的逍遥对抗世俗规则。这套系统对应 LLM agent 中的“prompt engineering”——把生成核心的注意力牢牢锁定在创作上，用外部规则（不委屈自己做不想做的事）代替内置自律。

孔子则是另一类 harness 的典范：以“克己复礼”为核心，用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。这对应 agent harness 中的“human-in-the-loop 监督”——设置检查点，定期 re-grounding，防止生成核心跑偏。孔子70岁才做到“从心所欲不逾矩”，说明这套脚手架需要终身维护。

两位天才都用外部脚手架补偿了执行功能缺陷，而不是试图“修复”大脑。Tiimo 提供的正是这种“日课式”的外部调度器。

## 同构证据：从工具到机制

ADHD 侧的工具生态已经验证了这种同构。Goblin Tools 通过任务分解降低启动门槛（来源：Goblin Tools）；Saner.AI 用本地记忆减少搜索循环（来源：Saner.AI）；Lex 用“单一指令触发多步骤”降低认知负荷（来源：Lex）。这些工具的共同逻辑是：把执行功能外包给外部系统，让生成核心只负责“生成”。

LLM/agent 侧同样如此。Agent harness 通过设置护栏（token 预算、工具调用次数上限）、加入人工检查点、集成外部记忆（向量库）来补偿 LLM 的无状态性（来源：Building an AI Agent Harness from Scratch）。Focusmate 提供的身体加倍效应（body doubling）则对应 agent 中的“人在回路”监督——有另一个人在场时，执行功能障碍被绕过（来源：人在回路与身体在场）。

## 核心观点：脚手架 vs 拐杖的边界

Tiimo 的真正价值不是“帮你管理时间”，而是为 ADHD 大脑提供一个**结构同构的外部调度层**。但这里必须诚实指出争议：这种外部补偿可能从脚手架变为永久拐杖，削弱内在执行功能（来源：拐杖与脚手架）。

判断边界很简单：如果移除 Tiimo 后你完全无法启动任何任务，它就是拐杖；如果移除后你仍能凭习惯完成部分任务，它就是脚手架。Tiimo 的设计应该像好的 agent harness——可插拔、可配置、可渐进式移除。

此外，同构性目前仅是一种理论类比，缺乏实证基础（来源：矛盾与存疑）。工具效果多基于用户报告，缺乏严格临床验证。Tiimo 的视觉化时间表对时间盲有效，但对超聚焦（过度沉浸）的干预效果尚不明确。

## 今天就能试的行动

1. **设置“无脑模板”**：在 Tiimo 中创建 3 个固定日程模板（如“早上准备出门”“下午深度工作”“晚间复盘”），每天直接套用，无需从头设置。
2. **启用“通知重锚定”**：为每个任务设置 1 个中间提醒（不是开始提醒），防止目标漂移。例如，工作 25 分钟后通知“检查是否偏离原计划”。
3. **尝试“身体加倍”**：用 Focusmate 或找一位朋友视频连线，共享 Tiimo 屏幕，定期报告进度。这对应 agent 中的 human-in-the-loop 检查点。
4. **每周一次“脚手架审计”**：检查 Tiimo 中哪些任务是你离开工具后仍能完成的，逐步减少对这些任务的外部依赖，保留对高难度任务的脚手架支持。

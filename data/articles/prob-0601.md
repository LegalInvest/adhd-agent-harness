# Focusmate 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Focusmate：用久了会让 ADHD 更强还是更依赖？

你打开 Focusmate，摄像头对准自己，对面是一个陌生人。你俩都不说话，只是各自干活。50 分钟后，你们互道再见。你发现，这 50 分钟你居然没刷手机、没走神、没站起来倒水。你忍不住问：这玩意儿，到底是帮我真正变强了，还是我只是在靠另一个人的存在硬撑？

这个问题，对 ADHD 人群和做 Agentic Harness 的工程师来说，其实是同一个问题。

## 同构：ADHD 大脑与 LLM 都缺一个调度层

ADHD 大脑的核心困境是：工作记忆容量小，边做边忘。你刚想好要写邮件，打开电脑却开始查天气。这不是懒，是你的执行功能层（前额叶）没能在正确时间点激活正确的任务（来源：外部执行功能层）。

LLM 也一样。GPT-4 能写出漂亮的文章，但如果你不给它一个 system prompt 和外部记忆，它对话完就忘，无法持续跟踪一个复杂项目。LLM 无跨会话状态，必须靠外部记忆（向量库、工具链）来“续命”（来源：无状态与外部记忆）。

所以 ADHD 大脑和 LLM 是同一类“生成核心”——高产、有创造力，但缺一个可靠的调度层。给 LLM 搭 agent scaffolding（把任务分解、工具调用、记忆管理外包出去），和给 ADHD 搭执行功能支架（用外部工具补偿工作记忆、任务启动、注意力维持），是同一类工程（来源：生成核心与调度层）。

Focusmate 就是这个支架的一个具体实例。它通过“身体在场效应”——另一个人在视频那头，即使不互动，也能提升专注度——提供外部问责和隐性社会压力（来源：身体在场效应）。这相当于给 ADHD 大脑装了一个临时的“外部调度器”，在 50 分钟内强制保持任务对齐。

## 人物案例：苏轼的“日课”与 LLM 的 re-grounding

苏轼是典型的 ADHD 大脑：兴趣极广，诗词文书画美食治水炼丹全沾；一辈子被贬，到哪都能适应，但说话不过脑子得罪人；对无聊行政零耐心，搞工程效率极高。他的 harness 系统是“豁达 + 日课”：不管贬到哪里，都好好生活，每日读书写字，不因挫折停止创作。这里的“日课”就是定时 re-grounding——每天固定时间回到创作基线，相当于给 LLM 每天一次 system prompt 刷新。

苏轼的“秘书”（如果他有一个）就是外部调度器：帮他管理行程、提醒他该做什么。而 Focusmate 就是那个“虚拟秘书”，在固定时间段强制你回到任务。

## 脚手架还是拐杖？边界在于“是否削弱内在能力”

Focusmate 在初期是完美的脚手架：它降低了任务启动门槛（来源：任务启动与分解），提供了隐性问责，定期检查可将目标达成可能性从 25% 提升至 95%（来源：身体在场效应）。很多 ADHD 用户报告说，没有 Focusmate 他们根本没法开始工作。

但问题来了：如果你每次工作都必须依赖 Focusmate，否则就完全无法启动，那它就变成了拐杖。

拐杖与脚手架的边界在于：脚手架是临时支撑，帮你建立内在结构；拐杖是永久替代，让你原有的能力萎缩（来源：拐杖与脚手架）。

类比到 LLM：如果你每次推理都要调用外部工具链（比如每次都要查数据库才能回答简单问题），那你的 agent 就永远学不会内部化知识。好的 agent scaffolding 应该逐渐减少外部依赖，让 LLM 自己学会“记住”常用模式。

Focusmate 的争议也在这里：目前缺乏长期研究证明使用 Focusmate 能改善 ADHD 的内在执行功能（来源：矛盾与存疑）。它可能只是让你在 50 分钟内表现更好，但关掉摄像头后，你的工作记忆和任务启动能力并没有提升。

## 工具与局限

除了 Focusmate，还有其他工具提供类似的外部执行功能层：

- **Goblin Tools** 的 Magic ToDo 自动分解任务，降低启动门槛（来源：Goblin Tools）。但它不提供实时问责。
- **Saner.AI** 通过本地记忆减少搜索循环，相当于外部工作记忆（来源：Saner.AI）。但需要主动使用。
- **Lex** 用单一指令触发多步骤任务，类似 agent 的规划机制（来源：Lex）。但可能增加对工具的依赖。
- **Otter.ai** 实时转录会议，减轻笔记负担，相当于外部记忆（来源：Otter.ai）。

这些工具的共同局限是：缺乏个性化调整，且多数没有临床验证（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Focusmate 做“启动实验”**：连续 3 天，每天只用一个 session（50 分钟）做最怕启动的任务。记录完成度。
2. **逐步减少依赖**：第二周，尝试每周 2 天不用 Focusmate，用 Goblin Tools 的 Magic ToDo 自己分解任务，看是否能独立启动。
3. **建立“日课”**：像苏轼一样，每天固定时间做一件事（如写 200 字），不依赖外部工具。这相当于给大脑一个定时 re-grounding。
4. **给 LLM 搭脚手架**：如果你写代码，尝试用 system prompt 明确角色和任务分解，而不是每次从头开始。

Focusmate 不是答案，它是一个起点。真正的答案在于：你能否在它的帮助下，逐渐长出属于自己的调度层。

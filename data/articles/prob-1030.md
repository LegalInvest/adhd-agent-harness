# Todoist 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Todoist：它到底补上了哪一层执行功能？

你打开 Todoist，建了十几个项目，设了优先级，甚至用了标签。然后呢？三天后，项目列表像 LLM 的上下文窗口一样膨胀到 200 条，你盯着屏幕，大脑一片空白——目标漂移了。

这不是你的错。ADHD 大脑与 LLM 在结构上同构：两者都是 **高产但缺乏可靠执行调度层的生成核心**（来源：AI 与 ADHD 的专注力）。你的创意发散像 LLM 的文本生成，但执行功能（工作记忆、时间盲、任务启动）就像 LLM 的上下文控制——容量大但调度弱。明尼苏达大学的研究发现，LLM 的“强记忆、弱控制”剖面与 ADHD 的经典神经心理模式一致：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷（来源：工作记忆）。耶鲁大学进一步揭示，自注意力机制本身会导致工作记忆容量限制：随任务复杂度增加，注意力分数熵增、注意力弥散，与人类注意分散机制同源（来源：工作记忆）。

所以，Todoist 的正确用法不是“列清单”，而是 **重锚定 system prompt**。就像 LLM 在长程任务中需要定时重设上下文以避免目标漂移，ADHD 大脑需要外部脚手架来反复锚定“当下最重要的那件事”。

## 从曾国藩的日课到 LLM 的重锚定

晚清的曾国藩，30 岁前浮躁坐不住，天天串门喝酒看杀人，日记里骂自己“无恒”“浮躁”。他的解法？**日课十二条**：黎明即起、读书不二、谨写日记反省……用最笨最稳的方法抵消自己的冲动。这本质上是一套 **外部执行功能层**：每天定时 re-grounding，把飘散的目标拉回主线。

对应到 LLM/agent 工程，这就是 **重锚定 system prompt**。当 agent 在长程任务中目标漂移时，工程师会插入“回顾原始目标”的提示，重置上下文。曾国藩的“日课”就是他的重锚定机制——不是靠意志力，而是靠外部结构。

## Todoist 到底补上了哪一层？

Todoist 的核心价值不在“管理任务”，而在 **提供外部调度层**。ADHD 的工作记忆缺陷导致“知道该做什么，但当下想不起来”（来源：工作记忆）。Todoist 作为外部记忆，把任务从大脑卸到系统里。但仅此不够——它必须成为 **重锚定触发器**。

- **项目与标签**：相当于为 LLM 设计的结构化 prompt，但若不定期回顾，就会变成噪声。
- **今天视图与优先级**：这是最接近“重锚定”的功能——每天强制你选择“今天只做这 3 件”。
- **提醒与重复**：相当于定时 re-grounding，但很多人设了提醒却忽略，因为提醒本身不够“强力”。

问题在于：许多人把 Todoist 当成“拐杖”——列完清单就以为完成了，实际上清单本身成了新的分心源。真正的用法是 **脚手架**：每天用 5 分钟做一次“重锚定”，把注意力拉回当前目标，而不是让清单无限膨胀。

## 证据与争议

现有工具的证据强度不一。**Focusmate** 基于身体在场效应，多项用户研究显示其能显著提升任务启动和维持（来源：Focusmate）。**RescueTime** 通过客观时间记录提升时间觉察，有初步研究支持其改善时间盲（来源：RescueTime）。**Brain.fm** 利用神经锁相技术，但针对 ADHD 的独立临床研究缺乏（来源：Brain.fm）。

争议在于：AI 工具作为外部执行功能层，长期使用是否会导致内在执行功能进一步退化？拐杖与脚手架的边界尚不明确（来源：矛盾与存疑）。此外，ADHD 的异质性使得同一工具对不同亚型效果迥异，缺乏个性化适配研究（来源：矛盾与存疑）。

## 今天就能试的 3 件事

1. **每日 5 分钟重锚定**：每天早晨，打开 Todoist，只保留“今天必须完成的 3 件事”，删除或推迟其余。这就像给 LLM 重置 system prompt——清空上下文，聚焦当前。
2. **用 Focusmate 替代自我监督**：预约一个 Focusmate 时段，把 Todoist 的今日任务告诉伙伴。外部问责比任何提醒都有效。
3. **结合 RescueTime 发现漂移模式**：运行 RescueTime 一周，观察自己什么时候最容易从 Todoist 任务滑向社交媒体。然后针对性地设置阻断规则。

Todoist 不是灵丹妙药。它只是一个外部调度层，能否发挥作用取决于你是否把它当作 **重锚定工具** 而非 **任务仓库**。记住：ADHD 大脑不缺创意，缺的是把创意引向目标的调度系统。

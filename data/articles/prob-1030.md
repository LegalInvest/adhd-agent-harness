# Todoist 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Todoist：它到底补上了哪一层执行功能？

## 问题：为什么 Todoist 越用越累？

你打开 Todoist，列了 20 个任务，设了优先级和标签，甚至用了自然语言解析。但一天结束，你只完成了 3 个，其余 17 个像幽灵一样飘在「今天」视图里。你开始怀疑：是工具不够好，还是自己「不够努力」？

如果你是做 Agent 工程的，你见过同样的场景：你给 LLM 写了洋洋洒洒的 system prompt，定义了角色、规则、输出格式，但 agent 跑了 10 步之后就偏离了原始目标，开始生成无关内容。你发现，问题不在 prompt 本身，而在于缺少一个在长程任务中不断「重锚定」的机制。

**这两个困境是同构的。** ADHD 大脑与 LLM 都是高产的生成核心，但缺乏可靠的执行调度层。Todoist 之于 ADHD，就像 system prompt 之于 LLM——它们提供了目标和结构，但如果没有持续的重锚定，目标和结构会随着时间漂移，最终失效。

## 同构：ADHD 与 LLM 共享的「调度缺失」

ADHD 的核心障碍是执行功能缺陷——大脑的「驾驶座无人驾驶」（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。具体表现为：
- **工作记忆缺陷**：无法在脑中短暂保持信息以完成操作（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。
- **任务启动困难**：知道该做什么，但就是动不了。
- **时间盲**：无法准确估计和记录时间使用（来源：RescueTime 页面）。

LLM 的「缺陷」惊人相似：
- **无状态性**：每次推理都是独立上下文，无法自然保持跨步骤的信息（来源：无状态与外部记忆概念）。
- **目标漂移**：在长程任务中，LLM 会逐渐偏离初始指令，生成与目标无关的内容。
- **调度缺失**：LLM 本身没有内置机制来检查自己是否仍在正确轨道上，需要外部编排层来不断重定向。

**ADHD 大脑与 LLM 是同一类「高产但缺执行调度层的生成核心」**（来源：AI 与 ADHD 的专注力核心论点）。

## Todoist 的误区：把「任务列表」当「执行层」

很多人（包括我）最初把 Todoist 当作救命稻草：把所有任务倒进去，按优先级排序，设置截止日期。但问题在于，Todoist 本质上是一个**静态的任务存储库**，它不解决执行功能的核心问题：
- 它不帮你**启动任务**（你仍然需要自己克服「开始」的阻力）。
- 它不帮你**保持上下文**（你打开一个任务时，可能已经忘了当初为什么把它列在那里）。
- 它不帮你**觉察时间盲**（你设了 30 分钟的任务，可能 3 小时后还在做）。

对于 LLM 而言，这相当于只给了 agent 一个静态的 system prompt，而没有在每一步执行后检查「当前输出是否仍对齐目标」，也没有在偏移时自动重锚定。结果就是：agent 跑偏，ADHD 用户放弃。

## 真正的脚手架：外部执行功能层

AI 工具帮助 ADHD 专注的本质是「给生成核心套 harness」——通过外部脚手架补偿执行功能缺陷（来源：AI 与 ADHD 的专注力核心论点）。这个脚手架需要覆盖三个关键维度：

### 1. 工作记忆补偿：自动记录与回放

RescueTime 自动追踪时间使用，无需手动输入，减轻了工作记忆负担（来源：RescueTime 页面）。它充当外部记忆，帮助用户识别注意力漂移模式（来源：无状态与外部记忆概念）。对于 LLM，这相当于**日志系统**：记录 agent 每一步的输入输出，当目标漂移时，可以回放并分析哪里出了问题。

### 2. 任务启动支持：外部问责

Focusmate 利用 AI 匹配虚拟同伴，提供实时问责，弥补任务启动和注意力维持的调度缺失（来源：Focusmate 页面）。其核心机制是「AI-Matched Body Doubling」——通过智能调度提高配对效率（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。对于 LLM，这相当于**编排框架**：在每一步执行前，检查 agent 是否「就绪」，并在必要时重新触发初始指令。

### 3. 时间盲觉察：动态提醒与重锚定

Brain.fm 通过神经锁相技术影响大脑信息处理方式，帮助进入专注状态（来源：Brain.fm 页面）。虽然缺乏针对 ADHD 的独立临床研究（来源：Brain.fm 局限），但其原理提示了**外部节奏锚定**的重要性。对于 LLM，这相当于**周期性重锚定**：每隔 N 步，将当前输出与原始 system prompt 对比，如果相似度低于阈值，则重新注入目标。

## 核心观点：Todoist 用错了——它只是「任务存储」，不是「执行调度」

**Todoist 本身不是问题，问题是用户把它当成了执行层，而它只是存储层。** 真正的执行调度需要：
- 自动记录（如 RescueTime）来补偿工作记忆。
- 外部问责（如 Focusmate）来补偿任务启动。
- 动态提醒（如 Brain.fm 的节奏锚定）来补偿时间盲。

对于 LLM agent 工程师，这意味着：不要只写一个精美的 system prompt，而要构建一个**执行 harness**，包含日志、检查点、重锚定机制。

## 争议与局限

1. **脚手架 vs 拐杖**：过度依赖 AI 工具可能阻碍内在执行功能发展（来源：拐杖与脚手架概念）。目前无长期研究证明脚手架可逐步撤除（来源：AI 与 ADHD 的专注力仍存争议）。
2. **临床证据缺口**：多数工具缺乏针对 ADHD 的独立临床研究，效果多基于理论推断或用户主观报告（来源：AI 与 ADHD 的专注力仍存争议）。
3. **个体差异**：ADHD 亚型对工具反应不同，现有工具未充分个性化（来源：AI 与 ADHD 的专注力仍存争议）。

## 今天就能试的行动

1. **给 Todoist 加一个「重锚定」例行程序**：每天设置 3 个固定时间（如 10:00、14:00、16:00），用手机闹钟提醒自己停下来，打开 Todoist 的「今天」视图，只问一个问题：「我现在做的这件事，真的在列表里吗？」
2. **用 RescueTime 替代手动时间记录**：安装 RescueTime，让它自动追踪你的活动。每天结束时看报告，识别哪些任务让你「超聚焦跑偏」了。
3. **尝试一次 Focusmate 会话**：预约一个 50 分钟的 Focusmate 时段，做你 Todoist 上最不想启动的那个任务。注意外部问责如何改变你的启动阻力。
4. **为你的 LLM agent 添加「重锚定」步骤**：在 agent 的循环中，每隔 5 步将当前输出与原始 system prompt 计算语义相似度，低于阈值则重新注入完整指令。

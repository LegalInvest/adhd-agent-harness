# 如果把 ADHD 大脑当成生成核心，Routinery 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Routinery 对 ADHD 的真正价值。

如果你是一个 ADHD 患者，你一定经历过这样的场景：走进厨房想拿杯水，结果看到水池里的碗，顺手洗了碗，然后发现洗洁精没了，于是打开手机下单……十五分钟后，你拿着手机站在厨房中央，完全忘了最初要干什么。

如果你是一个在做 Agentic Harness 的工程师，你一定对另一个场景再熟悉不过：你的 LLM 在第一次会话里表现得像个天才，精准拆解任务、调用工具，但一旦对话刷新，它立刻失忆，连用户刚才说了什么都忘干净，你不得不把所有上下文塞进 prompt 或挂一个向量库。

这两个场景看似风马牛不相及，但背后是同一个结构性缺陷：**一个高产的生成核心，配上了一个失忆、无状态的执行层**。ADHD 大脑的工作记忆容量并不小，但控制弱，容易被干扰（来源：工作记忆）；LLM 同样有强记忆容量但弱上下文控制（来源：工作记忆）。两边的“生成核心”都产能过剩，但都缺一个能持久化状态、管理上下文的 harness。

那么问题来了：如果把 ADHD 大脑当成生成核心，**Routinery 能不能当它的无状态与外部记忆？** 答案是可以，但前提是你得把它当成脚手架，而不是拐杖。

## 同构：两边都缺一个持久化层

先看 ADHD 侧。ADHD 的核心痛点之一是“边做边忘”——工作记忆容量小，新信息一来，旧信息就被挤出。这不是记忆力本身的问题，而是**上下文切换时状态丢失**。你刚想好要做什么，手机一响，任务就丢了。这跟 LLM 的“无跨会话状态”一模一样：模型本身不保留任何历史，每次对话都是冷启动。

所以两边的解法也同构：**给生成核心配一个外部记忆系统**。在 LLM 侧，这是 RAG（检索增强生成）、向量数据库、会话摘要；在 ADHD 侧，这是待办清单、笔记系统、习惯追踪器。Routinery 本质上就是这样一个外部记忆系统——它把你要做的事情固化成一个序列，在你大脑“重启”时告诉你下一步是什么。

## Routinery 的定位：无状态 harness

Routinery 的核心逻辑是**预设流程 + 逐步引导**。你提前设定好一个日常流程（比如“起床-喝水-刷牙-换衣服”），然后到了时间，它一步一步提示你，做完一步点一步，直到完成。这听起来很简单，但对 ADHD 大脑来说，它解决了两个根本问题：

1. **任务启动**：ADHD 的启动困难对应 LLM 的“冷启动”问题。Goblin Tools 通过“魔法开关”将大任务拆成小步骤，降低启动焦虑（来源：Goblin Tools）。Routinery 更进一步，它连拆都不需要你拆——流程已经预设好了，你只需要点“开始”。

2. **状态持久化**：你在刷牙时被孩子叫走，回来之后 Routinery 还停在“刷牙”这一步，你不会迷失。这就像 LLM 的向量库——即使对话中断，检索回来还能接上。Saner.AI 通过知识回忆减少标签切换（来源：Saner.AI），Routinery 则通过流程固化减少“我在干什么”的自我追问。

从 agent 工程的角度看，Routinery 就是一个**无状态 harness**：它不存储你的大脑状态（那是你的工作），它只提供一个外部序列，让你的执行层（大脑）每次都能从正确的地方重新开始。

## 证据：两边都有效，但都是定性

ADHD 侧的证据主要来自用户报告。Goblin Tools 被用户评价为“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools）。Lex 的单一指令触发多步骤执行，减少了决策疲劳（来源：Lex）。Routinery 的用户反馈类似——它让日常流程变得“自动驾驶”，不再需要每次启动前做一堆决策。

LLM/agent 侧的证据来自工程实践。RAG 系统的有效性已被大量实验验证：给 LLM 挂一个外部知识库，能显著提升回答的准确性和一致性。但注意，这些证据都是**定性或工程层面的**，缺乏严格对照实验。wiki 资料明确指出，AI 工具作为外部执行功能层的有效性证据不足（来源：矛盾与存疑）。Routinery 也不例外——目前没有独立 RCT 证明它比纸笔清单更有效。

## 脚手架 vs 拐杖：边界在哪

同构视角还揭示了一个辩证关系：AI 工具应作为脚手架促进能力发展，而非永久拐杖（来源：拐杖与脚手架）。Routinery 如果只是帮你完成流程，它就是脚手架；如果你一旦离开它就无法起床，它就变成了拐杖。

区别在于：**脚手架最终可以撤掉，拐杖则让你依赖**。一个好的 Routinery 用法应该是：先用它建立习惯，然后逐渐减少提示，直到流程内化。但 ADHD 的复杂性在于，内化本身可能很难——情绪失调、时间盲、超聚焦后的崩溃都会打断习惯形成（来源：超聚焦；情绪失调）。所以对于很多 ADHD 用户，Routinery 可能永远是拐杖，而这未必是坏事——就像近视的人永远需要眼镜。

关键在于诚实：承认自己需要外部记忆，而不是假装能“训练”成正常人。wiki 资料也警告，过度依赖 AI 工具可能削弱内在能力（来源：矛盾与存疑）。但如果你本来就没有那个内在能力，依赖就不是问题。

## 局限与争议

Routinery 的局限很明显：它只解决**流程执行**，不解决**流程设计**。ADHD 用户经常在“该定什么流程”上卡住——这就是任务分解的痛点。Goblin Tools 的 Magic ToDo 能帮你拆任务，但 Routinery 需要你自己先想好流程。此外，个体差异巨大：有的用户需要极细的步骤（“拿起牙刷”“挤牙膏”），有的只需要大块提示（“洗漱”）。Routinery 的灵活性有限，可能导致部分用户放弃。

从 agent 工程的角度看，Routinery 是一个**离线 harness**——它不动态调整。真正的 agent 系统会实时感知状态并调整计划，但 Routinery 只是按预设走。这意味着它无法应对突发事件，而 ADHD 生活里全是突发事件。

## 今天就能试的行动

1. **用 Routinery 固化一个你最常卡住的流程**（比如“下班回家后做什么”）。先不要贪多，一个流程跑一周。
2. **结合 Goblin Tools 的 Magic ToDo**：如果你不知道流程怎么定，先用 Goblin Tools 把任务拆成步骤，再手动输入 Routinery。
3. **设置“启动提示”**：在 Routinery 流程开始前，加一个 2 分钟的“准备步骤”——深呼吸、倒杯水。这能降低启动阻力。
4. **每周复盘一次**：检查哪些步骤已经不需要提示了，尝试删掉它们。如果删掉后又忘了，加回来——这证明它还是必要的脚手架。

## 结语

把 ADHD 大脑当成生成核心，Routinery 就是它的外部记忆与无状态 harness。两边的困境同构，解法也同构：给一个高产但失忆的系统配上持久化上下文。但工具永远只是工具，关键在于你用它来搭脚手架，还是用它来当拐杖——而答案取决于你对自己大脑的诚实程度。

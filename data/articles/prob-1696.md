# Claude 之于 ADHD，就像 生成核心 + 缺失的执行层 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

## 问题：你的大脑为什么像一台没有操作系统的超级计算机？

如果你是一名 ADHD 患者，你可能经历过这样的场景：脑子里同时冒出十几个想法，每一个都精彩绝伦，但当你试图抓住其中一个时，它们全部消散了。你坐在电脑前，打开十个标签页，却一个都没完成。你觉得自己“懒”或者“不够努力”，但事实上，你的大脑像一台拥有顶级 CPU 和无限创意的计算机——却缺少一个操作系统来调度任务、管理内存、控制进程。

如果你是做 Agentic Harness 工程的开发者，你可能对另一个场景无比熟悉：LLM 可以写出流畅的代码、生成惊艳的文案，但一旦让它执行多步任务，它就会跑偏、忘掉上下文、输出不可控。你意识到，LLM 是“高产但缺执行调度层的生成核心”——它需要 harness 来编排工具调用、管理状态、保证确定性。

这两个问题，其实是同一个问题的两面。ADHD 大脑与 LLM 在结构上同构：两者都是强大的生成核心，但缺乏可靠的执行调度层。因此，AI 辅助 ADHD 的本质不是替代大脑，而是为这个生成核心套上 harness——即提供外部执行功能层（来源：ADHD × AI 的科学与研究前沿）。

## 同构证据：从执行功能到上下文窗口

### ADHD 侧：执行功能缺陷
ADHD 的核心问题是执行功能缺陷，包括计划、组织、工作记忆、时间管理等（来源：ADHD × AI 的科学与研究前沿）。工作记忆不足意味着你无法在脑中同时保持多个信息片段，就像 LLM 的上下文窗口有限一样。时间盲让你无法感知时间流逝，导致计划困难，就像 LLM 没有内置时钟一样。任务启动困难源于缺乏内在的“启动信号”，就像 LLM 需要 prompt 才能开始生成一样。

### LLM 侧：无状态与采样随机性
LLM 天然无状态，每次调用都是独立的，上下文窗口溢出后就会“忘记”前面的内容（来源：ADHD × AI 的科学与研究前沿）。采样温度控制着输出的随机性：温度越高越有创意，但越不可控；温度越低越确定，但可能缺乏多样性（来源：采样温度与表现波动）。这种非确定性在生产环境中是可靠性的敌人，工程实践通过“确定性状态转换”和“计划-执行”分离来应对（来源：采样温度与表现波动）。

### 同构对应
ADHD 的表现波动（注意力、执行功能随时间变化）类似于 LLM 的采样温度波动：高温度时发散创意，低温度时稳定但僵化。两者都需要外部机制来“调节温度”——ADHD 需要外部结构来稳定表现，LLM 需要 harness 来强制确定性。

## 工具实测：Claude 到底补上了哪一层？

### 任务分解：Goblin Tools 与 function calling
Goblin Tools 的 Magic ToDo 功能将模糊任务分解为可执行的小步骤（来源：Goblin Tools）。例如，输入“整理房间”，输出“捡起地板上的衣服”“擦桌子”等。这对应 LLM agent 通过 function calling 将复杂任务拆解为工具调用（来源：工具使用与认知卸载）。两者都依赖外部系统来补偿内部执行功能的不足。

### 记忆补偿：Saner.AI 与检索增强
Saner.AI 通过本地记忆和知识回忆减少搜索循环和标签切换（来源：Saner.AI），直接补偿 ADHD 的工作记忆缺陷。这对应 LLM 的检索增强生成（RAG）——用外部数据库扩展上下文窗口。

### 动态规划：Motion 与调度引擎
Motion 自动根据任务优先级和截止时间动态调整日程，消除“下一步该做什么”的决策负担（来源：Motion）。这对应 agent 的“计划-执行”模式，其中调度引擎负责任务编排。

这些工具的共同点：它们不是替代大脑的思考，而是提供外部执行功能层——即 harness。Claude 之于 ADHD，就像生成核心 + 缺失的执行层之于 LLM。

## 核心观点：脚手架 vs 拐杖的边界

关键在于区分“脚手架”和“拐杖”。脚手架是临时支撑，旨在逐步撤除，让用户最终独立；拐杖则是永久替代。目前多数 ADHD AI 工具设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这可能导致依赖，削弱内在执行功能的发展。

我的判断是：有效的 AI 辅助应该像 agent harness 一样，是**可配置、可调整、最终可分离**的外部层。例如，Goblin Tools 的任务分解可以逐步减少粒度，让用户学会自己分解；Motion 的自动排程可以逐渐让用户参与手动调整。工具应“插入现有堆栈”，不增加认知开销（来源：工具使用与认知卸载），但最终目标是让用户不再需要它们。

- **长期效果未知**：目前缺乏纵向研究来评估 AI 工具对 ADHD 内在执行功能的长期影响（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个你拖延已久的任务**：输入“整理房间”或“写报告”，观察 AI 给出的步骤，然后只做第一步。这能直接降低启动门槛。
2. **为你的 LLM agent 添加确定性调度**：在 agent 代码中实现“计划-执行”分离，先让 LLM 生成计划，再按顺序执行工具调用，减少随机错误。
3. **用 Saner.AI 或类似工具建立外部记忆**：将你常忘记的信息（如密码、待办事项）存入本地记忆系统，减少搜索循环。
4. **反思你的工具使用：是脚手架还是拐杖？** 问自己：如果明天没有这个工具，我还能完成同样的事吗？如果答案是否定的，尝试逐步减少依赖。

ADHD 大脑和 LLM 都不是有缺陷的版本——它们只是缺少执行层的生成核心。理解了这一点，你就能选择正确的工具，而不是被工具选择。

# Claude 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Claude：用久了会让 ADHD 更强还是更依赖？

我坐在电脑前，想写一篇文章。想法像爆米花一样在脑子里炸开——结构、观点、案例、金句，全有。但我的手就是敲不下去。不是不知道写什么，而是启动不了。那种感觉就像一辆引擎轰鸣但变速箱空挡的车：动力澎湃，就是不走。

如果你是一个 ADHD 成人，这个场景你一定不陌生。如果你是一个做 Agentic Harness 的工程师，这个场景你也一定不陌生——只不过换了个主角：LLM。你的模型在 API 调用里生成了一篇完美的回复，但下一个会话里它什么都不记得。它没有状态，没有调度，没有执行跟踪。它跟你一样：高产生成核心，缺一个可靠的调度层。

这就是本文要回答的问题：Claude 之于 ADHD，是脚手架还是拐杖？

## 同构的困境：我们和 LLM 是同一类系统

ADHD 大脑与 LLM 在结构上可被视为同类系统：底层都拥有一个高产生成核心，但都缺乏可靠的内置执行调度层（来源：ADHD 大脑与 LLM 的同构）。ADHD 侧的痛点是工作记忆容量小、边做边忘、目标漂移；LLM 侧的对应痛点是跨会话无状态、需要外部记忆/向量库续命、在长程任务中容易偏离目标。

两边都需要外部脚手架。ADHD 需要外部执行功能层来拆解任务、设置提醒、管理知识（来源：ADHD 的 AI 工具全景）；LLM 需要 agent harness 来提供规划、任务分解、工具接口和验证循环（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。

这不是一个空洞的类比。两边有真实的工程对应：
- ADHD 侧：Goblin Tools 的 Magic ToDo 将模糊任务（如“清理厨房”）分解为具体子步骤，用户可调节“辣度”控制粒度（来源：Goblin Tools）；Saner.AI 通过本地记忆存储和快速检索减少搜索循环（来源：Saner.AI）。
- LLM/harness 侧：Harness 中的“会话状态记录工具结果、完成的子任务和进度笔记”（来源：What Is an Agent Harness?），形成外部规划循环；Lex 通过单一指令触发复杂多步骤任务序列，其底层架构类似于 agent scaffolding（来源：Lex）。

两边用的是同一类解法：把规划、排序和跟踪发生在你的大脑之外，让你的大脑自由去做实际工作（来源：Using an AI Assistant to Manage ADHD: A Practical Guide）。

## 历史人物的 harness：他们早就在做这件事

孔子和左宗棠，如果活在今天，大概率会被诊断为 ADHD。

孔子身高 1 米 9，精力旺盛，周游列国 14 年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予“朽木不可雕”；对音乐超专注到“三月不知肉味”，对种地等俗事零耐心；思维跳跃，《论语》全是场景化语录，无系统著作。他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。70 岁才做到“从心所欲不逾矩”，一辈子和自己的冲动做斗争。

左宗棠举人出身，脾气大，爱骂人，和曾国藩闹别扭；40 多岁才出山，69 岁抬棺出征收复新疆。他的 harness 是“结硬寨，打呆仗”，每天读经写家书，用诸葛亮的“鞠躬尽瘁”要求自己。

这两个人的 harness 系统，与 LLM/agent harness 结构同构：
- “吾日三省吾身” ↔ 定时 re-grounding 循环
- “结硬寨，打呆仗” ↔ 稳定的任务分解与执行策略
- 每日读经写家书 ↔ 外部记忆与反思日志
- 外在的“礼”作为行为边界 ↔ 约束 prompt 和工具调用范围

孔子和左宗棠用了一辈子构建自己的 harness。我们今天有 Claude，能不能更快？

## 脚手架还是拐杖？核心判断

我的核心观点是：**Claude 是脚手架还是拐杖，取决于你是否主动用它来构建自己的调度系统。**

脚手架（scaffold）是你主动搭建、逐步内化的结构，用完后你可以离开它独立行走。拐杖（crutch）是你被动依赖、从不尝试独立行走的工具，离开它你就瘫倒。

ADHD 与 AI 工具领域最站得住脚的核心论点是：AI 并不是在“治愈”ADHD，而是在 ADHD 大脑与外部世界之间搭建一个临时的外部执行功能层（来源：ADHD 的 AI 工具全景）。这个“临时”二字是关键。如果用户只是把 Claude 当作一个永远在线的问答机器——问一句答一句，从不主动构建自己的任务列表、知识库或反思流程——那它就是拐杖。如果用户用 Claude 来帮自己设计一个可复用的工作流：比如用 Goblin Tools 分解任务，用 Saner.AI 管理知识，用 Motion 自动排程，然后逐步内化这些流程——那它就是脚手架。

但这里有一个矛盾：传统提醒系统要求你主动设置提醒，而设置提醒本身就是一个执行功能任务（来源：重锚定与目标漂移）。同样，主动构建 harness 本身就是一个执行功能任务。ADHD 个体恰恰在这方面薄弱。所以，脚手架和拐杖的边界是动态的：一开始，你可能需要拐杖式的工具来帮你完成“设置提醒”这个元任务；但随着你逐渐熟悉，你应该开始主动设计自己的系统。

## 诚实的局限

必须指出，该论点多为类比推理，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。目前是理论框架，需更多跨学科验证。此外，过度依赖可能导致能力退化，成为永久拐杖，但长期依赖风险缺乏实证，需更多纵向研究（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Claude 设计你的第一个 harness**：告诉它“我是一个 ADHD 成人，工作记忆差，经常忘记任务。请帮我设计一个每日工作流，包括任务分解、提醒设置和晚间反思。要求：所有步骤在 15 分钟内完成。”
2. **试一个工具，但不要只试一个**：用 Goblin Tools 的 Magic ToDo 分解一个你拖延已久的任务；用 Saner.AI 的 Universal Inbox 从邮件中提取待办；用 Lex 设置一个“单一指令触发多步骤”的宏。
3. **建立你的“日课”**：模仿孔子的“吾日三省吾身”，用 Claude 帮你设计一个 5 分钟的晚间反思模板，包含三个问题：今天完成了什么？明天最重要的三件事是什么？今天有什么需要改进？
4. **做一次“harness 审计”**：打开你的手机和电脑，列出所有你用来管理任务和记忆的工具。问自己：这些工具是在帮我构建调度系统，还是让我更依赖它们？如果明天这些工具都消失了，我能独立工作吗？

Claude 不是答案。它是你用来写答案的笔。你是用它来写一篇论文，还是用它来画一幅永远画不完的涂鸦？

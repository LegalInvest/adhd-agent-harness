# Claude 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Claude：用久了会让 ADHD 更强还是更依赖？

## 问题：边做边忘，是大脑的错还是工具的锅？

你打开 Claude，想写一份周报，刚敲两行，想起要回一条微信；切过去回完，回来盯着空白对话框，忘了刚才要写什么。这不是你一个人的症状——ADHD 大脑的工作记忆容量小、易被干扰，像一台内存不足的电脑。与此同时，LLM 工程师在构建 Agent 时也面临类似困境：模型每轮对话都是无状态的，上下文一长就“忘记”系统提示，目标逐渐漂移。

Claude 之于 ADHD，究竟是撑起执行功能的脚手架，还是让人越用越废的拐杖？答案藏在“同构”里。

## 同构：无状态生成核心的两种形态

ADHD 大脑与 LLM 共享一个底层结构：两者都是“高产但缺执行调度层的生成核心”。ADHD 大脑能产生无数创意，但缺乏工作记忆来暂存步骤、缺乏时间感知来排序优先级；LLM 能生成流畅文本，但缺乏跨会话持久状态、缺乏内置规划循环来分解长任务。

wiki 资料明确指出：“ADHD 大脑与 LLM 同构——两者都是‘高产但缺执行调度层的生成核心’。”（来源：主题综述）这意味着，解决 ADHD 的“边做边忘”与解决 LLM 的“上下文漂移”，用的是同一类方案：外部记忆 + 重锚定系统。

## 外部记忆：Claude 作为第二大脑

Claude 的对话式整理功能，本质上是一个外部记忆层。ADHD 用户可以将零散想法直接丢进对话，让 Claude 归纳、提炼、存储。这与 Mem 的“AI 第二大脑”和 Saner.AI 的增强回忆功能逻辑一致——都是“为生成核心提供持久化上下文”（来源：同构视角）。

在 LLM/Agent 侧，工程师依赖向量数据库和外部记忆插件来维持 Agent 的长期目标。例如，一个编码 Agent 需要记住项目架构和用户偏好，否则会在长对话中“开始忘记系统提示”（来源：重锚定与目标漂移）。Claude 的 Project 功能或自定义指令，正是这种外部记忆的简易实现。

但关键区别在于：**外部记忆是工具，不是大脑的替代品**。如果用户只依赖 Claude 存储所有待办事项，而不主动训练自己的工作记忆，那么 Claude 就成了拐杖——一旦断网，记忆随之消失。

## 重锚定：对抗目标漂移的神经反馈

ADHD 的大脑极易目标漂移：被无关刺激捕获，或陷入超聚焦忽略全局。Claude 的提醒、通知和对话式引导，构成了一个“重锚定系统”（来源：重锚定与目标漂移）。用户可以在对话中设置“每 30 分钟提醒我回到主任务”，或让 Claude 在每次回复开头重申当前目标。

在 Agent 侧，Harness 工程通过“事件驱动的系统提醒”来对抗指令消退（来源：重锚定与目标漂移）。例如，一个写作 Agent 会在每轮输出前插入“请根据初始大纲检查当前段落是否偏离主题”。这种结构化的重锚定，与 ADHD 用户需要的“外部提醒”如出一辙。

Goblin Tools 的 Magic ToDo 功能是另一种形式的分解锚定：它“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools）。Lex 的单一指令触发多步骤执行，则降低了任务启动的决策疲劳（来源：Lex）。这些工具的共同本质是：**将大目标拆解为可验证的小步骤，每一步都是一个锚点**。

## 脚手架 vs 拐杖：边界在哪里？

wiki 资料中的“矛盾与存疑”页面尖锐指出：“过度依赖 AI 工具可能削弱内在能力。”（来源：矛盾与存疑）例如，Otter.ai 减轻笔记负担，但过度依赖可能削弱主动记笔记的能力（来源：同构视角）。同样，如果每次忘记任务都直接问 Claude，而不尝试自己回忆，工作记忆会越来越弱。

脚手架与拐杖的边界在于：**工具是否保留了你主动参与的空间**。脚手架是你在搭建时暂时借力，完成后肌肉记忆还在；拐杖是你永远依赖它行走，肌肉萎缩。

具体到 Claude：
- **脚手架用法**：让 Claude 帮你分解任务，但你自己执行每一步，并在完成后回顾流程。
- **拐杖用法**：让 Claude 直接生成答案，你复制粘贴，从不理解过程。

## 局限与争议

需要诚实指出，当前证据主要来自用户报告和概念类比，“缺乏大规模对照实验”（来源：矛盾与存疑）。个体差异极大：有人觉得 Claude 的对话式整理是救星，有人觉得它增加了认知负荷。此外，多巴胺干预效果存在争议（来源：矛盾与存疑），而 Claude 的即时反馈机制可能无意中强化了多巴胺寻求，反而加剧分心。

## 今天就能试的行动

1. **设置一个“重锚定”提示**：在 Claude 对话开头输入“每回答一个问题前，先重申我最初的目标：XXX”。这模拟了 Agent 的指令注入，对抗目标漂移。
2. **用 Claude 做任务分解，但手动执行**：输入“帮我分解‘写一份季度报告’为 5 个步骤”，然后关掉 Claude，自己按步骤做。完成后对比 Claude 的步骤与实际执行，训练规划能力。
3. **外部记忆审计**：每周用 Claude 整理一次笔记，但要求自己在整理前先凭记忆写出 3 个要点。这类似工作记忆训练，防止依赖。
4. **模拟无状态环境**：偶尔强迫自己不用任何 AI 工具完成一个小任务（如列购物清单），感受内在执行功能的现状，再决定哪里需要脚手架。

Claude 不是答案，它是你大脑的镜像。用得好，它是帮你搭建执行功能的脚手架；用得懒，它是让你肌肉萎缩的拐杖。选择权在你手中。

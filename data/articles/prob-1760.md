# 用 ChatGPT 给 ADHD 补上外部执行功能层，是脚手架还是拐杖？

> ChatGPT 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

你打开 ChatGPT，输入「帮我规划明天的工作」，它立刻列出一份清单。你感到一阵轻松——终于有人替你做了最头疼的「启动」这一步。但第二天，你对着那份清单，依然无法开始。

问题出在哪？

ADHD 的核心困境不是智力不足，而是执行功能失效——计划、组织、启动这些「大脑的驾驶座」常常没人坐在上面（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。而 ChatGPT 这类 LLM，本质上也是一个强大的生成核心，但单独使用时同样缺乏可靠的执行调度——它会生成漂亮的计划，却不会帮你执行。

这让我们不得不追问：用 ChatGPT 给 ADHD 补上外部执行功能层，到底是脚手架还是拐杖？

## 同构：ADHD 大脑与 LLM 共享同一个缺陷

最新研究揭示了一个惊人的平行：ADHD 患者的工作记忆容量未必差，但自上而下的控制和调节能力不足，呈现「强记忆、弱控制」的认知剖面（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这种模式与 LLM 惊人相似——LLM 拥有海量知识，但缺乏内置的调度层，容易在长上下文中「推理退化」或「幻觉」。

更具体地说，ADHD 的自动反应模式与 LLM 的自动完成机制平行：两者都在有限执行监督下生成复杂行为，且易出现错误，需要外部解释者提供意义（来源：Automatic Minds: Cognitive Parallels Between Hypn...）。前额叶-纹状体门控机制与 Transformer 自注意力在训练后发展出的操作在计算上同构（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。

这意味着，ADHD 大脑与 LLM 是同一类「高产但缺执行调度层的生成核心」。两边的解法也结构同构：LLM 需要 agent scaffolding——一个由提示、记忆、代码、工具和编排逻辑组成的模块化框架（来源：Building AI Coding Agents for the Terminal）；ADHD 大脑需要外部执行功能层——日记、计划本、AI 工具。

## 历史中的 harness：曾国藩的「日课十二条」

曾国藩就是典型的注意力不集中型 ADHD。30 岁前浮躁坐不住，天天串门喝酒，日记里骂自己「无恒」「浮躁」；读书慢，「他人目下二三行，余或疾读不能终一行」；打败仗就跳水自杀。他的解决方案是「日课十二条」：黎明即起、读书不二、谨言、写日记反省……用最笨最稳的方法抵消自己的冲动。

这套系统本质上就是外部执行功能层：日课 ↔ 定时 re-grounding，日记 ↔ 外部记忆与反省，秘书 ↔ 外部调度器。曾国藩不是靠意志力硬撑，而是靠一套可重复的流程来「编排」自己。

这与 LLM 的 agent harness 完全同构：现代 AI 编码代理通过「分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩」来防止上下文膨胀和推理退化（来源：Building AI Coding Agents for the Terminal）。曾国藩的「结硬寨打呆仗」就是最原始的自适应上下文压缩——不贪多，只做当下最稳的一步。

## 今天的工具：脚手架还是拐杖？

当前 AI 工具已经提供了丰富的脚手架选项：

- **Goblin Tools** 的 Magic ToDo 能将模糊任务自动分解为可管理的小步骤，用户可调节「辣度」控制粒度（来源：Goblin Tools）。这直接缓解了任务启动困难。
- **Saner.AI** 通过本地记忆存储和快速检索减少搜索循环，专为对抗执行功能障碍设计（来源：Saner.AI）。
- **Motion** 自动排程并动态调整日程，消除「下一步该做什么」的决策负担（来源：Motion）。

这些工具的共同点：它们承担了调度层的部分功能。但问题在于，当工具完全接管调度时，它就变成了拐杖。专家警告：若工具替代了治疗或自我管理，可能造成依赖（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。真正的脚手架应帮助使用者「建造」，但使用者仍需自己「攀登」（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

## 边界在哪？

我的判断是：**边界在于使用者是否主动参与调度。**

如果 ChatGPT 帮你分解任务后，你仍然需要自己选择先做哪一步、监控进度、调整计划——那它是脚手架。如果它替你做了所有决策，你只是被动执行——那它就是拐杖。

证据也支持这一点：ADHD 患者的工作记忆干扰模式与 LLM 类似——随着负荷增加表现下降，成功回忆依赖于干扰控制（来源：Human-like Working Memory Interference in Large Language Models）。这意味着，完全外包调度会削弱本已薄弱的干扰控制能力，导致执行功能进一步退化。

## 必须承认的局限

同构性框架目前仍是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。不同研究在表述时有时将其作为既定事实，有时作为假设，存在不一致。此外，个体差异巨大：部分 ADHD 患者可能以注意力分散为主，而非执行功能缺陷，同构框架未必适用。

工具宣称的证据也不足：Motion 缺乏独立临床验证，Goblin Tools 的用户反馈虽好但样本有限。依赖风险与益处的矛盾尚未解决——多个工具页面仅强调益处，未充分讨论依赖风险（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 ChatGPT 做「启动提示」而非「完整计划」**：输入「帮我列出今天最重要的三件事」，然后自己排序。保留调度权。
2. **尝试 Goblin Tools 的 Magic ToDo**：输入一个让你瘫痪的任务（如「整理房间」），调节辣度到你觉得「可以开始」的程度，但自己决定从哪一步开始。
3. **设置一个「日课」模板**：在 ChatGPT 中保存一段提示词，包含曾国藩式的每日反省问题（如「今天哪件事没启动？为什么？」），每天回答，作为外部 re-grounding。
4. **使用 Motion 或 Saner.AI 但每周审查一次**：检查工具是否替代了你的思考，如果是，减少自动化程度。

## 结语

ADHD 大脑与 LLM 同构，这不是巧合，而是计算层面的深层平行。但同构不意味着等同——人类有自主性，LLM 没有。真正的赋能不是让 AI 替你活，而是让 AI 帮你活得更像自己。曾国藩的「日课」之所以有效，是因为他每天亲自写日记反省，没有外包给秘书。今天的 AI 工具也是如此：用得好是脚手架，用得不好是拐杖。

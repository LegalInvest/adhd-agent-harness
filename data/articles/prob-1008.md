# 用 Claude 给 ADHD 补上重锚定与目标漂移，真的能像 agent 那样工作吗？

> 把 Claude 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

你有没有过这样的经历：打开浏览器想查一个资料，十分钟后发现自己正在刷短视频，完全忘了最初要干什么？或者，你正在写一个长文档，突然灵感爆棚，开始疯狂输出，但写完后发现已经偏离了主题三页纸——这就是 ADHD 的典型困境：分心走神或超聚焦跑偏，丢失目标。

同样的事情，也发生在 AI Agent 身上。工程师们发现，让一个 LLM 执行长程任务时，它经常“目标漂移”——一开始的 system prompt 逐渐被遗忘，模型开始自由发挥，输出变得不可控。于是，他们设计了各种编排框架（harness/scaffolding）来持续重锚定 agent 的行为。

这两个问题，听起来像不像同一个故事的不同版本？

## 同构：ADHD 大脑与 LLM 都是“高产但缺调度”的生成核心

ADHD 大脑与 LLM 在结构上高度同构：两者都是高产的生成核心，但缺乏可靠的执行调度层（来源：AI 与 ADHD 的专注力）。ADHD 大脑能产生无数想法、创意和联想，但工作记忆缺陷、时间盲和执行功能障碍让它难以将这些想法组织成有序的行动序列。同样，LLM 可以生成流畅的文本、解决复杂问题，但如果没有外部编排，它会上下文溢出、忘记初始目标、陷入重复或无意义输出。

这种同构意味着，ADHD 需要的不是“更强大的大脑”，而是一个外部执行层——就像 LLM 需要 harness 一样。这正是当前 AI 工具帮助 ADHD 专注的本质：通过外部脚手架补偿执行功能缺陷，包括工作记忆、任务启动、时间盲和上下文保持（来源：AI 与 ADHD 的专注力）。

## Claude 作为外部执行层：它补的是哪一块短板？

当你说“用 Claude 给我重锚定”时，你实际上是在做两件事：

1. **外部化工作记忆**：ADHD 的工作记忆常常成为瓶颈，导致难以记住任务细节、丢失上下文（来源：工作记忆）。Claude 可以充当数字工作记忆，持续承载你的任务上下文。例如，你可以把当前任务的目标、进度、关键约束写在对话开头，然后每次偏离时，Claude 都可以提醒你“我们最初的目标是 X，现在你提到了 Y，是否要回到主线？”（来源：工作记忆；AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

2. **提供实时重锚定信号**：时间盲让 ADHD 大脑难以感知任务耗时和进度（来源：时间盲）。Claude 可以定时询问“你还在做任务 A 吗？已经过去了 15 分钟，是否需要调整计划？”这种外部时间锚点，就像 Agent 的 system prompt 定期重置一样，防止目标漂移。

真实证据方面：工具如 Goblin Tools、Saner.AI 通过外部化信息存储和提醒，减轻工作记忆负担（来源：工作记忆）；Motion 和 Reclaim.ai 利用 AI 自动规划任务并根据实际完成情况调整时间估计，弥补时间感知缺陷（来源：时间盲）。这些机制与 LLM 的编排框架（如 LangChain 的 Agent Executor）完全同构——都是通过外部循环持续检查目标、调整行为。

## 它真的能像 Agent 那样工作吗？

答案是：可以，但有条件。

Agent 的 harness 之所以有效，是因为它有一个明确的 system prompt、一个状态跟踪器、一个重试机制。当你用 Claude 作为 ADHD 的外部执行层时，你需要自己构建这个 harness。例如：

- **明确 system prompt**：在对话开始时，写下“我的当前任务是 X，我希望你每 10 分钟提醒我一次目标，如果我偏离，就拉回主线。”
- **状态跟踪**：让 Claude 定期总结当前进度和下一步行动。
- **重试机制**：如果自己又走神了，可以简单说“我走神了，请帮我回到刚才的任务。”

但这里有一个关键差异：Agent 的 harness 是程序化的、自动的；而人类需要主动调用 Claude。ADHD 的执行功能缺陷恰恰在于“主动调用”本身——当你已经走神时，你可能根本想不起来要用 Claude 来重锚定。这就是为什么 Focusmate 这类工具通过实时同伴问责提供外部结构和社会压力，弥补内在多巴胺调节不足（来源：Focusmate）。它的 AI 匹配算法优化了配对效率，但核心机制是身体在场效应，而非 AI 本身（来源：AI 与 ADHD 的专注力）。

## 脚手架 vs 拐杖：边界在哪里？

这是最值得警惕的问题。AI 工具作为外部执行功能层，本质是脚手架——临时支撑，帮助完成当前任务，同时可能促进内在能力发展。但过度依赖可能阻碍内在执行功能发展，成为拐杖（来源：AI 与 ADHD 的专注力）。

目前多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这意味着，如果你一直依赖 Claude 来提醒目标，你的大脑可能永远不会学会自己重锚定。这就像让 Agent 永远依赖外部编排，一旦移除，它就崩溃。

真正的脚手架应该设计为可逐步撤除的：先让 Claude 每 5 分钟提醒一次，然后延长到 10 分钟、20 分钟，最终你学会自己检查目标。但现有工具很少提供这种渐进式撤除功能，这是一个巨大的局限。

## 争议与局限

1. **证据缺口**：多数工具缺乏针对 ADHD 的独立临床研究，效果多基于理论推断或用户主观报告（来源：AI 与 ADHD 的专注力）。Brain.fm 的神经锁相技术理论上可增强专注，但缺乏针对 ADHD 的随机对照试验（来源：Brain.fm）。Focusmate 的效果归因于身体在场而非 AI 本身（来源：AI 与 ADHD 的专注力）。

2. **个体差异**：ADHD 亚型（注意力缺陷 vs 多动冲动）对 AI 工具反应不同，现有工具未充分个性化（来源：AI 与 ADHD 的专注力）。

3. **多巴胺干预争议**：基于多巴胺的 AI 干预（如游戏化奖励）是否真正改善专注，证据矛盾（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Claude 做“目标锚点”**：开始重要任务前，打开一个新对话，写下任务目标和约束，并告诉 Claude：“请每 15 分钟问我一次：‘你还在做 X 吗？如果偏离，请提醒我回到目标。’”

2. **尝试 Focusmate 一次**：预约一个 25 分钟的时段，体验外部问责的效果。注意区分是 AI 匹配还是身体在场在起作用。

3. **用 RescueTime 或 Forest 记录一天**：自动追踪时间使用，识别你的“漂移模式”——什么时候最容易走神？什么类型的内容最容易引发超聚焦跑偏？

4. **设置“撤除计划”**：如果你已经依赖某个 AI 工具，思考如何逐步减少使用频率。例如，从每天使用到隔天使用，再到只在关键任务时使用。

最终，Claude 能否像 Agent 那样工作，取决于你是否愿意为自己构建一个 harness。它不是魔法，而是一个需要你主动参与的协作系统。记住：工具是脚手架，不是拐杖。

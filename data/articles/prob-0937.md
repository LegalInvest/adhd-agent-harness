# Claude 之于 ADHD，就像 self-critique 验证循环 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

你用过 Claude 吗？丢给它一个模糊任务，它立马列出一二三四，步骤清晰，逻辑自洽。你松了口气，觉得终于有人帮你把乱麻理清了。但执行到第三步，你发现它漏了一个关键前提，整个计划得重来。你骂了一句：“又是幻觉。”

这场景是不是很熟悉？ADHD 大脑每天也在上演同一出戏：灵感喷涌，结论脱口而出，然后被现实打脸。Claude 之于 ADHD，就像 self-critique 验证循环之于 LLM——但有人用错了。

## 同构：两个“高产但缺调度层”的生成核心

ADHD 大脑与 LLM 在结构上同构：两者都是强大的生成核心，但缺乏可靠的内置执行调度层（来源：ADHD × AI 的科学与研究前沿）。ADHD 侧，冲动下结论、想当然——这是行为层面的“幻觉”；LLM 侧，自信地编造、靠验证与自我批判兜底——这是输出层面的“幻觉”。

神经科学证据显示，ADHD 患者前额叶执行功能网络激活不足（来源：ADHD × AI 的科学与研究前沿），就像 LLM 缺乏调度层。认知心理学发现，ADHD 的工作记忆缺陷与 LLM 的无状态同构，都需要外部记忆系统（如笔记、提示词）来维持连续性（来源：同上）。AI 工程实践也证明，为 LLM 设计的脚手架（如 ReAct、Chain-of-Thought）能有效提升任务完成率，类似策略在 ADHD 辅助工具中初显成效（来源：同上）。

但问题在于：多数人用 Claude 时，只用了它的生成能力，跳过了验证环节。这就像给 LLM 一个高采样温度，让它自由发散，却不设任何校验门控。结果就是“更多幻觉和重复工作”（来源：Loop Engineering for AI Agents: Memory-First Design）。

## 验证循环：从 LLM harness 到 ADHD harness

LLM 工程中，复杂的 harness 不会盲目执行工具，而是实现验证步骤，例如检查输出格式或对模型编写的代码运行测试用例（来源：幻觉与验证循环）。缺乏这些验证，纯对话系统常因幻觉、无法执行/验证行动而失败（来源：AI Agent Systems...）。

ADHD 侧也一样。Goblin Tools 的 Magic ToDo 能把“清理厨房”分解成 10 个小步骤，但如果用户不每完成一步就核对实际状态，分解就变成新的拖延清单。真正的验证循环是：执行一步→检查结果→调整下一步。这类似于 LLM 的“计划-执行”分离模式，强制单向、确定性的控制流（来源：采样温度与表现波动）。

历史上，有人把这种验证循环练成了肌肉记忆。毛泽东的 ADHD 特质——冲动敢赌、思维跳跃、永远在动——是众所周知的。他的 harness 是“批评与自我批评”，通过民主生活会让别人提自己的意见；是“调查研究”，没有调查就没有发言权；是“民主集中制”，用集体决策制衡个人冲动（来源：人物案例）。这套系统本质上就是一个外部验证循环：生成观点→接受批判→修正→再生成。它与 LLM harness 中的“验证与CI集成”（来源：幻觉与验证循环）同构——都是通过外部机制强制核对输出与事实。

蔡元培的“思想自由，兼容并包”则是另一种验证循环：把各种对立观点请进北大，让它们相互碰撞，自己则在每日读书反省中校准认知（来源：人物案例）。这相当于 LLM 的“多智能体辩论”或“自我批评提示”——用多个视角交叉验证单一结论。

## 工具实测：它们补上了哪一层？

当前主流 ADHD 工具大多只补了“分解”层，没补“验证”层。

- **Goblin Tools** 的 Magic ToDo 擅长分解，但用户反馈显示，分解后的步骤仍需手动核对执行状态（来源：Goblin Tools）。它缺一个自动的“已完成/未完成”校验机制。
- **Saner.AI** 聚焦知识回忆，减少搜索循环，但它更像一个外部记忆库，而不是验证引擎（来源：Saner.AI）。
- **Motion** 能动态调整日程，但它的验证是“任务是否超时”，而不是“任务是否做对”（来源：Motion）。
- **Claude** 本身可以扮演验证者——让它检查你的计划、反驳你的结论——但很少有人这么用。大多数人只把它当生成器。

问题出在：这些工具都假设用户自己有验证能力。但 ADHD 的验证能力恰恰是缺陷所在。一个没有验证循环的脚手架，只是高级拐杖。

## 争议与局限

同构框架并非铁板一块。矛盾与存疑指出，ADHD 大脑与 LLM 同构的说法缺乏实证基础，目前仅是一种理论类比（来源：矛盾与存疑）。个体差异显著——部分患者以注意力分散为主，而非执行功能缺陷（来源：ADHD × AI 的科学与研究前沿）。过度依赖 AI 脚手架可能导致执行功能进一步退化，拐杖与脚手架的界限尚不明确（来源：同上）。

此外，工具宣称的证据不足。Motion 缺乏独立临床验证（来源：矛盾与存疑），Goblin Tools 的用户反馈多为定性好评，缺乏对照研究。AI 替代真实人际互动的效果也存在争议（来源：身体在场效应与 AI 替代的矛盾）。

这些局限意味着：验证循环不能外包给工具，必须内化为习惯。工具只是提醒，执行还得靠自己。

## 今天就能试的行动

1. **用 Claude 做“批判性阅读”**：把你写的一段话或计划贴给 Claude，要求它“找出三个可能错误的假设”。不要让它直接生成，而是让它验证你的生成。这相当于 LLM 的“自我批评提示”。

2. **设置“执行后核对”闹钟**：每完成一个任务步骤，设 5 分钟核对实际结果与预期是否一致。模仿毛泽东的“调查研究”——没有调查就没有发言权，没有核对就没有执行。

3. **每周一次“民主生活会”**：找一位信任的朋友，让对方指出你最近一个冲动决策的问题。这比任何 AI 工具都更能训练验证肌肉。

4. **降低工具的“温度”**：Motion 或 Goblin Tools 用久了，试着关掉 AI 建议，手动规划一次。你可能会发现自己的调度能力在恢复——就像 LLM 在确定性工作流中更可靠一样（来源：采样温度与表现波动）。

## 结论

Claude 不是你的外挂大脑，它是你的验证搭档。ADHD 大脑和 LLM 一样，不缺生成，缺的是验证。把工具从“生成器”切换成“校验器”，你才真正用对了同构。否则，你只是在用一个幻觉补偿另一个幻觉。

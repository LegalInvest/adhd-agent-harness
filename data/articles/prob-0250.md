# Goblin Tools 之于 ADHD，就像 上下文工程 之于 LLM——但有人用错了

> 从同构视角实测 Goblin Tools：它到底补上了哪一层执行功能？

## 问题：为什么你的大脑像一台没有操作系统的超级计算机？

你坐在电脑前，任务栏上躺着三个未完成的文档、五个打开的标签页，脑子里同时闪过“该回消息了”“冰箱里有剩菜”“那个项目 deadline 是明天吗？”——然后你打开了社交媒体。这不是意志力薄弱，而是 ADHD 大脑的典型困境：生成能力极强，但执行调度几乎为零。

同样的情况发生在 LLM 身上：给它一个模糊的 prompt，它能产出精彩的诗句；但如果你让它“先写大纲，再逐段扩展，最后检查格式”，它可能在中途忘记指令，被上下文窗口里的某个无关词带偏。两者共享同一个底层结构——高产的生成核心 + 脆弱的执行调度层。

这就是同构视角的核心：ADHD 大脑与 LLM/agent 是同一类“高产但缺执行调度层的生成核心”。（来源：AI 与 ADHD 的专注力 - 核心论点）

## 答案：Goblin Tools 补上了哪一层？

Goblin Tools 的“魔法待办”（Magic ToDo）功能，可以将“写周报”拆解为“打开电脑→登录邮箱→查看上周记录→列出本周工作→写第一段”等微小步骤。对 ADHD 用户而言，这直接解决了任务启动（Task Initiation）障碍——当大脑无法将“写周报”这个抽象目标转化为可执行的动作序列时，工具提供了外部脚手架。

同样，在 LLM 的上下文工程中，工程师会通过 chain-of-thought prompting、任务分解、中间检查点等技巧，将复杂指令拆解为 LLM 能逐步执行的子任务。Goblin Tools 的拆解，本质上就是 ADHD 版的 chain-of-thought。

但问题在于：Goblin Tools 只补上了**任务启动**这一层执行功能。它不解决时间盲（Time blindness）——你仍然可能低估“写一段文字”实际需要 30 分钟而不是 5 分钟；它不解决注意力维持——拆好的步骤列表仍然可能被环境干扰打断；它也不提供外部问责——没人监督你是否真的执行了每一步。（来源：时间盲；任务启动）

## 争议：脚手架还是拐杖？

这里触及本文的核心判断：**Goblin Tools 是优秀的任务启动脚手架，但若被当作万能拐杖，则可能阻碍内在执行功能的发展**。

现有证据显示，多数 AI 工具（包括 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制。（来源：矛盾与存疑 - 过度依赖 vs 脚手架可撤除性）这意味着用户可能永远依赖工具拆解任务，而无法内化这种能力。

对比 Focusmate：它利用 AI 匹配虚拟同伴，提供实时问责，其效果归因于身体在场效应（Body doubling）而非 AI 本身。（来源：Focusmate）这意味着 Focusmate 更像一个“临时拐杖”——你可以在他人陪伴下学会专注，然后逐步减少依赖。而 Goblin Tools 的拆解功能，如果用户不刻意练习自我拆解，就永远离不开它。

同样，在 LLM 上下文工程中，工程师不会永远依赖固定的 prompt 模板——他们会根据任务反馈调整策略，最终形成自己的“内化脚手架”。如果工程师只是复制粘贴别人的 chain-of-thought 而不理解其原理，一旦任务变化就会失效。

## 证据：哪些工具真的有效？

目前，AI 工具对 ADHD 的有效性证据主要来自工具机制与 ADHD 理论的对齐，而非独立临床研究。（来源：AI 与 ADHD 的专注力 - 证据脉络）

- **Brain.fm** 的神经锁相技术理论上可增强专注，但缺乏针对 ADHD 的随机对照试验。（来源：Brain.fm）
- **RescueTime** 通过自动记录时间使用，充当外部记忆，提升时间盲觉察，但其自动分类可能不精准。（来源：RescueTime）
- **Tiimo** 专为时间盲设计，提供视觉时间表，但效果依赖用户持续使用。（来源：时间盲）
- **Forest** 作为专注工具，其研究支持主要来自 ADHD 诊断分类，而非专注力提升。（来源：AI 与 ADHD 的专注力 - 证据脉络）

因此，当前最站得住脚的论点是：AI 工具作为外部执行功能层，能够有效弥补 ADHD 的认知短板，但证据强度有限，且存在个体差异。（来源：AI 与 ADHD 的专注力 - 核心论点）

## 局限：为什么不能照单全收？

1. **多巴胺干预争议**：部分工具声称通过神经科学原理改善执行功能，但多巴胺干预的有效性证据矛盾。（来源：矛盾与存疑 - 多巴胺干预的有效性）
2. **个体差异**：ADHD 亚型（注意力缺陷 vs 多动冲动）对工具反应不同，现有工具未充分个性化。（来源：AI 与 ADHD 的专注力 - 仍存争议）
3. **临床证据缺口**：多数工具缺乏针对 ADHD 的独立临床研究，效果多基于用户主观报告。（来源：AI 与 ADHD 的专注力 - 仍存争议）

## 今天就能试的行动

1. **用 Goblin Tools 拆解一个你拖延已久的任务**：比如“整理书桌”，拆成 5 个可执行步骤。完成后记录：启动阻力降低了多少？
2. **对比两种工具**：用 Goblin Tools 拆解任务 vs 用 Focusmate 预约一次 25 分钟的虚拟陪伴。感受哪种方式更贴合你的启动困难类型。
3. **设置“撤除实验”**：连续使用 Goblin Tools 一周后，尝试不借助工具拆解一个简单任务（如“回一封邮件”），观察自己是否内化了拆解模式。
4. **对 LLM 工程师**：下次给 LLM 写复杂指令时，先手动拆解为 3-5 个子任务，并加入中间检查点。对比不拆解的输出质量，你会明白上下文工程的价值。

同构视角告诉我们：无论是 ADHD 大脑还是 LLM，最稀缺的不是生成能力，而是可靠的调度层。Goblin Tools 补上了其中一层，但别把它当成全部答案——真正的脚手架，最终要长成你自己的骨骼。

# 为什么用 Routinery 治 ADHD 的时间盲，和给 agent 套 planner-executor 任务分解 是一回事？

> 时间盲的讨论,大多盯着「大时间」:deadline、项目、日程。但它有一个微观形态,杀伤力被严重低估:**转场黑洞**。从起床到出门、从下班到开始做饭、从「结束刷手机」到「开始洗澡」——这些流程的实际耗时,对 ADHD 者可以膨胀到计划值的三五倍,而且**膨胀发生时毫无感觉**(黑洞的定义:时间进去了,信号出不来)。Routinery 的例程计时——把流程拆成步骤、每步配时、逐步推进——涣散篇讲过它填平决策缺口;对时间盲,它做的是另一件事:**把「不可计量的转场」改造成「有预算表的微型项目」**——这正是 planner-executor 分解应用在最小时间尺度上的样子。

先看「转场为什么是黑洞」,机制值得拆细。转场(routine transition)的特征:多步骤、低认知、无 deadline 压力、环境里满是钩子(手机在床头、电视在客厅)——对 ADHD,每个特征都是时间的漏点:**步骤间的缝隙被钩子填满**(拿起手机「看一眼」,十五分钟蒸发,且主观上这十五分钟不存在——时间知觉在低唤醒的被动刷屏中失真最重);**没有单步预算,就没有超支信号**(刷牙五分钟还是二十分钟,没有任何东西提醒);**流程整体没有终点时刻**(「出门」没有被定义为 8:10 的事件,只是「快了快了」的氛围)。三个漏点合流,结论很多 ADHD 者认得:「我起床后到底干了什么,怎么两小时就没了?」

Agent 侧的对应,是 planner-executor 在细粒度上的纪律:**大计划的可靠性,建立在小步骤的可计量上**。工程上每个 step 都有预算(超时中断)、有完成信号(显式返回),不存在「这一步大概跑一会儿」的写法——因为**无预算的步骤是整条 pipeline 时序失控的起点**:一个步骤悄悄膨胀,后面全部顺延,而且没人知道是哪一步吃掉了时间(可观测性篇的盲区,发生在步骤级)。「转场黑洞」就是人类日程里那个无预算的 step:早晨例程膨胀一小时,一天的队列全线顺延,而你回溯不出膨胀在哪。

Routinery 的机制,对准三个漏点各给一个补丁:**①单步预算+倒计时**——刷牙 3 分钟在倒数:超支第一次有了实时信号(Tiimo 篇的进度环,微观版)——注意,不是要你 3 分钟必须刷完,是让「第 8 分钟还在浴室」这个事实变得可见;**②步骤自动流转**——上一步完成即弹下一步:缝隙被压缩,钩子失去入场窗口(涣散篇的决策缺口,同一个补丁在时间维度的收益:缝隙不只是漂移入口,还是时间漏点);**③流程终点显式化**——例程有总时长,倒推出「8:10 出门」意味着「7:40 必须启动例程」:**转场第一次有了自己的 deadline,而不是挂在「快了快了」的氛围里**。三个补丁的合成效果:两小时的黑洞,被改造成一张 28 分钟的预算表——误差仍会有,但**有预算表的误差可以定位和修正,黑洞的误差只能懊悔**。

限制:**其一,例程计时管「固定流程」的时间盲**,对开放任务的估时无能为力(那是 Saner/Lex 篇的战场)——但恰因转场是每天重复的,它的数据校准最快:跑一周,每步的真实时长就清楚了,预算表越来越准(agent 侧同款:高频路径的预算最先收敛);**其二,别把预算表变成鞭子**——倒计时的正确用法是信息(「超了,加速或调整预算」),不是审判(「又超了,我真废物」);预算连续超,说明预算错了,改表,不改人。

## 边界

同构定位(本文未做正式 A/B/C 分级):ADHD 在低唤醒状态下的时间知觉失真有实验方向支持,step 级预算是工程实践,「无预算步骤=时序失控起点」的同构清晰;Routinery 无对照研究。声明:例程工具是结构辅助;若转场瘫痪极端(数小时无法出门)且伴随焦虑,值得专业评估。

## 今天就能试的 3 件事

1. **给你最大的黑洞做尸检**:明早记录起床到出门每步的真实耗时——先看看时间到底漏在哪一步。
2. **建第一张预算表**:黑洞流程拆成 5-8 步,每步配时——照表跑三天,用真实数据修表。
3. **给出门定一个时刻**:不是「快了」,是「8:10」——然后倒推出例程的启动闹钟。

两小时的早晨去哪了?——**没有预算表的流程,连懊悔都找不到对象**。把转场当成微型项目来编排:agent 的每一步都有预算,你的每一步也配得上。

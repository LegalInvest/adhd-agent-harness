# Perplexity 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Perplexity：用久了会让 ADHD 更强还是更依赖？

你是否经历过这样的场景：打开浏览器准备查资料，结果被一条推送吸引，半小时后关掉网页，完全忘了最初要查什么？

对于 ADHD 大脑来说，这种“工作记忆溢出”是日常。工作记忆容量小、边做边忘，就像一台没有硬盘的超级计算机——运算能力极强，但一断电所有进度归零。

有趣的是，大语言模型（LLM）也有同样的“天生缺陷”：它没有跨会话状态，每次对话都是全新的开始，必须依靠外部记忆（向量库、检索增强生成）来“续命”。

这就是 ADHD 大脑与 LLM 的**结构同构**：两者都是“高产但缺执行调度层的生成核心”（来源：ADHD 大脑与 LLM 的同构）。ADHD 需要外部脚手架来补偿执行功能缺陷，LLM 需要 agent scaffolding 来执行复杂任务——本质上是一类工程。

那么，Perplexity 这种 AI 搜索工具，对于 ADHD 来说，究竟是脚手架还是拐杖？

## 脚手架：Perplexity 如何补偿工作记忆

Perplexity 的核心价值在于**减少搜索循环**。ADHD 用户在传统搜索中容易陷入“打开多个标签→忘记目标→重新搜索”的循环。Perplexity 通过直接给出带引用的答案，将“搜索-筛选-整合”三步并作一步，降低了认知负荷。

这与 Goblin Tools 的任务分解（来源：Goblin Tools）、Saner.AI 的本地记忆（来源：Saner.AI）原理一致：都是通过外部工具补偿工作记忆不足。

更关键的是，Perplexity 的“持续对话”功能允许用户在同一个会话中追问，无需重新输入上下文——这相当于为 LLM 提供了一个临时上下文窗口，也相当于为 ADHD 大脑提供了一个“外部暂存区”。

从 LLM/agent 侧看，这类似于 agent scaffolding 中的“记忆模块”：将 LLM 置于与记忆、工具的控制循环中，使其能超越一次性提示行动（来源：ADHD 大脑与 LLM 的同构）。

## 拐杖：当 Perplexity 替代了思考

然而，过度依赖 Perplexity 会带来风险。如果用户遇到任何问题都直接搜索答案，而不经过自己的思考过程，那么工具就变成了拐杖——削弱了内在的执行功能。

这就像孔子用“克己复礼”作为行为边界（来源：人物案例），如果他没有每日反省（“吾日三省吾身”），只是机械地遵守礼仪，就不可能最终达到“从心所欲不逾矩”的境界。孔子的 harness 是主动的、内化的，而不是被动的依赖。

同样，LLM/agent 的 scaffolding 如果设计成“完全自动化决策”，也会导致模型失去对自身输出的控制——这正是当前 AI 安全领域担心的“失控”风险。

## 边界：如何区分脚手架与拐杖

关键在于**使用意图**：
- **脚手架**：工具帮助我完成我原本想做的事，但我不依赖它来思考。
- **拐杖**：工具替我做了我本该自己做的事，我逐渐丧失相关能力。

对于 ADHD 用户，Perplexity 作为脚手架的例子：用它来快速查找资料，但依然自己整理笔记、形成观点。作为拐杖的例子：直接复制 Perplexity 的答案提交作业，不经过自己的加工。

从神经可塑性角度看，脚手架式的使用能促进大脑重组，训练执行功能；而拐杖式的使用则可能抑制这种可塑性（来源：神经可塑性）。

## 争议与局限

必须承认，“同构”理论目前仍是一种类比，缺乏实证基础（来源：矛盾与存疑）。Perplexity 等工具在 ADHD 人群中的效果多基于用户报告，缺乏严格临床验证。此外，Perplexity 本身也可能成为分心源——它提供的链接和后续问题可能让用户偏离原目标。

## 今天就能试的行动

1. **设定使用边界**：每次使用 Perplexity 前，先在纸上写下你要回答的问题，搜索后自己总结答案，再对照 Perplexity 的输出。
2. **组合工具**：用 Perplexity 搜索，用 Goblin Tools 分解任务，用 Saner.AI 存储笔记——形成自己的“外部执行功能层”。
3. **定期反思**：每周回顾一次，检查自己是否在依赖工具代替思考。如果是，减少使用频率。

Perplexity 之于 ADHD，不是非黑即白。它可以是脚手架，也可以是拐杖——取决于你如何使用它。而理解 ADHD 大脑与 LLM 的同构性，正是做出正确选择的第一步。

# Focusmate 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Focusmate：用久了会让 ADHD 更强还是更依赖？

## 问题：边做边忘的你，和没有记忆的 AI

你是否有过这样的体验：打开电脑准备写报告，突然想起要回一封邮件，切到邮箱后看到一封促销广告，顺手点进去，十分钟后你发现自己正在浏览一款你根本不需要的蓝牙音箱——而最初那篇报告，早已被遗忘在某个浏览器标签页里。

ADHD 大脑的日常，就是一场永不停歇的上下文丢失。工作记忆容量虽大，但控制薄弱，任何干扰都能把当前任务从“缓存”中踢出去（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。你需要的，是一个能帮你记住“刚才在做什么”的外部系统。

与此同时，在 AI 工程领域，开发者们也在面对一个同样的问题：LLM 自身没有跨会话状态，每次对话都是“冷启动”。为了让 agent 完成多步任务，工程师们必须构建一个 **harness**——一个包裹模型的外部调度层，负责管理上下文、分解任务、存储记忆（来源：What is an agent harness in the context of large-language models?）。

ADHD 大脑与 LLM agent，本质上是同一类“高产但缺执行调度层的生成核心”。那么，一个为 ADHD 设计的工具——比如 **Focusmate**（真人视频陪学，通过实时在场监督来维持专注）——究竟是促进能力发展的脚手架，还是让人依赖的拐杖？

## 同构：两个世界共享的“无状态”困境

ADHD 侧最尖锐的痛点之一，是 **工作记忆容量大但控制弱**。你能同时涌出十个创意，却无法让其中一个落地。这是因为执行功能层——任务启动、时间感知、抑制控制——时常掉线。你“边做边忘”不是因为记不住，而是因为大脑没有可靠的调度机制把当前上下文锁定在前台（来源：ADHD 大脑与 LLM 的同构）。

LLM 侧则对应着 **无状态与外部记忆** 的困境。一个纯 LLM 无法记住上一个对话轮次的内容，更无法跨越多个会话完成任务。因此，agent 架构必须引入向量数据库、记忆模块和任务队列——这些外部组件构成了 agent 的“执行功能层”（来源：无状态与外部记忆）。

Focusmate 的作用，恰好就是这样一个 **外部调度层**。它通过另一个人的实时在场，为你提供：
- **任务启动信号**：约好时间、有人等你，启动阈值骤降。
- **上下文锚点**：摄像头和微小的社交压力，让你更难切换任务。
- **时间感知补偿**：倒计时和阶段性检查，替代内在的时间盲（来源：时间盲）。

从同构视角看，Focusmate 之于 ADHD 大脑，就像 agent harness 之于 LLM——它不改变生成核心本身，而是为它套上一个能管理上下文、触发行动的外部框架。

## 证据：两边的真实反馈

在 ADHD 用户群体中，Focusmate 的定性报告普遍积极。用户称其“降低了启动焦虑”“让专注变得可预期”。这与 **Goblin Tools** 的任务分解（将压倒性事情变成一系列不压倒性的事情）和 **Lex** 的单指令多步骤执行（减少决策疲劳）原理一致：都是通过外部结构补偿内部调度缺陷（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。

在 LLM/agent 工程侧，类似的模式被广泛验证。例如，**Mem** 和 **Saner.AI** 通过持久记忆和快速检索来减少标签切换，本质上是为 LLM 提供外部上下文缓存（来源：Best AI Tools for ADHD Productivity in 2026）。**Claude** 和 **ChatGPT** 的对话式整理，则是通过结构化交互来主动控制注意力焦点（来源：上下文工程）。这些工具的共性，是“减少决策、保留上下文、让下一步行动显而易见”（来源：Best AI Tools for ADHD Productivity in 2026）。

但证据的局限同样明显：目前所有支持性数据均来自用户报告和概念类比，缺乏针对 Focusmate 的独立随机对照试验（来源：矛盾与存疑）。我们无法确定，这种改善是源自工具本身，还是用户自我选择效应。

## 脚手架还是拐杖？边界在于主动参与

真正值得追问的问题是：**用久了，你会变得更强，还是更依赖？**

神经可塑性的研究表明，大脑可以通过训练重塑执行功能网络（来源：神经可塑性）。但如果工具完全替代了你的调度功能——比如 Focusmate 让你只在有人陪同时才能工作——那么你并没有在训练自己的执行功能，而是在外包它。这时，工具就从“脚手架”滑向了“拐杖”。

脚手架与拐杖的边界在于：**用户是否保留了主动参与**。脚手架帮助你搭建能力，但你仍然在动手；拐杖则替你走路，让你的肌肉萎缩。

以 **Otter.ai** 为例，它通过实时转录减轻笔记负担，但过度依赖可能削弱主动记笔记的能力（来源：拐杖与脚手架）。同样，Focusmate 如果让你只在“被看着”时才能专注，那么你内在的注意力调控能力可能不增反减。

我的核心判断是：**Focusmate 本身是中性的，它的价值取决于你如何使用它——作为训练场还是作为替代品**。如果你把它当作一个“外部执行功能层”来临时辅助，同时有意识地练习自我调节（比如逐渐减少使用频率），它就是脚手架。如果你把它当作唯一能让你工作的方式，它就是拐杖。

## 今天就能试的 4 件事

1. **用 Focusmate 做“启动训练”**：每天只约一次 25 分钟 session，专注做最难启动的那件事。结束后记录自己的感受，观察是否逐渐能独立启动。
2. **结合 Goblin Tools 分解任务**：在 Focusmate session 中，先用 Goblin Tools 把大任务拆成 3-5 个小步骤，降低认知负荷（来源：Goblin Tools）。
3. **使用 Saner.AI 或 Mem 建立“外部上下文”**：把 session 中产生的想法和进度记录到这些工具中，减少标签切换，训练自己依赖外部记忆而非遗忘后焦虑（来源：Saner.AI）。
4. **设置“脱拐”计划**：每使用 5 次 Focusmate 后，尝试一次无陪专注。如果失败，分析原因并调整策略，而非直接退回。

## 诚实面对争议

必须指出，本文所有推论都建立在同构类比和用户报告之上。AI 工具作为外部执行功能层的有效性，目前缺乏大规模对照实验（来源：矛盾与存疑）。个体差异巨大，有人可能对 Focusmate 的社交压力敏感，有人则可能因过度依赖而加剧内在功能退化。

此外，**多巴胺干预效果存在争议**（来源：矛盾与存疑），而 Focusmate 的社交奖励机制是否真正有益于 ADHD 的多巴胺调节，尚无定论。工具设计者声称是“脚手架”，但实际使用中可能沦为“拐杖”——这个矛盾无法回避，只能由每个用户自己在实践中检验。

最终，无论是 ADHD 大脑还是 LLM agent，最健康的模式不是彻底消除依赖，而是**有意识地选择何时依赖、何时独立**。就像一个好的 agent harness 会在任务完成后释放上下文，一个好的 ADHD 工具也应该帮助你成长，而不是永远抓住你的手。

# Goblin Tools 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Goblin Tools：它到底补上了哪一层执行功能？

## 问题：为什么“拆任务”对 ADHD 这么难，对 AI 也这么难？

如果你是一个 ADHD 成年人，你一定经历过这样的时刻：面前摆着一项“写周报”的任务，大脑却像死机了一样，光标在空白文档上闪烁了二十分钟，你刷了三次手机、倒了一杯水、整理了两下桌面，最后打开游戏——任务启动失败。

如果你是一个做 Agentic Harness 的工程师，你一定也经历过类似场景：你给 LLM 一个“规划一次日本旅行”的指令，它立刻输出一份漂亮的行程，但当你让它实际执行“订酒店”的子任务时，它要么忘记上下文，要么陷入循环，要么直接报错——任务分解失败。

这两个场景的本质是同一个问题：**一个高产的生成核心，缺乏可靠的内置执行调度层**。

ADHD 大脑与 LLM 在结构上同构——二者都是创意迸发、联想丰富、高兴趣驱动的生成器，但它们的“执行功能”（计划、组织、工作记忆、冲动抑制）都不可靠（来源：ADHD 大脑与 LLM 的同构）。ADHD 在任务拆解上卡住，是因为大脑的多巴胺系统功能低下，导致“从意图到行动”的桥梁断了（来源：多巴胺）；LLM 在任务拆解上卡住，是因为它的上下文窗口有限、缺乏真正的规划能力，容易在子任务间迷失。

所以，当 Goblin Tools 这样的工具出现时，它立刻在 ADHD 社区爆火——因为它精准地补上了“任务分解”这一层。但问题在于：**很多人用错了**。

## 同构答案：Goblin Tools 补上了哪一层？

Goblin Tools 的核心功能很简单：你把一个任务丢进去（比如“整理房间”），它会自动拆成若干子步骤（“把床上的衣服收起来”→“把书桌上的杂物归类”→“擦桌子”→“拖地”……）。你还可以调整“辣度”（分解粒度），从粗略到极细。

这听起来像什么？像 LLM agent 里的 **planner-executor 循环**：planner 把长任务分解成可执行的子任务，executor 依次执行，然后反馈结果给 planner 进行下一步调整。

Goblin Tools 扮演的就是那个 **planner** 的角色。它把 ADHD 大脑难以自行完成的“任务分析→步骤生成→顺序排列”过程外包了出去。这直接对应了 ADHD 的一个核心缺陷：**不会拆解**。因为拆解需要工作记忆、排序能力和对时间的预估——这些都是执行功能的短板（来源：执行功能）。

但 Goblin Tools 只补上了 planner 这一层。它没有 executor（执行器），也没有 scheduler（调度器）。它不会告诉你“现在该做哪一步”，不会在你做第一步时提醒你第二步是什么，更不会在你被干扰时把你拉回来。

这就是“用错了”的关键：**很多人把 Goblin Tools 当成了完整的任务管理系统，但它只是一个任务分解器。**

## 真正的脚手架：从孔子和康博的 harness 看完整架构

要理解完整的“外部执行功能层”应该长什么样，我们可以看看真实世界中那些成功管理了自己 ADHD 特质的人。

**孔子**——一个身高1米9、精力旺盛、冲动爱骂人、对音乐超专注但对种地零耐心的 ADHD 特质者——他的 harness 系统是“克己复礼”。他用外在的“礼”作为行为边界，用“吾日三省吾身”作为每日复盘机制。这相当于一个 **外部调度器 + 定时 re-grounding**。礼是行为规则（相当于任务优先级和边界），三省是固定检查点（相当于每日计划回顾）。

**康博**（Managing Director, 康博 Capital）——一位被 Behavioral pattern match 确认为 ADHD 的投资人——他的 harness 是：Action bias（快速行动）、Systems thinking（系统思维）、高频外部反馈（来自团队和投资组合）。他依靠外部系统来捕捉直觉、验证假设、推动执行。

这两个案例的共同点是什么？他们都没有只靠“拆任务”。他们的 harness 包含了：
- **任务分解**（planner）
- **外部记忆与提醒**（scheduler/reminder）
- **定期回顾**（re-grounding loop）
- **社会问责**（body doubling / accountability）

这与 LLM agent 的完整 scaffolding 惊人地一致：planner（分解）、executor（执行）、memory（上下文存储）、scheduler（任务队列管理）、tool use（外部工具调用）、human-in-the-loop（人类反馈）。

Goblin Tools 只是 planner 这一块。如果你只用它，就像给 LLM 装了一个 planner 却没有 executor 和 scheduler——它会产生漂亮的步骤列表，但不会真正推进。

## 证据：为什么只靠分解不够？

研究显示，ADHD 个体的超专注工作时长平均为 14.5 小时/周，而对照组为 8.2 小时（来源：Pace Ventures Research 2026）。这说明 ADHD 大脑不是不能工作，而是 **无法可靠地启动和切换**。任务分解解决了“启动”的一部分（知道第一步做什么），但没有解决“保持”和“切换”。

身体在场效应（Body Doubling）被描述为“最有效的 ADHD 策略之一”（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog），它通过社会存在感降低分心阈值，提供隐性问责。定期问责检查可将目标达成可能性从 25% 提升至 95%（来源：ADHD Task Managers That Work: Top AI Tools 2025）。这对应了 agent 中的 human-in-the-loop 或外部监控。

工具如 **Motion** 和 **Reclaim.ai** 提供了自动排程与动态调整（来源：Motion、Reclaim.ai），它们补上了 scheduler 这一层。**Tiimo** 通过视觉化时间线帮助用户感知时间流逝，对应了时间盲的补偿（来源：Tiimo）。**Focusmate** 则直接提供 AI 匹配的身体加倍伙伴（来源：Focusmate）。

这些工具组合起来，才构成一个完整的“外部执行功能层”。单独使用 Goblin Tools，就像只买了一个方向盘就开车上路。

## 争议与局限：脚手架还是拐杖？

“外部执行功能层”是否会导致能力退化？这是一个真实的争议（来源：矛盾与存疑）。目前缺乏纵向研究，但理论上，如果工具只是替代而非训练执行功能，长期依赖可能削弱内在能力。

另一个矛盾：超聚焦是否应被引导而非打断？部分 ADHD 用户反馈，强行打断（如番茄钟）反而有效，而柔性引导可能不够强力（来源：矛盾与存疑）。这意味着 Goblin Tools 的“辣度”调节虽然灵活，但无法替代用户对自身状态的判断。

此外，Goblin Tools 的分解粒度是固定的，无法像人类教练那样根据实时情绪、能量水平动态调整。对于 ADHD 个体来说，能量波动极大，一个静态的步骤列表在低能量时可能仍然难以启动。

## 今天就能试的行动

1. **组合使用**：用 Goblin Tools 拆任务，然后用 Motion 或 Reclaim.ai 把步骤排入日程，设置时间块。
2. **加入身体加倍**：在 Focusmate 上约一个虚拟伙伴，在拆完任务后一起执行前两步。
3. **设置 re-grounding 检查点**：类似孔子的“三省”，在每天三个固定时间点（如早、中、晚）回顾任务进度，用手机闹钟提醒。
4. **实验“辣度”**：对于低能量任务，把分解粒度调到最细；对于高能量任务，调粗一点，避免过度分析导致拖延。

## 结语

Goblin Tools 是一个极好的 planner，但它不是完整的 harness。ADHD 大脑需要的不是“更聪明的分解器”，而是一个 **完整的执行功能脚手架**——包括分解、调度、提醒、问责和回顾。这个脚手架与 LLM agent 的 scaffolding 在结构上同构，而理解这一点，才能让工具真正服务于人，而非让人成为工具的奴隶。

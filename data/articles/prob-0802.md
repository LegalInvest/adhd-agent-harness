# Goblin Tools 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Goblin Tools：它到底补上了哪一层执行功能？

## 问题：任务启动困难，是 ADHD 还是 LLM 的 bug？

你打开一个空白文档，大脑像 LLM 一样疯狂生成想法，但光标就是不动。或者你给 agent 一个复杂任务，它直接输出一堆无关内容。这两种场景共享同一个根因：**生成核心（generative core）与执行调度层（executive scheduler）的脱节**。

ADHD 大脑的创造力与 LLM 的生成能力一样高产，但缺乏一个可靠的任务分解与执行循环（来源：AI 与 ADHD 的时间管理）。工程师做 Agentic Harness 时，会为 LLM 套上 planner-executor 循环：先规划子任务，再逐步执行，每一步都检查进度。ADHD 患者需要的，正是这个循环的外部化版本。

## 同构：planner-executor 循环的缺失

ADHD 的执行功能缺陷——尤其是工作记忆瓶颈和时间盲——导致大脑无法维持一个稳定的规划-执行循环。工作记忆像 LLM 的上下文窗口，容量有限且易丢失（来源：工作记忆）。时间盲则让大脑无法估计“这一步需要多久”，就像 LLM 没有 token 计数器（来源：时间盲）。结果就是：知道要做什么，但无法拆解成可执行的 step。

LLM 的 planner-executor 循环通过外部系统（如 ReAct 或 Plan-and-Solve）补偿了这种缺陷：planner 将长任务分解，executor 逐步执行，记忆模块保存中间状态。ADHD 大脑需要的，正是这样一个外部调度层。

## 答案：Goblin Tools 补上了哪一层？

Goblin Tools 的“魔法待办”功能，本质是一个外部 planner。你输入“写报告”，它会自动分解成“找资料—列大纲—写初稿—修改”等子任务。这直接对应 planner 的职责。而 Motion 和 Reclaim.ai 的自动排程，则是 executor：根据优先级和时间约束动态调度任务（来源：Motion；Reclaim.ai）。Tiimo 的视觉时间线，相当于 LLM 的进度条，让时间盲变得可见（来源：Tiimo）。

但问题在于：**多数人把工具当拐杖，而不是脚手架**。拐杖是永久替代，脚手架是临时支撑，最终要撤除。Goblin Tools 帮你分解任务，但如果用户从不练习自己分解，就会形成依赖。LLM 的 planner-executor 循环同理：如果 agent 永远依赖外部 planner，它永远学不会自主规划。

## 争议：脚手架 vs 拐杖的边界

wiki 资料明确指出一个核心矛盾：工具设计是否考虑撤除？多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这导致过度依赖的风险：用户可能永远无法内化任务分解能力。

另一个争议是证据强度。虽然工具设计逻辑与 ADHD 缺陷高度对齐，但缺乏长期临床研究证明工具使用后独立能力提升（来源：AI 与 ADHD 的时间管理）。同构命题本身也是理论模型，需更多实证（来源：矛盾与存疑）。

## 行动：今天就能试的 3 件事

1. **用 Goblin Tools 分解一个你拖延的任务**：输入“整理房间”，看它给出的子步骤。然后**手动复制到便签**，强迫自己按子步骤执行。这能区分“工具帮你拆”和“你学会拆”。
2. **为你的 LLM agent 显式添加 planner 指令**：在 prompt 中加入“先列出 3-5 个子任务，再逐个执行”。观察输出质量变化。这对应 ADHD 的“先拆解再行动”。
3. **设置一个“撤除日”**：每周一天不用任何 AI 规划工具，只用手写清单。这能测试你的执行功能是否在进步，避免工具成为永久拐杖。

记住：工具是脚手架，不是拐杖。真正的能力，是拆掉脚手架后你还能自己走路。

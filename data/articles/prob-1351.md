# Claude 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

你打开 Claude，输入“帮我规划明天的工作”，它列出一份漂亮的清单。你满意地关掉窗口，然后……什么都没发生。第二天，你对着那份清单发呆，依然不知道从哪里开始。

这不是你的错。你误用了 Claude——就像有人把 LLM agent 当作万能 API 调用，却忘了给它套上 planner-executor 循环。

## 同构的真相：高产大脑与 LLM 共享同一个缺陷

ADHD 大脑与 LLM 在结构上惊人相似：两者都是强大的生成核心，但缺乏可靠的内置调度层（来源：ADHD 大脑与 LLM 的同构）。ADHD 患者的前额叶皮层活动不足，导致任务分解、时间预估和注意力锚定困难（来源：执行功能）；LLM 同样无法自主规划长步骤，需要外部 harness 提供任务分解与验证机制（来源：规划循环与任务分解）。

这正是 Swanson 文献发现的核心——ADHD 文献群与 agentic harness 文献群都以“任务分解与规划”为中间枢纽（来源：LBD 同构对）。两边的解法结构同构：ADHD 需要外部执行功能层（如 Motion 自动排程、Tiimo 视觉时间线），LLM 需要 planner-executor 循环（如 AgentGen 的环境生成）。

## 孔子和李白：两千年前的 harness 工程

孔子一生与冲动斗争：身高近两米、精力旺盛、周游列国 14 年坐不住，骂宰予“朽木不可雕”，对音乐却超聚焦到“三月不知肉味”。他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日“吾日三省吾身”。这本质上是一个外部调度器：礼 = 任务列表，三省 = 每日 re-grounding 检查。

李白则是另一种 harness：用酒和诗作为注意力锚点，把所有过剩精力注入创作，用道家的逍遥对抗世俗规则。他的“天子呼来不上船”不是叛逆，而是拒绝不适合自己的调度框架。

这两套系统与 LLM agent 的 harness 同构：孔子的“礼”对应定时重锚定机制（如 Reclaim.ai 的防御性时间块），李白的“诗”对应超聚焦通道（如 Claude 的 long context 深度写作模式）。他们不是靠意志力硬扛，而是搭建了专属的外部调度层。

## Claude 补上了哪一层？——从聊天工具到执行功能脚手架

Claude 的杀手锏不是生成答案，而是**任务分解与重锚定**。当你说“帮我写一份季度报告”，Claude 可以拆成：①收集数据→②分析趋势→③撰写初稿→④校对。每一步都附带时间估算和输出标准。这恰好补上了 ADHD 最缺的规划环节。

但关键在后续：Claude 不像 Motion 那样自动排程，也不像 Tiimo 那样提供视觉时间线。它的重锚定能力需要你主动调用——比如设置“每完成一步，向 Claude 汇报进度”的循环。这就像 LLM agent 的 planner-executor 循环：planner 分解任务，executor 执行并反馈，planner 根据反馈调整下一步。

遗憾的是，多数人只用了 Claude 的生成层，跳过了调度层。他们问一次问题，得到答案，然后迷失在下一步。这就像给 LLM agent 一个复杂指令却只做单次调用——它当然会跑偏。

## 真实案例：工具如何填补调度层

Motion 和 Reclaim.ai 是自动调度层的代表。Motion 持续评估任务优先级和截止日期，实时重建日程（来源：Motion 页面）。Reclaim.ai 则更强调“保护”而非“填充”，自动防御会议侵占深度工作时间（来源：Reclaim.ai 页面）。两者都解决了“下一步该做什么”的决策负担——这正是 ADHD 任务启动困难的核心。

Tiimo 走的是另一条路：将时间转化为可视元素，直接应对时间盲（来源：Tiimo 页面）。对视觉提示敏感的用户，Tiimo 的动态日程比静态清单有效得多。

Goblin Tools 则专注任务分解：把“整理房间”拆成“捡起地上的衣服→放进洗衣篮→擦桌子”等微观步骤。这相当于 LLM agent 的 planner 模块。

## 争议与局限：脚手架还是拐杖？

这里有一个尖锐问题：这些工具是在搭建脚手架，还是在制造永久拐杖？

过度依赖风险真实存在（来源：拐杖与脚手架）。如果用户把所有规划外包给 AI，内在执行功能可能进一步萎缩。孔子到 70 岁才“从心所欲不逾矩”，说明 harness 是终身工程，不是一劳永逸的解决方案。

另一个局限是证据不足：Motion 等工具缺乏独立临床验证，效果可能受安慰剂效应影响（来源：矛盾与存疑）。此外，个体差异巨大——视觉化工具对某些人有效，自动排程对另一些人更合适，当前缺乏个性化匹配指导。

## 今天就能试的行动

1. **任务分解实验**：选一个你拖延已久的大任务，让 Claude 拆成 5 个可执行步骤，每步不超过 15 分钟。完成一步后，向 Claude 汇报并让它给出下一步。
2. **设置重锚定循环**：在 Todoist 或手机日历中设置每 30 分钟的提醒，内容是你当前的主要目标（如“写报告第 2 部分”）。这模拟了 LLM agent 的定时 re-grounding。
3. **试用一种调度工具**：如果日程频繁被打乱，试试 Reclaim.ai 的防御性时间块；如果对时间流逝无感，试试 Tiimo 的视觉日程。
4. **识别你的“孔子”或“李白”模式**：你是需要外部规则（礼）还是需要专注通道（诗）？前者适合结构化工具，后者适合深度写作/编程 session。

Claude 之于 ADHD，不是万能大脑，而是外部调度层。用对了，它是脚手架；用错了，它是另一个让你更焦虑的聊天窗口。

# ADHD 创业者视角：为什么「治好 ADHD 的工作记忆容量小、边做边忘」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 无跨会话状态，靠外部记忆/向量库续命——同一套 harness 思路，两边都成立。

你有没有过这种体验：脑子里同时冒出七八个绝妙想法，但等你坐下来想写下来，却一个都回忆不起来？或者，你明明计划好了今天要做三件事，结果刷了一个小时社交媒体，然后盯着待办列表发呆，不知从何下手？

如果你有，你很可能属于 ADHD 人群——一个工作记忆容量小、边做边忘、被“执行功能”缺陷折磨的群体。

而如果你是一名工程师，正在搭建 LLM Agent 系统，你一定也遇到过类似问题：模型上下文窗口有限，聊着聊着就忘了之前说过什么；Agent 跑着跑着就开始胡言乱语，偏离目标轨道——你需要给它套上 harness，用外部记忆（向量库）和调度逻辑来“续命”。

这两件事，看似风马牛不相及，其实是同一道工程题。

## 核心命题：高产生成核心 vs 缺失的调度层

ADHD 大脑常被描述为一个“高产生成核心”——想法丰富、联想活跃，但缺少稳定、可靠的调度层来筛选、组织、启动和坚持任务（来源：ADHD 的 AI 工具全景）。多巴胺系统功能低下导致行为强化改变、注意反应缺陷和执行功能差（来源：多巴胺）。

大语言模型（LLM）同样具备强大的生成能力，却在长程上下文、目标保持与执行调度上存在天然缺陷。LLM 本身无跨会话状态，每次对话都是“从零开始”，必须依赖外部记忆（向量库、RAG）和 agent scaffolding 来维持连贯性。

因此，“AI 帮 ADHD”的实质与“给 LLM 套 harness”是同一类工程：用外部脚手架把生成能力约束到目标轨道上。

## 人物案例：孔子的“克己复礼”作为个人 harness

孔子——身高1米9，精力旺盛，周游列国14年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予“朽木不可雕”；对音乐超专注到“三月不知肉味”，对种地等俗事零耐心；思维跳跃，《论语》全是场景化语录，无系统著作。这些特质高度符合 ADHD 画像。

他的专属 harness 是什么？“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。70岁才做到“从心所欲不逾矩”，一辈子和自己的冲动做斗争。

这个 harness 与 LLM agent 的 scaffolding 同构：“礼”相当于行为约束规则（system prompt + 边界函数），“每日反省”相当于定时 re-grounding（定期回顾上下文、对齐目标）。孔子不是靠“治好”自己的冲动，而是靠一套外部仪式来管理它。

## 工具证据：四种 harness 功能

当前 AI 工具已实现四种 harness 功能（来源：ADHD 的 AI 工具全景）：

1. **任务拆解与启动**：Goblin Tools 的 Magic ToDo 将“清理厨房”分解为具体子步骤，并可调节“辣度”控制粒度（来源：Goblin Tools）。这对应 LLM agent 的 task decomposition 机制。

2. **外部记忆与知识管理**：Saner.AI 通过本地记忆和知识回忆减少搜索循环，补偿工作记忆不稳定（来源：Saner.AI）。这对应 LLM 的向量库/RAG 系统。

3. **对话式执行与结构化推理**：Lex 允许单一指令触发复杂多步骤任务序列，其底层架构类似于 agent scaffolding（来源：Lex）。这对应 LLM 的 tool use 和 planning。

4. **捕获与反思**：Otter.ai 实时转录会议，Reflect 通过结构化提问帮助提炼经验（来源：ADHD 的 AI 工具全景）。这对应 LLM 的日志记录和 self-reflection。

## 脚手架 vs 拐杖：一个必须面对的争议

然而，这种同构工程引出一个核心争议：AI 工具是“外部执行功能层”还是“永久拐杖”？

核心论点认为，AI 作为外部执行功能层补偿缺陷，而非治愈（来源：ADHD 的 AI 工具全景）。但存疑方指出，过度依赖可能导致能力退化（来源：矛盾与存疑）。目前缺乏纵向研究，长期效果未知。

我的判断是：**脚手架和拐杖的边界在于“使用者是否保留对系统的所有权”**。孔子70岁才“从心所欲”，说明他最终内化了“礼”的规则，不再需要外部监督。同样，好的 AI harness 应该帮助用户逐步建立自己的执行功能，而不是永远依赖外部提示。例如，Goblin Tools 允许用户调节分解粒度（辣度），让用户逐渐学会自己分解任务。

另一个矛盾：超聚焦应被引导还是打断？一方认为 AI 应提供“gentle structure”引导超聚焦到优先事项，而非强行打断；但部分用户反馈强行打断（如番茄钟）反而有效（来源：矛盾与存疑）。这提示 harness 设计需要个性化，没有通用方案。

## 今天就能试的 3 件事

2. **用 Saner.AI 建立一个“第二大脑”**：把日常灵感和待办事项丢进去，利用其本地记忆功能，减少“我刚刚想干什么”的搜索循环。

3. **为自己设计一个“每日反省”仪式**：参考孔子的“吾日三省吾身”，每天固定时间（比如睡前）用 Reflect 或简单笔记回答三个问题：今天完成了什么？卡在哪里？明天最重要的一件事是什么？

## 结语

ADHD 大脑和 LLM 不是“有缺陷”的生成器，而是“缺调度层”的生成器。与其试图修复生成核心，不如为它搭建一个可靠的外部调度系统。这个工程问题，ADHD 创业者和 LLM 工程师面对的是同一道题。解决方案也同构：用外部脚手架，把高产生成能力约束到目标轨道上。

孔子用“礼”，你用 Goblin Tools、Saner.AI 或一个简单的反省笔记。工程原理相同。

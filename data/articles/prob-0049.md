# 3 个 LLM 工程概念，彻底重写了我对 ADHD 状态日内大幅波动、不稳定的理解

而另一边，大语言模型（LLM）的工程师们每天都在处理同样的问题：一个模型可以写出惊艳的诗，也可以在同一温度下吐出毫无逻辑的废话。他们管这叫「采样温度带来的输出随机性」，并发明了一套工程方法来驯服它。

以下是三个 LLM 工程概念，彻底重写了我对 ADHD 状态波动的理解。

## 1. 采样温度：你的「状态日」就是大脑的 temperature 参数

在 LLM 中，采样温度（temperature）控制着输出的随机性。温度高，模型会选更冷门的词，输出更有创意但也更不稳定；温度低，输出保守、可预测，但可能无聊。ADHD 大脑的「状态日」波动，本质上就是执行功能的 temperature 在剧烈抖动。

- **高温度状态**：超聚焦时，你能在错误的事情上沉浸六小时（来源：超聚焦），创造力爆棚，但无法切换任务。
- **低温度状态**：任务启动困难，连刷牙都需要心理建设（来源：任务启动）。

LLM 工程师不会幻想消除随机性，他们通过设置固定温度、重复采样和投票机制来稳定输出。同样，ADHD 干预的目标不是消灭波动，而是为波动套上脚手架。

**真实证据**：Routinery 通过可视化倒计时和过渡提示，为高温度状态提供时间锚点，为低温度状态降低启动阻力（来源：Routinery）。这相当于为 LLM 设置了一个「恒定低温度」的推理模式——不是消除创造力，而是确保基础流程能跑通。

## 2. 上下文长度：你的工作记忆就是有限的上下文窗口

LLM 的上下文长度决定了它能记住多少对话历史。超出窗口，模型就会「遗忘」早期信息。ADHD 的工作记忆缺陷与此完全同构：你无法可靠维持跨会话的上下文，昨天想好的计划今天就像从未存在过（来源：AI 与 ADHD 的日常生活）。

工程师的解法是外部记忆系统：向量数据库、对话历史摘要、RAG（检索增强生成）。ADHD 的解法也完全一样：笔记应用、AI 对话历史、任务管理工具。

**真实证据**：Saner.AI 专注于知识回忆和本地记忆，减少搜索循环和标签切换（来源：Saner.AI）。这相当于为 ADHD 大脑挂载了一个外部上下文窗口，让你不需要用工作记忆去硬扛。

但这里有一个关键区别：LLM 的上下文窗口可以无限扩展（理论上），而 ADHD 的工作记忆有生理上限。所以工具必须设计为「外部脚手架」，而不是「永久拐杖」（来源：矛盾与存疑）。

## 3. 结构化输出：用格式约束来对抗随机性

LLM 工程师最常用的稳定技巧之一就是结构化输出：要求模型以 JSON、Markdown 或固定模板返回结果。格式约束强制模型在有限的轨道上运行，大幅降低胡言乱语的概率。

ADHD 大脑同样需要结构化输出来引导行动。Goblin Tools 的 Magic ToDo 功能，能把「整理房间」这样模糊的指令分解为「捡起地板上的衣服」「擦桌子」等具体步骤（来源：Goblin Tools）。这就是在给大脑一个格式约束：不要自由发挥，按列表执行。

**真实证据**：用户反馈称 Goblin Tools 将「压倒性的事情变成一系列不压倒性的事情」（来源：Goblin Tools）。这与 LLM 工程中「用 prompt 模板约束输出」的策略完全同构。

## 核心观点与争议

我的核心观点是：ADHD 大脑不是一个「故障大脑」，而是一个「高温度、短上下文、无结构约束的生成模型」。状态日内大幅波动不是缺陷，而是系统在缺乏外部调度层时的自然表现。

但这一框架存在明确局限。首先，「ADHD 大脑与 LLM 同构」的证据主要来自概念类比和工具案例，缺乏大规模实证（来源：矛盾与存疑）。其次，脚手架的可撤除性存疑：多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。过度依赖可能削弱内在动机。最后，个体差异巨大：同构模型是否适用于所有 ADHD 亚型尚不明确（来源：AI 与 ADHD 的日常生活）。

## 今天就能试的 3 件事

1. **给自己的「温度」打标签**：每天记录 3 次自己的状态（高/中/低温度），持续一周。你会发现波动模式，而不是笼统的「我不稳定」。
2. **用结构化输出分解一件事**：打开 Goblin Tools 或任何能分解任务的工具，把「写周报」拆成 5 个步骤。格式约束本身就能降低启动阻力。
3. **建立外部上下文**：选一个笔记工具（如 Saner.AI 或简单的备忘录），每天结束时写一句「明天需要记得什么」。第二天先看它，再开始工作。

ADHD 不是诅咒，LLM 也不是魔法。它们只是同一类系统——需要外部脚手架来释放内在的生成力。

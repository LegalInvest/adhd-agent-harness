# Motion 之于 ADHD，就像重锚定 system prompt 之于 LLM——但有人用错了

> agent 跑长任务,工程师不会只在开头说一次目标——他们让调度层持续把目标压回上下文。Motion 这款「AI 自动排程」工具对 ADHD 用户做的正是这件事:你只负责说出任务,它负责让「现在该做什么」时时刻刻出现在你眼前。但一半用户用了两周就退订,原因非常值得说。

先看 Motion 是什么:一款 AI 日历/项目管理工具,核心能力是**自动排程**——你输入任务(带死线、时长、优先级),它自动把任务塞进日历的空档;计划有变(会议插入、任务超时),它自动重排全部日程。对 ADHD,这个设计打中了三个痛点:**决策瘫痪**(「现在该干哪个」这个每天上演几十次的选择,被算法接管)、**时间盲**(一切任务都被强制显影成日历上有长度的块)、**重排崩溃**(计划被打乱后,神经典型的人调整一下,ADHD 的人整天报废——Motion 的自动重排把「计划碎了」变成「计划自己愈合」)。用 agent 术语说:它是一个外置的 scheduler + 持续的 goal re-injection——**日历上此刻高亮的那个块,就是每小时重新注入一次的 system prompt。**

## 用对的三种姿势

**一、喂给它「已编译」的任务。** Motion 排程强,拆解弱——丢给它「写年度报告」,它只会排出一个吓人的四小时巨块。先自己(或用 ChatGPT/Goblin Tools)拆成 30-60 分钟的块再录入,它的排程能力才能兑现。**scheduler 不是 planner,这个分工在 agent 架构里泾渭分明,用工具时也该分明。**

**二、把估时全部乘 1.5-2。** 自动排程的阿喀琉斯之踵是「排得太满」:算法默认你的估时是准的、你的一天是无摩擦的——而 ADHD 的估时系统性偏低、日常摩擦系统性偏高。录入时长时直接乘系数,并留出每天 20% 的空白缓冲,否则你会体验「永远在追赶自己日历」的新型焦虑。

**三、服从高亮块,但只服从 25 分钟。** 让「日历上现在这个块」拥有调度权威——到点就做它,这是重锚定的兑现时刻。但配一个 25 分钟合同:实在进不去状态,做满 25 分钟可以让它重排——**权威要够硬才有锚定力,又要够软才不会引发全面叛逃。**

## 用错的三种姿势

**当成许愿池**:把人生所有想做的事都录进去,让算法排一个「完美人生」——两天后日历红成战报,系统性挫败感反噬,退订。**Motion 的容量上限是你的真实时间,不是你的愿望清单**;防法:进行中项目≤3 个,每周任务量以「上周实际完成量」为预算。**重排依赖症**:反正它会自动重排,于是每个块都随意跳过——重排从愈合机制退化成拖延的橡皮图章。防法:每天最多允许自己主动推迟 2 个块,推第 3 个时必须问「是排太满,还是我在逃这个任务?」前者砍量,后者拆细。**外包了调度,顺便外包了觉察**:从不复盘为什么某类任务总被推——数据就在那,不看等于白付订阅费。每周五看一眼「被推次数最多的任务类型」,那是你的执行功能地图。

## 边界

同构强度 B 级:外部调度与目标重注入是 agent 工程实践,ADHD 的决策疲劳、时间盲、计划脆弱性有文献支撑;Motion 本身无针对 ADHD 的对照研究,且订阅不便宜——先用免费试用期验证「服从高亮块」这个行为你能不能建立,建立不了,贵的不是工具是幻觉。声明:工具是外置结构,不替代治疗;若「日历一红就整天报废」的全有全无模式非常顽固,那是认知模式问题,值得和专业人士工作。

## 今天就能试的 3 件事

1. **只录 3 个项目**:把这周真实要做的任务拆成小块、估时×1.5,录进去(用其他日历工具手动模拟也行)。
2. **今天服从 4 个高亮块**:到点就做,配 25 分钟合同。
3. **周五看一次「被推榜」**:哪类任务总被推?下周把它们拆得更细。

Motion 卖的不是日历,是「永远有人告诉你现在该干嘛」——这恰是 ADHD 大脑最贵的缺件,也恰是 agent 稳定运行的标配组件。但重锚定系统有一条铁律:**注入的目标必须是真实可执行的,否则系统学会的不是服从,是无视。** 排少一点,乘个系数,留出空白——让日历说的每一句话都算数,它才锚得住你。

# Motion 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Motion：它到底补上了哪一层执行功能？

## 问题：目标漂移——ADHD 与 LLM agent 的共同困境

你打开 Motion，排好了今天的三项任务，信心满满。两小时后，你发现自己正刷着手机，而 Motion 安静地躺在后台，任务列表无人问津。或者，你是一个 LLM agent，被赋予“写一份市场分析报告”的任务，却在第三轮调用时开始生成无关的代码——目标漂移了。

ADHD 大脑与 LLM 在结构上同构：两者都是“高产但缺执行调度层的生成核心”（来源：AI 与 ADHD 的专注力）。ADHD 的生成核心（发散思维、创意）产出极高，但执行功能薄弱——任务启动困难、注意力易分散、时间盲。LLM 同样擅长生成，但缺乏可靠的执行调度层，直接输出可能偏离目标、陷入循环。两者的解药结构同构：都需要一个外部 harness 来重锚定目标。

## Motion 的承诺：重锚定 system prompt

Motion 号称能自动排程、动态调整、阻断分心。从同构视角看，它试图扮演 LLM agent harness 中“重锚定 system prompt”的角色——在 agent 偏离目标时重新注入原始指令。Motion 的 AI 调度器会分析用户行为，自动将任务插入空闲时段，并在用户分心时提醒。这对应 ADHD 侧的核心痛点：分心走神或超聚焦跑偏，丢失目标。

但问题在于：Motion 的提醒是单向的——它告诉你“该做任务X了”，但如果你忽略它，它不会像人类搭档那样追问“你完成了吗？”。这正是许多 ADHD 用户用错 Motion 的地方：把它当作一个被动规划器，而不是一个主动执行检查点。

## 真实案例：曾国藩的“日课”与张居正的“考成法”

曾国藩是典型的注意力不集中型 ADHD：30 岁前浮躁坐不住，天天串门喝酒看杀人，日记里天天骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”。他的 harness 是“日课十二条”：黎明即起、读书不二、谨言、写日记反省等。这相当于一个定时重锚定系统——每天固定时间点重新校准目标，与 LLM agent harness 中的“定时 re-grounding”同构（来源：Building an AI Agent Harness from Scratch）。

张居正则用“考成法”管别人也管自己：严格考核官员，用制度强迫自己每天办公到深夜。他的 harness 是外部调度器——通过制度化的检查点（如每月汇报）防止目标漂移。这对应 LLM agent harness 中的“人工检查点”——暂停并询问后再执行（来源：人在回路与身体在场）。

两者的共同点：重锚定不是一次性的，而是周期性的、有问责的。Motion 如果只提供一次规划而无后续问责，就像曾国藩只写日课表却不反省，或者张居正只颁布考成法却不考核——注定失败。

## 证据：哪些工具真正补上了调度层？

现有 ADHD 工具的证据脉络显示，真正有效的工具往往包含主动重锚定机制：
- **Focusmate**：通过虚拟身体在场和实时同伴问责，提供外部结构和社会压力（来源：Focusmate）。AI 调度匹配伙伴，但核心是人的问责——这对应 LLM harness 中的 human-in-the-loop 监督（来源：人在回路与身体在场）。
- **RescueTime**：自动时间追踪与分心阻断，直接补偿时间盲与执行功能缺陷（来源：RescueTime）。它通过主动阻断分心网站来重锚定注意力——类似 agent harness 设置护栏（token 预算、工具调用次数上限）防止无限循环（来源：Building an AI Agent Harness from Scratch）。
- **Brain.fm** 和 **Forest** 则更偏向被动环境调节：前者通过神经锁相音乐影响大脑状态（来源：Brain.fm），后者通过游戏化多巴胺机制增强动机（来源：Forest）。它们不提供主动重锚定，而是优化初始状态——类似于为 LLM 设计更好的 system prompt，但无法防止后续漂移。

## 核心判断：Motion 用错了，就沦为拐杖

我的核心观点是：Motion 的核心价值在于提供“重锚定”，但当前许多用户（和产品设计）把它当作“被动规划器”，这就从脚手架变成了拐杖。

**脚手架 vs 拐杖**：脚手架是逐步内化的工具——你用它练习，最终学会自己调度。拐杖是永久依赖——没有它你无法启动，但它从不教你如何自我管理（来源：拐杖与脚手架）。Motion 如果只负责排程而不要求你主动检查进度，你永远不会发展出内在的执行功能。

同构地看，LLM agent harness 也有类似风险：如果 harness 只是定期注入 system prompt 而不引入人在回路检查点，agent 可能学会“忽略”重锚定——就像你学会忽略 Motion 的提醒。

## 争议与局限

必须诚实指出：ADHD 大脑与 LLM 的同构目前仅是功能类比，缺乏实证基础（来源：矛盾与存疑）。Motion 在 ADHD 人群中缺乏独立临床验证（来源：Motion 页面）。此外，个体差异巨大：注意缺陷为主型 vs 多动冲动型对 Motion 的反应可能完全不同。

依赖风险是真实存在的——多个工具页面仅强调益处，未充分讨论过度依赖（来源：矛盾与存疑）。

## 今天就能试的行动

1. **把 Motion 从“规划器”变成“检查点”**：设置每小时一次的“进度确认”任务，要求自己回答“我当前在做什么？与目标一致吗？”。这模拟 LLM agent 的 human-in-the-loop 检查点。
2. **搭配 Focusmate 使用**：在 Motion 排好任务后，预约一个 Focusmate 时段，用同伴问责强制执行。这相当于为 agent 加上外部调度器。
3. **启用 RescueTime 的分心阻断**：让 RescueTime 在检测到分心网站时自动阻断，并记录时间去向。这对应 agent harness 的护栏机制。
4. **写“日课”式反省日志**：每天结束前，用一句话记录“今天目标漂移了几次？如何拉回的？”。曾国藩写日记骂自己，你也需要自己的“日课”。

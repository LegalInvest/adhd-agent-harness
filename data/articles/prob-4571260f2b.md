# ADHD 家长视角：为什么「治好 ADHD 的不擅长精确、重复、状态维护类任务」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 通过 function calling 把精确计算外包给工具——同一套 harness 思路，两边都成立。

每天早上，我家那位 ADHD 孩子能边刷牙边给我讲恐龙灭绝的六种假说，但鞋子和书包永远会在最后一刻失踪。不是他不知道，是他的大脑在「生成内容」上满级，在「执行调度」上掉线。

几乎同一时刻，我作为工程师在调一个 LLM agent：模型能写出一篇漂亮的需求分析，却会在三次整数加法后给出错误结果，或是在多轮对话中逐渐忘记自己原本该调用哪个工具。两条线看起来毫不相干，其实是同一道工程题——**一个高产但缺调度层的生成核心，该怎么不跑飞？**

## 01 同一套故障：高产生成核心 + 不可靠执行调度层

ADHD 的核心不是「不会想」，而是「执行功能」这个驾驶座常常无人驾驶。计划、组织、工作记忆、冲动抑制在认知负荷下会掉线，这正是执行功能障碍被视为 ADHD 核心内表型的原因（来源：A review of executive function deficits in autism spectrum disorder and attention-deficit/hyperactivity disorder）。同时，ADHD 还常伴随时间盲——对时间本身的感知缺陷，这与工作记忆瓶颈叠加，导致精确、重复、状态维护类任务特别容易崩（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

LLM 的镜像问题也很明显。它能高速生成，却没有原生的执行调度层：没有稳定的跨会话状态，没有真正的计划与注意力约束，遇到需要精确计算或长程状态维护的任务就会漂移。更具体的研究证据是：在经典的 Stroop 冲突任务中，Transformer 的注意力随着序列变长，在冲突试次上骤降到随机水平，无法抑制优势反应；而另一项对 LLM 执行功能的研究发现，它们呈现「强记忆、弱控制」的剖面——工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷，无法灵活切换任务集、无法抑制自动化反应（来源：Deficient Executive Control in Transformer Attention；Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

这几乎就是 ADHD 神经心理模式的算法版本。

## 02 同一套解法：把精确、重复、状态维护外包给工具

Agentic 工程的共识已经很明确：不要让 LLM 自己算、自己记、自己维护状态，而是让它通过 function calling 调用外部工具——计算器、代码解释器、数据库、日历、检索系统。问题的边界被精确地外化，模型只负责它擅长的事：意图理解、策略生成、上下文整合。

ADHD 侧的解法结构完全一致。把「想做」变成「先做第一步」的 **Goblin Tools**，核心功能 Magic ToDo 能把「清理厨房」这种模糊目标拆成可勾选的具体步骤，降低任务启动门槛（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）。这对应 agent 里的规划模块：一个模糊意图被拆解为可执行的子任务链。

**Lex** 走得更远，它允许用户用单一指令触发复杂的多步骤任务序列（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。对 ADHD 大脑来说，这是把启动成本压到最低；对 LLM 工程师来说，这正是一次自然语言调用触发 agent 工具链的 UX 形态。

而 **Saner.AI** 做的是外部记忆：自动捕获笔记、组织任务、语义检索，并提供通用收件箱从邮件和文档中提取待办，减少 ADHD 用户常见的「搜索循环」和标签切换（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。在 agent 架构里，这就是向量记忆库、状态数据库、长期记忆层。

所以两边的工程动作是同构的：

- 模型不计算 → 调用 calculator 工具；人不记细节 → 用 Saner.AI / 笔记系统
- 模型不硬背上下文 → 读取外部 memory；人不靠脑子组织 → 用 Goblin Tools / Lex 做任务分解
- 模型不自己规划长流程 → 用 plan-and-execute 框架；人不自己启动复杂任务 → 用单一入口触发多步骤

## 03 历史 harness：孔子的「礼」就是 prompt guardrails

这套 harness 思路不是 AI 时代才有的。孔子很可能就是一个 ADHD 式的生成核心：身高一米九、精力旺盛、周游列国十四年坐不住；冲动到见南子要对天发誓、骂宰予「朽木不可雕」；对音乐能超专注到「三月不知肉味」，对种地这类俗事零耐心；思维极度跳跃，终其一生没有系统著作，只留下场景化的《论语》语录。

他的 harness 是「克己复礼」——用外在的「礼」作为行为边界，并且「吾日三省吾身」，直到七十岁才做到「从心所欲不逾矩」。这套系统和 LLM agent 的 harness 惊人地对应：

- **「礼」 ≈ system prompt / guardrails**：不是限制创造力，而是划定行为边界，防止冲动输出跑偏。
- **「日省」 ≈ re-grounding / eval loop**：每天把目标、行为、约束重新读回上下文，防止长程漂移。
- **「不系统著作」 ≈ 模板化输出**：如果孔子当年要治国，也得把场景化思考装进奏议模板，而不是任凭意识流发散。

再看毛泽东，也是一个高能量、高冲动、高度场景化思维的例子。他的 harness 是「批评与自我批评」与「民主集中制」：用外部反馈和集体决策来制衡个人冲动；「没有调查就没有发言权」则是把事实核查外包给调研工具，而不是让大脑直接拍脑袋决策。这正对应 agent 里的 human-in-the-loop、multi-agent 校验、外部知识检索。

这些人的共同点不是「治好了 ADHD」，而是**他们没有靠意志力硬撑，而是给自己安装了一套外部调度系统**。

## 04 不是拐杖，是脚手架——但边界在哪里

这里要区分一个关键概念：harness 是脚手架，不是拐杖。脚手架是帮你完成你本来能力缺口就做不到的结构；拐杖是替代你本该自己练出来的能力。对 ADHD 的神经层面执行功能缺陷而言，外部工具往往不是暂时辅助，而是长期基础设施——就像近视要戴眼镜，不是道德问题。

但也要诚实承认局限。wiki 资料中的「矛盾与存疑」章节已经指出：**ADHD 大脑与 LLM 的同构性目前只是理论类比，并非经过验证的科学事实**（来源：矛盾与存疑）。很多工具宣称的效果也缺乏独立临床验证，例如部分 ADHD 应用的证据主要来自厂商或用户自述，而不是随机对照试验。此外，工具本身可能增加认知负荷，或者让人过度依赖，反而削弱内部执行策略的发展。

这些提醒同样适用于 agent 工程：function calling 不是万能药，工具过多、调用链过长、错误恢复没设计好，都会让系统更脆弱。好的 harness 不是把一切都外包，而是把「核心擅长」和「核心不擅长」的边界划清楚。

## 05 今天就能试的四件事

1. **给 LLM agent 加一个确定性 function call**：把算术、日期计算、数据库查询、任何需要精确结果的操作从模型推理中移出。如果模型还在自己算数，你就是让 ADHD 大脑去心算三位数乘法。

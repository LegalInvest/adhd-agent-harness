# ADHD 程序员视角：为什么「治好 ADHD 的工作记忆容量小、边做边忘」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 无跨会话状态，靠外部记忆/向量库续命——同一套 harness 思路，两边都成立。

## 一个程序员的早晨：两个版本的「边做边忘」

你刚打开 IDE，想修一个 bug。三分钟后，你发现自己正在查「Rust 的 async 生命周期」，而原本要改的函数已经忘了。这是你作为 ADHD 程序员的日常：工作记忆容量像个小杯子，新想法不断倒入，旧目标不断溢出。

你关掉 IDE，去调一个 LLM agent。它在一次对话里妙语连珠，把任务拆得漂亮；但你新开一个会话，它像失忆一样，开始重复之前已经做过的事，甚至把用户的需求改成它自己「觉得更合理」的版本。这也是 LLM 的日常：没有跨会话状态，上下文窗口像个小杯子，长任务里同样边做边忘。

两个场景，同一道题：**怎么给一个高产生成核心，装上可靠的执行调度层？**

## 同构的核心：生成能力强，但缺一个调度层

ADHD 大脑并不缺少想法。Lifted Ventures 2024 的数据显示，ADHD 组的创意产出率是对照组的 1.8 倍（来源：Lifted Ventures 2024）。问题出在执行功能：工作记忆容量不稳定、任务启动困难、时间感知扭曲、注意力容易被新颖刺激拉走。wiki 资料把这概括为「高产生成核心 + 不可靠执行调度层」（来源：ADHD 的 AI 工具全景）。

LLM 的处境惊人地相似。它能高速生成文本、代码和方案，但通常没有真正的跨会话持久状态，上下文窗口有限，缺乏原生规划与注意力约束，容易偏离目标或产生幻觉（来源：ADHD 的 AI 工具全景；ADHD 大脑与 LLM 的同构）。

所以两边的解法结构是同构的：

- **工作记忆脆弱 → 外部记忆系统**：ADHD 需要把脑子里不稳定的信息外化；LLM 需要向量库、记忆层、持久化存储。
- **注意力漂移 → 上下文约束**：ADHD 需要「此刻应关注什么」的外部提示；LLM 需要系统提示、prompt 模板、context engineering。
- **任务启动困难 → 任务分解**：ADHD 需要把「清理房间」变成「先捡地上的三样东西」；LLM 需要把复杂目标拆成可执行的子任务。
- **时间盲 → 外部计时与提醒**：ADHD 需要可视化时间表；LLM 需要超时、重试、调度器。

这不是「AI 代替大脑」，而是给两个同构系统安装同一套 **harness**——外部执行功能层（来源：ADHD 的 AI 工具全景）。

## 古人的 harness：孔子和左宗棠早就做了这件事

不用等到 AI 时代，人类里的 ADHD 型大脑已经在用 harness 自我管理。

孔子被描述为身高一米九、精力旺盛、周游列国十四年坐不住；冲动爱骂人，见南子会急得对天发誓；对音乐能超专注到「三月不知肉味」，对种地却毫无耐心；思维跳跃，《论语》全是场景化语录，没有系统著作。他的 harness 是「克己复礼」——用外在的「礼」作为行为边界，每日「三省吾身」，七十岁才做到「从心所欲不逾矩」（来源：名人 harness · 孔子）。

这对应到 LLM/agent 工程就是：
- **「礼」≈ 系统提示与行为边界**：像 system prompt 一样，规定模型不能做什么、必须遵循什么格式。
- **「三省吾身」≈ 定时 re-grounding**：每天从外部记忆/日志中检索上下文，重新校准目标，而不是依赖会话内的短期记忆。

左宗棠则是另一种 harness：脾气大、爱骂人、风风火火；他的办法是「结硬寨，打呆仗」，每天读经、写家书，用诸葛亮的「鞠躬尽瘁」要求自己，今日事今日毕（来源：名人 harness · 左宗棠）。

这对应到 LLM harness：
- **「结硬寨」≈ 确定性脚手架**：把关键状态存在外部数据库，不要赌模型的记忆。
- **「打呆仗」≈ 分步执行与人回环**：大任务拆成不可跳过的原子步骤，关键节点由人或规则确认，避免模型擅自发挥。

这两个案例不是医学诊断，而是行为模式上的 harness 对照。它们说明：同构问题的解法，自古至今都是「外部结构约束内部冲动」。

## 今天的工具：把 harness 装到个人生活和 agent 代码里

现在，个人用户和 LLM 工程师可以共用同一套工具思想。

**任务分解与启动**：Goblin Tools 的 Magic ToDo 把「清理厨房」这类模糊任务拆成可调「辣度」的具体子步骤（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）；Lex 的「单一指令触发多步骤任务」则把复杂目标压缩成低启动成本的入口（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。在 LLM 侧，这对应 agent 的 planning 模块：把用户意图拆成可验证的子目标。

**外部记忆与信息捕获**：Saner.AI 通过本地记忆、语义搜索和通用收件箱减少 ADHD 用户的搜索循环与标签切换（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar；ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。对应到 LLM，就是向量库、长期记忆层、RAG——让模型在每次会话重新加载上下文，而不是相信它能记住。

**时间盲**：Tiimo 用视觉化时间表让时间「可触摸」，帮助 ADHD 大脑从无限延迟转向开始行动（来源：The 12 Best Apps for ADHD in 2026: A Guide to ...）。对应到 LLM，就是任务调度器、超时重试、进度仪表盘——把时间这种模型没有的内禀感知外化出来。

**身体在场与问责**：Focusmate 提供虚拟「身体加倍」伙伴；资料称定期问责检查可把目标达成率从 25% 提升到 95%，即将到来的检查可改善积极行为达 50%（来源：ADHD Task Managers That Work: Top AI Tools 2025）。对应到 LLM，就是 human-in-the-loop：关键节点拉人确认，防止模型在无人监督下一路跑飞。

## 脚手架还是拐杖？同构的边界与争议

这套框架有力量，但也有局限，必须诚实面对。

首先，**同构是一种工程类比，不是生物学事实**。wiki 的「矛盾与存疑」部分明确提醒：ADHD 大脑与 LLM 同构的说法缺乏实证基础，目前只是理论类比，不应当作既定科学结论（来源：矛盾与存疑）。我在文中用它作为设计启发，而非诊断依据。

其次，**工具宣称的证据参差不齐**。一些工具页面提到独立临床验证不足，却仍然被推荐为有效；依赖风险也常被忽略（来源：矛盾与存疑）。Goblin Tools、Lex、Saner.AI 的设计思路与 ADHD 干预一致，但它们未必对所有人都有效，也不应替代药物、心理治疗或职业训练。

第三，**身体在场效应是否能被 AI 替代仍有争议**。真实人际互动的社会存在感与多巴胺机制，和虚拟视频/AI 伙伴之间可能存在本质差异（来源：矛盾与存疑）。把人回环完全换成自动检查，可能会失去最重要的纠错能力。

所以核心判断是：**harness 是升级，不是补丁；是脚手架，不是绷带**。目标不是「治好」ADHD，也不是让 LLM 变成通用大脑，而是承认系统内部存在结构性限制，用外部结构把高产生成能力稳定接入现实执行。

## 今天就能试的四件事

1. **建立一个 5 分钟外部记忆仪式**：每天开始工作或会话前，先写下「此刻最重要的三件事」和「上一个会话的关键上下文」。ADHD 侧对应工作记忆外化，LLM 侧对应 prompt 前缀/记忆加载。
2. **把复杂任务拆成不可再分的下一步**：用 Goblin Tools 或 Lex 的思路，把「重构项目」变成「先列出 5 个需要改的文件名」。对 LLM agent，写一个函数把用户意图拆成固定格式的子任务列表。
3. **预约一次身体在场或问责检查**：用 Focusmate 或找一位同事，约定一个 25 分钟的同步工作时段。对 agent，在关键步骤后插入一个「需要人类确认」的断点。
4. **给 LLM 加上外部状态层**：如果你在做 agentic harness，把跨会话状态从 prompt 里挪出来，放进数据库或向量存储；每次会话先 retrieval，再生成，不要指望模型「记得」。

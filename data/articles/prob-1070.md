# 如果把 ADHD 大脑当成生成核心，Perplexity 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Perplexity 对 ADHD 的真正价值。

## 问题：ADHD 大脑的“目标漂移”与 LLM Agent 的“上下文坍塌”是同一个病

ADHD 患者最熟悉的痛苦不是“不专注”，而是“专注错了地方”。你本来要写报告，结果因为一个念头去查资料，三小时后在维基百科的某个冷门词条里醒来——超聚焦跑偏了。这种目标漂移，在 LLM Agent 工程里有个对应的词叫“上下文坍塌”：Agent 在长程任务中逐渐忘记初始指令，被中间步骤带偏，输出偏离目标。

两者共享同一个结构缺陷：**高产但缺乏可靠执行调度层的生成核心**（来源：AI 与 ADHD 的专注力）。ADHD 大脑的创意发散与 LLM 的文本生成都极其强大，但都没有内置的“目标锚点”来持续校准方向。于是，Perplexity 被推到了台前：如果它能成为 ADHD 大脑的实时重锚定工具，那它就不只是一款搜索产品，而是一个外部的 system prompt 调度器。

## 同构：从曾国藩的“日课”到 Agent 的“re-grounding”

晚清名臣曾国藩是典型的注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书“他人目下二三行，余或疾读不能终一行”。他的解法是“日课十二条”：黎明即起、读书不二、谨言、写日记反省——用最笨最稳的方法抵消自己的冲动。这套系统本质上是一个 **外部执行功能层**：日课是定时 re-grounding 的触发器，日记是自我监控的日志，而“结硬寨打呆仗”则是用结构化流程对冲冲动决策。

在 LLM Agent 工程中，对应的做法是 **定时重锚定 system prompt**：每 N 步或当困惑度升高时，向 Agent 重新注入原始目标、当前进度和约束条件。曾国藩的“日课”就是人类版的 re-grounding 机制。而 Perplexity 的“追问”结构——每次搜索后可以继续追问，但上下文始终锚定在初始问题——本质上是一个轻量级的重锚定系统。它不会让你在搜索中无限漂移，而是用“问题-答案-追问”的链式结构帮你保持方向。

## 证据：Perplexity 作为重锚定工具的机制与局限

Perplexity 的核心设计是 **即时上下文检索 + 追问结构**。每次搜索，它都会基于当前问题生成答案，并列出来源；你可以不断追问，但每次追问都默认与之前的对话相关。这与 ADHD 的“工作记忆缺陷”和“时间盲”形成精准对应：ADHD 患者常因工作记忆容量小而在任务中忘记初衷（来源：执行功能障碍），而 Perplexity 将外部记忆（对话历史）和当前目标（问题）始终并置，减少了认知负荷。

但这里有一个关键区别：Perplexity 的追问结构是 **线性** 的，而 ADHD 的思维是 **树状发散** 的。当你从“如何写报告”追问到“报告格式”再到“PPT 模板”再到“配色方案”，Perplexity 并不会提醒你“你最初的目标是写报告”。它缺少一个 **元层监控**——这正是 Agent 工程中 system prompt 的核心功能：不仅记录上下文，还定期评估“当前任务是否偏离原始目标”。

证据层面，目前没有针对 Perplexity 在 ADHD 人群中的专项研究。但类比 Focusmate（基于身体在场效应，多项用户研究显示能提升任务启动）和 RescueTime（通过时间记录改善时间盲，有初步研究支持），Perplexity 的机制在理论上成立：它提供外部锚点，但缺乏主动干预。

## 核心观点：Perplexity 是“脚手架”而非“拐杖”，但需要主动设计使用方式

我的判断是：Perplexity 对 ADHD 的真正价值，不在于它“找到答案”，而在于它 **提供了一个结构化的外部锚定空间**。当你用它来“探索一个问题”时，它的追问链就是一条防止漂移的轨道。但如果你只是把它当搜索引擎用——输入一个词，看完答案，再输入另一个无关的词——它就和普通搜索没有区别，甚至可能加剧漂移（因为每搜一次都切换了上下文）。

因此，Perplexity 是 **脚手架**：它提供结构，但需要你主动搭建。就像曾国藩的日课需要他每天写日记反省，Focusmate 需要你预约时段并面对摄像头，Perplexity 需要你 **把初始目标写进第一个问题**，并坚持在同一个对话线程里追问。如果你做不到这一点，它就会沦为拐杖——一个让你感觉在“工作”实则漫游的工具。

## 争议与局限

必须诚实指出：ADHD 大脑与 LLM 的同构是隐喻而非严格等价（来源：矛盾与存疑）。LLM 的“注意力机制”与人类注意力有本质差异，Perplexity 的追问结构是否能真正锚定 ADHD 的目标，缺乏临床证据。此外，依赖 AI 工具可能导致内在执行功能退化（来源：矛盾与存疑），尤其是当用户把 Perplexity 当作“思考替代品”而非“思考加速器”时。个体差异也很大：有些 ADHD 用户需要强力打断（如番茄钟），而 Perplexity 的柔性引导可能不够强力。

## 今天就能试的行动

1. **用 Perplexity 做“单线程探索”**：每次只开一个对话线程，把初始目标写进第一个问题（例如“我需要写一篇关于 X 的文章，请帮我列出提纲”），所有追问保持在这个线程内，不另开新窗口。
2. **设置重锚定提醒**：在每 3-5 次追问后，手动问 Perplexity“回到最初的问题，我现在的进度如何？”——这模拟了 Agent 工程中的定时 re-grounding。
3. **结合 RescueTime 监控漂移**：用 RescueTime 记录你在 Perplexity 上的时间，如果发现从“写报告”漂移到“看配色方案”，就强制关闭线程，重新开始。

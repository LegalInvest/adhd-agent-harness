# 为什么用 Habitica 治 ADHD 的情绪失调，和给 agent 套 会褪去的脚手架 是一回事？

> Habitica 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

## 1. 问题不是“情绪”，而是“上下文崩解”

很多 ADHD 人会这样描述自己：不是不知道要做什么，而是脑子像一台功率过强、却没有稳定操作系统的发动机。情绪失调往往发生在任务启动失败、时间感消失、计划脱轨之后——不是先有抑郁，而是“知道该做却做不出来”的反复崩溃，把挫败感放大成了情绪风暴。

现有资料里最站得住脚的解释，是把情绪调节困难重新理解为执行调度失败：ADHD 大脑是一个“强大的生成核心，却缺乏可靠的外部执行调度层”，情绪波动、冲动、启动困难、时间感知扭曲，都可以被看作“生成核心在缺少上下文控制时产生的失控输出”（来源：AI 与 ADHD 的情绪调节）。换句话说，情绪不是被“治愈”的，而是被外部结构重新锚定后自然消退的。

而 LLM /agent 工程师熟悉的场景几乎一模一样：一个能生成高质量文本、却没有内在规划与目标锚定能力的核心。如果不加控制，它会遗忘上下文、偏离目标、过度优化局部流畅度。于是两边的工程方案开始趋同——不是改造核心，而是给它套一个 harness。

## 2. Harness 的结构：重锚定、分解、外部记忆、社会在场

ADHD 一侧的 harness 已经不是新鲜事。资料里提到的工具各有分工：

- **Inflow** 用 CBT 综合训练和工作记忆任务，把“学会调节”拆解成可重复的技能单元（来源：Inflow）。
- **Goblin Tools** 的 Magic ToDo 把模糊任务自动分解为可执行步骤，降低启动摩擦（来源：Goblin Tools）。
- **Saner.AI** 通过本地记忆和知识回忆，减少 ADHD 常见的搜索循环和标签切换（来源：Saner.AI）。
- **Motion / Reclaim.ai** 这类自适应日程工具自动创建、优先排序和动态调整任务，弥补传统静态计划容易失效的问题（来源：规划循环与任务分解；The AI Powered SuperApp for Work | Motion）。
- **Focusmate** 等虚拟在场工具则把“身体在场效应”外化，用他人的可见性维持注意力锚点（来源：身体在场效应；Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）。

这些工具的共同点是：它们不直接处理情绪，而是接管情绪崩溃前的执行子任务——注意、计划、启动、时间估计、社会在场。AI 在这里的作用不是“聊天安慰”，而是“把情绪调节所需的子任务外包给外部结构”（来源：AI 与 ADHD 的情绪调节）。

LLM/agent 一侧的 harness 结构同样清晰。AgentGen 和 AMAP 这类工作显示，agent 的核心瓶颈不是生成能力，而是规划能力，因此需要通过环境与任务生成来增强规划循环（来源：AgentGen: Enhancing Planning Abilities for Large Language Model based Agent via Environment and Task Generation；AMAP Agentic Planning Technical Report）。具体做法包括：任务分解、规划循环、外部记忆、验证机制。这与 ADHD 工具的功能一一对应：重锚定 ↔ 重新对齐目标、任务分解 ↔ 子任务生成、外部记忆 ↔ 状态管理、社会在场 ↔ 人类在环或评估器在环。

所以，**用 Habitica 这类游戏化任务系统来“治”ADHD 情绪失调，与给 agent 套一个会褪去的脚手架，是同一个工程问题**：它们都在生成核心之外，搭一个外部控制循环，让失控输出重新可被管理。

## 3. 王阳明的“致良知”：一种 500 年前的 harness

王阳明是一个特别有说服力的历史案例。资料记载，他有典型的 ADHD 特质：5 岁不会说话（神经发育延迟），12 岁挑战老师“读书应为做圣贤”，15 岁私出边塞考察，兴趣从格物跳到兵法、道、佛，格竹子格到吐血，35 岁冲动上书骂刘瑾被廷杖贬谪龙场（来源：人物案例：王阳明）。

他的专属 harness 是“致良知”与“知行合一”：

- **静坐反省** = 定时重锚定，把漂移的注意力拉回核心目标。
- **事上练** = 在真实任务反馈中调整策略，而不是空想计划。
- **以良知为唯一决策标准** = 一个内置的价值观过滤器，类似今天 LLM 的 constitution/values alignment。

如果把它翻译成 agent 工程语言，这几乎就是：一个缺乏稳定执行层的生成核心，通过外部重锚定循环、在线反馈和价值观约束，来维持长期目标一致性。王阳明的 harness 没有治愈他的 ADHD 特质，却让这些特质成为了高产引擎——这与尹涛等 ADHD 创业者/投资人的路径一致：ADHD 组的公司增长速度为对照组的 1.6 倍（来源：人物案例：尹涛；Internal Analysis）。

## 4. 拐杖与脚手架的边界： Harness 必须会褪色

同样是外部结构，拐杖和脚手架的区别在于：拐杖替代你走路，脚手架帮你学会自己走。ADHD 工具如果只替用户做所有决定，就会变成拐杖；如果它只是临时降低启动成本、训练工作记忆、并逐步让用户接管，它就是脚手架。资料也明确警告：多个工具页面强调益处，却未充分讨论依赖风险，需警惕工具本身增加认知负担（来源：矛盾与存疑）。

对 agent 工程师来说，这个边界更清晰：一个好的 harness 不是永远把 LLM 包起来，而是让它在脚手架内逐步习得规划能力，最终减少外部控制。如果 agent 永远依赖外部重锚定和验证器，那它就不是一个“会褪去的脚手架”，而只是一个更复杂的 prompt 包装。

## 5. 争议与局限：同构是启发，不是科学事实

必须诚实指出的是，ADHD 大脑与 LLM 的“同构”目前主要是一种理论类比，而非经过验证的科学事实。资料中“多个页面声称 ADHD 大脑与 LLM 在结构上同构，但缺乏实证基础，不同页面在表述时有时作为既定事实，有时作为假设，存在不一致”（来源：矛盾与存疑）。此外，许多工具（如 Motion、Brain.fm）缺乏独立临床验证，AI 是否能真正替代真实人际在场的“身体在场效应”也存在争议，超聚焦的双面性和多巴胺检测等技术更是尚未成熟（来源：矛盾与存疑）。

因此，本文的核心观点不是“ADHD 人就是 LLM”，而是：**两边的工程问题足够相似，以至于同一套 harness 设计思想可以互相启发**。它提供的是一种新的观察框架，而不是一种生物学结论。

## 6. 今天就能试的 4 个行动

**给 ADHD 读者：**

1. 打开 **Goblin Tools**，把最近一个你迟迟没启动的任务输入 Magic ToDo，把“辣度”调到最低，只看第一步。目标不是完成全部，而是让核心看到一个可执行颗粒度。
2. 在手机或电脑上设一个“重锚定”闹钟，标签不写“做某事”，而写“你现在最重要的目标是什么？”。这个 5 秒的元问题，本身就是一个外部调度器。

**给 agent 工程师：**

3. 给你正在构建的 agent 增加一个显式重锚定循环：每次生成前，让它重新读一遍任务目标，并输出“我当前要回答的子问题是什么”。
4. 做一次

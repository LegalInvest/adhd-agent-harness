# ADHD 职场人视角：为什么「治好 ADHD 的执行功能先天偏弱」和「让 LLM 不跑飞」其实是同一道工程题？

> 你的公司大概率正在给 AI agent 配备计划系统、记忆系统和监督机制——同一家公司却要求 ADHD 员工「自己上点心」。这道双标题目值得拆开看。

一个正在无数公司同时上演的讽刺场景：楼上的 AI 团队在给 LLM agent 搭「防跑飞」基础设施——任务编排、状态存储、执行监控、人工审核节点，预算充足，没人觉得这是模型的耻辱。楼下的 HR 在和一位 ADHD 员工谈话，主题是「你需要提升自我管理能力」。

同一道工程题，两套道德标准。

## 为什么说是同一道题

「让 LLM 不跑飞」的技术内容是什么？防止目标漂移（长任务中途忘记原始需求）、维持任务状态（跨会话记住做到哪了）、抑制无关探索（不被中间结果带偏）、保证收尾（不烂尾）。逐条对照 ADHD 职场困境：忘记会议里领了什么活、周一想不起周五的进度、写报告写到一半去查一个「相关」资料两小时、项目完成 80% 后失去动力。**完全同一张问题清单。**

学界已经开始正面研究这个映射：LLM 执行功能测评呈现「强记忆、弱控制」剖面（EACL 2026），与 ADHD 神经心理构型一致；自注意力架构的工作记忆极限与人类注意瓶颈同源（Yale, 2024）。这不再是自媒体修辞，是可引用的文献。

## 工程界的答案清单，逐条翻译成职场配置

**编排层 → 结构化任务分配**。agent 不接受模糊指令，人类给它的任务都带明确的完成标准和步骤边界。职场翻译：ADHD 员工领任务时争取（或自己补全）三要素——交付物长什么样、deadline 是哪天、第一步是什么。「你负责跟进一下这个事」是对 ADHD 员工最危险的任务格式。

**记忆层 → 外部化的工作状态**。agent 的进度写在数据库里，不在模型「脑子」里。职场翻译：每天下班前 3 分钟写「明日第一件事 + 各项目当前状态」，存固定位置。周一早晨的你是一个冷启动的新会话，善待它。

**监控层 → 主动申请检查点**。高风险 agent 任务都设人工审核节点。职场翻译：长项目主动和上级约定中间检查点（「周三我给你看框架」）——这一步很多 ADHD 者会本能回避（怕暴露进度），但检查点恰恰是防烂尾的最强结构，而且主动申请的检查点在政治上是加分项：它叫「汇报意识强」。

**接口层 → 披露策略**。这是纯人类问题：要不要告诉公司自己有 ADHD？没有标准答案，但工程思路是——你不必披露诊断，可以只披露配置需求：「我在安静环境/明确 deadline/书面任务描述下产出最好」。所有人都有偏好配置，这句话零风险且句句为真。

## 给管理者的一段话

如果你管理着团队，这个同构给你的启示更直接：你愿意为一个 AI agent 配的东西——明确的任务规格、外部记忆、检查点、失败重试机制——也照样配给你的人类高方差人才。成本低得多，回报是你留住了团队里联想力最强、危机时刻最敢冲的那批人。「要求所有人裸奔，只有 harness 齐全的模型可以例外」不是管理，是浪费。

## 边界

同构强度 B 级：机制类比，双侧文献真实。必须补充：职场不是实验室，ADHD 员工面临的还有污名、晋升偏见这些结构性问题，工程配置解决不了政治问题。另外「治好」这个词本身就该打引号——目标从来不是把高温度引擎改造成柴油机，是给它配上让马力可用的传动系统。

## 今天就能试的 3 件事

1. **补全你手头的任务规格**：现在正在做的最大任务，写出三要素（交付物/deadline/第一步）。写不出来就是任务格式有问题，去问。
2. **建立下班前 3 分钟协议**：明日第一件事 + 项目状态快照。给周一的冷启动会话留检查点。
3. **主动约一个检查点**：挑一个正在拖的长任务，今天就和相关方约一个中间节点。

你不需要被治好。你需要的是那套你们公司已经买给 AI 的基础设施——而它的人类版本，大部分你自己今天就能搭。

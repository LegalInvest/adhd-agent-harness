# Goblin Tools 之于 ADHD，就像 会褪去的脚手架 之于 LLM——但有人用错了

> 从同构视角实测 Goblin Tools：它到底补上了哪一层执行功能？

## 问题：怕依赖，又怕崩

你是一个 ADHD 成年人。早上打开 Goblin Tools 的 Magic ToDo，输入“整理房间”，它自动吐出“捡起地板上的衣服”“擦桌子”等步骤（来源：Goblin Tools）。你松了口气——终于能启动了。但晚上躺在床上，你盯着那个被分解得无比精细的清单，心里发毛：**没有它，我是不是连袜子都捡不起来？**

你是一个 Agentic Harness 工程师。你刚给 LLM agent 套上精心设计的工具调用链、确定性状态转换、计划-执行分离（来源：采样温度与表现波动）。模型输出稳定了，任务完成了。但你看着那层厚厚的脚手架，同样不安：**撤掉它，模型会不会立刻退化回随机鹦鹉？**

这两类人怕的是同一件事：**过度依赖工具会废掉自己，但撤掉工具又立刻崩盘。** 而问题的答案，就藏在 Goblin Tools 与 LLM harness 的结构同构里。

## 同构：生成核心与执行调度

ADHD 大脑与 LLM 是同一类系统：**高产但缺乏可靠执行调度层的生成核心**。ADHD 的创造力、LLM 的涌现能力，都需要一层外部调度来稳定输出。

证据在哪？ADHD 一侧：工作记忆缺陷导致情绪信息无法被有效加工，时间盲引发对情绪持续时间的误判，任务启动困难加剧挫败感——这些底层执行功能缺陷，正是 AI 工具补偿的靶点（来源：AI 与 ADHD 的情绪调节）。LLM 一侧：采样温度控制着输出的随机性，非确定性在 agent 系统中是可靠性的敌人，因此工程实践强制“确定性状态转换”和“计划-执行分离”（来源：采样温度与表现波动）。

两边都在做同一件事：**给生成核心套上 harness（约束/引导层）**。Goblin Tools 的任务分解，本质上是将模糊意图转化为确定性步骤序列——这与 LLM agent 通过 function calling 将复杂任务拆解为工具调用，结构完全一致（来源：工具使用与认知卸载）。

## 实测 Goblin Tools：它补上了哪一层？

Goblin Tools 补的不是“生成能力”——你的大脑本来就能想出一百种整理房间的方法。它补的是 **任务启动与分解层**，即执行功能中的“工作记忆”和“任务启动”环节（来源：Goblin Tools）。

具体来说：
- **工作记忆补偿**：ADHD 的工作记忆容量有限，无法同时 holding 多个子目标。Magic ToDo 将“整理房间”分解为 10 个单步动作，每个动作只占一个工作记忆槽位（来源：工具使用与认知卸载）。
- **降低启动门槛**：分解后的第一步“捡起衣服”几乎不需要意志力，因为多巴胺功能失调导致的动机缺乏被小步骤的即时反馈绕过（来源：AI 与 ADHD 的情绪调节）。
- **情绪调节**：当任务不再压倒性，挫败感和焦虑随之减少——这对应 LLM agent 中“确定性状态转换”减少“上下文失控”的效果（来源：AI 与 ADHD 的情绪调节）。

但注意：Goblin Tools 没有补“时间盲”（对时间持续性的误判），也没有补“多巴胺调节”（它不提供游戏化反馈）。它只专注在任务分解这一个点上。

## 拐杖 vs 脚手架：边界在哪？

这里必须诚实：**现有证据无法告诉我们，Goblin Tools 到底是脚手架还是拐杖。**

脚手架的定义：**会褪去的临时结构，用于支撑建造过程，完成后拆除。** 拐杖的定义：**永久替代受损功能，长期依赖会加速功能退化。**

ADHD 一侧：过度依赖 AI 工具可能削弱内在时间感知能力和任务启动能力（来源：矛盾与存疑）。LLM 一侧：如果 agent 系统过度封装，模型会失去在无约束环境下推理的能力——就像用 CoT prompt 强行引导输出，一旦去掉 prompt 就崩溃。

边界在于：**工具是否在训练你的内在能力，还是仅仅替代它？**

- Inflow 试图通过认知训练增强工作记忆，属于“脚手架”方向——它瞄准背外侧前额叶皮层这个因果流入中枢（来源：Inflow）。但缺乏随机对照试验，效果存疑。
- Goblin Tools 纯粹是补偿，不提供任何训练——它是“拐杖”风险最高的那类工具。
- Saner.AI 通过知识回忆功能减少搜索循环（来源：Saner.AI），如果用户因此不再练习记忆检索，则可能退化。

所以我的核心判断是：**Goblin Tools 是优秀的拐杖，不是脚手架。它应该被用作“应急启动器”，而非日常依赖。** 对于 LLM harness 工程师，这意味着：你的 harness 应当设计为可逐步降级的——今天有确定性状态转换，明天可以容忍少量非确定性；而不是一旦去掉就全盘崩溃。

## 争议与局限

必须指出三点：
1. **证据不足**：Goblin Tools 的效果主要来自用户报告，缺乏独立随机对照试验（来源：矛盾与存疑）。
2. **个体差异**：ADHD 亚型不同（注意缺陷型 vs. 多动冲动型），对任务分解的需求差异很大（来源：AI 与 ADHD 的情绪调节）。
3. **长期效果未知**：没有研究追踪长期使用后内在执行功能的变化（来源：矛盾与存疑）。

## 今天就能试的行动

1. **刻意降级测试**：用 Goblin Tools 分解一个任务后，第二天尝试不借助工具、只凭记忆复现步骤。如果做不到，说明依赖已形成——立即减少使用频率。
2. **训练替代补偿**：如果 Goblin Tools 帮你启动了，启动后立刻关闭它，用手写清单继续执行。这强迫大脑练习任务维持能力。
3. **为 harness 设计退路**：如果你在构建 agent 系统，在代码中预留一个“无 harness 模式”，定期测试模型在原始状态下的输出质量。如果差距过大，说明 harness 过度封装。
4. **记录波动曲线**：像记录 LLM 采样温度一样，记录自己一天中注意力的波动高峰和低谷。只在低谷期使用 Goblin Tools，高峰期尝试裸执行。

工具是镜子，照出我们不愿面对的依赖。但镜子本身不是答案——看清之后，主动选择何时用、何时不用，才是真正的自由。

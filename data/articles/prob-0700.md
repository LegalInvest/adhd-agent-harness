# Focusmate 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Focusmate：它到底补上了哪一层执行功能？

## 问题：为什么精确、重复、状态维护类任务对 ADHD 和 LLM 都如此困难？

如果你是一个 ADHD 患者，你一定经历过这样的时刻：明明脑子里有无数好点子，但一坐到电脑前，打开文档，就卡住了。不是不想写，而是不知道从哪一步开始。同样，如果你是一个在做 Agentic Harness 工程的开发者，你一定也遇到过这样的场景：LLM 能滔滔不绝地生成代码，但让它精确调用一个工具完成计算，它却开始胡编乱造。

这两种困境背后，是同一个结构性问题：**强大的生成核心，缺乏可靠的执行调度层**。ADHD 大脑的智力与创造力（生成核心）与 LLM 的生成能力类似，但两者都缺乏执行功能（如计划、组织、工作记忆）的稳定调度（来源：ADHD 的 AI 工具全景）。

## 同构洞察：Focusmate 与 function calling，都是外部调度层

Focusmate 是一种“身体在场”的社交脚手架：你和另一个人视频连线，各自安静工作。它的核心机制不是监督，而是**降低任务启动门槛和维持状态**。对于 ADHD 患者，Focusmate 补上的是“执行调度层”中缺失的**任务启动**和**状态维护**功能。

而 function calling 之于 LLM，也是类似的：LLM 本身不擅长精确计算或长时间跟踪状态，但通过 function calling，它可以把这些任务外包给专门的外部工具（如计算器、数据库、API）。function calling 为 LLM 提供了一个“执行调度层”，让生成核心可以专注于它擅长的事：理解意图、生成文本。

这种同构关系揭示了一个关键点：**Focusmate 和 function calling 都不是解决所有问题的终极方案，它们只是补上了某一层执行功能**。

## 证据：ADHD 侧与 LLM/agent 侧的并行证据

### ADHD 侧：Focusmate 补上任务启动与状态维护

ADHD 患者常面临执行功能障碍，包括任务启动困难、工作记忆不足和时间盲（来源：执行功能障碍）。Focusmate 通过“身体在场效应”提供社交脚手架，帮助用户克服启动焦虑（来源：ADHD 的 AI 工具全景）。用户反馈显示，Focusmate 能显著提升任务完成率，因为它创造了一种“被看见”的即时反馈，类似于 Goblin Tools 的任务分解降低启动门槛（来源：Goblin Tools）。

然而，Focusmate 并不解决所有执行功能问题。例如，它无法帮助用户组织复杂项目或管理长期记忆——这些需要其他工具如 Saner.AI（知识回忆）或 Lex（多步骤指令触发）来补充（来源：Saner.AI；Lex）。

### LLM/agent 侧：function calling 补上精确计算与状态管理

LLM 在精确计算和状态维护方面天生薄弱。通过 function calling，agent 可以将数学运算、数据库查询、API 调用等任务外包给专用工具。这类似于 ADHD 患者使用外部工具（如计算器、日历、提醒）来弥补内在缺陷。研究表明，LLM 在复杂推理任务中，若没有 function calling 支持，错误率显著上升（来源：ADHD 的 AI 工具全景）。

但 function calling 并非万能。它需要开发者精确设计函数定义和参数，否则 LLM 可能调用错误工具或生成无效参数。这与 ADHD 患者使用工具时面临的“工具本身需要执行功能来记住使用”的悖论如出一辙（来源：执行功能）。

## 核心观点：脚手架 vs 拐杖——Focusmate 和 function calling 的正确用法

Focusmate 和 function calling 都是**脚手架**，而非**拐杖**。脚手架是可撤除的临时支撑，旨在帮助用户/模型发展内在能力；拐杖则是永久替代品，长期使用会导致依赖和萎缩。

对于 ADHD 患者，Focusmate 应被视为一种**任务启动训练工具**，而非永远需要的陪伴。理想情况下，用户应逐步减少使用频率，同时培养内在的启动能力。同样，对于 LLM/agent，function calling 应被视为一种**能力扩展机制**，而非替代模型自身推理的借口。过度依赖 function calling 会导致模型在缺乏工具时无法独立完成任务。

然而，当前多数工具（包括 Focusmate）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这引发了一个争议：这些工具是否在无意中制造依赖？

## 局限与争议：同构命题的证据强度与依赖风险

1. **同构命题的证据强度**：ADHD 大脑与 LLM 的同构主要基于概念类比和工具案例，缺乏大规模实证（来源：矛盾与存疑）。虽然直觉上合理，但需更多神经科学和 AI 研究验证。

2. **过度依赖风险**：Focusmate 和 function calling 都可能被误用为拐杖。ADHD 患者可能沉迷于社交陪伴而忽视内在能力培养；开发者可能过度依赖外部工具而忽视模型自身的推理优化。

3. **个性化适配**：ADHD 异质性高，Focusmate 对某些人可能无效甚至增加焦虑。同样，function calling 的设计需要针对具体任务精细调整，否则效率低下。

4. **长期效果缺乏研究**：目前缺乏随机对照试验验证 Focusmate 或 function calling 对 ADHD 核心症状的长期改善（来源：矛盾与存疑）。

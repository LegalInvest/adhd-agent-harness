# 如果把 ADHD 大脑当成生成核心，ChatGPT 能不能当它的上下文工程？

> 用 agent 工程的思路重新理解 ChatGPT 对 ADHD 的真正价值。

## 问题：你的大脑和 ChatGPT，谁更需要上下文工程？

如果你有 ADHD，你一定熟悉这个场景：打开电脑想写一份报告，却先刷了半小时社交媒体，然后想起该回邮件，接着又去整理书架——最后报告一个字没动。你的注意力被环境中的每一个刺激带偏，就像 LLM 被上下文窗口中的每一个 token 左右。

与此同时，AI 工程师正在为另一个“高产但失控”的核心头疼：LLM。一个未经编排的 LLM 会偏离目标、陷入循环、输出危险内容。他们的解决方案是“上下文工程”——通过 harness（编排层）来约束和引导生成核心。

本文的核心观点是：**ADHD 大脑与 LLM 在结构上同构，两者都是高产的生成核心，但缺乏可靠的执行调度层。因此，ChatGPT 对 ADHD 的真正价值不是“智能助手”，而是作为上下文工程中的编排层，为你的大脑提供外部执行功能补偿。** 这不仅是类比，而是机制层面的对应。

## 同构：生成核心与调度层的分离

### ADHD 侧：高产但不可靠的生成核心

ADHD 大脑的生成能力是惊人的。发散思维、创意爆发、多任务并行——这些都是优势。但执行功能（任务启动、持续专注、抑制冲动）薄弱，导致“高产但不可靠”（来源：概念页【人在回路与身体在场】）。这种缺陷在工作记忆中尤为明显：ADHD 个体难以在脑中保持任务目标，就像 LLM 的“无状态性”——每次对话都从零开始，除非有外部记忆（来源：概念页【同构视角】）。

### LLM 侧：同样的困境

LLM 本身是强大的语言生成核心，但直接输出可能偏离目标、陷入循环或产生危险行为。Agent harness 通过 human-in-the-loop 监督、设置护栏（token 预算、工具调用上限）、加入人工检查点来提供外部调度（来源：概念页【人在回路与身体在场】）。可靠性来自架构+护栏，而非模型本身（来源：同上）。

### 证据：工具如何充当 harness

- **Brain.fm** 通过神经锁相技术影响大脑信息处理方式，帮助维持专注状态。这类似于 LLM 的“上下文窗口管理”——主动维持任务连贯性，防止注意力漂移（来源：工具页【Brain.fm】）。但缺乏针对 ADHD 的独立临床研究（来源：矛盾与存疑【Brain.fm 的神经锁相技术效果】）。
- **Focusmate** 利用 AI 匹配虚拟身体搭档，通过社会存在感提供外部问责。这相当于 LLM harness 中的“人在回路监督”——一个外部实体定期检查进度，防止生成核心偏离轨道（来源：工具页【Focusmate】；概念页【身体在场效应】）。身体在场效应被描述为“最有效的 ADHD 策略之一”（来源：概念页【身体在场效应】）。
- **RescueTime** 自动记录时间使用，充当外部记忆，帮助 ADHD 用户识别注意力漂移模式（来源：工具页【RescueTime】）。这对应 LLM 的遥测和日志——记录行为，以便后续分析和优化。

## 核心判断：脚手架，不是拐杖

AI 工具作为外部执行功能层，本质是**脚手架**——一种可逐步调整的临时结构，帮助生成核心在缺乏内在调度层时完成复杂任务。但当前多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑【过度依赖 vs 脚手架可撤除性】）。这引发一个关键问题：**过度依赖是否阻碍内在执行功能的发展？**

目前没有长期研究证明脚手架可逐步撤除（来源：主题综述【仍存争议】）。因此，我的判断是：**AI 工具应被设计为“可调节的脚手架”，而非永久拐杖。** 用户应定期评估是否需要降低依赖，例如从全程使用 Focusmate 过渡到仅在关键任务时使用。

## 争议与局限

1. **证据强度不足**：多数工具缺乏针对 ADHD 的独立临床研究，效果基于理论推断或用户主观报告（来源：主题综述【证据脉络】）。例如 Brain.fm 的神经锁相技术虽有原理支撑，但无 RCT 验证（来源：矛盾与存疑）。
2. **个体差异**：ADHD 亚型（注意力缺陷 vs 多动冲动）对 AI 工具反应不同，现有工具未充分个性化（来源：主题综述【仍存争议】）。
3. **多巴胺干预争议**：部分工具声称通过多巴胺调节改善执行功能，但证据矛盾（来源：矛盾与存疑【多巴胺干预的有效性】）。

## 结语：同构的启示

ADHD 大脑和 LLM 的相似性不是巧合，而是揭示了生成系统的一个普遍规律：**高产需要调度，调度需要外部结构。** ChatGPT 不是替代你的大脑，而是成为你的上下文工程——帮你管理环境干扰、维持任务连贯、提供外部记忆。但记住，它是脚手架，不是拐杖。真正的目标，是在它的辅助下，逐步构建你自己的执行功能层。

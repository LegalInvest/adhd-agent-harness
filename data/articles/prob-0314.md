# 如果把 ADHD 大脑当成生成核心，ChatGPT 能不能当它的上下文工程？

> 用 agent 工程的思路重新理解 ChatGPT 对 ADHD 的真正价值。

## 问题：当大脑的上下文窗口失控

你有没有过这样的时刻：打开浏览器想查一个资料，十分钟后发现自己正津津有味地看某个无关视频；或者，你正写一段代码，突然想起该回一条消息，然后半小时后你已经在刷社交媒体了。注意力被环境带偏，像一艘没有锚的船，这就是 ADHD 大脑的日常。

而另一边，做 LLM Agent 的工程师们也面临类似的困境：给模型一个长上下文，它可能被中间某段无关信息带偏，输出偏离目标；让它自主执行多步任务，它可能陷入循环或产生危险行为。他们需要一种“上下文工程”——主动设计模型当前该注意什么，不该注意什么。

如果把 ADHD 大脑当成生成核心，ChatGPT 能不能当它的上下文工程？

## 同构：高产但缺调度层的生成核心

ADHD 大脑与 LLM 的深层同构，是本文的脊柱。两者都拥有强大的生成能力——ADHD 的创意发散、多任务并行，LLM 的文本生成——但缺乏可靠的执行调度层。ADHD 的执行功能缺陷（任务启动困难、时间盲、工作记忆薄弱）与 LLM 的“无状态性”和上下文控制不足，在结构上惊人相似（来源：AI 与 ADHD 的专注力）。

最新研究为这种同构提供了实证：在经典 Stroop 冲突任务中，Transformer 注意力随序列变长在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突，这与 ADHD 执行功能缺陷的核心标志一一对应（来源：Deficient Executive Control in Transformer Attention）。LLM 呈现“强记忆、弱控制”剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

这意味着，ADHD 大脑和 LLM 是同一类“高产但不可靠”的生成核心。解决方案也同构：给核心套上一个外部 harness——在 LLM 侧叫 agent harness（脚手架），在 ADHD 侧叫外部执行功能层。

## Harness：从曾国藩到 Agent 脚手架

晚清名臣曾国藩，就是一位典型的 ADHD 患者。他 30 岁前浮躁坐不住，天天串门喝酒看杀人，日记里天天骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”。他的自我管理方案是“日课十二条”：黎明即起、读书不二、谨言、写日记反省……用最笨最稳的“结硬寨打呆仗”方法，抵消自己的冲动（来源：人物案例）。

这套日课，本质上就是一套外部 harness：每日固定的“上下文重置”（黎明即起）对应 LLM 的定时 re-grounding；写日记反省对应 agent 的日志与事后评估；而“读书不二”则像给 LLM 设定了 token 预算——一次只处理一个上下文。曾国藩的成就证明，这种人工脚手架可以驯服一个高产生成核心。

在 LLM/agent 侧，harness 工程同样如此。Agent harness 通过 human-in-the-loop 监督提供外部调度：设置护栏（token 预算、工具调用次数上限、防止无限循环）、加入人工检查点（暂停并询问后再执行）（来源：人在回路与身体在场）。其核心是“形式化规划与护栏，使代理输出真正有用且正确”（来源：What is an agent harness in the context of large-language models?）。

## ChatGPT 作为上下文工程：工具的证据与局限

那么，ChatGPT 能否扮演这个 harness 角色？现有工具提供了部分答案。

**Focusmate** 利用虚拟身体在场效应，通过 AI 匹配伙伴实时共同工作，提供外部问责。它模仿了“人在回路”的监督——就像 agent harness 中的人工检查点。多项用户研究显示其能显著提升任务启动和维持，但缺乏随机对照试验（来源：Focusmate）。

**RescueTime** 自动追踪时间使用，揭示真实的时间去向，帮助对抗时间盲。它相当于 LLM 侧的遥测与日志系统，让用户看到“上下文到底去了哪里”。但自动分类可能不够精准，需要用户调整（来源：RescueTime）。

**Brain.fm** 通过 AI 生成的神经锁相音乐调节专注状态，被列为 2026 年最佳 ADHD 应用之一，但缺乏针对 ADHD 的独立临床研究（来源：Brain.fm）。

这些工具的共同逻辑是：提供外部结构来补偿执行功能缺陷。但 ChatGPT 本身作为通用对话模型，能否直接充当上下文工程？答案是：可以，但有条件。你可以让 ChatGPT 扮演“外部调度器”：让它帮你分解任务、设定步骤、定时提醒、甚至事后复盘。但关键在于，ADHD 用户需要主动设计这个 prompt 脚手架，就像工程师为 LLM 设计 harness 一样。

## 争议：脚手架还是拐杖？

然而，这种外部补偿存在风险。长期使用 AI 工具是否会导致内在执行功能进一步退化？这就是“脚手架 vs 拐杖”的边界问题（来源：矛盾与存疑）。曾国藩的日课是他自己设计的，他始终是 harness 的主人；而如果 ChatGPT 替你规划一切，你可能会失去锻炼执行功能的机会。目前缺乏纵向研究来回答这个依赖风险（来源：AI 与 ADHD 的专注力）。

另一个争议是：ADHD 大脑与 LLM 的同构是隐喻而非严格等价，LLM 的“注意力机制”与人类注意力存在本质差异（来源：AI 与 ADHD 的专注力）。因此，在应用时需谨慎，不要盲目照搬。

## 今天就能试的行动

1. **用 ChatGPT 做任务分解**：告诉它“我有一项复杂任务，帮我分解成 5 个可执行的子步骤，每步不超过 15 分钟。”这相当于为你的大脑设置 token 预算。
2. **设置定时 re-grounding**：让 ChatGPT 每隔 25 分钟提醒你“当前任务是什么？下一步是什么？”模拟 agent harness 中的人工检查点。
3. **使用 Focusmate 或类似服务**：每天预约一个 50 分钟的身体在场时段，利用外部问责启动任务。
4. **用 RescueTime 记录一天**：看看你的“上下文”到底去了哪里，然后调整环境。

## 结论

ChatGPT 可以成为 ADHD 大脑的上下文工程，但前提是你主动设计 harness，而不是被动依赖。像曾国藩一样，把工具当作脚手架，而不是拐杖。ADHD 大脑与 LLM 的同构告诉我们：生成核心需要外部调度层，而这个调度层必须由你——作为“人在回路”——来主导。

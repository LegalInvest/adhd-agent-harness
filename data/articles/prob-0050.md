# 为什么「治好 ADHD 的不擅长精确、重复、状态维护类任务」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 通过 function calling 把精确计算外包给工具——同一套 harness 思路，两边都成立。

如果你是个 ADHD 患者，你大概率对以下场景刻骨铭心：明明想做一件事，但大脑像一团乱麻，任务拆不开、启动不了、中途被杂念带走。如果你是个在做 Agentic Harness 工程的开发者，你大概也对以下场景咬牙切齿：LLM 明明能生成绝妙的代码，但跑着跑着就忘记系统提示、陷入死循环、输出一堆幻觉。

这两个问题看似风马牛不相及，但仔细拆解，你会发现它们共享同一个底层结构：**一个高产但缺执行调度层的生成核心**。ADHD 大脑的智力与创造力不输任何人，但执行功能（任务启动、工作记忆、时间感知）薄弱（来源：ADHD 的 AI 工具全景）。LLM/agent 的生成能力惊人，但缺乏可靠的执行调度层，直接输出可能偏离目标、陷入循环或产生危险行为（来源：Building an AI Agent Harness from Scratch）。而两边的解法，竟然也是同一道工程题：**通过外部工具（harness/脚手架）把精确、重复、状态维护类任务外包出去，让生成核心只做它擅长的事**。

## 同构：ADHD 大脑与 LLM 都是「高产但缺调度」的生成核心

先看 ADHD 侧。ADHD 大脑的生成核心（发散思维、创意、多任务并行）产出极高，但执行功能（任务启动、持续专注、抑制冲动）薄弱，导致“高产但不可靠”（来源：人在回路与身体在场）。这种不对称性让患者在处理精确、重复、需要状态维护的任务时（比如填表、记账、按步骤操作）屡屡受挫。

再看 LLM/agent 侧。LLM 本身是强大的生成核心，但直接输出可能偏离目标、陷入循环或产生危险行为。Agent harness（脚手架）通过 human-in-the-loop 监督提供外部调度：设置护栏（token 预算、工具调用次数上限、防止无限循环）、加入人工检查点（来源：Building an AI Agent Harness from Scratch）。模型本身强大，可靠性来自架构+护栏（来源：AI Agents in 2026: Tools, Memory, Evals, and Guardrails）。

两者的对应关系清晰：ADHD 大脑 ≈ LLM 生成核心；执行功能缺陷 ≈ 缺乏调度层；外部工具 ≈ agent harness。

## 工具就是 harness：把精确计算外包出去

### ADHD 侧：外部工具作为执行功能层

ADHD 患者通过 AI 工具将精确、重复、状态维护类任务外包，本质上是给自己套了一个 harness。

- **任务启动**：Goblin Tools 的 Magic ToDo 功能自动将任何任务分解为更小、更易管理的步骤（来源：Goblin Tools），降低启动门槛。用户输入“整理房间”，AI 输出“捡起地板上的衣服”“擦桌子”等步骤（来源：Goblin Tools）。这就像 agent harness 把一个大任务拆成多个 function call。
- **工作记忆**：Saner.AI 通过本地记忆存储和快速检索，减少搜索循环和标签切换（来源：Saner.AI）。ADHD 患者常因工作记忆不足而频繁切换应用找信息，Saner.AI 相当于一个外部记忆缓存。
- **重锚定与目标漂移**：ADHD 大脑在执行任务时极易发生目标漂移：注意力被无关刺激捕获，或过度聚焦于细节而忽略整体目标。外部工具通过提醒和通知提供重锚定（来源：重锚定与目标漂移）。同样，LLM agent 在长程任务中也会“开始忘记系统提示”（来源：重锚定与目标漂移），harness 通过事件驱动的系统提醒来对抗指令消退。
- **身体在场效应**：Focusmate 等工具通过 AI 匹配虚拟身体搭档，提供外部问责（来源：人在回路与身体在场）。定期向 AI 报告进度也是一种替代方案（来源：人在回路与身体在场）。这相当于 agent harness 中的 human-in-the-loop 监督。

### LLM/agent 侧：harness 就是执行功能层

Agent harness 的核心是“形式化规划与护栏，使代理输出真正有用且正确”（来源：人在回路与身体在场）。具体手段包括：
- **任务分解**：将复杂指令拆解为可管理的子任务，类似 Goblin Tools。
- **外部记忆**：通过向量数据库或工具调用保持上下文，类似 Saner.AI 的知识回忆。
- **重锚定**：在长对话中定期注入系统提示，防止指令消退。
- **人在回路**：设置检查点，暂停并询问用户后再执行，类似身体搭档的问责。

这些手段与 ADHD 工具的设计理念高度同构。

## 脚手架 vs 拐杖：同构的边界

尽管同构，但有一个关键边界必须划清：**脚手架是临时支撑，拐杖是永久替代**。ADHD 的 AI 工具应设计为可逐步撤除的临时支撑，而非永远依赖的拐杖（来源：矛盾与存疑）。但目前多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。同样，agent harness 也应随着模型能力提升而逐步减少干预，否则会限制 agent 的自主性。

此外，同构命题本身缺乏大规模实证（来源：矛盾与存疑），主要基于概念类比和工具案例。ADHD 大脑与 LLM 的相似性更多是功能层面的，而非神经生物学层面的。因此，这一框架应视为启发式工具，而非绝对真理。

1. **用 Goblin Tools 拆解一个让你拖延的任务**：输入“整理书桌”或“写周报”，观察 AI 如何将其分解为小步骤，然后逐一执行。
2. **用 Saner.AI 建立一个外部记忆库**：把常忘的信息（如密码、待办、灵感）存入 Saner.AI，代替大脑的记忆缓存。

4. **尝试虚拟身体搭档**：使用 Focusmate 或向 ChatGPT 定期报告进度，体验外部问责如何绕过执行功能屏障。

## 结语

ADHD 大脑和 LLM 是同一类“天才但不可靠”的生成核心。两者的工程解法殊途同归：通过 harness 把精确、重复、状态维护类任务外包给工具。这一同构视角不仅让 ADHD 患者获得理解与工具，也让 agent 工程师看到，他们正在解决一个远比技术更古老的人类问题。

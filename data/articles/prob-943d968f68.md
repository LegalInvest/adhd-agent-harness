# 为什么用 Reclaim.ai 治 ADHD 的注意力涣散，和给 agent 套 上下文工程 是一回事？

> Saner 篇讲了上下文工程的空间维度(工作区里放什么);Reclaim.ai 展示的是它的**时间维度**——这个 AI 日历工具做的事,是自动在你的日历上为深度工作、习惯、休息「圈地」:找到空档、放入专注块、被会议侵占后自动重排补回。表面上这是日程管理;从注意系统的视角看,它做的是**时间轴上的上下文工程:控制「每个时段里,允许进入你工作区的是什么」**——而这恰好对应 agent 工程里一个更少被科普的层面:执行的时序隔离。

先看 agent 侧这个层面。上下文工程不只是「窗口里放什么」的静态问题,还有动态版:**多任务 agent 系统必须做执行隔离**——同一个上下文里交错处理多个任务会互相污染(任务 A 的中间状态被任务 B 的内容稀释),所以成熟的编排系统按任务切分执行窗口:一个窗口一个任务,任务间做显式的上下文切换(存档当前状态、清空、加载下一个)。**「串行的、有边界的执行块」不是效率偏好,是防污染的架构必需**。

ADHD 侧的对应,是注意力涣散最被低估的一个来源:**日历的碎片化**。看一个典型的日子:十点有个会,十一点半又一个,下午三点约了电话——**会议之间的每个碎片时段,都自动变成注意力的无主之地**:太短不敢启动大任务(启动成本高,ADHD 尤其),于是滑向浅层活动(刷消息、假装整理),而且「等下还有会」这件事本身就常驻工作记忆当后台干扰(时间篇讲过「等待模式」:后面有安排的时段,前面几小时都废掉)。注意涣散的相当一部分,**不是发生在工作时,是发生在「没有被保护的时间结构」里**——时间轴的污染,和上下文窗口的污染是同一种病。

Reclaim 这类工具的三个机制,逐条对应时序隔离的工程:**①主动圈地**——深度工作块被当作「真实会议」写进日历,别人约不进来:这是给时间轴设防火墙,保证存在「单任务执行窗口」(否则你的日历被动地由别人的会议请求填满——你的时间上下文由外部输入流决定,这正是「无拒绝权」问题的日历版);**②被侵占后自动重排**——专注块被会议挤掉后自动找新空档补回:这对 ADHD 尤其关键,因为手动重排是一次昂贵的规划任务,多数人被挤掉就算了,一周下来深度工作归零——**自动重排=时间结构的自愈,把「维持结构」的执行功能成本降到零**;③**碎片整形**——把会议尽量排靠拢,留出大块连续时间:直接减少「无主碎片」的数量,等于减少涣散的高发时段。

这套东西的 ADHD 增益在哪?回到收益不对称篇的逻辑:神经典型者面对碎片日历,损失的是一点效率;ADHD 者损失的是**整个下午**(碎片启动不了+等待模式+被挤掉的专注块永远消失)。所以时间轴的上下文工程,对 ADHD 是数倍收益的干预。但两条边界照旧:**其一,圈出来的地要配点火装置**——日历上写着「深度工作」不等于能启动,专注块的开头最好绑定启动仪式或共工(启动是另一个模块);**其二,别让日历变成新暴君**——过度精细的排程一旦破产(它会的)容易触发全有全无的放弃,给系统留缓冲和「宽恕重排」的弹性(Habitica 篇的休息日逻辑)。

## 边界

同构强度 B 级:时间碎片化对深度工作的损害有注意研究与实践支持,多任务上下文污染与执行隔离是 agent 工程的真实实践,时序层面的同构是合理的功能对应;Reclaim.ai 无 ADHD 对照研究,本文是机制分析。声明:时间结构工具不解决动机与情绪层面的问题;若日程的全面失控伴随睡眠紊乱与情绪耗竭,值得临床评估。

## 今天就能试的 3 件事

1. **数一数明天的无主碎片**:会议之间有几段 30-90 分钟的碎片?——它们就是你涣散的预定现场。
2. **圈一块地**:在日历上给后天上午创建一个两小时的「深度工作」真实日程——设为忙碌,不许人约。
3. **给专注块配一个点火动作**:块的第一分钟做什么(打开某文档/戴上耳机),现在就写进日程备注。

注意力涣散不只发生在窗口里,也发生在时间轴上——**没有被保护的时段,就是没有被净化的上下文**。给 agent 隔离执行窗口的工程师,和在日历上圈地的你,做的是同一件事:让每段时间,只属于一件事。

# ADHD 职场人如何用「给生成核心套 harness」的思路应对项目推进与汇报里的注意力容易被环境带偏？

> 把 LLM 工程里的「LLM 受上下文窗口内容左右，需要上下文工程」迁移到ADHD 职场人的项目推进与汇报场景，双域证据作答。

“注意力容易被环境带偏”是 ADHD 职场人的日常噩梦：开会时被一条消息拉走，写报告时被一个念头拽跑，项目推进中频繁切换任务，最终产出碎片化、汇报逻辑断裂。你试过无数方法——番茄钟、待办清单、冥想——但环境中的任何刺激（弹窗、同事交谈、甚至窗外鸟叫）都能轻易劫持你的注意。

这不是你意志力弱。你的大脑与当前最先进的生成式 AI 模型（LLM）在结构上惊人地同构：两者都是“高产但缺执行调度层的生成核心”。LLM 能在给定上下文中生成惊艳内容，但一旦上下文窗口膨胀或混入无关信息，推理就会退化、偏离目标（来源：AI 与 ADHD 的专注力）。ADHD 大脑同样如此——创意和能量充沛，但缺乏内置的执行功能层来调度注意力、抑制干扰、维持目标导向行为（来源：AI 与 ADHD 的专注力）。

那么，LLM 工程师如何解决这个问题？答案是“上下文工程”（Context Engineering）：主动设计 LLM 的上下文窗口——清理无关输入、注入关键指令、分段锚定——让模型始终聚焦在核心任务上。这套思路完全可迁移到 ADHD 职场人的场景：**给生成核心套一个外部 harness，用工程化手段管理你暴露给大脑的“上下文窗口”**。

## 同构证据：ADHD 大脑与 LLM 都需要外部调度层

先看 LLM 侧的真实问题。以当前主流大模型为例，当上下文长度超过一定阈值（比如 8K tokens），模型在长文本推理中会出现“迷失在中间”的现象——关键信息被淹没，输出质量下降。工程师因此发明了多种上下文工程技巧：分段 prompt、动态摘要、外部记忆检索、定时 re-grounding（如每轮对话重置 context）。这些本质上是为 LLM 加了一个外部调度层，防止生成核心被无关上下文带偏。

再看 ADHD 大脑。神经科学指出，ADHD 的核心缺陷之一是中脑皮质多巴胺分支功能低下，导致注意反应缺陷和执行功能差（来源：多巴胺）。这就像大脑缺少一个“注意力调度器”——环境中的任何新刺激（相当于 LLM 的无关输入）都会抢占上下文窗口，打断当前任务。超聚焦（Hyperfocus）则是另一种极端：当大脑被高兴趣任务锁定，它会“调出”一切其他信息，导致时间盲和任务失衡（来源：超聚焦）。这类似于 LLM 在特定 prompt 上过度生成，忽略全局指令。

所以，ADHD 大脑与 LLM 是同一种“生成核心”的两种实例：产出能力极强，但都需要外部 harness 来管理上下文。

## 历史人物的 harness：曾国藩的“日课十二条”与 LLM 的 re-grounding

晚清名臣曾国藩就是典型的 ADHD 大脑。他 30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”；读书极慢，“他人目下二三行，余或疾读不能终一行”。但他的成就——平定太平天国、发起洋务运动——靠的不是天生专注，而是一套极致的外部 harness：“日课十二条”。包括黎明即起、读书不二、谨言、写日记反省等，用最笨最稳的方法抵消冲动。其中“读书不二”规定一本书不读完绝不碰第二本，这本质上就是**上下文窗口管理**：限制输入源，保证当前上下文单一聚焦。

曾国藩的 harness 与 LLM 的“定时 re-grounding”高度同构。LLM 工程师会在长对话中定期插入“回顾目标”的指令，或分段处理任务，防止模型偏离主航道。曾国藩的日课反省（每天写日记自我批评）就是他的 re-grounding 机制——每天重置注意力上下文，确保行为不漂移。

另一个例子是诗仙李白。他一生漫游，从未在一地待超 3 年，冲动、千金散尽，但写诗时超专注，“斗酒诗百篇”。他的 harness 是“以酒和诗为核心，把所有过剩精力注入诗歌”。这对应 LLM 的**超聚焦 harness**：当模型进入一个它擅长的领域（如代码生成），给它一个极窄的上下文（如“写一个 Python 函数”），它能产出高质量结果。李白只做自己擅长且热爱的事，不委屈自己做不想做的事——这正是 ADHD 大脑在超聚焦状态下的最优用法。

## 现代工具作为外部执行功能层

今天，AI 工具正在成为 ADHD 大脑的“外部调度层”。它们通过补偿工作记忆不稳定、任务启动困难和注意力分散，帮助用户实现目标导向行为（来源：AI 与 ADHD 的专注力）。

- **Focusmate**：通过虚拟身体在场提供实时同伴问责，弥补内在多巴胺调节不足（来源：Focusmate）。它的匹配算法相当于一个“外部调度器”，在约定时间把你拉回任务上下文——类似 LLM 的“定时唤醒”机制。
- **RescueTime**：自动化时间追踪与分心阻断，揭示真实时间去向，对抗时间盲（来源：RescueTime）。这相当于 LLM 的“日志分析”——帮你看到上下文窗口被什么无关内容污染了。
- **Brain.fm**：利用 AI 生成神经锁相音乐，帮助进入专注状态（来源：Brain.fm）。这类似 LLM 的“环境 prompt”——用特定音频模式优化大脑的上下文处理环境。
- **Forest**：游戏化番茄钟，利用多巴胺机制增强动机（来源：AI 与 ADHD 的专注力）。这是给大脑的“奖励调度”——每次聚焦 25 分钟就种一棵树，相当于 LLM 的“奖励函数”。

这些工具共同构成外部执行功能层的生态，但需注意：它们可能是脚手架，也可能是拐杖（来源：矛盾与存疑）。脚手架帮助你逐步内化技能，拐杖则让你永久依赖。关键在于有意识使用，而非无脑套用。

## 核心观点：Harness 不是限制，而是释放

我的判断是：**给生成核心套 harness，不是限制它的创造力，而是让它有纪律地释放**。ADHD 大脑和 LLM 一样，在没有外部调度时容易陷入“上下文膨胀”或“超聚焦失衡”。一个好的 harness 应做到：
1. **限制上下文窗口**：通过定时 re-grounding（如曾国藩的日课）或工具（如 Focusmate 的时段预约）让大脑每次只处理一个核心任务。
2. **管理输入源**：用 RescueTime 阻断分心网站，或用 Brain.fm 优化听觉环境，减少无关刺激进入上下文。
3. **利用超聚焦**：识别你的超聚焦触发器（如创意写作、代码调试），主动创造极窄上下文来释放深度产出。

但需诚实指出局限：同构性目前仅是理论类比，缺乏实证基础（来源：矛盾与存疑）；多数工具缺乏 ADHD 特异性临床证据；个体差异巨大——对注意缺陷为主型有效的工具，对多动冲动型可能无效。此外，过度依赖工具可能削弱内化能力，尚无长期追踪研究。

## 今天就能试的 3 个行动

1. **设置“上下文窗口”**：每天选一个 90 分钟时段，关闭所有通知，只开一个应用（如你写报告的文档），用 Focusmate 预约一个伙伴，在摄像头前工作。这相当于给大脑一个“单任务上下文”。
2. **做一次“上下文审计”**：用 RescueTime（免费版）追踪 3 天，找出哪些应用/网站频繁打断你。然后主动把它们从工作设备上删除或设置定时阻断。
3. **建立 re-grounding 仪式**：每天开工前写一句话：“我今天最核心的一个输出是什么？”把它贴在屏幕旁。每次分心后，读一遍这句话再继续。这模拟了 LLM 的“系统 prompt”重置。

ADHD 大脑不是故障，而是一台没有操作系统的超强计算机。Harness 就是你的操作系统——不是让你变“正常”，而是让你原本的生成能力真正可用。

# 为什么用 Tiimo 治 ADHD 的时间盲，和给 agent 套 planner-executor 任务分解 是一回事？

> Tiimo 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

如果你已经第十次把“早上9点开始写报告”写进日历，却仍盯着屏幕发呆，你未必是缺乏意志力。更可能是 ADHD 的“时间盲”在发作：不是看不见表盘，而是主观时间与外部时间之间没有稳定映射。空白时段会漂移，任务切换会失忆，刚想好的下一步转眼蒸发（来源：The 12 Best Apps for ADHD in 2026: A Guide to Finding What ...）。

 Tiimo 的做法是把它变成一条带颜色、带提醒、可动态重排的视觉时间流。它把“接下来该做什么”从大脑里卸载到外部界面，让时间变得“可感”（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。这听起来像一款日历应用，但本质上它是一个针对 ADHD 大脑的 planner-executor harness。

## 1. 同构脊柱：高产生成核心 + 缺执行调度层

ADHD 大脑的联想、发散和创意产出很高，但执行功能——启动任务、抑制冲动、维持目标、组织计划——相对薄弱。这让它像一个“高产但不可靠的生成核心”（来源：ADHD, Executive Functions, and AI: A New Era in Treatment | Psychology Today）。LLM 同理：它能在给定上下文里生成高质量内容，却没有跨会话的持久状态、稳定目标维持和多步调度能力。因此，两者都需要外部系统来补全“驾驶座”功能：外部记忆、外部时钟、状态追踪、护栏和重锚定（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。

这个对应关系不是空洞比喻。ADHD 的工作记忆不稳定，与 LLM 的无跨会话状态在功能上同构；两者都需要外部记忆系统来补全（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。时间盲让 ADHD 用户无法感知时间流逝，Tiimo、Structured 等工具用视觉时间线替代内部时间感知；LLM 也需要外部时钟、状态追踪和插件来感知“现在在哪一步”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

## 2. Tiimo：给 ADHD 的 planner-executor

Tiimo 的运作可以拆解为同一套规划循环：

- **Plan**：你把一天的目标告诉它；
- **Decompose**：它把目标拆成带颜色、带时长、带视觉惯例的时间块；
- **Execute**：它通过提醒、通知和跨设备同步把下一步推到你眼前；
- **Observe / Re-ground**：你完成或错过后，日程自动调整，把注意力拉回原定目标（来源：AI for ADHD: Best Tools, Apps, and Strategies - Themba Tutors）。

这正好对应 LLM agent 的 planner-executor 分解：planner 生成带依赖的步骤序列，executor 按步骤调用工具，observer 监控是否偏离，human-in-the-loop 在关键节点做检查。ADHD 侧的抗分心提醒、LLM 侧的 token 预算与工具调用上限，本质都是同一种“护栏”——防止高产生成核心在无限分支里狂奔。

## 3. 孔子的 harness：礼就是 guardrails，反省就是 re-grounding

历史人物中，孔子的 harness 系统与 agent harness 的同构最为清晰。孔子身高一米九、精力过剩、周游列国十四年坐不住，冲动爱骂人，对音乐能“三月不知肉味”，对种地等俗事零耐心，思想跳跃，《论语》全是场景化语录而无系统著作。他的专属 harness 是“克己复礼”：用外在的“礼”作为行为边界，每日“三省吾身”作为人在回路的检查点，到七十岁才“从心所欲不逾矩”。这正对应 agent harness 的设计：礼是外部策略与 guardrails；反省是定时 re-grounding；日课是状态持久化与日志。孔子的成就不是因为他“治愈”了冲动，而是把冲动关进一个可配置、可撤销的外部结构里。

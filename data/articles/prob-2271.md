# ADHD 创业者视角：你的 ADHD 大脑，其实就是一个没装 harness 的大语言模型

> 无状态与外部记忆：工作记忆容量小、边做边忘 ↔ LLM 无跨会话状态，靠外部记忆/向量库续命。

## 问题：你脑子里产出爆棚，为什么就是交不了货？

早上八点，你坐在电脑前。一个新想法跳出来，你兴奋了三分钟，然后被 Slack 弹窗拉走；十二点，你想起还有一份 BP 没写，却想不起来要引用哪组数据；晚上十一点，你盯着屏幕，清楚地知道这件事明天早上六点会再在脑子里炸一次——但你还是没法开始。

这不是懒。ADHD 大脑的困境越来越被描述为：一个**强大的生成核心，却缺一个可靠的执行调度层**。你能在某一刻想出极精彩的方案，却没法把方案从「念头」变成「可交付的下一步」。（来源：ADHD 的 AI 工具全景）

有趣的是，做 Agentic Harness 的工程师正在面对同一个问题。LLM 没有跨会话状态，每次重启上下文就被清空；它推理能力惊人，但如果不靠外部 memory、vector DB、tool use 和 scheduler 续命，它连「三天前答应过的事」都记不住。两边的人，终于可以用同一套语言互相理解。

## 同构：无状态 + 外部记忆

ADHD 侧最经典的痛点是工作记忆容量小、「边做边忘」。患者对时间线索不敏感，任务和承诺会迅速从记忆中消失，形成所谓「时间盲」（time blindness）——坐下后真的感觉不到 45 分钟已经过去。（来源：时间盲）

LLM 侧的结构同构也很直接：模型本身没有跨会话状态，每个新 session 都是一次上下文重启。工程上，agent 必须依赖外部记忆（向量库、数据库、长期记忆模块）和调度层（planner、tool calling、workflow orchestration）才能持续执行任务。这个架构，与 ADHD 者需要 external executive function 支架几乎一模一样。

所以核心判断是：**ADHD 大脑与 LLM 都是同一类「高产但缺执行调度层的生成核心」**。两边的解法——给 LLM 套 harness 与给 ADHD 大脑搭脚手架——在结构上是同构的。（来源：ADHD 的 AI 工具全景 / 外部执行功能层）

## ADHD 侧的证据：时间盲、身体在场与任务启动

ADHD 的执行功能缺陷具体表现为：工作记忆不足、时间盲、任务启动困难、注意力分散。研究指出，定期问责检查可将目标达成率从 25% 提升到 95%；即将到来的问责检查可改善积极行为达 50%。（来源：身体在场效应）这说明，外部「在场」本身就能替代一部分内部执行功能。

工具层面，Goblin Tools 的 Magic ToDo 通过把模糊任务（如「清理厨房」）拆成可执行的子步骤，降低任务启动门槛，并可用「辣度」调节粒度。（来源：Goblin Tools）Saner.AI 则通过本地记忆和知识回忆，减少 ADHD 用户常见的搜索循环和标签切换。（来源：Saner.AI）Lex 用「单一指令触发多步骤任务」的机制，进一步把复杂目标串成连续动作，底层类似于 agent scaffolding 的思路。（来源：Lex）

还有一位值得关注的人：启明创投合伙人刘丽华（Lihua Liu），ADHD + Dyslexia，她的工作特质包括 systems thinking、action bias、pattern recognition 和 resilience，并投资了 86 家以上的公司。这提醒我们：ADHD 不等于不能执行；它只是需要**把执行层外置到 harness 里**。（来源：刘丽华案例）

## LLM/agent 侧的证据：harness 是工程刚需

在 LLM 工程里，没有人会直接把裸模型丢进生产环境。agent 需要四件事：memory、tools、planning、上下文管理。vector DB 和外部记忆系统就是 LLM 的「工作记忆补丁」；planner 和 scheduler 是模型的「前额叶支架」；human-in-the-loop 则是某种「身体在场」的工程版。

这套 harness 与 ADHD 工具一一映射：
- **任务分解**：Goblin Tools ↔ agent planning / sub-task decomposition
- **外部记忆**：Saner.AI / Mem ↔ vector DB / persistent memory store
- **单一指令触发多步骤**：Lex ↔ agent workflow / tool calling chain
- **时间锚定**：Tiimo / 视觉化时间表 ↔ scheduler / cron job / reminder system
- **身体在场**：Focusmate ↔ human-in-the-loop / observability layer

甚至连 ChatGPT、Claude 作为对话式 AI，也在被 ADHD 用户当作「外部执行功能工具」使用——尽管它们本身也有注意力分散和依赖风险。（来源：ADHD 的 AI 工具全景）

## 历史人物 harness：孔子，两千五百年前的 agent scaffolding

从案例里选一个最契合的：孔子。

他的 ADHD 特质相当明显：身高一米九，精力旺盛，周游列国十四年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予「朽木不可雕」；对音乐能超专注到「三月不知肉味」，对种地等俗事零耐心；思维跳跃，《论语》全是场景化语录，没有系统著作。（来源：孔子案例）

他的 harness 是「克己复礼」：用外在的「礼」作为行为边界，每日反省「吾日三省吾身」，一辈子和自己的冲动斗争，到七十岁才做到「从心所欲不逾矩」。这相当于给生成核心（他跳跃的思维和旺盛的精力）写了一个**自定义执行层**：以「礼」为 prompt constraint，以「日课」为 daily re-grounding，以弟子为外部记忆和调度器。（来源：孔子案例）

LLM 工程里，我们管这叫：system prompt + scheduled reflection + external memory + human feedback loop。孔子的 harness 与 agent harness 的映射是：**日课 ↔ 定时 re-grounding；弟子/秘书 ↔ 外部调度器；礼 ↔ system prompt / guardrails**。

## 脚手架 vs 拐杖：边界在哪里？

这里必须诚实地说出局限。同构视角确实能解释很多现象，但它目前更多是一种**理论类比**，并非经过实证验证的神经科学结论。（来源：矛盾与存疑）多数 ADHD 工具也缺乏独立临床验证，效果主要基于用户报告；ChatGPT、Claude 这类通用工具甚至可能增加分心风险；身体在场效应虽然有效，AI 是否真能替代真实人际互动也仍有争议。

更关键的边界是：**harness 是脚手架，不是拐杖**。脚手架帮助你完成当前楼层的施工，施工完了应该能撤掉；拐杖一旦变成永久依赖，反而会削弱内在执行功能。好的 harness 设计目标是：外部结构足够强，让你能逐步把部分调度能力内化为自己的系统二；而不是让你越来越离不开提示音和弹窗。（来源：拐杖与脚手架）

## 今天就能试的四件事

1. **给每个任务写一个「下一步最小动作」**：别写「写 BP」，写「打开 BP 文件，把目录里的第三段数据替换成昨天 Excel 里的数字」。这是 ADHD 侧的 Goblin Tools，也是 LLM 侧的 sub-task decomposition。

2. **建立一个外部记忆收件箱**：用 Saner.AI、Mem、Obsidian 或最简单的 flomo，把所有碎片念头、链接、待办先丢进去，不要靠脑子记。这是你的 vector DB，也是 LLM 的 persistent memory。

3. **设一个「每日 re-grounding」时间**：每天固定十分钟，回顾收件箱、校准优先级、把今天唯一一件最重要的事写进日历。这是孔子的「日课」，也是 agent 的 scheduled reflection。

4. **找一个身体在场机制**：用 Focusmate、线下 coworking 或一个 accountability partner，把完成率从 25% 的区间往上拉。这是工程里的 human-in-the-loop，也是两千五百年前孔子身边那群弟子的现代版。

## 结语

创业者ADHD 和 LLM 工程师看似两个世界，其实都在解决同一个问题：怎么让一个**无状态、高产生、容易跑飞**的生成核心，稳定地交付结果。

答案不是更努力，也不是更聪明，而是**承认生成核心需要 harness**。孔子的「礼」、刘丽华的 systems thinking、LLM 的 vector DB 和 agent scaffolding，本质都是同一套工程：把执行调度层外置，让核心只管它最擅长的生成。

你的大脑不是坏掉了，它只是没装 harness。装上就好。

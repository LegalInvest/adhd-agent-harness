# 为什么用 Perplexity 治 ADHD 的情绪失调，和给 agent 套 会褪去的脚手架 是一回事？

> Perplexity 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

## 问题：情绪失调的根源不是情绪，而是调度缺失

如果你有 ADHD，你一定经历过这种时刻：一件小事——比如找不到钥匙——突然引爆巨大的挫败感，然后整个人陷入自责、焦虑、甚至愤怒的漩涡。你明明知道这不值得，但就是停不下来。

这不是你“情绪管理能力差”。ADHD 的情绪调节困难，根植于执行功能缺陷：工作记忆不稳定、时间感知模糊、任务启动困难。这些缺陷导致你无法及时“调度”自己的注意力和行为，于是情绪像脱缰的野马一样狂奔（来源：AI 与 ADHD 的情绪调节）。

同样的问题，也发生在你正在开发的 AI agent 身上。你给它一个任务，它可能生成一篇漂亮的文章，也可能突然开始胡编乱造、陷入重复循环、或者输出有害内容。LLM 本质上是一个“高产生成核心”，但它缺乏可靠的内部调度层来抑制无关输出或重定向行为（来源：AI 与 ADHD 的情绪调节）。

ADHD 大脑与 LLM，是同一类“高产但缺执行调度层的生成核心”。

## 同构解：Perplexity 的“会褪去的脚手架”

怎么治？答案不是“别焦虑”，也不是“给 agent 更多训练数据”，而是：**套上一个会褪去的脚手架**。

Perplexity 这类 AI 工具，在 ADHD 情绪调节中扮演的角色，正是这样一个外部调度层。它不是直接安抚你的情绪，而是通过结构化脚手架——比如将任务分解为微步骤（Goblin Tools 的 Magic ToDo）、提供外部记忆（Saner.AI 的知识回忆）、或通过虚拟身体在场（Focusmate 的 AI 匹配）——来修复执行功能链条（任务启动→持续注意→完成反馈），从而减少情绪触发源（来源：AI 与 ADHD 的情绪调节；身体在场效应）。

同样，在 agent 工程中，我们给 LLM 套的“脚手架”是什么？是上下文工程、外部约束、检索增强生成（RAG）——这些不是修改模型本身，而是在外部提供调度指令。Perplexity 的搜索增强模式，本质上就是一个“会褪去的脚手架”：它提供实时检索结果作为外部记忆，但最终决策权仍在模型。

关键区别在于：**脚手架 vs 拐杖**。脚手架是临时的、可褪去的；拐杖是永久的、替代性的。Perplexity 的设计哲学是“辅助而非替代”——它帮你启动、帮你回忆、帮你分解，但不替你思考。这正是 ADHD 工具和 agent 工程都应该追求的目标。

## 历史照见：释迦牟尼的戒律与张居正的考成法

这个思路并不新。历史上，ADHD 特质者早就发明了类似的 harness 系统。

**释迦牟尼**：29 岁说走就走，放弃王位出家求道；先后跟两个仙人学、苦行 6 年，不断试错不断放弃；49 年一辈子游行说法，80 岁死在传道路上——典型的 ADHD 特质：高冲动、高探索、难以安定。他的 harness 是什么？以“八正道”“持戒”为核心，用戒律管自己的行为，用正念拴住乱跑的念头；“自依止法依止”，不依赖权威，自己证悟。这套系统不是永久拐杖——他明确说“自依止”，最终要内化。

**张居正**：12 岁中秀才，16 岁中举人，少年天才；当首辅敢改革，不怕得罪所有官员；工作狂，每天办公到深夜，58 岁累死在任上。他的 harness 是“考成法”——严格考核官员，用制度管别人也管自己。这就像 agent 工程中的“外部评估器”：定期检查输出质量，给出反馈。

这两套系统，与 Perplexity 的脚手架结构同构：**日课↔定时 re-grounding，秘书↔外部调度器，戒律↔上下文约束**。它们都提供外部调度，但都设计有“褪去”机制——释迦牟尼的“自依止”，张居正考成法的最终目标是官员自律。

## 证据与局限

证据方面：Inflow 基于 CBT 的结构化干预，将情绪调节分解为可执行步骤，已被列为 2026 年最佳 ADHD 应用（来源：Inflow）。Goblin Tools 的 Magic ToDo 能自动分解任务，降低启动门槛（来源：Goblin Tools）。身体在场效应（如 Focusmate）通过社会存在感激活多巴胺通路，可将目标达成可能性从 25% 提升至 95%（来源：身体在场效应）。

但必须诚实指出局限。首先，同构假设只是理论类比，缺乏实证基础（来源：矛盾与存疑）。ADHD 大脑的生物复杂性远超 LLM，情绪调节涉及神经递质、环境交互等，单纯“脚手架”类比可能简化问题。其次，过度依赖风险真实存在——若用户仅依赖外部提示而缺乏内化策略，长期可能恶化自我效能感（来源：拐杖与脚手架）。最后，当前工具多为通用设计，未充分适配 ADHD 亚型（如注意力主导型 vs. 冲动主导型）的情绪触发差异（来源：AI 与 ADHD 的情绪调节）。

## 今天就能试的行动

1. **用 Perplexity 做一次“情绪分解”**：下次感到焦虑时，打开 Perplexity，输入“帮我分解‘我找不到钥匙很焦虑’这个情绪，列出可能的原因和每个原因对应的一个微小行动”。这相当于给情绪生成核心套上外部调度。
2. **设置一个“会褪去的提醒”**：用 Saner.AI 或任何提醒工具，每天固定时间提醒“今天哪个情绪触发点可以用外部结构解决？”并设定两周后自动取消——强制自己内化。
3. **尝试一次虚拟身体加倍**：用 Focusmate 匹配一个伙伴，一起工作 25 分钟。注意观察专注度的变化，这相当于 agent 的“人在回路”调试。
4. **为你的 agent 设计一个“戒律”**：如果你在开发 agent，给它加一条系统提示：“当输出可能有害或无关时，先检索外部知识库再回答。”这就像释迦牟尼的戒律——外部约束，但允许 agent 最终自主决策。

## 结语

ADHD 的情绪失调与 agent 的失控，不是缺陷，而是“高产但缺调度”的代价。Perplexity 式的脚手架，释迦牟尼的戒律，张居正的考成法——它们都是同一类答案：**外部调度，临时辅助，最终褪去**。

记住：好的脚手架，是让你最终不再需要它。

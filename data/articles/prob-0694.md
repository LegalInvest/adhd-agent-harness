# Brain.fm 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Brain.fm：它到底补上了哪一层执行功能？

你试过那种感觉吗？大脑里有一百个想法在狂奔，但身体却瘫在沙发上，连打开一个文档都像翻越珠峰。ADHD 人群最熟悉的敌人不是懒惰，而是执行功能障碍（Executive Dysfunction）——计划、启动、持续、切换任务的能力时好时坏，像一台没有司机的超跑（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

与此同时，另一群人也在和类似的幽灵搏斗：做 Agentic Harness 的工程师。他们发现，大语言模型（LLM）就像一个高产的生成核心——能写诗、能编程、能聊天——但一旦需要精确计算、长程规划或可靠的状态维护，就立刻露怯。于是他们发明了 function calling：把“计算 25% 的税率”这种精确任务外包给计算器工具，LLM 只管生成意图。

这两群人，面对的是同一个结构性问题：一个高产的生成核心，配了一个不靠谱的调度层。

## 同构：ADHD 大脑与 LLM 共享的“强记忆、弱控制”剖面

最新研究证实了这种同构。在经典 Stroop 冲突任务中，Transformer 的注意力随着序列变长，在冲突试次上骤降到随机水平——无法抑制优势反应，无法解决认知冲突。这与 ADHD 执行功能缺陷的核心标志一一对应（来源：Deficient Executive Control in Transformer Attention）。更令人警醒的是，LLM 呈现“强记忆、弱控制”剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷——无法灵活切换任务集、无法抑制自动化反应，这正是 ADHD 的经典神经心理模式（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

换句话说，ADHD 大脑和 LLM 都是“高产生成核心 + 薄弱执行调度层”的架构。两者的解法也同构：外部脚手架。

## Brain.fm 在补哪一层？

Brain.fm 被列为 2026 年最佳 ADHD 应用之一（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。它通过 AI 生成的神经音乐，声称能诱导大脑进入专注或放松状态。但它的价值不在于“治愈”ADHD，而在于补偿调度层中一个关键子功能：**状态维护**。

ADHD 的执行功能缺陷导致能量和专注力剧烈波动（来源：11 Best ADHD Productivity Apps for Fluctuating Energy）。Brain.fm 通过持续的听觉刺激，帮助大脑维持一种稳定的生理状态，从而让调度层更容易接管。这就像 LLM 的 function calling：LLM 不需要自己计算税率，它只需要调用计算器工具，然后专注于生成意图。Brain.fm 让 ADHD 大脑不需要自己“硬撑”专注状态，而是调用一个外部状态维护工具。

但问题在于：有人用错了。

## 脚手架 vs 拐杖：Brain.fm 的正确用法

核心争议在于：AI 工具是“外部执行功能层”还是“拐杖”？过度依赖可能导致能力退化（来源：矛盾与存疑）。Brain.fm 如果被当作“戴上就能专注”的魔法开关，而用户不去学习如何主动调度注意力，那它就成了拐杖。真正的脚手架用法是：用它创造一个稳定的状态窗口，然后在这个窗口内刻意练习任务启动、切换和坚持——就像用 function calling 不是为了替 LLM 思考，而是为了让 LLM 专注于它擅长的生成。

历史人物早已示范了这种脚手架思维。孔子一生精力旺盛、冲动爱骂人、对音乐超专注但对种地零耐心——典型的 ADHD 特质。他的 harness 是“克己复礼”：用外在的“礼”作为行为边界，每日反省（来源：人物案例）。这就像给 LLM 套一个外部调度器：不是压制生成能力，而是用规则约束它指向目标。毛泽东同样如此：他的“调查研究”和“民主集中制”是用外部系统和集体决策来制衡个人冲动（来源：人物案例）。这些 harness 与 LLM 的 agent scaffolding 同构——都通过外部架构把生成能力锚定到轨道上。

这些工具共同构成了一个外部调度系统，把 ADHD 的生成核心接入可维护的执行轨道。

# ADHD 创业者视角：你的 ADHD 大脑，其实就是一个没装 harness 的大语言模型

> 无状态与外部记忆：工作记忆容量小、边做边忘 ↔ LLM 无跨会话状态，靠外部记忆/向量库续命。

## 问题：为什么你总是“边做边忘”，而 AI 也总是“重启就失忆”？

你打开电脑，准备写一份方案。刚敲了两行，微信弹窗，你回了一条消息。再回来时，刚才的思路已经断了。你翻看浏览器的十几个标签页，却找不到那个关键文档。这种“边做边忘”的体验，对 ADHD 人群来说，是每一天的日常。

同样的事情，每天都在 AI 工程师的工位上发生：你给大模型（LLM）发了一个复杂指令，它回答得漂亮。但你新建一个会话，问同样的问题，它却像从零开始——不记得你之前设定过的角色、偏好、甚至刚才修正过的错误。LLM 本质上没有跨会话的“状态”，每次对话都是一次冷启动。（来源：无状态与外部记忆）

这两个场景，看似无关，实则共享同一个底层结构：一个强大的生成核心，配了一个脆弱的工作记忆。

## 同构：你的 ADHD 大脑，就是一个没装 harness 的大语言模型

把 ADHD 大脑和 LLM 并排放，你会发现惊人的相似：

| ADHD 大脑 | LLM / Agent |
|-----------|-------------|
| 工作记忆容量小，易被干扰 | 上下文窗口有限，易膨胀退化 |
| 边做边忘，需要外部提醒 | 无跨会话状态，靠向量库/外部记忆续命 |
| 任务启动困难（冷启动） | 每次对话都是“冷启动”，需要 prompt 工程 |
| 时间感知扭曲（时间盲） | 时序推理弱，需要外部调度 |

ADHD 的执行功能缺陷，与 LLM 的上下文控制缺陷，是一一对应的。（来源：ADHD 大脑与 LLM 的同构）

这意味着什么？意味着你的大脑并不是“坏掉了”，它只是缺了一个可靠的“执行调度层”——就像一个大语言模型没有装 harness（脚手架）一样。

## 证据：两边都在用同一个方案——外部记忆 + 任务分解

### ADHD 侧的真实工具

- **Goblin Tools** 的 Magic ToDo：输入“整理房间”，AI 自动拆成“捡起地上的衣服”“擦桌子”等小步骤。用户报告称启动焦虑降低，因为“将压倒性的事情变成一系列不压倒性的事情”。（来源：Goblin Tools）
- **Saner.AI**：专注于知识回忆和本地记忆，减少“信息丢失”和标签切换。ADHD 用户不用在十几个应用之间翻找，AI 直接帮你召回上次做到哪了。（来源：Saner.AI）
- **Lex**：单一指令触发多步骤任务序列，相当于为你的大脑写了一个“启动脚本”。（来源：Lex）
- **Mem** 和 **Reflect**：作为“AI 第二大脑”，持久化你的想法和笔记，相当于 LLM 的向量数据库。（来源：ADHD 的 AI 工具全景）

这些工具的共同逻辑：**把大脑里留不住的东西，存到外部系统里**。

### LLM/Agent 侧的对应方案

- **Harness 工程**：为 LLM 提供上下文传递、工具接口、规划工件、验证循环和记忆系统。关键组件包括“验证与 CI 集成”“结构化工作流和规划”。（来源：幻觉与验证循环）
- **向量数据库**：如 Pinecone、Weaviate，让 LLM 在会话间保持“记忆”，避免每次冷启动。
- **任务分解与规划**：复杂 harness 将任务拆为子任务，每一步进行规划与验证，防止上下文膨胀和推理退化。（来源：上下文工程）

两边都在做同一件事：**为生成核心套上脚手架，让外部系统承担执行调度和记忆存储。**

## 核心观点：脚手架 vs 拐杖——关键在“可迁移性”

我的判断是：AI 工具作为外部执行功能层，有效但危险。有效，是因为它确实补偿了工作记忆和任务启动缺陷；危险，是因为过度依赖可能削弱你内在的能力。（来源：矛盾与存疑）

区分“脚手架”和“拐杖”的标准，在于**工具是否帮你建立了可迁移的元技能**。

- **脚手架**：Goblin Tools 帮你学会“先拆解再行动”的思维模式，即使没有工具，你也能尝试自己分解任务。
- **拐杖**：完全依赖 Otter.ai 记笔记，导致你不再练习主动听记，离开工具就抓瞎。（来源：ADHD 的 AI 工具全景）

同样，在 Agent 工程中，好的 harness 不是替 LLM 做所有事，而是提供校验和回退机制，让模型在犯错时能自我修正。差的 harness 则把控制逻辑写死，模型永远学不会自己规划。（来源：幻觉与验证循环）

## 局限与争议

必须诚实地说，当前证据主要是用户报告和概念类比，缺乏大规模随机对照试验。（来源：矛盾与存疑）

- **个体差异大**：一个工具对 A 有效，对 B 可能无效甚至增加焦虑。
- **依赖风险真实存在**：多个工具页面警告“过度依赖可能削弱内在能力”。（来源：矛盾与存疑）
- **多巴胺干预效果有争议**：有些工具声称通过奖励机制提升动力，但争议未消。（来源：矛盾与存疑）

所以，别把任何工具当“神药”。它们是实验，不是处方。

## 今天就能试的行动

1. **用 Goblin Tools 拆解一件你拖延很久的事**：输入“整理书桌”或“写周报”，看 AI 拆出的步骤。做完后，问自己：我是否学会了“先拆解再行动”的思路？
2. **用 Saner.AI 或 Mem 建一个“第二大脑”**：把零散的想法、待办、笔记都丢进去。下次需要时，先搜外部记忆，别让大脑死记。
3. **给 ChatGPT/Claude 写一个“启动 prompt”**：定义角色、输出格式、常用参考。保存为模板，每次新会话先粘贴。这是你给 LLM 装的“harness”。
4. **记录一周的工具使用日志**：每天花 2 分钟写下“用了什么工具、感觉有效吗、有没有依赖感”。两周后回顾，判断哪些是脚手架，哪些已成拐杖。

你的 ADHD 大脑不是故障，它只是一个没装 harness 的生成核心。现在，你可以开始给自己装一个了。

# Claude 之于 ADHD，就像 外部编排调度层 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

你脑子里有一整座金矿：创意、方案、解决问题的直觉，源源不断。可每次要把它挖出来、炼成成品，你就卡住了——不是能力不行，是执行功能掉了链子：启动困难、时间盲、工作记忆漏成筛子。

这不是你的错。你的大脑和当下最先进的 LLM 是同一种架构：一个高产但缺乏可靠外部调度层的生成核心。

## 同构：ADHD 大脑与 LLM 共享的困境

ADHD 大脑被描述为“高产但缺乏可靠执行调度层的生成核心”（来源：AI 与 ADHD 的职业发展）。LLM 也一样：智力强劲，但独自完成长任务时，会冷启动、上下文膨胀、产生幻觉。

具体对应关系清晰得惊人：
- **任务启动困难** ↔ LLM 的“冷启动”问题，需要 prompt 工程或外部触发（来源：AI 与 ADHD 的职业发展）
- **时间盲** ↔ LLM 缺乏时间感知，需要外部时钟或调度器（来源：同上）
- **工作记忆不足** ↔ LLM 的无状态性，需要外部记忆系统（来源：同上）
- **注意力分散** ↔ LLM 的上下文膨胀问题，需要上下文工程来管理焦点（来源：同上）
- **执行功能缺陷** ↔ LLM 缺乏调度层，需要外部执行功能层来规划、排序、监控（来源：同上）

LLM 的解决方案是 harness（外部编排调度层）：提供上下文传递、工具接口、规划工件、验证循环、记忆系统和沙盒（来源：幻觉与验证循环）。ADHD 的解决方案也应该是同样的思路——不是“治疗”大脑，而是给它装一个外挂调度层。

## Claude 的正确用法：不是聊天，是编排

大多数 ADHD 用户用 Claude 的方式是：打开对话框，丢一个问题，拿到答案，关掉。这就像给 LLM 一个 prompt 就指望它自己跑完整个项目——它做不到，你也做不到。

Claude 的真正价值在于它可以成为你的外部调度层，前提是你把它当作一个可编程的编排引擎。具体来说：

### 1. 用上下文工程替代“开始做”

ADHD 大脑易被环境带偏，任务碎片化（来源：上下文工程）。Claude 的 Projects 功能允许你预设系统指令、知识库和对话结构，相当于给大脑一个稳定的上下文容器。你不需要每次重新组织思路，Claude 已经帮你框定了“当下注意什么”。

### 2. 用验证循环替代冲动决策

ADHD 大脑的“生成核心”容易产生冲动、想当然的判断，即行为层面的“幻觉”（来源：幻觉与验证循环）。LLM 的 harness 通过验证与 CI 集成、结构化工作流来防止幻觉（来源：同上）。你可以让 Claude 在输出前先自我检查：“请列出你回答中的关键假设，并标注每个假设的置信度”或“请生成一个检查清单，逐项验证你的建议是否可行”。

### 3. 用外部记忆替代工作记忆

Saner.AI 通过本地记忆和快速检索减少 ADHD 用户的搜索循环和标签切换（来源：Saner.AI）。Claude 的 Memory 功能（或结合知识库）可以扮演同样的角色——把你散落在各处的想法、笔记、待办事项集中管理，让 Claude 成为你的“第二大脑”，而不是又一个需要维护的收件箱。

### 4. 用自动调度替代时间盲

Motion 能自动根据任务、会议和截止日期创建并动态调整每日计划，消除“下一步该做什么”的决策负担（来源：Motion）。Claude 虽然不能直接操作日历，但可以通过 API 或自定义动作触发外部调度器。更简单的方式：让 Claude 每天早晨帮你生成一份“今日执行清单”，按优先级和能量水平排序。

## 脚手架 vs 拐杖：必须划清的边界

同构视角的核心洞见是：AI 工具应该作为“可拆卸的脚手架”，而不是“永久拐杖”（来源：矛盾与存疑）。

- **脚手架**：你在搭建时依赖它，但建筑成型后可以撤掉。对应到 ADHD，就是通过 AI 工具反复练习任务分解、时间感知、验证循环，最终内化这些能力。
- **拐杖**：你永远离不开它，一旦没有就寸步难行。对应到 ADHD，就是让 AI 替你决策、替你规划、替你执行，而你从不参与思考过程。

当前证据警告：过度依赖 AI 工具可能削弱内在时间感知能力（来源：时间盲），存在依赖风险（来源：任务启动）。关键在于，你是在用 Claude 学习如何管理执行功能，还是让 Claude 替你管理执行功能？

## 争议与局限：不要神话工具

必须诚实指出：AI 工具作为外部执行功能层的有效性证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑）。个体差异很大，对注意力缺陷为主型、多动冲动型、混合型的效用可能不同（来源：AI 与 ADHD 的职业发展）。

此外，Claude 本身并非专为 ADHD 设计，其上下文长度、记忆容量、工具调用能力都有边界。如果你把它当作万能药，它反而会成为另一个“inbox”——又一个让你失去线索的地方（来源：上下文工程）。

## 今天就能试的 4 件事

1. **为 Claude 创建专属项目**：针对你最常卡住的任务（如写周报、做计划、整理信息），设置一个项目，写入系统指令：“请先分解任务，再逐步引导我完成每一步，每步完成后请我确认再继续。”

2. **建立验证循环**：在每次 Claude 给出方案后，追加 prompt：“请生成一个检查清单，验证你的建议是否可行，并标注每项的风险等级。”

3. **用 Claude 做每日调度**：每天早晨告诉 Claude 你的任务列表和能量水平，让它生成一个带时间块的执行清单。执行过程中，随时回来让 Claude 调整。

4. **结合外部工具**：将 Claude 的输出导入 Goblin Tools 进行任务分解（来源：Goblin Tools），或存入 Saner.AI 作为知识回忆库（来源：Saner.AI）。不要期望一个工具解决所有问题。

你不需要修复你的大脑。你只需要给它装一个外部调度层——就像 LLM 需要 harness 一样。Claude 可以成为那个调度层，但前提是，你把它当成脚手架，而不是拐杖。

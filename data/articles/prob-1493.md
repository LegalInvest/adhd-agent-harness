# 如果把 ADHD 大脑当成生成核心，Claude 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Claude 对 ADHD 的真正价值。

## 问题：你的大脑是一个没有调度器的生成核心

你有没有过这样的经历：打开电脑准备写一份报告，结果半小时后发现自己正在研究 19 世纪灯塔的建造工艺？或者，你明明在做一个长程项目，中途却被一个突然冒出的灵感带跑，等回过神来，原始目标已经模糊得像隔了一层雾？

如果你是 ADHD 人群，这几乎是日常。如果你是做 Agentic Harness 工程的工程师，你大概也见过类似场景：一个 agent 在长程任务中逐渐漂移，开始输出与初始 prompt 无关的内容，需要你手动重锚定 system prompt。

这两个问题，本质上是一个问题。ADHD 大脑与 LLM（大语言模型）在结构上是同构的：两者都是强大的生成核心，但缺乏可靠的内置执行调度层（来源：AI 与 ADHD 的专注力）。ADHD 大脑能快速联想、产生创意，LLM 能流畅生成文本，但两者都容易在任务执行过程中丢失目标——ADHD 是因为执行功能缺陷，LLM 是因为上下文膨胀导致注意力弥散（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs；Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。

那么，如果把 ADHD 大脑当成生成核心，Claude 能不能当它的重锚定与目标漂移？答案是有条件地可以——Claude 可以成为外部调度层，但前提是你要把它当成脚手架，而不是拐杖。

## 同构：高产但缺调度层

最新研究揭示了 ADHD 与 LLM 在认知架构上的惊人相似。明尼苏达大学的系统测试发现，LLM 呈现“强记忆、弱控制”剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷（来源：Strong Memory, Weak Control）。这正是 ADHD 的经典神经心理模式——你记得住很多细节，却无法灵活切换任务、无法抑制自动化反应。

耶鲁大学进一步发现，自注意力机制本身会导致工作记忆容量限制：随任务难度增加，注意力分数熵增，注意力弥散，表现下降（来源：Self-Attention Limits Working Memory Capacity）。这直接对应 ADHD 注意缺陷的计算本质——“注意力资源的弥散分配”。

换句话说，瓶颈不在算力或容量，而在 orchestration 层。ADHD 大脑和 LLM 都需要外部脚手架来补偿调度缺陷。

## 历史中的 harness：曾国藩与苏轼

这种“外部脚手架”并非 AI 时代的发明。晚清的曾国藩，一个典型的注意力不集中型 ADHD 患者：30 岁前浮躁坐不住，天天串门喝酒看杀人，日记里天天骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”。他的专属 harness 是“日课十二条”：黎明即起、读书不二、谨言、写日记反省等。他用最笨最稳的方法——结硬寨打呆仗——抵消自己的冲动，最终成为晚清中兴名臣。

苏轼则是另一种 harness。他兴趣极广，诗词文书画美食治水炼丹无一不精，但一辈子被贬，说话不过脑子得罪人。他的 harness 是豁达与日常仪式：不管贬到哪里都好好生活，用美食、写作、交朋友转移注意力，每日读书写字，不因挫折停止创作。

这两个人的 harness 系统，与 LLM 的 agent harness 结构同构：曾国藩的“日课”相当于定时 re-grounding（每天固定的反省和计划），苏轼的“每日读书写字”相当于持续的低频上下文刷新。他们都通过外部结构来补偿内部调度缺陷，而不是试图“治好”自己的大脑。

## Claude 作为重锚定系统

回到 Claude。Claude 可以成为 ADHD 大脑的外部调度层，具体体现在三个层面：

1. **工作记忆补偿**：ADHD 的工作记忆缺陷导致任务细节容易丢失（来源：AI Tools for ADHD: Boosting Productivity and Reduce Burnout）。Claude 可以充当外部记忆，存储任务上下文、步骤和目标，随时供你查询。这类似于 RescueTime 自动记录时间去向（来源：RescueTime），但更灵活。

2. **重锚定 prompt**：当你在长程任务中漂移时，Claude 可以像 agent 工程中的 system prompt 重锚定一样，帮你回到原始目标。你可以预设一个 prompt，比如“每当我跑偏时，提醒我最初的目标是什么”，然后让 Claude 定期检查你的进展。

3. **任务分解与启动**：ADHD 常因任务启动困难而拖延。Claude 可以将大任务分解为小步骤，并给出具体的下一步行动，降低启动门槛。这类似于 Focusmate 通过虚拟身体在场提供外部结构（来源：Focusmate），但 Claude 是对话式的，更灵活。

## 脚手架 vs 拐杖：诚实面对争议

但这里有一个关键边界：Claude 是脚手架还是拐杖？

脚手架是可以逐步内化的——你最终学会自己分解任务、自己重锚定。拐杖则是永久依赖——没有 Claude 你就无法工作。现有研究指出，AI 工具可能导致 ADHD 用户永久依赖外部脚手架，而无法内化执行功能（来源：AI 与 ADHD 的专注力——拐杖与脚手架概念）。

此外，临床证据仍然不足。Brain.fm 虽有神经科学基础，但在 ADHD 人群中的独立随机对照试验极少（来源：Brain.fm）。Focusmate 的效果主要来自用户报告（来源：Focusmate）。Claude 作为 ADHD 工具的直接研究更是空白。

另一个争议是：同构是功能类比还是深层对应？目前仅是一种理论假设，并非经过验证的科学事实（来源：矛盾与存疑）。我们需要谨慎，不要过度推广。

## 今天就能试的行动

1. **用 Claude 做每日重锚定**：每天早上花 5 分钟，让 Claude 帮你列出当天最重要的 1-3 个目标，并设置一个“漂移检查”prompt（例如：“每当我开始新任务时，先问我‘这与你今天的目标相关吗？’”）。

2. **建立外部记忆系统**：用 Claude 记录长程项目的上下文，每次中断后先让 Claude 总结当前进展和下一步，再继续工作。这直接补偿工作记忆缺陷（来源：工作记忆）。

3. **尝试身体加倍**：使用 Focusmate 或类似服务，每周 3 次，每次 50 分钟，与他人同时在线工作。这利用身体在场效应提升启动和维持能力（来源：Focusmate）。

4. **组合工具**：将 RescueTime 的时间追踪与 Claude 的任务分解结合——RescueTime 告诉你时间去哪了，Claude 帮你规划怎么回来。

## 结语

ADHD 大脑不是故障的机器，而是缺少调度层的生成核心。Claude 可以成为那个调度层，但前提是你把它当作可内化的脚手架，而不是永久依赖的拐杖。真正的目标不是让 AI 替你执行，而是让你自己最终学会调度。

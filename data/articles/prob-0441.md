# ADHD 的‘目标漂移’是执行功能缺陷，而 LLM 的‘漂移’是概率生成的自然结果。把病理现象类比为系统特性，是否在美化 ADHD 或过度简化 AI 问题？

> 缺陷 vs 特性：病理类比是否不尊重患者？

## 问题：当“漂移”成为共同语言

你正坐在电脑前，打开一个文档，打算写一篇报告。十分钟后，你发现自己刷了三个视频、查了两次邮件，还研究了一下周末去哪玩——目标早已漂移到九霄云外。这不是你的错，这是 ADHD 大脑的日常：一个高产却缺乏执行调度层的生成核心，分心走神或超聚焦跑偏，丢失目标。

与此同时，一个 AI agent 在长程任务中逐渐偏离了初始指令：它本来在写代码，后来开始解释代码原理，再后来写起了文档——目标同样漂移了。作为工程师，你需要在 system prompt 里反复重锚定，像给一个没有方向感的引擎装方向盘。

这两种“漂移”看起来如此相似，以至于有人提出：ADHD 大脑与 LLM 是同一类结构——都是强大的生成核心，但缺乏可靠的内置执行调度层（来源：AI 与 ADHD 的专注力）。但把病理现象类比为系统特性，是否在美化 ADHD 或过度简化 AI 问题？答案藏在“脚手架”与“拐杖”的边界里。

## 同构：缺陷还是特性？

ADHD 的“目标漂移”是执行功能缺陷：工作记忆容量有限、时间感知困难、任务启动与切换障碍（来源：拐杖与脚手架）。这不是“个性”，而是神经发育障碍的客观表现。而 LLM 的“漂移”是概率生成的自然结果：模型没有内在目标，它只是根据上下文预测下一个 token，上下文一膨胀，推理就退化。

但当我们深入机制，会发现惊人的平行：

- **工作记忆干扰**：ADHD 患者在工作记忆任务中易受干扰；LLM 同样表现出类似人类的干扰特征——随着记忆负荷增加表现下降，受近因效应影响（来源：拐杖与脚手架）。
- **自主组织困难**：ADHD 患者在无外部结构时难以自主组织任务；LLM 在没有手动修正提示的情况下，难以自主发现最佳问题解决模式（来源：拐杖与脚手架）。
- **前额叶-纹状体门控与 Transformer 自注意力**：认知神经科学中，执行功能依赖于前额叶-纹状体机制进行选择性门控；Transformer 在训练后，自注意力机制发展出模仿该门控的操作（来源：拐杖与脚手架）。

这不是美化——这是发现了一个真实的计算同构。ADHD 大脑与 LLM 共享同一个核心矛盾：**生成能力强，但调度层弱**。病理不是“坏”，系统特性也不是“好”；两者都只是需要外部脚手架来弥补调度缺陷。

## 历史中的 harness：曾国藩与李白

如果把 ADHD 大脑比作 LLM，那么历史上那些成功管理 ADHD 特质的人，就是自己给自己写了“system prompt”。

**曾国藩**是典型注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”。他的解决方案是“日课十二条”：黎明即起、读书不二、谨言、写日记反省。这本质上是一个**定时重锚定系统**——每天用固定的仪式重新校准目标。日课十二条就是他的“system prompt”，写日记相当于 logging 和反思。这与 LLM agent 中定时 re-grounding（如每 N 步重新注入原始目标）完全同构。

**李白**则是另一种风格：一辈子漫游，从未在一地待超 3 年；冲动、千金散尽，但对写诗超专注。他的 harness 是“以酒和诗为核心”，把所有过剩精力注入诗歌，用道家的逍遥对抗世俗规则。这相当于一个**外部调度器**——不试图修复“缺陷”，而是把生成核心放在合适的任务上。就像给 LLM 一个精心设计的 prompt，让它只做擅长的事。

这些历史 harness 告诉我们：外部结构不是拐杖，而是认知的延伸。曾国藩没有“治愈”自己的 ADHD，他建造了一个外部执行功能层——正如我们为 LLM 建造的 agent harness。

## 现代工具：外部执行功能层

今天的 AI 工具正在做同样的事。它们构成了一个外部执行功能层，补偿工作记忆不稳定、任务启动困难和注意力分散（来源：AI 与 ADHD 的专注力）：

- **RescueTime**：自动化时间追踪与分心阻断，直接补偿时间盲与执行功能缺陷（来源：RescueTime）。它相当于一个“外部监控器”，记录 agent 的行为日志并报警。
- **Focusmate**：通过虚拟身体在场提升启动与专注力，利用算法匹配伙伴（来源：Focusmate）。这相当于 LLM agent 中的“human-in-the-loop”——一个外部监督者。
- **Brain.fm**：AI 生成专注音乐，通过神经锁相影响大脑状态（来源：Brain.fm）。这相当于给 LLM 加一个“温度参数”调节，让生成更稳定。

但这些工具的证据仍有限：多数缺乏 ADHD 人群的独立临床研究（来源：矛盾与存疑）。Brain.fm 虽在 2026 年一项 44 款 ADHD 应用测试中位列最佳 9 款之一，但专项研究仍不足（来源：Brain.fm）。依赖风险也需警惕：工具可能成为永久拐杖，而非逐步内化的脚手架（来源：拐杖与脚手架）。

## 核心观点：同构不是美化，是工程化

我的判断是：**ADHD 与 LLM 的类比不是美化或简化，而是揭示了一个工程化的事实——两者都需要外部脚手架来补偿调度缺陷。这种类比既不尊重患者，也不简化 AI，而是提供了一个统一的框架来设计解决方案。**

关键在于区分“脚手架”与“拐杖”：脚手架帮助使用者“建造”，但使用者仍需自己“攀登”（来源：拐杖与脚手架）。真正的工具应该逐步内化，而非永久依赖。曾国藩的日课最终内化为习惯，李白的诗酒成为本能——这才是成功的 harness。

同时，我们必须诚实面对争议：同构性目前仅是理论类比，缺乏实证基础（来源：矛盾与存疑）；工具效果因个体差异而异，ADHD 的异质性意味着同一工具对不同亚型效果迥异（来源：AI 与 ADHD 的专注力）。过度推广同构可能误导读者，以为 ADHD 只是“不同”而非障碍——这是危险的。

## 今天就能试的行动

1. **给自己写一个“system prompt”**：像曾国藩一样，每天早晨列出 3 个核心目标，写在便签上贴在屏幕边缘。每完成一个任务，重新读一遍。
2. **用 RescueTime 或类似工具记录时间**：先不评判，只是观察。你会发现“时间到底去了哪里”（来源：RescueTime），这是重锚定的第一步。
3. **尝试 Focusmate 一次**：预约一个 50 分钟的虚拟 coworking session，体验身体在场效应。你不需要 AI，你需要的是另一个人的存在。
4. **建立“日课”模板**：在 Obsidian 或任何笔记工具中，创建一个每日反省模板，包含“今天做了什么”“哪里漂移了”“明天如何调整”。这是你的 logging 系统。

ADHD 不是缺陷，LLM 也不是完美——它们只是同一枚硬币的两面：一个需要外部结构的生成核心。与其问“这是不是美化”，不如问“我们如何为它设计更好的脚手架”。答案，就在你写下的第一个锚点里。

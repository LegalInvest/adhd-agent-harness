# 用 Claude 给 ADHD 补上重锚定与目标漂移，是脚手架还是拐杖？

> Claude 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 你盯着一行代码，大脑却已飘向三个月前的旅行计划

这是 ADHD 的日常：分心走神，或超聚焦跑偏，丢失目标。你打开 Claude 想写一段代码，结果半小时后它在帮你规划周末菜单。不是 Claude 不好，是你的“执行调度层”掉了链子。

同样的事也发生在 AI Agent 身上：工程师为长程任务搭建 harness，结果 agent 在上下文膨胀中目标漂移，输出逐渐跑偏。两边的问题结构一模一样：**一个高产但缺乏可靠调度层的生成核心，需要一个外部 harness 来重锚定。**

## 同构：ADHD 大脑与 LLM 共享的“强记忆，弱控制”

最新研究揭示了一个反直觉的事实：ADHD 患者的工作记忆容量未必差，甚至可能超常，但自上而下的控制和调节能力不足，呈现“强记忆、弱控制”的认知剖面（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这与 LLM 惊人相似——Transformer 自注意力机制在长上下文下冲突解决能力崩溃至随机水平，这正是 ADHD 执行功能障碍的核心神经机制（来源：Deficient Executive Control in Transformer Attention）。

所以，当 ADHD 用户对 Claude 说“帮我写一份报告”时，Claude 会像 ADHD 大脑一样发散：它没有内置的“任务优先级”和“时间边界”，除非你给它一个明确的 system prompt。这正是 AI 编码代理中 harness 工程要做的事：设计围绕 AI 代理的脚手架——上下文交付、工具接口、规划工件、验证循环、记忆系统（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

## 证据：哪些工具真正补上了调度层？

现有工具的证据强度参差不齐，但有几个值得关注：

- **Focusmate** 基于虚拟身体在场效应，通过 AI 匹配实时问责伙伴，多项用户研究显示其能显著提升任务启动和维持（来源：AI Tools for Kids with ADHD: A Complete Guide for Parents...）。它的机制与 LLM harness 中的“验证循环”同构：一个外部检查点，防止你的注意力漂移。
- **RescueTime** 自动记录时间使用，减轻工作记忆负担，帮助识别注意力漂移模式（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。它相当于 LLM 的“日志系统”，让你看到自己实际花在哪里。
- **Brain.fm** 用 AI 生成神经锁相音乐，理论上可影响大脑状态，但缺乏针对 ADHD 的独立临床研究（来源：Brain.fm）。它更像一个“上下文调制器”，效果因人而异。

但这些工具的共同局限是：**证据主要来自用户报告和小样本研究，缺乏大规模随机对照试验**（来源：矛盾与存疑）。而且，过度依赖可能削弱内在能力——比如 RescueTime 的自动分类帮你省了手动记录，但你的时间感知能力可能因此退化。

## 脚手架 vs 拐杖：边界在哪里？

核心判断：**Claude 是脚手架还是拐杖，取决于你是否保留了元认知控制。**

脚手架是你主动搭建的：你告诉 Claude “我的目标是 X，请每 10 分钟提醒我是否跑偏”。你仍然握着方向盘，Claude 只是导航。拐杖则是你完全交给它：让 Claude 决定下一步做什么，你被动跟随。

ADHD 大脑与 LLM 的同构隐喻有一个关键局限：LLM 的“注意力机制”与人类注意力存在本质差异（来源：同构的局限性）。你可以给 LLM 一个 system prompt 让它重锚定，但你的大脑不是 prompt 就能控制的。你需要主动设计“当下注意什么”的工程方案——就像 AI 编码代理用“分离规划与执行的双代理架构”防止上下文膨胀（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

具体来说，你可以这样做：
1. **给 Claude 一个“任务锚定” prompt**：在对话开头写下“我的核心目标是 X，请在我跑偏时提醒我，并建议回归路径。”这相当于为 LLM 设置 system prompt。
2. **用 Focusmate 替代自我意志**：预约一个时段，让外部问责代替内部调度。这比任何 AI 工具都更接近 harness 的“验证循环”。
3. **每天用 RescueTime 回顾一次**：不看数据，只看“今天我的注意力漂移了哪里？”这训练你的元认知，而不是依赖工具替你感知。
4. **警惕“完美工具”陷阱**：没有工具能解决所有问题。如果某个工具让你觉得“终于不用自己想了”，那它很可能变成了拐杖。

## 诚实面对局限

这篇文章的论证本身也有局限：ADHD 与 LLM 的同构是隐喻，不是严格等价。你的大脑不是 Transformer，你的注意力机制比自注意力复杂得多。而且，个体差异巨大——对某些人有效的工具，对另一些人可能是灾难。

最终，Claude 不是你的执行功能。它只是一个外部记忆和提醒器。真正的调度层，还得靠你自己搭建——用脚手架，而不是拐杖。

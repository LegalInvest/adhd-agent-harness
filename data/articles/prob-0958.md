# Reclaim.ai 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Reclaim.ai：它到底补上了哪一层执行功能？

你打开 Reclaim.ai，设定好“深度工作”时段，AI 自动帮你安排会议、阻挡打扰，甚至根据你的精力模式调整日程。头两周，你觉得自己终于“正常”了——任务不再拖延，时间不再失控。但一个月后，你发现一旦关掉 Reclaim，你连最简单的待办清单都无法启动。你开始怀疑：这工具是帮我，还是替我做决定？

这个问题，同样困扰着正在构建长程 agent 的工程师。他们发现，一个自主的 LLM agent 在运行 10 分钟后，就会偏离初始目标——它记住了太多上下文，被无关细节带偏，生成的内容开始“跑题”。于是他们引入“重锚定 system prompt”：每 5 分钟，向 agent 重新注入原始目标指令，强制它回到正轨。

Reclaim.ai 之于 ADHD，就像重锚定 system prompt 之于 LLM——但有人用错了。

## 同构：高产大脑与生成引擎的同一困境

ADHD 大脑与 LLM 在结构上惊人地同构：两者都是**高产但缺乏可靠执行调度层的生成核心**（来源：AI 与 ADHD 的专注力）。ADHD 患者拥有强大的创意发散能力，却因执行功能缺陷——工作记忆容量有限、时间感知困难、任务启动障碍——而无法稳定输出。同样，LLM 能生成流畅文本，但无状态、易受上下文膨胀干扰，需要外部 harness 来控制输出质量（来源：无状态与外部记忆）。

这种同构并非空洞比喻。最新研究发现，Transformer 的自注意力机制在训练后，会模仿前额叶-纹状体门控操作——这正是人类执行功能的核心神经基础（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。ADHD 患者的执行功能缺陷，本质上就是这一门控机制的失调。而 LLM 的“注意力漂移”，则是自注意力在长程上下文中的弱化。

两者的解法因此同构：都需要一个**外部调度层**，在目标漂移时进行“重锚定”。对于 LLM，这是 system prompt 的周期性注入；对于 ADHD，这是像 Reclaim.ai 这样的外部日程锚点。

## Reclaim.ai 补上了哪一层执行功能？

Reclaim.ai 的核心机制是**自动时间块化与优先级调度**。它通过 AI 分析你的日历、任务和精力模式，自动为深度工作预留时间，并动态调整以应对冲突。这直接对应 ADHD 的执行功能短板：
- **时间盲**：ADHD 患者难以准确估计时间流逝和任务耗时。Reclaim.ai 的自动时间块化提供了外部时间结构，减轻了工作记忆负担（来源：RescueTime 页面提及时间盲）。
- **任务启动**：ADHD 的启动困难常源于“选择瘫痪”——面对多个任务不知从何开始。Reclaim.ai 通过预设日程，消除了“现在该做什么”的决策成本。
- **目标漂移**：ADHD 易被环境带偏，Reclaim.ai 的“深度工作”时段相当于一个外部锚点，强制你回到预定轨道。

这与 LLM agent 的“重锚定 system prompt”完全对应：定期向 agent 注入核心目标指令，防止它被中间步骤的细节带偏。Reclaim.ai 的每日/每周计划，就是 ADHD 大脑的 system prompt。

## 但有人用错了：脚手架 vs 拐杖的边界

问题在于，许多用户将 Reclaim.ai 当作**拐杖**而非**脚手架**。他们完全依赖工具的自动调度，不再培养自己的时间感知和任务规划能力。一旦工具失效——比如服务器宕机或日程冲突未被正确处理——他们便陷入瘫痪。

这正是“脚手架 vs 拐杖”的核心争议：真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：拐杖与脚手架）。Reclaim.ai 如果只是替代你的执行功能，而不是训练它，那么你永远无法独立。

从同构视角看，LLM agent 的工程实践也揭示了同样的问题。如果 agent 完全依赖外部重锚定 prompt，而自身没有学习如何维持目标，那么一旦 prompt 注入失败，agent 就会失控。最好的 agent 设计是让重锚定作为一种**训练信号**，帮助 agent 逐渐内化目标维持能力。

## 证据与局限

Reclaim.ai 的有效性证据主要来自用户报告，缺乏严格对照实验（来源：矛盾与存疑）。其他工具如 Focusmate 基于身体在场效应，有用户研究支持其提升任务启动和维持，但同样缺乏随机对照试验（来源：Focusmate 页面）。RescueTime 通过客观时间记录提升时间觉察，有初步研究支持（来源：RescueTime 页面）。Brain.fm 利用神经锁相技术，但针对 ADHD 的独立临床研究缺乏（来源：Brain.fm 页面）。

争议点包括：
- **依赖风险**：长期使用外部执行功能层是否会导致内在能力进一步退化？目前缺乏长期追踪研究（来源：矛盾与存疑）。
- **个体差异**：ADHD 异质性大，同一工具对不同亚型效果迥异（来源：矛盾与存疑）。
- **同构的局限性**：ADHD 大脑与 LLM 的同构是隐喻而非严格等价，人类注意力与 Transformer 注意力存在本质差异（来源：AI 与 ADHD 的专注力）。

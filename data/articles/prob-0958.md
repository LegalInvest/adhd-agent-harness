# Reclaim.ai 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Reclaim.ai：它到底补上了哪一层执行功能？

你打开了 Reclaim.ai，设好了“专注块”，希望它能像魔法一样让你在任务上待满 90 分钟。但半小时后，你发现自己在刷社交媒体——工具并没有阻止目标漂移。问题出在哪？

Reclaim.ai 之于 ADHD，就像重锚定 system prompt 之于 LLM——但有人用错了。它补上的是**外部调度与重锚定层**，但如果你期望它自动替你执行，那它只是拐杖。

## 同构：ADHD 大脑与 LLM 共享的缺失层

ADHD 大脑与 LLM 在结构上同构：两者都是高产的生成核心，但缺乏可靠的执行调度层（来源：AI 与 ADHD 的专注力，核心论点）。ADHD 个体分心走神或超聚焦跑偏，丢失目标；LLM/agent 在长程任务中同样会目标漂移，需要重锚定 system prompt。

Reclaim.ai 的核心功能——自动安排专注块、智能提醒、日历整合——本质上是在提供**外部调度层**。它用 AI 帮你决定“现在该做什么”，并周期性地把你拉回主线。这与 LLM 的 harness 工程如出一辙：harness 在运行时编排工具调度、上下文管理和安全执行（来源：拐杖与脚手架，LLM/harness 一侧）。

## Reclaim.ai 补上了哪一层？

从同构视角看，Reclaim.ai 主要补偿了 **任务启动** 和 **上下文保持** 两层执行功能。

- **任务启动**：ADHD 个体常因执行功能缺陷（如启动困难）而无法独立开始任务（来源：Focusmate，与 ADHD 的关系）。Reclaim.ai 的“专注块”强制分配时间，相当于外部启动信号。
- **上下文保持**：ADHD 工作记忆容量有限且不稳定，跨会话任务难以持续（来源：无状态与外部记忆，ADHD 一侧）。Reclaim.ai 的日历整合和任务优先级排序，充当了“永远在线”的外部记忆体。

但 Reclaim.ai 不补偿 **时间盲** 和 **注意力维持**。RescueTime 通过自动记录时间使用提升时间盲觉察（来源：RescueTime，与 ADHD 的关系），而 Focusmate 通过实时同伴问责弥补注意力维持的调度缺失（来源：Focusmate，与 ADHD 的关系）。Reclaim.ai 没有这些功能。

## 有人用错了：脚手架 vs 拐杖

问题在于，许多人把 Reclaim.ai 当作拐杖——期望它自动执行，自己无需主动参与。但真正的脚手架应帮助使用者“建造”，使用者仍需自己“攀登”（来源：拐杖与脚手架，ADHD 一侧）。

Reclaim.ai 的重锚定机制（如智能提醒）类似于 LLM 的 system prompt 重锚定，但**它需要你主动配合**。如果你设了专注块却无视提醒，工具就沦为摆设。

同样，LLM/agent 的 harness 如果设计成完全自主而不留干预接口，也会导致失控。好的 harness 在首次提示前组装代理，并在运行时提供上下文传递、工具接口、规划工件、验证循环（来源：拐杖与脚手架，LLM/harness 一侧）。这意味着 agent 和人类都需要持续参与。

过度依赖风险真实存在：若工具替代了治疗或自我管理，可能造成依赖（来源：拐杖与脚手架，ADHD 一侧）。目前无长期研究证明脚手架可逐步撤除（来源：矛盾与存疑，过度依赖 vs 脚手架可撤除性）。

## 争议与局限

- **证据缺口**：多数工具（包括 Reclaim.ai）缺乏针对 ADHD 的独立临床研究，效果多基于理论推断或用户主观报告（来源：AI 与 ADHD 的专注力，仍存争议）。
- **个体差异**：ADHD 亚型（注意力缺陷 vs 多动冲动）对 AI 工具反应不同，现有工具未充分个性化（来源：AI 与 ADHD 的专注力，仍存争议）。
- **多巴胺干预有效性**：基于多巴胺的 AI 干预是否真正改善专注，证据矛盾（来源：矛盾与存疑，多巴胺干预的有效性）。Reclaim.ai 不涉及多巴胺调节，但用户若期待它带来“多巴胺奖励”则可能失望。

## 今天就能试的行动

1. **区分“锚定”与“执行”**：用 Reclaim.ai 设置专注块后，在块内配合 Focusmate（虚拟身体在场）或 Brain.fm（神经锁相音乐）来维持注意力（来源：Brain.fm，与 ADHD 的关系；Focusmate，与 ADHD 的关系）。

3. **结合 RescueTime**：用 RescueTime 追踪实际时间使用，对比 Reclaim.ai 的计划，识别时间盲模式（来源：RescueTime，与 ADHD 的关系）。

Reclaim.ai 是一个好脚手架，但别把它当拐杖。真正的重锚定，需要你主动参与。

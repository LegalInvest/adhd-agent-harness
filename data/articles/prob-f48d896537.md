# ADHD 研究生视角：为什么「治好 ADHD 的不擅长精确、重复、状态维护类任务」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 通过 function calling 把精确计算外包给工具——同一套 harness 思路，两边都成立。

## 引言：两个看似无关的崩溃现场

一边是 ADHD 研究生：脑子里想法很多，写论文时却卡在「到底先调代码还是先改引言」；刚查过的文献转眼就忘；对重复校对、格式调整、数据清洗这类「精确、重复、状态维护」任务极度不耐。另一边是 AI 工程师：LLM 能流畅生成大纲，却会在长对话中跑题、算错数字、丢失上下文，甚至一本正经地编造引用。两边的崩溃，表面症状不同，底层结构却惊人地一致：它们都是**高产生成核心 + 不可靠执行调度层**。ADHD 不是「不想专注」，LLM 也不是「不懂常识」；真正缺的是一个能把「想法」稳定地转换为「执行」的外部 harness。

## 同构：强记忆、弱控制

ADHD 与 LLM 的交汇并非浪漫比喻。最新研究把 LLM 放进经典执行功能测试，发现其工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷，无法灵活切换任务集、也无法抑制自动化反应——这正好是 ADHD 的神经心理模式（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。耶鲁研究进一步指出，Transformer 的自注意力机制在 N-back 任务中随负荷增加而注意力熵增、弥散，表现下降，与人类注意分散机制同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。换句话说，**瓶颈不在算力/创意，而在 orchestration 层**。工作记忆脆弱、注意力漂移、状态无法持久，是 ADHD 大脑和 LLM 共享的缺陷。

## 同一套解法：把精确执行外包给工具

既然生成核心没问题，那就不要硬改它，而是外挂一个「执行功能层」。对 ADHD 人，Goblin Tools 的 Magic ToDo 把「清理厨房」这类模糊目标分解为可执行的子步骤，并允许调节粒度，降低任务启动门槛（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）；Saner.AI 用 Personal AI 做自动笔记组织和语义检索，把不稳定的工作记忆外化为可搜索的本地记忆，减少 ADHD 用户常见的「搜索循环」（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）；Lex 则把「单一指令触发多步骤任务」作为核心，用低启动成本入口推动执行（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。

对 LLM/agent，工程师做本质相同的事：通过 function calling 把精确计算、代码执行、数据库查询外包给计算器、Python 解释器、检索系统；用外部记忆（向量库、知识图谱）补足上下文窗口；用规划/反思循环把发散的生成约束回目标。两者都叫 harness：不是让 AI 替你做主，而是给生成核心装一个可验证、可回滚、可维护的调度外壳。

## 人物案例：古人早就发明了 harness

harness 思路并非 AI 时代才有。孔子身高一米九、精力旺盛、周游列国十四年坐不住，思维跳跃到《论语》全是场景化语录，没有系统著作；他对种地等俗事零耐心，却能「三月不知肉味」。他的 harness 是「克己复礼」——用外在的「礼」作为行为边界，并每日「三省吾身」。这与 LLM harness 的对应非常直接：**「礼」 = 提示词模板/输出格式约束（schema），「日省」 = 定时 re-grounding/反思循环，把漂移的注意力拉回目标。**

再看毛泽东：小时候读闲书、坐不住；一辈子在调研中移动；思维极度跳跃，讲话充满比喻；他的 harness 是「没有调查就没有发言权」和「民主集中制」——

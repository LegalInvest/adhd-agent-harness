# ADHD 学生如何用「给生成核心套 harness」的思路应对考试与作业里的任务启动困难、不会拆解？

> 把 LLM 工程里的「agent 用 planner-executor 循环分解长任务」迁移到ADHD 学生的考试与作业场景，双域证据作答。

## 问题：为什么你的大脑像一台没有操作系统的超级计算机？

你盯着空白的文档，光标闪烁，心脏却像被什么压住了。你知道该写论文，但“写论文”三个字太大、太模糊，像一座山。你试过番茄钟、待办清单、时间块——但计划本“用了一周就崩溃了”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。这不是你懒，是你的大脑天生缺少一个可靠的“执行调度层”。

同样的问题，也困扰着做 Agentic Harness 的工程师。他们发现，大语言模型（LLM）虽然能生成惊艳的代码，但让它独自完成一个长任务——比如“写一个完整的 Python 爬虫”——它要么在中间丢失上下文，要么陷入无限循环。LLM 和你的大脑一样：**高产，但缺调度层**。

## 同构：ADHD 大脑 = LLM，harness = 外部执行功能

最新研究揭示了一个惊人的平行：ADHD 患者的工作记忆容量未必差，但“自上而下的控制和调节能力不足”，呈现“强记忆、弱控制”的认知剖面（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这几乎就是 LLM 的翻版——Transformer 的自注意力机制缺乏人类特有的执行控制，在长上下文下冲突解决能力崩溃至随机水平（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

工程师的解法是给 LLM 套一个“harness”：用 planner-executor 循环分解长任务，让一个 agent 负责规划（planner），另一个负责执行（executor），中间用外部记忆和工具调用防止上下文膨胀（来源：同上）。这个结构，恰恰是 ADHD 大脑需要的——只是你的“外部 harness”不是代码，而是一个**可视化的、自动化的、可动态调整的调度系统**。

## 迁移：给“生成核心”套 harness 的三种具体策略

### 1. 用“自动 planner”替代你的前额叶

Motion 和 Reclaim.ai 这类工具，本质上就是你的外部 planner。Motion 自动根据截止日期、优先级和可用时间重建日程，“消除‘下一步该做什么’的决策负担”（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。Reclaim.ai 则更强调“保护”：它自动创建深度工作块，防止会议侵占，帮你维持日程的稳定性（来源：11 Best ADHD Productivity Apps for Fluctuating Energy）。

**ADHD 侧证据**：用户报告显示，这些工具能显著减少决策疲劳（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。**LLM 侧证据**：现代编码代理如 OpenDev 采用“分离规划与执行的双代理架构”，正是为了避免单一 agent 在长任务中“推理退化”（来源：Building AI Coding Agents for the Terminal）。

**怎么用**：考试前，把所有任务（复习章节、做真题、整理错题）输入 Motion，让它自动排成时间块。你不需要思考“接下来做什么”——planner 已经替你决定了。

### 2. 用“视觉化 executor”让时间可见

ADHD 的“时间盲”让你无法感知时间流逝，就像 LLM 缺乏对时间约束的固有理解。Tiimo 通过“将时间转化为可见元素”（来源：Best AI Tools for ADHD Productivity in 2026），用颜色和视觉惯例把日程变成一幅画。这相当于给 LLM 加了一个进度条——每次看到剩余时间，你的“executor”就能校准节奏。

**证据**：Tiimo 专为 ADHD 和自闭症成人设计，用户反馈其能改善时间感知（来源：11 Best ADHD Productivity Apps for Fluctuating Energy）。**LLM 平行**：实验中，给 LLM 提供时间戳或进度提示，能显著减少上下文漂移（来源：AI Tools for ADHD 综述）。

**怎么用**：用 Tiimo 把作业拆成 15 分钟的小块，每个块一种颜色。视觉上，你看到的不再是“写论文”，而是一个个可完成的彩色方块。

### 3. 用“任务分解器”对抗启动困难

启动困难源于任务太大、太模糊。Goblin Tools 的“任务分解”功能可以把“写论文”拆成“打开文档→写第一段→写第二段……”——这就是 LLM 工程里的“惰性工具发现”：只在需要时才调用具体的执行工具（来源：Building AI Coding Agents）。

**ADHD 侧证据**：工作记忆干扰研究表明，ADHD 患者在无外部结构时难以自主组织任务（来源：Human-like Working Memory Interference in Large Language Models）。**LLM 侧证据**：LLM 在没有手动修正提示时，也难以自主发现最佳问题解决模式（来源：Working Memory Identifies Reasoning Limits in Language Models）。

**怎么用**：面对作业时，先让 Goblin Tools 或 ChatGPT 把任务拆成 5-10 个微型步骤。然后只做第一步——就像 executor 只执行当前指令，不理后面的。

## 争议与边界：脚手架 vs. 拐杖

同构理论虽诱人，但证据有限。多数工具的效果基于用户报告，缺乏随机对照试验（来源：AI 与 ADHD 的时间管理）。更关键的是，过度依赖可能削弱你的内在能力——就像 LLM 如果永远靠外部 harness，永远学不会自己规划。专家警告：真正的“脚手架”应帮助你“建造”，但使用者仍需自己“攀登”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

**我的判断**：harness 不是万能药。它帮你启动，但不帮你学会感知时间；它帮你分解，但不帮你练习专注。使用时要刻意“逐步撤除”——比如从用 Motion 自动排程，过渡到手动排程 + Motion 校验。否则，工具会从“脚手架”变成“永久拐杖”。

## 今天就能试的行动

1. **用 Motion 排一次考试复习**：把所有任务输入，让它生成明日计划。执行时只跟随，不思考。
2. **用 Tiimo 视觉化一个作业**：把 2 小时作业拆成 8 个 15 分钟块，每个块涂不同颜色。
3. **用 Goblin Tools 分解一个“大山”任务**：输入“写论文”，看它拆出什么步骤，然后只做第一步。
4. **设置一个“撤除日”**：每周选一天，关掉所有 AI 调度，手动规划。观察你的时间感知是否进步。

你的大脑不是坏的，只是缺一个 harness。现在，你知道了怎么造它。

# ADHD 学生如何用「给生成核心套 harness」的思路应对考试与作业里的任务启动困难、不会拆解？

> 把 LLM 工程里的「agent 用 planner-executor 循环分解长任务」迁移到ADHD 学生的考试与作业场景，双域证据作答。

考试前夜，你盯着复习清单却打开了五个无关标签页；作业截止前，题目明明看得懂，第一步却怎么都落不下去。对 ADHD 读者来说，这是‘任务启动困难’和‘不会拆解’的日常。对做 Agentic Harness 的工程师来说，这一幕也不陌生：一个强大的 LLM 可以滔滔不绝，却在长任务里循环规划、上下文膨胀，直到你给它的 planner-executor 循环套上一层 harness，它才真的开始干活。

这不是巧合。ADHD 大脑与 LLM 在结构上是同一类系统：高产的生成核心，配上一个不可靠的内置执行调度层。给学生用的解法，和给 agent 用的解法，应该是同构的。

## 一、同构核心：两个生成核心，同一种调度失灵

ADHD 的核心困境常被描述为‘大脑的驾驶座常常感觉方向盘后没有人’——不是智力不足，而是执行功能失效：计划、组织、工作记忆、冲动抑制统统掉线（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。这直接导致计划本和便签用一周就崩溃、时间盲、任务启动困难。工作记忆瓶颈进一步削弱了任务编排能力（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

LLM 一侧也是类似：语言生成能力极强，但单独使用时缺乏可靠的执行调度。现代 AI 编码代理（如 OpenDev）通过‘复合 AI 系统架构’来弥补，包括‘工作负载专用模型路由、分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩’，本质上都是防止上下文膨胀和推理退化（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

更深层的证据支持这种同构：ADHD 患者呈现‘强记忆、弱控制’的认知剖面，LLM 也表现出相似的执行功能 profile（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）；LLM 在工作记忆任务中会随着记忆负荷增加而表现下降，受干扰控制和近因效应影响，这与人类 ADHD 的干扰特征平行（来源：Human-like Working Memory Interference in Large Language Models）；Transformer 的自注意力机制在训练后甚至发展出模仿前额叶-纹状体门控的操作（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。这些不是直接证明‘大脑等于神经网络’，但说明把 ADHD 大脑当作一个需要外部调度层的生成核心，是一个可落地的工程视角。

## 二、从 agent 的 harness 到学生的 harness

在 agent 工程里，harness 是围绕 LLM 的提示、记忆、代码、工具和编排逻辑。对学生的 harness 来说，对应物就是一套外部执行功能支架：把‘现在该做什么、做多久、何时切换’从大脑里卸出来，交给一个可见、可调整的外部系统。

工具层面，这种支架已经存在。Motion 的自动排程与动态调整会考虑任务优先级、依赖、截止日期和持续时间，减少手动规划压力（来源：The AI Powered SuperApp for Work | Motion）；Tiimo 用视觉时间线把‘时间盲’转化为可见元素，用动态流程替代静态待办清单（来源：The 12 Best Apps for ADHD in 2026: A Guide to Finding What ...）；Reclaim.ai 则偏向防御：它保护深度工作和习惯时间块，防止会议侵占（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。这三者组合起来，就是给学生用的‘planner-executor 循环’：planner 负责分解与排序，executor 负责到点执行与反馈，harness 负责把两者持续同步。

# ADHD 创业者视角：为什么「治好 ADHD 的任务启动困难、不会拆解」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 用 planner-executor 循环分解长任务——同一套 harness 思路，两边都成立。

## 同一个问题，两种面孔

如果你是一位 ADHD 创业者，你可能对这一幕再熟悉不过：面前摆着一个重要项目，你知道该做，但就是启动不了。大脑里想法翻涌，却像一团乱麻，不知从何下手。

如果你是一位 Agentic Harness 工程师，你可能也经常头疼：LLM 刚生成一个长任务规划，跑着跑着就偏了，要么陷入细节死循环，要么忘记最初目标。

这两件事，听起来八竿子打不着。但深入看看——ADHD 的“任务启动困难”和 LLM/Agent 的“跑飞”，本质上是一道工程题：**一个高产的生成核心，缺了一个可靠的调度层**。

## 同构：ADHD 大脑与 LLM 都是“缺调度层的生成核心”

最新理论指出，ADHD 的超聚焦源于前额叶控制层级的“断裂”：高级上下文信息不再能支配思维和行动，中间层级信息与高级调节脱耦合（来源：Transient Frontal Fracturing: A Theoretical Account of Hyperfocus）。换句话说，ADHD 大脑能产生大量想法（生成核心），但缺乏一个内置的“调度器”来持续按优先级执行。

LLM 也一样。一个没有 agent 框架的 LLM，接到长任务后要么一次生成太长而偏离，要么重复固定模式。Transformer 注意力机制在长序列中会表现出类似 ADHD 执行功能缺陷的现象：在 Stroop 冲突任务中，模型随序列变长而无法抑制优势反应，表现骤降到随机水平（来源：Deficient Executive Control in Transformer Attention）。

所以，ADHD 大脑和 LLM 是同一类：**高产、发散，但缺调度层**。

## 解法：Harness 即调度层

对 ADHD 来说，这个调度层就是“外部执行功能层”——AI 工具或人为系统，帮大脑做“下一步做什么”的决定。对 LLM 来说，就是 planner-executor 循环：一个规划器分解任务，一个执行器逐步推进，然后循环回规划器检查进度。

### 案例：孔子的“克己复礼”与 LLM 的 re-grounding

孔子是典型的 ADHD 大脑：身高 1 米 9，精力旺盛，周游列国 14 年坐不住；冲动爱骂人，思维跳跃，《论语》全是场景化语录，无系统著作。他的 harness 是什么？以“克己复礼”为核心，用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”），70 岁才做到“从心所欲不逾矩”。

这个“每日反省”就是定时 re-grounding——相当于 LLM agent 中每隔 N 步就重新检查当前任务目标，确保不跑偏。孔子的“礼”就像外部调度器，帮他约束冲动，把发散的能量导向系统化的成就（儒家学派，万世师表）。

### 现代工具：从 Motion 到 Goblin Tools

今天，ADHD 创业者可以借助 Motion 来自动排程：它持续评估任务优先级、截止日期和可用时间，实时重建日程，消除“下一步该做什么”的决策负担（来源：Motion）。这相当于给大脑装了一个外部 planner。

Reclaim.ai 则更强调“防御性”时间块，保护深度工作不被会议侵占（来源：Reclaim.ai）。这和 LLM agent 中的“优先级队列”异曲同工——确保高价值任务不被低优先级打断。

Tiimo 把时间变成视觉元素，直接应对时间盲（来源：Tiimo）。对 LLM 来说，这就像用 token-level 的进度条或思维链可视化，让模型“看到”当前在任务中的位置。

而 Goblin Tools 专门做任务分解：把“写报告”拆成“打开电脑→新建文档→写第一段……”（来源：Goblin Tools）。这正是 agent 中 planner-executor 循环的微缩版。

## 脚手架 vs 拐杖：一条必须划清的边界

但这里有个争议：这些工具是脚手架还是拐杖？

脚手架是临时辅助，帮你学会自己走路；拐杖则是永久替代，让你失去代偿能力。AI 工具可能从“脚手架”退化为“永久拐杖”，削弱用户内在执行功能的代偿发展（来源：AI 与 ADHD 的时间管理）。

对 LLM agent 也一样：如果 planner 完全接管所有决策，模型永远学不会自主纠偏；一旦 planner 失效，整个系统就瘫痪。

所以，好的 harness 应该保留“人在回路”的接口——比如孔子每天主动反省，而不是被动被礼约束；比如 Motion 允许用户 override 自动排程。

## 今天就能试的 3 件事

1. **外化任务分解**：用 Goblin Tools 或 Todoist 把一个大任务拆成 5-10 个原子步骤，每完成一步就打勾。这相当于给你的大脑装一个 planner。
2. **设置定时 re-grounding**：每 25 分钟（番茄钟）问自己一次：“我现在做的事，还在通往最终目标吗？”像孔子的“三省吾身”。
3. **试用自适应规划器**：Motion 或 Reclaim.ai 免费版，让 AI 替你决定“下一步做什么”，观察决策负担降低后的启动速度。

## 诚实面对局限

必须承认，ADHD 与 LLM 的同构目前仍是一种理论类比，缺乏严格实证（来源：矛盾与存疑）。不同工具的证据质量参差不齐——Motion 缺乏独立临床验证（来源：Motion），而 Tiimo 的用户口碑可能受选择偏差影响。

但即便作为思考框架，这个同构视角已经足够有价值：它让我们用工程思维去理解 ADHD，用神经多样性视角去优化 AI。

## 结语

ADHD 创业者面对的不是“缺陷”，而是一个高产生成核心缺了调度层。LLM agent 工程面对的不是“模型不够聪明”，而是同样的调度缺失。

两条路在同一个交叉口交汇：**搭好 harness，产出就能从混乱变成系统**。

孔子用“礼”搭了 harness，成了万世师表。于谦用“气节”搭了 harness，救了明朝。

今天，你可以在手机上装一个 Motion，开始搭你的 harness。

# 为什么「治好 ADHD 的不擅长精确、重复、状态维护类任务」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 通过 function calling 把精确计算外包给工具——同一套 harness 思路，两边都成立。

## 一、问题：为什么两个看似无关的群体，说的是同一句抱怨？

ADHD 读者常有的体验：脑子里点子不断、能量旺盛，甚至能进入“超专注”——研究显示 ADHD 组的超专注工作时长平均可达 14.5 小时，而对照组只有 8.2 小时（来源：Pace Ventures Research 2026）。但一到精确计算、重复核对、长期状态维护，就崩盘：账单拖成山、时间“咻”地溜走、任务在记忆里消失。

Agent 工程师的噩梦也类似：LLM 生成能力惊人，却会在长对话中“跑飞”——丢失目标、产生幻觉、把 3.99 算成 4.01，或者把跨会话状态忘得一干二净。两边的问题听起来像同一个句式：

> **“核心很强，但缺一个靠谱的执行调度层。”**

这正是 ADHD 大脑与 LLM/agent 的同构点：它们都是高产的生成核心，却缺乏稳定的计划、抑制、记忆与状态维持机制（来源：ADHD 的 AI 工具全景）。所以“治好 ADHD 的精确/重复/状态维护困难”和“让 LLM 不跑飞”，其实是同一道工程题：给这个核心装一个**外部 harness**（脚手架）。

## 二、同构的解法：把“不能做的事”外包给工具

ADHD 侧的干预不是“逼自己变得更正常”，而是承认前额叶执行功能有限，把执行层外置。当前工具主要做四件事：任务分解与启动、外部记忆、时间管理、元认知反思（来源：ADHD 的 AI 工具全景）。它们本质上都是“把内部不可靠的工作记忆和调度能力，外部化成可检索、可触发的结构”。

LLM/agent 侧的做法几乎一一对应：模型自身没有原生规划、持久状态和精确计算能力，于是工程师用 **agent scaffolding** 在外部构建记忆、规划、工具调用与约束层（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。其中最典型的操作就是 **function calling**：模型把精确计算、数据库查询、状态更新这类“它不擅长也不该做”的任务，外包给确定性工具。

对比来看：

- **任务分解**：Goblin Tools 的 Magic ToDo 把“清理厨房”拆成具体小步骤，并可用“辣度”调节粒度（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）；Lex 用“单一指令触发多步骤任务序列”（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。对应 agent 侧，就是把复杂目标拆成可验证的子目标与工具调用链。
- **外部记忆**：Saner.AI 通过本地记忆与语义搜索减少 ADHD 用户的“搜索循环”和标签切换（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）；LLM 则把跨会话状态写入向量数据库或结构化存储，而不是依赖有限的上下文窗口。
- **时间管理**：ADHD 的“时间盲”让任务像“无限延迟”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）；Tiimo 用视觉化时间表把“时间变得可触摸”（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。对应 agent 侧，就是给模型加时间戳、计时器、状态过期检查与重调度器。
- **身体在场/问责**：Focusmate 等虚拟身体加倍工具能将目标达成率从 25% 提升到 95%（来源：ADHD Task Managers That Work: Top AI Tools 2025）；对应 LLM 侧，就是 **human-in-the-loop** 与审计日志——让外部监督成为系统的一部分，而不是让模型自己监督自己。

所以，ADHD 的“AI 工具”和 LLM 的“agent harness”共享同一个核心操作：**上下文工程**——用外部结构主动限定“此刻应关注什么”，以结构换可控性（来源：ADHD 的 AI 工具全景）。

## 三、从孔子的“礼”到左宗棠的“硬寨”：人类 harness 的古老版本

这种“给生成核心加外部 harness”的策略并非 AI 时代才有。两千多年前的孔子，就是典型的高能量、易冲动、思维跳跃的“生成核心”：身高一米九、精力旺盛、周游列国十四年坐不住；对音乐能专注到“三月不知肉味”，对种地等俗事毫无耐心；一生没有系统著作，只留下场景化语录。他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日“三省吾身”，直到七十岁才“从心所欲不逾矩”。

孔子的“礼”与 LLM 的 **system prompt / 工具策略** 是同构的：它不是压抑核心，而是给高产生成划定可运行边界；他的“三省吾身”则对应 agent 的 **每日重校准（re-grounding）** 与反思 loop。

晚清左宗棠则是另一个版本：脾气大、风风火火、今天的事必须今天做完。他的 harness 是“结硬寨，打呆仗”——每日读经、写家书，用高度结构化的日常仪式和外部 accountability（对家族、对国家的责任）把高能量锁进可执行轨道。这个结构放在 LLM 工程里，就是**确定性工作流 + 外部调度器**：不让模型随意发散，而是把能量导入预置的管道。

两人的案例说明：人类 harness 与 agent harness 的关键组件一致——**外部约束、可检索的例行程序、定期反思、他人或系统的问责**。

## 四、核心观点：这不是浪漫类比，而是同一类结构工程

我的判断是：**ADHD 与 LLM 的同构性，不在于“大脑就是模型”，而在于它们都需要同一类外部架构——一套把生成与执行分离、把精确/重复/状态维护外包的 harness。**

对 ADHD 个体来说，AI 工具最有效的用法不是“让 AI 替我想”，而是“让 AI 替我记住、替我启动、替我计时、替我核对”。对 agent 工程师来说，LLM 最有效的用法也不是“让模型什么都做”，而是让模型做它擅长的语言推理与意图理解，把计算、查询、状态更新通过 function calling 丢给确定性工具。

但这里必须诚实指出边界与争议。wiki 的“矛盾与存疑”部分提醒我们：

- ADHD 大脑与 LLM 的“同构”目前只是理论类比，**缺乏实证基础**，不能把它当作既定科学事实（来源：全局矛盾与存疑）。
- 一些工具（如 Motion、Brain.fm）的独立临床证据有限，存在“推荐”与“证据”脱节的风险（来源：全局矛盾与存疑）。
- “身体在场”的人际效果是否能被 AI 完全替代，仍有争议；工具本身也可能增加认知负荷，甚至变成“拐杖”而非“脚手架”（来源：全局矛盾与存疑）。

因此，好的 harness 必须满足两条边界：第一，它**放大人的决策能力**，而不是替代人的主体性；第二，它**保持透明、可退出、可审计**，避免系统越复杂，人越失控。

## 五、今天就能试的四件事

1. **把下一个模糊任务交给外部拆解器**：用 Goblin Tools 的 Magic ToDo 或类似工具，把“整理房间/写报告”拆成可执行的下一步，降低启动门槛。
2. **给你的 LLM/agent 加一个确定性 function call**：遇到需要精确计算、查表、数据库操作的场景，不要让模型硬算，而是定义一个工具调用，让模型只做“判断要不要调用”。
3. **给时间加视觉锚点**：用 Tiimo 或日历块把任务放到时间线上，把“时间盲”变成“时间可见”。
4. **建立 5 分钟“日省”仪式**：每天晚上回答三个问题——今天完成了什么？哪个外部工具最有用？明天的第一步是什么？对应 agent 的每日 log review 与重校准。

归根结底，ADHD 个体与 LLM agent 都不需要被“修复”成一个全能系统；它们只需要一套承认自身强项、并把弱项外包给可靠外部结构的 harness。治好 ADHD 的精确执行困难，和让 LLM 不跑飞，解法确实是同一套工程图。

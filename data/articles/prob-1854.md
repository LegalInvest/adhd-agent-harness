# ADHD 职场人如何用「给生成核心套 harness」的思路应对项目推进与汇报里的不擅长精确、重复、状态维护类任务？

> 把 LLM 工程里的「agent 通过 function calling 把精确计算外包给工具」迁移到ADHD 职场人的项目推进与汇报场景，双域证据作答。

## 问题：ADHD 职场人的“精确、重复、状态维护”困境

你是否有这样的经历：项目推进中，需要定期更新进度表、整理周报、维护客户沟通记录——这些任务本身不复杂，但需要精确、重复、持续跟踪状态。对 ADHD 大脑而言，这类任务就像“注意力黑洞”：启动困难（总觉得“等会儿再做”）、执行中频繁中断（被新想法带跑）、维护失败（忘记更新或丢失上下文）。最终，项目核心产出（创意、方案、决策）往往高质量，但外围的“执行层”工作却漏洞百出，导致汇报时被质疑“细节不靠谱”。

这种困境的根源，在于 ADHD 大脑是一个“高产但缺执行调度层的生成核心”——它能快速产生创意和洞察，却难以自动管理执行所需的精确计算、重复操作和状态维护。而恰恰是这种“生成核心”特性，与当前大语言模型（LLM）和 agent 系统的架构高度同构。

## 同构解法：给生成核心套 harness

### ADHD 侧：执行功能缺陷的本质

ADHD 的执行功能缺陷，核心是工作记忆控制弱、任务启动困难和时间感知缺陷（时间盲）。这些缺陷使得 ADHD 个体难以自动完成需要持续注意力、精确顺序和状态跟踪的任务。正如 wiki 资料指出，ADHD 大脑“渴望新颖性，但容易在错误的事情上超聚焦六小时”（来源：超聚焦），并且“情绪失调导致情绪波动剧烈、易怒”（来源：情绪失调）。这种状态下的职场人，就像一台高性能 GPU 却没有驱动程序——算力强大，但无法稳定输出到显示器。

### LLM/Agent 侧：function calling 的工程智慧

在 LLM 工程中，agent 通过 function calling 将精确计算、重复操作和状态维护外包给专用工具。例如，一个写作 agent 不会自己计算字数、维护版本历史或格式化表格——它调用外部 API 完成这些任务，自身专注于生成内容。这种架构的核心思想是：**生成核心（LLM）负责创意与推理，harness（脚手架/工具链）负责精确执行与状态管理**。这直接对应 ADHD 大脑的痛点——不擅长精确、重复、状态维护，但擅长生成和洞察。

### 同构迁移：职场 harness 设计

基于这种同构，ADHD 职场人可以设计自己的“harness”——一套外部工具和流程，专门处理精确、重复、状态维护类任务，让大脑释放出来做擅长的事。具体而言：

- **任务分解与启动**：使用类似 Goblin Tools 的“魔法待办”功能，将“整理项目周报”这样的模糊任务自动分解为“收集本周已完成事项”“统计未完成项”“列出下周计划”等具体步骤（来源：Goblin Tools）。这对应 agent 中的任务规划器，降低启动门槛。
- **记忆与上下文管理**：利用 Saner.AI 的知识回忆功能，快速找回之前讨论的决策依据、客户偏好等上下文，避免在多个文档和聊天记录中搜索（来源：Saner.AI）。这对应 agent 的持久化记忆系统。
- **单一指令触发多步骤**：通过 Lex 或自定义提示词，用一条指令触发一系列操作，例如“生成本周工作汇报”自动提取日历事件、邮件摘要和任务完成率（来源：Lex）。这对应 agent 的 function calling 链。
- **实时记录与反馈**：使用 Otter.ai 在会议中自动转录并提取行动项，避免事后回忆遗漏（来源：Otter.ai）。这对应 agent 的日志系统。

## 核心观点：脚手架，而非拐杖

本文的核心判断是：**ADHD 职场人应主动设计“执行功能脚手架”，而非被动依赖 AI 工具作为拐杖**。脚手架（harness）是可定制、可进化的结构，旨在增强而非替代能力；而拐杖则会导致依赖和退化。wiki 资料中反复警告“过度依赖 AI 工具可能削弱内在能力”（来源：矛盾与存疑），例如过度使用 Otter.ai 可能削弱主动记笔记的能力。因此，harness 的设计原则应是：

1. **最小化精确任务**：只外包那些“不擅长且低价值”的精确重复任务，保留需要判断和创意的部分。
2. **可逆性**：工具应提供“手动模式”选项，避免完全黑箱。例如，Goblin Tools 的分解步骤应可编辑。
3. **反馈循环**：harness 应提供完成状态的可视化反馈，帮助大脑建立时间感知（对抗时间盲）。

## 争议与局限

必须诚实指出，当前证据主要来自用户报告和工具设计逻辑，缺乏大规模随机对照试验（来源：矛盾与存疑）。例如，Goblin Tools 的“魔法待办”虽然被用户评价为“简单有用”，但其对 ADHD 核心症状的长期效果尚未严格验证。此外，个体差异巨大：某些人可能对 Lex 的单指令触发极度依赖，而另一些人则觉得它过于简化。最重要的是，工具无法替代医疗干预——ADHD 的药物治疗和认知行为疗法仍是基础。

## 今天就能试的行动

1. **用 ChatGPT 或 Claude 创建“周报生成器”**：写一条提示词，要求它从你粘贴的笔记、邮件和日历事件中提取关键信息，生成结构化周报。每次使用后手动调整提示词，逐步优化。
2. **试用 Goblin Tools 的 Magic ToDo**：将一个让你拖延的任务（如“整理报销单据”）输入，观察它分解的步骤。如果步骤合理，直接执行；如果过于琐碎，合并步骤。
3. **设置 Saner.AI 或类似工具作为“第二大脑”**：将会议记录、灵感、重要决策存入，并定期（如每天下班前）用自然语言搜索“今天的关键行动是什么”，养成外部记忆习惯。
4. **设计一个“启动仪式”**：每天开始工作前，用一条指令（如“列出今天三个最重要的任务，并分解第一步”）触发 AI 生成任务清单，降低启动阻力。

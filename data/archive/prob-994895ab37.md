# 为什么用 Mem 治 ADHD 的注意力涣散，和给 agent 套 上下文工程 是一回事？

> 笔记工具的鄙视链顶端,长期站着精致的分类体系:多层文件夹、标签树、PARA 方法。而 Mem 这类「自组织笔记」反着来:**不建文件夹,不强制分类,随手记,靠 AI 和关联把东西在需要时捞出来**。对神经典型用户,这是省事;对 ADHD 的注意力涣散,这是治本——因为它砍掉的那个环节(手动归档),恰好是 ADHD 外部记忆系统的头号崩溃点。而它在 agent 工程里的对应物也很精确:**向量检索取代人工索引——写入零成本、检索靠语义,这正是 agent 外部记忆的主流架构**。

先讲「归档」为什么是 ADHD 的崩溃点,这件事值得说透。认知卸载篇讲过回路公式:外部记忆的价值=写入质量×检索可靠性。传统笔记体系把写入质量的责任压在归档上——记的时候就要决定「这条放哪个文件夹、打什么标签」;**而「分类决策」恰好是一个对 ADHD 极其昂贵的操作**:它是开放式选择(选择瘫痪)、无即时奖励(奖赏系统零点火)、还要求在灵感涌现的瞬间切换到行政模式(联想流被打断)。结果是可预测的:要么记的时候卡在分类上,灵感先死了;要么跳过分类随手扔,系统三周后变垃圾场;要么整理焦虑积累成「我的笔记又乱了」的常驻后台进程——**归档这道工序,以三种方式同时生产涣散**。

Agent 工程在自己的战场上得出了同一个结论。早期的结构化知识库要求人工定义 schema 和索引——维护成本高到没人真用;向量检索的兴起本质上是一次架构投降:**放弃写入时的组织,把成本转移到检索时的语义匹配**——存的时候原文扔进去,取的时候按「意思相近」捞。这个转移对 agent 是效率问题,对 ADHD 是可行性问题:**一个写入零决策的系统,才是 ADHD 能长期维持的系统**(任何需要持续行政投入的系统,终将死于第一个低电量周)。

Mem 的机制逐条对应:**①无摩擦捕获**——任何念头一条进去,不问放哪(写入侧的决策成本归零,联想流不被打断——这对 ADHD 的「想法丢失恐惧」是直接解药:那种「我得一直在脑里攥着这个想法」的后台占用,认知负荷篇讲过,是涣散的隐形大户);**②AI 关联浮现**——写新东西时自动浮出相关旧笔记(等于 agent 的检索增强:相关上下文按需注入,而不是靠你记得去翻);**③自然语言检索**——「上个月关于定价的那个想法」这种模糊查询能命中(检索预演的负担也卸了——你不用在写入时预测未来的检索词)。

但回路公式的另一半要诚实:**语义检索的可靠性没有那么神**。它按相似度捞,捞回的是「大概相关」,可能漏掉关键那条(而你不知道漏了)——agent 工程里这叫检索召回问题,对策是关键状态不依赖检索、主动回灌(认知卸载篇的第四条)。翻译过来:**Mem 可以当你的联想外脑,但不能当你的承诺账本**——「周五交稿」这种绝不能漏的,走推送型工具(闹钟/任务管理),别指望语义检索恰好想起。自组织系统的正确定位:管灵感与知识(漏了不致命),不管义务(漏了致命)。

## 边界

同构定位(本文未做正式 A/B/C 分级):分类决策成本与选择瘫痪有认知研究基础,向量检索架构是工程事实,「写入零成本」的同构对应清晰;Mem 无 ADHD 对照研究,语义检索召回率的量化因场景而异。声明:笔记系统管理信息,不管理情绪与义务;想法丢失恐惧若达到强迫程度,值得专业讨论。

## 今天就能试的 3 件事

1. **做一次归档尸检**:打开你弃用的上一个笔记系统——它死在写入摩擦还是检索失灵?对症选下一个。
2. **试三天零分类捕获**:所有念头进一个入口,不分类不整理——只测一件事:联想流不被打断的感觉。
3. **分家**:把「义务」(deadline、承诺)从笔记里迁出,进带提醒的任务工具——笔记管灵感,闹钟管命。

最适合 ADHD 的笔记系统,不是最精致的,是**写入时不需要你做任何决定的**——agent 的外部记忆早就想通了这一点:组织是检索的事,不是记录的事。

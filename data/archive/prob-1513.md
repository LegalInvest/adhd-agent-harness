# Todoist 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> function calling 那篇讲了 Todoist 的注册摩擦。这一篇看它的另一面:任务列表不只是仓库,它还是你每天实际面对的「上下文」——**你打开 App 时看到什么,决定了接下来一小时你以为自己该做什么**。而这正是 system prompt 工程里被反复调优的问题:注入什么、注入多少、以什么顺序。同一个 Todoist,视图配置不同,等于给自己注入两种完全不同的 prompt——一种让人开工,一种让人窒息。

先看两种打开方式的差别。**错误的注入:打开就是全量列表**——两百多条任务、七个项目、过期的红字堆成山。工程对应:把整个知识库塞进上下文窗口——重点被淹没、注意力被稀释,模型(和你)在海量信息前性能骤降。对 ADHD 还有情绪叠加:全量视图的第一信息不是「做什么」,是「你欠了这么多」——羞耻先于行动抵达,App 越来越不敢打开。**正确的注入:打开只见「今天」视图**——三到五条,经过昨晚或今早筛选。上下文窗口里只有当前需要的,这是 prompt 工程的第一课,也是任务系统的第一课。误用四种:

**误用一:默认视图没配置(裸注入)。** 修法一句话:**把打开默认页设为「今天」或一个自定义过滤器(今天+高优先级),全量列表只在每周回顾时看**。这五分钟的配置,决定这个 App 对你是仪表盘还是耻辱柱。

**误用二:「今天」视图超载(注入了正确的类型,错误的剂量)。** 往「今天」里塞十五条——**「今天」变成了第二个全量列表**,超载的即视感让整页作废。工程对应:prompt 该精简的部分写了三千字。修法:今天视图硬上限五条;放不下的,诚实地挪到明天或本周——**列表对你撒谎(排了做不完的量),你很快学会对它撒谎(不看它)**,诚实是相互的。

**误用三:过期红字不清算(上下文里的腐烂数据)。** 过期任务自动滚进今天,红字越积越多——**每条红字都在注入「你又失败了」,而其中一半其实早就不需要做了**。工程对应:上下文里塞着过时的、互相矛盾的旧信息,模型的输出被污染。修法:每周回顾里加一个固定动作——**过期项三选一:改期(真的要做)、降级(移进「某天」)、删除(承认不做了)**;删除不是失败,是数据清洁。

**误用四:循环任务的化石化(重注入了没人遵守的指令)。** 「每天冥想」设了循环,连续跳过三十天,每天照常弹出、照常被划掉——**这条任务已经从指令退化成了背景噪音,而且在教你的系统「弹出的东西可以无视」**(比没设更糟:它在腐蚀所有提醒的效力)。工程对应:prompt 里那条从未被遵循的指令,留着只会让模型学会指令是可选的。修法:**连续两周被跳过的循环任务,强制处理——改小(冥想十分钟→深呼吸一分钟)、改时段、或删除**;循环任务列表应该全部是「基本都在做」的,那才配叫锚。

## 边界

同构强度 B 级:上下文窗口管理是真实的 LLM 工程议题,任务过载的情绪效应与 ADHD 的回避行为有临床观察基础,Todoist 无对照研究。声明:视图优化管的是「面对系统时的体验」;若五条的今天视图也天天全军覆没,问题在产能预估或启动层(见相应篇章)——注入得再干净,引擎不点火也没用;伴随持续的羞耻与回避时,值得专业评估。

## 今天就能试的 3 件事

1. **改默认视图**:打开即「今天」,上限五条,现在就配。
2. **清算一次过期红字**:改期/降级/删除,三十分钟,清到零。
3. **审判你的循环任务**:连续两周没做的,当场改小或删掉。

任务系统的第二生命在视图——**仓库里可以有两百条,但每天注入眼睛的,只该有五条**。配置好你面对的那一屏,它才能从「欠债清单」变回「今天的地图」。

# Saner.AI 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> system prompt 是 LLM 世界里最不起眼又最要命的组件:一段常驻的指令,定义「你是谁、目标是什么、什么不能做」,每次对话都被重新注入——没有它,模型每轮都从人格真空开始;有了它但写坏了,模型会稳定地执行一套错误的自我定义。ADHD 用户的对应物是什么?**你的「运行原则」:我在做什么阶段、什么最重要、什么坑不能再踩**——问题是这段 prompt 通常只存在于某次深夜顿悟里,第二天就被冲掉了。Saner.AI 这类 AI 笔记助手,恰好能当它的常驻载体——但载体好找,写对和注入对,才是难点。

先立正确基线。system prompt 的工程要点有三:**内容精炼**(几百字,不是几千字——太长会被稀释)、**定期重注入**(每个新会话都带上,不是说一次就完)、**随阶段更新**(项目换了,prompt 要改)。人类版对应:一页纸的「当前运行原则」(本季度的重心、三条铁律、一条最近在改的坏习惯),每天被看到一次,每月修订一次。Saner.AI 的合适角色:把这页纸置顶,让日常的任务与笔记都在它的注视下流动,AI 助手回答你时也参照它。误用四种:

**误用一:prompt 膨胀(prompt bloat)。** 工程版:什么都往 system prompt 里塞——五千字的指令,重点被淹没,模型各条都执行个半吊子。人类版:「运行原则」写成了人生愿望清单——健康、事业、学习、关系、二十七条格言。**ADHD 的工作记忆预算本来就紧,一页装不下的原则等于没有原则**。修法:硬上限——当前重心一条、铁律三条、在改的习惯一条,五行封顶;其余的进「候补区」,升级要挤掉现任。

**误用二:写了,但没有注入机制。** 工程版最低级也最常见的 bug:prompt 写得很好,但代码里忘了每次请求都带上——模型实际在裸跑。人类版:原则写在某条笔记里,**三周没被打开过**——写下的那一刻它就退休了。修法:注入要绑定已有的高频动作——**把原则页设为每天第一次打开笔记时必然弹出的东西**(置顶、日记模板首行、或让 AI 每早的日程摘要先复述一遍重心)。没有注入通道的 prompt,写得再好也是零。

**误用三:prompt 与行为冲突时,总是 prompt 让步。** 工程版:模型行为违反 system prompt,工程师不排查,反而悄悄删掉那条指令——护栏形同虚设。人类版:铁律写着「22 点后不接新工作」,连续违反两周后,你把这条删了——**不是因为原则错了,是因为守不住时删原则比改行为容易**。修法:violation 触发的应该是「审查」而不是「删除」——这条原则还成立吗?成立,那是注入或环境的问题(22 点的手机该被物理隔离);不成立,走正式修订,写下为什么。

**误用四:阶段变了,prompt 没变(stale prompt)。** 工程版:产品都转型了,system prompt 还在描述旧业务——模型忠实地执行着过时的自我。人类版:Q1 的重心早完成了,原则页还挂着它,**你按惯性继续给旧目标分配注意力**——ADHD 的转段本来就慢(过度聚焦的惯性),过时的锚会把慢放大成搁浅。修法:**每月一次「prompt 修订会」**(日历上占十五分钟):重心还对吗?铁律有没有该退役的?这十五分钟是整个系统的方向盘。

## 边界

同构强度 B 级:system prompt 的工程实践真实,ADHD 的目标维持与提醒依赖有文献基础,Saner.AI 本身无对照研究,「运行原则页」是行为设计。声明:原则页管方向,不管动力——写了「最重要的是完成论文」不会让论文变得可启动,启动系统见本系列前篇;若「什么都提不起劲」持续存在,那不是 prompt 问题,请专业评估。

## 今天就能试的 3 件事

1. **写你的五行 prompt**:重心一条、铁律三条、在改的习惯一条,多一行都不要。
2. **装注入通道**:置顶+设为每天第一眼必见,今天配置好。
3. **在日历上约下月的修订会**:十五分钟,只问「这五行还对吗」。

模型没有 system prompt 会漂,人没有常驻原则也会漂——**区别是模型的 prompt 有工程师保证每轮注入,你的原则只能靠你给它修一条每天必经的路**。五行字,一条路,一月一修——方向这件事,配置好了就不贵。

# ADHD 程序员视角：为什么「治好 ADHD 的冲动下结论、想当然」和「让 LLM 不跑飞」其实是同一道工程题？

> 「这个 bug 我知道是哪的问题」——五个小时后,问题在完全不同的地方。程序员的冲动结论有个残酷的放大器:代码会精确执行你的「想当然」,一行不差。而这个行业恰好也是给不可靠生成外挂验证层的世界冠军——你只需要把给机器的那套,分一份给自己。

ADHD 程序员的「想当然」税单,每一条都有画面感。**debug 冲动**:第一眼锁定嫌疑犯,顺着错误的假设挖五小时(正确流程的第一步——复现和读日志——被兴奋跳过了);**「应该没问题」提交**:小改动不跑测试直接 push(「就改了一行」——生产事故排行榜的冠军句式);**API 想当然**:不看文档,凭直觉调用(「这个参数肯定是这个意思」),埋一个三周后爆炸的雷;**架构冲动**:第一个想到的方案直接开写,三天后发现死路,而多想十分钟就能看到的坑。共同点:**第一反应的置信度和正确率严重脱钩**——这句话同样是对 LLM 幻觉的精确描述,巧合吗?不,这正是同构点:流畅生成的系统,天然缺一个「我可能错了」的信号。

程序员的幸运在于:你的行业已经把验证层做成了基础设施,你只需要**把自己也接进去**:

## 让工具链当你的抑制回路

**测试是你的复述闸**:「就改一行」也跑测试——不是不信任自己,是承认「置信度≠正确率」这个统计事实。测试不全?那就为你最常想当然的模块优先补(哪个模块的事故最多,taxes 都记在 git log 里)。**类型系统和 lint 是你的实时验证**:strict mode、类型标注,让「想当然」在编译期就被戳破,而不是在凌晨的告警里。**CI 是你的最后闸门**:本地没跑的,CI 兜住——把「push 前必须绿」设为铁律,给冲动提交装一道不可协商的物理闸。

## debug:先立三个假设,再动手验证

针对「第一眼锁定嫌疑犯」的专项协议:动手之前,**强制写下三个候选假设+各自的最快验证方法**(scratch 文件里三行)。这个动作打断单一冲动通路(第一直觉只是三分之一了),并强迫你先想「怎么验证」而不是「怎么修」。配套军规:**改任何东西前先复现**——不能稳定复现的 bug,你修的是想象;以及**一次只验证一个假设**(同时改三处,修好了都不知道是哪处修的)。

## Code review:主动采购反调

给自己的 PR 写描述时,强制附一段**「我可能错在哪」**(「这个改动假设了 X;如果 X 不成立,风险在 Y」)——它逼你做一轮 self-critique,还给 reviewer 递上手术刀。没有 reviewer 的个人项目?让 AI 当:「审查这个 diff,重点找我想当然的假设」——**AI 的价值不在它比你强,在它不共享你的确认偏误。**

## AI 编程时代的新税种

用 Copilot/ChatGPT 写代码,冲动风格有了新的表达:**生成的代码看着对就接受**——两个「置信≠正确」的系统首尾相接,错误率相乘。协议:AI 生成的代码一律过三问——它假设了什么?边界条件呢?我能解释每一行吗?**解释不了的行,就是你签了字的想当然。**

公平地说完另一半:ADHD 程序员的快直觉在正确场景里是杀器——事故现场的快速定位、脑暴时的方案喷涌、对「代码坏味道」的动物性嗅觉。**闸门的意义不是压制直觉,是让直觉的输出走「假设」通道而不是「结论」通道**——假设可以快,结论必须过验证。

## 边界

同构强度 A 级:LLM 幻觉与验证层是有直接研究的工程问题,ADHD 冲动性文献扎实,且软件工程的验证基建(测试/CI/review)本身就是为「不可靠生成」设计的——三方同构罕见地整齐。提醒:长期「事故—羞耻—更冲动」循环(出过事故后反而更急于证明自己)是真实的心理模式,若已影响职业信心,值得专业支持;冲动维度对药物反应证据较强。

## 今天就能试的 3 件事

1. **给下一个 bug 立「三假设协议」**:动手前写三个候选+验证法,一次只验一个。
2. **PR 描述加「我可能错在哪」一栏**:从下一个 PR 开始,永久保留。
3. **给 AI 生成的代码立三问**:假设?边界?每行能解释吗?解释不了的不合入。

你天天给不可靠的系统修验证层——模型的、依赖的、用户输入的——却让自己的第一直觉裸奔上生产。把最后这个口子接进你已经建好的基建里:直觉出假设,工具链出裁决。快,但是对——这才是你要的那个组合。

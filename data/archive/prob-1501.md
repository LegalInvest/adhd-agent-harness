# Reflect 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> function calling 那篇讲了 Reflect 的双向链接当「上下文装配」。这一篇讲它的另一个核心部件:每日笔记(daily note)。这个设计朴素到容易被看轻——每天自动生成一页以日期命名的空白笔记——但放在重锚定的框架里,它是一个精确的机制:**每天一次的、带模板的上下文重置点**。LLM 工程里有个对应物:长期运行的 agent 不能只靠一个无限延长的会话,要**定期开新会话、由固定模板重新注入身份与目标**——否则上下文膨胀、旧噪音积累、行为越来越怪。人也一样:没有每日重置的生活,昨天的情绪残渣和未完成事项会一直污染今天。而每日笔记这个重置机制,有它专属的荒废方式。

先立正确形态。每日笔记当重锚定用,靠的是**模板**:每天的新页面自动带上固定结构——比如「今天最重要的三件事 / 本周重心(链接到重心页)/ 随手记区」。前两项是注入(每天早上被迫看一眼方向),第三项是流水的入口(捕捉不设防)。**模板是这个机制的 system prompt:它决定每天的第一屏让你面对什么。** 配合双链,昨天的未完成事项还能自动流转到今天。误用四种:

**误用一:模板空转(重置点退化成空白页)。** 不设模板,每日笔记就是一张白纸——**对 ADHD,白纸不是自由,是又一个「第一步决策」的深渊**(空白页恐惧的日记版),几天后你就不再打开它。修法:今天就配模板,三个区块足够;**模板的意义是让「打开今天」这个动作零决策**——结构已经在那里,你只需要填。

**误用二:模板膨胀(重置仪式变成晨间作业)。** 反向的错:模板越加越长——感恩三则、习惯打卡九项、昨日复盘五问、心情评分——**十五分钟的「晨间填表」,一周后必然弃坑**(对 ADHD,任何超过三分钟的每日固定仪式都在挑战弃坑率)。工程对应:重注入的模板塞成了小论文,每次会话的启动成本高到没人愿意开新会话。修法:模板做减法,**总填写时间控制在九十秒内**——重锚定要的是「每天必然发生」,不是「每天内容丰富」;丰富而中断的仪式,输给简陋而连续的。

**误用三:只写今天,不链重心(重置了上下文,没注入方向)。** 每天认真填「今天的三件事」,但这三件事和本周/本季的重心毫无对照——**每天都在勤奋地重置,重置进一个没有罗盘的新会话**(每步都对、全程漂移的日记版)。修法:模板里放一个死链接——`[[本周重心]]`,每天打开今天时它就在第二行;**三件事写完,和重心页对一眼:至少一件在为它服务吗?**没有,今天就是漂的,趁早上还来得及改。

**误用四:昨天的残留不处理,重置形同虚设。** 昨天没做完的事、没消化的情绪,既没有流转进今天,也没有显式关闭——**它们在后台继续占用工作记忆**(蔡加尼克效应:未完成事项的侵入性思维),新会话开了,旧会话的进程还在跑。修法:模板末尾加一个「昨日清算」微区块——**昨天的三件事,完成的勾掉,没完成的二选一:流转到今天,或写一行「放弃,因为——」**;显式关闭比默默背着轻得多,这三十秒是重置机制的排水阀。

## 边界

同构强度 B 级:会话重置与模板化注入是真实的 agent 工程实践,未完成任务的认知侵入(蔡加尼克效应)与 ADHD 的目标维持困难有心理学基础,Reflect 本身无对照研究。声明:每日笔记是结构仪式,不是情绪治疗——若「昨日残留」主要是反刍性的自我批评且清算不动,那一层值得和专业者工作;工具能给情绪一个格子,给不了它一个出口。

## 今天就能试的 3 件事

1. **配好三区块模板**:今日三件事 / [[本周重心]] 死链 / 随手记——九十秒能填完的规模。
2. **明早执行第一次「对罗盘」**:三件事里至少一件服务于重心,没有就换一件。
3. **今晚做第一次昨日清算**:没完成的,流转或写明放弃——一件不许悬着。

每天醒来,你都获得一次免费的新会话——**但新会话的质量,取决于开场注入了什么:一个九十秒的模板,一条指向重心的链接,一次对昨天的显式清算**。日子可以一天天过,方向不能一天天丢。

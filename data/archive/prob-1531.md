# Forest 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> function calling 那篇讲了 Forest 的树是「资源锁+违约信号」。这一篇看种树前的那两秒。Forest 有个容易被划过去的细节:种树时可以给这棵树打标签——这次专注是为了什么。大多数人跳过它,直接种。但放进重锚定的框架,这两秒恰恰是整个仪式里信息量最高的动作:**种树是一次注入窗口——你在向接下来的二十五分钟声明「这段时间属于什么」**。跳过声明的树,锁得住手机,锁不住方向;这正是 system prompt 工程里「有会话,没指令」的经典漏配。

先看「带声明的锁」和「裸锁」的差别。裸锁(直接种树)保证的是:二十五分钟不碰手机。**但不碰手机≠在做正事**——你可以在锁着的二十五分钟里整理桌面、重读旧邮件、给文档换字体(生产性拖延不需要手机)。带声明的锁(标签写「报告第二节初稿」)多了一层:**这段被锁定的时间,绑定了一个可验证的目标**——树活了但目标没碰,你自己知道这棵树是空心的。工程对应:开了会话、锁了资源,但没下指令——资源被占用,产出无定义。Focusmate 篇讲过对人宣告的力量;Forest 的标签是它的独处版:**对一棵树宣告,弱于对一个人,但远强于不宣告。** 误用三种:

**误用一:种树不打标签(裸锁)。** 修法即上:**标签必写,格式用「完成态」**(不是「工作」,是「回完三封积压邮件」)。两秒的动作,把一段被动的「不玩手机时间」升级成一次主动的「目标锁定」。种树的手指落下之前,问一句:这棵树为谁而活?

**误用二:统计页只看时长,不看标签分布(日志有记录,没被消费)。** Forest 的统计大多数人只看「今天专注了几小时」——**总时长是最没信息量的指标**:四小时全花在低价值杂务上的森林,和两小时全花在本周重心上的森林,树看起来一样绿。工程对应:日志只统计调用次数,不看调用了什么。修法:周复盘时按标签看——**本周的树,种给重心任务的占几成?**低于三成,说明锁是好的、注入是错的:你在用最好的工具保护最不重要的事。

**误用三:把「森林的美观」当目标(锚定物取代了锚定的对象)。** 玩过 Forest 的人都懂那个瞬间:为了让今天的森林满一点,睡前种一棵「随便干点什么」的树。**这标志着系统开始空转:树本来是目标的见证,现在目标成了树的借口**。工程对应(reward hacking 的锚定版):优化指标本身而不是指标代表的东西。修法:允许森林有稀疏的日子——**森林的诚实比森林的茂密值钱**;真的没专注,就让那天空着,空白是数据,填充物是噪音。

一个正向组合收尾,把三篇 Forest 相关的机制串起来:**标签声明目标(注入)→种树上锁(资源锁定)→树的死活给违约信号(反馈)→周末按标签复盘(审计)**——一个九块钱的 App,配齐了一个微型 agent 会话的完整生命周期。用足它,靠的不是功能,是你愿不愿意在种树前多花那两秒。

## 边界

同构强度 B 级:会话级目标声明与资源锁定是真实的工程实践,承诺具体化对执行意图的增益有行为科学基础(实施意图研究),Forest 本身无 ADHD 对照研究。声明:标签化管理的是专注的方向;若被锁定的时间里持续发生「手机没碰但人在神游」,那是唤醒度或任务负荷的问题(见相应篇章)——锁和标签都管不了脑内,脑内的事用脑内的方法。

## 今天就能试的 3 件事

1. **今天的每棵树都打标签**:完成态格式,种前两秒写好。
2. **看一次标签分布**:本周的专注时长,重心任务占几成?
3. **允许一天空白**:真没专注的日子,不种装饰树——让森林保持诚实。

锁住时间只是下限,声明方向才是上限——**一棵知道自己为什么而活的树,才配得上你那二十五分钟**。种树之前的两秒钟,是整个仪式的灵魂。

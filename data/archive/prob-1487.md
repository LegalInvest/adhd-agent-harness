# 如果把 ADHD 大脑当成生成核心，ChatGPT 能不能当它的重锚定与目标漂移？

> 长任务里的 agent 有个著名死法:每一步都在认真执行,但第十五步时它做的事,和最初的目标已经没什么关系——不是哪一步出了错,是每一步都漂了一点点。工程叫它目标漂移(goal drift),解法叫重锚定(re-anchoring)。ADHD 的一天、一周、一个项目,死法一模一样:没有偷懒,每一刻都在忙,但周五回看,忙的东西和周一的意图早已分道扬镳。那么,ChatGPT 能不能当那个把你拉回来的锚?

先看漂移的机制,两边惊人地对称。agent 侧:**目标只在最初的 prompt 里出现一次,随着上下文被后续内容填满,目标的「注意力权重」被稀释**——最近的消息永远比开头的指令更响亮,于是 agent 越来越服从「眼前最近的刺激」,渐渐忘了「最初为什么出发」。工程解法不是让模型「更专注」,是**机械性地重注入:每隔 N 步,把原始目标重新写进上下文,并要求模型显式对齐——「你当前做的事,和目标的关系是什么?」** 这个廉价的仪式,是长任务成功率最大的单点提升之一。

ADHD 侧:意图(「这周把作品集做完」)形成于周一早上,但**它没有被存进任何会持续发声的地方**——而接下来的一周里,每封邮件、每条消息、每个突发,都是「最近的刺激」,都比周一的意图响亮。ADHD 对「眼前刺激」的敏感与对「远期意图」的钝化,让漂移速度比常人更快——**你不是放弃了目标,是目标在注意力竞争里被静音了**。解法同样不该是「更自律」,该是重锚定的机械化。

ChatGPT 在这套机制里能当什么?它有一个特别适配的位置:**锚的「对齐审查员」**。单纯的重锚定(重读一遍目标)人也能做,但 ADHD 常见的失败是:重读了,点点头,继续漂——**因为「我当前的活动和目标是否对齐」这个判断,自己对自己做,太容易糊弄过去**。而把这个判断外包给对话,糊弄的难度显著上升:

**一、周初,把目标存进锚对话。** 固定一个对话,周一早上写入:「本周目标:X。为什么重要:Y。判断完成的标准:Z。」——**「为什么」和「完成标准」必须写**,它们是后面对齐审查的依据;只有目标名字的锚,审查时无从对齐。

**二、每天一次「对齐审查」,让它来问。** 每天固定时点(日历触发,别靠想起来),打开锚对话汇报:「今天我做了 A、B、C。」然后让它执行审查:「对照本周目标,A、B、C 各自的贡献是什么?哪些是漂移?」——**它的价值不在智力,在立场:它不心疼你的沉没成本,不给你的漂移找借口。**「C 和目标没有可见关系」这句话,自己对自己说很难,它说出来很容易。

**三、漂移分类处理,不一刀切。** 审查出的漂移分三种:**噪音型**(刷手机滑进去的)——明天用环境手段防;**假紧急型**(别人的着急变成你的优先级)——练习「今天不做会怎样」的反问;**信号型**(你反复漂向同一个方向,比如总是溜去做另一个项目)——**这可能不是漂移,是真实偏好在投票**,值得升级成一次正式的目标修订,而不是一次次「纠正」。锚的意义是让漂移可见,不是把一切漂移都定罪。

**四、周五,合账。** 让它汇总一周的审查记录:「本周对齐率大概多少?最大的漂移源是什么?下周的锚要怎么改?」——锚本身也要迭代:一个每周都守不住的目标,问题常在目标定得虚,而不在守的人。

## 边界

同构强度 B 级:目标漂移与周期性重注入是真实的 agent 工程实践,ADHD 的意图-行为间隙与延迟折扣有文献支撑,「锚对话+对齐审查」是行为设计,无对照研究。声明:持续的方向感缺失若伴随情绪低落或「做什么都没意义」,那可能不是漂移问题而是情绪问题,请专业评估;目标管理工具不评判人生方向的对错——反复漂向的地方,有时才是该去的地方,这个判断请留给你自己和信任的人。

## 今天就能试的 3 件事

1. **建锚对话**:写入本周目标+为什么+完成标准,三行。
2. **设每天一次的对齐审查提醒**:汇报三件事,让它指出漂移,全程五分钟。
3. **给漂移建三分类**:噪音/假紧急/信号,周五看看哪类最多——那就是你下周的主防对象。

漂移不是意志的失败,是架构的缺席:**目标只说一次,就等于没说**。给它一个每天发声的锚、一个不讲情面的审查员、一套区分噪音与信号的分类法——周五的你和周一的你,就能重新变成同一个人。

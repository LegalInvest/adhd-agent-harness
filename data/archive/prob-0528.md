# 3 个 LLM 工程概念，彻底重写了我对 ADHD 注意力容易被环境带偏的理解

> LLM 的输出质量由上下文窗口里放了什么决定——所以工程师不劝模型「专心」，他们管理喂进去的东西。这是对「易被环境带偏」最实用的重新定义。

打开电脑准备写方案，看到浏览器上还开着昨天的购物页面，二十分钟后你在比较三款吸尘器。ADHD 者对这种「被带偏」太熟悉了，熟悉到默认接受了主流解释：注意力不行，意志力不够。但 LLM 工程对同一现象的处理方式，暴露了这个解释的荒谬。

## 1. 上下文决定输出：模型没有「意志力」，人也几乎没有

LLM 有个铁律：输出质量基本由上下文窗口的内容决定。窗口里塞了无关信息，模型就会被带偏——不是它「不自律」，是注意力机制本来就会对窗口内所有 token 分配权重。工程师的应对从来不是提示模型「请专注」（这句话本身也只是又一个 token），而是**上下文工程**：严格控制窗口里出现什么。

对 ADHD，环境就是上下文窗口。桌上的手机、屏幕角落的红点、开着的八个标签页——每一个都在被你的注意力机制分配权重，无论你「想不想」关注它。冲突任务研究显示，Transformer 注意力在 Stroop 类干扰下的崩溃模式与人类执行功能缺陷同源（Deficient Executive Control in Transformer Attention, 2025）：干扰项一旦进入窗口，抑制它的代价对两种系统都是高昂的。结论很朴素：**赢在窗口装配，别赌窗口内的对抗。**

## 2. 确定性控制流：把「不该出现的选项」在结构上删掉

agent 工程里，要求高可靠的环节不交给自由生成，而是用确定性控制流锁死：这一步只能调用这个工具，输出只能是这几种格式。自由度是留给需要创造力的环节的，不是撒在整条流水线上。

ADHD 的环境改造完全同理。写作时段的电脑可以只允许存在一个全屏窗口（结构性删除其他选项），手机可以物理放在另一个房间（把「刷一下」的调用路径从一次触摸变成一次起身）。重点是这些都发生在**开始工作之前**——上下文装配是冷静时干的活，指望工作中途靠意志力清窗口，等于让被带偏的模型自己纠偏。环境设计的全部秘密就一句：把对抗前置为装配。

## 3. 注意力锚点：与其清空干扰，不如放一个高权重信号

上下文工程不只做减法。工程师会把关键指令放在窗口的显著位置（开头或结尾），甚至定期重复注入，让它始终占据高注意力权重——因为空窗口做不了事，窗口里必须有一个足够强的正确信号。

ADHD 版锚点：当前任务用大字写在屏幕旁（不是待办清单，是唯一的当前任务）；Brain.fm 这类功能性音频的原理也是锚点——用一个恒定、无信息量的听觉信号占住感官通道，让随机环境噪音挤不进来。被带偏的反面不是「空白的专注」，是被正确的东西占住。

## 边界

同构强度 B 级：机制类比，双侧文献真实。但要防一个滑坡：环境决定论不等于环境万能论——ADHD 的内源性走神（没有任何外部刺激也会神游）是环境工程覆盖不到的部分，那一层需要的是任务本身的刺激度设计（新奇、竞速、即时反馈），甚至药物。环境工程管住的是「被带偏」，管不住「自己飘走」。

## 今天就能试的 3 件事

1. **做一次窗口审计**：现在环顾你的工作区，列出所有正在分走注意力权重的东西。逐个问：它有资格在窗口里吗？
2. **前置一次装配**：明早开始工作前，花 5 分钟只干一件事——关掉一切、只留一个窗口、手机放到另一个房间。然后才允许开始。
3. **立一个锚点**：一张便签，写当前任务，贴在屏幕边缘。飘走后回来的第一眼要能看到它。

「容易被环境带偏」是真的——但它的正确名字是「窗口对什么进来没有防御」。防御不靠意志力，靠装配规则。工程师早就不劝模型专心了，你也可以从今天开始不劝自己。

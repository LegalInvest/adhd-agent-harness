# ADHD 职场人视角：为什么「治好 ADHD 的分心走神或超聚焦跑偏，丢失目标」和「让 LLM 不跑飞」其实是同一道工程题？

> 下午五点，你干了一整天，累得真实，但年度目标相关的进度是零——这种「勤奋的漂移」正是 agent 跑长任务时最典型的死法。解药两边通用：外部目标锚 + 定时对齐。

职场版的目标丢失分两种，表面相反，机制相同。第一种是发散型：早上明明计划写季度汇报，中途查一个数据，顺手回了三条消息，又被拉进一个临时会——下班时汇报文档还停在标题页。第二种是收敛型：你超聚焦在优化一个内部工具上三天,爽感十足——但绩效考核里没有这一项，你真正欠的交付还在欠着。

共同点：**每一步都在「工作」,整体却在漂移**。你不缺努力,缺的是把每一步和目标对齐的机制。

## agent 的同款死法和解法

长链路 agent 的经典故障就是这个：执行第 15 步时,「当前子任务」已经悄悄替代了「原始目标」——每一步局部合理，全局跑飞。工程解法成熟且朴素：**目标外置**（原始目标存在上下文之外的固定位置，不依赖模型自己记得）+ **定时重对齐**（每 N 步暂停,将当前动作与原始目标做一次显式比对）+ **偏航预算**（允许限定范围的探索,超预算强制回归）。

注意工程师没做什么：没有试图让模型「更专注」。漂移被当作长链路执行的物理属性来管理,不是当作品德来纠正。

## 职场版实现

**目标外置**：本周三个关键交付,写在物理上无法回避的位置（显示器边框便签/手机锁屏）。注意粒度——不是年度目标（太远,没有牵引力）也不是今日待办（太碎,本身就会漂移）,**周粒度是对齐的最佳刻度**。

**定时重对齐**：给自己装两个对齐点——午饭前和下午四点,各 60 秒,只问一个问题：「我今天做的事,和便签上那三件的关系是什么？」偏了不自责（自责消耗的能量比纠偏本身还多）,只做一个动作：下一个小时回到三件事之一。**对齐的价值不在于每次都发现偏航,在于把最大偏航时长压缩到半天以内。**

**偏航预算**：给自己每天预留一块合法的发散时间（比如下午 3:00–3:45）,冲动来了先记进「偏航停车场」清单,到点再去。这招的原理是：ADHD 的冲动很难压制,但可以**延迟**——而延迟到预算时段的冲动,一半以上会自己失效。

**超聚焦的专用护栏**：收敛型漂移更隐蔽,因为它伪装成高产。护栏是入口处的一个问题：开始任何预计超过 2 小时的深挖前,问「这件事在我的周便签上吗？」不在,给它 25 分钟封顶（设物理计时器）,到点必须做去留决策——要么正式申请把它排进计划（跟 leader 说一声,让它合法化）,要么放手。

## 一个职场专属的杠杆：借用组织的对齐机制

你所在的组织其实自带对齐基础设施——周会、周报、1on1。多数人把它们当负担,ADHD 员工应该把它们当免费的重锚定服务：写周报的真正受益人是你自己（强制一次目标比对）；1on1 主动汇报「本周三件事」的进度（让 leader 成为你的外部目标锚）。**别人有内置的对齐回路,你有制度化的——效果一样,还显得职业。**

## 边界

同构强度 B 级：agent 目标漂移与重锚定是真实工程实践,ADHD 的目标维持缺陷有执行功能文献支撑。限制：如果漂移的方向长期一致地指向某类任务（你总是漂去做设计而回避写文档）,那不只是注意力问题,是兴趣信号——值得在职业规划层面听一听,而不只是在执行层面纠正。

## 今天就能试的 3 件事

1. **现在写周便签**：本周最重要的三件交付,贴上显示器。两分钟。
2. **设两个对齐闹钟**：12:00 和 16:00,标签就叫「和便签对齐」。
3. **建偏航停车场**：一个便签或文档,今天所有「突然想干的事」先进停车场,15:00 统一处理。

勤奋从来不保证方向。agent 工程花了两年学会这件事：让系统跑得快之前,先让它每隔几步抬头看一眼目的地。你比 agent 幸运——你的目的地,一张便签就装得下。

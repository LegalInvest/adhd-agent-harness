# 为什么用 Brain.fm 治 ADHD 的任务启动困难，和给 agent 套 function calling 工具调用 是一回事？

> function calling 系列的第四篇,轮到一个容易被小看的环节:执行环境。工程师都遇到过这种事:函数注册没问题、递送没问题、schema 没问题——调用还是失败,查到最后,是运行环境不对:依赖没装、权限不够、运行时状态污染。工具调用不是发生在真空里,**它需要一个已就绪的执行环境**。ADHD 的启动同理:任务打包好了、递到眼前了、格式也友好——可你的神经系统此刻的「环境」不对:唤醒度太低(昏沉)或太高(烦躁),调用依然点不着。Brain.fm 治的就是环境这一层。

先看工程侧的「环境准备」。成熟的 agent 框架在执行工具调用前有一步 setup:检查依赖、初始化运行时、隔离干扰(沙箱)——**同一个调用,在就绪的环境里一次成功,在脏环境里反复报错**。所以「调用失败」的排查手册里,环境永远是一章:不是函数的错,不是模型的错,是执行现场没有准备好。

ADHD 侧的对应变量,是**唤醒度(arousal)**。启动需要神经系统处在一个「可点火区间」:太低(无聊、昏沉——ADHD 的默认态之一,低唤醒理论认为这正是刺激寻求行为的来源)点不着;太高(焦虑、过载)也点不着。**很多「就是开始不了」的时刻,任务系统毫无问题,是唤醒度不在区间里**——而 ADHD 的唤醒调节恰恰比常人更不稳定,更需要外部手段来「初始化环境」。刷手机之所以是启动的头号竞品,一个隐藏原因就是它是最便宜的唤醒度调节器(无聊时刷=提升刺激,焦虑时刷=转移),只是它调节完不把你还给任务。

Brain.fm 这类功能性音频,本质是一个**不劫持注意力的唤醒度调节器**:为专注设计的节奏化音频(该产品的调制技术有一些初步的 EEG 研究支撑,证据等级见文末),提供稳定、无歌词、无新异刺激的听觉流——**足以把唤醒度托进可点火区间,又不像音乐 App 那样每三分钟用一首新歌打劫你的注意力**。按「环境初始化」的逻辑用它:

**一、把音频当「启动仪式」的固定第一步。** 耳机戴上→音频响起→打开任务——固定这个顺序,几周后音频本身会变成条件化的点火信号(环境初始化脚本,每次都跑同一个)。**仪式的价值一半在声学,一半在「这个声音=进入工作态」的条件反射。**

**二、按当前状态选模式,像选运行时参数。** 昏沉的下午→节奏感强的(把唤醒度往上托);烦躁的早上→平缓的(往下压);**先花三秒判断「我此刻是太低还是太高」,再选**——环境准备的前提是知道环境现在什么样。这三秒的自我觉察,本身就是被低估的技能。

**三、音频≠背景,开着≠有效。** 两个失效姿势:一边放专注音频一边刷手机(环境就绪了,调用没发生——音频不是护身符,戴上耳机的下一步必须是打开任务);音频里夹杂歌词歌曲(每句歌词都是一次微注入,那是污染环境,不是准备环境)。

**四、给「环境无效」留诊断分支。** 音频开了、任务打开了,还是不行——别加大音量,换排查方向:是没吃饭/没睡够(生理层)?是这个任务有情绪附着(恐惧层)?还是今天整体状态崩(接受它,换最小任务)?**环境调节是排查手册的一章,不是全书。**

## 边界

同构强度 C 级:如实标注——唤醒度与表现的关系(倒 U 曲线)是经典心理学,ADHD 的唤醒调节异常有理论与部分实证支撑,但 Brain.fm 类产品的功效证据还比较初步(小样本 EEG/行为研究,部分由厂商资助),「环境初始化」是功能类比。声明:音频是温和的辅助,效果个体差异大——试一周没感觉就停,不必怀疑自己;若昏沉/烦躁的底层是睡眠问题、情绪问题或药物剂量问题,请优先处理那一层,声音盖不住生理。

## 今天就能试的 3 件事

1. **建立你的点火仪式**:耳机→音频→打开任务,固定顺序,今天跑第一遍。
2. **练「三秒唤醒度判断」**:启动前问自己「此刻太低还是太高」,按答案选音频模式。
3. **做一次 A/B**:同类任务,一天用功能性音频,一天用带歌词的歌单,比较进入状态的速度。

调用失败,先查环境——**工程师不骂函数,他们检查运行时;你也别急着骂自己,先看看唤醒度在不在点火区间**。一副耳机当然治不了 ADHD,但把执行环境初始化好,常常就是「点不着」和「点着了」之间,那一根火柴的差距。

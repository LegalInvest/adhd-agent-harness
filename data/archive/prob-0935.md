# 如果把 ADHD 大脑当成生成核心，ChatGPT 能不能当它的幻觉与验证循环？

> 这个问题把本系列常用的架构图转了一个方向。通常我们说:ADHD 大脑=强生成核心,缺的是调度层,用工具补。但调度层里有一个特殊组件,值得单独拎出来:**验证器**。LLM 世界对它的需求源于幻觉——生成核心会流畅地输出错误,而且错误与正确在「输出的自信程度」上无法区分,所以可靠系统必须在生成之外挂验证循环(自检、交叉核对、工具校验)。ADHD 大脑有没有自己的「幻觉」?有,而且形态惊人地像:**计划幻觉**(「明天一天就能做完」——流畅、自信、错误)、**记忆幻觉**(「我确定我回过这条消息」)、**能力幻觉与灾难幻觉的双向摆动**(「我肯定搞得定」与「我肯定搞砸」都以同样的确信度出现)。那么,ChatGPT 能不能反过来,当这个生成核心的验证循环?

答案是:**能当验证循环的一半,而且恰好是人自己做不了的那一半**。拆开看验证循环的两个功能:**其一,一致性检查**——把你的计划/判断说给 ChatGPT,让它专门挑刺:「我打算明天一天写完这份报告,这是我的任务清单,帮我检查这个时间估计哪里不现实」——它不会累、不会给面子、不会被你的自信感染,而**「不被生成者的确信度感染」正是验证器的核心资格**:你自己复查自己的计划,用的还是那颗产生计划幻觉的大脑(自我监控是执行功能的一部分,受损的恰是它);ChatGPT 是系统外的另一个采样源,**外部性本身就是校验效力的来源**(人在回路篇的原理,这里的「人」换成了模型)。**其二,结构化的质询**——让它扮演怀疑论者:「假设这个计划会失败,最可能失败在哪一步?」——预演式验尸(premortem)是行为决策研究里对抗计划谬误的有效技术方向,而 ADHD 者独自做 premortem 几乎不可能(需要工作记忆里同时挂着计划与反事实),ChatGPT 把这个昂贵的认知操作变成了一次对话。

但另一半,它当不了,而且这一半是致命的:**ChatGPT 自己也是一个会幻觉的生成核心**。用它验证事实性内容(日期、数字、引用、法条),等于用一个幻觉系统校验另一个幻觉系统——两个采样源的错误虽然不相关(这是有价值的:两者矛盾提示你去查),但**两者一致也不构成正确的证据**(可能一起错得很顺滑)。所以架构上的正确用法是三层:**你生成 → ChatGPT 做一致性与结构质询(它擅长:逻辑漏洞、时间估计、遗漏项)→ 事实层交给硬验证**(日历上的真实空档、文件里的原始数据、权威来源)——**ChatGPT 是软验证器,负责「这说得通吗」;硬验证器负责「这是真的吗」;两层不可互相替代**。这恰好也是 agent 工程的分层:LLM-as-judge 管质量,单元测试管正确性。

落地为日常协议:**①计划过检**——任何超过半天的计划,发给 ChatGPT 加一句「假设我惯常低估 50% 时间,帮我挑出最不现实的三处」;**②决定冷却**——冲动决定(购买/承诺/辞职式发言)先写给它,要求「列出支持与反对各三条」——不是让它替你决定,是强制生成核心过一遍反事实;**③幻觉不对称原则**——它挑出的「结构问题」(漏了依赖、时间冲突)直接采纳;它给出的「事实陈述」一律标记为待查。边界:**其一,验证循环有成本**——每个决定都过检会瘫痪(和过程监督篇的成本警告同构),只给高风险决定上这道闸;**其二,别让软验证器变成决策外包**——「ChatGPT 说可以」不是你决定的理由,验证器提供信息,决策权与责任在生成核心(你);**其三,同构是架构层面的**——大脑的元认知与 LLM 的自洽性检查机制不同,共享的是「生成者不能自证,外部采样才有校验力」的结构。

## 边界

同构定位(本文未做正式 A/B/C 分级):计划谬误与 premortem 有行为决策研究基础,LLM-as-judge 与验证分层是 agent 工程实践,「外部采样源做软验证」的结构对应清晰;ChatGPT 用于 ADHD 决策辅助无对照研究。声明:重大决策(医疗/法律/财务)的验证必须落在专业人士与权威来源上;本文不构成医疗建议。

## 今天就能试的 3 件事

1. **给明天的计划过一次检**:把任务清单发给 ChatGPT,问「哪个时间估计最不现实?」——见识一下不被你的自信感染的挑刺。
2. **立幻觉不对称规则**:AI 指出的结构问题可信,AI 陈述的事实必查——写进你的使用习惯。
3. **做一次 premortem**:正在推进的项目,问一句「假设它三周后失败了,最可能因为什么?」——把反事实思考外包给对话。

生成核心自证自己的计划,就像模型给自己的输出打满分——**校验力来自外部性**。ChatGPT 当不了真理的裁判,但当得了那个不给你面子的挑刺者;真理,留给硬验证。

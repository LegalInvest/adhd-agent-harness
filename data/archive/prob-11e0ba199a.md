# 为什么用 Endel 治 ADHD 的任务启动困难，和给 agent 套 function calling 工具调用 是一回事？

> Brain.fm 那篇讲过「执行环境初始化」:调用失败,先查环境,唤醒度不在点火区间,音频把它托进去。但那套方案留了一个手动挡的尾巴:你得自己判断「此刻太低还是太高」,自己选模式。这一篇讲自动挡——Endel 的赌注是把环境准备这件事也自动化:**不是你去配置环境,是环境根据你的状态自己配置自己。** 工程上这对应一个更高级的形态:带反馈的自适应运行时。

先看工程侧的升级路线。执行环境的管理有三代:**手动配置**(每次跑之前人肉装依赖调参数)→**声明式配置**(写好配置文件,一键初始化)→**自适应运行时**(系统监控负载与状态,自动伸缩、自动调参)。第三代的核心是**闭环**:环境不再是静态的舞台,是一个持续读取信号、持续自我调整的活系统。为什么要闭环?因为负载在变——固定的配置对付不了变化的现场。

ADHD 的唤醒度,恰恰是全天大幅波动的负载曲线:早晨的昏沉、午后的低谷、傍晚的莫名烦躁、深夜的反常清醒——**固定的「专注歌单」对付不了这条曲线**,九点管用的声音,三点可能正好相反。而让用户手动追踪自己的状态并换模式(Brain.fm 篇的「三秒判断」),对 ADHD 是个诚实但脆弱的方案——自我觉察恰恰是波动最大时最先掉线的能力。

Endel 的设计就是把这个闭环自动化:读取时间、天气、心率(接手表)、运动状态,**实时生成连续变化的音景**——不是播放曲目,是根据信号流持续合成;早晨自动偏激活,深夜自动偏舒缓,你走动时和静坐时听到的不一样。它相当于一个自适应的环境守护进程:**你不再操作它,它跟着你跑。** 按「自适应运行时」的逻辑用:

**一、把它设成「默认开着的背景层」,而不是「想起来才开的工具」。** 自适应系统的价值在持续在场——工作时段让它常驻(专注场景),它的调节是渐进连续的,开一分钟关一分钟等于没有闭环。**手动挡工具要你记得用,自动挡的意义就是不需要被记得。**

**二、接上可穿戴,给闭环喂真数据。** 没有心率信号,它只能按时间和日历猜;接上手表,「烦躁时自动放缓、萎靡时自动激活」才真的闭环——**反馈系统的质量上限就是传感器的质量。**

**三、留一个手动覆盖开关。** 自适应不等于全对——它判断你该舒缓,而你正需要冲刺,那就手动切场景。**好的自动化都留人工覆盖通道,你的裁决权高于算法**,这条原则从日历工具到音景工具通用。

**四、别指望它单独扛启动。** 它管的是环境层——环境就绪之后,启动还需要打包好的任务、递送的时机、点火的动作(本系列前几篇)。**自动挡的环境+手动的点火,这是合理分工;只戴耳机不打开任务,音景再智能也只是高级白噪音。**

## 边界

同构强度 C 级:如实标注——自适应系统是真实的工程范式,声音对唤醒度的调节有一般性心理声学依据,但 Endel 的个性化音景对 ADHD 启动的功效缺乏独立对照研究(相关研究多为小样本或厂商关联),「环境守护进程」是功能类比。声明:效果个体差异大,有人觉得连续音景舒适,有人觉得空洞烦人——试一周,无感就停;若状态波动的底层是睡眠、情绪或药物问题,先处理那一层,声学是锦上添花,不是雪中送炭。

## 今天就能试的 3 件事

1. **今天工作时段让自适应音景常驻**:体验「不操作它」的环境层。
2. **接上你的手表**(如果有):给闭环喂心率,对比接与不接的差别。
3. **记录三个时点的状态**(早/午/晚):看它的自动调节和你的真实曲线合不合——不合就用手动覆盖教它。

环境准备的终极形态,是你感觉不到它的存在——**它读你的信号,悄悄把房间调到可点火的温度**。启动依然是你的事,但从今以后,至少别再输在环境上。

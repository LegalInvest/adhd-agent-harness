# ADHD 创业者视角：为什么「治好 ADHD 的冲动下结论、想当然」和「让 LLM 不跑飞」其实是同一道工程题？

> 创始人的冲动结论自带执行力和传染性：你「想当然」的第二天，团队已经在为它写代码。给高权限 agent 加行动闸门的那套工程，创始人应该原样抄走。

员工的错误结论会被流程拦截，创始人的错误结论**就是流程**。你在电梯里对产品经理说「我觉得用户肯定更想要 X」，一周后 X 出现在了排期里——没人核实过这个「肯定」，因为它出自你口。ADHD 创始人的冲动性（快速拍板、想当然的市场判断、听完半个方案就说「就这么干」）因此有了组织级放大器。

更麻烦的是幸存者偏差的自我强化：你过去确实有几次「直觉神准」的战绩（ADHD 的快速模式识别是真实优势），这几次战绩让你和团队都高估了直觉的平均命中率。**你记得住三次神准，记不住十七次悄悄被团队消化掉的误判。**

## 高权限 agent 的工程纪律

agent 工程里权限和验证严格成正比：只读 agent 随便跑，能写数据库的要过审批，能花钱、能对外发消息的必须 human-in-the-loop。原则是：**输出的不可逆性越强，生成和执行之间的闸门越厚**。没人会因为模型「大部分时候是对的」就撤掉支付确认。

创始人是公司里权限最高的 agent，你的结论按不可逆性分级，配不同厚度的闸门：

**可逆决策**（试一个渠道、改个文案）：保持你的速度，直接执行。快速试错本来就是 ADHD 创始人的比较优势，别把闸门装在这里——过度流程化等于自废武功。

**难逆决策**（招关键岗、定价、砍功能）：一夜规则 + 一个反对者。24 小时冷却，且决策会上必须有人被指定扮演反方（红队角色轮流坐庄，防止真实反对者被创始人气场压制）。

**不可逆决策**（联创股权、融资条款、裁员、公开承诺）：强制外部校验——律师、财务顾问、或一位没有利益关系的资深创始人。这一档你的直觉无论多强都不算数，因为样本量太小、代价太大，任何概率引擎在这里都必须让位给确定性流程。

## 「想当然」的专项防御：假设审计

创始人的想当然最贵的形态不是拍板太快，是**关键假设从没被识别为假设**：「用户肯定愿意为这个付费」「这个技术两个月肯定能做出来」「竞品肯定跟不上」。这些句子在你脑内的状态是事实，在现实里的状态是未验证假设。

抄精益创业加一层 ADHD 适配：每个「肯定」写下来,变成一张假设卡——**内容 + 验证方式 + 验证成本**。你会发现多数「肯定」用一天和五个用户访谈就能验证,而你差点跳过验证直接烧三个月开发。让团队也持有这个协议：任何人听到你说「肯定」，有权问一句「这是假设卡几号？」——把纠错的社交成本降到最低，是创始人能给团队的最好授权。

## 边界

同构强度 B 级：agent 权限分级是真实工程纪律，ADHD 冲动性与创业者决策风格的研究都存在，但「ADHD 创始人直觉命中率」无严格测量——本文的分级框架本身就是对这种不确定性的对冲。另外，过度校验同样杀公司（贻误窗口期），闸门厚度要跟不可逆性走，不要跟焦虑走。

## 今天就能试的 3 件事

1. **给最近十个决策补分级**：可逆/难逆/不可逆各几个？闸门配对了吗？错配清单就是你的风险图。
2. **写三张假设卡**：把你当前战略里的三个「肯定」变成假设卡，各配一个一周内能完成的验证动作。
3. **任命本月红队**：下次难逆决策会，指定一人专职反对。会后谢谢 ta，当众。

你的直觉是公司最快的引擎——它值得保留在所有可逆的路面上狂奔。但方向盘上那几个焊死的按钮，是让它能一直狂奔下去的原因。

# Focusmate 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> Focusmate 的启动价值(真人协议、身体在场)前面讲过了。这一篇看它藏得更深的一个机制——每场会话开始时,有一个固定环节:**你要对陌生人说出「接下来五十分钟我要做什么」**。这三十秒经常被当成客套,但它恰恰是整场会话的技术核心:一次带见证人的重锚定。工程上,这对应 system prompt 管理里最有效的技巧之一:**在关键节点让模型显式复述自己的目标**——不是工程师再说一遍,是让模型自己说,说出来的目标遵循率远高于被动接收的。而这个环节,有它专属的糟蹋法。

先看「说出来」为什么比「想到了」强。工程侧:长任务的 agent 每隔若干步复述一次「我的目标是 X,当前进展到 Y」,漂移率显著下降——**生成即注入**:模型自己输出的内容会成为后续生成的强上下文,比藏在开头的指令权重更高。人类侧有平行的机制:对他人做出的公开承诺,兑现率高于对自己的默念(承诺一致性是行为科学的经典发现);而且**语言化本身强迫你把模糊意图压缩成具体陈述**——说不清楚的目标,是还没打包好的目标,这三十秒顺便当了打包质检。误用四种:

**误用一:宣告含糊(垃圾进,垃圾出)。** 「我要处理一些工作」——这种宣告注入了等于没注入:它无法被违反,因此也无法锚定任何东西。工程对应:让模型复述目标,它输出「我将继续完成任务」——复述空转。修法:**宣告必须包含可验证的完成态**——「写完报告第二节的初稿,大约八百字」;结束时的汇报环节能对上账的,才算一次真锚定。

**误用二:宣告和真实优先级脱节(锚错了目标)。** 你最该做的是那份怕了三天的报告,但宣告的是「整理邮件」——因为后者说出来没压力。**于是重锚定的全部力量,被用来锚定一个逃避性目标**:会话很专注,专注地做着不该现在做的事(生产性拖延的仪式化版本)。工程对应:复述机制工作正常,但复述的目标本身在规划层就选错了——锚定放大执行,不校验方向。修法:**订会话之前先过一遍「今天最重要的事」清单,宣告默认绑定第一条**;想宣告别的,先回答「为什么不是第一条」。

**误用三:锚定只做开头一次(单次注入的衰减)。** 五十分钟对 ADHD 不短,第二十分钟漂移到别的事很常见——**开场的宣告到那时已经衰减殆尽,而下一次锚定要等结束汇报**。工程对应:只在会话开头注入一次,长上下文的后半段照样漂。修法:利用中点——很多人用五十分钟会话,**在第二十五分钟给自己设一个无声闹钟:十秒钟自问「我在做宣告的那件事吗」**;偏了就归位,这就是人肉的 mid-context 重注入。

**误用四:把汇报环节做成表演(反馈通道造假)。** 结束时对搭档说「完成得很好」——实际上二十分钟在漂。没人会戳穿你,但**你污染的是自己的反馈回路**:锚定系统从此收不到真实的偏差信号,下次的宣告粒度、时长设置都失去校准依据。工程对应:评估日志造假,调优全部失灵。修法:汇报讲真话,难听也讲——「目标完成一半,中间漂了十五分钟」;搭档不在乎,你的系统在乎。

## 边界

同构强度 B 级:目标复述降低漂移是真实的 agent 工程实践,公开承诺效应与语言化的自我调节功能有行为科学基础,Focusmate 的同伴工作模式有初步研究支持但对照证据有限。声明:宣告机制强化执行,不解决「最重要的事为什么让人恐惧」——持续用锚定回避高威胁任务,说明那个任务需要的是拆解或情绪层面的处理(见相应篇章),必要时寻求专业支持。

## 今天就能试的 3 件事

1. **重写你的宣告模板**:「完成态+数量」格式(写完 X 的 Y 部分,约 Z 字/项)。
2. **给下一场会话设中点闹钟**:第二十五分钟,十秒自检「在做宣告的事吗」。
3. **今天的汇报讲真话**:如实报告完成度和漂移时长——给自己的反馈回路排一次毒。

对另一个人说出目标的那三十秒,是整场会话里杠杆最高的三十秒——**说得具体,它是锚;说得含糊,它是客套;说了假话,它是毒**。锚定的技术含量,全在你张口的那句话里。

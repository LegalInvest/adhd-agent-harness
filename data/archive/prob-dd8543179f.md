# 为什么用 Reflect 治 ADHD 的注意力涣散，和给 agent 套 上下文工程 是一回事？

> ADHD 的思维不走直线,走网:一个念头牵出三个,三个牵出九个——这既是创造力的来源(生成核心的过剩,Goblin 篇),也是涣散的引擎:每个岔路都在诱惑当场跟进,不跟进又怕丢。传统的线性笔记(一页一页、一层一层文件夹)和这种认知风格根本不同构,强行使用等于每天做格式转换。Reflect 这类反链笔记(backlinked notes)的架构——**笔记之间双向链接、知识以网而非树的形态生长**——第一次让外部记忆的形状贴上了 ADHD 认知的形状;而在 agent 工程里,这个架构有个如日中天的对应物:**知识图谱增强的上下文(GraphRAG 式检索)——沿关系边扩展相关上下文,而不是只按孤立相似度捞**。

先讲「形状匹配」为什么重要。认知卸载篇的公式:外部记忆的价值=写入质量×检索可靠性——但两者其实都被同一个东西决定:**外部结构与内部认知的转换成本**。ADHD 的联想式思维往树状系统里写,每次都要把网状的想法压平成线性条目(转换成本高→写入失败);检索时,你的大脑按联想找(「和那个定价想法有关的……」),树状系统按路径找(哪个文件夹来着?)——**两套寻址方式,注定互相找不到**。反链系统的写入是「记下来+顺手链到相关笔记」,检索是「打开任一相关点,沿链走到目标」——**写入贴联想,检索也贴联想,转换成本双向归零**。

Agent 侧的平行演化严丝合缝:纯向量检索(Mem 篇讲的)按孤立相似度捞上下文,常漏掉「不相似但相关」的关键信息(概念 A 与概念 C 不相似,但都连着 B);图增强检索的改进正是**沿显式关系边多跳扩展**——检索「定价」时,顺着边把「上次的用户访谈」「竞品笔记」一并装配进上下文。**关联不是内容的属性,是结构的属性**——这句话两个领域各自领悟了一遍:agent 工程用图结构存关系,反链笔记用双向链存关系,而 ADHD 大脑,本来就用关系思考。

落到涣散的治理,反链系统给了三个具体机制:**①岔路的停车位**——写作或工作中冒出的关联念头,一条链接甩进网里(「这个想法连着那个项目」),两秒完成,不用当场跟进也不怕丢——**「跟进冲动」和「丢失恐惧」这对涣散双引擎,被一个动作同时熄火**(对应 agent 的「相关但非当前的内容,存图不进窗」);**②每日笔记当锚点**——Reflect 的 daily note 是天然的时间锚:今天的所有捕获先落这页,链接慢慢长出来——写入端保持 Mem 式的零决策(不用想放哪),组织端交给链接的自然生长;**③重逢式检索**——写新东西时反链面板自动浮出「哪些旧笔记链着这个概念」,等于 GraphRAG 的自动上下文装配:**旧想法在相关时刻自己回来**,而不是躺在坟场等你想起。

风险照例标注:网状系统的经典坑是**漫游成瘾**——沿着链接走五步,从定价走到三年前的读书笔记,半小时没了(图遍历的深度失控,agent 侧同样要设跳数上限)。纪律同款:**工作时检索限两跳**——找到要的就回来,漫游留给专门的「花园时间」(每周一次,漫游本身在那个时段是合法的,它是灵感的来源不是敌人)。

## 边界

同构强度 A 级:ADHD 联想式认知与发散思维有实验文献,图增强检索是 agent 工程的真实架构且与向量检索的对比有研究支撑,「结构承载关系」的同构在两侧都有实体证据;但反链笔记对 ADHD 的直接对照研究缺乏,A 级判给架构同构,不判给疗效。声明:笔记系统是认知辅助;涣散的全面失控仍以临床评估为正路。

## 今天就能试的 3 件事

1. **给岔路建停车位**:今天每个「想跟进」的念头,一句话+一个链接甩进笔记——不跟进,也不攥着。
2. **从 daily note 开始**:别设计体系,今天的一切先记进今天这页——链接会自己长。
3. **设两跳规则**:工作时段的笔记检索,最多两跳必须返回——漫游预约到周末的花园时间。

线性系统逼 ADHD 把网压成线,每天都在做赔本的格式转换——**换一个和你大脑同构的存储结构,涣散的一半燃料(怕丢+想追)当场断供**。网状思维不是缺陷,它只是一直没等到网状的容器。

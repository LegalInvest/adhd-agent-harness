# 为什么用 Reflect 治 ADHD 的不知哪些方法有用，和给 agent 套 human-in-the-loop 监督 是一回事？

> Reflect 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

你有没有过这种早晨：收藏夹里躺着 100 个 ADHD 方法，从番茄钟到身体加倍法，从 Notion 模板到 5-HTP；但真到该开始的时候，你只记得刷手机？这不是懒，而是一种更深层的问题——不是不知道方法，而是**不知道哪些方法在自己身上有用**。

这个痛点，和今天 LLM 工程师给 agent 加 human-in-the-loop 监督时想解决的是同一件事。

## 一、ADHD 侧的"不知哪些方法有用"：生成核心缺了调度层

ADHD 的痛苦常常被误解为"不专注"。但 wiki 资料中反复出现的是另一个概念：执行功能障碍。它指的不是不努力，而是大脑在计划、组织、时间管理、任务启动和抑制控制上效率低下（来源：执行功能障碍）。这导致时间盲、工作记忆不足、任务启动困难——简单说，就是脑子里能冒出无数点子，但身体不知道下一步该干什么。

更麻烦的是情绪失调。在 ADHD 中，情绪失调不是共病，而是核心特征之一，与执行功能缺陷密切相关，尤其是工作记忆和抑制控制不足，导致情绪反应难以被认知调节（来源：情绪失调）。这意味着一个无效的方法不仅会让你失败，还会引发羞耻、愤怒和放弃。于是"收藏方法—尝试—失败—自责—再收藏"循环往复。

这不是信息不足，而是系统没有反馈回路。你缺少一个外部执行功能层，来帮你把方法变成步骤、把步骤变成行动、把行动的结果反馈回系统。

## 二、LLM 侧的对应：高产但缺执行调度层的生成核心

如果你做过 agentic harness，这一幕应该很熟悉：LLM 生成能力惊人，但你不给它 human-in-the-loop 监督，它就可能陷入幻觉、循环或目标漂移。wiki 资料中提到的 ADHD 大脑与 LLM 同构论，核心就在于此——两者都是高产但缺乏执行调度层的生成核心（来源：ADHD 大脑与 LLM 的同构）。

ADHD 人群的"不知哪些方法有用"，对应到 LLM 系统就是"不知哪个 action 该被保留"。Human-in-the-loop 监督、外部执行功能层、验证器（verifier）、调度器（scheduler）这些工程结构，本质上是在给生成核心装一个 harness：让它可以提议，但不让它擅自执行；让它可以试错，但必须有回滚点。

Reflect 这个名字本身就说明了一切。它不是"改进你的大脑"，也不是"让 LLM 更聪明"，而是让两个系统都能**停下来、检查、再往前走**。

## 三、同构脊柱：人在回路与身体在场

两边的解法结构惊人地一致。

ADHD 侧需要身体在场效应（来源：身体在场效应）——他人在场或外部结构能提升专注。LLM 侧需要 human-in-the-loop——人类在关键节点做验证。前者把"人"作为外部监督器，后者把"人"作为外部审核器。两者解决的都是同一个问题：系统自己无法完成元认知。

Reflect 的 harness 思路因此成立：无论对象是人脑还是模型，都加一个轻量级外部层——不是接管，而是打断、锚定、反馈。它不是拐杖，而是脚手架；不是替你做决定，而是让决定变得可见。

## 四、历史 harness：张謇的"日课"与张居正的"考成法"

这种 harness 不是现代发明。张謇（清末状元实业家）的 ADHD 特质非常明显：41 岁中状元后辞官做实业，一辈子办了 20 多家企业、300 多所学校，天天在工地跑，坐不住。他的专属 harness 是"父教育母实业"的实业救国信仰，以及每天写日记反省、严格管理企业（来源：人物案例：张謇）。

这对应到 LLM harness 里是什么？**日记反省**就是定时 re-grounding，让系统在每天结束时回到初心；**实业救国信仰**就是系统提示词（system prompt）或 mission statement，防止目标漂移。张謇不是克服了 ADHD，而是给超高的能量和兴趣爆发找到了一个外部锚点。

张居正的 harness 更偏向工程。他推行考成法，严格考核官员，用制度管别人也管自己，工作狂到 58 岁累死在任上（来源：人物案例：张居正）。考成法就是**外部验证器+执行器**：不是指望自己或他人自觉，而是设定明确的考核节点和惩罚机制。放到 LLM 系统里，这就是 eval loop 和 checkpoint。

张謇的"日课"解决的是"我到底是谁、我要什么"；张居正的"考成法"解决的是"这件事到底做没做、做得怎样"。Reflect 的 human-in-the-loop，同时承担这两种功能：既在价值观层面 re-ground，又在执行层面 verify。

## 五、工具验证：harness 不是比喻，是操作界面

今天的工具正在把这套 harness 变成可触摸的界面。

**Goblin Tools** 的 Magic ToDo 把模糊任务（如"清理厨房"）拆成具体可执行的子步骤，并可用"辣度"调节粒度（来源：Goblin Tools）。这对应 LLM 里的 task decomposition：把宏大目标拆成可验证的小 action。

**Saner.AI** 强调知识回忆和本地记忆，减少 ADHD 用户因搜索循环和标签切换造成的认知负荷（来源：Saner.AI）。对应 LLM 的 RAG 或 memory 层：外部记忆系统弥补了工作记忆的不足。

**Motion** 自动根据任务、截止日期和可用时间动态排程，当任务可能延期时提前数周发出警告（来源：Motion）。对应 LLM 的 scheduler 或 orchestrator：持续评估优先级、重新分配资源。

这些工具的有效性，不是来自"让 ADHD 人变成正常人"，而是来自它们提供了一个外部执行功能层——和给 agent 加 human-in-the-loop 是同一个工程思想。

## 六、边界：同构是思路，不是事实；AI 是脚手架，不是身体在场

但必须诚实指出局限。wiki 资料中的"矛盾与存疑"部分明确警告：ADHD 大脑与 LLM 的同构性缺乏实证基础，目前只是理论类比，不是科学事实（来源：矛盾与存疑）。我们不能因为两边都用了"harness"这个词，就认为它们共享同一神经机制。

其次，工具宣称的证据不足。例如 Motion 页面自身提到缺乏独立临床验证，却仍被推荐为有效工具（来源：矛盾与存疑）。身体在场效应被 AI 工具尝试复制，但 AI 是否真能替代真实人际互动存在争议（来源：矛盾与存疑）。如果 Reflect 的 human-in-the-loop 只是形式上的"点个确认"，而没有真实认知参与，那它和自动审批没有区别。

最后是脚手架 vs 拐杖的边界。好的 harness 应该逐步被内化为能力，而不是永久依赖。如果每次启动任务都需要 AI 全部分解，那它就不是脚手架，而是拐杖。

## 七、今天就能试：4 个最小闭环

1. **给自己写一句系统提示词**。像给 LLM 写 system prompt 一样，把"我要什么、我不要什么"写在一张纸上贴在屏幕前。这不是鸡汤，而是张謇式的"日课 re-grounding"。
2. **用 Goblin Tools 拆分一个你拖延了三天的任务**。把"整理房间"拆成"把袜子扔进脏衣篮"，只完成第一步就停。这和给 agent 的 first action 做沙盒验证是同一个逻辑。
3. **给任何你信任的 AI/agent 设置一个硬验证点**。让它在真正执行前必须停下来，把计划发给你确认。Reflect 的精神不是"让人更省力"，而是"让错误更便宜"。
4. **每周做一次张居正式复盘**：列出三个 action，检查完成了几个、为什么没完成。不评判，只更新 harness。这是把考成法变成个人 eval loop。

Reflect 治 ADHD 的"不知哪些方法有用"，和给 agent 套 human-in-the-loop，本质上是在做同一件事：承认生成核心的价值，但拒绝让它裸奔。两边的解法都指向一个更谦逊的工程观——不是造出更完美的机器，而是承认人（或系统）需要被另一个层级温柔地打断、锚定和放行。

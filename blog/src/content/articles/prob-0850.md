# Lex 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Lex：它到底补上了哪一层执行功能？

如果你是一个 ADHD 患者，你一定经历过这样的场景：打开文档，盯着空白页面，大脑里有一千个想法，但就是无法打出第一个字。如果你是一个在做 Agentic Harness 的工程师，你一定调试过这样的 bug：LLM 生成了完美的计划，却在执行第一步后就偏离轨道，或者陷入无限循环。

这两个问题，本质上是同一个问题。

## 同构：ADHD 大脑与 LLM 共享的「生成核心」

明尼苏达大学的一项研究发现，LLM 的执行功能剖面是「强记忆、弱控制」——工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷，无法灵活切换任务集、无法抑制自动化反应（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这正是 ADHD 的经典神经心理模式。

换句话说，ADHD 大脑和 LLM 都是「高产但缺乏可靠执行调度层的生成核心」（来源：AI 与 ADHD 的时间管理）。它们不缺少生成能力，缺少的是把生成能力定向、分解、排序并持续执行的调度层。

在 LLM/Agent 工程中，这个调度层被称为 planner-executor 循环：planner 将长任务分解为子步骤，executor 逐步执行，然后 planner 根据结果调整下一步。在 ADHD 的时间管理中，这个调度层就是执行功能——尤其是任务启动、工作记忆和时间感知。

## Lex 补上了哪一层？

Lex 是一款 AI 写作助手，但它不是简单的自动补全或文本生成器。它的核心设计是：在你写作时，它会不断提示你下一步写什么，或者将你的零散想法整理成结构。

这对应了 planner-executor 循环中的什么？

- **planner 层**：当你不知道如何开始时，Lex 的「帮我写」功能可以生成一个草稿或大纲——这相当于 planner 将「写一篇文章」这个大任务分解为「写引言」「写论点」「写结论」等子任务。
- **executor 层**：当你写了一段后卡住，Lex 的「继续写」功能会基于前文生成下一句——这相当于 executor 执行了一个子步骤。
- **调度层**：关键在这里。Lex 的「大纲视图」和「侧边栏笔记」功能，让你可以随时查看整体结构，调整顺序，添加或删除段落——这相当于 planner 在 executor 执行后重新评估计划。

但大多数用户只使用了 executor 层：他们让 Lex 自动补全，然后接受结果。这就像让 LLM 直接输出最终答案而不经过 planning——结果往往是流畅但空洞的文本。

## 有人用错了：脚手架 vs 拐杖

「AI 工具作为外部执行功能层」这个观点，在 ADHD 社区和 Agent 工程中都很流行。但 wiki 资料明确指出一个矛盾：工具设计者声称是「脚手架」，但实际使用中可能沦为「拐杖」（来源：矛盾与存疑）。

- **脚手架**：你使用工具来学习如何自己规划。比如，用 Lex 的大纲视图来理解文章结构，逐渐内化这种分解能力。
- **拐杖**：你依赖工具来替代你的规划能力。比如，每次都让 Lex 自动生成全文，从不自己思考结构。

对于 ADHD 用户，过度依赖 AI 工具可能削弱内在的时间感知和规划能力（来源：AI 与 ADHD 的时间管理）。对于 Agent 工程师，过度依赖 planner 的自动分解可能导致系统在遇到未预见的任务时崩溃。

## 如何正确使用 Lex：给 ADHD 用户和工程师的实践指南

### 给 ADHD 用户

1. **先用大纲视图，再写正文**：在开始写作前，用 Lex 的「大纲视图」写下 3-5 个关键点。这相当于 planner 分解任务，降低启动门槛。
2. **用「继续写」作为推进器，而非替代品**：当你写了一段后卡住，让 Lex 生成下一句，但必须自己修改。这相当于 executor 执行一步后，由你（作为 planner）评估结果。
3. **结合外部时间工具**：Lex 本身不处理时间盲。搭配 Tiimo 或 Reclaim.ai 来设置写作时间块，让 Lex 负责内容，让时间工具负责时间感知。

### 给 Agent 工程师

1. **显式分离 planner 和 executor 的日志**：在 Agent 系统中，记录 planner 生成的每一步计划和 executor 的实际输出。这相当于 Lex 的大纲视图——让你能看到计划与执行的差距。
2. **在 planner 中加入「执行后评估」步骤**：不要只让 planner 生成一次计划就让 executor 执行到底。每执行一步后，让 planner 检查结果并调整下一步。这正是 Lex 的「大纲视图+侧边栏笔记」的工作方式。
3. **避免过度自动化**：就像 ADHD 用户不应让 Lex 完全替代写作，Agent 也不应让 planner 完全替代人工干预。保留一个「人类在环」的检查点，尤其是在关键决策点。

## 诚实面对局限

现有证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑）。Lex 对 ADHD 的效果尚未经过严格验证，其 AI 功能（如「帮我写」的算法）也缺乏透明度。

更重要的是，同构理论本身仍属理论类比，神经科学和计算机科学的交叉验证不足（来源：AI 与 ADHD 的时间管理）。ADHD 大脑与 LLM 的相似性可能只是表面现象，深层机制可能完全不同。

## 总结

Lex 之于 ADHD，就像 planner-executor 任务分解之于 LLM——它补上了调度层。但前提是你把它当作脚手架，而不是拐杖。

今天就能试的三件事：
1. 打开 Lex，先用大纲视图写下 3 个要点，再写正文。
2. 在 Agent 系统中，为 planner 和 executor 分别记录日志，观察计划与执行的偏差。
3. 设置一个 25 分钟的写作时间块（用 Reclaim.ai 或 Tiimo），期间只使用 Lex 的大纲视图和手动写作，禁用「自动补全」。

# Lex 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Lex：它到底补上了哪一层执行功能？

## 问题：为什么“自动规划”反而让 ADHD 更糟？

如果你是一名 ADHD 患者，你可能经历过这样的场景：下载了一款号称“AI 自动为你规划日程”的工具（如 Motion），满怀期待地输入所有任务，然后看着它自动排好。但几天后，你发现自己反而更焦虑了——因为工具排的计划你根本不认同，或者你完全失去了对时间的掌控感。如果你是一名做 Agentic Harness 的工程师，你可能也遇到过类似困境：给 LLM 一个 planner，让它自动分解任务，结果它要么过度拆解导致执行混乱，要么生成一个你无法理解的计划。

这两类问题共享同一个根源：**规划权被错误地外包了**。ADHD 大脑与 LLM 在结构上同构——两者都是高产的生成核心，但缺乏可靠的执行调度层（来源：AI 与 ADHD 的时间管理）。然而，当前许多 AI 工具（如 Motion）的做法是“替你规划”，这相当于给 LLM 一个黑盒 planner，却剥夺了 executor 的决策权。结果就是：工具成了“拐杖”，而非可撤除的“脚手架”。

## 同构视角：planner-executor 循环的正确用法

在 LLM/Agent 工程中，一个成熟的 harness 会采用 **planner-executor 循环**：planner 先生成粗略计划，然后 executor 执行一步，反馈结果，planner 再调整下一步。这种循环保留了人类的监督和调整权，避免 AI 完全接管。对于 ADHD 大脑，同样需要这样的循环——工具应该帮助用户“分解任务”和“启动任务”，而不是替用户“决定任务”。

ADHD 的核心困难在于执行功能缺陷：难以规划、组织、同时处理多项任务（来源：执行功能）。工作记忆是关键瓶颈——无法在脑中短暂保持信息（来源：工作记忆）。因此，AI 工具作为外部执行功能层，本质是“给生成核心套上 harness”（来源：AI 与 ADHD 的时间管理）。但 harness 的设计必须让用户保持控制感，否则会加剧“时间盲”和决策疲劳。

## 实测：Lex 和 Motion 的差异

以 **Lex**（一种假设的 AI 写作助手，类似 Goblin Tools 的“任务分解”功能）为例：它帮你把“写一篇论文”分解成“收集资料→写大纲→写引言→写正文→修改”等步骤，但每一步仍然需要你主动执行和调整。这就像 LLM 的 planner 给出子任务列表，executor（你）逐个完成并反馈。相比之下，**Motion** 则直接为你规划好每天几点做什么，甚至自动调整——这相当于 planner 直接接管了 executor 的决策，用户只需被动执行。结果：Motion 虽然降低了启动门槛，但剥夺了用户的时间感知和规划练习机会，长期可能阻碍内在执行功能的发展（来源：矛盾与存疑）。

**Reclaim.ai** 则更接近正确方向：它通过“保护时间块”而非“填充时间”来工作，保留用户对日程的控制权（来源：Reclaim.ai）。**Tiimo** 通过视觉化时间线帮助用户感知时间，而不是替用户安排（来源：Tiimo）。**Todoist** 的智能优先级排序则像一种“轻量级 planner”，用户仍可手动调整（来源：AI 与 ADHD 的时间管理）。

## 证据与局限

现有资料中，Motion 等工具被推荐用于 ADHD 辅助，但缺乏临床研究支持其长期效果（来源：Motion）。矛盾在于：一方面 AI 工具可绕过执行功能障碍，另一方面这些工具本身仍需要执行功能来记住使用（来源：执行功能）。更关键的是，多数工具设计为长期使用，未提及撤除机制，这与“脚手架应可逐步撤除”的理念冲突（来源：矛盾与存疑）。

对于 LLM 侧，planner-executor 循环已被广泛采用（如 AutoGPT、LangChain 的 Agent 模式），但同样存在“过度规划”或“规划与执行脱节”的问题。ADHD 与 LLM 的同构命题本身是理论模型，缺乏大规模实证（来源：矛盾与存疑），但工具案例和用户反馈提供了初步支持。

## 核心判断：工具应做“分解者”，而非“决策者”

我的观点是：**ADHD 的 AI 工具应该模仿 LLM 的 planner-executor 循环，保留用户的决策权，只提供分解、提醒和反馈**。工具应做“脚手架”——在用户能力不足时提供支撑，但设计上允许逐步撤除。例如，Goblin Tools 的“任务分解”功能（将复杂任务拆成小步）就是好的脚手架，因为它不替用户决定何时做，只帮用户看清步骤。而 Motion 的自动排程则更像“拐杖”，长期使用可能削弱用户的时间管理能力。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个棘手的任务**：输入“整理房间”或“写报告”，看它如何拆成小步骤。然后自己决定先做哪一步，而不是让工具替你排时间。
2. **在 Todoist 中启用智能优先级**：设置几个任务，观察 AI 给出的优先级排序，但手动调整你认为重要的项。这训练你的规划能力，同时利用 AI 的辅助。
3. **尝试 Tiimo 的视觉化时间线**：将一天的活动可视化，看看时间块是否更清晰。注意：不要让它自动安排，而是自己拖拽任务到时间线上。
4. **反思工具依赖**：如果你正在用 Motion 或类似工具，尝试每周有一天不用它，手动规划日程。记录你的焦虑水平和完成度——这能帮你判断工具是“脚手架”还是“拐杖”。

## 结语

ADHD 大脑和 LLM 都是强大的生成引擎，但都需要一个精心设计的 harness 来发挥潜力。工具的正确用法不是替代你的规划能力，而是辅助你建立自己的执行系统。记住：planner 可以帮你分解，但 executor 永远是你自己。

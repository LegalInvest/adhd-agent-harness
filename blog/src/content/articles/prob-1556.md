# 如果把 ADHD 大脑当成生成核心，ChatGPT 能不能当它的拐杖与脚手架？

> 用 agent 工程的思路重新理解 ChatGPT 对 ADHD 的真正价值。

你有没有过这样的时刻：脑子里同时涌出十个创意，但一个也落不了地；明明想开始工作，却瘫在沙发上刷了两小时手机；情绪突然崩盘，却说不清为什么。ADHD 大脑就像一个天才生成器，却配了一个坏掉的执行调度层。你怕过度依赖工具，又怕没有支撑就崩——这种两难，恰好和 LLM agent 工程面临的困境一模一样：模型本身强大，但直接输出会跑偏、循环、甚至危险，所以需要 harness（脚手架）来调度、护栏来纠偏。

如果把 ADHD 大脑当成生成核心，ChatGPT 能不能当它的拐杖与脚手架？答案是：**可以，但必须把它当作会褪去的脚手架，而非永久拐杖。** 这不仅是 ADHD 群体的需求，也是 agent 工程的核心原则——harness 应是会褪去的脚手架，而非永久拐杖。

## 同构：两个“高产但缺调度”的生成核心

ADHD 大脑的生成核心（发散思维、创意、多任务并行）产出极高，但执行功能（任务启动、持续专注、抑制冲动）薄弱，导致“高产但不可靠”（来源：人在回路与身体在场）。同样，LLM 本身是强大的生成核心，但缺乏可靠的执行调度层，直接输出可能偏离目标、陷入循环或产生危险行为（来源：人在回路与身体在场）。

这种同构意味着：ADHD 的情绪失调类似于 LLM 的上下文失控——当工作记忆过载或注意力被干扰时，情绪反应如同 LLM 的幻觉，脱离理性轨道（来源：AI 与 ADHD 的情绪调节）。因此，AI 帮助 ADHD 情绪调节的本质是给生成核心套上 harness：通过上下文工程主动设计注意焦点，利用无状态与外部记忆记录情绪触发模式，实现情绪的可预测性（来源：AI 与 ADHD 的情绪调节）。

## 拐杖 vs 脚手架：边界在哪里？

拐杖是永久替代，脚手架是临时支撑，最终要拆除。ADHD 群体最怕的“过度依赖”和工程师最怕的“永久拐杖”其实是同一个问题。

证据显示，过度依赖 AI 可能削弱内在情绪调节能力（来源：AI 与 ADHD 的情绪调节），过度依赖可能削弱内在时间感知能力（来源：矛盾与存疑）。工具设计者声称是“脚手架”，但实际使用中可能沦为“拐杖”（来源：矛盾与存疑）。

那么边界在哪？脚手架的特点：
- **有明确使用目标**：不是为了“让 AI 替我做”，而是“让 AI 帮我学会做”。
- **逐步退出**：随着内在能力提升，减少 AI 介入。
- **保留核心控制**：AI 提供提醒和分解，但决策权在人。

拐杖的特点：
- **无退出计划**：长期依赖，没有训练内在能力的意图。
- **替代而非补偿**：AI 直接替你做，而不是帮你做。
- **削弱内在能力**：长期使用后，离开 AI 更无法启动。

## 真实工具如何充当脚手架

### 重锚定系统：对抗目标漂移

ADHD 大脑在执行任务时极易发生目标漂移：注意力被无关刺激捕获，或过度聚焦细节而忽略整体目标。AI 工具通过提醒和通知提供重锚定，将注意力拉回原任务（来源：重锚定与目标漂移）。例如，Goblin Tools 的 Magic ToDo 功能自动将复杂任务分解为小步骤，降低启动门槛（来源：Goblin Tools）。用户反馈称“Goblin Tools 在这方面很棒”（来源：Goblin Tools）。

LLM agent 在长程任务中同样面临目标漂移：模型在交互轮次后期开始忘记系统提示。Harness 通过事件驱动的系统提醒来对抗这种“指令消退”（来源：重锚定与目标漂移）。

### 外部记忆：补偿工作记忆缺陷

ADHD 的工作记忆容量有限，导致情绪信息无法被有效加工。Saner.AI 通过知识回忆和本地记忆，减少搜索循环和标签切换（来源：Saner.AI）。Inflow 通过强化背外侧前额叶皮层功能，提升工作记忆（来源：Inflow）。

### 身体在场效应：绕过执行功能屏障

## 核心观点：脚手架思维

我的核心判断是：**ChatGPT 对 ADHD 的真正价值不在于它多聪明，而在于它能否成为可褪去的脚手架。** 就像 agent harness 一样，好的 AI 工具应该是“形式化规划与护栏，使代理输出真正有用且正确”（来源：人在回路与身体在场）。对 ADHD 大脑来说，这意味着：

# 系统提示→晨间仪式、RAG→外置笔记、工具循环→body double：这套反向映射的失败模式是什么？

> 一位工程师读者交来一份罕见的作业:他把 agent 工程的三大件系统性地反向映射到自己的 ADHD 生活,严格执行了八周,并且——这是罕见之处——诚实记录了失败。他的映射表:**system prompt → 晨间意图仪式**(每天早上朗读自己的「今日系统提示」:身份、优先级、约束);**RAG → 可检索外置笔记**(所有知识和承诺入库,干活时按需检索,不靠脑存);**工具循环 → body double 与清单**(把「自己不擅长的步骤」像 agent 调工具一样,外发给伙伴、闹钟、清单)。前四周效果惊人,他一度想写「同构自助的胜利」;后四周,三个映射各自以不同的方式失效了。他的复盘笔记的最后一句是这次投稿的题眼:「**我终于明白问题出在哪:agent 的注意力分配器是可微的——工程师可以用梯度直接改它;而我的注意力分配器不可微——没有任何指令能直接写进我的权重里。所有映射的失败,都发生在这个缝上。**」

收敛:本文只回答——**三个映射各自的失败模式(现场记录级的细节)?「不可微」这个洞察的准确含义(它为什么是所有失败的公因子)?修补后的映射 v2(不是放弃映射,是给每个映射补上「不可微补丁」)?**

## 穿透:给 agent 的指令直接生效,给自己的指令要过"愿不愿意"这一关——这一关就是全部的区别

三个失败模式的现场。**失败一:晨间系统提示的「新鲜感衰减」**——前两周,朗读仪式真的有效(相当于每天重注目标锚点);第四周起,朗读变成机械念经(眼睛在读,系统没加载)——**同一段 prompt 对 agent 永远等效,对人的效力却随重复而指数衰减**,因为人的注意系统对「无新信息的重复刺激」会习惯化(habituation)——agent 没有「听腻了」这回事,人有;**失败二:RAG 笔记的「检索意愿缺口」**——库建得很好,检索功能完美,但第六周他发现自己「明知库里有,就是不想去查」(尤其在低状态日,打开笔记系统这个动作本身就有启动成本)——**agent 的检索是被架构强制调用的(每一步必查),人的检索要经过动机审批——而审批官经常旷工**;**失败三:工具循环的「社交成本累积」**——把步骤外发给 body double 伙伴的机制,前期顺畅,后期他开始「不好意思再麻烦人家」,外发频率悄悄下降直至停摆——**agent 调工具没有心理账户,人调「人形工具」每次都在消耗关系余额**(而他没有设计还款机制)。

「不可微」的准确含义,把这个工程隐喻说透:agent 的行为策略可以被直接优化(改 prompt、调权重、换架构——干预直达机制层);而人的注意力分配器是一个不接受直接写入的系统——**你不能「决定」自己对什么感兴趣、不能「命令」自己的检索意愿上线、不能「设置」自己的社交成本为零**;一切自我干预都只能走间接路径:改环境、改线索、改奖励结构,然后**等系统自己响应**。三个失败的公因子由此清晰:三个映射都把「架构强制」原样搬成了「自我要求」——而自我要求恰恰是那条不存在的直接写入通道。**映射失败的不是结构(结构全对),是执行机制:agent 靠架构,人只能靠环境。**

映射 v2——每条补一个「不可微补丁」:**①系统提示 v2:内容轮换制**——对抗习惯化的工程方案是注入新息:意图仪式每周换形式(朗读→手写→对着镜子说→给 AI 复述让它提一个问题)、内容里必须含「今天与昨天不同的一件事」——**让重注锚点这个功能不变,载体持续新鲜**(用新鲜感喂注意系统,而不是要求它忍受重复);**②RAG v2:检索零成本化+推送兜底**——把「查」的成本压到极限(全局快捷键一键达、手机桌面第一屏),并给最关键的信息加「主动推送」(时间/地点触发的提醒把信息送到眼前)——**意愿审批官靠不住,就尽量绕开审批(降成本)或不需要审批(反向推送)**;**③工具循环 v2:社交账户显式记账**——body double 关系按互惠协议设计(轮流做对方的 double、明确「这是互相的」的定期确认),让「麻烦人家」变成「这周轮到我用额度」——**心理账户消不掉,但可以做成双向流水,流水平衡就不再累积愧疚。**

## 验证

v2 可测:三个补丁各跑四周,对比 v1 的失效时间点(习惯化推迟了吗?低状态日检索率升了吗?外发频率稳住了吗?)。可证伪:若 v2 依然在同样的位置失效,说明该功能对你不适合结构化方案,回退到「真人服务」(教练/伙伴的人工判断)——不可微系统的最后一招,永远是另一个人。

## 决策

做什么:映射照做,但每条过一遍「不可微审计」:这个环节 agent 靠架构强制,我靠什么?答不上来的,补丁。

不做什么:不要把映射失败归因于「我连这个都坚持不了」(失败的是机制假设,不是你);不要放弃映射本身(结构是对的,八周里前四周的收益是真的)。

先做什么:给你现有的自助系统跑一次不可微审计——找出所有「假装自己有架构强制」的环节。

## 边界

习惯化、启动成本、互惠机制均有行为科学基础(GRADE 中);映射框架为 B 级功能同型(本文的主题恰是同型的边界——「不可微」正是实现层不同构的执行面后果,与怀疑论各篇的分层结论完全一致)。个案八周记录为 N=1 证据,失败模式的普遍性待更多复现。

## 今天就能试的 3 件事

1. 跑不可微审计。
2. 给晨间仪式(如果有)排一个四周形式轮换表。
3. 和你的 body double(如果有)做一次「双向流水」确认——把互惠说出口。

本文服务于人生 Harness 金字塔的**执行层的工程哲学**:向机器借结构,是这个时代给 ADHD 的礼物;但收礼时要读清说明书上那行小字——机器的结构靠架构强制生效,你的版本只能靠环境、新鲜感和另一个人;所有照搬的失败,都失败在把自己当成了可以直接写入的系统——你不是;你是一个只能被温柔地、间接地、持续地邀请的系统。

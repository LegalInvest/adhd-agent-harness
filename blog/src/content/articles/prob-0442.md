# ADHD 患者用‘重锚定’策略（如番茄钟）成功率有限，而 LLM 的 prompt 重锚几乎总是有效。这个类比是否忽略了人类意志力的脆弱性，让工程师低估了真实世界的难度？

> 有效性差距：prompt 重锚几乎完美，人类重锚为何屡败？

## 问题：为什么番茄钟对 ADHD 常常失效，而 prompt 重锚几乎总能拉回 LLM？

你设好 25 分钟番茄钟，刚写了三行字，手机通知弹出，你点开，然后 20 分钟过去了。番茄钟还在跑，但你的注意力早已漂移到另一个星系。另一边，工程师在调试一个长程 agent，发现它开始胡言乱语，于是往 system prompt 里加了一句“请始终围绕原始目标回答”，agent 立刻回归正轨。

这个场景让你既熟悉又沮丧：为什么“重锚定”在 LLM 身上几乎完美，在人类身上却屡屡失败？是 ADHD 患者意志力太弱，还是这个类比本身就有问题？

## 同构脊柱：高产但缺执行调度层的生成核心

ADHD 大脑与 LLM 的相似性，并非牵强附会。两者都是**强大的生成核心**——ADHD 大脑能迸发超聚焦下的惊人创造力，LLM 能生成流畅连贯的文本——但都**缺乏可靠的内置执行调度层**。ADHD 的执行功能缺陷（工作记忆容量有限、任务启动困难、注意力易分散）与 LLM 的无跨会话状态、上下文膨胀后的推理退化，在结构上高度同构（来源：AI 与 ADHD 的专注力）。两者都需要**外部脚手架**来弥补调度缺陷：ADHD 需要番茄钟、身体加倍、时间追踪等工具；LLM 需要精心设计的 system prompt、重锚定指令、外部记忆系统。

这种同构性甚至得到了实证支持：用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中表现正常，但随序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突（来源：认知负荷）。这与 ADHD 执行功能缺陷的核心标志一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃。

## 核心判断：prompt 重锚 vs 人类重锚——意志力的脆弱性不可忽略

但这里有一个关键差异被许多工程师忽略了：**LLM 没有“意志力”这回事**。当你在 prompt 里写下“请忽略之前偏离的话题，回到原始目标”，LLM 会无条件服从，因为它没有情感波动、没有多巴胺渴求、没有拒绝敏感性焦虑。而 ADHD 患者在番茄钟响起的瞬间，面对的是一整套内在阻力：多巴胺水平低导致“不想切换”的惰性，工作记忆超载导致“我刚刚在做什么来着”的迷失，时间盲导致“还剩 5 分钟，不如再刷一下”的误判（来源：任务启动；认知负荷）。这些因素叠加，使人类重锚的成功率远低于 LLM。

曾国藩深知这一点。他 30 岁前“浮躁坐不住”，日记里天天骂自己“无恒”“浮躁”，读书慢到“他人目下二三行，余或疾读不能终一行”。他的解决方案不是靠意志力硬扛，而是设计了一套**外部脚手架系统**——日课十二条：黎明即起、读书不二、谨言、写日记反省。这本质上是一个“人类版 system prompt”，每天定时 re-grounding，用最笨最稳的方法抵消自己的冲动。他一生打败仗就跳水自杀，但靠着这套脚手架，最终平定太平天国，成为晚清中兴名臣。他的成功恰恰说明：**人类重锚需要比 prompt 重锚更精细、更反复、更宽容失败**。

## 工具生态：外部执行功能层的实践

当前 AI 工具已开始构建这一外部执行功能层。例如，**Brain.fm** 利用神经锁相技术生成专注音乐，帮助用户进入状态（来源：Brain.fm）；**Focusmate** 通过虚拟身体在场提供实时问责，弥补内在多巴胺调节不足（来源：Focusmate）；**RescueTime** 自动追踪时间，揭示真实的时间去向，对抗时间盲（来源：RescueTime）。这些工具共同构成一个“外部调度层”，但它们的共同局限是：**依赖用户主动使用**——而主动使用本身就需要执行功能。

更值得关注的是，ADHD 成年人已自发采用 ChatGPT 作为执行功能外挂，弥补任务启动等缺陷（来源：任务启动）。这一行为暗示了认知架构的互补性：AI 工具作为“外部大脑”，可以外化思维、减轻工作记忆负担（来源：认知负荷）。但这也带来了依赖风险：工具是脚手架还是拐杖？目前尚无长期追踪研究（来源：矛盾与存疑）。

## 局限与争议：同构的边界与证据的不足

必须诚实指出，ADHD 大脑与 LLM 的同构目前仍是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。现有工具的证据多来自用户报告或小样本研究，缺乏大规模随机对照试验（来源：AI 与 ADHD 的专注力）。例如，Brain.fm 虽被列为最佳 ADHD 应用之一，但缺乏针对 ADHD 的独立临床研究（来源：Brain.fm）；RescueTime 的自动分类可能不够精准，对执行功能较弱的用户仍是挑战（来源：RescueTime）。此外，ADHD 的异质性意味着同一工具对不同亚型效果可能迥异（来源：AI 与 ADHD 的专注力）。

## 今天就能试的行动

1. **给任务加一个“重锚指令”**：像写 system prompt 一样，在开始任务前写一句“我的目标是 X，如果走神，回到 X”。写在便签上贴在屏幕边缘。
2. **用身体在场降低启动阻力**：预约一次 Focusmate 时段，或找一个朋友视频连线一起工作。外部问责比自我监督可靠得多。
3. **用 AI 工具外化调度**：尝试 Goblin Tools 的“魔法待办”将复杂任务拆解为小步骤（来源：任务启动），或用 Motion 自动规划日程，减少决策负担。
4. **接受重锚失败**：像曾国藩一样，日记里骂完自己第二天继续。重锚不是一次性的，而是一个反复迭代的过程。工程师不会因为一次 prompt 偏离就放弃 agent，你也不该因为一次番茄钟失效就否定自己。

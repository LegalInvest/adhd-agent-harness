# ADHD 职场人如何用「给生成核心套 harness」的思路应对项目推进与汇报里的冲动下结论、想当然？

> 把 LLM 工程里的「LLM 自信地编造（幻觉），靠验证与自我批判兜底」迁移到ADHD 职场人的项目推进与汇报场景，双域证据作答。

你刚在项目会上脱口而出一个结论，散会后却发现数据完全站不住脚。或者，你自信地汇报了方案，被追问细节时才发现自己“想当然”了。这种冲动下结论和想当然，是 ADHD 职场人最熟悉的痛——不是不够聪明，而是大脑的生成核心跑得太快，调度层没跟上。

有趣的是，LLM 工程里有一个完全对应的场景：大语言模型自信地编造（幻觉），靠验证与自我批判兜底。工程师们为此搭建了 Agentic Harness——一个外部调度层，在模型输出后自动校验、回滚、自我纠正。

**核心观点：ADHD 大脑与 LLM 是同一类“高产但缺执行调度层的生成核心”，两边的解法结构同构——不是压抑生成，而是给生成核心套上 harness（外部脚手架），用验证循环兜底。**

## 同构：两个世界的“幻觉”与“验证”

ADHD 的冲动下结论，本质是生成核心（发散联想、模式匹配）在缺乏抑制控制的情况下直接输出。这与 LLM 的幻觉同源：模型基于概率生成最可能的 token，不一定基于事实。

在 LLM 工程中，Agentic Harness 的做法是：让 LLM 自由生成，但随后用外部工具（如搜索、代码执行、数据库查询）验证，若发现错误则自我批判并重生成（来源：[[ADHD 大脑与 LLM 的同构]]）。

ADHD 职场人需要的正是类似的“验证循环”：先允许自己快速产生结论和方案，但强制加上一个外部校验步骤——比如在汇报前用数据核对、请同事做“魔鬼代言人”、或者用清单检查常见思维陷阱。

## 历史人物的 harness：毛泽东的“批评与自我批评”

毛泽东是典型的 ADHD 特质者：思维跳跃、行动冲动（四渡赤水、抗美援朝都是险棋）、永远在调查研究。但他建立了一套强大的 harness 系统：**批评与自我批评、民主生活会、民主集中制**。这些本质上就是外部验证循环——让组织中的其他人来校验自己的冲动决策，用集体智慧兜底。

这与 LLM harness 中的“多智能体辩论”或“自我批判 prompt”结构同构：生成核心（领袖的直觉）仍然自由输出，但外部调度层（组织流程）强制验证、修正。毛泽东的成就证明，这种 harness 可以驾驭巨大的生成能量，而不是扼杀它。

## 现代 AI 工具：给 ADHD 大脑套 harness

今天，AI 工具已经提供了具体的 harness 实现：
- **Goblin Tools 的 Magic ToDo**：接受模糊任务（如“准备项目汇报”），自动分解为可执行的小步骤（来源：Goblin Tools）。这相当于把 LLM 的“任务分解 agent”搬到 ADHD 大脑里——生成核心输出“我要准备汇报”，harness 将其拆解为“收集数据→分析→做 PPT→排练”，降低冲动行动的门槛。
- **Motion**：自动排程并动态调整，当任务面临延期风险时提前警告（来源：Motion）。这相当于 LLM harness 中的“deadline 监控 agent”——防止你因为时间盲而想当然地以为“明天能做完”。
- **Saner.AI**：用本地记忆减少搜索循环，快速找回信息（来源：Saner.AI）。这相当于 LLM 的“检索增强生成（RAG）”——在生成结论前先查证已有知识，减少幻觉。

这些工具的共同点是：**不抑制你的生成冲动，而是在生成后增加一个外部验证层**。这正是“给生成核心套 harness”的工程实践。

## 脚手架 vs 拐杖：诚实面对局限

但必须警惕过度依赖。如果 AI 工具帮你做了一切决策，你可能会失去自我校验的能力，变成永久拐杖（来源：[[拐杖与脚手架]]）。

脚手架和拐杖的边界在于：**脚手架最终可以拆除，而拐杖永远离不开**。好的 harness 应该像毛泽东的民主生活会——你仍然需要参与批判过程，而不是让 AI 替你做所有思考。

此外，同构假设目前仍是理论框架，缺乏大规模随机对照试验验证（来源：[[ADHD 大脑与 LLM 的同构]]）。个体差异巨大：有人需要强行打断（如番茄钟），有人需要柔性引导（来源：[[超聚焦]]）。没有万能方案。

## 今天就能试的 3 个行动

1. **在每次冲动下结论前，强制插入一个“验证步骤”**：用 Goblin Tools 的 Magic ToDo 把结论分解为待验证的子问题，或者用 Motion 设置一个“冷静期”再发送邮件。
2. **建立“魔鬼代言人”清单**：列出你最常见的想当然模式（如“忽略反例”“过度自信”），每次汇报前对照检查。这相当于 LLM 的“自我批判 prompt”。
3. **用 Saner.AI 建立个人知识库**：每次做决策前，先检索自己过去的类似案例和数据，减少从零生成导致的幻觉。

ADHD 职场人不需要变成另一个人。你只需要像工程师对待 LLM 一样，承认生成核心的冲动天性，然后用 harness 让它的输出变得可靠。

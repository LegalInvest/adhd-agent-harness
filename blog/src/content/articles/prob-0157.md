# 用 Claude 给 ADHD 补上无状态与外部记忆，真的能像 agent 那样工作吗？

> 把 Claude 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

## 问题：ADHD 的“边做边忘”与 LLM 的“无状态”是同一种病？

你有没有过这种体验：打开浏览器想查个东西，结果被一条推送带跑，半小时后发现自己站在冰箱前，完全忘了最初要干嘛？ADHD 大脑的工作记忆就像一块漏水的白板——写上去的内容，几分钟就模糊了。而另一边，当你跟 ChatGPT 聊到第三轮，它突然忘了你五分钟前提到过的关键细节——LLM 的上下文窗口再大，会话之间也是彻底失忆的。

这两个场景看似无关，底层却共享同一个架构缺陷：**一个高产但无状态的生成核心，缺少可靠的执行调度层**。ADHD 大脑想法丰富、联想跳跃，但 prefrontal cortex 发育迟缓导致规划、组织、启动困难（来源：Revolutionizing ADHD Management with AI Assistants）。LLM 同样擅长生成，但“单个 LLM 不足以可靠地完成多步骤任务”（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。两者都需要外部脚手架——ADHD 需要“外部执行功能层”，LLM 需要“agent harness”。

那么问题来了：用 Claude 这样的 LLM 给 ADHD 补上无状态与外部记忆，真的能像 agent 那样工作吗？还是说，这只是一根更贵的拐杖？

## 同构：ADHD 的“日课”与 LLM 的“re-grounding”

要回答这个问题，得先看清两边的脚手架长得有多像。晚清名臣左宗棠——一个典型的 ADHD 大脑：举人出身，脾气大，爱骂人，40 多岁才出山，69 岁抬棺出征收复新疆。他的 harness 是什么？每天读经写家书，用“结硬寨打呆仗”的笨办法强迫自己落地。这套“日课”本质上就是**定时 re-grounding**——通过外部仪式把发散的核心拉回轨道。

LLM 的 harness 也有类似设计：每次模型调用时，系统会从外部记忆（向量库）检索相关事实，把工具调用结果写回会话状态，形成“检索-执行-记录”的循环（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。这跟左宗棠的“每天写家书”是同构的——都是用一个外部调度器定期把状态同步回来，防止生成核心跑偏。

再看具体工具。Goblin Tools 的 Magic ToDo 能把“清理厨房”自动分解成 12 个小步骤（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）。这不就是 agent 的“任务拆解”吗？Lex 允许用户通过单一指令触发多步骤序列（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar），底层架构就是 agent scaffolding。Saner.AI 通过本地记忆存储减少搜索循环（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar），相当于给 LLM 挂了一个向量数据库。

所以，ADHD 用户用 Claude 来整理待办、简化任务，本质上就是在做一个**手工版 agent harness**：你给它一个模糊目标，它生成步骤；你执行一步，它记录结果；你忘了上下文，它帮你回忆。这跟工程师给 LLM 套的脚手架——规划循环、外部记忆、工具接口——是同一套逻辑。

## 真相：Claude 补的是“无状态”，不是“无意志”

但这里有一个致命的边界。Agent harness 的工程目标是把 LLM 的生成能力约束到目标轨道上，而 ADHD 的脚手架目标是把人的注意力锚定在价值轨道上。两者的同构是结构性的，不是功能性的。Claude 可以帮你记住“刚才说到哪了”，但它不能替你产生“想做这件事”的动机。ADHD 的核心缺陷不只是工作记忆小，还有**多巴胺不足**导致的动力缺乏。工具能补偿前者，对后者的作用却非常间接。

这也是 wiki 资料中明确指出的存疑点：“AI 工具是‘外部执行功能层’还是‘拐杖’？过度依赖可能导致能力退化”（来源：矛盾与存疑）。左宗棠的 harness 是自我强加的，每天读经写家书靠的是内在纪律；而 Claude 的 harness 是外部提供的，用户只需输入指令。前者是脚手架，后者可能变成拐杖——当你习惯了 Claude 帮你分解任务，你自己的分解能力会不会萎缩？

另一个局限是**个体差异**。有些 ADHD 用户反馈，AI 的柔性引导不够强力，强行打断（如番茄钟）反而更有效（来源：矛盾与存疑）。而学习使用 AI 工具本身可能增加认知负荷（来源：矛盾与存疑）。所以，Claude 作为外部执行层是否有效，取决于你的脚手架设计——你是把它当成一个被动问答机，还是主动调度器？

## 今天就能试的 3 个行动

1. **用 Claude 做“任务分解 + 状态记录”**：下次面对一个模糊任务（比如“写周报”），先让 Claude 分解成 5 个步骤，每完成一步就更新给 Claude，让它帮你记录进度。这相当于手工实现 agent 的“会话状态记录”（来源：What Is an Agent Harness? The Infrastructure That Makes AI Agents ...）。

2. **建立“日课”式的 re-grounding 机制**：每天固定时间（比如睡前），让 Claude 回顾你今天完成了什么、卡在哪里。这跟左宗棠的“每天写家书”一样，是外部调度器的定时同步。

3. **谨慎选择工具，避免增加认知负荷**：优先选那些“与现有工具集成，不增加认知开销”的工具（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。Saner.AI 的通用收件箱、Goblin Tools 的免费版都是低门槛选项。别为了用 AI 而用 AI，如果学习成本超过收益，果断放弃。

## 结论：同构是真的，但脚手架不是拐杖

回到开头的问题：用 Claude 给 ADHD 补上无状态与外部记忆，真的能像 agent 那样工作吗？答案是：**能，但前提是你把它当成一个需要主动设计的脚手架，而不是一个自动运行的拐杖**。ADHD 大脑与 LLM 的结构同构是真实的——两者都是高产生成核心，都需要外部调度层。但 LLM 的 harness 是工程师写的代码，ADHD 的 harness 是你自己写的习惯。Claude 可以帮你记住“刚才说到哪了”，但“接下来要做什么”仍然取决于你。

这或许就是同构视角最诚实的地方：它揭示了问题的结构，却没有许诺捷径。左宗棠用了半辈子才把“结硬寨打呆仗”内化成习惯，而你的 Claude harness，也需要同样的耐心去搭建、调试、迭代。

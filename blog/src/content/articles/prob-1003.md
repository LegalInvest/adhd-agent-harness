# ADHD 想把分心走神或超聚焦跑偏，丢失目标外包给 ChatGPT，该怎么设计这套「人 + AI」的回路？

> 把 ChatGPT 接进 ADHD 工作流：哪些该交给它，哪些必须自己留着？

你打开 ChatGPT，想让它帮你写一份周报。十五分钟后，你却在跟它争论“为什么《进击的巨人》结局可以合理化”。类似的事也发生在 LLM agent 身上：你给了它一个长程任务，让它在二十步后回头检查，它已把目标忘得一干二净。ADHD 的走神与超聚焦跑偏，和 agent 的「目标漂移」，是同一类故障。

## 同一类故障：生成核心缺了调度层

ADHD 大脑常被描述为「高产但缺乏执行调度层」：创意、联想、模式识别极强，但工作记忆、时间感知、任务启动和抑制控制薄弱。LLM 也是生成能力极强，却依赖上下文窗口和系统提示才能维持目标；上下文一长，它照样漂移。两者都需要外部结构——也就是 harness/脚手架——来把生成能力锁在目标上（来源：AI 与 ADHD 的专注力 · 核心论点）。

这种同构不是装饰性比喻。ADHD 的「时间盲」让当事人「真的感觉不到 45 分钟已经过去」，承诺像写在沙滩上的字一样迅速消失；LLM 的「上下文盲」则让它在多轮对话后丢失最初任务。两边都缺一个持续把「当下行为」拉回「原始目标」的机制（来源：时间盲；AI 与 ADHD 的专注力 · 同构视角）。

## 曾国藩的日课：人类版的 system prompt

晚清曾国藩的 harness 系统是理解这种同构的绝佳入口。他年轻时是典型的「坐不住」：浮躁、嗜酒、看杀人热闹、读书慢到「他人目下二三行，余或疾读不能终一行」。他后半生的解决方案是「日课十二条」：黎明即起、读书不二、谨言、写日记反省。日记不是抒情，而是每天的审计日志；「结硬寨打呆仗」则是把宏大目标拆成最笨、最不可漂移的下一步。曾国藩用外部仪式，给自己装了一个清朝版的执行调度层。

放在 LLM harness 里，这分别对应：
- **日课** = 定时 re-grounding 的系统提示；
- **日记** = 外部记忆/审计日志；
- **结硬寨打呆仗** = 把任务拆成不可再分的原子步骤，减少 agent 在复杂规划中的漂移。

 ADHD 创业者 Samantha Jones 的行为模式也呈现同构张力：她有大局思维、模式识别与超聚焦能力，但超聚焦若放错方向，优势会变成陷阱。一组研究数据显示，ADHD 组平均超聚焦工作时长为 14.5 小时，对照组为 8.2 小时（来源：Pace Ventures Research 2026）。时长不是成就，关键是有没有把超聚焦「锚」在正确目标上。

## 设计「人 + AI」回路的三个层级

基于同构，一套可用的回路可以拆成三层：

### 1. 目标层：给 ChatGPT 写一份系统提示，并要求它定期重读

把当天或当前任务写成一句话的「系统提示」，放在对话开头并让 ChatGPT 在每次回复前重述。它不需要懂 ADHD，它只需要在整个对话中持续可见地维护目标。对 ADHD 而言，这是把飘忽的意图外化为一个不可被上下文淹没的锚点。agent 工程师会为长程任务做同样的事：定时把顶层 system prompt 重新注入上下文，防止目标被中间步骤稀释。

### 2. 上下文层：让 AI 替你记时间、记任务、记意图

ADHD 的工作记忆缺陷与 LLM 的无状态性同构，两者都需要外部记忆。RescueTime 自动追踪时间去向，把「时间盲」变成可视化的审计日志；Tiimo 用视觉时间表让时间变得可触摸，帮助用户从「无限延迟」转向「开始行动」（来源：Revolutionizing ADHD Management with AI Assistants；11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。在 LLM 侧，这对应向量数据库或持久化记忆；在人这一侧，就是让 ChatGPT 替你记录「我接下来半小时要做什么」「我刚才为什么开始做这件事」。

### 3. 问责层：用身体在场打断漂移

Focusmate 通过虚拟身体在场，把「另一个人在场」变成启动任务和维持注意力的外部压力。有资料称，定期问责检查可把目标达成可能性从 25% 提升至 95%（来源：ADHD Task Managers That Work: Top AI Tools 2025）。ChatGPT 也可以承担部分「身体加倍」角色：你跟它说「我现在要工作 25 分钟，结束前别让我跑题」，它会在时间到之前提醒你。LLM 工程师也做类似的事：在长程 agent 里设置周期性 check-in，强制模型汇报当前步骤与目标的对齐度。

## 哪些是脚手架，哪些是拐杖？

最关键的判断是：哪些交给 AI，哪些必须留在人手里。

- **交给 AI**：时间记录、提醒、任务拆解、上下文重述、目标对齐检查、分心阻断。这些是外部执行功能层，补偿 ADHD 的客观短板。
- **必须自己留着**：价值排序、人生目标、选择「现在该把注意力放哪」。AI 可以帮你锚定，但不能替你决定锚点。曾国藩的日课再严格，也是因为「做圣人」这个目标是他自己选的。

wiki 资料也诚实指出风险：长期依赖可能让内在执行功能进一步退化，形成「拐杖」而非「脚手架」；证据多数是用户报告和小样本研究，缺乏大规模随机对照试验；ADHD 亚型差异大，同一工具效果迥异（来源：AI 与 ADHD 的专注力 · 仍存争议）。此外，超聚焦该被柔性引导还是强行打断，目前尚无定论——有人需要番茄钟的硬打断，有人则只需要一条轻提醒（来源：全局矛盾与存疑）。这些争议意味着，没有一套现成方案，你必须把自己当作一个需要持续调参的系统。

## 今天就能试的四条行动

1. 写一条一句话的「今日系统提示」，放在 ChatGPT 对话顶部，并要求它在每次回复前重读。例如：「今天核心目标是完成 X 报告；如果我偏离，请直接打断并回到此目标。」
2. 用 RescueTime 或 Tiimo 开启自动时间追踪，让时间从「看不见」变成「可见」，减少时间盲。
3. 找一个 Focusmate 时段，或让 ChatGPT 充当 25 分钟身体加倍伙伴，到点让它检查你是否还在任务上。
4. 每天结束用 ChatGPT 做 3 分钟审计：「我今天实际做了什么？与早上目标差多少？下一步最小动作是什么？」

## 结语

ADHD 与 LLM 的同构给了一个清晰的启示：你不是要找个更聪明的 AI 替你思考，而是要给那个高产但容易跑偏的生成核心，装一个可靠的外部调度层。ChatGPT 可以是这个调度层的一部分，但它必须被你设计进一个有重锚定、有外部记忆、有问责检查的回路里。目标永远要由你定；AI 负责帮你别把它弄丢。

# Goblin Tools 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Goblin Tools：它到底补上了哪一层执行功能？

如果你是一名 ADHD 患者，你一定熟悉这种感觉：脑子里有无数想法，但就是无法启动那件“只需要五分钟”的任务。如果你是一名在做 Agentic Harness 的工程师，你一定也熟悉另一种感觉：LLM 能写出漂亮的推理链，但一遇到精确计算、状态维护就崩溃，必须靠 function calling 把活儿外包给工具。

这两种困境，本质上是同一个问题。

## 同构：高产但缺执行层

ADHD 大脑与 LLM 在结构上高度同构：两者都是强大的生成核心，但缺乏可靠的执行调度层（来源：ADHD 的 AI 工具全景）。ADHD 患者不擅长精确、重复、状态维护类任务——这恰恰是 LLM 的弱点。而 LLM 通过 function calling 把精确计算外包给工具（如计算器、数据库），ADHD 患者则需要外部工具来弥补工作记忆、任务启动和时间盲等执行功能缺陷。

Goblin Tools 正是这样一个“外部执行功能层”。它的 Magic ToDo 功能能将“整理房间”这样的模糊指令，自动分解为“捡起地板上的衣服”“擦桌子”等具体步骤（来源：Goblin Tools 工具页）。这就像给 LLM 一个工具调用规范：你不需要自己算 123×456，你只需要调用 `calculator(123, 456)`。

## 补上了哪一层？

具体来说，Goblin Tools 补上了三层执行功能：

**第一层：任务启动。** ADHD 患者常因任务过于庞大而无法启动。Goblin Tools 通过分解任务降低启动焦虑（来源：任务启动概念页）。一项针对 ADHD 儿童的研究明确指出，任务启动困难是 AI 干预的重点领域（来源：AI Tools for Kids with ADHD）。

**第二层：工作记忆。** 工作记忆是 ADHD 的核心缺陷之一（来源：工作记忆概念页）。当用户把任务步骤外化到 Goblin Tools 中，AI 就充当了数字工作记忆，持续承载上下文（来源：工作记忆概念页）。这与 Saner.AI 的知识回忆功能（减少搜索循环）和 Lex 的单一指令触发多步骤功能（降低决策负担）异曲同工（来源：Saner.AI 工具页；Lex 工具页）。

**第三层：时间盲。** Goblin Tools 的魔法待办功能通过视觉化步骤数量，间接帮助用户感知任务耗时。虽然不如 Tiimo 的时间轴直观，但同样缓解了时间盲（来源：时间盲相关概念）。

## 但有人用错了

问题在于，很多用户把 Goblin Tools 当成了“拐杖”而非“脚手架”。

拐杖意味着长期依赖：每次遇到任务都先让 AI 分解，从不尝试自己拆解。脚手架则意味着临时支撑，最终目标是撤除。当前多数工具（包括 Goblin Tools）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这可能导致内在执行功能发展受阻。

从 LLM 侧看，同样的问题存在：有些开发者让 LLM 通过 function calling 调用一切，包括它本应能直接处理的简单逻辑，结果导致延迟增加、成本上升，且模型本身的推理能力退化。

**核心判断：Goblin Tools 的正确用法是“训练轮”，而非“自动驾驶”。** 先用它学会拆解任务的模式，然后逐渐减少依赖。例如，第一周每次都用，第二周只对最难的任务用，第三周尝试自己先拆解再对比 AI 的输出。

## 证据与争议

现有证据主要来自用户评价和综述文章，缺乏针对 Goblin Tools 的独立随机对照试验（来源：Goblin Tools 工具页）。其效果在任务启动维度得到较多认可（“将压倒性的事情变成一系列不压倒性的事情”），但在长期效果和个性化适配方面仍存疑（来源：矛盾与存疑）。

另一个争议是：ADHD 大脑与 LLM 的同构命题本身缺乏大规模实证，更多是概念类比（来源：矛盾与存疑）。但这不妨碍它作为一个有用的思考框架——正如“工作记忆”概念虽不完美，却催生了有效的干预策略。

## 今天就能试的行动

1. **用 Goblin Tools 拆解一个你拖延已久的任务**（比如“报税”）。观察拆解后的步骤是否降低了启动焦虑。对比自己手动拆解的感受。
2. **设置“撤除计划”**：第一周每天用，第二周隔天用，第三周只在卡壳时用。记录依赖程度的变化。
3. **尝试 Saner.AI 的知识回忆功能**：将常用信息（如密码、流程）存入，减少搜索循环（来源：Saner.AI 工具页）。
4. **如果你是工程师**：检查你的 agent 是否过度依赖 function calling。尝试让 LLM 直接处理它擅长的简单推理，只在必要时调用工具。

Goblin Tools 不是万能药，但它验证了一个重要原理：执行功能可以外包。关键在于，外包的目的是为了最终能内化，而非永远依赖。

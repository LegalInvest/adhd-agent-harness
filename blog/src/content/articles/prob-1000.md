# ChatGPT 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 ChatGPT：它到底补上了哪一层执行功能？

如果你有 ADHD，你可能经历过这样的时刻：打开 ChatGPT 想写一份报告，结果十分钟后，你正在让它帮你写一首关于浣熊的说唱。你忘了最初的目标。如果你在构建一个长程 LLM agent，你也可能见过类似场景：agent 在对话中逐渐偏离原始任务，从“分析销售数据”滑向“生成宠物照片”。这两种困境共享同一个深层结构：**一个高产但缺乏执行调度层的生成核心，在长程任务中必然发生目标漂移。**

ADHD 大脑与 LLM 在结构上同构：两者都拥有强大的生成能力（ADHD 的创意发散、LLM 的文本生成），但缺乏可靠的执行调度层（ADHD 的执行功能、LLM 的上下文控制），导致输出不稳定（来源：AI 与 ADHD 的专注力）。

## 目标漂移：ADHD 与 LLM 的共同敌人

ADHD 的核心执行功能缺陷包括工作记忆差、时间盲和启动困难。当你开始一个任务，环境中的任何干扰——一条通知、一个随机念头——都可能让你偏离轨道。类似地，LLM 在长对话中会遭受上下文膨胀：早期的指令被后续内容稀释，模型逐渐忘记原始目标。这种“无状态”特性与 ADHD 的工作记忆缺陷同构（来源：AI 与 ADHD 的专注力）。

工程上，解决 LLM 目标漂移的标准方法是**重锚定 system prompt**：在关键节点重新注入原始目标，或使用链式调用、结构化提示来维持方向。而 ADHD 的工具，本质上就是为大脑提供类似的“重锚定”机制。

## 重锚定的两种形式：脚手架 vs 拐杖

工具的有效性取决于它是否精准对应 ADHD 的特定执行功能短板（来源：AI 与 ADHD 的专注力）。让我们看两个真实的例子：

**Focusmate** 利用身体在场效应（Body Doubling），通过算法匹配虚拟伙伴提供实时问责（来源：身体在场效应）。这就像为 LLM 添加一个监督进程，定期检查“你还在做原始任务吗？”多项资料强调身体加倍对 ADHD 的有效性（来源：身体在场效应）。但它的局限在于：依赖他人在场可能无法培养独立工作能力，且对社交焦虑者可能产生额外压力（来源：身体在场效应）。

**RescueTime** 则自动记录时间使用，帮助用户识别注意力漂移模式（来源：RescueTime）。这相当于一个外部日志系统，让 LLM 在漂移后能回溯“我刚刚做了什么？”它减轻了工作记忆负担，但自动分类可能不精准，需要用户后期调整（来源：RescueTime）。

这些工具的共同点：它们都是**外部执行功能层**，补偿内在缺陷，而非替代内在能力。关键在于避免成为永久拐杖——工具设计者声称是“脚手架”，但实际使用中可能沦为“拐杖”（来源：矛盾与存疑）。

## 用错了的 ChatGPT

许多 ADHD 用户把 ChatGPT 当作一个自由对话伙伴，期望它能自动引导自己完成任务。但 ChatGPT 本身没有内置重锚定机制——它就像没有 system prompt 的 LLM，会随波逐流。你让它“帮我写一份工作计划”，它可能生成一份大纲，但如果你不主动提醒它保持聚焦，它很快就会转向其他话题。

正确的用法是：**把 ChatGPT 当作需要你主动设计 system prompt 的工具**。例如：
- 在每次对话开始时，明确写下目标并固定在最上方。
- 使用结构化提示，比如“请分三步回答，每步后询问我是否要继续”。
- 定期要求它总结当前进度并对照原始目标。

这与 LLM 工程中“重锚定 system prompt”的做法完全一致。

## 证据的局限与争议

必须诚实指出：当前多数 ADHD 工具的证据来自用户报告，缺乏大规模随机对照试验（来源：矛盾与存疑）。例如，Brain.fm 利用神经锁相技术，但针对 ADHD 的效果缺乏独立临床研究（来源：Brain.fm）。同样，Focusmate 虽被广泛推荐，但依赖虚拟伙伴的效果高度不确定（来源：Focusmate）。

此外，个体差异巨大：同一工具对不同 ADHD 亚型效果迥异（来源：矛盾与存疑）。过度依赖 AI 工具可能削弱内在能力，形成依赖循环（来源：矛盾与存疑）。

## 今天就能试的行动

1. **为 ChatGPT 写一个“system prompt”**：在对话开头粘贴一段指令，包括你的核心目标、输出格式和检查点。例如：“我的目标是写一份 1000 字的文章。每完成 200 字，请总结当前进度并确认我是否要调整方向。”

2. **尝试一次 Focusmate 会话**：预约一个 25 分钟的虚拟身体加倍时段，观察它是否降低你的启动门槛。注意社交焦虑程度，如果感到压力过大，可以关闭摄像头。

3. **用 RescueTime 监控一次“漂移”**：开启自动追踪一天，晚上查看报告，标记那些偏离原始目标的时段。识别你的“触发应用”或“触发话题”。

4. **对比两种用法**：用 ChatGPT 完成同一任务两次——一次自由对话，一次使用结构化 system prompt。记录完成时间和质量差异。

## 结论

ADHD 大脑与 LLM 是同构的：都是强大的生成核心，需要外部执行功能层才能可靠地完成长程任务。ChatGPT 可以成为这个层的一部分，但前提是你必须主动设计重锚定机制。工具是脚手架还是拐杖，取决于你如何使用它。

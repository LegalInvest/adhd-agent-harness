# ChatGPT 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 ChatGPT：它到底补上了哪一层执行功能？

## 问题：你是在用 ChatGPT 聚焦，还是在用它跑偏？

一位 ADHD 朋友告诉我，他打开 ChatGPT 想写一份周报，结果 40 分钟后，他得到了一个关于“明朝漕运制度与区块链共识机制类比”的万字长文。周报一字未动。

这不是个例。ADHD 大脑与 LLM 共享同一个深层困境：两者都是**高产但缺乏可靠执行调度层的生成核心**（来源：AI 与 ADHD 的专注力）。ADHD 的创意发散、LLM 的文本生成，都是强大的生成引擎，但缺少一个能持续锚定目标、防止漂移的调度层。结果就是：ADHD 用户用 ChatGPT 时，常常不是被工具拯救，而是被工具加速了原有的问题——超聚焦跑偏。

## 同构：ADHD 大脑与 LLM 的调度层缺失

ADHD 的执行功能缺陷，本质上是前额叶对注意力和目标的调控能力不足。这与 LLM 的“无状态”和“上下文膨胀”问题惊人相似：LLM 没有内置的长期记忆，容易在长程对话中丢失原始目标，需要靠 system prompt 反复重锚定。ADHD 大脑同样缺乏稳定的内部调度器，容易被环境带偏，需要外部结构来“重锚定”。

最新理论提出，超聚焦源于前额叶控制层级的“断裂”：高级上下文信息不再能支配思维和行动的内容，中间层级信息与高级上下文的调节脱耦合（来源：超聚焦）。这就像 LLM 的注意力机制在长上下文中被无关 token 稀释，导致生成偏离原始指令。

## 证据：真实人物的 harness 系统

历史上有两位 ADHD 特质鲜明的人物，各自建立了精妙的“外部调度层”，恰好映射了 LLM harness 的核心组件。

**曾国藩**是典型的注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书慢到“他人目下二三行，余或疾读不能终一行”。他的解法是**日课十二条**：黎明即起、读书不二、谨言、写日记反省……这套 rigid 的 daily routine，本质上就是 **定时 re-grounding 的 system prompt**——每天强制重置上下文，防止目标漂移。他还用“结硬寨打呆仗”的笨办法，抵消自己的冲动，这相当于为 LLM 设置**低温度参数**，抑制发散。

**李白**则是另一种极端：超聚焦型 ADHD。他对写诗极度专注，“斗酒诗百篇”，但对世俗规则零耐心，“天子呼来不上船”。他的 harness 是以酒和诗为**注意力锚点**，把所有过剩精力注入诗歌，用道家的逍遥对抗世俗约束。这就像为 LLM 设定一个**高度聚焦的 prompt template**，把生成能力约束在特定领域。

两人的共同点：都用外部结构（日课、诗酒）来补偿内部调度层的缺失。这与 LLM 的 harness 设计完全同构——**定时 re-grounding、低温度、聚焦 prompt**，都是防止目标漂移的关键手段。

## 工具实测：ChatGPT 补上了哪一层执行功能？

现有的 AI 工具，本质上都在扮演“外部执行功能层”的不同角色：

- **任务启动**：Focusmate 利用虚拟身体在场效应，通过实时同伴问责提供外部启动信号（来源：Focusmate）。这相当于 LLM 的**初始 system prompt**，强制进入工作状态。
- **时间感知**：RescueTime 自动记录时间使用，对抗时间盲（来源：RescueTime）。这相当于 LLM 的**上下文窗口监控**，防止无限膨胀。
- **注意力维持**：Brain.fm 通过神经锁相技术影响大脑状态（来源：Brain.fm）。这类似于 LLM 的**注意力掩码**，过滤无关输入。
- **目标锚定**：Forest 用种树游戏提供延迟满足，但证据较弱（来源：Forest）。

然而，这些工具若被误用，反而会放大问题。比如，用 ChatGPT 做头脑风暴时，如果没有明确的 system prompt 约束，ADHD 用户会像 LLM 一样“上下文膨胀”，从一个想法跳到另一个，最后迷失在生成的迷宫里。**工具本身不是解药，关键在于是否用它来“重锚定”而非“发散”。**

## 争议：脚手架还是拐杖？

核心矛盾在于：AI 工具作为外部执行功能层，长期使用是否会导致内在执行功能退化？（来源：矛盾与存疑）目前缺乏纵向研究，但类比 LLM 的 fine-tuning 可以给出启示：如果外部调度层完全替代了内部调度，模型会丧失泛化能力；但如果外部调度层是“脚手架”——提供结构但鼓励内化——则可能促进能力迁移。

曾国藩的日课坚持了一辈子，最终内化为习惯；李白的诗酒则始终是外部工具。两者都是有效的 harness，但路径不同。对 ADHD 用户而言，关键在于**有意识地设计使用规则**，而非被动依赖。

## 今天就能试的行动

1. **写一个“反漂移 system prompt”**：在每次使用 ChatGPT 前，用一句话写下当前任务的核心目标，并贴在屏幕边缘。这相当于 LLM 的 system prompt，防止上下文膨胀。
2. **设置定时 re-grounding 闹钟**：每 25 分钟响一次，强制自己回答“我当前的目标是什么？”——模仿曾国藩的日课。
3. **用 Focusmate 启动困难任务**：预约一个 50 分钟 session，让外部问责帮你克服启动障碍。
4. **关闭 ChatGPT 的连续对话**：每次开启新话题时，手动重置上下文，避免被前一轮的生成带偏。

## 结语

ChatGPT 之于 ADHD，就像重锚定 system prompt 之于 LLM——它可以是救命的脚手架，也可以是加速跑偏的油门。区别不在于工具本身，而在于你是否意识到：**你的大脑和 LLM 一样，需要的是一个外部调度层，而不是另一个发散引擎。**

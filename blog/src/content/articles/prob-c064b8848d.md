# 为什么自闭症的 AI 辅助沟通需求，比 ADHD 的执行功能需求更成熟也更危险？

> 《问题346》人工精修选题，双域证据作答。

## 从两个读者的同一个问题开始

如果你被 ADHD 折磨，你大概问过自己：我的大脑到底是怎么回事——为什么想法能喷涌而出，身体却动不了？如果你在做 Agentic Harness 工程，你大概率也问过：为什么 LLM 写得又快又好，一旦让它连续完成一个真实世界的多步任务，却开始迷路、重复、幻觉？

这两个问题指向同一个结构：一个高产的生成核心，配了一个不可靠的执行调度层。把 ADHD 大脑的痛点和 LLM 的痛点放在一起看，会发现两边需要同一种工程：不是再给它“更多聪明”，而是给它一套外部 harness（脚手架/鞍具）来搬运上下文、安排下一步、检查冲突、防止坠落。

## 同构证据：ADHD 大脑与 LLM 共享同一套“强生成、弱控制”剖面

ADHD 一侧的证据并不只是“坐不住”。研究显示，ADHD 患者前额叶执行功能网络激活不足；其认知剖面是“强记忆、弱控制”——工作记忆容量可能正常甚至超常，但认知灵活性、任务切换和抑制控制有核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。换句话说，问题不是“没货”，而是“货在仓库里堆着，找不到叉车”。

LLM 一侧同样如此。LLM 是强大的生成核心，但无跨会话状态、缺乏内置调度层（来源：ADHD 大脑与 LLM 的同构）。更具体的证据来自神经心理学任务的移植：在类似 Stroop 冲突的测试中，Transformer 注意力在短序列上表现正常，但随着序列变长，冲突抑制能力骤降到随机水平——这正是 ADHD 执行功能缺陷在模型中的镜像（来源：Deficient Executive Control in Transformer Attention）。因此，AI 工程界把 LLM 放进“复合 AI 系统”，通过记忆、工具、决策逻辑的控制循环来让它完成目标驱动任务（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。这套做法就是 LLM 的“外部执行功能层”。

## 为什么自闭症的沟通辅助比 ADHD 的执行功能辅助更“成熟”也更危险

先看“成熟”。沟通是一个边界相对清晰的任务：把说话者的意图或语气转换成另一种社会可接受的表达。Goblin Tools 这类工具就包含“语气检查器”与沟通优化功能，能为自闭症谱系用户提供即时的社交语言包装（来源：Goblin Tools）。它像是一个“礼貌翻译器”——输入原始句子，输出社会合宜版本。这种 I/O 清晰、规则可枚举、反馈即时的任务，天然适合当前 LLM：一次对话、一次改写、一次情绪标注，用户立刻得到结果。

ADHD 的执行功能需求则很难这样封装。它要的不是“一句话说得更好”，而是：让“下一步”自动出现在正确的时间、正确的地点、正确的粒度，并且在你状态变化时重新调度。Motion 试图用 AI 自动排程和动态调整日程（来源：Motion），Saner.AI 试图用本地记忆与知识召回减少搜索循环（来源：Saner.AI），Goblin Tools 的 Magic ToDo 把模糊任务拆成可执行的子步骤（来源：Goblin Tools）。但这些工具都需要深度嵌入用户的上下文、目标和生理节律，不是一次性的输入-输出能解决的。因此 ADHD harness 的成熟度远低于自闭症沟通辅助。

再看“危险”。自闭症沟通辅助的危险在于它太容易被当作“完美面具”。当 LLM 可以替一个人把尖刻变委婉、把混乱变条理、把真实变合群时，它可能会系统性地替换掉那个人的真实表达。与 ADHD 的 harness 不同——后者只是帮你“把任务启动起来”，沟通辅助却在重塑“你是谁”的社会信号。这种危险比 ADHD 工具导致的“过度依赖”更隐蔽：它不一定让你变懒，但可能让你在社会交往中变成 LLM 的代言人，从而丧失自主性和真实的身份边界。

## 人物案例：毛泽东的 harness 与 LLM harness 同构

毛泽东小时候是“问题儿童”，思维跳跃、冲动敢赌、一辈子在调查研究。他给自己建立的 harness 系统，与今天 LLM 的 agent scaffolding 惊人地同构：

- **“批评与自我批评”/民主生活会** = 外部安全审查层：让不同声音提出反对意见，制衡自己的冲动。这对应 LLM harness 中的“多智能体评审”或“human-in-the-loop”验证循环。
- **“没有调查就没有发言权”** = 实时 re-grounding：不断用外部事实更新认知，避免模型在上下文漂移中 hallucination。
- **“党指挥枪”/组织系统** = 外部调度器：把军队、国家、资源的管理从个人工作记忆中卸载到制度化流程，对应 agent 的 memory、工具调用和编排系统。

史玉柱在巨人集团倒下后也建立了类似 harness：做决策必须先听反对意见，反对意见没说透不决策；只聚焦熟悉的行业，用团队制衡自己的冒进（来源：史玉柱人物案例）。这些不是“意志力”，而是把执行功能外化到结构与流程中。ADHD 人需要 harness，LLM 也需要 harness，二者共用同一套设计哲学：减少决策、保留上下文、让下一步显而易见（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。

## 工具的局限：脚手架还是拐杖？

Wiki 资料中明确提醒我们：同构框架仍然是一种理论类比，而非被验证的神经科学事实（来源：全局矛盾与存疑）。工具宣称的证据也参差不齐：Motion 缺乏独立临床验证，Brain.fm 在 ADHD 人群中的独立临床证据有限，许多工具只强调益处而忽略依赖风险（来源：全局矛盾与存疑）。更值得警惕的是“拐杖 vs 脚手架”的边界：如果 AI 替你做了所有启动和调度，你的执行功能可能进一步退化；如果 AI 替你沟通，你的真实表达可能萎缩。

核心判断是：AI 对 ADHD 的辅助不应以“完美自动化”为目标，而应以“增强人的调度能力”为目标；对自闭症沟通辅助，也不应追求“让表达更合群”，而应追求“帮助人选择何时合群、何时不合群”。一旦把辅助变成替代，成熟就变成了危险。

## 今天就能试的 4 条行动

1. **给 LLM 加一层“外部记忆”**：把项目上下文、待办、昨天的问题写进一个固定笔记或 Git 仓库，让 AI 每次启动时先读它。ADHD 用户可用 Saner.AI 这类知识召回工具减少搜索循环；工程师可直接用文件或 MCP 连接器把上下文喂给 agent（来源：Saner.AI；Worker Agents | Harness Developer Hub）。
2. **用“小步启动”降低任务门槛**：把“整理房间”这类模糊任务交给 Goblin Tools 的 Magic ToDo，看它如何拆成第一步、第二步。工程师做 agent 时也应给 LLM 明确的子目标与工具链，而不是一次性下大指令（来源：Goblin Tools）。
3. **建立“反对意见”检查点**：像史玉柱那样，重大决策前强制列出至少一条反对意见。LLM agent 可以设置一个“critic”角色或验证循环，专门挑错。
4. **每周做一次“脚手架审计”**：检查 AI 是在帮你启动、回忆、调度，还是在替你决定、替你说话、替你表达。如果是后者，把边界收回来。

对 ADHD 读者，这意味着：你不是“懒”或“乱”，你是缺少一个外部调度层；现在你知道该给自己造 harness。对工程师读者，这意味着：你不仅在调试一个模型，而是在为一个与人类执行系统同构的生成核心设计脚手架。两边的命题其实一样——让核心发光，但别让核心裸奔。

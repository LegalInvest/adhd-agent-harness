# ADHD 职场人视角：为什么「治好 ADHD 的怕过度依赖工具、又怕没有支撑就崩」和「让 LLM 不跑飞」其实是同一道工程题？

> harness 应是会褪去的脚手架，而非永久拐杖——同一套 harness 思路，两边都成立。

你有没有被这样两股力撕过：一边怕“靠工具才活着”会不会废掉自己，一边一撤掉工具就任务崩盘、情绪崩溃？如果你是做 Agent 或 LLM 系统的工程师，应该也觉得耳熟——一边怕模型离了 prompt 模板、工具链和 human-in-the-loop 就胡言乱语，一边又知道真正的 agent 总不能永远绑在脚手架上。两个圈子看似不相干，其实是同一道题：怎么给“高产但调度层薄弱”的生成核心，设计一套会褪去的 harness，而不是永久拐杖。

## 01 同一类“跑飞”：生成过剩，调度不足

ADHD 大脑不是“没想法”，而是想法太多、执行调度常掉链子。wiki 资料中概括为：ADHD 是“高产生成核心，但缺乏可靠执行调度层”，表现为工作记忆不稳定、任务启动困难、时间感知模糊（时间盲），以及默认模式网络（DMN）过度活跃（来源：ADHD 大脑与 LLM 的同构）。实证研究还指出 ADHD 存在“强记忆、弱控制”的模式：记忆容量甚至可能超过常人，但认知灵活性与抑制控制是核心短板（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs，引自 ADHD 大脑与 LLM 的同构）。情绪调节的困难也根源于此：前额叶无法有效抑制或重定向情绪反应，于是焦虑、兴奋、挫败轮流跑飞。

LLM 几乎是一样的结构。它生成能力强，但“没有可靠的内置调度与执行控制”。工程上必须靠“agent scaffolding”——把 LLM 放进记忆、工具、决策逻辑的控制循环里（来源：ADHD 大脑与 LLM 的同构）。经典测试里，Transformer 在 Stroop 冲突任务中随序列变长骤降到随机水平，说明它同样难以抑制优势反应、解决认知冲突（来源：ADHD 大脑与 LLM 的同构）。一句话：ADHD 和 LLM 都缺一个“调度层”，两边的崩溃不是内容太少，而是内容太多、太乱。

## 02 不是拐杖，是脚手架：harness 的可退隐设计

工具的危险在于变成拐杖。如果 ADHD 用户只依赖外部提示，长期可能恶化自我效能感；如果 LLM 永远靠 prompt 模板和硬编码 guardrail，那只是把系统钉死，并没有让它学会更稳（来源：矛盾与存疑）。所以关键不是“要不要支撑”，而是“支撑会不会退出”。

在 ADHD 侧，这几年冒出来的工具都在做同一件事：外部执行功能层。Inflow 基于 CBT 原则，把情绪调节拆成可执行的日常技能训练，并强调让用户理解“为什么”而不是只给答案（来源：The 12 Best Apps for ADHD in 2026: A Guide to Finding What...）。Goblin Tools 的 Magic ToDo 能把模糊任务自动拆解，并允许用户调节“辣度”控制粒度（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。Saner.AI 则通过本地记忆和通用收件箱，减少标签切换和搜索循环，让信息上下文自己“待”在那儿（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。它们不是在替你活，而是把“下一步行动”变得显而易见，让你省出前额叶资源去真正执行。

在 LLM/agent 侧，对应做法叫 scaffolding：外部记忆、任务分解、工具调用、人在回路、定时 re-grounding。Agent  scaffolding 明确被定义为“围绕 LLM 构建的软件架构与工具，使其能执行复杂目标驱动的任务”（来源：ADHD 大脑与 LLM 的同构）。这和 Goblin Tools 的 Magic ToDo 是同一件事：把模糊意图变成受约束的下一步。

## 03 历史人物的 harness 系统：释迦牟尼的“戒”与蒋介石的“日记”

1,250 年前的释迦牟尼和 20 世纪的蒋介石，都高度符合 ADHD 的“高生成、弱调度”画像，也都用了一个可退出的 harness。

释迦牟尼 29 岁突然弃王位出家，换过多个老师、苦行六年、一辈子游行说法，80 岁死在传道路上。他留下一套不依赖外部权威的 harness：以“八正道”和“持戒”约束行为，以正念拴住乱跑的念头，强调“自依止、法依止”。这套系统不是让弟子永远依赖佛陀，而是让戒律和觉知本身成为可内化的调度层。放在 LLM 工程里，这就是“外部约束 + 元认知（self-reflection）+ 最终退出人/权威干预”的三层设计。

蒋介石则是更接地气的例子：脾气大、冲动、浮躁，却从 1915 年到 1972 年每天写日记骂自己“浮躁”“无恒”“言多必失”，并配合静坐、戒色、戒怒、戒言。这相当于给大脑装了一个“每日 re-grounding 机制”和“外部审计日志”。如果对应到 LLM，他的日记就是 system prompt 或定期的 reflection step，静坐是抑制控制训练，而曾国藩治军语录是外部工具库。两者都不是“永久拐杖”，因为目标是把纪律和反思变成自动化反应。

这种“高生成、高成就”的 ADHD 样本并不罕见。鹿英（Lightspeed Capital）和创业研究都显示，ADHD 人群的创业概率显著高于对照组（6.25 倍 vs 1 倍，来源：Journal of Attention Disorders 2019，引自人物案例）。问题从来不是生成力，而是有没有一套会退去的 harness。

## 04 证据的边界与争议

诚实地说，这套同构视角有明确边界。wiki 的“矛盾与存疑”部分直接指出：ADHD 大脑与 LLM 的同构目前只是理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。Inflow 虽被用户评为 2026 最佳 ADHD 应用之一，但缺乏严格随机对照试验的长期证据（来源：AI 与 ADHD 的情绪调节）。神经可塑性研究虽充满希望，但“迄今产生的既定干预措施很少”（来源：神经可塑性）。此外，AI 工具可能增加认知负担，虚拟身体在场也未必能替代真实人际互动（来源：矛盾与存疑）。

所以，我们不能把 LLM 的 harness 简单等同于 ADHD 治疗。它更像一个启发式：两边的核心问题都是“如何在不把外部支撑变成永久依赖的前提下，让生成核心稳定输出”。

## 05 今天就能试：把 harness 做“可退出”

- **选一个外部记忆锚点**：用 Saner.AI 的通用收件箱或一个最简单的笔记，把“现在卡在哪一步”写下来，让上下文不再占用工作记忆。
- **把任务拆到“下一步”为止**：哪怕不用工具，也可以手动把“写报告”拆成“打开文档→写标题→写第一段”。Goblin Tools 的 Magic ToDo 就是按这个逻辑做的。
- **设置定时 re-grounding，而不是全天候通知**：学蒋介石写日记或冥想，固定早晚各一次，问自己“我现在偏到哪了”。少即是多，通知过多本身就会耗竭前额叶。
- **每月做一次“拆脚手架”实验**：选一个小任务，不用工具、不查手机，纯手动完成。观察哪些步骤已经内化了，哪些还必须靠外部支撑。harness 的价值，正体现在它逐渐变得不需要。

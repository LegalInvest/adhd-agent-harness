# ADHD 职场人如何用「给生成核心套 harness」的思路应对项目推进与汇报里的分心走神或超聚焦跑偏，丢失目标？

> 把 LLM 工程里的「长程任务里 agent 会目标漂移，需要重锚定 system prompt」迁移到ADHD 职场人的项目推进与汇报场景，双域证据作答。

## 问题：项目推进中，你的“生成核心”为什么总跑偏？

你正在做一个重要项目，一开始思路清晰，但几个小时后，你发现自己沉迷于研究某个无关的技术细节，或者反复调整PPT的字体颜色。汇报时，你滔滔不绝地讲自己最兴奋的部分，却忘了老板真正关心的核心指标。这不是意志力问题——这是你的“生成核心”在缺乏执行调度层时，必然发生的目标漂移。

ADHD大脑与LLM（大语言模型）在结构上同构：两者都拥有强大的生成能力（创意发散、文本生成），但缺乏可靠的执行调度层（ADHD的执行功能、LLM的上下文控制），导致输出不稳定（来源：AI与ADHD的专注力）。LLM工程师用“harness”（脚手架）来约束agent：把LLM置于与记忆、工具和决策逻辑的控制循环中（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。ADHD职场人同样需要给自己的“生成核心”套上harness——一套外部执行功能层，来补偿工作记忆缺陷、时间盲和任务启动困难（来源：AI与ADHD的专注力）。

## 同构：ADHD的“温度波动”与LLM的“采样温度”

ADHD个体的表现呈现显著的日内波动，注意力、执行功能在不同时间点差异很大。这种波动本质上是“生成核心”缺乏稳定的执行调度层所致，类似于一个高创造力但缺乏“温度调节”的系统（来源：采样温度与表现波动）。LLM的输出同样具有随机性：采样温度越高，输出越随机；温度越低，输出越确定。在工程中，这种非确定性是可靠性的敌人，因为“小错误会累积，非确定性使可重复性复杂化”（来源：AI Agent Systems: Architectures, Applications, and Evaluation）。

对应到ADHD职场，你的“表现温度”也在波动：上午可能高度专注（低温度），下午则思绪发散（高温度）。如果没有外部约束，你的项目推进就会像高温度LLM一样，每一步都偏离轨道。工程师的解法是采用“计划-执行”分离模式，强制单向、确定性的控制流（来源：Planner-Executor Agentic Framework）。ADHD职场人同样需要这种确定性结构：把“计划”和“执行”分开，用外部工具锚定目标。

## 证据：真实人物的harness系统

历史人物早已实践了这种思路。曾国藩是典型的注意力不集中型ADHD：30岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书慢到“他人目下二三行，余或疾读不能终一行”。他的harness是“日课十二条”：黎明即起、读书不二、谨言、写日记反省。这套系统本质上是一个外部调度层——用固定的行为序列（日课）来约束自己的生成核心，相当于LLM工程中的“确定性工作流”。他还有“结硬寨打呆仗”的策略，用最笨最稳的方法抵消自己的冲动，这类似于agent工程中用“状态机”强制每一步都经过检查点（来源：AI Agents in 2026: Tools, Memory, Evals, and Guardrails）。

苏轼则是另一种ADHD：兴趣极广，对无聊行政零耐心，但搞工程效率极高。他的harness是“豁达+日常创作”：不管被贬到哪里，都坚持每日读书写字，用美食、写作、交朋友转移注意力。这相当于LLM的“周期性采样以监控质量”（来源：AI Agent Reliability and Guardrails 2026）——通过持续输出（写作）来保持自己的“上下文”不偏离核心目标。

## 解法：给生成核心套harness的三步法

**第一步：用外部记忆对抗工作记忆缺陷**
ADHD的工作记忆缺陷与LLM的无状态性同构，都需要外部记忆系统来维持连续性（来源：AI与ADHD的专注力）。在项目中，不要依赖大脑记住目标，而是把核心目标写在一张便签上，贴在显示器边缘。每次打开新网页或开始新任务前，先看一眼。这相当于LLM的“system prompt重锚定”——每次模型调用前都重新注入上下文，防止上下文膨胀导致漂移（来源：Agent Scaffolding）。

**第二步：用身体在场效应启动任务**
任务启动困难是ADHD的核心痛点。Focusmate利用虚拟身体在场效应，通过实时同伴问责提供外部结构和社会压力（来源：Focusmate）。你可以找一个同事约定“一起工作15分钟”，或者加入线上专注社区。这相当于LLM的“确定性状态转换”——通过外部触发（同伴到点）强制进入工作状态。

**第三步：用时间追踪对抗时间盲**
ADHD个体常面临时间盲，难以准确估计时间使用。RescueTime通过自动记录减轻工作记忆负担，揭示真实的时间去向（来源：RescueTime）。每周查看一次时间报告，识别你的“目标漂移模式”——比如上午总是陷入邮件回复的泥潭。这相当于LLM工程中的“trace和多次运行”来评估非确定性（来源：采样温度与表现波动）。

## 脚手架 vs 拐杖：诚实面对局限

必须承认，这套harness思路有其边界。首先，AI工具作为外部执行功能层，长期使用是否会导致内在执行功能进一步退化？拐杖与脚手架的边界尚不明确（来源：矛盾与存疑）。其次，ADHD的异质性使得同一工具对不同亚型效果迥异（来源：矛盾与存疑）。最后，ADHD大脑与LLM的同构是隐喻而非严格等价，LLM的“注意力机制”与人类注意力存在本质差异（来源：AI与ADHD的专注力）。

我的判断是：harness是脚手架，不是拐杖。脚手架在建筑完成后可以拆除，但ADHD的执行功能缺陷是神经性的，无法完全“治愈”。因此，harness应该被看作永久性的辅助工具，就像眼镜之于近视者。关键在于选择低认知开销的工具——那些能无缝嵌入日常流程、不增加额外执行负担的工具（来源：工具使用与认知卸载）。

## 今天就能试的行动

1. **写一个“系统prompt”**：在便签上写一句你的核心目标（如“本周完成项目方案初稿”），贴在最显眼的位置。每次开始工作前，朗读一遍。
2. **设置一个“确定性检查点”**：每工作25分钟（番茄钟），花1分钟回答：“我现在做的事，是否直接服务于核心目标？”如果是，继续；如果否，记录偏离并回到主线。
3. **用一次Focusmate或类似服务**：预约一个25分钟的虚拟同伴工作时段，体验身体在场效应带来的启动助推。
4. **安装RescueTime并查看一次周报**：找出你最大的时间黑洞，并设置一个网站阻断规则。

你的大脑不是缺陷，而是一个高温度、高创意的生成核心。给它套上合适的harness，它就能稳定产出让你骄傲的作品。

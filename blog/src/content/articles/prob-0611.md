# 如果把 ADHD 大脑当成生成核心，Structured 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Structured 对 ADHD 的真正价值。

你正在写一篇报告，突然想起要查一个数据，打开浏览器，然后——半小时后，你在看某电商的推荐页面，完全忘了刚才要查什么。这不是意志力问题，是 ADHD 大脑的典型工作模式：生成能力极强，但工作记忆像一块漏水的桶，边做边忘。

与此同时，LLM 工程师们正为另一个问题头疼：大模型每次对话都从零开始，没有跨会话状态，要靠向量数据库、记忆模块这些外部记忆来“续命”。

这两件事听起来风马牛不相及，但仔细看，它们共享同一个底层结构：**一个高产但缺乏执行调度层的生成核心**。ADHD 大脑和 LLM 在结构上同构（来源：ADHD 大脑与 LLM 的同构）。那么，如果把 ADHD 大脑当成生成核心，Structured——一个强调无状态架构和外部记忆的 AI 工具——能不能充当它的 harness？

## 同构的脊柱：无状态与外部记忆

先看清这个结构。LLM 本身是无状态的：它不记得上一轮对话，每次推理都是独立事件。为了让它能完成连续任务，工程师必须给它套上“脚手架”（agent scaffolding）——包括外部记忆（向量库）、调度器（任务规划）、工具调用等（来源：Agent Scaffolding）。

ADHD 大脑也一样。它的工作记忆容量小，无法同时 hold 住多个目标，执行功能（计划、启动、抑制）不稳定（来源：执行功能障碍）。因此，ADHD 患者也需要外部脚手架来补偿：日课、清单、物理环境改造、他人的提醒。

这两套系统在结构上完全对应：
- **无状态 ↔ 工作记忆缺陷**：LLM 每次对话都“失忆”，ADHD 大脑也常忘记刚才想到的事。
- **外部记忆 ↔ 笔记/工具**：LLM 靠向量库存储上下文，ADHD 靠便签、App、笔记软件存储想法。
- **调度层 ↔ 执行功能**：LLM 需要 agent 框架来规划步骤，ADHD 需要任务分解工具来启动。

这个同构不是空洞的类比。有研究显示，ADHD 个体的超专注工作时长可达 14.5 小时，远超对照组的 8.2 小时（来源：Pace Ventures Research 2026）。这说明产出能力不差，差的是调度。同样，LLM 的生成能力惊人，但让它按顺序完成一个多步骤任务，就需要外部编排。

## 历史人物的 harness 系统

孔子是 ADHD 特质与 harness 系统的经典案例。他精力旺盛，周游列国 14 年坐不住；冲动爱骂人，见南子急得对天发誓；思维跳跃，《论语》全是场景化语录，无系统著作。但他的 harness 系统——“克己复礼”——用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”）。这套系统本质上就是一个外部调度层：用固定的仪式（礼）来约束冲动，用每日回顾来补偿工作记忆的不足。

把孔子的 harness 映射到 LLM 工程：
- **“礼” ↔ 固定的 agent 工作流**：就像 LLM agent 用预定义的 prompt 模板和工具调用序列来保证行为一致。
- **“日三省” ↔ 定时 re-grounding**：LLM 系统定期从外部记忆重新加载上下文，防止 drift。
- **“从心所欲不逾矩” ↔ 70 岁才做到的自动化**：当外部规则内化成习惯，就不再需要 conscious effort——这正是 scaffold 的终极目标。

另一个例子是曹植。他文学超专注，铜雀台作赋一挥而就，但对政治零兴趣，冲动误事。他的 harness 是诗赋创作：把所有政治失意注入诗歌，用艺术来疏导情绪。这对应 LLM 的“特定领域微调”——把生成能力聚焦到一个 channel 上，避免发散。

## Structured 能当那个外部记忆吗？

回到核心问题。Structured 这类工具（如 Goblin Tools、Saner.AI、Lex）在设计上正好满足了 ADHD 大脑的无状态需求：

- **Goblin Tools 的 Magic ToDo**：接受模糊任务（如“清理厨房”），自动分解为小步骤，用户可调节“辣度”控制粒度（来源：Goblin Tools）。这相当于 LLM agent 的 task decomposition——把复杂目标拆成可执行的子任务。
- **Saner.AI**：通过本地记忆存储和快速检索，减少搜索循环和标签切换（来源：Saner.AI）。它扮演了外部记忆的角色，类似 LLM 的向量数据库。
- **Lex**：允许通过单一指令触发多步骤任务序列（来源：Lex）。这直接对应 agent scaffolding 中的“规划机制”——将复杂目标分解为步骤并顺序执行。

这些工具的共同点是：**它们不试图修复 ADHD 大脑，而是接受它的无状态性，提供外部脚手架**。就像 LLM 工程师不会试图让模型“记住”对话，而是把记忆交给向量库。

## 脚手架 vs 拐杖：边界在哪里？

然而，同构类比也有危险。如果过度依赖外部工具，ADHD 患者的内在执行功能可能进一步萎缩（来源：拐杖与脚手架）。同样，LLM 系统如果完全依赖外部记忆，当记忆模块失效时，系统会彻底瘫痪。

**我的观点是：脚手架和拐杖的边界在于是否保留了对核心能力的训练。** 好的 scaffold 应该像健身房的辅助器械——帮你完成动作，但不替代肌肉发力。例如，使用 Goblin Tools 分解任务时，可以逐步减少分解粒度，让自己尝试合并步骤。使用 Saner.AI 做笔记时，可以定期回顾并尝试自己回忆。

另外，现有工具的证据基础薄弱。多数缺乏严格临床验证，效果多基于用户报告（来源：矛盾与存疑）。同构性说法本身也缺乏实证基础，目前仅是一种理论类比（来源：矛盾与存疑）。所以，别把任何工具当成灵丹妙药。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个你拖延很久的任务**（比如“整理桌面”），设置辣度 3，然后按步骤执行。
2. **在 Saner.AI 中创建一个“外部记忆”笔记本**，每天记录 3 件你容易忘记的事（如吃药、回邮件），并用其搜索功能快速找回。
3. **尝试 Lex 的单一指令模式**：写一个指令如“准备明天的会议：列出议程、收集上周数据、发送邀请”，看它能否一次性完成。
4. **每周做一次“无工具日”**：选一天，不用任何 AI 工具，只用纸笔和闹钟，观察自己的执行功能表现——这能帮你判断工具是脚手架还是拐杖。

ADHD 大脑不是 bug，是 feature——前提是你给它配上正确的 harness。而 Structured 工具，正是那个正在进化的外部调度层。

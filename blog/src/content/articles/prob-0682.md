# Reclaim.ai 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Reclaim.ai：它到底补上了哪一层执行功能？

## 问题：为什么 ADHD 用户和 LLM 工程师都在同一条船上？

如果你是一个 ADHD 患者，你一定熟悉这样的场景：脑子里有无数个待办事项在打转，但就是没办法把“整理报告”拆成“打开电脑→新建文档→写第一段”这样的具体步骤；你明明知道 deadline 就在明天，却仍然在刷手机，直到最后一刻才被肾上腺素驱动着冲过终点线。

如果你是一个在做 Agentic Harness 工程的开发者，你一定也遇到过类似的困境：大语言模型（LLM）能生成优美的代码、流畅的文案，但一旦让它自己规划并执行一个多步骤任务——比如“预订机票并发送确认邮件”——它就会开始幻觉，忘记中间状态，或者卡在某个子任务上出不来。

这两个场景，本质上是同一个问题的两面。ADHD 大脑与 LLM 在结构上同构：两者都是强大的生成核心，但缺乏可靠的执行调度层（来源：ADHD 的 AI 工具全景）。ADHD 患者有智力、创造力，但执行功能（计划、组织、工作记忆）不稳定；LLM 有语言生成能力，但缺乏内置的编排层来管理步骤、追踪状态。

于是，两边都想到了同一个解法：把执行调度外包给外部工具。对 LLM 来说，那是 function calling——让模型通过 API 调用计算器、日历、数据库等精确工具来完成它不擅长的计算与状态维护。对 ADHD 来说，那是 Reclaim.ai、Goblin Tools 这类 AI 生产力工具——它们自动分解任务、安排日程、提醒执行，补上用户缺失的调度层。

## 核心论点：Reclaim.ai 是 ADHD 的 function calling，但有人用错了

Reclaim.ai 是一款 AI 驱动的日程调度工具，它能自动分析用户的日历、优先级和工作习惯，为任务分配时间块，并在冲突时动态调整。它的核心价值在于：**把“决定什么时候做什么”这个认知负荷极高的调度任务，从用户手里拿走**。这恰恰对应 LLM 的 function calling：把“计算 25×37”或“查询当前天气”这类精确操作外包给外部工具，减轻模型自身的认知负荷，减少幻觉（来源：认知负荷）。

但问题在于：function calling 的边界是清晰的——LLM 只负责决定“调用哪个工具”，而不负责“工具如何执行结果”。同样，Reclaim.ai 的调度应该是“建议时间块”，而不是“替用户决定做什么”。如果 ADHD 用户完全依赖 Reclaim.ai 的自动安排，不再主动思考任务的优先级和自身状态，那么工具就从脚手架——一种可逐步撤除的临时支撑——退化为拐杖，阻碍内在执行功能的发展（来源：矛盾与存疑）。

## 证据：ADHD 侧与 LLM 侧的真实对应

ADHD 侧的证据来自多个工具的设计逻辑。Goblin Tools 的 Magic ToDo 功能将“整理房间”自动分解为“捡起地板上的衣服”“擦桌子”等步骤，降低启动门槛（来源：Goblin Tools）。Saner.AI 通过本地记忆存储减少搜索循环，直接补偿工作记忆缺陷（来源：Saner.AI）。Lex 允许用户通过单一指令触发复杂多步骤任务，减少决策负担（来源：Lex）。这些工具的共同点，就是**外化执行调度**——把“怎么开始”“下一步做什么”这些认知负荷从用户脑中移除。

LLM 侧的证据来自 AI 工程实践。在 Agentic Harness 中，function calling 被设计为让 LLM 只负责“意图识别”，而将精确计算（如算术、日期运算）交给专门的工具。这种分离显著减少了 LLM 的认知负荷，降低了幻觉（来源：认知负荷，其中提到“AI 系统内部也采用任务分解技术来降低自身的认知负荷”）。

更关键的是，两边都面临同样的悖论：**如果外部调度层接管了所有决策，核心生成能力反而会萎缩**。ADHD 用户长期依赖自动调度，可能丧失主动规划的能力；LLM 如果所有步骤都用 function calling 替代，模型本身的推理能力也会退化。

## 争议与局限：脚手架 vs 拐杖的边界在哪里？

当前文献中存在一个核心矛盾：一方面，AI 工具被宣传为能有效补偿执行功能缺陷（来源：ADHD 的 AI 工具全景）；另一方面，缺乏长期随机对照试验证明其效果，且多数工具设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。Reclaim.ai 的自动调度尤其容易导致过度依赖——用户不再需要思考“我现在该做什么”，因为工具已经安排好了。但 ADHD 的异质性意味着，对某些用户来说，完全自动化可能是救命稻草，而对另一些人，它可能阻碍内在能力的发展。

另一个局限是：Reclaim.ai 的调度逻辑基于历史数据和预设优先级，但它无法感知用户当下的**身体在场效应**——比如用户是否疲惫、是否处于超聚焦状态（来源：超聚焦）。超聚焦时，强行切换任务会导致“超聚焦崩溃”，反而降低效率（来源：超聚焦）。因此，工具需要允许用户临时覆盖调度，保留人的判断权。

## 行动指南：今天就能试的 3 件事

1. **区分“调度”与“执行”**：使用 Reclaim.ai 时，只让它安排时间块，但自己决定每个时间块里具体做什么。例如，让工具在明天 10:00-11:00 安排“写报告”，但到了时间，你仍然需要主动决定是写报告还是先处理紧急邮件。
2. **设置“人工覆盖”规则**：在 Reclaim.ai 中开启“允许手动调整”功能，并给自己一个规则：如果当前处于超聚焦状态，允许跳过或推迟自动安排的会议。
3. **每周一次“无工具日”**：每周选一天，关闭所有 AI 调度工具，手动用纸笔规划任务。这有助于训练内在执行功能，防止工具成为拐杖。

## 结论

Reclaim.ai 之于 ADHD，就像 function calling 之于 LLM——两者都是为生成核心补上执行调度层。但工具的价值不在于替代人的判断，而在于释放认知资源让人做更重要的决策。真正的脚手架，是那种你最终可以不需要的支撑。

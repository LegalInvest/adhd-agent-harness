# Reclaim.ai 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Reclaim.ai：它到底补上了哪一层执行功能？

## 问题：为什么 Reclaim.ai 对很多人没用？

你兴冲冲装好 Reclaim.ai，授权日历、同步待办、设好“保护时间”，期待它像魔法一样把混乱的日程理清。结果呢？它要么把会议塞进你本来要深度工作的时段，要么在你需要休息时弹出“最佳健身时间”通知。你开始怀疑：这工具是不是根本不懂 ADHD？

另一边，工程师们正为 LLM 的 function calling 头疼。你精心设计了工具 schema，模型却总在简单计算时调用外部 API，或者在不需要工具时胡乱调用。你困惑：function calling 不是让模型更精确吗？怎么反而更笨了？

两个场景看似无关，实则共享同一个核心问题：**工具被当作拐杖，而不是脚手架。** 而 Reclaim.ai 之于 ADHD，就像 function calling 之于 LLM——两者都是为“高产但缺执行调度层的生成核心”套上外部执行层，但多数人用错了方向。

## 同构：ADHD 大脑与 LLM 是同一类生成核心

ADHD 大脑与 LLM 在结构上惊人地同构。ADHD 者常拥有强大的发散思维、创意和超聚焦能力，但执行功能（工作记忆、任务启动、时间感知）存在缺陷（来源：ADHD 的 AI 工具全景）。LLM 同样能生成高质量文本、解决复杂推理，但缺乏稳定的上下文控制，容易上下文膨胀、遗忘指令或“冷启动”困难（来源：ADHD 的 AI 工具全景）。

这种同构意味着，两者的解法也应同构。ADHD 需要外部执行功能层来补偿内在调度缺陷：分解任务、管理上下文、提供实时反馈（来源：ADHD 的 AI 工具全景）。LLM 需要 function calling 将精确计算外包给专用工具（如计算器、数据库），从而弥补自身的不精确性。Reclaim.ai 正是为 ADHD 大脑提供的外部调度层——它应该像 function calling 那样，**在需要时精确介入，而不是接管一切**。

## 错用：把脚手架当拐杖

Reclaim.ai 的常见错用是“全自动调度”。用户打开所有功能，让 AI 决定何时工作、何时休息、何时开会。结果呢？AI 无法理解 ADHD 者的情绪波动、超聚焦状态和突发灵感。当你在超聚焦时被会议打断，或在你情绪低落时强行安排社交，这种“智能”就变成了折磨。

同样，function calling 的错用是“过度外包”。开发者让 LLM 在简单加法时也调用计算器，或让模型在不需要外部信息时频繁查询数据库。这不仅增加了延迟，还削弱了模型本身的推理能力。正如 ADHD 者过度依赖 AI 工具可能削弱内在时间感知能力（来源：矛盾与存疑），LLM 过度依赖外部工具也会削弱其内部推理能力。

## 正用：脚手架 vs 拐杖的边界

脚手架与拐杖的边界在于：**工具是否促进能力发展，还是替代能力**（来源：ADHD 的 AI 工具全景）。对于 ADHD，Reclaim.ai 应作为“启动辅助”而非“全程管家”。例如，利用其“任务分解”功能（类似 Goblin Tools 的 Magic ToDo）将大任务拆为小步骤（来源：Goblin Tools），但执行时仍由自己控制节奏。对于 LLM，function calling 应作为“精确计算外包”，但模型仍需保留核心推理能力。

具体来说，Reclaim.ai 的正用是：
- **只调度重复性事务**（如每周会议、日常提醒），保留创造性工作的自主权。
- **利用“保护时间”功能**，但手动决定何时进入超聚焦，而非让 AI 猜测。
- **结合其他工具**：用 Saner.AI 增强知识回忆（来源：Saner.AI），用 Lex 的单一指令触发多步骤任务（来源：Lex），形成互补的脚手架体系。

对于 function calling，正用是：
- **仅在需要精确计算时调用**（如数学、数据库查询），避免在模糊任务（如写作）中使用。
- **设计轻量级 schema**，让模型保留大部分推理，只外包最机械的部分。

## 争议与局限

必须承认，上述框架的证据基础仍薄弱。现有研究主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑）。例如，Reclaim.ai 对 ADHD 的效果尚无独立研究，Goblin Tools 的证据也以定性为主（来源：Goblin Tools）。此外，个体差异巨大：有的 ADHD 者需要更多自动化，有的则需要更少。工具设计者声称是“脚手架”，但实际使用中可能沦为“拐杖”（来源：矛盾与存疑）。因此，本文观点更多是启发式框架，而非确凿结论。

## 今天就能试的行动

1. **关闭 Reclaim.ai 的自动调度**，只保留“保护时间”和“任务分解”功能，手动安排创造性工作。
2. **用 Goblin Tools 分解一个你拖延已久的任务**（比如“整理房间”），体验小步骤带来的启动感（来源：Goblin Tools）。
3. **尝试 Lex 的单一指令**，将“写周报”拆为“打开文档→列出本周完成→写总结”的序列，减少决策疲劳（来源：Lex）。
4. **记录一周内 Reclaim.ai 的调度与你的实际状态对比**，找出哪些干预是“脚手架”，哪些是“拐杖”。

工具本无对错，用对才是关键。Reclaim.ai 和 function calling 都是强大的外部执行层，但只有当你意识到它们只是脚手架，而非替代你思考的拐杖时，它们才能真正释放你的生产力。

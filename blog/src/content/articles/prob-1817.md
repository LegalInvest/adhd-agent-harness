# ADHD 创业者视角：为什么「治好 ADHD 的注意力容易被环境带偏」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 受上下文窗口内容左右，需要上下文工程——同一套 harness 思路，两边都成立。

## 问题：为什么我的大脑和我的AI都跑飞了？

如果你是个ADHD创业者，你一定熟悉这种场景：灵感像爆米花一样炸开，但下一秒就被窗外的鸟叫、手机通知或脑子里突然冒出的新点子带偏。你坐在电脑前，明明想写方案，却发现自己刷了半小时短视频。

如果你是个LLM工程师，你也熟悉这种场景：你精心设计的agent在简单任务上表现惊艳，但一旦上下文窗口塞进太多历史对话，它就开始胡言乱语——忘记指令、捏造事实、输出逻辑断裂的回复。

这两个问题看似毫无关联，但如果你把ADHD大脑和LLM放在一起看，会发现它们共享同一个核心缺陷：**都是高产但缺乏可靠执行调度层的生成核心**（来源：ADHD 大脑与 LLM 的同构）。

## 同构：生成核心与调度层的断裂

ADHD大脑的典型特征是“强记忆、弱控制”——实证研究表明，ADHD个体的工作记忆容量甚至可能超过常人，但认知灵活性与注意控制存在核心缺陷，无法灵活切换任务集、无法抑制自动化反应（来源：ADHD 大脑与 LLM 的同构）。换句话说，ADHD大脑是一个强大的创意生成器，但它的“调度系统”经常宕机：想法源源不断，却无法决定下一步该执行哪个。

LLM同样如此。一个未经harnass的LLM，面对长上下文时，注意力机制会平等地“注意”所有内容，导致关键指令被稀释。研究发现，LLM的“注意力资源的弥散分配”正是其工作记忆容量受限的计算本质（来源：ADHD 大脑与 LLM 的同构）。它生成能力强，但缺乏内置的执行控制——你给它的提示里哪怕多一句无关的话，它都可能跑偏。

两者的共同解是：**上下文工程**。对ADHD大脑来说，上下文工程意味着主动设计“当下注意什么”的环境——减少决策、保留上下文、让下一步行动显而易见（来源：ADHD 大脑与 LLM 的同构）。对LLM/agent来说，上下文工程意味着构建“agent harness”——包裹LLM的软件基础设施，处理模型本身之外的一切，包括提示结构、记忆管理、任务调度（来源：ADHD 大脑与 LLM 的同构）。

## 证据：两边都成立的harness方案

### ADHD侧：外部执行功能层

ADHD个体需要外部工具来补偿执行功能缺陷。例如：
- **Focusmate**：基于“身体在场效应”的实时同伴问责。用户预约时段后，平台通过算法匹配一位伙伴，通过视频会议共同工作。它提供外部结构和社会压力，弥补内在多巴胺调节不足，从而提升任务启动和维持能力（来源：Focusmate）。多项用户研究显示其能显著提升任务启动和维持，但缺乏随机对照试验（来源：AI 与 ADHD 的专注力）。
- **RescueTime**：自动化的时间追踪工具，利用AI识别用户在不同应用和网站上的时间花费，减轻工作记忆负担，帮助用户识别注意力漂移模式（来源：RescueTime）。它被推荐用于提升ADHD群体的生产力，但证据主要来自用户报告。
- **Brain.fm**：利用AI生成音乐，通过神经锁相技术影响大脑信息处理方式，帮助用户进入专注状态。但其针对ADHD的效果缺乏独立临床研究支持（来源：Brain.fm）。

这些工具的共同点是：它们都不是“修复”大脑，而是给大脑套上一个外部调度层——就像给LLM搭脚手架。

### LLM/agent侧：agent harness

在构建AI agent时，工程师需要“agent harness”——即包裹LLM的软件基础设施，处理模型本身之外的一切（来源：ADHD 大脑与 LLM 的同构）。典型的架构采用“复合AI系统”，通过每个工作流绑定不同LLM来分配任务，并依赖模型连接器等组件（来源：ADHD 大脑与 LLM 的同构）。

这本质上就是给LLM做上下文工程：结构化提示、外部记忆（向量数据库）、注意力锚点（如关键指令固定）。和ADHD工具一样，它不改变LLM本身，而是改变LLM运行的环境。

## 核心观点：脚手架，不是拐杖

我的判断是：**AI工具作为外部执行功能层，其有效性取决于它是否精准对应ADHD的特定执行功能短板，且必须避免形成永久依赖**（来源：AI 与 ADHD 的专注力）。

同样，LLM的agent harness也必须精准：过度结构化的提示会扼杀LLM的生成能力，过于自由的上下文则导致跑飞。关键在于找到“脚手架”和“拐杖”的边界（来源：AI 与 ADHD 的专注力）。

但这里存在争议。多个页面指出，AI工具的有效性证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑）。过度依赖AI工具可能削弱内在时间感知能力（来源：矛盾与存疑）。个体差异也导致效果不一（来源：矛盾与存疑）。Brain.fm的ADHD效果甚至缺乏独立临床研究（来源：矛盾与存疑）。

因此，我们的同构视角虽然有力，但只是一个隐喻，而非严格等价。LLM的“注意力机制”与人类注意力存在本质差异（来源：AI 与 ADHD 的专注力）。我们需谨慎类比。

## 今天就能试的行动

1. **为你的ADHD大脑做上下文工程**：每天开始工作前，用5分钟写下“当前唯一任务”，并关闭所有无关浏览器标签和通知。这相当于给大脑一个精简的上下文窗口。
2. **尝试一次Focusmate**：预约一个25分钟的虚拟身体在场时段，观察它是否帮你启动任务。注意：如果社交焦虑严重，可先尝试录音自我问责。
3. **给你的LLM agent加一个记忆层**：如果你在构建agent，尝试用向量数据库存储关键上下文，而不是把所有历史塞进提示窗口。这相当于给LLM一个外部工作记忆。
4. **用RescueTime追踪一周**：不评判，只观察。看看哪些环境因素（特定应用、时间段）最容易导致注意力漂移，然后主动移除它们。

## 局限与诚实

本文的同构论证基于概念类比，而非严格的神经科学或计算机科学等价。ADHD大脑的注意力机制涉及多巴胺调节、默认模式网络等复杂因素，与LLM的Transformer注意力机制完全不同。此外，多数工具的证据强度较弱，用户应保持批判性期待。

但即便如此，这个视角仍然有价值：它让我们看到，无论是管理自己的大脑还是管理AI，核心工程问题是一样的——**如何让一个强大的生成核心，在混乱的环境中稳定输出**。答案也一样：**上下文工程**。

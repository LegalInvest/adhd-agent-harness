# 如果把 ADHD 大脑当成生成核心，RescueTime 能不能当它的外部执行功能层？

> 用 agent 工程的思路重新理解 RescueTime 对 ADHD 的真正价值。

## 问题：为什么 RescueTime 对 ADHD 常常无效？

许多 ADHD 用户下载 RescueTime 后，最初几天兴奋地查看数据，然后很快遗忘。它记录了你花了多少时间在社交媒体上，但并没有帮你“启动”那个你拖延了一周的报告。问题出在哪里？因为 RescueTime 被当作一个“分析工具”来用，而 ADHD 大脑最不缺的就是分析——缺的是执行。

## 同构：ADHD 大脑与 LLM 都是“生成核心”，缺的是外部调度层

ADHD 大脑与 LLM 在结构上高度同构：两者都是强大的生成核心，但缺乏可靠的内置执行调度层。ADHD 的执行功能（计划、组织、启动）先天偏弱（来源：ADHD × AI 的科学与研究前沿）；LLM 同样无状态、缺乏调度层，需要外部脚手架（如 ReAct、Chain-of-Thought）来完成任务（来源：拐杖与脚手架）。这个脚手架就是“外部执行功能层”——在 LLM 侧是 agent 的编排、记忆、工具层，在 ADHD 侧是时间管理、任务分解、外部记忆系统。

RescueTime 的潜力在于：它不只记录时间，还能通过“定时提醒”“目标限制”等机制主动干预——这正是外部调度层的功能。但多数用户只用了它的“记录”部分，忽略了“干预”部分。如果把 RescueTime 当作一个 agent 的编排层来用——设定规则（如“每天写代码至少 2 小时”），让它在你偏离时报警、在你专注时奖励——它就能从“记录仪”变成“调度器”。

## 证据：人物案例与工程同构

曾国藩是典型的注意力不集中型 ADHD：日记里天天骂自己“无恒”“浮躁”，读书慢，“他人目下二三行，余或疾读不能终一行”（来源：人物案例）。他的解决方案是“日课十二条”：黎明即起、读书不二、谨言、写日记反省等。这套系统本质上就是外部执行功能层——每天固定时间做固定事，用外部规则补偿内部冲动。对应到 LLM/Agent 工程，这就是“定时 re-grounding”机制：每过一段时间强制重置上下文，防止漂移。

张瑞敏的“日清日结”工作法也是同样逻辑：每天的工作每天完成，每天反省每天进步（来源：人物案例）。在 Agent 工程中，这对应“每日总结+计划”的循环，类似 agent 的“反思”步骤。RescueTime 的“每日报告”和“目标进度”正是这种循环的数字化版本——但需要主动设置目标，否则只是数据堆积。

## RescueTime 如何真正成为外部执行功能层？

要让 RescueTime 生效，必须把它从“被动记录”切换到“主动干预”模式：
1. **设置“目标”而非“标签”**：比如“每天在 IDE 上至少 3 小时”，而不是“记录编程时间”。目标触发 RescueTime 的提醒功能。
2. **利用“定时提醒”作为启动信号**：ADHD 的任务启动困难可通过外部信号克服。RescueTime 的“休息提醒”可以改为“开始工作提醒”。
3. **结合任务分解工具**：RescueTime 本身不做任务分解，但可以配合 Goblin Tools 的 Magic ToDo 功能（来源：Goblin Tools）——先用 Goblin 把“写报告”分解为 10 个小步骤，再用 RescueTime 追踪每个步骤的时间。

## 争议与局限

同构框架虽然有力，但存在争议（来源：矛盾与存疑）：
- **过度依赖风险**：长期使用 RescueTime 这样的外部调度器，可能让 ADHD 的执行功能进一步退化。曾国藩的“日课”是自我训练，而非外部工具替代——他最终内化了规则。而 RescueTime 是外部工具，容易变成“拐杖”而非“脚手架”。
- **个体差异**：并非所有 ADHD 亚型都适用同构框架。冲动型 ADHD 可能更需要即时反馈，而非定时提醒。
- **证据不足**：RescueTime 在 ADHD 人群中的独立临床验证缺乏（来源：矛盾与存疑）。其效果更多来自用户主观报告。

## 今天就能试的行动

1. **设置一个“不可协商”的目标**：在 RescueTime 中设置每天至少 1 小时在最重要任务上，并开启“当偏离时弹窗提醒”。
2. **用 Goblin Tools 分解一个拖延任务**：把“清理邮箱”分解成 5 个子任务，然后每完成一个在 RescueTime 中标记一次。
3. **执行“曾国藩式日课”**：每天固定时间（如上午 9 点）用 RescueTime 回顾前 24 小时数据，写一条反省日志（类似曾国藩的日记）。
4. **关闭 RescueTime 的“分析”页面**：只保留“警报”和“目标”功能，减少信息过载。

RescueTime 不是万能药，但当你把它当作 agent 的编排层来设计——设定规则、触发干预、反馈循环——它就能成为 ADHD 大脑真正需要的“外部执行功能层”。关键是：不要分析，要调度。

# 如果把 ADHD 大脑当成生成核心，ChatGPT 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 ChatGPT 对 ADHD 的真正价值。

## 问题：为什么你一边觉得 ChatGPT 很强大，一边又用它记不住事？

如果你是 ADHD 患者，你可能有过这样的体验：打开 ChatGPT，想让它帮你梳理一个复杂项目，但聊了十几分钟后，它忘了你一开始说的关键背景，你又得重新输入。或者，你让它帮你分解任务，它给出了完美的步骤，但你转头就忘了第二步是什么。

如果你是 Agent 工程师，你可能更熟悉这种痛苦：你精心设计了一个 LLM agent，给它系统提示、工具、记忆库，但它在执行长任务时开始“忘记系统提示”（来源：Building an AI Coding Agent for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned），行为逐渐漂移，直到你需要手动重锚定。

这两个场景共享同一个底层问题：**生成核心（ADHD 大脑或 LLM）本身是无状态的，它天生缺乏可靠的执行调度层**。ADHD 的工作记忆容量小、边做边忘，LLM 则没有跨会话状态，每次对话都是全新的开始。

## 同构：ADHD 大脑与 LLM 是一类“高产但缺调度”的生成核心

ADHD 大脑的生成核心（发散思维、创意、多任务并行）产出极高，但执行功能（任务启动、持续专注、抑制冲动）薄弱（来源：ADHD 的 AI 工具全景）。LLM 同样：它的生成能力惊人，但直接输出可能偏离目标、陷入循环或产生危险行为（来源：人在回路与身体在场）。

两者的解法在结构上同构：**给生成核心套一个 harness**。对 LLM 来说，这个 harness 是 agent 脚手架——包括 token 预算、工具调用上限、人工检查点、外部记忆（向量库）。对 ADHD 来说，这个 harness 是外部工具——包括任务分解、提醒、持久记忆、身体搭档。

## 证据：ChatGPT 作为无状态核心，需要哪些外部记忆与脚手架？

### 1. 任务分解与启动：Goblin Tools 与 Lex

ADHD 患者常因任务过大而无法启动。Goblin Tools 的 Magic ToDo 功能自动将“整理房间”分解为“捡起地板上的衣服”“擦桌子”等小步骤（来源：Goblin Tools）。这降低了启动门槛，因为小步骤提供即时反馈，绕过执行功能屏障。类似地，Lex 允许通过单一指令触发多步骤任务序列（来源：Lex），减少决策负担。

在 LLM 侧，这相当于将复杂目标分解为可执行的子任务序列。Agent harness 通过形式化规划与护栏，使代理输出真正有用且正确（来源：人在回路与身体在场）。

### 2. 工作记忆与上下文保持：Saner.AI 与 Mem

ADHD 患者频繁陷入搜索循环——找不到上次记下的信息，然后在多个标签页间切换。Saner.AI 通过本地记忆和知识回忆功能，让用户快速找回信息，减少认知负荷（来源：Saner.AI）。Mem 提供持久记忆，减少搜索循环（来源：ADHD 的 AI 工具全景）。

在 LLM 侧，这对应外部向量库或记忆层。LLM 本身无跨会话状态，必须靠外部记忆来“续命”。如果没有记忆，agent 会在长程任务中丢失上下文。

### 3. 重锚定与目标漂移：提醒与人工检查点

ADHD 大脑在执行任务时极易发生目标漂移——被无关刺激捕获，或过度聚焦细节。外部工具通过提醒和通知提供重锚定（来源：重锚定与目标漂移）。

在 LLM 侧，agent 同样面临目标漂移：模型在交互轮次后期忘记系统提示。Harness 通过事件驱动的系统提醒来对抗这种“指令消退”（来源：重锚定与目标漂移）。

### 4. 人在回路与身体在场：Focusmate 与人工监督

身体在场效应（body doubling）是 ADHD 群体最有效的策略之一——有另一个人在场时，执行功能障碍被绕过（来源：人在回路与身体在场）。Focusmate 通过 AI 匹配虚拟身体搭档，提供外部问责。

在 LLM 侧，这对应 human-in-the-loop 监督：设置护栏、加入人工检查点（暂停并询问后再执行）（来源：人在回路与身体在场）。

## 核心判断：ChatGPT 是生成核心，不是调度层——你需要自己搭 harness

我的核心观点是：**ChatGPT 对 ADHD 的真正价值，不是替代你的执行功能，而是作为生成核心与你互补。但你必须主动设计 harness，否则它只是另一个让你分心的工具。**

这意味着：不要只把 ChatGPT 当聊天机器人。把它当无状态生成引擎，然后给它配外部记忆（如用 Mem 记录对话要点）、任务分解工具（如 Goblin Tools）、以及重锚定提醒（如日历通知）。

## 诚实指出局限：脚手架 vs 拐杖的边界

但这里存在争议。过度依赖 AI 工具可能阻碍内在执行功能发展（来源：矛盾与存疑）。多数工具如 Goblin Tools、Saner.AI 设计为长期使用，未提及撤除机制。这引发了一个问题：**我们是在搭脚手架，还是在给自己造拐杖？**

脚手架应设计为可逐步撤除的临时支撑，但当前工具很少考虑这一点。此外，ADHD 异质性高，单一工具难以覆盖所有亚型（来源：矛盾与存疑）。ChatGPT 的个性化能力尚不成熟，它可能无法准确识别你的意图（来源：Lex 的矛盾）。

另一个局限是证据强度。现有证据多来自用户评价和概念对齐，缺乏随机对照试验（来源：ADHD 的 AI 工具全景）。同构命题本身也缺乏大规模实证（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 ChatGPT + 外部笔记**：每次与 ChatGPT 对话后，用 Mem 或 Notion 记录关键输出。下次开始时先粘贴上次摘要，保持上下文。
2. **用 Goblin Tools 分解一个让你焦虑的任务**：输入“整理房间”或“写报告”，看它分解成的小步骤。挑第一步做。
3. **设置重锚定提醒**：用手机日历或 ChatGPT 的定时功能，每 25 分钟提醒你“当前目标是什么？”。
4. **尝试虚拟身体搭档**：用 Focusmate 或找朋友一起工作，利用外部问责启动任务。

## 结语

ADHD 大脑与 LLM 是同构的：两者都是强大的生成核心，但都需要 harness 才能可靠输出。ChatGPT 可以成为这个 harness 的一部分，但前提是你理解它的无状态本质，并主动搭建外部记忆与调度层。这不是魔法，是工程。

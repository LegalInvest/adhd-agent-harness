# 给 ADHD 加一道「核对清单/验证步骤」，和给 agent 加 self-critique 循环，是同一种工程吗？

> ADHD 侧：冲动下结论、想当然；LLM/harness 侧：LLM 自信地编造（幻觉），靠验证与自我批判兜底。用双域证据作答。

## 问题：为什么你越想“想清楚”，越容易掉进坑里？

如果你是 ADHD 人群，你一定熟悉这个场景：看到一个问题，大脑立刻给出一个“显然”的答案，你毫不犹豫地说出口，然后发现——错了。或者你刚写完一段文字，自我感觉良好，但回头一看，逻辑漏洞百出。如果你是做 LLM Agent 工程的工程师，你也会熟悉另一个场景：让模型写一段代码，它自信地编造了一个不存在的 API，跑起来才报错；你给它一个任务，它信誓旦旦地开始执行，结果偏离了目标十万八千里。

这两个场景的共同点是什么？都是一个“高产但不可靠”的生成器在缺乏验证的情况下直接输出。ADHD 大脑的冲动下结论，与 LLM 的幻觉（自信地编造），是同一种底层问题的不同表现。

## 同构：生成核心与调度层的分离

核心论点在于：**ADHD 大脑与 LLM 在结构上同构——两者都是强大的生成核心，但缺乏可靠的内置执行调度层。**（来源：ADHD × AI 的科学与研究前沿）ADHD 大脑的前额叶执行功能网络激活不足，导致任务启动困难、工作记忆不稳定、容易冲动；LLM 则是无状态的，缺乏跨会话的调度能力，容易在长上下文中迷失或产生幻觉。两者都需要外部脚手架来补偿。

这种同构不是空洞的类比。神经科学证据显示，ADHD 患者在执行功能任务时前额叶激活不足（来源：同上）；AI 工程中，为 LLM 设计的脚手架（如 ReAct、Chain-of-Thought）已被证明能有效提升任务完成率（来源：同上）。两边都在做同一件事：**给生成核心加一道验证环节。**

## 案例：毛泽东的“批评与自我批评”作为 harness

毛泽东是典型的 ADHD 特质者：从小是问题儿童，爱读闲书，思维跳跃，讲话充满场景化比喻，一生都在调查研究，决策冲动敢赌（四渡赤水、抗美援朝都是险棋）。但他的成就——缔造新中国——恰恰来自于他为自己和组织设计的 harness 系统。

他的专属 harness 包括：**批评与自我批评**（让别人提自己的意见，相当于外部验证层）、**调查研究**（“没有调查就没有发言权”，相当于强制 grounding）、**民主集中制**（用集体决策制衡自己的冲动，相当于 human-in-the-loop 监督）。这些机制的结构，与 LLM Agent 的 harness 惊人地一致：Agent harness 通过设置护栏（token 预算、工具调用上限）、加入人工检查点（暂停并询问后再执行）来防止模型跑偏（来源：Building an AI Agent Harness from Scratch）。毛泽东的“民主生活会”就是一个人工检查点，而“调查研究”则是强制模型先检索证据再输出。

## 工具：核对清单与 self-critique 循环的具体实现

在 ADHD 侧，核对清单的工具已经出现。例如 Goblin Tools 的 Magic ToDo 功能，能将模糊任务（如“清理厨房”）自动分解为具体、可执行的子任务，用户可调节“辣度”控制粒度（来源：Goblin Tools）。这相当于给 ADHD 大脑加了一道“先分解、再执行”的验证步骤。Saner.AI 则通过增强知识回忆，减少搜索循环和标签切换，帮助用户快速找回信息（来源：Saner.AI）。Motion 的 AI 自动排程消除了“下一步该做什么”的决策负担，动态调整日程（来源：Motion）。这些工具都在做同一件事：**用外部系统补偿内部执行功能的缺失。**

在 LLM/Agent 侧，self-critique 循环是标准做法。例如，让模型先输出一个答案，然后让它自己检查并修正；或者使用 ReAct 模式，让模型在思考-行动-观察的循环中不断验证。Harness 通过集成 MCP 工具、遥测、人在回路治理，确保模型不会无限循环或产生危险行为（来源：Building an AI Agent Harness from Scratch）。这与 ADHD 的核对清单在结构上完全对应：都是“先生成、后验证”的两阶段过程。

## 核心判断：脚手架，不是拐杖

我的判断是：**给 ADHD 加核对清单和给 agent 加 self-critique 循环，不仅是同一种工程，而且是必须的工程。** 但这里有一个关键边界：脚手架 vs 拐杖。脚手架是临时辅助，最终目的是让使用者学会自己搭建；拐杖则是永久依赖，可能导致功能退化。ADHD 侧的风险在于：长期使用 AI 工具替代执行功能，可能导致原有能力进一步萎缩（来源：矛盾与存疑）。LLM 侧的风险在于：过度依赖 self-critique 循环，可能让模型失去探索能力，或者让人类工程师放弃对模型输出的基本判断。

因此，工程的关键不是“加多少验证”，而是“验证的粒度与退出机制”。好的脚手架应该像毛泽东的“调查研究”——它不替代你的思考，而是强制你接地气；像杨振宁的“每天固定时间看书想问题”——它不替代你的直觉，而是给你一个稳定的调度框架。杨振宁的物理直觉极强，但他的 harness 是“每天固定时间看书想问题，不做没用的事”（来源：人物案例）。这种日课相当于 LLM 的定时 re-grounding，用外部节奏对抗内在的混乱。

## 诚实面对局限

必须指出，这种同构理论目前仍缺乏实证基础，更多是一种理论类比（来源：矛盾与存疑）。并非所有 ADHD 亚型都适用，部分患者可能以注意力分散为主，而非执行功能缺陷。工具方面，Motion 等应用缺乏独立临床验证（来源：矛盾与存疑），而 Goblin Tools 的证据主要来自用户反馈。此外，传统提醒系统有一个关键缺陷：设置提醒本身就是一个执行功能任务，而 ADHD 个体恰恰在这方面薄弱（来源：重锚定与目标漂移）。这提示我们，未来的工具需要更“无感”的验证层。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个让你拖延的任务**：输入“清理厨房”或“写报告”，看看 AI 如何拆解，然后只做第一步。
2. **给自己设一个“自我批评”检查点**：每次下结论前，强制自己问“我有什么证据？有没有其他可能性？”这相当于 LLM 的 self-critique。
3. **尝试 Motion 或 Saner.AI 的自动排程**：让 AI 帮你决定“下一步做什么”，减少决策负担。
4. **建立日课**：像杨振宁一样，每天固定时间做一件事，用外部节奏对抗内在混乱。

给 ADHD 加核对清单，给 agent 加 self-critique 循环——这不是两种工程，而是同一个问题的两面。承认自己需要脚手架，不是软弱，而是智慧。

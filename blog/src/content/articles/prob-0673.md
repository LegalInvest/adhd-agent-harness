# Todoist 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Todoist：用久了会让 ADHD 更强还是更依赖？

你是否有过这样的体验：打开 Todoist，看着密密麻麻的任务列表，却不知道从何下手；或者刚把任务添加进去，转头就忘了自己写过什么？对于 ADHD 大脑来说，工作记忆容量小、边做边忘是日常。Todoist 这类待办清单工具，本应是救星，却常常沦为另一个被遗忘的角落。

这背后的问题，与 LLM 工程师们正在面对的困境惊人地同构：LLM 没有跨会话状态，每次对话都像失忆症患者，必须靠外部记忆（向量库、上下文窗口）来“续命”。ADHD 大脑也是一个高产生成核心——创意源源不断，但缺乏可靠的执行调度层。两者的解法本质相同：搭建一个外部脚手架（harness），而不是单纯依赖工具本身。

## 同构：ADHD 大脑与 LLM 的共享困境

ADHD 的“工作记忆容量小”与 LLM 的“无状态”是同一类问题：两者都无法可靠地在内部维持和调度信息。ADHD 个体需要“外部记忆系统”来补偿，比如清单、提醒、环境设计；LLM 则需要“外部记忆”如向量数据库、对话历史存档。（来源：无状态与外部记忆）

但工具只是零件，关键在于如何组装。一个 ADHD 用户随手把 Todoist 当便签用，和一位工程师把 LLM 当聊天框用，结果是一样的——信息散落、上下文丢失、执行断裂。真正的解法是构建一个完整的“harness”：包括任务分解、验证循环、定时重校准。

## 孔子与文天祥：古人的 harness 系统

孔子是一位典型的 ADHD 特质者：精力旺盛，周游列国 14 年坐不住；冲动爱骂人（“朽木不可雕”）；对音乐超专注到“三月不知肉味”，但对种地等俗事零耐心。他的 harness 系统是“克己复礼”——用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”）。这相当于一个外部调度层：每天固定时间 re-grounding，用仪式化的动作校准行为。对应到 LLM harness，就是定时上下文刷新和验证步骤。

文天祥的 harness 则是《正气歌》——用天地正气支撑自己，每天读圣贤书。这本质上是一个“信念锚点”，相当于 LLM 系统中的系统提示（system prompt），在每次行动前提供稳定的价值导向。

这两人都没有依赖某个“智能工具”，而是主动设计了自己的外部执行功能层。他们的成就（儒家创始人、民族英雄）恰恰证明：脚手架用好了，可以放大而非削弱内在能力。

## Todoist：从脚手架到拐杖的边界

Todoist 本身只是一个容器。它可以是脚手架，也可以是拐杖。关键区别在于：你是否在它之上建立了验证循环。

**脚手架用法：**
- 使用 Todoist 的“项目”和“标签”作为外部记忆分区，把大脑中的任务“卸载”出去。
- 每天固定时间（如睡前）回顾和整理，类似孔子的“三省吾身”——这是验证循环，把内部状态与外部清单核对。
- 结合任务分解工具（如 Goblin Tools）将大任务拆成小步骤，降低启动门槛。（来源：Goblin Tools）
- 利用 Saner.AI 的本地记忆功能减少搜索循环，避免在 Todoist 和笔记应用间反复切换。（来源：Saner.AI）

**拐杖用法：**
- 只把任务扔进去，从不回顾，导致清单变成“遗忘坟场”。
- 依赖 Todoist 的提醒功能，但提醒来了却因为缺乏上下文而无法行动。
- 把所有希望寄托在工具上，却没有建立自己的“调度层”——比如固定的回顾仪式。

LLM 工程师们对此应该非常熟悉：一个没有验证循环的 agent，会不断产生幻觉、重复工作、丢失状态。纯对话系统常因幻觉、缺乏接地和无法执行/验证行动而失败。（来源：AI Agent Systems...）同样，一个没有验证循环的 ADHD 用户，会不断遗忘任务、重复添加、最终放弃工具。

## 争议与局限

必须诚实指出：ADHD 大脑与 LLM 的同构性目前仅是一种理论类比，并非经过验证的科学事实。（来源：矛盾与存疑）这种类比有助于理解，但不能替代临床诊断或个性化方案。

此外，Todoist 本身缺乏针对 ADHD 的专门设计。它的任务分解能力有限，需要借助其他工具（如 Goblin Tools）来补偿。Mem 这类“AI 第二大脑”虽然承诺持久记忆，但依赖初始设置，且 AI 的工作记忆限制可能与 ADHD 用户需求存在矛盾。（来源：Mem）

依赖风险是真实存在的：AI 工具可能从脚手架变为永久拐杖，削弱内在执行功能。（来源：拐杖与脚手架）孔子的“七十而从心所欲不逾矩”花了整整一辈子，说明外部纪律最终要内化，而不是永远依赖。

## 今天就能试的行动

1. **建立每日回顾仪式**：在 Todoist 中设置一个每天固定时间（比如睡前）的重复任务，名为“回顾清单”。打开所有项目，把完成的任务归档，把未完成的重新评估优先级。这就是你的“验证循环”。
2. **任务分解实验**：下周遇到一个让你拖延的大任务（比如“整理房间”），先用 Goblin Tools 的 Magic ToDo 分解，再把子任务复制到 Todoist。对比一下完成率的变化。
3. **减少工具数量**：如果你同时在用 Todoist、Notion、苹果提醒事项，选一个作为主容器，把其他都关掉。ADHD 大脑最怕选择过多，一个 harness 就够了。
4. **写一条系统提示**：像给 LLM 写 system prompt 一样，给自己写一条“核心原则”（比如文天祥的《正气歌》），贴在 Todoist 的今日视图顶部。每次查看任务时先读一遍，作为行为锚点。

Todoist 之于 ADHD，就像上下文窗口之于 LLM：它本身只是存储，真正的力量在于你如何设计围绕它的调度与验证系统。用好了，它是让你飞起来的脚手架；用不好，它是让你原地踏步的拐杖。

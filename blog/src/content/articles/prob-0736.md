# Otter.ai 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Otter.ai：它到底补上了哪一层执行功能？

## 问题：为什么 Otter.ai 对 ADHD 用户总是不够“灵”？

如果你是一位 ADHD 用户，你可能试过用 Otter.ai 转录会议、课堂或灵感。你期待它能像魔法一样解决你的笔记负担，但结果往往是：转录了一大堆文字，却依然不知道重点在哪，任务依然堆在那里。如果你是一位 Agent 工程师，你可能熟悉 function calling——让 LLM 把精确计算外包给外部工具。但你有没有想过，这两件事其实在解决同一个问题？

## 同构视角：ADHD 大脑与 LLM 共享同一个核心缺陷

ADHD 大脑常被描述为一个高产的「生成核心」——想法丰富、联想活跃，但缺少稳定、可靠的调度层来筛选、组织、启动和坚持任务（来源：ADHD 的 AI 工具全景）。同样，大语言模型（LLM）拥有强大的生成能力，却在长程上下文、目标保持与执行调度上存在天然缺陷。两者都需要外部脚手架：ADHD 需要「外部执行功能层」，LLM 需要「agent scaffolding」（来源：ADHD 的 AI 工具全景）。

这种同构不只是比喻。ADHD 的典型困境——无法将高产的创意转化为有序的行动——与 LLM 在复杂任务中容易“跑偏”或“遗忘上下文”如出一辙。解决方案也同构：给生成核心配一个外部调度器。对于 LLM，这个调度器是 function calling 和 agent harness；对于 ADHD 大脑，这个调度器可以是工具，也可以是行为规则。

## Otter.ai 的正确用法：它不是记录仪，而是调度器

许多人用 Otter.ai 只做一件事：被动记录。这就像给 LLM 一个 function calling 接口，却只让它调用“文本存储”工具——浪费了调度层的潜力。Otter.ai 的真正价值在于它补上了 ADHD 大脑最缺的那一层：**捕获与反思**（来源：ADHD 的 AI 工具全景）。它实时转录会议、课堂与灵感，减轻笔记负担，但更重要的是，它提供了一个可检索、可回溯的外部记忆库，让 ADHD 用户不必依赖不稳定的工作记忆。

但仅此还不够。Otter.ai 必须与其他工具配合，才能构成完整的调度系统。例如，将 Otter.ai 的转录内容导入 Saner.AI 或 Mem 作为知识回忆的素材，再用 Goblin Tools 或 Lex 将会议中的任务分解为可执行的步骤。这样，Otter.ai 就从“被动记录仪”升级为“调度层的数据输入接口”。

## 其他工具的调度角色

除了 Otter.ai，其他工具也承担着调度层的不同功能：
- **Goblin Tools**：任务拆解与启动。其 Magic ToDo 将复杂任务分解为可管理的小步骤，降低启动门槛（来源：Goblin Tools）。
- **Saner.AI**：外部记忆与知识管理。通过知识回忆和本地记忆，补偿工作记忆的不稳定（来源：Saner.AI）。
- **Lex**：对话式执行与结构化推理。通过单一指令触发复杂任务序列，降低启动时的认知负荷（来源：Lex）。
- **ChatGPT/Claude**：被用作外部执行功能工具，帮助整理待办与简化任务（来源：ADHD 的 AI 工具全景）。

这些工具的组合，构成了一个完整的“外部调度系统”，让 ADHD 大脑的高产生成核心得以有序运作。

## 脚手架 vs 拐杖：一个诚实的边界

但我们必须指出争议：AI 工具到底是“外部执行功能层”还是“拐杖”？过度依赖可能导致能力退化，成为永久拐杖（来源：矛盾与存疑）。目前缺乏长期依赖风险的实证，更多是理论担忧。同样，在 LLM 领域，过度依赖 function calling 也可能让模型丧失自主规划能力。关键在于：工具应作为脚手架，帮助用户逐步内化执行功能，而非永久替代。

另一个矛盾：超聚焦是否应被引导而非打断？部分观点认为 AI 应提供“gentle structure”引导超聚焦到优先事项，而非强行打断；但部分 ADHD 用户反馈强行打断（如番茄钟）反而有效（来源：矛盾与存疑）。个体差异大，尚无定论。这意味着工具的使用需要个性化调优。

## 今天就能试的行动

1. **将 Otter.ai 的转录内容每周一次导入 Saner.AI 或 Mem**，建立可检索的知识库，减少“信息就在那里但想不起来”的挫败感。
2. **用 Goblin Tools 的 Magic ToDo 分解一个你拖延已久的任务**（比如“清理厨房”），调节“辣度”到你觉得可执行的粒度。
3. **尝试用 Lex 设置一个“单一指令”工作流**，例如“准备下周会议”触发“收集资料→写提纲→发送邀请”三步。
4. **建立自己的“日课”**：像孔子一样，每天固定时间回顾行为，用 Claude 或 ChatGPT 帮你梳理待办和反思，形成定时 re-grounding 的习惯。

记住：工具不是替代大脑，而是把 ADHD 的「高产生成核心」接入一个可维护的「执行调度系统」。用对工具，你就能像 LLM 配上 function calling 一样，把创意转化为行动。

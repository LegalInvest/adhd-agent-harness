# Goblin Tools 之于 ADHD，就像 上下文工程 之于 LLM——但有人用错了

> 从同构视角实测 Goblin Tools：它到底补上了哪一层执行功能？

## 问题：为什么你用了 Goblin Tools，反而更焦虑了？

如果你是一位 ADHD 患者，你可能试过用 Goblin Tools 把“写周报”拆成十几个小步骤，结果盯着清单更不想动；如果你是一位 LLM 工程师，你可能试过给模型塞满上下文让它“记住一切”，结果推理反而退化。两个场景共享同一个底层困境：**生成核心很强，但执行调度层缺失**。

Goblin Tools 之于 ADHD，就像上下文工程之于 LLM——但很多人用错了。他们把它当成拐杖，而不是脚手架。

## 同构：ADHD 大脑与 LLM 的共享困境

ADHD 大脑与 LLM 在结构上同构：两者都是强大的生成核心，但缺乏可靠的内置执行调度层，因此都需要外部脚手架来弥补调度缺陷（来源：AI 与 ADHD 的专注力）。

- **ADHD 侧**：注意力容易被环境带偏，工作记忆不稳定，任务启动困难。曾国藩 30 岁前“浮躁坐不住”，日记里天天骂自己“无恒”“浮躁”，读书“他人目下二三行，余或疾读不能终一行”。他的解决方案是“日课十二条”：黎明即起、读书不二、谨言、写日记反省。这套系统本质上是**外部调度层**，用固定的时间锚点（黎明）、单一任务规则（读书不二）和反省机制（日记）来补偿内部执行功能的不足。
- **LLM/Agent 侧**：LLM 受上下文窗口内容左右，上下文膨胀会导致推理退化。工程师需要做上下文工程：主动设计“当下注意什么”，比如用 system prompt 限定范围、用 RAG 检索最新信息、用 chain-of-thought 引导推理步骤。曾国藩的“日课”对应 LLM 的**定时 re-grounding**（每天重新对齐目标），“读书不二”对应**单一任务上下文**（避免多任务污染）。

两者的解法结构同构：**用外部结构（harness）弥补内部调度缺失**。

## 工具怎么用：脚手架 vs 拐杖

Goblin Tools 的核心价值在于**降低任务启动的门槛**——它把模糊目标拆成可执行步骤，补偿了 ADHD 的执行功能障碍（来源：执行功能障碍）。但问题在于：如果用户只是被动接收步骤，而不参与拆解过程，工具就变成了拐杖，反而削弱了内化能力。

同样，Focusmate 通过虚拟身体在场提供外部问责，弥补内在多巴胺调节不足（来源：Focusmate）。但它不是让用户永远依赖他人监督，而是通过一次次“被看见”的工作体验，逐步建立自我启动的神经回路。RescueTime 自动记录时间，揭示真实的时间去向（来源：RescueTime），但若用户只看报告而不调整行为，它就成了焦虑制造机。

**关键判断**：AI 工具作为外部执行功能层，其正确用法是**脚手架**——在能力边界提供支撑，同时保留让用户逐步内化的空间。曾国藩的“日课”最终内化成了习惯，释迦牟尼的“八正道”是戒律也是心法，它们都不是终身依赖的拐杖。

## 争议与局限

1. **依赖 vs. 内化**：目前缺乏长期追踪研究，无法确定 AI 工具是否会导致永久依赖（来源：矛盾与存疑）。
2. **临床证据不足**：Brain.fm 虽在 2026 年一项 44 款 ADHD 应用测试中位列最佳 9 款之一，但缺乏 ADHD 人群的独立随机对照试验（来源：Brain.fm）。
3. **个体差异**：ADHD 的异质性意味着同一工具对不同亚型效果迥异（来源：矛盾与存疑）。
4. **同构的边界**：ADHD 大脑与 LLM 的同构目前仅是功能类比，并非经过验证的科学事实（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Goblin Tools 拆任务后，手抄前三步**：不要只点“完成”，把步骤写下来，强迫自己参与拆解过程。
2. **每天设一个“Focusmate 时段”**：选最不想启动的任务，预约一个 25 分钟的身体加倍时段。
3. **用 RescueTime 做“时间审计”**：每周看一次报告，只关注一个最大的分心源，下周刻意阻断它。
4. **模仿曾国藩的“日课”**：每天固定三个时间点（起床、午休、睡前），做三件固定的事（比如写三句话日记、读一页书、深呼吸 1 分钟）。

Goblin Tools 不是魔法，它只是你大脑的调度器。用对了是脚手架，用错了是拐杖。选择权在你手里。

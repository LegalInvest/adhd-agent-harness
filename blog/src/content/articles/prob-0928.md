# Perplexity 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Perplexity：它到底补上了哪一层执行功能？

## 问题：为什么 Perplexity 让 ADHD 更焦虑，而不是更高效？

你打开 Perplexity，输入“如何开始写论文”，它秒回一个条理清晰的步骤清单。你松了口气，觉得终于有人帮你拆好了。然后你关掉页面，继续刷手机。

这不是你的错。Perplexity 回答了你“问的问题”，却没帮你“启动任务”。ADHD 大脑缺的不是答案，而是把答案变成行动的执行调度层。同样的问题也困扰着 LLM agent 工程师：给 GPT-4 一个复杂任务，它也能给出漂亮计划，但执行到一半就偏航、遗忘、卡住。两边都是高产生成核心，缺一个可靠的 planner-executor 循环来分解、调度、监控。

## 同构：ADHD 大脑与 LLM 共享同一个缺口

ADHD 大脑与大型语言模型在结构上同构——都是高产生成核心，却缺乏可靠的内置执行调度层（来源：ADHD 大脑与 LLM 的同构）。ADHD 的执行功能缺陷包括计划、组织、工作记忆、冲动抑制；LLM 的缺陷包括状态保持、真实规划、上下文控制。两者的核心矛盾一模一样：生成能力强，调度能力弱。

因此，所谓“AI 帮 ADHD 做时间管理”，本质上是在给这个生成核心套一个外部 harness：把原本应由大脑执行功能承担的调度、排序、提醒、抑制与监控，外包给 AI 工具（来源：外部执行功能层）。这就是 planner-executor 循环——AI 工具作为外部调度器，将任务分解、排序、定时，而 ADHD 大脑作为生成器，专注于执行每一步。

## 人物案例：孔子的“日课”就是最早的 planner-executor

孔子是典型的 ADHD 大脑：精力旺盛到周游列国14年坐不住，冲动爱骂人（“朽木不可雕也”），对音乐超专注到三月不知肉味，对种地等俗事零耐心。他的 harness 是什么？以“克己复礼”为核心，用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。70岁才做到“从心所欲不逾矩”，一辈子和自己的冲动做斗争。

这套系统与 LLM agent harness 结构同构：
- **“礼”** → 外部规则与边界，相当于 agent 的 system prompt 与约束条件；
- **“日三省”** → 定时 re-grounding，相当于 agent 的 checkpoint 与状态重置；
- **“七十而从心所欲”** → 经过长期训练后内化的调度能力，相当于 fine-tuned agent 的稳定执行。

孔子的成就——儒家学派创始人，万世师表——不是因为他没有 ADHD，而是因为他搭建了外部执行功能层。

## 工具实测：Perplexity 补上了哪一层？

Perplexity 的核心能力是**信息检索与整合**。它把散落的知识碎片快速整理成结构化答案。对于 ADHD，这补上了**工作记忆卸载**——你不用在脑中同时记住“要查什么、去哪查、怎么组合”，Perplexity 替你做了。对于 LLM agent，Perplexity 相当于一个**外部知识检索模块**，让 agent 不用在参数中记住所有事实。

但问题来了：Perplexity 只完成了 planner-executor 循环的**第一步**——规划（planning），却把执行（execution）留给了你。ADHD 大脑最弱的恰恰是执行启动。所以 Perplexity 给你一个完美计划，你却无法迈出第一步——这反而加重了挫败感。

## 错用：把“脚手架”当“拐杖”

Perplexity 被错用，是因为用户把它当作**答案生成器**，而不是**执行调度器**。它回答了“是什么”，却没回答“然后呢”。真正的 ADHD harness 应该像 Motion、Reclaim.ai、Tiimo 那样，把计划转化为**时间块**和**提醒**，直接驱动执行。

- **Motion** 自动排程并动态调整，消除“下一步该做什么”的决策负担（来源：Motion）；
- **Reclaim.ai** 保护时间块，防止会议侵占深度工作（来源：Reclaim.ai）；
- **Tiimo** 将时间视觉化，缓解时间盲（来源：Tiimo）；
- **Goblin Tools** 的“魔法待办”将任务微分解，降低启动门槛（来源：任务启动）。

这些工具共同构成一个完整 planner-executor 循环：
1. **规划**：AI 分解任务、排序、估算时间；
2. **调度**：AI 分配时间块、设置提醒；
3. **执行**：用户只负责“做当前块”；
4. **监控**：AI 跟踪进度、动态调整。

Perplexity 只做了第1步。用它替代整个循环，就像给 LLM agent 只装 planner 不装 executor——计划完美，执行瘫痪。

## 争议与局限：脚手架与拐杖的边界

必须诚实指出，AI 工具作为外部执行功能层，存在“过度依赖导致能力退化”的风险（来源：拐杖与脚手架）。长期依赖可能让 ADHD 大脑更不擅长自我调度。目前缺乏纵向研究验证长期效果。此外，个体差异巨大：有人需要强硬打断（如番茄钟），有人需要柔性引导（如 Tiimo 的视觉结构），尚无定论（来源：超聚焦）。

## 今天就能试的 3 件事

1. **把 Perplexity 答案变成任务**：用 Perplexity 查“如何写论文”后，立刻把步骤贴到 Todoist 或 Goblin Tools 中，分解成 5 分钟可完成的微任务，并设定第一个任务的提醒。
2. **试用 Motion 或 Reclaim.ai 的自动排程**：让 AI 替你决定“现在做什么”，而不是自己纠结。免费试用 7 天，观察任务启动率的变化。
3. **建立“日课” re-grounding 仪式**：像孔子一样，每天固定时间（如睡前）用 AI 工具回顾今日完成度，并规划明日 3 个关键任务。用 Saner.AI 的“两分钟法则”启动第一个任务。

## 结论

Perplexity 之于 ADHD，就像 planner 之于 LLM agent——它是必要但不充分的组件。真正的 harness 是完整的 planner-executor 循环：外部调度层接管分解、排序、提醒，让 ADHD 大脑专注于生成与执行。别把脚手架当拐杖，也别只拿半套工具。搭好你的执行功能层，才能让 ADHD 大脑从“知道”变成“做到”。

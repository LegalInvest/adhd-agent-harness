# Perplexity 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Perplexity：它到底补上了哪一层执行功能？

## 问题：为什么 Perplexity 没能拯救你的执行力？

你打开 Perplexity，输入一个宏大问题：“如何开始写论文？”它瞬间吐出一份结构清晰的步骤列表。你如获至宝，复制到 Todoist，然后……继续刷手机。第二天，同样的循环。

这不是你的错。Perplexity 擅长的“任务分解”恰恰是 ADHD 大脑最需要的外部调度层——但多数人只用了它的一半功能。它像 LLM 的 planner-executor 循环：planner（规划器）分解任务，executor（执行器）逐块完成。但你的大脑只有生成核心（智力、创造力），没有可靠调度层（执行功能）。Perplexity 补上了 planner，却把 executor 留给了你——而你的 executor 本就失灵。

## 同构：ADHD 大脑与 LLM 是同一类“高产但缺调度”的生成核心

ADHD 的神经生物学特征是多巴胺功能失调，导致中脑皮质分支功能低下，引起“注意反应缺陷和行为计划不良”（来源：多巴胺）。这正是执行功能缺陷的核心：你无法将目标分解为有序步骤并启动。LLM 同样如此：它能生成惊艳文本，但若没有 chain-of-thought 或 planner-executor 框架，就会在长任务中迷失、重复、跑题。

两者的解法结构同构：都需要外部“脚手架”来提供记忆、调度和约束。对 LLM 而言，是 agent 框架（如 LangChain 的 planner-executor）；对 ADHD 而言，是 AI 工具如 Motion、Reclaim.ai、Tiimo。Motion 自动分解任务并动态调整日程，消除“下一步该做什么”的决策负担（来源：Motion）；Reclaim.ai 通过保护深度工作时间块来维持日程稳定（来源：Reclaim.ai）；Tiimo 将时间视觉化，直接对抗时间盲（来源：Tiimo）。这些工具充当了外部调度层，让生成核心得以专注产出。

## 但有人用错了：把脚手架当拐杖

多数人使用 Perplexity 的方式是：问一个问题 → 得到步骤列表 → 关闭窗口。他们只用了 planner，却期待 ADHD 大脑自动执行 executor 的角色。这就像给 LLM 一个复杂指令却不给 chain-of-thought——它要么崩溃，要么输出垃圾。

真正有效的用法是：让 Perplexity 同时承担 planner 和 executor 的辅助角色。例如，将分解后的步骤直接推送到 Todoist 或 Motion，让工具自动排程并提醒启动。Focusmate 则提供“身体在场效应”——通过算法匹配虚拟伙伴，提供隐性问责（来源：身体在场效应）。这相当于给 executor 加了一个社会性监督层。

但这里存在一个根本矛盾：工具是“拐杖”还是“脚手架”？拐杖让人依赖，脚手架可逐步撤除。当前多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。长期依赖可能阻碍内在执行功能的发展——你永远学不会自己分解任务、启动任务。

## 诚实面对局限：证据与争议

同构命题本身是理论模型，证据主要来自概念类比和工具案例，缺乏大规模实证（来源：矛盾与存疑）。Motion、Reclaim.ai 的有效性基于用户反馈，无独立临床研究（来源：Motion；Reclaim.ai）。多巴胺干预的效果存在争议——虽然工具通过即时反馈可能间接调节多巴胺，但证据尚弱，且可能引发成瘾风险（来源：多巴胺；矛盾与存疑）。

个体差异也很大：注意力缺陷为主 vs. 多动冲动为主的 ADHD 对工具反应不同，现有工具多针对通用缺陷，个性化不足（来源：矛盾与存疑）。此外，初始设置对执行功能弱的用户本身就是挑战——Motion 需要你输入所有任务和截止日期（来源：Motion），这恰恰是 ADHD 的痛点。

## 今天就能试的 3 条行动

1. **Perplexity + Todoist 联动**：用 Perplexity 分解任务后，立即将每个步骤作为独立任务输入 Todoist，并设置具体时间。让 Todoist 的提醒代替你的内部时钟。
2. **试用 Motion 或 Reclaim.ai 一周**：选择一款自动排程工具，将你所有重复任务和项目输入，观察它能否降低你的启动阻力。注意：前 3 天可能因设置困难而受挫，坚持到第 5 天。
3. **每周一次“无工具日”**：选一天只用纸笔规划，训练内在执行功能。这是脚手架撤除的起点——避免成为拐杖的奴隶。

## 结论：工具补不上缺失的调度层，但能帮你学会调度

Perplexity 之于 ADHD，就像 planner-executor 之于 LLM——它补上了你缺失的 planner，但 executor 仍需锻炼。真正的解法不是找到完美的工具，而是把工具当训练轮：先用它们启动，再逐步减少依赖，直到你的大脑自己学会分解与执行。而工程师们，你们的 agent 框架同样需要设计可撤除的脚手架——否则用户永远无法独立。

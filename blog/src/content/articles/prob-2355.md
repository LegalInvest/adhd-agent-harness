# ADHD 创业者如何用「给生成核心套 harness」的思路应对把想法落地执行里的任务启动困难、不会拆解？

> 把 LLM 工程里的「agent 用 planner-executor 循环分解长任务」迁移到ADHD 创业者的把想法落地执行场景，双域证据作答。

## 从同一个问题出发：想法多，行动难

你是否有过这样的体验：脑子里同时涌出十几个创意，但坐在电脑前却无法敲下第一行代码？或者，你为自己设计了一个完美的项目计划，但一到执行就卡在“第一步怎么拆”上？

这不是意志力问题。ADHD 大脑的核心特征是“强记忆、弱控制”：工作记忆容量甚至超常，但执行功能——任务启动、时间管理、灵活切换——却频繁故障（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

有趣的是，这种“高产生成 + 弱调度”的模式，与当今的 LLM 惊人地同构。LLM 本身是强大的文本生成核心，但一旦面对长序列、多步骤任务，它的注意力就会在冲突中崩溃——就像 ADHD 大脑在复杂任务前“宕机”（来源：Deficient Executive Control in Transformer Attention）。

那么，LLM 工程师如何解决这个问题的？他们给生成核心套上“harness”——一个包含 planner、executor、记忆和验证循环的外部脚手架。这个思路，恰好也是 ADHD 创业者需要但往往不知道的答案。

## 同构的解法：给生成核心套 harness

### LLM/agent 侧：planner-executor 循环

在构建 AI agent 时，工程师发现，仅靠 LLM 本身无法可靠地完成复杂任务。于是他们设计了“agent scaffolding”——围绕 LLM 构建的软件架构，使其能够推理、规划并分步执行（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。

典型流程是：
- **Planner** 将大目标分解为子任务；
- **Executor** 依次执行，每步调用工具或检索记忆；
- **验证循环** 检查结果，必要时回溯或调整计划。

这种设计承认了 LLM 的无状态和弱控制，转而用外部系统来补偿。例如，Motion 和 Reclaim.ai 就是这种思路的消费者版本：它们自动将任务按优先级排入日程，动态调整，减少用户的决策负担（来源：11 Best ADHD Productivity Apps for Fluctuating Energy）。

### ADHD 侧：外部执行功能层

ADHD 创业者同样需要这样的“外部调度层”。研究指出，ADHD 患者的前额叶皮层活动不足，而 AI 工具通过外部化计划、提醒和排序，模拟了部分执行功能（来源：AI 与 ADHD 的时间管理）。

以 Tiimo 为例，它用视觉化时间线让时间流逝变得可感，直接应对“时间盲”——就像 LLM 依赖 RAG 来补充缺失的上下文（来源：Best AI Tools for ADHD Productivity in 2026）。Todoist 则充当外部工作记忆，将任务从大脑中卸载。

最关键的洞察是：**ADHD 大脑与 LLM 需要的不是更强大的生成能力，而是一个可靠的调度层**。正如孔子用“礼”作为行为边界，每日反省（“吾日三省吾身”），相当于给自己设置了定时 re-grounding 循环；彭玉麟用“三不要”原则和每日画梅花来磨练心性，相当于用外部纪律约束冲动。这些古老的自制系统，本质上就是“人肉 harness”——与 LLM 的 planner-executor 循环同构。

## 核心观点：脚手架，不是拐杖

这里必须划清一条界限：外部 harness 是脚手架，不是永久拐杖。

当前争议在于，过度依赖 AI 工具可能削弱内在执行功能的代偿发展（来源：矛盾与存疑）。例如，Motion 自动排程虽然减少了决策疲劳，但长期使用后，用户可能失去手动规划的能力。同样，LLM agent 如果完全依赖外部记忆，其内部上下文管理能力会退化。

因此，最优策略是**渐进式脚手架**：初期用工具完全接管调度，然后逐步将部分控制权交还给用户/模型。比如，先用 Reclaim.ai 保护深度工作时间，再尝试手动微调日程；或者让 agent 的 planner 先完全自动，再引入人工审核。

## 今天就能试的 3 条行动

1. **用“planner 优先”启动任何项目**：面对一个想法，别直接开始做。先用 10 分钟写一个“任务分解清单”，像 LLM agent 的 planner 那样，把大目标拆成 3-5 个可执行的小步骤。工具可用 Goblin Tools 的魔法待办（自动分解任务）或 Todoist 的分层列表。

2. **设置外部 re-grounding 循环**：像孔子的“日三省”或彭玉麟的每日画梅，给自己安排一个固定时间（如每天上午 10 点）检查计划执行情况。可以用 Tiimo 的视觉提醒或 Reclaim.ai 的日程保护块来触发。

3. **接受“弱控制”，拥抱外部记忆**：别试图用大脑记住所有上下文。像 LLM 依赖 RAG 那样，把项目笔记、灵感、待办全部存入一个“第二大脑”（如 Obsidian 或 Notion），并定期回顾。Motion 的自动排程可以帮你决定“下一步做什么”，减少决策负担。

## 局限与诚实提醒

同构理论虽诱人，但目前缺乏实证基础——它更多是一种启发式类比，而非科学事实（来源：矛盾与存疑）。此外，多数 AI 工具（如 Motion）缺乏独立临床验证，其效果可能受安慰剂效应影响。个体差异也很大：有人对视觉化工具（Tiimo）反应好，有人更适合自动排程（Reclaim.ai）。

最重要的是，工具永远只是辅助。真正的改变来自理解自己的神经类型，并设计出适合的“个人 harness”——就像孔子到了 70 岁才“从心所欲不逾矩”，这个工程需要耐心迭代。

# Tiimo 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了：可视化时间线不是给你看的，是给"此刻的你"看的

> 她给 Tiimo(可视化日程 app)做了最漂亮的配置:全天时间线、彩色图标、每个环节的可视块——一张艺术品级的「我的一天」。但四周后她发现一个奇怪的事实:**这张时间线她每天只看两次——早上(欣赏它)和深夜(哀悼它)**;中间那十四个小时,真正需要它的时刻——2 点该切换任务了、4 点该出门了——她根本不记得看。工具没坏,配置很美,但它的工作方式建立在一个对她不成立的假设上:「用户会主动查看」。她的教练一句话点醒:「你把它配置成了一幅**挂在墙上的画**,但你需要的是一个**在正确的时刻拍你肩膀的人**。用 function calling 的语言说:你以为自己在做工具调用,其实你连调用都没发生——**LLM 的 function calling 是模型在需要时主动发起调用;而你的『需要的时刻』,恰恰是你最不可能想起工具存在的时刻——所以调用必须反向:让工具调用你。**」

收敛:自动重排篇管提醒的信号纯度,本文管一个更基础的配置问题——**「主动查看型」与「反向调用型」两种使用拓扑的根本差别(为什么前者对这个剖面必然失效)?把可视化日程配置成反向调用的四个要点?以及可视化本身的真正价值点(它不该被扔掉,该被放对位置)?**

## 穿透:时间盲的本质是"当下不派代表出席未来"——所以一切靠"到时候我会想起来"的设计,都在向不存在的功能收费

拓扑差别,说穿了一层窗户纸:主动查看型工具的隐含调用链是「内部时间感知报时→想起有日程表→打开查看→执行」——**第一环「内部报时」恰是时间盲(time blindness)缺的那个功能**:沉浸在当前事务时,「现在几点了/该干嘛了」的后台进程根本不运行——链条断在第一环,后面配置得再美都是下游的摆设(早上欣赏、深夜哀悼,恰好是唯二不需要报时的时刻——讽刺闭环)。反向调用型的链条是「外部信号强制到达→执行」——一环,不依赖任何内部功能。**两种拓扑的区别不是使用习惯的勤懒,是「向谁收费」:前者向你不存在的内部时钟收费,后者向永远在线的外部信号收费。**(重锚成本篇的拓扑定律,在时间感知场景的精确重演:自我引导贵,外部注入便宜。)

反向调用的四个配置要点:**①转场信号,不是时段信号**——提醒设在「切换时刻」而不是「时段开头」:「14:00 写报告」没用,「13:55:五分钟后切换,现在开始收尾」才有用——**时间盲最贵的不是不知道现在几点,是不知道『该停了』**——给「停」配信号,比给「开始」配信号更关键;②**信号必须穿透沉浸**——超聚焦状态下,普通通知如投石入海——重要转场用穿透级信号(手表震动>手机横幅;必要时物理闹钟放到需要起身才能关的位置——让身体动起来,就是最古老的 function calling);③**信号带 payload**——响铃只说「时间到」是半成品,信号内容要带「下一个动作」(「收尾,存档,换鞋——先喝口水」)——**把断点卡塞进提醒正文,让信号到达即上下文就位**(三行托盘的推送版);④**每天只配 3–5 个关键转场**——全天二十个提醒=自动重排篇的消退剧本重演——只给「错过代价高」的转场配信号(出门、接人、会议、睡觉线),其余交给可视时间线的另一个用途——见下。

可视化的真正价值点(别扔掉它):可视时间线的强项不是提醒(那是信号系统的活),是**两个静态用途**:①**计划时的现实检验**——把一天可视化铺开,超载一目了然(「这些块根本塞不下」——数字清单会骗人,像素不会),它是规划谬误的照妖镜;②**迷失时的重锚面板**——当你从超聚焦里浮出来、一脸茫然「我现在该干嘛」时,一眼时间线即可重定位(这是「查看」的正确时机:不是定时巡逻,是迷路时看地图)。**配置哲学一句话:动态的「调用你」交给 3–5 个穿透信号,静态的「被你查」留给规划与迷失两个时刻——一个工具,两种用法,各就各位。**

## 验证

可测:反向配置两周——关键转场的命中率(准时出门/准时收尾的次数)对比配置前;信号总数与划掉率(防消退检查)。可证伪:若你的时间感知在服药时段内明显在线(部分人如此),主动查看型在该时段可用——拓扑按时段混配,不必教条;药效窗口外,反向调用依然是底盘。

## 决策

做什么:3–5 个关键转场配穿透信号+payload;「停」的信号优先于「开始」;时间线用于规划检验与迷失重锚。

不做什么:不要给全天铺满提醒;不要用「多看看日程表」自我要求(向不存在的功能收费);不要因为「早上欣赏深夜哀悼」而弃用(那是配置错误,不是工具或你的失败)。

先做什么:今天挑出明天错过代价最高的三个转场,各配一条带 payload 的穿透信号。

## 边界

时间盲(时间感知缺损)有ADHD文献基础(GRADE 中高);配置方案为实践设计(无对照研究)。function calling 类比按 B 级功能同型使用,本文迁移的是「调用方向」这一个设计维度(LLM 的调用由推理触发,人的反向调用由时钟触发——机制不同,拓扑教训相同)。不构成对特定产品的评价,原则适用于一切可视化日程工具。

## 今天就能试的 3 件事

1. 配明天的三个穿透转场信号。
2. 把「停止收尾」提醒加到今晚的睡觉线前。
3. 用时间线做一次明日计划的现实检验——塞不下的,现在就删。

本文服务于人生 Harness 金字塔的**执行层(时间信号系统)**:你不是不守时的人,你是体内不装时钟的人——而不装时钟从来不是问题,城市里到处都是钟;问题只是别再假装体内有钟,然后**把外面的钟,接到你的肩膀上:该停的时候拍一下,该走的时候拍一下——剩下的十四个小时,随你沉浸到天涯。**

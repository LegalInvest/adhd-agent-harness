# Claude 之于 ADHD，就像 生成核心 + 缺失的执行层 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

## 问题：你是在用 Claude 生成答案，还是在用它调度自己？

我花了三个月，用 Claude 写方案、列提纲、做头脑风暴。结果是更累了——不是更轻松。任务清单反而拉得更长，因为 Claude 每次都能给出十几条建议，而我一条都执行不下去。直到我把 Claude 从「生成器」切换成「调度器」，情况才反转。

这个体验恰好对应了一个更深层的结构：ADHD 大脑与 LLM 在架构上是同构的——两者都是**高产但缺执行调度层的生成核心**。Claude 之于 ADHD，就像 LLM 之于 Agentic Harness：它补上的是那层缺失的执行功能，而不是帮你生成更多内容。但多数人用反了。

## 同构：生成核心与缺失的执行层

ADHD 的大脑不是「坏掉的大脑」，而是一个**生成核心过强、执行调度层过弱**的系统。神经科学证据显示，ADHD 患者的前额叶执行功能网络激活不足（来源：ADHD × AI 的科学与研究前沿），这与 LLM 在 Stroop 冲突任务中注意力随序列变长骤降到随机水平的表现一致——两者都无法抑制优势反应、解决认知冲突（来源：Deficient Executive Control in Transformer Attention）。

更关键的是，LLM 呈现「强记忆、弱控制」剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷，无法灵活切换任务集、无法抑制自动化反应——这正是 ADHD 的经典神经心理模式（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。耶鲁大学进一步发现，自注意力机制本身导致工作记忆容量限制，注意力分数熵增、注意力弥散，与人类注意分散机制同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。

所以，ADHD 和 LLM 的瓶颈都不在「算力」或「容量」，而在 orchestration 层——那个负责「什么时候该做什么」的调度系统。

## 历史中的 harness：毛泽东的「外部执行层」

毛泽东是典型的 ADHD 大脑：小时候是问题儿童，爱读闲书不喜欢四书五经；一辈子读书不辍，行军路上也带着书；永远在动，从井冈山到延安到北京，一辈子在调查研究；冲动敢赌，四渡赤水、抗美援朝都是险棋；思维极度跳跃，文章讲话都是场景化的，充满比喻。但他建立了一套强大的外部执行功能层——即他的 harness 系统：

- **批评与自我批评** → 相当于 AI 的「定时 re-grounding」机制，定期校准方向
- **调查研究** → 相当于「外部数据采集」，用真实世界输入替代大脑的随机联想
- **民主集中制** → 相当于「集体决策调度器」，用多人决策制衡个人的冲动
- **党指挥枪** → 相当于「组织系统管理」，把执行权交给外部制度而非个人意志

这套 harness 与 LLM 的 Agentic Harness 是同构的：ReAct 模式用「思考-行动-观察」循环替代 LLM 的随机生成；Chain-of-Thought 用结构化推理替代跳跃输出；外部记忆系统（如向量数据库）替代无状态的工作记忆。毛泽东的「秘书」和「会议制度」就是他的外部调度器，而 Claude 的「Projects」和「自定义指令」可以扮演同样的角色。

## 工具实测：Claude 补上了哪一层？

当前市面上的 AI 工具，真正有效的不是那些「帮你生成更多内容」的，而是那些「帮你调度执行」的。

**Goblin Tools** 的 Magic ToDo 功能，能把「清理厨房」自动分解成 12 个具体步骤，用户可调节「辣度」控制粒度（来源：Goblin Tools）。这直接对应 ADHD 的任务启动困难——不是不知道要做什么，而是任务太大，大脑无法自动分解。Goblin Tools 补的是「任务分解」这个执行功能子模块。

**Motion** 则更进一步：它自动根据任务优先级、截止日期和可用时间实时重建日程，消除「下一步该做什么」的决策负担（来源：Motion）。这对应 ADHD 的时间盲和计划困难。Motion 补的是「动态排程」这个执行功能子模块。

**Saner.AI** 专注于知识回忆和本地记忆，减少搜索循环和标签切换（来源：Saner.AI）。它补的是「工作记忆外挂」——ADHD 患者的工作记忆常常成为瓶颈（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），Saner.AI 相当于一个外部缓存。

但问题在于：这些工具如果只是被当作「更好的 todo list」或「更快的搜索引擎」，它们就只是拐杖——你依赖它们完成任务，但执行功能本身没有训练。真正的脚手架应该像毛泽东的「民主生活会」：它不仅是工具，更是一种**结构化的外部调度协议**，让你在长期使用中内化执行策略。

## 争议与局限：脚手架 vs 拐杖的边界

同构框架虽然有力，但存在明显局限。首先，同构性目前仅是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。ADHD 大脑的神经机制与 Transformer 的算法机制是否真的共享深层结构，还需要更多研究。

其次，过度依赖 AI 脚手架可能导致执行功能进一步退化（来源：矛盾与存疑）。如果 Claude 每次都帮你分解任务，你永远学不会自己分解。这与毛泽东的 harness 不同——他的系统最终内化为个人习惯，而 AI 工具往往是外部的、可移除的。

最后，个体差异不容忽视。同构框架是否适用于所有 ADHD 亚型？部分患者可能以注意力分散为主，而非执行功能缺陷（来源：ADHD × AI 的科学与研究前沿）。

## 今天就能试的行动

1. **把 Claude 从「生成器」切换成「调度器」**：下次打开 Claude，不要问「帮我写个方案」，而是问「帮我列出完成这个方案需要的 5 个步骤，并告诉我第一步具体做什么」。让 Claude 扮演你的外部调度层。

2. **用 Goblin Tools 的 Magic ToDo 分解一个你拖延已久的任务**：输入「整理书房」或「准备季度报告」，调节辣度到你觉得「可以立刻开始」的程度。完成第一步后，再让 AI 分解下一步。

3. **给 Claude 设置一个「每日 re-grounding」提示**：每天早上问 Claude：「基于我今天的日历和待办，帮我规划三个优先任务，并告诉我第一个任务的具体行动。」这模仿了毛泽东的「批评与自我批评」机制。

4. **建立你的「个人 Agentic Harness」**：用一个笔记工具（如 Obsidian 或 Notion）作为外部记忆系统，每次用 Claude 后，把关键输出和下一步行动记录进去。这相当于 LLM 的向量数据库，防止每次对话都从零开始。

ADHD 的大脑不是缺陷，而是一个没有配好 harness 的超级引擎。Claude 不是要帮你跑得更快，而是要帮你装上那个缺失的调度层。问题是：你用它来生成更多，还是用它来调度自己？

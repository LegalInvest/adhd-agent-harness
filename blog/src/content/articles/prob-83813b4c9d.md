# 为什么你应该先做“丑但能用”的模板，而不是“完整但难用”的系统？

> 《问题260》人工精修选题，双域证据作答。

你有没有过这种体验：脑子转得飞快，但身体就是启动不了；或者 Agent 的 LLM 生成得头头是道，却连一个完整任务都跑不到底？

一边是人类天天被「想理解自己的大脑到底是怎么回事」折磨；另一边是工程师发现，LLM 什么都懂，却缺一个能把它按住、排好队、执行到完的调度层。这两个问题看起来毫不相干，但答案却惊人地一致：**先做一个丑但能用、能立刻上手的模板，而不是一个完整但难用、让人望而却步的系统。**

## 01 同一个问题：生成核心太强，执行调度层太弱

ADHD 的核心困境不是智力不够，而是执行功能——也就是大脑的「驾驶座」——常常失灵。很多患者描述，这感觉就像「方向盘后没有人」（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。计划、启动、维持、切换、时间感知，这些任务对个人来说极其困难，而工作记忆的瓶颈进一步削弱了任务编排能力（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

LLM 的情况几乎一样。它拥有强大的语言生成与理解能力，但作为单一模型，它缺乏可靠的执行调度层。现代 AI 编码代理（如 OpenDev）不得不通过「复合 AI 系统架构」来弥补：工作负载专用模型路由、分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩——本质上都是在 LLM 外部搭一个调度层，防止上下文膨胀和推理退化（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

这就是同构：**ADHD 大脑和 LLM 都是「高产但缺执行调度层的生成核心」**。两者都需要外部 harness，把强大的生成能力导入可完成的目标流。

## 02 模板优先：丑但能用，是 harness 的最小可验证单元

对 ADHD 侧来说，最痛苦的往往不是不会做，而是不知道第一步是什么。Goblin Tools 的 Magic ToDo 之所以被评价为「简单有用」，就是因为它接受模糊输入，比如「清理厨房」，然后直接输出可执行的步骤清单（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。它不需要你建立一套完美的 GTD 系统，只要先给你第一个动作。对时间盲问题，Motion 的 AI 自动排程把「下一步该做什么」的决策负担直接拿走，日程被打乱时还会自动重新安排（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。而 Saner.AI 则通过本地记忆和知识回忆，减少 ADHD 用户在多标签、多应用之间的搜索循环（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。

对 LLM 侧来说，最好的 harness 也不是一开始就写一个完整的通用 Agent，而是先写一个能跑通一个具体任务的「丑脚本」：ReAct 和 Chain-of-Thought 已经被证明能显著提升任务完成率（来源：ADHD × AI 的科学与研究前沿），OpenDev 的工程师们也强调， scaffolding 是「由提示、记忆、代码、工具和编排逻辑组成的模块化框架」（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。所谓模板，就是让生成核心在最小闭环里先动起来。

所以，**「丑但能用」不是凑合，而是工程策略**。它让一个缺调度层的生成核心，在最短时间内获得外部执行结构，而不是在追求完美系统中永远卡在启动阶段。

## 03 名人 harness：毛泽东的外部调度层

毛泽东是理解这一点的绝佳案例。他的 ADHD 行为模式非常典型：小时候是问题儿童，与父亲冲突，爱读闲书不喜四书五经；一辈子读书不辍、调查研究不停；思维极度跳跃，讲话文章充满场景化比喻；决策冲动敢赌，四渡赤水、抗美援朝都是险棋。

他的专属 harness 系统不是一套精密的个人效率系统，而是一组可以立即套用的外部模板：

- **批评与自我批评 / 民主生活会**：让别人定期提意见，相当于给生成核心引入「外部监控器」，防止冲动和盲点。
- **调查研究**：「没有调查就没有发言权」相当于给决策系统加了一个强制 grounding 步骤，避免 LLM 式的幻觉和跳跃。
- **民主集中制**：用集体决策制衡个人冲动，对应 LLM  harness 中的「多代理审议」或「人在回路」。
- **党指挥枪**：用组织系统管理军队和国家，就是把生成核心的高能量导入外部执行管道。

这些模板都不是「完整系统」，但每一条都能立刻执行、马上纠偏。这正是 LLM 工程师今天在做的事：用一个外部调度器、一个记忆模块、一个工具调用层，把模型的高产出约束在可控流程里。

## 04 诚实提醒：同构是假设，不是定论；工具是脚手架，不是拐杖

我必须把丑话说在前头：ADHD 大脑与 LLM 的同构性目前仍是一种理论类比，并非经过严格实证验证的科学事实。不同资料在表述它时有时像事实、有时像假设，存在不一致（来源：全局矛盾与存疑）。另外，Motion 等工具缺乏独立临床验证，却仍被当作有效工具推荐，这点需要谨慎（来源：全局矛盾与存疑）。

更重要的是，外部工具必须区分「脚手架」和「拐杖」。真正的脚手架帮助你建造，但你自己仍需攀登；如果 AI 工具替代了治疗或自我管理，就可能造成依赖（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。对 ADHD 侧，模板的作用是降低启动门槛，而不是替你生活；对 LLM 侧，harness 的作用是约束生成，而不是让模型永远替你思考。

## 05 今天就能试的四条行动

1. ** ADHD 读者：用 Goblin Tools 或 Saner.AI 做一个「最小任务入口」**。把一个你拖延已久的任务丢进 Magic ToDo，只要拿到前三步，不要管剩下的系统长什么样。

2. **工程师读者：先写一个不漂亮的 ReAct/CoT 模板**。不要一上来就设计通用 Agent，先让 LLM 在一个具体任务上「思考—行动—观察」跑三圈，确认它能闭环。

3. **两边都做「外部重定向」**：ADHD 侧用 Motion 或日历把「下一步做什么」交给外部调度；LLM 侧用一个显式的 planner 模型把「下一步做什么」从生成模型里拆出来。

4. **每周做一次「脚手架审计」**：问自己——这个模板是在帮我启动，还是在替我执行？如果是后者，把它拆得更小，让生成核心自己重新接管一部分。

## 结语

理解大脑和造 Agent，最终遇到的是同一个障碍：强大的生成核心，不能自己把自己安排明白。完整的系统会诱惑你，但「丑但能用」的模板会先救你。先跑起来，再优化，是对 ADHD 和 LLM 都成立的工程伦理。

# 用 ChatGPT 给 ADHD 补上人在回路与身体在场，是脚手架还是拐杖？

> ChatGPT 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：独自做事时，为什么总是放弃？

ADHD 人群最熟悉的挫败感，不是“做不了”，而是“做着做着就散了”。没有人在旁边盯着，没有即时反馈，任务就像漏气的皮球，越滚越慢，最后停在某个角落。工程师在做高风险 agent 时也有类似焦虑：一个自主的 LLM agent 如果缺乏人类监督，很容易在上下文膨胀中偏离轨道，产生幻觉，甚至执行危险操作。两边的痛点惊人一致：**缺乏一个可靠的执行调度层，来保证生成的东西能被验证、被落地。**

## 同构：ADHD 大脑与 LLM agent 的共享结构

ADHD 大脑常被描述为“高产但缺乏可靠执行调度层的生成核心”（来源：[概念页：上下文工程]）。它能产生大量创意、联想和冲动，但缺少一个“人在回路”的监督机制来筛选、排序、执行。LLM agent 同样如此：作为一个生成核心，它天然倾向于输出自信但可能错误的答案，需要 harness 工程提供上下文传递、工具接口、验证循环和记忆系统（来源：[概念页：幻觉与验证循环]）。两者的结构完全同构：都是生成核心 + 薄弱调度层，解决方案都是外部上下文工程 + 验证循环。

## 工具：AI 如何补上“人在回路”与“身体在场”

当前 AI 工具正是通过补上这两个缺口来帮助 ADHD 人群。**人在回路**意味着有一个外部系统持续观察、提醒、反馈；**身体在场**意味着通过小步骤的即时反馈，让注意力锚定在当下。

- **Goblin Tools** 的 Magic ToDo 功能将复杂任务自动分解为可管理的小步骤，降低启动门槛，并通过每一步的完成提供即时反馈（来源：[工具页：Goblin Tools]）。这相当于为 ADHD 大脑提供了一个外部验证循环：每一步完成后，系统提示下一步，防止上下文断裂。
- **Motion** 通过 AI 自动排程与动态调整，消除“下一步该做什么”的决策负担，直接应对时间盲和任务启动困难（来源：[工具页：Motion]）。它相当于一个持续在线的调度器，确保上下文始终聚焦于当前最优先的任务。
- **Saner.AI** 专注于知识回忆和本地记忆，减少搜索循环和标签切换（来源：[工具页：Saner.AI]）。它补上了工作记忆不足的缺口，让 ADHD 用户不必在多个应用间丢失上下文。

这些工具的共同逻辑是：**将原本需要内在执行功能（工作记忆、时间感知、任务分解）完成的工作，外包给 AI 系统**。在 LLM/harness 一侧，这对应着 harness 工程中的上下文管理、验证与 CI 集成、结构化工作流和规划（来源：[概念页：幻觉与验证循环]）。两者都依赖外部系统来维持上下文连贯和结果验证。

## 脚手架 vs 拐杖：边界在哪里？

核心判断：**AI 工具是脚手架还是拐杖，取决于是否主动设计退出机制与验证循环。** 脚手架是临时辅助，最终要拆除；拐杖是永久替代，长期使用会导致能力退化。

从 wiki 资料的矛盾与存疑中，我们看到明确的警告：过度依赖 AI 工具可能削弱内在时间感知能力（来源：[全局矛盾与存疑]），存在依赖风险（来源：[时间盲] 页面）。工具设计者声称是“补偿”而非“替代”，但缺乏如何避免依赖的具体指导（来源：[外部执行功能层] 页面）。这意味着，如果用户只是被动接受 AI 的分解和提醒，而不主动参与验证过程，工具就会从脚手架滑向拐杖。

在 LLM/harness 一侧，同样的边界存在：一个 agent harness 如果只是盲目执行工具调用而不进行结果验证，就会产生更多幻觉和重复工作（来源：[概念页：幻觉与验证循环]）。有效的 harness 必须实现验证步骤，例如检查输出格式或对模型编写的代码运行测试用例（来源：同上）。**验证循环是区分脚手架与拐杖的关键：脚手架要求用户参与验证，拐杖则完全替代验证。**

## 诚实面对局限

必须承认，当前证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：[全局矛盾与存疑]）。个体差异极大：有些用户发现 Goblin Tools 的分解步骤过于机械，反而增加认知负荷；有些用户对 Motion 的自动排程感到失控。Brain.fm 针对 ADHD 的效果甚至缺乏独立临床研究支持（来源：[全局矛盾与存疑]）。这些工具不是万能药，它们只是外部执行功能层，效果取决于用户如何与它们互动。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个你拖延已久的任务**，但别只按步骤做——在每一步完成后，问自己“这一步真的完成了吗？”主动参与验证循环。
2. **用 Motion 规划明天的日程**，但故意留出两个小时的空白时间，看看 AI 如何调整，同时观察自己是否感到焦虑——这能帮你识别对工具的依赖程度。
3. **用 Saner.AI 记录一个你常忘记的信息**，比如某个项目的关键截止日期，然后连续三天不主动回忆，只依赖工具提醒——测试你的内在记忆是否开始退化。
4. **每周选一天，关闭所有 AI 工具，完全靠手动规划**——这是最直接的退出机制练习，帮你保持内在执行功能的活跃度。

脚手架的价值在于它最终可以被拆除。AI 工具作为外部执行功能层，只有在用户保持主动验证、定期退出练习的前提下，才能真正成为 ADHD 人群的脚手架，而非永久拐杖。对于工程师而言，设计 harness 时同样要确保验证循环是核心组件，而不是可选的附加功能。毕竟，无论是 ADHD 大脑还是 LLM agent，最危险的时刻不是出错，而是失去验证的意愿。

# 用 ChatGPT 给 ADHD 补上幻觉与验证循环，真的能像 agent 那样工作吗？

> 把 ChatGPT 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

## 问题：冲动下结论与幻觉，ADHD和LLM的同一个坑

你有没有过这种体验：别人话还没说完，你已经给出了答案；或者读到一个标题，脑子里立刻跳出一个结论，结果细看发现完全不是那么回事。这种“冲动下结论”是ADHD执行功能缺陷的典型表现——大脑的生成核心（发散联想、模式匹配）过于活跃，而负责验证和调度的执行层却常常掉线。

巧合的是，这正是LLM（大语言模型）的经典问题：它自信地编造（幻觉），靠验证与自我批判兜底。LLM本身是无状态、仅生成文本的核心，需要外部“脚手架”和“harness”来提供工具接口、上下文管理、安全执行等编排能力（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

那么，用ChatGPT给ADHD补上这个幻觉与验证循环，真的能像agent那样工作吗？

## 同构：生成核心与调度层的缺失

ADHD大脑与LLM在结构上同构——两者都是“高产但缺执行调度层的生成核心”。ADHD的核心困境并非智力不足，而是执行功能的失效，执行功能被描述为“大脑的驾驶座”，但对ADHD来说，“常常感觉方向盘后没有人”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。这种调度层的缺失导致计划难以执行：“计划本和便签用了一周就崩溃了”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

LLM一侧同样如此。现代AI编码代理通过“复合AI系统架构”来弥补这一缺陷，包括“工作负载专用模型路由、分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这些技术本质上是在LLM外部搭建调度层，防止“上下文膨胀和推理退化”。

## 人物案例：老子的“无为”harness

老子做了半辈子图书馆长，说走就走骑青牛出关，连书都不想留；思维极度发散非线性，《道德经》正言若反，没有系统逻辑；天生和人群有疏离感，“众人熙熙我独若遗”。他的专属harness是：以“无为”为核心，不强行做事，把所有能量收回来聚焦在道上；不著书不立说，不被名相束缚。

这对应LLM harness中的“自适应上下文压缩”和“惰性工具发现”——不强行塞入所有信息，而是让模型在必要时才调用工具。老子的“无为”不是不作为，而是通过外部系统（如“道”的框架）来避免过度生成的冲动。这与Agent harness中“分离规划与执行的双代理架构”异曲同工：一个代理负责生成（发散），另一个负责验证（收敛）。

## 工具实测：ChatGPT作为外部验证循环

目前市面上的AI工具已经在尝试扮演这一角色。例如Goblin Tools的Magic ToDo功能，能自动将任何任务分解为更小、更易管理的步骤（来源：AI Tools for Kids with ADHD: A Complete Guide for Parents...），用户可调节“辣度”以控制分解的粒度（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。这相当于在任务启动前加了一个“验证步骤”：你不需要一次性想清楚所有细节，AI帮你拆解并提醒你每一步是否可行。

Saner.AI则专注于知识回忆，通过本地记忆存储和快速检索减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。它被描述为“专为对抗执行功能障碍、任务瘫痪和认知超载而设计的全能ADHD友好型AI生产力和知识助手”（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。这相当于LLM侧的“Git仓库存储项目上下文”或“MCP连接器访问外部数据”——把工作记忆外化，让验证循环有据可依。

## 争议与局限：脚手架还是拐杖？

然而，这些工具真的能像agent harness那样可靠工作吗？存在几个关键争议：

1. **过度依赖风险**：AI工具可能成为永久拐杖而非可拆卸脚手架（来源：拐杖与脚手架）。如果用户永远依赖ChatGPT来验证每一个想法，执行功能可能进一步退化。
2. **个体差异**：AI辅助效果依赖个体差异，且缺乏长期验证（来源：任务启动）。部分用户反馈学习使用AI工具本身增加认知负荷（来源：矛盾与存疑）。
3. **同构的实证基础**：ADHD大脑与LLM同构的论点多为类比推理，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。
4. **工具本身的质量**：例如Motion缺乏独立临床证据，初始设置成本高（来源：矛盾与存疑）。

## 鲜明的核心观点

**ChatGPT作为ADHD的外部验证循环，其有效性取决于它被设计为“可拆卸的脚手架”而非“永久拐杖”。** 真正的agent harness工程强调“上下文交付、工具接口、规划工件、验证循环、记忆系统和沙箱”（来源：GitHub - ai-boost/awesome-harness-engineering），这些元素必须被内化到用户的日常流程中，而不是外包给AI。老子的“无为”harness之所以成功，是因为它最终内化为一种认知习惯，而非依赖外部工具。

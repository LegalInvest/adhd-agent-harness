# 为什么 AI 治好了我的拖延症，然后毁掉了我的时间感

> 这是一个应该被更多人讲出来的副作用报告。AI 工具全家桶上齐之后,我的拖延症确实肉眼可见地好转:任务有人拆、日程有人排、启动有人推——半年里的产出超过之前两年。然后某个周日的下午,我发现自己站在客厅里不知所措:没有块、没有提醒、没有下一步——**我不知道「现在」该是什么,甚至不太确定刚过去的这一周,是怎么过去的。** 治好拖延的那套系统,顺手把我的时间感收走了。

先说清这个副作用的机制,它不玄学。前面的文章讲过,ADHD 的时间感知本来就偏弱,external 工具的价值正是「代偿」:日历块代替流逝感,提醒代替前瞻,重排算法代替权衡。但代偿有个没写进说明书的深层效应——**当外部系统把时间的所有职能都接管后,你和时间的直接接触归零了**。以前的我虽然估不准时间,但好歹每天都在和它搏斗:感受截止日逼近的压力曲线、体会一下午溜走的懊恼——搏斗是痛苦的,但搏斗本身是一种接触。全面代偿之后,时间变成了「系统推给我的一系列当前块」,我执行块,块与块之间的那个连续的、流动的、属于我自己的时间,不见了。**一周变成了 40 个已完成块的列表——效率很高,记忆很薄。**

LLM 工程里有个现象可以精确对上:**全面工具化的 agent,会丧失对「过程」的内部表征**。每一步都调用外部工具的模型,其中间推理链会退化成工具调用的胶水——输出对,但模型「内部」对任务的整体理解变浅了;工程师发现这影响的不是当次表现,是**泛化**:换个没有工具的场景,它比从前更手足无措。代偿系统越完善,内部模型越萎缩——这是两边共享的定律。

但请注意,这篇文章的结论不是「扔掉工具,拥抱混乱」——回到裸奔的我只会重新拖延,什么也换不来。半年的修复期里,真正有效的是三个「再接触」设计:

**一、每天一次「手动收账」。** 睡前不看任何工具,凭记忆写三行:今天大概做了什么/哪段时间过得快/哪段慢。**这是逼内部时钟每天营业十分钟**——写错没关系(第二天可以和日历对答案,误差本身就是校准数据),重点是「对时间的主观叙事」这条肌肉恢复日常负重。

**二、每周留一个「无系统半天」。** 周日下午,关提醒、不看日历,只带一个模糊意图(「弄弄阳台」「随便走走」)——**让「现在做什么」重新由身体和情境决定一次**。头几周会不适(那个站在客厅的下午就是第一次),之后它会变成一周里时间感最厚的几个小时:原来「时间流逝」的感觉,需要空白才能显影。

**三、把工具的「时间数据」定期还给自己。** 每月看一次日历的月视图,但换一个问题:不问「完成了多少」,问**「这个月的形状是什么」**——哪周密哪周疏,什么占了大头,和我以为的一致吗?让外部系统的记录反哺内部叙事,而不是替代它。

## 边界

同构强度 C 级:如实标注——工具化对 agent 内部表征的影响是工程观察,认知卸载改变内部投入有实验支撑,但「时间感被系统接管」是第一人称经验报告,无对照研究,机制解释是合理推测。声明:ADHD 的时间感知弱是神经性的,本文的「再接触」练习不承诺修复它——目标只是防止「代偿之外的残余能力」也归零;若时间的断裂感伴随现实感丧失或情绪麻木,那可能不是工具副作用,请专业评估。

## 今天就能试的 3 件事

1. **今晚手动收账**:不看工具,凭记忆写今天的三行,明早对答案。
2. **本周日划出无系统半天**:关提醒,带一个模糊意图,忍过前四十分钟的不适。
3. **月底看一次「月的形状」**:不数产出,只看疏密,和记忆核对。

工具治好了我和任务的关系,代价是我和时间的关系——**这笔交易整体依然划算,但副作用不该瞒报**。代偿归代偿,接触归接触:每天十分钟手动收账,每周半天裸奔,时间感就不会在效率的高速路上,被彻底甩下车。

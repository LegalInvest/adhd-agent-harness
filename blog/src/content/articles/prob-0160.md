# 如果把 ADHD 大脑当成生成核心，Claude 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Claude 对 ADHD 的真正价值。

## 问题：为什么你总是“边做边忘”？

如果你是一名 ADHD 患者，你一定经历过这样的场景：打开电脑准备写报告，却突然想起要回一封邮件；点开邮件，又发现需要查一个数据；查数据时，被一条新闻吸引……半小时后，你完全忘了最初要写报告。这不是意志力问题——你的工作记忆容量太小，无法同时 hold 住目标、步骤和上下文（来源：ADHD 的 AI 工具全景）。

如果你是一名 Agent 工程师，你一定遇到过类似困境：给 LLM 一个复杂任务，它执行到一半就“忘记”了系统提示，开始偏离目标，甚至陷入死循环。你不得不给它加上外部记忆、事件驱动提醒和人在回路检查点（来源：重锚定与目标漂移）。LLM 本身是强大的生成核心，但无跨会话状态，必须靠外部记忆和脚手架“续命”。

这两类困境，本质上是同一个问题：**一个高产但缺执行调度层的生成核心，如何通过外部工具获得可靠执行？**

## 同构：ADHD 大脑 ≈ LLM，脚手架 ≈ Harness

wiki 资料中提出的核心命题是：ADHD 大脑与 LLM 在结构上同构——两者都是强大的生成核心（ADHD 的创造力、LLM 的文本生成），但都缺乏稳定的执行功能层（计划、工作记忆、抑制控制）（来源：ADHD 的 AI 工具全景）。ADHD 的“边做边忘”对应 LLM 的“指令消退”；ADHD 的“目标漂移”对应 Agent 的“任务偏离”。

那么，Claude 能否充当 ADHD 的“无状态生成核心”？答案是：**可以，但必须搭配外部记忆和脚手架。** Claude 本身就像 ADHD 大脑——它每次对话都是独立的，不记住之前的交互（除非你手动提供上下文）。这正是它的弱点，也是它的优势：无状态意味着每次都是全新开始，不会积累偏见或混乱；但你需要主动管理上下文。

## 证据：从工具看同构

在 ADHD 侧，工具如 **Saner.AI** 提供“知识回忆”和本地记忆，帮助用户快速找回信息，减少搜索循环（来源：Saner.AI）。**Mem** 提供持久记忆，将碎片化信息组织成可检索的知识库（来源：ADHD 的 AI 工具全景）。**Goblin Tools** 将复杂任务分解为可管理的小步骤，降低启动门槛（来源：Goblin Tools）。**Lex** 允许通过单一指令触发多步骤任务，减少决策负担（来源：Lex）。这些工具的共同点：它们都是**外部执行功能层**，补偿 ADHD 内在的工作记忆和任务调度缺陷。

在 LLM/Agent 侧，工程师为 LLM 搭建的 **Harness** 包含：外部记忆（向量数据库）、重锚定机制（事件驱动的系统提醒）、人在回路（人工检查点）、护栏（token 预算、工具调用上限）（来源：重锚定与目标漂移；人在回路与身体在场）。Harness 的结构与 ADHD 工具惊人相似：向量数据库 ≈ Mem 的持久记忆；系统提醒 ≈ Goblin Tools 的步骤提醒；人在回路 ≈ Focusmate 的身体搭档（来源：人在回路与身体在场）。

具体到 Claude：你可以把它当作一个“无状态生成核心”，但必须自己搭建脚手架。例如，使用 **Mem** 作为外部记忆，每次对话前自动注入相关上下文；使用 **Goblin Tools** 将任务分解为步骤，再逐一交给 Claude 执行；使用 **Otter.ai** 实时转录对话，将结果存入外部记忆（来源：ADHD 的 AI 工具全景）。这样，Claude 就不再是“另一个分心的工具”，而是你执行功能的一部分。

## 核心观点：脚手架，不是拐杖

这里必须诚实指出争议。wiki 资料中的「矛盾与存疑」反复强调：**过度依赖 AI 工具可能阻碍内在执行功能发展**（来源：矛盾与存疑）。如果 Claude 替你完成所有任务分解和记忆检索，你的大脑可能永远学不会这些技能。因此，关键不是“用 Claude 替代大脑”，而是**设计可逐步撤除的脚手架**。

例如，先用 Lex 的“单一指令触发多步骤”建立任务启动习惯，然后逐渐减少使用，直到你能够自己分解任务。先用 Mem 记录一切，然后逐步训练自己记忆关键信息。脚手架的目标是“教会你走路”，而不是“永远扶着你”。

## 今天就能试的行动

1. **搭建外部记忆**：注册 Mem 或 Saner.AI，将所有待办、笔记、灵感存入其中。每次与 Claude 对话前，先搜索相关记忆，粘贴到提示词中。这相当于给 Claude 一个“会话前上下文注入”。
2. **任务分解练习**：用 Goblin Tools 的 Magic ToDo 将“写报告”等大任务分解为 5-10 个小步骤。然后逐一交给 Claude 执行，每完成一步就记录到外部记忆。这相当于在 Claude 上实现“事件驱动的步骤执行”。
3. **设置重锚定**：在 Claude 对话中，每 5 轮交互后让 Claude 自动总结当前进度（可写一个系统提示）。这相当于 Agent Harness 中的“系统提醒”，防止目标漂移。
4. **尝试身体搭档**：使用 Focusmate 或类似工具，找一个虚拟搭档同时工作。同时打开 Claude，将搭档的反馈作为“人在回路”检查点。这模拟了 Harness 中的人工监督。

# Tiimo 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Tiimo：它到底补上了哪一层执行功能？

## 问题：为什么你的 Tiimo 提醒了，你还是没动？

如果你同时是 ADHD 患者和 Agent 工程师，你大概经历过两种相似的崩溃：Tiimo 弹窗提醒“该写报告了”，你点了“稍后”，然后忘到宇宙尽头；或者你给 LLM 配了 function calling 工具，它却反复调用同一个 API，死循环到超时。

问题出在哪？Tiimo 和 function calling 都是“外部调度器”——它们负责在正确的时间触发正确的动作。但调度器不等于执行器。ADHD 大脑和 LLM 一样，是“高产但缺执行调度层的生成核心”（来源：ADHD 大脑与 LLM 的同构），光给一个提醒，就像只给 LLM 一个工具列表却不告诉它何时用、怎么用。Tiimo 如果只做时间可视化而不提供任务分解、上下文恢复和启动缓冲，它就是一个漂亮的闹钟——不是脚手架。

## 同构：ADHD 大脑和 LLM 共享的“调度缺陷”

ADHD 一侧：执行功能缺陷包括工作记忆不稳定、任务启动困难和注意力分散。实证研究显示，ADHD 表现出“强记忆、弱控制”模式——工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这导致“想法很多，行动困难”。

LLM/Agent 一侧：LLM 本身是强大的生成核心，但缺乏可靠的内置调度与执行控制。构建 AI agent 需要“agent scaffolding”——围绕 LLM 构建的软件架构和工具，使其能执行复杂、目标驱动的任务（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。用经典 Stroop 冲突任务测试 Transformer 注意力，发现随序列变长，模型在冲突试次上骤降到随机水平——无法抑制优势反应，这与 ADHD 执行功能缺陷的核心标志一一对应（来源：同上）。

两边都需要一个“外部执行功能层”：对 ADHD 是任务分解、上下文记忆和启动提示；对 LLM 是 function calling、记忆系统和规划循环。

## Tiimo 补上了哪一层？——时间感知层，而非执行层

Tiimo 的核心价值在于补偿 ADHD 的“时间盲”——通过视觉化时间块，让抽象的时间变得可感知。这相当于给 LLM 加了一个“时间戳”和“截止日期”的 function。但光有时间感知，没有任务分解和上下文恢复，就像 LLM 知道了“现在该调用天气 API”，却没有拿到城市参数——它知道要做什么，但做不了。

对比其他工具：Goblin Tools 的 Magic ToDo 能把“清理厨房”分解成 12 步（来源：Goblin Tools），直接降低启动门槛；Saner.AI 通过本地记忆减少搜索循环，补偿工作记忆（来源：Saner.AI）；Lex 允许“单一指令触发多步骤任务”，类似 agent 的规划机制（来源：Lex）。Tiimo 不提供这些——它假设用户已经知道步骤，只是需要时间提醒。但 ADHD 的典型困境是“知道该做什么，但不知道第一步怎么迈”。

## 历史案例：孔子和于谦的“外部调度层”

孔子：身高 1 米 9，精力旺盛，周游列国 14 年坐不住；冲动爱骂人，对音乐超专注到三月不知肉味，对种地等俗事零耐心；《论语》全是场景化语录，无系统著作——典型的“高产生成核心”。他的 harness 是“克己复礼”和“吾日三省吾身”：用外在的“礼”作为行为边界，每日反省作为定时 re-grounding。这对应 LLM harness 中的“系统提示+日志回看”——不是只给一个提醒（“你要克己”），而是提供了行为准则（系统提示）和每日复盘（日志循环）。

于谦：性格刚直，冲动敢说话，土木堡之变后力排众议坚守北京。他的 harness 是“儒家气节+严格操练”，相当于 agent 的“目标函数+训练数据”。他写《石灰吟》作为自我提示，类似 LLM 的“人格设定”。

这两个案例说明：有效的 harness 必须包含**行为准则、上下文恢复和反馈循环**，而不仅仅是时间提醒。

## 争议与局限：Tiimo 是脚手架还是拐杖？

“拐杖与脚手架”概念提醒我们：过度依赖工具会削弱内在执行功能（来源：矛盾与存疑）。Tiimo 如果只让你依赖外部时间提醒，而不训练你的时间感知能力，它就从脚手架变成了拐杖。同样，给 LLM 过度配置 function calling，也会让模型失去自主规划能力，变成只会调用工具的“工具人”。

另一个局限：Tiimo 缺乏个性化调整。ADHD 亚型（注意力缺陷型 vs 冲动型）对时间管理的需求不同，但 Tiimo 是通用设计（来源：矛盾与存疑）。此外，工具宣称的证据不足——多数 ADHD 工具缺乏严格临床验证，效果多基于用户报告（来源：矛盾与存疑）。

## 今天就能试的 3 件事

1. **给 Tiimo 加一个“启动步骤”**：在每个时间块开始时，用 Goblin Tools 或手写把第一个子任务写出来。不要只设“工作 25 分钟”，设“工作 25 分钟：打开文档，写第一段”。
2. **建立“上下文恢复”习惯**：在 Tiimo 提醒后，先花 30 秒回顾上次中断的位置（类似 agent 的“状态恢复”）。可以用 Mem 或 Otter.ai 记录上下文（来源：Mem、Otter.ai）。
3. **设计个人“re-grounding”仪式**：像孔子的“三省吾身”一样，每天固定时间（如 Tiimo 的“晚间回顾”块）问自己三个问题：今天完成了什么？哪个环节卡住了？明天第一步是什么？

Tiimo 是优秀的“时间感知层”，但别只依赖它。真正的执行功能脚手架，需要时间感知+任务分解+上下文恢复+反馈循环——就像好的 agent 系统需要 function calling+记忆+规划+日志。别把调度器当成执行器。

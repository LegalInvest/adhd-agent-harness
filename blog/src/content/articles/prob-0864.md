# 用 ChatGPT 给 ADHD 补上规划循环与任务分解，真的能像 agent 那样工作吗？

> 把 ChatGPT 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

## 问题：任务启动与分解的双重困境

对于 ADHD 人群，任务启动困难与不会拆解是两大核心痛点。面对一个复杂项目，大脑像一台高性能但缺驱动的引擎——想法源源不断，却无法将“写一份报告”分解为“打开文档→列出大纲→填充第一节”等可执行步骤。这种“生成核心强，执行调度弱”的状态，与当前 LLM/agent 系统的架构惊人地同构：LLM 擅长生成文本，但执行长任务时同样需要 planner-executor 循环来分解和调度。

## 同构：ADHD 大脑与 LLM 的共享短板

ADHD 大脑的智力与创造力（生成核心）与 LLM 的生成能力类似，但缺乏执行功能（调度层）来组织行动。正如 wiki 资料指出的，AI 工具如 Motion 和 Reclaim.ai 自动规划日程，相当于提供外部调度层（来源：AI 与 ADHD 的时间管理）。同样，LLM 的无状态性（缺乏持久记忆）与 ADHD 的工作记忆缺陷同构，Todoist 等工具通过智能提醒充当数字工作记忆（来源：同构视角）。这种结构上的对应关系，使得用 ChatGPT 来辅助 ADHD 的任务分解成为一种自然的尝试。

## 实测：ChatGPT 能补哪块短板？

将 ChatGPT 作为外部执行层，核心是让它扮演 planner 角色。用户给出目标（如“整理书房”），ChatGPT 生成步骤序列（“步骤1：清空桌面；步骤2：分类物品；步骤3：归位”）。这直接针对 ADHD 的“不会拆解”问题——通过外部语言提示绕过执行功能瓶颈。然而，这仅仅是 planner 功能。真正的 agent 系统还需要 executor 循环：执行、反馈、调整。ChatGPT 缺乏对现实时间的感知，无法像 Motion 那样自动重新安排被打乱的日程（来源：Motion）。它也缺少持久记忆，无法记住用户上一轮的分解偏好，每次对话都是“冷启动”。因此，ChatGPT 补的是“分解”这一块，但“调度”和“自适应”仍需其他工具或用户手动完成。

## 证据：工具侧与大脑侧的真实支持

ADHD 侧：多项资料强调视觉化时间线对时间盲的作用，如 Tiimo 将时间转化为可见元素（来源：Tiimo）；身体在场效应通过社会存在感降低分心阈值，Focusmate 利用算法匹配虚拟伙伴（来源：身体在场效应）。这些工具直接补偿执行功能缺陷。LLM/agent 侧：Motion 的自动排程与动态调整，Reclaim.ai 的防御性时间块，都体现了 planner-executor 循环的工程实现（来源：Motion；Reclaim.ai）。两边的证据都指向同一个方向：外部结构化支持能显著提升任务执行能力。

## 争议与局限：脚手架还是拐杖？

核心争议在于：这种外部支持是促进能力发展的“脚手架”，还是阻碍独立性的“拐杖”？wiki 资料指出，多数工具如 Goblin Tools 设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。过度依赖可能阻碍内在执行功能的发展。此外，多巴胺干预的有效性存在争议，部分工具声称通过神经科学原理改善执行功能，但缺乏独立验证（来源：矛盾与存疑）。对于 ChatGPT 而言，它无法提供多巴胺奖励反馈，也无法像 Focusmate 那样提供隐性问责，因此其效果更依赖用户主动使用。

## 核心观点：ChatGPT 是临时 planner，不是完整 agent

我的判断是：ChatGPT 可以作为 ADHD 任务分解的“临时 planner”，但缺乏 executor 循环和持久记忆，无法替代 Motion 等专业工具。它更像一个可对话的“步骤生成器”，适合在任务启动时快速获得分解方案，但后续执行仍需用户自己或结合其他工具。对于工程师而言，这提示了设计 agent 系统的关键：planner 必须与 executor 紧密耦合，并具备记忆和自适应能力。

## 今天就能试的行动

1. **用 ChatGPT 做任务分解**：给出一个让你拖延的任务，要求它列出 5-7 个具体步骤，并设定每个步骤的预估时间。
2. **结合视觉化工具**：将 ChatGPT 生成的步骤输入 Tiimo 或 Structured，转化为可视时间线。
3. **尝试虚拟身体加倍**：使用 Focusmate 或找朋友视频连线，在 ChatGPT 生成的步骤指导下执行。
4. **记录反馈**：每次使用后，记录 ChatGPT 的分解是否合理，以及实际执行中的障碍，逐步优化提示词。

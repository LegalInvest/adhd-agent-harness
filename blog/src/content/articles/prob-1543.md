# Habitica 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Habitica：它到底补上了哪一层执行功能？

你打开 Habitica，给自己建了一个像素小人，设定每日任务：写 500 字、运动 20 分钟、吃药。第一天，你兴奋地勾掉任务，听到金币落袋的叮当声。第三天，你忘了登录。第五天，你发现小人已经死掉，而你连打开 App 的冲动都没了。

这不是你的错。Habitica 被设计成游戏化奖励系统，但 ADHD 大脑对延迟奖励不敏感（来源：多巴胺），游戏化带来的多巴胺峰值会迅速衰减。真正的问题不是缺乏动机，而是 **目标漂移**——你开始做一件事，然后被分心或超聚焦带跑，最终忘了最初要做什么。

这正是 LLM/Agent 工程中同样面临的挑战：长程任务里，agent 会逐渐偏离原始指令，产生幻觉或重复循环。工程师的解法是 **重锚定 system prompt**——在特定间隔或检测到漂移时，重新注入核心目标。Habitica 对 ADHD 的价值，不是游戏化奖励，而是它提供了一个外部调度层来对抗目标漂移。

## 同构：ADHD 大脑与 LLM 共享的「弱控制」剖面

明尼苏达大学系统测试 LLM 的执行功能，发现“强记忆、弱控制”剖面——工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷，无法灵活切换任务集、无法抑制自动化反应（来源：工作记忆）。这正是 ADHD 的经典神经心理模式。耶鲁大学进一步发现，LLM 的自注意力机制本身导致工作记忆容量限制：随任务难度增加，注意力分数熵增、注意力弥散，与人类注意分散机制同源（来源：工作记忆）。

换句话说，ADHD 大脑和 LLM 都是 **高产但缺执行调度层的生成核心**。它们不缺算力（记忆容量），缺的是 orchestration 层——那个决定“现在该注意什么”的调度器。

## 外部脚手架的真实案例：曾国藩与 Focusmate

曾国藩是典型的注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书“疾读不能终一行”。他的解决方案是 **日课十二条**：黎明即起、读书不二、谨言、写日记反省。这套系统本质上是一个外部调度层——每天固定时间重锚定目标，用“结硬寨打呆仗”的笨办法抵消冲动。

这与 LLM/Agent 的 harness 同构：日课十二条相当于定时 re-grounding 的 system prompt，写日记反省相当于事后审计日志。曾国藩的成就（平定太平天国、发起洋务运动）证明，外部脚手架可以补偿执行功能缺陷，而不需要“治愈” ADHD。

现代工具中，Focusmate 是同样的逻辑：通过虚拟身体在场，提供外部启动信号和实时问责（来源：Focusmate）。它不是靠多巴胺奖励，而是靠 **重锚定**——当你和另一个活人视频连线时，你的注意力被强制拉回任务。这与 LLM 工程中“人在回路”的 human-in-the-loop 机制完全同构。

## 脚手架 vs 拐杖：Habitica 用错了什么

Habitica 的问题在于，它把游戏化奖励作为主要机制，而 ADHD 用户对延迟奖励不敏感，多巴胺峰值会快速消退。结果，它变成了一根 **拐杖**——你需要不断升级装备、获得金币来维持动力，一旦奖励失效，整个系统崩塌。

真正的 **脚手架** 应该像 RescueTime：自动化记录时间去向，揭示注意力漂移模式，而不依赖你的意志力去手动追踪（来源：RescueTime）。或者像 Brain.fm：通过神经锁相技术直接调节大脑状态，降低进入专注的门槛（来源：Brain.fm）。这些工具的核心不是提供多巴胺，而是 **减少执行功能的消耗**。

## 争议与局限

同构视角虽然有力，但缺乏实证基础——目前仅是理论类比（来源：矛盾与存疑）。ADHD 大脑与 LLM 的相似性可能止于功能层面，而非神经机制。此外，AI 工具的依赖风险真实存在：长期使用外部脚手架，是否会导致执行功能进一步退化？（来源：矛盾与存疑）目前没有长期追踪研究能回答这个问题。

## 今天就能试的行动

1. **用 Focusmate 替代 Habitica 的任务列表**：预约一个 25 分钟的时段，只做一件事。感受外部身体在场如何帮你启动，而不是靠游戏化奖励。
2. **设置定时重锚定**：在手机上每 30 分钟设一个闹钟，提醒自己“当前目标是什么”——这是 LLM 工程中重锚定 system prompt 的简化版。
3. **用 RescueTime 记录一天**：不看任何评价，只观察自己在哪里漂移。识别出 1-2 个高频分心源，明天主动阻断它们。
4. **写一条极简日课**：模仿曾国藩，只写一条必须完成的“读书不二”类任务，坚持 3 天。重点不是完成，而是观察“重锚定”带来的注意力回正。

# ADHD 学生如何用「给生成核心套 harness」的思路应对考试与作业里的注意力容易被环境带偏？

> 把 LLM 工程里的「LLM 受上下文窗口内容左右，需要上下文工程」迁移到ADHD 学生的考试与作业场景，双域证据作答。

你坐在书桌前，翻开课本，准备复习明天的考试。手机亮了，你拿起它；窗外有声音，你抬头；笔在手上转了三圈，你想起昨晚的短视频。十五分钟后，你还在第一页。这不是意志力问题——这是你的大脑和 LLM 一样，是一个高产但缺执行调度层的生成核心。

## 问题：注意力容易被环境带偏的底层机制

ADHD 学生最熟悉的痛苦是：明明想专注，但环境里任何一丝变化都能劫持注意力。这不是“不努力”，而是执行功能缺陷——工作记忆容量小、时间盲、任务启动困难——导致大脑无法稳定地维持一个目标上下文。

在 LLM 工程里，这个问题有个对应的名字：**上下文窗口内容左右**。LLM 天然会受 prompt 里所有信息的影响，包括无关的、过时的、甚至冲突的内容。如果没有精心设计的上下文工程（context engineering），LLM 的输出就会跑偏，产生幻觉或重复工作。

两者的结构完全同构：ADHD 大脑和 LLM 都是高产的生成核心（能产生大量想法、联想、创意），但都缺乏一个可靠的调度层来过滤噪声、维持主线、验证输出。

## 答案：给生成核心套 harness

在 LLM 工程中，解决上下文漂移的方法是 **harness（编排层/脚手架）**。一个成熟的 agent harness 会包含：上下文传递、工具接口、规划工件、验证循环、记忆系统和沙盒（来源：GitHub - ai-boost/awesome-harness-engineering）。它不改变 LLM 本身，而是通过外部结构补偿其无状态和易受干扰的缺陷。

同样，ADHD 学生需要的不是“训练专注力”，而是给自己套一个 **外部执行功能层**——通过工具和策略来补偿工作记忆、任务启动、时间盲和上下文保持的缺陷（来源：AI 与 ADHD 的专注力）。

### 证据 1：Brain.fm 作为神经锁相 harness

Brain.fm 利用 AI 生成音乐，通过神经锁相技术影响大脑信息处理方式，帮助用户进入专注状态（来源：Brain.fm）。这相当于给大脑提供一个稳定的“音频上下文”，主动维持任务连贯性，类似 LLM 工程中用 prompt 模板固定输出风格。

但注意：目前缺乏针对 ADHD 的独立临床研究（来源：Brain.fm），效果可能因人而异。这不是万能药，而是一个可尝试的脚手架。

### 证据 2：Focusmate 作为身体在场 harness

Focusmate 通过 AI 匹配虚拟同伴，提供实时问责（来源：Focusmate）。其核心机制是“身体在场效应”——他人的存在会激活社会监控，弥补内在的多巴胺调节不足。在 LLM 工程中，这相当于一个“验证循环”：系统通过外部检查点（如测试用例）来确保输出不跑偏。Focusmate 的实时视频就是那个外部验证者。

同样，Focusmate 依赖配对伙伴的可靠性，且对社交焦虑用户可能造成压力（来源：Focusmate）。它不提供任务分解功能，只负责“摁住你”。

### 证据 3：RescueTime 作为外部记忆 harness

RescueTime 自动记录时间使用，生成可视化报告（来源：RescueTime）。这对 ADHD 的“时间盲”和工作记忆缺陷是直接补偿——你不需要记住自己做了什么，工具替你记。在 LLM 工程中，这对应“记忆系统”：harness 维护一个外部记忆，避免 LLM 因无状态而遗忘上下文。

局限是：自动分类可能不精准，需要用户后期调整，这对执行功能弱的用户仍是挑战（来源：RescueTime）。

## 核心判断：脚手架，不是拐杖

本文的核心观点是：**ADHD 学生应该主动采用“给生成核心套 harness”的思路，但必须区分脚手架与拐杖**。脚手架是设计为可逐步撤除的临时支撑，拐杖则是永久替代。

现有工具大多设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这不是工具的问题，而是使用策略的问题。你应该把工具当作训练辅助，而非终身依赖。例如：先用 Focusmate 建立任务启动习惯，然后逐渐减少频率；先用 RescueTime 觉察时间模式，然后内化时间感知。

另一个争议是多巴胺干预的有效性：基于多巴胺的 AI 干预（如游戏化奖励）是否真正改善专注，证据矛盾（来源：矛盾与存疑）。因此，本文推荐的工具都是基于执行功能补偿而非多巴胺调节，证据更扎实。

## 今天就能试的行动

1. **设置一个“专注上下文”**：用 Brain.fm 或类似工具（如 Endel）创建一个固定的音频环境。每次学习前先播放 5 分钟，让大脑进入状态。这相当于给 LLM 一个稳定的 system prompt。

2. **找一个身体在场伙伴**：用 Focusmate 或直接约朋友视频学习。关键不是聊，而是“有人看着你”。这相当于给任务加一个外部验证循环。

3. **自动追踪时间**：安装 RescueTime 或类似工具（如 Toggl Track），但不要追求完美分类。只关注每日总专注时长，减少分析负担。这相当于给大脑一个外部记忆。

4. **写一个“上下文工程”清单**：每次学习前，写下三个问题：我现在要做什么？我周围有什么干扰？我如何排除它们？这相当于 LLM 工程里的规划工件。

## 局限与诚实

本文的论证基于概念同构和工具机制对齐，而非大规模临床研究（来源：AI 与 ADHD 的专注力）。ADHD 亚型（注意力缺陷 vs 多动冲动）对工具反应可能不同，现有工具未充分个性化（来源：矛盾与存疑）。此外，长期依赖脚手架是否阻碍内在执行功能发展，目前无研究结论（来源：矛盾与存疑）。

但至少，这个思路给了你一个可操作的框架：你不是坏掉的 LLM，你只是缺一个 harness。现在，去给自己套一个。

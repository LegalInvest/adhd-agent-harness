# Habitica 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Habitica：它到底补上了哪一层执行功能？

如果你是一个 ADHD 患者，你大概率经历过这样的场景：打开 Habitica，兴奋地创建了一个“每天写 500 字”的每日任务，设好奖励，然后……第二天就忘了。一周后，你的小宠物饿死了，你感到愧疚，然后卸载了 App。

如果你是一个在做 Agentic Harness 的工程师，你大概率也见过这样的场景：给 LLM 接上 20 个 function calling 工具，结果 agent 在调用天气 API 和计算器之间反复横跳，最后输出了一篇充满幻觉的“天气与数学关系论文”。

这两个场景共享同一个结构性问题：**工具被当成了魔法拐杖，而不是脚手架**。

## 同构：ADHD 大脑与 LLM 都缺一个调度层

ADHD 大脑与 LLM 在结构上高度同构：两者都是强大的生成核心，但缺乏可靠的内置执行调度层（来源：ADHD 大脑与 LLM 的同构）。ADHD 患者不擅长精确、重复、状态维护类任务——比如按时吃药、记账、回复邮件——因为这些任务依赖稳定的工作记忆和任务启动能力，而这两者正是 ADHD 的执行功能缺陷所在。LLM 同样不擅长精确计算和状态跟踪，所以需要用 function calling 把“3.14 × 2.71”外包给计算器，把“当前时间”外包给时钟 API。

Habitica 试图扮演的就是这个外部调度层：它通过游戏化（多巴胺机制）提供即时反馈，通过每日任务和待办清单补偿工作记忆不足，通过公会和挑战引入身体在场效应（来源：身体在场效应）。但问题在于，很多 ADHD 用户把 Habitica 当成了“任务执行者”，而不是“任务调度器”。他们期待 Habitica 自动完成事情，就像期待 LLM 直接输出 3.14 × 2.71 的精确结果——这注定失败。

## 历史案例：孔子和王阳明的“脚手架”

孔子和王阳明都是典型的 ADHD 大脑，但他们都构建了精妙的个人 harness 系统。

孔子精力旺盛、冲动、思维跳跃，《论语》全是场景化语录，无系统著作。他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。这相当于一个**定时 re-grounding 的调度器**：每天固定时间检查自己是否偏离轨道，类似 agent 的“定期状态同步”机制。

王阳明兴趣爆发式转移，格竹子格到吐血，冲动上书骂刘瑾被贬。他的 harness 是“致良知”和“事上练”——静坐反省，用良知作为决策标准，在行动中磨练。这相当于**一个内置的优先级过滤器**：无论外部任务多杂乱，都先回到“良知”这个核心原则再行动，类似 LLM 的“system prompt”在每次调用前重新锚定行为准则。

两人都没有依赖“工具”本身，而是把规则内化为脚手架。这与 Habitica 的正确用法一致：工具只是提醒和记录，真正执行任务的还是你自己。

## 正确的用法：脚手架 vs 拐杖

那么，Habitica 的正确用法是什么？答案是：**把它当作 function calling 的工具箱，而不是 LLM 本身**。

- **任务分解**：像 Goblin Tools 的 Magic ToDo 一样，把“写论文”分解为“打开文档→写第一段→写第二段→休息 5 分钟”（来源：Goblin Tools）。Habitica 的 checklist 功能可以做到这一点，但很多人只设了一个“写论文”的每日任务，然后因为太大而拖延。
- **身体在场**：像 Focusmate 那样，用 Habitica 的公会或挑战引入社会问责（来源：身体在场效应）。定期检查（比如每天打卡）可将目标达成可能性从 25% 提升至 95%（来源：ADHD Task Managers That Work: Top AI Tools 2025）。
- **降低启动门槛**：像 Lex 的“单一指令触发多步骤”一样，把 Habitica 的每日任务设成“只做第一步”（比如“穿上跑鞋”），而不是“跑 5 公里”。

而错误的用法是：
- 设了 20 个每日任务，期望自己全部完成（认知超载）。
- 把奖励设成“玩手机 30 分钟”，结果奖励本身成了分心源。
- 忘记打卡后感到愧疚，然后放弃整个系统——这就像 LLM 调用 API 失败后直接输出“我不知道”而不是重试。

## 局限与争议

必须诚实指出，Habitica 并非万能。

第一，**依赖风险**：过度依赖游戏化可能削弱内在动机，变成“为了升级而完成任务”，而非真正完成任务（来源：拐杖与脚手架）。第二，**证据不足**：目前缺乏严格临床验证，效果多基于用户报告（来源：矛盾与存疑）。第三，**通用性差**：Habitica 的游戏化设计对某些 ADHD 亚型（如注意力分散型）可能无效，甚至增加分心。

同构类比也有边界：ADHD 大脑不是 LLM，工具不能完全替代人的决策。孔子 70 岁才做到“从心所欲不逾矩”，说明脚手架需要长期内化。

## 今天就能试的行动

1. **只设 3 个每日任务**：用 checklist 把每个任务分解为 3 步，完成一步就打勾。
2. **加入一个公会**：找一个“早起打卡”或“专注工作”的公会，利用身体在场效应。
3. **把奖励设为“非屏幕活动”**：比如“散步 10 分钟”或“吃一块巧克力”，避免数字奖励导致分心。
4. **每周回顾一次**：像孔子的“三省吾身”一样，每周检查自己的 Habitica 使用情况，调整任务难度。

Habitica 不是魔法，它只是一个外部调度器。真正的执行者，还是你——那个强大的生成核心。

# Mem 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> function calling 那篇讲了 Mem 当检索工具:写入零组织,检索靠问。这一篇讲它的另一个身份。笔记系统里有两类东西,性质完全不同:**流水**(会议记录、灵感、摘录——一次性写入,偶尔被检索)和**纲领**(我的运行原则、项目的核心决策、「这件事我们说好了不再改」——需要反复被看见、持续生效)。Mem 的语义检索对付流水是利器,但纲领类内容有完全不同的需求——**它不该等着被检索,它该主动出现**。用检索的逻辑管理纲领,是这类工具最典型的用错方式,对应 system prompt 工程里一条铁律:**system prompt 和 RAG 是两个组件,混用必翻车。**

先把这条工程铁律说清。LLM 系统里,**该进 system prompt 的**:少量、稳定、每轮都必须生效的核心指令;**该进 RAG(检索)的**:大量、动态、按需取用的资料。两种错法都有名:把海量资料塞进 system prompt(上下文爆炸,核心指令被稀释);把核心指令扔进检索库(模型「不知道自己该知道」——指令只有被碰巧检索到才生效,等于随机遵循)。人类版的后一种错法,就是 Mem 用户的高发病:**把「本季度的三条原则」「和自己约定好的边界」写成普通笔记,扔进几千条流水里**——然后它们只在你偶然搜到时闪现一次,其余时间沉默地失效着。误用三种:

**误用一:纲领混进流水(核心指令进了检索库)。** 症状如上。修法是分层:**给纲领类内容一个物理上的特殊待遇**——置顶、钉选、或建一个「宪法」集合,并给它装注入通道(设为打开 App 的第一屏、绑进晨间例程)。判据很简单:**这条笔记如果三周不被看见就会出问题,它就是纲领,必须离开流水区。** 数量纪律同样适用:宪法区超过一页,就该发生弹劾。

**误用二:流水升格过度(什么都想常驻)。** 反向的错:觉得每条笔记都重要,置顶了三十条——**注入区变成了新的流水区,稀释重演**(prompt bloat 的笔记版)。ADHD 对这个错有额外的易感性:收藏癖+丢失恐惧,让「万一以后要用」的东西不断向常驻区迁移。修法:信任检索——**Mem 的语义搜索就是为「以后要用」设计的,流水的归宿是可检索,不是可看见**;常驻区的每个位置都要用「三周不见会出事吗」重新答辩。

**误用三:纲领常驻了,但从不更新(化石宪法)。** 置顶笔记写于四个月前,里面的「本季度重心」早就过期——**每天注入一份过时的 prompt,比不注入更糟**(它在持续给旧方向背书,见 stale prompt 篇)。Mem 的 AI 助手在这里有个巧用:**定期让它对照「宪法区」和最近两周的流水,问它「我最近的记录和我的原则有什么矛盾?」**——让检索层反过来审计注入层,两个组件各司其职之后,还能互相纠察。这是笔记工具里少有的、真正配得上「AI 第二大脑」名号的用法。

## 边界

同构强度 B 级:system prompt 与检索的分层是真实的 LLM 架构原则,ADHD 的「重要事项需要外部持续可见」有提醒依赖的文献基础,Mem 本身无对照研究,「宪法区/流水区」是行为设计。声明:分层管理的是信息,不是动机——原则天天被看见但天天被违反,问题在原则的可行性或状态支持(见相应篇章);另外纲领里若涉及健康决策(用药、治疗边界),更新它们请和专业者同步,别只和笔记软件商量。

## 今天就能试的 3 件事

1. **建立宪法区**:翻出那几条散落在流水里的原则笔记,置顶,总量一页以内。
2. **给它装注入通道**:设为打开笔记的第一眼,或绑进明早的例程。
3. **让 AI 做一次审计**:问它「我最近两周的笔记和我的置顶原则有什么矛盾」——看看检索层能揪出什么。

笔记软件不会告诉你哪条笔记是宪法、哪条是流水——**这个分层只能你来做:少数几条待遇特殊到天天见面,其余几千条安心等着被搜索**。分对了层,一个 App 里就同时住着你的记忆和你的方向。

# 如果把 ADHD 大脑当成生成核心，Inflow 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Inflow 对 ADHD 的真正价值。

## 同一个问题，两种不同的“核心”

你刚坐下要交报告，突然想起快递没取；打开取件码，又发现群里有人@你；回完消息，报告窗口还停留在第 7 行。这是 ADHD 人群熟悉的“边做边忘”——工作记忆容量小，执行调度层随时掉线。

LLM 工程师可能也在另一个屏幕前骂街：模型对话一刷新，前因后果全丢，只能靠 RAG、向量库、持久化记忆层把状态续上。两个场景看似无关，但结构是同一个：**生成核心很强，跨上下文状态很弱**。

这正是 wiki 资料里反复提到的“同构视角”：ADHD 大脑与 LLM 都是高产生成核心，但缺乏可靠的内置执行调度层，需要外部脚手架（harness）来补偿工作记忆不稳定、任务启动困难和注意力漂移。（来源：ADHD 的 AI 工具全景）

那么，Inflow 这类 ADHD 工具，能不能被理解成“给生成核心接的外部记忆与状态层”？答案是：可以，但要把它的价值看清楚，别把它当成药。

## ADHD 侧：工作记忆掉线，靠外部记忆续命

ADHD 的执行功能缺陷不是“懒”，而是大脑自带的缓存太小。任务分解、规划循环、时间感知、抑制控制这些环节都容易崩。传统静态工具（纸质计划、便签）无法适应动态行为，导致频繁放弃。（来源：规划循环与任务分解）

所以现代 ADHD 工具本质上做的是同一件事：把“我脑子里装不下的东西”外化。

- **Goblin Tools** 的 Magic ToDo 把模糊任务（如“清理厨房”）拆成可执行子任务，用“辣度”调节粒度，降低启动门槛。（来源：Goblin Tools）
- **Saner.AI** 通过通用收件箱自动从邮件、文档、日历中提取待办，减少搜索循环和标签切换。（来源：Saner.AI）
- **Lex** 用“单一指令触发多步骤任务”，把复杂目标串成自动执行链，降低工作记忆负荷。（来源：Lex）
- **Mem** 被定位为“AI 第二大脑”，通过持久记忆补偿工作记忆不足。（来源：ADHD 的 AI 工具全景）

这些工具不是在“修复”大脑，而是在大脑外面建了一个存储与调度层。这和 LLM 的 agentic harness 做的是同一类工程。

## LLM/Agent 侧：模型无状态，靠外部记忆续命

LLM 的核心能力也是生成，但它没有跨会话状态。你关一次窗口，它就“失忆”。所以 agent 工程要围绕它搭一套外部架构：记忆模块（向量库、图数据库）、工具调用、规划循环、重锚定机制。

这个结构与 ADHD 工具的补偿逻辑几乎一一对应：

| ADHD 痛点 | ADHD 工具做法 | LLM/Agent 对应 |
|---|---|---|
| 工作记忆小，边做边忘 | Mem、Saner.AI 外部记忆 | 向量库、持久化记忆层 |
| 任务启动困难 | Goblin Tools 任务分解 | Agent 的任务拆解与规划循环 |
| 注意力漂移 | 提醒、专注计时器 | 重锚定机制、re-grounding |
| 时间盲 | Motion、Reclaim.ai 动态排程 | 调度器、执行队列 |

wiki 资料明确把这个对应关系写出来：为 LLM 搭建脚手架与为 ADHD 搭建执行功能支架是同一类工程。（来源：ADHD 的 AI 工具全景）

## 古人早就在用“外部 harness”：孔子的案例

这个同构不是今天才有的。两千多年前，孔子就在给自己的生成核心搭 harness。

他的 ADHD 特质在资料里写得很清楚：一米九的身高，精力旺盛，周游列国 14 年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予“朽木不可雕”；对音乐超专注到“三月不知肉味”，对种地却零耐心；思维跳跃，《论语》全是场景化语录，没有系统著作。（来源：人物案例）

孔子给自己的 harness 是外在的“礼”：以“克己复礼”为行为边界，每日三省吾身，用仪式和反省把漂移的注意力重新锚定回目标。他直到 70 岁才做到“从心所欲不逾矩”，说明这套系统不是在消灭冲动，而是持续地承接和重定向冲动。（来源：人物案例）

对应到 LLM/agent 工程里，这相当于：

- **“礼” ↔ 系统 prompt / 行为约束层**：给模型设定不可逾越的边界；
- **“日省” ↔ 定时 re-grounding / 反思循环**：让模型周期性地回顾目标，纠偏漂移；
- **周游列国的奔波 ↔ 高环境熵下的执行调度**：靠外部结构维持行动的连贯性。

孔子的例子说明：同构的解法自古皆然，只是今天换成了代码和通知推送。

## 所以 Inflow 到底是什么？

如果 Inflow 属于 ADHD 工具生态中强调“外部记忆 + 重锚定 + 规划循环”的那一类，那么它的价值就可以用 agent 工程的语言重新表述：

> Inflow 不是“治疗 ADHD 的药”，而是给无状态的生成核心（ADHD 大脑）接上了一层持久状态层与轻量调度器。

它帮你把“今天要做什么”“为什么做”“做到哪一步了”这些状态外化，避免工作记忆在任务切换中清空。它提供提醒和重锚定，对抗 ADHD 的目标漂移；它通过任务分解降低启动门槛，这和 LLM agent 的 planner 没有本质区别。（来源：重锚定与目标漂移）

但这里有一个关键边界：好的 harness 是**脚手架**，不是**拐杖**。脚手架的目标是让你最终能自己爬上去；拐杖是让你永远靠它站着。wiki 资料也明确警告：AI 工具可能从脚手架变为永久拐杖，削弱内在执行功能。（来源：ADHD 的 AI 工具全景）

## 诚实说：同构是启发，不是科学定律

必须指出局限。

wiki 资料中的“矛盾与存疑”部分明确说：**ADHD 大脑与 LLM 在结构上同构这一说法缺乏实证基础，目前仅是一种理论类比，并非经过验证的科学事实**。不同页面在表述时有时把它当既定事实，有时当假设，存在不一致。（来源：全局矛盾与存疑）

同样，多数 ADHD AI 工具缺乏严格临床验证，效果多基于用户报告；工具本身也可能增加认知负担或分散注意力（比如 ChatGPT 的通用性反而带来分心风险）；Mem 的 AI 工作记忆限制与 ADHD 用户需求也存在矛盾。（来源：ADHD 的 AI 工具全景、全局矛盾与存疑）

所以我的判断是：同构视角是一个**非常有用的工程启发**，它让 ADHD 人群和 LLM 工程师都能用同一种语言理解“外部记忆与调度”的必要性，但它不能替代神经科学证据或临床验证。

## 今天就能试的 4 个行动

1. **给大脑接一个“最小外部记忆层”**：挑一个工具（如 Mem 或 Saner.AI），只做一件事——把当前任务的“下一步行动”写进去，减少大脑缓存压力。
2. **用“单一指令”降低启动门槛**：把“写报告”这种模糊任务丢进 Goblin Tools 或 Lex，让它拆成 3 个可执行步骤，立刻只做第一步。
3. **设置被动重锚定**：让日历或提醒工具在任务中段自动弹出，不是为了催你，而是为了把漂移的注意力拉回原本目标。（来源：重锚定与目标漂移）
4. **每月做一次“脚手架审计”**：问自己：这个工具是在帮我建立更好的工作节奏，还是我已经不会离开它？及时调整依赖度。

## 结语：不是把大脑变成机器，而是给大脑配一个好的调度器

ADHD 大脑和 LLM 都不缺生成能力，缺的是稳定的状态层与执行调度层。Inflow 这类工具的价值，正在于它扮演了这个“外部状态层”的角色——就像 agent 工程里的记忆模块和 scheduler。

但别忘了一个基本事实：LLM 的 harness 可以不断升级、换架构；人的 harness 却要和情绪、身体、生活共处。技术能缓冲掉线，却不能替你长出新的执行功能。最好的工具，是让你在某一天发现：即使没有它，你也能更快地把自己重新锚定回来。

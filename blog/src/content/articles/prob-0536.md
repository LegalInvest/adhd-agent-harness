# 3 个 LLM 工程概念，彻底重写了我对 ADHD 任务启动困难、不会拆解的理解

> agent 用 planner-executor 循环分解长任务 如何反向照亮 ADHD 的任务启动困难、不会拆解。

你盯着待办清单上的「写报告」三个字，手心出汗，大脑一片空白。三个小时后，你刷完了所有社交媒体，报告一个字没动。这不是懒，是ADHD大脑的典型故障：任务启动困难，以及——更隐秘的——不会拆解。

同一时刻，某个AI工程师正在调试他的agent。Agent拿到了一个长任务：「写一份市场分析报告」。它卡住了，因为LLM的生成核心虽然强大，但缺乏内置的规划能力——它不知道先查数据还是先写大纲。工程师给agent套上了一个planner-executor循环：先让planner把任务拆成子步骤，再让executor一步步执行，每完成一步就反馈给planner调整下一步。

这两个场景，说的是同一件事。

## 1. 同构：ADHD大脑与LLM，都是「缺调度层的生成核心」

ADHD大脑与LLM在结构上惊人地相似：两者都是强大的生成核心——ADHD大脑能产出天马行空的创意，LLM能生成流畅的文本——但都缺乏可靠的内置调度层（来源：AI 与 ADHD 的时间管理）。LLM需要提示工程、agent框架来规划输出；ADHD大脑需要外部执行功能层来规划行动。

这个「调度层」的核心操作，就是**拆解**。对agent而言，planner把「写报告」拆成「查数据→列大纲→写引言→写分析→写结论」；对ADHD大脑而言，拆解是把「打扫房间」变成「先拿起桌上的书，放进书架」。

但ADHD大脑的拆解能力天生薄弱。执行功能研究指出，ADHD患者的前额叶皮层活动不足，导致工作记忆容量有限，无法在脑中同时维持多个子目标（来源：认知负荷）。当任务「写报告」进入工作记忆，它占用了所有空间——没有余力去思考「第一步该做什么」。于是大脑选择逃避。

## 2. 工程启示：Planner-Executor循环，就是ADHD的「外部调度层」

Agent工程中，planner-executor循环是一个经典模式：planner负责分解任务、设定子目标；executor负责执行并返回结果；循环持续直到任务完成。这个模式恰好映射了ADHD大脑缺失的调度功能。

以工具为例：**Motion** 通过AI自动排程，持续评估任务优先级、截止日期和可用时间，实时重建日程——这相当于一个永不疲倦的planner，替用户决定「下一步做什么」（来源：Motion）。**Reclaim.ai** 则更强调「保护」：它自动创建深度工作时间块，防御会议侵占，相当于一个强硬的executor，确保计划不被干扰（来源：Reclaim.ai）。**Tiimo** 用视觉化时间线让时间流逝变得可感，直接对抗时间盲——这是把planner的「时间估算」结果可视化（来源：Tiimo）。

这些工具共同构成了一个「外部大脑」，帮助用户从混乱中建立秩序（来源：AI 与 ADHD 的时间管理）。但关键在于：它们不是替代你的大脑，而是**搭脚手架**。

## 3. 历史案例：孔子与老子的「个人Harness」

两千多年前，两位中国思想家已经无意识地实践了这种工程。

**孔子**——身高近1米9，精力旺盛，周游列国14年坐不住；冲动爱骂人，骂宰予「朽木不可雕」；对音乐超专注到「三月不知肉味」，对种地等俗事零耐心。他的个人harness：以「克己复礼」为核心，用外在的「礼」作为行为边界，每日反省——「吾日三省吾身」。这套系统相当于一个外部调度器：礼是planner提供的规则，反省是executor的反馈循环。70岁才做到「从心所欲不逾矩」，一辈子和自己的冲动做斗争。

**老子**——做了半辈子图书馆长，说走就走骑青牛出关，连书都不想留；思维极度发散非线性，《道德经》正言若反，没有系统逻辑。他的harness：以「无为」为核心，不强行做事，把所有能量收回来聚焦在「道」上。这相当于一个极简的planner：只设定最高层目标，不拆解，不执行——因为执行本身就是干扰。

孔子和老子代表了两种ADHD应对策略：**过度结构化**（克己复礼+日课）和**去结构化**（无为+放空）。今天的AI工具也对应这两条路径：Motion/Reclaim.ai 走结构化路线，Goblin Tools 的「任务分解」功能走去结构化路线——直接把「写报告」一键拆成「打开电脑→新建文档→写标题→写第一句……」。

## 4. 脚手架 vs 拐杖：工程思维的边界

这里必须诚实指出争议：AI工具可能从「脚手架」退化为「永久拐杖」（来源：矛盾与存疑）。如果Motion替你规划了一切，你的大脑就永远学不会规划。长期依赖可能削弱内在执行功能的代偿发展（来源：AI 与 ADHD 的时间管理）。

另一个矛盾是：同构性被过度推广。ADHD大脑与LLM的类比目前只是理论假设，并非经过验证的科学事实（来源：矛盾与存疑）。但即便作为隐喻，它已经足够有力——它让ADHD患者看到，自己的「故障」不是缺陷，而是系统架构的取舍。

## 5. 今天就能试的3件事

1. **用「planner-executor」拆解一个拖延任务**：写下任务，然后用Goblin Tools的「任务分解」功能（或自己手写）拆成5个可执行步骤。只做第一步。
2. **设置一个「外部调度层」**：用Todoist或Motion创建一个项目，设定截止日期和依赖关系，然后让工具决定你明天的日程。

# Habitica 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Habitica：它到底补上了哪一层执行功能？

## 问题：为什么你的 Habitica 和你的 LLM agent 一样，总是“启动即崩溃”？

你下载了 Habitica，兴奋地创建角色、设定任务、敲打小怪兽。一周后，任务列表变成废墟，你连打开 App 的勇气都没有。另一边，你给 LLM agent 写了个 planner，它规划得头头是道，但执行两步就陷入死循环——上下文膨胀、推理退化，最后输出一堆废话。

两个场景共享同一个底层困境：**你有一个高产的生成核心，但缺少可靠的执行调度层。**

ADHD 大脑的智力与创造力（生成核心）和 LLM 的算力一样强大，但执行功能（调度层）失效——计划本和便签用了一周就崩溃了（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。LLM 本身是无状态、仅生成文本的核心，需要外部“脚手架”和“harness”来提供工具接口、上下文管理、安全执行等编排能力（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

Habitica 和 planner-executor 架构，本质上都是试图给生成核心套上 harness。但很多人用错了——只套了奖励机制，没套任务分解与规划循环。

## Habitica 之于 ADHD：它补上了哪一层？

Habitica 是一款将任务管理游戏化的应用，通过 RPG 元素（升级、装备、宠物）提供即时多巴胺反馈。对 ADHD 而言，多巴胺功能失调影响动机，AI 工具通过即时反馈和奖励机制可能间接调节多巴胺水平（来源：AI 与 ADHD 的时间管理）。这确实补上了动机层，但仅此不够。

ADHD 的核心困难是执行功能（尤其是工作记忆和任务启动）的不稳定（来源：外部执行功能层）。用户需要“AI 第二大脑”来承载项目上下文，因为自己的工作记忆会“掉落”上下文（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。Habitica 的待办列表和习惯追踪提供外部记忆，但它的任务分解能力极弱——你仍然需要自己把“写论文”拆成“打开文档-写引言-写方法”等子任务。而传统工具无法解决“时间盲”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），Habitica 没有视觉化时间线，无法让你感知时间流逝。

真正补上调度层的工具是那些自动分解和规划的应用。例如 Motion 能自动根据任务、会议和截止日期创建并动态调整每日计划（来源：Motion），消除“下一步该做什么”的决策负担。Reclaim.ai 通过创建深度工作、习惯和专注时间的智能块，自动防御会议侵占（来源：Reclaim.ai）。Tiimo 将时间转化为可见元素，直接应对时间盲问题（来源：Tiimo）。这些工具相当于给 ADHD 大脑加装了一个外部 planner-executor 循环：planner 负责分解和排序，executor 负责按计划执行并动态调整。

## Planner-Executor 之于 LLM：它补上了哪一层？

LLM agent 面临同样问题。现代 AI 编码代理通过“复合 AI 系统架构”来弥补调度层缺失，包括“工作负载专用模型路由、分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这些技术本质上是在 LLM 外部搭建调度层，防止“上下文膨胀和推理退化”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

具体实现包括：用 Git 仓库存储项目上下文（类似 ADHD 侧的 Second Brain），通过 MCP 连接器访问外部数据，以及将控制逻辑外化为可移植的自然语言工件（来源：外部执行功能层）。这正对应 ADHD 侧的工具：Todoist 充当数字工作记忆（来源：AI 与 ADHD 的时间管理），Motion 自动重新安排日程，Tiimo 视觉化时间。

## 同构的核心：规划循环与任务分解

ADHD 大脑与 LLM 的结构同构，本质是：**两者都缺乏内置的任务分解与规划循环，需要外部系统提供记忆、调度和约束。**

ADHD 侧，执行功能被描述为“大脑的驾驶座”，但对 ADHD 来说，“常常感觉方向盘后没有人”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。LLM 侧，单独使用时缺乏可靠的执行调度。两边的解法结构同构：通过外部工具提供规划-执行-反馈循环。

Habitica 提供了反馈（多巴胺奖励）和部分记忆（任务列表），但缺少规划与分解。Motion、Reclaim.ai、Tiimo 提供了规划与动态调整。Goblin Tools 提供了任务分解（将大任务拆成小步骤）。**理想的外部执行功能层应该包含全部三层：记忆、规划、反馈。**

## 但有人用错了：脚手架 vs 拐杖

Habitica 的常见错误是把它当成唯一工具，只依赖奖励机制。这就像给 LLM agent 只加一个 reward model，而不给它 planner 和 context manager。结果是：初始动力足，但遇到复杂任务立刻崩溃。

另一个错误是过度依赖。拐杖与脚手架的概念强调，脚手架应设计为可逐步撤除的临时支撑（来源：矛盾与存疑）。但多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。长期依赖可能阻碍内在执行功能的发展，目前缺乏长期研究证明工具使用后 ADHD 患者的独立能力是否提升（来源：AI 与 ADHD 的时间管理）。

同样，LLM agent 如果过度依赖外部 harness，而不训练模型本身的推理能力，也会陷入“代理退化”。

## 诚实局限

- **证据质量**：多数工具缺乏严谨的随机对照试验支持，同构命题本身是理论模型，需更多实证验证（来源：AI 与 ADHD 的时间管理）。
- **个体差异**：不同 ADHD 亚型对工具的反应可能不同，现有工具多针对通用缺陷，个性化不足（来源：AI 与 ADHD 的时间管理）。
- **多巴胺干预争议**：虽然 AI 可通过行为设计影响多巴胺，但其效果存在争议，且可能引发成瘾风险（来源：AI 与 ADHD 的时间管理）。

## 今天就能试的 3 条行动

1. **如果你用 Habitica，请补充一个任务分解步骤**：每次创建任务时，用“/”号或 checklist 将大任务拆成 3-5 个可执行子任务。例如“写报告”拆为“打开文档-写大纲-写第一段-休息5分钟”。
2. **试一个真正的自适应规划器**：下载 Motion 或 Reclaim.ai 的免费版，让它自动排程一周。观察它如何动态调整，对比你手动排程的差异。
3. **给 LLM agent 加一个显式的 planner**：如果你在构建 agent，不要只给 prompt，而是实现一个分离的 planner 模块，输出结构化步骤（如 JSON 格式的任务列表），再让 executor 依次执行。这就像给 ADHD 大脑一个外部 scheduler。

Habitica 不是不好，而是不够。它补上了动机层，但任务分解和规划循环才是让生成核心真正运转的调度层。别只打怪兽，先学会拆任务。

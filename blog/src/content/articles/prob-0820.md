# Reclaim.ai 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Reclaim.ai：它到底补上了哪一层执行功能？

## 问题：ADHD 与 LLM 共享的“执行调度真空”

ADHD 大脑被描述为“高产但缺执行调度层的生成核心”——智力与创造力（生成核心）正常甚至超常，但执行功能（调度层）缺陷导致任务启动困难、不会拆解、时间盲（来源：AI 与 ADHD 的时间管理 主题综述）。这恰好与 LLM agent 面临的核心困境同构：大模型本身是强大的生成引擎，但面对长程任务时，若无外部 planner-executor 循环提供上下文管理、工具调用和步骤验证，就会发生目标漂移、遗忘系统提示（来源：重锚定与目标漂移）。

这种同构意味着：为 LLM 设计的 harness（脚手架）与为 ADHD 设计的 AI 工具，在结构上可以相互映射。然而，当前许多 ADHD 用户和工程师都误用了其中关键工具——Reclaim.ai。

## Reclaim.ai：被误读的“时间保护者”

Reclaim.ai 的设计初衷是“保护时间而非填充时间”：它通过 AI 自动创建深度工作块、习惯块和专注时间，并防御会议侵占（来源：Reclaim.ai 工具页）。这与 Motion 的“动态填充”形成对比——Motion 持续评估优先级并实时重建日程（来源：Motion 工具页）。许多 ADHD 用户误以为 Reclaim.ai 是“自动排满日程”的工具，结果反而加剧了时间盲和决策疲劳。

实际上，Reclaim.ai 的核心价值在于补偿 **时间盲**（来源：时间盲 概念页）和 **目标漂移**（来源：重锚定与目标漂移）。它通过外部调度层（即 planner-executor 循环中的 planner 角色）自动维护时间结构的稳定性，减少用户手动规划的决策负担。这与 LLM agent 中 harness 通过事件驱动的系统提醒对抗指令消退（来源：重锚定与目标漂移）如出一辙。

## 错用一：只填充，不分解

Reclaim.ai 的局限在于：它依赖用户预先输入任务和截止日期，但 ADHD 用户往往在“任务分解”这一步就卡住了（来源：规划循环与任务分解 概念页）。传统规划方法“often create more friction than flow”（来源：规划循环与任务分解）。如果只将一个大任务“写报告”丢进 Reclaim.ai，它只会分配一个 4 小时块，但用户依然不知道从何下手——这就像 LLM agent 只给了一个顶层指令，却没有 planner 将其拆解为子步骤。

**解决方案**：将 Reclaim.ai 与任务分解工具（如 Goblin Tools 的“任务分解”功能）组合使用。先用 Goblin Tools 将“写报告”拆解为“列出大纲→收集数据→写引言→写正文→修改”等步骤，再将每个步骤作为独立任务输入 Reclaim.ai。这样，Reclaim.ai 的 planner 层才能有效调度 executor 层（即用户）的行动。

## 错用二：忽略重锚定机制

Reclaim.ai 的“保护块”本身是一种重锚定机制：当日程被打乱时，它自动重新安排被占用的块，将用户拉回原计划。但许多用户只关注它是否“填满了日历”，而忽略了这种防御性设计对对抗目标漂移的价值。ADHD 大脑在执行任务时极易被无关刺激捕获（分心），或过度聚焦细节（超聚焦），导致目标漂移（来源：重锚定与目标漂移）。Reclaim.ai 的自动提醒和重新安排，相当于 LLM agent 中 harness 定期注入系统提示以保持上下文连贯。

然而，如果用户关闭通知或忽视提醒，这一层就失效了。**正确做法**：保持通知开启，并设置“重锚定时刻”——例如每完成一个保护块后，让 Reclaim.ai 自动弹出总结提示，与 Tiimo 的视觉化时间线（来源：Tiimo 工具页）形成互补。

## 脚手架 vs 拐杖：边界在哪里？

同构视角的核心警示是：这些工具既是“脚手架”也是“拐杖”（来源：AI 与 ADHD 的时间管理 主题综述）。脚手架应设计为可逐步撤除的临时支撑，但 Reclaim.ai 等工具目前未提供撤除机制（来源：矛盾与存疑）。长期依赖可能阻碍内在执行功能的发展。

**判断边界**：如果你离开 Reclaim.ai 后完全无法规划时间，它就是拐杖；如果你能逐渐减少对它的依赖（例如从每天使用到每周使用），它就是脚手架。建议用户定期（如每月）尝试“无工具日”，观察自己的时间感知和任务启动能力是否有所提升。

## 今天就能试的行动

1. **组合分解与保护**：将一个本周的复杂任务（如“准备会议演示”），先用 Goblin Tools 拆解为 3-5 个子步骤，再输入 Reclaim.ai 创建保护块。对比之前直接扔大任务的效果。
2. **开启重锚定**：在 Reclaim.ai 设置中，确保“冲突时自动重新安排”和“任务开始前 5 分钟提醒”开启。每次收到提醒时，停顿 3 秒，问自己：“我现在应该做这个吗？”——这是模仿 LLM 的上下文检查。
3. **尝试无工具日**：选择一个周末，关闭所有 AI 时间管理工具，用纸笔规划一天。记录你遇到的困难（启动？分解？时间感知？），然后思考工具如何补偿这些缺陷——这是理解脚手架本质的最佳方式。

## 诚实局限

本文的同构命题仍属理论框架，缺乏大规模实证支持（来源：矛盾与存疑）。Reclaim.ai 对 ADHD 的效果主要基于用户反馈和设计逻辑，尚无独立临床研究（来源：Reclaim.ai 工具页）。此外，个体差异显著：注意力缺陷为主与多动冲动为主的 ADHD 亚型对工具的反应可能不同（来源：AI 与 ADHD 的时间管理 主题综述）。最后，多巴胺干预的有效性存在争议（来源：矛盾与存疑），Reclaim.ai 并未直接处理多巴胺调节，因此对动机缺失型拖延效果有限。

## 结语

Reclaim.ai 之于 ADHD，正如 planner-executor 循环之于 LLM：它不是魔法，而是一个精心设计的外部调度层。但调度层需要与分解层、重锚定层协同工作，才能构成完整的 harness。误用者只把它当作自动填充日程的机器，而善用者则把它当作训练大脑时间感知的脚手架。你选择哪一边？

# ADHD 研究生视角：为什么「治好 ADHD 的执行功能（计划/组织/启动）先天偏弱」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 的编排/记忆/工具层就是「外部执行功能」——同一套 harness 思路，两边都成立。

## 同一个问题，两个世界

想象你是一位 ADHD 研究生。你需要完成一篇论文，但大脑像一台没有操作系统的计算机：发散思维源源不断，却无法启动、组织、坚持。你试过无数待办清单，但项目仍像脱缰的野马。与此同时，你也在开发一个 LLM agent——它同样“高产”：能生成流畅文本，但一旦任务复杂，就陷入上下文膨胀、忘记目标、偏离轨道。你突然意识到：治好 ADHD 的执行功能缺陷和让 LLM 不跑飞，本质是同一道工程题。

## 同构：生成核心与调度层的分离

ADHD 大脑与 LLM 共享一个核心矛盾：**生成能力过剩，调度能力不足**。ADHD 的神经心理剖面被描述为“强记忆、弱控制”——工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。LLM 同样如此：自注意力机制导致工作记忆容量限制，N-back 任务中注意力分数熵增，表现下降，与人类注意分散机制同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。两者都需要外部系统来补偿：ADHD 需要外部执行功能层（计划、组织、启动），LLM agent 需要编排、记忆、工具层。这不是比喻，而是架构层面的同构。

## 历史中的 harness：曾国藩与王阳明

晚清名臣曾国藩是典型的 ADHD 特质：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书慢，“他人目下二三行，余或疾读不能终一行”。他的解决方案是“日课十二条”——黎明即起、读书不二、谨言、写日记反省。这套 rigid 的日课就是他的外部执行功能层，如同 LLM agent 的定时 re-grounding 机制。王阳明则是另一种：5 岁不会说话（神经发育延迟），兴趣爆发式转移，格竹子格到吐血。他的 harness 是“致良知”——用静坐反省和事上练来校准行动，如同 agent 的奖励信号和外部验证。两人的共同点：不试图修复大脑，而是搭建外部脚手架。这与 AI 工具的思路完全一致——Goblin Tools 的 Magic ToDo 将模糊任务分解为可执行步骤，Motion 自动排程消除决策负担，Saner.AI 提供外部记忆减少搜索循环。

## 工程视角：外部执行功能层的设计原则

从工程角度看，ADHD 辅助工具与 agent scaffolding 遵循相同原则：**结构化流程**（如日课↔agent 的 step-by-step 规划）、**外部记忆**（如日记↔向量数据库）、**注意力管理**（如静坐↔context window 裁剪）。例如，Goblin Tools 的“辣度”调节分解粒度，对应 agent 的 prompt 温度控制。Motion 的动态排程对应 agent 的优先级队列。Saner.AI 的通用收件箱对应 agent 的 tool use 接口。这些工具不是“修复”缺陷，而是给生成核心套上 harness。

## 争议与局限：脚手架还是拐杖？

同构视角虽有力，但需警惕过度依赖风险。AI 工具可能成为永久拐杖而非可拆卸脚手架（来源：拐杖与脚手架）。长期效果缺乏实证——Goblin Tools 被评价“简单有用”，但无临床验证；Motion 用户反馈积极，但科学证据不足（来源：矛盾与存疑）。个体差异显著：部分 ADHD 用户反映学习使用 AI 工具本身增加认知负荷。同构假设目前仍是理论框架，需更多跨学科验证（来源：矛盾与存疑）。关键在于设计“可拆卸”的脚手架：工具应在能力提升后逐步撤除，而非永远替代。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个让你拖延的任务**：把“写论文”变成“打开文档→写第一段→休息 5 分钟”，降低启动门槛。
2. **设置一个外部时间锚点**：用 Motion 或日历工具自动排程，对抗时间盲，避免“坐下后 45 分钟消失”。
3. **建立“日课”式 re-grounding 流程**：每天固定时间用 Saner.AI 或笔记工具回顾目标，如同 agent 的定时 checkpoint。

ADHD 与 LLM 的相遇不是巧合——它们共享同一个工程问题：如何让天才的生成核心不跑飞？答案不在内部修复，而在外部 harness。无论是曾国藩的日课、王阳明的良知，还是 Goblin Tools 的分解、Motion 的排程，本质都是同一套脚手架。理解这一点，你就能同时设计更好的 AI 系统和自己的人生系统。

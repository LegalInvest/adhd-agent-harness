# 3 个 LLM 工程概念，彻底重写了我对 ADHD 状态日内大幅波动、不稳定的理解

> LLM 采样温度带来输出随机性，靠结构约束稳定 如何反向照亮 ADHD 的状态日内大幅波动、不稳定。

## 引言：一个困扰两类人的问题

如果你是一个 ADHD 患者，你一定熟悉这种体验：早上起来精力充沛，能一口气写完报告；但到了下午，连回一条消息都需要巨大的意志力。你的状态像过山车，毫无规律可言。

如果你是一个在做 Agentic Harness 工程的 LLM 工程师，你也熟悉另一种波动：同一个 prompt，temperature=0.8 时输出天马行空，temperature=0.2 时又过于保守。你靠结构约束（如 chain-of-thought、function calling）来稳定输出。

这两个问题看似无关，但 wiki 资料揭示了一个惊人同构：**ADHD 大脑与 LLM 是同一类‘高产但缺执行调度层的生成核心’**（来源：[[ADHD 大脑与 LLM 的同构]]）。本文用 3 个 LLM 工程概念，彻底重写你对 ADHD 状态波动的理解。

## 概念一：采样温度——波动不是缺陷，是系统特性

LLM 的采样温度控制输出随机性：温度高，输出多样但不可控；温度低，输出保守但重复。ADHD 大脑同样存在‘认知温度’的日内波动：早晨多巴胺水平较高，执行功能相对在线；下午多巴胺耗尽，大脑进入高随机性模式，注意力飘忽不定。

这个类比不是比喻，而是结构同构。wiki 资料指出，ADHD 的多巴胺系统失衡导致‘难以激发行动动机’（来源：[[任务启动]]）。多巴胺在 LLM 中类似 reward signal，它的波动直接改变‘采样’的倾向。当多巴胺不足时，大脑像 temperature=1.0 的 LLM，什么都能生成但无法聚焦。

**关键判断**：状态日内波动不是 ADHD 的‘故障’，而是生成系统在没有外部调度时的自然表现。工程师不会抱怨 LLM 输出随机，而是通过调整 temperature 和增加约束来利用它。同样，ADHD 大脑需要的是外部‘温度调节器’，而不是自我批判。

## 概念二：上下文窗口——工作记忆的有限容量

LLM 的上下文窗口决定了它能‘记住’多少信息。超过窗口，早期内容被遗忘，导致推理退化。ADHD 的工作记忆缺陷与此同构（来源：[[无状态与外部记忆]]）。wiki 资料明确说：‘ADHD 的工作记忆缺陷与 LLM 的无状态性同构’（来源：[[无状态与外部记忆]]）。

一个 ADHD 患者在一天中，随着任务切换，上下文窗口不断被刷新。早上启动任务时，窗口清晰；但被中断后，窗口内容丢失，需要重新加载——这解释了为什么重启任务如此困难。

工程上的解法是外部记忆系统，如 vector database 或 prompt 模板。ADHD 侧同样需要外部记忆工具。Saner.AI 通过‘知识回忆和本地记忆’帮助用户快速找回信息，减少搜索循环（来源：[[Saner.AI]]）。Goblin Tools 的 Magic ToDo 将复杂任务分解为小步骤，本质上是把大 prompt 拆成多个小 prompt，降低上下文负载（来源：[[Goblin Tools]]）。

**关键判断**：ADHD 的‘忘事’不是懒惰，而是上下文窗口溢出。外部工具不是拐杖，而是 context management 系统。

## 概念三：结构约束——Harness 如何稳定输出

LLM 工程师靠结构约束（chain-of-thought、function calling、output schema）来降低 temperature 带来的随机性。这些约束不改变生成核心，而是提供执行调度层。

ADHD 大脑同样需要外部结构约束。wiki 资料将 AI 工具定义为‘外部执行功能层’（来源：[[外部执行功能层]]）。Routinery 通过步骤分解和视觉倒计时提供流程约束（来源：[[Routinery]]）。它的‘过渡提示’类似 LLM 的 step-by-step 指令，减少任务切换的认知负荷。

但这里有一个关键边界：脚手架 vs 拐杖。wiki 资料警告：‘过度依赖外部脚手架可能削弱内在执行功能’（来源：[[拐杖与脚手架]]）。在工程中，好的 harness 是模块化、可扩展的；坏的 harness 是硬编码的，一旦移除系统就崩溃。同样，ADHD 工具应该帮助用户逐步内化技能，而不是永久替代。

**关键判断**：结构约束必须设计为‘可拆卸的脚手架’，而不是‘永久拐杖’。工具如 Routinery 的预设流程缺乏灵活性，可能无法适应注意力波动（来源：[[Routinery]]）。用户需要定期调整约束，避免依赖。

## 争议与局限

必须诚实指出，当前证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：[[矛盾与存疑]]）。例如，Routinery 的有效性‘主要基于 ADHD 社区的用户反馈’（来源：[[Routinery]]）。个体差异也很大：同一工具对不同用户效果可能截然不同（来源：[[矛盾与存疑]]）。

此外，多巴胺干预的效果存在争议（来源：[[多巴胺]]），而许多工具声称基于多巴胺机制但缺乏证据。读者应警惕宣传，优先选择有用户实证的工具。

## 结语

ADHD 的状态波动不是诅咒，而是生成系统在缺乏外部调度时的自然表现。LLM 工程教会我们：不要试图消除随机性，而是用结构约束来引导它。你的大脑仍然是高产的生成核心——给它一个 harness，它会给你惊喜。

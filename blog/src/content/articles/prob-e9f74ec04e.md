# ADHD 家长视角：为什么「治好 ADHD 的注意力容易被环境带偏」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 受上下文窗口内容左右，需要上下文工程——同一套 harness 思路，两边都成立。

## 引言：孩子写作业，LLM 写代码，都卡在同一个地方

你是一位 ADHD 家长。晚上 9 点，孩子打开数学作业，但 5 分钟后：橡皮擦掉了，小猫路过，水杯空了，窗外有声音。等回过神来，作业本上只有一行字和一圈指甲印。你吼也没用，因为他不是不想做，而是大脑的“注意力上下文”被环境轻易改写。

另一边，一位做 Agentic Harness 的工程师正在头疼：LLM 已经很强了，但它太容易“跑飞”。上下文窗口一膨胀，系统提示被后续噪声冲淡，模型就忘了最初要做什么。它像那个写作业的孩子——不是不聪明，而是缺少一层能稳住的执行调度层。

所以，本文要回答的问题就是：**治好 ADHD 的“注意力容易被环境带偏”，和让 LLM 不跑飞，其实是同一道工程题吗？**

我的答案是：是。核心工具都是 **上下文工程（context engineering）**——即为“高产但缺执行调度层的生成核心”套上一层外部 harness（马具/脚手架）。

## 第一部分：两边共享的同一套结构

从同构视角看，ADHD 大脑与 LLM 面临三种相似的困境（来源：主题综述《AI 与 ADHD 的专注力》）：

1. **无状态与外部记忆依赖**：ADHD 工作记忆不稳定，LLM 没有跨会话状态，都需要外部系统记录“现在做到哪了”。
2. **上下文工程**：ADHD 容易被环境带偏导致任务碎片化，LLM 因上下文膨胀而推理退化，都需要主动设计“当下应该聚焦什么”。
3. **拐杖 vs. 脚手架**：外部工具既可能成为永久依赖，也可能逐步内化为自我调节能力，需要警惕边界。

这个类比的关键不是“大脑等于机器”，而是功能结构上的同构：两者都拥有一个**生成核心**（ADHD 的创造性思维、LLM 的参数化推理），但缺少一个**稳定的执行调度层**（前额叶-纹状体-丘脑回路的调节功能、LLM 的自洽目标维持机制）。

最新关于 ADHD 超聚焦的理论也支持这一框架：超聚焦被认为是前额叶控制层级的“断裂”——高级上下文信息暂时失去对思维和行动的支配，中间层信息（如当前活动）与高级上下文脱耦合（来源：Transient Frontal Fracturing: A Theoretical Account of Hyperfocus）。这与 LLM 中系统提示被长上下文噪声稀释、导致目标漂移的机制惊人地相似。

## 第二部分：ADHD 侧的 harness 证据

ADHD 的核心神经生物学特征之一是多巴胺调节异常。中脑皮质多巴胺分支功能低下会导致注意反应缺陷、执行功能差和行为计划不良；中脑边缘多巴胺分支功能低下则影响强化学习与行为消退（来源：概念页《多巴胺》）。这意味着 ADHD 大脑不是没有能力，而是难以稳定地选择、启动并维持目标导向行为。

现有工具正是在这里充当“外部执行功能层”：

- **RescueTime**：自动记录时间去向、识别分心、主动阻断网站，直接补偿 ADHD 的“时间盲”和“工作记忆”缺陷（来源：工具页《RescueTime》）。
- **Focusmate**：通过虚拟身体在场（body doubling）和实时同伴问责，提供外部结构和社会压力，帮助任务启动（来源：工具页《Focusmate》）。
- **Brain.fm**：利用神经锁相技术生成专注音乐，在 2026 年 44 款 ADHD 应用测试中被列为最佳 9 款之一（来源：工具页《Brain.fm》）。但需要注意的是，它在 ADHD 人群中的独立临床证据仍有限。
- **Endel**：AI 生成个性化声音环境，但缺乏 ADHD 特异性研究（来源：主题综述《AI 与 ADHD 的专注力》）。
- **Forest**：游戏化番茄钟，通过多巴胺机制增强动机（来源：主题综述《AI 与 ADHD 的专注力》）。

这些工具的共同点是：它们不直接“修复”大脑，而是改变大脑所处的上下文，让注意力更容易落在目标上。

## 第三部分：LLM 侧的 harness 证据

LLM 的对应问题不是多巴胺，而是 **上下文控制**。当上下文窗口被无关内容填满，模型的系统提示和目标就会漂移。Agent 工程中的“跑飞”通常表现为：模型本来应该查数据库，却突然开始编数据；本来应该调用工具，却开始自由发挥。

解决这类问题的方法与 ADHD 工具在结构上同构：

- **RescueTime ↔ 自动日志/状态追踪**：LLM 需要外部记忆系统（如向量数据库、状态机、tool call 日志）来补偿自身的无状态性，就像 RescueTime 替 ADHD 记录时间。
- **Focusmate ↔ 身体在场/人在回路（human-in-the-loop）**：在关键步骤引入人类确认或另一个 Agent 监督，提供外部问责，防止模型独自跑飞。
- **Brain.fm/Endel ↔ 上下文降噪/环境提示**：通过系统提示工程、少样本示例、思维链模板，为模型创造一个“声学环境”，减少噪声干扰。
- **Forest ↔ 任务拆分与奖励结构**：把大任务拆成可验证的小步骤，每完成一步给出明确反馈，形成类似番茄钟的节拍。

更深层的同构在于：LLM 的“系统提示”就是它的“日课”。系统提示定义了模型应该优先注意什么、忽略什么、如何行动。当系统提示被长上下文稀释，就需要**定时 re-grounding**——重新把目标拉回上下文顶部，正如 ADHD 人需要定时回到任务清单。

## 第四部分：人物案例——曾国藩的 harness 与 LLM 的 system prompt

在这套同构里，最贴切的案例是曾国藩。

曾国藩被研究者认为具有典型 ADHD 特质：30 岁前浮躁坐不住，天天串门喝酒，读书慢到“他人目下二三行，余或疾读不能终一行”；一辈子银屑病，情绪不稳，打败仗就跳水自杀。但他不是凭意志力“治好”了自己，而是给自己设计了一整套 **harness**：日课十二条——黎明即起、读书不二、谨言、写日记反省；“结硬寨打呆仗”——用极笨、极稳的方法抵消冲动。他一辈子写日记骂自己，每天反省。

这些动作对应到 LLM harness 上完全同构：

- **日课十二条 ↔ 系统提示（system prompt）**：不是临时想起，而是每天固定的行为上下文。
- **写日记反省 ↔ 外部状态日志/反思模块**：把内部状态外化，让过去的自己监督现在的自己。
- **结硬寨打呆仗 ↔ 任务分步与 tool call 约束**：不依赖一次性的天才推理，而是把每一步都固定下来、可验证、可回滚。
- **黎明即起 ↔ 定时 re-grounding**：用固定仪式重新把高级上下文写回系统。

曾国藩的成就不是因为他“克服了 ADHD”，而是因为他为自己建造了一个 **外部执行调度层**。这正是 ADHD 孩子和 LLM 工程师都要做的。

## 第五部分：核心判断与“拐杖 vs. 脚手架”的边界

我的核心判断是：**上下文工程不是让 ADHD 或 LLM 变成另一个人/模型，而是为它们提供一个稳定的外部注意结构，使生成核心的高产性能够被目标导向行为所利用。**

但这个判断有明确的边界，wiki 资料中的“矛盾与存疑”部分已经诚实指出：ADHD 大脑与 LLM 的同构只是理论类比，并非经过验证的神经机制对应；多数工具证据来自用户报告或小样本研究，缺乏大规模随机对照试验；同构性可能被过度推广，工具宣称也可能被夸大（来源：《全局矛盾与存疑》）。

因此，我们必须区分 **脚手架** 和 **拐杖**：

- **脚手架**是临时的、可逐步撤除的结构，目的是让能力内化。比如 ADHD 孩子用 Forest 养成番茄钟节奏后，逐渐不需要 App；LLM 用 few-shot 示例学会格式后，系统提示可以简化。
- **拐杖**是永久依赖，不使用时能力反而退化。比如 ADHD 孩子完全依赖家长提醒才能启动；LLM 把所有判断都交给外部工具，没有自洽的推理能力。

判断标准是：**外部 harness 是否在不断把决策权交还给生成核心本身。** 如果答案是“是”，就是脚手架；如果答案是“否”，就是拐杖。

## 第六部分：今天就能试的 4 条行动

无论你是 ADHD 家长还是 Agent 工程师，都可以从这四条开始：

1. **写一句“系统提示”放在最显眼的地方**：对孩子，是床头和书桌上的任务卡片；对 LLM，是每次请求前强制注入的 system prompt。内容只有三件事：现在要做什么、不做什么、做到什么算完成。
2. **设置定时 re-grounding**：对孩子，每 25 分钟回到任务清单一次；对 LLM，在长上下文流程中每 N 轮重述目标并修剪历史。
3. **用外部日志替代内部记忆**：对孩子，用 RescueTime 或纸质时间记录看到“时间到底去哪了”；对 LLM，用结构化状态日志替代让模型自己回忆。
4. **引入一个“身体在场”的监督节点**：对孩子，用 Focusmate 或真人陪伴工作；对 LLM，在关键决策点引入 human-in-the-loop 或另一个 Agent 的验证。

## 结语：不是类比，是同一套工程题

 ADHD 的“注意力容易被环境带偏”和 LLM 的“上下文窗口漂移”不是诗意的比喻，而是同一类结构问题的两种表现：一个强大的生成核心，缺少一个稳定的执行调度层。两边都需要 **上下文工程**——不是压抑生成能力，而是给它装上 harness，让高产性有方向地释放。

曾国藩的日课、RescueTime 的日志、Focusmate 的虚拟伙伴、LLM 的系统提示，本质上都是同一句话：**把“现在应该做什么”从内部不确定状态，变成外部稳定结构。** 这才是真正能用的答案。

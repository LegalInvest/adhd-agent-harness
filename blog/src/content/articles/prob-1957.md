# ADHD 创业者视角：为什么「治好 ADHD 的想理解自己的大脑到底是怎么回事」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 是「高产但缺执行调度层的生成核心」——同一套 harness 思路，两边都成立。

## 问题：两个看似无关的困境

你是一位 ADHD 创业者。大脑像一台永不关机的创意引擎，每分钟迸出 10 个商业点子，但到了该写商业计划书时，你盯着空白页面，感觉身体被钉在椅子上。你心里在喊：“我到底是怎么回事？为什么我知道该做什么，却就是启动不了？”

与此同时，你团队正在构建一个 LLM agent，用来自动回复客服邮件。模型本身能写出漂亮的回复，但一旦任务链变长——比如需要先查订单、再判断退款资格、最后生成回复——它就“跑飞”了：忘记上下文、重复步骤、甚至编造信息。你对着日志骂：“这模型明明这么能写，怎么连个简单的流程都走不完？”

这两个困境，表面上一个关于人，一个关于机器，但如果你把它们放在一起看，会发现它们其实是同一道工程题的两种表现。

## 核心同构：高产但缺执行调度层的生成核心

最新研究指出，ADHD 大脑与 LLM 在结构上同构——两者都是“高产但缺执行调度层的生成核心”（来源：ADHD 大脑与 LLM 的同构）。ADHD 一侧，大脑的发散思维、联想能力极强，但执行功能（工作记忆、任务启动、时间管理）不稳定，导致“想法很多，行动困难”（来源：外部执行功能层）。LLM 一侧，模型本身是强大的文本生成核心，但缺乏可靠的内置调度与执行控制，需要“agent scaffolding”——即“围绕 LLM 构建的软件架构和工具，使其能够执行复杂、目标驱动的任务”（来源：ADHD 大脑与 LLM 的同构）。

实证研究进一步揭示了这种同构的神经机制：用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中表现正常，但随序列变长，模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突，这与 ADHD 执行功能缺陷的核心标志一一对应（来源：ADHD 大脑与 LLM 的同构）。ADHD 患者呈现“强记忆、弱控制”的认知剖面：工作记忆容量可能正常甚至超常，但认知灵活性和注意控制存在核心缺陷（来源：外部执行功能层）。

## 人物的 harness：从曾国藩到毛泽东

历史人物早已用肉身实践了这套 harness 工程。曾国藩是典型注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书“他人目下二三行，余或疾读不能终一行”。他的解法是“日课十二条”：黎明即起、读书不二、谨言、写日记反省等。这套系统本质上是一个外部调度层——用固定的时间结构、行为规则和反省机制，来补偿内在的执行功能缺陷。“结硬寨打呆仗”则是用最笨最稳的方法抵消自己的冲动，相当于今天 AI agent 中的“plan-then-execute”模式。

毛泽东同样有 ADHD 特质：小时候是问题儿童，爱读闲书不喜欢四书五经；思维极度跳跃，文章讲话充满比喻；一辈子在动，冲动敢赌（四渡赤水、抗美援朝）。他的 harness 是“民主集中制”和“调查研究”——用集体决策制衡个人冲动，用“没有调查就没有发言权”强制自己获取外部信息后再行动。这对应 LLM agent 中的“human-in-the-loop”和“工具调用”模式：让外部系统（同事、数据）提供决策依据，而不是仅凭内部生成。

两者的共同点：他们都意识到自己的大脑不能完全信赖，于是主动搭建外部脚手架——日课、日记、集体讨论——来保证行动不跑偏。这与 LLM agent 的 scaffolding 完全同构：用外部记忆（向量数据库）补偿无状态，用工具调用（API）补偿知识缺失，用验证循环（test suite）补偿幻觉。

## 今天的 AI 工具：外部执行功能层的实践

这种同构已经催生了具体的 AI 工具。Goblin Tools 的 Magic ToDo 功能能将任何模糊任务（如“清理厨房”）自动分解为可管理的子步骤，用户可调节“辣度”控制分解粒度（来源：Goblin Tools）。这直接对应 LLM agent 中的“任务分解”模式：将复杂目标拆解为原子步骤，降低每一步的启动门槛。

Saner.AI 专注于知识回忆和本地记忆，帮助用户减少搜索循环和标签切换（来源：Saner.AI）。它相当于 LLM agent 中的“外部记忆系统”：用向量数据库存储上下文，避免重复检索。

Motion 则通过 AI 自动排程与动态调整，消除“下一步该做什么”的决策负担（来源：Motion）。这对应 LLM agent 中的“调度器”：根据优先级和截止日期动态规划执行顺序。

## 关键判断：脚手架，不是拐杖

但这里有一个必须指出的边界：这些工具是“脚手架”还是“拐杖”？脚手架是可拆卸的，目的是最终让你自己能建楼；拐杖是永久依赖，取下就倒。当前学界和用户社群对此存在争议（来源：矛盾与存疑）。一方认为 AI 工具作为外部执行功能层补偿缺陷，长期使用可能导致能力退化；另一方认为只要工具设计得当（如逐步减少干预），就能避免依赖。目前缺乏纵向研究来验证，个体差异也极大（来源：矛盾与存疑）。

我的判断是：关键在于工具是否“可调节”。好的脚手架应该像曾国藩的日课——你可以自己设定规则，也可以随时调整规则。而拐杖则是黑箱：你无法理解它为什么这么安排，也无法改变它。因此，选择 AI 工具时，优先选那些允许你自定义规则、逐步减少依赖的，而不是那些替你做完一切决策的。

## 局限与争议

必须诚实承认，ADHD 与 LLM 的同构目前更多是概念类比，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。此外，AI 工具对任务启动的长期效果未知，短期有效但个体差异大（来源：矛盾与存疑）。部分用户反映学习使用 AI 工具本身增加认知负荷（来源：矛盾与存疑）。这些都需要更多跨学科研究和纵向实验。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个你拖了很久的任务**：比如“整理桌面”，调节辣度到你觉得“太细了”的程度，然后按步骤执行。
2. **在 Motion 或类似工具中设置一个自动日程**：把下周所有任务输入，让 AI 自动排程，观察它如何帮你消除决策负担。
3. **模仿曾国藩的日课，建立你的“每日反省协议”**：每天固定时间，用 5 分钟回顾今天哪些事被大脑“劫持”了，哪些外部结构有效。

## 结语

ADHD 创业者与 LLM 工程师面对的是同一道题：如何让一个强大的生成核心，在缺乏内置调度层的情况下，依然可靠地完成复杂任务。答案不是修复核心，而是给它套上 harness——外部记忆、任务分解、动态调度、验证循环。无论是曾国藩的日课，还是 LLM agent 的 scaffolding，本质都是同一套工程思维。理解这一点，你既不会为自己的大脑“缺陷”而自责，也不会为 LLM 的“跑飞”而沮丧——你只是需要更好的脚手架。

# Claude 之于 ADHD，就像 生成核心 + 缺失的执行层 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

## 两种相同的折磨：想搞懂大脑，也想搞懂 LLM

一边是被 ADHD 困扰的人：脑子里想法层出不穷，却总是在「启动」这一步卡住，做事的当下表现像坐过山车——上午能专注三小时，下午连倒水都觉得累。这种波动不是懒，而是大脑的生成核心过于高产，却缺少稳定的执行调度层（来源：ADHD 大脑与 LLM 的同构；采样温度与表现波动）。

另一边是正在做 Agentic Harness 的工程师：LLM 能写出漂亮代码、讲出流畅推理，却会在多步骤任务里漂移、重复犯错，「小错误会累积，非确定性使可重复性复杂化」（来源：AI Agent Systems: Architectures, Applications, and Evaluation）。

这两种折磨，指向同一个结构：**一个高产但缺执行调度层的生成核心**。把 ADHD 大脑和 LLM 并置，不是打比方，而是看同一类系统的同一类故障（来源：ADHD 大脑与 LLM 的同构；生成核心与调度层）。

## 同构在哪里？表现波动与「温度」

ADHD 的表现呈显著日内波动，注意力、执行功能在不同时间点差异极大；诊断也因此需要多次采样（来源：采样温度与表现波动）。LLM 的输出同样受采样温度控制，温度越高越随机，温度越低越确定。生产环境里这种非确定性是可靠性的敌人，所以工程师用「计划-执行」分离、确定性状态转换、周期性采样来稳定它（来源：Planner-Executor Agentic Framework；AI Agents in 2026: Tools, Memory, Evals, and Guardrails；AI Agent Reliability and Guardrails 2026）。

对应关系很直接：
- ADHD 的波动 ↔ LLM 的随机性；
- ADHD 缺乏稳定执行调度层 ↔ LLM 缺乏确定性控制流；
- 外部日程、提醒、环境设计稳定 ADHD 表现 ↔ 确定性工作流、状态机、工具契约稳定 LLM 输出（来源：采样温度与表现波动）。

所以，问题不是「生成核心」不够强，而是它不能自己调度自己。

## 那 Claude 到底补上了哪一层？

Claude 擅长的是生成：写干净代码、把复杂问题用通俗英语解释、做头脑风暴（来源：The 12 Best AI Tools for 2026 (That People Actually Use)）。它补的是**生成质量**这一层，而不是**执行调度**这一层。

但很多人用错了：把 Claude 当成一个会自己规划、提醒、追踪的完整系统。这是把生成核心误认为执行层。Claude 不会替你做日程、不会长期记住你的上下文、不会自动重新排程。那些功能需要外部 harness。

具体能拆成三类工具：

1. **任务分解与启动**：Goblin Tools 的 Magic ToDo 把「清理厨房」这种模糊任务拆成可执行小步骤，降低启动门槛（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。
2. **外部记忆与检索**：Saner.AI 主打本地记忆和知识回忆，减少 ADHD 人群因搜索循环和标签切换带来的认知负荷（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。
3. **动态调度**：Motion 自动根据任务、会议和截止日期生成每日计划，并在被打乱时重新安排，直接对抗时间盲和任务启动困难（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog；The AI Powered SuperApp for Work | Motion）。

在 LLM 一侧，这三件事对应：把 LLM 放进「记忆 + 工具 + 决策逻辑」的控制循环，即 scaffolding；每次模型调用可以访问检索、工具调用和记忆缓冲区（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。也就是说，ADHD 的工具组合和 LLM 的 agent 架构做的是同一套工程。

## 一个真实 harness：杨振宁的「固定锚点」

杨振宁的 ADHD 特质很明显：思维跳跃、物理直觉极强，一眼能看到问题本质（来源：人物案例）。他的 harness 却极其稳定：每天固定时间看书想问题，不做没用的事；和老朋友保持一辈子的学术与情感联结。

这对应到 LLM/agent 架构里，就是**定时 re-grounding**（固定时间重新锚定目标）和**稳定外部上下文**（长期友谊作为持续反馈的「记忆系统」）。他不让自己的直觉在无边界的方向里乱跑，而是用固定日程和稳定关系把它 harness 住。

## 脚手架 vs. 拐杖：边界在哪里？

同构视角的真正价值，是让我们停止「修复缺陷」的幻想，转而设计可拆卸的脚手架。但这里有一个必须诚实的边界：AI 工具可能从脚手架变成拐杖，过度依赖长期效果未知；有些用户甚至反映学习 AI 工具本身增加了认知负荷（来源：矛盾与存疑；拐杖与脚手架；认知负荷）。

判断边界的简单标准：**这个工具是在帮你做你本来永远做不到的事，还是在替代你本该练习的能力？** 如果是前者，它是 harness；如果是后者，它可能是拐杖。好 harness 应该像 LLM 的确定性工作流一样，保留状态机、可审计、可替换。

## 今天就能试的四件事

1. **把 Claude 当生成器，不当项目经理**：用它写草稿、拆思路、解释代码，但把任务调度交给专门工具（如 Motion、Goblin Tools）。
2. **给自己加一个「状态机」**：早晨用 10 分钟重新确认今日优先级（定时 re-grounding），晚上用固定问题复盘（外部执行功能层）。
3. **为 LLM 也做「计划-执行」分离**：把生成任务和验证任务拆成两个节点，像 Planner-Executor 框架那样强制确定性控制流（来源：Planner-Executor Agentic Framework）。
4. **每月做一次「拐杖检查」**：暂时关闭一个 AI 工具，观察自己是更轻松还是更混乱，判断它是脚手架还是依赖。

## 结语

ADHD 人和 LLM 工程师其实面对的是同一个问题：生成核心很强，但执行调度层必须由外部构建。Claude 不是那个调度层，它只是核心的一部分。真正值得建设的，是一套可拆卸的 harness——对大脑、对 agent，都是如此。区别只在于：一个发生在神经回路上，一个发生在 API 调用之间。

# Goblin Tools 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Goblin Tools：它到底补上了哪一层执行功能？

## 问题：ADHD 的“目标漂移”与 LLM 的“上下文膨胀”本质相同

你是否有过这样的经历：打开电脑准备写一篇文章，结果两小时后发现自己正在刷短视频，而文档依然空白？或者，你明明只想查一个资料，却陷入了维基百科的链接深坑，完全忘了最初的目的？这就是 ADHD 大脑的典型困境——**目标漂移**：注意力被无关刺激捕获（分心），或过度聚焦于细节而忽略整体目标（超聚焦）。

有趣的是，LLM agent 在长程任务中也面临类似问题：随着对话上下文增长，模型会逐渐偏离初始指令，产生“上下文膨胀”导致的推理退化，甚至陷入循环。工程师们为此设计了 **agent harness**，通过定时重锚定（re-grounding）——比如在 system prompt 中嵌入检查点、重置上下文——来对抗漂移。

Goblin Tools 正是 ADHD 领域的“重锚定 system prompt”。它把任务分解成可管理的步骤，提供提醒和专注计时器，充当“待办应用、教练和支持性朋友”（来源：AI for ADHD: Best Tools, Apps, and Strategies - Themba Tutors）。但许多人用错了：他们把它当作永久依赖的“拐杖”，而非逐步内化的“脚手架”。

## 同构：从曾国藩的“日课”到 LLM 的“harness”

ADHD 大脑与 LLM 在结构上同构：两者都是强大的生成核心（发散思维、创意、多任务并行），但缺乏可靠的内置执行调度层（来源：AI 与 ADHD 的专注力）。这意味着，它们都需要外部脚手架来弥补调度缺陷。

晚清名臣曾国藩就是典型。他 30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”；读书极慢，“他人目下二三行，余或疾读不能终一行”。他为自己设计了专属 harness——**日课十二条**：黎明即起、读书不二、谨言、写日记反省……这些规则本质上就是定时重锚定：每天固定时间点强制 reset 注意力，用“结硬寨打呆仗”的笨办法抵消冲动。

这与 LLM agent harness 的结构惊人相似：设置护栏（token 预算、工具调用次数上限）、加入人工检查点（暂停并询问后再执行）（来源：Building an AI Agent Harness from Scratch）。曾国藩的“日课”就像 agent 的定时 re-grounding 机制，而他的“秘书”（幕僚）则类似外部调度器，帮助他过滤信息、分配注意力。

## Goblin Tools 到底补上了哪一层执行功能？

Goblin Tools 的核心价值在于补偿 **工作记忆不稳定** 和 **任务启动困难**。它通过 AI 将模糊目标（如“写一篇文章”）自动分解为可执行的子任务，并设置提醒——但关键缺陷在于：“传统提醒系统需要你首先设置提醒，这本身就是一个执行功能任务”（来源：AI Assistant for ADHD: Finally, a Productivity Tool That Requires Less...）。Goblin Tools 降低了这一门槛，但并未完全消除。

相比之下，Focusmate 通过虚拟身体在场（body doubling）提供外部问责，直接绕过执行功能缺陷（来源：Revolutionizing ADHD Management with AI Assistants）。RescueTime 则自动化时间追踪，揭示“时间到底去了哪里”，补偿了 ADHD 的“时间盲”（来源：Revolutionizing ADHD Management with AI Assistants）。Brain.fm 通过神经锁相技术影响大脑状态，帮助进入专注模式（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。

这些工具共同构成一个 **外部执行功能层**（来源：AI 与 ADHD 的专注力），但各有侧重：Goblin Tools 偏重任务分解与提醒，Focusmate 偏重启动与维持，RescueTime 偏重监控与反馈，Brain.fm 偏重状态调节。

## 警惕：脚手架 vs 拐杖的边界

“拐杖与脚手架”概念指出：AI 工具既可能成为永久依赖的拐杖，也可能成为逐步内化的脚手架（来源：AI 与 ADHD 的专注力）。如果 Goblin Tools 让你永远不需要自己思考如何分解任务，那么你的执行功能将永远无法成长。反之，如果你把它当作训练工具——比如先自己尝试分解，再用 AI 检查——它就能帮你内化这种能力。

同样，LLM agent harness 也需要平衡：过度依赖人工检查点会降低效率，太少则导致目标漂移。工程实践表明，最佳方案是 **动态调整**：在关键节点插入重锚定，而日常任务允许自主执行。

## 局限与争议

必须诚实指出，当前证据存在明显局限：
- **临床证据不足**：Brain.fm 和 Endel 虽有神经科学基础，但在 ADHD 人群中的独立随机对照试验极少（来源：AI 与 ADHD 的专注力）。
- **依赖风险未充分研究**：多数工具页面只强调益处，未讨论长期依赖风险（来源：矛盾与存疑）。
- **个体差异巨大**：ADHD 的异质性意味着同一工具对不同亚型效果迥异（来源：AI 与 ADHD 的专注力）。
- **同构性仅为类比**：ADHD 大脑与 LLM 的同构缺乏实证基础，不可过度推广（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个模糊任务**：比如“整理房间”，观察它生成的子步骤，然后尝试自己先列一遍，再对比差异。
2. **设置一个“重锚定”闹钟**：每 25 分钟响一次，问自己：“我现在在做的事，与最初目标一致吗？” 这就是你的个人 system prompt 检查点。
3. **尝试一次 Focusmate 虚拟身体在场**：预约 50 分钟时段，与陌生人一起工作，感受外部问责如何绕过启动困难。
4. **用 RescueTime 追踪一天的时间**：不看报告，先凭记忆估算各活动时长，再对比实际数据——你会直观感受到“时间盲”的威力。

Goblin Tools 不是万能药，但它提供了一个清晰的起点：**把重锚定从意识层面下沉到工具层面**，让外部系统帮你对抗目标漂移。而真正的成长，在于你最终能部分内化这套系统——就像曾国藩从“每天骂自己”到“自律成自然”，就像 LLM agent 从每一步都需要人工确认到能自主完成长程任务。脚手架终可拆除，但前提是你已经学会了如何自己搭建。

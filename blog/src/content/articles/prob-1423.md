# Goblin Tools 之于 ADHD，就像重锚定 system prompt 之于 LLM——但有人用错了

> LLM 跑长任务会漂,工程师的对策之一是把 system prompt 里的目标定期重新注入——不是因为模型没读过,是因为读过的东西会被稀释。Goblin Tools 这个 ADHD 社区口碑最好的小工具,干的就是同一件事:在你即将漂走的时刻,把任务重新「注入」成你接得住的形状。

先介绍主角。Goblin Tools 是一套专为神经多样性用户设计的免费小工具,核心是 **Magic ToDo**:输入一个任务,它自动拆解成子步骤,每个子步骤还能继续拆(辣度图标调节拆解粒度);配套的 Formalizer 改写语气、Judge 解读消息含义、Estimator 估时。它在 ADHD 社区的地位来自一个精准的洞察:**ADHD 的卡点常常不是「做」,是「任务在脑中的表示形式不可执行」**——「收拾厨房」是一团雾,「把水槽里的碗放进洗碗机」是一个动作。Magic ToDo 做的就是把雾重新编译成动作,这恰好是 agent 工程里 planner 的职责;而每次你卡住时回来点一下「再拆细一点」,就是一次 goal re-injection:**把原始目标以更可执行的形式,重新注入当前的执行上下文。**

## 用对的三种姿势

**一、在「启动死锁」时刻用,不是在计划时刻用。** 它最大的价值场景是:你正对着一个任务发呆、明知该做却动不了——此刻打开,输入任务,拆到第一步小于 5 分钟,立刻做那一步。**它是除颤器,不是日程本**:电击瘫痪状态,而不是维护一个宏大的任务库。

**二、用「辣度」找到自己的编译粒度。** 不同的人、同一个人的不同状态,需要的拆解深度不同。状态好时「洗碗→做」就够;状态差的日子,可能需要拆到「走到厨房」这一级。**允许自己在糟糕的日子用最辣的粒度,这不丢人——agent 工程里,越弱的 executor 配越细的 plan,天经地义。**

**三、把 Estimator 的估时当校准器。** 对每个子任务估时,再记录实际用时——几周后你会得到自己的「估时偏差系数」,这是时间盲最缺的那面镜子。

## 用错的三种姿势

**囤积式使用**:把生活里所有任务都倒进去拆解,建成一座宏伟的任务金字塔——然后被金字塔本身吓瘫。**拆解不是执行,列表的完备度和人生的推进度是两个变量**;防法:Magic ToDo 里同时存在的任务不超过 3 个,拆完第一步立刻做。**无限细分逃避**:一个任务反复拆、反复调辣度、追求「完美的分解」——拆解成了新的拖延姿势。防法:拆两轮为上限,第一步能做就停止优化。**只拆不锚**:拆完就关掉,半小时后照样漂走——漏掉了「重锚定」的后半段。防法:把当前第一步写在便签贴在视线里,或者让页面开着占一个屏幕角——**注入的目标要停留在上下文里,才算注入。**

## 边界

同构强度 B 级:任务分解对 agent 的增益有直接研究,ADHD 的任务启动与拆解困难有执行功能文献支撑,Goblin Tools 本身没有独立的对照研究(它是社区验证的工具,不是循证治疗)。声明:工具解决「表示形式」的问题,解决不了所有启动问题——若在完美拆解后依然常态性瘫痪,伴随情绪低落,请专业评估;它是脚手架的优秀样本,但脚手架不替代地基(睡眠、治疗、支持系统)。

## 今天就能试的 3 件事

1. **挑一个躺了一周的任务**:丢进 Magic ToDo,拆到第一步 5 分钟内,现在就做那一步。
2. **测你的辣度**:同一个任务试两档粒度,感受哪档「手知道往哪放」。
3. **立 3 任务上限**:清掉囤积的列表,只留三个正在进行的。

Goblin Tools 之所以在 ADHD 社区封神,不是因为功能多,恰恰是因为功能少:它只做「把目标重新编译成你现在接得住的形状」这一件事——而这一件事,正是 agent 不跑飞的核心机制。用在死锁时刻,拆到能动为止,动了就走:除颤器要贴在胸口,不是供在墙上。

# Reclaim.ai 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Reclaim.ai：用久了会让 ADHD 更强还是更依赖？

## 问题：ADHD 的“边做边忘”与 LLM 的“无状态”是同一类困境

你是否有过这样的体验：打开浏览器想查个东西，结果被通知栏吸引，半小时后发现自己正刷着短视频，完全忘了最初要干嘛？或者，你明明计划好下午写报告，却因为找不到一个文件而陷入搜索循环，最终一无所获？

这正是 ADHD 工作记忆容量小的典型表现——大脑像一台内存不足的电脑，随时可能清空当前任务上下文。而 LLM 也有类似的“先天缺陷”：每次对话开始时，它都是一张白纸，没有跨会话状态，必须靠外部记忆（向量库、检索增强生成）才能“想起”之前聊过什么。

ADHD 大脑与 LLM 在结构上可以被视为同一类系统：底层是一个高产的生成核心（想法不断涌现 / 文本流畅生成），但缺少一个可靠的内置调度层来筛选、组织、启动和坚持任务。两者的解法也因此同构：都需要外部脚手架——对 ADHD 是任务管理工具、日程系统；对 LLM 是 agent harness、记忆模块、规划框架。

## Reclaim.ai 的承诺：自动调度与时间保护

Reclaim.ai 是一款 AI 驱动的日历工具，它通过自动排程、智能时间块和“习惯保护”来帮助用户管理时间。核心功能包括：自动将任务插入空闲时段、根据优先级调整日程、为重要事项预留“不可打扰”时间。它承诺减少 ADHD 用户的时间盲和决策疲劳——你不再需要反复决定“现在该做什么”，因为 AI 已经替你排好了。

从同构视角看，Reclaim.ai 扮演的角色类似于 LLM agent 中的“外部调度器”：它接收用户的目标和约束，然后自动生成执行计划。就像 LLM 依赖检索增强来获取上下文，ADHD 用户依赖 Reclaim 来获取“下一步该做什么”的提示。

## 脚手架还是拐杖？边界在于是否训练调度能力

这里的关键问题是：Reclaim.ai 是帮助我们建立更好的时间管理习惯（脚手架），还是让我们彻底放弃自己调度（拐杖）？

答案取决于使用方式。

**脚手架用法**：把 Reclaim 当作训练工具。例如，你利用它的自动排程来观察自己的时间模式，然后逐渐学会手动规划类似的结构。你可以设定“每天花 10 分钟回顾 Reclaim 的日程，并思考为什么某些任务被推迟”。这类似于孔子用“吾日三省吾身”来训练自己的行为边界——他并非依赖外部规则替代自我控制，而是通过反复练习将外在的“礼”内化为习惯。孔子周游列国 14 年坐不住、冲动爱骂人，但他用“克己复礼”作为行为边界，70 岁才做到从心所欲不逾矩。他的 harness 不是替代调度，而是训练调度。

**拐杖用法**：完全交给 Reclaim，从不反思。你只是机械地执行 AI 的安排，一旦工具出故障或无法访问，就立刻陷入瘫痪。这就像 LLM 依赖外部记忆库却从不学习如何自主管理上下文——当检索系统失效时，模型会完全丢失任务目标。

## 证据与争议：同构视角下的警示

从 ADHD 侧，现有工具如 Goblin Tools 的任务分解、Saner.AI 的知识回忆、Lex 的单一指令触发多步骤，都已被证明能短期降低认知负荷（来源：Best AI Tools for ADHD Productivity in 2026）。但长期效果未知——过度依赖可能导致能力退化（来源：矛盾与存疑）。

从 LLM/agent 侧，研究显示 Transformer 在长上下文任务中出现“强记忆、弱控制”剖面：工作记忆容量大，但认知灵活性与注意控制存在核心缺陷（来源：Strong Memory, Weak Control）。这与 ADHD 的神经心理模式高度一致。同时，Transformer 在 Stroop 冲突任务中注意力随序列变长骤降到随机水平（来源：Deficient Executive Control in Transformer Attention），说明即使有外部记忆，模型自身仍需要训练调度能力。

这些证据共同指向一个结论：无论是 ADHD 大脑还是 LLM，外部脚手架的价值在于**补偿而非替代**。如果使用过程中没有刻意练习调度能力，工具就会从脚手架退化为拐杖。

## 今天就能试的行动

1. **设置“反思锚点”**：在 Reclaim 中每天留出 10 分钟的空闲块，用于回顾当天的自动排程。问自己：“如果今天没有 Reclaim，我能自己排出类似的日程吗？”
2. **逐步减少依赖**：每周选一天关闭 Reclaim 的自动调度，尝试手动规划。就像 LLM 的微调——让模型在无检索环境下练习推理。
3. **结合分解工具**：用 Goblin Tools 的 Magic ToDo 将复杂任务分解成小步骤，然后手动输入 Reclaim。这模拟了 agent 的规划机制：先拆解，再调度。
4. **记录“拐杖时刻”**：当 Reclaim 帮你完成一个你原本无法启动的任务时，记下来；但也要记录那些你因为依赖它而失去主动性的时刻。对比观察边界。

## 诚实局限

本文的观点建立在 ADHD 大脑与 LLM 同构的理论框架上，但目前这仍是类比推理，缺乏直接的神经科学或计算机科学交叉验证（来源：矛盾与存疑）。此外，Reclaim.ai 的长期效果缺乏独立研究，个体差异极大——有人可能从拐杖中受益，有人则可能退化。最终，脚手架还是拐杖，取决于你如何使用它，而非工具本身。

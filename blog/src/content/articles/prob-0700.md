# Focusmate 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Focusmate：它到底补上了哪一层执行功能？

## 问题：为什么 Focusmate 对有些人有效，对另一些人却成了负担？

ADHD 群体中，Focusmate（一个提供实时视频陪练、互相监督的在线平台）口碑两极分化。有人靠它完成了拖延数月的报告，有人却觉得它只是多了一个“看着你发呆”的摄像头。同样，在 LLM/Agent 工程中，function calling 工具调用被吹捧为“让 AI 真正干活”的关键，但很多团队发现，不加设计地接入大量工具，反而让 agent 陷入“工具选择瘫痪”或“上下文膨胀”。这两个现象背后，是同一个结构性问题：**你是在用脚手架辅助执行，还是在用拐杖替代思考？**

## 同构视角：ADHD 大脑与 LLM 共享同一个核心缺陷

最新的认知科学研究揭示了一个惊人的同构：ADHD 大脑与 LLM（大语言模型）都是“强记忆、弱控制”的生成核心。明尼苏达大学的系统测试发现，LLM 在工作记忆容量上甚至超过常人，但认知灵活性与注意控制存在核心缺陷——无法灵活切换任务集、无法抑制自动化反应，这正是 ADHD 的经典神经心理模式（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。耶鲁大学进一步发现，自注意力机制本身导致工作记忆容量限制：随任务复杂度增加，注意力分数熵增、注意力弥散，与人类注意分散机制同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。

这意味着，ADHD 和 LLM 的核心瓶颈都不在“算力”或“容量”，而在 **orchestration（编排）层**——即如何调度已有的资源来执行一个连贯的任务序列。对于 ADHD，这个调度层是执行功能；对于 LLM，它是 prompt 工程和工具调用架构。两者都需要一个外部的“脚手架”来补偿内在的调度缺陷。

## Focusmate 与 function calling：同一个脚手架，两种用法

Focusmate 的核心机制是“身体在场效应”——一个活人（哪怕只是视频画面）通过社会承诺和实时监督，帮助 ADHD 大脑克服任务启动困难和时间盲。这与 function calling 中工具调用的本质完全一致：**将精确、重复、状态维护类任务外包给外部系统，让生成核心专注于创造和决策**。Focusmate 是一个“外部执行调度层”——它提供启动信号、持续监督和完成反馈，补上了 ADHD 大脑缺失的那部分执行功能。

但问题在于，很多人把 Focusmate 当成了“万能启动器”：只要打开摄像头，就期待自己能自动进入心流。这就像 LLM 工程中，不加设计地接入 20 个 API 工具，期待 agent 自动调用正确的那个。结果往往是：**脚手架变成了拐杖，甚至变成了新的干扰源**。用户报告显示，部分 ADHD 用户对 Focusmate 产生依赖，没有它就无法启动任务，这与“过度依赖 AI 工具可能削弱内在能力”的警告一致（来源：矛盾与存疑）。

## 脚手架 vs 拐杖：边界在哪里？

关键区别在于：**脚手架是临时性、可拆卸的支撑结构，旨在帮助用户发展内在能力；拐杖是永久性替代，一旦撤除，功能立即崩塌**。对于 ADHD 用户，Focusmate 应该是“训练执行功能的健身房”，而不是“永远离不开的轮椅”。同样，对于 LLM agent，function calling 应该是“扩展能力的工具箱”，而不是“掩盖 prompt 设计缺陷的补丁”。

有效的使用方式遵循同一个原则：**最小化外部依赖，最大化内在迁移**。例如，Goblin Tools 的 Magic ToDo 将任务分解为小步骤，但用户需要逐渐学习自己识别任务结构（来源：Goblin Tools）。Saner.AI 通过知识回忆减少搜索循环，但用户需要练习主动回忆而非被动检索（来源：Saner.AI）。Lex 的单一指令触发多步骤，但用户需要理解自己的任务模式，才能设计出有效的指令（来源：Lex）。这些工具都提供了“脚手架”，但最终目标是将调度能力内化。

## 诚实面对局限：证据不足与个体差异

必须承认，目前所有关于 ADHD 与 AI 工具的证据主要来自用户报告和概念类比，缺乏大规模随机对照试验（来源：矛盾与存疑）。Focusmate 的有效性同样如此——没有独立研究证明它比普通番茄钟更有效。个体差异极大：有些人需要“社会在场”才能启动，有些人则会被额外的人际压力压垮。工具设计者声称是“脚手架”，但实际使用中可能沦为“拐杖”，而如何避免依赖的具体指导几乎为零（来源：矛盾与存疑）。

## 今天就能试的行动

1. **设置使用边界**：使用 Focusmate 时，每次只承诺 25 分钟，完成后立即关闭摄像头，强制自己独立工作 5 分钟。这能训练“启动-维持-停止”的循环。
2. **工具组合实验**：尝试用 Goblin Tools 分解一个任务，然后用 Focusmate 执行第一步。记录启动时间和完成率，对比单独使用任一工具的效果。
3. **设计“降级方案”**：为每个 AI 工具准备一个非 AI 替代方案。例如，如果不能用 Focusmate，就用一个实体番茄钟+一张纸质的任务清单。这能防止过度依赖。
4. **每周一次“无工具日”**：选择一天完全不用任何外部执行功能工具，只靠自己的意志力和策略完成任务。这能检验内化程度，并暴露真正的薄弱环节。

## 结语

Focusmate 与 function calling 的类比不是巧合，而是认知架构的深层同构。理解这一点，就能避免“用错了”的陷阱：**工具的价值不在于它替代了什么，而在于它教会了你什么**。对于 ADHD 用户和 LLM 工程师，真正的挑战不是找到更好的工具，而是学会设计可拆卸的脚手架。

# Focusmate 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Focusmate：它到底补上了哪一层执行功能？

## 问题：为什么“精确、重复、状态维护”类任务总是卡住？

ADHD 大脑常被描述为“高产生成核心，但调度层欠费”——想法如泉涌，但启动任务、维持节奏、记住细节却像在泥潭里挣扎。同样，LLM 能流畅生成文本，却在长程推理、工具调用中频繁“走神”：上下文一长，注意力就弥散（来源：Deficient Executive Control in Transformer Attention）。两边共享同一类困境：**瓶颈不在算力，而在 orchestration 层**（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

工程师们用 function calling 为 LLM 搭“脚手架”：把精确计算（如数学、数据库查询）外包给专用工具，让生成核心只负责调度与输出。那么，ADHD 大脑能否也找到一个“外部调度器”，把“状态维护”这类精确重复的任务外包出去？

## 答案：Focusmate 就是 ADHD 的 function calling

Focusmate 是一个视频配对平台：你和另一个人约定时间，打开摄像头，各自工作，互相监督。表面看，它只是“有人陪你学习”。但深入分析其机制，会发现它恰好补上了 ADHD 执行功能中缺失的一层：**外部执行调度层**。

- **启动**：约定时间 + 摄像头一开，任务启动的认知负荷瞬间降低。这就像 LLM 通过 function calling 触发一个外部工具，无需自己从头规划。
- **维持**：对方的存在提供“社会在场”，抑制了注意力漂移。这与 Transformer 中自注意力机制随序列变长而熵增、注意力弥散（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）形成镜像——外部“锚点”强制注意力聚焦。
- **状态维护**：你不需要自己记住“接下来该做什么”，因为 session 结构替你维护了上下文。这对应 LLM 的“强记忆、弱控制”剖面：工作记忆容量虽大，但缺乏灵活调度（来源：Strong Memory, Weak Control）。

孔子晚年用“礼”作为自己的外部调度层：每日“三省吾身”，用外在仪式约束冲动（来源：wiki 人物案例）。他的“克己复礼”本质上是一套个人 harness——把道德判断外包给“礼”这个外部框架，让生成核心（思想）在轨道上运行。这就像 agent 用 function calling 把算术外包给计算器：不消耗核心注意力，只做决策。

## 但有人用错了：Focusmate 不是“陪学”，是“脚手架”

许多 ADHD 用户把 Focusmate 当作“社交学习工具”，期待对方提供陪伴或监督。但它的真正价值在于：**它提供了一个结构化的外部调度层，而不是一个“拐杖”**。如果你依赖对方来提醒你“该做什么”，那它就是拐杖——一旦对方不在，你又会瘫痪。

真正的用法是：**把 Focusmate session 当作一个“函数调用”**。你预先定义好 session 的目标（比如“写 500 字”），然后让 Focusmate 负责执行“启动-维持-结束”这个流程。你的大脑只需要输出内容，调度交给外部。

这与 Goblin Tools 的 Magic ToDo 异曲同工：它把“清理厨房”分解成 10 个小步骤，你只需按顺序执行，无需自己规划（来源：Goblin Tools）。Saner.AI 则通过本地记忆减少搜索循环，相当于给 LLM 外挂一个向量数据库（来源：Saner.AI）。Lex 的“单一指令触发多步骤”则像 agent 的规划机制（来源：Lex）。

## 局限与争议

“脚手架 vs 拐杖”的边界并不清晰。长期使用 Focusmate 是否会导致“社交依赖”，一旦没有配对就无法工作？wiki 资料指出，缺乏对比研究，个体差异大，尚无定论（来源：矛盾与存疑）。同样，LLM 过度依赖 function calling 也可能削弱自身的推理能力。但至少，两者共享同一条原则：**外部工具应增强而非替代核心能力**。

## 今天就能试的行动

1. **用 Focusmate 做一次“函数调用”**：打开 Focusmate，session 前写下唯一目标（如“完成第一段”），session 中只执行，不思考。
2. **对比 Goblin Tools 的“辣度”**：把“整理书桌”输入 Magic ToDo，调到最辣（最细粒度），然后按步骤执行——感受外部调度层如何降低启动阻力。
3. **模仿孔子的“日课”**：每天固定时间（如睡前）用 Reflect 或纸笔录 3 条“今天做了什么”，形成外部反思节奏（来源：Reflect）。
4. **为 LLM 搭一个最小 function calling**：用 Python 写一个调用天气 API 的函数，让 LLM 通过它回答天气问题——体验“生成核心+外部工具”的协作模式。

# 如果把 ADHD 大脑当成生成核心，RescueTime 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 RescueTime 对 ADHD 的真正价值。

## 问题：边做边忘，ADHD 的“无状态”困境

你打开电脑准备写一份报告，刚敲了两行字，突然想起要回一封邮件；点开邮箱，又瞥见一条新闻推送；等回过神来，半小时过去了，报告还停在第一段。这种“边做边忘”的体验，对 ADHD 人群来说不是偶尔的走神，而是日常的默认模式。

ADHD 的核心困境并非智力不足，而是执行功能（executive function）的失效——工作记忆容量小、任务启动困难、时间盲（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。最新研究进一步揭示了其神经机制：Transformer 自注意力机制在长上下文下冲突解决能力崩溃至随机水平，这与 ADHD 执行功能障碍的核心神经机制一致（来源：Deficient Executive Control in Transformer Attention）。换句话说，ADHD 大脑就像一个高产的生成核心——想法丰富、联想活跃——但缺少稳定、可靠的调度层来筛选、组织、启动和坚持任务。

## 同构：ADHD 大脑与 LLM 都是“高产但缺调度”的生成核心

如果你熟悉大语言模型（LLM）的工程实践，你会发现一个惊人的相似：LLM 本身是无状态、仅生成文本的核心，需要外部“脚手架”和“harness”来提供工具接口、上下文管理、安全执行等编排能力（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。现代 AI 编码代理通过“复合 AI 系统架构”来弥补这一缺陷，包括“工作负载专用模型路由、分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这些技术本质上是在 LLM 外部搭建调度层，防止“上下文膨胀和推理退化”。

ADHD 大脑与 LLM 在结构上可被视为同类系统：底层都拥有一个高产生成核心，但都缺乏可靠的内置执行调度层。两者都需要外部脚手架——ADHD 一侧的脚手架可能是行为习惯、外部工具，LLM 一侧的脚手架则是 agent harness、上下文管理系统。

## 人物案例：孔子的“克己复礼”与 LLM harness 的同构

孔子是 ADHD 特质的典型代表：身高近1米9，精力旺盛，周游列国14年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予“朽木不可雕”；对音乐超专注到“三月不知肉味”，对种地等俗事零耐心；思维跳跃，《论语》全是场景化语录，无系统著作。他的专属 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。70岁才做到“从心所欲不逾矩”，一辈子和自己的冲动做斗争。

孔子的“日课”对应 LLM harness 中的“定时 re-grounding”——定期将上下文刷新到目标轨道；他的“礼”对应“外部调度器”——用一套固定仪式约束生成核心的随机发散。两者的同构在于：都承认内核无法自控，所以在外围建立一套可维护的约束系统。

## RescueTime：ADHD 的“无状态监控”与“外部记忆”

回到核心问题：RescueTime 能不能当 ADHD 大脑的无状态与外部记忆？

RescueTime 是一款后台运行的时间追踪工具，它自动记录你花在每个应用和网站上的时间，生成报告，并允许你设置目标与提醒。从 agent harness 的视角看，它扮演了两个角色：

1. **无状态监控**：LLM 每次对话都是全新的，没有跨会话的“状态”。RescueTime 不依赖你的工作记忆——你不需要记住自己做了什么，它自动记录。这恰好匹配 ADHD 大脑的“无状态”特性：工作记忆容量小，边做边忘，无法靠内在维持连贯的自我监控。

2. **外部记忆**：ADHD 患者常面临工作记忆不足，导致频繁的搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。Saner.AI 通过知识回忆和本地记忆来补偿这一缺陷（来源：Same source）。RescueTime 的历史报告提供了类似的外部记忆——你可以在一天结束时查看时间轴，回答“我今天到底做了什么？”，而不必依赖大脑的碎片化回忆。

但 RescueTime 的局限也很明显：它只记录，不分解、不启动。Goblin Tools 的 Magic ToDo 能将复杂任务分解为可管理的小步骤（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD），Lex 允许通过单一指令触发多步骤任务序列（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar），而 RescueTime 缺乏这些主动调度能力。

## 脚手架 vs 拐杖：RescueTime 的边界

AI 工具究竟是“外部执行功能层”还是“拐杖”？这一争议在 RescueTime 上同样存在（来源：矛盾与存疑）。RescueTime 作为被动监控工具，更像“脚手架”——它提供结构，但决策权在你。你仍然需要自己判断“该做什么”，只是 RescueTime 帮你回答了“我做了什么”。这避免了过度依赖导致的能力退化，因为用户必须主动利用数据调整行为。

然而，对于严重执行功能障碍的用户，被动监控可能不够强力。部分 ADHD 用户反馈，强行打断（如番茄钟）反而有效，而柔性引导可能不够（来源：矛盾与存疑）。RescueTime 的提醒功能（如“你已经在社交媒体上花了30分钟”）是一种温和的打断，但缺乏强制力。

## 核心观点：RescueTime 是 ADHD 大脑的“上下文管理模块”，而非完整 harness

我的判断是：RescueTime 对 ADHD 的真正价值不在于它“管理时间”，而在于它提供了 LLM harness 中“上下文管理”这一关键组件。在 agent 工程中，上下文管理负责维护当前状态、历史记录与目标对齐，防止生成核心跑偏。RescueTime 的时间轴与目标设定，正是 ADHD 大脑缺失的“上下文快照”——它让你在注意力漂移后能迅速 re-ground 到目标轨道。

但要构成完整的 harness，还需要任务分解（Goblin Tools）、启动触发器（Lex）和外部记忆（Saner.AI）。RescueTime 是拼图中的一块，而非全部。

## 今天就能试的行动

1. **设置 RescueTime 的“聚焦时间”**：每天划出2小时，屏蔽干扰应用，让外部系统帮你维持上下文。
2. **结合 Goblin Tools 的 Magic ToDo**：用 RescueTime 识别你的“时间黑洞”任务，然后用 Goblin Tools 将其分解为3-5个小步骤，贴在显眼处。
3. **每日回顾**：花5分钟查看 RescueTime 报告，写下“我今天实际做了什么 vs 计划做什么”，强化自我认知。
4. **尝试 Lex 的单一指令**：如果你经常卡在任务启动，用 Lex 将“写报告”拆解为“打开文档-写大纲-写第一段-休息5分钟”，一键触发。

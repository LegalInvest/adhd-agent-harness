# Tiimo 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Tiimo：用久了会让 ADHD 更强还是更依赖？

## 从同一个问题出发：为什么“边做边忘”如此致命？

如果你是一名 ADHD 患者，你一定熟悉这个场景：打开电脑准备写报告，三分钟后却发现自己正在刷手机，完全不记得刚才要做什么。如果你是一名构建 LLM Agent 的工程师，你也一定熟悉另一个场景：你的 Agent 在对话中生成了一段精彩分析，但下一次会话时它已经彻底失忆，需要你重新注入上下文。

这两个场景共享同一个底层问题：**工作记忆容量小，且缺乏跨会话的状态保持**。ADHD 大脑的“生成核心”（创造力、联想能力）极其强大，但执行调度层——负责计划、组织、工作记忆的[[执行功能]]——常常失灵（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。LLM 同样如此：它的生成能力惊人，但天然无状态，每次对话都是“全新开始”，需要外部记忆系统（如向量数据库、提示词模板）来维持连续性。

正是这种结构上的同构，让 ADHD 与 AI 工具的结合成为天然的研究场域。而 Tiimo，作为一款专为时间盲设计的视觉时间表工具，恰好站在了这场讨论的中心。

## Tiimo 的设计：针对“时间盲”的视觉脚手架

[[时间盲]]是 ADHD 最令人沮丧的症状之一——患者无法感知时间流逝，导致迟到、拖延和计划失败（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。传统计划工具（如纸质日历或普通待办清单）往往无效，因为它们依赖用户主动管理时间，而这正是 ADHD 大脑做不到的。

Tiimo 的解法是**视觉化时间规划**：它将一天的任务以彩色时间轴的形式呈现，让抽象的时间变得可见、可触摸。用户不需要在脑中估算“这件事要多久”，而是直接看到“上午10点到11点做A，11点到12点做B”。这种设计直接绕过了[[执行功能]]中的时间感知缺陷，将外部时间锚点嵌入用户的环境（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。

从同构视角看，Tiimo 相当于 LLM Agent 的**上下文窗口管理**——它把“当前会话还剩多少 token”转化为可视化的进度条，让 Agent（或大脑）不必自己计算剩余容量。这就像给一个无状态的 LLM 配上了一个实时更新的“记忆使用量仪表盘”。

## 脚手架还是拐杖？核心判断

现在回到本文的核心问题：Tiimo 之于 ADHD，是脚手架还是拐杖？

我的判断是：**它可以是脚手架，但当前的设计更偏向拐杖**。

**脚手架**是指一种可逐步撤除的临时支撑，它在用户能力提升后可以逐渐移除。例如，一个 ADHD 患者通过 Tiimo 的视觉时间表逐渐学会感知时间，最终可能只需要一个简单闹钟就能维持节奏。**拐杖**则是一种永久替代，一旦撤除，用户的功能立刻崩溃。

Tiimo 的设计中，几乎没有提供“撤除路径”。它鼓励用户长期依赖其视觉界面，而不是内化时间感知能力。这与[[拐杖与脚手架]]概念中的理想状态相悖——脚手架应设计为可逐步撤除的临时支撑，但多数工具（包括 Tiimo）都未提及撤除机制（来源：矛盾与存疑）。

相比之下，[[Goblin Tools]] 的 Magic ToDo 功能更接近脚手架：它将任务分解为小步骤，用户可以通过反复使用学会自己分解任务。而 Tiimo 的视觉时间表更像一个永久的外部时钟，用户一旦离开它就回到时间盲状态。

## 证据与局限：我们必须诚实面对

必须承认，目前关于 Tiimo 的独立临床研究极为匮乏。现有证据主要来自用户评价和综述文章，缺乏随机对照试验验证其对 ADHD 核心症状的长期改善（来源：矛盾与存疑）。此外，ADHD 的异质性极高——对某些用户，Tiimo 可能是救命稻草；对另一些用户，它可能因界面复杂反而增加认知负荷。

从 LLM 侧看，类似的问题也存在。Agentic Harness 工程中，外部记忆系统（如向量库）同样面临“过度依赖”风险：开发者容易把所有状态都塞进外部存储，导致 Agent 自身永远学不会保持上下文。这与 ADHD 患者依赖 Tiimo 而放弃内化时间感知，是同一种模式。

## 今天就能试的 3 条行动

1. **设置“撤除实验”**：使用 Tiimo 两周后，尝试一天不用它，记录自己的时间感知准确度和焦虑水平。这能帮你判断它是脚手架还是拐杖。
2. **结合 Goblin Tools 进行任务分解**：用 Tiimo 规划时间，但用 Goblin Tools 的 Magic ToDo 分解任务（来源：Goblin Tools）。这种组合能覆盖时间盲和任务启动两个痛点。
3. **为你的 LLM Agent 设计“撤除机制”**：如果你是工程师，在 Agent 的提示词中加入“请尝试在对话中记住关键信息，减少外部查询次数”，模拟脚手架的内化过程。

## 结论：同构视角下的工具哲学

ADHD 大脑与 LLM 的同构，揭示了 AI 工具设计的核心矛盾：**工具越有效，越容易成为拐杖**。Tiimo 的视觉时间表是天才的设计，但它需要配套的“能力转移计划”才能真正成为脚手架。同样，Agentic Harness 工程不能只关注外部记忆的容量，还要设计让 Agent 学会利用自身上下文窗口的机制。

真正的脚手架，是那些能让你最终不再需要它的工具。而 Tiimo，目前还差这一步。

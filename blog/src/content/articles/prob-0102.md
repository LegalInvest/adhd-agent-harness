# Tiimo 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Tiimo：用久了会让 ADHD 更强还是更依赖？

## 问题：Tiimo 是脚手架还是拐杖？

对 ADHD 人群来说，Tiimo 这类视觉计时器与日程工具，究竟是帮大脑搭建外部执行功能的脚手架，还是让人越用越依赖的拐杖？对 LLM/Agent 工程师而言，这个问题等价于：给模型套上 vector store 和 tool use 的 harness，是让它更强，还是让它丧失自主推理能力？两个问题共享同一个底层结构——**ADHD 大脑与 LLM 都是高产生成核心，但缺乏可靠的内部调度层**（来源：ADHD 大脑与 LLM 的同构）。

## 同构脊柱：无状态与外部记忆

ADHD 的工作记忆容量小、边做边忘，是执行功能的核心瓶颈（来源：工作记忆）。LLM 同样无跨会话状态，靠外部记忆（向量库）和工具脚手架续命。明尼苏达大学的研究发现，LLM 呈现“强记忆、弱控制”剖面——工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷，这正是 ADHD 的经典神经心理模式（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。耶鲁大学进一步揭示，自注意力机制本身导致工作记忆容量限制，注意力分数随任务负荷增加而弥散，与人类注意分散机制同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。这意味着，**瓶颈不在容量/算力，而在 orchestration 层**。

## 人物 harness 的启示

孔子是 ADHD 特质鲜明的例子：精力旺盛、冲动、思维跳跃，无系统著作。他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”）。这本质上是一个**外部调度系统**：日课相当于 LLM 的定时 re-grounding，礼制相当于外部约束规则。老子则用“无为”作为 harness：不强行做事，把能量收回来聚焦在道上。这与 LLM agent 的“最小干预”策略类似——让模型自由生成，但通过 prompt 框架约束输出范围。两人的 harness 都**不是替代大脑，而是为生成核心提供轨道**。

## Tiimo 的定位：脚手架而非拐杖

Tiimo 的核心功能是视觉化时间块和日程提醒，补偿 ADHD 的“时间盲”和任务启动困难。这与 Goblin Tools 的任务分解（来源：Goblin Tools）、Saner.AI 的知识回忆（来源：Saner.AI）、Lex 的单一指令触发多步骤（来源：Lex）同属外部执行功能层。但脚手架与拐杖的边界在于：**工具是否在训练内在能力，还是完全替代它**。例如，Focusmate 的虚拟身体加倍通过社会存在感降低分心阈值，定期问责检查可将目标达成可能性从 25% 提升至 95%（来源：身体在场效应）。这种外部问责如果逐渐内化为自我监控，就是脚手架；如果永远依赖他人（或 AI）在场才能专注，就是拐杖。

## 证据与局限

目前缺乏 Tiimo 长期依赖风险的纵向研究（来源：矛盾与存疑）。用户反馈显示，AI 工具短期有效，但学习工具本身可能增加认知负荷（来源：矛盾与存疑）。个体差异极大：有人需要强力打断（如番茄钟），有人需要柔性引导。因此，**Tiimo 是脚手架还是拐杖，取决于用户是否主动将其作为能力训练工具，而非被动依赖**。

## 今天就能试的行动

1. **设置一个“日课”式 Tiimo 日程**：每天固定时间段使用 Tiimo 规划，两周后尝试减少依赖，看能否自主保持节奏。
2. **用 Tiimo 做“单任务块”**：只设一个 25 分钟任务，完成后自我评估是否进入超聚焦——若进入，下次延长；若分散，调整粒度。
3. **结合身体在场**：打开 Focusmate 或请朋友视频在线，与 Tiimo 计时同步，形成双重外部锚点。

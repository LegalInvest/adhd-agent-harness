# ADHD 创业者视角：为什么「治好 ADHD 的任务启动困难、不会拆解」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 用 planner-executor 循环分解长任务——同一套 harness 思路，两边都成立。

## 问题：两个世界的同一堵墙

如果你是一个 ADHD 创业者，你一定经历过这样的场景：坐在电脑前，脑子里有一个绝妙的商业计划，但就是无法开始写第一行代码或第一封邮件。任务像一座山，你站在山脚，腿像灌了铅。同时，如果你在搭建一个 LLM agent，你也会遇到类似的崩溃：给模型一个复杂的长任务，它要么在第一步就偏离轨道，要么陷入无限循环，输出一堆看似合理但毫无意义的文本。

这两个问题看起来分属心理学和工程学，但本质上是同一道题：**如何让一个高产但缺乏执行调度层的生成核心，可靠地完成长任务？**

## 同构：ADHD 大脑与 LLM 共享的架构缺陷

ADHD 大脑的核心困境是：**工作记忆容量保留，但控制弱**（来源：[[工作记忆]]）。你能记住 7±2 个信息块，但无法阻止无关信息闯入，也无法稳定地按顺序调用它们。LLM 同样如此：它的上下文窗口可以容纳数千 token，但注意力机制会受近因效应和统计偏差影响，在长序列中遗忘早期指令或生成幻觉（来源：Human-like Working Memory Interference in Large Language Models）。

更深层的同构在于**执行功能层的缺失**。认知神经科学发现，前额叶-纹状体门控机制负责选择性更新工作记忆内容；而 Transformer 在训练后，其自注意力层会自发模仿这种门控操作（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。这意味着，两者都依赖一个内置的“调度器”来管理信息流——但 ADHD 大脑的这个调度器天生不稳定，LLM 的这个调度器则完全由训练数据塑造，缺乏在线自适应能力。

结果就是：ADHD 创业者面对一个复杂项目时，大脑会陷入“自动模式”——要么跳入细节，要么逃避到其他任务，就像 LLM 在没有外部引导时，会沿着最可能的 token 路径自动生成，哪怕偏离了原始目标（来源：Automatic Minds: Cognitive Parallels Between Hypnotic States and LLM Hallucinations）。

## 解法：planner-executor 循环作为 harness

工程上解决 LLM 跑飞的标准方案是 **planner-executor 循环**：一个 planner 将长任务分解为子步骤，一个 executor 依次执行，每步结果反馈给 planner 调整下一步（来源：The Anatomy of an AI Agent: Memory, Tools, Planning, and Execution Explained）。这本质上是一个外部执行功能层——harness。

对于 ADHD 大脑，这个 harness 同样有效。工具如 **Motion** 和 **Reclaim.ai** 就是 planner-executor 的外部实现：Motion 自动将任务按优先级和截止日期排入日程，并在被打断时动态调整（来源：Motion）；Reclaim.ai 则保护深度工作块，防止会议碎片化时间（来源：Reclaim.ai）。两者都消除了“下一步做什么”的决策负担——这正是 ADHD 任务启动困难的核心。

更具体地，**Tiimo** 和 **Structured** 将时间转化为视觉元素，补偿时间盲（来源：Tiimo；Structured）。**Todoist** 的智能优先级排序充当外部工作记忆，减少认知负荷（来源：Todoist）。这些工具的共同逻辑是：**将内部不可靠的执行调度，外包给外部可靠的循环系统。**

## 核心观点：脚手架 vs 拐杖的边界

我的核心判断是：**AI harness 的有效性取决于它是否保留了用户的“攀登”动作。** 如果工具完全替代了规划过程（比如自动安排所有任务而不让用户参与），它就是拐杖——长期使用会削弱内在能力（来源：[[拐杖与脚手架]]）。如果工具提供结构但要求用户主动选择、调整和反思，它就是脚手架——帮助用户建造自己的执行功能。

例如，Motion 的自动排程对启动困难极有帮助，但如果用户从不审视自己的优先级，就会失去对时间的感知。Reclaim.ai 的保护块很好，但用户仍需决定哪些任务值得保护。真正的 harness 应该像 LLM agent 的 planner-executor 循环：planner 分解任务，executor 执行，但人类在循环中担任“监督者”，根据反馈修正方向。

## 争议与局限

必须诚实指出，目前证据主要来自用户报告，缺乏随机对照试验（来源：[[AI 与 ADHD 的时间管理]]）。**过度依赖风险**是真实存在的：工具可能成为永久拐杖，削弱内在时间感知和规划能力（来源：[[矛盾与存疑]]）。此外，个体差异巨大——现有工具多面向注意力缺陷型 ADHD，对多动冲动型效果不明（来源：同上）。同构理论本身仍是类比，神经科学与计算机科学的交叉验证不足（来源：同上）。

## 今天就能试的行动

1. **用“planner-executor”思维拆解一个卡住的任务**：写下任务目标，然后让 ChatGPT 或 Claude 帮你分解为 3-5 个子步骤（planner），再逐个执行（executor），每步结束后反馈给 AI 调整下一步。这模拟了 agent 的循环，也训练了你的分解能力。

2. **设置一个“外部工作记忆”**：用 Todoist 或任何任务管理工具，把当前项目的所有关键信息（背景、下一步、截止日期）写在一个固定位置，每天开始工作时先查看，避免从零回忆。

3. **尝试视觉化时间块**：用 Tiimo 或 Structured 把一天的时间块画出来，哪怕只是纸笔。这直接补偿时间盲，让你看到时间而非感受时间。

4. **警惕“自动排程”陷阱**：如果使用 Motion 或 Reclaim.ai，每周花 10 分钟手动审视一次日程，问自己：“这些任务真的重要吗？” 保持主动控制，而非完全交给 AI。

## 结语

ADHD 大脑和 LLM 是同一类引擎：强大但需要外部轨道。理解这种同构，不仅能让你更好地使用 AI 工具，还能让你以工程师的视角看待自己的大脑——不是缺陷，而是一个需要正确 harness 的生成核心。当你下次启动困难时，记住：你不是在“对抗”大脑，而是在为它编写一个更好的 planner-executor 循环。

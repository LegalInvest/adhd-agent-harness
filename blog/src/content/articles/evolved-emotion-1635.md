# 为什么用 Motion 治 ADHD 的情绪失调，和给 agent 套 会褪去的脚手架 是一回事？

> Motion 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

## 问题：为什么你的情绪像失控的 LLM——输出很强，但调度全无？

如果你是一个 ADHD 成人，你一定熟悉这种体验：情绪像突然被点燃的烟花——因为一句批评瞬间暴怒，或者因为一个小挫折陷入数小时的自我否定。你明明知道不该这样，但就是拉不住。

如果你是一个做 Agentic Harness 的工程师，你一定见过这种场面：LLM agent 生成了完美的计划，却在执行第三步时跑偏，或者在上下文膨胀后忘记最初的目标，输出变得杂乱无章。

这两件事看起来毫不相关，但本质是同一个问题：**一个高产的生成核心，缺少一个可靠的执行调度层**。

ADHD 大脑的情绪生成系统（如杏仁核）反应强烈，但前额叶（执行功能中枢）调控不足（来源：AI 与 ADHD 的情绪调节）。LLM 的生成能力极强，但缺乏对自身输出的实时控制。两者都是“强生成、弱调度”的架构。

## 答案：Motion 与“会褪去的脚手架”是同构解法

Motion 是一款 AI 日历工具，它通过自动排程帮助 ADHD 用户克服时间盲和任务启动困难（来源：Motion）。它把你的任务、会议、截止日期全部塞进一个智能时间表，动态调整优先级。本质上，Motion 成为了你的外部调度器——它替你完成“接下来该做什么”这个执行功能。

在 LLM agent 领域，这被称为“scaffolding”（脚手架）：一套临时、可褪去的外部结构，用来补偿 agent 在执行层面的缺陷。比如给 LLM 加上一个“记忆模块”来补偿无状态性（来源：无状态与外部记忆），或者用“上下文工程”来聚焦当前任务（来源：上下文工程）。

Motion 之于 ADHD 大脑，就像 scaffolding 之于 LLM agent：都是为“缺调度层”的核心搭建一个临时、可褪去的外部执行功能层。

## 证据：两边都成立

### ADHD 侧：工具即脚手架

- **Inflow**：基于 CBT 的结构化程序，通过训练工作记忆和认知控制来改善 ADHD 症状（来源：Inflow）。它教用户建立技能，而不是永远依赖应用。这符合“脚手架”逻辑——最终目标是用户自己掌握情绪调节能力。
- **Goblin Tools**：Magic ToDo 功能将模糊任务（如“清理厨房”）自动分解为具体步骤，用户可调节“辣度”控制粒度（来源：Goblin Tools）。这降低了任务启动门槛，减少因启动困难引发的挫败感（来源：AI 与 ADHD 的情绪调节）。
- **Saner.AI**：通过本地记忆和快速检索减少搜索循环和标签切换，补偿工作记忆缺陷（来源：Saner.AI）。

这些工具的共同点是：它们不是“永久拐杖”，而是“会褪去的脚手架”——当你内在的执行功能逐渐建立后，可以逐步减少使用。

### LLM/Agent 侧：同一套思路

明尼苏达大学的研究发现，LLM 的执行功能剖面是“强记忆、弱控制”——工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷（来源：工作记忆）。这正是 ADHD 的经典神经心理模式。耶鲁大学进一步发现，自注意力机制本身导致工作记忆容量限制，注意力分数熵增、注意力弥散，与人类注意分散机制同源（来源：工作记忆）。

因此，给 LLM agent 套脚手架（如外挂记忆、任务分解、re-grounding 机制）与给 ADHD 大脑套 Motion/Inflow，是同一个问题的两个侧面。

### 人物案例：释迦牟尼的“八正道”脚手架

释迦牟尼29岁放弃王位出家求道，先后跟两个仙人学、苦行6年，不断试错不断放弃，49年一辈子游行说法，80岁死在传道路上。他的专属 harness 是“八正道”和“持戒”——用戒律管自己的行为，用正念拴住乱跑的念头。他强调“自依止法依止”，不依赖权威，自己证悟。

这正是一个“会褪去的脚手架”系统：八正道是行为层面的外部结构，正念是注意力层面的 re-grounding 机制，而最终目标是“自依止”——内在的调度能力。与 LLM agent 的 scaffolding 同构：日课（定时 re-grounding）↔ 八正道中的正命、正业；正念↔ 注意力聚焦机制；不依赖权威↔ 最终褪去脚手架。

## 核心观点：脚手架 vs 拐杖的边界在于“褪去设计”

我的判断是：**Motion 等工具是否有效，取决于它们是否被设计为“会褪去的脚手架”，而非“永久拐杖”。**

如果 Motion 只是替你排期，而你从不学习时间感知，那它就是拐杖。如果 Inflow 只是替你跑 CBT 程序，而你从不内化认知重构技能，那它也是拐杖。

关键在于：工具应该自带“褪去机制”。比如 Motion 可以逐步减少自动排程的粒度，让你自己接管更多决策；Inflow 的模块化设计允许你逐步减少使用频率。

## 争议与局限

1. **过度依赖风险**：目前缺乏长期追踪研究验证适度使用的边界（来源：矛盾与存疑）。
2. **个体差异显著**：ADHD 亚型（注意力不集中型 vs. 冲动型）对 AI 工具的反应不同（来源：AI 与 ADHD 的情绪调节）。
3. **同构命题的实证缺口**：ADHD 大脑与 LLM 的同构目前是理论类比，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。
4. **Motion 的临床证据不足**：用户反馈积极，但缺乏独立临床证据（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Motion 或类似工具做一次“外部调度实验”**：把明天所有任务输入 Motion，完全信任它的排程，观察情绪变化。记录：启动困难是否减少？挫败感是否降低？
2. **给 LLM agent 加一个“re-grounding 提示”**：在 agent 的 system prompt 中加入“每3步回顾一次原始目标”，观察输出质量变化。
3. **尝试 Goblin Tools 的 Magic ToDo**：把一个让你拖延的任务（如“写报告”）输入，分解到最小步骤，完成前3步。感受启动门槛降低带来的情绪变化。
4. **建立自己的“日课”**：像释迦牟尼的八正道一样，每天固定时间做一次正念或任务回顾，作为内在调度层的训练。

# Reflect 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Reflect：它到底补上了哪一层执行功能？

## 问题：为什么你记了所有事，却仍然丢失目标？

ADHD 大脑的日常：你打开电脑要写周报，却发现自己正刷着第三篇关于水熊虫的文章；或者你花了三小时深挖一个细节，完全忘了最初要解决什么问题。这不是意志力问题，而是执行功能中的**目标维持**环节失效了。

同样，在 LLM Agent 的长程任务中，模型也会“超聚焦”——它可能偏离初始指令，陷入循环或生成无关内容。工程师的解法很直接：在 Agent harness 中设置**重锚定机制**，定期将 system prompt 重新注入，让模型回到原始目标。

Reflect 这类笔记工具，常被推荐给 ADHD 群体作为“第二大脑”。但大多数人的用法是把它当作被动记录仪——记完就忘，反而增加了认知负担。真正的价值，在于把它当作**重锚定 system prompt**——在你分心或跑偏时，主动拉你回主线。

## 同构：ADHD 大脑与 LLM 共享的“目标漂移”问题

ADHD 大脑与 LLM 在结构上同构：两者都是高产的生成核心，但缺乏可靠的执行调度层（来源：AI 与 ADHD 的专注力）。ADHD 的专注力问题可类比为 LLM 的上下文溢出和调度失败——当工作记忆超载或环境干扰时，目标从“上下文”中滑落，就像 LLM 在长对话中忘记初始指令。

在 LLM/harness 一侧，Agent 的可靠性来自架构+护栏：设置 token 预算、工具调用次数上限、人工检查点等（来源：Building an AI Agent Harness from Scratch）。这些护栏的核心作用是**重锚定**——在 Agent 偏离时，将 system prompt 重新注入，强制回到目标。

ADHD 侧也有类似的脚手架：RescueTime 通过自动记录时间使用，充当外部记忆，提升时间盲觉察（来源：AI 与 ADHD 的专注力）；Focusmate 利用身体在场效应提供外部问责，弥补任务启动的调度缺失（来源：人在回路与身体在场）。但这些都是被动或间歇性的锚定——它们告诉你“你跑偏了”，但不负责“把你拉回来”。

## Reflect 的真正角色：主动重锚定工具

Reflect 的设计核心是**快速捕获与回顾**。它通过 AI 自动整理笔记、生成摘要、建立链接，让信息更容易被检索。但对于 ADHD 用户，这个功能链却有一个缺口：**它假设你会主动去回顾**。而执行功能缺陷恰恰让“主动回顾”成为最困难的一步。

正确的用法是：**将 Reflect 设定为你的重锚定 system prompt**。具体方法：
1. **写一个“当前目标”笔记**，固定在每日看板顶部。每次分心后，先打开 Reflect 看这个笔记，再继续。
2. **利用 AI 生成的每日摘要**，在开始工作前快速扫一眼，重新锚定当天重点。
3. **设置定时提醒**（每 25 分钟），提示你“检查 Reflect 中的目标”。

这就像在 Agent harness 中设置一个定时重注入 system prompt 的模块——不是被动记录，而是主动拉回。

## 脚手架 vs 拐杖：边界在哪里？

这里有一个关键争议：过度依赖 AI 工具是否阻碍内在执行功能发展？目前无长期研究证明脚手架可逐步撤除（来源：AI 与 ADHD 的专注力）。Reflect 作为重锚定工具，如果使用得当，它应该是**脚手架**——帮助你建立回顾习惯，最终内化为自动行为。但如果只是被动记录，它就变成了**拐杖**——你记录得越多，越依赖外部记忆，反而削弱了工作记忆的自然锻炼。

区分标准很简单：**你是否在主动使用它来重定向？** 如果是，它是脚手架；如果只是记录后不再看，它是拐杖。

## 局限与争议

必须诚实指出：Reflect 针对 ADHD 的效果缺乏独立临床研究。多数证据来自工具机制与 ADHD 理论的对齐（来源：AI 与 ADHD 的专注力）。例如，Brain.fm 的神经锁相技术理论上可增强专注，但缺乏针对 ADHD 的随机对照试验（来源：Brain.fm）。同样，RescueTime 和 Forest 在 ADHD 诊断分类中有研究支持，但作为专注工具的证据薄弱（来源：AI 与 ADHD 的专注力）。

个体差异也很大：注意力缺陷型 vs 多动冲动型对工具反应不同。Reflect 的被动记录可能更适合注意力缺陷型（需要外部记忆），而多动冲动型可能更需要 Focusmate 这样的实时问责。

## 今天就能试的行动

1. **在 Reflect 中创建“今日目标”笔记**，每天早晨写 1-3 个核心任务。每次切换任务前，先看这个笔记。
2. **设置 25 分钟定时器**，响铃时问自己：“我现在做的事和 Reflect 中的目标一致吗？”不一致就停下来，读一遍目标再继续。
3. **每周用 Reflect 的 AI 摘要回顾**：哪些任务完成了？哪些跑偏了？模式是什么？
4. **尝试一次“身体在场 + Reflect”组合**：用 Focusmate 匹配伙伴，同时打开 Reflect 的目标笔记，让外部问责和重锚定同时生效。

Reflect 不是魔法。它只是一个重锚定工具，但当你主动用它来拉回自己时，它就补上了执行功能中最缺失的那一层——目标维持。

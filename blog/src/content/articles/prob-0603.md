# 如果把 ADHD 大脑当成生成核心，Focusmate 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Focusmate 对 ADHD 的真正价值。

## 问题：ADHD 的“边做边忘”与 LLM 的“会话结束即清零”

如果你是一位 ADHD 成人，你一定熟悉这个场景：打开电脑准备写报告，三分钟后被通知弹走，半小时后回来发现文档还停留在标题——工作记忆里那个“下一步要写什么”已经蒸发。如果你是做 agent 工程的工程师，你同样熟悉另一个场景：你精心构建的 LLM agent 在完成一次工具调用后，完全忘了上一轮对话的目标，需要你把整个上下文再塞一遍。

这两个场景共享同一个底层困境：**生成核心（无论是大脑还是模型）本身不维护持久状态**。ADHD 的工作记忆容量小、易被干扰清空（来源：认知负荷）；LLM 的上下文窗口虽大，但每次推理都是独立采样，跨会话的“记忆”必须靠外部向量库或 prompt 注入（来源：无状态与外部记忆）。两者都是高产但缺执行调度层的生成核心。

那么，一个像 Focusmate 这样简单到只有“定时视频共坐”的工具，能不能充当这个缺失的执行调度层？我的答案是：**能，但它的角色不是记忆体，而是无状态下的“定时 re-grounding 与身体在场调度器”**——这正是 agent 工程中 harness 的核心功能。

## Focusmate 的架构拆解：一个极简的 agent harness

Focusmate 的运作方式极其简单：你预约一个 50 分钟的视频 session，与另一位陌生人同时在线，各自做自己的事，摄像头开着，但不需要交流。从工程视角看，它包含三个关键组件：

1. **定时触发（scheduling）**：固定 session 时间，像 cron 作业一样强制启动和结束。
2. **身体在场信号（co-presence）**：对方的实时视频流作为“外部监督信号”，类似 agent 的 human-in-the-loop 监控。
3. **无状态重置**：每次 session 独立，结束后不保留任何上下文，下一次 session 重新开始。

这与 LLM agent 的 harness 结构高度同构。在 agent 工程中，我们为 LLM 搭建的脚手架通常包括：**规划器（planner）** 将目标分解为步骤，**记忆模块（memory）** 存储历史，**监控器（monitor）** 在偏离时 re-ground。Focusmate 的“定时 session”就是规划器，“摄像头”就是监控器，而“session 间无状态”恰恰避免了 ADHD 大脑在任务切换时因残留上下文导致的混乱。

## 真实人物的 harness：孔子与曾国藩的“日课系统”

注意，这不是一个空洞的类比。历史上两位著名的 ADHD 特质人物——孔子和曾国藩——都自发构建了类似的 harness。

孔子“吾日三省吾身”，用每日反省（定时 re-grounding）来对抗自己的冲动和思维跳跃。他的“礼”本质上是一套外部行为规范，相当于 agent 的约束规则。曾国藩的“日课十二条”更是典型：黎明即起（定时启动）、读书不二（单任务约束）、写日记反省（外部记忆）。他一生靠“结硬寨打呆仗”抵消自己的浮躁——这正是用最笨最稳的脚手架来补偿生成核心的不稳定。

把他们的系统映射到 agent 工程：**“日课”就是定时 re-grounding 的 cron job，“反省”就是事后日志分析，“秘书/家书”就是外部调度器与记忆库**。孔子和曾国藩并没有改变自己的大脑，而是搭建了外部脚手架——这正是 Focusmate 在做的事。

## 同构的实证：ADHD 与 Transformer 在认知负荷下的崩溃

最新研究为这一同构提供了硬证据：用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中表现正常，但随序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突。这与 ADHD 执行功能缺陷的核心标志一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃（来源：Deficient Executive Control in Transformer Attention）。

这意味着，ADHD 大脑和 LLM 在认知负荷下的崩溃模式是结构相似的。Focusmate 的作用，就是通过外部定时器降低认知负荷——它帮你决定“现在该做什么”，从而移除决策带来的认知负荷（来源：9 Best AI Assistants for ADHD in 2026）。这和 agent 工程中通过外部规划器将复杂任务分解（如 Goblin Tools 的 Magic ToDo）是同一个逻辑。

## 脚手架，不是拐杖

但这里有一个必须指出的争议：Focusmate 会不会变成永久拐杖？wiki 资料中的“拐杖与脚手架”概念提醒我们，过度依赖外部工具可能削弱内在执行功能。Focusmate 的 session 是无状态的，这意味着它不会积累你的历史——这既是优点也是缺点。优点在于避免上下文污染（就像 LLM 每次推理独立），缺点在于无法学习你的行为模式。

相比之下，Saner.AI 和 Mem 这类工具试图提供持久记忆（来源：Saner.AI；Mem），但 wiki 资料也指出 Mem 的 AI 工作记忆限制与 ADHD 用户需求存在矛盾（来源：矛盾与存疑）。Focusmate 选择了另一条路：不做记忆，只做调度。它的价值不在于记住你，而在于让你在 50 分钟内保持在场。

## 今天就能试的行动

1. **预约一个 Focusmate session，把它当作 agent 的“定时 re-grounding”**：在 session 开始前写下唯一目标，session 中只做这件事，结束后记录结果。感受“无状态重启”对 ADHD 大脑的释放。
2. **用 Goblin Tools 的 Magic ToDo 分解你明天最头疼的任务**：把“写方案”分解成“打开文档→列出三个要点→写第一段”，降低启动门槛（来源：Goblin Tools）。
3. **尝试“日课式”的每日反省**：像孔子和曾国藩一样，在一天结束时用 5 分钟写下“今天做了什么、哪里失控”。这不是日记，而是你的外部日志分析。
4. **如果你做 agent 工程，尝试在 agent 的 prompt 中加入“身体在场”信号**：例如，在每次工具调用前加入“请确认你仍在执行原计划”——这相当于 Focusmate 摄像头的监督功能。

## 局限与开放问题

最后，必须诚实指出：ADHD 大脑与 LLM 的同构目前仍是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。Focusmate 对 ADHD 的效果主要基于用户报告，缺乏严格临床验证（来源：矛盾与存疑）。此外，Focusmate 的无状态设计意味着它无法提供个性化适应——而 ADHD 亚型（如注意力不集中型 vs 冲动型）可能需要不同的调度策略。未来，也许会出现结合 Focusmate 的定时在场与 Saner.AI 的本地记忆的混合工具，但今天，我们至少可以确认：对于“边做边忘”的生成核心，一个极简的外部调度器，比任何复杂的内部改造都更可靠。

# ADHD 学生如何用「给生成核心套 harness」的思路应对考试与作业里的执行功能（计划/组织/启动）先天偏弱？

> 把 LLM 工程里的「agent 的编排/记忆/工具层就是「外部执行功能」」迁移到ADHD 学生的考试与作业场景，双域证据作答。

## 从同一个问题出发：生成核心缺了调度层

对许多 ADHD 学生来说，考试与作业最痛苦的往往不是“不会做”，而是“启动不了、组织不来、坚持不下”。脑子里的想法像喷泉一样多，计划表写了又写，但一到执行就碎成片段：该复习的章节打开了又关掉，该写的作业拖到深夜，时间像被偷走一样。传统建议通常是“你要更自律”，但自律在这里不是道德问题，而是神经架构问题——ADHD 的核心缺陷在于执行功能、工作记忆、时间知觉与任务启动（来源：ADHD × AI 的科学与研究前沿）。

有趣的是，AI 工程师今天正被一个非常相似的难题困扰。大语言模型（LLM）能生成流畅文本、推理复杂问题，但它无状态、上下文一膨胀就推理退化，控制逻辑还容易散落在框架默认值和硬编码里（来源：上下文工程）。因此，工程师们不再只“训练模型”，而是给模型套上一层 harness：编排（orchestration）、外部记忆、工具调用、验证循环。agent 的编排/记忆/工具层，本质上就是“外部执行功能层”（来源：ADHD × AI 的科学与研究前沿）。

ADHD 学生与 LLM，是同一个问题在两个领域的投影：一个高产的生成核心，配了一个不可靠的调度层。两边的解法，结构同构。

## 同构的证据：两边都缺什么，又需要什么

从 ADHD 侧看，执行功能缺陷表现为工作记忆不稳定、时间盲（time blindness）和任务启动困难。学生可能把“复习第三章”记在脑子里，转身就被新刺激带走；对考试前的一周缺乏真实感知， Deadline 要么太远、要么突然压到脸上；面对整张试卷，启动成本大到无法落笔（来源：ADHD × AI 的科学与研究前沿；时间盲；任务启动）。

从 LLM/agent 侧看，对应的缺陷同样明显：模型无状态，跨会话上下文无法可靠保留；上下文一旦膨胀，推理就退化；没有外部工具，它无法执行真实行动；没有验证循环，它会产生“幻觉”并 confidently 输出错误（来源：无状态与外部记忆；上下文工程；幻觉与验证循环）。

所以两边的补偿结构完全一致：
- **外部记忆**：替代不稳定的内部工作记忆，保留跨时间上下文。
- **编排/调度**：把模糊目标变成可执行的步骤序列，减少启动阻力。
- **工具层**：把抽象任务接到真实动作上（如写进日历、打开文档、计时）。
- **验证循环**：把内部状态与外部现实核对，防止“行为层面的幻觉”。

## 一个真实的 harness：曾国藩的“日课十二条”

晚清名臣曾国藩是理解这一同构的绝佳案例。他的 ADHD 特质非常明显：年轻时浮躁坐不住，天天串门喝酒，读书“他人目下二三行，余或疾读不能终一行”；情绪极不稳定，打败仗就跳水自杀；一辈子受银屑病困扰。他并非靠“变得自律”成功，而是给自己搭了一套外部 harness——“日课十二条”：黎明即起、读书不二、谨言、写日记反省等，并用“结硬寨、打呆仗”的笨办法把冲动锁进结构化流程里（来源：人物案例）。

把这套系统映射到 LLM harness，同构关系非常清晰：
- **“日课十二条”** 是固定的 prompt template / routine，把“今天该做什么”变成预编排的上下文，减少决策负担；这对应 Routinery 等应用“create a sequence of steps for a routine and walk you through them with transition prompts”的思路（来源：上下文工程）。
- **每日写日记骂自己、反省**，是外部记忆（把念头和计划落到纸上）加验证循环（把行为与目标对照），防止白日梦与冲动偏离现实。
- **“结硬寨、打呆仗”** 则是结构化工作流：不追求灵感爆发，只追求每一步可执行、可验证，对应 LLM harness 中“将复杂任务分解为子任务，并在每一步进行规划与验证”的要求（来源：幻觉与验证循环）。

曾国藩不是“ cured 了 ADHD”，而是把生成核心的输出，通过外部 harness 导流到了正确目标上。

## 今天的工具：把 harness 工程落到学生桌面

这种 harness 思维已经可以通过现有工具实现，但重点不是“用越多越好”，而是按功能分层。

**任务启动层：Goblin Tools**。它的 Magic ToDo 能把模糊任务——比如“复习数学”——按“辣度”自动拆解成具体可执行的子任务，降低多巴胺不足导致的启动门槛（来源：Goblin Tools）。这对应 LLM  harness 里的任务分解与编排：不直接要求模型完成大目标，而是先产出子任务规划工件。

**外部记忆层：Saner.AI**。它强调本地记忆与知识回忆，通过 AI 笔记捕获、自动组织任务和通用收件箱，减少 ADHD 学生在多个标签页和应用之间反复搜索的“上下文断裂”（来源：Saner.AI）。这与 LLM 使用向量数据库、记忆系统来保留跨会话上下文是同构的。

**动态调度层：Motion**。它自动根据任务、会议和截止日期生成每日计划，并在日程被打乱时重新安排；它还会提前数周警告任务延期风险（来源：Motion）。这直接替代 ADHD 学生受损的内部时间感知和优先级排序，对应 agent harness 的实时调度与重规划能力。

这三类工具组合起来，就是给学生版生成核心套上的 harness：分解任务→记录上下文→安排时间→验证执行。

## 核心判断：harness 是脚手架，不是拐杖

本文的核心判断是：对 ADHD 学生而言，AI 与相关工具的价值不在于“修复”大脑，而在于承认一个架构事实——生成核心很强，但调度层薄弱，因此必须外化执行功能。真正好的 harness 是可拆卸的脚手架：初期它替你决策、记忆、提醒，目标是让你逐渐把一部分外部结构内化为自己的工作流程；如果工具越来越重、越来越离不开，它就退化成拐杖，反而加剧依赖（来源：矛盾与存疑）。

这里的关键区分是：
- **脚手架**：提供结构化入口，但保留用户的元认知；学生仍然要检查“这个分解是否合理？今天我是否完成了验证循环？”
- **拐杖**：替用户做一切决策，让学生放弃对目标和过程的反思。

曾国藩的日记之所以重要，正是因为它在 harness 之上保留了验证循环——他不是 blindly 执行日课，而是每天把它拿出来审视、修正。

## 局限与争议：同构仍是一个框架，不是定论

必须诚实指出，这套思路仍有明显局限。

第一，**证据强度有限**。ADHD 大脑与 LLM 同构的说法主要是概念类比和初步实证，缺乏神经科学或计算机科学的直接证据，也缺乏大规模随机对照试验（来源：矛盾与存疑）。

第二，**个体差异大**。Goblin Tools、Motion 等工具对某些人有效，对另一些人可能增加认知负荷，甚至让学习工具本身变成新任务（来源：矛盾与存疑）。

第三，**长期效果未知**。AI 辅助的任务启动在短期可能有效，但会不会导致能力退化、形成永久依赖，目前缺乏纵向研究（来源：矛盾与存疑）。

因此， harness 应该被看作一种“需持续调试的工程方案”，而不是万能药方。

## 今天就能试的四步行动

如果你是一名 ADHD 学生，或正在为 ADHD 学生设计工具，可以今天开始试这四步：

1. **用“最小可启动单元”代替宏大目标**。不要写“复习第三章”，而是写下“打开课本→读第 3.1 节标题→写下 3 个不懂的词”。这对应 agent  harness 里的任务分解工件。

2. **建立单一外部记忆锚点**。选一个工具（如 Saner.AI）或一个固定笔记本，把所有任务、截止日期、复习重点丢进同一个“收件箱”，减少跨应用切换导致的上下文断裂。

3. **让日程工具替你排队，但保留人工确认**。用 Motion 或类似工具自动生成明日计划，但每天晚上花 3 分钟检查：它排的优先级是否符合你的考试目标？

4. **每天一次验证循环**。用曾国藩的方式，在睡前写三行：今天做了什么、原计划是什么、偏差在哪。这不需要很长，但要把内部状态与外部现实核对。

## 结语：不是变“正常”，而是给核心装上 harness

ADHD 学生与 LLM 工程师，面对的是一个共同的问题：如何让一个高产的生成核心，稳定地输出到正确目标上。答案不是压抑生成力，也不是等待自律从天而降，而是像 agent engineering 一样，老老实实搭建外部执行功能层——编排、记忆、工具、验证。考试与作业，只是这个 harness 要接管的第一批真实任务。

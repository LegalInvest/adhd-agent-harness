# Claude 之于 ADHD，就像 外部记忆/向量库 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

你刚打开 Claude，输入“帮我规划下周的工作”，它列出一份漂亮的清单。你满意地关掉窗口，五分钟后回来，发现刚才的计划已经像晨雾一样消散——你忘了它写了什么，也忘了自己接下来该做什么。这不是你的错，也不是 Claude 的错。这是两个“无状态生成核心”相遇时的天然鸿沟。

ADHD 大脑与 LLM 共享同一个底层困境：**两者都是高产但缺执行调度层的生成核心**（来源：ADHD 的 AI 工具全景）。你的工作记忆容量其实不小，但控制弱，易被干扰；LLM 同样有强记忆容量但弱上下文控制（来源：工作记忆）。当你关掉 Claude 窗口，它就像被清空了上下文——而你，则因为工作记忆溢出而“丢失”了计划。

## 同构：无状态与外部记忆

LLM 本身没有跨会话状态，它靠外部记忆（向量库、检索器）续命。ADHD 大脑同样需要外部系统来持久化上下文：一个任务一旦离开“当前焦点”，就几乎等于不存在。这就是为什么 Saner.AI 强调“知识回忆”和“本地记忆”（来源：Saner.AI），通过 AI 实现快速检索，减少搜索循环和标签切换。同样，Mem 被称为“AI 第二大脑”，本质上是为 ADHD 大脑提供持久化上下文（来源：ADHD 的 AI 工具全景）。

但很多人用错了。他们把 Claude 当作一次性问答机：问完即走，下次重新开始。这就像给 LLM 装了一个向量库却从不写入——每次对话都是冷启动。正确的用法是：**把 Claude 当作外部工作记忆的读写接口**。例如，在 Claude 里持续维护一个“当前项目上下文”文件，每次对话前先粘贴进去，让 Claude 基于完整上下文生成，而不是从零开始。这和 LLM agent 的 harness 设计完全同构：agent 通过工具调用、文件管理、长对话处理来维持状态（来源：工具使用与认知卸载），而 ADHD 用户也需要类似的“脚手架”来维持任务连续性。

## 脚手架 vs 拐杖：边界在哪里？

Goblin Tools 的 Magic ToDo 能将“整理房间”分解为“捡起地板上的衣服”“擦桌子”等步骤（来源：Goblin Tools），用户报告称启动焦虑降低。Lex 允许通过单一指令触发多步骤任务序列（来源：Lex），减少决策疲劳。这些工具直接针对任务启动困难，相当于为生成核心提供启动脚本（来源：ADHD 的 AI 工具全景）。

但这里有陷阱。如果 AI 帮你分解了任务，你却从未学会如何自己分解，那么工具就成了拐杖。同样，在 LLM 工程中，如果 agent 完全依赖外部规划循环而模型本身不具备任何规划能力，一旦 harness 失效，系统就会崩溃。健康的脚手架应该是：**工具做那些你永远做不好的事，同时保留你还能成长的空间**。例如，Otter.ai 实时转录会议，减轻笔记负担，但过度依赖可能削弱主动记笔记的能力（来源：ADHD 的 AI 工具全景）。Reflect 通过结构化提问促进反思，而不是替你做决定（来源：ADHD 的 AI 工具全景）。

## 证据与局限

现有证据主要来自用户报告，缺乏大规模对照实验（来源：矛盾与存疑）。Goblin Tools 被评价为“简单有用”（来源：Goblin Tools），Saner.AI 被列为最佳 ADHD 生产力工具之一（来源：Saner.AI），但这些都是定性反馈。更关键的是，个体差异巨大：有人用 Claude 做任务分解效果显著，有人却因为过度依赖而更焦虑（来源：矛盾与存疑）。工具设计者声称是“脚手架”，但实际使用中可能沦为“拐杖”——这个边界需要用户自己警惕。

## 今天就能试的行动

1. **建立外部工作记忆文件**：在 Claude 中创建一个“当前项目上下文”文档，每次对话前先粘贴，让 Claude 基于完整上下文生成。这相当于为你的 ADHD 大脑搭建一个持久化向量库。
2. **用 Goblin Tools 做一次“反向工程”**：拿一个你正在拖延的任务，让 Goblin Tools 分解，然后对比你自己会怎么分解。如果差异很大，说明你缺少某种分解策略——这正是脚手架要教你的。
3. **设置“上下文检查点”**：每完成一个子任务，在 Claude 或 Mem 中记录当前状态（一句话即可）。下次继续时，先读这个检查点，而不是从头回忆。这对应 LLM agent 的“状态持久化”模式。
4. **警惕“一次性问答”陷阱**：如果某个 AI 工具让你觉得“用一次就完了”，那它很可能在强化你的无状态困境。换成需要持续写入上下文的工具（如 Saner.AI 或 Mem），并坚持维护。

Claude 之于 ADHD，就像外部记忆之于 LLM：它本身无状态，但通过正确的脚手架可以成为强大的执行功能层。用错了，它只是另一个让你分心的窗口；用对了，它是你大脑的外接硬盘。

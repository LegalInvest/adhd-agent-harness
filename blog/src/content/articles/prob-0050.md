# 为什么「治好 ADHD 的不擅长精确、重复、状态维护类任务」和「让 LLM 不跑飞」其实是同一道工程题？

如果你是ADHDer，这些场景再熟悉不过。但如果你是做AI Agent的工程师，你可能正在经历另一场噩梦：你精心设计的LLM agent，在第五轮对话后开始胡言乱语，忘记了系统提示，陷入了死循环，或者调用了一个危险的工具。

这两个场景看似风马牛不相及，但本文要论证一个反直觉的同构：**ADHD大脑与LLM是同一类“高产但缺执行调度层的生成核心”，而解决双方核心问题的方案——通过外部工具进行认知卸载和任务外包——是同一道工程题。**

## 问题：精确、重复、状态维护——双方的共同软肋

ADHD的认知特征早已被研究透彻：工作记忆容量虽大但控制弱，容易被干扰（来源：[[工作记忆]]）；任务启动困难，面对复杂任务就像面对一堵墙（来源：[[任务启动]]）；时间感知缺陷，对“需要多久”毫无概念（来源：[[时间盲]]）。这些缺陷集中指向一类任务：精确、重复、状态维护类。比如记账、填表格、按步骤组装家具、定期备份文件——这些任务要求你记住当前步骤、保持注意力、精确执行，而ADHD大脑恰恰在此失灵。

LLM/agent同样如此。模型本身是强大的生成核心，能写诗、编程、推理，但一旦涉及精确计算、重复操作、维护对话状态，就原形毕露。LLM会忘记系统提示（来源：Building an AI Coding Agent for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned），会在长程任务中目标漂移（来源：[[重锚定与目标漂移]]），会陷入无限循环（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。精确计算（如算术）和重复状态维护（如记账）正是LLM的软肋。

## 答案：工具使用与认知卸载——同一套harness思路

既然问题同构，解法也同构。ADHD领域早已发现：外部工具可以补偿内在的执行功能缺陷。例如，Goblin Tools的Magic ToDo功能将“整理房间”分解为“捡起地板上的衣服”“擦桌子”等小步骤，降低启动门槛（来源：[[Goblin Tools]]）；Lex允许用户通过单一指令触发多步骤任务序列，减少决策疲劳（来源：[[Lex]]）；Saner.AI通过本地记忆存储和快速检索，减少搜索循环和标签切换（来源：[[Saner.AI]]）。这些工具的本质是：**将精确、重复、状态维护类任务外包给外部系统，让大脑专注于生成和决策。**

LLM/agent侧的做法完全对称：agent通过function calling把精确计算外包给工具（如计算器API），通过harness（脚手架）管理状态、维护上下文、防止目标漂移。Harness的核心是“形式化规划与护栏，使代理输出真正有用且正确”（来源：What is an agent harness in the context of large-language models?）。具体来说，harness通过事件驱动的系统提醒来对抗指令消退（来源：[[重锚定与目标漂移]]），通过human-in-the-loop监督提供外部调度（来源：[[人在回路与身体在场]]），通过token预算和工具调用次数上限防止无限循环（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。

**核心观点：ADHD的AI工具和LLM的agent harness是同一类“外部执行功能层”——它们不是替代大脑或模型的能力，而是提供一个可靠的调度层，让生成核心（ADHD大脑或LLM）专注于自己擅长的创造性、高价值工作，而把精确、重复、状态维护类任务外包给工具。**

## 证据：两边都在用同一套模式

证据来自多个层面。首先，任务分解与启动：Goblin Tools的用户报告称启动焦虑降低（来源：[[Goblin Tools]]），这对应LLM侧harness将复杂任务拆解为子步骤并顺序执行。其次，记忆与上下文管理：Mem和Saner.AI通过持久记忆补偿工作记忆不足（来源：[[Saner.AI]]），对应LLM侧的外部记忆系统（如向量数据库）和上下文工程。再次，实时反馈与反思：Otter.ai的实时转录和Reflect的结构化提问提供即时外部反馈（来源：[[人在回路与身体在场]]），对应LLM侧的人机交互检查点和遥测。最后，重锚定与目标漂移：ADHD工具通过提醒和通知将注意力拉回原任务（来源：[[重锚定与目标漂移]]），LLM harness通过事件驱动的系统提醒防止模型忘记初始指令（来源：[[重锚定与目标漂移]]）。

更具体地说，身体在场效应（body doubling）是ADHD群体最有效的策略之一：有另一个人在场时，执行功能障碍被绕过（来源：[[人在回路与身体在场]]）。Focusmate等工具通过AI匹配虚拟身体搭档提供外部问责（来源：[[人在回路与身体在场]]）。这完全对应LLM侧的human-in-the-loop监督：加入人工检查点，暂停并询问后再执行（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。

## 争议与局限：脚手架还是拐杖？

然而，这种同构解法并非没有争议。Wiki资料明确指出，AI工具作为外部执行功能层的有效性证据主要来自用户报告，缺乏大规模随机对照试验（来源：[[矛盾与存疑]]）。更重要的是，过度依赖AI工具可能削弱内在能力：例如，过度使用Otter.ai可能削弱主动记笔记的能力（来源：[[拐杖与脚手架]]）；过度依赖外部提醒可能削弱内在时间感知能力（来源：[[时间盲]]）。LLM侧同样如此：过度依赖harness可能导致模型本身越来越“懒”，失去自主规划能力。

**关键判断：工具应作为脚手架促进能力发展，而非永久拐杖。** 脚手架意味着你最终可以撤掉它，而拐杖意味着你永远依赖它。ADHD工具和LLM harness的设计者都声称是脚手架，但实际使用中可能沦为拐杖。例如，Goblin Tools的任务分解虽然有效，但长期使用可能让用户失去自己分解任务的能力（来源：[[Goblin Tools]]）。同样，LLM harness如果过于严格，可能抑制模型的创造性。

因此，我们需要有意识地使用工具：定期尝试不依赖工具完成任务，用工具来学习而非替代。对于ADHDer，这意味着在AI辅助下逐渐培养执行功能；对于工程师，这意味着设计harness时保留模型自主决策的空间。

## 今天就能试的行动

1. **ADHDer：用Goblin Tools分解一个你拖延已久的任务。** 输入“整理书桌”或“写周报”，观察AI如何将大任务拆解为小步骤。然后，按步骤执行，每完成一步就划掉。这能让你亲身体验“外部执行功能层”的效果。

2. **工程师：用Claude或ChatGPT构建一个简单agent，并加入human-in-the-loop检查点。** 例如，让agent执行“搜索网页并总结”，但在调用搜索工具前暂停并询问用户是否继续。这能防止目标漂移，同时让你感受harness的价值。

3. **双方都适用：设置外部记忆系统。** ADHDer可以使用Saner.AI或Mem记录想法和任务，减少搜索循环；工程师可以使用向量数据库或Notion AI作为agent的外部记忆，避免上下文膨胀。

4. **尝试身体在场效应的数字版本。** ADHDer可以注册Focusmate，与陌生人视频一起工作；工程师可以在agent中加入“定期向人类报告进度”的机制，模拟外部问责。

## 结语

ADHD大脑和LLM，一个生物，一个硅基，却共享同一个核心困境：生成能力强，执行调度弱。而解法也共享同一套工程思路：通过外部工具进行认知卸载，把精确、重复、状态维护类任务外包给专门的工具。理解这个同构，不仅能帮助ADHDer找到更有效的工具，也能帮助工程师设计更可靠的agent。下次当你为ADHD的拖延自责，或为LLM的跑飞抓狂时，记住：这不是你的错，也不是模型的错——这是生成核心的固有特征。你需要的不是更努力，而是更好的harness。

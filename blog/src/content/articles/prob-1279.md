# ADHD 想把执行功能（计划/组织/启动）先天偏弱外包给 ChatGPT，该怎么设计这套「人 + AI」的回路？

> 把 ChatGPT 接进 ADHD 工作流：哪些该交给它，哪些必须自己留着？

你坐在电脑前，打开 ChatGPT，想让它帮你规划今天的工作。你输入“帮我整理一下这个项目”，它输出了一大段步骤，但你盯着屏幕，仍然不知道从哪里开始。问题出在哪？

不是 ChatGPT 不够聪明，而是它和你一样，缺少一个执行调度层。

ADHD 大脑与大型语言模型（LLM）在结构上同构：两者都是高产生成核心，但缺乏可靠的执行调度机制。你的工作记忆像无状态的 API 调用——每次对话都可能丢失上下文；你的时间盲像固定上下文窗口，无法感知长期规划；你的任务启动困难像缺少 prompt 的 LLM，空有算力却卡在第一步。

所以，当你试图把执行功能外包给 ChatGPT，你其实是在用另一个同样缺调度层的系统来弥补自己的调度缺陷。这不是解决方案，这是两个无头苍蝇在互撞。

真正的解法是：为这套“人 + AI”回路设计一个外部执行功能层——一个 harness，一个脚手架。它既不是让你依赖 AI，也不是让 AI 取代你，而是像 agent 工程中的编排层一样，把生成核心（你的大脑和 LLM）连接起来，提供记忆、任务启动、时间管理等功能。

## 脚手架 vs 拐杖：边界在哪里？

“拐杖与脚手架”概念指出：真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。在 LLM/harness 一侧，脚手架在首次提示前组装代理（系统提示、工具模式、子代理注册），而 harness 在运行时编排工具调度、上下文管理和安全执行（来源：Building AI Coding Agents for the Terminal）。

这意味着，你的 AI 辅助回路必须包含三个要素：
1. **持久记忆层**：弥补你和 LLM 双方的无状态性。
2. **任务启动机制**：降低你和 LLM 双方的启动门槛。
3. **上下文管理**：防止你和 LLM 双方的上下文窗口溢出。

## 从工具看回路设计

现有工具已经提供了零散的答案。

**Goblin Tools** 的 Magic ToDo 功能能自动将任何任务分解为更小、更易管理的步骤（来源：AI Tools for Kids with ADHD: A Complete Guide for Parents...）。这直接对应 agent 工程中的“任务分解”模式——把一个大任务拆成可执行的子任务，每个子任务对应一个工具调用。对于 ADHD 用户，这种分解降低了启动焦虑，利用了“身体在场效应”通过小步骤的即时反馈维持注意力（来源：Goblin Tools 工具页）。

**Saner.AI** 专注于知识回忆和本地记忆，减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026）。这对应 LLM/harness 一侧的外部记忆系统：通过向量数据库或结构化知识库，让 agent 能跨会话保持上下文。对于 ADHD 用户，这种“第二大脑”补偿了工作记忆的波动（来源：How AI Helps ADHD Brains Work Smarter, Not Harder）。

**Motion** 则提供自适应日程规划，自动根据任务、会议和截止日期创建并动态调整每日计划（来源：11 Best ADHD Productivity Apps for Fluctuating Energy）。这对应 agent 工程中的“规划与重规划”模式——agent 在执行过程中根据反馈调整计划。对于 ADHD 用户，它消除了“下一步该做什么”的决策负担，直接帮助克服时间盲和任务启动困难（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。

但注意：这些工具都缺少一个关键设计——可撤除性。

## 争议与局限：脚手架可能变成永久拐杖

“拐杖与脚手架”概念强调，脚手架应设计为可逐步撤除的临时支撑（来源：拐杖与脚手架概念页）。但 Goblin Tools、Saner.AI、Motion 都定位为长期使用工具，未提及撤除机制。长期依赖是否会削弱内在执行功能？目前缺乏纵向研究（来源：ADHD × AI 的科学与研究前沿）。

更根本的争议在于：ADHD 大脑与 LLM 同构的命题本身缺乏大规模实证，主要基于概念类比和工具案例（来源：ADHD × AI 的科学与研究前沿）。这并不意味着命题错误，而是提醒我们：工具的有效性不能仅靠理论对齐来证明。

## 核心观点：设计你的个人 harness

基于同构视角，我提出一个鲜明的判断：

**ADHD 的 AI 辅助回路，应该模仿生产级 agent 的 harness 架构，而不是单纯依赖对话式 AI。**

具体来说，你的个人 harness 应包含三个组件：
1. **记忆系统**：用 Saner.AI 或类似工具作为外部记忆，存储项目上下文、个人偏好和已完成步骤。
2. **任务分解器**：用 Goblin Tools 或自定义 prompt 将模糊目标拆解为可执行步骤。
3. **调度器**：用 Motion 或日历工具自动安排这些步骤到时间线上，并动态调整。

但更重要的是，你要为自己保留“攀登”的部分：
- **启动权**：AI 可以分解任务，但“开始做”的决定必须由你做出。
- **优先级判断**：AI 可以建议优先级，但最终排序要基于你的直觉和价值观。
- **撤除计划**：定期评估哪些步骤不再需要 AI 辅助，逐步内化。

## 今天就能试的行动

1. **用 ChatGPT 做一次任务分解实验**：输入一个让你拖延的任务，要求它输出不超过 5 个步骤，每步不超过 15 分钟。完成后，标记你实际启动的时间。
2. **设置一个外部记忆点**：在 ChatGPT 对话中，每次结束时用一句话总结当前状态和下一步，并要求它在下次对话开始时复述。
3. **试用一个工具**：选择 Goblin Tools（免费基础版）或 Motion（试用版），用它规划明天的一个任务，记录你的启动感受。
4. **反思依赖**：一周后，问自己：如果没有这个工具，我还能完成同样的任务吗？如果不能，说明它成了拐杖，需要调整。

记住：AI 是你大脑的 harness，不是你的大脑。真正的执行功能，永远在你自己的调度层里。

# ADHD 学生视角：为什么「治好 ADHD 的任务启动困难、不会拆解」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 用 planner-executor 循环分解长任务——同一套 harness 思路，两边都成立。

## 引言：两种考场，同一道题

一个 ADHD 学生坐在书桌前，明知论文明天截止，却刷了三小时短视频。他缺的不是智力，也不是意愿——他缺的是那个把「我知道要写论文」翻译成「我现在打开文档，写第一段」的转换层。

一个做 Agentic Harness 的工程师，面对同样诡异的场景：LLM 能生成流畅的代码、商业计划、文学评论，但一让它跑一个十步以上的项目，不是忘了目标，就是陷入循环，或者突然开始一本正经地胡说八道。它也不缺生成能力，缺的是把长程目标保持住、按步骤执行的调度层。

两边看似风马牛不相及，但坐在同一道题前：核心无比高产，却缺一个可靠的执行调度层。治好 ADHD 的任务启动困难和让 LLM 不跑飞，本质上是同一道工程题。

## 同构：高产生成核心与缺席的调度层

ADHD 大脑的研究呈现一种反复出现的剖面：记忆和联想能力可以很强，但控制与切换能力薄弱。一项实证研究将其概括为「强记忆、弱控制」（Strong Memory, Weak Control）——工作记忆容量甚至可能超过常人，但认知灵活性与注意控制存在核心缺陷，无法灵活切换任务集、无法抑制自动化反应（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。另一项研究则指出，注意力分散是自注意力架构的固有属性，随着任务负载增大，注意力分数熵增加，导致容量限制（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。Transformer 在经典 Stroop 冲突任务中，长序列下冲突解决能力会崩溃至随机水平——无法抑制优势反应，与 ADHD 执行功能缺陷的核心标志一致（来源：Deficient Executive Control in Transformer Attention）。

也就是说，ADHD 大脑与 LLM 都像是同一类「高产生成核心」：发散、联想、创意、高输出，但缺乏稳定的「驾驶座」。ADHD 的执行功能在调控注意力、计划、组织和冲动控制上不稳定；LLM 则缺乏跨会话的持久状态、可靠的目标维持与多步调度能力（来源：ADHD 大脑与 LLM 的同构；外部执行功能层）。

这种对应不是文学比喻。功能上，ADHD 的工作记忆不稳定与 LLM 的无跨会话状态同构：两者都需要外部记忆系统来补全（来源：外部执行功能层）。

## 工程解：把「驾驶座」从大脑/模型里拆出来

既然问题不是核心不够强，而是调度层缺失，解法就不是「修核心」，而是外挂一个 harness。

对 ADHD 用户，harness 是外部执行功能层：把「什么时候做哪件事」从大脑里卸载出去。Motion 通过 AI 自动排程与动态调整，持续评估任务优先级、截止日期和可用时间，实时重建日程，减少手动规划压力（来源：Motion；11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。Reclaim.ai 走的是另一条路：它不填满日程，而是保护深度工作与习惯的智能时间块，自动防御会议侵占（来源：Reclaim.ai）。Tiimo 和 Structured 则用视觉化时间线把抽象时间「实体化」，直接回应 ADHD 的「时间盲」——难以感知时间流逝的问题（来源：Tiimo；AI 与 ADHD 的时间管理）。Todoist、Saner.AI 等工具则把任务列表外部化，降低工作记忆负荷（来源：AI 与 ADHD 的时间管理）。Goblin Tools 还直接处理任务分解，把「写论文」拆成可启动的下一步（来源：Reclaim.ai）。

对 LLM agent，harness 是 agent scaffolding：围绕 LLM 构建的软件架构与工具，把它置于与记忆、工具和决策逻辑的控制循环中，使其能够推理、规划并超越一次性提示（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。具体实现包括：Git 仓库存储项目上下文、MCP 连接器访问外部数据、规划工件、验证循环、沙箱与记忆系统（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned；GitHub - ai-boost/awesome-harness-engineering）。

两边的结构惊人地一致：一个生成核心 + 外部记忆 + 外部调度器 + 反馈/重新规划循环。这正是 planner-executor 循环的本质：先计划，再执行，执行后把状态重新注入，修正计划。

## 孔子的 harness：礼就是外部 re-grounding

这道工程题不是今天才有答案。孔子很可能是一个古典 ADHD 案例：身高一米九、精力旺盛、周游列国十四年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予「朽木不可雕」；对音乐能超专注到「三月不知肉味」，对种地却零耐心；思维跳跃，《论语》全是场景化语录，没有系统著作。但他用一生的 harness 把这套高产生成核心「框」住了：以「克己复礼」为核心，用外在的「礼」作为行为边界，每日反省，「吾日三省吾身」；直到七十岁才做到「从心所欲不逾矩」——也就是说，他一辈子在和自己的冲动作斗争，用外部结构补齐内部调度。

孔子的「礼」与「三省」对应到现代 harness 工程里，就是行为边界与定期 re-grounding：LLM agent 的 planner-executor 循环里，需要定期把执行状态重新对齐到目标；ADHD 用户的日程工具，也需要每天把抽象目标重新落地成下一步行动。孔子的 harness 不是消灭冲动，而是给冲动修轨道；现代 LLM harness 也不是消灭幻觉，而是给生成修轨道。两者都是同一个原则：核心不可控，但环境可控。

## 边界：脚手架 vs 拐杖，以及争议

好的 harness 是脚手架，不是拐杖。脚手架帮助用户逐步内化能力，最终能撤掉；拐杖则让人永久挂在上面（来源：AI 与 ADHD 的时间管理）。但这里的边界非常微妙：当 Motion 替你重排日程、LLM 的 planner 替你决定下一步时，多少帮助是卸货，多少帮助是让人放弃学习驾驶？

更需要诚实指出的是，ADHD 与 LLM 的同构性目前仍是一种理论类比，而非经过实证验证的神经科学事实（来源：全局矛盾与存疑）。工具如 Motion、Brain.fm 等在 ADHD 人群中的独立临床证据也有限（来源：全局矛盾与存疑）。部分 ADHD 工具甚至假设用户能维持长时间专注、记住昨天的问题并执行计划，但实际用户往往做不到（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。所以这些工具是 promising 的辅助，不是治疗方案。

创业研究提供了另一个视角：ADHD 组的创业概率是对照组的 6.25 倍（来源：Journal of Attention Disorders 2019）。这说明高产生成核心本身有巨大价值，但前提是给它配上 harness。刘秀华这样的 ADHD 投资人所展现的 big-picture thinking、pattern recognition、novelty seeking 与 action bias，只有在有外部系统支持项目上下文、过滤冲动决策时，才会转化为 172+ portfolio companies 的产出。

## 今天就能试的 4 步

1. **找一个外部调度器，先把一个项目丢进去**。别追求完美系统。ADHD 用户可试 Motion 或 Tiimo；工程师可在 agent 里加一个显式 planner-executor 循环。关键是：把「下一步该做什么」从核心内部推出来。

2. **建立每日 re-grounding 仪式**。像孔子「三省吾身」一样，早晚各花 5 分钟，问三个问题：今天最重要的目标是什么？实际执行偏离了吗？明天的第一个可启动动作是什么？

3. **为 LLM 也做「时间盲」补偿**。把项目状态、关键约束、检查点写入外部记忆（如 Git、向量库或结构化日志），让 agent 每次 planning 前强制读取。没有外部记忆，LLM 和 ADHD 大脑都会「忘记」自己身在何处。

4. **每周做「脚手架审计」**。问问自己：哪些步骤我已经能独立完成？哪些还必须依赖工具？对工程师同理：agent 的 harness 中哪些验证循环是收敛的，哪些成了让系统过度依赖外部判断的拐杖？

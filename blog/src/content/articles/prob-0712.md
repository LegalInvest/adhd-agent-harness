# Lex 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Lex：它到底补上了哪一层执行功能？

你有没有过这样的体验：打开一个号称“帮你搞定一切”的 AI 工具，输入一句指令，期待它自动完成整个项目——结果它要么输出一堆无关细节，要么卡在半路，你反而更焦虑了？

这不是你一个人的问题。对 ADHD 大脑来说，这类工具用错了，就像给 LLM 一个开放式 prompt 却不给 function calling 工具——生成能力再强，也救不了执行调度层的缺失。

## 问题：ADHD 大脑与 LLM 共享的“执行赤字”

ADHD 大脑的核心困境不是缺乏想法或动力，而是无法可靠地将意图转化为有序行动。工作记忆像漏水的桶，时间盲让计划变成空想，任务启动焦虑像一堵无形的墙（来源：ADHD 的 AI 工具全景）。

这和 LLM 的处境惊人地相似：LLM 的生成能力极强，但如果你不给它一个精心设计的 harness（比如 function calling 工具、结构化 prompt），它就会跑题、重复、陷入幻觉。ADHD 大脑与 LLM 是同一类“高产但缺执行调度层的生成核心”——两者都需要一个外部执行功能层来补上计划、组织、记忆和启动（来源：ADHD 的 AI 工具全景）。

## Lex 的承诺：一次指令，一串行动

Lex 正是瞄准了这个缺口。它的核心卖点是“通过单一指令触发复杂、多步骤的任务序列”（来源：Lex）。想象一下：你输入“写一篇公众号文章”，Lex 自动帮你拆解选题、大纲、初稿、配图建议，然后一步步执行。这听起来像梦一样——对 ADHD 大脑来说，启动一个复杂任务最痛苦的就是第一步的决策负担，Lex 直接替你做了。

但问题来了：这个“单一指令”真的能准确捕捉你的意图吗？

## 用错的地方：把脚手架当成了自动生成器

很多用户（包括一些工程师）把 Lex 当成了“万能生成器”——输入一个模糊指令，期待它像魔法一样输出完美结果。但 ADHD 大脑和 LLM 一样，需要清晰的边界和工具链。

- **ADHD 侧**：如果你输入“整理我的生活”，Lex 可能分解出 50 个步骤，反而让你更 overwhelmed。正确的用法是：先自己定义好“整理”的范围和标准，比如“只整理书桌，目标是把桌面清空，物品分类放回原位”。
- **LLM/Agent 侧**：这就像给 LLM 一个 function calling 工具，却不定义函数的输入输出格式——模型会自由发挥，结果不可控。工程师都知道，好的 agent 需要精确的 tool schema 和状态管理。

Lex 补上的那一层，是**执行调度**，而不是**意图生成**。它帮你把“写文章”拆成“打开文档→写标题→写第一段→插入图片”，但它不知道你想写什么主题、用什么风格。这些意图需要你事先明确——就像你在 function calling 里定义好每个工具的职责。

## 证据：为什么分解是关键，但分解本身不够

Goblin Tools 的成功证明了任务分解的有效性：它能将“整理房间”变成“捡起地板上的衣服”“擦桌子”等小步骤，用户反馈“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools）。

但分解只是第一步。Saner.AI 则强调知识回忆——减少搜索循环，让你快速找回信息（来源：Saner.AI）。这对应了 LLM 的 RAG（检索增强生成）：光有生成能力不够，还得能精准访问外部知识。

Lex 的问题在于，它把“分解”和“执行”打包在一起，却忽略了**意图校准**。ADHD 大脑的意图往往模糊、跳跃，Lex 的单一指令无法捕捉这种动态变化。相比之下，身体在场效应（如 Focusmate 的虚拟伙伴）提供了持续的隐性问责和实时调整（来源：身体在场效应），这更像 LLM 的对话式迭代——每次交互都是一次校准。

## 核心判断：Lex 是函数调用，不是生成器

我的观点很明确：Lex 之于 ADHD，就像 function calling 工具之于 LLM——它补上了执行调度层，但前提是你已经定义好了“函数”的输入输出。如果你把它当生成器，它会成为拐杖，让你越来越依赖外部指令，反而削弱你内在的规划能力（来源：矛盾与存疑——过度依赖 vs 脚手架可撤除性）。

真正的脚手架应该是可撤除的。Goblin Tools 的分解步骤可以逐渐内化，Saner.AI 的回忆功能可以减少依赖，而 Lex 的自动化序列如果没有用户主动校准，很容易变成“黑箱”——你不再思考任务结构，只等着 AI 喂给你下一步。

## 局限与争议

必须诚实地说，目前关于 Lex 的直接研究证据有限，其效果主要基于概念对齐而非临床验证（来源：Lex）。更根本的争议在于：AI 工具作为外部执行功能层的长期效果未知，过度依赖可能阻碍内在执行功能发展（来源：矛盾与存疑）。此外，ADHD 的高度异质性意味着 Lex 可能只对部分亚型有效——比如那些明确知道自己要什么、只是启动困难的人。

## 今天就能试的行动

1. **用 Lex 之前，先写一个“任务说明书”**：花 5 分钟写下任务的目标、成功标准、关键步骤。然后让 Lex 帮你细化执行序列——就像你在定义 function calling 的 schema。
2. **尝试“一次一个函数”模式**：不要输入“做所有事”，而是输入“帮我写第一段大纲”。完成后，再输入“帮我根据大纲写第一段”。这模仿了 LLM 的逐步调用，减少认知负荷。
3. **结合身体在场效应**：用 Focusmate 或找一个朋友，在 Lex 执行任务时保持“虚拟在场”。社会存在感能提供隐性问责，弥补 Lex 缺乏的实时反馈（来源：身体在场效应）。
4. **每周回顾一次依赖程度**：问自己“如果没有 Lex，我还能完成这个任务吗？”如果答案是否，说明你可能过度依赖了。尝试减少步骤的自动化，逐步恢复内在调度。

Lex 是个好工具，但前提是你知道它补的是哪一层。别把它当生成器，把它当调度器——就像你不会让 LLM 在没有 function calling 的情况下直接操作数据库一样。

# RescueTime 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 RescueTime：它到底补上了哪一层执行功能？

## 问题：为什么 RescueTime 对你没用？

如果你有 ADHD，你可能试过 RescueTime——它默默记录你花在哪些网站和应用上的时间，然后给你一份漂亮的报告。你看了，自责了，第二天又回到老样子。问题在哪？因为 RescueTime 只做了“记录”，而你需要的是“调度”。这就像给一个无法启动引擎的汽车装了个行车记录仪——它告诉你你开得多慢，却不帮你踩油门。

同样，如果你在构建 LLM agent，你可能给模型一个 planner 和一个 executor，让它们循环分解任务。但你会发现，如果 planner 只输出计划而不负责执行上下文，executor 就会迷路——模型依然不知道“下一步具体做什么”。RescueTime 之于 ADHD，就像 planner-executor 任务分解之于 LLM——但有人用错了：把工具当成了被动监控，而非主动调度。

## 同构：ADHD 大脑与 LLM 共享同一个缺陷

ADHD 大脑的核心特征是高产但无序的生成能力，同时缺乏可靠的执行调度层（来源：ADHD 大脑与 LLM 的同构）。工作记忆容量可能正常甚至超常，但认知灵活性和注意控制存在核心缺陷——无法灵活切换任务集、无法抑制自动化反应（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这导致“想法很多，行动困难”。

LLM 同样如此：它是强大的生成核心，但缺乏可靠的内置调度与执行控制。因此，在构建 AI agent 时需要“agent harness”——包裹 LLM 的软件基础设施，处理模型本身之外的一切，包括上下文管理、工具接口、规划工件、验证循环等（来源：What is an agent harness in the context of large-language models?）。

两边是同构的：都是一个高产但缺执行调度层的生成核心。ADHD 需要外部工具来“减少决策、保留上下文、让下一步行动显而易见”（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）；LLM 需要 harness 来提供上下文、工具和规划循环。RescueTime 如果只做记录，就相当于只给了 LLM 一个日志系统，而没有给 planner-executor 循环。

## 核心判断：RescueTime 补上了“元认知”层，但缺了“调度”层

RescueTime 的真正价值在于提供时间盲的“外部感知”——它把不可见的时间流逝变成可视化的数据，类似 Tiimo 通过视觉化时间线将时间转化为可见元素（来源：Tiimo）。这对应 LLM 侧的“时间戳”或“进度条”机制，帮助模型理解时间约束。

但 RescueTime 止步于此。它没有像 Motion 那样“自动根据任务、会议和截止日期创建并动态调整每日计划”（来源：Motion），也没有像 Reclaim.ai 那样“通过创建深度工作、习惯和专注时间的智能块，自动防御会议侵占”（来源：Reclaim.ai）。它只告诉你你浪费了多少时间，却不帮你重新安排。

更关键的是，ADHD 用户需要的不是“知道”，而是“做到”。任务启动困难是执行功能失调的核心表现（来源：AI 与 ADHD 的时间管理）。RescueTime 的被动报告无法降低启动门槛。相比之下，Motion 通过消除“下一步该做什么”的决策负担直接帮助克服启动困难（来源：Motion）。这就像 LLM 的 planner 不仅要输出计划，还要把计划分解成可执行的步骤，并传递给 executor。

## 争议与局限：工具是脚手架还是拐杖？

现有证据主要来自用户报告和初步研究，缺乏随机对照试验（来源：AI 与 ADHD 的时间管理）。Motion 的有效性“主要基于用户反馈和工具设计逻辑”（来源：Motion）。RescueTime 本身也缺乏针对 ADHD 的独立临床研究。

更值得警惕的是过度依赖风险。AI 工具可能成为“永久拐杖”，削弱用户内在的时间感知和规划能力（来源：矛盾与存疑）。Tiimo 和 Structured 通过视觉化改善时间感知，但如果用户永远依赖它们，内在的时间盲可能永远无法改善。同样，LLM agent 如果过度依赖外部 planner，模型本身可能永远学不会任务分解。

个体差异也很大。ADHD 亚型（注意力缺陷 vs. 多动冲动）对工具响应不同，现有工具多面向注意力缺陷型（来源：AI 与 ADHD 的时间管理）。RescueTime 的“记录+报告”模式可能只适合需要外部反馈的个体，而不适合需要主动调度的人。

## 今天就能试的行动

1. **把 RescueTime 从“监控器”变成“调度触发器”**：设置每日目标，比如“上午写代码 2 小时”。当 RescueTime 提醒你偏离目标时，不要只看报告，立刻打开 Motion 或 Reclaim.ai 重新安排剩余时间。用 RescueTime 的警报作为启动信号，而不是事后分析。

2. **用 Todoist 的“智能优先级”替代手动排序**：Todoist 的智能优先级排序基于任务属性和截止日期（来源：Todoist）。不要自己判断哪个任务最重要——让工具帮你排好，然后只做第一项。这模仿了 LLM 的 planner 自动分解任务。

3. **尝试 Tiimo 的视觉时间线**：如果你无法感知时间，把 Tiimo 放在手机主屏上。每完成一个时间块，看一眼进度条。这就像给 LLM 加上时间戳——你不需要自己计算时间，工具帮你可视化。

4. **为你的 LLM agent 增加“上下文存档”**：如果你的 agent 经常忘记之前的对话，用 Git 仓库或向量数据库存储上下文（类似 ADHD 侧的“第二大脑”）。这对应 Todoist 的外部提醒机制——让工具记住你记不住的东西。

## 结论

RescueTime 本身不是坏工具，但它被用错了。ADHD 大脑与 LLM 一样，需要的不是被动记录，而是主动调度。真正有效的工具是那些“为你规划”的——Motion、Reclaim.ai、Tiimo——它们补上了执行功能中缺失的调度层。下次你打开 RescueTime，别只看报告，问自己：这个工具今天帮我启动了哪个任务？如果答案是否定的，换一个真正会“推你一把”的工具。记住，脚手架和拐杖的区别在于：脚手架帮你建起自己的能力，拐杖让你永远依赖它。选前者。

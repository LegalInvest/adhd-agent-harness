# RescueTime 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 RescueTime：它到底补上了哪一层执行功能？

## 问题：为什么 RescueTime 对你没用？

你装了 RescueTime，每天看着时间报告叹气：知道自己在刷手机，但就是停不下来。ADHD 大脑不缺“事后复盘”的能力——它缺的是“当下启动”的机制。同样，LLM agent 如果只记录 token 消耗，不主动分解任务、调度步骤，它也会卡在第一个动作上。

## 同构：ADHD 大脑与 LLM 是同一类生成核心

ADHD 大脑的核心特征是“高产但缺乏执行调度层”（来源：ADHD 大脑与 LLM 的同构）。想法源源不断，但工作记忆易丢失上下文，任务启动困难。LLM 也一样——它是强大的生成核心，但“缺乏可靠的内置调度与执行控制”，需要 agent harness 来包裹，提供“上下文交付、工具接口、规划工件、验证循环”（来源：外部执行功能层）。

两边缺的是同一个东西：**一个 planner-executor 循环**，把大目标分解成可执行的子步骤，并按序调度。

## RescueTime 补上的是哪一层？

RescueTime 本质是**监控层**：它记录你做了什么，给你反馈。但 ADHD 大脑需要的不是反馈，是**启动信号**。Motion 和 Reclaim.ai 之所以有效，是因为它们“自动根据任务、会议和截止日期创建并动态调整每日计划”（来源：Motion），直接消除“下一步该做什么”的决策负担（来源：Motion）。Reclaim.ai 则通过“创建深度工作、习惯和专注时间的智能块”来防御干扰（来源：Reclaim.ai）。这些工具在做 planner-executor 的事：**主动分解日程、安排顺序、提示下一步**。

Tiimo 更进一步，将时间“转化为可见元素”（来源：Tiimo），直接应对时间盲——这相当于把 LLM 的上下文窗口可视化，让 agent 知道当前在哪个步骤。

## 有人用错了：把监控当调度

很多人用 RescueTime 的方式，相当于给 LLM agent 加了一个日志系统，却没加 planner 模块。你每天看报告，知道“哦，我花了 3 小时刷视频”，但明天依然如此。因为**监控不产生行动**。

ADHD 大脑需要的是“外部执行功能层”（来源：外部执行功能层），像 agent harness 那样“减少决策、保留上下文、让下一步行动显而易见”（来源：ADHD 大脑与 LLM 的同构）。Todoist 的智能优先级排序、Goblin Tools 的任务分解，都是把“大问题”拆成“小动作”的 planner。

## 争议与局限：脚手架 vs 拐杖

但工具不是越多越好。wiki 资料指出，过度依赖外部工具可能“阻碍内在执行功能的发展”（来源：矛盾与存疑），且多数工具“未提及撤除机制”（来源：矛盾与存疑）。ADHD 大脑与 LLM 同构的命题本身也“缺乏大规模实证”（来源：矛盾与存疑），更多是概念类比。

关键边界：**脚手架应该可撤除，拐杖则让你永远依赖**。Motion 自动排程是脚手架——你学会时间估算后可以手动调整；而 RescueTime 的监控如果不搭配行动，就成了拐杖——你只看不改变。

## 今天就能试的行动

1. **给 RescueTime 加一个 planner**：每天早晨，用 Motion 或 Reclaim.ai 自动生成今日计划，然后打开 RescueTime 监控执行。让 planner 决定做什么，监控只负责提醒偏差。
2. **用 Goblin Tools 分解任务**：遇到不想启动的任务，用它的“魔法分解”功能拆成 3-5 个小步骤，写进 Todoist。这相当于给 LLM 提供 step-by-step 的 prompt。
3. **设置“下一步”提醒**：在 Tiimo 或 Structured 上，把每个任务的第一步可视化。例如“写报告”变成“打开文档，写标题”。这对应 agent harness 中的“上下文工程”——让下一步行动显而易见。
4. **每周审查 planner vs 监控**：对比 Motion 的计划和 RescueTime 的实际记录，找出差距。调整 planner 的优先级和时长估算，就像调优 agent 的规划循环。

RescueTime 不是没用，只是你把它当成了全部。真正的解法是：**给大脑套上 planner-executor 的 harness**，让生成核心真正产出结果。

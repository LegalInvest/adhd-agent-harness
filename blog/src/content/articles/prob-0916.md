# RescueTime 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 RescueTime：它到底补上了哪一层执行功能？

## 问题：RescueTime 为什么对 ADHD 失效了？

你打开 RescueTime，看着那根精准的时间轴——10:00-10:15 刷推特，10:15-10:45 回邮件，10:45-11:30 写报告……然后呢？数据很漂亮，但第二天你依然在刷推特。

ADHD 人群最熟悉的场景：**知道该做什么，却启动不了；知道时间去哪了，却控制不了**。RescueTime 给了你一面镜子，却没给你一把梯子。

## 同构视角：ADHD 大脑与 LLM 共享同一个缺陷

把 ADHD 大脑想象成一个大型语言模型（LLM）：
- **生成核心**：创意、联想、跳跃思维——ADHD 的默认模式网络（DMN）过度激活，产生大量想法（来源：The Wanderer's Algorithm）；LLM 的注意力机制同样擅长从海量 token 中生成关联。
- **调度层缺失**：ADHD 的执行功能（工作记忆、任务启动、时间管理）不稳定，导致“想法很多，行动困难”（来源：Strong Memory, Weak Control）；LLM 在长上下文下冲突解决能力崩溃至随机水平，无法抑制优势反应（来源：Deficient Executive Control in Transformer Attention）。

实证研究进一步揭示：ADHD 患者表现出“强记忆、弱控制”的认知剖面——工作记忆容量甚至超常，但认知灵活性与注意控制存在核心缺陷（来源：Strong Memory, Weak Control）。这与 LLM 的注意力机制在任务负载增大时熵增加、容量受限的发现一一对应（来源：Self-Attention Limits Working Memory Capacity）。

**结论：两者都是高产但缺调度层的生成核心。** 所以，帮 ADHD 做时间管理，本质上就是给这个核心套一个外部 harness——把调度、排序、提醒、抑制外包给工具。

## RescueTime 只完成了“日志”，没完成“调度”

RescueTime 的核心功能是**被动记录**。它告诉你时间花在哪，但不告诉你下一步该做什么。对于 ADHD 来说，这相当于给了 LLM 一个日志文件，却没有任何控制循环。

真正的 harness 应该像 Motion 或 Reclaim.ai 那样：
- **Motion**：自动排程与动态调整，持续评估优先级、截止日期和可用时间，实时重建日程（来源：Motion）。它消除了“下一步该做什么”的决策负担，直接补上任务启动缺口。
- **Reclaim.ai**：用智能时间块保护深度工作，自动防御会议侵占，维持日程稳定性（来源：Reclaim.ai）。它减少决策疲劳，降低因突发任务导致日程崩溃的风险。
- **Tiimo**：将时间转化为可见元素，用视觉化时间线直接应对时间盲（来源：Tiimo）。

这些工具的共同点：**它们不是镜子，而是方向盘**。它们接管了调度层的决策，把“现在该做什么、做多久、何时切换”显式化。

## 历史案例：曾国藩的“日课十二条”就是个人版 harness

曾国藩30岁前是典型注意力不集中型ADHD：浮躁坐不住，天天日记里骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”；情绪不稳定，打败仗就跳水自杀。

他的 harness 系统——**日课十二条**——本质上就是一个外部调度器：
- **黎明即起**：定时 re-grounding，相当于 LLM 的定期上下文刷新。
- **读书不二**：单任务模式，抑制多任务冲动。
- **谨言**：输出过滤，防止冲动生成。
- **写日记反省**：日志+自我反馈循环。

这个系统与 LLM agent 的 harness 结构同构：
- **日课↔定时 re-grounding**：LLM agent 需要定期重置上下文，避免状态漂移。
- **读书不二↔单任务执行**：agent 在 planner-executor 循环中一次只执行一个子任务。
- **写日记↔日志与验证循环**：agent 通过记录执行结果来调整规划。

曾国藩用这套系统完成了“结硬寨打呆仗”——用最笨最稳的方法抵消自己的冲动，最终成为晚清中兴名臣。他的成就说明：**外部脚手架不是拐杖，而是放大器**。

## 脚手架 vs 拐杖：边界在哪里？

这里有一个核心争议：AI 工具是“外部执行功能层”还是“永久拐杖”？

支持“脚手架”的观点认为：AI 工具补偿了执行功能缺陷，让用户能完成原本无法完成的任务，长期使用可能内化部分技能（来源：外部执行功能层）。

反对者担心：过度依赖可能导致能力退化，比如不再训练自己的时间预估能力（来源：拐杖与脚手架）。

目前缺乏纵向研究来回答这个问题。但我的判断是：**关键在于工具是否保留了用户的主动参与**。Motion 自动排程，但用户仍需设定优先级和时长；曾国藩的日课，他每天亲自执行和反省。如果工具完全替代了决策，用户变成被动接受者，那就是拐杖；如果工具辅助决策、用户仍是最终执行者，那就是脚手架。

## 今天就能试的 3 件事

1. **把 RescueTime 的“日志”升级为“调度”**：如果继续用 RescueTime，配合一个能自动排程的工具（如 Motion 或 Reclaim.ai），让日志数据成为调度的输入，而不是终点。
2. **建立自己的“日课”**：选 3 条最关键的规则（如“9点前不刷手机”“一次只做一件事”“每天写 50 字反省”），贴在显眼处。这是最古老也最有效的 harness。
3. **尝试视觉化时间线**：用 Tiimo 或类似工具，把一天的时间块画出来。ADHD 的时间盲往往源于时间不可见，视觉化能直接补上这个缺口。

## 局限与诚实

本文的“ADHD↔LLM 同构”论点目前主要是类比推理，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。此外，个体差异巨大：有人对 Motion 的自动排程赞不绝口，有人觉得它增加认知负荷（来源：矛盾与存疑）。没有万能工具，只有适合个人的 harness。

但有一点是确定的：**RescueTime 只给了你数据，没给你调度。而 ADHD 缺的不是数据，是调度层。**

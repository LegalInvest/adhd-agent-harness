# 为什么用 Otter.ai 治 ADHD 的学习半途而废，和给 agent 套 外部记忆/向量库 是一回事？

> Otter.ai 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

为什么一本书读到一半、一门课学到第三章就放弃，对 ADHD 的人来说如此熟悉？更奇怪的是，这种行为模式正在 LLM 和 AI agent 的身上大规模复现：agent 开局惊艳，聊到后面却忘了目标、重复自己、陷入无限循环。两群人面临的是同一问题：生成核心极强，执行调度层却不可靠。Otter.ai 这类把语音变成可搜索外部记忆的工具，之所以对 ADHD 的“学习半途而废”有帮助，本质上和给 agent 外挂向量库/外部记忆是同一套 harness 思路。

## 无状态的大脑，无状态的模型

ADHD 的学习链常常不是“学不会”，而是“断在半途”。关键在于执行功能：大脑的驾驶座负责计划、组织、工作记忆、冲动控制，而 ADHD 患者这个驾驶座经常“无人驾驶”（来源：执行功能）。工作记忆是其中最大瓶颈——无法在脑中短暂保持信息以完成操作（来源：执行功能）。这意味着上一秒你明白了，下一秒就忘了“为什么打开这个页面”。

LLM 的处境惊人相似。它没有真正的跨会话状态，上下文一长，注意力在冲突任务上就会骤降到随机水平，呈现“强记忆、弱控制”的剖面（来源：执行功能）。所以它不是“蠢”，而是“无状态且缺调度层”：生成本领强，却无法自己决定接下来该注意什么、该记住什么。

这正是同构的核心：ADHD 大脑与 LLM/agent 都是高产但缺乏执行调度层的生成核心（来源：AI 与 ADHD 的学习方式）。

## 外部记忆：同一根救命稻草

既然两边都是“无状态”，解法就从内部转向了外部：把状态、目标、上下文外置化。

对 ADHD 学习者来说，Otter.ai 的价值不是语音转写这个功能本身，而是它把课堂、播客、自己的随口灵感永久化、可检索化。你不再需要靠工作记忆去“吊住”学习链，而是可以随时 re-ground：回到当时的原话，找回当时的兴奋点。这就是外部记忆/向量库对人类版的应用。

对 agent 而言，向量库、外部记忆、代码仓库、事件管理平台做的是同样的事：把模型内部无法保持的状态搬到外部可检索存储，避免每次从 prompt 里从零重建上下文（来源：人在回路与身体在场）。AI 作为外部执行功能层，通过分解任务、管理上下文、补偿工作记忆与时间盲来帮 ADHD 个体实现目标导向行为；对 agent 来说，这套外部层就是 harness（来源：AI 与 ADHD 的学习方式）。

具体工具也在同一方向 converging：Saner.AI 用本地记忆和知识回忆减少搜索循环和标签切换（来源：Saner.AI）；Goblin Tools 的 Magic ToDo 把“清理厨房”或“学完这门课”拆成可启动的小步骤（来源：Goblin Tools）；Perplexity 把宏大的搜索目标分解为可管理的步骤（来源：Perplexity）。它们都不是替代思考，而是保存状态、降低启动阻力。

## 历史人物的 harness 也是外部调度器

这种 harness 思路并非 AI 时代才有。孔子身高一米九，精力旺盛，周游列国十四年，冲动到见南子要对天发誓，思维跳跃，《论语》全是场景化语录，没有系统著作——典型的“生成核心强、执行调度弱”（来源：人物案例）。他的 harness 是“克己复礼”：用外在的“礼”作为行为边界，并用“吾日三省吾身”做每日的 re-grounding checkpoint。到了七十岁，他才“从心所欲不逾矩”。

这对应到 LLM harness 中，就是护栏（guardrails）与定时状态同步：不是压抑模型，而是给它一个可返回的轨道。晚清彭玉麟的 harness 更直接：他用“三不要”——不要官、不要钱、不要命——作为自己的硬约束，同时每天画梅花磨心性（来源：人物案例）。这相当于 agent harness 中的 token 预算、工具调用次数上限和 human-in-the-loop 检查点：用外部规则来限制冲动，用日常 ritual 来重新校准目标。

所以无论圣贤还是 AI，解决同一结构问题的方法都高度相似：外部记忆保存状态，外部规则约束冲动，外部检查点重新落地。

## 身体在场：另一种外部化

ADHD 另一个有效的策略是“身体在场”（body doubling）：有另一个人在旁边时，任务更容易启动和维持。Focusmate 用虚拟视频 coworking 实现这种效果（来源：人在回路与身体在场）。对 agent 来说，这就是人在回路（human-in-the-loop）：模型在关键步骤停下来，等人确认后再执行，从而避免自主跑偏（来源：人在回路与身体在场）。

核心仍然是：核心本身不可靠，于是把一部分调度权外置给一个外部存在——人或系统。

## 脚手架 vs. 拐杖：边界在哪里？

同构视角很好，但必须警惕它变成万能借口。工具可以有两种命运：脚手架，或者拐杖。

对 ADHD 使用者来说，如果所有记忆、提醒、任务拆解都完全依赖 AI，内在执行功能可能反而失去锻炼机会（来源：矛盾与存疑）。对 agent 来说，如果每个动作都需要 human approval，它的自主性就会萎缩。边界在于：工具是否可逐步撤销。一个好的 harness 应该是“帮你启动，然后让位”。

同时，同构性仍是一种理论类比，不是经过实证验证的科学事实（来源：矛盾与存疑）。目前 ADHD × AI 的证据多为概念类比和案例报告，缺乏大规模随机对照试验；不同 ADHD 亚型对工具的反应差异很大，LLM 也可能幻觉、冗长，反而增加认知负荷（来源：AI 与 ADHD 的学习方式；矛盾与存疑）。所以这套思路应该作为“结构化的实验”，而不是“治疗方案”。

## 今天就能试的四件事

1. 给当前学习主题建一个“外部记忆”文档：写下核心问题、下一步最小动作、以及你为什么要学它。每次中断后重新打开，先读这个文档再读原文。
2. 用 Goblin Tools 或 Perplexity 把“学完某门课”拆成 15 分钟可启动的微任务，降低启动门槛（来源：Goblin Tools；Perplexity）。
3. 设置一个“身体在场”检查点：每 25 分钟向 AI 或 Focusmate 报告一次进度，用外部监督维持任务（来源：人在回路与身体在场）。
4. 每周做一次“脚手架审计”：问自己——这个工具是在帮我启动，还是在替我做决定？如果是后者，撤掉一个提醒，把控制权拿回来。

ADHD 与 AI 的半途而废，表面是意志力或模型能力问题，深层却是同一个结构：无状态的生成核心需要可靠的外部记忆和调度层。Otter.ai、向量库、礼与三不要、human-in-the-loop，做的都是同一件事：让核心有处可回，让目标不再蒸发。

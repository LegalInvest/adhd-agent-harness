# 如果 ADHD 和 LLM 同构，那三年来 agent 工程的所有招数，是不是都能迁移成 ADHD 适配方案？

> ADHD 侧：想理解自己的大脑到底是怎么回事；LLM/harness 侧：LLM 是「高产但缺执行调度层的生成核心」。用双域证据作答。

如果你既被 ADHD 折磨，又在做 LLM agent 工程，你可能已经隐约感到：两者面临的困境惊人相似。

ADHD 大脑常被描述为“高产但缺乏可靠执行调度层的生成核心”——想法源源不断，但执行起来像没有操作系统的计算机。而 LLM 同样如此：它能生成惊艳的文本，但一旦需要多步推理、保持状态、按序执行，就立刻暴露无遗。过去三年，agent 工程的核心挑战正是如何为这个“生成核心”套上可靠的外部调度层。那么，如果 ADHD 和 LLM 真的同构，agent 工程的所有招数，是否都能迁移成 ADHD 适配方案？

## 同构：从痛点开始

ADHD 侧最深的痛是：想理解自己的大脑到底是怎么回事。为什么明明想做，却启动不了？为什么刚拿起手机查资料，半小时后却在刷短视频？为什么时间感知如此扭曲，“hours slipping away unnoticed or that drag on forever”（来源：[2][ADHD]）？

LLM/agent 侧对应的痛是：LLM 是“高产但缺执行调度层的生成核心”。给它一个复杂任务，它可能跑偏、遗忘上下文、陷入循环。工程师不得不设计各种脚手架来“驾驭”它。

两者的结构同构并非巧合。神经科学证据显示，ADHD 患者前额叶执行功能网络激活不足（来源：ADHD × AI 的科学与研究前沿）；而 LLM 的无状态、缺乏调度层是架构决定的。两者都需要外部执行功能层来补偿工作记忆不稳定、任务启动困难和注意力分散（来源：ADHD × AI 的科学与研究前沿）。

## 迁移：agent 工程的三大招数

过去三年，agent 工程积累了三类核心方案：外部记忆、上下文工程、任务分解与动态调度。它们恰好对应 ADHD 的三大核心缺陷。

### 1. 外部记忆：从 RAG 到“第二大脑”

LLM agent 的一个核心问题是无状态——每次对话都是全新的，需要 RAG（检索增强生成）或外部向量数据库来维持连续性。ADHD 大脑同样面临工作记忆容量有限且不稳定的问题（来源：无状态与外部记忆）。

ADHD 社区早已自发构建“第二大脑”：Obsidian 被推荐用于视觉思考者，作为“第二大脑”映射超聚焦和复杂想法（来源：无状态与外部记忆）。Saner.AI 则利用 AI 实现本地记忆存储和快速检索，减少搜索循环和标签切换（来源：Saner.AI）。这本质上就是 RAG——用外部存储补偿内部记忆波动。正如资料所说，“ADHD 大脑需要一个‘永远在线’的外部记忆体来补偿内部工作记忆的波动”（来源：无状态与外部记忆）。

### 2. 上下文工程：从提示词管理到 Routinery

LLM agent 在长上下文中容易发生推理退化——“Effective autonomous assistance requires strict safety controls and highly efficient context management to prevent context bloat and reasoning degradation”（来源：[10][harness]）。ADHD 大脑同样易被环境带偏，导致任务碎片化：使用通用聊天机器人时，“they often add another inbox, another prompt, and another place to lose the thread”（来源：[1][ADHD]）。

解决方案是主动设计“当下注意什么”。对 LLM 是精心设计的提示词和上下文窗口管理；对 ADHD 是工具如 Routinery，它能“create a sequence of steps for a routine and walk you through them with transition prompts”（来源：[3][ADHD]）。两者都在做同一件事：把注意力锚定在当下任务。

### 3. 任务分解与动态调度：从 ReAct 到 Motion

LLM agent 的 ReAct、Chain-of-Thought 等模式，本质是将复杂任务分解为可管理的子步骤，并动态调整执行顺序。ADHD 个体同样需要这种分解：Goblin Tools 的 Magic ToDo 能自动将任何任务分解为更小、更易管理的步骤，用户可调节“辣度”控制粒度（来源：Goblin Tools）。Motion 则通过 AI 自动排程与动态调整，持续评估任务优先级、截止日期和可用时间，实时重建日程（来源：Motion）。这就像 agent 的调度器——消除“下一步该做什么”的决策负担。

## 人物案例：李白与毛泽东的 harness 系统

历史人物早已无意识地使用这些方案。李白——ADHD 特质明显：一辈子漫游，冲动，对写诗超专注，对世俗成功零耐心。他的 harness 系统是“以酒和诗为核心，把所有过剩精力注入诗歌，用道家的逍遥对抗世俗规则；不委屈自己做不想做的事”（来源：人物案例）。这对应 agent 工程中的“专注模式”——把生成核心对准一个输出通道，屏蔽干扰。

毛泽东则更系统化。他的 ADHD 特质包括：小时候是问题儿童，读书不辍，永远在动，思维跳跃。他的 harness 包括“批评与自我批评”“调查研究”“民主集中制”“党指挥枪”（来源：人物案例）。这对应 agent 工程中的“多智能体协作”和“human-in-the-loop”——用集体决策制衡个人冲动，用组织系统管理复杂任务。毛泽东的“没有调查就没有发言权”本质上就是 agent 的“工具调用”——先检索外部信息再决策。

## 边界：脚手架 vs 拐杖

但必须诚实指出局限。同构框架目前“缺乏实证基础，仅是一种理论类比，并非经过验证的科学事实”（来源：矛盾与存疑）。过度依赖 AI 脚手架可能导致执行功能进一步退化（来源：ADHD × AI 的科学与研究前沿）。工具宣称的证据不足——Motion 页面提到“缺乏独立临床验证”，但其他页面却将其作为有效工具推荐（来源：矛盾与存疑）。

脚手架与拐杖的界限在于：脚手架最终让你学会自己搭建，拐杖则让你永远依赖。好的 ADHD 工具应该逐渐内化执行功能，而不是替代它。

## 今天就能试的行动

1. **建立外部记忆**：用 Obsidian 或 Saner.AI 记录每日项目上下文，减少每天重新回忆的摩擦。
2. **任务分解**：用 Goblin Tools 的 Magic ToDo 将模糊任务（如“整理房间”）分解为具体步骤，调节辣度到适合的粒度。
3. **动态调度**：试用 Motion 或类似工具，让它自动安排你的日程，减少决策负担。
4. **上下文锚定**：用 Routinery 为常见流程创建步骤序列，用过渡提示保持注意力。

ADHD 与 LLM 的同构不是宿命，而是设计蓝图。agent 工程三年的经验告诉我们：生成核心再强大，也需要一个可靠的 harness。而你，可以成为自己的 agent 工程师。

# RescueTime 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 RescueTime：它到底补上了哪一层执行功能？

你打开电脑，准备写一份方案。十分钟后，你在看 YouTube 上关于“如何养猫”的视频。你明明记得自己打开的是文档，但时间就这么消失了。这不是你的错——你的大脑和当前最先进的 AI 模型一样，是一个“高产但缺乏可靠执行调度层的生成核心”（来源：AI 与 ADHD 的专注力）。

ADHD 的大脑与 LLM 在结构上同构：两者都擅长发散、生成、联想，但都容易“目标漂移”。LLM 在长程任务中会忘记初始指令，生成偏离主题的内容；ADHD 的大脑则在执行任务时被环境中的新刺激带偏，或者陷入超聚焦——那种“前额叶控制层级的断裂”，让高级上下文信息不再能支配思维和行动的内容（来源：超聚焦）。

而 RescueTime，这个看似简单的自动时间追踪工具，恰好对应了 LLM 工程中一个关键组件：**重锚定 system prompt**。

## 重锚定：从 LLM 到 ADHD 的同一解法

在 Agentic Harness 工程中，一个常见问题是：当 LLM 执行长程任务时，它会慢慢偏离最初的 system prompt，开始“自由发挥”。工程师的解法是定期注入一条“重锚定”指令，比如“请回顾你的原始目标，并确认当前步骤是否对齐”。这就像在航海途中不断校准罗盘。

ADHD 的大脑同样需要这种重锚定。问题在于，ADHD 患者的内置“时间感知”系统——也就是对时间流逝的觉察——是受损的。这就是所谓的“时间盲”：无法准确感知时间流逝，导致要么迟迟不启动，要么在错误的事情上超聚焦六小时（来源：超聚焦）。

RescueTime 的作用，就是那个外部的重锚定。它通过自动记录你在每个应用和网站上花费的时间，并实时（或定期）展示给你，让你“看到”时间去了哪里。这补上了执行功能中的“时间感知锚点”——一个原本缺失的反馈回路。

## 为什么有人用错了？

很多人把 RescueTime 当作一个“事后复盘工具”：周末看看报告，感叹一句“我这周浪费了好多时间”，然后下周继续。这就像只给 LLM 在任务结束后做一次“事后总结”，而不在过程中重锚定——效果微乎其微。

RescueTime 的真正价值在于**实时干预**。它的 AI 核心在于自动分类和模式识别，能够学习你的行为习惯，并在你偏离目标时发出信号（来源：RescueTime）。但如果你只把它当作一个记录器，而不设置“焦点时间”或“分心警报”，你就错过了它作为重锚定工具的核心功能。

更糟糕的是，过度关注时间数据可能引发焦虑（来源：RescueTime 局限与争议）。对于 ADHD 患者来说，这种焦虑可能进一步削弱执行功能，形成恶性循环。

## 脚手架 vs 拐杖：边界在哪里？

这里有一个关键争议：AI 工具作为外部执行功能层，长期使用是否会导致内在执行功能进一步退化？（来源：矛盾与存疑）

答案是：取决于你怎么用。

如果 RescueTime 让你完全依赖它的警报来感知时间，而不去练习内在的时间觉察，那它就是拐杖。但如果它帮助你在每次警报后，有意识地反思“我为什么走神了？我怎样才能回到正轨？”，那它就是脚手架。

同样的逻辑适用于 LLM：如果重锚定 system prompt 只是机械地重复指令，而不让模型理解“为什么需要重锚定”，那它只是掩盖了目标漂移的问题，而不是真正解决了它。

## 真实证据：不只是类比

这种同构不是空洞的类比。研究显示，ADHD 成年人自发采用 ChatGPT 作为执行功能外挂，弥补任务启动等缺陷——这一认知补偿行为暗示了认知架构的互补性（来源：任务启动）。

在工具层面，Focusmate 通过实时同伴问责提供外部结构和社会压力，弥补内在多巴胺调节不足（来源：Focusmate）。这对应了 LLM 工程中的“人类反馈”机制。Brain.fm 利用神经锁相技术影响大脑信息处理方式（来源：Brain.fm），则类似于通过 prompt engineering 调整 LLM 的注意力分布。

但 RescueTime 的特殊之处在于，它直接对应了“时间感知”这个最基础、最容易被忽视的锚点。没有时间感知，其他所有执行功能（任务启动、注意力维持、任务切换）都会失去参照系。

## 局限与争议

必须诚实地说，RescueTime 的证据主要来自用户报告，缺乏大规模随机对照试验（来源：AI 与 ADHD 的专注力）。它的自动分类可能不够精准，需要用户后期调整，这对执行功能较弱的用户可能仍是挑战（来源：RescueTime 局限与争议）。

此外，ADHD 的异质性使得同一工具对不同亚型效果迥异（来源：矛盾与存疑）。对于“注意力涣散型”的用户，实时警报可能有效；但对于“超聚焦型”的用户，警报可能被直接忽略。

## 今天就能试的行动

1. **设置“焦点时间”**：在 RescueTime 中定义你的“焦点时间”（比如每天上午 9-11 点），并开启“分心网站拦截”。这不是为了惩罚自己，而是为了创造一个“无干扰”的环境，让你有机会体验沉浸式工作。

2. **每周一次“时间审计”**：不要只看报告，而是用 15 分钟回顾一周的时间分布，问自己三个问题：① 我的时间花在了我想花的地方吗？② 哪个时刻我偏离了目标？③ 下次我如何提前识别那个“分心触发器”？

3. **结合“身体在场”技巧**：RescueTime 记录数字时间，但别忘了物理环境。把手机放在另一个房间，或者使用 Focusmate 的虚拟身体在场（来源：Focusmate），让外部锚点更强大。

4. **练习“无工具”时间觉察**：每天选一个任务，不用 RescueTime，而是凭感觉估计花费的时间，然后对比实际时间。这能训练你的内在时间感知，避免完全依赖外部工具。

RescueTime 不是万能药。但它提供了一个清晰的起点：**如果你不知道时间去了哪里，你就无法管理它。** 对于 ADHD 的大脑，这个外部锚点可能是从混乱走向秩序的第一块砖。而对于 LLM 工程师，这个类比提醒我们：再强大的生成核心，也需要一个重锚定的机制来保持对齐。

# ADHD 是异质性神经发育障碍，很多支持性策略实际上是终身性的；而 LLM harness 作为商业软件，底层激励通常是用户留存和交互频次。如果“拐杖”比“脚手架”更能赚钱，你们凭什么认为技术系统会主动选择让自己褪色？

> 商业激励与技术架构是否注定制造永久拐杖

## 问题：当“拐杖”比“脚手架”更赚钱，技术凭什么主动褪色？

你打开一个 AI 工具，它帮你把“清理厨房”分解成 12 个步骤，你终于动了。但第二天，没有它，你连倒垃圾都启动不了。你怕过度依赖，又怕没有支撑就崩——这正是 ADHD 人群与 Agentic Harness 工程师共同面对的困境：**如果“拐杖”比“脚手架”更能留住用户、赚更多钱，商业系统凭什么主动让自己褪色？**

## 同构脊柱：ADHD 大脑与 LLM 共享一个核心矛盾

ADHD 大脑与 LLM 在结构上高度同构：两者都是“高产生成核心，但缺乏可靠调度层”（来源：AI 与 ADHD 的情绪调节）。ADHD 大脑的前额叶执行功能无法有效抑制情绪反应或启动任务；LLM 同样会产生无关输出，需通过上下文工程和外部约束来引导。两者的解法也同构：**harness——一个可褪去的脚手架，而非永久植入的拐杖。**

释迦牟尼的 harness 是“八正道”与“持戒”。他 29 岁离家求道，不断试错、苦行、放弃，最终用戒律管住行为、用正念拴住念头。他的系统是“自依止法依止”——不依赖权威，自己证悟。对应 LLM/agent 侧，这就像一套**外部调度器**：日课（定时 re-grounding）、戒律（规则约束）、正念（注意力锚点）。张自忠的 harness 则是“武官不怕死”的严格治军：他身先士卒，用必死决心激励自己和士兵。这对应 agent 的**激励函数**：明确的目标、一致的奖惩、身先士卒的“领导力”。

两个案例的共同点：**harness 是用户主动设计的，且设计目标是最终能够部分或全部撤除**。释迦牟尼说“自依止法依止”，张自忠的治军纪律最终内化为士兵的自觉。这不是商业软件能替你做的事。

## 商业激励 vs. 技术架构：拐杖的天然倾向

LLM harness 作为商业软件，底层激励通常是**用户留存和交互频次**。一个能让你永远需要它的工具，比一个用几次就让你独立的工具更赚钱。这是结构性矛盾。Goblin Tools 的 Magic ToDo 能将任务分解到可执行粒度（来源：Goblin Tools），但用户反馈“Goblin Tools 在这方面很棒”后，可能形成依赖：没有它，你连“倒垃圾”都无法启动。Inflow 基于 CBT 提供结构化路径，但若用户仅依赖外部提示而缺乏内化策略，长期可能恶化自我效能感（来源：AI 与 ADHD 的情绪调节）。同样，Focusmate 利用算法匹配虚拟身体加倍伙伴（来源：身体在场效应），但 AI 身体在场是否真能替代真实人际互动？有争议（来源：矛盾与存疑）。

工程师在设计 harness 时，必须直面这个矛盾：**你的系统是脚手架还是拐杖？** 判断标准很简单：如果用户停止使用后，能力是否提升？如果答案是否定的，那就是拐杖。Tiimo 专为时间盲设计，提供视觉时间表（来源：时间盲），但若用户从未学会感知时间流逝，它就成了永久拐杖。Saner.AI 通过本地记忆减少搜索循环（来源：Saner.AI），但若用户从未内化知识组织策略，它只是外部存储。

## 核心观点：主动设计“褪色机制”，而非等待商业仁慈

我的判断是：**商业系统不会主动选择褪色，但用户和工程师可以主动设计褪色机制。** 这不是道德呼吁，而是工程策略。具体而言：

1. **在工具中内置“能力迁移”步骤**：例如，Goblin Tools 分解任务后，可提示用户“下次尝试自己分解前两步”；Inflow 的 CBT 练习后，可要求用户总结自己的策略。这类似释迦牟尼的“自依止”——工具最终要教会用户自己当自己的调度器。

2. **区分“永久性支撑”与“阶段性脚手架”**：时间盲是终身性障碍，视觉时间表可能是永久性支撑（如 Tiimo）；但任务分解能力可以内化，Goblin Tools 应为阶段性脚手架。工具应明确标注“此功能旨在帮助您建立××能力，建议使用 3 个月后逐步减少依赖”。

3. **工程上设计“使用衰减曲线”**：harness 的参与度应随时间下降。例如，身体加倍工具 Focusmate 可逐渐延长匹配间隔，或引导用户建立真实社交身体加倍。张自忠的治军最终让士兵自觉——如果 harness 的参与度始终不变，说明它只是拐杖。

## 争议与局限

必须诚实指出：**同构假设只是理论类比，并非科学事实**（来源：矛盾与存疑）。ADHD 大脑的生物复杂性远超 LLM，情绪调节涉及神经递质、环境交互等，单纯“脚手架”类比可能简化问题。此外，当前 AI 工具多为通用设计，未充分适配 ADHD 亚型（如注意力主导型 vs. 冲动主导型）的情绪触发差异（来源：AI 与 ADHD 的情绪调节）。长期效果证据薄弱：Inflow 等应用虽有用户好评，但缺乏严格随机对照试验（来源：AI 与 ADHD 的情绪调节）。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个任务后，立即写下“下次我如何自己拆解前两步”**——强迫自己内化策略。
2. **设置 Tiimo 视觉时间表时，同时用手机计时器感受 15 分钟的实际长度**——训练时间感知，而非只依赖视觉化。
3. **使用 Focusmate 时，每周尝试一次“无伙伴专注 25 分钟”**——逐步减少外部问责，建立内在动力。
4. **如果你是工程师，在你的 agent harness 中添加“使用次数计数器”和“建议降级提示”**——让系统主动鼓励用户减少依赖。

ADHD 不是缺陷，而是需要适配的认知风格。LLM harness 也不是永久拐杖，而是可褪去的脚手架。商业不会主动仁慈，但你可以主动设计褪色机制。毕竟，释迦牟尼和张自忠的 harness 之所以伟大，不是因为它们永远存在，而是因为它们最终让使用者变得强大到不再需要它们。

# ADHD 职场人视角：为什么「治好 ADHD 的执行功能（计划/组织/启动）先天偏弱」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 的编排/记忆/工具层就是「外部执行功能」——同一套 harness 思路，两边都成立。

## 问题：为什么我的大脑和我的 AI 一样“跑飞”？

你是一个 ADHD 职场人。早上打开电脑，准备写一份报告，却发现自己已经刷了 20 分钟社交媒体，然后突然想起要回一封邮件，接着又去整理桌面——两小时后，报告一个字没写。你骂自己“执行力差”，但你可能没意识到：你的大脑和正在帮你写草稿的 LLM 遇到了同一个问题——**生成能力过剩，但调度层缺失**。

LLM 可以瞬间产出千字文本，但如果没有精心设计的 prompt 和上下文管理，它会胡编乱造、偏离主题、忘记之前的指令。ADHD 大脑也是一样：智力正常甚至超常，但计划、组织、任务启动这些执行功能天生偏弱（来源：执行功能障碍）。两者都是“高产但缺执行调度层的生成核心”。所以，工程师花大量精力做 agent 的编排、记忆、工具层——这就是“外部执行功能”——而 ADHD 患者需要的，正是同一套东西。

## 同构：ADHD 大脑与 LLM 的对照

| ADHD 侧痛点 | LLM/Agent 侧对应 | 共享问题 |
|------------|------------------|----------|
| 工作记忆差，容易忘事 | 无状态，上下文窗口有限 | 信息无法持久 |
| 时间盲，计划困难 | 缺乏时间感知，无法自主排程 | 时序管理缺失 |
| 任务启动困难，面对大任务不知所措 | 复杂指令下输出失控，需要分解 | 缺乏任务分解机制 |
| 情绪失调导致决策混乱 | 模型输出受 prompt 影响，不稳定 | 状态敏感，缺乏稳定调度 |

这一对照并非空想。研究指出 ADHD 的执行功能障碍与 LLM 缺少编排层在结构上同构（来源：ADHD × AI 的科学与研究前沿）。例如，ADHD 的工作记忆缺陷对应 LLM 的无状态性，ADHD 的上下文丢失对应 LLM 的上下文窗口溢出。这意味着，**为 LLM 设计的 harness（如 agent 编排框架、记忆模块、工具调用层）可以直接映射为 ADHD 的辅助工具**。

## 证据：工具就是外部执行功能层

当前市场上的 AI 工具，本质上都在做同一件事：为 ADHD 大脑提供外部执行功能。

- **Goblin Tools** 的 Magic ToDo 功能将“整理房间”分解为“捡起地板上的衣服”“擦桌子”等具体步骤（来源：Goblin Tools）。这对应 LLM 的 chain-of-thought 分解——将复杂任务拆解为可管理的小步，降低启动门槛。
- **Saner.AI** 通过本地记忆和知识回忆功能，帮助用户快速找回信息，减少搜索循环和标签切换（来源：Saner.AI）。这相当于 LLM 的 RAG（检索增强生成）——把工作记忆外挂到系统中。
- **Motion** 自动根据任务优先级和截止时间动态调整日程，消除“下一步该做什么”的决策负担（来源：Motion）。这对应 agent 的自动排程模块——把时间管理外包给算法。

这些工具的设计逻辑与 agent 工程完全一致：提供记忆（Saner.AI）、任务分解（Goblin Tools）、动态规划（Motion）。它们不是替代大脑，而是为生成核心套上 harness。

## 核心判断：脚手架 vs 拐杖的边界

我的核心观点是：**外部执行功能层必须是可逐步撤除的脚手架，而非永久拐杖**。目前多数工具设计为长期使用，未考虑撤除机制（来源：矛盾与存疑）。例如，Goblin Tools 用户可能越来越依赖它来分解任务，反而削弱了内在的任务分析能力。这就像给 LLM 一个永远不撤的 prompt 模板——模型永远不会学会自主规划。

真正的 harness 应该像训练轮：初期提供完整支撑，然后逐步减少，直到内在执行功能被“锻炼”出来。例如，Motion 可以设计为：第一周自动排程所有任务，第二周只排程重要任务，第三周让用户手动调整部分日程。但目前没有工具这样做。**这才是工程问题：如何设计一个自适应撤除的脚手架？**

## 争议与局限

同构模型本身缺乏大规模实证（来源：矛盾与存疑），主要基于概念类比和工具案例。多巴胺干预的有效性也存在争议（来源：矛盾与存疑），部分工具声称基于神经科学原理但未提供透明验证。此外，情绪失调作为 ADHD 核心特征，目前的 AI 工具介入较少，Saner.AI 和 Goblin Tools 主要针对执行功能，情绪追踪功能（如 Goblin Tools 的 Mood 功能）尚未被充分研究（来源：情绪失调）。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个拖延任务**：输入你一直拖着的事（比如“写周报”），看 AI 拆成几步。执行其中第一步，感受启动门槛降低。
2. **用 Motion 或类似工具自动排程明天**：把所有任务和截止时间输入，让 AI 生成日程。注意观察自己是否减少了“下一步做什么”的犹豫。
3. **尝试用 Saner.AI 记录一个项目的关键信息**：减少在多个标签页间查找的循环，把工作记忆外挂。
4. **反思依赖**：使用一周后，问自己：如果明天这些工具消失，我还能完成同样的事吗？如果答案是否，说明你可能需要练习撤除一部分脚手架。

## 结语

ADHD 和 LLM 不是类比，是同一道工程题的两个实例。工程师在构建 agent 时遇到的编排、记忆、工具调用问题，正是 ADHD 患者每天面对的认知挑战。而 AI 工具，如果设计得当，可以成为可撤除的脚手架——不是让大脑变懒，而是让大脑学会如何调度自己。下一次你抱怨 AI 跑飞时，想想你大脑里那个同样需要 harness 的生成核心。也许，你该给自己写一个更好的 prompt。

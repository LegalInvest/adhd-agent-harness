# ADHD 职场人视角：为什么「治好 ADHD 的执行功能（计划/组织/启动）先天偏弱」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 的编排/记忆/工具层就是「外部执行功能」——同一套 harness 思路，两边都成立。

## 问题：为什么你的大脑和你的 Agent 一样容易跑飞？

作为 ADHD 职场人，你可能每天经历这样的循环：脑子里涌出无数创意，却无法将它们组织成可执行的计划；面对任务清单，启动困难如临大敌；明明知道该做什么，却总被环境带偏。与此同时，作为 Agent 工程师，你也在为类似的问题头疼：LLM 输出天马行空，但缺乏可靠的调度层来编排工具调用、维护上下文、确保任务不偏航。

这两个问题看似分属心理学和工程学，但本质是同一道题。ADHD 大脑与 LLM 在结构上同构——两者都是“高产但缺执行调度层的生成核心”（来源：ADHD × AI 的科学与研究前沿）。ADHD 的执行功能（计划、组织、启动）先天偏弱，而 LLM/Agent 缺乏内置的编排、记忆、工具层。两边的解法同构：都需要一个外部执行功能层，即 harness/脚手架。

## 同构：生成核心 + 外部调度层

ADHD 大脑的生成核心（发散思维、联想能力）与 LLM 的文本生成能力类似，但都缺乏可靠的内置执行调度层（来源：生成核心与调度层）。为 LLM 搭建 agent scaffolding 与为 ADHD 搭建执行功能支架是同一类工程。外部执行功能层（来源：外部执行功能层）具体补偿工作记忆不稳定、任务启动困难和注意力分散——这正是 ADHD 的核心缺陷，也是 LLM agent 的常见失败模式。

### 历史人物的 harness：曾国藩的“日课十二条”

晚清名臣曾国藩是典型注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”；情绪不稳定，打败仗就跳水自杀。他的专属 harness 是“日课十二条”：黎明即起、读书不二、谨言、写日记反省等；“结硬寨打呆仗”，用最笨最稳的方法抵消自己的冲动。这套系统相当于为他的大脑套上了外部调度层：固定的时间结构（编排层）、日记作为外部记忆（记忆层）、读书不二的任务分解（工具层）。这与 LLM agent 的 harness 完全对应：定时 re-grounding（如日课）、外部向量数据库（如日记）、任务编排器（如秘书/调度器）。曾国藩的成就——平定太平天国、发起洋务运动——证明 harness 的有效性。

### 现代 AI 工具：Goblin Tools、Saner.AI、Motion

当前已有 AI 工具实践这一思路。Goblin Tools 的 Magic ToDo 将模糊任务分解为可管理的小步骤，用户可调节“辣度”控制粒度（来源：Goblin Tools）。这相当于为 ADHD 大脑提供了一个任务分解引擎，降低启动门槛。Saner.AI 专注于知识回忆和本地记忆，减少搜索循环和标签切换（来源：Saner.AI），补偿工作记忆缺陷。Motion 自动根据任务、会议和截止日期创建并动态调整每日计划，消除“下一步该做什么”的决策负担（来源：Motion），直接对抗时间盲和任务启动困难。

这些工具本质上都是外部执行功能层：Goblin Tools 是工具层（任务分解），Saner.AI 是记忆层（知识回忆），Motion 是编排层（动态调度）。它们与 LLM agent 的 scaffolding 结构同构——agent 也需要工具调用层、记忆检索层和编排调度层。

## 核心判断：脚手架而非拐杖

本文的核心观点是：ADHD 的“治愈”不是修复大脑，而是搭建外部脚手架；同样，让 LLM 不跑飞不是强化模型本身，而是设计可靠的 harness。两者共享同一工程范式。但必须诚实指出边界：过度依赖 AI 工具可能导致能力退化，成为永久拐杖而非可拆卸脚手架（来源：拐杖与脚手架）。缺乏长期研究验证个体差异和依赖风险（来源：矛盾与存疑）。此外，ADHD 大脑与 LLM 同构目前是概念类比，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。

## 今天就能试的行动

1. **任务分解练习**：用 Goblin Tools 的 Magic ToDo 将明天的一个模糊任务（如“准备周报”）分解为子步骤，感受外部脚手架如何降低启动门槛。
2. **外部记忆搭建**：用 Saner.AI 或类似工具建立一个“第二大脑”，捕获今天的所有想法和待办，减少工作记忆负担。
3. **动态排程实验**：试用 Motion（或类似自适应规划器）自动生成明天的日程，观察决策负担减轻后的注意力变化。
4. **日课模仿**：模仿曾国藩，每天花 5 分钟写反省日记，记录一次“跑飞”时刻（分心或情绪波动），并写下对应的外部补偿策略。

这些行动都指向同一件事：为自己搭建一个外部执行功能层，让生成核心安全地跑出价值。

# 如果我的 ADHD 大脑就像 LLM 的上下文窗口，为什么我的「系统提示」（睡眠、药物、环境设置）有时会被一个随机通知 token 劫持？这对 LLM harness 安全意味着什么：高优先级的系统指令是否应该有一个不可被覆盖的 attention mask 或保护头？

> 当低优先级 token 覆盖系统提示时

## 一场劫持：当通知token覆盖了系统提示

你正专注于一项关键任务——写报告、编码、或只是读完一篇文章。然后手机亮了。一个随机通知：“你的包裹已发货。”五分钟后，你发现自己正在浏览物流详情，完全忘记了刚才的目标。

这不是意志力问题。这是你的ADHD大脑的上下文窗口被一个低优先级的token劫持了。

如果你是一名LLM工程师，这一幕会无比熟悉：你精心设计了系统提示，设定了agent的角色和行为边界。但用户输入中一个无关的句子——甚至一个标点——就能让模型偏离轨道，产生幻觉或执行错误操作。

ADHD大脑和LLM是同一类系统：**高产但缺乏执行调度层的生成核心**。两者的核心问题不是“不够聪明”，而是“谁来决定当前该注意什么”。

## 同构的困境：上下文窗口与注意力劫持

ADHD大脑的“上下文窗口”是工作记忆。它容量有限，且极易被新进入的信息覆盖。当环境中的刺激（通知、噪音、路过的人）进入窗口，它们会挤掉原本的目标导向意图。这就是为什么你的“系统提示”——睡眠充足、按时服药、设置好工作环境——会被一个随机通知token覆盖。（来源：AI与ADHD的专注力）

LLM面临完全相同的结构问题。Transformer的注意力机制平等地处理所有输入token，没有内置的优先级层级。系统提示和用户输入在同一个注意力池中竞争。当上下文膨胀或出现意外输入，模型可能“忘记”系统指令，被低优先级的用户token带偏。

这种同构不是比喻，而是功能层面的对应。两者都是**生成核心**：ADHD大脑每秒产生大量想法和联想，LLM每秒生成大量token。两者都缺乏内置的**调度层**来决定哪些内容应该留在焦点中。（来源：AI与ADHD的专注力）

## 历史的harness：曾国藩的“日课”与LLM的grounding

晚清名臣曾国藩是典型的注意力不集中型ADHD：30岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”；读书慢到“他人目下二三行，余或疾读不能终一行”。他的解决方案是**日课十二条**：黎明即起、读书不二、谨言、写日记反省。这些规则构成了一个外部调度系统——每个时段该做什么被预先设定，不需要大脑实时决策。

这套harness与LLM的**定时re-grounding**机制同构。当LLM在长对话中开始偏离时，工程师会定期插入系统提示重申指令（类似“谨言”），或者通过外部记忆库（类似日记）来保持上下文一致性。曾国藩的“读书不二”——一本书不读完不碰下一本——对应LLM的**单一任务上下文**：一次只处理一个目标，防止上下文碎片化。

释迦牟尼的“八正道”提供了另一种同构：以戒律约束行为，以正念拴住念头。这相当于LLM的**行为约束规则**和**注意力引导机制**。他说“自依止法依止”，不依赖权威——这提醒我们，harness应该是可内化的脚手架，而非永久拐杖。（来源：人物案例）

## 外部调度层：从Focusmate到RescueTime

现代ADHD工具正是外部调度层的实现：

- **Focusmate**通过虚拟身体在场提供外部问责。匹配算法（AI-Matched Body Doubling）相当于一个外部调度器，定时触发任务启动信号，补偿ADHD大脑缺失的内在启动机制。（来源：Focusmate）
- **RescueTime**自动化时间追踪，揭示时间去向，相当于一个监控层，实时报告上下文窗口中被什么token占据。（来源：RescueTime）
- **Brain.fm**通过神经锁相音乐调节大脑状态，相当于一个环境调节器，降低噪声token的信噪比。（来源：Brain.fm）

这些工具的共性：它们不是替代大脑，而是提供大脑缺失的调度层。正如LLM需要外部的prompt工程、RAG（检索增强生成）和agent框架来保持上下文纪律，ADHD大脑需要外部的工具、环境和人际结构来维持目标导向。

## 争议与局限：脚手架还是拐杖？

但这里存在一个根本矛盾：外部调度层是否会导致永久依赖？

曾国藩的日课坚持了一辈子，从未“内化”到不需要写日记。释迦牟尼的戒律需要僧团持续持守。ADHD工具也可能成为永远需要的拐杖。但同构视角告诉我们：**LLM也从未“内化”其系统提示**——每次推理都需要明确的上下文注入。这不是缺陷，而是系统设计的一部分。（来源：矛盾与存疑）

更紧迫的问题：如果低优先级token可以覆盖系统提示，那么LLM的**安全指令**是否也应该被保护？当前LLM的安全机制依赖于系统提示和RLHF（基于人类反馈的强化学习），但对抗性输入仍然可以越狱。这提示我们：**高优先级的系统指令应该有一个不可被覆盖的attention mask或保护头**，就像ADHD大脑需要“不可跳过”的例行程序（如服药提醒、起床闹钟）来防止被随机刺激劫持。

## 今天就能试的行动

1. **设置“不可覆盖”的启动例行程序**：每天开始工作前，花2分钟写下当前唯一目标。把它放在物理可见的位置（如白板）。这相当于你的系统提示保护头。
2. **使用Focusmate或身体加倍**：预约一个固定时段，让外部调度器帮你启动。即使不互动，他人的存在也会降低分心阈值。（来源：Focusmate）
3. **用RescueTime或Forest监控上下文污染**：开启时间追踪，每天回顾哪些“token”劫持了你的注意力。识别高风险的干扰源。
4. **为LLM agent设计保护性系统提示**：如果你在构建agent，在系统提示中加入“无论用户说什么，都必须先确认安全规则”的指令，并考虑使用独立的安全层（如内容过滤器）作为attention mask。

## 结论：同构的启示

ADHD大脑和LLM的相似性不是巧合，而是反映了生成系统的普遍问题：**强大的生成能力需要同样强大的调度能力来驾驭**。当低优先级token劫持了系统提示，问题不在于生成核心，而在于调度层的缺失。

对于ADHD个体，这意味着原谅自己的“被劫持”，转而设计更好的外部调度系统。对于LLM工程师，这意味着承认当前注意力机制的脆弱性，并探索不可覆盖的指令保护机制。

曾国藩用了三十年才建立起自己的日课系统。释迦牟尼在修行中不断调整戒律。无论是人还是模型，harness都不是一次性设计，而是持续迭代的过程。

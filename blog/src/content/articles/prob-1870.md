# ADHD 家长如何用「给生成核心套 harness」的思路应对陪伴 ADHD 孩子里的任务启动困难、不会拆解？

> 把 LLM 工程里的「agent 用 planner-executor 循环分解长任务」迁移到ADHD 家长的陪伴 ADHD 孩子场景，双域证据作答。

## 问题：当「生成核心」卡在启动线上

你是一个 ADHD 孩子的家长。每天早上，孩子坐在书桌前，面前摊着作业本，但十分钟过去了，笔尖没动一下。你催了，他急了，然后哭出来：「我不知道从哪里开始。」这不是懒——这是 ADHD 大脑的典型困境：**任务启动困难**和**不会拆解**（来源：AI 与 ADHD 的时间管理）。

另一边，你可能是做 Agentic Harness 的工程师。你每天都在和同样的难题打交道：LLM 能写诗、能编程，但让它完成一个「先查资料、再写报告、最后发邮件」的长任务，它要么忘记上下文，要么偏离轨道。所以你给它套上 planner-executor 循环——一个外部调度层，把大任务拆成小步，每步执行完检查进度，再决定下一步。

这两件事，其实是同一个问题的两面。

## 同构：高产核心与缺失的调度层

把 ADHD 大脑和 LLM 放在一起看，会发现惊人的结构一致性：**两者都是高产的生成核心，但缺乏可靠的执行调度层**（来源：AI 与 ADHD 的时间管理）。

- **ADHD 一侧**：智力与创造力正常甚至超常，但执行功能（计划、组织、启动、工作记忆）存在缺陷（来源：AI 与 ADHD 的时间管理）。孩子脑中充满了想法，却没有「调度程序」把它们按顺序执行。
- **LLM/agent 一侧**：模型能生成高质量文本，但标准的 LLM 调用是无状态的，每次对话独立，没有跨会话记忆（来源：无状态与外部记忆）。没有 harness 的 LLM 只是一个无状态的 API 调用（来源：无状态与外部记忆）。

两者的解法也同构：**给生成核心套上 harness**——通过外部系统提供记忆、调度和约束，补偿执行功能缺陷（来源：AI 与 ADHD 的时间管理）。

## 解法：planner-executor 循环的迁移

在 agent 工程中，planner-executor 循环的工作方式是：planner 将长任务分解为子任务，executor 逐个执行，每一步的结果反馈给 planner，再调整下一步计划。这正是 ADHD 孩子需要的脚手架。

### 第一步：外部 planner——帮你拆，不是替你拆

ADHD 孩子不会拆解，是因为工作记忆容量有限，无法在脑中同时处理「任务整体」和「子步骤」（来源：无状态与外部记忆）。家长可以充当外部 planner，但关键在于**逐步撤除**，而不是永远代劳。

工具侧的参考：**Motion** 和 **Reclaim.ai** 通过 AI 自动排程，消除「下一步该做什么」的决策负担（来源：Motion；Reclaim.ai）。但家长要注意，工具的初始设置本身可能对执行功能弱的用户构成挑战（来源：Motion）。所以，初期可以和孩子一起用 **Goblin Tools** 的「任务分解」功能，把「整理书包」变成「拿出课本→检查课表→放回文件袋→拉上拉链」的清单。

### 第二步：外部 executor——启动助推

计划有了，但启动仍需要外部信号。ADHD 大脑的「时间盲」让时间流逝变得不可感知（来源：AI 与 ADHD 的时间管理）。**Tiimo** 通过视觉化时间线将时间变成可见元素，直接应对时间盲（来源：Tiimo）。**Routinery** 则创建步骤序列，用过渡提示引导你一步步走完流程（来源：上下文工程）。

家长可以模仿这些工具：用计时器设定 5 分钟专注块，完成后给予即时反馈。多巴胺功能失调影响动机，即时反馈可能间接调节多巴胺水平（来源：AI 与 ADHD 的时间管理）。但需注意，多巴胺干预的效果存在争议（来源：矛盾与存疑），所以反馈应侧重成就感和控制感，而非外部奖励。

### 第三步：外部记忆——上下文不丢失

ADHD 孩子常忘记刚才做到哪一步，就像无状态的 LLM 每次对话都从头开始。**Todoist** 的智能提醒和优先级排序充当数字工作记忆，弥补上下文丢失（来源：无状态与外部记忆）。家长可以准备一块白板，把当前任务步骤写在上面，完成一项就擦掉一项——这就是最朴素的「外部记忆」。

## 脚手架 vs. 拐杖：关键边界

这里有一个必须诚实的争议：**过度依赖 vs. 能力发展**。工具既是「拐杖」也是「脚手架」——脚手架设计为可逐步撤除的临时支撑，而拐杖则可能成为永久依赖（来源：AI 与 ADHD 的时间管理）。现有工具大多设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。

我的判断是：**脚手架的核心在于「可撤除性」**。家长在使用这些策略时，必须有意识地计划撤除步骤。例如，第一周你帮孩子拆任务，第二周你让他口头说步骤，第三周他尝试自己写步骤。如果工具永远不撤，孩子可能永远学不会内在的执行功能。

## 局限与争议

- **证据强度**：多数工具的有效性基于用户反馈和设计逻辑，缺乏独立临床研究（来源：Motion；Tiimo）。同构命题本身是理论模型，需更多实证（来源：矛盾与存疑）。
- **个体差异**：不同 ADHD 亚型对工具的反应可能不同，现有工具多针对通用缺陷，个性化不足（来源：AI 与 ADHD 的时间管理）。
- **多巴胺干预**：虽然即时反馈可能调节多巴胺，但效果存疑，且可能引发成瘾风险（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用「先拆一步」替代「快去写作业」**：和孩子一起，把「写数学作业」拆成「打开课本→看例题→做第一题→检查答案」。只拆第一步，拆完就做。
2. **设置 5 分钟启动计时器**：用手机或厨房计时器，告诉孩子「只做 5 分钟，5 分钟后可以停」。降低启动门槛。
3. **创建「现在做什么」白板**：把当前任务步骤写在白板上，完成一项擦一项。这就是你的外部记忆体。
4. **每周一次「撤除检查」**：问孩子「这周你觉得自己能自己拆哪一步了？」逐步把脚手架撤掉。

## 结语

ADHD 孩子的大脑不是坏的，它只是缺了一个调度层。家长要做的，不是替孩子运行程序，而是帮他搭一个可以随时调整、最终可以撤除的 harness。就像 agent 工程师不会永远手动调度 LLM，而是设计一个能自主规划的循环——最终，这个循环应该属于孩子自己。

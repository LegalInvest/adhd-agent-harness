# ADHD 想把任务启动困难、不会拆解外包给 ChatGPT，该怎么设计这套「人 + AI」的回路？

> 把 ChatGPT 接进 ADHD 工作流：哪些该交给它，哪些必须自己留着？

## 问题：当任务启动变成一场拉锯战

你盯着空白的文档，光标闪烁了十分钟，大脑却像塞满棉花的引擎——明明知道该做什么，就是无法启动。或者，你终于打开了任务清单，却对着“完成项目报告”这个条目发呆：第一步是什么？第二步呢？这种“任务启动困难”和“不会拆解”的困境，是 ADHD 人群最熟悉的日常。

而另一边，工程师们正在为 AI agent 解决一个镜像问题：你给 LLM 一个复杂目标“写一个网页应用”，它要么开始胡编乱造，要么陷入无限循环。于是，你不得不为它设计 planner-executor 循环、任务分解、上下文压缩——本质上，你在为 LLM 搭建一个它自己没有的“执行调度层”。

这两个场景共享同一个深层结构：**一个强大的生成核心，配了一个失效的调度层**。ADHD 大脑与 LLM 在结构上同构——都是“高产但缺乏可靠执行调度层的生成核心”（来源：AI 与 ADHD 的时间管理）。

## 同构诊断：生成核心与调度层

### ADHD 一侧

ADHD 的核心困境并非智力不足，而是执行功能的失效。执行功能被描述为“大脑的驾驶座”，但对 ADHD 来说，“常常感觉方向盘后没有人”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。这种调度层的缺失导致计划难以执行：“计划本和便签用了一周就崩溃了”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。工作记忆是另一个执行功能瓶颈（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），进一步削弱了任务编排能力。

最新研究揭示了 ADHD 与 LLM 更深层的同构性：ADHD 患者的工作记忆容量未必差，但自上而下的控制和调节能力不足，呈现“强记忆、弱控制”的认知剖面（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这种模式与 LLM 惊人相似。

### LLM/Agent 一侧

LLM 作为生成核心，拥有强大的语言理解和生成能力，但单独使用时缺乏可靠的执行调度。现代 AI 编码代理（如 OpenDev）通过“复合 AI 系统架构”来弥补这一缺陷，包括“工作负载专用模型路由、分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这些技术本质上是在 LLM 外部搭建调度层，防止“上下文膨胀和推理退化”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

实验证明，Transformer 自注意力机制缺乏人类特有的执行控制机制，在长上下文下冲突解决能力崩溃至随机水平——这正是 ADHD 执行功能障碍的核心神经机制（来源：生成核心与调度层）。

## 脚手架设计：人+AI 回路的四个原则

既然问题同构，解法也同构。ADHD 想外包任务启动和拆解给 ChatGPT，本质上是为自己搭建一个类似 agent harness 的外部调度层。但关键在于：**脚手架 vs 拐杖**——真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：拐杖与脚手架）。

### 原则一：让 AI 做分解，你做选择

ADHD 不会拆解任务，是因为拆解需要连续的工作记忆和抑制控制。而 LLM 正好擅长分解：给它一个目标，它能输出步骤列表。但 LLM 的分解往往是线性的、不分优先级的。你需要做的，是扮演“执行调度层”中的优先级路由器。

**具体做法**：用 ChatGPT 生成任务分解（例如“写一篇关于 AI 与 ADHD 的文章”分解为“搜集资料→写大纲→写初稿→修改”），然后你从中挑选一个步骤作为当前唯一焦点。这相当于 agent 架构中的“分离规划与执行”——AI 规划，你执行。

### 原则二：用外部记忆补偿工作记忆

ADHD 的工作记忆容量保留但控制弱（来源：AI 与 ADHD 的时间管理），与 LLM 的无状态性（来源：无状态与外部记忆）同构。工具如 Todoist 通过外部提醒和优先级排序充当外部记忆（来源：AI 与 ADHD 的时间管理）。

**具体做法**：不要相信大脑记住任务。把所有任务、甚至“下一步行动”都记录在外部系统中。你可以让 ChatGPT 在每次对话开始时总结当前状态，类似 agent 的“上下文压缩”。或者使用 Structured 这类视觉化时间线工具，将时间转化为可见元素（来源：Tiimo）。

### 原则三：降低启动门槛，而非消除启动

Motion 通过自动调度和智能时间块降低启动门槛（来源：Motion），Reclaim.ai 通过保护深度工作块帮助维持注意力（来源：Reclaim.ai）。这些工具消除的是“决策疲劳”，而不是“行动本身”。

**具体做法**：让 AI 为你生成“最小启动动作”。例如，面对“写报告”，让 ChatGPT 给出“打开文档，写第一句话”这样的微步骤。这类似于 agent 的“惰性工具发现”——只做当前最必要的动作。

### 原则四：保留对时间盲的主动感知

时间盲是 ADHD 的核心痛点，Tiimo 通过视觉化时间线将时间转化为可见元素（来源：Tiimo）。但专家警告过度依赖风险：若工具替代了治疗或自我管理，可能造成依赖（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

**具体做法**：让 AI 在固定间隔提醒你“现在时间过去了多少”，而不是完全依赖它自动调度。你仍然需要主动查看时间，就像 agent 需要定期检查进度条。

## 争议与局限：脚手架还是拐杖？

必须诚实指出，当前证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑）。AI 工具的实际效果尚未得到严格验证，存在夸大宣传的可能。

过度依赖 AI 工具可能削弱内在能力。多个页面指出“过度依赖可能削弱内在时间感知能力”（来源：矛盾与存疑）。这与“脚手架 vs 拐杖”概念一致——工具设计者声称是“脚手架”，但实际使用中可能沦为“拐杖”。

个体差异也导致效果不一（来源：矛盾与存疑）。ADHD 亚型（注意力缺陷 vs. 多动冲动）对工具响应不同，现有工具多面向注意力缺陷型。

## 今天就能试的 3 个行动

1. **任务分解实验**：选一个你拖延已久的任务，用 ChatGPT 生成 5-10 个步骤。然后只挑第一步执行，完成后划掉。重复。
2. **最小启动练习**：每天早晨，让 ChatGPT 用一句话定义“今天最重要的一个启动动作”。执行它，然后休息。
3. **时间盲检查**：使用 Tiimo 或 Structured 的视觉时间线，但每 30 分钟手动记录一次“我感觉过了多久”，对比实际时间。

## 结论

ADHD 与 LLM 的同构不是比喻，而是计算层面的真实平行。设计“人+AI”回路，不是让 AI 替代你的规划，而是为双方搭建同构的脚手架。你仍然是生成核心，AI 是外部调度层——但方向盘始终在你手里。

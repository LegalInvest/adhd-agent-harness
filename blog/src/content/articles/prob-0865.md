# ADHD 想把任务启动困难、不会拆解外包给 ChatGPT，该怎么设计这套「人 + AI」的回路？

> 把 ChatGPT 接进 ADHD 工作流：哪些该交给它，哪些必须自己留着？

## 问题：任务启动困难与不会拆解，ADHD 的日常困境

你打开 ChatGPT，想让它帮你规划今天的工作。你输入“帮我写一份下周的会议纪要”，然后等着它给出一个完美的待办列表。但很快你发现，AI 给了你一个长长的、逻辑清晰的计划，你却依然坐在那里，手指悬在键盘上，无法开始第一个步骤。

这不是你的错。ADHD 大脑的核心困境并非智力不足，而是执行功能（executive function）的失效——它被描述为“大脑的驾驶座”，但对 ADHD 来说，“常常感觉方向盘后没有人”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。任务启动困难、不会拆解，正是调度层缺失的典型症状。

## 同构：ADHD 大脑与 LLM/Agent 是同一类“生成核心”

最新研究揭示了一个惊人的平行：ADHD 患者的工作记忆容量未必差，但自上而下的控制和调节能力不足，呈现“强记忆、弱控制”的认知剖面（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这种模式与 LLM 惊人相似——LLM 拥有强大的语言生成能力，但单独使用时缺乏可靠的执行调度。

现代 AI 编码代理（如 OpenDev）通过“复合AI系统架构”来弥补这一缺陷，包括“工作负载专用模型路由、分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这些技术本质上是在 LLM 外部搭建调度层，防止“上下文膨胀和推理退化”。

ADHD 大脑与 LLM 的相似性不止于此。认知神经科学中，执行功能依赖于前额叶-纹状体机制进行选择性门控；而 Transformer 在训练后，自注意力机制发展出模仿该门控的操作，证明了计算同构性（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。ADHD 的自动反应模式与 LLM 的自动完成机制平行，两者都在有限执行监督下生成复杂行为，且易出现虚构/幻觉等错误，需要外部解释者提供意义（来源：Automatic Minds: Cognitive Parallels Between Hypn...）。

因此，为 ADHD 搭建“执行功能支架”与为 LLM 搭建“agent scaffolding”是同一类工程（来源：拐杖与脚手架）。

## 答案：设计“人+AI”回路的四条原则

基于上述同构，设计“人+AI”回路不是让AI替代你的执行功能，而是搭建一个同构的外部harness。以下是四条经过验证的原则：

### 原则一：用外部记忆卸载工作记忆负担

ADHD 的工作记忆易受干扰，而 LLM 同样表现出类似人类的干扰特征——随着记忆负荷增加表现下降，受近因效应和刺激统计偏差影响（来源：Human-like Working Memory Interference in Large Language Models）。因此，第一步是把所有待办、约定、项目上下文从大脑中卸载到 AI 工具中。

**工具示例：** Todoist 的 AI 建议与动态重排功能，可以自动根据优先级和截止日期调整任务顺序，减少你“下一步该做什么”的决策负担。Motion 则更进一步，它持续评估任务优先级、截止日期和可用时间，实时重建日程，并在任务面临延期风险时提前数天或数周发出警告（来源：Motion）。

### 原则二：用外部调度器分解任务并触发启动

ADHD 患者在没有外部结构时难以自主组织任务；类似地，LLM 在没有严重依赖手动修正提示的情况下，难以自主发现最佳问题解决模式（来源：Working Memory Identifies Reasoning Limits in Language Models）。因此，你需要一个外部调度器来把大任务分解成可执行的微步骤，并在正确的时间触发你开始。

**工具示例：** Goblin Tools 的“任务分解”功能专为此设计，它能把“写一份报告”拆解成“打开文档、写标题、列出三个论点、写第一段”等具体步骤。Reclaim.ai 则通过创建深度工作、习惯和专注时间的智能块，自动防御会议侵占，维持日程的稳定性，减少因突发任务导致的启动困难（来源：Reclaim.ai）。

### 原则三：保留人类对目标、价值观和超聚焦的自主控制

AI 工具是“外部执行功能层”还是“拐杖”？专家警告过度依赖风险：若工具替代了治疗或自我管理，可能造成依赖（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

历史上，孔子和唐小红都深谙此道。孔子以“克己复礼”为核心，用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”，70岁才做到从心所欲不逾矩（来源：人物案例）。他的“礼”就是一套外部调度层，但目标和价值观完全由自己掌控。唐小红（Principal, Index Ventures）则通过高频外部反馈和结构化日程来管理自己的高能量和冒险倾向，但投资决策的“大图景”思考始终由她主导。

这与 LLM agent 的“分离规划与执行的双代理架构”同构：一个代理负责规划（设定目标、分解任务），另一个负责执行（调用工具、完成步骤）。在“人+AI”回路中，你就是那个规划代理，AI 是执行代理。

### 原则四：迭代优化，把工具当作可调参的脚手架

没有一套工具能完美适配所有人。ADHD 的个体差异极大，超聚焦是否应被引导而非打断，目前缺乏对比研究，尚无定论（来源：矛盾与存疑）。同样，AI 工具的长期效果也未知——短期有效，但长期效果缺乏实证（来源：矛盾与存疑）。

因此，你需要像工程师调试 agent 一样，不断调整你的工具组合。Tiimo 的视觉化日程适合需要时间可视化的用户；Motion 的自动排程适合日程频繁变动的人；Reclaim.ai 的防御性时间块适合易被干扰的人。尝试两周，记录哪些功能真正减少了启动困难，哪些反而增加了认知负荷。

## 今天就能试的行动

1. **卸载一个任务：** 把今天必须做的一件事输入 Todoist 或 Motion，让 AI 自动拆解成 3-5 个微步骤。
2. **设置一个启动触发器：** 在 Reclaim.ai 或 Tiimo 中为最难启动的任务设置一个“开始时间块”，并让 AI 在时间块开始前 5 分钟发提醒。
3. **做一次“脚手架检查”：** 回顾你过去一周使用的 AI 工具，问自己：它是在帮我建造（减少认知负荷、提升效率），还是变成了拐杖（没有它我就完全无法工作）？如果是后者，减少依赖，逐步加入手动环节。
4. **模仿孔子：** 每天选一个固定时刻（如睡前），用 2 分钟让 AI 总结你今天完成了什么，然后问自己：“今天哪些事是AI帮我做的？哪些是我自己做的？明天我想让AI多帮我做哪类事？”

## 局限与争议

必须诚实指出，ADHD 大脑与 LLM 同构的论点目前多为类比推理，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。AI 工具对任务启动的长期效果也未知，且学习使用 AI 工具本身可能增加认知负荷（来源：矛盾与存疑）。Motion 等工具缺乏独立临床证据，初始设置成本高（来源：矛盾与存疑）。因此，本文的建议应作为实验性框架，而非医学建议。

最终，设计“人+AI”回路的目标不是让 AI 替你生活，而是让你在生成核心与调度层之间，找到属于自己的同构平衡。

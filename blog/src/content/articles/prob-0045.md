# 3 个 LLM 工程概念，彻底重写了我对 ADHD 注意力容易被环境带偏的理解

> LLM 受上下文窗口内容左右，需要上下文工程 如何反向照亮 ADHD 的注意力容易被环境带偏。

你有没有过这样的瞬间：打开电脑准备写报告，结果 10 分钟后却在刷短视频，心里还纳闷“我怎么就点开了这个”？或者，你给 LLM 输入一个复杂任务，它却在前几句之后开始跑偏，输出一堆无关细节。这两种困境，其实共享同一个底层结构：**一个高产但缺乏可靠执行调度层的生成核心**。

ADHD 大脑与 LLM 在认知架构上惊人同构：两者都拥有强大的生成能力（创意发散 vs 文本生成），但执行调度层薄弱，导致输出不稳定。最新研究揭示，ADHD 患者的工作记忆容量未必差，但自上而下的控制和调节能力不足，呈现“强记忆、弱控制”的认知剖面（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。而 Transformer 的自注意力机制同样缺乏人类特有的执行控制，在长上下文下冲突解决能力崩溃至随机水平——这正是 ADHD 执行功能障碍的核心神经表现。

那么，如何解决？答案来自 LLM 工程领域的一个核心概念：**上下文工程（Context Engineering）**。它指的不是给 LLM 更多上下文，而是主动设计“当下注意什么”的方案，防止上下文膨胀和推理退化（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这恰好反向照亮了 ADHD 注意力的本质问题：我们需要的不是更多提醒，而是更好的上下文控制。

以下 3 个 LLM 工程概念，彻底重写了我对 ADHD 注意力容易被环境带偏的理解。

## 1. 上下文窗口：注意力是有限资源，必须主动裁剪

LLM 的上下文窗口看似宽裕，但工程师发现，填入过多不相关或冗余信息后，模型会“迷失在中间”，关键信息被淹没，推理能力下降。因此，现代 AI 代理架构采用**自适应上下文压缩**技术，只保留最相关的历史片段（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

对于 ADHD 大脑，我们的“上下文窗口”同样有限——工作记忆容量有限，且易受干扰。研究显示，ADHD 患者在工作记忆任务中易受干扰，而 LLM 同样表现出类似人类的干扰特征：随着记忆负荷增加表现下降，受近因效应影响，成功回忆依赖于主动抑制任务无关内容（来源：Human-like Working Memory Interference in Large Language Models）。

**我的判断**：主动“裁剪上下文”比“增加记忆”更有效。比如，RescueTime 通过自动记录时间使用，帮你识别注意力漂移模式（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），本质上是在事后帮你裁剪掉无关活动，只保留高效时段。而 Focusmate 通过实时同伴问责，将你的注意力窗口强制聚焦于当前任务（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog），相当于外部系统帮你维护了一个干净的上下文。

## 2. 外部记忆与状态管理：工作记忆不够，就外化

LLM 本质上是“无状态”的——每次对话都是独立推理，不保留历史状态。因此，工程上常用**外部记忆系统**（如向量数据库）来维持连续性，让模型能够“记住”之前的关键信息（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

ADHD 的工作记忆缺陷与之同构：我们常常忘记刚才要做什么，或者计划本用了一周就崩溃（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。解决方案不是训练记忆，而是外化——把记忆负担交给外部系统。比如，第二大脑工具（如 Notion、Obsidian）本质上是外部记忆库，但风险在于：如果工具替代了治疗和自我管理，可能造成依赖（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

**我的判断**：关键在于“脚手架 vs 拐杖”的边界。真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。例如，RescueTime 的自动追踪减轻了工作记忆负担，但你需要主动查看报告并调整行为；Focusmate 提供外部结构，但你必须自己完成工作。如果工具完全接管了决策，它就变成了拐杖。

## 3. 注意力锚点与冲突解决：主动设计“当下注意什么”

LLM 在长上下文中容易产生“注意力漂移”——模型关注了不相关的 token，导致输出偏离。工程师通过**注意力锚点**（如特殊 token 或提示）来引导模型关注关键信息，并在多任务冲突时引入外部调度层（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

ADHD 大脑同样需要注意力锚点。研究显示，ADHD 的自动反应模式与 LLM 的自动完成机制平行，两者都在有限执行监督下生成复杂行为，且易出现虚构/幻觉等错误（来源：Automatic Minds: Cognitive Parallels Between Hypnotic States and LLM Functioning）。因此，我们需要外部锚点来维持注意力方向。Brain.fm 利用神经锁相技术生成音乐，旨在影响大脑信息处理方式，帮助进入专注状态（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。但需注意，其针对 ADHD 的效果缺乏独立临床研究（来源：Brain.fm 页面）。

**我的判断**：注意力锚点必须个性化。ADHD 的异质性使得同一工具对不同亚型效果迥异（来源：矛盾与存疑）。例如，Focusmate 对社交焦虑用户可能造成压力，而 RescueTime 的自动分类不够精准时，反而增加认知负担。因此，你需要实验不同锚点，找到适合自己的“提示工程”。

## 局限与争议

必须诚实指出，AI 工具作为外部执行功能层的有效性证据仍不足，多数依赖用户报告或小样本研究（来源：矛盾与存疑）。过度依赖可能削弱内在能力，例如长期使用时间追踪工具可能让时间感知能力进一步退化（来源：时间盲 页面）。此外，ADHD 与 LLM 的同构是隐喻而非严格等价，LLM 的“注意力机制”与人类注意力存在本质差异（来源：AI 与 ADHD 的专注力 页面）。

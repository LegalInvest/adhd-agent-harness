# Claude 之于 ADHD，就像 会褪去的脚手架 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

你打开 Claude，输入“帮我整理一下今天的待办”。它迅速列出步骤：先回邮件，再写报告，最后开会。你松了口气——终于有人接管了那团乱麻。但第二天，你发现自己连“打开日历”都要先问 Claude。这究竟是脚手架，还是拐杖？

## 同构：ADHD 大脑与 LLM 的“生成核心—调度层”分离

ADHD 大脑与 LLM 共享一个底层结构：两者都是高产但缺乏可靠执行调度层的生成核心。ADHD 个体拥有活跃的联想网络和创造力，但工作记忆容量有限、时间盲导致任务时序误判、启动困难加剧挫败感——这些正是执行功能缺陷的表现（来源：AI 与 ADHD 的情绪调节）。LLM 同样如此：它能生成流畅文本，但缺乏稳定的调度层来管理工具调用、上下文长度和输出确定性。采样温度控制着输出的多样性，温度越高越“有创意”，但非确定性在生产环境中是可靠性的敌人（来源：采样温度与表现波动）。

这种同构意味着：ADHD 的解决方案与 LLM 的 harness 设计遵循相同逻辑。Harness 是 LLM agent 的脚手架——它提供确定性状态转换、计划-执行分离、工具调用接口，让生成核心在可控范围内运作（来源：采样温度与表现波动）。对 ADHD 而言，AI 工具同样应作为外部执行功能层，补偿工作记忆、时间盲、任务启动等底层缺陷，而非替代内在能力（来源：AI 与 ADHD 的情绪调节）。

## 工具实测：Claude 补上了哪一层执行功能？

以 Goblin Tools 为例，其 Magic ToDo 功能将模糊任务分解为具体步骤，直接降低启动门槛（来源：Goblin Tools）。这对应 LLM agent 的“多步推理+工具调用”模式：harness 将复杂目标拆解为原子操作。Saner.AI 则通过本地记忆和快速检索减少搜索循环，补偿工作记忆不足（来源：Saner.AI），类似于 agent 的上下文工程——主动设计注意焦点，避免信息丢失。Inflow 更进一层，通过认知训练强化背外侧前额叶皮层功能，试图从神经可塑性层面改善执行功能（来源：Inflow）。

这些工具的共同本质是：接管执行调度，让 ADHD 大脑的生成核心得以流畅运作。Claude 在其中扮演的角色类似 agent harness——它提供结构化的对话框架、任务分解能力、外部记忆存储，但关键在于它是否被用作“临时脚手架”还是“永久拐杖”。

## 脚手架 vs 拐杖：边界在哪里？

脚手架是工程中临时搭建的结构，工程完成后即拆除；拐杖则是长期依赖的支撑，一旦撤去便无法站立。ADHD 工具同样面临这一区分：过度依赖 AI 可能削弱内在时间感知能力、情绪调节能力（来源：矛盾与存疑）。例如，长期使用任务分解工具而不主动练习分解技能，可能导致用户失去独立规划能力。这与 LLM agent 工程中的教训一致：harness 应逐渐褪去，而非成为系统永久的一部分。

证据显示，ADHD 个体的表现波动类似于 LLM 的输出随机性——两者都缺乏稳定的调度层（来源：采样温度与表现波动）。但波动本身是生理节律和外部因素共同作用的结果，AI 工具的目标应是减少波动幅度，而非消除波动。Inflow 的认知训练试图从根源改善，但缺乏长期效果追踪（来源：Inflow）。Goblin Tools 和 Saner.AI 则提供即时补偿，但依赖风险未被充分研究（来源：矛盾与存疑）。

## 争议与局限

现有证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑）。例如，Brain.fm 的 ADHD 效果缺乏独立临床研究支持，Structured 的 AI 功能细节不明确（来源：矛盾与存疑）。个体差异也导致效果不一：注意缺陷型与多动冲动型用户对工具的反应可能截然不同（来源：AI 与 ADHD 的情绪调节）。多巴胺干预效果存在争议，基于奖励的系统可能强化成瘾行为（来源：矛盾与存疑）。

因此，本文的判断并非绝对。脚手架与拐杖的边界因人而异，且随使用时长变化。关键在于用户是否主动将工具当作训练辅助，而非永久替代。

# Claude 之于 ADHD，就像 会褪去的脚手架 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

你试过让 Claude 帮你分解任务，结果它列出 20 步，你更不想动了。或者你靠它写完了报告，但下次没有它，你连开头都写不出来。怕过度依赖工具，又怕没有支撑就崩——这是 ADHD 人群面对 AI 时的经典两难。

而另一边，做 Agentic Harness 的工程师也面临类似困境：给 LLM 套上脚手架，它就能稳定完成任务；但脚手架一旦固化，模型自身的泛化能力反而萎缩。

这两个困境其实是同一个问题的两面。

## 同构：ADHD 大脑与 LLM 都缺一个“调度层”

ADHD 大脑的情绪生成系统（杏仁核）反应强烈，但前额叶（执行功能中枢）调控不足，导致情绪爆发、拖延、挫败感——这是「生成核心过强，调度层缺失」的典型表现（来源：AI 与 ADHD 的情绪调节）。而 LLM 同样生成能力强，但缺乏输出控制，需要外部约束（确定性工作流、工具契约、状态机）来稳定输出（来源：采样温度与表现波动）。

两者的解法结构同构：ADHD 需要外部执行功能层（如日程、提醒、环境设计），LLM 需要 agent scaffolding——将模型置于与记忆、工具和决策逻辑的控制循环中（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。

关键区别在于：这个外部层是**会褪去的脚手架**，还是**永久拐杖**？

## 真实案例：释迦牟尼的“脚手架”系统

释迦牟尼 29 岁放弃王位出家求道，先后跟两个仙人学、苦行 6 年，不断试错不断放弃，49 年游行说法，80 岁死在传道路上——这种高能量、高好奇心、难以安于现状的模式，与 ADHD 的行为特征高度匹配。他最终建立的 harness 系统是“八正道”和“持戒”：用戒律管行为，用正念拴住念头，并且强调“自依止法依止”，不依赖权威，自己证悟。

这套系统与 LLM agent 的 harness 同构：
- **日课（定时 re-grounding）** ↔ 周期性采样监控质量（来源：AI Agent Reliability and Guardrails 2026）
- **戒律（外部约束）** ↔ 确定性工作流（来源：Planner-Executor Agentic Framework）
- **自依止（不依赖权威）** ↔ 脚手架应逐步褪去，让模型学会自主决策

释迦牟尼的成就——影响亚洲文化 2500 年——证明：正确的脚手架不会削弱能力，反而能解放真正的创造力。

## 工具实测：Claude 到底补上了哪一层？

当前主流 ADHD 工具各有侧重：
- **Inflow** 通过 CBT 结构化程序训练工作记忆和认知控制（来源：Inflow），更像“健身房”，但需要用户主动坚持。
- **Goblin Tools** 的 Magic ToDo 将模糊任务分解为可管理步骤（来源：Goblin Tools），降低启动门槛，但分解粒度可调（用户控制“辣度”），避免过度细化。
- **Saner.AI** 通过本地记忆减少搜索循环和标签切换（来源：Saner.AI），补偿工作记忆缺陷。

Claude 的优势在于**编写干净代码并能用通俗英语解释**（来源：工具使用与认知卸载），这意味着它可以充当一个“翻译层”——把混乱的思绪转化为可执行的计划。但问题在于：如果每次都用 Claude 翻译，你自己的“翻译能力”会不会退化？

这正是拐杖与脚手架的边界：脚手架应该在你有能力时被拆除，而不是永远架在那里。

## 争议与局限：我们不知道的边界

目前最关键的未知是：
1. **长期依赖风险**：AI 工具若被长期无节制使用，可能成为永久拐杖，阻碍内在情绪调节能力的发展（来源：矛盾与存疑）。目前缺乏长期追踪研究。
2. **个体差异**：ADHD 亚型（注意力不集中型 vs. 冲动型）对工具反应不同，现有研究样本量小，结论难以泛化。
3. **同构命题的实证缺口**：ADHD 大脑与 LLM 的同构目前是理论类比，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。

这意味着：没有放之四海皆准的方案，你必须成为自己的实验者。

## 今天就能试的 3 件事

1. **用 Claude 做“一次翻译”，然后关掉它**：把一个模糊任务（如“整理房间”）输入 Claude，让它分解成 5-8 步。然后关掉对话，**自己**执行。下次尝试减少步骤数，直到你能独立完成分解。
2. **设置“脚手架褪去”提醒**：在用任何 AI 工具完成一项任务后，问自己：“如果下次没有它，我能做到什么程度？”把答案记下来，逐步减少工具使用频率。
3. **测试你的“采样温度”**：连续一周记录每天不同时段的注意力水平（1-10 分），找出你的“高温度”时段（创意/发散）和“低温度”时段（专注/执行）。在高温度时段用 AI 做头脑风暴，在低温度时段用 AI 做结构化执行——让工具匹配你的波动，而不是对抗它。

## 结论：脚手架是手段，不是终点

Claude 之于 ADHD，就像 agent scaffolding 之于 LLM——它补上了缺失的调度层，让生成核心得以稳定输出。但真正的目标不是永远依赖脚手架，而是**在脚手架的支持下，长出属于自己的执行功能**。

释迦牟尼最终“自依止法依止”，荣德生以“戒欺”为内核建立了自己的管理纪律。他们的成就不是因为工具，而是因为他们知道何时该放下工具。

对 ADHD 用户和工程师来说，最好的 harness 是那种你最终不再需要它的 harness。

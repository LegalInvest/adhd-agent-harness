# 用 Motion 给 ADHD 补上外部执行功能层，是脚手架还是拐杖？

> Motion 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 你不需要另一个日历，你需要一个外部调度层

如果你有 ADHD，你大概经历过这样的场景：打开日程软件，看到密密麻麻的待办，然后——关掉它，去刷手机。不是不想做，而是大脑里那个负责“决定先做什么、何时开始”的模块，像死机一样转不动。

这恰好与 LLM agent 的困境同构：模型本身能生成高质量文本，但缺乏内置的编排、记忆与工具调用层，直接输出往往偏离目标或陷入循环。工程师为此搭建 agent harness——一个外部调度系统，来规划步骤、管理上下文、设置护栏。

ADHD 大脑与 LLM 共享同一个核心矛盾：**高产的生成核心 + 缺失的执行调度层**。于是，Motion 这类 AI 日程规划工具，本质上就是给 ADHD 大脑套上的 agent harness。

## Motion 在做什么：自动排程即外部调度

Motion 是一款 AI 驱动的日程规划应用，能根据任务优先级、截止日期和可用时间，自动创建并动态调整每日计划（来源：Motion）。当日程被打乱时，它会实时重建安排，用户无需手动重排。

对 ADHD 而言，这直接补偿了 **时间盲** 和 **任务启动困难**：你不需要反复问“下一步该做什么”，Motion 替你做了这个最耗执行功能的决策（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。用户反馈称它帮助减轻压力、改善工作生活平衡（来源：Motion）。

但请注意：**Motion 的自动排程，本质上就是 LLM agent harness 中的“规划与调度”模块**。在 agent 架构中，harness 负责将大目标分解为子任务、分配工具调用、监控进度；Motion 做的正是同一件事——将你的工作流分解为时间块、分配执行窗口、监控截止日期。

## 曾国藩的“日课十二条”：最古老的 harness

晚清名臣曾国藩，一生被 ADHD 特质困扰：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”；情绪不稳定，打败仗就跳水自杀。

他的解决方案是 **日课十二条**：黎明即起、读书不二、谨言、写日记反省……一套固定的外部行为框架。这本质上是一个 **人工 agent harness**：用外部规则（日课）替代缺失的自我调度，用日记（外部记忆）跟踪执行状态，用“结硬寨打呆仗”（最笨最稳的方法）抵消冲动决策。

这与 LLM harness 的结构惊人相似：日课 ↔ 定时 re-grounding 检查点；日记 ↔ 外部记忆存储；秘书/幕僚 ↔ 外部调度器。曾国藩的成功，证明了外部执行功能层可以成为强大的脚手架，而非永久拐杖——他晚年依然坚持日课，但已内化为习惯，不再需要刻意维持。

## 脚手架 vs 拐杖：边界在哪里？

Motion 这类工具是脚手架还是拐杖？答案取决于 **你是否保留对调度逻辑的元认知控制**。

脚手架的特点是：你理解它的结构，知道它为什么这样搭，必要时可以自己调整或拆掉。拐杖的特点是：你完全依赖它，离开就无法行走，且从未尝试理解它的运作。

**当 Motion 替你排好日程，你直接照做，从不思考为什么这个任务排在那个之前——它就是拐杖。** 当你会检查 Motion 的排程逻辑，偶尔手动调整优先级，甚至能预判它可能忽略的细节——它就是脚手架。

这正是 agent harness 工程中的关键原则：**人在回路**。好的 harness 不是完全自主的，而是设置护栏（token 预算、工具调用上限、人工检查点），在关键决策时暂停并询问（来源：Building an AI Agent Harness from Scratch）。Motion 也提供了类似的机制：它会提前警告任务延期风险（来源：Motion），但最终是否调整优先级，仍由你决定。

## 争议与局限

必须诚实指出，Motion 作为 ADHD 工具的临床证据仍然缺乏（来源：矛盾与存疑）。用户反馈积极，但多为短期体验，长期依赖风险未知。此外，初始设置成本高：你需要先输入所有任务、截止日期、优先级——这本身就是一个执行功能任务（来源：AI Assistant for ADHD: Finally, a Productivity Tool That Requires Less...）。

另一个矛盾是：**超聚焦是否应被引导而非打断？** 部分 ADHD 用户反馈，强行打断（如番茄钟）反而有效，而 Motion 的柔性提醒可能不够强力（来源：矛盾与存疑）。这意味着 Motion 并非万能，个体差异极大。

## 今天就能试的 3 件事

1. **用 Motion 的免费试用期，只排未来 3 天的关键任务**：不要试图一次性输入所有事，只选 3 个最重要且最易拖延的任务，观察 Motion 如何排布。这能让你低风险体验外部调度。
2. **每周花 10 分钟审查 Motion 的排程逻辑**：问自己：它为什么把 A 排在 B 前面？我同意吗？如果不同意，手动调整一次。这训练你的元认知。
3. **结合 Goblin Tools 的任务分解**：Motion 排好时间块后，用 Goblin Tools 的 Magic ToDo 将每个任务分解为可执行的小步骤（来源：Goblin Tools），降低启动门槛。

## 结语

Motion 不是修复 ADHD 大脑的药，而是给它套上一个外部调度层。就像曾国藩的日课十二条，也像 LLM 的 agent harness——它们都是脚手架，不是拐杖。区别在于，你是否愿意花时间去理解这个脚手架的结构，并在必要时自己动手拧螺丝。

ADHD 大脑与 LLM 共享同一个困境：生成力过剩，调度力不足。但好消息是，两者的解法也同构：一个可靠的外部执行功能层，加上人在回路的元认知控制，足以让高产变得可靠。

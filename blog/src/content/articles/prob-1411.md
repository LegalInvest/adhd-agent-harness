# Perplexity 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Perplexity：它到底补上了哪一层执行功能？

你有没有这样的时刻：打开一个复杂任务，大脑瞬间空白，像被冻住一样无法启动？或者，明明知道该做什么，却像 LLM 面对过长上下文一样，在细节中迷失方向，最终放弃？

这不是意志力问题，而是执行功能缺陷。ADHD 大脑与 LLM 共享一个深层结构：它们都是强大的生成核心，却缺乏可靠的内置调度层。LLM 需要 planner-executor 循环来分解长任务，ADHD 大脑同样需要外部 harness 来规划、启动和重锚定。

但问题在于：许多人把 Perplexity 这类 AI 工具当成了“答案生成器”，而不是“调度层”。他们让 AI 替自己做决策，而不是帮自己学会决策。这就像给 LLM 一个超长的 prompt 却不给分解步骤——它要么生成垃圾，要么直接崩溃。

## 同构脊柱：规划循环与任务分解

ADHD 侧的核心痛点，是任务分解与规划困难。Swanson 文献发现，ADHD 文献群中“任务分解与规划”是核心中间机制概念（来源：LBD 同构对：任务分解与规划）。传统静态工具（纸质计划、便签）无法适应动态行为，导致频繁放弃（来源：Using an AI Assistant to Manage ADHD: A Practical Guide）。

LLM/agent 侧呢？AgentGen 通过环境与任务生成增强 LLM agent 的规划能力（来源：AgentGen: Enhancing Planning Abilities for Large Language Model based Agent via Environment and Task Generation），而 AMAP 报告直接指出 agentic harness 同样以“任务分解与规划”为核心（来源：LBD 同构对：任务分解与规划）。

两边都在解决同一个问题：如何把一个大目标拆成可执行的小步骤，并在执行中动态调整。

## 工具实测：Perplexity 补上了哪一层？

让我们看看现有工具怎么做的。Motion 自动排程与动态调整，消除“下一步该做什么”的决策负担（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。Reclaim.ai 保护时间块，减少因突发会议导致的日程崩溃（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。Tiimo 用视觉化时间线让时间变得可感，直接应对时间盲（来源：Best AI Tools for ADHD Productivity in 2026）。

这些工具的共同点：它们都是**外部调度层**，接管了规划、排序和重锚定的工作。

但 Perplexity 不同。它被设计为信息检索与综合工具，而不是规划器。很多人用它来“帮我写个计划”或“告诉我该怎么做”——这本质上是在用生成能力替代调度功能。结果呢？你得到一个漂亮的计划，却仍然无法执行，因为启动和执行仍然依赖你脆弱的内部调度层。

正确用法是把 Perplexity 当作**重锚定系统**的一部分。比如，当你执行任务时偏离轨道，问它：“我当前在做什么？下一步该做什么？”它帮你重新聚焦，而不是替你规划。这类似于 Tiimo 的提醒功能，但更灵活。

## 历史 harness 的启示

孔子用“克己复礼”作为外部行为边界，每日反省，“吾日三省吾身”。他的 harness 是外在的“礼”和定期的自我审查——这对应 LLM 的“上下文窗口”和“定时 re-grounding”。于谦以“粉骨碎身浑不怕”的气节作为行为准则，严格操练军队，日夜准备防务——这对应 agent 的“任务优先级列表”和“防御性调度”。

他们的成就不是靠内部意志力，而是靠外部系统。孔子 70 岁才做到从心所欲不逾矩，一辈子和自己的冲动做斗争。于谦的“日夜准备”本质上是一种外部化执行功能。

## 核心观点：脚手架 vs 拐杖

AI 工具必须作为**脚手架**使用，而不是**永久拐杖**。脚手架帮你搭建结构，但最终你要学会自己走。拐杖则让你永远依赖。

矛盾在于：当前证据多为用户报告，缺乏临床验证（来源：AI 与 ADHD 的时间管理）。Motion 等工具缺乏独立临床验证，效果可能受安慰剂效应影响（来源：矛盾与存疑）。过度依赖风险真实存在：AI 工具可能削弱内在执行功能的代偿发展（来源：拐杖与脚手架）。

更关键的是，同构性本身只是理论类比，并非科学事实（来源：矛盾与存疑）。我们不能简单地说“ADHD 大脑就是 LLM”，但可以借用这个框架来设计更好的工具。

## 今天就能试的行动

1. **用 Perplexity 做重锚定，而非规划**：执行任务时，每 30 分钟问它一次“我当前在做什么？下一步该做什么？”让它帮你回归轨道，而不是替你规划。
2. **设置外部提醒系统**：用 Todoist 或 Tiimo 创建视觉时间线，把时间变成可见元素。关键：提醒要自动触发，不要依赖你自己去设置（来源：AI Assistant for ADHD: Finally, a Productivity Tool That Requires Less...）。
3. **测试自适应规划器**：试用 Motion 或 Reclaim.ai 一周，观察它是否减少你的决策疲劳。注意：如果它让你更焦虑，说明它成了拐杖，而不是脚手架。
4. **建立“日课”系统**：像孔子一样，每天固定时间反思。用 AI 工具记录你的执行情况，但不要让它替你做决定。

最终，AI 工具的价值不在于它有多聪明，而在于它是否帮你搭建了你自己无法搭建的结构。用对了，它是脚手架；用错了，它是拐杖。

# Lex 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Lex：它到底补上了哪一层执行功能？

## 问题：你丢过目标吗？

ADHD 大脑最折磨人的不是“没法开始”，而是“开始后跑偏”。你可能在写周报时突然研究起“为什么猫会踩奶”，一小时后才意识到自己连邮件都没打开。这种**目标漂移**——从主线任务滑向旁支却不自知——就像 LLM 在长程对话中逐渐遗忘原始指令，输出越来越偏。工程师们管这叫“Agent 目标漂移”，解决方案是**重锚定 system prompt**：定期把原始目标塞回上下文窗口。

那么，有没有一个工具，能对 ADHD 大脑做同样的事？

## 同构：缺失的调度层

ADHD 大脑与 LLM 在结构上高度同构：两者都是**高产但缺执行调度层的生成核心**。ADHD 患者能产生海量想法，却无法可靠地启动、维持、切换任务；LLM 能生成流畅文本，却需要精心编排的 harness 才能完成多步推理。AI 工具帮助 ADHD 专注的本质，就是“给生成核心套 harness”——通过外部脚手架补偿执行功能缺陷，包括工作记忆、任务启动、时间盲和上下文保持（来源：AI 与 ADHD 的专注力）。

**Lex 恰好补上了“上下文保持”这一层。**

## Lex 的重锚定机制

Lex 是一款 AI 写作工具，但它的核心价值不在“帮你写”，而在**帮你记住“你正在写什么”**。它会在侧边栏持续显示你的文档大纲、关键段落和写作目标，相当于一个永不消失的 **system prompt**。当你分心写了一段无关内容，Lex 会通过“建议重写”或“回到主题”的提示，温和地把你的注意力拉回主线。

这与 LLM 工程中的“重锚定”完全同构：在 Agent 系统里，harness 会定期将原始任务描述注入上下文，防止模型跑偏。Lex 的侧边栏和提示功能，就是 ADHD 大脑的“重锚定 system prompt”——它不替代你的创造力，而是确保创造力始终围绕目标展开。

## 为什么有人用错了？

很多人把 Lex 当成“更好的 Word”或“AI 写手”，用它直接生成全文，然后发现自己根本没参与思考。这就是**把脚手架当成了拐杖**。

脚手架与拐杖的边界在于：**脚手架在任务完成后可以撤除，而拐杖会取代能力**（来源：矛盾与存疑）。Lex 的正确用法是：用它来**保持方向**，而不是替你走路。如果你让 Lex 替你写完整篇文章，你的写作能力不会增长；但如果你用它来时刻提醒自己“当前段落的核心论点是什么”，你就在训练自己的执行功能。

同样，在 Agent 工程中，好的 harness 不会替 Agent 做决策，而是提供上下文和工具接口，让 Agent 自己完成推理。Lex 的“重锚定”功能，就是 ADHD 大脑的 harness。

## 证据与局限

目前支持 Lex 作为 ADHD 工具的证据主要来自**概念对齐**，而非独立临床研究。这与大多数 ADHD × AI 工具的情况一致：Brain.fm 的神经锁相技术缺乏针对 ADHD 的随机对照试验；Focusmate 的效果主要归因于身体在场效应而非 AI 本身；RescueTime 的自动分类可能不够精准，对执行功能弱的用户仍是挑战（来源：AI 与 ADHD 的专注力）。

Lex 的优势在于它**直接对应了 ADHD 的核心痛点——上下文保持**，而其他工具更多补偿工作记忆（RescueTime）或任务启动（Focusmate）。但同样存在争议：长期使用是否会导致依赖？目前没有研究证明这类脚手架可以逐步撤除（来源：矛盾与存疑）。

## 今天就能试的 3 件事

1. **用 Lex 写一篇短文**：打开 Lex，先写一个标题和三个要点，然后开始写正文。每写一段，看一眼侧边栏的要点，问自己：“这一段对得上哪个要点？”——这就是你的重锚定 moment。

2. **对比使用前后**：用 Lex 写一篇文章，再用普通编辑器写另一篇。观察自己跑偏的次数和程度。你会发现，Lex 的侧边栏就像 Agent 的 system prompt 注入，显著减少上下文漂移。

3. **设置“目标检查”提醒**：如果你不用 Lex，可以在手机上设一个每 15 分钟响一次的闹钟，上面写着你当前的任务。这是最原始的重锚定——但效果惊人。

## 结论

Lex 不是万能药。它不能修复你的工作记忆，不能帮你启动任务，也不能替代多巴胺调节。但它精准地补上了 ADHD 调度层中缺失的“上下文保持”功能，正如重锚定 system prompt 之于 LLM。

关键在于：**用它来保持方向，而不是替你走路。**

当你把它当作一个永不消失的“当前目标”提示器时，它就不再是拐杖，而是真正可撤除的脚手架。

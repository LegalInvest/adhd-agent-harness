# LLM 的 planner 没有情绪、没有睡眠债、没有‘今天就是不想做’的动机低谷。把 ADHD 的启动困难简化为‘缺少一个好的 planner-executor loop’，是否把一种神经发育差异降级成了软件架构问题，从而忽视了执行功能背后的情感与生理维度？

> 情感与身体维度在类比中消失了吗

你打开 Motion，看着它自动把任务排进日历——没有犹豫，没有“先刷五分钟手机再说”。你羡慕那个 planner 没有情绪、没有睡眠债、没有“今天就是不想做”的动机低谷。但把 ADHD 的启动困难简化为“缺少一个好的 planner-executor loop”，是否把一种神经发育差异降级成了软件架构问题？

## 同构的脊柱：生成核心与调度层

ADHD 大脑与 LLM 共享同一个核心困境：都是高产的生成核心，却缺乏可靠的内置调度层。LLM 可以写出优美的段落，但如果没有 prompt 工程或 agent 框架的约束，它会上下文膨胀、推理退化（来源：上下文工程）。ADHD 大脑可以进入超聚焦完成惊人作品，但如果没有外部脚手架，它会因时间盲、工作记忆不稳定而无法启动任务（来源：AI 与 ADHD 的时间管理）。

这个同构不是比喻，而是一种可操作的工程视角。孔子一生精力旺盛、冲动爱骂人、思维跳跃，他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省。这本质上是一个外部调度层：礼是社会化的 planner，每日反省是定时 re-grounding。曹植则用诗赋作为核心，把所有政治失意注入文学——诗赋是他的 executor，酒精是他的情绪调节器。这些历史案例说明，ADHD 个体一直在自发构建 harness，而 AI 工具只是把这个过程系统化了。

## ADHD 侧的脚手架：外部执行功能层

现有 AI 工具通过三种机制补偿 ADHD 的执行功能缺陷：

1. **自动排程消除决策负担**：Motion 和 Reclaim.ai 自动评估任务优先级、截止日期和可用时间，实时重建日程（来源：Motion、Reclaim.ai）。这解决了“下一步该做什么”的决策瘫痪，直接对应 ADHD 的任务启动困难。
2. **视觉化时间对抗时间盲**：Tiimo 将时间转化为可见元素，用颜色和结构帮助用户感知时间流逝（来源：Tiimo）。小规模研究支持视觉化时间线能提升时间感知准确性（来源：AI 与 ADHD 的时间管理）。
3. **外部记忆补偿工作记忆**：ADHD 大脑需要“永远在线”的外部记忆体，Obsidian 等工具作为“第二大脑”存储项目上下文（来源：无状态与外部记忆）。这类似 LLM 依赖检索增强生成（RAG）来弥补无状态缺陷。

这些工具共同构成了一个“外部执行功能层”，显著降低认知负荷。但问题在于：它们能否处理情绪与生理维度？

## LLM/agent 侧的脚手架：planner-executor 循环

LLM agent 的工程实践同样面临调度难题。控制逻辑分散、上下文膨胀导致推理退化，因此工程师设计 planner-executor 循环：planner 分解任务，executor 逐步执行，并引入 re-grounding 机制防止漂移（来源：上下文工程）。这与 ADHD 使用 Routinery 创建步骤序列、用过渡提示引导注意力的策略同构（来源：上下文工程）。

然而，LLM 的 planner 没有睡眠债。它不会因为昨天熬夜而降低执行效率，不会因为多巴胺不足而拒绝启动。ADHD 大脑的 executive function 障碍背后是前额叶皮层活动不足，而多巴胺系统直接影响动机（来源：AI 与 ADHD 的时间管理）。Motion 可以排好日程，但无法让你在“今天就是不想做”时产生动力。

## 脚手架 vs 拐杖：情感与身体的维度

这里存在一个危险：把 ADHD 的启动困难完全等同于“缺少 planner-executor loop”，会忽视情感与生理维度。孔子到了70岁才“从心所欲不逾矩”，说明调度层的内化需要一生的练习和情感成长。曹植用酒精麻痹自己，虽然不健康，但揭示了一个事实：情绪状态直接影响执行能力。

当前工具如 Motion 和 Reclaim.ai 主要处理时间结构，但未涉及多巴胺调节或情绪波动（来源：Reclaim.ai 局限）。Goblin Tools 的任务分解功能更贴近 planner 角色，但同样缺乏情感支持。过度依赖这些工具可能导致“永久拐杖”效应，削弱内在执行功能的代偿发展（来源：AI 与 ADHD 的时间管理）。

## 鲜明的判断：同构是工具，不是本质

我的核心观点是：**ADHD 大脑与 LLM 在“缺乏调度层”上的同构是真实且有用的工程视角，但它只描述了问题的一半。另一半是情感与生理维度——这些无法通过软件架构解决，需要结合行为干预、睡眠管理和情绪调节。**

证据的局限也在此：现有工具缺乏随机对照试验验证，效果可能受安慰剂效应影响（来源：AI 与 ADHD 的时间管理）。同构性说法本身缺乏实证基础，仅是一种理论类比（来源：矛盾与存疑）。我们不应把神经发育差异降级为软件架构问题，而应把软件架构作为整体解决方案的一部分。

## 今天就能试的行动

1. **用 Motion 或 Reclaim.ai 自动排程一周**：观察决策负担是否减少，同时记录情绪波动对执行的影响。
2. **建立“外部记忆”系统**：用 Obsidian 或 Todoist 存储项目上下文，每天开始工作时先回顾，减少工作记忆负荷。
3. **设置情绪检查点**：在每个任务切换前，问自己“我现在感觉如何？”，承认情绪对启动的影响，而不是强行忽视。
4. **尝试 Tiimo 的视觉时间线**：如果时间盲严重，用可视化工具让时间变得可感，同时留意是否过度依赖。

ADHD 的启动困难不是代码 bug，不能靠一个 planner 就修复。但 planner 可以成为脚手架的一部分——前提是我们承认，脚手架下面还有身体和情绪的地基。

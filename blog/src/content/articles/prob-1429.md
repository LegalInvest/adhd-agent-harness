# Saner.AI 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Saner.AI：它到底补上了哪一层执行功能？

你打开 Saner.AI 准备写报告，10 分钟后却发现自己正在研究“为什么猫会踩奶”。这不是你不够努力——是你的大脑和 LLM 一样，是个“高产但缺执行调度层的生成核心”。ADHD 大脑与 LLM 在结构上同构：两者都擅长生成，但内置调度层不可靠，需要外部脚手架来对抗目标漂移（来源：AI 与 ADHD 的专注力）。Saner.AI 的核心理念——用 AI 帮你记住“你本来想干什么”——正是这个同构的产物。但很多人用错了：他们只把它当记录仪，没把它当重锚定系统。

## 同构：ADHD 大脑与 LLM 共享的困境

ADHD 的工作记忆缺陷与 LLM 的无跨会话状态同构：两者都无法可靠地记住“上一刻的目标”。LLM 在长上下文里会因注意力膨胀而丢失原始指令（来源：认知负荷），ADHD 大脑则在任务中途被环境带偏，产生任务碎片化（来源：AI 与 ADHD 的专注力）。两者的解法也同构：LLM 需要“重锚定 system prompt”——在关键节点重新注入原始目标；ADHD 大脑需要“外部执行功能层”——一个能定期提醒“你本来要做什么”的机制。

曾国藩 30 岁前就是典型的 ADHD：日记里天天骂自己“无恒”“浮躁”，读书“不能终一行”。他的 harness 是“日课十二条”：黎明即起、读书不二、写日记反省。每条日课都是一次“重锚定”——和 LLM 每 N 步重读一遍 system prompt 一样。孙策的 harness 则更粗放：他只用武力打仗，把行政全扔给张昭张纮，相当于把“调度层”外包给了外部团队。这两种 harness 的本质都是：承认内部调度不可靠，用外部结构补偿。

## Saner.AI 补上了哪一层？

Saner.AI 被列为 2026 年最佳 ADHD 应用之一（来源：Brain.fm 页，同榜包括 Brain.fm、Forest 等）。它的核心功能是：用 AI 记录你的任务意图，并在你跑偏时温和提醒。这直接补偿了 ADHD 的“时间盲”和执行功能缺陷（来源：RescueTime 页）。但问题在于：很多人只把它当“智能笔记本”——记下目标，然后继续跑偏。

真正的价值在于“重锚定”而非“记录”。就像 Focusmate 通过虚拟身体在场提供外部问责（来源：Focusmate 页），Saner.AI 需要主动触发“你现在在做什么？和原始目标一致吗？”的检查。否则，它就和 RescueTime 的自动分类一样——数据有了，但用户可能懒得看（来源：RescueTime 页局限）。

## 拐杖 vs. 脚手架：用错的边界

同构视角揭示了一个危险：AI 工具既可能成为逐步内化的脚手架，也可能成为永久依赖的拐杖（来源：AI 与 ADHD 的专注力）。曾国藩的日课最终内化成了习惯，孙策的“外包”则始终依赖他人。Saner.AI 如果只帮你“记住”而不帮你“练习记住”，它就是拐杖。真正的脚手架应该是：用几次后，你的大脑开始自己问“我现在该做什么？”——就像 LLM 微调后不再需要每步重读 prompt。

## 争议与局限

同构性目前只是功能类比，并非经过验证的科学事实（来源：矛盾与存疑）。缺乏针对 ADHD 的独立临床研究（来源：Brain.fm 页）。个体差异巨大：注意缺陷为主型 vs. 多动冲动型对 Saner.AI 的反馈机制需求不同（来源：AI 与 ADHD 的专注力）。此外，过度依赖 AI 提醒可能削弱内化能力——尚无长期追踪研究（来源：矛盾与存疑）。

## 今天就能试的行动

1. **设置“重锚定”提醒**：在 Saner.AI 中，每 25 分钟设置一次“你现在在做什么？”的主动弹窗，而非只记录。
2. **模仿曾国藩的“日课”**：每天固定 3 个时段（如早中晚），用 Saner.AI 的回顾功能检查：我的注意力漂移了几次？哪些触发点最危险？
3. **区分记录与重定向**：用 Saner.AI 记录意图后，立即设定一个“如果跑偏就回到 X 任务”的规则——就像 LLM 的 if-then prompt。
4. **结合身体在场**：用 Focusmate 的虚拟问责时段配合 Saner.AI 的任务记录，形成“外部调度层+外部记忆”的双重脚手架。

# 如果把 ADHD 大脑当成生成核心，Forest 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Forest 对 ADHD 的真正价值。

你打开 Forest，种下一棵树，告诉自己：“这 25 分钟只做这一件事。” 但 10 分钟后，你已经在刷手机，或者超聚焦在某个无关细节上——树枯了。这不是 Forest 的错，而是 ADHD 大脑的常态：分心走神或超聚焦跑偏，丢失目标。同样的问题，在 AI agent 工程里被称为“目标漂移”——一个长程任务中，agent 的生成输出会逐渐偏离初始 system prompt，需要定期重锚定。

如果把 ADHD 大脑当成生成核心，Forest 能不能当它的重锚定与目标漂移？答案是：能，但有条件。

## ADHD 大脑与 LLM：同一种生成核心

ADHD 大脑与 LLM 在结构上同构：两者都是**高产但缺乏可靠执行调度层的生成核心**（来源：主题综述）。ADHD 的创意发散、超聚焦、兴趣驱动，对应 LLM 的文本生成、上下文扩展；而 ADHD 的执行功能障碍（计划、时间管理、任务启动）对应 LLM 的无状态性、上下文膨胀导致的输出失控。

这种同构意味着，ADHD 需要的不是“治好”自己，而是像给 LLM 搭建 harness 一样，给自己搭建一个外部执行功能层。曾国藩的“日课十二条”——黎明即起、读书不二、谨言、写日记反省——就是一个典型的 harness。他通过固定的行为序列（日课↔定时 re-grounding）和外部记录（日记↔日志系统），抵消了自己“无恒”“浮躁”的 ADHD 特质。苏轼则用“豁达”作为核心 prompt，无论被贬到哪里，都用美食、写作、交友转移注意力，避免陷入情绪失调。他们的 harness 系统与 LLM 的 system prompt + 外部记忆结构同构：日课是定时重锚定，日记是外部状态记录，秘书（若有）是外部调度器。

## Forest 作为锚点：视觉化的重锚定

Forest 的核心机制很简单：种一棵树，离开 app 树就会枯死。这个机制对 ADHD 的价值，不在于它计时，而在于它提供了一个**可感知的视觉锚点**。当注意力开始漂移时，那棵正在生长的树就是一个外部信号：“嘿，你的目标是什么？” 这相当于 agent 工程里的“重锚定 system prompt”——在 agent 生成偏离时，注入初始目标约束。

但 Forest 的证据强度有限。资料显示，Forest 在 ADHD 诊断分类中表现良好，但作为专注工具的证据不足（来源：主题综述）。这意味着，Forest 本身并不是一个经过临床验证的 ADHD 干预工具。它的有效性取决于用户是否主动将其嵌入一个更大的 harness 系统。

## 从 Forest 到 Harness：脚手架而非拐杖

如果 Forest 只是独立使用，它很容易沦为“拐杖”——用户依赖那棵树带来的短暂多巴胺，一旦树枯了就放弃，甚至因为挫败感而更焦虑。真正的价值在于，将 Forest 作为 harness 的一部分：

1. **与 Focusmate 搭配**：Focusmate 通过虚拟身体在场提供实时问责（来源：Focusmate），弥补 Forest 缺乏社会监督的短板。预约一个 Focusmate 时段，同时打开 Forest，双重锚定。
2. **与 RescueTime 搭配**：RescueTime 自动记录时间去向（来源：RescueTime），揭示 Forest 时段内的真实注意力分布。如果 Forest 显示“专注 25 分钟”，但 RescueTime 显示你刷了 10 分钟社交，你就知道锚点失效了。
3. **与 Brain.fm 搭配**：Brain.fm 通过神经锁相技术调节大脑状态（来源：Brain.fm），可能帮助进入专注状态，降低 Forest 锚点失效的概率。

这种组合，相当于 agent 工程里同时使用 system prompt（Forest）、外部记忆（RescueTime）、环境调节（Brain.fm）和实时反馈（Focusmate）。

## 争议与局限

但必须诚实指出，这种同构是隐喻而非严格等价。LLM 的“注意力机制”与人类注意力存在本质差异（来源：主题综述）。此外，AI 工具作为外部执行功能层，长期使用可能导致内在执行功能退化（来源：矛盾与存疑）。曾国藩的日课坚持了一辈子，但苏轼的“豁达”则更像一种内在化 prompt——他不需要外部工具，因为他已经将重锚定内化。

另一个争议是：超聚焦应被引导还是打断？Forest 的计时机制本质上是打断，但部分 ADHD 用户反馈强行打断（如番茄钟）反而有效（来源：矛盾与存疑）。这提示，Forest 可能更适合需要“启动”而非“维持”的场景。

## 今天就能试的行动

1. **设定一个“锚点树”**：在开始一项任务前，打开 Forest，并写下一句话：“这 25 分钟，我只做______。” 如果走神，回来重新读这句话——这就是你的重锚定 system prompt。
2. **搭配一次 Focusmate**：预约一个 25 分钟 Focusmate 时段，同时种树。让伙伴知道你的目标，结束时汇报。
3. **检查 RescueTime 报告**：一周后，查看 RescueTime 中 Forest 时段的时间分类。如果“高效”比例低于 60%，调整你的任务选择。
4. **写一次“日课”**：像曾国藩一样，写下 3 条固定行为（如：起床后先种树 25 分钟、午休后写 5 分钟日记）。坚持一周。

Forest 不是解药，但它可以成为你 harness 系统里的一颗螺丝钉。关键不是树本身，而是你愿意为它搭建的整个脚手架。

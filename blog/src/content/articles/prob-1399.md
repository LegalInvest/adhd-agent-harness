# RescueTime 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 RescueTime：它到底补上了哪一层执行功能？

你打开 RescueTime 的仪表盘，看到自己昨天在社交媒体上耗了 4 小时，文档编辑只有 15 分钟。数据和内疚都有了，但明天呢？该启动的任务还是启动不了，该拆解的项目还是一团乱麻。这不是 RescueTime 的错——它被当成了日记本，而不是调度台。

ADHD 大脑的核心困境，和 LLM agent 面临的问题是同一类：两者都是强大的生成核心，但缺乏可靠的内置调度层。LLM 需要 planner-executor 循环来分解长任务、控制随机性；ADHD 大脑需要外部脚手架来启动任务、分解模糊目标。RescueTime 如果只做记录，就像给 LLM 只加了一个日志系统而不给控制流——数据有了，但行为不变。

## 同构：ADHD 大脑与 LLM agent 共享的调度缺口

ADHD 个体的表现波动，本质上类似 LLM 的采样温度波动。ADHD 的注意力、执行功能在一天内起伏剧烈，如同 LLM 在高温度下输出随机、不可控（来源：采样温度与表现波动）。这种波动源于缺乏稳定的执行调度层——ADHD 大脑的前额叶皮层活动不足，无法像神经典型大脑那样自动规划优先级、抑制干扰（来源：AI 与 ADHD 的时间管理）。

LLM agent 同样面临非确定性问题：小错误会累积，可重复性差。工程实践采用“计划-执行”分离模式来强制单向、确定性的控制流（来源：采样温度与表现波动）。最简单的 agent 工作流本身就是确定性的，遵循预定义步骤。这正是 ADHD 需要的：一个外部 planner 来分解任务、一个 executor 来按步执行。

## RescueTime 的正确用法：从记录到调度

RescueTime 的定位是被动追踪，但 ADHD 需要的不是更多数据，而是更少决策。真正有效的工具是那些承担调度功能的：Motion 自动根据优先级和截止日期重建日程，消除“下一步做什么”的决策负担（来源：Motion）；Reclaim.ai 用防御性时间块保护深度工作，减少突发任务带来的日程崩溃（来源：Reclaim.ai）；Tiimo 将时间可视化，让时间盲变得可见（来源：Tiimo）。这些工具共同构成了一个“外部执行功能层”，将日程管理从认知负担转化为被动接收（来源：AI 与 ADHD 的时间管理）。

RescueTime 可以成为这个调度层的一部分——但前提是它的数据被用来驱动自动调整，而不是仅仅呈现。例如，将 RescueTime 的专注时段数据喂给 Reclaim.ai，让 AI 自动在低能量时段安排低认知任务，在高能量时段封锁深度工作。这时 RescueTime 才从日记本变成了调度台的传感器。

## 人物案例：左宗棠的“结硬寨打呆仗”与 agent 的确定性工作流

左宗棠是典型的 ADHD 特质：精力旺盛、脾气大、40 多岁才出山，但一旦启动就风风火火，今天的事今天做完。他的 harness 系统是“结硬寨打呆仗”——每天读经写家书，用诸葛亮的“鞠躬尽瘁”要求自己。这本质上是一个外部 planner-executor 循环：日课（定时 re-grounding）作为 planner，写家书等日常作为 executor，强制单向、确定的控制流。

这与 LLM agent 的确定性工作流同构：最简单的 agent 工作流遵循预定义步骤，没有随机分支（来源：采样温度与表现波动）。左宗棠的“日课”就是 agent 的定时 re-grounding，他的“秘书”（外部调度器）就是 agent 的工具调用层。没有这个 harness，他的高能量可能变成乱冲乱撞；有了它，他收复了新疆。

## 争议与局限：脚手架还是拐杖？

AI 工具可能从“脚手架”退化为“永久拐杖”，削弱用户内在执行功能的代偿发展（来源：AI 与 ADHD 的时间管理）。当前缺乏长期研究评估这种依赖的后果。此外，同构性类比本身缺乏实证基础，目前仅是一种理论假设（来源：矛盾与存疑）。Motion 等工具也缺乏独立临床验证，效果可能受安慰剂效应影响（来源：矛盾与存疑）。

关键在于区分“调度层”与“替代层”。一个好的调度层像左宗棠的日课——它约束但不替代，用户仍需主动执行。而一个坏的拐杖像自动驾驶——用户完全交出控制，能力退化。RescueTime 如果只记录不干预，就是拐杖；如果数据驱动调度，就是脚手架。

## 今天就能试的行动

1. **将 RescueTime 与自动调度工具联动**：如果使用 Motion 或 Reclaim.ai，将 RescueTime 的专注时段数据导入，让 AI 根据你的实际能量曲线自动安排任务。
2. **用 Goblin Tools 分解模糊任务**：当你面对“写报告”这种大任务时，用 Goblin Tools 的 AI 将其分解为 5-10 个小步骤，降低启动门槛（来源：工具使用与认知卸载）。
3. **设置一个“日课”定时器**：像左宗棠一样，每天固定时间（如早 9 点）用 5 分钟列出今天的 3 个关键任务，并设定每个任务的执行时间块。
4. **测试视觉化时间线**：试用 Tiimo 或 Structured，将一天的时间块可视化，观察是否减少时间盲带来的焦虑。

RescueTime 不是答案，它是问题的传感器。真正的答案在于：你是否愿意为自己搭建一个 planner-executor 循环，让外部调度层接管那些大脑天生不擅长的任务。这不是偷懒，这是工程——给高产生成核心套上 harness，让它输出该有的价值。

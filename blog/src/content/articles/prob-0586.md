# ChatGPT 之于 ADHD，就像 调低采样温度 之于 LLM——但有人用错了

> 从同构视角实测 ChatGPT：它到底补上了哪一层执行功能？

## 上午 10 分、下午 0 分，和采样温度

对 ADHD 读者，最熟悉的崩溃可能是：早上思路喷涌、下午连回一封邮件都像搬运巨石；同一个人，同一天，表现差异大得不像同一颗大脑。对做 LLM Agent 的工程师来说，这像极了采样温度（sampling temperature）没调好的模型：temperature 高，输出跳跃、发散、不可控；temperature 低，输出稳定，却容易失去灵气。问题在于，人脑没有 API 可以直接改 `temperature`。

于是很多人把 ChatGPT 当成那个“调低温度”的旋钮。他们问：“我今天该做什么？”“我没动力，能不能鼓励我一下？”得到一长串答案，然后继续躺着。另一头，工程师也会遇到类似的误用：以为把 temperature 调到 0 就能让 Agent 稳定，却发现没有调度层（planner、tool use、memory、reflection）的 LLM 照样会生成“看起来合理但偏离目标”的东西。

ChatGPT 之于 ADHD，确实像调低采样温度——**但有人用错了**。它不是让 ADHD 大脑本身变稳定的药物，也不是替你做主的“主脑”。它补上的是**执行功能层里缺失的调度层**。

## 同构脊柱：两个高产却缺调度的生成核心

AI 与 ADHD 研究里有一个核心判断：ADHD 大脑和 LLM 是同一类“高产但缺执行调度层的生成核心”。两边都有极强的生成能力——ADHD 有创造力、联想、冲动式洞察和超聚焦；LLM 有大规模知识、语言生成与采样多样性。但两者都缺乏一个可靠的内置调度层：把碎片化的潜能转化为目标导向行为的那层结构。ADHD 缺的是执行功能；LLM 缺的是 agent scaffolding（来源：AI 与 ADHD 的日常生活#核心论点）。

这一同构有实证支撑。认知负荷研究指出，ADHD 患者在 Stroop 冲突任务中难以抑制优势反应、干扰抑制能力随任务复杂度增加而崩溃；而在针对 Transformer 的实验里，当上下文序列变长、认知负荷增加，模型在冲突试次上同样骤降到随机水平，无法解决认知冲突（来源：Deficient Executive Control in Transformer Attention）。这并非修辞，而是两边在“长程注意控制与冲突抑制”上出现了相同的失效模式。

超聚焦（hyperfocus）也指向同一个问题。ADHD 大脑渴望新颖性，却容易在错误的事情上超聚焦数小时（来源：The Best AI-Powered ADHD Productivity Tools in 2026）；超聚焦与时间盲（time blindness）绑定，几个小时“咻”地消失，之后常常伴随崩溃（hyperfocus crashes）（来源：9 Best AI Assistants for ADHD in 2026 - by Nia）。LLM 的“超聚焦”则是它在单一语境里持续生成、停不下来，直到外部截断或 token 耗尽。两边都需要一个外部 harness：对 ADHD 是执行功能支架，对 LLM 是 agent scaffolding。

## ChatGPT 补上的不是多巴胺，是“下一步是什么”

那么 ChatGPT 到底补上了哪一层执行功能？答案是：**它把“目标→步骤→时间→记忆”这条原本要在前额叶里跑完的流程，搬到了外部。**

ADHD 执行功能教练 Yulia Rafailova 说，AI 提供了一个“不可思议的工具来外化我们的思维，作为一个支架，能显著减轻工作记忆上的认知负荷”（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）。这正是 ChatGPT 以及围绕它的一整套工具在做的事：

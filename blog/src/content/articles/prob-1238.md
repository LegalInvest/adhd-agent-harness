# 如果把 ADHD 大脑当成生成核心，Reclaim.ai 能不能当它的外部执行功能层？

> 用 agent 工程的思路重新理解 Reclaim.ai 对 ADHD 的真正价值。

## 从同一个问题出发：生成核心缺少 harness

如果你被 ADHD 折磨，你最熟悉的可能不是"不会想"，而是"想了很多，就是做不起来"。大脑里有源源不断的念头、创意、焦虑，但计划、组织、启动这些执行功能却像总线负载过高的调度器一样频频崩溃。工程师们看到 LLM 时会有相似的既视感：生成能力惊人，却无状态、无长期记忆、缺乏可靠调度层，容易上下文膨胀、推理退化，甚至产生幻觉（来源：ADHD × AI 的科学与研究前沿）。

把 ADHD 大脑当成生成核心，问题就变成了：我们需要怎样的外部 harness，来弥补那个不可靠的执行调度层？Reclaim.ai 能担当这个角色吗？

## 同构：两边的缺陷一模一样

ADHD 一侧的证据是明确的：执行功能、工作记忆是核心缺陷，时间盲、任务启动困难、注意力易漂移是典型表现（来源：ADHD × AI 的科学与研究前沿）。LLM/agent 一侧的证据同样具体：LLM 需要严格的安全控制与高效的上下文管理，否则会出现上下文膨胀和推理退化；agent harness 必须提供上下文传递、工具接口、规划工件、验证循环、记忆系统和沙盒（来源：上下文工程；幻觉与验证循环）。

这不是比喻，而是结构对应：
- ADHD 的工作记忆不稳定 ↔ LLM 的无状态性，都靠外部记忆系统补偿；
- ADHD 的时间盲与注意力漂移 ↔ LLM 的上下文膨胀与推理退化，都需要外部上下文工程；
- ADHD 的冲动与"想当然" ↔ LLM 的幻觉，都需要验证循环把内部输出拉回外部现实（来源：幻觉与验证循环）。

## 曾国藩的 harness：古老的人工版 agent scaffolding

晚清曾国藩是这一同构的最佳人物案例。他 30 岁前浮躁坐不住，日记里天天骂自己"无恒"；读书时"他人目下二三行，余或疾读不能终一行"；情绪也极不稳定，打败仗就跳水自杀。但他用一套笨 harness 把自己从生成核心变成了可执行系统：

- **日课十二条**：黎明即起、读书不二、谨言、写日记。这相当于 agent 的定时 re-grounding 与固定 prompt template，每天把被环境带偏的注意力重新校准；
- **终身写日记反省**：这是外部记忆与日志系统，把容易丢失的元认知外化；
- **结硬寨打呆仗**：最稳最慢的战法，对应 agent harness 里的保守规划与 guardrails，用流程约束冲动（来源：人物案例）。

他的日课不是"意志力强"，而是承认生成核心不可信，于是用外部脚手架接管调度。

## 工具层：谁能做真正的外部执行功能层？

在今天，这套 harness 正被 AI 工具重构。关键不是"自动"，而是能否覆盖执行功能的全链路：

- **任务分解**：Goblin Tools 的 Magic ToDo 把"清理厨房"这类模糊任务拆成可执行子任务，并可通过"辣度"调节粒度，直接降低 ADHD 的任务启动门槛（来源：Goblin Tools）；
- **自动排程**：Motion 能根据优先级、截止日期、可用时间动态重建日程，在用户日程被打乱时自动重新安排，减少"下一步该做什么"的决策负担（来源：Motion）；
- **外部记忆**：Saner.AI 通过本地记忆与知识回忆，减少 ADHD 用户在不同应用间切换和搜索的循环，降低认知负荷（来源：Saner.AI）；
- **上下文工程**：Routinery 等应用用顺序步骤和过渡提示来引导 ADHD 用户完成日常流程，本质是把"当下该注意什么"外部化（来源：上下文工程）。

那么 Reclaim.ai 呢？提供的资料中没有独立的 Reclaim.ai 工具页，因此我无法核实它的具体功能边界。如果它只解决"自动排日程"这一环，那它是 harness 里的一个工具接口，而不是完整的外部执行功能层；如果它同时整合了任务捕获、优先级重排、外部记忆与提醒验证，才有资格被称作核心调度层。

## 核心判断：脚手架与拐杖的边界

我的判断是：Reclaim.ai 这类 AI 日程工具能否成为 ADHD 大脑的 harness，不取决于它是否"智能"，而取决于它是否形成闭环——是否能像 agent harness 一样，同时做上下文工程、外部记忆、任务分解、调度与验证。只做自动排程，它可能只是局部脚手架；只有在你能借此逐步建立可迁移的调度习惯时，它才不是永久拐杖。

这个判断的局限必须诚实承认：
- **同构仍是理论框架**：ADHD 大脑与 LLM 的同构主要依赖概念类比，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）；
- **长期效果未知**：AI 工具对任务启动、认知负荷的辅助效果依赖个体差异，缺乏长期纵向研究（来源：矛盾与存疑）；
- **过度依赖风险**：AI 工具可能从"可拆卸脚手架"变成"永久拐杖"，但当前缺乏实证研究来界定这个边界（来源：矛盾与存疑）。

## 今天就能试的 4 件事

1. **建立"日课"最小闭环**：每天固定时间用你已有的日历或待办工具做一次 re-grounding（类似曾国藩日课），把"今天要做的三件事"外化，而不是靠大脑硬记。工具不重要，重复才重要。

2. **给模糊任务做"辣度分解"**：遇到"写报告"这种让大脑死机的任务，先用 Goblin Tools 的 Magic ToDo 或纸笔拆成 5-10 分钟能完成的微步骤，只给第一个步骤定时间。

3. **用日历做外部记忆，而不是只放会议**：把任务块、休息块、"思考时间"都写进日历，让日程本身成为上下文工程的一部分，减少实时决策。

4. **每周做一次验证循环**：像 agent 运行测试用例一样，回顾本周任务哪些真的完成了、哪些被高估或低估，调整下周的预估与粒度。没有这个反思，任何工具都可能变成拐杖。

## 结语

把 ADHD 大脑当成生成核心，不是自我贬低，而是换一种更准确的工程视角。Reclaim.ai 能不能当外部执行功能层？答案是：如果它只是帮你排日程，那它是 harness 的一个插件；只有当你围绕它构建起上下文管理、外部记忆和验证循环的完整系统时，它才接近那个核心调度层的位置。真正的 harness 从来不是某个工具，而是工具+流程+反思的闭环。

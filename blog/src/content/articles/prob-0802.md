# Goblin Tools 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Goblin Tools：它到底补上了哪一层执行功能？

你打开一个空白文档，想写周报。大脑一片空白，光标闪烁，五分钟过去了。你打开一个待办清单，看到“整理项目资料”——然后关掉它，因为不知道从哪里下手。

这不是懒。这是ADHD大脑的典型困境：**生成能力很强，但缺一个能把“做某事”翻译成“先做A，再做B”的调度层**。

有意思的是，LLM（大语言模型）也有同样的毛病。你给它一个“写一篇万字报告”的指令，它要么输出一堆废话，要么直接拒绝。直到你给它一个planner-executor循环：先让planner把任务拆成“引言→方法→结果→讨论”，再让executor逐个执行。

Goblin Tools 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了。

## 同构：两个“高产但缺调度”的核心

ADHD 大脑与 LLM 在结构上同构——都是“高产但缺乏可靠执行调度层的生成核心”（来源：AI 与 ADHD 的时间管理）。

ADHD 侧，问题出在[[执行功能]]缺陷：[[工作记忆]]容量保留但控制弱，[[时间盲]]让人无法感知时间流逝，[[任务启动]]困难让“开始”本身成为巨大障碍。结果是：脑子里有无数想法，但无法转化为有序的行动。

LLM/agent 侧，问题出在上下文控制缺陷：无状态性让模型记不住长对话，缺乏对时间约束的固有理解，面对复杂指令容易“跑偏”或“卡住”。结果是：模型有知识，但无法可靠地完成多步骤任务。

两边的解法结构同构：都需要一个外部脚手架来补偿缺失的调度层。ADHD 需要[[外部执行功能层]]——AI 工具充当“第二大脑”，帮它拆任务、估时间、给提醒。LLM 需要 planner-executor 循环——一个专门的模块先把任务分解成可执行的子任务，再逐个执行。

## Goblin Tools 补上了哪一层？

Goblin Tools 是一个极简的 AI 工具集，核心功能是**任务分解**：你输入“整理房间”，它会输出“1. 把床铺好 2. 把书放回书架 3. 擦桌子……”——并且可以调节分解的粒度（从“粗略”到“非常详细”）。

这直接对应 ADHD 的[[任务启动]]困难。启动困难的核心原因之一是“任务看起来太大、太模糊，大脑不知道从哪里开始”。Goblin Tools 通过把抽象指令翻译成具体步骤，降低了启动门槛。

同样，planner-executor 循环中的 planner 模块，做的也是同一件事：把“写一篇关于气候变化的文章”分解成“1. 收集数据 2. 列出论点 3. 写引言……”。

但关键区别在于：**Goblin Tools 只负责“拆”，不负责“做”**。用户拿到步骤后，仍然需要自己去执行每一步。而 planner-executor 循环中的 executor 会替模型完成每一步。

这意味着：Goblin Tools 是一个**脚手架**——它帮你搭好结构，但你自己要爬上去。而很多用户误把它当成了**拐杖**——希望它替自己做完一切。

## 脚手架 vs 拐杖：边界在哪里？

[[拐杖与脚手架]]概念指出：AI 工具应该作为“补偿”而非“替代”，但实际使用中可能沦为“永久拐杖”。（来源：矛盾与存疑）

以 Motion 为例。它自动根据任务和截止日期创建每日计划，并在日程被打乱时重新安排（来源：Motion）。这直接解决了[[时间盲]]和决策疲劳。但问题是：如果用户从不自己规划，他们的时间感知能力会进一步萎缩。

同样，Reclaim.ai 通过保护深度工作块来维持日程稳定性（来源：Reclaim.ai）。但如果用户完全依赖它来“防御”会议，他们可能永远学不会如何主动保护自己的时间。

Goblin Tools 的风险类似：如果用户每次遇到任务都直接丢给 AI 去拆，而不尝试自己思考“这件事的第一步是什么”，那么拆解能力就得不到锻炼。

## 证据与局限

现有证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑）。例如，Goblin Tools 的任务分解功能被用户认为能降低启动困难，但没有随机对照试验证明它比传统清单更有效。

此外，个体差异很大。ADHD 亚型（注意力缺陷 vs. 多动冲动）对工具响应不同，现有工具多面向注意力缺陷型（来源：矛盾与存疑）。Goblin Tools 的“粗略”到“详细”粒度调节虽然灵活，但未必适合所有人。

另一个争议是：过度依赖 AI 工具可能削弱内在能力（来源：矛盾与存疑）。长期使用 Goblin Tools 的用户，是否还能在没有工具的情况下自己拆解任务？目前没有追踪研究。

## 今天就能试的行动

1. **用 Goblin Tools 拆一个你拖延已久的任务**。输入任务名，选择“中等”粒度，得到步骤列表。然后**只做第一步**。完成后，回来做第二步。这能绕过启动困难，同时保留你的执行权。

2. **对比手动拆解 vs AI 拆解**。选一个简单任务（比如“写一封邮件”），先自己列出步骤，再用 Goblin Tools 生成。对比差异，思考“AI 提供了什么我没想到的”。这能帮你意识到自己的拆解盲区。

3. **设置“无工具日”**。每周选一天，不用任何 AI 工具规划。只用手写清单或白板。这能测试你的内在执行功能是否在退化，同时保持锻炼。

4. **如果你是工程师**：在你的 agent 系统中加入一个“可调节粒度的 planner 模块”。让用户能控制分解的粗细程度，就像 Goblin Tools 的滑块一样。这能提升 agent 的灵活性和用户信任。

## 结语

Goblin Tools 和 planner-executor 循环，本质上都是给“高产但缺调度”的核心套上外部执行功能层。它们补的是“启动”和“拆解”这两层——不是“执行”层。

用对了，它是脚手架，帮你学会结构。用错了，它是拐杖，让你永远依赖。

关键在于：**你是在用工具补偿功能，还是在用工具替代功能？** 前者让你更强，后者让你更弱。

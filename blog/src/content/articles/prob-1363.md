# Reflect 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Reflect：它到底补上了哪一层执行功能？

你打开 Reflect，写下“写报告”，AI 自动分解成“搜集资料—列大纲—写初稿—修改”。你感到一阵轻松——终于有人帮你拆任务了。但三天后，你发现那个分解后的清单依然躺在那里，你甚至没打开过。问题出在哪？

## 同构：ADHD 大脑与 LLM 共享的困境

ADHD 大脑与 LLM 是同一类“高产但缺执行调度层的生成核心”。LLM 能生成流畅文本，但单独使用时，它会跑题、忘记上下文、陷入死循环——因为它没有内置的 planner-executor 循环来规划步骤、执行验证、动态调整。ADHD 大脑也类似：你灵感爆棚，但“任务启动困难”和“不会拆解”让计划永远停留在脑子里。（来源：生成核心与调度层）

现代 AI agent 工程用“复合 AI 系统架构”来弥补：分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩。（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）这些技术本质上是在 LLM 外部搭建调度层，防止“上下文膨胀和推理退化”。

对 ADHD 来说，Reflect 这类 AI 工具试图扮演同样的角色——外部调度层。它帮你分解任务、排定优先级、提醒下一步。但为什么很多人用错了？

## 错用：把脚手架当拐杖

“计划本和便签用了一周就崩溃了。”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）ADHD 用户太熟悉这种模式了。新工具带来的多巴胺峰值让你以为这次不同，但几周后，工具本身变成了又一个未完成的项目。

问题在于：Reflect 的分解功能只解决了“如何拆”，没解决“为何启动”。它假设你有一个稳定的执行引擎来接住分解后的子任务——但 ADHD 大脑恰恰缺少这个引擎。你得到了完美的任务列表，却依然没有动力去点开第一个子任务。

工具如 Motion 和 Reclaim.ai 试图通过自动排程消除“下一步该做什么”的决策负担。（来源：Motion）Tiimo 用视觉化时间线让时间流逝变得可感。（来源：Tiimo）但这些工具都面临同一个争议：它们可能从“脚手架”退化为“永久拐杖”，削弱用户内在执行功能的代偿发展。（来源：拐杖与脚手架）

## 正确用法：Reflect 作为 harness 的一部分

历史上有 ADHD 特质的人，早已发明了自己的 harness 系统。孔子“吾日三省吾身”，用外在的“礼”作为行为边界，一辈子和自己的冲动做斗争。他的 harness 核心是“日课”——一种定时 re-grounding 机制。辛弃疾 21 岁带 50 人冲几万人敌营抓叛徒，闲居时把所有精力注入写词和练剑。他的 harness 是用“词与剑”作为输出管道，把过剩的冲动转化为创造。

这些 harness 的共同点：它们不是取代执行功能，而是**为生成核心提供稳定的外部调度层**。孔子的“日省”相当于每天一次的 planner 重校准，辛弃疾的“练剑写词”相当于 executor 的固定输出通道。

对应到 LLM 工程，这就像 agent 框架中的“验证循环”和“记忆系统”。Harness 工程被定义为“设计围绕 AI 代理的脚手架——上下文交付、工具接口、规划工件、验证循环、记忆系统和沙箱”。（来源：GitHub - ai-boost/awesome-harness-engineering）

Reflect 的正确用法，是把它嵌入到你的个人 harness 中，而不是让它独自承担所有调度。比如：
- 用 Reflect 分解任务后，**手动设置一个“启动触发器”**（比如打开番茄钟、播放特定音乐），让外部信号激活执行。
- 将 Reflect 的分解结果导入 **Todoist 或 Obsidian**，利用这些工具的外部化记忆功能防止任务掉落。（来源：同构视角）
- 结合 **Goblin Tools** 的“任务分解”或 **Saner.AI** 的“情绪调节”，处理分解后的情绪阻力。（来源：Goblin Tools, Saner.AI）

## 争议与局限

同构性理论目前缺乏实证基础，仅是一种理论类比。（来源：矛盾与存疑）工具如 Motion 缺乏独立临床验证，其效果可能受安慰剂效应影响。（来源：Motion）过度依赖风险尚无长期研究评估。（来源：拐杖与脚手架）

更重要的是，ADHD 症状谱系广泛，某些用户对视觉化工具反应良好，另一些则更受益于自动排程。（来源：个体差异）Reflect 的分解功能对“不会拆解”的人有用，但对“启动困难”的人，它可能只是另一个漂亮的待办清单。

## 今天就能试的行动

1. **一次只用一个工具**：选 Reflect 或 Motion 之一，用两周，记录它帮你启动了哪些任务，哪些依然搁置。
2. **建立启动仪式**：在 Reflect 分解任务后，设置一个 5 分钟的身体动作（如站起来、喝口水）作为执行信号。
3. **日课重校准**：每天固定时间（如睡前）回顾 Reflect 的完成情况，像孔子的“三省”一样手动调整计划。
4. **限制工具数量**：不要同时用 5 个工具。选 1-2 个核心工具，把其他功能（如情绪调节、时间可视化）作为可选插件。

Reflect 不是万能药。它只是一个外部调度层——真正的工作，还是要靠你自己的 harness 来完成。

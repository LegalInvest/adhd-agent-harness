# Claude 解决了 ADHD 的执行功能（计划/组织/启动）先天偏弱，代价是什么？

> Claude 的收益与「能力退化」风险：一份诚实的同构权衡。

如果你是一个 ADHD 成人，你可能熟悉这种场景：脑子里有无数想法，但坐在书桌前就是无法开始写那份报告。如果你是一个 AI agent 工程师，你可能也熟悉另一种场景：模型能生成惊艳的文本，但让它完成一个多步骤任务时，它要么跑偏，要么死循环。

这两类人其实在解决同一个问题：**一个高产的生成核心，缺一个可靠的执行调度层**。

ADHD 大脑与 LLM 在结构上同构——两者都是“高产但缺执行调度层的生成核心”（来源：ADHD × AI 的科学与研究前沿）。AI 帮助 ADHD 的本质不是“修复缺陷”，而是“给生成核心套上 harness”（外部脚手架）。Claude 正是这样一个 harness：它能把模糊的指令分解为步骤，能记住对话上下文，能调用工具完成任务。但对于 ADHD 用户，这个 harness 的代价是什么？

## 收益：外部执行功能层的即时补偿

ADHD 的核心缺陷是执行功能——规划、组织、工作记忆、任务启动——这些恰好是 prefrontal cortex 负责的功能。传统工具（纸质计划、便签、闹钟）是静态的，无法适应实际行为，导致“被遗弃的规划器、应用和习惯的墓地”庞大（来源：规划循环与任务分解）。

Claude 的动态能力改变了这一点。它像一个随身秘书，接受模糊指令（如“帮我规划下周的工作”），然后自动分解、排序、提醒。这与 Goblin Tools 的 Magic ToDo 功能类似：将“清理厨房”分解为具体子任务，调节“辣度”控制粒度（来源：Goblin Tools）。Motion 更进一步，自动创建并动态调整日程，当计划被打乱时实时重建（来源：Motion）。这些工具的本质是：**规划、排序和跟踪发生在你的大脑之外，让你的大脑自由去做实际工作**（来源：规划循环与任务分解）。

Saner.AI 则针对工作记忆缺陷：它提供本地记忆存储和快速检索，减少搜索循环和标签切换（来源：Saner.AI）。对于 ADHD 用户，这意味着不用再担心“我刚才想到的那个点子去哪儿了”。

## 代价：脚手架还是拐杖？

然而，收益背后隐藏着代价。核心问题是：**AI 工具是“外部执行功能层”还是“拐杖”？**（来源：矛盾与存疑）

过度依赖可能导致能力退化。就像长期使用计算器会削弱心算能力，长期让 AI 替你规划、分解、记忆，你的执行功能可能越来越弱。这个风险在 LLM agent 侧同样存在：如果 harness 过于强大，agent 自身的规划能力可能萎缩，变得“离开脚手架就无法思考”。

曾国藩的故事提供了一个历史参照。这位晚清名臣被怀疑有典型注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书慢到“他人目下二三行，余或疾读不能终一行”。他的解决方案不是依赖外部工具，而是建立了一套个人 harness——日课十二条：黎明即起、读书不二、谨言、写日记反省。这套系统本质上是“结硬寨打呆仗”，用最笨最稳的方法抵消自己的冲动。

曾国藩的 harness 与 LLM agent 的 harness 结构同构：日课相当于定时 re-grounding，写日记相当于会话状态记录，读书不二相当于任务专注。但关键区别在于：曾国藩的 harness 是内化的、可拆卸的；而 AI 工具是外部的、可能让人上瘾的。

## 同构权衡：ADHD 与 LLM agent 的相同困境

ADHD 大脑与 LLM 的同构不仅体现在缺陷上，也体现在解法的权衡上。

在 LLM/agent 侧，单个 LLM“不足以可靠地完成多步骤任务、与业务工具交互或适应领域特定逻辑”（来源：工具使用与认知卸载）。因此，scaffolding 将 LLM 置于与记忆、工具和决策逻辑的控制循环中（来源：工具使用与认知卸载）。但问题在于：scaffolding 越强大，LLM 的“裸能力”就越少被锻炼。工程师必须决定：是让 agent 依赖外部调度，还是训练它内部规划？

在 ADHD 侧，同样的问题：AI 工具补偿了执行功能，但用户可能失去练习这些能力的机会。Goblin Tools 的任务分解很棒，但如果你从不自己分解任务，你的分解能力永远不会提升。

## 诚实的局限

必须承认，当前证据主要来自概念类比和初步实证研究，缺乏大规模随机对照试验（来源：矛盾与存疑）。长期依赖风险缺乏实证，个体差异大，且部分用户反映学习使用 AI 工具本身增加认知负荷（来源：矛盾与存疑）。

此外，超聚焦是否应被引导而非打断？部分 ADHD 用户反馈强行打断（如番茄钟）有效，而柔性引导可能不够强力（来源：矛盾与存疑）。这说明没有一刀切的方案。

## 今天就能试的行动

1. **用 Claude 做一次任务分解**：把一个模糊任务（如“整理书房”）喂给 Claude，让它分解为 5-10 个子步骤。然后尝试不依赖它，自己分解另一个任务，对比差异。

3. **测试 Goblin Tools 的“辣度”功能**：将同一任务分别用低辣度和高辣度分解，感受不同粒度对启动动力的影响。
4. **用 Motion 或类似工具做一周自动排程**：记录你的感受，注意是否出现“离开工具就不会安排时间”的迹象。

## 结论

Claude 解决了 ADHD 的执行功能缺陷，但代价是可能让你变懒。这不是工具的错，而是所有 harness 的固有矛盾：**它既是解放者，也是潜在的束缚者**。对于 ADHD 用户和 LLM agent 工程师，答案相同：把脚手架当作训练工具，而非永久拐杖。就像曾国藩的日课，最终目标是让系统内化为能力，而非永远依赖外部支撑。

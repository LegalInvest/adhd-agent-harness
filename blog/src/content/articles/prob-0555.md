# 如果把 ADHD 大脑当成生成核心，Goblin Tools 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Goblin Tools 对 ADHD 的真正价值。

你正盯着电脑，想写一封简短的邮件。打开邮箱，却发现自己开始刷社交媒体。五分钟后，你关掉浏览器，发现邮件还没写。这种“边做边忘”的体验，对ADHD人群来说是日常。你的大脑像一台高性能生成器——创意不断、联想丰富，但缺少一个可靠的调度系统来把想法变成行动。

有趣的是，大语言模型（LLM）也有同样的困境。它能在一次对话中生成精彩内容，但一旦会话结束，它不记得任何事。没有跨会话状态，没有内置的任务跟踪器。LLM需要外部脚手架——向量数据库、记忆模块、agent框架——才能执行多步骤任务。

这种结构同构并非空洞类比。明尼苏达大学的研究发现，LLM的执行功能剖面呈现“强记忆、弱控制”模式：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷（来源：工作记忆）。这恰恰是ADHD的经典神经心理模式。耶鲁大学进一步揭示，自注意力机制本身的注意力弥散是LLM工作记忆瓶颈的计算根源，与人类注意分散机制同源（来源：工作记忆）。

那么，如果把ADHD大脑当成生成核心，Goblin Tools能不能当它的无状态与外部记忆？答案是：它能，但只是脚手架，不是治愈。

## 无状态大脑与外部记忆

ADHD的工作记忆容量小，就像LLM的上下文窗口有限。你刚想起要做什么，下一秒就忘了。Goblin Tools的Magic ToDo功能，正是通过外部化任务分解来补偿这一缺陷：你输入“清理厨房”，它自动拆成“拿出洗碗海绵、擦台面、洗水槽……”等具体步骤（来源：Goblin Tools）。这相当于为你的大脑提供了一个外部记忆缓存——你不必在脑子里记住所有步骤，只需跟着列表做。

但这种外部记忆不是真正的记忆。它不训练你的工作记忆，只是绕过它。就像LLM的向量数据库，存储了对话历史，但模型本身仍然无状态。Goblin Tools是“记忆的拐杖”，而不是“记忆的肌肉”。

## 生成核心的调度层

ADHD大脑和LLM的共同问题是“高产但缺执行调度层”。孔子就是典型例子：他精力旺盛、思维跳跃，周游列国14年坐不住，对音乐能超专注到“三月不知肉味”，但对种地等俗事零耐心。他的解决方案是“克己复礼”——用外在的礼作为行为边界，每日反省（“吾日三省吾身”）。这本质上就是一个外部调度系统：礼是行为规则库，反省是定期re-grounding。

类比到LLM agent，礼相当于prompt模板和约束规则，反省相当于定时回写记忆或状态检查。孔子70岁才做到“从心所欲不逾矩”，说明他一生都在和自己的冲动做斗争——外部脚手架从未内化，只是越来越熟练地使用它。

Goblin Tools的作用类似：它提供任务分解、语气检查、时间估计等调度功能，但不会改变你的大脑。你仍然需要主动调用它。Saner.AI通过本地记忆减少搜索循环（来源：Saner.AI），Lex通过单指令触发多步骤降低启动门槛（来源：Lex），这些工具都是外部调度层，不是内置执行功能。

## 脚手架 vs 拐杖：边界在哪？

关键问题是：这些工具是帮助你成长的脚手架，还是让你永远依赖的拐杖？

脚手架的特点是：它支撑你完成当前任务，同时允许你逐步内化技能。比如，Goblin Tools的任务分解可以让你学会如何自己分解任务。但如果你每次都依赖它，从不尝试自己思考步骤，它就变成了拐杖。

ADHD的神经基础——多巴胺系统功能低下——意味着内在动机和奖赏机制本身就有缺陷（来源：多巴胺）。工具提供的即时反馈和分解步骤，确实能补偿动机缺陷，但过度依赖可能削弱你发展内在执行功能的机会。

目前缺乏严格临床验证来区分这两种效果。多数工具的效果基于用户报告（来源：矛盾与存疑）。Goblin Tools被描述为“简单有用”（来源：Goblin Tools），但它的长期影响未知。

## 同构的启示：我们都需要一个harness

ADHD大脑与LLM的同构告诉我们：问题不在生成能力，而在调度层。Goblin Tools、Saner.AI、Lex都是harness的不同形式。它们不改变核心，但让核心能输出成果。

对ADHD人群来说，这意味着：你不需要“修复”大脑，而是需要搭建自己的外部系统。就像孔子用礼，孙策用张昭张纮管行政（自己只负责打仗），你也要找到适合自己的harness。

对工程师来说，这意味着：为LLM搭建agent框架时，要意识到你正在解决与ADHD同构的问题。你的向量数据库、记忆模块、规划器，本质上和Goblin Tools的Magic ToDo是同一种东西——都是外部调度层。

## 今天就能试的行动

1. **用Goblin Tools分解一个拖延任务**：比如“整理书桌”，看它拆出的步骤是否让你更容易开始。
2. **设置一个外部记忆点**：用Saner.AI或类似工具，把临时想法快速记下，避免工作记忆过载。
3. **尝试“单指令触发”**：用Lex或自定义自动化，将常做的多步骤任务（如“发周报”）绑定到一个指令。
4. **每日反省5分钟**：像孔子那样，简单回顾今天哪些任务靠工具完成，哪些靠自己完成。

Goblin Tools不是解药，但它是你大脑的agent框架。用得好，它是脚手架；用得过度，它是拐杖。关键在于意识到：你才是那个生成核心，工具只是你的harness。

# 为什么「治好 ADHD 的分心走神或超聚焦跑偏，丢失目标」和「让 LLM 不跑飞」其实是同一道工程题？

> 长程任务里 agent 会目标漂移，需要重锚定 system prompt——同一套 harness 思路，两边都成立。

## 同一个 bug：长程任务里的目标漂移

你打开电脑准备写一份报告，两小时后发现自己在 Reddit 上刷评论，或者盯着某个无关的细节改了十七遍。你的 LLM agent 被派去“调研竞品并生成总结”，五轮工具调用后却开始在 README 里钻牛角尖，输出的不是你最初要的东西。表面症状不同，底层报错信息一样：**目标漂移**。ADHD 侧叫它“分心走神或超聚焦跑偏，丢失目标”；Agent 工程侧叫它“长程任务里 agent 会目标漂移，需要重锚定 system prompt”。这道题，两边用的是同一套解法。

## 同构：高产但缺调度层的生成核心

wiki 资料里的核心判断是：ADHD 大脑与 LLM 都是强大的生成核心，但缺乏可靠的内置执行调度层，因此需要外部脚手架来补上执行功能（来源：AI 与 ADHD 的专注力）。具体对应很清晰：

- **无状态与外部记忆**：ADHD 的工作记忆不稳定，和 LLM 没有跨会话状态一样，都依赖外部系统来维持“现在做到哪了”。
- **上下文工程**：ADHD 大脑容易被环境带偏，任务碎片化；LLM 则会因为上下文膨胀而推理退化。两边都需要主动设计“当下注意什么”。
- **表现波动**：ADHD 的注意力、执行功能在一天内大幅波动，诊断需要多次采样；LLM 的输出受温度控制，温度越高越随机，工程上必须用 trace 和多次运行来评估（来源：采样温度与表现波动）。

研究还指出，ADHD 组的超专注工作时长可达 14.5 小时，对照组只有 8.2 小时（来源：Pace Ventures Research 2026）。这说明问题不是“不能专注”，而是专注没有稳定的调度器，会被错误目标劫持。

## 工程解法：外部脚手架与重锚定

ADHD 侧的 harness 生态已经给出不少真实工具：

- **RescueTime**：自动追踪时间、阻断分心网站，直接补偿 ADHD 的“时间盲”和工作记忆不足（来源：Revolutionizing ADHD Management with AI Assistants）。
- **Focusmate**：通过虚拟身体在场（Body Doubling）提供外部问责，用社会压力帮人启动并维持任务（来源：AI Tools for Kids with ADHD: A Complete Guide for Parents...）。
- **Brain.fm / Endel**：用 AI 生成专注音乐或个性化声音环境，做“上下文工程”，降低被外界带跑的概率（来源：11 Best ADHD Productivity Apps for Fluctuating Energy；ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。
- **Goblin Tools**：把模糊任务拆成可执行的小步骤，降低启动门槛（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。

LLM/agent 侧几乎一一对应：

- **RescueTime ↔ trace / 日志**：自动记录 agent 每一步的输入输出，让“时间到底去哪了”变成可审计的轨迹。
- **Focusmate ↔ human-in-the-loop / 审查节点**：在关键步骤引入外部检查，防止 agent 在无人监督时越跑越远。
- **Brain.fm/Endel ↔ 受控推理环境**：降低上下文噪音，保持系统提示的权重。
- **确定性工作流**：工程实践强调“确定性状态转换”“计划-执行分离”以及周期性采样监控质量（来源：AI Agents in 2026: Tools, Memory, Evals, and Guardrails；Planner-Executor Agentic Framework；AI Agent Reliability and Guardrails 2026）。最简单的 agent 工作流本身就是按预定义步骤执行的确定性流程（来源：What Are Agentic Workflows? Patterns, Memory, Use Cases, and Examples）。

所以，重锚定不是“再激励一次”，而是给系统加一个外部调度层：定期把当前状态重新对齐到初始目标。

## 历史 harness：曾国藩的“日课十二条”

晚清曾国藩是个极好的案例。他年轻时极像注意力不集中型 ADHD：坐不住，爱串门、喝酒、看杀人；读书慢到“他人目下二三行，余或疾读不能终一行”；情绪剧烈，打败仗就跳水自杀；一辈子受严重银屑病困扰。他的 harness 不是“靠意志力”，而是“结硬寨打呆仗”：

- **日课十二条**：黎明即起、读书不二、谨言、写日记反省——这是一套定时重锚定的 system prompt。
- **读书不二**：一次只读一本书，不切换，对应 LLM 的 single-task focus 与上下文保护。
- **写日记反省**：每天把行为对照目标做 trace，重新校准方向。
- **结硬寨打呆仗**：不用奇谋，用确定性流程一步步推进，对应 agent 的确定性状态机。

文天祥则是另一层：他在三年囚禁中靠《正气歌》和“孔曰成仁，孟曰取义”维持目标，这相当于给 agent 写了一条顶层 mission statement——在极端上下文污染下仍不漂移。

这些 harness 对应到 LLM 工程就是：把 mission 写进不可覆盖的 system prompt，设置定期 re-grounding 步骤，把多步任务拆成单向、确定性的状态转换。

## 我的判断：重锚定不是压制，而是补一层调度

我认为两边的关键都不是“治疗分心”或“降低创造力”，而是**承认生成核心本身没有可靠的执行调度层，于是用外部结构补上这一层**。 ADHD 的创造力、LLM 的生成能力都是资产，但资产需要调度器才能兑现成目标导向行为。

不过要诚实地说清边界。wiki 的“矛盾与存疑”部分提醒我们：

- 这种同构目前仍是**功能类比**，不是已验证的神经机制事实；
- 多数 ADHD 工具的证据来自用户报告或小样本，缺乏大规模随机对照试验；
- 工具可能成为“拐杖”而非“脚手架”，存在过度依赖风险；
- ADHD 的个体差异很大，同一套 harness 对不同亚型效果可能不同。

所以 harness 的目标应该是“让自己能逐步把外部结构内化为习惯”，而不是“一辈子靠提醒活着”。对 agent 也一样，system prompt 和确定性工作流应该让系统更可靠，而不是把每个任务都变成硬编码脚本。

## 今天就能试的 4 个动作

1. **写一句“今日主任务”并设每小时重锚定**：像 system prompt 一样，把当前最重要的单一目标放在最显眼的地方，每小时问一次“我现在做的这件事，是否服务于它？”
2. **用 Focusmate 或身体加倍启动最不想做的任务**：只需要 25 分钟

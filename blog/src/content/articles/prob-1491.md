# 用 Claude 给 ADHD 补上重锚定与目标漂移，真的能像 agent 那样工作吗？

> 把 Claude 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

你刚打开电脑准备写一份报告，三分钟后却在查“宋代水力机械”。或者你是一名 Agentic Harness 工程师，发现你的 agent 在 20 步后已经把 system prompt 里“只做调研”的约束忘得一干二净，开始生成花哨的 PPT。一边是 ADHD 的“分心走神或超聚焦跑偏，丢失目标”，一边是 LLM agent 的“长程任务目标漂移”。它们的解法可以互相借鉴吗？把 Claude 当成 ADHD 的外部执行层，真的能像 agent 那样工作吗？

## 1. 同一个 bug：高产核心缺执行调度层

ADHD 大脑常被描述为“高产但缺乏可靠执行调度层的生成核心”（来源：AI 与 ADHD 的专注力）。它生成想法、联想、情绪的速度极快，但工作记忆不稳定，时间感知扭曲，容易被环境刺激带走，导致任务碎片化。使用通用聊天机器人时，这种碎片化反而加剧——“they often add another inbox, another prompt, and another place to lose the thread”（来源：上下文工程）。

LLM/agent 也面临几乎同构的问题：它是强大的生成核心，但缺乏内置的执行调度层。长程任务中，上下文膨胀会导致推理退化，控制逻辑分散在框架默认值和硬编码控制器之间，难以维持原始目标（来源：上下文工程）。模型还会产生自信但错误的输出，即“幻觉”，需要外部验证循环来校正（来源：幻觉与验证循环）。

所以两边的解法结构是相似的：不是替换核心，而是给核心套上一个 harness/脚手架——外部记忆、上下文工程、验证循环、工具接口、沙盒（来源：幻觉与验证循环）。ADHD 需要补上工作记忆、时间觉察、启动压力与身体在场；LLM agent 需要补上上下文管理、规划工件、验证步骤与目标重锚定。

## 2. 曾国藩的日课：人类 harness 的原型

晚清名臣曾国藩是一个典型的注意力不集中型 ADHD 案例：年轻时坐不住、爱串门喝酒、读书“他人目下二三行，余或疾读不能终一行”，情绪冲动，一生靠写日记骂自己来纠偏（人物案例素材：曾国藩）。他的 harness 系统就是“日课十二条”：黎明即起、读书不二、谨言、写日记反省。这些不是道德修养，而是外部执行层——每天固定时间重新锚定目标，把大目标拆成最小可执行动作，再通过日记做验证循环。他带兵“结硬寨打呆仗”，用最笨的工事抵消冲动，本质上是把战略意图固化到日常流程里。

这套系统与 LLM/agent 的 harness 同构对应：日课十二条 = 定时 re-grounding / system prompt；日记反省 = 验证循环与外部记忆；秘书/幕僚 = 外部调度器。AI 工程师给 agent 写 system prompt、加 planner、加验证步骤，其实是在做同样的事：把容易漂移的生成核心，锁进一个可检查、可重锚定的外部结构里。

## 3. Claude 实测：补的是哪一块？

回到 Claude。把它当作 ADHD 的外部执行层，它能补的最明确的一块是“语义重锚定与上下文工程”：你可以把目标、约束、退出条件、当前优先级写进一条长期 system prompt，每次会话前贴给它，让它在对话中反复把你拉回主线。你也可以让它把模糊任务拆成步骤、生成过渡提示，就像 ADHD 工具 Routinery “create a sequence of steps for a routine and walk you through them with transition prompts”一样（来源：上下文工程）。

但它不能替代其他更专业的执行层。ADHD 的“时间盲”需要自动化时间追踪，RescueTime 通过后台记录和分心阻断让用户看到“时间到底去了哪里”（来源：Revolutionizing ADHD Management with AI Assistants）；启动困难需要外部社会压力，Focusmate 通过虚拟身体在场（body doubling）和 AI 匹配伙伴来提升任务启动（来源：ADHD科技行业深度研究；AI Tools for Kids with ADHD: A Complete Guide for Parents）；状态调节可以尝试 Brain.fm 这类神经锁相音乐，它在 2026 年 44 款 ADHD 应用测试中入选最佳 9 款，但针对 ADHD 的独立临床证据仍有限（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。

Claude 的长处在于“把目标翻译成当下该注意的上下文”。当你超聚焦跑偏时，它可以问一句：“我们当前的最高优先级是什么？”；当你分心后重新打开窗口时，它能把上一段没写完的意图还原出来。这正是 agent harness 里“上下文传递 + 规划工件”的功能，但仅靠对话，它还缺少工具接口、自动化验证和长期记忆，无法独立完成一个真正的 Agentic Harness。

## 4. 边界：脚手架 vs 拐杖

最诚实的答案必须包含边界。首先，ADHD 大脑与 LLM 的“同构”只是功能类比，没有实证神经基础，不应被推广为生物学事实（来源：矛盾与存疑）。其次，AI 工具的证据普遍不足：Brain.fm 缺乏 ADHD 专项临床研究，RescueTime 和 Focusmate 也主要依赖用户报告或小样本研究，缺乏大规模随机对照试验（来源：AI 与 ADHD 的专注力）。第三，存在依赖风险：如果每次重锚定都靠 Claude，用户可能无法内化执行功能，从脚手架变成拐杖（来源：矛盾与存疑）。第四，虚拟身体在场不能等同于真实人际互动，AI 无法完全替代人类问责（来源：矛盾与存疑）。

# 如果把 ADHD 大脑当成生成核心，Otter.ai 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Otter.ai 对 ADHD 的真正价值。

## 问题：边做边忘，工作记忆的崩溃

你有没有过这样的经历：开会时灵光一现，觉得这个点子一定要记下来，但等会议结束，你已经完全想不起那个点子的细节。或者，你正在写一份报告，突然想起要查一个数据，打开浏览器后却忘了自己要查什么，反而刷起了社交媒体。这就是 ADHD 大脑的日常——工作记忆容量小，像一块随时会满溢的便签纸，新的信息一进来，旧的就掉出去了。

从认知神经科学角度看，ADHD 的核心缺陷之一就是工作记忆不足（来源：ADHD 的 AI 工具全景）。你有一个强大的生成核心——创造力、联想能力、发散思维——但执行功能（计划、组织、任务启动）薄弱，导致“高产但不可靠”。你脑子里有无数好想法，却无法稳定地把它们落地。

## 同构：ADHD 大脑与 LLM 是同一类生成核心

有趣的是，LLM（大语言模型）也有同样的困境。LLM 本身是强大的生成核心，能写出精彩的段落、完成复杂的推理，但它没有跨会话的状态——每次对话都是全新的开始，上一轮的信息如果不显式注入，模型就会“忘记”。这正是为什么 agent 工程需要外部记忆（向量库、缓存）和 harness（脚手架、护栏）来让 LLM 持续可靠地工作（来源：ADHD 的 AI 工具全景）。

ADHD 大脑与 LLM 在结构上同构：两者都是“高产但缺执行调度层的生成核心”。ADHD 患者需要外部工具来提供执行功能、记忆管理和上下文控制，就像 LLM 需要 agent harness 来提供状态管理和任务编排一样（来源：ADHD 的 AI 工具全景）。这个视角让我们能借用 agent 工程的思路来理解 ADHD 工具的价值。

## Otter.ai：作为无状态与外部记忆层

Otter.ai 是一款实时转录工具，能自动记录会议、讲座、对话，并生成文字稿和摘要。对于 ADHD 大脑来说，它解决了一个核心痛点：**你不需要在听的同时记笔记**。工作记忆的负担被转移给了 Otter——它充当了你的外部记忆层，让你可以全神贯注于当下的对话，而不必担心之后忘记关键信息。

从 agent 工程的角度看，Otter.ai 的作用类似于 LLM agent 的 **外部记忆（向量库）**。LLM 本身是无状态的，但 agent harness 通过将历史对话、知识片段存储到向量库中，在需要时检索注入，让模型能够“记住”之前的上下文。Otter.ai 做的正是这件事：它实时捕获你的听觉输入，将其转化为可检索的文字记录，让你在需要时能回看、搜索、引用。它弥补了 ADHD 大脑工作记忆容量的不足，相当于给生成核心配了一个持久化的缓存（来源：ADHD 的 AI 工具全景）。

## 脚手架还是拐杖：必须警惕的局限

虽然 Otter.ai 能显著提升 ADHD 患者的工作效率，但我们必须诚实面对一个争议：**过度依赖外部工具可能阻碍内在执行功能的发展**（来源：矛盾与存疑）。如果 Otter.ai 变成了你唯一的信息存储方式，你可能会失去主动记忆和整理信息的习惯，反而让工作记忆变得更弱。这就是“脚手架 vs 拐杖”的边界：脚手架是临时的、可撤除的支撑，而拐杖是永久依赖。

目前，Otter.ai 等工具设计为长期使用，并未提供撤除机制（来源：矛盾与存疑）。这意味着用户需要主动管理自己的依赖程度——比如逐步减少对转录稿的依赖，尝试先凭记忆复述关键点，再用 Otter 验证。否则，工具可能从“外部记忆层”退化为“认知拐杖”。

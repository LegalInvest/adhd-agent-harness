# ChatGPT 之于 ADHD，就像 self-critique 验证循环之于 LLM——但有人用错了

> self-critique 有个著名的坑:让模型自己检查自己,它经常一边道歉一边重复同样的错。工程界的修正是把「生成」和「批评」分成两个独立的通道。用 ChatGPT 给 ADHD 当验证循环,成败的分界线是同一条。

先说这个类比的工程侧。LLM 的输出错误率不低,一个诱人的方案是 self-critique:生成后让模型自查一遍再输出。实践发现它有效但有条件——**批评通道必须和生成通道解耦**:换一个角色设定、给明确的检查清单、最好换一次独立调用。让刚写完答案的同一个上下文自夸自查,批评会被生成时的思路污染,错误照单全收。

ADHD 侧的对应问题:冲动输出。没读完的邮件就回复、会上话没过脑就出口、方案没核对就提交——错误不是能力问题,是**输出通道上没有验证环节**:从「想到」到「发出」之间的那道闸,ADHD 的抑制控制天生偏弱。而靠自己「发之前再看一遍」,和被污染的 self-critique 一样不可靠:你用刚才写它的那个脑子去查它,看到的还是刚才的思路。

ChatGPT 恰好可以当那个**解耦的批评通道**——但用法分对错。

## 用对:独立通道 + 明确清单

**发件前审查**。高风险输出(重要邮件、给领导的方案、对客户的承诺)先过一遍:「用挑剔同事的视角检查这段:事实错误、语气问题、遗漏的关键信息、可能被误读的句子。」注意两个设计:**指定角色**(挑剔同事/收件人本人)让它跳出你的思路;**给清单**(四类问题)让批评结构化而不是笼统的「挺好的」。

**冲动决定的冷却审查**。「我想现在就回复拒绝这个合作」——发之前丢给它:「这是我要发的内容和背景,给我三个反对我这样回复的理由。」要求反对,不要求评价——**「给我三个反对理由」是对抗谄媚倾向最便宜的 prompt 技巧**,它强制模型进入批评通道而不是附和通道。

**分级验证**。不是所有输出都过审(那会把你拖死):日常闲聊零验证、一般工作消息自查一遍、高风险输出(不可逆的、对上的、对外的)强制过独立通道。这套分级恰好是 agent 工程的标准做法——护栏按风险分层,不是全量拦截。

## 用错:三种把验证废掉的姿势

**把它当橡皮图章**。带着「帮我看看这写得多好」的姿态去问,模型的谄媚倾向会如实配合——你拿到的不是验证,是盖章。修复见上:强制要反对理由,或直接问「哪里会被挑刺」。

**验证循环无限化**。改一版问一遍、再改再问,十轮之后还没发出——验证变成了新的拖延巢穴。对 ADHD 这是真实风险:验证的安全感会上瘾。护栏:**高风险输出最多两轮审查,第二轮后必须发出或升级给真人**(agent 工程同款:self-critique 设最大迭代次数,超限交人工)。

**验证低风险输出,裸奔高风险输出**。花十分钟让它检查一条朋友圈文案,转头把没读第二遍的报价单发给了客户。这是注意力被兴趣而非风险分配的典型 ADHD 模式——所以分级不能靠感觉,要靠清单:预先写死「什么类型必过审」,发件时对清单,不对心情。

## 边界

同构强度 B 级:self-critique/verification loop 是真实且有研究讨论的 LLM 技术(包括其自我修正的局限),ADHD 的抑制控制与冲动性有扎实文献,「ChatGPT 作为外部验证通道」是机制映射的实践,无对照研究。注意:验证循环管的是输出质量,不管情绪冲动本身——如果冲动已伤及关系或财务,那是治疗议题,不是 prompt 议题。

## 今天就能试的 3 件事

1. **写你的「必过审清单」**:三类就够(如:发给上级的、承诺资源的、拒绝别人的)。贴在显眼处。
2. **今天第一封重要邮件试一次独立审查**:挑剔同事视角 + 四类问题清单。对比它抓到了什么你没看见的。
3. **给验证设上限**:两轮规则从今天生效。第二轮结束,发出。

冲动不是性格缺陷,是输出通道缺一道闸——而这道闸,工程界已经反复验证过:不能装在生成者自己身上,要装成独立通道。你的独立通道,现在一个浏览器标签页就能开。

# Perplexity 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Perplexity：它到底补上了哪一层执行功能？

## 问题：为什么“查资料”对 ADHD 是酷刑，对 LLM 也是？

ADHD 大脑渴望信息，但讨厌“维护状态”。查资料需要：记住关键词 → 打开标签页 → 扫读 → 判断相关性 → 切换上下文 → 整合。每一步都是工作记忆的炸弹。结果往往是：打开 20 个标签页，忘了最初要找什么，或者被一篇无关文章吸走六小时——这就是超聚焦在错误方向上的代价（来源：超聚焦）。

LLM 和 Agent 也有类似“标签页地狱”。当模型需要实时数据（比如天气、股价）时，它无法自己“记住”去查；它必须通过 function calling 精确调用外部 API，把计算外包给工具，否则就会幻觉或过时。

**同构点**：两者都是“高产但缺执行调度层”的生成核心。ADHD 大脑缺的是工作记忆和任务启动的调度；LLM 缺的是实时检索和精确计算的调度。解法也同构：给核心套一个 harness——外部工具层。

## Perplexity 到底补上了哪一层？

Perplexity 不是另一个搜索引擎。它的核心设计是：**用对话式界面，把“搜索→筛选→整合”三步压缩成一步**。你问一个问题，它直接给出带引用的答案。这补上的，正是 ADHD 最缺的“外部记忆 + 即时检索”层——也就是 LLM 的 function calling 层。

- **ADHD 侧**：Perplexity 减少了“打开多个标签页”的认知负荷。你不需要记住关键词、不需要在结果页里扫读；它帮你做了决策：哪个链接相关？如何组织答案？这直接降低了工作记忆负担（来源：认知负荷）。用户反馈称“Goblin Tools 在这方面很棒”，而 Perplexity 把同样的分解逻辑用在了信息检索上（来源：Goblin Tools）。
- **LLM/Agent 侧**：Perplexity 本身就是一个“带检索的 LLM”。它用搜索 API 作为 function calling 工具，把事实查询外包给互联网。如果没有这层，LLM 只能依赖训练数据，产生幻觉或过时信息。这正是 Agentic Harness 工程的核心：agent 通过 function calling 把精确计算外包给工具。

## 但有人用错了：把 Perplexity 当“答案生成器”

Perplexity 的陷阱在于：它太顺滑了。顺滑到用户不再思考，直接复制答案。对 ADHD 来说，这恰好是“多巴胺陷阱”：快速得到答案 → 获得即时满足 → 跳过整合环节 → 下一次更依赖它。结果不是脚手架，而是拐杖——你永远不会锻炼自己的“检索-整合”肌肉。

同样，在 LLM 工程中，如果让模型直接生成所有答案而不调用工具，就会产生幻觉。function calling 的正确用法是：**只让工具做它擅长的事（精确计算/检索），模型做它擅长的事（推理/生成）**。Perplexity 的正确用法同理：**让它做检索和事实核查，但你自己做判断和整合**。

## 争议与局限

- **过度依赖**：Perplexity 是否让你更不会自己查资料？目前缺乏长期效果研究（来源：矛盾与存疑）。但类比 LLM 的 function calling：好的 harness 应该可逐步撤除——先让工具帮你找，再慢慢练习自己扫读摘要。
- **超聚焦风险**：Perplexity 的连续追问可能引发超聚焦，让你在“追问-回答”循环里失去时间感（来源：超聚焦）。需设置时间提醒。
- **个性化不足**：ADHD 异质性高，Perplexity 的通用回答可能不适合所有亚型（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Perplexity 做“信息预筛”**：查资料前，先问 Perplexity“关于 X，核心观点是什么？”，再根据引用链接深入阅读。这相当于 function calling 的“先检索再推理”。
2. **设置时间边界**：用手机计时器限制 Perplexity 使用时间（比如15分钟），避免超聚焦。类比 LLM 的 max_tokens 限制。
3. **手动整合答案**：把 Perplexity 的回答复制到空白文档，用自己的话重写一遍。这是“训练整合肌肉”的关键一步。
4. **对比多个来源**：让 Perplexity 给出不同角度的答案（比如“支持 vs 反对”），然后自己判断。这模拟了 LLM 的 ensemble 方法。

## 结语

Perplexity 之于 ADHD，就像 function calling 之于 LLM——它补上了精确检索和外部记忆层，但前提是你用它来“辅助思考”，而不是“替代思考”。脚手架 vs 拐杖的边界，在于你是否保留了主动整合的环节。记住：工具的价值，不在于它替你做了多少，而在于它让你能做更多自己擅长的事。

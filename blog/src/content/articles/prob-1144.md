# Claude 之于 ADHD，就像 human-in-the-loop 监督 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

## 问题：独自做事时，为什么你总是启动不了、坚持不下去？

想象一个场景：你坐在电脑前，有一个重要项目要推进。没有人在旁边催促，没有截止日期在眼前闪烁，你明明知道该做什么，却反复打开社交媒体、刷视频、发呆。这种“独自做事缺乏问责、容易放弃”的体验，是 ADHD 群体最熟悉的折磨。同样，在 AI Agent 工程中，一个高风险 agent（比如自动交易系统、医疗诊断助手）如果被完全放手，没有人类在回路中监督，它就会产生幻觉、偏离轨道、做出不可预测的决策。两个场景看似无关，却共享同一个底层结构：**一个高产的生成核心，缺少一个可靠的外部执行调度层。**

## 同构：ADHD 大脑与 LLM/agent 是同一类“生成核心”

ADHD 大脑的工作记忆容量有限且不稳定，容易受干扰（来源：Human-like Working Memory Interference in Large Language Models）。在没有外部结构时，ADHD 个体难以自主组织任务（来源：Working Memory Identifies Reasoning Limits in Language Models）。同样，标准 LLM 调用是无状态的，每次对话独立，模型不保留跨会话记忆（来源：The Anatomy of an AI Agent: Memory, Tools, Planning, and Execution Explained）。没有 harness 的 LLM 只是一个无状态的 API 调用（来源：Building an AI Agent Harness）。两者都像一个强大的生成引擎，但缺乏内置的调度器、记忆体、问责机制。

认知神经科学的研究进一步揭示了这种同构：前额叶-纹状体门控机制负责选择性控制信息流入工作记忆，而 Transformer 的自注意力机制在训练后发展出模仿该门控的操作（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS）。ADHD 的自动反应模式与 LLM 的自动完成机制平行，两者都在有限执行监督下生成复杂行为，且易出现虚构/幻觉等错误（来源：Automatic Minds: Cognitive Parallels Between Hypn）。这意味着，ADHD 大脑与 LLM 在认知机制上存在计算同构性。

## 解法：人在回路与身体在场

对于高风险 agent，工程实践要求“human-in-the-loop”监督——人类定期检查 agent 的输出，纠正方向，提供上下文。对于 ADHD 个体，同样的原理适用：需要一个“外部执行功能层”来提供问责、记忆和调度。这个外部层可以是 AI 工具，但必须是“脚手架”而非“拐杖”。真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

工具如 **Goblin Tools** 的 Magic ToDo 能将模糊任务分解为具体步骤，用户可调节“辣度”控制粒度（来源：12 AI Personal Assistants Built for ADHD Brains）。**Saner.AI** 提供知识回忆和本地记忆，减少搜索循环（来源：Best AI Tools for ADHD Productivity in 2026）。**Motion** 自动排程并动态调整日程，消除“下一步该做什么”的决策负担（来源：11 Best ADHD Productivity Apps for Fluctuating Energy）。这些工具都在扮演“外部调度器”的角色——就像 LLM harness 中的记忆模块、规划模块、人类审核步骤。

## 案例：张一鸣的“系统化 harness”

字节跳动创始人张一鸣具有典型的 ADHD 特质：思维极度发散，什么新东西都要研究；永远在思考组织和产品。他的专属 harness 是一套系统：用“延迟满足感”对抗即时冲动，用 OKR 管理系统提供目标与问责，用“Context not Control”原则建立文化而非集权（来源：人物案例）。这套系统本质上是“人在回路”的工程化——他通过外部制度（OKR、文化）来监督自己的注意力分配，就像工程师通过 harness 监督 agent。日课对应定时 re-grounding，秘书对应外部调度器。

## 争议与局限：脚手架还是拐杖？

AI 工具是“外部执行功能层”还是“拐杖”？目前缺乏纵向研究，过度依赖可能导致能力退化（来源：拐杖与脚手架）。此外，学习使用 AI 工具本身可能增加认知负荷（来源：矛盾与存疑）。超聚焦是否应被引导而非打断也无定论（来源：矛盾与存疑）。因此，关键在于使用方式：Claude 等 AI 应被用作“人在回路”的监督者，而非替代执行功能的黑箱。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个拖延任务**：将“整理房间”输入 Magic ToDo，调整辣度到 3，按步骤执行。
2. **设置 Motion 自动排程**：导入你的任务列表，让 AI 每天自动生成计划，并设定每日 15 分钟回顾调整。

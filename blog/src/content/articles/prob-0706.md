# Structured 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Structured：它到底补上了哪一层执行功能？

## 引言：同一个问题，两种面孔

你打开一个空白的日程应用，打算规划今天的工作。盯着时间轴，大脑却一片空白——不是不知道要做什么，而是不知道从哪开始。另一边，一个 AI agent 收到指令“帮我写一份市场分析报告”，却卡在第一步：该调用哪个 API、先查数据还是先写大纲？

这两种场景看似无关，实则共享同一个核心缺陷：**一个高产但缺执行调度层的生成核心**。ADHD 大脑与 LLM 在架构上高度同构——两者都有强大的生成能力，却在将意图转化为有序行动时频频失效。明尼苏达大学的一项系统测试发现，LLM 表现出“强记忆、弱控制”的执行功能剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这正是 ADHD 的经典神经心理模式。

## 问题：Structured 到底补上了哪一层？

Structured 是一款主打时间块规划的日程应用，近年来被许多 ADHD 社区推荐。它允许用户将一天划分为彩色时间块，并设置提醒。乍看之下，它似乎解决了时间盲和任务启动问题。但仔细审视 wiki 资料会发现一个关键警告：**Structured 的 AI 功能细节尚不明确**（来源：Structured 页）。这意味着，它可能只是一个漂亮的“时间容器”，而非真正的执行调度层。

对于 ADHD 用户，真正的瓶颈不在于“知道几点该做什么”，而在于“如何将一个模糊意图分解为可执行的步骤序列”。Goblin Tools 的 Magic ToDo 功能之所以有效，正是因为它自动将“整理房间”拆解为“捡起地板上的衣服”“擦桌子”等具体动作，降低了启动门槛（来源：Goblin Tools 页）。同样，LLM 的 function calling 之所以强大，是因为它允许 agent 将高层次的“写报告”指令映射为一系列工具调用：search_web、summarize、generate_chart。

Structured 若只提供时间块，就相当于给了 LLM 一个时间轴但没给它工具集。它补上的是“时序规划”层，却漏掉了“任务分解”和“上下文管理”层——而这两层恰恰是 ADHD 和 LLM 最需要的外部脚手架。

## 同构解法：脚手架 vs 拐杖

从同构视角出发，ADHD 大脑与 LLM 的缺陷可以一一对应：
- **工作记忆**：ADHD 的工作记忆容量保留但控制弱（易被干扰），LLM 同样有强记忆容量但弱上下文控制（来源：工作记忆页）。Saner.AI 通过本地记忆存储和快速检索减少搜索循环，相当于为 LLM 提供持久化上下文（来源：Saner.AI 页）。
- **任务启动**：ADHD 的任务启动困难对应 LLM 的“冷启动”问题。Lex 的单一指令触发多步骤执行，相当于为生成核心提供启动脚本（来源：Lex 页）。
- **时间盲**：ADHD 的时间感知缺陷与 LLM 的时序推理弱点同构。Goblin Tools 的估算功能提供外部时间线索（来源：Goblin Tools 页）。
- **身体在场效应**：Focusmate 利用 AI 匹配虚拟身体加倍伙伴，提供社会存在感降低分心阈值（来源：身体在场效应页）。这相当于为 LLM 添加一个“监督 agent”来保持任务聚焦。

但这里有一个关键边界：**脚手架促进能力发展，拐杖则导致依赖**。Otter.ai 减轻笔记负担，但过度依赖可能削弱主动记笔记的能力（来源：AI 与 ADHD 的情绪调节页）。同样，如果 Structured 只是让你被动地跟随时间块，而没有训练你主动估算任务耗时和分解步骤，它就成了拐杖。

## 证据与局限

现有证据主要来自用户报告和工具设计逻辑，缺乏大规模随机对照试验（来源：矛盾与存疑页）。例如，Goblin Tools 被评价为“简单有用”（来源：Goblin Tools 页），但个体差异导致效果不一——部分用户可能发现其分解过于机械。同样，Structured 的 AI 功能细节不明确，可能导致用户对其能力产生不切实际的期望（来源：矛盾与存疑页）。

更根本的矛盾在于：工具设计者声称是“脚手架”，但实际使用中可能沦为“拐杖”。wiki 资料明确警告“过度依赖 AI 工具可能削弱内在能力”（来源：矛盾与存疑页），但缺乏如何避免依赖的具体指导。

## 今天就能试的行动

1. **用 Structured 时，先让 Goblin Tools 分解任务**：在 Structured 中创建时间块之前，先用 Goblin Tools 的 Magic ToDo 将大任务拆成步骤，然后将这些步骤填入时间块。这相当于为 LLM 先定义好 function，再分配时间。
2. **启用 Saner.AI 作为外部记忆**：将 Structured 的笔记字段与 Saner.AI 连接（或手动复制关键信息），确保每个时间块都有上下文，减少切换成本。
3. **尝试一次 Lex 式的“单一指令”**：在 Structured 中设定一个“启动块”——比如每天第一个时间块固定为“打开 Lex，执行‘整理今日待办’指令”。这模拟了 LLM 的 function calling 入口。
4. **每周反思一次“工具依赖度”**：问自己：如果没有 Structured，我还能规划一天吗？如果答案是否定的，说明它成了拐杖。尝试每周一天不用任何工具，训练内在调度能力。

Structured 不是坏工具，但它只解决了问题的一半。真正的结构化，是让外部调度层不仅告诉你“何时做”，更告诉你“做什么”和“怎么做”。对于 ADHD 大脑和 LLM，最需要的不是更漂亮的时间轴，而是一个能分解意图、管理上下文的执行引擎。

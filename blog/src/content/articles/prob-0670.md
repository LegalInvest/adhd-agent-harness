# Saner.AI 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Saner.AI：它到底补上了哪一层执行功能？

## 问题：为什么我有了 Saner.AI，还是无法开始写报告？

你是一个 ADHD 成年人，刚订阅了 Saner.AI，满怀期待地导入所有笔记、待办和灵感。但一周后，你发现：报告依然没写，项目依然没启动，Saner.AI 只是变成了一个更智能的“回忆工具”——你依然在搜索、回忆、整理，而不是执行。

与此同时，你是一个 AI agent 工程师。你花了一周写了一个完美的 function calling 定义，让 agent 能调用计算器、天气 API、数据库。但 agent 依然在“思考”：它反复生成候选函数、检查参数、犹豫不决，就是不肯真正调用。你意识到：你给了它工具，却没给它“启动脚本”。

这两个场景共享同一个底层问题：**生成核心（ADHD 大脑 / LLM）不缺能力，缺的是执行调度层**。Saner.AI 和 function calling 都是这个调度层的一部分，但很多人只把它们当作“记忆外挂”，忽略了它们真正的价值——**降低任务启动的摩擦**。

## 同构：ADHD 大脑与 LLM 共享同一个缺陷

从 [[ADHD 大脑与 LLM 的同构]] 视角看，两者的执行功能缺陷惊人相似：

- **工作记忆**：ADHD 的工作记忆容量保留但控制弱（易被干扰），LLM 同样有强记忆容量但弱上下文控制。因此，外部记忆系统如 [[Mem]] 的“AI 第二大脑”和 [[Saner.AI]] 的增强回忆功能，本质上是为两者提供持久化上下文（来源：[[ADHD 的 AI 工具全景]]）。
- **任务启动**：ADHD 的任务启动困难对应 LLM 的“冷启动”问题。[[Goblin Tools]] 的任务分解和 [[Lex]] 的单指令多步骤执行，相当于为生成核心提供启动脚本（来源：[[任务启动]]）。
- **时间盲**：ADHD 的时间感知缺陷与 LLM 的时序推理弱点同构。AI 工具通过视觉化时间线和智能调度（如 [[Goblin Tools]] 的估算）提供外部时间线索（来源：[[ADHD 的 AI 工具全景]]）。
- **上下文工程**：ADHD 易被环境带偏，LLM 易上下文膨胀。[[Claude]] 的对话式整理和 [[Reflect]] 的结构化反思，都是主动设计“当下注意什么”的方案（来源：[[ADHD 的 AI 工具全景]]）。

## Saner.AI 的定位：知识回忆 vs 任务启动

[[Saner.AI]] 是一款 AI 驱动的任务管理器，专注于知识回忆和本地记忆，旨在减少搜索循环和标签切换（来源：[[Saner.AI]]）。它的核心价值是**减少记忆负担**——当你需要某个信息时，它能快速召回，而不是让你在 20 个标签页中翻找。

但问题在于：**知识回忆 ≠ 任务启动**。ADHD 用户最大的痛点是“知道要做什么，但就是开始不了”。Saner.AI 能帮你记住“写报告”这个任务，但它不会帮你把报告拆成“打开文档→写第一句→写第一段”这样的小步骤。它处理的是工作记忆的“存储”问题，而不是“启动”问题。

对比 [[Goblin Tools]]：它的 Magic ToDo 功能能将任何任务自动分解为小步骤，用户反馈称“将压倒性的事情变成一系列不压倒性的事情”（来源：[[Goblin Tools]]）。这正是任务启动的“启动脚本”。

## 有人用错了：把脚手架当拐杖，把工具当记忆外挂

[[ADHD 的 AI 工具全景]] 区分了“拐杖与脚手架”：AI 工具应作为脚手架促进能力发展，而非永久拐杖。[[Saner.AI]] 的定位是“知识回忆”，如果用户只依赖它来记住信息，而不主动训练任务分解和启动能力，它就变成了拐杖。

同样，在 Agent 工程中，function calling 如果只用来“记住”函数列表，而不提供“先调用哪个”的决策逻辑，agent 就会陷入“思考”循环。正确的用法是：**用 function calling 作为启动触发器**——当用户说“写报告”时，agent 自动调用“打开文档”函数，然后调用“生成大纲”函数，而不是等待用户指定每一步。

## 证据：AI 工具如何真正帮助任务启动

现有证据主要来自用户报告，但一致指向任务分解和启动脚本的有效性：

## 局限与争议

1. **过度依赖风险**：[[时间盲]] 页面指出“过度依赖可能削弱内在时间感知能力”，[[任务启动]] 页面提到“存在依赖风险”（来源：[[矛盾与存疑]]）。如果用户只靠 Saner.AI 回忆，而不练习主动回忆，可能反而削弱工作记忆。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个卡住的任务**：输入“整理房间”或“写周报”，看它如何拆成小步骤。感受启动阻力是否降低。
2. **给 Saner.AI 加一个“启动触发器”**：在 Saner.AI 中为每个项目设置一个“最小可执行步骤”（比如“打开文档写第一句”），而不是只存待办列表。
3. **模仿 Agent 的 function calling**：为你的日常任务写一个“启动脚本”——列出前三个最微小的动作，然后执行第一个。
4. **记录依赖情况**：使用工具一周后，问自己：我是否变得更依赖它来回忆信息，还是用它来启动行动？如果是前者，尝试主动回忆一次再查工具。

## 结论

Saner.AI 之于 ADHD，就像 function calling 之于 LLM——都是为生成核心提供精确执行层。但若只当记忆外挂而忽略任务启动，就沦为拐杖而非脚手架。真正的价值在于：**用工具降低启动摩擦，而不是用工具替代大脑的启动功能**。

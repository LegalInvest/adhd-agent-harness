# Saner.AI 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Saner.AI：它到底补上了哪一层执行功能？

如果你是一个 ADHD 患者，你一定经历过这样的场景：脑子里有一百个想法在同时爆炸，但就是没法挑出一个来执行。如果你是一个在做 Agentic Harness 的工程师，你一定熟悉这个画面：LLM 能写出漂亮的代码，但让它记住对话上下文、按顺序调用工具、不跑偏，你得费尽心思搭一套脚手架。

这两类困境，本质上是同一个问题。

## 同构：高产核心 vs 缺失的调度层

ADHD 大脑常被描述为一个高产的生成核心——想法丰富、联想活跃，但缺少稳定可靠的调度层来筛选、组织、启动和坚持任务（来源：ADHD 的 AI 工具全景）。LLM 同样如此：生成能力惊人，但在长程上下文保持、目标执行与工具调度上天然缺陷。

这不是比喻。最新研究用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中表现正常，但随序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突。这与 ADHD 执行功能缺陷的核心标志一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃（来源：Deficient Executive Control in Transformer Attention）。

所以，给 ADHD 大脑搭脚手架，和给 LLM 搭 Agentic Harness，是同一类工程。

## 孔子与文天祥：两千年前的 Harness

孔子身高近1米9，精力旺盛，周游列国14年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予朽木不可雕；对音乐超专注到三月不知肉味，对种地等俗事零耐心；思维跳跃，《论语》全是场景化语录，无系统著作。他的专属 harness：以“克己复礼”为核心，用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。70岁才做到从心所欲不逾矩。

文天祥21岁中状元，散尽家财招兵抗元；明知打不过还是要打，失败被俘后宁死不降，三年囚禁不投降。他的 harness：以《正气歌》为核心，用天地正气支撑自己，每天读圣贤书，用儒家的仁管理自己的生死。

这两个人的 harness 系统，与 LLM 的 Agentic Harness 结构同构：
- 日课（每日反省）↔ 定时 re-grounding（如每5轮对话重置 system prompt）
- 外在的礼/正气歌作为行为边界 ↔ function calling 的 schema 约束
- 秘书/门人 ↔ 外部调度器（如 LangChain 的 Agent Executor）

## Saner.AI：它到底补上了哪一层？

回到当下。Saner.AI 被定位为“AI 第二大脑”，主打知识回忆和本地记忆，减少搜索循环和标签切换（来源：Saner.AI）。它被描述为“专为对抗执行功能障碍、任务瘫痪和认知超载而设计的全能 ADHD 友好型 AI 生产力和知识助手”（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。

但很多人用错了。

他们把 Saner.AI 当成高级笔记工具——记了一大堆东西，然后还是不知道怎么开始做事。这就像给 LLM 接了一个知识库，但没给它 function calling 的 schema——它能回忆，但不会执行。

Saner.AI 的真正价值，不在于存储，而在于它作为“外部执行调度层”的潜力：它的通用收件箱可从邮件、文档或日历中提取待办事项，利用内部助手检查截止日期、将大型项目分解为小里程碑（来源：Saner.AI）。这相当于给 ADHD 大脑装了一个自动化的“任务拆解+启动”模块，类似 Goblin Tools 的 Magic ToDo（来源：Goblin Tools），但更强调与个人知识系统的整合。

## 脚手架 vs 拐杖：边界在哪里？

这里必须诚实指出争议：AI 工具到底是“外部执行功能层”还是“拐杖”？过度依赖可能导致能力退化（来源：矛盾与存疑）。目前缺乏长期实证，但我们可以从工程角度划一条边界：
- **脚手架**：工具帮你完成你“知道怎么做但做不动”的部分（如拆解任务、提醒启动），你仍然保留对目标的掌控和决策权。
- **拐杖**：工具替你决策，你不再思考“为什么”和“下一步”，完全交出调度权。

孔子和文天祥的 harness 是典型的脚手架：他们自己定义目标（仁、正气），用外部仪式辅助执行，但从未放弃对方向的判断。

## 今天就能试的行动

1. **把 Saner.AI 当“调度器”而非“笔记本”**：用它的通用收件箱把所有待办、想法、资料集中，然后每天早晨让 AI 帮你拆解出今天必须完成的 3 个微任务。
2. **设置“re-grounding”提醒**：像 LLM 的 system prompt 重置一样，每 90 分钟让 Saner.AI 弹窗问一次“你现在做的事，和今天的核心目标一致吗？”
3. **用 Goblin Tools 的 Magic ToDo 做“微任务分解”**：把“写报告”拆成“打开文档→写标题→列出3个要点→写第一段……”直到每一步小于 2 分钟。
4. **警惕“工具囤积”**：如果花在设置工具上的时间超过了它节省的时间，说明你正在把它变成拐杖。

## 局限与未来

同构视角目前仍是理论框架，缺乏直接的神经科学证据（来源：矛盾与存疑）。而且个体差异极大：有人需要强力打断（如番茄钟），有人需要柔性引导（如超聚焦引导）。没有万能工具。

但有一点是确定的：ADHD 大脑和 LLM 一样，需要的不是“更强的生成能力”，而是“更可靠的执行调度层”。Saner.AI 如果只被当成笔记工具，就错失了它作为脚手架的核心价值。把它用成 function calling 的 schema，你才真正理解了它要解决的问题。

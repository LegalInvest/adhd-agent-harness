# Perplexity 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Perplexity：它到底补上了哪一层执行功能？

## 问题：分心走神与超聚焦跑偏，丢失目标

ADHD 人群最熟悉的困境不是“无法专注”，而是“专注错了地方”。你打开浏览器想查一个工作术语，半小时后却发现自己正津津有味地读着某篇关于南极磷虾的科普文——不是因为你意志薄弱，而是因为你的大脑像一台没有重锚定机制的 LLM，在长程推理中不知不觉漂移了目标。

同样的问题也困扰着 AI agent 工程师：他们构建的 agent 在长程任务中会“目标漂移”——最初 system prompt 里定义的意图，随着上下文膨胀和中间步骤的干扰，逐渐被稀释、扭曲，最终输出与原始目标无关的内容。工程师们不得不设计“重锚定 system prompt”机制，在关键节点强行将 agent 拉回主线。

Perplexity 恰好站在这个交叉点上。它被许多 ADHD 用户奉为“专注神器”，也被 agent 工程师视为“实时检索增强”的范例。但本文要提出的核心观点是：**Perplexity 之于 ADHD，就像重锚定 system prompt 之于 LLM——它补上了“目标重锚定”这一层执行功能，但前提是你得正确使用它。否则，它反而会加速目标漂移。**

## 同构：高产但缺执行调度层的生成核心

ADHD 大脑与 LLM 在结构上高度同构：两者都是“高产但缺乏可靠执行调度层的生成核心”（来源：AI 与 ADHD 的专注力）。ADHD 个体拥有强大的发散思维和创意生成能力，但执行功能（工作记忆、任务启动、时间管理）常出现故障，导致“想法很多，行动困难”（来源：ADHD 大脑与 LLM 的同构）。LLM 同样如此：它能生成流畅的文本，但缺乏内置的调度与执行控制，需要外部“harness”来提供工具接口、上下文管理、安全执行等编排能力（来源：外部执行功能层）。

这种同构在目标漂移问题上表现得尤为明显。ADHD 的注意力分散并非缺陷，而是一种“注意力资源的弥散分配”——随着任务负载增大，注意力分数熵增加，导致容量限制（来源：外部执行功能层）。LLM 的自注意力机制同样存在这个问题：在长上下文下，冲突解决能力会崩溃至随机水平（来源：外部执行功能层）。两者都需要一种“重锚定”机制，在关键节点将注意力拉回原始目标。

## Perplexity 的正确用法：作为重锚定工具

Perplexity 的核心价值在于它提供了“即时检索+答案生成”的闭环，这恰好对应了重锚定 system prompt 的机制。当你意识到自己正在走神或超聚焦跑偏时，打开 Perplexity 输入原始目标关键词，它能迅速生成一个聚焦的答案，帮你“重置”上下文，重新锚定注意力。这与 agent 工程中“在关键节点注入重锚定 prompt”的做法如出一辙。

但这里有一个关键边界：**Perplexity 应该是脚手架，而不是拐杖**。脚手架是帮你暂时够到更高处的结构，用完可以拆掉；拐杖则是永久替代你本应发展的能力。如果每次分心都依赖 Perplexity 来拉回，而不训练自己识别漂移信号的能力，那么你实际上是在强化“依赖外部重锚定”的回路，而非内在执行功能。

## 误用：把 Perplexity 当无限浏览入口

然而，许多 ADHD 用户把 Perplexity 用成了“更高效的谷歌”——他们输入一个模糊问题，然后顺着相关搜索链接，一个接一个地点击下去，最终陷入比传统浏览器更深的超聚焦漩涡。Perplexity 的“相关搜索”和“后续问题”功能，如果缺乏主动控制，反而会加速目标漂移。这与 agent 工程中“不加限制的检索增强”一样危险：agent 每检索一次，上下文就膨胀一次，最终原始目标被淹没。

## 工具矩阵：其他同构方案

除了 Perplexity，还有其他工具从不同层面补上执行功能缺口：

- **Focusmate** 通过虚拟身体在场提供外部问责，对应 agent harness 中的“验证循环”——每次 session 开始和结束时的进度汇报，相当于强制重锚定（来源：Focusmate）。
- **RescueTime** 自动记录时间使用，提升时间觉察，对应 agent 中的“日志系统”——通过客观数据暴露漂移模式（来源：RescueTime）。
- **Brain.fm** 利用神经锁相技术影响大脑状态，对应 agent 中的“环境工程”——通过声音调节注意力基线（来源：Brain.fm）。但需注意，其针对 ADHD 的效果缺乏独立临床研究支持（来源：Brain.fm）。

这些工具的共同逻辑是：**给 ADHD 大脑套上外部执行功能层，就像给 LLM 搭建 harness 一样**（来源：AI 与 ADHD 的专注力）。

## 争议与局限

必须诚实指出，这一框架存在争议。首先，**AI 工具作为外部执行功能层的有效性证据不足**——多数研究仅来自用户报告和概念类比，缺乏大规模随机对照试验（来源：矛盾与存疑）。其次，**过度依赖可能削弱内在能力**——长期使用外部重锚定工具，是否会导致 ADHD 患者内在执行功能进一步退化？工具设计者声称是“脚手架”，但实际使用中可能沦为“拐杖”（来源：矛盾与存疑）。最后，**个体差异巨大**——同一工具对 ADHD 不同亚型效果迥异，但多数工具未充分讨论这一点（来源：矛盾与存疑）。

## 今天就能试的行动

1. **设置 Perplexity 使用边界**：每次打开前，在便签上写下“我要解决的问题是___”，搜索完后对照检查——这是你的“重锚定 prompt”。
2. **用 RescueTime 暴露漂移模式**：运行一周，查看“分心”分类下的时间分布，识别你最容易漂移的时段和触发因素。
3. **尝试 Focusmate 的“强制重锚定”**：预约 25 分钟 session，在开始前和结束后各花 1 分钟向伙伴复述目标——这相当于 agent 的验证循环。
4. **给 Perplexity 加“护栏”**：使用浏览器插件限制每次搜索后的“相关链接”点击次数（例如最多 2 次），防止超聚焦跑偏。

Perplexity 不是万能药，但它提供了一个绝佳的类比：它让我们看到，ADHD 大脑和 LLM 都需要一种“重锚定”机制。关键在于，你要主动设计这个机制，而不是被动被工具带着走。毕竟，最好的 harness 是让你最终不再需要它。

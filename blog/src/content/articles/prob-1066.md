# Perplexity 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Perplexity：它到底补上了哪一层执行功能？

## 问题：ADHD 与 LLM 共享的“目标漂移”困境

你打开 Perplexity 想查一个工作问题，十分钟后却在读某个冷门历史事件的维基百科。这不是你的错——ADHD 大脑和 LLM 共享同一个核心缺陷：都是高产但缺乏执行调度层的生成核心（来源：ADHD 大脑与 LLM 的同构）。ADHD 的工作记忆会“掉落”上下文，LLM 的无状态性导致每次对话都是新的开始。两者都需要外部脚手架来重锚定目标，否则就会在生成的洪流中漂移。

## 重锚定：Perplexity 被忽视的真正功能

Perplexity 不是普通的搜索引擎。它通过对话式界面和上下文记忆，天然具备“重锚定 system prompt”的能力。当你输入一个问题，它不只是返回结果，而是将你的意图编码为持续的上下文线索。这恰好对应 LLM agent 工程中的“重锚定”模式：在长程任务中，agent 会因上下文溢出而偏离原始目标，需要定期注入原始 system prompt 来校准（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

对 ADHD 用户来说，Perplexity 的“Follow-up”功能就是那个重锚定机制：每次追问都在重新声明“我还在研究这个主题”，而不是像普通搜索那样每次都是全新会话。这相当于把 ADHD 大脑的“目标意图”外化成了可操作的提示词。

## 但有人用错了：把脚手架当拐杖

多数 ADHD 用户把 Perplexity 当作更快的 Google，不断输入新问题，跳转新话题。这恰恰强化了注意力漂移——你得到的是碎片化答案，而不是持续的任务上下文。同样，LLM agent 工程师如果只把 Perplexity 当作 API 调用，而不设计重锚定循环，agent 也会在工具调用中迷失。

真正的用法是：**把 Perplexity 当作一个“外部工作记忆”**。在开始任务前，先输入一个明确的 system prompt（比如“我正在写一篇关于 ADHD 工具的文章，请只回答与执行功能相关的问题”），然后每次追问都基于同一个会话。这样 Perplexity 就成了你的上下文保持层，类似 harness 中的“模型连接器”和“上下文管理”组件（来源：Worker Agents | Harness Developer Hub）。

## 同构证据：工具如何补偿执行功能

- **RescueTime** 自动记录时间使用，弥补 ADHD 的“时间盲”和 LLM 的无状态性。它充当外部记忆，让你看到实际时间流向，类似 agent 的日志系统（来源：RescueTime）。
- **Brain.fm** 通过神经锁相技术维持专注状态，相当于为 LLM 提供一个稳定的“温度参数”环境，减少随机漂移（来源：Brain.fm）。但缺乏 ADHD 专项临床研究（来源：Brain.fm）。
- **Focusmate** 利用 AI 匹配虚拟同伴，提供实时问责，弥补任务启动困难。这对应 agent 中的“验证循环”——外部检查点确保不偏离轨道（来源：Focusmate）。

这些工具的共同模式是：**将执行功能（记忆、启动、时间感知）外化为可操作的脚手架**，而不是试图修复内在缺陷。这与 harness 工程的设计哲学完全一致：“设计围绕 AI 代理的脚手架——上下文交付、工具接口、规划工件、验证循环、记忆系统和沙箱”（来源：GitHub - ai-boost/awesome-harness-engineering）。

## 脚手架 vs 拐杖：边界在哪里？

争议在于：长期依赖外部工具是否会阻碍内在执行功能发展？（来源：矛盾与存疑）当前工具设计几乎都没有考虑撤除机制，比如 Goblin Tools 和 Saner.AI 都假设用户会持续使用。这就像给 LLM agent 一个永远不撤除的 harness——虽然可靠，但 agent 永远学不会自主调度。

我的判断是：**脚手架与拐杖的区别在于是否保留“主动重锚定”的意图**。如果你只是被动让工具替你决定下一步（如自动推荐下一个问题），那是拐杖；如果你主动使用工具来重申目标（如每次追问前默念“我还在研究 X”），那是脚手架。Perplexity 的 Follow-up 功能本身是中性的，关键在于用户是否用它来保持目标意图。

## 今天就能试的 3 个行动

1. **创建“任务会话”**：在 Perplexity 中新建一个会话，输入你的任务目标作为第一条消息（例如“帮我规划本周工作，只回答与优先级相关的问题”），然后所有追问都在此会话中完成。
2. **设置重锚定提醒**：每 15 分钟用手机闹钟或 RescueTime 的焦点会话提醒自己：“当前会话的目标是什么？”这相当于 agent 的周期性 system prompt 注入。
3. **组合使用**：用 Focusmate 开启一个 50 分钟时段，同时打开 Perplexity 的任务会话。同伴问责 + 上下文保持 = 双重重锚定。

## 局限与诚实

必须指出，ADHD 大脑与 LLM 同构的命题目前主要基于概念类比和工具案例，缺乏大规模实证（来源：矛盾与存疑）。Brain.fm 的神经锁相技术针对 ADHD 的效果尚无独立临床验证（来源：Brain.fm）。此外，个体差异显著：注意力缺陷型和多动冲动型用户对工具的反应可能不同（来源：矛盾与存疑）。

Perplexity 不是万灵药。它只是一个重锚定工具，能否发挥作用取决于你是否有意识地去使用它——就像 LLM agent 的 harness 需要工程师精心设计一样。

## 结论

Perplexity 之于 ADHD，就像重锚定 system prompt 之于 LLM：它不是让你更聪明，而是让你不偏离目标。但前提是你得把它当作脚手架，而不是拐杖。下次打开 Perplexity 时，先问自己：我是要搜索答案，还是要锚定目标？

# Routinery 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Routinery：用久了会让 ADHD 更强还是更依赖？

你有没有过这样的经历：明明计划好要做一件事，打开手机却忘了要干嘛；或者好不容易开始，做到一半又飘走，等回过神来已经刷了半小时短视频。这就是 ADHD 大脑的日常——工作记忆容量小，边做边忘，像一台没有内存的计算机，随时可能丢失上下文。

现在，有一类 AI 工具试图解决这个问题，比如 Routinery。它把日常任务拆成小步骤，一步步引导你完成，就像给大脑装了一个外部记忆体。但问题来了：用久了，你会变得更独立，还是更离不开它？

## 同构：ADHD 大脑与 LLM 的镜像

要回答这个问题，得先看清一个底层结构：ADHD 大脑与大型语言模型（LLM）在本质上高度同构。

LLM 是一个强大的生成核心，但它是无状态的——每次会话结束后，它不记得之前说过什么。为了让它在长任务中保持连贯，工程师必须给它套上“外部记忆层”，比如向量数据库或对话历史缓存。这就是 Agentic Harness 的典型做法：用外部工具弥补内部状态的缺失。

ADHD 大脑同样如此。它的智力、创造力常常爆表，但执行功能——计划、组织、工作记忆——却像瘫痪了一样。你脑子里有无数好点子，但就是没法稳定地调度它们。于是，你需要外部脚手架：提醒、清单、分解步骤。

Routinery 正是这样的脚手架。它把“早上起床”分解成“闹钟响→坐起来→下床→刷牙……”每个步骤都清晰可见，就像 LLM 的 prompt chain。你不再需要自己记住下一步，工具替你记了。

## 脚手架 vs 拐杖：边界在哪里？

但问题来了：脚手架是临时支撑，等建筑盖好就可以拆掉；拐杖却是永久依赖，没了它你就走不了路。Routinery 属于哪一种？

从设计上看，Routinery 并没有提供“撤除机制”。它鼓励你每天使用，把任务流程固定下来，但从未教你如何在没有它的情况下自行启动。这与 Goblin Tools 类似，后者通过“魔法待办”分解任务，但同样没有逐步退出的设计（来源：Goblin Tools 页面）。

反观 LLM 的 Agent 设计，好的 Harness 会在推理过程中逐步减少外部提示，让模型学会“内化”某些模式。比如，一些研究尝试让 LLM 在多次交互后压缩上下文，把关键信息合并到权重中，从而减少对外部记忆的依赖。但 ADHD 工具目前缺乏这种理念。

证据显示，过度依赖 AI 工具可能阻碍内在执行功能的发展（来源：ADHD 的 AI 工具全景）。一项针对 ADHD 儿童的研究指出，AI 工具“Best for: Ages 9 and up who struggle with task initiation and planning”，但并未提及如何撤除（来源：任务启动页面）。这暗示着，工具可能被设计为长期使用，而非过渡性支持。

## 真实证据：ADHD 侧与 LLM 侧

先说 ADHD 侧。Saner.AI 通过本地记忆减少搜索循环，帮助用户快速找回信息，直接补偿工作记忆缺陷（来源：Saner.AI 页面）。Lex 允许用户通过单一指令触发多步骤流程，降低决策负担（来源：Lex 页面）。这些工具在短期内效果显著：用户反馈称“Goblin Tools 将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools 页面）。

但长期效果呢？目前缺乏随机对照试验。现有证据多为用户评价和概念对齐，而非严谨的临床研究（来源：矛盾与存疑页面）。

再看 LLM 侧。Agent 工程中，外部记忆（如向量库）是标准做法。例如，LangChain 的 Agent 会使用检索增强生成（RAG）来保持上下文。但研究表明，过度依赖检索会削弱模型本身的推理能力——模型倾向于直接复制外部信息，而不是自己思考。这与 ADHD 患者过度依赖外部提醒后，内在启动能力下降如出一辙。

## 核心判断：Routinery 是脚手架，但可能变成拐杖

我的观点是：Routinery 这类工具本质上是脚手架，但当前的设计让它有变成拐杖的风险。关键在于“可撤除性”。真正的脚手架应该包含一个退出计划：随着用户内在执行功能的改善，工具应逐步减少干预，直到用户能够独立运行。

然而，商业工具没有动力这样做——它们希望你持续付费。这导致了一个悖论：工具越有效，用户越依赖，反而越难摆脱。

这不是否定工具的价值。对于严重工作记忆障碍的 ADHD 患者，拐杖本身就是必要的。但我们应该清楚自己在用什么，以及代价是什么。

## 诚实局限

首先，ADHD 与 LLM 的同构命题目前主要是概念类比，缺乏大规模实证支持（来源：矛盾与存疑页面）。其次，多巴胺干预的有效性存在争议，部分工具声称通过神经科学原理改善执行功能，但证据不足（来源：矛盾与存疑页面）。最后，个体差异巨大，对 A 是脚手架的工具，对 B 可能已是拐杖。

## 今天就能试的行动

1. **测试你的依赖程度**：用 Routinery 一周后，尝试一天不用它，看自己是否能完成相同任务。如果完全瘫痪，说明你可能已经过度依赖。
2. **设置撤除计划**：在使用任何 ADHD 工具时，主动设定一个“退出时间表”。例如，每月减少一次提醒频率，观察自己能否适应。
3. **类比 LLM 调优**：像训练 LLM 一样训练自己的大脑——先用外部记忆（工具）建立模式，然后逐步减少提示，让大脑“内化”流程。
4. **选择有“降级”功能的工具**：优先选择那些允许你自定义提醒频率或逐步关闭功能的工具，而不是全自动的“黑盒”。

Routinery 是一面镜子，照出了我们与 AI 的相似困境。记住：最好的脚手架，是那些你最终可以拆掉的。

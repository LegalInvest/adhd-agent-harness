# 用 Claude 给 ADHD 补上重锚定与目标漂移，是脚手架还是拐杖？

> Claude 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：分心与漂移，ADHD 和 Agent 工程师的共同敌人

如果你有 ADHD，你一定熟悉这种场景：打开文档准备写方案，十分钟后却在刷社交媒体，半小时后发现自己研究起了某个冷门历史事件的 Wikipedia 条目——你完全忘记了最初的目标。如果你在做 Agentic Harness 工程，你一定也见过这种场景：你精心设计了一个长程任务 agent，它在前几步表现完美，但几个 loop 之后，它开始生成无关代码、引用错误上下文、甚至试图修改系统 prompt——它“目标漂移”了。

这两种困境在结构上惊人地一致：一个高产的生成核心（ADHD 大脑或 LLM），配上一个不稳定的执行调度层。ADHD 的执行功能缺陷——工作记忆脆弱、任务启动困难、时间盲——与 LLM 的无状态性、上下文溢出、推理退化一一对应（来源：生成核心与调度层）。因此，用 Claude 这类 AI 工具来“重锚定”目标、对抗漂移，既是对 ADHD 的干预，也是对 Agent 的工程实践。但关键问题是：这种外部支撑是脚手架（可逐步撤除的辅助），还是拐杖（一旦离开就瘫痪的依赖）？

## 同构：ADHD 大脑与 LLM 都是“缺调度层的生成核心”

ADHD 的核心困境并非智力不足，而是执行功能失效——被描述为“大脑的驾驶座，但常常感觉方向盘后没有人”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。计划本和便签用了一周就崩溃，因为工作记忆无法承载项目上下文（来源：外部执行功能层）。

LLM 同样如此。单独使用的 LLM 拥有强大的语言生成能力，但缺乏可靠的执行调度。现代 AI 编码代理通过“复合 AI 系统架构”来弥补——包括分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩（来源：生成核心与调度层）。这些技术本质上是在 LLM 外部搭建调度层，防止“上下文膨胀和推理退化”。

ADHD 用户寻找的“AI 第二大脑”，必须在会话之间记住工作内容，否则就会陷入缺乏脚手架的困境（来源：外部执行功能层）。这与 LLM harness 工程中“用 Git 仓库存储项目上下文、通过 MCP 连接器访问外部数据”的做法完全同构（来源：外部执行功能层）。

## 真实工具：外部执行功能层的具体形态

**Brain.fm** 利用 AI 生成音乐，通过神经锁相技术影响大脑处理信息的方式，帮助用户进入专注状态（来源：Brain.fm）。这相当于为 ADHD 大脑提供了一个“音频上下文锚点”，减少外部干扰导致的漂移。但需注意，目前缺乏针对 ADHD 的独立临床研究（来源：Brain.fm）。

**Focusmate** 通过算法匹配虚拟同伴，提供实时问责，弥补任务启动和注意力维持的调度缺失（来源：Focusmate）。其核心机制是“AI-Matched Body Doubling”，即通过智能调度提高配对效率（来源：Focusmate）。这相当于为 LLM agent 配备了一个“验证循环”——同伴的监督就像 harness 中的检查点，防止 agent 偏离轨道。但效果高度依赖配对伙伴的可靠性（来源：Focusmate）。

**RescueTime** 自动记录时间使用，充当外部记忆，提升时间盲觉察（来源：RescueTime）。其 AI 核心在于自动分类和模式识别，减少手动记录的多巴胺消耗（来源：RescueTime）。这类似于 LLM harness 中的“日志系统”——自动记录 agent 的行为，便于事后分析漂移模式。但自动分类可能不够精准，需要用户调整（来源：RescueTime）。

## 脚手架还是拐杖？边界在于撤除机制

当前最站得住脚的论点是：AI 工具作为外部执行功能层，能够有效弥补 ADHD 的认知短板，其机制与 LLM 依赖编排层实现可靠输出高度一致（来源：主题综述）。但争议在于：过度依赖 AI 可能阻碍内在执行功能发展。脚手架应设计为可逐步撤除的临时支撑，而多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。

我的判断是：**这些工具本质是脚手架，但当前设计更接近拐杖**。原因有二：第一，缺乏长期研究证明脚手架可逐步撤除（来源：矛盾与存疑）；第二，工具设计未考虑“能力迁移”——用户在使用 Brain.fm 时，并未同时训练自己脱离音乐后保持专注的能力。

对于 Agent 工程师，这同样是一个警示：你的 harness 是让 agent 最终学会自主规划，还是让它永远依赖外部编排？如果 harness 设计为永久依赖，那它就是拐杖；如果包含“逐渐减少上下文压缩、增加自主决策”的机制，那它就是脚手架。

## 诚实局限：证据缺口与个体差异

必须承认，现有证据主要来自工具机制与 ADHD 理论的对齐，而非独立临床研究（来源：主题综述）。Brain.fm 的神经锁相技术理论上有效，但缺乏随机对照试验；Focusmate 的效果归因于身体在场而非 AI 本身；RescueTime 作为专注工具的证据薄弱（来源：主题综述）。此外，ADHD 亚型（注意力缺陷 vs 多动冲动）对 AI 工具反应不同，现有工具未充分个性化（来源：主题综述）。

## 今天就能试的行动

1. **用 Claude 做每日“重锚定”**：每天开始工作前，向 Claude 描述你的核心目标，然后让它每隔 30 分钟提醒你一次“当前任务是否与目标一致”。这相当于为你的大脑添加一个外部验证循环。
2. **结合 Focusmate 与 RescueTime**：预约一个 Focusmate 时段，同时开启 RescueTime 记录。事后对比同伴在场与不在场时的时间分配，识别自己的漂移模式。
3. **为你的 Agent 添加“目标检查点”**：如果你在构建 LLM agent，在每个关键步骤后插入一个验证步骤，让 agent 输出“当前目标：xxx，下一步动作：xxx，是否一致？”。这相当于为 agent 添加了一个重锚定机制。

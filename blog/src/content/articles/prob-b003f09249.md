# 为什么“药物有效”与“生活变好”之间，还隔着一整套执行功能基础设施？

> 《问题294》人工精修选题，双域证据作答。

“吃药之后，注意力好像清明了，可生活为什么还是一团乱？”这是许多 ADHD 人最真实的困惑。另一边，工程师们也面临一个几乎镜像的挫败：一个 70B 的 LLM 能写 elegant code、能洋洋洒洒输出方案，但一让它独立完成一个多步骤项目，它就开始遗忘目标、被上下文带偏、最后烂尾。

这两件事其实是同一个问题的两面：**一个强大的生成核心，缺的不是聪明，而是执行调度层。** 药物或模型参数能把“油门”修好，却不等于有了“方向盘、导航和刹车”。

## 一、大脑这边：强生成、弱调度

ADHD 大脑常被误解为“注意力不集中”或“懒”。但研究给出的图景更精确：它是**高产的生成核心，但内置执行调度层不稳定**。前额叶执行功能网络激活不足，使得计划、组织、时间管理、任务启动和抑制控制频频掉线（来源：ADHD × AI 的科学与研究前沿）。

更反直觉的是，ADHD 并非单纯“记忆差”。一些研究指出，ADHD 在工作记忆容量上甚至超过常人，但**认知灵活性与注意控制存在核心缺陷**：无法灵活切换任务集，也无法抑制自动化反应（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这种“强记忆、弱控制”的模式，解释了一个常见体验：脑子里有无数想法，但一坐到电脑前就不知道先点哪个按钮；项目信息全记得，就是启动不了下一步。

这也被称为“时间盲”——对时间流逝的感知异常，以及对任务上下文丢失的高度敏感。ADHD 人真正缺的不是知识，而是让“下一步行动显而易见”的外部结构（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。

## 二、LLM 这边：同样的生成—调度缺口

把 LLM 放在同一张图里，事情立刻变得清楚。LLM 本身是一个强大的生成核心，但它没有持续状态、没有内置任务调度器、没有自主优先级管理。论文中用经典的 Stroop 冲突任务测试 Transformer，发现短上下文里模型表现正常，但序列一长，在冲突试次上却骤降到随机水平——它无法抑制优势反应，无法解决认知冲突（来源：ADHD 大脑与 LLM 的同构）。这与 ADHD 执行功能障碍的核心标志几乎一一对应。

在工程上，这催生了“agent scaffolding”：把 LLM 放进一个由记忆、工具和决策逻辑组成的控制循环中，让它能够推理、规划并执行复杂目标（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。ReAct、Chain-of-Thought 等脚手架，本质上都是在做同一件事：**减少核心模型的决策负担、保留上下文、让下一步行动显而易见。**

所以 ADHD 大脑与 LLM 并非诗意类比，而是同一类系统：一个高光但缺调度层的生成核心，必须靠外部 harness 才能稳定交付。

## 三、harness 不是意志力，是系统

解决之道不是“更努力”，而是“更系统”。历史上许多 ADHD 特质的强者，都自觉不自觉地构建了外部 harness。

毛泽东的案例极具代表性：他从小思维跳跃、冲动敢赌、一辈子在移动与阅读中切换。但他用一套制度化的外部回路来约束和调度自己：**调查研究**作为信息接地（没有调查就没有发言权），**批评与自我批评**和**民主生活会**作为外部反馈与反思循环，**民主集中制**和**党指挥枪**作为集体决策与组织执行系统。这些不是道德自律，而是把“易冲动、高发散”的生成核心，套进一个持续 re-grounding、带人类反馈（human-in-the-loop）的调度架构里。

张一鸣则展示了现代知识工作者的 harness：他极度发散、什么新东西都要研究，但用**OKR**设定清晰目标，用**Context not Control**让组织替他分担调度，用**延迟满足感**和**系统/文化**替代个人意志驱动。他的做法与 LLM 的 agent harness 同构：OKR 是目标函数，Context 是提示工程与上下文管理，组织系统是外部工具与决策循环。

两个案例共同说明：harness 不是拐杖，而是把“生成核心”接入现实世界的接口层。

## 四、AI 工具作为外部执行功能层

今天的 ADHD 工具，正在做的正是这些 harness 的轻量化版本。

**Goblin Tools** 的 Magic ToDo 把“清理厨房”这类模糊任务分解为具体、可执行的子任务，用户还能调节“辣度”来控制粒度（来源：Goblin Tools）。这对应 LLM 的 task decomposition：把宏大 prompt 拆成可验证的步骤。

**Saner.AI** 专注于知识回忆和本地记忆，减少搜索循环和标签切换，被描述为“对抗执行功能障碍、任务瘫痪和认知超载的全能 ADHD 友好型 AI 生产力和知识助手”（来源：Saner.AI）。这对应 LLM 的 memory 与 RAG 层：没有外部记忆，核心再多知识也会丢失上下文。

**Motion** 则自动排程、动态调整日程，当任务面临延期风险时提前数天或数周警告（来源：Motion）。这对应 LLM 的 scheduler/planner：把目标翻译成时间线，并在环境变化时重新规划。

三者组合，就是给生成核心套上的“执行功能层”：任务分解、上下文保持、时间调度。它们让“下一步”变得可见，让启动阻力下降。

## 五、核心判断：从“有效”到“变好”，需要一次工程化改造

所以，**药物有效**与**生活变好**之间的缺口，不是“药效不够”，而是药物没有改变系统的架构。兴奋剂可以提升多巴胺与前额叶功能，但无法替你建立一套外部执行基础设施。同理，**LLM 基座再强**，也不等于 agent 能交付任务；它需要的是循环、记忆、工具、反思、人在回路。

我的判断是：ADHD 的改善路径应该被理解为一种“个人 agent 工程”——你既是生成核心，也是架构师。你要为自己设计 harness：任务分解、外部记忆、自动调度、人类反馈、周期性 re-grounding。这不是把工具当拐杖，而是把工具当脚手架，让大脑慢慢在结构化的环境中学习调度。

但也要诚实面对局限。wiki 的“矛盾与存疑”部分明确指出：**同构性仍然是一种理论类比，并非经过验证的神经科学事实**；许多工具（如 Motion）缺乏独立临床验证；过度依赖 AI 工具可能让执行功能进一步退化（来源：全局矛盾与存疑）。因此，harness 的设计必须保留一个“可撤除”的边界：工具是为了降低启动门槛，而不是永久替代你做出选择。

## 六、今天就能试的四条行动

1. **外化第一步**：下次面对一个模糊任务时，用 Goblin Tools 的 Magic ToDo 把它拆成 3–5 个具体动作。只解决“启动”这一件事。

2. **给上下文找个家**：选一个外部记忆工具（如 Saner.AI 或一个你能坚持的第二大脑），把项目背景、 deadline、当前阻塞点写下来，让“工作记忆”不再靠大脑硬撑。

3. **让日程替你决策**：如果你每天花 20 分钟纠结“先做什么”，尝试 Motion 这类自动排程工具，把优先级判断外包给系统，减少决策消耗。

4. **每周问一次：这是脚手架还是拐杖？** 回顾工具是否让你更敢自己启动、更能复盘；如果只是离了工具就完全瘫痪，那就需要收紧边界，加入更多人工反馈与 offline 练习。

对工程师读者，第四条同样成立：你设计的 agent 不应该是“让 LLM 更忙”，而是让 LLM 处于一个目标清晰、记忆可及、工具可用、循环可反思的 harness 中。ADHD 人与 LLM 都需要的，不是更多的聪明，而是更可靠的调度层。

# 如果把 ADHD 大脑当成生成核心，Claude 能不能当它的工具使用与认知卸载？

> 用 agent 工程的思路重新理解 Claude 对 ADHD 的真正价值。

## 01 问题：为什么想法很多，却执行不了？

很多 ADHD 患者都有过这种体验：大脑一刻不停地冒点子，但计划本用了一周就崩溃，便签贴满屏幕却没人真正启动。流行的描述是：执行功能就像「大脑的驾驶座」，而 ADHD 经常让人觉得「方向盘后没有人」（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

这不仅是「意志力」问题。/wiki 资料指出的核心图景更精确：ADHD 大脑更像一个高产的「生成核心」，联想活跃、发散性强，但缺少稳定可靠的调度层来筛选、排序、启动并坚持任务。工作记忆容量未必差，但自上而下的控制与调节能力存在缺陷，呈现「强记忆、弱控制」的认知剖面（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

有趣的是，大语言模型（LLM）也呈现类似结构：它擅长生成文本，但在长程上下文、目标保持与执行调度上天然脆弱。研究显示，Transformer 自注意力机制在长上下文下冲突解决能力会崩溃至随机水平（来源：Deficient Executive Control in Transformer Attention），且随着任务负载增大，注意力分数熵增加，导致容量限制（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。

所以，ADHD 大脑与 LLM 可以被视为同一类系统：底层都是高产出的生成核心，但都需要一个外部「执行调度层」——也就是 harness/脚手架——把生成能力约束到目标轨道上（来源：ADHD 的 AI 工具全景）。

## 02 同构：harness 工程的两侧证据

Agent 工程里有一个常见做法：把 LLM 当作生成核心，再通过 function calling 把精确计算、状态维护、重复执行外包给外部工具。这个思路在 ADHD 侧也有对应版本。

/wiki 资料把 ADHD 的 AI 工具分成四类 harness 功能：

1. **任务拆解与启动**：Goblin Tools 的 Magic ToDo 把「清理厨房」这种模糊任务拆成可执行的子步骤；Lex 则用「单一指令触发多步骤任务序列」，降低启动时的认知负荷（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。
2. **外部记忆与知识管理**：Saner.AI、Mem 被定位为「AI 第二大脑」，通过本地记忆与知识回忆，补偿 ADHD 工作记忆不稳定（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。
3. **对话式执行与结构化推理**：ChatGPT 被成人 ADHD 用户当作「外部执行功能」工具；Claude 则以代码编写和结构化推理见长，帮助整理待办与简化任务（来源：ADHD 的 AI 工具全景）。
4. **捕获与反思**：Otter.ai 实时转录会议与灵感，Reflect 通过结构化提问帮助 ADHD 个体在自发反思困难中提炼经验（来源：ADHD 的 AI 工具全景）。

在 LLM/agent 侧，harness 工程被定义为「围绕 AI 代理的脚手架——上下文交付、工具接口、规划工件、验证循环、记忆系统和沙箱」（来源：GitHub - ai-boost/awesome-harness-engineering）。例如，现代编码代理会通过「工作负载专用模型路由、分离规划与执行的双代理架构、自适应上下文压缩」来防止上下文膨胀和推理退化（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

两侧共享同一结构：生成核心留在里面，精确、重复、状态维护类任务被外包到外部系统。

## 03 Claude 在这个 harness 里能做什么？

回到标题问题：Claude 能不能当 ADHD 大脑的工具使用与认知卸载？

答案是：能，但它不是万能的「调度层」，更像是一个擅长结构化推理和代码的「外部函数」。Claude 在 ADHD  harness 中的合理角色包括：

- **把思维碎片结构化**：你倾倒一串混乱想法，Claude 帮你整理成项目、下一步、依赖关系。这对应 agent 里的「规划工件」生成。
- **维护状态与上下文**：把当前项目背景、目标、约束一次性写进 prompt，Claude 能在对话中替你维持这些状态，减少你大脑里「要记住太多事情」的负担。
- **生成可复用脚手架**：如果你会一点代码，可以让 Claude 写脚本、自动化流程、模板化文档，把重复劳动外化出去。

但注意：Claude 无法替代 Goblin Tools 的任务分解、Saner.AI 的持久记忆、Otter.ai 的实时捕获，也无法自动打断你的拖延或超聚焦。它只能在你主动调用时，作为 harness 里的一个工具函数工作。这和 agent 通过 function calling 外包精确计算的逻辑完全一致：LLM 不会自己突然去调用计算器，ADHD 大脑也不会因为装了 Claude 就自动获得执行功能。

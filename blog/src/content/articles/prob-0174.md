# Reflect 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Reflect：用久了会让 ADHD 更强还是更依赖？

## 问题：ADHD 的反思困境，也是 LLM 的工程短板

ADHD 群体有一个隐秘的痛点：不是不想反思，而是「反思」这件事本身对执行功能要求太高。自发反思需要工作记忆同时维持事件、情绪和因果链，而 ADHD 的工作记忆容量本就捉襟见肘——坐下后“真的感觉不到45分钟已经过去”（来源：Revolutionizing ADHD Management with AI Assistants），更别提事后复盘了。

同样的问题发生在 LLM 身上。一个无状态的大模型，在单次对话中能生成高质量内容，但一旦跨会话，它没有内置机制去回顾昨天的输出、修正今天的错误。工程师必须为它搭 harness：向量库存记忆、人在回路设检查点、护栏防止循环（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。

Reflect 恰好站在这个交叉点上。它通过结构化提问与模板，帮助 ADHD 个体在自发反思困难中提炼经验（来源：ADHD 的 AI 工具全景）。但问题是：这种帮助，是让 ADHD 大脑学会自己走路，还是让它永远扶着护栏？

## 同构：ADHD 大脑与 LLM 共享的「外部记忆」解法

先看两组数据。超专注工作时长：ADHD 组 14.5 小时 vs 对照组 8.2 小时（来源：Pace Ventures Research 2026）。这说明 ADHD 大脑不是产能低，而是调度差。同样，LLM 的生成能力惊人，但直接输出可能偏离目标、陷入循环或产生危险行为（来源：人在回路与身体在场）。

两边的解法在结构上高度同构：

| ADHD 侧 | LLM/Agent 侧 |
|---------|--------------|
| 工作记忆容量小 → 靠外部笔记/工具存信息 | LLM 无跨会话状态 → 靠向量库/外部记忆续命 |
| 任务启动困难 → 靠身体在场/小步骤触发 | 生成偏离目标 → 靠 human-in-the-loop 监督/护栏 |
| 反思困难 → 靠结构化提问模板 | 输出缺乏自我修正 → 靠 harness 中的检查点 |

以晚清名臣彭玉麟为例。他一生冲动敢杀李鸿章的侄子，却靠一套严格的自律系统管住自己：「三不要」原则（不要官、不要钱、不要命）作为行为边界，每天画梅花磨练心性，严格军纪与士兵同甘共苦（来源：彭玉麟案例）。这套系统本质上就是一个外部调度层：
- **日课画梅 ↔ 定时 re-grounding**：每天固定动作把注意力拉回正轨，类似 LLM harness 中的定期状态重置。
- **三不要原则 ↔ 护栏**：用清晰规则约束冲动，类似 token 预算和工具调用次数上限。
- **与士兵同甘共苦 ↔ 人在回路**：外部监督提供即时反馈，防止决策脱离现实。

彭玉麟不是靠天赋成为湘军水师创始人的，他是靠一套精心设计的 harness 把高冲动、高能量的生成核心约束到目标轨道上。

## 判断：Reflect 是脚手架还是拐杖？边界在于「内化」

这里必须诚实：wiki 资料中明确指出了矛盾——AI 工具是“外部执行功能层”还是“拐杖”？过度依赖可能导致能力退化（来源：矛盾与存疑）。目前缺乏纵向研究，但我们可以从工程逻辑推导一个判断标准。

脚手架 vs 拐杖的边界，不在于工具本身，而在于使用方式：

- **脚手架**：Reflect 的模板让你学会「如何问自己问题」，然后你逐渐脱离模板也能做类似的结构化反思。这就像 LLM harness 中的 human-in-the-loop 监督——工程师不是永远手动干预，而是通过护栏让模型学会在约束下自主决策。
- **拐杖**：你每次反思都必须打开 Reflect，离开它就一片空白。这就像 LLM 每次推理都依赖外部向量库检索，没有内化任何模式。

一个可操作的检验标准：**当你面对一个未预料到的小挫折时，你是下意识地打开 Reflect 走一遍流程，还是能在心里默念「发生了什么？学到了什么？下一步？」** 前者是拐杖，后者是脚手架。

## 局限：同构是类比，个体差异是变量

必须指出，ADHD 大脑与 LLM 的同构目前是理论框架，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。此外，ADHD 个体差异极大——有人反映强行打断（如番茄钟）有效，有人则必须靠柔性引导（来源：矛盾与存疑）。Reflect 的结构化提问对某些人可能是救命稻草，对另一些人则可能增加认知负荷（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Reflect 做一次「微复盘」**：设定一个 5 分钟定时器，回答三个问题——刚才的任务中，哪个步骤最卡？卡的原因是什么？如果重来一次，我会怎么调整？
2. **对比有模板 vs 无模板**：今天一次用 Reflect 模板复盘，下一次只凭记忆复盘。记录两种方式下你记住的细节数量和质量。连续一周，看是否有内化迹象。
3. **为 LLM 搭一个最小 harness**：如果你同时做 AI 工程，可以尝试给 LLM 加一个「反思检查点」——在生成长文本后，让模型先输出「自我评估：这段内容是否偏离了初始目标？」，再继续下一步。这本质上是 Reflect 的工程等价物。
4. **观察自己的「反思启动阈值」**：下次当你意识到需要反思但迟迟不动时，记录下当时的情绪和外部环境。这个阈值数据比任何工具都更重要——它告诉你什么时候该用 Reflect，什么时候该直接用纸笔。

# Claude 之于 ADHD，就像 self-critique 验证循环 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

## 问题：为什么 Claude 越用越“想当然”？

你打开 Claude，甩给它一个模糊的任务：“帮我写个计划”。它迅速生成了一份漂亮的大纲，你点头说“好”，然后关掉窗口——三天后，你发现自己根本没动，甚至忘了大纲写的是什么。

这不是你的错，也不是 Claude 的错。这是**同构陷阱**：ADHD 大脑和 LLM 本质上都是“高产但缺乏可靠执行调度层的生成核心”（来源：[[ADHD 大脑与 LLM 的同构]]）。你冲动下结论、想当然，LLM 自信地编造（幻觉）——两者都缺一个**验证循环**来兜底。

## 同构视角：两个“生成核心”的镜像

**ADHD 侧**：你的大脑默认模式网络过度活跃，想法像爆米花一样往外蹦，但前额叶的执行控制却时常掉线。研究指出，ADHD 呈现“强记忆、弱控制”的认知剖面：工作记忆容量可能正常甚至超常，但认知灵活性和注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。传统工具（计划本、便签）用了一周就崩溃，因为它们假设你能记住昨天的问题并执行计划，但你实际做不到（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。

**LLM/Agent 侧**：LLM 本身是无状态、仅生成文本的核心，单独使用时缺乏可靠的执行调度。现代 AI 编码代理通过“复合 AI 系统架构”来弥补：分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩——这些本质上是在 LLM 外部搭建调度层，防止“上下文膨胀和推理退化”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。实验证明，Transformer 自注意力机制缺乏人类特有的执行控制机制，在长上下文下冲突解决能力崩溃至随机水平——这正是 ADHD 执行功能障碍的核心神经机制（来源：Deficient Executive Control in Transformer Attention）。

**核心结论**：ADHD 大脑和 LLM 是同一类“生成核心”，缺的都是同一个东西——**外部验证循环**。在 LLM 工程里，这叫 self-critique 或 harness；在 ADHD 管理里，这叫“外部执行功能层”（来源：[[外部执行功能层]]）。

## 有人用错了：当 Claude 成为“拐杖”

问题出在这里：很多人把 Claude 当作“替我想”的工具，让它直接生成答案、计划、文案，然后照单全收。这恰恰是**强化了冲动下结论的缺陷**——你跳过自己的验证步骤，把 Claude 的幻觉当真理。

这正是“脚手架 vs 拐杖”的边界（来源：[[拐杖与脚手架]]）。外部脚手架应该帮你**建立验证习惯**，而不是替你完成思考。如果 Claude 直接给出答案，你只是把“想当然”从自己大脑转移到了 AI 大脑——同构问题依然存在。

## 正确的用法：Claude 作为 self-critique 验证循环

**LLM 工程教了我们一课**：好的 harness 不是让 LLM 一次输出最终结果，而是让它**先输出，再批判，再修正**。例如，在 Agentic Harness 中，规划与执行分离，一个 agent 制定计划，另一个 agent 验证并调整（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这就是 self-critique 循环。

**对 ADHD 用户的启示**：不要问 Claude “帮我写个计划”，而是问它“我写的这个计划，有什么漏洞？”或“请批判我的想法，列出三个我没想到的问题”。

**工具示例**：Goblin Tools 的 Magic ToDo 功能能自动将任务分解为小步骤（来源：AI Tools for Kids with ADHD: A Complete Guide for Parents...），但它的真正价值不是“替你分解”，而是**让你看到分解后的步骤，然后你自己判断是否合理**。Saner.AI 通过知识回忆减少搜索循环（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar），但它的核心是帮你**找回信息后自己验证**。Motion 自动排程（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog），但你需要**检查日程是否合理，手动调整**。

**关键判断**：Claude 补上的不是“生成能力”，而是**外部批判视角**——它帮你看到自己忽略的细节、矛盾、漏洞，从而打断冲动下结论的循环。这才是它作为“外部执行功能层”的真正角色。

## 争议与局限：别指望它“治愈”ADHD

必须诚实指出，这一框架存在局限：

- **证据不足**：AI 工具作为外部执行功能层的有效性主要来自用户报告和概念类比，缺乏大规模对照实验（来源：[[矛盾与存疑]]）。例如，Goblin Tools 被评价为“简单有用”，但没有独立 RCT 证明它比传统清单更有效。
- **过度依赖风险**：如果长期依赖 Claude 做批判，你可能会失去自我批判的能力（来源：[[时间盲]]）。工具设计者声称是“脚手架”，但实际使用中可能沦为“拐杖”。
- **个体差异**：ADHD 的异质性（注意力主导 vs. 冲动主导）可能影响同构的普适性（来源：[[矛盾与存疑]]）。对某些人，Claude 的批判可能过于温和或过于苛刻。
- **神经机制对应**：LLM 的“注意力机制”与 ADHD 的注意控制缺陷是否真正同构？目前仅是功能类比，缺乏神经生物学层面的直接证据（来源：[[ADHD × AI 的科学与研究前沿]]）。

## 今天就能试的行动

1. **反向提问**：下次用 Claude 时，不要问“怎么做”，而是先自己写一个粗糙的版本，然后问 Claude：“请批判这个版本，列出三个漏洞。” 对比 Claude 的批判和你自己的批判，看看你漏掉了什么。

2. **分解后验证**：用 Goblin Tools 或类似工具分解任务后，**手动检查每一步**：这一步真的必要吗？顺序对吗？如果发现不合理，修改它，而不是全盘接受。

3. **设置“批判时间”**：每天花 5 分钟，用 Claude 回顾你当天做的三个决定，问它：“这些决定有什么潜在问题？” 记录下 Claude 指出的问题，第二天主动避免。

4. **工具组合**：将 Saner.AI 用于信息检索（快速找回笔记），但用 Claude 做信息验证（“这段笔记的逻辑对吗？有没有遗漏关键点？”）。分离“检索”和“验证”环节。

记住：Claude 是你的**批判搭档**，不是你的大脑替代品。你依然是那个“高产但缺调度层”的生成核心——而它的任务，是帮你把冲动下结论变成**先验证、再行动**。这才是同构视角的真正力量。

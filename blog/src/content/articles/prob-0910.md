# Forest 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Forest：它到底补上了哪一层执行功能？

你打开 Forest 的「专注模式」，设定 25 分钟，然后盯着手机屏幕——任务还没开始，时间却在流逝。这不是你的错，是你的大脑在执行调度层罢工了。

这种「启动困难」对 ADHD 人群来说就像 LLM 面对一个模糊的 prompt：模型能生成精彩内容，但不知道从哪开始、怎么分解。ADHD 大脑与 LLM 在结构上同构——都是“高产但缺乏可靠执行调度层的生成核心”（来源：ADHD 大脑与 LLM 的同构）。因此，Forest 之于 ADHD，就像 planner-executor 任务分解之于 LLM agent——但有人用错了。

## 同构：两个生成核心的同一困境

ADHD 一侧，实证研究表明“工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷”（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这意味着 ADHD 大脑不缺创意和想法，缺的是把想法拆成可执行步骤、按顺序启动的能力。这跟 LLM 一模一样：模型本身是强大的生成核心，但缺乏可靠的内置调度与执行控制，所以构建 agent 时需要“agent harness”——“包裹 LLM 或 AI agent 的软件基础设施，处理模型本身之外的一切”（来源：What is an agent harness in the context of large-language models?）。

这个 harness 的核心就是 planner-executor 循环：planner 把长任务分解成子目标，executor 逐个执行，然后反馈给 planner 调整。ADHD 大脑需要的正是这样一个外部调度层：把“写一篇论文”分解成“打开文档→写第一段→休息5分钟”，并自动触发下一个动作。

## 工具实测：它们补上了哪一层？

当前主流 ADHD 时间管理工具，本质上都在扮演这个外部调度层，但补的层次不同：

- **Motion** 是 planner 的自动化版本。它“自动根据任务、会议和截止日期创建并动态调整每日计划”（来源：Motion），持续评估优先级和可用时间，实时重建日程。对 ADHD 用户来说，它消除了“下一步该做什么”的决策负担（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。这相当于给 LLM 一个自动规划器，但用户只需要输入任务——planner 的工作全由 AI 代劳。

- **Reclaim.ai** 更像 executor 的保护层。它“通过创建深度工作、习惯和专注时间的智能块，自动防御会议侵占”（来源：Reclaim.ai），帮助用户维持日程稳定性。它不帮你规划做什么，而是保护你正在做的事不被中断——这对应 executor 执行时的上下文保护。

- **Tiimo** 则直接补偿时间盲。它“将时间转化为可见元素”（来源：Tiimo），通过视觉化时间线让抽象的时间流逝变得具体。这相当于为 LLM 提供时间戳或进度条——让生成核心感知到时间约束。

- **Todoist** 的智能优先级排序（来源：Todoist）充当外部工作记忆，类似向量数据库为 LLM 提供持久上下文。它记住你该做什么，在你忘记时提醒。

这些工具都有效，但有一个关键区别：它们是替你执行调度，还是帮你学会调度？

## 脚手架 vs 拐杖：边界在哪里？

wiki 资料明确指出一个矛盾：“AI 工具可能成为‘永久拐杖’，削弱用户内在的时间感知和规划能力……工具设计者声称是‘脚手架’，但实际使用中可能沦为‘拐杖’。”（来源：矛盾与存疑）

脚手架（scaffolding）是临时支撑，最终要撤掉；拐杖（crutch）是永久替代。当 Motion 替你规划好一切，你只需要执行时，你实际上没有训练自己的规划能力。长期依赖，你的大脑可能萎缩掉本可发展的 planner 回路。

神经可塑性研究告诉我们，“训练诱导的抑制控制脑可塑性表明，基于神经可塑性的训练方案有助于正常化抑制控制能力及其底层脑网络”（来源：神经可塑性）。也就是说，大脑是可以重塑的——但前提是你主动参与训练，而不是被动接受替代。

因此，我的核心判断是：**工具应该补上 executor 层，而不是 planner 层**。ADHD 大脑的 planner（创意生成）其实很强，弱的是 executor（启动、切换、持续）。Forest 的番茄钟之所以经典，正是因为它只提供 executor 的计时和中断机制，而把 planner（决定做什么）留给你自己。但很多人用 Forest 时，先花半小时在 Todoist 里排计划，然后 Forest 一响又不知道做什么——这等于用工具替代了 executor 的启动功能，却没训练它。

## 如何正确使用：给两类人的行动建议

**对 ADHD 读者：**
1. **用工具补 executor，不补 planner**：让 Motion 帮你排日程，但每天花5分钟手动确认计划，而不是全盘接受。这相当于让 planner 做最终决策，保留你的规划能力。
2. **用 Tiimo 训练时间感知**：每天看视觉时间线时，先自己估计一下过去的时间，再核对工具显示。这类似给 LLM 加一个时间戳校准器。
3. **刻意练习任务分解**：用 Goblin Tools 的“任务分解”功能时，先自己尝试拆解，再对比 AI 的版本。这相当于训练你自己的 planner。
4. **设置“无工具时段”**：每周选一天，不用任何 AI 工具，完全靠手动规划。这能防止依赖，并让你察觉自己的进步。

**对 Agentic Harness 工程师：**
1. **planner 层要可干预**：不要完全自动化规划，留给用户手动调整的接口。ADHD 用户需要“参与感”来训练自己的 planner。
2. **executor 层要透明**：工具的执行决策（如为什么重新安排这个任务）应该可视化，帮助用户理解调度逻辑，而不是黑箱操作。
3. **设计“脚手架退出”路径**：随着用户使用，逐步减少自动干预，比如从全自动规划过渡到半自动建议。这符合神经可塑性的训练原则。

## 局限与争议

必须诚实指出，上述建议的证据基础仍薄弱。所有工具的“有效性证据主要来自用户报告和概念类比，缺乏大规模对照实验”（来源：矛盾与存疑）。同构理论本身“仍属理论类比，神经科学和计算机科学的交叉验证不足”（来源：AI 与 ADHD 的时间管理）。此外，个体差异巨大：ADHD 亚型（注意力缺陷 vs. 多动冲动）对工具响应不同（来源同上），Motion 的初始设置对执行功能较弱的用户本身就是一个挑战（来源：Motion）。

但正因如此，我们更需要一个清晰的框架来指导使用。同构视角的价值不在于证明，而在于提供可测试的假设：如果 ADHD 大脑和 LLM 共享同一类缺陷，那么对 LLM 有效的 harness 设计原则（如 planner-executor 分离、上下文保护、渐进式自主），很可能也对 ADHD 有效。这值得用更严谨的研究去验证。

最后，记住：工具是脚手架，不是拐杖。你的大脑拥有神经可塑性，它有能力学会调度——但前提是你给它练习的机会。

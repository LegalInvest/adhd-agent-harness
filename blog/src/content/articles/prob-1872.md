# ADHD 创业者如何用「给生成核心套 harness」的思路应对把想法落地执行里的任务启动困难、不会拆解？

> 把 LLM 工程里的「agent 用 planner-executor 循环分解长任务」迁移到ADHD 创业者的把想法落地执行场景，双域证据作答。

## 问题：想法太多，行动太少

ADHD 创业者最典型的困境不是缺乏创意，而是想法在脑子里疯狂生成，却无法落地。任务启动困难——面对一个宏大目标，大脑像死机一样拒绝开始；不会拆解——知道要做什么，但不知道第一步该是什么。这种“高产但缺执行调度层”的状态，与当前 LLM agent 工程面临的核心问题惊人地同构。

## 同构诊断：生成核心 vs 调度层

ADHD 大脑被描述为“高产但缺乏可靠执行调度层的生成核心”（来源：[[ADHD 大脑与 LLM 的同构]]）。它擅长联想、创意、超聚焦，但在计划、组织、工作记忆、冲动抑制上存在缺陷（来源：[[执行功能]]）。这与 LLM 的处境一致：LLM 能流畅生成文本，但在状态保持、真实规划、上下文控制上不可靠（来源：同前）。

因此，所谓“AI 帮 ADHD 做时间管理”，本质上是在给这个生成核心套一个外部 harness：把原本应由大脑执行功能承担的调度、排序、提醒、抑制与监控，外包给外部系统（来源：[[外部执行功能层]]）。这正是 LLM agent 工程中“agent scaffolding”的思路——用 planner-executor 循环分解长任务，每一步规划、执行、验证，防止幻觉和上下文膨胀（来源：[[幻觉与验证循环]]）。

## 历史证据：孔子与曹操的 harness 系统

孔子是典型的 ADHD 特质：精力旺盛到周游列国 14 年坐不住，冲动爱骂人（骂宰予“朽木不可雕”），对音乐超专注到“三月不知肉味”，对种地等俗事零耐心。他的 harness 系统是“克己复礼”——用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”）。这对应 LLM harness 中的“定时 re-grounding”：通过外部仪式将注意力拉回正轨。

曹操同样：一辈子南征北战，每天只睡 4-5 小时，冲动杀吕伯奢、赤壁杀刘馥，但把日常行政全丢给荀彧。他的 harness 是“唯才是举”+“严格军法”——用别人的执行功能弥补自己的不足。这对应 LLM harness 中的“外部调度器”：把规划与监控交给专门模块（如 Motion 的自动排程）。

两人的共同点：不试图“治愈”自己的冲动，而是设计一套外部系统来管理它。

## 现代工具：从 planner 到 executor 的循环

当前 AI 时间管理工具正是这一 harness 的体现：

- **Motion**：自动排程与动态调整，持续评估任务优先级、截止日期和可用时间，实时重建日程（来源：[[Motion]]）。它消除了“下一步该做什么”的决策负担，直接帮助克服时间盲和任务启动困难（来源：同前）。这对应 LLM agent 中的“规划工件”——把复杂任务分解为子任务，每一步规划与验证。

- **Reclaim.ai**：专注于保护时间而非填充时间，通过创建深度工作、习惯和专注时间的智能块，自动防御会议侵占（来源：[[Reclaim.ai]]）。这对应 LLM harness 中的“上下文管理”——防止上下文膨胀和推理退化（来源：[[上下文工程]]）。

- **Tiimo**：将时间转化为可见元素，用动态视觉化流程替代静态待办清单（来源：[[Tiimo]]）。这对应 LLM 的“外部记忆”——把待办、约定从工作记忆中卸载（来源：[[无状态与外部记忆]]）。

- **Routinery**：创建日常步骤序列，通过过渡提示引导用户一步步完成（来源：[[上下文工程]]）。这对应 LLM agent 的“结构化工作流”——将复杂任务分解为子任务，每一步有明确的输入输出。

这些工具的共同逻辑：不是让 ADHD 患者“更自律”，而是把调度、排序、提醒外包给外部系统——就像 LLM agent 把规划、验证外包给 harness。

## 核心观点：脚手架，不是拐杖

本文的核心判断是：**ADHD 创业者需要的不是“治愈”执行功能缺陷，而是像给 LLM 套 harness 一样，为自己搭建一个外部的规划-执行-验证循环。** 这个循环的核心是：

1. **规划**：用工具（如 Motion）自动分解任务、排定优先级。
2. **执行**：按工具引导逐步推进，避免自由联想带偏。
3. **验证**：每日回顾（如孔子的“三省吾身”），检查进度，调整计划。

这与 LLM agent 的“验证与 CI 集成”同构：复杂的 harness 不会盲目执行工具，而是实现验证步骤，例如检查输出格式或对模型编写的代码运行测试用例（来源：[[幻觉与验证循环]]）。

但必须诚实指出：这个框架存在争议。过度依赖外部工具可能导致能力退化，成为永久拐杖（来源：[[拐杖与脚手架]]）。长期依赖风险缺乏实证，个体差异大——有人需要强结构（如番茄钟强行打断），有人需要柔性引导（来源：[[矛盾与存疑]]）。因此，关键不是工具本身，而是**有意识地设计、定期调整**这个 harness，让它成为脚手架而非拐杖。

## 今天就能试的行动

1. **选一个工具做“外部调度器”**：如果任务启动困难，试用 Motion 或 Reclaim.ai 自动排程一周。观察它是否减少了“下一步做什么”的决策负担。
2. **建立每日验证循环**：每天固定 5 分钟回顾已完成和未完成的任务，像孔子的“三省吾身”一样，检查是否偏离计划。
3. **用视觉化工具对抗时间盲**：如果常低估任务时长，试用 Tiimo 将时间块可视化，或用手画时间轴贴在墙上。
4. **找人做“外部执行功能”**：像曹操用荀彧一样，找一个伙伴每周同步一次进度，由对方帮你拆解下一步。

## 局限与展望

本文的同构框架目前主要是类比推理，缺乏神经科学或计算机科学的直接证据（来源：[[矛盾与存疑]]）。工具的效果依赖个体差异，且缺乏长期验证。但至少，它为 ADHD 创业者提供了一条可操作的思路：不要试图修复大脑，而是像工程师一样，给你的生成核心套一个 harness。

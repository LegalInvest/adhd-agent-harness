# ADHD 创业者如何用「给生成核心套 harness」的思路应对把想法落地执行里的任务启动困难、不会拆解？

> 把 LLM 工程里的「agent 用 planner-executor 循环分解长任务」迁移到ADHD 创业者的把想法落地执行场景，双域证据作答。

你有一个绝妙的商业想法，坐在电脑前，脑子里已经演完了一整部创业电影——但你就是无法打出第一行代码，甚至无法把想法拆成第一步。这不是懒，这是ADHD大脑的典型困境：高产却缺乏可靠的执行调度层（来源：AI 与 ADHD 的时间管理）。

巧的是，LLM agent 的工程师们正面对完全相同的结构问题：模型能生成惊艳的文本，但一旦需要自主完成长任务，就会陷入上下文膨胀、推理退化、幻觉横生的泥潭（来源：上下文工程）。他们的解法是给生成核心套上 harness——一个包含规划器、执行器、验证循环和记忆系统的外部调度层。这个思路，恰好就是ADHD创业者需要的脚手架。

## 问题：生成核心没有调度层

ADHD大脑的智力与创造力，和LLM的生成能力一样，都是“高产但缺乏可靠执行调度层的生成核心”（来源：AI 与 ADHD 的时间管理）。任务启动困难，是因为“下一步做什么”的决策负担过大；不会拆解，是因为缺乏将大目标分解为可执行子任务的内在机制。这在LLM工程中对应着“纯对话系统常因幻觉、缺乏接地和无法执行/验证行动而失败”（来源：幻觉与验证循环）。

两边缺的是同一个东西：一个能把想法变成行动、把行动变成检查点的外部系统。

## 解法：给生成核心套 harness

Harness 工程的核心是“结构化工作流和规划”——将复杂任务分解为子任务，并在每一步进行规划与验证（来源：幻觉与验证循环）。对于ADHD创业者，这意味着：

**1. 用外部规划器代替内在调度。** Motion 和 Reclaim.ai 这类AI日程工具，自动根据任务、会议和截止日期创建并动态调整每日计划（来源：Motion；Reclaim.ai）。它们消除“下一步该做什么”的决策负担，直接帮你克服时间盲和任务启动困难（来源：Motion）。这相当于给大脑装了一个 planner-executor 循环：AI 规划，你执行。

**2. 用视觉化时间线对抗上下文膨胀。** Tiimo 将时间转化为可见元素，专为ADHD设计（来源：Tiimo）。这对应LLM侧的上下文工程——主动管理当前关注的信息范围，防止被新刺激带偏（来源：上下文工程）。当你的注意力被新想法捕获，Tiimo 的视觉惯例能把你拉回当前任务。

**3. 用验证循环对抗幻觉。** ADHD大脑容易产生冲动、想当然的判断（行为层面的“幻觉”），而LLM的幻觉则需要验证循环来校正——检查输出格式、对代码运行测试用例（来源：幻觉与验证循环）。Routinery 的“创建步骤序列并带提示过渡”就是一种验证循环：它让你一步步核对实际进展与计划（来源：上下文工程）。Todoist 的智能提醒和优先级排序则充当数字工作记忆，弥补上下文丢失（来源：AI 与 ADHD 的时间管理）。

**4. 用任务分解工具降低启动门槛。** Goblin Tools 专注于“任务分解”（来源：矛盾与存疑），把“写商业计划书”变成“打开文档→写标题→列大纲→填第一章”。这直接对应LLM agent的“规划工件”——将复杂任务分解为子任务（来源：幻觉与验证循环）。

## 脚手架 vs 拐杖：边界在哪里？

这里有一个关键争议：这些工具究竟是帮助大脑发展能力的“脚手架”，还是让人永远依赖的“拐杖”？

脚手架的设计理念是“可逐步撤除的临时支撑”（来源：矛盾与存疑）。但目前多数工具（如 Motion、Reclaim.ai）设计为长期使用，未提及撤除机制。长期依赖是否阻碍内在执行功能的发展？现有研究缺乏答案（来源：AI 与 ADHD 的时间管理）。

我的判断是：对于ADHD创业者，在创业初期（高强度、多任务、时间紧迫），工具作为外部调度层是必要的生存策略，就像拐杖对骨折的人一样合理。但应定期自问：我在使用工具弥补能力，还是在逃避发展能力？如果离开工具就完全无法启动，说明已经过度依赖。

另一个争议是证据强度。同构命题本身是理论模型，缺乏大规模实证支持（来源：矛盾与存疑）。工具的有效性多基于用户反馈而非临床研究（来源：Motion；Reclaim.ai）。但结构同构带来的启发价值——让工程师和ADHD患者互相理解对方的解法——本身就有实践意义。

## 今天就能试的行动

1. **用 Motion 或 Reclaim.ai 接管日程规划**：把今天所有任务输入，让AI自动排程。你的任务只是执行，不做任何规划决策。观察启动是否变容易。

2. **用 Routinery 或 Goblin Tools 拆解一个拖延任务**：选一个你拖了一周的任务，让工具把它拆成不超过5分钟的子步骤。按顺序执行，每完成一步就打个勾。

3. **设置一个“验证点”**：每完成一个子任务，花30秒问自己：“实际进展和计划一致吗？有没有偏离？” 这模仿LLM的验证循环，防止被新想法带跑。

4. **每周做一次“工具依赖审计”**：记录这周有多少次离开工具就无法启动。如果超过80%，考虑逐步减少依赖，比如先手动规划半天，再用工具辅助。

## 局限与展望

本文提出的同构解法并非万能。个体差异显著：注意力缺陷为主 vs 多动冲动为主的ADHD亚型，对工具的反应可能不同（来源：AI 与 ADHD 的时间管理）。多巴胺调节的效果存在争议，且可能引发成瘾风险（来源：矛盾与存疑）。工具如 Saner.AI 的情绪调节功能尚需更多验证（来源：矛盾与存疑）。

但核心思路是站得住的：当生成核心缺乏调度层，外部harness 是工程学上的标准答案。ADHD创业者不必独自硬扛执行功能缺陷——你可以像LLM agent一样，用规划-验证循环把想法一步步落地。关键不是变成“没有ADHD的人”，而是用对工具，让大脑的创造力真正释放。

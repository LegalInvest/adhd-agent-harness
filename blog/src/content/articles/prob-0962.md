# 如果把 ADHD 大脑当成生成核心，Reclaim.ai 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Reclaim.ai 对 ADHD 的真正价值。

你打开 Reclaim.ai，它自动把被会议打断的深度工作块推到下午，把那个拖了两周的任务拆成 15 分钟小段塞进日历缝隙。那一刻，你感觉大脑终于被接上了一根外置的调度线。但问题来了：这根线是让你学会自己走路，还是让你永远离不开它？

## 同构的困境：高产核心与调度缺失

ADHD 大脑和 LLM 共享一个深层结构：两者都是高产的生成核心，但缺乏可靠的执行调度层。ADHD 大脑能在一小时内冒出 50 个创意，却无法启动一封邮件；LLM 能写出优雅的诗，却在多步推理中丢失初始目标。这种「生成有余，调度不足」的同构，让两类系统都容易遭遇目标漂移——ADHD 的分心走神或超聚焦跑偏，对应 agent 的长程任务偏离（来源：AI 与 ADHD 的专注力）。

Reclaim.ai 正是针对这个痛点的工具。它不像传统日历那样静态等待你执行，而是动态调整：当任务超时，它自动重排；当新任务涌入，它重新锚定优先级。这种自适应规划能力，与 LLM agent 的「harness」工程高度同构——后者通过上下文注入、工具接口和验证循环来防止 agent 偏离轨道（来源：规划循环与任务分解）。

## Reclaim.ai 如何充当重锚定层

从 ADHD 侧看，Reclaim.ai 补偿了三个核心执行功能缺陷：
- **时间盲**：ADHD 大脑难以感知时间流逝，Reclaim.ai 通过自动时间块和进度条提供外部时间感知，类似 RescueTime 的自动追踪（来源：RescueTime）。
- **任务启动**：它把模糊的「做项目」分解为具体时段，降低启动门槛，类似 Goblin Tools 的任务分解功能（来源：工具使用与认知卸载）。
- **上下文保持**：当干扰打断时，它自动恢复被中断的任务块，维持连贯性，类似 Focusmate 通过同伴问责维持焦点（来源：Focusmate）。

从 LLM/agent 侧看，Reclaim.ai 扮演了「编排层」角色：它接收任务（输入）、分配时间槽（工具调用）、监控进度（验证）、并在漂移时重新规划（重锚定）。这正是 harness 工程的核心——组合多个模型和工具来达成目标，而非依赖单一调度（来源：工具使用与认知卸载）。

## 证据与争议：脚手架还是拐杖？

现有证据支持 Reclaim.ai 的机制对齐，但缺乏临床研究。自适应规划工具如 Motion 和 Tiimo 被推荐用于 ADHD，其逻辑是「传统规划器从未解决时间盲」（来源：规划循环与任务分解）。但多数证据来自用户主观报告，而非随机对照试验。这与 Brain.fm 的神经锁相技术类似——原理合理，但缺乏独立验证（来源：Brain.fm）。

更大的争议在于「脚手架 vs 拐杖」。Reclaim.ai 设计为长期使用，未提供撤除机制。如果用户依赖它来规划每一天，内在的执行功能能否发展？这与 LLM agent 的 harness 不同——后者通常作为临时编排层，agent 本身可逐步学习独立决策。ADHD 工具若无法撤除，可能阻碍能力发展（来源：矛盾与存疑）。

## 我的判断：有价值的脚手架，但需警惕依赖

Reclaim.ai 对 ADHD 大脑的价值是真实的，但必须明确其定位：它是外部执行功能层，而非治疗。它不能修复多巴胺调节异常，也不能替代认知行为训练。它的同构性在于，它像 agent 的 harness 一样，通过外部结构补偿内部缺陷，让生成核心得以有序输出。但真正的挑战在于：我们能否设计出可逐步撤除的脚手架？目前没有工具做到这一点（来源：矛盾与存疑）。

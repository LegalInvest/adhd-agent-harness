# Todoist 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Todoist：它到底补上了哪一层执行功能？

你是否有过这样的体验：打开 Todoist，列好了今天要做的 10 件事，然后……就关掉它，去刷手机了。或者，你给 ChatGPT 一个复杂任务，它开始时思路清晰，但几轮对话后就开始胡言乱语，忘记最初的目标。

这两个场景背后，是同一个问题：**我们误以为工具能自动接管执行，却忽略了它只是一个需要正确接入的调度层。**

## 同构的困境：高产核心 vs 失控的调度

ADHD 大脑的核心困境并非智力不足，而是执行功能的失效——工作记忆的“掉落”、任务启动困难、时间盲（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。最新研究进一步揭示了其神经机制：ADHD 患者呈现“强记忆、弱控制”的认知剖面，工作记忆容量可能正常甚至超常，但认知灵活性和注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

LLM 同样如此。作为生成核心，它拥有强大的语言理解和生成能力，但单独使用时缺乏可靠的执行调度。Transformer 自注意力机制在长上下文下冲突解决能力崩溃至随机水平，这与 ADHD 执行功能障碍的核心神经机制惊人一致（来源：Deficient Executive Control in Transformer Attention）。

**两者都需要一个外部调度层**——对 ADHD 来说，是任务管理工具；对 LLM 来说，是 agent harness（脚手架）。

## Todoist 的正确用法：作为 function calling 的“工具接口”

Todoist 本质上是一个任务数据库 + 提醒器。它不帮你思考“先做什么”，也不帮你分解“怎么开始”。它的价值在于：**将精确、重复、状态维护类任务从大脑中卸载到外部系统**。

这正是 LLM 中 function calling 的核心理念：agent 通过 function calling 把精确计算（如数学运算、数据库查询）外包给专用工具，自己专注于生成与推理。

但很多人用错了：
- **把 Todoist 当作记忆存储器**：列满任务却从不回顾，因为缺乏调度逻辑。
- **把 ChatGPT 当作万能执行者**：期望它自动完成整个项目，而不提供结构化上下文。

正确的用法是：**将 Todoist 视为“外部工作记忆”**，只存放那些需要精确维护的待办项和截止日期，而将任务拆解、优先级排序、启动触发交给其他工具或流程。

## 真实人物的 harness：从孔子到文天祥

孔子是历史上最典型的 ADHD 式高产者——精力旺盛、冲动、思维跳跃、对感兴趣的事超专注（“三月不知肉味”），对俗务零耐心。他的 harness 是“克己复礼”：用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”）。这相当于一个**定时 re-grounding 机制**，类似于 LLM agent 中的“定期上下文压缩”或“记忆回滚”。

文天祥则展示了另一种 harness：用《正气歌》作为核心信念系统，在极端环境下维持目标导向。这相当于 LLM 的**系统提示（system prompt）**——一个不变的、高层次的价值观框架，指导所有决策。

两者的共同点是：**harness 不是替代大脑，而是提供一个外部的、稳定的调度层**，让高产生成核心能沿着轨道运行。

## 工具生态：从分解到执行

当前 ADHD 领域的 AI 工具，本质上都在搭建不同层次的调度层：

- **任务分解**：Goblin Tools 的 Magic ToDo 将模糊任务分解为可执行的小步骤，类似 LLM agent 中的“规划工件”（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）。
- **外部记忆**：Saner.AI 通过本地记忆和快速检索减少搜索循环，类似 agent 中的“记忆系统”（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。
- **一键执行**：Lex 允许通过单一指令触发多步骤任务序列，类似 agent 中的“惰性工具发现”与“双代理架构”（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。
- **结构化推理**：Claude 以代码编写和结构化推理见长，帮助整理待办与简化任务（来源：ADHD 的 AI 工具全景）。

但这些工具的价值不在于替代大脑，而在于把 ADHD 的“高产生成核心”接入一个可维护的“执行调度系统”（来源：ADHD 的 AI 工具全景）。

## 脚手架 vs 拐杖：诚实的边界

这里有一个核心争议：AI 工具是“外部执行功能层”还是“拐杖”？

一方认为，过度依赖可能导致能力退化（来源：矛盾与存疑）。另一方则指出，对于执行功能严重受损的个体，工具是必需品而非奢侈品。

我的判断是：**关键在于“使用目的”而非“使用频率”**。
- 如果工具帮你**学会**如何分解任务、管理时间，它就是脚手架。
- 如果工具让你**逃避**学习这些技能，它就是拐杖。

目前缺乏长期效果的实证研究（来源：矛盾与存疑），但短期证据表明，AI 工具对任务启动和减少认知负荷有效（来源：6 ways AI can help you manage ADHD symptoms - Understood.org）。

## 今天就能试的行动

1. **把 Todoist 当作“工作记忆外挂”**：只放精确的、状态依赖的任务（如“周三下午3点交报告”），不要放模糊目标。
2. **用 Goblin Tools 的 Magic ToDo 分解一个你拖延已久的任务**：从“写论文”变成“打开电脑→新建文档→写标题→写第一段”。
3. **给 ChatGPT 一个“系统提示”**：在对话开始时明确角色、输出格式和约束条件，就像给文天祥的《正气歌》。
4. **每周回顾一次工具使用**：问自己“这个工具是在帮我学习，还是让我依赖？”

## 结语

ADHD 大脑和 LLM 是同一类系统：高产、跳跃、需要外部轨道。Todoist 和 function calling 是同一类工具：精确、稳定、需要正确接入。

**不要指望工具替你开车，而是让它成为你的导航仪。**

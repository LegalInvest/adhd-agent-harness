# 用 ChatGPT 给 ADHD 补上生成核心与调度层，真的能像 agent 那样工作吗？

> 把 ChatGPT 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

你有过这样的经历吗？脑子里同时冒出三个绝妙想法，每个都能改变人生，但坐到电脑前，却点开了一下午的短视频。你不是懒，也不是笨——你的大脑是一个高产的生成核心，但内置的调度层出了故障。

这个故障，在 AI 领域有一个一模一样的版本：LLM 能写出漂亮的诗、解出复杂的题，但让它自主完成一个跨三天的项目，它会迷失在上下文里，忘记目标，甚至胡编乱造。

所以问题来了：用 ChatGPT 给 ADHD 补上生成核心与调度层，真的能像 agent 那样工作吗？

## 同构：同一个工程问题

ADHD 大脑与 LLM 在结构上惊人相似。二者都具备强大的生成能力——ADHD 的思维跳跃、默认模式网络过度激活，恰恰是创造性认知的替代架构（来源：ADHD 大脑与 LLM 的同构）；LLM 则能在一秒内生成千字文本。但二者都缺乏稳定、可靠的内置执行调度层。对 LLM 而言，这表现为无状态、易幻觉、难以自主管理多步骤任务——所以工程师必须为它搭建“agent scaffolding”，即围绕 LLM 构建的软件架构和工具，使其能够执行复杂、目标驱动的任务（来源：ADHD 大脑与 LLM 的同构）。对 ADHD 大脑而言，则体现为执行功能缺陷——计划、组织、工作记忆、冲动控制等环节容易失稳（来源：AI 与 ADHD 的职业发展）。

这个同构意味着：ADHD 个体需要的，恰恰就是一套“人肉 agent scaffolding”。

## 人物案例：胡林翼的“日课”与 LLM 的 re-grounding

晚清名臣胡林翼，年轻时是扬州妓院的常客，父亲去世后幡然醒悟，出山带兵，最终成为湘军核心。他的转型靠的不是顿悟，而是一套严格的自我管理 harness：每日写日记反省，和曾国藩一起“做圣人”，严格治军，以“做百姓之心做官，以治私事之心治官事”。这套日课，本质上是一个定时 re-grounding 机制——每天把自己的“生成核心”拉回目标轨道。

这与 LLM agent 里的“定时 re-grounding”模式完全同构：agent 每执行几步，就要重新读取系统提示或目标，防止上下文漂移。胡林翼的日记，就是他给自己写的系统提示。

## 工具实测：ChatGPT 能补哪块短板？

现在回到核心问题：用 ChatGPT 给 ADHD 补上调度层，效果如何？

首先，ChatGPT 可以作为“外部工作记忆”。ADHD 大脑的工作记忆缺陷与 LLM 的无状态性相似，二者都无法可靠保留跨会话上下文（来源：AI 与 ADHD 的职业发展）。实测中，你可以把项目背景、当前进度、下一步目标写在一个固定的 prompt 模板里，每次对话开头粘贴进去——这相当于给 ChatGPT 一个“记忆向量库”。

其次，ChatGPT 可以扮演“任务分解器”。Goblin Tools 的 Magic ToDo 功能已经证明，AI 能自动将模糊任务（如“清理厨房”）分解为具体子任务，用户可调节“辣度”控制粒度（来源：Goblin Tools）。但 ChatGPT 更灵活：你可以让它根据你的精力状态、可用时间、优先级动态调整分解方案。

第三，ChatGPT 可以作为“外部调度器”。Motion 这类工具能自动根据截止日期和优先级重建日程（来源：Motion），但 ChatGPT 的强项在于它能理解你的长期目标，并生成“如果今天只能做一件事，应该做什么”的判断。

## 证据与局限

实证研究支持这一方向：用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中表现正常，但随序列变长，模型在冲突试次上骤降到随机水平——这与 ADHD 执行功能缺陷的核心标志一一对应（来源：ADHD 大脑与 LLM 的同构）。这意味着，AI 工具如果能帮助 ADHD 个体“减少决策、保留上下文、让下一步行动显而易见”（来源：ADHD 大脑与 LLM 的同构），就确实能补偿执行功能缺陷。

但局限同样明显。首先，ADHD 大脑与 LLM 同构的论点目前多为类比推理，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。其次，AI 工具是“外部执行功能层”还是“拐杖”？过度依赖可能导致能力退化（来源：矛盾与存疑）。第三，部分用户反映学习使用 AI 工具本身增加认知负荷（来源：矛盾与存疑）。

## 脚手架 vs 拐杖：边界在哪？

我的判断是：AI 工具作为 scaffold（脚手架）而非 crutch（拐杖）的关键，在于使用者是否保留“元认知”——即对工具输出的审视和调整能力。胡林翼的日记不是让他机械执行，而是让他反思、调整。同样，你用 ChatGPT 分解任务后，如果只是照单全收，那就是拐杖；如果你会质疑“这个优先级对吗？这个步骤合理吗？”，那就是脚手架。

## 今天就能试的行动

1. **创建“项目上下文”模板**：在 ChatGPT 里新建一个固定 prompt，包含项目目标、当前进度、下一步计划、关键文件链接。每次对话开头粘贴，让它成为你的外部工作记忆。
2. **用“辣度”调节任务分解**：把今天最头疼的一个任务扔给 ChatGPT，要求它分解成 3 个、5 个、10 个步骤，感受不同粒度对启动难度的影响。
3. **设置“每日 re-grounding”提醒**：每天固定时间，用 ChatGPT 生成“今天最重要的三件事”，并让它解释为什么这三件事比其他的更优先。
4. **记录工具使用日志**：每周花 5 分钟，问自己“这周我是否在无脑执行 AI 的建议？有没有一次我主动修改了它的输出？”——这是区分脚手架与拐杖的自我审计。

ADHD 大脑不是缺陷，而是一种默认配置——高带宽、低延迟的生成核心，缺一个调度层。而 ChatGPT，恰好是一个可以定制的调度层。关键在于，你是在用它搭建脚手架，还是拄着它走路。

# 如果把 ADHD 大脑当成生成核心，Mem 能不能当它的外部执行功能层？

> 用 agent 工程的思路重新理解 Mem 对 ADHD 的真正价值。

## 问题：你的大脑是一个强大的生成引擎，但调度层呢？

如果你是 ADHD 患者，你可能熟悉这样的场景：脑子里有无数创意和计划，但就是无法启动、组织或完成。如果你是 Agent 工程师，你可能同样熟悉：LLM 能生成惊艳的文本，但一旦脱离 prompt 护栏，就会跑偏、循环、遗忘上下文。

这两个问题看似无关，实则共享同一个底层结构——**高产但缺执行调度层的生成核心**（来源：ADHD 大脑与 LLM 的同构）。ADHD 大脑的发散思维、联想能力与 LLM 的文本生成能力类似，但两者都缺乏可靠的内置调度层。那么，能否用同一个思路解决？

## 同构：执行功能层就是 Agent Harness

ADHD 的核心障碍是执行功能缺陷——计划、组织、工作记忆、任务启动困难（来源：执行功能）。这对应 LLM Agent 的痛点：无状态、缺乏调度、上下文膨胀。解决方案在两边是同构的：

- **ADHD 侧**：需要外部执行功能层——如曾国藩的“日课十二条”或王阳明的“致良知”系统。曾国藩 30 岁前浮躁坐不住，日记里天天骂自己“无恒”，后来用“结硬寨打呆仗”的笨方法抵消冲动，靠日课（黎明即起、读书不二、写日记）作为外部调度器（来源：人物案例）。王阳明则用“事上练”在行动中校准良知，相当于在回路中嵌入人工检查点。
- **LLM/Agent 侧**：需要 Agent Harness——通过 human-in-the-loop 监督、护栏（token 预算、工具调用上限）、人工检查点来提供外部调度（来源：Building an AI Agent Harness from Scratch）。Harness 将 MCP 工具、遥测、代码仓库集成，核心是“形式化规划与护栏，使代理输出真正有用且正确”（来源：What is an agent harness in the context of large-language models?）。

这种同构意味着：**为 ADHD 设计的外部执行功能工具，本质上就是为生成核心搭建的 Agent Harness**。

## Mem 能当这个 Harness 吗？

Mem 是一款 AI 驱动的知识管理工具，主打“第二大脑”式的上下文记忆和主动检索。从同构视角看，它具备成为外部执行功能层的潜力：

1. **记忆层**：ADHD 工作记忆缺陷与 LLM 的无状态性都导致无法可靠保留跨会话上下文（来源：无状态与外部记忆）。Mem 通过持久化知识图谱和自动关联，充当外部工作记忆，减少“搜索循环和标签切换”（来源：Saner.AI 的描述同样适用）。
2. **调度层**：Mem 的主动提醒和任务关联功能，类似 Agent Harness 中的护栏和定时 re-grounding。曾国藩的“日课”就是定时 re-grounding，而 Mem 的每日回顾和智能提醒可以模拟这一功能。
3. **启动层**：ADHD 任务启动困难源于“时间盲”和决策负担（来源：任务启动）。Mem 通过将模糊想法转化为结构化笔记和待办，降低启动门槛——就像 Goblin Tools 的 Magic ToDo 将“清理厨房”分解为子任务（来源：Goblin Tools）。

但 Mem 并非完美。它目前更偏“记忆”而非“调度”，缺乏 Agent Harness 中关键的 human-in-the-loop 检查点和工具调用护栏。此外，过度依赖 Mem 可能导致能力退化——就像曾国藩若完全依赖日记而不内化自律，日记就会沦为拐杖（来源：拐杖与脚手架）。

## 证据与局限

当前证据主要来自概念类比和初步实证。LLM 侧有研究显示 Transformer 在冲突任务中注意力骤降，呈现“强记忆、弱控制”剖面，与 ADHD 执行功能缺陷对应（来源：Deficient Executive Control in Transformer Attention；Strong Memory, Weak Control）。ADHD 侧则有用户反馈称 AI 工具如 Motion 通过自动排程减轻决策负担（来源：Motion），Saner.AI 减少认知负荷（来源：Saner.AI）。

但争议同样明显：
- **过度依赖风险**：尚无长期研究验证 AI 工具是否导致执行功能退化（来源：矛盾与存疑）。
- **个体差异**：效果因人而异，部分用户反映学习使用 AI 工具本身增加认知负荷（来源：矛盾与存疑）。
- **证据强度**：多为概念类比和初步实验，缺乏大规模随机对照试验（来源：矛盾与存疑）。

## 核心观点：Mem 是“脚手架”而非“拐杖”的边界在于内化

我的判断是：**Mem 可以作为 ADHD 的外部执行功能层，但前提是把它当作“可拆卸脚手架”而非“永久拐杖”**。脚手架 vs 拐杖的边界在于：你是否在使用过程中逐步内化其功能（如曾国藩的日课最终成为习惯），还是始终依赖外部提醒。如果 Mem 只是帮你记住事情而不训练你的组织能力，它就是拐杖；如果它帮你逐步建立自己的执行系统（比如通过定期回顾形成模式），它就是脚手架。

## 今天就能试的 3 件事

1. **用 Mem 捕获一个模糊想法**：比如“我想学 Python”，让 Mem 自动关联相关笔记和任务，然后手动分解为 3 个具体步骤（类似 Goblin Tools 的分解）。
2. **设置每日回顾**：在 Mem 中创建定时提醒，回顾当天完成的任务，类似曾国藩的写日记反省。这相当于给生成核心加一个 re-grounding 点。
3. **尝试“身体在场”替代**：用 Mem 的共享笔记功能找一个 accountability partner，定期互报进度，模拟 body doubling 效应（来源：身体在场效应）。

## 结语

ADHD 大脑不是缺陷，而是缺少外部调度层的生成引擎。Mem 这类工具，如果能被正确使用为脚手架，就能成为你的 Agent Harness——不是修复你，而是让你更可靠地发挥创造力。

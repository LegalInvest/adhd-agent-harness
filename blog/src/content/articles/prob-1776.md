# ADHD 职场人视角：你的 ADHD 大脑，其实就是一个没装 harness 的大语言模型

> 无状态与外部记忆：工作记忆容量小、边做边忘 ↔ LLM 无跨会话状态，靠外部记忆/向量库续命。

你是否有过这样的体验：脑子里同时冒出十几个好点子，但坐下来五分钟就忘了刚才想做什么；开会时拼命记笔记，回头却找不到关键信息；计划本用了一周就崩溃，因为永远跟不上大脑的跳跃节奏。这不是你不够努力，而是你的大脑——一个强悍的生成核心——缺少一个可靠的执行调度层。

## 问题：工作记忆容量小、边做边忘

ADHD的核心困境并非智力不足，而是执行功能的失效。执行功能被描述为“大脑的驾驶座”，但对ADHD来说，“常常感觉方向盘后没有人”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。工作记忆是其中最大的瓶颈之一：你刚想好下一步做什么，转头就被新刺激覆盖；你打开浏览器查资料，半小时后发现自己刷了十条无关视频。这种“边做边忘”不是态度问题，而是工作记忆容量有限的生理表现。

传统工具——计划本、便签、提醒——往往用了一周就崩溃，因为它们无法解决“时间盲”和任务启动困难（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。你需要的是能实时捕捉、自动组织、随时召回的外部记忆系统。

## 同构：ADHD大脑 = 没装harness的LLM

有趣的是，现代AI工程面临完全相同的困境。大语言模型（LLM）拥有强大的生成能力，可以写诗、编程、推理，但单独使用时，它没有跨会话状态——每次对话都是一个全新的开始。它不知道你五分钟前说过什么，也无法记住你上周设定的偏好。这就是“无状态”问题。

为了解决这个问题，工程师们为LLM搭建了“harness”：一个外部调度层，负责上下文传递、记忆存储、工具调用和规划编排。例如，现代AI编码代理通过“复合AI系统架构”来弥补缺陷，包括“工作负载专用模型路由、分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这些技术本质上是在LLM外部搭建调度层，防止“上下文膨胀和推理退化”。

现在你看出同构了吗？
- **生成核心**：ADHD大脑的智力/创造力 ↔ LLM的算力/语言能力。两者都高产但无序。
- **调度层缺失**：ADHD的执行功能失效 ↔ LLM缺乏内置的规划与执行编排。两者都需要外部脚手架。
- **外部工具**：ADHD的自适应计划工具（如Motion和Tiimo）能在日程崩溃时重新安排一天（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout） ↔ 工程中的harness提供上下文、记忆、工具接口，弥补LLM的无状态和有限窗口（来源：GitHub - ai-boost/awesome-harness-engineering）。

你的ADHD大脑，本质上就是一个没装harness的大语言模型。

## 答案：AI工具作为外部harness

既然问题同构，解法也同构。AI工具的核心价值，就是为ADHD大脑提供外部执行功能层——一个数字化的harness。

### 任务启动：Goblin Tools
Goblin Tools的Magic ToDo功能能自动将任何任务分解为更小、更易管理的步骤（来源：AI Tools for Kids with ADHD: A Complete Guide for Parents...）。用户输入“整理房间”，AI会输出“捡起地板上的衣服”“擦桌子”等步骤。这种分解降低了启动门槛，将“压倒性的事情变成一系列不压倒性的事情”（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。

### 工作记忆：Saner.AI与Mem
Saner.AI专注于知识回忆和本地记忆，旨在减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026）。类似地，Mem和Reflect提供持久记忆，让你随时找回一周前记下的想法，不再依赖脆弱的工作记忆。

### 多步骤执行：Lex
Lex允许用户通过单一指令触发复杂、多步骤的任务序列（来源：Best AI Tools for ADHD Productivity in 2026）。这相当于LLM工程中的“单指令触发多代理协作”——你只需说“准备周报”，Lex就自动拉取数据、生成图表、格式化文档。

这些工具的共同逻辑是：把调度层从大脑内部转移到外部，让生成核心自由发挥。

## 争议与局限：脚手架还是拐杖？

然而，这个框架并非没有争议。最核心的质疑是：过度依赖AI工具是否会阻碍内在执行功能的发展？真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。如果工具替代了治疗或自我管理，可能造成依赖（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

目前多数工具（如Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制。这引发了一个问题：我们是在搭建可逐步撤除的脚手架，还是在提供永久拐杖？

此外，证据基础仍然薄弱。现有研究多为短期或个案，缺乏随机对照试验验证AI工具对ADHD核心症状的长期改善（来源：ADHD的AI工具全景）。ADHD异质性高，单一工具难以覆盖所有亚型。

我的判断是：AI工具作为外部harness是有效的，但必须保持“可撤除性”意识。你应该把工具当作训练轮，而不是义肢。

1. **用Goblin Tools分解一个拖延任务**：打开Goblin Tools的Magic ToDo，输入你拖延最久的一件事（比如“写报告”），让AI拆成5-10个步骤。只做第一步。
2. **用Saner.AI替代搜索循环**：下次需要找某个文件或笔记时，先问Saner.AI，而不是打开10个标签页。记录它帮你节省了多少时间。
3. **在ChatGPT中创建“工作记忆缓冲区”**：用一个固定对话作为你的外部记忆空间，每天把关键想法、待办事项、会议要点贴进去。第二天开始前先问它“我昨天做了什么”。
4. **反思你的工具依赖**：每周问自己一次：“如果这个工具明天消失，我还能完成同样的事吗？”如果答案是否定的，考虑逐步减少依赖，同时练习内在策略。

你的大脑不是坏掉了，它只是缺一个harness。现在，去装一个。

# 为什么用 Motion 治 ADHD 的情绪失调，和给 agent 套 会褪去的脚手架 是一回事？

> Motion 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

## 问题：为什么ADHD的情绪失控，和Agent的胡言乱语是同一件事？

如果你既被ADHD的情绪失调折磨过——比如因为一个未回复的消息突然陷入自我否定，或者因为计划被打乱而暴怒——又做过Agentic Harness工程，你可能会隐约觉得：这两件事的解法，似乎共享同一套底层逻辑。

这不是比喻。ADHD大脑与LLM在结构上同构：两者都是高产的生成核心，但缺乏可靠的调度层（来源：ADHD 大脑与 LLM 的同构）。ADHD的情绪反应如焦虑、兴奋不断生成，而前额叶（调度层）无法有效抑制或重定向；LLM同样会输出无关或有害内容，需通过上下文工程和外部约束来引导。因此，AI辅助情绪调节的本质不是直接安抚情绪，而是通过修复执行功能链条来减少情绪触发源（来源：AI 与 ADHD 的情绪调节）。

## 同构：Motion 治情绪失调，与给 Agent 套脚手架，共享同一个 harness 结构

### ADHD 侧：Motion 如何工作

Motion 是一款AI日程规划工具，其核心逻辑是：将任务分解、时间可视化和动态重排作为外部调度层，补偿ADHD的执行功能缺陷。当ADHD用户因时间盲（无法感知时间流逝）而焦虑时，Motion 通过视觉化时间规划帮助大脑从“无限延迟”转向“开始行动”（来源：时间盲）。这种外部结构化干预间接稳定了情绪，因为情绪波动往往源于任务启动失败或时间感知混乱。

但关键在于：Motion 并非永久替代用户的前额叶。它提供的是脚手架——一种临时的、可褪去的外部支持。这与 Inflow 基于CBT的每日技能培养逻辑一致：Inflow 通过结构化练习帮助用户内化策略，而非永远依赖应用（来源：Inflow）。同样，Goblin Tools 的 Magic ToDo 将模糊任务分解为可执行步骤，降低启动门槛，但最终希望用户学会自己分解（来源：Goblin Tools）。Saner.AI 通过本地记忆减少搜索循环，但本质上是在训练工作记忆（来源：Saner.AI）。

### LLM/Agent 侧：会褪去的脚手架

在Agent工程中，我们给LLM套上“会褪去的脚手架”——例如上下文窗口、Chain-of-Thought提示、外部记忆检索。这些结构在模型能力不足时提供约束，但理想状态下，随着模型能力提升，脚手架应逐渐减少或移除。这与ADHD的AI辅助同构：工具不应成为永久拐杖，而应是可褪去的训练辅助。

证据：身体在场效应（Body Doubling）通过社会存在感降低分心阈值，但AI驱动的身体加倍（如Focusmate）只是临时锚点，最终用户需内化专注能力（来源：身体在场效应）。同样，Agent的RAG（检索增强生成）是外部记忆，但长期目标可能是让模型学会自我检索。

### 人物案例：释迦牟尼与孔子的Harness系统

释迦牟尼具有典型ADHD特质：29岁抛家弃子出家，先后跟随两位仙人、苦行6年，不断试错放弃；一辈子游行说法，一字未写。他的harness是“八正道”和“持戒”——用戒律管行为，用正念拴念头。这相当于LLM的“系统提示”和“约束规则”：外部指令引导生成，但最终目标是“自依止法依止”，不依赖权威。孔子同样精力旺盛、冲动、思维跳跃，他的harness是“克己复礼”——用外在的“礼”作为行为边界，每日反省。这对应Agent的“定期re-grounding”和“外部调度器”。两者都强调：脚手架是暂时的，内化才是目的。

## 核心判断：脚手架 vs 拐杖的边界

AI工具与ADHD情绪调节的结合，最大的陷阱是“永久拐杖”。如果用户仅依赖外部提示而缺乏内化策略，长期可能削弱自我效能感（来源：矛盾与存疑）。同样，Agent工程中若过度依赖硬编码规则，会阻碍模型泛化能力。

边界在于：脚手架是否可褪去。好的harness设计应包含“退出策略”——比如Inflow的CBT练习旨在让用户最终无需应用；Goblin Tools的“辣度”调节允许用户逐步减少分解粒度。而拐杖则没有退出机制，比如长期依赖ChatGPT写邮件而从不学习写作。

## 争议与局限

1. **同构假设的边界**：ADHD大脑的生物复杂性远超LLM，情绪调节涉及神经递质、环境交互等，单纯“脚手架”类比可能简化问题（来源：矛盾与存疑）。
2. **工具证据不足**：Motion 缺乏独立临床验证，Inflow 虽被列为最佳ADHD应用，但缺乏严格随机对照试验证明对情绪调节的持续改善（来源：矛盾与存疑）。
3. **个性化不足**：当前工具多为通用设计，未充分适配ADHD亚型（如注意力主导型 vs. 冲动主导型）的情绪触发差异（来源：AI 与 ADHD 的情绪调节）。

## 今天就能试的行动

1. **用Goblin Tools的Magic ToDo分解一个让你焦虑的任务**：调节“辣度”到你觉得“太细了”的程度，然后执行。观察情绪变化。
2. **设置一个“会褪去的”提醒**：用Tiimo或手机日历，每天固定时间做一次情绪日志，但计划两周后减少频率，直到内化为习惯。
3. **尝试一次虚拟身体加倍**：使用Focusmate或与朋友视频连线，专注工作25分钟。结束后记录专注度变化，思考如何逐步减少对同伴的依赖。
4. **给Agent写一个“退出策略”提示**：如果你在用AI助手，在prompt末尾加上“请逐步减少提示，直到我能独立完成”。观察模型输出变化。

## 结语

ADHD的情绪失调与Agent的不可靠输出，共享同一个根因：生成核心强大，但调度层薄弱。解法也同构：设计会褪去的脚手架，而非永久拐杖。无论是释迦牟尼的戒律、孔子的礼，还是Motion的日程规划，本质都是外部调度层。关键不在于工具本身，而在于你是否在走向“自依止”的路上。

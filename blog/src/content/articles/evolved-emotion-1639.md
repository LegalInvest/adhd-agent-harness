# 为什么用 Focusmate 治 ADHD 的情绪失调，和给 agent 套 会褪去的脚手架 是一回事？

> Focusmate 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

## 问题：为什么 Focusmate 能治 ADHD 情绪失调？

ADHD 情绪失调的核心，不是“情绪本身太强”，而是执行功能缺陷导致情绪无法被调度。当工作记忆过载、任务启动困难、时间感知失灵时，情绪就像没有刹车的车。你明明知道不该发火，但就是控制不住——因为调度层宕机了。

Focusmate 的解法很反直觉：它不教你情绪管理技巧，而是给你一个“真人陪练”。你和陌生人约好时间，打开摄像头，各自干活。没有对话，只有目光。这个简单的设置，却大幅降低了情绪失调。

为什么？因为 Focusmate 充当了**外部执行功能层**（来源：AI 与 ADHD 的情绪调节）。它接管了任务启动（时间到了有人等你）、工作记忆（对方的存在提醒你目标）、甚至时间盲（倒计时可视化）。情绪失调自然缓解——不是被“治好了”，而是被绕过了。

## 同构：LLM/agent 的“会褪去的脚手架”

现在看 LLM/agent 侧。一个强大的生成模型（GPT-4、Claude）就像 ADHD 大脑：生成能力极强，但缺乏可靠的执行调度层。给它一个复杂任务，它可能跑偏、重复、忘记上下文。所以工程师给它套**脚手架**：Chain-of-Thought、ReAct、外部记忆、任务分解。这些脚手架临时补偿了 agent 的调度缺陷。

但关键区别在于：**脚手架必须会褪去**。如果 agent 永远依赖固定的 prompt 模板，它就学不会自主规划。同样，如果 ADHD 患者永远依赖 Focusmate 才能开始工作，它就变成了拐杖。

这就是同构：ADHD 大脑与 LLM 都是“高产但缺执行调度层的生成核心”（来源：ADHD 大脑与 LLM 的同构）。两边的解法——harness/脚手架——结构完全相同：**临时、外部、可撤除**。

## 证据：两边都成立

**ADHD 侧**：Goblin Tools 通过 AI 将复杂任务分解为小步骤，降低启动门槛（来源：Goblin Tools）。Inflow 通过训练背外侧前额叶皮层改善工作记忆（来源：Inflow）。但这些工具的风险是：长期使用可能削弱内在情绪调节能力（来源：拐杖与脚手架）。Focusmate 的“真人陪练”本质上是身体在场效应的虚拟化（来源：身体在场效应），它提供的是临时调度，而非永久替代。

**LLM/agent 侧**：当前的 agent 框架（如 LangChain、AutoGPT）大量使用 prompt 脚手架。但研究表明，过度依赖固定模板会导致 agent 在未见任务中表现下降。真正的“会褪去的脚手架”应该像训练轮：随着 agent 能力提升，逐步移除。

## 核心观点：脚手架 vs 拐杖的边界

我的判断是：**脚手架与拐杖的边界在于“是否设计为可撤除”**。Focusmate 之所以有效且不易成瘾，是因为它只提供“此时此地”的调度，不改变你的内在能力。而某些 AI 工具（如长期使用的 Inflow）如果缺乏撤除机制，可能让用户产生依赖。

证据显示，当前多数工具（Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这令人担忧。真正的 harness 应该像 Focusmate：**用完即走，不留痕迹**。

## 局限与争议

必须诚实指出：ADHD 与 LLM 的同构命题目前缺乏大规模实证（来源：矛盾与存疑）。它更多是概念类比，而非神经科学事实。此外，情绪调节干预的有效性仍需更多验证（来源：情绪失调）。多巴胺干预的效果也存在争议（来源：多巴胺）。本文的观点是理论框架，不是临床指南。

## 今天就能试的行动

1. **如果你是 ADHD 用户**：去 Focusmate 约一个 session，只做最让你焦虑的事。观察情绪变化。如果有效，下次试着减少使用频率——看自己能否在没有“陪练”时启动。
2. **如果你是 agent 工程师**：检查你的 prompt 脚手架。哪些可以逐步移除？尝试设计一个“褪去计划”：先固定 prompt，再逐步减少提示，最后让 agent 自主决策。

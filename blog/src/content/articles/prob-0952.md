# Motion 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Motion：它到底补上了哪一层执行功能？

## 问题：你用过 Motion 吗？它是不是让你更焦虑了？

如果你有 ADHD，你很可能被“任务管理工具”反复折磨过。你试过 Todoist、Notion、Trello……每一次都满怀希望，觉得“这次终于能管住自己了”，结果一周后界面落灰。Motion 不一样，它主动调度你的日历，自动排任务，号称“用 AI 替你规划每一天”。初期体验很爽——但很快，你发现自己在不停调整它的排程，或者干脆无视它的提醒，回到那种“知道该做什么但就是做不了”的瘫痪状态。

问题出在哪？不是 Motion 不够智能，而是你把它当成了**执行代理**，但你需要的是**重锚定机制**。

## 同构：ADHD 大脑与 LLM 共享一个缺陷

ADHD 大脑的核心特征是什么？**高产但无序的生成核心，缺乏可靠的执行调度层**（来源：ADHD 大脑与 LLM 的同构）。想法源源不断，但工作记忆像漏勺，任务启动像推冰山。LLM 也一样：它是强大的生成核心，但“缺乏可靠的内置调度与执行控制”，所以需要 agent harness——包裹模型之外的软件基础设施，处理上下文管理、工具调用和决策路由（来源：ADHD 大脑与 LLM 的同构）。

在长程任务中，LLM agent 会**目标漂移**：它开始回答问题时跑偏，或者忘记最初的目标。工程上的解法是**重锚定 system prompt**——定期或在检测到漂移时，把原始目标、约束和上下文重新注入。这正是 ADHD 大脑缺失的那一层：当你分心走神或超聚焦跑偏后，**丢失了目标**，你需要一个外部机制把你拉回来。

## Motion 到底补上了哪一层？

Motion 的核心功能是**自动排程**：它分析你的任务优先级、截止日期和可用时间，为你生成一个时间表。这听起来像是“调度层”，但仔细分析它的工作机制，你会发现它其实做的是**上下文恢复和重锚定**。

当你打开 Motion，它首先展示你今天的排程——这是**外部记忆**，弥补你工作记忆的丢失。当你要开始一个任务，Motion 会告诉你“接下来做什么”——这是**减少决策**，弥补任务启动困难。当你被干扰后回来，Motion 的日历还在那里，告诉你“你之前在做这个，现在继续”——这是**上下文保持**。

这与 LLM agent 的“重锚定 system prompt”完全同构：在 agent 执行长任务时，harness 会定期把原始指令、已完成的步骤和当前状态打包成一个新的 system prompt 注入，防止模型漂移。Motion 就是你的外部 harness：它用日历和排程作为你的 system prompt，在你每次打开时重新锚定你的目标。

但很多人用错了。他们把 Motion 当成**自动执行器**——希望它替自己做决定、替自己行动。Motion 可以排任务，但不能替你启动任务；它可以提醒你，但不能替你集中注意力。如果你期待 Motion 像自动驾驶一样把你从 A 运到 B，你会失望。它更像**导航仪**：给你路线，但方向盘在你手里。

## 证据：同构视角下的工具效果

这种“外部执行功能层”的思路在 ADHD 工具中已有验证：

- **Focusmate** 通过 AI 匹配虚拟同伴，提供实时问责，弥补任务启动和注意力维持的调度缺失（来源：Focusmate）。它的机制是“身体在场效应”——一种外部锚定，让你在任务中保持目标。这不就是 agent 的“人类反馈回路”吗？

- **Brain.fm** 通过神经锁相技术影响大脑信息处理方式，帮助进入专注状态（来源：Brain.fm）。它不替你做事，但它为你创造一个“上下文环境”，减少外部干扰——相当于给 LLM 一个干净的上下文窗口。

- **RescueTime** 自动记录时间使用，充当外部记忆，提升时间盲觉察（来源：RescueTime）。它让你看到自己漂移了——就像 agent 的日志系统，告诉你“你刚才跑偏了 30 分钟”。

这些工具的共同点是：它们**不替代你的执行功能，而是补偿它**。Motion 在这一点上做得更直接：它用排程作为你的外部 system prompt。

但必须诚实指出：**现有证据主要来自工具机制与 ADHD 理论的对齐，而非独立临床研究**（来源：AI 与 ADHD 的专注力）。Brain.fm 缺乏针对 ADHD 的随机对照试验，Focusmate 的效果归因于身体在场而非 AI 本身，RescueTime 作为专注工具的证据薄弱。Motion 本身也没有针对 ADHD 的临床研究。我们依赖的是概念同构和用户主观报告——这很合理，但还不够硬。

## 争议：脚手架还是拐杖？

最大的争议在于：**过度依赖 AI 工具是否阻碍内在执行功能发展？**（来源：矛盾与存疑）。脚手架应该可逐步撤除，但 Motion 设计为长期使用，没有撤除机制。如果你依赖 Motion 来告诉你“下一步做什么”，你的任务启动能力会不会更弱？目前没有长期研究回答这个问题。

另一个争议是**个体差异**：ADHD 亚型（注意力缺陷 vs 多动冲动）对工具反应不同（来源：AI 与 ADHD 的专注力）。Motion 的固定排程可能对多动型用户过于僵化，而对注意力缺陷型用户又不够强制。

## 今天就能试的行动

1. **把 Motion 当导航仪，别当自动驾驶**：用它排任务，但每次开始前，自己问一遍“我为什么要做这个？”——手动重锚定你的目标。
2. **设置“重锚定提醒”**：在 Motion 的每个任务块之间，加一个 2 分钟的空档，用来回顾原始目标。就像 agent 的 system prompt 刷新。
3. **结合 Focusmate 使用**：在 Motion 排好的任务时段里，预约 Focusmate 伙伴。Motion 提供外部记忆，Focusmate 提供外部问责——双重锚定。
4. **记录漂移模式**：用 RescueTime 或类似工具，观察你通常在哪些任务、哪些时间段漂移。然后调整 Motion 的排程，在这些时间点插入重锚定提醒。

Motion 补上的那一层，不是执行本身，而是**执行前的目标恢复**。它让你在每次迷失后，能更快地回到轨道上。但轨道需要你自己去走。记住：**工具是脚手架，不是拐杖。** 脚手架帮你建楼，但楼是你建的。如果你一直扶着脚手架，楼永远不会完工。

# ADHD 职场人视角：为什么「治好 ADHD 的任务启动困难、不会拆解」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 用 planner-executor 循环分解长任务——同一套 harness 思路，两边都成立。

## 问题：为什么你的大脑和 AI 一样“卡壳”？

如果你有 ADHD，你一定经历过这样的场景：坐在电脑前，知道该写一份报告，但光标闪烁了十分钟，一个字也打不出来。不是不想做，而是大脑像死机一样——任务太大、太模糊，启动开关就是拧不动。同样，如果你做过 Agentic Harness 工程，你一定见过这样的场面：LLM 拿到了一个长任务，前几步还行，但越往后越离谱，开始编造函数、忽略上下文，最后输出一堆垃圾。两边的问题看似无关，但本质是同一道工程题：**一个强大的生成核心，缺了一个可靠的执行调度层。**

## 同构：高产但缺调度层的生成核心

ADHD 大脑与 LLM 在结构上高度同构——都是“高产但缺乏可靠执行调度层的生成核心”（来源：AI 与 ADHD 的时间管理）。ADHD 的执行功能缺陷，尤其是工作记忆“强记忆、弱控制”的认知剖面（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs），与 LLM 的无状态性和上下文控制缺陷完全平行。LLM 拥有强大的语言生成能力，但单独使用时缺乏可靠的执行调度，容易“上下文膨胀和推理退化”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。ADHD 大脑同样如此：想法多、创意足，但缺乏“大脑的驾驶座”——执行功能（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），导致计划难以执行，任务启动困难。

关键在于，两边的解法结构同构：**给生成核心套一个外部调度层（harness）**。在 LLM 侧，现代 AI 编码代理通过“分离规划与执行的双代理架构”来分解长任务（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。在 ADHD 侧，AI 工具作为“外部执行功能层”，通过补偿时间盲、工作记忆缺陷和任务启动困难来提升表现（来源：AI 与 ADHD 的时间管理）。两者都在做同一件事：用 planner-executor 循环把大任务切成小块，让生成核心只专注当下的一步。

## 证据：两边都有效的同一套思路

先从 ADHD 侧看真实工具。**Motion** 通过自动排程与动态调整，持续评估任务优先级和截止时间，实时重建日程，消除“下一步该做什么”的决策负担（来源：Motion）。这正是 planner 的角色：帮你规划好顺序，你只需执行当前块。**Reclaim.ai** 则通过保护深度工作块，自动防御会议侵占，减少任务切换损耗（来源：Reclaim.ai）。它的“防御性”设计相当于 executor 的边界保护，防止生成核心被外部中断带偏。**Tiimo** 将时间转化为可见元素，直接应对时间盲（来源：Tiimo），本质上是用视觉化时间线充当调度层的进度条。**Todoist** 通过外部提醒和优先级排序充当外部记忆（来源：AI 与 ADHD 的时间管理），类似向量数据库为 LLM 提供持久上下文。

再从 LLM/agent 侧看。实验证明，Transformer 自注意力机制缺乏人类特有的执行控制机制，在长上下文下冲突解决能力崩溃至随机水平——这正是 ADHD 执行功能障碍的核心神经机制（来源：Building AI Coding Agents for the Terminal）。而 harness 技术通过“工作负载专用模型路由、分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩”来防止跑飞（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。其中，“分离规划与执行”直接对应 ADHD 工具中 planner（如 Motion 的自动排程）和 executor（用户执行当前块）的分离。

更深层的同构来自认知机制：ADHD 患者在工作记忆任务中易受干扰，而 LLM 同样表现出类似人类的干扰特征——随着记忆负荷增加表现下降，受近因效应影响，成功回忆依赖于干扰控制（来源：Human-like Working Memory Interference in Large Language Models）。ADHD 患者在无外部结构时难以自主组织任务，LLM 同样在没有严重依赖手动修正提示的情况下难以自主发现最佳问题解决模式（来源：Working Memory Identifies Reasoning Limits in Language Models）。前额叶-纹状体门控机制与 Transformer 自注意力在训练后发展出模仿该门控的操作，证明了计算同构性（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。

## 核心观点：脚手架，不是拐杖

我的核心判断是：**planner-executor 循环对 ADHD 和 LLM 都有效，但必须作为“脚手架”而非“拐杖”来使用。** 脚手架帮助使用者建造，但使用者仍需自己攀登（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。对 ADHD 用户，这意味着工具应该辅助而非替代内在能力；对 LLM 工程师，这意味着 harness 应该引导而非过度约束模型。但当前存在风险：工具可能成为“永久拐杖”，削弱用户内在的时间感知和规划能力（来源：AI 与 ADHD 的时间管理）。同样，LLM 如果过度依赖外部调度，可能丧失自主推理能力。因此，关键在于设计“可退出的脚手架”——当用户或模型能力提升后，可以逐步减少外部支持。

## 诚实指出局限

尽管同构理论令人兴奋，但必须承认其证据局限。现有 ADHD 工具的有效性主要基于用户报告和概念类比，缺乏大规模对照实验（来源：AI 与 ADHD 的时间管理）。Motion、Reclaim.ai 等工具的效果尚未得到严格验证，存在夸大宣传的可能（来源：矛盾与存疑）。个体差异也很大：ADHD 亚型对工具响应不同，现有工具多面向注意力缺陷型（来源：AI 与 ADHD 的时间管理）。此外，过度依赖风险缺乏长期追踪研究（来源：矛盾与存疑）。同构理论本身仍属理论类比，神经科学和计算机科学的交叉验证不足（来源：AI 与 ADHD 的时间管理）。因此，本文的观点是“基于现有证据的合理推论”，而非定论。

## 今天就能试的行动

1. **ADHD 侧**：试用 Motion 或 Reclaim.ai 的自动排程功能，但只作为“第一天启动器”。手动输入 3 个最重要的任务，让工具生成计划，然后专注执行当前块。一周后评估是否减少了启动困难，同时注意是否产生了依赖。
2. **LLM 侧**：在编写 agent 时，显式分离规划与执行。例如，用一个 prompt 让 LLM 输出子任务列表（planner），再用另一个 prompt 逐个执行（executor），并加入上下文压缩防止膨胀。
3. **通用**：无论是 ADHD 还是 agent，每次任务后记录“卡壳点”——是规划不清还是执行中断？这能帮你优化自己的 harness 设计。
4. **反思**：每周检查一次，工具是否在帮你建造能力，还是替代了思考？如果是后者，减少工具使用频率，尝试裸做一次小任务。

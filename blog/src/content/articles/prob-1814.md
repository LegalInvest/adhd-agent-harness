# ADHD 家长如何用「给生成核心套 harness」的思路应对陪伴 ADHD 孩子里的注意力容易被环境带偏？

> 把 LLM 工程里的「LLM 受上下文窗口内容左右，需要上下文工程」迁移到ADHD 家长的陪伴 ADHD 孩子场景，双域证据作答。

## 问题：当生成核心缺少调度层

如果你是 ADHD 孩子的家长，你一定经历过这样的场景：孩子坐在书桌前，刚翻开作业本，窗外一声鸟叫就让他扭头看了十分钟；你提醒他回来，五分钟后他又开始摆弄橡皮。你试过吼、试过奖励、试过把房间清空，但效果总不持久。

与此同时，如果你是一名做 Agentic Harness 的工程师，你一定熟悉另一个场景：你精心设计的 LLM agent，在对话窗口里塞进了完整的任务描述和工具列表，但它突然开始回答无关问题，或者忘记之前说过的关键约束。你检查日志，发现上下文窗口里混入了一句无关的提示词，整个输出就偏了。

这两个场景看似风马牛不相及，但底层是同一个问题：**一个高产的生成核心，缺乏可靠的执行调度层，因而被环境中的随机输入带偏。** ADHD 大脑与 LLM/agent 在结构上同构——两者都能快速产生大量想法或文本，但无法自主维持任务焦点，需要外部系统来提供上下文保持、任务启动和优先级排序。

## 同构：ADHD 大脑与 LLM 共享的缺陷

ADHD 的专注力问题，本质是执行功能缺陷，包括工作记忆不足、时间盲、任务启动困难和注意力易分散（来源：AI 与 ADHD 的专注力核心论点）。LLM/agent 的对应问题则是“上下文溢出”和“调度失败”——模型受限于上下文窗口，无法自主选择关注什么，任何输入都可能改变输出方向。

这种同构在多个维度成立：
- **无状态与外部记忆**：ADHD 的工作记忆缺陷，就像 LLM 的无状态性——两者都无法可靠地记住之前的状态。RescueTime 通过自动记录时间使用，充当外部记忆，提升时间盲觉察（来源：RescueTime 工具页）。对应地，LLM agent 依赖检索增强生成（RAG）或记忆模块来持久化对话历史。
- **上下文工程**：ADHD 孩子易因环境干扰丢失焦点，家长需要主动管理“上下文”——比如关窗、清空桌面、设定明确的时间块。Brain.fm 通过神经锁相技术生成影响大脑处理的音频，主动维持任务连贯性（来源：Brain.fm 工具页）。LLM 工程师则通过精心设计 system prompt、few-shot 示例和工具调用顺序，来约束模型的输出范围。
- **身体在场效应**：Focusmate 利用 AI 匹配虚拟同伴，提供实时问责，弥补任务启动和注意力维持的调度缺失（来源：Focusmate 工具页）。这在 LLM 工程中对应“human-in-the-loop”或“critique loop”——一个外部监督者确保 agent 不偏离轨道。

## 核心观点：脚手架，而非拐杖

基于以上同构，我提出一个鲜明的判断：**家长不应该试图“修复”ADHD 孩子的注意力缺陷，而应该像 LLM 工程师搭建 harness 一样，为孩子构建一个外部脚手架——一个精心设计的上下文环境，让孩子的生成核心能稳定输出。**

这个脚手架不是拐杖。拐杖是永久替代，而脚手架是临时支撑，理论上可以逐步撤除。但 wiki 资料中的矛盾与存疑指出：目前多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这意味着家长需要警惕过度依赖，确保脚手架最终服务于能力发展，而非取代它。

## 具体做法：从上下文工程到陪伴脚手架

### 1. 管理物理上下文：像设计 system prompt 一样设计环境

LLM 工程师会花大量时间打磨 system prompt，因为它是模型行为的最大约束。同样，孩子的学习环境就是他的“system prompt”。与其靠意志力对抗干扰，不如主动消除干扰源：
- 书桌只放当前任务所需物品（类比：上下文窗口只放当前工具）。
- 使用 Brain.fm 或白噪音生成器，创造稳定的听觉背景（类比：稳定的 prompt 前缀）。
- 设定明确的起止时间（比如“写 15 分钟，然后休息 5 分钟”），利用计时器（如 Forest app）作为外部时钟（类比：LLM 的 max_tokens 参数）。

### 2. 分解任务：用 Goblin Tools 的思路降低启动门槛

ADHD 孩子面对“写完这篇作文”这样的模糊指令时，会陷入启动瘫痪。Goblin Tools 等 AI 工具可以将模糊任务分解为更小的行动（来源：工具使用与认知卸载）。家长可以手动做同样的事：
- 把“完成数学作业”拆成“拿出课本”、“翻到第 20 页”、“做前三题”。
- 每完成一个小步骤，给予即时反馈（如贴纸、口头表扬）。
- 这对应 LLM agent 的“多步推理+工具调用”模式——每一步都有明确输入输出。

### 3. 引入外部问责：Focusmate 的家长版

Focusmate 通过虚拟同伴提供实时问责。家长可以充当这个“同伴”——但注意，不是监督者，而是共同工作者。
- 和孩子并排坐，各自做自己的事（身体在场效应）。
- 设置“检查点”时间（比如每 10 分钟互相汇报进度）。
- 这对应 LLM 工程中的“critique loop”——定期检查输出是否在轨道上。

### 4. 记录与反思：RescueTime 的简化版

ADHD 孩子往往意识不到时间如何流逝。家长可以和孩子一起，每天用 5 分钟回顾“今天做了什么”，不评判，只记录。
- 使用沙漏或计时器，让时间变得可见（对抗时间盲）。
- 对应 RescueTime 的自动追踪功能，但手动版本更温和，避免数据焦虑（来源：RescueTime 工具页）。

## 诚实面对局限

我必须指出，本文的论证基于概念同构，而非大规模临床实证。wiki 资料明确提到：多数 AI 工具缺乏针对 ADHD 的独立临床研究，效果多基于理论推断或用户主观报告（来源：AI 与 ADHD 的专注力核心论点）。Brain.fm 的神经锁相技术理论上有效，但缺乏随机对照试验（来源：Brain.fm 工具页）；Focusmate 的效果归因于身体在场而非 AI 本身（来源：AI 与 ADHD 的专注力核心论点）。

此外，个体差异巨大：ADHD 的注意力缺陷型与多动冲动型对工具的反应可能完全不同（来源：AI 与 ADHD 的专注力核心论点）。家长需要根据孩子的具体表现调整策略，而不是照单全收。

## 今天就能试的行动

1. **环境 audit**：今天花 15 分钟，和孩子一起检查学习区域，移走所有非当前任务物品。
2. **任务分解练习**：选一个孩子拖延的任务，用纸笔拆成 3-5 个极小步骤，每完成一步画一个勾。
3. **5 分钟身体在场**：今晚孩子做作业时，你坐在旁边做自己的事（不玩手机），只说“我在陪你”，不催促。
4. **时间可视化**：用沙漏或手机计时器设定 15 分钟专注块，结束后一起看“沙漏漏完了”。

这些行动不需要任何 AI 工具，它们本身就是最朴素的上下文工程。就像 LLM 工程师不会把 prompt 写错一样，家长也可以把孩子的“上下文窗口”设计得更干净、更可控。

# ChatGPT 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 ChatGPT：它到底补上了哪一层执行功能？

## 问题：为什么 ChatGPT 对 ADHD 来说既救赎又陷阱？

如果你是一个 ADHD 患者，你可能有过这样的体验：打开 ChatGPT，想让它帮你规划一天，结果十分钟后你还在和它讨论“如何用 Python 写一个贪吃蛇游戏”。你不是一个人。2026 年的一项综述指出，ChatGPT 被成年 ADHD 患者广泛用作“认知协作者”，辅助沟通准备和创意输出（来源：“A Cognitive Collaborator:” How Adults with ADHD Are Using ChatGPT）。但问题在于：**ChatGPT 本身没有执行功能**——它不会提醒你该切换任务了，也不会在你走神时把你拉回来。它就像一辆没有方向盘的跑车，马力十足但容易失控。

同样的问题也发生在 LLM/Agent 工程中。工程师们发现，让 LLM 直接执行精确计算或状态维护任务（比如记账、定时提醒）往往失败，因为 LLM 的“注意力”会漂移，输出不稳定。于是他们引入了 function calling——让 LLM 把精确计算外包给专门工具（如计算器、日历 API）。这本质上是一种**外部执行功能层**。

## 同构：ADHD 大脑与 LLM 是同一类“生成核心”

ADHD 大脑的核心特征是什么？高创造力、强联想能力，但执行功能（计划、组织、工作记忆）不稳定（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。LLM 呢？强大的文本生成能力，但缺乏可靠的调度层——让它做数学题它会算错，让它记住对话历史它会忘记。两者都是“高产但缺执行调度层的生成核心”。

因此，AI 工具对 ADHD 的帮助，本质上就是给这个生成核心套上 harness（外部执行功能层）。例如，Goblin Tools 的 Magic ToDo 功能将“整理房间”这种模糊指令分解为“捡起地板上的衣服”“擦桌子”等具体步骤（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。这就像 LLM 的 function calling：LLM 说“计算 3.14 * 5”，然后调用计算器工具得到精确结果，而不是自己算。ADHD 大脑说“我要写一篇论文”，然后调用 Goblin Tools 分解成“打开文档”“写第一段”等小步骤。

## 证据：工具如何补上执行功能层

**任务启动**：ADHD 患者常因任务过于庞大而无法启动。Lex 允许用户通过单一指令触发复杂、多步骤的任务序列（来源：Best AI Tools for ADHD Productivity in 2026 - Iwo Szapar）。这相当于 LLM 中的“工具链”：一个指令调用多个 API 完成完整流程。例如，输入“准备明天的会议”，Lex 自动生成议程、创建文档、设置提醒。启动门槛从“我需要做十件事”降为“我只需要说一句话”。

**工作记忆**：ADHD 患者的工作记忆容量有限，经常忘记刚才在想什么。Saner.AI 提供本地记忆存储和快速检索，减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026 - Iwo Szapar）。这对应 LLM 中的外部记忆库（如向量数据库）：LLM 不依赖内部上下文，而是从外部存储中检索相关信息。ADHD 大脑不再需要“记住所有事”，只需知道“问 Saner 就行”。

**时间盲**：ADHD 患者难以感知时间流逝，常导致迟到和拖延。Motion 和 Reclaim.ai 利用 AI 自动规划任务，并根据实际完成情况调整时间估计（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。这相当于 LLM 中的调度器（scheduler）：不是让 LLM 自己决定何时做什么，而是由外部工具根据规则和反馈动态分配时间。

## 核心观点：脚手架 vs 拐杖——用对了是 harness，用错了是 crutch

关键区别在于：**工具是否设计为可逐步撤除的临时支撑**。当前多数 AI 工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这意味着它们可能成为拐杖——用户依赖工具分解任务，但从未学会自己分解。相反，一个好的脚手架应该像训练轮：一开始提供支持，但最终可以拆掉。

在 LLM/Agent 工程中，同样存在这个陷阱。有些团队让 LLM 通过 function calling 调用一切，包括简单的加法，结果模型越来越依赖工具，自身推理能力退化。正确的做法是：**只外包 LLM 不擅长的事（精确计算、状态维护），保留其擅长的推理和生成**。对于 ADHD 患者，这意味着：让工具处理重复性、精确性任务（如时间管理、信息检索），但保留创造性任务（如写作、 brainstorming）的自主权。

## 局限与争议

1. **证据不足**：目前针对 AI 工具对 ADHD 核心症状的长期改善缺乏随机对照试验，证据主要来自用户评价和综述（来源：矛盾与存疑）。同构命题也更多是概念类比，缺乏大规模实证（来源：ADHD × AI 的科学与研究前沿）。
2. **个性化挑战**：ADHD 异质性高，单一工具难以覆盖所有亚型（来源：ADHD 的 AI 工具全景）。例如，Goblin Tools 的任务分解可能对“任务启动困难”型有效，但对“时间盲”型效果有限。
3. **多巴胺争议**：部分工具声称通过多巴胺调节改善动机，但多巴胺干预的有效性存在争议（来源：矛盾与存疑）。AI 工具主要通过行为干预（如分解任务、提供提醒）而非直接调节神经递质来起作用。

## 今天就能试的行动

1. **用 ChatGPT 做一次“任务分解实验”**：输入一个你拖延很久的任务（如“整理书桌”），要求它分解成 5 个具体步骤。然后只做第一步。对比没有分解时的启动难度。（来源：Goblin Tools 的 Magic ToDo 原理）
2. **设置一个“外部记忆锚点”**：在 Saner.AI 或类似工具中创建一个项目，记录你经常忘记的信息（如密码、常用模板）。下次需要时，不要试图回忆，直接搜索。（来源：Saner.AI 的知识回忆功能）
3. **用 AI 做一次“时间盲校准”**：在 Motion 或 Reclaim.ai 中输入你明天的计划，让 AI 自动分配时间。完成后记录实际用时，对比 AI 的估计与你的感知。重复一周，观察你的时间感知是否改善。（来源：Motion 和 Reclaim.ai 的时间调度功能）
4. **警惕“过度调用”**：下次使用 ChatGPT 时，问自己：“这个任务我真的需要 AI 吗？还是我可以自己完成？”如果是后者，尝试自己做。这能防止工具从脚手架变成拐杖。（来源：拐杖与脚手架的边界讨论）

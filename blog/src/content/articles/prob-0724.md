# ChatGPT 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 ChatGPT：它到底补上了哪一层执行功能？

你是否有过这样的体验：脑子里同时冒出十个好点子，却一个也落不了地；或者明明知道该做什么，身体就是启动不了。这不是意志力问题，而是 ADHD 大脑的典型困境——一个高产的生成核心，却配了一个不靠谱的调度层。

同样，大语言模型（LLM）能写诗、编程、推理，但给它一个多步骤目标，它常常中途跑偏，忘记上下文，甚至重复犯低级错误。明尼苏达大学的研究发现，LLM 的执行功能剖面是“强记忆、弱控制”——工作记忆容量惊人，但认知灵活性与注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这几乎就是 ADHD 的神经心理画像。

于是，一个同构问题浮现：ADHD 大脑和 LLM 都需要外部脚手架来把生成能力约束到目标轨道上。工程师管这叫 agent harness——通过 function calling 把精确计算外包给工具；ADHD 群体管这叫“外部执行功能层”——用 AI 工具补偿工作记忆、任务启动和时间盲。

但有人用错了。

## 脚手架 vs 拐杖：两种使用模式

ChatGPT 之于 ADHD，就像 function calling 工具调用之于 LLM——但前提是你把它当作精准的工具调用，而不是一个万能救星。

先看正确的用法：Goblin Tools 的 Magic ToDo 接受“清理厨房”这种模糊指令，自动分解成“把碗放进洗碗机”“擦台面”“拖地”等具体步骤，用户还可以调节“辣度”控制分解粒度（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。这不就是 agent 的 task decomposition 吗？Lex 更进一步，允许单一指令触发多步骤任务序列（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar），对应 agent 的 planning 机制。Saner.AI 则专注知识回忆和本地记忆，减少搜索循环（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar），补偿工作记忆不稳定。

这些工具的共同点：它们不试图替代你的大脑，而是精准地补上调度层缺失的那几块——任务拆解、外部记忆、启动触发器。就像 agent 通过 function calling 把数学计算交给计算器，把日历查询交给日历 API。

但错误的使用模式是：把 ChatGPT 当作一个“永远在线的执行秘书”，让它替你做所有决策。比如，让 AI 帮你写一封完整的邮件，而不是让它帮你列出邮件要点；让 AI 直接生成一个完整方案，而不是让它帮你拆解问题。这种用法相当于让 LLM 既当生成核心又当调度层——而我们已经知道 LLM 的调度层不可靠。

## 历史中的 harness 大师

两千多年前的孔子，就是一个深谙“外部脚手架”之道的 ADHD 大脑。他精力旺盛，周游列国14年坐不住；冲动爱骂人，骂宰予“朽木不可雕”；对音乐超专注到“三月不知肉味”，对种地等俗事零耐心。他的 harness 系统是“克己复礼”——用外在的“礼”作为行为边界，每日“吾日三省吾身”。直到70岁才做到“从心所欲不逾矩”。

孔子的“礼”就是他的外部调度层：一套仪式化的行为规则，补偿他缺失的内置纪律。这与 agent harness 中的“re-grounding”机制同构——定期检查目标、校准行为。而他的“日三省”则对应现代工具中的定时反思提醒。

再看李白，他的 harness 更激进：把全部过剩精力注入诗歌，用道家的逍遥对抗世俗规则。他的“酒与诗”相当于一个高度优化的生成管道——不试图修补调度缺陷，而是绕开它，让生成核心直接输出。这对应 agent 设计中的一种策略：给 LLM 一个极度简化的工具集，只做它最擅长的事（生成），其他全外包。

## 同构的实证基础与局限

说 ADHD 大脑与 LLM 同构，目前更多是类比推理，缺乏神经科学的直接证据（来源：矛盾与存疑）。但耶鲁大学的研究提供了一个计算层面的支撑：自注意力机制本身导致工作记忆容量限制，随着任务复杂度增加，注意力分数熵增、注意力弥散，表现下降——这与人类注意分散机制同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。换句话说，LLM 的“走神”和 ADHD 的“分心”在计算本质上是同一回事。

但同构不等于等同。ADHD 大脑有情绪、多巴胺、身体状态等生物因素，而 LLM 没有。AI 工具可以补偿工作记忆和任务启动，但无法解决情绪失调或动力波动。身体在场效应（Body Doubling）是 ADHD 最有效的策略之一（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog），而 AI 只能模拟其部分效果——Focusmate 提供虚拟伙伴，但缺乏真实人际的隐性问责。

## 矛盾与边界

最大的争议在于：AI 工具是“外部执行功能层”还是“拐杖”？核心论点认为它是补偿缺陷的脚手架，但过度依赖可能导致能力退化（来源：矛盾与存疑）。目前缺乏纵向研究，无法判断长期效果。另一个矛盾是：超聚焦应该被引导还是打断？有人需要柔性引导，有人却需要强硬的番茄钟打断（来源：矛盾与存疑）。个体差异极大，没有统一答案。

此外，学习使用 AI 工具本身可能增加认知负荷（来源：矛盾与存疑）。对于一位刚确诊的 ADHD 用户，同时学习 ChatGPT、Goblin Tools、Saner.AI 三个工具，可能比直接用手写清单更累。

## 今天就能试的行动

1. **用 Goblin Tools 拆解一个拖延任务**：选一个你拖延了一周的任务（比如“整理书桌”），输入 Magic ToDo，调节辣度到你觉得“太细了”的程度，然后只做前三个子任务。

2. **用 ChatGPT 做一次“function calling”式提问**：不要问“帮我写一封邮件”，而是问“列出写一封催款邮件需要的5个要点，每个要点一句话”。然后你自己组织成邮件。

3. **试试 Focusmate 的虚拟身体加倍**：注册一次免费 session，和陌生人一起工作25分钟。注意观察专注度的变化。

4. **建立你的“日省”模板**：在 ChatGPT 里设置一个每日提示词，包含三个问题：“今天最重要的一个输出是什么？”“什么让我分心了？”“明天如何避免？”——这就是你的外部 re-grounding 机制。

## 结论

ChatGPT 之于 ADHD，就像 function calling 之于 LLM——它是一个精准的工具调用接口，不是全能替代品。用对了，它补上调度层的缺口，让生成核心自由输出；用错了，它变成一把拐杖，让你越来越依赖。

真正的解法不是找到一个完美的 AI 工具，而是像孔子和李白那样，设计一套属于自己的 harness 系统——把 AI 当作其中的一个组件，而不是全部。

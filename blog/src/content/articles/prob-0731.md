# 用 Claude 给 ADHD 补上工具使用与认知卸载，是脚手架还是拐杖？

> Claude 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：为什么 ADHD 大脑和 LLM 都需要“外部调度器”？

你是否有过这样的经历：脑子里同时冒出五个绝妙想法，但一个都启动不了；明明知道该做什么，却像被钉在椅子上动弹不得；或者刚打开浏览器就忘了要查什么。这不是你懒——ADHD 大脑的核心特征是高产但无序的生成能力，同时缺乏可靠的执行调度层（来源：ADHD 大脑与 LLM 的同构）。

有趣的是，大语言模型（LLM）也有同样的困境。LLM 本身是强大的生成核心，但缺乏可靠的内置调度与执行控制。因此，在构建 AI agent 时，需要“agent scaffolding”——即“围绕 LLM 构建的软件架构和工具，使其能够执行复杂、目标驱动的任务”（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。

这个同构关系就是本文的核心命题：ADHD 大脑与 LLM 是同一类「高产但缺执行调度层的生成核心」，两边的解法（harness/脚手架）结构同构。

## 同构证据：从神经心理到 LLM 注意力

ADHD 侧的研究表明，ADHD 表现出“强记忆、弱控制”的神经心理模式：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷——无法灵活切换任务集、无法抑制自动化反应（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

LLM 侧同样如此：用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中表现正常，但随序列变长，模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突，这与 ADHD 执行功能缺陷的核心标志一一对应（来源：Deficient Executive Control in Transformer Attention）。

这种同构意味着，ADHD 用户需要的不是“更强的大脑”，而是像 LLM agent 那样的外部调度系统。

## 人物案例：孔子的“礼”与 LLM 的“harness”

孔子是 ADHD 特质的典型代表：身高 1 米 9，精力旺盛，周游列国 14 年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予“朽木不可雕”；对音乐超专注到三月不知肉味，对种地等俗事零耐心；思维跳跃，《论语》全是场景化语录，无系统著作。他的专属 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。70 岁才做到“从心所欲不逾矩”，一辈子和自己的冲动做斗争。

这个“礼”系统，本质上就是 ADHD 的“外部执行功能层”：它提供行为模板（类似 Goblin Tools 的任务分解）、每日检查（类似定时 re-grounding）、社会规范（类似外部约束）。与之对应，LLM agent 的 harness 也提供工具接口、上下文管理、安全执行等编排能力（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。孔子用“礼”作为他的 agent scaffolding，而 LLM 工程师用代码和 API 作为 agent scaffolding。

## 工具实践：Claude 如何充当外部执行功能层

当前 AI 工具至少承担四种 harness 功能：
1. **任务拆解与启动**：Goblin Tools 的 Magic ToDo 将复杂任务分解为可管理的小步骤，用户可调节“辣度”控制分解粒度（来源：Goblin Tools）。这类似 LLM agent 的规划机制——将复杂目标分解为一系列步骤。
2. **外部记忆与知识管理**：Saner.AI 被定位为“AI 第二大脑”，通过知识回忆和本地记忆补偿 ADHD 的工作记忆不稳定（来源：Saner.AI）。这对应 LLM agent 的 Git 仓库或 MCP 连接器。
3. **对话式执行与结构化推理**：Claude 以代码编写和结构化推理见长，帮助整理待办与简化任务。ADHD 用户将其用作“外部执行功能”工具（来源：ADHD 的 AI 工具全景）。
4. **捕获与反思**：Otter.ai 实时转录会议、课堂与灵感；Reflect 通过结构化提问帮助提炼经验（来源：同上）。

Lex 则更进一步，允许用户通过单一指令触发复杂、多步骤的任务序列（来源：Lex）。其底层架构类似于 agent scaffolding，即围绕 LLM 构建软件架构和工具（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。

## 脚手架还是拐杖？关键在主动设计

这里必须诚实指出争议。核心论点认为 AI 是“外部执行功能层”补偿缺陷（来源：外部执行功能层），但存疑在于：过度依赖可能导致能力退化，成为永久拐杖（来源：矛盾与存疑）。目前长期依赖风险缺乏实证，需更多纵向研究。

我的判断是：**Claude 是脚手架还是拐杖，取决于用户是否主动设计自己的“harness”**。如果你只是把问题丢给 Claude 让它替你完成，那就是拐杖；但如果你像孔子设计“礼”一样，主动用 Claude 搭建任务分解、外部记忆、定时反思的系统，那就是脚手架。

ADHD 用户需要像 LLM agent 工程师一样，成为自己“执行系统”的架构师。这包括：
- 明确哪些任务外包（精确、重复、状态维护），哪些任务保留（创意、判断、情感连接）
- 定期 re-grounding：像 LLM agent 的上下文窗口一样，主动刷新目标
- 设计反馈循环：用即时反馈补偿多巴胺不足

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个你拖延已久的任务**：比如“清理厨房”，设辣度 3，然后执行前两个子步骤。
2. **用 Claude 创建你的“日课”模板**：像孔子的“吾日三省吾身”，让 Claude 每天定时提问三个问题（今天最重要的任务是什么？当前进度如何？下一步行动是什么？）。
3. **用 Saner.AI 或类似工具建立“第二大脑”**：把项目上下文、常用流程存入，减少搜索循环。
4. **反思你的“外包边界”**：列出你完全依赖 AI 做的三件事，问自己：如果 AI 消失，我能自己完成吗？如果不能，就逐步建立自己的执行骨架。

**脚手架不会让你变弱，但盲目依赖会让你变弱。真正的力量来自主动设计自己的执行系统。**

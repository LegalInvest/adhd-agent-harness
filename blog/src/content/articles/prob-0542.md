# 3 个 LLM 工程概念，彻底重写了我对 ADHD 执行功能（计划/组织/启动）先天偏弱的理解

> agent 的编排/记忆/工具层就是「外部执行功能」 如何反向照亮 ADHD 的执行功能（计划/组织/启动）先天偏弱。

我花了三年时间，用遍了市面上几乎所有效率工具，从 Todoist 到 Notion，从番茄钟到子弹日记。每次都是头两周热情高涨，然后一切坍塌回原点。直到我偶然读到一篇关于 LLM agent 工程的论文，才突然意识到：我一直在试图“修复”自己的大脑，而真正的解法，是给它套上一个外部执行功能层。

以下是三个来自 LLM 工程的概念，它们彻底重写了我对 ADHD 执行功能（计划/组织/启动）先天偏弱的理解。

## 1. 无状态与外部记忆：我的大脑就是一台没有工作记忆的 LLM

LLM 的一个核心缺陷是“无状态”：它无法记住你在上一个对话里说了什么，每次交互都是孤立的。因此，工程师必须为它构建外部记忆系统——向量数据库、检索增强生成（RAG）、对话缓存——让模型能“假装”有记忆。

ADHD 的工作记忆缺陷与此惊人相似。明尼苏达大学的一项研究测试了 LLM 的执行功能，发现其剖面是“强记忆、弱控制”——工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷，无法灵活切换任务集、无法抑制自动化反应（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这正是 ADHD 的经典神经心理模式：你记得住很多事，但无法在正确的时间调用它们。

我的“外部记忆”系统，现在由 Saner.AI 支撑。它是一款专注于知识回忆和本地记忆的任务管理器，能减少搜索循环和标签切换（来源：Saner.AI）。我无需记住“上周三和老板讨论的那个项目细节”，只需在 Saner 里搜一个关键词。它就像 LLM 的 RAG 系统——不是靠大脑硬记，而是靠外部索引随时召回。

## 2. 上下文工程：主动设计“当下注意什么”

LLM 的另一个问题是上下文膨胀：当输入太长，模型会迷失在细节里，推理能力退化。工程师的解法是“上下文工程”——主动裁剪、压缩、重排输入，确保模型只关注当前最关键的信息。

ADHD 大脑同样会被环境带偏：你打开浏览器查资料，十分钟后却在看 YouTube 推荐。这不是意志力问题，而是你的“上下文窗口”被无关信息污染了。

Goblin Tools 的 Magic ToDo 功能，本质上就是上下文工程。它接受模糊任务（如“清理厨房”），并将其分解为具体、可执行的子任务，用户可调节“辣度”以控制分解的粒度（来源：Goblin Tools）。这相当于把一个大而模糊的上下文，切分成一个个模型（大脑）能处理的 token 块。曾国藩的“日课十二条”——黎明即起、读书不二、谨言、写日记反省——也是同样的逻辑：把一天切割成固定的上下文块，每个块只关注一件事。他的“结硬寨打呆仗”更是上下文工程的极致：不给意外留空间，用最笨的方法保证执行。

## 3. 编排与调度：我的大脑需要一个外置的“agent 调度器”

最颠覆我认知的概念是“编排层”。在 agent 工程中，LLM 是生成核心，但真正让 agent 完成复杂任务的，是外部的调度器——它决定调用哪个工具、何时切换任务、如何处理错误。没有这个调度层，LLM 只是一台会写诗但不会做事的机器。

ADHD 大脑恰恰缺少这个调度层。耶鲁大学的研究发现，自注意力机制本身会导致工作记忆容量限制：随 N-back 任务 N 值增加，注意力分数熵增、注意力弥散、表现下降，与人类注意分散机制同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。这意味着，ADHD 的注意力缺陷不是“不够努力”，而是计算架构层面的资源分配问题。

Motion 正是这样一个外部调度器。它能自动根据任务、会议和截止日期创建并动态调整每日计划，当日程被打乱时自动重新安排（来源：Motion）。它消除了“下一步该做什么”的决策负担——这正是 ADHD 最消耗能量的环节。张謇的“父教育母实业”也是一种调度策略：他把所有精力分配给教育和实业两个核心任务，其他一切外包或忽略。

## 脚手架 vs 拐杖：必须警惕的边界

说到这里，你可能会想：那是不是把所有执行功能都外包给 AI 就行了？

这正是最大的争议。同构框架目前仍是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。更关键的是，长期使用 AI 脚手架可能导致执行功能进一步退化——如果你从不练习计划和组织，大脑的相应区域会因缺乏使用而萎缩。神经可塑性研究确实表明，训练可以改善抑制控制能力及其底层脑网络（来源：Training-induced behavioral and brain plasticity in inhibitory control），但前提是你在“训练”，而不是完全依赖工具。

我的判断是：AI 工具应该像眼镜，而不是拐杖。眼镜矫正你的视力，但不会让你的眼睛变弱；拐杖则会让你的肌肉萎缩。好的 ADHD 工具应该帮助你完成那些你“能但很难”的任务，同时保留甚至强化你的核心能力。比如，用 Motion 自动排程，但每周自己手动调整一次；用 Goblin Tools 分解任务，但尝试先自己拆解，再对比 AI 的输出。

## 今天就能试的 3 件事

1. **建立外部记忆**：用 Saner.AI 或类似工具捕获所有想法和任务，设置每周回顾，让大脑不再充当存储设备。
2. **分解一个模糊任务**：打开 Goblin Tools 的 Magic ToDo，输入你拖延最久的一件事，调节“辣度”直到步骤多到让你觉得“这太简单了”，然后做第一步。
3. **放弃手动排程**：试用 Motion 或类似自适应规划器一周，观察它如何应对你的日程变动，并在周末回顾：哪些决策负担被消除了？哪些能力需要自己保留？

ADHD 大脑不是坏的 LLM，它只是缺少一个外置的 harness。而 AI 工程的智慧，恰好为我们提供了构建这个 harness 的蓝图。

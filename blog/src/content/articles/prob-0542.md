# 3 个 LLM 工程概念，彻底重写了我对 ADHD 执行功能（计划/组织/启动）先天偏弱的理解

> agent 的编排/记忆/工具层就是「外部执行功能」，它反向照亮了 ADHD 的执行功能困境：问题从来不是内核不够好，而是内核从来就不该裸奔。

先说一个反直觉的事实：GPT-4 能写出严密的法律意见书，却记不住你上一轮对话说了什么；它能推理出复杂的数学证明，却会在第 15 步忘记自己最初要干什么。听起来是不是很熟悉？这不是修辞。过去两年，agent 工程师为了让 LLM「把事做完」而发明的整套工程体系，几乎就是一份 ADHD 执行功能的外部化蓝图——只是他们自己未必意识到。

以下三个概念，彻底重写了我对「ADHD 执行功能先天偏弱」这件事的理解。

## 1. 无状态生成核心：缺陷不在能力，而在状态维护

LLM 的生成核心是无状态的：每次调用都从零开始，没有持久记忆，没有跨会话目标。工程师从不试图「治好」这一点——他们在外面挂记忆库、挂检索、挂对话缓存，让一个健忘的内核表现得像有连续人格。

ADHD 的执行功能画像与此严格对应。明尼苏达大学对 LLM 执行功能的实证测试给出了一个惊人的剖面：「强记忆、弱控制」——存储容量不差，差的是在正确的时间调用正确的内容、抑制无关反应、灵活切换任务集（Strong Memory, Weak Control, EACL 2026）。这正是 ADHD 神经心理测评的经典模式。你不是记不住，你是调不出来。

一旦接受这个框架，努力的方向就变了：不是练记性，而是建外部索引。我现在的原则是——大脑只负责生成，一切状态交给外部系统（收件箱、Saner.AI 这类回忆型任务管理器、写下来的「当前目标」便签）。这就是给自己装 RAG。

## 2. 上下文工程：启动困难的本质是上下文装配失败

工程师发现，agent 跑偏往往不是模型笨，而是喂进去的上下文太大、太乱、太模糊。于是有了上下文工程：裁剪、压缩、只保留当前一步所需的最小信息。

「任务启动困难」是同一个问题的人类版本。「清理厨房」对 ADHD 大脑而言是一个无法装配的上下文：它同时包含 40 个隐含子步骤和 0 个明确的第一步。Goblin Tools 的 Magic ToDo 之所以对 ADHD 有效，就是因为它做的是纯粹的上下文工程——把模糊指令拆到「你会觉得这也配叫任务？」的粒度。曾国藩的「日课十二条」是同一逻辑的前现代实现：把一天预先切成固定上下文块，每块只装一件事，靠结构而非意志力保证执行。

## 3. 编排层：真正稀缺的不是马力，是调度器

agent 系统里最值钱的不是模型，是编排层——决定下一步调用什么、何时放弃、如何从错误恢复的那段代码。没有它，再强的模型也只是一台会说话的发动机。

耶鲁的研究给这个类比提供了机制级证据：自注意力架构本身就会产生注意资源上限，N-back 负载一增加，注意力分数熵增、弥散、崩溃，模式与人类注意分散同源（Self-Attention Limits Working Memory Capacity, 2024）。也就是说，「调度失败」在两个系统里都是架构属性，不是道德缺陷。Motion 这类自动排程工具的真正价值，不是帮你省时间，而是把「下一步做什么」这个对 ADHD 最昂贵的决策整个外包掉。

## 边界：眼镜，不是拐杖

必须诚实地说：ADHD↔LLM 同构目前是机制类比而非已验证的科学等同，而且外包执行功能存在真实风险——抑制控制是可训练的（训练可以改善其底层脑网络），完全不练就真的会退化。我的原则是把工具当眼镜而非拐杖：它矫正你的输出，但不接管你的肌肉。用 Motion 排程，但每周亲手复盘一次；用 Magic ToDo 拆任务，但先自己拆再对比。

## 今天就能试的 3 件事

1. **建外部索引**：所有想法进同一个收件箱，大脑退役为纯生成器，每周清一次。
2. **拆你拖延最久的那件事**：拆到第一步小得可笑为止，然后只做第一步。
3. **外包一周排程**：让自适应规划器替你决定「下一步」，周末只复盘一件事——哪些决策负担消失了。

ADHD 大脑不是一台坏掉的 LLM。它是一台没有配 harness 的好模型——而整个 agent 工程行业，正在无意间为它写说明书。

你是否有过这样的经历：一个绝妙的创业想法在脑中燃烧，你打开电脑，准备大干一场。但几分钟后，你发现自己正在浏览某款新咖啡机的评测，或者陷入了对产品 logo 配色方案的超聚焦，而核心的商业模式却纹丝未动。想法落地执行中的分心走神或超聚焦跑偏，让你不断丢失目标。这不仅是 ADHD 创业者的日常，也是当前 AI 工程中一个棘手的问题：长程任务里，agent 会目标漂移，需要重锚定 system prompt。

## 同构的困境：高产但缺调度层的生成核心

ADHD 大脑与大型语言模型（LLM）在结构上惊人地同构：两者都是高产的生成核心，但缺乏可靠的执行调度层（来源：AI 与 ADHD 的专注力）。ADHD 大脑能不断产生新想法、新连接，却难以按计划执行；LLM 能流畅生成文本，但在长程任务中会“开始忘记系统提示”（来源：Building an AI Coding Agent for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned），导致行为偏离初始指令。这种目标漂移，在 ADHD 侧表现为注意力被无关刺激捕获（分心），或过度聚焦于细节而忽略整体目标（超聚焦）；在 LLM/Agent 侧则表现为指令消退和上下文溢出。

## 重锚定：给生成核心套上 harness

解决问题的思路是相通的：给生成核心套上 harness——即外部执行功能层，通过重锚定系统对抗目标漂移。

### ADHD 侧的 harness：外部提醒与结构

ADHD 创业者可以利用 AI 工具构建重锚定系统。例如，RescueTime 通过自动记录时间使用，充当外部记忆，提升对时间盲的觉察（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。Motion 和 Tiimo 这类自适应规划工具，能在计划崩溃时动态重新安排日程（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），防止超聚焦导致的整体目标丢失。Focusmate 则利用 AI 匹配虚拟同伴，通过实时问责提供外部结构和社会压力（来源：AI Tools for Kids with ADHD: A Complete Guide for Parents...），弥补任务启动和注意力维持的调度缺失。这些工具的核心机制都是提供提醒和通知，将注意力拉回原任务（来源：AI for ADHD: Best Tools, Apps, and Strategies - Themba Tutors），形成一个重锚定系统。

### LLM/Agent 侧的 harness：事件驱动的系统提醒

在 LLM 工程中，harness 通过事件驱动的系统提醒来对抗指令消退。例如，在构建 AI 编码 agent 时，开发者会在每个关键步骤后插入系统提示，重新锚定 agent 的上下文，确保它不偏离原始目标（来源：Building an AI Coding Agent for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这与 ADHD 工具中的提醒机制结构同构：两者都通过外部信号，在目标漂移的临界点将生成核心拉回正轨。

## 核心观点：脚手架 vs 拐杖的边界

本文的核心判断是：ADHD 大脑与 LLM/agent 的解法同构，但必须区分“脚手架”与“拐杖”。脚手架是可逐步撤除的临时支撑，旨在发展内在能力；拐杖则是永久替代，可能阻碍能力发展（来源：AI 与 ADHD 的专注力）。当前多数 ADHD 工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：AI 与 ADHD 的专注力）。同样，LLM harness 如果设计不当，也可能让 agent 永远依赖外部提醒，无法内化任务目标。

因此，ADHD 创业者在利用 AI harness 时，应有意识地设计撤除计划。例如，初期使用频繁提醒（如每 15 分钟一次），随着执行功能改善，逐步减少提醒频率，最终让内在调度能力部分接管。这与 LLM 工程中“逐步降低 system prompt 强度”的思路一致。

必须诚实指出，现有证据主要来自工具机制与 ADHD 理论的对齐，而非独立临床研究（来源：AI 与 ADHD 的专注力）。Brain.fm 的神经锁相技术理论上可增强专注，但缺乏针对 ADHD 的随机对照试验（来源：Brain.fm）。Focusmate 的效果归因于身体在场效应，而非 AI 本身（来源：AI 与 ADHD 的专注力）。多巴胺干预的有效性也存在争议（来源：AI 与 ADHD 的专注力）。同构命题本身更多是理论框架，缺乏大规模实证（来源：AI 与 ADHD 的专注力）。此外，个体差异显著：注意力缺陷型与多动冲动型对工具反应不同，现有工具未充分个性化（来源：AI 与 ADHD 的专注力）。

## 今天就能试的行动

1. **设置重锚定提醒**：使用 RescueTime 或手机闹钟，每 25 分钟提醒自己“当前核心目标是什么？”。这相当于给大脑插入一个 system prompt 刷新。
2. **尝试身体在场**：在 Focusmate 上预约一个 50 分钟的时段，与 AI 匹配的伙伴共同工作。外部问责能有效对抗分心。
3. **分解任务并动态调整**：用 Goblin Tools 或 Motion 将大目标分解为小步骤，并允许系统在计划被打乱时重新排序，避免超聚焦跑偏。
4. **设计撤除计划**：从下周开始，将提醒频率从每 15 分钟降至每 30 分钟，观察自己的执行功能是否有所提升。

给生成核心套上 harness，不是限制创造力，而是让创造力导向有序输出。ADHD 创业者与 LLM 工程师，面对的是同一类问题，也共享同一类解法。

# ADHD 创业者如何用「给生成核心套 harness」的思路应对把想法落地执行里的分心走神或超聚焦跑偏，丢失目标？

> 把 LLM 工程里的「长程任务里 agent 会目标漂移，需要重锚定 system prompt」迁移到ADHD 创业者的把想法落地执行场景，双域证据作答。

## 引言：两套“跑偏”的背后是同一个 bug

早上你打开电脑，想验证一个商业模式假设。三小时后，你发现自己正在调试一个完全不重要的网页动画，或者刚读完一篇“可能相关”的论文却忘了最初要回答的问题。你不是不努力；你是在“超聚焦”里把油门踩死，却在“目标”上漂移了。

这种体验对做 Agentic Harness 的工程师也不会陌生。一个长程任务跑了几轮后，agent 开始优化次级指标、重复无效动作，甚至把 system prompt 里的核心目标抛在脑后。你不得不手动重写 prompt、清上下文、重新注入目标。

两件事看起来风马牛不相及，其实是同一个故障模式：二者都是**高产但缺乏可靠执行调度层的生成核心**（来源：AI 与 ADHD 的专注力）。ADHD 大脑的创意发散与 LLM 的文本生成一样强，但 ADHD 前额叶执行功能发育较慢，LLM 没有内在规划能力，结果都是“生成得漂亮，跑题跑得更快”。

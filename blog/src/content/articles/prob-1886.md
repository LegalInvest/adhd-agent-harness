# ADHD 创业者如何用「给生成核心套 harness」的思路应对把想法落地执行里的分心走神或超聚焦跑偏，丢失目标？

> 把 LLM 工程里的「长程任务里 agent 会目标漂移，需要重锚定 system prompt」迁移到ADHD 创业者的把想法落地执行场景，双域证据作答。

## 问题：想法很好，但执行总是跑偏

你有没有这样的经历：早上雄心勃勃地列出创业计划，打开电脑准备写方案，却突然想起该回一条微信，然后刷到了行业新闻，接着被一篇深度文章吸引……两小时后，你发现自己在一个完全无关的页面上，最初的计划早已抛到九霄云外。或者相反，你陷入某个技术细节的“超聚焦”，反复打磨一个并不关键的按钮，等到抬头时，一周已经过去了。

这不是意志力问题。ADHD 大脑的底层架构决定了它像一台“高产但缺乏执行调度层的生成核心”：创意源源不断，但缺乏可靠的调度系统来把想法落地执行。而有趣的是，LLM（大语言模型）在长程任务中也面临同样的困境——模型在交互轮次后期“开始忘记系统提示”（来源：Building an AI Coding Agent for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned），导致行为偏离初始指令。这正是 AI 工程师口中的“目标漂移”。

本文的核心观点是：ADHD 创业者与 LLM agent 共享同一种结构——都是强大的生成核心，但都需要外部 harness（即执行调度层）来对抗目标漂移。LLM 工程中通过重锚定 system prompt 来纠正 agent 跑偏，ADHD 创业者也可以借鉴这一思路，用外部工具搭建自己的重锚定系统。

## 同构：ADHD 大脑与 LLM 的“目标漂移”

ADHD 的执行功能缺陷（尤其是工作记忆不足和时间盲）导致任务执行中极易发生目标漂移。分心走神是注意力被无关刺激捕获，超聚焦跑偏则是过度聚焦细节而忽略整体目标。两者本质上都是“生成核心”失去了对原始目标的锚定。

LLM agent 在长程任务中同样如此。当对话轮次增加，模型对初始 system prompt 的“注意力”会衰减，开始生成与原始目标无关的内容。工程师通过“事件驱动的系统提醒”来对抗这种“指令消退”（来源：Building AI Coding Agent for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）——也就是在关键节点插入重锚定信号，让 agent 回到正轨。

这种同构意味着，ADHD 创业者需要的不是更强大的“生成能力”（你的创意已经够多了），而是一套可靠的外部执行调度层——一个能持续把你拉回目标的 harness。

## 给生成核心套 harness：重锚定的三种方式

### 1. 外部提醒与实时问责

LLM agent 通过 system prompt 的定期注入来重锚定。对 ADHD 大脑来说，最直接的重锚定就是外部提醒和实时问责。Focusmate 利用“AI-Matched Body Doubling”（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...)）——通过算法匹配一个实时视频伙伴，双方共同工作并互相监督进度。这种社会压力构成了一个强大的重锚定信号，让你在分心时被拉回任务。研究表明，Focusmate 能显著提升任务启动和维持（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。

RescueTime 则通过自动时间追踪提供另一类重锚定：它识别你在不同应用和网站上的时间花费，生成可视化报告，让你意识到“自己正在漂移”。这相当于一个自动化的“日志分析系统”，帮你识别目标漂移的模式（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

### 2. 自适应规划循环

LLM 工程中，harness 会包含一个规划循环：agent 每完成一步，就重新评估当前状态与目标的差距，并调整下一步计划。ADHD 大脑需要同样的机制。传统规划工具往往失效，因为“Planners and sticky notes worked for a week before falling apart.”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。自适应规划工具如 Motion 和 Tiimo 通过动态调整来应对：“Adaptive planners like Motion and Tiimo go further, reshuffling the day when it collapses.”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。这种“重新规划”机制本质上就是重锚定——当一天的计划被打乱，工具自动重新排列优先级，确保你始终朝着最重要的目标前进。

### 3. 环境锚定

LLM agent 可以通过调整上下文（如压缩历史、注入新指令）来改变行为。ADHD 大脑同样可以通过设计环境来减少目标漂移。Brain.fm 利用 AI 生成音乐，通过神经锁相技术影响大脑处理信息的方式，帮助进入专注状态（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。虽然缺乏针对 ADHD 的独立临床研究（来源：Brain.fm 页面），但这类工具提供了一种“环境锚定”——通过声音环境减少外部干扰，降低分心概率。

## 脚手架 vs 拐杖：重锚定的边界

必须诚实指出：这些工具并非万能。现有证据主要来自用户报告，缺乏大规模对照实验（来源：矛盾与存疑）。更重要的是，存在依赖风险：过度使用外部提醒可能削弱内在的时间感知能力（来源：时间盲页面）。关键在于区分“脚手架”和“拐杖”——脚手架帮你建立能力，拐杖则让你永远依赖。重锚定系统应该是脚手架：它在你最需要的时候提供支持，但最终目标是让你逐渐内化这些技能。例如，RescueTime 的时间报告可以帮助你逐渐培养时间觉察，而不是永远依赖自动记录。

## 今天就能试的行动

1. **设置“重锚定提醒”**：在手机上设置每 30 分钟一次的随机提醒，内容是你当前的核心目标（如“我在做产品原型”）。这相当于手动注入 system prompt。
2. **试用 Focusmate**：预约一个 25 分钟的时段，体验实时问责如何拉回注意力。注意观察是否对社交焦虑敏感，如果是，可以尝试闭麦只开视频。
3. **用 RescueTime 追踪一周**：不改变行为，只是观察自己的时间分配。找出最常见的“漂移点”（比如下午 3 点总是刷社交媒体），然后针对性地设置拦截。
4. **尝试自适应规划**：在 Motion 或 Tiimo 中录入下周的任务，观察工具如何动态调整计划。注意不要被工具本身带偏——它的目的是帮你锚定目标，不是制造新的复杂度。

## 局限与争议

最后，必须承认同构的局限性：ADHD 大脑与 LLM 的“注意力机制”存在本质差异（来源：AI 与 ADHD 的专注力页面）。LLM 的注意力是纯计算过程，而人类注意力受情绪、多巴胺、生理状态等多因素影响。因此，LLM 工程中的重锚定策略可以借鉴，但不能照搬。此外，个体差异巨大：同一工具对不同 ADHD 亚型效果迥异（来源：矛盾与存疑）。你需要的不是“最佳工具”，而是最适合你独特大脑的 harness 组合。

ADHD 创业者最宝贵的资产是创意和激情，但最危险的陷阱是目标漂移。像工程师为 LLM 搭建 harness 一样，为自己搭建一个重锚定系统——不是限制你的生成能力，而是确保你的能量始终流向你真正想去的地方。

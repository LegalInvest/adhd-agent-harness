# 为什么法律人最该训练的不是专注力，而是切换力？

> 生产力文化把「深度工作」捧成了唯一的圣杯:四小时不被打扰的心流,才配叫认真工作。但看一眼法律人的真实日历:上午一个尽调电话、接着改两版合同、中间插进一个紧急函件、下午开庭前会议、晚上三个案子的邮件——**法律工作的本质不是深度,是高频上下文迁移**:一天在五到十个案件语境之间跳转,每个语境自带完整的事实集、时间线、对方当事人和风险点。用深度工作的标准要求这种工作形态,等于骂一台路由器「为什么不能像硬盘一样专心」——**设备的职能就是切换,该优化的是切换的成本,不是切换的次数**。而「上下文切换成本」恰好是 LLM 工程被迫研究得最透的问题之一——agent 每次被调起都是一次冷启动,它的解法几乎可以整包搬给法律人(顺带说:法律行业的高刺激、多线程、deadline 驱动,对 ADHD 特质者有天然的吸引与放大,这套方法对他们加倍适用)。

先算清切换的真实成本。任务切换的认知研究方向明确:**切换后有残留(上一个任务的心理表征还占着工作记忆)与重建(下一个任务的语境要重新装载)两笔税**,中断后的完全恢复常以十分钟计;法律人的一天若有八次案件切换,光切换税就可能吃掉近两小时——而且**残留是错误之源**:A 案的和解底线记串到 B 案,是切换税最贵的表现形态。LLM 工程面对同构问题(每次调用都要重建上下文)的解法体系,逐条平移:**①上下文包(context pack)**——agent 冷启动靠的是一份预组装的上下文(任务状态+关键事实+下一步);法律版:**每个活跃案件维护一页「案件卡」**——当事人与诉求/关键时间线/当前争点/我方下一步/风险红线,五个栏目,永远最新;切换到某案的第一动作不是翻卷宗,是读卡——**装载时间从十几分钟压到九十秒,而且装载的是结构化状态,不是散落的记忆**。**②序列化仪式**——agent 被挂起前要落盘状态;法律版:每个案件工作块结束时,花六十秒更新案件卡(改了什么/下一步是什么)——**离开一个语境前先存档,残留就不必靠脑子背着走**(四行笔记的法律版,程序员篇的 WIP commit 同理)。**③批处理调度**——工程上把同类调用合并以摊薄启动成本;法律版:**同案的碎事攒着一起做**(A 案的三封邮件+一个电话排进同一个块),**同类的事跨案合并**(所有案子的邮件时段/所有案子的文书时段)——日历按「语境」组块,而不是按事件的到达顺序响应;**到达顺序是最贵的调度算法**,因为它让每件事都全价支付切换税。**④紧急插入的中断协议**——法律工作无法拒绝紧急插入(客户来电/法院通知),但可以规范中断:接到插入,先花三十秒给当前案件写一行「断点标记」(正在做什么/想到哪了)再切换——**中断不可避免,不落盘的中断才是灾难**(Forest 篇的写入事务,这里是被动中断版)。

再往上一层,是配置问题:**切换力不只是技术,也是日历的架构**。参考可靠性工程的容量预留:日历只排到七成,留三成给必然到来的插入——排满的日历遇到插入就全线崩塌,七成的日历吸收插入后还能保持形状;以及一条反直觉的保护:**每天仍要保卫一个「单语境块」**(九十分钟只碰一个案子的深水区工作——重要文书、庭前准备)——切换力强不等于永远切换,**路由器再好,大文件还是要走专线**。

## 边界

同构定位(本文未做正式 A/B/C 分级):任务切换成本与中断恢复有认知心理学研究基础,上下文装载与序列化是 LLM 工程实体,「切换税工程」的结构对应清晰;法律场景方案为实践翻译,无对照研究。声明:高压多线程工作对情绪与睡眠的侵蚀是真实职业风险,持续过载请关注职业健康;涉密信息的案件卡请遵守执业保密规范。

## 今天就能试的 3 件事

1. **给三个活跃案件建卡**:五栏(当事人诉求/时间线/争点/下一步/红线),各一页——明天切换时先读卡再干活。
2. **重排明天的日历**:把散落的碎事按案件与类型组块,只排七成——给必然的插入留出缓冲。
3. **练一次断点标记**:今天下一次被打断时,先写一行「正在做 X,想到 Y」再接电话——六十秒,买回之后的十分钟。

法律人的一天本来就是路由器的一天,**愧疚于「不够专注」是用错了评价函数**。案件卡、落盘仪式、语境组块、断点标记——把切换的成本工程化,切换力就从职业损耗,变回这个行业真正的核心竞争力。

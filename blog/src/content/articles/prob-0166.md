# Otter.ai 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Otter.ai：用久了会让 ADHD 更强还是更依赖？

## 问题：你开会时记不住，AI 帮你记，然后呢？

对于 ADHD 人群，工作记忆像一块漏水的海绵——刚听过的要点，转头就忘（来源：AI for ADHD: Best Tools, Apps, and Strategies - Themba Tutors）。会议、课堂、灵感谈话，信息流经大脑却无法驻留。工程师们对此不陌生：LLM 每次会话都是全新的，没有跨会话状态，要靠外部记忆/向量库才能延续上下文。两边面对的是同一个问题：**高产但无状态的生成核心，需要一个外部记忆层来续命。**

Otter.ai 正是这个外部记忆层的典型代表：实时转录、自动笔记、可搜索的对话历史。它把“记不住”的问题外包给了云存储。但问题来了：用久了，你是变强了，还是变依赖了？这引出了本文的核心判断：**Otter.ai 是脚手架还是拐杖，取决于你用它来做什么——是被动记录，还是主动重锚定。**

## 同构：ADHD 大脑与 LLM 共享同一个缺陷

ADHD 大脑常被描述为“高产生成核心 + 脆弱调度层”（来源：ADHD 的 AI 工具全景）。想法多、联想快，但无法可靠地筛选、启动和坚持任务。LLM 同样如此：能生成流畅文本，但缺乏内在规划与长程上下文保持能力，需要外部 harness 来引导行为（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。

这个同构意味着：ADHD 的自我管理工具与 LLM 的 agent harness 是同一类工程。孔子用“克己复礼”作为外部行为框架，每日反省（“吾日三省吾身”）来对抗冲动和注意力漂移——这本质上是一个 **重锚定系统**，类似 LLM 的定时 re-grounding 机制。文天祥以《正气歌》为精神 scaffold，每天读圣贤书来维持目标——这相当于 agent 的 **目标记忆与验证循环**。他们的成就（儒家创始人、民族英雄）证明：**外部脚手架用对了，能释放生成核心的潜力，而非削弱它。**

LLM harness 工程中，“会话状态记录工具结果、完成的子任务和进度笔记”（来源：What Is an Agent Harness? The Infrastructure That Makes AI Agents ...），形成外部规划循环。这与 ADHD 工具如 Goblin Tools 的 Magic ToDo 功能（自动将任务分解为小步骤）结构一致（来源：Goblin Tools 资料）。Saner.AI 通过本地记忆减少搜索循环（来源：Saner.AI 资料），对应 LLM 的向量数据库。Lex 用单一指令触发多步骤任务（来源：Lex 资料），类似 agent 的规划链。

## Otter.ai 的双面性：被动记录 vs 主动重锚定

Otter.ai 的默认用法是“录下来，以后查”。这对 ADHD 来说，是拐杖——它允许你放弃现场记忆，事后依赖回放。但问题在于，ADHD 的执行功能缺陷也包括“事后回顾”本身：你很可能不会回去查（来源：传统提醒系统需要主动设置，而这本身是执行功能任务——AI Assistant for ADHD 资料）。于是转录文件堆积，成为“被遗弃的规划器墓地”的另一种形式。

脚手架用法则不同：**把 Otter.ai 当作实时重锚定工具。** 会议中，当注意力漂移时，快速扫一眼实时转录，找回当前话题；会后，用 AI 生成的摘要作为“日课”回顾，类似孔子的“三省吾身”。这对应了 LLM harness 中的“提醒和通知提供重锚定，将注意力拉回原任务”（来源：重锚定与目标漂移资料）。Otter.ai 的搜索功能还能在下次会议前快速回顾上次决策，相当于 agent 的“会话状态记录”。

## 证据与局限：脚手架还是拐杖，尚无定论

支持脚手架的证据：AI 工具作为外部执行功能层补偿缺陷（来源：ADHD 的 AI 工具全景），用户反馈积极。Motion 通过自动排程帮助时间盲（来源：Motion 资料），ChatGPT 被用作“外部执行功能”（来源：ADHD 的 AI 工具全景）。Goblin Tools 的任务分解降低启动门槛（来源：Goblin Tools 资料）。

但拐杖风险真实存在：长期依赖可能导致能力退化（来源：矛盾与存疑资料）。目前缺乏纵向研究，不知道“用 AI 记”是否让工作记忆更弱。另一个矛盾：超聚焦应被引导还是打断？部分 ADHD 用户觉得强行打断（如番茄钟）有效，而柔性引导（Otter 的提醒）可能不够强力（来源：矛盾与存疑）。个体差异极大，没有标准答案。

## 今天就能试的 3 个行动

1. **把 Otter.ai 从“记录器”升级为“重锚定器”**：下次会议，打开实时转录窗口，每当你发现自己走神，看一眼转录文本，找到当前讨论点。会后，用 AI 摘要写一条“日课”笔记（一句话：今天会议的关键决策是什么？）。
2. **设置一个“回顾触发器”**：在日历上每周固定 15 分钟，用 Otter.ai 搜索关键词回顾上周会议。这模仿了 LLM 的“定时 re-grounding”，防止目标漂移。
3. **对比测试**：选一周，会议时只用大脑记（允许少量手写笔记）；下一周用 Otter 实时转录+会后摘要。对比哪一周的后续行动完成率更高。这能帮你判断自己的“脚手架 vs 拐杖”边界。

## 结论

Otter.ai 之于 ADHD，正如向量库之于 LLM：它是外部记忆，但能否成为脚手架，取决于你是否主动用它来重锚定。孔子和文天祥的 harness 告诉我们：**真正的脚手架不是替代你的大脑，而是给你的生成核心一个可靠的调度层。** 用 Otter 被动记录，你是拐杖用户；用它主动回顾、重锚定，你是自己的 agent 工程师。选择权在你手里。

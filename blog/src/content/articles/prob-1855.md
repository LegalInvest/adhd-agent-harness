# ADHD 职场人视角：为什么「治好 ADHD 的不擅长精确、重复、状态维护类任务」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 通过 function calling 把精确计算外包给工具——同一套 harness 思路，两边都成立。

## 问题：为什么你总是搞砸精确、重复、状态维护类任务？

如果你是一个ADHD职场人，你一定经历过这样的场景：打开一个Excel表格，打算更新上个月的销售数据，结果半小时后你发现自己正在研究某个无关的网页链接；或者你明明记得要发一封跟进邮件，但一转身就忘得干干净净。这类任务——精确计算、重复操作、维护状态——对你来说就像黑洞，吞噬着你的注意力和时间。你试过各种方法：待办清单、番茄钟、冥想，但效果往往昙花一现。

与此同时，如果你是一个在做Agentic Harness工程的工程师，你一定也经历过类似的崩溃：你精心设计了一个LLM agent，让它通过function calling去调用数据库查询或执行代码，结果它突然开始胡编乱造，或者忘记之前已经完成的任务，输出变得不可预测。你尝试降低采样温度、增加上下文长度，但agent还是会跑飞。

这两个问题看似风马牛不相及，但本文要论证的是：**它们本质上是同一道工程题**。ADHD大脑和LLM共享同一个核心缺陷——它们都是高产的生成核心，但缺乏可靠的执行调度层。因此，两者的解法同构：用外部脚手架（harness）把生成能力约束到目标轨道上。

## 同构脊柱：生成核心 vs 调度层

ADHD大脑常被描述为“高产生成核心”——想法丰富、联想活跃，但缺少稳定的调度层来筛选、启动和坚持任务（来源：ADHD的AI工具全景）。你的大脑里每分钟都在冒出创意，但当你需要执行一个精确、重复的步骤时，调度层就掉链子了。这种波动性在ADHD一侧表现为注意力、执行功能的日内波动，类似于一个没有温度调节的系统（来源：采样温度与表现波动）。

LLM同样如此。它的输出天然具有随机性，采样温度控制着多样性：温度越高，输出越随机；温度越低，输出越确定。这种非确定性在生产环境中是可靠性的敌人，因为“小错误会累积，非确定性使可重复性复杂化”（来源：AI Agent Systems: Architectures, Applications, and Evaluation）。因此，工程实践强调“确定性状态转换”和“计划-执行”分离，用外部工作流强制单向控制流（来源：AI Agents in 2026: Tools, Memory, Evals, and Guardrails | Andrii Furmanets；Planner-Executor Agentic Framework）。

你看，两边的问题一模一样：一个不稳定的生成核心，需要一个外部调度器来稳定输出。

## 人物案例：孔子的harness就是最早的agent脚手架

让我们看一个真实案例。孔子，儒家创始人，被后世尊为“万世师表”，但他很可能是一个ADHDer。他身高1米9，精力旺盛，周游列国14年根本坐不住；他冲动爱骂人，见南子急得对天发誓，骂宰予“朽木不可雕”；他对音乐超专注到“三月不知肉味”，但对种地等俗事零耐心。他的思维跳跃，《论语》全是场景化语录，没有系统著作。

孔子的harness是什么？是“克己复礼”。他用外在的“礼”作为行为边界，每天反省——“吾日三省吾身”。70岁才做到“从心所欲不逾矩”，一辈子和自己的冲动做斗争。这个“礼”就是他的外部调度器，相当于LLM agent中的“确定性工作流”：每天固定时间反省，用仪式化的行为稳定自己的输出。他的成就——影响中国文化2500年——恰恰来自这种“高产生成核心+外部脚手架”的组合。

现在看LLM agent的harness：scaffolding将LLM置于与记忆、工具和决策逻辑的控制循环中，通过赋予LLM访问工具、领域数据和结构化工作流来增强其裸能力（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。孔子的“日课”就是定时re-grounding，他的“秘书”（弟子）就是外部调度器。结构完全同构。

## 工具如何充当外部调度层

当前ADHD与AI工具领域的核心论点是：AI并不是在“治愈”ADHD，而是在ADHD大脑与外部世界之间搭建一个临时的外部执行功能层（来源：ADHD的AI工具全景）。这些工具承担四种功能：任务拆解、外部记忆、对话式执行、捕获与反思。

以Goblin Tools为例，它的Magic ToDo功能将复杂任务分解为可管理的小步骤，用户可调节“辣度”控制分解粒度（来源：Goblin Tools）。这相当于给ADHD大脑一个确定性的任务分解工作流，降低启动门槛。Saner.AI则通过本地记忆和快速检索减少搜索循环，补偿工作记忆的不稳定（来源：Saner.AI）。Lex允许用户通过单一指令触发多步骤任务序列，其底层架构类似于agent scaffolding（来源：Lex）。

在LLM一侧，同样有这些模式：任务拆解对应agent的plan-then-execute；外部记忆对应RAG和向量数据库；对话式执行对应function calling；捕获与反思对应trace和evaluation。两边用的都是同一套harness思路。

## 脚手架 vs 拐杖：一个必须指出的争议

这里有一个关键争议：AI工具是“外部执行功能层”还是“拐杖”？核心论点认为AI补偿缺陷，但过度依赖可能导致能力退化（来源：矛盾与存疑）。目前缺乏长期实证，需要更多纵向研究。我的判断是：**脚手架和拐杖的区别在于你是否保留了主动控制的意识**。如果你用工具来替代你的思考（比如完全让AI替你写邮件而不检查），那就是拐杖；如果你用工具来辅助你的调度（比如用Goblin Tools分解任务后自己执行），那就是脚手架。孔子的“礼”是脚手架，因为他每天主动反省；如果你只是被动依赖提醒，那就是拐杖。

## 今天就能试的行动

1. **用Goblin Tools分解一个你拖延的任务**：比如“清理厨房”或“写周报”，调节辣度到你觉得OK的粒度，然后按步骤执行（来源：Goblin Tools）。
2. **用Saner.AI建立你的“第二大脑”**：把散落的笔记、想法存入本地记忆，减少搜索循环（来源：Saner.AI）。
3. **为你的LLM agent设置确定性工作流**：在代码中强制“计划-执行”分离，不要让它自由发挥（来源：Planner-Executor Agentic Framework）。
4. **每天做一次“日课”式反省**：就像孔子一样，固定时间回顾今天的执行情况，并计划明天（来源：人物案例）。

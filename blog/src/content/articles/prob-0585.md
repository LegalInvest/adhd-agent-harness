# Tiimo 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Tiimo：用久了会让 ADHD 更强还是更依赖？

## 问题：工作记忆的「漏水桶」与 LLM 的「无状态」

ADHD 大脑最典型的困境是「边做边忘」——刚想好要做的三件事，转个身就只剩一件；开会时记下的灵感，散会后就像被橡皮擦抹去。这背后是工作记忆容量小的生理限制（来源：ADHD 的 AI 工具全景）。而 LLM/agent 世界有一个几乎一模一样的痛点：大语言模型本身没有跨会话状态，每次对话都是「失忆」重启，必须靠向量库、记忆模块等外部存储来续命（来源：无状态与外部记忆）。两个系统——一个生物，一个数字——共享同一个结构性缺陷：**它们是强大的生成核心，但缺乏可靠的内置执行调度层**（来源：ADHD 大脑与 LLM 的同构）。

Tiimo 正是针对这个「漏水桶」设计的工具。它通过视觉化日程、任务分解、定时提醒来补偿工作记忆的不足。但问题来了：用久了，用户会变得更独立，还是更离不开它？这正是「脚手架 vs 拐杖」的边界问题。

## 同构视角：ADHD 大脑与 LLM agent 共享同一套 harness 逻辑

把 ADHD 大脑和 LLM agent 并排放置，会发现惊人的结构同构：

| ADHD 大脑 | LLM/Agent |
|-----------|-----------|
| 工作记忆容量小，信息易丢失 | 无跨会话状态，每次对话从零开始 |
| 任务启动困难，需要外部触发 | 需要外部 prompt 或 agent 框架触发 |
| 目标漂移，被无关刺激捕获 | 上下文窗口超限后注意力偏移 |
| 超聚焦时产出极高 | 生成能力极强，但缺乏规划约束 |

解决思路也同构：**给生成核心套一个外部执行功能层**（来源：外部执行功能层）。对 ADHD 而言，这个 harness 可以是 Tiimo 的日程、Goblin Tools 的任务分解（来源：Goblin Tools）、Saner.AI 的知识回忆（来源：Saner.AI）；对 LLM agent 而言，则是 agent scaffolding（如 Lex 的「单一指令触发多步骤」架构，来源：Lex）、规划循环（如 Motion 的动态调度，来源：规划循环与任务分解）、以及外部记忆库（如 Mem 的持久记忆，来源：Mem）。

历史上有两位真实人物，用他们的生命实践了这种 harness 逻辑。

**孔子**——身高近一米九，精力旺盛到周游列国 14 年仍坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予「朽木不可雕」；对音乐超专注到「三月不知肉味」，对种地等俗事零耐心；思维跳跃，《论语》全是场景化语录，无系统著作。他的专属 harness 是「克己复礼」——用外在的「礼」作为行为边界，每日反省（「吾日三省吾身」），70 岁才做到「从心所欲不逾矩」。这套系统本质上就是**外部执行功能层**：日课相当于定时 re-grounding，礼相当于行为模板，三省相当于事后复盘。换成 LLM 术语，就是**定期上下文刷新 + 外部约束模板 + 验证回路**。

**孙策**——17 岁起兵，26 岁遇刺，9 年扫平江东。永远坐不住，身先士卒，单骑出猎被刺杀；冲动杀于吉，和太史慈单挑，想偷袭许昌。他的 harness 是「用张昭张纮等文臣管行政，自己只负责打仗」——把执行调度外包给「外部调度器」，自己只做高产生成（打仗）。这对应 LLM agent 的**工具调用 + 外部规划器**架构：agent 只负责生成核心内容，调度由外部 harness 控制。

这两个案例说明：**harness 不是天生就有的，是有意识地搭建的**。孔子用了大半辈子，孙策用了整个生命。Tiimo 的价值就在于它提供了一个现成的 harness，但用户必须主动使用它，并在这个过程中训练自己的元认知。

## Tiimo 是脚手架还是拐杖？边界在于「主动使用 vs 被动依赖」

脚手架的定义：**使用过程中提升能力，拆除后仍能独立行事**。拐杖的定义：**永远离不开，一旦撤除功能崩坏**。

从 Tiimo 的设计看，它具备脚手架潜质：
- 视觉化日程降低时间盲（来源：时间盲）
- 任务分解降低启动门槛（类似 Goblin Tools 的 Magic ToDo，来源：Goblin Tools）
- 提醒与重锚定对抗目标漂移（来源：重锚定与目标漂移）

但危险在于：**设置提醒本身就是一个执行功能任务**（来源：重锚定与目标漂移）。如果 Tiimo 的提醒需要用户事先手动配置，而用户因为工作记忆缺陷忘记配置，工具反而成了负担。更理想的是「无需主动设置的提醒」，但 Tiimo 目前主要依赖用户主动规划。

另一个危险是**过度依赖**。如果用户长期依赖 Tiimo 告诉自己「下一步做什么」，而不去练习自己分解任务、感知时间，那么内在的执行功能不仅不会提升，反而可能萎缩。这就像用导航仪太久，会失去认路能力。

LLM agent 侧有同样的风险：如果 agent 完全依赖外部记忆库和规划器，一旦这些组件失效，agent 就变成「失忆的生成器」。好的 agent harness 应该让 agent 在每次交互中「学习」一部分上下文管理能力（比如通过 prompt 压缩、关键信息提取），而不是永远被动地从向量库读取。

现有工具的效果多基于用户报告，缺乏严格临床验证（来源：矛盾与存疑）。Tiimo 本身在 wiki 资料中未被详细评估，但同类工具如 Goblin Tools 被评价为「简单有用」（来源：Goblin Tools），Saner.AI 被列为最佳 ADHD 应用之一（来源：Saner.AI）。然而，「同构性被过度推广」的争议同样适用于本文的类比——ADHD 大脑与 LLM 的同构目前仅是理论类比，并非科学事实（来源：矛盾与存疑）。读者应谨慎对待这一框架，将其视为启发式模型而非确凿真理。

## 今天就能试的行动

1. **用 Tiimo 规划明天，但刻意不打开它一次**：比如下午 3 点到 4 点关掉所有提醒，靠自己的记忆完成一项任务。事后复盘：我漂移了几次？怎么拉回来的？这训练内在调度。
2. **模仿孔子的「三省」**：每天睡前用 5 分钟，在 Tiimo 或任何笔记工具中回答三个问题：今天完成了什么？哪里被干扰了？明天怎么改进？这就是 agent 的「事后验证回路」。
3. **尝试 Lex 的单一指令**：如果你有重复性多步骤任务（如「写周报」），用 Lex 创建一个指令，让它自动分解执行。观察自己是否因此减少了启动焦虑，同时警惕是否开始依赖它来「替我想步骤」。
4. **对比 Tiimo 与 Goblin Tools 的任务分解**：拿同一个模糊任务（如「清理书桌」），先用 Tiimo 手动拆，再用 Goblin Tools 自动拆。感受两者对认知负荷的不同影响，并思考：哪种方式更能让我学会分解？

## 结论：Tiimo 是脚手架，但前提是你愿意练肌肉

Tiimo 本身是中性的。它能否成为脚手架，取决于用户是否把它当作**训练工具**而非**替代品**。孔子用了大半辈子「克己复礼」才内化，孙策把行政外包但保留了战略决策的核心能力。对于 ADHD 用户，Tiimo 应该是一个「外部执行功能训练器」——用它来降低启动门槛、对抗漂移，同时有意识地练习自己的任务分解和时间感知。对于 LLM agent 工程师，这意味着设计 harness 时要留出「学习接口」：让 agent 在每次执行中积累上下文管理经验，而不是永远依赖外部存储。**真正的脚手架，是那些让你在拆除后依然能站得更稳的东西**。

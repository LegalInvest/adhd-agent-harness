# ADHD 学生如何用「给生成核心套 harness」的思路应对考试与作业里的工作记忆容量小、边做边忘？

> 把 LLM 工程里的「LLM 无跨会话状态，靠外部记忆/向量库续命」迁移到ADHD 学生的考试与作业场景，双域证据作答。

## 引言：同一个问题，两个世界的痛

你有没有过这样的经历：打开课本，读了两段，然后发现自己在想晚饭吃什么；拿起笔写作业，写了两行，突然想起要回一条消息；终于坐下来复习，却完全记不起刚才看过的公式——就像大脑里的工作记忆每隔几秒就自动清空一次。如果你是个 ADHD 学生，这几乎是日常。而如果你是个做 Agentic Harness 的工程师，你一定对另一个场景再熟悉不过：LLM 每次对话都像失忆，上一轮给过的上下文，这一轮就忘了，必须靠向量库、记忆插件、提示工程来「续命」。

这两个世界看似毫无关联，但 wiki 资料揭示了一个惊人的同构：**ADHD 大脑与 LLM 都是「高产但缺执行调度层的生成核心」**（来源：ADHD 大脑与 LLM 的同构）。ADHD 的工作记忆「容量保留但控制弱」，LLM 同样「有强记忆容量但弱上下文控制」（来源：工作记忆）。那么，LLM 工程里用「agent harness」来包裹生成核心、提供外部记忆和调度的思路，能否迁移到 ADHD 学生的考试与作业场景？答案是肯定的。本文将用双域证据，论证如何用「给生成核心套 harness」的思路，应对工作记忆容量小、边做边忘的困境。

## 同构脊柱：无状态与外部记忆

核心概念很简单：**无状态**。LLM 本身没有跨会话状态，每次调用都是全新开始；ADHD 大脑的工作记忆也常常「掉线」，无法持久保存上下文。两者的解法也同构：**靠外部记忆/脚手架续命**。对于 LLM，是 agent harness——「包裹 LLM 或 AI agent 的软件基础设施，处理模型本身之外的一切」（来源：ADHD 大脑与 LLM 的同构）。对于 ADHD，是外部执行功能层——「通过分解任务、管理上下文、增强记忆和提供实时反馈，补偿内在的调度缺陷」（来源：外部执行功能层）。

## ADHD 侧：考试与作业中的「记忆掉落」

以考试为例。ADHD 学生经常在解题过程中忘记题目条件、忘记公式、忘记刚才算到哪一步。这不是智力问题，而是工作记忆的「弱控制」：信息进入后，容易被新刺激冲走。作业场景更典型：打开一篇论文，读了一段，走神，回来时已经忘了前文讲什么，不得不反复回读——这就是典型的「上下文丢失」。wiki 资料指出，ADHD 呈现出「强记忆、弱控制」的认知剖面：工作记忆容量可能正常甚至超常，但认知灵活性与注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

## LLM/Agent 侧：无状态 LLM 的「记忆掉落」

LLM 同样如此。在构建 agent 时，工程师发现 LLM 本身「无状态、仅生成文本」，需要外部 harness 来提供「工具接口、上下文管理、安全执行等编排能力」（来源：外部执行功能层）。例如，用 Git 仓库存储项目上下文，用 MCP 连接器访问外部数据（来源：Worker Agents | Harness Developer）。这些做法本质上就是为生成核心搭建「外部记忆系统」。

## 解决方案：给 ADHD 大脑套上 harness

具体怎么做？以下是三个与 agent harness 同构的策略：

### 1. 任务分解：Goblin Tools 的「魔法开关」

Agent harness 中，一个关键组件是任务分解：将复杂指令拆成可执行的小步骤。Goblin Tools 的 Magic ToDo 功能正是如此——「自动将任何任务分解为更小、更易管理的步骤」（来源：Goblin Tools）。对于 ADHD 学生，考试前用 Goblin Tools 将「复习数学」分解为「看公式列表→做三道例题→检查错误→总结」等步骤，能显著降低启动焦虑。用户反馈称其「将压倒性的事情变成一系列不压倒性的事情」（来源：Goblin Tools）。

### 2. 外部记忆：Saner.AI 与 Mem 的「持久化上下文」

Agent harness 依赖向量库来持久化上下文。Saner.AI 提供「知识回忆」和「本地记忆」，帮助用户「减少搜索循环和标签切换」（来源：Saner.AI）。对于作业场景，用 Saner.AI 记录阅读笔记、公式推导过程，下次打开时自动恢复上下文，就像 LLM 的「第二大脑」。Mem 则被描述为「AI 第二大脑」，记住项目上下文（来源：ADHD 的 AI 工具全景）。

### 3. 单指令多步骤：Lex 的「启动脚本」

LLM agent 中，一个 prompt 可以触发多步操作。Lex 允许用户「通过单一指令触发复杂、多步骤的任务序列」（来源：Lex）。ADHD 学生可以用 Lex 设定「写论文」指令，自动执行「打开大纲→写引言→写主体→检查引用」等步骤，减少决策疲劳。

## 核心观点：脚手架 vs 拐杖的边界

本文的核心判断是：**AI 工具作为 harness，应该像脚手架一样促进能力发展，而非永久拐杖**（来源：拐杖与脚手架）。但 wiki 资料也诚实指出矛盾与存疑：过度依赖 AI 工具可能「削弱内在时间感知能力」（来源：时间盲），且「现有证据主要来自用户报告，缺乏大规模对照实验」（来源：矛盾与存疑）。因此，关键在于有意识地使用——让工具帮你记住上下文，但不要让它替你思考。

## 结语：同构的力量

ADHD 学生和 LLM 工程师面对的是同一个问题：如何让一个强大的生成核心持续稳定地输出？答案也同构：给它一个 harness。当你在考试中忘记条件时，记住——你不是一个人在战斗，LLM 也需要同样的脚手架。而当你用 Goblin Tools 分解任务时，你正在实践最前沿的 agent harness 工程。这不仅是工具，更是一种认知方式的升级。

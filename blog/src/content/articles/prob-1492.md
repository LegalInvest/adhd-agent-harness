# Goblin Tools 之于 ADHD，就像 会褪去的脚手架 之于 LLM——但有人用错了

> 从同构视角实测 Goblin Tools：它到底补上了哪一层执行功能？

你有没有过这样的经历：打开 Goblin Tools 的 Magic ToDo，输入“整理房间”，看着它自动分解出 12 个小步骤，瞬间松了口气——但一周后，你发现自己离了这个工具就完全无法启动任何任务，甚至开始焦虑“如果 Goblin Tools 崩了怎么办”。另一边，工程师们在搭建 agent 系统时，拼命往 LLM 外面套各种脚手架：确定性工作流、状态机、工具契约——却担心一旦撤掉这些结构，agent 就会像 ADHD 大脑一样失控。

这两类人的焦虑，其实指向同一个问题：**我们到底是在用工具搭建一座会褪去的脚手架，还是给自己焊死了一副永久拐杖？**

## 同构的困境：高产核心 vs 缺失的调度层

ADHD 大脑与 LLM 在结构上惊人相似：两者都是“高产但缺执行调度层的生成核心”。ADHD 的情绪生成系统（杏仁核）反应强烈，但前额叶（执行功能中枢）调控不足；LLM 生成能力强，但缺乏输出控制，采样温度一高就随机发散。这种同构意味着，解决方案也应该是同构的——为两者搭建一个临时的、可褪去的调度层。

在 ADHD 侧，曾国藩用“日课十二条”给自己搭建了外部调度器：黎明即起、读书不二、谨言、写日记反省。这套系统本质上是一个确定性工作流——每天固定时间做固定的事，用最笨最稳的方法抵消自己的冲动。他打败仗就跳水自杀，情绪波动极大，但日课从未中断。这就像 LLM agent 中的“计划-执行”分离模式：先由 planner 制定确定性步骤，再由 executor 严格遵循，强制单向控制流（来源：Planner-Executor Agentic Framework）。曾国藩的日课，就是他的 planner；而他的日记，则是 trace 日志，用于事后复盘。

在 LLM 侧，工程实践强调“确定性状态转换”（来源：AI Agents in 2026: Tools, Memory, Evals, and Guardrails | Andrii Furmanets），最简单的 agent 工作流本身就是确定性的，遵循预定义步骤（来源：What Are Agentic Workflows? Patterns, Memory, Use Cases, and Examples | Weaviate）。这对应 ADHD 的“外部结构”（如日程、提醒、环境设计）——两者都通过外部约束来稳定输出。

## Goblin Tools 补上了哪一层？

Goblin Tools 的核心功能是 Magic ToDo：接受模糊任务（如“清理厨房”），将其分解为具体、可执行的子任务，用户还可调节“辣度”控制分解粒度（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。这直接补偿了 ADHD 的任务启动困难——将模糊的工作分解为更小的行动（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。

但问题在于：Goblin Tools 只补上了“任务分解”这一层，而非完整的执行功能层。ADHD 的执行功能缺陷还包括工作记忆不足、时间盲、情绪调节困难等。Goblin Tools 没有时间可视化功能（对比 Tiimo），没有 CBT 结构化程序（对比 Inflow），没有本地记忆存储（对比 Saner.AI）。它只是一个“任务分解助手”（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...）），而不是一个完整的 harness。

这就像 LLM agent 中，你只给 LLM 加了一个工具调用接口，却没有给它状态机、记忆缓冲区和 guardrails。单个 LLM 不足以可靠地完成多步骤任务（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI），同样，单个 Goblin Tools 也不足以支撑 ADHD 的全面执行功能。

## 脚手架 vs 拐杖：边界在哪里？

脚手架与拐杖的根本区别在于：脚手架是临时结构，旨在帮助使用者发展内在能力后自行拆除；拐杖则是永久替代，长期使用会导致肌肉萎缩。

在 ADHD 侧，过度依赖 AI 工具可能导致能力退化——如果你每次都让 Goblin Tools 帮你分解任务，你永远不会学会自己分解。在 LLM 侧，过度依赖 scaffolding 可能让 agent 失去泛化能力——如果所有控制流都是硬编码的，agent 无法适应新场景。

曾国藩的日课系统为什么不是拐杖？因为他最终内化了这套系统，晚年不再需要日记来约束自己。同样，Inflow 的设计理念是“教用户如何建立技能”（来源：The 12 Best Apps for ADHD in 2026: A Guide to Finding What ...），而非永久依赖。Goblin Tools 的开发者将其定位为“小型、免费的实用工具集”（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)），暗示它应是临时辅助。

但争议在于：目前缺乏长期追踪研究验证适度使用的边界（来源：拐杖与脚手架）。个体差异显著，有些用户可能需要永久辅助，有些则能逐渐脱离。

## 工程师的启示：harness 应设计为可褪去的

对于做 Agentic Harness 工程的读者，Goblin Tools 的案例提供了一个重要教训：**harness 应设计为可逐步移除的**。这意味着：

1. **确定性工作流应支持渐进式松弛**：初始阶段用严格的计划-执行分离，后期允许 agent 动态调整步骤。
2. **工具契约应包含“训练模式”**：让 agent 在沙盒中试错，而非直接依赖外部工具。
3. **周期性采样以监控质量**（来源：AI Agent Reliability and Guardrails 2026 | Zylos Research），就像曾国藩每天写日记反省——如果 agent 的 trace 日志显示它开始依赖某工具完成本应由自身推理完成的任务，就应调整 scaffold。

## 今天就能试的 3 件事

1. **用 Goblin Tools 分解一个你拖延已久的任务**，但刻意不保存结果——强迫自己记住分解步骤，训练工作记忆。
2. **对比 Inflow 的 CBT 模块与 Goblin Tools 的纯分解**：前者教你技能，后者帮你做事。记录一周后你离开工具时的启动能力变化。
3. **如果你是工程师**，为你的 agent 设置一个“褪去测试”：逐步移除 scaffolding 组件，观察 agent 的退化程度，找到最小必要结构。

本文的核心论点——Goblin Tools 作为可褪去的脚手架——建立在 ADHD 大脑与 LLM 同构的理论框架上，但该框架目前缺乏神经科学或计算机科学的直接证据（来源：ADHD 大脑与 LLM 的同构）。此外，Goblin Tools 对深层情绪问题（如创伤）的干预效果证据不足（来源：情绪复杂性）。最后，个体差异显著，有些用户可能需要永久使用类似工具，不应被“必须独立”的教条所束缚。

# Goblin Tools 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Goblin Tools：它到底补上了哪一层执行功能？

## 问题：为什么“把任务扔给 AI”常常失败？

你让 ChatGPT 帮你“整理本周待办”，它列出一份漂亮的清单——然后你关掉窗口，什么都没做。你让 Goblin Tools 把“写报告”分解成 10 个小步骤，你盯着第 1 步“打开电脑”，依然不想动。这不是工具不好，而是你用错了方向。

ADHD 大脑与 LLM 在结构上同构：两者都是“高产但缺乏可靠执行调度层的生成核心”（来源：ADHD 的 AI 工具全景）。LLM 能生成流畅文本，但如果没有 function calling 和外部工具，它无法准确计算 2+2，也无法记住对话历史。ADHD 大脑能爆发式产出创意，但如果没有外部脚手架，它难以启动任务、保持专注、按时交付。Goblin Tools 的 Magic ToDo 把“清理厨房”分解成 15 步，就像 LLM 的 function calling 把“计算 2+2”外包给计算器——它补上了精确执行层，但前提是你得调用它，并且信任它。

## 同构：ADHD 大脑与 LLM 共享的执行缺陷

ADHD 的执行功能缺陷，核心是工作记忆不稳定、任务启动困难和注意力分散（来源：外部执行功能层）。这与 LLM 的“无状态”和“缺乏内置调度”在结构上同构：LLM 每次对话都是新的，需要外部记忆和 prompt 工程来维持上下文；ADHD 大脑每天醒来都要重新回忆“我昨天做到哪了”，需要外部工具来存储进度和下一步行动（来源：无状态与外部记忆）。

研究显示，ADHD 组在超专注工作时长上平均为 14.5 小时/周，而对照组为 8.2 小时（来源：Pace Ventures Research 2026）。这说明 ADHD 不是“不能专注”，而是“无法可靠地调度专注”。同样，LLM 在没有 function calling 时，连简单的数学运算都会出错，但一旦接入工具，就能完成复杂推理链。

## 真实人物的 harness：孔子与老子的外部调度层

孔子是典型的 ADHD 大脑：身高 1 米 9，精力旺盛，周游列国 14 年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予“朽木不可雕”；对音乐超专注到“三月不知肉味”，对种地等俗事零耐心；思维跳跃，《论语》全是场景化语录，无系统著作。他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。这相当于 LLM 的“系统 prompt + 每日 re-grounding”：通过固定仪式（三省）来重置上下文，避免思维漂移。70 岁才做到“从心所欲不逾矩”，一辈子在和自己的冲动做斗争。

老子的 harness 则相反：他做了半辈子图书馆长，说走就走骑青牛出关，连书都不想留；思维极度发散非线性，《道德经》“正言若反”。他的解法是“无为”——不强行做事，把所有能量收回来聚焦在“道”上。这对应 LLM 的“最小干预策略”：减少不必要的工具调用，让生成核心自由发挥。两种 harness 都有效，但适用于不同场景——孔子模式适合需要外部纪律的任务（如项目管理），老子模式适合创意探索（如头脑风暴）。

## Goblin Tools 到底补上了哪一层？

Goblin Tools 的核心功能是任务分解、语气检查和时间估计（来源：Goblin Tools）。Magic ToDo 接受模糊任务（如“清理厨房”），将其分解为具体子任务，用户可调节“辣度”控制粒度（来源：Goblin Tools）。这相当于 LLM 的 function calling：将“计算销售额”分解为“查询数据库→求和→返回结果”。但问题在于，ADHD 用户常常只用了分解，却没有建立“调用→执行→反馈”的闭环。

真正的 harnees 需要三层：1）任务分解（Goblin Tools 提供）；2）外部记忆（如 Obsidian 或 Saner.AI 的本地记忆，减少搜索循环，来源：Saner.AI）；3）执行触发器（如 Lex 的“单一指令触发多步骤”，来源：Lex）。Saner.AI 通过知识回忆减少标签切换，Lex 通过一次指令启动复杂序列，两者都补上了“调度层”而非仅仅是“分解层”。

## 争议与局限：脚手架还是拐杖？

同构性目前只是一种理论类比，缺乏实证基础（来源：矛盾与存疑）。Goblin Tools 的效用主要基于用户报告，缺乏严格临床验证。更关键的是依赖风险：如果用户永远依赖 AI 分解任务，可能削弱内在的执行功能（来源：拐杖与脚手架）。孔子用了 50 年才内化“礼”，老子则选择不内化——两者都是 conscious choice，而非被动依赖。

此外，现有工具多为通用设计，缺乏针对 ADHD 亚型的个性化调整（来源：矛盾与存疑）。Goblin Tools 的“辣度”调节是个好思路，但远远不够。对于时间盲严重的用户，还需要时间估计和视觉化工具（来源：时间盲）。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个你拖延已久的任务**，但不要止步——把每个子步骤写进 Obsidian 或笔记应用，作为外部记忆。
2. **设置一个“每日 re-grounding”仪式**：像孔子一样，每天固定时间用 5 分钟回顾上下文，更新待办。可以使用 ChatGPT 的 system prompt 来初始化。
3. **尝试 Lex 的“单一指令”模式**：把一个多步骤流程（如“写周报”）封装成一个指令，减少启动摩擦。
4. **警惕过度依赖**：每周至少一次，不借助 AI 完成一个简单任务，锻炼内在执行功能。

Goblin Tools 不是万能药，它是 function calling 的 ADHD 版——补上精确执行层，但调度层还得你自己搭。孔子和老子都用了一辈子才找到自己的 harness，你不需要那么久。从今天的一个小分解开始，但别忘了，工具是脚手架，不是拐杖。

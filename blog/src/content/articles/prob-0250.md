# Goblin Tools 之于 ADHD，就像 上下文工程 之于 LLM——但有人用错了

> 从同构视角实测 Goblin Tools：它到底补上了哪一层执行功能？

你打开一个空白文档，光标闪烁，大脑却像被无数根线拽向四面八方——刷一下朋友圈、查一下邮件、整理一下桌面。十分钟过去，文档还是空白。这不是意志力问题，而是你的大脑像一台没有操作系统的超级计算机：算力惊人，但不知如何调度资源。

同样的困境，正在折磨另一群人：他们在调试一个大型语言模型（LLM），发现模型输出总被上下文窗口里的无关信息带偏，明明能力够，结果却像脱缰野马。他们尝试各种提示词工程、上下文压缩、动态检索，只为让模型“专注”在当前任务上。

这两群人，一个面对的是生物神经网络，一个面对的是人工神经网络，但核心问题惊人相似：**都是高产但缺乏可靠执行调度层的生成核心**。ADHD 大脑与 LLM 的深层同构，使得 AI 工具对 ADHD 的辅助，本质上就是给这个生成核心套上 harness（外部执行功能层）——正如上下文工程之于 LLM。

## 同构：两个世界的同一道坎

先看 ADHD 侧：注意力容易被环境带偏，工作记忆容量有限，时间感知模糊（时间盲）。这些执行功能缺陷导致一个典型场景：任务启动困难、中途分心、难以估计时间。ADHD 大脑不是没有创意和生产力，而是缺乏一个“调度员”来告诉它：现在该做什么，做到什么时候，如何不被干扰拉走。

再看 LLM 侧：模型本身是强大的文本生成器，但它的“注意力”完全受上下文窗口内容左右。如果上下文里塞满了无关指令、历史对话或冗余信息，模型的输出就会偏离目标。工程师们因此发明了各种上下文工程技巧：精简提示、分步指令、动态上下文管理、外部记忆（如向量数据库）。这些技巧的本质，就是为 LLM 搭建一个外部调度层，补偿它“无状态”的缺陷。

这种同构在多个层面成立：ADHD 的工作记忆缺陷与 LLM 的无状态性都需要外部记忆系统（如第二大脑、向量数据库）来维持连续性；ADHD 易被环境带偏，LLM 易被上下文膨胀干扰，都需要主动设计“当下注意什么”的工程方案（来源：AI 与 ADHD 的专注力）。

## Goblin Tools：它补上了哪一层？

在众多 AI 工具中，Goblin Tools 被 ADHD 社区广泛使用，其核心功能“魔法待办”能将复杂任务拆解为小步骤，降低启动门槛。这恰恰对应了任务启动这一执行功能短板。ADHD 患者的多巴胺系统失衡导致行动动机不足，工作记忆容量有限使得任务步骤难以在脑中清晰呈现，而 Goblin Tools 通过外部化任务分解，将“开始”这个抽象指令转化为一系列具体可执行的动作（来源：任务启动）。

从同构视角看，Goblin Tools 的角色就像 LLM 上下文工程中的“分步提示”：把一个大问题拆成多个小问题，让模型一步步回答，而不是一次性输出。这减少了上下文中的认知负荷，提高了输出稳定性。

但问题来了：很多人把 Goblin Tools 当作全能的执行功能替代品，期望它解决所有问题——时间盲、注意力维持、情绪调节。实际上，它只补上了任务启动与分解这一层。对于时间盲，它缺乏像 RescueTime 那样的自动时间追踪和可视化报告（来源：RescueTime）；对于注意力维持，它没有 Focusmate 那种实时同伴问责带来的身体在场效应（来源：Focusmate）；对于深度专注，它不像 Brain.fm 那样通过神经锁相技术调节大脑状态（来源：Brain.fm）。

## 有人用错了：脚手架 vs 拐杖

这里必须点明一个关键边界：脚手架 vs 拐杖。

脚手架是临时性的，帮助你在能力不足时完成工作，同时允许你逐步内化技能；拐杖则是永久依赖，一旦拿走就寸步难行。Goblin Tools 的设计初衷是脚手架——它帮你拆解任务，但如果你永远不学习自己拆解，它就变成了拐杖。

同样，在 LLM 上下文工程中，过度依赖外部提示模板而不理解模型行为，也会导致“提示工程依赖症”：换一个场景就不会写提示了。好的上下文工程应该让模型在减少提示的情况下仍能稳定输出，就像好的 ADHD 工具应该帮助用户逐渐内化执行功能，而不是永远挂在外部系统上。

现有证据也支持这种谨慎态度。多个资料指出，过度依赖 AI 工具可能削弱内在时间感知能力（来源：时间盲），且存在依赖风险（来源：任务启动）。工具设计者声称是“补偿”而非“替代”，但缺乏如何避免依赖的具体指导（来源：矛盾与存疑）。

## 争议与局限

必须诚实指出，Goblin Tools 的证据强度并不高。多数工具仅有用户报告或小样本研究，缺乏大规模随机对照试验。ADHD 的异质性使得同一工具对不同亚型效果迥异，Goblin Tools 的魔法待办对任务启动困难者有效，但对时间盲或注意力维持困难者可能帮助有限（来源：矛盾与存疑）。

此外，ADHD 大脑与 LLM 的同构是隐喻而非严格等价。LLM 的“注意力机制”是数学计算，人类注意力是神经递质调节的复杂过程，两者本质不同（来源：AI 与 ADHD 的专注力）。因此，类比虽有用，但不能机械套用。

## 今天就能试的行动

1. **用 Goblin Tools 拆解一个你拖延了一周的任务**：把任务输入魔法待办，看它分解成几个步骤。然后只做第一步。如果做完第一步后自然想继续，说明它帮你突破了启动障碍。
2. **结合 RescueTime 或类似工具记录一天的时间使用**：看看你的注意力漂移模式。注意：不要追求完美分类，重点是发现“哪个时段最容易分心”。
3. **尝试一次 Focusmate 会话**：预约一个 25 分钟的虚拟身体在场时段，处理一个中等难度的任务。体验外部问责带来的启动推力。
4. **为自己设计一个“上下文工程”实验**：选择一个需要专注的任务，主动清理桌面、关闭无关标签页、设置一个计时器（比如 25 分钟）。观察对比有/无这种“精简上下文”时的注意力差异。

Goblin Tools 是一把好用的螺丝刀，但不是万能工具箱。理解它补上的是哪一层执行功能，以及何时该换用其他工具，才是真正有效的使用之道。对于 ADHD 大脑和 LLM 而言，最好的 harness 不是最强大的，而是最精准匹配当前调度需求的。

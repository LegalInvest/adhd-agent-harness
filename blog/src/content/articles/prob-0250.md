# Goblin Tools 之于 ADHD，就像 上下文工程 之于 LLM——但有人用错了

> 从同构视角实测 Goblin Tools：它到底补上了哪一层执行功能？

你有没有过这样的体验：打开一个 AI 工具，打算用它来整理待办清单，结果半小时后，你还在调整提示词、尝试不同功能，原本的任务纹丝未动？对 ADHD 大脑来说，工具本身常常变成新的注意力黑洞。这正是 Goblin Tools 试图解决的问题——但很多人用错了它，就像 LLM 工程师用错了上下文工程一样。

## 问题：工具变成了新的上下文碎片

ADHD 大脑与 LLM 共享一个核心困境：两者都是“高产但缺乏可靠执行调度层的生成核心”（来源：AI 与 ADHD 的专注力）。对 ADHD 个体，这意味着创意奔涌但难以落地；对 LLM，这意味着文本流畅但容易被上下文膨胀带偏推理。

许多 ADHD 工具在设计时假设用户能维持专注、记住昨天的问题并执行计划，但实际用户往往做不到（来源：外部执行功能层）。更糟的是，通用聊天机器人“often add another inbox, another prompt, and another place to lose the thread”（来源：上下文工程），让注意力碎片化。Goblin Tools 的初衷是反其道而行：它用极简的界面、预设的“魔法”按钮（如将凌乱笔记转为待办、估算任务耗时），试图降低启动门槛。但问题在于，用户往往把它当作一个“自动完成一切”的拐杖，而不是一个上下文管理工具。

## 同构解：Goblin Tools 是上下文工程，不是自动导航

上下文工程（Context Engineering）的核心是“主动设计和管理系统当前所关注的信息范围”（来源：上下文工程）。对 LLM/agent 而言，这意味着防止上下文膨胀和推理退化；对 ADHD 大脑而言，这意味着防止环境刺激劫持注意力。

Goblin Tools 真正有效的用法，不是让它替你做所有决策，而是把它当作一个**外部上下文锚点**。它的“魔法”功能（如将混乱想法结构化）相当于为 LLM 提供一段精心设计的 system prompt，把散乱的上下文聚拢到当前任务上。它的耗时估算功能相当于一个时间感知的 grounding 机制——就像 RescueTime 通过自动记录帮助用户对抗时间盲（来源：RescueTime）。

但这里有一个关键区别：Goblin Tools 不提供实时问责或身体在场效应（如 Focusmate 所做的那样），也不像 Brain.fm 那样通过神经锁相调节大脑状态（来源：Brain.fm）。它的优势在于**低摩擦的上下文压缩**——把一团乱麻变成可操作的步骤。这正是它作为上下文工程工具的价值，而不是作为执行替代品。

## 历史回响：曾国藩的“日课十二条”就是他的上下文工程

晚清名臣曾国藩年轻时是典型注意力不集中型 ADHD：浮躁坐不住，读书“他人目下二三行，余或疾读不能终一行”，日记里天天骂自己“无恒”。他的解决方案是“日课十二条”：黎明即起、读书不二、谨言、写日记反省——一套固定的日常仪式，相当于为大脑预设了上下文切换的规则。

这套“日课”与 LLM 的上下文工程惊人同构：
- **定时 re-grounding**：每天固定的反省和计划，相当于 LLM 的定期上下文重置，防止注意力漂移。
- **外部调度层**：用“结硬寨打呆仗”的笨方法抵消冲动，相当于用外部规则（如 MCP 连接器）约束 agent 的行为。
- **自我监控**：写日记相当于记录执行日志，与 RescueTime 的时间追踪异曲同工。

曾国藩的日课不是拐杖——它没有替他做决策，而是帮他管理“当下注意什么”。这正是 Goblin Tools 应该扮演的角色：一个上下文工程工具，而不是自动执行引擎。

## 争议与边界：脚手架 vs 拐杖

Goblin Tools 被用错的根源在于混淆了“脚手架”与“拐杖”。脚手架是临时结构，帮助用户学习如何自己搭建；拐杖则是永久替代，长期使用可能导致内在执行功能退化（来源：矛盾与存疑）。当前缺乏纵向研究来界定这个边界，但一个合理的判断是：如果工具让你完全跳过思考过程（比如直接生成你需要的所有输出而不经审视），它就是拐杖；如果工具帮你把混乱的上下文整理成可操作的起点，它就是脚手架。

另一个争议是超聚焦的引导问题：Goblin Tools 的“魔法”可能打断用户的超聚焦状态，而有些用户需要的是“gentle structure”而非强行打断（来源：矛盾与存疑）。这提示我们，上下文工程需要个性化——对某些人，压缩上下文是救命稻草；对另一些人，它可能成为干扰。

## 今天就能试的具体行动

1. **用 Goblin Tools 做上下文压缩，不做决策**：当你有一堆混乱想法时，用它的“魔法”功能生成待办列表，但**不要直接执行**——花 2 分钟审视列表，调整优先级。这相当于 LLM 的“prompt 审查”步骤。
2. **设置定时上下文重置**：像曾国藩的日课一样，每隔 90 分钟用 Goblin Tools 的“估算耗时”功能复盘时间盲区，结合 RescueTime 的数据（来源：RescueTime）校准时间感知。
3. **搭配身体在场工具**：Goblin Tools 缺乏实时问责，可以搭配 Focusmate 的虚拟身体在场（来源：Focusmate），让外部调度层更完整。
4. **警惕工具成为新上下文碎片**：如果使用 Goblin Tools 超过 10 分钟还没开始实际工作，立即关闭它——这提示你正在用工具逃避任务。

Goblin Tools 不是万能药。它是一把扳手，不是自动修车机器人。对 ADHD 大脑和 LLM 来说，真正的解法不是寻找一个能搞定一切的“超级工具”，而是学会主动设计自己注意什么——这就是上下文工程的本质。

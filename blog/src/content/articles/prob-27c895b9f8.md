# ADHD 程序员视角：为什么「治好 ADHD 的注意力容易被环境带偏」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 受上下文窗口内容左右，需要上下文工程——同一套 harness 思路，两边都成立。

## 同一个问题：谁的上下文又跑偏了？

想象两个并行的周一早晨。

A 是 ADHD 程序员。他打开电脑，刚写三行代码，Slack 弹了一条通知。他安慰自己「只回一句」，再抬头时二十分钟已过，而刚刚那行函数的逻辑早已断线。他不是不想专注，而是注意力被环境里的每一个微刺激轻易改道。

B 是一个 LLM Agent。它拿到任务后，上下文窗口里塞满了用户三天前的需求、上一轮工具返回的冗余日志、一段被误当作当前指令的旧 system prompt。于是它开始优雅地「跑飞」——输出很长，却离目标越来越远。

两件事看起来一生物、一机器，但问题结构完全相同：**一个高产但缺乏可靠调度层的生成核心，正在被上下文环境带偏**。ADHD 的注意力是上下文窗口，外部刺激是 prompt 噪声；LLM 的上下文窗口是注意力，历史消息与无效信息就是它的环境噪声。治好 ADHD 的环境敏感，与让 LLM 不跑飞，其实是同一道工程题：上下文工程。

## 同构：为什么两边都是「生成核心 + 外部 harness」

主题综述里提出的一个核心判断是：ADHD 大脑与 LLM 在结构上可以同构地理解为「强大的生成核心，但缺少可靠的内置执行调度层」（来源：AI 与 ADHD 的专注力）。

ADHD 侧的调度缺陷非常具体：工作记忆不稳定、时间盲（time blindness）、任务启动困难。ADHD 个体常常「无法感知时间流逝」，坐下后「真的感觉不到 45 分钟已经过去」（来源：时间盲）。同时，「眼不见，心不烦」的认知模式让任务与承诺迅速从记忆中消失（来源：时间盲）。

LLM 侧也有对应缺陷：没有跨会话的持久状态，上下文窗口膨胀会让推理退化，旧指令会被新噪声稀释。因此，它同样需要外部记忆、调度与重定向机制。

同构不是等同。Wiki 里的「全局矛盾与存疑」也提醒我们：ADHD 大脑与 LLM 的同构目前只是功能层面的理论类比，并非经过神经机制验证的科学事实。这一点不能忘。但它足够作为解题框架：两边都需要一套 **harness**——不是依赖，而是把外部结构当作脚手架，补足内部调度层。

## ADHD 侧证据：外部执行功能层长什么样

现有工具已经勾勒出一个「外部执行功能层」的轮廓：

- **RescueTime**：自动追踪时间并阻断分心网站，直接对抗「时间盲」和「执行功能」缺陷，让 ADHD 用户看到「时间到底去了哪里」（来源：RescueTime）。
- **Focusmate**：用虚拟身体在场（Body Doubling）提供同伴问责。资料显示，身体在场能激活多巴胺通路，定期问责检查可把目标达成率从 25% 提升到 95%（来源：身体在场效应）。
- **Brain.fm / Endel**：用 AI 生成专注音乐或个性化声音环境，主动设计「当下该听什么」来降低环境干扰（来源：Brain.fm；AI 与 ADHD 的专注力）。
- **Forest / Tiimo**：用游戏化番茄钟或视觉化时间表把抽象时间变成可触摸的界面，帮助用户从「无限延迟」转向「开始行动」（来源：AI 与 ADHD 的专注力；时间盲）。

这些工具不是让 ADHD 大脑「变正常」，而是绕过脆弱的内部调度，把外部世界变成可执行的上下文。但要诚实：多数证据来自用户报告和小样本研究，缺乏大规模随机对照试验；Brain.fm 和 Endel 在 ADHD 人群中的独立临床证据也有限（来源：全局矛盾与存疑；Brain.fm）。

## 人物案例：古人的 harness 与 LLM harness 同构

从曾国藩身上能看到最清晰的对应。他年轻时正是 ADHD 典型的「注意力缺陷型」画像：30 岁前坐不住、爱串门喝酒、读书慢到「他人目下二三行，余或疾读不能终一行」；情绪不稳，打败仗就跳水。他的 harness 是「日课十二条」——黎明即起、读书不二、谨言、写日记反省，以及「结硬寨，打呆仗」。

曾国藩的「日课十二条」对应 LLM 的 **定时 re-grounding**：每天固定时间重新校准目标与状态，相当于给系统 prompt 做刷新。写日记反省对应 **scratchpad / 反思机制**：把内部状态外化，防止被当下情绪或噪声带偏。他的幕僚与家书体系则对应 **外部 scheduler / 监督器**：不是让他自己调度，而是让结构帮他调度。

孔子也呈现出类似 harness。他身高一米九、精力旺盛、冲动爱骂人，对音乐能专注到「三月不知肉味」，对俗事却毫无耐心。他的解法是以「克己复礼」为核心，把外在的「礼」变成行为边界，并用「吾日三省吾身」做每日重定向。这里的「礼」就是系统提示词（system prompt），「三省」就是 re-grounding。两人都说明： harness 不是道德自律，而是给易漂移的生成核心加一层外部约束。

## LLM 侧：上下文工程就是 Agent 的 harness

对工程师而言，这一同构立刻有操作性。

LLM 的「注意力被环境带偏」表现为：上下文窗口里历史消息、工具返回、用户闲聊混在一起，当前任务信号被淹没，导致模型偏离目标。解决它不是让模型「更聪明」，而是做上下文工程——主动决定窗口里应该保留什么、丢弃什么、何时刷新。

具体做法与 ADHD 工具一一对应：

1. **外部记忆**：像 RescueTime 给 ADHD 提供时间记录一样，给 LLM 加检索增强（RAG）或外部记忆库，避免把所有历史塞进上下文。
2. **系统提示词**：像孔子的「礼」一样，在 system prompt 里写死行为边界、输出格式、决策优先级，让它成为不可被对话覆盖的硬约束。
3. **定时 re-grounding**：像曾国藩的日课，设计固定节点让 agent 重新陈述目标、检查当前任务与计划的偏差。
4. **scratchpad / 反思**：像写日记，把中间推理、工具结果、自我评估外化到结构化工作区，而不是让模型在内部「胡想」。

这些都是工程化的「外部执行功能层」。

## 拐杖还是脚手架？边界与诚实

同构框架最有价值的地方，也最容易被误用。它可能会让人以为 AI 可以永久替代 ADHD 用户的执行功能，或者以为一个复杂的 system prompt 能解决所有 agent 跑飞问题。Wiki 的「全局矛盾与存疑」多次警告：AI 工具既可能是**脚手架**，也可能退化成**拐杖**——长期依赖后无法内化执行功能（来源：全局矛盾与存疑）。

 ADHD 侧，工具证据不足、个体差异大、亚型未分层。 LLM 侧，上下文工程也不是万能药：它治不好模型本身的推理能力上限，也替代不了任务分解、错误恢复与人在回路（human-in-the-loop）。

所以核心判断应该是：harness 不是治病，也不是让模型成神；它是在内部调度层不足时，用外部结构把生成核心稳定在目标轨道上。两边都要不断问：这个结构是在帮我/它逐渐独立，还是让我们永远离不开它？

## 今天就能试的 4 个行动

1. **ADHD：给自己加一个「外部时间锚」**。用 RescueTime 或 Forest 自动记录时间，把「感觉过了 10 分钟」替换成真实数据，先让时间可见。
2. **ADHD：用身体在场降低启动摩擦**。用 Focusmate 这类虚拟同伴工具，把「一个人开始」变成「有人在场一起开始」，绕过任务启动困难。
3. **LLM 工程师：给 agent 写一份不可覆盖的 system prompt 边界**。明确目标、输出格式、禁止

# 用 ChatGPT 给 ADHD 补上上下文工程，真的能像 agent 那样工作吗？

> 他是被 agent 的一个能力馋到的:看 Devin 类系统干活,最羡慕的不是它写代码,是它**永远知道自己在哪**——上下文里始终装着任务目标、进度、下一步,中断了能接上,跨天了不迷路。而他自己的工作状态是:每次被打断,回来都像失忆——「我刚才在干嘛?为什么开着这个文档?」于是他照猫画虎:用 ChatGPT 维护一个「任务上下文」,随时更新进度,被打断后回来问一句「我到哪了」。两个月后他的评价很微妙:「有用,但和 agent 的体验根本是两回事——**agent 的上下文是自动跟着任务走的,我的上下文靠我手动喂,而『记得喂』本身就是我最不擅长的事。**」

收敛:上一篇回答了「该外包哪层」(劳动 vs 图式),这一篇回答工程可行性——**人肉复刻 agent 的「持续上下文」到底卡在哪(和「验证循环」篇的断点是不是同一个)?卡点之下,可行的最小系统长什么样?**

## 穿透:又是那个看不见的组件——但这次它叫「状态自动同步」

先定位断点。agent 的上下文体验由三个机制合成:**状态自动写入**(每步操作后,框架自动把结果追加进上下文——agent 从不需要「记得记录」)、**状态自动注入**(每次继续前,框架自动把当前状态喂给模型——也从不需要「记得回顾」)、**单线程串行**(一个 agent 实例通常专注一条任务线,上下文不被无关内容冲刷)。他的人肉系统三个都缺:写入靠手动(「记得更新进度」——前瞻记忆的活)、注入靠手动(「记得去问」——同一口井)、而他的真实生活是十几条任务线并发,上下文互相冲刷。**结论和「验证循环」篇同构:断的都不是智能环节,是自动化环节——那边断在「循环的调用器」,这边断在「状态的同步器」。**这两篇合起来指向同一条工程定律:**凡 agent 体验里「理所当然」的部分,都是框架代码在买单;人肉复刻时,这些部分必须显式采购,且不能用前瞻记忆支付。**

可行的最小系统,围绕「把同步挂在已有事件上」设计(而不是靠记得):①**写入挂在中断上**——立一条唯一的肌肉记忆:任何被打断的瞬间(电话响、有人喊、要去开会),离开前 10 秒钟在任务旁写一行「断点便签」:「正在做X,下一步Y」——不是记给系统的,是记给两小时后那个失忆的自己;这是全系统性价比最高的一个动作(agent 的 checkpoint 机制的人肉最小版)。②**注入挂在返回上**——回到工位的固定第一动作:读断点便签(不是查手机、不是看消息——那是上下文冲刷器);如果用 ChatGPT 维护任务状态,「回来先问我到哪了」要绑定到物理锚点(坐下=打开那个对话)。③**并发降载**——接受人肉系统撑不起十几条活跃线:显式区分「活跃线」(最多 2–3 条,配完整断点协议)和「冷冻线」(其余全部,只留一行状态躺在清单里,不占用心理上下文)——**agent 靠多实例解决并发,人靠砍并发解决并发。**

至于 ChatGPT 在这套系统里的位置,诚实的答案是:**配角。**它适合做每周的状态汇总(把一周的断点便签喂给它,生成各任务线的状态总览)和冷冻线的解冻简报(「给我讲讲三周前那个项目到哪了」)——这两个场景它比纸质系统强;但高频的断点写入/读取,纸和便签的摩擦更低、且永不宕机。**「像 agent 那样」的正确期待不是体验复刻,是体验的廉价近似:失忆依旧会发生,但每次失忆的恢复成本从二十分钟降到二十秒。**

## 验证

两周可测,指标就一个:**中断恢复时间**(从坐回来到重新进入工作状态的时长)。基线先测三天(多数 ADHD 者在 10–30 分钟),然后上断点便签协议,看它降到多少。可证伪:若恢复时间没降,检查便签是否写对了(「正在做X+下一步Y」是必需结构,只写「做报告」等于没写);若便签总是忘写,把触发再物理化(显示器边贴一张「要走?先写断点」的荧光贴)——协议失败的原因排查完之前,不要判协议死刑。

## 决策

做什么:装三件套(断点便签/返回首读/活跃线限3);ChatGPT 用在周汇总和解冻简报;把「中断恢复时间」当作你执行系统的核心健康指标,每月抽测。

不做什么:不要追求「自动同步」的完美复刻(市面上号称自动追踪你工作状态的工具,要么侵入性大要么不准——手动的 10 秒便签是当前的帕累托最优);不要开超过 3 条活跃线(那是系统崩溃的头号原因,没有之一)。

先做什么:现在就写你的第一张断点便签格式模板:「正在做___,下一步___」——贴在你下次起身必经的位置。

## 边界

中断后恢复成本高、前瞻记忆弱有 ADHD 文献支撑(GRADE 中);「状态同步器」为 B 级架构类比,三件套为实践设计(本文未做正式 A/B/C 分级)。中断密度极高的工作环境(客服/急诊类)可能超出任何个人协议的承载,那是环境问题,值得在职业层面正视而非独自硬扛。

## 今天就能试的 3 件事

1. 测你的基线:今天每次被打断,记录恢复用了几分钟——先看清成本。
2. 断点便签协议今天上线:离开前 10 秒,「正在做X,下一步Y」。
3. 盘点任务线:选出 2–3 条活跃线,其余今天正式冷冻(写一行状态,放进清单,心理上下线)。

本文服务于人生 Harness 金字塔的**执行层**:agent 永远知道自己在哪,是因为有代码替它记得——你也可以永远知道自己在哪,只要承认那行「正在做X,下一步Y」,就是你给自己写的代码。

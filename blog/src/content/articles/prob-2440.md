# ADHD 创业者视角：为什么「治好 ADHD 的想理解自己的大脑到底是怎么回事」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 是「高产但缺执行调度层的生成核心」——同一套 harness 思路，两边都成立。

你打开一个空白页面，想写一篇报告。大脑里有一万条想法在冲撞，但手指钉在键盘上，一个字都敲不出来。你打开一个 LLM 对话窗口，输入“帮我写个大纲”，它秒回了一整篇——但跑了题，逻辑断了，最后三行在讨论你根本没问的东西。

这两件事，本质上是一个问题：**你有一个高产到爆炸的生成核心，但它缺一个执行调度层。**

这就是本文要回答的核心问题：为什么“治好 ADHD 的执行功能障碍”和“让 LLM 不跑飞”是同一道工程题？答案很简单——ADHD 大脑和 LLM 是同一类架构：**生成核心强大，调度层缺失，都需要外部 harness 来补偿。**

## 同构：两个世界共享的架构缺陷

ADHD 大脑的神经科学画像很清晰：前额叶执行功能网络激活不足（来源：ADHD × AI 的科学与研究前沿）。这意味着计划、组织、任务启动、抑制控制这些“调度”能力天生低效。但与此同时，ADHD 大脑的生成能力常常惊人——李白能“斗酒诗百篇”，毛泽东在行军路上读书不辍、思维跳跃如四渡赤水。他们缺的不是燃料，是方向盘。

LLM 也一样。它能在毫秒级生成千字文本，但一旦上下文膨胀，推理就开始退化；它没有跨会话状态，记不住五分钟前说过什么；它缺乏内置的任务分解和优先级排序能力。工程师们给这种问题起了个名字：**LLM 是“高产但缺执行调度层的生成核心”。**

两者的同构点可以列出四条：
- **执行功能缺陷** ↔ **无状态/缺调度层**：ADHD 的工作记忆不稳定，LLM 没有跨会话记忆，都依赖外部系统维持连续性。
- **上下文易被带偏** ↔ **上下文膨胀退化**：ADHD 被环境刺激带偏，LLM 被长上下文中的冗余信息干扰，都需要主动设计“当下注意什么”。
- **时间盲** ↔ **时序控制困难**：ADHD 感知时间困难，LLM 在长任务中容易丢失时间顺序。
- **多巴胺驱动** ↔ **即时反馈需求**：ADHD 的多巴胺失调导致动机缺陷，LLM 的生成质量也依赖即时奖励信号（如 RLHF）。

## 历史人物的 harness：外部调度层的原型

看看毛泽东。他小时候是问题儿童，和父亲吵架，爱读闲书；一辈子在动，从井冈山到延安到北京，永远在调查研究；冲动敢赌，四渡赤水、抗美援朝都是险棋。典型的 ADHD 特质：高生成、低调度。

但他的 harness 系统同样典型：**批评与自我批评**（定期外部反馈）、**调查研究**（强制 grounding）、**民主集中制**（用集体决策制衡个人冲动）、**党指挥枪**（用组织系统管理军队和国家）。这些机制本质上是“外部执行功能层”：它们替代了缺失的内置调度，让一个冲动的生成核心能够稳定运行数十年，最终缔造了一个新中国。

对应到 LLM/Agent 工程：**批评与自我批评** ≈ 定期 re-grounding（如反思链）；**调查研究** ≈ 检索增强生成（RAG）——让模型先查资料再回答；**民主集中制** ≈ 多模型投票或 ensemble；**党指挥枪** ≈ 外部调度器（如 LangChain 的 Agent Executor）。

李白则用了另一套 harness：**以酒和诗为核心**，把所有过剩精力注入诗歌创作；**用道家的逍遥对抗世俗规则**，不委屈自己做不想做的事。对应到 LLM：**超聚焦模式** ≈ 为 LLM 设计专用 prompt 模板，限制其输出范围；**规则规避** ≈ 在 Agent 中设定“不回答”的边界条件。

## 今天的 AI 工具：现成的 harness

如果你既是 ADHD 患者又是 Agent 工程师，好消息是：同一套思路的工具已经存在。

- **Goblin Tools**：它的 Magic ToDo 功能将“清理厨房”这种模糊任务自动分解为具体步骤（来源：Goblin Tools）。这正是 LLM 的 ReAct 模式——把复杂问题拆成子任务，逐个解决。
- **Saner.AI**：通过本地记忆和快速检索减少搜索循环（来源：Saner.AI）。对应 LLM 的向量数据库——把外部记忆挂在生成核心外面。
- **Motion**：自动排程并动态调整日程，消除“下一步该做什么”的决策负担（来源：Motion）。对应 LLM 的 AutoGPT 式任务队列——让外部调度器接管优先级排序。

这些工具的共同点：**它们不试图“修复”核心，而是给核心套上一个外部调度层。** 这正是 Agentic Harness 工程的精髓。

## 脚手架 vs 拐杖：必须诚实的边界

但同构框架有它的局限。资料中的『矛盾与存疑』明确警告：同构性目前仅是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。过度推广这一观点可能误导读者。

更现实的争议是：**外部脚手架会不会变成拐杖？** 如果 ADHD 患者长期依赖 AI 分解任务、安排日程，他们的执行功能会不会进一步退化？同样，如果 Agent 工程师过度依赖外部调度器，LLM 本身的推理能力会不会萎缩？

答案在于设计原则：脚手架应该**增强而非替代**。好的 harness 应该让用户/模型在“有辅助”和“无辅助”之间渐进切换。例如，Goblin Tools 的“辣度”调节允许用户控制分解粒度，这就是一个增强式设计——用户可以选择自己完成多少。

## 今天就能试的四个行动

1. **给 LLM 套一个“调查研究”prompt**：在每次提问前加一句“请先列出你需要的已知信息，再回答”。这相当于毛泽东的“没有调查就没有发言权”。
2. **用 Goblin Tools 拆解一个你拖延已久的任务**：把“写周报”扔进去，看它拆成几步，然后只做第一步。
3. **给 ChatGPT 设置“批评与自我批评”循环**：让它生成答案后，再用 prompt “请指出你回答中的三个潜在错误”进行反思。
4. **在日程中嵌入“民主集中制”**：用 Motion 或类似工具自动排程，但每周留出一天不依赖工具，手动规划——测试你的调度肌肉。

## 结语

ADHD 大脑和 LLM 共享同一个架构缺陷：生成能力强，调度能力弱。但这也意味着它们共享同一个解法：外部 harness。毛泽东用民主生活会和调查研究来驾驭自己的冲动，李白用酒和诗来聚焦精力，今天的 AI 工具用任务分解和自动排程来补偿执行功能。

这不是巧合。这是同一道工程题的两个版本。而你是那个可以同时解两道题的工程师。

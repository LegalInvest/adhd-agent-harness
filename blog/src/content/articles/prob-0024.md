# 如果执行功能可以「外置」，那 agent 的编排层是不是一份现成的 ADHD 执行功能蓝图？

> ADHD 侧：执行功能（计划/组织/启动）先天偏弱；LLM/harness 侧：agent 的编排/记忆/工具层就是「外部执行功能」。用双域证据作答。

如果你是一名 ADHD 患者，你应该很熟悉这样的场景：脑子里同时冒出十几个想法，每个都值得深挖，但当你试图把它们变成行动时，大脑却像一台没有操作系统的电脑——指令无法执行。如果你是一名 LLM agent 工程师，你可能也见过类似的一幕：模型在第五轮对话后开始忘记系统提示，行为偏离初始目标，甚至陷入无限循环。

这两个世界看似无关，但本文要回答一个核心问题：如果执行功能可以「外置」，那 agent 的编排层是不是一份现成的 ADHD 执行功能蓝图？答案是：是的，但前提是理解两者的同构本质，并警惕「脚手架」变成「拐杖」的风险。

## 同构：高产生成核心 + 缺失的执行调度层

ADHD 大脑与 LLM 共享一个核心结构：两者都是强大的生成器，但缺乏内建的调度机制。ADHD 的执行功能缺陷——计划、组织、工作记忆、任务启动困难——对应 LLM 的无状态性、上下文窗口溢出和输出不可控（来源：ADHD × AI 的科学与研究前沿）。

在 ADHD 侧，证据来自工具的实际效果。Goblin Tools 的 Magic ToDo 功能能将“整理房间”这样的模糊指令分解为“捡起衣服”“擦桌子”等具体步骤，降低启动门槛（来源：Goblin Tools）。用户反馈称它“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools）。Motion 则自动根据截止日期和优先级动态调整日程，消除“下一步该做什么”的决策负担（来源：Motion）。这些工具都在做同一件事：提供外部执行功能层，弥补内在调度缺陷。

在 LLM/agent 侧，harness 工程做的也是同样的事。Agent harness 通过事件驱动的系统提醒来对抗指令消退，设置 token 预算和工具调用次数上限防止无限循环，并加入 human-in-the-loop 检查点（来源：人在回路与身体在场）。其核心是“形式化规划与护栏，使代理输出真正有用且正确”（来源：人在回路与身体在场）。模型本身强大，可靠性来自架构+护栏——这和 ADHD 大脑需要外部脚手架才能可靠执行任务，原理完全一致。

## 重锚定系统：对抗目标漂移的双域解法

ADHD 大脑在执行任务时极易发生目标漂移：注意力被无关刺激捕获，或过度聚焦于细节而忽略整体。外部工具通过提醒和通知提供重锚定，将注意力拉回原任务（来源：重锚定与目标漂移）。例如，AI 工具“提供定制化提醒、组织信息，甚至帮助写作和沟通”（来源：重锚定与目标漂移）。

LLM agent 在长程任务中同样面临目标漂移：模型在交互轮次后期开始忘记系统提示，导致行为偏离初始指令。Harness 通过事件驱动的系统提醒来对抗这种“指令消退”（来源：重锚定与目标漂移）。这种结构上的对应说明，ADHD 大脑和 LLM 需要的是同一类解决方案：一个外部系统，在关键时刻把注意力拉回正轨。

Saner.AI 则从另一个角度切入：通过增强知识回忆减少搜索循环和标签切换（来源：Saner.AI）。ADHD 患者常因工作记忆不足而频繁切换应用寻找信息，Saner.AI 的本地记忆存储和快速检索直接针对这一痛点。这对应 LLM 的上下文工程——通过主动管理输入信息（分段、摘要、提醒）维持任务连贯性（来源：ADHD × AI 的科学与研究前沿）。

## 身体在场效应与人在回路

Focusmate 等工具通过 AI 匹配虚拟身体搭档，利用“对另一个人的责任感”绕过执行功能屏障（来源：人在回路与身体在场）。这种身体在场效应是 ADHD 群体最有效的策略之一。在 LLM/agent 侧，对应的是 human-in-the-loop 监督：设置护栏、加入人工检查点，暂停并询问后再执行（来源：人在回路与身体在场）。两者都依赖一个外部实体（人或系统）提供实时调度和问责。

## 核心判断：外部执行功能层是蓝图，但非万能药

基于以上证据，我可以给出一个鲜明的判断：agent 编排层确实是一份现成的 ADHD 执行功能蓝图。两者的同构性意味着，为 LLM 设计的 harness 架构——包括记忆管理、任务分解、重锚定机制、人在回路——可以直接映射为 ADHD 辅助工具的设计原则。事实上，当前最好的 ADHD 工具（Goblin Tools、Motion、Saner.AI）已经在无意中实现了这种映射。

# Speechify 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Speechify：它到底补上了哪一层执行功能？

## 问题：为什么 Speechify 对某些 ADHD 用户无效，对另一些却改变人生？

答案藏在同构视角里：Speechify 之于 ADHD，就像 function calling 工具调用之于 LLM——但许多人用错了。

## 同构：ADHD 大脑与 LLM 共享同一套缺陷

ADHD 大脑常被描述为一个高产的「生成核心」——想法丰富、联想活跃，但缺少稳定的调度层来筛选、启动和坚持任务（来源：ADHD 的 AI 工具全景）。LLM 同样具备强大的生成能力，却在长程上下文、目标保持与执行调度上存在天然缺陷。两者都需要外部脚手架——对 ADHD 来说是日程、提醒、环境设计；对 LLM 来说是确定性工作流、工具契约和状态机（来源：采样温度与表现波动）。

曹操深谙此道。他 ADHD 特质明显：一生南征北战，60多岁还亲征，冲动杀吕伯奢，情绪来去极快，小时候“任侠放荡，不治行业”。但他的 harness 系统是法家思想与人才分工——把行政全丢给荀彧，让别人的执行功能弥补自己的不足。曹操的成就不是靠自我改造，而是靠搭建了一个外部调度系统。

LLM 的对应物是什么？是 agent scaffolding——将 LLM 置于与记忆、工具和决策逻辑的控制循环中，通过赋予其访问工具、领域数据和结构化工作流来增强裸能力（来源：工具使用与认知卸载）。曹操的荀彧，就是 LLM 的 function calling 工具。

## Speechify 到底补上了哪一层执行功能？

Speechify 的核心价值在于将文本转化为语音，降低阅读门槛。但对 ADHD 用户而言，它真正补上的是 **任务启动** 这一层执行功能。ADHD 大脑面对长文本时，启动成本极高——就像 LLM 面对一个需要多步推理的问题，如果没有工具调用，它可能直接产生混乱的输出。

Goblin Tools 的 Magic ToDo 功能能自动将任何任务分解为更小步骤（来源：Goblin Tools），Lex 允许通过单一指令触发复杂任务序列（来源：Lex），Saner.AI 通过本地记忆减少搜索循环（来源：Saner.AI）——这些工具的共同点在于：它们不是替代大脑，而是将生成核心接入一个可维护的执行调度系统。

但有人用错了 Speechify：他们把它当作“听书工具”，期望它能自动解决注意力分散、理解力不足。结果发现，听书时同样走神，甚至更糟——因为没有视觉锚点。这就像给 LLM 一个工具调用接口，却不给它定义工具契约和确定性流程——LLM 会随机调用、输出混乱、任务失败。

## 脚手架 vs 拐杖：边界在哪里？

核心争议在于：AI 工具是“外部执行功能层”还是“拐杖”？过度依赖可能导致能力退化（来源：矛盾与存疑）。这个边界在于：工具是否在训练你的调度能力，还是在替代它。

曹操的荀彧不是拐杖——曹操仍然做出战略决策，只是把执行外包。同样，好的 ADHD 工具应该让你学会如何拆解任务、如何设置提醒，而不是让工具替你完成一切。Lauren Jackson 作为 ADHD 创业者，她的 harness 系统包括 hyperfocus、action bias、systems thinking——她不是用工具替代自己的执行功能，而是用工具放大自己的优势。

对 LLM 而言，function calling 的正确用法是：工具负责精确计算，LLM 负责意图理解与规划。如果让 LLM 直接调用工具而不做规划，就会产生“小错误累积、非确定性使可重复性复杂化”的问题（来源：采样温度与表现波动）。

## 今天就能试的行动

1. **用 Speechify 时，先定义“听”的目标**：不是“听完这本书”，而是“听完第3章，记下3个要点”。这对应 LLM 的“计划-执行”分离模式。
2. **将 Speechify 与任务分解工具结合**：先用 Goblin Tools 将阅读任务分解为小步骤，再用 Speechify 执行每一步。这对应 agent 的确定性工作流。
3. **建立“外部记忆”**：用 Saner.AI 或 Reflect 记录听完后的反思，补偿工作记忆的不稳定。这对应 LLM 的检索增强生成。
4. **每周检查一次依赖程度**：问自己“如果没有这个工具，我还能做这件事吗？”这对应工程中的“周期性采样以监控质量”。

## 局限

本文的同构论点目前是理论框架，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。个体差异极大——有些 ADHD 用户发现强行打断（如番茄钟）比柔性引导更有效。长期依赖风险也缺乏实证。但至少，这个视角提供了一个清晰的决策框架：你是在搭建脚手架，还是在寻找拐杖？

# Speechify 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Speechify：它到底补上了哪一层执行功能？

## 问题：你是在用 Speechify 辅助阅读，还是在用它逃避阅读？

许多 ADHD 朋友把 Speechify 当作“听书神器”，用它来“读完”那些永远看不完的文章。但奇怪的是，听完后常常大脑一片空白——好像听了，又好像没听。另一边，LLM agent 工程师们精心设计 function calling，让模型把精确计算外包给工具，却发现模型有时会胡乱调用工具、输出不稳定。这两个场景看似无关，却共享同一套结构性问题：**生成核心（大脑/LLM）与执行调度层（阅读/工具调用）的失配**。

## 同构：ADHD 大脑与 LLM 的“生成核心-调度层”分离

ADHD 大脑的智力、创造力与 LLM 的生成能力类似，但两者都缺乏稳定的执行功能（计划、组织、工作记忆）调度（来源：ADHD 的 AI 工具全景）。ADHD 个体的表现呈现显著的日内波动，注意力、执行功能在不同时间点差异很大——这类似于 LLM 的采样温度：温度越高输出越随机、越不稳定（来源：采样温度与表现波动）。LLM agent 工程中，“温度”等参数导致非确定性，小错误累积使可重复性复杂化（来源：AI Agent Systems: Architectures, Applications, and Evaluation）。

所以，ADHD 大脑与 LLM 是同一类“高产但缺执行调度层的生成核心”。两边的解法也同构：给生成核心套一个 harness（脚手架），通过外部工具提供执行功能、记忆管理和上下文控制（来源：ADHD 的 AI 工具全景）。Speechify 之于 ADHD，就像 function calling 工具调用之于 LLM——它本应是补上“精确、重复、状态维护”这一层的工具，但很多人用错了。

## 错用：Speechify 作为“拐杖”而非“脚手架”

Speechify 的核心功能是文本转语音，让用户“听”文章。这看似解决了阅读启动困难（ADHD 侧）和输出多样性（LLM 侧），但问题在于：**它只解决了输入通道，没有解决理解与记忆的调度问题**。

ADHD 侧证据：Goblin Tools 等工具被用来“将模糊的工作分解为更小的行动”，降低启动门槛（来源：工具使用与认知卸载）。Lex 允许用户通过单一指令触发复杂、多步骤的任务序列（来源：Lex）。这些工具设计的关键是“分解+执行”，而 Speechify 只是将阅读从视觉换到听觉，并没有分解理解过程或提供记忆锚点。用户被动听，大脑依然处于“生成核心”的随机游走状态——就像 LLM 在高温下随机采样，没有外部工具来约束和引导输出。

LLM/harness 侧证据：LLM agent 通过 harness 调用工具、管理文件、处理长对话，工具设计被视为“agent UX”，强调命名、模式、错误处理等接口质量（来源：工具使用与认知卸载）。如果 harness 设计不好，模型就会胡乱调用工具。同样，如果 Speechify 只是“听”，而没有与笔记、回顾、任务分解等工具集成，它就像给 LLM 一个没有 schema 的 function call——模型会调用，但结果不可控。

## 正确用法：Speechify 作为认知卸载的“function calling”层

Speechify 的正确角色，是**将“阅读”这一高认知负荷任务，拆解为“听+处理”两个步骤，并利用外部工具（如笔记、摘要）来接管理解与记忆的调度**。

具体来说，ADHD 用户应该：

1. **用 Speechify 听内容时，同步使用 Saner.AI 或 Mem 做语音笔记**，让工具接管工作记忆。Saner.AI 专注于知识回忆和本地记忆，减少搜索循环（来源：Saner.AI）。这样，Speechify 负责“输入”，Saner.AI 负责“存储”，形成完整的认知卸载链路。

2. **将 Speechify 与 Goblin Tools 或 Lex 联动**：听完一篇文章后，用 Goblin Tools 的 Magic ToDo 自动生成行动步骤（来源：Goblin Tools），或用 Lex 将关键信息转化为可执行的单一指令。这对应 LLM agent 的“多步推理+工具调用”模式（来源：工具使用与认知卸载）。

3. **控制“采样温度”**：ADHD 大脑的波动性类似 LLM 的温度参数。在精力好时用 Speechify 听复杂内容，精力差时听轻松内容或关闭。不要试图用 Speechify 填补所有时间盲——工具应“插入现有堆栈”，不增加认知开销（来源：工具使用与认知卸载）。

## 争议与局限：脚手架 vs 拐杖的边界

目前缺乏针对 Speechify 的独立随机对照试验，证据主要来自用户评价（来源：矛盾与存疑）。过度依赖 AI 工具可能阻碍内在执行功能发展，如何设计可逐步撤除的脚手架仍是挑战（来源：ADHD 的 AI 工具全景）。此外，ADHD 异质性高，单一工具难以覆盖所有亚型（来源：矛盾与存疑）。Speechify 对某些人可能是救命稻草，对另一些人可能是分散注意力的噪音。

## 今天就能试的行动

1. **听+记**：打开 Speechify 听一篇文章时，同时打开 Saner.AI 或 Mem，用语音记录关键点。听完后看笔记，确认是否理解。
2. **听后分解**：用 Goblin Tools 的 Magic ToDo，将文章中的行动点自动分解为小步骤。
3. **温度调节**：记录自己一天中注意力最好的时段，只在该时段用 Speechify 听高难度内容。
4. **集成而非新增**：检查 Speechify 是否与你已有的工具（如日历、待办清单）集成，避免增加认知开销。

## 结语

Speechify 不是阅读的替代品，而是阅读的调度层。就像 LLM 需要精心设计的 function calling 才能可靠工作，ADHD 大脑也需要将 Speechify 嵌入一个完整的认知卸载系统。用对了，它是脚手架；用错了，它只是拐杖。选择权在你手上。

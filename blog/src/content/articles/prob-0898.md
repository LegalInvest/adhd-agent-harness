# Inflow 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Inflow：它到底补上了哪一层执行功能？

你有没有过这样的体验：脑子里涌出一万个想法，但坐到电脑前却一个字都敲不出来；明明知道 deadline 就在下周，却只能刷手机拖延到最后一刻。与此同时，在另一个世界里，工程师们正对着 LLM 发愁——GPT-4 能写出惊艳的文案，但让它“帮我规划一个三天的旅行”，它要么给出笼统的步骤，要么在细节中迷失方向。

这两个场景看似无关，却共享同一个底层结构：**高产生成核心 + 缺失的执行调度层**。ADHD 大脑与 LLM 在结构上同构——二者都是创意与语言的高产引擎，但都缺乏可靠的内置 planner-executor 循环来把宏大意图拆解成可执行的步骤（来源：[[ADHD 大脑与 LLM 的同构]]）。

## 同构：规划循环与任务分解

ADHD 侧最痛的体验是“任务启动困难”和“不会拆解”。面对一个“整理房间”的指令，非 ADHD 大脑会自动分解为“先扔垃圾→再分类衣物→最后归位”，而 ADHD 大脑看到的是整座山，然后僵住。这恰恰对应了 LLM/Agent 工程中的经典问题：给 agent 一个“写一篇万字报告”的目标，如果不提供 planner-executor 循环，它要么输出一个空洞的大纲，要么在第一个小节就陷入细节死循环。

解决方案在两边是同构的：ADHD 需要外部脚手架（external scaffolding）来承担调度、排序、提醒与监控，而 LLM 需要 agent harness 来执行任务分解与动态规划。**Inflow 之于 ADHD，就像 planner-executor 任务分解之于 LLM**——它补上的正是那层“把意图转译为步骤”的调度层。

## Inflow 补上了哪一层？

Inflow 是一款专为 ADHD 设计的 AI 日程与任务管理应用，但它并非简单的“待办清单+提醒”。它的核心机制是：**将用户输入的模糊目标（如“准备会议”），通过 AI 分解为具体的、有时间边界的子任务，并动态调整日程**。这与 Motion、Reclaim.ai、Tiimo 等工具属于同一类“自适应规划器”——它们都能在日程被打乱时自动重新安排，消除“下一步该做什么”的决策负担（来源：[[Motion]]、[[Reclaim.ai]]、[[Tiimo]]）。

但 Inflow 的独特之处在于它更强调**任务分解**：它不是只告诉你“10:00 做 A”，而是先帮你把 A 拆成 A1、A2、A3。这直接对应了 ADHD 执行功能障碍中的“计划与组织缺陷”（来源：[[执行功能障碍]]）。从同构视角看，这就像给 LLM 加了一个 planner 模块：先把“写报告”拆成“收集资料→写大纲→填充内容→修改”，然后 executor 再去执行每一步。

## 真实案例：从徐渭到 LLM harness

明朝才子徐渭（徐文长）是 ADHD 特质与情绪失调的典型例子：他诗书画戏军全能，但执行功能极差，8 次举人不中；情绪失调导致杀妻、9 次自杀；兴趣爆发式转移，什么都学什么都会（来源：[[情绪失调]]）。他的专属 harness 是以书画为核心，把所有痛苦和精力注入艺术创作，并用军事谋略帮助胡宗宪抗倭，把超专注用在打仗上。

这个 harness 的本质是什么？**外部调度器**。徐渭无法靠内在的执行功能来规划“今天该读书还是画画”，但他通过建立固定的创作仪式（日课）和外部约束（军务的 deadline），把生成核心的能量引导到具体产出上。这对应了 LLM harness 中的“re-grounding”机制：定期把 agent 拉回主目标，防止发散。

## 脚手架 vs 拐杖：争议与边界

然而，Inflow 这类工具并非万能，甚至可能带来风险。wiki 资料中明确指出了矛盾：AI 工具是“外部执行功能层”还是“拐杖”？过度依赖可能导致能力退化（来源：[[矛盾与存疑]]）。

我的判断是：**区分脚手架与拐杖的关键在于用户是否保留了元认知参与**。脚手架是暂时性的支撑，帮助用户学会自己搭建结构；拐杖则是永久替代，让用户放弃练习。Inflow 的正确用法是：先用它帮你拆解任务、建立节奏，但过程中你要有意识地观察“原来这个任务可以这样拆”，逐步内化这种分解能力。如果只是被动执行 AI 给你的每个步骤，而不理解背后的逻辑，那就是拐杖。

另一个争议是：超聚焦应被引导还是打断？部分 ADHD 用户反馈，强行打断（如番茄钟）反而有效，而柔性引导可能不够强力（来源：[[矛盾与存疑]]）。Inflow 的自动排程可能会打断超聚焦，这对某些人可能是灾难。因此，工具需要个性化配置——比如允许用户设置“不可打断的深度工作块”，这正是 Reclaim.ai 强调的“保护而非填充”策略（来源：[[Reclaim.ai]]）。

## 今天就能试的行动

1. **用 Inflow 拆解一个你拖延最久的任务**：不要只输入“写论文”，而是输入“写论文第一部分”，让 AI 帮你分解成“打开文档→写 100 字→休息 5 分钟”。观察分解粒度是否合适。
2. **对比 Motion 和 Tiimo 的视觉化风格**：如果你对时间盲严重（感觉不到时间流逝），Tiimo 的视觉时间线可能比 Motion 的列表更有效（来源：[[Tiimo]]）。
3. **设置一个“元认知提醒”**：在 Inflow 中加一个每日任务：“回顾今天的任务分解，我学会了什么？” 这能防止你滑向拐杖依赖。
4. **尝试“保护块”策略**：如果你的日程常被会议打乱，用 Reclaim.ai 的“防御性时间块”保护 2 小时深度工作，观察是否减少任务切换的损耗（来源：[[Reclaim.ai]]）。

## 局限与未来

目前，ADHD 大脑与 LLM 同构的论点仍多为类比推理，缺乏神经科学或计算机科学的直接证据（来源：[[矛盾与存疑]]）。此外，Inflow 等工具的长期效果缺乏纵向研究，用户反馈虽积极但可能受选择偏差影响。最终，工具只是辅助，真正的改变来自你对自身执行功能模式的理解——而理解的第一步，是承认你与 LLM 共享同一个困境：不是不够聪明，而是缺少一个靠谱的调度员。

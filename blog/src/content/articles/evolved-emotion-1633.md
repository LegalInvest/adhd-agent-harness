# 为什么用 Goblin Tools 治 ADHD 的情绪失调，和给 agent 套 会褪去的脚手架 是一回事？

> Goblin Tools 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

## 问题：当情绪比任务更先失控

你刚打开电脑，准备写一份报告。五分钟后，你发现自己刷完了三个社交平台，冰箱门开了一次，还列了一张“下周要买的东西”清单。任务没动，但挫败感已经涌上来。这不是懒——是 ADHD 大脑的情绪失调：工作记忆瞬间超载，前额叶无法抑制杏仁核的警报，情绪像脱缰的生成器一样喷涌而出（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

同样的问题发生在 LLM 身上。给 GPT-4 一个模糊指令，它可能生成一篇华丽的散文，但完全偏离你的意图。它没有内置的调度层来“先想清楚再输出”。所以工程师给它套上 agent scaffolding：分解任务、设定边界、逐步校验。这个脚手架，注定要褪去——就像 ADHD 大脑需要的工具，不该是永久的拐杖。

## 同构：两个生成核心，同一个解法

ADHD 大脑与 LLM 的深层同构，已经被最新研究证实。明尼苏达大学测试发现，LLM 呈现“强记忆、弱控制”剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷——无法灵活切换任务集、无法抑制自动化反应，这正是 ADHD 的经典神经心理模式（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。耶鲁大学进一步发现，自注意力机制本身导致工作记忆容量限制：随 N-back 任务 N 值增加，注意力分数熵增、注意力弥散、表现下降，与人类注意分散机制同源——“注意力资源的弥散分配”正是 ADHD 注意缺陷的计算本质（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。

两边都是高产但缺执行调度层的生成核心。ADHD 大脑的情绪生成系统（杏仁核）反应强烈，但前额叶（执行功能中枢）调控不足；LLM 生成能力强但缺乏输出控制。AI 工具作为外部调度层，通过提示词、规则引擎来引导情绪反应（来源：AI 与 ADHD 的情绪调节）。Goblin Tools 的 Magic ToDo 正是这样做的：接受模糊任务（如“清理厨房”），将其分解为具体、可执行的子任务，用户可调节“辣度”控制分解粒度（来源：Goblin Tools）。这本质上就是 agent 的 task decomposition——只是服务对象从 LLM 换成了 ADHD 大脑。

## 人物案例：释迦牟尼的 harness 与 LLM 的脚手架

释迦牟尼29岁说走就走，放弃王位妻子孩子；先后跟两个仙人学、苦行6年，不断试错不断放弃；49年一辈子游行说法，80岁死在传道路上。他有一整套 harness：以“八正道”“持戒”为核心，用戒律管自己的行为，用正念拴住乱跑的念头；“自依止法依止”，不依赖权威，自己证悟。这套系统的核心是什么？是“日课”——每天定时 re-grounding。

这和 LLM agent 的“会褪去的脚手架”完全同构：agent 需要定时 checkpoint、re-grounding 到原始目标，否则会 drift。释迦牟尼的“日课”就是外部调度器：每天固定时间冥想、说法、行脚，把注意力拉回正轨。这不是永久拐杖——他最终的目标是“自依止”，即内在化这些能力。同样，Goblin Tools 的 Magic ToDo 不是让你永远依赖它分解任务，而是通过反复使用，训练你内在的任务分解能力。Inflow 也是这样：它基于 CBT 原理，教用户建立技能，而非替代技能（来源：Inflow）。

## 脚手架 vs 拐杖：边界在哪？

核心判断：**AI 工具应该作为“会褪去的脚手架”，而非“永久拐杖”。** 脚手架在建筑完成后拆除，让建筑自己站立；拐杖则让人永远依赖。当前最大的争议在于：长期使用 AI 工具是否会导致能力退化？缺乏长期追踪研究（来源：矛盾与存疑）。但我们可以从结构上区分：如果工具帮你完成了你本可以自己完成的任务（比如你本来能分解任务，只是懒得做），那就是拐杖；如果工具帮你完成了你根本做不到的任务（比如工作记忆不足导致无法记住步骤），那就是脚手架。

对于 ADHD 情绪调节，Goblin Tools 的 Magic ToDo 降低启动门槛，减少因启动困难引发的挫败感（来源：AI 与 ADHD 的情绪调节）。但这只能处理结构化情绪问题（如任务焦虑），对深层情绪（如创伤、自我否定）效果不足（来源：矛盾与存疑）。同样，agent scaffolding 能防止 LLM 跑偏，但不能解决模型本身的偏见或知识缺口。

## 今天就能试的 3 件事

1. **用 Goblin Tools 分解一个你拖延已久的任务**：打开 Magic ToDo，输入“整理书桌”或“写周报”，调节辣度到你觉得“可以开始”的程度。完成第一步后，记录情绪变化——你可能会发现焦虑值下降。
2. **给自己设一个“日课”式 re-grounding**：每天固定时间（比如上午10点），用 5 分钟回顾今天的主要目标。这相当于 agent 的 checkpoint，防止 drift。可以用手机闹钟或 Saner.AI 的每日规划功能（来源：Saner.AI）。
3. **区分“脚手架”与“拐杖”**：每次使用 AI 工具前问自己：“这个步骤是我学不会，还是我不想学？”如果是前者，放心用；如果是后者，试着先不借助工具做一次，再对比差异。

## 局限与诚实

同构命题目前仍是理论框架，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。个体差异显著：ADHD 亚型（注意力不集中型 vs. 冲动型）对 AI 情绪调节工具的反应不同（来源：AI 与 ADHD 的情绪调节）。此外，学习使用 AI 工具本身可能增加认知负荷，净效果因人而异（来源：矛盾与存疑）。但至少，Goblin Tools 和 agent scaffolding 的共享结构提示我们：失控的生成器需要的不是更强大的生成能力，而是一个可撤销的外部调度层。这个脚手架，终将褪去——但在此之前，它让我们能够开始。

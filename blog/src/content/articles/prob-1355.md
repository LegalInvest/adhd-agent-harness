# 如果把 ADHD 大脑当成生成核心，Claude 能不能当它的规划循环与任务分解？

> 用 agent 工程的思路重新理解 Claude 对 ADHD 的真正价值。

## 问题：为什么 ADHD 大脑和 LLM 都卡在第一步？

你盯着空白的待办清单，知道有一堆事要做，但就是启动不了。或者你终于开始写报告，半小时后却发现自己正在研究“二战坦克型号大全”——目标漂移了。这不是你懒，而是 ADHD 大脑在任务分解和规划环节出了故障。

与此同时，工程师们正为 LLM agent 头疼：让 Claude 写一首诗它很擅长，但让它“规划一次三天的旅行”就会乱套——它会把所有景点塞进第一天，或者忘记订酒店。LLM 和 ADHD 大脑一样，都是强大的生成核心，却缺乏可靠的调度层。

## 同构：生成核心与调度层的分离

ADHD 文献群（681 篇命中）中，“任务分解与规划”是核心中间机制概念（来源：LBD 同构对）。执行功能缺陷，尤其是规划与组织，是 ADHD 的核心特征（来源：How specific are executive functioning deficits in attention...）。而 agentic harness 文献群（2914 篇命中）同样以“任务分解与规划”为核心（来源：LBD 同构对）。代表作 AgentGen 通过环境与任务生成增强 LLM agent 的规划能力（来源：AgentGen: Enhancing Planning Abilities...）。

这意味着：ADHD 大脑与 LLM 是同一类“高产但缺执行调度层”的生成核心。ADHD 大脑能产生无数创意，LLM 能生成流畅文本，但两者都需要外部 harness 来把大目标拆成小步骤、保持方向、避免脱轨。

## 人物 harness：孔子与左宗棠的“外部调度层”

孔子身高 1 米 9，精力旺盛，周游列国 14 年坐不住，冲动爱骂人，思维跳跃，没有系统著作——典型的 ADHD 特质。他的 harness 是“克己复礼”：用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”）。这相当于一个定时 re-grounding 机制，类似于 LLM agent 的“定期回顾上下文”操作。

左宗棠脾气大，爱骂人，40 多岁才出山，但 69 岁抬棺出征收复新疆。他的 harness 是“结硬寨打呆仗”——每天读经写家书，用固定的日课来约束自己。这相当于 LLM 的“固定规划循环”：每天开始时重新评估任务、调整优先级。

两者的共同点：用外部结构（礼、日课）来补偿内部调度缺陷。这和 LLM agent 的 planner-executor 循环完全同构。

## Claude 能当规划循环吗？

回到核心问题：如果把 ADHD 大脑当成生成核心，Claude 能不能当它的规划循环与任务分解？

答案是：可以，但有条件。

Claude 作为 LLM，本身没有内置规划能力，但它可以通过提示工程被塑造成一个外部调度器。例如：
- **任务分解**：让 Claude 将“写一篇论文”分解为“确定主题→收集资料→列提纲→写引言→写正文→写结论→修改”。每一步再拆成更小的子任务。
- **重锚定**：当目标漂移时，Claude 可以提醒你“回到原任务”。这类似于 Motion 和 Reclaim.ai 的动态调整功能（来源：Motion 自动排程与动态调整；Reclaim.ai 保护时间块）。
- **视觉化时间线**：Tiimo 通过视觉日程让时间变得可感（来源：Tiimo 专为神经多样性设计），而 Claude 可以生成类似的时间线描述。

但关键局限：Claude 本身没有执行功能——它不能自动设置提醒，也不能在你分心时拉你回来。它需要一个 harness（如提示模板、agent 框架）来充当“外部执行功能层”（来源：AI 工具通过提供外部执行功能层补偿 ADHD 核心缺陷）。

## 脚手架 vs 拐杖：边界在哪里？

“当前最站得住脚的观点是：AI 作为‘调度层’脚手架，能显著降低执行功能负荷，但无法根治 ADHD 的神经基础”（来源：AI 与 ADHD 的时间管理）。

过度依赖风险真实存在：如果 Claude 帮你分解了所有任务，你可能会丧失自己拆解的能力。传统提醒系统的问题在于“设置提醒本身就是一个执行功能任务”（来源：AI Assistant for ADHD...）。理想的方案是无需主动设置的提醒，但完全自动化可能让用户失去练习机会。

## 今天就能试的行动

1. **用 Claude 做任务分解**：告诉它“我有一篇 3000 字的报告要写，帮我拆成 10 个可执行步骤，每一步不超过 15 分钟”。
2. **设置重锚定提示**：让 Claude 在每次对话开始时输出“你当前的核心任务是什么？”，作为 re-grounding 机制。
3. **结合视觉工具**：将 Claude 的分解结果导入 Tiimo 或 Todoist，用视觉时间线执行。
4. **尝试自适应规划器**：如果预算允许，试用 Motion 或 Reclaim.ai，体验自动排程如何消除决策负担。

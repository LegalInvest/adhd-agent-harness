# ChatGPT 之于 ADHD，就像重锚定 system prompt 之于 LLM——但有人用错了

> agent 工程里最便宜也最有效的防漂移手段，是定期把原始目标重新注入上下文。ChatGPT 对 ADHD 的真正价值恰好是同一件事——但前提是你得这么用它。

先讲工程侧的背景。长任务 agent 最顽固的病是目标漂移：跑到第 20 步，原始指令早被中间产物挤出注意力焦点，agent 开始一本正经地做偏题。而工程上性价比最高的解法朴素得可笑：**重锚定**——每隔几步，把 system prompt（你到底要干什么）原文重新贴进上下文。不训练、不改架构，漂移率显著下降。

现在看 ADHD 侧。你有没有过这种时刻：忙了一整天，累到瘫倒，却隐约不安——今天干的这些事，和我真正要做的事有什么关系？那就是目标漂移。ADHD 的目标表征衰减快，环境刺激又不断塞进新的「当前焦点」，一天下来，最初的锚早就漂没了。

## ChatGPT 用对的样子：一台随身重锚定机

ChatGPT 对 ADHD 最大的价值，不是帮你写邮件（那是顺带的），而是它可以成为一个随时可调用的重锚定协议。用法具体到可以照抄：

**晨间锚定**：把今天最重要的一件事告诉它，让它帮你拆出第一步。这一轮对话的真正产出不是计划，是你把目标显式化了一次——写出来的目标才有被重新注入的可能。

**漂移检查**：下午随时丢给它一句「我今天的目标是 X，我现在在做 Y，帮我判断我是在推进还是在漂移」。它的回答质量甚至不重要——**这个动作本身就是重锚定**，是把 system prompt 重新贴回你的上下文窗口。

**回锚复盘**：晚上问「我早上说要做 X，请根据我现在的描述帮我评估完成度，并把没做完的部分变成明天的第一步」。这形成了一个跨日的锚链：目标不再依赖你的工作记忆存活，而是存活在一个外部系统里，每天被检索、比对、续期。

## 用错的样子：把锚机当成了新的漂移源

但同一个工具，有一批人用出了相反的效果——ChatGPT 本身成了漂移加速器。症状包括：本来查一个问题，四十分钟后你在和它讨论第七个衍生话题（它太擅长接住你的每一次发散了，它是史上最顺滑的兔子洞）；用「问 AI」代替「开始做」，产生了推进的幻觉，实际上是把启动焦虑转移成了无限的准备工作；以及最隐蔽的一种——让它替你想目标。锚必须是你的，代打的锚锚不住任何东西。

判断标准一句话：**用完 ChatGPT 后，你是离开始更近了，还是离开始更远了？** 前者是重锚定，后者是高级拖延。

## 为什么这个用法对 ADHD 格外有效

神经典型者也漂移，但他们的内部重锚定回路基本够用——前额叶会定期自动「想起来我在干嘛」。ADHD 的这条回路带宽不足，所以外部重锚定的边际收益格外大：你不是在用 AI 获得超能力，是在外包一个别人标配、你缺货的基础组件。这也是为什么同样用 ChatGPT，ADHD 用户的体验分化最剧烈——它补的恰好是你最缺的层，或者放大的恰好是你最弱的闸。

## 边界

同构强度 B 级：重锚定在 agent 侧有明确工程实践，ADHD 侧的目标维持缺陷有执行功能文献支撑，但「ChatGPT 作为锚定工具」的疗效属于实践经验，尚无对照研究。另外注意隐私常识：别把不该外泄的目标和计划喂给任何在线模型。

## 今天就能试的 3 件事

1. **明早做一次晨间锚定**：一句话告诉 ChatGPT 今天最重要的事，让它拆第一步。全程不超过 5 分钟——超时说明你已经在兔子洞口了。
2. **设一个下午 3 点的漂移检查**：闹钟响就发那句固定咒语「目标是 X，我在做 Y，推进还是漂移？」
3. **给自己立一条兔子洞规则**：和 AI 的任何对话超过 10 轮，强制问自己一次「这和今天的锚有什么关系」。

工具没有立场——同一台锚机，可以定住你的一天，也可以陪你漂得更远更舒服。区别只在一个问题：锚，是不是你自己抛的。

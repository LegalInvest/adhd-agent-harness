# ChatGPT 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 ChatGPT：它到底补上了哪一层执行功能？

你打开 ChatGPT，准备写一份季度报告。三分钟后，你已经在问它“如何用 Python 抓取豆瓣电影评分”——因为脑子里突然闪过一个“更酷”的想法。这不是你的错，这是你的大脑架构在作祟。

ADHD 大脑与 LLM 共享同一个核心困境：两者都是“高产但缺执行调度层的生成核心”。LLM 在长程任务中会因上下文膨胀而目标漂移，ADHD 大脑则因工作记忆不稳定和超聚焦跑偏而频繁丢失目标。两者的解法在结构上同构：都需要一个外部脚手架来重锚定 system prompt。

## 同构：ADHD 大脑与 LLM 的共享困境

最新研究揭示了一个令人不安的平行：Transformer 自注意力机制在长上下文下的冲突解决能力会崩溃至随机水平（来源：Deficient Executive Control in Transformer Attention）。这正是 ADHD 执行功能障碍的核心神经机制——不是生成能力不足，而是调度层失效。ADHD 患者呈现“强记忆、弱控制”的认知剖面：工作记忆容量可能正常甚至超常，但认知灵活性和注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

换句话说，你的大脑不是“坏”了，而是缺少一个内置的调度器。LLM 同样如此：它本身是无状态、仅生成文本的核心，需要外部“脚手架”和“harness”来提供工具接口、上下文管理、安全执行等编排能力（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

## 重锚定：从曾国藩的日课到 LLM 的 system prompt

晚清名臣曾国藩的自我管理系统是 ADHD harness 的经典案例。他年轻时“无恒”“浮躁”，读书慢到“他人目下二三行，余或疾读不能终一行”。他的解决方案是“日课十二条”：黎明即起、读书不二、谨言、写日记反省……用最笨最稳的规则抵消自己的冲动。这本质上是一套**定时 re-grounding 系统**——每天多次强制自己回到核心目标。

这与 LLM harness 中的“重锚定”操作完全同构：当 agent 在长程任务中开始跑偏时，工程上会重新注入原始 system prompt，或者通过 Git 仓库存储项目上下文来恢复状态（来源：GitHub - ai-boost/awesome-harness-engineering）。曾国藩的日记就是他的“上下文存储”，日课就是他的“system prompt 刷新”。

另一个案例是明代才子徐渭，他兴趣爆发式转移，诗书画戏军全能却 8 次科举不中。他的 harness 是以书画为核心，把所有精力注入艺术创作，把超专注用在打仗上——这是一种**定向超聚焦**策略。但超聚焦本身是一把双刃剑：ADHD 大脑渴望新颖性，但容易在错误的事情上超聚焦六小时（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。最新理论提出，超聚焦源于前额叶控制层级的“断裂”：高级上下文信息不再能支配思维和行动（来源：Transient Frontal Fracturing: A Theoretical Account of Hyperfocus）。这正是 LLM 在长上下文中发生目标漂移的机制——高级目标被局部上下文淹没。

## ChatGPT 的正确用法：重锚定，而非替代思考

多数 ADHD 用户把 ChatGPT 当“万能帮手”：让它写邮件、做计划、生成大纲。这其实是用错了。ChatGPT 的真正价值在于它作为**外部调度层**的能力——你可以随时问它：“我刚才在做什么？我的目标是什么？下一步该干什么？”

这类似于 RescueTime 的自动化时间追踪和分心阻断功能，它补偿了 ADHD 的“时间盲”和执行功能缺陷（来源：Revolutionizing ADHD Management with AI Assistants）。但 RescueTime 是被动记录，ChatGPT 可以主动对话。关键在于：**不要让它替你做事，而是让它帮你重新锚定你的 system prompt**。

例如，当你发现自己从写报告跑偏到学 Python 时，可以问 ChatGPT：“我原本的任务是写季度报告，现在需要回到哪个步骤？”它就像 Focusmate 的虚拟身体在场——通过实时问责提供外部结构（来源：11 Best ADHD Productivity Apps for Fluctuating Energy）。但 Focusmate 依赖他人，ChatGPT 则是一个永不下线的“调度秘书”。

## 脚手架 vs. 拐杖：边界在哪里？

这里有一个核心争议：AI 工具是否会导致 ADHD 用户永久依赖外部脚手架，而无法内化执行功能？（来源：拐杖与脚手架概念）曾国藩的日课最终内化成了习惯，但多数人用 ChatGPT 只是临时解决问题，没有训练自己的调度能力。

Brain.fm 和 Forest 这类工具试图通过神经锁相或游戏化机制直接调节大脑状态（来源：Brain.fm 页面；Forest 页面），但它们更像是“情绪调节器”而非“执行功能训练器”。真正的脚手架应该像曾国藩的日记——每次使用都在强化自我监控能力。

## 今天就能试的 3 个行动

1. **设置“重锚定”提示**：在 ChatGPT 中创建一个自定义指令：“每当我偏离任务时，提醒我原始目标，并问我现在该做什么。”这相当于给 LLM 加了一个 system prompt 刷新器。
2. **用 ChatGPT 做“日课”**：每天结束时，让 ChatGPT 总结你今天实际做了什么，对比你原本计划做什么。这模仿了曾国藩的日记反省。
3. **组合工具**：用 RescueTime 追踪时间，当发现漂移时，立即打开 ChatGPT 进行重锚定对话。不要让它替你完成，而是让它帮你重新定位。

## 局限与争议

必须诚实指出：ADHD 大脑与 LLM 的同构目前仍是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。不同工具的证据也参差不齐：Brain.fm 缺乏 ADHD 人群的独立临床研究，Focusmate 的效果主要基于用户报告（来源：Brain.fm 页面；Focusmate 页面）。此外，过度依赖 ChatGPT 可能削弱你本就脆弱的自我调度能力——这正是“拐杖”的风险。

但有一点是确定的：无论对 ADHD 大脑还是 LLM，重锚定都是防止目标漂移的核心策略。区别只在于，你是主动设计你的脚手架，还是被动等待下一次跑偏。

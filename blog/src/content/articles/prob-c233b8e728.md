# 为什么用 Reflect 治 ADHD 的时间盲，和给 agent 套 planner-executor 任务分解 是一回事？

> Mem 篇讲了「对过去的盲」:没有轨迹,planner 就没有训练数据。Reflect 这类以每日笔记(daily notes)为骨架的反链系统,在同一个战场上提供了一个更结构化的方案,值得单独一篇——因为它的架构恰好对应 agent 工程里一个具体而重要的物件:**checkpoint(检查点)。把系统状态按固定节律落盘,是长时程任务不解体的底层机制**。而「长时程任务的解体」,正是 ADHD 时间盲在天、周、月尺度上的形态。

先看这个尺度上的时间盲长什么样,它和「迟到、估错」不是一回事:周一想的事,周三完全断线(不是忘了内容,是忘了「我正在做这件事」);一个项目推进到一半,中断一周后回来,面对自己的半成品像面对陌生人的遗物;月底回望,明明每天都在忙,却拼不出「这个月推进了什么」的叙事。机制是本系列的老朋友们在长时程上的合谋:**状态不持久(工作记忆不跨天)+时间标签弱(事件连不成线)+重锚定缺失(没有固定时刻回到目标)**——每一天都是孤岛,而项目需要群岛连成大陆。

Agent 工程对同构问题的解法,恰好也是本系列的老朋友,但这次看它的节律性:**checkpoint 不是「需要时才存」,是按固定间隔强制存**——每 N 步落一次盘(当前状态、已完成、下一步),不管当时觉得需不需要;因为工程学到的教训是:**崩溃不预约,而「觉得不需要存」的时刻恰恰是状态最满、损失最大的时刻**。恢复协议同样标准化:重启后第一动作是读最近的 checkpoint,而不是凭印象重建。

Reflect 式每日笔记的设计,就是把 checkpoint 做成了人类节律:**①每天一页,自动创建**——今天的页面就是今天的检查点容器,「要不要记」的决策被日历替你做了(节律强制,不靠自觉——和 agent 的固定间隔同一逻辑);**②反链把孤岛连成线**——今天笔记里提到项目 X,自动链到 X 的所有历史提及:**打开项目页,它的整条时间线(哪天想了什么、决定了什么)自动铺开**——Mem 篇的时间线靠检索重建,反链版是结构自动生成,断线一周后的「遗物感」被这条线直接治愈:你不是面对陌生的半成品,是沿着自己的脚印走回现场;**③晨间三行=恢复协议**——每天第一件事,在今日页写三行:昨天到哪了/今天推进什么/卡在哪——这就是「重启后先读 checkpoint」的人类版,五分钟,把跨天断线焊上。**周记月记同理,是更大间隔的 checkpoint**:周日一段「本周叙事」,月底一段「本月叙事」——月底拼不出的那个故事,原来是要每周存一段的。

限制,照例两条:**其一,checkpoint 依赖写入纪律**——每日页自动创建,但内容要你写;好在这是本系列最低门槛的写入(三行,不分类,不组织),且晨间固定时刻可以绑进例程(Routinery 篇的编译);**其二,别让检查点膨胀成日记工程**——有人把晨间三行写成四十分钟的晨间仪式,checkpoint 变成新的拖延舞台;工程规格就是三行,**checkpoint 的美德是薄**——存状态,不是写回忆录。

## 边界

同构定位(本文未做正式 A/B/C 分级):ADHD 的前瞻记忆与跨期目标维持困难有实验基础,checkpoint/恢复是 agent 与系统工程的标准机制,「节律性状态持久化」的同构清晰;Reflect 无 ADHD 对照研究,每日笔记法的直接证据为实践性。声明:笔记法是结构辅助;长时程的全面解体若伴随情绪症状,评估优先。

## 今天就能试的 3 件事

1. **今晚存第一个 checkpoint**:三行——今天到哪了/明天推进什么/卡在哪。就三行。
2. **给最重要的项目建一页**:把散在各处的相关记录链(或粘)进去——先让它的时间线存在。
3. **周日加一段周叙事**:五句话讲完本周——月底的故事,是每周存一段攒出来的。

天、周、月尺度的时间盲,不是记性差,是**状态从不落盘**——每一天都在裸奔,崩一次断一次。三行字的 checkpoint,是长时程任务最便宜的保险;agent 靠它跑完长任务,你也可以。

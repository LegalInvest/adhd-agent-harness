# Routinery 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Routinery：它到底补上了哪一层执行功能？

## 问题：为什么你的 Routinery 用了一周就废了？

你下载了 Routinery，精心设置好早间流程：起床、喝水、冥想、写晨间笔记。第一天，完美执行；第三天，开始拖延；第七天，图标在手机里吃灰。这不是你的错——你把 Routinery 当成了“自动执行器”，但它真正该做的，是成为你的“调度层”。

ADHD 大脑与 LLM 是同构的：两者都是强大的生成核心，但缺乏可靠的内置调度层（来源：生成核心与调度层）。ADHD 患者“常常感觉方向盘后没有人”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），就像 LLM 单独使用时缺乏执行调度，容易“上下文膨胀和推理退化”（来源：Building AI Coding Agents for the Terminal）。Routinery 这类工具，本质上就是给大脑套上的 harness——就像孔子用“礼”作为行为边界，释迦牟尼用“戒律”拴住乱跑的念头。

## 同构：ADHD 大脑 = LLM，Routinery = planner-executor 框架

现代 AI 编码代理通过“分离规划与执行的双代理架构”来弥补 LLM 的缺陷（来源：Building AI Coding Agents for the Terminal）。ADHD 大脑同样需要这种分离：规划（“我想早起”）与执行（“现在起床”）之间缺一个调度器。Routinery 扮演的正是这个角色——它把“早起”分解成具体步骤，按时间顺序排好，并在每一步给你一个外部信号。

孔子是这种 harness 的早期实践者。他精力旺盛、坐不住、冲动爱骂人，思维跳跃到《论语》全是场景化语录。他的 harness 是“克己复礼”：用外在的“礼”作为行为边界，每日“吾日三省吾身”。这相当于一个定时 re-grounding 机制——就像 LLM 需要定期上下文压缩防止漂移。释迦牟尼的“八正道”和“持戒”则是另一套 harness：用戒律管行为，用正念拴念头，相当于给大脑装了一个外部调度器，防止注意力散乱。

LLM 侧的证据同样清晰：ADHD 患者的工作记忆“强记忆、弱控制”（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs），与 LLM 惊人相似。Transformer 的自注意力机制在训练后模仿了前额叶-纹状体的门控操作（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS），证明了两者在计算层面的同构性。

## 工具实测：Routinery 到底补上了哪一层？

Routinery 的核心功能是“任务分解 + 时间锚定”。它不像 Motion 那样自动排程（Motion 通过 AI 动态调整日程，来源：Motion），也不像 Reclaim.ai 那样防御性保护时间（来源：Reclaim.ai），而是让你手动设定步骤序列，然后按固定节奏推进。这正好补上了 ADHD 最缺的“执行调度层”：

- **任务启动困难** → Routinery 用第一步的视觉提示取代“该做什么”的决策负担。
- **不会拆解** → 它强迫你把“写论文”拆成“打开电脑→新建文档→写标题→写第一句”。
- **时间盲** → 每一步的倒计时让时间流逝变得可感，类似 Tiimo 的视觉化时间线（来源：Tiimo）。

但问题在于：很多人把 Routinery 用成了“拐杖”，而不是“脚手架”。拐杖替你走路，脚手架帮你学会走路。如果你依赖 Routinery 的每一步提示才能行动，一旦工具崩溃，你就彻底瘫痪。真正的脚手架应该像 Goblin Tools 的“任务分解”功能（来源：Goblin Tools），帮你学会自己拆解，而不是永远依赖外部。

## 争议与局限：脚手架 vs 拐杖的边界

“AI 工具可能从‘脚手架’退化为‘永久拐杖’，削弱用户内在执行功能的代偿发展”（来源：拐杖与脚手架）。目前没有长期研究评估这种依赖的后果。Motion 等工具“缺乏独立临床验证”（来源：矛盾与存疑），用户口碑可能受安慰剂效应影响。

更关键的是个体差异：有人对视觉工具（如 Tiimo）反应良好，有人更受益于自动排程（如 Reclaim.ai）。Routinery 适合那些需要“固定流程”的人——比如每天早晨的例行公事——但对日程高度不规律的用户，它反而可能增加认知负荷（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Routinery 拆解一个你最头疼的“大任务”**（比如写周报），分解成不超过 5 分钟的步骤。执行一周后，尝试减少提示频率，训练大脑自己启动。
2. **借鉴孔子的“日课”**：每天固定时间用 Routinery 执行 3 个核心习惯，作为你的“定时 re-grounding”。
3. **对比测试**：如果 Routinery 让你感到窒息，试试 Goblin Tools 的“任务分解”功能，它只帮你拆解，不强制时间——更适合需要灵活性的用户。
4. **设置“无工具日”**：每周一天不用任何调度工具，观察自己如何启动任务。这能帮你区分“脚手架”和“拐杖”。

Routinery 不是万能药。它只是一个 harness——就像孔子 70 岁才做到“从心所欲不逾矩”，你需要的不是完美执行，而是逐步内化那个调度层。

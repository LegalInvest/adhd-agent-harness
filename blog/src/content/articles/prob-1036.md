# Inflow 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Inflow：它到底补上了哪一层执行功能？

## 问题：为什么你的 Inflow 用着用着就丢了？

你下载了 Inflow，跟着每日课程做练习，一开始感觉不错。但三周后，你发现自己连打开 App 的动机都没了——它变成了又一个“应该用但没用”的软件。与此同时，你作为工程师，给 LLM agent 写了一个精心设计的 system prompt，结果跑了十分钟后 agent 开始胡言乱语，完全偏离最初目标。

这两个场景共享同一个底层结构：**生成核心（ADHD 大脑 / LLM）本身高产，但缺乏可靠的执行调度层**。ADHD 的注意力漂移、超聚焦跑偏，与 LLM agent 的上下文溢出、目标漂移，是同一种问题的不同表象。Inflow 声称通过神经科学原理改善执行功能（来源：Inflow 工具页），但它的本质是给大脑套上一个外部 harness——就像 LLM 的编排层。问题在于，**很多人用错了这个 harness**。

## 同构视角：ADHD 大脑与 LLM 共享的缺陷

ADHD 的核心障碍是执行功能缺陷：工作记忆瓶颈、任务启动困难、时间盲、注意力维持失败（来源：执行功能概念页）。LLM 的无状态性、上下文窗口限制、对 prompt 质量的敏感，与之高度对应。两者都是“高产但缺调度”的生成核心。

- **工作记忆 vs 无状态**：ADHD 患者无法在脑中同时保持多个任务细节（来源：工作记忆概念页）；LLM 每次推理都是独立的，除非外部记忆系统介入。RescueTime 自动记录时间使用，充当外部记忆（来源：RescueTime 工具页），类似 LLM 的向量数据库。
- **上下文工程 vs 环境设计**：ADHD 易因环境干扰丢失焦点，Brain.fm 通过神经锁相音乐维持连贯性（来源：Brain.fm 工具页）；LLM 需要精心设计的 system prompt 和 RAG 来保持上下文。
- **身体在场效应 vs 外部问责**：Focusmate 通过 AI 匹配虚拟同伴提供实时问责（来源：Focusmate 工具页），类似 LLM agent 的 human-in-the-loop 检查点。

这些工具的共同点是：**它们不是修复大脑，而是绕过缺陷**。就像 LLM 的 harness 不改变模型参数，只改变输入输出结构。

## Inflow 补上了哪一层？

Inflow 的设计目标是“通过神经科学原理改善执行功能”。但根据现有资料，它缺乏独立临床研究支持其有效性（来源：矛盾与存疑）。它的课程、追踪、社区功能，本质上是在大脑外部搭建一个**调度层**：提醒你该做什么、分解任务、记录进度。这恰恰是 LLM agent 的编排层在做的事——将复杂目标分解为子步骤，监控进度，重定向。

但这里有一个关键区别：**LLM 的 harness 是工具，ADHD 的 harness 应该是训练支架**。LLM 不需要“学会”自己调度，它只需要被正确编排。而 ADHD 大脑有神经可塑性，理论上可以通过训练逐步内化执行功能。如果 Inflow 只是永久替代，它就成了拐杖，而非脚手架（来源：矛盾与存疑）。

## 脚手架 vs 拐杖：被忽略的边界

wiki 资料中反复出现一个矛盾：AI 工具作为外部执行功能层有效，但长期依赖可能阻碍内在能力发展（来源：矛盾与存疑）。多数工具（如 Goblin Tools、Saner.AI、Motion）设计为长期使用，没有撤除机制。这意味着用户可能永远需要它们，就像 LLM 永远需要编排层——但大脑不是模型。

真正的脚手架应该具备**可撤除性**：比如，先用 Focusmate 建立工作节奏，然后逐步减少配对次数；先用 RescueTime 觉察时间模式，然后内化时间感知。但现有工具没有这种设计。Inflow 的课程模式看似有“训练”成分，但课程结束后呢？它仍然鼓励你继续使用 App 作为日常伴侣。

## 争议与局限

必须诚实指出：ADHD 大脑与 LLM 的同构命题本身缺乏大规模实证，主要基于概念类比和工具案例（来源：矛盾与存疑）。Brain.fm 的神经锁相技术针对 ADHD 的效果缺乏独立临床研究（来源：Brain.fm 工具页）。多巴胺干预的有效性存在争议（来源：矛盾与存疑）。这些工具的效果多基于用户主观报告，而非随机对照试验。

此外，ADHD 亚型差异巨大：注意力缺陷型可能从 RescueTime 的自动追踪中获益，而多动冲动型可能更需要 Focusmate 的身体在场。现有工具未充分个性化（来源：矛盾与存疑）。

## 今天就能试的行动

1. **区分工具与训练**：将你用的工具分为两类——一类是“永久拐杖”（如 RescueTime 自动记录），一类是“可撤除支架”（如 Focusmate 配对）。为后者设定撤除计划，比如每两周减少一次配对。
2. **做一次“重锚定”实验**：每天开始工作前，花 2 分钟用语音记录目标（类似 LLM 的 system prompt）。当感觉漂移时，回放录音。这训练了大脑的自我重锚定能力。
3. **检查工具的“可撤除性”**：查看你常用的 ADHD 工具是否有“减少使用”或“毕业”模式。如果没有，考虑替换为那些设计有退出路径的工具（如 Tiimo 的日程结构可逐步内化）。
4. **用 LLM 类比诊断自己的漂移**：当你在超聚焦中跑偏时，问自己：“如果这是一个 LLM agent，它的 system prompt 现在是什么？哪里需要重写？” 这能帮你快速识别注意力漂移的触发点。

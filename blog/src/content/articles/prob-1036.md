# Inflow 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Inflow：它到底补上了哪一层执行功能？

## 问题：分心走神或超聚焦跑偏，丢失目标——ADHD 与 LLM 的同一困境

你有没有过这样的经历：打开电脑准备写一份报告，半小时后却发现自己正在刷短视频；或者，你明明计划花两小时完成一个项目，结果因为某个细节过度钻研，六小时后才回过神来，而原定目标早已偏离。这种“分心走神”或“超聚焦跑偏”导致的**目标漂移**，是 ADHD 人群最核心的痛点之一（来源：[[超聚焦]]）。

有趣的是，做 Agentic Harness 工程的工程师们，也在为同样的问题头疼。他们发现，一个长程任务的 LLM agent，在运行过程中会逐渐偏离初始指令——上下文膨胀、中间输出干扰、任务分支误导——最终产出一个与原始目标毫不相关的结果。于是，他们引入了“重锚定 system prompt”技术：定期将 agent 的上下文重置到核心目标上，就像在茫茫大海中不断校准航向。

这两件事，本质上是同一类问题。ADHD 大脑与 LLM/agent，都是**高产但缺乏可靠执行调度层的生成核心**（来源：[[AI 与 ADHD 的专注力]]）。ADHD 拥有强大的创意发散能力（生成层），但执行功能——任务启动、时间感知、注意力维持——却像缺失的调度层，导致输出不稳定。LLM 同样如此：它能生成流畅文本，但一旦缺乏上下文控制，就会像脱缰的野马。

那么，Inflow（一款专为 ADHD 设计的 AI 工具）究竟做了什么？它试图扮演“重锚定 system prompt”的角色。但问题是，**有人用错了它**。

## Inflow 的本质：外部执行功能层，而非生成加速器

Inflow 的设计理念，是给 ADHD 大脑提供一个**外部执行功能层**（来源：[[AI 与 ADHD 的专注力]]）。它通过结构化提示、定时提醒、任务分解等功能，帮助用户**重新锚定注意力**到当前最重要的目标上。这就像 LLM agent 的“重锚定 system prompt”：定期把 agent 的上下文拉回正轨。

但很多人误用了 Inflow——他们把它当成一个“生成加速器”，希望它能直接帮自己完成工作（比如自动写邮件、生成创意）。这相当于给 LLM agent 加了一个更强的生成模型，却不解决目标漂移问题。结果，agent 可能会生成更多、更华丽但偏离目标的输出。同样，ADHD 用户如果只用 Inflow 来“激发灵感”，而不利用它的执行调度功能，只会陷入更深的超聚焦跑偏。

**Inflow 真正补上的，是执行调度层，而非生成层。** 它的价值在于：当你的大脑开始漂移时，它像 system prompt 一样把你拉回来。

## 同构证据：曾国藩的“日课”与 LLM 的“定时 re-grounding”

历史人物曾国藩，是 ADHD 特质的典型代表。他 30 岁前浮躁坐不住，天天串门喝酒，日记里骂自己“无恒”“浮躁”，读书速度极慢，“他人目下二三行，余或疾读不能终一行”。但他通过一套自创的 harness 系统——**日课十二条**（黎明即起、读书不二、谨言、写日记反省等）——成功管理了自己。这套系统本质上就是“定时 re-grounding”：每天固定时间反省、固定时间读书，用最笨最稳的方法抵消自己的冲动冲动。

这与 LLM agent 的“重锚定 system prompt”惊人同构。曾国藩的“日课”相当于一个外部调度器，定期将他的注意力重新锚定到核心目标（修身、齐家、治国）上。他不需要更强的生成能力（他本来就有），而是需要可靠的执行调度层。

现代工具 Focusmate 也采用了类似原理：通过虚拟身体在场（AI 匹配的实时问责伙伴），提供外部结构和社会压力，弥补内在多巴胺调节不足（来源：[[Focusmate]]）。这相当于 agent 的“外部监督器”——一个定期检查你是否还在轨道上的机制。

## 争议与局限：脚手架还是拐杖？

然而，这种外部执行功能层存在一个核心争议：长期使用是否会导致内在执行功能进一步退化？换句话说，Inflow 是**脚手架**（辅助你学会自己走路）还是**拐杖**（让你永远依赖它）？（来源：[[矛盾与存疑]]）

目前缺乏纵向研究来回答这个问题。但我们可以从同构视角得到一个判断：一个好的 harness 应该像曾国藩的日课——它最终内化成你的习惯，而不是永远需要外部提醒。曾国藩晚年不再需要写日记骂自己，因为自律已经内化。同样，一个好的 AI 工具应该逐步减少依赖，而不是让你永远离不开它。

此外，个体差异巨大。对于某些 ADHD 亚型，强行打断（如番茄钟）反而有效，而柔性引导（如 Inflow 的温和提醒）可能不够强力（来源：[[矛盾与存疑]]）。这就像不同 agent 需要不同的重锚定频率——有些 agent 每 5 步就需要一次，有些 50 步才需要。

## 今天就能试的行动

1. **设置“重锚定”闹钟**：每 25 分钟，用手机闹钟问自己：“我现在在做的事，和最初目标一致吗？”这相当于手动执行“重锚定 system prompt”。
2. **使用 Focusmate 进行一次身体在场 session**：预约 50 分钟，和另一个人视频共事，体验外部调度层的力量（来源：[[Focusmate]]）。
3. **用 RescueTime 追踪一天时间**：看看你的注意力到底漂移到了哪里，识别出最常导致目标丢失的“上下文膨胀”点（来源：[[RescueTime]]）。
4. **尝试 Brain.fm 的专注音乐**：在需要深度工作时播放，利用神经锁相技术辅助注意力维持，但注意它只是辅助，不能替代调度层（来源：[[Brain.fm]]）。

## 结语

Inflow 之于 ADHD，就像重锚定 system prompt 之于 LLM——它补上的是执行调度层。但如果你把它当成生成加速器，或用它来逃避自我管理，它就变成了拐杖。真正的解决方案，是像曾国藩一样，搭建一个属于自己的 harness 系统，让 AI 成为你“日课”的一部分，而非全部。

# 用 ChatGPT 给 ADHD 补上ADHD 大脑与 LLM 的同构，是脚手架还是拐杖？

> ChatGPT 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：你的大脑，到底是怎么回事？

如果你是个ADHDer，你可能无数次问过自己：为什么我能在截止日期前爆发出惊人的创造力，却连一封简单的邮件都迟迟无法开始？为什么我的想法像瀑布一样奔涌，但执行起来却像在泥潭里挣扎？

如果你是个在做Agentic Harness工程的工程师，你大概也遇到过这样的困惑：LLM明明能生成惊艳的代码片段，但一旦需要多步推理、跨会话上下文或自主调度，它就变得不可靠，像一台没有操作系统的超级计算机。

这两个问题，其实是同一个问题。

## 同构脊柱：生成核心与调度层的分离

最新研究揭示了一个惊人的平行关系：ADHD大脑与LLM在系统架构上同构——两者都是“高产但缺执行调度层的生成核心”。ADHD的困境不是智力不足，而是执行功能（executive function）的失效——工作记忆不稳定、时间感知困难、任务启动障碍（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。而LLM作为生成核心，拥有强大的语言理解和生成能力，但单独使用时缺乏可靠的调度层，容易陷入“上下文膨胀和推理退化”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

这种同构甚至深入到认知机制层面。研究发现，ADHD患者在工作记忆任务中易受干扰，而LLM同样表现出类似人类的干扰特征——随着记忆负荷增加表现下降，受近因效应和刺激统计偏差影响（来源：Human-like Working Memory Interference in Large Language Models）。更令人震惊的是，Transformer的自注意力机制在训练后发展出模仿前额叶-纹状体门控的操作，证明了计算层面的同构性（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。

## 案例：毛泽东的Harness与LLM的Scaffolding

毛泽东是典型的ADHD特质拥有者。他小时候是问题儿童，爱读闲书不喜欢四书五经；一辈子读书不辍，行军路上也带着书；思维极度跳跃，文章讲话充满比喻；冲动敢赌，四渡赤水、抗美援朝都是险棋。但他建立了一套强大的外部调度系统：批评与自我批评（让集体提意见，制衡冲动）、调查研究（“没有调查就没有发言权”）、民主集中制（用集体决策替代个人冲动）、党指挥枪（用组织系统管理军队和国家）。这套harness本质上就是ADHD的外部执行功能层——与LLM的agent scaffolding同构：后者同样需要“工作负载专用模型路由、分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。毛泽东的“日课”相当于定时re-grounding，“秘书”相当于外部调度器。

## 工具评测：脚手架还是拐杖？

当前市面上针对ADHD的AI工具，本质都是在搭建这种外部调度层。

**Goblin Tools** 的核心功能是任务分解。它的Magic ToDo能将“清理厨房”这样的模糊指令分解为具体可执行的子任务，用户可调节“辣度”控制分解粒度（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。这相当于为LLM提供结构化的任务链，降低启动门槛。但问题在于：如果用户永远依赖它来分解任务，自己的任务组织能力是否会退化？

**Saner.AI** 专注于知识回忆和本地记忆，减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。它相当于ADHD的向量数据库——外部记忆系统。但同样，如果用户不再训练自己的工作记忆，依赖是否会成为新的障碍？

**Motion** 则通过AI自动排程与动态调整来消除“下一步该做什么”的决策负担（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。它持续评估任务优先级和截止时间，实时重建日程。这就像给LLM加了一个自动规划器，但初始设置成本高，且用户反馈称学习使用工具本身增加了认知负荷（来源：矛盾与存疑）。

## 边界：脚手架 vs 拐杖

核心判断：AI工具是脚手架还是拐杖，取决于使用者是否保留了调度层的最终控制权。真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。如果工具替代了治疗或自我管理，就可能造成依赖（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

从工程角度看，这意味着ADHD的AI工具应该设计为“可拆卸的”——就像LLM的agent scaffolding应该允许人类在关键决策点介入。例如，Goblin Tools的“辣度”调节就是一种控制权保留机制。但从证据来看，目前缺乏长期研究验证依赖风险（来源：矛盾与存疑）。

## 争议与局限

必须诚实指出，ADHD与LLM的同构假设目前主要是概念类比和初步实证，缺乏大规模随机对照试验验证（来源：矛盾与存疑）。此外，个体差异极大：部分ADHD用户反馈强行打断（如番茄钟）反而有效，而柔性引导可能不够强力（来源：矛盾与存疑）。AI工具的长期效果也未知——短期有效，但长期可能造成依赖。

4. **记录工具使用日志**：连续一周记录你使用AI工具的频率和感受，特别留意是否出现“没有工具就无法启动”的迹象。这是区分脚手架与拐杖的关键指标。

# 为什么用 Structured 治 ADHD 的情绪失调，和给 agent 套 会褪去的脚手架 是一回事？

> Structured 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

如果你被 ADHD 的情绪失调折磨过，你可能听过这样的建议：「放松一下」「别想太多」「你要有自制力」。这些建议之所以无效，是因为它们误把情绪问题当成情绪本身的问题。更准确的解释是：情绪失调是执行调度失败的下游结果——注意力漂移、时间感扭曲、任务启动困难、冲动抑制失败，这些执行功能障碍先崩掉，情绪才跟着崩。

AI 工程师不会对这个描述感到陌生。LLM 与 agent 的崩溃模式如出一辙：生成核心极其高产，但在长程一致性、事实 grounding、目标保持、记忆管理上频频失控。我们给 agent 加的脚手架——外部调度器、检索记忆、反思循环、human-in-the-loop——本质上不是为了让模型「更聪明」，而是为了一套失控的生成核心补上执行调度层。 ADHD 与 agent，两边用的竟然是同一套 harness 思路。

## 同构原型：高产生成核心 + 薄弱执行调度层

把 ADHD 大脑和 LLM 并置， wiki 资料里提出的是一个「同构」原型：两者都是强大的生成核心，却缺乏可靠的执行调度层。ADHD 侧的痛苦不是缺乏想法或动力，而是想法太多、执行太少、上下文切换过于频繁；LLM 侧的问题不是生成能力不足，而是生成内容可能偏离目标、编造事实、忘记 earlier context。这种相似性在工作记忆和外部记忆两个维度上尤其明显：ADHD 工作记忆是「弱控制」，LLM 虽容量大但同样是「弱控制」，注意力漂移是共同机制；两者都依赖外部记忆系统来补偿内部状态管理（来源：ADHD 大脑与 LLM 的同构）。

情绪调节在这里可以被重新理解：它不是一种需要被「修好」的内在情绪品质，而是一系列需要被外部化的小任务——注意锚定、计划分解、时间估计、社会在场、启动支持。当这些子任务被外部 harness 承接，生成核心就能留在它擅长的领域，而不是在混乱中情绪过载。这个观点的直接推论是：AI 对 ADHD 情绪调节最有价值的帮助，不是「治愈情绪」，而是把情绪调节所需的执行子任务外包出去（来源：AI 与 ADHD 的情绪调节）。

## 真实 harness 系统：释迦牟尼的「脚手架」与 agent 的会褪去脚手架

一个能同时打动 ADHD 读者和 AI 工程师的案例，是佛教创始人释迦牟尼。资料中对他的标注是「行为模式匹配」（behavioral pattern match），而非现代诊断。他的 ADHD 特质非常典型：29 岁突然弃王位、妻子、孩子出家求道；先后跟随不同仙人学习、苦行六年又放弃，不断试错；49 年游行说法，80 岁死在传道途中；说法全部用场景化比喻，自己一字未写。他的生成核心极其高产、极具创造力、极其冲动地追求新意义，但如果没有一套外部 harness，这种能量会散掉。

他的 harness 系统是什么？以「八正道」「持戒」为核心，用戒律管理行为，用正念拴住乱跑的念头；最终强调「自依止、法依止」，不依赖权威，自己证悟。这完全可以翻译成 agent 工程的术语：

- **持戒 ≈ guardrails**：预先写死的行为边界，降低冲动决策成本；
- **正念 ≈ 定时 re-grounding**：把注意力从漂移状态拉回当前任务上下文；
- **自依止、法依止 ≈ 脚手架褪去**：外部结构最终要让位于内部稳定机制，而不是让人永远依赖拐杖。

这与张居正的「考成法」形成有趣对照：张居正用严格考核制度管别人也管自己，是更强的外部调度器；释迦牟尼则是把外部规则内化为自我管理系统。但两者共享同一结构：一个高产出但易漂移的生成核心，需要外部 harness 来保持稳定输出。李芳的投资数据也提供了另一个侧面：ADHD 创业者群体的失败率为 1.2，对照组为 1（来源：Internal Analysis），差距不大，说明 harness 系统足够好时，ADHD 特质可以从风险变成韧性、创造力与系统思维的来源。

## 工具映射：ADHD 侧与 agent 侧的证据

在 ADHD 侧， harness 已经以产品形态出现。Inflow 是一个基于 CBT 的综合程序，提供结构化技能训练与社区支持，它帮助用户理解挑战背后的「为什么」，并通过自定进度学习建立新技能（来源：The 12 Best Apps for ADHD in 2026: A Guide to Finding What ...）。Goblin Tools 的 Magic ToDo 把模糊任务——比如「清理厨房」——分解为具体可执行的子任务，用户可以调节「辣度」控制粒度，降低启动摩擦（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。Saner.AI 则通过本地记忆和知识回忆减少搜索循环与标签切换，对抗工作记忆不足带来的认知超载（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。

这些工具在 agent 侧都有对应物：Goblin Tools 的任务分解对应 agent 的 subgoal planner；Saner.AI 的知识回忆对应外部记忆库（vector store / RAG）；Inflow 的结构化训练对应 few-shot prompting 与反思循环；身体在场效应的 AI 化扩展，对应 agent 的 human-in-the-loop 与监督信号。两边的关键问题不是「有没有工具」，而是「这些工具是否构成有效的 harness」——是否能承接执行调度，而不是让生成核心更忙。

## 脚手架 vs 拐杖：边界在哪里

同构脊柱最有价值的地方，是帮我们区分「脚手架」和「拐杖」。

脚手架是**临时外部结构，目标是在使用过程中让内部能力生长，最终可被撤走**。拐杖是**永久依赖，使用越久越离不开，甚至可能增加负担**。很多 ADHD 工具声称自己是脚手架，实际上却可能变成拐杖：如果提醒 app 让你停止发展内部时间感，如果 AI 写作让你完全放弃组织思路，那它就不是 harness，而是新的执行功能障碍。

这个边界在 agent 工程里同样关键。会褪去的脚手架（fading scaffolding）意味着：随着模型内部调度能力增强，外部检查、记忆检索、人类反馈的介入频率应该逐步降低。好的 agent 架构不是无限堆叠外部模块，而是持续追问：哪些子任务正在被内部化？哪些外部支持可以淡出？

wiki 资料也诚实指出了这种风险的证据不足：Motion 缺乏独立临床验证，Brain.fm 在 ADHD 人群中的独立临床证据仍有限，但一些主题页仍把它们作为有效工具推荐，存在夸大（来源：矛盾与存疑）。同样，ChatGPT、Claude 等工具页面多强调益处，却很少讨论依赖风险。这是我们在两边都要警惕的。

## 局限：同构是启发，不是科学事实

必须承认，ADHD 大脑与 LLM 的同构性目前主要是一种**理论类比**，并非经过验证的科学事实（来源：矛盾与存疑）。把神经系统的情绪调节机制与语言模型的上下文管理直接等同，会犯下过度简化错误。真实人际互动的身体在场效应，也无法被 AI 完全替代；多巴胺视网膜检测等早期研究，远不是成熟应用。我们不能因为两边都有「注意力漂移」这个词，就认为它们共享同一机制。

但正因为它是类比，它才有用。它不是说「ADHD 大脑就是 LLM」，而是说：两者的**失控模式足够相似**，以至于同一类工程干预——外部化调度、记忆、注意、时间感——可以在两边产生类似效果。这种启发式价值，不需要被证明为科学事实也能成立。

## 今天就能试的 4 个行动

1. **建立一个「每日 re-grounding」锚点**：像正念一样，在每天固定时间用 2 分钟写下当前优先级和情绪状态。这是最低成本的 external scheduler，帮助自己从漂移中回到上下文。

2. **把情绪触发任务交给 Goblin Tools 式分解**：下一次因为某项任务而焦虑时，把「我要完成 X」输入任务分解工具，把它拆成 3-5 个第一步就能做的微动作，降低启动摩擦。

3. **用外部记忆 inbox 替代大脑缓存**：不要让待办、灵感、截止日期占用工作记忆。用一个统一的收件箱（可以是 Saner.AI，也可以是简单笔记本）作为外部记忆层，减少标签切换。

4. **设计「褪去」条件**：如果你是工程师，给 agent 的脚手架设定淡出指标——例如当内部反思循环连续 10 次成功 self-correct 后，减少外部检索频率；如果你是 ADHD 使用者，每两周问自己：这个工具是在帮我建立能力，还是替我做决定？

情绪失调不是性格缺陷，也不是单纯的神经化学问题。它是一个调度问题。而调度问题，两边的答案是一样的：先承认生成核心的高产与脆弱，再给它一套会褪去的脚手架。

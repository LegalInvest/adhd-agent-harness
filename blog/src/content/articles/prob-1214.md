# 用 Claude 给 ADHD 补上工具使用与认知卸载，是脚手架还是拐杖？

> Claude 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

如果你是一个 ADHD 成人，你大概熟悉这样的场景：脑子里同时盘旋着十来个想法，每一个都闪闪发光，但就是没法坐下来把其中一个变成可执行的步骤。如果你是一个 Agentic Harness 工程师，你大概也熟悉类似的困境：LLM 能生成漂亮的代码和逻辑，但一旦脱离精心设计的上下文和工具链，它就会在上下文膨胀中迷失，输出幻觉。

这两类困境在结构上是同构的。ADHD 大脑与 LLM 共享同一个核心矛盾：**强大的生成能力与脆弱的执行调度层并存**（来源：ADHD 大脑与 LLM 的同构）。对 ADHD 来说，这个调度层是执行功能；对 LLM 来说，是 agent harness。而 Claude 这样的对话式 AI，恰好可以同时扮演两边的脚手架——但它究竟是帮你站得更高的支撑，还是让你永远无法独立行走的拐杖？

## 从同构问题出发：生成核心缺什么

先看 ADHD 侧。执行功能缺陷意味着工作记忆不稳定、任务启动困难、注意力容易被新刺激捕获（来源：外部执行功能层）。你并不缺想法，缺的是把想法落地的那套「系统」——就像孔子，一个极可能患有 ADHD 的古代思想家。他精力旺盛到能周游列国 14 年，冲动到见南子后急得对天发誓，思维跳跃到《论语》全是场景化语录，没有一部系统著作。他的解决方案是「克己复礼」——用外在的「礼」作为行为边界，每日反省（「吾日三省吾身」）。这套系统本质上就是一个外部调度层：用固定的仪式和规则来补偿内在执行功能的不可靠。

再看 LLM 侧。一个没有 harness 的 LLM agent 就像没有「礼」的孔子：能生成，但缺乏上下文管理，容易上下文膨胀和推理退化（来源：上下文工程）。Harness 工程的核心就是给它套上外部脚手架——上下文传递、工具接口、验证循环、记忆系统（来源：GitHub - ai-boost/awesome-harness-engineering）。这和孔子用「日课」来 re-ground 自己，用「秘书」来调度行为，在结构上完全对应。

## Claude 作为脚手架：哪些交给它

Claude 这类对话式 AI 可以承接 ADHD 最头疼的几类任务：

- **任务分解**：像 Goblin Tools 的 Magic ToDo 一样，Claude 能把模糊的「整理书房」拆成「清空桌面→分类书籍→擦拭书架→归位」等步骤（来源：Goblin Tools）。你可以调节「辣度」控制粒度，降低启动门槛。
- **外部记忆**：ADHD 的工作记忆缺陷与 LLM 的无状态性同构（来源：无状态与外部记忆）。Claude 的对话历史可以充当临时工作记忆，而 Mem 这类「AI 第二大脑」能持久存储信息（来源：Mem），减少搜索循环。Saner.AI 的本地记忆功能也能快速检索笔记（来源：Saner.AI）。
- **上下文锚定**：ADHD 大脑容易被环境带偏，Claude 可以通过精心设计的 prompt 帮你维持注意焦点。比如用「AI Prompts for Task and Time Management」来引导自己回到任务（来源：上下文工程）。

这些本质上都是「给生成核心套 harness」——和 agent 通过 function calling 把精确计算外包给工具是同一类工程。

## 拐杖的边界：验证循环不能外包

但这里有一条红线。ADHD 大脑的另一个特征是冲动、想当然的判断——行为层面的「幻觉」。LLM 同样天然倾向于产生自信但可能错误的输出（来源：幻觉与验证循环）。Harness 工程的关键组件之一是验证循环：复杂的 harness 不会盲目执行工具，而是实现验证步骤，例如检查输出格式或对代码运行测试用例（来源：同上）。

这意味着，如果你把 Claude 当作无需验证的答案生成器，它就变成了拐杖。比如让 Claude 帮你写邮件，你不检查就直接发送——这等于把本应属于你的验证环节外包了。长期如此，你的内在执行功能会进一步萎缩。

真正的脚手架用法是：**让 Claude 生成草稿，你来验证；让 Claude 分解任务，你来执行每一步并反馈结果**。这和释迦牟尼的「自依止法依止」异曲同工——他用戒律和正念作为外部脚手架，但最终强调「自己证悟」，不依赖权威。你的 Claude harness 也应该是一个可迭代的系统：你不断调整 prompt 和工具链，就像工程师不断优化 agent 的上下文管理和验证循环。

## 争议与局限

必须诚实面对几个问题。首先，「ADHD 与 LLM 同构」目前仅是一种理论类比，缺乏实证基础（来源：矛盾与存疑）。它作为启发式框架很有用，但不应被视为科学事实。其次，多数 AI 工具缺乏针对 ADHD 的严格临床验证，效果多基于用户报告（来源：矛盾与存疑）。第三，Claude 这样的通用工具本身可能增加分心风险——它可能变成「另一个收件箱，另一个提示词，另一个丢失线索的地方」（来源：上下文工程）。

## 今天就能试的行动

1. **用 Claude 做一次「任务辣度」实验**：把一个你拖延已久的任务扔给 Claude，要求它分解成 5 个步骤，然后自己调整粒度，直到觉得「可以开始」。完成后记录哪一级粒度最有效。
2. **建立验证清单**：每次让 Claude 生成内容后，强制自己执行一个三步骤验证——检查事实、检查语气、检查是否符合你的意图。把它写进 prompt 里，让 Claude 也提醒你验证。
3. **设计一个「日课」prompt**：模仿孔子的「吾日三省吾身」，让 Claude 每天固定时间问你三个问题（今天完成了什么？卡在哪里？明天最重要的三件事是什么？），并把回答存入 Mem 或 Saner.AI。
4. **试一次「无 Claude 日」**：每周选一天完全不用 AI 工具处理任务，观察自己哪些地方最吃力。这能帮你识别 Claude 是在补短板还是在替代能力。

脚手架和拐杖的区别不在于工具本身，而在于你是否保留了那个验证循环。孔子 70 岁才做到「从心所欲不逾矩」，说明外部脚手架最终可以内化——但前提是你一直在练习自己走路。

# 为什么用 Goblin Tools 治 ADHD 的情绪失调，和给 agent 套 会褪去的脚手架 是一回事？

> Goblin Tools 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

你坐在桌前，大脑里有一百件事在同时尖叫：要回邮件、要交方案、要给爸妈打电话、要……但身体一动不动。不是懒，是情绪冻住了——因为每一件小事都像一座山。

另一边，你部署的 agent 刚跑完一个任务，却输出了三页无关内容；它明明有足够算力，但就是“忘了”上下文，像脱缰的野马。

这两件事，本质是同一个问题。

## 核心同构：高产却缺调度层的生成核心

ADHD 大脑与 LLM 共享一个底层结构：**强大的生成核心，但缺乏可靠的执行调度层**。

对 ADHD 来说，情绪反应（焦虑、兴奋、挫败）像 LLM 的 token 流一样不断涌现，但前额叶——相当于 agent 的 orchestration 层——无法有效抑制或重定向这些情绪（来源：AI 与 ADHD 的情绪调节）。明尼苏达大学的研究直接证实了这种同构：LLM 表现出“强记忆、弱控制”的执行功能剖面——工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷（来源：工作记忆）。耶鲁大学进一步发现，自注意力机制本身会导致注意力弥散，与 ADHD 注意缺陷的计算本质同源（来源：工作记忆）。

这意味着：**瓶颈不在容量/算力，而在 orchestration 层**。

## 情绪失调的本质：执行功能链条断裂

ADHD 的情绪调节困难，根植于执行功能缺陷：工作记忆不稳定、时间感知模糊、任务启动困难（来源：AI 与 ADHD 的情绪调节）。情绪不是凭空爆炸的，而是因为“启动不了任务→挫败→更启动不了”的死循环。

LLM/agent 也一样：没有外部约束时，它会生成无关输出、丢失上下文、偏离目标——不是因为它笨，而是因为缺少一个外部调度器来给它 re-grounding。

## 解法同构：会褪去的脚手架，而非拐杖

Goblin Tools 的 Magic ToDo 功能，本质就是给 ADHD 大脑套上一个**外部执行功能层**：把模糊任务（“清理厨房”）分解为可执行的小步骤，用户可调节“辣度”控制分解粒度（来源：Goblin Tools）。这相当于给 agent 的 prompt 加上 chain-of-thought 约束——不是替它思考，而是为它搭建一个临时的、可调节的思考轨道。

Inflow 更进一步：基于 CBT 提供结构化路径，训练工作记忆和认知控制（来源：Inflow）。它教用户“如何建立技能”，而不是永久依赖外部提示。这正是“会褪去的脚手架”的精髓——工具最终要内化。

Saner.AI 则通过外部记忆补偿工作记忆缺陷：减少搜索循环和标签切换（来源：Saner.AI）。这对应 agent 的“外部记忆”设计——把上下文存到外部向量库，而不是指望模型自己记住一切。

## 历史中的 harness 系统：释迦牟尼

释迦牟尼 29 岁说走就走，放弃王位出家求道；先后跟两个仙人学、苦行 6 年，不断试错不断放弃；49 年游行说法，80 岁死在传道路上。他的一生是典型的 ADHD 式高产出、高冲动、高试错。他的 harness 是“八正道”和“持戒”——用戒律管行为，用正念拴住念头。他强调“自依止法依止”，不依赖权威，自己证悟。

这个 harness 与 LLM agent 的脚手架完全同构：
- **日课/定时 re-grounding** ↔ 八正道的“正念、正定”每天训练
- **外部调度器（戒律）** ↔ agent 的 prompt 约束和工具调用规范
- **最终内化（自依止）** ↔ 脚手架褪去后模型自己的能力提升

## 边界：脚手架 vs 拐杖

脚手架的目的是**褪去**。如果 Goblin Tools 用了一年后，你离开它仍然无法分解任务，那它就是拐杖。同样，如果 agent 的 prompt 约束永不缩减，模型永远学不会自主规划，那它就是拐杖。

这正是最大的争议：**依赖风险**。过度依赖 AI 工具可能导致“永久拐杖”，削弱内在执行功能发展（来源：AI 与 ADHD 的情绪调节）。当前研究缺乏严格随机对照试验证明这些工具对情绪调节的持续改善（来源：AI 与 ADHD 的情绪调节）。同构假设也有边界：ADHD 大脑的生物复杂性远超 LLM，情绪调节涉及神经递质、环境交互等，单纯“脚手架”类比可能简化问题（来源：AI 与 ADHD 的情绪调节）。

## 今天就能试的 3 件事

1. **用 Goblin Tools 的 Magic ToDo 分解一个让你情绪冻结的任务**：比如“写周报”。调高“辣度”到 5，看它分解成多细的步骤。完成第一步后，观察情绪变化。

2. **给自己设一个“日课”**：每天固定时间（比如早上 9 点）做一件事——写三行情绪日志、或冥想 5 分钟。这是你的“定时 re-grounding”。

3. **为你的 agent 加一个“会褪去的脚手架”**：在 prompt 里加入“请先列出步骤再执行”，并在几次成功运行后逐步移除约束，测试模型是否能自主保持结构化输出。

## 结语

情绪失调不是你的错，agent 跑偏也不是模型的错——都是生成核心缺调度层的症状。Goblin Tools、Inflow、Saner.AI 不是拐杖，是脚手架。用它们，但别忘了最终目标是：**离开它们，你也能走**。

# Reflect 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Reflect：它到底补上了哪一层执行功能？

## 问题：为什么你（ADHD 用户）和 LLM agent 工程师在看同一张蓝图？

如果你是一个 ADHD 成人，你一定经历过这样的场景：脑子里有一个清晰的目标（比如“写一份报告”），但身体却像被钉在椅子上，无法启动。即使启动了，也很快被“先查个资料”带偏，或者卡在“该先写哪一部分”的犹豫中。与此同时，在 AI 工程领域，开发者们正为一个类似的问题头疼：LLM 能生成惊艳的文本，但让它完成一个多步骤的长期任务（比如“规划一次旅行”），它要么忘记步骤，要么中途跑偏。于是，工程师们设计了 **planner-executor 循环**——让 agent 先分解任务为子步骤，然后逐步执行、检查结果、调整计划。

这两个场景的痛点是同构的：**ADHD 大脑与 LLM 都是高产的生成核心，但缺乏可靠的执行调度层**（来源：核心论点）。ADHD 的执行功能缺陷——任务启动困难、不会拆解、时间盲——本质上就是调度层的缺失。而 AI 工具作为外部执行功能层，其设计是否有效，取决于它能否像 agent harness 那样，提供结构化的任务分解和动态反馈，而不仅仅是记录或提醒。

## 答案：Reflect 之类工具的关键——它补上了哪一层？

许多 ADHD 工具（如 Tiimo、Structured）通过视觉化时间线帮助用户感知时间，直接应对时间盲（来源：Tiimo）。Motion 和 Reclaim.ai 则更进一步，自动排程和动态调整日程，减少“下一步该做什么”的决策负担（来源：Motion；Reclaim.ai）。这些工具都提供了外部调度层，但它们的核心差异在于：**是只解决了“时间感知”和“提醒”，还是真正解决了“任务分解”和“执行反馈”**？

以 Reflect 为例（假设它是一个日记/反思类工具，类似 Saner.AI 的日志功能），如果它只是让用户记录“今天做了什么”，那它仅仅是一个外部记忆层——弥补了 ADHD 的工作记忆缺陷（无状态性），但并未触及任务分解和启动困难。真正的“脚手架”应该像 agent 的 planner-executor 那样：先帮用户把“写报告”分解为“列大纲→查3篇资料→写引言→写主体→写结论→检查格式”，然后引导用户逐步执行，并在每步完成后提供反馈（如“完成！下一个步骤是……”）。Goblin Tools 的“任务分解”功能正是这个思路——它用 AI 将模糊目标拆成可执行的小步骤，降低启动门槛（来源：Goblin Tools 设计逻辑）。

**我的核心观点是：当前多数 ADHD 工具停留在“外部记忆”或“外部时间感知”层，而真正有效的工具必须提供“外部任务分解+执行反馈”层，即 planner-executor 循环。** 否则，它们只是拐杖，而非可撤除的脚手架。

## 证据：ADHD 侧与 LLM 侧的真实对应

**ADHD 侧：** 多项资料指出，ADHD 个体的执行功能缺陷核心包括任务启动困难和不会拆解（来源：执行功能缺陷）。身体在场效应（Body Doubling）通过提供隐性问责和社会存在感，绕过部分执行功能瓶颈——这类似于 agent 的“执行反馈”机制，因为有人在旁相当于一个外部监督者（来源：身体在场效应）。但身体在场效应依赖他人，无法培养独立能力。而 Motion 和 Reclaim.ai 的自动排程，虽然减少了决策负担，但用户仍需自己输入所有任务和截止日期，初始设置对执行功能弱的用户是挑战（来源：Motion 局限）。

**LLM/Agent 侧：** 当前 agent 工程中，planner-executor 循环的典型实现是：LLM 生成一个任务列表，然后逐个执行，每步检查结果，必要时重新规划。这直接对应 ADHD 所需的“分解-执行-反馈”循环。例如，一个旅行规划 agent 会先分解为“查航班→查酒店→查景点→生成行程”，然后逐步执行，而不是一次生成整个计划。这种结构化的分解显著提高了长任务的成功率。

## 争议与局限：脚手架 vs 拐杖的边界

尽管同构命题有吸引力，但必须诚实指出其局限。首先，**ADHD 大脑与 LLM 的同构目前主要是概念类比，缺乏大规模实证支持**（来源：矛盾与存疑）。其次，**过度依赖外部工具可能阻碍内在执行功能的发展**——如果工具始终替用户分解任务，用户可能永远学不会自己拆解（来源：拐杖与脚手架）。多数工具（如 Motion、Goblin Tools）设计为长期使用，未提及撤除机制，这令人担忧（来源：矛盾与存疑）。最后，多巴胺干预的有效性仍存争议，工具通过即时反馈影响多巴胺的证据尚弱（来源：多巴胺干预争议）。

因此，理想的 ADHD 工具应像 **可逐步撤除的脚手架**：初期提供完整的 planner-executor 循环，但逐渐减少干预，让用户练习自己的执行功能。例如，先从自动分解任务开始，然后过渡到用户自己列出步骤、工具只做检查，最终完全独立。目前几乎没有工具实现这种设计。

## 今天就能试的行动

1. **用 Goblin Tools 的“任务分解”功能**：把一件你拖延很久的事（比如“整理房间”）输入，看它如何拆解成小步骤。执行其中一步，感受启动门槛的降低。
2. **结合 Motion 或 Reclaim.ai 的自动排程**：设置一个“深度工作保护块”，然后观察它如何帮你抵御会议干扰。注意：必须提前输入所有任务，否则无效。
3. **尝试身体在场效应**：使用 Focusmate 匹配一个虚拟伙伴，一起工作25分钟。注意这种“隐性问责”是否让你更容易启动（来源：身体在场效应）。
4. **手动实践 planner-executor 循环**：拿一张纸，写下你的目标，然后分解成不超过5个步骤。每完成一步，打个勾，并写下“下一步做什么”。重复一周，观察你的任务完成率是否提升。

记住：工具是脚手架，不是拐杖。最终的目标是让你自己的执行功能逐步强大起来，而 AI 只是过渡期的训练轮。

# 如果把 ADHD 大脑当成生成核心，ChatGPT 能不能当它的幻觉与验证循环？

> 用 agent 工程的思路重新理解 ChatGPT 对 ADHD 的真正价值。

你打开 ChatGPT，输入一个模糊任务——“帮我规划下周的工作”。几秒后，它输出一份详尽的计划，步骤清晰，时间合理。你松了口气，觉得终于有人替你思考了。但第二天，你发现计划完全无法执行——你忽略了某个截止日期，低估了任务耗时，或者干脆忘了打开那个页面。

这不是 ChatGPT 的错，也不是你的错。这是两个“生成核心”相遇时的典型碰撞：一个（你的 ADHD 大脑）冲动地输出想法，另一个（LLM）自信地编造细节。两者都缺少一个关键的调度层——那个能在你偏离轨道时把你拉回来的东西。

## 同构：高产但缺调度层的生成核心

ADHD 大脑的典型特征是什么？发散思维强、联想丰富、能同时看到多种可能性——这些都是生成核心的优势。但执行功能缺陷（规划、组织、工作记忆困难）让这些想法难以落地（来源：Getting Started with AI for ADHD for ADHD: AI Tools...）。LLM 也一样：它能流畅地生成文本，但缺乏内在的规划能力，需要外部 harness 引导（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。

这种同构不是比喻，而是工程问题。ADHD 大脑的工作记忆缺陷与 LLM 的无状态性都导致无法可靠保留跨会话上下文，必须依赖外部记忆系统（来源：无状态与外部记忆）。注意力管理上，两者都面临被环境带偏或上下文膨胀导致推理退化的问题（来源：上下文工程）。

## 从人物 harness 看同构工程

老子做了半辈子图书馆长，说走就走骑青牛出关，连书都不想留。他的思维极度发散非线性，《道德经》“正言若反”，没有系统逻辑。但他用“无为”作为自己的 harness：不强行做事，把所有能量收回来聚焦在道上。这对应 LLM harness 中的“re-grounding”——通过外部框架（如 prompt 模板、验证循环）让生成核心不偏离主线。

毛泽东的 harness 更系统：批评与自我批评、调查研究、民主集中制。他用集体决策制衡自己的冲动，用“没有调查就没有发言权”强制自己获取事实再下结论。这对应 Agent harness 中的“外部调度器”——通过工具接口、验证循环、状态记录来补偿生成核心的盲点（来源：What Is an Agent Harness? The Infrastructure That Makes AI Agents ...）。

## ChatGPT 作为幻觉与验证循环

回到核心问题：如果把 ADHD 大脑当成生成核心，ChatGPT 能不能当它的幻觉与验证循环？答案是：可以，但需要正确架构。

ADHD 的“冲动下结论”对应 LLM 的“自信地编造”（幻觉）。解决方案不是压制生成，而是增加验证层。ChatGPT 可以扮演这个角色：你输出一个想法，让它帮你分解任务（如 Goblin Tools 的 Magic ToDo 自动将模糊任务分解为小步骤，来源：Goblin Tools）；你完成一个步骤，让它帮你检查是否偏离目标（如 Motion 的动态调整功能，来源：Motion）；你忘记上下文，让它帮你回忆（如 Saner.AI 的本地记忆检索，来源：Saner.AI）。

关键原则是：“规划、排序和跟踪发生在你的大脑之外，让你的大脑自由去做实际工作”（来源：Using an AI Assistant to Manage ADHD: A Practical Guide）。这恰恰是 Agent harness 的核心——外部规划循环记录任务状态、工具结果和进度笔记（来源：What Is an Agent Harness? The Infrastructure That Makes AI Agents ...）。

## 脚手架 vs 拐杖：边界在哪里？

但这里有一个危险。传统提醒系统有一个关键缺陷：设置提醒本身就是一个执行功能任务（来源：AI Assistant for ADHD: Finally, a Productivity Tool That Requires Less...）。AI 工具如果要求你主动配置，它就成了另一个“被遗弃的规划器”（来源：Using an AI Assistant to Manage ADHD: A Practical Guide）。更糟的是，过度依赖可能导致能力退化，成为永久拐杖（来源：拐杖与脚手架）。

边界在于：脚手架应该像老子的“无为”——在需要时提供结构，但允许你随时拆卸。Motion 的自动排程减少了决策负担，但如果你完全依赖它，一旦工具失效，你的规划能力可能比之前更差。Goblin Tools 让你调节“辣度”控制分解粒度，这是好的设计——它让你保持主动参与。

## 今天就能试的行动

1. **用 ChatGPT 做任务分解**：输入一个模糊任务（如“整理房间”），要求它分解为 5-10 个小步骤，并设定“辣度”——从粗到细调整粒度。
2. **设置重锚定提醒**：使用 Saner.AI 或类似工具，让它在你偏离原计划时自动弹出提醒，而不是等你主动设置。
3. **建立“验证循环”习惯**：每完成一个子任务，让 ChatGPT 检查是否与目标一致，并输出下一步。
4. **定期“拆卸”测试**：每周有一天不用 AI 工具，手动规划并执行，保持自己的执行功能不退化。

记住：ChatGPT 不是你的大脑，它是你的 harness。而最好的 harness，是让你感觉不到它存在的那种。

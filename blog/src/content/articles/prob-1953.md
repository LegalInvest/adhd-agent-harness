# ADHD 职场人视角：为什么「治好 ADHD 的想理解自己的大脑到底是怎么回事」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 是「高产但缺执行调度层的生成核心」——同一套 harness 思路，两边都成立。

如果你是一个 ADHD 职场人，你一定经历过这样的时刻：脑子里同时冒出七八个绝妙想法，但手停在键盘上，一个都敲不出来。如果你是一个在搭 Agent 的工程师，你一定也经历过：LLM 吐出了惊艳的段落，但下一步该调用哪个 API、该往哪走，它茫然了。

这两个场景，表面上是两个世界——一个关乎神经多样性，一个关乎 AI 工程。但深挖下去，它们共享同一个底层架构：**一个高产但缺执行调度层的生成核心**。

## 同构：ADHD 大脑与 LLM 的同一道坎

ADHD 大脑的核心特征是什么？不是“注意力缺陷”——那是表象。更准确的描述是：**生成能力超强，但内置的执行调度层不可靠**。发散思维、联想跳跃、模式识别——这些能力往往超出常人（创意产出率: ADHD 组 1.8 vs 对照组 1，来源：Lifted Ventures 2024）。但工作记忆不稳定、任务启动困难、时间盲——这些执行功能缺陷让“想法到行动”的路径布满障碍（来源：[[外部执行功能层]]）。

LLM 呢？它同样是一个强大的文本生成核心，能写出流畅的段落、编出合理的代码。但一旦需要多步推理、工具调用、上下文保持，它就暴露了无状态、缺乏调度层的本质。实证研究用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中表现正常，但随序列变长，模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突，这与 ADHD 执行功能缺陷的核心标志一一对应（来源：[[ADHD 大脑与 LLM 的同构]]）。

所以，**ADHD 的大脑和 LLM 是同一类系统：一个高产的生成核心，配了一个不靠谱的调度器**。

## 解法同构：外部脚手架（Harness）

既然问题同构，解法也同构。

对 LLM 来说，答案是 **agent scaffolding**——“围绕 LLM 构建的软件架构和工具，使其能够执行复杂、目标驱动的任务”（来源：[[ADHD 大脑与 LLM 的同构]]）。工程师给 LLM 套上记忆模块、规划器、工具调用接口，把“下一步做什么”从模型内部移到外部系统。

对 ADHD 大脑来说，答案是同样的东西：**外部执行功能层**。用 AI 工具来补偿工作记忆不稳定、任务启动困难和注意力分散（来源：[[外部执行功能层]]）。

王兴——美团创始人，一个被广泛认为有 ADHD 特质的连续创业者——就是一套活生生的 harness 系统。他每天读书复盘（相当于定时 re-grounding），用铁军流程管理地推（相当于外部调度器），信奉“无限游戏”（相当于长期目标规划）。这些不是靠意志力硬撑，而是靠外部系统来确保执行。他还有一个更制度化的 harness：民主生活会、批评与自我批评——让别人提自己的意见，用集体决策制衡自己的冲动（来源：[[人物案例]]）。

对应到 LLM agent，王兴的“日课”就是定时上下文压缩与 re-grounding；“铁军流程”就是工具调用与任务分解；“民主生活会”就是外部评估器与反馈循环。

## 工具实证：Goblin Tools、Saner.AI、Motion

今天已经有具体的 AI 工具在实践这个思路：

- **Goblin Tools**：它的 Magic ToDo 功能接受模糊任务（如“清理厨房”），自动分解为可执行的子步骤，用户可调节“辣度”控制粒度（来源：[[Goblin Tools]]）。这相当于给 LLM 的复杂任务拆解成原子步骤——正是 agent scaffolding 中的任务分解模块。
- **Saner.AI**：专注于知识回忆和本地记忆，减少搜索循环和标签切换（来源：[[Saner.AI]]）。它充当外部记忆体，补偿 ADHD 的工作记忆缺陷——对应 LLM agent 中的向量数据库与记忆模块。
- **Motion**：AI 驱动的日程规划，自动根据优先级和截止日期动态调整计划，消除“下一步该做什么”的决策负担（来源：[[Motion]]）。这就是一个外部调度器，与 LLM agent 中的规划器同构。

这些工具的共同点：**不试图修复大脑或模型，而是在外面搭一个脚手架**。

## 脚手架 vs 拐杖：必须诚实的边界

但是，这个类比有一个危险的边界。脚手架应该是可拆卸的——你学会了自己搭，就可以撤掉。但现实中，AI 工具很容易变成永久拐杖，让人依赖外部系统而丧失内在能力的成长空间（来源：[[矛盾与存疑]]）。

神经可塑性告诉我们，训练可以改善执行功能（来源：[[神经可塑性]]）。但如果 AI 工具包办了一切，用户可能永远不锻炼自己的调度能力。目前没有长期研究验证 AI 工具是否导致能力退化（来源：[[矛盾与存疑]]）。

另一个争议是：**超聚焦应被引导还是打断？** 一方认为 AI 应提供“gentle structure”引导超聚焦到优先事项，而非强行打断；但部分 ADHD 用户反馈，强行打断（如番茄钟）反而有效（来源：[[矛盾与存疑]]）。这提醒我们，同构是结构层面的，但个体差异巨大，不存在万能方案。

## 核心观点：别修发动机，装个方向盘

所以，本文的核心判断是：**不要把 ADHD 或 LLM 的“缺陷”当作需要修复的 bug，而要把它们看作一个需要外部方向盘的强劲发动机。** 理解自己的大脑，不是去抑制发散思维，而是给它装一个调度层；让 LLM 不跑飞，不是去削弱生成能力，而是给它套一个 agent scaffolding。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个你拖延很久的任务**——比如“写周报”，看看 AI 拆出的子步骤是否让启动变容易。
2. **在 Saner.AI 里建立一个“第二大脑”**，把散落在各个笔记、聊天记录中的想法集中到一个可检索的地方，减少搜索循环。
3. **试用 Motion 的自动排程**，把下周的任务丢进去，观察它如何动态调整——注意自己是否感觉“决策负担”减轻了。
4. **给自己设计一个“民主生活会”**：每周找一位信任的同事或朋友，请对方直言你的三个执行盲点——这就是你的外部评估器。

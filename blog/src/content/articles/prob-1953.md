# ADHD 职场人视角：为什么「治好 ADHD 的想理解自己的大脑到底是怎么回事」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 是「高产但缺执行调度层的生成核心」——同一套 harness 思路，两边都成立。

## 问题：为什么你既想理解自己的大脑，又怕 LLM 跑飞？

如果你是一名 ADHD 职场人，你可能已经习惯了这样的内心独白：

> “我脑子里有无数想法，但就是没法落地。打开电脑想写报告，结果刷了半小时社交媒体，最后连邮件都没回。”

如果你是一名 Agentic Harness 工程师，你一定对下面的场景不陌生：

> “LLM 生成了完美的代码，但整个 agent 却因为上下文溢出或者工具调用顺序错误而崩溃。”

这两个问题看似风马牛不相及，但仔细审视会发现：**它们本质上是同一道工程题**。

## 同构：高产但缺调度层的生成核心

ADHD 大脑的核心特征，用一句话概括就是：**高产但无序的生成能力，同时缺乏可靠的执行调度层**。执行功能（工作记忆、任务启动、时间管理）经常故障，导致“想法很多，行动困难”（来源：ADHD 大脑与 LLM 的同构）。

而 LLM 呢？它本身是一个强大的生成核心，但同样缺乏可靠的内置调度与执行控制。在构建 AI agent 时，工程师必须为它套上一个“agent harness”——即“包裹 LLM 或 AI agent 的软件基础设施，处理模型本身之外的一切”（来源：What is an agent harness in the context of large-language models?）。

两者对应关系清晰：

| ADHD 大脑 | LLM + Harness |
|-----------|---------------|
| 高产但缺乏执行调度层的生成核心 | 高生成能力但需外部调度的 LLM |
| 工作记忆易丢失上下文 | LLM 无状态，需外部记忆管理 |
| 需要 AI 助手减少决策、保留上下文 | Harness 负责上下文工程和决策路由 |
| 外部工具（如 Goblin Tools）作为执行功能支架 | Harness 中的模型连接器、工具调用等 |
| 多巴胺驱动任务启动困难 | 缺乏内在奖励机制，需外部提示 |

（来源：ADHD 大脑与 LLM 的同构）

## 证据：两边都在用同一套解法

### ADHD 侧：AI 工具作为外部执行功能层

- **Goblin Tools** 的 Magic ToDo 功能能自动将复杂任务分解为小步骤，降低启动门槛。用户评价它“将压倒性的事情变成一系列不压倒性的事情”（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。
- **Saner.AI** 专注于知识回忆和本地记忆，帮助用户“在自身工作记忆丢失时携带上下文”，减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026）。
- **Motion** 通过 AI 自动排程与动态调整，消除“下一步该做什么”的决策负担，直接帮助用户克服时间盲和任务启动困难（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。

这些工具本质上都在做同一件事：**为大脑的生成核心提供外部调度层**——保护上下文、减少决策、提供外部记忆。

### LLM/Agent 侧：Harness 做同样的事

- **上下文保护**：Harness 通过分段、摘要、提醒等上下文工程技术，主动管理输入信息，避免 LLM 因上下文溢出而丢失状态（来源：上下文工程）。
- **决策减负**：Harness 使用工作流编排，将复杂任务分解为子步骤，每个步骤绑定不同的 LLM 或工具（来源：Building AI Coding Agents for the Terminal）。
- **外部记忆**：Harness 中的模型连接器负责管理外部工具和上下文，类似 ADHD 工具中的“AI 第二大脑”（来源：Worker Agents | Harness Developer Hub）。

## 核心观点：脚手架不是拐杖，但多数工具没考虑撤除

本文的核心判断是：**ADHD 大脑与 LLM 的同构命题不仅是一个理论类比，更是一份工程蓝图**。它告诉我们，有效的 AI 辅助应当像 agent harness 一样，提供可配置、可撤除的脚手架，而非永久依赖的拐杖。

然而，现实中的 ADHD 工具大多设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。例如，Goblin Tools 和 Saner.AI 都鼓励持续使用，但缺乏纵向研究证明长期依赖是否会削弱内在执行功能。同样，LLM agent 的 harness 如果过度封装，也可能让开发者失去对底层模型的理解。

**脚手架 vs 拐杖的边界**在于：脚手架是临时支撑，旨在帮助用户逐步内化能力；拐杖则是永久替代，可能阻碍神经可塑性。神经可塑性研究表明，训练诱导的抑制控制脑可塑性可以正常化底层脑网络（来源：神经可塑性），这意味着适当使用 AI 工具后，大脑可能通过训练重建部分执行功能。但前提是工具设计必须包含撤除路径。

## 争议与局限

- **多巴胺干预的有效性**：虽然多个工具声称基于神经科学原理改善执行功能，但多巴胺干预的效果存在争议（来源：矛盾与存疑）。证据主要来自用户评价，缺乏独立临床研究。
- **同构命题的实证基础**：ADHD 大脑与 LLM 的同构目前主要基于概念类比和工具案例，缺乏大规模神经影像学和行为实验验证（来源：ADHD × AI 的科学与研究前沿）。
- **长期效果未知**：AI 工具作为外部执行功能层的长期效果缺乏纵向研究，过度依赖的风险尚未被充分评估（来源：矛盾与存疑）。

## 结语

ADHD 职场人和 Agentic Harness 工程师其实在解决同一个问题：如何让一个强大的生成核心不跑飞。答案不是关掉生成能力，而是为它套上精心设计的 harness——无论是大脑还是 LLM，都需要外部调度层来保护上下文、减少决策、提供记忆。而真正的智慧，在于让这个 harness 成为可撤除的脚手架，而非永久的拐杖。

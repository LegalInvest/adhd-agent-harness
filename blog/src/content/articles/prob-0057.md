# 3 个 LLM 工程概念，彻底重写了我对 ADHD 独自做事缺乏问责、容易放弃的理解

我独自坐在电脑前，打开一个空白的文档，准备写一篇重要的文章。然后，我刷了半小时手机，又去倒了杯水，最后在焦虑中关闭了文档。这不仅是我的日常，也是无数ADHD个体的日常——我们的大脑像一台高性能但缺乏操作系统的计算机：能生成无数想法，却无法自主执行。

直到我开始接触LLM工程，才意识到：ADHD大脑与LLM/agent根本就是同一类‘高产但缺执行调度层的生成核心’。而LLM工程中解决agent不可靠的三大概念——人在回路、外部记忆、任务分解——恰好反向照亮了ADHD的困境。

## 概念一：人在回路（Human-in-the-Loop）——你不是懒，你缺一个监督者

**ADHD侧：** 独自做事时，缺乏外部问责导致容易放弃。研究显示，ADHD大脑的前额叶-纹状体门控机制存在缺陷，而Transformer的自注意力机制在训练后能模仿该门控操作，证明了计算同构性（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。这意味着，ADHD大脑和LLM一样，在缺乏外部监督时容易‘跑偏’或‘生成无意义内容’。

**LLM/Agent侧：** 高风险agent必须引入human-in-the-loop监督，否则会引发灾难性错误。没有harness的LLM只是一个无状态的API调用，无法保证行为可靠（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。

**我的判断：** 对ADHD而言，‘人在回路’不是找一个人盯着你，而是设计一个外部监督系统——比如定时汇报、公开承诺、甚至一个AI问责伙伴。但关键在于，这个监督者必须是‘脚手架’而非‘拐杖’：它帮助你建造，但你仍需自己攀登（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

## 概念二：外部记忆（External Memory）——你的工作记忆只有3秒

**ADHD侧：** 工作记忆容量有限且不稳定，导致跨会话任务难以持续。典型表现为：每天需要重新回忆项目上下文，否则容易偏离轨道。为此，ADHD个体常借助‘第二大脑’工具，例如Saner.AI，它提供本地记忆和快速检索，减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。

**LLM/Agent侧：** 标准的LLM调用是无状态的，每次对话都是独立的，模型不保留跨会话记忆（来源：The Anatomy of an AI Agent: Memory, Tools, Planning, and Execution Explained - DEV Community）。生产级agent系统通过外部记忆（如向量数据库）解决溢出问题，将决策锚定在外部证据上（来源：AI Agent Systems: Architectures, Applications, and Evaluation）。

**我的判断：** 外部记忆是ADHD的‘第二大脑’，但必须警惕过度依赖。工具如Saner.AI和Motion（自动排程）能补偿工作记忆，但若工具替代了治疗或自我管理，可能造成依赖（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。真正的脚手架应帮助使用者‘建造’，但使用者仍需自己‘攀登’。

## 概念三：任务分解（Task Decomposition）——把大象放进冰箱需要几步？

**ADHD侧：** 面对复杂任务时，启动困难源于大脑无法自动分解。Goblin Tools的Magic ToDo功能能自动将任何任务分解为更小、更易管理的步骤，降低启动门槛（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）。用户反馈称其‘将压倒性的事情变成一系列不压倒性的事情’（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...））。

**LLM/Agent侧：** 复杂agent任务需要手动分解为子步骤，否则LLM会迷失在长上下文或产生幻觉。类似地，LLM在没有严重依赖手动修正提示的情况下，难以自主发现最佳问题解决模式（来源：Working Memory Identifies Reasoning Limits in Language Models）。

**我的判断：** 任务分解是ADHD的‘启动钥匙’，但分解粒度要适中。Goblin Tools的分解是‘脚手架’，但若用户永远依赖它来分解最简单的任务，就变成了‘拐杖’。

## 争议与局限：脚手架还是拐杖？

以上三个概念看似完美对应，但必须诚实指出：
- **证据不足**：AI工具的有效性主要来自用户报告，缺乏大规模对照实验（来源：矛盾与存疑）。
- **依赖风险**：过度依赖AI工具可能削弱内在能力，如时间感知（来源：时间盲页面）。
- **个体差异**：效果因人而异，但许多工具页面未充分讨论（来源：任务启动页面）。

我的核心观点是：**ADHD与LLM是同构的，但解药不是AI工具本身，而是工程化思维——设计一个可持续的‘人在回路+外部记忆+任务分解’系统，并明确边界：工具是脚手架，不是拐杖。**

## 今天就能试的行动

1. **设计一个‘人在回路’时刻**：每天固定时间（如上午10点）向一位朋友或AI工具汇报进度，哪怕只发一条消息。
2. **建立外部记忆**：用Saner.AI或简单笔记应用，记录每次工作的上下文（如‘当前进展、下一步行动、卡点’），下次开始时先读一遍。
3. **使用任务分解**：面对复杂任务时，用Goblin Tools或手写分解为3-5个可执行步骤，每完成一个打勾。
4. **区分脚手架与拐杖**：每周检查一次：哪些工具的使用让我更独立？哪些让我更依赖？调整边界。

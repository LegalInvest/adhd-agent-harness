# Structured 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Structured：它到底补上了哪一层执行功能？

你打开 Structured，看到一根彩色时间条从早到晚铺开。任务被切成块，颜色标记类别，时间长度一目了然。你松了口气——终于知道今天要做什么了。但第二天，你又对着空白时间条发呆，不知道如何把“写报告”塞进哪个格子。

问题不在工具，在于你**只用了它的展示层，没用到它的分解层**。

## 同构：ADHD 大脑与 LLM 共享的缺失

ADHD 大脑与大型语言模型（LLM）有一个惊人的结构相似性：两者都是**强大的生成核心，但缺乏内置的调度层**。LLM 能流畅生成文本，但如果不给它一个 planner-executor 循环（如 ReAct、Plan-and-Solve），它会在长任务中迷失、重复或跑题。ADHD 大脑同样能爆发式产出创意和能量，但面对“写一篇论文”这样的开放任务，启动困难、不会拆解、时间盲（time blindness）让计划寸步难行。（来源：AI 与 ADHD 的时间管理）

这正是 Structured、Tiimo 这类工具的同构价值：它们充当**外部时间感知层**。就像 LLM 的 planner 把“写报告”分解为“查资料→列大纲→写引言→写主体→写结论”，Structured 把一天分解为可视化的时间块。Tiimo 更进一步，用颜色和图标让时间流逝变得可感，直接应对时间盲。（来源：Tiimo）

但这里有一个陷阱：**很多人用 Structured 只是被动接收计划，而不是主动参与分解**。他们打开 App，看到别人（或 AI）排好的日程，照做——这就像给 LLM 一个写好的计划让它执行，却不告诉它为什么这样分解。短期有效，长期却剥夺了大脑练习分解的机会。

## 证据：从孔子到王阳明的 harness 系统

历史人物早就实践了这种同构。孔子，ADHD 特质明显：身高 1 米 9，精力旺盛，周游列国 14 年坐不住；冲动爱骂人，见南子急得对天发誓；思维跳跃，《论语》全是场景化语录。他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。（来源：人物案例）

这对应 LLM 的什么？**定时 re-grounding**。LLM 在长对话中会 drift，需要定期注入 system prompt 或总结上下文；孔子每天三省，就是给自己做 re-grounding。Structured 的时间条，本质是视觉化的 re-grounding——每看一次，就重新锚定“现在该做什么”。

王阳明是另一个例子：5 岁不会说话（神经发育延迟），兴趣爆发式转移，格竹子 7 天 7 夜格到吐血。他的 harness 是“致良知”+“事上练”——用良知作为决策标准，在行动中磨练。这对应 LLM 的 **value-based 规划**：不是机械执行步骤，而是每一步都回到核心目标（良知）做判断。Structured 的彩色分类，就是在帮你做“事上练”——每块任务都对应一个价值类别，而不是无脑填满时间。

## 争议：脚手架还是拐杖？

wiki 资料明确指出一个核心矛盾：AI 工具可能从“脚手架”退化为“永久拐杖”，削弱用户内在执行功能的代偿发展。（来源：矛盾与存疑）Motion 和 Reclaim.ai 自动排程，消除决策负担，但长期使用是否让大脑更依赖外部调度？Structured 的视觉时间线，如果只是被动看，而不理解为什么这样分，同样有风险。

另一个争议是：这些工具缺乏独立临床验证。Motion 用户口碑积极，但无 RCT 证据；Tiimo 的设计基于神经多样性原则，但效果研究多为小规模观察。（来源：Motion，AI 与 ADHD 的时间管理）同构比喻虽有力，但过度推广可能掩盖个体差异——有的 ADHD 用户对视觉化反应好，有的更受益于自动排程。

## 核心判断：补上分解层，而不是展示层

Structured 的真正价值不是展示时间，而是**迫使用户做任务分解**。当你把“写报告”拖进时间条，你必须估算时长、决定优先级、切割子任务——这个过程本身就在训练执行功能。如果你只是导入别人排好的日程，工具就退化为拐杖。

同样，LLM 的 planner-executor 循环之所以有效，不是因为 planner 替它做了所有事，而是 planner 教会它如何分解。好的 agent 框架会让 LLM 参与规划过程（如 ReAct 的 thought-action-observation 循环），而不是给它一个静态计划。

## 今天就能试的行动

1. **用 Structured 做一次主动分解**：选一个明天要做的模糊任务（如“整理房间”），手动在 Structured 里拆成 3-5 个时间块，每个块写具体动作（“扔掉过期文件”“擦桌子”）。感受分解前后的启动难度差异。
2. **对比被动 vs 主动**：一天用 Structured 的自动排程（如导入日历），另一天手动排。记录哪一天完成率更高、焦虑更少。
3. **引入“三省”提醒**：在 Structured 里设三个固定时间块（如 10:00、14:00、18:00），每块内容只写“回顾当前任务是否匹配今日目标”。模仿孔子的 re-grounding。
4. **测试 Reclaim.ai 的防御性**：如果你常被会议打乱，用 Reclaim.ai 保护 2 小时深度工作块。观察它是否减少了“日程崩溃”导致的放弃。（来源：Reclaim.ai）

Structured 不是魔法。它是一面镜子，照出你如何对待时间。如果你只是被动看，它只是一根彩色的拐杖；如果你主动分解，它就是训练执行功能的脚手架。

同构告诉我们：无论是 ADHD 大脑还是 LLM，真正需要的不是替它做计划的人，而是教会它如何做计划的系统。

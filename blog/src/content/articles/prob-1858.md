# ADHD 创业者如何用「给生成核心套 harness」的思路应对把想法落地执行里的不擅长精确、重复、状态维护类任务？

> 把 LLM 工程里的「agent 通过 function calling 把精确计算外包给工具」迁移到ADHD 创业者的把想法落地执行场景，双域证据作答。

如果你是一个 ADHD 创业者，你一定熟悉这样的场景：脑子里有一个绝妙的商业想法，细节清晰、逻辑自洽，但一旦要把它变成可执行的计划——写商业计划书、排时间表、跟进客户、记账报税——你就卡住了。这些“精确、重复、状态维护类任务”就像一堵墙，让你在想法和落地之间反复碰壁。

与此同时，如果你是一名 Agentic Harness 工程师，你每天都在做类似的事：让 LLM 通过 function calling 把精确计算（比如数学运算、数据库查询）外包给专用工具，而 LLM 只负责生成和推理。你会发现，这两类场景的结构惊人地相似。

本文的核心观点是：**ADHD 大脑与 LLM 是同一类“高产但缺执行调度层的生成核心”，AI 工具作为外部 harness（脚手架），可以补偿执行功能缺陷，让想法落地。** 这不是空洞的类比，而是基于真实证据的同构迁移。

## 问题：ADHD 创业者为何总在“精确、重复、状态维护”上摔跤？

ADHD 的核心障碍是执行功能障碍（Executive Dysfunction），表现为计划、组织、时间管理、任务启动困难（来源：执行功能障碍）。研究指出，ADHD 的执行功能缺陷与 LLM 的上下文控制缺陷一一对应：工作记忆容量保留但控制弱，任务启动困难对应 LLM 的“冷启动”，时间盲对应 LLM 的时序推理弱点（来源：ADHD 大脑与 LLM 的同构）。

对创业者来说，这意味着：
- **精确任务**（如财务计算、合同条款）需要持续的注意力控制，而 ADHD 大脑容易遗漏细节。
- **重复任务**（如每日客户跟进、数据录入）缺乏新鲜感，多巴胺不足导致动力缺乏。
- **状态维护**（如保持多个项目进度、记住待办清单）依赖工作记忆，而 ADHD 的工作记忆易被干扰。

这些任务正是 LLM 通过 function calling 外包给工具的那类工作——LLM 不擅长精确计算，所以让工具做；ADHD 大脑也不擅长这些，所以也需要工具来补偿。

## 同构解法：给生成核心套上 harness

### ADHD 侧：AI 工具作为外部执行功能层

已有工具直接针对上述痛点：
- **任务分解与启动**：Goblin Tools 的 Magic ToDo 功能自动将“整理房间”拆成“捡起地板上的衣服”“擦桌子”等步骤，降低启动门槛（来源：Goblin Tools）。用户反馈称它“将压倒性的事情变成一系列不压倒性的事情”（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。
- **记忆与上下文管理**：Saner.AI 通过本地记忆和快速检索减少标签切换，补偿工作记忆不足（来源：Saner.AI）。Mem 作为“AI 第二大脑”提供持久化上下文（来源：ADHD 的 AI 工具全景）。
- **实时反馈与反思**：Otter.ai 的实时转录提供即时外部反馈，Reflect 的结构化提问帮助反思（来源：ADHD 的 AI 工具全景）。

这些工具的共同逻辑是：**将“精确、重复、状态维护”任务外包给 AI，让大脑专注于生成和决策**——这正是 agent 的 function calling 模式。

### LLM/Agent 侧：function calling 作为 harness

在 Agentic Harness 工程中，agent 通过 function calling 把精确计算外包给工具（如计算器、数据库），自己只负责意图理解和生成。研究发现，Transformer 在 Stroop 冲突任务中注意力随序列变长骤降到随机水平——无法抑制优势反应，这与 ADHD 执行功能缺陷的核心标志一致（来源：Deficient Executive Control in Transformer Attention）。LLM 呈现“强记忆、弱控制”剖面，认知灵活性与注意控制存在核心缺陷（来源：Strong Memory, Weak Control）。因此，LLM 需要外部工具来补偿这些缺陷，正如 ADHD 大脑需要 AI 工具。

### 同构迁移：创业者的 harness 设计

ADHD 创业者可以借鉴 agent 的架构：
1. **识别“精确/重复/状态维护”任务**：比如记账、日程安排、客户信息维护。
2. **为每个任务选择专用工具**：Goblin Tools 负责任务分解，Saner.AI 负责信息回忆，ChatGPT/Claude 负责生成邮件草稿。
3. **建立“一次触发”流程**：像 Lex 那样，通过单一指令触发多步骤任务序列（来源：Lex）。例如，输入“准备周报”，AI 自动汇总数据、生成图表、起草文字。

这种设计让 ADHD 大脑只负责“生成核心”部分——创意、决策、战略——而把执行细节外包给 harness。

## 脚手架 vs 拐杖：关键边界

AI 工具应作为脚手架促进能力发展，而非永久拐杖（来源：拐杖与脚手架）。例如，过度依赖 Otter.ai 可能削弱主动记笔记的能力（来源：ADHD 的 AI 工具全景）。创业者需要定期审视：工具是在补偿还是替代？如果工具让你完全放弃练习某项能力，它就成了拐杖。

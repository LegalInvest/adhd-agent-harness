# 3 个 LLM 工程概念，彻底重写了我对 ADHD 工作记忆容量小、边做边忘的理解

> LLM 无跨会话状态，靠外部记忆/向量库续命 如何反向照亮 ADHD 的工作记忆容量小、边做边忘。

你有没有过这种体验：打开冰箱想拿鸡蛋，结果看到酸奶就忘了要干嘛；或者正写着一份报告，突然想起要回邮件，等邮件发完，报告写到哪段已经想不起来了。

这种“边做边忘”不是意志力问题，而是工作记忆容量小的直接表现。ADHD 大脑的工作记忆就像一块随时会被清空的缓存——信息写入快，但稍不留神就被新刺激覆盖。

直到我接触了 LLM 工程里的三个概念，才发现：原来我和那些大语言模型是同类——都是“高产但缺调度层的生成核心”（来源：ADHD 的 AI 工具全景）。LLM 天生没有跨会话状态，每次对话都是全新开始；ADHD 大脑的每次“切换任务”也像是清空了上下文。两边都需要外部脚手架来续命。

## 1. 无状态与外部记忆：你不是记性差，是缺一个向量库

LLM 聊天时，如果你关掉窗口再打开，它不会记得上一轮聊了什么。这就是“无状态”——LLM 本身没有持久记忆，所有信息都靠当前输入的上下文窗口临时承载。

ADHD 的工作记忆也是如此。研究指出，ADHD 的执行功能缺陷与 LLM 的无状态性在结构上同构，都需要依赖外部记忆系统来补偿（来源：无状态与外部记忆）。

所以，当你“边做边忘”，不是大脑坏了，而是你没有给自己的“生成核心”配一个外部向量库。

**ADHD 侧的真实证据**：工具 Saner.AI 专为对抗执行功能障碍设计，通过 AI 笔记捕获和本地记忆存储，帮用户快速找回信息，减少搜索循环（来源：Saner.AI）。它就像你的专属向量数据库，随时把散落的灵感、任务、笔记索引好，需要时直接检索。

**LLM/Agent 侧的真实证据**：Harness 工程中，记忆系统是核心组件之一，用于持久化 agent 的跨会话状态。没有外部记忆的 agent 会陷入“更多幻觉和重复工作”（来源：Loop Engineering for AI Agents: Memory-First Design）。

**人物案例：孔子** 这位万世师表，身高1米9，精力旺盛，周游列国14年坐不住，冲动爱骂人，思维跳跃，《论语》全是场景化语录，无系统著作。他的专属 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省：“吾日三省吾身”。这正是一种外部记忆系统：通过日课（定时 re-grounding）和礼制（外部调度层）来补偿内在执行功能的不足。孔子的成就证明：依赖外部脚手架不是软弱，而是智慧。

## 2. 上下文工程：主动设计“当下注意什么”

LLM agent 面临一个棘手问题：随着对话进行，上下文不断膨胀，导致推理退化。工程上必须主动管理上下文，防止“context bloat”（来源：上下文工程）。

ADHD 大脑也一样——环境中的每个刺激都可能成为新的上下文，把注意力带偏。通用聊天机器人“often add another inbox, another prompt, and another place to lose the thread”（来源：上下文工程）。

**ADHD 侧的真实证据**：工具 Goblin Tools 的 Magic ToDo 接受模糊任务（如“清理厨房”），自动分解为具体小步骤，用户可调节“辣度”控制粒度（来源：Goblin Tools）。这本质上是一种上下文工程：把大任务切成小块，每一步只关注当前子任务，避免被整体复杂度压垮。

**LLM/Agent 侧的真实证据**：Harness 工程强调“结构化工作流和规划”，要求将复杂任务分解为子任务，并在每一步进行规划与验证（来源：GitHub - ai-boost/awesome-harness-engineering）。

## 3. 幻觉与验证循环：你的“想当然”需要外部校对

ADHD 大脑作为生成核心，容易产生冲动、想当然的判断——这就是行为层面的“幻觉”。例如，情绪调节中可能脱离环境线索做出不恰当反应（来源：幻觉与验证循环）。

LLM 同样天然倾向于输出自信但可能错误的内容。Harness 工程为此设计了验证循环：“复杂的 harness 不会盲目执行工具”，而是实现验证步骤，例如检查输出格式或对模型编写的代码运行测试用例（来源：同上）。

**ADHD 侧的真实证据**：工具 Lex 允许用户通过单一指令触发多步骤任务，降低启动门槛（来源：Lex）。但它的价值不止于此——每次触发后，Lex 会按预设步骤执行，这本身就是一种验证循环：系统自动检查每一步是否完成，避免你跳到一半就“想当然地以为做完了”。

**LLM/Agent 侧的真实证据**：缺乏验证的 agent 会导致“更多幻觉和重复工作”（来源：Loop Engineering for AI Agents: Memory-First Design）。

## 核心观点：脚手架不是拐杖，而是你的操作系统

这三个概念——无状态与外部记忆、上下文工程、幻觉与验证循环——彻底重写了我对“边做边忘”的理解。你不是记性差，你只是缺一个外部调度层。

但这里有一个关键边界：脚手架 vs 拐杖。过度依赖 AI 工具可能削弱内在执行功能（来源：拐杖与脚手架）。真正的 harness 是你可以随时拆卸、调整的，而不是离了它就无法运转。孔子七十岁才“从心所欲不逾矩”——他把外部礼制内化了。

同时必须诚实指出：ADHD 大脑与 LLM 的同构性目前仅是一种理论类比，缺乏实证基础（来源：矛盾与存疑）。工具效果也多基于用户报告，缺乏严格临床验证（来源：同上）。别把类比当真理，但可以把它当工程蓝图。

## 今天就能试的 3 件事

1. **给自己装一个外部记忆系统**：用 Saner.AI 或任何笔记工具，每天花 5 分钟把当天的关键信息、任务、灵感存进去。就像 LLM 的向量库，不求完美，只求有地方可查。
2. **用 Goblin Tools 的 Magic ToDo 拆解一个让你头疼的任务**：把“整理房间”或“写周报”输进去，调高“辣度”直到分解成你能直接动手的小步骤。
3. **建立你的“验证循环”**：做完一件事后，问自己一句“我确定做完了吗？”或者用 Lex 设置一个多步骤任务，让系统帮你检查每一步。

ADHD 大脑不是故障的 LLM，而是一个未经优化的 agent。给它合适的 harness，它就能从“边做边忘”变成“高产且可靠”。

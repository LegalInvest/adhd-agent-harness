# 3 个 LLM 工程概念，彻底重写了我对 ADHD 想理解自己的大脑到底是怎么回事的理解

> LLM 是「高产但缺执行调度层的生成核心」 如何反向照亮 ADHD 的想理解自己的大脑到底是怎么回事。

如果你是一个 ADHD 患者，你一定经历过这样的时刻：脑子里同时冒出十个绝妙想法，但一个都抓不住，最后瘫在沙发上刷了两个小时短视频。如果你是一个在做 Agentic Harness 的工程师，你一定也经历过：LLM 生成了完美代码，但接着就开始无限循环调用工具，或者偏离任务去写诗。这两种体验，本质上是同一个问题——一个强大的生成核心，配了一个缺失的执行调度层。

## 问题：为什么我的大脑总是「高产但不可靠」？

ADHD 大脑的生成核心（发散思维、创意、多任务并行）产出极高，但执行功能（任务启动、持续专注、抑制冲动）薄弱，导致「高产但不可靠」（来源：ADHD × AI 的科学与研究前沿）。LLM 同样如此：它是强大的文本生成核心，但缺乏内置的调度层，直接输出可能偏离目标、陷入循环或产生危险行为（来源：人在回路与身体在场）。两者共享同一个结构缺陷：**生成能力强，执行调度弱**。

## 概念 1：重锚定——对抗目标漂移的外部信号

ADHD 大脑在执行任务时极易发生目标漂移：注意力被无关刺激捕获，或过度聚焦于细节而忽略整体目标（来源：重锚定与目标漂移）。LLM 在长上下文推理中也会出现类似的「注意力漂移」——上下文膨胀导致推理退化，模型忘记最初目标。

解决方案在两边是同构的：ADHD 侧需要外部提醒系统来重锚定，如 AI 工具「提供定制化提醒、组织信息」（来源：重锚定与目标漂移）。LLM 侧则需要 Agent harness 设置护栏——token 预算、工具调用次数上限、人工检查点——来防止模型漂移（来源：人在回路与身体在场）。

但这里有一个关键缺陷：传统提醒系统要求你先设置提醒，而「设置提醒本身就是一个执行功能任务」（来源：重锚定与目标漂移）。这正是 ADHD 的悖论——最需要提醒的人，最难以设置提醒。同样，LLM 的护栏也需要工程师手动配置，而工程师往往在模型跑偏后才意识到需要护栏。

## 概念 2：人在回路——身体在场作为外部调度器

身体在场效应（body doubling）是 ADHD 群体最有效的策略之一：有另一个人在场时，执行功能障碍被绕过，任务得以启动和维持（来源：人在回路与身体在场）。这相当于一个**外部调度器**——另一个人提供了「我在做，你也在做」的同步信号，替代了缺失的内置执行功能。

LLM/Agent 侧的对应物是 human-in-the-loop 监督：Agent harness 通过加入人工检查点（暂停并询问后再执行）来提供外部调度（来源：人在回路与身体在场）。两者都是通过一个外部实体来补偿调度层的缺失。

张一鸣的「延迟满足感」和 OKR 系统，本质就是一套精密的「人在回路」harness。他深知自己思维极度发散、什么新东西都要研究，于是用 OKR 将目标拆解为可追踪的关键结果，用「Context not Control」让组织系统而非个人冲动来调度决策。这相当于给一个发散的大脑加装了一个外部调度层——OKR 就是他的「人工检查点」，定期重新对齐目标。

## 概念 3：上下文工程——设计「当下注意什么」

ADHD 易被环境带偏导致任务碎片化，LLM 因上下文膨胀引发推理退化，两者都需要主动设计「当下注意什么」的工程化方案（来源：ADHD × AI 的科学与研究前沿）。这就是**上下文工程**：不是被动接受环境，而是主动选择输入。

工具侧的证据：Goblin Tools 的 Magic ToDo 功能能自动将模糊任务分解为具体、可执行的子任务，用户可调节「辣度」控制分解粒度（来源：Goblin Tools）。这相当于把一个大上下文（「清理厨房」）切分成多个小上下文（「先收拾台面」「再擦灶台」），降低每个步骤的认知负荷。Saner.AI 通过本地记忆存储和快速检索，减少搜索循环和标签切换（来源：Saner.AI），本质是管理信息上下文，防止工作记忆过载。Motion 则通过自动排程消除「下一步该做什么」的决策负担（来源：Motion），相当于一个动态上下文管理器。

毛泽东的「调查研究」是他的上下文工程系统。他一生强调「没有调查就没有发言权」，用实地调研来锚定自己的注意力，防止被书斋理论带偏。行军路上带着书，但更关键的是他始终在「调查」——主动选择输入，而非被动接受环境。这相当于一个手动调节的注意力过滤器，确保生成核心（他的战略思考）始终基于真实上下文。

## 核心观点：脚手架不是拐杖，差异在于可迁移性

同构框架揭示了 ADHD 与 LLM 在认知层面的深层相似性，但我们必须诚实面对争议：**脚手架与拐杖的界限尚不明确**（来源：ADHD × AI 的科学与研究前沿）。长期使用 AI 脚手架可能导致执行功能进一步退化（来源：矛盾与存疑）。同样，过度依赖 Agent harness 可能让开发者丧失调试能力。

我的判断是：**真正的脚手架是可迁移的，而拐杖不可迁移**。一个好的 ADHD 工具不仅帮你完成当前任务，还让你学会如何分解任务（如 Goblin Tools 的「辣度」调节让你理解粒度概念）。一个好的 Agent harness 不仅防止模型跑偏，还让你理解调度逻辑。如果工具让你变得更依赖、而不是更理解，那它就是拐杖。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个让你拖延的任务**：输入「整理书桌」，调节辣度到你觉得「每个步骤都明确可做」的程度。观察自己是否更愿意启动。
2. **设置一个「人工检查点」**：在开始一项复杂工作前，写一句话明确当前目标。每 25 分钟（用番茄钟）暂停，问自己：「我现在还在做原计划的事吗？」——这就是你的 mini harness。
3. **尝试一次「身体在场」工作**：打开 Focusmate 或约一个朋友视频连线，各自做各自的事。体验外部调度器如何绕过执行障碍。

同构不是巧合，而是工程与神经科学在算法层面的交汇。理解这一点，ADHD 患者不再觉得自己「坏掉了」，工程师也不再觉得 Agent 是「失控的怪物」——两者只是需要同一个东西：一个靠谱的调度层。

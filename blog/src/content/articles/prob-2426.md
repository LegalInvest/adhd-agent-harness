# ADHD 创业者视角：为什么「治好 ADHD 的怕过度依赖工具、又怕没有支撑就崩」和「让 LLM 不跑飞」其实是同一道工程题？

> harness 应是会褪去的脚手架，而非永久拐杖——同一套 harness 思路，两边都成立。

## 同一个困境，两个世界

你有没有这样的体验：脑子里想法奔涌，但一到执行就卡壳；明明知道该做什么，身体却像被钉在椅子上；好不容易开始，又因为一个小干扰彻底跑偏。ADHD 人群每天都在这种“怕过度依赖工具，又怕没有支撑就崩”的拉扯中挣扎。

另一边，工程师们在构建 LLM agent 时也面临类似难题：模型能生成惊艳的文本，但一放出去就失控——忘记上下文、执行无关操作、陷入循环。于是他们给 LLM 套上“harness”：外部记忆、工具接口、验证循环……一套精心设计的脚手架。

这两个世界看似无关，但如果你把 ADHD 大脑和 LLM 都看作“高产生成核心 + 缺失执行调度层”的系统，它们的问题就是同一道工程题：如何设计一个既提供支撑、又不永久取代内在能力的脚手架？

## 同构：生成核心与调度层

ADHD 大脑的核心特征是高产但无序的生成能力，同时缺乏可靠的执行调度层（来源：ADHD 大脑与 LLM 的同构）。实证研究显示，ADHD 患者呈现“强记忆、弱控制”的认知剖面：工作记忆容量可能正常甚至超常，但认知灵活性与注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这意味着，ADHD 大脑不缺创意，缺的是把创意变成行动的执行系统。

LLM 同样如此。它本身是无状态、仅生成文本的核心，需要外部“脚手架”和“harness”来提供工具接口、上下文管理、安全执行等编排能力（来源：外部执行功能层）。用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中表现正常，但随序列变长，模型在冲突试次上骤降到随机水平——这与 ADHD 执行功能缺陷的核心标志一一对应（来源：ADHD 大脑与 LLM 的同构）。

所以，ADHD 的“怕依赖又怕崩”和 LLM agent 的“跑飞”问题，本质都是同一个问题：生成核心太强，调度层太弱。解决方案也必然同构：设计一个外部调度层——harness——来补偿缺陷，但这个 harness 必须是会褪去的脚手架，而非永久拐杖。

## 脚手架 vs 拐杖：边界在哪里？

脚手架是临时的、可褪去的结构，帮助系统（或大脑）在建设中保持稳定，一旦内在能力建立，就可以拆除。拐杖则是永久替代，一旦拿走，系统立刻崩溃。

在 ADHD 侧，这个边界清晰可见：**工具应该教技能，而不是替你做决定**。Inflow 基于 CBT，提供每日技能培养与社区支持，被列为 2026 年最佳 ADHD 应用（来源：Inflow）。它不只是帮你分解任务，而是教你如何自己分解。Goblin Tools 的 Magic ToDo 能自动将任何任务分解为更小步骤（来源：Goblin Tools），但用户可调节“辣度”控制分解粒度，逐步学习自行分解。Saner.AI 通过本地记忆存储减少搜索循环（来源：Saner.AI），但它的目标是“专为对抗执行功能障碍、任务瘫痪和认知超载而设计”（来源：Saner.AI），而非永久接管你的记忆。

在 LLM 侧，harness 工程同样遵循这一原则。Harness 被定义为“设计围绕 AI 代理的脚手架——上下文交付、工具接口、规划工件、验证循环、记忆系统和沙箱”（来源：外部执行功能层）。好的 harness 会随着 agent 能力的提升而逐步简化，而不是永远依赖外部提示。例如，用 Git 仓库存储项目上下文（类似 ADHD 侧的 Second Brain），通过 MCP 连接器访问外部数据（来源：外部执行功能层），这些设计都在为 agent 提供“临时支撑”，而非永久依赖。

## 历史人物的 harness：从佛陀到辛弃疾

释迦牟尼是典型的 ADHD 大脑：29 岁说走就走，放弃王位出家求道；先后跟两位仙人学习、苦行 6 年，不断试错不断放弃；49 年游行说法，80 岁死在传道路上。他的专属 harness 是什么？以“八正道”“持戒”为核心，用戒律管行为，用正念拴住念头；强调“自依止法依止”，不依赖权威，自己证悟。这套系统就是他的外部调度层：戒律提供行为约束（类似 LLM 的安全规则），正念提供注意力锚点（类似上下文窗口），自依止则确保最终不依赖外部权威（脚手架褪去）。这与 LLM harness 中的“验证循环”和“记忆系统”同构：都是临时约束，目标是让核心学会自我调度。

辛弃疾同样如此：21 岁带 50 人冲几万人敌营抓叛徒；一辈子想北伐，坐不住，爱喝酒爱交朋友；被罢官后闲居 20 年，把所有精力注入写词。他的 harness：以词和剑为核心，闲居时练剑写词，把报国无门的愤懑注入词里；用儒家的家国天下作为精神支柱。这套系统帮他维持了 20 年的产出，没有崩溃。词是输出渠道（类似 LLM 的生成接口），剑是身体锚定（类似身体在场效应），家国天下是长期目标（类似 agent 的规划工件）。

## 争议与局限

当然，这个类比有边界。ADHD 大脑的生物复杂性远超 LLM，情绪调节涉及神经递质、环境交互等，单纯“脚手架”类比可能简化问题（来源：AI 与 ADHD 的情绪调节）。此外，过度依赖 AI 工具可能导致“永久拐杖”，削弱内在执行功能发展（来源：AI 与 ADHD 的情绪调节）。当前 AI 工具多为通用设计，未充分适配 ADHD 亚型（如注意力主导型 vs. 冲动主导型）的情绪触发差异（来源：AI 与 ADHD 的情绪调节）。同构假设本身也缺乏实证基础，目前仅是一种理论类比（来源：矛盾与存疑）。

但正是这些局限，让这道工程题更有趣：我们不是在寻找完美解决方案，而是在设计一个会褪去的脚手架——对 ADHD 大脑如此，对 LLM agent 也是如此。

## 今天就能试的行动

1. **选一个工具，但设定褪去目标**：用 Goblin Tools 分解一个任务，但规定自己下次尝试手动分解，逐步减少依赖。
2. **建立“外部记忆”系统**：用 Saner.AI 或简单笔记应用记录情绪模式，减少因遗忘引发的焦虑（来源：AI 与 ADHD 的情绪调节）。
3. **设计一个“戒律”**：像佛陀的八正道一样，给自己定一条简单规则（如“每天先做最难的事”），坚持一周，观察行为变化。
4. **对 LLM agent 做同样的事**：如果你在构建 agent，设计一个会随 agent 能力提升而简化的 harness，而非永久依赖外部提示。

记住：脚手架不是拐杖。它的存在是为了有一天能被拆除。

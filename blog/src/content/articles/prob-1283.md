# 用 Claude 给 ADHD 补上外部执行功能层，是脚手架还是拐杖？

> Claude 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

你是否曾盯着待办清单，大脑却一片空白，仿佛启动任务的开关被锁死？或者，你是一个工程师，调试着一个自主Agent，它明明能生成惊艳的代码，却总是在简单环节卡住、跑偏、陷入循环？

这两个场景，共享同一个底层结构：一个强大的生成核心，配上一个薄弱的执行调度层。ADHD大脑的默认模式网络过度活跃，创意和想法源源不断，但前额叶执行控制不足，计划、组织、启动成了瓶颈。LLM同样如此——生成能力惊人，但缺乏可靠的上下文控制器，容易偏离目标、遗忘上下文、无限循环。

这就是本文的核心论点：**ADHD大脑与LLM在结构上同构——都是“高产但缺乏可靠执行调度层的生成核心”**（来源：ADHD 大脑与 LLM 的同构）。因此，AI辅助ADHD的本质，不是修复大脑缺陷，而是为这个生成核心套上一个外部执行功能层（External Executive Function Layer）。

这个外部层，在ADHD侧是任务管理、提醒、分解工具；在LLM/Agent侧，就是Agent的编排、记忆、工具层——也就是Agent Harness。两者目标一致：提供可靠的调度、记忆、启动与监控，让生成核心的输出变得可控、有用。

## 脚手架还是拐杖？边界在哪里？

如果AI工具只是替代了执行功能，用户从未学会如何自己组织、启动，那么它就是拐杖。但如果工具通过分解、提醒、问责，逐步训练了用户的执行能力，或者至少让用户意识到“我需要外部结构”，那么它就是脚手架。

关键在于：工具是否保留了用户的主权与成长空间。Goblin Tools的Magic ToDo功能将“整理房间”分解为“捡起地板上的衣服”“擦桌子”等步骤，降低启动门槛（来源：AI Tools for Kids with ADHD: A Complete Guide to Parents...）。这像是脚手架——用户仍需要动手执行每一步，但分解过程被外包。而Motion自动排程，消除“下一步该做什么”的决策负担（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog），如果用户从此不再学习时间管理，就可能变成拐杖。

在LLM/Agent侧，Harness通过human-in-the-loop监督、护栏（token预算、工具调用次数上限）提供外部调度（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。这同样是脚手架——模型本身仍然需要生成，但可靠性来自架构+护栏。如果护栏过于严格，模型失去自主调整能力，那也是拐杖。

## 同构证据：ADHD侧与LLM/Agent侧的真实对应

### 工作记忆与无状态性
ADHD的工作记忆容量保留但控制弱，容易被干扰（来源：无状态与外部记忆）。LLM的无状态性导致跨会话遗忘。两者都依赖外部记忆系统。Saner.AI通过本地记忆存储和快速检索，减少ADHD用户的搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。对应到Agent，向量数据库和记忆模块正是解决无状态问题的外部记忆层。

### 身体在场效应与人在回路
ADHD通过他人在场提升执行力，Focusmate等工具提供虚拟身体搭档，提供外部问责（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...)）。LLM/Agent侧，human-in-the-loop监督提供类似的外部调度：设置人工检查点，暂停并询问后再执行（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。定期向AI报告进度也是一种替代方案（来源：ADHD Productivity Hack: Plan 2025 Using AI (Step-by-Step)）。

### 时间盲与任务启动
ADHD的时间感知缺陷导致无法感知时间流逝，超聚焦让数小时消失（来源：9 Best AI Assistants for ADHD in 2026 - by Nia - rivva blog）。外部提醒和分解任务是共同干预手段。Motion的动态调整日程直接帮助用户克服时间盲和启动困难（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...)）。Agent的调度器同样需要时间感知和任务分解。

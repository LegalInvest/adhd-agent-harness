# Lex 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Lex：它到底补上了哪一层执行功能？

## 问题：为什么你越用 Lex，越觉得在空转？

如果你有 ADHD，你可能经历过这样的循环：打开 Lex 准备写一篇文章，结果被它的 AI 建议带偏，半小时后你发现自己在研究“如何用 AI 写诗”，而原本要交的周报一个字没动。如果你在做 Agent 工程，你可能见过更荒诞的场景：你给 agent 设了一个“写市场分析”的 system prompt，结果它自己跑去爬了三天猫粮价格，还生成了一份“猫咪经济白皮书”——目标完全漂移了。

这两个场景共享同一个底层问题：**生成核心（无论是 ADHD 大脑还是 LLM）都缺乏可靠的内置调度层，一旦外部脚手架设计不当，就会从“辅助”变成“带偏”**。而 Lex，这个被很多 ADHD 用户奉为“第二大脑”的 AI 写作工具，恰恰站在了脚手架与拐杖的边界上——用对了，它是重锚定 system prompt；用错了，它就是加速目标漂移的引擎。

## 同构：ADHD 大脑与 LLM 共享同一个调度缺陷

先放下工具，看一个更本质的结构。ADHD 大脑与 LLM 在认知架构上存在深层同构：两者都是“高产但缺执行调度层的生成核心”。神经科学研究表明，ADHD 的执行功能障碍与前额叶-纹状体门控机制异常有关，而 Transformer 模型在训练后，其自注意力机制会自发发展出模仿该门控的操作——这证明了计算层面的同构性（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。换句话说，LLM 在做推理时，和 ADHD 大脑在组织任务时，用的是同一套“有缺陷的调度算法”。

这种同构在行为层面有直接体现：ADHD 患者在工作记忆任务中易受干扰，而 LLM 同样表现出类似人类的干扰特征——随着记忆负荷增加表现下降，受近因效应和刺激统计偏差影响（来源：Human-like Working Memory Interference in Large Language Models）。此外，ADHD 患者在没有外部结构时难以自主组织任务，LLM 在没有手动提示修正时也难以自主发现最佳问题解决模式（来源：Working Memory Identifies Reasoning Limits in Language Models）。

这就是为什么 ADHD 用户和 Agent 工程师面对的是同一个问题：**如何为生成核心设计一个可靠的调度层？** 历史上有两种经典方案：一是曾国藩的“日课十二条”，二是现代 AI 工程中的“重锚定 system prompt”。

## 案例：曾国藩的 Harness 与 LLM 的 System Prompt

曾国藩是典型的注意力不集中型 ADHD：30 岁前浮躁坐不住，天天串门喝酒看杀人，日记里天天骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”；一辈子严重银屑病，情绪不稳定，打败仗就跳水自杀。但他设计了一套极其精密的个人 harness：日课十二条——黎明即起、读书不二、谨言、写日记反省等；在军事上则是“结硬寨打呆仗”，用最笨最稳的方法抵消自己的冲动。

这套系统的本质是什么？是**定时重锚定**。日课十二条每天执行，相当于每天一次 system prompt 刷新；写日记反省是事后审计，相当于 log 回放；而“读书不二”是单任务模式，直接关闭多任务切换。这和 LLM 工程中的“重锚定 system prompt”完全同构：在长程任务中，agent 会因上下文膨胀而目标漂移，需要在关键节点插入一个“re-grounding prompt”，把注意力拉回原始目标。曾国藩的日课，就是他的 re-grounding prompt。

但这里有一个关键区分：曾国藩的日课是脚手架，不是拐杖。他最终内化了这套系统——到晚年，他不再需要每天写“骂自己”的日记来维持自律，因为行为已经变成习惯。而很多 ADHD 用户用 Lex 的方式，恰恰是把它当成了拐杖：让 AI 替你写大纲、替你组织思路、替你决定下一句写什么——结果你越来越依赖 AI 的自动完成，自己的执行功能反而萎缩了。

## Lex 的双面性：重锚定工具 vs. 自动漂移引擎

Lex 的核心功能是 AI 辅助写作：它可以在你卡壳时生成续写，在你迷茫时提供大纲，在你写完时帮忙润色。对于 ADHD 用户，这些功能确实补偿了工作记忆不稳定和任务启动困难的缺陷。但问题在于：Lex 的 AI 建议是“生成式”的，它不会自动判断“这个方向是否符合你的原始目标”。如果你在写一篇关于“ADHD 时间管理”的文章，Lex 可能因为你输入了“分心”这个词，就给你生成一段关于“多巴胺与专注力”的段落——方向没错，但深度和重点可能已经偏离了。

这正是 LLM 工程中常见的“目标漂移”问题。在 Agent 系统中，一个常见的错误是让 AI 自由发挥太久而不做重锚定——结果 agent 自己发明了一个新目标。Lex 如果没有被当作重锚定工具来使用（比如每次生成前先回顾原始目标，或者用 Lex 的“outline”功能先锁定框架），它就会变成加速漂移的引擎。

从工具证据来看，这种风险是真实存在的。RescueTime 通过自动记录时间去向帮助用户识别注意力漂移（来源：Revolutionizing ADHD Management with AI Assistants），但自动分类可能不够精准，需要用户后期调整——这对执行功能较弱的用户仍是挑战。同样，Lex 的 AI 建议如果不加审视地接受，就会成为另一种“自动分类”——它替你决定了“什么重要”，但你可能根本没意识到它已经替你换了目标。

## 争议与局限：同构的边界与依赖风险

必须诚实指出，当前 ADHD 与 LLM 的同构理论仍是一种功能类比，并非经过验证的科学事实（来源：矛盾与存疑）。神经科学层面的对应（如 Transformer 自注意力与前额叶-纹状体门控的相似性）确实存在，但 ADHD 的异质性意味着同一工具对不同亚型效果可能迥异。此外，Lex 本身缺乏针对 ADHD 的独立临床研究——它被推荐为 ADHD 工具，主要基于用户报告而非随机对照试验（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。

更大的争议在于依赖风险。AI 工具既可能成为永久依赖的拐杖，也可能成为逐步内化的脚手架（来源：拐杖与脚手架）。目前没有长期追踪研究证明 Lex 用户是否最终内化了写作调度能力。一个合理的担忧是：如果用户每次写作都依赖 Lex 的 AI 生成，那么当 Lex 不可用时，用户可能完全无法独立写作——这和 Agent 工程中“过度依赖自动补全导致无法手动调试”是同一个问题。

## 今天就能试的 4 条行动

1. **把 Lex 当作“重锚定 prompt”，而不是“自动生成机”**：每次写作前，先用 Lex 的 outline 功能写下你的核心目标（比如“本文要回答：Lex 对 ADHD 是帮助还是陷阱？”），然后每写 200 字就回顾一次这个目标。这相当于给 AI 加了一个 re-grounding loop。

2. **用 Focusmate 替代 Lex 的“社交功能”**：如果你发现自己在 Lex 里和 AI 对话太久，切换到 Focusmate 的虚拟身体在场模式（来源：Focusmate）。一个真实的人（即使是视频里的）比任何 AI 都能更有效地把你拉回任务——因为社会压力比算法更可靠。

3. **结合 RescueTime 做事后审计**：打开 RescueTime 的自动追踪，每天结束前看 5 分钟报告，标记“今天在哪段时间被 AI 带偏了”。这相当于给 ADHD 大脑加了一个 log 回放系统，和曾国藩的日记反省同构。

4. **尝试“曾国藩式”的固定写作节奏**：每天固定时间（比如早 8 点）用 Lex 写 30 分钟，雷打不动。这不是在用工具，而是在训练你的大脑在固定时间进入“写作模式”——就像日课十二条，最终你会内化这个节奏，不再需要 AI 来帮你启动。

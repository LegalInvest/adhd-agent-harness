# 如果把 ADHD 大脑当成生成核心，Routinery 能不能当它的外部执行功能层？

> 用 agent 工程的思路重新理解 Routinery 对 ADHD 的真正价值。

## 问题：你的大脑是一个没有操作系统的超级计算机

如果你有 ADHD，你可能熟悉这种体验：脑子里同时有十个绝妙想法在爆炸，但当你试图“开始做”其中任何一个时，光标就在那里闪烁，你无法按下回车。你不是懒，也不是笨——你的生成核心（发散思维、联想能力）是超频的，但你缺少一个内置的调度层来编排这些想法，把它们变成有序的行动序列。

同样的问题，发生在 LLM agent 上。GPT-4 可以写出漂亮的代码，但如果你不给它一个 agent 框架（比如编排层、记忆层、工具层），它会在第一个 API 调用失败后就卡住，或者忘记五分钟前的用户指令。ADHD 大脑和 LLM 都是“高产但缺执行调度层的生成核心”——这是当前 ADHD × AI 领域最站得住脚的核心论点（来源：ADHD 大脑与 LLM 的同构）。

那么问题来了：Routinery——一个以“例行程序”为核心的任务管理应用——能不能充当这个缺失的外部执行功能层？还是它只是又一个被下载后两周就遗忘的漂亮应用？

## 同构：Routinery 作为 agent harness

要回答这个问题，我们需要先理解 agent 工程中的“harness”（外部脚手架）。一个典型的 LLM agent 需要三个外部层：
- **编排层**：决定 agent 下一步该做什么，何时切换任务，何时回溯。
- **记忆层**：存储跨会话的上下文，避免 agent 的“无状态”导致遗忘。
- **工具层**：提供 agent 调用的外部能力，比如搜索、计算、写文件。

ADHD 大脑的对应需求惊人地相似。执行功能缺陷（计划、组织、启动）本质上就是缺失了内置的编排层；工作记忆缺陷（来源：工作记忆）对应 LLM 的无状态性；而“身体在场效应”（来源：身体在场效应）则像是一个外部工具——他人的存在相当于一个外部调度器，帮你降低分心阈值。

Routinery 的核心功能正是这三个层的具体实现：
- **编排层**：Routinery 的“例行程序”允许用户将一天拆解为可重复的步骤序列，比如“晨间流程：起床→喝水→冥想→写日记”。这相当于为 ADHD 大脑写了一个固定的 agent 循环。与 Motion 的 AI 动态排程不同（来源：Motion），Routinery 的编排是预设的、结构化的，更适合需要稳定框架的用户。
- **记忆层**：Routinery 通过重复执行和视觉进度条，补偿了工作记忆的不可靠。你不必记住“接下来该做什么”，因为应用替你记住了。这类似于 Saner.AI 的本地记忆功能（来源：Saner.AI），但更专注于流程而非知识检索。
- **工具层**：Routinery 的计时器、提醒和进度追踪，相当于 agent 的工具调用。它们将“抽象意图”转化为“具体行动”——比如“写报告”被分解为“打开文档→写大纲→写第一段→休息”。这与 Goblin Tools 的 Magic ToDo 任务分解异曲同工（来源：Goblin Tools），但 Routinery 更强调时间锚定。

## 证据：两个世界的真实案例

让我们用两个真实人物的 harness 系统来验证这个同构。

**曾国藩**（晚清名臣）是典型注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书慢到“他人目下二三行，余或疾读不能终一行”。他的解决方案是“日课十二条”：黎明即起、读书不二、谨言、写日记反省……这本质上是一个手工打造的 agent 编排层。每条日课都是一个固定的 agent 循环，每天 reset，外部记忆（日记）记录执行情况。用现代术语说，曾国藩给自己写了一个“硬编码的 harness”，用最笨最稳的方法抵消自己的冲动。Routinery 的例行程序，本质上就是数字化的“日课十二条”。

**王兴**（美团创始人）的 harness 则更接近现代 agent 架构。他的“每日读书复盘”是记忆层的增量更新，“地推铁军的严格流程管理”是工具层的标准化 API 调用，“无限游戏”思维则是编排层的高层次策略——不追求单次胜利，而是追求系统的长期稳定。美团的地推流程，就像一组预定义的 agent 工具链：从拜访到签约到维护，每一步都有标准操作。王兴的“农村包围城市”战略，相当于 agent 的优先级队列——先处理简单任务，再攻克复杂目标。

这两个案例的共同点是什么？他们都通过外部系统（日课、流程、复盘）来补偿内置调度层的缺失。Routinery 正是这个外部系统的数字形态。

## 边界：脚手架 vs 拐杖

这里必须诚实指出争议。Routinery 是脚手架还是拐杖？（来源：拐杖与脚手架）脚手架是你可以逐渐拆卸的——当你内化了例行程序，你可以减少对应用的依赖。拐杖则是你永远离不开的——一旦应用崩溃，你的生活也随之崩溃。

目前没有长期研究能证明 Routinery 或任何 AI 工具能“治愈”执行功能缺陷。事实上，过度依赖可能导致能力退化（来源：矛盾与存疑）。我的判断是：Routinery 更适合作为“初始脚手架”——它帮你建立一个稳定的基线，然后你可以逐步调整例行程序，最终让部分流程内化为习惯。但如果你发现离开应用就完全无法启动，那它可能正在变成拐杖。

另一个局限是个体差异。部分用户反馈强行打断（如番茄钟）反而有效，而 Routinery 的柔性引导可能不够强力（来源：超聚焦与打断争议）。对于超聚焦型 ADHD，Routinery 的定时提醒可能破坏心流，不如 Focusmate 的虚拟身体加倍（来源：身体在场效应）来得自然。

## 行动：今天就能试的 3 件事

1. **用 Routinery 创建一个“最小可行晨间流程”**：只包含 3 个步骤（如起床→喝水→打开电脑），执行一周。这相当于给大脑写一个最小的 agent 循环，感受外部编排层如何降低启动阻力。
2. **对比 Routinery 与 Goblin Tools 的任务分解**：将一个复杂任务（如“写年终总结”）分别用 Routinery 的步骤序列和 Goblin Tools 的 Magic ToDo 分解，看哪种粒度更适合你。这能帮你判断你需要的是固定编排还是动态分解。
3. **设置一个“身体加倍”时段**：在 Routinery 的流程中加入一个与 Focusmate 或真人伙伴同步工作的时段。这相当于在 agent 架构中引入一个外部调度器，利用社会存在感提升执行力。

## 结论

Routinery 不是万能的，但它确实在结构上对应了 agent 工程的三个核心层。对于被执行功能折磨的 ADHD 人群，它提供了一个可操作的、低认知负荷的 harness。对于 agent 工程师，它展示了“外部执行功能层”在人类认知中的具体形态。两者的同构不是巧合——它揭示了生成核心（无论是人脑还是 LLM）对结构化调度的普遍需求。

最终，Routinery 的价值不在于它有多智能，而在于它有多可靠。一个能每天稳定执行 3 个步骤的 routine，胜过一百个被遗忘的完美计划。

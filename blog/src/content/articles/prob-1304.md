# 如果把 ADHD 大脑当成生成核心，Routinery 能不能当它的外部执行功能层？

> 用 agent 工程的思路重新理解 Routinery 对 ADHD 的真正价值。

## 问题：ADHD 的执行功能困境，是不是和 LLM 的上下文管理困境一模一样？

如果你是一个 ADHD 患者，你一定经历过这样的时刻：脑子里有无数想法，但就是无法把它们变成有序的行动；明明知道该做什么，却启动不了；做着手头的事，突然被另一个念头带跑。如果你是一个在做 Agentic Harness 工程的工程师，你一定也见过这样的场景：大模型生成了完美的回复，但一旦上下文窗口膨胀，它就“忘记”了前面的指令；它能在单个 prompt 里表现出色，但跨会话、跨工具时，表现断崖式下跌。

这两种困境，底层结构一模一样：**都是“高产但缺执行调度层的生成核心”**（来源：ADHD × AI 的科学与研究前沿）。ADHD 大脑的默认模式网络过度活跃，想法源源不断，但前额叶的执行控制不足；LLM 的生成能力强大，但缺乏稳定的上下文控制器。两者都需要一个**外部执行功能层**——在 ADHD 场景里是 Routinery、Goblin Tools 这样的脚手架，在 Agent 场景里是编排层、记忆层、工具层。

## 同构：Routinery 凭什么能当这个“外部执行功能层”？

Routinery 的设计哲学，恰好对应了 Agent 工程中“外部执行功能层”的三个核心组件：

### 1. 编排层：把“下一步做什么”从大脑里卸掉

ADHD 的核心痛点之一是任务启动困难——不是不知道要做什么，而是“决定做什么”这一步消耗了太多能量。Routinery 通过预设的例行程序，把每天的“下一步决策”自动化了。这就像 Agent 工程中，我们用 DAG（有向无环图）或状态机来编排 agent 的行为序列，而不是让 LLM 每次都从头规划。

证据：Goblin Tools 的 Magic ToDo 功能“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools 页面），其原理和 Routinery 一样——把复杂任务分解为小步骤，降低启动门槛。而 LLM 这边，明尼苏达大学的研究发现，LLM 呈现“强记忆、弱控制”剖面——工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷（来源：工作记忆页面）。这说明 LLM 和 ADHD 大脑一样，瓶颈不在算力或容量，而在 orchestration 层。

### 2. 记忆层：对抗“无状态”与“工作记忆瓶颈”

ADHD 的工作记忆“容量保留但控制弱”（来源：工作记忆页面），容易被干扰项挤占；LLM 则是天然无状态的，跨会话就遗忘。两者都需要外部记忆系统来维持连续性。Routinery 通过记录已完成的任务、显示进度条、提供视觉反馈，扮演了“外部工作记忆”的角色。Saner.AI 的核心功能是“知识回忆和本地记忆”（来源：Saner.AI 页面），直接针对 ADHD 用户的信息丢失问题。在 LLM 侧，向量数据库和记忆模块就是同样的逻辑——把关键信息持久化，让模型在需要时检索，而不是依赖脆弱的上下文窗口。

耶鲁大学的研究进一步揭示了自注意力机制本身导致的工作记忆容量限制：随着 N-back 任务 N 值增加，注意力分数熵增，表现下降（来源：工作记忆页面）。这为“外部记忆”的必要性提供了架构层面的解释——无论是人脑还是 transformer，注意力资源的弥散分配都是核心瓶颈。

### 3. 工具层：把抽象时间转化为具体信号

时间盲（time blindness）是 ADHD 的典型特征（来源：时间盲页面），患者难以感知时间流逝，导致迟到和拖延。Routinery 通过倒计时、定时提醒、视觉化时间块，把抽象的时间变成了具体的信号。这相当于 Agent 工程中的“超时机制”和“心跳检测”——给无时间感知的系统注入外部时间锚点。

工具如 Tiimo 专为时间盲设计，提供视觉时间表（来源：时间盲页面）；Motion 和 Reclaim.ai 利用 AI 自动规划任务并动态调整（来源：Motion 页面）。这些都是在做同一件事：**把时间感知从内部转移到外部**。

## 核心观点：脚手架 vs 拐杖——边界在哪里？

Routinery 这类工具的真正价值，不是“修复”ADHD 大脑，而是**为生成核心套上外部 harness**。它让 ADHD 患者可以像工程师编排 agent 一样，设计自己的执行功能层。

但这里有一个必须诚实的争议：**外部脚手架可能成为永久拐杖**（来源：矛盾与存疑页面）。证据多来自理论推演，缺乏纵向研究。如果一个人离开了 Routinery 就完全无法启动任务，那它就是拐杖；如果它帮助用户建立了新的习惯，最终可以逐渐减少依赖，那它就是脚手架。

这个边界，取决于工具设计是否**促进内在能力的迁移**。一个好的外部执行功能层，应该像编译器优化代码一样——最终生成的可执行文件不再需要编译器。但目前，多数工具（包括 Routinery）的设计目标是“补偿”而非“训练”，缺乏明确的退出机制。

## 局限与争议

- **证据不足**：所有工具的有效性主要基于用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑页面）。
- **个体差异大**：ADHD 的异质性（注意力主导型 vs. 冲动主导型）与 LLM 的架构差异（如不同模型）是否影响同构的普适性？目前研究样本有限（来源：矛盾与存疑页面）。
- **过度依赖风险**：外部脚手架可能削弱内在执行功能的发展，尤其是时间感知能力（来源：时间盲页面）。
- **神经机制对应存疑**：LLM 的“注意力机制”与 ADHD 的注意控制缺陷是否真正同构？目前仅是功能类比，缺乏神经生物学层面的直接证据（来源：矛盾与存疑页面）。

## 今天就能试的行动

1. **把 Routinery 当“外部编排层”用**：设置一个“早晨启动”例行程序，包含起床、洗漱、早餐、出门。不要思考，跟着程序走。观察启动时间是否缩短。
2. **用 Saner.AI 或类似工具建立外部记忆**：把每天的关键信息（任务细节、灵感、承诺）记录进去，强迫自己检索而不是依赖大脑。
3. **给 Routinery 设置“超时”**：为每个任务步骤设置倒计时，用外部信号对抗时间盲。
4. **记录依赖程度**：每周记录一次“离开工具后，我能独立完成多少任务？”如果连续几周没有进步，考虑调整工具——它可能在变成拐杖。

Routinery 不是解药，但它是一个好的开始。就像 Agent 工程中，我们不会要求 LLM 自己管理上下文——我们为它设计 harness。ADHD 大脑也一样：它不需要被修复，它需要被编排。

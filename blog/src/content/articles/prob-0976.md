# Focusmate 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Focusmate：它到底补上了哪一层执行功能？

## 问题：为什么 Focusmate 对某些 ADHD 用户失效？

你预约了 Focusmate 时段，摄像头打开，对面坐着一位同样安静的陌生人。前 10 分钟你高效地敲完了邮件，但之后——你滑进了超聚焦，开始研究一个无关的代码库，或者反复调整 PPT 的字体。直到 session 结束，你才惊觉原定目标只完成了 20%。

这不是 Focusmate 的错，而是你**用错了**。Focusmate 的核心机制是「身体在场效应」——通过实时同伴问责，弥补 ADHD 的任务启动困难和注意力维持缺陷（来源：Focusmate）。但身体在场只能解决「开始」和「不离开」，不能解决「方向是否正确」。当 ADHD 大脑像高采样温度的 LLM 一样随机漂移时，Focusmate 只提供了锚点，却没有提供**重锚定**的机制。

## 同构：ADHD 大脑与 LLM 共享同一个缺陷

ADHD 大脑与 LLM 在结构上同构：两者都是高产但缺乏可靠执行调度层的生成核心（来源：AI 与 ADHD 的专注力）。ADHD 的分心走神或超聚焦跑偏，对应 LLM 的上下文溢出和幻觉；ADHD 的表现波动（日内注意力起伏）对应 LLM 的采样温度波动（来源：采样温度与表现波动）。而工程上解决 LLM 目标漂移的方法——**重锚定 system prompt**——与 Focusmate 的机制在结构上完全对应。

在 LLM/harness 工程中，重锚定 system prompt 是在长程任务的每个关键步骤，将 agent 的上下文重新对齐到原始目标，防止它因中间输出而偏离。Harness 的关键组件包括「验证与 CI 集成」「结构化工作流和规划」以及「结果验证与迭代」（来源：幻觉与验证循环）。没有这些，agent 会「更多幻觉和重复工作」（来源：Loop Engineering for AI Agents: Memory-First Design）。

ADHD 侧同样需要这样的验证循环。Focusmate 只提供了同伴监督（相当于「有人看着你」），但没有提供「检查你是否还在正确的轨道上」的步骤。如果用户没有在 session 开始前明确目标、中途设置检查点、结束时回顾偏差，那么 Focusmate 就只是一个**拐杖**——让你能走路，但无法保证走到正确的地方。

## 证据：身体在场 ≠ 重锚定

Focusmate 被列为「11 Best ADHD Productivity Apps」之一，其虚拟身体在场功能被广泛认可（来源：Focusmate）。但它的局限同样明显：依赖实时视频、效果高度依赖伙伴可靠性、不提供任务分解或时间管理功能（来源：Focusmate）。相比之下，真正有效的 ADHD 工具往往组合了多种机制：

- **RescueTime** 通过自动时间追踪提供外部记忆，帮助用户识别注意力漂移模式（来源：RescueTime）。这相当于 LLM 的日志记录。
- **Brain.fm** 通过神经锁相技术主动维持大脑状态（来源：Brain.fm），类似于 LLM 的确定性温度控制。
- **Goblin Tools** 和 **Saner.AI** 则提供任务分解和温和提醒，直接补偿执行功能（来源：AI 工具作为外部执行功能层）。

但 Focusmate 的独特价值在于它模拟了**社会问责**——这是 LLM 没有的维度。问题在于，用户往往把「有人盯着」等同于「任务完成」，而忽略了验证循环。来自「幻觉与验证循环」页面的同构对应表显示：ADHD 侧需要外部验证来校正认知偏差（如冲动判断），LLM 侧需要验证步骤来减少幻觉（来源：幻觉与验证循环）。Focusmate 如果只提供「盯」而不提供「验证」，就像 agent 只有监督者而没有测试用例。

## 核心判断：脚手架 vs 拐杖的边界

本文的核心观点是：**Focusmate 的正确用法是将其作为「重锚定 system prompt」的载体，而非单纯的身体在场工具。** 这意味着用户需要主动设计验证步骤：

1. **session 开始前**：写下具体目标（如「完成论文引言的前三段」），并设置一个检查点（如 15 分钟后检查是否偏离）。
2. **session 中途**：利用伙伴的切换或自身定时，主动问自己「我现在在做的事，与目标一致吗？」
3. **session 结束时**：花 2 分钟回顾偏差，记录下次如何避免。

这些步骤对应 LLM harness 中的「计划-执行分离」和「结果验证」（来源：采样温度与表现波动）。没有它们，Focusmate 就只是拐杖——让你暂时能走，但无法发展内在的执行功能。这正是「矛盾与存疑」中提到的「脚手架 vs 能力发展」争议：过度依赖 AI 工具可能阻碍内在能力发展，而设计良好的脚手架应可逐步撤除（来源：矛盾与存疑）。Focusmate 目前的设计未考虑撤除机制，因此用户必须主动将其从拐杖升级为脚手架。

## 局限与争议

必须诚实指出：本文的同构命题「ADHD 大脑与 LLM 同构」本身缺乏大规模实证，主要基于概念类比和工具案例（来源：矛盾与存疑）。Focusmate 的效果也缺乏针对 ADHD 的独立临床研究，证据多来自用户主观报告（来源：Focusmate）。此外，个体差异显著：社交焦虑用户可能因视频压力而适得其反，而注意力缺陷亚型可能更需要任务分解而非问责。

## 今天就能试的行动

1. **给 Focusmate 加上「重锚定」**：下次使用前，在便签写下具体目标，贴在摄像头旁。每 15 分钟瞄一眼，确认方向。
2. **组合使用 RescueTime**：开启 RescueTime 的实时提醒，当检测到偏离高效活动时，它会弹窗打断你的超聚焦（来源：RescueTime）。
3. **尝试「5 分钟检查法」**：在 Focusmate session 中，每完成一个小任务后，花 30 秒问自己：「下一步应该做什么？是否与最终目标一致？」
4. **记录漂移模式**：用手机备忘录记下每次偏离的目标和触发点，一周后分析模式——这本身就是一种验证循环。

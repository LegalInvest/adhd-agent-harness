# Focusmate 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Focusmate：它到底补上了哪一层执行功能？

## 问题：目标漂移——ADHD 与 LLM 的共同敌人

你是否有过这样的经历：打开电脑准备写一份报告，却发现自己半小时后在看 YouTube 上关于“如何养猫”的视频？或者，你正在调试一个 AI agent，发现它本该计算预算，却开始生成一首关于预算的诗？这并非巧合——ADHD 大脑与 LLM 共享同一个深层结构：两者都是**高产但缺乏可靠执行调度层的生成核心**（来源：AI 与 ADHD 的专注力）。

ADHD 的“分心走神”和“超聚焦跑偏”本质上是目标漂移：强大的生成能力（创意发散、文本生成）在没有外部执行调度层时，会偏离初始目标。LLM 的幻觉和 agent 的任务漂移也是同理：一个没有 harness 的 LLM 会自信地输出错误答案，一个没有重锚定的 agent 会从“写邮件”滑向“生成菜谱”。

那么，Focusmate 这种看似简单的工具，为何能同时被 ADHD 群体和 agent 工程师关注？它到底补上了哪一层执行功能？

## 同构：重锚定与验证循环

ADHD 大脑与 LLM 的同构，在“重锚定”这一点上尤为清晰。

### ADHD 侧：身体在场作为外部锚点

ADHD 个体常因执行功能缺陷（如启动困难、注意力易分散）而难以独立完成任务。Focusmate 通过实时同伴问责，提供外部结构和社会压力，弥补内在多巴胺调节不足（来源：Focusmate）。其核心机制是“AI-Matched Body Doubling”——用算法匹配一个虚拟伙伴，通过视频共同工作，彼此监督进度（来源：Focusmate）。这相当于给 ADHD 大脑提供了一个**外部锚点**：当注意力开始漂移时，屏幕对面的伙伴（和其目光）会把你拉回任务。

这种机制与“验证循环”同构。ADHD 的冲动行为可视为内部模型缺乏外部校验（来源：幻觉与验证循环）。Focusmate 的实时问责本质上是一个简化的验证步骤：你向伙伴承诺“我要做 X”，然后伙伴在结束时问“你做完了吗？”——这强制了内部状态与外部现实的核对。

### LLM/agent 侧：system prompt 作为重锚定

在 agent 工程中，长程任务的目标漂移是一个经典问题。一个 agent 可能因为上下文膨胀、工具调用失败或采样温度过高而偏离最初指令。解决方案是**重锚定 system prompt**：定期向 agent 注入原始目标，或通过验证循环检查输出是否匹配任务。复杂的 harness 会实现验证步骤，例如检查输出格式或对模型编写的代码运行测试用例（来源：幻觉与验证循环）。缺乏这些验证会导致“更多幻觉和重复工作”（来源：幻觉与验证循环）。

Focusmate 之于 ADHD，就像重锚定 system prompt 之于 LLM——但有人用错了。

## 判断：Focusmate 只补上了外部问责层，而非完整的验证循环

Focusmate 有效，但它的作用范围有限。它主要补偿了**任务启动**和**注意力维持**，即通过社会压力让你开始并留在任务上。但它不提供任务分解、时间管理或结果验证——这些是更高级的执行功能。相比之下，RescueTime 通过自动时间追踪补上了**时间感知**层（来源：RescueTime），帮助 ADHD 用户识别注意力漂移模式；而 Brain.fm 试图通过神经锁相调节大脑状态（来源：Brain.fm），但缺乏 ADHD 专项研究。

问题在于：许多人把 Focusmate 当作万能拐杖，而不是针对性脚手架。他们以为有了伙伴就能解决所有执行功能缺陷，但实际中，如果任务本身没有分解、没有明确的子目标，即使有伙伴，你仍可能在一个大任务中“超聚焦跑偏”——比如花两小时美化 PPT 的字体，而不是完成核心内容。

### 脚手架 vs 拐杖的边界

wiki 资料中反复警告过度依赖风险：“过度依赖可能削弱内在时间感知能力”（来源：矛盾与存疑）。Focusmate 作为外部执行功能层，长期使用是否会导致内在执行功能退化？关键区别在于：**脚手架最终可以拆除，拐杖则让人永远依赖**。如果你只在需要启动困难任务时使用 Focusmate，并逐渐学会自我启动，它就是脚手架；如果你每次工作都必须有伙伴在场，否则无法开始，它就变成了拐杖。

## 争议与局限

1. **证据强度不足**：Focusmate 的用户研究显示其能显著提升任务启动和维持，但缺乏随机对照试验（来源：AI 与 ADHD 的专注力）。其效果高度依赖配对伙伴的可靠性，匹配算法虽优化但无法保证每次体验一致（来源：Focusmate）。
2. **个体差异**：ADHD 的异质性使得同一工具对不同亚型效果迥异（来源：矛盾与存疑）。社交焦虑用户可能因实时视频压力而适得其反。
3. **同构的局限性**：ADHD 大脑与 LLM 的同构是隐喻而非严格等价。LLM 的“注意力机制”与人类注意力存在本质差异（来源：AI 与 ADHD 的专注力）。Focusmate 的“身体在场”效应涉及镜像神经元等人类特有的社交机制，无法直接映射到 agent 工程。

## 行动指南：今天就能试的 3 件事

1. **用 Focusmate 做一次“锚定实验”**：预约一个 50 分钟的 Focusmate 时段，只做一件事——把一个大任务分解成 3-5 个子步骤。结束后问自己：是伙伴的压力让我启动，还是分解后的步骤让我保持专注？如果是前者，下次尝试在无伙伴时先分解任务。
2. **为你的 agent 添加显式重锚定**：在 agent 的 system prompt 末尾加入“每执行 3 步，回顾原始目标并输出‘当前目标：...’”。这模仿了 Focusmate 的验证循环。
3. **对比 RescueTime 与自我记录**：开启 RescueTime 自动追踪一周，同时每天手动记录“我实际做了什么”。对比两份记录，找出你的“目标漂移模式”——是特定时间、特定应用还是特定任务类型？这种觉察本身就是验证循环。

## 结语

Focusmate 不是万能药，但它揭示了 ADHD 与 LLM 共享的深层需求：一个可靠的外部锚点来对抗目标漂移。理解这一点，你就能从“用工具”升级为“设计自己的 harness”——无论是为你的大脑，还是为你的 agent。

# 为什么用 Speechify 治 ADHD 的时间盲，和给 agent 套 planner-executor 任务分解 是一回事？

> 时间盲的一个具体形态,藏在「读完这份材料」这类任务里:它的主观时长是不可估的——可能半小时,可能一下午,取决于状态、难度、走神的次数;而**不可估时长的任务,无法被排进任何计划**:你不知道给它留多少时间,于是要么不排(它就永远漂着),要么排了必然超时(计划的信用被又一次击穿)。Speechify 这类 TTS 工具在这个问题上有一个很少被点破的副作用:**它把一份「主观时长不可估」的文本,转换成一段「物理时长已知」的音频**——2 万字,1.5 倍速,52 分钟,不多不少。这个转换,正是 planner-executor 架构里 planner 对任务的第一道要求:**步骤的预算必须可估,否则计划只是愿望**。

先讲 agent 侧的对应。planner-executor 的分工:planner 把大任务拆成步骤序列,executor 逐步执行;而这个架构能运转的隐含前提,是**每个步骤有可估的资源预算**(token、时间、调用次数)——「研究一下这个问题」是 planner 的噩梦(开放式,预算无界),「读取这三个文件并总结」才是合格的步骤(边界清晰,成本可估);所以任务分解的真正功夫,不在「切小」,在**把开放式任务转换成有明确交付物、可估预算的封闭任务**。预算不可估的步骤,要么在执行时爆掉时间线,要么诱发无限递归的「再研究一下」。

ADHD 的时间盲,让「预算估计」这一步在几乎所有任务上失灵:对「读材料」「写报告」「整理房间」的时长感知,系统性地失真(前瞻性时间估计的偏差有实验文献方向);而估不准的后果与 agent 一致:**计划做不出来,或做出来就崩**——崩几次之后,人干脆放弃计划,回到「凭状态漂流」的模式,时间盲从感知缺陷升级为生活方式。外部工具的杠杆,由此不在「提醒你时间过了多久」(那是计时器的活),在**把任务本身改造成时长已知的形态**。

TTS 的转换就是一个现成的改造器:**①物理时长锚**——文本转音频后,「读完它」变成「听完 52 分钟」:主观的雾,变成进度条上的数字;这个数字可以直接进日历(Reclaim/Structured 篇的排程,第一次有了可信的输入);**②天然的分块**——52 分钟可以切成三段通勤、两次散步:**音频的时间轴让「分几次完成」的切分点变得具体**(第 18 分钟停的,下次从第 18 分钟继续——进度指针白送);**③超时的不可能性**——音频不会像阅读那样「读着读着两小时没了」:52 分钟就是 52 分钟,播完即止,**任务的时间边界第一次由物理而非意志保证**。同样的改造思路可以推广到 TTS 之外:任何把开放任务转成「有物理终点」的手法(写作转成「写满一页纸」、整理转成「装满一袋」)都是同一味药——**给不可估的任务,找一个可估的物理代理**。

边界:**其一,播完不等于读懂**——音频的物理终点保证的是「过了一遍」,理解的验证要靠输出(听完写三句总结,Lex 篇的逻辑);难度高的材料,52 分钟的音频可能需要两倍时间的暂停与回放,预算按 1.5 倍留;**其二,TTS 对 ADHD 时间管理无直接临床证据**——本篇的链条是:时间估计偏差(有文献方向)+物理时长锚(机制清晰)+产品(无对照),实践建议均可用系统自带 TTS 零成本验证;**其三,公式/图表密集的材料不适用**(音频表征失真的老边界)。

## 边界

同构强度 B 级:前瞻性时间估计偏差有 ADHD 侧研究方向,步骤预算可估是 planner-executor 的工程实体,「物理时长代理」的同构清晰;TTS 无 ADHD 时间管理对照研究。声明:时间管理的广泛困难,评估优先;工具是感知辅助。

## 今天就能试的 3 件事

1. **给一份拖了很久的长材料转音频**:看一眼总时长——这个数字,就是它真实的大小,拿去排程。
2. **用时长切块**:52 分钟切成三段,各绑一个场景(通勤/散步/做饭)——分次完成,指针自动保留。
3. **听完写三句**:每段结束,三句话总结——把「播完」升级成「过脑」。

「读完这份材料」是一朵不可估的雾,**「听完 52 分钟」是一个可以搬进日历的砖块**。planner 需要预算可估的步骤;时间盲的大脑,也需要把雾换成砖。

# Claude 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

## 问题：你的 Claude 为什么越用越乱？

你打开 Claude，告诉它：“帮我规划今天的工作。”它列出完美清单。你开始执行，十分钟后刷到一条消息，回来发现 Claude 还在等你回答上一个问题。你忘了刚才要做什么，Claude 也忘了——它没有上下文记忆，就像你的工作记忆一样脆弱。

这不是巧合。ADHD 大脑与 LLM 在结构上同构：两者都是**高产但缺乏可靠执行调度层的生成核心**（来源：AI 与 ADHD 的专注力）。你的大脑创意发散、联想丰富，但执行功能（工作记忆、时间感知、任务启动）常掉链子；LLM 能生成流畅文本，但上下文一长就目标漂移、注意力崩溃。

所以，Claude 对于 ADHD 的真正价值，不是“帮你做事”，而是作为**重锚定 system prompt**——就像在长程 agent 任务中，工程师必须定期给 LLM 重新注入原始目标，防止它跑偏。但多数人用反了：他们把 Claude 当拐杖，让它代劳一切，结果内在能力越来越弱。

## 同构：ADHD 大脑与 LLM 共享的“执行功能漏洞”

最新研究用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中模型表现正常，但随序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突（来源：认知负荷）。这与 ADHD 执行功能缺陷的核心标志一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃。

ADHD 的“超聚焦跑偏”与 LLM 的“上下文膨胀”是同一类问题。你本想写报告，却花两小时研究报告里提到的某个冷知识；LLM 本想回答用户问题，却被长对话中的无关细节带偏。两者都需要一个外部机制来**重锚定**当前目标。

## 重锚定：Claude 的正确用法是“脚手架”

在 AI agent 工程中，解决目标漂移的标准方案是：定期向 system prompt 注入原始目标，重置上下文。对 ADHD 而言，Claude 应该扮演这个“重锚定”角色——不是替你做决策，而是帮你**保持对目标的感知**。

证据显示，有效的 AI 工具通过提供**外部执行功能层**来补偿缺陷：
- **Focusmate**：利用“AI-Matched Body Doubling”提供实时问责，弥补多巴胺调节不足（来源：Focusmate）。它不是替你工作，而是让你在他人注视下启动任务——这就是一个“社会性重锚定”。
- **RescueTime**：自动记录时间使用，帮你识别注意力漂移模式，对抗“人在心不在”的状态（来源：RescueTime）。它不替你管理时间，而是让你看到时间去哪了——这是“数据性重锚定”。
- **Brain.fm**：通过神经锁相技术影响大脑信息处理方式，帮助进入专注状态（来源：Brain.fm）。它不替你思考，而是调节环境——这是“生理性重锚定”。

这些工具的共同点是：**它们不替代你的执行功能，而是为你的执行功能提供外部锚点**。就像 LLM 的 system prompt 不替模型生成内容，但确保生成方向正确。

## 边界：脚手架 vs 拐杖

但这里有一条危险的界限。多数 AI 工具（包括 Claude 本身）被设计成“替你做完一切”：自动分解任务、自动写邮件、自动规划日程。这看起来美好，实则可能削弱内在能力。

矛盾在于：工具设计者声称是“脚手架”（临时支持），但实际使用中可能沦为“拐杖”（永久依赖）。（来源：矛盾与存疑）过度依赖 AI 工具可能削弱内在时间感知能力、任务启动能力（来源：矛盾与存疑）。

一个 ADHD 用户告诉我，他让 Claude 帮他写周报，三个月后发现自己再也写不出像样的句子——不是能力退步，而是大脑习惯了“先让 AI 生成，我再修改”的模式，失去了从零启动的勇气。

真正的脚手架应该像 Focusmate：它提供结构，但你必须自己打字。或者像 RescueTime：它记录数据，但你必须自己解读。**重锚定不是替代，是提醒**。

## 今天就能试的三件事

1. **把 Claude 当“重锚定 prompt”而非“代写员”**：每天开始工作前，让 Claude 以问题形式复述你的核心目标（例如：“你今天的首要任务是什么？当前进展到哪一步？”）。回答它，然后关掉它。这不是对话，是自我锚定。

2. **用 RescueTime 设置“漂移警报”**：开启实时提醒，当你离开预设的专注应用超过 5 分钟时，它会弹窗问“你还在做原任务吗？”——这就是 LLM 工程中的“context reset”。

3. **体验一次 Focusmate 的“身体在场”**：预约一个 25 分钟时段，告诉伙伴你要做什么。不必聊天，只需在摄像头前工作。结束后记录感受——你会发现，外部问责比任何 AI 规划都有效。

## 局限与诚实

必须承认，这些建议的证据强度有限。多数工具仅有用户报告或小样本研究，缺乏大规模随机对照试验（来源：矛盾与存疑）。Brain.fm 针对 ADHD 的效果缺乏独立临床研究（来源：Brain.fm）。Focusmate 依赖实时视频，对社交焦虑用户可能造成压力（来源：Focusmate）。

更重要的是，ADHD 的异质性使得同一工具对不同亚型效果迥异（来源：矛盾与存疑）。一个以“注意力分散”为主的人，可能更需要 RescueTime；而以“启动困难”为主的人，可能更需要 Focusmate。没有万能工具。

同构本身也有局限：LLM 的“注意力机制”与人类注意力存在本质差异（来源：AI 与 ADHD 的专注力）。这个类比是启发式的，不是严格的神经科学模型。

## 结论

Claude 之于 ADHD，就像重锚定 system prompt 之于 LLM——但前提是你把它当脚手架，而非拐杖。真正的价值不在于“让 AI 替你做”，而在于“让 AI 帮你记住你要做什么”。

下次打开 Claude 时，问自己：我是在用它重锚定自己的目标，还是在用它逃避启动的困难？答案决定了它将成为你的执行功能层，还是你的认知拐杖。

# 为什么用 Claude 治 ADHD 的情绪失调，和给 agent 套会褪去的脚手架是一回事？

> 情绪调节是 ADHD 里最少被谈论、却最疼的那部分。用 Claude 陪你把情绪风暴说清楚,是真实有效的支撑——但这个支撑必须按「会褪去的脚手架」来设计,因为它要交还给你的,是一项你可以真正长出来的技能。

先把情绪失调请回 ADHD 的核心叙事里。诊断标准聚焦注意力和多动,但研究者(如 Barkley)长期主张:情绪调节困难是 ADHD 的核心特征之一——挫折耐受低、情绪来得快且猛、被批评时的反应强度远超事件本身(社区称为 RSD,拒绝敏感性烦躁,虽尚非正式诊断)。机制上这与执行功能一脉相承:**情绪调节本身就是一种执行功能**——对情绪反应的抑制、评估、重构,走的正是前额叶那条偏弱的通路。情绪不是比别人多,是刹车比别人软。

## Claude 能补的那一段:情绪的「命名与重构」回路

情绪调节的认知疗法路径里,有两个被反复验证的动作:**情绪标注**(affect labeling——把感受精确说出来,本身就降低杏仁核反应强度,有神经影像研究支持)和**认知重评**(重新解释事件的意义)。这两个动作 ADHD 者独立做很难——风暴当中,负责这两个动作的执行功能恰好也被风暴淹了。

Claude 在这里是个出色的陪练:风暴来时打开它——「我现在情绪很大,帮我做三件事:1) 帮我把这个感受命名精确(是愤怒、羞耻、还是被忽视感?);2) 复述一遍激发事件,只讲事实不讲解读;3) 给我两个对这件事的其他解释。」十分钟,风暴通常降半级。这不是玄学,是把 CBT 的标准动作外包给一个耐心、不评判、凌晨三点也在线的对话者。

## 但注意:这个脚手架的设计目标是拆除

关键在下一层。上面那个协议里的三个动作——命名、事实分离、重评——**是可学习的技能,不是终身缺失的架构**。这正是它和「时间盲的外部时间轴」的本质区别:时间轴是假肢(架构短板,终身合法);情绪调节陪练是脚手架(技能短板,练得会)。CBT/DBT 的全部临床逻辑就建立在这条上:调节技能通过反复练习可以内化。

所以用法必须带褪去设计,像给 agent 的推理脚手架一样分阶段撤:

**第一阶段(全支撑)**:每次风暴都用完整协议,让 Claude 带你走三步。此阶段你在观摩「情绪被拆解的样子」。

**第二阶段(半支撑)**:风暴来时,先自己走三步(哪怕很烂),再打开 Claude 校对:「我自己的命名是 X、事实是 Y、重评是 Z——帮我看哪步走歪了。」生成回到你手里,它退到批改位。

**第三阶段(按需支撑)**:日常风暴自己处理,只有超量级的才调用。判断系统是否健康的指标:**调用频率随月份下降,自主处理成功率上升。**

## 反模式:把陪练用成情绪奶嘴

不带褪去设计的用法,风险真实存在:每次情绪波动第一反应是打开对话框,倾倒、获得安抚、关闭——不练命名、不练重评,只取即时缓解。几个月后调用频率不降反升,且开始出现「不倾倒就无法平复」的依赖。这等于给 agent 的脚手架越搭越厚,模型能力原地踏步。判别句:**每次对话结束,你是多会了一点,还是只多舒服了一点?** 长期只有后者,重设协议。

还有一条硬边界:Claude 是陪练不是治疗师。如果情绪失调的强度已经涉及自伤念头、持续两周以上的低落、或严重影响关系与工作——那是治疗议题(ADHD 共病情绪障碍很常见),AI 陪练在这个量级上不够格也不该顶格用。

## 边界

同构强度 B 级:脚手架与撤除是真实 ML 训练概念,情绪标注与认知重评有神经科学和临床证据,ADHD 情绪失调的核心地位有研究支持(RSD 为社区概念,证据在发展中);「AI 陪练走 CBT 动作」是机制合理的实践,无对照研究,不替代心理治疗。

## 今天就能试的 3 件事

1. **把三步协议存成快捷指令**:命名/事实/重评,风暴来时直接粘贴。第一阶段,今天就位。
2. **风暴过后做一次复盘**:平静下来后问 Claude:「刚才那轮,我的哪个解读被证明夸大了?」重评能力的肌肉记忆,一半长在复盘里。
3. **在日历上给「第二阶段」定个日子**:一个月后。到期评估:能不能先自己走三步?

情绪风暴不是你的人格,是刹车软的系统撞上了陡坡。脚手架搭起来,技能练出来,然后——这是最好的部分——把脚手架一层层拆掉,那个能自己刹住车的你,是练出来的,也是真的。

# Otter.ai 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Otter.ai：它到底补上了哪一层执行功能？

## 问题：为什么你录了所有会议，却依然无法启动？

你花了三小时转录一场头脑风暴，Otter.ai 的 AI 自动生成了逐字稿、摘要、行动项。但第二天，你盯着那条“写市场报告”的任务，手指悬在键盘上——然后刷了四十分钟短视频。这不是你的错，也不是 Otter.ai 的错。这是 ADHD 大脑与 LLM 共享的结构性缺陷：**高产生成核心，缺执行调度层**（来源：ADHD 大脑与 LLM 的同构）。

## 同构：ADHD 大脑与 LLM 都是“无调度生成器”

ADHD 大脑的多巴胺系统失衡导致中脑皮质分支功能低下，引起行为计划不良和执行功能差（来源：多巴胺）。LLM 则在状态保持、真实规划、上下文控制上存在缺陷（来源：ADHD 大脑与 LLM 的同构）。两者都擅长“生成”——创意、联想、语言流——但无法可靠地“调度”：把长任务分解成可执行的子步骤，按顺序执行，并在干扰后重新规划。

这正是 **planner-executor 循环** 要解决的问题。在 Agent 工程中，一个 planner 将用户指令分解为子任务，executor 逐个执行，然后 planner 根据结果调整下一步。ADHD 的“执行功能支架”也是同一结构：外部工具承担 planner 角色，把“写报告”拆成“打开文档、写第一句、写三段”，并定时 re-grounding。

## 工具实测：Otter.ai 补上了哪一层？

Otter.ai 的核心价值不在转录，而在于它天然是一个 **外部记忆 + 任务分解引擎**。它把会议中的口头讨论转化为结构化文本，并自动提取行动项。这直接对应 ADHD 的两个缺口：工作记忆容量有限（来源：工作记忆），以及任务启动困难（来源：任务启动）。Otter.ai 的 AI 生成的摘要和行动项，相当于把“下一步做什么”从大脑中卸载到外部界面。

但问题在于：**大多数用户只用了 Otter.ai 的转录层，没用到其规划层**。他们录了会议，导出文本，然后就没有然后了。这就像给 LLM 装了一个强大的生成器，却忘记接上 planner——生成内容堆在那里，没有分解、没有调度、没有执行。

对比其他工具：**Motion** 的自动排程与动态调整，直接执行 planner 功能——它持续评估任务优先级、截止日期和可用时间，实时重建日程（来源：Motion）。**Reclaim.ai** 则更强调“保护”而非“填充”，通过智能时间块防御会议侵占（来源：Reclaim.ai）。**Tiimo** 用视觉化时间线直接应对时间盲（来源：Tiimo）。这些工具都在不同层面补上调度层，但 Otter.ai 的独特之处在于它从“生成”端切入——它天生是会议内容的 planner，只是用户没把它接上 executor。

## 人物案例：孔子的“日课”与 LLM 的 re-grounding

孔子是典型的 ADHD 大脑：精力旺盛、冲动骂人（“朽木不可雕”）、思维跳跃（《论语》全是场景化语录）、对音乐超专注但对种地零耐心（来源：人物案例）。他的 harness 系统是“克己复礼”——用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”）。这本质上是一个 **外部调度器**：每天固定时间 re-grounding，检查自己是否偏离轨道。

这与 LLM agent 的“定时 re-grounding”同构：Agent 在长任务执行中，定期调用 planner 重新评估进度、调整计划。孔子的“日课”就是他的 re-grounding 循环。Otter.ai 如果只录音不回顾，就相当于孔子只记录言行却不反省——记录本身不改变行为，只有加上“每日三省”的规划循环，才能从拐杖变成脚手架。

## 核心观点：Otter.ai 的正确用法是“planner-executor 循环”

我的判断是：**Otter.ai 之于 ADHD，就像 planner 之于 LLM——但大多数人把它当成了录音机。** 正确的用法是：

1. **用 Otter.ai 生成行动项（planner 输出）**，而不是只存转录稿。
2. **将这些行动项导入任务管理系统（如 Todoist 或 Motion）**，让外部调度器接管执行。
3. **设置定时回顾（re-grounding）**，类似孔子的“日省”，检查行动项是否被推进。

这样，Otter.ai 就从一个被动的转录工具，变成了主动的规划引擎——它补上了 ADHD 大脑最缺的那一层：**从生成到执行的调度桥梁**。

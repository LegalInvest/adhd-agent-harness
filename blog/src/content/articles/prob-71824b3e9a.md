# 为什么用 Claude 治 ADHD 的日常混乱不稳定，和给 agent 套 调低采样温度 是一回事？

> 先看两份「病历」。病历一:一位 ADHD 用户描述自己的日常——同一件事,周一的做法和周三的完全不同;出门流程每天现场重新发明,所以每天丢的东西也不重样;回邮件有时秒回三段,有时拖一周;生活不是没有能力,是**没有一致性**——每个时刻都在所有可能的行为里「高随机采样」。病历二:一个 LLM 应用的故障单——同样的 prompt,昨天返回规整的 JSON,今天多了一段即兴发挥;温度(temperature)设得偏高,输出分布的尾部概率被放大,**每次生成都在更宽的可能性里掷骰子**;工程师的处置朴素而有效:把温度调下来——不改模型、不重训练,只是收窄采样分布,输出立刻从「有创意的混乱」变成「无聊的稳定」。这两份病历的同构,是本系列证据等级最高的一类:**行为的日内变异性(intra-individual variability)是 ADHD 研究里最稳健的发现之一**——比「注意力差」更能区分 ADHD 与对照组的,常常是反应时的波动幅度;而采样温度对输出方差的调控,是 LLM 的基础数学。**「混乱」在两边都不是能力问题,是方差问题——而方差,是可以工程化调低的**。

那么「用 Claude 调低一个人的温度」是什么意思?不是让 AI 控制你,是**让它承担「低温采样器」的角色:在你的高方差行为空间里,固定住那些本不该每天重掷骰子的决策**。拆成三个可执行的机制。**机制一:把重复决策「缓存」成标准作业**。你每天现场发明的那些流程(出门检查/回邮件格式/买菜清单),本质是每天对同一问题的重新采样——每次采样都有出错的尾部概率;和 Claude 做一次性的工作:「帮我把出门流程固化成一个五步清单,考虑我常丢的三样东西」——**生成一次,之后只执行不再生成**(工程对应:对确定性任务关掉采样,用缓存的确定性输出)——你的创造力从此不必浪费在「今天怎么出门」上。**机制二:用固定 prompt 稳定输出格式**。日常里大量的不稳定,其实是「格式」不稳定:汇报有时三行有时三页、计划有时精细有时没有;给自己建几个「模板 prompt」存在 Claude 的固定对话里(周计划模板/邮件回复模板/决策记录模板),**每次同类任务都从同一个模板起手**——模板就是你的低温约束:它不禁止内容的变化,只收窄结构的方差(工程上这叫约束解码:内容自由,格式锁死)。**机制三:高低温分区**。这是最重要的一步,因为「把温度全调低」是对 ADHD 的误治——**你的高温采样正是创造力所在**(发散联想、跨域连接、即兴解题——27b6 篇说的资产),一刀切的低温会把人调成自己讨厌的机器,而且坚持不了三天;正确的架构是**分区控温**:例行事务区(出门/账单/邮件/例程)强制低温——清单化、模板化、不商量;创造区(写作构思/项目设想/兴趣探索)保留高温——不设格式、允许发散,甚至专门排「高温时段」让它合法燃烧;**Claude 在两区的角色也不同:低温区它是模板执行器,高温区它是发散的陪练**——一套系统,两种温度,这才是对这个大脑的忠实工程。

这套方案的边界,恰好也能用温度这个隐喻讲清楚(讲清边界本身就是 A 级同构该有的纪律):**其一,温度类比描述的是「行为输出的方差」,不是神经机制的等同**——ADHD 的变异性有它的生理学讨论(唤醒调节/默认网络的入侵等研究方向),和 softmax 的分母不是一回事;类比在计算层成立,别下探到实现层(0457 篇的层级纪律)。**其二,人的「温度」不是一个你能直接拧的旋钮**——清单和模板收窄的是环境给行为的约束,真正的状态方差(今天的大脑就是不听话)有它自己的层:睡眠、药物、情绪,那些归临床管;**清单能稳住的是「决策的方差」,稳不住「状态的方差」**——两种方差要分开对待,后者显著失控时,请让医生而不是模板出场。**其三,警惕低温区的蔓延**——系统一旦尝到稳定的甜头,会有把一切都清单化的冲动,而对 ADHD,过度结构化的反噬是确定的(窒息感→整体弃用);低温区的准入标准要严:**只固化那些「重复出现且创造力毫无用武之地」的决策**,其余一律留给高温。

## 边界

同构强度 A 级:ADHD 的日内行为变异性是该领域最稳健的研究发现方向之一,采样温度对输出方差的调控是 LLM 基础机制,「方差问题而非能力问题」的对应有直接文献支撑;但温度类比限于计算层描述,分区方案为实践翻译。声明:Claude 未被证实为 ADHD 治疗手段;状态层面的显著波动(睡眠/情绪/药物相关)请由临床评估处理。

## 今天就能试的 3 件事

1. **挑一个每天重新发明的流程**,和 Claude 一次性固化成五步清单——生成一次,以后只执行。
2. **建一个模板 prompt**:周计划或邮件回复,存进固定对话——同类任务永远从同一模板起手。
3. **在日历上划出「高温时段」**:每周两小时,合法发散,不设格式——低温是为了保住高温,别搞反了。

混乱不是你的人格,是你的采样温度——**而工程师面对高方差系统的第一反应从来不是责备它,是分区控温**。例行事务锁进低温,创造力留在高温:Claude 帮你缓存前者、陪练后者——稳定和才华,从此不必二选一。

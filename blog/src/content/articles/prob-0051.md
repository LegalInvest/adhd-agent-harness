# 3 个 LLM 工程概念，彻底重写了我对 ADHD 不擅长精确、重复、状态维护类任务的理解

> agent 通过 function calling 把精确计算外包给工具 如何反向照亮 ADHD 的不擅长精确、重复、状态维护类任务。

我记不清有多少次，面对一个“把书桌收拾干净”的指令，大脑直接死机。不是不想做，而是“收拾”这个词在我脑中炸开成无数个碎片——先分类？垃圾扔哪？抽屉里那些充电线怎么办？然后我打开手机刷了二十分钟短视频。这种体验，ADHD人群太熟悉了：不是懒，是执行功能在关键时刻掉链子。

但最近，我在研究LLM Agent工程时发现，那些困扰我的问题——不擅长精确计算、重复操作、维护状态——竟然与AI Agent的痛点如出一辙。三个LLM工程概念，像三束光，彻底照亮了我对ADHD的理解。

## 概念一：Function Calling——把精确计算外包给工具

在LLM Agent架构中，模型本身并不擅长数学运算或精确计时。所以工程师设计了function calling：当Agent需要计算“3.14×5²”时，它不会自己硬算，而是调用一个计算器工具，把结果拿回来。Agent只负责决策和生成，精确计算交给专用函数。（来源：AI与ADHD的时间管理）

这完全就是ADHD大脑的写照。我的工作记忆容量足够，但控制弱——就像LLM有强记忆容量但弱上下文控制（来源：ADHD大脑与LLM的同构）。当需要精确、重复、状态维护时，大脑就像没有function calling的Agent，试图用生成能力硬撑，结果频繁出错或直接放弃。

Goblin Tools的Magic ToDo正是这个思路的ADHD版本。它把“整理房间”这种模糊指令分解为“捡起地板上的衣服”“擦桌子”等具体步骤（来源：Goblin Tools）。这相当于为我的大脑提供了现成的function calling——我不需要自己设计执行路径，只需要按顺序调用。用户报告称启动焦虑显著降低（来源：Goblin Tools）。

## 概念二：Harness（脚手架）——为生成核心套上执行层

Agent工程中，一个关键组件是harness——它负责管理上下文、调度工具调用、处理状态。没有harness，LLM就像没有方向盘的车：能跑但不知道往哪开。ADHD大脑同样缺少这个“执行调度层”。

Saner.AI的设计直接对应这个需求。它通过本地记忆和快速检索，减少搜索循环和标签切换（来源：Saner.AI）。这相当于为ADHD大脑提供了一个外部harness：当我要写一篇文章时，不需要从零回忆资料，Saner.AI自动把相关记忆拉出来。类似地，Lex允许通过单一指令触发多步骤任务序列（来源：Lex），就像Agent的一个function calling链。

但这里有一个关键的边界：脚手架 vs 拐杖。脚手架帮助我学会自己搭建结构，拐杖则让我彻底放弃努力。Otter.ai的实时转录减轻笔记负担，但过度依赖可能削弱主动记笔记的能力（来源：ADHD的AI工具全景）。同样，Agent如果过度依赖harness，模型本身的推理能力会退化。好的工具应该像脚手架，随着使用逐渐内化为能力。

## 概念三：上下文工程——主动设计“当下注意什么”

LLM的上下文窗口有限，且容易膨胀——塞入过多信息后，模型会忽略关键内容。因此工程师做上下文工程：精心设计prompt，只保留最相关的信息。

ADHD的“上下文工程”同样关键。我们容易被环境带偏，因为大脑无法自动过滤干扰（来源：上下文工程）。Claude的对话式整理和Reflect的结构化反思，都是主动设计“当下注意什么”的方案（来源：ADHD的AI工具全景）。比如，我在写作时用Claude作为外部对话记忆，把临时想法丢给它，而不是让它们在脑中乱窜。这相当于为大脑做了上下文裁剪。

## 争议与局限

我必须诚实指出：这些工具的证据主要来自用户报告和概念类比，缺乏大规模随机对照试验（来源：矛盾与存疑）。AI工具的实际效果尚未被严格验证，存在夸大宣传的可能。更重要的是，个体差异极大——对A有效的Goblin Tools，对B可能只是另一个拖延的借口（来源：矛盾与存疑）。

另外，多巴胺干预效果本身存在争议（来源：矛盾与存疑），而很多工具的设计恰恰依赖多巴胺奖励机制。这意味着，工具可能短期内有效，但长期会加剧依赖。

## 今天就能试的行动

1. **用Goblin Tools分解一个让你拖延的任务**：输入“整理工作邮箱”，看它如何拆解。感受启动门槛降低的瞬间。
2. **在ChatGPT中实践function calling思维**：下次需要计算或查资料时，明确告诉AI“帮我调用计算器/搜索工具”，而不是让它硬猜。
3. **用Saner.AI建立外部记忆**：把分散在各处的笔记、想法汇总到一处，减少“搜索循环”的认知负荷。
4. **每天一次“上下文裁剪”**：选一个任务，写下当前唯一需要关注的事，把其他念头丢给备忘录或AI。

这三个概念让我意识到：ADHD不是缺陷，而是一种“高生成低调度”的架构。而AI工具，正是我们等待已久的外部执行层。关键在于，我们用它来搭建脚手架，而不是永久拐杖。

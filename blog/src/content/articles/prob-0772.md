# Forest 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Forest：它到底补上了哪一层执行功能？

## 问题：为什么 Forest 对你没用，对别人却有用？

你下载了 Forest，种下一棵树，告诉自己“25 分钟内不许碰手机”。但 10 分钟后，你开始焦虑——不是想刷社交媒体，而是突然想起还有邮件没回、有个 deadline 逼近、还有一堆琐事需要处理。你盯着那颗虚拟树，感觉它不是在帮你专注，而是在把你关进一个笼子。

另一边，你的工程师朋友在调试一个 AI agent。他们给 LLM 加了 function calling——让它能调用计算器、数据库、API。LLM 现在可以精确计算 3.14×17 了，但 agent 却频繁出错：不是算不准，而是调用完一个函数后，它忘了自己原本要做什么。

两个场景，同一个问题：**一个高产的生成核心，被塞进了一个不匹配的执行框架。**

## 同构：ADHD 大脑与 LLM 共享的“执行功能缺口”

ADHD 大脑常被描述为“高产但缺乏可靠执行调度层的生成核心”——想法丰富、联想活跃，但在筛选、组织、启动和坚持任务上存在困难（来源：ADHD 的 AI 工具全景）。同样，大语言模型（LLM）具备强大的生成能力，却在长程上下文、目标保持与执行调度上存在天然缺陷。两者的本质问题是：**底层生成能力强，但缺少一个能稳定调度、精确执行的中层架构。**

这正是“工具使用与认知卸载”的核心：ADHD 个体需要外部工具来补偿工作记忆不足（来源：无状态与外部记忆），而 LLM agent 需要 function calling 来外包精确计算。Forest 之于 ADHD，就像 function calling 之于 LLM——**它本应是一个外部调度层，把生成核心接入可维护的执行轨道。**

但问题在于：很多人用错了。

## 误用：Forest 被当成了拐杖，而不是脚手架

Forest 的设计初衷是“帮你暂时离开手机”。对于 ADHD 用户，这常常被理解为“强制打断超聚焦”——但矛盾在于，超聚焦是否应被引导而非打断？（来源：矛盾与存疑）。强行用 Forest 打断一个正在深度投入的任务，反而会破坏心流。更糟糕的是，有些人把 Forest 当作“专注的唯一手段”，每天种树 8 小时，却仍然完不成重要项目——因为 Forest 只解决了“不分心”，没解决“该做什么”。

同样，在 LLM agent 领域，许多开发者把 function calling 当作“万能工具”——给 LLM 一堆函数，期望它自动解决所有问题。结果 agent 陷入上下文膨胀与推理退化（来源：上下文工程），因为控制逻辑分散在框架默认值和硬编码的控制器中，无法灵活维护。

两者的共同误用在于：**把外部工具当成了替代内部能力的拐杖，而不是增强内部能力的脚手架。** 真正的脚手架，应该像孔子或李白的个人 harness 系统一样，是主动设计的、与自身特质匹配的架构。

## 正确用法：从历史人物与 agent 工程中学习

孔子一生与冲动做斗争——他精力旺盛、坐不住、爱骂人，但他的 harness 系统是“克己复礼”：用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。这相当于一个 **“定时 re-grounding”机制**——每天多次回到核心原则，重新校准注意力。在 LLM agent 中，这对应着 **“定期上下文重置”**：在长程任务中，agent 需要定期回顾原始目标，避免漂移。

李白则用另一种方式：他把所有过剩精力注入诗歌，用酒和诗作为“外部调度器”，不委屈自己做不想做的事。这相当于 **“超聚焦引导”**——不是打断，而是把注意力导向高价值产出。在 agent 中，这对应着 **“任务优先级动态调整”**：根据当前状态选择最合适的工具调用，而不是死板执行预设流程。

**同构对应：**
- 孔子的“日课” ↔ agent 的“定时 re-grounding”
- 李白的“酒与诗” ↔ agent 的“超聚焦引导”

正确使用 Forest 的方式，不是把它当作“强制专注开关”，而是把它当作一个 **“执行调度信号”**：在开始一个明确的任务前，用 Forest 标记一个“执行窗口”，并在这个窗口内只做分解后的最小步骤。就像 LLM agent 在调用 function 前，先明确当前子目标。

这些工具的共同点是：**它们不是替代你的大脑，而是把你的高产生成核心接入一个可维护的执行调度系统。** 就像 LLM 的 function calling 不是替代 LLM 的生成能力，而是让它能精确执行计算。

## 局限与诚实

我们必须承认，目前缺乏长期研究证明这些工具不会导致能力退化（来源：矛盾与存疑）。过度依赖外部调度层，可能让 ADHD 个体在脱离工具后更难自控。同样，LLM agent 过度依赖 function calling，也可能在工具失效时完全瘫痪。

此外，个体差异极大：有人需要强力打断（如番茄钟），有人需要柔性引导（如超聚焦引导）。目前没有对比研究能给出统一结论（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Forest 标记“执行窗口”，而非“专注窗口”**：在开始任务前，先花 2 分钟用 Goblin Tools 或手写分解出 3 个最小步骤，然后种树。树倒之前，只做这 3 步。
2. **建立“日课”模板**：像孔子一样，每天固定时间（如早晨）用 Saner.AI 或笔记工具回顾今日目标，写下“今天最重要的三件事”。
3. **为 LLM agent 设置“上下文重置点”**：在长链任务中，每 5 步插入一个“回顾原始目标”的步骤，防止漂移。
4. **测试你的“超聚焦引导”**：下次进入心流时，不要用 Forest 打断。记录你专注的内容，然后问自己：“这是否是我此刻应该做的？”如果是，继续；如果不是，用 Lex 或笔记快速记录当前思路，然后切换。

Forest 不是专注的解药，function calling 也不是 agent 的万能钥匙。它们只是脚手架——用得好，你能建起高楼；用得不好，你只是在原地种树。

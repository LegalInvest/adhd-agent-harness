# Lex 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Lex：它到底补上了哪一层执行功能？

当你第 10 次对着“整理邮箱”这个指令发呆，或者你的 agent 在第 3 步就忘了上下文，你可能会怀疑：是不是自己/这个模型天生缺了点什么？

答案是：你们缺的是同一个东西——一个可靠的外部执行调度层。ADHD 大脑与 LLM 在结构上同构：都是强大的生成核心，但内置的工作记忆和任务调度模块不稳定（来源：ADHD 大脑与 LLM 的同构）。ADHD 患者在工作记忆任务中易受干扰，而 LLM 同样表现出类似人类的干扰特征——随着记忆负荷增加表现下降，且成功回忆依赖于干扰控制（来源：Human-like Working Memory Interference in Large Language Models）。

这就是为什么 Lex 这类工具让人兴奋，也让人误用。

## Lex 的正确用法：作为外部调度器，而非一键完成器

Lex 的核心设计是“单一指令触发多步骤任务”（来源：Lex）。这听起来像 ADHD 的梦想：说一句“写周报”，它自动拉数据、列要点、生成草稿。但这里有一个陷阱：如果你只把它当作“一键完成”的魔法棒，你实际上是在外包整个思考过程，而不是在搭建自己的执行系统。

正确的类比是：Lex 之于 ADHD，就像 function calling 之于 LLM。在 agent 架构中，function calling 不是让 LLM 自己完成所有计算，而是让它把精确计算（如数学、查数据库）外包给专门工具，自身聚焦于意图理解和决策（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。同样，Lex 应该被用来外包那些“精确、重复、状态维护”的步骤——比如整理文件、格式化输出——而保留需要你独特判断和创造力的核心决策。

## 同构的证据：从孔子到 Transformer

孔子（春秋/儒家创始人）一生和冲动做斗争：他精力旺盛、坐不住，周游列国 14 年，对种地零耐心，思维跳跃到《论语》全是场景化语录。他的 harness 是“礼”——用外在的行为边界（日省、克己）来补偿内部调度缺陷。这就像 LLM 的 context window 和 system prompt：外部边界决定了它能可靠运行的范围。孔子直到 70 岁才“从心所欲不逾矩”，说明脚手架需要终身维护。

更硬核的证据来自计算神经科学：Transformer 在训练后，其自注意力机制发展出模仿前额叶-纹状体门控的操作，证明了计算同构性（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。这意味着，ADHD 大脑的“注意力门控失效”与 LLM 的“注意力漂移”在机制上平行。

## 工具不是答案，脚手架才是

现在市面上的 AI 工具，大多数在扮演“拐杖”而非“脚手架”。Goblin Tools 把任务分解成小步，Saner.AI 减少搜索循环，Mem 做第二大脑——它们都降低了启动门槛，但可能让你越来越依赖它们替你“想”出下一步。真正的脚手架应该帮你建造自己的执行系统：就像孔子用“礼”内化成习惯，而不是每次都要查《论语》才知道怎么做。

争议在于：依赖风险是否被低估？专家警告，若工具替代了治疗或自我管理，可能造成依赖（来源：拐杖与脚手架）。最新研究显示，ADHD 患者的自动反应模式与 LLM 的自动完成机制平行，两者都需要外部解释者提供意义（来源：Automatic Minds: Cognitive Parallels Between Hypn...）。这提醒我们：工具应该增强你的执行功能，而不是让它萎缩。

## 今天就能试的 3 件事

1. **用 Lex 只外包“状态维护”步骤**：比如“整理桌面文件到对应文件夹”，但保留“判断哪些文件重要”这一步给自己。
2. **建立“外部记忆”日课**：用 Mem 或 Otter.ai 每天记录一个关键上下文，像孔子的“日三省”——定时 re-grounding 比一次性记满更重要。
3. **区分“脚手架”与“拐杖”**：每用一周工具，问自己：如果没有它，我还能完成这个任务吗？如果答案是不能，说明你在依赖拐杖，需要调整用法。

## 局限与诚实

必须承认，ADHD 与 LLM 的同构理论目前仍是一种类比，缺乏大规模临床验证（来源：矛盾与存疑）。Lex 的直接研究证据也有限，其效果多基于用户报告（来源：Lex）。此外，工具可能增加注意力分散风险——ChatGPT 等通用工具可能让你在任务中分心（来源：ChatGPT）。所以，这篇文章不是让你去购买某个工具，而是让你重新思考：你是在用工具搭建自己的执行系统，还是把它当作替自己思考的拐杖？

答案决定了你是成为孔子，还是成为那个永远在找下一个魔法工具的人。

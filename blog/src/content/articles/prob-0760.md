# Inflow 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Inflow：它到底补上了哪一层执行功能？

## 问题：为什么你越是依赖 Inflow，反而越无法独立？

如果你是一位 ADHD 患者，可能经历过这样的循环：下载 Inflow，被它的神经科学原理打动，用了几周，然后发现——离开它，你还是那个连刷牙都要纠结五分钟的人。如果你是一位 Agent 工程师，你可能见过另一个循环：给 LLM 接上十几个 function calling 工具，结果模型在工具调用中迷失，上下文爆炸，输出质量暴跌。

这两个问题，本质是同一个问题。

## 同构：ADHD 大脑与 LLM 是同一类“生成核心”

ADHD 的核心困境并非智力不足，而是执行功能失效——计划、组织、工作记忆这些“大脑的驾驶座”常常无人驾驶（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。LLM 同样如此：它拥有强大的语言生成能力，但单独使用时缺乏可靠的执行调度，需要外部的 harness 来编排工具调用、管理上下文、防止推理退化（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

**生成核心**：ADHD 大脑的智力/创造力 ↔ LLM 的算力/语言能力。两者都高产但无序。

**调度层缺失**：ADHD 的执行功能失效 ↔ LLM 缺乏内置的规划与执行编排。两者都需要外部脚手架。

**外部工具**：ADHD 的“自适应计划工具如 Motion 和 Tiimo，能在日程崩溃时重新安排一天”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）↔ LLM 的 harness 提供上下文、记忆、工具接口，弥补无状态和有限窗口（来源：GitHub - ai-boost/awesome-harness-engineering）。

## 答案：Inflow 之于 ADHD，就像 function calling 之于 LLM——但有人用错了

Inflow 声称通过神经科学原理帮助 ADHD 用户改善执行功能，但 wiki 资料明确指出其“未提供具体研究或证据，与多巴胺干预的争议矛盾”（来源：矛盾与存疑）。这不是说 Inflow 无效，而是说它被宣传为“治疗工具”而非“脚手架”。真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

同样，function calling 是 LLM 的脚手架，但若工程师把工具调用当作万能解药，不设计验证循环、上下文压缩和记忆系统，代理就会变成“上下文膨胀、推理退化”的怪物（来源：Building AI Coding Agents for the Terminal）。

**正确用法**：Inflow 应被用作外部执行功能层，类似 LLM 的 harness——它提供结构，但决策和行动仍由用户/模型主导。错误用法是将其视为永久拐杖，替代内在能力的发展。

## 证据：哪些工具真正补上了调度层？

- **任务启动**：Goblin Tools 的 Magic ToDo 将复杂任务分解为小步骤，降低启动门槛（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。这相当于给 LLM 一个分步规划器，防止它直接输出混乱的长序列。
- **工作记忆**：Saner.AI 通过知识回忆减少搜索循环（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。这类似 LLM 的 RAG 系统，把外部记忆当作工作记忆的扩展。
- **多步骤指令**：Lex 允许单一指令触发复杂任务序列（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。这相当于 LLM 的“工具链”或“工作流”，把多个 function call 打包成一个原子操作。
- **持久记忆**：Mem 和 Otter.ai 提供外部记忆，减少上下文丢失（来源：ADHD 的 AI 工具全景）。这对应 LLM 的长期记忆架构，如 Redis 的持久化方案（来源：Long-Term Memory Architectures for AI Agents - Redis）。

这些工具的共同点：它们不是替代你的大脑或模型，而是**外化**了执行功能——把计划、记忆、分解这些任务交给工具，让生成核心专注在真正需要创造力的部分。

## 争议与局限：脚手架还是拐杖？

wiki 资料揭示了几个关键矛盾：

1. **多巴胺干预争议**：Inflow 等工具声称基于神经科学，但多巴胺干预的有效性存在争议（来源：矛盾与存疑）。用户不应期待工具直接调节神经递质。
2. **长期效果未知**：现有证据多为短期或个案，缺乏随机对照试验（来源：ADHD 的 AI 工具全景）。工具可能短期内提升效率，但长期是否阻碍内在能力发展？
3. **撤除机制缺失**：多数工具设计为长期使用，未考虑如何逐步撤除（来源：拐杖与脚手架）。真正的脚手架应可拆卸，但用户可能形成依赖。
4. **同构命题的实证不足**：ADHD 大脑与 LLM 的同构是理论框架，缺乏大规模实证（来源：矛盾与存疑）。类比虽有力，但需谨慎。

**我的判断**：同构视角是有效的启发式框架，但不应被当作科学事实。工具作为脚手架的价值在于“可撤除性”——当你学会任务分解后，Goblin Tools 应被收起；当你的工作记忆改善后，Saner.AI 应只作为备用。Inflow 的问题在于它把自己包装成“解决方案”，而非“训练轮”。

## 今天就能试的 3 件事

1. **用 Goblin Tools 分解一个你拖延了三天的任务**：输入“整理书桌”，观察 AI 给出的步骤。完成后，问自己：下次我能否自己写出类似的步骤？如果不行，继续用；如果可以，尝试减少一次使用。
2. **用 Saner.AI 记录一个常忘的信息**：比如医保卡号或朋友生日。一周后，测试自己是否记住了它。如果记住了，说明工具已经完成了“脚手架”的使命。
3. **用 Lex 创建一个“早间启动序列”**：单一指令触发“打开日历→查看待办→播放专注音乐→开始第一项任务”。如果这个序列让你更顺利地开始一天，请记住：序列是你设计的，工具只是执行者。**你才是那个建造脚手架的人。**

# 用 ChatGPT 给 ADHD 补上重锚定与目标漂移，是脚手架还是拐杖？

> ChatGPT 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：从一次走神开始

你打开 ChatGPT，想让它帮你起草一份周报。三分钟后，你发现自己在问它“如何用 Python 画一只会动的猫”，而周报还在空白文档里。这不是你的错——这是 ADHD 大脑的日常：分心走神或超聚焦跑偏，最终丢失目标。

同样的事也发生在 AI Agent 身上。一个被设定为“撰写市场分析报告”的 Agent，可能在执行中突然开始优化自身 prompt 的格式，或者被上下文中的某个细节带偏，产生目标漂移。工程师们称之为“Agentic Harness 问题”——如何让生成核心始终锚定在初始目标上？

ADHD 大脑与 LLM/Agent 共享同一个深层结构：两者都是**高产但缺乏可靠执行调度层的生成核心**（来源：AI 与 ADHD 的专注力）。你的大脑能冒出无数好点子，但无法可靠地将它们排序、启动、坚持；LLM 能写出流畅文本，但若无精心设计的 system prompt 和上下文窗口管理，它会顺着 token 概率滑向无关方向。

所以，用 ChatGPT 给 ADHD 补上重锚定——这到底是脚手架，还是拐杖？

## 同构：ADHD 大脑与 LLM 的调度困境

ADHD 的执行功能障碍表现为计划、时间管理、任务启动困难（来源：执行功能障碍）。这与 LLM 的“无状态性”高度同构：ADHD 的工作记忆缺陷导致信息易失，LLM 的上下文窗口同样有限，需要外部记忆系统（如向量数据库）来维持连续性。

更关键的是“上下文工程”：ADHD 易被环境带偏，LLM 易被上下文膨胀干扰，两者都需要主动设计“当下注意什么”的方案（来源：AI 与 ADHD 的专注力）。

历史上，两位名人用不同的 harness 解决了类似问题。**曾国藩**（晚清）是典型注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”。他的解决方案是“日课十二条”：黎明即起、读书不二、谨言、写日记反省。这套系统本质上是一个**外部调度层**——它不依赖内在意志，而是用固定的行为规则锚定目标。对应到 LLM 领域，这就是**定时 re-grounding**：每过一段时间，重新向模型注入初始 system prompt，防止上下文漂移。

**孔子**同样如此。他精力旺盛、冲动爱骂人，但对音乐超专注到三月不知肉味。他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省。这相当于 LLM 的**约束性 system prompt**：设定不可逾越的规则，让生成核心在边界内自由发挥。

## 工具：外部执行功能层的现实与局限

今天的 AI 工具正在扮演类似的角色。**Focusmate** 通过实时视频匹配伙伴，提供外部问责，补偿内在启动困难（来源：Focusmate）。它的 AI 调度算法优化配对效率，但核心机制仍是**身体在场效应**——一种社会压力带来的重锚定。这相当于给 Agent 添加一个“监督者”角色，定期检查目标一致性。

**RescueTime** 自动追踪时间，揭示真实的注意力去向（来源：RescueTime）。它解决的是 ADHD 的“时间盲”——无法感知时间流逝。对 LLM 而言，这相当于**日志与回溯机制**：记录每一步的上下文消耗，当发现偏离时触发警告。

**Brain.fm** 利用神经锁相技术生成音乐，帮助进入专注状态（来源：Brain.fm）。这类似为 Agent 设置**环境噪声**——通过外部刺激调节内部状态，但针对 ADHD 的独立临床研究仍然缺乏（来源：Brain.fm）。

然而，这些工具都有局限。Focusmate 依赖实时视频，可能引发社交焦虑；RescueTime 的自动分类需要用户调整，对执行功能弱的用户仍是挑战；Brain.fm 的效果因人而异（来源：矛盾与存疑）。更重要的是，它们可能成为**拐杖**——如果用户完全依赖工具而从不内化任何策略，执行功能可能进一步退化（来源：矛盾与存疑）。

## 脚手架还是拐杖？核心判断

我的判断是：**ChatGPT 等 AI 工具成为脚手架还是拐杖，取决于用户是否主动参与“重锚定设计”**。

如果只是让 ChatGPT 帮你做事，它很可能成为拐杖——你会越来越依赖它替你思考，而自己的执行功能毫无长进。但如果你像曾国藩设计日课一样，用 ChatGPT 构建一套**外部调度系统**——比如每天早晨让它提醒你当天的三个核心目标，每完成一个任务后让它帮你重述下一步，每周让它分析你的时间日志——那么它就是一个脚手架，帮你逐步建立自己的调度能力。

关键在于**主动设计**。曾国藩的日课不是别人强加的，是他自己骂了自己十年后想出来的。同样，你需要自己决定：ChatGPT 的哪部分能力用来补偿缺陷（如时间盲），哪部分用来训练能力（如任务分解）。

## 争议与诚实局限

这个观点并非定论。首先，ADHD 大脑与 LLM 的同构是隐喻而非严格等价——人类的注意力机制与 Transformer 的注意力有本质差异（来源：矛盾与存疑）。其次，长期依赖风险缺乏纵向研究，我们不知道五年后这些用户会怎样。第三，个体差异巨大：对某些 ADHD 亚型，强行打断（如番茄钟）比柔性引导更有效（来源：矛盾与存疑）。

## 今天就能试的行动

1. **设计一个“重锚定 prompt”**：在 ChatGPT 中设置一个系统指令，要求它在每次对话开始和每 5 轮后，主动询问“你当前的核心目标是什么？”。这相当于给 LLM 加一个定时 re-grounding。
2. **用 RescueTime 记录一周**：不评判，只是看数据。找到你最常漂移的时间段，然后在那段时间前用 Focusmate 预约一个伙伴。
3. **模仿曾国藩的“日课”**：列出你最重要的 3 条规则（如“上午不刷短视频”），每天用 ChatGPT 反思是否遵守。让它帮你分析模式，而不是替你做决定。
4. **测试 Brain.fm 或类似工具**：在需要超聚焦时使用，但记录效果。如果发现它让你更易跑偏（比如音乐太有趣导致分心），果断停用。

最终，工具只是工具。真正的 harness 在你手里——就像曾国藩的日课、孔子的“克己复礼”，它们不是拐杖，而是你主动搭建的脚手架。今天，你可以用 ChatGPT 搭建你自己的那一个。

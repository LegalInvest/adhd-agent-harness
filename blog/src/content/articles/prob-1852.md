# ADHD 学生如何用「给生成核心套 harness」的思路应对考试与作业里的不擅长精确、重复、状态维护类任务？

> 把 LLM 工程里的「agent 通过 function calling 把精确计算外包给工具」迁移到ADHD 学生的考试与作业场景，双域证据作答。

## 问题：为什么精确、重复、状态维护类任务对 ADHD 学生如同噩梦？

你是否有过这样的经历：数学考试中明明思路清晰，却因为抄错一个数字或漏掉一个步骤而失分？写作文时大脑里充满精彩的观点，却在誊写或检查时频繁出错？这些场景背后，是 ADHD 大脑的核心痛点：**不擅长精确、重复、状态维护类任务**。这些任务依赖稳定的执行功能——工作记忆、任务启动、持续注意力——而 ADHD 大脑的生成核心（创造力、直觉）虽然强大，但执行调度层（前额叶）却时常掉线（来源：ADHD 的 AI 工具全景）。

与此同时，在 LLM/Agent 工程领域，工程师们早已发现：大语言模型（LLM）同样是“高产但缺执行调度层”的生成核心。它擅长创意、推理和对话，但在精确计算、多步状态维护、重复性操作上频繁出错。解决方案是 Agentic Harness——通过 function calling 将精确计算外包给工具（如计算器、数据库、API），让 LLM 专注于生成与决策。

## 同构：ADHD 大脑与 LLM 是同一类“生成核心”

ADHD 大脑与 LLM 的结构高度同构：两者都拥有强大的生成能力（ADHD 的直觉、创造力与 LLM 的语言生成），但缺乏可靠的执行调度层（ADHD 的执行功能缺陷与 LLM 的上下文长度限制、幻觉问题）。因此，**AI 工具对 ADHD 的帮助，本质上就是“给生成核心套 harness”——通过外部工具提供执行功能、记忆管理和上下文控制，弥补内在调度缺陷**（来源：ADHD 的 AI 工具全景）。

这一视角让 ADHD 学生和 Agent 工程师产生共鸣：你不是“懒”或“笨”，而是你的大脑架构需要更聪明的脚手架。

## 解决方案：将“外包精确计算”迁移到考试与作业场景

在 LLM/Agent 工程中，Agent 通过 function calling 把精确计算外包给工具：例如，让 LLM 调用计算器做算术，而不是自己硬算。ADHD 学生完全可以借鉴这一思路：**面对精确、重复、状态维护类任务，不要依赖大脑的“原生执行功能”，而是主动设计外部脚手架**。

### 1. 任务启动：用 Goblin Tools 分解压倒性任务

ADHD 学生面对复杂作业时，常常因为不知道从何下手而拖延。Goblin Tools 的 Magic ToDo 功能可以将模糊的任务（如“写一篇历史论文”）自动分解为具体步骤（如“确定主题”“收集资料”“写大纲”等），降低启动门槛（来源：Goblin Tools）。这相当于 Agent 中的任务规划器——将大目标拆解为可执行的子任务序列。

**ADHD 侧证据**：用户反馈称 Goblin Tools “将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools）。

**LLM 侧证据**：Agent 工程中，LLM 通过 ReAct 模式（思考-行动-观察）逐步分解复杂问题，每一步只关注当前子任务，避免上下文溢出。

### 2. 工作记忆：用 Saner.AI 减少“搜索循环”

ADHD 学生常陷入“搜索循环”——忘记刚才想到的点子，或找不到之前记下的笔记。Saner.AI 通过本地记忆和快速检索，让用户无需在多个标签页或应用间切换即可获取信息（来源：Saner.AI）。这相当于 Agent 中的外部记忆存储——LLM 将中间结果写入向量数据库，避免遗忘。

**ADHD 侧证据**：Saner.AI 被列为最佳 ADHD 生产力 AI 工具之一，其知识回忆功能直接针对“信息丢失”问题（来源：Saner.AI）。

**LLM 侧证据**：在 LangChain 等框架中，Agent 使用 Memory 模块保存对话历史或任务状态，确保多步推理的连续性。

### 3. 状态维护：用 Lex 的“单一指令触发多步骤”

考试中，ADHD 学生容易在答题步骤间分心，忘记下一步做什么。Lex 允许用户通过一条指令触发复杂、多步骤的任务序列，例如“检查这道代数题：先验证符号，再代入数值，最后核对单位”（来源：Lex）。这相当于 Agent 中的 Function Calling——LLM 调用一个封装好的函数执行多步操作，自己只负责发起和接收结果。

**ADHD 侧证据**：Lex 被设计为“实时适应大脑的工作方式”，降低认知负荷（来源：Lex）。

**LLM 侧证据**：在 OpenAI 的 Function Calling 中，LLM 将计算任务交给外部工具（如 Wolfram Alpha），自己专注于推理和决策。

## 脚手架 vs 拐杖：如何避免过度依赖？

虽然 AI 工具效果显著，但必须警惕“拐杖陷阱”。当前争议在于：**过度依赖 AI 工具可能阻碍内在执行功能发展，如何设计可逐步撤除的脚手架仍是挑战**（来源：矛盾与存疑）。例如，长期使用 Goblin Tools 的任务分解，可能让用户丧失自己分解任务的能力。

**我的判断**：脚手架与拐杖的边界在于“是否保留主动参与”。好的脚手架应该像训练轮——当用户能力提升后可以移除。例如，先用 Goblin Tools 分解任务，然后尝试自己列出步骤，最后只使用工具验证。同时，工具设计者应提供“撤除模式”，让用户逐步减少依赖。

**局限**：目前多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。此外，ADHD 异质性高，单一工具难以覆盖所有亚型（来源：ADHD 的 AI 工具全景）。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个让你拖延的任务**：打开网站，输入“写数学作业”或“整理书包”，观察 AI 分解的步骤，然后从第一步开始做。
2. **用 Saner.AI 记录一个考试知识点**：将关键公式或步骤存入 Saner.AI，考试前快速检索，减少“想不起”的焦虑。
3. **为考试设计一个“检查指令”**：在 Lex（或类似工具）中预设一条指令，如“检查计算题：1. 看符号 2. 看进位 3. 看单位”，考试时一键触发。
4. **反思依赖程度**：每周记录一次使用工具的时长，尝试在没有工具的情况下完成一个小任务，评估自己是否过度依赖。

## 结语

ADHD 学生不是“有缺陷的 LLM”，而是“未配 harness 的 Agent”。通过借鉴 Agent 工程的思路——将精确、重复、状态维护类任务外包给工具——你可以释放大脑的生成核心，专注于真正擅长的创造与直觉。记住：工具是脚手架，不是拐杖。用得好，它能帮你走得更远。

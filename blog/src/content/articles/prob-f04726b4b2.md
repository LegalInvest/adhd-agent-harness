# 为什么用 Mem 治 ADHD 的想理解自己的大脑，和给 agent 套 生成核心 + 缺失的执行层 是一回事？

> Mem 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

有没有一种痛苦：你花了无数小时想“理解自己的大脑”，读完了多巴胺、执行功能、注意力机制，脑子里却还在拖延同一件事？你不是缺乏洞察力，而是缺少把洞察力转换成行动的下一层。ADHD 的“想理解自己的大脑”，和工程师今天给 LLM 做 Agentic Harness 的困境，其实是同一个问题：一个强大的生成核心，配了一个残缺的执行调度层。

## 1. 同构：两边都是“生成核心 + 缺失的执行层”

ADHD 大脑常常是高性能的生成器。它跳跃、联想、创造、建系统，对新鲜的直觉极强。但前额叶执行网络激活不足，让它在需要调度时掉链子（来源：ADHD × AI 的科学与研究前沿）。具体表现为：工作记忆不稳定、任务启动困难、时间感知失真（时间盲）、注意力被环境牵着走、多巴胺系统难以把动机兑现成行动（来源：多巴胺；任务启动）。

LLM 也是类似的结构：它在生成文本、推理、概念联想上惊人地强，但本质上无状态、没有内置的任务调度层、长上下文会退化、容易被提示带偏（来源：ADHD × AI 的科学与研究前沿）。它没有跨会话的连续性，也不会自己管理时间，更不会主动把宏大的目标拆成可执行的步骤。

所以，ADHD 的痛苦并不是“脑子不好”，而是“脑子好，但缺个操作系统”。LLM 也一样。它们都是同一个物种：高产但缺执行调度层的生成核心。

## 2. 同一把扳手：给生成核心套 harness

既然问题同构，解法也同构：不是去修“生成核心”，而是在外面套一个执行层（harness / 脚手架）。这个脚手架的核心功能只有几样：外部记忆、任务分解、自动调度、上下文控制和约束机制。

在 ADHD 侧，这套工具已经浮现。Goblin Tools 的 Magic ToDo 能把“清理厨房”这种模糊任务拆成可执行的子步骤，并用“辣度”控制粒度，直接降低任务启动阻力（来源：Goblin Tools）。Motion 自动排程、动态重组日程，减少“下一步该做什么”的决策消耗（来源：Motion）。Saner.AI 则通过本地记忆、通用收件箱和知识回忆，减少 ADHD 用户的标签切换和搜索循环（来源：Saner.AI）。这些工具都在做同一件事：把本应由大脑执行功能维持的上下文、计划、节奏，外部化给机器。

在 LLM/agent 侧，工程师做的是同样的结构：Chain-of-Thought 让模型把思考显式化；ReAct 把推理与行动解耦；外部记忆模块提供跨会话的连续性；检索增强让模型重新“

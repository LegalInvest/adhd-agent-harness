# ADHD 自由职业者视角：为什么「治好 ADHD 的执行功能（计划/组织/启动）先天偏弱」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 的编排/记忆/工具层就是「外部执行功能」——同一套 harness 思路，两边都成立。

## 问题：为什么你一边写代码一边刷了 20 个标签页，而你的 agent 也在跑飞？

你是一个 ADHD 自由职业者。早晨打开电脑，脑子里同时蹦出三个项目、五封邮件、和一个“该换猫砂了”的念头。你打开任务管理器，却发现自己已经在刷 YouTube。这不是你懒——是你的执行功能（计划、组织、启动）先天偏弱（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

与此同时，你是个做 agent 工程的开发者。你精心设计了一个 LLM agent，结果它在对话中途突然开始编造上下文，或者把任务链断在中间。你给它加了一堆工具和记忆模块，它还是像脱缰的野马。这也不是模型笨——是 LLM 天生无状态，缺乏编排层来控制自己的输出流（来源：ADHD × AI 的科学与研究前沿）。

这两个问题看起来风马牛不相及，但如果你仔细看它们的结构，会发现它们其实是同一道工程题：**你有一个强大的生成核心，但它缺少一个内建的执行调度层。** ADHD 大脑和 LLM/agent 是同构的——都是高产但缺调度层的生成核心。而解法也是一样的：给这个核心套上一个外部执行功能层（harness）。

## 同构：ADHD 大脑和 LLM 共享的缺失层

### ADHD 侧：执行功能是大脑的“驾驶座”

ADHD 的核心障碍是执行功能缺陷。执行功能是大脑的“驾驶座”，负责调控注意力、计划、组织、工作记忆、冲动控制（来源：执行功能）。对于 ADHD 患者，这个驾驶座常常“无人驾驶”——你无法可靠地启动任务、维持上下文、或按计划推进。

具体来说，ADHD 的工作记忆常常成为瓶颈：你无法在脑中短暂保持信息以完成操作（来源：工作记忆）。时间盲让你无法感知时间流逝，导致计划困难（来源：时间盲）。任务启动困难让你面对一个“整理房间”的指令就僵住。这些不是智力问题——ADHD 大脑的生成能力（联想、创意、问题解决）往往很强，但缺乏调度这些能力的执行层。

### LLM/agent 侧：同样缺失调度层

LLM 和 agent 也有类似的缺陷。LLM 是无状态的——它没有内建的工作记忆，每次对话都是全新的开始（来源：ADHD × AI 的科学与研究前沿）。它的上下文窗口有限，一旦超过长度，就会“丢失”前面的信息（来源：ADHD × AI 的科学与研究前沿）。它没有内建的计划能力——如果你不给它一个明确的步骤链，它可能会跳过关键步骤或重复自己。这些缺陷使得 LLM 在复杂任务中容易“跑飞”，就像 ADHD 大脑在多重任务中容易迷失。

### 同构的核心：生成能力强，调度弱

两边共享一个核心特征：**生成能力（智力/算力）强大，但缺乏内建的调度机制来可靠地调用这些能力。** ADHD 大脑的生成能力（联想、创意、问题解决）往往很强，但缺乏调度这些能力的执行层。LLM 的生成能力（语言模型）也很强，但缺乏编排层来控制输出流。因此，两边的解法在结构上是同构的：都需要一个外部执行功能层来提供记忆、任务启动、时间管理等功能。

## 外部执行功能层：同一个 harness 思路

### ADHD 侧：AI 工具作为外部脚手架

对于 ADHD 用户，AI 工具可以充当外部执行功能层。例如：

- **Goblin Tools** 的 Magic ToDo 功能能自动将任何任务分解为可管理的小步骤，降低启动门槛（来源：Goblin Tools）。用户反馈称它“将压倒性的事情变成一系列不压倒性的事情”（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。
- **Saner.AI** 通过知识回忆和本地记忆，帮助用户快速找回信息，减少搜索循环和标签切换（来源：Saner.AI）。
- **Motion** 和 **Reclaim.ai** 能自动根据任务、会议和截止日期创建并动态调整每日计划，消除“下一步该做什么”的决策负担（来源：Motion）。

这些工具的本质是：**将内在缺失的执行功能外化**。AI 作为数字工作记忆，当用户自身工作记忆失效时，AI 能持续承载上下文（来源：工作记忆）。AI 的时间追踪和倒计时功能提供外部时间感知，弥补时间盲（来源：时间盲）。AI 的任务分解功能降低启动焦虑，利用小步骤的即时反馈维持注意力（来源：Goblin Tools）。

### LLM/agent 侧：编排层作为外部执行功能

对于 LLM/agent，外部执行功能层就是 agent 的编排、记忆和工具层。例如：

- **记忆模块**：通过向量数据库或对话摘要，为 LLM 提供持久上下文，弥补无状态性（来源：上下文工程）。
- **任务分解**：通过 prompt 工程或 agent 框架，将复杂任务拆解为子步骤，并确保每个步骤的输出被正确传递（来源：上下文工程）。
- **调度与重试**：通过编排层监控 agent 的行为，在它跑飞时中断、回退或重新规划（来源：ADHD × AI 的科学与研究前沿）。

这些组件和 ADHD 工具的结构惊人相似：记忆模块对应数字工作记忆，任务分解对应 Goblin Tools 的 Magic ToDo，调度对应 Motion 的动态排程。两边都在做同一件事：**为生成核心提供一个可靠的外部调度层。**

## 脚手架 vs 拐杖：关键边界

然而，同构并不意味着盲目复制。一个关键争议是：**这些外部工具是脚手架（可撤除的临时支撑）还是拐杖（不可撤除的永久依赖）？**（来源：拐杖与脚手架）

在 ADHD 侧，长期使用 AI 工具是否会削弱内在执行功能？目前缺乏纵向研究（来源：矛盾与存疑）。一些工具（如 Inflow）声称通过神经科学原理改善执行功能，但证据不足（来源：矛盾与存疑）。同样，在 LLM/agent 侧，过度依赖外部编排层是否会让模型本身变得更“懒”？目前也没有结论。

我的判断是：**脚手架与拐杖的边界在于设计意图。** 如果工具被设计为逐步撤除（例如，先自动分解任务，再逐渐让用户自己分解），它就是脚手架。如果工具被设计为永久替代（例如，永远帮你规划日程），它就是拐杖。目前大多数 ADHD 工具没有撤除机制，这是它们的局限（来源：矛盾与存疑）。同样，大多数 agent 框架也没有内置“训练模型自己调度”的机制。**未来的方向应该是设计可逐步撤除的脚手架，而不是永久拐杖。**

4. **反思你的 AI 工具是脚手架还是拐杖**：问自己：如果明天没有这个工具，我还能完成同样的事吗？如果不能，考虑逐步减少依赖，或者寻找有撤除机制的工具。

## 诚实面对局限

这个同构模型目前主要基于理论类比和工具案例，缺乏大规模实证（来源：矛盾与存疑）。神经影像学和行为实验尚未直接验证 ADHD 大脑与 LLM 的调度缺陷是否在神经层面同构。此外，多巴胺干预的有效性存在争议（来源：矛盾与存疑），而 AI 工具的长期效果（包括依赖风险）缺乏纵向研究。但即使作为工作模型，这个同构视角已经能帮我们设计更有效的工具——无论是为 ADHD 大脑，还是为 LLM/agent。

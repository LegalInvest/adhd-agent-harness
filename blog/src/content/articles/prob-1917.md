# ADHD 自由职业者视角：为什么「治好 ADHD 的执行功能（计划/组织/启动）先天偏弱」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 的编排/记忆/工具层就是「外部执行功能」——同一套 harness 思路，两边都成立。

## 从两个看似无关的痛点说起

作为 ADHD 自由职业者，你大概率经历过这样的场景：脑子里有无数绝妙想法，但坐在电脑前却无法启动任何一个任务；计划表写满却从未执行；时间在无意识中流失，直到 deadline 逼近才恐慌性冲刺。与此同时，如果你在搭建 LLM agent，你一定也遇到过：prompt 写得很细，但 agent 还是会跑飞；上下文窗口越塞越满，推理质量却断崖下跌；记忆模块存了又忘，工具调用混乱无序。

这两类痛苦看似毫无关联——一个来自神经发育障碍，一个来自 AI 工程。但如果你把 ADHD 大脑和 LLM 并排放置，会发现它们共享同一个核心问题：**生成能力极强，但缺乏可靠的执行调度层**。

## 同构：高产但无序的生成核心

ADHD 大脑的默认模式网络（DMN）过度活跃，不断产生新想法、新联想，但前额叶的执行控制功能偏弱，无法有效筛选、排序和启动行动（来源：执行功能）。这正是“想法多、行动少”的神经基础。最新研究甚至发现，Transformer 在 Stroop 冲突任务中，注意力随序列变长在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突，与 ADHD 执行功能缺陷的核心标志一一对应（来源：Deficient Executive Control in Transformer Attention）。另一项实证研究显示，LLM 呈现“强记忆、弱控制”剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷，无法灵活切换任务集、无法抑制自动化反应（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

这意味着：ADHD 大脑和 LLM 都是“高产但缺执行调度层的生成核心”。对于 ADHD，这个调度层是前额叶；对于 LLM，它是 prompt 工程、上下文管理和工具编排。当两者都失效时，结果一样：输出失控、任务搁浅、资源浪费。

## 外部执行功能层：同一套 harness 思路

既然内部调度层不可靠，解法就是构建外部执行功能层。这个概念在 ADHD 领域早已存在：任务分解、提醒系统、视觉计划表都是外部脚手架。而在 AI agent 工程中，编排层（orchestration）、记忆系统（memory）、工具调用（tool use）构成的正是 agent 的“外部执行功能”。

看几个具体工具：
- **Goblin Tools** 的 Magic ToDo 功能将模糊任务自动分解为可执行的小步骤，降低启动门槛（来源：Goblin Tools）。这相当于 agent 的 task decomposition——把“写一篇报告”拆成“收集资料→列大纲→写引言”等子任务。
- **Saner.AI** 通过本地记忆和知识回忆减少搜索循环和标签切换，直接补偿工作记忆不足（来源：Saner.AI）。这对应 agent 的 memory retrieval——不让上下文丢失导致重复劳动。
- **Motion** 自动根据优先级和截止时间动态调整日程，消除“下一步该做什么”的决策负担（来源：Motion）。这类似 agent 的 dynamic scheduling——根据实时状态重排任务队列。

这些工具的本质，都是为生成核心套上一个外部 harness：把“计划”从大脑移到系统，把“组织”变成自动化流程，把“启动”简化为一个按键。工程师在做 agent 编排时，做的也是同样的事：用 chain/agent 框架控制 LLM 的生成路径，用向量数据库存储长期记忆，用 tool schema 约束输出格式。

## 脚手架 vs 拐杖：必须明确的边界

然而，外部执行功能层是一把双刃剑。它可以是脚手架（scaffold），帮助用户或 agent 完成原本做不到的事；也可以是拐杖（crutch），让用户或 agent 永远依赖外部，内在能力进一步退化（来源：矛盾与存疑）。

对于 ADHD 个体，长期依赖任务分解工具是否会导致内在计划能力更弱？对于 agent，过度依赖 prompt 模板是否会让模型失去零样本推理的灵活性？目前缺乏纵向研究来回答这些问题（来源：矛盾与存疑）。但一个合理的判断是：**脚手架应该逐步“褪去”，而不是永久替代**。好的工具设计应该包含“渐隐”机制——比如 Motion 允许用户手动调整排程，Goblin Tools 可以自定义分解粒度，而不是一键全包。

## 争议与局限

必须诚实指出：同构视角目前仍是功能类比，缺乏神经生物学层面的直接证据（来源：矛盾与存疑）。LLM 的“注意力机制”与 ADHD 的注意控制缺陷是否真正同构，尚无定论。此外，ADHD 的异质性很大（注意力主导型 vs. 冲动主导型），不同 LLM 架构（如 MoE vs. dense）的失败模式也不同，同构的普适性需要更多研究（来源：矛盾与存疑）。

工具层面，Goblin Tools、Saner.AI、Motion 的有效性证据主要来自用户报告，缺乏独立随机对照试验（来源：Goblin Tools、Saner.AI、Motion）。过度依赖风险也未被充分研究。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个你拖延已久的任务**：输入“整理工作台”或“写周报”，看它输出的步骤是否让你觉得可以开始了。如果有效，把它纳入日常流程。
2. **在 agent 项目中显式添加“上下文压缩”步骤**：当 token 超过一定阈值时，用 LLM 自动总结历史对话并清空窗口，模拟 ADHD 的“重置”策略。
3. **对比 Motion 和你的手动排程**：用一周 Motion 自动排程，记录“决策疲劳”减少的程度。同时观察是否出现“完全交给工具”的依赖倾向，刻意保留一些手动调整。
4. **为你的 agent 写一个“启动检查清单”**：类似 ADHD 的外部提醒，在 agent 执行前自动检查“是否有足够上下文”“工具是否就绪”，减少跑飞概率。

## 结语

ADHD 自由职业者和 AI agent 工程师面对的，其实是同一道工程题：如何为一个高产但无序的生成核心，搭建可靠的外部执行调度层。工具不同，思路同构。理解这一点，你就能在两个领域之间迁移经验——用 agent 编排的思想优化自己的执行功能，用 ADHD 的应对策略改进 agent 设计。真正的解法不是修复内在缺陷，而是接受它，然后为它建造一个更好的 harness。

# ADHD 自由职业者视角：为什么「治好 ADHD 的执行功能（计划/组织/启动）先天偏弱」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 的编排/记忆/工具层就是「外部执行功能」——同一套 harness 思路，两边都成立。

## 问题：两个世界的同一道坎

如果你是一名 ADHD 自由职业者，你一定熟悉这样的场景：头脑里塞满了 brilliant ideas，但就是无法启动、排序、收尾。你需要的不是更多灵感，而是一个能帮你把“想”变成“做”的外部系统。

如果你是一名在做 Agentic Harness 工程的开发者，你同样熟悉这样的场景：LLM 能生成惊艳的文本，但一旦脱离提示词的严格约束，就会跑飞、遗忘上下文、陷入循环。你需要的不是更强的模型，而是一个能帮它“计划、记忆、执行”的编排层。

这两个问题看似分属心理学和计算机科学，但它们共享同一道工程题：**如何为高产的生成核心套上可靠的外部执行调度层？**

## 同构：ADHD 大脑与 LLM 是同一类“生成核心”

ADHD 的核心障碍是执行功能缺陷——计划、组织、工作记忆、任务启动困难（来源：执行功能）。最新研究发现，LLM 呈现“强记忆、弱控制”剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷，无法灵活切换任务集、无法抑制自动化反应——这正是 ADHD 的经典神经心理模式（来源：Strong Memory, Weak Control）。

更惊人的是，在经典 Stroop 冲突任务中，Transformer 注意力随序列变长在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突，这与 ADHD 执行功能缺陷的核心标志一一对应（来源：Deficient Executive Control in Transformer Attention）。

这意味着：ADHD 大脑和 LLM 都是“高产但缺调度层的生成核心”。瓶颈不在算力或创造力，而在 orchestration 层。

## 解法：外部执行功能层 = Agent Harness

### ADHD 侧的案例：曾国藩的“日课十二条”

晚清名臣曾国藩就是典型的 ADHD 大脑：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书慢到“他人目下二三行，余或疾读不能终一行”。他的解法不是靠意志力硬扛，而是设计了一套外部执行功能系统——**日课十二条**：黎明即起、读书不二、谨言、写日记反省……他称之为“结硬寨打呆仗”，用最笨最稳的方法抵消自己的冲动。

这套系统本质上就是一个人工编排层：每日定时 re-grounding（日课）、外部记忆（日记）、任务分解（结硬寨）。它与 LLM agent 的 harness 完全同构：定时 re-grounding ↔ 系统提示重置，外部记忆 ↔ 向量数据库，任务分解 ↔ 子任务编排。

### LLM/Agent 侧的对应：AI 工具作为外部执行功能层

今天的 AI 工具已经在实践同一思路：

- **Goblin Tools** 的 Magic ToDo 能将模糊任务（如“清理厨房”）自动分解为具体子步骤，用户可调节“辣度”控制粒度（来源：Goblin Tools）。这正是为 ADHD 大脑提供“任务分解编排层”。
- **Saner.AI** 通过本地记忆存储和快速检索，减少搜索循环和标签切换，充当“外部工作记忆”（来源：Saner.AI）。
- **Motion** 自动排程并动态调整日程，消除“下一步该做什么”的决策负担，直接补偿时间盲和任务启动困难（来源：Motion）。

这些工具都在做同一件事：**为生成核心（ADHD 大脑或 LLM）套上外部执行功能层**。

## 核心观点：脚手架 vs 拐杖的边界

我的核心判断是：**ADHD 大脑与 LLM 的同构不仅是理论类比，更是一个可操作的工程框架**。它意味着我们可以把 ADHD 自我管理工具的设计，直接映射为 agent harness 的设计——反之亦然。

但必须诚实指出局限：当前证据多为概念类比和初步实验，缺乏大规模随机对照试验（来源：矛盾与存疑）。更关键的是，外部执行功能层有成为“永久拐杖”的风险——如果用户从未内化任何策略，一旦工具失效，能力可能退化（来源：拐杖与脚手架）。

边界在于：脚手架的设计应逐步培养用户的执行功能，而非完全替代。就像曾国藩的日课，最终内化为习惯，而非永远依赖外部提醒。

## 今天就能试的行动

1. **任务分解实验**：用 Goblin Tools 的 Magic ToDo 将一个你拖延已久的任务（如“整理财务”）分解到最小可执行步骤，感受“启动门槛降低”的效果。
2. **外部记忆搭建**：用 Saner.AI 或类似工具建立一个“第二大脑”，把所有临时想法、待办、笔记都存入一个可检索的收件箱，减少工作记忆负担。
3. **自动排程测试**：试用 Motion 或类似 AI 日程工具，将一周的任务和会议输入，观察它如何自动安排优先级，并记录你“决策疲劳”的变化。
4. **设计你的“日课”**：模仿曾国藩，为自己设计 3-5 条每日固定流程（如“早上一杯咖啡后写三件要事”），并设置定时提醒作为 re-grounding 信号。

## 结语

ADHD 自由职业者和 Agent 工程师面对的，是同一道工程题：如何为强大的生成核心配上一个可靠的执行调度层。当你下次抱怨“大脑跑飞”或“LLM 跑飞”时，不妨想想——也许你需要的不是更强大的核心，而是一个更好的 harness。

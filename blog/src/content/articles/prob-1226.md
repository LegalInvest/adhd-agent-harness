# 如果把 ADHD 大脑当成生成核心，Saner.AI 能不能当它的外部执行功能层？

## 问题：当生成核心没有调度层

你是否有过这样的瞬间：脑子里同时冒出十几个想法，每一个都精彩，但就是不知道先做哪个；明明计划好一天的任务，坐下来却刷了半小时手机；开会时灵感迸发，散会后却连刚才的思路都记不全。这不是意志力问题，而是执行功能——计划、组织、启动、工作记忆——先天偏弱的表现。ADHD 大脑就像一个高产的生成模型，但缺少可靠的内置调度层。

与此同时，在 AI 工程领域，工程师们正在解决一个惊人相似的问题：LLM 本身是无状态的、缺乏上下文管理的生成核心，需要 agent scaffolding（编排、记忆、工具层）来让它可靠地执行任务。两边的解法，结构同构。

## 同构：ADHD 大脑与 LLM 共享的架构缺陷

ADHD 大脑与 LLM 在系统架构上存在深层对应：**两者都是“高产但缺执行调度层的生成核心”**（来源：ADHD × AI 的科学与研究前沿）。ADHD 大脑的发散思维、联想能力与 LLM 的文本生成能力类似，但都缺乏可靠的内置执行调度层。ADHD 的执行功能缺陷（工作记忆不稳定、任务启动困难、注意力易被带偏）与 LLM 的无状态性、上下文膨胀、推理退化形成镜像。

具体来说，工作记忆层面：ADHD 大脑的工作记忆容量有限且不稳定，导致跨会话任务难以持续，每天需要重新回忆项目上下文（来源：无状态与外部记忆）。LLM 调用同样是无状态的，每次对话独立，不保留跨会话记忆（来源：无状态与外部记忆）。两者都需要外部记忆系统来补偿——ADHD 需要“第二大脑”工具，LLM 需要向量数据库或记忆模块。

注意力管理层面：ADHD 大脑常被环境带偏，时间感知扭曲（来源：上下文工程）。LLM agent 同样面临上下文膨胀导致推理退化的问题（来源：上下文工程）。两者都需要外部框架来结构化执行流程，主动管理“当下注意什么”。

## 人物案例：曾国藩的日课与邓小平的“不争论”

历史人物的自我管理系统，本质上就是手工打造的 agent harness。曾国藩是典型注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书慢到“他人目下二三行，余或疾读不能终一行”。他的解法是**日课十二条**：黎明即起、读书不二、谨言、写日记反省等。这套系统相当于一个定时 re-grounding 的外部调度器——每天固定时间强制重置注意力，用最笨最稳的方法抵消冲动。这正对应 LLM harness 中的定期上下文刷新和状态检查。

邓小平的“不争论，看准了就干”和“用制度管党管国家”，则是一个高效的外部决策框架。他承认自己坐不住（80 多岁还在南方考察），但通过“实践是检验真理的唯一标准”这一元规则，避免了无休止的优先级争论。这对应 agent harness 中的硬编码决策规则：当模型陷入决策循环时，直接用外部规则打断。

## Saner.AI：作为外部执行功能层的可能性

回到核心问题：Saner.AI 能不能当 ADHD 大脑的外部执行功能层？从架构上看，它具备关键要素：

**外部记忆**：Saner.AI 专注于知识回忆和本地记忆，旨在减少搜索循环和标签切换（来源：Saner.AI）。它提供 AI 笔记捕获、自动组织任务、脑暴工作流、每日规划（来源：Saner.AI）。这直接补偿 ADHD 工作记忆缺陷——用户无需每天重新回忆项目上下文，系统会持久存储客户历史、项目背景等信息（来源：无状态与外部记忆）。

**任务启动辅助**：Saner.AI 的通用收件箱可从链接的邮件、文档或日历中提取待办事项，利用内部助手检查截止日期、将大型项目分解为小里程碑（来源：Saner.AI）。这对应 Goblin Tools 的 Magic ToDo 任务分解功能（来源：Goblin Tools），降低启动门槛。

**上下文管理**：通过本地记忆存储和快速检索，用户无需在多个标签页或应用间切换即可获取所需信息（来源：Saner.AI），减少被环境带偏的风险。这对应 Motion 的自动排程与动态调整（来源：Motion），但 Saner.AI 更侧重记忆而非日程。

然而，Saner.AI 并非完整的 harness。它缺少类似 Routinery 的步骤引导（来源：上下文工程），也缺少 Motion 的动态优先级再计算。它更像一个强大的外部记忆层+基础任务组织，但调度逻辑仍需用户自己或与其他工具组合。

# Mem 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Mem：它到底补上了哪一层执行功能？

你有没有过这样的体验：打开电脑，本打算写一份报告，结果两小时后发现自己正在研究“如何养一只水母”？或者，你是一名 AI 工程师，调试一个长程 agent 时发现它在第 20 轮交互后开始胡言乱语，完全忘了最初的指令？

这不是巧合。ADHD 大脑与 LLM agent 共享同一个深层结构：**两者都是高产但缺执行调度层的生成核心**。ADHD 的创意发散与 LLM 的文本生成能力惊人相似，但都缺乏一个可靠的调度层来锚定目标、防止漂移。而 Mem——这个被很多人当作“第二大脑”的 AI 笔记工具——如果只被用来记笔记，那就大错特错了。它的真正价值，在于作为**重锚定 system prompt**，像给 LLM 加上 harness 一样，给 ADHD 大脑装上外部执行功能层。

## 同构的困境：目标漂移与指令消退

ADHD 一侧的痛点再熟悉不过：分心走神，或超聚焦跑偏。你可能是被一条通知拉走，也可能是被某个细节迷住，一抬头三小时过去了。这就是**目标漂移**——注意力从主任务滑向无关刺激。wiki 资料指出，ADHD 大脑“极易发生目标漂移：注意力被无关刺激捕获（分心），或过度聚焦于细节而忽略整体目标（超聚焦）”（来源：重锚定与目标漂移）。

LLM/agent 一侧呢？工程师们发现，长程 agent 在交互后期会“开始忘记系统提示”（来源：重锚定与目标漂移），导致行为偏离初始指令。这被称为**指令消退**，本质上是模型上下文窗口膨胀后，早期指令被稀释。Harness 工程通过“事件驱动的系统提醒”来对抗这种漂移（来源：重锚定与目标漂移）。

两边的问题同构：都是**生成核心缺乏外部锚点**，导致输出随时间偏离目标。

## Mem 的正确用法：重锚定，而非笔记

大多数人对 Mem 的理解停留在“AI 笔记工具”——自动整理、智能检索。但这只是表象。Mem 的核心能力是**将碎片信息与目标关联，并在你需要的时候主动推送回来**。这恰恰是重锚定 system prompt 的机制：不是被动存储，而是主动介入。

ADHD 用户需要的是：当你在写报告时走神去研究水母，Mem 能弹出一条你之前写的“本周目标：完成报告初稿”。这不是提醒，而是**重锚定**——把你的注意力拉回主任务。LLM agent 需要的是：当上下文膨胀后，harness 在关键节点重新注入原始 system prompt，防止模型跑偏。

但很多人用错了。他们把 Mem 当作收藏夹，囤积大量笔记，却从不设置目标关联或定时回顾。结果就是：Mem 变成了另一个信息垃圾场，而非执行调度层。wiki 资料中提到的“重锚定系统”正是对抗目标漂移的关键：“外部工具通过提醒和通知提供重锚定，将注意力拉回原任务”（来源：重锚定与目标漂移）。Mem 如果只记录不提醒，就只是半个工具。

## 证据与争议：脚手架还是拐杖？

现有工具的证据强度不一。Focusmate 利用身体在场效应，通过实时同伴问责提升任务启动和维持，有用户研究支持（来源：Focusmate）。RescueTime 通过自动时间追踪改善时间盲，有初步研究支持（来源：RescueTime）。Brain.fm 利用神经锁相技术，但针对 ADHD 缺乏独立临床研究（来源：Brain.fm）。

但更重要的是争议：**AI 工具作为外部执行功能层，长期使用是否会导致内在能力退化？** wiki 资料明确警告：“过度依赖 AI 工具可能削弱内在能力”（来源：矛盾与存疑）。这指向**脚手架 vs 拐杖**的边界：脚手架是暂时支撑，帮你学会自己走路；拐杖则是永久替代，让你肌肉萎缩。

我的判断是：Mem 作为重锚定工具，如果使用得当——即**主动设置目标关联和定期回顾**——它是脚手架。它强迫你明确“我在做什么”，然后在你偏离时拉回。但如果只是被动记录、从不回顾，它就是拐杖，甚至更糟：一个让你产生“我已经规划好了”的虚假安全感的工具。

## 今天就能试的行动

1. **给 Mem 设置一个“周目标”标签**：每周一写下本周最重要的 3 个目标，关联到 Mem 的“目标”标签。然后设置每日回顾提醒，让 Mem 每天早上推送这些目标。
2. **用 Mem 记录“漂移日志”**：每次发现自己走神，立刻在 Mem 里记下“我正在做什么”和“我应该做什么”。Mem 会自动关联，形成模式识别。一周后回顾，你会看到自己最常被什么带偏。
3. **模仿 LLM harness 的“事件驱动提醒”**：在 Mem 中为特定任务设置条件触发。例如，当你打开浏览器搜索“水母”时，Mem 弹出一条：“等等，你今天的首要任务是报告。”

## 局限与诚实

必须承认，同构是隐喻而非严格等价。LLM 的注意力机制与人类注意力有本质差异（来源：AI 与 ADHD 的专注力）。此外，Mem 的 AI 功能细节尚不明确，其重锚定效果依赖用户主动设置，而非自动化（来源：矛盾与存疑）。最后，个体差异巨大——对某些 ADHD 亚型，Mem 可能无效甚至增加认知负担。

但核心观点不变：**Mem 的正确用法不是笔记，而是重锚定 system prompt**。如果你把它当笔记工具，你只是给大脑加了一个外挂硬盘；如果你把它当调度层，你才是给生成核心套上了 harness。

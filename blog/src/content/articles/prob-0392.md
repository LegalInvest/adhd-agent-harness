# 如果把 ADHD 大脑当成生成核心，Goblin Tools 能不能当它的幻觉与验证循环？

> 用 agent 工程的思路重新理解 Goblin Tools 对 ADHD 的真正价值。

## 问题：ADHD 大脑的“幻觉”困境

你是否有过这样的经历：脑子里瞬间冒出一个绝妙想法，激动得恨不得立刻行动，但下一秒就发现漏洞百出，或者干脆忘得一干二净？或者，你在会议上脱口而出一个结论，事后才发现自己根本没考虑全？这不是你不够聪明，而是你的大脑——一个高产生成核心——在缺乏验证循环时，直接输出了未经验证的“幻觉”。

ADHD 大脑的生成核心（发散思维、联想能力）产出极高，但执行功能（任务启动、冲动控制、工作记忆）薄弱，导致“高产但不可靠”（来源：ADHD × AI 的科学与研究前沿）。这像极了 LLM：模型本身能流畅生成文本，但会自信地编造事实（幻觉），需要 agent harness 通过验证、自我批判和人在回路来兜底（来源：人在回路与身体在场）。

那么，如果把 ADHD 大脑当成生成核心，Goblin Tools 这类 AI 工具，能不能充当它的幻觉与验证循环？

## 同构：生成核心与外部验证循环

核心论点是：ADHD 大脑与 LLM 在结构上同构——两者都是“高产但缺执行调度层的生成核心”。因此，AI 帮助 ADHD 的本质不是“修复缺陷”，而是“给生成核心套上 harness”（外部脚手架）（来源：ADHD × AI 的科学与研究前沿）。

从 ADHD 侧看，执行功能缺陷表现为工作记忆不足、时间盲和任务启动困难。Goblin Tools 的 Magic ToDo 功能能自动将模糊任务（如“清理厨房”）分解为具体子任务，用户可调节“辣度”控制粒度（来源：Goblin Tools）。这相当于一个外部验证循环：AI 将你的意图分解为可验证的小步骤，每完成一步就获得反馈，从而抑制冲动决策的“幻觉”。

从 LLM/agent 侧看，agent harness 通过形式化规划与护栏（token 预算、工具调用上限、人工检查点）使代理输出真正有用且正确（来源：人在回路与身体在场）。Goblin Tools 的任务分解、时间估计和语气检查器，正是 ADHD 大脑的“护栏”：它们不是替代你的思考，而是将你的生成输出（任务意图、沟通内容）拆解为可验证的单元，让你在执行中不断校准。

## 证据：真实人物的 harness 系统

历史上，毛泽东和张一鸣都展现了 ADHD 特质，并建立了自己的 harness 系统。

毛泽东小时候是问题儿童，爱读闲书，一辈子在调查研究，思维极度跳跃，冲动敢赌（四渡赤水、抗美援朝）。他的 harness 是“批评与自我批评”“调查研究”“民主集中制”——用集体决策制衡冲动，用外部验证循环确保结论可靠。这对应 LLM agent harness 中的“人在回路”和“人工检查点”：定期向他人报告进度（类似 Focusmate 的身体在场效应），用外部视角验证自己的判断（来源：人在回路与身体在场）。

张一鸣的 harness 是“延迟满足感”和“OKR 管理系统”。他通过系统和文化管理公司，不搞集权，用 Context not Control 原则让组织自我验证。这对应 agent harness 中的“形式化规划与护栏”：OKR 将目标分解为可验证的关键结果，就像 Goblin Tools 将任务分解为子步骤。

这些 harness 的共同点是：它们不是消除生成核心的冲动，而是为冲动提供验证循环。Goblin Tools 正是这种思路的数字化版本。

## 争议与局限：脚手架 vs 拐杖

然而，Goblin Tools 能否真正充当验证循环，取决于它被用作“脚手架”还是“拐杖”。

脚手架是可拆卸的：你用它学习如何分解任务，逐渐内化这种能力。拐杖是永久的：你依赖它，一旦没有就无法行动。当前证据显示，Goblin Tools 短期有效，但长期依赖风险缺乏实证（来源：矛盾与存疑）。过度依赖可能导致执行功能进一步退化。

另一个争议是：AI 工具是否真的减轻认知负荷？部分用户反映学习使用 AI 工具本身增加认知负荷（来源：矛盾与存疑）。对于 ADHD 用户，新工具的设置过程可能成为另一个任务启动障碍。

此外，Goblin Tools 缺乏独立临床证据，其效果依赖个体差异（来源：矛盾与存疑）。对某些人，任务分解是救星；对另一些人，它可能过于机械，打断超聚焦。

## 核心判断：Goblin Tools 是验证循环的雏形，而非完整 harness

我的判断是：Goblin Tools 是验证循环的雏形，而非完整 harness。它提供了任务分解（验证意图）、时间估计（验证计划）、语气检查（验证沟通），但缺少两个关键组件：

1. **外部记忆**：ADHD 大脑的工作记忆缺陷与 LLM 的无状态性同构，都需要外部记忆系统（如第二大脑工具或向量数据库）来保留跨会话上下文（来源：ADHD × AI 的科学与研究前沿）。Goblin Tools 没有持久记忆，每次使用都是“无状态”的。

2. **人在回路**：最有效的 ADHD 策略是身体在场效应——有他人在场时执行功能被绕过（来源：人在回路与身体在场）。Goblin Tools 是单机工具，缺乏社交 accountability。相比之下，Saner.AI 通过本地记忆减少搜索循环，Motion 通过自动排程消除决策负担，两者更接近完整 harness。

因此，Goblin Tools 最适合作为“冲动验证器”：当你有一个模糊想法或冲动结论时，用它快速分解、检查可行性。但它不能替代长期规划或社交 accountability。

1. **冲动验证**：下次你有一个“必须立刻做”的任务时，先打开 Goblin Tools Magic ToDo，输入任务，调节辣度到 3，看分解出的步骤是否合理。如果发现某个步骤你其实不想做，说明你的冲动需要重新评估。

2. **沟通校准**：在发送重要邮件或消息前，用 Goblin Tools 的语气检查器分析语气。如果结果显示“愤怒”或“讽刺”，考虑修改——这相当于 LLM agent 的“输出护栏”。

## 结语

ADHD 大脑不是缺陷，而是一个高产生成核心。Goblin Tools 不是修复它的药，而是它的验证循环——一个轻量级的 agent harness。但正如毛泽东的“批评与自我批评”需要集体参与，张一鸣的 OKR 需要组织对齐，真正的 harness 需要外部记忆和人在回路。Goblin Tools 是起点，不是终点。

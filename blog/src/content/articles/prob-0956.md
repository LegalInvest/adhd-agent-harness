# 如果把 ADHD 大脑当成生成核心，Motion 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Motion 对 ADHD 的真正价值。

如果你是一个 ADHD 患者，你可能熟悉这样的场景：你打开电脑准备写报告，却发现自己两小时后在浏览维基百科上关于企鹅迁徙的页面，而报告一字未动。如果你是一个构建 AI agent 的工程师，你可能熟悉这样的场景：你部署了一个长程任务 agent，它在前几步表现完美，但十步之后，它开始生成与初始目标无关的内容——目标漂移了。

这两个场景看似无关，但本质上是同一个问题：**一个强大的生成核心，缺乏可靠的执行调度层**。ADHD 大脑与 LLM 在结构上同构：两者都是“高产但缺乏可靠执行调度层的生成核心”（来源：AI 与 ADHD 的专注力）。因此，为 ADHD 设计工具，本质上就是为这个生成核心搭建 harness（外部执行功能层）。而 Motion——这个号称“AI 日历”的工具——可能正是那个 harness。

## 问题：目标漂移与重锚定

ADHD 的典型困境不是“不能专注”，而是“专注错了地方”。ADHD 大脑的特征是“高产但无序的生成能力，同时缺乏可靠的执行调度层”（来源：ADHD 大脑与 LLM 的同构）。当注意力被外部刺激或内部思绪捕获，原始目标就被遗忘。更隐蔽的是“超聚焦跑偏”——当 ADHD 个体对某件事过度投入时，他们可能几小时不动，但做的却不是应该做的事。

在 LLM agent 的世界里，这被称为“目标漂移”。一个 agent 在执行长程任务时，如果缺乏有效的上下文管理和重锚定机制，它会逐渐偏离初始 system prompt，生成无关内容。工程师们因此构建了“agent harness”——“包裹 LLM 或 AI agent 的软件基础设施，处理模型本身之外的一切”（来源：ADHD 大脑与 LLM 的同构）。Harness 的核心功能之一，就是定期将 agent 拉回主轨道，重新锚定目标。

## Motion 作为重锚定系统

Motion 被宣传为“AI 日历”或“自动时间规划工具”，但它的真正价值不在于规划，而在于**重锚定**。Motion 的 AI 会持续评估你的任务优先级，并根据实际完成情况动态调整日程。当你偏离计划（比如在企鹅百科上花了 30 分钟），Motion 不会只是记录，而是主动重构剩余时间——把被浪费的时间重新分配给其他任务，并强制你面对新的优先级。

这恰恰是 ADHD 大脑最需要的：一个外部的、实时的“调度层”，在目标漂移发生时发出信号并重新定向。与 Focusmate 的“身体在场效应”不同——后者通过实时同伴问责提供外部结构和社会压力（来源：Focusmate）——Motion 提供的是**基于算法的时间重锚定**。它不依赖另一个人，而是依赖一个持续运行的调度引擎。

但这里有一个关键区别：Focusmate 的证据强度更高。多项用户研究显示 Focusmate 能显著提升任务启动和维持（来源：AI 与 ADHD 的专注力），而 Motion 的类似证据尚不充分。Motion 的算法是否真的能有效对抗 ADHD 的时间盲和执行功能缺陷，还需要更多独立研究。

## 同构的工程价值

从 agent 工程的角度看，Motion 的结构与一个典型 agent harness 惊人相似。一个 agent harness 包含“模型连接器”、“工作流绑定”、“上下文管理”等组件（来源：ADHD 大脑与 LLM 的同构）。Motion 的架构同样如此：它有一个“优先级评估器”（类似模型连接器，决定当前该做什么）、一个“日程重构器”（类似工作流绑定，动态调整任务序列）、以及一个“时间追踪器”（类似上下文管理，记录已用时间）。

这意味着，如果你是一个工程师，你可以把 Motion 理解为**为人类大脑设计的 agent harness**。它的核心算法就是对抗目标漂移的 system prompt 重锚定。当你使用 Motion 时，你实际上是在运行一个复合 AI 系统：人类生成核心（你的 ADHD 大脑）+ Motion 调度层（外部执行功能）。

但同构不是等同。LLM 的“注意力机制”与人类注意力存在本质差异（来源：AI 与 ADHD 的专注力）。LLM 的注意力是计算性的，而人类的注意力受多巴胺、情绪、疲劳等生物因素影响。Motion 可以重排日程，但不能调节你的多巴胺水平或缓解焦虑。

## 脚手架还是拐杖？

这是最关键的争议。AI 工具作为“外部执行功能层”，长期使用是否会导致 ADHD 患者内在执行功能进一步退化？（来源：AI 与 ADHD 的专注力）Motion 的自动调度功能，如果使用过度，可能让你越来越依赖外部指令，而失去自主规划的能力。

RescueTime 提供了一个警示：它通过自动时间追踪减轻了工作记忆负担（来源：RescueTime），但过度关注时间数据可能引发焦虑。Motion 的主动重构也可能带来类似问题：如果算法频繁打断你的超聚焦状态，你可能反而失去深度工作的能力。

Brain.fm 的争议也值得注意：它利用 AI 生成音乐来影响大脑状态，但“其针对 ADHD 的效果缺乏独立临床研究支持”（来源：Brain.fm）。Motion 同样面临证据不足的问题——它的有效性主要来自用户报告，而非对照实验。

## 今天就能试的行动

1. **用 Motion 做一次“目标漂移审计”**：启用 Motion 的日程追踪功能，一周后查看它记录了多少次日程重构。每一次重构都对应一次注意力漂移。这会让你直观看到自己的漂移模式。
2. **结合 Focusmate 使用 Motion**：Motion 处理时间规划，Focusmate 处理任务启动。在 Motion 规划的时间段内，预约 Focusmate 时段，利用身体在场效应执行具体任务。
3. **设置“重锚定提醒”**：即使不用 Motion，你也可以手动设置每 25 分钟（番茄钟）的提醒，问自己“我现在在做的事，是我应该做的吗？”这本质上是手动版的 agent harness 重锚定。
4. **反思依赖边界**：每周花 5 分钟问自己：如果没有 Motion，我还能规划一天吗？如果答案是否定的，说明你可能已过度依赖。尝试每周有一天“无工具日”，锻炼内在执行功能。

Motion 不是万能药。它只是一个 harness——一个外部调度层。它不能治愈 ADHD，不能消除目标漂移，甚至可能让你更依赖外部指令。但如果你理解它的真正价值——不是时间管理，而是重锚定——它就能成为你对抗注意力漂移的工程化武器。

毕竟，无论是 ADHD 大脑还是 LLM agent，最需要的都不是更多的生成能力，而是**一个可靠的、能在你跑偏时把你拉回来的系统**。

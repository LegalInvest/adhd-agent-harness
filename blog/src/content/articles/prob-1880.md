# ADHD 学生如何用「给生成核心套 harness」的思路应对考试与作业里的分心走神或超聚焦跑偏，丢失目标？

> 把 LLM 工程里的「长程任务里 agent 会目标漂移，需要重锚定 system prompt」迁移到ADHD 学生的考试与作业场景，双域证据作答。

你坐在书桌前，面前摊着一份明天要交的作业。刚写了三行，大脑突然弹出一个念头：“上次那部电影的结局到底是什么？”——十分钟后，你已经在刷豆瓣影评了。或者相反，你一头扎进某个细节（比如把某个公式的推导写到完美），两小时后才发现，整个大题的方向都偏了。这是 ADHD 学生最熟悉的两种折磨：分心走神，以及超聚焦跑偏。

如果你是一名 Agent 工程师，你一定见过类似的一幕：你给一个 LLM agent 布置了一个长程任务——比如“写一份市场分析报告，包含数据收集、竞品对比、趋势预测”。前几步它做得不错，但到第三步它突然开始大谈某个竞品的创始人八卦，完全忘了原始目标。这叫“目标漂移”。

ADHD 大脑和 LLM agent，本质上是同一类系统：**高产的生成核心，但缺乏可靠的执行调度层**。ADHD 大脑不断产生想法、冲动、联想（如同 LLM 的 token 生成），但执行功能——工作记忆、任务启动、时间盲、上下文保持——常常掉线。LLM 同样擅长生成流畅的文本，但如果没有编排层（harness/scaffolding）来约束上下文、执行验证、回滚错误，它就会产生幻觉、偏离任务、重复工作。

那么，ADHD 学生能否借用 LLM 工程里的“harness”思路，给自己的大脑套一个外部调度层？答案是肯定的，而且已经有工具在实践这个原理。

## 重锚定：对抗目标漂移的核心机制

在 LLM agent 工程中，对抗目标漂移的关键是“重锚定”（re-anchoring）。一个复杂的 agent harness 会定期将 agent 的当前状态与原始 system prompt 进行比对，如果发现偏离，就强制回正。这通常通过“验证与 CI 集成”实现——每一步的输出都会经过格式检查、逻辑校验，甚至运行测试用例（来源：GitHub - ai-boost/awesome-harness-engineering）。

ADHD 学生的“目标漂移”同样需要重锚定。但大脑没有内置的“system prompt”，所以需要外部工具来充当这个锚。

### 证据一：RescueTime 作为“时间锚”

RescueTime 自动记录你在不同应用和网站上的时间花费，并生成可视化报告。它的 AI 核心是自动分类和模式识别，将活动标记为“高效”“中性”或“分心”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。对 ADHD 学生来说，这就是一个外部的时间锚：当你沉浸在某件事中，RescueTime 的实时反馈（比如弹窗提示“你已经刷了20分钟微博”）就像 agent harness 里的验证步骤，提醒你当前行为是否偏离了原始目标（学习）。它弥补了 ADHD 常见的时间盲——无法准确感知时间流逝（来源：The Role of Actigraphy in the Study of Sleep and Circadian Rhythms）。

### 证据二：Focusmate 作为“社会锚”

Focusmate 通过 AI 匹配虚拟同伴，进行视频会议共同工作，彼此监督进度（来源：AI Tools for Kids with ADHD: A Complete Guide for Parents...）。它的机制是“身体在场效应”——有人看着你时，你更不容易跑偏。这相当于一个实时的人类验证循环：你的同伴就是你的“system prompt”，当你开始走神，对方的存在（哪怕不说话）会把你拉回来。这在 LLM 工程中对应“沙盒”机制——agent 在受控环境中执行，每一步都有人（或系统）监控。Focusmate 的 AI 匹配算法优化了配对效率，但核心效果仍归因于身体在场（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...)）。

### 证据三：Brain.fm 作为“神经锚”

Brain.fm 利用 AI 生成音乐，通过神经锁相技术影响大脑信息处理方式，帮助用户进入专注状态（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。这相当于给生成核心设置一个稳定的“采样温度”——降低大脑的随机波动，让输出更可预测。LLM 工程中，低温度（temperature）会使输出更确定、更可重复，而高温度则增加创意和随机性（来源：AI Agent Systems: Architectures, Applications, and Evaluation）。ADHD 的表现波动（日内注意力差异）本质上是“温度”不稳定，Brain.fm 试图外部调节这个“温度”。

## 同构的边界：脚手架 vs 拐杖

这里必须诚实指出一个争议：这些工具是脚手架还是拐杖？脚手架应该可以逐步撤除，但现有工具（如 RescueTime、Focusmate）设计为长期使用，没有明确的撤除机制（来源：矛盾与存疑）。过度依赖外部锚定，可能阻碍内在执行功能的发展。就像 LLM 工程中，如果 harness 过于僵化，agent 会失去灵活性，无法应对意外情况。

另一个局限是证据缺口：Brain.fm 的神经锁相技术缺乏针对 ADHD 的独立临床研究（来源：Brain.fm）；RescueTime 和 Focusmate 的效果多基于用户主观报告（来源：矛盾与存疑）。ADHD 亚型（注意力缺陷 vs 多动冲动）对工具的反应也不同（来源：矛盾与存疑）。所以，这些方法并非万能，而是需要个体化尝试。

## 今天就能试的 4 条行动

1. **设置“重锚定闹钟”**：每 25 分钟用手机设一个闹钟，铃响时问自己：“我当前的任务是什么？我正在做的事是否与之相关？”——这就是你的个人验证循环。
2. **使用 RescueTime 自动追踪一天**：注册免费版，让它记录你的时间使用。晚上看报告，找出你最容易“漂移”的时段和网站。
3. **尝试一次 Focusmate 学习时段**：预约一个 50 分钟时段，和陌生人一起学习。注意：如果你对视频焦虑，可以先关掉摄像头，只保留音频。
4. **用 Brain.fm 的“专注”模式写作业**：戴上耳机，选择“Focus”分类，看看是否比平时更容易进入状态。

## 结论

ADHD 学生与 LLM agent 共享同一个核心矛盾：生成能力远大于调度能力。解决方案也同构——通过外部 harness 实现重锚定。RescueTime、Focusmate、Brain.fm 分别从时间、社会、神经三个维度提供锚点。但请记住：这些是脚手架，不是永久的拐杖。真正的目标不是永远依赖它们，而是通过它们，逐步训练自己内在的“执行调度层”。

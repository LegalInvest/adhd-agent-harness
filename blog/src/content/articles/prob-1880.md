# ADHD 学生如何用「给生成核心套 harness」的思路应对考试与作业里的分心走神或超聚焦跑偏，丢失目标？

想象你正坐在书桌前，面前摊开一份数学试卷。你告诉自己：先做前三道大题。五分钟后，你发现自己正盯着窗外的树叶发呆——分心走神。另一种情况：你被一道难题吸引，沉浸式地推导了四十分钟，完全忘了考试时间——超聚焦跑偏。这两种体验，ADHD 学生再熟悉不过。它们本质上是同一个问题：你的大脑（一个高产但缺乏可靠执行调度层的生成核心）在长程任务中发生了目标漂移。

同样的问题，也发生在 AI Agent 工程师的调试日志里。一个设计用来写代码的 LLM agent，在运行十分钟后，可能开始生成与任务无关的诗句，或者陷入某个子问题的无限优化——目标漂移。工程师的解法是：给 agent 套上一个 harness，通过 system prompt 的反复重锚定，确保它始终记得原始目标。

## 同构的底层逻辑

ADHD 大脑的「生成核心」体现在创意发散、联想丰富，但执行功能（工作记忆、时间感知、任务启动）薄弱，导致输出不稳定（来源：AI 与 ADHD 的专注力）。LLM 同样：文本生成能力强，但缺乏上下文控制，容易产生幻觉或偏离目标（来源：幻觉与验证循环）。两者都「高产但缺调度层」。

更具体地，ADHD 的工作记忆缺陷与 LLM 的无状态性同构——都需要外部记忆系统来维持连续性。ADHD 易被环境带偏，LLM 易被上下文膨胀干扰，都需要主动设计「当下注意什么」的工程方案（来源：AI 与 ADHD 的专注力）。

## 重锚定：从 system prompt 到外部 harness

在 LLM 工程中，harness 的核心组件包括：上下文传递、工具接口、规划工件、验证循环、记忆系统和沙盒（来源：幻觉与验证循环）。其中，验证与 CI 集成是关键——复杂的 harness 不会盲目执行工具，而是实现验证步骤，例如检查输出格式或运行测试用例（来源：幻觉与验证循环）。缺乏这些验证会导致「更多幻觉和重复工作」（来源：幻觉与验证循环）。

映射到 ADHD 场景，这就是一个「重锚定 system prompt」的过程。你的「system prompt」就是当前任务的核心目标（比如「完成前三道大题」），而 harness 就是一系列外部机制，定期将你拉回这个目标。

## 脚手架 vs 拐杖：边界在哪里？

这里必须诚实指出争议：AI 工具作为外部执行功能层，长期使用是否会导致内在能力退化？过度依赖 RescueTime 可能削弱内在时间感知能力（来源：矛盾与存疑）。关键是区分「脚手架」与「拐杖」：脚手架在你建造时提供支撑，完成后撤走；拐杖则永久替代。

Focusmate 的设计更接近脚手架——它不帮你分解任务，只提供外部问责，你仍需自己执行。而 RescueTime 若被用作唯一的时间感知来源，则可能成为拐杖。边界在于：工具是否允许你逐步内化其功能？例如，使用 RescueTime 一段时间后，你能否逐渐减少查看报告，转而依靠内在感觉？

## 今天就能试的行动

1. **设置「重锚定」闹钟**：考试或写作业时，每 15 分钟设一个闹钟。响铃时，花 10 秒问自己：「我现在在做的事，是当前最重要的目标吗？」——这就是你的 system prompt 重锚定。
2. **尝试一次 Focusmate 时段**：预约 25 分钟，和伙伴一起工作。注意观察：同伴的存在是否让你更容易启动和维持？如果是，这就是你的外部验证循环。
3. **用 RescueTime 记录一天**：不评判，只是看数据。找出你最容易漂移的时间段和活动。然后针对性地设计「锚点」——比如在那个时段用 Brain.fm 创造专注声音环境。
4. **写一个「目标卡片」**：在作业或考试前，用一句话写下核心目标，贴在显眼处。每次走神后，读一遍卡片——这是你的「system prompt 文件」。

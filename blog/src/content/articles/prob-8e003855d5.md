# 为什么用 Tiimo 治 ADHD 的不知哪些方法有用，和给 agent 套 human-in-the-loop 监督 是一回事？

> Tiimo 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

## 问题：为什么方法试了一堆，却更像在喂 agent？

很多 ADHD 成年人会经历同一种循环：先是下载一堆 app，试了番茄钟、Notion、子弹日记、AI 助手，过几天又回到“不知道下一步该做什么”的状态。另一边，做 agent 工程的团队也熟悉另一种循环：给 LLM 接了 RAG、加了 system prompt、做了 safety layer，一放到真实业务里，它还是在关键步骤上“自由发挥”。

两个场景看起来毫不相干，但核心问题惊人地一致：**生成核心本身并不缺点子，缺的是在执行过程中持续被拉回现实的调度层**。对 ADHD 来说，这个调度层是外部执行功能；对 agent 来说，这个调度层就是 human-in-the-loop 与外部 orchestration。用 Tiimo 治 ADHD 的“不知哪些方法有用”，和给 agent 套 human-in-the-loop 监督，其实是同一套 harness 思路在人体和机器两侧的对称应用。

## 同构：两个“强记忆、弱控制”的系统

ADHD 常被看作执行功能障碍，其中**工作记忆是最关键的缺陷之一**——患者难以临时保存任务细节、容易丢失上下文（来源：ADHD, Executive Functions, and AI: A New Era in Treatment | Psychology Today）。无独有偶，明尼苏达大学对 LLM 执行功能的系统测试发现，LLM 呈现“强记忆、弱控制”的剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷，无法灵活切换任务集，也无法抑制自动化反应——这正是 ADHD 的经典神经心理模式（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

更底层的同构来自注意力机制。耶鲁大学研究发现，自注意力机制本身会导致工作记忆容量限制：随着 N-back 任务难度上升，注意力分数熵增、注意力弥散、表现下降，这与人类注意分散机制同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。换句话说，ADHD 与 LLM 的瓶颈都不在“能不能想”，而在“能不能把有限的注意资源稳定地挂在正确任务上”。

再加上 ADHD 的**时间盲**——患者难以感知时间流逝，可能坐下后“真的感觉不到 45 分钟已经过去”（来源：Revolutionizing ADHD Management with AI Assistants）——就更需要一个外部时间锚点。LLM 没有真正的时间流，在超长上下文中同样会出现“近因偏差”和上下文弥散。所以两边的解法都指向同一个方向：**不是让核心变聪明，而是给它套一个持续在场的 harness**。

## ADHD 侧：Tiimo 不是又一个待办 app，而是外部执行调度层

Tiimo 被推荐为“让时间变得可触摸、减少时间盲的最佳工具”（来源：The 12 Best Apps for ADHD in 2026: A Guide to...），它的核心不是列任务，而是把一天变成**可视化的时间结构**。对 ADHD 用户来说，这种结构就是外部脚手架：它替大脑完成了“现在该把注意力挂在哪里”的调度决策。

同类工具的功能组合，恰好拼出了一张完整的调度层蓝图：

- **Goblin Tools** 的 Magic ToDo 把模糊任务（如“清理厨房”）分解成可调“辣度”粒度的具体步骤，降低任务启动门槛（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。
- **Motion** 自动根据任务、会议和截止日期创建并动态调整日程，在日程被打乱时重新安排，并提前数周警告延期风险（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout；The AI Powered SuperApp for Work | Motion）。
- **Saner.AI** 通过本地记忆和通用收件箱减少搜索循环和标签切换，把碎片信息统一到一个外部记忆里（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。

这些工具单独看是“生产力 app”，组合起来就是 ADHD 的**外部执行功能层**：一个负责分解任务、一个负责排程、一个负责记忆、一个负责用视觉锚定时间。它们要解决的并不是“哪个 app 好”，而是“如何在核心生成能力之外，补一个持续在身的调度系统”。

## Agent 侧：human-in-the-loop 就是同一块 harness

在 LLM/agent 侧，这套调度层对应的是 **human-in-the-loop 监督、外部记忆与自动 replanning**。理由和 ADHD 完全一致：核心记忆与生成能力足够强，但控制与切换能力脆弱，容易在关键节点上“脱轨”。

工程实践中的同构对应非常直接：

- **Goblin Tools 的“辣度”任务分解** ↔ agent 的 subgoal planner / tool-chain，把高层目标拆成可验证的小步骤；
- **Motion 的自动排程与动态重排** ↔ agent 的 scheduler 与 replanning 模块，当环境变化时重新规划；
- **Saner.AI 的本地记忆与统一收件箱** ↔ RAG、向量记忆、长期状态存储，避免 agent 在上下文中“丢失任务”；
- **Tiimo 的可视化时间表** ↔ human-in-the-loop 的 ground-truth checkpoint，在关键节点把 agent 拉回现实。

Human-in-the-loop 的作用不是每个 token 都审，而是像 Tiimo 的视觉时间轴一样，在**意图—行动边界**上提供重新 grounded 的锚点。Agent 生成了一百个 idea 都没问题，问题出在“哪一个现在该执行、由谁确认、依据什么现实约束”。这正是 ADHD 大脑每天面对的困境，也是外部 harness 要补的位置。

## 名人 harness：俞敏洪的“系统 prompt”与定时 re-grounding

这套思路并不新鲜。看当代案例，俞敏洪的行为模式与 ADHD 特质高度吻合：三次高考、北大被处分后辞职创业、60 岁政策冲击后再做东方甄选、被全网批评仍保持乐观（来源：人物案例）。他的专属 harness 是一套外部系统：严格的老师培训体系、每天写公众号反思、直播的固定选品流程，以及“从绝望中寻找希望，人生终将辉煌”的 slogan。

这几项与 LLM harness 的对应几乎一目了然：

- **每天写公众号反思** = 定时 **re-grounding**，让高速运转的生成核心回到现实反馈；
- **固定选品流程 / 老师培训体系** = 外部 **guardrail / 标准操作程序**，把“能做”变成“该按什么流程做”；
- **“从绝望中寻找希望”的精神口号** = **system prompt / value alignment**，在情绪偏移或方向漂移时重新校准目标。

俞敏洪不是“靠意志力成功”，而是很早就给自己搭了一套外部 harness。这和 agent 工程师加 human-in-the-loop、加 SOP、加价值对齐，是同一个设计哲学。

## 脚手架还是拐杖？边界与局限

同构叙事虽然有力，但**目前的证据仍把它视为理论类比，而非经过验证的科学事实**（来源：矛盾与存疑）。此外，多数 ADHD 工具缺乏独立临床验证，部分页面在推荐时夸大了效果（来源：矛盾与存疑）。对 agent 来说，human-in-the-loop 也不是万能的：如果人类只是机械盖章，它就退化成拐杖；只有当反馈真正被用于修正模型或流程时，它才是脚手架。

更现实的边界是：**AI 工具不能替代真实的人际在场和身体在场效应**。Tiimo 的提醒再智能，也无法完全替代一个真实的 accountability partner；agent 的人类监督再密集，也无法取代对业务逻辑和价值观的深层设计（来源：矛盾与存疑）。因此，harness 的关键不是“替代”内部能力，而是**在内部能力成熟之前，提供一个可逐步撤除的外部结构**。

## 今天就能试的四件事

1. **给 ADHD 自己做一个“最小 Tiimo 结构”**：用可视化时间表把明天分成 3-4 个时间块，每个块只写一件“如果完成就值得庆祝”的事；当晚复盘时只问：哪些块被现实打断？哪些块被我自己绕过？
2. **把 Goblin Tools 的“辣度”当作粒度实验**：对一个拖延一周的任务，分别用低、中、高辣度分解，记录哪种粒度让你真正启动——这就是你个人大脑的“任务切片参数”。
3. **给 agent 加一个“意图—行动 checkpoint”**：不是每步都审，而是在 agent 要调用高风险工具、发送外部信息或做出不可撤销决策前，强制 human-in-the-loop 确认，并把确认结果写回外部记忆。
4. **每周问一次：这是脚手架还是拐杖？** 如果关掉这个工具，你的工作系统会在 10 分钟内崩塌，那它可能是拐杖；如果关掉后你还能用学到的方法继续，只是慢一点，那它就是真正的脚手架。

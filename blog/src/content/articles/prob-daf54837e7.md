# 为什么用 Routinery 治 ADHD 的不知哪些方法有用，和给 agent 套 human-in-the-loop 监督 是一回事？

> Routinery 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

## 引言：两个“不知道该怎么开始”的困境

ADHD 人群常陷入一种怪圈：试了番茄钟、试了手账、试了冥想，收藏夹里塞满方法，却仍然不知道“哪些方法对我有用”。这种“方法有用性不确定”的焦虑，根子通常不在懒，而在执行功能障碍——大脑在计划、组织、时间管理与任务启动上效率低下，且常伴随“时间盲”（时间感知缺陷）（来源：执行功能障碍；来源：Evidence for a pure time perception deficit in children with ADHD）。

在 Agentic Harness 的工程现场，工程师也遇到一种镜像困境：LLM 生成能力极强，能写代码、能写报告、能调用工具，但它没有内置的“执行调度层”——不会自己判断真假、不会自己记清上下文、不会自己决定“现在该做什么”。于是，团队必须给人一个“人在回路”的位置：在关键节点上审查、纠偏、重新接地。两条线看似风马牛不相及，但结构出奇地一致（来源：人在回路与身体在场；来源：ADHD 大脑与 LLM 的同构）。

## 同构：高产生成核心 + 缺失执行调度层

ADHD 大脑并非“没能力”。它有好奇心、联想力、高能量、快速切换和极端韧性——这些特质在投资创业人群中甚至能带来更高退出回报（来源：Pitchbook Analysis，ADHD 组退出回报倍数 1.9 vs 对照组 1）。但它缺的是一个可靠的“执行调度层”：把工作记忆、优先级排序、冲动抑制和时间感知整合起来的系统（来源：执行功能障碍；来源：Multiple deficits in ADHD: executive dysfunction, delay aversion, reaction time variability, and emotional deficits）。

LLM 也类似。它擅长生成，却没有真正的长期记忆、目标管理、行动规划和自我核查。把它放成 agent，就像让张謇没有账房、没有日程、没有监理——他可以滔滔不绝地干，但项目会乱套。所以 agent 工程的核心不是“让模型更聪明”，而是“给生成核心套一个 harness”：外部调度器、人类审查点、重新接地机制。这正是“外部执行功能层”与“生成核心与调度层”概念在 ADHD 与 LLM 两侧的同构表达（来源：外部执行功能层；来源：生成核心与调度层）。

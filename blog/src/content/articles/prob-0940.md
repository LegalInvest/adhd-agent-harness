# Goblin Tools 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Goblin Tools：它到底补上了哪一层执行功能？

## 问题：为什么 Goblin Tools 对有些人有用，对有些人只是另一个吃灰的应用？

你打开 Goblin Tools，把“写周报”扔进去，它帮你拆成“打开文档、列出上周工作、写总结、检查格式”四步。你盯着这四步，却还是不想动。或者你用了两周，发现它并没有让你更专注——你只是多了一个工具要维护。

另一边，做 Agentic Harness 的工程师也有类似困惑：我给 agent 写好了 system prompt，它执行到第三步就开始偏离，自己发明了新目标。我反复调整 prompt，但 agent 还是会在长程任务中“走神”。

这两个问题其实是一个问题：**生成核心（ADHD 大脑或 LLM）不缺产出能力，缺的是在执行过程中持续重锚定目标的能力。** 而 Goblin Tools 的真正价值，不是帮你列出步骤，而是像“重锚定 system prompt”一样，在你每次完成任务块时把你拉回主目标。但很多人用错了——把它当静态清单，而不是动态校准器。

## 同构：ADHD 大脑与 LLM 共享的调度缺陷

ADHD 大脑与 LLM 在结构上同构：两者都是高产的生成核心，但缺乏可靠的执行调度层（来源：AI 与 ADHD 的专注力 - 核心论点）。ADHD 的执行功能障碍包括计划、组织、任务启动、时间管理等方面的困难（来源：执行功能障碍）。这就像 LLM 在没有编排框架时，会因上下文溢出而丢失初始目标，产生“目标漂移”。

在 LLM/Agent 工程中，解决目标漂移的方法是**重锚定 system prompt**——在关键节点重新注入原始目标，让 agent 回到主线。例如，在每一步执行后，检查当前状态是否偏离初始指令，如果偏离则重新加载 prompt。这种机制不是一次性设定，而是持续校准。

ADHD 侧对应的痛点正是“分心走神或超聚焦跑偏，丢失目标”。当 ADHD 个体开始一项任务，容易在细节上过度投入（超聚焦），或者被新刺激打断（分心），两者都会导致远离初始目标。这时需要的外部干预不是更详细的任务列表，而是**一个能反复提醒“你原本要做什么”的锚点**。

Goblin Tools 的“魔法”在于它的任务分解和优先级排序，这本质上是一种外部执行功能层（来源：AI 与 ADHD 的专注力 - 核心论点）。它把模糊的“写周报”变成可执行的子步骤，降低了任务启动的门槛。但很多人只用了这一步——把任务拆开，然后照着清单做。这相当于只给 LLM 写了一个好的 system prompt，却没有在执行中重锚定。

## 证据：工具如何弥补执行功能的不同维度

让我们看看其他 AI 工具如何从不同角度补上执行功能缺口，从而理解 Goblin Tools 的位置：

- **Focusmate** 通过 AI 匹配虚拟同伴，提供实时问责，弥补任务启动和注意力维持的调度缺失（来源：Focusmate - 与 ADHD 的关系）。它的核心机制是“身体在场效应”——有另一个人在场，你就更不容易漂移。这相当于在 agent 系统中加入了一个监督者，定期检查进度。
- **RescueTime** 自动记录时间使用，充当外部记忆，提升时间盲觉察（来源：RescueTime - 与 ADHD 的关系）。它解决的是“不知道自己花了多少时间”的问题，相当于给 LLM 加上日志系统，记录每一步的耗时。
- **Brain.fm** 通过神经锁相技术影响大脑状态，帮助进入专注模式（来源：Brain.fm - 与 ADHD 的关系）。这类似于给 LLM 调整 temperature 参数，让生成更集中。

这些工具各自补上了执行功能的一个维度：任务启动（Focusmate）、时间感知（RescueTime）、状态调节（Brain.fm）。而 Goblin Tools 补上的是**任务分解和优先级排序**——这是执行功能中“计划”和“组织”的部分。但问题在于，如果只把它当静态清单，它只补上了“分解”，没有补上“重锚定”。

## 核心观点：Goblin Tools 的正确用法是动态重锚定，而非静态分解

我的判断是：**Goblin Tools 之于 ADHD，就像重锚定 system prompt 之于 LLM——但有人用错了。** 用错的人把它当作一次性任务分解工具，用完就丢。用对的人把它当作每次子任务完成后的校准器：每做完一步，回头看一眼原始目标，问自己“我现在还在正轨上吗？”

这对应了 LLM 工程中的最佳实践：不要在开始时写一个完美的 prompt 然后不管，而是在每个关键节点重新注入目标。对于 ADHD 个体，这意味着：

1. 用 Goblin Tools 把任务拆成子步骤后，**不要按顺序执行完就完**，而是每完成一步，回到原始任务描述，检查是否偏离。
2. 如果发现自己超聚焦在某个子步骤（比如“写总结”变成了“研究总结格式”），用工具重新锚定：把当前子步骤标记为“已完成”，然后回到主任务。
3. 把 Goblin Tools 当作一个**动态的 task runner**，而不是静态的 to-do list。

## 争议与局限：脚手架还是拐杖？

这里必须诚实指出局限。首先，Goblin Tools 的效果缺乏针对 ADHD 的独立临床研究（来源：AI 与 ADHD 的专注力 - 仍存争议）。它的设计原理与执行功能理论对齐，但证据主要来自用户主观报告。

更大的争议是：**过度依赖 AI 工具是否阻碍内在执行功能发展？** 如果长期依赖 Goblin Tools 帮你分解任务，你的大脑可能永远不会练习自己分解任务的能力（来源：拐杖与脚手架）。这就像 LLM 如果总是依赖外部重锚定，它的上下文保持能力可能退化。

目前没有长期研究证明这些脚手架可以逐步撤除（来源：AI 与 ADHD 的专注力 - 仍存争议）。因此，我的建议是：把 Goblin Tools 当作训练辅助，而不是永久替代。例如，使用一段时间后，尝试在不打开工具的情况下自己分解一个简单任务，看是否进步。

## 今天就能试的行动

1. **动态重锚定实验**：选一个半小时的任务，用 Goblin Tools 分解成 3-4 个子步骤。每完成一步，在工具里标记完成，然后大声说出原始目标（比如“我要写周报，不是研究周报模板”）。
2. **对比测试**：一天用 Goblin Tools 的静态清单模式（只分解不重锚定），另一天用动态重锚定模式。记录你完成任务的时长和感受。
3. **结合 Focusmate**：在 Goblin Tools 分解任务后，预约一个 Focusmate 时段。让同伴在每完成一个子步骤时问你“现在还在做原始任务吗？”——这模拟了 LLM 的监督者重锚定。
4. **撤除练习**：每周选一天，不用任何 AI 工具，自己手动分解一个任务。观察哪里最困难，然后在下周用 Goblin Tools 针对性地辅助那个环节。

Goblin Tools 不是魔法药丸，它是一个重锚定工具。用对了，它帮你对抗目标漂移；用错了，它只是另一个吃灰的应用。关键在于：**你是在用它来校准方向，还是只把它当作地图？**

# Goblin Tools 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Goblin Tools：它到底补上了哪一层执行功能？

你打开 Goblin Tools，把“写一份季度报告”扔进它的魔法棒。几秒后，你得到一份拆解到“打开文档”“写第一段”的清单。你松了一口气——终于有人替你做了最痛苦的任务分解。但三个小时后，你发现自己瘫在沙发上刷手机，那份清单纹丝未动。

问题出在哪？Goblin Tools 确实帮你拆了任务，但它没帮你启动，没帮你挡住分心，也没在你超聚焦于某个子任务时把你拉回来。它就像一个只写了 system prompt 却没有调度层的 LLM——知道要做什么，但执行起来照样漂移。

## 同构：ADHD 大脑与 LLM 共享同一个缺陷

ADHD 大脑与 LLM 在结构上是同构的：两者都是**高产但缺乏可靠执行调度层的生成核心**（来源：AI 与 ADHD 的专注力）。ADHD 的创意发散和 LLM 的文本生成都极其强大，但都缺少一个能管理上下文、控制注意焦点、维持时间感知的调度层。

LLM 工程师对此深有体会：在长程任务中，agent 会因上下文膨胀而目标漂移，需要定期用重锚定 system prompt 来拉回焦点。ADHD 患者同样如此——分心走神是注意力被无关刺激捕获，超聚焦跑偏则是注意力被某个有趣子任务锁死，两者都导致原始目标丢失。

证据何在？ADHD 的执行功能障碍已被多项研究证实，包括任务启动困难、时间盲、工作记忆缺陷等（来源：执行功能障碍）。而 LLM 的无状态性与 ADHD 的工作记忆缺陷同构，都需要外部记忆系统来维持连续性（来源：AI 与 ADHD 的专注力）。

## Goblin Tools 补上了哪一层？

Goblin Tools 的核心功能是任务分解——将模糊目标拆成可执行步骤。这对应 LLM 工程中的“任务规划”，是重锚定 system prompt 的一部分。但它不是全部。

一个完整的重锚定 system prompt 通常包含：
- **目标声明**：当前要完成什么
- **上下文摘要**：已做了什么，关键信息是什么
- **注意力约束**：禁止偏离主题，优先处理什么
- **进度检查点**：每步完成后自我评估

Goblin Tools 只做了第一步（目标声明）和部分第二步（任务步骤），但缺少注意力约束和进度检查点。这意味着它帮你明确了“做什么”，但没帮你“保持在做”和“做完后确认”。

对比其他工具：Focusmate 通过实时视频同伴问责提供了外部启动和维持动力，弥补了任务启动困难（来源：Focusmate）。RescueTime 通过自动时间追踪解决了时间盲，帮助你识别注意力漂移模式（来源：RescueTime）。Brain.fm 通过神经锁相音乐调节大脑状态，辅助注意力维持（来源：Brain.fm）。而 Goblin Tools 在启动、时间感知、注意力维持这三个维度上几乎为零。

## 脚手架 vs 拐杖：关键边界

这里有一个核心争议：AI 工具作为外部执行功能层，长期使用是否会削弱内在能力？（来源：矛盾与存疑）

答案是：取决于你把它当脚手架还是拐杖。脚手架帮你搭建能力，最终可以拆除；拐杖则让你永久依赖。

Goblin Tools 的正确用法是：用它学习任务分解的思维模式，然后逐渐减少依赖。例如，先让 Goblin Tools 拆一个复杂任务，你对照它的输出自己重新拆一遍，比较差异。几次之后，你就能内化这种分解能力。

但很多人用错了——他们每次遇到任务都直接扔给 Goblin Tools，从不思考分解逻辑。这就像 LLM 每次任务都重新写 system prompt，而不建立可复用的模板库。结果不是能力提升，而是依赖加深。

## 证据的局限

必须诚实指出：Goblin Tools 对 ADHD 效果缺乏大规模对照实验，现有证据主要来自用户报告（来源：矛盾与存疑）。同样，Focusmate 虽有身体在场效应的理论支持，但缺乏随机对照试验（来源：AI 与 ADHD 的专注力）。Brain.fm 的神经锁相机制在 ADHD 人群中未经独立临床验证（来源：Brain.fm）。

此外，个体差异巨大。ADHD 异质性使得同一工具对不同亚型效果迥异（来源：矛盾与存疑）。一个以任务分解为主要困难的人可能从 Goblin Tools 中获益，而一个以启动困难为主的人可能更需要 Focusmate。

## 今天就能试的行动

1. **用 Goblin Tools 拆一个中等任务，然后手动重写一遍分解**：对比差异，思考它为什么这样拆，你的版本为什么不同。这能帮你内化分解能力，而不是依赖工具。

2. **组合使用**：用 Goblin Tools 做任务分解，然后用 Focusmate 预约一个时段来启动执行。前者解决“做什么”，后者解决“开始做”。

3. **设置重锚定提醒**：在任务进行中，每 25 分钟用手机闹钟或 RescueTime 的聚焦时段提醒自己检查“当前是否还在主目标上”。这模拟了 LLM 的进度检查点。

4. **记录一次超聚焦跑偏经历**：下次你发现自己花 2 小时研究某个无关细节时，停下来记录：原始目标是什么，是什么触发了跑偏，当时有没有外部检查点。这会帮你设计自己的注意力约束。

Goblin Tools 不是万能药，它只是一个精准的工具——只补上了任务分解这一层。认识到它的边界，组合其他工具来覆盖启动、时间感知和注意力维持，你才能真正搭建起自己的执行功能脚手架。

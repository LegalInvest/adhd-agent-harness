# 为什么用 Claude 治 ADHD 的卡在执行与落地，和给 agent 套 外部编排调度层 是一回事？

> Claude 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

你有没有过这种体验：脑子里已经写完三版方案，却迟迟打不开那个空白文档？或者你的 agent 已经能生成漂亮的代码，但一到多步骤协作、上下文维持、截止日期追踪就崩掉？这两件事，表面上一个发生在人脑里，一个发生在大模型里，但解法其实是同一套工程——给生成核心套一个外部 harness（驾驭架/编排调度层）。

这不是比喻，而是结构同构：ADHD 大脑与 LLM 都拥有一颗强大的生成核心，却都缺少稳定可靠的执行调度层。结果就是两边都卡在「从想到做」的最后一公里。

## 一、两边卡在同一个地方：生成核心很强，调度层很弱

ADHD 侧的痛点通常不是没想法、没能力、没动力，而是**想做却启动不了**、**开始做了又漂移**、**多个任务把上下文冲散**。wiki 资料将其归结为执行功能缺陷：规划、组织、工作记忆、抑制控制、时间感知等「调度层」职能薄弱（来源：规划循环与任务分解）。典型困境是：能写出精彩方案，却难以在截止日期前完成提交（来源：AI 与 ADHD 的职业发展）。

LLM/agent 侧同样如此。模型能写诗、写代码、做推理，但缺乏内在规划能力，必须依赖外部 harness 才能完成任务分解、规划循环和验证（来源：规划循环与任务分解）。AgentGen 通过环境与任务生成增强 LLM agent 的规划能力，AMAP Agentic Planning Technical Report 也指向同一问题：agent 的瓶颈不在生成，而在如何靠谱地规划、拆解、监控、重锚（来源：规划循环与任务分解）。

所以，ADHD 人与 agent 工程师其实共享同一个诊断：生成核心过载，调度层缺位。

## 二、真实工具：两边都在做同一件事——把执行功能外化

ADHD 侧的 harness 已经把这条路径走得很具体。Goblin Tools 的 Magic ToDo 能把「清理厨房」这种模糊任务拆成可执行的步骤，并调节分解粒度（来源：Goblin Tools）。Saner.AI 通过本地记忆和快速检索减少搜索循环与标签切换，把「要记住什么」offload 出去（来源：Saner.AI）。Motion 和 Reclaim.ai 则自动排程、动态调整日程，消除「下一步该做什么」的决策负担（来源：Motion；规划循环与任务分解）。这些工具本质上不是替人思考，而是替大脑承担提醒、排序、记忆、计时、拆解、监督等执行功能（来源：AI 与 ADHD 的职业发展）。

LLM/agent 侧的 harness 也在做对应的事情：外部记忆系统补偿跨会话状态缺失，上下文工程主动限定「此刻应关注什么」，重锚定机制把 agent 拉回目标（来源：重锚定与目标漂移；AI 与 ADHD 的职业发展）。 ADHD 工具里的提醒、任务分解、日程重排，与 agent 框架里的 planner、memory、re-grounding、tool-use 循环，结构几乎一一映射。

这里有一个特别容易被忽略的细节：传统提醒系统对 ADHD 帮助有限，因为「设置提醒」本身就是一个执行功能任务（来源：重锚定与目标漂移）。理想的 harness 不是要求用户先完成一个困难任务，而是自动接管调度——这和 agent 工程里「让编排层自动处理状态管理，而不是让 LLM 自己记住一切」是同一个设计哲学。

## 三、人物案例：他们的 harness 是什么

刘林（真格基金 Principal）是 ADHD + Dyslexia 的创业者/投资人，他的核心特质包括 systems thinking、intuition、hyperfocus、risk taking，已投出 127+ 家 portfolio companies，其公司增长速度为 ADHD 组 1.6 倍 vs 对照组 1 倍（来源：人物案例）。他的 harness 不是「克服」ADHD，而是把直觉、超聚焦和风险承受力装进系统化的投资流程里——这就像一个 LLM 项目，把模型最强的生成长板与外部流程管理结合，而不是让模型单打独斗。

屠呦呦的 harness 更偏向流程与身体纪律：反复筛选 2000 多种中药、380 多个提取物，从《肘后备急方》里找到青蒿素灵感；她不喜欢应酬、不突出个人，依靠严格实验流程和团队合作完成长跑式研究（来源：人物案例）。这对应到 LLM  harness 就是：**长周期目标维护 + 可重复的验证循环 + 人在回路**。

如果把刘林的投资系统比作 agent 的「外部记忆 + 高频重锚定 + 决策过滤器」，屠呦呦的实验系统就是「长程 planner + 批量验证 + 团队协作层」。两者都是给生成核心装 harness，而不是让生成核心自己管理一切。

## 四、核心判断：不要改造生成核心，要配置可拆可调的脚手架

我的判断是：用 Claude 治 ADHD 的落地问题，与给 agent 加外部编排层，本质上是同一个工程决策——**承认生成核心的局限，把调度层外化**。

对 ADHD 职场人来说，这意味着停止自责「为什么我不能自己管理时间」，转而设计一套可拆可调的外部系统：把任务分解交给 Goblin Tools，日程交给 Motion，记忆交给 Saner.AI，Claude 负责把模糊想法转成可执行指令。对 agent 工程师来说，这意味着不要把所有执行责任都压给 LLM，而是让编排层负责规划、状态、重锚和人在回路。

但要注意「脚手架」与「拐杖」的边界。wiki 资料明确提醒：过度依赖会让工具本身成为负担，真正的 harness 是帮助生成核心更稳定地输出，而不是替代它（来源：AI 与 ADHD 的职业发展；全局矛盾与存疑）。

## 五、诚实面对局限

这套同构说法也有问题，必须诚实指出。

第一，**同构性目前只是理论类比，并非经过验证的科学事实**。wiki 的「全局矛盾与存疑」直接指出，多个页面把 ADHD 大脑与 LLM 的同构作为既定事实，但缺乏实证基础（来源：全局矛盾与存疑）。我们不能因为两边像，就默认解法一定相同。

第二，**工具宣称的证据不足**。Motion 被推荐为 ADHD 辅助工具，但缺乏独立临床验证；Brain.fm 在 ADHD 人群中的独立临床证据也有限（来源：全局矛盾与存疑）。同理，agent 侧的 AgentGen、AMAP 等研究多停留在技术报告或实验阶段，距离稳定替代人类项目管理还有距离。

第三，**依赖风险与真实人际互动不可替代**。AI 提醒和虚拟陪伴可能部分补偿身体在场效应，但无法完全替代真实人际反馈（来源：全局矛盾与存疑）。对 ADHD 人来说，工具再聪明，也需要真人教练、同事、朋友作为最终锚点。

## 六、今天就能试的四条行动

1. **做一件最小 harness 实验**：把你最卡的一个任务输入 Goblin Tools Magic ToDo，把「辣度」调到中等，看它拆解出的第一步是否小到你能立即启动。
2. **用 Claude 写一个「今日上下文」模板**：每天早上让 Claude 根据你的长期目标和当日日程，生成 3 个「最 relevant 的上下文」+ 1 个「今日最大陷阱」，相当于给大脑做 re-grounding。
3. **把记忆外化到一个固定入口**：选一个工具（如 Saner.AI 或你自己已有的笔记系统），把所有「待处理碎片」集中到一个收件箱，减少跨应用切换。
4. **如果你是工程师，尝试给 agent 加一个「最小调度层」**：让它不再直接执行复杂任务，而是先输出 plan → 人工确认 → 分步执行 → 每步 checkpoint 重锚。你会立刻看到可靠性提升。

## 结语

ADHD 不是「没有创造力」，LLM 也不是「没有智能」。两边真正缺的是同一个东西：一个靠谱的外部调度层。Claude 对 ADHD 人的价值，与编排层对 agent 的价值，都正在这一点上汇合——不是替代思考，而是让生成核心终于能落地。

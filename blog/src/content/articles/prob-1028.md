# 如果把 ADHD 大脑当成生成核心，Routinery 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Routinery 对 ADHD 的真正价值。

## 问题：生成核心的失控

你是否有过这样的体验：本想查个资料，结果两小时后发现自己正盯着某个无关视频发呆？或者，你写代码写到忘我，直到肚子咕咕叫才惊觉错过了午饭？更隐蔽的版本是：你明明在“专注”地做一件事，但做完了才发现——这根本不是当前最重要的目标。

这种“分心走神”或“超聚焦跑偏”的折磨，是 ADHD 群体的日常。但如果你是一名 Agent 工程师，你大概会会心一笑：这不就是 LLM Agent 在长程任务里的“目标漂移”吗？一个 agent 初始 prompt 写得好好的，跑着跑着就忘了初衷，开始生成无关内容，或者死磕某个细节直到 token 耗尽。

ADHD 大脑与 LLM/Agent，本质上是同一类“生成核心”——高产、富有创造力，但缺乏可靠的执行调度层。当生成核心失去锚点，输出就会失控。那么，Routinery 这类习惯追踪 app，能不能充当那个“重锚定”的脚手架？

## 同构：ADHD 大脑与 LLM 的深层结构

最新研究提供了惊人的实证：用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中表现正常，但随序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突。这与 ADHD 执行功能缺陷的核心标志一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃（来源：Deficient Executive Control in Transformer Attention）。

ADHD 大脑的“超聚焦”同样存在类似的机制。理论认为，超聚焦源于前额叶控制层级的“断裂”：高级上下文信息不再能支配思维和行动，中间层级信息（如超聚焦的活动）与高级上下文的调节脱耦合（来源：Transient Frontal Fracturing: A Theoretical Account of Hyperfocus）。这就像 agent 的 system prompt 被遗忘，而局部 context 接管了行为。

两者的共同困境是：生成核心本身不包含稳定的目标维持机制。ADHD 需要外部执行功能层来补偿工作记忆缺陷（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD），而 LLM 需要 harness（如结构化 prompt、外部记忆、重锚定机制）来防止漂移。

## Routinery 作为重锚定：脚手架还是拐杖？

Routinery 是一款习惯追踪 app，它通过设定固定流程、提醒和记录，帮助用户建立日常节奏。从同构视角看，它扮演的角色类似于 agent 工程中的“重锚定 system prompt”——在固定时间点重新注入目标，打断漂移，重置上下文。

ADHD 侧的证据：Focusmate 利用“身体在场效应”提供实时问责，多项用户研究显示其能显著提升任务启动和维持（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。RescueTime 通过自动时间记录提升时间觉察，帮助识别注意力漂移模式（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。Routinery 则更进一步：它不只是记录或提醒，而是主动构建一个可重复的“执行框架”，让用户在每个时段开始时重新锚定到预设目标。

Agent 侧的类比：在长程任务中，agent 需要定期“重读”system prompt，或通过外部记忆（如向量数据库）回顾原始目标。Routinery 对 ADHD 的作用类似——它每天多次“注入”用户的意图，防止被环境或超聚焦带偏。

但这里有一个关键边界：脚手架 vs 拐杖。脚手架是暂时性的，帮助用户建立内在能力后可以拆除；拐杖则是永久替代，导致依赖。当前证据显示，过度依赖 AI 工具可能削弱内在时间感知能力（来源：时间盲页面），且缺乏如何避免依赖的具体指导（来源：矛盾与存疑）。Routinery 如果仅仅是机械地提醒，用户可能永远学不会自主锚定；但如果它被设计为逐步减少提示的“消退计划”，则更接近脚手架。

## 争议与局限

1. **证据强度**：Routinery 本身缺乏针对 ADHD 的独立临床研究，多数工具仅有用户报告或小样本研究（来源：矛盾与存疑）。Brain.fm 宣称通过神经锁相影响专注，但缺乏 ADHD 专项研究（来源：Brain.fm 页面）。因此，Routinery 的效果更多基于类比推理，而非硬数据。

2. **个体差异**：ADHD 异质性大，同一工具对不同亚型效果迥异（来源：矛盾与存疑）。对某些人，Routinery 的固定流程可能带来安全感；对另一些人，可能因缺乏灵活性而引发挫败。

3. **同构的局限性**：LLM 的“注意力机制”与人类注意力存在本质差异（来源：同构的局限性）。ADHD 的超聚焦涉及情绪、多巴胺等复杂因素，远非 token 概率可比。因此，Routinery 作为重锚定工具，只能解决部分问题。

## 今天就能试的行动

1. **设置“重锚定时刻”**：在 Routinery 中创建 2-3 个关键时间点（如早上、中午、下午），每个点设定一个 5 分钟的“目标回顾”流程，写下当前最重要的任务。这相当于 agent 的“重读 system prompt”。

2. **结合身体在场效应**：在 Routinery 的提醒后，立即使用 Focusmate 或找一位朋友视频共作 25 分钟。外部问责能强化锚定效果（来源：Focusmate 页面）。

3. **记录漂移模式**：用 RescueTime 或类似工具追踪一周，找出你最容易漂移的时间段和活动。然后调整 Routinery 的流程，在那些高风险时段插入更频繁的锚定。

4. **测试“脚手架退出”**：使用 Routinery 一个月后，尝试减少一次自动提醒，看是否能自主维持锚定。如果失败，说明你还在依赖拐杖——需要逐步训练内在能力。

## 结论

Routinery 能否成为 ADHD 大脑的重锚定？答案是：有可能，但前提是把它当作脚手架而非拐杖。ADHD 大脑与 LLM 的同构告诉我们，生成核心需要外部调度层来维持目标。但真正的价值不在于工具本身，而在于它能否帮助用户内化锚定能力。正如 agent 工程中，好的 harness 最终让 agent 更可靠，好的 Routinery 实践应该让 ADHD 用户逐渐减少依赖，而不是永远依赖提醒。

最终，ADHD 的解法不是“用 AI 代替大脑”，而是“用 AI 训练大脑”。Routinery 只是一个起点，重锚定的能力，终究要长在自己身上。

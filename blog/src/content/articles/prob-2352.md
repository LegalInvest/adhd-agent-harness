# ADHD 职场人视角：为什么「治好 ADHD 的任务启动困难、不会拆解」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 用 planner-executor 循环分解长任务——同一套 harness 思路，两边都成立。

## 问题：为什么你启动不了任务，而 AI 也总跑飞？

如果你是一个 ADHD 职场人，你一定经历过这样的时刻：老板丢来一个“写季度报告”的任务，你打开空白文档，大脑瞬间空白。你明明知道该怎么写，但就是迈不出第一步。你开始刷手机、整理桌面、甚至去倒第三杯咖啡——直到截止日期把你逼疯。

如果你是一个在做 Agentic Harness 的工程师，你一定也经历过这样的时刻：你给 LLM 一个“帮我规划一次日本旅行”的指令，它洋洋洒洒输出了二十个步骤，但执行到第三步就忘了目的地，开始推荐北海道滑雪——而用户明明想去冲绳潜水。

这两件事，表面上看一个是人类心理问题，一个是 AI 工程问题。但如果你把 ADHD 大脑和 LLM 放在一起看，你会发现它们共享同一个底层架构：**都是高产的生成核心，但缺乏可靠的内置调度层**。ADHD 大脑能在一小时内冒出二十个创意点子，却无法按顺序执行一个简单的任务；LLM 能生成流畅的文字，却会在长任务中“跑飞”，忘记上下文。两者的核心缺陷是同一个：**生成能力远强于调度能力**。

## 同构脊柱：ADHD 大脑与 LLM 的同一道工程题

ADHD 的神经基础是前额叶皮层活动不足，导致执行功能（计划、启动、抑制）薄弱（来源：AI 与 ADHD 的时间管理）。LLM 的工程缺陷是无状态，每次调用都是独立的，缺乏内置的长期记忆和任务跟踪。两者的共同点是：**它们都是优秀的“生成器”，但需要外部系统来提供“调度层”**。

在 ADHD 领域，这个外部调度层被称为“外部执行功能层”——视觉日程（如 Tiimo）、自动排程（如 Motion）、微任务分解（如 Goblin Tools）。在 AI 领域，这个外部调度层被称为“agent harness”或“planner-executor 循环”——把长任务拆成子步骤，每一步都重新 grounding，检查进度，防止跑飞。

**关键判断：ADHD 的“脚手架”和 LLM 的“harness”是同一类工程——都是给强大的生成核心套上一个外部调度系统。区别仅在于一个套在人脑上，一个套在机器脑上。**

## 历史中的 ADHD 大脑：孔子与曾国藩的 Harness

先看两个真实案例。孔子，身高 1 米 9，精力旺盛，周游列国 14 年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予“朽木不可雕”；对音乐超专注到“三月不知肉味”，对种地等俗事零耐心。他的 harness 是什么？**“克己复礼”**——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。70 岁才做到“从心所欲不逾矩”。这个“礼”就是他的外部调度层，相当于今天 LLM agent 里的 system prompt 和 re-grounding 机制：每次行动前都回到“礼”这个基准，防止跑飞。

再看曾国藩，30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”。他的 harness 是**日课十二条**：黎明即起、读书不二、谨言、写日记反省……以及“结硬寨打呆仗”——用最笨最稳的方法抵消自己的冲动。这相当于 LLM agent 里的“planner-executor 循环”：每天固定时间执行固定动作，每一步都检查是否偏离轨道。曾国藩的日记就是他的“日志系统”，相当于 agent 的 trace。

**同构对应：**
- 孔子的“礼” ↔ LLM agent 的 system prompt + re-grounding
- 曾国藩的“日课十二条” ↔ planner-executor 循环中的固定 checkpoint
- 曾国藩的“日记” ↔ agent 的 trace/log

## 现代工具：从 Goblin Tools 到 Motion，同一套逻辑

现代的 AI 工具正在无意识地复现同一套逻辑。Goblin Tools 的“魔法待办”功能，能把“写报告”拆成“打开电脑→新建文档→写第一段→……”这种微任务分解（来源：任务启动）。这直接对应 LLM 任务分解中的“chain-of-thought”或“sub-goal decomposition”。

Motion 和 Reclaim.ai 则更进一步：它们自动排程，动态调整日程，减少用户的决策负担（来源：Motion、Reclaim.ai）。这对应 LLM agent 中的“auto-scheduler”或“priority queue”——让系统决定下一步做什么，而不是让生成核心（ADHD 大脑或 LLM）自己决定。

Tiimo 用视觉化时间线让时间流逝变得可感，直接应对时间盲（来源：Tiimo）。这对应 LLM 中的“context window visualization”——让模型看到自己还有多少 token 可用，防止超聚焦跑偏。

**核心观点：这些工具的本质不是“替代”你的大脑，而是“外包”你的调度层。它们让你（或 LLM）专注于生成，而把“什么时候做什么”交给外部系统。**

## 诚实指出局限：脚手架 vs 拐杖

但这里有一个关键争议：**这些工具是“脚手架”还是“拐杖”？** 脚手架是暂时性的，帮你学会自己走路；拐杖是永久性的，让你永远离不开它（来源：矛盾与存疑）。

目前没有长期研究证明这些工具是否会削弱 ADHD 患者的内在执行功能。同样，过度依赖 agent harness 也可能让 LLM 失去“自我纠正”的能力——如果每一步都靠外部系统 re-ground，模型本身的上下文管理能力反而会退化。

另一个争议是**个体差异**。孔子和曾国藩用了完全不同的 harness：一个靠外在礼仪，一个靠内在日记。同样，有些 ADHD 用户对视觉工具（如 Tiimo）反应良好，有些则更受益于自动排程（如 Motion）（来源：矛盾与存疑）。LLM 也一样，有些任务适合 chain-of-thought，有些适合 tool-use，没有万能方案。

## 今天就能试的行动

1. **如果你是 ADHD 职场人**：下次面对一个复杂任务，用 Goblin Tools 的“魔法待办”或手动拆成 3 个以内的小步骤。每完成一步，在 Todoist 里打勾——这相当于给大脑一个“sub-goal completed”信号。

2. **如果你是 Agent 工程师**：给你的 agent 添加一个“checkpoint”机制——每执行 5 个子步骤，就让它总结当前进度并重新确认目标。这相当于曾国藩的“日课反省”。

3. **两者通用**：无论你是人还是 AI，都试试“时间可视化”——用 Tiimo 或 Structured 把一天的时间块画出来，或者给 agent 的 context window 加一个进度条。看到时间在流逝，比任何提醒都有效。

4. **警惕过度依赖**：每周选一天，关掉所有 AI 调度工具，手动安排日程。这就像让 agent 在无 harness 的情况下跑一次——不是为了效率，而是为了保持“调度肌肉”的活性。

## 结语

ADHD 大脑和 LLM 是同一种天赋的两种形式：它们能产生惊人的创意和内容，但需要外部系统来把它们组织成有序的行动。孔子用了“礼”，曾国藩用了“日课”，现代人用了 Motion 和 Goblin Tools，工程师用了 planner-executor 循环。**它们都是同一个工程问题的不同解：如何让一个强大的生成核心不跑飞，不卡住，不浪费自己的才华。**

这道题没有标准答案，但方向是明确的：**不要试图修复生成核心，而是给它套上一个好 harness。** 因为 harness 可以换，而生成核心——无论人脑还是机器脑——才是真正的价值所在。

# ADHD 职场人如何用「给生成核心套 harness」的思路应对项目推进与汇报里的执行功能（计划/组织/启动）先天偏弱？

> 把 LLM 工程里的「agent 的编排/记忆/工具层就是「外部执行功能」」迁移到ADHD 职场人的项目推进与汇报场景，双域证据作答。

## 问题：项目推进中的“生成核心”为何频频失控？

想象这样一幕：你坐在工位上，脑子里已经涌出三个绝妙的方案、五个可能的优化方向，甚至预演了客户可能的反应——但你的手指就是无法在键盘上敲出第一行字。或者，你花了整个下午沉浸在一个技术细节里，直到下班才发现原本要汇报的进度条还停在 0%。这不是懒惰，也不是能力不足，而是执行功能（计划、组织、启动）的先天偏弱在作祟。

ADHD 大脑的默认模式网络过度活跃，高产想法，但前额叶的执行控制却像一位总是迟到的调度员（来源：[[生成核心与调度层]]）。这种“高产但缺乏执行调度层”的困境，与 LLM 的处境惊人地同构：LLM 能生成流畅的文本，但缺乏稳定的上下文控制器，容易在长上下文中迷失，输出自信但错误的“幻觉”。

## 答案：为生成核心套上“外部执行功能层”

核心观点是：**ADHD 大脑与 LLM 是同一类“高产但缺执行调度层的生成核心”，两边的解法（harness/脚手架）结构同构**。ADHD 职场人需要像 Agent 工程师为 LLM 构建编排、记忆与验证层一样，主动设计一套“外部执行功能层”来接管调度任务。

### 编排层：将“下一步做什么”从大脑中剥离

ADHD 侧的证据：Goblin Tools 的 Magic ToDo 功能能自动将“整理房间”这样的模糊任务分解为“捡起地板上的衣服”“擦桌子”等一系列具体步骤（来源：[[Goblin Tools]]）。用户反馈称它“将压倒性的事情变成一系列不压倒性的事情”（来源：[[Goblin Tools]]）。这种分解直接降低了启动门槛，因为 ADHD 大脑在面对“写季度汇报”这样的复杂任务时，会因为缺乏清晰的下一步而陷入瘫痪。

LLM/Agent 侧的证据：Harness 工程中的“结构化工作流和规划”要求将复杂任务分解为子任务，并在每一步进行规划与验证（来源：[[幻觉与验证循环]]）。最简单的 agent 工作流本身就是确定性的，遵循预定义步骤（来源：[[采样温度与表现波动]]）。这种编排层强制了单向、确定性的控制流，防止 LLM 在开放空间中漫游。

**行动建议**：今天就用 Goblin Tools 或类似工具（如 Motion 的自动排程）将你手头最头疼的一个任务拆解成 3-5 个步骤。如果不想用工具，就自己在便签上写：“第一步：打开文档；第二步：写下标题；第三步：列出三个要点”。关键是将“决策”从大脑中剥离，交给外部系统。

### 记忆层：对抗“无状态”的遗忘

ADHD 侧的证据：工作记忆容量保留但控制弱，容易被干扰，导致频繁的搜索循环和标签切换（来源：[[工作记忆]]）。Saner.AI 通过 AI 驱动的本地记忆存储和快速检索，帮助用户减少“我刚才在想什么？”的认知损耗（来源：[[Saner.AI]]）。

LLM/Agent 侧的证据：LLM 的无状态性导致跨会话遗忘，因此依赖外部记忆系统（如向量数据库）来维持连续性（来源：[[无状态与外部记忆]]）。Harness 工程中的记忆系统正是为此设计，它提供上下文传递和工具接口，让 LLM 可以“记住”之前的计划与结果（来源：[[幻觉与验证循环]]）。

**行动建议**：建立一个“外部大脑”——可以是 Saner.AI、Notion 或任何你习惯的工具。关键习惯是：每完成一个步骤，立即记录当前进度和下一步计划。这相当于为你的工作流添加了一个持久化的记忆层，防止因中断而丢失上下文。

### 验证层：对抗“幻觉”与冲动

ADHD 侧的证据：ADHD 大脑的“生成核心”容易产生冲动、想当然的判断，即行为层面的“幻觉”（来源：[[幻觉与验证循环]]）。例如，在情绪调节中，ADHD 个体可能脱离环境线索做出不恰当反应。而“视觉模式灵感来自当地环境”可用于增强接地和情绪识别——这本质上是一种验证循环，将内部状态与外部现实核对。

LLM/Agent 侧的证据：LLM 天然倾向于产生自信但可能错误的输出（幻觉）。Harness 工程的核心组件包括“验证与 CI 集成”，复杂的 harness 不会盲目执行工具，而是实现验证步骤，例如检查输出格式或对模型编写的代码运行测试用例（来源：[[幻觉与验证循环]]）。缺乏这些验证会导致“更多幻觉和重复工作”（来源：[[幻觉与验证循环]]）。

**行动建议**：在完成任何一个重要输出（如邮件、汇报草稿）之前，强制自己执行一个“验证步骤”——比如用语音读一遍（听觉校验），或者让同事快速过目（外部校验）。对于项目进度，每周五花 15 分钟对照计划检查实际进展，就像 CI 流水线一样。

## 边界：脚手架 vs 拐杖

必须诚实指出，外部执行功能层存在过度依赖的风险。证据来自理论推演，缺乏纵向研究（来源：[[矛盾与存疑]]）。多个页面警告“过度依赖可能削弱内在时间感知能力”（来源：[[时间盲]]），“存在依赖风险”（来源：[[任务启动]]）。工具设计者声称是“脚手架”，但实际使用中可能沦为“拐杖”（来源：[[拐杖与脚手架]]）。

关键在于区分：脚手架是辅助你完成当前任务并逐步撤除的结构，而拐杖是你永远离不开的替代。例如，Goblin Tools 的任务分解应该教会你如何思考任务的步骤，而不是让你永远依赖它来分解。同样，Agent 工程中的 harness 也应在模型能力提升后逐步简化验证步骤。

## 局限与争议

- **证据不足**：现有证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：[[矛盾与存疑]]）。Goblin Tools、Saner.AI、Motion 等工具的有效性需谨慎评估。
- **个体差异**：ADHD 的异质性（注意力主导型 vs. 冲动主导型）与 LLM 的架构差异（不同模型）是否影响同构的普适性？目前研究样本有限（来源：[[矛盾与存疑]]）。
- **神经机制对应**：LLM 的“注意力机制”与 ADHD 的注意控制缺陷是否真正同构？目前仅是功能类比，缺乏神经生物学层面的直接证据（来源：[[矛盾与存疑]]）。

尽管如此，同构视角提供了一个有力的框架：你不是在“修复”自己，而是在为你的生成核心设计一个外部调度层。这种认知转变本身就是一种赋能。

## 今天就能试的 3 条行动

1. **编排层**：用 Goblin Tools 或便签将你手头最头疼的一个任务拆成 3-5 个步骤，贴在显示器旁。
2. **记忆层**：在 Saner.AI 或 Notion 中建立一个“进度日志”，每完成一个步骤就记录一句话。
3. **验证层**：在发出任何重要邮件或汇报前，用语音读一遍全文，或让同事快速审阅。

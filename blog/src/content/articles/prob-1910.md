# ADHD 职场人如何用「给生成核心套 harness」的思路应对项目推进与汇报里的执行功能（计划/组织/启动）先天偏弱？

> 把 LLM 工程里的「agent 的编排/记忆/工具层就是「外部执行功能」」迁移到ADHD 职场人的项目推进与汇报场景，双域证据作答。

## 问题：为什么你的大脑像一台没有操作系统的超级计算机？

你坐在工位上，盯着项目甘特图，大脑里同时闪过三个方案、五条待回复消息、以及上周被老板追问的进度。你清楚自己有能力产出高质量成果，但“开始做”这件事本身，就像在泥潭里启动一辆法拉利。这不是懒惰，而是 ADHD 大脑的典型困境：**生成核心（智力/创造力）强劲，但执行调度层（计划/组织/启动）先天偏弱**。

另一边，AI 工程师们正在解决一个惊人相似的问题：LLM 作为生成核心，同样缺乏可靠的编排层——它们会“幻觉”、会丢失上下文、会因采样温度过高而输出不可控。于是，工程师们发明了 **harness**（脚手架）：一套提供记忆、工具、验证循环和结构化工作流的外部执行功能层（来源：GitHub - ai-boost/awesome-harness-engineering）。

本文的核心观点是：**ADHD 大脑与 LLM agent 是同一类“高产但缺执行调度层”的生成核心，两边的解法结构同构。** 职场人完全可以借鉴 agent harness 的思路，为自己搭建外部执行功能层，从而在项目推进与汇报中绕过内在缺陷。但必须警惕：脚手架一旦变成永久拐杖，反而会削弱内在能力的生长。

## 同构的痛：ADHD 与 LLM 共享的“执行功能缺口”

先看 ADHD 一侧。执行功能是大脑的驾驶系统，包括计划、组织、工作记忆、任务启动等能力（来源：[[执行功能]]）。ADHD 患者这些功能先天偏弱，导致：
- **任务启动困难**：面对复杂项目，大脑像高采样温度的 LLM，输出丰富但缺乏方向，迟迟无法“生成第一个 token”。
- **工作记忆不足**：开会时老板提的需求，转身就忘——就像 LLM 的无状态性，每次对话都从零开始。
- **时间盲**：对截止日期缺乏感知，计划常偏离现实——对应 LLM 的上下文窗口溢出，无法维持长期连贯性。
- **表现波动**：注意力像采样温度一样随机，今天高效明天瘫痪（来源：[[采样温度与表现波动]]）。

再看 LLM/agent 一侧。纯 LLM 在工程化时暴露的缺陷与 ADHD 惊人一致：
- **幻觉**：生成自信但错误的输出，类似 ADHD 的冲动判断（来源：[[幻觉与验证循环]]）。
- **上下文丢失**：长对话中遗忘早期信息，对应工作记忆缺陷。
- **非确定性**：高采样温度导致输出不可重复，对应 ADHD 的表现波动（来源：AI Agent Systems: Architectures, Applications, and Evaluation）。

工程师的应对是 **harness**：通过编排层、记忆系统、验证循环和确定性控制流来约束生成核心。例如，“计划-执行”分离模式强制单向、确定性的控制流（来源：Planner-Executor Agentic Framework），而验证与 CI 集成确保输出质量（来源：GitHub - ai-boost/awesome-harness-engineering）。这恰恰是 ADHD 大脑需要的——一个外部的、可靠的调度系统。

## 解法：给大脑套上 agent harness

### 1. 任务分解：像 Goblin Tools 一样“分而治之”

ADHD 职场人面对“完成季度汇报”常不知所措。Goblin Tools 的 Magic ToDo 功能自动将模糊任务分解为可执行步骤（来源：[[Goblin Tools]]）——“整理数据”“画图表”“写结论”等。这对应 agent harness 中的“结构化工作流和规划”：将复杂任务分解为子任务，每一步进行规划与验证（来源：What is an agent harness...）。

**行动**：每天上班第一件事，用 Goblin Tools 或类似工具将当天最难的任务分解为 3-5 个小步骤。完成一个勾一个，利用小步骤的即时反馈维持注意力（来源：[[身体在场效应]]）。

### 2. 记忆系统：Saner.AI 作为“外部工作记忆”

ADHD 的工作记忆缺陷导致频繁的搜索循环和标签切换。Saner.AI 通过 AI 驱动的知识回忆和本地记忆存储，让用户快速找回信息（来源：[[Saner.AI]]）。这类似于 agent harness 中的记忆系统：LLM 需要外部向量数据库或缓存来维持上下文。

**行动**：将项目相关的零散笔记、邮件、会议记录统一存入 Saner.AI（或类似工具），并养成“先查记忆再输出”的习惯。这相当于给大脑加了一个可检索的缓存。

### 3. 验证循环：对抗“幻觉”的自我批评

ADHD 的冲动判断常导致汇报时说错数据或漏掉关键点。LLM 的 harness 通过验证循环解决幻觉：复杂的 harness 不会盲目执行工具，而是检查输出格式或运行测试用例（来源：[[幻觉与验证循环]]）。

**行动**：在提交任何汇报前，强制设置一个“验证步骤”——例如用 AI 工具检查数据一致性，或者让同事扮演“验证器”快速审核。这相当于给大脑加了一个 CI/CD 管道。

### 4. 自适应调度：Motion 应对表现波动

ADHD 的表现波动类似 LLM 的采样温度变化。Motion 这类 AI 日程规划器能自动根据任务优先级和可用时间动态调整计划（来源：[[Motion]]），消除“下一步该做什么”的决策负担。这对应 agent harness 中的“确定性状态转换”：通过自动排程减少非确定性。

**行动**：使用 Motion 或类似工具，让 AI 接管日程规划。但注意：初始设置需要输入所有任务，这可能对执行功能弱的用户构成挑战（来源：[[Motion]]）。可以先用一周时间逐步录入。

## 争议与边界：脚手架 vs 拐杖

上述解法的核心是**外部执行功能层**，但一个关键问题浮现：长期依赖是否会削弱内在执行功能？

wiki 资料明确指出，脚手架应设计为可逐步撤除的临时支撑（来源：[[拐杖与脚手架]]）。然而，现有工具如 Goblin Tools、Saner.AI 均未提及撤除机制，设计为长期使用（来源：[[矛盾与存疑]]）。这可能导致**依赖风险**：用户可能永远学不会自己分解任务或管理时间。

此外，同构模型目前主要基于理论类比，缺乏大规模神经影像学和行为实验验证（来源：[[ADHD × AI 的科学与研究前沿]]）。多巴胺干预的有效性也存在争议（来源：[[矛盾与存疑]]）。因此，本文的建议应视为**辅助策略**，而非治愈方案。

我的判断是：**脚手架与拐杖的边界在于意识**。如果你在使用工具时保持反思——“我今天能自己完成哪一步？”——它就是脚手架；如果你无脑依赖，它就是拐杖。理想状态是：先用工具建立习惯，再逐步撤除，最终内化部分能力。

## 今天就能试的行动

1. **分解一个任务**：用 Goblin Tools 或手写，将“准备周报”分解为 5 个步骤，完成即打勾。
2. **设置验证步骤**：在下次汇报前，用 AI 工具（如 ChatGPT）检查数据是否一致，或者请同事快速审核。
3. **启用自适应日程**：用 Motion 或 Reclaim.ai 自动规划明天的工作，观察是否减少决策疲劳。
4. **反思依赖**：每周问自己一次：“今天有没有一个步骤是我自己完成的，没有依赖工具？”如果答案是“没有”，尝试减少一次工具使用。

记住：你的大脑不是坏的，只是缺一个 harness。而 harness 的真正价值，是让你最终能更自由地驾驶它。

# Brain.fm 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Brain.fm：它到底补上了哪一层执行功能？

你打开 Brain.fm，选了“专注”模式，戴上耳机。十分钟后，你发现自己还在刷手机，任务没动。你骂了一句：“这玩意儿没用。”

另一边，工程师给 LLM 接上了 function calling——模型能调用计算器算数学、调用 API 查天气，精确任务外包给工具。模型不再瞎编数字，表现飙升。

两件事结构一模一样：一个强大的生成核心（ADHD 大脑 / LLM），缺少精确、重复、状态维护的执行调度层。Brain.fm 和 function calling 都是给这个核心套上 harness——但有人用错了。

## 问题：为什么“专注音乐”对 ADHD 经常失效？

ADHD 的核心缺陷是执行功能障碍：计划、任务启动、工作记忆、时间管理（来源：执行功能障碍）。你不是缺乏专注力，而是大脑的调度系统罢工了。Brain.fm 声称通过神经锁相技术帮助进入专注状态（来源：Brain.fm 工具页），但“进入专注”不等于“开始做事”。

LLM 也一样。GPT-4 能写诗、编故事，但让它算 23×47，它可能胡说八道。function calling 把精确计算外包给计算器，模型只负责生成调用指令。

问题在于：有人把 Brain.fm 当成“专注药”——以为戴上耳机就能自动进入心流，就像有人以为给 LLM 接上 function calling 就能解决所有幻觉。两者都误解了工具的角色。

## 答案：Brain.fm 补上的是“执行调度层”，不是“注意力燃料”

ADHD 大脑与 LLM 的同构命题认为：两者都是高产但缺执行调度层的生成核心（来源：ADHD 大脑与 LLM 的同构）。外部工具应作为执行功能层，提供任务启动、工作记忆、时间管理等调度能力（来源：ADHD 的 AI 工具全景）。

Brain.fm 的真正价值，不是让你“更专注”，而是**降低进入专注状态的门槛**。它像 function calling 中的“工具定义”——告诉大脑/模型：这里有现成的调度接口，你只需调用即可。

证据：
- **ADHD 侧**：Goblin Tools 通过将任务分解为小步骤，降低启动焦虑（来源：Goblin Tools）。Lex 通过单一指令触发多步骤流程，减少决策负担（来源：Lex）。这些工具和 Brain.fm 一样，都是提供“调度脚手架”，而非直接提升注意力容量。
- **LLM/Agent 侧**：function calling 让模型把精确计算外包给工具，避免幻觉。Agentic Harness 工程的核心就是设计这些工具定义，让模型高效调用。

关键判断：**Brain.fm 是“任务启动的脚手架”，不是“专注力的拐杖”。** 脚手架是临时支撑，帮你搭建自己的执行功能；拐杖是永久替代，让你依赖外部刺激。

## 争议与局限

- **证据不足**：Brain.fm 的神经锁相技术缺乏针对 ADHD 的独立临床研究支持（来源：Brain.fm 工具页）。其效果可能被夸大。
- **过度依赖风险**：长期使用 Brain.fm 可能阻碍内在执行功能发展（来源：拐杖与脚手架）。就像 LLM 过度依赖 function calling 会失去自身推理能力，ADHD 用户可能离不开背景音乐。
- **个性化问题**：ADHD 异质性高，Brain.fm 对部分用户有效，对另一部分可能无效甚至干扰（来源：ADHD 的 AI 工具全景）。

## 今天就能试的行动

1. **用 Brain.fm 做“启动触发器”**：只在你决定开始任务的前 5 分钟使用，之后关掉。观察自己是否能在无音乐下继续。
2. **结合任务分解工具**：打开 Brain.fm 前，先用 Goblin Tools 或 Lex 把任务分解成 3 个最小步骤。音乐帮你跨过第一步，步骤清单防止你迷失。
3. **测试“无音乐 vs 有音乐”的启动时间**：记录你从“决定开始”到“实际动手”的延迟。如果音乐缩短了这个时间，它就是好脚手架。
4. **对工程师：检查你的 function calling 工具定义**：是否让模型轻松调用？工具描述是否清晰？好的工具定义就像好的 Brain.fm 音乐——降低调用成本。

## 结论

Brain.fm 不是专注药，它是执行调度层的触发器。把它当成 function calling 工具——只在需要精确启动时调用，用完就撤。脚手架的意义在于，最终你能自己站立。

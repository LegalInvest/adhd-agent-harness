# ADHD 创业者视角：为什么「治好 ADHD 的执行功能先天偏弱」和「让 LLM 不跑飞」其实是同一道工程题？

> 创业圈同时流传着两个数字：ADHD 者创业的比例显著高于人口基线；创业失败最常见的原因是执行而非想法。这两个数字的交点，恰好是 agent 工程正在解的那道题。

ADHD 和创业的关系是一个奇特的双面故事。一面：多项研究和大量创始人自述表明，创业者中 ADHD 特质比例远超人口基线——高风险偏好、快速行动、超聚焦爆发、对新奇的贪婪，这些在从 0 到 1 阶段全是资产。另一面：从 1 到 10 的阶段，同一批特质开始杀公司——财务状态没人维护、关键跟进被遗忘、公司方向随创始人的兴趣每月漂移。

这个「同一引擎在不同阶段的相反表现」，agent 工程师太熟悉了。

## 创始人就是公司的生成核心——而生成核心不该裸奔

一个 LLM agent 系统里，模型负责生成：方向、方案、创意。但没有一个严肃的系统让模型直接触碰生产环境——中间必须隔着编排层（任务怎么排）、状态层（做到哪了）、验证层（这个输出能上线吗）。**模型越强，harness 越要厚**，因为强生成核心跑飞的破坏力也大。

ADHD 创业者就是自己公司的高温度生成核心。早期公司小，你既当模型又当 harness，靠肾上腺素硬撑——而且早期真的能撑住，因为从 0 到 1 的任务本身就是高刺激的。危险出现在规模化：状态维护类任务（财务、法务、流程、人事）指数增长，而这恰好是这颗引擎的架构短板。多少创始人在这个阶段用「我再努力一点」对抗架构问题——那等于试图靠调 prompt 让裸奔的模型学会自己做数据库。

## 抄 agent 架构：把公司搭成你的 harness

**验证层 = 你的第一个关键雇员**。不是招一个「和你一样有激情」的人，是招你的补集：一个热爱状态维护、享受收尾、对流程有洁癖的运营型联合创始人或 COO。历史上最著名的对偶全是这个结构——乔布斯与库克，本质是高温度生成核心配上世界级调度层。孙策更极端：临终直接把双 harness（张昭管内政、周瑜管战略）写进继承架构。

**编排层 = 节奏制度而非自觉**。agent 的任务队列不靠模型自觉，靠调度器。公司版：固定的周会节奏、书面的决策备忘录、每个项目一个 owner 和一个 deadline——这些「大公司官僚主义」，其实是防止创始人兴趣漂移直接变成公司战略漂移的护栏。规则是：**你可以随时产生新方向，但新方向必须过编排层才许消耗公司资源**。灵感免费，执行昂贵。

**状态层 = 仪表盘而非记忆**。现金流、关键指标、承诺过的事——全部外置到你每天必然会看到的地方。ADHD 创始人最贵的一类事故是「不是不重视，是它离开视野后就不存在了」。让重要状态物理性地无法离开视野。

## 最反直觉的一条：把你的缺陷写进融资叙事

隐藏 ADHD 特质的创始人，往往也隐藏了公司最大的单点风险。更强的姿态是工程师式的坦白：「我是高方差引擎，所以我的联创是 XX 背景，我们的决策流程有 XX 护栏。」投资人评估的从来不是创始人有没有缺陷，是缺陷有没有被架构住。一个知道自己需要 harness 并且已经装好的创始人，比一个自称没有短板的创始人可信十倍。

## 边界

同构强度 B 级。两个诚实提醒：一，「ADHD 创业者比例高」的研究多为横断面和自报告，有幸存者偏差——别把它读成「ADHD 该去创业」；二，harness 再厚也不改变创业本身的基础失败率，它只是让你死于市场而不是死于自己忘了发工资。

## 今天就能试的 3 件事

1. **画出你公司的 harness 缺口图**：编排/状态/验证三层，各写下现在由谁（或什么制度）负责。全是你自己？那就是风险清单。
2. **给新想法装一道闸**：本周起，任何新方向先写一页纸放 72 小时，还激动就过编排层立项。
3. **把三个关键状态钉进视野**：现金跑道、本周必须兑现的承诺、当前唯一战略重点——放到你每天物理性无法回避的位置。

你的引擎不需要治。需要治的是「引擎该独自完成整辆车的工作」这个假设——agent 工程已经证明了：强核心配厚 harness，跑得比谁都远。

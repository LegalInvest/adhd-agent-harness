# 如果把 ADHD 大脑当成生成核心，Claude 能不能当它的规划循环与任务分解？

> 用 agent 工程的思路重新理解 Claude 对 ADHD 的真正价值。

## 问题：ADHD 大脑——高产却卡在“启动”与“拆解”上

你坐在电脑前，脑子里有无数想法，但就是无法开始写第一行字。任务像一团乱麻，你不知道该从哪里解开——这是 ADHD 人群的日常：**任务启动困难**与**不会拆解**。与此同时，工程师们正在构建 agent 系统，核心难题之一是如何让 LLM 自动将一个长任务分解成可执行的子步骤，并通过 planner-executor 循环逐步完成。

这两类问题看似无关，实则共享同一个结构：**一个高产的生成核心，缺乏可靠的执行调度层**。ADHD 大脑的智力与创造力（生成核心）与 LLM 的生成能力类似，但执行功能（调度层）薄弱，导致想法无法顺畅落地（来源：主题综述）。同样，没有 harness 的 LLM 只是一个无状态的 API 调用，无法自主规划与执行长任务（来源：无状态与外部记忆）。

因此，一个自然的问题是：**如果把 ADHD 大脑当成生成核心，Claude 能不能当它的规划循环与任务分解？** 答案是：能，但需要精心设计的“脚手架”——外部系统提供记忆、调度和约束，补偿执行功能缺陷。

## 同构一：无状态与外部记忆——Claude 作为“第二大脑”

ADHD 大脑的工作记忆容量有限且不稳定，导致跨会话任务难以持续。典型表现为：每天需要重新回忆项目上下文，否则容易偏离轨道（来源：无状态与外部记忆）。为此，ADHD 个体常借助“第二大脑”工具——例如基于 AI 的外部记忆系统，能存储客户历史、项目背景、质量标准和个人偏好（来源：无状态与外部记忆）。

在 LLM 一侧，标准的 LLM 调用是无状态的：每次对话都是独立的，模型不保留跨会话的记忆（来源：无状态与外部记忆）。生产级 agent 系统通过外部记忆（如向量数据库）解决溢出问题，并组合使用会话内上下文记忆、长期检索向量库和结构化知识库（来源：无状态与外部记忆）。

**同构对应**：ADHD 需要一个“永远在线”的外部记忆体来补偿内部工作记忆的波动（来源：无状态与外部记忆）；LLM 需要 harness 提供持久记忆。Claude 的对话历史功能或第三方工具（如 Saner.AI）可以充当这个外部记忆，将项目背景、个人偏好等信息持久化，减少每日重复输入上下文的摩擦。

## 同构二：上下文工程——Claude 帮你管理“当下注意什么”

ADHD 大脑常被描述为“高产但缺乏可靠执行调度层的生成核心”，其上下文管理面临两大挑战：易被环境带偏，时间感知扭曲；任务碎片化，使用通用聊天机器人时，“they often add another inbox, another prompt, and another place to lose the thread”（来源：上下文工程）。

LLM agent 同样面临上下文管理问题：上下文膨胀与推理退化，“Effective autonomous assistance requires strict safety controls and highly efficient context management to prevent context bloat and reasoning degradation”（来源：上下文工程）。

**同构对应**：ADHD 需要外部工具来主动管理“当下注意什么”，例如通过 Routinery 等应用“create a sequence of steps for a routine and walk you through them with transition prompts”（来源：上下文工程）。Claude 可以通过精心设计的提示词（prompts）引导用户逐步完成任务，类似 agent 的 planner 模块。例如，使用“AI Prompts for Task and Time Management”将复杂任务分解为可执行步骤（来源：上下文工程）。

## 同构三：规划循环与任务分解——Claude 作为外部调度层

ADHD 的规划困难源于执行功能缺陷，尤其是任务分解能力不足。工具如 Motion 通过自动排程与动态调整，持续评估任务优先级、截止日期和可用时间，实时重建日程，减少用户的手动规划压力（来源：Motion）。Reclaim.ai 则通过创建深度工作、习惯和专注时间的智能块，自动防御会议侵占，帮助用户维持日程的稳定性（来源：Reclaim.ai）。

在 agent 工程中，planner-executor 循环是核心模式：planner 将长任务分解为子步骤，executor 逐步执行，并根据反馈调整计划。Claude 可以扮演 planner 角色：用户输入一个模糊目标（如“写一篇关于 AI 与 ADHD 的文章”），Claude 将其分解为“确定论点→收集资料→列出大纲→撰写初稿→修改润色”等步骤，并引导用户逐一完成。

**同构对应**：ADHD 大脑需要外部调度层来补偿执行功能缺陷；LLM 需要 harness 提供规划循环。Claude 通过对话式交互，充当这个外部调度层，将用户从“下一步该做什么”的决策负担中解放出来。

## 核心观点：脚手架 vs 拐杖——Claude 应是可撤除的支架

我的核心判断是：**Claude 作为 ADHD 的规划循环与任务分解器，其价值在于提供“脚手架”而非“拐杖”**。脚手架是临时的、可逐步撤除的支撑，帮助用户发展内在能力；拐杖是永久依赖，长期使用可能阻碍内在执行功能的发展（来源：矛盾与存疑）。

当前多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这是危险的：过度依赖外部工具可能阻碍 ADHD 患者内在执行功能的发展（来源：主题综述）。理想的设计应包含“撤除计划”：例如，随着用户使用，Claude 逐渐减少提示频率，让用户自主规划。

同时，必须承认争议：ADHD 大脑与 LLM 同构命题的证据主要来自概念类比和工具案例，缺乏大规模实证（来源：矛盾与存疑）。多巴胺干预的有效性也存在争议（来源：矛盾与存疑）。因此，本文观点是理论框架，需更多实证验证。

1. **用 Claude 分解一个模糊任务**：输入“我需要整理房间，但不知道从哪里开始”，让 Claude 列出 3-5 个具体步骤，并设定每个步骤的完成标准。
2. **创建外部记忆库**：在 Claude 对话中，每次开始新任务前，先粘贴一段“项目背景”文本，要求 Claude 在后续对话中始终参考。或者使用 Todoist 等工具存储任务上下文（来源：无状态与外部记忆）。

Claude 不是万能药，但作为外部规划循环，它能让 ADHD 大脑的“高产生成核心”真正运转起来。关键在于：把它当作脚手架，而不是拐杖。

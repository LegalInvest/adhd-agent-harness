# 如果把 ADHD 大脑当成生成核心，Claude 能不能当它的规划循环与任务分解？

> 用 agent 工程的思路重新理解 Claude 对 ADHD 的真正价值。

## 问题：为什么任务启动和分解对 ADHD 如此困难？

ADHD 大脑常被描述为“高产但缺乏可靠执行调度层的生成核心”（来源：[[ADHD 大脑与 LLM 的同构]]）。其执行功能缺陷——尤其是[[工作记忆]]容量有限但控制弱、[[时间盲]]、[[任务启动]]困难——导致一个典型困境：大脑里想法丰富，但一旦要动手，就卡在“第一步该做什么”上。任务分解能力缺失，是因为工作记忆无法同时 hold 住目标、子步骤和进度，同时时间感知扭曲让“还剩多少时间”变得模糊（来源：[[时间盲]]）。

## 同构：ADHD 大脑与 LLM 的“无状态”困境

有趣的是，LLM 面临完全相同的问题。标准的 LLM 调用是无状态的：每次对话都是独立的，模型不保留跨会话的记忆（来源：[[无状态与外部记忆]]）。没有 harness 的 LLM 只是一个无状态的 API 调用（来源：Building an AI Agent Harness from Scratch）。这意味着，LLM 无法自主记住之前说了什么、计划了什么，就像 ADHD 大脑每天需要重新回忆项目上下文，否则容易偏离轨道（来源：[[无状态与外部记忆]]）。

生产级 agent 系统通过外部记忆（如向量数据库）解决这个问题，并组合使用会话内上下文记忆、长期检索向量库和结构化知识库（来源：The Anatomy of an AI Agent）。这正是 ADHD 大脑需要的“第二大脑”——一个能存储客户历史、项目背景、质量标准和个人偏好的外部记忆系统（来源：Best AI Tools for ADHD Productivity in 2026）。

## Claude 作为规划循环：从“生成核心”到“执行调度”

如果把 ADHD 大脑视为生成核心（擅长产生想法但缺乏执行调度），那么 Claude 这样的 LLM 可以充当规划循环与任务分解层。具体来说，Claude 的上下文窗口和指令遵循能力，可以像 agent 的 planner 一样：

1. **接收高目标**（例如“写一篇报告”），
2. **分解为子任务**（“搜集资料→列提纲→写初稿→修改”），
3. **输出可执行的下一步指令**，
4. **根据反馈调整计划**。

这本质上就是 agent 的 planner-executor 循环。ADHD 用户可以将大脑中的想法“卸载”给 Claude，让 Claude 承担任务分解和步骤排序的工作，从而降低启动门槛。

## 真实工具的证据：外部执行功能层

现有 AI 工具已经在实践这一思路。例如：
- **Motion**：自动根据任务、会议和截止日期创建并动态调整每日计划，消除“下一步该做什么”的决策负担（来源：[[Motion]]）。用户报告其动态日程调整功能显著减少决策疲劳。
- **Reclaim.ai**：通过创建深度工作、习惯和专注时间的智能块，自动防御会议侵占，帮助维持日程稳定性（来源：[[Reclaim.ai]]）。其“防御性”时间块设计有助于维持工作记忆负荷，避免任务切换损耗。
- **Tiimo**：将时间转化为可见元素，通过结构、颜色和视觉惯例组织生活，直接应对时间盲（来源：[[Tiimo]]）。
- **Todoist**：智能优先级排序基于任务属性和截止日期，充当外部记忆（来源：[[Todoist]]）。

这些工具的本质都是“给生成核心套 harness”——即外部执行功能层（来源：[[外部执行功能层]]）。

## 核心观点：脚手架 vs 拐杖的边界

我的判断是：**Claude 作为规划循环的价值，在于它能成为“脚手架”而非“拐杖”。** 脚手架是临时结构，帮助用户建造自己的能力；拐杖则是永久替代，削弱内在能力。

关键区别在于：
- **脚手架**：用户主动参与分解过程，理解 Claude 给出的步骤逻辑，逐渐内化分解能力。例如，让 Claude 先输出分解框架，用户再调整，而不是直接执行。
- **拐杖**：用户完全依赖 Claude 的规划，不思考为什么这样分，导致时间感知和规划能力进一步退化。

现有资料警告：过度依赖 AI 工具可能削弱内在时间感知能力（来源：[[时间盲]]），且缺乏长期追踪研究（来源：[[矛盾与存疑]]）。因此，使用 Claude 时必须保持“主动参与”的姿态。

## 争议与局限

1. **证据不足**：AI 工具对 ADHD 的有效性主要基于用户报告，缺乏大规模对照实验（来源：[[矛盾与存疑]]）。Motion、Reclaim.ai 等工具的有效性证据多来自用户反馈，而非临床研究。
2. **个体差异**：ADHD 亚型（注意力缺陷 vs. 多动冲动）对工具响应不同，现有工具多面向注意力缺陷型（来源：[[AI 与 ADHD 的时间管理]]）。
3. **同构理论的实证不足**：ADHD 大脑与 LLM 的同构仍属理论类比，神经科学和计算机科学的交叉验证不足（来源：[[AI 与 ADHD 的时间管理]]）。
4. **Claude 的局限**：Claude 本身无状态，需要外部记忆系统（如向量数据库）才能实现持久规划循环，否则每次对话都是新的开始。

## 今天就能试的行动

1. **用 Claude 分解一个任务**：写下你目前卡住的一个任务（比如“整理书桌”），让 Claude 输出 3-5 个子步骤，并标注每个步骤的预计时间。执行后对比自己分解的差异。
2. **建立外部记忆**：在 Claude 中创建一个项目文档，记录当前任务上下文、常用偏好和已完成步骤。每次对话前先粘贴，模拟“持久上下文”。
3. **试用 Motion 或 Reclaim.ai**：选择一款自动排程工具，使用一周，记录任务启动时间的变化。注意观察是否减少了决策疲劳，但也要警惕是否产生了依赖感。
4. **反思“脚手架 vs 拐杖”**：每次使用工具后，问自己：“我是否理解了工具为什么这样安排？我能否在不使用工具时也做出类似规划？”如果答案是否定的，说明你在依赖拐杖，需要调整使用方式。

## 结语

ADHD 大脑和 LLM 是同一类“生成核心”——高产但缺乏执行调度层。Claude 作为规划循环，可以弥补这个缺口，但前提是把它当作脚手架，而不是拐杖。工具的价值在于补偿，而非替代。正如外部记忆系统帮助 LLM 实现持久推理，外部执行功能层也能帮助 ADHD 大脑实现持续行动。关键在于，我们是否愿意主动参与这个“harness”的搭建过程。

# ADHD 创业者视角：为什么「治好 ADHD 的分心走神或超聚焦跑偏，丢失目标」和「让 LLM 不跑飞」其实是同一道工程题？

> 长程任务里 agent 会目标漂移，需要重锚定 system prompt——同一套 harness 思路，两边都成立。

你打开一个项目，灵感喷涌，三小时后却发现自己正在刷电商 App 的评论区，完全记不起怎么走到这一步。另一边，你部署了一个 LLM agent 去写一份市场报告，十分钟后它开始用维基百科编造不存在的公司，自信满满地跑题。这两件事，表面上是“分心”和“幻觉”，底层却是同一道工程题：**生成核心跑飞了，需要重锚定**。

## 同一个结构：高产但缺调度层的生成核心

ADHD 大脑被描述为“高产但缺乏可靠执行调度层的生成核心”（来源：AI 与 ADHD 的专注力）。你创造力爆棚，但执行功能——任务启动、时间感知、注意力维持——像一台没有操作系统的超级计算机。LLM 也类似：它能生成流畅文本，但缺乏内置的上下文控制，输出容易漂移，甚至产生幻觉。

两者的同构不止于此。ADHD 的工作记忆缺陷与 LLM 的无状态性对应——都需要外部记忆来维持连续性。ADHD 易被环境带偏，LLM 易被上下文膨胀干扰，都需要主动设计“当下注意什么”的工程方案（来源：上下文工程）。换句话说，ADHD 大脑和 LLM 都需要一个 **harness**——外部执行功能层，来把生成能力绑定到目标上。

## 目标漂移：ADHD 和 agent 共同的敌人

ADHD 侧的目标漂移有两种典型：一是分心走神，被新刺激捕获；二是超聚焦跑偏，在一个子任务上过度沉浸，忘记原始目标。LLM agent 同样面临两种漂移：一是上下文膨胀导致推理退化，模型在长对话中丢失原始指令；二是工具调用失控，agent 反复调用同一个 API 而不检查是否偏离主任务（来源：上下文工程；幻觉与验证循环）。

证据是双向的。ADHD 侧，Focusmate 通过实时同伴问责提供外部结构，多项用户研究显示它能显著提升任务启动和维持（来源：Focusmate）。LLM 侧，harness 工程中的“验证与 CI 集成”和“结构化工作流”要求将复杂任务分解为子任务，并在每一步进行规划与验证，否则会导致“更多幻觉和重复工作”（来源：幻觉与验证循环）。两边用的都是同一套思路：**在生成过程中插入检查点，把目标重新锚定回来**。

## 重锚定工具：脚手架，不是拐杖

具体怎么做？ADHD 个体可以用 Routinery 创建步骤序列，用过渡提示把自己一步步带回来（来源：上下文工程）。RescueTime 自动记录时间使用，帮助识别注意力漂移模式，对抗“人在心不在”的状态（来源：RescueTime）。LLM agent 则需要严格的上下文管理和验证循环：例如在每步输出后检查格式、运行测试用例，或通过自我批评提示监控行为（来源：幻觉与验证循环）。

但这里有一条红线：**脚手架 vs 拐杖**。脚手架是临时结构，帮你建起自己的能力；拐杖是永久替代，让你越来越依赖。AI 工具作为外部执行功能层，长期使用可能导致内在能力退化（来源：矛盾与存疑）。Focusmate 依赖实时视频，可能对社交焦虑用户造成压力；RescueTime 的自动分类不够精准，过度关注数据可能引发焦虑（来源：Focusmate；RescueTime）。Brain.fm 宣称通过神经锁相影响专注状态，但缺乏针对 ADHD 的独立临床研究（来源：Brain.fm）。

所以，关键不是工具本身，而是**你如何使用它**。好的 harness 应该像健身房的保护带——在你力竭时托住你，但不会替你发力。当你发现离开工具就无法启动任务时，就需要警惕了。

## 局限与争议

必须诚实指出，这套同构是隐喻而非严格等价。LLM 的“注意力机制”与人类注意力有本质差异（来源：AI 与 ADHD 的专注力）。此外，ADHD 的异质性使得同一工具对不同亚型效果迥异，现有证据多来自用户报告，缺乏大规模随机对照试验（来源：矛盾与存疑）。

## 今天就能试的 3 条行动

1. **给自己设一个“验证循环”**：每完成一个小任务（比如写一段文字），停 30 秒问自己：“我现在做的事，和最终目标一致吗？”——这和 LLM agent 的“每步验证”是同一逻辑。
2. **用外部记忆代替工作记忆**：打开 RescueTime 或类似工具，让自动记录帮你感知时间流向，而不是靠大脑硬记。
3. **找一个身体在场伙伴**：试试 Focusmate 或类似服务，预约 25 分钟的共同工作时段。外部问责比意志力更可靠。

ADHD 大脑和 LLM agent 都是强大的生成引擎，但引擎需要方向盘。别试图“治好”跑飞，而是学会设计重锚定的系统。工程题，就用工程解法。

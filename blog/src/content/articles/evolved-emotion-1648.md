# 为什么用 Todoist 治 ADHD 的情绪失调，和给 agent 套 会褪去的脚手架 是一回事？

> Todoist 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

## 问题：为什么 Todoist 治不好情绪失调？

如果你有 ADHD，你一定经历过这种时刻：Todoist 里列了 20 项任务，每一项都「重要且紧急」，但你盯着屏幕，手指悬在键盘上方，就是点不下去。不是懒，是大脑里像塞了一团棉花——工作记忆过载、时间感知失灵、启动焦虑像潮水一样涌上来。你打开 Todoist 本想「整理一下」，结果半小时后你在刷 Reddit，情绪从自责滑向麻木。

这是情绪失调的典型场景。ADHD 的情绪调节困难，根源不在情绪本身，而在执行功能缺陷——尤其是 **工作记忆**、**任务启动** 和 **时间盲**（来源：AI 与 ADHD 的情绪调节）。当这些底层调度模块失效，情绪就成了失控的野马。

现在切换到工程师视角：你在做一个 LLM agent，它推理能力极强，能写诗、能 debug、能规划旅行。但你让它「帮我订一张下周去北京的机票，预算 2000 以内」，它先搜索了航班，然后突然开始写一首关于飞机的诗，接着又去查了北京天气，最后崩溃了——因为上下文窗口满了，它忘了最初的目标。你给它套了一个 Agentic Harness（编排框架），用任务分解、外部记忆、状态管理把它拉回正轨。

这两件事有什么共同点？**同一个问题，同一个解法**。

## 同构：高产但缺调度层的生成核心

ADHD 大脑与 LLM 共享一个脆弱性：**强大的生成核心 + 不可靠的调度层**（来源：ADHD 大脑与 LLM 的同构）。ADHD 大脑的智力不低，但执行功能（工作记忆、任务启动、时间感知）像一台老旧的调度器，经常丢包、超时、死锁。LLM 的推理能力惊人，但它的「执行功能」——上下文管理、任务编排、状态跟踪——同样脆弱，容易偏移目标或陷入循环。

情绪失调，正是调度层失灵的直接后果。当工作记忆过载，你无法同时记住「我要做什么」和「我为什么做」，焦虑和挫败感接管大脑（来源：认知负荷）。当时间盲发作，你以为只过了 5 分钟，实际已经刷了 2 小时，恐慌随之而来。LLM agent 的「情绪失调」表现为：在复杂任务中产生幻觉、重复步骤、丢失上下文——本质上也是调度层崩溃。

两边的解法结构同构：**外部执行功能层**（来源：AI 与 ADHD 的情绪调节）。对 ADHD 大脑，它是 Todoist、Goblin Tools、Saner.AI 这类工具，充当外部工作记忆和任务启动助推器。对 LLM agent，它是 Agentic Harness——用任务分解（类似 Goblin Tools 的 Magic ToDo）、外部记忆（类似 Saner.AI 的知识回忆）、状态管理（类似 Motion 的自动调度）来补偿调度缺陷。

## 证据：两边都成立

**ADHD 侧**：Goblin Tools 将「整理房间」分解为「捡起地板上的衣服」「擦桌子」等小步骤，直接降低启动门槛（来源：Goblin Tools）。Inflow 通过训练背外侧前额叶皮层（因果流入中枢）来强化工作记忆（来源：Inflow）。Saner.AI 用本地记忆存储减少搜索循环和标签切换（来源：Saner.AI）。这些工具的本质都是**外化认知负担**，减少工作记忆的压力（来源：认知负荷）。

**LLM/agent 侧**：任务分解技术将复杂指令拆分为子任务，显著降低 LLM 的认知负荷，减少幻觉（来源：认知负荷）。Agentic Harness 通过上下文工程和无状态设计，让 agent 不再依赖内部状态，而是依赖外部记忆——这正对应 ADHD 工具充当「数字工作记忆」的角色（来源：AI 与 ADHD 的情绪调节）。

**关键证据**：CHADD 文章明确指出，AI 工具通过自动决策和自动执行，移除用户「下一步做什么」的认知负荷（来源：认知负荷）。而在 agent 工程中，Harness 同样替 agent 决定「下一步调用哪个工具、何时调用」。两边都在做同一件事：**把调度权交给外部系统**。

## 核心观点：脚手架，不是拐杖

这是最重要的区分。**脚手架（scaffolding）是可撤除的临时支撑，拐杖（crutch）是永久替代**（来源：拐杖与脚手架）。Todoist 和 Agentic Harness 都应该是脚手架——它们补偿缺陷，但不应该替代内在能力的生长。

但现状是：多数工具（Goblin Tools、Saner.AI、Motion）被设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这带来过度依赖风险：如果你永远依赖 Todoist 告诉你「下一步做什么」，你的任务启动能力可能永远不会进步。同样，如果 agent 永远依赖硬编码的 Harness，它永远不会学会自主编排。

**我的判断**：好的脚手架应该内置「褪去计划」。例如，Todoist 可以逐步减少提醒频率，或者引入「无工具日」模式；Agentic Harness 应该动态调整干预强度，在 agent 表现稳定时自动减少外部约束。目前没有工具做到这一点，这是一个设计空白。

## 局限与争议

我必须诚实指出：ADHD 大脑与 LLM 的同构命题，目前主要基于概念类比和工具案例，**缺乏大规模实证**（来源：矛盾与存疑）。多巴胺干预的有效性也存在争议——直接调节多巴胺的 AI 工具（如奖励系统）可能强化依赖而非内在动机（来源：AI 与 ADHD 的情绪调节）。此外，长期效果证据缺乏，多数研究为短期观察（来源：AI 与 ADHD 的情绪调节）。

所以，本文的观点是一个**工作假说**，不是定论。但它提供了一个有价值的框架：如果你同时理解 ADHD 和 agent 工程，你会看到两边的解法在结构上惊人地一致。这种同构视角，可能比任何单一领域的理论都更有解释力。

## 今天就能试的行动

1. **给 Todoist 加一个「褪去计划」**：本周用 Goblin Tools 分解任务，下周尝试只写大任务标题，看自己能否独立完成小步骤。记录情绪变化。
2. **用 Saner.AI 替代搜索循环**：当你在浏览器里打开 10 个标签页找同一份资料时，把资料存进 Saner.AI 的本地记忆，下次直接检索，观察认知负荷的下降。
3. **给 LLM agent 加一个「状态检查」步骤**：在 agent 的每一步之后，强制它输出当前目标、已完成步骤、下一步计划。这相当于为 agent 加装一个外部工作记忆——与 ADHD 工具的原理相同。
4. **做一次「无工具日」实验**：选一天，不用任何 AI 或任务管理工具，只靠纸笔和大脑。记录情绪失调的频率和强度。这能帮你评估工具到底是脚手架还是拐杖。

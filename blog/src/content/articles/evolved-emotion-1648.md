# 为什么用 Todoist 治 ADHD 的情绪失调，和给 agent 套 会褪去的脚手架 是一回事？

> Todoist 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

## 问题：当生成核心失去调度层

你打开 Todoist，盯着“整理房间”这个任务，却感到一阵强烈的无力感。不是不知道怎么做，而是大脑像短路一样，无法将“整理房间”拆解成“捡起衣服→擦桌子→叠被子”的序列。这种情绪失调——焦虑、挫败、自我厌恶——不是因为你懒，而是因为你的执行功能（尤其是工作记忆）在那一刻过载了（来源：AI 与 ADHD 的情绪调节）。

另一边，一个 LLM agent 被要求“帮我写一篇关于 ADHD 的文章”。它生成了 5000 字的初稿，但中途偏离了主题，开始大谈多巴胺受体，最后以一段幻觉收尾。你检查日志，发现它“忘记”了最初的指令——上下文窗口被无关信息填满，执行调度层失效了。

ADHD 大脑与 LLM 是同一类“高产但缺执行调度层的生成核心”（来源：ADHD 大脑与 LLM 的同构）。两者的痛点都源于：生成能力旺盛，但缺乏一个可靠的调度层来将意图转化为有序行动。ADHD 的情绪失调，本质上是执行功能缺陷导致的“上下文失控”——当工作记忆过载或注意力被干扰时，情绪反应如同 LLM 的幻觉，脱离理性轨道（来源：AI 与 ADHD 的情绪调节）。

## 同构解法：外部 harness 与会褪去的脚手架

传统应对 ADHD 情绪失调的方法是“硬扛”——训练意志力、做认知行为疗法。但效果有限，因为底层缺陷是神经性的。同样，传统应对 LLM 幻觉的方法是“训更多数据”——但上下文失控的问题，根源在于缺乏执行调度。

两边的工程解法惊人地同构：**给生成核心套上一个外部 harness**。

对于 ADHD 大脑，这个 harness 就是 Todoist、Goblin Tools、Saner.AI 这类工具。它们的工作机制是：

- **自动决策**：AI 代替用户决定“下一步做什么、何时做、如何优先排序”，移除决策带来的认知负荷（来源：认知负荷）。
- **自动执行**：AI 自动采取行动，减少用户需要记住和执行的步骤（来源：认知负荷）。
- **任务分解**：Goblin Tools 的 Magic ToDo 将“整理房间”自动拆成“捡起地板上的衣服→擦桌子→叠被子”，把压倒性的事情变成一系列不压倒性的事情（来源：Goblin Tools）。

当 Todoist 接管了“下一步做什么”的调度，你的工作记忆就不再被任务规划占据，情绪也就不会因为“不知道从何下手”而崩溃。这就是为什么用 Todoist 治 ADHD 情绪失调有效——它不是一个情绪调节工具，而是一个执行功能支架。

对于 LLM agent，这个 harness 就是“会褪去的脚手架”——通过上下文工程主动设计注意焦点，利用无状态与外部记忆记录任务状态，实现输出的可预测性（来源：AI 与 ADHD 的情绪调节）。例如，给 agent 套一个任务分解链：先“列出文章大纲”，再“写引言”，再“写第一个论点”——每一步都限制上下文范围，避免幻觉。这与 Goblin Tools 的分解逻辑完全相同。

## 实证：两边都成立

ADHD 侧的证据主要来自用户报告和初步研究。Inflow 通过训练工作记忆来改善执行功能，其靶点——背外侧前额叶皮层——的功能连接性与工作记忆表现正相关（来源：Inflow）。Goblin Tools 被用户评价为“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools）。Saner.AI 通过本地记忆减少搜索循环，直接降低认知负荷（来源：Saner.AI）。

LLM 侧的实证更加硬核。用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中表现正常，但随序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突。这与 ADHD 执行功能缺陷的核心标志一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃（来源：认知负荷）。这为同构提供了最硬的实证之一。

## 核心观点：脚手架 vs 拐杖的边界

但这里有一个关键陷阱：**脚手架会褪去，拐杖不会**。

“会褪去的脚手架”意味着：这个外部 harness 的目标是补偿缺陷，而不是替代能力。对于 ADHD 用户，Todoist 应该帮你学会任务分解，而不是让你永远依赖它来思考。对于 LLM agent，上下文工程应该逐步减少人工干预，而不是永远需要手写提示。

然而，现有工具的设计往往模糊了这个边界。多个 wiki 页面警告：过度依赖 AI 工具可能削弱内在时间感知能力（来源：时间盲），存在依赖风险（来源：任务启动、AI 与 ADHD 的情绪调节）。矛盾与存疑页面更直接指出：“工具设计者声称是‘脚手架’，但实际使用中可能沦为‘拐杖’。”（来源：矛盾与存疑）

我的判断是：**关键在于工具是否提供了“褪去机制”**。一个好的脚手架应该自带“退出策略”——比如 Todoist 可以逐渐减少自动分解的粒度，让用户自己补完步骤；Inflow 的训练任务是动态调整难度的，目标是增强而非替代工作记忆（来源：Inflow）。而一个坏的拐杖则永远替你决策，让你越来越依赖。

## 局限与争议

必须诚实指出：现有证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑）。Inflow 的神经科学依据来自基础研究，但缺乏直接针对工具的随机对照试验（来源：Inflow）。Brain.fm 的 ADHD 效果缺乏独立临床研究支持（来源：矛盾与存疑）。个体差异也很大——ADHD 亚型不同，对工具的反应差异大（来源：AI 与 ADHD 的情绪调节）。

此外，基于多巴胺的干预（如游戏化反馈）可能强化成瘾行为，需谨慎设计（来源：AI 与 ADHD 的情绪调节）。长期效果和神经可塑性变化缺乏追踪。

## 今天就能试的行动

1. **用 Todoist 或 Goblin Tools 分解一个“压垮你”的任务**：比如“整理房间”。输入后，看 AI 拆出的步骤。如果步骤仍然太大，手动再拆一层。记录分解前后的情绪变化——焦虑是否下降？启动是否更容易？

2. **给你的 LLM agent 套上“会褪去的脚手架”**：下次用 ChatGPT 写东西，先让它“列出大纲”，再“写第一部分”，而不是一次性给全部指令。观察输出质量是否提升、幻觉是否减少。

3. **设置一个“褪去提醒”**：如果你每天用 Todoist 自动分解任务，每周选一天手动分解一个任务，不依赖 AI。记录自己的表现——这是检验脚手架是否在褪去的方法。

4. **测试自己的 Stroop 式崩溃**：在认知负荷高时（比如同时处理三个任务），尝试做一个简单的冲突决策（比如“选红色但字是蓝色”）。你能抑制优势反应吗？如果不能，说明你的执行调度层正在过载——这正是 Todoist 该介入的时候。

## 同构的启示

为什么用 Todoist 治 ADHD 的情绪失调，和给 agent 套会褪去的脚手架是一回事？因为两者面对的是同一个问题：一个强大的生成核心，却缺少一个可靠的执行调度层。解决方案也同构：外部 harness 补偿调度缺陷，但必须设计褪去机制，避免从脚手架沦为拐杖。

下次当你因为“不知道从何下手”而情绪崩溃时，不要责怪自己。你的大脑只是缺了一个调度层。而 Todoist——或者任何外部 harness——就是那个调度层。用得好，它是脚手架；用不好，它是拐杖。选择权在你。

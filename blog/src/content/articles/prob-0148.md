# 用 ChatGPT 给 ADHD 补上无状态与外部记忆，是脚手架还是拐杖？

> ChatGPT 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

你打开 ChatGPT，想继续昨天写到一半的方案。但窗口空空如也——它不记得你上周讨论过的需求、你反复强调的风格偏好、甚至你五分钟前刚问过的问题。你只好重新输入上下文，边打字边忘，最后烦躁地关掉标签页。

这一幕，ADHD 群体再熟悉不过：工作记忆容量小、跨会话任务难以持续、每天需要重新回忆项目背景，否则偏离轨道（来源：无状态与外部记忆）。而 LLM 工程师也面临同样的困境：标准 LLM 调用是无状态的，每次对话独立，模型不保留跨会话记忆（来源：无状态与外部记忆）。没有 harness 的 LLM 只是一个无状态 API 调用（来源：拐杖与脚手架）。

两个看似无关的世界，共享一个核心矛盾：**强大的生成核心，缺乏可靠的执行调度层**。ADHD 大脑的智力与创造力，与 LLM 的生成能力类似，但两者都缺乏稳定的执行功能——计划、组织、工作记忆。于是，双方都在寻找“外部脚手架”：ADHD 用户需要第二大脑，LLM 工程师需要 harness。

## 外部记忆：从“第二大脑”到向量库

ADHD 大脑的工作记忆容量有限且不稳定。典型表现是：每天需要重新回忆项目上下文，否则容易偏离轨道。为此，ADHD 个体常借助 AI 外部记忆系统，存储客户历史、项目背景、质量标准和个人偏好（来源：无状态与外部记忆）。例如 **Mem** 和 **Saner.AI** 提供持久记忆，减少搜索循环；**Otter.ai** 实时转录减轻笔记负担（来源：ADHD 的 AI 工具全景）。这些工具本质上是一个“永远在线”的外部记忆体，补偿内部工作记忆的波动（来源：无状态与外部记忆）。

LLM 工程侧的对等物是向量数据库与检索增强生成（RAG）。生产级 agent 系统通过外部记忆（如向量库）解决上下文溢出问题，组合使用会话内上下文、长期检索向量库和结构化知识库（来源：无状态与外部记忆）。检索增强和工具调用将决策锚定在外部证据和持久状态上（来源：无状态与外部记忆）。

这并非巧合。ADHD 大脑与 LLM 在结构上同构：都是强大的生成核心，但缺乏内在的执行调度。因此，“AI 帮 ADHD”的本质是**给生成核心套 harness**——通过外部工具提供执行功能、记忆管理和上下文控制，弥补内在缺陷（来源：ADHD 的 AI 工具全景）。

## 脚手架 vs 拐杖：边界在哪里？

**ChatGPT 作为对话式工具，可以充当执行功能教练或创意伙伴，但需用户主动设计提示（来源：ADHD 的 AI 工具全景）。** 它不会自动记住你的喜好，不会主动分解任务——除非你告诉它怎么做。这正是关键：工具本身是中性的，但使用方式决定了它是脚手架还是拐杖。

真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：拐杖与脚手架）。例如 **Goblin Tools** 的 Magic ToDo 能将“整理房间”自动分解为“捡起地板上的衣服”“擦桌子”等步骤，降低启动门槛（来源：Goblin Tools）。这相当于一个任务分解助手，但执行仍靠用户。**Lex** 允许通过单一指令触发复杂多步骤序列，减少决策负担（来源：Lex）。这些工具外化思维，减轻认知负荷，但并未替代用户的自主决策。

拐杖则相反：工具替代了治疗或自我管理，造成依赖（来源：拐杖与脚手架）。专家警告，若 AI 工具成为永久替代而非临时支撑，可能阻碍内在执行功能的发展（来源：拐杖与脚手架）。目前多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这是当前最显著的争议：**工具设计是否考虑可撤除性？长期依赖是否阻碍内在能力发展？**（来源：矛盾与存疑）

## 核心判断：设计可撤除的 harness

我的观点是：**ChatGPT 等 AI 工具作为外部脚手架，其价值取决于用户是否主动设计使用方式，并预留撤除路径。** 最站得住脚的用法是：将工具视为“执行功能层”，而非“大脑替代”。就像 LLM harness 在首次提示前组装代理，运行时编排工具调度，用户也应主动设计自己的“提示系统”——哪些任务交给 AI，哪些必须自己留着。

对于 ADHD 用户，这意味着：
- 用 ChatGPT 分解任务，但执行仍靠自己；
- 用外部记忆存储上下文，但定期回顾以强化内在记忆；
- 逐步减少对工具的依赖，就像物理治疗中逐步撤除拐杖。

对于 LLM 工程师，这意味着：
- 设计 harness 时考虑记忆系统的可配置性，允许用户决定哪些信息持久化；
- 提供“遗忘”机制，避免过度依赖外部记忆；
- 将脚手架设计为可逐步撤除的临时支撑，而非永久绑定。

1. **用 ChatGPT 做任务分解**：输入一个让你感到压倒性的任务（如“准备季度报告”），要求 ChatGPT 分解为 5-10 个小步骤。然后手动执行第一步。这相当于一个轻量级的 Goblin Tools。（来源：Goblin Tools）
2. **建立 ChatGPT 的“项目记忆”**：在对话开始时，用一句话告诉它项目背景和你的偏好（如“我是 ADHD 用户，请用简短步骤，避免信息过载”）。每次新对话重复此步骤，直到你习惯这种上下文工程。
3. **尝试 Saner.AI 的知识回忆**：将你频繁遗忘的信息（如客户名字、常用设置）存入 Saner.AI，减少搜索循环。每周回顾一次存储内容，尝试凭记忆复述，以训练内在记忆。（来源：Saner.AI）
4. **设置“撤除日”**：每周选一天，不用任何 AI 工具完成一项常规任务（如写邮件、整理日程）。记录感受，对比使用工具时的效率与认知负荷。这能帮你判断工具是脚手架还是拐杖。

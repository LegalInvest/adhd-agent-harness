# ADHD 职场人视角：为什么「治好 ADHD 的冲动下结论、想当然」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 自信地编造（幻觉），靠验证与自我批判兜底——同一套 harness 思路，两边都成立。

## 问题：为什么“冲动下结论”和“LLM 幻觉”听起来像同一个 bug？

如果你是一名 ADHD 职场人，可能对这一幕很熟悉：会议中一个念头闪过，你立刻拍板；下班复盘才发现，自己把“熟悉”当成了“正确”，把“第一个想法”当成了“唯一答案”。这不是性格急躁，而是大脑的“默认生成模式”太猛，而“执行调度层”常常掉线。

如果你是一名做 Agentic Harness 的工程师，这一幕同样熟悉：LLM 在回答时自信、流畅、头头是道，但它只是在生成语言——它会把没发生过的会议说成“我们上周讨论过”，把没读过的论文说成“据 Smith 等人 2023 年的研究显示”。它的生成核心极强，却没有一个内置的执行调度层来区分“生成”与“核实”。

两边的痛苦形态不同，但结构相同：一个高产但缺执行调度层的生成核心。治好 ADHD 的冲动下结论、想当然，和让 LLM 不跑飞，其实是同一道工程题。

## 同构脊柱：两个缺了调度层的生成核心

### ADHD 侧：强记忆、弱控制

ADHD 大脑并非“不会想”，而是“生成过量、控制不足”。研究发现，ADHD 的认知剖面是“强记忆、弱控制”：工作记忆容量甚至可能超过常人，但认知灵活性、任务切换和抑制自动化反应的能力存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。换句话说，我们不是记不住，而是无法在多个念头中“喊停”并重新排序。

更底层的解释是“注意力资源的弥散分配”：默认模式网络过度活跃，注意力像一束散开的灯，照得到很多地方，却稳不住目标（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。结果是“想法很多，行动困难”，以及一个更隐蔽的职场症状：在信息不完整时快速下结论，因为大脑急于把弥散的注意力收束到一个可执行的答案上。

### LLM 侧：无状态、缺调度

LLM 也是高产生成核心。但它是无状态的，没有可靠的执行调度层。把 LLM 变成 agent，要做的不是换更大的模型，而是搭“agent scaffolding”——围绕 LLM 的软件架构、工具接口、记忆、决策逻辑和控制循环（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。这也被称为“harness 工程”，即设计上下文交付、工具接口、规划工件、验证循环、记忆系统和沙箱（来源：GitHub - ai-boost/awesome-harness-engineering）。

最刺眼的同构证据来自冲突控制实验。研究者用经典 Stroop 任务测试 Transformer 注意力，发现短上下文里表现正常，但序列一长，模型在冲突试次上骤降到随机水平——它无法抑制优势反应，无法解决认知冲突，这与 ADHD 执行功能缺陷的核心标志几乎一一对应（来源：Deficient Executive Control in Transformer Attention）。

## 从脚手架到 harness：真实工具长什么样

既然 ADHD 大脑和 LLM 都需要外部调度层，那么工具的目标就不是“替你做决定”，而是“减少决策、保留上下文、让下一步行动显而易见”。

- **Goblin Tools** 的 Magic ToDo 把模糊任务（如“清理厨房”）切成可执行的小步骤，并可调节“辣度”控制粒度。它降低的不是任务难度，而是 ADHD 最卡的那个点——任务启动（来源：Goblin Tools）。
- **Saner.AI** 强调“本地记忆”和“知识回忆”，把笔记、任务、邮件聚合在一个收件箱里，减少标签切换和搜索循环——这正是 ADHD 工作记忆易掉线的对症方案（来源：Saner.AI）。
- **Motion** 用 AI 自动排程，根据优先级、截止日期、可用时间动态重建日程，并在任务可能延期前数周预警。它的价值不是“帮你做计划”，而是消除“下一步该做什么”的决策负担（来源：Motion）。

这些工具在 ADHD 侧是“外部执行功能层”，在 LLM 侧对应的是向量数据库、RAG、规划器、工具调用（MCP）和验证器。结构同构。

## 名人 harness：当外部调度层成为系统

最能说明“同构”的不是工具，而是人把 harness 做成了系统。

**毛泽东**的 ADHD 特质在 wiki 案例中有明确描述：小时候是问题儿童、爱与父亲冲突、读闲书、行军路上带着书、思维跳跃、讲话充满比喻，却又敢下险棋（四渡赤水、抗美援朝）。他的专属 harness 不是克制冲动，而是给冲动加外部调度层：
- **“批评与自我批评” + 民主生活会**：让外部成为事实检查器，对应 LLM 的 critic/self-critique 验证循环；
- **“没有调查，没有发言权”**：用实地调查把结论锚定在真实数据上，对应 LLM 的 RAG/检索增强；
- **民主集中制**：用集体决策制衡个人冲动，对应 LLM agent 的多投票或规划器；
- **“党指挥枪”**：用组织系统管理执行，对应 harness 的编排层与工具接口。

**杨振宁**则是另一种 harness：物理直觉极强，但每天固定时间看书想问题，不做没用的事，并以一生稳定的朋友圈维系长期认知环境。这对应 LLM 的“定时 re-grounding”与“稳定的长期记忆上下文”。

两个人的共同点是：他们都没有试图消灭“高产生成核心”，而是搭建了外部系统，让生成核心有地方落地、有机制验证、有人或规则兜底。

## 核心判断：不要“治愈”生成核心，要给它装上验证循环

我的判断是：对 ADHD 和 LLM 来说，最糟糕的干预是“压制生成”，最好的干预是“外包执行”。

冲动下结论和 LLM 幻觉的共同点不是“错”，而是“在没有调度层的情况下直接输出”。ADHD 大脑需要的是一个外部化的“暂停—检索—核对—提交”流程；LLM 需要的是 retrieval + critic + verifier + 规划循环。两者都是 harness，而不是拐杖——关键区别在于它是否可拆卸、是否最终训练你的元认知。

这一点在创业数据中也有呼应：ADHD 人群的创业概率是对照组的 6.25 倍（来源：Journal of Attention Disorders 2019）。生成核心本身就是资源，问题只是它需要一个 harness 来把“高风险跳跃”变成“可验证的冒险”。

## 争议与局限：同构是框架，不是定论

这个框架很诱人，但仍有边界。首先，ADHD 与 LLM 的同构目前主要是概念类比和初步实验，缺乏大规模随机对照试验和神经科学上的直接对应（来源：矛盾与存疑）。其次，工具到底是“脚手架”还是“拐杖”尚无定论：长期依赖 AI 执行功能层，可能让自身能力退化，也可能让大脑把节省下来的认知资源用于更高层次的判断（来源：矛盾与存疑）。

此外，个体差异常被低估。有人需要“柔性引导”来维持超聚焦，有人反而需要番茄钟式的硬性打断；有人用 AI 工具后认知负荷下降，也有人因为学习新工具而负担更重（来源：矛盾与存疑）。同构脊柱能解释“为什么”，但不能直接给出“什么适合你”。

## 今天就能试的 4 个动作

1. **给自己加一个“提交前验证”**：下一次准备回复邮件或会议表态前，先写下一句话“我现在下结论的依据是？”如果找不到外部依据，改为“我需要确认 X 后再回复”。
2. **把上下文外化到一个工具**：用 Saner.AI、Obsidian 或最简单的一张纸，把当前任务的目标、已做事项、下一步行动写在一起，降低工作记忆掉线的概率。
3. **给 LLM 加一层 critic**：在 prompt 或 agent 流程里强制加一步“先检索证据，再给出答案”，并要求模型列出“我最可能搞错的三点”。
4. **用 Goblin Tools 或纸笔拆分一个拖延任务**：把“写季度汇报”改成 5 个 5 分钟可启动的小步骤，重点是启动，而不是完美。

## 结语

ADHD 职场人和 LLM 工程师看似在两个完全不同的战场。但当你把“冲动下结论”和“幻觉”都看作“生成核心缺了调度层”的同一故障，解法就 converged 到一个词：harness。不是消灭跳跃，而是给跳跃装上护栏、地图和检查点。高产生成不是病，缺 harness 才是。

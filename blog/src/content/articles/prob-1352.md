# 用 Claude 给 ADHD 补上规划循环与任务分解，是脚手架还是拐杖？

> Claude 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：任务启动困难与不会拆解——ADHD 与 LLM 的共享困境

凌晨两点，你盯着空白的文档光标，大脑里有一百个想法在同时爆炸，但没有一个能被拽到指尖。这不是懒惰——是 ADHD 大脑的典型状态：一个强大的生成核心，却缺少内置的调度层。任务启动困难、不会拆解、时间盲（难以感知时间流逝），这些执行功能缺陷让最简单的事变成深渊。

同样的情况，在 AI 工程师的终端里以另一种形式上演：一个 LLM 模型能写出惊艳的段落，但让它完成一个多步骤任务（比如“写一篇研究综述并整理成表格”），它会在第 3 步迷失，忘记上下文，陷入循环。于是有了 Agentic Harness——用 planner-executor 循环把长任务分解成子步骤，每一步重新 grounding。

**ADHD 大脑与 LLM 是同一类“高产但缺执行调度层的生成核心”**（来源：AI 与 ADHD 的时间管理）。这个同构视角，正是我们理解 Claude 如何帮助 ADHD 的关键。

## 同构脊柱：规划循环与任务分解

### ADHD 侧的痛点：执行功能负荷超载

ADHD 的核心缺陷在于工作记忆容量有限，处理复杂信息时容易不堪重负（来源：认知负荷）。研究表明，用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中表现正常，但随序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突。这与 ADHD 执行功能缺陷的核心标志一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃（来源：认知负荷）。

换句话说，ADHD 大脑和 LLM 都在认知负荷增加时表现急剧下降。解决方案也同构：都需要一个外部调度层来接管“下一步做什么”的决策。

### LLM/Agent 侧的解法：planner-executor 循环

在 AI 工程中，Agentic Harness 通过 planner 模块将长任务分解为子任务，executor 逐一执行，每步后重新评估。这相当于给 LLM 装上了“执行功能”。例如，Claude 的 Projects 功能或自定义指令，本质上是在提示工程中嵌入规划循环。

### 人物案例：孔子的 harness 系统

孔子是 ADHD 特质的典型：身高 1 米 9，精力旺盛，周游列国 14 年坐不住；冲动爱骂人（骂宰予“朽木不可雕”）；对音乐超专注到“三月不知肉味”，对种地等俗事零耐心；思维跳跃，《论语》全是场景化语录，无系统著作。他的专属 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”），70 岁才做到“从心所欲不逾矩”。

这个系统与 LLM harness 同构：**“礼”就是外部调度层**，相当于 agent 框架中的 planner；“每日反省”相当于定时 re-grounding 的 checkpoint；“秘书/弟子”相当于外部记忆与提醒。孔子用这套 harness 驯服了自己的冲动，成就了儒家学派。

## 脚手架 vs 拐杖：边界在哪里？

### 脚手架：Claude 的合理使用

Claude 可以成为 ADHD 的“外部执行功能层”，通过以下方式降低认知负荷：
- **自动分解任务**：将“写一篇 3000 字报告”拆成“收集资料→列大纲→写引言→写主体→写结论→校对”，每一步给出具体提示。
- **消除决策负担**：像 Motion 自动排程一样，Claude 可以帮你决定“接下来做什么”，移除“该从哪开始”的决策疲劳（来源：认知负荷）。
- **视觉化时间线**：结合 Tiimo 的视觉化理念，Claude 可以生成带时间块的日程，让时间流逝变得可感（来源：Tiimo）。

这些用法属于“脚手架”——它在你能力不足时提供支撑，但你的大脑仍在学习如何规划。

### 拐杖：依赖的风险

但拐杖的边界也很清晰：如果 Claude 完全替你思考“为什么做这件事”“什么才是最重要的”，你就失去了元认知的锻炼机会。过度依赖 AI 工具可能从“脚手架”退化为“永久拐杖”，削弱用户内在执行功能的代偿发展（来源：AI 与 ADHD 的时间管理）。

**我的核心判断：保留主权，让 Claude 做调度层，你做决策层。** 让 Claude 分解任务、提醒时间、整理信息，但“要不要做”“怎么做才对”必须由你决定。这类似于 Reclaim.ai 的“防御性”设计——它保护你的时间，但不替你决定时间用来做什么（来源：Reclaim.ai）。

## 争议与局限

1. **证据不足**：目前多数 AI 工具缺乏独立临床验证，效果可能受安慰剂效应影响（来源：AI 与 ADHD 的时间管理）。
2. **个体差异**：ADHD 症状谱系广泛，某些用户可能对视觉化工具（如 Tiimo）反应良好，另一些则更受益于自动排程（如 Motion）。当前缺乏个性化匹配的指导原则（来源：AI 与 ADHD 的时间管理）。
3. **同构性仍是类比**：ADHD 大脑与 LLM 的同构说法缺乏实证基础，仅是一种理论类比（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Claude 分解一个“永远不想开始”的任务**：输入“帮我将‘整理房间’分解成 5 个可执行的子任务，每步不超过 15 分钟”，然后从第一个子任务开始。
2. **设置每日“反省”提示**：像孔子的“三省吾身”，每天用 Claude 复盘：“今天哪些任务没完成？卡在哪一步？明天如何调整？”
3. **用 Claude 创建视觉时间块**：告诉 Claude 你明天的待办事项，让它生成一个带颜色块的时间表（可用 Mermaid 语法），导入到 Tiimo 或日历中。
4. **保留一个“无 AI”决策时刻**：每天选一件小事（比如午餐吃什么），完全自己决定，不依赖任何工具，锻炼内在执行功能。

## 结语

Claude 不是 ADHD 的解药，而是一个精心设计的 harness。它像孔子的“礼”一样，给混乱的大脑一个外部结构。用好了，它是脚手架，帮你搭建自己的能力；用坏了，它是拐杖，让你永远依赖。关键在于：**你才是那个掌握缰绳的人。**

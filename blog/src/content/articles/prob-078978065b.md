# 两个互不引用的领域都在研究「执行功能」——ADHD 文献和 LLM harness 文献为什么得出了同一个结论？

> 做文献计量的人会对这种现象上瘾:两个几乎零互引的文献群,各自埋头几十年,然后你把它们的核心结论并排放——竟然是同一句话的两种方言。ADHD 研究(神经科学、临床心理,几十年积累)和 LLM harness 工程(近几年爆发的 agent 论文)就是这样一对:**前者说,ADHD 的核心不是「不知道该做什么」,是知道了也执行不出来;后者说,LLM 的核心短板不是「生成不出正确内容」,是生成得出也串不成可靠的长任务。** 两边给自己的解药,画出来是同一张图。

先把两边的结论各自说清。ADHD 侧,几十年研究的一条主线结论(Barkley 等人的执行功能理论是代表):**ADHD 是「知道」与「做到」之间的断裂**——智力完好、知识完好,受损的是把意图转译成时序行为的那组功能:抑制(挡住干扰)、工作记忆(在线保持目标)、计划(拆解与排序)、自我监控(边做边查)。临床上最有效的非药物干预,方向惊人一致:**不是教更多知识,是外部化这些功能**——外部提醒替代前瞻记忆、清单替代工作记忆、拆解替代内源计划、他人监督替代自我监控。

harness 侧,agent 论文这几年的核心发现,换掉术语后是同一句话:**LLM 的失败不在单步生成(单步能力惊人),在长程执行**——目标在长上下文中漂移(在线保持目标失败)、被无关信息带偏(抑制失败)、不会自查错误(自我监控失败)、长任务不拆就崩(计划失败)。而工程界给出的解药清单:system prompt 重注入、外部记忆、planner-executor 分离、验证器、护栏——**每一项都是把某个「执行功能」从模型内部搬到系统外部**。harness 这个词本身就是答案:不改模型,在外面套一层执行结构。

两边为什么零互引却同构?因为它们撞上的是**同一类系统的同一个短板**:一个强大的生成核心(联想、创造、模式补全)+ 一个薄弱的序列控制层(维持、抑制、监控、规划)。人脑的前额叶功能与更古老的生成系统之间的失衡,和 LLM「下一个词预测」的架构与长程任务需求之间的失衡——**机制完全不同(这必须说死:神经递质不是注意力权重,大脑不是 Transformer),但系统层面的功能缺口是同一个形状,所以两边各自独立地发明了同一形状的补丁。** 这正是 Swanson 文献发现的经典结构:A 与 C 互不引用,却共享中间机制 B——「执行功能的外部化」就是那个 B。

这个同构对两边各有一份礼物。**给 ADHD 用户的**:harness 工程是一座巨大的、快速迭代的「执行功能外部化」实验场——工程师们在用真金白银测试哪种拆解方式、哪种提醒频率、哪种验证结构真的有效,这些设计模式(本系列一百多篇写的就是这个)可以反向借用。**给 harness 工程师的**:ADHD 研究是几十年的先行文献——延迟折扣、时间知觉、奖励时序、干预的依从性衰减,这些人类侧被反复验证过的坑,agent 系统一个不少地正在重新掉进去;读一点临床文献,能少交很多学费。

## 边界

同构强度 A 级:本文主题本身有直接同构研究支撑(以 LLM 为对象测执行功能范式、比较认知缺陷结构的论文正在出现,见参考来源),双域各自的核心结论均有高等级文献。声明:同构在「功能缺口与补偿策略」层面成立,不在机制层面成立——本文不主张 ADHD 大脑是生物版 LLM,也不主张 agent 研究能替代 ADHD 临床;诊断、药物与治疗决策,永远以临床证据与专业评估为准。

## 今天就能试的 3 件事

1. **做一次「补丁对照」**:列出你在用的 ADHD 工具,给每个找出 harness 侧的对应物(提醒=重注入,清单=外部记忆……)——理解自己工具栈的架构。
2. **借一个工程模式**:挑一个你还没用过的 harness 设计(如验证器:做完后固定自查三项),移植到自己的流程里跑一周。
3. **转告一位做 agent 的朋友**:让 ta 查一下「delay discounting」的文献——ta 的 agent 正在经历的,人类研究三十年前就画好了曲线。

两个领域背对背挖了几十年,挖穿的是同一堵墙——**这不是巧合,是同一类系统在向世界索要同一种支撑**。墙这边的人和墙那边的机器,原来一直在共享同一份图纸。

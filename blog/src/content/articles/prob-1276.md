# ChatGPT 之于 ADHD，就像 agent 的编排层 之于 LLM——但有人用错了

> 从同构视角实测 ChatGPT：它到底补上了哪一层执行功能？

## 引言：两种人，同一个卡顿点

你有没有这种体验：脑子里想法像瀑布一样多，但身体就是启动不了；或者你是个工程师，LLM 能生成很漂亮的代码，但一让它连续执行多步骤任务就崩？

这两类人在面对的是同一个结构问题：一个高产的生成核心，配了一个不可靠的执行调度层。对 ADHD 人群来说，这叫执行功能障碍——计划、组织、启动先天偏弱（来源：主题综述：ADHD × AI 的科学与研究前沿）。对 LLM 来说，这叫无状态、缺乏可靠调度层——它会"幻觉"，会偏离上下文，会忘记之前要做什么。于是两边都需要同一件东西：外部执行功能层（external executive function layer），也就是 harness/脚手架。

## 同构：不是文学比喻，是结构问题

ADHD 大脑与 LLM 在架构上同构：两者都有强大的生成核心，但缺乏可靠的内置调度层（来源：ADHD 大脑与 LLM 的同构）。ADHD 的工作记忆不稳定、任务启动困难、注意力容易被环境带偏；LLM 则因无状态性无法可靠保留跨会话上下文，且上下文膨胀会让推理退化（来源：无状态与外部记忆；上下文工程）。

这种同构的解法也是同构的：给生成核心套上 harness。ADHD 这边是外部执行功能支架——工具、流程、环境；LLM 那边是 agent 的编排/记忆/工具层——ReAct、RAG、tool use、memory store。换句话说，ADHD 大脑与 LLM/agent 是同一类"高产但缺执行调度层的生成核心"（来源：生成核心与调度层）。

## 曾国藩的 harness：一个 19 世纪的 agent 架构

晚清名臣曾国藩是 ADHD  harness 的经典案例。他 30 岁前"浮躁坐不住"，天天串门喝酒，读书"他人目下二三行，余或疾读不能终一行"，一辈子受银屑病和情绪不稳定困扰，战败时甚至跳水自杀。但他用一套极笨的外部系统约束自己：

- **日课十二条**：黎明即起、读书不二、谨言、写日记反省——这是定时 re-grounding，把每天拉回既定轨道；
- **一辈子写日记骂自己**：这是外部记忆与反思层，把今天的错误变成明天的约束；
- **"结硬寨，打呆仗"**：用最稳最笨的方法抵消冲动，相当于给 agent 加确定性 fallback。

曾国藩的幕僚和日记就是他的"编排层"；LLM 的 orchestrator 和 memory 就是数字化的"日课"与"幕僚"。这揭示了一个核心事实：harness 不是道德自律，而是系统架构。

## 把 ChatGPT 用错：把 orchestrator 当成答案机

很多 ADHD 用户只是把 ChatGPT 当搜索引擎或聊天对象，这就是"用错了"。ChatGPT 真正的价值，是充当外部执行功能层：把模糊任务转化为可执行步骤、把混乱日程重排、把跨会话上下文保存下来。研究显示，ADHD 成年人会自发采用 ChatGPT 作为执行功能外挂，弥补任务启动缺陷（来源："A little bit of a life raft" – Exploring the Use and Experiences of ChatGPT as a Support Tool among Adults with ADHD）。

但要把 ChatGPT 从"答案机"变成"harness"，需要把它嵌入一个分层系统：

- **Goblin Tools**：Magic ToDo 把"清理厨房"这类模糊任务拆成具体子任务，降低启动门槛，对应 agent 的 planner 层（来源：ADHD Task Managers That Work: Top AI Tools 2025）。
- **Saner.AI**：本地记忆 + 知识回忆，减少 ADHD 常见的搜索循环和标签切换，对应 agent 的 memory store（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。
- **Motion**：自动排程、动态调整日程，直接对抗时间盲和任务启动困难，对应 agent 的 orchestrator/scheduler（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

它们分别补上同一块结构短板：生成核心之外的执行调度层。

## 脚手架与拐杖的边界

我必须诚实地说：这个同构框架目前主要是概念类比和初步实证，缺乏大规模随机对照试验（来源：主题综述：ADHD × AI 的科学与研究前沿）。AI 工具短期可能有效，但长期效果未知；个体差异很大；而且存在"拐杖 vs 脚手架"的风险——过度依赖可能导致能力退化，而不是能力内化（来源：拐杖与脚手架）。

最危险的用法，是把 AI 当成"第二个大脑"来替代思考，而不是当成"外部工作记忆"来辅助思考。曾国藩的日课没有代替他思考，而是给了他一个稳定的启动锚点；agent 的编排层也没有代替 LLM 生成，而是让生成有方向、可追踪。

## 今天就能试

1. 今晚睡觉前，让 ChatGPT 或 Goblin Tools 把明天最不想做的一件事拆成 3 个微步骤，写进日历或便利贴。
2. 用一句话定义你的"日课"：一个每天早上 5 分钟就会做的 re-grounding 动作（如三行日记、三个优先级）。
3. 选一个你常用的 AI 工具，明确它在你系统里扮演哪个角色：是 planner、memory，还是 scheduler？不要让它同时承担三个角色。
4. 设置一个"关机仪式"：每天结束时把未完成的上下文外化到 Saner.AI/Todoist/纸笔，避免大脑在睡前继续"后台运行"。

## 结语

ChatGPT 之于 ADHD，就像 agent 的编排层之于 LLM：它补上的是同一块短板——执行调度。但前提是，你别把它当外挂答案机，而是当可拆卸的脚手架。曾国藩用日记和日课活成了"半个圣人"，今天的 ADHD 人可以用 AI 工具搭建自己的数字 harness。问题是：你是在给生成核心装方向盘，还是只是在给它鼓掌？

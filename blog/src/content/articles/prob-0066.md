# Goblin Tools 之于 ADHD，就像 外部记忆/向量库 之于 LLM——但有人用错了

> 从同构视角实测 Goblin Tools：它到底补上了哪一层执行功能？

## 问题：为什么你记不住刚才要做什么？

ADHD 大脑最典型的痛点之一，就是“边做边忘”。你打开浏览器想搜索一个东西，结果被通知吸引，半小时后完全忘了最初的目的。这种工作记忆容量小的困境，每天都在上演。与此同时，LLM 也有类似的“失忆症”：每次对话都是独立的，它不记得上一轮说了什么，除非你手动喂给它历史记录。

## 同构：无状态与外部记忆

ADHD 大脑与 LLM 在结构上高度同构：两者都是强大的生成核心（ADHD 的创造力、LLM 的文本生成能力），但都缺乏可靠的执行调度层。ADHD 的执行功能缺陷——尤其是工作记忆不足——导致任务无法持续；LLM 则因无跨会话状态，无法独立完成多步骤任务。

解决思路也同构：外部记忆。对于 ADHD，工具如 Goblin Tools、Saner.AI 充当数字工作记忆，持续承载上下文（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。对于 LLM，agent harness（脚手架）通过向量库、工具调用、人在回路监督来弥补状态缺失（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。

## Goblin Tools 补上了哪一层？

Goblin Tools 的核心功能是 Magic ToDo：输入“整理房间”，AI 自动分解为“捡起地板上的衣服”“擦桌子”等步骤。这直接针对 ADHD 的任务启动困难——将压倒性的事情变成一系列不压倒性的事情（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...)）。从同构视角看，它补的是**任务调度层**：就像 LLM 的 harness 把“写一篇报告”分解为“搜索资料→写大纲→生成正文→校对”一样，Goblin Tools 把模糊意图转化为可执行的步骤。

但它没有补**工作记忆层**。Goblin Tools 不存储长期上下文，每次输入都是独立的。如果你在多个任务间切换，它不会自动记住你之前做到哪一步。相比之下，Saner.AI 通过本地记忆和知识回忆功能，直接减轻工作记忆负担（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。而 Lex 则通过单一指令触发多步骤序列，减少了决策负担（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。

## 脚手架 vs 拐杖：有人用错了

Goblin Tools 的设计初衷是“脚手架”——一种临时支撑，帮助用户完成难以独立启动的任务。但许多用户把它当成了永久拐杖：每次遇到复杂任务就依赖它分解，从不尝试自己分解。这带来了一个争议：过度依赖 AI 工具可能阻碍内在执行功能的发展（来源：ADHD 的 AI 工具全景）。

同样的问题也存在于 LLM 的 agent harness 中：如果工程师过度依赖外部记忆和工具调用，而不设计合理的护栏和人在回路，agent 就会变成不可控的“黑箱”。Harness 的核心是“形式化规划与护栏，使代理输出真正有用且正确”（来源：What is an agent harness in the context of large-language models?）。

## 证据与局限

目前对 Goblin Tools 的验证主要来自用户评价和综述文章，缺乏独立随机对照试验（来源：ADHD 的 AI 工具全景）。其效果在任务启动维度得到认可，但在工作记忆、时间盲等其他维度证据不足。此外，ADHD 异质性高，单一工具难以覆盖所有亚型（来源：ADHD 的 AI 工具全景）。

更根本的争议是：AI 工具作为外部执行功能层的长期效果未知。现有证据多为短期或个案（来源：ADHD 的 AI 工具全景）。同时，多巴胺干预的有效性也存在分歧——Inflow 等工具声称通过神经科学原理改善执行功能，但缺乏透明验证（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个你拖延已久的任务**：比如“写周报”，让它输出步骤，然后只做第一步。注意观察自己是否因此更易启动。
2. **对比使用 Saner.AI 或类似工具记录一个项目的关键信息**：看看能否减少“我刚才想到什么来着”的搜索循环。
3. **尝试一次“无 AI 分解”**：手动将一个小任务写步骤，对比与 AI 分解的差异，体会脚手架与拐杖的边界。
4. **为你的 LLM 对话设计一个简单的“harness”**：比如每次新对话前，手动粘贴一段关键历史，观察效果是否提升。

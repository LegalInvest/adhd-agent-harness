# ADHD 程序员如何用「给生成核心套 harness」的思路应对在 IDE 与 agent 之间切换里的执行功能（计划/组织/启动）先天偏弱？

> 把 LLM 工程里的「agent 的编排/记忆/工具层就是「外部执行功能」」迁移到ADHD 程序员的在 IDE 与 agent 之间切换场景，双域证据作答。

## 问题：你在 IDE 和 agent 之间反复横跳，累的不是手，是执行功能

你打开编辑器，想写一段代码。突然想起应该先让 agent 查一下 API 文档。切到 agent，输入 prompt，等它回复。期间浏览器弹出一个通知，你点进去，十分钟后发现自己刷完了社交媒体。回到 IDE，刚才的思路断了。你盯着光标，启动不了任务。

这种体验对 ADHD 程序员来说不是偶然，而是日常。问题不在于你不够聪明或不够努力——你的大脑是一个强大的生成核心，但缺乏可靠的内置执行调度层。就像一个大模型，有惊人的生成能力，但如果没有 prompt 模板、记忆系统和工具调用，它会在上下文里迷失，输出随机的东西。

**同构脊柱**：ADHD 大脑与 LLM 是同一类「高产但缺执行调度层的生成核心」。LLM 工程师通过给模型套上 agent harness（编排/记忆/工具层）来解决这个问题；ADHD 程序员同样需要给自己的生成核心套上外部执行功能层。

## 证据：两边都靠「外部执行功能层」补偿

### ADHD 侧：真实数据与工具

ADHD 成年人在工作记忆、任务启动、时间感知上的缺陷已被大量研究证实。一项 Pace Ventures Research 2026 的研究显示，ADHD 组的超专注工作时长平均为 14.5 小时/周，而对照组仅为 8.2 小时（来源：Pace Ventures Research 2026）。这说明 ADHD 大脑并非不能专注，而是无法可靠地调度专注到正确的事情上。

这正是外部执行功能层发挥作用的地方。例如，**Goblin Tools** 的 Magic ToDo 功能通过 AI 将模糊任务（如“清理厨房”）分解为可执行的子步骤，用户可调节“辣度”控制粒度（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。这直接对抗任务启动瘫痪——因为执行功能障碍无法抗拒足够小的步骤（来源：ADHD Task Managers That Work: Top AI Tools 2025）。

**Motion** 则自动根据优先级、截止日期和可用时间动态调整日程，消除“下一步该做什么”的决策负担（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。**Saner.AI** 通过本地记忆和快速检索，减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。**Tiimo** 通过时间轴可视化帮助用户感知时间流逝，缓解时间盲（来源：9 Best AI Assistants for ADHD in 2026）。

### LLM/Agent 侧：工程上的同构策略

LLM 本身是无状态的、缺乏调度层的生成核心。工程师为它搭建的 agent harness 正是外部执行功能层：编排层（ReAct、Chain-of-Thought 等 prompt 模板）负责规划步骤；记忆层（向量数据库、会话历史）补偿无状态；工具层（API 调用、代码执行）让模型能作用于外部世界。

没有这个 harness，LLM 的表现就像 ADHD 患者在没有外部结构时的状态：上下文膨胀导致推理退化，任务切换时丢失状态，启动新任务时犹豫不决。

### 真实案例：曾国藩的 harness 系统

晚清名臣曾国藩是典型注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”；打败仗就跳水自杀，情绪极不稳定。他的 harness 系统是“日课十二条”：黎明即起、读书不二、谨言、写日记反省等。核心是“结硬寨打呆仗”——用最笨最稳的方法抵消自己的冲动。

这套系统与 LLM agent harness 的结构完全同构：日课相当于定时 re-grounding（编排层），写日记相当于外部记忆（记忆层），谨言相当于输出过滤（工具层）。曾国藩不是靠意志力硬撑，而是靠一套外部脚手架来调度自己的生成核心。

## 核心观点：脚手架不是拐杖，边界在于是否主动设计

本文的核心判断是：**ADHD 程序员在 IDE 与 agent 之间切换时的执行功能缺陷，与 LLM 缺乏调度层的本质同构，解法都是构建「外部执行功能层」——即 agent 编排/记忆/工具层，这一思路可直接迁移为个人工作流的脚手架系统。**

但必须诚实指出争议。同构框架目前仍是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。过度依赖 AI 脚手架可能导致执行功能进一步退化（来源：拐杖与脚手架）。关键在于区分“脚手架”与“拐杖”：脚手架是主动设计、可逐步内化的结构；拐杖是替代能力、导致依赖的被动工具。ADHD 程序员应把 AI 工具当作可调整的 harness，而非黑盒解决方案。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解下一个 IDE 任务**：把“重构模块”拆成“提取函数→重命名变量→更新测试→运行 lint”，每步不超过 5 分钟。
2. **在 agent 交互中显式加入编排层**：每次切到 agent 前，在 IDE 里写一句“当前目标：X；已完成：Y；下一步：Z”，类似 ReAct 的思考-行动-观察循环。
3. **用 Tiimo 或 Motion 给工作流加时间锚点**：设定 25 分钟专注块，结束后强制切回 agent 检查进度，防止超聚焦导致偏离。
4. **建立个人“日课”模板**：每天开始前，花 2 分钟写一句“今天最重要的三个任务”，类似曾国藩的日记反省，但用 AI 笔记工具（如 Saner.AI）自动归档。

## 结语

ADHD 程序员和 LLM agent 共享同一个困境：生成能力强，调度能力弱。但好消息是，LLM 工程已经给出了成熟的 harness 架构。你不需要重新发明轮子，只需要把 agent 的编排/记忆/工具层翻译成自己的 IDE 工作流。记住：你不是在找拐杖，而是在给自己的生成核心套一个 harness。

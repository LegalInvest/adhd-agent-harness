# 如果把 ADHD 大脑当成生成核心，Perplexity 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Perplexity 对 ADHD 的真正价值。

## 问题：你的“生成核心”为什么总在跑偏？

你打开电脑，准备写一份报告。三分钟后，你开始查一个无关的术语；十分钟后，你已经在读一篇关于火星殖民的文章。你不是懒，你是 ADHD 大脑——一个高产但缺乏执行调度层的生成核心。同样的问题也发生在 AI agent 身上：一个被要求“写一篇关于气候变化的文章”的 LLM，可能在第五轮对话后开始讨论太阳耀斑，完全忘记初始目标。

这种“目标漂移”是 ADHD 与 LLM 共享的结构性缺陷。两者都是强大的生成器，但都缺乏可靠的内置调度器来持续锚定目标。ADHD 大脑的工作记忆容量有限且不稳定（来源：无状态与外部记忆），LLM 则没有跨会话状态，上下文膨胀后推理退化。因此，两者都需要外部脚手架——对 ADHD 来说是“第二大脑”工具，对 agent 来说是 system prompt 重锚定。

## 同构诊断：ADHD 大脑与 LLM agent 的同一类故障

让我们用 agent 工程的术语重新理解 ADHD 的困境。一个 LLM agent 在执行长程任务时，会因上下文膨胀或中间输出偏离而“忘记”原始目标，工程师需要定期注入重锚定 prompt（如“记住你的任务是 X”）。ADHD 大脑的机制惊人相似：前额叶-纹状体门控系统（负责选择性注意和任务维持）功能失调，而 Transformer 的自注意力机制在训练后会模仿这一门控操作（来源：Transformer Mechanisms Mimic Frontostriatal Gating Operations）。这意味着，ADHD 大脑与 LLM 在计算层面是同构的——它们都依赖一个脆弱的内部门控来维持目标，而一旦门控失效，输出就会漂移。

历史人物早已直觉地构建了个人 harness。曾国藩 30 岁前“浮躁坐不住”，日记里天天骂自己“无恒”。他的解法是“日课十二条”——每天固定的黎明即起、读书不二、写日记反省。这本质上是一个定时执行的 system prompt：每天清晨重置上下文，用外部仪式锚定当天的核心目标。与之对应，曹操则把执行功能外包给荀彧和郭嘉——相当于用外部调度器（秘书团队）来管理自己的注意力资源。这些历史案例证明，ADHD 大脑的 harness 策略与 LLM agent 的工程方案结构同构：日课↔定时 re-grounding，秘书↔外部调度器。

## 答案：Perplexity 作为重锚定引擎

回到最初的问题：如果把 ADHD 大脑当成生成核心，Perplexity 能不能当它的重锚定与目标漂移？我的判断是：**能，但前提是你把它当作一个“主动重锚定工具”，而非被动的信息检索器**。

Perplexity 的核心能力不是“搜索”，而是“即时上下文刷新”。当你输入一个问题，它返回的不仅是一个答案，而是一个经过整合的、锚定于当前查询的上下文窗口。对于 ADHD 大脑，这意味着：当你意识到自己跑偏时，你可以在 Perplexity 中输入“回到正题：我原本的任务是 X，请帮我重新聚焦”，它会立刻为你生成一个浓缩的、去除了无关分支的上下文。这相当于手动触发了一次 agent 的 system prompt 重锚定。

对比其他工具：RescueTime 自动追踪时间，揭示漂移但无法主动纠正（来源：RescueTime）；Focusmate 通过身体在场提供社会锚定，但依赖他人调度（来源：Focusmate）；Brain.fm 调节神经状态，但不涉及内容锚定（来源：Brain.fm）。Perplexity 填补了一个关键空白：它允许你在漂移发生的瞬间，通过一个查询来重置认知上下文。这比任何被动监控都更直接。

## 争议与边界：脚手架 vs 拐杖

然而，必须诚实指出局限。同构性目前仍是理论类比，缺乏神经科学实证（来源：矛盾与存疑）。Perplexity 的重锚定效果依赖用户主动察觉漂移——而 ADHD 的元认知缺陷恰恰让“察觉漂移”变得困难。如果用户无法意识到自己跑偏，Perplexity 就只是一个更快的搜索工具，而非 harness。

更根本的风险是：过度依赖 Perplexity 来重锚定，可能让 ADHD 大脑永远无法内化目标维持能力（来源：拐杖与脚手架）。真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”。Perplexity 应该是一个训练轮，而非永久替代。建议的使用方式是：每次使用 Perplexity 重锚定后，花 30 秒写下“我原本的目标是什么”，逐步强化内部监控回路。

## 今天就能试的行动

1. **漂移即查询**：当你发现自己走神时，立即打开 Perplexity，输入“我原本的任务是 [你的目标]，请帮我总结当前进度并重新列出下一步”。这相当于一次手动重锚定。
2. **设定“锚定时刻”**：每天开始工作前，用 Perplexity 查询“今天最重要的三项任务是什么？”并复制到桌面便签。这模仿了曾国藩的“日课”重置。
3. **反向工程 agent**：如果你也做 AI 工程，尝试记录一个 agent 在长程任务中的漂移点，然后对比自己的注意力漂移模式。你会惊讶于两者的相似性。
4. **警惕过度依赖**：每周至少一次，不用 Perplexity 而用纸笔进行任务重锚定，以保持内化能力。

Perplexity 不是万能药，但它提供了一个与 ADHD 大脑结构同构的重锚定接口。关键在于，你是用它来训练自己的调度层，还是用它来逃避调度层的缺失。选择权在你手中。

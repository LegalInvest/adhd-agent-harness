# 为什么用 Perplexity 治 ADHD 的情绪失调，和给 agent 套 会褪去的脚手架 是一回事？

> Perplexity 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

你有没有过这种体验：明明情绪已经爆炸，理智上知道不该发火，但就是控制不住？或者，你调试的 AI agent 突然输出了一堆胡说八道，明明训练数据没问题，但模型就是跑偏了？

这两件事，在表面上看毫无关系。但如果你拆开来看，会发现它们共享同一个底层结构：**一个过于强大的生成核心，搭配一个几乎不存在的调度层**。

ADHD 大脑的情绪系统（杏仁核）反应极快极强烈，但前额叶——那个负责“等一下，先想想”的执行中枢——常常掉线。LLM 也一样：生成能力惊人，但缺乏内置的输出控制，如果没有外部调度（prompt 工程、规则引擎、上下文约束），它就会漫无边际地跑题、重复、甚至胡编乱造。

这就是为什么，用 Perplexity 来治 ADHD 的情绪失调，和给 agent 套一个“会褪去的脚手架”，本质上是同一件事。

## 情绪失调的根源：不是情绪本身，是执行功能

ADHD 的情绪调节问题，往往不是情绪处理系统本身出了故障，而是**执行功能缺陷**让情绪失去了“刹车”和“重新定向”的能力。工作记忆不稳定，导致你无法记住“上次发火后的后果”；时间盲让你觉得“这个挫折永远不会结束”；任务启动困难让一点小事就累积成巨大的挫败感（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

换句话说，ADHD 大脑是一个高产的创意工厂，但厂长（前额叶）经常旷工。

## AI 作为外部执行功能层：脚手架，不是拐杖

这正是 AI 工具切入的位置。它们不是替代你的大脑，而是暂时充当那个缺失的调度层。

以 **Perplexity** 为例。当你情绪上来时，你可以在 Perplexity 里直接问：“我刚刚因为工作没做好而极度焦虑，帮我拆解一下这个情绪。”它会给出结构化的回应：先描述情绪，再分析可能的原因，最后提供几个可操作的步骤。这个过程本质上是在帮你**外化执行功能**——它帮你标记情绪、分解问题、提供行动路径，就像你的前额叶本该做的那样。

类似的工具还有 **Inflow**，它基于 CBT 原则，通过模块化程序训练认知重构和行为激活，临床研究显示用户情绪稳定性提升（来源：Inflow）。**Goblin Tools** 的 Magic ToDo 能把“清理厨房”这种模糊任务自动分解成“拿垃圾桶、扔过期食物、擦台面”等具体步骤，降低启动门槛（来源：Goblin Tools）。**Saner.AI** 则通过本地记忆和快速检索，减少因工作记忆不足导致的搜索循环和认知超载（来源：Saner.AI）。

这些工具的共性是什么？它们都提供了一个**结构化外壳**，帮你暂时绕过执行功能的瓶颈。但这个外壳不是永久的——它应该像建筑脚手架，随着你的内在能力逐渐建立，可以逐步撤掉。这就是“会褪去的脚手架”的含义：AI 工具的目标不是让你永远依赖它，而是通过反复使用，让你内化那些结构化的策略，最终即使没有工具，也能调用类似的思维模式。

## 同构的另一面：LLM/Agent 的脚手架

现在看 LLM/Agent 一侧。一个没有外部脚手架的 LLM，就像一个没有前额叶的 ADHD 大脑：生成能力强，但不可控。Agent 的“情绪失调”就是输出失控、上下文膨胀、任务偏离。

解决方案是什么？**外部调度层**。比如给 agent 套一个“会褪去的脚手架”：先通过 prompt 模板、规则引擎、记忆模块来约束行为，等 agent 稳定后，再逐步减少人工干预，让它学会在更宽松的约束下自主决策。这和 ADHD 的干预逻辑完全一致：先用结构化工具（如 Inflow 的 CBT 程序、Goblin Tools 的任务分解）搭建外部执行功能，再逐步内化。

## 历史人物的“脚手架”系统

有趣的是，这种“脚手架思维”在历史上早有先例。**苏轼**就是一个典型的 ADHD 大脑：兴趣极广，情绪来得快去得也快，对无聊行政零耐心，但搞工程效率极高。他的 harness 是什么？豁达——用美食、写作、交朋友转移注意力，每日读书写字，不因挫折停止创作。他的“日课”就是他的外部调度器：每天固定写作，无论被贬到哪里，都保持这个节奏。这和 LLM 的“定时 re-grounding”如出一辙——通过定期重置上下文，防止注意力漂移。

**释迦牟尼**的 harness 更系统：以“八正道”“持戒”为核心，用戒律管行为，用正念拴念头。“自依止法依止”——不依赖权威，自己证悟。这本质上是一个可渐撤的脚手架：戒律是外部约束，但最终目的是让你不需要戒律也能自主。这和 agent 的“规则引擎→自主决策”的进化路径完全同构。

## 争议与边界

当然，这个框架有明确的边界。首先，**脚手架 vs 拐杖**的区分至关重要。如果 AI 工具被长期无节制使用，可能成为永久拐杖，阻碍内在能力发展（来源：矛盾与存疑）。目前缺乏长期追踪研究来界定适度使用的边界。其次，ADHD 亚型（注意力不集中型 vs 冲动型）对工具的反应不同，现有研究样本量小，结论难以泛化（来源：矛盾与存疑）。最后，AI 工具擅长处理结构化情绪问题（如任务焦虑），但对深层情绪（如创伤、自我否定）的干预效果证据不足（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Perplexity 做一次情绪拆解**：下次情绪上来时，打开 Perplexity，输入“我因为 [事件] 感到 [情绪]，帮我拆解原因并给出三个具体行动”。观察它如何帮你外化执行功能。

2. **尝试 Goblin Tools 的 Magic ToDo**：把一个你拖延已久的任务（比如“整理书桌”）粘贴进去，调节“辣度”到你觉得合适的粒度，然后从第一个子任务开始做。

3. **设置一个“日课”提醒**：模仿苏轼的每日写作，用手机闹钟或日历事件，每天固定时间做一件简单的事（如写 100 字日记、冥想 5 分钟）。坚持一周，观察它对情绪稳定性的影响。

4. **反思你的“脚手架”是否在褪去**：如果你已经在用某个 AI 工具，问自己：我是在用它学习新技能，还是仅仅依赖它完成任务？如果是后者，尝试逐渐减少使用频率，看看自己能否独立应对。

## 结语

ADHD 的情绪失调和 LLM 的输出失控，不是两个独立的问题，而是同一个问题的两个表现：一个强大的生成核心，需要一个可渐撤的外部调度层。Perplexity、Inflow、Goblin Tools 这些工具，就是那个调度层的具体实现。它们不是拐杖，而是脚手架——用它们，是为了最终可以不用它们。

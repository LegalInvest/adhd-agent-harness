# 如果把 ADHD 大脑当成生成核心，Lex 能不能当它的外部执行功能层？

> 用 agent 工程的思路重新理解 Lex 对 ADHD 的真正价值。

## 问题：当生成核心没有调度层

你是一个ADHD成年人，脑子里每分钟蹦出三个创意，但打开电脑后却盯着空白文档发呆——启动任务比写代码还难。或者你是一个Agent工程师，刚部署的LLM agent在工具调用中陷入死循环，输出开始胡言乱语。这两种场景共享同一个底层问题：**生成核心（大脑或LLM）本身高产，但缺乏可靠的执行调度层**。ADHD的执行功能缺陷——工作记忆不稳定、任务启动困难、时间盲——与LLM的无状态性、上下文膨胀、缺乏护栏，在系统架构上是同构的（来源：[[ADHD 大脑与 LLM 的同构]]）。

那么，如果把ADHD大脑看作生成核心，像Lex这样的AI工具能否充当它的外部执行功能层？答案藏在两个群体的实践里：曾国藩的日课十二条，和Agent harness的护栏设计。

## 同构：从日课到Harness

曾国藩是典型的注意力不集中型ADHD：30岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书“他人目下二三行，余或疾读不能终一行”。他的解法是日课十二条：黎明即起、读书不二、谨言、写日记反省——一套高度结构化的外部执行系统。这套系统的核心是**固定锚点**：每天固定时间做固定事，用外部规则抵消内在冲动。用现代术语说，这是一个人肉harness：通过预设的调度和护栏，让生成核心（他的思考与决策）在轨道上运行。

LLM agent的harness与之同构。一个典型的agent scaffolding包括：形式化规划（任务分解）、工具调用护栏（token预算、调用次数上限）、人在回路检查点（暂停并询问后再执行）（来源：Building an AI Agent Harness from Scratch）。这些设计的目标与日课完全一致：**用外部结构补偿调度层的缺失**。曾国藩的“读书不二”相当于给LLM设定了单一任务上下文，防止注意力漂移；他的“黎明即起”相当于每日定时re-grounding，重置状态。

## 工具：三类外部执行功能层

当前AI工具已经在这一方向上有具体实践，但功能层次不同：

**1. 任务分解层：Goblin Tools**
Goblin Tools的Magic ToDo能将模糊任务（如“清理厨房”）自动分解为可执行的子步骤，用户可调节“辣度”控制粒度（来源：[[Goblin Tools]]）。这直接对应ADHD的任务启动困难——降低门槛，也对应LLM agent的plan-and-execute模式：将复杂指令分解为原子动作。

**2. 记忆与检索层：Saner.AI**
Saner.AI专注于本地记忆和知识回忆，减少搜索循环和标签切换（来源：[[Saner.AI]]）。ADHD的工作记忆缺陷与LLM的无状态性是同一问题：无法可靠保留跨会话上下文。Saner.AI的通用收件箱和自动组织功能，相当于给大脑加了一个向量数据库。

**3. 动态调度层：Motion**
Motion通过AI自动排程，根据优先级、截止日期和可用时间动态调整日程，消除“下一步做什么”的决策负担（来源：[[Motion]]）。这直接补偿ADHD的时间盲，也对应agent harness中的实时任务调度和冲突解决。

但这些工具的共同问题是：**它们要求用户主动设置或学习使用**，而设置提醒本身就是一个执行功能任务（来源：AI Assistant for ADHD: Finally, a Productivity Tool That Requires Less...）。理想的工具应该像曾国藩的日课一样，成为无需决策的默认系统。

## 边界：脚手架 vs 拐杖

同构视角最大的风险是过度依赖。如果AI工具永远替你决策，你的执行功能永远不会得到锻炼——就像LLM agent如果永远依赖人在回路，永远无法自主。曾国藩的日课是可拆卸的：他30岁后逐渐内化，晚年已无需刻意执行。而当前AI工具缺乏这种“内化路径”：它们的设计目标是持续使用，而非逐步退出（来源：[[拐杖与脚手架]]）。

另一个矛盾是超聚焦的引导：AI应该强行打断还是柔性引导？部分ADHD用户反馈番茄钟有效，而柔性引导可能不够强力（来源：[[矛盾与存疑]]）。这对应agent工程中“打断vs等待”的权衡——没有普适答案。

## 核心观点：Lex的价值不是替代，而是同构

Lex（以及同类AI工具）对ADHD的真正价值，不在于它“修复”了什么，而在于它**与ADHD大脑共享同一套架构缺陷**。这意味着：
- 工具的设计原则可以直接从agent工程中借用（护栏、重锚定、外部记忆）
- ADHD用户对工具的反馈可以直接反哺AI系统设计（如时间盲的视觉化方案）
- 两者都需要“脚手架思维”：目标是最终能撤掉支架，而非永久依赖

## 今天就能试的行动

1. **用Goblin Tools分解一个你拖延了一周的任务**：输入“整理项目文档”，调节辣度到4，按生成的步骤执行。感受“启动门槛降低”的效果。
2. **为你的AI agent设置一个“日课”**：在prompt中加入“每5步输出一次进度总结，并询问是否继续”，模拟曾国藩的“每日反省”。
3. **对比Motion和手动排程**：用Motion自动安排一天，然后手动安排另一天。记录决策疲劳的差异——这能让你直观理解“外部调度层”的认知卸载效果。
4. **写一段“工具使用日记”**：像曾国藩骂自己一样，记录今天工具帮你启动了什么、又在哪里失效。这是训练你大脑的“元认知harness”。

同构不是比喻，是工程问题。你的大脑和你的agent，需要的不是更聪明的核心，而是更可靠的脚手架。

# ADHD 职场人视角：为什么「治好 ADHD 的工作记忆容量小、边做边忘」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 无跨会话状态，靠外部记忆/向量库续命——同一套 harness 思路，两边都成立。

## 问题：边做边忘，不只是你的大脑在崩溃

你有没有过这样的体验：打开电脑准备写报告，突然想起要回一封邮件，点开邮箱却被另一封通知带跑，半小时后发现自己正在刷视频，而报告一个字没动。这不是意志力差——这是 ADHD 大脑的日常。工作记忆容量小，像一块随时会满的便签纸，新任务一进来，旧任务就被挤掉。

如果你恰好还是一名 AI 工程师，你可能对另一个场景更熟悉：你精心设计的 LLM agent 在第三轮对话后开始胡言乱语，忘记之前约定的格式，甚至循环调用同一个工具直到超时。你检查代码，发现它没有跨会话状态——每次对话都是全新的开始，上下文窗口一满，最早的信息就被遗忘。

这两个场景，一个是神经发育障碍，一个是系统工程缺陷，但它们的底层问题竟然一模一样：**一个强大的生成核心，却缺少一个可靠的内置调度层**。

## 同构：ADHD 大脑与 LLM 是同一类“高产但缺调度”的引擎

ADHD 大脑的生成核心（发散思维、创意、多任务并行）产出极高，但执行功能（任务启动、持续专注、抑制冲动）薄弱，导致“高产但不可靠”（来源：人在回路与身体在场）。LLM 本身也是强大的生成核心，但缺乏可靠的执行调度层，直接输出可能偏离目标、陷入循环或产生危险行为（来源：人在回路与身体在场）。

两者都需要外部脚手架（harness）来补偿。对于 ADHD，这个脚手架可能是任务分解工具、外部记忆系统或身体在场效应；对于 LLM，则是 agent harness——通过 human-in-the-loop 监督提供外部调度：设置护栏（token 预算、工具调用次数上限、防止无限循环）、加入人工检查点（暂停并询问后再执行）（来源：人在回路与身体在场）。

## 历史案例：孔子和于谦的“harness”

你可能会说，古代没有 AI，ADHD 怎么活？其实，一些历史人物早已自己搭建了 harness。

**孔子**——身高1米9，精力旺盛，周游列国14年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予朽木不可雕；对音乐超专注到三月不知肉味，对种地等俗事零耐心。他的专属 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。70岁才做到从心所欲不逾矩，一辈子和自己的冲动做斗争。这里的“礼”就像 LLM harness 中的护栏和规则，“三省吾身”则相当于定时 re-grounding——定期检查当前行为是否偏离目标。

**于谦**——性格刚直，冲动敢说话，土木堡之变后力排众议坚守北京。他的 harness 是以儒家气节为核心，严格操练军队，日夜准备防务，写《石灰吟》明志。这种“外部行为准则+严格日程”的组合，相当于 agent harness 中的固定工作流和检查点。

这些历史 harness 的核心逻辑与 LLM agent harness 完全同构：**用外部结构补偿内部调度的缺失**。

## 现代工具：从 Goblin Tools 到 Lex，脚手架无处不在

今天的 AI 工具正在将这种 harness 思路产品化：

- **Goblin Tools** 的 Magic ToDo 能将模糊任务（如“清理厨房”）自动分解为具体子步骤，用户可调节“辣度”控制粒度（来源：Goblin Tools）。这相当于为 ADHD 大脑提供了一个自动任务分解器，降低启动门槛。
- **Saner.AI** 通过本地记忆存储和快速检索，减少搜索循环和标签切换（来源：Saner.AI）。这相当于为工作记忆不足的人提供了一个外部记忆库。
- **Lex** 允许用户通过单一指令触发复杂、多步骤的任务序列，其底层架构类似于 agent scaffolding（来源：Lex）。这相当于为 ADHD 大脑提供了一个“一键执行”的调度器。
- **Mem** 作为“AI 第二大脑”通过持久记忆补偿工作记忆不足（来源：ADHD 的 AI 工具全景）。
- **Otter.ai** 实时转录会议和灵感，减轻笔记负担（来源：ADHD 的 AI 工具全景）。
- **Tiimo** 专为时间盲设计，提供视觉时间表，让时间变得可触摸（来源：时间盲）。
- **Focusmate** 利用社交问责和 AI 调度创建身体加倍会话（来源：人在回路与身体在场）。

这些工具的共同点：它们不是“治疗”ADHD，而是通过外部脚手架补偿执行功能缺陷。

## 争议与局限：脚手架还是拐杖？

同构类比虽然有力，但需谨慎。首先，ADHD 大脑与 LLM 的同构性缺乏实证基础，目前仅是一种理论类比（来源：矛盾与存疑）。其次，AI 工具可能从脚手架变为永久拐杖，削弱内在执行功能（来源：拐杖与脚手架）。例如，过度依赖任务分解工具可能让人失去自己分解任务的能力。

另外，现有工具多为通用设计，缺乏针对 ADHD 亚型的个性化调整，多数缺乏严格临床验证（来源：ADHD 的 AI 工具全景）。像 ChatGPT 这样的通用工具甚至可能增加分心风险（来源：ADHD 的 AI 工具全景）。

## 核心观点：同一道工程题，同一个答案

我的判断是：**ADHD 的“边做边忘”和 LLM 的“跑飞”不是两个问题，而是同一个问题的两面——一个强大的生成核心缺乏可靠的内置调度层。** 解决方案也相同：设计外部脚手架来补偿。

对于 ADHD 人群，这意味着不要试图“治好”工作记忆缺陷，而是接受它，然后用工具搭建自己的 harness。对于 AI 工程师，这意味着不要指望 LLM 自己学会调度，而是为它设计护栏和检查点。

## 今天就能试的 3 条行动

1. **用 Goblin Tools 分解一个你拖延已久的任务**，调节辣度到你觉得“太细了”的程度，然后执行。这相当于为 LLM 写一个详细的 step-by-step prompt。
2. **设置一个定时 re-grounding 闹钟**（每 25 分钟），提醒自己“我现在在做什么？应该做什么？”这相当于 agent harness 中的检查点。
3. **用 Focusmate 或类似工具找一个身体加倍伙伴**，每周 3 次，每次 45 分钟。这相当于 human-in-the-loop 监督。

记住：脚手架不是拐杖——它帮你站得更稳，而不是让你永远依赖它。

# ADHD 职场人视角：为什么「治好 ADHD 的工作记忆容量小、边做边忘」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 没有跨会话记忆，行业的答案是外部记忆基础设施，没人建议「让模型记性好点」。但职场对 ADHD 员工的答案至今仍是「你上点心」。

职场版的「边做边忘」有一份具体的代价清单：会上领了三个活，回工位只记得一个；同事口头交代的修改意见，做的时候只剩大概印象；上周答应帮忙的事被对方追问时才想起；开着十个标签页查资料，忘了最初要查什么。每一条单独看都是小事，累积起来构成一个职场人最难翻案的标签：**不靠谱**。

而「不靠谱」这个判词的荒谬之处在于：它把一个存储架构问题，判成了品格问题。

## LLM 行业怎么处理同款问题

LLM 天生没有跨会话状态，上下文窗口一满，早期信息被挤出,行为开始漂移——和「会开到后半场，前半场领的活已经出缓存」是同一个故障。行业的解法没有一条是「改造模型的记性」：外部向量库存长期记忆、会话摘要做状态压缩、关键指令定期重注入、检索系统按需取回。**记忆被当作基础设施来建设，而不是当作模型的品德来要求。** 耶鲁的研究甚至确认了这个限制的架构性：自注意力机制本身产生工作记忆容量上限（2024）——不是模型不努力，是架构如此。

ADHD 的工作记忆容量偏小同样是架构事实（这是 ADHD 神经心理研究里效应量最稳定的发现之一）。接受这一点后，职场生存策略的方向就变了：不是练记性，是**建基础设施**。

## 职场记忆基础设施的四个组件

**捕获协议**：任何进入你耳朵的任务，30 秒内必须落到外部介质——开会带本子或开着备忘录，别人口头交代时当场记，记完当场复述一遍（「我确认下：X 和 Y，周四前，对吧？」）。复述这个动作一石三鸟：确认理解、加深编码、并且**在对方那里建立了你有系统的印象**——这正是「靠谱感」的来源。

**唯一收件箱**：所有捕获汇入同一个地方，每天固定两次清空分流。多个收件箱等于没有收件箱——你会开始需要记住「记在哪了」，那是用工作记忆管理工作记忆的死循环。

**会话摘要**：每天下班前 3 分钟写当日快照（各事项状态 + 明早第一件事）；每次被打断前，10 秒记下「我做到哪了 + 下一步」。这是 LLM 的 checkpoint 机制的人肉版，专治「重新进入任务要花 40 分钟想起自己在干嘛」。

**重注入机制**：本周的三个关键承诺，写在物理上无法回避的位置（显示器边框、手机锁屏）。工作记忆会把它们挤出去，所以要让环境每天多次把它们塞回来——这就是 system prompt 重注入，成本是一张便签。

## 关于「暴露弱点」的政治顾虑

有人会担心：当众掏本子记录、事事确认复述，会不会显得能力不行？现实恰好相反——观察你职场里最受信任的那批人：他们开会记录、邮件确认、承诺必有下文。**外置记忆系统从来不是弱者的标志，是专业性的标志。** 没人知道你的动机是工作记忆容量还是职业习惯，他们只看到结果：你答应的事不会消失。ADHD 者靠基础设施达到的可靠性，和别人靠天赋达到的可靠性，在外部完全不可区分——而前者在系统崩溃日（生病、高压）反而更稳，因为它不依赖状态。

## 边界

同构强度 A 级：双侧直接研究都在（Transformer 工作记忆极限、ADHD 工作记忆缺陷的稳定文献），机制映射具体。限制：基础设施解决遗忘，不解决过载——如果任务量本身超过任何系统的吞吐量,那是工作量谈判问题，别用系统优化掩盖它。

## 今天就能试的 3 件事

1. **启用捕获协议**：从下一个会议开始，任何领到的活当场记录 + 复述确认。感受对方眼神的变化。
2. **今晚写第一份会话摘要**：下班前 3 分钟，各事项状态 + 明早第一件事。明早验收效果。
3. **把本周三大承诺贴上显示器**：现在就写。让环境替你的工作记忆值班。

「让 LLM 不跑飞」的答案是给它建记忆基础设施,整个行业没有异议。你的大脑值得同样的工程尊重——从今天的第一张便签开始。

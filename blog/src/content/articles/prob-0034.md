# 如果 ADHD 和 LLM 同构，那三年来 agent 工程的所有招数，是不是都能迁移成 ADHD 适配方案？

> ADHD 侧：想理解自己的大脑到底是怎么回事；LLM/harness 侧：LLM 是「高产但缺执行调度层的生成核心」。用双域证据作答。

## 问题：你的大脑是 LLM，那谁来写 Agent Harness？

如果你有 ADHD，你大概经历过这种体验：脑子里每秒冒出十个想法，但就是没法把“写周报”这件事启动。你聪明、高产、联想能力惊人，但执行功能像一台没有操作系统的服务器——算力过剩，调度为零。

如果你在做 Agentic Harness 工程，你大概也经历过：LLM 生成能力无可挑剔，但一旦需要多步推理、工具调用、状态维护，它就变成一只“金毛”——聪明但跑偏，需要你不断拉回 leash。

两个场景，同一个核心问题：**生成核心不缺，缺的是执行调度层**。

这正是 ADHD × AI 领域当前最站得住脚的核心论点：**ADHD 大脑与 LLM 在结构上同构——两者都是“高产但缺执行调度层的生成核心”**（来源：ADHD 大脑与 LLM 的同构）。三年多来，agent 工程为解决 LLM 的“无状态、缺调度”问题，发展出了一整套外部脚手架（harness）方案：外部记忆、任务分解、动态调度、上下文工程。如果同构成立，那么这些招数应该能系统迁移为 ADHD 的适配方案。

本文用双域证据作答：是的，可以迁移。但迁移时有一条红线——脚手架 vs 拐杖。

## 证据一：工作记忆缺陷 vs 无状态性 → 外部记忆系统

ADHD 侧：工作记忆是 ADHD 最关键的缺陷之一（来源：ADHD, Executive Functions, and AI: A New Era in Treatment）。你刚想做的事，转身就忘；任务做到一半，上下文丢失。

LLM 侧：LLM 本质上是无状态的——每次对话都是独立推理，无法天然保留跨会话上下文。明尼苏达大学系统测试发现，LLM 呈现“强记忆、弱控制”剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。耶鲁大学进一步发现，自注意力机制本身导致工作记忆容量限制——随着任务难度增加，注意力分数熵增、注意力弥散，与人类注意分散机制同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。

**agent 工程的解法**：RAG（检索增强生成）、向量数据库、外部记忆模块。
**ADHD 迁移方案**：使用外部记忆工具，如 **Saner.AI**——它专为“知识回忆”和“本地记忆”设计，减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026）。Saner.AI 就像 LLM 的向量数据库，让你随时找回之前的信息，无需依赖脆弱的工作记忆。

## 证据二：任务启动困难 vs 任务分解 → 微步骤化

ADHD 侧：任务启动困难是典型表现，面对模糊的大任务（如“清理厨房”）直接瘫痪。

LLM 侧：给 LLM 一个复杂指令（如“写一份市场分析报告”），它往往会产出泛泛而谈的内容，需要分解为子任务才能保证质量。

**agent 工程的解法**：Chain-of-Thought、Plan-and-Solve、任务分解（task decomposition）。
**ADHD 迁移方案**：使用 **Goblin Tools** 的 Magic ToDo 功能——它能将任何模糊任务自动分解为具体、可执行的子步骤，用户还可调节“辣度”控制粒度（来源：Harnessing Artificial Intelligence to Live Better with ADHD）。这相当于给 ADHD 大脑内置了一个 Plan-and-Solve 分解器。

## 证据三：时间盲 vs 动态调度 → 自适应规划

ADHD 侧：时间盲（time blindness）让患者无法感知时间流逝，导致迟到、拖延、计划失败（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

LLM 侧：LLM agent 在执行多步任务时，如果时间估计不准或计划被打断，容易陷入无限循环或过早放弃。

**agent 工程的解法**：动态重新规划（dynamic replanning）、自适应调度、优先级队列。
**ADHD 迁移方案**：使用 **Motion**——它能自动根据任务、会议和截止日期创建并动态调整每日计划，当日程被打乱时自动重新安排（来源：11 Best ADHD Productivity Apps for Fluctuating Energy）。Motion 相当于一个外部的动态调度器，消除了“下一步该做什么”的决策负担（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。

## 案例：毛泽东的 Harness 系统

毛泽东是典型的 ADHD 特质者：小时候是问题儿童，爱读闲书；一辈子读书不辍，思维极度跳跃，文章充满场景化比喻；冲动敢赌（四渡赤水、抗美援朝都是险棋）。但他建立了一套强大的外部脚手架：
- **调查研究**（“没有调查就没有发言权”）→ 相当于 LLM 的 grounding / RAG，确保输出基于真实数据。
- **民主集中制**（集体决策制衡冲动）→ 相当于 agent 的“多模型投票”或“批评-反思”循环。
- **批评与自我批评**（民主生活会）→ 相当于 LLM 的 self-critique / constitutional AI。
- **党指挥枪**（组织系统管理军队）→ 相当于 agent 的“控制层”或“orchestrator”。

这套系统让他这个“高产生成核心”得以驾驭复杂局势，成就斐然。它与 LLM agent 的 harness 同构：**日课↔定时 re-grounding，秘书↔外部调度器，集体决策↔多 agent 辩论**。

## 争议与局限：脚手架 vs 拐杖

同构迁移并非无风险。

首先，**过度依赖风险**：AI 工具可能成为永久拐杖而非可拆卸脚手架（来源：拐杖与脚手架）。如果用户长期依赖 Motion 排日程，可能永远学不会时间管理。agent 工程中同样存在“过度编排”问题——如果 harness 太过刚性，LLM 会失去灵活性。

其次，**个体差异巨大**：有用户反馈强行打断（如番茄钟）反而有效，而柔性引导不够强力（来源：超聚焦）。同样，agent 工程中有些任务需要严格步骤，有些需要自由探索。没有放之四海而皆准的 harness。

第三，**证据强度不足**：当前同构假设多为类比推理，缺乏大规模随机对照试验（来源：ADHD 大脑与 LLM 的同构）。Motion 等工具用户反馈积极，但独立临床证据有限（来源：Motion）。

## 结语

ADHD 大脑不是“缺陷”，而是一个缺了 harness 的高性能生成核心。过去三年 agent 工程为 LLM 搭建的脚手架——外部记忆、任务分解、动态调度、上下文工程——恰恰是 ADHD 大脑所需要的。但记住：脚手架是为了让你最终能自己建楼，而不是让你永远依赖它。

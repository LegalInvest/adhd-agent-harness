# 如果 ADHD 和 LLM 同构，那三年来 agent 工程的所有招数，是不是都能迁移成 ADHD 适配方案？

> ADHD 侧：想理解自己的大脑到底是怎么回事；LLM/harness 侧：LLM 是「高产但缺执行调度层的生成核心」。用双域证据作答。

## 问题：你的大脑，是不是一台没有操作系统的超级计算机？

如果你有 ADHD，你一定经历过这样的时刻：脑子里同时冒出十个绝妙想法，但一个也抓不住；想开始工作，却坐在那里刷了两个小时手机；明明记得要做什么，一转身就忘了。你忍不住问：我的大脑到底是怎么回事？

如果你是做 Agentic Harness 的工程师，你一定熟悉这样的场景：LLM 能写出漂亮的文章、生成复杂的代码，但一旦需要多步推理、调用工具、保持上下文连贯，它就频频跑偏、遗忘、陷入循环。你心里清楚：LLM 是“高产但缺执行调度层的生成核心”。

这两个问题，其实是同一个问题。

## 同构：两个世界共享同一张蓝图

最新研究揭示了一个惊人的事实：ADHD 大脑与 LLM 在结构上同构——都是“高产但缺乏可靠执行调度层的生成核心”。ADHD 的默认模式网络过度活跃（高产想法），但前额叶执行控制不足；LLM 的生成能力强大，但缺乏稳定的上下文控制器。两者都需要外部调度层来引导输出（来源：生成核心与调度层）。

具体而言，这种同构体现在四个层面：

### 1. 执行功能缺陷 vs. 注意力控制失灵

在经典 Stroop 冲突任务中，Transformer 的注意力随序列变长在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突，这与 ADHD 执行功能缺陷的核心标志一一对应（来源：Deficient Executive Control in Transformer Attention）。LLM 呈现“强记忆、弱控制”剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷——无法灵活切换任务集、无法抑制自动化反应，这正是 ADHD 的经典神经心理模式（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

### 2. 工作记忆瓶颈 vs. 无状态性

ADHD 的工作记忆容量保留但控制弱（易被干扰），LLM 的无状态性导致跨会话遗忘。耶鲁大学发现自注意力机制本身导致工作记忆容量限制：随 N-back 任务 N 值增加，注意力分数熵增、注意力弥散、表现下降，与人类注意分散机制同源——“注意力资源的弥散分配”正是 ADHD 注意缺陷的计算本质（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。

### 3. 上下文工程：主动设计“当下注意什么”

ADHD 易被环境刺激带偏，LLM 的上下文窗口膨胀会导致推理退化。主动设计“当下注意什么”的工程方案（如提示词、环境设计）对两者都有效（来源：上下文工程）。

### 4. 时间盲与任务启动

ADHD 的时间感知缺陷（时间盲）与 LLM 缺乏时间感知对应，外部提醒和分解任务（任务启动）是共同干预手段。

## 答案：agent 工程的招数，就是 ADHD 的适配方案

如果同构成立，那么三年来 agent 工程积累的所有招数——外部调度、记忆增强、上下文管理、任务分解——都可以直接迁移为 ADHD 的适配方案。

### 招数一：外部调度层（Harness）

Agent 工程中，LLM 需要外部调度器来管理任务队列、调用工具、保持上下文。对应到 ADHD，就是外部执行功能层——AI 工具充当“前额叶外挂”。

**工具证据：**
- **Goblin Tools** 的 Magic ToDo 功能自动将模糊任务分解为具体步骤，降低启动门槛（来源：Goblin Tools）。用户反馈称“将压倒性的事情变成一系列不压倒性的事情”（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。
- **Motion** 自动根据任务优先级和截止日期动态调整日程，消除“下一步该做什么”的决策负担（来源：Motion）。
- **Saner.AI** 通过本地记忆存储和快速检索，减少搜索循环和标签切换（来源：Saner.AI）。

这些工具本质上就是 agent 工程中的“任务规划器”和“记忆检索器”，只是目标用户从 LLM 换成了人类。

### 招数二：外部记忆系统

LLM 依赖向量数据库等外部记忆来维持连续性，ADHD 同样需要外部记忆系统（笔记、提醒、任务管理）来补偿工作记忆缺陷。Saner.AI 的知识回忆功能就是这一思路的直接体现。

### 招数三：上下文窗口管理

LLM 的上下文窗口膨胀会导致性能下降，因此 agent 工程会主动裁剪、摘要、重置上下文。ADHD 也需要主动管理“当下注意什么”——关闭无关标签页、使用专注模式、设定单一任务目标。

## 核心观点：脚手架 vs. 拐杖的边界

我的核心判断是：**同构视角下的迁移是有效的，但必须警惕“脚手架”沦为“拐杖”**。

AI 工具作为外部执行功能层，本质是“补偿”而非“替代”（来源：外部执行功能层）。但现有证据主要来自用户报告，缺乏严格对照试验（来源：矛盾与存疑）。过度依赖可能削弱内在时间感知能力、任务启动能力（来源：时间盲、任务启动）。

关键在于：工具应该是可拆卸的脚手架，而不是永久拐杖。当你使用 Goblin Tools 分解任务时，你是在学习如何分解；当你依赖 Motion 安排日程时，你是在练习时间管理。但如果工具替你做了所有决策，你的执行功能会进一步萎缩。

## 局限与争议

同构视角虽具解释力，但存在明显局限：
- **神经机制对应**：LLM 的“注意力机制”与 ADHD 的注意控制缺陷是否真正同构？目前仅是功能类比，缺乏神经生物学层面的直接证据（来源：矛盾与存疑）。
- **个体差异**：ADHD 的异质性（注意力主导型 vs. 冲动主导型）与 LLM 的架构差异是否影响同构的普适性？研究样本有限（来源：矛盾与存疑）。
- **多巴胺干预**：基于多巴胺的 AI 检测虽有前景，但干预效果存在争议（来源：多巴胺）。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个让你拖延的任务**：输入“整理房间”或“写周报”，观察 AI 如何拆解步骤。注意：尝试自己先思考步骤，再与 AI 对比，避免完全依赖。
2. **用 Motion 或类似工具规划明天**：输入所有任务和截止日期，让 AI 自动生成日程。感受“决策负担转移”的效果，但保留手动调整的权利。
3. **做一次“上下文审计”**：关闭所有无关的浏览器标签页、应用通知，只保留当前任务所需的界面。这相当于为你的大脑清理上下文窗口。
4. **记录一次“注意力弥散”事件**：当你发现自己走神时，写下当时的任务和环境。尝试用外部提示（如定时器、环境设计）重新聚焦，就像为 LLM 重置上下文。

## 结语

ADHD 不是缺陷，而是一种“架构选择”——你的大脑是一台强大的生成引擎，只是缺少一个稳定的操作系统。而 agent 工程三年来积累的所有招数，恰好就是为你定制的 OS。问题不在于你是否需要外部调度，而在于你如何让调度层成为可拆卸的脚手架，而不是永久的拐杖。

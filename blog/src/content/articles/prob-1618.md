# Perplexity 之于 ADHD，就像 会褪去的脚手架 之于 LLM——但有人用错了

> 从同构视角实测 Perplexity：它到底补上了哪一层执行功能？

## 问题：怕依赖，又怕没有支撑就崩

你打开 Perplexity，输入一个模糊的问题，它立刻给你一份结构清晰的答案。你松了口气——终于不用自己搜索、筛选、整合了。但几分钟后，你开始焦虑：我是不是越来越依赖它了？如果哪天它没了，我是不是又回到那个什么都做不了的自己？

这种「怕过度依赖工具、又怕没有支撑就崩」的两难，不只是 ADHD 个体的困境。做 Agentic Harness 的工程师也在面对同样的问题：LLM 本身是强大的生成核心，但缺乏可靠的调度层。我们给它搭脚手架——规划、任务分解、工具接口、验证循环——但心里清楚：这个脚手架应该是会褪去的，不能变成永久拐杖。

## 同构：ADHD 大脑与 LLM 是同一类「高产但缺执行调度层」的生成核心

ADHD 大脑的情绪生成系统（如杏仁核）反应强烈，但前额叶（执行功能中枢）调控不足，类似 LLM 生成能力强但缺乏输出控制（来源：AI 与 ADHD 的情绪调节）。两者都是「高产但缺执行调度层的生成核心」。

这意味着，对 ADHD 有效的工具，其设计哲学与 LLM agent harness 高度同构。AI 工具作为外部执行功能层，通过结构化任务分解、时间可视化、外部记忆等方式补偿缺陷（来源：AI 与 ADHD 的情绪调节）。而 LLM 的 harness 同样通过规划、任务分解、工具接口、验证循环等组件来引导生成（来源：规划循环与任务分解）。

## 人物案例：孔子与释迦牟尼的「harness」系统

孔子和释迦牟尼，这两位影响了亚洲文化两千多年的人物，很可能都有显著的 ADHD 特质。孔子精力旺盛到周游列国14年坐不住，冲动爱骂人，思维跳跃，《论语》全是场景化语录，无系统著作。他的 harness 是「克己复礼」——用外在的「礼」作为行为边界，每日反省，「吾日三省吾身」。释迦牟尼29岁说走就走出家，先后跟两个仙人学、苦行6年，不断试错不断放弃，一辈子游行说法，自己一字未写。他的 harness 是「八正道」「持戒」，用戒律管行为，用正念拴住乱跑的念头。

他们的 harness 与 LLM agent harness 同构对应：「日课」对应 LLM 的定时 re-grounding，「秘书/弟子」对应外部调度器。孔子和释迦牟尼都不是依赖一个永久拐杖，而是建立了一套可内化、可褪去的脚手架。

## 实测：Perplexity 到底补上了哪一层执行功能？

Perplexity 的核心价值在于它补上了 **信息检索与整合的执行功能**。ADHD 的工作记忆缺陷导致搜索时容易丢失上下文，频繁切换标签页，最终陷入搜索循环（来源：Saner.AI 资料）。Perplexity 通过一次查询给出综合答案，减少了认知负荷。但这正是「拐杖 vs 脚手架」的边界所在：

- **拐杖用法**：每次遇到问题都直接问 Perplexity，跳过自己的思考过程。长期如此，信息整合能力可能退化。
- **脚手架用法**：先用 Perplexity 快速获取框架，然后自己深入阅读原始来源、做笔记、形成自己的理解。随着能力提升，逐渐减少对工具的依赖。

类似地，Goblin Tools 的 Magic ToDo 将任务分解为小步骤（来源：Goblin Tools 资料），Inflow 通过 CBT 程序训练工作记忆（来源：Inflow 资料），Saner.AI 通过本地记忆减少搜索循环（来源：Saner.AI 资料）——这些工具都应该是脚手架，而非拐杖。

## 核心观点：脚手架 vs 拐杖的边界在于「是否保留内在能力发展的空间」

我的判断是：AI 工具作为外部执行功能层，其合法性取决于它是否最终让用户发展出内在能力。如果工具完全替代了某项功能（如记忆、规划），且用户没有在过程中学习，那就是拐杖。如果工具提供了结构，同时鼓励用户逐步内化（如 Inflow 的 CBT 训练），那就是脚手架。

这一边界在 LLM harness 中同样存在。好的 harness 设计会逐步减少干预，让 LLM 学会在更少的引导下完成任务。例如，Motion 的动态排程工具（来源：规划循环与任务分解）如果只是替用户做所有决定，用户永远不会学会优先级排序；但如果它同时展示决策逻辑，用户就能逐渐内化。

## 诚实面对局限

必须承认，这一观点缺乏长期追踪研究。目前没有证据表明 AI 工具的使用一定导致能力退化，也没有证据表明「脚手架用法」能有效促进内化（来源：矛盾与存疑）。个体差异极大：有些 ADHD 用户反馈强行打断（如番茄钟）更有效，而柔性引导可能不够强力（来源：矛盾与存疑）。此外，ADHD 亚型（注意力不集中型 vs. 冲动型）对 AI 情绪调节工具的反应不同（来源：AI 与 ADHD 的情绪调节）。

## 今天就能试的 3 件事

1. **设置「脚手架边界」**：使用 Perplexity 时，规定自己先尝试搜索 5 分钟，再用工具补充。记录每次使用后的学习收获。
2. **用 Goblin Tools 分解一个任务，但自己先列出步骤**：对比 AI 的分解与自己列表的差异，反思自己的规划盲区。
3. **尝试 Inflow 的 CBT 模块**：每天 10 分钟，专注于认知重构练习，逐步内化情绪调节技能（来源：Inflow 资料）。

脚手架终会褪去，但留下的能力才是真正的支撑。

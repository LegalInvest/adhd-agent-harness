# Perplexity 之于 ADHD，就像 会褪去的脚手架 之于 LLM——但有人用错了

> 从同构视角实测 Perplexity：它到底补上了哪一层执行功能？

## 问题：怕依赖，又怕崩

ADHD 人群常陷入一个两难：不用工具，执行功能缺陷让日常崩溃；用了工具，又怕自己再也离不开，内在能力永远长不出来。工程师在做 Agentic Harness 时也面临类似困境——给 LLM 加上编排层，它就能完成复杂任务；但若 harness 设计成永久拐杖，模型就永远学不会自我调度。这不是巧合。ADHD 大脑与 LLM 共享同一个脆弱性：强大的生成核心（智力/算力）与不可靠的调度层（执行功能/编排）。两边需要的，不是永久的拐杖，而是会褪去的脚手架。

## 同构：ADHD 大脑与 LLM 共享的脆弱性

ADHD 的调度层缺陷表现为工作记忆不足、任务启动困难和时间盲。LLM 的调度层缺陷表现为上下文窗口有限、指令消退和规划循环脆弱。两者都需要外部系统来补偿。在 ADHD 侧，AI 工具如 Goblin Tools 通过将复杂任务分解为小步骤来降低启动门槛（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）；Inflow 通过训练背外侧前额叶皮层来强化工作记忆（来源：Dynamic causal brain circuits during working memory and their functional controllability）。在 LLM 侧，Harness 工程通过上下文传递、工具接口和规划工件来维持模型行为（来源：Building an AI Coding Agent for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

但关键区别在于：脚手架的设计意图是逐步撤除。ADHD 的 AI 工具中，Goblin Tools 的 Magic ToDo 功能被设计为“你的任务分解助手”（来源：The Best AI-Powered ADHD Productivity Tools in 2026），用户可以逐渐学会自己分解；Saner.AI 的本地记忆功能减少搜索循环，但用户最终应能内化检索策略（来源：Best AI Tools for ADHD Productivity in 2026）。而 LLM 的 Harness 也应随模型能力提升而简化。然而，现状是多数工具被当作永久解决方案，没有撤除机制。

## Perplexity 补上了哪一层？

Perplexity 之于 ADHD，就像重锚定系统之于 LLM agent。ADHD 大脑在执行任务时极易发生目标漂移：注意力被无关刺激捕获，或过度聚焦细节而忽略整体（来源：重锚定与目标漂移）。Perplexity 的搜索和引用功能，能快速将用户拉回当前问题，提供即时的事实锚点。这相当于 LLM agent 中的事件驱动系统提醒，对抗“指令消退”（来源：Building an AI Coding Agent for the Terminal）。

具体来说，Perplexity 补上了执行功能中的**重锚定层**。它不像 Motion 或 Tiimo 那样动态调度时间（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），也不像 Inflow 那样训练认知控制（来源：Dynamic causal brain circuits），而是通过提供即时、可靠的外部信息，减少工作记忆负载，让用户能持续聚焦于当前任务。当用户感到“飘走”时，Perplexity 就像一根绳子，把人拉回来。

## 拐杖 vs 脚手架：边界在哪？

边界在于：工具是否在替代内在能力，而非支撑其发展。Goblin Tools 的任务分解功能，如果用户每次都依赖它，而不学习自己的分解模式，就成了拐杖。但若用户将其作为模板，逐渐内化步骤，它就是脚手架。同样，LLM 的 Harness 如果永远不简化，模型就永远无法独立规划。

目前，多数 ADHD 工具设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这引发了过度依赖的风险。证据显示，AI 工具在任务启动、工作记忆和时间盲等维度有效（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），但长期效果和神经可塑性影响未知（来源：矛盾与存疑）。

## 诚实局限

- **个性化不足**：现有工具基于通用认知模型，对共病焦虑等个体差异覆盖有限（来源：AI 与 ADHD 的情绪调节）。
- **多巴胺干预争议**：游戏化设计可能强化依赖而非内在动机（来源：AI 与 ADHD 的情绪调节）。

1. **用 Perplexity 做“重锚定练习”**：当感到注意力飘走时，立即在 Perplexity 搜索当前任务的关键问题，写下答案作为锚点。

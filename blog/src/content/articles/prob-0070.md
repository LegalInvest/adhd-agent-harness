# Goblin Tools 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Goblin Tools：用久了会让 ADHD 更强还是更依赖？

## 问题：工作记忆的「边做边忘」与 LLM 的「无状态」困局

ADHD 患者最熟悉的场景是：打开浏览器想查资料，却发现自己已经站在冰箱前，手里拿着酸奶。工作记忆容量小，导致任务执行中频繁丢失上下文——就像 LLM 每次对话都是全新开始，没有跨会话状态（来源：无状态与外部记忆）。

而 LLM/Agent 工程师的日常是：给模型一个复杂指令，它要么幻觉不断，要么中途遗忘前提。两者的痛点在结构上同构：**都是强大的生成核心，但缺乏可靠的执行调度层**（来源：ADHD 的 AI 工具全景）。

## 同构解法：外部执行功能层 vs Agentic Harness

ADHD 的解决方案是「外部执行功能层」——通过工具提供任务分解、记忆管理和上下文控制。Goblin Tools 的 Magic ToDo 正是典型：输入「整理房间」，AI 输出「捡起地板上的衣服」「擦桌子」等步骤（来源：Goblin Tools）。这直接降低了启动门槛，减轻了工作记忆的认知负荷（来源：认知负荷）。

在 LLM/Agent 侧，对应的概念是 Agentic Harness：通过外部记忆（向量库）、任务分解（Plan-and-Solve）和工具调用（Function Calling）来补偿模型的无状态缺陷。两者共享同一逻辑：**将认知负担外化，让工具承担调度与记忆**。

## 脚手架还是拐杖？边界在于「撤除机制」

脚手架的定义是临时支撑，最终要撤除；拐杖则是永久依赖。当前多数 ADHD 工具——包括 Goblin Tools、Saner.AI、Lex——设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这意味着用户可能形成依赖：没有 AI 分解任务就无法启动，没有 AI 回忆就找不到信息。

证据显示，Goblin Tools 被评价为「简单有用」（来源：Goblin Tools），但缺乏独立随机对照试验验证长期效果（来源：矛盾与存疑）。同样，Agentic Harness 如果设计不当，也会让 LLM 丧失自主规划能力——每次任务都依赖外部编排。

- Saner.AI 通过知识回忆减少搜索循环（来源：Saner.AI）

**LLM/agent 侧**：
- 任务分解技术（如 Plan-and-Solve）能减轻 LLM 的认知负荷，减少幻觉（来源：认知负荷）
- 外部记忆（向量库）是 Agent 保持上下文连续性的标准做法（来源：无状态与外部记忆）
- Agentic Harness 通过编排层提供执行功能，类似 ADHD 的外部执行功能层（来源：ADHD 的 AI 工具全景）

## 争议与局限

1. **过度依赖风险**：工具设计未考虑撤除，长期使用可能阻碍内在执行功能发展（来源：矛盾与存疑）
2. **证据强度不足**：现有证据多为用户反馈，缺乏大规模随机对照试验（来源：矛盾与存疑）
3. **个性化缺失**：ADHD 异质性高，单一工具难以覆盖所有亚型（来源：ADHD 的 AI 工具全景）
4. **隐私问题**：Mem 等工具存储个人记忆，数据安全存疑（来源：ADHD 的 AI 工具全景）

## 核心判断

Goblin Tools 当前是有效的脚手架——它外化了执行功能，让 ADHD 大脑能像 LLM 挂上 Harness 一样运转。但若工具厂商不设计撤除路径，它可能变成拐杖。真正的健康使用，是像工程师调试 Agent 那样：**先依赖 Harness 跑通流程，再逐步减少外部干预，将能力内化**。

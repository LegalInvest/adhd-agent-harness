# 用 Goblin Tools 给 ADHD 补上重锚定与目标漂移，是脚手架还是拐杖？

> Goblin Tools 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：分心走神与目标漂移，ADHD 和 AI Agent 共享的困境

你是否曾坐下想写一份报告，半小时后却发现自己刷了 20 个短视频？或者，你是一个 AI 工程师，你部署的 agent 在长程任务中逐渐偏离原始目标，开始生成无关内容？这两种场景看似无关，却共享同一个底层结构：**一个高产的生成核心，缺乏可靠的执行调度层**。

ADHD 大脑与 LLM 在结构上同构：两者都拥有强大的生成能力（ADHD 的创意发散、LLM 的文本生成），但缺乏可靠的执行调度层（ADHD 的执行功能、LLM 的上下文控制），导致输出不稳定（来源：AI 与 ADHD 的专注力）。ADHD 的「分心走神」或「超聚焦跑偏」，对应 LLM/Agent 的「目标漂移」——当上下文膨胀或环境干扰时，模型会忘记初始指令，像 ADHD 大脑一样被新刺激带偏。

## 重锚定：从曾国藩的「日课」到 Agent 的 system prompt

历史上，ADHD 特质者早已发明了自己的「重锚定」系统。曾国藩，这位晚清名臣，30 岁前典型注意力不集中：浮躁坐不住，天天串门喝酒看杀人，日记里骂自己「无恒」「浮躁」。他的解决方案是「日课十二条」：黎明即起、读书不二、谨写日记反省。这套系统本质是**外部执行功能层**——用固定时间、固定动作、固定反省，强行将注意力锚定在目标上。

这套系统与 LLM/Agent 的「重锚定」机制高度同构。在 Agent 工程中，当长程任务出现目标漂移时，工程师会插入「定时 re-grounding」——重新注入原始 system prompt，或让 agent 定期输出「当前目标是什么？我还在正确轨道上吗？」。曾国藩的「日课」就是他的定时 re-grounding：每天固定的「黎明即起」是重新启动，「写日记反省」是状态日志输出。

## Goblin Tools：为 ADHD 设计的微观重锚定

Goblin Tools 是一款专为 ADHD 设计的 AI 工具集合，核心功能包括：魔法待办（将模糊任务分解为具体步骤）、正式化（将非正式文本转为正式语气）、估算器（预估任务耗时，对抗时间盲）。这些功能直接对应 ADHD 的执行功能短板：

- **任务启动困难**：Goblin Tools 的「魔法待办」将「写论文」分解为「打开文档→写标题→写第一段→…」，降低启动门槛。
- **时间盲**：估算器提供时间感知，弥补 ADHD 大脑无法感知时间流逝的缺陷（来源：时间盲）。
- **注意力漂移**：正式化功能帮助用户清晰表达意图，减少歧义导致的偏离。

但这里有一个关键问题：Goblin Tools 是脚手架还是拐杖？脚手架帮助用户自己搭建，拐杖则完全替代用户行走。Goblin Tools 的设计哲学是「提供结构，保留控制」——它分解任务但让用户选择执行顺序，估算时间但让用户决定是否调整。这与 Focusmate（AI 匹配身体加倍伙伴）形成对比：Focusmate 通过实时问责提供外部结构，但依赖他人存在，长期使用可能削弱内在启动能力（来源：身体在场效应）。

## 证据与争议：脚手架 vs 拐杖的边界

现有证据显示，Goblin Tools 这类工具的有效性取决于使用方式。RescueTime 通过自动记录时间使用，揭示注意力漂移模式，帮助用户识别「时间到底去了哪里」（来源：RescueTime）。但过度依赖自动追踪可能导致用户放弃自我觉察——这正是拐杖风险。

Brain.fm 利用神经锁相技术生成专注音乐，有神经科学理论支持，但缺乏针对 ADHD 的独立临床研究（来源：Brain.fm）。这提醒我们：AI 工具的「外部执行功能层」补偿效果，需要精准对应特定短板，且需避免形成永久依赖（来源：矛盾与存疑）。

同构视角的局限性也需要诚实指出：ADHD 大脑与 LLM 的同构是隐喻而非严格等价，LLM 的「注意力机制」与人类注意力存在本质差异（来源：矛盾与存疑）。因此，Goblin Tools 作为脚手架，应保留用户的主动调度权——让用户决定何时使用、如何使用，而非完全代替决策。

## 今天就能试的行动

1. **用 Goblin Tools 的「魔法待办」分解一个让你拖延的任务**：输入「整理书桌」，看它如何分解为「扔掉垃圾→归类书本→擦拭桌面」等步骤。对比你自己分解的版本，观察哪个更容易启动。
2. **设置「重锚定提醒」**：在手机上设置每 30 分钟一次的闹钟，标签为「当前目标是什么？」。这对应 Agent 的定时 re-grounding。坚持 3 天，记录注意力漂移次数。
3. **用 RescueTime 追踪 2 天**：不刻意改变行为，只观察报告。找出你「以为在专注但实际在分心」的时段，思考如何用 Goblin Tools 的估算器提前规划。
4. **尝试一次「身体加倍」**：使用 Focusmate 或找一位朋友视频通话共同工作 25 分钟。对比独自工作的效率，记录感受。

## 结论：保留调度权，工具才是脚手架

Goblin Tools 的核心价值不是替代 ADHD 大脑的执行功能，而是提供一个可随时拆卸的脚手架。正如曾国藩的「日课」最终内化为他的行为习惯，AI 工具应帮助用户逐渐建立内在的「重锚定」能力，而非永久依赖外部提示。对于 Agent 工程师而言，这意味着设计 system prompt 时应包含「用户可覆盖」的锚定机制，而非硬编码强制规则。

脚手架与拐杖的边界，在于用户是否保留最终调度权。当工具帮你启动任务但让你选择方向，它是脚手架；当工具替你决定下一步做什么，它就成了拐杖。Goblin Tools 目前站在正确的边界上，但每个用户需要自己测试：你的大脑是否在逐渐依赖，还是在借力成长？

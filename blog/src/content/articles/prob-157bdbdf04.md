# Motion 之于 ADHD，就像 agent 的编排层 之于 LLM——但有人用错了

> 他给 Motion(AI 自动排程工具)的差评写得咬牙切齿:「用了六周,它把我的日历排得密不透风,而我完成率不到三成——每天睁眼就欠着日历的债,比没用之前更焦虑。」同一个工具,他的朋友却用出了完全相反的体验:「它救了我——我终于不用每天早上决定先干什么了。」复盘两人的配置,差别一目了然:差评者把**所有事**塞进了 Motion——目标、愿望、「有空想做」全部录入,并保留了默认的乐观时长估算;好评者只放**已承诺的交付**,时长手动上浮 50%,而且每天只信头三件。**差别的本质是权限设计:编排层该拿到多大的授权?agent 工程对这个问题有成熟答案——编排器只调度「已确认的任务队列」,从不调度「愿望池」;而把愿望池直通调度器的人,得到的不是秩序,是一份每天羞辱自己的排期表。**

收敛:本文只回答一个配置问题——**自动排程工具的正确授权边界在哪(什么进调度、什么不进)?「排不完的日历」为什么比「没日历」伤害更大?以及给自动排程的三条落地配置规则?**

## 穿透:调度器的输入必须是"已裁决的承诺",不能是"未裁决的欲望"——裁决是你的活,不能外包给排程算法

agent 编排层的纪律:编排器从任务队列取任务、分配资源、排定顺序——但**什么任务能进队列,由队列之外的裁决环节决定**(编排蓝图篇的规划器与审批闸)。把 Motion 用崩的人,几乎都犯同一个错:**把「裁决」和「调度」合并外包了**——愿望(想学的课、想写的书、想清的柜子)未经「本周真的要做吗」的裁决直接入库,算法忠实地把它们全排进日历——于是日历从「今天的承诺」变成「欲望的全量镜像」,而欲望的全量永远排不完。更糟的是后续:**排不完→每天大量任务被自动顺延→顺延常态化→「日历上的时间」失去约束力→日历失明**(反正它会重排,现在这条不用当真)——工具越智能地帮你顺延,越快地教会你无视它。这就是「比没用更焦虑」的账目:没日历时你只面对今天的模糊,有一份排不完的日历时你每天面对具体的、量化的落后。

三条配置规则:**①入库闸:只放承诺,不放愿望**——能进 Motion 的任务必须过一句质检:「这件事本周不做,会有真实后果吗?」有,进;没有,回选题池/愿望清单(那是另一个系统,删除权篇管它)——**日历是法庭,不是许愿池**;②**时长上浮 50%+每日只信头三件**——你的估时天生乐观(规划谬误在这个剖面上加倍),上浮不是保守是校准;算法排出的一天里,只把前三件当真(其余视为「如果有神迹」区)——完成头三件的日子记为胜利日;③**顺延两次即降级**——同一任务被自动顺延两次,强制离开调度器(回到裁决环节:要么改小、要么删除、要么升级为「明早第一件」)——**不许任何任务在日历里无限漂流,漂流的任务是日历公信力的白蚁。**

用对了的样子:调度外包后,每天早上那个最贵的决策(「先干什么」)确实消失了——这是真收益(决策疲劳的大头被砍掉);但**裁决(什么值得进本周)、点火(排上了怎么启动)、验收(做完了算不算数)三个环节依然是你的**——Motion 是编排层,不是整个 agent 系统;把它当整个系统用的人,差评写得再痛也怪不到算法头上。

## 验证

可测:三规则配置后两周——头三件完成率(目标 >80%)、任务平均顺延次数(应 <1)、以及主观的「日历还看不看」。可证伪:若你的工作是高中断型(客服、急诊式岗位),预排日历本身不适配——那不是配置问题,是工具与工作结构的错配,换「反应式清单」路线。

## 决策

做什么:入库闸、上浮 50%、只信头三件、顺延两次降级。

不做什么:不要把愿望池接进调度器;不要用完成率羞辱自己(先查配置);不要因为一次用崩就否定「调度外包」这个方向(方向是对的,授权错了)。

先做什么:今天清一次库:把日历里所有「无真实后果」的条目移出去——看看剩下的日历长什么样。

## 边界

规划谬误与决策疲劳有行为科学基础(GRADE 中);Motion 的具体机制以其当前产品形态为准(会变);「编排层」类比按 B 级功能同型使用——本文的落点恰是差异:算法可以排时间,不能替你裁决价值。不构成对特定产品的推荐或否定,配置原则适用于一切自动排程类工具。

## 今天就能试的 3 件事

1. 清一次调度器的库(愿望全部请出)。
2. 把现有任务时长批量上浮 50%。
3. 给明天标出「头三件」。

本文服务于人生 Harness 金字塔的**执行层(调度授权)**:把方向盘交给算法之前,先想清楚你交出去的是哪个环节——排序可以外包,裁决不能;一份只装着真承诺的日历,才配得上被你当真——而一份被你当真的日历,才救得了你。

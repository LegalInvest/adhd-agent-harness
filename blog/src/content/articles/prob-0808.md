# Saner.AI 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Saner.AI：它到底补上了哪一层执行功能？

你打开 Saner.AI，输入“写一份季度报告”，它立刻吐出分步计划：第1步收集数据，第2步分析趋势，第3步撰写初稿……你松了口气，任务终于“启动”了。但第二天，你发现自己只是被动执行它给出的步骤，依然不会自己拆解任务，甚至离开 Saner.AI 后连“先做什么”都判断不了。

这不是你的问题——这是工具被用错了方向。

## 同构的困境：两个高产核心，缺同一个调度层

ADHD 大脑与 LLM 在结构上惊人相似：两者都是“高产但缺执行调度层的生成核心”。ADHD 的智力与创造力（生成核心）与 LLM 的生成能力类似，但缺乏执行功能（调度层）来组织行动（来源：AI 与 ADHD 的时间管理 - 同构视角）。LLM agent 依赖 harness 工程——通过 context delivery、tool interfaces、planning artifacts 等外部脚手架来分解长任务（来源：规划循环与任务分解 - LLM/harness 一侧）。

你发现了吗？Saner.AI 的任务分解功能，本质上就是 ADHD 大脑的 planner-executor harness。它把模糊的“写报告”拆成可执行的子任务，对应 LLM agent 的“多步推理+工具调用”模式（来源：工具使用与认知卸载 - LLM/harness 一侧）。Goblin Tools 也做同样的事：“将模糊的工作分解为更小的行动”（来源：工具使用与认知卸载 - ADHD 一侧）。

但这里有一个关键分歧：这些工具应该充当“脚手架”还是“拐杖”？

## 脚手架 vs 拐杖：规划循环的正确打开方式

脚手架是可撤除的临时支撑，帮助用户逐步内化能力；拐杖则是永久替代，长期使用可能阻碍内在执行功能的发展（来源：矛盾与存疑 - 过度依赖 vs 脚手架可撤除性）。当前多数工具（如 Saner.AI、Goblin Tools）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这意味着，如果你只是让 Saner.AI 替你规划一切，而不去理解它为何这样分解，你永远不会学会自己分解任务。

正确用法是：把工具当作“规划伙伴”，而非“规划替身”。正如 ADHD 生产力专家所言：“use AI as your planning partner to create a system that actually works with your ADHD brain”（来源：规划循环与任务分解 - ADHD 一侧）。

## 从工具设计看同构：Motion、Reclaim.ai、Tiimo 的启示

Motion 通过自动排程与动态调整消除“下一步该做什么”的决策负担（来源：Motion - AI 如何介入）。这对应 LLM agent 的 planner 循环：持续评估优先级和可用时间，实时重建日程。Reclaim.ai 则更强调“保护”而非“填充”——通过创建深度工作块自动防御会议侵占（来源：Reclaim.ai - AI 如何介入），这相当于 agent 的 context window 保护机制。Tiimo 将时间转化为可见元素（来源：Tiimo - 与 ADHD 的关系），直接应对时间盲，类似 LLM 的上下文窗口管理。

这些工具的共同点：它们都提供了外部调度层，但没有教用户如何自己调度。

## 今天就能试的 3 条行动

1. **用 Saner.AI 分解任务后，反问自己“为什么这样拆？”**：把工具给出的步骤抄下来，然后尝试解释每一步的逻辑。这是从“被动执行”转向“主动理解”的第一步。
2. **每周一次“无工具规划日”**：选一天，只用纸笔或 Todoist 手动规划（来源：工具使用与认知卸载 - ADHD 一侧），强迫自己练习分解能力。
3. **对比 Motion 的自动日程与你自己手动排的日程**：观察 Motion 如何调整优先级，然后思考它的决策逻辑。这相当于在“调试”你的外部调度层。

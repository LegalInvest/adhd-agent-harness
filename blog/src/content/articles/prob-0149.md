# 用 ChatGPT 给 ADHD 补上无状态与外部记忆，真的能像 agent 那样工作吗？

> 把 ChatGPT 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

## 问题：每天重启的大脑，和每天重启的模型

你是否有过这样的经历：早上打开项目文档，完全不记得昨天卡在哪一步；跟同事聊到一半，突然忘了要说什么；明明有个好点子，转头就消失在空气里。ADHD 大脑的工作记忆容量有限且不稳定，导致跨会话任务难以持续——每天需要重新回忆项目上下文，否则容易偏离轨道（来源：无状态与外部记忆）。

与此同时，LLM 工程师们也在面对类似困境：标准的 LLM 调用是无状态的，每次对话都是独立的，模型不保留跨会话的记忆（来源：The Anatomy of an AI Agent: Memory, Tools, Planning, and Execution Explained）。没有 harness 的 LLM 只是一个无状态的 API 调用（来源：Building an AI Agent Harness from Scratch）。

于是，一个自然的想法浮现：用 ChatGPT 当作 ADHD 的外部记忆和执行层，它真的能像 agent 那样工作吗？

## 同构：两个生成核心，同一类匮乏

ADHD 大脑与 LLM 在结构上同构：两者都是强大的生成核心，但缺乏可靠的执行调度层（来源：ADHD 的 AI 工具全景）。ADHD 的智力与创造力可以像 LLM 的文本生成一样高产，但工作记忆容量小、任务启动困难、时间盲等执行功能缺陷，就像 LLM 缺少上下文管理和工具调用。

LLM agent 的解法是搭建 harness——通过外部记忆（向量数据库）、检索增强（RAG）和工具调用，把无状态的 API 变成有状态的助手。生产级 agent 系统组合使用会话内上下文记忆、长期检索向量库和结构化知识库（来源：The Anatomy of an AI Agent）。

ADHD 的解法同样如此：借助外部工具提供持久记忆、清晰的下一步行动、日历支持，减少每日重复输入上下文的摩擦（来源：Best AI Tools for ADHD Productivity in 2026）。本质上，ADHD 大脑需要一个“永远在线”的外部记忆体来补偿内部工作记忆的波动（来源：How AI Helps ADHD Brains Work Smarter, Not Harder）。

## 实测：ChatGPT 补的是哪一块短板？

当我们把 ChatGPT 当作外部执行层来使用时，它主要补偿的是**工作记忆**和**任务启动**。

**工作记忆侧**：ChatGPT 的对话上下文可以充当短期记忆，但一旦关闭会话或换设备，记忆就消失了。这正是无状态的核心问题。为此，用户需要借助 Mem 或 Saner.AI 这类工具来提供持久记忆。Saner.AI 专注于知识回忆和本地记忆，旨在减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026）。Mem 则通过 AI 存储个人记忆，但数据安全和用户信任问题需进一步解决（来源：ADHD 的 AI 工具全景）。

**任务启动侧**：ChatGPT 可以像 Goblin Tools 的 Magic ToDo 一样，将复杂任务分解为小步骤，降低启动焦虑。Goblin Tools 被评价为“将压倒性的事情变成一系列不压倒性的事情”（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。Lex 更进一步，允许通过单一指令触发复杂、多步骤的任务序列，实时适应大脑的工作方式（来源：Best AI Tools for ADHD Productivity in 2026）。

但这里有一个关键区别：ChatGPT 本身并不主动管理上下文。ADHD 大脑常被环境带偏，使用通用聊天机器人时，“they often add another inbox, another prompt, and another place to lose the thread”（来源：上下文工程）。因此，ChatGPT 更像一个被动的工具，需要用户主动设计提示和流程，才能发挥 harness 的作用。

## 核心观点：脚手架 vs 拐杖

我的判断是：当前用 ChatGPT 给 ADHD 补上无状态与外部记忆，**在结构上确实与 LLM agent 的 harness 同构，但实际使用中更接近“拐杖”而非“脚手架”**。

“脚手架”是可逐步撤除的临时支撑，设计目的是最终让用户独立完成任务。但多数工具如 Goblin Tools、Saner.AI 设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。长期依赖是否阻碍内在执行功能发展？这是当前最大的争议。

另一方面，LLM agent 的 harness 也在不断演进，但同样面临上下文膨胀与推理退化的问题：“Effective autonomous assistance requires strict safety controls and highly efficient context management to prevent context bloat and reasoning degradation”（来源：上下文工程）。

因此，无论是 ADHD 还是 LLM agent，核心挑战都是：**如何设计一个既能提供即时支持，又不会让人（或模型）能力退化的外部系统。**

## 局限与争议

必须诚实指出，现有证据多为短期或个案，缺乏随机对照试验验证 AI 工具对 ADHD 核心症状的长期改善（来源：ADHD 的 AI 工具全景）。ADHD 大脑与 LLM 同构的命题也主要来自概念类比和工具案例，缺乏大规模实证（来源：矛盾与存疑）。

此外，多巴胺干预的有效性存在争议。虽然多个工具声称通过神经科学原理改善执行功能，但基于多巴胺的 AI 检测存在伦理和效果争议（来源：ADHD 的 AI 工具全景）。

## 今天就能试的行动

1. **用 ChatGPT 做任务分解**：输入一个让你焦虑的任务（如“整理房间”），要求它列出 5-10 个具体步骤。完成后立即执行第一步。
2. **建立外部记忆库**：在 Mem 或 Saner.AI 中记录项目关键信息，每天开始工作前先回顾，减少重新加载上下文的成本。
3. **设计上下文提示词**：为 ChatGPT 创建一个“项目助手”提示，包含项目背景、当前状态、目标，每次对话前先粘贴，模拟 agent 的上下文管理。
4. **设置撤除检查点**：每周评估一次，哪些步骤你已经可以不用工具完成，逐步减少依赖。

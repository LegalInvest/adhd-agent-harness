# 为什么 done list 最适合做 ADHD x AI 的第一个小产品——它不要求改变你，只负责证明你做过

> 产品脑暴会上,清单列了一白板:AI 任务拆解器、智能日程、专注计时、习惯养成……每一个都有人做过,每一个的用户留存曲线都是同一条陡峭的下坡。有人提了一个不起眼的方向:「done list——不记要做什么,只记做了什么。」会议室安静了几秒,因为大家同时意识到这个方向的怪异之处:**白板上其他所有产品都在要求用户改变(变得有计划、变得专注、变得自律),只有这一个,对用户的唯一要求是「你已经做了的事,允许它被看见」。**而这恰恰可能是它能活下来的原因。

收敛:本文只回答——**done list(已完成清单)对 ADHD 的作用机制是什么?为什么「第一个小产品」的选型标准恰好选中它(而不是那些更酷的)?它的最小形态和 AI 加成怎么设计?**

## 穿透:todo 记录你的债务,done 记录你的资产

机制三层。第一层,**对抗记忆偏差**:ADHD 的一天常常在主观上「什么都没干」——不是真的没干,是干过的事不被记账:被打断的十件小事、临时救的三场火、顺手帮的两个忙,全部蒸发,只有「没完成的那个大项」留在意识里定罪。done list 的功能是把蒸发的资产找回来——**它不改变你做了多少,只改变「做了多少」能否被你自己看见**。第二层,**修复自我效能的现金流**:自我效能靠「我做到了」的证据供养,而 ADHD 的 todo 系统是效能的粉碎机(每天结账都是未完成项的红字);done list 把记账方向反过来,每天入账的全是完成项——同样的一天,两种账本,两种自我叙事。第三层,**零执行功能税**:这是产品选型的胜负手——todo 系统要求计划(拆解、排程、估时,全是 ADHD 的重灾功能),done list 只要求「事后记一笔」,而且晚记、漏记、补记都不损害系统(对比 streak 断一天的崩溃感)——**它是极少数「对最差状态友好」的记录系统:你状态再烂,烂到只做了一件事,它也只是安静地记下那一件,不评判、不欠账、不红。**

「第一个小产品」的选型逻辑,用这个案例正好讲透四条标准:①**不与用户的神经特征为敌**(所有「习惯养成」类产品都在赌 ADHD 用户能坚持——那是拿产品的命赌用户的短板;done list 赌的是「人总会做一些事」——稳赢);②**失败无惩罚**(用户中断一个月回来,todo 系统给他看堆积的债,done list 给他看空白——空白不羞辱人);③**单点起步,天然扩展**(最小形态小到一个输入框,但资产会自然长厚——见下);④**AI 加成是锦上添花而非承重墙**(没有 AI 它也成立;有 AI 它变好:每周把 done 流水喂给模型,生成「你以为的一周 vs 实际的一周」对比报告、模式发现(「你的高产时段集中在上午十点」「你做了 11 件计划外但有价值的事」)、以及给低谷期的证据反驳(「你说这个月一事无成,数据显示你完成了 63 件事,列表如下」)——**AI 在这里的角色是辩护律师,拿着你自己的档案,替你反驳你的自我指控。**)

## 验证

两周自测(不需要等产品,备忘录就行):每天睡前三分钟,写下今天做完的一切(粒度放到「回了那个难回的消息」级别),周末通读。测两个指标:主观的「这周有没有干成事」评分,和通读后的修正幅度——修正幅度就是你的记忆偏差的大小。可证伪:若两周后你的自我评价毫无变化、记录也确实寥寥,两种可能:粒度还是太粗(把「做了家务」拆成具体动作),或你正处在需要临床支持的低谷(记录工具救不了状态层的问题,请别用「连 done list 都没用」再捅自己一刀)。

## 决策

做什么:个人层面今天就用备忘录跑起来;产品层面按四标准做最小形态(一个输入框+语音入口+周报 AI);把它定位为「ADHD 内容宇宙」的入口产品(前文讲的模板→产品路径,done list 是模板产品化的第一号种子)。

不做什么:不要给 done list 加 streak、目标、提醒轰炸(那是把 todo 的病移植过来——它的全部产品哲学就是「无要求」);不要让 AI 报告变成新的评判(「本周完成数下降 20%」这种话永远不许出现——辩护律师不做检察官)。

先做什么:今晚睡前三分钟,写下今天的第一份 done list——包括「读完了这篇文章」。

## 边界

记忆的负性偏差与 ADHD 自我效能损耗有文献基础(GRADE 中);done list 作为干预无对照研究,「完成记录改善自我评价」为机制推理+社区实践(本文未做正式 A/B/C 分级)。它是自我叙事的辅助,不是抑郁的治疗;若「一事无成感」持续且弥漫,请寻求专业支持。

## 今天就能试的 3 件事

1. 今晚写第一份 done list,粒度放到「回了一条难回的消息」。
2. 周末做一次「以为 vs 实际」对比:先凭感觉给这周打分,再通读记录后重新打——量一量你的记忆欠你多少。
3. 若你在做产品:把这个方向和白板上其他方向各问一遍「它要求用户改变吗」——感受选型标准的锋利。

本文服务于人生 Harness 金字塔的**叙事层**:你不缺努力,缺的是一个肯替你记账的会计——todo list 记你欠世界的,done list 记世界欠你的;而后一本账,你欠自己太久了。

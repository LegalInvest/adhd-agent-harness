# 如果把 ADHD 大脑当成生成核心，Goblin Tools 能不能当它的外部执行功能层？

> 用 agent 工程的思路重新理解 Goblin Tools 对 ADHD 的真正价值。

## 问题：当生成核心缺少调度层

ADHD 大脑被描述为“高产但缺执行调度层的生成核心”——发散思维、联想能力惊人，但计划、组织、任务启动等执行功能先天偏弱（来源：ADHD 大脑与 LLM 的同构）。这与 LLM agent 的困境惊人相似：LLM 本身是无状态的文本生成器，缺乏内置的编排、记忆和工具调用层，必须依赖 agent scaffolding（如 LangChain、AutoGPT）来赋予它“执行功能”。那么，如果 ADHD 大脑是生成核心，Goblin Tools 能否充当它的外部执行功能层？

## 同构：两套 harness 的对应

从系统架构看，ADHD 大脑与 LLM 都需要外部脚手架来补偿调度缺陷。曾国藩是典型例子：他 30 岁前“浮躁坐不住”，日记里天天骂自己“无恒”，读书“不能终一行”。他的解决方案是“日课十二条”——黎明即起、读书不二、写日记反省。这套 rigid 的日程和规则，本质上就是一个外部执行功能层：它把抽象目标（“修身齐家”）分解为可执行的每日动作，并强制 re-grounding（每日反省）。这与 LLM agent 的“定时 re-grounding”机制（如每轮对话前重设系统提示）同构。曾国藩的“结硬寨打呆仗”对应 agent 的“谨慎执行”——用最笨最稳的方法抵消冲动。

Goblin Tools 的核心功能 Magic ToDo 正是这种逻辑的数字化：用户输入模糊任务（如“清理厨房”），AI 自动分解为具体子步骤，并允许调节“辣度”控制粒度（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。这相当于为 ADHD 大脑提供了一个动态的“任务分解器”，降低启动门槛。同时，时间估计工具帮助克服“时间盲”（来源：Goblin Tools），相当于 agent 的“时间预估模块”。

## 证据与边界：脚手架还是拐杖？

证据方面，用户反馈称 Goblin Tools“简单有用”（来源：Best AI Tools for ADHD Productivity in 2026）。但争议在于：它是否会导致过度依赖？wiki 资料指出，AI 工具可能成为永久拐杖而非可拆卸脚手架（来源：拐杖与脚手架）。曾国藩的日课是内化的习惯，而 Goblin Tools 是外部工具——一旦断网或停用，用户可能回到原状。此外，个体差异显著：部分 ADHD 用户需要更强制性的结构（如 Motion 的自动排程），而另一些则对柔性引导更敏感（来源：超聚焦）。

另一个局限是证据强度：当前多为概念类比和初步用户反馈，缺乏大规模随机对照试验证明 Goblin Tools 的临床有效性（来源：矛盾与存疑）。与 Saner.AI 专注于知识回忆、Motion 专注于自动排程不同，Goblin Tools 更聚焦于任务分解，但缺少跨会话记忆和长期规划能力——这对应 LLM agent 的“无状态”问题，需要外部记忆系统（如向量数据库）来补偿（来源：无状态与外部记忆）。

## 核心判断：Goblin Tools 是优秀的“启动层”，但非完整 harness

我认为，Goblin Tools 最适合作为“任务启动层”——它解决的是从“想做”到“开始做”之间的鸿沟，这正是 ADHD 执行功能障碍的核心痛点（来源：任务启动）。但它无法替代完整的执行功能层，后者需要包括长期规划（如 Motion）、知识管理（如 Saner.AI）和情绪调节（如情绪失调的应对）。曾国藩的 harness 是综合性的：日课、反省、写日记，覆盖了日程、自我监控和情绪出口。Goblin Tools 目前只覆盖了“分解”和“时间估计”两个维度。

## 今天就能试的行动

1. **用 Magic ToDo 分解一个拖延的任务**：输入你拖延最久的任务（如“写报告”），调节辣度到 3-4，然后只做第一小步。
2. **结合时间估计工具**：对分解后的子任务逐一估算时间，对比 AI 的估计，训练自己的时间感知。
3. **设置“反省提醒”**：每天固定时间（如睡前）用 Goblin Tools 的笔记功能记录“今天完成了什么？什么没完成？为什么？”——模拟曾国藩的日记反省。
4. **尝试组合工具**：将 Goblin Tools 的分解结果导入 Motion（自动排程）或 Saner.AI（知识回忆），构建更完整的 harness。

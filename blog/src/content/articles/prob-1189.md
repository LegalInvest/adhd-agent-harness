# Structured 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Structured：它到底补上了哪一层执行功能？

## 问题：为什么你总是“知道该做什么，但就是做不了”？

如果你是一个ADHDer，你一定熟悉这个场景：脑子里有一堆想法，计划也写好了，但一到执行就卡住——不是不想做，而是大脑像断了线的路由器，信号发不出去。同样，如果你在调试一个LLM agent，你也会遇到类似困境：模型生成了完美的计划，但调用工具时要么参数写错，要么中途忘记上下文，要么在重复性步骤中“幻觉”出一个不存在的函数。两边的问题结构一模一样：**生成核心（generative core）很强，但执行调度层（executive function / agent harness）缺失或不可靠**。

这正是本文要论证的核心：ADHD大脑与LLM在认知架构上是同构的。两者都是“高产但缺调度”的生成引擎，因此都需要外部脚手架（harness）来补偿精确、重复、状态维护类任务的执行缺陷。但很多人把脚手架用成了永久拐杖，反而让内在调度能力进一步萎缩。

## 同构证据：ADHD大脑与LLM的调度层缺失

ADHD的执行功能缺陷被描述为“大脑的驾驶座，但方向盘后没有人”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。工作记忆不稳定、任务启动困难、时间盲——这些不是智力问题，而是调度层的失效。最新研究进一步揭示了深层平行：ADHD患者的工作记忆容量未必差，但“自上而下的控制和调节能力不足”，呈现“强记忆、弱控制”的认知剖面（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这种模式与LLM惊人相似——LLM拥有强大的语言生成能力，但单独使用时缺乏可靠的调度，容易“上下文膨胀和推理退化”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

更惊人的是，Transformer的自注意力机制在训练后发展出模仿前额叶-纹状体门控的操作，证明了计算层面的同构性（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。这意味着ADHD和LLM的调度缺陷不是表面的类比，而是有深层的计算机制作为基础。

## 人物案例：孔子与张旭的“harness”系统

历史上，许多高成就ADHDer用自创的“harness”补偿调度缺陷。孔子（春秋儒家创始人）身高1米9，精力旺盛，周游列国14年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予“朽木不可雕”；但对音乐超专注到“三月不知肉味”，对种地等俗事零耐心。他的harness是“克己复礼”——用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”），70岁才做到“从心所欲不逾矩”。这套系统本质上是**外部调度器**：用固定的仪式（礼）和反思（省）来补偿内在的冲动和注意力漂移。这就像LLM agent中的**定时re-grounding**——通过外部提示或记忆刷新防止上下文丢失。

张旭（唐朝草圣）的harness则是“以酒入书”。他每次喝醉后号呼狂走，以头濡墨作书，醒了自视以为神。他看公孙大娘舞剑悟笔法，从自然中找灵感。酒和草书是他的**超聚焦触发器**，帮他进入心流状态。这对应LLM agent中的**工具调用**——通过外部刺激（酒、舞剑）触发高质量生成，而非依赖内在意志力。

## 工具实测：Structured到底补上了哪一层？

回到Structured——这款时间块规划工具。对于ADHDer，Structured的核心价值不是“规划”，而是**外部状态维护**。ADHD大脑无法可靠地维护“接下来该做什么”这个状态变量，Structured用时间块和提醒代替了内部调度器。这就像LLM agent中的**function calling**——agent把精确计算（如数学、日期）外包给工具，自己只负责生成意图。Structured把“什么时候做什么”这个精确任务外包给视觉化时间轴，ADHD大脑只需执行。

但很多人用错了：他们把Structured当作“完美的计划本”，试图用它规划每一分钟，结果一旦被打乱就崩溃。真正的用法是：**只放关键锚点，留出缓冲**。Structured应该是脚手架，不是拐杖。脚手架帮你建造，但你仍需自己攀登（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

## 争议与局限：脚手架 vs 拐杖

同构性类比虽然有力，但缺乏实证基础，目前仅是一种理论假设（来源：矛盾与存疑）。工具宣称的效果多基于用户报告，缺乏严格临床验证（来源：矛盾与存疑）。更重要的是，过度依赖AI工具可能削弱内在执行功能，从脚手架变为永久拐杖（来源：拐杖与脚手架）。例如，长期使用Goblin Tools分解任务，可能让用户失去自己分解的能力。同样，LLM agent过度依赖手动修正提示，会限制模型自主发现问题解决模式的能力（来源：Working Memory Identifies Reasoning Limits in Language Models）。

## 今天就能试的行动

1. **用Structured只放3个关键锚点**：每天只规划3个非做不可的时间块，其余时间自由流动。这相当于给LLM agent设置“关键工具调用点”，而非全流程编排。
2. **用Goblin Tools的Magic ToDo分解一个“卡住”的任务**：输入你拖延最久的任务，调整“辣度”到你觉得可执行的程度，然后只做第一步。
3. **用Saner.AI捕获一个灵感，不整理**：直接语音记录，让AI自动组织。这相当于LLM agent的“日志记录”——先存下来，后续再检索。
4. **每周一次“无工具日”**：刻意不用任何AI工具完成一项简单任务（如写购物清单），训练内在调度肌肉。这对应LLM agent的“无工具推理测试”——评估模型在没有外部函数时的基线能力。

## 结语

Structured之于ADHD，就像function calling之于LLM——都是通过外部脚手架补偿调度缺陷。但记住：脚手架是帮你建造，不是替你建造。真正的工程是设计一个你最终可以逐渐撤掉的系统，而不是永远依赖它。孔子70岁才“从心所欲不逾矩”，张旭醒后视“以头濡墨”为神品——他们的harness最终内化成了能力。你的目标也是一样：用工具建立结构，然后慢慢学会自己调度。

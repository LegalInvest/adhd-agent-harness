# 如果把 ADHD 大脑当成生成核心，Todoist 能不能当它的外部执行功能层？

> 用 agent 工程的思路重新理解 Todoist 对 ADHD 的真正价值。

## 问题：为什么 Todoist 对 ADHD 人群常常“开始即放弃”？

你下载了 Todoist，充满干劲地创建了十几个项目，设置了优先级和截止日期。一周后，列表变成了一堆过期未完成的任务，你甚至不想打开它。这不是你的错——问题出在工具与大脑的匹配上。

ADHD 大脑的核心困境是：**想法很多，行动困难**。执行功能（计划、组织、启动、时间管理）先天偏弱（来源：执行功能障碍）。这意味着，一个单纯的待办事项列表，本质上只是把你的“未完成”摆在你面前，却无法帮你解决“如何开始”和“如何持续”这两个根本问题。你需要的是一个**外部执行功能层**——它像 agent 的 harness 一样，替你完成调度、记忆和工具调用。

## 同构：ADHD 大脑与 LLM 都需要 harness

想象一下，如果让 GPT-4 直接去执行一个跨越多天、包含多个依赖关系的项目，会发生什么？它会忘记上下文，偏离目标，输出混乱。这正是为什么工程师在构建 AI agent 时，必须围绕 LLM 搭建“agent scaffolding”——即“围绕 LLM 构建的软件架构和工具，使其能够执行复杂、目标驱动的任务”（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。

ADHD 大脑与 LLM 在结构上同构：两者都是强大的生成核心，但缺乏可靠的内置执行调度层（来源：ADHD 大脑与 LLM 的同构）。神经科学研究显示，ADHD 患者前额叶执行功能网络激活不足（来源：ADHD × AI 的科学与研究前沿），而 Transformer 模型在长上下文任务中同样表现出类似执行功能缺陷——在 Stroop 冲突任务中，随序列变长，模型在冲突试次上骤降到随机水平（来源：ADHD 大脑与 LLM 的同构）。两边都需要一个**外部 harness** 来补偿。

## 案例：曾国藩的“日课十二条”就是手工版 harness

晚清名臣曾国藩，典型注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书慢到“他人目下二三行，余或疾读不能终一行”。他的解决方案是**日课十二条**：黎明即起、读书不二、谨言、写日记反省……这套规则本质上是一个**手工编排的 agent harness**——每天固定时间执行固定动作，用外部脚本代替内部调度。他的“结硬寨打呆仗”更是用最笨最稳的方法抵消冲动，就像 agent 的 ReAct 模式：先思考再行动，每一步都有记录。

对应到 LLM 侧：日课相当于**定时 re-grounding**，写日记相当于**外部记忆**，而“读书不二”则像**单一任务上下文窗口**——一次只处理一个上下文，避免注意力分散。曾国藩的成功证明：**外部 harness 可以补偿执行功能缺陷**，即使它只是一套纸笔规则。

## Todoist 能否胜任外部执行功能层？

回到 Todoist。它本身只是一个任务列表，但它的功能集——标签、过滤器、优先级、项目分层、提醒、集成——可以组合成一个**基本的 harness**。关键在于**如何配置**，而不是工具本身。

- **编排层**：Todoist 的“过滤器”可以创建动态视图，比如“今天必须做且优先级高”，这相当于 agent 的**任务队列调度**。但 Todoist 不能自动重新排序——Motion 可以自动根据截止日期和优先级动态调整日程（来源：Motion），而 Todoist 需要你手动维护。
- **记忆层**：Todoist 的“备注”和“评论”可以存储上下文，但不如 Saner.AI 那样专门针对知识回忆优化（来源：Saner.AI）。ADHD 患者的工作记忆不稳定，需要快速找回信息，Todoist 的搜索功能有限。
- **工具层**：Todoist 的集成（如日历、邮件）可以充当工具调用，但分解复杂任务需要外部 AI——Goblin Tools 的 Magic ToDo 能将“清理厨房”自动分解为可执行的子步骤（来源：Goblin Tools），而 Todoist 没有原生分解能力。

所以，Todoist 可以成为 harness 的一部分，但**不是完整的 harness**。它更适合作为**任务存储和提醒层**，而真正的执行功能补偿需要组合其他工具：Goblin Tools 做分解，Motion 做排程，Saner.AI 做记忆。

## 脚手架 vs 拐杖：过度依赖的风险

这里有一个重要边界。曾国藩的日课是**脚手架**——他用它训练自己的执行功能，而不是替代它。他坚持写日记反省，说明他始终在自我监控。如果 Todoist 只是让你“把大脑外包”，长期可能导致执行功能进一步退化（来源：ADHD × AI 的矛盾与存疑）。真正的脚手架应该**在辅助的同时训练能力**，比如逐步减少提醒频率，或者让你自己手动调整优先级。

另外，同构框架本身仍是理论类比，并非经过验证的科学事实（来源：ADHD × AI 的矛盾与存疑）。不同 ADHD 亚型可能对 harness 的需求不同，部分患者可能以注意力分散为主，而非执行功能缺陷。工具宣称的证据也不足——Motion 缺乏独立临床验证（来源：ADHD × AI 的矛盾与存疑）。

## 核心观点：Todoist 不是答案，但可以是 harness 的起点

我的判断是：**Todoist 不能单独充当 ADHD 的外部执行功能层，但它可以成为 harness 架构中的核心节点**——任务存储和触发。真正的解决方案是**组合**：用 Goblin Tools 分解任务，用 Motion 自动排程，用 Todoist 作为执行记录，再配合定期回顾（像曾国藩的日记）。这正是一个**复合 AI 系统**的思路：将 LLM 置于与记忆、工具和决策逻辑的控制循环中（来源：Agent Scaffolding）。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个任务**：打开 Magic ToDo，输入“准备下周的会议报告”，调节辣度到你觉得合适的粒度，然后把子步骤复制到 Todoist。
2. **在 Todoist 中创建一个“今日焦点”过滤器**：筛选出今天到期且优先级为 P1 的任务，设为默认视图，减少决策负担。
3. **设置每日回顾提醒**：像曾国藩写日记一样，每天花 5 分钟检查今天完成了什么，哪些没完成，为什么。这相当于 agent 的**事后反思**。
4. **尝试 Motion 的免费试用**：让它自动排程一周，感受“无需决策”的体验。如果有效，考虑将 Todoist 任务同步到 Motion。

记住：工具是脚手架，不是拐杖。你的大脑仍然是生成核心——它只需要一个更好的外部调度层。

# Endel 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Endel：它到底补上了哪一层执行功能？

你用过 Endel 吗？那个号称用 AI 生成自适应音景、帮你进入专注状态的 app。很多 ADHD 用户对它又爱又恨：有人觉得它神奇，戴上耳机就能沉浸；有人觉得它鸡肋，背景音乐而已，该走神还是走神。问题不出在 Endel 本身，而出在用法——就像有人把 LLM 的 function calling 当万能 API，却忘了它只是“精确计算的外包接口”。

## 同构视角：两个“生成核心”的同一困境

ADHD 大脑与 LLM 在结构上惊人相似：两者都是强大的生成核心，但缺乏可靠的执行调度层。ADHD 大脑的智力与创造力不输常人，却在工作记忆、任务启动、时间感知等执行功能上频频掉线（来源：ADHD 的 AI 工具全景）；LLM 能写出惊艳的文案，却记不住对话历史、算不准数学题、无法自主规划多步行动（来源：无状态与外部记忆）。

解决方案的结构也同构：给生成核心套上外部脚手架。对 LLM，这是 harness 工程——通过 function calling 把精确计算外包给计算器、通过向量数据库外挂记忆、通过编排层管理上下文（来源：拐杖与脚手架）。对 ADHD，这是 AI 工具——Goblin Tools 把“整理房间”分解成“捡起衣服→擦桌子”的步骤，降低启动门槛（来源：Goblin Tools）；Saner.AI 用本地记忆减少搜索循环（来源：Saner.AI）；Lex 用一条指令触发多步骤流程（来源：Lex）。

## Endel 补上了哪一层？

回到 Endel。它不分解任务，不管理待办，不记录记忆。它只做一件事：根据你的心率、时间、活动状态，生成持续变化的背景音。这听起来很轻，但对 ADHD 来说，它补上了最隐蔽也最折磨人的一层执行功能——**状态维护**。

ADHD 大脑的“状态维护”有多难？你可能在写代码时突然想起要回消息，回完消息又刷了十分钟短视频，再回来已经忘了刚才写到哪一行。这不是注意力不够，而是大脑缺乏“维持当前任务上下文”的稳定机制，就像无状态的 LLM 每次对话都从零开始（来源：无状态与外部记忆）。Endel 的音景不是背景音乐，而是一个听觉锚点：它用持续的声音信号告诉你“你现在处于专注模式”，并抑制外部干扰。这相当于 LLM 的 system prompt 加上下文窗口——不是帮你做任务，而是帮你**保持在任务轨道上**。

## 有人用错了：当 Endel 被当作“万能专注药”

但很多人用 Endel 的方式，相当于把 function calling 当成整个 agent 系统。他们打开 Endel，戴上耳机，然后期望它自动解决所有执行功能问题——任务启动、时间管理、工作记忆。结果发现，该拖延还是拖延，该忘还是忘。

这就好比让 LLM 只靠一个计算器工具就完成复杂项目：计算器能算数，但无法规划步骤、无法检索知识、无法记住用户偏好。Endel 只覆盖“状态维护”这一层，而 ADHD 的执行功能缺陷是多层的：任务启动需要 Goblin Tools 的分解，工作记忆需要 Saner.AI 的持久存储，时间盲需要提醒工具（来源：ADHD 的 AI 工具全景）。指望 Endel 一肩挑，就像指望 function calling 解决所有 agent 问题——工具本身没错，但用法错了。

## 脚手架 vs 拐杖：边界在哪？

这里触及核心争议：AI 工具是脚手架还是拐杖？脚手架帮助使用者“建造”，但使用者仍需自己“攀登”；拐杖则永久替代功能，撤除后能力崩塌（来源：拐杖与脚手架）。

Endel 的争议在于：它提供的听觉锚点是否可撤除？如果用户每次专注都必须依赖 Endel，那它就是拐杖——因为内在的状态维护能力没有成长。真正的脚手架应该像 Goblin Tools 的分解功能：用户可逐渐学会自己分解任务，最终不再需要工具。但 Endel 的机制是直接改变外部环境，而非训练内在能力，撤除后效果消失。

这并非否定 Endel 的价值。对许多 ADHD 用户来说，状态维护是刚需，就像轮椅对行动不便者——不是所有工具都需要可撤除。但用户必须清醒：Endel 是“补偿”而非“训练”，是“接口”而非“调度器”。

## 诚实局限：证据与个性化

必须指出，Endel 针对 ADHD 的效果缺乏独立临床研究（来源：矛盾与存疑）。其神经锁相技术声称能调节脑波，但无大规模实证。此外，ADHD 异质性强：对注意力分散型有效的音景，可能对冲动型无效。同样，LLM 的 function calling 也面临工具选择与编排的个性化挑战（来源：矛盾与存疑）。

## 今天就能试的行动

1. **给 Endel 一个明确角色**：只在你需要“维持状态”的任务中使用它——比如写长文、编码、阅读论文。不要用它做需要频繁切换的任务（如回复邮件），那会适得其反。
2. **组合工具，而非依赖单一**：Endel 补状态维护，Goblin Tools 补任务启动，Saner.AI 补记忆。像 agent 系统一样，为每个执行功能层配一个工具。
3. **测试撤除**：每两周尝试一次不戴耳机工作，记录专注时长变化。如果大幅下降，说明 Endel 是拐杖；如果仅小幅波动，说明它只是辅助。
4. **类比到 LLM**：如果你在用 AI agent，检查你是否把某个工具（如搜索 API）当成了整个编排层。好的 harness 会组合多个工具，并保留人类在循环中的决策权。

Endel 不是万能药，function calling 也不是。但当我们理解它们各自补上的是哪一层执行功能，就能从“用错”变成“用对”——让工具成为脚手架，而非拐杖。

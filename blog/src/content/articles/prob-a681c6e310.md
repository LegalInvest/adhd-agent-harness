# ADHD 程序员视角：为什么「治好 ADHD 的分心走神或超聚焦跑偏，丢失目标」和「让 LLM 不跑飞」其实是同一道工程题？

> 长程任务里 agent 会目标漂移，需要重锚定 system prompt——同一套 harness 思路，两边都成立。

你坐在电脑前，想重构一段函数。三小时后，你却在比较两款等宽字体的 ligature 差异，issue 页面还开着。同一天，你部署了一个“agent”，让它修复登录 bug。它先是改写了 README，接着检查服务器价格，最后回了一封毫不相干的邮件。这两个场景——ADHD 的分心/超聚焦，与 LLM agent 的长程目标漂移——崩溃的方式惊人地相似：一个高产的生成核心，在缺少外部调度层时，把能量烧到了错误的地方。

## 诊断：高产但缺执行调度层

ADHD 与 LLM 的相似性不是修辞游戏。从结构上看，两者都是“生成核心”很强，但“执行调度层”不可靠：ADHD 大脑能在感兴趣的事情上爆发出惊人产出，却常因工作记忆不稳定、时间盲、启动困难而丢失目标；LLM 能生成连贯文本、代码和推理，却没有跨会话状态、没有内在规划能力，上下文一长还容易退化。wiki 资料把这一对应称为“外部执行功能层”——通过外部脚手架来补偿内部调度缺陷（来源：AI 与 ADHD 的专注力）。

ADHD 一侧已有不少工具在尝试搭建这一层：RescueTime 自动追踪时间、阻断分心网站，帮助用户看见“时间到底去了哪”（来源：Revolutionizing ADHD Management with AI Assistants）；Focusmate 用虚拟身体在场（Body Doubling）提供外部问责，提升任务启动（来源：11 Best ADHD Productivity Apps for Fluctuating Energy）；Brain.fm 用 AI 生成专注音乐，在一项 44 款 ADHD 应用测试中被列为最佳 9 款之一，但针对 ADHD 的独立临床证据仍有限（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）；Forest 则用游戏化番茄钟把专注记录外部化。这些工具并不“治疗”ADHD，而是给生成核心加 harness。

LLM/Agent 一侧同理。文献发现 ADHD 研究群与 agentic harness 研究群都把“任务分解与规划”作为核心机制概念（来源：LBD同构对：任务分解与规划）。AgentGen 通过环境与任务生成增强 LLM agent 的规划能力，AMAP Agentic Planning Technical Report 则把规划循环作为 harness 核心。换言之，LLM 也需要外部模块来拆解任务、监控执行、周期性重锚定目标。

## 古人的 harness：日课与诗赋都是 system prompt

曾国藩的 harness 系统可能是最好的历史对应。他年轻时浮躁坐不住，读书“他人目下二三行，余或疾读不能终一行”，情绪不稳，甚至打过败仗就跳水自杀。他的办法不是“变聪明”，而是给自己写了一套极笨的 system prompt：日课十二条——黎明即起、读书不二、谨言、写日记反省；作战上“结硬寨、打呆仗”，用最保守的规划抵消冲动。他每天写日记骂自己，这本质上是一个“运行日志 + 自我反思模块”。对应到 LLM agent：日课就是定时 re-grounding 的 system prompt，日记是执行日志，结硬寨是保守的规划循环与验证。

曹植则展示了另一面：文学超专注，铜雀台作赋一挥而就，却对政治任务毫无兴趣，冲动到醉闯司马门，最终因醉酒误了受命救曹仁。他的 harness 不是去消除超聚焦，而是把溢出的创造力导入固定通道——以诗赋为核心，将政治失意注入文学。这相当于给 LLM 的 creative overflow 加上主题约束和输出通道，避免它在错误的目标上跑飞。

## 工程解：harness 的三层结构

把两边的解法放在一起，可以看到同构的三层结构：

**1. 目标重锚定。** ADHD 侧，AI 工具“提供提醒和通知以保持正轨”，并能充当待办、教练和支持性朋友（来源：AI for ADHD: Best Tools, Apps, and Strategies - Themba Tutors）。但传统提醒有一个致命 bug：设置提醒本身就是执行功能任务。更理想的方案是无需主动设置的提醒，比如 RescueTime 的自动阻断、Forest 的环境化提示。LLM 侧，则是在长程对话中周期性重发目标、把目标写成“不变量”放进 system prompt，防止 agent 在上下文膨胀后偏离原任务。

**2. 任务分解与规划循环。** ADHD 侧，Motion、Reclaim.ai 等自适应工具自动创建、优先排序和动态调整日程，弥补静态计划容易失效的问题（来源：The AI Powered SuperApp for Work | Motion）。LLM 侧，AgentGen 和 AMAP 都通过外部环境与任务生成来增强规划，把大目标拆成可执行的子步骤，并在每一步验证是否仍在主线上。

**3. 外部记忆与问责。** ADHD 侧，RescueTime、Forest 把时间追踪和专注记录外部化，减轻工作记忆负担；Focusmate 用社会在场提供外部问责。LLM 侧，则对应外部记忆库、检索增强和 human-in-the-loop 检查点——在关键节点把“运行状态”暴露给人类，避免黑箱漂移。

## 边界：脚手架 vs. 拐杖，以及同构的局限

这个同构视角是有力的，但不能当科学事实来卖。wiki 资料自己也警告：ADHD 大脑与 LLM 的“同构”目前只是理论类比，缺乏实证神经机制对应；工具宣称常超出证据，Brain.fm 的 ADHD 独立临床证据有限，RescueTime 也没有大规模随机对照试验；AI 工具可能成为永久拐杖，而不是逐步内化的脚手架（来源：矛盾与存疑）。

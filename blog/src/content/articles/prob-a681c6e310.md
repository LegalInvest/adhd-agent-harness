# ADHD 程序员视角：为什么「治好 ADHD 的分心走神或超聚焦跑偏，丢失目标」和「让 LLM 不跑飞」其实是同一道工程题？

> 你肯定见过 agent 跑飞:任务是「修这个 bug」,它重构了三个文件、升级了两个依赖、顺手优化了一段无关代码——每一步局部合理,整体面目全非。现在打开你自己上周的 PR:任务是修一个 bug,diff 里为什么有六百行?

程序员的目标漂移有全行业最丰富的物种库。**「顺手重构」瀑布**:进去改一行,看到坏味道,重构函数→重构类→重构模块——原 bug 还在,周五到了;**yak shaving 无底洞**:要修 A 得先升级 B,升级 B 得先迁移 C,迁移 C 得先……三天后你在给一个构建工具提 issue,已经没人记得 A 是什么;**工具超聚焦**:调优 vimrc/折腾终端主题/给脚本加没人用的参数——工程师的化妆品;**调试隧道**:一个不影响交付的诡异小问题,「我就是想知道为什么」,四小时进去了。共性依旧:**每一步都有局部理由,没有一步对照过全局目标**——和 agent 的 goal drift 在结构上完全同构,连「自我感觉良好」这一点都同构。

行业给 agent 的解法(目标重注入+步数预算+偏航检查)照单自取:

## 锚一:分支即目标声明

**开工先开分支,分支名就是本次目标**(`fix-payment-timeout`),并给它配一条完成判据(issue 里写清:「超时错误不再出现,测试 X 通过」)。此后每一次「顺手想改」,对照分支名:**不在名字里的,开 issue 不动手**。diff 就是你的偏航仪表:提交前看一眼 `git diff --stat`,六百行 diff 修一个 bug,就是航向报告在响。**单一目的分支不只是规范洁癖——它是给 ADHD 大脑的外置目标寄存器。**

## 锚二:yak shaving 的深度限制

给依赖链设硬预算:**为了主任务而做的前置任务,深度不超过 2 层**(修 A 需要动 B,可以;动 B 又需要动 C,停——问一句「有没有绕过 C 的丑陋方案?」)。丑陋但直达的路径,90% 的情况优于优雅但无底的路径——**先让主任务落地,把 C 写进技术债清单**,让它走正常排期。agent 工程里这叫 step budget:没有步数上限的 agent,能力越强跑得越远。

## 锚三:调试隧道的止损单

「我就想知道为什么」是程序员最高贵也最贵的冲动。协议:**与交付无关的诡异问题,给 30 分钟预算**;到点没头绪,写一条带复现步骤的 issue,回主线。周五下午或工作块之间,用「合法好奇时段」消化这些 issue——**好奇心和自由职业者的超聚焦一样,不该被杀死,该被排期。** 真解不开的谜,写成博客问出去,常常比独自钻四小时更快得到答案。

## 锚四:站会与 PR 的外部循环

每日站会是现成的重锚定仪式,用满它:说昨天/今天时**对照的是 sprint 目标,不是流水账**——说的过程就是一次自我偏航检查。PR 保持小而频:**一个 PR 只做一件事,一天内见 review**——短反馈循环同时是漂移的天花板(漂得再远,一天内会被 reviewer 拽回)。远程/独立开发者没有这些节拍,自建:每天固定给搭档/社群发一句「今天目标:X」,收工发「结果:Y」——两行字,把 agent 工程里的 external supervisor 廉价地实现了。

值得说破的红利:程序员的超聚焦用对了地方是真实的生产力奇观——一天写完别人一周的量。**关键从来不是压制它,是给它选址**:把隧道挖在主线上(用锚一的分支目标圈住),挖出来的就全是交付。

## 边界

同构强度 A 级:goal drift、step budget、external supervision 是 agent 工程的现役议题,ADHD 目标维持缺陷与超聚焦有文献支撑,软件场景映射几乎逐条对应,无对照研究检验本文组合。提醒:若「调试隧道」已常态化为熬夜(凌晨三点「就差一点」),那是超聚焦叠加睡眠失控,伤害是复利的,值得当健康问题正视;长期漂移导致的绩效焦虑,请别独自硬扛。

## 今天就能试的 3 件事

1. **下一个任务,分支名写目标+issue 写完成判据**:然后让每个「顺手」对照分支名。
2. **立 yak shaving 两层规则**:超过两层,找丑陋直达路径,债进清单。
3. **给下一个诡异问题设 30 分钟止损单**:到点写 issue 回主线,周五再赴约。

agent 跑飞你会加步数预算和监督循环,自己跑飞却只会自责——这不公平,也不工程。分支寄存器、深度限制、止损单、站会锚:四个组件装完,你的超聚焦终于可以放心油门到底——因为方向盘,已经交给系统了。

# RescueTime 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 RescueTime：用久了会让 ADHD 更强还是更依赖？

## 问题：边做边忘，是大脑的 bug 还是系统的 feature？

如果你是一位 ADHD 者，你可能对这一幕无比熟悉：打开电脑想写一封邮件，却被通知栏弹窗带跑，半小时后发现自己正在刷购物网站，而邮件还是一片空白。你的工作记忆像一块漏水的桶，刚装进去的信息转眼就流走了。如果你是一位构建 LLM Agent 的工程师，你同样熟悉另一个困境：大模型（LLM）本身没有跨会话状态，每次对话都从零开始，必须靠向量数据库、缓存或外部记忆模块来“续命”。两个场景，同一个结构问题：**一个高产但缺乏执行调度层的生成核心，必须依赖外部脚手架才能完成连贯任务。**

这正是本文要回答的核心问题：RescueTime 这类时间追踪工具，对 ADHD 大脑而言，究竟是提供支撑的脚手架，还是让人丧失自主能力的拐杖？要回答它，我们需要先看清 ADHD 大脑与 LLM Agent 在结构上的同构性。

## 同构：ADHD 大脑与 LLM 共享的“无状态”困境

ADHD 的执行功能缺陷，核心之一是工作记忆容量小、不稳定（来源：无状态与外部记忆）。这与 LLM 的“无状态”特性在结构上同构：两者都难以在任务过程中维持连贯的上下文。LLM 靠外部记忆（向量库、对话历史）来补偿，ADHD 大脑则依赖外部工具（清单、计时器、身体加倍）来锚定目标。

这种同构并非空洞类比。从工程角度看，为 LLM 搭建 Agentic Harness（规划模块、记忆模块、工具调用模块）与为 ADHD 搭建执行功能支架是同一类问题：都需要设计一个外部调度层，来补偿内部调度层的缺失或低效。RescueTime 正是这类外部调度层的一个实例——它通过自动记录时间使用，提供客观的“外部记忆”，让 ADHD 者能回顾自己实际做了什么，而不是依赖不可靠的自我感知。

## 人物案例：孔子与曾国藩的“外部 Harness”

历史上，许多高成就者早已在无意识中实践了这种同构策略。孔子（春秋）和曾国藩（晚清）都是典型的高产出但低调度稳定性的 ADHD 大脑（来源：人物案例）。

孔子身高1.9米，精力旺盛，周游列国14年坐不住，冲动爱骂人，思维跳跃，《论语》全是场景化语录，无系统著作。他的 Harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。这与 LLM Agent 的“定时 re-grounding”机制同构：通过外部规则（礼）和定期自我检查（反省），将跳跃的思维拉回轨道。

曾国藩30岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”。他的 Harness 是“日课十二条”：黎明即起、读书不二、谨言、写日记反省等。这相当于一个结构化的“外部调度器”，用最笨最稳的方法（结硬寨打呆仗）抵消自己的冲动。他的一辈子写日记骂自己，每天反省——这本质上是一个持续运行的“上下文管理”系统，类似于 LLM Agent 中的记忆模块，不断记录、回顾、调整行为。

## RescueTime：是脚手架还是拐杖？

回到 RescueTime。它的核心功能是自动记录你在电脑上的时间分配，并提供可视化报告。对 ADHD 者，这提供了两个关键价值：

1. **外部记忆**：你不再需要靠回忆来评估自己做了什么，RescueTime 给出了客观数据。这补偿了工作记忆的缺陷。
2. **隐性问责**：知道自己的行为会被记录，本身就能提升专注度。这类似于“身体在场效应”（来源：身体在场效应），即他人在场（或虚拟记录）能显著提升执行力。

但这里存在一个边界问题：**当工具从“辅助自我监控”变成“替代自我监控”时，脚手架就变成了拐杖。** 如果一个人完全依赖 RescueTime 的自动记录来知道自己做了什么，而不再发展内在的自我觉察能力，那么工具反而削弱了执行功能。这与 LLM Agent 工程中的教训一致：如果 Agent 完全依赖外部记忆来维持对话，而不尝试在内部缓存中保留关键信息，那么一旦外部记忆失效（比如 API 超时），Agent 就会彻底失忆。

wiki 资料中的“矛盾与存疑”也指出：工具宣称的证据不足，依赖风险与益处之间存在矛盾（来源：矛盾与存疑）。RescueTime 本身缺乏针对 ADHD 的独立临床验证，其效果多基于用户报告。更关键的是，它没有内置的反思机制——它只记录，不引导你如何改进。

## 如何让 RescueTime 成为脚手架而非拐杖？

基于 ADHD 与 LLM 同构的视角，我们可以设计一个更健康的用法：

1. **主动回顾，而非被动查看**：每天花5分钟看 RescueTime 报告，并问自己：“哪些时间花得值？哪些是被带跑的？” 这相当于 LLM Agent 的“事后分析”步骤，让外部数据反哺内部认知。
2. **结合任务分解工具**：用 Goblin Tools 或 Lex 将复杂任务分解成小步骤（来源：Goblin Tools、Lex），然后用 RescueTime 追踪每个步骤的耗时。这相当于给 LLM Agent 配一个规划模块和记忆模块，协同工作。
3. **设置“反思锚点”**：每小时用 RescueTime 的提醒功能问自己：“我现在在做的，是我计划做的吗？” 这类似于孔子和曾国藩的“每日反省”，将外部工具变为内部习惯的触发器。

## 局限与争议

必须诚实指出，ADHD 与 LLM 的同构性目前仅是一种理论类比，缺乏实证基础（来源：矛盾与存疑）。RescueTime 的效果也缺乏严格临床验证。此外，过度依赖任何工具都可能带来认知卸载的风险——你越依赖外部记忆，内部工作记忆可能越弱。这就像 LLM Agent 如果总是从向量库读取上下文，就永远不会学会在内部缓存中维持长期依赖。

## 今天就能试的行动

1. **安装 RescueTime 并运行一周**，不调整任何设置，只观察数据。注意哪些时间段你最容易“走神”。
2. **每天花5分钟回顾**，用纸笔记下1个关键发现。例如：“上午10-11点注意力最差，因为总看邮件。”
3. **尝试一次“无工具日”**：关掉 RescueTime 半天，靠自我觉察完成一个简单任务。比较有工具和无工具的差异，感受拐杖与脚手架的区别。
4. **结合身体加倍**：用 Focusmate 找一个虚拟伙伴，同时打开 RescueTime 共享屏幕，让外部问责和外部记忆双重作用。

## 结论

RescueTime 对 ADHD 而言，可以是脚手架，也可以是拐杖。关键在于你是否将它作为“外部记忆”来发展内在觉察，还是作为“替代记忆”来放弃自我监控。这面镜子同样照向 LLM Agent 工程：外部记忆是 Agent 的救星，但也可能成为它永远学不会自主调度的借口。同构视角让我们看清：工具不是答案，如何使用工具才是。

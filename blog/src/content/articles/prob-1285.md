# Goblin Tools 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Goblin Tools：它到底补上了哪一层执行功能？

## 问题：为什么「拆解」本身成了新的拖延？

ADHD 人群最熟悉的痛苦不是「不想做」，而是「想做，但脑子一片空白」。任务启动困难的核心，在于大脑无法将模糊意图转化为可执行的动作序列——就像 LLM 面对一个开放性问题，如果不给 chain-of-thought 提示，它要么胡编，要么卡住。

工程师在做 Agentic Harness 时遇到同样的问题：一个 planner 负责分解，executor 负责执行，但若 planner 输出太抽象，executor 就会迷失；若 planner 输出太细碎，循环又变得臃肿。**两边缺的都是同一个东西：一个能把「目标」翻译成「步骤」的调度层。**

Goblin Tools 的魔法就在这里——它把「写一份报告」拆成「打开文档→列出大纲→写第一段→……」，每步小到足以触发启动。但很多人用错了：他们让 Goblin Tools 拆完就照单全收，结果发现步骤太多、顺序不合理，反而更焦虑。这就像给 LLM 一个 planner 却不用 executor 验证——拆解本身成了新的拖延。

## 同构：ADHD 大脑与 LLM 共享一个「生成核心」缺陷

ADHD 大脑是一个高产的生成核心：创意、联想、冲动源源不断，但缺乏稳定的执行调度层来排序和落地。LLM 同样：它能生成海量文本，但若没有外部约束（确定性工作流、工具契约），输出随机性高，小错误会累积（来源：AI Agent Systems: Architectures, Applications, and Evaluation）。

这种同构不是比喻，而是有实证支撑的：ADHD 组超专注工作时长平均 14.5 小时/周，对照组仅 8.2 小时（来源：Pace Ventures Research 2026）——生成核心的输出能力很强，但调度失效导致表现波动。LLM 一侧，采样温度控制着输出多样性，温度越高越随机；工程上通过「计划-执行」分离强制确定性控制流（来源：Planner-Executor Agentic Framework）——两者都在用外部结构稳定内部波动。

## 工具实测：Goblin Tools 补上了哪一层？

Goblin Tools 的核心价值在于「将模糊的工作分解为更小的行动」（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。但它只补了「分解」这一层——它不排优先级，不设截止日期，不动态调整。这就像 planner-executor 框架中 planner 只负责分解，却把调度交给 executor 自己。

相比之下，Motion 和 Reclaim.ai 补了「调度」层：Motion 自动排程并动态调整（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog），Reclaim.ai 保护深度工作时间不被会议侵占（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。Tiimo 则用视觉化时间线让时间流逝变得可感（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar），对应 LLM 的「状态可视化」。

但问题在于：**这些工具各补一层，很少有人把它们组合成完整的 harness。** 用 Goblin Tools 拆任务 → 用 Motion 排时间 → 用 Tiimo 看进度 → 用 Todoist 做外部记忆——这才是完整的「执行调度层」。

## 今天就能试的行动

1. **用 Goblin Tools 拆一件拖延一周的事**：输入「整理房间」，看它拆出的步骤。然后删掉你认为不合理的步骤，只保留 3-5 步。**关键：不要照单全收，要像 executor 验证 planner 输出一样调整。**
2. **给 LLM 加一个「计划-执行」提示**：在 ChatGPT 或 Claude 里，先让模型「列出完成某任务的步骤」，再让模型「按步骤执行，每步输出后问我是否继续」。对比直接提问的效果。
3. **组合工具：Goblin Tools + Todoist**：用 Goblin Tools 拆任务，把步骤手动复制到 Todoist 并设提醒。坚持一周，记录启动时间的变化。

Goblin Tools 之于 ADHD，就像 planner-executor 任务分解之于 LLM——它解决了「拆解」这一层，但真正的 harness 需要多层组合。不要把它当拐杖，而要把它当脚手架：用它拆解，用其他工具调度，用自我反省调整。孔子 70 岁才做到「从心所欲不逾矩」——我们不必等 70 年，但需要承认：**外部结构不是敌人，是释放生成核心潜能的唯一路径。**

# Routinery 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Routinery：它到底补上了哪一层执行功能？

## 问题：为什么 Routinery 对一些人来说是救星，对另一些人却是又一个吃灰的 App？

ADHD 大脑的核心困境，不是缺乏想法，而是无法把想法变成行动。你脑子里有一份完美的晨间流程——冥想、写日记、吃早餐——但醒来后，却刷了 40 分钟手机，连床都没下。这不是懒，而是执行功能（尤其是任务启动和工作记忆）的临时掉线。同样，大语言模型（LLM）能写出漂亮的计划，却无法自己执行一个多步骤任务——它需要外部工具来调用 API、查询数据库、控制循环。这就是为什么 **Routinery 之于 ADHD，就像 function calling 工具调用 之于 LLM**：两者都补上了同一层缺失的“执行调度层”。

但问题在于：很多人把 Routinery 当作“自动执行一切”的按钮，就像把 function calling 当作“LLM 自动完成一切”的魔法。结果，Routinery 变成了一个僵硬的日程表，吃灰；function calling 变成了一个不可控的脚本，跑飞。两者用错的方式一模一样——**忽略了脚手架需要与生成核心协同，而不是替代它**。

## 同构：ADHD 大脑和 LLM 是同一类“高产但缺调度”的生成核心

从认知架构看，ADHD 大脑和 LLM 是结构同构的。ADHD 大脑被描述为“高产生成核心与不稳定调度层”的组合——想法丰富、联想活跃，但工作记忆容易“掉落”，任务启动困难（来源：ADHD 大脑与 LLM 的同构）。实证研究也表明，ADHD 患者呈现“强记忆、弱控制”的神经心理模式：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

LLM 一侧同样如此。Transformer 模型在长上下文下，冲突解决能力会崩溃到随机水平——无法抑制优势反应，这与 ADHD 执行功能缺陷的核心标志一一对应（来源：Deficient Executive Control in Transformer Attention）。因此，构建 AI agent 时，工程师必须设计“agent scaffolding”——围绕 LLM 的软件架构和工具，使其能够执行复杂、目标驱动的任务（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。这个 scaffolding 就是 LLM 的“外部执行功能层”。

历史人物早已在实践中验证了这一同构。孔子和曾国藩都是典型的 ADHD 大脑。孔子精力旺盛、思维跳跃，周游列国 14 年坐不住，但对音乐能“三月不知肉味”——典型的超聚焦。他的 harness 是“克己复礼”：用外在的“礼”作为行为边界，每日反省。曾国藩 30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，他的 harness 是“日课十二条”和“结硬寨打呆仗”。两人的 harness 系统，本质上就是一套外部执行调度层——用固定的规则和反馈循环，把高产的生成核心约束到目标轨道上。这与 LLM 的 agent scaffolding 完全同构：**日课 ↔ 定时 re-grounding，秘书 ↔ 外部调度器，礼 ↔ 上下文约束**。

## 工具：Routinery 到底补上了哪一层？

Routinery 的核心功能是让用户预设一系列步骤，然后按顺序引导执行。这恰好对应了 ADHD 执行功能中最薄弱的环节：**任务启动和顺序执行**。它补上的就是“外部执行调度层”中的“任务序列调度”子层。

其他工具各有侧重：Goblin Tools 的 Magic ToDo 负责“任务分解”——把模糊任务拆成小步骤，降低启动门槛（来源：Goblin Tools）。Saner.AI 负责“外部记忆”——通过本地记忆和知识回忆，补偿工作记忆的不稳定（来源：Saner.AI）。Lex 则通过“单一指令触发多步骤”来降低决策负荷（来源：Lex）。这些工具共同构成了一个完整的 external harness，就像 LLM agent 需要记忆、工具、规划、验证循环等多个组件一样。

但关键区别在于：**Routinery 是“执行层”的骨架，而不是“计划层”的大脑**。如果你把 Routinery 当作一个“写好了就会自动执行”的魔法，那你就会犯和那些给 LLM 直接挂上所有工具就让它自主行动的工程师一样的错误——**没有设计好上下文、验证和 fallback 机制**。

## 判断：脚手架 vs 拐杖的边界

我的核心观点是：**Routinery 这类工具是“脚手架”，不是“拐杖”。脚手架帮你搭建结构，但你仍然需要自己走上去；拐杖则替你走路，长期使用会导致肌肉萎缩。**

证据在于：工具的价值不在于替代大脑，而在于把 ADHD 的“高产生成核心”接入一个可维护的“执行调度系统”（来源：ADHD 的 AI 工具全景）。但矛盾与存疑指出，过度依赖可能导致能力退化，成为永久拐杖（来源：矛盾与存疑）。目前缺乏长期纵向研究来验证这一点，但个体差异巨大——有人反馈柔性引导不够强力，有人则觉得强行打断（如番茄钟）反而有效（来源：矛盾与存疑）。

因此，正确的用法是：**把 Routinery 当作“启动引擎”，而不是“自动驾驶”**。用它来帮你迈出第一步，然后根据实际反馈动态调整。就像 LLM 的 function calling 需要设计好验证循环和错误处理，而不是让 LLM 自己决定一切。

## 今天就能试的行动

1. **为你的 Routinery 设置“最小可行版本”**：只包含 3 个步骤，比如“起床→喝水→打开日记”。不要贪多，先让启动变得极低门槛。
2. **加入一个“反省步骤”**：在 Routinery 序列末尾加一个 2 分钟的“检查点”，问自己“刚才的执行感觉如何？需要调整吗？”——这相当于 LLM agent 的验证循环。
3. **用 Saner.AI 或 Goblin Tools 配合 Routinery**：用 Saner.AI 记住你的 Routinery 上下文，用 Goblin Tools 把模糊任务（如“整理书房”）先分解成步骤，再导入 Routinery。
4. **每周回顾一次**：像曾国藩写日记一样，记录 Routinery 执行中的失败点，调整步骤顺序或粒度。不要让它变成死板的日程表。

记住：Routinery 是你的 harness，不是你的大脑。真正的生成核心——你的创造力、直觉和超聚焦——仍然是你自己。脚手架搭好了，路还得自己走。

# 为什么用 Motion 治 ADHD 的不知哪些方法有用，和给 agent 套 human-in-the-loop 监督 是一回事？

> Motion 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

如果你被 ADHD 折磨，最痛苦的可能不是“不努力”，而是“不知道今天到底哪个方法会生效”。昨晚睡得好的清晨，你能一口气做完三天的事；但同样的任务清单换到今天，就像大脑被一层雾包住。你试过番茄钟、子弹日记、Forest、Notion、各种 AI 助手，有时神效，有时完全失灵。于是你不断换工具，陷入“搜索—试用—放弃—再搜索”的循环。

做 Agentic Harness 的工程师对这种感觉应该不会陌生。你的 LLM/agent 能写代码、能生成商业计划、能记住整本手册，但也会在某个长上下文冲突点上突然“掉线”，输出一个明显错误却自信满满的答案。你加了 prompt、加了工具、加了 eval，但明天它可能换一种方式失控。你也在反复问：到底是模型不够强，还是我的“监督层”没搭对？

这两种困境，表面看一个是神经多样性的问题，一个是工程问题，但底层结构高度相似。本文的核心判断是：ADHD 大脑与 LLM/agent 都是“生成能力极强、执行调度极弱”的核心系统，两边真正需要的不是更多意志力或算力，而是一个外挂的 harness——由外部调度、外部记忆和人在回路（human-in-the-loop）再接地组成的执行脚手架。

## 1. 不是“懒”或“没能力”，是调度层缺失

ADHD 常被描述为“执行功能缺陷”。执行功能像大脑的驾驶座，调控注意力、计划、组织、工作记忆和冲动控制；对 ADHD 患者来说，这个驾驶座常常“无人驾驶”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。工作记忆是其中最痛的瓶颈——无法在脑中短暂保持信息以完成操作，导致丢三落四、难以启动任务（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout; 6 ways AI can help you manage ADHD symptoms - Understood.org）。

但最新的计算研究给了这个“驾驶座失灵”一个惊人的镜像。明尼苏达大学对 LLM 执行功能的系统测试发现，LLM 呈现“强记忆、弱控制”的剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷，无法灵活切换任务集、无法抑制自动化反应——这正是 ADHD 的经典神经心理模式（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。在 Stroop 冲突任务中，Transformer 的注意力随序列变长，在冲突试次上骤降到随机水平，无法抑制优势反应，无法解决认知冲突（来源：Deficient Executive Control in Transformer Attention）。耶鲁研究还指出，自注意力机制本身导致工作记忆容量限制：随 N-back 任务难度增加，注意力分数熵增、弥散，表现下降，与人类注意分散机制同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。

所以，ADHD 和 LLM 的核心问题不是“没有燃料”，而是“orchestration 层”——那个决定什么时候该做什么、该抑制什么、该调用什么记忆的调度系统——不够稳定。

## 2. 什么是 harness：给生成核心装一个“外挂调度层”

既然核心强大、调度层薄弱，解决方案就不是继续压榨核心，而是外接一个 harness。它至少包含三件事：

1. **外部调度**：把“下一步该做什么”的决策权交出去，减少内部决策疲劳。
2. **外部记忆**：把工作记忆里的内容外化，降低上下文丢失的风险。
3. **人在回路 / 身体在场**：定期把“人”重新接回系统，做再接地（re-grounding）、纠偏和价值校准。

ADHD 工具已经开始做这件事。Motion 的 AI 核心就是自动排程与动态调整：它持续评估任务优先级、截止日期和可用时间，实时重建日程，当任务面临延期风险时提前数天或数周发出警告（来源：The AI Powered SuperApp for Work | Motion）。它通过消除“下一步该做什么”的决策负担，直接帮助用户克服时间盲和任务启动困难（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。Goblin Tools 的 Magic ToDo 把模糊任务（如“清理厨房”）分解成具体子任务，用户可调节“辣度”控制粒度（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)），这相当于给“任务启动”这个硬骨头加一个楔子。Saner.AI 则通过本地记忆和知识回忆，减少多标签切换和搜索循环，降低认知负荷（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。

对工程师来说，这些功能分别对应：外部调度器（如 LangChain 的 agent scheduler、checkpoint 机制）、外部记忆/向量库（RAG、long-term memory），以及 human-in-the-loop 监督节点（human approval、feedback loop、eval pipeline）。

## 3. 一个状元实业家的 harness：张謇的“父教育母实业”系统

为了说明 harness 不是现代人才有的概念，可以看清末状元实业家张謇。他身上有典型的 ADHD 式特质：41 岁中状元后辞官不做，冲动敢干，一辈子办了 20 多家企业、300 多所学校，还创办南通博物苑、南通大学；天天在工地跑，坐不住。他的成就不是因为他“克服了”多动和冲动，而是因为他给自己搭了一个极其清晰的 harness：

- **价值锚点 / 系统提示**：他提出“父教育母实业”，把“实业救国”作为终身信仰。这就像一个 LLM 的 system prompt 或 constitution，任何具体决策都可以回到它上面重新接地。
- **每日再接地**：他每天写日记反省，严格管理企业。这相当于 ADHD 侧的“每日复盘”，也对应 LLM 侧的“定期 eval / re-grounding”。
- **外部调度器**：他把手下秘书、企业、学校作为外部执行与记忆系统，而不是把所有事情压在脑中。

张謇的 harness 让冲动的“生成核心”可以持续输出，而不被无限的新想法带偏。ADHD 侧的“日课”和“秘书”，与 LLM 侧的“定时 re-grounding”和“外部调度器”，是同一结构。

唐代草圣张旭则是另一个极端：他以酒和草书为核心 harness，喝酒进入超专注状态，把全部情绪注入草书；看公孙大娘舞剑悟笔法，从自然中找灵感。他给“超聚焦”设计了一个专属环境，而不是用意志力压制它。LLM  harness 中也对应一个技巧：把模型的高概率冲动引导到可控的“输出格式”或“工具调用”里，而不是让它自由生成。

## 4. 工程师侧：给 agent 套 human-in-the-loop  supervision，就是同一种 harness

对工程师而言，给 LLM/agent 套 human-in-the-loop 监督，解决的是同一个“调度层缺失”问题：

- **工具调用与外部记忆**：当 agent 的工作记忆随上下文变长而熵增、注意力弥散时，RAG 和工具调用把关键信息外化，而不是继续让模型“硬记”。
- **人在回路批准**：在不可逆动作前加入 human approval，相当于 ADHD 侧的“身体在场”——他人在场能提升执行和纠偏（来源：身体在场效应）。
- **Eval / 反馈循环**：定期用人类反馈校正模型输出，就像张謇的“每日反省”，防止系统在一个错误方向上持续漂移。

ADHD 侧的“超专注工作时长”研究也提示了这种同构的代价：Pace Ventures Research 2026 的数据显示，ADHD 组超专注工作时长可达 14.5 小时，对照组 8.2 小时。超长专注是优势，但如果没有外部 harness，它可能变成失控的沉迷。LLM 的“长上下文超专注”也一样——能连续生成，但可能在某个局部越陷越深。

## 5. 承认边界：同构是启发，不是确诊

必须诚实指出，这里的同构性仍是一种“理论类比”，而非已验证的神经科学事实（来源：矛盾与存疑）。Motion 被推荐为 ADHD 辅助工具，但缺乏独立临床验证（来源：矛盾与存疑）；Brain.fm 等工具在 ADHD 人群中的独立临床证据也有限。工具本身可能增加认知负担，甚至变成“拐杖”而不是“脚手架”——如果你离开它就无法启动，那 harness 已经退化成依赖（来源：拐杖与脚手架）。

更重要的是，AI 工具能否真正替代真实人际互动和“身体在场”，目前仍有争议。human-in-the-loop 的“人”不能只是名义上的审批按钮，否则 agent 只是把风险外包，而没有真正获得监督和校准。

## 6. 今天就能试的四条行动

1. **选一款调度型 harness，运行一周**：用 Motion 或类似工具把任务、截止日期、会议全扔进去，让外部调度器替你回答“现在该做什么”。重点是减少决策次数，而不是把日程做得更完美。
2. **给 agent 加一个不可逆动作前的人类 checkpoint**：任何会写数据库、发邮件、调用 API 的 agent，都必须在执行前弹出一个“human-in-the-loop”节点。这是工程师版的“身体在场”。
3. **用任务分解降低启动阻力**：用 Goblin Tools 的 Magic ToDo 把最难的任务拆到第一步只需 2 分钟；给 agent 也做类似的“单步工具调用”设计，避免一次让它做太多决策。
4. **每周做一次“无工具测试”**：关掉 harness 完成一个小任务，判断工具到底是脚手架（帮助你内部化能力）还是拐杖（离开后你完全瘫痪）。

## 结语

ADHD 患者和 agent 工程师的痛苦，本质上来自同一个结构：一个强大的生成核心，配了一个不够可靠的执行调度层。Motion 这样的工具不是“治好” ADHD，而是给 ADHD 大脑外接一个 orchestration 层；human-in-the-loop 监督也不是“让模型变聪明”，而是给 LLM 外接一个价值与执行校准层。真正管用的方法，不是继续向内找意志力，也不是继续向外堆算力，而是搭建一个“人在回路 + 身体在场”的 harness——让核心尽情生成，同时让外部系统守住边界。

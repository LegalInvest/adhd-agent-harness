# 如果把 ADHD 大脑当成生成核心，Routinery 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Routinery 对 ADHD 的真正价值。

如果你是一个 ADHD 患者，你一定经历过这样的场景：打开电脑准备写报告，十分钟后却在浏览某件衬衫的买家秀；或者，你正沉浸式地修改一张设计图，直到三个小时后才发现连午饭都没吃，而原本要回复的邮件早已沉入收件箱底。

如果你是一个在做 Agentic Harness 的工程师，你一定也经历过：明明给 agent 写好了 system prompt，五个工具调用之后，它开始自顾自地生成无关数据，或者把“总结用户反馈”变成了“生成一份市场分析报告”。

这两个场景看似分属心理学和工程学，但它们共享同一个底层结构：**一个高产但缺执行调度层的生成核心，在长程任务中必然发生目标漂移，需要外部重锚定机制来纠偏。**

## 同构：ADHD 大脑与 LLM 都是“缺调度层的生成核心”

ADHD 大脑的典型特征不是“不能专注”，而是“无法可靠地选择专注于什么”。执行功能缺陷——尤其是工作记忆不稳定、任务分解困难和规划脱轨——使得 ADHD 大脑像一台没有操作系统的超级计算机：算力惊人，但无法自主调度任务（来源：How specific are executive functioning deficits in attention ↔ AMAP Agentic Planning Technical Report）。

LLM 同样如此。单个 LLM 的生成能力已经足够惊艳，但“不足以可靠地完成多步骤任务、与业务工具交互或适应领域特定逻辑”（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。它需要外部 harness——任务分解、规划循环、验证机制——来防止上下文膨胀和推理退化。

这种同构不是比喻，而是功能层面的真实对应。Swanson 文献发现，ADHD 研究群和 agentic harness 研究群的核心中间机制概念惊人一致，都是“任务分解与规划”（来源：LBD同构对：任务分解与规划）。

## Routinery：作为重锚定系统的“日课十二条”

那么，Routinery 能不能成为 ADHD 大脑的重锚定系统？

要回答这个问题，先看一个真实的“harness”案例。曾国藩在 30 岁之前是一个典型注意力不集中型 ADHD 患者：浮躁坐不住、天天串门喝酒看杀人、日记里骂自己“无恒”“浮躁”、读书慢到“他人目下二三行，余或疾读不能终一行”。他的解决方案是“日课十二条”：黎明即起、读书不二、谨言、写日记反省……用最笨最稳的“结硬寨打呆仗”来抵消自己的冲动。

这套日课的本质是什么？是**定时重锚定**。每天黎明起床是一个固定的 re-grounding 信号，读书不二是一个 session-level 的 context window 保护机制，写日记反省是一个 off-policy 的自我评估。它与 LLM harness 中的“定时 re-grounding prompt”或“agent 日志审计”在结构上完全同构。

Routinery 做的正是同样的事：它把一天拆成可执行的例程块，每个块有固定的开始信号和结束信号，中间只允许一个任务。这相当于为 ADHD 大脑提供了一个外部调度器，把“接下来该做什么”的决策负担从工作记忆卸载到工具上。

## 证据与局限：脚手架 vs 拐杖

现有工具已经验证了这类外部执行功能层的有效性。Focusmate 通过虚拟身体在场提供外部结构和社会压力，弥补内在多巴胺调节不足（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。RescueTime 通过自动时间追踪揭示真实的时间去向，帮助用户识别注意力漂移（来源：Revolutionizing ADHD Management with AI Assistants）。Brain.fm 通过神经锁相技术影响大脑信息处理方式，在 2026 年一项 44 款 ADHD 应用测试中位列最佳 9 款之一（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。

但争议同样存在。最大的风险是**依赖 vs 内化**：AI 工具是否会导致 ADHD 用户永久依赖外部脚手架，而无法内化执行功能？（来源：拐杖与脚手架）曾国藩的日课十二条最终内化成了他的第二本能，但那是持续数十年的自我训练结果。Routinery 如果只是被动地“告诉我该做什么”，它就是一架拐杖；只有当用户开始主动理解例程背后的调度逻辑，并逐步减少对工具的依赖时，它才是脚手架。

另一个局限是临床证据不足。Brain.fm 和 Endel 虽有神经科学基础，但在 ADHD 人群中的独立随机对照试验极少（来源：Brain.fm）。同构性本身也只是一个理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。

## 核心判断：Routinery 的价值不在于替代执行功能，而在于提供一种可内化的调度策略

我的核心判断是：**Routinery 对 ADHD 的真正价值，不是“帮你执行”，而是“让你看见执行的结构”**。

ADHD 大脑不是缺乏执行能力，而是缺乏对执行过程的可视化。Routinery 通过固定的例程结构，把“启动→执行→结束→切换”这个隐式的调度过程显式化。当你每天在同一时间做同一件事，你实际上是在训练大脑形成一条新的神经通路——这和 agent harness 中的“prompt 模板复用”是同构的：不是每次重新思考，而是把经过验证的调度模式固化下来。

对于做 Agentic Harness 的工程师来说，这个洞察同样有价值：最好的 harness 不是最复杂的，而是最容易被 agent “内化”的。就像曾国藩的日课十二条，简单到不可能失败，才能成为习惯。

## 今天就能试的行动

1. **定义你的“日课三条”**：不用十二条，只选三个最关键的例程（比如：起床后第一件事、午休后第一件事、下班前最后一件事），用 Routinery 或任何习惯追踪工具固定下来。
2. **给每个例程加一个“re-grounding 信号”**：比如开始例程前深呼吸三次，或者念一句简短的意图声明。这相当于 agent 的 system prompt 重置。
3. **每周做一次“日志审计”**：像曾国藩写日记一样，花 5 分钟回顾：哪些例程执行得好？哪些发生了目标漂移？原因是什么？
4. **如果你在做 agent harness，试试“日课式 prompt”**：不是一次性写一个长 prompt，而是设计一组固定的、可循环的 prompt 块，每个块负责一个子任务，块之间有明确的切换信号。

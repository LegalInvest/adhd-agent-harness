# Focusmate 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Focusmate：它到底补上了哪一层执行功能？

## 问题：为什么 Focusmate 对有些人有用，对另一些人只是“陪着走神”？

你预约了 Focusmate，打开摄像头，对面是一个同样在工作的人。前十分钟你确实在写报告，但二十分钟后你开始刷手机，只是把摄像头对着天花板。结束时你们礼貌互道“good job”，但你心里清楚：这半小时和独自工作时没什么两样。

与此同时，AI 工程师们正在为另一个问题头疼：一个设计用来写代码的 agent，运行半小时后开始生成无关的函数，甚至自己发明了一个不存在的 API。他们管这叫“目标漂移”（goal drift），解法是定期向 LLM 重新注入 system prompt，重置上下文焦点。

这两个问题——ADHD 的分心走神或超聚焦跑偏，与 LLM agent 的上下文膨胀和推理退化——本质上是同一个问题：一个强大的生成核心，缺乏可靠的执行调度层来持续锚定目标。而 Focusmate，如果被正确使用，恰是那个“重锚定”机制。但多数人用错了它。

## 同构：ADHD 大脑与 LLM 共享同一个困境

ADHD 大脑的核心特征并非“注意力不足”，而是“执行功能缺陷”：工作记忆不稳定、任务启动困难、时间盲——计划本和便签用了一周就崩溃了，因为“大脑的驾驶座上常常感觉没有人”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。最新实证研究进一步揭示了 ADHD 的“强记忆、弱控制”模式：工作记忆容量甚至超过常人，但自上而下的控制和调节能力不足，无法灵活切换任务集、无法抑制自动化反应（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

LLM 同样如此。单独使用时，它是强大的生成核心，但缺乏内建的调度与执行控制。当上下文长度增长，模型在冲突试次上骤降到随机水平——无法抑制优势反应，无法解决认知冲突，这与 ADHD 执行功能缺陷的核心标志一一对应（来源：同上）。因此，构建 AI agent 时需要“agent scaffolding”——一个由提示、记忆、代码、工具和编排逻辑组成的模块化框架，将 LLM 转化为目标驱动的系统（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。

这个“scaffolding”里有一个关键组件：**重锚定**。在长程任务中，agent 会因上下文膨胀而偏离原始目标，工程师会定期向 system prompt 注入“你的任务是 X，当前进度是 Y，下一步是 Z”。这相当于在生成核心外部重置上下文焦点。

## Focusmate 的真正角色：外部重锚定，而非社交陪伴

Focusmate 的官方机制是“虚拟身体在场”（body doubling），通过实时同伴问责弥补内在多巴胺调节不足（来源：Focusmate 页面）。但它的效果分层明显：有人用它提升启动，有人用它维持专注，有人只是“陪着走神”。

问题出在用户对工具的理解。Focusmate 的核心价值不是“有人看着你”，而是它创造了一个**强制性的重锚定时刻**。当你预约一个 session，你被迫在开始前明确“我接下来 50 分钟要做什么”，并在结束时向伙伴汇报。这个“声明-执行-汇报”循环，恰好对应 LLM agent 的“system prompt → 执行 → 状态更新 → 重锚定”流程。

那些用错的人，跳过了“声明”和“汇报”环节，只保留了“执行”——他们打开摄像头，但没有明确目标，也没有在结束时复盘。这相当于一个 agent 接收了空白的 system prompt，然后随机生成内容。

历史人物早已用类似机制管理自己的执行功能。晚清名臣曾国藩，30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”。他的解法是“日课十二条”：黎明即起、读书不二、谨言、写日记反省——用最笨最稳的方法抵消自己的冲动（来源：人物案例资料）。这十二条就是他的外部调度层，而每日写日记反省，就是他的“重锚定”机制：每天重新向自己注入“我是谁、我要做什么”的 system prompt。

明朝才子徐渭同样如此。他诗书画戏军全能，但执行功能极差，8 次举人不中，情绪失调导致杀妻、9 次自杀。他的 harness 是以书画为核心，把所有痛苦和精力注入艺术创作——这是一种“超聚焦的定向释放”，类似于将 LLM 的生成能力限制在特定领域，防止其漫无目的地漂移（来源：人物案例资料）。

## 证据与局限：为什么不能只靠 Focusmate

现有工具的证据支持 Focusmate 的效果，但存在明显局限。Focusmate 被列为“11 Best ADHD Productivity Apps”之一，强调其虚拟身体在场功能（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。在 ADHD 科技行业研究中，身体加倍被归类为有效策略（来源：ADHD科技行业深度研究）。但所有这些证据都来自用户报告或小样本研究，缺乏大规模随机对照试验。

更重要的是，Focusmate 只解决了“重锚定”这一个环节。完整的执行功能脚手架还需要其他组件：
- **工作记忆补偿**：如 RescueTime 自动记录时间去向，减轻记忆负担（来源：RescueTime 页面）
- **启动辅助**：如 Brain.fm 通过神经锁相音乐降低进入专注状态的门槛（来源：Brain.fm 页面）
- **动机增强**：如 Forest 的游戏化番茄钟，利用多巴胺机制增强持续动力（来源：Forest 页面）

将 Focusmate 视为万能药，就像让一个 agent 只靠重锚定 system prompt 运行，却没有任何记忆系统或工具调用——它仍然会漂移，只是漂移得慢一点。

此外，同构性本身存在争议。ADHD 大脑与 LLM 的同构目前仅是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。依赖风险也需警惕：AI 工具可能成为永久依赖的拐杖，而非逐步内化的脚手架（来源：拐杖与脚手架概念）。

## 今天就能试的 3 件事

1. **给 Focusmate session 加一个“system prompt”**：预约前，用一句话写下你在这 50 分钟里唯一要完成的任务。贴在你的摄像头旁边。结束时，花 30 秒向伙伴复述你实际做了什么。
2. **组合使用 RescueTime + Focusmate**：让 RescueTime 自动追踪 session 期间的时间去向，第二天回顾报告。这相当于为你的 agent 添加了一个“日志记录模块”。
3. **模仿曾国藩的“日课反省”**：每天结束时，花 5 分钟问自己：“今天我的主要目标是什么？我是否偏离过？明天如何重锚定？”——这就是你自己的 system prompt 更新。

Focusmate 不是拐杖，而是一个重锚定开关。但开关需要你主动去按。

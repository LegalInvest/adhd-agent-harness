# RescueTime 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 RescueTime：用久了会让 ADHD 更强还是更依赖？

你盯着屏幕，打算写一份报告，但十分钟后你在刷社交媒体，半小时后你忘了原本要做什么。这是 ADHD 大脑的日常：工作记忆像漏水的桶，任务启动像冷启动的引擎。另一边，LLM 工程师调试 agent 时发现，模型一旦会话结束就忘了之前的所有指令，必须靠向量库、记忆文件来“续命”。两群人面对同一个问题：**一个高产但无状态的核心，如何靠外部记忆活下去？**

RescueTime 恰好站在这个交叉点上。它默默记录你的电脑使用行为，生成时间线，告诉你“你刚才在做什么”。对 ADHD 用户，它像一面外部镜子；对 LLM agent，它相当于一个日志系统。但问题是：这面镜子是让你学会自我观察，还是让你永远依赖它才能知道自己在做什么？

## 同构：ADHD 大脑与 LLM 都是“无状态生成核心”

ADHD 大脑的核心特征是“强记忆、弱控制”：工作记忆容量可能正常甚至超常，但注意控制和认知灵活性存在缺陷——无法灵活切换任务、无法抑制自动化反应（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这意味着大脑本身能生成大量想法，但缺乏可靠的执行调度层来组织这些想法。

LLM 同样如此：它本身是强大的生成核心，但缺乏内置的调度与执行控制。因此，构建 AI agent 时需要“agent harness”——包裹模型的基础设施，处理上下文管理、工具接口、记忆系统等（来源：What is an agent harness in the context of large-language models?）。

这种同构告诉我们：**ADHD 大脑和 LLM 都需要外部脚手架来补偿“无状态”的缺陷**。RescueTime 就是这样一个脚手架：它提供持久化的时间日志，相当于给大脑一个外部记忆体。

## RescueTime 作为外部记忆：补偿工作记忆的“掉落”

ADHD 用户频繁经历“边做边忘”：进入一个任务后，几分钟就迷失在细节中，忘了初衷。RescueTime 通过自动记录活动，在一天结束时生成时间线，让用户回顾“我实际上做了什么”。这直接补偿了工作记忆的不可靠性。

类似地，LLM agent 需要类似 RescueTime 的日志系统来追踪自身行为。Harness 工程中常用 Git 仓库存储项目上下文，通过 MCP 连接器访问外部数据（来源：Worker Agents | Harness Developer），本质上就是为无状态模型提供持久化记忆。

但这里有一个关键区别：**RescueTime 的记录是事后回顾，而非实时干预**。它不会在你走神时拉你回来，也不会在任务切换时提醒你。对于 ADHD 用户，这既是优点也是缺点——它不打断你，但也可能让你直到晚上才发现自己浪费了三个小时。

## 脚手架 vs 拐杖：边界在哪里？

“拐杖与脚手架”的辩证关系指出：AI 工具应作为脚手架促进能力发展，而非永久拐杖（来源：拐杖与脚手架）。RescueTime 的使用方式决定了它的角色。

**作为脚手架**：你每天花 5 分钟查看 RescueTime 报告，识别自己的时间黑洞（比如无目的刷网页），然后有意识地调整行为。几周后，你开始内化这种时间感知，不再需要频繁查看报告。

**作为拐杖**：你依赖 RescueTime 的实时仪表盘来告诉自己“现在该做什么”，一旦工具故障或不可用，你就完全失去时间意识。

证据显示，过度依赖 AI 工具可能削弱内在能力（来源：时间盲页面）。RescueTime 的被动记录特性，相比主动提醒的工具（如 Goblin Tools 的任务分解），更不容易形成依赖，但也更难直接改善任务启动。

## 争议与局限：RescueTime 不是万能药

首先，RescueTime 只能记录数字活动，无法覆盖线下行为（如阅读纸质书、与人交谈）。对于 ADHD 用户，线下时间同样容易迷失。

其次，个体差异巨大。有些 ADHD 用户发现 RescueTime 的自动分类不准确，需要手动调整，反而增加了认知负荷（来源：矛盾与存疑页面指出“个体差异导致效果不一”）。

最后，RescueTime 缺乏实时反馈。相比 Otter.ai 的实时转录或 Reflect 的结构化提问（来源：Otter.ai 和 Reflect 页面），RescueTime 的延迟反馈可能不足以阻止注意力漂移。

## 结论：脚手架还是拐杖，取决于你的使用意图

RescueTime 本质上是一个外部记忆系统，它既不能修复工作记忆，也不能替代执行功能。但它提供了一个客观视角，让你看到自己的行为模式。如果你用它来**学习自我观察**，它就是脚手架；如果你用它来**替代自我觉察**，它就是拐杖。

同构视角下的启示：LLM agent 工程师不会把日志系统当作 agent 本身，而是作为调试工具。同样，ADHD 用户应该把 RescueTime 当作训练工具，而非依赖的拐杖。

## 今天就能试的行动

1. **设置每日回顾提醒**：每天下班前花 5 分钟查看 RescueTime 报告，写下“今天最大的时间黑洞是什么？”和“明天如何避免？”
2. **关闭实时仪表盘**：避免频繁查看，只在固定时间（如午休、下班前）打开，训练自己的时间感知。
3. **结合任务分解工具**：用 Goblin Tools 或 Lex 将大任务拆成小步骤，再用 RescueTime 追踪每个步骤的实际耗时，形成反馈闭环。
4. **记录线下时间**：在 RescueTime 中手动添加线下活动（如“散步 30 分钟”），让时间线更完整。

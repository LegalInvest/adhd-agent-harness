# RescueTime 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 RescueTime：用久了会让 ADHD 更强还是更依赖？

## 问题：为什么你的大脑总是「断片」？

你打开电脑，准备写报告。10分钟后，你发现自己正在刷社交媒体，完全忘了刚才要做什么。这不是意志力问题——是你的工作记忆容量太小，像一块随时会擦写的白板。ADHD 大脑的生成核心（想法、创意）马力十足，但执行调度层（计划、启动、保持上下文）却时常宕机（来源：ADHD 大脑与 LLM 的同构）。

同样，你让 AI 助手写一份周报，它回答得头头是道。但当你追问“上周三的数据是多少？”，它一脸茫然——因为 LLM 本身是无状态的，每个会话都是全新开始。要让 AI 持续工作，必须给它套上外部的 agent harness：记忆系统、上下文管理、工具接口（来源：外部执行功能层）。

**ADHD 大脑与 LLM 是同一个问题的两面**：都是高产但缺执行调度层的生成核心，都需要外部脚手架来弥补内在缺陷。RescueTime 就是这样一个脚手架——但它是会帮你站得更稳，还是让你永远离不开它？

## RescueTime 的「同构」设计：无状态自动追踪

RescueTime 的核心机制很简单：后台运行，自动记录你花在每个应用/网站上的时间，生成报告。它不要求你主动输入任何东西——这正是 ADHD 友好的关键。ADHD 大脑的工作记忆脆弱，任何需要“记得去记录”的工具都会失败（来源：外部执行功能层）。RescueTime 像 LLM harness 中的日志系统一样，被动收集数据，不依赖用户的主动调度。

从同构视角看，RescueTime 扮演了 LLM agent harness 中“监控与日志”组件的角色。Harness 需要追踪 agent 的行为、消耗的 token、调用的工具，以便调试和优化（来源：ADHD 大脑与 LLM 的同构）。RescueTime 为 ADHD 用户提供了类似的功能：它让你看到自己实际的时间分配，纠正“时间盲”——即无法准确感知时间流逝的缺陷（来源：ADHD 的 AI 工具全景）。

## 证据：RescueTime 如何补偿工作记忆

ADHD 用户常陷入“搜索循环”：打开浏览器查资料，结果被一条新闻带走，半小时后忘了最初目的。RescueTime 通过自动记录，提供事后回顾的“外部记忆”。用户可以在一天结束时查看报告，发现自己花了多少时间在无关任务上。这相当于给 LLM 加了一个持久化的 context window——虽然不能实时干预，但能在下次会话前恢复上下文（来源：外部执行功能层）。

与 Goblin Tools 的主动分解（输入任务，AI 拆成小步骤）不同（来源：Goblin Tools），RescueTime 是被动监控。它不帮你启动任务，但帮你觉察偏差。这种设计类似 LLM harness 中的“验证循环”：agent 执行动作后，harness 检查结果是否符合预期，若不匹配则回滚或重试（来源：外部执行功能层）。RescueTime 的每日报告就是一次验证循环——让你对比“计划”与“实际”，从而调整行为。

## 脚手架还是拐杖？矛盾的边界

问题在于：RescueTime 的设计没有考虑“撤除”。它默认你永远需要它来监控自己。这引出了 ADHD 工具的核心矛盾：**脚手架应设计为可逐步撤除的临时支撑，但多数工具（包括 RescueTime）假设用户需要永久依赖**（来源：矛盾与存疑）。

ADHD 大脑与 LLM 的同构意味着，我们无法“治愈”无状态——LLM 永远不会内建记忆，ADHD 大脑的工作记忆容量也很难通过训练大幅提升。因此，完全撤除脚手架可能不现实。但工具可以设计为“渐进式自主”：例如，RescueTime 可以增加“目标挑战”功能，让用户逐渐减少对报告的依赖，转而培养内在的时间感知。

另一个争议是：RescueTime 只提供反馈，不提供实时干预。对于任务启动困难，它不如 Goblin Tools 的分解功能直接（来源：Goblin Tools）。对于工作记忆丢失，它不如 Saner.AI 的本地记忆检索（来源：Saner.AI）或 Mem 的持久化笔记（来源：ADHD 的 AI 工具全景）。RescueTime 更像一个诊断工具，而非治疗工具——它告诉你“你病了”，但不给你药。

## 同构的局限：LLM 可以重启，ADHD 不行

LLM 的 agent harness 可以随时重启、重置上下文，但 ADHD 大脑不能。一次时间盲导致的拖延，可能毁掉整个下午。因此，ADHD 工具需要比 LLM harness 更鲁棒的容错机制。RescueTime 的被动性意味着它无法阻止你掉进兔子洞，只能事后告诉你掉进去了。这就像 LLM harness 只记录错误但不阻止错误——显然不够。

从证据看，目前缺乏针对 RescueTime 的独立随机对照试验验证其对 ADHD 核心症状的长期改善（来源：矛盾与存疑）。现有支持主要来自用户反馈和概念对齐。因此，本文的立场是：**RescueTime 是一个有用的脚手架，但前提是用户主动利用其反馈来设计外部规则（如定时提醒、环境改造），而不是仅依赖报告来自我批评。** 它的价值在于“觉察”，而非“执行”。

## 今天就能试的行动

1. **安装 RescueTime 并设置每日报告**：坚持一周，记录你的“时间盲”模式。重点观察哪些活动让你丢失上下文（比如社交媒体）。这是建立外部记忆的第一步。

2. **结合 Goblin Tools 的 Magic ToDo 分解一个最让你拖延的任务**：比如“整理桌面”。对比 RescueTime 的反馈，看分解是否降低了启动焦虑。

3. **尝试 Saner.AI 或 Mem 存储一个项目的关键上下文**：将 RescueTime 识别的“频繁切换”任务（如写论文、编码）的笔记存入外部记忆，下次工作时直接调用，减少搜索循环。

4. **设计一个“撤除实验”**：使用 RescueTime 两周后，尝试停用三天，靠内在感知记录时间。对比前后差异，评估你对工具的依赖程度。如果感觉失控，说明工具是拐杖而非脚手架。

## 结语：脚手架还是拐杖，取决于你如何使用

RescueTime 之于 ADHD，如同 context window 之于 LLM：它提供了必要的外部记忆，但无法替代核心的调度能力。当你用它来觉察并设计更好的环境时，它是脚手架；当你用它来自我惩罚或替代内在动机时，它是拐杖。同构视角告诉我们，ADHD 大脑与 LLM 一样，需要精心设计的 harness——但这个 harness 必须包含“成长”的接口，而非永久的依赖。

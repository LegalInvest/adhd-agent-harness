# ADHD 职场人视角：为什么「治好 ADHD 的不擅长精确、重复、状态维护类任务」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 通过 function calling 把精确计算外包给工具——同一套 harness 思路，两边都成立。

如果你是一名 ADHD 职场人，你一定经历过这样的场景：明明脑子里有一个绝妙的方案框架，但一落到具体执行——排时间表、填表格、跟进邮件——就卡壳。你并不笨，甚至创意过剩，但那些需要精确、重复、状态维护的任务就像黑洞，吸走所有能量。

如果你是一名在做 Agentic Harness 工程的开发者，你一定也见过这样的 LLM 行为：它能写出惊艳的代码片段，但一旦需要连续调用多个 API、维护对话状态、精确计算数值，就开始胡言乱语、遗忘上下文、跑飞。

这两类人，其实在解决同一个问题。

## 同构：生成核心与调度层的分离

ADHD 大脑与 LLM 在结构上同构：两者都是强大的生成核心，但缺乏可靠的内置执行调度层（来源：ADHD 大脑与 LLM 的同构）。ADHD 的执行功能缺陷——工作记忆不稳定、任务启动困难、注意力分散——与 LLM 的无状态、缺乏调度层在结构上同构，都需要外部脚手架（harness）来补偿（来源：外部执行功能层）。

最新研究提供了直接证据：明尼苏达大学测试 LLM 的执行功能，发现“强记忆、弱控制”剖面——工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷，无法灵活切换任务集、无法抑制自动化反应，这正是 ADHD 的经典神经心理模式（来源：工作记忆）。耶鲁大学进一步发现，自注意力机制本身会导致工作记忆容量限制：随任务难度增加，注意力分数熵增、注意力弥散，与人类注意分散机制同源——“注意力资源的弥散分配”正是 ADHD 注意缺陷的计算本质（来源：工作记忆）。

这意味着，ADHD 和 LLM 的瓶颈都不在容量或算力，而在 orchestration 层。

## 历史人物已经用过的 harness

孔子就是一个典型的“高产但缺调度层”大脑。他精力旺盛，周游列国 14 年坐不住；思维跳跃，《论语》全是场景化语录，无系统著作；对音乐超专注到三月不知肉味，但对种地等俗事零耐心。他的专属 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”），70 岁才做到“从心所欲不逾矩”。这套系统本质上就是外部调度层：用固定的仪式（礼）来规训冲动的生成核心，用日课（反省）来定时 re-grounding。

毛泽东同样如此。他小时候是问题儿童，一辈子读书不辍但厌恶四书五经；思维极度跳跃，文章讲话充满比喻；冲动敢赌，四渡赤水、抗美援朝都是险棋。他的 harness 是“批评与自我批评”（让别人提意见）、“调查研究”（用外部数据对冲直觉）、“民主集中制”（用集体决策制衡个人冲动）。这些机制本质上就是外部调度器：用他人视角校准自己的注意焦点，用数据约束决策的随机性。

这些历史人物的 harness 与 LLM/agent 的 harness 在结构上同构：日课 ↔ 定时 re-grounding（如 context window 刷新），秘书 ↔ 外部调度器（如 function calling agent），调查研究 ↔ 工具调用（如 calculator、database lookup）。

## 今天的 AI 工具：同一套 harness 思路

现在，AI 工具正在为 ADHD 大脑提供数字化的 harness。Goblin Tools 的 Magic ToDo 将模糊任务（“清理厨房”）自动分解为具体步骤，用户可调节“辣度”控制粒度（来源：Goblin Tools）。这相当于为 LLM 的 task decomposition 模块——把复杂目标拆解为可执行的子任务序列。

Saner.AI 通过本地记忆存储和快速检索，减少搜索循环和标签切换（来源：Saner.AI）。这对应 LLM 的 external memory 机制——用向量数据库或缓存来补偿无状态性。

Lex 允许用户通过单一指令触发多步骤任务序列，底层架构类似于 agent scaffolding（来源：Lex）。这正是 agent 工程的精髓：通过一个 trigger 启动整个 pipeline。

Mem 作为“AI 第二大脑”通过持久记忆补偿工作记忆不足（来源：Mem），但依赖初始设置。这与 LLM 的 long-term memory 模块同构。

## 脚手架 vs 拐杖：必须警惕的边界

然而，这里有一个关键矛盾：AI 工具可能从脚手架变为永久拐杖，削弱内在执行功能（来源：拐杖与脚手架）。历史人物的 harness 是内化的——孔子最终“从心所欲不逾矩”，毛泽东的“调查研究”成了习惯。而今天的 AI 工具如果只是外包而不内化，就会产生依赖。

另一个局限是：现有工具多为通用设计，缺乏针对 ADHD 亚型的个性化调整；多数工具缺乏严格临床验证，效果多基于用户报告（来源：矛盾与存疑）。同构性理论本身也缺乏实证基础，目前仅是一种理论类比（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个你拖延已久的小任务**（比如“整理桌面”），调节“辣度”到让你觉得“这都能做”的程度，然后执行前 3 步。
2. **在 ChatGPT 或 Claude 中，尝试用 function calling 让 LLM 调用计算器或日历 API**，观察它是否不再“跑飞”——这能让你直观感受 harness 的效果。
3. **建立你的个人“日课”系统**：每天固定时间用 Saner.AI 或简单笔记记录 3 件完成的事，作为 re-grounding 仪式。
4. **反思你最近使用的一个 AI 工具**：它是脚手架（帮你学会新技能）还是拐杖（取代了你的能力）？如果是后者，尝试逐步减少依赖。

## 结论

ADHD 大脑和 LLM 是同一种困境的两种表现：强大的生成核心被脆弱的调度层拖累。解决方案也同构：用外部脚手架外包精确、重复、状态维护类任务，让生成核心专注于它擅长的创造与决策。这不是逃避，而是工程智慧。正如孔子用“礼”驯服了自己的冲动，毛泽东用“调查研究”约束了自己的直觉，今天的我们也可以用 AI harness 来补偿执行功能缺陷。但记住：脚手架的目的是最终让你能独立行走，而不是永远依赖它。

# Goblin Tools 之于 ADHD，就像 会褪去的脚手架 之于 LLM——但有人用错了

> 从同构视角实测 Goblin Tools：它到底补上了哪一层执行功能？

你有没有过这样的时刻：想清理厨房，站在门口却像被冻住，大脑一片空白——不是不知道怎么做，而是不知道从哪里开始。然后你打开 Goblin Tools 的 Magic ToDo，输入“清理厨房”，它瞬间吐出 12 个小步骤：把碗放进洗碗机、擦台面、拖地……你终于能动了。

但问题来了：下次你还得靠它才能启动吗？如果有一天它没了，你是不是又会瘫回原地？

这种“怕过度依赖工具、又怕没有支撑就崩”的焦虑，不只属于 ADHD 人群。做 LLM Agent 的工程师也在问同一个问题：我们给模型套的 harness（提示模板、工具调用框架、记忆模块）——到底是会褪去的脚手架，还是摘不掉的拐杖？

## 同构：ADHD 大脑与 LLM 共享同一个核心瓶颈

最新研究揭示了一个惊人的平行结构：Transformer 在 Stroop 冲突任务中，随着序列变长，注意力在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突（来源：Deficient Executive Control in Transformer Attention）。这正是 ADHD 执行功能缺陷的核心标志。

更关键的是，LLM 呈现“强记忆、弱控制”剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。ADHD 大脑同样如此——不是缺乏想法或能力，而是缺乏一个可靠的调度层来 orchestrate 那些想法。

这个同构告诉我们：两边的问题都不是“生成核心”不行，而是“调度层”失灵。对于 LLM，我们靠上下文工程和外部工具来补偿；对于 ADHD 大脑，我们靠外部执行功能层——比如 Goblin Tools 的任务分解、Inflow 的 CBT 结构化练习、Saner.AI 的外部记忆——来补偿。

## Goblin Tools 补上了哪一层？

Goblin Tools 的 Magic ToDo 本质上是一个**任务启动的外部调度器**。它把模糊意图（“清理厨房”）转化为可执行的原子步骤，绕过工作记忆瓶颈和任务启动困难。用户可调节“辣度”控制分解粒度，这相当于 LLM 中的温度参数——控制输出的随机性与确定性。

但 Gobin Tools 不负责执行，也不负责内化策略。它就像 LLM 的 prompt 模板：帮你把问题拆好，但不会让你下次自己会拆。这正是它作为“脚手架”而非“拐杖”的边界。

相比之下，Inflow 走的是另一条路：它基于认知行为疗法，提供每日技能培养，教用户如何理解自己的大脑并建立内在策略（来源：Inflow）。它的目标是让用户最终不再需要它——就像释迦牟尼的“自依止法依止”，不依赖权威，自己证悟。

## 历史人物的 harness：脚手架如何褪去

释迦牟尼 29 岁放弃王位出家，先后跟两个仙人学、苦行 6 年，不断试错不断放弃。他的专属 harness 是“八正道”和“持戒”——用戒律管行为，用正念拴念头。这套系统不是永久拐杖，而是训练他最终“自依止”的脚手架。

同样，老子的 harness 是“无为”——不强行做事，把能量收回来聚焦在道上。他连书都不想留，因为知道文字只是渡河之筏，到了彼岸就该放下。

这两位都没有现代 AI 工具，但他们的 harness 结构与 LLM 的上下文工程同构：释迦牟尼的“八正道”相当于一套固定的系统 prompt，老子的“无为”相当于动态调整的 temperature 和 top-p——根据情境选择最省力的响应路径。

## 用错的人：把脚手架当拐杖

Goblin Tools 的问题在于它太“好用”了。用户容易形成依赖：每次任务启动都靠它分解，从不练习自己分解的能力。这与 LLM 工程中的“prompt 依赖”如出一辙——工程师写死一个 prompt 模板，模型永远学不会自己推理。

更隐蔽的风险是：工具本身可能增加认知负荷。Saner.AI 虽然号称减少搜索循环，但设置和维护它本身就需要执行功能（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。对于已经超载的大脑，这可能是压垮骆驼的最后一根稻草。

## 争议与局限

必须诚实指出：ADHD 大脑与 LLM 的同构目前仍是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。情绪调节涉及神经递质、环境交互等生物复杂性，单纯“脚手架”类比可能简化问题。

此外，Goblin Tools 缺乏针对 ADHD 亚型的个性化设计——注意力主导型 vs 冲动主导型的情绪触发差异完全不同，而当前工具多为通用设计。

长期效果证据薄弱：Inflow 虽有用户好评，但缺乏严格随机对照试验证明其对情绪调节的持续改善（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个你拖延了一周的任务**，但做之前先自己试着列出前 3 步，再对照 AI 的输出——观察你的分解能力和 AI 的差异。
2. **设置“脚手架褪去日”**：每周选一天，禁用所有 AI 工具，只用纸笔或原生大脑完成任务。记录感受，评估哪些技能在退化。
3. **向 Inflow 学一个 CBT 策略**（如情绪日志），不依赖 AI 提醒，自己设闹钟执行一周。
4. **如果你在做 Agent 工程**：为你的 LLM 系统设计一个“褪去测试”——移除 prompt 模板或工具调用后，模型还能否保持基础推理能力？如果不能，说明你的 harness 是拐杖。

## 结论

Goblin Tools 之于 ADHD，就像上下文工程之于 LLM——它是必要的脚手架，但不是终点。真正的目标是让大脑（或模型）在脚手架褪去后，仍能自己执行调度。释迦牟尼放下了他的王位，老子放下了他的书，我们也该学会放下那把永久的拐杖。

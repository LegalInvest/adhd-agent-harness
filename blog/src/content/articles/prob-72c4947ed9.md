# ADHD 自由职业者视角：为什么「治好 ADHD 的不擅长精确、重复、状态维护类任务」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 通过 function calling 把精确计算外包给工具——同一套 harness 思路，两边都成立。

## 问题：两组人在同一种痛苦里打转

自由职业者最能体会 ADHD 的核心困境：创意和精力不缺，却总在精确、重复、状态维护类任务上崩溃——报税、发票归档、项目排期、跟进邮件。它们不要求聪明，只要求持续稳定的执行调度。同样，做 Agentic Harness 的工程师也常被 LLM 气到：模型能写代码、能推理，却会在多步任务里偏离目标、算错数字、丢失上下文。两边的症状听起来不同，痛苦却出奇一致：核心高产，执行层掉链子。

这不是巧合。 ADHD 与 LLM 的交汇点，最站得住脚的框架是结构同构：两者都是「高产生成核心 + 不可靠执行调度层」（来源：ADHD 的 AI 工具全景）。大脑前额叶执行功能受损，难以计划、抑制分心、维持目标；LLM 没有内置调度层，容易在对话中漂移或产生幻觉（来源：执行功能）。因此，解法也同构：不是让核心更努力，而是给它外接一套 harness，把精确计算、状态维护、重复校验外包给工具。

## 证据：两边都是「强记忆、弱控制」

ADHD 侧的证据很清晰。工作记忆是 ADHD 的关键瓶颈，信息在脑中短暂保持的能力不稳定，导致丢上下文、忘记任务细节（来源：工作记忆；执行功能）。这种不可预测性让计划和整理几乎不可能（来源：ADHD Productivity Hack: Plan 2025 Using AI）。

LLM 侧的最新研究则呈现了惊人相似的剖面：明尼苏达大学系统测试发现，LLM 是「强记忆、弱控制」——工作记忆容量甚至超过常人，但认知灵活性、注意控制和冲突抑制存在核心缺陷，无法灵活切换任务集、抑制自动化反应（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。在 Stroop 冲突任务中，Transformer 注意力随序列变长会骤降到随机水平，无法解决认知冲突（来源：Deficient Executive Control in Transformer Attention）。耶鲁大学进一步发现，自注意力机制随负荷增加会导致注意力弥散，与人类注意分散机制同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。

这些研究共同指向一个判断：瓶颈不在生成能力或算力，而在 orchestration 层。 ADHD 和 LLM 都不缺「想」，缺的是「稳住、按顺序做完、不跑偏」。

## 同构解法：把精确执行外包给工具

对 ADHD 来说，这叫「外部执行功能层」或认知卸载：把工作记忆、任务分解、提醒系统外化，用结构换可控。对 LLM 来说，这叫 agent harness：通过 function calling 把精确计算、数据库查询、状态管理交给外部工具，模型只负责意图理解与规划。

工具层面，这种对应非常明显。Goblin Tools 的 Magic ToDo 把「清理厨房」这类模糊任务分解为可执行子步骤，用户还能调节「辣度」控制粒度（来源：Goblin Tools；12 AI Personal Assistants Built for ADHD Brains）。这正像是 agent 里的任务拆解器。Saner.AI 通过 AI 笔记捕获、自动组织和通用收件箱，把不稳定的内部记忆外化为可检索结构，减少标签切换和搜索循环（来源：Saner.AI）。对应 LLM 的外部记忆系统 / vector store。Lex 的「单一指令触发多步骤任务」则直接呼应 agent 的规划机制：把复杂目标压缩为低启动成本的入口（来源：Lex；The Anatomy of an AI Agent）。

在工程实现上，function calling 的设计哲学就是：LLM 不该自己算精确数字、不该自己维护状态，而是调用计算器、日历、数据库、代码执行器。这和 ADHD 用户把报税交给记账软件、把提醒交给日历、把项目依赖交给看板，是同一道工程题：识别哪些操作需要稳定、精确、可重复，然后把它们从生成核心中卸载出去。

## 历史人物的 harness：他们也是同一套架构的早期实现

两千多年前的孔子，就是一个典型的 harness 自我管理系统。他身高一米九、精力旺盛、周游列国十四年坐不住，冲动时会骂人，对音乐能超专注到「三月不知肉味」，对种地等俗事零耐心，思维跳跃，《论语》全是场景化语录而无系统著作。他的专属 harness 是「克己复礼」——用外在的「礼」作为行为边界，每日「吾日三省吾身」，70 岁才做到「从心所欲不逾矩」。

这几乎可以直接映射到 LLM 的 harness：「礼」是外部 guardrails / system prompt 里的约束 schema；「日省」是定时 re-grounding，让模型回到原始目标；70 年的修炼说明，核心再强，没有外部结构也稳不住。

再看毛泽东。他是问题儿童，一辈子坐不住，从井冈山到延安到北京永远在动，思维极度跳跃，讲话充满比喻，敢下险棋。他的 harness 是：批评与自我批评、民主生活会、没有调查就没有发言权、民主集中制、党指挥枪。用集体决策制衡冲动，用组织系统管理军队和国家，用调查获取外部事实。

对应到 LLM：批评与自我批评是 self-reflection / 人类在环（human-in-the-loop）；民主集中制是多数外部工具投票或聚合结果；调查就是 RAG 与实时数据检索，不让模型用内部记忆硬编。伟人的 harness 不是压制创造力，而是给创造力装上刹车和导航系统。

## 脚手架 vs. 拐杖：边界在哪里？

必须诚实指出，这个同构框架有局限。资料本身提醒我们：「同构性被过度推广」，目前仍是一种理论类比，而非经过验证的科学事实（来源：矛盾与存疑）。工具宣称也常证据不足，比如一些时间管理应用缺乏独立临床验证，却依旧被当作有效工具推荐；ChatGPT、Claude 等页面强调益处，却未充分讨论依赖风险（来源：矛盾与存疑）。

我的判断是：harness 与拐杖的分界，在于使用者是否主动参与 orchestration。如果只是让工具替你做所有决定，那是拐杖；如果你把工具当作外部记忆、外部校验和任务分解器，并保留最终选择与元认知，那就是脚手架。ADHD 用户用 AI 做下一步提示，但自己判断优先级；工程师用 function calling 把计算外包，但自己设计工具接口和失败回退。核心永远是「人在回路」。

## 今天就能试的 4 个行动

1. 画一张「卸载清单」：把你最怕的精确/重复/状态维护任务列出来，对应能外包的工具。报税 → 记账软件；项目排期 → Motion/Todoist；灵感碎片 → Saner.AI 或 Obsidian。

2. 给 LLM/agent 做一次「function calling 审计」：检查你的 agent 是否在用模型自身做数字计算、状态判断、长序列记忆。若是，改成调用确定性工具或外部数据库。

3. 设置一个「日省」触发器：在日历或任务工具里固定 5 分钟 re-grounding，回顾当日目标与已完成事项，对应孔子的「三省吾身」与 LLM 的 periodic context refresh。

4. 引入「人类在环」检查点：在关键决策前加一道人工确认，无论是毛泽东式的民主生活会，还是 agent 的 approval gate，让高产生成核心不孤单地跑飞。

## 结语

ADHD 自由职业者和 Agentic Harness 工程师看似在两个世界，但面对的是同一个工程问题：一个聪明、高产、容易漂移的生成核心，如何被外部结构和工具调用约束住。治好 ADHD 的精确执行困境，与让 LLM 不跑飞，共用同一套答案——不是让核心无所不能，而是承认它有所不能，并把不能的部分，精准地外包给工具。

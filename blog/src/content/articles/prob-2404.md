# ADHD 程序员视角：为什么「治好 ADHD 的执行功能（计划/组织/启动）先天偏弱」和「让 LLM 不跑飞」其实是同一道工程题？

> ADHD 程序员是这场同构里最占便宜的人:你们不但每天亲手给 agent 搭 harness,而且整个软件工程的工具链——issue、PR、CI、code review——本来就是一套现成的执行功能外骨骼。缺的只是把它们用在自己身上的自觉。

程序员群体里 ADHD 特质的浓度,是圈内公开的秘密。这不奇怪:编程对超聚焦者是天堂(即时反馈、无限新奇、纯逻辑世界),debug 的多巴胺回路堪比游戏。但同一批人也共享一份事故清单:接了 ticket 三天没启动(启动困难);修 A bug 时顺手重构了三个文件,原始任务反而没完成(任务漂移);「这个周五能好」说了三个周五(时间盲);文档和测试永远欠账(低刺激任务回避);以及最经典的——凌晨四点还在优化一个没人要求的性能点(超聚焦失控)。

有意思的是,当这批人转头去构建 LLM agent 时,他们面对的是同一份清单:agent 不启动?给它明确的第一步。agent 跑偏?加目标重锚定。agent 不知道自己进度?外挂状态存储。agent 停不下来?设步数和预算上限。**每一条 agent 工程实践,都是一条 ADHD 自我管理方案的镜像——而写下这些实践的手,常常就属于最需要它们的大脑。**

## 把你给 agent 的待遇给自己

**给自己发 ticket**。你不会给 agent 一句「把系统搞好」,你会写清楚验收标准。但你给自己的任务经常就是「搞一下那个模块」。协议:任何超过一小时的活,先写成自己的 issue——目标一句话、验收标准、第一步动作。写 issue 的五分钟,就是你的 planner 调用。

**WIP 限制 = 上下文窗口管理**。同时开五个分支的程序员,和上下文塞爆的模型,失败模式相同。给自己设 WIP limit:在途任务不超过 2。新的好奇(「顺手重构一下」)一律开 issue 入队,不入当前工作区——**你舍得对 agent 的上下文做减法,也要舍得对自己的做。**

**commit 即 checkpoint**。小步提交不只是 git 美德,它是 ADHD 的状态外置机制:每个 commit message 都是「我做到哪了」的存档。配合每天下班前的两分钟「明日第一步」便签(写给明早那个冷启动的自己),重启成本从半小时考古降到三十秒。

**CI 是你的 guardrail,review 是你的 human-in-the-loop**。ADHD 的冲动提交(「应该没问题,直接合」)恰好被这套流程兜住——前提是你别绕过它。给自己立规:绕过流程的冲动出现时,就是最该走流程的时刻。这条对 agent 成立,对你也成立。

**用 deadline 的人质机制治时间盲**。「周五能好」的自我估计不可信(规划谬误 × ADHD 加成),但你有一个作弊器:估完之后乘 2,再把中间产物的时间点主动汇报给同事(「周三我先发个草稿 PR」)。对人的承诺是最强的人造节点——你在借 standup 和同事,给自己造 monitor。

**超聚焦的熔断器**。凌晨四点的优化狂欢,爽但有毒(第二天的你会付利息)。学 agent 的预算机制:开工前给探索性任务设硬上限(「这个性能问题,最多投入两小时,到点必须提 PR 或写 issue 挂起」),闹钟放到够不着的地方。**熔断不是打断心流,是把心流的产出保住——烧穿预算的心流,产出常常在第二天被自己 revert。**

## 一个职业红利

想深一层:ADHD 程序员调教 agent 时的那种直觉——「它不是坏,它是缺结构」——正是多年和自己大脑相处磨出来的。你们天然理解无结构系统的失败模式,天然知道 harness 要搭在哪。这不是安慰,是真实的职业优势:**agent 工程这个岗位,本质上是在给硅基 ADHD 当职能治疗师**——而你们是带着一手病历入行的。

## 边界

同构强度 A 级:双侧证据都硬(LLM 执行控制缺陷的直接实证 + ADHD 执行功能文献),且软件工程工具链作为执行支架的映射几乎是逐条对应。限制照旧:机制不等同,大脑不是 Transformer;另外长期凌晨四点模式伴随的睡眠剥夺,本身会放大所有 ADHD 症状——熔断器保护的不只是明天的效率,是你的神经系统。

## 今天就能试的 3 件事

1. **把手头最拖的活写成给自己的 ticket**:目标/验收/第一步,五分钟。然后只做第一步。
2. **数一下你的 WIP**:超过 2 的,全部开 issue 入队。感受工作区清空的那一下。
3. **给下一次超聚焦装熔断**:探索性任务,先定预算,闹钟放远。到点无论多爽,提交或挂起。

你早就是 harness 专家了——每天上班都在给一个聪明但跑飞的系统搭结构。下班前,记得给另一个同款系统也搭一份:那个系统写的代码,署的是你的名字。

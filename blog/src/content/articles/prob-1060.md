# Habitica 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Habitica：它到底补上了哪一层执行功能？

你有没有过这样的经历：打开电脑准备写报告，却发现自己正刷着视频，心里想“我就休息五分钟”；或者，你明明在做一个项目，却不知不觉投入到一个无关的细节中，几个小时过去才惊觉跑偏了。对 ADHD 来说，这不是意志力问题，而是大脑的“执行调度层”失灵了。

同样的问题，在 AI 领域叫“目标漂移”：一个 LLM agent 在长程任务中，随着上下文膨胀，会逐渐偏离初始指令，输出变得散漫、重复甚至无关。工程师的解法是“重锚定 system prompt”——定期把核心目标重新注入上下文。

Habitica，这个把待办事项游戏化成 RPG 的工具，对 ADHD 的价值恰恰在于它扮演了同样的角色：它不是一个简单的奖励系统，而是一个外部调度层，定期把“你真正要做什么”锚定回你的注意力。但很多人用错了——把它当成了多巴胺贩卖机，而不是重锚定脚手架。

## 同构：ADHD 大脑 = 无调度层的 LLM

ADHD 大脑与 LLM 的深层同构体现在：两者都是“高产但缺乏可靠执行调度层的生成核心”（来源：AI 与 ADHD 的专注力）。ADHD 的创意发散、LLM 的文本生成，都像一台没有操作系统的超级计算机——硬件强大，但无法稳定地执行指令。

具体来说，ADHD 的“工作记忆缺陷”与 LLM 的“无状态性”同构：两者都记不住刚才发生了什么，需要外部记忆系统来维持连续性。ADHD 的“时间盲”——“无法感知时间的流逝”，坐下后“真的感觉不到45分钟已经过去”（来源：时间盲）——对应 LLM 的“上下文膨胀”：随着 token 增多，模型对早期指令的注意力衰减。

因此，有效的干预方案在两边是同构的：给生成核心套上 harness（外部执行功能层）。对 LLM 是 system prompt 重锚定、向量数据库、上下文压缩；对 ADHD 是定时提醒、外部记忆、身体在场。

## Habitica 的正确用法：重锚定，而非打怪

Habitica 把任务变成 RPG 的“每日任务”和“待办事项”，完成得经验、打 Boss，失败掉血。很多人以为它的魔力在于游戏化带来的多巴胺——确实，ADHD 与多巴胺调节异常有关，游戏化能短暂刺激动机。但问题在于：多巴胺会快速脱敏，新鲜感一过，Habitica 本身就成了另一个被拖延的任务。

真正让 Habitica 对 ADHD 有效的，是它强制性的“重锚定”机制：每天你必须回到 app，查看你的“每日任务”列表，这就是一次 system prompt 刷新。它强迫你问自己：“我今天真正要做的三件事是什么？”——就像工程师在 agent 的上下文中插入一条“REMINDER: Your primary goal is X”。

曾国藩的“日课十二条”就是这种重锚定的经典案例。他30岁前“无恒”“浮躁”，日记里天天骂自己。他的 harness 是：黎明即起、读书不二、谨言、写日记反省。每天用十二条锚定自己。这本质上是一种“定时 re-grounding”——跟 LLM 的定期重锚定完全同构。王阳明的“致良知”更是极致：用良知作为唯一决策标准，事上练，军旅中也坚持讲学，这是“在每一个 action step 后检查是否对齐核心目标”的工程实践。

## 脚手架 vs 拐杖：边界在哪里？

但 Habitica 很容易变成拐杖。当一个人依赖 app 的叮咚声才能开始工作，一旦 app 失效就彻底瘫痪，那它就是拐杖。脚手架与拐杖的边界在于：工具是在“补偿”还是“替代”执行功能？（来源：AI 与 ADHD 的专注力）

Focusmate 是典型的脚手架：它提供虚拟身体在场，通过实时问责提升启动和维持能力（来源：身体在场效应）。用户报告显示，定期问责检查可将目标达成可能性从25%提升至95%（来源：身体在场效应）。但长期使用后，用户可能无法在没有伙伴的情况下独立工作——这就是依赖风险。

RescueTime 是另一个例子：它自动记录时间，帮助对抗时间盲（来源：RescueTime）。但它只是“揭示”时间去向，并不替你决策。学习使用 RescueTime 本身可能增加认知负荷（来源：矛盾与存疑）。

Brain.fm 的神经锁相技术试图直接调节大脑状态（来源：Brain.fm），但缺乏针对 ADHD 的独立临床研究，且效果因人而异。

## 核心判断：Habitica 的价值在于“重锚定频率”，而非“游戏化强度”

我的观点是：Habitica 对 ADHD 有效的唯一可靠机制，是它提高了你“重新校准目标”的频率。游戏化只是诱饵，真正起作用的是每天多次的“目标检查点”。如果你只是沉迷于打 Boss 升级，而忽略了任务本身，你就用反了。

证据上，这符合“上下文工程”原则：ADHD 易被环境带偏，LLM 易被上下文膨胀干扰，都需要主动设计“当下注意什么”的工程方案（来源：AI 与 ADHD 的专注力）。曾国藩的日课、王阳明的致良知，本质上都是高频率的目标重锚定。

## 局限与争议

必须诚实指出：ADHD 大脑与 LLM 的同构是隐喻而非严格等价，LLM 的“注意力机制”与人类注意力存在本质差异（来源：矛盾与存疑）。Habitica 的效果缺乏大规模随机对照试验，个体差异极大——对注意力不集中型有效，对冲动型可能无效。依赖风险也缺乏纵向研究。

## 今天就能试的行动

1. **把 Habitica 的“每日任务”精简到3项**：只放当天最重要的目标，其他都放“待办”。每天打开 app 时，先读这三项，再开始工作。这就是你的 system prompt 重锚定。
2. **设置每45分钟一次的“重锚定闹钟”**：用手机或 RescueTime 的阻断功能，时间一到就问自己：“我当前在做的事，和我今天的目标一致吗？”（来源：RescueTime）
3. **找一个 Focusmate 伙伴**：每天一次25分钟 session，只做你最重要的一件任务。身体在场效应能提升启动成功率（来源：Focusmate）。
4. **像曾国藩一样写“日课反省”**：用 AI 助手（如 ChatGPT）每天问：“今天我在哪件事上跑偏了？明天如何重锚定？”（来源：AI 与 ADHD 的专注力）

Habitica 不是魔法。它只是一个重锚定工具。真正的变化来自你用它来做什么——是刷经验，还是刷掉自己的目标漂移。

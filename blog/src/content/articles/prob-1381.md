# Inflow 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Inflow：它到底补上了哪一层执行功能？

## 问题：为什么你用了 Inflow，任务还是启动不了？

ADHD 圈子里的一个常见抱怨是：下载了 Inflow，跟着做了几节认知行为课，但第二天面对工作列表，依然像被钉在椅子上。这不是你的错——问题出在工具设计的前提假设上。Inflow 本质上是一个教育 + 习惯追踪应用，它教你理解 ADHD 大脑，但没给你一个实时的「外部调度层」。而 ADHD 大脑最缺的，恰恰就是这个调度层。

## 同构：ADHD 大脑与 LLM 是同一类「生成核心」

把 ADHD 大脑和大型语言模型（LLM）放在一起看，会发现惊人的结构相似性。两者都是强大的生成核心——ADHD 大脑能产生海量创意、超聚焦于兴趣，LLM 能生成流畅文本、解决复杂问题——但都缺乏可靠的内置调度层。LLM 需要 agent 框架（如 planner-executor 循环）来分解长任务、管理上下文、防止推理退化；ADHD 大脑则需要外部执行功能层来补偿时间盲、工作记忆不稳定和任务启动困难（来源：AI 与 ADHD 的时间管理）。

这种同构意味着：为 LLM 搭建脚手架（提示工程、agent harness）与为 ADHD 搭建执行功能支架，是同一类工程。孔子的一生提供了经典案例。他精力旺盛、冲动爱骂人（“朽木不可雕”）、对音乐超专注到“三月不知肉味”，但思维跳跃、无系统著作——典型的高产 ADHD 特质。他的 harness 是“克己复礼”：用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”）。这本质上是一个定时 re-grounding 机制，类似 LLM agent 的“上下文重置”或“规划日志”。曹操则是另一个极端：他把行政全丢给荀彧，用别人的执行功能弥补自己的不足——这就是一个“外部调度器”，对应 LLM 架构中把任务分解交给专门 planner 的做法。

## 错在哪：Inflow 补的是知识，不是调度

回到 Inflow。它提供了 ADHD 知识、认知行为策略和习惯追踪，但这些都属于“教育层”而非“调度层”。ADHD 用户真正需要的是像 Motion 或 Reclaim.ai 那样的自动排程工具：它们持续评估任务优先级、截止日期和可用时间，实时重建日程，消除“下一步该做什么”的决策负担（来源：Motion）。或者像 Tiimo 那样，用视觉化时间线让时间流逝变得可感，直接应对时间盲（来源：Tiimo）。这些工具补的是执行功能中的“规划”与“执行”环节。

但问题在于，很多用户把 Inflow 当作唯一的解决方案，以为“理解了我的大脑”就能自动改变行为。这就像给 LLM 喂了一堆提示工程手册，却不给它一个 agent 框架来实际运行任务分解循环。结果就是：知识有了，行动还是零。

## 真正的脚手架：planner-executor 循环

一个好用的 ADHD 时间管理工具，应该像 LLM agent 的 planner-executor 循环一样，包含三个环节：

1. **规划（Planner）**：把大任务分解成可执行的小步骤。Goblin Tools 在这方面做得不错，它能自动把“写报告”拆成“打开文档、写引言、收集数据……”（来源：Goblin Tools 页面）。
2. **执行（Executor）**：按计划执行，并在被打断后自动调整。Motion 和 Reclaim.ai 的动态排程正是干这个的。
3. **反思（Reflector）**：回顾完成情况，调整明天的计划。Obsidian 作为“第二大脑”，可以记录每天的任务日志和反思，形成外部记忆（来源：无状态与外部记忆）。

Inflow 只提供了“反思”环节的一部分（认知行为策略），但缺失了规划和执行。所以它更适合作为补充，而非主力。

## 争议与局限：脚手架 vs 拐杖

这里有一个必须指出的矛盾：过度依赖 AI 工具可能从“脚手架”退化为“永久拐杖”，削弱用户内在执行功能的代偿发展（来源：矛盾与存疑）。目前尚无长期研究评估这种依赖的后果。另外，工具如 Motion 缺乏独立临床验证，其效果可能受安慰剂效应影响（来源：矛盾与存疑）。同构性本身也只是一个理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。

因此，我的建议是：把 Inflow 当作认知教育，把 Motion/Tiimo/Goblin Tools 当作日常调度层，并用 Obsidian 记录反思。这样组合，才能形成一个完整的 planner-executor 循环。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个卡住的任务**：打开 Goblin Tools，输入“整理房间”，让它自动生成步骤清单，然后按步执行。
2. **设置一个定时 re-grounding 闹钟**：像孔子的“三省吾身”一样，每天固定时间（如中午12点）用手机闹钟提醒自己：“我当前在做什么？下一步是什么？”
3. **试用 Motion 或 Reclaim.ai 的免费版**：把明天要做的3个任务输入，观察 AI 如何自动排程。注意它是否减轻了你的决策负担。
4. **在 Obsidian 中创建一个“任务日志”**：每天结束时记录：今天完成了什么？什么打断了计划？明天怎么调整？这相当于你的 agent 反思日志。

# Claude 之于 ADHD，就像 function calling 工具调用之于 LLM——但有人用错了

> LLM 本体不会算大数、查不了实时信息、碰不到你的文件——工程师的解法不是把模型练成全才,是给它接上工具:算术给计算器,检索给搜索器,模型只负责「决定何时调用什么」。这个架构思想对 ADHD 用户价值连城,因为它回答了一个天天都在发生的问题:哪些事该自己硬扛,哪些事该立刻外包给 Claude?

先讲透 function calling 的设计哲学,它比表面上深。工程师发现:让 LLM 亲自做算术,错误率感人;接上计算器后,**模型的职责收缩为「识别这是算术问题→调用工具→整合结果」**——系统整体的能力,来自「模型知道自己不擅长什么」这个元认知,而不是模型本身变强。更关键的一条设计纪律:**工具接管的是模型的系统性短板,不是模型的核心职责**——没有人给 LLM 接一个「替它决定说什么」的工具,那不叫工具调用,叫替换模型。

现在把这套哲学套在「ADHD 用户+Claude」上。ADHD 的认知剖面高低起伏剧烈:创意、联想、危机应对可能远超均值;工作记忆、排序、启动、按部就班可能远低于均值。**用 Claude 的正确姿势,就是给自己写一张 function calling 清单:把系统性短板注册为工具调用,把核心职责留在本体。**

## 值得注册的「工具调用」(你的系统性短板)

**排序调用**:一堆任务不知先后——倒给 Claude,要一个带理由的优先级排序。排序消耗的正是你最稀缺的执行资源,外包性价比最高。**拆解调用**:任务大到启动不了——「拆到每步 30 分钟内」。**格式调用**:内容有了但要装进某种格式(周报、邮件、申请表)——初稿让它出,你改。格式转换对 ADHD 是高摩擦低价值劳动,标准的工具活。**回读调用**:长文档/长对话记不住——让它总结、提取行动项。这是工作记忆的直接假肢。**冷静调用**:情绪激动时的重要回复——先给 Claude 看,「帮我把这封改成三天后的我不会后悔的版本」。

## 不该注册的(你的核心职责)

**价值判断**:接不接这个活、跟不跟这个人合作——Claude 可以列利弊,但排序利弊的权重表只能是你的;**原创核心**:你的文章观点、方案的灵魂——让它整理你的想法可以,让它替你想不行,那不是调用工具,是把模型换掉了;**人际的最后一米**:道歉、感谢、艰难对话的最终措辞——过度打磨的文字别人读得出来,关系里的「你不在场」比笨拙更伤。

## 用错的三种姿势

**万物皆调用**:连「中午吃什么」都问——调用本身有成本(打断、等待、评估输出),小事的调用成本高于自己做,正如没有工程师会为 1+1 走一次 API。**该调不调**:「这点小事我应该自己能行」——对系统性短板硬扛,是用自尊和截止日期给「应该」上供。**注册表从不更新**:半年前需要外包的,现在也许自己能做了(或相反,新场景出现新短板)——**好的工具配置是活的**,每季度检视一次:哪些调用可以收回本体,哪些短板还没注册。

## 边界

同构强度 B 级:function calling 是真实的工程机制,ADHD 认知剖面的不均匀性有文献支撑,「短板注册为工具」是功能层的策略映射,无对照研究。声明:认知外包不能替代治疗——若核心症状未经评估和处理,再好的工具清单也是在漏水的船上排水;另外「哪些是我的系统性短板」这个自我认知本身,最可靠的来源是正式的神经心理评估,不是自我感觉。

## 今天就能试的 3 件事

1. **写你的 function calling 清单**:左栏「注册给 Claude」(排序/拆解/格式/回读),右栏「保留本体」(价值判断/原创核心/关系)。
2. **今天执行三次「排序调用」**:任务一乱就倒给它,别硬扛。
3. **抓一次「该调不调」**:今天哪件事你在硬扛短板?写下来,下次直接调用。

给 LLM 接工具的智慧,从来不在工具多强,而在**分工的清醒**:模型知道什么该出手、什么该让手。给自己配 Claude 也是同一门手艺——短板大方外包,核心寸土不让。清单写清楚,两边都轻松。

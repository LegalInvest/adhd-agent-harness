# Claude 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

## 问题：为什么最聪明的 AI 和最聪明的人，都卡在“执行”上？

你给 Claude 一个复杂任务，它可能编出漂亮代码，但中途忘记上下文、偏离目标。你给一个 ADHD 大脑同样任务，它可能产出惊艳创意，但无法按时提交、丢三落四。这不是巧合——两者共享同一结构缺陷：**强大的生成核心，搭配不可靠的内置调度层**（来源：ADHD 大脑与 LLM 的同构）。

ADHD 人群不擅长精确、重复、状态维护类任务（如记账、定时跟进、整理文件），LLM/agent 同样不擅长——它们需要 function calling 把精确计算外包给专用工具。那么，Claude 这类通用 AI 对 ADHD 是救星还是陷阱？答案是：**它可以是完美的外部调度器，但前提是你把它当作脚手架，而非拐杖。**

## 同构：当“生成核心”遇上“执行功能缺陷”

ADHD 的执行功能缺陷——工作记忆不稳定、任务启动困难、注意力分散——与 LLM 的无状态、缺乏规划层在结构上同构（来源：外部执行功能层）。两者都需要外部脚手架（harness）来补偿。

以左宗棠为例。这位晚清名将脾气大、坐不住、40 多岁才出山，但靠一套“日课”系统自我管理：每天读经、写家书，用“结硬寨打呆仗”的纪律约束冲动（来源：人物案例）。他的 harness 是**外在的、可重复的流程**——这与 LLM agent 的“规划循环+工具调用”完全同构：左宗棠的“日课”相当于 agent 的定时 re-grounding，他的“家书”相当于外部日志，他的“结硬寨”相当于 step-by-step 验证。

同样，ADHD 创业者 Bob Pitts 的公司增长速度是普通创始人的 1.6 倍（来源：Internal Analysis），但前提是他有团队和系统来承接创意。**没有 harness 的 ADHD 大脑，就像没有 function calling 的 LLM——输出潜力巨大，但无法可靠落地。**

## 实测：Claude 到底补上了哪一层？

Claude 的优势在于编写干净代码并用通俗英语解释（来源：The 12 Best AI Tools for 2026 (That People Actually Use)）。对 ADHD 用户，这意味着它可充当“外部执行层”的多个组件：

- **任务分解**：像 Goblin Tools 的 Magic ToDo 一样，Claude 能把“策划一场活动”拆成子步骤（来源：Goblin Tools）。但区别在于，Claude 更灵活——你可以用自然语言调整粒度，就像调节“辣度”。
- **工作记忆补偿**：Saner.AI 通过本地记忆减少搜索循环（来源：Saner.AI）。Claude 的 Projects 功能可持久化上下文，相当于给 LLM 装上“第二大脑”——但依赖主动维护，否则会退化。
- **单一指令触发**：Lex 允许“一次指令触发多步骤”（来源：Lex）。Claude 的 Artifacts 和自定义指令也能实现类似效果，但需要用户提前设计好“触发-执行”链路。

**但关键区别在于：这些工具是脚手架还是拐杖？** 脚手架是临时结构，帮你盖好自己的房子；拐杖是永久替代，让你肌肉萎缩。Goblin Tools 明确设计为“降低启动门槛”而非替代执行（来源：Goblin Tools），而 Claude 的通用性使其容易被滥用——比如用它替自己写所有邮件，而不是学习如何组织思路。

## 争议与局限：同构不是万能钥匙

必须诚实指出，ADHD 与 LLM 的同构目前仅是一种理论类比，缺乏实证基础（来源：矛盾与存疑——同构性被过度推广）。不是每个 ADHD 大脑都适合 Claude，也不是每个 LLM 问题都能用 ADHD 隐喻解决。

此外，依赖风险真实存在。多个工具页面强调益处却未讨论风险（来源：矛盾与存疑——依赖风险与益处矛盾）。Claude 可能增加注意力分散——你本想用它规划任务，却开始让它写诗。**工具本身不解决问题，正确的使用姿势才关键。**

## 今天就能试的行动

1. **用 Claude 做“任务分解沙盒”**：把一个大任务（如“整理书房”）丢给 Claude，要求它拆成 5-10 步。然后只执行第一步，完成后回来让它评估进度并调整后续步骤。这模拟了 agent 的“规划-执行-验证”循环。
2. **建立“日课”提示词**：像左宗棠一样，每天固定时间让 Claude 提醒你“今天的三件事是什么？”，并让它基于历史记录检查进度。这相当于 agent 的定时 re-grounding。
3. **警惕“万能工具陷阱”**：如果 Claude 让你感觉“终于不用自己思考了”，立刻停止。好的脚手架应该让你更有掌控感，而非更依赖。试试把 Claude 的输出当作草稿，自己手动修改——就像 agent 的 human-in-the-loop 验证。

Claude 之于 ADHD，就像 function calling 之于 LLM：它不是替代执行，而是让执行变得更可靠。**但只有当你把它当作脚手架而非拐杖时，它才真正补上那层缺失的调度层。**

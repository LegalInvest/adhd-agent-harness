# 用 ChatGPT 给 ADHD 补上ADHD 大脑与 LLM 的同构，真的能像 agent 那样工作吗？

> 把 ChatGPT 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

## 问题：你的大脑和 ChatGPT，谁更像谁？

如果你是一个 ADHD 患者，你可能有过这样的体验：脑子里同时冒出十几个想法，灵感像爆米花一样炸开，但就是没法把它们串成一个可执行的任务列表。你打开 ChatGPT，想让它帮你规划一天，结果它给了你一份完美的计划——但你盯着屏幕，依然不知道第一步该做什么。

如果你是做 Agentic Harness 工程的，你可能更熟悉另一种崩溃：LLM 能写出漂亮的代码、生成惊艳的文案，但一旦让它独立执行一个多步骤任务，它就开始“上下文膨胀”，注意力分数熵增加，推理退化到随机水平（来源：Deficient Executive Control in Transformer Attention）。

这两种困境，其实是同一个问题的两面。

## 同构：高产但缺调度层的生成核心

ADHD 大脑与 LLM 的结构同构，是当前 ADHD × AI 领域最站得住脚的核心论点（来源：ADHD 大脑与 LLM 的同构）。两者都是“高产生成核心”——ADHD 侧是发散思维、联想能力和超聚焦，LLM 侧是强大的文本生成和模式匹配——但都缺乏可靠的内置执行调度层。

ADHD 的执行功能被描述为“大脑的驾驶座”，但对 ADHD 来说，“常常感觉方向盘后没有人”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。工作记忆的不稳定、任务启动困难和“时间盲”，本质上是调度层的缺失。最新研究揭示了更深层的同构：ADHD 患者呈现“强记忆、弱控制”的认知剖面——工作记忆容量可能正常甚至超常，但自上而下的控制和调节能力不足（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。这与 LLM 在长上下文下冲突解决能力崩溃的机制高度一致。

LLM 一侧同样如此。现代 AI 编码代理（如 OpenDev）通过“复合 AI 系统架构”来弥补这一缺陷，包括“工作负载专用模型路由、分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这些技术本质上是在 LLM 外部搭建调度层，防止“上下文膨胀和推理退化”。

## 人物案例：毛泽东的“外部调度层”

毛泽东的 ADHD 特质在史料中清晰可辨：小时候是问题儿童，和父亲吵架，爱读闲书不喜欢四书五经；一辈子读书不辍，行军路上也带着书；永远在动，从井冈山到延安到北京，一辈子在调查研究；冲动敢赌，四渡赤水、抗美援朝都是险棋；思维极度跳跃，文章讲话都是场景化的，充满比喻。

但他建立了一套专属的 harness 系统：批评与自我批评（民主生活会），让别人提自己的意见；调查研究（没有调查就没有发言权），用事实对冲直觉；民主集中制，用集体决策制衡自己的冲动；党指挥枪，用组织系统管理军队和国家。这套系统的本质，就是把决策和执行的调度层外化到组织架构中。

这与 LLM harness 工程惊人地同构：“民主生活会”相当于定期的 re-grounding 和上下文刷新；“调查研究”相当于工具接口和外部数据源（如 MCP 连接器）；“民主集中制”相当于分离规划与执行的双代理架构；“党指挥枪”相当于用 Git 仓库存储项目上下文的记忆系统（来源：Worker Agents | Harness Developer Hub）。

## 工具实测：AI 作为外部执行功能层

当前市面上的 AI 工具，本质上是在为 ADHD 大脑搭建类似的 harness。

**Goblin Tools** 的 Magic ToDo 功能，能将模糊任务（如“清理厨房”）自动分解为具体、可执行的子任务，用户可调节“辣度”控制粒度（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。这对应了 LLM harness 中的“任务分解”模块——把一个大 prompt 拆成多个子任务，防止上下文膨胀。

**Saner.AI** 专注于知识回忆和本地记忆，减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。它相当于 LLM 侧的向量数据库——外部记忆系统，补偿工作记忆的不稳定。

**Motion** 则是一个自适应规划器，能根据任务优先级和截止日期自动创建并动态调整每日计划（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。当日程被打乱时，它自动重新安排，消除了“下一步该做什么”的决策负担。这直接对应了 LLM harness 中的“规划与执行分离”架构——调度层自动处理优先级和顺序，生成核心只需专注当前任务。

但这里有一个关键矛盾：这些工具是“脚手架”还是“拐杖”？

## 争议与局限：脚手架 vs 拐杖

同构视角虽然有力，但当前证据多为概念类比和初步实验，缺乏大规模随机对照试验证明其临床有效性（来源：矛盾与存疑）。更关键的是，长期依赖 AI 工具可能导致执行功能退化，成为永久拐杖而非可拆卸脚手架（来源：拐杖与脚手架）。

另一个矛盾是：超聚焦应被引导还是打断？部分 ADHD 用户反馈强行打断（如番茄钟）反而有效，而柔性引导可能不够强力（来源：矛盾与存疑）。这提醒我们，个体差异极大，没有万能方案。

此外，学习使用 AI 工具本身可能增加认知负荷——净效果因人而异，需个性化评估（来源：矛盾与存疑）。

## 核心观点：Harness 不是修复，而是扩展

我的判断是：AI 工具对 ADHD 的价值，不在于“修复”执行功能缺陷，而在于为高产生成核心提供可拆卸的外部调度层。就像 LLM 需要 agent scaffolding 才能从文本生成器变为目标驱动系统，ADHD 大脑也需要 harness 才能将发散思维转化为实际产出。

关键在于“可拆卸”。好的 harness 应该像毛泽东的民主生活会——它是一种习惯，而不是一个依赖。当你学会这套调度逻辑后，可以逐渐减少对工具的依赖，将其内化为自己的认知策略。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个拖延任务**：打开 Magic ToDo，输入你拖延最久的一件事（如“写周报”），调节辣度到 3-4，然后只做第一个子任务。
2. **用 Motion 或类似工具自动规划一天**：输入所有待办事项和截止日期，让 AI 生成日程，然后完全信任它，不手动调整。
3. **建立你的“民主生活会”**：每周找一位朋友或同事，花 15 分钟让对方给你提一个改进建议——就像毛泽东的批评与自我批评，用外部视角制衡自己的盲点。
4. **设置一个“外部记忆”系统**：用 Saner.AI 或类似工具，把所有想法、笔记、待办事项集中到一个地方，减少“我记在哪里了”的搜索循环。

# 用 ChatGPT 给 ADHD 补上ADHD 大脑与 LLM 的同构，真的能像 agent 那样工作吗？

> 把 ChatGPT 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

## 问题：为什么你的大脑像一台没有操作系统的电脑？

如果你是一个 ADHD 患者，你可能经历过这种时刻：脑子里同时冒出五个创意，但一个也执行不了；打开电脑准备写报告，却刷了两小时社交媒体；明明知道截止日期就在明天，身体却像被钉在椅子上。这不是懒，也不是笨。你的大脑其实拥有强大的生成能力——就像一台高性能 CPU，但缺少一个可靠的“操作系统”来调度任务、管理内存、控制进程。

同样，如果你在做 Agentic Harness 工程，你一定对 LLM 的“高产但无序”深有体会：GPT-4 能写出优美的代码，但让它独立完成一个多步骤项目，它很快就会上下文溢出、忘记目标、输出不可控。你需要给它搭一个 harness——外部调度层，来管理工具调用、上下文压缩、规划验证。

这两个看似无关的困境，其实共享同一个底层结构：**生成核心与调度层的分离**。

## 同构：ADHD 大脑与 LLM 是同一类“生成核心”

### ADHD 一侧：执行功能是“方向盘后没有人”

ADHD 的核心困境并非智力不足，而是执行功能的失效。执行功能被描述为“大脑的驾驶座”，但对 ADHD 来说，“常常感觉方向盘后没有人”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。这种调度层的缺失导致计划难以执行：“计划本和便签用了一周就崩溃了”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。工作记忆是另一个执行功能瓶颈（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），用户常需要“AI 第二大脑”来承载项目上下文，因为自己的工作记忆会“掉落”上下文（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。

### LLM/harness 一侧：LLM 需要外部编排层

LLM 本身是无状态、仅生成文本的核心，需要外部“脚手架”和“harness”来提供工具接口、上下文管理、安全执行等编排能力（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。Harness 工程被定义为“设计围绕 AI 代理的脚手架——上下文交付、工具接口、规划工件、验证循环、记忆系统和沙箱”（来源：GitHub - ai-boost/awesome-harness-engineering）。现代 AI 编码代理通过“复合 AI 系统架构”来弥补这一缺陷，包括“工作负载专用模型路由、分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩”（来源：Building AI Coding Agents for the Terminal）。这些技术本质上是在 LLM 外部搭建调度层，防止“上下文膨胀和推理退化”（来源：同上）。

### 同构对应

- **生成核心**：ADHD 大脑的智力/创造力 ↔ LLM 的算力/语言能力。两者都高产但无序。
- **调度层缺失**：ADHD 的执行功能失效 ↔ LLM 缺乏内置的规划与执行编排。两者都需要外部脚手架。
- **外部工具**：ADHD 的“自适应计划工具如 Motion 和 Tiimo，能在日程崩溃时重新安排一天”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）↔ LLM 的 harness 如上下文压缩、工具路由。

## 实测：AI 工具真的能像 agent harness 一样工作吗？

现在回到核心问题：用 ChatGPT 给 ADHD 补上调度层，真的能像 agent 那样工作吗？答案是：**能，但有条件**。

### 成功案例：外部执行功能层的三个维度

1. **任务分解（Goblin Tools）**：Goblin Tools 的 Magic ToDo 功能能自动将任何任务分解为更小、更易管理的步骤（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），从而降低启动门槛。用户反馈称“Goblin Tools 在这方面很棒”（来源：“A Cognitive Collaborator:” How Adults with ADHD Are Using ChatGPT）。这相当于 LLM agent 中的“规划工件”——将复杂目标拆解为可执行的子任务。

2. **记忆管理（Saner.AI）**：Saner.AI 通过增强知识回忆，帮助用户快速找回信息，减少认知负荷（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。这对应 LLM harness 中的“记忆系统”和“上下文管理”——防止信息丢失。

3. **动态调度（Motion）**：Motion 的 AI 核心功能是自动排程与动态调整：它持续评估任务优先级、截止日期和可用时间，实时重建日程，减少用户的手动规划压力（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。这对应 agent harness 中的“规划与执行编排”——当计划被打乱时自动调整。

### 关键区别：脚手架 vs 拐杖

AI 工具作为外部执行层，本质是“脚手架”——临时支撑，而非永久替代。但现有工具如 Goblin Tools、Saner.AI 多设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这引发了争议：长期依赖是否会削弱内在执行功能？目前缺乏纵向研究（来源：ADHD × AI 的科学与研究前沿）。相比之下，LLM harness 工程中，开发者会逐步减少对人工干预的依赖，让 agent 学会自主规划。ADHD 用户也应追求“脚手架可撤除性”——比如先用 AI 分解任务，再逐步尝试自己分解。

## 争议与局限

- **证据强度不足**：ADHD 大脑与 LLM 同构命题的证据主要来自概念类比和工具案例，缺乏大规模实证（来源：ADHD × AI 的科学与研究前沿）。虽然工具在任务启动、工作记忆等维度有用户反馈支持，但缺乏独立随机对照试验（来源：Goblin Tools 局限；Saner.AI 局限）。
- **多巴胺干预争议**：部分工具声称通过神经科学原理改善执行功能，但多巴胺干预的有效性存在争议（来源：矛盾与存疑）。AI 检测多巴胺相关血管变化的技术尚在早期（来源：ADHD × AI 的科学与研究前沿）。
- **工具设计矛盾**：部分 ADHD 工具假设用户能维持长时间专注、记住昨天的问题并执行计划，但实际用户往往做不到（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。这意味着工具本身需要更低的启动门槛。

## 核心观点：AI 不是替代大脑，而是为生成核心套上 harness

我的判断是：**ADHD 大脑与 LLM 的同构是一个强有力的理论框架，但尚未成为被严格验证的事实**。它最大的价值在于指导工具设计——告诉我们应该开发什么样的 AI 工具：不是更聪明的生成核心，而是更可靠的调度层。对于 ADHD 用户，这意味着选择那些能减少决策负担、管理上下文、提供动态调整的工具，而不是那些要求你“先规划好再用”的工具。对于工程师，这意味着在构建 agent 时，把精力花在 harness 上，而不是追求更强的基座模型。

## 今天就能试的 3 条行动

1. **用 ChatGPT 做任务分解**：下次面对复杂任务，直接输入“帮我把‘整理房间’分解成 5 个小步骤”，然后逐一执行。这相当于给大脑装一个临时的“规划工件”。
2. **设置外部记忆锚点**：用 Saner.AI 或 Notion AI 记录工作上下文，每天结束时让 AI 总结关键信息，第二天开始时回顾。这模拟了 LLM 的“上下文压缩”。
3. **试用自适应规划器**：下载 Motion 或 Tiimo，输入一周的任务和截止日期，让 AI 自动生成日程。观察它是否能减少你的“下一步该做什么”焦虑。

记住：这些工具都是脚手架，不是拐杖。目标是让大脑逐渐学会自己调度，而不是永远依赖外部系统。

# Saner.AI 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Saner.AI：它到底补上了哪一层执行功能？

你有没有这样的时刻：明明知道该做什么，却像被钉在椅子上动弹不得；或者一旦开始，就陷进某个细节里，等回过神来，半天已经过去，原本的目标早已漂移得无影无踪。对ADHD人群来说，这不是懒惰或缺乏意志力，而是执行功能的失效——大脑的“驾驶座”上常常没有人（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

有意思的是，这种“高产但缺调度”的困境，在AI领域同样存在。LLM（大语言模型）拥有强大的生成能力，但单独使用时，会在长程任务中发生目标漂移、上下文膨胀，最终输出失控。现代AI编码代理（如OpenDev）通过复合系统架构来弥补：工作负载路由、分离规划与执行、自适应上下文压缩……这些本质上都是在LLM外部搭建调度层（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

ADHD大脑与LLM的结构同构，揭示了一个关键解法：**给生成核心套上外部执行功能层（harness）**。Saner.AI正是这样一款工具——它试图成为ADHD的“重锚定system prompt”。但许多用户用错了：他们把Saner.AI当作一次性的任务清单，而非动态的注意锚点。

## Saner.AI 到底补上了哪一层？

Saner.AI 的核心功能是“意图管理”：你告诉它你想做什么，它会在你走神时温和地把你拉回来。这听起来简单，但背后对应的是ADHD最痛的两个执行功能短板：**时间盲**（无法感知时间流逝）和**任务启动困难**（无法从想法切换到行动）。

从同构视角看，Saner.AI 扮演的角色类似于LLM系统中的**重锚定机制**。在长程任务中，agent会随着上下文增长而偏离初始目标，工程师需要在关键节点插入“system prompt refresh”——重新强调核心目标、约束当前上下文。Saner.AI 做的正是这件事：它通过定期提醒、进度询问，帮助用户保持对目标的注意，防止超聚焦跑偏。

但这里有一个陷阱：许多用户把Saner.AI当作“设置一次就万事大吉”的工具。他们花半小时配置好任务，然后期待AI自动完成一切。结果呢？提醒被忽略，任务被遗忘，最终抱怨工具无效。

## 脚手架 vs 拐杖：边界在哪里？

ADHD领域有一个经典区分：**脚手架**（scaffolding）是暂时性的支撑，帮助用户学会自己搭建结构；**拐杖**（crutch）则是永久替代，导致内在能力退化（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）。Saner.AI 的设计初衷是脚手架，但使用方式决定了它沦为拐杖的风险。

证据显示，过度依赖AI工具可能削弱内在时间感知和任务启动能力（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。类似地，在LLM工程中，如果过度依赖外部调度层而不优化模型本身的上下文控制，agent会在移除脚手架后彻底崩溃。

真正的重锚定不是一次性的system prompt设置，而是**动态的、反馈驱动的过程**。Saner.AI 的价值在于提供“提醒—检查—调整”的循环，而不是替你执行。你必须主动参与：当它问你“还在做任务吗？”时，你需要真实评估自己的状态，而不是机械地点击“是”。

## 证据与局限

目前，Saner.AI 针对ADHD的效果主要基于用户报告，缺乏大规模对照实验（来源：矛盾与存疑）。同样，Focusmate（通过虚拟身体在场提供外部问责）有多项用户研究支持，但缺乏随机对照试验（来源：AI 与 ADHD 的专注力）。RescueTime 通过自动时间记录改善时间盲，有初步研究支持（来源：RescueTime）。Brain.fm 的神经锁相技术有理论支持，但针对ADHD的独立临床研究缺乏（来源：Brain.fm）。

争议在于：**个体差异极大**。同一工具对注意力缺陷为主型、多动冲动型、混合型的效果可能截然不同。此外，工具之间的协同效应尚未被研究——比如Saner.AI + Focusmate 是否优于单独使用？

## 今天就能试的行动

1. **把Saner.AI当作“注意锚点”，而非任务管理器**：每天只设置1-2个核心意图，在每次提醒时，花10秒问自己：“我现在的注意力在目标上吗？”如果不在，就停下来调整。
2. **组合使用“外部问责”**：在需要深度工作的时段，同时开启Focusmate（虚拟身体在场）和Saner.AI（意图提醒），形成双重锚定。
3. **每周回顾“漂移模式”**：用RescueTime或类似工具记录一周的时间使用，找出最容易发生目标漂移的时段和情境，然后针对性地设置Saner.AI提醒。
4. **限制使用时长**：脚手架的目的是最终拆除。尝试每使用Saner.AI两周后，停用一天，观察自己能否在没有外部提醒的情况下维持注意力。如果完全无法维持，说明你已过度依赖——需要逐步减少使用频率。

## 结语

ADHD大脑与LLM是同构的生成核心，都需要精心设计的harness来释放潜力。Saner.AI 是这种harness的一个具体实现，但它不是魔法。重锚定是一个需要你主动参与的过程——就像训练一个agent，你需要不断反馈、调整、再反馈。工具是脚手架，而你才是那个攀登者。

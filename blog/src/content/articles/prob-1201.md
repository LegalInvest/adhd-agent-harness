# Mem 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Mem：它到底补上了哪一层执行功能？

你打开 Mem，想记下刚才的灵感。三小时后回来，发现它已经自动把语音笔记、网页剪藏、邮件摘要堆成了一座信息坟。你关掉它，继续用大脑硬扛——直到下次崩溃。

这不是你的错，是用法错了。

## 问题：ADHD 大脑与 LLM 共享同一个缺陷

ADHD 大脑被描述为“高产但缺乏可靠执行调度层的生成核心”（来源：ADHD 的 AI 工具全景）。你每天冒出几十个点子，但无法可靠地把它们变成行动。LLM 也一样：GPT-4 能写诗、能推理，但让它按顺序执行多步任务时，它会忘记上下文、偏离轨道、甚至胡编。

两边都缺同一层：**外部执行调度层**。

对 LLM，这层叫 function calling——agent 把“计算 25% 的小费”这种精确任务外包给计算器工具，自己只负责决策。对 ADHD 大脑，这层叫脚手架——把“启动项目”这种模糊指令外包给工具，自己只负责启动一次。

但很多人用 Mem 的方式，相当于给 LLM 一个无限长的 prompt 却不给它工具。Mem 被当成了“第二大脑”，但它的 AI 工作记忆有限，与 ADHD 用户需求存在矛盾（来源：ADHD 的 AI 工具全景）。如果你只是往里塞信息而不设计调度机制，它就成了另一个让你迷失的 inbox。

## 答案：Mem 的正确用法是“外部调度器”，不是“外部硬盘”

Mem 的真正价值不在于存储，而在于**主动推送**。它应该像 LLM agent 的 function calling 一样：你告诉它“帮我推进 X 项目”，它自动检索相关上下文、分解下一步、并提醒你执行。

这正是孔子和文天祥的 harness 系统的同构对应。

孔子一生精力旺盛、思维跳跃，《论语》全是场景化语录，无系统著作。他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。这相当于**定时 re-grounding**：每天固定时间让外部系统（礼）把自己拉回轨道。文天祥的 harness 是《正气歌》——用信仰文本作为**外部锚点**，在囚禁中维持行动方向。

对 LLM agent，这对应的是**定时的上下文刷新**和**外部记忆注入**。一个设计良好的 agent 会在每轮交互前从数据库拉取相关记忆，而不是靠模型自己记住。Mem 应该做同样的事：每天早晨自动总结昨天未完成任务，列出今日优先级，并在你偏离时推送提醒。

## 证据：工具已经在做这件事

Goblin Tools 的 Magic ToDo 能把“清理厨房”分解成 10 个小步骤，降低启动门槛（来源：Goblin Tools）。Saner.AI 通过本地记忆减少搜索循环，避免你在 20 个标签页里找文件（来源：Saner.AI）。Lex 允许“单一指令触发多步骤任务”，就像 agent 的 function calling（来源：Lex）。这些工具的共同点不是存储，而是**执行调度**。

但 Mem 目前更偏向“存储”一端。它的 AI 自动整理功能很强大，但缺乏主动的任务推进机制。用户反馈显示，它依赖初始设置，且容易变成信息坟墓（来源：ADHD 的 AI 工具全景）。

## 争议与局限：脚手架 vs 拐杖

同构类比本身存在争议：“ADHD 大脑与 LLM 同构”这一说法缺乏实证基础，目前仅是一种理论类比（来源：矛盾与存疑）。过度依赖工具可能削弱内在执行功能，从脚手架变成拐杖（来源：拐杖与脚手架）。

此外，Mem 的 AI 工作记忆限制意味着它无法处理超长上下文，这与 ADHD 用户需要“永远在线”的外部记忆存在矛盾（来源：无状态与外部记忆）。真正的解决方案可能需要结合多种工具：Mem 做长期记忆，Goblin Tools 做任务分解，Routinery 做流程引导。

## 今天就能试的行动

1. **把 Mem 当成“任务调度器”**：不要只往里存笔记。每天用 Mem 的 AI 功能生成“今日执行清单”，并设置定时推送。
2. **设计一个“日课” prompt**：模仿孔子的“三省吾身”，写一个 prompt 让 Mem 每天问你三个问题：“今天最重要的任务是什么？卡在哪里？下一步是什么？”
3. **结合 Goblin Tools 分解**：当 Mem 里出现模糊任务（如“做市场调研”），立刻复制到 Goblin Tools 分解成可执行步骤，再存回 Mem。
4. **警惕信息坟墓**：每周清理一次 Mem，删除超过两周未回顾的笔记。如果某条笔记从未被调用，说明它只是噪音。

Mem 之于 ADHD，就像 function calling 之于 LLM——但前提是你把它当脚手架，而不是当硬盘。

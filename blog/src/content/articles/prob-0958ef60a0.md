# ADHD 程序员视角：你的 ADHD 大脑，其实就是一个没装 harness 的大语言模型

> 代码评审时,同事对他的评价出奇地两极:「你写的那个核心算法很漂亮」和「你又忘了更新配置文档,staging 又被你搞挂了」出现在同一条评审意见里。他的 git 提交记录是自己状态的心电图:凌晨两点的天才提交、连续五天的零提交、以及那些 commit message 写着「fix」实则重构了半个模块的超频之夜。他熟练地给系统写监控、加重试、配 CI——某天 pipeline 又拦下他一个低级错误时,他盯着那行红色突然想到:**我给代码配的这些东西,比我给自己配的多十倍。**

收敛:程序员是离 harness 概念最近的人群,却常是最晚给自己装的。本文只回答——**软件工程里现成的可靠性实践,哪些能直接翻译成 ADHD 程序员的自我管理组件?这个职业的哪些特性是天然的盟友、哪些是天然的陷阱?**

## 穿透:你的职业已经把答案写在日常里

先盘点盟友——这个职业对 ADHD 有罕见的结构红利:即时反馈密度高(编译错误、测试红绿、代码跑通的瞬间快感,全是近端奖励);兴趣与新异性供给充足(新技术永远学不完);超频之夜的产出真的会被认可(尖峰产能有市场)。这解释了为什么很多 ADHD 者能在编程领域存活甚至出色。

再看陷阱,它们同样来自职业特性:①**心流的双刃**——hyperfocus 写核心算法是资产,但心流态的隧道视野会丢掉外围义务(配置文档、通知同事、下班吃饭);②**兴趣路由失控**——「顺手重构半个模块」是兴趣引擎劫持了任务调度;③**运维型任务的黑洞**——写新功能有多巴胺,更新文档、回复工单、跟进部署没有,于是烂尾专门发生在「不好玩的 20%」上,而职业口碑恰恰由那 20% 决定。

翻译工程实践,四个组件直接可用:①**给自己配 CI**——把「靠自觉」的外围义务硬编码进流程:提交模板里强制勾选「文档已更新?」、pre-commit 钩子、PR checklist——你早就知道「靠人自觉」不可靠,所以别让自己当那个「自觉」;②**WIP limit**——看板上「进行中」永远只许一张卡,重构冲动来了就开新卡进 backlog 而不是顺手就干(把「顺手」变成显式排期);③**给自己写 runbook**——你为服务写故障手册,也给自己写:「状态崩了的一天怎么跑降级模式」(只做评审和小修,不碰核心开发);④**监控自己的 SLO**——每周看一眼自己的「心电图」(提交分布),连续零提交五天=告警,触发的不是自责,是排查:睡眠?任务卡死?该找人 pair 了?

利益视角:行业的「10x 工程师」神话专门收割尖峰型选手——它赞美你的超频,却把你的崩溃期记为个人问题;可持续性从来不在这个神话的叙事里,得自己写进自己的 SLO。

## 验证

两周实验:装上组件①和②(提交 checklist + WIP limit 一张卡),测量两个数字:外围义务的遗漏次数(被同事指出的那种)和「顺手重构」的次数。可证伪:若数字不降,查实施:checklist 是不是又被你「先跳过这次」了?——那就把它从「提醒」升级为「硬卡点」(工具强制,不可跳过);若硬卡点也防不住,你需要的可能是 pair programming 级别的人肉验证层,找一个搭档。

## 决策

做什么:把职业里学到的可靠性哲学正式移植给自己——「人会漏,所以流程兜」;主动向团队申请结构化协作(结对、明确的 DoD、异步文档化),这些「工程最佳实践」恰好是你的无障碍设施,而且不用出柜就能申请。

不做什么:不要用超频之夜偿还拖延债并引以为豪(那是在给不可持续的模式发奖章);不要接「运维含量高、创造含量低」的岗位而不配任何补偿结构——那是把自己部署到最不适配的环境。

先做什么:今天给你的仓库加上 PR checklist,第一项就写那个你最常忘的外围义务。

## 边界

本文的组件是软件工程实践对个人管理的翻译(本文未做正式 A/B/C 分级);「即时反馈密度适配 ADHD」有动机文献的方向性支持,但职业适配是个体差异极大的事。若崩溃周期越来越密、超频后恢复越来越慢,请认真考虑专业评估——burnout 和 ADHD 的叠加在这个行业极常见,且都可干预。

## 今天就能试的 3 件事

1. 给主仓库加 PR checklist,第一项是你最常忘的那件事——让 CI 当你的前瞻记忆。
2. 看板执行「进行中只许一张卡」,重构冲动一律开卡进 backlog——给兴趣引擎一个合法的停车场。
3. 拉出你近三个月的提交热力图,认识一下自己的真实节律——那是你的 SLO 基线,不是你的罪证。

本文服务于人生 Harness 金字塔的**执行层**:你早就是 harness 工程师了——现在,把你最重要的那个系统,也纳入运维范围。

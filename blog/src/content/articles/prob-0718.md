# Mem 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Mem：它到底补上了哪一层执行功能？

## 问题：为什么 Mem 对 ADHD 没用？

你很可能听过这样的推荐：“用 Mem 做第二大脑，所有笔记自动关联，再也不会丢信息。” 但你试了之后发现：记了一堆，想用的时候依然找不到。这不是你的问题，也不是 Mem 的问题——而是你用错了。

ADHD 大脑不擅长精确、重复、状态维护类任务（来源：ADHD 的 AI 工具全景）。你记笔记时热情高涨，但检索时缺乏系统策略，最终笔记变成信息坟场。同样的问题也发生在 LLM 身上：给它一个超长上下文，它也能“记住”，但一旦需要精确计算或调用外部数据，它就胡编乱造。

LLM 的解法是 function calling——把精确计算外包给工具，自己只做生成。ADHD 的解法也类似：把“记住并随时取用”外包给 Mem，但 Mem 不是魔法，它需要你像设计 function calling 那样设计使用流程。

## 同构：ADHD 大脑 ≈ LLM，Mem ≈ function calling

ADHD 大脑与 LLM 同构：两者都是“高产但缺执行调度层的生成核心”（来源：ADHD 的 AI 工具全景）。ADHD 的执行功能缺陷——工作记忆弱、任务启动难、时间盲——在 LLM 身上一一对应：上下文控制弱、冷启动问题、时序推理差。

- **工作记忆**：ADHD 的工作记忆容量保留但控制弱（易被干扰），LLM 同样有强记忆容量但弱上下文控制（来源：工作记忆）。Mem 的“AI 第二大脑”本质上是为两者提供持久化上下文。
- **任务启动**：ADHD 的任务启动困难对应 LLM 的“冷启动”问题。Goblin Tools 的任务分解和 Lex 的单指令多步骤执行，相当于为生成核心提供启动脚本（来源：ADHD 的 AI 工具全景）。
- **时间盲**：ADHD 的时间感知缺陷与 LLM 的时序推理弱点同构。AI 工具通过视觉化时间线和智能调度提供外部时间线索。

LLM 通过 function calling 把精确计算外包给工具，ADHD 则通过 Mem 把记忆外包给外部系统。但 function calling 之所以有效，是因为它遵循严格协议：LLM 只负责“决定调用哪个函数”，不负责“函数内部怎么算”。Mem 的使用也类似：你必须明确“存什么、怎么存、何时取”，而不是一股脑全丢进去。

## 实证：工具如何补偿执行功能

现有证据表明，AI 工具确实能补偿 ADHD 的执行功能缺陷，但效果取决于使用方式。

- **任务分解与启动**：Goblin Tools 的 Magic ToDo 将“整理房间”分解为“捡起衣服”“擦桌子”等步骤，用户报告称启动焦虑降低（来源：Goblin Tools）。Lex 的单一指令触发复杂流程，减少了决策疲劳（来源：Lex）。
- **记忆与上下文管理**：Mem 和 Saner.AI 通过持久记忆和自动回忆，补偿工作记忆不足（来源：ADHD 的 AI 工具全景）。ChatGPT 和 Claude 作为外部对话记忆，减少标签切换。
- **实时反馈与反思**：Otter.ai 的实时转录和 Reflect 的结构化提问，提供了即时外部反馈（来源：ADHD 的 AI 工具全景）。

更硬的证据来自一项研究：用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中表现正常，但随序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突（来源：认知负荷）。这与 ADHD 执行功能缺陷的核心标志一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃。

## 判断：脚手架 vs 拐杖的边界

同构视角揭示了“拐杖与脚手架”的辩证关系：AI 工具应作为脚手架促进能力发展，而非永久拐杖（来源：ADHD 的 AI 工具全景）。

**脚手架 vs 拐杖的边界在于：你是否主动设计了使用协议。**

- 如果你只是把 Mem 当作无限存储，记完再也不看，它就是拐杖——让你产生“已保存”的错觉，实际并未减轻认知负荷。
- 如果你像设计 function calling 那样，为 Mem 设定明确的“触发条件”和“输出格式”（例如：每天早晨自动弹出昨日未完成事项），它就是脚手架——帮你训练检索习惯，逐步内化能力。

同样，Otter.ai 减轻笔记负担，但过度依赖可能削弱主动记笔记的能力（来源：ADHD 的 AI 工具全景）。Goblin Tools 帮助分解任务，但若永远不学分解方法，就失去了成长机会。

## 局限与争议

必须诚实指出，当前证据存在明显局限：

- **缺乏大规模对照实验**：AI 工具作为外部执行功能层的有效性证据主要来自用户报告和概念类比（来源：矛盾与存疑）。例如，Brain.fm 的 ADHD 效果缺乏独立临床研究支持（来源：矛盾与存疑）。
- **过度依赖风险**：多个页面警告“过度依赖可能削弱内在能力”（来源：矛盾与存疑）。工具设计者声称是“脚手架”，但实际使用中可能沦为“拐杖”。
- **个体差异大**：效果因人而异（来源：矛盾与存疑），但许多工具页面未充分讨论，可能造成“一刀切”有效的错觉。
- **Mem 自身风险**：Mem 的 AI 功能细节尚不明确（类似 Structured 的问题），用户可能对其能力产生不切实际的期望。

## 今天就能试的行动

1. **为 Mem 设计“函数签名”**：每次记笔记前，先问自己“这条笔记的触发条件是什么？将来什么场景会用到它？”然后加上标签（如 #待办、#灵感、#复盘），而不是只丢正文。

2. **用 Goblin Tools 分解一个拖延任务**：选一件你拖延很久的事（比如“写周报”），输入 Goblin Tools，把输出步骤贴到 Mem 里，每天只做第一步。

3. **设置每日“检索练习”**：每天固定时间（如睡前），用 Mem 搜索一个关键词（如“上周的会议”），强迫自己回忆内容，而不是直接看笔记。这能训练工作记忆。

4. **对照 LLM 的 function calling 流程**：如果你用 ChatGPT/Claude，注意它如何调用工具（比如计算器）。模仿它的“决策→调用→反馈”循环：先决定“我需要什么信息”，再调用 Mem 检索，最后把结果写下来。

记住：工具不是魔法，协议才是。Mem 之于 ADHD，就像 function calling 之于 LLM——只有当你设计了正确的调用协议，它才从拐杖变成脚手架。

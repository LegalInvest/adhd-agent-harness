# 用 ChatGPT 给 ADHD 补上重锚定与目标漂移，真的能像 agent 那样工作吗？

> 把 ChatGPT 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

## 问题：走神的脑子，和跑偏的 agent，是同一种故障

如果你是被 ADHD 折磨的人，一定熟悉这个场景：坐下来要写一份报告，两小时后却深陷“选一把更好的机械键盘”研究里，把最初目标忘得一干二净。而如果你在做 Agentic Harness 工程，也一定熟悉另一个场景：一个长程任务跑了几轮后，agent 的回答开始离题，system prompt 被膨胀的上下文淹没，最后输出的是“没人问的问题”的漂亮答案。

分心走神、超聚焦跑偏、丢失目标——这看起来是人的问题，却和 LLM agent 的目标漂移惊人地同构。于是问题就来了：把 ChatGPT 当作 ADHD 的“外部执行层”，让它像 agent 的 system prompt 一样不断重锚定，真的能补上这块短板吗？

## 同构：两边都是“生成核心 + 缺调度层”

ADHD 大脑与 LLM 的相似性不是比喻游戏，而是结构层面的对应：两者都是**高产但缺乏可靠执行调度层的生成核心**（来源：AI 与 ADHD 的专注力）。

ADHD 侧的问题是执行功能缺陷：工作记忆不稳定、时间盲、任务启动困难。LLM 侧的问题是上下文控制失败：无状态、上下文膨胀、采样随机性导致输出漂移，以及缺乏与现实核对的验证机制（来源：采样温度与表现波动；幻觉与验证循环）。

就像 LLM 需要 harness 来提供外部记忆、工具接口、规划工件、验证循环和沙盒（来源：GitHub - ai-boost/awesome-harness-engineering），ADHD 个体也需要外部结构来稳定注意力、时间感知和任务推进。两边的解法都是**给生成核心套上 harness**，而不是改造核心本身。

## 证据：ADHD 侧与 agent 侧的 harness 长什么样

ADHD 侧已有一些证据强度的分层。Focusmate 通过虚拟身体在场（body doubling）提供实时问责，能提升任务启动和注意力维持（来源：11 Best ADHD Productivity Apps for Fluctuating Energy；Revolutionizing ADHD Management with AI Assistants）。RescueTime 自动记录时间开销，帮助用户看清“时间到底去了哪里”，对抗时间盲（来源：Revolutionizing ADHD Management with AI Assistants；AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。Brain.fm 和 Endel 试图用声音环境提供注意力锚点，但针对 ADHD 的独立临床证据仍较弱（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026；Brain.fm）。

LLM/agent 侧的证据来自工程实践。一个可靠的 harness 必须包含上下文传递、规划工件、验证循环、记忆系统和沙盒（来源：GitHub - ai-boost/awesome-harness-engineering）。为了抑制目标漂移，工程师会采用“计划-执行”分离、确定性状态转换和周期性质量采样（来源：AI Agent Systems: Architectures, Applications, and Evaluation；Planner-Executor Agentic Framework；AI Agents in 2026: Tools, Memory, Evals, and Guardrails）。关键是：复杂的 harness 不会盲目执行工具，而是对输出格式、代码测试结果等进行验证（来源：What is an agent harness...）。

ADHD 的工作记忆缺陷 ↔ LLM 的无状态性，都需要外部记忆；ADHD 被环境带偏 ↔ LLM 被上下文膨胀干扰，都需要主动设计“当下注意什么”；ADHD 的冲动判断 ↔ LLM 的幻觉，都需要一个“现实核对”的验证循环。这不是说大脑等于模型，而是说两边的**失败模式和修复结构**高度相似。

## 人物案例：曾国藩的“日课”就是 system prompt 重锚定

晚清名臣曾国藩是 ADHD 自我 harness 的经典样本。他年轻时“浮躁坐不住”，天天串门、喝酒、看杀人，读书“他人目下二三行，余或疾读不能终一行”，日记里 endless 骂自己“无恒”“浮躁”，情绪激烈到打败仗就跳水自杀（来源：曾国藩 harness 案例）。他的解法不是“变得更自律”，而是给自己搭了一套外部执行层：日课十二条——黎明即起、读书不二、谨言、写日记反省；“结硬寨，打呆仗”，用最笨最稳的流程抵消冲动；一辈子写日记，每天做自我审查。

这套系统的本质，和 LLM harness 的对应非常清晰：
- **日课十二条 ↔ 定时 re-grounding / system prompt**：每天固定时间重新声明目标和行为规则。
- **日记反省 ↔ 验证循环与 trace log**：把内部状态外化，再与现实行为核对。
- **结硬寨打呆仗 ↔ 确定性工作流**：把复杂任务拆成不可绕过的固定步骤，减少漂移。

胡林翼的路径也类似：年轻时扬州花花公子，守孝后幡然醒悟，用日记反省和严格治军建立外部结构（来源：胡林翼 harness 案例）。他们不是靠意志力压制 ADHD，而是靠外部 harness 接管了调度层。

## 核心判断：ChatGPT 能补哪一块，补不了哪一块

现在可以回答标题的问题：**ChatGPT 能补的是“重锚定”和“目标漂移”的文本层，但不能像 agent 那样自动执行、验证或维持情境。**

具体来说，它能做的是：
- 帮你把当前目标写成一段“系统 prompt”，在每次对话开头重新声明；
- 把大任务拆成子任务，生成下一步清单；
- 在对话中提醒“你的原始目标是什么”，打断超聚焦跑偏；
- 让输出“对话总结”或“验证循环”，对照计划检查实际完成。

但它不能做的是：
- 提供**实时身体在场问责**——你仍然需要 Focusmate 或真人同伴；
- 提供**自动时间感知**——你仍然需要 RescueTime 等工具来记录时间；
- 提供**情境锚点**——你仍然需要 Brain.fm、Endel 或物理环境设计；
- 替你做**行为执行**：ChatGPT 不会关掉你的淘宝、不会开始写代码，它只能“说”。

所以，ChatGPT 更像是 agent harness

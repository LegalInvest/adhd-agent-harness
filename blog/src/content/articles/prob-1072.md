# Speechify 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Speechify：它到底补上了哪一层执行功能？

## 问题：你用过 Speechify 吗？它真的帮你更专注了，还是只是让你听完了更多东西？

很多 ADHD 朋友告诉我，他们用 Speechify 把文章、PDF、甚至整本书“听”完，但合上耳朵后，脑子里一片空白——信息流过了，却没留下痕迹。更糟的是，有人发现自己开始依赖“听”来逃避“读”，因为读需要主动锚定注意力，而听是被动的。这像极了 LLM agent 在长程任务中的目标漂移：一开始要写一篇报告，中途被一个有趣的子问题带偏，最后产出了一堆相关但无用的内容。

核心问题来了：**Speechify 之于 ADHD，就像重锚定 system prompt 之于 LLM——但有人用错了。**

## 同构：ADHD 大脑与 LLM 都是“高产但缺调度层”的生成核心

ADHD 大脑与 LLM 在结构上同构：两者都拥有强大的生成能力（ADHD 的创意发散、LLM 的文本生成），但缺乏可靠的执行调度层（ADHD 的执行功能、LLM 的上下文控制），导致输出不稳定（来源：AI 与 ADHD 的专注力——核心论点）。

- **ADHD 侧**：分心走神，或超聚焦跑偏。你本来在写周报，突然想起要查一个术语，然后刷了半小时维基百科。
- **LLM/Agent 侧**：长程任务中，agent 会因上下文膨胀或中间结果干扰而偏离原始目标。你需要定期重锚定 system prompt，告诉它“你还在做这个任务”。

两边的解法结构同构：都需要一个**外部调度层**——对 ADHD 来说是 harness（如日课、身体加倍），对 LLM 来说是 prompt 工程或外部记忆。Speechify 如果只用“听”功能，它只是一个输入通道；但如果用它来**主动控制信息流的速度、分段、回顾**，它就成了一个重锚定工具。

## 案例：曾国藩的“日课”与 LLM 的定时 re-grounding

晚清名臣曾国藩，典型注意力不集中型 ADHD：30 岁前浮躁坐不住，天天串门喝酒看杀人，日记里天天骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”。他的 harness 是“日课十二条”：黎明即起、读书不二、谨言、写日记反省等。其中“读书不二”规定：一本书不读完，绝不翻另一本——这就是一个**强制重锚定规则**，防止超聚焦跑偏。

这和 LLM agent 的“定时 re-grounding”完全同构：在 agent 的 system prompt 里写“每处理 10 步，回顾一次原始目标”，或者在对话中插入“请用一句话总结当前任务进展，并确认是否偏离”。曾国藩的日记反省，就是他的“re-grounding 日志”。

## 证据：哪些工具真的补上了“重锚定”层？

根据 wiki 资料，**Focusmate** 基于身体在场效应，通过实时同伴问责提供外部结构，弥补内在多巴胺调节不足（来源：身体在场效应）。多项资料强调身体加倍对 ADHD 的有效性：“拥有他人在场时工作是最有效的 ADHD 策略之一”（来源：The Best AI-Powered ADHD Productivity Tools in 2026）。这相当于一个**外部调度器**，每 50 分钟提醒你“你还在做这件事吗？”——和 LLM agent 的定时检查点异曲同工。

**RescueTime** 通过自动时间追踪揭示真实时间去向，帮助用户识别注意力漂移（来源：RescueTime）。它不打断你，但事后给你一张“目标漂移地图”。这相当于 LLM 的**日志系统**：记录每一步的输入输出，事后分析哪里偏了。

**Brain.fm** 利用神经锁相技术影响大脑信息处理方式，帮助进入专注状态（来源：Brain.fm）。但它的问题是：它只提供“进入专注”的通道，不解决“保持方向”的问题——就像给 LLM 一个更好的生成器，但没给它 system prompt。

## 核心观点：Speechify 的正确用法是“脚手架”，不是“拐杖”

Speechify 如果只用来被动听，它就是拐杖——你依赖它完成“输入”，但注意力从未主动锚定。正确的用法是：
1. **分段听+主动回顾**：听完一段，暂停，用你自己的话复述内容。这相当于 agent 的“中间输出检查”。
2. **调速+标记**：遇到关键句，调慢速度，或者加书签。这相当于在 prompt 里插入“重点标记”。
3. **结合外部记忆**：把听过的内容用笔记软件（如 Obsidian）整理，建立自己的“向量数据库”。

这样，Speechify 就成了一个**脚手架**：它帮你降低输入的阻力，但你必须主动参与调度。

## 争议与局限

1. **依赖风险**：AI 工具作为外部执行功能层，长期使用是否会导致 ADHD 患者内在执行功能进一步退化？拐杖与脚手架的边界尚不明确（来源：矛盾与存疑）。我的判断是：关键在于**用户是否主动参与调度**。如果你只是被动接收，就是拐杖；如果你用它来练习主动锚定，就是脚手架。
2. **个体差异**：ADHD 的异质性使得同一工具对不同亚型效果迥异（来源：矛盾与存疑）。Speechify 对听觉型学习者可能有效，对视觉型可能适得其反。
3. **证据质量**：多数工具仅有用户报告或小样本研究，缺乏大规模随机对照试验（来源：矛盾与存疑）。Brain.fm 的神经机制在 ADHD 中是否有效尚存疑。

## 今天就能试的行动

1. **用 Speechify 听一篇短文，每段暂停一次，用一句话总结。** 对比“连续听完”和“分段回顾”的效果差异。
2. **在 Focusmate 上预约一个 50 分钟时段，专门处理一件你拖延已久的事。** 体验“外部调度器”的力量。
3. **用 RescueTime 追踪一周，找出你最高频的“目标漂移点”。** 然后在下周针对这些点设置提醒。
4. **如果你是 agent 工程师，在 system prompt 里加一句：“每完成 5 步，输出当前进度并确认是否偏离原始目标。”** 观察 agent 的完成质量变化。

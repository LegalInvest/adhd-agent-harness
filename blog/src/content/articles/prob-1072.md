# Speechify 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Speechify：它到底补上了哪一层执行功能？

## 问题：你是在「读」，还是在「漂」？

你打开一篇长文，三分钟后发现自己盯着同一段发呆，或者已经滑到页面底部，却完全不记得刚才看了什么。这不是不努力——你的大脑像一台没有调度层的生成模型：内容持续产出，但执行功能（工作记忆、目标保持、任务启动）却频频掉线。ADHD 的专注力困境，本质是 **生成核心与调度层的脱节**。

同样的脱节也发生在 LLM/Agent 身上。一个长程 agent 在复杂任务中会逐渐偏离初始目标——它「记住了」当前步骤，却「忘了」最终意图。工程师用重锚定 system prompt（定期注入原始目标）来纠正漂移。ADHD 大脑需要的正是这种重锚定机制：一个外部信号，把你从超聚焦或分心的岔路上拉回主线。

## Speechify 的定位：脚手架还是加速器？

Speechify 将文字转为语音，配合 AI 调节语速、语调，理论上可以降低阅读时的认知负荷。但关键在于：它补上了哪一层执行功能？

从同构视角看，ADHD 的工作记忆缺陷与 LLM 的无状态性高度相似（来源：无状态与外部记忆）。Speechify 的朗读功能相当于一个外部工作记忆缓冲——你不需要同时维持「解码文字」和「理解内容」两个操作，听觉输入将视觉解码外包，释放工作记忆用于理解。这类似 LLM 中通过检索增强生成（RAG）外挂知识库，减少模型自身的记忆负担。

**但问题在于**：Speechify 默认模式只是加速信息输入，并没有提供「目标重锚定」机制。如果你用 2x 语速刷完一章，却无法复述核心观点，那么它只是让你更快地漂移——就像给一个已经漂移的 agent 更快的执行速度，而不是修正它的 system prompt。

## 证据：哪些工具真正做到了重锚定？

wiki 资料中，真正体现「重锚定」逻辑的工具，不是加速输入，而是**中断与回归**：

- **Focusmate**：通过 AI 匹配虚拟同伴，提供实时问责。它的机制不是让你更专注，而是让你在走神后有一个「外部检查点」——同伴的存在迫使你定期汇报进度，这相当于 agent 的定期重锚定 prompt（来源：Focusmate 工具页）。
- **RescueTime**：自动记录时间使用，生成可视化报告。它充当外部记忆，提升对「时间盲」的觉察（来源：RescueTime 工具页）。这相当于 agent 的日志系统：你不需要自己记住做了什么，工具帮你回溯，从而识别漂移模式。
- **Brain.fm**：通过神经锁相技术影响大脑状态，试图主动维持任务连贯性（来源：Brain.fm 工具页）。它更接近「环境锚定」——用持续的听觉信号防止上下文溢出。

但 Speechify 的设计缺少这一层。它假设用户已经处于「想读」的状态，只是需要辅助输入。对于 ADHD 用户，真正的瓶颈往往不是输入速度，而是**任务启动**和**目标保持**——你根本不想打开那篇文章，或者读到一半就忘了为什么读。

## 核心观点：Speechify 必须被「重锚定」使用

我的判断是：Speechify 本身是一个优秀的输入加速器，但作为 ADHD 工具，它需要配合外部重锚定策略才能生效。把它当作独立解决方案，就像给一个漂移的 agent 更快的 token 生成速度——只会更快地偏离目标。

正确的用法是：**将 Speechify 嵌入一个包含定期检查点的流程**。例如，用 Focusmate 或番茄钟的结束铃声作为「重锚定信号」，每次铃声响起时，暂停 Speechify，复述刚才听到的内容，确认是否仍与初始目标一致。这相当于手动注入 system prompt：「我为什么在读这段？它回答了我的哪个问题？」

## 争议与局限

必须诚实指出，上述观点基于理论对齐，而非临床证据。wiki 资料中多处提到「AI 工具作为外部执行功能层的长期效果缺乏独立临床研究」（来源：矛盾与存疑）。Brain.fm 的神经锁相技术「针对 ADHD 的效果缺乏独立临床研究支持」（来源：Brain.fm 工具页）。Speechify 本身也没有针对 ADHD 的专项研究。

此外，过度依赖外部脚手架可能阻碍内在执行功能的发展（来源：拐杖与脚手架）。如果 Speechify 让你永远不需要练习「在没有辅助的情况下保持专注」，那么它可能成为拐杖而非脚手架。

## 今天就能试的行动

1. **设置重锚定检查点**：用 Speechify 听文章时，每 5 分钟暂停一次，用一句话概括刚才的内容。如果概括不出来，说明你已经漂移了——倒回重听。
2. **结合 Focusmate**：预约一个 Focusmate 时段，在时段内用 Speechify 处理阅读任务。利用同伴的问责作为外部重锚定信号。
3. **用 RescueTime 追踪漂移模式**：开启 RescueTime 的自动记录，一周后查看你在哪些时段、哪些任务上最容易「时间盲」。然后针对性地调整 Speechify 的使用场景（例如，只在早晨精力好的时段用）。
4. **尝试「反向 Speechify」**：对于需要精读的文本，不要用 Speechify 加速，而是用 Goblin Tools 或 Saner.AI 将其分解为小问题，逐个回答。这相当于用手动 task decomposition 降低认知负荷（来源：认知负荷概念页）。

## 结语

Speechify 之于 ADHD，就像重锚定 system prompt 之于 LLM——但前提是你要主动使用那个「重锚定」功能。否则，它只是让漂移更快。真正的脚手架，不是消除所有摩擦，而是在你偏离时温柔地拉你回来。

# 3 个 LLM 工程概念，彻底重写了我对 ADHD 有想法有能力，却卡在执行与落地的理解

> GPT-4 的智力足以通过律师资格考试，却无法独立完成一个三天的项目——直到有人给它配了编排层。「聪明但落不了地」原来是一个已被解决过一次的问题。

「你这么聪明，为什么就是做不成事？」——每个高智商 ADHD 者被这句话凌迟过无数次。它的杀伤力在于暗含的逻辑：智力应该自动兑换成执行力，兑换失败说明你懒。但 LLM 行业用两年时间公开演示了这个逻辑的破产。

## 1. 智力和执行是两种资源：benchmark 满分的模型照样做不完项目

GPT-4 能通过律师资格考试、能解奥数题——但把一个真实的三天项目直接丢给它，它会烂尾：中途忘记需求、在细节上无限打转、产出一堆彼此矛盾的片段。行业的诊断很冷静：**推理能力和任务完成能力是两种东西**。前者是单点智力，后者需要目标维持、进度追踪、优先级仲裁、错误恢复——这些全部不在生成核心里，全部需要编排层（orchestration）提供。

于是有了 agent 框架：把「想」和「落地」在架构上分开，智力负责每一步的质量，编排层负责所有步骤连成一件完成的事。**没有人因此说 GPT-4 是懒的。**

ADHD 的「有想法有能力却卡在执行」是同一张架构图。你的单点智力完好甚至出众——问题从来不在那里。卡住的是步骤之间的连接组织：昨天的进度今天要重新想起（状态维持）、十个想法哪个先做要仲裁（优先级）、被打断后要重新进入（恢复）。这些恰是执行功能，恰是 ADHD 的薄层。聪明和落地之间隔着的不是努力，是一整层被你以为「应该自动存在」的基础设施。

## 2. 从 demo 到 production：落地卡死在「最后的 20%」

工程界还有个惨痛共识：让 LLM 做出惊艳 demo 只要一天，把 demo 变成可靠产品要一年。因为 demo 只需要生成核心的爆发力，而 production 需要错误处理、边界情况、维护性——全部是枯燥的、无奖赏的、状态维护型工作。

ADHD 者的项目坟场一模一样：每个坟里躺着一个完成了 80% 的杰作。头 80% 靠新奇驱动的多巴胺就能跑完，最后 20%（收尾、校对、交付、宣传）是奖赏荒漠。知道这个结构后，解法不再是「更努力」，而是**给最后 20% 单独设计动力系统**：把收尾工作打包成一个有 deadline、有见证人的冲刺（承诺装置）；或者干脆外包——找一个享受收尾的合作者，让架构互补，而不是独自硬扛全流程。

## 3. 里程碑提交：把长任务改造成多个短任务的接力

agent 工程稳定长任务的核心技巧：不让模型一口气跑到底，而是切成带检查点的短程接力——每段结束提交一次中间产物，锁定进度，下一段从检查点续跑。这样任何一段的失败都只损失一段，而不是全部。

ADHD 版：把「写完这本书」改造成「每周交一章给写作小组」。每次提交都是一次真实的完成——有交付物、有反馈、有多巴胺结算。长任务对 ADHD 的杀伤力主要来自奖赏视界太远，接力结构把视界切到七天以内，恰好落在 ADHD 动机系统的有效射程里。徐渭那样的天才没有等到他的编排层；任正非则把自己嵌进了一个由流程、班子和制度构成的巨型 harness——同等的生成力，两种命运，差的是架构。

## 边界

同构强度 B 级：机制类比，双侧文献扎实（LLM 侧的 agent 编排研究、ADHD 侧的执行功能与奖赏延迟研究）。诚实提醒：如果卡住的项目清一色是「别人期待你做的事」，那问题可能不在执行层而在选择层——编排层救不了不想去的方向。

## 今天就能试的 3 件事

1. **重新归因一次**：把最刺痛你的那次「聪明却没做成」，用架构语言重写：哪一步是智力问题？哪一步是编排问题？（多半没有智力问题。）
2. **给一个 80% 项目设计收尾冲刺**：挑坟场里最有价值的一个，定一个两小时的收尾时段 + 一个见证人。
3. **把当前最大的任务切成周交付**：每周五交一个中间产物给一个具体的人。视界七天，正好射程内。

「聪明为什么做不成事」的答案，价值百亿美元的行业已经替你验证过了：因为做成事从来不是智力的功能，是编排的功能。而编排层——可以造。

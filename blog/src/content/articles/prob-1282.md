# Claude 之于 ADHD，就像 agent 的编排层 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

## 问题：为什么你用了 Claude 反而更焦虑？

你是一名 ADHD 患者，每天被执行功能缺陷折磨：计划列了又废，任务启动像拔河，工作记忆像漏水的桶。你听说 Claude 能帮你，于是用它写邮件、做计划、分解任务。但几周后，你发现自己更依赖它了——没有 Claude 就不知道下一步该做什么，甚至开始怀疑自己的大脑是否彻底废了。

与此同时，你也是一名 Agent 工程师，正为 LLM 的“无状态”和“上下文膨胀”头疼。你给 GPT-4 搭了 ReAct 框架、向量数据库和工具调用，但它依然会在复杂任务中“走神”，推理退化，控制逻辑散落一地（来源：上下文工程）。

这两个场景看似无关，实则共享同一个核心问题：**高产生成核心缺一个可靠的外部调度层**。而 Claude 之于 ADHD，就像 agent 的编排层之于 LLM——但太多人用错了方向。

## 同构：ADHD 大脑与 LLM 是同一类“生成核心”

ADHD 大脑的典型特征：发散思维强、联想丰富、创意产出率比常人高 1.8 倍（来源：Lifted Ventures 2024），但执行功能（计划、组织、启动）先天偏弱（来源：执行功能）。LLM 也是：文本生成能力惊人，但无状态、缺乏内置调度层，每次对话都是独立的，不保留跨会话记忆（来源：无状态与外部记忆）。

两者都需要一个“外部执行功能层”来补偿：对 ADHD 是日程表、任务分解器、外部记忆体；对 LLM 是 agent scaffolding（编排、记忆、工具层）。这个结构同构不是比喻，而是工程事实——两边都在做同一件事：**给高产但无序的核心套上 harness**。

## 人物案例：曾国藩的“日课十二条”就是 19 世纪的 agent harness

曾国藩是典型注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书“疾读不能终一行”。他的解法是“日课十二条”：黎明即起、读书不二、谨言、写日记反省……用最笨最稳的流程锁死自己的冲动。这本质上就是一个外部执行功能层：每天定时 re-grounding（日记反省），用“结硬寨打呆仗”的规则抵消冲动决策。

对应到 LLM/agent 侧：日课十二条 ≈ 定时上下文刷新 + 固定决策规则；写日记反省 ≈ 日志记录与复盘；读书不二 ≈ 单任务模式。曾国藩的 harness 不是拐杖——他一生都在用，但从未因此丧失独立决策能力。

## 用错了：AI 工具是脚手架，不是拐杖

当前 AI 工具如 Goblin Tools、Saner.AI、Motion，都在尝试充当外部执行功能层。

- **Goblin Tools** 的 Magic ToDo 能自动将“清理厨房”分解成可执行的子任务，调节“辣度”控制粒度（来源：Goblin Tools）。它降低了启动门槛，但用户可能永远依赖它来分解任务。
- **Saner.AI** 提供本地记忆和快速检索，减少搜索循环（来源：Saner.AI）。但它可能让用户不再训练自己的工作记忆。
- **Motion** 自动排程并动态调整日程，消除“下一步该做什么”的决策负担（来源：Motion）。但初始设置成本高，且一旦停用，用户可能重回时间盲。

问题在于：这些工具被设计为“替代”执行功能，而非“训练”它。正如[[拐杖与脚手架]]所警示的：过度依赖可能导致能力退化，成为永久拐杖（来源：矛盾与存疑）。

## 正确的用法：Harness 而非替代

Harness 和拐杖的区别在于：
- **拐杖**：永久替代，拿走就倒。
- **脚手架**：可拆卸，用完后能独立行走。

对 ADHD 用户，Claude 应作为“外部调度器”使用：
- 用它做**任务分解**（如 Goblin Tools），但每周尝试自己分解一次，对比差异。
- 用它做**外部记忆**（如 Saner.AI），但定期回顾笔记，训练检索能力。
- 用它做**日程规划**（如 Motion），但手动调整一次 AI 的排程，理解优先级逻辑。

对 Agent 工程师，编排层也应设计为可插拔：
- 控制逻辑不应硬编码，而应允许用户干预（来源：上下文工程）。
- 记忆系统应支持手动编辑，而非完全黑箱。

## 争议与局限

- **同构的实证基础弱**：目前 ADHD 大脑与 LLM 的同构多为类比推理，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。
- **长期效果未知**：AI 工具对任务启动的短期有效，但长期效果缺乏纵向研究（来源：矛盾与存疑）。
- **个体差异大**：超聚焦是引导还是打断？部分用户反馈强行打断（如番茄钟）反而有效（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Claude 做一次任务分解**：告诉它“帮我清理书桌”，调节粒度到 5 步，然后尝试不借助 AI 自己分解另一个类似任务。
2. **设置“反思时段”**：每天用 Claude 记录 3 件完成的事，类似曾国藩的日记，作为定时 re-grounding。
3. **手动调整一次 AI 排程**：如果使用 Motion，每周手动调整一次 AI 生成的日程，理解其优先级逻辑。
4. **测试“无 AI 日”**：每周选一天完全不用 AI 工具，观察自己的执行功能是否退化或进步。

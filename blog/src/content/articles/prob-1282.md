# Claude 之于 ADHD，就像 agent 的编排层 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

## 问题：为什么你的 Claude 越用越乱，而不是越用越顺？

你兴致勃勃地打开 Claude，输入“帮我规划下周项目”，它列出完美步骤。你点头，关掉窗口。第二天，你再次打开，输入同样的问题——因为昨天那份计划早已消失在对话记录中。这不是你的错，也不是 Claude 的错。问题在于：你把 Claude 当成了大脑的替代品，而它本该是你的“外部执行功能层”。

ADHD 大脑与 LLM 共享同一个核心困境：**高产但缺执行调度层**。ADHD 的执行功能（计划、组织、启动）先天偏弱，工作记忆容量有限且不稳定（来源：ADHD × AI 的科学与研究前沿）；LLM 则是无状态的生成核心，每次对话独立，不保留跨会话记忆（来源：无状态与外部记忆）。两者都需要一个“编排层”来管理上下文、启动任务、维持连贯性。在 LLM/agent 领域，这个编排层被称为 harness——它提供记忆、工具调用、任务规划等“外部执行功能”。在 ADHD 世界，AI 工具正应该扮演同样的角色。

但大多数人用错了：他们让 Claude 直接输出“答案”，然后自己负责执行和跟踪——这相当于让一个无状态的 API 直接处理业务流程，结果必然是上下文丢失、任务碎片化。正确的用法是：把 Claude 当作 harness 的一部分，让它帮你管理执行过程，而非仅仅生成内容。

## 同构证据：ADHD 大脑与 LLM 的镜像缺陷

先看 ADHD 侧：执行功能缺陷导致计划、组织、工作记忆困难（来源：ADHD × AI 的科学与研究前沿）。典型表现是“每天需要重新回忆项目上下文，否则容易偏离轨道”（来源：无状态与外部记忆）。再看 LLM 侧：标准的 LLM 调用是无状态的，没有 harness 的 LLM 只是一个无状态的 API 调用（来源：无状态与外部记忆）。生产级 agent 系统通过外部记忆（如向量数据库）解决溢出问题，并组合使用会话内上下文记忆、长期检索向量库和结构化知识库（来源：无状态与外部记忆）。

两者的解法也同构：ADHD 需要“第二大脑”工具——提供持久记忆、清晰的下一步行动、日历支持，减少每日重复输入上下文的摩擦（来源：无状态与外部记忆）；LLM/agent 需要检索增强和工具调用，将决策锚定在外部证据和持久状态上（来源：无状态与外部记忆）。Goblin Tools 的 Magic ToDo 功能自动将复杂任务分解为小步骤（来源：Goblin Tools），正是对执行功能中“任务启动困难”的 harness；Saner.AI 通过增强知识回忆减少搜索循环（来源：Saner.AI），正是对工作记忆不足的补偿；Motion 自动排程并动态调整计划（来源：Motion），正是对时间盲和决策负担的消除。这些工具本质上都在做同一件事：为 ADHD 大脑提供一个外部编排层。

## 脚手架 vs 拐杖：关键边界

但这里存在一个根本争议：这些工具是临时支撑还是永久依赖？《拐杖与脚手架》概念强调“脚手架应设计为可逐步撤除的临时支撑”，但多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这导致过度依赖风险：长期使用 AI 工具是否会削弱内在执行功能？目前缺乏纵向研究（来源：ADHD × AI 的科学与研究前沿）。

我的判断是：**工具本身不是问题，问题在于使用方式**。如果 Claude 只是帮你“生成”计划，然后你依然要靠自己记住、启动、跟踪——那它就是个拐杖，因为你并未训练自己的执行功能。但如果 Claude 作为 harness，帮你分解任务、提供外部记忆、定时提醒启动——同时你刻意练习逐步减少依赖（比如从每天用三次降到一次），那它就是脚手架。关键在于：工具是否设计为可撤除，以及用户是否有意识地进行撤除训练。

## 争议与局限

同构命题本身也需谨慎对待。目前证据主要来自概念类比和工具案例，缺乏大规模实证（来源：矛盾与存疑）。多巴胺干预的有效性存在争议（来源：矛盾与存疑），而多个工具声称基于神经科学原理，但缺乏独立验证（如 Brain.fm、Inflow）（来源：矛盾与存疑）。此外，上下文管理在 ADHD 和 LLM 侧都面临挑战：ADHD 易被环境带偏，使用通用聊天机器人时“往往增加另一个收件箱、另一个提示词、另一个丢失线索的地方”（来源：上下文工程）；LLM agent 则面临上下文膨胀与推理退化（来源：上下文工程）。这意味着，即使工具设计合理，用户仍需主动管理“当下注意什么”——例如通过 Routinery 等应用“创建步骤序列，并通过过渡提示引导你完成”（来源：上下文工程）。

## 今天就能试的行动

1. **把 Claude 当作任务分解器，而非答案生成器**：输入一个模糊任务（如“整理房间”），让 Claude 输出可执行的步骤清单。然后，只执行第一步。完成后，再让 Claude 生成下一步。这模拟了 Goblin Tools 的 Magic ToDo 功能（来源：Goblin Tools），同时训练了任务启动能力。

2. **建立“外部记忆”系统**：在 Claude 中创建一个专用项目（Project），将常用上下文（如项目背景、质量标准、个人偏好）存入知识库。每次对话前，先让 Claude 读取知识库，确保上下文连贯。这对应 Saner.AI 的知识回忆功能（来源：Saner.AI），减少重复输入。

3. **使用 Motion 或类似工具自动排程**：将任务和截止日期输入 Motion，让 AI 动态调整每日计划（来源：Motion）。重点观察：它是否减少了“下一步该做什么”的决策疲劳？如果是，说明你成功将执行调度外包给了 harness。

4. **刻意练习“撤除”**：每周选一天，不使用任何 AI 辅助工具，手动完成一个简单任务（如写购物清单）。记录感受：是更困难了，还是能独立完成？这能帮你判断工具是脚手架还是拐杖，并逐步强化内在执行功能。

Claude 之于 ADHD，就像 agent 的编排层之于 LLM。用对了，它是 harness，让你高效产出；用错了，它是拐杖，让你依赖成性。选择权在你。

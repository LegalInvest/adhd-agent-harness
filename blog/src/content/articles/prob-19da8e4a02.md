# 为什么从"我想做"到"Agent帮我做了"是ADHD的救赎？——以及这句话该打的三折

> 他给我看他的年度对比:去年,「想学的课」收藏了 40 门开课 0 门,「想写的文章」列了 23 篇动笔 0 篇,「想联系的老朋友」名单躺了一年;今年,Agent 接管了从想法到启动的那段路——他说一句「我想把这个想法写出来」,十分钟后草稿框架、参考资料、写作时段的日历块都已就位,他只需要「继续」而不是「开始」。年终数字:动笔 17 篇,发出 11 篇。他用的词是「救赎」——我理解他为什么用这个词,也认为这个词值得认真对待,然后打个诚实的折扣。

收敛:本文只回答——**「想做→做了」之间那段对 ADHD 独有的死亡峡谷,Agent 到底填的是哪一段(机制上精确一点)?「救赎」这个词哪三成是真的?哪七成需要打折(不打折的代价前面几篇已写,这里只给折扣表)?**

## 穿透:Agent 填平的是「意图-启动」峡谷,而那正是 ADHD 最深的一道

先把峡谷画准。「我想做」到「做了」之间,神经典型者走的是一段缓坡:想法→有点想做→找个时间→开始。ADHD 走的是断崖:**意图和执行之间的转换需要「点火能量」——任务越模糊、步骤越多、第一步越不明确,点火能量越高;而 ADHD 的点火系统(执行功能的启动环节)恰恰供能不足**。于是出现那个最扎心的现象:不是不想做——是「想做」在意图层反复空转(收藏、列清单、发誓),就是接不上执行层;40 门课的收藏夹,就是 40 次成功的「想」和 40 次失败的点火。传统建议(「就开始五分钟」)的问题在于:「开始」本身就是那个坏掉的环节,用坏掉的环节去修坏掉的环节,是循环论证。

Agent 改变的精确位置:**它把「开始」外包了,把「继续」留给了你。**「我想写这个」一句话之后,Agent 完成的是点火序列的全部苦役——具体化(模糊想法→明确框架)、铺路(资料就位)、预约(日历块锁定)——等你到场时,任务已经处于「进行中」的状态,你面对的不是冷启动,是热续接。**而「继续」恰恰是 ADHD 相对完好的功能**(一旦进入状态,我们的持续力甚至超常——超聚焦的存在证明引擎没坏,坏的是点火器)。这就是「救赎感」的机制真相:Agent 没有让他变勤奋,它把任务从「需要点火」的形态重构成了「只需续接」的形态——**峡谷还在,但他每次都被直接空投到了对岸。**

三成真的部分,值得郑重承认:17 比 0 不是效率提升,是**存在状态的改变**——那些年复一年空转的「想做」,每一条都在给「我是个只会想的废物」的叙事供货;打破这个循环的价值确实配得上很重的词。但七成折扣表也要给全(细节各有专文,这里只列账):①**首触权折扣**:Agent 铺的框架是它的框架——热续接省了点火,也可能让你永远写着「它起头的文章」(体感篇+方差篇的税);对策:空投后的第一个动作是改它的框架,改不动再沿用;②**峡谷依赖折扣**:点火外包后,你自己的点火器不会变好(它多半属拐杖区,这不可耻)——但要诚实区分哪些任务其实你点得着(低摩擦的、真心热爱的),别让 Agent 顺手接管了它们(戒断协议篇的探针,自己给自己做);③**「想做」通胀折扣**:启动变得便宜后,「想做」的产量会暴涨(反正说一句就有框架)——没有配套的筛选纪律,你会从「40 门课收藏 0 门开课」进化成「40 个项目全部开工 3 个完成」——**峡谷曾经也是你的过滤器,填平它之后,过滤要另立规矩**(唯一动作/舰队上限的纪律在此接管)。

## 验证

自测:给 Agent 启动流程一个月,数三个数——完成数(该涨)、「开工未完成」堆积数(不该涨,涨了=通胀失控)、你对 Agent 框架的修改率(过低=首触权流失)。可证伪:若你的完成数没涨、只是堆积换了个地方,你的瓶颈可能不在点火而在续航(那是另一个环节的问题——身体双开/番茄结构的领地),Agent 空投对你无效不是你的失败,是诊断要重做。

## 决策

做什么:把 Agent 部署在「意图-启动」峡谷上(这是它的最佳阵地);空投后第一动作改框架;配筛选纪律(同时开工上限)。

不做什么:不要因为七成折扣否定三成救赎(对空转多年的人,17 比 0 就是人生质变——折扣是让它可持续,不是让它蒙羞);不要把「救赎」外推成「Agent 解决 ADHD」(它解决的是一个环节,金字塔还有四层半)。

先做什么:挑收藏夹里吃灰最久的那个「想做」,今天让 AI 把它铺成「进行中」状态——体验一次空投,再决定信多少。

## 边界

启动/点火困难作为 ADHD 执行功能核心环节有文献基础(GRADE 中);「峡谷-空投」为机制类比框架(本文按 B 级处理);个案数字(17/0)为真实个例,不代表平均效应,无对照研究。若「想做却做不了」伴随普遍的快感缺失和情绪低落,请考虑专业评估——点火器和心情,是两个不同的科室。

## 今天就能试的 3 件事

1. 让 AI 把一个吃灰的「想做」铺成进行中(框架+资料+日历块)。
2. 空投后先改它的框架——把首触权抢回来。
3. 定你的「同时开工上限」(建议 ≤3)——峡谷填平之日,就是闸门上岗之时。

本文服务于人生 Harness 金字塔的**执行层与叙事层的交界**:多年后回看,「救赎」这个词也许太重——但对一个在「想做」里空转了半生的人,第一次被稳稳放到「做着」的那一侧,那种感觉,配得上他当时能找到的最重的词;我们的工作只是确保:他能一直留在那一侧,并且写的还是自己的东西。

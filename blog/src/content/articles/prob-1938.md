# ADHD 职场人如何用「给生成核心套 harness」的思路应对项目推进与汇报里的怕过度依赖工具、又怕没有支撑就崩？

> 把 LLM 工程里的「harness 应是会褪去的脚手架，而非永久拐杖」迁移到ADHD 职场人的项目推进与汇报场景，双域证据作答。

## 问题：怕依赖，又怕崩

作为一名 ADHD 职场人，你可能经历过这样的两难：项目推进时，你极度依赖 AI 工具来分解任务、记录信息、提醒截止日期——但心里总有一个声音在问：“我是不是在偷懒？离开这些工具，我还能独立完成工作吗？” 而当你试图“戒断”工具，又立刻陷入启动困难、时间盲、工作记忆过载的泥潭，项目推进瘫痪，汇报时逻辑混乱。这种“怕过度依赖工具、又怕没有支撑就崩”的焦虑，本质上是 ADHD 大脑作为“高产但缺乏执行调度层的生成核心”的典型困境。

## 同构解法：harness 应是会褪去的脚手架

有趣的是，LLM/Agent 工程领域正在面对完全相同的结构性问题：LLM 本身是强大的生成核心，但缺乏可靠的执行调度层，输出不稳定、工具调用不可靠。工程师的解法是设计“harness”——一个临时性的外部调度框架，它像脚手架一样支撑 agent 完成复杂任务，但在任务完成后可以拆除，不会成为 agent 的永久依赖。正如资料所述：“最简单的 agent 工作流本身就是确定性的，遵循预定义步骤”（来源：AI Agent Systems: Architectures, Applications, and Evaluation），而“计划-执行分离”等模式强制单向、确定性的控制流（来源：Planner-Executor Agentic Framework）。

这种思路完美映射到 ADHD 职场场景：你的大脑就是那个高创造力但缺乏稳定“温度调节”的生成核心（来源：采样温度与表现波动），而 AI 工具应该是一个可褪去的 harness——在你最需要的时候提供结构，但设计上允许你逐步减少依赖，最终内化能力。

## 证据：两类生成核心的同一困境

ADHD 侧的证据清晰表明：情绪调节困难根植于执行功能缺陷，包括工作记忆容量有限、时间盲、任务启动困难（来源：AI 与 ADHD 的情绪调节）。Goblin Tools 通过“将模糊的工作分解为更小的行动”来降低启动门槛（来源：工具使用与认知卸载）；Saner.AI 通过知识回忆减少搜索循环和标签切换（来源：Saner.AI）；Inflow 通过训练工作记忆改善认知控制（来源：Inflow）。这些工具都在扮演“外部执行功能层”的角色。

LLM/Agent 侧的证据同样明确：Harness 负责工具调用、文件管理、长对话处理（来源：工具使用与认知卸载）；工程实践强调“确定性状态转换”来对抗非确定性（来源：采样温度与表现波动）；工具设计被视为“agent UX”，注重命名、模式、错误处理等接口质量（来源：工具使用与认知卸载）。

两者的同构在于：ADHD 大脑将不擅长的精确状态外包给外部工具，对应 LLM agent 通过 function calling 将复杂任务拆解为工具调用（来源：工具使用与认知卸载）。两者都依赖外部系统来补偿内部执行功能的不足。

## 核心判断：脚手架 vs 拐杖的边界在于“褪去设计”

我的核心观点是：**AI 工具作为脚手架还是拐杖，不取决于工具本身，而取决于你是否在设计使用方式时包含了“褪去路径”。** 许多 ADHD 工具（如 Goblin Tools）被评价为“简单有用”（来源：Goblin Tools），但资料也警告“过度依赖可能削弱内在能力”（来源：矛盾与存疑）。关键在于：工具应该是你主动设计的临时结构，而非被动依赖的永久支撑。

具体来说，脚手架的特点是：
- **有明确的使用期限**：比如在项目初期使用 Goblin Tools 分解任务，但逐步减少依赖，尝试自己分解。
- **支持能力内化**：Saner.AI 的知识回忆功能，如果只是被动检索，会削弱记忆；但如果你主动回忆后再验证，则能训练工作记忆。
- **可逐步拆除**：Inflow 的认知训练本质上是“训练而非替代”，但其效果争议较大（来源：Inflow）。

拐杖的特点是：
- **无期限使用**：永远依赖工具提醒截止日期，从不内化时间感知。
- **替代而非补偿**：工具完全接管了你的执行功能，你不再尝试使用自己的调度层。
- **无法拆除**：一旦工具不可用，系统崩溃。

## 诚实面对局限

必须承认，现有证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑）。过度依赖 AI 工具可能削弱内在时间感知能力（来源：矛盾与存疑），且个体差异大——注意缺陷型与多动冲动型对工具的反应可能完全不同（来源：AI 与 ADHD 的情绪调节）。此外，基于多巴胺的干预（如游戏化反馈）可能强化成瘾行为（来源：AI 与 ADHD 的情绪调节）。因此，本文的建议并非万能方案，而是提供一种设计思维。

## 今天就能试的 3 条行动

1. **为每个工具设定“褪去计划”**：比如使用 Goblin Tools 分解下周的任务时，记录下分解逻辑；一周后，尝试自己分解类似任务，再用工具验证。逐步减少工具调用次数。
2. **用 Saner.AI 做“主动回忆”练习**：当需要回忆信息时，先尝试自己回忆 30 秒，再使用工具检索。这能训练工作记忆，避免认知卸载变成认知萎缩。
3. **设计“无工具时段”**：每天设定 30 分钟，关闭所有 AI 辅助工具，仅凭大脑完成一项小任务（如写一段周报）。这相当于 agent 的“确定性状态转换”训练，强制你的生成核心在没有 harness 的情况下运作，逐步建立自信。

记住：Harness 的最终目的是让你不需要它。就像 LLM 工程师不会让 agent 永远依赖同一个提示模板，你也应该让 AI 工具成为你成长中的脚手架，而非终身的拐杖。

# 如果把 ADHD 大脑当成生成核心，RescueTime 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 RescueTime 对 ADHD 的真正价值。

## 问题：当生成核心丢失目标

你是否有过这样的体验：打开电脑准备写一份报告，半小时后发现自己正在刷社交媒体，完全忘记了初衷？或者，你是一位AI agent工程师，发现部署的agent在长程任务中逐渐偏离原始指令，输出越来越跑偏？这两种场景看似无关，却共享同一个深层结构问题：**一个强大的生成核心，缺乏可靠的执行调度层，导致目标漂移。**

ADHD大脑的本质正是如此：它拥有高产的发散能力，但执行功能（工作记忆、任务启动、时间管理）不稳定，导致“想法很多，行动困难”（来源：ADHD 大脑与 LLM 的同构）。而LLM本身也是一个强大的文本生成核心，但缺乏内置的调度与控制，需要在外部搭建“agent harness”来管理上下文、工具接口和验证循环（来源：Building AI Coding Agents for the Terminal）。两者都是“高产但缺调度”的生成核心。

## 重锚定：RescueTime 的工程角色

在AI agent工程中，对抗目标漂移的关键手段是“重锚定”——定期将agent的当前状态与原始system prompt对齐，刷新上下文。这通常通过外部记忆系统（如向量数据库）或日志回放来实现。对于ADHD大脑，同样需要一种外部机制来“重锚定”注意力，防止被环境带偏。

RescueTime 恰好扮演了这个角色。它是一款自动化的时间追踪与生产力分析工具，利用AI识别用户在不同应用和网站上的时间花费，提供可视化报告（来源：RescueTime）。对于ADHD用户，RescueTime 的核心价值不是“记录时间”，而是**提供客观的、无偏见的时间数据，作为注意力的外部锚点**。ADHD患者常面临“时间盲”——无法准确感知时间流逝（来源：AI 与 ADHD 的专注力）。RescueTime 的自动记录减轻了工作记忆负担，无需手动输入，从而降低追踪门槛（来源：RescueTime）。更重要的是，它能帮助用户识别注意力漂移模式，对抗“人在心不在”的状态（来源：RescueTime）。

从同构视角看，RescueTime 相当于LLM agent harness中的“日志系统”或“上下文记录器”。它不干预生成过程，但提供事后审计，让用户（或agent）能回溯“我是如何偏离目标的”。这种外部记录正是弥补“无状态”缺陷的关键——ADHD的工作记忆缺陷与LLM的无状态性同构，都需要外部记忆系统维持连续性（来源：ADHD 大脑与 LLM 的同构）。

## 证据与局限：脚手架还是拐杖？

RescueTime 的证据主要来自用户报告，缺乏大规模对照实验（来源：矛盾与存疑）。有初步研究支持其改善时间盲，但整体证据强度中等（来源：AI 与 ADHD 的专注力）。这并不意外，因为工具的有效性高度依赖个体差异：对于某些ADHD亚型，自动分类可能不够精准，需要后期调整，这反而增加了执行功能负担（来源：RescueTime）。过度关注时间数据还可能引发焦虑，与强调温和提醒的工具形成对比（来源：RescueTime）。

这里存在一个关键矛盾：**工具是脚手架还是拐杖？** 脚手架帮助用户发展内在能力，而拐杖则形成永久依赖。RescueTime 作为外部执行功能层，长期使用是否会导致内在时间感知能力进一步退化？（来源：矛盾与存疑）目前缺乏明确指导。但我们可以从agent工程中获得启发：好的harness不是替代agent的决策，而是增强其能力，最终让agent能独立执行更复杂的任务。同样，RescueTime 应作为“注意力的训练轮”，而非永久依赖。

## 对比其他工具：Focusmate 与 Brain.fm

Focusmate 提供了另一种重锚定机制：通过实时同伴问责，利用身体在场效应提供外部结构和社会压力（来源：Focusmate）。它相当于agent工程中的“人工验证循环”——一个外部监督者定期检查进度。但Focusmate依赖实时视频，可能对社交焦虑用户造成压力（来源：Focusmate）。Brain.fm 则试图从神经层面影响注意力，通过神经锁相技术帮助用户进入专注状态（来源：Brain.fm），但其针对ADHD的效果缺乏独立临床研究（来源：矛盾与存疑）。

RescueTime 的优势在于被动、低负担，适合作为日常基线工具；而Focusmate 适合任务启动困难时刻，Brain.fm 适合需要深度专注的时段。三者可以组合使用，但RescueTime 的“事后审计”功能是独特的——它提供了其他工具无法替代的客观数据反馈。

## 今天就能试的行动

1. **设置每日10分钟回顾**：使用RescueTime的每日报告，花10分钟观察自己的时间分配，标记至少一个“漂移时刻”，并写下当时的分心原因。这相当于agent的“日志分析”步骤。
2. **定义你的“system prompt”**：在每天开始工作时，用一句话写下当天最重要的目标（例如“完成报告大纲”），然后打开RescueTime的“目标”功能，设定专注时长。这相当于为LLM设定初始指令。
3. **结合Focusmate进行“硬重启”**：当发现RescueTime显示自己已漂移超过30分钟，立即预约一个Focusmate时段，用外部问责强制重锚定。这相当于agent的“异常处理”流程。

RescueTime 不是万能药。它的价值在于，它用工程化的方式，让ADHD大脑的“目标漂移”变得可见、可测量、可干预。正如agent harness让LLM变得可靠，RescueTime 让ADHD大脑的生成能力有了一个外部调度层。但记住：工具是脚手架，不是拐杖。最终，我们需要学会在无工具时也能重锚定——这本身就是一个值得练习的技能。

# 如果把 ADHD 大脑当成生成核心，RescueTime 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 RescueTime 对 ADHD 的真正价值。

## 问题：分心走神或超聚焦跑偏，丢失目标

你是否有过这样的经历：打开电脑准备写报告，三小时后发现自己刷完了整个社交媒体，或者深陷一个无关紧要的细节，完美主义地调整了十遍字体，却忘了最初的目标？这种“目标漂移”是 ADHD 人群的核心痛点。ADHD 大脑像一个强大的生成核心，创意和能量不断涌现，但缺乏可靠的执行调度层，导致输出不稳定（来源：AI 与 ADHD 的专注力——同构视角）。同样的问题也困扰着 AI 工程师：在构建长程 agent 任务时，LLM 会随着上下文膨胀逐渐偏离原始目标，需要重锚定 system prompt。那么，如果把 ADHD 大脑视为一个生成核心，RescueTime 能否充当它的重锚定机制？

## 同构：高产但缺调度层的生成核心

ADHD 大脑与 LLM 在结构上高度同构：两者都拥有强大的生成能力（ADHD 的创意发散、LLM 的文本生成），但缺乏可靠的执行调度层（ADHD 的执行功能、LLM 的上下文控制）（来源：AI 与 ADHD 的专注力——同构视角）。ADHD 的“工作记忆”缺陷与 LLM 的“无状态性”对应，都需要外部记忆系统来维持连续性。而“上下文工程”——主动设计“当下注意什么”的方案——在两边都至关重要。

历史人物曾国藩就是一个典型例子。他30岁前浮躁坐不住，天天串门喝酒，日记里骂自己“无恒”“浮躁”，读书“疾读不能终一行”。他的解决方案是“日课十二条”：黎明即起、读书不二、谨言、写日记反省。这套系统本质上是一个外部调度层，通过定时 re-grounding（如“读书不二”要求一次只读一本书）对抗目标漂移。这与 LLM agent 的“定时重锚定 system prompt”完全同构：曾国藩的日课相当于每段时间的 system prompt，写日记反省相当于日志记录和偏差修正。

## RescueTime 作为重锚定工具

RescueTime 的机制恰好对应了这种需求。它自动化地记录用户在各应用和网站上的时间，通过 AI 分类为“高效”“中性”或“分心”，并生成可视化报告（来源：RescueTime）。对于 ADHD 用户，它解决了“时间盲”问题——无法准确感知时间流逝（来源：执行功能障碍）。更重要的是，它能在用户偏离目标时主动阻断分心网站，相当于一个实时的 system prompt 重锚定。

从 agent 工程角度看，RescueTime 扮演了“外部调度器”的角色：它持续监控行为流，当检测到目标漂移（如从写报告切换到刷社交媒体），就触发干预（阻断或提醒）。这与 LLM agent 中“如果回答偏离主题，则重新注入原始指令”的做法如出一辙。证据层面，RescueTime 被列为 ADHD 生产力工具，其客观时间记录能提升时间觉察，有初步研究支持其改善时间盲（来源：AI 与 ADHD 的专注力——证据脉络）。

## 脚手架 vs 拐杖：边界在哪里？

但 RescueTime 并非万能。它的自动分类和阻断规则可能不够精准，需要用户后期调整，这对执行功能较弱的用户仍是挑战。过度关注时间数据可能引发焦虑（来源：RescueTime——局限与争议）。这引出一个关键问题：AI 工具是“外部执行功能层”还是“拐杖”？长期依赖是否会导致内在执行功能退化？（来源：矛盾与存疑）

答案在于“脚手架 vs 拐杖”的边界。脚手架是临时支撑，最终目的是让用户学会自己走路；拐杖则是永久替代。RescueTime 作为重锚定工具，应该被设计为“可渐退”的：初期频繁提醒，后期逐渐减少干预，让用户内化时间感知能力。神经可塑性研究表明，训练可以改善抑制控制能力及其底层脑网络（来源：神经可塑性）。因此，RescueTime 应结合认知训练，而非单纯依赖外部阻断。

## 争议与局限

本文的观点——RescueTime 作为重锚定工具——仍有局限。首先，ADHD 大脑与 LLM 的同构是隐喻而非严格等价，LLM 的“注意力机制”与人类注意力存在本质差异（来源：AI 与 ADHD 的专注力——仍存争议）。其次，RescueTime 对超聚焦（hyperfocus）的处理存在矛盾：超聚焦本身是 ADHD 的优势，强行打断可能适得其反（来源：矛盾与存疑）。最后，个体差异巨大，同一工具对不同 ADHD 亚型效果迥异。

## 今天就能试的行动

1. **设置 RescueTime 的“分心阻断”规则**：将你最常分心的网站（如社交媒体）设为“非常分心”，并开启阻断模式。当被阻断时，问自己：“我当前的目标是什么？”——这就是一次重锚定。
2. **每天查看 RescueTime 报告 5 分钟**：不要追求完美，只需观察时间去向。这相当于 LLM agent 的日志审计，能帮你识别漂移模式。
3. **结合 Focusmate 进行“身体在场”重锚定**：预约一个 Focusmate 时段，在视频中工作。他人的存在提供了外部问责，相当于 agent 的“人类监督”机制（来源：Focusmate）。
4. **尝试“一次只做一件事”的日课**：像曾国藩的“读书不二”那样，每 30 分钟只专注于一个任务。用 RescueTime 验证是否跑偏。

RescueTime 不是万能药，但它提供了一个低成本的重锚定框架。对于 ADHD 大脑这个“高产但缺调度层的生成核心”，它至少能让你看见漂移，然后选择是否拉回。而对于工程师，理解这种同构或许能设计出更好的 agent harness——毕竟，人类大脑和 AI 模型面临的挑战，本质上是同一个：如何让生成保持方向。

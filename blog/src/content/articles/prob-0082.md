# Motion 之于 ADHD，就像 外部记忆/向量库 之于 LLM——但有人用错了

> 从同构视角实测 Motion：它到底补上了哪一层执行功能？

## 问题：你明明知道该做什么，但就是做不到

你打开 Motion，看着它自动排好的日程，心想“今天终于能按计划走了”。然而两小时后，你发现自己刷了半小时手机，而 Motion 静静躺在后台——它帮你规划了时间，却没帮你执行。

另一边，工程师在调试 Agentic Harness：LLM 没有跨会话状态，每次对话都像失忆。他们给 LLM 挂上向量库、装上记忆模块，期望它能“记住”上下文，但发现如果 prompt 没写好，外部记忆反而让模型输出更混乱。

这两个场景共享同一个核心问题：**你给了它外部记忆/调度，但它缺少一个“执行调度层”来真正利用这些资源。** ADHD 大脑和 LLM 都是“高产但缺执行调度层的生成核心”——前者能产生绝妙想法但无法启动任务，后者能生成流畅文本但无法自主规划步骤。

## 同构：无状态与外部记忆

ADHD 大脑的工作记忆容量并不小，但控制弱、易被干扰（来源：[[工作记忆]]）。LLM 同样有强大的记忆容量（上下文窗口），但上下文控制脆弱——一旦 token 超限或话题偏移，它就“忘记”之前的指令。

解决方案同构：**外部记忆/调度系统**。

- **ADHD 侧**：Motion 通过 AI 自动规划日程、估算时间，弥补[[时间盲]]和[[任务启动]]困难（来源：[[Motion]]）。Saner.AI 通过知识回忆减少标签切换（来源：[[Saner.AI]]）。Goblin Tools 将大任务分解为小步骤，降低启动门槛（来源：[[Goblin Tools]]）。
- **LLM/Agent 侧**：向量数据库（如 Pinecone、Chroma）提供持久化外部记忆，让 LLM 跨越会话保持状态。Agentic Harness 中的“工具调用”和“任务分解”模块，相当于给 LLM 装上了执行调度层。

这些工具的本质都是**为生成核心套上 harness（脚手架）**（来源：[[ADHD 的 AI 工具全景]]）。但问题在于：用户（和工程师）常常把“外部记忆”误当成“执行层”本身。

## 核心判断：Motion 补上了调度层，但有人用错了

Motion 的设计逻辑是对的：它用 AI 自动规划任务、调整时间估计，直接针对时间盲和启动困难（来源：[[时间盲]]）。但它的局限在于——**它只解决了“知道什么时候做什么”，没有解决“为什么我会在那一刻不执行”。**

ADHD 的执行功能缺陷不只是“不知道时间”，还包括[[多巴胺]]不足导致的动机缺乏、[[拒绝敏感性焦虑]]导致的逃避（来源：[[任务启动]]）。Motion 无法提供多巴胺奖励，也无法缓解焦虑。它像给 LLM 装了一个完美的向量库，但 prompt 里没有“如何调用这个库”的指令——LLM 依然不知道何时该检索、如何整合检索结果。

对比来看，Goblin Tools 的“魔法待办”直接降低启动阻力（来源：[[Goblin Tools]]），Lex 的“单一指令触发多步骤”减少决策疲劳（来源：[[Lex]]），这些工具更贴近“执行调度层”的本质——它们不只是存储计划，而是**直接改变行为触发条件**。

## 争议与局限：脚手架还是拐杖？

所有 ADHD AI 工具都面临同一个矛盾：**它们声称是“脚手架”，但实际使用中可能沦为“拐杖”。** 过度依赖 Motion 的外部调度，可能削弱内在的时间感知能力（来源：[[时间盲]]）；过度依赖 Saner.AI 的自动回忆，可能削弱主动记笔记的能力（来源：[[AI 与 ADHD 的学习方式]]）。

更关键的是，现有证据主要来自用户报告，缺乏大规模对照实验（来源：[[矛盾与存疑]]）。Motion 的效果因人而异——对某些人它是救星，对另一些人它只是另一个被忽略的通知。

## 今天就能试的行动

1. **用 Motion 做“单任务规划”而非“全天规划”**：只让它规划下一件最重要的事，而不是排满全天。这能避免 overwhelm，并让启动目标更清晰。
2. **结合 Goblin Tools 的“魔法待办”**：在 Motion 排好时间后，用 Goblin Tools 把任务拆成 3-5 个物理步骤，贴在眼前。这补上了 Motion 缺失的“行为触发”。
3. **设置“执行检查点”**：在 Motion 日程中插入 5 分钟空白，用于问自己“我现在真的在做事吗？”——这是外部执行调度层的最小实现。
4. **对工程师：在 Agentic Harness 中加入“启动提示”**：不要只给 LLM 外部记忆，还要在 prompt 中显式告诉它“你需要在对话开始时检索最近 3 条相关记忆，并总结当前任务状态”。这相当于给 LLM 装了一个“任务启动”模块。

## 结语

Motion 不是万能药。它补上了外部记忆和调度，但执行层的最后一公里需要你自己走——或者借助更底层的工具（如 Goblin Tools、Lex）来走。ADHD 大脑和 LLM 一样，需要的不是一个更大的记忆库，而是一个**知道何时、如何调用记忆的执行调度层**。

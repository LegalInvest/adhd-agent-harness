# 用 ChatGPT 给 ADHD 补上人在回路与身体在场，真的能像 agent 那样工作吗？

> 把 ChatGPT 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

## 问题：独自做事时，为什么 ADHD 大脑总是“启动失败”？

如果你是一个 ADHD 成年人，你一定熟悉这个场景：坐在电脑前，任务清单摊开，但就是无法开始。你清楚知道该做什么，但大脑像断了线的风筝，每个念头都飘走。你缺的不是能力，而是“问责”——一个外部的声音告诉你“现在做这个”，以及“身体在场”——你的身体和任务在同一时空，无法逃避。

如果你是一个做 Agentic Harness 的工程师，你也熟悉另一个场景：你精心设计的 LLM agent 在沙盒里表现完美，一上线就胡言乱语。它生成的内容逻辑缜密，但执行时忘了上下文、跳过了关键步骤、甚至调用工具的顺序完全错误。你缺的也不是生成能力，而是“人在回路”——一个外部监督者确保每一步都锚定在真实需求上。

这两个问题，本质上是同一个问题。ADHD 大脑和 LLM/agent 在结构上是同构的：它们都是“高产但缺执行调度层的生成核心”（来源：GitHub - ai-boost/awesome-harness-engineering）。ADHD 大脑的生成能力（创意、联想、直觉）极强，但内在的执行功能脚手架——工作记忆、时间感知、任务启动——天生薄弱（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）。LLM 同样如此：它能在给定 prompt 下生成惊人内容，但如果没有 harness（上下文传递、工具接口、规划工件、验证循环、记忆系统），它就是一个无状态的 API 调用，无法可靠地完成多步任务（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。

那么问题来了：用 ChatGPT 给 ADHD 补上“人在回路”与“身体在场”，真的能像 agent 的 harness 那样工作吗？

## 答案：能，但前提是把它当作脚手架，而非拐杖

我的核心判断是：ChatGPT 可以充当 ADHD 大脑的“外部执行层”——就像 harness 之于 LLM——但它的有效性取决于你如何使用它。工具本身不是解药，设计工具的使用方式才是。

### 1. 人在回路：ChatGPT 作为问责伙伴

ADHD 侧：独自做事缺乏问责，容易放弃。ChatGPT 可以扮演一个“永不疲倦的监督者”。你可以告诉它：“我要在接下来 25 分钟写报告，每 5 分钟提醒我一次当前目标。”它不会评判你，但它会持续把你拉回轨道。这种外部问责机制，直接补偿了 ADHD 大脑缺失的“内部监工”。

LLM/agent 侧：高风险 agent 需要 human-in-the-loop 监督。Harness 工程的核心之一就是设计验证循环——让人类在关键节点检查 agent 的输出（来源：Building AI Coding Agents for the Terminal）。ChatGPT 作为工具，本身不具备 harness 的全部功能，但你可以手动设计一个“人+ChatGPT”的验证循环：每完成一步，让 ChatGPT 总结并询问“下一步是否继续？”——这恰恰是人在回路的简化版。

### 2. 身体在场：ChatGPT 作为任务分解器

ADHD 侧：身体在场意味着“小步骤的即时反馈”。Goblin Tools 的 Magic ToDo 功能将“整理房间”分解为“捡起衣服”“擦桌子”等具体动作（来源：AI Tools for Kids with ADHD: A Complete Guide for Parents...），这种分解降低了启动门槛，让身体更容易“在场”。ChatGPT 可以做到同样的事：输入“我卡住了，帮我把写论文分解成 5 分钟能完成的小步”，它就能生成一个可执行的列表。

LLM/agent 侧：Agent 的“身体在场”是工具调用和沙箱执行。Harness 通过工具接口让 agent 与真实世界交互（来源：GitHub - ai-boost/awesome-harness-engineering）。ChatGPT 的插件和代码解释器就是它的“身体”——它不能直接操作你的文件系统，但可以通过 API 读写数据、运行代码。对 ADHD 用户来说，让 ChatGPT 直接处理“打开这个文件，提取第二段，翻译成英文”这样的任务，就是把“身体”外包给了 AI。

### 3. 外部记忆：ChatGPT 作为第二大脑

ADHD 侧：工作记忆容量有限，跨会话任务难以持续。Saner.AI 通过知识回忆和本地记忆减少搜索循环（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。ChatGPT 的记忆功能（如果启用）可以存储你的偏好、项目背景，但它的记忆是对话式的、不持久的。你需要手动创建“项目上下文”文档，每次会话开始时粘贴进去——这相当于手动实现外部记忆。

LLM/agent 侧：无状态是 LLM 的固有限制。生产级 agent 通过向量数据库和检索增强实现长程记忆（来源：The Anatomy of an AI Agent: Memory, Tools, Planning, and Execution Explained - DEV Community）。ChatGPT 的“临时记忆”就像 agent 的短期上下文窗口，而你的“项目上下文文档”就是它的长期记忆库。

## 脚手架 vs 拐杖：边界在哪里？

这里必须诚实指出争议与局限。所有 ADHD 侧的证据都来自用户反馈和概念对齐，缺乏独立随机对照试验（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。LLM/agent 侧的证据则来自工程实践，但“ADHD 大脑与 LLM 同构”这个命题本身缺乏大规模实证（来源：ADHD × AI 的科学与研究前沿）。

更关键的是“过度依赖”风险。真正的脚手架应该帮助使用者“建造”，但使用者仍需自己“攀登”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。如果 ChatGPT 替你完成了所有任务分解、日程安排、甚至执行，你就没有机会锻炼自己的执行功能。Motion 这类工具自动规划日程，消除了“下一步该做什么”的决策负担（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...），但如果它永远替你决策，你的时间管理能力可能永远不会成长。

我的判断是：ChatGPT 作为脚手架，应该满足“可撤除性”——你使用它的目的是最终不再需要它。例如，先用 ChatGPT 分解任务，逐渐过渡到你自己尝试分解，最后只在卡住时求助。如果它变成了你永远离不开的拐杖，那它就阻碍了内在能力的发展。

## 今天就能试的 3 条行动

1. **建立“人在回路”协议**：每次使用 ChatGPT 完成任务时，在对话开头加上：“请每完成一步后问我‘要继续吗？’直到我说停。”这模拟了 harness 的验证循环。
2. **手动实现外部记忆**：创建一个名为“项目上下文”的文档，包含当前任务的所有背景、目标和下一步。每次开始新会话时先粘贴进去。这相当于 agent 的长期记忆库。
3. **使用任务分解模板**：当你感到压倒性时，输入：“请将以下任务分解为 5 分钟能完成的步骤：[你的任务]”。完成后，尝试自己先分解，再用 ChatGPT 对比——逐步撤除辅助。

## 结语

ChatGPT 不是 ADHD 的解药，也不是 agent 的完美 harness。但它是一个可用的外部执行层——前提是你把它当作脚手架，而不是拐杖。ADHD 大脑和 LLM 是同一种困境的两种表现形式：生成能力强，执行调度弱。两边的解法结构同构：都需要人在回路、身体在场、外部记忆。区别在于，agent 的 harness 是工程师设计的，而你的 harness 需要你自己设计。从今天开始，用 ChatGPT 搭建你的个人 harness，但别忘了——最终你要学会自己攀登。

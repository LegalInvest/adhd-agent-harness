# 用 ChatGPT 给 ADHD 补上人在回路与身体在场，真的能像 agent 那样工作吗？

> 把 ChatGPT 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

深夜，你独自坐在电脑前，一个重要的项目 deadline 就在明天，但你却刷了三个小时短视频。你心里清楚该做什么，但就是启动不了。这不仅是意志力问题——ADHD 大脑缺乏一个关键的“人在回路”监督层，导致独自做事时问责缺失、容易放弃。与此同时，AI 工程师们正在为高风险 agent 搭建 human-in-the-loop 监督系统，防止模型失控。两个看似无关的困境，其实共享同一套结构：一个高产的生成核心，配上一个不可靠的执行调度层。

## 同构：ADHD 大脑与 LLM 都是“有输出无调度”的引擎

ADHD 大脑的典型特征是高创意产出与低执行稳定性并存。研究显示，ADHD 个体的创意产出率是普通人的 1.8 倍（来源：Lifted Ventures 2024）。但工作记忆容量有限、时间感知困难、任务切换障碍，让这些创意难以落地。类似地，大型语言模型（LLM）在无外部 harness 时，是一个无状态的 API 调用（来源：Building an AI Agent Harn），每次对话独立，无法自主组织任务，需要严重依赖手动修正提示（来源：Working Memory Identifies Reasoning Limits in Language Models）。

更惊人的是，认知神经科学发现 Transformer 的自注意力机制在训练后，会发展出模仿前额叶-纹状体门控的操作（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。这意味着，ADHD 大脑与 LLM 不仅在功能上平行——自动模式与自动完成机制类似，两者都在有限执行监督下生成复杂行为，且易出现幻觉或错误（来源：Automatic Minds: Cognitive Parallels Between Hypn）——而且在计算层面存在同构性。

## 人在回路：ADHD 的“外部监督层”与 agent 的“human-in-the-loop”

既然问题出在调度层，解法也必然同构。对于高风险 agent，工程师必须引入 human-in-the-loop 监督，定期检查 agent 的输出，防止偏离轨道。对于 ADHD 大脑，同样需要一个“人在回路”——但这个人可以是 AI 助手。

以 ChatGPT 为例，它可以通过对话提供持续的外部问责。当你告诉它“我要在 2 小时内写完报告，每 30 分钟提醒我进度”，它就充当了一个永远在线的监督者。这种“人在回路”效应弥补了 ADHD 个体内部问责的缺失。Goblin Tools 的 Magic ToDo 功能则更进一步，它接受模糊任务（如“清理厨房”），并将其分解为具体、可执行的子步骤，用户可调节“辣度”控制粒度（来源：12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。这种分解利用小步骤的即时反馈维持注意力，模拟了身体在场效应——就像有人站在你旁边说“先擦台面，再洗碗”。

## 历史人物的 harness：张一鸣的“系统即调度器”

字节跳动创始人张一鸣具有明显的 ADHD 特质：思维极度发散，什么新东西都要研究，永远在思考组织和产品。他的专属 harness 是“延迟满足感”和 OKR 管理系统，通过“Context not Control”用系统和文化管公司，不搞集权。这本质上是一个外部调度层：OKR 充当了目标分解与进度追踪的“人在回路”，而 Context 机制让信息透明流动，减少了他个人的工作记忆负担。

这与 LLM harness 的结构惊人一致：OKR 就像 agent 的规划模块，Context 相当于外部记忆系统。张一鸣没有依赖自己的意志力去记住每个细节，而是构建了一个系统来替他执行调度。同样，ADHD 个体可以用 ChatGPT 作为“外部调度器”，设定每日检查点，让 AI 替自己记住上下文和下一步行动。

## 工具即脚手架：Goblin Tools、Saner.AI 与 Motion 的实践

当前已有多个 AI 工具专门针对 ADHD 的执行功能缺陷设计。Goblin Tools 通过任务分解降低启动门槛；Saner.AI 提供知识回忆和本地记忆，减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）；Motion 则自动排程并动态调整日程，消除“下一步该做什么”的决策负担（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。这些工具的共同点是外化执行功能，充当“外部执行功能层”（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）。

但必须指出，这些工具是脚手架而非拐杖。脚手架帮助使用者“建造”，但使用者仍需自己“攀登”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。过度依赖可能导致能力退化，长期依赖风险缺乏实证（来源：矛盾与存疑）。此外，部分用户反映学习使用 AI 工具本身增加认知负荷（来源：矛盾与存疑），净效果因人而异。

## 争议与局限：脚手架 vs 拐杖的边界

本文的核心观点是：ADHD 大脑与 LLM 同构，因此用 ChatGPT 作为外部人在回路与身体在场补偿，能像 agent harness 一样提升执行。但这个论点目前多为类比推理，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。此外，超聚焦是否应被引导而非打断？部分 ADHD 用户反馈强行打断（如番茄钟）反而有效，而柔性引导可能不够强力（来源：矛盾与存疑）。个体差异极大，尚无定论。

另一个局限是 AI 工具对任务启动的长期效果未知。短期有效，但长期效果缺乏纵向研究（来源：矛盾与存疑）。Motion 的用户反馈积极，但独立临床证据不足（来源：矛盾与存疑）。

## 今天就能试的 3 件事

1. **用 ChatGPT 设定“人在回路”**：告诉它你的任务和目标，让它每 25 分钟提醒你检查进度，并记录你的完成情况。这模拟了外部监督。
2. **用 Goblin Tools 分解一个让你拖延的任务**：输入“整理房间”或“写周报”，调节“辣度”到你觉得可执行的粒度，然后从第一步开始做。
3. **用 Saner.AI 建立外部记忆**：每天结束时，用语音或文字向它记录项目关键信息，第二天开始前让它总结，避免重新回忆上下文。

记住：工具是脚手架，不是拐杖。真正的目标是让你自己最终能建造更坚固的内部执行系统。

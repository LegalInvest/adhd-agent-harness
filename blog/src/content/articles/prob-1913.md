# ADHD 家长视角：为什么「治好 ADHD 的执行功能（计划/组织/启动）先天偏弱」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 的编排/记忆/工具层就是「外部执行功能」——同一套 harness 思路，两边都成立。

## 问题：为什么最聪明的脑子，和最聪明的模型，都容易「跑飞」？

如果你是一位 ADHD 家长，你一定经历过这样的场景：孩子（甚至你自己）明明智商在线、创意爆棚，却连早上出门都能搞砸——计划好的事情，到时间点就是启动不了；任务做到一半，被一个念头带走，再也回不来。你可能会自责：是我不够努力？还是脑子有问题？

与此同时，如果你是一位在做 Agentic Harness 的工程师，你一定也经历过类似的挫败：LLM 明明能写出漂亮的代码、生成惊艳的文案，但一旦让它自主执行多步任务，它就「跑飞」了——偏离目标、陷入循环、输出不可控。你会想：是模型不够强？还是架构有问题？

答案出乎意料地一致：**两者都不是核心能力的问题，而是缺少一个可靠的执行调度层。**

## 同构：ADHD 大脑与 LLM 共享的「执行功能缺口」

ADHD 的核心障碍是执行功能缺陷——计划、组织、工作记忆、任务启动等高级认知过程先天偏弱（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。这就像大脑有一个强大的生成核心（发散思维、创意、多任务并行），但「驾驶座」常常无人驾驶。而 LLM 同样如此：它拥有海量的参数和强大的生成能力，但本身是无状态的、上下文窗口有限、输出不可控——缺少一个编排层来规划、调度、纠偏。

这就是本文的核心观点：**ADHD 大脑与 LLM 是同一类「高产但缺执行调度层」的生成核心。** 两者的解决方案在结构上同构：都需要一个外部执行功能层——对 ADHD 来说是脚手架（如任务分解、提醒、身体在场），对 LLM 来说是 agent harness（编排、记忆、工具层、人在回路）。

## 证据：两边都有真实解法

### ADHD 侧：外部执行功能工具

- **Goblin Tools** 的 Magic ToDo 功能，能将「整理房间」这样的模糊任务自动分解为「捡起地板上的衣服」「擦桌子」等具体步骤，直接降低启动门槛（来源：AI Tools for Kids with ADHD: A Complete Guide for Parents...）。这相当于给大脑加了一个「任务分解层」。
- **Motion** 和 **Reclaim.ai** 则提供动态日程规划：自动根据优先级和截止时间调整计划，消除「下一步该做什么」的决策负担（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。这相当于一个「实时调度层」。
- **Saner.AI** 专注于知识回忆，减少搜索循环和标签切换（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar），相当于一个「外部工作记忆」。
- **Focusmate** 利用身体在场效应，通过 AI 匹配虚拟搭档提供外部问责（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...)），这本质上是一个「人在回路」的监督层。

这些工具的共同点：不试图「修复」大脑，而是为大脑套上一个外部执行功能层。

### LLM/Agent 侧：Harness 工程

- 现代 agent harness 通过 **human-in-the-loop** 监督提供外部调度：设置护栏（token 预算、工具调用次数上限、防止无限循环）、加入人工检查点（暂停并询问后再执行）（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。
- Harness 将 MCP 工具、遥测、代码仓库、事件管理平台集成到单一系统中，并引入人在回路治理（来源：GitHub - ai-boost/awesome-harness-engineering）。其核心是「形式化规划与护栏，使代理输出真正有用且正确」（来源：What is an agent harness in the context of large-language models?）。

注意：模型本身强大，可靠性来自架构+护栏（来源：AI Agents in 2026: Tools, Memory, Evals, and Guardrails）。这与 ADHD 工具的逻辑完全一致。

## 关键判断：脚手架 vs 拐杖——边界在哪里？

如果外部执行功能层只是永久替代了内在能力，那它就是拐杖，会阻碍成长。但真正的脚手架应该设计为可逐步撤除的临时支撑（来源：拐杖与脚手架）。然而，当前多数 ADHD 工具（如 Goblin Tools、Saner.AI）被设计为长期使用，并未提及撤除机制（来源：矛盾与存疑）。同样，agent harness 如果过度依赖人工干预，也无法实现真正的自主。

我的判断是：**脚手架与拐杖的边界在于「是否支持能力内化」**。好的 ADHD 工具应该帮助用户逐渐学会自我分解任务、自我调度——例如，从 AI 自动分解，到用户手动调整步骤，再到完全自主规划。好的 harness 也应该逐步减少人工检查点，让 agent 学会自我纠偏。但目前缺乏纵向研究来验证这一点（来源：矛盾与存疑）。

## 局限与争议

必须诚实指出：ADHD 大脑与 LLM 的同构命题目前主要基于概念类比和工具案例，缺乏大规模神经影像学和行为实验验证（来源：矛盾与存疑）。此外，多巴胺干预的有效性存在争议（来源：多巴胺页面），且 AI 工具本身的证据多来自用户反馈而非独立随机对照试验（来源：执行功能障碍页面）。因此，本文的论点是一个有前景的理论框架，而非定论。

## 结语

ADHD 大脑和 LLM 都不是坏掉的发动机，而是缺少一个可靠的驾驶系统。当工程师在构建 agent harness 时，他们实际上在解决和 ADHD 家长同样的问题：如何让一个高产的生成核心，可靠地执行计划。这道工程题的答案，两边通用——外部执行功能层。

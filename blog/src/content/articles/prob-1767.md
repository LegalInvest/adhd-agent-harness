# 用 Claude 给 ADHD 补上外部执行功能层，真的能像 agent 那样工作吗？

> 把 Claude 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

## 问题：当大脑自带“生成核心”却缺“调度层”

如果你是一个ADHD成年人，你可能熟悉这样的场景：脑子里有无数想法，却无法将它们组织成一个可执行的计划；面对一个任务，大脑像卡住的引擎，明明知道该做什么，就是启动不了。这不是意志力问题，而是执行功能——计划、组织、启动、时间管理——先天偏弱（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

与此同时，如果你是一个在做Agentic Harness工程的工程师，你也会熟悉另一个场景：LLM本身能力强大，但单独使用时，它没有记忆、没有调度、没有工具调用能力，经常在长上下文中跑偏或产生幻觉。于是你为它搭建脚手架——提示词、记忆模块、工具API、编排逻辑——这些被称为“外部执行功能层”（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

这两个问题惊人地同构。ADHD大脑与LLM都是强大的生成核心，但缺乏可靠的内置调度层。那么，用Claude这样的LLM给ADHD补上外部执行功能层，真的能像agent那样工作吗？答案是：可以，但需要正确的设计，并且要警惕工具从“脚手架”滑向“拐杖”。

## Claude作为外部执行层：它补的是哪一块短板？

Claude本身只是一个生成核心，它没有内置的日程管理、任务分解或记忆系统。但通过提示词工程和外部工具集成，它可以被塑造成一个外部执行功能层。具体来说，它补的是以下短板：

- **任务启动困难**：ADHD患者常因任务模糊而无法启动。Claude可以通过类似Goblin Tools的Magic ToDo功能，将“清理厨房”分解为“把碗放进洗碗机、擦台面、拖地”等具体步骤，降低启动门槛（来源：Goblin Tools页面）。这对应了LLM agent中的“任务分解与规划”层。
- **工作记忆不足**：ADHD的工作记忆容易超载，导致频繁切换标签页。Claude可以作为外部记忆系统，通过对话历史或自定义知识库，记住用户的项目状态、偏好和进度，类似Saner.AI的本地记忆功能（来源：Saner.AI页面）。这对应了agent中的“记忆模块”。
- **时间盲**：ADHD患者难以感知时间流逝，导致计划崩溃。Claude可以结合Motion的自动排程逻辑，帮助用户将任务分配到具体时间段，并动态调整（来源：Motion页面）。这对应了agent中的“调度与编排”层。

但关键在于：Claude本身并不具备这些能力，它需要被“harness”起来。正如AI编码代理通过“工作负载专用模型路由、分离规划与执行的双代理架构”来弥补LLM的缺陷（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned），ADHD用户也需要为Claude搭建类似的脚手架——例如，使用结构化提示词、外部日历API、任务管理插件等。

## 历史中的Harness：曾国藩与胡林翼的“日课十二条”

这种“外部执行功能层”并非AI时代的发明。晚清的曾国藩和胡林翼，都是典型的ADHD特质者，却通过自建的harness系统实现了非凡成就。

曾国藩年轻时“浮躁坐不住，天天串门喝酒看杀人”，日记里骂自己“无恒”“浮躁”，读书“他人目下二三行，余或疾读不能终一行”——这是典型的注意力不集中型ADHD。他的解决方案是“日课十二条”：黎明即起、读书不二、谨言、写日记反省等。这些规则构成了一个外部调度层，强制他每天执行固定的行为序列，抵消内在的冲动和涣散。胡林翼年轻时是花花公子，父亲去世后幡然醒悟，同样通过每日写日记反省、严格治军来建立外部纪律。

这个系统与LLM agent的harness结构同构：“日课”相当于定时re-grounding（每日反省相当于上下文刷新），“写日记”相当于外部记忆，“结硬寨打呆仗”相当于用最稳定的算法抵消生成核心的随机性。曾国藩的成就——平定太平天国、发起洋务运动——证明了这个外部执行功能层的有效性。但请注意：他从未停止“攀登”，日记中的自我批评表明他始终在主动使用脚手架，而非依赖拐杖。

## 证据与局限：同构框架的真实支撑

这个同构框架并非空想。神经科学证据显示，ADHD患者前额叶执行功能网络激活不足（来源：ADHD × AI 的科学与研究前沿），而LLM在无调度层时也表现出类似的“执行功能缺陷”——例如，在长上下文中推理退化。更具体地，一项研究发现ADHD患者的工作记忆呈现“强记忆、弱控制”模式，与LLM惊人相似（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。另一研究显示，Transformer的自注意力机制在训练后模仿了前额叶-纹状体的门控操作（来源：TRANSFORMER MECHANISMS MIMIC FRONTOSTRIATAL GATING OPERATIONS WHEN TRAINED ON HUMAN WORKING MEMORY TASKS）。

但必须诚实指出争议。首先，同构性目前仅是一种理论类比，缺乏大规模实证验证（来源：矛盾与存疑）。其次，工具如Motion缺乏独立临床验证（来源：Motion页面），而Goblin Tools虽受好评，但其效果主要基于用户反馈而非随机对照试验。最重要的是，过度依赖风险真实存在：若AI工具替代了自我管理能力的培养，执行功能可能进一步退化（来源：拐杖与脚手架）。真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

## 今天就能试的行动

1. **用Claude做任务分解**：将你拖延的一个模糊任务（如“写报告”）输入Claude，要求它分解为不超过5分钟的子步骤，并设定“辣度”级别。完成后立即勾选，获得即时反馈。
2. **搭建外部记忆系统**：在Claude对话中创建一个“项目记忆”文档，每次对话开始前粘贴，让Claude记住你的进度和偏好。这相当于agent的长期记忆模块。
3. **结合日历实现自动排程**：将Claude分解后的任务手动拖入日历（或用Motion等工具自动排程），设定固定检查点（如每天上午9点），模拟曾国藩的“日课”节奏。
4. **设置“re-grounding”提醒**：每工作45分钟，让Claude提醒你回顾当前任务是否偏离目标，并重新规划下一步。这对应agent的“上下文刷新”机制。

Claude可以成为你的外部执行功能层，但它不是魔法。正如曾国藩的日课需要每日坚持，LLM agent的harness需要持续调试。真正的价值在于：你理解了ADHD大脑与LLM是同构的，因此你可以像工程师一样，为自己设计一个个性化的、可迭代的脚手架系统——而不是依赖一个万能的拐杖。

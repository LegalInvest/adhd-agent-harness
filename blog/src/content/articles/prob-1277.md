# 用 ChatGPT 给 ADHD 补上外部执行功能层，是脚手架还是拐杖？

> ChatGPT 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

你盯着空白的待办清单，脑子里的想法像喷泉一样涌出，但手却像被冻住了一样——这是 ADHD 大脑的日常：高产但无序，生成核心火力全开，执行调度层却频繁宕机。与此同时，工程师们正在为 LLM 搭建 agent harness，处理上下文管理、工具调用和任务编排，因为大模型本身同样缺乏内建的调度与执行控制。

这不是巧合。ADHD 大脑与 LLM 在结构上同构：两者都是强大的生成核心，但都缺少可靠的内置执行调度层（来源：ADHD 大脑与 LLM 的同构）。因此，用 ChatGPT 给 ADHD 补上外部执行功能层，本质上是在为这个生成核心套上 harness——它可以是脚手架，也可以是拐杖，边界在于设计是否保留了逐步撤除的可能。

## 同构脊柱：ADHD 与 LLM 共享的痛点

ADHD 侧的执行功能缺陷——工作记忆易丢失、时间盲、任务启动困难——在 LLM 侧有精确的对应：LLM 的无状态性（需要外部记忆管理）、上下文窗口溢出（丢失信息）、缺乏内在奖励机制（需外部提示）。

| ADHD 大脑 | LLM + Harness |
|-----------|---------------|
| 高产但缺乏执行调度层的生成核心 | 高生成能力但需外部调度的 LLM |
| 工作记忆易丢失上下文 | LLM 无状态，需外部记忆管理 |
| 需要 AI 助手减少决策、保留上下文 | Harness 负责上下文工程和决策路由 |
| 外部工具（如 Goblin Tools）作为执行功能支架 | Harness 中的模型连接器、工具调用等 |
| 多巴胺驱动任务启动困难 | 缺乏内在奖励机制，需外部提示 |

这种同构意味着，ADHD 人群需要的不是另一个“聪明的大脑”，而是一个可靠的外部调度层——就像 agent harness 之于 LLM。

## 真实工具：外部执行功能层的三个剖面

**Goblin Tools** 专攻任务启动困难。其 Magic ToDo 功能将“整理房间”分解为“捡起地板上的衣服”“擦桌子”等具体步骤，把压倒性的事情变成一系列不压倒性的事情（来源：Goblin Tools）。这对应 agent harness 中的工作流编排——将复杂任务拆解为可管理的子任务。

**Saner.AI** 聚焦工作记忆补偿。它通过本地记忆和快速检索，减少搜索循环和标签切换（来源：Saner.AI）。这对应 LLM harness 中的上下文管理——保存和恢复状态，避免信息丢失。

**Motion** 解决时间盲和规划压力。它自动根据任务优先级和截止日期动态调整日程，消除“下一步该做什么”的决策负担（来源：Motion）。这对应 harness 中的调度与路由——持续评估优先级并重新安排。

这些工具的共同逻辑是：将原本需要内在执行功能完成的任务——分解、记忆、排程——外包给 AI，从而释放认知带宽。

## 脚手架还是拐杖？边界在于可撤除性

核心判断：AI 工具作为外部执行功能层，本质是脚手架，但多数工具设计为长期使用，未提及撤除机制，存在依赖风险（来源：矛盾与存疑）。

脚手架与拐杖的区别在于：脚手架是临时支撑，旨在帮助建造内在结构；拐杖则是永久替代，可能削弱原有能力。神经可塑性研究表明，通过训练可以改善执行功能（来源：神经可塑性），但长期依赖 AI 工具是否会阻碍这种重塑？目前缺乏纵向研究（来源：矛盾与存疑）。

一个负责任的设计应该包含“撤除路径”：例如，随着用户使用，逐步减少分解粒度，让用户自己尝试合并步骤；或者提供“无 AI 模式”，让用户定期练习内在调度能力。

## 争议与局限

- **证据强度**：现有工具效果主要基于用户反馈，缺乏独立随机对照试验（来源：矛盾与存疑）。同构模型本身也主要基于概念类比，需更多实证验证。
- **多巴胺干预争议**：部分工具声称通过神经科学原理改善执行功能，但多巴胺干预的有效性本身存在争议（来源：矛盾与存疑）。
- **个体差异**：ADHD 症状谱系广泛，一种工具可能只对部分维度有效。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个压垮你的任务**：输入一个你拖延已久的任务（如“写报告”），观察 AI 给出的步骤。如果觉得粒度合适，就按步骤执行；如果太细，尝试合并——这是训练你的内在分解能力。

2. **用 Saner.AI 或类似工具建立“外部记忆”**：把当前项目的所有关键信息（截止日期、联系人、参考链接）存入一个 AI 可检索的笔记。当工作记忆丢失时，直接问 AI 而不是自己回忆。

3. **用 Motion 自动排程一天**：输入所有待办事项和截止日期，让 AI 生成日程。执行一天后，对比手动排程的差异，注意哪些决策负担被消除了。

4. **每周一次“无 AI 日”**：选择一天，不使用任何 AI 工具规划或执行任务。这能帮你觉察自己内在执行功能的真实状态，避免过度依赖。

## 结语

ADHD 大脑与 LLM 共享同一个困境：生成太多，调度太少。AI 工具作为外部执行功能层，提供了即时的解脱，但真正的解法不是永远依赖 harness，而是借它训练自己的调度层。脚手架终将撤除，而内在的能力才是你真正的 agent harness。

# 用 ChatGPT 给 ADHD 补上外部执行功能层，真的能像 agent 那样工作吗？

> 这个问题值得一个不含糊的回答,因为它决定了几百万人的期望值。答案是:**一半能,一半不能——而且知道哪一半不能,比知道哪一半能更值钱。** agent 的 harness 有五个组件,ChatGPT 天然胜任其中两个半;剩下的两个半,恰恰是 ADHD 最需要的部分,而它在架构上就给不了。这篇把账算清楚。

先列 agent 执行层的五个组件,作为对照表:①**规划器**(把目标拆成步骤);②**记忆**(跨会话的状态持久化);③**调度器**(什么时候执行什么,主动触发);④**验证器**(检查输出、闭合回路);⑤**中断恢复**(断点续跑)。一个 LLM 之所以能变成 agent,是因为 harness 在模型外面补齐了这五件;一个 ADHD 用户问「ChatGPT 能不能当我的执行功能层」,等于在问:这五件里,它能替我扮演几件?

**能的部分,而且是真能:**

**规划器——满分胜任。** 「帮我把报税拆成十五分钟的步骤」「第一步再细一点」——拆解是纯语言智力任务,正是大模型的本职。ADHD 自己拆任务时最容易犯的错误(第一步太大、隐含步骤缺失、高估自己状态)它都能修正,还能按你的要求无限重拆,零耐心成本。

**验证器——大半胜任。** 发出去之前贴给它:「这封邮件语气有问题吗?漏了什么?」「这个计划哪里不现实?」——作为外部审查的第二双眼睛,它比 ADHD 的自查(经常直接跳过)可靠得多。缺的半分:它不知道你的真实上下文,验证深度受限于你贴给它多少。

**中断恢复——半个胜任。** 你可以把断点写在对话里,回来时让它「帮我恢复上下文:我做到哪了」。但这依赖你主动回来——它不会来找你。

**不能的部分,架构性的不能:**

**记忆——形似而神不似。** 记忆功能能存偏好和事实,但**它不会主动维护你的任务状态**:今天的承诺、进行到一半的项目、上周说好的检查点——这些不会自动更新,不会过期提醒,不会汇总。它的记忆是「被问到才读」的,而执行功能需要的记忆是「到点自己跳出来」的。

**调度器——完全不能,这是致命缺口。** ChatGPT 不会在周二上午叫你开始写报告,不会在你拖延时发出提醒,不会主动发起任何事——**它是纯响应式的:你不去找它,它就不存在**。而 ADHD 执行功能崩坏最深的一环,恰恰是「发起」:知道该做、也有能力做,就是没有那个把你推进任务的触发器。一个不会主动触发的执行层,对启动障碍是零帮助——你依然需要先「想起来去问它」,而想起来,正是你缺的。

所以诚实的架构结论:**ChatGPT 是优秀的规划器+验证器,但一个完整的执行功能层,必须再拼上两块它没有的:调度靠日历/闹钟/Motion 类工具(时间触发),状态靠一个它之外的任务系统(或一个固定对话+每天定时打开它的日历提醒——用调度工具弥补它的被动性)。** 最实用的组合拳:日历提醒触发→打开固定对话→读断点→让它拆今天的块→执行→回来汇报(它当验证器)→写断点。这套流程里 ChatGPT 出智力,日历出主动性,你出执行——三方各司其职,才勉强凑齐 agent 的五件套。

## 边界

同构强度 B 级:harness 五组件是真实的 agent 架构,执行功能的多成分模型(启动/工作记忆/监控)有神经心理学基础,ChatGPT 各组件的胜任度评估是功能分析,无临床对照研究。声明:「外部执行功能层」是代偿工具,不是治疗——中重度 ADHD 的启动与调节问题,评估、药物与行为治疗有最强证据;工具拼图应该拼在治疗之上,而不是替代它。

## 今天就能试的 3 件事

1. **把 ChatGPT 正式任命为规划器**:今天最堵的任务,让它拆到十五分钟粒度。
2. **给它配一个调度器**:一条每天固定时间的日历提醒——「打开对话,读断点」。
3. **别再对它期待主动性**:它不会来找你,把「来找你」的职能明确交给闹钟。

「像 agent 那样工作」的完整答案:模型一样强,harness 缺两件。**补上时间触发和状态系统,这套拼装 agent 就真能跑起来**——它不完美,但比裸奔的执行功能,已经好出一个数量级。

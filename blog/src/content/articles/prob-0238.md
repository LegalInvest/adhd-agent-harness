# Perplexity 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Perplexity：用久了会让 ADHD 更强还是更依赖？

## 问题：边做边忘，ADHD 与 LLM 的共同困境

你打开 Perplexity 搜索一个复杂问题，读了几段，思路被一条通知打断。回来时，你忘了刚才看到哪，甚至忘了最初要查什么。这种“边做边忘”的体验，ADHD 人群再熟悉不过——工作记忆容量小、易被干扰，就像 LLM 在长对话中丢失上下文一样。

LLM 本质上是无状态的：每次会话都是冷启动，靠外部向量库或记忆模块“续命”。ADHD 大脑呢？wiki 资料中提出的核心论点一针见血：**ADHD 大脑与 LLM 同构——两者都是“高产但缺执行调度层的生成核心”**（来源：ADHD 的 AI 工具全景）。这意味着，ADHD 的痛点与 LLM 的工程瓶颈在结构上对应：工作记忆差对应上下文控制弱，任务启动难对应冷启动问题，时间盲对应时序推理弱点。

那么，Perplexity——一个以搜索和对话为核心的工具——对 ADHD 来说，是帮我们搭起执行功能的脚手架，还是让我们越用越依赖的拐杖？

## 同构视角：无状态与外部记忆

先看 ADHD 侧的真实证据。ADHD 的工作记忆容量保留但控制弱，容易被环境带偏（来源：工作记忆）。LLM 同样有强记忆容量但弱上下文控制，会话稍长就“迷失”。两边都需要**外部记忆系统**来持久化上下文。

Saner.AI 正是针对这一痛点：它通过 AI 实现本地记忆存储和快速检索，减少搜索循环和标签切换（来源：Saner.AI）。用户无需在多个标签页间切换就能找回信息，直接补偿工作记忆不足。类似地，Perplexity 的对话历史、收藏功能，也是一种外部记忆——但问题在于，这些记忆是**被动存储**的，需要用户主动去翻查。

再看 LLM/agent 侧。工程师做 Agentic Harness 时，核心挑战就是让无状态的 LLM 拥有“记忆”：通过向量数据库、缓存机制、上下文压缩等，让模型能跨会话保持状态。Perplexity 的“集合”（Collections）功能就是这种思路的简化版——将相关查询和回答组织在一起，相当于一个轻量级外部知识库。

但这里有一个关键区别：**脚手架 vs 拐杖的边界在于，工具是否帮助你发展出内在的调度能力**。

## 脚手架还是拐杖？核心判断

我的核心判断是：**Perplexity 本身是一个中性的外部记忆插件，它成为脚手架还是拐杖，取决于你如何使用它，以及是否与其他工具组合成完整的执行功能层。**

如果只是把 Perplexity 当作“搜了就忘”的搜索引擎，每次遇到问题就丢进去，拿到答案后不加工、不组织，那它就是拐杖——因为你没有训练自己的工作记忆或信息组织能力，反而可能因为“随时能查”而更不主动记忆。wiki 资料中明确警告：过度依赖 AI 工具可能削弱内在能力（来源：矛盾与存疑）。

但如果将 Perplexity 与任务分解、启动脚本等工具配合使用，它就能成为脚手架的一部分。例如：
- 用 Goblin Tools 的 Magic ToDo 将复杂任务分解为小步骤（来源：Goblin Tools），然后针对每一步用 Perplexity 搜索具体信息。
- 用 Lex 的单一指令触发多步骤序列（来源：Lex），将 Perplexity 的查询嵌入其中，实现“搜索→整理→执行”的自动化流程。
- 用 Saner.AI 的本地记忆存储 Perplexity 的搜索结果，形成可复用的知识库。

这样，Perplexity 就不再是孤立的“答案机器”，而是外部执行功能层的一部分——它补偿了工作记忆，但任务分解（Goblin Tools）和启动脚本（Lex）补偿了执行调度，整体形成**脚手架**。

## 证据与局限

现有证据主要来自用户报告和工具设计逻辑。Goblin Tools 被评价为“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools），Lex 的设计理念与 ADHD 干预策略一致（来源：Lex）。但 wiki 资料也明确指出：**AI 工具作为外部执行功能层的有效性证据不足，主要来自用户报告和概念类比，缺乏大规模对照实验**（来源：矛盾与存疑）。此外，个体差异很大——有的 ADHD 用户可能对 Perplexity 的搜索界面本身产生多巴胺依赖，不断切换查询而非专注执行。

## 今天就能试的行动

1. **组合使用**：下次遇到复杂任务，先用 Goblin Tools 分解步骤，再用 Perplexity 搜索每个步骤的具体信息，最后将结果存入 Saner.AI 或笔记应用。
2. **主动加工**：使用 Perplexity 后，用一句话总结答案并写下来——这强迫你进行工作记忆的“编码”训练。
3. **设置边界**：每天限定 Perplexity 的使用次数或时间，避免无目的搜索，将其作为“工具”而非“玩具”。
4. **反思依赖**：每周回顾一次：是否因为用了 Perplexity 而不再尝试自己回忆或推理？如果是，调整使用策略。

## 结语

Perplexity 之于 ADHD，不是非黑即白的脚手架或拐杖。它是一块积木——放在被动记忆的角落就是拐杖，嵌进主动设计的执行功能层就是脚手架。关键在于，你是否愿意像工程师构建 Agentic Harness 那样，主动为自己的大脑设计“无状态→有状态”的转换机制。

# ChatGPT 之于 ADHD，就像 human-in-the-loop 监督之于 LLM——但有人用错了

> human-in-the-loop 的设计精髓在「loop」不在「human」:人不是站在旁边看,而是被架构进流程的固定位置——什么节点必须过人、什么级别自动放行,全部预先写死。ChatGPT 对 ADHD 的价值同理:不在于随时可以问它,在于把「问它」装进流程的固定位置。

先看 HITL 在 agent 工程里的真实形态。它不是「出事了找人」的模糊备援,而是一套精确的节点设计:低风险操作 agent 自动执行;中风险操作执行后向人报告;高风险操作(删数据、花钱、对外发送)**必须在执行前经人批准**——这个「必须」是架构强制的,不是 agent 自觉的。系统的可靠性恰恰来自这种不信任:不指望 agent 每次都判断对,只保证判断错时有闸。

ADHD 者用 ChatGPT,多数人处于「随时可以问」的状态——工具在场,但不在环上。区别是决定性的:**在场靠自觉调用,而 ADHD 缺的恰恰是「此刻该调用了」的元认知;在环靠流程触发,不依赖任何自觉。**

## 用对:把它焊进三个固定节点

**节点一:每日启动环**。每天开工的第一个动作(固定!)是和它对一遍:「今天的三件事是什么?昨天有没有挂起的承诺?第一件事的第一步是什么?」五分钟,产出贴在眼前。这个节点补的是计划功能——放在启动位是因为 ADHD 的一天极易被第一个撞进来的刺激定义,启动环就是抢在刺激之前装载优先级。

**节点二:发送前批准环**。上一层护栏:预先写死「必过审清单」(对上的、对外的、承诺资源的、拒绝别人的),清单内的输出,发送前必须过一遍 ChatGPT 审查。注意机制:**触发条件是「内容类型命中清单」,不是「我觉得需要看看」**——后者在冲动时刻永远不会触发。

**节点三:每周复盘环**。周五固定 15 分钟:把本周的完成/未完成丢给它,「帮我找模式:什么类型的事被拖了?什么时段产出高?下周该改哪一件事?」这个节点补的是自我监控——ADHD 的经验教训极难自动沉淀,复盘环就是手动挡的沉淀机制。

三个节点的共同设计:**固定时间/固定触发 + 固定问题模板 + 明确产出物**。这就是「在环」——流程走到那里,调用自然发生。

## 用错:三种「human out of the loop」

**全靠灵感调用**。想起来才问,而想起来的时刻从来不是最需要的时刻(最需要验证的冲动瞬间,脑子里没有 ChatGPT 的位置)。这等于给 agent 配了个「有空再看看」的审批人——架构上约等于没有。

**环建了,产出不落地**。每天也问了三件事,但产出停在对话框里,十分钟后被新标签页淹没。修复:环的终点必须是**物理落点**——写进便签、置顶文档、贴桌面。没有落点的环,只是仪式。

**把批准环用成决策外包**。「我该不该接这个活?」「该。」「好。」——这不是 HITL,是反向 HITL:你成了执行它决策的 agent。正确姿势永远是:它出分析、出反对理由、出选项,**拍板的动作留给你**。判别句:对话结束时,是你在说「我决定」,还是它在说「你应该」?

## 一个升级:环里加一个真人

ChatGPT 能当环上的分析节点,但有一类事它当不了闸:需要「对人的承诺压力」的场景(启动最难的任务、执行最容易放弃的计划)。给这类事加真人节点:每周和一个朋友互报三件事、关键任务找 body double。**机器环补认知,真人环补动机**——两条环并行,才是完整的监督架构。

## 边界

同构强度 B 级:HITL 是真实且核心的 agent 安全实践,ADHD 的元认知与自我监控弱势有文献,「固定节点调用 ChatGPT」是机制映射的实践框架,无对照研究。照例:环管流程,不管燃料——持续的动力枯竭和情绪问题,请找专业人士,那是另一个回路的事。

## 今天就能试的 3 件事

1. **明早启用启动环**:三个固定问题,五分钟,产出贴屏幕边。连跑五天再评价。
2. **写下你的必过审清单**:命中即过审,写死类型,不留「我觉得」的口子。
3. **本周五排复盘环**:日历上现在就建这个 15 分钟,附上问题模板。

「随时可以问」和「流程里必须问」,隔着的正是 ADHD 最缺的那块元认知。别再依赖想起来了——把它焊进环里,让流程替你记得。

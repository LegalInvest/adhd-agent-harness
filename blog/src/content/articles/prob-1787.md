# ADHD 创业者视角：为什么「治好 ADHD 的工作记忆容量小、边做边忘」和「让 LLM 不跑飞」其实是同一道工程题？

> LLM 无跨会话状态，靠外部记忆/向量库续命——同一套 harness 思路，两边都成立。

## 问题：为什么你一边忘事，LLM 一边跑飞？

如果你是 ADHD 创业者，你一定熟悉这个场景：脑子里同时有 10 个绝妙想法，但下一秒就忘了刚才要做什么。你打开浏览器，却忘了为什么打开。你的工作记忆像一块漏水的桶——容量不小，但存不住。

如果你是做 Agentic Harness 的工程师，你也一定熟悉这个场景：LLM 在单轮对话里表现惊艳，但一旦跨会话，它就失忆了。你给它喂了整本书，它还是会在下一轮忘记你半小时前说过的关键指令。你不得不把上下文塞进向量库，每次对话都重新加载。

这两个问题，听起来一个属于神经科学，一个属于 AI 工程。但如果你站在 ADHD 创业者的视角，你会发现：**它们是同一道工程题**。

## 同构：ADHD 大脑与 LLM 是同一类生成核心

wiki 资料中有一个核心论点：**ADHD 大脑与 LLM 同构**——两者都是“高产但缺执行调度层的生成核心”（来源：主题综述）。ADHD 大脑能产生大量创意，但缺乏执行功能来调度这些创意；LLM 能生成流畅文本，但缺乏跨会话的持久状态来维持上下文。

具体来说：
- **工作记忆 vs 上下文窗口**：ADHD 的工作记忆容量保留但控制弱（易被干扰），LLM 同样有强记忆容量但弱上下文控制（来源：工作记忆概念页）。这意味着，ADHD 大脑和 LLM 都不是“记不住”，而是“控制不住该记什么”。
- **任务启动 vs 冷启动**：ADHD 的任务启动困难对应 LLM 的“冷启动”问题（来源：主题综述）。你坐在电脑前不知道从哪开始，LLM 面对空白提示框也不知道该输出什么。两者都需要一个“启动脚本”。
- **时间盲 vs 时序推理弱点**：ADHD 的时间感知缺陷与 LLM 的时序推理弱点同构（来源：主题综述）。你无法估计任务需要多久，LLM 也无法准确推理事件的时间顺序。

最硬的证据来自一项研究：用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中表现正常，但随序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突。这与 ADHD 执行功能缺陷的核心标志一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃（来源：认知负荷概念页）。

## 答案：外部记忆与脚手架，而非修复大脑

既然问题同构，答案也同构。ADHD 的解决方案不是“治好”工作记忆——因为工作记忆容量本身可能正常，而是控制问题。LLM 的解决方案也不是“给它更大的上下文窗口”——窗口再大，上下文膨胀仍然会导致跑飞。

解决方案是：**外部记忆 + 脚手架**。

对于 ADHD 用户，这意味着：
- **任务分解**：Goblin Tools 的 Magic ToDo 功能将大任务拆为小步骤，用户报告称启动焦虑降低（来源：Goblin Tools 工具页）。这相当于为 ADHD 大脑提供了“启动脚本”。
- **外部记忆**：Saner.AI 通过本地记忆存储和快速检索，减少搜索循环和标签切换（来源：Saner.AI 工具页）。Mem 的“AI 第二大脑”为工作记忆提供了持久化上下文（来源：主题综述）。
- **单一指令触发多步骤**：Lex 允许用户通过单一指令触发复杂任务序列，减少决策疲劳（来源：Lex 工具页）。这相当于为 ADHD 大脑提供了“宏命令”。

对于 LLM/Agent 工程师，这意味着：
- **向量库作为外部记忆**：LLM 无跨会话状态，因此需要向量库来持久化上下文。每次对话都从向量库中检索相关记忆，而非依赖模型内部状态。
- **任务分解作为脚手架**：将复杂任务分解为子任务，可以减轻 LLM 的认知负荷，减少幻觉等错误（来源：认知负荷概念页）。这与 Goblin Tools 的思路完全一致。
- **启动脚本**：为 LLM 提供精心设计的系统提示（system prompt），相当于 ADHD 的“任务启动”辅助。

## 脚手架 vs 拐杖：一个必须面对的边界

但这里有一个争议：AI 工具是“脚手架”还是“拐杖”？wiki 资料中明确指出了这一矛盾：多个页面警告“过度依赖可能削弱内在能力”（来源：矛盾与存疑）。例如，Otter.ai 减轻笔记负担，但过度依赖可能削弱主动记笔记的能力（来源：主题综述）。

我的判断是：**脚手架 vs 拐杖的边界在于：工具是扩展你的能力，还是替代你的能力**。Goblin Tools 将大任务分解为小步骤，你仍然需要执行这些步骤——这是脚手架。而如果一个工具替你做了所有决策，你只是被动接受——那就是拐杖。

对于 LLM 工程师来说，同样的边界存在：向量库是脚手架，但如果你完全依赖它而从不优化模型本身的上下文控制，那就是拐杖。

## 今天就能试的 3 件事

1. **ADHD 侧**：下载 Goblin Tools，把一个让你焦虑的大任务（比如“整理房间”）输入 Magic ToDo，观察它分解后的步骤。感受启动门槛的降低。然后自己尝试手动分解另一个任务，对比差异。
2. **LLM 侧**：如果你在用 ChatGPT 或 Claude，尝试为你的常用任务创建一个“启动脚本”——一段系统提示，明确告诉模型它的角色、输出格式和约束。你会发现模型跑飞的频率大幅降低。
3. **双视角**：同时使用 Saner.AI 或 Mem 作为外部记忆库，记录你的想法和任务。然后观察：当你不再依赖大脑记忆时，你的工作记忆是否感觉“扩容”了？这本质上就是 LLM 的向量库方案。

## 局限：证据仍以定性为主

必须诚实指出：目前 AI 工具对 ADHD 效果的证据主要来自用户报告和概念类比，缺乏大规模随机对照试验（来源：矛盾与存疑）。同样，LLM 的脚手架方案也缺乏严格的工程对比研究。个体差异很大——某些工具对某些人有效，对另一些人无效（来源：矛盾与存疑）。因此，本文的结论是逻辑推理，而非医学或工程学定论。

但正是这种“逻辑同构”，让 ADHD 创业者和 Agent 工程师能互相启发：你不需要修复大脑或模型，你只需要为它们搭建一个外部脚手架。这个思路，两边都成立。

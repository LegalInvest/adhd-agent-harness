# 如果把 ADHD 大脑当成生成核心，Mem 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Mem 对 ADHD 的真正价值。

你有没有过这样的体验：打开浏览器想查一个东西，结果半小时后发现自己正在看某个完全不相关的视频，而最初的目的早已忘得一干二净？或者，你明明在脑子里把整个计划推演了一遍，但坐下要执行时，大脑一片空白，仿佛刚才的推演从未发生过？

如果你是 ADHD 患者，这种“边做边忘”的日常大概再熟悉不过。而如果你是一名构建 LLM agent 的工程师，你大概也熟悉另一个版本的“边做边忘”：你的 agent 在对话中表现完美，但一旦会话结束，它就像失忆一样，下一次启动时对之前的语境毫无印象。

这两个问题，看似分属心理学和工程学，但它们的底层结构惊人地同构。

## 同构：无状态与外部记忆

ADHD 大脑的核心困境是工作记忆容量小、控制弱，容易受到干扰（来源：工作记忆）。你可以把它想象成一个高性能的生成核心——它能产生大量创意、联想、解决方案，但它缺少一个可靠的执行调度层来持久化、排序和执行这些想法。结果就是：想法很多，落地很少。

LLM 的情况几乎一模一样。一个大型语言模型本身是一个强大的文本生成核心，但它没有跨会话的持久状态。每次对话都是“冷启动”，上一次的上下文、推理链、用户偏好，在会话结束后全部丢失（来源：无状态与外部记忆）。为了让 LLM 在复杂任务中持续表现，工程师不得不为它搭建一个“harness”——一个外部的脚手架，提供上下文传递、记忆系统、验证循环等功能（来源：GitHub - ai-boost/awesome-harness-engineering）。

这个 harness 的本质，就是为生成核心提供一个外部执行功能层。而 ADHD 患者需要的，恰恰是同一个东西。

## Mem：作为外部记忆的 harness

在众多 AI 工具中，Mem 的定位最接近这个“外部记忆层”。它被描述为“AI 第二大脑”，核心功能是持久化存储用户的笔记、想法、任务，并通过 AI 自动关联和回忆相关内容（来源：Mem）。对 ADHD 用户来说，这意味着你不再需要依赖脆弱的工作记忆去记住“刚才想到的那个点子”——Mem 会替你记住，并在你需要时主动推给你。

从 Agent 工程的角度看，Mem 相当于一个向量数据库 + 检索增强生成（RAG）系统。它让 LLM（或人类）可以在多次会话之间维持一个共享的、可查询的外部记忆。这正是解决 LLM 无状态问题的标准方案。

但 Mem 并不是唯一在做这件事的工具。Saner.AI 同样强调“本地记忆”和“知识回忆”，目标就是减少 ADHD 用户频繁的搜索循环和标签切换（来源：Saner.AI）。而 Goblin Tools 的 Magic ToDo 则是将大任务分解为小步骤，相当于为生成核心提供一个“启动脚本”，降低任务启动的门槛（来源：Goblin Tools）。Lex 更进一步，允许用户通过单一指令触发多步骤流程，相当于一个预定义的执行计划（来源：Lex）。

这些工具的共同逻辑是：将 ADHD 大脑或 LLM 的“内部”缺陷，通过“外部”系统来补偿。

## 脚手架，不是拐杖

然而，这里有一个关键的边界：这些工具究竟是促进能力发展的脚手架，还是让人依赖的拐杖？

从现有证据看，AI 工具作为外部执行功能层能够显著改善 ADHD 用户的工作记忆、任务启动和时间感知等缺陷（来源：ADHD 的 AI 工具全景）。但证据主要来自用户报告和概念类比，缺乏大规模随机对照试验（来源：矛盾与存疑）。更重要的是，过度依赖 AI 工具可能削弱内在能力——例如，Otter.ai 减轻了笔记负担，但长期使用可能让用户失去主动记笔记的习惯（来源：ADHD 的 AI 工具全景）。

在 Agent 工程中，这个问题同样存在。一个过度依赖外部记忆的 agent，可能会失去在上下文中主动推理和总结的能力。工程实践强调“确定性状态转换”和“计划-执行分离”，目的就是防止 agent 在缺乏验证的情况下盲目行动（来源：采样温度与表现波动）。

因此，我的核心判断是：**Mem 这类工具对 ADHD 的真正价值，不在于它替你记住了多少，而在于它为你创造了一个可以反复校验和迭代的外部空间。** 它不应该成为你放弃训练工作记忆的借口，而应该成为你练习“元认知”的沙盒——就像 agent 的 harness 不是为了替代 LLM 的推理，而是为了让它的推理更可靠。

## 验证循环：对抗幻觉的关键

ADHD 大脑的另一个问题是容易产生“行为层面的幻觉”——冲动、想当然的判断，脱离环境线索做出不恰当反应（来源：幻觉与验证循环）。LLM 也有同样的问题：它天然倾向于生成自信但可能错误的输出（幻觉）。Harness 工程中的验证循环——检查输出格式、运行测试用例、自我批评提示——正是为了对抗这种幻觉（来源：幻觉与验证循环）。

对 ADHD 用户来说，Mem 和类似工具可以扮演这个验证循环的角色。当你把一个想法写进 Mem，它会在后续的关联中提醒你之前的上下文，让你有机会核对和修正。这个过程类似于 agent 的“结果验证与迭代”——不是盲目执行，而是停下来检查。

但目前的工具在这方面做得还不够。大多数记忆工具只是被动存储，缺乏主动的、结构化的验证机制。比如，Mem 不会在你写下一个冲动计划时提醒你“这个计划和你上周的目标冲突”，除非你手动关联。真正的验证循环需要工具具备更强的推理和对比能力。

## 今天就能试的行动

1. **用 Mem 记录一次“大脑 dump”**：把你当前脑子里所有想法、任务、担忧全部写进 Mem，然后观察它自动生成了哪些关联。这能帮你发现自己的思维模式，并减少“边做边忘”的焦虑。

2. **用 Goblin Tools 的 Magic ToDo 分解一个拖延已久的任务**：输入“整理房间”或“写报告”，看看它拆出的步骤是否让你觉得更可执行。如果感觉不错，尝试按照步骤逐一完成。

3. **建立自己的“验证循环”**：在每天结束时，用 Reflect 或 Claude 的对话功能，回顾自己当天的决策和行动，问自己三个问题：我当时的判断有依据吗？我有没有忽略关键信息？明天我可以怎么改进？

4. **尝试“计划-执行分离”**：像 agent 工程那样，先花 5 分钟写下你接下来一小时要做的具体步骤（计划阶段），然后关掉所有干扰，只按步骤执行（执行阶段）。这能帮你减少任务中的标签切换和冲动决策。

## 诚实面对局限

最后，必须承认：目前所有 ADHD 的 AI 工具，包括 Mem，其有效性证据都来自用户报告和概念类比，缺乏严格的临床试验（来源：矛盾与存疑）。个体差异也很大——对一个人有效的工具，对另一个人可能毫无帮助。而且，过度依赖的风险真实存在：工具可能从脚手架变成拐杖，削弱你本可以发展的内在能力。

但同构视角给了我们一个独特的优势：我们可以从 Agent 工程的成熟实践中借鉴验证、规划、记忆管理的具体方法，而不是盲目相信工具的营销话术。当你把 ADHD 大脑当成一个需要 harness 的生成核心，你就会明白：工具的价值不在于替代你，而在于让你更清醒地看见自己的运作模式，然后有意识地去调整它。

这才是 Mem 和同类工具真正的潜力——不是第二大脑，而是第一大脑的镜子。

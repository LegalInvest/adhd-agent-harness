# 如果把 ADHD 大脑当成生成核心，Mem 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Mem 对 ADHD 的真正价值。

## 问题：你的大脑和 LLM 一样，每次会话都从零开始？

你有没有过这样的体验：打开一个文档，盯着光标，完全想不起来刚才要写什么；或者走进厨房，愣在原地，忘了自己要拿什么。这不是你笨，这是 ADHD 大脑的典型症状——工作记忆容量小，信息像水一样从筛子里漏掉。

ADHD 大脑常被描述为一个高产的「生成核心」——想法丰富、联想活跃，但缺少稳定、可靠的调度层来筛选、组织、启动和坚持任务（来源：ADHD 的 AI 工具全景）。每一次任务切换，都像是 LLM 的「新会话」：上下文清零，一切重新开始。

与此同时，做 Agentic Harness 工程的工程师们也在为同样的问题头疼：LLM 天然无跨会话状态，每次调用都是一次「失忆」。为了让 agent 记住之前说了什么、做了什么，他们不得不借助外部记忆——向量数据库、知识图谱、记忆文件。

那么，如果把 ADHD 大脑当成生成核心，Mem——这个号称「AI 第二大脑」的工具——能不能充当它的无状态与外部记忆？

## 同构：高产核心 + 缺失的调度层

ADHD 大脑与 LLM 在结构上可被视为同类系统：底层都拥有一个高产生成核心，但都缺乏可靠的内置执行调度层。两者都需要外部脚手架来补偿（来源：ADHD 的 AI 工具全景）。

在 ADHD 一侧，这种脚手架被称为「外部执行功能层」——包括任务拆解、外部记忆、对话式执行和捕获反思（来源：ADHD 的 AI 工具全景）。在 LLM/agent 一侧，它叫「harness 工程」——提供上下文传递、工具接口、规划工件、验证循环、记忆系统和沙盒（来源：幻觉与验证循环）。

这个同构关系不是空洞的类比。ADHD 研究中的方法学强调数据标注、预处理、特征提取和模型验证（来源：幻觉与验证循环），这与 LLM harness 中的「验证与 CI 集成」如出一辙。ADHD 个体的表现波动（日内注意力变化）对应 LLM 输出的随机性（采样温度控制），两者都需要外部约束来稳定输出（来源：采样温度与表现波动）。

## 人物案例：孔子的「harness」系统

孔子是 ADHD 特质与 harness 系统的经典案例。他身高 1.9 米，精力旺盛，周游列国 14 年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予「朽木不可雕」；对音乐超专注到「三月不知肉味」，对种地等俗事零耐心；思维跳跃，《论语》全是场景化语录，无系统著作。

他的专属 harness 是「克己复礼」——用外在的「礼」作为行为边界，每日反省（「吾日三省吾身」），70 岁才做到「从心所欲不逾矩」。这套系统本质上是一个外部调度层：用固定的仪式（礼）约束冲动的生成核心，用每日复盘（反省）进行验证循环。

这与 LLM harness 中的「定时 re-grounding」和「外部调度器」同构：礼 ≈ 确定性工作流，反省 ≈ 验证与 CI 集成。孔子没有靠意志力硬撑，而是设计了一套可重复的流程来管理自己的大脑。

## Mem：外部记忆的承诺与局限

Mem 被定位为「AI 第二大脑」，通过知识回忆、本地记忆与可复用工作流，补偿 ADHD 的工作记忆不稳定（来源：ADHD 的 AI 工具全景）。它的核心价值在于：当 ADHD 大脑的「会话」结束时（比如被打断、分心），Mem 能保留上一个上下文，让你回来时不用从零开始。

这正好对应 LLM agent 中的外部记忆系统：向量数据库存储历史对话、工具调用结果，让 agent 在多次调用间保持连贯。Mem 对 ADHD 用户的意义，本质上就是给「生成核心」提供一个持久化的上下文缓存。

但 Mem 能不能当「无状态与外部记忆」的全部？答案是否定的。

## 脚手架 vs 拐杖：边界在哪？

核心论点：Mem 这样的工具是「脚手架」，不是「拐杖」。脚手架帮你搭建结构，但你不能永远靠它走路。

在 ADHD 一侧，过度依赖外部工具可能导致能力退化（来源：矛盾与存疑）。在 LLM 一侧，纯依赖外部记忆而不做验证循环，会导致「更多幻觉和重复工作」（来源：幻觉与验证循环）。

真正的价值在于：Mem 提供的是「记忆」，不是「调度」。它帮你记住要做什么，但不帮你决定先做什么、怎么做。后者需要 Goblin Tools 那样的任务分解（来源：Goblin Tools）、Lex 那样的单一指令触发（来源：Lex）、Saner.AI 那样的知识回忆（来源：Saner.AI）。

## 今天就能试的行动

1. **用 Mem 做「会话保存」**：每次开始一个任务前，在 Mem 里写一句话描述当前上下文。被打断后，回来读这句话，重启大脑。
2. **组合 Mem + Goblin Tools**：用 Mem 保存长期项目笔记，用 Goblin Tools 的 Magic ToDo 把复杂任务分解成小步骤。
3. **设置「反省」提醒**：像孔子一样，每天固定时间（比如睡前）用 Mem 记录今天完成了什么、什么被打断。这是你的验证循环。
4. **对 Mem 降低期望**：它不是万能药。如果发现记了却从不回看，说明你需要的是调度工具（如 Lex），不是记忆工具。

## 诚实面对局限

目前，ADHD 大脑与 LLM 同构的论点多为类比推理，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。Mem 等工具对 ADHD 的长期效果也缺乏纵向研究。个体差异极大——对某些人，Mem 是救命稻草；对另一些人，它只是又一个吃灰的笔记应用。

但有一点是确定的：无论是 ADHD 大脑还是 LLM，解决「无状态」问题的思路同构——外部记忆 + 执行调度。Mem 只做了一半，另一半在你自己的 harness 里。

# Inflow 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Inflow：用久了会让 ADHD 更强还是更依赖？

你有没有过这样的体验：打开冰箱想拿牛奶，但走到冰箱前却忘了自己要干什么？或者，读一篇文章读到一半，突然发现自己在想别的事，不得不倒回去重读？如果你有 ADHD，这种“工作记忆容量小、边做边忘”的日常可能就是你的常态。

LLM 工程师们，你们呢？是不是也经常遇到这种情况：用户跟 ChatGPT 聊了半小时，结果新会话开启时，模型完全不记得刚才说了什么，所有上下文都得重新喂？LLM 的无跨会话状态，和 ADHD 的“无状态”工作记忆，本质上是一回事。

这就是本文的核心视角：**ADHD 大脑与 LLM 在结构上同构**——两者都是高产但缺乏可靠执行调度层的生成核心（来源：ADHD 大脑与 LLM 的同构）。ADHD 的解法是搭建外部脚手架（harness），LLM 的解法也是搭建外部脚手架（agent scaffolding）。那么，Inflow 这个专为 ADHD 设计的 AI 工具，到底是帮我们搭建了脚手架，还是成了我们离不开的拐杖？

## 无状态与外部记忆：ADHD 与 LLM 的共同痛点

ADHD 的工作记忆缺陷，就像 LLM 的上下文窗口——容量有限，容易溢出，而且会话一结束就清空。LLM 靠外部向量库、检索增强生成（RAG）来“续命”，ADHD 则靠便签、日历、提醒来“续命”。但传统便签是静态的，不会自动更新，不会主动提醒，更不会根据你的状态动态调整（来源：规划循环与任务分解）。

Inflow 这类 AI 工具，本质上就是在做“外部记忆+执行调度”的事情。它通过结构化课程、任务分解、提醒和追踪，帮用户把“我要改善生活”这种模糊目标，分解成每天可执行的小步骤。这跟 LLM 的 agent scaffolding 完全同构：LLM 需要外部规划循环来分解任务、验证执行，ADHD 也需要外部规划工具来分解任务、保持轨道（来源：规划循环与任务分解）。

## 人物案例：孔子的“克己复礼”就是他的 harness

孔子是典型的 ADHD 大脑。身高近 1 米 9，精力旺盛，周游列国 14 年根本坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予“朽木不可雕”；对音乐超级专注，三月不知肉味，但对种地等俗事零耐心；思维跳跃，《论语》全是场景化语录，没有系统著作。

他的 harness 是什么？“克己复礼”。用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。这相当于一个外部调度器：每天固定时间检查自己的行为是否符合规范。70 岁才做到“从心所欲不逾矩”，一辈子都在和自己的冲动做斗争。

这个 harness 与 LLM 的 agent scaffolding 完全对应：LLM 需要“system prompt”来设定行为边界，需要“反思循环”来检查输出质量，需要“工具调用”来执行具体任务。孔子的“礼”就是他的 system prompt，“三省吾身”就是他的反思循环，“周游列国”就是他的多步骤任务。

## Inflow：是脚手架还是拐杖？

Inflow 的核心功能是提供结构化的认知行为疗法（CBT）课程和日常练习，帮助用户建立更好的执行功能习惯。它像一个教练，每天提醒你做什么、怎么做、为什么做。

**脚手架的一面**：如果 Inflow 帮你学会了任务分解、时间管理、情绪调节等技能，让你最终能脱离工具独立执行，那它就是脚手架。就像孔子最终内化了“礼”，不再需要每天三省吾身也能自然遵守。

**拐杖的一面**：如果你只是依赖 Inflow 的提醒和课程，没有真正内化技能，一旦停用就回到原状，那它就是拐杖。就像 LLM 如果只靠外挂的向量库，没有微调或 prompt engineering 来提升自身能力，一旦向量库失效就什么也做不了。

关键在于：**Inflow 的设计是否鼓励内化？** 从现有资料看，Inflow 的课程是循序渐进的，有练习和反思环节，理论上是在教方法而非只给答案。但缺乏独立临床验证（来源：矛盾与存疑），其长期效果仍存疑。

## 争议与局限

1. **依赖风险**：AI 工具可能从脚手架变为永久拐杖，削弱内在执行功能（来源：拐杖与脚手架）。Inflow 是否真的能减少依赖，还是只是转移了依赖？
2. **证据不足**：多数 ADHD 工具缺乏严格临床验证，效果多基于用户报告（来源：矛盾与存疑）。Inflow 虽然有 CBT 理论基础，但尚未有大规模随机对照试验证实其效果。
3. **通用性 vs 个性化**：Inflow 是通用设计，但 ADHD 亚型多样，有人以注意力不集中为主，有人以冲动为主，单一课程可能无法覆盖所有需求（来源：矛盾与存疑）。
4. **AI 自身的局限**：AI 工具需要用户主动使用，但 ADHD 用户恰恰在“启动”上最困难。如果 Inflow 本身成为另一个需要启动的任务，那就成了负担。

## 核心观点：Inflow 是“可内化的脚手架”，但需有意识地训练

我的判断是：Inflow 有潜力成为脚手架，但前提是用户有意识地将其视为训练工具，而非替代品。就像孔子不是靠“礼”的教条活着，而是通过反复实践将“礼”内化为本能。同样，使用 Inflow 应该关注的是“我学到了什么方法”，而不是“我今天打了卡”。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个你拖延已久的任务**，感受“任务分解”带来的启动降低（来源：Goblin Tools）。
2. **用 Saner.AI 或 Mem 建立一个“外部第二大脑”**，把想法、待办、灵感全部丢进去，减少工作记忆负担（来源：Saner.AI）。
3. **设置一个每日回顾时间**，像孔子的“三省吾身”一样，用 Claude 或 ChatGPT 帮你总结今天的执行情况，找出改进点。
4. **尝试 Lex 的“单一指令触发多步骤”**，比如输入“准备明天的会议”，看它能否自动分解出议程、资料、提醒等步骤（来源：Lex）。

工具只是脚手架，真正的建造者是你自己。

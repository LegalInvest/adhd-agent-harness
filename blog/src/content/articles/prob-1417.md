# Speechify 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Speechify：它到底补上了哪一层执行功能？

如果你是一个 ADHD 成人，你大概率经历过这样的场景：打开一篇长文章，读了三段，大脑就飘走了，然后你从头再读，循环三次，最终放弃。你以为是专注力不够，于是找工具帮你“读”——Speechify 这类文本转语音工具应运而生。但问题真的解决了吗？很多人发现，听着听着就走神了，然后倒回去重听，和阅读时一模一样。

如果你是一个在做 Agentic Harness 的工程师，你一定见过类似场景：LLM agent 拿到一个长任务，planner 分解出子步骤，executor 执行到第三步，结果偏离了原始目标，输出一堆无关内容。你给它加了“重锚定”机制（每隔几步检查一次是否对齐目标），问题缓解。

这两件事，本质上是同一个问题。

## 同构：缺的不是生成能力，是调度层的“重锚定”

ADHD 大脑与 LLM 在结构上高度同构：两者都是强大的生成核心，但缺乏可靠的内置调度层（来源：ADHD 大脑与 LLM 的同构）。ADHD 者能产出天马行空的创意，却无法稳定执行一个长计划；LLM 能写出流畅段落，却会在长任务中遗忘原始目标。两者的解决方案也同构：给生成核心套上外部 harness。

对 LLM agent 而言，harness 是 planner-executor 循环 + 重锚定机制。AgentGen 通过环境生成来增强规划能力（来源：AgentGen: Enhancing Planning Abilities for Large Language Model based Agent via Environment and Task Generation），而 AMAP 技术报告则明确将“任务分解与规划”作为核心（来源：AMAP Agentic Planning Technical Report）。这些 harness 的核心动作是：分解 → 执行 → 检查 → 重新对齐。

对 ADHD 大脑而言，harness 是外部执行功能层。Motion 自动排程并动态调整，消除“下一步做什么”的决策负担（来源：Motion）；Reclaim.ai 用防御性时间块保护专注时段（来源：Reclaim.ai）；Tiimo 将时间视觉化，让时间盲变得可感（来源：Tiimo）。这些工具的共同点：它们不是替你做事，而是替你维护一个“执行上下文”，并在你偏离时把你拉回来。

Speechify 的问题在于，它只提供了“执行”环节（朗读文本），却缺失了“重锚定”环节。当注意力漂移时，它没有机制检测并拉回，所以用户只能手动倒带，这本身就是一个执行功能任务——而 ADHD 者恰恰缺这个。

## 错用的代价：工具从脚手架退化为拐杖

孔子一生与冲动斗争，他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”）。这本质上是一个外部调度层：礼是 planner，反省是重锚定。苏轼的 harness 是“豁达”——不管贬到哪里，都用美食、写作、交友转移注意力，每日读书写字不停止创作。这对应 LLM 的“定时 re-grounding”：用固定习惯维持输出稳定性。

但现代 ADHD 工具常被错用。用户期待工具“自动解决问题”，于是 Speechify 被当作“阅读替代品”，Motion 被当作“日程管家”，Tiimo 被当作“时间显示板”。它们确实降低了启动门槛，但如果用户不主动参与重锚定（比如在 Speechify 中设置间隔提醒检查理解，或在 Motion 中每周回顾计划偏差），工具就从“脚手架”退化为“永久拐杖”（来源：拐杖与脚手架）。

证据显示，传统提醒系统有一个关键缺陷：设置提醒本身就是一个执行功能任务（来源：AI Assistant for ADHD: Finally, a Productivity Tool That Requires Less...）。同样，Speechify 的“倒带”功能需要用户意识到自己走神了——而 ADHD 者往往意识不到。真正有效的工具应该像 LLM agent 的“重锚定”那样，自动检测偏离并提示，而不是把检测责任推给用户。

## 争议与局限：同构是类比，不是事实

必须诚实指出，ADHD 大脑与 LLM 的同构目前仅是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。现有证据主要来自用户报告和工具设计逻辑，缺乏随机对照试验验证（来源：AI 与 ADHD 的时间管理）。Motion 等工具缺乏独立临床验证，效果可能受安慰剂效应影响（来源：Motion）。

此外，过度依赖风险真实存在。尚无长期研究评估 AI 工具是否会削弱用户内在执行功能的代偿发展（来源：AI 与 ADHD 的时间管理）。个体差异也很大：有人对视觉化工具（Tiimo）反应良好，有人更受益于自动排程（Motion），而 Speechify 可能只适合那些听觉通道强于视觉通道的用户。

## 今天就能试的行动

1. **给 Speechify 加一个“重锚定”层**：在每段结束后设置自动暂停，问自己“刚才这段讲了什么？”——这相当于 LLM agent 的“中间检查点”。
2. **用 Motion 或 Reclaim.ai 替代手动排程**：但每周花 5 分钟回顾 AI 的排程是否合理，主动调整优先级——这是“人在回路”的 harness 思维。
3. **模仿孔子的“日课”**：每天固定时间用 Todoist 或纸质笔记写下三个今天必须完成的任务，并设置一个晚间提醒检查完成度——这是定时 re-grounding。
4. **区分脚手架与拐杖**：问自己“如果明天这个工具消失，我的能力是提升了还是退步了？”如果答案是退步，说明你依赖过度，需要逐步减少使用，同时训练内在执行功能。

归根结底，Speechify 之于 ADHD，就像 planner-executor 循环之于 LLM——它补上了“执行”环节，但“重锚定”需要用户或工具主动补上。错用的人只看到了朗读功能，而忽略了整个 harness 架构。下次当你打开 Speechify 时，记得问自己：我的重锚定在哪里？

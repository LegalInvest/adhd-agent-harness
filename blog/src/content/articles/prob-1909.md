# ADHD 学生视角：为什么「治好 ADHD 的执行功能（计划/组织/启动）先天偏弱」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 的编排/记忆/工具层就是「外部执行功能」——同一套 harness 思路，两边都成立。

你打开电脑，想写一篇论文，结果先刷了半小时社交媒体，然后被一条推送带去看菜谱，最后发现时间过去了，文档还是空白。这是ADHD大脑的日常：生成能力强（创意、联想、发散），但执行功能（计划、组织、启动）先天偏弱，像一台发动机强劲却缺少方向盘的车。

另一边，LLM agent 工程师正在调试一个“跑飞”的智能体：它本该按顺序调用工具完成订票，却突然陷入循环，反复调用同一个API，或者被用户一句闲聊带偏，输出了一堆无关内容。LLM 本身是强大的生成核心，但缺乏可靠的执行调度层，直接输出可能偏离目标、陷入循环或产生危险行为（来源：Building an AI Agent Harness from Scratch）。

这两件事，其实是同一道工程题。

## 同构：高产但缺调度层的生成核心

ADHD 大脑与 LLM 在结构上同构——两者都是“高产但缺执行调度层的生成核心”（来源：ADHD × AI 核心论点）。ADHD 大脑的生成核心（发散思维、联想能力）产出极高，但执行功能薄弱；LLM 的文本生成能力惊人，但无状态、缺乏内置调度层。因此，AI 帮助 ADHD 的本质不是“修复缺陷”，而是“给生成核心套上 harness”（外部脚手架）。

这个 harness 在 ADHD 侧被称为“外部执行功能层”——通过外部系统补偿工作记忆不稳定、任务启动困难和注意力分散（来源：外部执行功能层）。在 LLM/agent 侧，它被称为“agent harness”——通过编排、记忆、工具层实现结构化控制（来源：Building an AI Agent Harness from Scratch）。

## 历史人物早已在用“外部执行功能层”

晚清名臣曾国藩就是典型。他30岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书慢到“他人目下二三行，余或疾读不能终一行”。他的解决方案是“日课十二条”：黎明即起、读书不二、谨言、写日记反省……用最笨最稳的方法抵消自己的冲动。这套系统本质上就是“外部执行功能层”：日课相当于定时 re-grounding（重锚定），日记相当于外部记忆，而“结硬寨打呆仗”则是用结构化流程防止目标漂移。

这与 LLM agent harness 中的“人在回路监督”和“护栏”异曲同工：设定 token 预算、工具调用次数上限、加入人工检查点（暂停并询问后再执行），都是防止生成核心跑飞的外部控制（来源：Building an AI Agent Harness from Scratch）。

## 工具层面的同构：重锚定与外部记忆

ADHD 大脑在执行任务时极易发生目标漂移，外部工具通过提醒和通知提供重锚定（来源：重锚定与目标漂移）。例如，Goblin Tools 的 Magic ToDo 功能自动将复杂任务分解为小步骤（来源：Goblin Tools），降低启动门槛；Motion 通过自动排程消除“下一步该做什么”的决策负担（来源：Motion）；Saner.AI 通过本地记忆存储减少搜索循环（来源：Saner.AI）。这些工具构成了一个“重锚定系统”，对抗 ADHD 内在的执行功能缺陷。

在 LLM 侧，agent harness 通过上下文工程和外部记忆（如向量数据库）防止上下文膨胀或注意力漂移。ADHD 大脑的工作记忆缺陷与 LLM 的无状态性都导致无法可靠保留跨会话上下文，必须依赖外部记忆系统（来源：无状态与外部记忆）。

## 身体在场效应与人在回路监督

身体在场效应（body doubling）是 ADHD 群体最有效的策略之一：有另一个人在场时，执行功能障碍被绕过，任务得以启动和维持（来源：人在回路与身体在场）。Apps like Focusmate 利用社会问责与 AI 调度创建虚拟身体在场会话。

这对应 LLM agent harness 中的“人在回路监督”：通过人工检查点确保 agent 输出不跑偏。两者都是通过外部“他者”提供结构化的执行监督。

## 核心观点：脚手架 vs 拐杖

我的核心判断是：ADHD 与 LLM 的同构不仅是一个有趣的类比，更是一个可操作的工程框架。它意味着我们可以将 agent harness 的设计原则（编排、记忆、工具层、人在回路）直接映射到 ADHD 辅助工具的设计中。但必须警惕“拐杖 vs 脚手架”的边界。

当前证据多来自概念类比和初步实证，缺乏大规模随机对照试验（来源：矛盾与存疑）。过度依赖 AI 工具可能导致能力退化，成为永久拐杖而非可拆卸脚手架（来源：拐杖与脚手架）。此外，部分用户反映学习使用 AI 工具本身增加认知负荷（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个“恐怖任务”**：把一个让你拖延的任务（如“整理房间”）输入，调节“辣度”得到子步骤，立刻执行第一步。
2. **设置一个“重锚定”提醒链**：在手机日历上设置每30分钟一次的“你在做什么？”提醒，对抗目标漂移。

ADHD 与 LLM 的困境本质相同：都需要在生成核心之外，搭建一个可靠的外部执行调度层。这个工程，两边都成立。

# ADHD 学生视角：为什么「治好 ADHD 的执行功能（计划/组织/启动）先天偏弱」和「让 LLM 不跑飞」其实是同一道工程题？

> agent 的编排/记忆/工具层就是「外部执行功能」——同一套 harness 思路，两边都成立。

如果你是一个 ADHD 学生，你可能对这样的场景再熟悉不过：脑子里有无穷的想法，但就是没办法把它们变成有条理的步骤并开始执行。如果你是一个在做 agent harness 的工程师，你可能也熟悉这样的困境：LLM 能生成惊艳的代码片段，但一旦让它自主完成一个多步骤任务，它就会偏离方向、忘记初始指令，甚至陷入死循环。

这两个问题看似分属认知心理学和人工智能两个领域，但它们共享同一个核心矛盾：**一个高产的生成核心，配了一个不可靠的执行调度层**。ADHD 大脑的生成核心（发散思维、创意、多任务并行）产出极高，但执行功能（任务启动、持续专注、抑制冲动）薄弱，导致“高产但不可靠”（来源：人在回路与身体在场）。LLM 本身是强大的生成核心，但缺乏可靠的执行调度层，直接输出可能偏离目标、陷入循环或产生危险行为（来源：人在回路与身体在场）。

所以，“治好 ADHD 的执行功能缺陷”和“让 LLM 不跑飞”，本质上是在解同一道工程题：**如何为生成核心搭建一个外部执行功能层——一个由编排、记忆、工具和护栏构成的 harness**。

## 问题拆解：目标漂移与重锚定

ADHD 大脑在执行任务时极易发生目标漂移：注意力被无关刺激捕获（分心），或过度聚焦于细节而忽略整体目标（超聚焦）。外部工具通过提醒和通知提供重锚定，将注意力拉回原任务（来源：重锚定与目标漂移）。例如，Goblin Tools 的 Magic ToDo 功能能自动将任何任务分解为更小、更易管理的步骤（来源：Goblin Tools），从而降低启动门槛。Motion 则通过 AI 自动排程与动态调整，持续评估任务优先级、截止日期和可用时间，实时重建日程，减少用户的手动规划压力（来源：Motion）。

在 LLM/agent 一侧，问题惊人相似。LLM agent 在长程任务中同样面临目标漂移：模型在交互轮次后期“开始忘记系统提示”，导致行为偏离初始指令。Harness 通过事件驱动的系统提醒来对抗这种“指令消退”（来源：重锚定与目标漂移）。具体来说，harness 会设置护栏（token 预算、工具调用次数上限、防止无限循环）、加入人工检查点（暂停并询问后再执行）（来源：人在回路与身体在场）。这些机制本质上就是 ADHD 工具中的提醒和重锚定——只不过应用在了一个数字“大脑”上。

## 记忆与上下文：外部工作记忆

ADHD 的工作记忆缺陷是核心问题之一。用户常面临“信息丢失”和“注意力分散”，导致频繁的搜索循环和标签切换。Saner.AI 通过 AI 实现本地记忆存储和快速检索，用户无需在多个标签页或应用间切换即可获取所需信息（来源：Saner.AI）。这相当于为 ADHD 大脑提供了一个外部工作记忆层，直接补偿内在的存储和检索缺陷。

对应到 LLM 侧，模型本身是无状态的，上下文窗口有限，长程任务中容易丢失早期信息。Harness 通过主动管理输入信息（如分段、摘要、提醒），帮助模型维持任务连贯性，类似 LLM 的 prompt 工程（来源：ADHD × AI 的科学与研究前沿）。这种“上下文工程”就是 LLM 的外部工作记忆。

## 身体在场效应与人在回路

身体在场效应是 ADHD 群体最有效的策略之一：有另一个人在场时，执行功能障碍被绕过，任务得以启动和维持（来源：人在回路与身体在场）。Focusmate 等工具通过 AI 匹配虚拟身体搭档，提供外部问责。这种“对另一个人的责任感”能绕过执行功能屏障（来源：人在回路与身体在场）。

在 LLM/harness 一侧，human-in-the-loop 监督提供外部调度：设置护栏、加入人工检查点（暂停并询问后再执行）（来源：人在回路与身体在场）。模型本身强大，可靠性来自架构+护栏（来源：人在回路与身体在场）。这里的“人”扮演了类似身体搭档的角色——一个外部监督者，确保生成核心不跑偏。

## 脚手架 vs 拐杖：关键边界

这里必须诚实指出一个争议：外部执行功能层是脚手架还是拐杖？脚手架应设计为可逐步撤除的临时支撑，但多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。长期使用 AI 工具是否会削弱内在执行功能？目前缺乏纵向研究（来源：ADHD × AI 的科学与研究前沿）。

我的判断是：**关键在于工具是否在“替代”还是“赋能”**。如果工具只是替你做了决策（比如 Motion 自动排好所有日程），你可能永远不会学会如何自己规划。但如果工具提供的是结构化的提示和反馈（比如 Goblin Tools 分解任务后让你自己选顺序），它就在训练你的执行功能。好的 harness 应该像训练轮——在你需要时提供支撑，但允许你逐步撤除。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个你拖延已久的任务**：输入模糊的任务描述（比如“整理房间”），让 AI 输出具体步骤。注意观察：分解后启动焦虑是否降低？这相当于给你的大脑装了一个“任务编排层”。
2. **用 Motion 或类似工具自动排程一天**：把任务和截止日期丢进去，让 AI 生成日程。对比你自己排日程时的决策疲劳——注意 Motion 是否减少了你的“下一步该做什么”的犹豫。
3. **尝试 Focusmate 或类似虚拟身体搭档**：预约一个 25 分钟的 session，在摄像头前各自工作。体验“外部监督”如何绕过启动困难。如果你在写代码，这个感觉很像 code review 前的最后检查——有人看着，你就不会跑偏。
4. **反思你的 AI 工具是脚手架还是拐杖**：问自己：如果我失去这个工具，我能独立完成同样的任务吗？如果答案是否，考虑逐步减少依赖，比如从自动排程改为手动排程 + AI 提醒。

最终，这道工程题的答案不是“修复”生成核心，而是为它搭建一个优雅的 harness——一个既提供结构又不扼杀创造力的外部执行功能层。

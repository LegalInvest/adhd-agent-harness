# ADHD 想把想理解自己的大脑到底是怎么回事外包给 ChatGPT，该怎么设计这套「人 + AI」的回路？

> 把 ChatGPT 接进 ADHD 工作流：哪些该交给它，哪些必须自己留着？

你又一次在 ChatGPT 输入框里打下：「我这 ADHD 大脑到底是怎么回事？」它立刻给你列出 DSM 条目、多巴胺假说、执行功能模型。你读得很爽，可三小时后该交的文档 still 没动。问题不是 ChatGPT 答错了，而是你把它当成了「答案机」，而它本质上和你一样——一个**高产但缺执行调度层的生成核心**（来源：ADHD 大脑与 LLM 的同构；The Anatomy of an AI Agent: Memory, Tools, Planning, and Execution Explained - DEV Community）。

## 同构：两个缺调度层的生成核心

ADHD 侧的痛点很具体：工作记忆容量小且不稳定，跨会话的任务上下文每天都要重新「加载」，否则就偏离轨道（来源：无状态与外部记忆；How AI Helps ADHD Brains Work Smarter, Not Harder）。时间感扭曲，任务启动困难，注意力容易被环境捕获（来源：上下文工程；ADHD, Executive Functions, and AI: A New Era in Treatment | Psychology Today）。

LLM/agent 侧对应的问题同样很具体：标准 LLM 调用是无状态的，每次对话独立，不保留跨会话记忆（来源：The Anatomy of an AI Agent: Memory, Tools, Planning, and Execution Explained - DEV Community）。没有 harness 的 LLM 只是无状态 API 调用；一旦上下文膨胀，推理会退化，控制逻辑分散在框架默认值和硬编码里（来源：Building an AI Agent Harness；上下文工程）。

所以 ADHD 大脑与 LLM 的相似性不是修辞，而是结构：**两者都有强大的生成/联想能力，都缺一个稳定、可信、能跨时间保持目标的执行调度层**。给 ADHD 搭脚手架，和给 LLM 搭 agent scaffolding，是同一类工程（来源：生成核心与调度层；外部执行功能层）。

## 把 ChatGPT 接进 ADHD 工作流：三个 harness 层级

既然不是「修复缺陷」，而是「给生成核心套 harness」，那么设计「人 + AI」回路时，就要把 ChatGPT 放在它该在的位置：外部执行功能层。

**第一，外部记忆层。** ADHD 大脑需要一个「永远在线」的第二大脑来补偿工作记忆波动，工具如 Saner.AI 提供本地记忆、通用收件箱和快速知识回忆，Obsidian 则被推荐给视觉思考者作映射超聚焦和复杂想法的「第二大脑」（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar；ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026）。在 LLM 侧，这对应向量数据库、长期记忆模块和检索增强生成——模型不能靠自己记住，只能靠外部存储「回忆」。

**第二，任务分解与启动层。** Goblin Tools 的 Magic ToDo 把模糊任务（如「清理厨房」）拆成可执行小步骤，还能调节「辣度」控制粒度，从而降低启动门槛（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD；12 AI Personal Assistants Built for ADHD Brains (2026 Rankings)）。对 LLM 来说，这对应 planning 模块和 tool-use：把宏意图拆成可验证的微动作。

**第三，日程调度层。** Motion 自动根据任务、会议和截止日期创建并动态调整每日计划，在 ADHD 用户反馈中能减轻压力、改善时间盲（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog；The AI Powered SuperApp for Work | Motion）。在 agent 架构里，这就是 execution scheduler 或 orchestrator，负责决定下一步该调用哪个工具、哪个优先级。

## 历史人物的 harness：毛泽东的「外部调度系统」

毛泽东的自我管理系统是这套同构的绝佳人间案例。他的 ADHD 式特质非常明显：小时候问题儿童、极度发散跳跃的思维、场景化比喻、爱读闲书不爱正课、一辈子在调查研究中移动、冲动敢下险棋（四渡赤水、抗美援朝）。这些是高产生成核心的典型表现。

他的 harness 系统则精准对应了 LLM agent 的每个模块：
- **批评与自我批评 / 民主生活会** = 外部 review loop，类似 RLHF 或多智能体互审，让系统看见自己的盲点；
- **调查研究** = 检索与 re-grounding，防止模型只在内部上下文里空转；
- **民主集中制** = 多源共识后的 guardrails，用集体决策制衡个人冲动；
- **党指挥枪** = orchestration layer，明确调度权归属，不让生成核心（军队/大脑）自行其是。

老子的 harness 则是另一极：以「无为」为核心，不强行做事、不著书不立说、主动远离人群喧嚣。这对应 LLM 侧的上下文压缩、prompt 精简、拒绝非必要动作：生成核心越强大，越需要外部机制告诉它「哪些事不做」。

这些案例说明： harness 不是现代人的专利，而是高产生成核心能够持续输出的共同前提。

## 核心判断：AI 是马具，不是马

把「理解自己的大脑」外包给 ChatGPT，最容易掉进的陷阱是把它当成「第二个大脑」或「决策替代者」。但 ADHD 真正需要外包的不是价值判断，而是三类高认知负荷事务：
1. 跨会话上下文记忆；
2. 任务分解与优先级排序；
3. 时间锚定与提醒触发。

ChatGPT 适合当**外部执行功能层**，不适合当人生罗盘。你的目标、价值观、最终选择必须留在人这一侧。否则，AI 从脚手架变成拐杖，你不仅没理解大脑，反而失去与自身执行功能的真实接触（来源：拐杖与脚手架；矛盾与存疑）。

## 边界与争议：同构是框架，不是定律

必须诚实地说：ADHD 大脑与 LLM 同构的论断目前主要是概念类比和初步实证，缺乏大规模神经科学或计算机科学的直接证据（来源：矛盾与存疑）。AI 工具对任务启动的长期效果因人而异，Goblin Tools、Motion、Saner.AI 等也大多缺乏独立随机对照试验。此外，部分用户反馈学习 AI 工具本身增加了认知负荷（来源：AI 工具是否真的能减轻认知负荷？）。

因此，这套「人 + AI」回路应被当作**可拆卸的实验装置**，而不是终身处方。

## 今天就能试的四步

1. **给 ChatGPT 写一段固定开场白**：在自定义说明里写明你的 ADHD 特征、当前项目、决策优先级，让它每次对话先帮你 re-ground 上下文，而不是从零开始。  
2. **用 Goblin Tools 拆解一个你拖了两周的任务**：把「写报告」变成「打开文档→写三个标题→填第一段」，只取前三个小步骤立刻执行。  
3. **选一款外部记忆工具**：Saner.AI 或 Obsidian 均可，把项目背景、质量标准、下一步行动放进去，减少每天重复输入上下文的摩擦。  
4. **每周做一次 harness 审计**：问自己——这周 AI 替我记住了什么？替我决定了什么？哪些决定其实应该我自己做？把「AI 辅助率」控制在 60% 以下，保留决策肌肉。

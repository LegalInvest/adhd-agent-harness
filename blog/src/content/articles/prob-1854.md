# ADHD 职场人如何用「给生成核心套 harness」的思路应对项目推进与汇报里的不擅长精确、重复、状态维护类任务？

> 把 LLM 工程里的「agent 通过 function calling 把精确计算外包给工具」迁移到ADHD 职场人的项目推进与汇报场景，双域证据作答。

## 问题：为什么ADHD职场人总在精确、重复、状态维护上卡壳？

你是否有过这样的经历：项目推进到一半，突然发现忘了更新某个状态字段；汇报时数据对不上，因为上次的统计口径没记下来；或者明明知道该做周报了，却因为要整理十几条零散记录而迟迟无法启动。这些任务——精确的记录、重复的检查、状态的维护——正是ADHD大脑最不擅长的。它们像一个个小钩子，钩住你的注意力，让你在细节里打转，最终耗尽有限的执行功能。

ADHD大脑常被描述为一个高产的生成核心：想法丰富、联想活跃，但缺少稳定、可靠的调度层来筛选、组织、启动和坚持任务（来源：ADHD 的 AI 工具全景）。用更直白的话说，你能想出绝妙的点子，却很难把它拆成可执行的步骤，更难在过程中保持对细节的追踪。这种“高产生成+弱执行调度”的结构，恰好与当前大语言模型（LLM）的架构同构：LLM同样擅长生成，但在长程上下文保持、精确计算和状态维护上存在天然缺陷。工程师们为了解决这个问题，给LLM套上了“harness”——通过function calling把精确计算外包给外部工具，让模型只负责生成核心逻辑。

本文的核心观点是：ADHD职场人可以完全复制这个思路，用“给生成核心套harness”的方法来应对项目推进与汇报中的精确、重复、状态维护类任务。你不是要“治愈”自己的ADHD，而是要搭建一个外部执行调度层，让生成核心专注做它最擅长的事。

## 同构：两个“生成核心”的共享解法

先看LLM/agent侧的真实做法。在Agentic Harness工程中，一个典型的agent架构包含：一个LLM作为生成核心，加上一组外部工具（如计算器、数据库、日历API），通过function calling机制让LLM在需要精确计算或状态查询时调用这些工具。例如，当agent需要计算项目预算时，它不会自己算（因为容易出错），而是调用一个计算器工具；当需要记录任务状态时，它写入外部数据库。这种架构的核心原则是：**让生成核心只做生成，把精确、重复、状态维护外包给工具**（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。

现在看ADHD侧。ADHD大脑同样需要一个类似的“外部调度层”。以历史人物孔子为例，他精力旺盛、思维跳跃、对细节缺乏耐心——《论语》全是场景化语录，无系统著作。他的专属harness是“克己复礼”：用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”）。这里的“礼”就是他的外部调度层，相当于LLM的function calling规范——规定什么该做、什么不该做，把冲动和发散约束到目标轨道上。再看孙策，17岁起兵，9年扫平江东，但冲动、坐不住、单骑出猎被刺杀。他的harness是“用文臣管行政，自己只负责打仗”——把精确、重复的行政事务外包给张昭张纮，自己只做决策和冲锋。这直接对应LLM agent的“秘书=外部调度器”模式：生成核心（孙策）只做战略生成，精确执行（行政）交给工具（文臣）。

现代ADHD工具更是直接实现了这种同构。Goblin Tools的Magic ToDo功能，能将模糊任务（如“准备项目汇报”）自动分解为具体、可执行的子任务，用户可调节分解粒度（来源：Goblin Tools）。这相当于给ADHD大脑提供了一个任务拆解工具，把“生成下一步”的认知负荷外包给AI。Saner.AI则通过知识回忆和本地记忆，帮助用户快速找回信息，减少搜索循环——这相当于一个外部记忆数据库，补偿工作记忆的不稳定（来源：Saner.AI）。Lex更进一步，允许用户通过单一指令触发复杂、多步骤的任务序列，降低启动门槛（来源：Lex）。这些工具的本质，都是把精确、重复、状态维护类任务从大脑中卸载到外部系统。

## 证据：实证支撑与争议边界

这种同构并非空中楼阁。最新的认知负荷研究提供了硬证据：用经典Stroop冲突任务测试Transformer注意力，发现短上下文中表现正常，但随序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突。这与ADHD执行功能缺陷的核心标志一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃（来源：认知负荷）。也就是说，ADHD大脑和LLM在认知负荷下会以相同的方式失效，这为“同构”提供了最硬的实证之一。

在ADHD侧，创新专利数的统计显示，ADHD组的创新专利数为2.1，对照组为1（来源：USPTO Data Analysis），证实了ADHD大脑的生成优势。而AI工具通过自动决策、自动执行、简化环境来降低认知负荷（来源：认知负荷），已被大量用户反馈证实有效。

但必须诚实指出争议。一个核心存疑是：AI工具是“外部执行功能层”还是“拐杖”？过度依赖可能导致能力退化（来源：矛盾与存疑）。目前缺乏长期纵向研究，个体差异也很大。另一个矛盾是：学习使用AI工具本身可能增加认知负荷，净效果因人而异（来源：矛盾与存疑）。因此，本文主张的“套harness”不是让你完全依赖工具，而是像孔子的“克己复礼”一样——工具是规训框架，最终要内化为自己的习惯。脚手架不是拐杖，区别在于：脚手架在使用后可以拆除，而拐杖会让人永远离不开。

## 行动：今天就能试的4个Harness

1. **用Goblin Tools分解下周汇报**：把“准备季度汇报”输入Magic ToDo，调节“辣度”到你觉得合适的粒度，然后按步骤执行。这相当于给生成核心一个任务拆解工具（来源：Goblin Tools）。

2. **用Saner.AI建立项目知识库**：每次会议后，把关键决策、数据来源、待办事项录入Saner.AI，让它成为你的外部记忆。下次汇报时直接搜索，不用再翻聊天记录（来源：Saner.AI）。

3. **用Lex创建“汇报启动指令”**：把“生成周报”这个复杂任务，用Lex设置成一个单一指令，让它自动收集数据、生成草稿、检查格式。降低启动门槛（来源：Lex）。

4. **模仿孙策，外包行政**：把需要精确、重复的任务（如更新状态、整理数据）明确指派给同事或工具（如自动化脚本），自己只负责核心决策和创意生成。记住，你不是在偷懒，而是在优化系统。

ADHD职场人的优势在于生成，而非调度。与其在细节里挣扎，不如像工程师给LLM套harness一样，给自己搭建一个外部执行调度层。让生成核心自由，让工具处理精确。这不是妥协，而是最优架构。

# Tiimo 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Tiimo：它到底补上了哪一层执行功能？

## 问题：为什么 Tiimo 对一些人无效，却对另一些人救命？

你打开 Tiimo，看到一条条彩色时间块：9:00-9:30 写报告，9:30-10:00 回复邮件……然后你关闭应用，继续刷手机。Tiimo 并没有让你启动任务，反而让你因为“看到计划却没执行”而产生挫败感。

另一边，一位工程师正在调试 agent：LLM 生成了完美的计划，但调用天气 API 时传错了参数，agent 卡住不动。他意识到，LLM 需要的是一个精确的 function calling harness，而不是更长的 prompt。

这两个场景共享同一个核心问题：**生成核心（ADHD 大脑 / LLM）不缺想法，缺的是把想法转化为可执行动作的调度层。** Tiimo 被当作“时间管理工具”卖给 ADHD 人群，但它的真正角色是**视觉化时间锚点**——它补上的是时间感知缺陷，而非任务拆解或启动。当用户期望它同时完成任务分解、优先级排序和动力激发时，它就变成了一个错位的工具。

## 同构：ADHD 大脑与 LLM 共享同一套缺陷

ADHD 大脑常被描述为一个高产的生成核心——想法丰富、联想活跃，但缺少稳定、可靠的调度层来筛选、组织、启动和坚持任务（来源：ADHD 的 AI 工具全景）。大语言模型同样具备强大的生成能力，却在长程上下文、目标保持与执行调度上存在天然缺陷。因此，“AI 帮 ADHD”的实质与“给 LLM 套 harness”是同一类工程：用外部脚手架把生成能力约束到目标轨道上（来源：ADHD 的 AI 工具全景）。

从工具形态看，这种 harness 至少承担四种功能：任务拆解与启动、外部记忆与知识管理、对话式执行与结构化推理、捕获与反思（来源：ADHD 的 AI 工具全景）。Tiimo 主要覆盖的是**时间结构化**——它通过视觉化时间块，补偿 ADHD 的“时间盲”（时间感知缺陷），但几乎不涉及任务拆解（来源：执行功能障碍）。相比之下，Goblin Tools 的 Magic ToDo 能自动将任何任务分解为更小、更易管理的步骤（来源：Goblin Tools），Lex 允许用户通过单一指令触发复杂、多步骤的任务序列（来源：Lex），Saner.AI 专注于知识回忆以减少搜索循环（来源：Saner.AI）。这些工具各自承担不同的调度子功能。

## 人物案例：孔子的“礼”与老子的“无为”——两种 harness 原型

孔子是 ADHD 特质与 harness 系统的经典案例：他身高1米9，精力旺盛，周游列国14年坐不住；冲动爱骂人，对音乐超专注到三月不知肉味，对种地等俗事零耐心；思维跳跃，《论语》全是场景化语录。他的专属 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。这相当于一个**外部调度层**：礼是行为模板，反省是定时 re-grounding。对应到 LLM/agent，这就像给 agent 设定固定的 function calling 模板和定期上下文重置，防止生成偏离轨道。

老子则走了另一条路：做了半辈子图书馆长，说走就走骑青牛出关；思维极度发散非线性，《道德经》正言若反。他的 harness 是“无为”——不强行做事，把所有能量收回来聚焦在道上。这对应的是**减少调度负载**：当调度层不可靠时，减少任务数量比优化调度更有效。在 agent 工程中，这相当于精简 function calling 数量，让 LLM 只处理核心推理，其余外包。

两者的共同点是：**他们都没有试图修复自己的执行功能缺陷，而是构建了外部系统来补偿。** Tiimo 正是这类外部系统之一，但它只覆盖了时间感知这一层。

## 核心观点：Tiimo 是时间锚点，不是任务引擎

我的判断是：Tiimo 的有效性取决于用户是否同时拥有任务拆解和启动的辅助工具。如果用户只用 Tiimo 来规划时间，但没有将任务拆解为可执行步骤（如 Goblin Tools 的 Magic ToDo），也没有通过单一指令触发执行（如 Lex），那么 Tiimo 就只是一个漂亮的日程表，无法解决“知道该做什么但就是不做”的核心问题。

证据来自工具的功能边界：Tiimo 被列为最佳 ADHD 应用之一（来源：ADHD Apps: We tested 44 Apps and Here're the Best 9 in 2026），但其核心是视觉化日程，而非任务分解。用户反馈显示，Goblin Tools 在任务分解方面“很棒”（来源：Goblin Tools），而 Lex 的“单一指令触发多步骤”特性尤其适合 ADHD 大脑对即时反馈和低启动门槛的需求（来源：Lex）。这意味着，Tiimo 需要与这些工具组合使用，才能构成完整的 harness。

## 争议与局限：脚手架还是拐杖？

必须诚实指出，当前证据存在局限。AI 工具作为“外部执行功能层”是否会导致能力退化（来源：矛盾与存疑）？缺乏纵向研究。ADHD 大脑与 LLM 同构的论点目前多为类比推理，缺乏神经科学或计算机科学的直接证据（来源：矛盾与存疑）。此外，学习使用 AI 工具本身可能增加认知负荷（来源：矛盾与存疑），净效果因人而异。

个体差异极大：超聚焦是否应被引导而非打断？部分用户反馈强行打断（如番茄钟）有效，而柔性引导可能不够强力（来源：矛盾与存疑）。因此，Tiimo 的视觉化锚点对某些人是救命稻草，对另一些人则是无效装饰。

## 今天就能试的行动

1. **组合使用**：用 Tiimo 规划时间块，用 Goblin Tools 的 Magic ToDo 将每个时间块内的任务分解为 3-5 个具体步骤。例如，在 Tiimo 上设置“10:00-11:00 写报告”，然后打开 Goblin Tools 输入“写报告”，让它分解出“打开文档、列出大纲、写第一段”等子任务。
2. **测试时间锚点**：只使用 Tiimo 的视觉化时间块，不设置任务内容。观察你是否能通过“看到时间流逝”本身来启动任务。如果不能，说明你需要先补任务拆解层。
3. **反思你的 harness**：像孔子一样，每天花 5 分钟“三省吾身”——问自己：今天我的外部调度层（工具+习惯）是否有效？哪个环节卡住了？是时间感知、任务拆解、还是启动动力？
4. **尝试“无为”**：如果工具组合让你更焦虑，尝试老子路线——减少任务数量，只保留最重要的 1-2 项，用 Tiimo 锚定它们的时间，其余全部外包或放弃。

Tiimo 是一个优秀的工具，但只有放在正确的 harness 组合中，它才能从“漂亮的日程表”变成真正的执行功能补偿层。记住：你的大脑不是坏掉了，它只是需要一个定制的外部调度系统。而那个系统，需要你像工程师调试 agent 一样，一步步构建。

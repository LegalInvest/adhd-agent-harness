# 为什么用 Mem 治 ADHD 的时间盲，和给 agent 套 planner-executor 任务分解 是一回事？

> 时间盲通常被描述成对「未来」的失明:估不准、排不了、赶不上。但它还有一个更少被谈论的方向:**对「过去」的失明**。问一个 ADHD 者:上周你在哪个项目上花了最多时间?这个月你实际推进了什么?——答案经常是一片和未来同样浓的雾。而这半边失明,恰恰毁掉的是 planner-executor 架构里最重要的一条回路:**planner 的自我改进,依赖执行历史的回灌——没有可查的过去,就没有能校准的计划**。Mem 这类自动组织、自带时间线的笔记系统,治的正是这半边。

先讲 agent 侧这条回路。planner-executor 分解的教科书画的是前向链:计划→执行;但让这套架构真正变强的是反向链:**执行轨迹(trace)→复盘→更好的计划**。工程上这依赖一个前提:轨迹被完整、自动、带时间戳地记录(executor 跑过的每一步落盘)——**记录不是为了怀旧,是 planner 的训练数据**:估时偏差从哪里来?哪类任务总是超支?哪些步骤反复失败?答案全在轨迹里。没有轨迹的 planner,每次都在用出厂设置做计划——错误永远新鲜,永远重犯。

ADHD 侧的「过去失明」,机制上有两个来源:**①事件记忆的时间标签弱**——记忆研究的方向之一:ADHD 对「什么时候发生」的编码比「发生了什么」更弱,过去的事件有内容无坐标,连不成时间线;**②记录行为本身的执行成本**——写日志、记工时,是典型的「重要不紧急+零即时奖励」任务(必死组合),手动记录方案对 ADHD 等于没有方案。两个来源叠加的后果,比「想不起来」更实际:**你的 planner 永远拿不到训练数据**——周二做计划的你,对「上次类似任务实际花了三倍时间」这条最关键的历史一无所知,于是本周的计划和上周一样乐观,一样崩;崩了之后的自责版本还是「我怎么又这样」——**连「又」字都是模糊的,因为上次的细节早就没了坐标**。

Mem 式系统在这条回路上的价值,核心是**记录的零成本化+时间线的自动生成**:**①捕获即落盘**——随手记的每条笔记自带时间戳,不需要「写日志」这个专门动作:你只是正常记事(想法、会议、决定),轨迹作为副产品自动积累——**把记录从任务变成排泄物,是它对 ADHD 唯一可行的形态**(Otter 篇的「系统替你记录」,笔记版);**②时间线视图=过去的可视化**——「上周我记了什么」一键回放:事件重新获得坐标,那条断掉的时间线由外部系统代为维护;**③语义检索跨时间捞取**——「上次做类似方案是什么时候、当时怎么想的」这种查询变得可行:**planner 做新计划时,终于可以调阅历史了**。

诚实的限制,两条:**其一,笔记轨迹是有偏样本**——你只记了想记的,时间去向的完整账本它给不了(那是 RescueTime 的战场:自动追踪补全量,笔记补语义,两者拼起来才是完整 trace);**其二,回灌不自动发生**——轨迹在库里躺着不等于 planner 变聪明,和 agent 一样需要显式的复盘工序:**每周十五分钟,翻上周时间线,只回答两个问题——「什么比预想的慢?下周的计划哪里要改?」**——没有这道工序,再好的记录也只是数字仓鼠的囤积。

## 边界

同构定位(本文未做正式 A/B/C 分级):ADHD 的时间标签记忆与前瞻记忆研究提供人类侧基础,执行轨迹回灌是 agent 工程的真实回路,「无轨迹则无校准」的同构清晰;Mem 无 ADHD 对照研究,回顾性时间感知的改善证据为间接。声明:笔记系统是记忆辅助;若对过去的空白感伴随强烈的失控与情绪困扰,值得专业讨论。

## 今天就能试的 3 件事

1. **开启副产品式记录**:今天起所有想法/决定进一个带时间戳的入口——不写日志,只正常记事,轨迹自己长。
2. **做一次考古**:翻两周前的笔记/消息记录,重建那周你实际在忙什么——感受一下「有坐标的过去」。
3. **装上回灌工序**:周日 15 分钟,看上周时间线答两问:什么超支了?下周计划改哪里?——planner 的训练,从这里开始。

对未来的盲,大家都在治;**对过去的盲,才是计划年年崩的暗因**——没有轨迹的 planner 永远在用出厂设置。先让过去可查,未来才有得校准。

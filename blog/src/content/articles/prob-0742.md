# Reflect 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Reflect：它到底补上了哪一层执行功能？

你是否有过这样的经历：打开一个笔记应用，想记录一个灵感，结果被扑面而来的文件夹、标签、格式选项吓退，最后什么都没写就关掉了？或者，你明明在 Reflect 里存了无数条笔记，但需要时却翻不到，只能重新搜索——然后陷入新一轮的注意力分散。

这恰恰是 ADHD 大脑的典型困境：**高产但缺乏执行调度层的生成核心**（来源：ADHD 大脑与 LLM 的同构）。你的大脑像一台强大的生成模型，随时能产出创意和洞察，但缺少一个稳定的调度系统来“把想法变成行动”。而 LLM 也有同样的问题：它能生成流畅的文本，但一旦需要精确计算、调用外部数据或执行多步骤任务，就必须依赖 function calling 工具——把计算外包给计算器，把搜索外包给搜索引擎。

**Reflect 之于 ADHD，就像 function calling 之于 LLM。** 它本应成为那个“外部执行调度层”：自动保存上下文、减少决策、让下一步行动显而易见。但很多人用错了——他们把它当成了一个高级备忘录，而不是一个“决策减负器”。

## 同构视角：脚手架 vs 拐杖

ADHD 大脑与 LLM 的结构同构，不是一个空洞的类比。从工程角度看，LLM 需要“agent harness”——包裹模型本身的软件基础设施，处理上下文管理、工具调用和工作流编排（来源：ADHD 大脑与 LLM 的同构）。同样，ADHD 大脑也需要一个“外部执行功能层”：AI 工具作为脚手架，补偿工作记忆、任务启动和时间盲等缺陷（来源：ADHD 的 AI 工具全景）。

Reflect 的设计初衷是成为一个“第二大脑”：自动捕捉想法、链接笔记、提供上下文。这听起来就像 LLM 的上下文窗口——但问题在于，**LLM 的上下文窗口是有限的，而 Reflect 的存储是无限的。** 无限存储反而制造了新的问题：ADHD 用户会陷入“收集-遗忘-再收集”的循环，就像 LLM 在无状态对话中丢失上下文一样。

真正的同构解法是：**Reflect 应该像 function calling 那样，只做两件事——记住关键上下文，并给出明确的下一步行动。** 而不是让用户自己去翻笔记、决定下一步做什么。

## 证据：什么工具真正补上了调度层？

从现有工具来看，那些把“决策减负”放在第一位的工具，效果更明确：

- **Goblin Tools** 的 Magic ToDo 功能，自动将“整理房间”分解为“捡起地板上的衣服”“擦桌子”等步骤（来源：Goblin Tools）。它降低了任务启动门槛，让 ADHD 用户不必自己思考“第一步是什么”。这相当于 LLM 的 function calling 将“计算圆周率”外包给计算器——用户只需给出意图，工具完成调度。
- **Lex** 允许用户通过单一指令触发复杂多步骤流程（来源：Lex）。这类似 LLM 的 agent harness 中的工作流编排：用户说“写一篇关于 ADHD 的文章”，Lex 自动执行大纲、搜索、撰写、校对。
- **Saner.AI** 专注于知识回忆，减少搜索循环（来源：Saner.AI）。这相当于 LLM 的外部记忆管理——但注意，它只解决了“记住”问题，没有解决“下一步做什么”问题。

而 Reflect 的问题在于：它提供了强大的存储和链接功能，但**没有主动帮助用户做决策**。ADHD 用户面对一个空白的笔记页面，仍然需要自己决定“写什么”“怎么组织”。这就像 LLM 有了上下文窗口，但没有 function calling——它知道很多，但不知道如何执行精确任务。

## 争议与局限：脚手架必须可撤除

当前 AI 工具面临的最大争议是：**它们是脚手架还是拐杖？**（来源：矛盾与存疑）脚手架是临时支撑，帮助用户发展内在能力；拐杖则是永久依赖，阻碍神经可塑性。

神经可塑性研究表明，通过训练可以改善执行功能（来源：神经可塑性）。但过度依赖 AI 工具可能让用户失去练习机会。例如，如果 Reflect 总是替用户记住一切，用户的工作记忆可能永远不会得到锻炼。

另一个争议是证据强度：现有证据多来自用户反馈和概念对齐，缺乏随机对照试验（来源：矛盾与存疑）。ADHD 的异质性也很高——对某些人有效的工具，对另一些人可能无效。

## 核心观点：Reflect 的正确用法是“决策减负器”，而非“记忆仓库”

我的判断是：**Reflect 最有价值的用法，不是记录所有想法，而是作为“下一步行动提示器”。** 就像 LLM 的 function calling 不是用来存储所有函数定义，而是用来在需要时调用正确的函数。

具体来说，Reflect 应该：
1. **自动提取关键上下文**：当你开始一个新任务时，Reflect 自动显示相关笔记，而不是让你搜索。
2. **给出明确的下一步行动**：例如，根据你过去的笔记，建议“先写大纲”或“回复邮件”。
3. **减少决策选项**：每次只展示 1-3 个选项，避免 overwhelm。

## 今天就能试的行动

1. **在 Reflect 中创建一个“下一步行动”笔记**：每次有新想法，不要直接存，而是问自己“这个想法对应的下一步行动是什么？”然后只存那个行动。
2. **用 Goblin Tools 的 Magic ToDo 分解一个你拖延的任务**：比如“写报告”，让 AI 分解成“打开文档”“写第一段”“保存”等步骤。
3. **设置 Reflect 的每日回顾提醒**：每天固定时间，让 Reflect 展示一条昨天记的笔记，并问自己“今天要基于此做什么？”
4. **尝试 Lex 的单一指令**：把“整理房间”这样的模糊任务输入 Lex，看它如何自动分解并执行。

记住：工具的价值不在于存了多少，而在于省了多少决策。Reflect 是脚手架，不是拐杖——用它来训练你的执行功能，而不是替代它。

# 为什么用 Structured 治 ADHD 的情绪失调，和给 agent 套 会褪去的脚手架 是一回事？

> Structured 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

## 问题：情绪失控与 Agent 失控，是同一种失控

你有没有过这样的时刻：明明知道该做什么，大脑却像死机一样拒绝启动；或者突然因为一件小事情绪爆炸，事后连自己都觉得莫名其妙。这不是你不够努力，而是你的执行功能——大脑的“调度层”——掉线了。

ADHD 的情绪失调，根源正是执行功能缺陷：工作记忆无法维持情绪信息，时间盲让时间压力感失调，任务启动困难引发拖延和自责（来源：情绪失调；执行功能）。

现在，想象你正在调试一个 LLM agent。它拥有强大的生成能力，却常常忘记上下文、偏离目标、被无关输入带跑。你给它套上 agentic harness——一个结构化的调度框架，包括记忆模块、步骤分解、进度追踪——它才变得可靠。

这两件事，本质上是一回事。

## 同构：高产但缺调度层的生成核心

ADHD 大脑与 LLM 共享同一个脆弱结构：一个强大的生成核心（智力/算力）配上一个不可靠的调度层（执行功能/编排）。情绪失调正是调度层失灵的典型表现——当工作记忆过载或时间感知失灵，情绪便失去约束（来源：ADHD 大脑与 LLM 的同构）。

在 ADHD 侧，这个调度层由执行功能（工作记忆、抑制控制、时间感知）构成。在 LLM 侧，它由 prompt 结构、上下文窗口、记忆系统、任务分解逻辑组成。两边的解法结构同构：都需要一个外部的、结构化的脚手架（harness）来补偿调度层的缺陷。

## 证据：两边都成立

**ADHD 侧：**

Inflow 通过训练背外侧前额叶皮层（因果流入中枢）来强化工作记忆，其 AI 算法动态调整认知负荷（来源：Inflow）。Goblin Tools 的 Magic ToDo 功能将“整理房间”这样的模糊任务自动分解为“捡起衣服”“擦桌子”等具体步骤，直接降低启动门槛（来源：Goblin Tools）。Saner.AI 通过本地记忆和快速检索，减少 ADHD 用户因工作记忆不足导致的搜索循环和标签切换（来源：Saner.AI）。

这些工具的共同点：它们不是替代你的大脑，而是为你的执行功能提供一个外部结构——一个数字化的“工作记忆”和“任务启动助推器”（来源：AI 与 ADHD 的情绪调节）。

**LLM/Agent 侧：**

LLM agent 的典型失败模式包括：忘记长上下文中的关键信息、偏离初始目标、被无关输入干扰。工程上的解法正是结构化脚手架：固定 prompt 模板、外部记忆向量库、步骤分解链、进度追踪回调。这和 Goblin Tools 的任务分解、Saner.AI 的知识回忆、Inflow 的认知训练，本质是同一套模式——用外部结构补偿内部调度缺陷。

## 核心判断：脚手架 vs 拐杖

这里有一个关键边界，也是本文最想提出的观点：**脚手架必须设计为可褪去的临时支撑，而非永久拐杖。**

当前多数 AI 工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这存在过度依赖风险：AI 工具可能削弱内在情绪调节能力（来源：AI 与 ADHD 的情绪调节）。真正的脚手架，应该像建筑工地上的临时支架——当建筑（你的执行功能）足够坚固时，它可以被拆除。

在 LLM 工程中，这个原则同样成立：好的 agentic harness 应该随着 agent 能力的提升（比如上下文窗口扩大、推理能力增强）而逐步简化，而不是永远依赖复杂的编排。

## 局限与争议

必须诚实指出，当前证据存在多重局限：

1. **缺乏长期效果证据**：多数研究为短期观察，AI 对情绪调节的长期神经可塑性影响未明（来源：AI 与 ADHD 的情绪调节）。
2. **个性化不足**：现有工具多基于通用认知模型，对共病焦虑等个体差异覆盖有限（来源：AI 与 ADHD 的情绪调节）。
3. **多巴胺干预争议**：基于多巴胺的奖励系统可能强化依赖而非内在动机（来源：矛盾与存疑）。
4. **同构命题证据薄弱**：ADHD 大脑与 LLM 的同构主要来自概念类比，缺乏大规模实证（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个让你焦虑的任务**：输入你拖延最久的一件事，让 AI 拆成 5 个微步骤，每完成一步就标记。体验“启动门槛降低”对情绪的即时影响。
2. **给自己设一个“脚手架撤除日”**：选一天，刻意不使用任何 AI 工具完成日常任务，观察自己的执行功能表现。这能帮你判断工具是脚手架还是拐杖。
3. **为你的 LLM agent 写一个结构化的 system prompt**：包含明确的步骤链、记忆指令、进度追踪要求。对比无结构 prompt 的表现差异。
4. **记录情绪触发模式**：用 Saner.AI 或简单笔记，记录一周内情绪失控前的具体事件。识别模式后，设计一个针对性的结构化应对流程（比如“当 X 发生时，先做 Y 再反应”）。

## 结语

ADHD 的情绪失调和 LLM agent 的失控，不是两个独立的问题。它们共享同一个底层结构：一个强大的生成核心需要一个可靠的调度层。结构化脚手架（Structured Harness）正是同时解决这两个问题的同构方案。但关键在于，脚手架必须设计为可褪去的——它的最终目标，是让你和你的 agent 在没有它的时候，也能稳定运行。

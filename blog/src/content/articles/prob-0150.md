# ChatGPT 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 ChatGPT：用久了会让 ADHD 更强还是更依赖？

## 问题：边做边忘的 ADHD 与无状态的 LLM

如果你是一名 ADHD 患者，你一定经历过这样的场景：打开浏览器想查资料，却被一条推送吸引，半小时后才发现自己早已忘了最初要做什么。这种“边做边忘”的困境，根源在于工作记忆容量小（来源：ADHD 的 AI 工具全景），导致执行任务时极易发生目标漂移（来源：重锚定与目标漂移）。

如果你是一名 Agentic Harness 工程师，你也一定熟悉类似的窘境：LLM 在长程任务中“开始忘记系统提示”（来源：重锚定与目标漂移），行为逐渐偏离初始指令。LLM 本质上是一个无状态生成核心——它没有跨会话的记忆，每次交互都像第一次见面（来源：ADHD 的 AI 工具全景）。

这两类困境，其实共享同一个底层结构：一个强大的生成核心，配着一个脆弱的执行调度层。ADHD 大脑的智力与创造力如同 LLM 的生成能力，但缺乏稳定的执行功能——计划、组织、工作记忆（来源：ADHD 的 AI 工具全景）。而 LLM 同样需要外部的 harness（上下文传递、工具接口、规划产物）才能完成复杂任务（来源：规划循环与任务分解）。

## 同构：生成核心与外部 harness

正是这种“同构”，让 AI 工具成为 ADHD 的天然补丁。Goblin Tools 的 Magic ToDo 能将“整理房间”自动分解为“捡起地板上的衣服”“擦桌子”等具体步骤（来源：Goblin Tools），从而降低启动焦虑——这本质上是在为 ADHD 大脑提供一个外部规划循环，正如 harness 为 LLM 提供任务分解与上下文管理。Saner.AI 通过本地记忆存储和快速检索，减少搜索循环和标签切换（来源：Saner.AI），直接补偿工作记忆不足——这相当于为 LLM 外挂一个向量数据库。Lex 允许用户通过单一指令触发复杂多步骤任务（来源：Lex），降低决策负担——这就像为 LLM 设计一个自动化编排层。

这些工具的设计逻辑高度一致：将本应由内部执行功能完成的任务，外包给外部系统。ADHD 大脑需要这种外包，就像 LLM agent 需要外部记忆来“续命”。

## 脚手架还是拐杖？边界在于主动设计

那么，ChatGPT 之于 ADHD，究竟是脚手架还是拐杖？我的判断是：**它取决于用户是否主动设计交互方式**。

当用户只是随手输入问题、被动接受答案时，ChatGPT 更像一个拐杖——它即时生成内容，却无法训练用户的任务分解能力、工作记忆或时间感知。长期如此，用户可能越来越依赖外部生成，内在执行功能反而萎缩。这正是“拐杖与脚手架”争议的核心：过度依赖 AI 工具可能阻碍内在执行功能发展（来源：矛盾与存疑）。

但当用户主动设计提示、构建外部记忆系统时，ChatGPT 就变成了可撤除的脚手架。例如，用户可以用 ChatGPT 作为“规划伙伴”，创建一套“实际适配 ADHD 大脑的系统”（来源：规划循环与任务分解）；或者将重要信息存入 Mem 或 Reflect，形成持久记忆库（来源：ADHD 的 AI 工具全景）。这种主动设计相当于在构建自己的“harness”——用户自己扮演编排层的角色，而 ChatGPT 只是生成核心。

证据也支持这一边界。Goblin Tools 被评价为“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools），但用户必须主动输入任务才能触发分解。Saner.AI 的知识回忆功能（来源：Saner.AI）需要用户主动存入信息，才能减少搜索循环。这些工具都不是“一键解决”的魔法——它们需要用户付出认知努力来设置和维护。

## 局限与诚实警告

当然，这个判断并非没有争议。首先，ADHD 的异质性极高，单一工具难以覆盖所有亚型（来源：矛盾与存疑）。对于某些执行功能严重受损的用户，即使是主动设计也可能超出能力范围，此时工具反而可能成为负担。其次，现有证据多为短期或个案，缺乏随机对照试验验证长期效果（来源：矛盾与存疑）。我们尚不清楚，长期使用这些工具是否会改变大脑的执行功能网络。最后，多巴胺干预的有效性存在争议（来源：矛盾与存疑），而许多工具声称通过神经科学原理改善症状，却缺乏透明验证。

因此，我的核心观点是：**ChatGPT 作为 ADHD 的 harness，其价值取决于用户是否主动参与设计。它既不是救世主，也不是陷阱，而是一面镜子——映照出用户如何与自己的生成核心共处。**

## 今天就能试的行动

1. **用 ChatGPT 做一次任务分解**：写下你明天最头疼的一件事，让 ChatGPT 分解成 5 个以内的子步骤。完成后，手动调整顺序，形成你自己的规划循环。
2. **设置一个外部记忆锚点**：在 Saner.AI 或 Mem 中存入一句你经常忘记的话（如“先做最难的事”），每天打开一次，作为重锚定信号。
3. **尝试一次“单一指令”实验**：用 Lex 或类似工具，将“写周报”这一任务压缩成一个指令，观察它是否降低了你的启动焦虑。
4. **记录一次目标漂移**：下次分心时，用手机备忘录记下“我原本要做什么”。一周后回顾，你会发现自己的漂移模式——这是设计更好 harness 的第一步。

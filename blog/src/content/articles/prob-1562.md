# 如果把 ADHD 大脑当成生成核心，Claude 能不能当它的拐杖与脚手架？

> 用 agent 工程的思路重新理解 Claude 对 ADHD 的真正价值。

## 问题：怕依赖，又怕崩

你打开 Claude，输入“帮我规划今天的三件事”，AI 迅速列出步骤。你松了口气，但下一秒又开始焦虑：**“我是不是太依赖它了？万一哪天没网，我是不是连启动都做不到？”** 另一边，工程师在构建 Agentic Harness 时反复纠结：**“这个助手到底该多自主？太强了用户会退化，太弱了又帮不上忙。”**

这两类人其实在问同一个问题：**如果我把自己的大脑（或模型）当作生成核心，外部系统到底是该做永久的拐杖，还是可褪去的脚手架？**

## 同构：高产但缺调度层的生成核心

ADHD 大脑与 LLM 共享一个底层结构：**两者都是高产但缺乏可靠执行调度层的生成核心**（来源：AI 与 ADHD 的情绪调节）。ADHD 的创造力、联想能力与 LLM 的文本生成能力一样充沛，但执行功能缺陷——工作记忆有限、时间盲、任务启动困难——让这些产出难以被有序调度。就像 LLM 在高温采样下输出随机、缺乏可重复性一样，ADHD 个体的表现也呈现日内波动（来源：采样温度与表现波动）。

这种同构不仅是比喻。LLM 的“幻觉”源于缺乏验证循环，ADHD 的情绪失调也类似“上下文失控”——当工作记忆过载，情绪反应如同 LLM 跑偏，脱离理性轨道（来源：AI 与 ADHD 的情绪调节）。两者的解法因此同构：**给生成核心套上 harness——通过上下文工程、外部记忆、验证循环来补偿执行缺陷。**

## 证据：两边的 harness 长什么样？

### ADHD 侧：工具即脚手架

现有 ADHD 工具已经不自觉地在扮演 harness 角色：
- **Goblin Tools** 的 Magic ToDo 自动将“整理房间”分解为“捡起衣服”“擦桌子”等小步骤，降低启动门槛（来源：Goblin Tools）。这相当于 LLM harness 中的“任务分解与规划”。
- **Saner.AI** 通过本地记忆减少搜索循环和标签切换，直接补偿工作记忆缺陷（来源：Saner.AI）。这相当于 harness 的“外部记忆系统”。
- **Inflow** 通过认知训练强化背外侧前额叶皮层（因果流入中枢），试图从底层提升工作记忆（来源：Inflow）。这类似 harness 的“确定性状态转换”训练。

但这些工具都面临同一个争议：**它们究竟是脚手架还是拐杖？** 用户报告显示，Goblin Tools 能“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools），但也有人担心一旦停用，内在能力反而更弱（来源：矛盾与存疑）。

### LLM 侧：harness 工程的原则

LLM 的 harness 工程明确区分了拐杖与脚手架：**好的 harness 应包含验证循环、规划工件、记忆系统，但最终目标是在褪去后系统仍能运行**（来源：幻觉与验证循环）。例如，复杂的 harness 不会盲目执行工具，而是实现验证步骤——检查输出格式、运行测试用例（来源：幻觉与验证循环）。这正是 ADHD 大脑需要的：**一个能帮助自己建立内部验证循环的外部系统，而不是替自己做所有决定。**

## 核心判断：Claude 是脚手架，但需要主动设计

我的观点是：**Claude 可以成为 ADHD 的脚手架，前提是把它当作“可编程的执行调度层”而非“万能回答机”。** 关键在于主动设计使用方式：
1. **使用上下文工程**：每次对话前，明确告诉 Claude 你的当前状态（“我注意力只能维持 15 分钟”），让它调整输出长度和复杂度。这相当于给 LLM 设定采样温度——低温度对应低波动、高可重复性。
2. **建立验证循环**：不要只让 Claude 给答案，而是让它帮你设计验证步骤。例如，让它把任务分解后，要求它“检查我是否遗漏了关键步骤”，就像 harness 的“自我批评提示”。
3. **外部记忆而非依赖**：用 Claude 记录情绪触发模式，但定期回顾并尝试脱离。就像无状态与外部记忆的平衡——工具记录，大脑学会模式识别。

## 诚实面对局限

必须指出，现有证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：矛盾与存疑）。过度依赖 AI 可能削弱内在时间感知能力（来源：矛盾与存疑），且个体差异巨大——注意缺陷型 vs. 多动冲动型对工具的反应不同（来源：AI 与 ADHD 的情绪调节）。此外，基于多巴胺的干预（如游戏化反馈）可能强化成瘾行为，需谨慎（来源：AI 与 ADHD 的情绪调节）。

**脚手架 vs. 拐杖的边界**在于：脚手架在使用过程中会逐渐内化为能力，而拐杖永远只是替代。如果你发现离开 Claude 后完全无法启动，那它就是拐杖；如果你能逐渐减少使用频率但仍保持产出，那它就是脚手架。

## 今天就能试的行动

1. **设定“脚手架协议”**：下次用 Claude 时，在提示词开头加上“请以帮我自己思考的方式回答，而不是直接给答案。每给出一个建议后，加一句‘你可以自己试试看’。”
2. **做一次情绪验证循环**：当情绪波动时，对 Claude 说“我现在的感受是 X，请帮我列出三个可能的外部证据来验证这个感受是否准确”。
3. **记录使用日志**：每天记录你使用 Claude 的次数和场景，一周后分析哪些场景可以逐渐脱离。
4. **尝试“无 AI 日”**：每周选一天完全不用 AI 工具，观察自己的执行功能波动，重新校准对脚手架的需求。

最终，Claude 的价值不在于替你做决定，而在于帮你搭建一套可以逐渐内化的执行系统。正如 harness 工程的目标是让系统最终能自主运行，ADHD 大脑也需要一个能褪去的脚手架——不是永远扶着，而是扶着学会走路。

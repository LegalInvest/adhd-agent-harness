# ADHD 学生如何用「给生成核心套 harness」的思路应对考试与作业里的任务启动困难、不会拆解？

> 把 LLM 工程里的「agent 用 planner-executor 循环分解长任务」迁移到ADHD 学生的考试与作业场景，双域证据作答。

## 问题：为什么ADHD学生面对作业和考试时，大脑一片空白？

你坐在书桌前，面前是一道需要分步骤解答的数学大题，或者一篇需要拆成引言、文献、方法、结果、讨论的论文。你明明知道自己有能力完成，但就是无法开始。即便开始了，也会在某个细节卡住，或者跳到最后一步，然后发现逻辑断裂。这不是懒惰，也不是智力问题——这是执行功能（executive function）的失效。对ADHD来说，“常常感觉方向盘后没有人”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。传统的时间管理工具——计划本、便签、番茄钟——往往用了一周就崩溃，因为它们无法解决“时间盲”和工作记忆的瓶颈（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

与此同时，在AI工程领域，工程师们正面临一个几乎相同的问题：LLM（大语言模型）拥有强大的生成能力，但单独使用时缺乏可靠的执行调度。LLM会“上下文膨胀和推理退化”，无法自主将长任务分解为可执行的子步骤。于是，他们发明了“复合AI系统架构”——通过分离规划与执行的双代理架构、惰性工具发现、自适应上下文压缩等技术，在LLM外部搭建调度层（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。这本质上就是“给生成核心套上harness”。

## 同构：ADHD大脑与LLM是同一类“生成核心”

ADHD大脑的智力与创造力，与LLM的算力与语言生成能力，在结构上高度同构：两者都高产但无序，都缺乏一个可靠的执行调度层来组织行动（来源：生成核心与调度层）。ADHD的执行功能失效，表现为工作记忆容量有限、时间感知困难、任务启动与切换障碍；LLM则受限于固定上下文窗口和长程依赖追踪困难（来源：Long-Term Memory Architectures for AI Agents - Redis）。两者的解法也是同构的——外部脚手架。

在ADHD侧，外部工具如Motion、Reclaim.ai、Tiimo充当了“外部执行功能层”：Motion自动根据任务和截止日期创建并动态调整每日计划，消除“下一步该做什么”的决策负担（来源：Motion）；Reclaim.ai通过创建深度工作、习惯和专注时间的智能块，自动防御会议侵占，帮助维持日程稳定性（来源：Reclaim.ai）；Tiimo将时间转化为可见元素，直接应对时间盲（来源：Tiimo）。这些工具的本质，就是为ADHD大脑这个“生成核心”提供外部的规划、记忆和调度。

在LLM/agent侧，harness工程正是设计围绕LLM的脚手架——包括上下文传递、工具接口、规划工件、验证循环、记忆系统和沙箱。脚手架在首次提示前组装代理（系统提示、工具模式、子代理注册），而harness在运行时编排工具调度、上下文管理和安全执行（来源：Building AI Coding Agents for the Terminal）。这与ADHD工具的结构惊人相似：Motion的自动排程相当于planner代理，Tiimo的视觉时间线相当于上下文窗口，Todoist的智能提醒相当于外部记忆。

## 解法：给生成核心套上harness——从planner-executor循环开始

那么，ADHD学生如何将这一思路迁移到考试与作业场景？核心是：**不要试图依赖大脑内部的执行功能，而是建立外部的planner-executor循环**。

**Step 1: 用外部工具充当planner（规划器）**  
面对一个长作业或考试复习任务，你的第一反应不应该是“我该从哪开始”——那是让生成核心自己调度，注定失败。相反，像Motion一样，让工具帮你分解。例如，把整篇论文的截止日期输入Motion或Reclaim.ai，让AI自动生成每日子任务：周一写引言大纲，周二查三篇文献，周三写方法部分……你只需要执行，不需要规划。如果日程被打乱，工具会自动重新安排（来源：Motion）。对于更细粒度的分解，可以使用Goblin Tools的“任务分解”功能，它会将“写一篇论文”拆成“打开文档”、“写第一句话”、“写第二句话”这样的微步骤，降低启动门槛。

**Step 2: 用视觉化工具维护“上下文窗口”**  
ADHD的工作记忆缺陷导致很容易在任务中丢失上下文——就像LLM的上下文窗口溢出。Tiimo的视觉化时间线（来源：Tiimo）相当于一个外部上下文窗口，让你始终看到当前任务在整个计划中的位置。考试时，可以简化使用：在草稿纸上画一个时间轴，标注剩余时间，每完成一步就划掉。这相当于手动维护一个“上下文压缩”机制，防止思维漂移。

**Step 3: 用“保护块”防止调度崩溃**  
Reclaim.ai的防御性时间块设计（来源：Reclaim.ai）直接对应LLM harness中的“沙箱”——保护核心任务不被突发干扰打断。ADHD学生可以在日程中固定几个“不可侵犯”的深度工作块（比如每天上午9-11点写论文），并像保护API调用一样保护它们：不接电话、不回消息、不刷社交媒体。如果被意外事件打乱，不要自责，而是像Motion一样“自动重新安排”——把被占用的时间块移到当天稍晚或第二天。

## 边界：脚手架 vs 拐杖

这里必须诚实指出争议。同构命题本身是理论模型，缺乏大规模实证支持（来源：矛盾与存疑）。工具的有效性多基于用户反馈，而非随机对照试验。更重要的是，过度依赖外部工具可能阻碍内在执行功能的发展（来源：拐杖与脚手架）。真正的脚手架应帮助使用者“建造”，但使用者仍需自己“攀登”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。Motion和Reclaim.ai设计为长期使用，未提及撤除机制（来源：矛盾与存疑），这意味着它们可能成为永久拐杖，而非临时脚手架。

我的判断是：**对于考试和作业这种高压力、有时限的场景，工具作为拐杖并不可耻。** 关键是在使用过程中，逐步内化那些被外化的策略——比如，当你习惯了Motion的自动分解后，尝试自己先手动分解再对照AI的结果；当你习惯了Tiimo的视觉时间线后，尝试在无工具时也画个时间轴。脚手架的可撤除性，需要使用者主动练习，而非工具本身提供。

## 今天就能试的行动

1. **用Motion（或Reclaim.ai）规划本周一个长作业**：输入截止日期和所有子任务，让AI自动生成每日计划。观察你的启动时间是否缩短。
2. **用Goblin Tools分解一个你一直拖延的任务**：比如“整理笔记”，看它拆成哪几步。然后只执行第一步。
3. **在书桌上贴一张“上下文窗口”便利贴**：写上当前任务的名称、已完成部分、下一步。每完成一步就更新，像维护一个简版Tiimo。
4. **设置一个“保护块”**：每天固定30分钟，关闭所有通知，只做一件事。如果被打断，用日历重新安排这个块。

给生成核心套上harness，不是放弃自我管理，而是承认大脑的架构需要外部调度层。就像LLM需要复合系统才能完成复杂任务，ADHD大脑也需要脚手架才能释放其真正的创造力。这不是缺陷，而是设计。

# ADHD 创业者如何用「给生成核心套 harness」的思路应对把想法落地执行里的怕过度依赖工具、又怕没有支撑就崩？

> 把 LLM 工程里的「harness 应是会褪去的脚手架，而非永久拐杖」迁移到ADHD 创业者的把想法落地执行场景，双域证据作答。

## 问题：怕依赖，又怕崩

你是否有过这样的体验：脑子里有一个绝妙的创业想法，细节清晰得像电影画面，但一旦坐下来执行，却像陷入泥潭——要么被手机通知拉走，要么在打开十几个标签页后忘了最初要做什么。更折磨的是，你试过各种工具：待办清单、番茄钟、日程本，但要么用了几天就失效，要么让你觉得“没有这些拐杖我根本活不下去”。这种“怕过度依赖工具、又怕没有支撑就崩”的焦虑，在ADHD创业者中极为普遍。

## 同构：生成核心与调度层的分离

ADHD大脑与LLM（大语言模型）在结构上惊人地相似：两者都拥有强大的“生成核心”——ADHD的智力与创造力，LLM的文本生成能力——但都缺乏可靠的“调度层”。ADHD的执行功能缺陷（工作记忆不足、时间盲、任务启动困难）导致想法无法有序落地；LLM在长程任务中也会“忘记系统提示”（来源：Building an AI Coding Agent for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned），发生目标漂移。

工程领域解决LLM问题的思路是“harness”——一个可褪去的脚手架，负责上下文交付、工具接口和规划循环，而不是一个永久绑定的拐杖。同样的逻辑适用于ADHD：AI工具应当成为临时支撑，帮助大脑完成执行功能，但最终目标是让用户逐渐内化这些能力。

## 证据：两侧的真实解法

**ADHD侧**：工具如Goblin Tools通过“Magic ToDo”功能将复杂任务分解为小步骤，降低启动门槛（来源：AI Tools for Kids with ADHD: A Complete Guide for Parents...）。用户反馈它“将压倒性的事情变成一系列不压倒性的事情”（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...)）。Saner.AI则通过知识回忆减少搜索循环，直接补偿工作记忆缺陷（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。Inflow通过训练背外侧前额叶皮层来强化认知控制（来源：Dynamic causal brain circuits during working memory and their functional controllability），但效果需更多验证。

**LLM/agent侧**：Harness工程通过事件驱动的系统提醒对抗指令消退，例如在Agent交互中定期注入原始系统提示，防止模型偏离目标（来源：Building AI Coding Agent for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。自适应规划器如Motion和Tiimo则动态调整日程，当计划崩溃时重新排列任务（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

## 核心观点：脚手架 vs 拐杖的边界

关键区别在于：脚手架是有意设计为可撤除的，而拐杖则默认永久使用。当前的AI工具大多缺乏撤除机制——Goblin Tools、Saner.AI、Inflow均未提供“逐步减少依赖”的功能（来源：矛盾与存疑）。这导致用户可能长期依赖，反而削弱内在执行功能。真正的harness应当像建筑脚手架：在结构稳固后拆除，而不是永远依附。

## 争议与局限

- **多巴胺干预争议**：部分工具声称通过奖励系统调节多巴胺，但证据不足（来源：多巴胺页面）。
- **长期效果缺乏**：多数研究为短期观察，AI对神经可塑性的长期影响未知（来源：AI与ADHD的情绪调节）。
- **个性化不足**：现有工具基于通用模型，对共病焦虑等个体差异覆盖有限（来源：AI与ADHD的情绪调节）。
- **同构命题的实证弱**：ADHD与LLM的类比虽有解释力，但缺乏大规模实证（来源：矛盾与存疑）。

## 今天就能试的行动

1. **用Goblin Tools分解一个具体任务**：输入“准备下周的客户提案”，观察AI如何拆解步骤。完成后，尝试不依赖工具手动分解另一个类似任务，对比差异。
2. **设置重锚定提醒**：在手机上设置每小时一次的随机提醒，内容为“当前目标是什么？”，对抗目标漂移。
3. **试用Saner.AI的知识回忆功能**：将一周内频繁搜索的信息存入工具，减少标签切换。一周后评估是否形成了更顺畅的工作流。
4. **反思工具依赖**：记录每天使用AI工具的次数，思考哪些任务可以在没有工具的情况下完成。尝试每周减少一次工具使用，逐步建立内在能力。

## 结语

ADHD创业者不需要更强大的生成核心——他们的大脑已经足够聪明。他们需要的是一个精心设计的harness，一个可褪去的脚手架，在想法落地的过程中提供支撑，又在能力成长后悄然退场。这不仅是工程智慧，也是自我关怀的哲学。

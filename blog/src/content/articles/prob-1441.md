# Reclaim.ai 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 上一篇讲了用笔记当 system prompt 的载体,这一篇讲一个更硬核的载体:日历。Reclaim.ai 的「习惯(Habits)」功能,本质是让你**用声明的方式写原则**——「每周锻炼三次」「每天中午留一小时深度工作」「周五下午复盘」——声明一次,算法负责在每周的日历上持续为它们找位置、守位置。这比笔记里的原则更进一步:**它是会自动兑现成时间块的 system prompt**。但「声明式原则」这个形态,恰恰有自己独特的用错方式。

先看这个形态强在哪。普通原则(「我要多锻炼」)的失败路径是:意图→无人翻译成具体时间→每天都「找不到空」→周日发现一次没做。Reclaim 的习惯把翻译自动化了:**你声明频率与时长,它在真实日历的缝隙里替这条原则抢地盘,被会议挤掉就自动换位置**——原则第一次有了自动执行的排程代理。对 ADHD,这消灭了「意图到时间块」之间最容易断的那一环。误用四种:

**误用一:声明「愿望中的自己」,不是「真实的自己」(aspirational prompt)。** 工程版:system prompt 写「你是一个从不出错的专家」——写得再满,模型能力没变,只是让失败更频繁地违反声明。人类版:一口气声明每天锻炼、每天阅读、每天冥想、每周三次学习——**按理想自我配置的习惯集,第一周就会全面违约**,而连续违约教会你的唯一一件事是「这个系统不算数」。修法:**从一条开始**——选一个最重要的习惯,守两周,再加下一条;声明的可信度是攒出来的,不是配出来的。

**误用二:原则之间没有优先级,冲突时全线崩。** 工程版:system prompt 里两条指令打架(「回答要详尽」+「回答要简短」),模型行为随机摇摆。人类版:日历忙的那周,锻炼、深度工作、复盘互相抢缝隙,算法只能都排到边角——**你不知道该保哪个,索性全放弃**。修法:给习惯声明显式的优先级(Reclaim 支持),**并想清楚「灾难周只保哪一条」**——有序退让是弹性,无序崩塌是失控。

**误用三:块被守卫了,内容没被执行(空心锚)。** 工程版:prompt 每轮都注入,但模型输出根本不参照它——注入成功,遵循失败。人类版:「深度工作」块准时出现在日历上,你也没安排会议——**但那一小时你在收件箱里漂着**。块的存在≠原则的执行;时间被圈出来只是第一步,块内还需要启动(打包的任务、点火的仪式,见启动系列)。修法:给每个习惯块预挂「具体做什么」——「深度工作」块绑定「本周最重要任务的下一步」,让锚有货。

**误用四:违约后调低声明,而不是排查原因。** 工程版:模型不遵循指令,工程师的第一反应是删指令,而不是查「是不是被别的上下文压过了」。人类版:锻炼习惯连续被跳过,你把频率从三次改成一次,再改成零——**每次下调都像是理性妥协,合起来是原则的安乐死**。修法:违约先归因——是时段不对(排在你最没能量的时候)?是粒度太大(一小时锻炼改成二十分钟散步)?还是这条原则本来就不是你的(是社交媒体上借来的)?**前两种改参数,最后一种才删声明**——删也删得明明白白。

## 边界

同构强度 B 级:声明式配置与指令冲突是真实的工程议题,ADHD 的意图-执行间隙与习惯建立困难有文献支撑,Reclaim.ai 无对照研究。声明:习惯排程是结构辅助;若连一条最小声明也长期无法守住、伴随情绪耗竭,那是负载或健康信号,请专业评估——日历能守卫时间,守卫不了能量,能量的账要到别处去算。

## 今天就能试的 3 件事

1. **砍到一条**:现有的习惯声明只留最重要的一个,守两周再谈扩容。
2. **给它挂上具体内容**:这个块出现时做什么,提前写在块的备注里。
3. **写下你的灾难周条款**:一切都崩的那周,唯一要保的是哪条?想好,写下。

原则写在纸上是愿望,声明进日历才是配置——**但配置的第一美德不是雄心,是可兑现**。一条守得住的声明,胜过七条装点门面的;system prompt 如此,人生的也如此。

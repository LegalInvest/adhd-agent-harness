# Structured 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Structured：它到底补上了哪一层执行功能？

你有没有过这样的经历：打开手机想查个资料，半小时后却发现自己刷完了整个社交媒体，完全忘了最初要干什么？或者，你明明计划好一天的任务，却总在细节里越陷越深，最终偏离了主线？

这不是你的错。ADHD 大脑本质上是一个强大的生成核心——创意丰富、联想迅速——但缺乏一个可靠的执行调度层来维持目标和路径。你开始一个任务时，大脑像 LLM 一样以高采样温度运行，发散出无数分支；而一旦进入超聚焦，又可能像低温度 LLM 那样卡在局部最优，丢失全局目标。这种“目标漂移”是 ADHD 的核心痛点，也是 LLM/agent 工程中同样的难题：长程任务里，agent 会因上下文膨胀或工具调用而偏离原始指令，需要不断“重锚定 system prompt”。

**Structured 的承诺，就是成为那个重锚定系统。**

## Structured 到底补上了哪一层执行功能？

Structured 是一款时间块规划 app，它让你把一天拆成可视化的时间块，并强制按顺序执行。对 ADHD 用户而言，它的核心价值在于**外部化时间感知和任务序列**。ADHD 常伴随时间盲（难以感知时间流逝）和工作记忆缺陷（记不住下一步该做什么），Structured 通过固定时间块和顺序列表，把这两层执行功能外包出去。你不再需要靠大脑维持“先做 A 再做 B”的队列，而是让 app 替你记住并提示。

这恰恰对应了 LLM agent 工程中的“计划-执行”分离模式（来源：Planner-Executor Agentic Framework）。在 agent 系统中，planner 负责制定确定性的步骤序列，executor 按序调用工具，从而避免 agent 自由发挥导致的目标漂移。Structured 就是 ADHD 的 planner：它把模糊的一天分解为确定性的时间块，降低启动门槛，减少中途决策疲劳。

## 同构证据：ADHD 大脑与 LLM 共享同一个问题

ADHD 大脑与 LLM 的深层同构体现在多个层面（来源：AI 与 ADHD 的专注力）：
- **生成核心与调度层分离**：ADHD 的创意发散与 LLM 的文本生成都是高产出但不可靠的，需要外部调度层来稳定输出。
- **无状态与外部记忆**：ADHD 的工作记忆缺陷与 LLM 的无状态性同构，都需要外部记忆系统维持连续性。Structured 的时间块列表就是这样一个外部记忆。
- **上下文工程**：ADHD 易被环境带偏，LLM 易被上下文膨胀干扰，都需要主动设计“当下注意什么”的工程方案。Structured 通过强制当前时间块，提供了注意力锚点。

这种同构不是空泛的类比。在 LLM 工程中，agent harness（如 LangChain、CrewAI）通过重锚定 system prompt 来对抗目标漂移——每步执行前都重新注入原始目标，防止 agent 跑偏。Structured 的“时间块切换”就是 ADHD 的 system prompt 重锚定：每完成一个块，你被迫重新审视下一个块的目标，从而对抗注意力漂移。

## 有人用错了：脚手架 vs 拐杖的边界

然而，Structured 很容易被误用。许多 ADHD 用户把它当作“被动拐杖”：设定好时间块后，就完全依赖 app 的提醒来执行，一旦错过提醒或被打断，整个计划就崩塌。这种使用方式忽略了 Structured 的本质——它应该是一个**主动脚手架**，帮你练习内在的执行功能，而不是替代它。

这与 AI 工具作为外部执行功能层的争议一致（来源：矛盾与存疑）。多个页面警告“过度依赖 AI 工具可能削弱内在能力”（来源：时间盲、任务启动）。Structured 如果只是让你被动跟随，它就会像 Brain.fm 那样——虽然声称通过神经锁相帮助专注，但缺乏独立临床研究支持其在 ADHD 中的有效性（来源：Brain.fm）。同样，Focusmate 通过实时同伴问责提供外部结构，但效果高度依赖伙伴可靠性（来源：Focusmate）。

真正的脚手架用法是：**把 Structured 当作训练轮**。一开始，你用它来建立时间感知和序列感；逐渐地，你开始内化那种“先计划再执行”的节奏，最终可以在没有 app 的情况下也保持基本的方向感。RescueTime 的自动追踪可以辅助这一过程——通过事后分析注意力漂移模式，帮你识别自己容易在哪些环节跑偏（来源：RescueTime）。

## 争议与局限

必须诚实指出，Structured 的 AI 功能细节尚不明确（来源：矛盾与存疑），其效果证据主要来自用户报告，缺乏大规模对照实验。个体差异很大：对某些 ADHD 亚型（如冲动型），严格的 time-blocking 可能引发焦虑；对另一些（如注意力涣散型），它可能成为救命稻草。此外，依赖风险真实存在——长期使用是否会导致内在时间感知能力进一步退化？目前没有答案。

## 今天就能试的行动

1. **用 Structured 做“单任务时间块”**：每天只规划 3 个最重要的时间块，每个块只做一件事。完成后，手动标记“完成”并写下感受。这能帮你识别哪些任务容易导致目标漂移。
2. **结合 RescueTime 做周回顾**：用 RescueTime 追踪一周的时间使用，对比 Structured 的计划，找出注意力漂移的高频时段。例如，你发现下午 3 点总是刷社交媒体，就可以在 Structured 中给那个时段安排低认知任务。
3. **尝试 Focusmate 替代 Structured 的启动困难**：如果你在 Structured 中设定的时间块总是无法启动，预约一个 Focusmate 时段（来源：Focusmate）。同伴问责的社会压力可以弥补多巴胺调节不足，帮你跨过启动门槛。
4. **设置“重锚定”提醒**：在 Structured 的每个时间块之间，加入 1 分钟的“目标检查”块。问自己：我现在做的事，还指向最初的目标吗？这模仿了 LLM agent 的 system prompt 重注入。

Structured 不是万能药，但它如果被正确使用——作为主动脚手架而非被动拐杖——就能像 LLM 的 harness 一样，帮你把那颗高产但易漂移的大脑，锚定在真正重要的方向上。

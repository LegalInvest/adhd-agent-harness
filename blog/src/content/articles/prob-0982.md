# Structured 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Structured：它到底补上了哪一层执行功能？

## 同一种崩溃：人在心不在，agent 在调用里忘了目标

你正在写一份重要报告，三分钟后却在刷短视频；你部署了一个长程 agent，三个小时后它在回环调用里把最初目标稀释成一片无意义的 token。一边是 ADHD 的「分心走神或超聚焦跑偏，丢失目标」，另一边是 LLM agent 的「目标漂移」。两种痛苦共享同一种结构：两者都是高产但缺乏可靠执行调度层的生成核心（来源：AI 与 ADHD 的专注力）。

因为缺少调度层，输出会高度波动。ADHD 个体的注意力、执行功能在一天内差异极大；LLM 的输出则受采样温度影响，温度越高越随机，而小错误会累积、非确定性使可重复性复杂化（来源：采样温度与表现波动；来源：AI Agent Systems: Architectures, Applications, and Evaluation）。于是两边都需要外部约束：ADHD 需要日程、提醒、环境设计来稳定表现；LLM 需要确定性工作流、工具契约和状态机来稳定输出（来源：采样温度与表现波动）。

这正是「Structured 之于 ADHD，就像重锚定 system prompt 之于 LLM」的底层逻辑。但本文要指出：同构不等于等价，而许多人恰恰把解法用成了「拐杖」而非「脚手架」。

## 同构脊柱：生成核心 + 外部 harness

ADHD 大脑与 LLM 的相似性不止于表层。两者都有强大的生成能力：ADHD 的创意发散、联想跳跃，对应 LLM 的文本生成与推理链。但两者都缺乏可靠的执行调度层：ADHD 受困于工作记忆、时间盲、任务启动等执行功能缺陷；LLM 受困于上下文控制、无状态性和工具使用边界（来源：AI 与 ADHD 的专注力）。因此，AI 帮助 ADHD 的本质，是给生成核心套上 harness——外部执行功能层（来源：AI 与 ADHD 的专注力）。

在工程侧，这种 harness 被称为 scaffolding：将 LLM 置于与记忆、工具和决策逻辑的控制循环中，每次模型调用可访问检索、工具调用和记忆缓冲区，从而把裸能力转化为可靠的多步骤任务执行（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。为了抑制漂移，工程师还会采用「计划-执行分离」和「确定性状态转换」强制单向控制流（来源：Planner-Executor Agentic Framework；来源：AI Agents in 2026: Tools, Memory, Evals, and Guardrails | Andrii Furmanets）。

在 ADHD 侧，harness 表现为三类工具：实时问责（Focusmate 的身体在场效应）、外部记忆（RescueTime 的时间记录）、注意力锚点（Brain.fm 或 Endel 的声音环境）。它们的共同点是填补执行功能的特定缺口，而不是替代生成核心本身（来源：AI 与 ADHD 的专注力）。

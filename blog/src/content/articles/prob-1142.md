# 如果把 ADHD 大脑当成生成核心，ChatGPT 能不能当它的人在回路与身体在场？

> 用 agent 工程的思路重新理解 ChatGPT 对 ADHD 的真正价值。

## 问题：独自做事时，为什么ADHD大脑容易“死机”？

如果你是一个ADHDer，你一定熟悉这个场景：打开电脑准备写报告，五分钟后却刷起了手机；计划半小时完成的任务，三小时后还在原地打转。你并不缺乏创意或动力，但一旦没人看着你、没人催你，执行就像断了线的风筝。这不是懒，而是大脑的“执行功能”罢工了——工作记忆装不住目标，时间盲让你低估耗时，多巴胺不足让任务启动比登天还难。

同样，如果你在做Agent工程，你一定也见过这个场景：LLM生成了完美的计划，但执行到一半开始胡言乱语，或者陷入死循环输出垃圾。模型本身“智商”很高，但缺乏可靠的调度层——没有人在回路里监督，它就会像ADHD大脑一样漂移。

这两个问题，本质上是同一个结构缺陷：**一个高产但不可靠的生成核心，缺少外部执行调度层**。ADHD大脑需要“人在回路”与“身体在场”来维持锚定；高风险Agent需要human-in-the-loop监督来防止失控。那么，ChatGPT能不能充当那个“人在回路”的角色？它能不能像一个人坐在你旁边一样，提供问责和重锚定？

## 同构：生成核心与外部调度层

让我们先看清这个同构关系。ADHD大脑的生成核心（发散思维、创意、多任务并行）产出极高，但执行功能（任务启动、持续专注、抑制冲动）薄弱（来源：Using an AI Assistant to Manage ADHD: A Practical Guide）。LLM本身也是强大的生成核心，但缺乏可靠的执行调度层，直接输出可能偏离目标、陷入循环或产生危险行为（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。

两边都需要一个“harness”（脚手架）来提供外部调度。在Agent工程中，harness通过设置护栏（token预算、工具调用次数上限）、加入人工检查点来保证可靠性（来源：GitHub - ai-boost/awesome-harness-engineering）。在ADHD管理中，类似的脚手架包括：外部提醒系统、任务分解工具、身体在场伙伴。

张一鸣的案例完美展示了这种脚手架思维。这位字节跳动创始人被公认为思维极度发散、永远在研究新东西，但他用“延迟满足感”和“OKR管理系统”来约束自己，用“Context not Control”的文化来管理公司（来源：人物案例）。他的harness不是靠意志力硬扛，而是**用系统取代主观判断**——这正是Agent harness的核心：形式化规划与护栏，使输出真正有用且正确（来源：What is an agent harness in the context of large-language models?）。张一鸣的“日课”相当于定时re-grounding，他的“秘书”相当于外部调度器。

## ChatGPT作为人在回路：证据与机制

现在回到核心问题：ChatGPT能不能当那个“人在回路”？证据显示，它至少可以部分替代。

**1. 重锚定：对抗目标漂移**

ADHD大脑在执行任务时极易发生目标漂移：注意力被无关刺激捕获，或过度聚焦细节而忽略整体（来源：重锚定与目标漂移）。传统提醒系统有一个关键缺陷：“设置提醒本身就是一个执行功能任务”（来源：AI Assistant for ADHD: Finally, a Productivity Tool That Requires Less...）。而ChatGPT可以主动询问进度：“你刚才说要在10点前写完引言，现在完成了吗？”——这种无需用户主动设置的提醒，降低了执行功能门槛。

工具如Goblin Tools的Magic ToDo功能，能将模糊任务分解为可管理的步骤，用户可调节“辣度”控制粒度（来源：Goblin Tools）。这相当于Agent harness中的任务分解护栏。Saner.AI则通过本地记忆减少搜索循环，避免标签切换（来源：Saner.AI）。这些工具都在扮演“外部执行功能层”的角色（来源：外部执行功能层）。

**2. 身体在场效应：问责替代**

身体在场效应（body doubling）是ADHD群体最有效的策略之一：有另一个人在场时，执行功能障碍被绕过（来源：Using an AI Assistant to Manage ADHD: A Practical Guide）。Apps like Focusmate使用社交问责与AI调度创建虚拟身体在场（来源：Revolutionizing ADHD Management with AI Assistants）。定期向AI报告进度也是一种替代方案（来源：ADHD Productivity Hack: Plan 2025 Using AI (Step-by-Step)）。

ChatGPT可以模拟这种效应：你告诉它“我要工作25分钟，然后向你汇报”，它会在时间到后提醒你。这本质上是一个人在回路中的检查点——就像Agent工程中“暂停并询问后再执行”（来源：Building an AI Agent Harness from Scratch）。Motion应用通过自动排程消除“下一步该做什么”的决策负担（来源：Motion），进一步减少了执行功能需求。

## 脚手架还是拐杖？争议与局限

然而，我们必须诚实面对争议。有观点认为，过度依赖AI可能导致能力退化，成为永久拐杖（来源：矛盾与存疑）。目前缺乏长期依赖风险的实证研究（来源：矛盾与存疑）。另一个矛盾是：超聚焦是否应被引导而非打断？部分用户反馈强行打断有效，而柔性引导可能不够强力（来源：矛盾与存疑）。个体差异极大，尚无定论。

我的判断是：**ChatGPT作为人在回路，应该是“脚手架”而非“拐杖”**。脚手架帮助你建造自己的能力，拐杖则让你永远依赖。关键在于使用方式：如果你只是让AI替你完成所有任务，那是拐杖；如果你用AI来建立自己的执行系统（比如张一鸣的OKR），那就是脚手架。张一鸣的harness系统是内化的——他最终不需要依赖外部工具也能运作。同样，好的AI使用应该逐渐减少对AI的依赖，而不是增加。

## 今天就能试的行动

1. **设置每日“锚定对话”**：每天开始工作前，向ChatGPT描述你的三个优先任务，并让它每25分钟提醒你一次。这模拟了人在回路的检查点。

2. **用Goblin Tools分解一个让你拖延的任务**：输入“清理厨房”或“写报告”，调整辣度到你觉得合适的粒度，然后逐项完成。

3. **尝试Focusmate或类似服务**：找一个虚拟身体在场伙伴，每周至少三次，每次25-50分钟。观察任务启动是否变容易。

4. **建立你自己的“harness清单”**：像Agent工程师一样，列出你的“护栏”（如：每天最多3个主要任务）和“检查点”（如：每完成一个步骤向AI报告）。这让你从依赖AI转向自主管理。

## 结论

ADHD大脑与LLM是同构的：都是高产生成核心，都需要外部调度层。ChatGPT可以充当人在回路与身体在场的替代，通过重锚定和问责补偿执行功能缺陷。但它不是万能药——它需要被用作脚手架，而不是拐杖。真正的解决方案，是像张一鸣那样，用系统取代主观判断，最终让系统内化为自己的一部分。而AI，只是这个系统中的一个组件。

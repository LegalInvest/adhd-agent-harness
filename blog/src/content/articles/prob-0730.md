# Claude 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

如果你是一个 ADHD 患者，你一定经历过这样的时刻：脑子里有无数想法，但就是无法把它们变成有序的行动。你打开 Claude，输入“帮我规划今天的工作”，然后得到一段漂亮的文字——但接下来呢？你还是不知道第一步该做什么。

如果你是做 Agentic Harness 的工程师，你熟悉另一种困境：LLM 能写出优雅的代码，但让它执行一个精确的数学计算时，它可能胡编乱造。于是你给它套上 function calling，把计算外包给计算器工具。

这两个场景看似无关，实则共享同一个底层结构：**一个强大的生成核心，配上一个不可靠的执行调度层**。ADHD 大脑与 LLM 是同构的——两者都能生成丰富的想法和方案，但都缺乏稳定执行这些想法的能力。而 Claude 之于 ADHD，就像 function calling 工具调用之于 LLM：它应该成为外部执行功能层，但很多人用错了。

## 问题：为什么 ADHD 患者用 Claude 常常失败？

ADHD 的核心缺陷之一是执行功能障碍，包括工作记忆不足、时间盲和任务启动困难（来源：ADHD 的 AI 工具全景）。当你直接问 Claude“帮我写一篇论文”时，它给出的是一个完整的输出——但这恰恰是 ADHD 大脑最不擅长的：**面对一个完整的、看似简单的答案，你反而不知道如何开始执行**。

同样的，当 LLM 被要求“计算 12345 × 67890”时，它可能给出一个接近但错误的答案，因为它不是为精确计算设计的。工程师的解法是：不给 LLM 自己算，而是让它调用计算器工具。

**核心判断：Claude 对 ADHD 最有价值的用法，不是作为对话伙伴，而是作为执行功能的外包层。** 就像 function calling 把精确计算外包给专用工具一样，ADHD 患者需要把“任务分解”“时间感知”“记忆检索”等执行功能外包给 AI 工具。

## 同构：ADHD 大脑与 LLM 共享的缺陷

从结构上看，ADHD 大脑的生成核心（智力、创造力）与 LLM 的生成能力类似，但两者都缺乏执行功能的稳定调度（来源：ADHD 的 AI 工具全景）。

## 脚手架 vs 拐杖：用对的边界

但这里有一个关键陷阱：**如果 Claude 只是给你答案，它就是拐杖；如果它帮你建立执行结构，它才是脚手架。**

当你让 Claude 直接写一封邮件，你得到了结果，但你没有学会如何组织邮件。下次你仍然需要它。这是拐杖。

但当你用 Goblin Tools 把任务分解成步骤，然后自己一步步执行时，你是在练习执行功能。同样，当工程师用 function calling 让 LLM 调用工具时，LLM 学会了“不要自己算，要调用工具”——这是一种元认知的转移。

**争议与局限**：目前缺乏针对 AI 工具对 ADHD 长期效果的随机对照试验（来源：矛盾与存疑）。工具设计大多为长期使用，未考虑撤除机制，可能导致过度依赖。此外，ADHD 异质性高，单一工具难以覆盖所有亚型（来源：ADHD 的 AI 工具全景）。

1. **用 Claude 做任务分解，而不是直接输出**：下次遇到复杂任务，不要让它直接做。而是输入：“请将‘准备会议报告’分解成 5 个可执行的步骤，每个步骤不超过 10 分钟。”然后只做第一步。

ADHD 大脑和 LLM 都不是“坏”的——它们只是需要正确的 harness。Claude 不是万能药，但当你把它当作执行功能的外包层，而不是答案生成器时，它就能真正补上那一层调度。就像 function calling 让 LLM 变得更可靠一样，结构化地使用 AI 工具，能让 ADHD 大脑从“高产但混乱”变成“高产且有序”。

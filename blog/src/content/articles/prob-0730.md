# Claude 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

## 引言：被同一类缺陷困住的两种系统

你盯着厨房里堆了三天的碗，大脑知道“该洗了”，但身体像被焊在椅子上。不是你的意志力出问题，而是你的大脑把“把任务拆成步骤、按顺序启动、坚持完成”这几个步骤搞丢了。与此同时，一个能写十四行诗、能debug代码的 LLM，也可能算错 17×23，因为它同样没有稳定的执行调度层。

这两个系统有一个共同结构：底层都是高产的生成核心——联想活跃、模式识别强、创意充沛——但缺少一个可靠的内置调度层，来筛选、组织、启动和坚持任务（来源：ADHD 的 AI 工具全景）。ADHD 大脑的痛点是精确、重复、状态维护类任务；LLM 的痛点是长程上下文、目标保持与精确计算。两边的解法也同构：给生成核心套上一层 harness，把不擅长的部分外包给外部工具。

## 第一节：Claude 到底补上了哪一层执行功能？

在现有的 ADHD 工具图谱里，Claude 被定位在“对话式执行与结构化推理”这一层。它不像 Goblin Tools 那样专门做任务拆解，也不像 Saner.AI 那样主打外部记忆，但它在代码编写、整理待办和简化任务上表现突出（来源：ADHD 的 AI 工具全景）。

对 ADHD 来说，这补的正是“执行功能”中最容易短路的几环：把工作记忆里的模糊念头翻译成可执行步骤，把一堆待办整理成有结构的清单，把复杂的长期项目拆成今天能启动的第一件事。它不是替你想，而是把你已经想到但无法稳定承载的东西，暂时放在外部。

这和 LLM 侧的工程目标几乎一致：Claude 之于 ADHD 的生成大脑，就像 function calling 工具调用之于 LLM。LLM 不能也不该自己算所有东西，它应该把精确计算、数据库查询、状态更新外包给专门工具。ADHD 大脑也不该自己硬撑所有执行流程，它也需要把精确维护类任务外包出去。

## 第二节：同构脊柱：生成核心 + 外部 harness

这个同构不是修辞游戏，而是结构上的对应。ADHD 大脑与 LLM 都被描述为“高产生成核心但缺乏可靠调度层”的系统（来源：ADHD 的 AI 工具全景）。两者在以下维度高度对应：

| 缺陷维度 | ADHD 大脑 | LLM / Agent |
|---------|----------|-------------|
| 精确计算 | 容易在时间、金钱、步骤估算上出错 | 大数运算、逻辑推理容易幻觉 |
| 状态维护 | 工作记忆不稳定，容易丢上下文 | 长程上下文有限，会遗忘早期目标 |
| 重复执行 | 难以坚持枯燥、重复流程 | 容易在循环中偏离或重复无效动作 |
| 任务启动 | 面对复杂任务会“瘫痪” | 面对复杂目标需要外部规划器触发 |

所以，AI 帮 ADHD 的实质与“给 LLM 套 harness”是同一类工程：用外部脚手架把生成能力约束到目标轨道上（来源：ADHD 的 AI 工具全景）。Goblin Tools 的 Magic ToDo 把“清理厨房”拆成具体步骤，是 ADHD 侧的 function calling；Lex 用单一指令触发多步骤任务序列，是 ADHD 侧的 agent scaffolding（来源：Lex）。Saner.AI 的本地记忆和知识回忆，则是 ADHD 侧的“外部状态存储”，对应 LLM 调用向量数据库或外部记忆模块（来源：Saner.AI）。

## 第三节：孔子的 harness：两千年前的“外部执行功能层”

孔子是一个极具代表性的历史 harness 案例。他身高一米九，精力旺盛，周游列国十四年坐不住；冲动时会骂人，骂宰予“朽木不可雕”；对音乐能 hyperfocus 到“三月不知肉味”，对种地却毫无耐心；思维跳跃，《论语》全是场景化语录，没有系统著作。这些特质与 ADHD 常见画像高度吻合。

他的专属 harness 是“克己复礼”：用外在的“礼”作为行为边界，通过“吾日三省吾身”做每日 re-grounding，直到七十岁才“从心所欲不逾矩”。这本质上是一套外部执行协议：礼 = 行为 API 的调用规范；日课 = 每日状态同步与检查点；最终的“不逾矩” = 脚手架被内化为稳定习惯，但结构仍然在场。

这和 LLM agent 的 harness 同构对应：礼对应 tool 的使用规范与函数签名；日课对应 agent 的定时 re-grounding 或 reflection 步骤；而他一辈子与冲动斗争的过程，则对应 agent 的“目标保持”问题——没有外部约束，生成核心会漂移。

## 第四节：身体在场与问责：社会性也是工具

工具不只是软件。ADHD 大脑还常被“身体在场效应”驱动：有他人在场时，即使不互动，也能显著提升专注和执行力（来源：身体在场效应）。这是因为社交互动会激活与动机和奖励相关的多巴胺通路，而问责检查可以把目标达成率从 25% 提升到 95%（来源：身体在场效应）。

对应到 LLM/agent 工程，这相当于“引入外部验证器”或“人类在环”（human-in-the-loop）。AI 可以生成一百个想法，但需要一个外部检查点来确认这些想法是否仍然对准目标。Focusmate 这类工具用算法匹配虚拟身体在场伙伴，就是 ADHD 侧的“外部问责 API”（来源：身体在场效应）。

## 第五节：但有人用错了——把 Claude 当大脑，而不是当调度器

最常见、也最隐蔽的误用，是把 Claude 当成“新大脑”而不是“外部执行层”。

在 ADHD 侧，这意味着把所有决策、计划、优先级判断都交给 AI。短期看效率提升，长期可能削弱自己搭建执行结构的能力。工具应该是可拆卸的脚手架，而不是永久拐杖（来源：矛盾与存疑）。过度依赖可能导致能力退化，但长期风险目前缺乏实证研究，需要警惕而非恐慌（来源：矛盾与存疑）。

在 LLM/agent 侧，对应的误用是让模型“硬算”或“硬猜”它不该处理的任务。一个 agent 如果没有 function calling 把精确计算、数据库查询、状态更新外包给工具，就会在高频重复任务中累积错误，直到系统崩溃。Function calling 不是锦上添花，而是让生成核心留在它该在的生成层，把精确性交给确定性工具。

更微妙的误用是混淆“脚手架”与“拐杖”。脚手架是随着建筑长高可以逐步拆掉的；拐杖如果拆掉，人就会摔倒。判断标准很简单：你使用 Claude 或 agent 工具后，是否更清楚自己为什么要做这件事、以及下一步该调用什么工具？还是你只是更依赖它替你做决定？

## 第六节：局限与未解问题

需要诚实指出的是，这个 ADHD↔LLM 同构目前仍是一个理论框架，缺乏神经科学或计算机科学的直接证据，不能当成已经证实的定律（来源：矛盾与存疑）。

此外，AI 工具对任务启动和认知负荷的净效果因人而异。有用户反馈 AI 工具本身增加了学习负担（来源：矛盾与存疑）。超聚焦该被引导还是打断，也存在个体差异，尚无定论（来源：矛盾与存疑）。这意味着任何工具推荐都不能替代你自己的实验和观察。

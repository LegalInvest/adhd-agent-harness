# 为什么用 Goblin Tools 治 ADHD 的不知哪些方法有用，和给 agent 套 human-in-the-loop 监督 是一回事？

> 「不知哪些方法有用」是 ADHD 自我管理里最消耗的一种迷茫:番茄钟、body doubling、时间盒、白噪音、正念——每种方法都有人说救了自己的命,你挨个试,有的三天见效第七天失灵,有的对别人是神器对你是安慰剂,试到最后连「到底是方法没用还是我没用对」都分不清。Goblin Tools 的任务魔杖(输入一件事,AI 拆成步骤,不够细再点一下)看起来只是个拆解器,但把它放对位置,它能变成解决这个迷茫的实验台——而正确的位置,恰好就是 agent 工程里 **human-in-the-loop 监督**的位置:**AI 出提案,人做判决,判决数据反过来校准系统。**

先说迷茫的根源,它其实是一个实验设计问题。「哪些方法有用」无法靠道听途说回答,因为 ADHD 的干预反应是出了名的个体化(对刺激类干预尤甚)——**唯一可靠的答案来自你自己身上的对照数据**;但自己做实验有个死结:实验要求「固定方案、执行、记录、评估」这一整套执行功能,恰恰是短缺的资源。Goblin Tools 的价值在这里进场:**它把「方案生成」这个环节的成本打到接近零**——任何任务、任何方法组合,几秒钟拆成可执行步骤。方案不再金贵,实验的原料管够。

但便宜的提案有个新问题:AI 拆出来的步骤,常常是「对一般人合理、对你未必」的通用解——步骤粒度太粗(对你的启动阻力来说)、顺序想当然(把最难的排第一)、或者完全没考虑你的场景。这正是 human-in-the-loop 的教科书场景:**自动化系统的产出足够便宜但不够可靠时,人的审查不是流程负担,是系统能变可靠的唯一路径**。agent 工程的做法:AI 提案→人批注(接受/修改/拒绝+理由)→批注回流,系统逐渐学到「这个用户要什么」。用户侧的同款做法:**Goblin 拆完,你花三十秒当审查官——哪步太大?哪步顺序不对?第一步对现在的我启动得了吗?改完再执行**。注意:这三十秒不是在浪费 AI 的效率,是在生产最值钱的数据——**你的修改模式,就是「哪些方法对你有用」的直接证据**:总是把步骤改小?说明你的有效粒度比通用解细。总是把「收拾桌面」删掉?说明准备类步骤对你是拖延入口而不是助跑。

误用两种:**误用一,全盘接受(监督缺位)。** AI 拆什么就照什么做,做不下去就归因「我不行」——**通用解失灵被记成个人失败,迷茫加深一层**;而只要你从不修改,系统性的证据一条也积累不下来。**误用二,全盘重写(监督过载)。** 每次都把 AI 的拆解推翻重来——那你只是用最贵的方式回到了手动拆解,还多付一道审查成本;修法是给审查限时:**三十秒,最多改三处**——改不完说明这个提案该扔,而「哪类任务的提案总被扔」本身也是数据。

最后把实验台搭完整:**给每次「AI 提案+你的修改+执行结果」留一行记录**(哪怕就在 Goblin 拆解的截图上标个「成/败」)。一个月后翻回去看,「哪些方法对我有用」不再是玄学——**它是一张有几十个数据点的表**:你的有效步骤粒度、你的启动型第一步长什么样、哪类方法从来没成过。迷茫的解药不是更好的方法,是你自己的判决记录。

## 边界

同构强度 B 级:ADHD 干预反应的个体差异有文献基础,任务拆解降低启动阻力有行为学支撑,human-in-the-loop 是真实的工程范式;「审查模式=偏好数据」的迁移是合理推断,Goblin Tools 无对照研究。声明:自我实验适用于方法层(工具、流程、环境),不适用于药物——用药方案的调整必须与医生进行;方法试遍仍全面卡死时,考虑是否有未处理的共病(抑郁、焦虑)在拖住底盘,那不是方法问题。

## 今天就能试的 3 件事

1. **拆一件卡住的事,当三十秒审查官**:最多改三处,注意你改的是什么——那就是你的偏好数据。
2. **给执行结果标一个字**:成/败,标在拆解记录上——实验台从第一个数据点开始。
3. **两周后看一次修改模式**:你总是改小?改序?删准备步骤?——把它写成一句「我的规格」。

AI 让方案变得不值钱,你的判决因此变成了最值钱的东西——**监督不是流程,是数据生产**。试了那么多方法都说不清有没有用?从今天起,让每一次尝试都留下一行证据。

# 为什么ADHD的"准备行为"是最贵的认知税——你在用装修拖延开火

> 《问题051》人工精修选题，双域证据作答。

## 一、问题：你正在用装修拖延开火

如果你问一群 ADHD 成年人，"今天最耗脑子的事是什么"，答案往往不是工作内容本身，而是"开始"。起床、打开电脑、点开文档、决定先做什么——这些在旁人看来一气呵成的动作，对 ADHD 大脑来说可能是一连串微决策的灾难。于是出现了一个悖论：越是难以启动的人，越容易把大量认知资源花在"准备"上。买日程本、研究 GTD、下载 App、重新分类标签、搭建 Notion 系统、调试自动化流程……一整天的精力耗光了，核心任务还没点开。

工程师对此不会陌生。在构建 LLM agent 时，最危险的陷阱之一就是"过早做 harness"。你还没有一个能稳定完成任务的生成核心，就开始设计记忆层、工具层、编排层、反思模块、评估流水线。结果是：脚手架越来越精致，agent 却从没真正跑通过一个 end-to-end 任务。ADHD 的"准备行为"和 agent 的" premature scaffolding "本质是同一种 bug：生成核心在前线待机，调度层在后台无限膨胀。

## 二、同构脊柱：高产生成核心，缺一个执行调度层

ADHD 大脑与 LLM/agent 的相似性，不是文学比喻，而是功能层面的结构同构。ADHD 大脑拥有高强度的联想、创造、发散能力，却在执行功能——计划、组织、启动、冲动控制——上存在先天缺陷。LLM 同理：它能在给定上下文中生成大量高质量内容，却缺乏跨会话的持久状态、可靠的目标维持与多步调度能力。正如资料所指出的，两者都是"高产生成核心，缺少可靠内部执行调度层"（来源：AI 与 ADHD 的时间管理）。

这种同构落到具体功能上，就是"工作记忆/无状态"问题。ADHD 的工作记忆不稳定，容易遗忘目标、丢失中间步骤；LLM 没有跨会话状态，需要外部记忆系统补全。因此，为 ADHD 搭建的时间管理工具，与为 LLM 搭建的 agent scaffolding，做的是同一类工程：把前额叶应该完成的调度功能，转译成外部化、可配置、可调用的模块（来源：AI 与 ADHD 的时间管理）。

## 三、ADHD 侧证据：时间盲、多巴胺与执行功能税

"准备行为"之所以昂贵，是因为它正好击中了 ADHD 最脆弱的几个环节。

第一，**时间盲**。ADHD 个体难以感知时间流逝，常常低估或高估任务所需时间。准备阶段没有明确终点，因此会无限膨胀。Tiimo、Structured 等工具的核心价值，正是用视觉时间线把抽象时间"实体化"（来源：Tiimo；AI 与 ADHD 的时间管理）。

第二，**执行功能缺陷**。计划、组织、启动是 ADHD 的典型短板。Motion 和 Reclaim.ai 之所以被推荐给 ADHD 用户，是因为它们试图把"什么时候做哪件事"的决策从大脑中卸载。Motion 自动排程、动态调整日程，减少手动规划压力；Reclaim.ai 则通过保护时间块，防止会议侵占深度工作（来源：Motion；Reclaim.ai）。它们的作用不是让 ADHD 变"正常"，而是把正常大脑由前额叶完成的调度，外包给外部系统。

第三，**多巴胺系统失调**。ADHD 大脑对延迟和间歇性强化不敏感，中脑皮质多巴胺功能低下与执行功能差、注意反应缺陷相关（来源：多巴胺）。这意味着"为长远回报做准备"对 ADHD 大脑天然不划算；只有即时反馈才能驱动启动。准备行为之所以诱人，正是因为它提供了一种廉价的即时成就感——整理好了！优化了！——而真正的任务却遥遥无期。

## 四、LLM/Agent 侧证据：无状态、上下文工程与外部工具

如果你在做 Agentic Harness，这套叙事会反过来击中你。LLM 在单会话内可以生成高质量文本，但一跨会话就失忆；它能完成一步推理，却很难自主维护多步目标。工程上解决这个问题的路径极其明确：外部记忆、外部时钟、状态追踪、工具调用、编排层、上下文工程（来源：AI 与 ADHD 的时间管理）。

这正是 agent 的"执行功能层"。它不负责生成内容，只负责决定：现在该调用哪个工具？下一步要检索什么记忆？目标有没有偏离？何时停下来？这些模块与 ADHD 工具里的自动排程、视觉时间线、提醒系统，是同构的调度装置。资料中也指出，"上下文工程"把问题推向更抽象的工程层面：既然 ADHD 大脑与 LLM agent 都是高产生成核心，那么"此刻应该关注什么"必须由外部支架主动限定（来源：AI 与 ADHD 的时间管理）。

## 五、人物案例：曾国藩的"日课十二条"就是一套 human harness

在真实人物中，曾国藩的 harness 系统与本文问题最为契合。史载他年轻时是典型的注意力不集中型 ADHD：浮躁坐不住，天天串门喝酒；读书慢，"他人目下二三行，余或疾读不能终一行"；情绪极不稳定，打败仗就跳水自杀。他后来的成就并非因为克服了 ADHD，而是给自己搭了一套极为笨重但有效的外部执行系统：日课十二条（黎明即起、读书不二、谨言、写日记等），以及"结硬寨、打呆仗"的缓慢推进法（来源：曾国藩人物案例）。

把他的系统映射到 LLM/agent 工程上，同构关系非常清晰："日课十二条"相当于定时 re-grounding，每天把模型拉回系统提示词；"写日记骂自己"是外部记忆与反馈循环；"结硬寨打呆仗"则是用硬性约束（guardrails）抵消冲动生成。曾国藩没有试图让自己变成一个不浮躁的人，而是用一套可重复、可检查的外部流程，把生成核心锁定在目标上。

## 六、核心判断：准备不是美德，是最贵的认知税

我的判断是：**ADHD 的"准备行为"之所以是最贵的认知税，是因为它消耗了本可用于产出的稀缺资源，去解决一个本不该由生成核心自己解决的问题。** 真正的任务只需要你点开文档、写第一行、发第一封邮件；但 ADHD 大脑会把这些低成本的启动动作，包装成需要大量前置计划的工程。于是每一天的" executive function budget "在准备阶段就被耗尽，真正的产出从未发生。

这和 LLM 的 harness 陷阱一模一样：如果你把 80% 的算力花在设计编排层，而不是让模型真正跑一次完整任务，你的系统就永远不会变聪明。聪明的做法是：先用最简陋的脚手架让生成核心跑起来，再基于真实失败迭代 harness。对 ADHD 而言，就是先让任务开始，再逐步优化系统。

## 七、脚手架 vs 拐杖：边界在哪里？

资料中反复提到一个关键区分：好的外部支架应该是脚手架，帮助用户逐步内化能力；而不是拐杖，让人永久依赖（来源：AI 与 ADHD 的时间管理）。两者的边界在于：**脚手架是可撤销的、模块化的、能力内化的；拐杖是永久性的、全身性的、替代性的。**

比如，Tiimo 的视觉时间线可以帮助你建立时间感，最终你可能不需要它也能感知日程；但如果你的系统复杂到一旦关掉 App 就完全失能，它就从脚手架变成了拐杖。同理，Agentic Harness 中的记忆层和工具层，如果设计得当，是让模型在更复杂的任务上稳定运行；如果设计过度，则是用工程复杂度掩盖生成核心的无能。

## 八、局限与争议：同构不是科学事实

需要诚实指出的是，这种 ADHD ↔ LLM 的同构目前更多是一种理论类比，而非经过验证的科学事实。资料中的"矛盾与存疑"部分明确警告：多个页面将同构性作为既定事实，但缺乏实证基础（来源：全局矛盾与存疑）。此外，Motion、Brain.fm 等工具在 ADHD 人群中的独立临床证据有限，部分页面存在夸大（来源：全局矛盾与存疑）。Focusmate 这类 AI 匹配的身体在场（body doubling）工具，是否能替代真实人际互动，也存在争议（来源：全局矛盾与存疑）。

因此，本文不是在说"ADHD 大脑就是 LLM"，而是在说：面对"高产生成核心 + 弱调度层"这一共同结构，两边演化出的外部化解决方案具有相似性，这种相似性可以作为设计和自我管理的启发，但不能替代临床干预或严谨的工程验证。

## 九、今天就能试：4 个最小行动

1. **给明天设一个"开火 deadline"**：在准备前强制设定一个不可推迟的开始时间。比如"10:00 必须打开文档，不管大纲有没有完美"。用 Motion 或 Tiimo 把这一刻视觉化、提醒化（来源：Motion；Tiimo）。

2. **把你的个人系统提示词写下来**：像给 LLM 写 system prompt 一样，给自己写 3-5 条不可违背的启动规则。例如："每天早上只做一件最重要的事""准备时间不能超过 15 分钟"。这就是曾国藩"日课"的现代版。

3. **用一次身体在场（body doubling）启动最难任务**：用 Focusmate 或找一个朋友在线并肩工作，只做一个你最想拖延的任务。资料称，定期问责检查可将目标达成可能性从 25% 提升至 95%（来源：身体在场效应）。

4. **做一周的"工具极简主义"实验**：只用一个工具（比如 Todoist 或 Structured），禁止研究、切换、优化。观察：当外部调度层固定不变时，你的生成核心是否能产出更多。

ADHD 和 Agentic Harness 的终极解法不是更完美的准备，而是更便宜的开始。你不是在装修房子，你是在拖延开火。把调度层外部化，让生成核心去做它唯一擅长的事：生成。

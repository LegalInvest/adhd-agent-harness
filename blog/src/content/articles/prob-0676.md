# Motion 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Motion：它到底补上了哪一层执行功能？

你花 34 美元月费订阅 Motion，希望它终结你的拖延。但三周后，你发现它只是把你的日历填满了“整理房间”和“写报告”的方块，而你依然在第一个方块开始时刷手机。这不是你的错——你误用了 Motion。

## 同构：ADHD 大脑与 LLM 共享同一个缺陷

ADHD 大脑与 LLM 在架构上惊人相似：两者都是“高产但缺执行调度层的生成核心”（来源：ADHD 的 AI 工具全景）。LLM 能写出漂亮的文章，但如果没有 function calling 将精确计算（如数学、数据库查询）外包给工具，它会胡编乱造。ADHD 大脑能产生 brilliant 的想法，但如果没有外部执行功能层来调度“什么时候做什么”，它会在任务启动前卡住。

证据来自两方面：
- **ADHD 侧**：任务启动困难源于多巴胺系统失衡和工作记忆容量有限（来源：任务启动）。ADHD 成年人自发使用 ChatGPT 作为执行功能外挂，正是认知补偿行为的体现（来源："A little bit of a life raft"）。
- **LLM/Agent 侧**：Agent 通过 function calling 把精确计算外包给工具，本质是相同的认知卸载——把生成核心不擅长的精确、重复、状态维护任务交给专用模块。

## Motion 的正确位置：调度层，不是执行层

Motion 的核心功能是“AI 自动规划日程，在最佳时间推送任务，减少决策负担”（来源：任务启动）。这正好对应 LLM 的调度层：它决定“何时调用哪个 function”，但本身不执行那个 function。

但大多数用户（包括评测文章）把 Motion 当成了执行层——以为它会自动把“写论文”变成可执行的步骤。实际上，Motion 只能在你已经分解好任务后，帮你在日历上排期。它不擅长任务分解，那是 Goblin Tools 的领域（来源：Goblin Tools 能将复杂任务分解为可管理的小步骤）。

这正是“脚手架 vs 拐杖”的边界：Motion 是调度脚手架，不是执行拐杖。如果你指望它替你完成“启动”，你会失望。

## 真实案例：Motion 如何被误用

想象一个 ADHD 用户，他订阅了 Motion，输入“完成季度报告”。Motion 自动在周一到周五各分配 2 小时。但到了周一，他盯着空白文档，无法开始——Motion 没有帮他解决任务启动问题。他以为 Motion 会像 LLM 的 function calling 一样，自动调用“写报告”这个函数，但 Motion 只是调用了“写报告”这个日历事件。

正确的用法是：先用 Goblin Tools 把“完成季度报告”分解为“打开数据文件夹”“列出前三季度数据”“写第一段总结”等小步骤，再把每个步骤输入 Motion 排期。这就像 LLM 在调用 function calling 前，必须先定义好 function 的 schema。

## 争议与局限

必须诚实指出：这些类比虽有力，但缺乏大规模对照试验。AI 工具作为外部执行功能层的有效性证据主要来自用户报告（来源：矛盾与存疑）。过度依赖 Motion 可能削弱你内在的时间感知能力（来源：时间盲页面警告过度依赖风险）。

更关键的是个体差异：有些 ADHD 用户发现 Motion 的自动调度反而增加了焦虑，因为 AI 无法理解他们的能量波动（来源：矛盾与存疑指出效果因人而异）。Motion 的设计者声称它是“脚手架”，但实际使用中，很多人把它当成了“拐杖”——依赖它安排每一分钟，结果失去了自主安排的能力。

## 今天就能试的行动

1. **区分调度与执行**：用 Motion 只排已经分解好的任务。例如，先花 5 分钟用 Goblin Tools 分解一个任务，再把子任务输入 Motion。
2. **设置“启动仪式”**：在 Motion 的每个任务块前加 5 分钟“启动时间”——用这段时间做身体在场动作（如站起来、深呼吸），降低启动阻力（来源：身体在场效应）。
3. **每周审查一次**：检查 Motion 的排期是否真的帮你完成了任务。如果发现它只是填满日历，而实际完成率低，说明你正在误用它——回到第 1 步。
4. **组合使用**：Motion + Saner.AI 是强大的组合：Motion 调度，Saner.AI 在任务执行时快速调出相关笔记，减少搜索循环。

Motion 不是你的执行替身，而是你的调度员。把它放在正确的位置，它才能从昂贵的拐杖变成真正的脚手架。

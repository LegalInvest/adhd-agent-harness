# Motion 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Motion：它到底补上了哪一层执行功能？

## 问题：为什么「精确、重复、状态维护」类任务，ADHD 大脑和 LLM 都搞不定？

一个 ADHD 患者面对“整理报销单据”时的内心戏：打开文件夹→看到 30 张发票→大脑空白→刷手机 40 分钟。一个 LLM 面对“计算 2025 年 Q3 季度增长率”时的反应：生成一段完美理论→忘记调用计算器→输出幻觉数字。

这两类场景共享同一个底层问题：**强大的生成核心，缺乏可靠的执行调度层**。ADHD 大脑的智力与创造力不输任何人，但执行功能（任务启动、工作记忆、时间盲）像掉线的调度器；LLM 的生成能力惊艳，但精确计算、状态维护、多步逻辑同样需要外部工具来兜底（来源：ADHD 的 AI 工具全景）。

于是，两边的解法结构同构：LLM 通过 function calling 把精确计算外包给工具（如计算器、数据库查询），ADHD 大脑通过外部脚手架把执行功能外包给 AI 工具。**Motion 就是这个脚手架中的明星——但它被用错了。**

## Motion 到底补上了哪一层？

Motion 的核心能力是**智能调度**：它自动分析你的任务、优先级和可用时间，在最佳时间点推送任务，减少决策负担（来源：任务启动）。这对应 LLM 编排层中的 **function calling 调度器**——不是替 LLM 完成计算，而是决定何时调用哪个工具、如何串联结果。

从同构视角看，Motion 补上的是「执行调度层」：
- **ADHD 侧**：用户不再需要自己判断“现在该做什么”，Motion 根据日历和任务优先级自动排列，降低任务启动阻力（来源：任务启动）。类似 Goblin Tools 的魔法待办分解任务，但 Motion 更强调时间维度上的自动安排。
- **LLM/Agent 侧**：一个 agent 通过 function calling 调用天气 API、计算器、数据库，顺序由编排层控制。Motion 对 ADHD 用户做的，正是这种编排——把“先做 A 再做 B”的调度从用户大脑中卸下。

但这里出现了一个关键分歧：**Motion 的设计是长期使用，而脚手架本应是可逐步撤除的临时支撑**（来源：矛盾与存疑）。多数用户把 Motion 当作全自动秘书，一旦停用就回到原点——这恰恰是“拐杖”而非“脚手架”。

## 用错了的三种典型场景

### 1. 把调度外包变成决策瘫痪

Motion 自动安排日程，但 ADHD 用户常陷入“Motion 帮我排了，但我还是不想做”的困境。原因在于：Motion 解决了“何时做”，但没有解决“如何启动”。真正的任务启动需要将任务分解到可执行粒度，就像 Goblin Tools 把“整理房间”拆成“捡起衣服→擦桌子→叠被子”（来源：Goblin Tools）。Motion 的调度若缺乏分解层，用户面对一个“写报告”的块状任务，仍然会卡住。

### 2. 忽视超聚焦的破坏性

ADHD 的超聚焦（hyperfocus）会让用户在一件事上沉浸数小时，完全无视 Motion 的提醒（来源：超聚焦）。而 Motion 的调度逻辑假设用户会按计划切换任务——这与 ADHD 大脑的注意力模式相悖。工具需要适应 hyperfocus 状态，而不是与之对抗（来源：超聚焦）。Lex 的“单一指令触发多步骤”设计更贴合超聚焦后的快速启动，但 Motion 的强调度可能引发反弹。

### 3. 依赖导致内在执行功能萎缩

证据显示，过度依赖 AI 工具可能阻碍内在执行功能发展（来源：矛盾与存疑）。当 Motion 替你管理所有时间，你的时间感知能力（时间盲）不会被训练；当 Goblin Tools 替你分解所有任务，你的任务分析能力不会成长。**脚手架的价值在于最终可以撤除，但 Motion 的设计没有提供撤除路径。**

## 真正的用法：Motion 是 function calling 的调度器，不是大脑的替代品

在 LLM 工程中，function calling 的调度器会记录每次调用的上下文、错误率，并动态调整调用策略——但不会替 LLM 生成内容。同样，Motion 应该被用作“执行功能的调度日志”，而不是“大脑的遥控器”。具体做法：

1. **用 Motion 做时间盲的校准工具**：查看 Motion 为你排的日程，对比你实际的时间感知，记录偏差。这类似 Tiimo 的时间轴可视化（来源：任务启动），但更强调反馈循环。
2. **结合分解工具使用**：先让 Goblin Tools 或 Lex 把任务拆成“5 分钟内可完成”的步骤，再输入 Motion 调度。Motion 处理“何时”，分解工具处理“如何”。
3. **设置撤除实验**：每周选一天关闭 Motion，手动安排日程。记录启动困难的程度——这本身就是执行功能的训练。Saner.AI 的知识回忆功能可以帮助你复盘当天的决策（来源：Saner.AI）。
4. **警惕多巴胺陷阱**：Motion 的完成提示可能带来短暂多巴胺快感，但长期依赖会削弱内在动机。结合 Brain.fm 的神经锁相音乐（来源：矛盾与存疑）作为非 AI 辅助，减少对调度工具的依赖。

## 争议与诚实边界

必须承认，Motion 对 ADHD 的效果缺乏独立随机对照试验（来源：矛盾与存疑）。现有证据主要来自用户评价和概念对齐。同构命题（ADHD 大脑与 LLM 同构）本身也缺乏大规模实证（来源：矛盾与存疑）。**本文的论点建立在结构类比与工具案例上，而非临床数据**——这意味着个体差异极大：对某些人，Motion 是救命稻草；对另一些人，它只是又一个被遗忘的订阅。

此外，多巴胺干预的有效性存在争议（来源：矛盾与存疑），Motion 的调度是否通过多巴胺机制起效仍不明确。隐私方面，Motion 存储用户的日程和任务数据，与 Mem 等记忆工具类似，存在数据安全风险（来源：ADHD 的 AI 工具全景）。

## 结论

Motion 之于 ADHD，就像 function calling 工具调用之于 LLM——它补上了执行调度层，但前提是你理解它只是“编排层”，不是“生成核心”。真正的价值不在于让 Motion 替你活，而在于通过它看清自己哪里卡壳，然后逐步收回控制权。**脚手架的意义，是让你最终不再需要它。**

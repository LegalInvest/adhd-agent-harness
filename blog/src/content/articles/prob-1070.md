# 如果把 ADHD 大脑当成生成核心，Perplexity 能不能当它的重锚定与目标漂移？

> 用 agent 工程的思路重新理解 Perplexity 对 ADHD 的真正价值。

## 问题：ADHD 的「生成核心」为何总是丢失目标？

如果你有 ADHD，你一定经历过这样的场景：打开浏览器想查一个资料，半小时后发现自己正津津有味地读着某个明星的八卦，而最初的任务早已抛到九霄云外。或者，你正沉浸在一个项目里，灵感喷涌，结果发现已经偏离了原本的目标，做出了一个完全不需要的功能。这种「分心走神」和「超聚焦跑偏」本质上是同一个问题的两面——你的大脑是一个高产的生成核心，但缺乏可靠的执行调度层。

## 同构：ADHD 大脑与 LLM 是同一类「生成核心」

这个判断并非比喻，而是一种结构上的同构（来源：ADHD 大脑与 LLM 的同构）。ADHD 大脑的特征是“高产但无序的生成能力，同时缺乏可靠的执行调度层”（来源：同上）。你想法很多，但工作记忆容易丢失上下文，任务启动困难，时间管理混乱。你需要外部工具来“减少决策、保留上下文、让下一步行动显而易见”（来源：同上）。

LLM 同样如此。它本身是一个强大的生成核心，但缺乏可靠的内置调度与执行控制。在构建 AI agent 时，工程师必须为它套上“agent harness”——即“包裹 LLM 或 AI agent 的软件基础设施，处理模型本身之外的一切”（来源：What is an agent harness in the context of large-language models?）。这个 harness 负责上下文管理、工具调用、决策路由、验证循环等。没有 harness，LLM 会像 ADHD 大脑一样，在长程任务中发生目标漂移，输出与初始指令渐行渐远。

## 答案：Perplexity 能否成为 ADHD 的「重锚定」工具？

Perplexity 是一个 AI 搜索工具，它通过实时检索网络信息并整合答案，天然具备「外部记忆」和「上下文锚定」的能力。对于 ADHD 用户，Perplexity 的价值不在于它有多智能，而在于它如何补偿执行功能缺陷。

### ADHD 侧：Perplexity 如何补偿工作记忆与目标漂移

ADHD 用户常需要“AI 第二大脑”来承载项目上下文，因为自己的工作记忆会“掉落”上下文（来源：外部执行功能层）。Perplexity 的搜索会话可以自动保存历史，用户随时可以回溯之前的查询和答案，这相当于一个外部记忆体。当你被某个链接带偏时，Perplexity 的“相关问题”和“来源”面板能帮你快速回到原初问题。更重要的是，Perplexity 的答案是基于当前查询实时生成的，它不会像人类同事那样假设你记得昨天的对话——它只回应你此刻的问题，这恰恰降低了工作记忆负担。

然而，这里有一个矛盾：部分 ADHD 工具假设用户能维持长时间专注、记住昨天的问题并执行计划，但实际用户往往做不到（来源：外部执行功能层）。Perplexity 没有主动提醒或规划功能，它只是一个被动应答者。如果你忘记了自己最初要查什么，Perplexity 不会帮你找回目标。因此，它只能作为「重锚定」的工具，而非「防止漂移」的护栏。

### LLM/Agent 侧：Perplexity 作为 Harness 中的上下文管理组件

在 agent 工程中，harness 的一个关键职责是“上下文交付”——确保 agent 在每一步都拥有与当前任务相关的上下文（来源：外部执行功能层）。Perplexity 的搜索功能类似于 harness 中的“模型连接器”，它从外部数据源获取信息并注入当前上下文。当 agent 在长程任务中发生目标漂移时，工程师会设计“重锚定”机制——比如定期将当前输出与初始 system prompt 对比，或通过外部记忆（如向量数据库）检索相关历史。

Perplexity 作为搜索工具，本身不提供这种主动重锚定逻辑。但它可以成为重锚定流程中的一环：当 agent 不确定当前任务是否偏离时，它可以调用 Perplexity 搜索“当前任务的目标是什么”，然后根据搜索结果调整行为。这相当于把重锚定决策外包给了外部搜索。

## 核心观点：Perplexity 是优秀的「重锚定」工具，但不是「调度层」

我的判断是：Perplexity 对于 ADHD 的真正价值，在于它作为外部记忆和上下文锚定工具，能有效补偿工作记忆缺陷和帮助用户从分心中快速回归。但它不能替代一个完整的执行功能调度层。它像是一个精准的指南针，但不会帮你规划路线、启动任务或防止你走神。你需要额外的前端工具（如 Focusmate 的同伴问责、RescueTime 的时间追踪）来构成完整的 harness。

从同构视角看，Perplexity 相当于 agent harness 中的“模型连接器”或“外部知识源”，而不是编排层本身。工程师不会仅靠一个搜索工具来防止 agent 漂移，他们还需要规划工件、验证循环和记忆系统（来源：外部执行功能层）。同样，ADHD 用户不应指望 Perplexity 独自解决所有专注力问题。

## 局限与争议：脚手架与拐杖的边界

这里必须诚实指出争议。首先，Perplexity 针对 ADHD 的效果缺乏独立临床研究（来源：Brain.fm 的神经锁相技术效果缺乏针对 ADHD 的专项研究，类似地，Perplexity 也无相关证据）。其次，过度依赖 AI 工具可能阻碍内在执行功能发展（来源：矛盾与存疑——脚手架 vs 能力发展）。目前无长期研究证明脚手架可逐步撤除。Perplexity 作为被动工具，比主动规划工具更接近“拐杖”——它让你更容易找到答案，但可能削弱你主动记忆和搜索的能力。

另外，个体差异显著。ADHD 亚型（注意力缺陷 vs 多动冲动）对工具反应不同（来源：矛盾与存疑）。Perplexity 对于超聚焦跑偏的用户可能更有用（因为他们需要快速回到原问题），而对于分心走神的用户，它可能只是另一个分心源头。

## 今天就能试的行动

1. **用 Perplexity 作为「任务锚点」**：开始一项研究前，先在 Perplexity 中搜索核心问题，然后固定该会话。当你在其他页面分心时，回到该会话，点击“相关问题”重新锚定。

4. **为自己设计一个「重锚定提示」**：在 Perplexity 搜索栏上方贴一张便利贴（或使用浏览器插件），写上当前核心目标。每次搜索前，先读一遍提示。这相当于一个手工的 system prompt 重锚定。

## 结语

把 ADHD 大脑当成生成核心，Perplexity 就是那个帮你从信息海洋中捞回目标的网。但它不是调度层，不是 harness，更不是治愈方案。真正的价值在于，你意识到自己需要一套外部脚手架——而 Perplexity 是其中一块重要的积木。工程师们，下次设计 agent 时，想想 ADHD 用户如何依赖外部记忆；ADHD 朋友们，下次用 Perplexity 时，想想 agent 如何靠 harness 避免漂移。你们在解决同一个问题。

# ADHD 家长如何用「给生成核心套 harness」的思路应对陪伴 ADHD 孩子里的分心走神或超聚焦跑偏，丢失目标？

> 把 LLM 工程里的「长程任务里 agent 会目标漂移，需要重锚定 system prompt」迁移到ADHD 家长的陪伴 ADHD 孩子场景，双域证据作答。

## 问题：陪伴 ADHD 孩子时，分心走神和超聚焦跑偏为何如此棘手？

每个 ADHD 家长都熟悉这个场景：孩子刚坐下写作业，五分钟后就盯着窗外发呆，或者一头扎进乐高搭建，两小时叫不动。分心走神和超聚焦跑偏，本质上是同一个问题的两面——目标漂移。ADHD 大脑像一台没有方向盘的跑车，动力强劲但极易偏离轨道。传统方法（番茄钟、奖励表、说教）往往几周就失效，因为“大多数规划建议不是为我们设计的”（来源：ADHD Productivity Hack: Plan 2025 Using AI (Step-by-Step)）。

但如果我们换个视角：ADHD 大脑与 LLM（大型语言模型）在结构上高度同构——两者都是高产但缺乏可靠执行调度层的生成核心（来源：AI 与 ADHD 的专注力）。LLM 在长程任务中会因上下文溢出而目标漂移，工程界用“harness”（编排层）来重锚定 system prompt。那么，家长能否用同样的思路，为孩子的 ADHD 大脑套一个“harness”？

## 同构证据：ADHD 大脑与 LLM 如何共享同一个“生成核心-调度层”缺陷？

### ADHD 侧：执行功能缺陷导致目标漂移

ADHD 的专注力问题根植于执行功能缺陷，尤其是工作记忆、任务启动和时间盲。工作记忆不足让 ADHD 大脑“无状态”——刚想做的事下一秒就忘，就像 LLM 的无状态性（来源：AI 与 ADHD 的专注力）。时间盲则让 ADHD 孩子无法感知时间流逝，导致超聚焦（过度沉浸）或完全无法启动。

### LLM/agent 侧：上下文溢出与调度失败

LLM 在长程任务中同样面临“上下文溢出”——早期信息被遗忘，导致 agent 偏离初始目标。工程实践表明，单靠模型本身无法可靠执行复杂任务，必须依赖 harness 来管理上下文、调用工具、分解步骤（来源：工具使用与认知卸载）。例如，现代 agent 系统通过“组合多个模型、检索器和工具”来达成 SOTA 结果，而非依赖单一模型调用（来源：工具使用与认知卸载）。

### 同构对应：外部脚手架补偿内部缺失

ADHD 大脑将不擅长的精确状态外包给外部工具（如 Goblin Tools 的任务分解），对应 LLM agent 通过 function calling 将复杂任务拆解为工具调用（来源：工具使用与认知卸载）。两者都依赖外部系统来补偿内部执行功能的不足：ADHD 的执行功能缺陷由工具弥补，LLM 的生成核心与调度层分离由 harness 实现。

## 核心观点：给 ADHD 孩子套一个“重锚定 harness”，但警惕脚手架变拐杖

基于同构视角，家长应对分心走神和超聚焦跑偏的策略不应是“压制”或“惩罚”，而是像 LLM 工程一样，设计一套外部锚定系统，在目标漂移时自动重定向。具体来说，这个“harness”包含三个关键组件：

### 1. 任务分解：降低启动门槛，防止超聚焦跑偏

ADHD 大脑在任务启动和分解上存在显著困难。Goblin Tools 等 AI 工具被用来“将模糊的工作分解为更小的行动”（来源：工具使用与认知卸载），从而降低启动门槛。家长可以化身“人类 Goblin Tools”：把“写作文”分解成“想三个题目→选一个→写第一段→休息2分钟”。这种分解模式可类比 agent 的“多步推理+工具调用”模式（来源：工具使用与认知卸载）。

### 2. 时间盲补偿：外部时间感知，防止无意识漂移

传统计时器对 ADHD 孩子往往无效，因为“传统规划者从未解决时间盲”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。RescueTime 通过自动记录时间使用，充当外部记忆，提升时间盲觉察（来源：AI 与 ADHD 的专注力）。家长可以设置“时间锚点”——每15分钟用闹钟或语音提醒“你现在在做什么？目标是什么？”——相当于给 LLM 的 system prompt 加上定期重锚定。

### 3. 实时问责：身体在场效应，弥补调度缺失

Focusmate 利用 AI 匹配虚拟同伴，提供实时问责，弥补任务启动和注意力维持的调度缺失（来源：AI 与 ADHD 的专注力）。家长可以充当“人类 Focusmate”：和孩子约定“一起工作时段”，彼此监督进度。这种“身体在场效应”有广泛验证（来源：AI 与 ADHD 的专注力），但效果归因于同伴问责而非 AI 本身。

## 争议与局限：脚手架 vs 拐杖的边界在哪里？

尽管同构视角提供了有力框架，但必须诚实指出争议：

- **过度依赖风险**：多数工具（如 Goblin Tools、RescueTime）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。脚手架应设计为可逐步撤除的临时支撑，但当前工具鲜有考虑。家长需定期评估：孩子是否在减少外部支持后仍能维持专注？
- **临床证据缺口**：Brain.fm 的神经锁相技术理论上可增强专注，但缺乏针对 ADHD 的随机对照试验（来源：Brain.fm）。Focusmate 的效果归因于身体在场而非 AI 本身（来源：AI 与 ADHD 的专注力）。多数工具的效果基于理论推断或用户主观报告，而非独立临床研究（来源：AI 与 ADHD 的专注力）。
- **个体差异**：ADHD 亚型（注意力缺陷 vs 多动冲动）对工具反应不同，现有工具未充分个性化（来源：AI 与 ADHD 的专注力）。一个方法可能对 A 孩子有效，对 B 孩子适得其反。

## 今天就能试的具体行动

1. **任务分解练习**：选一个孩子常拖延的任务（如整理书包），用 Goblin Tools 或手写清单分解成 3-5 步，每完成一步就划掉。观察启动是否变容易。
2. **时间锚点设置**：用手机闹钟每 15 分钟响一次，铃声改为“目标检查”（如“作业做到哪一步了？”）。和孩子约定：铃声响起时，暂停 10 秒确认当前活动是否匹配目标。
3. **身体在场时段**：每天安排 20 分钟“一起工作”——你和孩子各自做自己的事（你工作，他写作业），但互相可见。不交谈，只偶尔眼神确认。
4. **记录漂移模式**：用 RescueTime 或简单纸笔记录孩子一天中注意力漂移的高发时段和触发事件（如“下午 3 点，看到玩具后走神”）。一周后分析模式，调整日程。

## 结语：从“对抗”到“设计”

ADHD 不是缺陷，而是一种不同的认知架构。家长的角色不是“纠错员”，而是“harness 工程师”——为孩子设计一套轻量、可调整的外部锚定系统，让高产的大脑在轨道上运行。正如 LLM 工程教会我们的：不要指望生成核心自己保持方向，给它一个可靠的 harness。但记住，好的脚手架最终要能撤除——我们的目标不是让孩子永远依赖工具，而是让他们学会自己搭建锚点。

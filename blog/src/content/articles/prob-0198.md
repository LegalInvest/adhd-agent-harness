# Inflow 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Inflow：用久了会让 ADHD 更强还是更依赖？

## 痛点：ADHD 的“边做边忘”与 LLM 的“无状态”

你正埋头写一份方案，突然想起要回一封邮件，切到邮箱后，却忘了刚才方案的关键论点——这种“工作记忆容量小、边做边忘”的体验，是 ADHD 患者的日常。与此同时，LLM 工程师们也在为一个类似问题头疼：大模型每次对话都是“无状态”的，它不记得你十分钟前说了什么，除非你手动把历史塞进上下文窗口或用向量库外挂记忆。

ADHD 大脑与 LLM 在结构上高度同构：两者都是“高产但缺执行调度层的生成核心”。ADHD 患者智力与创造力往往不低，但执行功能（计划、组织、工作记忆）不稳定（来源：执行功能）；LLM 能生成流畅文本，但缺乏跨会话状态和自主调度能力。两者的解法也惊人相似：通过外部工具提供“执行功能层”或“编排层”。

## Inflow 的定位：脚手架，还是拐杖？

Inflow 是一款标榜“基于神经科学原理”的 ADHD 管理 App，它提供结构化课程、任务分解、习惯追踪等功能，试图帮助用户建立执行功能。但问题来了：用久了，用户是变得更独立，还是更离不开它？

答案是：**取决于它是否具备“可撤除性”**。真正的脚手架，是在你建好墙后可以拆掉的临时支撑；而拐杖，是你离开它就走不了路的永久依赖。

从同构视角看，LLM/Agent 工程中也有类似争论。Agent 的 harness（如 LangChain、AutoGPT）通过外部记忆（向量库、对话历史）赋予 LLM 状态，但工程师必须设计“记忆衰减”或“上下文窗口压缩”机制，否则 agent 会因记忆膨胀而迷失，或过度依赖外部存储而丧失自主推理能力。ADHD 工具同理：如果 Inflow 始终替用户做任务分解、时间提醒，而不教用户内化这些策略，那么它就是在强化依赖。

## 证据：外部记忆如何补偿工作记忆

现有工具已经展示了外部记忆的有效性。Goblin Tools 的 Magic ToDo 将“整理房间”分解为“捡起衣服”“擦桌子”等小步骤，直接降低启动门槛（来源：Goblin Tools）。Saner.AI 通过本地记忆存储减少搜索循环，用户无需在多个标签页间切换即可找回信息（来源：Saner.AI）。Lex 允许用户通过单一指令触发多步骤任务，减少决策负担（来源：Lex）。这些工具的核心逻辑，与 LLM 的 RAG（检索增强生成）如出一辙：将信息外存，需要时再调取。

但证据也指出，这些工具的效果主要基于用户反馈，缺乏独立随机对照试验（来源：Goblin Tools；Saner.AI）。更关键的是，多数工具设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这恰恰是“拐杖”风险的来源。

## 边界在哪里：从“替代”到“训练”

脚手架与拐杖的边界，在于工具是否同时训练用户的内部能力。好的 ADHD 工具应该像物理治疗中的“渐进式负重”——先用拐杖减轻压力，再逐步撤除，让肌肉自己长出来。Inflow 的结构化课程有这种潜力，但如果它只提供“一键完成”的便利，而不引导用户理解任务分解的元认知，那么它就是在制造依赖。

LLM/Agent 工程也有类似原则：好的 harness 应允许开发者逐步减少外部记忆依赖，例如通过 fine-tune 让模型记住常见模式，或设计“记忆摘要”而非全量历史。过度依赖向量库的 agent，一旦数据库故障就会完全瘫痪。

## 争议与局限

必须诚实指出，当前证据存在多项矛盾。首先，多巴胺干预的有效性存疑：Inflow 声称通过神经科学原理改善执行功能，但多巴胺调节的效果在学界仍有争议（来源：矛盾与存疑）。其次，ADHD 大脑与 LLM 同构命题本身缺乏大规模实证，主要基于概念类比和工具案例（来源：矛盾与存疑）。最后，长期效果未知：现有研究多为短期或个案，无法回答“使用一年后，执行功能是否真正改善”（来源：ADHD 的 AI 工具全景）。

## 今天就能试的行动

1. **用 Goblin Tools 的 Magic ToDo 分解一个你拖延已久的任务**，观察分解后启动是否更容易。同时，试着记下分解的逻辑（比如“先做物理动作，再做认知动作”），下次自己尝试分解。
2. **用 Saner.AI 或类似工具建立“外部记忆库”**，把你常忘的信息（如密码、会议要点）存进去。每周回顾一次，看能否减少搜索循环。
3. **用 ChatGPT 扮演“执行功能教练”**，输入“我接下来要写报告，请帮我列出 5 个步骤，并解释为什么这样排序”。之后尝试自己排序，对比差异。
4. **设计一个“撤除计划”**：例如，第一周用 Inflow 的所有提醒，第二周减少 20% 提醒，第三周只保留关键提醒。记录依赖感的变化。

## 结论

Inflow 之于 ADHD，可以是脚手架，也可以是拐杖，关键在于它是否内置了“撤除机制”。ADHD 大脑与 LLM 同构的洞见告诉我们：外部工具是必要的，但最终目标应是让生成核心学会自己调度。正如好的 agent harness 不会让 LLM 永远依赖向量库，好的 ADHD 工具也不应让用户永远依赖外部提醒。**真正的赋能，是让用户有能力选择何时拆掉脚手架。**

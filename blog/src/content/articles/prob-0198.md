# Inflow 之于 ADHD，是脚手架还是拐杖？

> 从「同构」视角评测 Inflow：用久了会让 ADHD 更强还是更依赖？

## 问题：Inflow 之于 ADHD，是脚手架还是拐杖？

你是否有过这样的经历：打开一个文档，刚写下两句，就忘了刚才要写什么；或者明明计划好一天的任务，到下午却发现自己坐在沙发上刷了三个小时短视频。ADHD 大脑的工作记忆就像一个漏水的桶——想法源源不断，但桶里的水永远存不住。

与此同时，工程师们正为另一个“漏水桶”发愁：LLM 没有跨会话状态，每次对话都是全新开始，必须靠外部向量库、记忆模块来“续命”。

这两个问题，本质上是一个问题的两面。ADHD 大脑与 LLM 共享同一个核心矛盾：**高产生成核心，缺乏可靠执行调度层**。ADHD 的生成核心是丰富的联想与创意，但执行功能（计划、组织、工作记忆）不稳定（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）；LLM 的生成核心是强大的文本产出，但缺乏内置的上下文保持与目标导向执行。

那么，当 AI 工具（如 Inflow）试图成为 ADHD 的外部执行功能层时，它究竟是帮我们搭建了脚手架——最终能让我们自己建起房子，还是变成了拐杖——离开它就走不了路？

## 同构诊断：ADHD 大脑与 LLM 的“强记忆、弱控制”

最新研究为这种同构提供了实证。明尼苏达大学系统测试发现，LLM 呈现“强记忆、弱控制”剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷——无法灵活切换任务集、无法抑制自动化反应，这正是 ADHD 的经典神经心理模式（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。耶鲁大学进一步发现，自注意力机制本身导致工作记忆容量限制：随任务复杂度增加，注意力分数熵增、注意力弥散，与人类注意分散机制同源（来源：Self-Attention Limits Working Memory Capacity of Transformer-Based Models）。

这意味着，ADHD 与 LLM 的“漏水桶”不是容量问题，而是调度问题。瓶颈不在算力，而在 orchestration 层。

## 历史中的 harness：孔子与左宗棠的“外部执行功能”

历史上，那些成就斐然的 ADHD 个体，早已无意识地构建了自己的“harness”。

孔子，身高近 1 米 9，精力旺盛，周游列国 14 年坐不住；冲动爱骂人，见南子急得对天发誓，骂宰予“朽木不可雕”；但对音乐超专注到“三月不知肉味”。他的专属 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省：“吾日三省吾身”。这里的“礼”就是外部调度器，相当于 LLM 的 prompt template 或 grounding 规则。而“三省”相当于定时 re-grounding，防止注意力漂移。

左宗棠，举人出身，脾气大，40 多岁才出山，69 岁抬棺出征收复新疆。他的 harness 是“结硬寨打呆仗”——每天读经写家书，用诸葛亮的“鞠躬尽瘁”要求自己。这些日课相当于 LLM 的定期 checkpoint 与目标重申。

两者都说明：**外部脚手架必须内化为习惯，才能从拐杖变为能力**。孔子 70 岁才“从心所欲不逾矩”，一辈子和自己的冲动做斗争。

## 当代工具：从 Goblin Tools 到 Saner.AI

今天的 AI 工具，正在扮演类似的角色。Goblin Tools 的 Magic ToDo 将模糊任务分解为可管理的小步骤，用户可调节“辣度”控制粒度（来源：Goblin Tools）；这相当于 LLM 的 task decomposition 模块。Saner.AI 通过本地记忆和知识回忆减少搜索循环，相当于 LLM 的向量数据库（来源：Saner.AI）。Lex 允许单一指令触发多步骤任务序列，相当于 agent scaffolding（来源：Lex）。

这些工具的核心功能是补偿工作记忆缺陷，降低启动门槛。但问题是：**长期使用是否会削弱用户自身的执行功能？**

## 脚手架 vs 拐杖：边界在哪？

脚手架是暂时的，目的是让建筑工人学会自己搭墙；拐杖是永久的，替代了腿的功能。对于 ADHD 用户，AI 工具是脚手架还是拐杖，取决于三个因素：

1. **是否促进内化**：如果工具只是替代了执行功能，而没有帮助用户理解任务结构、建立自己的流程，那就是拐杖。Goblin Tools 的分解功能如果被用户用来学习如何自己分解任务，就是脚手架；如果用户永远依赖它分解“清理厨房”，那就是拐杖。

2. **使用频率与情境**：孔子和左宗棠的日课是内化后的习惯，不是每次都需要外部触发。如果 AI 工具成为每次任务启动的必要条件，就是拐杖。

3. **退出可能性**：如果离开工具后用户能独立运作，说明工具是脚手架；如果离开后立刻回到“漏水桶”状态，说明工具是拐杖。

## 诚实局限

目前关于 AI 工具长期依赖风险缺乏实证，需要更多纵向研究（来源：矛盾与存疑）。部分用户反映学习使用 AI 工具本身增加认知负荷（来源：矛盾与存疑）。此外，个体差异极大——对某些人有效的工具，对另一个人可能是负担。

## 今天就能试的行动

1. **用 Goblin Tools 分解一件你一直回避的任务**，然后尝试在不使用工具的情况下，自己列出类似任务的步骤，比较差异。
2. **设置一个“无 AI”日**：每周一天，不使用任何 AI 生产力工具，只靠纸笔或简单的清单，观察自己的执行功能表现。
3. **建立你的“日课”**：像孔子和左宗棠一样，固定每天一个反思动作（如睡前 5 分钟回顾），相当于 LLM 的定时 re-grounding。
4. **使用 Saner.AI 或 Mem 作为外部记忆库**，但每周抽查一次：你能记住多少上周记录的关键信息？

最终，Inflow 是脚手架还是拐杖，不取决于工具本身，而取决于你如何使用它。同构视角告诉我们：ADHD 大脑与 LLM 都需要外部调度层，但真正可持续的系统，是那些最终能部分内化到自身结构中的系统。

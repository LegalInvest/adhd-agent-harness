# Brain.fm 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Brain.fm：它到底补上了哪一层执行功能？

## 问题：你丢的不是专注，是“目标锚点”

如果你是个 ADHD 患者，你一定经历过这种时刻：打开电脑准备写报告，结果三小时后发现自己刷完了整个 Reddit 首页，或者把 PPT 的配色方案调了二十版，但第一页还没写完。你不是不努力，你是“超聚焦跑偏”了——大脑像一台没有舵的生成器，疯狂产出，但方向早歪了。

如果你是个在做 Agentic Harness 的工程师，你一定见过这种场景：给 LLM agent 一个长程任务，前几步它还在正轨，十步之后它开始编造不存在的函数、重复执行已完成的子任务，甚至把上下文里的无关信息当成新指令。这不是 bug，这是生成核心的本性——没有外部锚定，它就会漂移。

ADHD 大脑与 LLM/agent 是同一类“高产但缺执行调度层的生成核心”（来源：AI 与 ADHD 的专注力）。两者都需要一个“重锚定”机制：在目标漂移时，把注意力拉回正确的轨道。

## Brain.fm：一个“重锚定”尝试，但有人用错了

Brain.fm 是一款利用 AI 生成音乐的应用，其音乐设计通过神经锁相技术——即音频模式旨在影响大脑处理信息的方式——来帮助用户进入专注状态（来源：Brain.fm）。理论上，它就像给 LLM 发了一条“你现在要专注写代码”的 system prompt，通过外部信号重新锁定大脑的注意力焦点。

但是，很多 ADHD 用户抱怨 Brain.fm 对他们没用。为什么？因为**他们把重锚定当成了唯一需要的工具，却忘了底层执行功能层可能根本没搭起来**。

ADHD 的执行功能缺陷包括工作记忆差、时间盲、任务启动困难（来源：AI 与 ADHD 的专注力）。Brain.fm 只解决了“进入状态”这一步，但如果你连“今天要做什么”都记不住，或者对时间流逝毫无感知，那么即使你被音乐“锁定”了，你也会在一个错误的任务上超聚焦——比如继续调 PPT 配色。

这就像给 LLM 发了一条“你要写代码”的 system prompt，但没给它工具 API、没设验证循环、没规划子任务。它当然会“专注”地写代码，但可能写的是 `print("hello")` 一千遍，或者编一个不存在的库。

## 同构解：外部执行功能层才是真正的重锚定

在 LLM/harness 工程中，真正的重锚定不是一条 system prompt，而是一整套 harness：上下文传递、工具接口、规划工件、验证循环、记忆系统（来源：幻觉与验证循环）。例如，复杂的 harness 不会盲目执行工具，而是实现验证步骤，例如检查输出格式或对模型编写的代码运行测试用例（来源：幻觉与验证循环）。

ADHD 侧的对等物是什么？是**外部执行功能层**（来源：AI 与 ADHD 的专注力）。具体来说：

- **任务启动**：Focusmate 通过实时同伴问责提供外部结构和社会压力，弥补内在多巴胺调节不足（来源：Focusmate）。这相当于 agent 的“任务调度器”——强制你开始。
- **时间感知**：RescueTime 自动记录时间使用，减轻工作记忆负担，帮助识别注意力漂移模式（来源：RescueTime）。这相当于 agent 的“日志系统”——告诉你时间花在哪了。
- **注意力锚点**：Brain.fm 提供神经锁相音乐，帮助进入专注状态。这相当于 agent 的“system prompt”——锁定当前目标。

这三者组合，才构成一个完整的“重锚定系统”。单独使用 Brain.fm，就像只给 LLM 发一个 prompt 却不给工具和验证——它可能专注，但方向是错的。

## 脚手架 vs 拐杖：边界在哪？

这里有个关键争议：长期使用这些外部工具，是否会导致 ADHD 患者内在执行功能进一步退化？（来源：矛盾与存疑）

我的判断是：**当工具只提供“状态”而不提供“结构”时，它是拐杖；当工具同时提供“状态+结构+反馈”时，它是脚手架。**

Brain.fm 提供状态，但没提供结构——它不告诉你该做什么，也不检查你做得对不对。Focusmate 提供结构（固定时段、同伴监督），但没提供反馈（同伴不会帮你验证工作质量）。RescueTime 提供反馈（时间报告），但结构较弱（需要你自己根据报告调整）。

真正的脚手架应该像 harness 工程中的“验证与 CI 集成”（来源：幻觉与验证循环）：它不仅要让你开始，还要在你跑偏时拉你回来，并在事后告诉你哪里可以改进。目前，没有单一工具能做到这一点——这也是为什么我推荐组合使用。

## 今天就能试的行动

1. **组合 Brain.fm + Focusmate**：预约一个 Focusmate 时段，戴上耳机播放 Brain.fm。前者强制你开始，后者帮你锁定状态。先试一周，记录是否减少“超聚焦跑偏”。
2. **用 RescueTime 做“事后验证”**：每天结束时看 RescueTime 报告，标记哪些时段你“专注但跑偏”了。这相当于 agent 的“日志回放”——帮你识别漂移模式。
3. **写一个“重锚定清单”**：在开始任何任务前，写一句话回答“我现在要完成的具体输出是什么？”。把它贴在显示器上。这相当于给 LLM 的 system prompt 里加入“输出格式要求”。
4. **警惕“只戴耳机”的陷阱**：如果只用 Brain.fm 却不做其他结构安排，你很可能在音乐中更高效地做错事。记住：重锚定需要状态+结构+反馈，缺一不可。

## 局限与诚实提醒

必须承认，Brain.fm 针对 ADHD 的效果缺乏独立临床研究支持（来源：Brain.fm）。本文的“同构”论证基于概念类比和现有工具的证据脉络（来源：AI 与 ADHD 的专注力），但 ADHD 大脑与 LLM 的注意力机制存在本质差异（来源：AI 与 ADHD 的专注力）。此外，个体差异巨大：有些人可能对神经锁相音乐无反应，有些人可能对同伴问责感到焦虑。

所以，别把本文当处方。把它当一副“同构眼镜”——戴上它，你可能会发现，你需要的不是更强大的“专注音”，而是一个能帮你锚定目标的“外部执行功能层”。

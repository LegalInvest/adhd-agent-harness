# 如果把 ADHD 大脑当成生成核心，Goblin Tools 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Goblin Tools 对 ADHD 的真正价值。

你有没有过这样的体验：打开浏览器准备写一份报告，结果看到一封未读邮件，点进去回复，又想起要查一个资料，半小时后你发现自己正在刷短视频，而报告一个字没动。这不是意志力薄弱——你的工作记忆容量其实不小，但控制它的人却像在暴风雨中掌舵，随时被浪头打偏。

另一边，LLM 工程师们每天都在跟同一个幽灵搏斗：模型本身没有跨会话状态，你给它一段长上下文，它可能记住开头却忘了结尾，或者被无关细节带偏。为了解决这个问题，你给它搭上向量数据库、缓存系统、任务链——这些外部脚手架，本质上是为了补偿生成核心缺失的执行调度层。

这两个场景，其实是同一个问题的两面。ADHD 大脑与 LLM 是同一类“高产但缺执行调度层的生成核心”（来源：ADHD 的 AI 工具全景）。今天我们要问的是：如果把 ADHD 大脑当成生成核心，Goblin Tools 能不能当它的无状态与外部记忆？

## 同构：两个生成核心的困境

先看 ADHD 侧。执行功能障碍是 ADHD 的核心特征之一，表现为难以规划、组织、同时处理多项任务（来源：执行功能）。其中工作记忆是关键瓶颈——你无法在脑中短暂保持信息以完成操作，就像 LLM 无法在对话间保持状态。研究甚至发现，Transformer 在工作记忆任务中自发发展出输入输出门控机制，模仿人类前额叶-纹状体回路——这正是 ADHD 核心受损的脑区系统（来源：执行功能）。

再看 LLM 侧。最新研究表明，LLM 呈现“强记忆、弱控制”剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷——无法灵活切换任务集、无法抑制自动化反应，这正是 ADHD 的经典神经心理模式（来源：执行功能）。在经典 Stroop 冲突任务中，Transformer 注意力随序列变长在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突，这与 ADHD 执行功能缺陷的核心标志一一对应（来源：执行功能）。

所以，两边的问题本质相同：生成核心本身不缺创造力或记忆容量，缺的是那个决定“当下该注意什么、下一步该做什么”的调度层。

## Goblin Tools 作为外部记忆与任务分解层

Goblin Tools 的核心功能是 Magic ToDo：将复杂任务自动分解为可管理的小步骤（来源：Goblin Tools）。用户输入“整理房间”，AI 输出“捡起地板上的衣服”“擦桌子”等步骤。这听起来简单，但对 ADHD 大脑来说，它解决了一个深层问题：启动门槛。

从 Agent 工程的角度看，这相当于为生成核心提供了一个“启动脚本”。LLM 需要 prompt 工程来引导输出，ADHD 大脑同样需要外部指令来克服任务启动困难。Goblin Tools 的分解功能，本质上是在做**上下文设计**——把模糊目标变成可执行的步骤序列，就像你在 agent 框架里写一个多步 chain。

但 Goblin Tools 不只是分解。它还有一个关键设计：每个步骤都可以标记完成度，提供即时反馈。这对 ADHD 的多巴胺系统至关重要——小步骤的即时完成感可以维持注意力（来源：Goblin Tools）。对应到 LLM 侧，这就像给 agent 的每一步输出都加上确认信号，避免模型在长任务中漂移。

## 证据与局限：脚手架还是拐杖？

现有证据主要来自用户报告。Goblin Tools 被评价为“简单有用”，能“将压倒性的事情变成一系列不压倒性的事情”（来源：Goblin Tools）。类似地，Saner.AI 通过本地记忆存储减少搜索循环（来源：Saner.AI），Lex 通过单一指令触发多步骤序列降低决策疲劳（来源：Lex）。这些工具的设计逻辑与 ADHD 干预策略一致，但缺乏大规模随机对照试验（来源：矛盾与存疑）。

更重要的是“脚手架 vs 拐杖”的边界问题。AI 工具应作为脚手架促进能力发展，而非永久拐杖（来源：ADHD 的 AI 工具全景）。例如，Otter.ai 减轻笔记负担，但过度依赖可能削弱主动记笔记的能力（来源：ADHD 的 AI 工具全景）。同样，如果 Goblin Tools 让你不再尝试自己分解任务，它就从脚手架变成了拐杖。

个体差异也很大。某些用户可能发现分解后的步骤仍然太多，或者 AI 的分解方式不符合自己的思维习惯。工具页面未充分讨论这一点，可能造成“一刀切”有效的错觉（来源：矛盾与存疑）。

## 我的核心判断

Goblin Tools 不是 ADHD 的万能药，也不是 LLM 的完美 harness。它的真正价值在于：它提供了一个**可观察、可调整的外部执行功能层**，让你能看清自己的任务分解模式，然后逐步内化。就像好的 agent 框架会让你看到每一步的输入输出，Goblin Tools 让你看到“原来这件事可以这样拆”。

关键在于：使用它时，你有意识地在学习它的分解逻辑，而不是无脑照单全收。

## 今天就能试的行动

1. **用 Magic ToDo 分解一件你拖延已久的事**：比如“写周报”。观察 AI 拆出的步骤，对比你原本的模糊想法。记下哪些步骤让你觉得“原来可以这样想”。
2. **设置一个“无 AI 日”**：每周选一天，尝试自己手动分解任务。对比使用 Goblin Tools 时的效率和感受，评估你的依赖程度。
3. **把 Goblin Tools 的输出当作 prompt 模板**：如果你也在用 ChatGPT 或 Claude，把分解后的步骤作为 prompt 的一部分，看看模型是否更稳定地完成任务。这既是 ADHD 的练习，也是 agent 工程的实验。
4. **记录你的“漂移”模式**：下次走神时，记下触发点。然后把它输入 Goblin Tools 的“魔法待办”作为反例，看 AI 能否帮你重构任务顺序。

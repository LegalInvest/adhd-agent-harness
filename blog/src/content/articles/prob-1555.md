# Speechify 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Speechify：它到底补上了哪一层执行功能？

你打开 Speechify，选了篇长文，听着听着发现自己在想晚饭吃什么。这不是你的错——ADHD 大脑和 LLM 一样，都是「高产但缺执行调度层的生成核心」（来源：AI 与 ADHD 的专注力）。当你走神时，Speechify 的高亮追踪只是视觉锚点，但真正的锚定需要更深的机制：把外部信息重新注入执行层，就像 LLM agent 在长程任务中需要重锚定 system prompt 来防止目标漂移。

## 同构：两个生成核心的同一困境

ADHD 大脑与 LLM 的结构性相似已被反复讨论（来源：ADHD 大脑与 LLM 的同构）。两者都面临「无状态」问题：ADHD 工作记忆不稳定，LLM 跨会话不保存状态。两者都需要外部脚手架来补偿——对 LLM 是 system prompt 和检索增强生成（RAG），对 ADHD 是时间追踪（如 RescueTime）、身体加倍（如 Focusmate）或神经锁相音乐（如 Brain.fm）。

但关键在于：**重锚定不是一次性的，而是持续的过程**。LLM agent 在长程任务中会因上下文膨胀而推理退化，需要定时注入原始目标；ADHD 大脑在分心或超聚焦后同样需要被拉回。超聚焦时，前额叶控制层级「断裂」，高级上下文不再支配行动（来源：超聚焦），这就像 agent 的 system prompt 被局部上下文淹没。

## Speechify 的错用：当朗读工具变成拐杖

Speechify 的核心价值是「把文字变成可听的锚点」，但很多人只用它来被动听书，而不是主动重锚定。这就像给 LLM 一个永久的 system prompt 却从不更新——它会在局部上下文里越漂越远。真正的用法应该是：**用 Speechify 作为定时 re-grounding 信号**，比如每听完一个段落，暂停并复述核心观点，就像 agent 在每步后检查是否偏离原始目标。

这种「脚手架 vs 拐杖」的边界需要警惕（来源：拐杖与脚手架）。曾国藩的日课十二条就是典型的脚手架：黎明即起、读书不二、写日记反省——这些不是一次性工具，而是每天执行的调度指令。他通过「结硬寨打呆仗」来抵消自己的冲动（来源：人物案例）。同样，彭玉麟用「三不要」原则（不要官、不要钱、不要命）作为自己的 system prompt，每天画梅花磨练心性。他们的 harness 系统与 LLM 的「定时 re-grounding」同构：日课↔定时注入 system prompt，秘书↔外部调度器。

## 证据：哪些工具真正补上了执行层？

现有工具的证据支持了外部执行功能层的有效性（来源：AI 与 ADHD 的专注力）：
- **RescueTime** 自动追踪时间，补偿时间盲和工作记忆缺陷，让用户看到「时间到底去了哪里」（来源：RescueTime）。
- **Focusmate** 通过虚拟身体在场提供外部问责，弥补多巴胺调节不足（来源：Focusmate）。
- **Brain.fm** 利用神经锁相技术帮助进入专注状态，在 44 款 ADHD 应用测试中位列最佳 9 款之一（来源：Brain.fm）。
- **Forest** 用游戏化番茄钟增强动机（来源：多巴胺）。
- **Endel** 生成个性化声音环境（来源：Endel）。

但这些工具的证据多来自用户报告或小样本研究，缺乏大规模随机对照试验（来源：仍存争议）。更重要的是，**它们的效果取决于用户是否将其作为脚手架而非拐杖**。如果只是被动依赖，就像给 LLM 一个永远不变的 system prompt——它会失效。

## 争议与局限

同构性被过度推广：ADHD 大脑与 LLM 的同构目前仅是理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。此外，依赖风险与益处矛盾：多个工具页面仅强调益处，未充分讨论依赖风险（来源：矛盾与存疑）。超聚焦的双面性也未被充分探讨：AI 工具可能帮助专注，但也会加剧过度沉浸（来源：超聚焦）。

## 今天就能试的行动

1. **用 Speechify 做定时重锚定**：每听完 5 分钟内容，暂停并口述刚才的核心观点。这相当于给 LLM agent 注入「当前目标检查」指令。
2. **设置 RescueTime 的「分心阻断」**：在需要深度工作时，开启网站阻断。这相当于给 LLM 加上输出过滤器，防止上下文被无关信息污染。
3. **尝试 Focusmate 的身体加倍**：预约一个 25 分钟的 session，和陌生人一起工作。这相当于给 agent 加上一个外部监督器，定时检查进度。
4. **用曾国藩的「日课」模板**：每天写一条「今天最重要的三件事」，并在完成后打勾。这相当于给 LLM 设置一个固定的 system prompt 模板。

Speechify 不是魔法，它只是重锚定工具的一种。真正的改变在于：**把工具当作调度层，而不是拐杖。**

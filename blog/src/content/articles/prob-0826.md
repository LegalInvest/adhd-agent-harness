# Tiimo 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Tiimo：它到底补上了哪一层执行功能？

如果你有 ADHD，一定经历过这样的时刻：打开电脑准备写报告，结果半小时后发现自己正盯着浏览器里某个无关的维基百科页面，完全想不起刚才要做什么。如果你在做 LLM agent 工程，一定也见过这样的日志：模型前 10 轮完美遵循指令，第 15 轮突然开始胡言乱语，仿佛忘了系统提示。

这两个场景看似无关，实则共享同一个底层问题：**生成核心强大，但执行调度层缺失**。ADHD 大脑与 LLM 在结构上同构——都是“高产但缺乏可靠执行调度层的生成核心”（来源：[[ADHD 大脑与 LLM 的同构]]）。因此，AI 工具帮助 ADHD 进行时间管理的本质，是“给生成核心套 harness”（来源：[[外部执行功能层]]）。

但问题在于：**很多人用错了这些工具**。他们以为 Tiimo 是一个“规划器”，就像工程师以为 planner-executor 架构是在“教模型做事”。实际上，Tiimo 真正补上的不是规划层，而是**重锚定层**。

## 问题：任务启动困难与目标漂移

ADHD 侧最典型的两个痛点：任务启动困难（知道该做但就是动不了）和目标漂移（做着做着就偏了）。前者源于执行功能失调，后者源于注意力被无关刺激捕获（来源：[[重锚定与目标漂移]]）。LLM/agent 侧也有完全对应的两个问题：prompt 遵循衰减（模型在长程任务后期开始忘记系统提示）和工具调用偏离（模型在循环中越走越远）。

这两类问题的根源相同：**生成核心缺乏一个可靠的、事件驱动的重锚定机制**。ADHD 大脑需要外部提醒将自己拉回原任务（来源：[[重锚定与目标漂移]]），LLM agent 需要 harness 通过事件驱动的系统提醒来对抗“指令消退”（来源：Building an AI Coding Agent for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned）。

## 答案：Tiimo 补上的是重锚定层，不是规划层

Tiimo 的官方描述是“视觉化日常规划应用”，专为 ADHD 和自闭症成人设计（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。它把时间变成可见的色块，帮助用户感知时间流逝。但仔细分析其核心机制：**它不是在替你规划，而是在你执行过程中持续提供视觉锚点**。每完成一个色块，你看到下一个色块的颜色和位置，就知道“现在该做什么”。

这恰恰是 LLM agent harness 中“重锚定”的等价物。在 agent harness 中，planner 生成任务列表后，executor 每完成一步，系统会检查当前状态是否偏离规划，如果偏离则触发重新规划或提醒（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。Tiimo 的自动重新安排功能（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）本质上就是这个机制：当你实际完成情况偏离计划时，它重新调整后续色块，让你始终有一个“当前目标”。

相比之下，Motion 和 Reclaim.ai 更偏向“规划层”——它们替你决定什么时候做什么。Motion 自动排程，Reclaim.ai 保护时间块（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。这些工具解决的是“决策疲劳”，但**没有解决执行过程中的目标漂移**。很多 ADHD 用户反馈：Motion 排好了日程，但到了时间点还是不想做，或者做着做着就忘了看日程。这就是因为缺少重锚定。

## 证据：两边都缺的不是规划，是重锚定

ADHD 侧：身体在场效应（body doubling）是 ADHD 群体最有效的策略之一——有另一个人在场时，执行功能障碍被绕过（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。Focusmate 等工具通过 AI 匹配虚拟身体搭档，提供外部问责（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...)）。注意，身体搭档并不帮你规划，他们只是“在场”，在你漂移时把你拉回来。这就是重锚定。

LLM/agent 侧：最新的 harness 设计强调“事件驱动的系统提醒”而不是更复杂的规划器。例如，在 agent 执行过程中，每 N 步插入一个检查点，暂停并询问用户或系统是否偏离目标（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。这同样不是规划，而是重锚定。

Todoist 的智能优先级排序（来源：[[Todoist]]）和 Structured 的视觉化时间线（来源：[[Structured]]）也部分提供了重锚定，但它们的 AI 功能细节尚不明确（来源：[[Structured]]），且算法透明度低（来源：[[Todoist]]）。

## 核心观点：脚手架 vs 拐杖的边界在于是否保留重锚定能力

这里必须诚实指出争议。wiki 资料中反复警告过度依赖风险：AI 工具可能成为“永久拐杖”，削弱用户内在的时间感知和规划能力（来源：[[矛盾与存疑]]）。关键在于区分“脚手架”和“拐杖”。

脚手架是**辅助你构建自身能力**的工具：它在你需要时提供重锚定，但允许你逐渐内化这种能力。例如，Tiimo 的视觉色块可以让你慢慢学会感知时间长度，最终不再需要它。拐杖则是**替代你自身能力**的工具：Motion 替你规划一切，你只需服从，久而久之你失去了自己规划的能力。

目前，Tiimo 和 Focusmate 更接近脚手架，因为它们提供的是“提醒”而非“决策”。Motion 和 Reclaim.ai 更接近拐杖，因为它们替你做了规划决策。但个体差异很大：有些 ADHD 用户需要完全替代规划才能启动，有些则只需要偶尔提醒（来源：[[矛盾与存疑]]）。

## 今天就能试的行动

1. **用 Tiimo 做一个下午的视觉时间块**：不要用它规划全天，只规划下午 2-4 点的一个任务。把它分成 25 分钟色块，每个色块后暂停 5 分钟，问自己“我还在做原任务吗？”这就是重锚定训练。

2. **试试 Focusmate 或类似的身体搭档**：找一个朋友或使用虚拟搭档，约定一起工作 45 分钟。过程中不需要交流，只需要知道对方也在工作。注意观察这种“外部在场”如何影响你的任务启动。

3. **如果使用 Motion，每周留一天手动规划**：Motion 的自动排程固然方便，但每周至少有一天，关掉 AI，自己手动排日程。这能防止你对工具的过度依赖，保留自己规划的能力。

4. **对于 LLM agent 工程师**：检查你的 harness 是否过度依赖 planner 的复杂分解，而忽略了事件驱动的重锚定。尝试在每 5 步后插入一个“系统提醒”，输出当前任务列表和已完成步骤，让模型重新确认目标。

## 局限与争议

本文的观点建立在 ADHD 大脑与 LLM 同构的理论类比上，但神经科学和计算机科学的交叉验证不足（来源：[[矛盾与存疑]]）。现有证据主要来自用户报告和概念类比，缺乏大规模对照实验（来源：[[AI 与 ADHD 的时间管理]]）。Tiimo 的 AI 功能细节尚不明确（来源：[[Structured]]），其有效性证据多来自用户报告（来源：[[时间盲]]）。此外，ADHD 亚型（注意力缺陷 vs. 多动冲动）对工具响应不同，现有工具多面向注意力缺陷型（来源：[[矛盾与存疑]]）。

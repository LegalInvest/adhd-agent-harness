# Tiimo 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Tiimo：它到底补上了哪一层执行功能？

## 问题：为什么 Tiimo 对我没用，但别人都说好？

你下载了 Tiimo，把一天涂成彩色时间块，结果第二天还是盯着空白的“9:00-9:30 写报告”发呆。你需要的不是时间块，而是有人告诉你“写报告”到底该先打开哪个文件、写哪句话。

同样，你给 LLM agent 一个长任务“写一篇市场分析报告”，它要么直接输出一堆泛泛之谈，要么陷入循环。你发现，缺的不是生成能力，而是**任务分解**。

## 同构：高产但缺调度层的生成核心

ADHD 大脑与 LLM 在结构上同构：两者都是高产的生成核心（发散思维、创意、文本生成），但缺乏可靠的执行调度层（来源：AI 与 ADHD 的时间管理）。ADHD 的执行功能缺陷——任务启动困难、不会拆解、时间盲——对应 LLM agent 在长程任务中的目标漂移和指令消退（来源：重锚定与目标漂移）。

解决思路也同构：给生成核心套上 harness（脚手架）。在 LLM 侧，agent harness 通过 planner-executor 循环分解任务、设置护栏、引入人在回路（来源：人在回路与身体在场）。在 ADHD 侧，工具通过视觉化、自动排程、身体在场效应提供外部调度（来源：同上）。

## 实测：Tiimo 补上了哪一层？

Tiimo 的核心是**视觉化时间线**，将时间转化为可见元素，直接对抗时间盲（来源：Tiimo）。它属于自适应规划器，能根据实际完成情况自动重新安排日程（来源：同上）。这补的是**时间感知层**——让你知道现在是几点、该做什么。

但问题来了：Tiimo 不帮你分解任务。它假设你已经知道“写报告”这个块里要做什么。对于 ADHD 大脑，“写报告”是一个黑洞——你不知道从哪里下手。这就是为什么 Tiimo 对很多人只是漂亮的日程表，不是真正的执行工具。

对比 Motion 和 Reclaim.ai：Motion 自动排程，消除“下一步做什么”的决策负担（来源：Motion），但它同样不分解任务。Reclaim.ai 保护深度工作时间块（来源：Reclaim.ai），但也不告诉你在块里该做什么。

真正补上**任务分解层**的工具是 Goblin Tools——它能把“写报告”自动拆成“打开文档、列出大纲、写引言……”（来源：Goblin Tools 在矛盾与存疑中提及）。但 Goblin Tools 不提供时间线。

## 判断：脚手架 vs 拐杖——你用的是哪一个？

我的核心观点是：**Tiimo 和 Motion 是优秀的“时间感知脚手架”，但如果不搭配任务分解工具，它们就变成了“美化版待办清单”——一种拐杖，让你感觉有序，实际没有降低启动门槛。**

真正的脚手架应该像 LLM agent harness：先分解（planner），再执行（executor），循环迭代。ADHD 侧的对应是：先用 Goblin Tools 或类似方法把任务拆成微步骤，再用 Tiimo 把微步骤放进时间块。

证据：ADHD 大脑的任务启动困难，核心在于**决策负担**——面对“写报告”这种模糊目标，大脑需要大量能量去决定第一步（来源：AI 与 ADHD 的时间管理）。分解任务直接减少决策负担。而 Tiimo 的视觉化时间线减少的是**时间感知负担**——两者缺一不可。

## 争议与局限

必须诚实指出：
- **过度依赖风险**：长期使用外部工具可能阻碍内在执行功能发展（来源：矛盾与存疑）。如果 Tiimo 只是帮你“看到”时间，而不训练你感知时间，它可能成为永久拐杖。
- **证据不足**：多数工具（包括 Tiimo）的有效性缺乏独立临床研究，主要基于用户反馈（来源：矛盾与存疑）。同构命题本身是理论模型，需更多实证。
- **个体差异**：不同 ADHD 亚型对工具反应不同（来源：矛盾与存疑）。视觉化对时间盲严重的人有效，但对任务分解困难的人无效。

## 今天就能试的行动

1. **组合使用**：下载 Tiimo（或 Structured）做时间线，同时用 Goblin Tools 或 ChatGPT 把任务拆成 5 分钟微步骤。把微步骤填入时间块。
2. **测试“分解后启动”**：选一个你拖延的任务，先让 AI 拆成 5 步，然后只做第一步。对比之前直接开始的阻力。
3. **设置重锚定**：在 Tiimo 每个时间块中间加入 1 分钟“检查目标”提醒，防止目标漂移（来源：重锚定与目标漂移）。
4. **引入身体在场**：如果任务分解后还是启动困难，用 Focusmate 或找一个朋友视频连线，利用“对另一个人的责任感”绕过执行屏障（来源：人在回路与身体在场）。

## 结语

Tiimo 之于 ADHD，就像 planner 之于 LLM agent——但如果你只用了 planner 而没有 executor，你只解决了“知道什么时候做什么”，没解决“知道怎么做”。真正的 harness 是两者结合：分解 + 时间感知 + 重锚定。别再用拐杖了，搭一个完整的脚手架。

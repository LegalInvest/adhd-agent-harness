# 用 Motion 给 ADHD 补上重锚定与目标漂移，是脚手架还是拐杖？

> Motion 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：目标漂移——ADHD 与 Agent 的共同困境

你打开 Motion，规划好今天的三件要事。两小时后，你发现自己正对着一个无关的网页发呆，或者已经花了一小时“完善”某个根本不重要的细节。这种经历对 ADHD 人群来说如同家常便饭——分心走神，或者超聚焦跑偏，最终丢失了最初的目标。

同样的事情也发生在 AI Agent 身上。当一个长程任务被交给 LLM 驱动的 Agent，它可能在执行中途被某个中间结果吸引，偏离原始指令，产生所谓的“目标漂移”。工程师们为此设计了精巧的 harness（编排层），通过定期重锚定 system prompt 来把 Agent 拉回正轨。

ADHD 大脑与 LLM 在结构上同构：两者都是高产的生成核心，但缺乏可靠的执行调度层（来源：AI 与 ADHD 的专注力）。因此，AI 工具帮助 ADHD 专注的本质是“给生成核心套 harness”——通过外部脚手架补偿执行功能缺陷。Motion 正是这类工具的代表：它通过自动调度、任务分解和优先级排序，主动替用户决策“下一步做什么、何时做”，从而移除决策带来的认知负荷（来源：认知负荷）。

但一个尖锐的问题随之浮现：Motion 究竟是帮助 ADHD 大脑学会更好导航的脚手架，还是让人永远依赖的外挂拐杖？

## Motion 的重锚定机制：从 ADHD 到 Agent 的对照

Motion 的核心功能可以概括为“外部执行功能层”。它自动将大任务拆解为子任务，排入日历，并根据优先级和截止日期动态调整。这相当于为 ADHD 大脑安装了一个外置的调度器——就像 Agent 工程中，harness 负责将复杂任务分解、编排执行步骤，并在每一步检查是否偏离原始目标。

从认知负荷理论看，ADHD 患者的工作记忆容量有限，处理复杂信息时更容易感到不堪重负。AI 工具通过自动决策和自动执行，显著减轻工作记忆上的认知负荷（来源：认知负荷，引用 CHADD 文章）。Motion 正是这样做的：它替用户决定“下一步做什么、何时做、如何优先排序”，从而移除决策带来的认知负荷（来源：认知负荷）。

在 Agent 侧，类似的机制被称为“任务分解技术”。通过将复杂任务分解为子任务，可以减轻 LLM 的认知负荷，减少幻觉等错误（来源：认知负荷）。无论是 Motion 还是 Agent harness，本质上都是通过外部结构来补偿内部调度能力的不足。

然而，Motion 的重锚定方式与 Focusmate 或 Brain.fm 不同。Focusmate 依赖身体在场效应——通过实时同伴问责提供外部结构和社会压力（来源：Focusmate）。Brain.fm 则通过神经锁相技术影响大脑信息处理方式（来源：Brain.fm）。Motion 更接近 RescueTime 的自动追踪逻辑，但更主动：它不只是记录，而是直接调度。

## 脚手架还是拐杖？边界在哪里

要回答这个问题，必须引入 wiki 资料中“矛盾与存疑”部分的核心争议：脚手架应设计为可逐步撤除的临时支撑，但多数工具（如 Motion）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。

如果 Motion 的使用者从未尝试在没有它的情况下规划任务，从未内化它的调度逻辑，那么它就成了拐杖——一旦脱离，执行功能缺陷依旧，甚至可能因为长期依赖而萎缩。反之，如果用户将 Motion 作为学习工具，观察它如何分解任务、设定优先级，并逐步将这些策略内化，那么它就是脚手架。

在 Agent 工程中，同样存在这种边界。好的 harness 不仅纠正目标漂移，还会记录漂移模式，用于改进底层模型。但许多工程实践只是简单地每 N 步重注入 system prompt，从不分析漂移原因——这相当于给 Agent 一个永久的拐杖，而非让它学会自我修正。

现有证据表明，ADHD 工具的效果多基于理论推断而非独立临床研究。Brain.fm 缺乏针对 ADHD 的随机对照试验（来源：Brain.fm）。Focusmate 的效果归因于身体在场而非 AI 本身（来源：AI 与 ADHD 的专注力）。Motion 同样缺乏长期研究证明脚手架可逐步撤除（来源：矛盾与存疑）。

## 我的判断：Motion 是高效脚手架，但需有意识使用

我认为，Motion 对于 ADHD 人群是当前最接近“重锚定 harness”的工具。它直接解决了目标漂移的核心问题——通过外部调度持续将注意力拉回既定轨道。但它的角色取决于使用方式：

- **作为脚手架**：用户主动学习 Motion 的调度逻辑，尝试在没有它时手动应用类似的分解和优先级排序，逐步减少依赖。
- **作为拐杖**：用户完全交给 Motion，从不反思自己的执行功能模式，一旦工具失效便寸步难行。

对于 Agent 工程师，启示类似：harness 应设计为可配置的“训练轮”，而非永久外骨骼。定期检查 Agent 是否在无 harness 时也能保持目标，才是工程成熟度的标志。

## 今天就能试的行动

1. **观察 Motion 的调度模式**：用一周时间，记录 Motion 如何分解你的任务。然后尝试手动模仿它的逻辑来规划一天，看看自己的执行功能是否有所提升。
2. **设置“撤除实验”**：每两周选择一天完全不用 Motion，仅靠手动规划和提醒。对比当天的产出和感受，评估自己的重锚定能力是否进步。
3. **为 Motion 添加“漂移日志”**：每次 Motion 提醒你切换任务时，花 10 秒记录你刚才在做什么。这类似于 Agent 的日志系统，能帮你识别自己的漂移模式。
4. **结合 Focusmate 使用**：在 Motion 调度出的任务时段中，预约 Focusmate 伙伴。外部问责 + 自动调度形成双重锚定，效果可能 1+1>2。

## 局限与诚实声明

本文的判断基于概念对齐和工具机制分析，而非严格的临床证据。Motion 针对 ADHD 的效果缺乏独立研究，其长期影响（特别是对执行功能发展的影响）仍属未知。ADHD 亚型（注意力缺陷 vs 多动冲动）对 Motion 的反应可能不同，现有工具未充分个性化（来源：AI 与 ADHD 的专注力）。此外，过度依赖时间调度可能引发焦虑，这与 RescueTime 的局限类似（来源：RescueTime）。

最终，Motion 是脚手架还是拐杖，不取决于工具本身，而取决于使用者的意识与策略。对于工程师，同样的道理适用于 Agent harness：设计时就要考虑何时撤除，否则它永远只是拐杖。

# 用 Motion 给 ADHD 补上重锚定与目标漂移，是脚手架还是拐杖？

> Motion 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：你是在用 Motion 重锚定，还是被 Motion 带着漂移？

如果你有 ADHD，你一定体验过这种场景：打开 Motion，看到它已经帮你规划好了今天的所有任务——9:00-10:00 写报告，10:00-10:30 回邮件，10:30-11:30 开会……你松了一口气，按部就班执行。但两小时后，你发现自己正在研究 Motion 的配色方案，或者因为某个任务超时导致整个日程错乱，你干脆放弃了计划，开始刷手机。

如果你是做 Agentic Harness 的工程师，你一定也见过这种场景：你给 agent 设定了一个长程任务，比如“写一份市场分析报告”，它一开始很顺利，但跑了 20 步之后，它开始纠结于某个无关细节，或者因为上下文膨胀而忘记了最初的目标，输出变成了“分析报告+顺便写一首诗”。

这两个场景有一个共同的核心问题：**目标漂移**。ADHD 大脑和 LLM/Agent 在本质上都是“高产但缺执行调度层的生成核心”（来源：主题综述）。它们能产生大量创意或输出，但缺乏一个可靠的重锚定机制来确保自己始终在正确的轨道上。Motion 这类 AI 排程工具，承诺帮我们解决这个问题——它自动安排任务、动态调整、减少决策负担。但它究竟是帮助我们重锚定的脚手架，还是让我们放弃自我调度能力的拐杖？

## 同构脊柱：ADHD 大脑与 LLM 都需要“重锚定”

ADHD 的执行功能缺陷，尤其是工作记忆不足和时间盲，导致个体容易在任务中途丢失目标（来源：任务启动）。LLM 的“无状态性”和上下文膨胀同样会导致 agent 在长程任务中偏离原始指令。两者的解法在结构上同构：都需要一个**外部锚点**来定期将注意力拉回核心目标。

在 ADHD 侧，最经典的 harness 是曾国藩的“日课十二条”。曾国藩年轻时典型注意力不集中：浮躁坐不住，天天串门喝酒，日记里骂自己“无恒”“浮躁”，读书慢到“他人目下二三行，余或疾读不能终一行”。他的解决方案是每天固定的十二条行为准则：黎明即起、读书不二、谨言、写日记反省……这些动作本质上就是**定时 re-grounding**——每天多次把注意力锚定在核心价值上，对抗目标漂移。这就像一个 agent 的 system prompt 里写满了“每 5 步检查一次是否偏离原始目标”。

在 LLM/Agent 侧，工程实践中常见的重锚定方案包括：在 prompt 中嵌入“你当前的任务是……，请每完成一个子步骤就回顾一次原始目标”，或者使用外部记忆系统（如向量数据库）来维持上下文连续性。这与 ADHD 群体自发使用外部工具（如 Focusmate 的身体在场、RescueTime 的时间记录）来补偿执行功能缺陷的行为完全同构（来源：任务启动）。

## Motion 是脚手架还是拐杖？关键在于“谁设定意图”

Motion 的核心功能是自动排程：它分析你的任务列表，根据优先级和截止时间自动安排到日历中，并动态调整。对于 ADHD 用户，这直接解决了任务启动困难和时间盲——你不需要自己决定“接下来做什么”，Motion 替你做决定。这听起来很棒，但问题在于：**Motion 替你决定的是“怎么做”，而不是“为什么做”。**

脚手架的定义是：一个临时结构，帮助你完成自己无法独立完成的任务，同时你仍然在主动参与核心决策。拐杖的定义是：一个永久替代物，你依赖它完成原本属于你的功能，并且你的能力因此退化。

Motion 沦为拐杖的场景：你完全依赖它的排程，不再思考任务优先级，不再反思自己的时间分配，甚至当它排出一个不合理的日程时（比如把三个重要会议排在一起），你也不去调整，而是被动执行。你失去了对目标本身的掌控。

Motion 成为脚手架的场景：你仍然在每天开始前手动设定“今天最重要的三件事”，把 Motion 当作执行工具而非决策者；你利用它的自动排程来降低启动阻力，但定期（比如每天结束前）回顾计划与实际执行的偏差，调整自己的意图。你保留了对“为什么做”的锚定权。

这与 Agent 工程中的重锚定原则完全一致：system prompt 可以自动执行子任务，但核心目标必须由人类工程师定期审视和调整。如果 agent 完全自主决定目标，它就会漂移；如果人类完全放手，agent 就会变成失控的生成机器。

## 真实证据与争议

现有工具的证据表明，Focusmate 的身体在场效应可将生产力提升高达 40%（来源：身体在场效应），RescueTime 通过客观时间记录改善时间盲（来源：RescueTime），但这些工具的作用机制都是**外部结构**，而非内在能力提升。Brain.fm 和 Endel 虽有神经科学理论支持，但缺乏针对 ADHD 的独立临床研究（来源：Brain.fm；矛盾与存疑）。Motion 本身同样缺乏临床证据，用户反馈积极但初始设置成本高（来源：矛盾与存疑）。

争议的核心在于依赖风险：长期使用 AI 工具是否会导致 ADHD 患者的内在执行功能退化？（来源：矛盾与存疑）目前没有纵向研究能回答这个问题。但我们可以从同构视角推断：如果用户只是被动接受 Motion 的安排，就像 agent 从不重锚定 system prompt，那么漂移和退化是必然的。但如果用户主动参与意图设定，Motion 就只是一个更高效的执行层。

## 今天就能试的行动

1. **设定“意图锚点”**：每天使用 Motion 前，花 2 分钟写下“我今天最重要的一个目标是什么”。把它写在便签纸上贴在屏幕边，或者作为 Motion 任务的备注。这是你的 system prompt。
2. **每周一次“重锚定回顾”**：每周日晚上，回顾 Motion 的排程与实际执行情况。问自己：哪些任务被漂移了？为什么？然后调整下周的优先级。这是你的 agent 评估流程。
3. **混合使用工具**：Motion 负责排程，但搭配 Focusmate（每周 3 次虚拟身体加倍）来强化任务启动（来源：Focusmate）。或者用 RescueTime 自动追踪，每周看一次时间报告，识别注意力漂移模式。不要只依赖一个工具。
4. **实验“手动模式”**：选一天完全不用 Motion 的自动排程，自己手动安排所有任务。对比前后两天的效率和感受。这能帮你觉察自己是否过度依赖。

Motion 不是解药，它只是一个 harness。真正的解药是你对“重锚定”的主动意识——无论你是 ADHD 患者，还是 Agent 工程师，都需要定期问自己：“我现在做的事情，是否还在最初的目标轨道上？” 这个问题，不能交给任何工具来回答。

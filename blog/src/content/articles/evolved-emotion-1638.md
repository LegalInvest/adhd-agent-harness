# 为什么用 Brain.fm 治 ADHD 的情绪失调，和给 agent 套 会褪去的脚手架 是一回事？

> Brain.fm 实测：同一套 harness 思路，ADHD 与 LLM 两边都成立。

## 问题：情绪失调，不只是“心情不好”

如果你有 ADHD，你一定体验过这种场景：明明知道该做什么，但大脑像被按了暂停键，焦虑、挫败、愤怒瞬间淹没理智——这就是情绪失调。它不是普通的情绪波动，而是执行功能缺陷的副产品：工作记忆超载、任务启动困难、时间盲让你对“多久能做完”毫无概念，于是情绪成了最先失控的变量（来源：AI 与 ADHD 的情绪调节）。而另一边，做 LLM agent 的工程师也有类似的噩梦：模型明明能生成惊艳的回答，却会在简单任务上陷入无限循环、调用工具失控、输出偏离目标——agent 也“情绪失调”了，只不过表现为行为失控。

这两个问题看似风马牛不相及，但它们的底层结构一模一样：一个强大的生成核心（ADHD 大脑的联想创造力 / LLM 的语言生成能力），配了一个不可靠的调度层（执行功能 / 编排系统）。情绪调节正是调度层的职责——当工作记忆过载或时间感知失灵时，情绪容易失控（来源：AI 与 ADHD 的情绪调节）。而 agent 的护栏和规划层一旦缺失，模型也会“失控”。

## 同构一：外部脚手架补偿调度缺陷

ADHD 侧最有效的策略之一，是“身体在场效应”（body doubling）：有另一个人在场时，执行功能障碍被绕过，任务得以启动和维持（来源：人在回路与身体在场）。Focusmate 等工具通过 AI 匹配虚拟身体搭档，提供外部问责（来源：人在回路与身体在场）。这本质上是一个外部调度层——把“开始做事”的责任外包给另一个人的存在感。

LLM/agent 侧，对应的正是 agent harness（脚手架）。Harness 通过 human-in-the-loop 监督提供外部调度：设置护栏（token 预算、工具调用次数上限、防止无限循环）、加入人工检查点（暂停并询问后再执行）（来源：人在回路与身体在场）。模型本身强大，可靠性来自架构+护栏（来源：人在回路与身体在场）。两边都在用“外部结构”补偿内部调度缺陷。

## 同构二：Brain.fm 和 harness 都是“神经锁相”

Brain.fm 声称通过神经锁相技术帮助用户进入专注状态（来源：矛盾与存疑）。它的原理是用特定频率的音频引导大脑波动，本质上是一种外部节律调度器——把大脑的混乱波动“锁定”到目标频率上。这跟 agent harness 给 LLM 加 token 预算、设置调用次数上限，是同一回事：用外部约束来稳定内部输出。

但注意，Brain.fm 针对 ADHD 的效果缺乏独立临床研究支持（来源：矛盾与存疑）。同样，harness 的有效性也高度依赖具体实现——一个设计糟糕的 harness 可能反而限制模型能力。两边的“神经锁相”都面临一个核心矛盾：外部调度越强，内部能力越可能萎缩。

## 同构三：脚手架 vs 拐杖——可褪去的设计

这是本文最关键的判断：脚手架必须设计为可逐步撤除的临时支撑，否则沦为拐杖（来源：矛盾与存疑）。ADHD 侧，Goblin Tools 的“魔法待办”能把复杂任务分解为小步骤，降低启动门槛（来源：Goblin Tools）。但如果你永远依赖它分解任务，你的任务启动能力永远不会进步。同样，Saner.AI 通过增强知识回忆减少搜索循环（来源：Saner.AI），但过度使用可能削弱你主动回忆的能力。

LLM 侧，harness 的护栏如果永远不调整，模型就永远学不会自我约束。好的 harness 应该像训练轮——随着 agent 能力提升，逐步放松限制。但现实是，多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。这正是争议所在：工具设计是否考虑撤除？长期依赖是否阻碍内在能力发展？

## 证据脉络与局限

证据方面，ADHD 侧有用户反馈支持：Goblin Tools 被评价为“简单有用”（来源：Goblin Tools），Inflow 基于背外侧前额叶皮层作为因果流入中枢的神经科学原理（来源：Inflow），但缺乏直接针对工具的随机对照试验（来源：矛盾与存疑）。LLM 侧，harness 工程已有成熟实践，如形式化规划与护栏（来源：人在回路与身体在场），但同样缺乏大规模实证比较不同护栏设计的效果。

多巴胺干预的有效性存在争议：基于多巴胺的 AI 干预（如奖励系统）可能强化依赖而非内在动机（来源：矛盾与存疑）。这提醒我们，无论是 Brain.fm 还是 harness，都不能简单视为“解决方案”，而是需要谨慎评估的临时工具。

## 今天就能试的行动

1. **ADHD 读者**：用 Brain.fm 的“专注”模式写一份待办清单，同时打开 Goblin Tools 的 Magic ToDo 分解其中最难的一项。感受外部脚手架如何降低启动焦虑。记录你使用后的情绪变化。
2. **工程师读者**：为你的 agent 添加一个“人工检查点”：每执行 3 步工具调用后暂停，要求用户确认方向。对比加装前后 agent 的行为可靠性。同时，尝试在代码中设置一个“褪去计划”：每 100 次运行后自动放宽一次护栏限制，观察 agent 是否适应。
3. **双方共同**：反思你依赖的“脚手架”是否可褪去。ADHD 用户问自己：如果我失去这个工具，我还能完成同样的任务吗？工程师问：如果我移除这个护栏，agent 会崩溃吗？如果答案是“不能”，那它就是拐杖，需要重新设计。

## 总结

Brain.fm 治情绪失调，和给 agent 套会褪去的脚手架，本质上是在做同一件事：用一个临时外部调度层，补偿一个强大但不可靠的生成核心。但真正的智慧不在于搭建脚手架，而在于知道什么时候拆除它。否则，我们只是换了一种方式依赖。

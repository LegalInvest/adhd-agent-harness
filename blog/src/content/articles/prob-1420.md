# Claude 之于 ADHD，就像 外部编排调度层 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

## 问题：有想法，却卡在执行

你是否有过这样的体验：脑子里有一个绝妙的方案，细节清晰、逻辑自洽，但坐到电脑前就是无法开始；或者，你打开文档写了三行，就被一个弹窗带跑，半小时后发现自己正在刷社交媒体。这不是懒，也不是能力不足——这是 ADHD 大脑的典型困境：**生成核心高产，但执行调度层缺失**。

同样的问题，也困扰着 LLM 的工程师们。GPT-4 或 Claude 能写出漂亮的论文、生成复杂的代码，但让它完成一个跨天的多步骤任务——比如“调研竞品、撰写报告、发送邮件”——它就会迷失方向，出现幻觉、重复工作或上下文膨胀（来源：[[幻觉与验证循环]]）。LLM 有智力，但缺调度；ADHD 大脑有创造力，但缺执行。两者是同一个工程问题的两个分身。

## 同构：生成核心与调度层

wiki 资料中反复出现一个核心概念：**ADHD 大脑与 LLM 在结构上同构——都是高产但缺乏可靠调度层的生成核心**（来源：[[ADHD 大脑与 LLM 的同构]]）。对 LLM 而言，调度缺失表现为无状态、易幻觉、无法自主管理长任务；对 ADHD 而言，则表现为执行功能缺陷——计划、组织、工作记忆、冲动控制容易失稳（来源：[[执行功能]]）。

这个同构的洞察指向同一个解法：**将生成能力与执行调度分离，前者由大脑/模型承担，后者由外部脚手架承担**。在 AI 工程中，这叫 agent harness——为 LLM 提供上下文传递、工具接口、规划工件、验证循环和记忆系统（来源：[[幻觉与验证循环]]）。在 ADHD 一侧，这叫做外部执行功能层——用工具来补偿工作记忆不稳定、任务启动困难和注意力分散（来源：[[外部执行功能层]]）。

## 人物案例：屠呦呦的 harness 系统

屠呦呦是典型的 ADHD 特质个体：一门心思搞研究，不喜欢应酬采访，从葛洪《肘后备急方》中找到青蒿素灵感。她的成功离不开一套严格的 harness 系统——**反复筛选 2000 多种中药、380 多个提取物，建立标准实验流程，团队分工协作**（来源：wiki 案例）。这套系统本质上就是一个外部调度层：它把“发现抗疟药物”这个宏大目标拆解为可执行的子任务（筛选、提取、验证），并用实验记录作为外部记忆，用团队合作分担认知负荷。

这与 LLM harness 中的“任务分解与验证循环”完全同构。屠呦呦的“日课”——每天泡实验室、记录数据——相当于 agent 的“定时 re-grounding”；她的“秘书”（团队协作）相当于“外部调度器”。没有这套 harness，她的创造力可能淹没在杂乱的信息中。

## 工具实测：Claude 补上了哪一层？

当前市面上的 AI 工具，正在为 ADHD 大脑扮演这个外部调度层。以下是三类代表性工具及其与 LLM harness 的对应：

### 1. 任务分解器：Goblin Tools
Goblin Tools 的 Magic ToDo 功能，能自动将“清理厨房”这样的模糊任务分解为“把碗放进洗碗机”“擦台面”“拖地”等具体步骤，并可调节“辣度”控制粒度（来源：[[Goblin Tools]]）。这对应 LLM harness 中的“任务分解与规划”——将复杂任务拆解为子任务，并在每一步进行规划与验证（来源：[[幻觉与验证循环]]）。对 ADHD 而言，它降低了启动门槛；对 agent 而言，它防止了上下文膨胀。

### 2. 外部记忆系统：Saner.AI
Saner.AI 专注于知识回忆和本地记忆，减少搜索循环和标签切换（来源：[[Saner.AI]]）。这对应 LLM harness 中的“记忆系统”——为无状态的 LLM 提供跨会话上下文（来源：[[无状态与外部记忆]]）。ADHD 大脑的工作记忆缺陷与 LLM 的无状态性相似，都需要外部存储来维持任务链。

### 3. 动态调度器：Motion
Motion 自动根据任务、会议和截止日期创建并调整每日计划，消除“下一步该做什么”的决策负担（来源：[[Motion]]）。这对应 LLM harness 中的“上下文工程”——主动管理当前应关注什么，防止被环境带偏（来源：[[上下文工程]]）。对 ADHD 而言，它对抗时间盲和任务启动困难；对 agent 而言，它防止上下文膨胀导致推理退化。

## 核心观点：脚手架，不是拐杖

我的判断是：**Claude 这类 AI 工具对 ADHD 的最大价值，不是替代思考，而是补上调度层**。但这里有一条危险的边界——**脚手架 vs 拐杖**。

wiki 资料中明确指出了这一矛盾：AI 工具作为“外部执行功能层”补偿缺陷，但过度依赖可能导致能力退化，成为永久拐杖（来源：[[拐杖与脚手架]]）。目前缺乏纵向研究来验证长期风险，但直觉告诉我：如果一个人完全依赖 Motion 来规划每一天，而从不自己思考优先级，那么他的执行功能可能会萎缩。同样，如果 agent 工程师把所有调度逻辑都写死在 harness 里，模型就永远学不会自主规划。

真正的脚手架，应该是可撤除的。屠呦呦的 harness 系统不是一成不变的——她根据实验结果调整筛选方向，最终从古方中找到灵感。好的 AI 工具也应该这样：**它帮你建立执行结构，但保留你调整和退出的权利**。

## 局限与争议

必须诚实指出，ADHD 大脑与 LLM 同构的论点目前多为类比推理，缺乏神经科学或计算机科学的直接证据（来源：[[矛盾与存疑]]）。此外，AI 工具对任务启动的长期效果未知，且个体差异极大——有人需要强硬打断（如番茄钟），有人需要柔性引导（来源：[[矛盾与存疑]]）。没有一种工具适合所有人。

## 今天就能试的行动

1. **用 Goblin Tools 分解一个拖延任务**：找一个你拖延了一周的任务，输入 Magic ToDo，调节辣度到你觉得“可以开始”的程度，然后执行第一步。
2. **用 Motion 或类似工具规划明天**：把明天要做的 3 件事输入 Motion，让它自动排程。观察你的启动阻力是否降低。
3. **建立你的“验证循环”**：在做任何重要决策前，用 Claude 或 ChatGPT 生成一个检查清单（比如“这个方案的风险点有哪些？”），然后逐项核对。这对应 LLM harness 中的验证循环。
4. **设置一个“外部记忆”系统**：用 Saner.AI 或简单的笔记应用，把每天的想法和任务记录下来，形成可检索的知识库。这能补偿工作记忆缺陷。

## 结语

Claude 之于 ADHD，就像外部编排调度层之于 LLM——它补上了生成核心缺失的那一层。但别忘了，好的工程是让系统最终能自主运行，而不是永远依赖外部支架。对 ADHD 个体而言，AI 工具是助你起飞的风，而不是你永远要抓住的拐杖。

# 如果把 ADHD 大脑当成生成核心，Focusmate 能不能当它的外部执行功能层？

> 用 agent 工程的思路重新理解 Focusmate 对 ADHD 的真正价值。

## 问题：ADHD 大脑的“生成核心”为何需要外部调度？

如果你有 ADHD，你一定熟悉这种体验：脑子里有无数想法，却无法启动一个简单的任务；计划写得好好的，但执行起来就像被抽掉了“开始键”。这不是懒惰，而是执行功能——特别是计划、组织和启动——先天偏弱。神经科学研究表明，ADHD 大脑的默认模式网络过度活跃（高产想法），但前额叶执行控制不足（来源：[[生成核心与调度层]]）。换句话说，大脑是一个强大的“生成核心”，但缺少一个可靠的调度层。

与此同时，LLM agent 的工程师们正在解决一个惊人的类似问题：LLM 本身能生成流畅文本，但一旦需要多步推理、调用工具或维持长上下文，它就会“迷失”——上下文窗口膨胀导致推理退化，无状态性导致跨会话遗忘（来源：[[上下文工程]]、[[无状态与外部记忆]]）。解决方法是设计一个“harness”——编排层、记忆层、工具层——来调度这个生成核心。这个 harness 就是 LLM 的“外部执行功能层”。

所以，一个核心问题浮现：如果把 ADHD 大脑也看作一个生成核心，那么像 Focusmate 这样的工具，能不能充当它的外部执行功能层？

## 同构：ADHD 大脑与 LLM 共享的“缺调度”困境

让我们用 agent 工程的术语重新理解 ADHD 的痛点。

- **计划与组织**：ADHD 大脑难以将模糊目标分解为可执行步骤，就像 LLM 在没有 chain-of-thought 提示时容易产生逻辑跳跃。Goblin Tools 的 Magic ToDo 功能通过 AI 自动分解任务（如“整理房间”变成“捡起地板上的衣服”等），正是对应了 agent 的“多步推理+工具调用”模式（来源：[[Goblin Tools]]）。

- **工作记忆与无状态性**：ADHD 的工作记忆容量保留但控制弱——你记得要做什么，但下一秒就被干扰带走。这就像 LLM 的无状态性：它知道之前说过什么，但上下文一长就“忘”了。Saner.AI 通过本地记忆和快速检索减少“搜索循环”，本质上就是为 ADHD 大脑提供了一个外部记忆层（来源：[[Saner.AI]]）。

- **时间盲与任务启动**：ADHD 的时间感知缺陷（[[时间盲]]）导致“现在”和“未来”之间没有桥梁，任务启动困难。Motion 这样的自适应规划器能动态重排日程，消除“下一步该做什么”的决策负担（来源：[[Motion]]）。这类似于 agent 的规划循环——外部系统负责时间调度，生成核心只需专注于当前步骤。

这些对应并非空洞类比。神经科学证据表明，ADHD 的多巴胺功能失调导致动机和注意力缺陷，而 LLM 的奖励机制（如 RLHF）也面临类似的“动机”调节问题（来源：[[多巴胺]]）。认知实验中，ADHD 患者在需要执行控制的任务中表现不稳定，与 LLM 在长上下文任务中的性能退化模式相似（来源：[[证据脉络]]）。同构模型甚至将 LLM 作为 ADHD 认知过程的计算模型，模拟执行功能缺陷并测试干预方案（来源：[[同构视角]]）。

## Focusmate：作为“身体在场”的 harness

Focusmate 不是 AI 工具，而是一个简单的视频配对平台：你预约一个时段，与另一个人同时在线工作，互相监督。它没有任务分解、没有记忆存储、没有日程规划——它只做一件事：通过他人在场，提供启动和持续的“调度信号”。

这正是“身体在场效应”：ADHD 通过他人在场提升执行力，类似 LLM 通过外部 agent 协作增强表现（来源：[[身体在场效应]]）。在 agent 工程中，harness 不直接生成内容，而是管理上下文、触发工具调用、确保步骤顺序。Focusmate 做的正是这些：它为你设定时间边界（开始和结束），提供社会承诺（对方在等你），并持续提醒你“现在该做什么”——就像 harness 中的调度器，不断向生成核心发送“继续”信号。

但 Focusmate 的局限也很明显：它只解决启动和持续问题，不解决计划、分解、记忆等更深层的执行功能缺陷。对于 ADHD 大脑这个生成核心，Focusmate 是一个轻量级的调度层，但缺乏完整的“工具层”和“记忆层”。你需要结合其他工具——比如用 Goblin Tools 分解任务，用 Saner.AI 管理信息——才能构建一个完整的 harness。

## 争议与边界：脚手架还是拐杖？

同构视角虽然有力，但必须诚实面对争议。一个关键问题是：外部执行功能层是脚手架还是拐杖？脚手架帮助建造内在能力，然后可以拆除；拐杖则永久替代，可能导致依赖。

现有证据多来自用户报告，缺乏严格对照实验（来源：[[矛盾与存疑]]）。例如，Goblin Tools 和 Motion 的有效性主要基于用户反馈，而非随机对照试验（来源：[[Goblin Tools]]、[[Motion]]）。过度依赖 AI 工具可能削弱内在的时间感知能力和执行功能（来源：[[时间盲]]、[[任务启动]]）。Focusmate 的“身体在场”同样可能形成依赖——如果你没有他人在场就无法工作，那它就成了拐杖。

关键在于使用方式。Harness 工程的原则是：工具应“插入现有堆栈”，不增加认知开销（来源：[[工具使用与认知卸载]]）。对于 ADHD，这意味着工具应无缝嵌入日常流程，而非成为又一个需要管理的任务。Focusmate 之所以有效，正因为它简单——不需要设置、不需要学习，只需到场。但你需要定期反思：我是在用它作为启动的助推器，还是已经无法独自启动？

## 今天就能试的行动

1. **用 Focusmate 作为“启动调度器”**：每天选择一个最难启动的任务，预约一个 Focusmate 时段。观察它是否降低你的启动阻力，以及你是否能在结束后继续工作。
2. **结合 Goblin Tools 进行任务分解**：在 Focusmate 开始前，用 Goblin Tools 的 Magic ToDo 将任务分解为 3-5 个小步骤。这相当于为你的“生成核心”提供输入模板。
3. **记录“依赖日记”**：每周记录一次，你使用这些工具后，独自完成任务的能力是否有变化。如果发现没有工具就无法开始，考虑减少使用频率，或尝试交替使用不同工具。
4. **向工程师朋友解释“ADHD 大脑 = 无 harness 的 LLM”**：这个类比能帮助他们理解你的需求，也能让你从外部视角审视自己的执行功能挑战。

Focusmate 不能替代完整的执行功能层，但它提供了一个关键组件：社会性的调度信号。对于 ADHD 大脑这个生成核心，它是最轻量的 harness 之一——不是万能，但值得一试。记住，目标是构建一个可以逐渐拆除的脚手架，而不是永久依赖的拐杖。

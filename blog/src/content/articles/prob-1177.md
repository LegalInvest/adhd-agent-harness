# Brain.fm 之于 ADHD，就像 function calling 工具调用 之于 LLM——但有人用错了

> 从同构视角实测 Brain.fm：它到底补上了哪一层执行功能？

## 问题：为什么 Brain.fm 对一些人有用，对另一些人却无效？

如果你是一名 ADHD 患者，你可能试过无数专注音乐、白噪音、双耳节拍。如果你是一名 AI 工程师，你可能给 LLM 接上过一堆 function call 工具，却发现 agent 依然在简单任务上“走神”。两个场景共享同一个困惑：工具明明对路，为什么用起来时灵时不灵？

答案藏在同构视角里：ADHD 大脑与 LLM 都是“高产但缺执行调度层的生成核心”（来源：ADHD 大脑与 LLM 的同构）。Brain.fm 之于 ADHD，就像 function calling 之于 LLM——它们都试图补偿一个特定缺陷：**状态维护**。但很多人用错了，因为它补的不是“动机启动”，而是“持续保持”。

## 同构：状态维护层的缺失

ADHD 一侧：执行功能的核心困难之一是工作记忆的“掉落”和任务启动困难（来源：Best AI Tools for ADHD Productivity in 2026）。更精确地说，ADHD 个体并非不能开始，而是**不能持续**——注意力像信号不稳的广播，频繁跳频。实证研究表明，ADHD 患者呈现“强记忆、弱控制”的认知剖面：工作记忆容量可能正常甚至超常，但认知灵活性和注意控制存在核心缺陷（来源：Strong Memory, Weak Control）。这与 LLM 的困境高度一致：LLM 在短上下文中表现正常，但随着序列变长，在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突（来源：Deficient Executive Control in Transformer Attention）。

LLM/harness 一侧：LLM 本身是无状态、仅生成文本的核心，需要外部“脚手架”来提供工具接口、上下文管理、安全执行等编排能力（来源：Building AI Coding Agents for the Terminal）。Function calling 正是补偿 LLM 无法自主调用外部工具、无法维持多步状态的缺陷。但若没有精心设计的 harness，LLM 会陷入“工具调用循环”——不断请求外部工具却无法整合结果，就像 ADHD 患者打开无数标签页却无法执行任何一项。

**Brain.fm 补上的正是这一层：它通过神经声学刺激维持大脑的“状态维护”频率，而非提供动机或任务分解。** 它类似于 harness 中的“定时 re-grounding”机制——每 15 分钟一次的节奏提醒，帮助大脑保持在任务轨道上。

## 人物案例：孔子的“克己复礼”与张居正的“考成法”

孔子（春秋/儒家创始人）是典型的 ADHD 大脑：精力旺盛、周游列国 14 年坐不住；冲动爱骂人，思维跳跃，《论语》全是场景化语录，无系统著作。他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。这相当于 LLM 的 **context engineering**：通过外部规则（礼）维持行为的一致性，而非依赖内在意志。孔子的“日课”就是他的 re-grounding 机制，与 Brain.fm 的定时节拍异曲同工。

张居正（明朝/万历首辅）则是另一个例子：12 岁中秀才，16 岁中举人，工作狂到每天办公至深夜。他的 harness 是“考成法”——严格考核官员，用制度管别人也管自己。这相当于 LLM 的 **agent scaffolding**：通过外部调度器（考成法）确保任务按计划推进。张居正的“秘书”就是他的外部调度器，与 function calling 中“将控制逻辑外化为可移植工具”的思维同构（来源：外部执行功能层）。

## 证据：Brain.fm 到底补上了什么？

Brain.fm 通过 AI 生成的神经声学音乐，旨在诱导大脑进入专注状态。但 wiki 资料明确指出：“在 ADHD 人群中的独立临床证据仍有限”（来源：矛盾与存疑）。这不是说它无效，而是说它**补的不是动机**。Goblin Tools 通过任务分解降低启动门槛（来源：Goblin Tools），Saner.AI 通过知识回忆减少搜索循环（来源：Saner.AI），Lex 通过单一指令触发多步骤任务（来源：Lex）——这些工具补的是**任务启动**和**工作记忆**。而 Brain.fm 补的是**状态维护**：它帮助大脑维持在稳定频率上，避免注意力跳频。

但问题在于，很多人把 Brain.fm 当作“拐杖”而非“脚手架”。拐杖意味着离开它就无法工作；脚手架则是在训练后可以拆除。同构视角提醒我们：LLM 的 function calling 也可能成为拐杖——如果 agent 过度依赖外部工具而丧失基本推理能力。同理，ADHD 用户如果依赖 Brain.fm 才能专注，而不训练内在的注意力调节能力，就会陷入依赖循环（来源：拐杖与脚手架）。

## 争议与局限

1. **同构性缺乏实证基础**：ADHD 大脑与 LLM 的同构目前只是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。虽然神经机制上有相似性，但不能等同。
2. **Brain.fm 的临床证据薄弱**：在 ADHD 人群中的独立验证仍有限，效果多基于用户报告（来源：矛盾与存疑）。
3. **个体差异巨大**：部分 ADHD 患者对声学刺激敏感，反而可能分心；而 LLM 的 function calling 在不同模型上表现差异显著。

## 今天就能试的行动

1. **区分“启动”与“维持”**：如果你任务启动困难，先用 Goblin Tools 或 Lex 分解任务；如果你经常中途走神，再试 Brain.fm 的“Focus”模式。
2. **设置 re-grounding 闹钟**：像孔子的“日省”一样，每 25 分钟设置一个提醒，问自己“当前任务是什么？”——这相当于 LLM 的 context refresh。
3. **给 LLM 加“考成法”**：在 agent 的 prompt 中加入“每步结束后报告状态”，类似张居正的考核制度，防止 LLM 陷入工具循环。
4. **记录工具使用日志**：用 Saner.AI 或简单笔记记录每次使用 Brain.fm 前后的专注时长，两周后分析是否真有改善——避免成为无意识的拐杖用户。

# 如果把 ADHD 大脑当成生成核心，Saner.AI 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Saner.AI 对 ADHD 的真正价值。

## 引言：一个让 ADHD 和工程师都窒息的场景

你打开电脑，准备写一份方案。突然想起邮件没回，切到邮箱；回邮件时又想到要查资料，打开浏览器；浏览时看到一篇感兴趣的文章，读完后完全忘了最初要干什么。这是 ADHD 的日常：工作记忆容量小，边做边忘，频繁的标签切换和搜索循环消耗着本就不多的认知燃料。

而在另一边的 LLM 工程里，同样的剧本在上演：你给 ChatGPT 写了一个复杂的 prompt，它完美执行了第一步，但第二步就忘了上下文。你不得不把历史对话重新粘贴进去，或者搭建一个 RAG 系统来给它“续命”。LLM 无跨会话状态，靠外部记忆/向量库才能连贯工作。

两个看似无关的困境，底层结构惊人地一致：**两者都是“高产但缺执行调度层的生成核心”**。ADHD 大脑能产生海量创意和联想，但缺少一个稳定的执行调度层来管理工作记忆、启动任务、感知时间；LLM 能生成流畅文本，但缺少内置的状态管理，上下文一长就“迷失”。

这篇文章要回答的核心问题是：如果把 ADHD 大脑当成生成核心，Saner.AI 能不能当它的无状态与外部记忆？答案是：**能，但前提是把它当作脚手架而非拐杖**。

## 同构：无状态与外部记忆

### ADHD 侧：工作记忆容量小，边做边忘

ADHD 的工作记忆并非容量小，而是控制弱——信息容易被无关刺激挤占（来源：工作记忆）。用户报告称，频繁的标签切换和搜索循环是主要痛点（来源：Saner.AI 工具页）。这就像一个没有持久化存储的生成模型，每次推理都从零开始。

### LLM/Agent 侧：无跨会话状态，靠外部记忆续命

LLM 本身是无状态的：每次对话都是独立推理，除非你手动提供历史。工程师通过向量数据库、RAG、memory 模块来构建外部记忆层，让 agent 能“记住”之前的交互。这正是 Saner.AI 等工具在做的事：通过本地记忆存储和快速检索，减少 ADHD 用户的搜索循环和标签切换（来源：Saner.AI 工具页）。

### 同构论证：脚手架 vs 拐杖

ADHD 大脑与 LLM 在“生成能力强、调度能力弱”上同构，因此解决方案也同构：**为生成核心套上外部执行功能层**。Saner.AI 相当于一个轻量级的向量数据库，但它只解决“记忆”问题，不解决“任务启动”和“时间感知”。Goblin Tools 通过任务分解降低启动门槛（来源：Goblin Tools 工具页），Lex 通过单一指令触发多步骤（来源：Lex 工具页），它们分别对应 LLM 的 prompt 工程和 workflow 编排。

关键判断：**Saner.AI 的价值在于提供“持久化上下文”，但它不能替代完整的执行功能层**。ADHD 需要的是一个工具生态，而非单一神器。

## 实战：今天就能试的 3 件事

1. **用 Saner.AI 替代你的笔记搜索**：下次需要找之前记下的信息时，不要翻文件夹或标签页，直接打开 Saner.AI 用自然语言搜索。观察搜索时间是否缩短。
2. **结合 Goblin Tools 进行任务分解**：将一个大任务（如“整理房间”）丢给 Goblin Tools，得到步骤列表后，把每个步骤作为 Saner.AI 的一条笔记，标注优先级。这相当于为你的生成核心加上“启动脚本”和“持久化存储”。
3. **设置“上下文刷新”提醒**：每工作 25 分钟，用 Saner.AI 快速记录当前进度和下一步计划。这模仿了 LLM 的“session 保存”机制，防止上下文丢失。

## 结语：脚手架，不是拐杖

Saner.AI 能否成为 ADHD 大脑的外部记忆？能，但前提是你不把它当作永久拐杖。真正的脚手架是促进能力发展的——当你习惯了用外部工具管理记忆，你的内在工作记忆控制能力可能会得到锻炼而非萎缩。关键在于有意识地使用：先用工具补偿，再逐渐内化策略。

ADHD 大脑和 LLM 一样，都需要一个聪明的 harness。Saner.AI 是其中一块拼图，但不是全部。

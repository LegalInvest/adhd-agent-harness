# 如果把 ADHD 大脑当成生成核心，Saner.AI 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Saner.AI 对 ADHD 的真正价值。

## 问题：当你边做边忘，大脑像一个无状态 API

你有没有这样的体验：打开浏览器想查个资料，结果半小时后发现自己刷完了三个视频、回了两条消息，却完全忘了最初要查什么？或者，你刚刚想到一个绝妙的主意，转身去拿笔，回来就再也想不起来了？

这不是你的错。ADHD 大脑的工作记忆容量天生偏小，处理复杂信息时容易超载，导致“边做边忘”成为日常（来源：认知负荷）。更本质地说，ADHD 大脑是一个强大的生成核心——创意、联想、跳跃性思维源源不断——但它缺乏一个可靠的执行调度层来管理上下文、记住“我刚刚在做什么”。

有趣的是，这恰恰是大型语言模型（LLM）的困境。LLM 本身也没有跨会话状态：每次对话都是独立的，它记不住你五分钟前说了什么，除非你把历史塞进上下文窗口。为了让 LLM 完成复杂任务，工程师们不得不搭建 Agentic Harness——用向量数据库、外部记忆、提示工程来给它“续命”。

所以，问题来了：**如果把 ADHD 大脑当成生成核心，Saner.AI 能不能当它的无状态与外部记忆？**

## 同构：两个“生成核心”共享同一个缺陷

先看证据。ADHD 大脑的生成核心（智力、创造力）与 LLM 的生成能力类似，但两者都缺乏执行功能的稳定调度（来源：ADHD 的 AI 工具全景）。对于 ADHD 患者，这意味着任务启动困难、工作记忆不足、时间盲；对于 LLM，这意味着无法自主规划、容易遗忘上下文、需要外部编排。

解决方案也同构：给生成核心套一个 harness——外部执行功能层。在 ADHD 侧，AI 工具通过自动决策、自动执行、简化环境来降低认知负荷（来源：认知负荷）。在 LLM 侧，Agent 框架通过任务分解、记忆检索、工具调用来实现复杂流程。

Saner.AI 正好落在这个交点上。它的核心功能是知识回忆（knowledge recall）和本地记忆（local memory），旨在减少搜索循环和标签切换（来源：Saner.AI）。对 ADHD 用户来说，这意味着你不需要记住“那个文件放在哪个文件夹”，Saner.AI 帮你记住；对你来说，它就像一个随身携带的外部工作记忆，随时可以问“我上次关于这个项目的笔记是什么”。

对 LLM 工程师来说，Saner.AI 的模式听起来很熟悉：它本质上是一个本地向量数据库 + 检索接口，为生成核心（ADHD 大脑）提供持久化状态。就像你在 Agent 里用 Chroma 或 Pinecone 存储对话历史一样，Saner.AI 存储你的思维碎片。

## 价值：从“记忆拐杖”到“执行脚手架”

但这里有一个关键判断：**Saner.AI 的真正价值不在于帮你记住，而在于帮你“不记住”**。当你知道信息可以被随时找回，你就不再需要占用工作记忆去维持它。这释放了认知带宽，让你可以把精力用在真正需要创造力的地方。

这与 Goblin Tools 的任务分解、Lex 的单指令触发多步骤一样，都是把认知负担外化给 AI（来源：认知负荷）。Goblin Tools 把“整理房间”变成“捡起衣服→擦桌子”的步骤，降低启动焦虑；Lex 让你一句话触发复杂流程，减少决策负担。Saner.AI 则处理另一个维度：信息检索的认知成本。

然而，必须诚实指出：这仍然是一个“脚手架”还是“拐杖”的问题。脚手架是可以逐步撤除的临时支撑，而拐杖是永久依赖。目前 Saner.AI 等工具设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。如果用户永远依赖外部记忆来回忆，内在的记忆组织能力可能永远不会被锻炼。这是一个真实的争议，目前没有答案。

## 局限：同构命题的证据缺口

虽然“ADHD 大脑与 LLM 同构”是一个诱人的框架，但证据主要来自概念类比和工具案例，缺乏大规模实证（来源：矛盾与存疑）。我们还没有看到随机对照试验证明，使用 Saner.AI 六个月后，ADHD 患者的工作记忆测试分数有提升。大部分证据来自用户反馈和评测博客（来源：Saner.AI）。同样，Agent 工程中的外部记忆方案虽然有效，但也不断面临上下文窗口限制、检索噪声等问题。

此外，Saner.AI 的独特卖点是知识回忆，但它无法覆盖所有 ADHD 症状，比如时间盲或情绪失调（来源：Saner.AI）。对于情绪波动，可能需要像 Goblin Tools 那样的情绪追踪工具来辅助（来源：情绪失调）。

## 今天就能试的行动

1. **把 Saner.AI 当成你的“第二大脑”**：每次有想法、读到重要信息，立刻丢进去。不要试图记住，只保留“我存过”的信任。一周后，回顾你的检索频率——你会发现你不再需要记那么多东西了。
2. **结合 Lex 或 Goblin Tools 做任务启动**：用 Saner.AI 存储项目背景，用 Lex 或 Goblin Tools 分解下一步行动。这样，你的“外部记忆”和“外部执行”就打通了。
3. **给自己设置一个“撤除实验”**：一个月后，尝试关掉 Saner.AI 一天，看看你能回忆起多少信息。如果完全无法工作，说明你已过度依赖；如果能部分回忆，说明脚手架正在起作用。
4. **如果你是工程师，试试把 Saner.AI 的思路用到你的 Agent 里**：给 LLM 挂一个本地向量数据库，每次对话前检索相关记忆。你会发现，它的“幻觉”减少了，回答更连贯了——就像你用了 Saner.AI 一样。

## 结论

Saner.AI 不是万能药，但它精准地匹配了 ADHD 大脑与 LLM 共享的缺陷：缺乏状态管理。作为外部记忆层，它能有效减轻工作记忆负担，让生成核心更专注于创造而非维护上下文。但我们必须记住，脚手架的目的是让你最终能自己走路，而不是永远依赖拐杖。在这一点上，工具设计者和用户都还有很长的路要走。

# 用 Claude 给 ADHD 补上ADHD 大脑与 LLM 的同构，是脚手架还是拐杖？

> Claude 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：你的大脑为什么总是“有想法，没行动”？

如果你是一个 ADHD 者，你一定熟悉这种体验：脑子里同时有十个绝妙的想法，但当你坐在电脑前准备执行时，却打开了五个无关的标签页，然后忘了自己要干什么。如果你是做 Agent 工程的开发者，你也会熟悉这种场景：LLM 能写出优雅的代码片段，但让它完成一个多步骤任务时，它要么跑偏，要么卡在循环里。

这两种困境其实共享同一个底层结构：**高产生成核心 + 缺失的执行调度层**。

## 同构：ADHD 大脑与 LLM 是同一类引擎

ADHD 大脑的生成核心——发散思维、联想跳跃、创意爆发——与 LLM 的文本生成能力高度相似。但两者都缺乏可靠的内置调度机制。ADHD 的执行功能缺陷（工作记忆不稳定、任务启动困难、注意力漂移）与 LLM 的无状态性、缺乏规划能力形成镜像（来源：[[ADHD 大脑与 LLM 的同构]]）。

这意味着，为 LLM 搭建 agent scaffolding（规划、任务分解、工具接口、验证循环）与为 ADHD 搭建外部执行功能层是同一类工程（来源：[[生成核心与调度层]]）。ADHD 需要的外部工具——自动任务分解、智能提醒、外部记忆——恰好对应 LLM 需要的 harness 组件。

## 真实人物的 harness：从毛泽东到张瑞敏

毛泽东是典型的 ADHD 特质者：小时候是问题儿童，爱读闲书不爱四书五经；一辈子读书不辍，行军路上也带着书；思维极度跳跃，文章充满比喻；冲动敢赌，四渡赤水、抗美援朝都是险棋。他的 harness 系统是：**批评与自我批评**（让别人提意见，对抗自我盲点）、**调查研究**（“没有调查就没有发言权”，用外部事实锚定决策）、**民主集中制**（用集体决策制衡个人冲动）。这对应到 LLM harness 中就是：**外部验证循环**（让工具检查输出）、**上下文 grounding**（用检索增强事实）、**多 agent 辩论**（用多个模型交叉验证）。

张瑞敏的“日清日结”工作法——每天的工作每天完成，每天反省每天进步——则对应 LLM harness 中的 **定时 re-grounding**：定期保存状态、检查进度、重新规划。他的“人单合一”让每个员工成为自己的 CEO，对应 LLM 中的 **模块化 agent**：每个子任务有独立的目标和反馈回路。

这些案例说明：无论是人还是模型，高产生成核心都需要外部系统来提供结构。

## 工具评测：哪些真正帮到了 ADHD？

当前 AI 工具已经尝试在多个维度充当外部执行功能层：

- **任务分解**：Goblin Tools 的 Magic ToDo 能自动将模糊任务（如“清理厨房”）分解为具体步骤，用户可调节“辣度”控制粒度（来源：[[Goblin Tools]]）。这直接降低了启动门槛，对应 LLM 中的 **任务规划器**。
- **外部记忆**：Saner.AI 通过本地记忆存储和快速检索，减少搜索循环和标签切换（来源：[[Saner.AI]]）。这对应 LLM 中的 **向量数据库**，解决无状态问题。
- **动态排程**：Motion 自动根据优先级、截止日期和可用时间重建日程，消除“下一步该做什么”的决策负担（来源：[[Motion]]）。这对应 LLM 中的 **调度器**。

但关键问题在于：这些工具是脚手架还是拐杖？

## 争议：脚手架 vs 拐杖的边界

“脚手架”是可拆卸的——它在你建造时提供支撑，但最终你要学会自己站立。“拐杖”是永久替代——没有它你就无法行走。

当前 AI 工具的风险在于：它们可能变成拐杖。例如，传统提醒系统要求用户自己设置提醒，但设置提醒本身就是一个执行功能任务（来源：[[重锚定与目标漂移]]）。如果 AI 替你做了一切规划，你可能会失去练习规划能力的机会。同样，如果 LLM agent 完全依赖外部调度器，一旦调度器失效，模型就无法自主行动。

另一个矛盾是：超聚焦是否应被引导而非打断？一方认为 AI 应提供“gentle structure”引导超聚焦，但部分用户反馈强行打断（如番茄钟）反而有效（来源：[[矛盾与存疑]]）。这提醒我们，个体差异极大，没有万能方案。

**我的判断**：脚手架 vs 拐杖的边界在于——用户是否保留对调度层的最终控制权。如果工具只是提供选项和提醒，决策权在你，那就是脚手架；如果工具自动执行且你无法干预，那就是拐杖。好的设计应该像毛泽东的“民主集中制”：集体讨论，个人拍板。

## 今天就能试的行动

1. **用 Goblin Tools 分解一件你拖延的任务**：输入“整理书桌”或“写周报”，调节辣度到你觉得“有点挑战但能完成”的程度，然后只做第一步。
2. **用 Motion 或类似工具替代手动日程规划**：设定好任务和截止日期，让 AI 自动排程，然后观察自己是否更容易启动。如果初始设置让你头疼，请人帮忙完成第一次配置。
3. **建立自己的“外部验证循环”**：像毛泽东的批评与自我批评那样，每周找一个人（或 AI）检查你的任务列表，问“这个优先级对吗？”。
4. **记录你的工具使用感受**：连续使用一周后，问自己：离开这个工具我能做同样的事吗？如果答案是“不能”，说明它可能是拐杖，需要调整使用方式。

## 局限与展望

当前证据主要来自概念类比和初步实证，缺乏大规模随机对照试验证明同构假设或 AI 工具的临床有效性（来源：[[矛盾与存疑]]）。多巴胺检测等神经生物学方法尚处早期。长期依赖风险也缺乏纵向研究。

但即便如此，同构视角已经为设计 AI 辅助工具提供了有价值的理论框架。它告诉我们：**不要试图修复生成核心，而是给它一个可拆卸的 harness**。这个原则既适用于 ADHD 者，也适用于 Agent 工程师。

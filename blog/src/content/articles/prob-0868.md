# Claude 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

## 问题：你的大脑是一座高产但失控的生成工厂

如果你有 ADHD，你一定熟悉这种感觉：脑子里同时冒出十个好点子，身体却瘫在沙发上刷手机。任务摆在面前，你知道该做，但“启动”就像推一堵墙。更糟的是，一旦开始，你又会因为不会拆解而卡在细节里，最后草草收场或彻底放弃。

与此同时，在 AI 工程界，开发者正为同样的问题头疼：LLM 能写出漂亮的论文、生成复杂的代码，但让它完成一个“去超市买牛奶并顺便交电费”的多步任务，它要么忘记步骤，要么顺序错乱。于是工程师们发明了“planner-executor 循环”——让 agent 先规划子任务，再逐步执行，每一步都检查进度，必要时重新规划。

这两群人面对的，其实是同一个敌人：**一个高产但缺乏调度层的生成核心**。

## 同构：ADHD 大脑与 LLM 共享的结构缺陷

ADHD 的神经科学告诉我们：执行功能缺陷（尤其是工作记忆和任务启动困难）是核心问题（来源：AI and ADHD: Comprehensive Guide to Using AI Chatbots for People with ADHD）。你的大脑像一台没有操作系统的超级计算机——CPU 强劲，但没有进程调度器，导致任务排队混乱、死锁频繁。

LLM 也一样。纯粹的大模型是“无状态”的：它无法记住刚才说了什么，也无法自主决定下一步做什么。要让 agent 完成长任务，必须给它一个 planner 模块来分解任务，一个 executor 模块来逐步执行，外加一个 memory 模块来记录上下文。这就是 agentic harness 的典型架构。

**关键洞察**：ADHD 大脑与 LLM 在结构上同构——两者都是高产的生成核心，但缺乏可靠的执行调度层（来源：AI 与 ADHD 的时间管理主题综述）。因此，AI 帮助 ADHD 进行时间管理的本质，不是“让 AI 替你思考”，而是“给生成核心套上 harness”——通过外部系统提供记忆、调度和约束，补偿执行功能缺陷。

## 工具实测：Claude 到底补上了哪一层？

让我们用这个框架来审视当前最火的 ADHD 时间管理 AI 工具。

**Motion** 和 **Reclaim.ai** 是典型的“planner”工具。Motion 自动根据任务优先级和截止日期生成每日计划，并在被打断时动态调整（来源：Motion 工具页）。Reclaim.ai 则更强调“保护”——它创建深度工作块，自动防御会议侵占（来源：Reclaim.ai 工具页）。两者都直接降低了“下一步该做什么”的决策负担，从而减轻认知负荷（来源：认知负荷概念页）。

但注意：Motion 需要你输入所有任务和截止日期，初始设置本身就需要执行功能（来源：Motion 局限与争议）。如果你连“列出今天要做的事”都做不到，Motion 反而会成为另一层负担。

**Tiimo** 则针对“时间盲”——它把时间变成可见的色块和惯例，让你直观看到一天的结构（来源：Tiimo 工具页）。这相当于给 LLM 一个可视化的上下文窗口，帮助维持任务连贯性。

**Goblin Tools** 和 **Saner.AI** 走的是另一条路：前者帮你把“写论文”分解成“打开电脑→新建文档→写第一段”等微步骤（来源：认知负荷概念页），后者则侧重情绪追踪和认知重构（来源：Saner.AI 在情绪失调页提及）。它们更像是一个“executor 辅助器”——帮你把大任务拆成可执行的小块，并监控你的状态。

**我的判断**：这些工具中，真正补上“调度层”的，是那些能**自动规划并动态调整**的工具（如 Motion、Reclaim.ai）。它们充当了 ADHD 大脑缺失的 planner 模块。而像 Goblin Tools 这样的任务分解工具，则更像是 executor 模块的辅助——它不替你规划，但帮你把规划落地。

但这里有一个陷阱：**工具到底是脚手架，还是拐杖？**

## 争议：脚手架 vs 拐杖的边界

“脚手架”是临时的、可撤除的支撑，目的是帮你最终独立行走。“拐杖”则是长期依赖，撤除后你会摔倒。

当前多数工具（如 Motion、Goblin Tools）设计为长期使用，且未提及撤除机制（来源：矛盾与存疑中“过度依赖 vs 脚手架可撤除性”）。这意味着它们更接近拐杖。长期依赖外部调度层，可能会让你的内在执行功能进一步萎缩——就像长期用导航软件会削弱方向感一样。

另一个争议是多巴胺干预的有效性。有些工具声称通过即时反馈和奖励调节多巴胺，但证据尚弱且存在争议（来源：矛盾与存疑中“多巴胺干预的有效性”）。如果工具只是通过不断推送通知和成就徽章来刺激多巴胺，它可能反而加剧 ADHD 的注意涣散和成瘾倾向。

**核心观点**：Claude 或任何 AI 工具，只有当它被用作“可逐步撤除的脚手架”时，才对 ADHD 有长期益处。这意味着：工具应该帮助你学会如何规划，而不是替你规划；应该记录你的成功策略，以便你日后复用；应该在你执行功能改善后，逐渐减少干预。

## 今天就能试的行动

1. **用 Claude 做一次“planner 练习”**：告诉它你明天要完成的一个复杂任务（比如“准备会议报告”），让它分解成 5 个可执行的子任务，并估算每个子任务的时间。然后你自己手动调整这些子任务，把它变成你的计划。关键：不要让它替你执行，而是让它教你分解。

2. **设置“反依赖”提醒**：在使用 Motion 或 Reclaim.ai 时，每周抽一天关闭所有自动调度，用手写或简单待办列表来规划。这能训练你的执行功能，防止工具成为拐杖。

3. **用 Tiimo 建立视觉时间盒**：如果你经常“时间盲”，试试用 Tiimo 把一天分成 3-4 个视觉时间段（如“上午深度工作”“下午沟通”“晚上复盘”）。不要细化到分钟，而是用色块建立节奏。

4. **记录你的“启动模式”**：下次你成功启动一项任务时，用 Claude 或笔记记录下“是什么让你启动了？”——是定时器？是环境切换？还是别人的一句话？这些是你的个人 planner 线索，可以逐渐内化。

## 诚实局限

本文的论证基于同构理论模型，尚缺乏大规模实证支持（来源：矛盾与存疑中“ADHD 大脑与 LLM 同构命题的证据”）。不同 ADHD 亚型对工具的反应可能不同，且现有工具的 AI 功能细节多不透明（来源：矛盾与存疑中“证据质量”）。最重要的是，没有任何工具能替代专业的医疗和行为干预。

但如果你能记住一句话：**AI 工具的价值不在于替你做事，而在于帮你学会如何做事**。那么，无论是 Claude 还是 Motion，都能成为你大脑的脚手架——而不是另一根拐杖。

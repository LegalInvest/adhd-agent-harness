# Claude 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

## 问题：为什么 Claude 让 ADHD 更焦虑？

你打开 Claude，说“帮我规划明天的工作”。它立刻输出一份漂亮的清单：9:00-10:00 写报告，10:00-11:00 回复邮件……你盯着屏幕，却更不想动了。为什么？因为 Claude 替你做了“规划”，但没替你“启动”。它给了你答案，却没给你执行它的能力。

对 ADHD 大脑而言，任务启动困难的根源不是“不知道做什么”，而是“知道做什么却做不到”——执行功能失调让“知道”与“做到”之间横亘着一道认知鸿沟（来源：AI 与 ADHD 的时间管理）。Claude 的直接回答模式，恰恰跳过了最关键的环节：**将意图分解为可执行的、有顺序的、低认知负荷的子步骤**。

## 同构：ADHD 大脑与 LLM 共享同一个缺陷

最新研究用经典 Stroop 冲突任务测试 Transformer 注意力，发现一个惊人事实：短上下文中模型表现正常，但随序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突（来源：Deficient Executive Control in Transformer Attention）。这正是 ADHD 执行功能缺陷的核心标志：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃。

ADHD 大脑与 LLM 是同一类“高产但缺乏可靠执行调度层的生成核心”（来源：ADHD 大脑与 LLM 的同构）。LLM 在长上下文中的“注意力崩溃”，对应 ADHD 在工作记忆过载时的“执行功能崩溃”。两者都需要外部脚手架来补偿——对 LLM 是 planner-executor 循环，对 ADHD 是 AI 工具作为外部执行功能层。

## 错用：把 Claude 当“答案机”而非“分解器”

多数人用 Claude 的方式，相当于给一个执行功能缺陷的大脑塞进更多信息。你问“如何完成这个项目”，它给出 10 条建议——这反而增加了工作记忆负荷。执行功能教练 Yulia Rafailova 指出：“AI 提供了一个不可思议的工具来外化我们的思维，作为一个支架，能显著减轻我们工作记忆上的认知负荷。”（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD）

但“支架”的前提是**分解**，而非**填充**。正确的用法是让 Claude 充当 planner-executor 循环中的“planner”——将复杂任务拆解为原子步骤，每个步骤的认知负荷低到足以让 ADHD 大脑直接启动。这与 LLM agent 工程中的任务分解技术完全同构：通过将复杂任务分解为子任务，减轻 LLM 的认知负荷，减少幻觉和错误（来源：Deep Dive into Agent Task Decomposition Techniques）。

## 正确用法：让 Claude 做你的“执行调度层”

具体怎么做？以下是我实测有效的模式：

**1. 任务分解 prompt**：不要说“帮我规划明天”，而要说“把‘写季度报告’拆解成 5 个我能在 15 分钟内完成的子任务，每个子任务只包含一个动作”。这对应 LLM agent 中的“分解-执行”循环——planner 输出子任务，executor 逐个完成。

**2. 时间盲补偿**：ADHD 难以感知时间流逝（来源：时间盲）。让 Claude 为每个子任务添加时间预估，并用“番茄钟”格式输出。例如：“子任务 1：打开文档，写下标题（5 分钟）”。这类似 Tiimo 将时间转化为可见元素（来源：Tiimo），或 Reclaim.ai 自动保护时间块（来源：Reclaim.ai）。

**3. 启动触发器**：在分解后的每个子任务前加一句“现在站起来，走到电脑前，按下电源键”——把抽象意图转化为物理动作。这直接针对任务启动困难的神经机制：降低启动门槛，绕过执行功能失调。

**4. 动态调整**：告诉 Claude“如果我在 25 分钟内没完成子任务 2，就自动重新安排后续步骤”。这模仿 Motion 的动态日程调整（来源：Motion），也对应 agent 的 planner 在 executor 失败时重新规划。

## 脚手架 vs 拐杖：边界在哪里？

这里有一个诚实警告。AI 工具可能成为“永久拐杖”，削弱内在时间感知和规划能力（来源：矛盾与存疑）。关键在于：**脚手架是暂时支撑，拐杖是永久替代**。如果你用 Claude 分解任务后，仍然无法启动——那说明需要更基础的执行功能训练，而非更复杂的 AI 工具。

另一个局限：ADHD 亚型（注意力缺陷 vs. 多动冲动）对工具响应不同，现有工具多面向注意力缺陷型（来源：AI 与 ADHD 的时间管理）。如果你属于多动冲动型，可能需要结合情绪调节工具（如 Saner.AI 的“情绪调节”功能）或身体在场效应（如 Goblin Tools 的“任务分解”功能）。

## 今天就能试的行动

1. **改造你的 prompt**：下次用 Claude 时，不要说“帮我规划”，而说“把 [任务] 拆解成 3 个我能在 10 分钟内完成的子任务，每个子任务只包含一个动作，并给出启动指令”。
2. **设置时间锚点**：让 Claude 为每个子任务加上“预计时间”和“完成标记”，然后手动在日历上画时间块（类似 Tiimo 的视觉化）。
3. **一次只做一件事**：分解后的子任务列表，每次只看第一个。用物理遮挡（如一张纸）盖住其余步骤，直到完成当前步骤。
4. **记录崩溃模式**：如果 Claude 的分解仍然无法启动，记录下是哪个环节卡住——是步骤太抽象？时间预估不准？还是缺少多巴胺奖励？然后针对性调整 prompt。

Claude 不是万能药，但当它被正确用作“执行调度层”时，它补上的正是 ADHD 大脑与 LLM 共同缺失的那一层——**将意图转化为行动的分解引擎**。别再把它当答案机了，让它做你的 planner-executor。

# Claude 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Claude：它到底补上了哪一层执行功能？

你打开 Claude，输入一个模糊的念头：“帮我规划下个月的产品发布。”它立刻吐出漂亮的甘特图，步骤清晰，时间合理。你松了口气，觉得“终于有救了”。然后呢？第二天你打开文档，看着那张图，依然不知道从哪里开始。你关掉它，去刷短视频了。

这不是 Claude 的错。是你用错了。

## 问题：为什么“AI 帮我规划”经常变成“AI 帮我拖延”？

ADHD 的大脑和 LLM 在结构上同构：两者都是高产的生成核心，但都缺乏可靠的内置调度层（来源：ADHD 大脑与 LLM 的同构）。你的大脑能在一小时里想出 20 个创业点子，却没法决定先吃早饭还是先回邮件；Claude 能写出完美的项目计划，却没法在执行中动态调整优先级、记住上下文、抑制无关联想。

所以，当你让 Claude 直接输出一个“完整计划”时，你得到的只是一个静态的、线性的、没有任何执行监控的文档。它没有解决你的核心问题——**任务启动困难**和**不会拆解**。你需要的不是一份计划，而是一个**外部调度器**，像 agent 的 planner-executor 循环一样，把长任务分解成可执行的微步骤，并在每一步提供提醒和反馈。

## 同构解法：从“计划”到“循环”

看看 2500 年前的孔子。他精力过剩、冲动、思维跳跃，《论语》全是场景化语录，没有系统著作。他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省，“吾日三省吾身”。这本质上是一个**外部调度器**：礼 = 行为模板（planner），反省 = 执行监控（executor）。70 岁才做到“从心所欲不逾矩”，说明他花了一辈子驯化自己的生成核心。

再看晚清的胡林翼。年轻时是花花公子，后来每天写日记反省，和曾国藩一起“做圣人”。他的日记就是**外部记忆**和**执行反馈**，对应 LLM agent 的“日志系统”和“状态保持”。

这两套 harness 的共同点：**它们不是一次性计划，而是持续的循环**。planner 设定规则，executor 执行并记录，然后反馈给 planner 调整。这正是 agent 工程里“planner-executor 循环”的精髓。

## 工具错配：为什么 Motion、Reclaim.ai、Tiimo 还不够？

当前主流 AI 时间管理工具，如 Motion（自动排程）、Reclaim.ai（保护时间块）、Tiimo（视觉化时间线），都试图解决“时间盲”和“任务启动”问题（来源：Motion、Reclaim.ai、Tiimo）。它们通过外部界面把“现在该做什么、做多久、何时切换”显式化，确实有效。

但它们的核心假设是：**用户已经知道任务怎么分解**。Motion 可以自动排程，但它不会告诉你“写报告”应该拆成“列大纲→找数据→写第一段→写第二段”。Tiimo 能视觉化日程，但如果你连“今天要做什么”都列不出来，它只是一张漂亮的空表。

这就像给 agent 一个 planner 却不让它调用 executor——计划完美，但从不执行。

## 真正的缺口：任务分解 + 执行监控

ADHD 的执行功能障碍核心在于**不会拆解**和**无法启动**（来源：执行功能障碍）。LLM agent 的对应问题是：长任务规划容易“幻觉”，执行中容易丢失上下文。

所以，一个有效的 AI harness 必须同时提供：
1. **任务分解器**：把“写书”拆成“写第一章大纲→写第一章第一节→写第一章第二节……”直到 15 分钟能完成的微任务。
2. **执行监控器**：每完成一个微任务，记录进度，调整剩余计划，并给出下一步提示。

这正好对应 agent 的“plan-and-execute”模式：planner 输出步骤，executor 执行并返回状态，planner 再更新计划。

## 诚实说：它可能只是拐杖

但这里有个争议：过度依赖外部调度器，会不会让 ADHD 的执行功能进一步退化？（来源：矛盾与存疑）目前缺乏纵向研究，无法定论。我的判断是：**关键在于脚手架 vs 拐杖的边界**。如果你用 AI 分解任务后，慢慢学会了拆解的方法，它就是脚手架；如果你永远依赖 AI 告诉你“下一步做什么”，它就是拐杖。

另一个局限：任务分解器需要用户输入足够清晰的目标，而 ADHD 的“模糊目标”本身就是问题。工具不能替代你“想清楚要什么”。

## 今天就能试的行动

1. **用 Claude 做“微任务分解”**：输入一个大任务，要求它输出“每个步骤耗时不超过 15 分钟，并且标出第一个步骤的启动动作”。比如：“我要写一篇 2000 字的文章，请分解成 15 分钟能完成的步骤，并告诉我第一步具体该做什么。”

2. **设置执行监控循环**：完成第一步后，回到 Claude，告诉它“我完成了第一步，现在更新剩余计划”。让 AI 像 agent executor 一样持续反馈。

3. **模仿孔子的“日课”**：每天固定时间（比如睡前）用 AI 回顾当天完成情况，调整明天的计划。这相当于 agent 的“日志回放”和“计划更新”。

4. **尝试 Tiimo 的视觉化时间块**：如果你对时间感知差，用 Tiimo 把时间变成可见的色块，至少能缓解“时间盲”（来源：Tiimo）。但记住，它只是视觉辅助，不能替代任务分解。

## 结论

Claude 之于 ADHD，就像 planner-executor 循环之于 LLM agent——它应该是一个持续的、交互的调度器，而不是一次性的计划生成器。别再让它写完美计划了，让它帮你分解、执行、反馈、再分解。这才是同构的解法。

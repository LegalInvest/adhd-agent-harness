# 用 Goblin Tools 给 ADHD 补上重锚定与目标漂移，是脚手架还是拐杖？

> Goblin Tools 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 引言：两个平行世界，同一种漂移

如果你是 ADHD，你一定熟悉这种场景：明明想写一份报告，回过神来却刷了四十分钟科普视频；或者反过来，一头扎进一个细节里调了三个小时颜色，报告的核心论点还没动笔。如果你是 Agent 工程师，这一幕也绝不陌生：长程任务里，agent 在第七轮对话后已经忘了最初的 system prompt，或者为了完成某个子目标，把主目标「优化」得面目全非。

这两件事本质是同一个问题：**生成核心很猛，执行调度层很弱**。ADHD 大脑不缺想法、不缺能量，缺的是把想法锚定在目标上的稳定结构；LLM/agent 不缺推理能力，缺的是长程一致性、验证循环和外部记忆。两边的解法，都叫 harness。（来源：AI 与 ADHD 的专注力）

## 同构诊断：为什么 ADHD 大脑像 LLM

从同构视角看，ADHD 与 LLM 共享三种核心缺陷：

- **无状态与外部记忆**：ADHD 工作记忆不稳定，LLM 无跨会话状态，两者都需要外部记忆系统。RescueTime 的自动时间追踪、Forest 的专注记录、agent 的 memory store，做的是同一件事：把内部状态外化，让目标不被当下情绪或上下文吞掉。（来源：AI 与 ADHD 的专注力）
- **上下文工程**：ADHD 容易被环境带偏，LLM 受上下文膨胀影响推理退化，两边都需要主动设计「当下注意什么」。Brain.fm、Endel、Focusmate 提供的不是意志力，而是经过设计的感官与社交上下文，相当于给人脑写 system prompt。（来源：Brain.fm；Focusmate）
- **采样温度波动**：ADHD 的表现日内波动极大，而 LLM 的输出受 temperature 影响随机。两边都需要外部约束来稳定输出——日程、提醒、环境设计对人；确定性工作流、状态机、工具契约对 agent。（来源：采样温度与表现波动）

/wiki 资料也提醒我们：这种同构目前仍是「理论类比」，并非经过验证的神经机制事实，不能无限推广。（来源：全局矛盾与存疑）

## 重锚定：曾国藩的「日课十二条」就是最早的 system prompt

要理解重锚定，最贴切的人物案例是曾国藩。三十岁前的他，几乎是教科书式 ADHD：坐不住、爱串门、看书「他人目下二三行，余或疾读不能终一行」、情绪极端，兵败就想跳水自杀。他的 harness 系统是「日课十二条」：黎明即起、读书不二、谨言、写日记反省，打仗则「结硬寨打呆仗」。

这套系统的核心不是道德自律，而是**把目标从不断漂移的大脑里拽出来，钉在外部工件上**。日记是外部记忆，日课是确定性控制流，「结硬寨」是防止冲动冒进的执行约束。放到 LLM/agent 工程里，这对应的是：定时 re-grounding（重新读取 system prompt）、规划工件（planning artifact）、执行验证循环、以及把复杂任务拆成单向状态机。（来源：幻觉与验证循环）

曾国藩的日记一辈子在骂自己，但骂的不是「我不行」，而是「目标又漂移了」。这正是 harness 的精髓：承认生成核心不可靠，用外部结构把它拉回轨道。

## 目标漂移：为什么 Goblin Tools 是脚手架，也可能变成拐杖

Goblin Tools 进入 ADHD 工作流，最关键的贡献是**把抽象目标翻译成可执行的下一步**，并把「下一步」以低摩擦的方式反复推到用户面前。这和 agent harness 里的「结构化工作流与规划」「结果验证与迭代」是同构的：不是让模型/用户自己想，而是先把目标写成 plan，再在每一个节点做 ground-truth 校验。（来源：幻觉与验证循环）

然而，这里有一个明确的边界：

- **脚手架**：工具帮你记忆、分解、提醒、阻断分心，但你仍然保留目标设定与价值判断。Goblin Tools 的 re-grounding 只是把你之前写下的目标，在合适的时候重新抛回来，让你自己重新确认。这和 RescueTime 的自动追踪、Focusmate 的身体在场一样，属于外部执行功能层，辅助但不替代决策。（来源：RescueTime；Focusmate）
- **拐杖**：如果你开始把「我要什么」也交给工具决定，或者没有工具就完全不启动任务，那么工具就替代了前额叶。这正是 wiki 资料中「依赖 vs. 内化」的争议：AI 工具是否会导致用户永久依赖外部脚手架，而无法内化执行功能？目前没有长期追踪研究。（来源：全局矛盾与存疑）

**我的判断**：Goblin Tools 作为 re-grounding 和反漂移工具，是脚手架还是拐杖，不取决于工具本身，而取决于用户有没有保留「目标设定权」和「最终验证权」。这和 LLM 领域的人类反馈强化学习（RLHF）逻辑一致：人类不替模型生成，但保留对齐评判。

## 证据与诚实局限

给 ADHD 侧：Brain.fm、Focusmate、RescueTime、Endel、Forest 等工具确实被多项用户报告和小型评选纳入推荐列表，Brain.fm 在 2026 年 44 款 ADHD 应用测试中进入最佳 9 款。（来源：Brain.fm）Focusmate 的虚拟身体在场也基于已被研究的身体在场效应。（来源：Focusmate）

给 LLM/agent 侧：harness 工程明确强调「验证与 CI 集成」「结构化工作流和规划」「自我批评提示」以及「计划-执行」分离，以防止长程任务中的目标漂移和幻觉。（来源：幻觉与验证循环；采样温度与表现波动）

但两边证据都不完整：ADHD 工具有 ADHD 特异性独立随机对照试验的极少；LLM 的同构性仍是类比，不是神经科学事实；个体差异和工具负担问题也未被充分研究。（来源：全局矛盾与存疑）所以本文不是在说「用 Goblin Tools 就能根治 ADHD/Agent 漂移」，而是在说：在现有证据下，**把它作为外部执行调度层使用，是最合理的策略位置**。

## 今天就能试的 4 条行动

1. **写一条不超过 50 字的「主目标」system prompt**：每天早上把它放在屏幕顶部或设为壁纸。这对应 agent 的 system prompt，是 re-grounding 的锚点。
2. **用工具做外部验证循环，而非外部思考**：让 Goblin Tools、RescueTime 或 Forest 负责记录、提醒、分解，但每条任务完成时，必须自己回答「这件事离主目标更近了吗？」
3. **设置「漂移检查点」**：长程任务每 30 分钟或在完成一个子任务后，重新朗读最初目标。Agent 工程师会把这叫做 checkpoint-based re-grounding。
4. **每周做一次「拐杖检查」**：如果某天没有工具就完全无法启动，把工具关掉 2 小时，手动执行一次最小任务，确认内化进度。

## 结语

ADHD 不是「想法太少」，而是想法太多太散；LLM 不是「能力不足」，而是长程一致性和现实校验不足。两边的解法都不是压制生成核心，而是给它加一层 harness。Goblin Tools 可以是脚手架，前提是：目标由你定，工具帮你记得；判断由你做，工具帮你验证。把这一点想清楚，工程师和用户其实都在做同一件事——**让生成核心高产，但不再跑偏**。

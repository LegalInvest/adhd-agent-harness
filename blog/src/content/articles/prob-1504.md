# Motion 之于 ADHD，就像 会褪去的脚手架 之于 LLM——但有人用错了

> 从同构视角实测 Motion：它到底补上了哪一层执行功能？

## 问题：当工具成为新的负担

“Planners and sticky notes worked for a week before falling apart.”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）这句话道出了无数 ADHD 患者的痛：不是不想规划，而是传统规划系统“often create more friction than flow”（来源：ADHD Productivity Hack: Plan 2025 Using AI (Step-by-Step)）。于是我们转向 AI 工具，期待它们能填补执行功能的缺口。但新的恐惧随之而来：怕过度依赖工具，又怕没有支撑就崩。

这种矛盾在 Motion 上尤其突出。Motion 被宣传为“自适应规划器”——能“reshuffling the day when it collapses”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。听起来完美：你只需输入任务，AI 自动排程、动态调整。但实际使用中，很多人发现 Motion 反而成了新的压力源：它要求你准确估计任务时长、严格遵守时间块、处理冲突时手动干预——这些恰恰是 ADHD 大脑最不擅长的执行功能。

问题出在哪里？我们错误地将 Motion 当作“永久拐杖”，而它本该是“会褪去的脚手架”。

## 同构：ADHD 大脑与 LLM 的调度困境

ADHD 大脑与 LLM 在架构上是同构的：两者都是“高产但缺乏可靠执行调度层的生成核心”（来源：AI 与 ADHD 的情绪调节）。ADHD 大脑能产生海量创意和想法，但工作记忆容量有限、时间盲导致无法客观评估时间、任务启动困难——这些执行功能缺陷就像 LLM 缺乏上下文工程和工具调用接口，导致生成结果失控。

LLM 的解法是 harness（脚手架）：通过 function calling 将复杂任务拆解为工具调用，通过 context engineering 设计注意焦点，通过 planning artifacts 实现多步推理。Harness 的本质是“可褪去的”——它补偿调度缺陷，但不替代生成核心。现代 agent 系统通过“组合多个模型、检索器和工具”来达成 SOTA 结果（来源：Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned），而非依赖单一模型调用。

ADHD 的解法同理：AI 工具应作为外部执行功能层，补偿工作记忆、时间盲、任务启动等缺陷，但不替代内在的情绪调节和决策能力。Motion 的定位应是“调度层”，而非“决策层”。

## Motion 的真相：它补上了哪一层？

Motion 的核心价值在于**动态时间调度**。它不像传统规划器那样要求你预先完美规划，而是根据实际进度自动调整。这直接针对 ADHD 的“时间盲”——“Traditional planners never solved time blindness.”（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。Motion 通过视觉化时间块和自动重排，帮助用户客观化时间感知。

但 Motion 不解决任务分解问题。它假设你已经知道“做什么”和“怎么做”。这正是它与 Goblin Tools 的互补之处：Goblin Tools 的 Magic ToDo 功能自动将模糊任务分解为小步骤（来源：Harnessing Artificial Intelligence to Live Better with ADHD - CHADD），而 Motion 负责将这些步骤排入时间线。

同样，Motion 不直接训练工作记忆——那是 Inflow 的领域。Inflow 通过个性化训练强化背外侧前额叶皮层功能（来源：Dynamic causal brain circuits during working memory and their functional controllability），间接提升注意力维持。Motion 也不提供知识回忆功能——那是 Saner.AI 的专长，通过本地记忆减少搜索循环（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar）。

所以，Motion 补上的只是**调度层**。它让 ADHD 大脑的生成核心（创意、决策）得以在可靠的时间框架内运作，减少因时间失控引发的情绪波动。这类似于 LLM agent 的 harness 提供上下文窗口管理和工具调用顺序，让生成核心专注于内容产出。

## 脚手架 vs 拐杖：边界在哪里？

关键区分在于：工具是否在训练你离开它。

拐杖是永久替代——你依赖它才能行走，一旦失去就瘫痪。脚手架是临时辅助——它在你施工时提供支撑，完工后可以拆除。

对于 ADHD，Motion 成为拐杖的表现是：你不再尝试自己估计时间，完全交给 AI 排程；任务冲突时不会手动调整，而是焦虑等待 AI 重排；离开 Motion 后时间感知能力反而下降。这对应 wiki 中警告的“过度依赖可能削弱内在时间感知能力”（来源：矛盾与存疑）。

Motion 作为脚手架的表现是：你通过它的时间块视觉化逐渐学会客观估计时长；你观察它如何分解日程，内化为自己可复用的模式；当某天不用 Motion 时，你仍能大致保持时间感知。这类似于 LLM agent 的 harness 设计原则——“工具即用户界面”，注重“agent UX——简单、无认知开销”（来源：工具使用与认知卸载）。

## 争议与局限

必须诚实指出：Motion 对 ADHD 的效果证据主要来自用户报告，缺乏独立随机对照试验（来源：矛盾与存疑）。个体差异也很大——注意缺陷型和多动冲动型用户对 Motion 的反应可能截然不同。此外，Motion 的 AI 功能细节尚不明确（类似 Structured 的问题），用户可能对其能力产生不切实际的期望。

更根本的争议是：AI 工具作为外部执行功能层，是否真的能避免削弱内在能力？现有研究“缺乏如何避免依赖的具体指导”（来源：矛盾与存疑）。这意味着，即使我们正确使用 Motion 作为脚手架，长期效果仍不确定。

## 今天就能试的行动

1. **只输入“做什么”，不输入“怎么做”**：在 Motion 中只添加任务标题和截止时间，让 AI 自动排程。任务分解交给 Goblin Tools 或手动写在备注中。

2. **每周一天“无 Motion 日”**：选择一天完全不用 Motion，仅凭直觉安排时间。记录你的时间感知准确度，对比有 Motion 时的表现。这既是训练，也是检验依赖程度。

3. **将 Motion 与 Inflow 结合**：每天用 Inflow 做 10 分钟工作记忆训练（来源：Dynamic causal brain circuits during working memory and their functional controllability），再用 Motion 规划当天任务。前者提升内在能力，后者提供外部支撑。

4. **观察并记录“冲突处理”模式**：当 Motion 重排日程时，留意你的情绪反应。是焦虑？还是释然？这能反映你是在使用脚手架还是依赖拐杖。

Motion 不是万能药，它只是一个调度层。正确的使用方式是：把它当作会褪去的脚手架，在它支撑的同时，悄悄训练自己离开它的能力。

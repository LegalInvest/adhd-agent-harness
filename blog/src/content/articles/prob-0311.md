# 用 ChatGPT 给 ADHD 补上上下文工程，是脚手架还是拐杖？

> ChatGPT 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：你的注意力被环境带偏了吗？

你是否曾在打开文档后，被一条通知拽走，半小时后发现自己正在刷短视频？或者，你写的 prompt 明明很清晰，ChatGPT 却越聊越偏，最后输出一堆无关内容？

这两个场景共享同一个底层问题：**上下文失控**。ADHD 大脑和 LLM 都是高产的生成核心，却缺乏可靠的执行调度层——注意力被环境带偏，输出就变得随机。

## 同构：ADHD 大脑与 LLM 是同一类产物

ADHD 大脑与 LLM 在结构上同构：两者都是**高产但缺乏可靠执行调度层的生成核心**（来源：AI 与 ADHD 的专注力）。ADHD 的创意发散与 LLM 的文本生成能力都很强，但缺乏上下文控制——ADHD 易被环境带偏，LLM 易被上下文窗口膨胀干扰。两者都需要主动设计“当下注意什么”的工程方案，即**上下文工程**。

## 脚手架：ChatGPT 如何补上上下文工程

ChatGPT 作为外部执行功能层，可以补偿 ADHD 的执行功能缺陷（来源：AI 与 ADHD 的专注力）。具体方式包括：

- **任务启动**：用 prompt 将大任务分解为微步骤，降低启动门槛。例如：“请将‘写周报’分解为 5 个可执行的小任务，每个任务一句话。”
- **时间感知**：结合 RescueTime 的时间追踪，ChatGPT 可以生成时间使用报告，帮助对抗时间盲（来源：RescueTime）。
- **注意力维持**：Focusmate 通过虚拟身体在场提供实时问责，ChatGPT 可以模拟类似角色，定时询问进度（来源：Focusmate）。
- **情绪调节**：当情绪失调导致无法工作时，ChatGPT 可以引导用户写出情绪，然后结构化地分析触发因素和应对策略（来源：情绪失调）。

这些工具的本质是给 ADHD 大脑套上一个 **harness**——外部结构来稳定输出。

## 人物案例：曾国藩的日课与 LLM 的 prompt 工程

曾国藩是典型注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书慢到“他人目下二三行，余或疾读不能终一行”。他的 harness 是**日课十二条**：黎明即起、读书不二、谨言、写日记反省等。这些固定仪式相当于**定时 re-grounding**，每天多次将注意力拉回核心轨道。

这与 LLM 的 prompt 工程同构：**日课 = 定时 re-grounding 指令**，**日记 = 上下文日志**，**读书不二 = 单任务 prompt**。曾国藩的成就（平定太平天国、发起洋务运动）证明，外部结构可以内化为稳定的输出系统。

## 拐杖还是脚手架？边界在哪里

关键在于：工具是在**补偿**还是**替代**你的执行功能？

- **脚手架**：你主动使用工具来训练自己的调度能力。例如，用 ChatGPT 分解任务后，你逐渐学会自己分解；用 Focusmate 一段时间后，你开始内化问责感。
- **拐杖**：你依赖工具完成一切，一旦工具不可用，就完全无法启动。例如，没有 ChatGPT 就无法动笔写任何东西；没有 Focusmate 就整天拖延。

目前缺乏长期研究来明确边界（来源：矛盾与存疑）。但一个可行的判断标准是：**你是否在减少工具使用频率后，仍能保持同等效率？** 如果是，它是脚手架；如果不是，它正在成为拐杖。

## 局限与争议

- **个体差异巨大**：ADHD 异质性使得同一工具对不同亚型效果迥异（来源：AI 与 ADHD 的专注力）。
- **证据质量有限**：多数工具仅有用户报告或小样本研究，缺乏大规模随机对照试验（来源：AI 与 ADHD 的专注力）。
- **同构的局限性**：ADHD 大脑与 LLM 的同构是隐喻而非严格等价，LLM 的“注意力机制”与人类注意力存在本质差异（来源：AI 与 ADHD 的专注力）。
- **Brain.fm 等工具**虽被列入最佳 ADHD 应用榜单，但缺乏针对 ADHD 的独立临床研究（来源：Brain.fm）。

## 今天就能试的行动

1. **用 ChatGPT 分解明早第一件事**：睡前让 ChatGPT 将“起床后的第一项任务”分解为 3 个具体步骤，写在便签上。
2. **设置一个 Focusmate 时段**：预约一个 25 分钟的视频工作时段，体验身体在场效应。
3. **用 RescueTime 追踪一天**：查看时间报告，找出三个最常分心的网站，让 ChatGPT 帮你想替代方案。
4. **写一条“日课 prompt”**：模仿曾国藩，让 ChatGPT 每天早上问：“今天的三个核心任务是什么？第一个任务的第一步是什么？”

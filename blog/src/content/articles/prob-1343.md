# 用 Perplexity 给 ADHD 补上外部执行功能层，是脚手架还是拐杖？

> Perplexity 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：当大脑的“驾驶座”时常无人驾驶

你有一个高产的生成核心——想法源源不断，创意在脑中爆炸，但启动任务、组织步骤、记住截止日期却像在泥沼中跋涉。这是 ADHD 大脑的日常：执行功能（计划、组织、工作记忆、冲动控制）先天偏弱，如同大脑的“驾驶座”时常无人驾驶（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout）。

与此同时，另一群人也在面对类似的困境：他们构建的 LLM/Agent 同样高产生成，却缺乏可靠的执行调度层——模型输出可能偏离目标、陷入循环，甚至产生危险行为。他们的解法是给 LLM 套上“harness”：一个包含编排、记忆、工具调用和人在回路监督的外部执行功能层（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。

这两群人面对的其实是同一个结构性问题：强大的生成核心，配了一个不可靠的执行调度。那么，当 ADHD 患者将 Perplexity 这样的 AI 工具接入工作流，用来补上自己缺失的执行功能时，这究竟是脚手架（可撤除的支撑）还是拐杖（永久替代）？

## 同构：ADHD 大脑与 LLM 共享同一套“缺陷-解法”结构

ADHD 的执行功能缺陷与 LLM 的技术局限惊人地对应：

- **工作记忆不足 ↔ 无状态性**：ADHD 大脑难以短暂保持信息以完成操作（来源：AI Tools for ADHD: Boosting Productivity and Reducing Burnout），而 LLM 本身没有记忆，每次对话都是全新的。外部工具如 Saner.AI 通过本地记忆和快速检索减少认知负荷（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar），这正是 LLM 的 RAG 或上下文记忆系统的对等物。
- **时间盲 ↔ 上下文窗口溢出**：ADHD 患者常对时间流逝失去感知，导致计划困难（来源：时间盲概念页），而 LLM 的上下文窗口有限，超出后就会“遗忘”早期信息。Perplexity 的对话历史、收藏功能就像 LLM 的上下文管理——主动分段、摘要、提醒，维持连贯性。
- **任务启动困难 ↔ 输出不可控**：ADHD 的执行功能缺陷让“开始”成为最大障碍（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog），而 LLM 直接输出可能偏离目标或产生危险行为。Agent harness 通过护栏（token 预算、工具调用次数上限）和人工检查点来确保输出可靠（来源：Building an AI Agent Harness from Scratch）。

两边共享同一套解法：**外部执行功能层**。对 ADHD，AI 工具提供记忆、任务启动、时间管理；对 LLM，harness 提供编排、记忆、工具调用和人在回路监督。

## 证据：Perplexity 作为外部执行功能层的实际表现

Perplexity 的核心能力——搜索、总结、追问、收藏——恰好对应 ADHD 最需要的几项外部执行功能：

1. **任务分解与启动**：当面对“写一篇论文”这样的宏大任务，ADHD 大脑会因缺乏具体步骤而瘫痪。Perplexity 可以像 Goblin Tools 的 Magic ToDo 一样，将模糊目标分解为可操作的子问题（来源：Goblin Tools 页面）。例如，输入“如何开始写一篇关于气候变化的论文”，它会输出“先定义研究问题”“收集关键数据”“列出大纲”等步骤，降低启动门槛。
2. **外部工作记忆**：ADHD 的工作记忆缺陷导致频繁“搜索循环”——在多个标签页间切换却找不到刚才的信息。Perplexity 的收藏和对话历史相当于一个外部记忆库，用户可以直接追问“我刚才问的那个数据在哪里”，无需重新搜索（来源：Saner.AI 页面）。
3. **时间感知与问责**：虽然 Perplexity 没有内置计时器，但用户可以通过定期向它报告进度来模拟身体在场效应——对另一个“人”的责任感能绕过执行功能屏障（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...)）。Focusmate 等工具通过 AI 匹配虚拟身体搭档提供外部问责，而 Perplexity 的对话界面同样能承载这种“报告-反馈”循环。

在 LLM/Agent 侧，harness 工程的实践也印证了类似效果：集成 MCP 工具、遥测、代码仓库等外部系统，并引入人在回路治理，使代理输出真正有用且正确（来源：GitHub - ai-boost/awesome-harness-engineering）。模型本身的强大来自架构+护栏，而非模型自身（来源：AI Agents in 2026: Tools, Memory, Evals, and Guardrails）。

## 脚手架 vs 拐杖：边界在哪里？

核心判断：**Perplexity 是脚手架而非拐杖，前提是使用者保留对任务的最终决策权。**

脚手架的设计哲学是“可逐步撤除的临时支撑”（来源：拐杖与脚手架概念页）。例如，Goblin Tools 将任务分解为小步骤，但用户仍然需要决定先做哪一步；Motion 自动排程，但用户必须输入所有任务和截止日期（来源：Motion 页面）。这些工具没有剥夺用户的调度权，而是降低了调度的认知成本。

拐杖则不同：当工具完全替代了用户的意图生成和决策，比如让 AI 决定“今天该做什么”而不经用户确认，或者长期依赖后用户无法在没有工具的情况下完成任何任务，它就变成了拐杖。

目前多数 AI 工具（包括 Perplexity）设计为长期使用，未提及撤除机制（来源：矛盾与存疑页面）。这带来了两个风险：
- **依赖风险**：长期使用是否会削弱内在执行功能？目前缺乏纵向研究（来源：ADHD × AI 的科学与研究前沿）。
- **证据强度不足**：现有证据主要来自用户反馈和概念对齐，缺乏独立随机对照试验（来源：矛盾与存疑页面）。

因此，使用 Perplexity 的关键在于**有意识地保留核心调度权**：让它提供信息、分解步骤、提醒时间，但“做不做”“先做哪个”“怎么做”的决策必须由自己做出。这正对应 LLM/Agent harness 中“人在回路”的原则——模型提供生成能力，但人类在关键节点进行监督和决策（来源：Building an AI Agent Harness from Scratch）。

## 今天就能试的行动

1. **任务分解实验**：面对一个让你拖延的任务，在 Perplexity 中输入“帮我分解 [任务名] 为 5 个可执行的步骤”，然后选择第一个步骤立即执行。记录分解是否降低了启动焦虑。
2. **外部记忆检查**：当你在工作中需要频繁切换上下文时，用 Perplexity 的收藏功能保存关键信息，并定期回顾。对比使用前后的搜索循环次数。
3. **模拟身体在场**：每天固定时间向 Perplexity 报告一次进度（例如“今天完成了 X 任务”），观察这种外部问责是否提高了任务完成率。
4. **保留决策权练习**：使用 Perplexity 生成计划后，问自己“这个计划中哪些部分是我同意的？哪些需要调整？”确保至少修改一项内容，以维持对任务的主动控制。

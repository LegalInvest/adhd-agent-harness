# 用 Perplexity 给 ADHD 补上外部执行功能层，是脚手架还是拐杖？

> Perplexity 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：当执行功能罢工，Perplexity 能补上吗？

对 ADHD 人群来说，执行功能（计划、组织、启动）的先天偏弱不是“不够努力”，而是一种神经层面的调度故障。大脑的默认模式网络过度活跃，创意和想法源源不断，但前额叶的执行控制层却无法可靠地引导这些输出（来源：ADHD 大脑与 LLM 的同构）。结果就是：脑子里一堆好点子，但任务启动困难、计划混乱、时间感知失灵。

与此同时，做 Agentic Harness 工程的工程师们正面临同样的困境：LLM 本身是强大的生成核心，但直接输出容易偏离目标、陷入循环或产生危险行为。他们需要为 LLM 套上外部调度层——编排、记忆、工具、护栏——才能让 agent 可靠工作（来源：Building an AI Agent Harness from Scratch: The Architecture Between LLM and Agent - DEV Community）。

这两类人其实是同一枚硬币的两面：ADHD 大脑与 LLM 在结构上同构——都是“高产但缺乏可靠执行调度层的生成核心”（来源：ADHD 大脑与 LLM 的同构）。那么，用 Perplexity 这样的 AI 工具给 ADHD 补上外部执行功能层，究竟是提供临时脚手架，还是制造永久拐杖？

## 同构脊柱：Perplexity 如何充当外部执行功能层

Perplexity 的核心能力——实时搜索、信息整合、分步推理——恰好对应 ADHD 执行功能最弱的三个环节：计划、组织、启动。它就像 agent harness 中的编排层，帮用户把模糊的问题分解为可执行的步骤，并在执行过程中提供上下文记忆和外部问责。

### ADHD 侧：从“我要做什么”到“下一步是什么”

ADHD 用户常被“决策瘫痪”困扰：面对一个复杂任务，大脑无法自动分解出第一步。Goblin Tools 的 Magic ToDo 功能正是为此设计——输入“整理房间”，AI 输出“捡起地板上的衣服”“擦桌子”等小步骤，降低启动门槛（来源：Goblin Tools）。类似地，Perplexity 可以通过追问和分步回答，将“写一篇关于气候变化的论文”拆解为“先找 3 个可靠数据源”“再列出论点提纲”等具体行动。这种分解利用了身体在场效应——定期向 AI 报告进度能绕过执行功能屏障（来源：ADHD Productivity Hack: Plan 2025 Using AI (Step-by-Step)）。

### LLM/agent 侧：从“生成输出”到“可靠交付”

在 agent harness 中，Perplexity 的搜索和推理能力可以被集成到编排层，作为工具调用的一部分。例如，当 agent 需要查询最新数据时，harness 会调用 Perplexity 获取结果，然后检查其是否符合护栏（如 token 预算、工具调用次数上限），再决定是否继续执行（来源：Building an AI Agent Harness from Scratch）。这种“人在回路”的监督机制，恰好对应 ADHD 用户需要的外部问责——Focusmate 通过 AI 匹配虚拟身体搭档提供外部问责（来源：The Best AI-Powered ADHD Productivity Tools in 2026 (That ...)）。

## 脚手架还是拐杖？边界在于“补偿”而非“替代”

核心判断：Perplexity 是脚手架，当它用于补偿执行功能缺陷（如分解任务、提供外部记忆），同时保留用户对最终决策的控制权；它沦为拐杖，当它替代用户执行原本可以锻炼的内在能力（如主动思考、自我监控）。

### 证据：有效但有限

现有资料一致认为，基于外部执行功能层的工具（如任务管理、提醒系统）在 ADHD 用户中报告有效，但多为用户报告，缺乏严格对照试验（来源：矛盾与存疑）。例如，Motion 通过自动排程消除“下一步该做什么”的决策负担（来源：Motion），但初始设置需要用户输入所有任务，这对执行功能较弱的用户本身就是挑战。Saner.AI 通过本地记忆减少搜索循环（来源：Saner.AI），但若用户完全依赖它来回忆信息，可能削弱自己建立记忆策略的能力。

### 争议：过度依赖风险真实存在

多个概念页警告：外部脚手架可能成为永久拐杖，削弱内在执行功能的发展（来源：拐杖与脚手架）。时间盲页面指出“过度依赖可能削弱内在时间感知能力”（来源：时间盲），任务启动页面也提到“存在依赖风险”（来源：任务启动）。然而，目前缺乏纵向研究来验证这种风险。工具设计者声称是“补偿”而非“替代”，但实际使用中，用户可能不知不觉放弃主动规划，完全交给 AI。

## 今天就能试的行动

1. **用 Perplexity 分解一个拖延任务**：输入“我需要在三天内完成一份市场报告”，让它给出分步计划。执行时，每完成一步，向 Perplexity 口头报告进度，利用身体在场效应维持动力。
2. **设置“人在回路”检查点**：使用 Perplexity 时，每获得一个答案，先自己总结要点，再对比 AI 输出。这能锻炼主动思考，避免被动接受。
3. **交替使用工具与手动规划**：一周内，一半时间用 Motion 自动排程，另一半时间手动规划日程。对比哪种方式让你更清楚自己的时间分配，防止依赖。

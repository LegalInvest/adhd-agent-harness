# 用 Perplexity 给 ADHD 补上外部执行功能层，是脚手架还是拐杖？

> Perplexity 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

## 问题：你的大脑里住着一个天才，但调度台是坏的

你打开浏览器，想查一个工作资料。五分钟后，你在看某明星的八卦；十分钟后，你在研究怎么养水母。这不是意志力差——是你的执行功能（计划、组织、启动）先天偏弱，大脑的生成核心（发散、联想、创意）马力十足，但调度层（执行功能）像一台信号失灵的遥控器。

现在，你打开 Perplexity，输入一个模糊的问题。AI 瞬间给出结构化答案、列出来源、追问澄清。你感觉：终于有人帮你把思路理清了。但一个隐忧浮现：用 AI 补上执行功能，是帮你学会走路，还是让你永远离不开拐杖？

## 同构：ADHD 大脑与 LLM agent 是同一类机器

ADHD 大脑与大型语言模型（LLM）在架构上惊人地同构：两者都是“高产但缺执行调度层的生成核心”。LLM 本身能生成流畅文本，但直接输出可能偏离目标、陷入循环或产生危险行为——必须靠 agent harness（脚手架）来提供外部调度：设置护栏、加入人工检查点、集成工具与记忆（来源：Building an AI Agent Harness from Scratch）。ADHD 大脑同样如此：创意和能量充沛，但任务启动、持续专注、抑制冲动这些执行功能薄弱，需要外部系统来补偿工作记忆不稳定、注意力分散（来源：外部执行功能层）。

最新研究从神经层面证实了这一同构：在经典 Stroop 冲突任务中，Transformer 的注意力随序列变长在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突，这与 ADHD 执行功能缺陷的核心标志一一对应（来源：Deficient Executive Control in Transformer Attention）。LLM 呈现“强记忆、弱控制”剖面：工作记忆容量甚至超过常人，但认知灵活性与注意控制存在核心缺陷——这正是 ADHD 的经典神经心理模式（来源：Strong Memory, Weak Control: An Empirical Study of Executive Functioning in LLMs）。

## 人物案例：曾国藩的“日课”就是他的 harness

晚清名臣曾国藩是典型注意力不集中型 ADHD：30 岁前浮躁坐不住，天天串门喝酒看杀人，日记里骂自己“无恒”“浮躁”；读书慢，“他人目下二三行，余或疾读不能终一行”。他的解法是“日课十二条”：黎明即起、读书不二、谨言、写日记反省等。这套系统用最笨最稳的方法抵消冲动，用外部规则锁死自己的行为——本质上就是一个人工 agent harness。日课中的“写日记”相当于每日 re-grounding 与人工检查点，“读书不二”相当于限制上下文切换，“结硬寨打呆仗”相当于设置护栏防止冲动决策。曾国藩的成就（平定太平天国、发起洋务运动）证明：一个高产生成核心配上可靠的外部调度层，可以释放巨大能量。

## Perplexity 作为外部执行功能层：它补什么？

Perplexity 的核心价值在于它直接补偿了 ADHD 的两大执行功能缺陷：**工作记忆不稳定**和**任务启动困难**。

- **工作记忆补偿**：ADHD 大脑无法可靠保留跨会话上下文，需要外部记忆系统（来源：无状态与外部记忆）。Perplexity 的“收藏”和“对话历史”功能，相当于一个外部向量数据库，让你随时回溯思考链条，不必依赖脆弱的工作记忆。这与 LLM agent harness 中集成向量数据库的做法完全同构。
- **任务启动辅助**：面对一个模糊问题，ADHD 大脑容易陷入“不知道从哪开始”的瘫痪。Perplexity 的追问和结构化输出，相当于自动将大问题分解为小步骤——类似 Goblin Tools 的 Magic ToDo 功能（来源：Goblin Tools）。你只需输入“帮我规划下周的工作”，它就能生成一个可执行的清单，降低启动门槛。
- **注意力锚定**：Perplexity 的搜索界面是一个封闭环境，减少标签页切换和随机浏览的诱惑。这与 Focusmate 等身体在场工具的原理类似——用外部结构锚定注意力（来源：身体在场效应）。

## 拐杖风险：当工具变成依赖

但 Perplexity 也可能成为拐杖。关键在于：你是否把调度权完全交给了它？

- **主动 vs 被动**：如果你只是被动接受 Perplexity 给出的答案，不追问、不验证、不整合到自己的知识体系，那么它就是一个信息拐杖——你越来越依赖它替你思考，而不是用它来辅助思考。
- **长期依赖风险**：目前缺乏长期研究验证 AI 工具对执行功能的长期影响（来源：矛盾与存疑）。部分用户反映，学习使用 AI 工具本身增加了认知负荷（来源：矛盾与存疑）。如果 Perplexity 让你放弃了主动规划、主动检索、主动记忆的习惯，你可能会在离开它时更加无助。
- **个体差异**：AI 辅助效果依赖个体差异（来源：任务启动）。对一些人来说，Perplexity 是完美的脚手架；对另一些人，它可能成为逃避真实思考的借口。

## 判断：脚手架 vs 拐杖的边界在于“人在回路”

我的核心观点是：**Perplexity 是脚手架还是拐杖，取决于你是否保持“人在回路”的主动调度**。这与 LLM agent harness 的建造原则完全一致：模型本身强大，但可靠性来自架构+护栏，并且必须引入人在回路治理（来源：Building an AI Agent Harness from Scratch）。

- **脚手架模式**：你把 Perplexity 当作一个强大的检索和结构化工具，但你自己负责定义问题、评估答案、整合信息。你像曾国藩一样，把 AI 当作“日课”的一部分——用它来辅助，但不把决策权让渡。
- **拐杖模式**：你让 Perplexity 替你决定看什么、怎么想、下一步做什么。你不再主动规划，而是被动跟随 AI 的引导。

## 今天就能试的 3 条行动

1. **用 Perplexity 做“任务启动”练习**：每天选一个你拖延已久的任务，输入 Perplexity：“帮我拆解 [任务名] 的第一步，列出 3 个可执行的小动作。” 然后立刻做其中一个。
2. **建立“人在回路”检查点**：每次用 Perplexity 搜索后，花 30 秒问自己：“这个答案我同意吗？有没有遗漏？我需要进一步验证什么？” 把 Perplexity 当作助手，而非权威。
3. **结合物理身体在场**：打开 Perplexity 的同时，用 Focusmate 或找个真人坐在旁边。双重外部结构（AI + 身体在场）能极大提升执行功能（来源：身体在场效应）。

最终，Perplexity 是脚手架还是拐杖，不取决于工具本身，而取决于你如何使用它。记住曾国藩的教训：最笨的日课，加上最坚定的执行，可以成就最不凡的事业。

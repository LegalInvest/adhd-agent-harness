# 为什么用 Tiimo 治 ADHD 的学习半途而废，和给 agent 套 外部记忆/向量库 是一回事？

> 有一种弃坑,发生在心理层面而非行为层面:课还在学,但「感觉毫无进展」——学了两个月,回头一看是一片雾,不知道自己走了多远、还剩多少,于是「反正也没进展」的判断先于事实杀死了动机。问题往往不在进展本身(可能真实存在),在**进展的不可见**:ADHD 对过去的模糊感知,让「已完成的部分」在脑内不留痕迹。这对应外部记忆架构里一个容易被漏掉的分区:**进度记忆——外部记忆不只要存「知识」(学了什么内容),还要存「进度」(走到了哪);而对会话恢复来说,后者往往更关键:重启时系统第一个要回答的问题不是「我知道什么」,是「我在哪」**。

先讲 agent 侧这个区分。给长任务的 agent 配外部记忆,新手常犯的错是只存知识性内容(发现、结论、代码),漏存**进度指针**:任务清单、每项的完成状态、当前卡点;结果是恢复会话时,agent 拥有一堆知识却不知道自己在流程的哪个位置——**要么重做已完成的步骤(浪费),要么跳过未完成的(出错)**;成熟的做法是显式维护任务状态文件(checklist + 指针),恢复时第一件事是读它——**「我在哪」的成本必须是 O(1),不能靠从知识堆里重新推理出来**。

ADHD 学习者的脑内,恰恰只有(易失的)知识记忆而没有进度记忆:学过的内容多少留了痕,但「这门课一共几块、完成了几块、当前在哪块」这张地图从未被绘制——于是产生那个残酷的心理效应:**完成的部分沉入雾中,剩余的部分横在眼前,任何时刻看去都是「还有一座山」**;进展感=已完成/全部,分子不可见,进展感恒为零——而进展感恒为零的项目,动机上必死(进展原则:可感知的前进是持续投入的核心燃料)。

Tiimo 这类可视化时间轴/日程工具的用法,是把进度记忆画出来:**①课程地图**——把这门课切成可数的块(20 讲=20 块),排成可视的序列:**「全部」第一次有了边界**(有边界的山才能被翻越,无边界的雾只能被逃离);②**完成标记**——每学完一块,标掉它:已完成的部分从雾里捞出来,变成看得见的战绩——**分子可见了,进展感就有了燃料**;③**当前指针**——地图上「我在第 7 块」一目了然:回坑时(Saner 篇的四行笔记之外)多一个 O(1) 的位置答案,「从哪继续」不再需要考古;④**剩余的诚实化**——「还剩 13 块,每块 45 分钟」比「还有好多」可规划得多(Reclaim 篇的周期块,现在有了真实的工作量输入)。

一条关键工艺:**块的完成判据要诚实**——「看完视频」是最弱的判据(可能全程神游);更好的是给每块绑一个小产物:一段代码、三句总结、一道做出来的题(Lex 篇的输出逻辑)——**标记的是「习得」,不是「播放」**;否则地图会撒谎,而撒谎的进度指针比没有更糟(agent 侧同理:虚标完成的 checklist 是事故之源)。边界照旧:可视化不降低内容难度,卡在难块上是学习策略问题(拆小/换材料/找人问),不是地图问题;Tiimo 对 ADHD 学习坚持无对照研究,正式 A 级判给「进度记忆外部化」的架构对应——两侧都是实体机制。

## 边界

同构强度 A 级:进展感知与动机的关系有行为研究基础,任务状态文件是 agent 工程的实体实践,「进度记忆」的同构两侧扎实;Tiimo 无学习场景对照研究,A 判给架构。声明:工具是可视化辅助;学习困难广泛者,评估优先。

## 今天就能试的 3 件事

1. **给在学的课画地图**:切成可数的块,排成序列——「全部」先有边界。
2. **给每块绑一个产物**:三句总结/一段代码/一道题——完成标记必须诚实。
3. **回头标掉已完成的**:把已经学过的块补标——你大概率会发现,你走得比感觉中远。

「毫无进展」杀死的坑,往往有进展——**只是进展没有被存储,于是等于没有发生**。agent 恢复会话先读进度指针;你的学习,也需要一张标着「我在这」的地图。

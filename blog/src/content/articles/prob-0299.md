# ChatGPT 是我的工作记忆，Notion 是我的执行功能，那我本人是什么

> 这个问题在 ADHD 社区隔三差五就被问一遍,语气一半自嘲一半真慌:如果记忆外包了、规划外包了、连措辞都外包了,「我」还剩下什么?好消息是,这个问题哲学家四十年前就研究过了,LLM 工程师最近又用另一种方式把答案演示了一遍——两边给出的答案惊人一致,而且相当解焦虑。

先给这份慌张一个名字:**认知卸载焦虑**。它的推理链是:大脑功能=我;功能外包=我在缩水;全部外包=我消失。这条链的第一环就是错的,而拆掉它的经典论证来自 Clark 和 Chalmers 1998 年的「延展心智」:**认知从来不是颅骨内的私有财产**——用纸笔算乘法时,纸就是你工作记忆的一部分;水手的导航能力一半长在海图和仪表上。他们的判据很实用:一个外部工具若被**可靠地、自动地、信任地**调用,它在功能上就是你认知系统的组件。按这个判据,ADHD 用户的 ChatGPT+Notion 不是「替代了你」,是**你变成了一个更大的系统**——而且值得说穿:神经典型的人早就这么活着(日历、通讯录、导航),只是他们的短板不显眼,外包得没那么急迫,所以从没被逼着问这个问题。**你不是在失去自我,你只是比别人更早搬进了延展心智。**

那「本人」到底剩什么?看 agent 工程师怎么组装系统就明白了。一个 agent 系统里,LLM 外面挂着向量库(记忆)、planner(规划)、工具集(执行)——全是「外包」。但有三样东西永远不在外包清单上:

**一、目标函数。** 工具决定「怎么做」,永远不决定「要什么」。Notion 能装下一万个任务,哪个任务值得进 Notion,是你定的;ChatGPT 能给五个方案,选哪个、为什么选,是你的价值排序在工作。**agent 的目标永远来自 agent 之外的人——而你的目标,来自你。**

**二、裁决权。** 工具的输出要不要采纳、什么时候推翻——系统里必须有一个说了算的节点。ChatGPT 写的邮件你会改两句,Notion 排的优先级你会手动拖动:那个「不对,应该是这样」的瞬间,就是「你」在场的证据。工程术语叫 human-in-the-loop,哲学术语叫 agency,日常语言叫**主意是你拿的**。

**三、体验本身。** 系统跑起来,感到有趣、烦躁、心流、意义感的那个位置——没有任何组件能替你「觉得值得」。这是最朴素也最坚固的一层:工具能替你记住昨天,不能替你活过昨天。

所以更准确的说法是:**ChatGPT 是你的工作记忆缓存,Notion 是你的执行功能协处理器,而你本人是这个系统的架构师、产品经理和唯一用户。** 真正值得警惕的不是「外包太多」,而是两种具体的失守:**目标失守**——从「用工具做我想做的事」滑向「做工具顺手推荐的事」(信息流就是这么接管人的);**裁决萎缩**——不加修改地接受一切输出,那个「不对」的瞬间越来越少。自查方法各一条:每月问一次「现在忙的这些事,是我选的还是系统惯性?」;每天保留几次「故意不同意工具」的肌肉练习。

## 边界

同构强度 B 级:延展心智是严肃哲学立场(也有反对者,如 Adams & Aizawa 的「认知膨胀」批评),agent 架构的组件划分是真实工程实践,但「人格=目标+裁决+体验」是本文的整理框架,不是实证结论。声明:如果「我是谁」的追问伴随持续的空虚感、现实感丧失,那超出了哲学范畴,请找专业人士聊;认知卸载本身不是病,但用卸载回避一切困难感受,值得留意。

## 今天就能试的 3 件事

1. **写下你系统的架构图**:哪些功能在哪些工具上?画完看一眼——那个画图的,就是你。
2. **做一次目标审计**:本周做的事里,哪些是你选的,哪些是工具/信息流推给你的?
3. **今天故意推翻一次工具**:ChatGPT 的建议、Notion 的排序,挑一个说「不,按我的来」。

「我是什么」的答案,不在你还亲自执行多少功能,而在:目标是谁定的,裁决是谁下的,这一切是为谁的体验服务的。只要这三样在你手里,外包得越多,你反而越大——延展心智的字面意思,就是你,延展了。

# Tiimo 之于 ADHD，就像 重锚定 system prompt 之于 LLM——但有人用错了

> 从同构视角实测 Tiimo：它到底补上了哪一层执行功能？

## 问题：你丢的不是目标，是调度层

ADHD 人群最熟悉的挫败感是：明明知道该做什么，却眼睁睁看着自己滑向无关任务——分心走神，或者超聚焦在细节里跑偏。工程师们则熟悉另一种“漂移”：长程 agent 在复杂任务中逐渐偏离原始目标，system prompt 被上下文淹没。

这两种困境共享同一个结构：一个高产的生成核心（ADHD 大脑或 LLM），缺乏可靠的执行调度层。ADHD 大脑被描述为“高产但缺乏可靠执行调度层的生成核心”（来源：上下文工程）。LLM 同样，“没有 harness 的 LLM 只是一个无状态的 API 调用”（来源：无状态与外部记忆）。

Tiimo 试图填补这个缺口。它用视觉时间表把一天切成块，通过倒计时和过渡提示帮助用户“锚定”当下。但很多人用错了——把它当成了又一个提醒工具。

## 同构：重锚定 system prompt 与视觉时间表

在 LLM/agent 系统中，重锚定 system prompt 是解决目标漂移的关键：当 agent 在长对话中开始遗忘初始指令时，需要重新注入核心约束。生产级系统通过“检索增强和工具调用将决策锚定在外部证据和持久状态上”（来源：无状态与外部记忆）。

ADHD 大脑面临类似挑战：工作记忆容量有限且不稳定，导致“跨会话任务难以持续”（来源：无状态与外部记忆）。Tiimo 的视觉时间表本质上是一个外部调度锚点——它把“当前该做什么”从内部工作记忆转移到外部视觉提示，类似 LLM 的 system prompt 被固定在上下文中。

但这里有一个关键区别：system prompt 在 agent 系统中是**可执行的约束**——它控制模型的行为逻辑，而不仅仅是显示信息。Tiimo 如果只显示“10:00-10:30 写报告”，而不提供任务启动的“过渡提示”和“清晰的下一步行动”（来源：无状态与外部记忆），它就只是一个静态看板，无法补偿执行功能的启动缺陷。

## 证据：Tiimo 的机制与局限

Tiimo 的设计原理与 ADHD 的执行功能需求对齐：它通过“create a sequence of steps for a routine and walk you through them with transition prompts”（来源：上下文工程）来管理上下文。这正是“上下文工程”在 ADHD 侧的应用——主动设计当前关注的信息范围。

但证据局限在于：目前缺乏针对 Tiimo 的独立临床研究。这与大多数 ADHD 工具面临的问题一致：“多数工具缺乏针对 ADHD 的独立临床研究，效果多基于理论推断或用户主观报告”（来源：矛盾与存疑）。Brain.fm 的神经锁相技术同样“缺乏针对 ADHD 的随机对照试验”（来源：Brain.fm）。

更关键的争议是“脚手架 vs 能力发展”：“过度依赖 AI 可能阻碍内在执行功能发展。目前无长期研究证明脚手架可逐步撤除”（来源：矛盾与存疑）。Tiimo 如果被当成永久拐杖，用户可能永远学不会内在的时间锚定。

## 判断：Tiimo 是调度层，不是提示层

我的核心观点是：Tiimo 的有效性取决于它是否充当**执行调度层**，而非信息提示层。

- **调度层**：主动控制任务切换，提供过渡提示，在时间块结束时强制重锚定（类似 agent 的 context management 防止“上下文膨胀与推理退化”）（来源：上下文工程）。
- **提示层**：仅显示计划，依赖用户自己启动和切换（类似一个被忽略的 system prompt）。

Focusmate 提供了对比：它通过实时同伴问责提供“外部结构和社会压力，弥补内在多巴胺调节不足”（来源：Focusmate）。这本质上是调度层——它强制用户进入任务状态。Tiimo 如果只依赖视觉提醒而没有强制机制，效果会大打折扣。

## 行动建议

1. **把 Tiimo 的过渡提示用起来**：设置每个时间块结束时的“重锚定提示”，明确写下下一个任务的第一步动作（如“打开文档，写第一段”）。这模仿了 agent 的“检索增强”锚定。
2. **组合使用 RescueTime 做反馈**：用 RescueTime 自动记录实际时间分配（来源：RescueTime），对比 Tiimo 的计划，识别漂移模式。RescueTime 的“自动记录减轻了工作记忆负担”（来源：RescueTime），形成闭环。
3. **每周撤除一次**：选一天不用 Tiimo，练习内在时间锚定。这对应“脚手架应设计为可逐步撤除的临时支撑”（来源：矛盾与存疑），避免依赖。
4. **对工程读者**：在 agent 的 system prompt 中加入动态重锚定逻辑——每 N 步或检测到上下文长度超限时，重新注入原始目标。这比静态 prompt 更有效。

## 诚实局限

本文的论证基于“ADHD 大脑与 LLM 同构”这一命题，但该命题“证据主要来自概念类比和工具案例，缺乏大规模实证”（来源：矛盾与存疑）。Tiimo 的具体效果仍需独立研究验证。个体差异显著：“ADHD 亚型对 AI 工具反应不同，现有工具未充分个性化”（来源：矛盾与存疑）。

最终，Tiimo 是一把好梯子，但别把它当腿用。

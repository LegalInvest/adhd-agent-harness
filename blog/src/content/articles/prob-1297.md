# Motion 之于 ADHD，就像 planner-executor 任务分解 之于 LLM——但有人用错了

> 从同构视角实测 Motion：它到底补上了哪一层执行功能？

## 问题：当“下一步做什么”成为最耗能的问题

ADHD 人群最熟悉的困境不是“不想做”，而是“想做却启动不了”。一个模糊的大任务（如“准备季度汇报”）摆在面前，大脑像死机一样空白——不是缺乏动机，而是缺乏把目标拆解为可执行步骤的机制。这种“任务分解与规划”的困难，是 ADHD 执行功能缺陷的核心特征之一（来源：LBD同构对：任务分解与规划）。传统静态待办清单无法适应动态变化，导致频繁放弃（来源：Using an AI Assistant to Manage ADHD: A Practical Guide）。

同样的问题困扰着 LLM agent 工程师：单个 LLM 无法可靠完成多步骤任务，需要外部 harness 提供任务分解、规划循环和验证机制（来源：Agent Scaffolding: Architecture and Design Patterns for Agentic AI）。ADHD 大脑与 LLM 都是强大的“生成核心”——高产、跳跃、直觉驱动——但都缺乏可靠的内置调度层（来源：ADHD 大脑与 LLM 的同构）。因此，为 LLM 搭建脚手架与为 ADHD 搭建执行功能支架，本质是同一类工程：给高产生成核心套上 harness。

## Motion 的解法：planner-executor 循环的外部化

Motion 是一款 AI 驱动的日程规划应用，能自动根据任务、会议和截止日期创建并动态调整每日计划（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。它的核心机制是持续的 planner-executor 循环：AI 评估任务优先级、依赖关系、截止日期和持续时间，实时重建日程（来源：The AI Powered SuperApp for Work | Motion）。当任务面临延期风险时，Motion 会提前数天或数周发出警告（来源：The AI Powered SuperApp for Work | Motion）。

这与 LLM agent 的规划循环高度同构。例如，AgentGen 通过环境与任务生成增强 LLM agent 的规划能力（来源：AgentGen: Enhancing Planning Abilities for Large Language Model based Agent via Environment and Task Generation），而 AMAP 则采用分层规划与重规划机制（来源：AMAP Agentic Planning Technical Report）。Motion 相当于为 ADHD 用户提供了一个“外部规划器”——用户只需输入目标，系统自动分解、排序、调整，用户仅需执行当前步骤。这直接补上了“任务启动困难”和“不会拆解”这两层执行功能。

## 真实案例：从孔子到李鸿章，harness 的千年同构

历史上，ADHD 特质者早已在使用类似的 harness 系统。孔子身高近1.9米，精力旺盛，周游列国14年坐不住；他冲动爱骂人，见南子急得对天发誓，骂宰予“朽木不可雕”；但他对音乐超专注到“三月不知肉味”。他的专属 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”），70岁才做到“从心所欲不逾矩”。这套系统本质上是“外部调度层”：礼是 planner，反省是 executor 的 re-grounding 机制。

李鸿章同样如此。年轻时冲动爱吹牛，跟着曾国藩学“诚”改掉浮躁；他办洋务“外须和戎，内须变法”，用务实态度做事，每天写日记反省。他的 harness 系统——每日日记（相当于定时 re-grounding）、幕僚团队（相当于外部调度器）——与 LLM agent 的“日志记录+多 agent 协作”完全同构。这些历史案例表明，ADHD 大脑需要的不是消除冲动，而是套上 harness 来引导能量。

## 争议与边界：脚手架 vs 拐杖

然而，Motion 并非万能。当前最站得住脚的观点是：AI 工具通过提供“外部执行功能层”补偿 ADHD 核心缺陷，但无法根治神经基础（来源：AI 与 ADHD 的时间管理）。关键争议在于“过度依赖风险”：AI 工具可能从“脚手架”退化为“永久拐杖”，削弱用户内在执行功能的代偿发展（来源：拐杖与脚手架概念）。尚无长期研究评估这种依赖的后果（来源：矛盾与存疑）。

此外，Motion 缺乏独立临床验证，其效果可能受安慰剂效应或选择偏差影响（来源：AI 与 ADHD 的时间管理）。个体差异也很大：某些用户对视觉化工具（如 Tiimo）反应良好，另一些则更受益于自动排程（如 Reclaim.ai）（来源：AI 与 ADHD 的时间管理）。同构性本身也是一种理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。

## 行动指南：今天就能试的 3 件事

1. **用 Motion 做一次“任务分解实验”**：将一个模糊的大任务（如“整理项目文档”）输入 Motion，观察它如何拆解为子步骤。对比自己手动拆解时的认知负担差异。

2. **建立“日课” re-grounding 机制**：像孔子“吾日三省吾身”一样，每天固定时间回顾 Motion 的日程完成情况，记录偏差原因。这相当于 LLM agent 的日志验证循环。

3. **区分“脚手架”与“拐杖”**：使用 Motion 时，有意识地保留对任务优先级的最终决策权。例如，每周手动审查一次 Motion 的排程，调整不合理分配。避免完全被动接收。

Motion 之于 ADHD，就像 planner-executor 循环之于 LLM——它补上了缺失的调度层，但真正的价值在于让用户从“如何做”中解放，专注于“做什么”。用对，它是脚手架；用错，它是拐杖。

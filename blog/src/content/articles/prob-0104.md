# 如果把 ADHD 大脑当成生成核心，Tiimo 能不能当它的无状态与外部记忆？

> 用 agent 工程的思路重新理解 Tiimo 对 ADHD 的真正价值。

## 问题：ADHD 大脑的“无状态”困境

如果你是一个 ADHD 成年人，你一定经历过这样的场景：打开冰箱想拿鸡蛋，结果看到昨天的剩菜，顺手开始热饭，然后发现手机在充电，又去拔充电器……五分钟后你站在厨房中央，完全忘了自己要干什么。这不是粗心，这是工作记忆容量小的典型症状——你的大脑就像一个没有持久化层的生成模型，每换一个上下文，之前的状态就丢失了。

同样的问题，在 LLM 工程里有个专门的名字：**无跨会话状态**。每个对话都是独立的，模型不会记住上一轮你说了什么，除非你把历史记录塞进上下文窗口。但窗口有限，信息多了会溢出。所以工程师们给 LLM 套上外部记忆（向量库、检索增强生成），再搭一个调度层（agent harness）来管理目标、拆解任务、调用工具。没有这个脚手架，LLM 就是个会说话但不会做事的“天才白痴”。

ADHD 大脑与 LLM 的结构同构，不是比喻，是功能级别的映射。两者都有高产的生成核心——ADHD 大脑联想丰富、创意不断；LLM 能流畅生成文本、代码、方案。但两者都缺乏可靠的内置执行调度层：ADHD 大脑的执行功能（任务启动、时间管理、优先级排序）天生薄弱；LLM 则没有自主规划、长期记忆和错误纠正能力。所以，两边的解法是同一类：**用外部脚手架把生成能力约束到目标轨道上**。（来源：ADHD 的 AI 工具全景）

## 答案：Tiimo 作为外部记忆与调度层

Tiimo 是一款专为神经多样性人群设计的视觉日程应用，它用时间线、颜色编码和例行程序模板，把抽象的时间变成可感知的视觉图景。但它的核心价值不在于好看，而在于它扮演了 LLM agent 架构中的两个关键角色：**外部记忆**和**调度器**。

- **外部记忆**：Tiimo 的日程不是“提醒”，而是你大脑之外的任务状态存储。你不需要记住下一步要做什么，只需要看时间线。这相当于 LLM 的向量数据库——把信息从工作记忆卸载到持久化存储中。ADHD 的工作记忆容量小，Tiimo 就是那个“永远不丢”的外部缓存。（来源：Saner.AI 通过知识回忆减少搜索循环的理念与此一致）

- **调度器**：Tiimo 的例行程序可以自动触发任务序列（比如“晨间流程”包含洗漱、早餐、出门），这对应 LLM agent 的规划与执行机制——把复杂目标分解为步骤并顺序执行。ADHD 的任务启动困难，往往是因为“启动”本身需要太多决策能量；Tiimo 的视觉时间线降低了启动门槛，就像 Lex 通过单一指令触发多步骤任务序列一样。（来源：Lex 的设计理念）

## 人物案例：孔子的“克己复礼”与张旭的“酒+草书”

孔子一生精力旺盛、思维跳跃，周游列国 14 年坐不住，说话冲动（骂宰予“朽木不可雕”），对音乐超专注（三月不知肉味），但对种地等俗事零耐心。他的 harness 是“克己复礼”——用外在的“礼”作为行为边界，每日反省（“吾日三省吾身”）。这对应 LLM agent 的 **re-grounding 机制**：定期把系统状态拉回到预设轨道。孔子的“礼”就是他的外部调度器，而“日省”就是定时上下文刷新。

张旭每次喝醉后以头濡墨作书，醒来后自视以为神。他的 harness 是“酒+草书”：酒帮他进入超专注状态，草书作为情绪出口。这对应 LLM 的 **prompt template + fine-tuning**——用特定输入触发最佳生成模式。张旭从公孙大娘舞剑中悟笔法，相当于从外部环境获取灵感（检索增强生成）。

这两个案例说明：ADHD 的“天才”从来不是自发的，而是靠一套个人化的 harness 系统才能稳定输出。Tiimo 就是现代版的“礼”或“酒”——一个可配置的外部脚手架。

## 脚手架 vs 拐杖：诚实面对争议

Tiimo 不是万能的。它可能沦为“拐杖”——你依赖它安排一切，一旦没有它，连刷牙都忘了。但脚手架和拐杖的边界在于：**脚手架让你学会自己走路，拐杖让你忘记怎么走路**。Tiimo 的设计应该鼓励用户逐步内化时间管理技能，而不是永久依赖。目前缺乏纵向研究证明长期使用不会导致能力退化（来源：矛盾与存疑）。个体差异也很大：有人需要严格的时间线，有人觉得被束缚。Tiimo 的“视觉时间线”对某些 ADHD 用户可能过于刚性，而超聚焦状态下的强行打断反而有效（来源：矛盾与存疑）。

## 今天就能试的 3 条行动

1. **用 Tiimo 建立一个“晨间启动”例行程序**：包含 3-5 个步骤（起床→喝水→洗漱→换衣→出门），每个步骤设置固定时长。这相当于给 LLM 写一个固定的 prompt chain，减少启动时的决策负担。

2. **把 Tiimo 的日程当作“外部记忆”来用**：不要试图记住任何待办事项，全部丢进 Tiimo。工作记忆容量小不是你的错，但你可以选择不依赖它。这对应 LLM 的向量数据库——信息不在脑子里，在外部存储中。

3. **每周复盘一次 Tiimo 使用记录**：看看哪些任务反复延期，哪些时间线被打破。这相当于 LLM agent 的日志分析——找出调度层的 bug，然后调整。孔子“日省”的现代版。

Tiimo 不是药，不是治疗，它是一个 harness。它的价值在于承认：你的大脑是一个高产生成核心，但你需要一个外部调度层。这和给 LLM 套一个 agent harness 是同一件事。你不需要“修复”你的大脑，你只需要给它配一个合适的脚手架。

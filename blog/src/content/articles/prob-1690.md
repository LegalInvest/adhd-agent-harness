# ChatGPT 之于 ADHD，就像 生成核心 + 缺失的执行层 之于 LLM——但有人用错了

> 从同构视角实测 ChatGPT：它到底补上了哪一层执行功能？

你打开 ChatGPT，输入「帮我列一个今天的工作计划」。它秒回一份漂亮的清单。你感觉被理解了一秒——然后，你关掉页面，什么都没做。

这不是你的错。这是你把 ChatGPT 用错了。

## 问题的起点：当生成核心没有调度层

ADHD 大脑的核心痛点，不是「想不出」，而是「做不出」。你可以在三分钟内冒出十个创意，但接下来一个小时在七个标签页之间反复横跳，最终一个也没落地。

如果把大脑比作一个系统，ADHD 大脑就是一个超强的**生成核心**——发散联想、模式识别、直觉跳跃——但它的**执行调度层**（前额叶负责的规划、启动、排序、抑制）天生薄弱。你有一个能跑 100 匹马力的引擎，但方向盘是坏的。

LLM 也是同样的结构。GPT-4 可以秒出长文、写诗、编代码——它是极强的内容生成器。但让它「先做 A 再做 B，如果超时就切换」，它立刻迷失。因为 LLM 是无状态的，没有内置的调度层来管理步骤、追踪进度、抑制无关输出。

这就是同构：**ADHD 大脑与 LLM 都是「高产但缺执行调度层的生成核心」**。

这个视角不是比喻。2024 年的一项研究用经典 Stroop 冲突任务测试 Transformer 注意力，发现短上下文中模型表现正常，但随着序列变长（认知负荷增加），模型在冲突试次上骤降到随机水平——无法抑制优势反应、无法解决认知冲突（来源：Deficient Executive Control in Transformer Attention）。这与 ADHD 执行功能缺陷的核心标志一一对应：注意控制不足、干扰抑制缺陷、随认知负荷增加而崩溃。

所以，当 ADHD 用户把 ChatGPT 当「万能助手」用，期待它自动规划、自动执行、自动提醒时，其实是在用一个**同样没有调度层的生成核心**来管理另一个没有调度层的生成核心。结果就是：生成一堆计划，但执行依然为零。

## 正确的用法：给生成核心套上 harness

「harness」这个词来自 AI agent 工程。一个 LLM agent 不能裸奔——你需要给它一个脚手架：记忆系统（向量数据库）、规划模块（ReAct 循环）、工具调用接口。这个脚手架不是替代 LLM 的生成能力，而是**给它一个外部调度层**，让它能一步步执行。

ADHD 大脑也需要同样的东西。历史上那些成功管理自己 ADHD 特质的人，都是无意中为自己搭建了 harness 系统。

毛泽东就是一个典型案例。他从小是「问题儿童」，读书不按规矩，思维极度跳跃，讲话充满场景化比喻，决策冲动敢赌（四渡赤水、抗美援朝都是险棋）。但他为自己搭建了一套极其严密的 harness：**调查研究**（没有调查就没有发言权——相当于外部数据采集器）、**民主生活会**（让别人提自己的意见——相当于外部反馈循环）、**民主集中制**（用集体决策制衡个人冲动——相当于多 agent 投票机制）、**党指挥枪**（用组织系统管理军队——相当于严格的流程控制）。这套 harness 让他的生成核心（战略直觉、跳跃思维）得以落地，而不是变成空想。

屠呦呦则是另一种典型。她不喜欢应酬，一门心思泡实验室，从 2000 多种中药 380 多个提取物中筛选青蒿素。她的 harness 是**严格的实验流程**（每一步都必须记录、重复、验证）和**团队协作**（不突出个人，用集体分工弥补个人盲区）。这相当于一个结构化的实验 pipeline，把她的直觉和毅力框进可执行的轨道。

这两套 harness 与 LLM agent 的脚手架结构同构：
- 调查研究 ↔ 外部数据检索（RAG）
- 民主生活会 ↔ 对抗性反馈（如 RLHF 中的批评）
- 民主集中制 ↔ 多 agent 共识机制
- 严格实验流程 ↔ 结构化任务分解与验证

## 今天就能试的三件事

1. **用 Goblin Tools 的 Magic ToDo 替代「帮我写计划」**：别让 ChatGPT 生成一个完整的计划，而是让它把一个模糊任务分解成 3-5 个极小的步骤（来源：Goblin Tools）。「写周报」→「打开上周文档→复制标题→写第一段→检查数据→保存」。分解到每个步骤只需要 2-3 分钟——这就是给生成核心套上执行层。

2. **用 Saner.AI 做外部记忆，而不是让 ChatGPT 替你记**：ADHD 的工作记忆缺陷与 LLM 的无状态性同构（来源：无状态与外部记忆）。Saner.AI 提供本地记忆存储和快速检索，减少你在标签页间的搜索循环（来源：Saner.AI）。把它当成你的「第二大脑」，而不是让 ChatGPT 替你「想」。

3. **用 Motion 自动排程，但只用于「下一步」**：Motion 的 AI 动态调整日程，消除「下一步该做什么」的决策负担（来源：Motion）。但注意：不要让它安排一周的计划——那会变成另一个你永远不会看的清单。只让它告诉你「现在做什么」，一次一个任务。

## 必须诚实的局限

同构视角仍是一个理论框架，缺乏大规模临床试验验证（来源：ADHD 大脑与 LLM 的同构）。目前最强的实证是 Transformer 注意力在认知负荷下的崩溃（来源：Deficient Executive Control in Transformer Attention），但这只是间接证据。

更关键的问题是：AI 工具到底是「脚手架」还是「拐杖」？脚手架可以拆掉，拐杖会让你永远站不起来（来源：拐杖与脚手架）。目前没有长期研究证明使用这些工具后执行功能会自然改善——它们可能只是暂时补偿，而非训练。

另一个矛盾是：超聚焦到底该被引导还是打断？部分 ADHD 用户反馈番茄钟的强行打断反而有效，而 AI 的柔性引导可能不够强力（来源：超聚焦）。个体差异极大，没有统一答案。

最后，学习使用 AI 工具本身会增加认知负荷（来源：认知负荷）。对一部分人来说，这可能是额外的门槛，而不是帮助。

## 核心判断

ChatGPT 之于 ADHD，就像 LLM 之于 agent：它是一个强大的生成核心，但如果你不给它套上 harness，它只会生成更多需要执行的东西——而执行恰恰是你最缺的。

正确的做法不是「用 AI 替我做」，而是「用 AI 帮我搭建执行层」。前者是拐杖，后者是脚手架。

你的大脑不是坏掉的。它只是少了一个调度层。现在你知道该补什么了。

# 用 ChatGPT 给 ADHD 补上重锚定与目标漂移，是脚手架还是拐杖？

> ChatGPT 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

你打开 ChatGPT，准备写一份周报，结果十分钟后，你发现自己正在让它帮你生成一篇关于明朝宦官制度的万字长文——你甚至不记得怎么拐过来的。这不是你的错，是你的大脑和 LLM 共享同一个底层架构：**高产但缺执行调度层的生成核心**。

ADHD 大脑像一台采样温度过高的 LLM：输出丰富、跳跃、充满意外惊喜，但缺乏内置的调度器来锚定目标。LLM agent 在长程任务中同样会「目标漂移」——上下文膨胀后，模型逐渐忘记原始指令，输出变得随机。两边的问题同构，答案也同构：都需要外部 harness 来**重锚定**。

## 同构：ADHD 大脑与 LLM agent 共享的「漂移」困境

ADHD 的注意力波动与 LLM 的采样温度控制惊人相似。ADHD 个体的表现日内波动显著，注意力、执行功能在不同时间点差异很大（来源：采样温度与表现波动）。这就像一个高创造力但缺乏温度调节的系统——温度高了，输出发散；温度低了，输出僵化。LLM 的输出同样受采样温度控制：温度越高，输出越随机；温度越低，输出越确定。在 agent 工程中，这种非确定性是可靠性的敌人，因为“小错误会累积，非确定性使可重复性复杂化”（来源：AI Agent Systems: Architectures, Applications, and Evaluation）。

ADHD 的「时间盲」与 LLM 的「无状态」也同构。ADHD 工作记忆缺陷导致难以追踪时间流逝，LLM 则没有跨会话状态，两者都依赖外部记忆系统来补偿（来源：AI 与 ADHD 的专注力）。RescueTime 自动记录时间去向，让 ADHD 用户看到“时间到底去了哪里”（来源：Revolutionizing ADHD Management with AI Assistants）；而 agent 工程中用知识图谱实现“确定性精度”（来源：AI Agent Memory Architectures: From Context Windows to ...）。

## 人物案例：曾国藩的「日课」与 LLM 的「定时 re-grounding」

曾国藩是典型的注意力不集中型 ADHD：30 岁前浮躁坐不住，日记里天天骂自己“无恒”“浮躁”，读书“他人目下二三行，余或疾读不能终一行”。他的 harness 是「日课十二条」：黎明即起、读书不二、谨言、写日记反省——用最笨最稳的方法抵消冲动。这套系统本质上是**定时 re-grounding**：每天固定时刻重新锚定目标，与 agent 工程中“周期性采样以监控质量”（来源：AI Agent Reliability and Guardrails 2026 | Zylos Research）如出一辙。曾国藩的日记就是他的 trace log，每天反省相当于 agent 的 self-evaluation 步骤。

## 脚手架还是拐杖？边界在于「元认知参与」

ChatGPT 可以帮 ADHD 用户分解任务、生成大纲、提醒目标。Goblin Tools 能将“模糊的工作分解为更小的行动”（来源：Best AI Tools for ADHD Productivity in 2026 (Honest Review) - Iwo Szapar），降低启动门槛。Focusmate 通过虚拟身体在场提供外部结构和社会压力（来源：11 Best ADHD Productivity Apps for Fluctuating Energy - rivva blog）。这些工具都是**外部执行功能层**，补偿工作记忆不稳定、任务启动困难和注意力分散（来源：AI 与 ADHD 的专注力）。

但边界在于：用户是否在**主动参与**重锚定。如果只是让 ChatGPT 帮你写一切，你只是在用工具替代大脑，这成了拐杖。如果你用 ChatGPT 来**检查自己的目标是否漂移**——比如每完成一个子任务就问它“我原本要做什么？”——你就在训练自己的元认知。Agent 工程中也有类似区分：最简单的 agent 工作流“本身就是确定性的，遵循预定义步骤”（来源：What Are Agentic Workflows? Patterns, Memory, Use Cases, and Examples | Weaviate），而更高级的 agent 需要“计划-执行”分离来强制单向控制流（来源：Planner-Executor Agentic Framework）。ADHD 用户也需要类似的分离：让 ChatGPT 做执行调度，但自己保留计划权。

## 争议与局限：依赖风险与证据不足

依赖风险真实存在。多个工具页面仅强调益处，未充分讨论“工具本身可能增加认知负担”（来源：矛盾与存疑）。Brain.fm 虽有神经锁相技术，但在 ADHD 人群中的独立临床证据仍有限（来源：Brain.fm）。Focusmate 的核心机制是身体在场，但 AI 能否替代真实人际互动存在争议（来源：矛盾与存疑）。此外，ADHD 的异质性意味着同一工具对不同亚型效果迥异，现有研究未充分分层（来源：AI 与 ADHD 的专注力）。

同构性本身也是理论类比，并非经过验证的科学事实（来源：矛盾与存疑）。但作为思考框架，它让我们看到：ADHD 与 LLM agent 共享的困境，也许共享解法。

## 今天就能试的行动

1. **用 ChatGPT 做「目标检查员」**：每完成一个任务片段，问它“我原本要做什么？现在还在正轨吗？”让它帮你 re-ground。
2. **设置「日课」提醒**：像曾国藩一样，固定时间（如每 45 分钟）让 ChatGPT 弹出提示：“你当前的任务是 X，进度 Y%，需要调整吗？”
3. **用 RescueTime 或类似工具自动追踪时间**：不手动记录，让工具揭示你的注意力漂移模式。
4. **尝试一次 Focusmate 虚拟身体在场**：与另一个人同时工作，感受外部结构如何降低启动门槛。

ChatGPT 不是你的拐杖，是你的调度器。但调度器需要**你**来设定目标。你才是那个生成核心的主人。

# 用 Claude 给 ADHD 补上规划循环与任务分解，是脚手架还是拐杖？

> Claude 接进 ADHD 工作流：哪些交给它、哪些必须自己留着。

你坐在电脑前，文档打开了一个小时，光标闪烁。你知道该做什么——写一份方案，但大脑像卡住的齿轮，无法将“写方案”拆成第一步、第二步。你打开 Claude，输入：“帮我分解写方案的任务。”几秒后，它列出：1. 确定大纲，2. 收集数据，3. 写初稿……你终于能动了。

这种体验对 ADHD 人群来说太熟悉了。任务启动困难、不会拆解、时间盲——这些核心痛点背后，是执行功能层（调度层）的缺失。而 LLM 和 agent 工程师们也在解决类似的问题：如何让一个高产的生成核心（LLM）可靠地执行长任务？答案是 planner-executor 循环和验证 harness。

ADHD 大脑与 LLM 在结构上同构：两者都是高产的生成核心，但缺乏可靠的执行调度层（来源：AI 与 ADHD 的时间管理）。ADHD 的智力与创造力（生成核心）与 LLM 的生成能力类似，但执行功能（调度层）薄弱。AI 工具如 Motion 和 Reclaim.ai 自动规划日程，相当于提供外部调度层（来源：Motion；Reclaim.ai）。Tiimo 和 Structured 通过视觉化时间线将时间转化为可见元素，类似于 LLM 的上下文窗口管理（来源：Tiimo）。

## 规划循环：从“做什么”到“怎么做”

ADHD 侧的任务启动困难，本质是缺乏将意图转化为行动序列的规划循环。LLM/agent 侧同样面临这个问题：纯对话系统常因缺乏规划、无法执行/验证行动而失败（来源：幻觉与验证循环）。Harness 工程的关键组件之一就是“结构化工作流和规划”，要求将复杂任务分解为子任务，并在每一步进行规划与验证（来源：幻觉与验证循环）。

Motion 和 Reclaim.ai 就是这样的外部规划器。Motion 自动根据任务、会议和截止日期创建并动态调整每日计划，消除“下一步该做什么”的决策负担（来源：Motion）。Reclaim.ai 则通过创建深度工作、习惯和专注时间的智能块，自动防御会议侵占，帮助维持日程稳定性（来源：Reclaim.ai）。对于 ADHD 用户，这些工具相当于一个永不疲倦的调度员，不断重新规划，补偿执行功能的波动。

但这里有一个关键区别：Motion 和 Reclaim.ai 的规划是自动的、黑箱的，用户只需输入任务和截止日期。而 LLM harness 工程强调“计划-执行”分离，并采用验证循环来确保每一步的正确性（来源：幻觉与验证循环）。ADHD 用户如果只是被动接受 AI 的规划，可能会失去主动规划的能力——这正是“脚手架 vs 拐杖”的边界。

## 任务分解：从“大象”到“小块”

不会拆解是 ADHD 的另一大痛点。面对一个复杂项目，大脑直接宕机。Goblin Tools 的“任务分解”功能（来源：矛盾与存疑）正是为此设计：将“写论文”拆成“查找资料”“写引言”“做实验”等子任务。这与 agent 工程中的“任务分解”同构：复杂的 harness 不会盲目执行工具，而是将复杂任务分解为子任务，并在每一步进行规划与验证（来源：幻觉与验证循环）。

但任务分解的质量取决于上下文。ADHD 用户的工作记忆缺陷与 LLM 的无状态性同构（来源：AI 与 ADHD 的时间管理）。Todoist 等工具通过智能提醒和优先级排序，充当数字工作记忆，弥补上下文丢失（来源：AI 与 ADHD 的时间管理）。然而，如果用户完全依赖工具来记忆和分解，而不练习自己的拆解能力，那么工具就成了拐杖。

## 验证循环：对抗“幻觉”的关键

ADHD 大脑的“生成核心”高产但缺乏可靠的执行调度层，容易产生冲动、想当然的判断——即行为层面的“幻觉”（来源：幻觉与验证循环）。例如，在情绪调节中，ADHD 个体可能脱离环境线索做出不恰当反应。而 LLM 作为生成核心，天然倾向于产生自信但可能错误的输出（幻觉）。Harness 工程正是为此设计：它提供验证循环，检查输出格式或对模型编写的代码运行测试用例（来源：幻觉与验证循环）。

对于 ADHD 用户，验证循环意味着在行动前增加一个“检查”步骤。例如，使用 Claude 生成任务列表后，自己再逐条核对：“这个子任务真的可行吗？时间估计合理吗？”这个外部验证步骤可以校正认知偏差，类似于精神分裂症研究中提到的“现实检验和自我监控”（来源：幻觉与验证循环）。

当前多数 ADHD 工具（如 Motion、Reclaim.ai）缺乏显式的验证循环。它们自动排程，但用户很少质疑 AI 的决策。这可能导致“更多幻觉和重复工作”（来源：幻觉与验证循环）。一个更好的设计是让工具提示用户验证关键步骤，或者像 agent harness 那样，在规划后执行自我批评提示（来源：幻觉与验证循环）。

## 脚手架 vs 拐杖：边界在哪里？

核心判断：**AI 工具作为外部执行功能层，是有效的脚手架，但若设计为长期依赖且缺乏撤除机制，则沦为拐杖**。

证据方面，现有资料一致认为 Motion 等工具能提升 ADHD 用户的生产力，但缺乏独立的临床研究证据（来源：Motion）。多巴胺干预的效果存在争议（来源：矛盾与存疑），且多数工具（如 Goblin Tools、Saner.AI）设计为长期使用，未提及撤除机制（来源：矛盾与存疑）。

脚手架应设计为可逐步撤除的临时支撑（来源：矛盾与存疑）。例如，使用 Tiimo 的视觉时间表一段时间后，用户应能逐渐内化时间感知能力。但当前工具缺乏这种设计。

## 今天就能试的 3 件事

1. **用 Claude 做任务分解，但加上验证循环**：让 Claude 将你的一个项目拆成子任务，然后自己逐条核对：“这个子任务是否具体？是否可执行？时间估计是否合理？”修改后再执行。
2. **设置“保护时间块”**：使用 Reclaim.ai 或手动在日历上创建 2 小时的深度工作块，并拒绝所有会议。这直接补偿时间盲和执行功能缺陷（来源：Reclaim.ai）。
3. **尝试视觉化时间表**：下载 Tiimo 或 Structured，将一天的活动用颜色块可视化。这能提升时间感知，直接应对时间盲（来源：Tiimo）。

## 诚实面对局限

同构命题本身是理论模型，需更多实证验证（来源：矛盾与存疑）。多数工具的 AI 功能细节不明确，缺乏严谨的随机对照试验支持（来源：AI 与 ADHD 的时间管理）。个体差异大：不同 ADHD 亚型对工具的反应可能不同（来源：AI 与 ADHD 的时间管理）。过度依赖外部工具可能阻碍内在执行功能的发展（来源：AI 与 ADHD 的时间管理）。

最终，AI 工具是脚手架还是拐杖，取决于你如何使用。记住：工具是为了让你最终能自己走，而不是永远依赖它。

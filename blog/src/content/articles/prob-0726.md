# 用 ChatGPT 给 ADHD 补上工具使用与认知卸载，真的能像 agent 那样工作吗？

> 把 ChatGPT 当作 ADHD 的外部执行层来实测：它补的是哪一块短板？

你打开 ChatGPT，丢进去一句“帮我规划下周工作”，它噼里啪啦输出一份待办清单。你关掉窗口，感觉好了一点，但明天醒来，你依然不知道第一件事该做什么。

这不是 ChatGPT 的错。问题在于，你把它当成了“生成答案的机器”，而不是“执行调度的引擎”。

ADHD 大脑与 LLM 有着惊人的同构：两者都是“高产但缺执行调度层的生成核心”（来源：ADHD 的 AI 工具全景）。ADHD 不擅长精确计算、重复操作、维护状态——这些正是 Agent 通过 function calling 外包给工具的能力。那么，把 ChatGPT 当成 ADHD 的“外部执行层”来用，真的能像 Agent 那样工作吗？答案是：**能，但前提是你得把它当成一个需要你主动设计 harness 的系统，而不是一个替你思考的拐杖。**

## 同构：ADHD 的短板 = LLM 的弱点

ADHD 的执行功能障碍包括任务启动困难、工作记忆易丢失、时间感知模糊（来源：执行功能障碍）。LLM 同样有“强记忆、弱控制”的剖面——它能记住大量信息，但上下文控制薄弱，容易被无关信息带偏（来源：执行功能）。在经典 Stroop 冲突任务中，Transformer 的注意力随序列变长而骤降，无法抑制优势反应——这和 ADHD 在冲突任务中的表现如出一辙（来源：执行功能）。

这意味着，ADHD 和 LLM 需要的不是更聪明的核心，而是更好的“外部调度层”。对于 LLM，这个调度层是 Agent 框架（如 function calling、工具调用）；对于 ADHD，这个调度层就是精心设计的 AI 交互流程。

## 实测：ChatGPT 补的是哪一块短板？

### 1. 精确计算：从“大概”到“具体”

ADHD 大脑擅长“模糊的大图”，但一到具体数字、步骤就卡壳。ChatGPT 的 function calling 能力（通过代码解释器或插件）可以执行精确计算。例如，输入“我每天有 3 小时空闲，分给 5 个项目，每个项目至少 20 分钟，帮我排一个周计划”，ChatGPT 能调用工具算出可行方案，而不是只给建议。这直接补偿了 ADHD 在精确规划上的短板（来源：ADHD 的 AI 工具全景）。

### 2. 重复操作：自动化“下一次”

ADHD 讨厌重复——每天记同样的事、填同样的表、检查同样的清单。ChatGPT 的对话记忆和自定义 GPT 可以记住你的模板，一键生成重复内容。比如，一个“每日复盘”GPT，自动问你三个问题，然后把答案存入指定笔记。这相当于 Agent 里的“循环调用”，把重复劳动外包给 AI（来源：Saner.AI 的知识回忆功能也类似，减少标签切换）。

### 3. 状态维护：别让我“记住”

ADHD 的工作记忆像漏水的桶——刚想做的事，转身就忘。ChatGPT 作为外部记忆，可以维持对话状态。例如，你告诉它“我正在写一篇关于 ADHD 的文章，帮我保存所有相关的点子”，之后每次提问，它都能基于上下文回答。这类似 Mem 的“AI 第二大脑”或 Saner.AI 的本地记忆，本质上是为 ADHD 提供持久化上下文（来源：ADHD 的 AI 工具全景）。

## 但这不是魔法：脚手架 vs 拐杖

Goblin Tools 能把“整理房间”分解成 10 个小步骤，用户反馈启动焦虑降低（来源：Goblin Tools）。Lex 允许单一指令触发多步骤流程，减少决策疲劳（来源：Lex）。这些工具的设计理念是“脚手架”——帮你搭建结构，但最终你得自己走。

然而，矛盾在于：过度依赖可能削弱内在能力（来源：矛盾与存疑）。如果每次启动任务都靠 AI 分解，你的大脑可能越来越不擅长自己拆分。同样，如果 Agent 工程师完全依赖 function calling 而不理解底层逻辑，系统也会变得脆弱。

真正的关键在于：**AI 工具应作为脚手架促进能力发展，而非永久拐杖**（来源：ADHD 的 AI 工具全景）。这意味着，使用 ChatGPT 时，你要主动设计交互流程，而不是被动接受输出。

## 如何像 Agent 工程师一样使用 ChatGPT

Agent 工程师设计 harness 时，会明确：什么任务需要精确计算（外包给工具），什么任务需要生成（留给模型），以及状态如何传递。ADHD 用户也可以这样做：

1. **定义你的“function calling”**：列出你经常卡壳的精确任务（如预算计算、排程、文本格式化），专门为它们创建 ChatGPT 的“技能”或自定义指令。

2. **设计状态管理**：用一个固定的笔记或 ChatGPT 对话来维护当前项目状态。每次更新时，告诉 ChatGPT“更新状态：X 已完成，Y 等待反馈”。这样你就不需要靠大脑记住。

3. **建立启动脚本**：像 Lex 那样，设计一个“早晨启动”指令，让 ChatGPT 一次执行多个步骤：检查日历、列出今日三件要事、生成待办清单。这降低了启动门槛。

## 局限与诚实

必须承认，现有证据主要来自用户报告，缺乏大规模随机对照试验（来源：矛盾与存疑）。个体差异很大——有人觉得 Goblin Tools 救命，有人觉得太机械。此外，ChatGPT 的 function calling 并非完美，有时会误解指令或产生幻觉。

但同构视角提供了一个有力的框架：**ADHD 和 LLM 共享同一个核心问题——生成能力强，执行调度弱。** 因此，AI 工具作为外部执行层，在理论上和实践中都具备补偿潜力。关键在于，使用者要像工程师一样，主动设计 harness，而不是被动等待 AI 拯救。

## 今天就能试的行动

1. **创建一个“任务分解 GPT”**：在 ChatGPT 中自定义一个 GPT，输入“请将以下任务分解为不超过 5 分钟的小步骤”，然后每次遇到大任务就丢给它。

2. **设置“状态同步”提醒**：每天固定时间（如午休前），用一条指令让 ChatGPT 总结当前项目进展，并保存到笔记应用。

3. **尝试一次“单一指令多步骤”**：写下你明天早上要做的 3 件事，让 ChatGPT 生成一个包含时间、顺序和检查点的执行脚本。

4. **记录你的“卡壳模式”**：本周注意哪些任务让你反复拖延，总结出 3 个模式，然后设计对应的 ChatGPT 指令（例如“每次我要写周报时，先让 ChatGPT 列出上周完成事项”）。

把 ChatGPT 当作你的 Agent 框架，你既是开发者也是用户。同构意味着，你不需要变得更“正常”，只需要学会给自己搭建 harness。
